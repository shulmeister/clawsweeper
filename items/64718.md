## Summary

Keep open. Current main still lacks the PR’s sender-aware auto-reply exec hardening: exec overrides are resolved without `senderIsOwner`, the auto-reply call site does not pass owner status, and non-owner chat turns can still inherit permissive host exec defaults when no explicit exec override is set. The PR is not obsolete, but it should be revised before merge to address the remaining review concern about partial `agentExecDefaults` still falling through to permissive downstream defaults.

## Best Possible Solution

Keep this PR open for maintainer/security review. The implementation path should pass `command.senderIsOwner` into `resolveReplyExecOverrides`, make non-owner auto-reply defaults fail closed unless an explicit non-owner-safe operator policy is configured, merge or require safe `security`/`ask` when agent defaults are partial, ignore user-injectable inline/session exec overrides for non-owner senders, and add focused unit coverage for those cases before merge.
