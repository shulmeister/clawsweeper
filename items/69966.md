## Summary

Keep #69966 open. Current main still treats the bootstrap profile knobs as global `agents.defaults` settings: `agents.list[]` does not accept `contextInjection`, `bootstrapMaxChars`, `bootstrapTotalMaxChars`, or named profile references; the embedded and CLI runtime paths resolve bootstrap budgets from global defaults only; and the Control UI presets patch `agents.defaults` only. The issue is recent, concrete, unprotected by maintainer labels, and not covered by the related per-agent context/compaction reports.

## Best Possible Solution

Add first-class per-agent bootstrap profile support in core config rather than closing this. A conservative path is to extend `AgentConfig`/`AgentEntrySchema` with per-agent `contextInjection`, `bootstrapMaxChars`, and `bootstrapTotalMaxChars` (or a clearly designed named-profile indirection), resolve those through `resolveAgentConfig` using the session agent id, update embedded and CLI bootstrap callers, then update Control UI presets/docs/tests so global defaults still work and agent overrides inherit cleanly.
