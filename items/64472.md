## Summary

Keep open. Current main still enforces the workspace root as the canonical boundary for bootstrap-file reads, so a workspace symlink resolving to a sibling such as ~/.openclaw/kb remains rejected and then reported as missing. The linked fix PR #64473 is still open, and the earlier similar PR #51277 was closed unmerged in the provided context.

## Best Possible Solution

Keep the issue open and review PR #64473 or a safer maintainer-authored variant. The fix belongs near src/infra/boundary-path.ts and src/agents/workspace.ts with regression coverage for workspace bootstrap files symlinked to an allowed OpenClaw-owned location, plus negative tests for arbitrary paths outside the trusted boundary.
