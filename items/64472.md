## Summary

Keep open. Current main still routes workspace bootstrap files through the boundary reader with the workspace directory as the root, and the boundary resolver still rejects symlink targets that resolve outside that root. The provided GitHub context also shows a same-author open PR, #64473, using closing syntax for this issue, so the issue should remain open until that PR is reviewed, landed, or rejected.

## Best Possible Solution

Keep this issue open and review #64473 as the active implementation candidate. If maintainers accept `~/.openclaw` sibling targets as trusted for workspace bootstrap reads, the fix should update `src/infra/boundary-path.ts` with focused regression coverage for `workspace/AGENTS.md -> ~/.openclaw/kb/AGENTS.md` while preserving rejection of arbitrary outside targets. If that trust-boundary relaxation is rejected, close or revise the PR with a clear security rationale and, if appropriate, point the remaining broader symlink behavior to the existing related workspace-symlink thread.
