## Summary

Current main already implements Telegram completion feedback through the shipped lifecycle status-reactions path. It covers queued/done/error reactions, allowed-reaction fallback, removeAckAfterReply handling, and error/fallback classification, so this PR’s direct ACK-replacement patch is obsolete.

## Best Possible Solution

Close this PR and keep the shipped Telegram status-reactions implementation. If maintainers want 👎 specifically as the default failure emoji, that should be a small follow-up against the existing messages.statusReactions emoji configuration/defaults, not this direct ACK replacement patch.
