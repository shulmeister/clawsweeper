## Summary

Keep open. Current main still routes chat assistant text through the Control UI markdown renderer, but that renderer only uses markdown-it plus task-list handling and sanitization; there is no KaTeX, MathJax, or math-delimiter renderer/dependency on main. This is also a core WebUI rendering request, not something I would close as ClawHub without an existing plugin surface or linked plugin that can alter chat markdown rendering.
