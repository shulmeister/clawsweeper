## Summary

Current `main` already provides a CLI cleanup path for stale plugin config entries via `openclaw doctor --fix`, with repair logic, tests, docs, and a shipped changelog entry. The exact requested subcommand name (`openclaw plugins clean`) is not present, but the requested capability is already implemented well enough in core.

## Evidence

- **Stale plugin repair implementation:** `maybeRepairStalePluginConfig` scans and removes stale `plugins.allow` and `plugins.entries` ids from config.
  - file: [src/commands/doctor/shared/stale-plugin-config.ts:120](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/commands/doctor/shared/stale-plugin-config.ts#L120)
- **CLI repair wiring:** The doctor repair sequence applies stale-plugin cleanup as part of `openclaw doctor --fix` / `--repair`.
  - file: [src/commands/doctor/repair-sequencing.ts:58](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/commands/doctor/repair-sequencing.ts#L58)
- **Discoverable fix hint coverage:** Regression tests assert stale plugin warnings include `Run "openclaw doctor --fix"` for cleanup.
  - file: [src/commands/doctor/shared/preview-warnings.test.ts:228](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/commands/doctor/shared/preview-warnings.test.ts#L228)

  - file: [src/commands/doctor/shared/stale-plugin-config.test.ts:63](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/commands/doctor/shared/stale-plugin-config.test.ts#L63)

## Risks / Open Questions

- The shipped UX is `openclaw doctor --fix`, not the exact proposed `openclaw plugins clean --dry-run` surface.
- `src/config/validation.ts:1022` and `src/config/validation.ts:1029` still tell users to remove stale entries from plugin config manually; that discoverability gap is separate from whether a cleanup command exists.
