## Summary

Keep #68180 open. Current main still lacks the Linux fnm `aliases/default/bin` resolver path that #68169 reports, so the PR is not obsolete. The PR does need maintainer cleanup before merge because it bundles unrelated fixes and lacks Linux regression assertions for the daemon PATH change.

## Best Possible Solution

Keep the PR open, but narrow or cherry-pick the daemon change into `src/daemon/service-env.ts`, add focused assertions in `src/daemon/service-env.test.ts` for Linux `FNM_DIR/aliases/default/bin` and `${HOME}/.fnm/aliases/default/bin`, and handle the unrelated Feishu/Lark, ACP, Telegram, and Slack commits separately. Once that lands, close linked issue #68169 as fixed.
