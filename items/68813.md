## Summary

PR #68813 is superseded by the shipped Comfy config-path fix from #63058. Current main uses and validates `plugins.entries.comfy.config` for Comfy workflow configuration, documents that path publicly, and shipped the fix in v2026.4.23; the remaining `providerConfigSchema` idea is a separate manifest/API design question rather than needed cleanup for this PR.

## Best Possible Solution

Close #68813 as superseded by #63058/v2026.4.23. Keep the shipped Comfy contract on `plugins.entries.comfy.config`, and only revisit `providerConfigSchema` through a separate maintainer-scoped plugin API design if generic provider-owned `models.providers.*` validation is still wanted.
