## Summary

PR #68813 is superseded by the shipped Comfy config resolution on current main. The PR proposed a new providerConfigSchema path for validating models.providers.comfy, but main now documents and validates Comfy workflow config under plugins.entries.comfy.config, the runtime reads that plugin config first, and the changelog records the Comfy validation fix as shipped via #63058 in 2026.4.23. The linked #66784 is already closed in the provided context, so the remaining generic provider-ownership/providerConfigSchema idea should be handled only as a separate maintainer-designed follow-up if still desired.

## Best Possible Solution

Close #68813 as superseded. Keep the shipped Comfy configuration contract at plugins.entries.comfy.config, rely on the current manifest configSchema validation path, and track any future generic provider-owned model-provider schema API in a separate maintainer-scoped design issue or PR.
