## Summary

Keep PR #42174 open. Current main still rejects Windows drive-letter Docker sandbox bind sources at config validation and runtime security validation, so the reported user-visible bug is not implemented on main. The PR is not merge-ready as-is because the reviewed surface is security-sensitive and prior bot review raised allowlist/canonicalization concerns, but the underlying bug remains real and focused.

## Best Possible Solution

Keep the item open and replace or revise the PR with a tested Windows bind-path implementation: reuse `splitSandboxBindSpec` in config validation, accept `^[A-Za-z]:[\\/]` sources in runtime validation, normalize drive-letter case without rewriting path segments, resolve Windows paths through existing ancestors where feasible, enforce `bindSourceRoots` for Windows paths, and add regression tests for uppercase/lowercase drive letters, binds with and without mode, reserved targets, outside-root rejection, and symlink/junction-style escape handling.
