## Summary

Keep open. Current `main` does not already implement this PR: Feishu DM config still lacks `dms.*.tools`/`toolsBySender`, direct-session tool policy resolution still goes through group-only paths, and the empty sender-override fallback fix is still absent.

## Evidence

- **Feishu DM schema still lacks tool-policy fields:** `DmConfigSchema` only contains `enabled` and `systemPrompt`; there is no `tools` or `toolsBySender` support on current `main`. An exact-path search for `channels.feishu.(accounts.*.)?dms.*.(tools|toolsBySender)` returned no matches.
  - file: [extensions/feishu/src/config-schema.ts:32](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/config-schema.ts#L32)
  - command: `rg -n "channels\.feishu\.(accounts\.\*\.)?dms\.\*\.(tools|toolsBySender)" docs src extensions`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Core resolver is still group-oriented:** `resolveGroupToolPolicy` derives `groupIds` and then delegates to `plugin?.groups?.resolveToolPolicy` and `resolveChannelGroupToolsPolicy`; there is no DM/direct-specific resolver in this file on current `main`.
  - file: [src/agents/pi-tools.policy.ts:367](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/pi-tools.policy.ts#L367)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Legacy session-key parsing still rejects non-group kinds:** In the fallback parser path, only `group` and `channel` kinds are accepted; `direct`/`dm` are not handled there.
  - file: [src/agents/pi-tools.policy.ts:225](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/pi-tools.policy.ts#L225)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Empty sender override still short-circuits fallback:** `resolveChannelGroupToolsPolicy` still returns any truthy `groupSenderPolicy` / `defaultSenderPolicy` directly. Because `{}` is truthy, the empty-override fallthrough fix proposed by the PR is not present on current `main`.
  - file: [src/config/group-policy.ts:426](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/config/group-policy.ts#L426)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Feishu plugin resolver still reads group tools only:** The Feishu channel adapter's resolver returns `groupConfig?.tools` only; it does not inspect DM config or sender-scoped DM policies.
  - file: [extensions/feishu/src/policy.ts:151](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/policy.ts#L151)
