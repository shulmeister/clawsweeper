## Summary

Keep open. Current main still flattens the Mattermost inbound system-event preview with `/\s+/g`, and there is no Mattermost config/docs/test evidence of an optional newline-preserving preview mode. The adjacent Markdown ingestion issue was addressed separately: the full inbound body preserves newlines, but this issue specifically asks for an opt-in faithful preview string.

## Best Possible Solution

Keep the issue open until Mattermost gains a default-preserving opt-in for faithful system-event previews, likely in the Mattermost plugin config schema and docs, with a focused monitor test proving multi-line input can enqueue a newline-preserving preview while compact one-line preview remains the default.
