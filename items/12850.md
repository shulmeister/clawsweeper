## Summary

Keep open. Current `main` still flattens Mattermost inbound system-event previews with `/\s+/g` before enqueueing the event, while the full agent-facing body remains newline-preserving. I found no Mattermost config/schema/docs surface for an opt-in faithful preview mode, and the related PR/discussion continues to identify #12850 as the remaining feature request rather than already-shipped behavior.

## Best Possible Solution

Keep #12850 open. Implement this in the bundled Mattermost plugin as an opt-in top-level/account setting that preserves newlines for inbound system-event previews while keeping the current compact one-line default. Add focused Mattermost tests for default flattening and opt-in newline-preserving previews, and document the setting on `docs.openclaw.ai/channels/mattermost`.
