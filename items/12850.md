## Summary

Keep open. Current main still computes the Mattermost inbound system-event preview with `bodyText.replace(/\s+/g, " ").slice(0, 160)`, so the specific requested optional newline-preserving preview mode is not implemented. The related Markdown/newline ingestion work appears to be addressed separately, but this issue is about the compact preview string and that remaining behavior is still present.
