## Summary

Keep open. Current main still flattens the Mattermost inbound system-event preview with `/\s+/g`, while the full agent-visible body is preserved separately. I found no Mattermost config, docs, or tests for an opt-in newline-preserving preview mode, so the requested feature remains unimplemented. The related Markdown/newline ingestion work is adjacent, not a completion of this issue.

## Best Possible Solution

Keep the issue open. Add a Mattermost plugin config setting, with top-level and per-account support, that defaults to the current compact one-line preview but can opt into a faithful preview helper that preserves `\n`, normalizes spaces/tabs within lines, and caps by chars/lines. Document it in the public Mattermost channel docs and add focused monitor tests proving both default flattening and opt-in newline-preserving previews.
