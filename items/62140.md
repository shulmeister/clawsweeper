## Summary

Keep open. Current main has partial SecretRef/token-handling improvements, but the issue's core request is not fully implemented: service install still has an already-loaded fast path that can skip refresh, and the service install plan still treats state-dir `.env` values as durable service metadata without a SecretRef-specific exclusion for `OPENCLAW_GATEWAY_TOKEN`. The related #70752 is useful and narrower, but it remains open and does not prove this broader post-update recovery item is resolved.

## Evidence

- **Loaded gateway install still fast-returns:** When the gateway service is loaded and `--force` is not set, `runDaemonInstall` only refreshes for the Node extra CA case; otherwise it emits `already-installed` and returns. There is no equivalent refresh check for an embedded `OPENCLAW_GATEWAY_TOKEN`.
