## Summary

Keep open. Current main has a good manifest-backed fix path for command aliases, but the issue is specifically about external lossless-claw/lcm ownership, and current core still falls back to the old misleading plugins.allow guidance when alias ownership is not known. A maintainer also explicitly left this open on current main-adjacent code and pointed to #64732 as related, not canonical closure.

## Best Possible Solution

Keep #65319 open until a maintainer or lossless-claw owner verifies the external plugin publishes manifest commandAliases for lcm and /lossless in the current OpenClaw shape, or until core changes the unresolved-command fallback so it cannot recommend command tokens as plugins.allow entries for plugin-owned surfaces. If #64732 becomes the accepted canonical core fallback issue, then close this manually as a duplicate with that maintainer context.
