## Summary

Keep this PR open. Current main still uses the fragile `[^)]*` parser for `Exec denied (...)` and `Exec finished (...)`, while the gateway can emit nested denial metadata such as `approval-timeout (allowlist-miss)`. That still misroutes denied async exec followups into the generic completion path. The related #72148 is adjacent resume-fallback wording work, not a parser replacement.

## Best Possible Solution

Keep this PR open and review or land the focused parser fix. The best product/code path is to replace the denied/finished metadata regex extraction with balanced parsing, keep the nested-parentheses parser and followup-routing regression tests, and handle #72148 as separate resume-fallback wording/logging work.
