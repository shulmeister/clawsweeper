## Summary

Keep open. Current main does not contain the warning/flag from this PR: the context-window guard only returns low-window/block flags, runtime setup only logs the existing low-context warning, and Bedrock discovery still bakes fallback contextWindow values into model metadata. The PR is a plausible implementation candidate, but it needs revision because the review discussion correctly identifies that a source === "default" warning alone would not cover the Bedrock fallback case without the later optional-context-window/discovery change, which is not present on main.

## Best Possible Solution

Keep the PR open and ask for changes. The best path is a source-aware implementation that distinguishes true provider/discovery metadata from hardcoded fallback values, likely by carrying optional contextWindow or metadata-source information through Bedrock discovery and model config, then warning in resolveEffectiveRuntimeModel with targeted tests for both generic default resolution and Bedrock fallback discovery.
