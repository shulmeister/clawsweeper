## Summary

Keep open. Current main still has the exact discoverability gap reported in #67060: provider request proxying is explicit and documented, but gateway startup does not warn when HTTP_PROXY/HTTPS_PROXY is present and remote providers lack request.proxy configuration. The linked PR #68023 attempted the right shape of fix but was closed unmerged, and its warning/tests are not present on current main.

## Best Possible Solution

Implement the non-breaking gateway startup warning proposed in #67060: detect HTTP_PROXY/HTTPS_PROXY variants, identify configured remote model providers without explicit request.proxy, emit one actionable warning from the startup logging path, and add focused tests for remote provider warning, configured env-proxy suppression, no proxy env suppression, and local-provider suppression. PR #68023 is a useful prior attempt to reference, but current main still needs the change.
