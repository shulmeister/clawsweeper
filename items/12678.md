## Summary

Keep open. Current main has useful narrower controls, including tool allow/deny policy, gateway HTTP default-deny for selected dangerous tools, and plugin hooks that can block or require approval at call time. It does not yet implement the requested manifest-declared, capability-based permission model with default-deny grants/revocations and audit semantics. The linked @vainplex/openclaw-governance plugin is relevant prior art, but the issue discussion itself notes that true default-deny/invisible-tool enforcement would need API-layer support.
