## Summary

Remote ACPX execution on paired nodes is a useful capability, but current main already exposes the plugin/runtime seams needed for it and OpenClaw scope keeps optional runtime integrations out of core. The issue has no protected label or maintainer author signal, and the related remote-runtime PR context already points this shape toward an external plugin/ClawHub path rather than core special-casing `tools.exec.host=node`.

## Risks / Open Questions
