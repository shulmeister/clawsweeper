## Summary

Current main still lacks the requested generic model rate limiter. OpenClaw has adjacent reactive handling for 429s, Retry-After, auth-profile cooldown/rotation, model fallback, and docs for those paths, but no documented config or scheduler that enforces per-model tokens-per-interval and requests-per-interval before model calls.

## Best Possible Solution

Keep #30828 open for a core model request-path design: add a documented per-provider/per-model, and possibly per-auth-profile, schema for request and token budgets with interval windows; enforce it before shared model calls; reconcile actual usage after responses/streams; optionally ingest provider headers such as OpenAI x-ratelimit-*; and add focused tests/docs covering scheduler, fallback, retry, streaming, and auth-profile interactions.
