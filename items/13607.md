## Summary

Keep #13607 open. Current main still implements only per-agent `agents.list[].reasoningDefault`, while the requested global `agents.defaults.reasoningDefault` is absent from the config type, strict schema, generated schema/help surface, docs, and main reply resolution path. The open related PR #71817 is explicitly partial for Telegram/Feishu per-agent behavior and does not supersede the global default request.

## Best Possible Solution

Keep #13607 open until OpenClaw implements a schema-backed global `agents.defaults.reasoningDefault` with clear precedence such as inline `/reasoning` directive and session override > per-agent default > global default > model-capability/off fallback. The implementation should update config types, zod schema, generated schema/help/docs/status behavior, reconcile the gateway-tool allowlist, and add regression coverage for the main reply path plus relevant channel adapters. Treat #71817 as a useful partial per-agent Telegram/Feishu follow-up, not as the canonical global-default fix.
