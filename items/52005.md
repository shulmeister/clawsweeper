## Summary

Current `main` does not show a strong implemented-on-main fix for this report. The Discord bound-thread suppression path on `main` is still keyed to explicit webhook identity, inbound replay dedupe is still keyed to message ID, and the Discord process path still unconditionally wires typing for routed replies. That means the reported original+PluralKit-proxy double-handle shape remains plausible, especially when proxied copies do not collapse onto the same webhook identity early enough.

## Evidence

- **Bound-thread suppression is still webhook-id based:** `shouldIgnoreBoundThreadWebhookMessage` only drops when the inbound `webhookId` matches the bound thread webhook id, or when a recently-unbound thread saw that same webhook id again. There is no broader content-clone or PluralKit-specific collapse in this path.

  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Inbound dedupe is still message-id based:** `buildDiscordInboundReplayKey` uses only `accountId`, resolved `channelId`, and `message.id`. Original and proxied copies with different Discord message ids would not dedupe here.
  - file: [extensions/discord/src/monitor/inbound-dedupe.ts:22](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/extensions/discord/src/monitor/inbound-dedupe.ts#L22)

- **Typing path has no PluralKit/bound-thread exception:** The Discord reply pipeline still always starts typing via `sendTyping` for the resolved channel target. I did not find a PluralKit-specific or bound-thread duplicate-pressure suppression branch in the process path.
  - file: [extensions/discord/src/monitor/message-handler.process.ts:558](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/extensions/discord/src/monitor/message-handler.process.ts#L558)

  - file: [extensions/discord/src/monitor/message-handler.preflight.test.ts:452](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/extensions/discord/src/monitor/message-handler.preflight.test.ts#L452)
