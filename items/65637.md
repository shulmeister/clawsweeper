## Summary

Keep this PR open. Current main and the latest release still serialize the raw parsed config for both full and path-specific `/config show` replies, while the existing gateway `config.get` path already uses the schema-aware redaction pipeline. The linked bug #65623 remains open and this PR is the paired implementation candidate, though the bot review correctly flags a test-quality fix before landing.

## Best Possible Solution

Keep this PR open, address the review comment by making the mocked schema hints explicitly sensitive, then review and land this PR or apply the same small fix directly. The fix belongs in `src/auto-reply/reply/commands-config.ts` so `/config show` uses the existing schema-aware config redaction pipeline for both full-config and path-specific replies, with regression coverage in `src/auto-reply/reply/commands-gating.test.ts`.
