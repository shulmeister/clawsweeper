## Summary

Keep #70588 open. Current main does not expose a Discord `webhook-create` message-tool action: the shared action enum lacks it, Discord discovery advertises channel/thread/admin actions but not webhooks, and the Discord message-action handler/runtime would reject it. There is private Discord webhook creation for thread bindings, but it is not a general agent-callable action and does not return the requested webhook URL.

## Best Possible Solution

Keep the issue open and implement this in the bundled Discord channel plugin if maintainers want the capability. The likely path is to add `webhook-create` to the shared message-action vocabulary and target-mode plumbing, advertise it from `extensions/discord/src/channel-actions.ts`, add Discord runtime handling that calls `Routes.channelWebhooks`, accept target channel plus optional name/avatar, return the generated URL carefully, add an explicit Discord action gate, and cover it with Discord action/schema/runtime tests plus docs.
