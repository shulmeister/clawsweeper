## Summary

Keep open. Current main has strong adjacent failover behavior: rate-limit classification, auth-profile rotation, model-scoped cooldowns, ordered model fallback, same-provider cooldown probes, and cross-provider fallback. It still does not implement the issue's requested dynamic model-tier policy, health/metrics state, round_robin strategy, proactive header handling, or gradual upgrade-back controller. The aashari fork linked in the issue is useful prior art, but that behavior is not shipped upstream.

## Best Possible Solution

Keep this issue open for maintainer product/design work in OpenClaw's core model fallback and auth-routing layer. A good implementation would add a documented routing policy such as failover versus round-robin/health-aware routing, per-provider or per-model health state, observable downgrade and recovery decisions, conservative primary recovery probes, config/schema/docs coverage, and regression tests. The aashari fork's round_robin/model-health behavior is useful prior art but should not be treated as shipped upstream behavior.
