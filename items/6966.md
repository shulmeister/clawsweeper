## Summary

Keep open. Current main has strong adjacent failover work, including rate-limit classification, model-scoped auth cooldowns, same-provider fallback probes, and ordered model fallback, but it still does not implement the issue’s requested dynamic model-tier policy, metrics/health strategy, or the aashari fork’s round_robin/model-health behavior.

## Best Possible Solution

Keep this issue open for maintainer product/design work in core model fallback and auth state. The likely path is a documented policy such as failover vs round-robin/health-aware routing, per-provider/model-tier cooldown or health tracking, observable downgrade/upgrade decisions, conservative primary recovery probes, and config/docs/tests. The aashari fork’s round_robin/model-health implementation is useful prior art but should not be treated as shipped upstream behavior.
