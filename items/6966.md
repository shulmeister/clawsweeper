## Summary

Keep open. Current main has strong adjacent failover behavior, including rate-limit classification, auth-profile rotation, model-scoped cooldown metadata, ordered fallback candidates, cooldown probes, and documentation/tests for those paths. It still does not implement the issue’s requested dynamic model-tier switching policy: no strategy/round_robin config, no model-health state, no metrics/header-driven downgrade controller, and no gradual upgrade-back policy. The aashari fork linked in the issue is useful prior art, but that implementation is external and not shipped in upstream main.

## Best Possible Solution

Keep this issue open for maintainer product/design work in OpenClaw’s core model fallback and auth-routing layer. A good implementation would add a documented routing policy such as failover versus round-robin/health-aware routing, per-provider/model/profile health state, mark-success/mark-failure recovery semantics, conservative primary recovery probes, observable downgrade and upgrade-back events, config/schema/docs coverage, and regression tests. The aashari fork’s `round_robin` and model-health behavior is useful prior art but should not be treated as shipped upstream behavior.
