## Summary

Keep open. Current main has ordered model failover, auth-profile rotation, model-scoped cooldowns, and cooldown probing, but it still does not implement the requested dynamic model-tier switching policy with health-aware routing, downgrade thresholds, round-robin strategy, or gradual upgrade-back behavior. The aashari fork linked in the discussion is useful external prior art, not upstream shipped behavior.

## Best Possible Solution

Keep this issue open for core model-routing design. A good upstream solution would define a documented routing policy such as failover versus health-aware/round-robin, add per-provider/model/profile health state with success/failure recovery semantics, expose observable downgrade and upgrade-back events, update config schema/docs, and add regression tests. Treat the aashari fork linked in the discussion as prior art, not as shipped upstream behavior.
