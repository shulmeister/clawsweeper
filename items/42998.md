## Summary

Keep #42998 open. Current main has made the duplicate-rerun portion mostly obsolete by treating empty cron output as non-interim, including in the latest release tree, but it does not implement the PR's remaining raw-stdout recovery behavior: cron output is still derived from run payloads/final assistant text, not from the latest successful transcript toolResult. The PR also has unresolved review feedback about gating the retry skip on recovered tool-result text and test timing fragility, so the useful path is to rework it against the current `run-executor.ts` split rather than close it as fully implemented.

## Best Possible Solution

Keep the PR open or ask the contributor to rework it: preserve current main's empty-output non-retry behavior, add a structural transcript/tool-result recovery path for successful raw-stdout cron runs, gate any retry suppression on actual recovered tool-result text rather than prompt keywords alone, and add a deterministic regression test against the current `run-executor.ts`/`run.ts` split.
