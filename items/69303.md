## Summary

Keep open. Current main still releases the inbound dedupe claim on every dispatch catch path without stamping the dedupe cache, and the existing regression test explicitly expects the same MessageSid to retry after a dispatch failure. The provided GitHub context shows #72291 and #71819 are open PRs for this exact poisoned-dedupe fix, with #72291 maintainer-labeled, so the issue should remain open until an equivalent fix lands on main.

## Best Possible Solution

Keep #69303 open as the canonical bug until #72291, #71819, or an equivalent maintainer-approved fix lands. The fix should preserve release-style retry for failures before processing, but stamp a terminal/poisoned dedupe state once a turn has made user-visible progress or external side effects, with regression tests covering both paths.
