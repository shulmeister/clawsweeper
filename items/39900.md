## Summary

Keep #39900 open. Current main still does not preserve or forward `extraBody`/`extra_body` for OpenAI-compatible TTS: the config schema can now accept object-valued provider fields, but the bundled OpenAI speech provider normalizes only known TTS fields and `openaiTTS` serializes only the fixed OpenAI `/audio/speech` payload.

## Best Possible Solution

Keep the issue open and implement a narrow bundled OpenAI speech-provider change: accept `extraBody` and `extra_body` as JSON records in `messages.tts.providers.openai`, preserve them through `resolveConfig`, merge them into normal and telephony `/audio/speech` request bodies with documented precedence, add focused request-body tests, and document the field on https://docs.openclaw.ai/providers/openai.
