## Summary

Current main still matches #71646: `OpenClawChannelBridge` keeps `pendingClaudePermissions` and `pendingApprovals` as plain instance Maps, clears only `pendingWaiters` on close, and has no TTL/sweeper/cap or focused regression tests for these pending maps. The linked fix PR #71648 is the active path and has not landed in the checked-out main revision.

## Best Possible Solution

Keep #71646 open until #71648 or an equivalent maintainer-approved fix lands. The fix should clear both pending maps on bridge close, expire stale Claude permission and approval entries, preserve the existing MCP protocol/tool surface, and add focused regression tests. A hard cap/FIFO guard can land in the same PR or be tracked as a follow-up if maintainers want to split slow-leak cleanup from burst protection.
