## Summary

Current main still matches the reported MCP channel-bridge leak shape: `pendingClaudePermissions` and `pendingApprovals` are plain instance Maps with narrow success-only delete paths, no TTL/sweeper/cap, and no close-time clear. The provided GitHub context also shows same-author PR #71648 is open and uses closing syntax for this issue, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #71648, or an equivalent maintainer-approved patch. The fix should clear both pending maps on bridge close, prevent post-close writes, expire stale Claude permission and approval entries using existing approval expiry metadata where available, preserve MCP notification/tool behavior, and add focused regression tests. Maintainers should decide whether a hard cap/FIFO guard belongs in the same patch or a narrow follow-up.
