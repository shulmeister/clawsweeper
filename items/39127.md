## Summary

Keep #39127 open. Current main still does not provide the requested gateway-owned per-session activity RPCs or WebSocket statechange event. The only nearby runtime activity fields are channel-account scoped, and the related implementation PRs in the provided context were closed unmerged.

## Best Possible Solution

Keep #39127 open as the canonical core/gateway API item. The best path is an additive gateway protocol implementation for `sessions.activity.get`, `sessions.activity.list`, and `sessions.activity.statechange`, with READ-scope authorization, schema exports/validators, gateway handlers, a gateway-owned activity tracker, explicit attention-state transitions, stale-run cleanup, bounded child rollups, reconnect/list resync semantics, and focused tests covering run transitions, stale terminal events, attention clear/set behavior, and cross-agent child sessions. Closed PRs #39438 and #41399 are useful prior art but should not supersede the issue because neither merged to main.
