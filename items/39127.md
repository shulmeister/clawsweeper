## Summary

Keep #39127 open. Current main at 67b9167b still lacks the requested gateway-owned per-session activity RPCs and WebSocket statechange event; nearby status surfaces are either channel-account scoped or display/session-list metadata, not the explicit runtimeActivity/attention/stale-run/children-rollup contract requested here. The related implementation PRs in the provided context were closed unmerged, so they are useful prior art but do not supersede the issue.

## Best Possible Solution

Keep #39127 open as the canonical core Gateway API item. The best path is an additive protocol implementation for sessions.activity.get, sessions.activity.list, and sessions.activity.statechange with exported schemas/types, READ-scope authorization, gateway handlers, a gateway-owned activity tracker, run-id based activeRuns, stale-run cleanup, explicit attention state transitions, bounded child rollups, reconnect/list resync semantics, docs, and focused tests. Use closed PRs #39438 and #41399 as prior art, but do not treat them as shipped or superseding because neither merged to main.
