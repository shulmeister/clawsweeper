## Summary

Keep open. Current main still does not implement the requested gateway-owned `sessions.activity.get/list` RPCs or a `sessions.activity.statechange` WebSocket event. Existing activity fields are channel-scoped, and the related implementation PRs #39438 and #41399 were closed unmerged, so they do not supersede the issue.

## Best Possible Solution

Keep #39127 open as the canonical core/gateway API tracking item. The best path is to implement a gateway-owned session activity contract in `src/gateway/protocol/schema/sessions.ts`, wire `sessions.activity.get/list` through `src/gateway/server-methods/sessions.ts`, `src/gateway/server-methods-list.ts`, and `src/gateway/method-scopes.ts`, add a scoped WebSocket statechange event, document it in the public gateway protocol docs, and cover run transitions, stale-run cleanup, explicit attention states, and cross-agent child rollups with focused gateway tests. The closed PRs #39438 and #41399 are useful prior art, but neither shipped on main.
