## Summary

Keep open. Current main has adjacent tool-failure plumbing, but the reported default `exec` command-not-found/fabricated-success case is not solved: non-timeout `exec` failures are still intentionally suppressed when verbose details are not enabled, and the CLI repro does not pass `--verbose on`. The linked NemoClaw sandbox-side python symlink fix is useful related context, but it does not resolve OpenClaw's agent-loop/reporting behavior. The related PR #60507 is also adjacent security approval work, not a canonical fix for this issue.

## Evidence

- **Default payload policy suppresses exec failures without verbose detail:** `resolveToolErrorWarningPolicy` computes detail inclusion, then immediately returns `showWarning: false` for `exec`/`bash` when details are not enabled. This happens before the general mutating-tool warning path, so a failed exec can be hidden from the user-facing payload.
