## Summary

Current main still matches the reported core problem closely enough that this should remain open. The session-history helper still returns true for the first assistant message anywhere in the inspected JSONL, the retry prompt is still hard-coded in core, and the related PR #69109 was closed unmerged while explicitly leaving the retry-prompt hook request out of scope.

## Best Possible Solution

Keep the issue open. The narrow core fix should update `sessionFileHasContent`/`jsonlFileHasAssistantMessage` to track assistant-after-latest-user and add the exact fresh-user regression test. Separately, maintainers should choose a supported retry-prompt override surface, likely a config field or retry-aware plugin hook payload, and document it alongside model fallback behavior.
