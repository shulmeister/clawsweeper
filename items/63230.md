## Summary

Current main does not implement the PR's requested per-channel Slack `thread.requireExplicitMention` override. Slack supports the setting only at the account level today: channel config schema lacks `thread`, the channel resolver has no thread override field, and mention gating still uses only `ctx.threadRequireExplicitMention`. The PR remains a focused, useful change, though the review comment about narrowing the channel-level schema should be addressed before landing.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a scoped Slack plugin/config change that accepts `channels.slack.channels.<id>.thread.requireExplicitMention`, resolves channel entry then wildcard then account default, applies that value in mention gating, adds override-direction tests, and documents the new per-channel setting. Address the existing review comment by using a narrow channel thread schema rather than reusing the full account-level `SlackThreadSchema`.
