## Summary

Keep #9155 open. Current main still does not implement `cron.defaults.delivery` or an equivalent primary cron delivery default; it only supports explicit per-job delivery and separate failure-notification defaults, while omitted isolated cron delivery still resolves through `last`.

## Best Possible Solution

Keep the issue open until OpenClaw implements a real primary cron delivery default, likely `cron.defaults.delivery`, or maintainers choose and document a deliberate alternate product direction. A complete fix should add config types/schema/help/docs, apply precedence as explicit job delivery first then global primary default then `last`, and cover service normalization, delivery planning, gateway protocol, CLI, and Control UI paths with tests.
