## Summary

Keep #9155 open. Current main at 7a85c1a8226eb7d3a96906d2721a189768d3034a still does not implement `cron.defaults.delivery`: cron config has no `defaults` block, the strict config schema would reject it, cron creation only preserves explicit delivery or defaults isolated agent-turn jobs to `{ mode: "announce" }`, and delivery planning still falls back to `channel: "last"` when no target is present. Related fixes for `last` route/session-context bugs mitigate #3520/#45806-style failures, but they are not a global config-level default delivery target.

## Best Possible Solution

Keep the issue open until OpenClaw either implements a real config-level primary cron delivery default or a maintainer explicitly chooses a different product direction. A complete fix would add `cron.defaults.delivery` to `CronConfig`, zod/schema/generated config docs/help, cron creation/normalization, CLI/UI/docs surfaces as needed, and regression tests proving job-level delivery overrides the global default before falling back to `last`.
