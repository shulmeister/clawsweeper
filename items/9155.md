## Summary

Keep open. Current main does not implement `cron.defaults.delivery`; it only defaults isolated agent-turn jobs to announce delivery and then falls back to `last` when no explicit channel/target is present. Main does include newer session-key delivery inference for cron jobs created from an active chat, which helps the related `last` routing bugs, but it is not a global default delivery target for all cron jobs.
