## Summary

Keep #9155 open. Current main at d74f897c1c6b2de756c98eb3bc7de45f411d6a5c still does not implement `cron.defaults.delivery`: the cron config type/schema have no `defaults` block, job creation only preserves explicit delivery or defaults isolated agent-turn jobs to `{ mode: "announce" }`, and delivery planning still falls back to `channel: "last"` when no target is present. The recent session-context inference mitigates related `last` routing bugs for jobs created from an active chat, but it is not a global config-level default delivery target. The existing April 26 maintainer/Codex discussion also explicitly kept this issue open.

## Best Possible Solution

Keep the issue open until OpenClaw either implements a real config-level primary cron delivery default or a maintainer explicitly chooses a different product direction. A complete fix would add `cron.defaults.delivery` to `CronConfig`, zod/schema/generated config docs/help, cron creation/normalization, CLI/UI/docs surfaces as needed, and regression tests proving job-level delivery overrides the global default before falling back to `last`.
