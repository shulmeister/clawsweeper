## Summary

Keep #9155 open. Current main still does not expose `cron.defaults.delivery` or an equivalent global default for normal cron announce delivery; only failure-notification defaults exist, while omitted or targetless announce delivery still resolves through `last`. The closed same-scope item #41164 points back here, so this issue remains the canonical tracker.

## Best Possible Solution

Keep #9155 open as the canonical product issue. Implement a primary cron delivery default, likely `cron.defaults.delivery`, with precedence of job-level `delivery` first, then global cron default, then existing `last` fallback. The change should update config types, zod and generated schema/help, docs, CLI creation/edit flows, gateway/service normalization, Control UI surfaces, and regression tests covering explicit job overrides, default application, and compatibility fallback.
