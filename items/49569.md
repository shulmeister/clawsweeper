## Summary

Keep open. Current `main` still lacks the PR’s 429 `retry_after` backoff handling in Telegram command-menu sync, so this is not implemented on `main` and there is not strong evidence to close it as obsolete.

## Evidence

- **429s still fall through to the generic failure path:** `syncTelegramMenuCommands` retries only `BOT_COMMANDS_TOO_MUCH`. Any other `setMyCommands` error is rethrown, and the outer `catch` only logs `Telegram command sync failed`, which matches the PR’s reported 429 failure mode.
