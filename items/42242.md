## Summary

PR #42242 adds a useful GoHighLevel CRM channel/skill, but current OpenClaw policy and SDK surfaces put optional third-party service integrations on the external plugin/ClawHub path rather than in the core repository. Current main already exposes channel, webhook, inbound/outbound, account, and runtime APIs sufficient for this kind of plugin, and the PR discussion still has implementation concerns that should be handled in an external plugin or split into focused core PRs if still relevant.

## Best Possible Solution

Close this core-repo PR and move the GoHighLevel work to an external native plugin published through ClawHub or npm using the current generic channel, webhook, inbound/outbound, account, and runtime SDK surfaces. If a concrete SDK gap appears while extracting it, track that as a narrow plugin API issue or PR; split unrelated core fixes separately if they are still needed.
