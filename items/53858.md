## Summary

Current main still matches the reported Nostr channel restart loop. The DM subscription still passes an array through `subscribeMany`, the gateway auto-restarts any `startAccount` promise that settles, and the Nostr gateway start path still returns immediately after creating the bus. The open related PR #42832 is an implementation candidate, not a reason to close this issue before merge.

## Best Possible Solution

Keep this issue open as the canonical Nostr restart-loop/subscription tracker. Review and rebase #42832 or prepare a fresh patch against current main: pass a plain `Filter` object to `SimplePool.subscribeMany`, remove the unsafe cast, keep `startNostrGatewayAccount` pending until `ctx.abortSignal` aborts, close both the subscription and SimplePool relay connections during cleanup, and add regression coverage for the serialized REQ filter shape plus gateway lifecycle so a successful Nostr start does not auto-restart.
