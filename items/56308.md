## Summary

Keep open. Current `main` has partial session-title plumbing in gateway/TUI code and supports manual session labels through `sessions.patch`, but the user-facing `openclaw sessions` CLI still lists technical keys only, the docs still describe key-only output, and there is no `/title` slash command on main. The issue discussion’s linked Clawdi-AI fork PRs are useful prior art, but they are not present in this checkout.

## Evidence
