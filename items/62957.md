## Summary

This issue should stay open. Current main still enforces the 16,000-token context-window minimum as a hard runtime block, and the surrounding tests, custom-provider onboarding behavior, and local-model docs still encode that policy.

## Best Possible Solution

Keep this issue open for maintainer product judgment. A complete fix would replace the runtime hard block with warning-only behavior or a deliberate override, then update src/agents/context-window-guard.ts, src/agents/pi-embedded-runner/run/setup.ts, custom-provider onboarding normalization, the guard/onboarding tests, and docs/gateway/local-models.md so sub-16k experimentation is consistently allowed while still warning users about limitations.
