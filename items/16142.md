## Summary

Keep #16142 open. Current main still routes main-session cron `systemEvent` reminders through the heartbeat model path: cron enqueues the event, heartbeat builds a cron relay prompt, and `runHeartbeatOnce` calls `getReplyFromConfig`. Existing tests assert that behavior, and docs describe cost mitigations rather than the requested deterministic string-formatting bypass.

## Best Possible Solution

Keep the issue open until main-session cron `systemEvent` reminders have a deterministic delivery path for simple reminder text that avoids `getReplyFromConfig`, while preserving model-mediated behavior for events that need reasoning. The likely implementation belongs around `executeMainSessionCronJob`, `resolveHeartbeatRunPrompt`, or a small delivery helper, with regression tests proving simple reminders avoid the model and docs clarifying which cost controls are mitigations versus true bypasses.
