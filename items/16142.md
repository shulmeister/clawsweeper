## Summary

Keep #16142 open. Current main still routes main-session cron/systemEvent reminders through the heartbeat model path: cron enqueues the text, heartbeat turns it into a scheduled-reminder prompt, and runHeartbeatOnce calls getReplyFromConfig. The prior same-day review also kept it open, and there is no relevant diff in the inspected cron/heartbeat source, tests, or docs between that review SHA and current main.

## Best Possible Solution

Keep #16142 open until main-session cron systemEvent reminders have a deterministic delivery path for simple reminder text that avoids getReplyFromConfig, while preserving the model path for events that need reasoning. The likely implementation belongs around executeMainSessionCronJob, resolveHeartbeatRunPrompt, or a small delivery helper, with tests proving simple reminders avoid the model and docs clarifying deterministic bypasses versus remaining cost mitigations.
