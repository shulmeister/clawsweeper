## Summary

Keep open. Current `main` does not implement native Feishu card table rendering, and the PR still has a plausible regression in interactive-card text extraction for reply/quote context.

## Evidence

- **Current main still sends card markdown as one raw markdown element:** `buildMarkdownCard()` on `main` returns a schema 2.0 card whose `body.elements` is a single `{ tag: "markdown", content: text }` entry. There is no table-component conversion in the current implementation.
