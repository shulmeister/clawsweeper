## Summary

Current `main` already persists Control UI uploads into transcript `MediaPath`/`MediaPaths` fields and the Control UI history renderer already displays those fields. This PR changes a different `chat.history` inline-image sanitization path (`{ type: "image", data, mimeType }` blocks), so it does not line up with the regression described in #70507.

## Evidence

- **Issue report points at transcript media-path round-trip:** #70507 says the optimistic UI uses inline/base64 preview content, but persisted history is expected to round-trip through transcript `MediaPath` / `MediaPaths`, and the suspected failure is the transcript rewrite/persistence step.
