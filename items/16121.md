## Summary

Keep #16121 open. Current main still lacks first-class Telegram custom emoji formatting for normal Markdown-rendered messages/captions and action sends. The latest maintainer/Codex comment in the issue also explicitly kept it open, and no relevant Telegram formatter, Markdown IR, send, action runtime, or Telegram docs files changed between that reviewed SHA and current main.

## Best Possible Solution

Keep #16121 open. Implement custom emoji support in the bundled Telegram plugin formatting path with a structured representation or explicit Markdown-level syntax carrying `emoji-id` plus fallback text; render safe Telegram HTML as `<tg-emoji emoji-id="...">fallback</tg-emoji>`; preserve plain-text fallback; and cover normal messages, captions, replies/quotes, chunking, preview streaming, action sends, docs, and formatter/delivery tests.
