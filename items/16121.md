## Summary

Keep #16121 open. Current main has Telegram HTML transport and a trusted `textMode: "html"` escape hatch, but it still lacks first-class custom emoji parsing/rendering for normal markdown/model output, captions, replies, and preview paths. The issue discussion explicitly asks for `<tg-emoji emoji-id="...">` plus a markdown-level syntax, and a maintainer Codex review already kept it open on 2026-04-26.

## Best Possible Solution

Keep #16121 open. Implement this in the bundled Telegram plugin formatting path: add a documented custom emoji syntax or structured entity, validate and escape `emoji-id`, render Telegram HTML as `<tg-emoji emoji-id="...">fallback emoji</tg-emoji>`, preserve sensible plain-text fallback, and cover messages, captions, replies, chunking, parse-fallback behavior, preview streaming, and docs/tests under `extensions/telegram` and `docs/channels/telegram.md`.
