## Summary

Keep #16121 open. Current main still does not provide first-class Telegram custom emoji support for normal markdown/model output, captions, or replies. The issue discussion specifically asks for `<tg-emoji emoji-id="...">` and a markdown-level syntax; current Telegram formatting still escapes raw HTML by default, and the only raw-HTML path is a trusted caller escape hatch.

## Best Possible Solution

Implement this in the bundled Telegram plugin formatting path: add a documented custom emoji syntax or entity representation, validate and escape `emoji-id` safely, render Telegram HTML as `<tg-emoji emoji-id="...">fallback emoji</tg-emoji>`, preserve sensible plain-text fallbacks, and cover messages, captions, replies, chunking, parse-fallback behavior, and docs/tests under `extensions/telegram` and `docs/channels/telegram.md`.
