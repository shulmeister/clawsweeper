## Summary

Keep open. Current main has adjacent docs discovery surfaces (`docs:list`, Docs hubs, live docs search, and public `llms.txt`), but it does not implement the requested public `docs_map.md`/`docs-map.md` file with per-page H2/H3/H4 heading hierarchy. The issue is recent, has no protected maintainer label or maintainer author signal, and the requested docs artifact is concrete enough to track in this repo.

## Best Possible Solution

Keep the issue open and implement a repo-owned docs map generator: parse docs/**/*.md for H2/H3/H4 headings, order pages consistently with docs/docs.json or docs:list output, publish the generated map through docs-sync-publish, and add a docs check so the public artifact cannot drift. The existing Docs hubs and llms.txt surfaces should remain, but they do not replace the requested heading hierarchy map.
