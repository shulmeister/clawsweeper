## Summary

Keep open. Current main has useful docs navigation surfaces (`pnpm docs:list`, docs hubs, live docs search, and agent prompt pointers), but none implement the requested public `docs_map.md` sibling with a nested H2/H3/H4 heading hierarchy for every docs page. The public docs currently serve `llms.txt`, while `docs_map.md` and `docs-map.md` return 404.

## Best Possible Solution

Keep the issue open. The best path is to add a generated, published docs map artifact, likely by extending the existing docs tooling to parse `docs/**/*.md`/`*.mdx` frontmatter plus H2-H4 headings, wire it into the docs publish/check workflow, and link it from `llms.txt` or the docs hub so agents can discover it reliably.

- **No tracked docs map artifact on current main:** No tracked `docs_map.md`, `docs-map.md`, `llms.txt`, or `llms-full.txt` file exists in this checkout; history also has no tracked docs-map file for those names.
