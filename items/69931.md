## Summary

Keep #69931 open. Current main supports sending WhatsApp reactions and ack reactions, but I could not find a plugin-visible or agent-visible inbound WhatsApp reaction event/read surface for reactions on historical messages. This is not a good ClawHub close because the reporter’s central point is a missing OpenClaw/WhatsApp API surface that an external skill would need before it could read reactions reliably.

## Best Possible Solution

Keep the issue open as a WhatsApp/plugin API gap. The likely maintainer path is to add a typed inbound reaction surface for WhatsApp, ideally a generic channel/plugin reaction event contract with fields like channel/account/conversation, target message id, emoji, sender, timestamp, and added/removed direction, then wire the WhatsApp Baileys reaction event into that surface. Once core exposes this, a ClawHub skill/plugin can build the historical-reaction automation on top of it.
