## Summary

Keep open. Current main still resolves WhatsApp outbound targets through config-backed directory entries and generic id/name matching, but the WhatsApp peer directory only exposes normalized `allowFrom` ids. That means a DM request like “tell Jack …” remains plausibly unresolved unless the model already knows Jack’s exact WhatsApp target id. I found no post-report fix on the relevant paths.
