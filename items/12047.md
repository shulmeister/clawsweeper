## Summary

Keep #12047 open. Current main has partial coverage: Telegram document attachments are downloaded and forwarded through shared media context, and text-like files plus PDFs can be extracted. The requested Office-style document support remains missing because default file MIME allowlists omit DOCX/XLSX/PPTX, the shared media pass skips application/vnd Office MIME types as binary before extraction, and the bundled document extractor only declares PDF support.

## Best Possible Solution

Keep this open as an OpenClaw product/contract issue. The best fix is shared and channel-independent: keep Telegram forwarding attachments through MediaPath/MediaType, extend safe file MIME handling so Office document MIME types can reach document extractor plugins, generalize non-PDF document-extractor dispatch, add or bundle an Office extractor implementation, and cover Telegram attachments plus shared extraction with tests and docs.
