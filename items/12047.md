## Summary

Current main only partially satisfies #12047. Telegram document attachments are downloaded and forwarded into the shared media path, and plain text/Markdown/JSON/PDF can be extracted, but Office-style document extraction remains missing: Office MIME types are treated as binary before extraction, default file MIME allowlists omit Office formats, the document extraction dispatcher only routes PDFs, and the bundled document-extract plugin declares/supports only PDF. The existing issue discussion also contains a same-day Codex review by steipete keeping this open for the same remaining Office/.docx gap.

## Best Possible Solution

Keep this open as an OpenClaw product/contract issue. Extend the shared media/input-file extraction path so safe Office MIME types can reach document extractors, add a bundled or plugin-owned Office extractor implementation, and cover Telegram attachments plus generic shared extraction with tests and docs. Telegram should continue forwarding MediaPath/MediaType into the shared path rather than parsing documents itself.
