## Summary

Keep #12047 open. Current main partially addresses Telegram document attachments for text-like files and PDFs, but the requested .docx/Office-style extraction remains unresolved: Telegram passes downloaded document paths and MIME types into the shared media path, yet shared file extraction explicitly treats application/vnd.* Office MIME types as binary and the default file extractor only supports text/JSON/PDF.

## Best Possible Solution

Keep this as an OpenClaw issue. Extend the shared media/document extraction path so supported Office MIME types such as .docx can pass through a safe document-extractor contract with size limits, untrusted-content wrapping, tests, and docs. Telegram should continue to supply MediaPath/MediaType into that shared path rather than owning document parsing itself.
