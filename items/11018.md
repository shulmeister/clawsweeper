## Summary

#11018 is a useful notification-router idea, but current main already exposes the needed plugin/event/task surfaces for this to live as a ClawHub plugin rather than a new native core system. The exact proposed native events_subscribe/policy JSON is not implemented, so this is not an implemented-on-main close.

## Risks / Open Questions

- A future maintainer may still want a first-party notification-router plugin or a smaller documented sample based on the linked userland implementation.
- If a plugin author needs richer direct channel delivery than session-queued system events and existing reply/channel helpers provide, that should be tracked as a focused plugin API gap rather than this broad core feature.
