## Summary

Current `main` has partial adjacent pieces: internal message hooks, internal agent event plumbing, and push-based completion for spawned subagents/background tasks. It does not implement the requested native event-driven notification system end to end: the documented internal hooks do not expose `session:progress`/`session:completed` or tool lifecycle events, there is no agent-facing `events_subscribe` API, and subagent task delivery is still intentionally limited rather than a general proactive notification bus.

## Evidence

- **Documented internal hooks are narrower than the request:** The hooks docs list `command:*`, `session:compact:*`, `session:patch`, `agent:bootstrap`, `gateway:startup`, and message events (`message:received`, `message:transcribed`, `message:preprocessed`, `message:sent`). They do not list `session:spawn`, `session:progress`, `session:completed`, `session:error`, or `tool:*` events.
  - file: [docs/automation/hooks.md:34](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/docs/automation/hooks.md#L34)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Current hook type surface stays generic/internal:** `InternalHookEventType` is only `command | session | agent | gateway | message`; the current publicized internal-hook surface does not define the requested session/tool lifecycle event family.
  - file: [src/hooks/internal-hook-types.ts:1](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/hooks/internal-hook-types.ts#L1)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Subagent push path is completion-only in docs:** Subagent docs say `/subagents spawn` sends one final completion update back to the requester chat when the run finishes, and describe completion as push-based. That matches the issue comment about completion announce, but not the broader proactive event bus requested.
  - file: [docs/tools/subagents.md:40](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/docs/tools/subagents.md#L40)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Task delivery intentionally excludes normal subagent terminal auto-delivery:** `shouldAutoDeliverTaskTerminalUpdate()` returns false for `runtime === "subagent"` unless the status is `cancelled`, and the test suite explicitly asserts a succeeded subagent task should not auto-deliver. That means current main still does not provide general native proactive subagent completion/progress notification delivery.
  - file: [src/tasks/task-executor-policy.ts:92](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/tasks/task-executor-policy.ts#L92)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **No agent-facing subscription API found:** A repo search for `session:completed|session:progress|session:error|tool:started|tool:completed|tool:error|events_subscribe` returned no matches in `src` or `docs`, so the requested agent-accessible event subscription surface is not present on current `main`.
  - command: `rg -n 'session:completed|session:progress|session:error|tool:started|tool:completed|tool:error|events_subscribe' src docs || true`
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **There is internal event plumbing, but it is not the requested user-facing feature:** `agent-events.ts` exposes an internal listener/emit bus (`emitAgentEvent`, `onAgentEvent`) used for runtime wiring. This is internal infrastructure, not a documented agent-usable notification/event-subscription product surface.
  - file: [src/infra/agent-events.ts:201](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/infra/agent-events.ts#L201)
