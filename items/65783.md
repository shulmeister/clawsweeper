## Summary

Keep open. Current main still has the unsafe coarse split loop in both memory-host-sdk implementations and still sends raw chunk text/structured inputs to embedding providers without unpaired-surrogate sanitization. The PR is tied to the still-open same-author bug report #65782 and addresses a plausible real bug, but it should be revised before merge because the shown diff only patches the package-local chunker while the published plugin SDK storage barrel re-exports the canonical src/memory-host-sdk implementation.

## Best Possible Solution

Keep this PR open and ask for a focused rebase/revision: patch the canonical `src/memory-host-sdk/host/internal.ts` implementation, keep any package mirror aligned if it remains present, add the coarse-boundary regression in the relevant current test suite, and add/embed-boundary sanitizer coverage in `extensions/memory-core`. Once that corrected version is reviewed and lands, it can resolve #65782 and the #27753 class of failures.
