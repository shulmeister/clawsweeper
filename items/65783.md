## Summary

Keep #65783 open. Current main still lacks both requested behaviors: the memory chunker can still cut a UTF-16 surrogate pair at the outer coarse split boundary, and the memory embedding boundary still forwards raw text to providers without a lone-surrogate sanitizer. The PR discussion is concrete and favorable: Greptile considered the fix correct, the author addressed the double-sanitization nit, and a Codex review comment identified that the canonical runtime chunker also needed patching. That work remains useful rather than obsolete.

## Best Possible Solution

Keep the PR or a maintainer-owned successor open and update it to patch both chunker copies, especially the canonical `src/memory-host-sdk/host/internal.ts` path exported through the plugin SDK, plus add the embed-boundary sanitizer and regression tests for the coarse surrogate split and sanitizer behavior. After that, run the changed memory-core and memory-host-sdk lanes before landing.
