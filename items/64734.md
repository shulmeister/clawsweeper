## Summary

Keep this PR open. Current main still has the 10s Discord thread-title timeout and 512-token completion budget, so the proposed 60s/4096-token change is not obsolete or implemented on main. The related merged #64172 only raised the token budget to the current 512 value and did not address the timeout path described here.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The likely maintainer path is to review or revise the narrow Discord plugin change, run the targeted Discord extension tests, and decide whether the 60s/4096 constants are the right shipped defaults. The observability concern in the PR body can be handled as a separate follow-up rather than blocking this focused fix.
