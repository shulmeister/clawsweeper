## Summary

Keep this issue open. Current main still intentionally suppresses assistant output classified as `commentary` before channel partial/block delivery, so early commentary cannot be committed as a standalone durable Discord message before restart-sensitive work. Discord's draft stream and reply dispatcher only operate on partial/block/final payloads that reach the channel path, while current tests continue to assert that commentary-phase output is not emitted.

## Best Possible Solution

Keep this issue open for maintainer product and channel-delivery work. Decide whether assistant commentary is intended to be user-visible; if it is, add a channel-safe materialization and flush path for approved commentary at assistant-message, tool, or restart boundaries, route it through Discord's reply dispatcher and restart-deferral machinery, and add regression tests. If commentary suppression is intentional policy, document that explicitly before closing.
