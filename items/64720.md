## Summary

Keep this PR open. Current main still only normalizes and neutralizes spoofed inbound system markers for channel bodies; it does not wrap channel message bodies or thread context with `wrapExternalContent` boundaries. The PR is a plausible implementation candidate, but the provided review discussion identifies remaining P1 trust-classification issues around `OriginatingChannel`, `heartbeat`, and `cron-event` that should be fixed before merge rather than closing the PR as obsolete.

## Best Possible Solution

Keep the PR open and ask for a revision: derive wrapping from ingress trust/source provider rather than reply-routing `OriginatingChannel`, include trusted internal providers such as `heartbeat` and `cron-event`, keep the raw body for queue summaries, wrap channel thread context deliberately, and land focused regression tests around channel bodies, thread context, and internal scheduler prompts.
