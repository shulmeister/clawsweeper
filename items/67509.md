## Summary

Keep this PR open. Current main still has no equivalent root/effective-root CLI guard, and the linked #67478 failure mode remains plausible through the modern CLI entrypoint, legacy entrypoint, default state directory resolution, and Linux systemd user-service install path. The PR remains the active implementation candidate and has incorporated review feedback for legacy entry coverage, container forwarding, narrowed bypass semantics, and effective UID handling.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent core fix that guards both src/entry.ts and src/index.ts before state/config side effects, checks real and effective UID, preserves intentional container forwarding, includes focused regression tests, and updates DigitalOcean/install guidance or the wrapper flow so root SSH users get a clear supported command path.
