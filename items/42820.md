## Summary

Current `main` still matches the reported failure mode. The shared `message` tool schema still includes poll creation fields, the outbound runner still throws on `action: "send"` when any poll creation param is present, and regression tests still enforce that behavior. The only related fix on `main` is narrower: zero-valued numeric poll defaults are treated as unset, which does not address non-empty poll strings/arrays or `true` booleans that models may still auto-populate.
