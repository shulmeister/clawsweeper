## Summary

Current main does not implement the requested sessions_send sender identity display. The UI can render a top-level senderLabel when present, but sessions_send neither accepts nor propagates a sender/displayName/avatar field; it only attaches inter_session provenance, which the UI does not use for the displayed sender name/avatar. This remains a valid core/API/UI feature request, not a ClawHub item.
