## Summary

Keep this issue open. Current main still has only per-agent `reasoningDefault` plus an implicit model-capability fallback of `on`/`off`; it does not expose the requested per-model `reasoningDefault` configuration under `agents.defaults.models`, and the linked open PR only partially addresses per-agent channel handling for Telegram/Feishu rather than this schema/runtime feature.

## Best Possible Solution

Add a first-class per-model reasoning visibility setting, likely `reasoningDefault?: "on" | "off" | "stream"` on `AgentModelEntryConfig` / `agents.defaults.models[modelRef]`, validate it in the Zod/generated config schemas, document its precedence, and update reply/model-selection resolution so directive and session overrides still win, per-agent defaults remain explicit, and selected-model config can override the catalog-derived `on`/`off` fallback. Add focused tests for a MiniMax model defaulting to `stream` and a reasoning-capable GPT model defaulting to `off`. The adjacent Telegram/Feishu per-agent PR should be reviewed on its own merits but should not close this issue unless it expands to cover the per-model schema and runtime behavior.
