## Summary

PR #64865 is superseded by merged PR #64779, which closed #64748 by making the canonical parent plugin id (`memory-wiki`) sufficient for `openclaw wiki` under `plugins.allow`. Current main has manifest-backed `wiki` alias metadata, CLI allow-guard coverage, validation guidance that points alias entries back to `memory-wiki`, and release notes for the shipped fix. The exact alias-preservation behavior proposed by #64865 is not the shipped direction and the PR patch is stale against the current structured `commandAliases[].name` contract.

## Best Possible Solution

Close #64865 as superseded by merged #64779. Keep the shipped parent-plugin allowlist behavior: `openclaw wiki` should work when `memory-wiki` is in `plugins.allow`, and config validation should point alias entries back to the owning plugin id. If maintainers later want doctor to preserve alias entries as a separate compatibility choice, that should be opened as a new narrow product decision with tests against structured `commandAliases[].name`.
