## Summary

Close PR #41931 as superseded. The underlying Windows installer bug is still real on current main, but this PR’s `[void](Main)` approach drops failure signaling for direct PowerShell script runs. Open PR #48983 now tracks the same fix with a guarded top-level completion path, and #38054 remains the canonical issue for the broader Windows installer hard-exit behavior.

## Best Possible Solution

Close #41931 as superseded by open PR #48983, while keeping the remaining product bug consolidated on canonical issue #38054. The eventual landed fix should avoid `exit` inside `Main` for `iex`/scriptblock installs while preserving non-zero exit codes for direct `powershell -File` / `pwsh -File` execution, with focused regression coverage or manual Windows proof.
