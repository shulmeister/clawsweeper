## Summary

Keep #17211 open. Current main still has compaction checkpoint snapshot listing, branch, and restore, but not the requested native checkpoint engine with checkpoint.create/list/resume/diff APIs, deterministic replay packets, structured task-boundary contracts, diffing, or missing-field validation hooks; the existing issue discussion also includes a maintainer/Codex comment explicitly keeping this umbrella open.

## Best Possible Solution

Keep #17211 open as the native checkpoint/replay umbrella. A complete solution likely belongs in versioned Gateway/session protocol and session persistence: checkpoint.create/list/resume/diff, a structured done/saved/dropped/next task contract, deterministic replay packet generation, diff/audit data, validation status/hooks, UI/docs updates, and protocol/runtime/UI tests. If maintainers prefer plugin ownership, first define an explicit checkpoint/context-engine API so a plugin can implement it without reaching into core internals.
