## Summary

Keep #17211 open. Current main still implements compaction-scoped snapshot recovery, not the requested native checkpoint engine with checkpoint.create/list/resume/diff, deterministic replay packets, done/saved/dropped/next task contracts, diffing, or validation hooks. The existing maintainer-side Codex comment on the issue also explicitly keeps this umbrella open, and current main still matches that assessment.

## Best Possible Solution

Keep #17211 open as the umbrella for native checkpoint/replay design. A good implementation should add versioned Gateway/session APIs for checkpoint.create/list/resume/diff, persist a structured task contract with done/saved/dropped/next plus validation state, generate deterministic replay packets, expose diff/audit data, document the workflow, and add protocol/runtime/UI contract tests. If maintainers prefer a plugin-owned direction, first add an explicit checkpoint/context-engine API instead of relying on the current compaction provider or compact-only context-engine seam.
