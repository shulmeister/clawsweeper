## Summary

Keep open. Current main has compaction-scoped checkpoint snapshots and restore/branch support, plus plugin-owned compaction hooks, but it still lacks the requested native task-boundary checkpoint engine: generic create/resume/diff APIs, deterministic replay packet generation, and done/saved/dropped/next contract validation.

## Best Possible Solution

Keep #17211 open as the umbrella for a native checkpoint/replay design. A good implementation would add a versioned Gateway/session protocol for checkpoint.create/list/resume/diff, persist a structured task contract with done/saved/dropped/next plus validation, generate deterministic replay packets, provide source/diff auditability, and add docs plus contract tests. If parts should be plugin-owned, first extend the context-engine/plugin API explicitly rather than treating the current compaction provider seam as sufficient.
