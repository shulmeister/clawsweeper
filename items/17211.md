## Summary

Keep #17211 open. Current main (6a688e33f66f3965968343922e874cc0e9440998) has compaction-scoped checkpoint snapshots with list/get/branch/restore, but it still lacks the requested native task-boundary checkpoint engine: checkpoint.create/resume/diff APIs, deterministic replay packets, and done/saved/dropped/next validation.

## Best Possible Solution

Keep #17211 open as the umbrella for native checkpoint/replay design. A good implementation would add versioned Gateway/session APIs for checkpoint.create/list/resume/diff, persist a structured task contract with done/saved/dropped/next plus validation, generate deterministic replay packets, provide source/diff auditability, and add docs plus contract tests. If parts should be plugin-owned, first add an explicit context-engine/plugin API instead of relying on the current compaction provider seam.
