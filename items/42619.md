## Summary

Current `main` does not show a code path where a config-driven gateway restart rewrites `~/.openclaw/exec-approvals.json` to restrictive defaults; the report no longer matches the implementation.

On current `main`, `config.apply` only writes `openclaw.json` and schedules an authorized in-process `SIGUSR1` restart (`src/gateway/server-methods/config.ts:629`, `src/cli/gateway-cli/run-loop.ts:230`). Exec approvals remain host-local at `~/.openclaw/exec-approvals.json` (`src/infra/exec-approvals.ts:174`, `docs/cli/approvals.md:185`). Closing as not reproducible on current `main`. If you can still reproduce this on a newer build, please open a fresh issue with the exact restart path, active profile/HOME/OPENCLAW_HOME values before and after restart, and a before/after diff of the approvals file.
