## Summary

Keep #31583 open. Current main still blocks the reported `GOG_KEYRING_PASSWORD` path: sensitive-looking keys from `skills.entries.<skill>.env` only pass when declared by skill metadata, and the bundled `gog` skill still does not declare that env key. The linked issue discussion also identifies open PR #50960 as candidate fix work, not shipped behavior.

## Best Possible Solution

Keep #31583 open until #50960 or an equivalent maintainer-reviewed fix lands. The narrow product fix belongs in `src/agents/skills/env-overrides.ts`: allow explicitly configured `skills.entries.<skill>.env` sensitive-pattern keys such as `GOG_KEYRING_PASSWORD` through the skill env sanitizer while preserving the dangerous host-env blocklist, then add regression tests for undeclared configured `_PASSWORD`/`_TOKEN` keys. Track sandbox env behavior separately if maintainers want to change the documented host-only contract.
