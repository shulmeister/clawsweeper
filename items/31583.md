## Summary

Keep #31583 open. Current main still blocks the reported `GOG_KEYRING_PASSWORD` path: user-configured `skills.entries.<skill>.env` keys are sanitized against secret-looking names unless the skill metadata declares them, and the bundled `gog` skill does not declare `GOG_KEYRING_PASSWORD`. The related discussion points to open PR #50960 as candidate work, not shipped behavior.

## Best Possible Solution

Keep #31583 open until #50960 or an equivalent maintainer-reviewed fix lands. The narrow fix should allow explicitly configured `skills.entries.<skill>.env` keys such as `GOG_KEYRING_PASSWORD` through the sensitive-name sanitizer while preserving the dangerous host-env blocklist, add regression coverage for an undeclared password/token-style key, and separately decide whether sandbox exec should receive skill env or remain documented as host-only.
