## Summary

Keep #31583 open. Current main still has the reported sensitive skill-env blocker: configured `skills.entries.<skill>.env` values are collected, but sensitive-looking keys such as `GOG_KEYRING_PASSWORD` are only allowed when declared by skill metadata, and the bundled `gog` skill does not declare that env var.

## Best Possible Solution

Keep the issue open until #50960 or an equivalent maintainer-reviewed fix lands. The likely code path is `src/agents/skills/env-overrides.ts`: explicitly allow user-configured `skills.entries.<skill>.env` keys that match sensitive patterns while preserving the hard block for dangerous host/runtime override variables. Add regression coverage for undeclared configured `_PASSWORD`/`_TOKEN` keys and verify the host `exec` child-process path inherits the applied env.
