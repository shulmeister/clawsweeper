## Summary

Keep open. Current `main` still hardcodes Control UI uploads to `image/*`, serializes outbound dashboard attachments as `{ type: "image" }`, and the gateway rejects non-image attachment MIME types. The issue discussion comment pointing to #57707 is not reflected in `main` at `95b7a85f0607784d8e33b4ce7e42617174196490`.

## Evidence

- **Control UI file picker still image-only:** The chat composer imports `CHAT_ATTACHMENT_ACCEPT`, and the file input uses that constant. The constant is still `"image/*"`, so the browser picker does not allow generic documents/code files.
