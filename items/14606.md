## Summary

Current `main` already routes the UI Compact action through `compactEmbeddedPiSession()` and no longer uses raw transcript truncation unless a caller explicitly supplies `maxLines`. Slack also has supported manual compaction paths via either the single configured slash command flow or native Slack slash commands.

## Evidence

- **Gateway default compaction path is LLM-backed:** `sessions.compact` checks `maxLines`; when it is omitted, it interrupts any active run and calls `compactEmbeddedPiSession(...)` for the target session.
  - file: [src/gateway/server-methods/sessions.ts:1552](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/gateway/server-methods/sessions.ts#L1552)
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **UI uses the default smart-compaction path:** The UI slash command executor sends only `{ key: sessionKey }` to `sessions.compact`, so the request does not opt into the legacy line-truncation branch.
  - file: [ui/src/ui/chat/slash-command-executor.ts:156](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/ui/src/ui/chat/slash-command-executor.ts#L156)

## Close Comment

Closing as implemented on `main` after Codex review. The current UI compact path calls `sessions.compact` with only the session key (`ui/src/ui/chat/slash-command-executor.ts:156`), and the Gateway now routes that default path through `compactEmbeddedPiSession(...)` (`src/gateway/server-methods/sessions.ts:1552-1589`) instead of raw JSONL truncation. The old `slice(-maxLines)` truncation logic still exists only behind the explicit `maxLines` maintenance branch (`src/gateway/server-methods/sessions.ts:1636-1654`), so it is no longer what the UI button uses. The Slack accessibility concern is also covered on current `main`: Slack supports either single-command slash mode or native slash commands (`docs/channels/slack.md:635-665`), and native commands are routed back to the target chat session (`src/auto-reply/reply/session.ts:263-268`). I also verified the same smart-compaction implementation is present in the current release commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`.
