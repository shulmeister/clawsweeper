## Summary

Current main has partial mitigations for local custom-provider auth, but the reporter’s stated failing case remains plausible: an explicit custom-provider `apiKey` value of `"ollama-local"` is still classified as a non-secret marker and rejected instead of being used as a literal credential or migrated away. That is not strong enough evidence to close this issue.

## Evidence

- **Reporter’s placeholder value is still a registered marker:** The bundled Ollama plugin still declares `ollama-local` as a `nonSecretAuthMarker`, so that exact string is treated specially rather than as an ordinary API key.
  - file: [extensions/ollama/openclaw.plugin.json:7](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/extensions/ollama/openclaw.plugin.json#L7)
  - command: `nl -ba extensions/ollama/openclaw.plugin.json | sed -n '1,20p'`
  - sha: [320d52a23e41](https://github.com/openclaw/openclaw/commit/320d52a23e41a326287826219a20f1f29f334554)
- **Bundled plugin markers are globally treated as non-secret API-key markers:** `listKnownNonSecretApiKeyMarkers()` imports bundled plugin markers, and `isNonSecretApiKeyMarker()` returns true for them. That means `ollama-local` is globally recognized as a marker on current main.
  - file: [src/agents/model-auth-markers.ts:46](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/src/agents/model-auth-markers.ts#L46)

  - sha: [320d52a23e41](https://github.com/openclaw/openclaw/commit/320d52a23e41a326287826219a20f1f29f334554)
- **Custom providers still reject explicit marker-like apiKey strings:** `resolveUsableCustomProviderApiKey()` returns `null` when a custom provider’s configured key is a non-secret marker that is not also a known env-var marker. With `apiKey: "ollama-local"`, execution reaches lines 157-161 and rejects it.
  - file: [src/agents/model-auth.ts:153](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/src/agents/model-auth.ts#L153)

  - file: [src/agents/model-auth.ts:329](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/src/agents/model-auth.ts#L329)
