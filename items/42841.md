## Summary

Keep open. Current main still does not implement the PR's central behavior: `openclaw node run` and `openclaw node install` have no repeatable `--header` option, node-host config/run state has no gateway header field, and `GatewayClient` does not forward custom headers into the `ws` constructor. The linked issue #42792 was closed, but the observable code/docs behavior on main still lacks the requested Cloudflare Zero Trust WebSocket upgrade header path, and the PR discussion shows a concrete repro plus follow-up commits addressing review feedback rather than an obsolete or duplicate change.

## Best Possible Solution

Keep the PR open or replace it with a refreshed canonical implementation. The right product/code path is still in core node/gateway CLI: add a repeatable `--header` option for `node run` and `node install`, persist/resolve headers consistently for installed services, forward them through `GatewayClient` into `ws` upgrade options, document the Cloudflare Zero Trust workflow on the public node CLI docs page, and keep focused regression tests for parsing, env/config precedence, status redaction, install-plan environment handling, and WebSocket client options.
