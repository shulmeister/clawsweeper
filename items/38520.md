## Summary

Keep #38520 open. Current main still has adjacent compaction safeguards, but not the requested lifecycle: configurable pre-compaction warning thresholds, a visible structured handoff turn/window, or a bounded/audited compaction deferral tool/API. The related issues in the provided context are closed or partial, and the latest local diff since the prior ClawSweeper keep-open review is unrelated WhatsApp work.

## Best Possible Solution

Keep this open for maintainer product/API design. A complete solution likely belongs in core compaction configuration and runtime event handling: validated warning/critical threshold config, defined channel/UI delivery for pre-compaction warnings, a structured pre-compaction handoff turn with timeout and prompt policy, and a bounded/audited deferral contract if maintainers want agents or plugins to pause imminent compaction. It should update config schema/types/docs, compaction runner/event paths, any plugin/tool contracts needed, and focused tests for threshold warnings, handoff timeout behavior, and deferral limits.
