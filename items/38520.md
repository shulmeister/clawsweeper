## Summary

Keep #38520 open. Current main implements adjacent compaction safeguards, but the central requested lifecycle is still missing: configurable pre-compaction warning thresholds/events, a general structured handoff window with timeout policy, and an agent-callable bounded deferral mechanism. The issue is eligible for review, has no protected labels, and related closed items do not provide an open canonical tracker for the remaining warning/deferral work.

## Best Possible Solution

Keep this issue open as core compaction lifecycle/product work. The best path is either to implement narrower core controls for threshold warnings, a structured pre-compaction handoff turn with timeout policy, and a bounded audited deferral API, or for maintainers to explicitly split/decline those remaining parts. This should not be closed as implemented or ClawHub while compaction scheduling and deferral still require core behavior.
