## Summary

Keep #38520 open. Current main has adjacent safeguards for compaction, including silent memory flush, start/end user notices, post-compaction reinjection, plugin observation hooks, and context-engine maintenance, but it still does not implement configurable pre-compaction warning events, a structured handoff turn/window, or an agent-callable bounded deferral mechanism.

## Best Possible Solution

Keep this open for maintainer product/API design. A good implementation would add validated compaction lifecycle config for warning thresholds and handoff policy, define where warning events are emitted to agents/channels/UI, and add a bounded deferral contract only if maintainers want agents or plugins to pause imminent compaction. It should update `src/config/*`, compaction event/runner paths, plugin or tool contracts as needed, public docs, and focused tests for threshold warnings, handoff timeout behavior, and deferral limits.
