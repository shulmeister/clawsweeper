## Summary

Keep #6625 open. Current main at 9089e6b595ba33b4afc27b99c56ace80c4fbbc7b still exposes hard sub-agent run timeouts and post-timeout announce/partial-progress handling, but I found no configurable grace period, pre-timeout warning field, or timer that injects a checkpoint warning before killing native or ACP sub-agent runs. The existing maintainer/Codex comment correctly distinguishes this from the related partial-result persistence work.

## Best Possible Solution

Keep #6625 open for core sub-agent lifecycle work. Add a documented agents.defaults.subagents grace/warning setting, expose any needed sessions_spawn override deliberately, schedule a warning before runTimeoutSeconds, deliver it through the child-session control/message path for both native and ACP sub-agents where feasible, and add focused tests/docs while preserving current post-timeout partial-progress and fallback-delivery behavior.
