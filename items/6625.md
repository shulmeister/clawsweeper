## Summary

Keep #6625 open. Current main still has sub-agent run timeouts and post-timeout partial-progress recovery, but no configurable pre-timeout grace period, no `sessions_spawn` warning/grace parameter, and no native or ACP path that injects a checkpoint warning before the hard timeout.

## Best Possible Solution

Keep #6625 open for core sub-agent lifecycle work. Add a documented sub-agent timeout-warning/grace setting, optionally a `sessions_spawn` override, schedule a warning before `runTimeoutSeconds` expires, deliver it through an appropriate child-session control/message path for native and ACP runs where feasible, and cover it with focused tests and docs while preserving existing post-timeout partial-progress recovery.
