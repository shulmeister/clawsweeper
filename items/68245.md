## Summary

Current `main` and the shipped `v2026.4.20` release already changed Active Memory to fail open on recall/prompt-build failures instead of hanging the whole reply turn.

## Evidence

- **Prompt-build hook now fails open:** The Active Memory `before_prompt_build` hook wraps recall in `try/catch`, logs a warning, and returns `undefined` so the main reply can continue without injected memory context.
  - file: [extensions/active-memory/index.ts:2044](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/extensions/active-memory/index.ts#L2044)
