## Summary

Current `main` still omits `knownLongFlags` from the user-facing `safeBinProfiles` schema, generated schema, fixture type, and normalization path, while the runtime validator still consumes `profile.knownLongFlags` when present. The report matches current code, so this should stay open; the linked PR #56806 may still be the right vehicle.
