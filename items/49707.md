## Summary

Keep open. Current `main` still treats all-zero usage as absent in multiple places, so this PR is not already implemented on `main`, and the underlying bug remains plausible. The proposed one-line change from the PR would also be incomplete against current code because zero usage is still filtered before accumulation and again before persistence.

## Evidence

- **Current subscribe path still drops zero-usage snapshots before accumulation:** `resolveAssistantUsage()` only returns a usage object when `hasNonzeroUsage(usage)` is true, and `getUsageTotals()` still returns `undefined` when every accumulated counter is `0`. That means a provider response like `{ input: 0, output: 0, total: 0 }` is still treated as no usage on `main`.
  - file: [src/agents/pi-embedded-subscribe.ts:382](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/agents/pi-embedded-subscribe.ts#L382)
