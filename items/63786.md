## Summary

Current main does not yet implement the central Copilot replay fix from this PR. The GitHub Copilot plugin marks Claude models with `dropThinkingBlocks`, but both the session-history replay path and mid-turn stream wrapper still call the generic `dropThinkingBlocks()` helper, which intentionally preserves the latest assistant thinking blocks. Current tests also assert that GitHub Copilot preserves latest thinking, so the reported regression remains live. The PR should stay open, but it needs review/updates for the later provider-normalization feedback and for the extra ACP/guard-script changes that appear partially superseded by the current generic conversation-resolution path.

## Best Possible Solution

Keep this PR open and ask the author to update it against current main: retain the Copilot-specific replay sanitizer and regression tests, choose that sanitizer from a normalized provider id or policy-derived strategy, align context-pruning estimates with the same Copilot behavior, and either split or justify the ACP/guard-script changes against the newer `resolveInboundConversationResolution` path.
