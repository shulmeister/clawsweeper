## Summary

Current `main` at `9ece33c50506ba031df382f69bde65dfda19bea5` does not implement a per-run embedded cost cap. The config/schema exposes heartbeat controls like `timeoutSeconds`, `lightContext`, and `isolatedSession`, but no `maxCostPerRun`; the heartbeat path dispatches straight into `getReplyFromConfig` without a pre-dispatch cost check; and isolated cron runs only compute `estimatedCostUsd` after the run finishes.

## Evidence

- **No config field for per-run cost cap:** `AgentDefaultsConfig["heartbeat"]` lists `every`, `activeHours`, `model`, `session`, `target`, `ackMaxChars`, `timeoutSeconds`, `lightContext`, `isolatedSession`, and `includeReasoning`, but no `maxCostPerRun` or similar budget field.

  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Validation schema also lacks cost-cap support:** `HeartbeatSchema` validates the same heartbeat keys and is `.strict()`, so current config parsing would reject an added `maxCostPerRun` key unless code changed. No cost-cap field is present.
  - file: [src/config/zod-schema.agent-runtime.ts:19](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/config/zod-schema.agent-runtime.ts#L19)
