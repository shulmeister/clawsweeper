## Summary

Keep #62364 open. Current main has relevant startup/model-catalog mitigations, but the reported problem is not proven fixed: `models.list` still waits on the cold gateway model catalog, `chat.history` can share that same wait, provider catalog hooks still run serially, and normal Gateway discovery has no default timeout outside live modes. A same-day issue comment also reports 38-56s `models.list` round trips on v2026.4.24 after the related #70186 fix path.

## Best Possible Solution

Keep the issue open. The best maintainer path is to make the Gateway model-catalog path bounded and less blocking: avoid `chat.history` depending on the full catalog for a thinking default, use stale/static catalog rows where possible, parallelize or time-box provider catalog hooks in normal Gateway operation, and add regression coverage with a deliberately slow provider catalog hook.
