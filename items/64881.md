## Summary

Keep open. Current `main` still treats `fullAccessAvailable` as true whenever elevated exec is configured and allowed, even when no runtime/provider probe supplied that signal. The present tests also codify that optimistic fallback, so the issue's requested ambiguity-safe behavior and provider/runtime-backed regression coverage are not implemented yet.
