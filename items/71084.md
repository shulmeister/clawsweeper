## Summary

Keep #71084 open. Current main still explicitly excludes CLI providers from preflight compaction and memory flush, the regression test asserts the CLI memory-flush skip, and docs document that CLI backends skip the flush. Related PR #71081 was closed unmerged in favor of this discussion, so there is no shipped implementation or canonical superseding item to close against.

## Best Possible Solution

Keep the issue open for maintainer direction. If maintainers accept CLI-backed long-lived agents as eligible for this housekeeping, the best implementation is the narrow core change from #71081: remove the CLI gates in `src/auto-reply/reply/agent-runner-memory.ts`, update the CLI-provider memory-flush test to assert flush/session rotation, add or adjust preflight-compaction coverage for CLI providers, and update the session-management/config docs so the supported behavior is explicit.
