## Summary

Keep open. Current main does not implement the PR's main behavior: ACP task completions on deliverable channels still go through visible background-task delivery, ACP session turns do not get the proposed normalization prompt, and completion announces still seed from cached round-one text before attempting live output capture. The PR also has substantive unresolved review concerns, so it is not a conservative close or merge candidate.

## Best Possible Solution

Keep this PR open, but require a rebase/rework before maintainer consideration. The safe path is to split or coordinate it with the broader #65242 completion-delivery effort: preserve a single, persisted completion-delivery routing contract for parent-owned ACP runs, prefer live terminal output without trusting stale interim chatter, escape or omit untrusted findings from the steer message so structured internalEvents remain authoritative, and add regression tests covering deliverable Telegram/Discord channels, restart persistence, blocked outcomes, and ACP extraSystemPrompt behavior if that product guidance is still desired.
