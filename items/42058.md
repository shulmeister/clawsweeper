## Summary

PR #42058 is a duplicate/superseded implementation path for configurable image compression. Current main still lacks the feature, but open PR #42259 explicitly tracks the same remaining user problem with image-tool quality/max-side config, while #42058 has unresolved review issues in production logging, config semantics, alpha handling, and native prompt-image behavior.

## Best Possible Solution

Close #42058 as superseded by #42259 and keep one canonical implementation thread for image-tool compression quality/max-side configuration. The canonical path should fix its review comments, keep the config contract simple and documented, preserve existing media behavior such as PNG alpha handling, and add targeted tests for configured quality and side limits before merge.
