## Summary

Current main covers the reported WebChat image-drop path: configured provider/model refs are resolved to provider plus bare model id, qwen3.6-plus is image-capable on supported Qwen Standard endpoints, the Gateway checks the catalog input capability before parsing attachments, and attachment parsing now preserves images instead of emitting the old silent-drop behavior.

## Best Possible Solution

Close this item as implemented for the Gateway/WebChat attachment-drop path. Keep the current normalized model-resolution and attachment-preservation behavior, and handle remaining Qwen Coding Plan availability separately in the canonical Qwen endpoint/catalog issues.
