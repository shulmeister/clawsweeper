## Summary

Keep open. The issue discussion points to PR #59315 as the intended fix, but that PR was closed unmerged, and current `main` still lacks the scroll-body modal structure that PR introduced. The current Web UI approval views still render long content directly in the card ahead of the action row, so there is not strong evidence that the reported off-screen/blocking approval bug is fixed on `main`.

## Evidence

- **Current exec approval modal still uses the old structure:** `renderExecApprovalPrompt` renders the approval content directly inside `.exec-approval-card`, followed by `.exec-approval-actions`; there is no `.exec-approval-body` wrapper to isolate scrolling from the action row.
