## Summary

Keep open. Current main still does not implement a cron-scoped `tools.cron.tools.deny` policy or make cron sessions inherit `tools.subagents.tools.deny`. Cron has global tool policy support and a per-job `toolsAllow` allowlist, plus delivery-mode-specific message-tool behavior, but the requested system-wide cron denylist remains missing.

## Best Possible Solution

Keep tracking the issue until a maintainer chooses and implements the product direction. A complete fix would add a cron-scoped tool policy such as `tools.cron.tools.allow/deny` or explicitly define inheritance from `tools.subagents.tools`, wire it into isolated cron tool construction alongside global/provider/agent policy and runtime `toolsAllow`, then update config schema/types, docs, and cron tool-policy tests covering denied `message` for announce and none delivery modes.
