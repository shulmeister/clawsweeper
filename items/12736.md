## Summary

Current main still does not implement a cron-scoped `tools.cron.tools.deny` config option or make `cron:<jobId>` sessions inherit `tools.subagents.tools.deny`. The existing cron surface supports global tool policy, subagent-only policy, delivery-mode message-tool behavior, and per-job `toolsAllow`, but not the requested system-wide cron denylist.

## Best Possible Solution

Keep the issue open until a maintainer chooses the product direction. A complete fix would add a cron-scoped tool policy such as `tools.cron.tools.allow/deny` or explicitly define inheritance from `tools.subagents.tools`, wire it into isolated cron tool construction alongside global/provider/agent/group/sandbox policy and per-job `toolsAllow`, then update config schema/types, docs, help metadata, and cron tool-policy tests covering denied `message` in announce and none delivery modes.
