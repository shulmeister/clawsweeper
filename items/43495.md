## Summary

Keep #43495 open. Current main still lacks `<notts>` or equivalent visual-only TTS preprocessing: TTS supports `[[tts...]]` directives and hidden speech blocks, but it does not remove visible-only blocks from spoken text while preserving them in chat. The PR addresses a real product gap, but it should be rebased and narrowed because the provided review context flags unresolved implementation gaps and the PR bundles unrelated gateway, macOS, media, and TTS changes.

## Best Possible Solution

Keep the item open, but ask for a refreshed and focused PR against current main. The best path is to implement visual-only TTS preprocessing in the current speech-core TTS path, add regression coverage for visible/spoken text splitting and tagged-mode behavior, update `docs/tools/tts.md`, and split or drop the unrelated gateway/macOS/media changes. The unresolved bot review comments should be addressed before merge.
