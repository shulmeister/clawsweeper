## Summary

Keep PR #69982 open. Current main still has the document click listener cleanup bug in the delete confirmation popover, and the PR is a focused fix attempt. It should likely be revised before merge because the existing-popover toggle path also removes the popover without removing the listener, and there is no regression test for listener cleanup yet.

## Best Possible Solution

Keep the PR open and ask for a small update: use one cleanup/dismiss path for Cancel, Delete, outside click, and same-button toggle dismissal, then add a focused grouped-render test that proves document click listeners are removed for each dismissal path.
