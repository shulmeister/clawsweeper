## Summary

Keep #69533 open. Current main still hardcodes the Dream Diary narrative system prompt and input prompt, the memory-core config schema rejects unknown dreaming keys such as `prompt`, docs list only `enabled` and `frequency` as user-facing dreaming settings, and the Dreams UI only patches the enabled toggle. This is a real bundled memory-core feature request, not a ClawHub close, because there is no current extension point to customize memory-core's internal narrative prompt without changing this repo.

## Best Possible Solution

Keep the issue open. A good implementation would add a typed memory-core dreaming narrative prompt/template contract under `plugins.entries.memory-core.config.dreaming`, validate it in `openclaw.plugin.json`, resolve it through `src/memory-host-sdk/dreaming.ts`, apply it in `extensions/memory-core/src/dreaming-narrative.ts` with safe placeholder expansion for date/time/count/themes, expose it in the Dreams/settings UI if product wants a UI knob, and document the new setting at https://docs.openclaw.ai/reference/memory-config and https://docs.openclaw.ai/concepts/dreaming with focused tests for schema, resolver, prompt construction, and UI patching.
