## Summary

Keep #13479 open. Current main still only partially covers the request: the cron dashboard renders the prompt/system payload inline and the cron model persists `description`, but clicking a cron job still selects run history, and the main cron job card does not render `job.description` as the requested human-readable summary. The existing Apr 26 Codex review comment reached the same conclusion, and the relevant cron UI/protocol files are unchanged from that reviewed state to current main.

## Best Possible Solution

Keep the issue open. Implement this in the Control UI cron surface: add selected-job detail or expansion behavior in `ui/src/ui/views/cron.ts`, render `job.description` as the human-readable summary next to the existing System/Prompt/Delivery payload, preserve the separate run-history controls, add English i18n copy, and cover the behavior with focused `ui/src/ui/views/cron.test.ts` assertions.
