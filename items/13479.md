## Summary

Keep open. Current main partially satisfies the request by rendering cron payload text inline on the Web UI job card, but clicking a job still only loads run history and the job description/summary is not displayed in the job list/detail surface. The prior Codex review comment on #13479 reached the same conclusion, and current main still matches that behavior.

## Best Possible Solution

Keep #13479 open. The best implementation path is to add a selected-job detail or expansion flow in the Control UI cron view, surface `job.description` as the requested summary alongside the prompt/system payload, add English i18n strings and focused `ui/src/ui/views/cron.test.ts` coverage, and preserve the existing run-history controls.
