## Summary

Keep #13479 open. Current main still only partially satisfies the request: the cron job cards show payload text inline and clicking a job loads run history, but there is no selected-job detail/expansion flow and the job `description` summary is not rendered in the cron dashboard surface.

## Best Possible Solution

Keep #13479 open. The best implementation path is to add a selected-job detail or expansion state in the Control UI cron view, render `job.description` as the human-readable summary alongside the existing prompt/system payload display, add English i18n strings, preserve the existing run-history loading controls, and cover the behavior in focused `ui/src/ui/views/cron.test.ts` tests.
