## Summary

Keep open. Current main still defaults fresh QMD collection adds to `--glob`, and the retry path only switches to `--mask` when QMD rejects the flag. That does not cover the PR's reported QMD 2.1.0 compatibility problem where collection patterns can be accepted but registered incorrectly. The PR overlaps #65480/#65481, but this PR has its own repro, in-the-wild confirmation, and useful test changes, so it should not be auto-closed as maintainer cleanup.

## Best Possible Solution

Keep this PR open or explicitly supersede it with one canonical QMD compatibility PR. The best product/code path is to prefer `--mask` for fresh `qmd collection add` calls, preserve fallback for binaries that actually require `--glob`, add regression coverage for silent `--glob` pattern loss, and update QMD memory docs/changelog wording to match the chosen compatibility contract.
