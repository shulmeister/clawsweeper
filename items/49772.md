## Summary

Keep open. Current `main` has partial multi-agent Feishu fan-out, but it does not implement agent-to-agent group chat well enough: only the active mentioned agent gets a real Feishu reply dispatcher, while the other broadcast agents run with a no-op dispatcher and do not reply back into the group.

## Evidence

- **Feishu broadcast is single-visible-reply only:** `extensions/feishu/src/bot.ts:1056-1145` sets one `activeAgentId`; that agent gets `createFeishuReplyDispatcher(...)`, while non-active agents are routed through a `noopDispatcher` whose `sendBlockReply`/`sendFinalReply` return `false`. That is not autonomous bot-to-bot chatting in the group.
  - file: [extensions/feishu/src/bot.ts:1056](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/extensions/feishu/src/bot.ts#L1056)
  - command: `nl -ba extensions/feishu/src/bot.ts | sed -n '1038,1182p'`
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Broadcast still depends on a user mention:** `extensions/feishu/src/bot.broadcast.test.ts:189-229` verifies Feishu broadcast dispatch happens when the bot is mentioned and is skipped when it is not mentioned. That matches fan-out from one user-triggered turn, not ongoing autonomous inter-agent conversation.
  - file: [extensions/feishu/src/bot.broadcast.test.ts:189](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/extensions/feishu/src/bot.broadcast.test.ts#L189)
  - command: `nl -ba extensions/feishu/src/bot.broadcast.test.ts | sed -n '180,240p'`
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Feishu public config surface lacks bot-to-bot controls:** `docs/channels/feishu.md:56-72` and `extensions/feishu/src/config-schema.ts:223-224` document/schema only `groupPolicy` and `requireMention` for group behavior. There is no Feishu `allowBots` or inter-agent room mode like the issue requests.
  - file: [docs/channels/feishu.md:56](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/docs/channels/feishu.md#L56)
  - command: `nl -ba docs/channels/feishu.md | sed -n '35,90p'`
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Existing broadcast model isolates agents instead of letting them converse:** `docs/channels/broadcast-groups.md:164-172` says broadcast groups only change which agents run when a message is already eligible, and each agent keeps separate conversation history that does not include the other agents' messages.
  - file: [docs/channels/broadcast-groups.md:164](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/docs/channels/broadcast-groups.md#L164)
