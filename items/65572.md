## Summary

Current `main` already handles this race. `GatewayChatClient.loadHistory()` now retries `chat.history` during gateway startup for up to 60 seconds when the gateway returns retryable `UNAVAILABLE`, and the shipped changelog explicitly records this fix as `#69164` under release `2026.4.20`.

## Evidence

- **Current TUI gateway client retries startup `chat.history` failures:** `GatewayChatClient.loadHistory()` loops until success, retrying retryable `UNAVAILABLE` errors for `chat.history` within a 60s startup window and sleeping based on `retryAfterMs` (bounded to 100-5000ms) before retrying.
