## Summary

Current main already contains Feishu reply-dispatch logic and regression coverage that match this report: plain-text finals are sent through `sendMessageFeishu`, and Feishu now forces `disableBlockStreaming: true` specifically to avoid silent final-reply drops. The same implementation is present in the shipped `v2026.4.22` release.
