## Summary

Keep open. Current main still contains the exact reported array-state data-loss path in both device and node pairing: `loadState()` accepts parsed arrays through `pending ?? {}` / `paired ?? {}`, then later persists those values through `JSON.stringify`, which drops UUID string keys on arrays. The related PR discussion identifies the right fix direction, but the linked PRs are not merged into current main.

## Best Possible Solution

Keep the issue open and land a narrow runtime normalization fix in both `src/infra/device-pairing.ts` and `src/infra/node-pairing.ts`, preferably through a shared helper that coerces non-record JSON values such as arrays to `{}`. Add regression coverage that seeds `pending.json` and `paired.json` with `[]` for device pairing, and either a parallel node-pairing test or shared-helper coverage plus a node loader check. PR #63081 is the closest current implementation vehicle, with #63078 providing the useful round-trip test shape.
