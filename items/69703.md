## Summary

Close as superseded. Current main handles the documented `models.providers.ollama` + `api: "ollama"` path through the bundled native Ollama provider and runtime stream registration, while the PR's proposed OpenAI-completions alias conflicts with the current docs. The remaining arbitrary custom/multiple Ollama provider-id work is already tracked by narrower open items such as #34252 and #62735.

## Best Possible Solution

Close this PR as superseded. Keep the shipped native Ollama provider path for `api: "ollama"`, keep `api: "openai-completions"` as the explicit documented legacy/proxy mode, and handle multi-Ollama/custom-provider-id support in the existing canonical issues instead of merging a generic transport alias for native Ollama.
