## Summary

Keep open. Current main still does not implement agent-scoped internal hook policy overrides, and the open PR remains the active implementation attempt rather than obsolete cleanup. The PR discussion also identifies unresolved behavior concerns that need author/maintainer follow-up before this can be treated as shipped or superseded.

## Best Possible Solution

Keep the PR open, rebase it onto current main, and either implement the agent-scoped internal hook policy with clear semantics or have a maintainer decide against the product surface. A complete path would add typed config and zod/generated schema support for `agents.defaults.hooks` and `agents.list[].hooks`, wire runtime agent identity into internal hook dispatch only for agent-scoped events, document exact or wildcard event matching, and add regression tests for global-only behavior, default overlays, per-agent overrides, catch-all registrations, and non-agent events.
