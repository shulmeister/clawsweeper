## Summary

Keep PR #38670 open. Current main still does not implement agent-scoped internal hook policy overrides: internal hook dispatch remains global-only, the config/types/schema lack `hooks.internal.events`, `agents.defaults.hooks`, and `agents.list[].hooks`, and the docs still describe only global internal hook configuration plus the separate plugin hook surface. The related merged `before_agent_reply` plugin hook is useful context but does not supersede this core config/runtime policy proposal.

## Best Possible Solution

Keep PR #38670 open for maintainer judgment. If OpenClaw wants this capability, the best path is a core config/runtime change that adds explicit strict schema/types for `hooks.internal.events`, `agents.defaults.hooks`, and `agents.list[].hooks`; resolves global/default/per-agent precedence only for real agent-scoped events; documents exact event matching; and adds focused tests for precedence, hook-name gating, and non-agent hook behavior. If maintainers reject the product direction, close it explicitly with that rationale rather than automated cleanup.
