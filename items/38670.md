## Summary

Keep PR #38670 open. Current main still does not implement agent-scoped internal hook policy overrides: internal hook dispatch is global-only, config/schema lacks `hooks.internal.events`, `agents.defaults.hooks`, and `agents.list[].hooks`, and resolved agent config does not carry hook policy. The related shipped `before_agent_reply` plugin hook is a different plugin lifecycle surface and does not supersede this core internal-hook policy change.

## Best Possible Solution

Keep PR #38670 open for maintainer judgment. If OpenClaw wants this capability, implement it as a core config/runtime change with strict policy types and schemas for `hooks.internal.events`, `agents.defaults.hooks`, and `agents.list[].hooks`; thread config plus real runtime agent identity into internal hook dispatch; record hook names for named-hook gating; apply agent overlays only to agent-scoped events; document exact event matching; and add focused tests for precedence, named hook gating, non-agent hook behavior, and backward compatibility. If maintainers reject the product direction, close it explicitly with that rationale.
