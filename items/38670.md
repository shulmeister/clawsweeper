## Summary

Keep open. Current main at 73e21511075d7ed5651faae305f439d65b6ef47c still does not implement agent-scoped internal hook policy overrides; the PR remains a coherent core config/runtime proposal rather than an obsolete duplicate, ClawHub-only item, or stale cleanup candidate.

## Best Possible Solution

Keep PR #38670 open for maintainer judgment. If OpenClaw wants agent-scoped internal hook policy, the best implementation path is a core change that adds explicit policy types, zod schema support for `hooks.internal.events`, `agents.defaults.hooks`, and `agents.list[].hooks`, runtime policy resolution only for real agent-scoped events, documented event-matching semantics, and focused tests for default/per-agent precedence and non-agent hook behavior. If maintainers reject this product direction, close it explicitly with that rationale rather than automated cleanup.
