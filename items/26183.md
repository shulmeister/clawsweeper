## Summary

Keep open. Current main still has the WhatsApp ack reaction send path, but it remains fire-and-forget and is not paired with post-reply cleanup. The config schema also still rejects the requested lifecycle keys, while docs explicitly scope `messages.removeAckAfterReply` to Slack, Discord, and Telegram rather than WhatsApp.

## Best Possible Solution

Keep the issue open and implement WhatsApp ack lifecycle support in the bundled WhatsApp plugin: make ack sending return a trackable success promise, honor `messages.removeAckAfterReply` or a clearly documented WhatsApp-specific option, clear the reaction with `sendReactionWhatsApp` using an empty emoji after successful reply delivery, and update schema/types/docs plus WhatsApp monitor tests.
