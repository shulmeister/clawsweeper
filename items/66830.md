## Summary

Current main still has the reported reserve-token asymmetry: memoryFlush and preflight thresholds are computed from `reserveTokensFloor`, while the documented/user-facing `reserveTokens` knob is preserved in config and used by Pi compaction settings. The linked fix PRs #67297 and #69379 are open and unmerged, so this issue remains a valid bug report rather than a cleanup close.

## Best Possible Solution

Keep #66830 open until #69379 or an equivalent patch lands on main. The fix should make preflight and memoryFlush gates resolve an effective reserve from `max(agents.defaults.compaction.reserveTokens, reserveTokensFloor)`, preserve plugin plan compatibility, update threshold logs, and add regression tests for `reserveTokens > reserveTokensFloor`.
