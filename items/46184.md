## Summary

Keep open. Current main still routes Gemini CLI OAuth token exchange, userinfo, and Code Assist project discovery through `fetchWithSsrFGuard` without opting into `trusted_env_proxy`, while the guard’s default strict mode explicitly ignores env proxy settings. The linked PR #47518 is a direct attempted fix, but it is still open/unmerged in the provided context and current main has no equivalent implementation.
