## Summary

Current OpenClaw already supports custom workspace prompt injection through the supported `agent:bootstrap` hook surface, including workspace/plugin hooks that can append arbitrary files before Project Context is built.

## Evidence

- **Hook API exposes mutable bootstrap file list:** The hooks docs explicitly document `agent:bootstrap` and say `context.bootstrapFiles` is a mutable array; the same page also documents workspace hooks and plugin hook packs as supported extension surfaces.
  - file: [docs/automation/hooks.md:123](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/docs/automation/hooks.md#L123)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Bootstrap list is passed through hook overrides before prompt assembly:** `applyBootstrapHookOverrides` wraps the bootstrap file list in an `agent:bootstrap` event and returns the mutated `bootstrapFiles` array.
  - file: [src/agents/bootstrap-hooks.ts:7](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/bootstrap-hooks.ts#L7)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Runtime does not require bootstrap files to be from the fixed default set after hook mutation:** After hook overrides, `sanitizeBootstrapFiles` only requires a valid `path`; it does not re-check the file basename against the built-in bootstrap filename allowlist.
  - file: [src/agents/bootstrap-files.ts:147](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/bootstrap-files.ts#L147)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Injected Project Context builder accepts generic file entries:** The bootstrap context builder injects each file from `path` and `content`, so a hook-added file like `PROJECTS.md` will be included like any other Project Context entry.
  - file: [src/agents/pi-embedded-helpers/bootstrap.ts:282](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/pi-embedded-helpers/bootstrap.ts#L282)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Docs describe bootstrap-file mutation as a supported feature:** The system prompt docs state that internal hooks can intercept `agent:bootstrap` to mutate or replace injected bootstrap files before prompt construction.
  - file: [docs/concepts/system-prompt.md:137](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/docs/concepts/system-prompt.md#L137)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)

## Risks / Open Questions

- The bundled `bootstrap-extra-files` hook is narrower than the issue proposal because it only auto-loads recognized bootstrap basenames; operators wanting arbitrary filenames like `PROJECTS.md` need a custom workspace/plugin hook instead of a declarative config key.
- Subagent sessions still pass through session filtering before hook overrides, so a custom hook must intentionally re-add any extra context needed for subagents.

## Close Comment

Closing from a Codex review. Current OpenClaw already supports this capability through the supported `agent:bootstrap` hook surface rather than a new core config key. The docs mark `context.bootstrapFiles` as a mutable array for bootstrap events (`docs/automation/hooks.md:123`), `src/agents/bootstrap-hooks.ts:7` passes the bootstrap list through that hook before Project Context is built, and the injection path only requires a valid `path` after hook mutation (`src/agents/bootstrap-files.ts:147`, `src/agents/pi-embedded-helpers/bootstrap.ts:282`). In practice, a workspace or plugin hook can append `PROJECTS.md` or similar domain files today. This behavior is already present in the shipped `v2026.4.22` code (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
