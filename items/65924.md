## Summary

Current main already prevents this retry-corruption path from poisoning the next provider request: failed assistant turns are removed from replay, and session-history repair drops errored assistant tool-call turns before rebuilding strict provider history. Moonshot/Kimi native tool-call IDs are intentionally preserved, so the fix is replay filtering/repair rather than forcing Moonshot into strict ID rewriting.

## Best Possible Solution

Close #65924 as implemented on current main. Keep the replay-time failed-turn filtering and Moonshot native ID preservation; broader Moonshot duplicate-ID reports such as #51148/#51593 can remain as separate trackers if they still reproduce outside this retry-corruption pattern.
