## Summary

Keep open. Current `main` (`ac063568d354c14e0a0a43bcffb9725bb3fca651`, still present in release `v2026.4.22`) has the same unsafe `job.payload.kind` dereferences the PR is trying to guard, and the cron load/list path can still surface malformed jobs with no repaired `payload` before the UI/CLI render them. The PR is still relevant, although it likely needs follow-up on the edit-form fallback and test coverage before merge.

## Risks / Open Questions

- The PR's `payloadKind: job.payload?.kind ?? "agentTurn"` fallback can silently reclassify a malformed legacy job if a user opens it in the form and saves it.
- `CronJobBase` still declares `payload` as required in `src/cron/types-shared.ts:15`, so TypeScript will not help catch future unguarded access sites.
