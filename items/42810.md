## Summary

Keep PR #42810 open. Current main still does not contain the PR's ACP audit logger, audit interfaces, JSONL storage/query path, or AcpSessionManager audit dependency. Main has narrower actor logging for config/update control-plane writes and a task-ledger health audit, but those do not cover ACP session lifecycle security audit events. The PR discussion also shows substantive unresolved design/correctness review comments, so this should be a maintainer/security design decision rather than a conservative cleanup close.

## Best Possible Solution

Keep the PR open for maintainer/security product judgment, or supersede it with a canonical audit-design issue/PR if maintainers want to consolidate audit work. The best implementation would define the core audit contract across ACP, gateway control-plane writes, task/background execution, and tool invocation; make audit writes best-effort so they cannot break session lifecycle operations; provide a real configured sink instead of an unreachable default; document retention/query behavior; and add focused regression tests for session init/close success, failure, closeSession compatibility, retention, buffer failure handling, and query consistency.
