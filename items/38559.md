## Summary

Keep open. Current `main` still routes Feishu mp4 outbound messages as `msg_type: "media"` but only sends `content: { file_key }`; it does not implement the PR's claimed `image_key`/`file_name` payload, so there is not enough evidence to close this as already implemented or obsolete. The PR also appears partially superseded by later host-edit work, but the Feishu video fix itself is not on `main`.
