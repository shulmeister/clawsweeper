## Summary

Current main already surfaces tool-call activity on messaging channels via verbose mode: `/verbose on` sends per-tool metadata bubbles with argument detail, `/verbose full` forwards tool output after completion, and preview-streaming channels also show live tool-progress updates.

## Risks / Open Questions

- The current implementation is message-based verbose surfacing, not richer per-channel UI like collapsible embeds or a separate debug thread/channel.
- If the requester wanted always-on tool visibility without using `/verbose`, that narrower product request is not implemented here.
