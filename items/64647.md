## Summary

Keep open: current main still lacks the PR's per-agent `agents.list[].memory` config surface and per-agent memory backend resolution. The PR is a focused implementation candidate for the linked rollout request, and the provided discussion supports the design while identifying a small type-check follow-up rather than showing that the work is obsolete.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is to refresh it on current main, fix the reported typing issue, preserve the intended boundary (`agents.list[].memory.backend` and `agents.list[].memory.qmd.*` with `memory.citations` remaining global), regenerate the config schema, and land it with the included schema/runtime/docs regression coverage or an equivalent maintainer-owned patch.
