## Summary

PR #64660 should stay open. Current main partially overlaps with the streaming-usage part, but it does not implement the PR's core local-LLM plain-text tool-call promotion, the timeout-compaction fallback, or the proposed local endpoint capability attribution. The PR also has substantive review concerns that need maintainer/contributor follow-up rather than automated closure.

## Best Possible Solution

Keep the PR open for maintainer review, but ask for a rebase onto current main and a narrowed implementation: safe local/provider-scoped plain-text tool-call promotion with tests for Gemma markers, Qwen/function_calls XML, false-positive JSON answers, finish_reason length/error preservation, streaming delta hygiene, and text ordering; separately decide whether generic localhost streaming-usage attribution and timeout message-count estimation are still desired.
