## Summary

Keep #71584 open. Current main still rejects `/think max` for Ollama unless a provider thinking profile advertises `max`, and the bundled Ollama plugin does not currently provide such a profile or forward Ollama `options.reasoning_effort`. The related Ollama reports in the provided context cover adjacent native `think`/usage/config behavior, not this `max` reasoning-effort gap.

## Best Possible Solution

Keep the issue open. The best fix likely belongs in the bundled Ollama provider: expose a model-aware `resolveThinkingProfile` that includes `max` for Ollama cloud models known or discovered to support provider max reasoning, and map/forward that level through native Ollama payloads, likely via `options.reasoning_effort`. If user-declared compat is intended to control this, also align `ModelCompatSchema`, generated schema/docs, and the reasoning-effort resolver so `supportedReasoningEfforts: ["max"]` is preserved and honored.
