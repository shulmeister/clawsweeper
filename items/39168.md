## Summary

Keep PR #39168 open. The reported `mcp_exec`/`mcp_read`/`mcp_write` dispatch problem is still plausible on current main, but the PR cannot merge as-is because it patches obsolete `@mariozechner/pi-agent-core@0.55.3`; current main uses 0.70.2 and already has an OpenClaw stream-normalization wrapper where this repair should land.

## Best Possible Solution

Keep the PR open but request rework against current main: add guarded `mcp_` prefix normalization in `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts`, only when the stripped name resolves to an allowed registered canonical tool and the raw name does not already resolve. Preserve legitimate `mcp__bundle__*` names, avoid carrying a stale pnpm patch, and add focused tests for `mcp_exec`, `mcp_read`, `mcp_write`, nonexistent `mcp_foo`, bare `mcp_`, and registered `mcp__bundle__*` names.
