## Summary

Keep PR #39168 open. Current main still does not show an `mcp_` tool-name normalization path, while the PR branch patches an obsolete `@mariozechner/pi-agent-core` version and needs rework against OpenClaw's current stream-normalization wrapper instead of merging as-is.

## Best Possible Solution

Keep the PR open, but ask for a rework against current main: add guarded `mcp_` prefix normalization in `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts` only when the stripped name resolves to an allowed canonical tool, and add regression coverage in the existing embedded-runner normalization tests. Avoid carrying a stale pnpm patch for pi-agent-core 0.55.3.
