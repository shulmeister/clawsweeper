## Summary

Keep PR #39168 open. The PR’s current patch is stale because main uses @mariozechner/pi-agent-core 0.70.2 and has no pi-agent-core patchedDependency, but the underlying mcp_exec/mcp_read/mcp_write dispatch bug still appears unimplemented on current main. The right fix is not this obsolete 0.55.3 pnpm patch; it should land in OpenClaw’s existing tool-call normalization wrapper with guarded allowed-tool resolution.

## Best Possible Solution

Request rework against current main: add guarded mcp_ prefix normalization in src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts, preserving real mcp__bundle__* tool names and only rewriting when the stripped name resolves to an allowed registered tool. Add focused tests for mcp_exec, mcp_read, mcp_write, nonexistent mcp_foo, bare mcp_, and registered mcp__bundle__* names. Do not carry the stale pnpm patch.
