## Summary

Keep PR #39168 open. The submitted dependency patch is obsolete against current main, but the reported `mcp_exec` / `mcp_read` / `mcp_write` dispatch failure is not implemented on main either. Current OpenClaw has the right stream-normalization seam before pi-agent-core dispatch, but it still lacks a guarded `mcp_` stripped-name lookup.

## Best Possible Solution

Keep the PR open, but rework it against current main by adding guarded `mcp_` normalization in `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts`. Resolve the raw name first, then strip `mcp_` only when the suffix is non-empty, not a bundled `mcp__bundle__*` name, and resolves unambiguously to an allowed tool such as `exec`, `read`, or `write`. Add focused tests for `mcp_exec`, `mcp_read`, `mcp_write`, nonexistent `mcp_foo`, bare `mcp_`, and registered `mcp__bundle__*` names. Do not use a pnpm dependency patch for this on current main.
