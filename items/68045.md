## Summary

Keep PR #68045 open. Current main has not implemented the PR's central fixes: grouped Control UI bubbles still derive display text from normalized message content instead of the shared `extractText`/`processMessageText` path, inbound metadata stripping still only handles structured sentinels and timestamp prefixes, and group-level rendering still lacks an all-empty suppression guard. The PR also has unresolved P1 review feedback about preserving normalized assistant directive cleanup and stripping multiline system-event continuations safely, so it remains useful but needs revision before merge rather than closure.

## Best Possible Solution

Keep the PR open and revise it against current main. The best fix is to centralize grouped bubble text extraction so it preserves `normalizeMessage` assistant directive/media cleanup while also applying the shared assistant/user stripping contract; extend `stripInboundMetadata` and `stripLeadingInboundMetadata` to remove only verified injected system-event and internal-instruction lines, including continuation lines from `drainFormattedSystemEvents`; add the group-level empty suppression guard; and cover these paths with focused tests in `ui/src/ui/chat/grouped-render.test.ts`, `ui/src/ui/chat/message-extract.test.ts`, and `src/auto-reply/reply/strip-inbound-meta.test.ts`.
