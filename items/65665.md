## Summary

Kept open. Current `main` shows Telegram inbound images are saved under fresh UUID-backed filenames and the media-understanding path reads file bytes directly before calling providers, but I did not find a targeted fix or regression test for sequential Telegram images returning stale vision results. With a recent report, no logs/sample images, and no strong reproduction proof either way, this is not a safe conservative close.
