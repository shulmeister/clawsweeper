## Summary

Keep open. Current `main` still treats setup discovery/runtime as provider- and CLI-backend-specific, while the generic plugin configuration step in onboarding explicitly skips sensitive plugin fields like API keys and sends users to `openclaw config set` or the Web UI instead. That means the requested first-class tool-plugin credential/setup hook is not implemented well enough to close, and VISION points toward extending the plugin API in core when a capability cannot yet be built cleanly as a plugin.
