## Summary

Keep this open. Current main includes useful partial fixes for provider-scoped request timeouts, auth-profile timeout poisoning, deferred context-engine maintenance, QA readiness/cleanup, and Telegram polling recovery, but it still does not expose the requested phase-aware timeout attribution through the live gateway or agent.wait surface. The draft OpenClaw PR #65511 that claimed the full fix was closed unmerged, while the related lossless-claw work remains external.

## Best Possible Solution

Keep this open until the OpenClaw side lands an additive timeout-attribution contract. The best path is to rework the useful pieces of #65511 into focused PRs: add phase-aware timing/timeout metadata to lifecycle events, gateway dedupe snapshots, agent.wait responses, protocol schema/docs, and tests; preserve the existing auth-profile timeout guard, provider-scoped request timeout support, deferred maintenance behavior, QA cleanup/readiness behavior, and Telegram polling recovery; and track remaining owner-specific scheduling work through #54488 and the external lossless-claw PR.
