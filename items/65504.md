## Summary

Keep this issue open. Current main has meaningful partial fixes around auth-profile timeout poisoning, deferred context-engine maintenance, QA gateway cleanup, and Telegram polling recovery, but the central requested behavior, phase-aware timeout attribution through the live gateway/agent.wait surface, is not present. The draft PR that claimed to close this issue was closed unmerged, and the related lossless-claw work remains outside this repo.

## Best Possible Solution

Keep this open until the OpenClaw side lands an additive timeout-attribution contract. The best path is to rework the useful parts of #65511 into focused PRs: add phase-aware run timing/timeout metadata to lifecycle, dedupe, agent.wait, protocol docs, and tests; preserve the existing auth-profile timeout guard; keep the deferred maintenance and QA cleanup behavior; and link #54488 plus the lossless-claw PR for any remaining owner-specific scheduling work.
