## Summary

Keep #37706 open. Current main still treats Feishu/Lark raw content containing `@_all` as a bot mention, exposes that through `mentionedBot`, and then uses that flag for group `requireMention` gating. The provided related PR context shows multiple unmerged fix attempts and an open fix PR, but no shipped/main implementation resolves the bug.

## Best Possible Solution

Keep #37706 open as the canonical Feishu/Lark bug. Land a focused Feishu plugin fix, likely through #44256 or a refreshed equivalent, that removes the raw `@_all` bot-mention shortcut, relies on actual bot mention metadata and post `at` tags, and adds regression coverage for broadcast-only and broadcast-plus-real-bot-mention cases.
