## Summary

Current main still implements the reported false positive: Feishu/Lark raw content containing `@_all` is treated as `mentionedBot=true`, and group mention gating uses that flag to allow processing. The issue should remain open as the canonical bug until the shortcut is removed and covered by regression tests.

## Best Possible Solution

Keep #37706 open as the canonical Feishu/Lark bug. The best fix is a focused Feishu plugin change that removes the raw `@_all` bot-mention shortcut, preserves real bot mentions through top-level mention metadata and post `at` tags, and adds regression coverage for `@_all`/`@all`-only broadcasts plus a broadcast combined with a real bot mention.
