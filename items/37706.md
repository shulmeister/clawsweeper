## Summary

Keep open. Current main still contains the exact Feishu/Lark false-positive path: raw message content containing `@_all` makes `checkBotMentioned()` return true, and `parseFeishuMessageEvent()` passes that value into `mentionedBot`. The issue is actionable, has a concrete repro/root cause, is not maintainer-authored, has no protected labels, and the related open PR #44256 is a fix attempt rather than a shipped or canonical replacement for the original bug report.

## Best Possible Solution

Keep #37706 open as the canonical bug until the Feishu plugin removes the raw `@_all` shortcut from `checkBotMentioned()` and adds regression coverage for `@all`/`@_all` broadcast messages not setting `mentionedBot=true` unless the bot is also explicitly mentioned. The existing open PR #44256 can be used as a starting point if it is still viable, but closing the issue before a fix lands would be premature.
