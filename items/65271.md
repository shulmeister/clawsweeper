## Summary

Keep this PR open. Current main still hardcodes Windows exec to PowerShell and does not expose custom Windows shell selection, while docs/config likewise omit a shell override. The PR is a live implementation candidate, but its proposed OPENCLAW_SHELL override collides with documented runtime-marker semantics and has unresolved review concerns about validating configured shell names/paths. The related open #39413 tracks a broader config-backed version of the same problem, so maintainers should consolidate/review rather than auto-close this PR.

## Best Possible Solution

Keep the PR open for maintainer review, but consolidate the product direction with #39413. The best implementation should choose a stable public surface, likely config-backed tools.exec.shell / per-agent override or an explicitly named env var, validate Windows explicit paths and bare names against PATH/PATHEXT, preserve PowerShell defaults, and update config schema, docs, and regression tests together.
