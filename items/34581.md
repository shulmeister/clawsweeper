## Summary

Keep open. Current main still uses strict `express.json()` parsing for Microsoft Teams webhooks, only handles `413` parse middleware errors, and does not contain the PR's raw-body JSON repair or explicit 200-ack path for irrecoverable malformed JSON. The PR also has substantive unresolved review feedback on the revised repair regex, so it needs maintainer review or a corrected implementation rather than cleanup closure.

## Best Possible Solution

Keep this PR open for Microsoft Teams maintainer review. The best path is to land an equivalent fix in `extensions/msteams/src/monitor.ts` that preserves auth/body limits, parses raw JSON safely, repairs only truly bare invalid escapes, returns a deliberate 200 acknowledgement for unrecoverable Bot Framework payloads when that is the intended Teams behavior, and adds regression tests for malformed escapes, mixed valid/invalid backslashes, and irrecoverable JSON.
