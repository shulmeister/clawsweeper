## Summary

Keep open. The requested stale-total/post-compaction status behavior is not implemented on current main: the active status builder still uses persisted totalTokens even when totalTokensFresh is false and does not consult compactionCheckpoints or tokensAfter. The PR is plausible, but it needs to be rebased onto the current src/status/status-message.ts location and should address the bot-noted unreachable branch.

## Best Possible Solution

Keep the PR open and ask for a small rebase: implement the fallback in src/status/status-message.ts by preferring resolveFreshSessionTotalTokens(entry), falling back to the newest valid compactionCheckpoints[].tokensAfter when totalTokensFresh is false, and add the proposed regression test in src/auto-reply/status.test.ts. The dead else-if branch from the bot review should be removed during that update.
