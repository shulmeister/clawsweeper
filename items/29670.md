## Summary

Keep open. Current main accepts Tailscale Serve identity as a Control UI WebSocket auth method, but the pairing-skip path still does not include a Tailscale-authenticated Control UI operator branch. The provided issue discussion also has recent maintainer guidance explicitly saying this should stay open until a narrower fix with regression tests lands.

## Best Possible Solution

Keep #29670 open. The landable path is a narrow gateway WebSocket fix: add a Tailscale-authenticated Control UI operator pairing bypass alongside the trusted-proxy path, keep device identity enforcement for device-less clients, wire it through message-handler, and add regression tests for the positive device-present Tailscale path and the negative missing-device path. Docs should be reconciled after the product behavior lands.
