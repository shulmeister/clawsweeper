## Summary

Keep open. Current main still lacks the requested channel-level `channels.whatsapp.heartbeat.target` / `targets` contract. The shipped config surface only supports visibility flags, while WhatsApp heartbeat recipient resolution still uses explicit runtime `to`, `--all`, sessions, and `allowFrom` fallback.

## Best Possible Solution

Keep the issue open until maintainers either implement an additive channel-level WhatsApp heartbeat routing contract or explicitly choose a different product direction. A complete fix should add typed `target`/`targets` support in the channel heartbeat config schema/types/docs, validate it for WhatsApp account scope and allowlists, apply it in the WhatsApp heartbeat recipient path without changing alert routing semantics, and add regression tests for single admin, co-admin targets, omitted-target backward compatibility, account-scoped config, and invalid target handling.
