## Summary

Keep #12394 open. Current main still has only heartbeat visibility controls for channel/account heartbeat config, and WhatsApp heartbeat recipients are resolved from explicit runtime opts, session history, and allowFrom/pairing state. The existing agent-level heartbeat target/to setting can route the whole heartbeat delivery, but it does not provide OK-only admin routing while preserving alert/content routing for relevant users.

## Best Possible Solution

Keep #12394 open until maintainers either implement or explicitly reject an additive WhatsApp/channel heartbeat recipient contract. A complete implementation should add typed/schema/generated-doc support for `channels.whatsapp.heartbeat.target` or `targets`, validate those recipients against WhatsApp account and allowlist policy, apply them only to `HEARTBEAT_OK` delivery, preserve alert/content routing, and add regression coverage for single target, multiple targets, omitted-target backward compatibility, per-account overrides, and invalid targets.
