## Summary

Current main still lacks the requested channel-level WhatsApp heartbeat recipient routing. Channel heartbeat config is still visibility-only, WhatsApp reuses that strict shape, the visibility resolver returns only showOk/showAlerts/useIndicator, and heartbeat delivery still resolves a single agent-level target/to for both OK acknowledgments and alert content. The prior issue discussion already identified the same gap, and the current checkout still matches it.

## Best Possible Solution

Keep #12394 open until maintainers implement or explicitly reject an additive channel-level heartbeat recipient contract. A complete fix should extend schema/types/generated docs for channels.whatsapp.heartbeat.target or targets, validate normalized recipients against the WhatsApp account and allowlist/pairing policy, apply those recipients only to HEARTBEAT_OK delivery when showOk is enabled, preserve alert/content routing, and add tests for omitted-target backward compatibility, single target, multiple targets, account overrides, and invalid targets.
