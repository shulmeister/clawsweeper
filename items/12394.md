## Summary

Keep open. Current main still lacks the requested `channels.whatsapp.heartbeat.target` / `targets` contract: channel heartbeat config is visibility-only, WhatsApp wires that same visibility schema for root and account config, and recipient resolution still comes from agent-level `to`, session state, and `allowFrom` fallback rather than channel-level heartbeat recipients.

## Best Possible Solution

Keep #12394 open until maintainers implement or explicitly reject an additive channel-level WhatsApp heartbeat recipient contract. A complete fix should add typed/schema/generated-doc support for `channels.whatsapp.heartbeat.target` and/or `targets`, validate targets against WhatsApp account and allowlist policy, apply the target list to `HEARTBEAT_OK` delivery without changing alert routing semantics, and add regression tests for single admin target, co-admin targets, omitted-target backward compatibility, per-account config, and invalid target handling.
