## Summary

Current main has an agent-level single-recipient heartbeat route (`agents.defaults.heartbeat.target` + `to`) that can reduce the WhatsApp allowFrom broadcast problem for one admin, but the requested channel-level `channels.whatsapp.heartbeat.target/targets` contract and multi-target/co-admin routing are not implemented. The remaining requested capability is meaningful, so this should stay open.

## Evidence

- **Channel heartbeat config remains visibility-only:** `ChannelHeartbeatVisibilitySchema` is strict and only accepts `showOk`, `showAlerts`, and `useIndicator`; it does not accept `target` or `targets` under `channels.<channel>.heartbeat`.
