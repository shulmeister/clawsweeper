## Summary

Keep open. Current main still only supports visibility flags under `channels.whatsapp.heartbeat`; the only routed heartbeat override is the agent-level single `target` + `to` path. WhatsApp's channel-level heartbeat recipient resolver still falls back to session recipients and `allowFrom`, including multiple recipients, and there is no `target` or `targets` channel config contract.

## Best Possible Solution

Keep this issue open until maintainers either implement an additive channel-level WhatsApp heartbeat routing contract or explicitly choose a different product direction. A complete fix likely belongs in the WhatsApp/channel heartbeat config surface: add typed `target`/`targets` support, validate it in config schema/types/docs, apply it in `resolveWhatsAppHeartbeatRecipients`/channel heartbeat sending while preserving alert behavior, and add regression tests for single admin, co-admin targets, omitted-target backward compatibility, account-scoped config, and allowlist validation.
