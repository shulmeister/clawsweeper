## Summary

Current `main` already covers both reported paths. Control UI auth now explicitly honors `gateway.controlUi.dangerouslyDisableDeviceAuth` for operator Control UI sessions, and the macOS LaunchAgent stop/restart flow now disables relaunch, verifies shutdown, falls back to `bootout` when needed, and cleans stale listener PIDs before restart. The same code is already present in release `v2026.4.22`.

## Close Comment

Closing from a Codex review because this is already implemented on current `main`, and the same behavior is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). For the Control UI path, `src/gateway/server/ws-connection/connect-policy.ts:23` and `:121` now honor `gateway.controlUi.dangerouslyDisableDeviceAuth` for operator Control UI sessions, and `src/gateway/server.auth.control-ui.suite.ts:420` exercises that bypass successfully. For the macOS daemon path, `src/daemon/launchd.ts:466` disables relaunch before stopping and falls back to `bootout` if the service does not actually stop, while `src/daemon/launchd.ts:651` plus `src/infra/restart-stale-pids.ts:511` clean stale gateway listeners and wait for the port to clear before restart. That matches the two failure modes reported here, so there is no remaining maintainer cleanup item to track on `main`.
