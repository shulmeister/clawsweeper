## Summary

Keep open. Current `main` does not implement this feature, and the proposed PR conflicts with explicit current tests and fallback behavior in the dispatch path, so it is not safe to auto-close as implemented/cannot-reproduce/clawhub/incoherent/stale.

## Evidence

Outside that branch it proceeds to `message_received` fire-and-forget hooks and normal agent routing; there is no global `runInboundClaim(...)` broadcast today.
