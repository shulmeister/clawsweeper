## Summary

Current `main` does not show the claimed fix. The browser extension still opens a persistent Playwright CDP browser connection via `chromium.connectOverCDP(...)`, and the global unhandled-rejection handler still exits the Gateway for generic assertions. I found no `shared_worker`/`service_worker` suppression logic or targeted regression test on `main`, so there is not strong evidence to close this issue.
