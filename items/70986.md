## Summary

Keep open. Current main still has both reported memory-wiki lint defects: wikilink extraction scans raw markdown with regexes, and broken-link resolution only accepts path-derived targets rather than frontmatter titles. The related #64696 is open and narrower, so it does not supersede this broader parser/resolver issue.

## Best Possible Solution

Keep this issue open and fix it in extensions/memory-wiki. Make wikilink extraction markdown-aware by excluding frontmatter, fenced code, inline code, and comments before regex extraction. Replace lint's path/basename-only target set with a normalized resolver that includes relative path, basename, page title, slugified title, id, and future aliases. Add regression tests for fenced and inline [[...]] literals, imported source fences, natural Unicode or special-character frontmatter-title links, and relative Markdown links if included in scope. Coordinate with #64696, but do not treat that narrower open issue as a substitute for this broader fix.
