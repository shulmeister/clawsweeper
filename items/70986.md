## Summary

Current main still has both reported lint paths: wikilinks are extracted with a raw regex over the full markdown minus only the managed related block, and broken-link validation only accepts relative paths and basenames, not page titles. The related #64696 remains broader/background rather than a shipped canonical fix.

## Best Possible Solution

Keep #70986 open. The maintainer path is to fix `extensions/memory-wiki/src/markdown.ts` so wikilink extraction ignores markdown structures such as fenced code, inline code, YAML frontmatter, and comments, then update `extensions/memory-wiki/src/lint.ts` to build normalized target keys from path, basename, title, slugified title, id, and any future alias metadata. Add focused regression tests in `extensions/memory-wiki/src/lint.test.ts` or `markdown.test.ts` for fenced/inline literals, `[[reply_to_current]]` transcript text, and `[[<frontmatter title>]]` resolution.
