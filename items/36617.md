## Summary

Keep #36617 open. Current main has meaningful Japanese support for the browser Control UI and generated/public docs i18n infrastructure, but the issue asks for first-class Japanese localization across core UX, including onboarding/help/templates. Current CLI onboarding, wizard prompts, and workspace templates still contain English-only source text without a shared key-based i18n path, so conservative cleanup should not close it as implemented.

## Best Possible Solution

Keep #36617 open until maintainers either define a narrower scope or implement the missing core localization path. The likely product/code path is to keep the existing Control UI i18n and docs publish pipeline, then extend key-based localization to CLI onboarding/wizard/help text and workspace templates, with tests covering locale selection, fallback behavior, and generated bundle/schema drift. If maintainers decide the issue should only cover Control UI, close or split it only after creating narrower follow-up issues for CLI onboarding and templates.
