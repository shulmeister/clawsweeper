## Summary

Keep #36617 open. Current main has meaningful ja-JP support for the browser Control UI and a docs i18n publish pipeline, but the issue asks for first-class Japanese localization across core UX, explicitly including onboarding, help, templates, and fallback behavior. Current CLI onboarding/help and workspace templates still use English source strings directly, and there is no shared core i18n path outside the UI.

## Best Possible Solution

Keep #36617 open until maintainers either define a narrower localization scope or add a core/key-based i18n path beyond the existing Control UI and docs pipeline. A good implementation path is to preserve current UI ja-JP support, then add locale selection and English fallback for CLI onboarding/wizard/help text and workspace templates, with tests for locale resolution, fallback, and generated bundle/schema drift.
