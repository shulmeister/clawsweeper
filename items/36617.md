## Summary

Keep #36617 open. Current main has Japanese localization for the browser Control UI and docs publish pipeline, but the requested core UX scope still includes onboarding, CLI help, templates, and common labels, which remain English-first and do not share a core i18n layer.

## Best Possible Solution

Keep #36617 open until maintainers either narrow it to already-shipped Control UI/docs support or implement shared non-browser core i18n. A concrete path is locale resolution outside the browser UI, English fallback, ja-JP resources for CLI help, onboarding/wizard prompts, common terminal labels, and workspace templates, plus focused tests for fallback behavior, help rendering, wizard prompts, and template generation.
