## Summary

Keep #65767 open. Current main still has the flow registry migration shape that can leave a legacy `owner_session_key` NOT NULL column in `flow_runs`, while current inserts only write `owner_key`; that matches the linked #65563 failure and means the PR is still addressing real behavior. The PR may need maintainer review/update, but it is not obsolete or safely closeable.

## Best Possible Solution

Keep the PR open or replace it with an equivalent maintainer-owned patch that backfills `owner_key` from legacy `owner_session_key` for all reachable schema states, rebuilds/drops the obsolete legacy column so future inserts are writable, and lands the regression tests for full legacy and partially migrated `flow_runs` databases.

## Evidence
