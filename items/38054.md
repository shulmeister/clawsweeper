## Summary

Keep open. Current main still documents the Windows PowerShell one-liner and `scripts/install.ps1` still exits directly from `Main` on failure paths, with no top-level catch/pause before invoking `Main`. The reported fresh Windows failure path still returns false from `Install-Node` when no package manager is found, then reaches `exit 1`. Related fix PRs #38068 and #38634 were closed unmerged, so they do not supersede the issue.

## Best Possible Solution

Keep #38054 open until the Windows installer adds a guarded pause/readable-exit path for interactive or piped PowerShell failures. A good fix would replace direct `exit 1` calls in `Main` with return/throw handling under a top-level wrapper, preserve noninteractive/CI behavior, and include manual Windows Sandbox or Parallels evidence. The newer overlapping issue #41797 should be deduped to this original report if maintainers consolidate.
