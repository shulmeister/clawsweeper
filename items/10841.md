## Summary

Keep open. Current `main` has partial mitigations for time awareness, but there is not strong evidence that issue #10841 is fixed: `session_status` still does not pass a current-time line or machine-readable current timestamp into the status card, so the report's core complaint remains plausible.

## Evidence

- **`session_status` does not supply current time:** `src/agents/tools/session-status-tool.ts:551-599` calls `buildStatusText(...)` with model/task/session fields only. It does not pass `timeLine`, epoch, ISO time, or similar current-time data before returning `statusText`.
  - file: [src/agents/tools/session-status-tool.ts:551](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/agents/tools/session-status-tool.ts#L551)
  - command: `nl -ba src/agents/tools/session-status-tool.ts | sed -n '560,640p'`
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Status card only renders time when caller provides `args.timeLine`:** `src/status/status-message.ts:881-899` includes `args.timeLine` in the output array, but nothing in the `session_status` call path above sets it.
  - file: [src/status/status-message.ts:881](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/status/status-message.ts#L881)
  - command: `nl -ba src/status/status-message.ts | sed -n '876,902p'`
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **A precise current-time helper exists, but is wired into cron rather than `session_status`:** `src/agents/current-time.ts:23-30` builds `Current time: <local> (<tz>) / <UTC>`, and cron appends it in `src/cron/isolated-agent/run.ts:580-613`. That helps cron turns, but it is separate from the `session_status` path named in the issue.
