## Summary

Keep open. The requested Control UI feature is message-level folding for long assistant responses, with Show more/expand-collapse behavior. Current main only has collapse affordances for tool messages, tool cards, and pure JSON blocks; normal assistant markdown, including long code blocks/tables, is still rendered directly in the chat thread.

## Evidence

- **Issue asks for assistant message folding, not only tool output folding:** The issue body requests long assistant responses to collapse by default with a Show more toggle, and the comment reinforces a 30-40 line threshold plus keeping the latest response expanded.
- **Normal assistant messages render full markdown directly:** In the non-tool message branch, pure JSON uses a details block, but otherwise `markdown` is rendered as a `chat-text` div. There is no long-message threshold, collapsed state, or Show more/Show less control around normal assistant text.
