## Summary

Keep open. Current `main` still forces a rebuild whenever `git status` reports a dirty watched source path, before the source-mtime fallback runs, and the test suite explicitly locks that behavior in. The linked PR #57236 is still open, so there is no strong evidence that the issue is already fixed on `main`.

## Evidence

- **Current main still short-circuits dirty trees to rebuild:** `resolveBuildRequirement()` returns `{ shouldBuild: true, reason: "dirty_watched_tree" }` as soon as `hasDirtySourceTree(deps)` is true, and only reaches `hasSourceMtimeChanged()` afterward if git is unavailable or status cannot be read.
