## Summary

Current main implements the missing Gateway failure handling and stale transcript-lock recovery needed for this report: accepted chat.send turns are surfaced to session history/subscribers, dispatch failures emit chat error events instead of an empty final-only path, active chat run state is cleaned up, and stale .jsonl.lock files are reclaimed on gateway startup with main-session recovery.
