## Summary

Keep #67580 open. Current main has some mitigation for daily-memory ingestion, but the reported raw Candidate leak is not fully fixed: the contamination detector still misses inline metadata before `Candidate:`, rehydration has no managed dreaming fence overlap guard, promotion writes the rehydrated snippet directly to `MEMORY.md`, and the default recall gate still uses aggregate signal count rather than real recalls only. The linked PR #68774 is relevant but remains unmerged in the provided GitHub context, so it is not enough to close the issue as implemented.

## Evidence
