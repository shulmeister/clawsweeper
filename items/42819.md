## Summary

Keep open. Current `main` still bakes Playwright Chromium into `/home/node/.cache/ms-playwright` in the Docker image, and the docs/tests still document and assert that cache-path behavior. I found no implementation in the checkout that moves browser assets to `/opt/playwright`, exposes a stable `chrome`/`chromium` symlink on `PATH`, or otherwise resolves the issue request.

## Evidence

- **Dockerfile still installs browsers into the cache path:** `Dockerfile` creates `/home/node/.cache/ms-playwright`, sets `PLAYWRIGHT_BROWSERS_PATH` to that directory, installs Chromium there, and only fixes ownership. There is no `/opt/playwright` path and no symlink/PATH export for the browser binary.
