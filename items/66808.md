## Summary

ZeroEntropy embeddings are not implemented on the checked-out main, but current OpenClaw has a plugin-facing memory embedding provider seam, manifest ownership/env-var metadata, and docs that direct optional capabilities and plugin promotion to ClawHub. This PR is useful provider work, but it should move to a standalone ZeroEntropy memory embedding provider plugin unless it exposes a missing generic SDK seam.

## Best Possible Solution

Close this core PR as ClawHub/plugin work. Carry the provider implementation and review notes into a standalone ZeroEntropy memory embedding provider plugin using `api.registerMemoryEmbeddingProvider`, `contracts.memoryEmbeddingProviders`, manifest `providerAuthEnvVars`, and plugin-specific config. If that work discovers a missing generic SDK seam, open a narrow plugin-API PR instead of adding ZeroEntropy-specific core code.
