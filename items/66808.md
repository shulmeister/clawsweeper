## Summary

PR #66808 is useful ZeroEntropy memory embedding provider work, but current main already has a generic plugin-facing memory embedding provider seam, manifest capability/env metadata, open-ended config strings, and provider-plugin precedent. With no maintainer/protected signal and no missing core API identified, the best cleanup is to close the core PR and move the provider to ClawHub/plugin distribution.

## Best Possible Solution

Close this core PR as ClawHub/plugin work. Carry the provider implementation into a standalone ZeroEntropy memory embedding provider plugin using definePluginEntry, api.registerMemoryEmbeddingProvider, contracts.memoryEmbeddingProviders, and manifest setup.providers[].envVars for ZEROENTROPY_API_KEY. If that plugin uncovers a missing generic memory-embedding SDK seam, open a narrow plugin-API PR for the seam instead of adding ZeroEntropy-specific core code.
