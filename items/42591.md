## Summary

Keep open. Current `main` does not implement the requested installer modularization: the Unix installer is still a single `scripts/install.sh` entrypoint, the repo has no `scripts/install/main.sh` or `scripts/install/lib/*.sh` split, and the same file still mixes downloader, UI, platform, build-tool, npm, and git install responsibilities.

## Evidence

- **Single Unix installer file still exists:** The installer surface under `scripts/` still contains `scripts/install.sh` and `scripts/install.ps1`; there is no `scripts/install/main.sh` or `scripts/install/lib/*.sh` tree matching the proposal.

## Risks / Open Questions

- This is a maintainability/refactor request, not a resolved bug report; closing it would discard a still-valid cleanup target without evidence that the requested modularization landed.
- The repo currently still exposes `scripts/install.sh` as the documented primary Unix installer path, so any future refactor would need to preserve that public surface even if internals move behind sourced modules.
