## Summary

Keep open. Current `main` still swallows `QueuedFileWriter` append failures silently, so the bug this PR targets is not implemented on `main`. The branch also appears stale against current code because `main` now exposes `flush()` and additional safe-append hardening, not the PR’s earlier `drain()`-centric shape.
