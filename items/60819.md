## Summary

Keep #60819 open. Current main improves part of the doctor path by consulting the active memory runtime before saying no memory plugin is registered, but the `openclaw status` path can still return a null memory snapshot before querying the runtime manager when the expected memory-search store/config checks fail. That null snapshot is rendered exactly as `enabled (plugin <slot>) · unavailable`, so the reported status false-positive is not clearly fixed on main.

## Evidence
