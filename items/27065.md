## Summary

Keep #27065 open. Current main still renders the Control UI New session button as a direct click to the supplied `onNewSession` callback, and the parent still wires that callback straight to `handleSendChat("/new", { restoreDraft: true })`. I found no confirmation dialog, relocation, danger treatment, or undo/safety step on the current main path. The direct fix PR #27110 is closed and unmerged, while the existing maintainer/Codex review comment also explicitly kept this issue open.

## Best Possible Solution

Keep #27065 open and implement a focused Control UI safety step before the toolbar New session action dispatches `/new`. The likely change belongs in `ui/src/ui/chat/run-controls.ts` or the `onNewSession` wiring in `ui/src/ui/app-render.ts`, with regression coverage in `ui/src/ui/chat/run-controls.test.ts` for both confirm and cancel behavior. Broader `/new` and `/reset` policy can remain tracked separately; this specific toolbar misclick issue is still real on current main.
