## Summary

Keep #27065 open. Current main still renders the Control UI New session toolbar button as a direct click next to Send, and the parent immediately dispatches `/new` without a confirmation gate. The direct fix PR #27110 is closed and unmerged in the provided GitHub context, so the requested guard has not shipped.

## Best Possible Solution

Keep #27065 open and implement a focused Control UI confirmation before the toolbar New session action dispatches `/new`. The likely change belongs in `ui/src/ui/chat/run-controls.ts` or the `onNewSession` wiring in `ui/src/ui/app-render.ts`, with regression coverage in `ui/src/ui/chat/run-controls.test.ts` for both confirm and cancel behavior. Broader `/new` and `/reset` confirmation policy can remain tracked separately.
