## Summary

Keep open. Current main still has the reported failure mode: a matched route binding whose agent id is absent from `agents.list` is normalized to the configured default agent, so Telegram ingress can still deliver to `main` instead of failing closed or using an explicit catch-all. This is a real behavioral gap, not implemented-on-main, duplicate, or stale-insufficient-info.

## Evidence
