## Summary

Keep open. Current main still does not implement the `tools.exec.shell` configuration surface proposed by PR #39413, and the PR discussion identifies actionable PTY and gateway-approval propagation gaps rather than evidence that the work is obsolete or already shipped.

## Best Possible Solution

Keep the PR open for maintainer review or revision, or have a maintainer explicitly consolidate it into #49931 after preserving the useful implementation notes. A complete solution should define the shell override contract, add typed/global/per-agent config and docs, thread the selected shell through direct exec, PTY supervisor, and gateway approval follow-up runs, and add tests for precedence, fallback, Windows shell args, and validation behavior.
