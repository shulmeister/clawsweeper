## Summary

Keep open. Current main still does not implement the proposed `tools.exec.shell` / per-agent shell override, and the PR remains a useful implementation attempt even though it needs revision for PTY, gateway approval follow-up, docs, and tests. The open related issue #49931 tracks the broader product request, but the provided discussion does not show maintainer consolidation or a superseding merged fix.

## Best Possible Solution

Keep the PR open for maintainer review or explicit consolidation into #49931. A complete solution should define the supported shell override contract, add typed global and per-agent config plus docs, thread the selected shell through direct exec, PTY supervisor, and gateway approval follow-up runs, and add regression tests for precedence, fallback, Windows shell arguments, validation, and approval paths.
