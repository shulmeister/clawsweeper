## Summary

Keep #38808 open. Current main still has the Feishu P2P thread reply gap: it parses `root_id` and `thread_id`, but all thread-aware reply routing remains group-only, and direct-message dispatch still strips reply metadata before sending. The linked bug #38806 remains open, while related replacement PRs #38814 and #38996 are closed unmerged, so this PR is still useful cleanup-safe work rather than obsolete.

## Best Possible Solution

Keep #38808 open and either rebase/merge it or reimplement its narrow Feishu plugin fix on current main: detect `isDirect && Boolean(ctx.threadId)`, target `ctx.rootId` for P2P thread replies, set `replyInThread` and `threadReply`, preserve reply metadata through the dispatcher by not skipping reply-to for that case, keep plain P2P quote replies unchanged, and land the two focused regression tests. Treat #38808 as the canonical fix for #38806 unless a maintainer lands a newer replacement.
