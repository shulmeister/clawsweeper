## Summary

Keep #38808 open. Current main still lacks the Feishu P2P thread reply routing described by this PR: direct messages with `thread_id` still do not set reply-in-thread behavior, do not target the thread root, and still strip reply metadata before sending. The linked bug #38806 is still open, while related replacement attempts #38814 and #38996 are closed unmerged, so this remains useful canonical work rather than cleanup-close material.

## Best Possible Solution

Keep #38808 open and either rebase/merge it or adapt its narrow Feishu plugin fix on main: detect `isDirect && Boolean(ctx.threadId)`, reply to the P2P thread root with `replyInThread`/`threadReply` enabled, preserve plain P2P quote-reply behavior, keep reply metadata through the dispatcher/send path, and land the targeted P2P thread/plain-reply tests. Treat #38808 as the canonical fix for #38806 unless a maintainer opens or lands a newer replacement.
