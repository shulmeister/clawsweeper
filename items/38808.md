## Summary

Current main still lacks the Feishu P2P thread-reply behavior from PR #38808. Direct messages still force non-thread reply flags, keep reply targeting on the triggering message, and skip reply metadata before the send layer, so the reported bug remains plausible on the live code path. The related duplicate PR #38814 was closed in favor of #38808, and the linked bug #38806 remains open, making this PR the canonical remaining fix rather than cleanup material.

## Best Possible Solution

Keep #38808 open, or land an equivalent targeted Feishu fix in `extensions/feishu/src/bot.ts`: detect `isDirect && Boolean(ctx.threadId)`, reply to `ctx.rootId` for P2P thread messages, set `replyInThread`/`threadReply` true, and avoid `skipReplyToInMessages` only for that P2P thread case. Keep the plain P2P quote-reply behavior unchanged and add the two regression tests described in the PR before closing #38806.
