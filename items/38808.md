## Summary

Keep #38808 open. Current main still has the Feishu P2P thread reply gap: inbound parsing preserves `root_id` and `thread_id`, but reply routing remains group-only, direct messages still force `replyInThread=false`, direct-message dispatcher calls still strip reply metadata, and there is no current `isDirect && Boolean(ctx.threadId)` branch. The linked bug #38806 is still open in the provided context, while related replacement PRs #38814 and #38996 are closed unmerged. Greptile's review of this PR also positively identifies the same narrow fix shape.

## Best Possible Solution

Keep #38808 open and either rebase/merge it or reimplement the same narrow Feishu plugin fix on current main: detect `isDirect && Boolean(ctx.threadId)`, use `ctx.rootId` as the P2P thread reply target, set `replyInThread` and `threadReply`, avoid stripping reply metadata for that direct-thread case, preserve plain P2P quote-reply behavior, and land focused regression tests for both direct-thread and direct-plain-reply cases.
