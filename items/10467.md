## Summary

Keep #10467 open. Current main still lacks a public `sessions_spawn.lane` parameter and per-lane subagent concurrency config. Both native and ACP spawn paths still dispatch child runs through the fixed `subagent` lane, and docs/tests still describe/assert one global subagent lane cap. The maintainer-posted Codex review in the issue already kept it open, and the relevant files are unchanged from that review SHA to current HEAD.

## Best Possible Solution

Keep #10467 open until core adds an optional validated `sessions_spawn.lane` field, forwards it through both native and ACP spawn paths, defaults omitted lane to `subagent`, supports backward-compatible per-lane subagent concurrency configuration with hot reload, and updates docs plus tests for default lane, custom lanes, invalid names, and config fallback behavior.
