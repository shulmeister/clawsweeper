## Summary

Keep #10467 open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still does not implement multi-lane `sessions_spawn`: the tool schema has no `lane` parameter, native and ACP spawn dispatch both hard-code the `subagent` lane, gateway concurrency configuration remains a single scalar for `agents.defaults.subagents.maxConcurrent`, and docs/tests still describe/assert one global subagent lane. The provided issue discussion also includes a user +1 and a recent maintainer-posted Codex review reaching the same keep-open conclusion.

## Best Possible Solution

Keep #10467 open until core adds an optional validated `sessions_spawn.lane` field, forwards it through native and ACP spawn paths to the gateway agent call, supports backward-compatible per-lane subagent concurrency configuration with default fallback to `subagent`, and updates docs plus regression tests for omitted lane, custom lanes, invalid lane names, and config reload behavior.
