## Summary

Keep open. Current main still exposes `sessions_spawn` without a `lane` parameter, native subagent spawning still sends every run through the hardcoded `subagent` lane, and config still has only one global `agents.defaults.subagents.maxConcurrent` setting. The request remains actionable core/gateway behavior, not a ClawHub/plugin cleanup close.

## Best Possible Solution

Keep tracking #10467 until core implements optional `sessions_spawn.lane` plumbing, forwards validated lane names to the gateway `agent` call, adds a per-lane subagent concurrency config shape with defaults/fallback to `subagent`, and updates docs plus regression tests. Related closed issues (#8995, #11018) and the external koi per-lane concurrency reference are useful context but do not supersede this OpenClaw request.
