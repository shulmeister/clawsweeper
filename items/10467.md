## Summary

Keep open. Current main still does not implement `sessions_spawn` lane selection or per-lane subagent concurrency; the request remains a concrete core/gateway feature rather than a cleanup close.

## Best Possible Solution

Keep #10467 open until core implements an optional validated `sessions_spawn.lane` field, forwards it through native and ACP spawn paths to the Gateway `agent` call, adds a backwards-compatible per-lane subagent concurrency config shape with default fallback to `subagent`, and updates docs plus regression tests for default lane behavior, custom lanes, invalid lane names, and config reload.
