## Summary

PLaMo support is a useful provider integration, but it fits OpenClaw's external provider-plugin path rather than a bundled core plugin. Current main has the needed provider hooks and SDK subpaths for catalog/auth, dynamic model resolution, tool-schema normalization, custom stream transports, stream wrappers, and guarded provider fetches; no concrete missing core API is evident from this review.

## Best Possible Solution

Close this core PR as a ClawHub/plugin candidate. The best path is to publish the PLaMo provider as an external provider plugin on ClawHub or npm using the existing provider SDK; if publication exposes a specific missing generic SDK seam, open a narrow follow-up issue or PR for that seam only.
