## Summary

Keep open. Current main still treats a non-existent `--output` path without a trailing slash as an explicit archive filename, so the PR is addressing a real unresolved backup behavior. The patch likely needs to be rebased/adapted because the resolver now lives in `src/infra/backup-create.ts` rather than the older `src/commands/backup.ts` target shown in the PR diff.

## Best Possible Solution

Keep the PR open, adapt its resolver change to `src/infra/backup-create.ts`, and add focused backup command tests that distinguish non-existent directory-style outputs from explicit archive filenames/custom suffixes. If maintainers prefer not to take the branch as-is, the same fix should still be applied in main because the underlying behavior remains present.

## Evidence

- **Current resolver still has old behavior:** `resolveOutputPath` appends the timestamped `.tar.gz` basename only for trailing-slash paths or existing directories.
