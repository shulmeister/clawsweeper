## Summary

Close as ClawHub/plugin work. PR #66425 adds a self-contained third-party AstraFlow model-provider plugin and does not identify a missing OpenClaw core API. Current main documents and exposes the provider-plugin surfaces needed for API-key auth, provider registration, catalogs, and onboarding, while VISION.md sets a high bar for adding optional plugins to core.

## Best Possible Solution

Close the core PR and ask the contributor to publish AstraFlow as an external ClawHub/npm provider plugin using the existing provider-plugin SDK. Revisit OpenClaw core only if publishing it externally exposes a concrete missing provider, catalog, auth, or onboarding API seam, or if maintainers later choose to promote it as an official bundled provider.
