## Summary

Keep this issue open. Current main still matches the reported bug: the active `enqueue-followup` path distinguishes stale active snapshots for draining, but then unconditionally calls `typing.cleanup()` before returning, so a real queued-behind-active-run case still loses its typing controller. The prior Codex review comment reached the same conclusion, and the current checkout at 9626ef274ae2346b3d519f99c31856be98aab570 has not changed that behavior.

## Best Possible Solution

Keep this issue open and fix `src/auto-reply/reply/agent-runner.ts` by computing whether the enqueued followup is behind a still-active run after enqueue. If the snapshot is stale, drain immediately as today; if the original run is still active, keep or refresh typing, for example through the existing typing signaler, instead of calling `typing.cleanup()`. Add focused tests proving stale snapshots drain and still-active queued followups do not cleanup typing immediately.
