## Summary

Keep this issue open. Current main still matches the reported bug: the enqueue-followup branch distinguishes stale active snapshots for draining, but then unconditionally calls `typing.cleanup()` before returning, so a real queued-behind-active-run case still loses its typing controller. The existing tests cover enqueue and stale-snapshot drain, but not preservation or refresh of typing while the active run remains live.

## Best Possible Solution

Keep the issue open and fix `src/auto-reply/reply/agent-runner.ts` by computing whether the followup is queued behind a still-active run after enqueue. If the snapshot is stale, drain immediately as today; if the original run is still active, keep or refresh the queued run's typing signal instead of calling `typing.cleanup()`. Add focused regression coverage for both branches: stale active snapshot drains, real active queued followup preserves/starts typing and does not cleanup immediately.
