## Summary

Current main implements the plain IDENTITY.md avatar fallback, but it still preserves backtick-wrapped avatar values and has no regression coverage for that case. The PR discussion and provided diff show the branch added that missing parser behavior and tests, so the PR is not fully obsolete.

## Best Possible Solution

Keep this PR or linked issue #38439 open until current main handles inline-code/backtick-wrapped `IDENTITY.md` avatar values, or until a narrower canonical issue tracks that parser gap. The maintainer path is to rebase and narrow the PR to identity-file avatar value normalization plus focused tests for backtick-wrapped values and custom workspace avatar resolution.
