## Summary

Keep #20173 open. Current main still only handles Discord message-create events in the bundled Discord monitor, does not expose the requested edit-handling config/schema/docs surface, and uses a replay key that would suppress a naive same-message edit re-run.

## Best Possible Solution

Keep #20173 open and implement it in the bundled Discord plugin. Add opt-in config/type/schema/generated metadata/docs/UI-hint coverage, register a Discord message-update listener, normalize edited payloads through the existing preflight and inbound worker pipeline, preserve bot/self/access filters, make replay dedupe edit-aware, define debounce/rate-limit and in-flight cancellation semantics, and add regression tests for listener registration, config validation, and edited-message handling.
