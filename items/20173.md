## Summary

Keep #20173 open. Current main still has a create-only Discord inbound listener path, no MESSAGE_UPDATE/messageUpdate handling, no edited-message config or docs surface, and the existing replay key is account/channel/message-id based, which would suppress a same-message edit without explicit edit-aware handling. The existing maintainer-side Codex comment also kept this open, and the relevant files are unchanged since that review.

## Best Possible Solution

Keep #20173 open and implement it in the bundled Discord plugin. Add opt-in config/type/schema/generated metadata/docs coverage, register a Carbon MessageUpdateListener or equivalent MESSAGE_UPDATE gateway path, normalize edited-message payloads through the existing Discord preflight and inbound worker pipeline, preserve current bot/self/access/mention filters, make replay dedupe edit-aware, define debounce/rate-limit and in-flight cancellation behavior, and add focused regression tests for listener registration, config validation, and edited-message reprocessing.
