## Summary

Current `main` does not contain this fix. A failed `openai-codex` OAuth login still falls through to the normal post-auth wizard path, so the PR should stay open.

## Evidence

- **OpenAI Codex OAuth failure is still swallowed as an empty auth result:** `runOpenAICodexOAuth` catches OAuth errors and null credentials and returns `{ profiles: [] }`, so the setup flow gets no explicit failure/retry signal.
  - file: [extensions/openai/openai-codex-provider.ts:308](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/extensions/openai/openai-codex-provider.ts#L308)

  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Plugin auth application has no skip-model-prompt outcome on main:** After provider auth runs, the main path only returns `config` plus an optional `agentModelOverride`; it does not surface a `skipDefaultModelPrompt`-style result for failed OAuth.
  - file: [src/plugins/provider-auth-choice.ts:281](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/plugins/provider-auth-choice.ts#L281)

## Risks / Open Questions

- Closing this PR as already implemented would also miss other unmerged work in the branch; the current `main` tree does not show the proposed skip-model-prompt behavior.
