## Summary

Keep open. Current main still rejects the self-hosted Firecrawl workflow, while this PR is a focused implementation candidate for the linked bug and has active follow-up addressing prior review concerns.

## Best Possible Solution

Review and land this PR or a maintainer-owned equivalent in `extensions/firecrawl/src/firecrawl-client.ts` that permits admin-configured self-hosted Firecrawl endpoints, allows HTTP only for private/local hosts, routes those private/local Firecrawl API calls through the existing trusted web-tools guard, keeps public hosts on strict HTTPS behavior, preserves the restored SecretRef tests, adds endpoint/SSRF regression coverage, and updates `docs/tools/firecrawl.md`. After that ships, close the linked bug #63877 as implemented and resolve or supersede #59666.
