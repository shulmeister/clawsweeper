## Summary

Keep open. Current main still has the channel auto-reply tool exposure described by this PR, and the PR remains a plausible security fix path, but its latest review context flags real regressions that need maintainer review or changes before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The likely path is to revise the patch so dangerous filesystem/exec tools are denied only for untrusted external channel auto-reply contexts, add focused regression tests for those providers plus internal-provider exemptions, and separately wire any `exec` owner-only hardening through the actual runtime authorization policy.
