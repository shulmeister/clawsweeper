## Summary

Keep open. Current `main` already added ambient proxy support for the WhatsApp plugin, so the issue body's old `makeWASocket({ authDir })` description is stale, but there is still no user-facing config surface that forwards Baileys socket timing options or an explicit socket config object into `makeWASocket()`.

## Evidence

- **`createWaSocket` still hardcodes the Baileys option bag:** The current WhatsApp socket factory accepts only `authDir` and `onQr` in `opts`, then calls `makeWASocket()` with a fixed inline object (`auth`, `version`, `logger`, `printQRInTerminal`, `browser`, `syncFullHistory`, `markOnlineOnConnect`, `agent`, `fetchAgent`). There is no config-derived passthrough for `defaultQueryTimeoutMs`, `keepAliveIntervalMs`, `connectTimeoutMs`, or arbitrary socket options.
  - file: [extensions/whatsapp/src/session.ts:121](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/whatsapp/src/session.ts#L121)

## Risks / Open Questions

- The issue body is partially outdated: `main` no longer passes only `{ authDir }` to `makeWASocket()`, so any maintainer follow-up should narrow the remaining gap to missing timing/raw-socket config passthrough rather than proxy support in general.
- If this is fixed later, the compat question is still open: whether Baileys socket tuning belongs under legacy `web.*`, current `channels.whatsapp.*`, or a new plugin-owned config surface.
