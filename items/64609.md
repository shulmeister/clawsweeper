## Summary

Current main still appears to have the inconsistent Telegram config resolution described here. The account merge intentionally drops channel-level `groups` in multi-account mode, and the prompt/skills/topic path still reads only that merged account config, while `requireMention` has a separate channel-level fallback. Related PR #64650 addressed the same bug but is closed and unmerged in the provided GitHub context, so it is not a reason to close this issue.

## Best Possible Solution

Keep this issue open. The best fix is to update the Telegram prompt/skills/topic resolver in `extensions/telegram/src/bot-core.ts` so group-context reads use account-level `groups` first and fall back to channel-level `channels.telegram.groups` when no account override exists, without changing multi-account routing inheritance. Add a regression test in `extensions/telegram/src/bot.create-telegram-bot.test.ts` for two configured accounts plus channel-level group/topic `systemPrompt` and `skills`; optionally add a doctor warning if maintainers decide channel-level group prompts should remain unsupported in multi-account configs.
