## Summary

Keep open. Current `main` overlaps with the PR's `/model` cache work, but it does not implement the PR's core raw-alias-preservation behavior for ambiguous bare WebUI model IDs. The current UI logic still falls back to provider-qualifying ambiguous or catalog-missing plain IDs, so there is not strong evidence this PR is already implemented on `main`.
