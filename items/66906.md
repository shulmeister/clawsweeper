## Summary

Current main does not implement the stale Discord exec approval click suppression requested by #66906. PR #66918 directly targeted the issue, but it was closed unmerged, and the current Discord button handler still turns any failed gateway resolve into the exact ephemeral “Failed to submit approval decision” message. Gateway not-found responses are now structured enough to distinguish stale approvals, but the Discord handler does not use that classification.

## Best Possible Solution

Keep #66906 open and implement the narrow #66918-style fix on current main: make the Discord approval button path distinguish approval-not-found/already-resolved gateway failures from real failures, suppress stale-click follow-up errors, preserve the existing user-facing error for genuine submission failures, and add focused tests in `extensions/discord/src/monitor/exec-approvals.test.ts` for both branches. The existing structured `APPROVAL_NOT_FOUND` helper should be reused rather than matching ad hoc strings.
