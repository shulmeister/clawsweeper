## Summary

Current `main` and the shipped `v2026.4.23` release already print assistant reply payload text in default non-JSON mode for `openclaw agent`; only `--json` reserves stdout for structured output, so this issue is obsolete on supported code.

## Evidence

- **Released CLI prints reply payloads in text mode:** In the `v2026.4.23` release, `agentViaGatewayCommand` writes each `result.payloads` entry with `runtime.log(...)` whenever `--json` is not set, and only falls back to a summary when there are no payloads.
