## Summary

Keep open. Current main still lacks the cron humanization proposed in this PR: cron jobs render raw `Cron <expr>` text, and Dreaming phase rows still show only the next-run timestamp even though the normalized phase status carries the cron expression. The provided PR discussion also shows the concrete bot-review findings were addressed by follow-up commits.

## Best Possible Solution

Keep this PR open for maintainer review, rebase, CI, and possible landing. The best path is to preserve the focused cron-description utility and regression tests, keep the follow-up fixes for non-zero minute offsets and missing-cron fallback, and decide during review whether the Dreaming copy should describe configured phase cron expressions or the current unified managed dreaming sweep.
