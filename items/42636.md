## Summary

Keep open. Current main and the latest release still use a 200-message Control UI chat history request and a 200-message render window, so the PR's central performance change has not been implemented. The PR remains a plausible focused UI performance fix, but it needs updating for the current render helper path.

## Best Possible Solution

Keep the PR open and ask for an update against current main: introduce a shared initial chat history window constant, use it in `ui/src/ui/controllers/chat.ts` for the `chat.history` request limit and in `ui/src/ui/chat/build-chat-items.ts` for the render slice/notice, then update controller and build-chat-items tests to assert the 80-message behavior. The server-side `chat.history` default can remain 200 because the PR is specifically a Control UI initial-load guard.
