## Summary

Keep open. Current `main` still has the heartbeat `responsePrefix` template interpolation gap that PR #43065 targets: heartbeat resolves the configured prefix as a raw string, calls `getReplyFromConfig()` without `onModelSelected`, and passes the raw prefix into `normalizeHeartbeatReply()`. The normal reply path has the required prefix context and interpolation machinery, but `runHeartbeatOnce()` is not wired into it. The linked duplicate PRs #43133 and #43158 were closed unmerged, and the underlying issue #43064 remains open in the provided context, so there is no canonical shipped replacement to close this PR against.

## Best Possible Solution

Keep PR #43065 open or replace it with an equivalent maintainer-owned patch that wires `runHeartbeatOnce()` into `createReplyPrefixContext()`, passes `onModelSelected` through `getReplyFromConfig()`, resolves the raw heartbeat prefix with `resolveResponsePrefixTemplate()` after the model run, and adds a focused heartbeat regression test for a template such as `{model}: `. The related duplicate PRs are closed unmerged, so they should not be treated as the canonical implementation.
