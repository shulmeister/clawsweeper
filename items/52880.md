## Summary

The reported 2026.3.22 regression is already fixed on current main and was shipped in the 2026.3.23 line: current gateway/onboarding code auto-builds missing Control UI assets, and release packaging now explicitly refuses npm publishes that omit `dist/control-ui` assets.

## Close Comment

Closing as implemented on current main. This Codex review found that OpenClaw now auto-recovers missing Control UI assets during gateway startup (`src/gateway/server-control-ui-root.ts:39`) by calling the asset builder in `src/infra/control-ui-assets.ts:292`, and the `2026.3.23` changelog explicitly records the packaging fix: published npm installs now keep bundled plugins and Control UI assets (`CHANGELOG.md:2202`). The release path also has guards now: prepack refuses to publish without a built Control UI bundle (`scripts/openclaw-prepack.ts:71`) and the npm release checks fail if `dist/control-ui` assets are missing from the tarball (`test/openclaw-npm-release-check.test.ts:292`). If this still reproduces on a release newer than `2026.3.22`, please open a fresh issue with the exact version and install path.
