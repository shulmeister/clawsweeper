## Summary

Keep open. Current `main` has partial mitigation for the report: agent-turn cron jobs default to `sessionTarget: "isolated"`, and isolated runs get fresh per-run session keys. But the cleanup half of the issue is not fully implemented: cron job removal only deletes the job record, and the cron session reaper explicitly prunes only `...:cron:<jobId>:run:<uuid>` entries while keeping the base `...:cron:<jobId>` session row. That means disabled/removed cron jobs can still leave persisted cron session state behind.

## Evidence

- **Agent-turn cron jobs now default to isolated sessions:** Normalization defaults `payload.kind === "agentTurn"` to `sessionTarget = "isolated"`, which addresses accidental shared-session reuse for newly created agent-turn cron jobs unless the caller explicitly opts into `current` or `session:...`.
  - file: [src/cron/normalize.ts:527](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/src/cron/normalize.ts#L527)
  - command: `nl -ba src/cron/normalize.ts | sed -n '520,540p'`
  - sha: [320d52a23e41](https://github.com/openclaw/openclaw/commit/320d52a23e41a326287826219a20f1f29f334554)
- **Tests prove isolated cron runs get fresh session identity per run:** The session identity test asserts two runs of the same isolated cron job receive different `sessionId` values and different `agent:main:cron:job-1:run:...` keys, so current `main` no longer reuses one long-lived run session for isolated jobs.
  - file: [src/cron/isolated-agent.session-identity.test.ts:127](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/src/cron/isolated-agent.session-identity.test.ts#L127)
  - command: `nl -ba src/cron/isolated-agent.session-identity.test.ts | sed -n '127,171p'`
  - sha: [320d52a23e41](https://github.com/openclaw/openclaw/commit/320d52a23e41a326287826219a20f1f29f334554)
- **Reaper explicitly keeps the base cron session row:** `src/cron/session-reaper.ts` documents that only `...:cron:<jobId>:run:<uuid>` records are ephemeral and that the base `...:cron:<jobId>` session is kept as-is; the sweep loop only deletes keys matched by `isCronRunSessionKey()`.
  - file: [src/cron/session-reaper.ts:1](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/src/cron/session-reaper.ts#L1)
  - command: `nl -ba src/cron/session-reaper.ts | sed -n '1,145p'`
  - sha: [320d52a23e41](https://github.com/openclaw/openclaw/commit/320d52a23e41a326287826219a20f1f29f334554)
- **Cron job removal does not clean session state:** `remove()` only filters the cron job out of `state.store.jobs`, persists the cron store, and emits a removal event. It does not call any session-store or transcript cleanup path.
  - file: [src/cron/service/ops.ts:332](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/src/cron/service/ops.ts#L332)
