## Summary

Keep open. Current main still has the reported bug surface: early abort replies can be sent before model selection populates the response-prefix context, and the routed abort path applies `messages.responsePrefix` without any model/thinking context, so `{model}` and `{thinkingLevel}` can still remain literal. The linked PRs #45315 and #45454 are useful attempted fixes, and #45567 was closed unmerged, but none of that fix is present on current main.

## Evidence

- **Fast-abort path sends abort text before normal model selection:** `dispatchReplyFromConfig` checks `fastAbortResolver`; when handled, it creates a payload from `formatAbortReplyTextResolver(...)` and immediately sends it through `routeReplyToOriginating(payload)` or `dispatcher.sendFinalReply(payload)`. There is no fallback call to `replyOptions.onModelSelected` or response-prefix context construction in this branch.

  - sha: [28497515fe1e](https://github.com/openclaw/openclaw/commit/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071)
