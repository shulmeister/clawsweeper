## Summary

Keep #45427 open. Current main carries Discord sender names into the runtime context, but the prompt metadata path still deliberately treats human sender names as untrusted user-context metadata and does not expose a trusted `sender_display_name`/`sender_name` field in the inbound trusted metadata envelope. The issue and follow-up comment describe a real identity-misrouting failure mode, and the provided related items do not supersede this specific trusted sender-name request.
