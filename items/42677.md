## Summary

Keep open. On current `main` (`564f820efa2bb81ae49b8f55a93e41492ad31917`), automatic stale-session rollover exists, but bundled `session-memory` is still wired only to explicit `command:new` / `command:reset` hooks. The PR's requested behavior is not implemented on `main` yet.

## Evidence

- **Automatic stale-session rollover still happens on main:** `initSessionState()` preserves `previousSessionEntry` for stale sessions even when `resetTriggered` is false, so the automatic rollover path is real and distinct from explicit resets.
  - file: [src/auto-reply/reply/session.ts:483](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/auto-reply/reply/session.ts#L483)
  - command: `nl -ba src/auto-reply/reply/session.ts | sed -n '470,540p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Rollover path archives transcripts but does not emit a bundled hook:** Later in the same flow, main archives the previous transcript on rollover, but this code path does not emit any internal `session:rollover` hook for bundled hooks to consume.
  - file: [src/auto-reply/reply/session.ts:801](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/auto-reply/reply/session.ts#L801)
  - command: `nl -ba src/auto-reply/reply/session.ts | sed -n '780,890p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **`getReplyFromConfig()` only backfills explicit reset hooks:** After session init, `getReplyFromConfig()` has a `maybeEmitMissingResetHooks()` helper that only emits hooks when `resetTriggered` is true and the command body matches `/new` or `/reset`. There is no automatic-rollover hook emission on main.
  - file: [src/auto-reply/reply/get-reply.ts:497](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/auto-reply/reply/get-reply.ts#L497)
  - command: `nl -ba src/auto-reply/reply/get-reply.ts | sed -n '480,530p'; nl -ba src/auto-reply/reply/get-reply.ts | sed -n '530,690p'`

- **Bundled `session-memory` still ignores non-command rollover events:** The handler returns unless the event is a `command` with action `new` or `reset`, and the bundled hook metadata/docs still advertise only those two events. Automatic stale-session rollover therefore will not create a memory snapshot on current main.
  - file: [src/hooks/bundled/session-memory/handler.ts:50](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/hooks/bundled/session-memory/handler.ts#L50)
