## Summary

Keep open. Current main still does not make bundled `session-memory` persist on automatic stale-session rollover: there is no `session:rollover` internal hook, the bundled hook is registered only for `command:new` and `command:reset`, and its handler returns for non-command events. Main does have newer session lifecycle plugin hooks for stale rollovers, but that is adjacent plumbing and does not run the bundled internal `session-memory` hook. The PR remains useful, though it needs rebase/rework against current main and should address the existing ACP duplicate-write review concern.

## Best Possible Solution

Keep the PR open for maintainer review or contributor rework. The best fix is to implement automatic stale-session persistence in the current architecture: either add a typed internal automatic rollover event and register/update bundled `session-memory` for it, or adapt bundled memory persistence to the newer session lifecycle hook path, while preserving no-double-write behavior for explicit `/new`/`/reset` including ACP-bound stale sessions. Update `HOOK.md`, docs, onboarding expectations, and focused tests for automatic rollover persistence and explicit reset de-duplication.
