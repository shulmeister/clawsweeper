## Summary

Keep open. Current `main` has static `SecretRef` resolution and a narrower provider-only external-auth hook, but it does not expose a generic credential-provider capability or registration surface spanning providers, channels, tools, and plugins. The RFC is recent, concrete, and backed by actionable discussion and prior implementation experience, so it is not a conservative cleanup close candidate.

## Evidence

## Risks / Open Questions

- Existing provider-only external auth hooks could be mistaken for full implementation, but they do not satisfy the RFC's cross-domain credential-provider contract.
- Closing as `clawhub` would be premature because the current SDK does not yet expose the generic extension point the RFC is asking to standardize.
