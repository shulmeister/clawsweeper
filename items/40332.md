## Summary

Keep #40332 open. Current main still does not implement declarative per-binding or per-agent ACPX `permissionMode` / `nonInteractivePermissions` overrides. ACP binding and agent runtime config only carry `agent`, `backend`, `mode`, `cwd`, and `label`; ACPX permission policy is still sourced from global plugin config, with only a separate per-session runtime `permissionProfile` control available after session creation.

## Best Possible Solution

Implement the requested declarative permission scoping as a core ACP/runtime contract change: extend `AgentRuntimeAcpConfig`, `bindings[].acp` schema/docs, configured binding metadata, ACP session runtime options or backend ensure input, and ACPX runtime wiring so precedence is `bindings[].acp` over `agents.list[].runtime.acp` over global `plugins.entries.acpx.config`. Include validation and regression tests covering both `permissionMode` and `nonInteractivePermissions`.
