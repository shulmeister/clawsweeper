## Summary

Keep PR #38237 open. Current main still has the Slack draft-preview identity gap: `dispatchPreparedSlackMessage` resolves `slackIdentity` and normal delivery forwards it, but `createSlackDraftStream` has no identity parameter and its initial `sendMessageSlack` call omits identity. The PR’s central fix is still useful, but it needs to be ported to the current `extensions/slack` layout and split from unrelated changes before merge.

## Best Possible Solution

Keep PR #38237 and linked issue #38235 open until the identity-only fix is ported cleanly to current `extensions/slack`: add optional `SlackSendIdentity` support to `createSlackDraftStream`, pass `slackIdentity` from `dispatchPreparedSlackMessage`, forward identity only on the initial draft `sendMessageSlack` call, and add a focused regression test in `extensions/slack/src/draft-stream.test.ts`. Split out the unrelated daemon cleanup and any already-superseded duplicate-race work before merge.
