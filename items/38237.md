## Summary

Keep #38237 open. Current main still resolves Slack agent identity and forwards it through normal Slack delivery, but the draft preview stream path still does not accept or pass that identity to the initial `chat.postMessage`. The PR's core Slack identity fix is still useful, though it should be rebased and split from unrelated/superseded changes before merge.

## Best Possible Solution

Keep this PR or its linked issue #38235 open until the identity fix is ported cleanly to current `extensions/slack`: add an optional `SlackSendIdentity` to `createSlackDraftStream`, pass `slackIdentity` from `dispatchPreparedSlackMessage`, forward it only to the initial `sendMessageSlack` call, and add a focused regression test in `extensions/slack/src/draft-stream.test.ts`. Drop unrelated daemon cleanup and already-shipped duplicate-race changes before merge.
