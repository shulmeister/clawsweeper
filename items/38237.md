## Summary

Keep PR #38237 open. Current main still drops the resolved Slack agent identity on the draft/partial streaming initial message, while normal delivery and sendMessageSlack already support identity. The PR targets a real remaining bug, but should be rebased to current extensions/slack paths and narrowed because the provided diff also includes unrelated daemon cleanup and a separate draft-stream duplicate-race change.

## Best Possible Solution

Keep PR #38237 and linked issue #38235 open until the identity-only fix is ported cleanly to current extensions/slack: add optional SlackSendIdentity support to createSlackDraftStream, pass slackIdentity from dispatchPreparedSlackMessage, forward identity only on the initial sendMessageSlack call, and add a focused regression test in extensions/slack/src/draft-stream.test.ts. Rebase and split out the unrelated daemon cleanup and already-superseded duplicate-race work before merge.
