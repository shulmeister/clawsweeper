## Summary

Keep #69443 open. Current main has adjacent subagent announcement and delivery-context hardening, but the exact reported low-level RPC path still does not treat `params.to` as a session key. In the Gateway `agent` handler, session routing is driven by `params.sessionKey` or an explicit `agentId`; `params.to` is later used as a delivery target/source hint, so a call with only `to: "agent:main:openclaw-weixin:..."` can still fall back to the main session.

## Best Possible Solution

Add a narrow core fix or explicit validation for low-level `agent` RPCs: either treat `params.to` as `params.sessionKey` when it is a canonical `agent:<agentId>:...` session key and no `sessionKey` is provided, or reject that shape with a clear error telling callers to use `sessionKey`. Cover it with a Gateway regression test using an external-plugin-shaped key such as `agent:main:openclaw-weixin:direct:user@im.wechat`, and keep the existing channel delivery fields separate from session routing.
