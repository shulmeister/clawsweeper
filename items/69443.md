## Summary

Current main still treats `agent` RPC `sessionKey` as the transcript/session selector and `to` as an outbound delivery target. I did not find code or tests that normalize `to: "agent:main:openclaw-weixin:direct:..."` into the requested session, so #69443 remains actionable. The related #69576 context is useful but adjacent: it concerns account-scoped delivery-mirror isolation, not this raw `gateway call agent` callback-routing path.

## Best Possible Solution

Keep the issue open for a targeted gateway/session routing decision. Maintainers should either reject `agent` RPC `to` values that parse as OpenClaw session keys unless `sessionKey` is provided, or intentionally normalize those values into `sessionKey` with clear precedence for `deliver`, `replyTo`, channel, accountId, and existing delivery context. Add a regression test for `method: "agent"` with `to: "agent:main:openclaw-weixin:direct:..."` and no `sessionKey`, plus docs/tool guidance to use explicit `sessionKey` or `sessions_send` for cross-session callbacks.
