## Summary

Keep open. Current `main` has substantial restart-resume plumbing for gateway-controlled restarts, config apply/patch, and updates, but it does not fully implement the issue's broader request to persist arbitrary in-flight task context before any reboot/process-killing command and then automatically verify/report the outcome after boot. The shipped behavior is partial, not a clear end-to-end match.

## Risks / Open Questions

- Closing as implemented would overstate the current scope: arbitrary shell-initiated reboots or other process-killing commands still do not have a clear persisted breakpoint/resume path in core.
- Config/update restart reports may look similar to the requested feature, but they currently persist restart metadata rather than a fully general task-context-and-verification workflow.
