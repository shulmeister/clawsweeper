## Summary

Keep open. Current `main` does not implement granular per-tool `tools.fs.workspaceOnly` policy. The code and docs still expose a single `tools.fs.workspaceOnly` boolean for filesystem tools, with only a separate special-case override for `tools.exec.applyPatch.workspaceOnly`. That does not satisfy the reported need to allow reads outside the workspace while keeping write/edit confined to it.

## Evidence

## Risks / Open Questions

- There is a partial mitigation today: `tools.exec.applyPatch.workspaceOnly` can be configured separately, but that only covers `apply_patch`, not `read` or `edit`/`write`.
- Tool allow/deny lists can disable whole tools, but they do not provide the requested mixed policy of read outside workspace while write/edit remain workspace-confined.
