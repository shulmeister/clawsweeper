## Summary

Keep this PR open. Current main still ignores OPENCLAW_STATE_DIR for the default/main agent workspace path, so the requested behavior is not implemented. The PR is relevant but needs revision before merge because the submitted patch uses the env value raw, and review comments correctly flagged that tilde/relative values should be normalized like the existing state-dir/config path code.

## Best Possible Solution

Keep the PR open and request a focused revision in the current src/agents/workspace-default.ts path: derive the default workspace base from resolveStateDir(env, homedir) or equivalently normalize OPENCLAW_STATE_DIR with resolveUserPath, preserve existing HOME/OPENCLAW_HOME fallback behavior, append the OPENCLAW_PROFILE suffix as before, and add tests for absolute, tilde/relative state dir, default profile, non-default profile, and unset env cases. If maintainers prefer workspace isolation to remain explicit, close only after documenting that product decision.
