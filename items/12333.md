## Summary

Keep open. Current `main` offers a different workaround via multi-agent isolation and agent switching, but it does not clearly implement the requested project/context switching inside one workspace or scoped memory filtering within that workspace.

## Evidence

- **Single workspace remains the core model:** The current docs still describe one workspace per agent as the only working directory for tools and context: `OpenClaw uses a single agent workspace directory ... as the agent’s only working directory (cwd)`.
  - file: [docs/concepts/agent.md:12](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/docs/concepts/agent.md#L12)

- **Current supported solution is separate agents, not project switching within one workspace:** Multi-agent docs say each agent gets its own workspace, state directory, and session history, and recommend `openclaw agents add work` / `openclaw agents add coding` to create isolated contexts.
  - file: [docs/concepts/multi-agent.md:8](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/docs/concepts/multi-agent.md#L8)

That is adjacent functionality, but it is not a `workspace switch <project-name>` feature inside one workspace.
