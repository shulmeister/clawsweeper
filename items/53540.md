## Summary

Keep open. Current main still leaves the reported OpenRouter-to-Anthropic large tool-parameter path on the OpenAI-compatible streaming transport, with a default 120s no-chunk LLM idle watchdog and no source/test/release evidence that the edited Write-style large-argument reproducer succeeds by default.

## Best Possible Solution

Keep this issue open as the canonical tracker for OpenRouter/Anthropic large tool-parameter generation failures. A maintainer fix should add a synthetic regression that withholds tool-call argument chunks longer than the default idle window, and ideally a live OpenRouter replay. The fix likely belongs in the LLM idle-timeout policy, OpenRouter/Anthropic tool-argument streaming support where the proxy supports it, or a core/tool-loop strategy that avoids requiring one large silent tool-call parameter payload.
