## Summary

Keep open. Current main still does not initialize nvm before Node detection in `scripts/install.sh`, so the reported `curl | bash` PATH gap is not implemented away. The proposed PR also has an unresolved correctness risk: sourcing `nvm.sh` before `check_node()` can change the active `node` being detected, so this is not a strong close candidate.

## Evidence

- **Current installer still checks Node before any nvm bootstrap:** `main()` goes straight from `install_homebrew` to `check_node()`/`install_node()` and only later calls `ensure_default_node_active_shell()`. There is no pre-`check_node()` `NVM_DIR`/`nvm.sh` setup on current main.
  - file: [scripts/install.sh:2326](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/scripts/install.sh#L2326)

  - sha: [320d52a23e41](https://github.com/openclaw/openclaw/commit/320d52a23e41a326287826219a20f1f29f334554)
- **nvm handling is only advisory after failure:** `ensure_default_node_active_shell()` only prints nvm recovery instructions if nvm is already detectable via `NVM_DIR`, `command -v nvm`, or an active `node` path under `/.nvm/`. That does not solve the original `curl | bash` case where shell startup files were not sourced before `check_node()`.
  - file: [scripts/install.sh:1341](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/scripts/install.sh#L1341)
