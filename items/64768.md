## Summary

Keep open. Current main has generic Discord pre-lifecycle gateway cleanup, which partially overlaps the PR title, but the bot-identity behavior behind this PR is still unresolved: fetchUser('@me') failures still return an undefined botUserId, startup continues, and mention-required guild handling still depends on botId being present. The PR also has unresolved review concerns, so it needs maintainer judgment rather than cleanup closure.

## Best Possible Solution

Keep the PR open or supersede it with a focused maintainer-owned fix for the canonical Discord bot-identity startup bug. The fix should resolve botUserId reliably or fail closed, preserve the existing pre-lifecycle gateway cleanup, explicitly handle reconnect suppression if needed, remove unsafe botId short-circuits where appropriate, and add deterministic regression coverage for thrown fetch failures and missing user ids.
