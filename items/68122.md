## Summary

Current `main` already covers this path: `session_status` resolves the configured default model into its allowed set, accepts legacy `codex/*` compatibility refs, and clears stale runtime model fields when applying a session override so the session no longer stays pinned to old model identity.

## Evidence

- **`session_status` now validates against the configured default/current model context:** The override path resolves the configured default for the agent, uses that as the current provider/model basis, builds the allowed model set from that context, and only rejects when the resolved key is outside that set. That directly covers `session_status(model="codex/gpt-5.4")` when the configured/default model is Codex-backed.
  - file: [src/agents/tools/session-status-tool.ts:211](https://github.com/openclaw/openclaw/blob/b13545355d42701a1b2a5838b18101777c1bd808/src/agents/tools/session-status-tool.ts#L211)
  - command: `git blame -L 211,241 -- src/agents/tools/session-status-tool.ts`
  - sha: [b13545355d42](https://github.com/openclaw/openclaw/commit/b13545355d42701a1b2a5838b18101777c1bd808)
- **Allowed-model computation explicitly preserves the configured default key:** When no explicit `agents.defaults.models` allowlist exists, `buildAllowedModelSetWithFallbacks(...)` adds the resolved default model key into `allowedKeys`; when an allowlist does exist, it also adds the resolved default key after parsing compatibility aliases. That prevents the configured default Codex model from being rejected as 'not allowed'.
  - file: [src/agents/model-selection-shared.ts:515](https://github.com/openclaw/openclaw/blob/b13545355d42701a1b2a5838b18101777c1bd808/src/agents/model-selection-shared.ts#L515)
  - command: `git blame -L 515,576 -- src/agents/model-selection-shared.ts`
  - sha: [b13545355d42](https://github.com/openclaw/openclaw/commit/b13545355d42701a1b2a5838b18101777c1bd808)
- **Applying a session override now clears stale runtime model identity and marks a pending live switch:** `applyModelOverrideToSessionEntry(...)` deletes stale persisted `model`/`modelProvider` fields when the selected model changes or runtime identity is stale, and sets `liveModelSwitchPending`. `session_status` uses that helper when mutating the session entry. This addresses the stale-session-state concern described in the report.
  - file: [src/sessions/model-overrides.ts:56](https://github.com/openclaw/openclaw/blob/b13545355d42701a1b2a5838b18101777c1bd808/src/sessions/model-overrides.ts#L56)

- **Current main still supports legacy `codex/*` config and routes it to the Codex harness:** Tests on `main` cover both auto-enabling the bundled Codex plugin from `codex/gpt-5.4` config and skipping PI warmup when the Codex harness claims that model. That matches the issue's legacy compatibility setup instead of treating it as unsupported config.
