## Summary

Keep #42294 open. Current main has partially moved/covered the generic `audioAsVoice` routing, and #42060 is closed for the main user-visible TTS path, but the Telegram outbound adapter still does not translate `audioAsVoice` into Telegram's `asVoice` option on the generic outbound media/sendPayload path. The PR needs rebasing to the current `extensions/telegram` layout and should address the review comment about gating the `sendPayload` route on actual media presence, but its core bugfix intent is still relevant.

## Best Possible Solution

Keep the PR open but require a rebase/narrow update: port the Telegram-owned fix to `extensions/telegram/src/outbound-adapter.ts`, map `audioAsVoice` to `asVoice` for both `sendPayload` media sequencing and direct `sendMedia`, gate any `sendPayload` routing changes on actual media content, and add Telegram adapter plus delivery regression tests for voice-note media.
