## Summary

Keep PR #72128 open. Current main still does not implement configurable `sessions_spawn` label templates, and the PR is the active unmerged implementation candidate for the linked request rather than obsolete cleanup. It also has substantive unresolved review concerns around resolved target-agent labels and requester-scoped subagent config, so conservative maintainer cleanup should not close it.

## Best Possible Solution

Keep PR #72128 open for maintainer review or contributor revision. The best path is to implement label templating in the core `sessions_spawn` path using the resolved target agent for `{agentId}`, read per-agent `subagents.labelTemplate` from requester-scoped config with defaults fallback, preserve explicit-label precedence, and update tests, docs, config types, Zod schemas, and generated schema artifacts. Close only if a separate merged implementation supersedes it or a maintainer explicitly rejects the feature direction.
