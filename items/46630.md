## Summary

Keep #46630 open. Current main has related slash-command discoverability through the chat slash menu and command palette, including plugin/skill commands from `commands.list`, but it does not implement the requested configurable WebChat shortcut/keyword sidebar or `webchat.shortcuts`/`shortcuts.json` style prompt insertion surface. The related #46604 remains a separate split-dashboard layout request, not a canonical tracker for this shortcut panel.

## Evidence

- **Issue request:** #46630 asks for a collapsible, scrollable WebChat sidebar of user-defined shortcut keywords with label, prompt, description tooltip, optional icon/category, click-to-insert behavior, and configuration via `webchat.shortcuts` or `shortcuts.json`. It also links #46604 only as a possible layout integration point.
- **Existing command discovery surface:** The UI refreshes slash commands from the gateway `commands.list` RPC with `scope: "text"` and replaces the in-memory slash command list with built-in plus runtime command entries. This supports registered commands, not arbitrary user-defined prompt shortcuts.
