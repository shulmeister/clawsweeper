## Summary

Current main does not implement issue #7679. Telegram still defaults to `dmPolicy: "pairing"` in schema, tests, runtime fallback, and user docs. There is partial QuickStart behavior that can force an allowlist prompt and write `dmPolicy: "allowlist"` with manually entered numeric Telegram IDs, but that is not a general default and does not auto-capture the owner ID during Telegram onboarding.
