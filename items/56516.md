## Summary

Keep open. Current `main` still keeps `sessions_spawn(mode:"run")` subagent sessions by default, the Control UI session page is backed by `sessions.list` rather than a separate web-only registry, and the documented cleanup/archive behavior does not automatically remove completed subagent sessions from the UI immediately after completion.

## Evidence

- **`sessions_spawn` still defaults to persistent cleanup:** The tool resolves `cleanup` to `"keep"` unless the caller explicitly passes `"delete"`, so run-mode subagent sessions are not auto-deleted on completion by default.
  - file: [src/agents/tools/sessions-spawn-tool.ts:193](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/tools/sessions-spawn-tool.ts#L193)

- **Gateway session listing still includes stored subagent sessions:** `listSessionsFromStore` starts from all session-store entries and only applies generic filters (`global`, `unknown`, `agentId`, `spawnedBy`, search, `activeMinutes`, limit). There is no branch that drops completed subagent sessions from the main session list just because they finished.
  - file: [src/gateway/session-utils.ts:1511](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/gateway/session-utils.ts#L1511)
