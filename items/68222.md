## Summary

Current main already treats `sessions_yield` as an intentional end-turn path and hides its synthetic abort artifacts, but I did not find evidence of a Kimi-specific fix that would stop the model from spuriously choosing that tool during long-running tasks. The report is still plausible as a current model/provider behavior issue, so this is not a high-confidence close.
