## Summary

Current `main` already supports cross-channel DM/session sharing and WebChat inspection through the existing session-routing model (`session.dmScope` and `session.identityLinks`); the remaining gap is mostly default configuration, not missing core functionality.

## Risks / Open Questions

- Behavior is config-dependent: new local onboarding now prefers `session.dmScope: "per-channel-peer"` for safety, so users on default secure settings can still experience isolated histories unless they opt into shared modes.
- WeChat is an external plugin, so any plugin-specific UX limitations beyond OpenClaw's shared session model are outside the core evidence reviewed here.
