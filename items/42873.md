## Summary

Keep open. Current main still strips a trailing own-line `NO_REPLY` from auto-reply text and delivers the preceding narration, and current tests explicitly encode that strip-and-deliver behavior. Some cron announce delivery paths have since gained separate trailing-token suppression, but the PR's core normalization behavior is not implemented and the linked follow-up #46626 indicates additional delivery paths still need coverage.

## Best Possible Solution

Keep the PR open for maintainer or author follow-up. The best path is a focused shared silent-token helper, regression tests that flip the current `normalizeReplyPayload` expectation for own-line trailing `NO_REPLY`, and a coordinated follow-up with #46626 for the remaining delivery paths while preserving inline `😄 NO_REPLY` stripping behavior.
