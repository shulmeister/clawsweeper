## Summary

Current main still delivers the preamble text for replies ending with a standalone trailing `NO_REPLY`, so PR #42873 is not implemented on main and should remain open.

## Evidence

- **Current normalization still strips trailing token and keeps prior text:** `normalizeReplyPayload` handles mixed-content `NO_REPLY` by calling `stripSilentToken(text, silentToken)` and only skips when the stripped result is empty. There is no own-line suppression branch, so `"text\n\nNO_REPLY"` remains deliverable as `"text"`.
