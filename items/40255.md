## Summary

Keep #40255 open. Current main still resolves user-configured heartbeat prompts and then unconditionally passes the resulting prompt through the HEARTBEAT.md workspace-path hint appender, so any custom prompt mentioning HEARTBEAT.md can still be mutated. That conflicts with the documented custom-prompt behavior, which says custom heartbeat bodies are sent verbatim.

## Best Possible Solution

Keep the issue open and fix `resolveHeartbeatRunPrompt` so the workspace HEARTBEAT.md path hint is appended only for the built-in default heartbeat prompt or internally generated prompts where that hint is intended. Add a regression test with a configured custom prompt containing `HEARTBEAT.md` and assert the body sent to the model is unchanged, preserving the documented verbatim custom-prompt contract.
