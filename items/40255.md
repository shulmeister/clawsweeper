## Summary

Keep #40255 open. Current `main` still resolves a configured heartbeat prompt and then unconditionally passes the resulting text through the workspace `HEARTBEAT.md` hint appender, so a user-authored prompt containing `HEARTBEAT.md` is still modified before it is sent as the heartbeat user message. The open PR #40756 remains the active fix attempt; earlier related PRs were closed unmerged duplicates, not shipped fixes.

## Best Possible Solution

Keep the issue open and land a focused fix, likely through PR #40756 or an equivalent maintainer patch. `resolveHeartbeatRunPrompt` should apply `appendHeartbeatWorkspacePathHint` only to built-in/default or internally generated heartbeat prompts, not to user-configured `heartbeat.prompt` text from agent overrides or defaults. Add a regression test configuring a custom prompt that mentions `HEARTBEAT.md` and assert the `Body` passed to `getReplyFromConfig` remains unchanged, while preserving the existing default-prompt hint behavior.
