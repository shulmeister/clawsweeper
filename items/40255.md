## Summary

Current main still appears to reproduce the regression: a configured heartbeat prompt can become the heartbeat run prompt, and `resolveHeartbeatRunPrompt` still passes that prompt through `appendHeartbeatWorkspacePathHint` without distinguishing user-authored prompt text from the built-in/default/internal prompt paths. The provided GitHub context also shows open PR #40756 as an unmerged implementation candidate with closing syntax for this regression, so the issue should stay open until that PR or an equivalent patch lands.

## Best Possible Solution

Keep this issue open and resolve it by reviewing and landing PR #40756 or an equivalent maintainer patch. The fix should preserve the workspace-path hint for built-in/default/internal heartbeat prompts, skip it for user-authored configured prompt text, and add a focused regression test for a custom prompt that mentions `HEARTBEAT.md`.
