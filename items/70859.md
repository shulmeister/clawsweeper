## Summary

Keep open. Current main partially covers the setup/status-scan direction, but it has not implemented this PR's `status --all` path change, and the existing review concern about snapshot notices suppressing hook-derived compatibility warnings is still grounded in current loader behavior.

## Best Possible Solution

Keep the PR open but ask for a narrower revision: either add a real metadata-backed compatibility source that preserves hook-only and `before_agent_start` notices without loading full plugin modules, or limit the change to already-proven cold paths. The `status --all` diagnostics call in `src/commands/status-all/report-data.ts` is still the remaining useful target, but it needs correctness coverage against the real loader behavior and the unrelated test/runtime changes should be split out.
