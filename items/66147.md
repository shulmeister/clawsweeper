## Summary

Keep this PR open. Current main still has no `model.tool` config, no split-model routing contract, and no equivalent plugin hook for per-tool-continuation model swaps. The PR is a plausible implementation candidate, but the existing review discussion identifies real cross-provider correctness defects that need design and code changes rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or author changes. The best path is to decide the product contract for split model routing, then rework the implementation so the effective per-call model controls tool support, provider transport, auth profile resolution, transcript policy, provider-specific stream wrappers, context budgeting, docs/config schema, and generated config baselines. If maintainers prefer a plugin/API design instead of core `model.tool` config, use #13225 as the canonical design discussion rather than closing this PR as already handled.
