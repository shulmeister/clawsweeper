Keep open. Current main still does not implement `model.tool` or equivalent split tool-turn routing, and the PR has unresolved review findings around primary-bound tool enablement, transport/auth, and provider-specific stream wrappers. None of the conservative close reasons fit for this open PR.

## Best Possible Solution

Keep the PR open only if maintainers want core split-model routing, but require a rework around the effective per-call model: config/schema/docs/generated baseline updates, tool enablement and tool schema normalization based on the routed tool-capable model, per-call stream transport/auth/provider-wrapper selection, explicit fallback/auth-profile/context-window policy, and integration coverage for a non-tool-capable primary with a cross-provider tool model. If maintainers prefer plugin/API direction, supersede it with the existing pre-tool-call routing/delegation design discussion rather than auto-closing this PR now.
