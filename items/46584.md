## Summary

Keep open. Current main still appears to reproduce the reported false positive: `noteMemorySearchHealth` has explicit branches for `local` and `lmstudio`, but no `ollama` branch, so explicit `memorySearch.provider: "ollama"` falls into the remote-provider API-key warning path. The Ollama embedding runtime still treats the API key as optional, which supports the original report and the existing issue comment. Related PRs #46893 and #47078 attempted the right fix but are closed/unmerged per the provided GitHub context, and their changes are not present in this checkout.
