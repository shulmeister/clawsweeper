## Summary

Current main still lacks a Compose build context for `openclaw-gateway`, and a read-only `docker compose up --build --dry-run openclaw-gateway` against the checked-out compose file still tries to pull `openclaw:local` and fails with `pull access denied`. The PR is a focused candidate for a real remaining Docker workflow gap, though maintainers should verify the interaction with `OPENCLAW_IMAGE` and existing setup-script build args before landing.

## Best Possible Solution

Keep this PR open for maintainer review or replace it with an equivalent Docker change that lets source-checkout users run `docker compose up --build` without a prior manual `docker build`, while preserving the setup-script path, remote `OPENCLAW_IMAGE` behavior, and documented Docker build args.

## Risks / Open Questions

- Adding unconditional `build: .` to the root compose file should be checked against the documented `OPENCLAW_IMAGE=ghcr.io/openclaw/openclaw:latest` path so Compose does not unexpectedly rebuild or tag a remote image during setup-script runs.
- The root setup script passes build args such as `OPENCLAW_DOCKER_APT_PACKAGES`, `OPENCLAW_EXTENSIONS`, and `OPENCLAW_INSTALL_DOCKER_CLI`; a final fix may need Compose `build.args` or documentation if maintainers expect `docker compose build` to preserve those options.
