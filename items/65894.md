## Summary

Keep open. Current main still does not include the PR's requested `build: .` context in `docker-compose.yml`, so the exact change is not implemented or superseded. The existing documented/setup paths build `openclaw:local` before Compose starts, but that does not solve the PR's narrower `docker compose up --build` behavior against the repo-root Compose file.

## Best Possible Solution

Keep the PR open for maintainer review. If OpenClaw wants repo-root `docker compose up --build` to be a supported local path, add an explicit build context to `openclaw-gateway` and update/validate Docker docs so the remote-image `OPENCLAW_IMAGE` path remains clear. If raw Compose is not supported, close only after documenting that the setup script/manual `docker build` flow is the intended path.
