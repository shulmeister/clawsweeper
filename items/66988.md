## Summary

Keep #66988 open. Current main still has the reported behavior: the gateway redaction fallback sets `raw` to `null`, and the Control UI interprets that as Raw mode unavailable, forces Form mode, disables the Raw tab, and shows the exact disabled message from the issue. The related fix PRs in the provided context (#67010 and #67212) were closed unmerged, so the issue is not implemented or superseded on main.

## Best Possible Solution

Keep this open unless maintainers intentionally want Raw mode disabled for unsafe round-trips. A product-quality fix would separate raw display availability from raw editability: keep a safely redacted raw string for read-only viewing when possible, add an explicit editability/reason flag to the config snapshot contract, keep save/apply blocked when round-trip restore is unsafe, and cover the gateway redaction path plus UI controller/view behavior with regression tests.
