## Summary

Keep #70753 open. Current main has related Undici timeout hardening, but I did not find a targeted fix or regression test for the reported AgentMail bundled MCP failure. The repro uses `type:"http"`, while embedded Pi still consumes OpenClaw's `transport` field directly and defaults URL MCP entries to SSE unless `transport:"streamable-http"` is set.

## Best Possible Solution

Keep this open until current main is verified against the reported AgentMail case. The best path is to reproduce with `transport:"streamable-http"` and with the reporter's `type:"http"` command, then either normalize `type:"http"` to OpenClaw's streamable HTTP transport for embedded Pi or add clear validation/guidance. Add a focused regression test for hosted streamable HTTP MCP initialization and ensure the bundled client no longer hits the 15s/30s timeout path.
