## Summary

Keep #39690 open. Current main still preserves the reported failure mode for matching explicit+implicit provider models: explicit `input: ["text", "image"]` is overwritten by implicit/catalog `input`, while Gateway image handling still relies on that merged catalog metadata. The related text-only image offload work helps #67606/#67634-style failures, but it does not implement the issue's central requested behavior. The direct fix PR #39785 is closed unmerged in the provided GitHub context, and current source still lacks its proposed precedence change.

## Best Possible Solution

Keep #39690 open until `mergeProviderModels` preserves explicit `models.providers.*.models[].input` when the field is present, using the same explicit-override pattern already used for `reasoning`. Add or update merge coverage so explicit `input: ["text", "image"]` survives an implicit text-only match, then verify Gateway/native inline image handling for a custom Ollama or OpenAI-compatible vision model still works alongside the existing text-only offload fallback.
