## Summary

Keep #39690 open. Current `main` still overwrites an explicitly configured model `input` value with implicit/catalog metadata when the model IDs match, so `input: ["text", "image"]` can still be lost before Gateway decides whether to preserve inline image blocks. The text-only offload path helps a related failure mode but does not implement the reported explicit-input precedence behavior.

## Best Possible Solution

Keep the issue open. The best fix is the narrow precedence change described by #39785: update `mergeProviderModels` so an explicitly present `models.providers.*.models[].input` wins over implicit/catalog metadata, matching the existing `reasoning` override pattern. Add regression coverage where an explicit image-capable model survives a text-only implicit match, then verify Gateway still sends native inline image blocks for that configured model while preserving the text-only offload fallback.
