## Summary

Keep PR #71594 open. Current main still lacks the PR's IPv4-only BYOH documentation and onboarding hint, while the current gateway bind and Control UI link paths remain IPv4-oriented for `lan`, `tailnet`, and valid `custom` hosts. The PR is focused docs/comment/test work rather than obsolete or already implemented work.

## Best Possible Solution

Keep PR #71594 open for maintainer review or merge after normal validation. The best product path is to document the current IPv4-only BYOH behavior in gateway CLI/config/onboarding surfaces now, while treating broader native IPv6 BYOH support as a separate behavior change if maintainers want it.
