## Summary

Keep open. Current main still lacks the proposed IPv4-only BYOH clarification in the gateway CLI docs, gateway config type docs, and no-GUI onboarding SSH hint, while current gateway code supports the PR premise that the BYOH-facing lan, tailnet, and custom paths are IPv4-oriented today.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase the patch onto current main, keep the docs/TSDoc/onboarding clarification, place any implementation-adjacent comment near `src/gateway/control-ui-links.ts`, and treat native IPv6 BYOH support as a separate behavior change if maintainers want it.
