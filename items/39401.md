## Summary

Keep #39401 open. Current main still lacks the proposed keyed/remount boundary or an equivalent selected-agent reset for the Agents overview model controls, while the linked bug #39392 remains open with concrete reproduction details. Related PR #39495 is closed unmerged in the provided GitHub context, so it does not supersede this open PR as a shipped or canonical fix.

## Best Possible Solution

Keep #39401 open or explicitly nominate a fresh canonical PR for #39392. The best implementation path is to add a per-selected-agent remount/reset boundary around the Agents overview model controls, then add a focused regression test that switches between agents with different model settings and verifies the displayed primary/fallback controls reflect the selected agent.
