## Summary

Keep #39401 open. Current `main` still renders the Agents overview without a per-agent `keyed(selectedAgent.id, ...)` remount or an equivalent config-form reset, while the overview panel still contains stateful model/fallback form controls. The linked bug #39392 has concrete reproduction details and another open same-surface PR (#39495), so this is useful pending UI bug-fix work rather than a conservative cleanup close.

## Best Possible Solution

Keep tracking #39401 or explicitly choose a canonical PR with #39495, then rebase the keyed overview remount or equivalent selected-agent reset onto current `ui/src/ui/views/agents.ts` and add a regression test for `main -> other agent -> main` model-control state on the Agents overview panel.
