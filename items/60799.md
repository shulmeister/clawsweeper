## Summary

Keep open. Current main still appears to reproduce the core data-loss trigger: `loadCronStore()` treats a top-level JSON array as an empty object, so legacy plain-array `jobs.json` files load with zero jobs and a later save can replace the store from that empty in-memory state. The related PR #61113 proposed the right normalization/test coverage, but the provided GitHub context says it was closed unmerged, and current main does not contain that behavior.
