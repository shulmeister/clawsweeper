## Summary

Keep open. Current main still ships the narrower workspace-only filesystem guard, not the requested `tools.fileAccess` sandbox/full mode with allow/deny path lists and exec-command filesystem enforcement. The latest issue discussion also explicitly preserves this remaining scope rather than treating `tools.fs.workspaceOnly` as completion.

## Best Possible Solution

Keep #7722 open until OpenClaw either implements a real filesystem access policy surface, likely under `tools.fs` or an equivalent versioned config contract, with allow/deny path rules enforced across read/write/edit/apply_patch and a defensible exec strategy, or a maintainer explicitly narrows or rejects that product direction. Treat the shipped `tools.fs.workspaceOnly` support as useful partial hardening, not completion of this issue.
