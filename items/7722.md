## Summary

Keep open. Current main has a narrower `tools.fs.workspaceOnly` guard for read/write/edit/apply_patch, which addresses the #5948-style workspace-only subset mentioned in the discussion, but it does not implement the requested `tools.fileAccess` allowedPaths/denyPaths model or path enforcement for exec commands. #12202 and huss-mo's comment reinforce that the remaining request is tool-layer path access control, not prompt guidance.
