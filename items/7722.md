## Summary

Current main still only supports the narrower `tools.fs.workspaceOnly` guard. It does not accept the requested `tools.fileAccess` shape, has no `mode`, `allowedPaths`, or `denyPaths` config contract, and does not pass any filesystem allow/deny policy into exec command execution. The latest issue discussion also explicitly kept this remaining scope open.

## Best Possible Solution

Keep #7722 open until OpenClaw either implements a versioned filesystem access policy surface, likely under `tools.fs` or an equivalent config contract, with allow/deny path rules enforced across read/write/edit/apply_patch and a defensible exec strategy, or a maintainer explicitly narrows or rejects that product direction. Treat shipped `tools.fs.workspaceOnly` as useful partial hardening, not completion of this issue.
