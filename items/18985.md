## Summary

Keep open. Current main still does not implement the requested Windows 11 MSYS2 install.sh support or fish-specific PATH guidance. The Unix installer still accepts only macOS, Linux, or WSL before exiting unsupported, docs route native Windows users to install.ps1 or WSL2, and the related Microsoft tracker PR #49126 was closed unmerged. A maintainer-authored Codex review comment on the issue already reached the same keep-open conclusion on 2026-04-26, and current main has not changed the checked installer/docs surface.

## Best Possible Solution

Keep #18985 open until maintainers either implement explicit MSYS2/MINGW install.sh support with fish config.fish PATH persistence/warnings plus docs and installer smoke coverage, or make a documented product decision that Windows Unix-like environments should use WSL2 or install.ps1 instead.

## Risks / Open Questions

- A real fix should be validated in an actual Windows 11 MSYS2/MINGW plus fish environment because OSTYPE/MSYSTEM values, path conversion, package-manager behavior, and shell startup files are environment-sensitive.
- Maintainers could choose to document MSYS2 as unsupported and route users to WSL2 or install.ps1, but that product decision is not currently encoded in code/docs as a closeable resolution for this feature request.
