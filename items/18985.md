## Summary

Keep #18985 open. Current main still rejects Windows MSYS2/MINGW-style `install.sh` execution before installation work starts, and the installer PATH repair/warning path remains bash/zsh-only rather than fish-aware. Docs and smoke surfaces still route Windows users to WSL2 or `install.ps1`; existing fish completion support is adjacent but does not solve installer PATH persistence.

## Best Possible Solution

Keep #18985 open until maintainers either implement explicit MSYS2/MINGW `install.sh` support with correct OS detection, Windows/MSYS path and package-manager handling, fish `config.fish` PATH persistence or guidance, docs, and installer smoke coverage; or make a documented product decision that Windows Unix-like environments should use WSL2 or `install.ps1` instead.
