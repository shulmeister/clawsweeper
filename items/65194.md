## Summary

Keep this PR open. Current main still has the iMessage duplicate default/named watcher path, the linked same-author issue #65141 is still open, and the PR is an implementation candidate rather than something already shipped. The submitted patch also needs revision because review feedback correctly points out that the startup path does not use the new filtered helper and that disabled or case-variant account entries can produce regressions.

## Best Possible Solution

Keep the PR open and request a narrower iMessage-owned revision. The fix should affect the account list actually used by gateway monitor startup, skip `default` only when it is fallback-only and a startup-eligible named account covers the same local backend, preserve `default` when it has distinct backend/runtime identity such as `cliPath` or `dbPath`, handle normalized default keys, and add focused regression tests for the reported config plus disabled-placeholder and distinct-runtime cases.
