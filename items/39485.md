## Summary

Keep PR #39485 open. Current main still routes Discord account traffic through explicit binding tiers and then the configured default agent; it has not implemented the requested Discord accountId-to-agentId fallback. The PR remains useful for #39428, but the proposed patch is not merge-ready because existing review comments identify a real empty-agent-list phantom-routing edge case and an ambiguous matchedBy label.

## Best Possible Solution

Keep this PR open or supersede it with a maintainer-owned fix that adds a narrow Discord-only implicit accountId-to-existing-agent fallback after explicit binding tiers, skips accountId "default", verifies the matched agent exists in agents.list, uses a distinct match reason such as "implicit.account", and adds regression coverage for configured-agent matching, default-account behavior, and the empty-agent-list case. Keep #39428 as the canonical bug until that guarded behavior lands.
