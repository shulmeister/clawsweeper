## Summary

Keep PR #39485 open. Current main still does not implement Discord `accountId` to matching agent fallback, and the PR remains useful but not merge-ready because its review comments identify real routing and observability issues that still need changes.

## Best Possible Solution

Keep this item open or supersede it with a maintainer-owned fix that adds a guarded Discord `accountId` to existing-agent fallback, uses a distinct match reason such as `implicit.account`, preserves the `default` account guard, and adds regression coverage for configured-agent matching plus the empty-agent-list edge case.
