## Summary

Keep PR #39485 open. Current main still lacks the requested Discord accountId-to-agentId fallback, the linked canonical bug #39428 remains open, and this PR is useful but not merge-ready because existing review comments identify a real empty-agent-list routing regression plus an observability issue in the proposed matchedBy label.

## Best Possible Solution

Keep this PR open or supersede it with a maintainer-owned fix that adds a narrow Discord accountId-to-existing-agent fallback after explicit binding tiers, skips the default account id, verifies the candidate exists in agents.list, uses a distinct match reason such as implicit.account, and adds regression coverage for configured-agent matching, default-account behavior, and the empty-agent-list edge case. Keep #39428 as the canonical bug until guarded behavior lands.
