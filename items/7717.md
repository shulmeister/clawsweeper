## Summary

Keep #7717 open. Current main still lacks a Discord role-mention wake trigger: role mentions are parsed as metadata and counted for other-mention filtering, but there is no `triggerOnRoles` config field and the wake decision does not compare mentioned role IDs against configured trigger role IDs.

## Best Possible Solution

Keep #7717 open and implement this in the bundled Discord plugin as an additive guild/channel config surface, for example `triggerOnRoles?: string[]`. Preflight should compare Discord `mentionedRoles`/`mention_roles` against the effective configured role IDs and treat a match as a wake mention while preserving `requireMention`, `ignoreOtherMentions`, @everyone, explicit bot mentions, reply-to-bot behavior, sender allowlists, and role-based routing. Add focused schema/type metadata tests, preflight tests for matched and unmatched role mentions, and a Discord docs update at https://docs.openclaw.ai/channels/discord.
