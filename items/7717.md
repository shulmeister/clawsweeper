## Summary

Keep #7717 open. Current main still does not implement Discord role-mention wake triggers: role mentions are captured as Discord metadata and can affect other-mention filtering, but the wake decision only considers @everyone, explicit bot mentions, configured text mention patterns, and reply-to-bot behavior. The existing `roles` config is a sender allowlist/routing input, not a list of mentioned role IDs that should wake the bot.

## Best Possible Solution

Implement this in the bundled Discord plugin as an additive config surface, likely `triggerOnRoles?: string[]` on guild/channel config. Preflight should compare Discord `mentionedRoles`/`mention_roles` against effective configured role IDs and treat a match as a wake mention while preserving existing `requireMention`, `ignoreOtherMentions`, @everyone, explicit bot mention, reply-to-bot, sender allowlist, and role-based routing behavior. Add focused schema/type metadata tests, preflight tests for matched and unmatched role mentions, and a Discord docs note at https://docs.openclaw.ai/channels/discord.
