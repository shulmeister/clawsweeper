## Summary

Keep #7717 open. Current main still lacks a Discord role-mention trigger setting; existing `roles` fields are sender allowlists, and Discord preflight treats role mentions only as generic mention metadata/other-mention gating, not as a wake trigger. The prior Apr 26 Codex review comment in the issue reached the same keep-open conclusion, and current main still matches that gap.

## Best Possible Solution

Keep #7717 open and implement it in the bundled Discord plugin as an additive guild/channel config surface, for example `triggerOnRoles?: string[]`. Wire it through config types, strict Zod schema, generated metadata/help, public Discord docs, and preflight. Preflight should compare `message.mentionedRoles` / `mention_roles` against configured role IDs and treat a match as a wake mention while preserving existing `requireMention`, `ignoreOtherMentions`, @everyone/@here, reply-to-bot, sender `users`/`roles` allowlist, and routing behavior. Add focused config/schema tests plus preflight tests for matched and unmatched role mentions.
