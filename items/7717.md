## Summary

Keep open. Current main still lacks first-class Discord role-mention trigger support like the requested per-guild `triggerOnRoles` role-ID list. Role mentions are recognized as mention metadata, but they are not treated as a bot wake trigger unless the user separately configures generic text mention patterns.

## Evidence

- **Discord guild/channel config has no role-trigger field:** The strict Discord guild and channel schemas expose `requireMention`, `ignoreOtherMentions`, `users`, and `roles`, but no `triggerOnRoles` or equivalent role-mention trigger list. The existing `roles` field is part of sender allowlisting, not wake-trigger matching.

  - command: `nl -ba src/config/zod-schema.providers-core.ts | sed -n '420,520p'`
  - sha: [1dfa52d0711d](https://github.com/openclaw/openclaw/commit/1dfa52d0711dfc86e1c04538fe46e3d43961766f)
- **Role mentions are counted as metadata but not as the bot mention trigger:** Discord preflight sets `explicitlyMentioned` only when `mentionedUsers` contains the bot ID. `hasAnyMention` includes `mentionedRoles`, but the mention decision later skips guild messages when `requireMention` is true and the computed `wasMentioned` is false.
