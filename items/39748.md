## Summary

Keep open. Current main still has the Nostr relay URL validation gap: the config schema accepts arbitrary relay strings, CLI and Control UI setup only enforce ws/wss schemes, and resolved relay values flow into the Nostr runtime. The PR remains relevant, but should be refreshed against current main and broadened to share one validator across the schema and setup paths.

## Best Possible Solution

Keep the PR open and refresh it against current main. The maintainer path should be a shared Nostr relay URL validator that allows remote `wss://`, permits `ws://` only for localhost/loopback development hosts, and is reused by `NostrConfigSchema`, CLI setup, Control UI setup, generated config metadata, and focused regression tests in the current Nostr test structure. Drop unrelated baseline churn during the refresh.
