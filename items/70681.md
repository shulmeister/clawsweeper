## Summary

Keep #70681 open. Current main still has the Unix `lsof -Fpc` command-name filter that the PR targets, so the macOS `p_comm` vs rewritten `argv[0]` bug is not implemented or superseded on main. The PR is a plausible core infra fix, but it should address the existing review finding about `pollPortOnce` verifier timeouts before merge.

## Best Possible Solution

Keep #70681 open, or land an equivalent core fix: stop using lsof `c` records for Unix gateway classification, verify candidate PIDs with ps argv through the existing gateway argv matcher, apply the short `POLL_SPAWN_TIMEOUT_MS` on polling paths, and keep regression tests for macOS `cnode` gateway output plus unrelated listeners.
