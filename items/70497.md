## Summary

Keep PR #70497 open. Current main still does not strip JSON Schema `if`/`then`/`else` from Gemini tool schemas: the Gemini cleaner's unsupported-keyword set stops at existing keywords such as `not`, and the cleaning loop preserves unknown keys. The linked issue #69768 remains open and includes concrete Gemini cron/tool-schema failures plus a triage comment pointing to these exact missing keywords. PR #70497 is a focused re-open of closed-unmerged PR #69964 and adds the missing keywords plus a targeted regression test, so it is not obsolete or superseded.

## Best Possible Solution

Keep the PR open and have a maintainer validate or land the focused fix: add `if`, `then`, and `else` to `GEMINI_UNSUPPORTED_SCHEMA_KEYWORDS`, keep the regression test from PR #70497, run the targeted schema test/lint/format checks from the PR body, and then use the merged PR to resolve the linked Gemini cron/tool-schema issue #69768 if the maintainer agrees that conditional schemas are the confirmed root cause.
