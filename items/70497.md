## Summary

PR #70497 is still relevant. Current main does not strip JSON Schema conditional keywords `if`, `then`, or `else` from Gemini tool schemas, while the PR adds exactly those keywords plus a targeted regression test. The PR is a reopened copy of #69964 after the original fork became private, not obsolete or superseded.

## Best Possible Solution

Keep the PR open and integrate the narrow fix: add `if`, `then`, and `else` to the Gemini unsupported keyword set, keep the regression test, then run the targeted schema test/lint/format checks and the changed gate before merge.
