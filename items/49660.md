## Summary

Keep open. Current `main` does not implement the PR's startup orphan-self-lock cleanup, so there is no strong basis to close this PR as already fixed or obsolete.

## Evidence

- **Startup cleanup still only removes locks already marked stale:** `cleanStaleLockFiles()` inspects each `.jsonl.lock` via `inspectLockPayload()` and removes it only when `lockInfo.stale` is already true; it does not apply orphan-self-lock detection during startup cleanup.
  - file: [src/agents/session-write-lock.ts:418](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/src/agents/session-write-lock.ts#L418)
  - command: `nl -ba src/agents/session-write-lock.ts | sed -n '418,472p'`
  - sha: [320d52a23e41](https://github.com/openclaw/openclaw/commit/320d52a23e41a326287826219a20f1f29f334554)
- **Orphan-self-lock helper on main still rejects any lock with valid `starttime`:** `shouldTreatAsOrphanSelfLock()` returns `false` as soon as `payload.starttime` is a valid integer, so current-PID locks with recorded starttime are not treated as orphaned.
  - file: [src/agents/session-write-lock.ts:403](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/src/agents/session-write-lock.ts#L403)
  - command: `nl -ba src/agents/session-write-lock.ts | sed -n '403,415p'`
  - sha: [320d52a23e41](https://github.com/openclaw/openclaw/commit/320d52a23e41a326287826219a20f1f29f334554)
- **Fresh live lock payloads are considered non-stale:** `inspectLockPayload()` only marks locks stale for missing/dead/recycled PID or invalid/too-old timestamp. A current-PID lock with fresh `createdAt` remains non-stale on main.
  - file: [src/agents/session-write-lock.ts:325](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/src/agents/session-write-lock.ts#L325)

  - sha: [320d52a23e41](https://github.com/openclaw/openclaw/commit/320d52a23e41a326287826219a20f1f29f334554)
- **Gateway startup still relies on `cleanStaleLockFiles()`:** `startGatewaySidecars()` calls `cleanStaleLockFiles()` for each sessions directory during startup, so the missing orphan-self-lock handling affects the exact path described by the PR.
  - file: [src/gateway/server-startup-post-attach.ts:149](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/src/gateway/server-startup-post-attach.ts#L149)
