## Summary

Keep open. Current main supports explicit agent creation/update and hot `agents.list` reloads, but it still lacks the requested `agents.autoDiscover` config surface and does not scan external workspace globs for agent config files.

## Best Possible Solution

Keep the issue open and narrow it around external workspace agent discovery now that explicit zero-restart agent management exists. A concrete design should define the `agents.autoDiscover` config/API shape, trusted directory rules, schema validation, deterministic precedence versus explicit `agents.list`, hot-reload/watch behavior, docs, and tests for unsafe workspace config injection.
