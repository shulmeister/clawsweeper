## Summary

Keep open. Current main addresses the related zero-restart pain with explicit agent creation/update RPCs and hot `agents.list` reloads, but it still does not implement external workspace auto-discovery or an `agents.autoDiscover` config surface. Agent config remains explicit through `agents.defaults` and `agents.list`, and runtime agent enumeration still starts from configured agent entries rather than scanning workspace globs for `agent.json` or `openclaw-agent.json`.

## Best Possible Solution

Keep the issue open, but narrow it around external workspace agent discovery now that explicit zero-restart agent management is partly available. A concrete implementation should add a reviewed config/API design, trusted-directory and schema validation rules, deterministic merge precedence with explicit `agents.list`, hot-reload behavior, docs, and tests around unsafe or untrusted workspace config injection.
