## Summary

Keep #32530 open. Current main has better manual/runtime agent management than the original report assumed, including hot handling for `agents.list` changes and Gateway RPC methods for agent create/update/delete, but it still does not implement external workspace auto-discovery, an `agents.autoDiscover` config contract, scanning for `agent.json`/`openclaw-agent.json`, or a plugin API that can contribute discovered agent definitions. TinyTb's #27706 comment is useful related zero-restart context, not a superseding implementation of workspace discovery.

## Best Possible Solution

Keep #32530 open and narrow it around external workspace agent discovery. The best maintainer path is either a first-class `agents.autoDiscover` config contract or a documented plugin-backed agent-definition discovery API that runs before normal agent enumeration, with trusted roots, accepted filenames, schema validation, deterministic ordering, explicit precedence against `agents.list`, watch/hot-reload behavior, docs, and regression tests. Treat #27706 as related zero-restart context, not as a superseding tracker.

## Risks / Open Questions

- A local script or plugin CLI can approximate the workflow by rewriting `agents.list`, and current hot reload makes that less disruptive, but that remains an out-of-band workaround rather than supported discovery.
- Any implementation still needs trusted roots, schema validation, deterministic ordering, precedence against explicit `agents.list`, and watch/reload behavior to avoid config injection and nondeterministic routing.
