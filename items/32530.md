## Summary

Keep issue #32530 open. Current main supports explicit agent registration and hot reload for `agents.list`, but it still has no `agents.autoDiscover` config surface, no external workspace glob scanner for agent config files, and no merge/precedence contract for discovered agents. TinyTb's #27706 comment is useful context for zero-restart agent management, but that related closed issue does not supersede the requested workspace-discovery behavior.

## Best Possible Solution

Keep #32530 open and narrow it around external workspace agent discovery. A good implementation should define either a core `agents.autoDiscover` schema or an explicitly plugin-backed discovery API, trusted roots, accepted filenames, schema validation, deterministic ordering, precedence versus explicit `agents.list`, hot reload/watch behavior, docs, and tests for unsafe workspace config injection.

## Risks / Open Questions

- External workspace discovery has real security and operations design questions: trusted scan roots, arbitrary config injection, deterministic scan/merge order, precedence against explicit `agents.list`, and watch/hot-reload behavior.
- A plugin could potentially help by scanning and writing explicit config through existing runtime services, but current main does not document a stable agent-discovery/merge contract; that needs maintainer product/API design before closing this as ClawHub-only.
