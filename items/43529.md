## Summary

Keep PR #43529 open. Current main still has no timeout-driven session reset in `runReplyAgent`; it only resets sessions for compaction failure, role-ordering conflict, and Gemini session corruption. Main does include timeout-triggered compaction, but that path explicitly falls through to timeout error payloads when compaction fails, is skipped, or reaches its cap, so it does not fully cover the linked #20910 death-spiral class. The PR is not merge-ready as-is because current main has diverged and the latest review context identifies an unhandled timeout error-payload surface, but the work is still useful and not obsolete.

## Best Possible Solution

Keep the PR open, but ask for a rebase and a narrower current-main patch: add a timeout session-rotation path beside the existing compaction/role-order reset callbacks, gate it to definitive all-candidate timeout exhaustion after transient retry and outside heartbeat turns, handle both thrown fallback failures and embedded timeout error payloads, and add regression tests in the current `agent-runner` test files.
