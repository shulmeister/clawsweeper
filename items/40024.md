## Summary

Keep #40024 open. Current main still does not satisfy the reporter's baseUrl-only OpenAI-compatible local proxy case: custom provider fallback resolution defaults to `openai-responses`, Chat Completions is selected only for `openai-completions`, and loopback/private endpoint fetches still require explicit `allowPrivateNetwork` opt-in.

## Best Possible Solution

Keep the issue open and make the provider contract explicit. Either infer/select `openai-completions` for custom local OpenAI-compatible `/v1` providers that do not support Responses, or fail early with an actionable configuration error requiring `models.providers.<id>.api: "openai-completions"` and `models.providers.<id>.request.allowPrivateNetwork: true` for loopback/private endpoints. Add a regression test for `local-agent-proxy` at `http://127.0.0.1:3000/v1` covering the selected transport path and error behavior.
