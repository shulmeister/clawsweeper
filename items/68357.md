## Summary

Current main still has the Feishu auto-mention-forward behavior reported in #68357, but the same remaining work is already tracked by the exact linked PR #68444. That PR is open and unmerged, explicitly closes #68357, adds `mentionForwardMode: "auto" | "none"`, wires the policy into both agent prompt injection and outbound reply delivery, updates Feishu docs, and adds schema/policy/bot tests. #56692 is related broader group-chat context work, not the canonical tracker for this Feishu-specific opt-out.

## Best Possible Solution

Close #68357 as superseded by the exact open implementation PR #68444. Continue code review and any required rebase/nit fixes on #68444 or a direct successor, then merge the additive Feishu `mentionForwardMode` implementation with its docs and regression tests.
