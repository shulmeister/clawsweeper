## Summary

Current main still lacks the PR's requested behavior: the visible topbar hamburger toggles drawer state, but chat focus mode suppresses the drawer/nav, so the click remains visually ineffective while user-initiated focus mode is active.

## Best Possible Solution

Keep this PR open or replace it with an equivalent current-main patch: update the `.topbar-nav-toggle` click path so user-initiated `chatFocusMode` exits focus mode and reveals the nav/drawer, preserve onboarding-controlled focus behavior, and add a browser regression test for clicking the hamburger while `.shell--chat-focus` is active.
