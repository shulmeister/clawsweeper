## Summary

Keep PR #38635 open. Current main still does not implement the PR's central behavior: the visible mobile hamburger toggles drawer state only, while focus mode masks the drawer/nav, so clicking it in user-initiated focus mode remains ineffective. Main has a separate `Exit focus mode` button, which mitigates discoverability but does not supersede the requested hamburger behavior. The PR also has useful discussion: Greptile identified onboarding and test-null-guard issues, and the PR head context says those were addressed in commit ce4d9fbb88f8af840482903976088259c7f37c98.

## Best Possible Solution

Keep this PR open or replace it with an equivalent current-main patch: update the current `topbar-nav-toggle` click path so user-initiated `chatFocusMode` exits focus and reveals the nav/drawer, preserve onboarding-controlled focus behavior, and add a browser regression test for clicking the hamburger while `shell--chat-focus` is active.
