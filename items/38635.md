## Summary

Current `main` does not implement this PR’s hamburger-click behavior. In `ui/src/ui/app-render.ts`, focus mode still forces `navDrawerOpen` false (`chatFocus` blocks the drawer at lines 674-675), while the visible mobile hamburger at lines 1236-1246 only toggles `state.navDrawerOpen`. That means clicking the hamburger during chat focus still has no visible effect. The PR’s original patch also targets the old sidebar `nav-collapse-toggle` at lines 1289-1297, but mobile now uses `topbar-nav-toggle`, so the branch is stale rather than already implemented.
