## Summary

Keep open. Current `main` still has the same skill-download code path that can hang after headers arrive: `src/agents/skills-install-download.ts` passes `timeoutMs` into `fetchWithSsrFGuard(...)` and then waits on `await pipeline(readable, file)` with no separate pipeline abort wiring or mid-stream stall regression test. The Telegram follow-up queueing part of the report is also consistent with the current per-chat/per-topic sequential lane design, so the report still maps to plausible current behavior rather than an obsolete bug.

## Evidence

- **Skill download path still blocks on raw stream pipeline:** `downloadFile()` calls `fetchWithSsrFGuard({ url, timeoutMs })`, then immediately does `await pipeline(readable, file)`. Cleanup/release only happens in the `finally` after that await returns, so there is no explicit pipeline-level timeout/cancel handling in this function.
