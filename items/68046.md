## Summary

Keep open. Current `main` still strips non-text MCP tool-result blocks down to `{type,text}` in `src/gateway/mcp-http.handlers.ts`, so the underlying bug described by this PR is still present. The PR also has unresolved review feedback: its proposed passthrough guard would accept arbitrary or malformed typed blocks, so it is not yet a conservative maintainer-close candidate.

## Evidence
