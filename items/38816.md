## Summary

Keep #38816 open. Current main has improved explicit provider-prefixed image-model routing, including Ollama media-understanding support, but the reported bare `agents.defaults.imageModel.primary` case is still not clearly implemented. Bare image-model refs are still resolved through a default provider in the image tool fallback path, and related fix PRs in the provided GitHub context were closed unmerged rather than shipped.

## Best Possible Solution

Keep this issue open until the image model fallback resolver either infers a unique configured provider for bare `agents.defaults.imageModel` refs, documents and enforces provider-prefixed imageModel values as the intended contract with an explicit maintainer decision, or a new canonical maintainer issue/PR tracks the remaining bare-ID behavior.
