## Summary

Keep #38816 open. Current main still normalizes a bare `agents.defaults.imageModel.primary` through the image fallback default provider, so `primary: "moondream"` can still become `anthropic/moondream` instead of inferring the configured Ollama/LiteLLM provider. Main has fixes for provider-prefixed/custom configured image models, but not the providerless imageModel-primary path reported here.

## Best Possible Solution

Keep this issue open until OpenClaw either infers a unique configured provider for bare `agents.defaults.imageModel.primary` / fallback entries using the same configured-model evidence as chat model defaults, or explicitly rejects providerless imageModel refs with validation and documentation. Continue recommending explicit `provider/model` values such as `ollama/qwen2.5vl:7b` as the safe workaround.
