## Summary

Keep open. Current main has related image-tool improvements for fully qualified provider/model refs and configured provider metadata, but the specific reported case still is not clearly fixed: a bare `agents.defaults.imageModel.primary` such as `moondream` is still normalized through a default provider rather than inferred from Ollama/LiteLLM configuration.

## Best Possible Solution

Keep tracking this issue until maintainers either implement provider inference for bare `agents.defaults.imageModel` refs from configured provider/model rows, or explicitly decide that image model config must always use full `<provider/model>` refs and add clear validation/error messaging. A focused regression should cover `imageModel.primary: "moondream"` with a configured Ollama/LiteLLM image-capable model.
