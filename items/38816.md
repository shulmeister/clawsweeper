## Summary

Keep #38816 open. Current main documents and supports explicit provider-prefixed image models such as `ollama/qwen2.5vl:7b`, but the reported bare `agents.defaults.imageModel.primary: "moondream"` path still resolves through the image fallback default provider and can become `anthropic/moondream` instead of inferring a configured Ollama/LiteLLM provider. Related fix PRs in the provided context (#47077 and #71458) are closed and unmerged, so there is no canonical shipped fix to close against.

## Best Possible Solution

Keep this issue open until maintainers either add imageModel-specific bare-provider inference from `agents.defaults.models` / `models.providers`, or explicitly define bare `imageModel` refs as unsupported and enforce that contract with validation plus a clear configuration/runtime error. Continue documenting explicit `provider/model` imageModel values as the safe workaround.
