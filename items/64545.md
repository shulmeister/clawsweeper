## Summary

Keep open. Current main still derives the Mattermost interaction signing secret from the bot token using the public hardcoded HMAC key, and the latest release contains the same behavior. The callback route is plugin-managed auth; after a valid `_token`, the handler can enqueue a system event. The linked PR #64546 is a relevant proposed fix but is open/unmerged in the provided GitHub context, so this issue remains the canonical tracker.

## Best Possible Solution

Keep #64545 open as the canonical Mattermost security bug. The best fix belongs in the Mattermost plugin: generate and persist a per-account random interaction signing secret separate from the bot token, update schema/types/docs/tests, treat source-IP allowlisting as defense-in-depth, and review PR #64546 or a successor before closing only after the fix lands and ships.
