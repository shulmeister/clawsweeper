## Summary

Keep open. Current main still derives the Mattermost interaction signing secret from the bot token using the public hardcoded HMAC key, and current docs still publish that derivation. The same-author PR #64546 directly proposes a fix with closing syntax, but it is open and unmerged, so this issue should remain the tracker until a reviewed fix lands.

## Best Possible Solution

Keep this issue open as the canonical Mattermost security tracker. Review and revise PR #64546 or a successor so the Mattermost plugin uses a persistent, SecretRef-aware per-account interaction signing secret separate from the bot token; update schema/types/secret contract/docs/tests, and close only after the fix lands and ships.
