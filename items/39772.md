## Summary

Keep PR #39772 open. Current main at e672b61417af5c45b0431df6d9109a1f4b618ef5 still does not implement the PR's first-preview-only reply gating in Telegram draft/message streaming, and the PR remains a plausible fix direction for stream rotations. It should not be closed as implemented or superseded, but it also should not land unchanged because later review concerns about first-send failure recovery and `replyToMode: "all"` semantics are still valid.

## Best Possible Solution

Keep the PR open for maintainer or author revision. The best fix should suppress the transient Telegram `Deleted message` preview artifact while preserving reply-threading semantics: distinguish `replyToMode: "first"` from `"all"`, preserve `message_thread_id` after stream rotation, avoid consuming reply context before the first successful send, and add regression coverage for `forceNewMessage`, thread params, first-send failure/materialize recovery, and all-mode behavior.
