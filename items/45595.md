## Summary

Keep open. Current main still has the reported symlink/realpath asymmetry: approval matching is driven by the resolved candidate path, while execution pins to the real path. The discussion comment accurately identifies the same code-path mismatch, and the checked-in test coverage only proves that a symlink allowlist entry can still execute the canonical target, not that real-path allowlisting or symlink-target drift is fixed.
