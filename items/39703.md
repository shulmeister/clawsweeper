## Summary

Keep #39703 open. Current main still has FIFO cron schedule cache behavior: cache hits return without moving the key, and capacity eviction deletes the first Map key. Existing tests cover cache reuse/size only, not key-level LRU survival. The PR remains the focused fix, and its second commit addresses the bot review by adding key-level cache assertions.

## Best Possible Solution

Keep #39703 open as the canonical cron cache LRU fix, or land an equivalent maintainer patch that promotes cache hits with delete/re-set and includes key-level regression coverage proving the promoted expression survives eviction while the next-oldest entry is evicted.

## Evidence

  - file: [src/cron/schedule.ts:23](https://github.com/openclaw/openclaw/blob/e672b61417af5c45b0431df6d9109a1f4b618ef5/src/cron/schedule.ts#L23)
