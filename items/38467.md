## Summary

Keep #38467 open. Current main still backs WhatsApp directory peers from configured allowFrom entries only, has no Baileys contacts.upsert handler or persisted contact store, and has no WhatsApp contact-storage config/docs/tests. The related PR #33924 was closed unmerged and addressed outbound allowSendTo gating, not contact-directory ingestion or name lookup.

## Best Possible Solution

Keep this issue open for the WhatsApp plugin owner/product decision. A complete fix likely belongs in the WhatsApp plugin: add an opt-in local contact store scoped by account/authDir, register and sanitize a Baileys contacts.upsert handler, merge/dedupe saved contacts into directory.listPeers with query support, make outbound WhatsApp target resolution search peer contacts for name inputs, and add privacy docs plus unit/contract tests.
