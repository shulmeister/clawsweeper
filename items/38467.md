## Summary

Keep #38467 open. Current main still does not ingest Baileys `contacts.upsert`, persist WhatsApp phone contacts, expose them through the WhatsApp directory, or provide an `allowSendTo` substitute from the related closed PR. The existing directory path remains config-backed by `allowFrom` entries.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin: add account-scoped local contact persistence populated from Baileys `contacts.upsert`, add explicit privacy/config handling such as an opt-in `storeContacts` if maintainers want persistence gated, merge stored contacts into `directory.listPeers` or `listPeersLive` with `id`/`name`/`handle`/phone metadata, adjust outbound/direct target resolution so unique contact names can resolve safely, and add tests plus docs for ingestion, filtering, ambiguity, privacy, and message-tool behavior.
