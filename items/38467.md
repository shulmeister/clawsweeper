## Summary

Keep #38467 open. Current main still exposes WhatsApp directory peers from configured allowFrom entries only; it does not persist Baileys contact-list events, expose saved contact names/phone numbers through the WhatsApp directory, or resolve outbound WhatsApp targets by contact name. The prior ClawSweeper comment and related PR #33924 context align with this finding: #33924 was closed unmerged and covered outbound allowlist policy, not contact-directory ingestion.

## Best Possible Solution

Keep the issue open and implement the feature in the bundled WhatsApp plugin: add account-scoped local contact persistence populated from Baileys contact events, gate persistence with explicit privacy/config handling, merge stored contacts into the WhatsApp directory with name/handle/phone metadata, make outbound target resolution use unique contact-name matches safely, and add tests/docs for ingestion, filtering, ambiguity, privacy, and message-tool behavior.
