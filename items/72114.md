## Summary

Keep #72114 open. Current main has adjacent bundled runtime-deps hardening and update follow-up work, but I do not see a source-backed fix for the specific cold-cache browser staging failure plus service-manager SIGTERM/SIGKILL shutdown race. The related #72058 tracker and #72084 PR remain relevant, but the provided discussion does not clearly make either one the canonical owner for this narrower race condition.

## Best Possible Solution

Keep #72114 open as a focused runtime-deps/update restart-race report. The likely core fix is to make bundled runtime-deps staging observable as pending restart/shutdown work, avoid service-manager restarts while an npm staging subprocess is active, harden cleanup/retry for interrupted staging, and add regression coverage for cold-cache browser runtime-deps staging during an update-driven gateway restart. If maintainers want #72058 to own this, mark #72114 as a duplicate only after that tracker explicitly covers the shutdown/SIGKILL failure mode.
