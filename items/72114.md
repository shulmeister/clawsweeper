## Summary

Keep #72114 open. Current main has adjacent bundled-runtime-deps hardening, but I do not see a source-backed fix for the reported cold-cache browser staging failure or the update/systemd shutdown race. Related #72058 is open and similar, but the provided discussion does not clearly make it the canonical owner of this narrower SIGTERM-to-SIGKILL failure mode.

## Best Possible Solution

Keep #72114 open as a focused runtime-deps/update restart-race report, or explicitly fold it into #72058 only after a maintainer confirms #72058 owns the shutdown/SIGKILL failure mode. The likely core fix is to make bundled runtime-deps staging visible to restart/shutdown deferral, avoid service-manager restarts while a staging npm subprocess is active, and add regression coverage for cold-cache bundled runtime-deps staging during an update-driven gateway restart.
