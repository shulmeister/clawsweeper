## Summary

Current main no longer persists outbound TTS/media into channel session transcripts as inline base64, and any assistant-side audio blocks that do exist are skipped during model replay, so the reported context-overflow path is already addressed on main.

## Evidence

- **Channel transcript mirroring writes text-only assistant entries:** Outbound delivery mirrors successful sends through `appendAssistantMessageToSessionTranscript`, passing only the resolved mirror text into the transcript append path.
  - file: [src/infra/outbound/deliver.ts:1076](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/infra/outbound/deliver.ts#L1076)
  - command: `nl -ba src/infra/outbound/deliver.ts | sed -n '1074,1090p'`
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Mirrored media becomes filenames, not inline payload bytes:** `resolveMirroredTranscriptText` collapses `mediaUrls` to decoded basenames (or `"media"`), so a TTS file path is mirrored as text like `reply.ogg`, not as embedded base64 audio.
  - file: [src/config/sessions/transcript-mirror.ts:34](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/config/sessions/transcript-mirror.ts#L34)
  - command: `nl -ba src/config/sessions/transcript-mirror.ts | sed -n '1,80p'`
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Transcript mirror persists delivery-mirror text content:** `appendAssistantMessageToSessionTranscript` stores a `delivery-mirror` assistant message whose content is a single text block built from the mirrored text; it does not write media blocks here.
  - file: [src/config/sessions/transcript.ts:109](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/config/sessions/transcript.ts#L109)
  - command: `nl -ba src/config/sessions/transcript.ts | sed -n '94,132p'`
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **OpenAI replay skips assistant audio blocks:** During replay to OpenAI-compatible transports, assistant history only emits `thinking`, `text`, and `toolCall` blocks; other assistant block types are ignored, so assistant audio content does not re-enter model context.
  - file: [src/agents/openai-transport-stream.ts:274](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/agents/openai-transport-stream.ts#L274)
  - command: `nl -ba src/agents/openai-transport-stream.ts | sed -n '272,320p'`
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Anthropic replay also skips assistant audio blocks:** Anthropic replay likewise only serializes assistant `text`, `thinking`, and `toolCall` blocks, leaving assistant audio blocks out of the provider request.
  - file: [src/agents/anthropic-transport-stream.ts:306](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/agents/anthropic-transport-stream.ts#L306)
  - command: `nl -ba src/agents/anthropic-transport-stream.ts | sed -n '306,346p'`
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Chat history sanitization strips inline audio data:** For gateway chat history rendering, `sanitizeChatHistoryContentBlock` deletes `source.data` from `audio` blocks and replaces it with size metadata, preventing base64 audio from being surfaced back out through history APIs.
  - file: [src/gateway/server-methods/chat.ts:954](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/gateway/server-methods/chat.ts#L954)
  - command: `nl -ba src/gateway/server-methods/chat.ts | sed -n '949,964p'`
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)

## Risks / Open Questions

- This review is based on current `main` only; I did not tie the fix to a released tag, so release pickup remains unknown.
- Control-UI `gateway-injected` transcript files can still contain embedded audio blocks for UI purposes, but current replay and history-sanitization paths prevent those blocks from causing the reported context-bloat behavior.
