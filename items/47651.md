## Summary

Keep open. Current main partially addresses the idea with opt-in Gemini Embedding 2 structured inputs for image/audio files from configured extraPaths, but #47651 asks for broader raw media memory indexing: images, video, audio, PDFs, and media a user sends and marks worth remembering. The current config/schema/docs still scope multimodal memory to image/audio extraPaths and keep default memory roots Markdown-only, so meaningful requested capability remains.
