## Summary

Keep #20460 open. Current main still stores config backups beside the config file as `${configPath}.bak` plus numbered `.bak.1` through `.bak.4`; it does not create a `config-backup/` directory or datetime-suffixed config backup names. The linked PR #20877 is relevant prior art, but the provided GitHub context shows it was closed unmerged.

## Best Possible Solution

Keep this open as a core config enhancement. Implement a dedicated backup directory and datetime naming in `src/config/backup-rotation.ts`, then update `writeConfigFile`, include mutation writes, recovery/audit paths, retention pruning, chmod handling, tests, and user-facing docs. Use PR #20877 as prior art, but fix its retention issue and define compatibility for existing adjacent numbered backups.
