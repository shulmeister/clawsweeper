## Summary

Keep #20460 open. Current main still stores automatic config-write backups beside the config file as `.bak` plus a numbered `.bak.N` ring; it does not create a dedicated `config-backup/` directory or datetime-named backup files. Related PR #20877 was closed unmerged, and #39982/#40392 remain open candidate implementations rather than shipped behavior.

## Best Possible Solution

Keep #20460 open as the core config-backup ergonomics request unless maintainers explicitly consolidate it into a canonical follow-up. A complete fix should update `src/config/backup-rotation.ts`, config write/recovery/audit paths, doctor/docs, and tests to define the dedicated `config-backup/` directory, datetime naming, retention cap, collision handling, and compatibility or migration behavior for existing adjacent numbered backups, coordinating with #20877, #39923, #39982, and #40392.
