## Summary

Keep open. Current main still implements adjacent numbered config backups (`openclaw.json.bak`, `.bak.1` through `.bak.4`) and ordinary config writes still call that maintenance path. The linked PR #20877 is useful prior art for this request, but it was closed unmerged and the requested `config-backup/` directory plus datetime naming is not present on main.

## Best Possible Solution

Keep this as an open core config enhancement. Implement the directory and datetime naming in `src/config/backup-rotation.ts` and the config write/mutation/recovery paths, update tests and user-facing docs/log wording, and use PR #20877 as prior art while fixing its retention-count issue and preserving compatibility with existing adjacent backups.
