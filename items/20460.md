## Summary

Keep open. Current main still stores config backups beside the config file as `openclaw.json.bak`, `.bak.1`, etc., and the tests/docs still describe or assert that adjacent numbered behavior. The related PR #20877 directly addressed this request but was closed unmerged, so it is useful prior art rather than a shipped fix.

## Best Possible Solution

Continue tracking this as an open core config enhancement. The likely fix belongs in `src/config/backup-rotation.ts` plus the config write, mutation, audit, and recovery paths, with tests updated to assert `config-backup/openclaw.json.bak.<datetime>` behavior and docs updated away from adjacent `.bak` wording. Preserve or explicitly migrate legacy adjacent backups for recovery compatibility.
