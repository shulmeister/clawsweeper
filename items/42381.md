## Summary

Keep open. Current main still does not implement the PR's remote node-host OAuth/login guard: `modelsAuthLoginCommand` proceeds from the TTY check into provider resolution and `runProviderAuthMethod`, and the provider result is still persisted to the resolved local agent auth store. The provided PR discussion and linked #42291 describe a real remaining bug path, and the current tests cover normal local OAuth writes but have no remote node-host blocking regression.

## Best Possible Solution

Keep the PR or linked bug open and land a rebased current-main fix that detects a tokened node-host config pointing at a non-loopback gateway before local auth writes. Prefer the existing `src/gateway/net.ts` `isLoopbackHost()` helper, add a focused regression test for a remote gateway such as `10.30.10.20`, and decide whether the guard should apply only to OAuth login or all `models auth` write commands that would not update the gateway.
