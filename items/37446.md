## Summary

Keep #37446 open. Current main still does not provide a stable caller-supplied or deterministic recovery identity for `sessions_spawn`; native and ACP sub-agent paths mint fresh child session keys and gateway idempotency keys on each spawn attempt. The linked ClawHub skill is useful operational guidance, but it cannot make core `sessions_spawn` retries converge on an existing child run after a lost acknowledgement.

## Best Possible Solution

Keep the issue open until OpenClaw core adds an additive idempotent recovery contract for sub-agent orchestration. A strong fix would add a caller-supplied or deterministic `sessions_spawn` recovery key, persist the mapping from that key to the child session/run, make native and ACP spawn paths return or reuse the existing child run after timeout/lost acknowledgement, and add regression coverage for both paths. Keep recommending the linked ClawHub skill as operational guidance, but not as a substitute for the missing core contract.
