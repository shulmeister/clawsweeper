## Summary

Keep open. Current main still has the exact parser gap reported in #60380: Feishu interactive card content parsing only reads flat `markdown` and `div` elements, so the reported post-format fallback payload with nested arrays and `tag: "text"` would still collapse to `[Interactive Card]`. The linked PR #60383 is useful context, but it is still open/unmerged in the provided GitHub context, so the bug remains in main.

## Evidence

- **Current parser lacks fallback handling:** `parseInteractiveCardContent` only accepts top-level `elements` or `body.elements`, returns `[Interactive Card]` if neither exists, and only extracts flat `div.text.content` or `markdown.content`. It does not extract top-level `title`, `header.title`, nested array children, `tag: "text"`, `tag: "a"`, or `plain_text`.
