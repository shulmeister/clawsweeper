## Summary

Keep open. Current `main` still has the same structural bug the issue describes: `openclaw wiki ...` uses primary-command-scoped plugin CLI loading, that scope resolves to `memory-wiki` rather than `memory-core`, and the wiki bridge/status paths still read public artifacts only from the in-process memory capability singleton. That singleton returns `[]` when no memory capability was registered, and non-activating CLI snapshot loads explicitly restore the previous memory state instead of initializing it. The relevant files are unchanged between `v2026.4.23` and current `main`, so there is no evidence this was fixed on main or in the latest release. The issue discussion is consistent with that reading, and the referenced follow-up PR still exists on the remote as PR #69952.
