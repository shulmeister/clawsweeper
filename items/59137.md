## Summary

Keep open. Current `main` still lacks the retry-state restoration, pending full-session retry tracking, and cache-mirroring behavior described in PR #59137, and the regression tests on `main` still only cover basic temp-DB swap semantics.

## Evidence

- **No cache-table existence guard on main:** `seedEmbeddingCache()` still reads `embedding_cache` unconditionally. That means the PR's compatibility fix for older indexes without the cache table is not present on current `main`.
  - file: [extensions/memory-core/src/memory/manager-sync-ops.ts:298](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/memory-core/src/memory/manager-sync-ops.ts#L298)

  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Safe reindex still lacks retry-state snapshot/restore:** `runSafeReindex()` seeds cache into the temp DB, runs the reindex, and on failure only restores DB/vector/FTS handles. There is no `sessionFullRetryPending`, sync-state snapshot, or rollback recovery path preserving full-session retry intent.
  - file: [extensions/memory-core/src/memory/manager-sync-ops.ts:1161](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/memory-core/src/memory/manager-sync-ops.ts#L1161)

- **Unsafe/full reindex still clears session retry state:** `runUnsafeReindex()` and `resetIndex()` clear `sessionsDirtyFiles`; the only session retry signal on main is `sessionsDirty`. That matches the PR's claim that failed full session rebuilds can lose the signal needed for the next full retry.
  - file: [extensions/memory-core/src/memory/manager-sync-ops.ts:1233](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/memory-core/src/memory/manager-sync-ops.ts#L1233)

There is no separate pending-full-retry flag, so an empty dirty-file set still suppresses a needed full session retry.
  - file: [extensions/memory-core/src/memory/manager-session-reindex.ts:1](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/memory-core/src/memory/manager-session-reindex.ts#L1)

There is no second write path to the original DB, so successful batches can still be lost if a later batch fails before the temp DB swaps in.
