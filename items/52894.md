## Summary

Keep open. Current main still has an onboarding path that only recommends Homebrew and offers to display the install command; it does not auto-install Homebrew before attempting brew-backed skill dependency installs. That means the requested behavior is not implemented for the flow described by the issue.

## Evidence

- **Onboarding prompt is still manual-only:** `setupSkills` detects missing `brew`, shows a `Homebrew recommended` note, asks `Show Homebrew install command?`, and if confirmed only prints the shell command. There is no `brew` install call in this branch.
  - file: [src/commands/onboard-skills.ts:111](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/commands/onboard-skills.ts#L111)

  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Selected skills proceed to dependency install without a Homebrew bootstrap step:** After the prompt, the code loops selected skills and calls `installSkill(...)`; there is no preceding Homebrew installation step inserted into the onboarding flow.
  - file: [src/commands/onboard-skills.ts:159](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/commands/onboard-skills.ts#L159)
