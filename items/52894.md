## Summary

Keep open. Current main still only shows the Homebrew install command during skill onboarding, then sends brew-backed installs to `skills.install`, which returns a `brew not installed` failure when no brew executable is available. The separate top-level macOS shell installer can install Homebrew, but that does not cover the reported skill-onboarding path.

## Best Possible Solution

Keep this issue open and address the skill dependency onboarding path directly. A conservative fix would add an explicit macOS-only confirmation path in `src/commands/onboard-skills.ts` that installs Homebrew before running brew-backed `skills.install`, re-detects `brew`, and reuses the admin/failure-handling expectations from `scripts/install.sh`. If maintainers do not want onboarding to install Homebrew, change the prompt and public docs so users are not led to expect automatic installation; coordinate Docker/Linux handling with #14593 and skill feedback/fallback behavior with #57555.
