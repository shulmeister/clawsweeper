## Summary

Keep open. Current `main` still has the reported macOS installer behavior: `scripts/install.sh` runs the Homebrew setup step before Node detection, and that step can rewrite `PATH` via `brew shellenv`. The subsequent Node check resolves `node` from the current `PATH` and does not source `nvm.sh` or otherwise restore/version-manager-prefer the user's active Node. I did not find evidence on `main` that this issue is implemented or obsolete.

## Evidence

- **Homebrew setup can rewrite PATH before Node detection:** `install_homebrew()` runs before `check_node()`. On macOS it evaluates `brew shellenv`, which changes `PATH` for the running shell before any Node version check.
  - file: [scripts/install.sh:1217](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/scripts/install.sh#L1217)
  - command: `nl -ba scripts/install.sh | sed -n '1180,1248p'`
  - sha: [320d52a23e41](https://github.com/openclaw/openclaw/commit/320d52a23e41a326287826219a20f1f29f334554)
- **Node detection is plain PATH lookup:** `parse_node_version_components()` and `check_node()` use `command -v node` and `node -v`; they do not source `nvm.sh`, `fnm`, `volta`, or similar manager env before deciding whether Node is too old.
  - file: [scripts/install.sh:1240](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/scripts/install.sh#L1240)
  - command: `nl -ba scripts/install.sh | sed -n '1230,1388p'`
  - sha: [320d52a23e41](https://github.com/openclaw/openclaw/commit/320d52a23e41a326287826219a20f1f29f334554)
- **nvm is only mentioned in the failure hint, not loaded:** The only `nvm` handling in `install.sh` is post-check detection for an error message (`NVM_DIR`, `/.nvm/`, `command -v nvm`). There is no `nvm.sh` sourcing path in the installer.
  - file: [scripts/install.sh:1353](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/scripts/install.sh#L1353)

  - sha: [320d52a23e41](https://github.com/openclaw/openclaw/commit/320d52a23e41a326287826219a20f1f29f334554)
- **Main flow still performs the problematic order:** `main()` calls `install_homebrew` first, then `check_node`, then `ensure_default_node_active_shell`. That ordering preserves the path-rewrite-before-detection behavior described above.
  - file: [scripts/install.sh:2321](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/scripts/install.sh#L2321)
