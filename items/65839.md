## Summary

Keep open. Current main still contains the exact user-facing context-limit reset message that tells users to set `agents.defaults.compaction.reserveTokensFloor` to `20000 or higher`, and there is no test or docs-backed replacement guidance for model-specific context windows. The issue is recent, concrete, and not maintainer-authored/protected, so it should remain as actionable product/docs cleanup.

## Best Possible Solution

Keep this issue open and update the context-limit reset guidance in `src/auto-reply/reply/agent-runner-execution.ts` so it is not a hard-coded `20000` recommendation. The best fix is likely a shared helper that formats model/context-window-aware guidance, backed by tests for both overflow reset paths, plus aligned docs under the public compaction/config pages.
