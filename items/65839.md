## Summary

Keep open. Current main still emits the exact context-limit reset guidance challenged by the report, telling users to set `agents.defaults.compaction.reserveTokensFloor` to `20000 or higher` in both overflow reset paths. The runtime setting itself is supported, but the user-facing guidance has not been replaced with model/context-window-aware advice; related #72106 was already closed as a duplicate of this issue, and #67314 remains a separate overlapping guidance issue.

## Best Possible Solution

Keep this issue open and fix the user-facing context-limit reset guidance in `src/auto-reply/reply/agent-runner-execution.ts`. The best implementation is a shared helper for both reset paths that includes the active model/context window and practical reserve guidance instead of hard-coding `20000`, with focused tests and aligned public compaction/config docs.
