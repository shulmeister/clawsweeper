## Summary

Keep PR #72290 open. Current main does not already implement the two proposed protections: inbound dedupe still requires a stable `MessageSid`, and the eager chat user transcript update still emits a user message without an `idempotencyKey`. The PR is not merge-ready as submitted because it adds a full `chat.ts.backup` copy and lacks regression coverage, but the linked #72176 regression remains plausible and the PR targets the relevant code paths.

## Best Possible Solution

Keep #72290 open as a plausible targeted fix path for #72176, but request author revisions before merge: remove the backup file, add focused tests for missing/unstable inbound message ids and chat user transcript idempotency, and tighten or justify the content-based fallback semantics. If a maintainer prefers a different dedupe boundary, track that on #72176 rather than auto-closing this PR.
