## Summary

Keep #7661 open. Current main still lacks the requested `openclaw message channel create` CLI path and Slack `channel-create` implementation. The generic action vocabulary includes `channel-create`, and Discord has provider-specific runtime support behind its `channels` gate, but Slack discovery/dispatch/runtime/docs still expose only existing messaging, reaction, pin, member, emoji, and file actions, with no `channels:manage` setup path.

## Best Possible Solution

Keep #7661 open. Implement Slack-owned `channel-create` support in `extensions/slack` with a documented `channels:manage` setup path, discovery/action gating, a `conversations.create` runtime wrapper, dispatch/runtime tests, and `docs/channels/slack.md` updates. Add an `openclaw message channel create` CLI mapping through the existing message-action runner with provider-appropriate flags for Slack and Discord.
