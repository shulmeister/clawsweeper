## Summary

Keep #7661 open. Current main already has the generic `channel-create` action name and Discord-side runtime support, but the Slack-centered request is still unresolved: Slack does not advertise, dispatch, or implement channel creation, the Slack docs/scopes do not document channel-management creation support, and `openclaw message channel` still only registers `info` and `list`. The provided issue discussion also already kept this open for the same gap.

## Best Possible Solution

Keep #7661 open. The best path is to implement Slack-owned `channel-create` in the Slack plugin with a scoped action gate, a Slack `conversations.create` runtime wrapper, optional topic handling if product wants to honor the proposed flag, config/schema/docs/scope updates, tests for discovery and dispatch, and a `openclaw message channel create` CLI mapping through the existing message-action runner. Keep the existing Discord support as partial prior work, not as a reason to close the Slack-centered issue.
