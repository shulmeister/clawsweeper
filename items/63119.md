## Summary

Keep open. Current main has reset/deleted transcript archiving and a low-level `archivePath` option in the compaction truncation helper, but the actual compaction flow never passes an archive path and there is no compaction-archive retention config/schema/docs. The related PR #63161 was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open and implement the missing compaction-specific archival path: add a documented config field under `agents.defaults.compaction` or `session.maintenance`, generate a deterministic `.compaction.<timestamp>` archive path before `truncateSessionAfterCompaction`, wire retention cleanup through the existing session archive maintenance helpers, and add integration coverage proving compaction creates and prunes those archives. Reuse the already-shipped reset/deleted archive primitives where possible.
