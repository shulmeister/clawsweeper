## Summary

Keep open. Current main still emits the open plugin allowlist warning for auto-discovered workspace/global plugins when `plugins.allow` is empty, and the only maintainer comment explicitly keeps the issue open as a broader UX/policy problem. Some narrower false positives are improved, but the central first-run/actionability concern remains valid.

## Best Possible Solution

Keep #68780 open and make a narrow core UX/docs fix in the plugin loader: keep the security warning, but distinguish first-run/no explicit plugin config from mismatched allowlists, include the exact plugin IDs and a copyable `plugins.allow` snippet, and point to `openclaw plugins list --enabled --verbose` / `openclaw plugins inspect <id>`. Add/update loader warning tests for the new wording and first-run/configured cases.
