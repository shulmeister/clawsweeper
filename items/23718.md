## Summary

Keep #23718 open. Current main still does not implement `heartbeat.model: { primary, fallbacks }`: config validation, public types, generated schema, docs, and runtime options remain string-only for heartbeat-specific model selection. Existing model fallback machinery and per-agent model fallback plumbing exist, but heartbeat does not expose or forward its own fallback chain. Related PR #40875 is open and docs-only, so it does not supersede the runtime feature request.

## Best Possible Solution

Keep the issue open and implement heartbeat-specific fallback-chain support in core: change heartbeat schema and public config types to accept `AgentModelConfig`, regenerate config schema/help, update the public heartbeat docs, resolve heartbeat primary/fallbacks with the existing model-input helpers, pass a heartbeat fallback override into the reply/run path and `runWithModelFallback` while preserving explicit empty-array semantics, and add focused config plus heartbeat runtime tests. PR #40875 may remain useful as docs context but should not be treated as the implementation.
