## Summary

Keep open. Current main has a partial #66019 fix that clears unresolved cron next runs to stop the immediate refire loop, but it does not implement this PR's central schedule-error accounting behavior or the same-pass duplicate-count protections discussed in the PR review. The PR discussion shows the author addressed those review threads through timer, startup, maintenance, and manual-run paths, and current main still lacks those surfaces.

## Best Possible Solution

Keep the PR open, but ask for a tight rebase onto current main. The best path is to preserve the already-shipped no-refire behavior while porting the narrow missing pieces from the PR: promote cron `undefined` next-run computations to `recordScheduleComputeError`, keep successful recovery clearing stale counts, suppress only duplicate schedule-error recording within the same timer/startup/manual pass, retain the cron refire floor for recovered recomputes, and keep regression coverage for success, error-backoff, maintenance, startup, and manual `cron run` paths. Remove unrelated update test fixture alignment unless it is still required by the rebased merge ref.
