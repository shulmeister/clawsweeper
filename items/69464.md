## Summary

Keep open. Current main still lacks the PR's core behavior: provider model config does not accept `params`, runtime model resolution does not preserve or merge configured params, and both Ollama native and OpenAI-compatible `num_ctx` paths still derive from `contextWindow` rather than `params.num_ctx`. The PR context is focused and not superseded; related #44678 is closed unmerged.

## Best Possible Solution

Keep #69464 open for maintainer review. The best path is to land a current-main version of this fix: accept provider model `params`, thread params from `agents.defaults.models` and `models.providers.<id>.models[]` into the runtime model with explicit config precedence, make native and OpenAI-compatible Ollama use a validated `params.num_ctx` override, and keep the added docs plus regression tests. Broader Ollama option forwarding from #48010 can remain a follow-up.
