## Summary

Keep PR #71482 open. Current main still has the data-loss-prone shape described by the PR: `loadSessionStore()` can collapse failed/invalid reads to `{}`, and `scrubDreamingNarrativeArtifacts()` then treats an empty store as authoritative while archiving old dreaming transcript files. The PR should not be closed as implemented or superseded, but the existing patch likely needs revision because the proposed unconditional empty-store guard disables legitimate orphan cleanup for valid empty stores.

## Best Possible Solution

Keep the PR open and ask for a tighter fix: make the dreaming cleanup use a trusted/strict session-store read path that can distinguish read/parse failure from a successfully parsed empty `{}` store, skip destructive archival only when the store load is untrusted, and add regression coverage for both the failed-load data-loss path and the legitimate-empty-store cleanup path.

## Evidence

There is no guard on current main for an empty store object before orphan archival.
