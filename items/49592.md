## Summary

Current `main` already supports Telegram ACP `mode:"session"` spawns via current-conversation bindings, and `sessions_spawn({ runtime:"acp", ... })` routes through that path. The old child-thread-only failure no longer matches the code or tests on `main`, and the shipped `v2026.4.22` release already contains this support.

## Evidence

- **`sessions_spawn` uses the ACP spawn path:** The `sessions_spawn` tool hands `runtime:"acp"` requests directly to `spawnAcpDirect`, so the ACP Telegram behavior is governed by the current ACP spawn implementation.
  - file: [src/agents/tools/sessions-spawn-tool.ts:255](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/agents/tools/sessions-spawn-tool.ts#L255)

- **ACP spawn now honors channel-defined placement:** `spawnAcpDirect` resolves the channel's default binding placement and only errors if that placement is unsupported. The old Telegram-specific child-thread assumption is gone; Telegram can use `current` placement.
  - file: [src/agents/acp-spawn.ts:610](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/agents/acp-spawn.ts#L610)

## Risks / Open Questions

- This behavior still depends on `channels.telegram.threadBindings.spawnAcpSessions=true`; a disabled flag on an older/local config would still fail.
- The docs emphasize Telegram topics/DM-topic support more clearly than plain DM current-conversation binding, even though the direct-chat regression test exists.

## Close Comment

Closing as implemented on current `main` after a Codex review. `sessions_spawn({ runtime: "acp" })` goes through `spawnAcpDirect` (`src/agents/tools/sessions-spawn-tool.ts:255`), and the ACP spawn path now uses the channel’s declared binding placement instead of assuming Telegram needs child-thread creation (`src/agents/acp-spawn.ts:610`). Telegram now advertises current-conversation bindings with `defaultTopLevelPlacement: "current"` (`extensions/telegram/src/channel.ts:620`), and there are explicit regression tests for both Telegram topic binding and Telegram direct-chat binding for ACP session mode (`src/agents/acp-spawn.test.ts:2308`, `src/agents/acp-spawn.test.ts:2347`). The docs also list Telegram as a supported ACP thread-binding channel (`docs/tools/acp-agents.md:151`). This support is present in the shipped `v2026.4.22` release. If you still hit the old error on `v2026.4.22` or newer, please open a fresh issue with the exact Telegram chat type and whether `channels.telegram.threadBindings.spawnAcpSessions=true` is enabled.
