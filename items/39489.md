Keep open. Current main still does not wire Feishu runtime connection/activity signals into the gateway health status path, and no canonical related PR has superseded this work. The PR remains useful, but it needs revision for the current `lastTransportActivityAt` health contract and focused Feishu lifecycle/status tests before it would be merge-ready.

Keep the PR open and ask for a rebase/revision that threads `ctx.setStatus` through Feishu startup, monitor, account handlers, and WebSocket/webhook transport lifecycle; emits `connected`, `mode`, `lastEventAt`, and `lastTransportActivityAt` according to the current gateway health contract; preserves the webhook guard-ordering fix from the PR discussion; and adds focused Feishu tests for WebSocket/webhook lifecycle and activity reporting. Maintainers should also pick a canonical branch among the overlapping Feishu health PRs.
