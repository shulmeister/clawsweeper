## Summary

Keep open. Current main still has no admin-configured `docker.env` bypass for sandbox env sanitization, so the PR is not implemented or obsolete. The provided PR discussion is useful: the author’s later commit appears to address the earlier `/root`, strict-mode, and no-op validation concerns, but the PR still needs changes because the skill-filter change can bypass `enabled: false` safeguards.

## Best Possible Solution

Keep the PR open for author or maintainer follow-up. The best path is to narrow and harden it: add `forceAllowKeys` for admin-configured `sandbox.docker.env` at the Docker create path, add regression tests showing configured credential-shaped keys pass while non-configured/process env credentials remain blocked, and either remove the skill eligibility bypass or implement it so explicit `enabled: false` and bundled-skill policy still win. Do not change sandbox host-path denylist entries.
