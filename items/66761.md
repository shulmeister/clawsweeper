## Summary

Keep this PR open. Current main still has the unguarded `provider?.models?.some(...)` call that #66744 reports, the current unit test only covers array-shaped provider models, and the PR is a narrow fix with focused regression tests for the malformed runtime shapes described in the issue discussion.

## Best Possible Solution

Keep the PR open for maintainer review and either merge its `Array.isArray(provider?.models)` guard with the regression tests, or apply the same fix directly on main. The remaining broader provider-shape cleanup should be tracked separately if maintainers want stricter plugin output normalization.
