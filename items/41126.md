## Summary

Current `main` already installs an `EnvHttpProxyAgent` globally when proxy env vars are present and does so early in CLI startup and embedded-run/OAuth paths, which addresses the reported OpenRouter region/proxy regression. The shipped release commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5` contains this implementation, so the issue can be closed as fixed on main.
