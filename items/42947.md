## Summary

Current `main` already fixes the `/status` zero-context display by backfilling prompt/token usage from the session transcript when the session store snapshot is sparse or zero. The changelog says this shipped in `v2026.4.5`.

## Evidence

- **Shipped fix noted in changelog:** `CHANGELOG.md` includes a `2026.4.5` entry: `/status` and `session_status` now fall back to transcript token totals when the session meta store stayed at zero, so compatible providers stop showing `Context: 0/...`.
