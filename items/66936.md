## Summary

Keep open. Current main fixes the `agents list` unresolved-SecretRef crash and several plugin-preload hang surfaces, but the whole issue is not fully implemented: `agents add` still inherits broad `agents` plugin preloading, non-JSON `agents list` still loads plugins for provider enrichment, and CLI teardown is still memory-runtime-only rather than a generic external-plugin handle cleanup. The related closed reviews (#68632, #66227) and current changelog cover important parts, but they do not prove the reported `native-tools-bridge`-style hang is obsolete for every reported `agents` subcommand.

## Best Possible Solution

Keep the issue open but narrow it to the remaining CLI exit-hang surface. The best fix is either to keep the remaining config-only `agents` paths, especially non-interactive `agents add` and non-JSON `agents list` where possible, off broad runtime plugin loading, or to add a real plugin-runtime teardown contract that closes external plugin handles after short-lived CLI commands. The fixed SecretRef crash and already-covered bind/delete/list-json pieces can be referenced as resolved by #68632, #66227, #71739, and #71743.
