## Summary

Current main does not appear to implement the requested Copilot Premium/Chat quota in the per-response /usage footer. The data exists and /status can display it, but the response footer path still formats only token/cache/cost/session details. The related PR #43098 attempted this exact change but was closed unmerged, so it does not supersede the issue.

## Evidence

There is no provider-usage/quota parameter or Premium/Chat formatting in this footer formatter.
