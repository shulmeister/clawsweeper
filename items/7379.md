## Summary

Keep open. Current main still does not add GitHub Copilot Premium/Chat quota to the per-response `/usage` footer; the quota data exists and `/status` displays provider usage, but the reply footer path only formats token/cache/cost/session details. Prior PR #43098 targeted this issue but was closed unmerged, so it is useful prior work rather than a shipped fix.

## Best Possible Solution

Keep #7379 open until the response footer path intentionally loads or reuses cached provider usage for GitHub Copilot, appends Premium/Chat quota without breaking `/usage tokens` versus `/usage full` semantics, and adds a focused regression test. PR #43098 is the best prior implementation to review as a starting point.
