## Summary

Keep #7379 open. Current main still keeps provider quota usage on the `/status` path while the per-response `/usage` footer only formats turn token/cache/cost data and optional session key. GitHub Copilot Premium/Chat quota data is available through the provider-usage hook, but it is not wired into the reply footer. The related PR #43098 attempted the right feature but is closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep #7379 open. The best implementation is a focused reply-footer change that reuses the active provider's provider-usage summary for GitHub Copilot OAuth sessions, appends formatted Premium/Chat quota only when available, degrades silently on auth/fetch errors, and adds a regression test in `src/auto-reply/reply/agent-runner.misc.runreplyagent.test.ts`. PR #43098 can be treated as prior art, but it needs current-main review because it was closed unmerged.
