## Summary

Keep open. Current main still does not append GitHub Copilot Premium/Chat quota to the per-response `/usage` footer. The data exists and `/status` renders provider usage windows, but the reply footer path still formats only token/cache/cost/session details. Prior PR #43098 attempted this exact change but was closed unmerged, so it remains useful prior work rather than a shipped fix.

## Best Possible Solution

Keep the issue open until the response footer path intentionally loads or reuses cached provider usage for GitHub Copilot, appends Premium/Chat quota without breaking `/usage tokens` versus `/usage full` semantics, and adds a focused regression test. PR #43098 is the best prior implementation to review as a starting point.
