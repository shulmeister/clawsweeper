## Summary

Keep #38939 open. Current main still lacks Feishu group mention arbitration across sibling bot accounts, and the related #39853 work is open and narrower rather than a shipped replacement. The PR remains relevant, but should be revised before merge because the existing review comments identify real correctness issues.

## Best Possible Solution

Keep the PR open or replace it with a corrected Feishu plugin implementation. The maintainer path should perform dispatch-time arbitration from structured mentioned bot open_ids, derive the coordinator from Feishu default-account selection, cover specialist-only, coordinator-plus-specialist, and multi-specialist group mentions, avoid substring-only implicit mention matching, and add regression tests for those cases.
