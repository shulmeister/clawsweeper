## Summary

Current main still does not implement the PR's Feishu inbound mention arbitration. Feishu message delivery is handled per account, current code only checks whether the current bot was mentioned, and open group policy can still accept an event without a mention, so the duplicate-reply scenario remains actionable. The PR also has unresolved review concerns, so it should stay open rather than be auto-closed.

## Best Possible Solution

Keep the PR open and either update it or replace it with a maintainer-ready Feishu plugin change. The fix should arbitrate in the inbound `im.message.receive_v1` path before `handleFeishuMessage()`, extract mentioned bot open_ids from both top-level mentions and post content, derive the coordinator from Feishu default-account resolution instead of hardcoding `main`, and add regression tests for single-specialist, default+specialist, multiple-specialist, post-message mentions, and non-`main` default-account deployments.
