## Summary

Keep #38939 open. Current main still does not implement Feishu group mention arbitration across sibling bot accounts, and the PR remains a focused attempt to fix a real duplicate-reply/drop-risk gap. The PR should not be closed as implemented, duplicate, or stale cleanup; it needs a corrected implementation before merge.

## Best Possible Solution

Keep the PR open or replace it with a corrected Feishu implementation. The maintainer path should derive the coordinator from Feishu default-account selection, arbitrate on structured mentioned bot open_ids before per-account dispatch, avoid substring-only implicit mention matching, and add regression coverage for specialist-only, coordinator-plus-specialist, and multi-specialist group mentions.
