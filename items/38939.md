## Summary

Keep PR #38939 open. Current main at e672b614 still lacks Feishu dispatch-time arbitration across sibling bot accounts: all enabled accounts are started, receive handling is account-scoped, mention parsing only marks whether the current bot was mentioned, and group gating only checks that current-account flag. Related PRs #39853 and #39001 do not supersede this duplicate-reply fix. The PR also has review-noted issues, so it should stay open or be replaced by a corrected implementation rather than closed.

## Best Possible Solution

Keep the PR open or replace it with a corrected Feishu plugin implementation. The maintainer path should arbitrate before dispatch using structured mentioned bot open_ids, derive the coordinator from Feishu default-account selection rather than a literal main id, handle specialist-only, coordinator-plus-specialist, and multi-specialist group mentions, avoid substring-only implicit mention matching, and add regression tests for those cases.
