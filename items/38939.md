## Summary

Keep PR #38939 open. Current main still lacks Feishu dispatch-time arbitration across sibling bot accounts: every enabled Feishu account is started, receive handling is account-scoped, message dedupe is namespaced by account, and group gating only checks whether the current account's bot was mentioned. The related open PRs #39853 and #39001 do not supersede this because they address inbound metadata exposure and merge-forward formatting, not single-dispatch arbitration across bot accounts. The PR should not be merged unchanged because review comments identify hardcoded coordinator and substring implicit-mention risks, but the underlying bug remains plausible on current main.

## Best Possible Solution

Keep the PR open or replace it with a corrected Feishu plugin implementation. The maintainer path should arbitrate before ordinary dispatch using structured mentioned bot open_ids from text and post events, derive the coordinator from Feishu default-account selection rather than a literal main id, handle specialist-only, coordinator-plus-specialist, and multi-specialist group mentions, avoid substring-only implicit mention matching, and add focused multi-account regression tests for the receive handler and normal single-agent dispatch path.
