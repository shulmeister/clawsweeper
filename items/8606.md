## Summary

Keep open. Current main still does not emit a `session:prune` or `context:prune` hook when context pruning trims tool output, and the directly related implementation PR #12581 remains open rather than shipped. The most recent discussion also already reached the same keep-open conclusion after reviewing the pruning path and hook surfaces.

## Best Possible Solution

Keep #8606 open until PR #12581 or a replacement maintainer-approved implementation lands. The likely implementation belongs in the context-pruning path under `src/agents/pi-hooks/context-pruning/*`, with tests for hook timing and payload contents, plus public docs updates for automation/plugin hooks if the event is exposed.
