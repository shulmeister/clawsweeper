## Summary

Keep open. Current main still prunes context but does not expose a `session:prune` or `context:prune` lifecycle hook through either internal hooks or plugin hooks. The directly related implementation PR #12581 is still open in the provided GitHub context, while the earlier linked PR #9761 was closed unmerged. The existing owner-authored Codex review also kept this issue open, and current code still supports that conclusion.

## Best Possible Solution

Keep #8606 open until PR #12581 or a replacement maintainer-approved implementation lands. The likely path is an additive pruning lifecycle event around `src/agents/pi-hooks/context-pruning/*`, with tests proving when it fires and what pruned metadata/content is exposed, plus docs updates for the public hook surface at `docs/automation/hooks.md` and any plugin hook docs if the event is exposed there too.
