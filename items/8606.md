## Summary

Keep open. Current main still performs context pruning but does not emit a `session:prune` or `context:prune` hook through internal hooks or plugin hooks. The earlier linked PR #9761 was closed unmerged, and the directly related implementation PR #12581 is still open in the provided GitHub context.

## Best Possible Solution

Keep #8606 open until PR #12581 or a replacement maintainer-approved implementation lands. The likely implementation belongs in the context-pruning path under `src/agents/pi-hooks/context-pruning/*`, with tests proving when the prune event fires and what metadata/content it exposes, plus docs updates for `https://docs.openclaw.ai/automation/hooks` and `https://docs.openclaw.ai/plugins/hooks` if the event is exposed to plugin authors.
