## Summary

Keep #39142 open. Current main still has no configurable `messages.modelExhausted` or equivalent user-facing exhaustion message. The runtime has fixed built-in friendly copy for pure billing and pure transient rate-limit fallback exhaustion, but it does not read config for custom text, and mixed billing/rate-limit exhaustion remains intentionally surfaced as the raw `All models failed` failure in regression coverage.

## Best Possible Solution

Keep the issue open until core either adds a supported configurable exhaustion-message surface, such as `messages.modelExhausted`, or a maintainer explicitly chooses a different product direction. A complete fix should update `MessagesConfig`, runtime Zod/schema/help metadata, docs, and `runAgentTurnWithFallback` so configured copy is used for the intended all-models-exhausted cases, with tests covering pure billing, pure rate-limit, and the mixed billing/rate-limit policy decision.
