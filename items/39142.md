## Summary

Keep #39142 open. Current main no longer blindly exposes the raw `All models failed` summary in the checked fallback branches, but it still does not provide a supported configurable `messages.modelExhausted` or equivalent user-facing exhaustion-message setting.

## Best Possible Solution

Keep the issue open until core either adds a supported configurable exhaustion-message surface or a maintainer explicitly chooses a different product direction. A complete implementation should add a field to `MessagesConfig`, `MessagesSchema`, generated config help/labels/reference docs, and the fallback failure branch in `runAgentTurnWithFallback`, with focused tests for pure billing exhaustion, pure transient rate-limit exhaustion, and the intended mixed billing/rate-limit behavior.
