## Summary

Keep #39142 open. Current main has sanitized, hard-coded billing/rate-limit fallback replies, but it still does not expose `messages.modelExhausted` or an equivalent supported config surface for customizing the user-facing all-models-exhausted message.

## Best Possible Solution

Keep the issue open until core adds a supported configurable model-exhaustion reply surface or maintainers explicitly choose a different product direction. A complete fix likely belongs in `MessagesConfig`, `MessagesSchema`, generated schema help/labels/docs, public config docs, and the fallback failure branch in `runAgentTurnWithFallback`, with focused tests for pure billing exhaustion, pure rate-limit exhaustion, and mixed billing/rate-limit behavior while preserving sanitized defaults.
