## Summary

Keep open. Current main still discovers/selects all succeeded Microsoft Foundry deployments, but the microsoft-foundry provider only models OpenAI-compatible Foundry APIs and still builds `/openai/v1` base URLs. There is no built-in Anthropic Messages routing or guardrail for Claude deployments yet.

## Evidence

- **Foundry API union excludes Anthropic Messages:** `FoundryProviderApi` is limited to `openai-completions` and `openai-responses`; `isFoundryProviderApi` only accepts those two values.
  - file: [extensions/microsoft-foundry/shared.ts:74](https://github.com/openclaw/openclaw/blob/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54/extensions/microsoft-foundry/shared.ts#L74)
