## Summary

Current `main` shows that WeChat image send support is owned by the external `@tencent-weixin/openclaw-weixin` channel plugin, not by OpenClaw core, so this is not actionable in `openclaw/openclaw`.

## Evidence

  - file: [docs/channels/wechat.md:10](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/channels/wechat.md#L10)

## Close Comment

Closing after a Codex review of current `main`. The WeChat integration in this repo is intentionally plugin-scoped, not core-scoped: `docs/channels/wechat.md:10` and `docs/channels/wechat.md:27` route WeChat to the external `@tencent-weixin/openclaw-weixin` package, and `docs/channels/wechat.md:41` says media upload/download is owned by that external plugin. `VISION.md:54` also states that optional capability should usually ship as plugins maintained outside core. I did not find a WeChat runtime implementation in this checkout, only docs and generic plugin-registry references. Because this request is about WeChat image sending behavior, it belongs with the external WeChat plugin rather than `openclaw/openclaw` core. If there is a specific OpenClaw core contract break in `src/channels/*` or `src/plugins/*`, please open a new core issue with a concrete reproduction and the plugin version involved.
