## Summary

Keep #50998 open. Current main still has the underlying iOS permission gap: contacts and calendar node commands are routed/advertised, but first-use Contacts/Calendar authorization is not requested and the app bundle lacks the Contacts/Calendar usage-description keys. The related #51011 is an open follow-up settings UX PR, not a merged replacement for this invoke-path bug fix.

## Best Possible Solution

Keep #50998 open for maintainer review. The best path is to land a scoped iOS fix in ContactsService, CalendarService/EventKit authorization helpers, Info.plist, and project.yml that restores explicit first-use permission flow without regressing headless invoke cancellation behavior, with focused tests for the authorization helper/continuation logic plus device or simulator verification.
