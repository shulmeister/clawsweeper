## Summary

Keep open. Current `main` still does not thread upstream provider request IDs through `FailoverError`, `model.usage` diagnostics, queue error logs, or the bundled OTEL plugin, and the issue already has maintainer-backed follow-up discussion plus an open linked PR for the additive core change.

## Risks / Open Questions

- The needed change crosses core diagnostics types, failover normalization, queue logging, and the bundled `diagnostics-otel` plugin, so a partial fix could leave some failure paths without the correlation id.
- Current lower-layer error formatting already sometimes embeds provider request ids in message strings; any future implementation should avoid duplicate reporting and keep additive compatibility for plugin listeners.
