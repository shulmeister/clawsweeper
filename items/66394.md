## Summary

Current main does not implement the requested production log rotation. The logger writes a dated file and suppresses additional writes once maxFileBytes is exceeded; docs still tell users to use external log rotation. The linked PR #66457 is useful related work, but it is open/unmerged in the provided GitHub context, so the original feature request still tracks real remaining work.
