## Summary

Keep #7648 open. Current main still implements bottom-follow chat scrolling in Control UI/WebChat, with only a downward “New messages” affordance and no UI, persisted setting, gateway config, top anchoring mode, or Scroll to Top control. The prior Codex/maintainer review comment also kept this open, and the related #44130 is TUI-specific rather than a replacement for this issue.

## Best Possible Solution

Keep #7648 open and implement an explicit Control UI/WebChat chat-scroll preference. A conservative fix would add a persisted UI setting or documented gateway.webchat/controlUi config, thread it into scheduleChatScroll, support bottom-follow plus off/top behavior, optionally add a Scroll to Top control, and cover send, streaming, refresh, and manual-scroll states with focused UI tests.
