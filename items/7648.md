## Summary

Keep #7648 open. Current main still does not provide a Control UI/WebChat preference to disable chat auto-follow or a Scroll to Top action. Browser Control UI continues to force/follow bottom on send, initial/tab load, refresh, and streaming when pinned; native SwiftUI WebChat also pins to bottom on load/send/user messages/streaming. Settings, gateway config, docs, and tests show only existing bottom-follow behavior and no scroll-mode option. The related #44130 remains a separate TUI issue, not a browser/native WebChat duplicate.

## Best Possible Solution

Keep #7648 open. A conservative fix should add an explicit Control UI/WebChat chat scroll preference, likely persisted in UiSettings and optionally backed by documented gateway.controlUi or gateway.webchat config if maintainers want config-level control. Thread bottom/off/top behavior through scheduleChatScroll, send/refresh/render lifecycle paths, and OpenClawChatView; expose a clear UI affordance such as a Scroll to Top action when appropriate; add focused tests for send, streaming, initial load, manual scroll, refresh, and the visible action/preference. Keep TUI-specific behavior tracked separately in #44130.
