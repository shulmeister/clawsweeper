## Summary

Keep #7648 open. Current main still implements Control UI/WebChat bottom-follow behavior with only a downward New messages affordance. There is no UI toggle, persisted browser setting, gateway.controlUi/gateway.webchat config, top anchoring mode, or Scroll to Top control that satisfies the requested feature.

## Best Possible Solution

Keep #7648 open and implement an explicit Control UI/WebChat chat-scroll preference. A conservative path would add a persisted UI setting or documented gateway.webchat/controlUi config, thread it through scheduleChatScroll, support bottom-follow plus off/top behavior, optionally add a Scroll to Top control, and cover send, streaming, refresh, and manual-scroll states with focused UI tests.
