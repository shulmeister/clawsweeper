## Summary

Keep open. Current main has partial auto-follow behavior that respects a user who has already scrolled up, and it exposes a “New messages” jump-to-bottom affordance, but the requested Control UI/WebChat capability is still missing: there is no config or UI toggle to disable auto-scroll or keep new responses anchored at the top, and the send path still explicitly resets scroll state and forces the thread to the bottom for the incoming response.

## Evidence

- **Send path forces bottom auto-scroll:** When sending a chat message, current main resets chat scroll state and calls scheduleChatScroll(..., true), with comments stating this ensures auto-scroll works for the response and forces the viewport to the bottom for the incoming stream.
