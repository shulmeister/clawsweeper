## Summary

Keep #7648 open. Current main still implements bottom-follow chat scrolling and a bottom-oriented “New messages” affordance, but there is no Control UI/WebChat setting to disable auto-scroll, no top anchoring mode, and no Scroll to Top control. The prior issue comment already reached the same keep-open conclusion, and current main still matches it.

## Best Possible Solution

Keep this issue open and implement the behavior in the Control UI/WebChat chat surface. A conservative path is to add an explicit chat-scroll preference, thread it through render state into scheduleChatScroll, support bottom-follow plus off/top behavior, optionally add a Scroll to Top affordance, and add focused tests for send, streaming, refresh, and manual scroll states.
