## Summary

Keep #42797 open. Current main still does not implement the PR's gateway-side auth/usable-provider narrowing: `models.list` returns the configured allowlist or the full catalog, and active UI/TUI picker paths still consume that RPC. The related #59693 work is broader, still open, and not a clear complete replacement for this narrower PR, while the review comments on #42797 identify real fix-before-merge issues rather than an auto-close reason.

## Best Possible Solution

Keep the PR open for maintainer review or fold it into a clearly chosen canonical successor. The best implementation is a shared gateway/model-picker filtering path that preserves the full-catalog fallback for unconfigured installs, matches the source-of-truth provider auth checks, covers Bedrock AWS SDK and synthetic/local provider auth, and adds regression tests for gateway `models.list` plus the UI/TUI picker consumers.
