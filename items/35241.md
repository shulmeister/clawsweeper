## Summary

Keep #35241 open. Current main still does not expose `--session-key` on `openclaw agent`, while the embedded agent command, session resolver, and Gateway protocol/server already understand `sessionKey`. The PR addresses a real CLI surface gap with downstream demand, but it should be refreshed and should address validation parity before merge.

## Best Possible Solution

Keep the PR open. The best path is a small core CLI change that registers `openclaw agent --session-key <key>`, adds `sessionKey` to `AgentCliOpts`, treats it as a valid selector, passes it through Gateway and embedded fallback paths, mirrors malformed-key and `--agent` conflict validation consistently, and updates CLI help/docs plus focused tests. Handle the `--no-fallback` idea as a separate follow-up.
