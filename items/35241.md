## Summary

Keep open. Current main still does not expose `--session-key` on `openclaw agent`, even though the lower-level resolver, embedded command, Gateway schema, and Gateway server already support explicit session keys. The PR addresses a real CLI parity gap, and the discussion includes downstream demand plus review feedback that should be addressed before merge.

## Best Possible Solution

Continue tracking this PR or an equivalent maintainer-authored change. The best implementation is to add `--session-key <key>` to `openclaw agent`, include it in `AgentCliOpts`, pass it through `resolveSessionKeyForRequest()` and embedded fallback, add consistent `--agent`/`--session-key` validation before dispatch or in both paths, and update CLI docs/help plus focused tests.
