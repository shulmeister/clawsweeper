## Summary

Keep #71537 open. Current main partially handles reset archives when an active transcript path exists but is empty, but it still lacks the PR's archived-only previous-session discovery and the session-logs skill guidance for reset/deleted archive searches. The related #60409 is complementary gateway RPC work, not a replacement for this PR's hook and skill surfaces.

## Best Possible Solution

Keep the PR open and have maintainers review the remaining scoped value: add archived-only reset transcript discovery for session-memory, keep the regression tests for direct archive/missing-live/latest-archive cases, and update session-logs documentation to include active plus reset/deleted archive searches. Coordinate with #60409 only for gateway history behavior; it should not block or replace this hook/docs work.
