## Summary

Keep #45494 open. Current main has related failover and cron retry work, but it does not implement the requested fast-fail behavior for Anthropic-style `api_error: Internal server error` outages. Main still classifies transient JSON/server API errors as `timeout`, timeout failures do not mark auth-profile cooldown, and the fallback loop still evaluates candidates sequentially rather than detecting a uniform provider outage and aborting early. The linked PR #45625 was closed unmerged, so it is not a canonical implemented fix.
