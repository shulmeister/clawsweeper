## Summary

Keep open. Current main covers parts of the request, but not the whole item: memory search already supports extra indexed paths and optional session-transcript indexing, session/status surfaces already expose usage/cost/latency/tool-count data, and gateway startup can run BOOT.md hooks. But I found no implemented user-facing conversation-chaining flow like `openclaw agent --continue <sessionId>` or automatic inherited last-session summary, and the only built-in startup-context preload is for `/new` and `/reset`, not general post-restart core-context prewarming.
