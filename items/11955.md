## Summary

Keep #11955 open. Current main implements several adjacent pieces, including memory_search over MEMORY.md/memory/*.md, configurable extra paths, opt-in session transcript indexing, usage/cost/session aggregate APIs, and first-turn startup context for bare /new and /reset. The broader requested capabilities still remain: an agent-facing self-evaluation metrics API with task/satisfaction/success signals, automatic conversation chaining or --continue inheritance, and a gateway-restart boot/preload sequence before the first user message. The provided discussion also shows #11956 was closed as a duplicate of #11955, making this the canonical tracker rather than a superseded item.

## Best Possible Solution

Keep #11955 as the canonical umbrella unless a maintainer splits it. The remaining product path is to define and implement a core metrics/self-evaluation API, a documented conversation continuation or summary-handoff mode, and an explicit gateway startup prewarm/preload mechanism, while preserving the already-shipped memory search and usage-reporting surfaces.
