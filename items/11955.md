## Summary

Keep #11955 open. Current main implements adjacent pieces: semantic memory over MEMORY.md and memory/*.md, configurable extra paths, opt-in transcript indexing, usage/cost/session aggregation, first-turn startup context for bare /new and /reset, and an opt-in BOOT.md startup hook. The umbrella still has meaningful missing core work: an agent-facing self-evaluation metrics API with task/satisfaction/success signals, automatic conversation chaining or summary handoff for new sessions, and default gateway-restart preload of core context before the first user message. The provided discussion also already identifies #11955 as the canonical tracker after duplicate #11956 was closed.

## Best Possible Solution

Keep #11955 as the canonical umbrella unless a maintainer splits it. The best product path is to preserve the already-shipped memory search, usage reporting, startupContext, and boot-md pieces while separately defining a first-class metrics/self-evaluation API, a documented conversation continuation or compact-summary handoff mode, and an explicit gateway startup preload/prewarm mechanism for core context.
