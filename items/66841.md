## Summary

Current main still has the behavior reported in #66841: `openclaw logs` calls `logs.tail`, the gateway handler reads the configured rolling file log, and the loopback pairing-required fallback reads the local configured log file. There is no current implementation of Linux/systemd journal preference, service PID/ExecStart/service-version provenance, or mismatch warnings, so this concrete bug report should remain open.

## Best Possible Solution

Keep the issue open and implement the requested Linux/systemd behavior in the `openclaw logs` CLI: before following, print explicit provenance, prefer the active gateway service journal when the local managed service and PID are available, warn on CLI/service/log-source mismatches, and retain the configured file-tail fallback only with clear disclosure and regression tests.
