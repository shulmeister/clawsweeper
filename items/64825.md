## Summary

Keep open. Current main has the later TUI streaming watchdog, but it still does not implement this issue's specific inactive-final/no-active-run recovery path: `finalizeRun()` only calls `setActivityStatus()` when `wasActiveRun` is true, and local BTW/unbound deltas can still set the footer to `streaming` without arming the watchdog or clearing it on the empty final path.

## Best Possible Solution

Keep this issue open and implement a narrow TUI state-machine fix in `src/tui/tui-event-handlers.ts`: after final/terminal cleanup, transition to the terminal status when the finishing run was active or when there is no `activeChatRunId` and no tracked session run remains. Include the local BTW empty-final branch, and add focused tests for inactive-final clearing, local BTW delta plus empty final, concurrent active-run preservation, and terminal error/aborted status preservation.
