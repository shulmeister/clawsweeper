## Summary

Keep open. Current `main` still exposes the chat transcript as a polite live region while streaming text is rendered into that same transcript, so the reported screen-reader behavior remains plausible and is not shown as fixed on `main`. Related PRs were referenced on the issue, but their intended suppression behavior is not present in the checked-out code.
