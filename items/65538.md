## Summary

Keep open. Current main still renders the Control UI chat transcript as a polite live log during streaming, so the reported behavior is still plausible. There is also an open implementation PR, #65575, that directly targets this issue and changes the same chat-thread aria-live behavior with focused tests, so this issue should stay open until that PR is reviewed and merged or rejected.

## Best Possible Solution

Keep this issue open and use #65575 as the current implementation candidate. The likely product fix belongs in `ui/src/ui/views/chat.ts`, with focused tests in `ui/src/ui/views/chat.test.ts` proving idle transcript announcements remain polite while sending/streaming updates are suppressed. If maintainers reject #65575, an equivalent main-branch fix should still address the same live-region behavior before closing this issue.
