## Summary

Keep open. Current main still does not implement configurable context-file elevation for SOUL.md, IDENTITY.md, USER.md, or MEMORY.md: the prompt builder only gives known bootstrap files deterministic ordering, then renders them as ordinary `## <path>` entries under `# Project Context`. The config types/schema/docs expose bootstrap injection, truncation, and context-engine/plugin hooks, but not the requested `context.elevation`/`contextElevation` surface or built-in elevated section mapping. The related PR #18110 was closed unmerged in the provided GitHub context, and the latest maintainer/Codex discussion already explicitly kept this issue open.

## Best Possible Solution

Continue tracking #17810 for maintainer product direction. If accepted, implement the behavior in the agent prompt/bootstrap path with a documented config/schema surface, deterministic matching/order, elevated sections before Project Context, exclusion from the flat context unless explicitly retained, and matching system-prompt tests plus docs. If maintainers prefer externalization, close only after an explicit plugin/API design or ClawHub implementation is agreed, not from the current generic hook availability alone.
