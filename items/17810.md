## Summary

Keep #17810 open. Current main still prioritizes SOUL.md/IDENTITY.md/USER.md/MEMORY.md for ordering, but renders them as normal files under `# Project Context`; there is no `context.elevation`/`contextElevation` config surface, tests and docs still assert flat Project Context behavior, and related PR #18110 is closed unmerged.

## Best Possible Solution

Continue tracking #17810 for maintainer product direction. If accepted in core, add a documented config/schema surface for deterministic context-file elevation before Project Context, preserve current behavior when unset, exclude elevated files from the flat block unless explicitly retained, and cover prompt rendering plus config/docs tests. If maintainers prefer externalization, document a plugin/hook-pack path that can both add stable prompt sections and remove duplicated bootstrap files without replacing the whole system prompt.
