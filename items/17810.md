## Summary

Keep open. Current main still does not implement configurable context-file elevation for SOUL.md/IDENTITY.md/USER.md/MEMORY.md: prompt assembly orders bootstrap files and renders them under Project Context, tests/docs assert that flat behavior, and the proposed config surface/section labels are absent. The related PR #18110 was closed unmerged in the provided GitHub context, so it does not supersede this feature request.

## Best Possible Solution

Continue tracking #17810 until maintainers choose a product direction. The likely implementation belongs in the agent prompt/bootstrap path: add a documented config/schema surface for context-file elevation, apply deterministic matching before Project Context rendering, exclude elevated files unless explicitly retained, and update system-prompt tests plus docs. If maintainers prefer hooks/plugins instead, the issue should be resolved with an explicit plugin/API design rather than auto-closed.
