## Summary

Keep #17810 open. Current main still sorts SOUL.md, IDENTITY.md, USER.md, and MEMORY.md into a deterministic order and adds a SOUL-specific hint, but it does not implement configurable elevated prompt sections or exclude elevated files from the flat Project Context block. The related implementation PR #18110 is closed unmerged, and the current config schema/docs/tests still reflect the flat Project Context behavior.

## Best Possible Solution

Continue tracking #17810 for explicit maintainer product direction. If accepted, implement a documented config/schema surface for deterministic context-file elevation before `# Project Context`, preserve current behavior when unset, exclude elevated files from the flat block unless explicitly retained, and add focused prompt-rendering plus config/docs coverage. If maintainers want this externalized, first define or document a safe public plugin/hook API for restructuring bootstrap prompt sections without requiring brittle whole-prompt replacement.
