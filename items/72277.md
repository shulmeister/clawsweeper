## Summary

Keep #72277 open. It is a fresh, detailed v2026.4.24 regression report filed after #72151 requested a new issue if the Slack async exec completion leak reproduced on that release. Current main has generic exec-event delivery-context plumbing, but I did not find source, test, or post-release history that proves the specific Slack channel-thread async exec completion path is fixed.

## Best Possible Solution

Keep the issue open and fix it with a Slack-specific regression test: start a Slack channel-thread turn, background an exec completion, assert the queued exec-event delivery context retains the original thread_ts, and assert the final Slack outbound payload includes that thread_ts instead of posting at channel root. If the context is lost before heartbeat relay, preserve the immutable originating route at exec-tool creation or event enqueue time.
