## Summary

Current `main` already fixes the reported hot-reload staleness via cache invalidation rather than eager refresh: model-related hot reloads clear the cached gateway model catalog, and the next catalog read rebuilds it and rewrites `models.json`. Current regression tests cover both the invalidation and the post-reload visibility of newly added models.

## Evidence

## Risks / Open Questions

- Current main fixes this lazily on the next catalog read, not by eagerly rebuilding during the reload handler. If someone specifically wanted immediate on-reload disk rewrite with no subsequent catalog consumer, that exact timing is different.
- The fix appears newer than the latest tagged release `v2026.4.22`, so this looks implemented on `main` but not yet in a shipped release.
