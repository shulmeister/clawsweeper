## Summary

Keep open. Current `main` does not implement this request on the `session_status` tool: the tool still returns a `/status`-style text card plus a small `details` object, with no structured `remaining_seconds`, `remaining_tokens`, `elapsed_seconds`, or session-timeout fields. There is related structured token data elsewhere (`/status` summary and `sessions_list`), but not on the requested tool surface.

## Evidence

- **`session_status` still returns text-oriented details only:** The tool definition exposes only `sessionKey` and `model` params, and its success payload returns `details.ok`, `details.sessionKey`, `details.changedModel`, and `details.statusText`. No structured budget fields are returned.
