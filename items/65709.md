## Summary

The issue is an upstream Tesla OAuth/signup research task for a prospective integration, not a current OpenClaw core bug or missing core behavior. OpenClaw’s current scope pushes optional third-party service integrations into external plugins published via ClawHub/npm, and current `main` has no Tesla-owned surface to fix.

## Evidence

- **Issue scope is upstream Tesla integration research:** The issue asks to verify Tesla developer signup availability, app registration, scopes, redirect URIs, auth constraints, and implementation gotchas. That is research about Tesla’s external platform requirements rather than a reproducible OpenClaw core defect.
- **Vision routes optional capabilities to plugins:** OpenClaw’s vision says core stays lean, optional capability should usually ship as plugins, code plugins are the path for providers/channels/tools, and plugin promotion belongs in ClawHub.
