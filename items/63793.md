## Summary

Keep #63793 open. Current main still lacks the PR's central ACP lifecycle fixes: `/acp spawn` does not probe the freshly initialized runtime before reporting success, and stale `Gateway disconnected` failures are not treated as recoverable in close, cancel, active-turn cancel, or early-turn retry paths. The related `/acp` command-routing fix is present on main, but it is a separate surface and does not supersede this PR.

## Best Possible Solution

Keep #63793 open for ACP/Gateway maintainer review. The best path is a narrow fix that adds a direct post-init spawn liveness/status check with cleanup, treats resolved dead/no-session status as failed spawn, and applies an explicit stale gateway-disconnect close-code allowlist across close, passive cancel, active-turn cancel, and early-turn retry with the focused tests described in the PR.
