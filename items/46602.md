## Summary

Keep open. Current main does show a minimal TTS line in `/status`, but it does not include the requested configured model, voice, or custom `baseUrl`/backend visibility. The directly related PR #47353 proposed this exact behavior but was closed unmerged, so it does not supersede the issue.

## Evidence

- **/status output omits requested backend fields:** `formatVoiceModeLine` currently renders only `autoMode`, `provider`, `limit`, and `summary`; there is no model, voice, display name, baseUrl, or custom backend indicator in the `/status` line.
  - file: [src/status/status-message.ts:465](https://github.com/openclaw/openclaw/blob/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071/src/status/status-message.ts#L465)
  - command: `nl -ba src/status/status-message.ts | sed -n '451,466p'`

- **TTS status snapshot lacks model/voice/baseUrl data:** The snapshot type used by `/status` only carries `autoMode`, `provider`, `maxLength`, and `summarize`, and the resolver returns only those fields. That means the status formatter cannot currently display the requested OpenAI-compatible backend details.
