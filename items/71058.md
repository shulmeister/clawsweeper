## Summary

Keep open. Current main still treats `channels.msteams` as one top-level Microsoft Teams bot configuration/default account, and the runtime resolves one credential set and one webhook listener. The issue is fresh, author is not protected, labels are not protected, and the provided discussion only shows interest in working on it, not an implementation, workaround plugin, or superseding item.
