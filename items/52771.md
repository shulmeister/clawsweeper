## Summary

Current `main` explicitly reads `~/.openclaw/.env` / `$OPENCLAW_STATE_DIR/.env`, merges those durable service env vars into gateway-install service metadata, and writes them into the LaunchAgent plist. The reporter also confirmed the original redaction bug was fixed in `v2026.3.22`.

## Close Comment

Closing from a Codex review: this specific LaunchAgent env-var redaction bug is already implemented on current `main`. OpenClaw now reads durable service vars from `~/.openclaw/.env` / `$OPENCLAW_STATE_DIR/.env` in `src/config/state-dir-dotenv.ts:45`, merges them into the gateway install environment in `src/commands/daemon-install-helpers.ts:215`, passes that environment through `src/cli/daemon-cli/install.ts:157`, and writes it into the LaunchAgent plist via `src/daemon/launchd.ts:539` and `src/daemon/launchd-plist.ts:26`. The reporter also confirmed on this issue that `v2026.3.22` includes the fix and that provider keys are now present in the plist after install. The separate plist-permissions concern noted later is a different issue, but the original credential-redaction regression described here is fixed, so I’m closing this as implemented on main.
