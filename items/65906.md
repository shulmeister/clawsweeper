## Summary

Keep PR #65906 open. Current main still lacks `{context}` / `{contextPercent}` response-prefix support, so the PR is not obsolete or implemented-on-main. The PR also still has a real review concern on its latest revision: its runtime wiring appears to prefer aggregate `lastCallUsage.total`, which can include output tokens, while current main’s usage helpers treat context usage as prompt-side tokens only.

## Best Possible Solution

Keep the PR open and ask for a focused revision: compute `{context}` / `{contextPercent}` from prompt-side context tokens using existing usage helpers, wire the callback through shared reply dispatch paths, add regression tests for provider usage shapes where `total` includes output, and document the new variables in the response-prefix docs/config help.
