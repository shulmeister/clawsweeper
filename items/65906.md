## Summary

Keep this PR open. Current main still does not support `{context}` or `{contextPercent}` in response prefix templates, and this PR is the only concrete implementation candidate in the provided context. The PR is not ready to close as obsolete or implemented because main lacks the feature; it also should not be merged as-is until the remaining review concern is fixed: context usage should be derived from prompt/context tokens, not aggregate totals that may include output tokens.

## Best Possible Solution

Keep this PR open and ask for a focused revision. The implementation should derive `{context}` / `{contextPercent}` from prompt-side context usage, preferably reusing existing helpers such as `deriveContextPromptTokens` or `deriveSessionTotalTokens`, wire the usage callback through the reply prefix pipeline so gateway and plugin/channel deliveries receive it, and update the responsePrefix docs/config type comments and tests before maintainer review/merge.
