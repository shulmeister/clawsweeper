## Summary

Issue #67107 should stay open. Current main still requires Node >=22.14.0, the macOS installer still defaults to installing Node 24 through Homebrew without a macOS-version preflight, and the public install docs still describe macOS support broadly without a Big Sur/Monterey floor. The discussion also explicitly identifies the remaining work as installer/docs UX, which is actionable in this repo.

## Best Possible Solution

Keep the issue open and treat it as an installer/docs UX bug, not a request to lower the runtime to Node 18. The best fix is to add an early macOS version preflight in scripts/install.sh before Homebrew/Node installation, emit a clear Big Sur/macOS 11 message with supported alternatives such as Docker or running the gateway on another machine, and update public install/Node docs to state the practical macOS floor implied by the Node 22.14+/24 requirement.
