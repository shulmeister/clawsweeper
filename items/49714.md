## Summary

Current `main` already supersedes PR #49714 with a model-aware thinking-block replay/pruning implementation: Anthropic replay policy enables `dropThinkingBlocks` where needed, outbound embedded-runner calls sanitize replayed messages before every provider request, and context-pruning estimates can ignore non-latest thinking blocks that will be dropped before send. The relevant implementation is on `main` in commit `9f8b4006307d8c840814610e0d664155178aab12`, which post-dates the latest tagged release.

## Risks / Open Questions

- The current fix prevents the Anthropic replay mismatch on the send/pruning paths rather than adding the exact user-facing error text/reset flow proposed in this PR. If a future path bypasses `dropThinkingBlocks` and still surfaces the raw invalid-request payload, `formatAssistantErrorText()` would still fall back to generic invalid-request formatting.
- The implementation appears to have landed after the latest tagged release (`v2026.4.22`), so closing as implemented-on-main does not imply the fix is in the latest release build yet.
