## Summary

Keep open. Current `main` still documents and implements in-place TUI streaming, with no CLI flag, shortcut, or settings toggle to preserve intermediate assistant blocks during multi-tool turns.

## Evidence

- **TUI docs still describe overwrite-style streaming:** The current docs say tool partial updates stream into the same card and that streaming responses update in place until finalized; the documented settings/shortcuts only cover tool expansion and thinking visibility.
  - file: [docs/web/tui.md:183](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/docs/web/tui.md#L183)
  - command: `nl -ba docs/web/tui.md | sed -n '78,210p'`
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **No preserve flag in TUI CLI surface:** `openclaw tui` exposes `--local`, `--url`, `--token`, `--password`, `--session`, `--deliver`, `--thinking`, `--message`, `--timeout-ms`, and `--history-limit`; there is no preserve-intermediate style option. The `TuiOptions` type also has no corresponding field.
  - file: [src/cli/tui-cli.ts:10](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/cli/tui-cli.ts#L10)
  - command: `nl -ba src/cli/tui-cli.ts | sed -n '1,120p'; nl -ba src/tui/tui-types.ts | sed -n '1,30p'`

- **Assistant streaming reuses one message component per run:** `ChatLog.startAssistant`, `updateAssistant`, and `finalizeAssistant` all mutate the same `AssistantMessageComponent` stored in `streamingRuns`, which means intermediate text is replaced rather than appended as separate transcript entries.
  - file: [src/tui/components/chat-log.ts:186](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/tui/components/chat-log.ts#L186)

- **Event handling feeds deltas into the same assistant bubble:** On chat delta events, the TUI calls `chatLog.updateAssistant(...)`; on final events it calls `chatLog.finalizeAssistant(...)`. There is no alternate path that appends intermediate assistant blocks persistently.
  - file: [src/tui/tui-event-handlers.ts:313](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/tui/tui-event-handlers.ts#L313)
