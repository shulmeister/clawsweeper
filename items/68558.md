## Summary

Keep open. Current main still has the reported `/compact` output behavior, but the PR should not be cleaned up as obsolete or merged as-is: its proposed `currentTokenCount` forwarding appears to pass the model context cap rather than observed prompt usage, matching the P1 review concern in the PR discussion.

## Best Possible Solution

Keep PR #68558 open for author or maintainer revision. The product bug still exists on current main, but the fix should pass a real observed session/prompt token count if available, or omit `currentTokenCount`; guard label rendering when `before <= after`; add focused regression tests for small skipped sessions, `before <= after`, and stale stored totals; and then address the minor async/stat and duplicate path-review comments.
