## Summary

Keep open. Current `main` does not contain this PR's model-selection/display changes: slash-containing server model refs are still preserved without provider qualification on catalog miss, `/model` and usage details still display raw `session.model`, and the dropdown helper still only reads `agents.defaults.models` instead of merging `agents.defaults.model` primary/fallback entries.

That means `provider + model-with-slash` rows can still surface without the provider prefix when the catalog is empty.

## Risks / Open Questions

- The provider-prefix bug remains plausible on current `main` whenever the server stores `modelProvider` separately and the model id itself contains `/`, especially if the catalog is unavailable or incomplete.
- The PR's current approach still appears to need one more pass on stale/mismatched provider metadata before it is safe to land unchanged.
- The dropdown half of the PR is clearly not implemented on `main`, but the exact user-visible duplicate reproduction should be revalidated because current code seems to omit `agents.defaults.model` values rather than merge them.
