## Summary

Keep open. Current `main` still builds the agent-to-agent `sessions_send` prompts without the requester identity name, and the call sites still only pass requester session/channel data. The issue remains unfixed on `main` and in the latest tagged release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

## Evidence

- **A2A prompt builders still omit requester identity name:** `buildAgentSessionLines()` only emits requester session/channel plus target session/channel. `buildAgentToAgentMessageContext()`, `buildAgentToAgentReplyContext()`, and `buildAgentToAgentAnnounceContext()` all reuse that helper, so none can include `Agent 1 (requester) name: ...` today.

- **Initial sessions_send call site passes no identity information:** The main `sessions_send` path calls `buildAgentToAgentMessageContext()` with only `requesterSessionKey`, `requesterChannel`, and `targetSessionKey`. There is no requester name or config-derived identity lookup in this path.
