## Summary

Keep open. Current main still does not include the requester agent's configured identity name in sessions_send agent-to-agent message, reply, or announce prompts. The prior linked fix PR #38579 directly addressed this but was closed unmerged, so it does not make current main fixed.

## Best Possible Solution

Implement the narrow core fix in sessions_send: resolve the requester agent id from the requester session key, look up the configured agents.list[].identity.name, and include an `Agent 1 (requester) name: <name>.` line in the initial send, ping-pong reply, and final announce contexts when a non-empty name exists. Add focused tests that cover the initial prompt plus reply/announce prompts and preserve existing behavior when no requester identity name is configured.
