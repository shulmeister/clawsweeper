## Summary

Keep open. Current main still does not thread the requester agent identity name into `sessions_send` agent-to-agent prompts. The helper contract only supports requester session/channel plus target session/channel, the initial send and reply/announce paths only pass those fields, and current tests assert A2A prompt presence without covering `Agent 1 (requester) name`. The related fix PR #38579 was closed unmerged in the provided GitHub context, so it does not prove main is fixed.

## Best Possible Solution

Implement the requester identity plumbing in core: resolve the requester agent id from the requester session key/config, read the configured `identity.name`, add an optional requester-name field to the sessions_send A2A prompt helpers, pass it through initial send, reply, and announce flows, and add regression tests asserting `Agent 1 (requester) name: <name>.` appears when configured and is omitted when absent.
