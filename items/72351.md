## Summary

Current main still has the reported bug: `--history-limit 0` is accepted by the CLI, preserved by the TUI session loader, sent unchanged to `chat.history`, and rejected by the Gateway protocol schema. This PR remains a focused implementation candidate with matching regression coverage, and the existing review comments support that it is narrow rather than obsolete or superseded.

## Best Possible Solution

Keep this PR open and review or land the narrow TUI-side guard with its regression test. The Gateway protocol should continue rejecting `chat.history.limit: 0`; the TUI should interpret explicit `--history-limit 0` locally as no-history mode and avoid sending `chat.history` at all.
