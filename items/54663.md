## Summary

Keep this issue open. Current main still suppresses ordinary group-chat tool-result summaries, while Telegram/WhatsApp/channel default config surfaces do not expose a schema-backed opt-in such as allowVerboseToolResults. The related #44027 is still only an unmerged implementation candidate and proposes a /verbose behavior change rather than the requested config option.

## Best Possible Solution

Keep this issue open. The best path is a narrow, typed, schema-backed opt-in such as channels.defaults.allowVerboseToolResults plus provider/group/topic overrides, defaulting to false. Feed that policy into dispatch-from-config's shouldSendToolSummaries and ACP projector gates, add regression tests proving ordinary groups stay silent by default but send text summaries when explicitly enabled, preserve forum/media/exec-approval behavior, update public docs with a trusted-group warning, and coordinate the behavior with #44027.
