## Summary

Keep this PR open. Current main still starts Feishu accounts through the gateway per-account path under `Promise.all`, and that path still calls `fetchBotIdentityForMonitor` directly without a shared startup probe queue. The PR remains a focused implementation candidate for the linked Feishu multi-account startup timeout bug.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #63475. Maintainers should review or rebase it against current main, preserve the abort-while-waiting behavior from the follow-up commit, keep regression coverage for concurrent `fetchBotIdentityForMonitor` callers, and run the Feishu startup/changed gates before merging.
