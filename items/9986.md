## Summary

Keep #9986 open. Current main still does not trigger configured model fallback when context length is exceeded: overflow is handled by active-model compaction/tool-result truncation, then reported as a blocked `context_overflow`, while the outer model-fallback loop explicitly rethrows likely context-overflow errors instead of trying the configured fallback chain.

## Best Possible Solution

Keep #9986 open until maintainers either explicitly document a compaction-only policy for context overflow or implement a guarded fallback path after compaction and oversized-tool-result truncation are exhausted. The implementation should choose only configured fallback candidates with known larger context headroom, log the switch, preserve replay safety, and add regression coverage plus docs for smaller-primary to larger-fallback behavior.
