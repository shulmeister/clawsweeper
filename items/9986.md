## Summary

Current main has better context-overflow recovery than the report described: it detects context overflow, auto-compacts, retries, and eventually returns a user-facing error instead of freezing. However, the specific requested capability remains missing: configured model fallbacks are explicitly not used for context-overflow errors, and the current terminal path tells users to reset or choose a larger-context model manually. Related items do not cleanly supersede this: #28632 is output-token stopReason length, #34005 is graceful Ollama overflow handling, and #29014 is a broader overloaded/retry issue that only references this context-fallback request.
