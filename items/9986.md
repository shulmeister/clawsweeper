## Summary

Keep #9986 open. Current main still treats context overflow as compaction/tool-result-truncation recovery on the active model, then returns a blocked context_overflow result if recovery is exhausted. The outer model fallback loop, failover reason model, result classifier, docs, and tests still exclude context overflow from fallback routing, and the latest issue discussion already contains a Codex/maintainer comment explicitly keeping this item open.

## Best Possible Solution

Keep #9986 open until maintainers either intentionally document context overflow as compaction-only for model fallback, or implement a guarded larger-context fallback path. A complete fix likely needs replay-safety gating, trustworthy candidate context-window metadata, fallback candidate selection based on prompt size/headroom, logging, docs, and regression tests for smaller-primary to larger-fallback behavior after compaction/truncation recovery is exhausted.
