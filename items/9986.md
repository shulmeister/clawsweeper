## Summary

Keep #9986 open. Current main still does not implement context-overflow-triggered model fallback: context overflow is explicitly excluded from the outer model fallback loop, handled by embedded-runner compaction/truncation, and then returned as a terminal context_overflow error. The related issues in the provided context are narrower or different surfaces: #28632 is output-token length, #34005 is graceful Ollama overflow messaging/recovery, and #29014 covered overloaded/retry fallback rather than selecting a larger configured fallback after context overflow.

## Best Possible Solution

Keep #9986 open until maintainers either explicitly decide the documented policy is intentional or implement a guarded context-overflow fallback path. The likely implementation belongs around embedded-runner overflow exhaustion and model-fallback policy: after compaction and tool-result truncation are exhausted, replay only when safe, select a configured fallback with sufficient context headroom, log the transition, and add docs/tests for smaller-primary to larger-fallback behavior.

## Risks / Open Questions

- A naive implementation could replay side-effecting turns or completed tool work after overflow; any fallback path needs replay-safety checks and side-effect gating.
- Selecting a larger fallback requires reliable context-window metadata for the primary and fallback models, including tests for equal or smaller fallback windows.
