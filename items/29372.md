## Summary

Keep #29372 open. Current main still does not implement `agents.list[].subagents.workspace`: the per-agent config type, strict runtime schema, generated schema, and docs only expose existing subagent policy/model fields. The runtime also still treats omitted `agentId` as same-agent and deliberately preserves the caller/inherited workspace for same-agent spawns. The related cross-agent workspace bug is fixed on main, but that covers explicit `agentId` cross-agent spawns, not the requested default subagent workspace override.

## Best Possible Solution

Keep #29372 open for a core config/runtime implementation if maintainers want this isolation behavior. A complete fix should add `agents.list[].subagents.workspace` to TypeScript types, zod/runtime/generated schemas, public docs, path normalization/help, and spawn workspace resolution for omitted `agentId` and explicit same-agent spawns, with tests for omitted `agentId`, explicit same-agent, `requireAgentId`, cross-agent preservation, sandbox inheritance, and bootstrap isolation.
