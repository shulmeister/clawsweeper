## Summary

Keep open. Current main still does not implement the requested `agents.list[].subagents.workspace` override for default/same-agent subagents. The related cross-agent workspace inheritance bug from #40171 is addressed in current code, but the remaining #29372 behavior is still absent: omitted `agentId` resolves to the requester agent, same-agent spawns preserve the caller's inherited workspace, and the config type/schema/docs expose subagent allow/model/thinking/require settings but no workspace override.

## Best Possible Solution

Keep #29372 open for maintainer/product judgment. A complete implementation would add a documented `agents.list[].subagents.workspace` config surface, decide whether a default-level counterpart is needed, update TypeScript types, zod/runtime/generated schema/help/docs, and change same-agent/default spawn workspace resolution so omitted-`agentId` subagents can use the configured child workspace without breaking the already-fixed cross-agent behavior. Add regression tests for omitted `agentId`, same-agent explicit `agentId`, strict schema acceptance/rejection, and interaction with `requireAgentId`/cross-agent target workspaces. If maintainers reject the feature, the alternative should be explicitly documented as `requireAgentId` plus separate target agent profiles/workspaces.
