## Summary

Keep #12208 open. Current main still does not implement a bounded inner scroll view for every large chat bubble body; the native shared chat UI and Android chat UI render bubble contents directly inside the outer transcript scroll, and the old `.large`/truncation symbols are absent. The existing maintainer-posted Codex review also favored keeping this open, and the only newer local main commit touches a generated plugin SDK baseline rather than chat UI.

## Best Possible Solution

Keep the issue open for a focused native chat UI/product decision. The likely implementation belongs in `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMessageViews.swift` and possibly the Android chat surface too: define the current equivalent of a large bubble, wrap only that bounded body area in a vertical scroll container, preserve outer transcript scrolling, and add render/smoke coverage or manual mobile verification for tall-but-not-truncated content.
