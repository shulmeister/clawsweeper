## Summary

Keep #12208 open. Current main still does not implement an inner scroll body for all large or tall chat bubbles; Swift and Android chat bubbles render body content in plain stacks/columns inside only the transcript-level scroll containers. The prior Codex/steipete review comment correctly maps this to the native mobile chat surface and the unresolved behavior remains present on current main.

## Best Possible Solution

Keep this open for a focused native chat UI fix. Define the current equivalent of a large/tall bubble, wrap the bounded message body area in an inner vertical scroll container in `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMessageViews.swift`, add Android parity in `apps/android/app/src/main/java/ai/openclaw/app/ui/chat/ChatMessageViews.kt`, preserve transcript-level scrolling, and add focused render/smoke coverage or documented manual mobile verification for tall-but-not-truncated content.
