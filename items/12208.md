## Summary

Keep #12208 open. Current main still does not implement inner scrolling for every large bubble body; the native chat bubble bodies render direct stacks inside the outer message-list scroll, and the old `sizeClass == .large` / truncation symbols are absent from the current chat UI. The issue remains a concrete UI behavior request rather than a conservative cleanup close.

## Best Possible Solution

Keep the issue open for a focused native chat UI change. The likely implementation belongs in `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMessageViews.swift`: define the product rule for when a bubble is `.large`, wrap that bubble body in a bounded vertical `ScrollView`, and add render/smoke coverage or manual iPhone verification for tall-but-not-truncated message content.
