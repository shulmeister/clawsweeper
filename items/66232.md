## Summary

Close as duplicate/superseded. The same remaining heartbeat fallback work is already tracked by open issue #69434 and open PR #69495, while current main still has the string-only heartbeat model schema/runtime path, so this should not be marked implemented.

## Best Possible Solution

Close #66232 as superseded by the active heartbeat fallback tracker #69434 and implementation PR #69495. The implementation should continue there: change heartbeat config to use the shared model selection shape, carry explicit fallback lists through the heartbeat reply/run path, preserve `fallbacks: []` semantics, and add schema/runtime/docs tests.
