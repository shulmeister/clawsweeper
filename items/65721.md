## Summary

Current OpenClaw already recovers invalid config edits from a last-known-good snapshot during both gateway startup and config reload, and the same startup recovery code is present in release `v2026.4.23`, so the reported manual `openclaw doctor --fix` recovery path is obsolete for the shipped/current codebase.

## Evidence

- **Startup auto-recovers invalid config from last-known-good:** `loadGatewayStartupConfigSnapshot()` restores the config from `recoverConfigFromLastKnownGood(...)` before failing startup validation, and only throws after recovery is unavailable.
  - file: [src/gateway/server-startup-config.ts:71](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/gateway/server-startup-config.ts#L71)
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **Startup recovery is covered by regression tests:** Test asserts an invalid snapshot is replaced with a recovered valid snapshot, logs the restore, and enqueues a recovery notice; a separate test confirms failure is only kept when recovery is unavailable.
  - file: [src/gateway/server-startup-config.recovery.test.ts:57](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/gateway/server-startup-config.recovery.test.ts#L57)
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **Runtime reload auto-recovers invalid external config edits:** The managed config reloader restores last-known-good after an invalid external edit and continues with the recovered snapshot instead of restarting into a broken state.
  - file: [src/gateway/config-reload.ts:229](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/gateway/config-reload.ts#L229)
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **Reload recovery is also regression-tested:** Test verifies invalid external edits trigger `recoverSnapshot(..., "invalid-config")`, reload the recovered snapshot, and avoid a gateway restart.
  - file: [src/gateway/config-reload.test.ts:673](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/gateway/config-reload.test.ts#L673)
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **Healthy startups promote a last-known-good snapshot and normal config writes keep backups:** Gateway startup promotes the accepted config to last-known-good, and `writeConfigFile()` rotates `openclaw.json.bak` backups before replacing the config file.
  - file: [src/gateway/server.impl.ts:903](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/gateway/server.impl.ts#L903)
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **Backup rotation on config writes:** `writeConfigFile()` calls `maintainConfigBackups(...)` when the config already exists, preserving a backup before overwriting.
  - file: [src/config/io.ts:1847](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/config/io.ts#L1847)
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **Same startup recovery code is already in the latest shipped release:** The `v2026.4.23` tag contains the same `recoverConfigFromLastKnownGood(...)` startup path in `src/gateway/server-startup-config.ts`, so this is not just an unreleased main-branch behavior.
  - file: [src/gateway/server-startup-config.ts:71](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/gateway/server-startup-config.ts#L71)
