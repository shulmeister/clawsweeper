## Summary

Keep PR #31172 open. Current main still does not implement a Feishu `streamingThrottleMs` config path: the runtime uses a private hardcoded 160ms throttle, the dispatcher does not pass a config-derived value, and the schema/manifest/docs omit the setting. The stale-closed linked issue #31136 does not supersede this open PR, though the PR needs maintainer review because its proposed 100ms default no longer matches current main's 160ms behavior.

## Best Possible Solution

Keep the PR open for maintainer review or a rebased patch. A complete implementation should add `streamingThrottleMs` to the Feishu schema and plugin metadata, pass the resolved account value into `FeishuStreamingSession`, document the option on the Feishu docs page, and add focused tests for the default and config plumbing. Maintainers should explicitly choose whether the default stays at current main's 160ms or changes to the PR's proposed 100ms.
