## Summary

Current `main` already applies an explicit `sessions_spawn.model` before child startup and persists the child runtime model into session state before the `agent` run begins, with matching tests and docs. The latest release I could directly confirm containing this behavior is `v2026.4.23` (`a9797214338ba31c52c796adbb75afb16e0684a9`).
