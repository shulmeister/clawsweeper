## Summary

Current main still does not implement this PR's central behavior: there is no `/theme` slash command, no interactive theme picker, no `/theme list`, and no runtime theme application API. Main only supports startup light/dark palette selection through `OPENCLAW_THEME` and `COLORFGBG`, with docs and tests focused on that startup behavior. The PR remains a focused TUI feature with useful branch follow-up in the discussion, so it is not safely closable as implemented, duplicate, obsolete, incoherent, or ClawHub-scoped.

## Best Possible Solution

Keep this PR open for maintainer/product judgment. If the feature is accepted, integrate it in the core TUI: add `/theme`, `/theme list`, completions, a searchable picker using the existing select-list surfaces, live preview with cancel/commit semantics, docs, and focused tests for runtime switching, terminal color reset, and repaint of preformatted header/footer/status UI.
