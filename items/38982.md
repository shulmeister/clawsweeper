## Summary

Keep open. Current main still does not implement this PR's central behavior: no `/theme` slash command, no interactive theme picker, no `/theme list`, and no runtime theme switching API. Main only has startup-time light/dark palette selection through `OPENCLAW_THEME`/`COLORFGBG`, while the PR discussion shows focused follow-up work rather than an obsolete or incoherent change.

## Best Possible Solution

Keep this PR open unless a maintainer intentionally supersedes it with a clearer TUI theming plan. The implementation path belongs in the TUI: add `/theme` registration in `src/tui/commands.ts`, handle runtime theme selection in `src/tui/tui-command-handlers.ts`, provide a well-tested theme mutation/apply API in `src/tui/theme/theme.ts`, and cover live preview, cancel revert, `/theme list`, terminal color reset, and full repaint of header/footer/status surfaces.
