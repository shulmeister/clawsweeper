## Summary

Keep #38982 open. Current main still does not ship the PR's central TUI behavior: no `/theme` slash command, no `/theme list`, no runtime theme picker/live preview, and no runtime theme switching API. The PR remains substantive prior work, and the overlapping #44382 is still open rather than a shipped or explicitly canonical replacement.

## Best Possible Solution

Keep this PR open unless a maintainer explicitly supersedes it with a single TUI theming plan. The likely path is to coordinate #38982 with #44382, then land one focused implementation that registers `/theme`, handles `/theme`, `/theme list`, searchable picker preview, commit, and cancel/revert in the TUI command path, adds a tested runtime theme API, and ensures editor, markdown, overlays, header, footer, and status surfaces repaint consistently.
