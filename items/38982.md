## Summary

Keep #38982 open. Current main still ships only startup light/dark terminal-background handling, not the PR’s central runtime TUI theme-picker behavior: there is no `/theme` slash command, no `/theme list`, no runtime theme-switching API, and no picker live-preview/cancel/commit path. The overlapping #44382 is useful related work but remains open and is not a shipped or explicitly canonical replacement in the provided discussion.

## Best Possible Solution

Keep #38982 open unless a maintainer explicitly chooses #44382 as the canonical TUI theming path. The likely product path is to coordinate both PRs, then land one focused implementation that adds a runtime theme API, registers `/theme`, supports `/theme list`, opens a searchable picker with live preview, commits on select, reverts on cancel, repaints editor/header/footer/status/overlay surfaces consistently, updates public TUI docs, and adds targeted command/theme tests.
