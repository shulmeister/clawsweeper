## Summary

Current `main` has shipped substantial parts of the roadmap: SecretRef-backed credential storage across supported config surfaces, `openclaw secrets` audit/configure/apply/reload flows, and `models.json` marker persistence instead of resolved provider secrets. But the implementation is explicitly additive and opt-in, not a mandatory protection layer: `docs/gateway/secrets.md` still says plaintext storage works, so the broad "protecting API keys from agent access" problem is not fully closed by default for existing plaintext configs or other readable secret files.
