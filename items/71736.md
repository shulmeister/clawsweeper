## Summary

Keep #71736 open. Current main still lacks the requested Control UI contribution/descriptor registry; it only has adjacent plugin slash-command discovery and generic plugin approval rendering. The newer #72287 PR is an open implementation candidate that supersedes #72082, but it is not merged or shipped, so this remains an active core SDK/UI design item rather than cleanup-closeable.

## Best Possible Solution

Keep #71736 open as the focused Control UI contribution-slot RFC. Review #72287 as the implementation candidate; if it lands, close this as implemented with the landed commit/release. The desired implementation should add a safe data-only Control UI descriptor contract, Gateway projection such as plugins.uiDescriptors, host-owned rendering/catalog semantics, enable-disable cleanup, sanitization, and fixture tests for chat modes, approvals, input guards, status surfaces, and UI actions.
