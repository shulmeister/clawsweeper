## Summary

Keep open. Current `main` supports OpenAI TTS `speed` in provider config and Talk requests, but the reported `[[tts:speed=...]]` directive path is still not implemented for the OpenAI speech provider.

## Evidence

- **OpenAI API layer already accepts speed:** `openaiTTS` takes an optional `speed` parameter and includes it in the `/audio/speech` JSON body when present.
  - file: [extensions/openai/tts.ts:111](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/extensions/openai/tts.ts#L111)
  - command: `nl -ba extensions/openai/tts.ts | sed -n '111,147p'`
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Provider config and Talk overrides pass speed through:** The OpenAI speech provider reads `config.speed`, maps Talk `params.speed`, and passes `overrides.speed ?? config.speed` into `openaiTTS`. That covers config-driven and Talk-driven speed, not directive parsing.
  - file: [extensions/openai/speech-provider.ts:115](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/extensions/openai/speech-provider.ts#L115)
  - command: `nl -ba extensions/openai/speech-provider.ts | sed -n '99,145p'; nl -ba extensions/openai/speech-provider.ts | sed -n '220,254p'`
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **OpenAI directive parser does not handle speed:** `parseDirectiveToken` for the OpenAI speech provider only recognizes voice/model keys and falls through on everything else; there is no `speed` case.
  - file: [extensions/openai/speech-provider.ts:148](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/extensions/openai/speech-provider.ts#L148)
  - command: `nl -ba extensions/openai/speech-provider.ts | sed -n '148,178p'`
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Generic directive routing sends speed to whichever provider handles it:** The shared directive parser walks providers until one returns `handled`. In current `main`, `speed` is implemented for ElevenLabs and MiniMax, and the routing tests only cover those providers. With OpenAI lacking a `speed` handler, `[[tts:speed=...]]` still will not become an OpenAI override.
  - file: [src/tts/directives.ts:142](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/tts/directives.ts#L142)
  - command: `nl -ba src/tts/directives.ts | sed -n '136,168p'; nl -ba extensions/elevenlabs/speech-provider.ts | sed -n '216,225p'; nl -ba extensions/minimax/speech-provider.ts | sed -n '114,122p'; nl -ba src/tts/directives.test.ts | sed -n '81,168p'`
