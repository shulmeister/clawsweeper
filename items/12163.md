## Summary

Keep #12163 open. Current main forwards OpenAI TTS speed from provider config and Talk/provider overrides into the `/audio/speech` request, but the reported `[[tts:speed=1.5]]` path is still missing because the OpenAI directive parser handles only voice/model tokens and returns unhandled for `speed`.

## Best Possible Solution

Keep the issue open. Implement OpenAI-owned speed directive handling in `extensions/openai/speech-provider.ts`: accept `speed` only when `policy.allowVoiceSettings` is enabled, parse a finite number, validate OpenAI's `0.25..4.0` range, return `{ speed }` as provider overrides, and add focused tests for preferred OpenAI `[[tts:speed=1.5]]`, explicit `[[tts:provider=openai speed=1.5]]`, disabled voice settings, invalid/out-of-range values, and final OpenAI request body serialization.

## Evidence
