## Summary

Keep #12163 open. Current main already passes configured/OpenAI runtime `speed` through to `/audio/speech`, but the issue’s reported per-reply directive path still is not implemented for OpenAI: the OpenAI speech provider only parses voice/model directive tokens, so `[[tts:speed=1.5]]` cannot create an OpenAI provider override. The prior user PRs #12177 and #22618 remain useful context, but both were closed unmerged and do not supersede the issue.

## Best Possible Solution

Implement the missing OpenAI directive handler in `extensions/openai/speech-provider.ts`: parse `speed` when `allowVoiceSettings` is enabled, validate against OpenAI’s supported `0.25..4.0` range, return `{ speed }`, and add focused tests for `[[tts:speed=...]]` with OpenAI as the active or declared provider. The existing request-body and synthesize plumbing can stay.
