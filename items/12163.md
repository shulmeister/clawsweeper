## Summary

Keep #12163 open. Current main partially supports OpenAI TTS speed through provider config, Talk overrides, and the OpenAI `/audio/speech` request body, but the reported `[[tts:speed=1.5]]` directive path is still not implemented for the bundled OpenAI speech provider. The related PRs in the provided context (#12177 and #22618) were closed unmerged, and the relevant files are unchanged since the earlier keep-open review.

## Best Possible Solution

Keep the issue open. Implement OpenAI-owned directive handling in `extensions/openai/speech-provider.ts`: handle `speed` only when `policy.allowVoiceSettings` is enabled, parse and validate finite numbers against OpenAI's `0.25..4.0` range, return `{ speed }` as provider overrides, and add focused tests proving the directive reaches the OpenAI request body for both preferred-provider and explicit-provider forms while preserving the existing config and Talk speed plumbing.

## Risks / Open Questions

- A narrow fix that copies ElevenLabs' `0.5..2.0` directive range would still under-support OpenAI's documented `0.25..4.0` range.
- Regression coverage should include both preferred OpenAI provider `[[tts:speed=1.5]]` and explicit `[[tts:provider=openai speed=1.5]]`, with `allowVoiceSettings` disabled/enabled behavior.
