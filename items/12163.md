## Summary

Keep #12163 open. Current main already forwards configured/OpenAI runtime speed to the OpenAI `/audio/speech` request, but the reported per-reply directive path still is not implemented for OpenAI: `[[tts:speed=1.5]]` cannot create `providerOverrides.openai.speed` because the OpenAI speech provider only parses voice/model directive tokens.

## Best Possible Solution

Implement the missing OpenAI directive handler in `extensions/openai/speech-provider.ts`: parse `speed` when `allowVoiceSettings` is enabled, validate OpenAI's supported `0.25..4.0` range, return `{ speed }`, and add focused tests proving `[[tts:speed=...]]` routes to `providerOverrides.openai` when OpenAI is the active or declared provider. Keep the existing request-body and synthesize plumbing.
