## Summary

Current `main` already implements this request as `browser.extraArgs`: the config schema accepts it, the browser resolver preserves it, the Chromium launcher appends it to launch flags, tests cover it, and the changelog says it shipped in `v2026.2.17`.

## Risks / Open Questions

- Current implementation is append-only; it does not add a separate first-class config for removing built-in flags, so future reports should be reopened only if appended flags are insufficient on current builds.

## Close Comment

Closing as implemented. Codex review against current `main` found the user-facing `browser.extraArgs` config field in `src/config/types.browser.ts:68` and `src/config/zod-schema.ts:411`, with the Chromium launcher appending those flags in `extensions/browser/src/browser/chrome.ts:135`. The behavior is covered by tests in `extensions/browser/src/browser/chrome.internal.test.ts:206`, documented in `docs/gateway/configuration-reference.md:3008`, and recorded in `CHANGELOG.md:4611` as shipped in `v2026.2.17`. If you are still hitting a current-build case where appended flags are not enough and a built-in flag must be explicitly suppressed, please open a new issue with the exact config and conflicting launch flags.
