## Summary

Keep #72252 open. Current main still has the reported heartbeat exec-completion relay behavior: structured exec completions are classified as exec events, all exec events are treated as relayable whenever channel alert delivery is enabled, and exec completions explicitly bypass normal HEARTBEAT_OK skip handling. The directly linked PR #72253 tracks the exact fix, but it is still open/unmerged, so this should not be closed as implemented yet.

## Best Possible Solution

Keep #72252 open until PR #72253, or an equivalent main-branch change, lands. The fix should classify structured successful exec completions and restart-cleanup SIGTERM completions as internal-only, preserve relay behavior for real non-SIGTERM failures, and update focused heartbeat filter/runner tests for code 0, SIGTERM, code 1, and mixed-event batches.
