## Summary

Keep open. Current main still rejects `auth-profiles.json` entries whose credential `type` is `aws-sdk`; the linked PR #69753 proposed the right-shaped narrow fix, but it is closed unmerged and the current checkout lacks that marker-skip behavior.

## Best Possible Solution

Keep #69708 open and land a narrow auth-profile loader fix: treat `type: "aws-sdk"` as a non-secret SDK-managed marker rather than a malformed credential, avoid the invalid-entry warning for that marker, preserve real invalid-entry warnings, and add regression coverage matching PR #69753. Also verify Bedrock `auth: "aws-sdk"` still reaches the runtime AWS SDK default chain without requiring a static API key.
