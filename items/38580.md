## Summary

Keep open. Current main still does not implement on-demand Telegram pinned-message reads. Generic `list-pins` plumbing exists, but the bundled Telegram plugin does not advertise, map, or handle `list-pins`/`getpinned`; its config/schema lacks a `pins` gate; its `getChat` helper only returns a chat id; and docs/tests still list pin reads for Discord/Slack/Matrix but not Telegram. Related PRs #38613 and #38718 are useful prior art, but the provided GitHub context shows both were closed unmerged.

## Best Possible Solution

Keep #38580 open until the bundled Telegram plugin implements an on-demand pinned-message read path, preferably by opting Telegram into the existing `list-pins`/`pins` message action and optionally adding a native `/getpinned` command. The implementation should add Telegram action discovery/runtime plumbing, a config/schema gate, docs, and regression tests for no pin, API errors, target/account resolution, and the Bot API's chat-level pin limitation. Use closed PRs #38613 and #38718 as prior art, not canonical fixes.
