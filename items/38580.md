## Summary

Keep #38580 open. Current main still has Telegram outbound delivery pinning, but it does not expose a Telegram pinned-message read path backed by Bot API `getChat.pinned_message`. The generic `list-pins` action exists for other channels, while Telegram discovery/runtime/config/docs still omit it; the related PRs #38613 and #38718 were closed unmerged in the provided context.

## Best Possible Solution

Keep #38580 open until the bundled Telegram plugin implements a pinned-message read path, likely a `list-pins` or `read-pinned` message action backed by Telegram Bot API `getChat.pinned_message`, with Telegram action discovery/runtime plumbing, config/schema metadata, CLI/tool formatting, tests, and public docs. If maintainers decide this belongs outside the bundled Telegram plugin, record that explicit product decision before closing.
