## Summary

Keep open. Current main still does not expose a Telegram pinned-message reader through the Telegram message action surface, runtime aliases, native slash commands, or docs. Existing Telegram pin support is outbound delivery pinning / pin-unpin helpers, not `getChat.pinned_message` retrieval. The related PRs cited in the issue context were closed unmerged, so no canonical shipped fix supersedes this issue.

## Best Possible Solution

Keep the issue open and implement the feature in the bundled Telegram plugin: add an on-demand pinned-message read action backed by Telegram Bot API `getChat.pinned_message`, expose it through the channel message action surface and/or a native `/getpinned` command, gate it with Telegram action config, and add docs plus tests for pinned message, no pin, missing token, disabled action, and target/account resolution.
