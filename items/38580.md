## Summary

Keep open. Current `main` has Telegram support for outbound pinning and a `getChat` helper for username lookup, but it does not implement reading pinned Telegram messages or expose a Telegram `list-pins`/`read pinned` action.

## Evidence

- **Telegram `getChat` helper only returns chat id:** `fetchTelegramChatId()` calls Telegram `getChat`, but it parses only `result.id` and discards the rest of the payload, so there is no current path that surfaces `pinned_message` from that API response.
  - file: [extensions/telegram/src/api-fetch.ts:34](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/extensions/telegram/src/api-fetch.ts#L34)
  - command: `nl -ba extensions/telegram/src/api-fetch.ts | sed -n '1,90p'`
  - sha: [51186d272543](https://github.com/openclaw/openclaw/commit/51186d2725439646f7fea0e59bb466f957404c46)
- **Telegram action map omits any pinned-message read action:** The Telegram message-action map only wires `send`, `poll`, `react`, `delete`, `edit`, `sticker`, `sticker-search`, `topic-create`, and `topic-edit`. The advertised Telegram actions likewise omit `read`, `pin`, `unpin`, and `list-pins`.
  - file: [extensions/telegram/src/channel-actions.ts:40](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/extensions/telegram/src/channel-actions.ts#L40)
  - command: `nl -ba extensions/telegram/src/channel-actions.ts | sed -n '40,160p'`
  - sha: [51186d272543](https://github.com/openclaw/openclaw/commit/51186d2725439646f7fea0e59bb466f957404c46)
- **Generic CLI has `pins`, but Telegram does not back it:** Core CLI registers generic `message pin`, `message unpin`, and `message pins` commands. Because Telegram does not expose `pin`/`unpin`/`list-pins` in its channel action adapter, that generic surface is not implemented for Telegram on current `main`.
  - file: [src/cli/program/message/register.pins.ts:4](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/src/cli/program/message/register.pins.ts#L4)
  - command: `nl -ba src/cli/program/message/register.pins.ts | sed -n '1,60p'`
  - sha: [51186d272543](https://github.com/openclaw/openclaw/commit/51186d2725439646f7fea0e59bb466f957404c46)
- **No documented Telegram command for reading pinned messages:** The bundled slash-command docs list current built-in commands, and none provide a Telegram pinned-message read flow such as `read pinned`/`getpinned`.
  - file: [docs/tools/slash-commands.md:148](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/docs/tools/slash-commands.md#L148)

  - sha: [51186d272543](https://github.com/openclaw/openclaw/commit/51186d2725439646f7fea0e59bb466f957404c46)
- **Cross-plugin search shows no Telegram `list-pins` implementation:** A repo search for `"list-pins"|listPins` finds implementations in Slack, Matrix, Discord, Feishu, and MSTeams, but none in `extensions/telegram`, which matches the missing Telegram action wiring above.
  - command: `rg -n '"list-pins"|listPins|action === "list-pins"|case "list-pins"' src extensions --glob '!**/dist/**'`
