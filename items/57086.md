## Summary

Keep open. Current `main` still carries the behavior this PR is trying to change: gateway agent and node-event dispatch both forward `sessionId` alongside a canonical `sessionKey`, and downstream session resolution still gives an explicit `sessionId` precedence over freshness rollover. The linked bug therefore still appears real and unshipped on `main`.
