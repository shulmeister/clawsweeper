## Summary

PR #66615 is superseded by open PR #66650 for the same #66573 failure. Current main still keeps the `chat.history` protocol limit at 1000, and the observed failure boundary is the `sessions_history` tool forwarding an oversized model-supplied limit. #66650 tracks the narrower caller-side clamp with regression coverage instead of expanding the gateway protocol limit to 2000.

## Best Possible Solution

Close #66615 as superseded and continue #66573 in #66650 or an equivalent patch: keep the current 1000-entry `chat.history` contract, clamp/validate `sessions_history` to that maximum, and keep the regression test. Only raise the gateway limit through an explicit protocol/product decision.
