## Summary

PR #66615 should close as superseded by open PR #66650. Current main still has the 1000-entry `chat.history` contract and the reported `sessions_history` caller path can still forward an oversized limit, but #66650 tracks the same #66573 bug with the narrower caller-side clamp and tests instead of expanding the gateway protocol limit to 2000.

## Best Possible Solution

Close #66615 as superseded and continue the #66573 fix in #66650 or an equivalent patch: clamp/validate `sessions_history` to the current `chat.history` maximum and add regression coverage. Only raise the gateway `chat.history` maximum if maintainers explicitly choose a broader protocol change and update tests/docs/stubs with it.

## Evidence
