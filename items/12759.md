## Summary

Keep #12759 open. Current main still has only individual `channel:*` labeler rules and no `channel: ALL`, `channel: NEW`, or collapse logic that would replace many channel labels with one catch-all label. The related PR #71548 attempted this work but was closed unmerged, so it does not supersede or ship the requested behavior.

## Best Possible Solution

Keep the issue open until repo-controlled labeling implements the catch-all behavior: add the maintainer-approved label and metadata, update `.github/labeler.yml` or the relevant labeler/agent logic so broad channel changes receive one catch-all label instead of many individual `channel:*` labels, sync the GitHub label, and verify with a PR touching shared or multiple channel paths.
