## Summary

Keep #12759 open. Current main still has only individual channel labels in repo-controlled labeler automation, no `channel: ALL` or `channel: NEW` catch-all, and no workflow/script collapse logic. The related PR #71548 attempted the requested behavior but was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open until repo-controlled labeling implements the catch-all behavior: add the maintainer-approved label and metadata, update `.github/labeler.yml` and/or labeler workflow logic so broad channel changes receive one catch-all label instead of many individual `channel:*` labels, sync the GitHub label, and verify against a representative PR touching multiple channel/plugin paths.
