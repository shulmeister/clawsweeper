## Summary

PR #64600 adds a draft bundled Servepath provider. Current main does not ship Servepath, but the request is an optional hosted OpenAI-compatible provider integration that can be implemented with the current provider plugin API and distributed through ClawHub/npm. The PR body also says Servepath already works through the generic custom-provider path; the remaining value is first-class discovery/onboarding, which current external provider-plugin surfaces cover without a missing core API or protected maintainer signal.

## Best Possible Solution

Close the core PR and have the Servepath work live as an external provider plugin published through ClawHub/npm. Keep core changes limited to generic plugin/provider SDK improvements if extracting Servepath uncovers a real missing API seam.
