## Summary

Keep #56674 open. Current main still does not implement the PR's central request: `/v1/responses` does not collect model thinking/reasoning into non-streaming `type: "reasoning"` output items, and the streaming API does not emit `response.reasoning.delta` or `response.reasoning.done`. The PR discussion is substantive, and the author responded to the bot review findings with fixes on the PR head, so this should remain available for maintainer review rather than be cleaned up as obsolete.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase or port the PR onto current main, preserve its internal non-broadcast reasoning capture guard, add/keep regression coverage for non-stream output, streaming `response.reasoning.delta`/`done`, tool-call ordering, duplicate terminal events, and shared-prefix reasoning segments, then update the OpenResponses docs to document the shipped reasoning output/events.
