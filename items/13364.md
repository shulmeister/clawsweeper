## Summary

Keep #13364 open. Current main still exposes `before_tool_call` / `after_tool_call` through the plugin hook runner, but managed/internal `HOOK.md` hooks remain void-returning internal events without a tool-call event type, blocking return contract, parameter rewrite contract, or bridge from tool execution. The issue also has recent maintainer signal to keep it open as the canonical tracker; #17008 was closed as a duplicate and #29590 was closed unmerged as only a partial after-tool-call attempt.

## Best Possible Solution

Keep #13364 open for maintainer product/API design. A complete implementation should add typed managed/internal before/after tool-call events, define blocking and parameter-rewrite return semantics, validate or reject unsupported HOOK.md event names, bridge the relevant tool execution paths without double-firing existing plugin hooks, and add focused internal-hook tests plus docs. If maintainers decide tool-call interception should stay plugin-only, close later with an explicit product decision and point users to plugin hooks and ClawHub packaging.
