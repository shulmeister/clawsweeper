## Summary

Keep this PR open. Current main still suppresses text-only tool summaries, plan updates, and tool-start statuses for non-forum group sessions, regardless of `CommandAuthorized`, while the PR targets that remaining behavior with a provider-agnostic condition. The latest PR context also shows the author addressed the bot feedback by removing Feishu/Lark hardcoding, making tool-start statuses symmetric with summaries, and scoping the bypass to real control commands.

## Best Possible Solution

Keep this PR open and review it as an implementation candidate. The best path is to rebase/adapt it to current `main`, preserve the provider-agnostic condition, allow summaries and tool-start statuses only for non-forum group turns where `CommandAuthorized === true` and the current command body is a real control command, and keep the regression coverage for authorized, unauthorized, mention-prefixed, and path-like slash inputs.
