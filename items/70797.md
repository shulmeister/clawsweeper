## Summary

Keep open. Current main still matches the reported leak class: the shared visible-text sanitizer only enters downgraded tool-call stripping for `[Tool ...]` or `[Historical context ...]` markers, so a bare `{"tool_calls":[...]}` payload returns unchanged, and the partial reply path passes `cleanedText` to `onPartialReply` without a streaming-aware tool-call JSON guard. The linked #71954 was closed unmerged, and #72011 is still an open fix attempt rather than shipped main behavior.

## Best Possible Solution

Keep #70797 open until #72011 or an equivalent fix lands on main. The fix should add a bare `tool_calls` JSON sanitizer for final visible text and a streaming-aware guard before `onPartialReply`, with regressions in `src/shared/text/assistant-visible-text.test.ts` and `src/agents/pi-embedded-subscribe.handlers.messages.test.ts` covering incomplete and complete Telegram preview payloads while preserving ordinary JSON/code examples.
