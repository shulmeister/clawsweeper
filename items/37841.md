## Summary

Current main still matches the reported bug. A successful same-target media-only message-tool send records target/media metadata, and `buildReplyPayloads` still converts same-target suppression into `replyPayloads = []` without requiring any message-tool text to have been sent. PR #37900 described the right direction, but it was closed unmerged and the current source lacks that gate.

## Best Possible Solution

Keep the issue open. Implement the PR #37900 behavior or an equivalent fix: media-only same-target sends may dedupe already-sent media, but must not set `replyPayloads` to `[]` unless the message tool actually sent text that should suppress the final session reply. Add a same-target media-only regression test in `src/auto-reply/reply/agent-runner-payloads.test.ts`.
