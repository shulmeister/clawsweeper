## Summary

Current main still appears to have the reported same-target media-only suppression bug. The reply path passes message-tool text/media/target telemetry into `buildReplyPayloads`, but same-target suppression is still driven by target metadata and then drops all final payloads without requiring that the message tool sent text. A media-only message tool call can commit target/media metadata while leaving `messagingToolSentTexts` empty. The exact proposed fix in #37900 was closed unmerged.

## Best Possible Solution

Keep #37841 open. Implement the narrow reply-payload fix so same-target final/follow-up session text is suppressed only when the message tool actually sent text to that same origin target. Media-only sends should continue to dedupe already-sent media URLs while preserving the final session text. Add focused regression coverage for same-target media-only sends in `buildReplyPayloads` and the follow-up delivery path.
