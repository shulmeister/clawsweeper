## Summary

Keep #37841 open. Current main still suppresses final and follow-up session text using same-target message-tool metadata even when the tool only sent media and recorded no sent text. The useful linked fix attempt in PR #37900 targeted the right behavior, but the provided GitHub context shows it was closed unmerged, and current tests still do not cover same-target media-only text preservation.

## Best Possible Solution

Keep #37841 open. Patch `buildReplyPayloads` and `resolveFollowupDeliveryPayloads` so same-origin target suppression only drops final or follow-up session text when the message tool actually sent text to that same target. Media-only sends should still dedupe already-sent media URLs while preserving session text. Add focused regression tests for same-target media-only sends preserving text, same-target text sends still suppressing duplicates, and the follow-up delivery path.

## Evidence

A same-target media-only send therefore still drops the final session text.
