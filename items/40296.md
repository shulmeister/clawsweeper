## Summary

Do not close this PR. Current main still places the channel/session-sensitive Messaging and Reactions sections before the large Project Context block and before the internal cache boundary, so the exact prefix-cache instability reported by #40256 is not fully implemented away on main.

## Best Possible Solution

Keep the PR or linked #40256 open and refresh the implementation against current main. The best path is to preserve the existing `SYSTEM_PROMPT_CACHE_BOUNDARY`, move or split channel/session-sensitive prompt text (`## Messaging` inline-button/runtime-channel lines, `## Reactions`, and any comparable per-session additions) so large stable Project Context remains before volatile content, then add a regression test in the current test suite asserting Project Context/cache-boundary ordering for Messaging, Group Chat Context, Reactions, and runtime-specific channel guidance.
