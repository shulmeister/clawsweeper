## Summary

Keep #71728 open. Current main has adjacent truthfulness guidance, input provenance, memory citations, and memory-wiki claim metadata, but it does not implement the RFC's requested claim-level confidence/source signaling for every assistant assertion or autonomous completion reporting for unverified claims. The issue is fresh, coherent, non-maintainer-authored, and has no protected labels; the prior ClawSweeper comment also correctly identified the same remaining product/API gap.

## Best Possible Solution

Keep the RFC open for maintainer product/API design. A good path is to define a core or SDK-visible trust signal for assistant output, with optional claim-level evidence/source metadata, known-vs-inferred wording guidance, and autonomous completion-event fields for unverified claims. Existing memory citations, input provenance, and memory-wiki claim metadata can feed that design, but they are not a complete implementation on current main.
