## Summary

Keep open. Current main still does not provide a macOS LaunchDaemon/system-scope Gateway mode: managed macOS install/start/restart remains LaunchAgent-only, the CLI has no daemon/scope flag, the macOS app calls the same LaunchAgent install path, and docs still say headless macOS needs a custom LaunchDaemon that is not shipped. The existing `gateway status --deep` path can detect system LaunchDaemons, but it does not install or manage one.

## Best Possible Solution

Keep this issue open for maintainer product/security review. The likely implementation belongs on the managed gateway service surface, preferably an explicit macOS system-scope mode on `openclaw gateway install` and matching lifecycle/status/doctor flows, not only `gateway start`. It should install a `/Library/LaunchDaemons` plist with deliberate `UserName`, state/config/env handling, sudo install/uninstall behavior, `launchctl system/...` lifecycle support, docs, and regression tests while preserving per-user LaunchAgent as the default.
