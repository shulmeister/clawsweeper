## Summary

Keep open. Current main still has the Telegram message-action token lookup path described by #61012: omitted accountId reaches action-runtime as undefined, and action-runtime resolves the token with resolveTelegramToken rather than the default-account-aware resolveTelegramAccount path. The linked replacement PR #61144 was closed unmerged in the provided GitHub context, and its described code change is not present on main.
