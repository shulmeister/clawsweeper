## Summary

Keep open. Current main mitigates the narrow Anthropic-default-without-auth path, but it does not implement this PR's general fail-fast startup auth guard. Gateway sidecars still prewarm the configured primary model and then start channels; prewarm only checks model resolution and logs warnings, not provider credentials. The PR also has unresolved review concerns because its proposed helper checks only env/profile auth and can miss valid runtime auth modes.

## Best Possible Solution

Keep the PR open for contributor or maintainer revision. The best path is to implement the startup preflight against the same auth contract used by runtime model execution, likely by resolving the selected default model and then using hasAvailableAuthForProvider/resolveApiKeyForProvider or an equivalent non-mutating wrapper before channel startup. Preserve the skipChannels bypass, avoid circular model suggestions, and add focused tests for missing auth, valid config-backed auth, Bedrock/aws-sdk or local synthetic auth, and the openai-codex example edge case.

## Risks / Open Questions

- Merging the PR as-is could block valid configurations that authenticate through models.providers.<provider>.apiKey, AWS SDK/Bedrock fallback, plugin-owned auth, or synthetic local auth.
- The PR's user-facing example can suggest switching to the same failing openai-codex model, so the error text needs conditional or generic guidance.
- Closing the PR as implemented would lose a still-unimplemented general startup guard for explicitly configured default models with missing auth.
