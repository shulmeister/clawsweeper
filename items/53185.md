## Summary

Keep this issue open. Current main has meaningful web_fetch sanitization and now has a trusted bundled tool-result middleware seam, but it still does not implement the requested web_fetch/web_search content-sanitizer.py pipeline, config-driven script preprocessing, external plugin interception, web_search threat_level metadata, or HIGH/CRITICAL block decision path. The related #54508 is broader taint/risk-class security work, not a full replacement for this narrower preprocessing-hook request.

## Best Possible Solution

Keep this issue open but narrow it. Treat web_fetch hidden-content stripping as substantially done, then decide whether OpenClaw should add a trusted pre-model tool-result preprocessing API for web_fetch/web_search, extend the existing bundled-only middleware into a documented external trust policy, or fold only the taint/risk-class parts into #54508 while leaving the custom script/preprocessing question tracked here.
