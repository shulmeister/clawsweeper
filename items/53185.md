## Summary

Keep open. Current `main` already has partial web-content hardening, so the issue’s problem statement is partly outdated, but the specific requested feature is not implemented as proposed. `web_fetch` sanitizes HTML into readable text, removes many hidden/offscreen elements, strips invisible Unicode, and wraps output as untrusted external content. Separately, external-content wrapping sanitizes spoofed boundary markers and common LLM special tokens for both `web_fetch` and `web_search`. However, there is no in-repo `content-sanitizer.py`, no generic tool-output preprocessing hook, and no `threat_level`/block-or-translate pipeline described in the issue. That is not strong enough evidence for any close reason.
