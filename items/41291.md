## Summary

Current `main` already bounds the relevant agent retry paths and surfaces terminal errors instead of allowing open-ended retry loops after failed tool/use-model transitions; the same guards are present in shipped release `v2026.4.22`.

## Evidence

- **Planning-only retry is explicitly bounded:** The embedded runner now limits planning-only retries to 1 by default (2 for strict-agentic), which directly covers the reported repeated `I'll ... next` behavior after a single tool action.
  - file: [src/agents/pi-embedded-runner/run/incomplete-turn.ts:80](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/agents/pi-embedded-runner/run/incomplete-turn.ts#L80)
  - command: `nl -ba src/agents/pi-embedded-runner/run/incomplete-turn.ts | sed -n '74,90p'`
- **Regression test matches the reported symptom shape:** There is a test for a `read` tool call followed by `I'll analyze the structure next.`, and it routes that into the bounded planning-only retry path rather than an open-ended loop.
  - file: [src/agents/pi-embedded-runner/run.incomplete-turn.test.ts:1040](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/agents/pi-embedded-runner/run.incomplete-turn.test.ts#L1040)
  - command: `nl -ba src/agents/pi-embedded-runner/run.incomplete-turn.test.ts | sed -n '1028,1050p'`

  - file: [src/agents/pi-embedded-runner/run.ts:703](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/agents/pi-embedded-runner/run.ts#L703)

## Close Comment

Closing this as implemented on current `main`. Codex review checked the current runner, tests, docs, and the shipped `v2026.4.22` release.
