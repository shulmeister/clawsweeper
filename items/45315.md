## Summary

Keep open. Current `main` still has the fast-abort/template-placeholder bug this PR targets, and the proposed fix is not implemented on `main`. The review comments identify follow-up accuracy gaps for session/channel overrides and catalog-backed thinking defaults, but those do not make the underlying bug obsolete or already fixed.

## Evidence

  - file: [src/channels/reply-prefix.ts:31](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/channels/reply-prefix.ts#L31)

That path does not itself populate model/thinking context before sending the abort message.
  - file: [src/auto-reply/reply/dispatch-from-config.ts:643](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/auto-reply/reply/dispatch-from-config.ts#L643)
