## Summary

Keep open. Current main still preserves existing provider apiKey/baseUrl from agent models.json in merge mode, so the broader Onboard overwrite/persistence request remains unresolved. The issue discussion already says #39103 fixed only a narrower stale baseUrl precedence bug and intentionally kept this issue open for the broader path.

## Best Possible Solution

Keep the issue open and implement an explicit Onboard update path: when a user edits an existing provider, persist the new apiKey/baseUrl and make models-json regeneration honor that update instead of preserving stale merge-mode runtime fields. Cover same-provider same-API edits, SecretRef-managed credentials, auth-profiles interaction, and the documented replace/merge behavior in tests/docs.
