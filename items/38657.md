## Summary

Not safe to close. Current main has the newer onboard/provider patch path that overwrites provider config in `openclaw.json` and upserts auth profiles, but the `models.json` merge path still explicitly preserves an existing provider `baseUrl`, and current tests/docs still assert that behavior. That leaves the reported stale-provider overwrite problem at least partially plausible on current `main`.
