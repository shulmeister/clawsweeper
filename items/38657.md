## Summary

Keep open. Current main still does not implement the broader Onboard overwrite/persistence behavior reported in #38657: the Onboard custom-provider path can build a replacement provider entry, but the downstream merge-mode models.json planner still preserves existing runtime apiKey/baseUrl values in common same-API-surface cases. The provided maintainer discussion also explicitly says #39103 fixed only the narrower stale baseUrl precedence bug and kept this issue open for the broader config/state overwrite path.

## Best Possible Solution

Keep #38657 open for a focused fix to the Onboard/configure persistence path. The best implementation would make explicit user-supplied provider edits overwrite stale runtime provider state for apiKey/baseUrl and any auth profile state that participates in resolution, while preserving existing fallback behavior only when the new config omits those fields. Add regression coverage that follows the Onboard save path into models.json/auth-profiles.json planning for same-provider same-API-surface updates.
