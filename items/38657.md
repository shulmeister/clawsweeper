## Summary

Current main still preserves existing generated provider state over explicit same-provider edits in merge mode. The narrower stale baseUrl/API-surface case from #39103 was addressed, but #38657’s broader Onboard overwrite/persistence problem remains supported by current source, tests, docs, and the maintainer discussion.

## Best Possible Solution

Keep #38657 open. The best fix is an explicit provider update path for Onboard/configure/model-auth that distinguishes deliberate same-provider edits from omitted fields: new apiKey and baseUrl values should persist through openclaw.json, regenerated agents/*/agent/models.json, and auth-profile handling, while preserving existing runtime values only when the new config truly omits those fields or SecretRef/auth-profile ownership requires it. Add regression coverage for Moonshot .ai/.cn base URL edits, same-API-surface custom baseUrl changes, plaintext key replacement, SecretRef-managed credentials, and auth-profile precedence.
