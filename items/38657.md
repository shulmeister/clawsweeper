## Summary

Current main still has the reported same-provider overwrite gap. Onboard/custom config writes the new provider `baseUrl` and `apiKey` into config, but merge-mode `models.json` regeneration still preserves existing non-empty runtime `apiKey` and `baseUrl` for matching providers, so a deliberate Onboard edit can be masked by stale agent state. The issue discussion also explicitly kept this broader overwrite/persistence issue open after the narrower #39103 baseUrl precedence fix.

## Best Possible Solution

Keep this issue open. The best fix is an explicit Onboard/provider-auth update path where a deliberate same-provider edit persists new `apiKey` and `baseUrl` through `openclaw.json`, agent `models.json` regeneration, and auth-profile handling, while preserving existing runtime values only when the new config truly omits those fields. Add regression coverage for same-provider plaintext edits, SecretRef-managed credentials, baseUrl changes with the same API surface, and auth-profiles interactions.
