## Summary

Keep open. Current main still routes pre-prompt compaction token accounting through the pinned pi-coding-agent estimator, and that estimator still has unchecked `.length` reads for the malformed message shapes described here. The provided GitHub context also shows #63636 is open, unmerged, and uses closing syntax for this issue, so the issue should stay open until that PR is reviewed and merged or superseded. The prior comments are useful evidence: jokemanfire described the same null-guard fix locally, and cygnostik linked #64053 as a likely Telegram manifestation of the same malformed-history/token-estimation family.

## Best Possible Solution

Keep this issue open and review/land or close #63636. The durable fix should add a safe local token estimator or normalize all malformed replay shapes before estimation, covering missing/non-array assistant content, missing `text`/`thinking`/`name`, non-array toolResult/custom content, and missing bash/summary fields with regression tests in the compaction and preemptive-compaction surfaces. After that lands on main, close this issue as implemented.
