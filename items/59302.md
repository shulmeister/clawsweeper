## Summary

Current `main` already implements cron restart catch-up for recurring jobs, including stale-running recovery and missed-slot replay on gateway startup, with regression coverage matching the report.

## Evidence

- **Startup clears stale running markers and triggers catch-up:** `start()` clears persisted `runningAtMs` markers, skips replay only for interrupted one-shot jobs, and then calls `runMissedJobs(...)` during gateway startup.
  - file: [src/cron/service/ops.ts:87](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/cron/service/ops.ts#L87)

  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Missed-slot replay no longer depends only on nextRunAtMs:** Startup catch-up treats a cron job as runnable either when `nextRunAtMs` is already due or when the most recent scheduled cron slot is newer than `lastRunAtMs`, which covers the issue's case where persisted state already advanced to a future slot after restart.
  - file: [src/cron/service/timer.ts:894](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/cron/service/timer.ts#L894)
