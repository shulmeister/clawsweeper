## Summary

Current main already implements the PR's central ACP transcript persistence behavior with a different, stronger path: ACP dispatch persists the canonical session turn after runTurn, resolves the transcript store from the session key owner, and existing tests cover the persistence path. The PR patch is obsolete and its bot-review concerns are superseded by main's implementation.

## Best Possible Solution

Close this PR as implemented on current main and keep the current persistAcpDispatchTranscript-based implementation. It solves the transcript persistence problem while avoiding the PR's catch-all error transcript append and runtime-agent/store mismatch concerns; any remaining smoke wording preference should be a separate narrow docs/skill change.
