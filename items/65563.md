## Summary

This issue should stay open. Current main still has the legacy `flow_runs.owner_session_key` migration shape described by the reporter: it copies legacy values into `owner_key`, but new inserts omit `owner_session_key`, so an existing table that still has `owner_session_key TEXT NOT NULL` can still reject `create_flow`. There is also an open paired PR, #65767, from the same author, so the best maintainer path is to review that implementation candidate rather than close the issue now.

## Best Possible Solution

Keep the issue open and review #65767, or an equivalent maintainer patch, to rebuild/migrate legacy `flow_runs` tables so `owner_session_key` no longer blocks inserts, preserve existing owner data in `owner_key`, and add a focused regression test that exercises `create_flow` or `createManagedTaskFlow` against the legacy sqlite schema.
