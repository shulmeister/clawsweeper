## Summary

Keep open. Current `main` still appears to have the reported migrated-schema bug: the webhooks plugin binds `create_flow` to the configured session and creates a managed TaskFlow, but the SQLite flow store only inserts `owner_key` while the migration logic merely copies from legacy `owner_session_key` and does not rebuild/drop that old column. On databases that still have `flow_runs.owner_session_key TEXT NOT NULL`, inserts can still fail exactly as reported. The linked PR #65767 also suggests active fix work rather than an obsolete report.
