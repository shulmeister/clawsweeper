## Summary

Keep open. The issue reports that enabling elevated makes non-elevated exec route to the gateway by default; current main still falls back to elevated level "on" when elevated is allowed, passes that into exec defaults, and treats the default as an elevated request. The linked PR #46908 claimed to fix this by defaulting to "off", but the provided context shows it was closed unmerged, and current main still matches the reported behavior.
