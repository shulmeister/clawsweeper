## Summary

Keep #68344 open. Current main resolves the synthetic local Ollama auth marker path, but it still does not parse Modelfile `PARAMETER num_ctx` from Ollama `/api/show` responses and still does not auto-mark models as `compat.supportsTools: false` when Ollama reports capabilities without `tools`. The linked PR #68349 described the right fix but is closed and unmerged; PR #69464 is still open and covers a narrower configured-params context issue, not the full reported local Ollama workflow.

## Best Possible Solution

Keep the issue open or retitle/split it around the remaining actionable work: teach `extensions/ollama/src/provider-models.ts` to parse `/api/show.parameters` for positive last-wins `num_ctx`, set `compat.supportsTools: false` when a non-empty Ollama capabilities array omits `tools`, add focused tests in `extensions/ollama/src/provider-models.test.ts`, and update the public Ollama docs to clarify local auth is synthetic/optional while documenting the automatic context/tool behavior once shipped.
