## Summary

Keep open. Current `main` still lacks the two substantive fixes from the report: Ollama `/api/show` parsing does not honor Modelfile `PARAMETER num_ctx` overrides, and discovered Ollama models are not marked with `compat.supportsTools: false` when `/api/show` omits `tools`. The API-key part is already handled as a synthetic local marker and documented as a placeholder, but that only resolves one sub-issue.

## Evidence

- **`num_ctx` override still not implemented on `main`:** `queryOllamaModelShowInfo` only reads `model_info[*].context_length` and `capabilities` from `/api/show`. It does not read or parse a `parameters` field, so a Modelfile `PARAMETER num_ctx 32768` override would still be ignored by discovery.
  - file: [extensions/ollama/src/provider-models.ts:119](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/extensions/ollama/src/provider-models.ts#L119)
  - command: `nl -ba extensions/ollama/src/provider-models.ts | sed -n '119,150p'`
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Tool-support fallback wiring is still missing:** Discovered Ollama models are built without any `compat` block; only `id`, `name`, `reasoning`, `input`, `cost`, `contextWindow`, and `maxTokens` are set. Separately, core tool gating defaults to tool support unless `compat.supportsTools === false`, so current discovery does not express the 'no tools' capability state needed for fallback behavior.
  - file: [extensions/ollama/src/provider-models.ts:214](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/extensions/ollama/src/provider-models.ts#L214)

- **The API-key complaint is mostly doc/UX, not a remaining core bug:** Local Ollama discovery already auto-injects the synthetic `ollama-local` marker when no real key is configured, and there is a regression test covering auto-registration from a local Ollama instance. The user-facing Ollama docs also state that any placeholder value works for local-only mode.
