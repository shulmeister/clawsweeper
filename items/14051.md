## Summary

Current main still does not implement activity-based heartbeat idle pausing. The heartbeat config and strict runtime schema expose interval, active-hours, delivery/session/context, timeout, and reasoning options, but no `idleTimeoutMinutes` or equivalent recent-activity window. The runner and scheduler remain interval/active-hours/queue/preflight gated, while existing `lastInteractionAt` tracking is for session lifecycle freshness and is not read by the heartbeat scheduler. The prior issue discussion already kept this open, and the adjacent linked issues are related but not canonical replacements.

## Best Possible Solution

Keep #14051 open. A proper fix should add a product-approved heartbeat idle-timeout config key to exported types, strict runtime schema, schema help/generated docs, config-honor inventory, user docs, and focused tests. Runtime should track or reuse last allowed inbound user activity at the correct agent/session/channel boundary, then gate heartbeat runs or scheduling on that activity window while preserving existing active-hours, retry, targeted wake, task wake, light-context, and isolated-session behavior.
