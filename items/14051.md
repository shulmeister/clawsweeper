## Summary

Keep open. Current main still does not implement activity-based heartbeat pausing: the heartbeat config/schema/docs do not expose `idleTimeoutMinutes` or an equivalent activity window, and the runner/scheduler remains interval, active-hours, and queue gated rather than gated on recent allowed inbound activity. The issue is concrete and the related open items are adjacent, not canonical replacements.

## Best Possible Solution

Keep #14051 open. A proper fix should add a product-approved heartbeat idle-timeout config key to the exported types, strict runtime schema, schema help/labels/generated docs, docs, config-honor inventory, and tests. Runtime should track or reuse last allowed inbound user activity at the correct agent/session/channel boundary, gate heartbeat runs/scheduling on that window, and define interactions with `activeHours`, requests-in-flight retry behavior, manual/system wakes, task-triggered heartbeats, light context, and isolated sessions.
