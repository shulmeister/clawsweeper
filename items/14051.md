## Summary

Keep open. Current main still lacks activity-based heartbeat pausing: the heartbeat config and strict runtime schema do not expose `idleTimeoutMinutes` or an equivalent activity window, and the runner remains interval/active-hours/queue-gated rather than gated on recent allowed-sender activity. The related open items are adjacent: #40611 names this issue as an alternative solution for heartbeat contention, and #46844 is a Talk Mode idle-timeout request, not a canonical duplicate for heartbeat scheduling.

## Best Possible Solution

Keep #14051 open. A proper fix should add a product-approved heartbeat idle-timeout config key to the type/schema/help/generated docs path, track last allowed inbound activity at the right agent/session/channel boundary, and gate heartbeat scheduling/runs so heartbeats resume on user activity and pause after the configured inactivity window. It should also define interaction with `activeHours`, `requests-in-flight` retry behavior from #40611, manual wakes, task-triggered heartbeats, and isolated/light-context token-saving modes.
