## Summary

Keep #14051 open. Current main still does not implement activity-based heartbeat idle pausing: heartbeat config/types/schema/docs/tests expose interval, active-hours, delivery, timeout, light-context, and isolation options, but no idle-timeout or idle-only activity gate. The heartbeat runner still schedules by interval and only skips for disabled, quiet-hours, and busy queue/session lanes, not for lack of recent allowed inbound activity. The related #59567 was closed as a duplicate in favor of this tracker, so it does not supersede it.

## Best Possible Solution

Keep #14051 open as the canonical product/API tracker. A proper fix should add a maintainer-approved heartbeat idle-timeout or idle-only config key to exported types, strict schema, generated config help/docs, config-honor inventory, and focused tests. Runtime should gate heartbeat scheduling or execution on recent allowed inbound activity at the correct agent/session/channel boundary while preserving existing interval, active-hours, targeted wake, task wake, requests-in-flight retry, light-context, and isolated-session behavior.
