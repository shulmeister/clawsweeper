## Summary

Keep open. Current `main` still implements heartbeat as a fixed-interval scheduler with optional `activeHours` gating; it does not expose or enforce an activity-based idle timeout like `idleTimeoutMinutes`.

## Evidence

- **Heartbeat config surface lacks idle-timeout field:** The public heartbeat config type lists `every`, `activeHours`, delivery/prompt options, `lightContext`, `isolatedSession`, and `includeReasoning`, but no `idleTimeoutMinutes` or similar activity-window field.
  - file: [src/config/types.agent-defaults.ts:331](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/config/types.agent-defaults.ts#L331)
  - command: `nl -ba src/config/types.agent-defaults.ts | sed -n '331,385p'`
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **Schema would reject the proposed key:** `HeartbeatSchema` is a strict object containing `every`, `activeHours`, `model`, `session`, `target`, `directPolicy`, `to`, `accountId`, `prompt`, `ackMaxChars`, `suppressToolErrorWarnings`, `timeoutSeconds`, `lightContext`, `isolatedSession`, and `includeReasoning`. There is no activity-based idle-timeout field.
  - file: [src/config/zod-schema.agent-runtime.ts:19](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/config/zod-schema.agent-runtime.ts#L19)
  - command: `nl -ba src/config/zod-schema.agent-runtime.ts | sed -n '19,50p'`
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **Runtime gating is not activity-based:** `runHeartbeatOnce()` checks whether heartbeats are enabled, whether the interval resolves, whether the current time is within `activeHours`, and whether queues are busy. There is no check against recent inbound activity, sender activity, `lastActivityAt`, or a configurable idle window before deciding to run.
  - file: [src/infra/heartbeat-runner.ts:743](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/infra/heartbeat-runner.ts#L743)

- **Docs describe active-hours, not idle-after-activity behavior:** Heartbeat docs present `activeHours` as the time-based control and say heartbeats are skipped only outside that fixed window. They also note heartbeat replies restore `updatedAt` so the session idle-expiry behavior is preserved, which is different from pausing scheduled heartbeats after inactivity.
