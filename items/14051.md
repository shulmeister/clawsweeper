## Summary

Keep #14051 open. Current main still has interval/active-hours heartbeats and session idle metadata, but no heartbeat config key or runtime path that starts heartbeats on allowed inbound activity and pauses them after an idle window. The existing discussion also keeps this as the canonical tracker: #59567 is closed as a duplicate in favor of #14051, and #40611 references this issue as the more complete fix.

## Best Possible Solution

Keep #14051 open as the canonical heartbeat idle-timeout tracker. A proper fix should add a maintainer-approved heartbeat activity/idle config key to exported types, strict schema, schema help/generated docs, config-honor inventory, and https://docs.openclaw.ai/gateway/heartbeat. Runtime should gate heartbeat scheduling or execution on recent allowed inbound user/channel activity at the correct agent/session/channel boundary, likely using lastInteractionAt or a dedicated activity ledger, while preserving existing interval, activeHours, wake retry, requests-in-flight handling, lightContext, isolatedSession, delivery, and per-agent merge behavior.
