## Summary

Keep #15634 open. Current main has shipped major Mattermost parity pieces, including reactions, interactive buttons, threading, markdown-aware delivery, media upload/handling, and typing indicators, but the read receipts / presence portion of the umbrella request is still absent from source, config, docs, and tests. The latest provided issue discussion also explicitly kept the umbrella open for that remaining gap.

## Best Possible Solution

Keep #15634 open as the Mattermost parity umbrella, or split the remaining read receipts / presence work into a narrower Mattermost issue and link it back. Do not close this item until that remaining scope is implemented, documented, or explicitly descoped by a maintainer.
