## Summary

Keep #15634 open. Current main has implemented much of the Mattermost parity umbrella, including bundled-plugin docs, markdown-preserving delivery, reactions, threading, typing indicators, media, and interactive buttons. The remaining read receipts / presence scope is still not implemented or documented for Mattermost, and the existing issue discussion already identifies that unresolved remainder.

## Best Possible Solution

Keep #15634 open as the Mattermost parity umbrella for now, or split the unresolved read receipts / presence work into a narrower canonical Mattermost issue and close this umbrella only after that remaining scope is implemented, documented, or explicitly descoped. Treat PR #19957 as canonical evidence for the completed rich-interactions/buttons portion.
