## Summary

Current main already places telephony in plugin space, and the SDK exposes the hooks/routes/tools/realtime surfaces needed for outbound task-call orchestration. This request is better tracked as a ClawHub/plugin enhancement, not an OpenClaw core issue.

## Evidence

- **Vision is plugin-first, not core-first:** OpenClaw’s vision says optional capability should usually ship as plugins, plugin promotion belongs in ClawHub, and new core skills that can live on ClawHub are out of scope.

  - file: [docs/plugins/building-plugins.md:11](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/docs/plugins/building-plugins.md#L11)

## Risks / Open Questions

- There is not an already-linked ClawHub plugin in the thread yet, so closing this core issue does not mean the feature is implemented.
- If maintainers want this in the official `@openclaw/voice-call` plugin rather than a third-party plugin, that should be tracked in the plugin’s own scope, not as a core feature request.
