## Summary

Keep open. Current main has a real Control UI/Gateway image attachment path and several adjacent WebChat image fixes, but I did not find code, tests, changelog, or related-item evidence that specifically resolves the reported iPad/Safari/Tailscale `chat.send` failure after `Image resized to fit limits`. ## Best Possible Solution

Keep this issue open. The best maintainer path is to reproduce on current main with Control UI in iPad Safari over Tailscale, or add a targeted WebChat image regression that drives an image through `chat.send`, attachment parsing, agent image sanitization/resizing, and the selected runner.
