## Summary

Keep #7281 open. Current main still has no configurable `fallbackChannel`/`fallbackChannels` support for hook mappings: the hook config/schema accepts one `channel` and one `to`, mapping resolution preserves only that primary target, hook dispatch creates a single cron delivery target, and cron delivery sends to that resolved target without an ordered fallback chain.

## Best Possible Solution

Keep #7281 open until maintainers either implement configurable hook delivery fallback chains or explicitly reject the product behavior. A complete fix should add typed `fallbackChannel` or `fallbackChannels` config and validation, propagate the fallback chain through hook mapping resolution and `dispatchAgentHook` into cron/direct delivery, retry ordered alternate targets only after primary delivery failure, keep it distinct from `channel: "last"` and `failureDestination`, document it on the public configuration reference, and add focused regression tests.
