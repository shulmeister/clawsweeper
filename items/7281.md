## Summary

Keep open. Current main does not implement hook-mapping fallbackChannel/fallbackChannels, and the existing delivery fallback behavior is either "last" target resolution or best-effort/session-only handling, not an alternate-channel retry chain. The related #54029 cross-reference is useful context for channel-unavailable failures, but it tracks Slack outbound degradation/logging rather than superseding this hook mapping feature request.
