## Summary

Keep #7281 open. Current main still has no configurable `fallbackChannel`/`fallbackChannels` hook mapping support: hook config/schema, hook mapping resolution, hook dispatch, cron delivery planning, and direct delivery dispatch all carry one primary delivery target only. The provided maintainer discussion also already kept this open and distinguished the adjacent Slack availability report #54029 from this requested fallback-chain feature.

## Best Possible Solution

Keep #7281 open until maintainers either implement configurable hook delivery fallback chains or explicitly decide against the product behavior. A complete implementation would add typed `fallbackChannel`/`fallbackChannels` config and validation, propagate it through hook mapping resolution and `dispatchAgentHook` into cron/direct delivery, retry ordered alternate targets only after primary delivery failure, keep it distinct from `channel: "last"` and `failureDestination`, document it at https://docs.openclaw.ai/gateway/configuration-reference, and add focused regression tests.
