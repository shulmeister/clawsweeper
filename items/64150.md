## Summary

Keep this PR open. Current main still does not implement the central OAuth `Glob` -> local `find` alias or register `find`/`grep`/`ls` in OpenClaw’s coding tool factory, so the PR is not obsolete. However, the exact patch also has real unresolved safety/policy concerns from review: raw filesystem discovery tools need workspace/sandbox guards and catalog/group policy coverage before merge.

## Best Possible Solution

Keep this PR open as a real but incomplete implementation candidate. The best path is a focused revision or successor PR that registers `find`/`grep`/`ls` through guarded OpenClaw wrappers, adds them to the core filesystem catalog/profile/group policy model, adds tests for `Glob` -> `find`, `Grep` -> `grep`, and `LS` -> `ls` under Anthropic OAuth, and covers `tools.fs.workspaceOnly` plus `group:fs` allow/deny behavior.
