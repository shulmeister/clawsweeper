## Summary

Current `main` already hot-applies model-catalog changes: the gateway clears the cached model catalog on `models`/`agents.defaults.model[s]` reloads, the next catalog load regenerates `models.json`, regression tests cover newly added models becoming visible after hot reload, and the same handler logic is present in release `v2026.4.22`.

## Evidence

- **Hot reload invalidates model catalog cache:** `applyHotReload` resets the model-catalog cache when changed paths touch `models`, `models.*`, `agents.defaults.model`, or `agents.defaults.models`, which is the missing step described in the issue.
  - file: [src/gateway/server-reload-handlers.ts:107](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/src/gateway/server-reload-handlers.ts#L107)
  - command: `nl -ba src/gateway/server-reload-handlers.ts | sed -n '101,121p'`
  - sha: [320d52a23e41](https://github.com/openclaw/openclaw/commit/320d52a23e41a326287826219a20f1f29f334554)
- **Next catalog read regenerates `models.json`:** `loadModelCatalog()` calls `ensureOpenClawModelsJson(cfg)` before loading the registry, so once the cache is cleared the next model-menu/catalog read rebuilds `models.json` from current config.
  - file: [src/agents/model-catalog.ts:81](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/src/agents/model-catalog.ts#L81)
