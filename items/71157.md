## Summary

Keep this issue open. Current main still has no WhatsApp channel/account disappearing-message expiration setting, no resolved account state for an effective expiration, and no outbound send option path that would pass Baileys `ephemeralExpiration` consistently across text, quoted replies, media, voice-note companion text, payload/chunked sends, or polls. The provided related reports cover mentions, quote threading, and delivery, not this expiration feature.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin. Add channel-level and account-level `disappearingMessagesSeconds` to config types, strict Zod schema, generated metadata/help, and docs; resolve a positive effective value with account override over channel default and `0`/`null`/omitted meaning no OpenClaw override; thread that value through active send and poll options; merge it into Baileys message generation options for normal text, quoted replies, media, voice-note companion text, payload/chunk/fallback sends, and polls; add targeted tests for configured, account override, unset, quoted, media/voice-note, payload/chunked, and poll paths.
