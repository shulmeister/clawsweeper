## Summary

Keep #71157 open. Current main has a bundled WhatsApp channel and Baileys supports an `ephemeralExpiration` send option, but OpenClaw does not expose the requested `channels.whatsapp.disappearingMessagesSeconds` config, the strict WhatsApp schema would reject it, and the outbound text/media/poll send paths never pass expiration metadata to Baileys. The related items in the provided context cover quote replies, mentions, and delivery reliability, not disappearing-message expiration.

## Best Possible Solution

Keep the issue open and implement it inside the bundled WhatsApp plugin. Add a typed `disappearingMessagesSeconds?: number` field to WhatsApp shared/account config, schema, generated metadata/docs, and account resolution; resolve account-level over channel-level values; extend `ActiveWebSendOptions` and the Baileys send wrapper to merge `ephemeralExpiration` with quoted options; apply it to text, media, visible text after voice notes, polls, channel/tool sends, and quoted replies; add focused tests for channel default, account override, unset/zero behavior, and quoted/media/poll paths.
