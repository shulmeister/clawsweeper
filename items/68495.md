## Summary

This PR proposes optional script-backed skill slash commands as a new core frontmatter dispatch mode. Current main already has plugin command and plugin tool surfaces that can provide deterministic, no-LLM slash commands, while the project direction keeps optional skills/plugins in ClawHub rather than core. The PR discussion also flags a real shell-injection risk in the proposed direct shell path, so the best cleanup is to close this core PR and move concrete /ctx, /ping, or /version helpers into a plugin/ClawHub package.

## Best Possible Solution

Close this core PR. Keep the existing plugin command and skill tool-dispatch surfaces as the product boundary, and publish concrete script-backed helpers as a ClawHub plugin that owns its commands/tools, timeout/stdout limits, and shell safety. If that work exposes a specific missing plugin API, open a focused plugin API design issue instead of adding command-dispatch: exec directly to core skills.
