## Summary

PR #68495 adds a new core skill frontmatter mode for direct shell execution, but current main already has supported no-LLM plugin command and skill tool-dispatch surfaces, and VISION.md directs optional skills/plugins through ClawHub. The PR discussion also identified a shell-injection risk in the proposed `sh -c` path, so the best maintainer cleanup is to close this core PR and move concrete `/ctx`, `/ping`, or `/version` helpers to a ClawHub plugin.

## Best Possible Solution

Close this core PR. Keep current main's supported no-LLM paths: plugin `api.registerCommand(...)` for command-owned behavior and skill `command-dispatch: tool` for deterministic skill-to-tool dispatch. Publish concrete helpers such as `/ctx`, `/ping`, or `/version` as a ClawHub plugin; if that work exposes a specific missing plugin API, open a focused plugin API issue instead of adding generic `command-dispatch: exec` to core skills.
