## Summary

Keep open. Current `main` still does not implement this PR's requested behavior: web UI chat rendering does not extract `{ type: "image", data, mimeType }` tool-result image blocks, still hides tool-message images inside the collapsed tool-output disclosure, and still only reads top-level tool-result text instead of nested `content` arrays.

## Evidence

- **Current image extraction still misses tool-result `{data,mimeType}` blocks:** `extractImages()` only handles `image.source.data`, `image.url`, `image_url`, and `input_image`; there is no branch for top-level `image.data`/`mimeType` payloads from tool results.
  - file: [ui/src/ui/chat/grouped-render.ts:131](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/ui/src/ui/chat/grouped-render.ts#L131)
  - command: `nl -ba ui/src/ui/chat/grouped-render.ts | sed -n '120,230p'`
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Tool-message images are still rendered only inside the expandable tool-output body:** For tool messages, `renderMessageImages(images, ...)` is inside the `toolMessageExpanded ? ... : nothing` branch, so images are hidden by default until the disclosure is opened.
  - file: [ui/src/ui/chat/grouped-render.ts:1425](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/ui/src/ui/chat/grouped-render.ts#L1425)

- **Nested tool-result text is still not extracted:** `extractToolText()` still returns only top-level `item.text` or string `item.content`; it does not recurse into nested tool-result `content` arrays, so caption/text paired with nested images would still be lost.
