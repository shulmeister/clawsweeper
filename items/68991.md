## Summary

Current main already prevents automatic fallback from becoming the next-turn selected model, but through the newer selected-model override architecture rather than this PR's `persisted*` API. Runtime `model`/`modelProvider` can still record what actually ran for accounting/status, while selected model state lives in `modelOverride`/`providerOverride`; auto-sourced fallback overrides are cleared before the next turn so the configured/user-selected primary is retried.

## Best Possible Solution

Close this PR as obsolete on current main. Keep the shipped selected/runtime split: runtime fields may reflect the model that actually served the turn, while `modelOverride`/`providerOverride` and auto-override cleanup control the next selected model. Any remaining cleanup should be handled as a narrow follow-up against that architecture, not by merging this older `persisted*` API branch.
