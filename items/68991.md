## Summary

Current main already fixes the sticky automatic fallback problem through the selected/runtime model split. Fallback runtime fields can still record what served the turn, while next-turn selection uses modelOverride/providerOverride, auto-sourced fallback overrides are cleared before retrying the primary, and tests/docs cover that behavior. PR #68991's persisted* API is now obsolete.

## Best Possible Solution

Close this PR as obsolete on current main. Keep the shipped selected/runtime split using modelOverride/providerOverride plus modelOverrideSource, where automatic fallback is transient and explicit user selections remain durable. Any remaining cleanup should be a narrow follow-up against that architecture, not a merge of this older persisted* API branch.
