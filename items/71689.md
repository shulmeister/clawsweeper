## Summary

Keep open. Current main still restores `tasks/runs.sqlite` directly through the SQLite store and only logs restore failures; there is no corruption detection, bad-DB snapshot, `.recover`-style rebuild, replacement, or operator event path. The issue is recent, concrete, non-maintainer-authored, has no protected labels, and the provided related items do not supersede this malformed-SQLite task registry failure.

## Best Possible Solution

Keep the issue open and implement a task-registry SQLite corruption recovery path in `src/tasks/task-registry.store.sqlite.ts`: detect malformed-image SQLite errors during open/load, close the DB, snapshot `runs.sqlite` and sidecars, attempt a safe recovery/rebuild or initialize a clean registry when recovery is impossible, emit an operator-visible event, and add focused tests with a deliberately malformed DB plus docs/troubleshooting notes.
