## Summary

Keep #14438 open. Current main still has no supported `openclaw plugins reload` or plugin cache-clear workflow, public docs explicitly tell native plugin authors to restart the live Gateway after plugin code/config changes, and the loader/Jiti cache helpers remain internal rather than a safe live re-registration path. The existing 2026-04-26 maintainer comment also explicitly says to keep this issue open.

## Best Possible Solution

Keep #14438 open as core plugin loader and Gateway lifecycle developer UX work. The likely path is a supported Gateway RPC plus `openclaw plugins reload <id>` and/or `openclaw plugins clear-cache` that invalidates loader/Jiti/module state, safely disposes plugin-owned registrations and services, refreshes tools/hooks/providers/runtime hooks, and documents Docker/container behavior. If hot reload is intentionally out of scope, update the public plugin and CLI docs with a clear supported restart/cache workflow instead.
