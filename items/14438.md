## Summary

Keep #14438 open. Current main still does not provide a supported plugin runtime hot-reload, file watcher, or user-facing jiti cache invalidation command. The code has internal loader/cache reset paths used for activation/tests/startup, but the documented and implemented developer path remains restarting the Gateway after plugin code/config changes. The existing 2026-04-26 maintainer-context Codex comment by steipete reached the same keep-open conclusion, and the newer current main SHA still matches that behavior.

## Best Possible Solution

Keep the issue open as core plugin loader/gateway lifecycle work. A complete solution should add an explicit developer reload path, such as a Gateway RPC plus `openclaw plugins reload <id>` or `openclaw plugins clear-cache`, that invalidates plugin registry/jiti/module state, safely clears plugin-owned registrations, reloads the selected plugin, updates the active registry/gateway methods, and documents container/wrapper process behavior. If maintainers decide native plugin code hot reload is intentionally unsupported, the issue should be closed only after that product decision is recorded clearly in docs and the CLI guidance.
