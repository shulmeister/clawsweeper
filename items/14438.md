## Summary

Current main has a repo-dev `gateway:watch` loop for bundled code, but it does not implement the requested installed-plugin hot-reload / jiti-cache invalidation workflow. There is still no `openclaw plugins reload` or `clear-cache` CLI, and current plugin management commands still tell users to restart the gateway to apply plugin changes.

## Evidence

- **No plugin reload/clear-cache CLI exists:** The `plugins` CLI registers `list`, `inspect`, `enable`, `disable`, `uninstall`, `install`, `update`, `doctor`, and `marketplace list`; there is no reload or cache-clearing command.
  - file: [src/cli/plugins-cli.ts:139](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/cli/plugins-cli.ts#L139)
