## Summary

Keep #11414 open. Current main still only implements Telegram forum topic create/edit through the bundled Telegram message action surface; the requested close, reopen, and delete topic lifecycle actions are absent from the generic action enum, Telegram adapter, runtime aliases/handlers, send helpers, config schema, docs, and contract tests. The issue is also marked as the dedupe parent, and the related open PR #41410 only covers deletion, not close/reopen.

## Best Possible Solution

Keep this dedupe-parent issue open until the bundled Telegram plugin supports the remaining forum-topic lifecycle actions. A complete fix likely belongs in the existing Telegram plugin/action path: add generic action names such as topic-close, topic-reopen, and topic-delete; map them in extensions/telegram/src/channel-actions.ts; implement Telegram runtime/send helpers for closeForumTopic, reopenForumTopic, and deleteForumTopic; add config/schema/docs coverage; and extend Telegram action contract/runtime tests. PR #41410 may help with deletion, but close and reopen still need implementation.
