## Summary

Current `main` only implements Telegram forum-topic create/edit support. The requested close/reopen/delete topic-management actions are still absent from the message-tool surface, runtime handlers, config, and docs, so the issue should remain open.

## Evidence

- **Message action names stop at create/edit:** The shared channel message action list includes `topic-create` and `topic-edit`, but there are no `topic-close`, `topic-reopen`, or `topic-delete` actions.
  - file: [src/channels/plugins/message-action-names.ts:46](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/channels/plugins/message-action-names.ts#L46)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Telegram adapter only maps create/edit topic actions:** Telegram's message-action map exposes `topic-create` -> `createForumTopic` and `topic-edit` -> `editForumTopic`; no close/reopen/delete aliases are defined.
  - file: [extensions/telegram/src/channel-actions.ts:48](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/extensions/telegram/src/channel-actions.ts#L48)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Runtime only handles create/edit forum-topic methods:** `executeTelegramAction` has explicit branches for `createForumTopic` and `editForumTopic`, then falls through to `Unsupported Telegram action`, so close/reopen/delete are not implemented here.
  - file: [extensions/telegram/src/action-runtime.ts:598](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/extensions/telegram/src/action-runtime.ts#L598)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Telegram send/runtime helpers only export create/edit helpers:** The public Telegram runtime API exports `createForumTopicTelegram`, `editForumTopicTelegram`, and `renameForumTopicTelegram`; there are no helpers for close/reopen/delete forum topics.
  - file: [extensions/telegram/runtime-api.ts:65](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/extensions/telegram/runtime-api.ts#L65)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Repo-wide search found no close/reopen/delete topic support:** A read-only search for `closeForumTopic|reopenForumTopic|deleteForumTopic` across `extensions/telegram`, `src`, and `docs` returned no matches.
  - command: `rg -n "closeForumTopic|reopenForumTopic|deleteForumTopic" extensions/telegram src docs`
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)

## Risks / Open Questions

- The issue body is partially obsolete because `createForumTopic` and `editForumTopic` already exist on `main`; the remaining gap is specifically close/reopen/delete support.
- Telegram docs appear slightly behind the code: `docs/channels/telegram.md` documents `createForumTopic` and `topic-create`, but not the existing `editForumTopic`/`topic-edit` surface.
