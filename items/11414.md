## Summary

Keep #11414 open. Current main partially supports Telegram forum topics through create/edit, but the requested close, reopen, and delete forum-topic actions are still absent from the message action surface, Telegram adapter, runtime handlers, config gates, docs, and tests. The issue is also the dedupe parent; the related reports in the provided context are closed children/related items, not an open canonical replacement.

## Best Possible Solution

Keep this dedupe-parent issue open until the bundled Telegram plugin supports the remaining forum-topic lifecycle actions. The likely implementation path is to add generic action names such as `topic-close`, `topic-reopen`, and `topic-delete`, map them in `extensions/telegram/src/channel-actions.ts`, implement Telegram runtime/send helpers for `closeForumTopic`, `reopenForumTopic`, and `deleteForumTopic`, add config/schema/docs coverage, and extend the Telegram action contract and runtime tests. Related PR #41410 may be useful for the delete portion, but close and reopen still need coverage.
