## Summary

Keep #11414 open. Current main has Telegram forum topic create/edit support, but the requested close, reopen, and delete lifecycle actions are still missing from the shared action surface, Telegram discovery/action mapping, runtime dispatch, transport exports, config schema, docs, and contract tests. The linked PR #41410 is useful deletion-only prior art, but it is open/unmerged and does not cover close or reopen.

## Best Possible Solution

Keep #11414 open as the canonical Telegram forum topic lifecycle tracker. A complete fix should add close, reopen, and delete support through the shared message action contract, Telegram action discovery/map, runtime dispatch, transport helpers, config schema/metadata, public docs, and focused Telegram contract/runtime/transport tests. PR #41410 can be linked or expanded for the delete portion, but it does not replace the parent issue.
