## Summary

Keep this issue open. Current main still has no typed or validated per-agent `delivery.defaultTarget` / `delivery.defaultChannel` contract, and the message tool still infers channel/target only from explicit tool arguments or trusted run/tool context. The related cron-default and announce-model reports are narrower; they do not supersede this agent-level cross-session delivery-default request.

## Best Possible Solution

Keep this issue open for a core product/design fix. The likely path is a typed, documented per-agent delivery default contract, wired into trusted tool-context construction and message-tool channel/target inference for programmatic sessions, with tests for `sessions_send`, webchat-routed sessions, cron-created sessions, multi-channel ambiguity, explicit override precedence, account/thread handling, and no-config backward compatibility.

## Risks / Open Questions

- A generic default delivery target can silently route messages to the wrong chat if precedence and safety checks are underspecified.
- A complete fix should preserve explicit tool args and per-run delivery overrides ahead of any agent default, and should account for accountId/threadId plus session sendPolicy and cross-context policy.
