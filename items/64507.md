## Summary

Keep #64507 open. Current main has partial delivery-context improvements, including preserving active chat context when an agent creates cron jobs, but it still does not implement the requested per-agent `delivery.defaultTarget` / `delivery.defaultChannel` fallback for cross-session or programmatic agent runs. The strict agent config schemas have no `delivery` field, and message sends still infer target/channel only from explicit args or the current session/tool context.

## Best Possible Solution

Keep the issue open for maintainer product/design judgment. A likely core fix would add a typed, documented per-agent delivery default contract, wire it into trusted tool-context construction and message-tool channel/target inference for programmatic sessions, and cover `sessions_send`, webchat-routed sessions, cron-created sessions, multi-channel ambiguity, account/thread handling, and no-config backward compatibility with focused tests.

## Risks / Open Questions
