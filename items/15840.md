## Summary

Keep #15840 open. Current main still has no opt-in config to suppress media placeholders, and the Signal/iMessage outbound paths still synthesize visible `<media:${kind}>` text for media-only sends. The request is concrete, still reproducible from source, and prior related PRs were closed unmerged rather than shipped.

## Best Possible Solution

Keep this issue open as the canonical product/config request. A conservative fix would add a documented default-off `messages` config option, validate it in the strict config schema/generated config metadata, thread it through the bundled Signal and iMessage outbound send paths that synthesize placeholders, and add focused regression tests proving default placeholder behavior is preserved while opt-in suppression sends media-only audio without visible `<media:audio>` text.
