## Summary

Keep #15840 open. Current main still has no `messages.suppressMediaPlaceholders` or granular `mediaPlaceholders` config contract, and bundled Signal/iMessage send paths still synthesize visible `<media:...>` fallback text for media-only sends. The existing maintainer-side issue comment also identifies this as the canonical cross-channel product/config request, while the related implementation PRs were closed unmerged.

## Best Possible Solution

Keep #15840 open as the canonical product/config request. A conservative fix should add a documented default-off `messages` option, update `MessagesConfig`, strict zod schema, generated config metadata/docs, and route the setting through the bundled Signal and iMessage media-only send paths while preserving current fallback behavior by default and adding focused tests for audio/media-only sends.
