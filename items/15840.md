## Summary

Keep #15840 open. Current main still has no opt-in config to suppress media placeholder text, and the Signal/iMessage outbound paths still synthesize visible `<media:audio>`/`<media:${kind}>` text for media-only sends. Related fix PRs were closed unmerged, and the latest issue discussion already identifies this as the canonical remaining product/config request.

## Best Possible Solution

Keep this issue open as the canonical product/config request. A conservative fix should add a documented default-off `messages` config option, validate it in the strict config schema and generated config metadata, thread it through bundled outbound send paths that synthesize media placeholders, and add focused tests proving default placeholder behavior is preserved while opt-in suppression sends media-only audio without visible `<media:audio>` text where the channel transport supports that.
