## Summary

Keep open. Current main still does not implement the requested opt-in suppression for media-only placeholder text. Signal and iMessage still synthesize visible `<media:${kind}>` text when outbound media is sent without a message, and the shared `messages` config schema/type still has no `suppressMediaPlaceholders` or granular `mediaPlaceholders` option. Telegram’s current outbound path appears to send empty-caption media without inventing this placeholder, so the remaining issue is narrower than the original examples but still real for bundled messaging channels.

## Best Possible Solution

Keep #15840 open until maintainers either implement an opt-in placeholder suppression policy or decide the product direction. A conservative implementation would add a documented, default-off `messages` config field, validate it in the strict config schema, thread it through the bundled channel outbound paths that synthesize visible placeholders, and add focused regression tests for Signal/iMessage media-only audio sends while preserving current default accessibility/fallback behavior.

That value is then placed in the Signal request params as the message body.
