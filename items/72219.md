## Summary

Keep #72219 open. Current main still writes a misleading `delivery.resolved` failure for bare `delivery.mode: "none"` cron runs, while the PR is a focused fix with a regression test for the linked open issue #72210.

## Best Possible Solution

Keep the PR open for maintainer review and land #72219 or an equivalent targeted patch that omits `delivery.resolved` only for intentionally disabled fallback delivery, while preserving existing message-tool target behavior and adding the regression test.
