## Summary

Current main already has post-conversation/session hook primitives and shipped examples that cover this request well enough: internal hooks, plugin `agent_end`/`session_end` lifecycle hooks, a bundled `session-memory` hook that writes transcript summaries to `memory/`, and a memory plugin that auto-captures from `agent_end`.

## Risks / Open Questions

## Close Comment

Closing from a Codex review because current `main` already implements the requested post-conversation/session hook mechanism. OpenClaw now has internal hooks plus plugin lifecycle hooks for `agent_end` and `session_end` (`src/plugins/hook-types.ts:390`, `docs/concepts/agent-loop.md:97`), and the gateway emits `session_end` with the end reason and archived transcript path (`src/gateway/session-reset-service.ts:107`). There is also a shipped example, `session-memory`, which reads recent transcript content and writes `memory/YYYY-MM-DD-slug.md` (`src/hooks/bundled/session-memory/handler.ts:53`), and `memory-lancedb` already auto-captures memories from `agent_end` (`extensions/memory-lancedb/index.ts:612`). That covers the core request well enough, so this can be closed as implemented on `main`.
