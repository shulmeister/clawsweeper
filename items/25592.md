## Summary

Keep #25592 open. Current main suppresses commentary/final_answer phase leaks and has optional final-tag enforcement plumbing, but it still allows ordinary unphased assistant text before or between tool calls to enter block-reply and final-payload delivery. The issue also has recent repro reports and an Apr 26 maintainer review explicitly keeping it as the broader canonical bug.

## Best Possible Solution

Keep #25592 open as the canonical product bug until OpenClaw implements or explicitly rejects a channel-safe delivery policy, such as final-only delivery for channel sessions, configurable enforceFinalTag for customer-facing channels, or an opt-in replyMode/tool-only explicit-send mode. Add regression coverage for ordinary unphased assistant text before/between tool calls in both block-reply delivery and final payload assembly.
