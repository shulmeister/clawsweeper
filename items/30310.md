## Summary

Current main still does not implement PR #30310's central behavior. `createAccountListHelpers().listAccountIds()` returns only configured account IDs when an `accounts` map exists, with `default` used only as the empty-list fallback. Gateway startup starts exactly the account IDs returned by that helper, and Slack/Discord/Zalo account resolution still supports top-level credential/config fallback for named accounts. The PR is therefore still a plausible focused channel startup fix, not conservative cleanup material.

## Best Possible Solution

Keep #30310 open. Maintainers should either land a token-aware account-listing fix with regression coverage in `src/channels/plugins/account-helpers.test.ts`, or make an explicit product decision that named `accounts` should suppress base/default startup and document that contract. The PR remains the canonical focused work item for this behavior until one of those paths happens.
