## Summary

Keep PR #30310 open. Current main still does not implement the PR's central startup behavior: the shared account helper lists only configured account IDs when an accounts map exists, so a base/default credential set is omitted whenever named accounts are present. Gateway startup consumes that list directly, and Slack/Discord/Zalo still have top-level credential fallback or merged base config semantics, so this remains a plausible focused channel startup fix rather than cleanup material.

## Best Possible Solution

Keep #30310 open as the canonical focused work item. Maintainers should either land a token-aware account-listing fix with regression coverage in src/channels/plugins/account-helpers.test.ts, or explicitly decide that named accounts suppress base/default startup and document that contract. Before landing, resolve the security/product question around implicit default startup and the remaining channel-specific edge cases from the review discussion.
