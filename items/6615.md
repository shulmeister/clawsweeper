## Summary

Keep #6615 open. Current main at 9089e6b595ba33b4afc27b99c56ace80c4fbbc7b still does not implement exec-approval denylist or `ask: "on-match"` behavior. The issue discussion links useful prior art, including SafeClaw and CRE/claude-rule-enforcer, and there were closed unmerged PR attempts, but the shipped OpenClaw core still exposes only deny/allowlist/full plus off/on-miss/always.

## Best Possible Solution

Keep #6615 open until OpenClaw either implements first-class exec approval denylist support across the approvals file model, config/types/Zod schema, runtime policy, gateway protocol, CLI/Control UI, docs, and tests, or maintainers explicitly redirect it to ClawHub/plugin policy with any missing authorization hooks tracked separately.
