## Summary

Keep #6615 open. Current main still does not implement the requested exec-approvals denylist mode, `ask: "on-match"`, or denylist argument matching. The SafeClaw and CRE/claude-rule-enforcer links in the discussion are useful prior art/workarounds, but the current OpenClaw core exec-approval policy surface remains allowlist-centric and the related implementation PRs were closed unmerged.

## Best Possible Solution

Keep #6615 open until OpenClaw either implements denylist/on-match/argument-aware deny matching across the exec-approvals model, config/schema, runtime policy, gateway protocol, CLI/session surfaces, macOS/web UI, docs, and tests, or maintainers explicitly redirect the capability to ClawHub with any missing plugin authorization API tracked separately.
