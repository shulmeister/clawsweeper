## Summary

Keep #6615 open. Current main still does not implement the requested exec-approvals denylist mode, `ask: "on-match"`, or `argsMatch` deny matching. The linked CRE/SafeClaw work is useful prior art/workaround evidence, but it has not replaced the missing core exec-approval policy surface, and the prior implementation PRs were closed unmerged.

## Best Possible Solution

Keep #6615 open until OpenClaw either implements denylist/on-match support across the core exec-approvals stack, runtime policy, gateway/node/system.run paths, CLI/session directives, macOS/web UI, docs, and tests, or maintainers explicitly redirect the capability to a ClawHub/plugin policy-engine path and track any missing extension API separately.
