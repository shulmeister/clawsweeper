## Summary

Keep #6615 open. Current main still lacks the requested core exec-approval denylist mode and `ask: "on-match"` behavior. There is some argument-pattern work for allowlist entries, but the shipped policy surface, runtime decision path, CLI/session directives, docs, macOS app, and web UI remain allowlist-centric. The linked CRE/SafeClaw work is useful prior art, and PRs #43222 and #60684 attempted related fixes, but those PRs were closed unmerged and no core implementation has shipped.

## Best Possible Solution

Keep #6615 open until OpenClaw either implements denylist/on-match support in the core exec-approvals stack across types, normalization, runtime policy, gateway/node/system.run paths, CLI/session directives, macOS/web UI, docs, and tests, or maintainers explicitly redirect the feature to a ClawHub/plugin policy-engine path and track any missing plugin API separately.
