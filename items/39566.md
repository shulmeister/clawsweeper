## Summary

Keep #39566 open. Current main still resolves Telegram topic config by exact `message_thread_id` only; `topics."*"` fallback is not implemented in runtime, docs, or targeted tests. The PR needs porting/cleanup before merge, but the requested behavior is still useful and missing.

## Best Possible Solution

Keep the PR open, but require a refreshed patch that implements exact-topic-then-`topics["*"]` fallback in `extensions/telegram/src/bot-core.ts`, adds focused Telegram runtime/schema tests for group and DM wildcard defaults plus exact-topic priority, and updates the Telegram docs/types. If maintainers decide wildcard topic defaults are not desired, close only after that product decision is explicit.
