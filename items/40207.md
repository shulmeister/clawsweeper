## Summary

Keep open. Current main still generates LaunchAgent StandardOutPath/StandardErrorPath from the OpenClaw state directory, which defaults to `$HOME/.openclaw/logs`, and there is no symlink realpathing or move to a stable macOS user log directory for launchd stdout/stderr. The issue has two concrete macOS confirmations and remains actionable in core LaunchAgent generation.

## Best Possible Solution

Keep the issue open. The likely fix belongs in the macOS LaunchAgent log path generation path: either default LaunchAgent stdout/stderr to a non-symlinked macOS user log directory such as `~/Library/Logs/OpenClaw`, or resolve symlinked state/log paths before writing `StandardOutPath` and `StandardErrorPath`. Add regression coverage for a symlinked `~/.openclaw`/state dir and for plist regeneration, then update the gateway/mac logging docs.
