## Summary

Keep open. Current main has active-memory tuning knobs and a short result cache, but it does not implement the requested warm-standby or reusable embedded-agent session. Each active-memory recall still creates a fresh subagent session id/session key and session file before calling the embedded runner, and the runner still performs the per-invocation setup path described in the issue.

## Best Possible Solution

Keep this issue open and treat it as an active-memory/embedded-runner performance feature. The likely fix belongs in the bundled active-memory plugin plus the supported embedded-agent runtime API: add a bounded reusable session or warm-standby path with TTL/config/model/auth invalidation, preserve isolation from the main user transcript, and add tests proving no cross-query memory leakage plus benchmarks for first and subsequent recall latency.
