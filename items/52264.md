## Summary

Current `main` does not already implement this startup hardening. Relative `OPENCLAW_STATE_DIR` remains cwd-sensitive on every resolution, and neither gateway startup entrypoint pins that env override to an absolute path before later startup work.

## Evidence

- **State dir override still resolves against current cwd:** `resolveStateDir()` reads `OPENCLAW_STATE_DIR` and immediately passes it through `resolveUserPath()` on each call, so a relative override is re-resolved at call time instead of being pinned once at startup.
  - file: [src/config/paths.ts:65](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/config/paths.ts#L65)
  - command: `nl -ba src/config/paths.ts | sed -n '60,97p'`
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Relative path resolver is cwd-sensitive:** `resolveHomeRelativePath()` falls back to `path.resolve(trimmed)` for non-`~` inputs, which means `./state` follows the process current working directory whenever it is resolved.
  - file: [src/infra/home-dir.ts:121](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/infra/home-dir.ts#L121)
  - command: `nl -ba src/infra/home-dir.ts | sed -n '102,122p'`
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **CLI gateway startup has no normalization hook:** `runGatewayCommand()` imports `resolveStateDir` from config, but there is no `normalizeStateDirEnv` import or startup call before config loading and server startup.
  - file: [src/cli/gateway-cli/run.ts:11](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/cli/gateway-cli/run.ts#L11)
  - command: `nl -ba src/cli/gateway-cli/run.ts | sed -n '1,20p'; nl -ba src/cli/gateway-cli/run.ts | sed -n '358,416p'`
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Direct server startup also lacks normalization and regression coverage is absent:** `startGatewayServer()` begins by loading config without first normalizing state-dir env overrides, and the current tests do not include the PR's call-order or cwd-pinning coverage.
  - file: [src/gateway/server.impl.ts:242](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/gateway/server.impl.ts#L242)

## Risks / Open Questions

- A relative `OPENCLAW_STATE_DIR` can still drift if any later startup path or plugin code changes the process cwd before another state-dir/config-dir resolution.
- The related `OPENCLAW_CONFIG_PATH` override is also still resolved via the same cwd-sensitive path helper, so the broader class of startup path-drift bugs is not fully eliminated on `main`.
