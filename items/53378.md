## Summary

Keep this issue open. Current main still renders the Control UI Sessions table around the raw session key, optional label input, generic kind badge, and optional displayName text; it does not show a dedicated channel icon/name or derive Telegram/Feishu channel identity for the session list. The related implemented work is for the chat session dropdown/displayName path, not this Sessions table.

## Best Possible Solution

Implement this in the Control UI Sessions page: derive a stable channel display from GatewaySessionRow.channel, origin.provider, and session-key parsing; render a compact channel name/icon or Channel column next to the key; preserve the existing editable Label field; add Feishu to the channel label mapping if needed; and add focused ui/src/ui/views/sessions.test.ts coverage for Telegram and Feishu direct/group rows.
