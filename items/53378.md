## Summary

Current `main` does not implement this request in the Control UI sessions list. The backend already exposes session `channel` metadata, but the sessions table still renders the raw session key as the primary identifier and never displays a channel name or icon.

## Evidence

- **Sessions table still shows raw key:** The Control UI sessions table uses a `Key` column and renders `row.key` directly as the linked primary text. The only secondary text is optional `displayName`; there is no channel badge, label, or icon rendered in the row.
  - file: [ui/src/ui/views/sessions.ts:380](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/ui/src/ui/views/sessions.ts#L380)
  - command: `nl -ba ui/src/ui/views/sessions.ts | sed -n '260,560p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Channel metadata exists in session row contract:** `GatewaySessionRow` already includes `channel?: string` alongside `displayName`, so the data model can carry channel identity to the UI.
  - file: [src/gateway/session-utils.types.ts:18](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/session-utils.types.ts#L18)
  - command: `nl -ba src/gateway/session-utils.types.ts | sed -n '1,120p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Backend populates channel field for sessions.list:** `buildGatewaySessionRow` derives `channel` from the stored entry or parsed key and returns it in the row object, along with `displayName`. This means the missing behavior is in the sessions-list presentation, not the gateway data source.
  - file: [src/gateway/session-utils.ts:1248](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/session-utils.ts#L1248)

  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Another UI surface already maps direct-session keys to channel labels:** The chat session controls parse keys like `agent:<agent>:<channel>:direct:<id>` and format a human-readable fallback such as `<ChannelLabel> · <identifier>`. That logic exists elsewhere, but it is not used by the sessions table.
