## Summary

Current main already guards the BlueBubbles macOS Contacts SQLite lookup by normalizing all phone lookup keys to digit-only strings before constructing the SQL list. That behavior is present in the latest release tag and avoids the proposed PR branch's dot-parameter regression.

## Evidence

- **Current main normalizes lookup keys before SQL construction:** `normalizePhoneLookupKey` strips all non-digits and `uniqueNormalizedPhoneLookupKeys` only returns those normalized values for later lookup use.
