## Summary

Keep open. Current main still lacks the PR's central behavior: overflow detection recognizes several provider-shaped messages, but observed token extraction only handles three older formats, and overflow compaction only receives `currentTokenCount` when extraction succeeds. The related #70479 is explicitly a separate open PR for the later session-remap/orphaning behavior, so it does not supersede this overflow-budget extraction work.

## Best Possible Solution

Keep #70473 open and land this PR or an equivalent core runner/helper change that broadens provider overflow token extraction, forwards parsed counts into overflow compaction, adds the minimally over-budget fallback for confirmed but unparseable overflow messages, and preserves the accompanying regression tests/docs. Keep #70479 as the separate canonical track for the later session preservation/orphaning fix.
