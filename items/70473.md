## Summary

Keep open. Current main still has the narrower overflow token extractor and only forwards currentTokenCount when a provider count is parseable, so PR #70473 is not implemented or obsolete. The linked #70479 work is explicitly separate and does not supersede the overflow-budget extraction/fallback change.

## Best Possible Solution

Keep PR #70473 open and either merge or adapt it after maintainer review: broaden `extractObservedOverflowTokenCount`, pass a synthetic `ctxInfo.tokens + 1` budget for confirmed unparseable overflows, and keep the added tests/docs. Treat #70479 as the separate session-preservation follow-up, not as a replacement for this PR.
