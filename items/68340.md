## Summary

Current `main` already defers `config.patch`/SIGUSR1 restarts until active work drains, covering the reported in-flight-turn kill path with code, docs, and regression tests. The fix appears present on `main` after the issue was filed, but it is not in the latest tagged release yet.

## Evidence

- **`config.patch` now routes restart requests through the shared SIGUSR1 scheduler:** The `config.patch` handler writes the config, builds the restart sentinel payload, and then calls `scheduleGatewaySigusr1Restart(...)` instead of directly forcing a restart.
  - file: [src/gateway/server-methods/config.ts:596](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/gateway/server-methods/config.ts#L596)
  - command: `nl -ba src/gateway/server-methods/config.ts | sed -n '579,607p'`
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Shared restart scheduler defers until pending work drains:** `deferGatewayRestartUntilIdle(...)` polls a pending-work callback and only emits the restart once pending work reaches zero, or after the configured timeout. The default timeout is 300000 ms (5 minutes).
  - file: [src/infra/restart.ts:251](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/infra/restart.ts#L251)

  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Gateway startup registers the deferral check against active runs and pending replies:** On startup the gateway installs `setPreRestartDeferralCheck(...)` with a total of command-queue work, pending replies, active embedded runs, and active tasks, so a restart waits for live work instead of killing it immediately.
  - file: [src/gateway/server.impl.ts:315](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/gateway/server.impl.ts#L315)

  - file: [src/gateway/server-restart-deferral.test.ts:39](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/gateway/server-restart-deferral.test.ts#L39)

## Risks / Open Questions

- If a run exceeds `gateway.reload.deferralTimeoutMs` or that timeout is configured very low, the gateway can still force a restart; current `main` fixes the reported immediate-restart path but does not add a separate interrupted-turn transcript marker.
