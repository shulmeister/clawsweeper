## Summary

Current main still only has partial WebChat input-history support. It stores locally submitted drafts and recalls only from an empty composer; it does not implement the requested ArrowUp/ArrowDown navigation mode, draft preservation/restoration, caret/selection/modifier/IME guards, or history derived from current session user messages. The linked PR #38709 is relevant and explicitly targets this gap, but it remains open and unmerged in the provided GitHub context.

## Best Possible Solution

Keep #38702 open until WebChat implements the full in-session input-history semantics, likely by merging or reworking PR #38709: derive recallable entries from current session user messages, preserve and restore the active draft, enforce modifier/IME/selection/caret guards, reset safely around loading/reload boundaries, and add focused UI/controller tests for ArrowUp/ArrowDown behavior.
