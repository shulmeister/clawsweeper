## Summary

Keep open. Current `main` has only a partial in-memory ArrowUp/ArrowDown recall for an empty composer; it does not implement the issue's requested message-derived history, draft restoration, or guarded activation rules.

## Evidence

- **History navigation only runs when the draft is empty:** `handleKeyDown` gates ArrowUp/ArrowDown recall behind `!props.draft.trim()`. That means navigation does not work while editing a non-empty draft, and the code does not preserve/restore a draft buffer when leaving history mode.
  - file: [ui/src/ui/views/chat.ts:1012](https://github.com/openclaw/openclaw/blob/ac063568d354c14e0a0a43bcffb9725bb3fca651/ui/src/ui/views/chat.ts#L1012)
  - command: `nl -ba ui/src/ui/views/chat.ts | sed -n '1008,1068p'`
  - sha: [ac063568d354](https://github.com/openclaw/openclaw/commit/ac063568d354c14e0a0a43bcffb9725bb3fca651)
- **No transcript-backed history hydration:** Per-session history is created as a fresh `new InputHistory()` cache entry. The only tracked operations in `chat.ts` are `up`, `down`, `push`, and `reset`; there is no code that seeds history from `props.messages` or current session chat history.
  - file: [ui/src/ui/views/chat.ts:130](https://github.com/openclaw/openclaw/blob/ac063568d354c14e0a0a43bcffb9725bb3fca651/ui/src/ui/views/chat.ts#L130)
  - command: `rg -n "new InputHistory|inputHistory\.(up|down|push|reset)" ui/src/ui/views/chat.ts`
  - sha: [ac063568d354](https://github.com/openclaw/openclaw/commit/ac063568d354c14e0a0a43bcffb9725bb3fca651)
- **Stored history is just a trimmed local string stack:** `InputHistory` keeps an internal `string[]`, dedupes adjacent pushes, and returns `null` when walking past the newest item. It has no concept of a saved draft, caret position, selection boundaries, or IME/modifier guards for arrow-key navigation.
  - file: [ui/src/ui/chat/input-history.ts:3](https://github.com/openclaw/openclaw/blob/ac063568d354c14e0a0a43bcffb9725bb3fca651/ui/src/ui/chat/input-history.ts#L3)
  - command: `nl -ba ui/src/ui/chat/input-history.ts | sed -n '1,120p'`
  - sha: [ac063568d354](https://github.com/openclaw/openclaw/commit/ac063568d354c14e0a0a43bcffb9725bb3fca651)
- **History entries are only pushed from composer actions:** Current `main` records history only when sending the current draft or when run controls call `onStoreDraft`; it does not reconstruct history from existing user-role transcript entries as requested by the issue.
  - file: [ui/src/ui/views/chat.ts:1051](https://github.com/openclaw/openclaw/blob/ac063568d354c14e0a0a43bcffb9725bb3fca651/ui/src/ui/views/chat.ts#L1051)
  - command: `nl -ba ui/src/ui/views/chat.ts | sed -n '1008,1068p' && nl -ba ui/src/ui/views/chat.ts | sed -n '1330,1346p'`
  - sha: [ac063568d354](https://github.com/openclaw/openclaw/commit/ac063568d354c14e0a0a43bcffb9725bb3fca651)
