## Summary

Current main still does not implement #38702's requested WebChat input-history behavior. The UI has a partial local send-time stack, but ArrowUp/ArrowDown recall only runs when the draft is empty, does not derive entries from current session user messages, does not preserve/restore a non-empty draft, and lacks the requested modifier/IME/selection/caret guards. The related PR #38709 is still open/unmerged in the provided GitHub context, so this issue remains actionable.

## Best Possible Solution

Keep #38702 open until #38709 or an equivalent focused UI change lands. The implementation should derive recall entries from current session user-role text messages, preserve and restore the active draft, enforce modifier/IME/selection/caret guards, avoid stale recall during loading/reload boundaries, and add targeted WebChat tests for ArrowUp/ArrowDown navigation.
