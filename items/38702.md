## Summary

Keep #38702 open. Current main still has only partial WebChat input history: it stores sent drafts in a per-session in-memory helper and recalls them only when the composer is empty. It does not implement the requested draft preservation/restoration, session-message-derived history, caret boundary rules, modifier/IME/selection guards, or loading/reset hardening. The related PR #38709 directly targets this issue, but it remains open and unmerged in the provided context.

## Best Possible Solution

Keep the issue open until the WebChat composer implements the requested in-session history semantics, likely by merging or reworking PR #38709: derive recallable entries from the rendered/current session user messages, capture and restore the active draft, enforce modifier/IME/selection/caret guards, reset on session reload boundaries, and add focused UI/controller tests for ArrowUp/ArrowDown behavior.
