## Summary

Keep this PR open. Current main still only supports `session.reset.mode` values `daily` and `idle`, so the requested `off` mode is not implemented. The PR is a plausible implementation candidate, but it still needs maintainer review/fixes rather than cleanup closure.

## Best Possible Solution

Keep the PR open and review it as the implementation candidate. The best path is a focused patch that adds `session.reset.mode: "off"` consistently across exported types, zod validation, generated schema/help/docs, and `evaluateSessionFreshness`, includes a unit test for the no-reset behavior, regenerates `docs/.generated/config-baseline.sha256`, and removes or splits the unrelated runtime-logger cleanup.
