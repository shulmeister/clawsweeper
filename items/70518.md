## Summary

Keep this PR open. Current main still does not implement a heartbeat-specific skill allowlist: the strict heartbeat schema and exported config type lack `allowSkills`, the heartbeat runner does not pass a `skillFilter` into reply runs, and the docs/generated config surfaces do not expose the proposed key. The PR is a focused additive implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent small additive change that adds `agents.defaults.heartbeat.allowSkills` and `agents.list[].heartbeat.allowSkills`, preserves unset versus empty-array behavior, threads the value into `getReplyFromConfig` as `skillFilter`, and keeps Zod schema, exported types, generated config help/labels/schema, docs, config-honor inventory, and targeted regression tests aligned.
