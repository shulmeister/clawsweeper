## Summary

Current main still does not deliver messages pushed by `message:received` or `message:sent` internal hook handlers. The issue remains a real current bug, and the likely maintainer fix is still the open PR #41375 rather than shipped code on main.

## Best Possible Solution

Keep #39688 open until PR #41375 or an equivalent maintainer-approved fix lands. The fix should drain internal hook replies only on reply-capable surfaces, add regression coverage for hook-produced reply delivery, and clarify the public hooks contract for which events can auto-send `event.messages`.
