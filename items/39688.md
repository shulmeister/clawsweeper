## Summary

Keep #39688 open. Current main still emits `message:received` and `message:sent` internal hooks as fire-and-forget events without retaining the event or draining hook-produced `event.messages` back through reply delivery. Open PR #41375 is the active maintainer fix path, but it is still open and unmerged.

## Best Possible Solution

Keep #39688 open until PR #41375 or an equivalent maintainer-approved fix lands. The fix should drain internal hook replies only where a safe reply target exists, add regression coverage for hook-produced reply delivery, and update the hooks docs so `event.messages` auto-delivery matches the actual reply-capable surfaces.
