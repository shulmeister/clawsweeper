## Summary

Keep open. Current main still keys Control UI session tokens by the exact normalized URL host, so `ws://127.0.0.1:18789`, `ws://localhost:18789`, and `ws://[::1]:18789` remain separate token scopes. The PR discussion is substantive: the author addressed the IPv6 coverage/dead-branch feedback, but a remaining review concern about broad `127.*` matching should be fixed before merge.

## Best Possible Solution

Keep the PR open, rebase it onto current main, and implement a robust loopback-host canonicalizer for Control UI token/settings scope. It should share scopes for `localhost`, numeric `127.0.0.0/8`, and `[::1]`, preserve protocol/port/path boundaries, leave all non-loopback hosts exact-scoped, and add regression tests for alias reuse, IPv6 reuse, port/path isolation, and DNS names beginning with `127.`.
