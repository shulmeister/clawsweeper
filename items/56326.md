## Summary

Keep open. Current `main` still exposes ACP-only `sessions_spawn` fields in the public tool schema and still rejects those fields for `runtime="subagent"`, so the reported failure mode remains implemented rather than obsolete.

## Evidence

- **Public schema still mixes ACP-only and subagent fields:** `SessionsSpawnToolSchema` on current `main` includes `resumeSessionId`, `cwd`, and `streamTo` in the same public schema as subagent fields like `task`, `agentId`, `mode`, `context`, and attachments.
  - file: [src/agents/tools/sessions-spawn-tool.ts:100](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/tools/sessions-spawn-tool.ts#L100)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Execution path still errors before subagent dispatch:** The tool reads `resumeSessionId` and `streamTo` up front, then returns an error when either is present and `runtime !== "acp"`; this matches the issue's reported failure mode for schema-following models.
  - file: [src/agents/tools/sessions-spawn-tool.ts:187](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/tools/sessions-spawn-tool.ts#L187)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Guard clauses explicitly reject the reported payloads:** Current guards return `streamTo is only supported for runtime=acp` and `resumeSessionId is only supported for runtime=acp` instead of stripping or ignoring ACP-only fields for subagent spawns.
  - file: [src/agents/tools/sessions-spawn-tool.ts:229](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/tools/sessions-spawn-tool.ts#L229)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Tests codify the bug as current behavior:** Unit tests on `main` assert that `resumeSessionId` without `runtime=acp` is rejected, and that `streamTo` with `runtime: "subagent"` is rejected, so the issue is not fixed accidentally elsewhere.
  - file: [src/agents/tools/sessions-spawn-tool.test.ts:354](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/tools/sessions-spawn-tool.test.ts#L354)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Docs still describe these fields as ACP-only:** Subagent docs list the expected `sessions_spawn` params without `resumeSessionId` or `streamTo`, while ACP docs explicitly mark `resumeSessionId` as requiring `runtime: "acp"` and describe `streamTo` as ACP behavior. That mismatch supports the report that the public schema is too broad for subagent use.
  - file: [docs/tools/acp-agents.md:329](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/tools/acp-agents.md#L329)
