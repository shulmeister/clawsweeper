## Summary

Keep open. Current `main` still advances model fallback one candidate at a time, documents only auth-profile rotation within a candidate, classifies HTTP 502 as `timeout`, and tests explicitly enforce a single same-provider cooldown probe before moving to cross-provider fallback. The issue discussion also narrowed the remaining gap after PR #55118: per-auth/per-agent retry controls would not satisfy per-fallback-candidate retry behavior for pooled proxy providers.

## Evidence

There is no per-candidate retry budget or retry-delay field in this path.
