## Summary

Keep #38966 open. Current main still emits a gateway-side `sessions.changed` notification on reset, but the TUI does not subscribe to session change events and its event dispatch only handles `chat`, `chat.side_result`, and `agent`, so the externally reset session still will not refresh in the TUI. The linked PR #40472 is the proposed fix, but it is open and unmerged in the provided GitHub context.

## Best Possible Solution

Keep the issue open until PR #40472 or an equivalent fix lands. The fix should have the TUI subscribe to session change events, handle `sessions.changed` reset/new events for the current session, reload history/session metadata, clear stale token and run state, ignore unrelated sessions, and include focused TUI handler coverage.
