## Summary

Keep #38966 open. Current main still has the gateway-side `sessions.changed` event path, but the TUI does not subscribe to session events and its event dispatcher/handler factory still has no `sessions.changed` reset handler. The linked fix PR #40472 is still the canonical unmerged work in the provided GitHub context, so the observable TUI external-reset refresh bug is not implemented on main.

## Best Possible Solution

Keep #38966 open until PR #40472 or an equivalent fix lands. The implementation should make the TUI subscribe to session change events, handle `sessions.changed` reset/new events for the current session only, reload history and session metadata, clear stale token/run/stream state, ignore unrelated sessions, and add focused TUI handler/client coverage.
