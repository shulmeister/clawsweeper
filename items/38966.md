## Summary

Keep #38966 open. Current main still does not make `openclaw tui` refresh after an external `/new` or `/reset`: the gateway can emit `sessions.changed` for `sessions.reset`, but the TUI neither subscribes to those events nor dispatches/handles them. The linked fix PR #40472 is open/unmerged in the provided GitHub context, so the issue is not implemented on main yet.

## Best Possible Solution

Keep #38966 open until PR #40472 or an equivalent fix lands. The fix should make the TUI subscribe to `sessions.changed`, dispatch and handle `new`/`reset` events for the current session only, reload history/session metadata, clear stale token/run/stream/local-run state, ignore unrelated sessions, and add focused TUI/gateway regression coverage.
