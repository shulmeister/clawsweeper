## Summary

Keep #39680 open. Current main still has the reported value-only `/^[A-Za-z0-9+/=]{80,}$/` heuristic, still passes no env-var key context into `validateEnvVarValue`, and current tests still assert a warning for a non-credential `SAFE_TEXT` key. The related PRs in the provided context were closed unmerged, so they do not supersede the issue.

## Best Possible Solution

Keep this issue open. Tighten `src/agents/sandbox/sanitize-env-vars.ts` by making the base64 warning structurally aware and/or env-key aware, then add regression tests for legitimate long alphanumeric, hex-like, and JWT-segment-like values while preserving explicit secret blocking and null-byte blocking behavior.
