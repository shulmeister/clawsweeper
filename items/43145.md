## Summary

Keep #43145 open. Current main still does not expose the proposed `agent.timeline` gateway method or retained in-memory `AgentRunTraceTimeline` store. Main does have trajectory bundles and some overlapping block-reply/chunker fixes, but those do not replace the PR's gateway run-id timeline API. The PR is also broad and still has unresolved correctness/security review concerns, so the right maintainer path is split/rework rather than auto-close.

## Best Possible Solution

Keep the PR open as related prior art, but do not merge it as-is. The best path is to split a narrow agent timeline proposal from unrelated changes, decide how it relates to existing trajectory/event surfaces and #68204's broader run-trace schema work, then implement a versioned, authorized, bounded, redacted `agent.timeline` or equivalent with protocol/docs/client/test coverage.
