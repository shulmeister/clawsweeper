## Summary

Keep open. I did not find strong evidence that current `main` fixes or obsoletes this Windows gateway HTTP failure. The implicated startup path still prewarms the configured primary model before channel startup, there is still no `OPENCLAW_SKIP_MODEL_PREWARM` escape hatch, and the repo still mutates Undici's global dispatcher on startup while shipping `undici` 8.1.0. This is also a supported surface: docs still say native Windows Gateway use is supported, even though WSL2 is recommended.
