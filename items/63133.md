## Summary

Keep open. The checked-out main still has the reported behavior: lean `status --json` skips memory inspection by returning `memory: null`, but no scan or JSON field distinguishes an intentional skip from an attempted probe that returned no snapshot, and the shared formatter still renders enabled/null memory as `unavailable`. The PR remains a focused implementation candidate, with the latest PR head still present upstream and the provided diff addressing the earlier review concern by reserving `not checked` for an explicit skipped state.

## Best Possible Solution

Keep this PR open for maintainer review, rebase, and CI. The best path is to land this focused checked/skipped memory-collection state, or reimplement the same behavior on main: lean `status --json` without `--all` should render skipped memory collection as `not checked`, while attempted memory probe failures and unavailable memory runtimes should continue to render as `unavailable`.
