## Summary

Current main already fixes the reported v2026.4.24 Bonjour/CIAO crash loop. The plugin now wires scoped process handlers into the advertiser, CIAO PROBING/ANNOUNCEMENT cancellations are classified as handled, the host unhandled-rejection path exits early for registered plugin handlers, and the latest main keeps those handlers registered across advertiser restarts so late probing cancellations do not crash the gateway. Related ClawSweeper reviews for #72103 and #72152 reached the same implemented-on-main conclusion for the same Bonjour/CIAO crash family.

## Best Possible Solution

Close this issue as fixed on current main. Keep the current Bonjour/CIAO implementation: scoped rejection/exception handlers, shared host handler registration, handler lifetime across advertiser restarts, bounded advertiser recovery, and the documented disable-Bonjour workaround for WSL/Docker or blocked multicast environments. Any backport or patch-release decision should go through normal release handling rather than keeping this duplicate crash report open.
