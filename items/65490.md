## Summary

Keep this issue open. Current main has meaningful partial coverage, especially model fallback notices and status output, but the central production-operations gap is still not fully implemented: startup primary-model handling is a warn-only static warmup, there is no compact agent-readable diagnostic surface with current model plus last error/fallback status, and cron add still lacks a dry-run/preflight path and does not require an announce delivery target before persisting a job.

## Best Possible Solution

Keep this issue open, but narrow it into a production execution-discipline tracker: add a configurable startup/provider preflight that can live-probe the configured primary model and surface or block degraded startup; add a safe agent-visible diagnostic RPC/tool with selected model, active/responding model, last provider/fallback error category/code, and fallback state; and add cron add dry-run/preflight plus clearer announce delivery target validation or preview/fail-closed semantics. Reference closed related fallback and cron items as completed subwork rather than treating this as wholly unfixed.
