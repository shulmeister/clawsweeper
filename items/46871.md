## Summary

Keep open. Current main still has the reported split: `nodes status` reads the live/known-node catalog, while `nodes list` reads only the separate node-pairing store. A connected device-paired node can therefore still appear as `Paired: 1 · Connected: 1` in status and be absent from list, which matches the issue and the confirming comments. Related #49719 is another duplicate report, but this issue is the earlier canonical exact report, and #50057 is a broader umbrella rather than a narrower fix tracker.
