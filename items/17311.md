## Summary

Keep #17311 open. Current main supports SecretRefs through env, file, and exec providers, and documents 1Password as an exec-provider recipe, but it still does not implement native keyring/Keychain or native 1Password provider sources. The latest issue discussion also contains a maintainer-authored Codex review explicitly keeping this open for the same remaining scope.

## Best Possible Solution

Keep #17311 open until native keyring/Keychain and 1Password SecretRef provider sources are implemented across the type contract, config schema, Gateway protocol schema, resolver runtime, CLI/setup UX, docs, and tests; or until a maintainer explicitly chooses env/file/exec plus documented exec recipes as the final supported direction and opens or links focused follow-up work for any remaining macOS/keyring needs.
