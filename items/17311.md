## Summary

Keep open. Current main ships the SecretRef foundation for env/file/exec and documents 1Password through an exec-provider recipe, but it still does not implement native keyring/Keychain or native 1Password provider sources. The provided discussion also includes a same-day steipete Codex review keeping this open for that remaining scope, and no narrower canonical follow-up for both native providers is identified.

## Best Possible Solution

Keep #17311 open, but narrow it to the remaining native keyring/Keychain and native 1Password provider design. If maintainers want exec-provider recipes to be the final product direction, record that decision and open/link focused follow-ups for any remaining macOS operator pain before closing this broad issue.
