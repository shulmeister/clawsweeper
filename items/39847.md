## Summary

Keep #39847 open. Current main still lacks the requested shared outbound safety net: the common outbound planner parses raw reply text before any inbound-metadata stripping, delivery only projects that plan plus optional adapter sanitizers, and the Discord adapter does not provide a metadata-stripping sanitizer. The normal assistant-reply path has an upstream sanitizer now, so the remaining bug is narrower than the original report, but shared/direct outbound planning can still preserve echoed metadata.

## Best Possible Solution

Keep #39847 open as the canonical bug. Land a focused source-level fix in `src/infra/outbound/payloads.ts` that strips inbound metadata before directive parsing/formatting in the shared outbound plan, and add regression tests in `src/infra/outbound/payloads.test.ts` covering Conversation/Sender metadata while preserving MEDIA directives, reply tags, audioAsVoice, silent replies, presentation/interactive payloads, and channelData. Then close or supersede the duplicate open fix PRs with that rationale.
