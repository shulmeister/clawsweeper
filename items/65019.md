## Summary

Keep open. Current `main` does not implement a Slack `suppressBareText`/tools-only delivery setting, and the existing Slack send path still delivers ordinary assistant text unless it is exactly the silent token `NO_REPLY`. This remains a plausible channel-plugin feature request rather than an obsolete or ClawHub-only item.

## Evidence

- **Slack config schema has no bare-text suppression field:** `SlackAccountSchema` and `SlackConfigSchema` define Slack knobs such as `streaming`, `replyToMode`, `thread`, `actions`, `dmPolicy`, `heartbeat`, and reactions, but there is no `suppressBareText`, `toolsOnly`, or equivalent channel-level delivery filter.
  - file: [src/config/zod-schema.providers-core.ts:897](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/src/config/zod-schema.providers-core.ts#L897)
  - command: `nl -ba src/config/zod-schema.providers-core.ts | sed -n '897,1005p'`
  - sha: [ae5c6573670b](https://github.com/openclaw/openclaw/commit/ae5c6573670b3742b156899bd2b9c36b8677f191)
- **Slack send path only suppresses the silent token:** `sendMessageSlack(...)` suppresses delivery only when `isSilentReplyText(trimmedMessage)` is true and there is no media/blocks. Otherwise non-empty text proceeds; empty text without other content errors. That matches the report that ordinary narration still posts to Slack.
  - file: [extensions/slack/src/send.ts:349](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/extensions/slack/src/send.ts#L349)

  - sha: [ae5c6573670b](https://github.com/openclaw/openclaw/commit/ae5c6573670b3742b156899bd2b9c36b8677f191)
- **Slack message transform does not strip plain assistant text:** The Slack channel plugin's `transformReplyPayload` only compiles interactive replies when enabled; otherwise it returns the payload unchanged. There is no current plugin-side transform that drops bare assistant text before delivery.
  - file: [extensions/slack/src/channel.ts:398](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/extensions/slack/src/channel.ts#L398)

- **User-facing docs describe streaming/chunking, not bare-text suppression:** Current docs explain that block streaming emits normal channel messages and that preview streaming is message-based. They do not document any Slack option to discard non-tool assistant text before posting.
