## Summary

Current `main` has manual building blocks for nested subagents and agent-to-agent result delivery, but it does not implement the requested gateway-level pre-run auto-escalation/classification and synchronous specialist→coordinator→specialist handoff policy. The issue still describes missing product behavior rather than something already fixed on `main`.

## Evidence

- **Inbound routing is static binding-based, not task-classified:** Agent bindings only match `channel`, `accountId`, `peer`, `guildId`, `teamId`, and `roles`, and route resolution only evaluates those fields plus default-agent fallback. I found no config or route input for content/domain classification before the model runs.

- **Current spawn primitive is explicitly non-blocking:** Docs on current `main` say `sessions_spawn` is always non-blocking and only returns a `runId` plus `childSessionKey`; the implementation also rejects channel-delivery parameters and tells callers to use `message` or `sessions_send` for delivery.
  - file: [docs/concepts/session-tool.md:101](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/docs/concepts/session-tool.md#L101)
