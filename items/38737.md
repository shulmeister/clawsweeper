## Summary

The request is useful, but it is an optional multi-agent orchestration policy rather than an OpenClaw core gap. Current main does not implement a built-in `autoEscalate` config surface, but it does expose enough plugin surface for an external code plugin to classify inbound specialist-channel turns before the specialist model runs, short-circuit the default dispatch, run/wait/read coordinator or specialist subagent sessions, and return the final reply. VISION.md also points optional capabilities to plugins/ClawHub and explicitly guards against core agent-hierarchy/heavy-orchestration layers. The supplied related issues (#17324, #23885, #48784) are narrower closed agent/session targeting requests and do not need to remain open as this broader core feature request.

## Best Possible Solution

Close the core issue as ClawHub/plugin-scope. The best path is an external code plugin with plugin-owned config for domain ownership and escalation policy, using `before_dispatch` or `before_agent_reply` plus `api.runtime.subagent.run`/`waitForRun`/`getSessionMessages` to perform the coordinator handoff. Reopen or file a narrow SDK issue only if that plugin uncovers a concrete missing extension API.
