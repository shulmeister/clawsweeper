## Summary

Keep #39631 open. Current main still builds a user-facing `⚠️ ... failed` error payload for mutating tool failures such as `edit`, even when assistant output already exists, and the Telegram dispatcher still delivers those `isError` payloads rather than suppressing or normalizing them. The related fix PRs #39636 and #39717 were closed unmerged in the provided GitHub context, so they do not supersede the issue.

## Best Possible Solution

Keep the issue open until OpenClaw deliberately changes the agent reply payload policy. The likely fix belongs around `resolveToolErrorWarningPolicy` and `buildEmbeddedRunPayloads`: suppress or normalize failed mutating-tool warning payloads when a user-facing assistant reply already exists, preserve a fallback warning when no assistant reply exists, and keep full details in logs/debug surfaces. Update payload tests and Telegram dispatch tests so failed `edit()` no longer leaks a raw tool warning into user chat.
