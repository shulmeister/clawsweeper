## Summary

Keep #39631 open. Current main still surfaces failed mutating tool calls such as `edit` as user-facing `⚠️ ... failed` payloads, and the Telegram dispatcher still sends final `isError` payloads instead of suppressing or normalizing them. The related fix PRs #39636 and #39717 were closed unmerged, while broader #39406 remains open rather than superseding this Telegram-specific report.

## Best Possible Solution

Keep the issue open until OpenClaw deliberately changes the user-facing policy for failed mutating tools. The likely fix belongs around `resolveToolErrorWarningPolicy` and `buildEmbeddedRunPayloads`: suppress or normalize failed mutating-tool warning payloads when a user-facing assistant reply already exists, keep full details in logs/debug surfaces, preserve a clean fallback when no assistant reply exists, and add regression coverage in both payload tests and Telegram dispatch tests for failed `edit()` not leaking a raw warning into chat.
