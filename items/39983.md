## Summary

Issue #39983 should remain open. Current main still implements the bundled Ollama memory embedding provider with Ollama's deprecated `/api/embeddings` endpoint, sends `prompt`, and parses a single `embedding` array; the tests also assert that old contract. The issue is a real bundled-plugin behavior bug, not stale cleanup, and the remaining fix has not landed on main.

## Best Possible Solution

Keep the issue open and land a focused bundled Ollama plugin fix: update `extensions/ollama/src/embedding-provider.ts` to call `/api/embed`, send `input`, parse `embeddings[0]` or batch `embeddings`, preserve the existing base URL/auth/header/SSRF behavior, and update `extensions/ollama/src/embedding-provider.test.ts` to assert the current Ollama API contract. After the fix lands, close this issue and close or supersede duplicate PR #40205 against the merged implementation.
