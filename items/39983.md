## Summary

Keep #39983 open. Current main at e672b61417af5c45b0431df6d9109a1f4b618ef5 still implements the bundled Ollama memory embedding provider with `/api/embeddings`, sends `prompt`, parses `embedding`, and has tests that lock in that old contract. The current Ollama API documents `/api/embed` with `input` and `embeddings`, so the reported bug remains actionable in this repository.

## Best Possible Solution

Keep the issue open and land a focused bundled Ollama plugin fix. Update `extensions/ollama/src/embedding-provider.ts` to call `/api/embed`, send `input` (preferably using batch input where appropriate), parse `embeddings[0]` or batch `embeddings`, and preserve the existing base URL, auth/header, timeout, and SSRF behavior. Update `extensions/ollama/src/embedding-provider.test.ts` to assert the current Ollama API shape. PR #40205 remains a candidate, but the issue should close only after the fix lands on main.
