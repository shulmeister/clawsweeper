## Summary

Close PR #66457 as duplicate/superseded. Current main still implements the old maxFileBytes suppression behavior, so this PR is not implemented-on-main; however the same remaining log-rotation work is already tracked by open issue #66394 and the older, narrower open PR #58621. PR #66457 also bundles unrelated ACPX runtime/service/config changes and has review findings that should not become the canonical logging fix vehicle.

## Best Possible Solution

Keep size-based log rotation tracked through #66394 and the narrower open PR #58621. The eventual fix should be logging-only, update src/logging/logger.ts, the logging tests, config type/help text, and public logging docs, and deliberately address custom-path retention, rotate races, failure-warning reset, and any new rotation options. The ACPX queue-owner/client-pooling changes in #66457 should be split into a separate PR if they are still desired.
