## Summary

Current `main` does not inject context-window usage into the system prompt Runtime section. The Runtime builder still only emits agent/host/repo/os/node/model/default_model/shell/channel/capabilities/thinking, while context usage remains exposed through `/status` and `session_status` instead.

## Evidence

- **Runtime section still built without context usage:** `buildAgentSystemPrompt()` emits `## Runtime` via `buildRuntimeLine(...)`, and `buildRuntimeLine()` has no context-usage field. It only joins agent, host, repo, os, node, model, default_model, shell, channel, capabilities, and thinking.
