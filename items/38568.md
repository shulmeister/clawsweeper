## Summary

Keep #38568 open. Current main at c7a0d9b1889ca09ab5bfd84cda39a8d50d3d5768 still does not inject context-window usage into the system-prompt Runtime line, and the prompt parameter/call paths do not carry a current context usage snapshot. The issue remains a concrete core prompt/runtime enhancement, not a duplicate or ClawHub-scoped request.

## Best Possible Solution

Implement this in core prompt assembly. Add an explicit context usage field to the system-prompt runtime params, populate it from the current session/model prompt snapshot before prompt construction, render a compact Runtime field such as context=49% or context=98k/200k (49%), and add focused tests for buildRuntimeLine plus embedded/CLI prompt wiring while preserving existing status and session_status behavior.
