## Summary

Keep open. Current main still does not inject context window usage into the system prompt Runtime line. The request is concrete core prompt/runtime behavior, the prior discussion correctly identifies it as useful, and current code still only exposes context usage through status/session/UI surfaces rather than automatic prompt metadata.

## Best Possible Solution

Keep the issue open. The likely implementation belongs in core prompt assembly: add a structured context usage field to the runtime prompt params/runtimeInfo contract, wire it from the runner's current token accounting and context window, render a compact Runtime field such as context=49%, and add focused system-prompt tests plus runner wiring coverage. This should remain core-owned rather than moved to ClawHub because the requested behavior is automatic system prompt metadata, not an optional plugin capability.
