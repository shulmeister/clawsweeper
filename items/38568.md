## Summary

Current main still does not inject context window usage into the system prompt Runtime line. The request remains a concrete core prompt/runtime enhancement, not a stale, duplicate, ClawHub-only, or already-shipped cleanup candidate.

## Best Possible Solution

Keep the issue open and implement it in core prompt assembly: carry the current prompt/context usage snapshot into the system-prompt runtime params, render a compact context percentage or used/budget percentage in buildRuntimeLine, and add focused tests for both the prompt Runtime line and existing status/session_status behavior.
