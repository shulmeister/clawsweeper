## Summary

Keep open. Current main still builds the gateway install plan with literal managed .env/config/auth-profile values and then serializes those values into the LaunchAgent plist, so PR #70968 is not obsolete or implemented on main. The PR addresses the closed #70612 report, but its current patch still needs maintainer review because the review thread identified a plausible PATH/service-owned-variable purge regression.

## Best Possible Solution

Keep PR #70968 open and ask for a small revision: remove literal values for managed secret/durable keys before service plist serialization, but do not purge generated service-owned environment entries. Add regression coverage for both secret removal and PATH preservation, then run the targeted install-plan test and changed gate before merge.
