## Summary

Keep open. Current `main` still does not implement `payload.model: "default"` as a cron runtime-default sentinel. The cron path treats any non-empty `payload.model` as an explicit model override, and downstream model parsing resolves bare strings like `default` as a literal model id under the effective default provider. The related PRs #37847 and #37871 proposed the expected guard, but the provided GitHub context shows both were closed unmerged, and the guard is absent from current source.

## Best Possible Solution

Implement the sentinel in the current cron model-selection path, likely by treating trimmed `payload.model === "default"` the same as an omitted/empty model in `src/cron/isolated-agent/model-selection.ts`. Also review `src/cron/isolated-agent/run-fallback-policy.ts` so `default` does not accidentally count as an explicit model override for fallback resolution. Add focused regression coverage for `payload.model: "default"`, whitespace around `default`, omitted/empty model, specific provider/model overrides, and fallback behavior.
