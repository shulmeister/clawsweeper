## Summary

Keep #37843 open. Current main at c7a0d9b1889ca09ab5bfd84cda39a8d50d3d5768 still treats `payload.model: "default"` as a non-empty cron model override, sends it through normal model resolution, and separately classifies it as a payload override for fallback policy. The related fix PRs in the provided GitHub context were closed unmerged, and current tests/docs cover omitted, empty, whitespace, explicit, invalid, and disallowed models, but not the requested `default` sentinel.

## Best Possible Solution

Keep the issue open. Implement `default` in the current cron model-selection path by treating trimmed `payload.model === "default"` as unset before `resolveAllowedModelRef`, update `resolveCronFallbacksOverride` so the sentinel preserves omitted-model fallback behavior, and add focused tests for omitted, empty, whitespace-trimmed `default`, explicit provider/model, disallowed, invalid, and fallback cases. Update cron docs if the sentinel becomes a supported CLI/config contract.
