## Summary

Keep open. Current main still does not implement `payload.model: "default"` as a cron runtime-default sentinel. Cron trims any non-empty `agentTurn` payload model and resolves it as a model override; a bare string like `default` is parsed as `<defaultProvider>/default`, and the allowed-model resolver has no special branch to skip override resolution. The related PRs #37847 and #37871 proposed the requested guard, but the provided GitHub context shows both were closed unmerged.

## Best Possible Solution

Keep #37843 open until cron runtime model selection reserves `payload.model: "default"` as an explicit request to use the configured agent/default model at run time, with aligned fallback-policy behavior and a regression test for `"default"` plus whitespace-trimmed `" default "`. Docs for `openclaw cron add|edit --model` should mention the sentinel if maintainers accept that product direction.
