## Summary

Keep #37843 open. Current main still treats any non-empty trimmed cron `payload.model` as an explicit model override, so `model: "default"` is not implemented as a runtime-default sentinel. The related PRs in the provided issue context are closed unmerged, and current docs/tests do not document or verify the sentinel behavior.

## Best Possible Solution

Keep the issue open. Add a shared cron payload-model normalization helper that trims `payload.model` and treats exact `default` as unset, then use it in both `resolveCronModelSelection` and `resolveCronFallbacksOverride`. Add focused cron tests for undefined, empty, whitespace-only, exact `default`, trimmed `default`, explicit provider/model, invalid/disallowed models, and fallback-chain behavior. Update CLI help/docs if `--model default` is intended as a supported user-facing value.
