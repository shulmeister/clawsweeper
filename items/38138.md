## Summary

Keep #38138 open. Current main still resolves the Discord agent route from channel/account/peer/parentPeer before mention detection, then uses mentions only as response gating for the already selected route. The requested behavior, where an explicit @mention of another agent overrides a parent channel/thread binding, is not implemented on main d74f897c1c6b2de756c98eb3bc7de45f411d6a5c.

## Best Possible Solution

Keep the issue open and implement a narrow mention-aware routing override for Discord multi-agent threads. The fix should map explicit Discord bot mentions and/or per-agent mentionPatterns to an agent before parentPeer/configured binding wins, add a matchedBy/debug path or equivalent preflight override, preserve explicit /focus/thread-bound session semantics intentionally, and add regression coverage for a parent-bound thread where @Dex routes to Dex instead of the parent-bound agent. If maintainers want one general design, link it to the broader content-based routing work (#13304), but #38138 is still a concrete missing Discord behavior today.
