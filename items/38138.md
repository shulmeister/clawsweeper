## Summary

Keep #38138 open. Current main still resolves Discord agent routing from static binding inputs before mention detection, and mention handling is only used for gating the already selected route. The prior ClawSweeper keep-open comment remains consistent with current main at 7e13f3f5140168e31027b39a9855ade3fadf24d5.

## Best Possible Solution

Implement a Discord mention-based routing override before mention gating, either as an explicit highest-priority route tier or as a documented preflight override that can map native Discord bot identity and per-agent mentionPatterns to an agent. Add regression coverage for a thread inheriting a parentPeer binding while explicitly mentioning another configured agent, and update the public Discord/multi-agent routing docs once the behavior is supported.
