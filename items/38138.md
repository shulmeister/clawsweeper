## Summary

Keep #38138 open. Current main at 82262387651ab270b565f71533ce4937c9c00dc6 still routes Discord messages from static channel/account/guild/role/peer/parentPeer inputs before mention detection, and native Discord mentions plus mentionPatterns only gate the already selected effective route. The existing ClawSweeper comment on the issue reached the same conclusion, and the current routing, Discord preflight, tests, and docs still support keeping the request open.

## Best Possible Solution

Keep the issue open. Implement a Discord mention-based routing override before mention gating, either as a highest-priority route tier or a documented Discord preflight override that maps native bot mentions and per-agent mentionPatterns to the target agent. Add regression coverage for a Discord thread inheriting a parentPeer binding while explicitly mentioning another configured agent, then update https://docs.openclaw.ai/channels/channel-routing, https://docs.openclaw.ai/channels/discord, and https://docs.openclaw.ai/concepts/multi-agent with the final precedence rules.
