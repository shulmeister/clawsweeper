## Summary

Keep #38138 open. Current main still resolves Discord routing from static channel/account/peer/parentPeer inputs before mention detection, and mention handling gates the already selected effective route instead of selecting the mentioned agent. The prior ClawSweeper keep-open comment remains consistent with current main at 631552c554095b0c520376db9131dac0f5427f94.

## Best Possible Solution

Keep the issue open. Implement a Discord mention-based routing override before mention gating, either as a highest-priority route tier or as a documented Discord preflight override that maps native bot identity and per-agent mentionPatterns to an agent. Add regression coverage for a Discord thread inheriting parentPeer while explicitly mentioning another configured agent, then update the public Discord and multi-agent routing docs.
