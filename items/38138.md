## Summary

Keep #38138 open. Current main still chooses the Discord agent from static routing inputs before mention detection, then uses the selected agent's mention patterns only for gating. The provided related context does not identify a canonical duplicate that covers this Discord thread-parent override, and the prior Codex review remains consistent with the current 46b9044 main code.

## Best Possible Solution

Implement an explicit Discord mention routing override in the routing/preflight path, with tests for a thread that inherits a parentPeer binding but mentions another configured agent. The fix should map native Discord bot identity and/or per-agent mentionPatterns to the intended agent, recompute the route/session for that agent before gating and processing, and update channel routing docs if mention override becomes a supported route tier.
