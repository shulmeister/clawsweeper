## Summary

Keep open. Current `main` does not provide strong evidence that the reported `/new` vs `/reset` behavior is resolved across OpenClaw. The TUI now distinguishes them, but core Gateway/docs still describe `/reset` as an alias for `/new` and still test bare `/new` as resetting the same session.
