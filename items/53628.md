## Summary

Keep open. Current main still does not expand `${XDG_CONFIG_HOME}` or other `$VAR` placeholders in shared user-path resolution, and the provided GitHub context shows an open unmerged PR with closing syntax for this issue.

## Best Possible Solution

Keep this issue open. Review and either land or close PR #58281, reconciling duplicate PR #53718 as part of the same triage. The likely fix belongs in `src/infra/home-dir.ts` with regression coverage in `src/infra/home-dir.test.ts` for `${VAR}` and `$VAR` expansion through `resolveUserPath`/`resolveHomeRelativePath`.

## Evidence
