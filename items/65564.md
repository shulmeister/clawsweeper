## Summary

Keep open. Current main appears to have fixed the original stale sessionFile carryover for forced-new cron sessions, but the later issue discussion identifies a remaining heartbeat isolated-session rotation leak. The current heartbeat runner still archives only stale nested `:heartbeat` keys and does not capture/archive the previous transcript for the normal isolated-session key before overwriting the store entry.
