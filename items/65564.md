## Summary

Keep open. Current `main` appears to have fixed the original stale-`sessionFile` carryover, but the isolated-heartbeat rotation path still overwrites the same isolated session key with a new `sessionId` without archiving the previous transcript from that same key. That matches the later issue comments about lingering orphan transcript accumulation, and the existing doctor/dreaming cleanup code still assumes those orphan `.jsonl` files can exist.
