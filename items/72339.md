## Summary

Current main addresses the v2026.4.24 Bonjour/CIAO crash. The Bonjour plugin now registers scoped CIAO rejection/exception handlers through the plugin runtime, classifies CIAO probing/announcement cancellations as handled, keeps those handlers alive across advertiser restarts, and the global unhandled-rejection path exits early when a registered handler handles the reason. Tests and docs cover the reported stuck probing/announcing and process-disable behavior. The related PR #72332 correctly identified the Bonjour/CIAO restart path, but current main already contains the maintained fix.

## Best Possible Solution

Close this issue as implemented on main. Keep the maintained Bonjour/plugin-runtime handler approach, and let any remaining release tracking happen through the normal release process rather than keeping this v2026.4.24 crash report open.
