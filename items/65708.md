## Summary

Current `main` and shipped release `v2026.4.23` no longer document an invalid `cacheTtlMs: 300000` Active Memory example. The manifest and runtime still enforce a `120000` ms ceiling, so the docs/schema mismatch described in #65708 is resolved.

## Evidence

- **Manifest schema still caps cache TTL at 120000 ms:** `plugins.entries.active-memory.config.cacheTtlMs` is defined with `minimum: 1000` and `maximum: 120000` in the Active Memory plugin manifest.
  - file: [extensions/active-memory/openclaw.plugin.json:57](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/extensions/active-memory/openclaw.plugin.json#L57)
  - command: `nl -ba extensions/active-memory/openclaw.plugin.json | sed -n '45,75p'`
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **Runtime clamp matches the manifest ceiling:** The runtime config resolver clamps `raw.cacheTtlMs` to a maximum of `120_000`, so current runtime behavior matches the manifest ceiling.
  - file: [extensions/active-memory/index.ts:656](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/extensions/active-memory/index.ts#L656)
  - command: `nl -ba extensions/active-memory/index.ts | sed -n '640,665p'`
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
