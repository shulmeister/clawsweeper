## Summary

Keep PR #43404 open. Current main at a57d681 still has both reported behaviors: the reminder guard only matches enabled cron jobs by sessionKey, and compaction retry still resets successfulCronAdds to 0. The PR is a focused fix for open bug #43292, and its final diff addresses the earlier bot-review concern by limiting the agentId fallback to sessionTarget="isolated" jobs.

## Best Possible Solution

Keep this PR open as useful implementation work for #43292. Maintainers should rebase it onto current main, preserve successfulCronAdds across compaction retries, match existing isolated cron jobs by agentId only when sessionTarget is "isolated" and no sessionKey is present, keep the added regression tests, then close #43292 when the fix lands.
