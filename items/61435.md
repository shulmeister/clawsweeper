## Summary

Current main has partial Google Chat replyToMode plumbing, but it does not implement the requested DM-specific thread control and the default typing-indicator reply path can still place the actual first response inside the inbound Google Chat thread. A meaningful requested capability remains missing.

## Evidence

- **DM config lacks requested control:** GoogleChatDmConfig only exposes enabled, policy, and allowFrom; there is no dm.threadReplies or DM-scoped reply/threading option. The only nearby field is top-level replyToMode, documented in the type comment as controlling reply threading when reply tags are present.

  - file: [extensions/googlechat/src/monitor.ts:259](https://github.com/openclaw/openclaw/blob/4069c81b159087cb71f6b64676d533cf0d897d68/extensions/googlechat/src/monitor.ts#L259)
