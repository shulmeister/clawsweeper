## Summary

Keep PR #68186 open. Current main still contains the reported `screen.snapshot` problems, and the PR is a focused bug/security-hardening change with tests, but its latest review context still has an unresolved P2 concern about dynamic `id`/`nodeId` envelope sizing before it is safe to land.

## Best Possible Solution

Keep the PR open and ask for the remaining size-guard fix: compute the limit from the real outgoing `node.invoke.result` frame, or at least include actual request/node identifier lengths, while preserving the PR's useful fail-closed malformed-param handling, sanitized capture errors, validation-error mapping, changelog entry, and focused macOS runtime regression tests. After that, the PR can serve as the canonical fix for #68181.
