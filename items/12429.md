## Summary

Current main does not implement the requested Kubernetes ServiceAccount Trust pairing path. It has a minimal single-pod Kubernetes gateway deployment and an optional CIDR-based node auto-approval mechanism, but no TokenReview auth mode, no k8s token field in the gateway connect protocol, no node-host `--k8s-trust` flow, no worker deployment/Helm chart for autoscaled nodes, and no secure headless worker pairing bypass. The linked PR #12434 was closed unmerged, and the broader open RFC #42026 is related but does not precisely supersede the ServiceAccount TokenReview pairing request.
