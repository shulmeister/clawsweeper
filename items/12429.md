## Summary

Current `main` still does not implement the requested Kubernetes ServiceAccount Trust pairing path for autoscaled worker/node-host pods. The issue is not safe to close as implemented, duplicate, or ClawHub/plugin work: the remaining work requires a gateway connect-protocol/auth change plus node-host CLI/runtime support, and the referenced implementation PR #12434 was closed unmerged while the broader RFC #42026 does not precisely supersede this narrow K8s pairing request.

## Best Possible Solution

Keep this issue open until OpenClaw either implements a secure, opt-in headless Kubernetes pairing path across gateway config/schema, connect protocol, server TokenReview validation, node-host CLI/runtime, tests, docs, and deployment assets, or a maintainer explicitly redirects the work to a narrower canonical design issue. If the team wants this outside core, first define a documented gateway handshake/auth extension point; the current plugin surfaces are not enough to replace WebSocket connect authentication safely.
