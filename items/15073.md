## Summary

Keep open. Current `main` still supports only model-list failover plus shared bootstrap-context controls. I found no native support for switching to a different agent/workspace when a fallback model is used, and no per-model bootstrap context/profile config that would satisfy this request.

## Evidence

- **Model config only exposes primary plus fallback model refs:** `AgentModelConfig` is just a string or `{ primary, fallbacks }`. There is no `fallbackAgent`, fallback workspace, or model-specific context-profile field in the shared agent model schema.
  - file: [src/config/types.agents-shared.ts:8](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/config/types.agents-shared.ts#L8)

- **Per-agent config has workspace and model, but no fallback-agent linkage:** `AgentConfig` supports `workspace`, `agentDir`, and `model`, but nothing that maps model failover to another agent id or alternate workspace/context.
  - file: [src/config/types.agents.ts:70](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/config/types.agents.ts#L70)
