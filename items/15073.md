## Summary

Keep #15073 open. Current main still supports per-agent model fallback lists only; fallback attempts do not switch to another agent, workspace, agentDir, session file, skills snapshot, or context profile. The provided issue discussion also includes a recent Codex/maintainer comment explicitly keeping this issue open for the same reason.

## Best Possible Solution

Keep #15073 open until maintainers choose a first-class core design: either fallbackAgent/fallbackWorkspace semantics or a narrower model-specific context-profile/budget mechanism. A complete implementation likely belongs in config types and strict schemas, schema help/docs, fallback candidate metadata, reply-runner/session plumbing that can safely swap agentDir/workspaceDir/sessionFile/skills/context per attempt, and regression tests proving fallback retries use alternate context without corrupting the original session.
