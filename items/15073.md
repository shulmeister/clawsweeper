## Summary

Keep open. Current main still implements fallback as provider/model retry within the same active agent session, not as a switch to a different agent/workspace/context profile. The provided issue discussion also contains a same-day maintainer/Codex comment explicitly keeping #15073 open, and the related closed issues are narrower model-context-budget/profile ideas rather than a canonical replacement for this broader fallback-agent request.

## Best Possible Solution

Continue tracking #15073 until OpenClaw either adds an explicit fallback-agent/workspace/context-profile design or a maintainer chooses a narrower model-specific bootstrap/context-budget design as the canonical product direction. The likely implementation spans the agent config schema, fallback candidate model, agent-command retry plumbing, bootstrap/context assembly, docs, and regression tests proving a fallback can use a different agent/workspace without corrupting the original session.
