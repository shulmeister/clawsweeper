## Summary

Keep open. Current main still implements model fallback as provider/model retry inside the same agent run, not as a switch to another agent/workspace/context profile. The issue discussion also includes a same-day maintainer/Codex review saying to keep this open; the related closed issues are narrower context-budget/profile ideas, not a canonical replacement for this broader fallback-agent request.

## Best Possible Solution

Continue tracking #15073 until OpenClaw either adds an explicit fallback-agent/workspace/context-profile design, or a maintainer chooses a narrower model-specific bootstrap/context budget solution as the canonical product direction. The likely implementation spans the agent config schema, fallback candidate model, agent-command retry plumbing, bootstrap/context assembly, docs, and regression tests proving that a fallback can use a different agent/workspace without corrupting the original session.
