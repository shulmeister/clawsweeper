## Summary

Keep #69110 open. Current main still does not pass resolved provider/model identity into `message_sending` delivery hooks, and there is no core enforcement that compares an agent-emitted model tag with the actual runtime model. The code does know the resolved model/provider after a run, and `llm_output` exposes it, but that path remains separate from the outbound delivery hook the issue targets. Related PRs such as #65149 and #39761 are open/unmerged design candidates, not shipped fixes.

## Best Possible Solution

Keep the issue open and coordinate a single additive provenance design. The likely product/code path is to thread resolved provider/model/run provenance from the agent result into final reply assembly and outbound delivery hooks, then add regression coverage for a fallback run where the selected session model differs from the actual model used. #65149 and #39761 are relevant implementation candidates, but neither currently ships on main or fully closes the core enforcement question.
