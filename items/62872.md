## Summary

Current main still leaves the reported persistent-session `sessions_send` duplicate-message behavior unresolved. The shipped ACP-child fix only skips A2A for the parent-owned one-shot ACP child case; persistent ACP targets and normal peer sessions still run the generic ping-pong plus announce flow, and `sessions_send` still has no per-call ping-pong disable/max option.

## Best Possible Solution

Keep this issue open as the canonical tracker. The likely fix belongs in `src/agents/tools/sessions-send-tool.ts` and `src/agents/tools/sessions-send-tool.a2a.ts`: add a per-call ping-pong max/disable option or make A2A context-aware so persistent peer-session sends return the direct inline reply without framework-generated announce/ping-pong turns by default, while preserving the existing parent-owned one-shot ACP child skip and adding regression tests for persistent peer sessions.
