## Summary

Keep open. Current `main` still has no watchdog for a sync loop that simply stops producing progress without emitting `STOPPED` or `sync.unexpected_error`, and the relevant Matrix monitor files are unchanged between the latest release `v2026.4.23` and current `main`. The issue discussion shows one local non-repro, but it also requested targeted container-side `/sync` evidence that has not been provided yet, so there is not high-confidence evidence to close conservatively.

## Evidence

- **Monitor waits only for abort or explicit fatal sync events:** `monitorMatrixProvider()` races shutdown against `syncLifecycle.waitForFatalStop()`. It records `healthySyncSinceMs` from sync-state events, but there is no timer or watchdog that fails the monitor when sync activity simply stops arriving.
  - file: [extensions/matrix/src/matrix/monitor/index.ts:258](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/extensions/matrix/src/matrix/monitor/index.ts#L258)
  - command: `nl -ba extensions/matrix/src/matrix/monitor/index.ts | sed -n '250,470p'`
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Status updates depend on SDK sync-state events:** `createMatrixMonitorStatusController()` marks Matrix healthy on `SYNCING` and disconnected on `RECONNECTING`/`ERROR`/`STOPPED`, but it does not infer failure from missing sync activity.
  - file: [extensions/matrix/src/matrix/monitor/status.ts:58](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/extensions/matrix/src/matrix/monitor/status.ts#L58)
