## Summary

Current main now covers the reported silent Matrix sync-stall failure mode with Matrix-owned sync lifecycle errors plus transport-activity-based stale-channel recovery. Matrix only refreshes transport liveness on successful SYNCING /sync responses, and the gateway health monitor restarts channels whose provider-proven transport activity goes stale. This shipped after the reported 2026.4.15 version in v2026.4.22.

## Best Possible Solution

Close as implemented on main. Keep the shipped Matrix sync lifecycle and transport-activity health-monitor behavior; if a current release still misses Matrix events, ask for a new focused report with the in-container /sync result, OpenClaw/Node versions, proxy/Synapse logs showing whether OpenClaw continues /sync requests, and current channel health snapshots.
