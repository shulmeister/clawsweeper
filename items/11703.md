## Summary

Keep open. Current `main` still batches WhatsApp inbound by receive-time debounce only. The WhatsApp monitor records `messageTimestampMs` on each inbound message, but the debounce key and flush timing do not use that timestamp, and the current config/schema expose only `debounceMs`, not any timestamp-based coalescing option.

## Evidence

- **WhatsApp batching key ignores messageTimestamp:** The WhatsApp inbound monitor builds the debounce key from account/chat/sender only (`${msg.accountId}:${conversationKey}:${senderKey}`), so messages with the same WhatsApp server timestamp are not grouped unless they arrive within the active receive-time debounce window.
  - file: [extensions/whatsapp/src/inbound/monitor.ts:175](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/extensions/whatsapp/src/inbound/monitor.ts#L175)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **messageTimestamp is preserved but not used for batching:** `msg.messageTimestamp` is converted to `messageTimestampMs` and then copied to the outbound `timestamp` field on the queued message, but there is no logic in this path that uses it to extend or join debounce buffers.
  - file: [extensions/whatsapp/src/inbound/monitor.ts:400](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/extensions/whatsapp/src/inbound/monitor.ts#L400)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Core debouncer flushes from enqueue time:** The shared inbound debouncer starts/reset a `setTimeout(..., buffer.debounceMs)` when each item is enqueued. That means batching is driven by arrival time in OpenClaw, not by any source timestamp embedded in the message payload.
  - file: [src/auto-reply/inbound-debounce.ts:149](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/auto-reply/inbound-debounce.ts#L149)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **No timestamp-coalescing config exists on current main:** The WhatsApp config type/schema still expose only `debounceMs` for batching. There is no `coalesceByTimestamp`, `coalesceWindowMs`, or similar timestamp-based control in the current config surface.
  - file: [src/config/types.whatsapp.ts:104](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/config/types.whatsapp.ts#L104)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
