## Summary

Keep #71677 open. Current main still handles Feishu card.action.trigger callbacks as a void, awaited card-action path with dedupe-only mitigation; there is no immediate callback card response, no ocf1 disable-state field, and no tests/docs showing instant disabled-button updates.

## Best Possible Solution

Keep the issue open and implement it in the bundled Feishu plugin: add an opt-in disable-state field or equivalent typed metadata to the ocf1 interaction contract, make card.action.trigger return an immediate Feishu callback response with a transitional disabled card/toast while dispatching the synthetic command asynchronously, preserve duplicate-token protection, and add focused tests for WS/webhook callback return payloads plus existing quick-action and approval-card behavior.
