## Summary

Keep this issue open. Current main still treats Feishu card actions as a void synthetic-command path: the handler does not return an immediate Feishu callback response, `handleFeishuCardAction` is still `Promise<void>`, and the `ocf1` envelope has no disable-state metadata for transitional disabled buttons.

## Best Possible Solution

Implement this in the bundled Feishu plugin: add a typed opt-in disable-state or transitional-response contract to the Feishu `ocf1` card interaction envelope, return a valid immediate Feishu callback card/toast response from the `card.action.trigger` handler, continue synthetic command dispatch asynchronously, and add targeted tests for WebSocket/webhook callback payloads plus quick-action, approval-card, and duplicate-token behavior.
