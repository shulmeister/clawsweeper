## Summary

Current main still lacks WhatsApp reaction-query support. The WhatsApp plugin advertises and dispatches outbound `react`, but there is no WhatsApp `reactions` query/list action, no persisted historical reaction store, and no inbound WhatsApp reaction event handling surfaced to agents. The related PR #13641 was closed unmerged and only covered inbound event surfacing, so it does not satisfy the query/history request.

## Best Possible Solution

Keep this issue open until the WhatsApp plugin implements a complete reaction-read path: persist or retrieve reaction history from the WhatsApp backend, advertise and handle the generic `reactions` message action for WhatsApp, surface inbound reaction events with correct conversation routing/access checks, and add focused tests/docs for the CLI and WhatsApp plugin behavior.
