## Summary

Keep open. Current `main` still implements WhatsApp outbound `react` only; it does not implement the requested `reactions` listing action, and I did not find a WhatsApp reaction-notification/query surface for agents or CLI history lookup.

## Evidence

- **WhatsApp plugin only advertises `react`:** `extensions/whatsapp/src/channel.ts` wires WhatsApp message actions with `supportsAction: ({ action }) => action === "react"` and gateway execution only for `react`, so `reactions` is not supported on current `main`.
  - file: [extensions/whatsapp/src/channel.ts:143](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/extensions/whatsapp/src/channel.ts#L143)
  - command: `nl -ba extensions/whatsapp/src/channel.ts | sed -n '125,160p'`
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **WhatsApp action handler rejects non-`react` actions:** `extensions/whatsapp/src/channel-react-action.ts` throws unless `params.action === "react"`, which rules out the requested message-reaction query action in the channel implementation itself.
  - file: [extensions/whatsapp/src/channel-react-action.ts:13](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/extensions/whatsapp/src/channel-react-action.ts#L13)

  - file: [docs/cli/message.md:87](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/docs/cli/message.md#L87)
