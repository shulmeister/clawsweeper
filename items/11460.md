## Summary

Keep open. Current main still supports WhatsApp outbound `react` only, not `message reactions` listing/querying, searchable historical reaction storage, or inbound WhatsApp reaction events surfaced to agents. The related PR #13641 was closed unmerged and only attempted incoming event surfacing, not the requested query/history path.

## Best Possible Solution

Keep tracking this issue until the WhatsApp plugin implements a real reaction-read path: decide where reaction history is stored, add a WhatsApp `reactions` message-action handler and discovery entry, surface or persist inbound reaction events with correct routing/access checks, then update CLI docs, WhatsApp docs, and focused tests. PR #13641 can be used as background for the event-surfacing part, but it does not supersede the query/history request and was closed unmerged.
