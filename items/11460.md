## Summary

Keep #11460 open. Current main still only supports outbound WhatsApp `react`; it does not advertise or dispatch the generic `reactions` query action, does not persist or retrieve WhatsApp reaction history, and does not subscribe to inbound WhatsApp reaction events. The related PR #13641 was closed unmerged, and the provided context shows PR #48570 is still only an open/unmerged implementation reference.

## Best Possible Solution

Keep #11460 open until the bundled WhatsApp plugin implements a complete reaction read path: advertise and handle the generic `reactions` message action, retrieve or persist historical reactions where feasible, surface inbound add/remove reaction events through normal routing and access checks, and add focused WhatsApp tests plus docs. PR #48570 can be used as implementation input, but it should not be treated as a shipped superseding fix.
