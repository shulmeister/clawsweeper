## Summary

Keep open. Current `main` still hardcodes Mattermost DMs to `replyToMode="off"`, and the docs still say DMs stay non-threaded, but the outbound Mattermost path still falls back from `threadRootId` to `payload.replyToId` and forwards that as Mattermost `root_id`. Because the shared reply filter allows explicit reply tags to survive even when mode is `off`, a DM reply payload with `replyToId` can still thread on current `main`.
