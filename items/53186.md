## Summary

Keep open. Current main still has the reported Mattermost DM threading bug: DMs are documented and resolved as non-threaded, but the outbound root resolver can still fall back from `payload.replyToId` to Mattermost `root_id`. Open maintainer PR #72305 now tracks the narrow fix path, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and use PR #72305, or an equivalent Mattermost-plugin change, as the implementation path: make `resolveMattermostReplyRootId` chat-kind aware, pass `kind` from all Mattermost delivery call sites, drop `replyToId`/`threadRootId` for direct-message sends, and add focused regression tests. After that merges to main, close this issue as implemented.
