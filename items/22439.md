## Summary

Keep PR #22439 open. Current main still does not implement the proposed schema-backed `agents.defaults.bootstrapTier` feature: bootstrap filtering is binary by session type, config/docs expose only existing controls such as `contextInjection` and `lightContext`, and the extra bootstrap hook has no tier-aware path. The linked issue #22438 is still open, and the recent maintainer-side review comment also explicitly kept this PR open rather than treating it as implemented or superseded.

## Best Possible Solution

Keep #22439 open for maintainer product review alongside #22438. If maintainers accept the direction, the implementation should add a schema-backed `agents.defaults.bootstrapTier`, wire tier resolution through bootstrap file loading and the extra-bootstrap hook, regenerate config metadata/docs, and add focused tests for `minimal`, `standard`, `full`, and backward-compatible defaults. Close only if maintainers explicitly reject the product direction or a canonical follow-up fully covers configurable tiered bootstrap behavior.
