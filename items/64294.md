## Summary

Keep open. Current main still does not expose the proposed `api.runtime.llm.complete()` Plugin SDK runtime capability, and context-engine runtime contexts still do not receive an LLM completion helper. This PR remains a plausible implementation candidate for the still-open Plugin SDK inference gap tracked by #45865, but it needs normal maintainer review rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or replacement by a revised implementation. The best path is an additive `api.runtime.llm.complete()` core runtime seam with exported SDK types, context-engine injection where appropriate, docs/API-baseline updates, and tests that prove model/auth resolution uses the intended active agent. If maintainers prefer a different shape, keep #45865 as the canonical product/API discussion rather than closing this work as obsolete.

## Risks / Open Questions
