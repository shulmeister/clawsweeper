## Summary

Keep open. Current `main` still matches the reported bug: `src/gateway/config-reload-plan.ts` has no top-level `auth` reload rule, so `auth.profiles.*` falls through the unmatched-path branch and triggers a gateway restart. The latest shipped tag I inspected still has the same omission, and PR #68257 with the narrow fix and regression tests is still open and unmerged.

## Evidence

- **Missing `auth` reload rule on current main:** `BASE_RELOAD_RULES_TAIL` explicitly classifies `meta`, `identity`, `wizard`, `logging`, `agents`, `tools`, `bindings`, `audio`, `agent`, `routing`, `messages`, `session`, `talk`, `skills`, `secrets`, `plugins`, `ui`, `gateway`, `discovery`, and `canvasHost`, but there is no `auth` prefix.
  - file: [src/gateway/config-reload-plan.ts:95](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/gateway/config-reload-plan.ts#L95)
