## Summary

Keep open. Current main still has no reload rule for top-level `auth`, so `buildGatewayReloadPlan(["auth.profiles.<provider>:<id>"])` falls through to the unmatched-path restart branch. The linked PR #68257 directly tracks the narrow fix, but it is still open/unmerged in the provided context, and current tests do not cover the requested `auth.profiles.*` no-op behavior.

## Best Possible Solution

Keep the issue open and land or recreate the narrow PR #68257 fix: add a top-level `auth` no-op reload rule that does not shadow `gateway.auth.*`, and add regression tests for `auth.profiles.*` no-op behavior plus `gateway.auth.token`/`gateway.auth.mode` restart behavior in the same batch.
