## Summary

Keep this PR open. Current main still rejects owner-prefixed ClawHub skill slugs at the shared skills boundary, and the PR is a focused implementation candidate for the linked user bug rather than obsolete, duplicated, or already shipped.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a narrow parser in `src/agents/skills-clawhub.ts` that accepts optional `<owner>/<slug>` for requested ClawHub install/update inputs, validates both segments under the existing requested-slug rules, canonicalizes to the bare slug before registry/download/install-dir/lockfile use, preserves legacy tracked-slug compatibility, and keeps the focused regression tests plus changelog entry. If maintainers reject that ClawHub namespace contract, close it with an explicit product decision and update the ClawHub install snippets/docs accordingly.
