## Summary

Keep PR #71857 open. Current origin/main still rejects owner-prefixed ClawHub skill slugs before registry lookup, while the PR addresses a concrete CLI/gateway mismatch reported in #71767 and is not implemented, duplicated, or superseded on main.

## Best Possible Solution

Keep PR #71857 open for maintainer review. The best fix is the narrow shared parser change proposed by the PR: accept optional <owner>/<slug> only for requested ClawHub skill install/update, validate both segments under the existing ASCII slug rules, canonicalize to the bare slug for registry/install-dir/lockfile behavior, preserve legacy tracked-slug update behavior, and keep the focused regression tests plus changelog entry.
