## Summary

Keep PR #71857 open. Current main still rejects owner-prefixed ClawHub skill slugs before any registry lookup, while the linked bug #71767 is concrete and the proposed PR is a narrow fix for the CLI/gateway mismatch rather than obsolete, duplicate, or out-of-repo work.

## Best Possible Solution

Keep this PR open for maintainer review. The best implementation path is the narrow parser change proposed by #71857: accept optional `<owner>/<slug>` only for requested ClawHub skill install/update, validate both segments under the existing ASCII slug guard, strip the owner before registry/download/install-dir/lockfile use, preserve tracked legacy behavior, and keep the focused regression tests plus changelog entry.
