## Summary

Keep this issue open. Current main still resolves exec approvals storage through the home-based `~/.openclaw` defaults instead of `OPENCLAW_STATE_DIR`, and open PR #62938 remains an unmerged implementation candidate with closing syntax for this exact bug.

## Best Possible Solution

Keep this issue open and review, update, land, or replace PR #62938. The fix should live in `src/infra/exec-approvals.ts`: resolve the approvals file and socket under the explicit `OPENCLAW_STATE_DIR` / `resolveStateDir()` contract first, then fall back to the existing `OPENCLAW_HOME` and default-home behavior. Add focused regression coverage in `src/infra/exec-approvals-store.test.ts` for state-dir precedence and keep the existing symlink-safety protections intact.
