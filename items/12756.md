## Summary

Keep #12756 open. Current main still has the generic `read` message action, but the bundled Telegram plugin does not advertise or implement `read`/`readMessages`; the related implementation PR was closed unmerged, and the latest maintainer/Codex discussion also kept this as the canonical open tracker.

## Best Possible Solution

Keep #12756 open and implement the feature in the bundled Telegram plugin: add `read` discovery, action mapping, runtime handling, parameter compatibility for `openclaw message read --target`, docs, and tests. Because Telegram bots do not expose arbitrary chat history, the most likely product path is a bounded inbound message store like PR #14367 explored, wired end-to-end through the channel action adapter.
