## Summary

Keep #12756 open. Current main still exposes the generic `read` message action, but Telegram does not advertise, map, gate, document, or implement a Telegram `read`/`readMessages` path. The related implementation PR #14367 was closed unmerged, and the latest maintainer/Codex discussion already treats #12756 as the canonical tracker.

## Best Possible Solution

Keep #12756 open and implement the feature in the bundled Telegram plugin: add `read` discovery, CLI/action mapping, runtime handling, config/docs/tests, and parameter compatibility for `openclaw message read --channel telegram --target ...`. The likely viable design is a bounded inbound message store like PR #14367 explored, because Telegram Bot API cannot fetch arbitrary per-chat history for bots.
