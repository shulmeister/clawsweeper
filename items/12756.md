## Summary

Keep #12756 open. Current main still has the shared `read` message action, but the bundled Telegram plugin does not advertise, map, dispatch, document, or test Telegram message reads. The prior implementation PR #14367 is closed unmerged and remains useful prior art, not shipped behavior; the latest maintainer/Codex issue comment also explicitly recommended keeping this issue open.

## Best Possible Solution

Keep #12756 open and implement Telegram `message read` end-to-end in the bundled Telegram plugin: advertise the action in discovery, map `read` to a runtime `readMessages` handler, add a bounded inbound message store or equivalent Telegram-safe source, align CLI/tool parameters (`--target`, `--limit`, `--before`, `--after`), add config/docs, and cover the contract/runtime behavior with focused tests. Use closed PR #14367 as implementation history, not as shipped proof.
