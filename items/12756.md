## Summary

Keep #12756 open. Current main still does not implement Telegram `message read`: the shared action name exists, but the bundled Telegram plugin neither advertises `read` nor maps it to a runtime handler, and the runtime has no Telegram read/inbound-store path. The prior PR #14367 is useful prior art for an inbound-store design, but the provided GitHub context says it was closed unmerged and current main does not contain that implementation.

## Best Possible Solution

Keep #12756 open. Implement Telegram `message read` end-to-end in the bundled Telegram plugin: advertise `read` in discovery, map the shared action to a Telegram runtime handler, store or otherwise expose recent inbound Telegram messages within Telegram platform limits, align CLI/tool parameters (`target`/`to`, `limit`, `before`, `after`, topic/thread handling), document limitations, add config/schema support if gating is needed, and cover adapter/runtime/contract behavior in focused tests. Treat closed PR #14367 as prior art, not shipped proof.
