## Summary

Keep #12756 open. Current main still has the generic `read` message action and CLI command, but the Telegram plugin does not advertise or implement Telegram `read`/`readMessages`. The related PR #14367 contained useful analysis and a possible inbound-store approach, but it was closed unmerged and the current code still lacks the requested end-to-end behavior.

## Evidence

- **Telegram discovery omits read:** Telegram message-tool discovery starts with `send` and conditionally adds poll, react, delete, edit, sticker, sticker-search, topic-create, and topic-edit. It never adds `read`, so agents/automation will not discover Telegram read support.
