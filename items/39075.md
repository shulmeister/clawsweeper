## Summary

Keep open. Current main still does not contain the central Telegram/runtime performance optimizations from PR #39075. The PR is stale and broad, but it remains useful prior art rather than obsolete: Telegram preflight is still sequential, command detection and envelope formatting lack the proposed fast paths/caches, typing keepalive still uses the old interval loop, and workspace bootstrap reads remain sequential.

## Best Possible Solution

Keep PR #39075 open as useful prior art, but ask for a focused rebase onto current main. Split Telegram-specific pipeline optimizations from shared runtime/agents changes, preserve current routing/config-refresh behavior, and add scoped tests or benchmarks for each optimization that still belongs.
