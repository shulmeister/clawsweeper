## Summary

Current `main` rebuilds workspace bootstrap context per run and defaults to injecting bootstrap files on every turn, so the reported stale bootstrap-after-gateway-restart behavior is implemented away on `main` (though not yet in the latest release tag).

## Evidence

- **Default behavior now re-injects bootstrap context:** `resolveContextInjectionMode()` defaults `agents.defaults.contextInjection` to `"always"`, making per-turn bootstrap injection the default rather than a one-time session-only behavior.
  - file: [src/agents/bootstrap-files.ts:53](https://github.com/openclaw/openclaw/blob/0a3da5cd8a6e712de6418008e01b11fb2afd7aea/src/agents/bootstrap-files.ts#L53)

## Risks / Open Questions

- This implementation is newer than the latest release tag `v2026.4.22`, so users on released builds may still see the old behavior until they update past commit `0a3da5cd8a6e712de6418008e01b11fb2afd7aea`.
