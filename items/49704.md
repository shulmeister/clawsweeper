## Summary

Current `main` does not contain this PR‚Äôs fix set. The isolated-cron delivery path still depends on already-loaded channel plugins to parse explicit thread targets, Telegram target parsing still lacks the legacy `chatId/topicId` form, and cron run results still do not carry a separate `deliveryError` field through to persisted state. This is not safe to close as implemented or obsolete.
