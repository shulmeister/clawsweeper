## Summary

Keep open. Current main has local cost estimates and some transcript/session cost plumbing, but it does not capture OpenRouter's provider-reported `usage.cost` / `cost_details` from streaming responses and expose that exact per-message cost to agent runtime. The linked ClawRouter workaround is useful prior discussion, and #7006/#9696 overlap, but neither supersedes this exact OpenRouter provider-reported cost request.

## Evidence

- **OpenRouter stream usage ignores provider cost fields:** `parseTransportChunkUsage` reads token counts from the streaming usage object, initializes cost to zero, then calls `calculateCost(model, usage)`. It does not read OpenRouter-specific `usage.cost` or `usage.cost_details`, so provider-reported exact cost is not preserved at the transport boundary.
