## Summary

Keep open. Current `main` still implements the reported behavior: when any agent has `agents.list[].heartbeat`, heartbeat enablement is restricted to only those explicit agents, even if `agents.defaults.heartbeat` is set. The same semantics are documented and covered by current tests, so this is not already fixed on `main` or in the latest release.
