## Summary

Keep open. Current main still has the behavior this PR targets: CLI profile setup derives `$OPENCLAW_HOME/.openclaw-<profile>`, while the generic config path resolver under `OPENCLAW_HOME` defaults to `$OPENCLAW_HOME/.openclaw`. The PR remains a focused implementation candidate, and the provided discussion shows later commits addressed the review feedback around sentinel `OPENCLAW_HOME` values and dev-profile coverage.

## Best Possible Solution

Keep this PR open for maintainer review. The likely path is to land a normalized version if `OPENCLAW_HOME` is intended to be the physical isolation boundary, with matching docs/help updates and a quick review of service install/start paths so CLI and daemon state resolution stay aligned.
