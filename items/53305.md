## Summary

Current `main` still treats memory flush as a pre-compaction path only. I found no session-end, idle-expiry, max-age-expiry, or day-boundary trigger that runs a flush for long-lived thread sessions, and the exposed config still lacks any periodic/session-end flush setting.

## Evidence

- **Memory flush is only invoked in the normal reply path:** `runMemoryFlushIfNeeded(...)` is called from the main reply runner immediately after preflight compaction handling; I did not find another call site for memory flush in reset, expiry, or thread-binding lifecycle code.
  - file: [src/auto-reply/reply/agent-runner.ts:1104](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/auto-reply/reply/agent-runner.ts#L1104)
  - command: `nl -ba src/auto-reply/reply/agent-runner.ts | sed -n '1098,1138p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Flush gating is compaction-threshold/transcript-size based, not session-end or periodic:** `runMemoryFlushIfNeeded(...)` computes `shouldFlushMemory` from projected token threshold or `forceFlushTranscriptBytes`, then runs the flush turn. There is no branch for session expiry, idle timeout, max-age timeout, or UTC day rollover.
  - file: [src/auto-reply/reply/agent-runner-memory.ts:700](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/auto-reply/reply/agent-runner-memory.ts#L700)

- **Public config still exposes only pre-compaction flush knobs:** `AgentCompactionMemoryFlushConfig` contains `enabled`, `softThresholdTokens`, `forceFlushTranscriptBytes`, `prompt`, and `systemPrompt`. There is no `periodicIntervalHours` or session-end flush option.
  - file: [src/config/types.agent-defaults.ts:481](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/config/types.agent-defaults.ts#L481)

The reset helper updates session ids/files and may delete old transcripts, but it does not invoke memory flush.
