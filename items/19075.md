## Summary

Keep #19075 open. Current main at 82262387651ab270b565f71533ce4937c9c00dc6 still lacks a configurable Brave Search baseUrl: the Brave runtime hardcodes both Brave API endpoints, the Brave plugin config/schema/docs expose only apiKey and mode, and previous Brave baseUrl PR attempts are closed unmerged while duplicate issues were closed back to this canonical tracker.

## Best Possible Solution

Keep #19075 open as the canonical bundled Brave Search baseUrl tracker. Implement plugin-owned config at plugins.entries.brave.config.webSearch.baseUrl, update the Brave manifest UI hints/schema, shared config type, docs/help, and focused tests, normalize and validate the URL, derive both /res/v1/web/search and /res/v1/llm/context from the resolved base while preserving the default Brave endpoint, and include the resolved base in the search cache key.
