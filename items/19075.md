## Summary

Keep open. Current main still does not implement a Brave Search baseUrl override: the bundled Brave runtime constructs both Brave request URLs from hardcoded api.search.brave.com endpoint constants, the Brave plugin manifest only accepts webSearch.apiKey and webSearch.mode, and the public Brave docs do not document any baseUrl option. The issue is the canonical tracker for this missing behavior, and the related PR #45843 is still only an open/unmerged implementation attempt.

## Best Possible Solution

Keep #19075 open as the canonical tracker until the bundled Brave plugin supports a provider-owned baseUrl. The likely fix belongs in extensions/brave: add plugins.entries.brave.config.webSearch.baseUrl to the manifest/UI hints and config types, normalize and validate it, construct both /res/v1/web/search and /res/v1/llm/context URLs from the normalized base while preserving the default api.search.brave.com behavior, include the base URL in the cache key, update https://docs.openclaw.ai/tools/brave-search, and add focused tests. PR #45843 or a refreshed successor is the natural implementation path.
