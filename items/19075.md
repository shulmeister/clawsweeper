## Summary

Keep open. Current main still does not implement a Brave Search baseUrl override, and #19075 remains the canonical issue for the missing provider-owned Brave proxy/custom-endpoint support. The related PR #45843 is an open implementation attempt, not shipped behavior.

## Best Possible Solution

Keep #19075 open as the canonical tracker until the bundled Brave plugin supports a provider-owned baseUrl. The likely fix belongs in extensions/brave: add plugins.entries.brave.config.webSearch.baseUrl to the manifest/UI hints and config types, normalize and validate it, construct both /res/v1/web/search and /res/v1/llm/context URLs from the normalized base while preserving the default api.search.brave.com behavior, include the base URL in cache identity, update https://docs.openclaw.ai/tools/brave-search, and add focused tests. PR #45843 or a refreshed successor is the natural implementation path.
