## Summary

Keep #19075 open. Current main still does not implement a configurable Brave Search baseUrl: the bundled Brave runtime hardcodes both Brave API endpoints, the Brave plugin schema/UI config only expose apiKey and mode, top-level tools.web.search remains strict without baseUrl, and the docs do not document any Brave proxy/base URL setting. Related PR attempts are closed unmerged, and duplicate reports were closed back to #19075, so this remains the canonical tracker.

## Best Possible Solution

Keep #19075 open as the canonical Brave Search baseUrl tracker. Implement it in the bundled Brave plugin by adding plugins.entries.brave.config.webSearch.baseUrl to the manifest/UI hints/config type, normalizing and validating it, deriving both /res/v1/web/search and /res/v1/llm/context URLs from the configured base while preserving the default Brave endpoint, including the resolved base URL in cache identity, updating https://docs.openclaw.ai/tools/brave-search, and adding focused schema/runtime tests. This is missing bundled Brave behavior, not a ClawHub-only request.

- Endpoint validation must preserve the existing trusted web-search endpoint guard and deliberately handle whether http loopback/private-network URLs are allowed.
