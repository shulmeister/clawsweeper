## Summary

Keep open. Current main has moved Ollama implicit discovery into the bundled Ollama plugin and now supports a canonical Cloud-only setup through `models.providers.ollama`, but it does not implement this PR's specific behavior for a separate explicit remote provider key such as `ollama-cloud` using `api: "ollama"`. With `OLLAMA_API_KEY` set and no `models.providers.ollama` entry, current discovery still builds the default local Ollama provider and can still emit the local unreachable warning.

## Best Possible Solution

Keep the PR open or ask the contributor to rebase/port it to current `extensions/ollama/src/discovery-shared.ts`. The best implementation is a current-plugin regression fix that suppresses ambient local Ollama discovery when another configured provider uses the native Ollama API with a non-loopback base URL, while preserving canonical local `ollama` warning behavior and adding coverage in `extensions/ollama/provider-discovery.test.ts`.
