## Summary

Keep #70472 open. Current main still has the compaction/context-overflow path that rotates the active session after failure, the checkpoint code still only warns when the session entry is missing, and the related PR #70479 is open/unmerged rather than shipped on main.

## Best Possible Solution

Keep the issue open and use #70479, or an equivalent rebased fix, as the implementation direction: preserve `agent:main:main` across overflow/compaction failures, fail the turn with explicit retry or `/compact` guidance instead of silently rotating, and add a regression test proving the active WebChat session id/store entry remains unchanged after compaction failure.
