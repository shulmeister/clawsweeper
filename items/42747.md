## Summary

Keep open. Current `main` does not implement Slack OAuth token rotation: the Slack config surface only supports static `botToken`/`appToken`/`signingSecret`/`userToken`, runtime account resolution only reads tokens from config or env, startup still requires those tokens directly, setup only prompts for bot/app tokens, and the Slack docs/UI still describe a static token model with no rotation status or init flow.

## Evidence

- **Slack config has no token-rotation fields:** `SlackAccountConfig` defines `signingSecret`, `botToken`, `appToken`, and `userToken`, but no `tokenRotation`, `refreshToken`, `expiresAt`, or refresh-buffer fields.
  - file: [src/config/types.slack.ts:109](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/config/types.slack.ts#L109)
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Secret registry only knows the four static Slack credentials:** Slack secret targets are limited to `appToken`, `botToken`, `signingSecret`, and `userToken` at top-level and per-account paths. There is no managed refresh token or expiry secret/state surface.
  - file: [extensions/slack/src/secret-contract.ts:11](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/extensions/slack/src/secret-contract.ts#L11)
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Account resolution is config/env only:** Resolved Slack accounts pull `botToken`, `appToken`, and `userToken` from merged config and env fallbacks, then record only `config`/`env`/`none` as sources. No refresh workflow or persisted rotation state exists here.
  - file: [extensions/slack/src/accounts.ts:45](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/extensions/slack/src/accounts.ts#L45)
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Runtime still requires static tokens at startup:** Slack monitor startup resolves `botToken` and `appToken` directly and throws if required tokens are missing. There is no background refresh loop or Slack OAuth refresh call on the runtime path.
  - file: [extensions/slack/src/monitor/provider.ts:126](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/extensions/slack/src/monitor/provider.ts#L126)
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Setup flow only captures bot/app tokens:** The Slack setup adapter and wizard require `--bot-token` and `--app-token`, and prompt only for those two credentials. There is no `openclaw slack token-rotation init` or equivalent bootstrap flow in the setup surface.
  - file: [extensions/slack/src/setup-core.ts:136](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/extensions/slack/src/setup-core.ts#L136)
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Docs and code search show static token model, no refresh implementation:** Slack docs still document a static token model (`botToken` + `appToken` or `signingSecret`) and probe fields for credential availability only. A repo search over Slack/config/commands/docs returned no matches for `tokenRotation`, `refreshBufferSeconds`, `oauth.v2.access`, or `oauth.v2.exchange`.
  - file: [docs/channels/slack.md:400](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/docs/channels/slack.md#L400)
