## Summary

Keep open. Current main has partial improvements that reduce the exact stale-lastGood fallback risk, but the issue’s remaining requested cleanup is not fully implemented: generic auth-profile rotation/upsert paths still do not clear stale `lastGood` or `usageStats`, and there is no supported `auth rotate` or `auth clear-cache` command. The issue is recent and has an active same-day follow-up, so it should not be conservatively auto-closed.

## Evidence

- **lastGood is no longer used to prioritize profile selection:** `resolveAuthProfileOrder` explicitly says `lastGood` is not prioritized and orders by profile type/lastUsed with cooldowned profiles pushed to the end. This is useful partial progress against the reported stale fallback mechanism, but it is not the whole requested fix.
