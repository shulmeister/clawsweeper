## Summary

Current main already avoids the stale-`lastGood` fallback described here: auth selection is driven by ordered eligible profiles, billing failures disable the failing profile, and merged auth stores repair stale profile references so `lastGood`/order point at the replacement profile ID. The same implementation is present in release `v2026.4.23`.

## Evidence
