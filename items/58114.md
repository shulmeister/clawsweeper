## Summary

Keep this issue open. Current main still has the gap described: the message-channel TTS path parses `[[tts:...]]` directives and forwards overrides, but the voice-call telephony stream path passes raw text into telephony synthesis with a static merged TTS config and no per-utterance overrides.

## Best Possible Solution

Implement this in the voice-call owner path by reusing the existing TTS directive parser and policy. A good fix would parse/strip directives before Twilio stream synthesis, preserve visible text behavior, and extend the telephony synthesis boundary to accept per-utterance provider overrides (or an equivalent per-call merged config) so `textToSpeechTelephony` can pass them into `prepareSpeechSynthesis`/`synthesizeTelephony`. Add focused tests in `extensions/voice-call/src/telephony-tts.test.ts` or Twilio provider tests plus a speech-core telephony override contract test.
