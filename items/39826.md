## Summary

Keep PR #39826 open. Current main still emits the Discord no-mention guild-drop structured log with only `channelId` and `reason`; the requested `accountId` and `guildId` values are available in the same preflight scope but are not included in that specific log payload. The PR is stale against the old `src/discord/...` path, but the underlying Discord plugin change remains valid in `extensions/discord/...`.

## Best Possible Solution

Keep the PR open or reapply the same log-only change on current main in `extensions/discord/src/monitor/message-handler.preflight.ts`: add `accountId: params.accountId` and `guildId: params.data.guild_id` to the no-mention `logger.info` payload, ideally with a focused test assertion for that structured log output.
