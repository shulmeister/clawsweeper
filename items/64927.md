## Summary

Keep open. Current `main` still enforces a generic critical install block for code that combines `process.env` access with network-send context, with no evidence of an exception for official/community plugins like Lossless Claw. User-facing docs still show plain `openclaw plugins install @martian-engineering/lossless-claw`, while the only documented escape hatch is the break-glass `--dangerously-force-unsafe-install` flag.
