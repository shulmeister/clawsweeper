## Summary

Keep #7359 open. Current main still keeps Slack DMs/main and Slack channel sessions as separate buckets. Message-tool sends can now be mirrored into the destination session, but default session-tool visibility remains tree-scoped, so a DM session still cannot read an unrelated Slack channel session unless broader visibility is explicitly configured. The existing related closed Slack issues address thread context/routing, not this default cross-context self-activity recall request.

## Best Possible Solution

Keep #7359 open. Implement an explicit same-agent Slack self-activity recall path, likely a scoped outbound activity ledger or session-history bridge keyed by agent, Slack account, target channel/thread, and sent message id. Expose that from DM context without broadly granting all cross-session history, and add regression coverage for sending from a DM to a Slack channel, then asking from the DM whether the agent sent it while unrelated channel history remains gated.
