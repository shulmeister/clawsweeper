## Summary

Keep #7359 open. Current main still routes Slack DMs/main sessions separately from Slack channel sessions, mirrors outbound `message` sends only into the destination Slack session, and default session-tool visibility does not give a DM/main session access to unrelated same-agent channel transcripts. The recent issue discussion also includes an explicit maintainer/Codex keep-open comment for this exact gap, and the related closed Slack thread-context issues do not supersede this cross-session self-activity request.

## Best Possible Solution

Keep #7359 open and implement a scoped Slack self-activity recall path. The safest shape is an outbound activity ledger or session-history bridge keyed by agent id, Slack account/team, destination channel/thread, and Slack message id/timestamp, exposed from DM/main context only for the agent's own outbound Slack sends. Add regression coverage for a Slack DM/main turn that sends a `message` tool post to a Slack channel, then asks from the DM whether the agent sent it and can recall the sent message without enabling broad cross-session visibility.
