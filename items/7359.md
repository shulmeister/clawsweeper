## Summary

Keep #7359 open. Current main still separates default Slack DMs from Slack channel sessions, and outbound Slack message-tool sends are mirrored into the resolved destination session rather than a DM-visible self-activity record. Default session-history visibility also remains limited to the current session tree, so it does not provide arbitrary same-agent channel recall from a DM.

## Best Possible Solution

Keep #7359 open and implement a scoped Slack self-activity recall path. The likely maintainable shape is an outbound activity ledger or session-history bridge keyed by agent id, Slack team/account, destination channel/thread, and Slack message timestamp, exposed from DM/main context only for the agent's own outbound Slack sends. Add regression coverage for a Slack DM/main turn that posts to a Slack channel through the message tool and later recalls that sent message without granting broad cross-session transcript visibility.
