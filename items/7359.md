## Summary

Keep #7359 open. Current main records outbound Slack message-tool sends into the destination Slack session, but Slack DMs and Slack channel sessions still remain distinct session buckets and the default session-tool visibility is `tree`, so a DM/main session cannot safely recall an unrelated Slack channel transcript by default. The earlier 2026-04-26 maintainer/Codex comment keeping the issue open still matches current main at d54d2d6.

## Best Possible Solution

Keep #7359 open. Implement a scoped same-agent Slack self-activity recall path, such as an outbound activity ledger or session-history bridge keyed by agent, Slack account, destination channel/thread, and sent Slack message id. Expose only the agent's own outbound activity from DM context by default, keep unrelated channel history gated, and add regression coverage for sending from a Slack DM to a Slack channel and then asking from the DM whether the agent sent it.
