## Summary

Keep this issue open. Current main still has the reported shape: isolated cron agent turns run through the embedded agent path, OpenAI-compatible completions tool-call argument chunks are appended directly into `partialArgs`, completed tool-call blocks retain that scratch field, and the existing malformed-argument repair does not accept a duplicated full JSON object suffix. The related context covers other cron/streaming failures, not this specific duplicate-JSON case.

## Best Possible Solution

Keep this open and fix the OpenAI-compatible streaming/tool-call boundary. The fix should detect cumulative or repeated complete argument payloads before appending, or deterministically repair duplicate balanced JSON objects, then add a focused regression test for an isolated `agentTurn` cron path using an OpenRouter/Kimi-shaped stream that emits the same complete arguments twice. Completed tool-call blocks should not expose malformed `partialArgs` scratch state.
