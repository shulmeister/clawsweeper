## Summary

Keep #8969 open. Current main has implemented the first quick win as `openclaw skills check`, but the requested plugin manifest-registry diagnostics in `openclaw status`/`--json`/`--all` and an operator-visible warning for model-routed skill-command prompt rewrites are still missing.

## Best Possible Solution

Keep #8969 open, or split it into two focused follow-ups. Preserve `openclaw skills check` as the shipped solution for the first quick win. Add manifest-registry diagnostics to `openclaw status`, `openclaw status --json`, and `openclaw status --all`, then emit an operator-visible diagnostic when a non-tool skill command rewrites the request for model-directed execution. Cover both with focused status and skill-command tests plus public docs updates.
