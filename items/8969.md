## Summary

Keep #8969 open. Current main implements the first quick win as `openclaw skills check`, but the other two requested operator diagnostics are still not on the requested core surfaces: `openclaw status` carries plugin compatibility notices, not manifest-registry diagnostics, and non-tool `/skill` invocations still rewrite the prompt without an operator-visible warning. The issue discussion also includes useful local validation/config/cost-report workarounds from the reporter, and the latest maintainer comment already identified the same remaining scope.

## Best Possible Solution

Keep the issue open, or split it into two focused follow-ups. The remaining product work is to surface manifest-registry diagnostics in `openclaw status` human and JSON output, and to emit an operator-visible warning/diagnostic when a non-tool `/skill` invocation rewrites the prompt for model-directed execution. Once those land with focused tests and docs, the umbrella issue can close as implemented.
