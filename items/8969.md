## Summary

Keep open. Current main appears to cover the first request with `openclaw skills check`, including JSON output and missing requirement reporting, but the remaining requested core diagnostics are still meaningful: `openclaw status` only renders plugin compatibility notices rather than manifest-registry loading diagnostics, and the non-tool `/skill` path rewrites the prompt silently without a warning. This is a clear core CLI/status/slash-command UX issue, not a ClawHub-only plugin request.
