## Summary

Keep #8969 open. Current main implements the first requested quick win as `openclaw skills check`, but the two remaining core operator diagnostics are still missing: `openclaw status` does not surface manifest-registry diagnostics, and non-tool skill command invocation still rewrites the prompt for model-directed execution without an operator-visible warning.

## Best Possible Solution

Keep the issue open, or split it into two focused follow-ups. The remaining core work is to add manifest-registry diagnostics to `openclaw status` human and JSON surfaces, and to emit an operator-visible warning or diagnostic when a non-tool `/skill` invocation rewrites the request for model-directed execution. The already-shipped `openclaw skills check` path should remain as the implementation for the first quick win.
