## Summary

Keep #8969 open. Current main implements the first quick win as `openclaw skills check`, but the two remaining operator diagnostics are still not implemented: `openclaw status` does not surface manifest-registry diagnostics, and non-tool skill command invocation still rewrites the prompt without an operator-visible warning.

## Best Possible Solution

Keep #8969 open, or split it into two focused follow-ups. Preserve `openclaw skills check` as the implementation for the first quick win. Add manifest-registry diagnostics to `openclaw status` human, JSON, and `--all` surfaces, and emit an operator-visible diagnostic when a non-tool `/skill` invocation rewrites the request for model-directed execution. Cover both with focused status/skill-command tests and update the public status/slash-command docs.
