## Summary

Current main intentionally keeps the gateway control plane on narrow, purpose-built RPCs (`config.*`, `logs.tail`, `gateway.identity.get`, `agents.files.*`) and documents broad state-dir filesystem exposure as a security risk. OpenClaw's project direction says optional capabilities like this should ship as plugins/ClawHub items, and the plugin SDK already supports custom gateway RPC methods or HTTP routes for a scoped implementation.
