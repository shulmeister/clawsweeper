## Summary

Current `main` already reports subagent completion details back to the requester session and preserves deleted subagent transcript usage for dashboard/session usage views. The implementation ships in `v2026.4.22`.

## Close Comment

Closing as implemented on `main` after Codex review. OpenClaw now pushes subagent completion details back into the requester session in `src/agents/subagent-announce.ts:449` and formats them with `session_key`, `session_id`, `status`, result text, and runtime/token stats in `src/agents/internal-events.ts:43` plus `src/agents/subagent-announce-output.ts:531`. Separately, deleted subagent transcripts are still counted for usage/cost reporting: archive files with `.jsonl.deleted.*` / `.jsonl.reset.*` are intentionally included in usage discovery in `src/config/sessions/artifacts.ts:37`, `src/infra/session-cost-usage.ts:457`, and surfaced through `sessions.usage` for dashboard/session views. This behavior is covered by regression tests in `src/infra/session-cost-usage.test.ts:245`. The implementation is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
