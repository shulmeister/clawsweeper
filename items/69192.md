## Summary

Keep open. Current main still returns many hook gateway 4xx responses without a hook warn/journal line, including invalid JSON, missing/invalid wake or agent payloads, mapping validation failures, unauthorized requests, and not-found/method errors. The related PR #69267 targeted this exact issue but is closed and unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open and implement a small gateway fix in `src/gateway/server-http.ts`: centralize hook rejection responses through a helper that emits a single structured `logHooks.warn` line with path, status, stable reason code, content/body metadata that avoids secrets or raw payloads, and a request id when available. Add focused tests that inject a mock `logHooks.warn` and cover invalid JSON, `/hooks/agent` validation, mapped-hook validation, auth/method/not-found, and the ambiguous `hook mapping requires message` path; consider splitting user-facing error text from stable internal reason codes in `src/gateway/hooks-mapping.ts`.
