## Summary

Keep #34400 open. Current main partially supports recursive memory discovery and the builtin watcher now watches directories, but the request is not fully implemented: nested dated memory files still miss temporal decay, dirty searches still sync in the background, QMD watch targets still combine collection paths with glob patterns, and several user-facing strings/docs still advertise flat `memory/*.md`.

## Best Possible Solution

Keep #34400 open and land a focused memory-core follow-up. The best path is to support nested dated memory paths in temporal decay, decide whether dirty `memory_search` should await sync for first-call freshness, repair or explicitly validate QMD collection watch targets under chokidar 5, update tool/prompt/config/docs wording from `memory/*.md` to recursive `memory/**/*.md`, and add regression tests for nested dated files plus QMD/builtin watcher target construction.
