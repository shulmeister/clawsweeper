## Summary

Keep #34400 open. Current main recursively discovers default memory files and indexes them, but the issue discussion identified remaining runtime and user-facing gaps that are still present on current main: nested dated memory files are not recognized by temporal decay, dirty builtin searches still start sync asynchronously, extraPath directory watching still uses recursive glob paths with chokidar 5, and several user-facing strings still describe only `memory/*.md`. The related PRs #34585 and #35691 were closed unmerged, so there is no shipped complete fix to close against.

## Best Possible Solution

Keep #34400 open and land a focused memory-core fix or maintainer decision. The likely path is to support nested dated memory paths in temporal decay, decide whether dirty builtin `memory_search` must await sync for first-call freshness, review extraPath watcher behavior under current chokidar, and align tool descriptions, prompt guidance, config help, CLI output, and public docs with the recursive behavior. Add regression coverage for nested dated paths and whichever freshness behavior is chosen.
