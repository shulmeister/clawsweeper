## Summary

Keep #34400 open. Current main recursively collects/indexes default memory directory files, but the issue discussion identified remaining runtime and user-facing gaps that are still present: dated files under memory subdirectories are not recognized for temporal decay, dirty search sync remains fire-and-forget, QMD watcher paths still include glob patterns, and prominent memory_search/memory_get guidance still says memory/*.md.

## Best Possible Solution

Keep #34400 open and either land a focused memory-core fix or have a maintainer explicitly split the remaining work. The likely implementation should update dated-memory path parsing for nested memory directories, decide whether first dirty search must await sync, replace QMD chokidar glob watch paths with directory roots plus filtering where needed, and align memory_search/memory_get prompt, config, CLI, and docs strings with the supported recursive behavior.
