## Summary

Keep #34400 open. Current main already recursively collects default memory files, but the issue discussion’s runtime and user-facing gaps remain: nested dated files under memory subdirectories are still treated as evergreen, dirty search sync is still fire-and-forget, and multiple tool/prompt/docs strings still describe only memory/*.md. The related fixing PRs in the provided GitHub context (#34585 and #35691) were closed unmerged, so there is no shipped fix to close against.

## Best Possible Solution

Keep #34400 open and land a focused memory-core fix or maintainer decision. The likely implementation is to support nested dated memory paths in temporal decay, decide whether first dirty search must await sync for freshness, review watcher paths that still combine collection path + glob patterns, and align tool descriptions, prompt guidance, config help, CLI output, and public docs with the shipped recursive behavior. Add regression coverage for nested dated files and the selected first-search freshness behavior.
