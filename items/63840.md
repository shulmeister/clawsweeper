## Summary

Keep open. Current main still lacks the requested Slack Agents & Assistants DM root-message thread preservation for non-all reply modes, and this PR remains a useful implementation candidate. It should not be closed as implemented or superseded, but the branch needs revision because the latest implementation gates on raw `channel_type` and includes an unrelated agents harness change.

## Best Possible Solution

Update this PR rather than close it: set `MessageThreadId` for Slack Agents & Assistants DM root messages using normalized direct-message detection, preserve existing channel/group top-level `thread_ts == ts` behavior for `off`/`first`/`batched`, add focused tests for `channel_type: "im"`, missing, and wrong D-prefix DM events, and remove or split the unrelated agents harness change.
