## Summary

Keep this issue open. Current main still scopes the `message` action enum and plugin-provided schema contributions, but the shared core schema groups are still added unconditionally, so the requested action-aware trim has not shipped. The existing ClawSweeper comment remains accurate against current main `9626ef274ae2346b3d519f99c31856be98aab570`.

## Best Possible Solution

Keep this issue open and implement action-aware core schema trimming in `src/agents/tools/message-tool.ts`: thread the resolved action set into the property builder, include shared core schema groups only when active actions can use them, preserve always-needed routing/send fields and cross-channel behavior, and add focused schema tests for minimal send/react-only and disabled-action configurations that assert unsupported core fields are absent while supported fields remain.
