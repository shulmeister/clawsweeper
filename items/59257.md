## Summary

Keep open. Current `main` contains partial hardening for isolated cron model selection, but it still explicitly warns and falls back to agent/default models when `payload.model` is rejected by the allowlist instead of failing loudly. That means the issue’s core safety complaint is not fully resolved, even though some related hidden-fallback and telemetry gaps have improved.

## Evidence

- **Rejected cron model override still succeeds with fallback:** `resolveCronModelSelection(...)` still returns `ok: true` and a warning when `resolveAllowedModelRef(...)` reports `model not allowed:`. It keeps the previously resolved agent/default model instead of failing the run.
  - file: [src/cron/isolated-agent/model-selection.ts:114](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/cron/isolated-agent/model-selection.ts#L114)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Current tests codify fallback-to-default behavior:** The isolated-cron test suite still expects a disallowed `payload.model` to warn and then use agent defaults, which matches the open safety gap rather than a full fix.
  - file: [src/cron/isolated-agent/run.skill-filter.test.ts:213](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/cron/isolated-agent/run.skill-filter.test.ts#L213)
