## Summary

Current main implements the requested cleanup. Compaction checkpoint persistence now tracks removed checkpoint metadata, unlinks trimmed `.checkpoint.<uuid>.jsonl` snapshots, and session disk-budget cleanup recognizes unreferenced checkpoint artifacts. Regression tests cover both immediate trim cleanup and disk-budget orphan cleanup. This appears to be after the latest tagged release, so no fixed release is identified yet.

## Best Possible Solution

Close as implemented on current main. Keep the main implementation that unlinks checkpoint snapshots as they fall out of retained metadata and lets disk-budget maintenance reclaim older unreferenced checkpoint artifacts; any remaining concern about default disk-budget policy or one-time historical cleanup should be tracked as a narrower follow-up.
