## Summary

Current main still does not provide a Slack upload option to preserve original image bytes. Slack outbound file sends call the optimized outbound media loader, Slack account config/docs expose only media size limits, and the only related implementation attempt, #53988, is closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep this issue open and implement the opt-out in the bundled Slack plugin. A conservative fix should add an explicit documented Slack config control, preserve the current optimized behavior by default, route uploads through a raw media path or an SDK optimizeImages=false option when disabled, and add schema/types/metadata/docs plus regression coverage proving PNG uploads can preserve original MIME/bytes while retaining maxBytes, SSRF, proxy, and local-root safeguards.
