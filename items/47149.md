## Summary

Keep open. Current main has general outbound Telegram chunking, but the requested announce fallback is not fully implemented: completion announce delivery can still return `didAnnounce=false` when the direct announce-agent delivery throws and no queue fallback is available, and the registry then retries/gives up rather than sending a guaranteed short completion notice.

## Evidence

- **Direct completion delivery failure remains observable:** The current regression test explicitly asserts that completion-mode announce returns false when direct delivery fails and queue fallback is unavailable.

- **Thrown direct announce only direct-fallbacks with thread content:** When the announce-agent call throws, `sendCompletionFallback` is called with content only for thread targets; non-thread channel/DM failures rethrow and are reported as an undelivered direct announce.
