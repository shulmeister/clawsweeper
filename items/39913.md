## Summary

Current main still has the reported inconsistency. Direct reply routing resolves and applies messages.responsePrefix, but message(action="send", ...) keeps the outbound text raw through message-tool handling, plugin/core dispatch, sendMessage, and gateway delivery. The prior Codex review in the issue reached the same conclusion, and the latest local source still matches that finding.

## Best Possible Solution

Keep #39913 open. Implement prefixing once in the shared message-tool outbound send boundary, resolving responsePrefix with agent, channel, and account context, applying the same idempotent text normalization used by direct replies before plugin/core/gateway dispatch, preserving media-only sends, and adding regression tests for runMessageAction, executeSendAction/sendMessage, and gateway send.
