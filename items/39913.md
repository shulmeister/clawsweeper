## Summary

Keep #39913 open. Current main still applies `messages.responsePrefix` on the normal reply path, but message-tool `send` continues to pass the raw message through plugin dispatch, core `sendMessage`, and gateway/direct delivery without resolving or applying the effective prefix. The prior Codex keep-open comment remains accurate against current main `e672b61417af5c45b0431df6d9109a1f4b618ef5`.

## Best Possible Solution

Keep #39913 open. Implement prefixing once at the shared message-tool outbound send boundary before plugin/core/gateway dispatch, resolving `responsePrefix` with agent, channel, and account context and using the same idempotent/template semantics as direct replies. Preserve media-only sends, cross-context decoration, mirrors/transcripts, plugin dispatch behavior, and add focused regression coverage for `runMessageAction`/`handleSendAction`, `executeSendAction`, and core/gateway send delivery.
