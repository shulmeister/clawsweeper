## Summary

Keep open. Current `main` appears to have fixed the heartbeat-driven stale-session problem, but the requested turn-count TTL is not implemented, and there is still no separate conversation timestamp or `heartbeatRefreshesSession`-style config in the session reset contract.

## Evidence

- **System events no longer trigger session expiry:** `initSessionState()` now treats `heartbeat`, `cron-event`, and `exec-event` as system events and bypasses freshness checks for them before session reuse is decided.
  - file: [src/auto-reply/reply/session.ts:254](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/auto-reply/reply/session.ts#L254)
  - command: `nl -ba src/auto-reply/reply/session.ts | sed -n '250,470p'`
- **Heartbeat no-reset behavior is covered by tests:** Tests explicitly assert that heartbeat runs do not reset stale sessions under idle or daily reset policies, while normal providers still do.
  - file: [src/auto-reply/reply/session.heartbeat-no-reset.test.ts:75](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/auto-reply/reply/session.heartbeat-no-reset.test.ts#L75)
  - command: `nl -ba src/auto-reply/reply/session.heartbeat-no-reset.test.ts | sed -n '1,220p'`
- **Docs say heartbeat-only replies restore `updatedAt`:** Heartbeat docs now state that heartbeat-only replies do not keep the session alive and that the previous `updatedAt` is restored so idle expiry still works.
  - file: [docs/gateway/heartbeat.md:268](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/docs/gateway/heartbeat.md#L268)
  - command: `nl -ba docs/gateway/heartbeat.md | sed -n '262,272p'`
- **Session reset schema still lacks requested feature:** `session.reset` only accepts `mode`, `atHour`, and `idleMinutes`; there is no `maxTurns`, `heartbeatRefreshesSession`, or equivalent field.
  - file: [src/config/zod-schema.session.ts:17](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/config/zod-schema.session.ts#L17)
  - command: `nl -ba src/config/zod-schema.session.ts | sed -n '1,80p'`
- **Freshness logic is still timestamp-only:** `evaluateSessionFreshness()` computes staleness only from `updatedAt`, `atHour`, and `idleMinutes`. There is no turn counter and no separate `lastConversationAt`/maintenance timestamp.
  - file: [src/config/sessions/reset-policy.ts:72](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/config/sessions/reset-policy.ts#L72)
