## Summary

Keep open. Current main still lacks a Slack shortcut listener for global shortcuts or message-action shortcuts, and the existing plugin surfaces do not appear to let a separate plugin subscribe to raw Slack Bolt shortcut payloads from the bundled Slack app. This is actionable in the Slack plugin rather than closeable as implemented or ClawHub-only.

## Best Possible Solution

Implement this in `extensions/slack` as a Slack-owned interaction path: add a shortcut event module, register `ctx.app.shortcut` for global and message-action shortcuts, apply the same mismatch/auth/session routing policy used by existing Slack system events, enqueue a structured shortcut system event with stable context keys, and update Slack interaction tests plus `docs/channels/slack.md`. If maintainers want this handled by external plugins instead, first add a typed Slack shortcut extension API; the current plugin surfaces only cover generated button/select interactions and do not provide raw Bolt shortcut subscription.
