## Summary

Current `main` still does not implement per-agent `agents.list[].compaction` overrides. The agent-entry type/schema do not accept `compaction`, compaction remains exposed under `agents.defaults`, and runtime compaction setup still reads global defaults without an agent-specific merge path. Related PRs #14598 and #19329 were closed unmerged, so they do not supersede the issue.

## Best Possible Solution

Keep the issue open. The likely implementation should add an optional `compaction` object to `agents.list[]`, validate it in `AgentEntrySchema`, type it in `AgentConfig`, resolve it by merging per-agent values over `agents.defaults.compaction` including nested `memoryFlush`, and thread the effective agent id through reserve-token, safeguard, compaction provider, memory-flush, and related runtime call sites with focused resolver/schema tests. Closed PRs #14598 and #19329 are useful prior art but do not close the request.
