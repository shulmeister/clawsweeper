## Summary

Keep #13736 open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still does not implement per-agent compaction overrides: the agent list type/schema do not accept `compaction`, the resolved agent config does not merge it, docs describe defaults-only compaction, and runtime compaction setup still reads `agents.defaults.compaction`. Related PRs #14598 and #19329 were closed unmerged, while #69988 is an open implementation path rather than shipped behavior.

## Best Possible Solution

Keep #13736 open unless maintainers explicitly designate a canonical replacement such as #69988. The implementation should add typed and strict-schema support for optional `agents.list[].compaction`, deep-merge it over `agents.defaults.compaction` including nested `memoryFlush`, thread the effective agent-specific config through reserve-token, safeguard/runtime, memory-flush, and embedded setup paths, and add focused schema/resolver/runtime tests plus config docs.
