## Summary

Current main still does not implement per-agent `agents.list[].compaction` overrides. The type/schema only expose compaction under `agents.defaults`, runtime compaction paths still read defaults-only config, and the related implementation PRs were closed unmerged, so this issue remains actionable.

## Best Possible Solution

Keep the issue open. A good implementation should add optional `agents.list[].compaction` typing and validation, reuse or extract the defaults compaction schema, resolve effective per-agent compaction by merging `agents.list[]` values over `agents.defaults.compaction` including nested `memoryFlush`, thread the effective agent id/config through reserve-token, safeguard, memory-flush, compaction-provider, and runtime call sites, and add focused resolver/schema/runtime tests plus docs. Closed PRs #14598 and #19329 are useful prior art but are not shipped.
