## Summary

Keep #13736 open. Current main still only supports compaction configuration under `agents.defaults.compaction`; `agents.list[]` entries do not type or validate a `compaction` field, resolver/runtime paths still read defaults-only compaction settings, and docs/generated config help only expose defaults compaction. The related implementation history is useful context, but PRs #14598 and #19329 are closed unmerged and PR #69988 is still an open candidate path rather than shipped main behavior.

## Best Possible Solution

Keep #13736 open until the feature lands on main or maintainers explicitly designate a canonical replacement. The implementation should add optional `agents.list[].compaction` to the public agent config type and strict schema, reuse the defaults compaction schema, deep-merge agent compaction over `agents.defaults.compaction` including nested `memoryFlush`, thread the effective config through Pi reserve-token setup, safeguard/runtime compaction settings, compaction model selection, and memory-flush/preflight paths, then update docs/generated config metadata and focused schema/resolver/runtime tests.
