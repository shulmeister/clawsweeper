## Summary

Keep PR #39888 open. Current main still does not emit a trusted `model.usage` diagnostic event from the `openclaw agent` CLI path, while the reply-runner path does and diagnostics plugins depend on that event for token/cost telemetry. The PR cannot merge as-is because it targets the now-obsolete `src/commands/agent.ts` implementation, but the underlying behavior gap remains valid and the PR’s second commit addressed useful review feedback around `channel: "cli"` and context-token fallback.

## Best Possible Solution

Refresh this PR or replace it with a small current-main change in `src/agents/agent-command.ts` or a shared helper after `updateSessionStoreAfterAgentRun`. Emit a trusted `model.usage` diagnostic for CLI/local agent runs with `channel: "cli"`, session/agent/provider/model metadata, usage and last-call usage, context limit/used, cost, duration, and a focused regression test while preserving reply-runner behavior.
