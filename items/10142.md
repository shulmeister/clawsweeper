## Summary

Keep open. Current main still does not emit the requested internal `session:end` hook event for HOOK.md/internal hook handlers. It has related typed plugin lifecycle hooks (`session_end`) and internal `session:compact:*`/`session:patch` events, but those are separate surfaces and do not satisfy the issue’s requested internal session completion event. The existing maintainer review comment also explicitly kept this open, and the related implementation PRs are either closed unmerged or still open rather than shipped on main.

## Best Possible Solution

Continue tracking this as an OpenClaw core/internal hooks feature until maintainers either implement `session:end` in the internal hook system or explicitly decide that plugin `session_end` is the supported product direction. A complete implementation should wire `createInternalHookEvent("session", "end", ...)` at the relevant session completion paths, define/test the event context, and update `docs/automation/hooks.md`; if the intended answer is plugin-only, document that direction and close with maintainer product judgment rather than automated cleanup.
