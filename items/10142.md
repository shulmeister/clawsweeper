## Summary

Keep #10142 open. Current main still does not provide an internal `session:end` HOOK.md event; the adjacent shipped surface is the typed plugin `session_end` hook, and the related implementation attempts in the provided context are either closed unmerged (#6885, #37103) or still open and adjacent (#56672, #61675). The latest maintainer comment also explicitly kept this issue open for the same boundary reason.

## Best Possible Solution

Keep #10142 open until maintainers either implement a documented internal `session:end` hook contract or explicitly decide that typed plugin `session_end` is the supported replacement for this workflow-integration use case. A complete internal-hook path should add typed event/guard coverage, emit from gateway agent, cron isolated-agent, reset/delete, and subagent completion paths as appropriate, and update tests plus the public hooks docs.
