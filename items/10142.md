## Summary

Keep open. Current main still does not implement or document the requested internal `session:end` hook event for `HOOK.md`/internal hook handlers. Main has adjacent plugin lifecycle support through `session_end`, plus internal `session:compact:*` and `session:patch`, but those are separate surfaces and do not satisfy this issue's requested internal session completion event. The existing maintainer review comment also explicitly kept #10142 open, and the related implementation PRs in the provided context are either closed unmerged or still open.

## Best Possible Solution

Continue tracking #10142 as a core/internal hooks feature until maintainers either implement `session:end` in the internal hook system or make an explicit product decision that plugin `session_end` is the supported direction. A complete core implementation should define the event contract, emit `createInternalHookEvent("session", "end", ...)` from the relevant completion paths, add regression coverage, and update `docs/automation/hooks.md`; if plugin-only is intended, document that migration path and close with maintainer judgment.
