## Summary

Keep this issue open. Current main appears to handle the runtime auto-enable safety case, but the requested BlueBubbles-specific documentation and composed safe local example are still missing; the guidance is only available indirectly across generic plugin and gateway docs.

## Best Possible Solution

Add a focused section to docs/channels/bluebubbles.md that compares channels.bluebubbles.enabled=false with plugins.entries.bluebubbles.enabled=false, explains which layer controls channel startup versus plugin activation/runtime dependency repair, includes a JSON5 safe local example with gateway.mode="local", gateway.bind="loopback", gateway.auth.mode="token", channels.bluebubbles.enabled=false, and plugins.entries.bluebubbles.enabled=false, and links to the public plugin and gateway configuration docs while briefly stating what startup can repair or mutate.
