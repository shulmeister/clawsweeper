## Summary

Keep #8081 open. Current main has useful partial controls such as per-agent sandbox/tool policy, owner-only surfaces, trusted-proxy identity allowlists, and operator scopes, and the discussion already links external wrappers like Pinchy, teamclaw, and openclaw-mission-control. Those do not implement the requested shared-gateway admin/standard/guest RBAC, per-user credential visibility/isolation, or audit model in OpenClaw core. The latest maintainer-authored issue comment also explicitly keeps this tracker open.

## Best Possible Solution

Keep #8081 open as the broad product/security tracker. A complete path should define gateway/channel caller identity, role- or scope-based authorization for sensitive config and credential operations, per-user or per-tenant secret isolation, audit logging, channel/group identity mapping, documentation of the trust boundary, and regression coverage across gateway auth, config read/write paths, HTTP APIs, and channel command surfaces. If maintainers choose an external-wrapper or ClawHub-first direction, document that explicitly and identify any missing plugin/API surfaces needed to support it outside core.
