## Summary

Keep #8081 open. Current main still does not implement the requested multi-user RBAC/auth/secrets-isolation model; it documents a trusted-operator personal-assistant boundary with partial hardening controls. The issue discussion also has an explicit maintainer review keeping it open, plus external wrapper examples and unmerged/closed related PRs that do not ship the core behavior.

## Best Possible Solution

Keep #8081 open as the broad product/security tracking issue. A core solution should define an explicit identity and authorization model for gateway/API/channel callers, role- or scope-based permissions for sensitive config and credentials, per-user secret isolation, audit logging, and updated docs/tests for the personal-assistant trust model. If maintainers choose the external-wrapper or ClawHub path, document that direction and confirm which plugin/API surfaces are sufficient or still missing.

## Risks / Open Questions
