## Summary

Keep #8081 open. Current main still does not implement the requested multi-user RBAC/auth/secrets-isolation system; it documents a one-trusted-operator gateway model with partial hardening around gateway auth, operator scopes, DM isolation, sender tool policies, and multi-user warnings. The issue discussion also has an explicit maintainer Codex review from 2026-04-26 keeping it open, plus external wrapper projects and prior unmerged PRs that show demand but do not ship the core behavior.

## Best Possible Solution

Keep #8081 open as the broad product/security tracking issue. A core solution should define an explicit identity and authorization model for gateway/API/channel callers, role/scoped permissions for sensitive config and credentials, per-user secret isolation, audit logging, and docs/tests that update the personal-assistant trust model. If maintainers later choose the external-wrapper path, document that direction and confirm no missing plugin/API surface blocks it.
