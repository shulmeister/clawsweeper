## Summary

Keep #8081 open. Current main has useful lower-level controls, but it still does not implement the requested shared-gateway admin/user/guest RBAC, per-user credential isolation, or audit model. The issue discussion also contains a maintainer-authored Codex review from 2026-04-26 explicitly keeping this tracker open; linked external projects such as Pinchy, TeamClaw, and openclaw-mission-control are useful reference/workaround implementations, not a replacement for a core product/security decision.

## Best Possible Solution

Keep #8081 open as the broad product/security tracker. A complete path should define gateway and channel caller identity, role/scope authorization for sensitive config and credential operations, per-user or per-tenant secret isolation, audit logging, channel/group identity mapping, documented trust boundaries, and regression coverage across gateway auth, config read/write, HTTP APIs, and channel command surfaces. External projects linked in the thread can inform the design, but they should not close the core tracker unless maintainers explicitly choose an external-wrapper or ClawHub-first direction and identify any missing plugin/API surfaces.

## Risks / Open Questions

- The request is broad and product/security-sensitive; it likely needs maintainer design before implementation or an explicit decision to split core API work from external-wrapper/plugin work.
