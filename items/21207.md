## Summary

Keep #21207 open. Current main has adjacent pieces, including provider `parallel_tool_calls`, context-engine lifecycle/compaction, `update_plan`, and strict-agentic plan-only retry handling, but it still does not implement the requested native batched file/search APIs, AI-callable named context block save/load/unload/list tools, or submitted-plan step enforcement API.

## Best Possible Solution

Keep #21207 open and split the work into concrete maintainer tracks: native or gateway-level batching for file/search tools, an explicit AI-callable named context block API if save/load/unload semantics are desired, and runner-owned opt-in plan state with strict step gating if enforceable sequential execution is a product goal. Treat the existing `parallel_tool_calls`, context-engine lifecycle/compaction, `update_plan`, and strict-agentic plan-only retries as groundwork rather than closure.
