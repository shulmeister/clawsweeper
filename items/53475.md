## Summary

Keep open. Current main still treats a direct external SIGTERM without OpenClaw's restart-intent marker as a graceful stop, while the LaunchAgent plist delegates availability to launchd KeepAlive and the macOS integration tests still do not cover the reporter's raw SIGTERM -> new PID -> healthy gateway path.

## Best Possible Solution

Keep this issue open. Add a macOS launchd integration reproduction that installs a test Gateway LaunchAgent, reads the launchd runtime PID, sends SIGTERM to that PID, and asserts launchd returns to `state = running` with a different PID and a healthy gateway endpoint. If that fails, fix the macOS launchd lifecycle or document the exact GUI-session/on-demand-domain limitation; if it passes on current main, close later with direct proof.
