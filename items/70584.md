## Summary

Keep open. Current main still has the PR's reported behavior: Claude Opus 4.7 is treated as an adaptive-thinking Anthropic Messages model, but `minimal` and `low` still map to `output_config.effort: "low"`. The adaptive path then emits that effort into the request payload, including the GitHub Copilot provider path. Current tests cover Opus 4.7 `xhigh`, but not the low/minimal rejection this PR targets. The linked bug #70322 is therefore not implemented on main, and the PR remains useful, though it should be reconciled with the related #69928 report about whether Copilot accepts only `medium` or also `high`/`xhigh`.

## Best Possible Solution

Keep #70584 open. The best path is a targeted Anthropic Messages compatibility fix in `src/agents/anthropic-transport-stream.ts` with regression tests for Opus 4.7 low/minimal and unaffected Opus 4.6 behavior, then verify whether Copilot Opus 4.7 supports `high`/`xhigh` or should be clamped more broadly to `medium`. Do not close as implemented or duplicate; current main still emits the reported unsupported low effort.
