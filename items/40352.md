## Summary

Keep #40352 open. Current main still has the reported Control UI stale model-selection paths: explicit Agents reload still calls `loadConfig(state)` without clearing dirty config form state, `applyConfigSnapshot` still refuses to replace `configForm` while dirty, and the Agents model `<select>` still relies on Lit `.value` without marking the matching `<option>` selected. The issue is concrete, non-maintainer-authored, and only has the `stale` label; the linked fixes (#40441, #40443, #52948) are useful evidence but remain open rather than shipped on main.

## Best Possible Solution

Keep the issue open and land one canonical fix on main. The likely product path is to add explicit reload semantics that clear/discard stale config form state for the Agents/Config/Channels reload buttons, add option-level `?selected` bindings for the affected `<select>` renderers, and cover the Agents model dropdown with a regression test showing the configured primary model after initial render and after Reload Config.
