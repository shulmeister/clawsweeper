## Summary

Keep open. Current main has meaningful partial progress, but this recent consolidated meta issue still contains multiple unsolved core/security/ops requests, and the only explicit follow-up in the discussion (#70976 / PR #70990) covers just the model-failover hook slice of item #1.

## Best Possible Solution

Keep #65824 open for maintainer triage, but split it into narrower canonical issues. Mark already-covered slices against #70976/#70990 or the existing hook surfaces, then decide focused core work for non-verbose fallback visibility/context reinjection, config health rollback, crash recovery, doctor dry-run/diff, path-level file policy, subagent failure circuit breaking, budget gates, and scheduled model health routing. Optional integration/tooling ideas should move to plugin or ClawHub/API design only after core extension seams are confirmed sufficient.
