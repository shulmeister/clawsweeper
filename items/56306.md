## Summary

Current main does not support attachment delivery via a successful `read(file=...)` call; it delivers Telegram media through the outbound `mediaUrl` / `MEDIA:` path, and that path is implemented and covered by tests. The report’s reproduction therefore does not match current Telegram send behavior on main.

## Evidence
