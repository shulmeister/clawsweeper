## Summary

Keep #69319 open. Current main still lacks a `skills.uninstall` Gateway RPC in the method registry, handler surface, protocol schema, scope map, and Gateway docs. The PR addresses the same real operator-UI gap described in #69299, but it should be revised before merge because review comments flagged destructive non-ClawHub deletion and invalid-slug error classification.

## Best Possible Solution

Keep #69319 open, or replace it with a maintainer-owned patch that adds `skills.uninstall` through the Gateway protocol, method registry, `operator.admin` scope, handler, generated client models, docs, and focused tests. The mergeable implementation should delete only ClawHub-tracked skill folders or lock entries, treat malformed slugs as client errors, preserve idempotent not-installed behavior, and cover the reviewed edge cases in backend and handler tests.
