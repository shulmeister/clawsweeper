## Summary

Do not close PR #69201. Current main still derives `sessions_spawn` child routing from the live tool/request context only, while the PR targets the unresolved case where that context is empty but the parent session entry already has `deliveryContext.to` and `deliveryContext.threadId`. The PR discussion also shows active, substantive review and follow-up fixes through the latest head, not an obsolete or low-signal drive-by.

## Best Possible Solution

Keep the PR open for maintainer review or land an equivalent core fix: backfill missing delivery fields from the parent session before subagent/ACP requester-origin resolution, keep explicit ctx values authoritative, guard channel/to/root-level cases, and include persisted child-session regression coverage.
