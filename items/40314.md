## Summary

Keep open. Current main still triggers `message:received` and `message:sent` internal hooks but does not deliver text appended to `hookEvent.messages`, while the docs describe `event.messages.push(...)` as the way a hook sends a message to the user. PR #40314 is a focused fix attempt for open bug #39688, but it is not implemented on main and the provided review comments show follow-up hardening is still needed rather than cleanup closure.

## Best Possible Solution

Keep the PR or a canonical successor open, rebase it onto current main, and finish the implementation by collecting `hookEvent.messages` for `message:received` and `message:sent`, routing those replies with session context, suppressing message-hook re-emission, persisting any suppression flag needed for queued recovery, and adding regression tests for both delivery paths and loop prevention.
