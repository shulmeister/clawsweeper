## Summary

Keep open. Current main still has the Feishu-specific docs gap this PR targets, but the submitted wording is not safe to auto-close or treat as already implemented because routing code treats omitted configured `match.accountId` as the literal `default` account, while Feishu `defaultAccount` is separate account-selection behavior.

## Best Possible Solution

Keep the PR open and revise the Feishu docs wording. The best fix is to document the actual configured-binding contract precisely: for hand-written bindings, omitted `match.accountId` matches the literal `default` account id in the generic matcher; use an explicit Feishu account id such as `main` for a named account, or `accountId: "*"` to match any configured Feishu account. Coordinate with the broader binding docs if #49769 or a successor becomes the canonical cleanup.
