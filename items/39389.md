## Summary

Keep PR #39389 open. Current main still has the Feishu-specific docs gap this PR targets, and the submitted patch remains useful, but its wording should be revised before merge because Feishu `defaultAccount` is outbound/default-selection behavior while binding omission currently indexes against the literal `default` account key.

## Best Possible Solution

Keep the PR open and revise the Feishu docs patch. The best version should document the actual contract: omitted `match.accountId` matches the literal `default` binding account key, explicit Feishu account ids such as `main` match named accounts, and `accountId: "*"` matches any configured Feishu account. Coordinate with #49769 for broader CLI/type/docs wording, but keep Feishu-specific guidance on the Feishu channel page.
