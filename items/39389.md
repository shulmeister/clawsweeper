## Summary

Keep PR #39389 open. Current main still lacks the Feishu-channel documentation this PR targets: the Feishu routing section lists channel and peer fields but not `match.accountId` or `accountId: "*"`. The submitted change remains useful, but its wording should be revised because Feishu `defaultAccount` is documented and implemented as default account selection, while routing omission currently indexes against the literal `default` account key.

## Best Possible Solution

Keep the PR open and revise the Feishu docs patch. The best version belongs in `docs/channels/feishu.md`: document that omitted `match.accountId` follows the routing contract for the literal `default` account key, explicit Feishu account ids such as `main` match named accounts, and `accountId: "*"` matches any configured Feishu account. Also clarify that `channels.feishu.defaultAccount` is default account selection for outbound/default account resolution and should not be conflated with inbound binding omission. Coordinate wording with #49769 if that broader PR proceeds, but keep the Feishu-specific guidance on the Feishu channel page.
