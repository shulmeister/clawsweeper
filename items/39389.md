## Summary

Keep PR #39389 open. Current main still has the Feishu-specific docs gap: the Feishu routing section lists channel and peer fields but does not document `match.accountId` omission or `accountId: "*"`. The requested semantics are real in current routing code and tests, and Feishu inbound routing passes the concrete runtime account id, so this is not implemented on main or safely superseded. The PR should be revised, though, because wording like “default account” is ambiguous beside `channels.feishu.defaultAccount`, which the Feishu docs describe as outbound/default API selection.

## Best Possible Solution

Keep the PR open and revise the Feishu docs patch. Add `match.accountId` to the Feishu routing fields, explain that omitted means the literal default routing scope/account id (`default`) rather than all accounts, explicit ids such as `main` match that named Feishu account, and `accountId: "*"` matches any configured Feishu account. Also distinguish this from `channels.feishu.defaultAccount`, which controls outbound/default account selection. Coordinate wording with #49769, but keep a Feishu-specific example on `docs/channels/feishu.md`.

## Risks / Open Questions

- Merging the current wording exactly could make users with `channels.feishu.defaultAccount: "main"` expect a binding without `match.accountId` to match inbound events from the named `main` account, which current routing does not do.
