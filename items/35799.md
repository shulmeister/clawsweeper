## Summary

Keep #35799 open. Current main partially addresses the heartbeat herd problem with an internal stable per-agent phase, but the requested user/product surface is still missing: no prime cadence mode, no exposed heartbeat phase controls, and cron `every` schedules remain fixed intervals without jitter/stagger support.

## Best Possible Solution

Keep the issue open and narrow or implement the remaining product surface deliberately: preserve the internal stable heartbeat phase, add backward-compatible heartbeat controls for deterministic/user-selectable phase and optional prime cadence if maintainers want them, and extend cron `every` type/schema/normalizer/CLI/UI/status/docs/tests with explicit jitter/stagger semantics rather than relying on cron-expression-only staggering.
