## Summary

Keep #39400 open. Current main still exposes provider quota in `session_status` as an optional human-readable status line, not as stable structured usage fields with explicit availability, source, or reset metadata. Codex Day/Week window normalization exists, but the monitor-facing contract requested by the issue is still missing.

## Best Possible Solution

Keep the issue open. If cron/monitor use of `session_status` is supported, extend the provider usage snapshot/status contract with a structured `usage` object in `session_status.details`, including provider/source, availability or unavailable reason, normalized windows with percent/resetAt, and reset explanation where known. Preserve the existing human-readable status line and add focused tests for no auth, fetch error, empty window, timeout, and Codex 5h/Day/Week mapping behavior.
