## Summary

Keep #39400 open. Current main still does not provide the requested stable structured `session_status` usage contract: provider quota windows are fetched and formatted for a human status card, but the tool result `details` only returns `statusText` plus basic metadata. The open related #8635 is relevant programmatic usage work, but it does not fully supersede this provider-window availability/reset-metadata request.

## Best Possible Solution

Keep #39400 open. If cron/monitor use of `session_status` is supported, add structured usage data to the tool result `details`, including provider/source, explicit availability or unavailable reason, normalized 5h/day windows with percent and `resetAt`, and reset explanation where upstream data allows it. Preserve the existing human-readable status card and coordinate with #8635 only if that work explicitly covers provider quota-window availability and reset metadata.
