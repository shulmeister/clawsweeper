## Summary

Keep PR #70391 open. Current main still performs the redundant `chat.history` reload after a final chat event even when that event carries an assistant message, so the PR is not implemented on main. The PR needs maintainer review and likely revision for the noted deferred `session.message` and role-validation edge cases, but no conservative close reason applies.

## Best Possible Solution

Keep the PR open and revise it into a narrow UI fix: skip the final-event `chat.history` reload only when the final event carries a validated displayable assistant message that the UI already appended, while preserving deferred `session.message` replay semantics for unrelated transcript updates. Add/keep targeted tests in `ui/src/ui/chat-event-reload.test.ts` and `ui/src/ui/app-gateway.node.test.ts`.
