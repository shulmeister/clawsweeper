## Summary

Keep this PR open. Current main still computes the preflight compaction and memory-flush token gates from `reserveTokensFloor` only, while config help and the Pi settings path treat `reserveTokensFloor` as a floor under `reserveTokens`. The PR remains the active implementation candidate for #66830, and the related earlier PR #67297 is closed unmerged.

## Best Possible Solution

Keep this PR open as the canonical implementation path for #66830. Land this helper or an equivalent `max(reserveTokens, reserveTokensFloor)` calculation, thread it through the preflight compaction and memory-flush gates, keep the focused regression coverage and changelog entry, then close #66830 after merge.
