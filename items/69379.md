## Summary

Keep PR #69379 open. Current main still computes both preflight compaction and memoryFlush thresholds from `reserveTokensFloor` only, while docs and the Pi settings path treat `reserveTokensFloor` as a floor for the effective `reserveTokens`. The related duplicate PR #67297 is closed unmerged and the linked bug #66830 remains open, so this PR is not implemented or superseded.

## Best Possible Solution

Keep #69379 open for maintainer review or land an equivalent fix. The best implementation direction is to compute a single effective reserve for preflight and memoryFlush gates as `max(agents.defaults.compaction.reserveTokens, reserveTokensFloor)`, preserve the existing `MemoryFlushPlan` public contract if possible, and add focused regression coverage for `reserveTokens > reserveTokensFloor` plus the threshold-clamp edge case.
