## Summary

Keep open. Current main has runtime and documentation support for .env loading, ${VAR} substitution, and structured SecretRef objects, but the central requested UX remains incomplete: the Control UI config form still renders env SecretRefs as a generic read-only “Structured value (SecretRef)” placeholder, does not show config/env/unset source metadata or a “From env” badge/id, and its analyzer explicitly supports only the string editing path for now. The related #55034 context is a narrower redaction round-trip bug, not a merged or canonical replacement for this first-class env-backed secret UX request.
