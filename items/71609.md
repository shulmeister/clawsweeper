## Summary

Keep this issue open. Current main still leaves the Control UI browser path able to reuse stale stored device credentials after `AUTH_DEVICE_TOKEN_MISMATCH`, and the reconnect/message/docs behavior has not been updated to provide stale-token cleanup or non-looping recovery.

## Best Possible Solution

Fix the Control UI browser auth recovery path in `ui/src/ui/gateway.ts`: clear stale cached browser device auth on `AUTH_DEVICE_TOKEN_MISMATCH` for the stored-token primary path, avoid unbounded reconnects when no safe retry remains, preserve explicit shared token/password precedence, update the mismatch wording and public troubleshooting guidance toward stale-identity recovery, and add focused `GatewayBrowserClient` regression tests for scope-upgrade/device-token-mismatch cleanup and non-looping recovery.
