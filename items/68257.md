## Summary

Keep open. Current main still has no top-level `auth` reload rule, so `auth.profiles.*` remains unmatched and falls through to the conservative restart path. The PR is a focused gateway bug fix with concrete reproduction details, linked issue #68258, and an independent confirmation for `auth.profiles.microsoft:<email>` on OpenClaw 2026.4.11.

## Best Possible Solution

Keep the PR open and land a focused gateway reload-plan fix, with regression tests proving `auth.profiles.<provider>:<email>` does not restart the gateway while `gateway.auth.*` continues to restart. If maintainers are uncomfortable with a broad `auth` no-op rule, adjust the patch to a narrower `auth.profiles` rule rather than closing the item.
