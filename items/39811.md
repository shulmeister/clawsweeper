## Summary

Keep #39811 open. Current main still accepts arbitrary configured model refs at schema/config-validation time, `openclaw config validate` does not perform catalog-aware verification, and selection/listing paths intentionally preserve configured refs that are absent from the catalog. The related fallback-notification work is partly addressed elsewhere, and PR #40355 is a plausible implementation path, but it is open/unmerged and not shipped on current main.

## Best Possible Solution

Keep #39811 open. The best path is to add catalog-aware warning or validation for configured model refs across defaults, per-agent overrides, fallbacks, channel/hook/tool surfaces, aliases, and provider-owned model entries. PR #40355 can be used as a starting point if rebased and broadened to the current doctor/config-validation structure, while preserving forward-compatible/custom-provider refs as warnings rather than hard errors where appropriate.
