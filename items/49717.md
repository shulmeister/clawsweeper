## Summary

Close as implemented on main: the Feishu inbound path already runs message handling in fire-and-forget mode and applies persistent per-message dedupe, and the latest release v2026.4.22 contains the same runtime behavior.

## Evidence

- **Inbound handler returns before long-running work:** The Feishu message handler claims the message ID, starts `processMessage()` in fire-and-forget mode, and returns without awaiting downstream handling when `fireAndForget` is enabled. That is the immediate-ack pattern the issue asks for.
  - file: [extensions/feishu/src/monitor.message-handler.ts:323](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/extensions/feishu/src/monitor.message-handler.ts#L323)

- **Feishu monitor enables fire-and-forget by default:** `monitorSingleAccount()` wires the registered Feishu handlers with `fireAndForget: params.fireAndForget ?? true`, so the default WebSocket path does not wait for full message processing before returning control to the SDK dispatcher.
  - file: [extensions/feishu/src/monitor.account.ts:662](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/extensions/feishu/src/monitor.account.ts#L662)

## Close Comment

Closing from a Codex review. Current OpenClaw already implements the two core mitigations requested here: Feishu inbound messages are handled in fire-and-forget mode by default (`extensions/feishu/src/monitor.account.ts:662` and `extensions/feishu/src/monitor.message-handler.ts:323`), and retransmitted `message_id` values are filtered through the persistent Feishu dedupe store (`extensions/feishu/src/dedup.ts:9` and `extensions/feishu/src/monitor.message-handler.ts:315`). This same runtime behavior is already present in the latest shipped release, `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), so there is not a remaining core cleanup item to track here. If you still see the seq-gap warning on 2026.4.22 or newer, please open a fresh issue with exact gateway logs and whether the event was duplicated or only warned.
