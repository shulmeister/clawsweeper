## Summary

Keep #72141 open. Current main has the intended isolated-cron plumbing, but I did not find a source/test/history-backed fix for the reported parent-session assistant-message mirroring. The issue is fresh, detailed, labeled bug/regression, includes concrete timing/log evidence from 2026-04-24 and 2026-04-26, and the existing discussion identifies a plausible isolation/cache/bootstrap failure path that still needs maintainer investigation.

## Best Possible Solution

Keep the issue open and add a focused regression harness that simulates an active parent `agent:main:main` Telegram/DM session with recent assistant content, then triggers an isolated Google `agentTurn` cron with `delivery.mode="none"` and asserts the isolated run prompt, assembled messages, context-engine inputs, prompt-cache state, and final assistant text contain only the cron payload/tool output. Fix the concrete leak once that test identifies whether it is bootstrap, transcript/session lookup, prompt-cache reuse, active-run routing, or sessions_send feedback.
