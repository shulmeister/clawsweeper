## Summary

Keep open. Current main still lacks the one-shot exit behavior this PR adds: the hooks report commands build plugin diagnostics, plugin diagnostics load plugin modules and run plugin register handlers, and the hooks CLI actions return without any post-output exit or teardown. The provided PR discussion also shows the review feedback was handled in follow-up commits, including the clearer one-shot runner name and shared Vitest env detection.

## Best Possible Solution

Keep the PR open for maintainer review and likely landing after normal validation. The best path is the PR's scoped one-shot hooks CLI runner: run hooks list/info/check/enable/disable and bare hooks through a wrapper that drains stdout, calls defaultRuntime.exit(0) outside test runtimes via isVitestRuntimeEnv, leaves deprecated hooks install/update unchanged, and keeps focused regression coverage for the exit guard.
