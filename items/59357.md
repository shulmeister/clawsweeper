## Summary

Current `main` already ships OpenClaw `User-Agent` attribution through the centralized provider request policy, with request-time native-host gating that supersedes this PR's model-header injection approach.

## Evidence

- **Request-time attribution is centralized on main:** `resolveProviderRequestPolicyConfig()` merges provider attribution headers into final request headers, and `resolveProviderRequestHeaders()` exposes that shared path for LLM/provider requests.
  - file: [src/agents/provider-request-config.ts:643](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/provider-request-config.ts#L643)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Model resolution now intentionally avoids header injection:** `resolveProviderRequestConfig()` documents that attribution headers are excluded during model resolution and applied later at transport/request time so native-host gating follows the final resolved route.
  - file: [src/agents/provider-request-config.ts:682](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/provider-request-config.ts#L682)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Media-understanding requests use the same shared policy:** `resolveProviderHttpRequestConfig()` builds `Headers` from the same resolved request policy before transcription/HTTP helpers send requests.
  - file: [src/media-understanding/shared.ts:198](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/media-understanding/shared.ts#L198)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
