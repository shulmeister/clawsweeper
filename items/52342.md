## Summary

Keep open. Current `main` does not implement the PR’s Feishu OpenChat handling, so there is no basis to close as already fixed or obsolete. The checked-out code still classifies inbound Feishu chats purely from `chat_type`, while current docs and target parsing still treat `oc_` as chat/group IDs. At the same time, current tests also model some `oc_*` chats as `p2p` DMs, so the exact heuristic in the PR still needs maintainer review rather than cleanup closure.
