## Summary

Keep #69054 open. Current main treats `delivery.accountId` as a supported cron delivery field and has adjacent channel ambiguity validation, but it does not implement the requested `openclaw cron lint` command, gateway startup warning/fail-closed path, or add/edit validation requiring `delivery.accountId` for announce jobs targeting multi-account channel configs. The linked #69055 is a companion log-attribution request, not a duplicate or shipped replacement.

## Best Possible Solution

Keep #69054 open and implement it as a cron-owned validation surface: add a reusable lint helper that detects announce-mode jobs missing `delivery.accountId` when the resolved channel has multiple configured accounts; wire `openclaw cron lint --warn-only`, gateway startup warnings, and add/edit mutation validation; document the migration path and add unit/integration coverage. Preserve the existing explicit `accountId: "default"` escape hatch for jobs that should remain on the default bot.
