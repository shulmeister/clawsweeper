## Summary

Keep #70341 open. Current main does not contain the PR's Windows cmdlet allowlist fallback, and the PR is not safe to close as implemented-on-main. The linked issue #52952 was closed as already handled by wrapper parsing and durable exact-command approvals, but this PR proposes a narrower extra behavior: matching unresolved bare PowerShell cmdlets against a PowerShell executable allowlist entry. That behavior is still absent on main, and the PR also has an unresolved P1 execution-path review noting that allowlisting a bare cmdlet would later try to spawn `Get-ChildItem` directly instead of preserving the PowerShell host.

## Best Possible Solution

Keep the PR open for maintainer judgment, but require a rework before merge: either scope it down/close it manually as unnecessary because durable exact-command approvals solve #52952, or implement the cmdlet fallback together with a Windows execution plan that preserves the PowerShell host and adds node-host regression coverage for bare cmdlet allow-always reruns.
