## Summary

Keep #70341 open. Current main still lacks the Windows PowerShell cmdlet allowlist fallback described by the PR, and the linked #52952 report gives a concrete Windows gateway/Telegram reproduction. The PR is not safe to close as implemented or obsolete; it likely needs revision because review feedback identified a reachable execution-path problem after bare cmdlets become allowlist-satisfied.

## Best Possible Solution

Keep the PR open or supersede it with a maintainer-owned fix. The best implementation should add a narrowly scoped Windows cmdlet fallback in allowlist evaluation and allow-always persistence, keep command resolution honest for cmdlets, preserve the PowerShell execution wrapper when a bare cmdlet is allowed, and add regression coverage for both gateway and node allowlist execution paths.

## Evidence

- **Current allowlist matching still requires a resolved executable path:** `matchAllowlist` returns `null` when `resolution.resolvedPath` is missing, except for a bare `*` wildcard. PowerShell cmdlets such as `Get-ChildItem` are shell builtins, so they do not naturally provide a filesystem executable path for a normal path-scoped allowlist entry.
