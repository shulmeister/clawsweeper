## Summary

Current `main` already removes the startup-blocking behavior behind this request by making Gateway model-pricing bootstrap asynchronous and failure-tolerant, so the requested disable switch is no longer needed to avoid startup timeouts/hangs.

## Evidence

- **Gateway startup no longer waits on pricing bootstrap:** `startGatewayRuntimeServices` starts pricing refresh via `startGatewayModelPricingRefresh(...)` and returns immediately; it does not await pricing fetches on the startup path.
  - file: [src/gateway/server-runtime-services.ts:87](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/gateway/server-runtime-services.ts#L87)
  - command: `nl -ba src/gateway/server-runtime-services.ts | sed -n '70,110p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Pricing bootstrap is explicitly deferred:** `startGatewayModelPricingRefresh` wraps the initial refresh in `queueMicrotask(...)`, so network fetches begin only after the starter returns.
  - file: [src/gateway/model-pricing-cache.ts:654](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/gateway/model-pricing-cache.ts#L654)

## Risks / Open Questions

- There is still no explicit user-facing config flag to disable pricing bootstrap entirely.
- Timeout warnings can still appear when upstream pricing endpoints are slow or unavailable; the fix addresses startup blocking, not all pricing-log noise.
