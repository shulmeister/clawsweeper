## Summary

Keep #37804 open. Current main has meaningful partial support for config-injected and forward-compatible models, especially in runtime resolution and `models list`, but it still does not implement the RFC’s central shared resolved-model descriptor across status, list, runtime, capability gates, and docs.

## Best Possible Solution

Keep #37804 open as the umbrella design tracker until maintainers either land a typed shared resolved-model descriptor or create/link narrower canonical follow-ups. The likely implementation should combine registry entries, `models.providers`, provider dynamic/forward-compat hooks, transport, capabilities, provenance, and availability, then migrate `models status`, `models list`, runtime execution, picker/auto-reply consumers, thinking gates, and docs to consume that one result.
