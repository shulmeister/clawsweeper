## Summary

Keep open. Current main has strong partial fixes for config-injected provider models in runtime resolution, catalog/list visibility, and provider-owned thinking hooks, but the broader RFC invariant is still not implemented as one canonical resolved model descriptor consumed by status, list, runtime, and capability gates. The issue discussion also framed this as phased work, with later CLI/auto-reply/picker migration explicitly deferred.

## Best Possible Solution

Keep #37804 open as the umbrella design/product tracking item until either a maintainer explicitly narrows the accepted invariant or OpenClaw lands a shared resolved model descriptor consumed by status, list, runtime, and capability/docs surfaces. If maintainers want smaller work, split the remaining status/provenance and thinking/capability alignment into canonical follow-up issues, then close this as superseded once those are tracked.
