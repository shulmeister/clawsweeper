## Summary

Keep open. Current main still drops ordinary Teams `message` activities with empty text before using non-poll `activity.value`, while the existing shipped fix only handles `invoke` activities named `adaptiveCard/action`. The PR targets a distinct messageBack/Action.Submit path, includes a concrete reproduction and follow-up tests in the provided PR context, and is not obsolete or superseded by #60431.

## Best Possible Solution

Keep the PR open and land this PR or an equivalent focused fix in the bundled `msteams` message handler: serialize non-poll `activity.value` for empty-text messageBack Action.Submit activities before the empty-body drop, with regression tests for object/string/primitive payloads and empty values while preserving the existing poll and `adaptiveCard/action` invoke paths.
