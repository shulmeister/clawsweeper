## Summary

Current main supports disabling Telegram native commands wholesale and disabling text commands on native-command surfaces, but it does not support selectively hiding or disabling `/help` while keeping other Telegram slash commands available. The feature request is still open work, not implemented cleanup.

## Evidence

- **Built-in registry always defines /help as a core command:** `help` is hard-coded as a built-in chat/native command with text alias `/help`; there is no per-command config gate for it in the registry.
