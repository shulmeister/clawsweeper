## Summary

Keep PR #72033 open. Current main still does not implement `diagnostics.pricing`: the diagnostics handler, operator read-scope table, advertised Gateway method list, tests, CLI/docs, and protocol docs only cover `diagnostics.stability`. The PR remains a focused core Gateway observability change because the pricing cache metadata helper exists in core and the startup refresh path can still log pricing bootstrap failures, but that state is not exposed through any current RPC.

## Best Possible Solution

Keep the PR open for maintainer review. If maintainers want this operator diagnostic surface, the best path is the PR's small core Gateway change: add `diagnostics.pricing` to the diagnostics handler, READ_SCOPE table, advertised method list, focused cache-state tests, and public gateway protocol/operator docs if the RPC is intended to be supported.
