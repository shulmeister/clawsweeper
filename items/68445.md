## Summary

Keep PR #68445 open. Current main has related memory-core cleanup and dreaming is now documented/configured as opt-in, but the central PR changes are not implemented on main: exec progress updates still do not handle rejected async `onUpdate` callbacks, dreaming narrative still uses the millisecond timestamp session key as both `sessionKey` and `idempotencyKey`, session corpus appends still have no file-size cap, and short-term recall writes still have no 4096-entry LRU cap. The PR is recent, contributor-authored, and not protected by maintainer labels, so it should receive maintainer review or be rebased/split rather than auto-closed.

## Best Possible Solution

Keep the PR open but ask for maintainer review or an updated, rebased version. The best path is to split or refresh the still-missing exec `onUpdate` rejection fix and the memory-core bounded-growth changes, with regression coverage for async `onUpdate`, split narrative keys, corpus size limiting, and short-term recall eviction. Close only after an equivalent mainline fix or a canonical replacement PR lands.
