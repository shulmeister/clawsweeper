## Summary

Keep #7707 open. Current main has related source/provenance metadata, temporal decay, session visibility filtering, and untrusted prompt framing, but it still does not implement the requested source-based memory trust tiers, review/expiration policy by trust level, or trust-weighted retrieval. The issue discussion also already contains a maintainer Codex review explicitly keeping this exact issue open.

## Best Possible Solution

Keep #7707 open for maintainer/product design. A complete fix likely belongs across memory ingestion, storage, retrieval, config, and docs: classify memory origin at ingestion, persist structured trust metadata, expose a documented trust policy, enforce review/expiration/filtering/downweighting during recall, and add tests. HMAKT99's AKF comment and existing provenance/Memory Wiki work are useful prior art, but they do not currently satisfy the core request.
