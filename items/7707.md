## Summary

Keep #7707 open. Current main has related provenance/source labels, session visibility filtering, temporal decay, active-memory untrusted prompt framing, and Memory Wiki provenance, but it still does not implement source-based memory trust tiers, trust metadata persistence, review/expiration policy by trust level, or trust-weighted retrieval. The April 26 steipete Codex review in the issue already reached the same keep-open conclusion, and the current checkout still matches that assessment.

## Best Possible Solution

Keep #7707 open for maintainer/product/security design. A complete solution likely needs structured origin trust metadata at memory ingestion, schema/storage support, documented config policy, retrieval-time filtering/downweighting/review gates, expiration by trust tier, UI/docs visibility, and regression tests. HMAKT99's AKF comment and the existing provenance/Memory Wiki work are useful prior art, but current main does not yet satisfy the core request.
