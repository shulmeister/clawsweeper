## Summary

Keep #7707 open. Current main has coarse memory source attribution, temporal decay, active-memory untrusted prompt framing, and inter-session provenance, but it still lacks source-based memory trust tiers, review/expiration policies by trust level, and trust-weighted retrieval. The latest issue discussion also already contains a maintainer Codex review saying to keep this exact issue open.

## Best Possible Solution

Keep #7707 open for maintainer/product design. A complete fix likely belongs across memory ingestion, storage, retrieval, config, and docs: classify memory origin at ingestion, persist structured trust metadata, expose documented trust policy, enforce review/expiration/filtering/downweighting during recall, and add tests. HMAKT99's AKF comment and existing provenance/Memory Wiki work are useful prior art, but they do not currently satisfy the core request.
