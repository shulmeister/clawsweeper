## Summary

Keep #7707 open. Current main has adjacent source labels, session visibility filtering, temporal decay, Active Memory untrusted prompt framing, and Memory Wiki provenance concepts, but it still lacks persistent memory trust metadata, trust-policy config, review/expiration enforcement by trust tier, and trust-weighted retrieval/downranking.

## Best Possible Solution

Keep #7707 open for maintainer/product/security design. A complete solution should add structured source trust metadata at memory ingestion, persist it through memory schemas/backends and SDK contracts, document trust-policy config, enforce retrieval-time filtering/downweighting/review gates, support expiration by trust tier, preserve provenance through compaction and Memory Wiki flows, expose trust state where useful in UI/docs, and add regression coverage across builtin, QMD, Active Memory, session-memory, and compaction paths. HMAKT99's AKF comment is useful prior art, not shipped behavior.
