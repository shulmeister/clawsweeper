## Summary

Keep #7707 open. Current main has adjacent source labels, session visibility filtering, Active Memory untrusted framing, optional memory-lancedb prompt-injection filtering, and Memory Wiki provenance/claims, but it still does not implement persistent memory trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval for the default memory path.

## Best Possible Solution

Keep #7707 open for maintainer/product/security design. A complete solution should add structured source trust metadata at ingestion, persist it through memory-core, QMD, session-memory, Memory Wiki bridge, and SDK contracts, document trust-policy config, enforce retrieval-time filtering/downweighting/review gates, support expiration by trust tier, preserve provenance through compaction/dreaming/Active Memory flows, expose trust state where useful in UI/docs, and add regression coverage across builtin, QMD, Active Memory, memory-lancedb, session-memory, and Memory Wiki paths. HMAKT99's AKF comment is useful prior art, not shipped behavior.

## Risks / Open Questions

- Closing would drop a real product/security design gap: durable memory can still be searched, promoted, and injected without persisted source trust tiers that retrieval can enforce.
- A complete fix likely needs SDK/config/schema and memory backend changes, not only a bundled plugin, because current contracts do not carry trust fields.
- Adjacent provenance and prompt-framing features reduce risk in specific flows but do not prevent trust laundering through durable memory, compaction, dreaming, session indexing, or wiki bridge paths.
