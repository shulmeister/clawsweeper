## Summary

Keep #34986 open. Current main still does not implement the requested configurable heartbeat pre-flight script/command that can return HEARTBEAT_OK and skip agent session creation. The related PR #35504 proposed that exact `agents.defaults.heartbeat.preFlightCommand` behavior but was closed unmerged; current main only has built-in heartbeat skips such as empty HEARTBEAT.md, no due tasks, busy lanes, quiet hours, and disabled visibility.

## Best Possible Solution

Keep the issue open for maintainer product judgment or implementation. A complete fix would add an optional heartbeat pre-flight hook such as `agents.defaults.heartbeat.preFlightCommand`, merge it through default and per-agent heartbeat config, update exported types, strict Zod schema, labels/help/generated config docs, and run it in `runHeartbeatOnce` before session/model work. Tests should cover HEARTBEAT_OK skip, non-OK output, command failure, timeout, unset command, default/per-agent merging, and unchanged existing heartbeat behavior.
