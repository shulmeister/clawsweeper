## Summary

Keep #9820 open. Current main still does not provide native JSONL support for `memorySearch.extraPaths`: builtin file discovery admits Markdown plus enabled multimodal media only, `memory_get` rejects non-`.md` paths, watcher globs remain Markdown-oriented, and docs still describe extra-path directory scanning as `.md` files. The linked message-hook use case is plausible because message hooks exist, but current session JSONL indexing is a separate transcript source and does not cover arbitrary raw JSONL logs from hooks.

## Best Possible Solution

Keep #9820 open until maintainers either add native JSONL support to builtin memory indexing or explicitly document a narrower product direction. A complete fix should update file admission, JSONL chunking/line mapping, watcher globs, `memory_get` rendering, config help/docs, and focused tests; if arbitrary JSONL is out of scope, document the intended session-memory, QMD custom collection, or plugin-owned converter path.
