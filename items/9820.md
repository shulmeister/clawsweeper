## Summary

Keep #9820 open. Current main still does not implement native JSONL support for builtin `memorySearch.extraPaths`: extra path discovery admits Markdown plus enabled multimodal media, `memory_get` still rejects non-Markdown paths, watcher globs are Markdown/media-oriented, and docs/tests still describe Markdown-only extra path behavior. The issue discussion also includes a 2026-04-26 steipete/Codex review explicitly keeping this open for the same gap, and I rechecked the relevant current-main paths at `7a85c1a8226eb7d3a96906d2721a189768d3034a`.

## Best Possible Solution

Keep tracking this issue until maintainers either add native JSONL support to builtin `memorySearch.extraPaths` or explicitly document a narrower product direction. A complete core fix should extend extra-path admission and watcher globs, add JSONL parsing/chunking with line mappings, make `memory_get` safely read JSONL hits, add focused lister/index/read tests, and update the public memory docs. If arbitrary JSONL should remain outside builtin memory, document the intended alternative through session transcript memory, QMD/custom collections, or a plugin-owned converter path.
