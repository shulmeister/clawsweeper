## Summary

Keep #9820 open. Current main still does not provide native JSONL support for builtin `memorySearch.extraPaths`: file discovery admits Markdown plus enabled multimodal files, indexing chunks non-multimodal content as Markdown, `memory_get` rejects non-`.md` targets, watcher filtering still ignores JSON-like files when stats are available, and public docs describe extra paths as recursively scanned for `.md` files. The existing session transcript JSONL path is separate opt-in session-memory indexing, not arbitrary JSONL ingestion from extra paths.

## Best Possible Solution

Keep #9820 open until maintainers either add native JSONL support to the builtin memory indexer or explicitly document the supported alternative. A complete core fix should update file admission, JSONL parsing/chunking with line mapping, watcher filtering, `memory_get`, config help/docs, and focused tests. If arbitrary raw logs are out of scope, document the intended QMD/session-memory/converter-plugin workflow and close against that product decision.
