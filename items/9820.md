## Summary

Keep open. Current main still does not implement native JSONL support for builtin `memorySearch.extraPaths`: extra path discovery admits Markdown plus enabled multimodal media only, `memory_get` still rejects non-Markdown files, and docs/tests continue to describe and assert Markdown-only extra path behavior. OpenClaw does have related message hook support and session transcript JSONL indexing, but that path is scoped to OpenClaw-owned session transcripts, not arbitrary hook-created `.jsonl` logs in `extraPaths`.

## Best Possible Solution

Keep tracking this issue until maintainers either implement native builtin JSONL support for `memorySearch.extraPaths` or explicitly narrow the product direction. A complete core fix should extend file admission and watcher globs, add JSONL-aware parsing/chunking with line mappings, make `memory_get` read JSONL hits safely, update focused lister/index/read tests, and revise the public memory docs. If maintainers prefer not to support arbitrary JSONL in the builtin engine, document the intended alternative through OpenClaw session transcript memory, QMD/custom collections, or a plugin-owned converter path.
