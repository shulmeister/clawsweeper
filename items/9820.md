## Summary

Keep open. Current main has related session-transcript memory support, but the requested capability remains missing for `memorySearch.extraPaths`: arbitrary `.jsonl` files are still not accepted by the builtin memory file lister/read path, and docs still describe extra paths as Markdown-only for normal indexing.

## Evidence

- **Builtin memory file filter still excludes JSONL:** `isAllowedMemoryFilePath` returns true for `.md` files or enabled multimodal files only; `listMemoryFiles` uses that predicate for extra path files and directories.
  - file: [src/memory-host-sdk/host/internal.ts:94](https://github.com/openclaw/openclaw/blob/cbe5515b70abaeba98550b58d739ca0896907c43/src/memory-host-sdk/host/internal.ts#L94)
  - command: `nl -ba src/memory-host-sdk/host/internal.ts | sed -n '80,200p'`
  - sha: [cbe5515b70ab](https://github.com/openclaw/openclaw/commit/cbe5515b70abaeba98550b58d739ca0896907c43)
- **`memory_get` path still rejects non-Markdown files:** `readMemoryFile` only marks a direct extra file as allowed when it ends with `.md`, then rejects any absolute path that does not end with `.md`. Native JSONL search results would still lack a corresponding read path here.
  - file: [src/memory-host-sdk/host/read-file.ts:49](https://github.com/openclaw/openclaw/blob/cbe5515b70abaeba98550b58d739ca0896907c43/src/memory-host-sdk/host/read-file.ts#L49)
