## Summary

Keep open. Current main still does not expose the requested `openclaw sessions rm` or `openclaw sessions clear` CLI surface. The issue remains actionable because core already has `sessions.delete` support and the Control UI uses it, but the CLI only wires list and cleanup maintenance. The linked PR #35735 appears to be the right related implementation track, but it is open/unmerged in the provided context and no matching delete/clear CLI files or commits are present on main.

## Best Possible Solution

Keep #35682 open and use PR #35735 as the active implementation reference. The best product path is a maintained CLI implementation that wraps the existing session deletion capability with `sessions rm <key>` and scoped bulk clear behavior, plus tests and docs for `--agent`, `--all-agents`, `--dry-run`, `--json`, transcript handling, and safe store consistency.
