## Summary

Keep open. Current main still has `openclaw sessions` for listing and `openclaw sessions cleanup` for maintenance-policy pruning, but it does not expose the requested intentional delete commands (`sessions rm` / `sessions clear`). The linked PR #35735 is useful related implementation work and should remain the likely path, but it is still open and its head commit is not present in the checked-out main tree.

## Best Possible Solution

Keep #35682 open until maintainers merge PR #35735 or choose a different CLI design. The best product path is to add `openclaw sessions rm <key>` and `openclaw sessions clear` on top of the existing session store/gateway delete behavior, with `--agent`, `--all-agents`, `--store`, `--dry-run`, `--json`, docs, and focused CLI tests.
