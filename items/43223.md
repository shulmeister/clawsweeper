## Summary

Keep open. Current main has not absorbed this PR’s destination change: `scripts/copy-hook-metadata.ts` still copies bundled hook metadata to `dist/bundled`, while `resolveBundledHooksDir()` still looks for npm-installed bundled hooks at `dist/hooks/bundled`. The linked regression is still plausible from current source and docs, but the exact PR may need refresh because current tsdown config/tests also place bundled hook handlers under `dist/bundled`.

## Best Possible Solution

Keep the PR open for maintainer follow-up. Refresh it into a complete bundled-hook packaging fix that aligns `scripts/copy-hook-metadata.ts`, `resolveBundledHooksDir()`, `tsdown.config.ts`, and the tsdown/build regression tests so a packaged install discovers bundled hooks like `session-memory` with both `HOOK.md` and `handler.js` in the same directory.
