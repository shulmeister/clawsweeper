## Summary

Keep open. Current `main` still replays `thinkingSignature` reasoning items in `convertResponsesMessages` without checking whether the assistant history came from a different provider/API, so the reported mixed-provider Responses API failure is not shown as fixed on `main`.

## Evidence

- **Current replay path still pushes reasoning signatures unconditionally:** In the assistant-history replay branch, `buildOpenAIResponsesParams` calls `convertResponsesMessages`, and `convertResponsesMessages` still does `if (block.thinkingSignature) output.push(JSON.parse(block.thinkingSignature));` for `thinking` blocks with no provider/API guard before replaying Responses items.
