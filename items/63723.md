## Summary

Current main already implements the reported sessions_send timeout/routing fix path: pending Gateway RPCs such as agent.wait no longer get their websocket closed by the tick watchdog, sessions_send drives a target agent run and waits for the updated assistant reply, and the A2A announce path preserves external routing metadata for follow-up delivery. A matching duplicate report, #63724, was already closed with the same implemented-on-main conclusion, while the broader structured-signal idea linked in #50072 is separate and already closed.

## Best Possible Solution

Close this issue as implemented. Keep the shipped Gateway pending-RPC watchdog and sessions_send A2A route-preservation behavior; handle broader structured-signal/task-ownership ideas in their own product/API discussion rather than keeping this concrete timeout/routing report open.
