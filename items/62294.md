## Summary

Current main does not implement the requested heartbeat behavior. Non-interval wake reasons still bypass the interval gate before running, action-driven runs still push nextDueMs forward, and the generated macOS launchd plist still lacks ProcessType=Interactive. The discussion usefully split the work into #62308, #62310, and #66992, but those are open in the provided GitHub context and do not prove the issue is fixed or clearly superseded.
