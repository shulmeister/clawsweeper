## Summary

Keep #8495 open. Current main still only blocks/skips unauthorized WhatsApp group traffic; it does not auto-leave unauthorized groups, expose a group-join hook/event, or support a WhatsApp leave-group message action. The existing issue comment from steipete also explicitly kept this open, and the broader related #11489 remains open rather than superseding this WhatsApp-specific request.

## Best Possible Solution

Keep #8495 open. The best path is a bundled WhatsApp plugin change with explicit opt-in config, schema/docs updates, a policy-checked Baileys leave-group path for unauthorized group membership, focused tests around blocked group joins/reconciliation, and a deliberate decision on whether the group-join notification belongs in a WhatsApp-specific event, the broader #11489 group pairing flow, or a generic plugin hook. Manual leaveGroup support should be added only if the WhatsApp channel action/runtime surface can enforce account and group authorization safely.
