## Summary

Keep #8495 open. Current main still blocks unauthorized WhatsApp group traffic but does not auto-leave the group, expose a WhatsApp group-join hook/event, or support a WhatsApp leave-group message action. The broader #11489 group-pairing request remains useful context, but it is open and does not supersede this concrete WhatsApp auto-leave/manual-leave gap.

## Best Possible Solution

Keep #8495 open. The best path is a bundled WhatsApp plugin change: add explicit root/account config and strict schema/docs for an operator-controlled auto-leave policy, listen for Baileys group membership/add events or reconcile participating groups safely, call the platform leave-group API only after policy validation, and add focused tests. A group-join hook/event can be coordinated with #11489, but that should not replace the concrete WhatsApp auto-leave and leaveGroup action request unless maintainers choose a different product direction.
