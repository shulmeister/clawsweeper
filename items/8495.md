## Summary

Keep #8495 open. Current main still implements WhatsApp group blocking/allowlisting, but it does not auto-leave unauthorized WhatsApp groups, expose a WhatsApp group-join webhook/event, or provide a WhatsApp leave-group message action. The broader related #11489 remains open and does not supersede this WhatsApp-specific request.

## Best Possible Solution

Keep #8495 open for a deliberate bundled WhatsApp plugin feature. Add opt-in config/schema/docs for unauthorized-group auto-leave, implement a verified Baileys group leave operation with account/target authorization, listen for the appropriate group participant/join update, expose a clear owner notification or hook shape, and cover both groupAllowFrom sender blocks and channels.whatsapp.groups admission blocks with tests. Coordinate the notification/pairing shape with open #11489 rather than closing this WhatsApp-specific request.
