## Summary

Keep #8495 open. Current main still blocks unauthorized WhatsApp group traffic but does not auto-leave unauthorized groups, expose a WhatsApp group-join hook/event, or support a WhatsApp leave-group message action. The related #11489 is broader and still open, so it does not supersede this concrete WhatsApp gap.

## Best Possible Solution

Keep #8495 open. The best path is a bundled WhatsApp plugin change with explicit opt-in config, strict schema/docs, safe group-add or participating-group reconciliation, a policy-checked WhatsApp leave call, a manual leaveGroup/message action if maintainers want that surface, and focused tests. Coordinate the group-join notification/hook portion with the broader open #11489 work rather than treating #11489 as a superseding replacement.
