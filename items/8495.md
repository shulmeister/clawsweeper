## Summary

Keep #8495 open. Current main still implements WhatsApp group allowlist handling as message blocking/skipping only; it does not auto-leave unauthorized WhatsApp groups, expose a group-join hook/event, or add a WhatsApp leave-group message action. The related #11489 is broader and open rather than a shipped replacement, and the existing maintainer-side Codex review comment also explicitly keeps this item open.

## Best Possible Solution

Keep #8495 open for a deliberate bundled WhatsApp plugin change: add opt-in config/schema/docs for auto-leaving unauthorized groups, implement a Baileys-backed group leave path with account and target authorization, cover group admission and unauthorized-group behavior with tests, and decide whether join notification belongs in a WhatsApp-specific event or the broader open #11489 group-pairing design. A manual leave action should ship only with safe channel action/runtime authorization.
