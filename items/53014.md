## Summary

Keep open. Current `main` does not contain the PR's stale-session heartbeat preflight, and the existing code path still makes heartbeat/cron/exec turns reuse the existing session while writing a fresh `updatedAt`. Normal user session resolution still keys rollover off `updatedAt`, so the reported daily-reset interference remains plausible on `main`. There is also evidence in the PR review context that the proposed patch needs follow-up for bypassed exec/cron/wake heartbeats rather than being safely closable as-is.
