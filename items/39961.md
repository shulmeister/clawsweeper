## Summary

Current main still does not implement the PR's requested streaming text-repetition guard. Existing loop detection is scoped to tool calls, assistant text streaming/final flush paths do not abort or suppress repeated text output, and config/docs do not expose a textRepetitionGuard setting. The PR discussion shows useful contributor follow-up and fixes to bot-review findings, but the underlying capability is still missing and needs maintainer engineering review rather than cleanup closure.

## Best Possible Solution

Keep #39961 open for maintainer engineering review. The best path is to land a robust core streaming text-repetition guard with config/schema/docs/tests covering abort, queued-event, chunked, non-chunked, final-flush, and lifecycle fallback behavior, or have a maintainer explicitly decide OpenClaw should not add this core guard.
