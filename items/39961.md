## Summary

Keep #39961 open. Current main still does not implement a streaming text-repetition guard, and the existing loop guard is scoped to tool-call loops. The PR is unique unmerged work for a real core streaming gap, but the abort/final-flush lifecycle needs maintainer engineering review before landing.

## Best Possible Solution

Keep #39961 open for maintainer engineering review. The best path is to land a robust core streaming text-repetition guard with config/schema/docs/tests covering abort behavior, queued events, chunked and non-chunked replies, final flush, and lifecycle fallback behavior, or have a maintainer explicitly decide OpenClaw should not add this core guard.
