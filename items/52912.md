## Summary

Keep open. Current `main` still does not implement the requested Anthropic setup-token sibling sync, and PR #52912 introduces an unresolved regression by enabling sibling sync for all OAuth credentials in generic provider auth flows, including explicit non-default agent setups.

## Evidence

- **Current main still writes plugin-provider auth to one agent dir only:** `runProviderPluginAuthMethod` still calls `upsertAuthProfile` once with the current `agentDir`; there is no sibling-agent propagation in this path.
  - file: [src/plugins/provider-auth-choice.ts:163](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/plugins/provider-auth-choice.ts#L163)
  - command: `nl -ba src/plugins/provider-auth-choice.ts | sed -n '120,185p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Sibling sync exists only for OAuth helper on main:** `writeOAuthCredentials` resolves sibling agent dirs and best-effort writes to each sibling only when `syncSiblingAgents` is enabled; no equivalent generic token helper exists on `main`.
  - file: [src/plugins/provider-auth-helpers.ts:269](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/plugins/provider-auth-helpers.ts#L269)
  - command: `nl -ba src/plugins/provider-auth-helpers.ts | sed -n '250,335p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **PR broadens sync in a path used with explicit agentDir:** `runProviderPluginAuthMethod` is called from flows that pass an explicit `agentDir`, so making every OAuth credential sync to siblings would leak credentials across agent boundaries rather than staying scoped to the selected agent.
  - file: [src/flows/model-picker.ts:389](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/flows/model-picker.ts#L389)

  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Auth profile ids default to shared `provider:default` names:** `buildAuthProfileId` falls back to `default` when no profile name is provided, which makes cross-agent overwrites plausible for OAuth providers if sibling sync is applied generically.
  - file: [src/agents/auth-profiles/identity.ts:17](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/auth-profiles/identity.ts#L17)
