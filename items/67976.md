## Summary

Keep open. Current main still does not show source, docs, or test evidence that `agents.defaults.imageModel.primary = "volcengine-plan/doubao-seed-2.0-pro"` works for image understanding. The Volcengine coding provider catalog remains text-only, the reported model is absent from the bundled catalog, and there is no Volcengine-owned media-understanding provider or regression test for the reported Ark coding endpoint path.

## Best Possible Solution

Keep the issue open. The best maintainer path is to verify the current Volcengine Ark coding image endpoint, then either add a Volcengine-owned image-capable catalog/media-understanding path for the supported `doubao-seed-2.0-pro` ref with a regression test, or explicitly document that `volcengine-plan` imageModel is unsupported and point users to a supported `volcengine/*` image-capable model or custom provider config.
