## Summary

Keep open. Current main does not implement the PR's central behavior: tool-enabled/shell-like agent turns still only get `temperature` when it is explicitly configured. The PR is also not merge-ready as-is because the branch contains unrelated secrets/lockfile/changelog changes and review comments identify remaining agent-runner edge cases.

## Best Possible Solution

Keep the PR open but require a narrower rework before maintainer review: split out unrelated secrets/lockfile/changelog changes, rebase on current main, implement the tool-temperature heuristic through the agent extra-params/runtime-plan path for both normal and compaction runs, preserve explicit user temperatures, avoid changing prompt-cache defaults, skip only when actual provider thinking requires it, and add focused tests for plain chat, generic tools, shell-like tools, explicit temperature, thinking-enabled cases, and compaction.
