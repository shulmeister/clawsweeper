## Summary

Keep open. Current main still does not implement the PR's central thrown context-overflow session reset, and the final embedded-error fallback still tells the user to use /new instead of resetting the active session. The PR is plausible and linked to a real open Telegram overflow report, but existing review comments identify changes needed before it is merge-ready.

## Best Possible Solution

Keep the PR open for maintainer review or request changes. The best path is to preserve the useful thrown-overflow recovery idea, add a pure context-overflow summary guard, align the user-facing reset copy with existing compaction overflow guidance, and add tests that cover both the thrown-overflow and reachable final embedded-error reset paths. Coordinate with #68494 and the broader #70744 session-recovery issue if maintainers prefer a fresh follow-up PR.
