## Summary

Keep open. Current main still implements Discord/generic channel recovery as restart plus cooldown/hourly cap, without the cross-lifecycle escalation or Discord clean-restart behavior requested here. The issue also has fresh current-version evidence and an open PR (#40413) explicitly tracking the proposed fix.

## Best Possible Solution

Keep #38596 open until a maintainer lands a focused fix or decides the product direction. The likely implementation path is to add monitor-level repeated-restart escalation, wire a Discord-owned clean restart/fresh IDENTIFY hook through an appropriate plugin/channel boundary, and cover it with health-monitor plus Discord lifecycle regression tests. PR #40413 is the current canonical implementation discussion to evaluate rather than closing this issue as stale.
