## Summary

Keep open. Current main still has generic health-monitor restart/cooldown/hourly-cap behavior, but no monitor-level repeated-restart escalation, Discord clean-restart/fresh-IDENTIFY hook, or Discord reconnect catch-up behavior that would resolve the reported post-connection instability. The supplied issue context also includes fresh current-version evidence and an open PR (#40413) that explicitly proposes the missing implementation.

## Best Possible Solution

Keep #38596 open until a maintainer lands or rejects a focused Discord recovery design. The likely path is to evaluate PR #40413 or a narrower replacement: add monitor-level repeated-restart escalation, expose a Discord clean-restart/fresh-IDENTIFY path across the channel boundary, reset escalation after a stable period, and cover it with health-monitor plus Discord lifecycle regression tests. Treat #71546 as related user-visible ingest-lag evidence, not a replacement for this monitor-escalation issue.
