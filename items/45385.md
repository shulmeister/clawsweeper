## Summary

Keep open. Current main partially addresses the suspected transcript-repair side by stripping dangling Anthropic tool-use blocks, including last-message cases, but the issue’s titled user-facing bug remains: `formatAssistantErrorText()` still lets JSON `invalid_request_error` payloads fall through to `LLM request rejected: ${message}` with the provider’s raw `tool_use`/`tool_result` text. The linked PR #45393 is still open and unmerged, so it is not a basis to close the issue.
