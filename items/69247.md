## Summary

Current main already fixes the task audit false positive that #69247 targeted, using registry-level timestamp normalization from merged PR #71871 instead of this PR's 1-second audit tolerance. The shipped direction preserves strict audit checks for real timestamp ordering violations while preventing benign create/update/restore skew from reaching audit-visible task records.

## Best Possible Solution

Close #69247 as implemented on main. Keep the merged #71871 registry-level timestamp normalization and strict audit invariant rather than merging this PR's broad 1-second audit-side tolerance; track any separate legacy-data cleanup only if maintainers still want it.
