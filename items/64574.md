## Summary

#64574 is a broad Nextcloud Talk parity wishlist rather than a core regression or a concrete missing OpenClaw API. Current main already treats Nextcloud Talk as a plugin-owned channel surface, and the remaining requested status, typing, receipt, message-operation, thread, and rich-message features are optional service-specific channel capabilities that fit the existing plugin/ClawHub path.

## Best Possible Solution

Close the broad core tracker as plugin/ClawHub scope. Keep the shipped bundled Nextcloud Talk plugin as the baseline, and handle individual Talk API enhancements as focused plugin-owned work or a ClawHub/community plugin; open a narrow SDK issue only if implementation proves a specific missing extension seam.
