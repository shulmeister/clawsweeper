## Summary

Keep #71638 open. Current main still does not surface Telegram forward, via-bot, or reply provenance on `MessageReceivedHookEvent.context.metadata`. Telegram already normalizes reply/forward context for the agent prompt path, but the received-hook canonical path and mappers only emit generic routing/sender metadata, and there is no `via_bot`/`viaBot` hook implementation.

## Best Possible Solution

Keep the issue open and implement a Telegram-owned additive fix: derive bounded `metadata.forward`, `metadata.viaBot`, and `metadata.replyTo` from inbound Telegram messages, carry it through the canonical received-hook path for both plugin and internal hooks, and add focused tests for the mapper plus Telegram received-hook emit paths.
