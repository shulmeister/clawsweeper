## Summary

Keep open. Current `main` only has targeted early parameter checks for the file tools (`read`/`write`/`edit`). The requested general pre-call validation flow, including config like `tools.validation.enabled` / `autoPrompt`, is not present, and `exec` still rejects a missing `command` only inside its `execute` path.

## Evidence

- **Validation helper only covers read/write/edit:** `REQUIRED_PARAM_GROUPS` defines early required-parameter checks only for `read`, `write`, and `edit`; there is no generic all-tools validation registry.
  - file: [src/agents/pi-tools.params.ts:75](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/pi-tools.params.ts#L75)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **File tools are explicitly wrapped with early validation:** `write` and `edit` are wrapped with `wrapToolParamValidation(...)`, and `read` calls `assertRequiredParams(...)` before executing the base tool.
  - file: [src/agents/pi-tools.read.ts:637](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/pi-tools.read.ts#L637)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Lazy exec tool is not wrapped:** `createLazyExecTool()` exposes `execSchema` but forwards directly to the loaded tool's `execute(...)` without any `wrapToolParamValidation(...)` step.
  - file: [src/agents/pi-tools.ts:99](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/pi-tools.ts#L99)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Exec still errors inside tool execution:** `createExecTool()` checks `if (!params.command) throw new Error("Provide a command to start.")` inside `execute`, so the missing required field is handled in the tool body rather than by a general pre-call validator.
  - file: [src/agents/bash-tools.exec.ts:1374](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/bash-tools.exec.ts#L1374)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **No matching runtime config exists:** `ToolsSchema` enumerates supported `tools.*` config keys, but there is no `tools.validation`, `enabled`, or `autoPrompt` setting matching the issue's proposal.
  - file: [src/config/zod-schema.agent-runtime.ts:868](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/config/zod-schema.agent-runtime.ts#L868)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Tests only prove the file-tool path:** Current tests assert missing-parameter failures for `read`/`write`/`edit`, but there is no equivalent coverage showing a generalized validation layer for `exec` or all tools.
  - file: [src/agents/pi-tools.create-openclaw-coding-tools.test.ts:578](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/pi-tools.create-openclaw-coding-tools.test.ts#L578)
