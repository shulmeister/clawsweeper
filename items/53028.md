## Summary

Keep open. Current `main` still treats LM Studio as a normal localhost HTTP provider, but I did not find a code change, test, or changelog entry that specifically fixes the reported WiFi-off/local-loopback failure on macOS. The issue is also not old enough for `stale_insufficient_info`.
