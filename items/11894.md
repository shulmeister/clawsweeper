## Summary

Keep #11894 open. Current main has stronger cost and usage surfaces, and it carries auth profile ids internally during runner setup/rotation, but the requested user-facing behavior is still missing: no per-call transcript/session-safe auth profile attribution, no per-profile cost aggregation, and no user-visible fallback/profile event that identifies which profile paid for a call. The issue also has a fresh maintainer comment explicitly saying to keep it open; the burn0 workaround is useful for network-level cost tracking but does not solve OpenClaw-level profile attribution.

## Best Possible Solution

Keep the issue open until OpenClaw records the actual auth profile used for each model call in transcript/session-safe metadata, exposes cost totals grouped by auth profile in /usage cost or session_status, and emits or stores a clear profile-rotation/fallback event that user-facing surfaces can consume. The burn0 link can remain as a useful interim workaround for network-level per-request costs, but it is not sufficient for native auth-profile attribution.
