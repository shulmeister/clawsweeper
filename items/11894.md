## Summary

Keep #11894 open. Current main has native per-response/session cost reporting and internal auth-profile rotation plumbing, and the linked burn0/#55379 discussion is a useful interim cost workaround, but OpenClaw still does not expose the resolved auth profile for each completed model call, aggregate costs by auth profile, or emit a user-facing auth-profile rotation/fallback event. The provided April 26, 2026 maintainer-thread Codex comment also explicitly keeps this issue open for the same remaining gap.

## Best Possible Solution

Keep #11894 open until OpenClaw records the resolved authProfileId for each completed model call in a session-safe form, persists or exposes it through transcript/usage metadata or diagnostics, aggregates session_status and/or usage.cost totals by auth profile, and emits a clear auth-profile rotation/fallback event. The existing runner auth-controller/rotation plumbing, transcript cost parser, gateway usage summaries, and model-call hook types are the likely implementation points; burn0 can remain an interim external cost tool but is not a complete native solution.
