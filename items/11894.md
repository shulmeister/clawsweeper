## Summary

Keep #11894 open. Current main has meaningful usage and cost visibility, but it still does not expose the actual auth profile used for each model call, does not aggregate cost by auth profile, and does not provide a user-visible fallback/profile event that satisfies the issue. A maintainer review comment on the issue also explicitly kept it open after noting the burn0 workaround does not solve OpenClaw-level auth-profile attribution.

## Best Possible Solution

Keep the issue open until OpenClaw records the actual auth profile used for each model call in a transcript/session-safe shape, exposes per-profile cost attribution in `/usage cost` or `session_status`, and emits or stores a clear fallback/profile-rotation event that user-facing surfaces can consume. The burn0 workaround can remain a useful interim note for network-level cost tracking, but it is not a replacement for native auth-profile attribution.
