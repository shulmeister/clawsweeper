## Summary

Keep #11894 open. Current main has meaningful cost/usage visibility, including per-response `/usage full`, `/usage cost`, transcript-backed `usage.cost`, and `/status` cost estimates, but it does not satisfy the full request because actual per-call auth profile attribution and a user-visible fallback/profile event remain missing. The related burn0 comment is a useful interim workaround for network-level cost tracking, but the commenter explicitly notes it does not solve OpenClaw-level auth profile attribution, and closed #55379 is narrower than this issue.
