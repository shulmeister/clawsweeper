## Summary

Keep open. Current main partially addresses the request: `/status`/`session_status` now surface the current auth profile label and fallback auth label, and transcript usage data carries estimated cost. But OpenClaw still does not persist the auth profile actually used on each assistant turn, and its cost summaries/time series have no auth-profile dimension, so the requested per-call attribution and per-profile breakdown are not implemented well enough to close.

## Evidence

- **Current status card shows profile labels, not just generic auth mode:** `resolveModelAuthLabel()` resolves the preferred/current auth profile from the auth-profile store and formats it with the exact profile id/display label, and `status-message.ts` injects that label into the model line and fallback line for `/status` and `session_status`.
  - file: [src/agents/model-auth-label.ts:31](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/agents/model-auth-label.ts#L31)

- **Profile display includes the concrete profile id:** The display label is the profile id, optionally followed by display name or email, so current-session attribution is visible at status time.
  - file: [src/agents/auth-profiles/display.ts:5](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/agents/auth-profiles/display.ts#L5)
