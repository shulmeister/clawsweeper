## Summary

Keep #11894 open. Current main has adjacent cost reporting, usage aggregation, auth-profile rotation, and structured failover logs, but it still does not attribute completed model calls or session cost totals to the resolved auth profile. The existing burn0/#55379 discussion is useful as an interim per-request cost workaround, but it explicitly does not solve native OpenClaw auth-profile attribution.

## Best Possible Solution

Keep #11894 open until OpenClaw records the resolved authProfileId for each completed model call in safe metadata, exposes it through transcript/usage/session_status or diagnostics, aggregates costs by auth profile, and emits a clear user-facing fallback/profile-rotation event. burn0/#55379 can remain an external cost workaround, but it is not a native replacement for OpenClaw auth-profile attribution.
