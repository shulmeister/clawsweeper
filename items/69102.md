## Summary

Keep #69102 open. Current main does not expose Gemini Flex `service_tier`/`serviceTier` through the bundled Google Gemini text transport, config extra-params layer, Google provider docs, or tests. The request is recent, unprotected, not maintainer-authored, and not superseded by the provided related items, which cover other provider/service features.

## Best Possible Solution

Keep the issue open and implement this in the bundled Google plugin path: normalize `serviceTier`/`service_tier` from `agents.defaults.models["google/..."] .params`, pass it into the Google transport as REST `service_tier: "flex"`, add focused transport/extra-params tests, and document it on the public Google provider page.

## Evidence

- **Official Gemini Flex contract:** Google's Flex inference docs say Flex is selected by `service_tier: "flex"` in the REST request body and `config: { serviceTier: "flex" }` in the JavaScript SDK examples. ([ai.google.dev](https://ai.google.dev/gemini-api/docs/flex-inference))
  - command: `web.open https://ai.google.dev/gemini-api/docs/flex-inference#javascript`
- **Google transport omits service tier:** The native Gemini request type includes `cachedContent`, `contents`, `generationConfig`, `systemInstruction`, `tools`, and `toolConfig`, but no `service_tier` or `serviceTier`; `buildGoogleGenerativeAiParams` only populates those existing fields.
  - file: [extensions/google/transport-stream.ts:62](https://github.com/openclaw/openclaw/blob/e672b61417af5c45b0431df6d9109a1f4b618ef5/extensions/google/transport-stream.ts#L62)

  - file: [src/agents/pi-embedded-runner/extra-params.ts:280](https://github.com/openclaw/openclaw/blob/e672b61417af5c45b0431df6d9109a1f4b618ef5/src/agents/pi-embedded-runner/extra-params.ts#L280)

## Risks / Open Questions

- Implementation should confirm the exact REST streaming payload placement for `streamGenerateContent`; Google docs show REST `service_tier` at the top level and SDK `serviceTier` inside `config`.
- Flex traffic can require longer timeout handling; the narrow issue asks for service tier selection, but a complete user-facing fix may also need documented timeout guidance.
