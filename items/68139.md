## Summary

Current main and the official docs surface do not reproduce this report. The issue targets `openclaw-ai.com`, while repo docs policy and live docs point to `docs.openclaw.ai`, where the German provider index and provider pages resolve correctly. Current i18n tests also lock localized links to `/<locale>/...`, not `/<locale>/docs/...`.

## Evidence

- **Official docs host and link contract:** Docs policy says OpenClaw docs are hosted on Mintlify at `https://docs.openclaw.ai` and internal links in `docs/**/*.md` must remain root-relative like `/providers/ollama`. That matches the current provider index source.
  - file: [docs/AGENTS.md:7](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/docs/AGENTS.md#L7)
  - command: `sed -n '1,240p' docs/AGENTS.md`
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Localized docs are a separate publish surface:** The repo explicitly says foreign-language docs are not maintained in this repo and are generated in the separate `openclaw/docs` publish repo, so a breakage on an alternate localized host is not evidence of a core-docs bug by itself.
  - file: [docs/AGENTS.md:21](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/docs/AGENTS.md#L21)
  - command: `sed -n '1,240p' docs/AGENTS.md`
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Current provider index source intentionally uses root-relative links:** `docs/providers/index.md` still links providers with root-relative paths such as `/providers/ollama`, consistent with the Mintlify rules above.
  - file: [docs/providers/index.md:27](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/docs/providers/index.md#L27)
  - command: `nl -ba docs/providers/index.md | sed -n '1,140p'`
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Current i18n tests expect `/<locale>/...`, not `/<locale>/docs/...`:** `scripts/docs-i18n/localized_links_test.go` verifies localized rewrites like `/zh-CN/gateway/configuration` and `/zh-CN/gateway/sandboxing`. The current main contract does not include an extra `/docs` path segment for localized routes.
  - file: [scripts/docs-i18n/localized_links_test.go:22](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/scripts/docs-i18n/localized_links_test.go#L22)
  - command: `nl -ba scripts/docs-i18n/localized_links_test.go | sed -n '1,260p'`
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Official localized docs reproduce as working:** Live official docs pages currently resolve at `https://docs.openclaw.ai/de/providers/index` and `https://docs.openclaw.ai/de/providers/ollama`; the `/de/docs/...` variants error. That matches current main's i18n contract and contradicts the report's expected path shape.
  - file: [https://docs.openclaw.ai/de/providers/index:112](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/https%3A//docs.openclaw.ai/de/providers/index#L112)
  - command: `web.open docs.openclaw.ai localized provider pages`
- **Prior issue/PR investigation already reached the same conclusion:** Issue comment 4273442204 says the reported breakage only occurs on `openclaw-ai.com` and not on the official docs site. Follow-up PR #68250 was later closed with a note that the change targeted a non-official mirror rather than the supported docs host.
  - file: [https://github.com/openclaw/openclaw/pull/68250:568](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/https%3A//github.com/openclaw/openclaw/pull/68250#L568)
  - command: `web.open GitHub issue #68139 and PR #68250`
