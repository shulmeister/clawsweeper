## Summary

Keep #69512 open. Current main only maps OpenClaw's YOLO exec policy to Claude CLI `--permission-mode bypassPermissions`; it does not forward `tools.exec.security: "allowlist"` or `exec-approvals.json` allowlist entries to Claude CLI sessions. The native OpenClaw exec path still enforces allowlist misses, but the Claude CLI backend remains configured through Claude-specific args and has no policy bridge for OpenClaw exec approvals.

## Best Possible Solution

Keep the issue open and implement a policy bridge at the Claude CLI backend/runtime boundary. The fix should translate the effective per-agent `tools.exec` policy plus `exec-approvals.json` allowlist state into Claude CLI permissions where Claude supports that, or add an OpenClaw-owned Claude permission/pre-exec interception path. Add regression coverage showing a Claude CLI session with `security: "allowlist"` allows allowlisted commands and denies misses, without relying on raw user-supplied `--permission-mode` args.
