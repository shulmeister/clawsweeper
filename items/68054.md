## Summary

Keep open. Current main has relevant improvements around cron visibility, ephemeral system events, and legacy nano-banana migration, but none conclusively fixes or supersedes the reported user-visible problem: an active OpenClaw agent on 2026.3.2 sending unexpected images while `openclaw cron list` and `cron/jobs.json` show no jobs. The provided related items are title-search overlaps, not canonical duplicates for this behavior.

## Best Possible Solution

Keep the issue open for maintainer diagnosis. The next best path is to ask for a reproduction bundle from a current release: `openclaw version`, `openclaw cron status`, `openclaw cron list --all --json`, `openclaw cron runs` for any suspected job IDs, `openclaw system heartbeat last`, `openclaw memory status --json`, relevant redacted gateway logs around the unexpected send, the affected channel/plugin, and whether `openclaw doctor --fix` migrated any legacy `nano-banana-pro` config. If reproduced on current main, likely investigation surfaces are `src/infra/heartbeat-runner.ts`, `src/infra/system-events.ts`, `src/cron/service/*`, and image/message tool delivery paths.
