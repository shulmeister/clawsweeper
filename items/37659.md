## Summary

Current main still does not implement the requested safe-mode recovery documentation. The existing TOOLS templates remain generic, OAuth docs describe normal auth-profile storage only, and exact searches found no safe-mode recovery guidance for habitat-parsed.env, visible OAuth re-auth, auth-profiles.provisioned.json, or clearing the safe-mode flag.

## Best Possible Solution

Keep the issue open. The best next step is for a maintainer to either add safe-mode recovery instructions to the generated TOOLS.md/public docs in this repo or route the issue to the external installer/operations surface that owns setup-safe-mode-workspace.sh. The eventual guidance should cover both token sources, visible OAuth re-auth without detached tmux, and the safe-mode flag/recovery-attempt reset exit procedure.
