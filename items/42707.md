## Summary

Keep open. Current `main` still does not implement the PR’s Telegram subagent-session fix end to end: Telegram does not register subagent lifecycle hooks, the spawn path still requires those hooks for `thread=true`, and the resumed agent path still does not hydrate routing from persisted session delivery context.

## Evidence

- **Telegram entry lacks subagent hook registration:** `extensions/telegram/index.ts` only defines the bundled channel entry (`plugin`, `secrets`, `runtime`, `accountInspect`). Unlike Discord, it has no `registerFull(api)` hook registration step for subagent lifecycle wiring.
  - file: [extensions/telegram/index.ts:3](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/extensions/telegram/index.ts#L3)
  - command: `nl -ba extensions/telegram/index.ts | sed -n '1,80p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Discord still uses explicit subagent hook registration:** `extensions/discord/index.ts` calls `registerDiscordSubagentHooks(api)` from `registerFull(api)`, which is the wiring pattern the PR proposed for Telegram but is absent on current `main`.
  - file: [extensions/discord/index.ts:21](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/extensions/discord/index.ts#L21)
  - command: `nl -ba extensions/discord/index.ts | sed -n '1,80p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Spawn path still requires channel subagent hooks:** `src/agents/subagent-spawn.ts` still rejects `thread=true` when no channel plugin registered `subagent_spawning` hooks, returning `thread=true is unavailable because no channel plugin registered subagent_spawning hooks.` Telegram currently has no such registration on `main`.
  - file: [src/agents/subagent-spawn.ts:495](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/agents/subagent-spawn.ts#L495)
  - command: `nl -ba src/agents/subagent-spawn.ts | sed -n '480,540p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Resumed agent run context still ignores persisted delivery context:** `src/agents/command/run-context.ts` only derives channel/account/thread from `opts` and existing `runContext`, and `src/agents/command/attempt-execution.ts` still forwards `messageTo: params.opts.replyTo ?? params.opts.to` and `messageThreadId: params.opts.threadId`. The PR’s session-resume hydration from stored `deliveryContext` is not present.
  - file: [src/agents/command/run-context.ts:5](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/agents/command/run-context.ts#L5)
  - command: `nl -ba src/agents/command/run-context.ts | sed -n '1,140p'; nl -ba src/agents/command/attempt-execution.ts | sed -n '398,412p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **User-facing docs still say Telegram subagent thread sessions are unsupported:** `docs/tools/subagents.md` still says Discord is currently the only supported channel for persistent thread-bound subagent sessions, while `docs/channels/telegram.md` documents only ACP topic-bound spawn for Telegram. That matches the missing implementation on `main`, not the PR’s claimed Telegram subagent support.
  - file: [docs/tools/subagents.md:115](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/docs/tools/subagents.md#L115)

That is partial overlap with the PR, but it does not replace the missing Telegram subagent lifecycle hooks or the missing resume-context hydration.
  - file: [extensions/telegram/src/thread-bindings.ts:610](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/extensions/telegram/src/thread-bindings.ts#L610)
