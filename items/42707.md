## Summary

Keep PR #42707 open. Current main has generic subagent lifecycle hooks and Telegram conversation-binding primitives, but it still lacks the Telegram-specific subagent hook registration and session-resume routing hydration that the PR is trying to add. The PR should not be closed as obsolete; it needs a current-main rebase/adaptation and should address the existing review findings before merge.

## Best Possible Solution

Keep the PR open and rework it against current main: add a Telegram subagent hook API and `registerFull` wiring like Discord/Matrix, prewarm or create the Telegram conversation-binding manager before capability checks, hydrate `resolveAgentRunContext` from `sessionEntry.deliveryContext` on resumes, fall back to hydrated `currentChannelId/currentThreadTs` when invoking the embedded agent, and add Telegram-specific lifecycle/resume tests including prefixed topic targets.
