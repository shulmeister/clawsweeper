## Summary

Keep open. Current main still does not implement the requested orphan `</think>` fallback: the shared sanitizer preserves accumulated text before an unmatched closing reasoning tag, so malformed `[reasoning prose] </think> [final answer]` can still become visible assistant text. The two PRs linked from the issue targeted this exact fix but were closed unmerged, so they do not supersede the open issue.

## Best Possible Solution

Keep #67092 open and fix the shared sanitizer. The narrow product fix should make `stripReasoningTagsFromText` discard accumulated visible text when it sees an unmatched closing reasoning tag outside code, keeping only the tail after the last orphan close when there is final content. Add regression coverage in `src/shared/text/reasoning-tags.test.ts` and `src/shared/text/assistant-visible-text.test.ts` for delivery/history profiles, plus a focused extraction or stream test if the maintainer wants proof for UI/block reply paths.

## Evidence

For an orphan close like `reasoning </think> final`, that keeps `reasoning` and `final` rather than discarding the reasoning prefix.
