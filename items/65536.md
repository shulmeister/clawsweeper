## Summary

Keep open. Current `main` still contains the two direct `os.homedir()` calls this PR replaces in `src/agents/sandbox-paths.ts`, so the change is not implemented on `main`. The repo also already has shared home-resolution helpers and tests that prefer `HOME`/`USERPROFILE` before `os.homedir()`, which makes this PR still plausible rather than obsolete. The only caveat is that the PR body's claim about Node `os.homedir()` behavior does not match the Node v22 docs or a local probe, so the exact reproduction should be maintainer-verified before merge instead of auto-closing it.
