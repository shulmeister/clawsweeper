## Summary

Keep this PR open. Current main still has the exact sandbox tilde-expansion behavior the PR targets: `src/agents/sandbox-paths.ts` expands `~` with `os.homedir()` directly and also abbreviates errors with `os.homedir()`. The PR is a narrow, plausible fix with useful reproduction detail, and the author already addressed the bot review's `shortPath` follow-up in the second PR commit. It is not implemented on main and is not superseded by the broader open `OPENCLAW_HOME` issue.

## Best Possible Solution

Review this PR as a valid implementation candidate. The best path is to keep it open, request or add a focused regression test for sandbox `~` expansion and `shortPath`, and consider changing the implementation to reuse the existing OS-home helper. The broader `OPENCLAW_HOME` report should remain separate because this PR targets OS-home tilde expansion in sandbox paths, not the full internal-state home override policy.
