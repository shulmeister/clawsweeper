## Summary

Keep #9544 open. Current main still uses Baileys `useMultiFileAuthState` for the WhatsApp socket and only hardens `creds.json` persistence; it has not replaced the multi-file credential and signal-key store with a SQL/consolidated production auth store.

## Best Possible Solution

Keep #9544 open until the bundled WhatsApp plugin replaces `useMultiFileAuthState` with a production auth store, migrates existing multi-file credentials and signal keys safely, updates docs/config/auth presence/logout behavior, and adds focused fresh-install plus migration tests.
