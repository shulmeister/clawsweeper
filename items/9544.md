## Summary

Keep #9544 open. Current main still uses Baileys `useMultiFileAuthState(authDir)` for WhatsApp auth and only hardens queued/atomic `creds.json` writes; it has not replaced the multi-file signal-key store with SQL, SQLite/Postgres, or another consolidated production auth store. The related PR #66863 is useful prior art, but it was closed unmerged per the provided GitHub context.

## Best Possible Solution

Keep #9544 open until the bundled WhatsApp plugin replaces `useMultiFileAuthState` with a production auth store, safely migrates existing Baileys multi-file credentials and signal keys, updates status/login/logout/docs/config behavior, and adds focused tests for fresh install, migration, corruption recovery, and key persistence. PR #66863 can be treated as prior art, not shipped code.
