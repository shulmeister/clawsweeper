## Summary

Keep open. Current main still uses Baileys `useMultiFileAuthState` for the WhatsApp auth store, so the requested move away from the multi-file signal-key store is not implemented. The referenced PR #66863 is useful related context and proposed an atomic consolidated auth store, but it was closed unmerged and its referenced commit is not present in this checkout.
