## Summary

Keep #35447 open. Current main still stores cron job ownership and stamps it on cron.add, but cron list/update/remove/run/runs paths do not accept caller context or enforce agent/session ownership. The related implementation PRs are closed unmerged, and the open related issues are narrower or broader rather than canonical replacements.

## Best Possible Solution

Keep #35447 open and implement a core cron caller-context contract across the agent tool, gateway protocol/handlers, CronService contract, and cron service ops. Add focused tests proving list/listPage filtering plus update/remove/run/enqueueRun denial across agent/session boundaries, with explicit admin override and documented legacy-job behavior. Track the sandbox execution comment separately if it still reproduces.
