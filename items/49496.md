## Summary

Current `main` already contains the generic runtime/tool-loading fix for plugin tools disappearing from model-visible tool lists in channel sessions, and the fix shipped in `v2026.4.22`.

## Evidence

- **Live agent runs opt into gateway-bound plugin loading:** The embedded agent execution path for user/group replies now passes `allowGatewaySubagentBinding: true`, so normal channel sessions use the gateway-bound plugin runtime instead of a detached tool load.
  - file: [src/auto-reply/reply/agent-runner-execution.ts:1076](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/auto-reply/reply/agent-runner-execution.ts#L1076)

  - file: [src/agents/pi-tools.ts:608](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/agents/pi-tools.ts#L608)

## Close Comment

Closing as implemented on `main` after Codex review. The current runtime now threads `allowGatewaySubagentBinding: true` through live embedded agent runs and session tool assembly (`src/auto-reply/reply/agent-runner-execution.ts:1076`, `src/agents/pi-tools.ts:608`), and plugin tool resolution reuses the gateway-owned active plugin registry when that gateway-bindable mode is active (`src/plugins/tools.ts:61`, `src/gateway/server-plugins.ts:487`). There is also direct regression coverage for this path in `src/plugins/tools.optional.test.ts:356`, and the shipped `v2026.4.22` changelog calls out the plugin-runtime fix for plugin tools/channels disappearing from mismatched runtime loads (`CHANGELOG.md:2001`, release commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). If this still reproduces on a newer build, please open a fresh issue with the exact version and an affected-session tool inventory dump so maintainers can check for a new channel-specific regression.
