## Summary

Keep open. Current `main` still serves WebChat/Control UI chat history through a bounded `chat.history` path, and the UI still requests only the latest 200 messages. The codebase also documents that this history is intentionally truncated/bounded, while the newer paginated session-history API is separate and not wired into the chat UI. That is not strong evidence that #49777 has already been fixed on `main`.
