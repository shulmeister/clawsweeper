## Summary

Keep this PR open. Current main still does not expose effective sandbox mounts from `openclaw sandbox explain` in JSON or text output, while the PR remains a focused implementation candidate for the feature requested by the now-closed #62988. The PR should be revised before landing because review discussion identifies real correctness gaps around canonical sandbox status, backend-specific mount roots, and unrelated test churn.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Revise it to wire backend-aware effective mounts into `openclaw sandbox explain`, reuse `resolveSandboxRuntimeStatus` for sandboxed/direct decisions, emit empty mounts or an explicit direct-runtime hint when not sandboxed, cover JSON and text output plus Docker binds, agent workspace overrides, main-session aliases, and SSH/OpenShell remote roots, and drop unrelated test-file churn before review.
