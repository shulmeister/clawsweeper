## Summary

Keep #10957 open. Current main has Android companion-node functionality, pairing docs, capability coverage, Play/third-party flavors, and signed AAB build mechanics, but it still does not provide or document Android beta/public access for an existing-Gateway phone node. The existing maintainer-thread context already reached that keep-open conclusion, and current main at 631552c554095b0c520376db9131dac0f5427f94 still matches it.

## Best Possible Solution

Keep #10957 open until maintainers either provide and document an Android beta/public distribution path for existing-Gateway phone nodes, or explicitly route Android beta-access requests to a support/auto-response process with clear public instructions. Continue Android node capability, flavor split, signed bundle, and release-hardening work separately in the Android app and release docs.
