## Summary

Keep #10957 open. Current main has the Android companion-node app, pairing docs, capability registry, flavor split, and local release-bundle mechanics, but it still does not provide or document Android beta/tester access for a phone node paired to an existing Gateway. The prior issue comment also kept it open for this remaining access/documentation gap.

## Best Possible Solution

Keep #10957 open until maintainers either publish and document an Android beta/public distribution path for existing-Gateway Android phone nodes, or explicitly route Android beta-access requests to a support/auto-response process with clear public instructions. Continue Android app capability and release-hardening work in the Android app/release surfaces separately.
