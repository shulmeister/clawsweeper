## Summary

Keep #10957 open. The issue is a concrete Android beta/access request for using an Android phone as a node with camera, location, SMS, and canvas against an existing gateway. Current main has substantial Android node implementation, but the user-facing docs still say the Android app has not been publicly released and the Android README still marks release hardening as incomplete, so the requested beta access is not implemented on main.

## Best Possible Solution

Keep tracking #10957 until maintainers either provide an Android beta/public distribution path and update the Android docs, or make an explicit product/triage decision that Android beta-access requests should use a separate support or auto-response path. Do not close as implemented while the public docs still say the Android app is unreleased.
