## Summary

Keep open. Current main still has fixed workspace bootstrap templates, ClawHub skill/plugin installs, and bundle-style plugin mapping, but it does not ship the proposed Agent Pack export/install/list workflow or PACK.md format. The PR remains potentially useful but needs maintainer product judgment and fixes for the review-blocked path traversal/YAML escaping issues before it could land.

## Best Possible Solution

Keep the PR open until a maintainer decides the product direction. If Agent Packs belong in core, the best path is a smaller, security-reviewed implementation that sanitizes pack names, quotes/validates PACK.md frontmatter, documents the format, and adds focused tests for install/export/list behavior. If maintainers prefer ClawHub/plugin scope, the next step should be an explicit plugin/API design for safe workspace-template scaffolding rather than closing this as already handled by current bundle support.
