## Summary

Current main still lacks the requested webchat/WebSocket `message_sending` hook. The Gateway chat final path builds the final payload and broadcasts it directly, while the external outbound delivery path and inbound reply dispatcher do run `runMessageSending`. The optional `sessionKey` context field already exists, but the actual webchat hook invocation, content rewrite, cancel handling, and failure isolation requested here are still missing.

## Best Possible Solution

Keep this issue open. The likely fix belongs in the Gateway WebChat finalization path, preferably via a small shared helper that calls `getGlobalHookRunner().runMessageSending` only when `message_sending` hooks are registered, passes `channelId: "webchat"`, `sessionKey`, run/conversation correlation, applies rewritten final text, honors `{ cancel: true }`, isolates hook failures so chat delivery continues, and adds focused `src/gateway/server-chat.*.test.ts` coverage for rewrite, cancel, thrown hook, and no-hook behavior. The implementation also needs an explicit decision on whether only the final payload or both flushed delta and final payload should be hook-visible.
