## Summary

Keep open. Current main still has the webchat final delivery path build the final `chat` payload and call `broadcast("chat", payload)` / `nodeSendToSession(...)` directly, with no `message_sending` hook invocation. The shared external outbound path does run `applyMessageSendingHook`, and the plugin docs define `message_sending` as the hook that can rewrite or cancel outbound content. The related closed PR #38340 discussed the same class of bypasses and explicitly listed web chat as follow-up work, but that PR was not merged and did not implement this path.

## Best Possible Solution

Keep the issue open and implement the core gateway wiring: before the webchat final `chat` broadcast in `src/gateway/server-chat.ts`, run `message_sending` with `channelId: "webchat"`, `sessionKey`, and run/conversation context; apply rewritten content, honor `{ cancel: true }`, isolate hook failures, and add regression coverage for WebSocket final delivery.
