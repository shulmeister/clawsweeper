## Summary

Keep PR #70676 open. Current main still has the exact silent `socket.close()` revocation failure paths described in the PR, and I did not find an equivalent log-and-terminate fallback or regression tests on main. The PR is a focused gateway/auth hardening change with useful discussion, including a bot review issue that the author addressed in the PR branch.

## Best Possible Solution

Keep the PR open. The best path is to land a focused log-then-terminate implementation for shared-auth revocation and device-token removal, wire gateway loggers through the shared-auth call sites, preserve the addressed aux-handler logger binding fix, and add the proposed unit coverage for throwing `socket.close()` plus loop-continuation behavior.
