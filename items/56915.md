## Summary

Current `main` still only discovers skills at the root itself or one directory below each configured skills root. That covers repo-style `extraDirs` roots like `dir/skills/<skill>/SKILL.md`, but it does not cover grouped managed skills like `~/.openclaw/skills/coze/koze-retrieval/SKILL.md`. The April 21 comment documents a workaround (`skills.load.extraDirs` or moving the skill), not an implementation on `main`, so this issue should remain open.

## Evidence
