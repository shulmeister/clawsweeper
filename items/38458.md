## Summary

Keep open. Current main still does not provide the requested per-channel or broader tool-output visibility policy. The existing global `messages.suppressToolErrors` only suppresses non-mutating warnings, while the reported `message send` class is mutating and remains user-visible by design. Related #71282 was a narrower TTS leak, and #51091 is still open/unmerged and only addresses global all-tool suppression.

## Best Possible Solution

Keep the issue open for a core runtime/config design: define an explicit tool-output visibility policy with global defaults and per-channel overrides, apply it in embedded-run payload shaping and streaming chat delivery, keep tool errors available to the agent, update schema/docs, and add focused tests for mutating `message`/`exec` failures plus Telegram/Discord streaming delivery.
