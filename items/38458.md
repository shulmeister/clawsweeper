## Summary

Keep open. Current main still lacks the requested per-channel tool-error visibility control, and the existing global `messages.suppressToolErrors` intentionally does not suppress mutating tool failures such as the reported `message` send failure.

## Best Possible Solution

Keep tracking this in core until OpenClaw has a documented effective tool-output visibility policy, likely combining a global default with per-channel overrides, that can suppress user-facing tool-error payloads while still returning errors to the agent. The implementation should cover mutating tools such as `message` and `exec`, update config schema/docs, and add focused tests around `buildEmbeddedRunPayloads` plus at least one streaming channel path.
