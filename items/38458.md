## Summary

Keep open. Current main still lacks the requested per-channel or broader tool-output visibility policy, and the shipped `messages.suppressToolErrors` behavior remains narrower: mutating failures such as `message send` stay user-visible, with tests asserting that contract. The related #71282 TTS leak was a narrower fix, and open PR #51091 is not merged or shipped.

## Best Possible Solution

Keep this issue open for a core runtime/config design: define an explicit tool-output visibility policy with a global default and per-channel overrides, wire it through embedded-run payload shaping and streaming chat dispatch so mutating `message`/`exec` failures can be hidden from chat while still returned to the agent, then update schema/docs and add focused tests for `buildEmbeddedRunPayloads` plus a Telegram streaming/final-delivery path.
