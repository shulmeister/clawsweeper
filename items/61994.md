## Summary

Current main implements the requested recovery behavior: a new lifecycle start reactivates a completed session by setting it back to running and clearing terminal fields, and the gateway persists that lifecycle-start update for the session key. The Feishu inbound path dispatches into the same chat/session pipeline, and the send path does not reject an existing session merely because its previous stored status was done.
