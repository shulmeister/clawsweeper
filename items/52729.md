## Summary

Current `main` already contains a shipped gateway/plugin webhook-route fix that matches this LINE 404 report from `v2026.3.13`: plugin HTTP routes are pinned to the gateway startup registry, plugin dispatch resolves from that live HTTP-route registry, and LINE still returns `400 Missing X-Line-Signature header` when its route is reached.

## Close Comment

Closing as implemented on current `main`. Codex review found that OpenClaw now pins plugin HTTP/webhook routes to the gateway startup registry (`src/gateway/server-runtime-state.ts:108`, `src/plugins/runtime.ts:138`), and the shipped changelog for `v2026.3.22` explicitly calls out this webhook-route churn fix (`CHANGELOG.md:2395`). Current LINE webhook handling still returns `400 Missing X-Line-Signature header` when the route is reached (`extensions/line/src/webhook-node.ts:69`), and there is regression coverage for webhook routes surviving registry changes (`src/gateway/server/plugins-http.test.ts:300`). This makes the `v2026.3.13` LINE 404 report obsolete on supported current releases. If you can still reproduce this on `v2026.3.22` or newer, please open a fresh issue with current startup logs and the exact config.
