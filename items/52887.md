## Summary

Keep open. Current `main` has not implemented the PR's requested WhatsApp behavior: the live WhatsApp inbound dispatch still suppresses every `tool` payload, while the shared reply pipeline still produces media-bearing tool payloads. The original branch also targeted an older file layout and would need a refresh against the current dispatcher split, with care to avoid leaking interim tool text as media captions.
