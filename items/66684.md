## Summary

Keep open. Current main still contains the concrete remote-script execution pattern reported in `Dockerfile:50`, and the same Bun `curl | bash` example is still documented for Docker rebuilds. Several credential-pattern examples appear to be stale line numbers or documentation/changelog false positives, but the remaining Dockerfile and docs surface is real enough that this should not be conservatively closed.

## Best Possible Solution

Keep the issue open and narrow it to supply-chain/docs hardening: either replace the Docker Bun install with a pinned or verified installation path, or document why the project accepts the current build-time risk, and update the Docker docs example accordingly. The stale/false-positive credential-pattern hits can be closed out in the issue discussion once the actionable Docker/script surfaces are decided.
