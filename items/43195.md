## Summary

Keep #43195 open. Current main partially supersedes the PR by sharing the plugin unhandled-rejection handler registry through `globalThis`, but it still does not make `installUnhandledRejectionHandler()` idempotent: each call still attaches a new `process.on("unhandledRejection", ...)` listener. The PR remains useful if rebased onto the current implementation and covered with an idempotency regression test.

## Best Possible Solution

Keep the PR open or replace it with an equivalent maintainer patch that preserves the current global handler registry, adds a shared/global installed flag for `installUnhandledRejectionHandler()`, and adds a focused regression test proving repeated installs attach only one `unhandledRejection` listener while plugin-registered handlers remain shared.
