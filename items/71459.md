## Summary

Keep open. Current main still emits the generic missing register/activate diagnostic in both plugin loader paths and does not log field-level bundled channel entry contract failures. The PR addresses a real remaining diagnostic gap, but its current patch also drops the existing OPENCLAW_PLUGIN_LOAD_DEBUG module-shape detail for non-bundled missing-register failures, as noted in the PR review comments, so it needs revision rather than closure.

## Best Possible Solution

Keep #71459 open and ask for a small revision: preserve `formatMissingPluginRegisterError(mod, env)` for non-bundled missing-register failures in both loader paths, detect bundled channel entries from the already-normalized `record.kind` with the existing kind helper, and add focused tests for the bundled wrong-loader message plus field-level bundled entry validation logging. If that lands cleanly, this is the right code-path fix rather than a ClawHub/external item.
