## Summary

Keep #72146 open. It is a fresh regression report against v2026.4.24 with concrete `gateway probe` output and logs showing the LaunchAgent-started gateway repeatedly reaches `ready` and advertises Bonjour, while the local loopback WebSocket probe still times out and TUI stays disconnected. Current main has LaunchAgent install/start/restart recovery paths and diagnostic probes, but I did not find source, tests, docs, history, or a canonical related item proving this specific LaunchAgent-ready-but-unreachable/TUI-disconnected regression is fixed.

## Best Possible Solution

Keep the issue open and triage it as a current macOS LaunchAgent regression. Ask for `openclaw gateway status --deep --json`, `openclaw gateway diagnostics export`, `launchctl print gui/$UID/ai.openclaw.gateway`, the generated LaunchAgent plist ProgramArguments/EnvironmentVariables with secrets redacted, and port ownership around 18789 while the TUI is disconnected. The likely fix belongs in the LaunchAgent lifecycle/restart-health/status-probe path or the gateway WebSocket startup/handshake path, with a regression test that distinguishes `launchd says running/log says ready` from `loopback probe can actually connect`.
