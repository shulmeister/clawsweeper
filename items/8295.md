## Summary

Keep #8295 open. Current main still does not implement Telegram allowBots parity: the Telegram config schema/types do not accept allowBots, the Telegram group/topic inbound path has no allowBots policy comparable to Discord/Slack, and the existing channel_post/Bot-to-Bot QA support is a separate channel-oriented path rather than the requested group/topic control.

## Best Possible Solution

Keep #8295 open until Telegram implements a real allowBots policy in the bundled Telegram plugin/config contract. The likely fix should add account/group/topic allowBots schema and types, update generated config metadata/docs/UI surfaces, apply bot-authored group/topic filtering consistently with Discord/Slack while always dropping self echoes, preserve channel_post behavior, and add regression tests plus live QA coverage around BotFather Bot-to-Bot Communication Mode.
