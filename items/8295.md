## Summary

Keep #8295 open. Current main still does not implement Telegram allowBots parity for groups/topics: the Telegram schema, public config types, generated metadata, and runtime path all lack an allowBots policy. The Discord/Slack parity surfaces exist, while Telegram's current bot-to-bot support is a separate channel_post path. The Plaiground ClawHub skill mentioned in the discussion is a useful Discord-based workaround, not a fix for Telegram groups/topics.

## Best Possible Solution

Keep #8295 open until the bundled Telegram plugin implements a real allowBots policy in schema, public types, generated config metadata/UI, docs, and runtime filtering. The fix should support account/group/topic precedence, always drop self echoes, preserve channel_post behavior, add regression tests, and include live QA coverage around BotFather Bot-to-Bot Communication Mode.
