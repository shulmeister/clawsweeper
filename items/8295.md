## Summary

Keep #8295 open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still does not expose Telegram allowBots for groups/topics in the public config types, strict schema, generated metadata/UI help, docs, or inbound group runtime path. The Plaiground ClawHub skill mentioned in the issue is a useful Discord workaround, and PR #68325 is a relevant open implementation attempt, but neither ships the requested bundled Telegram plugin behavior yet.

## Best Possible Solution

Keep #8295 open until the bundled Telegram plugin implements allowBots across public config types, strict schema, generated config metadata/UI help, docs, inbound runtime filtering, and regression tests. PR #68325 or a successor is the likely implementation path, but it needs maintainer review for account/group/topic precedence, self-echo loop prevention, mention gating, and channel_post compatibility.
