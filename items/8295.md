## Summary

Keep #8295 open. Current main at 82262387651ab270b565f71533ce4937c9c00dc6 still does not expose Telegram `allowBots` in config types, strict schema, generated metadata, user docs, or the inbound group/topic runtime filter. The linked Plaiground ClawHub skill is a useful Discord workaround, PR #8310 was closed unmerged, and PR #68325 is still an open candidate rather than shipped behavior.

## Best Possible Solution

Keep #8295 open until the bundled Telegram plugin implements `allowBots` across account/group/topic config types, strict schema, generated metadata/help, docs, and inbound group/topic filtering with regression tests. PR #68325 is the current candidate to review; if maintainers reject Telegram group support, document the BotFather mode/admin/privacy requirements and supported alternatives such as Telegram channels or the Plaiground Discord/ClawHub workaround.
