## Summary

Keep #8295 open. Current main still does not implement Telegram `allowBots` parity for groups/topics: the Telegram schema, public config types, generated metadata, docs, and runtime path still lack a bot-authored group/topic policy. The existing Telegram bot-to-bot behavior is the separate `channel_post` path, and the Plaiground ClawHub skill mentioned in the issue is a useful Discord workaround, not a Telegram fix.

## Best Possible Solution

Keep #8295 open until the bundled Telegram plugin implements a real `allowBots` policy across schema, public config types, generated metadata/UI, docs, and runtime filtering. The implementation should support account/group/topic precedence, always drop self echoes, preserve existing `channel_post` behavior, add regression tests, and include live Telegram QA around BotFather Bot-to-Bot Communication Mode.
