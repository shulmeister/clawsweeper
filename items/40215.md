## Summary

Keep open. Current main still implements `/usage` as a per-response token/cache/cost footer, with `full` only adding the session key. It does not show model plus cumulative context used/window/percent in the reply footer, and `agents.defaults.responseUsage` is not present in the config schema. The related PRs #48635 and #69618 are open/unmerged attempts at this area, so #40215 remains useful as the canonical tracking issue.

## Best Possible Solution

Keep #40215 open as the canonical request. The best path is to either land a reviewed version of #69618 or implement the narrow core behavior directly: enrich the shared response usage formatter with model/context-window data, add an inherited `agents.defaults.responseUsage` config contract with schema/docs/tests, preserve session override semantics, and cover normal reply plus block-streaming/footer composition paths.
