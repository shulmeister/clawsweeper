## Summary

Keep #40215 open. Current main still treats `/usage` as a per-response token/cache/cost footer, with `full` adding only the session key. It does not show active model plus cumulative context used/window/percent in the reply footer, and there is no `agents.defaults.responseUsage` config contract. Related #65487 was closed as a duplicate of this issue, while #48635 and #69618 are still open/unmerged proposals rather than shipped fixes.

## Best Possible Solution

Keep #40215 open as the canonical request. The best implementation path is to enrich the shared response usage footer with active model and fresh context used/window/percent data, add an inherited `agents.defaults.responseUsage` config contract with schema/types/docs/tests, preserve session-level override semantics, and coordinate with #69618 or a narrower reviewed replacement before closing this issue.
