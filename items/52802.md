## Summary

Keep open. Current `main` still routes OpenAI-family streaming through the `openai` SDK, and the shipped SDK versions still contain the same `event`-only SSE -> `JSON.parse(sse.data)` path described in #52802. I found no repo-side guard that would prevent an empty-data SSE frame from reaching the SDK parser.
