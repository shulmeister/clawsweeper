## Summary

Keep open. Current main still has provider docs, provider plugin catalogs, model auth/probe status, and plugin-contract compatibility/deprecation tracking, but it does not publish the requested public provider compatibility matrix with provider x model-family x capability lifecycle statuses, deprecation deadlines, tracked broken-issue links, or doctor output for that matrix. The Matrix plugin mismatch linked in the discussion is relevant motivation, but that separate closed bug does not supersede this broader policy/docs/CLI request.

## Best Possible Solution

Keep the issue open until OpenClaw has a maintainer-approved provider compatibility policy and a machine-readable matrix, likely backed by provider/plugin metadata or generated docs data, surfaced in public docs and `openclaw doctor`/models status output. The implementation should distinguish operational auth/probe failures from lifecycle support states and should link broken/deprecated entries to canonical issues or removal timelines.
