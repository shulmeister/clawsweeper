## Summary

Keep open. Current main has provider lists, plugin compatibility/deprecation tracking, model-catalog listing statuses, and runtime auth/probe status, but it still does not publish the requested provider x model-family x capability compatibility matrix with lifecycle states, deprecation deadlines, broken-issue links, or doctor output. The Matrix plugin mismatch linked in the discussion is relevant motivation, but that closed bug is narrower and does not supersede this policy/docs/CLI proposal.

## Best Possible Solution

Keep the issue open until maintainers approve and ship a provider compatibility policy plus a machine-readable matrix, likely extending provider/plugin model metadata with lifecycle status, capability scope, deprecation deadline, replacement, and canonical issue links. Generate public docs from that data and surface a concise view in doctor or models status while keeping runtime probe failures distinct from lifecycle support states.
