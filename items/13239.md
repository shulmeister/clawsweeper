## Summary

Keep #13239 open. Current main has provider docs, auth/probe status, model catalog metadata, and plugin compatibility/deprecation registries, but it still does not ship the requested public provider x model-family x capability compatibility matrix with stable/experimental/deprecated/broken states, deadlines, tracked issue links, generated docs, and doctor output. The linked Matrix plugin issue is useful motivation but is a specific closed plugin regression, not a canonical replacement for this broader provider policy request.

## Best Possible Solution

Keep the issue open until maintainers approve and ship a provider compatibility policy plus a machine-readable registry keyed by provider, model family, capability, lifecycle status, deadline, replacement, and canonical issue link. Generate the public docs page from that registry and surface a concise doctor or models-status view while keeping live auth/probe health separate from support lifecycle state.
