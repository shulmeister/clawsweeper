## Summary

Current main has provider documentation, model list/status auth surfaces, live auth probes, and plugin compatibility diagnostics, but it does not implement the requested public provider compatibility matrix, provider/model-family capability statuses, deprecation deadlines, or doctor output for that matrix. The Matrix plugin mismatch linked in the issue discussion is relevant evidence for the need, but that separate closed bug does not supersede this broader policy/docs request.
