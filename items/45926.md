## Summary

Keep open. Current main still has the reported timeout gap: for `timeoutSeconds > 0`, `sessions_send` starts the target run and waits, but if `agent.wait` returns timeout it immediately returns `status: "timeout"` without calling `startA2AFlow`. The A2A helper can wait on a `waitRunId`, and the fire-and-forget path already calls it that way, but the timed-wait timeout branch does not.
