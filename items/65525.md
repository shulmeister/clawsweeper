## Summary

Keep this issue open. Current main does not auto-propagate gateway `NODE_OPTIONS` or proxy env into MCP stdio children: stdio launch config still derives `env` only from the server config, and the transport only merges MCP SDK safe defaults with that explicit env. Separately, an open unmerged PR (#65574) explicitly claims to fix this issue, so maintainer workflow says the issue should remain open until that PR is reviewed, landed, or closed.

## Best Possible Solution

Keep the issue open and review #65574 as the implementation candidate, updating it as needed for the current stdio env safety policy. If maintainers do not want automatic startup-hook propagation, the safer path is to land a narrower proxy-env solution and update the MCP docs to explicitly document what stdio children do and do not inherit.
