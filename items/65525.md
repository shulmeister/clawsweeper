## Summary

Keep open. Current `main` still launches MCP stdio children with the SDK default inherited environment plus only explicitly configured `mcp.servers.<name>.env`, so ambient proxy variables and gateway-level `NODE_OPTIONS` are not propagated. The report also has a related open fix PR (#65574), which is additional evidence that this is not already resolved on `main`.

## Evidence

- **Current stdio launch config only forwards configured env:** `resolveStdioMcpServerLaunchConfig()` returns `env: toMcpEnvRecord(raw.env, ...)` and does not merge any parent-process proxy variables or `NODE_OPTIONS` defaults.
  - file: [src/agents/mcp-stdio.ts:41](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/agents/mcp-stdio.ts#L41)
