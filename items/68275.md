## Summary

Keep open. Current `main` still rejects `sessions_spawn` calls when `runtime!="acp"` and either `streamTo` or `resumeSessionId` is present, and the only identified fix is PR #68397, which is still open and unmerged. The latest release `v2026.4.23` still contains the same hard-error path.
