## Summary

Close as superseded by open PR #68397. Current main still has the reported `runtime="subagent"` hard failure when `streamTo:"parent"` is present, but #68397 is an explicit, narrower fix PR for this exact behavior and updates the same implementation and tests to silently drop ACP-only fields for subagent spawns.

## Best Possible Solution

Close this issue as superseded and keep the remaining work concentrated in PR #68397, which is the canonical fix path for silently dropping ACP-only `streamTo` and `resumeSessionId` fields from `runtime=subagent` spawns while preserving ACP behavior.
