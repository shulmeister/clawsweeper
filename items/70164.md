Keep open. The issue is recent, has reporter follow-up, and current main does not provide issue-specific proof that the MiniMax/tool-call websocket sequence is fixed. The Gateway can still emit `state:"final"` without a message when no assistant text stream reaches the chat buffer, which matches the reported symptom.

## Best Possible Solution

Keep the issue open. Add a focused regression for a MiniMax/OpenAI-compatible tool-call turn that emits tool result, then assistant message_end/final text, then lifecycle end, and assert Gateway WS clients receive an assistant event plus chat delta/final message content before `state:"final"`.
