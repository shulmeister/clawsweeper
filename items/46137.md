## Summary

Keep open. Current main has useful improvements for this area: the legacy chrome-relay path was removed/migrated, `tabs` now exposes `suggestedTargetId`/`tabId`, and explicit `/navigate` responses try to resolve renderer-swap target ids. But the reported post-submit path is not fully fixed: `/act` still executes against the resolved current tab and returns that same target id without the post-navigation target reconciliation used by `/navigate`; follow-up focus/screenshot/evaluate calls with a stale raw target id still route through `ensureTabAvailable()` and fail with `tab not found`. The related #63343 covers a different `wsUrl` discovery race, so this should not be auto-closed as a duplicate.

## Evidence

- **Post-submit act path does not reconcile targetId after navigation:** The `/act` route calls `executeActViaPlaywright` with `targetId: tab.targetId` and then returns `jsonOk()` using the same `tab.targetId`; unlike `/navigate`, it does not call `resolveTargetIdAfterNavigate()` after clicks/evaluate/type actions that may submit a form or trigger navigation.
  - file: [extensions/browser/src/browser/routes/agent.act.ts:624](https://github.com/openclaw/openclaw/blob/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071/extensions/browser/src/browser/routes/agent.act.ts#L624)
  - sha: [28497515fe1e](https://github.com/openclaw/openclaw/commit/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071)
- **Follow-up stale target ids still fail selection:** `ensureTabAvailable()` resolves the caller's target id against the current tab list and throws `BrowserTabNotFoundError` when the id is no longer present. That preserves the reported `tab not found` behavior for stale raw target ids.
  - file: [extensions/browser/src/browser/server-context.selection.ts:51](https://github.com/openclaw/openclaw/blob/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071/extensions/browser/src/browser/server-context.selection.ts#L51)
  - sha: [28497515fe1e](https://github.com/openclaw/openclaw/commit/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071)
- **Stable tab aliases are keyed by live raw targetId:** `assignTabAliases()` removes alias entries whose raw `targetId` is no longer live, so a renderer-swap/new-target replacement does not obviously preserve the prior `tabId` across the stale-target boundary.
  - file: [extensions/browser/src/browser/server-context.tab-ops.ts:118](https://github.com/openclaw/openclaw/blob/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071/extensions/browser/src/browser/server-context.tab-ops.ts#L118)
  - sha: [28497515fe1e](https://github.com/openclaw/openclaw/commit/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071)
- **Target reconciliation exists only for explicit navigate:** The browser `/navigate` route calls `resolveTargetIdAfterNavigate()` and returns the resolved current target id, showing a narrower fix for explicit navigation rather than the broader form-submit/action path described in #46137.
  - file: [extensions/browser/src/browser/routes/agent.snapshot.ts:292](https://github.com/openclaw/openclaw/blob/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071/extensions/browser/src/browser/routes/agent.snapshot.ts#L292)
  - sha: [28497515fe1e](https://github.com/openclaw/openclaw/commit/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071)
- **Tool-level stale retry is narrow:** The agent tool retries stale Chrome MCP target errors only for `hover`, `scrollIntoView`, or `wait`, and only when exactly one tab remains. It does not cover the issue's focus, screenshot, or evaluate failures after form submission.
  - file: [extensions/browser/src/browser-tool.actions.ts:528](https://github.com/openclaw/openclaw/blob/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071/extensions/browser/src/browser-tool.actions.ts#L528)
  - sha: [28497515fe1e](https://github.com/openclaw/openclaw/commit/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071)
- **Current tests still assert stale target rejection in one profile path:** Remote-profile tests explicitly expect `ensureTabAvailable("STALE_TARGET")` to reject with `tab not found` even when only one tab remains, confirming current main does not have a universal stale-target recovery contract.
  - file: [extensions/browser/src/browser/server-context.remote-profile-tab-ops.playwright.test.ts:137](https://github.com/openclaw/openclaw/blob/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071/extensions/browser/src/browser/server-context.remote-profile-tab-ops.playwright.test.ts#L137)
