## Summary

Keep open. Current main still does not implement direct `params.user` passthrough, and the linked issue was closed specifically to keep review on this PR, not because the behavior shipped. Main has a guarded `extra_body` escape hatch for static OpenAI-compatible Completions payload patches, but that is not the same API, does not cover the proposed `agents.defaults.params.user` config, and does not resolve the PR review feedback about provider/API gating and tests.

## Best Possible Solution

Keep the PR open and request changes, or replace it with a maintainer-owned patch. The best implementation is a guarded `params.user` payload wrapper limited to provider/API families that actually support the field, with string normalization and tests proving injection for supported OpenAI/OpenRouter-style payloads and no mutation for unsupported providers. If maintainers decide the existing `extra_body` escape hatch is sufficient, close this PR only with that explicit product rationale and a docs pointer.
