## Summary

Keep this issue open. Current main still lacks both the requested pinned facts editor and the broader persistent Control UI panel/plugin registration seam described in the discussion. The existing UI exposes raw core-file editing for USER.md/MEMORY.md-style context, while the plugin API can register tools/routes/gateway methods/memory hooks but not mount persistent sidebar/dashboard panels in the Control UI.

## Best Possible Solution

Keep this issue open unless maintainers split a canonical Control UI panel extension API issue. The best implementation path is an additive gateway/plugin-SDK registration and SPA mount contract for persistent Control UI panels, followed by a pinned facts editor with explicit storage and guaranteed context-injection semantics.
