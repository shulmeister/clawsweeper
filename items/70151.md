## Summary

Keep PR #70151 open. Current main still builds terminal payloads from the attempt-local assistant snapshot and immediately surfaces the `payloads=0` incomplete-turn error without polling the persisted session file for a late assistant message. The shipped #71467 final-answer fallback is adjacent, but it only helps when `lastAssistant` is already present in the snapshot; it does not cover this PR's stale-snapshot / late persisted assistant race.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it on current main, adapt the recovery to coexist with the existing #71467 final-answer fallback, and keep a focused regression test proving a late assistant message persisted to the session file prevents the false `payloads=0` incomplete-turn error.
