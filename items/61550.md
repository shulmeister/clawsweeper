## Summary

Keep open. Current main still has a Telegram preview-streaming final-delivery path that can finalize by editing/materializing the preview instead of going through `deliverReplies`, while Telegram also overrides the generic auto-reply `message_sending` before-delivery hook with an identity callback. The related #69807 closure and #70118-style generic hook evidence do not cover this specific streaming-preview path.

## Evidence

- **Telegram disables the generic auto-reply hook for this dispatcher:** `dispatchTelegramMessage` passes `beforeDeliver: async (payload) => payload` into `dispatchReplyWithBufferedBlockDispatcher`, so the generic `message_sending` before-delivery hook is not installed for this Telegram path.
  - file: [extensions/telegram/src/bot-message-dispatch.ts:759](https://github.com/openclaw/openclaw/blob/4069c81b159087cb71f6b64676d533cf0d897d68/extensions/telegram/src/bot-message-dispatch.ts#L759)
  - command: `nl -ba extensions/telegram/src/bot-message-dispatch.ts | sed -n '740,835p'`
  - sha: [4069c81b1590](https://github.com/openclaw/openclaw/commit/4069c81b159087cb71f6b64676d533cf0d897d68)
- **Generic hook only applies when no channel beforeDeliver is provided:** The auto-reply helper uses `params.dispatcherOptions.beforeDeliver ?? buildMessageSendingBeforeDeliver(...)`; Telegram's identity `beforeDeliver` therefore wins over the generic hook wiring added for inbound reply delivery.
  - file: [src/auto-reply/dispatch.ts:151](https://github.com/openclaw/openclaw/blob/4069c81b159087cb71f6b64676d533cf0d897d68/src/auto-reply/dispatch.ts#L151)
