## Summary

Close as obsolete/cannot reproduce on current main. PR #64444 documents a global npm install failure caused by @lancedb/lancedb on s390x, but current main and the latest release package no longer install LanceDB as part of the root OpenClaw package. LanceDB is plugin-local to memory-lancedb, postinstall has no eager bundled-plugin dependency targets, and the loader stages bundled runtime deps only when an enabled bundled plugin is being loaded. Current docs do not contain an s390x page, but this PR's proposed --force workaround is stale against the current install path.

## Best Possible Solution

Close this stale PR rather than merge the --force install guide. Keep the current plugin-local runtime dependency design; if s390x documentation is still desired, write a new short docs note that says normal OpenClaw install follows the standard Linux/Node path and separately documents any memory-lancedb limitation when users opt into that plugin.
