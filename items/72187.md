## Summary

Keep #72187 open. Current main still lacks the proposed public plugin service lifecycle test harness, does not export a lifecycle helper from `openclaw/plugin-sdk/testing`, and `startPluginServices` still hardcodes `STATE_DIR` instead of accepting a caller-provided test `stateDir`. The PR is a focused SDK/testing seam, not a ClawHub-only plugin request.

## Best Possible Solution

Keep #72187 open for normal maintainer review. If maintainers want this public SDK testing seam, the best path is a focused core change that adds `createPluginServiceLifecycleTestHarness` under `openclaw/plugin-sdk/testing`, threads an optional test `stateDir` through `startPluginServices`, documents the helper, and updates the SDK API/export baselines with targeted lifecycle tests.
