## Summary

Current `main` already implements configurable inbound image summarization through the media-understanding pipeline: inbound images can be rewritten into `[Image]` description blocks for agent/session context while the original attachment remains accessible, and the changelog shows this shipped in release `2026.1.16-1`.

## Close Comment

Closing as implemented on current `main` after Codex review. OpenClaw already has configurable inbound media understanding under `tools.media.*`: it can rewrite inbound images into `[Image] ... Description:` text before the reply pipeline (`src/media-understanding/apply.ts`), keep the original attachment reachable (`src/auto-reply/media-note.ts`), and explicit image-understanding config can force that path even when the active chat model already supports vision (`src/media-understanding/runner.ts`, `src/media-understanding/runner.vision-skip.test.ts`). The changelog shows the feature shipped in `2026.1.16-1` (`CHANGELOG.md`). The config shape is different from the proposed `imageSummaryOnIngest: true`, but the requested capability is already present.
