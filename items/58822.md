## Summary

Keep open. Current main still appears to implement the reported precedence bug: both the normal subagent resolver and the cron isolated-agent resolver check the parent/agent model before `agents.defaults.subagents.model`, while the docs and issue discussion say the global subagent default should win unless a per-agent subagent model is set.

## Evidence

- **Main resolver still shadows global subagent default:** `resolveSubagentConfiguredModelSelection` resolves `agentConfig?.subagents?.model`, then `agentConfig?.model`, then `params.cfg.agents?.defaults?.subagents?.model`, matching the reported regression rather than the requested precedence.
  - file: [src/agents/model-selection.ts:226](https://github.com/openclaw/openclaw/blob/7bbd47349ec37f01a8f2b1dc219bf950615f361b/src/agents/model-selection.ts#L226)
  - command: `GitHub current-main source inspection`
  - sha: [7bbd47349ec3](https://github.com/openclaw/openclaw/commit/7bbd47349ec37f01a8f2b1dc219bf950615f361b)
- **Cron isolated-agent path has the same precedence:** `subagentModelRaw` resolves `params.agentConfigOverride?.subagents?.model`, then `params.agentConfigOverride?.model`, then `params.cfg.agents?.defaults?.subagents?.model`, so the second affected path is also still present.
  - file: [src/cron/isolated-agent/model-selection.ts:66](https://github.com/openclaw/openclaw/blob/7bbd47349ec37f01a8f2b1dc219bf950615f361b/src/cron/isolated-agent/model-selection.ts#L66)
  - command: `GitHub current-main source inspection`
  - sha: [7bbd47349ec3](https://github.com/openclaw/openclaw/commit/7bbd47349ec37f01a8f2b1dc219bf950615f361b)
- **Public docs support the reporter's expected behavior:** The Sub-Agents docs say the default model inherits the caller unless `agents.defaults.subagents.model` or per-agent `agents.list[].subagents.model` is set, and explicit `sessions_spawn.model` still wins.
  - file: [docs/tools/subagents.md](https://github.com/openclaw/openclaw/blob/7bbd47349ec37f01a8f2b1dc219bf950615f361b/docs/tools/subagents.md)
  - command: `docs.openclaw.ai/tools/subagents inspection`
  - sha: [7bbd47349ec3](https://github.com/openclaw/openclaw/commit/7bbd47349ec37f01a8f2b1dc219bf950615f361b)
- **Issue discussion confirms real-world regression:** Commenter SonicBotMan confirmed that after v2026.3.31, subagents inherited the parent agent model instead of the configured subagent default in a multi-model setup, and agreed the fix is to swap `agentConfig?.model` and `cfg.agents.defaults.subagents.model`.
