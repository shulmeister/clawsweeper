## Summary

Current `main` no longer uses the old hardcoded env map the report referenced, but there is still no code or manifest entry for provider id `bailian` or env var `BAILIAN_API_KEY`. The exact report is not strongly proven fixed on `main`, so this should stay open for maintainer triage rather than be conservatively closed.

## Evidence

- **Auth env lookup is now manifest-driven:** `resolveEnvApiKey()` normalizes the provider id, loads provider env candidates, and checks those candidates instead of consulting a static `envMap` in `model-auth.ts`. That removes the exact old implementation the issue cited, but does not by itself prove `bailian` support.
