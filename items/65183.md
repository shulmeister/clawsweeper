## Summary

Current `main` and the latest stable release already use a synchronous `webhooks` plugin `register(api)` implementation, test that it returns `undefined`, and reject async plugin registration at the loader boundary, so the reported warning path is no longer present.

## Evidence

- **Bundled webhooks plugin is synchronous on main:** `extensions/webhooks/index.ts` exports `register(api: OpenClawPluginApi) { registerWebhookRoutes(api); }` with no `async` and no returned promise.
  - file: [extensions/webhooks/index.ts:50](https://github.com/openclaw/openclaw/blob/a5db42862d76fce18e5d7fa57f3907a543e61137/extensions/webhooks/index.ts#L50)
  - command: `nl -ba extensions/webhooks/index.ts | sed -n '1,220p'`
  - sha: [a5db42862d76](https://github.com/openclaw/openclaw/commit/a5db42862d76fce18e5d7fa57f3907a543e61137)
- **Regression test asserts synchronous webhooks registration:** `extensions/webhooks/index.test.ts` calls `plugin.register(...)`, asserts the result is `undefined`, and verifies route registration happened immediately.
  - file: [extensions/webhooks/index.test.ts:34](https://github.com/openclaw/openclaw/blob/a5db42862d76fce18e5d7fa57f3907a543e61137/extensions/webhooks/index.test.ts#L34)
