## Summary

Keep open. Current main still classifies `gateway` action `config.schema.lookup` as mutating because `isMutatingToolCall` only compares the full dotted action string against the flat `READ_ONLY_ACTIONS` set. The gateway method is valid and read-only, and downstream payload policy still shows warnings for mutating tool failures even when the assistant produced a user-facing reply, so this PR remains a useful focused bug fix rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open and either land this root-cause classification fix or apply the same gateway-only leaf-verb logic on main with regression coverage for `config.schema.lookup`, read-only dotted gateway actions, mutating gateway writes such as `config.set`, and invalid cron/canvas actions. After that ships, close linked issue #42856 and treat narrower warning-suppression PRs as redundant if they no longer add behavior.
