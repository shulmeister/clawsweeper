## Summary

Keep #43634 open. Current main still has the exact export HTML template/replacement mismatch described by the PR: the checked-in template contains formatter-expanded MARKED_JS, HIGHLIGHT_JS, and JS blocks, while the generator still performs literal compact `{{...}}` replacements. The latest release tag in this checkout has the same affected template and replacement logic, and `.gitattributes` still lacks the proposed protection for `template.html`. The related bug reports #43616 and #43620 therefore remain real, and this PR is not implemented on main.

## Best Possible Solution

Keep this PR or a canonical equivalent open until main restores the compact export-html placeholders and adds regression coverage against the real `template.html` file. The final fix should also decide whether `.gitattributes`, formatter ignore rules, or a more tolerant replacement helper is the best prevention against future formatter damage.
