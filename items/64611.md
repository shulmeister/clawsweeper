## Summary

Current main still has the heartbeat task clock-skew bug this PR describes. The PR remains a focused implementation candidate: `isTaskDue` still treats a future `lastRunMs` as not due, and heartbeat task mode can then skip with `no-tasks-due` instead of running the task after a backward clock correction.

## Best Possible Solution

Keep this PR open and have a maintainer review or apply the narrow equivalent change: add `lastRunMs > nowMs` handling in `src/auto-reply/heartbeat.ts`, keep a focused `isTaskDue` regression test for future timestamps, refresh the branch if needed, then run the targeted heartbeat test and changed gate before landing.
