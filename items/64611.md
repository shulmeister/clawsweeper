## Summary

Keep open. Current main still has the reported clock-skew behavior in `isTaskDue`: future `lastRunMs` values are not treated as due, and there is no `isTaskDue` regression coverage on main. The PR's final commit directly addresses that remaining gap with a minimal guard and unit tests.

## Best Possible Solution

Keep the PR open or recreate its final minimal change on main: update `src/auto-reply/heartbeat.ts` so future `lastRunMs` values are treated like never-run tasks, and add the focused `isTaskDue` unit coverage in `src/auto-reply/heartbeat.test.ts`. The related ClawSweeper heartbeat reports cover different interruption/routing/cron-state bugs, so they do not supersede this clock-skew fix.
