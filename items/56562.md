## Summary

Keep open. Current `main` has a partial hardcoded mitigation for malformed built-in file-tool arguments, but it does not implement the requested config surface (`tools.validate_before_call`, `tools.retry_on_missing_params`) and it still surfaces validation failures as ordinary tool-error results rather than doing an automatic hidden retry.

## Evidence

- **No requested config keys in tool config types:** `AgentToolsConfig` exposes `loopDetection`, `exec`, `fs`, and related policy fields, but there is no `validate_before_call` or `retry_on_missing_params` entry anywhere in the tool config types.
  - file: [src/config/types.tools.ts:304](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/config/types.tools.ts#L304)
  - command: `nl -ba src/config/types.tools.ts | sed -n '300,360p'`

- **Docs only document loop detection, not validation/retry knobs:** The public tools config doc documents `tools.loopDetection` and other existing `tools.*` sections, but there is no documented `tools.validate_before_call` or `tools.retry_on_missing_params` section.
  - file: [docs/gateway/config-tools.md:114](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/gateway/config-tools.md#L114)
