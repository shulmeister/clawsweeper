## Summary

Keep #40311 open. Current main has not implemented Brave Goggles support: the bundled Brave web-search plugin schema, runtime URL builders, cache key, tests, and docs do not mention `goggles`. The PR discussion is useful and the author addressed bot-review concerns around runtime coverage and cache-key collisions, but the branch now targets an older core-web-search shape; the feature should be ported to the current provider-owned Brave plugin under `extensions/brave` rather than closed as obsolete.

## Best Possible Solution

Keep the item open and ask for a rebase/port: add `goggles` to the current bundled Brave provider schema in `extensions/brave/src/brave-web-search-provider.ts`, thread it through `extensions/brave/src/brave-web-search-provider.runtime.ts` for both web and llm-context modes, include it in the Brave cache key without lossy normalization collisions, and update Brave/web docs plus focused extension tests.
