## Summary

Keep open. Current main has soft prompt guardrails plus configurable hard controls, but the issue discussion is asking for a native hard/default dangerous-operation gate. Main still defaults trusted host exec to no approval prompts, documents that prompt injection is not solved, and explicitly says there is no extra heuristic dangerous-command gate in YOLO mode.

## Best Possible Solution

Keep #6731 open until maintainers either implement a native hard/default dangerous-operation gate in core exec/tool policy or make an explicit product decision that documented opt-in exec approvals, tool policy, and sandboxing are the final safety model. A concrete path would add tests and docs for defaults, dangerous command classification, approval behavior, sandbox interaction, and the Windows PowerShell production scenario raised in the comments.
