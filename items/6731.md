## Summary

Keep #6731 open. Current main has exec approvals, sandboxing, and advisory safety prompt text, but it still documents no-approval host exec as the default and explicitly says prompt injection is not solved. The issue discussion is asking for a native hard/default dangerous-operation gate, which remains core safety/product scope rather than conservative cleanup-closeable.

## Best Possible Solution

Keep #6731 open until maintainers either implement a native hard/default dangerous-operation gate in core exec/tool policy or explicitly decide that documented opt-in exec approvals, tool policy, and sandboxing are the intended final model. The likely path is core exec/tool policy plus docs and tests covering defaults, dangerous command classification, approval behavior, and sandbox interaction.
