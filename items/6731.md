## Summary

Keep #6731 open. Current main has configurable exec approvals, sandboxing, allowlists, strict inline-eval hardening, and advisory safety prompt text, but it still defaults host exec to no approval in trusted single-operator mode and explicitly documents no separate dangerous-command heuristic gate in YOLO mode. The issue asks for a native hard/default dangerous-operation gate, which remains core safety/product scope rather than conservative cleanup-closeable.

## Best Possible Solution

Keep #6731 open until maintainers either implement a native hard/default dangerous-operation gate in core exec/tool policy, or explicitly decide that the current opt-in model of exec approvals, sandboxing, tool policy, and docs is the final intended product behavior. A concrete path would cover default safe mode, dangerous-command classification, approval behavior, sandbox interaction, and regression tests/docs.
