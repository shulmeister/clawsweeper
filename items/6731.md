## Summary

Keep #6731 open. Current main still documents and implements no-approval host exec as the default trusted-operator behavior unless operators opt into stricter approvals/sandboxing, and the issue has a recent maintainer-side review comment explicitly keeping it open as core safety/product scope.

## Best Possible Solution

Keep #6731 open until maintainers either implement a native hard/default dangerous-operation gate in core exec/tool policy or explicitly decide that the current opt-in exec approvals, sandboxing, and tool-policy model is the final product stance. A concrete implementation path should cover default safe mode, Windows PowerShell destructive-command classification such as Remove-Item, credential/config destructive operations, approval UX, sandbox interaction, and regression tests/docs.
