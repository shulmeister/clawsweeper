## Summary

Keep open. Current main has useful sandbox, tool-policy, and exec-approval controls, but the issue discussion asks for a native hard safety/dangerous-operation gate. Current documented defaults still allow gateway/node host exec without approval unless the operator tightens policy, and docs explicitly frame prompt-injection hardening as operator-configurable tool policy, exec approvals, sandboxing, and channel allowlists rather than a solved default gate. This is core safety scope, not a ClawHub-only plugin item.

## Risks / Open Questions

- Some users can already get much of the desired behavior by enabling sandbox mode and setting exec approvals to `allowlist`/`on-miss` or `always`, so any future fix should avoid breaking intentional high-power operator workflows.
