## Summary

Keep open. Current `main` still has the reported routing mismatch: implicit `--to` resolution flows through a helper that hardcodes `DEFAULT_AGENT_ID = "main"`, while explicit `--agent` resolution uses the requested/configured agent main session key. The same logic is present in the latest shipped release `v2026.4.23`, so this is neither implemented-on-main nor obsolete.

## Evidence

- **Implicit `--to` still hardcodes `main`:** `resolveSessionKey()` builds direct and group session keys with `DEFAULT_AGENT_ID`, so a plain `--to` path resolves to `agent:main:*` instead of the configured default agent.
  - file: [src/config/sessions/session-key.ts:29](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/config/sessions/session-key.ts#L29)
