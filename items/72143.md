## Summary

Keep #72143 open. Current main still contains the reported user-facing fallback prefix, tests explicitly assert that prefix is sent after a session resume failure, and there is no retry/backoff around the single `callGatewayTool("agent")` follow-up attempt. The linked PR #72148 is relevant and should be acknowledged, but it is still open and, by its own body, only addresses prefix removal while deferring retry/backoff work.

## Best Possible Solution

Keep #72143 open until a merged fix removes the internal fallback prefix from user-visible delivery, logs resume failures internally with useful metadata, and either adds bounded retry/backoff before direct fallback or creates a clear canonical follow-up for that remaining resilience work. PR #72148 is the current relevant implementation attempt and should be reviewed or expanded rather than using this cleanup pass to close the issue.
