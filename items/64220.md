## Summary

Keep open. Current main still has the specific Control UI stale-state failure described here: a compaction start event sets an active status with no active-state watchdog, the renderer shows active/retrying compaction without an age check, and several terminal/reconnect/reset paths clear chat run state without consistently clearing compactionStatus/fallbackStatus. Related #39908 covered a broader stuck-compaction symptom and backend overflow/retry cleanup, but it does not supersede this narrower missed-end/stale-Control-UI state issue.

## Best Possible Solution

Keep this issue open and fix it in the Control UI state layer. Add a run/session-scoped stale timeout or watchdog for active compaction, centralize cleanup of compactionStatus/fallbackStatus plus their timers, invoke that cleanup on terminal chat events, reconnect/disconnect, session/agent switch, clear/reset, and send-failure paths, and add regression tests for a missed compaction end after a fully rendered reply plus reconnect/session-switch cleanup.
