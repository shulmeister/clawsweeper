## Summary

Current main renders Control UI HTTP links as https:// and Gateway WS links as wss:// when gateway.tls.enabled is true, and the onboard/status paths now pass that TLS state into the shared link resolver. The merged #71499 fix covers this same behavior on main; it is listed in the unreleased 2026.4.25 changelog, so no shipped release is recorded yet.
