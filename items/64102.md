## Summary

Keep open. Current main only suppresses the hook-pack fallback for security-scan plugin install errors when `--dangerously-force-unsafe-install` is set; the PR’s central no-flag blocked npm plugin install case is still not implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review or adaptation. The best code path is to make security-scan install failures terminal before hook-pack fallback for both local and npm plugin install paths, regardless of `--dangerously-force-unsafe-install`, while preserving fallback for non-security plugin install errors and keeping the focused CLI regression test for the no-flag blocked npm install case.
