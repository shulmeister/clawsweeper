## Summary

Current main already addresses this by staging allowed inbound media from the managed media store into the active sandbox workspace and rewriting agent-facing `MediaPath`/`MediaUrl` values to `media/inbound/...` before the reply run starts.

## Evidence

- **Agent run now stages inbound media before execution:** `get-reply` invokes `stageSandboxMedia(...)` whenever a session has inbound media, before handing control to the agent runtime.
