## Summary

Current main does not implement the requested inter-tool text coalescing. The related hold/resume PR was closed unmerged, and the active code still flushes pending block replies before tool execution instead of holding the coalescer across tool calls.

## Best Possible Solution

Keep #15022 open as the canonical product/code issue. The likely implementation is a core reply-pipeline change that can hold or otherwise defer block-reply flushing across tool execution, then resume/flush once the tool-interleaved assistant turn reaches a safe boundary, with regression coverage for sequential and parallel tool calls.
