## Summary

Keep #15022 open. Current main still has the reported behavior: the block reply pipeline flushes buffered text before tool execution and across assistant-message boundaries, and there is no shipped coalesceTextBlocks/toolTextCoalesce/final-only config surface for merging interleaved text/tool/text output into one outbound message.

## Best Possible Solution

Keep #15022 open as the focused core reply-pipeline issue. The likely fix belongs in the block reply pipeline and embedded subscribe path: hold or defer block-reply flushing across tool execution, support sequential and parallel tool calls, and add regression tests proving text/tool/text turns can deliver as one outbound message without breaking media, reasoning, duplicate suppression, and explicit boundary behavior.
