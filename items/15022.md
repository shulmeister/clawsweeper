## Summary

Current `main` does not implement inter-tool text coalescing. The live code explicitly flushes pending block replies before every `tool_execution_start`, the coalescer only supports idle-time text merging with no hold/resume around tool execution, and targeted tests assert preservation of message boundaries before tools. The issue remains actionable on `main`.

## Evidence

- **Tool start forces a flush before each tool call:** `handleToolExecutionStart()` invokes `flushBlockReplyBuffer()` and then `onBlockReplyFlush()` specifically to preserve message boundaries before tool execution, which is the opposite of coalescing text across tool calls.
  - file: [src/agents/pi-embedded-subscribe.handlers.tools.ts:696](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/agents/pi-embedded-subscribe.handlers.tools.ts#L696)
  - command: `nl -ba src/agents/pi-embedded-subscribe.handlers.tools.ts | sed -n '540,730p'`

- **Coalescer has no tool-execution hold/resume support:** `createBlockReplyCoalescer()` only exposes `enqueue`, `flush`, `hasBuffered`, and `stop`; buffering is governed by `idleMs`, `minChars`, and `maxChars`.
