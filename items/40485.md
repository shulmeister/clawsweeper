## Summary

Keep PR #40485 open. The linked Telegram bug (#40442) is closed, but current main does not contain the PR’s narrow reply-threading override: native Telegram `/new` and `/reset` still flow through the account-level `replyToMode`, and generated replies still default their `replyToId` to the triggering command message. The PR remains a plausible focused bug fix, though it needs rebasing from the old `src/telegram` paths to the current `extensions/telegram` plugin layout and should add `/reset` coverage noted in review.

## Best Possible Solution

Keep the PR open and port its narrow behavior to `extensions/telegram/src/bot-native-commands.ts`: allow `buildCommandDeliveryBaseOptions` to override `replyToMode`, set it to `"off"` only for direct-chat native `new` and `reset`, preserve existing group/topic and other-command threading behavior, and add regression tests for both `/new` and `/reset` with global Telegram `replyToMode: "first"`.
