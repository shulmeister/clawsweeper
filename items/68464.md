## Summary

Keep #68464 open. Current main still does not contain the PR's central hardening: completion dispatch is still direct-first, subagent run records do not persist a delivery claim or user-safe delivery payload, and iMessage outbound sanitization/chunking lacks the proposed internal-context stripping and fence-normalization path. The PR also has unresolved review findings, so it is not cleanup-close material.

## Best Possible Solution

Keep the PR open or replace it with a current successor that ports the useful hardening into main: persisted per-run delivery claims, explicit user-safe completion payloads, queue-first/fail-closed active-parent completion dispatch, and iMessage internal-context/fence-safe outbound normalization. Before landing, address the existing review findings and coordinate the remaining missing-payload race with #69767.
