## Summary

PR #42479 should close as superseded by PR #47631, not as implemented. Current main still forces Telegram DM streaming lanes onto message transport and current tests still assert that old behavior, so the user-visible request remains unresolved. However, #47631 is open, targets the current `extensions/telegram` plugin layout, covers the same DM answer-lane native draft restoration, and adds the follow-up draft cleanup / ambiguous materialize guards that #42479 does not include. #42479 is narrower and targets the older `src/telegram` path, so the best maintainer path is to keep the newer canonical PR or land an equivalent current-layout patch.

## Best Possible Solution

Close #42479 as superseded and use #47631, or an equivalent current-layout maintainer patch, as the canonical path for restoring native draft transport on Telegram DM answer lanes while preserving reasoning-lane message transport and duplicate-final safeguards.
