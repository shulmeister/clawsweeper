## Summary

Current main already provides dashboard access to archived session transcripts through the Control UI Usage view, including discovery of reset/deleted transcript files, archived-only detail fallback, searchable conversation rendering, and regression coverage.

## Evidence

- **Archived transcripts are included in usage discovery:** `discoverAllSessions()` scans usage-counted transcript files, including reset/deleted archives, extracts first-user-message labels, and deduplicates by session ID while keeping the newest relevant transcript.
  - file: [src/infra/session-cost-usage.ts:457](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/infra/session-cost-usage.ts#L457)
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Archived-only sessions resolve for detail views:** `resolveExistingUsageSessionFile()` falls back from a missing primary transcript to the latest `.jsonl.reset.*` or `.jsonl.deleted.*` archive, which is then used by session summary/log/timeseries loaders.
  - file: [src/infra/session-cost-usage.ts:299](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/infra/session-cost-usage.ts#L299)
