## Summary

Keep PR #41983 open. Current main still shares the control-plane write budget by device/IP whenever either identity is present, so the PR's requested per-connection isolation is not implemented or superseded. The PR is a focused gateway behavior change with a regression test, but it needs a rebase/rework to preserve current main's bucket pruning and hard cap and to address the stale test description called out by Greptile.

## Best Possible Solution

Keep the PR open or ask the contributor to rebase it onto current main, update the stale test name, add/adjust the concurrent-session isolation test, and preserve the existing stale-bucket pruning plus 10,000-entry cap. If maintainers decide shared device/IP budgeting is intentional, close with that explicit product rationale rather than as automated cleanup.
