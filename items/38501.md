## Summary

Current main still lacks the requested Ctrl+C active-run clarification. The TUI Ctrl+C path only clears/warns/exits and does not inspect `activeChatRunId` or mention Esc/`/abort`, while the active-run abort behavior remains on Esc and `/abort`. Related fix PRs in the provided GitHub context are not merged, so the issue is still actionable.

## Best Possible Solution

Keep #38501 open until a focused TUI text-only change lands on main. The best fix is to update `src/tui/tui.ts` so Ctrl+C status text includes an active-run hint when `activeChatRunId` is set, while preserving current Ctrl+C clear/exit behavior and leaving Esc/`/abort` as the abort paths. A small regression test around the Ctrl+C status builder or handler would make the UX contract explicit.
