## Summary

Current main still has Ctrl+C wired only to clear input, warn, or exit the TUI. It does not mention that an active run continues, while Esc and `/abort` remain the implemented abort paths. The directly linked fix PR #38502 and related PR #38661 are still open/unmerged in the provided GitHub context, so the user-facing issue is not implemented on main.

## Best Possible Solution

Keep #38501 open until a focused TUI UX patch lands on main. The best implementation is to update `src/tui/tui.ts` so Ctrl+C clear/warn status text includes an active-run hint when a run is in progress, while preserving existing Ctrl+C clear/exit behavior and leaving Esc plus `/abort` as the actual abort controls. Add a small regression test around Ctrl+C status construction or handler behavior.
