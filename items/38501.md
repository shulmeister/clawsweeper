## Summary

Current main still lacks the requested TUI Ctrl+C active-run hint. Ctrl+C only clears/warns/exits, while Esc and `/abort` are still the abort paths, and the related fix PRs in the provided context are open or closed unmerged rather than shipped.

## Best Possible Solution

Keep #38501 open until a focused TUI UX patch lands on main. The likely fix belongs in `src/tui/tui.ts`: when `state.activeChatRunId` is set, preserve the existing Ctrl+C clear/warn/exit behavior but append an explicit active-run hint such as `run still active (Esc or /abort to stop)`. Add a small regression test covering Ctrl+C status text while a run is active, then use or supersede the existing focused PR #38502.
