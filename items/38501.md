## Summary

Keep #38501 open. Current main still implements Ctrl+C in the TUI as clear/warn/exit status flow only, with no active-run hint, while Esc and `/abort` remain the actual active-run abort paths. The issue discussion also points to focused fix PR #38502, but the provided context shows it is still open and unmerged.

## Best Possible Solution

Keep #38501 open until a focused TUI UX patch lands on main. The best fix is to update the Ctrl+C status path in src/tui/tui.ts so that when state.activeChatRunId is set, Ctrl+C preserves the existing clear/warn/exit behavior but explicitly says the run is still active and that Esc or /abort stops it. Add a small regression test for the status/handler behavior; PR #38502 is the narrowest linked candidate in the provided context.
