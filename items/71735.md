## Summary

Keep open. Current main still lacks the requested scoped plugin command API, trusted/reserved command ownership model, and command-result continuation semantics. The related RFC PR #71731 is still open and explicitly described as docs-only, not a runtime hook implementation.

## Evidence

- **Plugin command definition lacks requested fields:** OpenClawPluginCommandDefinition currently exposes name, nativeNames/nativeProgressMessages, description, acceptsArgs, requireAuth, and handler; there is no requiredScopes field or trusted ownership metadata.
  - file: [src/plugins/types.ts:1845](https://github.com/openclaw/openclaw/blob/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54/src/plugins/types.ts#L1845)

- **Command result has no continuation contract:** PluginCommandResult is just ReplyPayload, and ReplyPayload has no continueAgent-style field.
  - file: [src/plugins/types.ts:1833](https://github.com/openclaw/openclaw/blob/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54/src/plugins/types.ts#L1833)
