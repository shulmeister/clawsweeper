## Summary

Keep PR #71062 open. Current main does not implement the requested /v1/responses client-tool handoff fix: the Pi adapter still returns a normal pending tool result without `terminate: true`, the gateway still emits function-call args only on `output_item` bookends, and the upstream Responses stream parser still ignores `response.function_call_arguments.done`. The PR is a focused, recent bug fix with tests and follow-up commits responding to review feedback, so it is not stale, duplicate, or superseded.

## Best Possible Solution

Keep the PR open and finish normal maintainer review. The product path is to land or adapt this focused fix: add `terminate: true` to the delegated client-tool stub, emit `response.function_call_arguments.delta` and `.done` between function-call `output_item` bookends while preserving populated args in the completed response snapshot, handle `.done`-only upstream Responses arguments, and keep the adapter/gateway/parser regression tests. Also update the changelog entry to include the required `Thanks @PopFlamingo` attribution before landing.
