## Summary

Keep PR #71062 open. Current main still has the behavior the PR is trying to change: client-tool execute results do not set Pi's termination hint, `/v1/responses` streaming emits function-call arguments only on `output_item` bookends, the OpenResponses schema/docs omit `response.function_call_arguments.{delta,done}`, and the upstream Responses stream parser handles `.delta` but not `.done`-only arguments. This is a focused, recent bug-fix PR with concrete regression-test targets, not an obsolete or low-signal cleanup candidate.

## Best Possible Solution

Keep the PR open for maintainer review. The best implementation path is the PR's current direction: add `terminate: true` to the client-tool handoff result, emit `response.function_call_arguments.delta` and `.done` between the function_call `output_item` bookends, update the OpenResponses schema/parser/tests, and ensure the final `response.completed` snapshot carries the completed function_call item with real arguments.
