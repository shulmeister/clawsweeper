## Summary

Keep #12911 open. Current main has a browser Talk loop in `/chat`, but it is still OpenAI WebRTC/browser-session specific, while the requested xAI/Grok realtime voice path is explicitly not registered or exposed. Recent maintainer comments in the provided issue context also say the OpenAI-backed Talk work is only partial and keep the issue open for the xAI/Grok provider work.

## Best Possible Solution

Keep #12911 open as the tracker for Grok/xAI Live Voice Mode in `/chat`. The concrete path is to make browser Talk provider-aware beyond the current OpenAI WebRTC client, add xAI realtime voice/browser-session support if upstream supports it, extend the shared realtime voice contract where needed, and update provider docs plus focused UI/provider tests. If maintainers decide the shipped OpenAI Talk path is sufficient, close only after that explicit product decision.
