## Summary

Keep #12911 open. Current main has a browser Talk path in /chat, but it is still not the requested Grok/xAI Live Voice Mode: the browser path posts directly to OpenAI Realtime WebRTC, xAI does not register a realtime voice/browser-session provider, and docs explicitly describe xAI realtime voice as not exposed yet.

## Best Possible Solution

Keep #12911 open as the tracker for Grok/xAI Live Mode in /chat. The best path is to make browser Talk provider-aware, add xAI realtime voice and browser-session support if the upstream API supports it, wire that through the bundled xAI plugin or an explicit plugin API extension, and update Control UI docs/tests. If maintainers decide OpenAI Talk satisfies the product direction, close only after that explicit product decision.
