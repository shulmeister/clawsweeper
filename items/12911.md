## Summary

Keep #12911 open. Current main has a browser Talk loop in `/chat`, but the shipped path is still OpenAI Realtime/WebRTC-backed and the xAI/Grok Live Voice Mode requested by the issue remains missing. Maintainer comments in the provided thread also explicitly keep the item open for xAI/Grok provider work.

## Best Possible Solution

Keep #12911 open as the tracker for Grok/xAI Live Mode in `/chat`. The likely path is to make browser Talk provider-aware beyond the current OpenAI-specific WebRTC client, add xAI realtime voice/browser-session support if upstream supports it, wire that through the bundled xAI plugin or an explicit plugin API extension, and update Control UI docs/tests. If maintainers decide OpenAI Talk satisfies the product direction, close only after that explicit product decision.
