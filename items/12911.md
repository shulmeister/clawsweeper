## Summary

Keep open. Current main partially implements browser Talk in /chat, but it is still OpenAI-WebRTC-specific and the bundled xAI plugin does not expose Grok/xAI realtime voice or browser-session support. The issue discussion already called out this exact partial state and explicitly kept the xAI/Grok request open.

## Best Possible Solution

Keep #12911 open as the tracker for Grok/xAI Live Mode in /chat. The best implementation path is to make Control UI Talk genuinely provider-aware, add or extend the realtime voice/browser-session contract for xAI if the upstream API supports it, register that support in the bundled xAI plugin, and update docs/tests. If maintainers decide OpenAI Talk satisfies the product direction, close only after that explicit product call.
