## Summary

Keep #71709 open. Current main still lets the session-memory slug generator run through the embedded agent with the default shared auth-profile failure policy. The helper-local policy exists and is already used by another internal helper, but the slug-generator call site does not pass it, while Anthropic extra-usage text is still intentionally classified as billing and can be persisted to shared auth-profile state.

## Best Possible Solution

Keep the issue open. Fix `src/hooks/llm-slug-generator.ts` by passing `authProfileFailurePolicy: "local"` to the slug-generator `runEmbeddedPiAgent` call, then add a focused regression test in `src/hooks/llm-slug-generator.test.ts` asserting the slug helper uses local auth-profile failure policy and falls back without mutating shared profile health. Preserve the broader billing classifier unless maintainers decide separately to change provider-wide semantics.
