## Summary

Current main intentionally keeps `GET /v1/models` agent-first. The runtime model catalog exists through the Gateway RPC `models.list` and CLI/UI surfaces, but the requested OpenAI-compatible HTTP discovery endpoint for actual provider model IDs is still missing.

## Best Possible Solution

Keep this issue open. The best path is to add a documented authenticated HTTP model-catalog discovery surface, likely by reusing the existing `models.list` runtime-allowed catalog and tests, without breaking the current agent-routing semantics of `/v1/models` unless maintainers explicitly choose that product change.
