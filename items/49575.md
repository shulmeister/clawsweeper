## Summary

Current main already implements per-client gateway broadcast sequencing for the reported false-gap case, and it includes regression coverage for both scope-filtered broadcasts and `dropIfSlow` gaps. The same gateway code and tests are present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

## Evidence

- **Per-client seq only advances for delivered or intentionally dropped eligible broadcasts:** `broadcastInternal` skips non-targeted and scope-filtered clients before computing or storing the next seq, and it advances seq on `dropIfSlow` before continuing. That directly addresses false gaps from scope filtering while preserving real gaps for slow-consumer drops.
