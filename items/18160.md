## Summary

Keep #18160 open. Current main still does not implement a direct exec/shell cron payload: cron types, Gateway schema, validation, runtime dispatch, CLI, UI, docs, and tests remain built around `systemEvent` and `agentTurn`. The linked implementation PR #51276 was closed unmerged in the provided GitHub context, and the latest maintainer comment explicitly kept #18160 as the substantive core tracker.

## Best Possible Solution

Keep #18160 open as the canonical core cron direct-exec tracker. The best implementation path is an additive `exec` or `shell` payload contract across cron types, Gateway schema/validators, normalization, runtime dispatch, CLI add/edit, Control UI, docs, and focused tests, with a controlled process runner and clearly documented security and delivery semantics. Broader direct-tool or conditional-pipeline ideas should remain separate follow-ups unless maintainers choose to fold them into this core cron payload design.
