## Summary

Keep #18160 open. Current main still has no direct `exec`/`shell` cron payload: the cron type contract, Gateway schema, validation, runtime, CLI, agent tool schema, Control UI, and docs all remain limited to `systemEvent` and `agentTurn`. The linked implementation PR #51276 is closed/unmerged, and the issue has concrete production-use comments plus an existing maintainer/Codex comment explicitly keeping it open.

## Best Possible Solution

Keep #18160 open as the active tracker unless maintainers explicitly consolidate it. The concrete path belongs in core cron: add a versioned `exec`/`shell` payload contract across `src/cron/types.ts`, Gateway schema/normalization, CLI, agent tool schema, Control UI, docs, and tests; implement a controlled process-runner branch with timeout/abort, stdout/stderr capture, run-log/state updates, retry/failure behavior, channel/webhook delivery, and security policy aligned with existing exec approvals/sandbox expectations.

## Risks / Open Questions

- A direct cron exec mode would add scheduled host command execution outside the agent loop, so the implementation needs explicit security policy, timeout/abort handling, logging, and delivery semantics rather than a narrow schema-only change.
- Closing this now would lose the most substantive tracker for a still-missing cron capability; related open items are narrower or broader variants rather than a clearly canonical replacement.
