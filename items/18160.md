## Summary

Keep #18160 open. Current main still does not implement a direct `exec`/`shell` cron payload: cron payload types, Gateway schema, CLI creation/editing, agent tool schema, UI form state, docs, and runtime execution are all still limited to `systemEvent` and `agentTurn`. The related implementation PR #51276 is closed and unmerged, while #37651 and #50558 are duplicate open requests rather than a clear canonical superseding tracker. The issue discussion also includes concrete production use cases and a maintainer/Codex comment already keeping this item open.

## Best Possible Solution

Keep #18160 open as the active tracker unless maintainers explicitly consolidate it into a canonical duplicate. The concrete implementation path belongs in core cron: add a versioned `exec`/`shell` payload contract across `src/cron/types.ts`, Gateway schema/normalization, CLI, agent tool schema, Control UI, docs, and tests; implement a runtime branch that executes through a controlled process runner with timeout/abort handling, run-log/state updates, delivery/failure behavior, and security policy aligned with existing exec approvals.
