## Summary

Keep open. Current main still does not thread the resolved `--agent` auth directory into the `models status --probe` path. The non-probe status path resolves and reads the target agent store, but `runAuthProbes` accepts no `agentDir`, `buildProbeTargets` still calls `ensureAuthProfileStore()` with no argument, and probe execution independently resolves the default agent id/dir. The linked PR #67263 targeted this exact bug, but it was closed unmerged in the provided GitHub context and the proposed plumbing is absent from current `main`.

## Best Possible Solution

Keep #67235 open and either revive the substance of #67263 or implement the same fix on current main: pass the `modelsStatusCommand` resolved `agentDir` and target `agentId` through `runAuthProbes`, `buildProbeTargets`, and the probe execution path, then add regression tests that use distinct main and target-agent auth stores/orders for `models status --probe --agent <id>`.
