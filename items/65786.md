## Summary

Current main does not fully resolve the reported Feishu config-invalid regression. The Feishu schema now includes several reported keys such as dmPolicy/groupPolicy/allowFrom, but it is still strict, omits contextVisibility, and only accepts boolean streaming, while the reporter's config uses contextVisibility and streaming: "progress". Doctor also does not currently expose a Feishu-specific compatibility normalizer to migrate that legacy shape.

## Best Possible Solution

Keep this issue open. The best fix is in the Feishu plugin/config contract: add contextVisibility to FeishuConfigSchema and account config if intended, regenerate/update bundled channel config metadata, and add a Feishu doctor compatibility migration or schema support for legacy scalar streaming values such as "progress". Add a regression test using the reported config and a doctor --fix path test so future upgrades do not re-break it.
