## Summary

Keep open. Current `main` still treats startup secret-resolution failures as fatal, with no built-in startup backoff/circuit-breaker for exec secret providers, and the issue timeline points to an open follow-up PR rather than a landed fix.

## Evidence

- **Exec secret providers still fail immediately on timeout/exit/error:** `resolveExecRefs` runs the configured exec provider once per provider group, then throws immediately on timeout, no-output timeout, non-zero exit, or parse failure. There is batching of ids within one provider alias, but no retry, cooldown, circuit breaker, or rate-limit handling in this path.
  - file: [src/secrets/resolve.ts:648](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/secrets/resolve.ts#L648)
  - command: `nl -ba src/secrets/resolve.ts | sed -n '648,912p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Startup still aborts when secret activation fails:** `createRuntimeSecretsActivator` converts startup secret-resolution errors into `Startup failed: required secrets are unavailable...` and rethrows them during startup. That preserves fail-fast behavior instead of keeping the process alive on secret-provider failure.
  - file: [src/gateway/server-startup-config.ts:160](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/gateway/server-startup-config.ts#L160)
  - command: `nl -ba src/gateway/server-startup-config.ts | sed -n '160,205p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Startup path resolves secrets during preflight and final activation:** `prepareGatewayStartupConfig` may resolve active gateway-auth SecretRefs once for startup preflight and then again for final activation. That means a failed startup can repeatedly re-hit external secret providers on each fresh supervisor restart.
  - file: [src/gateway/server-startup-config.ts:237](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/gateway/server-startup-config.ts#L237)
  - command: `nl -ba src/gateway/server-startup-config.ts | sed -n '224,282p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Docs and tests still codify fail-fast startup behavior:** The public secrets docs still state `Startup failure aborts gateway startup`, and the startup secrets test still expects unresolved required secrets to throw `Startup failed: required secrets are unavailable...` on startup.
  - file: [docs/gateway/secrets.md:388](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/gateway/secrets.md#L388)
