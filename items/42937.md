## Summary

Keep open. Current main has a related per-binding `bindings[].session.dmScope` capability, but it still does not implement this PRâ€™s requested `agents.list[].dmScope` contract. The per-agent type/schema field is absent, strict agent config parsing would reject it, and route resolution only considers global `session.dmScope` plus matched binding session overrides. The PR discussion remains useful: Greptile found the change well-scoped, and the PR head addressed the duplicated-schema review note by extracting a shared `DmScopeSchema`.

## Best Possible Solution

Keep the PR open or replace it with a maintainer-authored follow-up that rebases onto current main and integrates `agents.list[].dmScope` with the existing binding-level override. The concrete implementation should add the AgentConfig type, strict zod schema, labels/help/docs/generated schema updates, and route tests covering precedence: binding override > per-agent override > global session default.
