## Close Comment

Codex review against current `main` (`8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf`) points this to ClawHub/plugin space rather than OpenClaw core. The roadmap explicitly says new skills should go to ClawHub first (`VISION.md:68`) and lists both `agent-hierarchy frameworks` and `heavy orchestration layers that duplicate existing agent and tool infrastructure` under items not being merged into core (`VISION.md:101`). Current `main` already has the extension surfaces for this pattern: reusable workspace and agent-scoped skills (`docs/tools/skills.md:9`), ClawHub as the registry for skills and plugins (`docs/tools/clawhub.md:10`), and the bundled `skill-workshop` plugin that captures repeatable workflows into reviewed `SKILL.md` files (`docs/plugins/skill-workshop.md:16`, `extensions/skill-workshop/openclaw.plugin.json:2`). Because this is a useful workflow-packaging idea but not a fit for core per the current project scope, closing as `clawhub` is the conservative maintainer outcome.
