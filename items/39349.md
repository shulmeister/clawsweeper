## Summary

Keep #39349 open. Current main still silently disables an explicit `pty:true` request whenever exec is routed through the sandbox, and the PR remains a focused fix for the still-open linked bug #38601.

## Best Possible Solution

Keep #39349 open or land an equivalent narrow fix on main. The best landing path is to add the warning on the existing `warnings` path when `params.pty === true && sandbox`, plus a targeted regression test proving the warning is included when sandbox routing suppresses PTY and absent when PTY is not suppressed.
