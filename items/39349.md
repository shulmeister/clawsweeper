## Summary

Current main still disables an explicit `pty:true` request when sandbox routing is active and does not warn the user, so PR #39349 remains a valid focused fix for the still-open linked bug #38601.

## Best Possible Solution

Keep #39349 open or land an equivalent narrow fix on main, ideally with a targeted regression test around `createExecTool` when `pty:true` is requested and sandbox routing suppresses PTY. A docs clarification at https://docs.openclaw.ai/tools/exec would also be useful, but it should not replace the runtime warning because the silent behavior remains user-visible.
