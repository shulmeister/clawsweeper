## Summary

Keep open. Current main still computes `usePty` as `params.pty === true && !sandbox` and has no warning branch for `params.pty === true && sandbox`, so the PR still addresses a real unresolved exec UX gap rather than obsolete work.

## Best Possible Solution

Keep the PR open or land an equivalent fix on main with a focused regression test for `pty:true` plus sandbox. If maintainers prefer documentation instead of a runtime warning, update `docs/tools/exec.md`, but current behavior remains surprising and unresolved.
