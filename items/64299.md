## Summary

Keep this issue open. The reporter’s follow-up verified the immediate 400 was resolved by granting and publishing the Feishu `docx:document.block:convert` scope, but current main still leaves a repo-side documentation/setup gap: the public Feishu docs only say to grant required scopes, and the manual setup prompt lists chat/contact scopes but not the doc conversion scope needed by `feishu_doc` write/append. Also, the related payload-normalization PR #64331 was closed unmerged, and current main still has a small-insert `toDescendantBlock` path where normalized `children` can be overwritten by the raw block object, so there is not enough evidence to close as implemented.

## Best Possible Solution

Keep the issue open and turn it into a focused Feishu docs/setup cleanup: add `docx:document.block:convert` to the public Feishu channel setup guidance and the manual credential help, and consider a clearer runtime error or diagnostic through `feishu_app_scopes` when write/append fail because the convert scope is missing. Separately, either revive a narrow regression/fix for the descendant `children` normalization path from #64331 or explicitly close that payload theory after verification.
