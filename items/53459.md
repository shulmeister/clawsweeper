## Summary

Keep open. Current `main` still lacks Docker Compose passthrough for `EXA_API_KEY` and `FIRECRAWL_API_KEY`, while the runtime and docs expect those keys to be present in the gateway environment. This PR still addresses a real gap; it is not implemented on `main` or in `v2026.4.22`.
