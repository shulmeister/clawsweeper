## Summary

Keep #39102 open. Current main still only supports the global `tools.agentToAgent.allow` gate for cross-agent session tools; the requested `agents.list[].tools.agentToAgent.allow` requester-specific outbound override is not implemented in policy, call sites, config schema/types/help/docs, prune behavior, or tests.

## Best Possible Solution

Keep the PR open or replace it with an updated implementation on current main. The right path is to add a strict `agents.list[].tools.agentToAgent.allow` schema/type/help/docs surface, update A2A policy so global participation is checked first and requester outbound overrides second, use consistent structured denial formatting across `sessions_list`, `sessions_history`, `sessions_send`, and `session_status`, prune deleted agent IDs from per-agent allowlists immutably, and add focused tests for fallback, deny-all, outbound-only behavior, global participation, prune immutability, and existing `session_status` visibility/model/sessionId behavior.
