## Summary

Keep #39102 open. Current main still implements only the global `tools.agentToAgent.allow` gate for cross-agent session tools; the proposed `agents.list[].tools.agentToAgent.allow` outbound override is not present in source, schema, docs, prune behavior, or tests.

## Best Possible Solution

Keep the PR open or replace it with an updated implementation on current main. The product path should add a strict `agents.list[].tools.agentToAgent.allow` schema/type/docs surface, update the plugin-sdk A2A policy to evaluate global participation first and requester outbound overrides second, wire all four session tools through structured denial messages, scrub deleted agent ids from per-agent lists immutably, and add focused tests for fallback, deny-all, outbound-only semantics, global participation, and prune behavior.
