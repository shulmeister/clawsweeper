## Summary

Current main still implements A2A session access as a global `tools.agentToAgent` gate only. The PR proposes a real missing config surface, policy evaluator, error path, tests, and pruning behavior for `agents.list[].tools.agentToAgent.allow`, so it is not implemented on main and is not a conservative auto-close candidate.

## Best Possible Solution

Keep the PR open. The best path is to finish review on the branch, ensure the per-agent config type/schema/help/docs/generated schema stay aligned, wire the structured A2A decision path through all session-tool call sites, preserve existing sandbox/session-status semantics, and land focused regression tests for requester-specific allow, deny-all `[]`, global participation gating, and deletion pruning.
