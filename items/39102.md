## Summary

Keep #39102 open. Current main still implements only the global `tools.agentToAgent.allow` participation gate for cross-agent session tools; the requested `agents.list[].tools.agentToAgent.allow` requester-specific outbound override is not present in policy, schema/types, docs, prune behavior, or call-site denial handling.

## Best Possible Solution

Keep the PR open or replace it with an updated implementation on current main. The implementation should add a strict `agents.list[].tools.agentToAgent.allow` schema/type/help/docs surface, evaluate global participation before requester outbound overrides, share structured denial formatting across `sessions_list`, `sessions_history`, `sessions_send`, and `session_status`, prune deleted agent IDs from per-agent allowlists immutably, and add focused tests for fallback, deny-all `[]`, outbound-only semantics, global participation, prune immutability, sandbox visibility, model fallback, and sessionId resolution.
