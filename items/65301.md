## Summary

Keep open. Current main still uses the broad poll-creation detector on action="send", so metadata-only poll fields such as non-zero duration or true visibility flags can still trigger the poll-only rejection. This PR remains a useful implementation candidate and appears to be the canonical clean branch after the older same-author #65295 was closed as superseded.

## Best Possible Solution

Keep this PR open for maintainer review. If maintainers agree that metadata-only poll fields should not block normal sends, land this stricter real-poll-intent guard or an equivalent action-scoped/sanitizing fix, preserving real poll rejection on action="send" and the poll action validation path. Broader schema overexposure work can remain tracked by the related open issues if this PR intentionally fixes only the runtime false-positive guard.
