## Summary

Keep PR #64608 open. Current main still does not forward `appliedTags` through Discord `channel-edit`: the shared parser, edit payload type, and REST PATCH body only cover existing channel/thread fields and `availableTags`, while Discord’s Modify Channel API supports `applied_tags` for forum/media threads. The PR is a focused implementation of a still-missing behavior, and its later commits address the review concern about preserving empty arrays and accepting `applied_tags` snake_case input. ([github.com](https://github.com/openclaw/openclaw/pull/64608))

## Best Possible Solution

Keep the PR open and review/land a rebased version that adds `appliedTags?: string[]` to `DiscordChannelEdit`, parses camelCase and snake_case tag arrays while preserving explicit `[]`, forwards `body.applied_tags` in `editChannelDiscord`, and covers runtime, guild-admin, and send-layer tests for non-empty and empty arrays.
