## Summary

Keep this PR open. Current main still does not implement the requested Discord channel-edit `appliedTags` pipeline: `DiscordChannelEdit` has no `appliedTags` field, the shared channel-edit parser does not read it, and `editChannelDiscord` never writes `applied_tags` to the PATCH body. Existing `appliedTags` support is limited to thread creation, so the PR is still a relevant implementation candidate rather than obsolete.

## Best Possible Solution

Keep the PR open and review it as a live implementation candidate. The best current fix is to add `appliedTags?: string[]` to `DiscordChannelEdit`, read camelCase and snake_case tag arrays in the shared channel-edit parser while preserving explicit empty arrays for clearing tags, forward the field from both action paths through that parser, map it to `applied_tags` in `editChannelDiscord`, and keep focused tests for non-empty, empty, and snake_case inputs.
