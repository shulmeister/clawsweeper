## Summary

Keep #71737 open. Current main has only adjacent raw/runtime surfaces: raw `api.runtime.events.onAgentEvent`, internal host run context, generic prompt hooks, and low-level `gateway_start` cron access. The requested typed plugin lifecycle contracts, namespaced run-context storage, session-owned scheduler cleanup, and heartbeat-specific contribution hook are not implemented on main. Open PR #72082 is the right implementation candidate, but it is still open/unmerged, so closing #71737 now would be premature.

## Best Possible Solution

Keep #71737 open as the focused RFC/tracking issue until maintainers either accept and land the generic host-hook implementation in #72082 or choose a different SDK design. The likely code path belongs in plugin SDK/runtime contracts, plugin registry/cleanup ownership, gateway scheduler cleanup, and heartbeat prompt assembly, with fixture coverage for sanitized event subscription, plugin-owned run context cleanup, session scheduler cleanup, plugin disablement cleanup, and heartbeat contribution ordering.
