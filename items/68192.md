## Summary

Current main already implements the requested RFC 2544 opt-in for `web_fetch`, and it now threads the same `tools.web.fetch.ssrfPolicy` into remote `pdf` loads as well. The `web_fetch` portion was already present in the released `v2026.4.23` tree; the remaining `pdf` wiring is present on current `main` and covered by tests.

## Evidence

- **Released web_fetch path already honors the RFC 2544 opt-in:** The `v2026.4.23` tree already reads `params.ssrfPolicy?.allowRfc2544BenchmarkRange` and passes `{ allowRfc2544BenchmarkRange }` into `fetchWithWebToolsNetworkGuard(...)`, so the core `web_fetch` request in this issue was already implemented in the latest release cited by the report.

  - file: [src/config/zod-schema.agent-runtime.ts:339](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/config/zod-schema.agent-runtime.ts#L339)
