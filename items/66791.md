## Summary

Keep #66791 open. Current main still only exposes Discord scheduled-event list/create support, while event-edit/event-delete and eventId handling are absent from the shared action registry/schema, Discord discovery and handler paths, runtime cases, REST helpers, CLI, and docs. The reporter's follow-up comment accurately maps the existing implementation and describes a concrete, contained Discord plugin enhancement rather than a stale or duplicate request.

## Best Possible Solution

Keep the issue open and implement it as a bundled Discord plugin enhancement: add `event-edit` and `event-delete` to the shared action list/spec and Discord discovery, add `eventId` to the message-tool schema, map the actions in `extensions/discord/src/actions/handle-action.guild-admin.ts`, add `eventEdit`/`eventDelete` runtime cases under the existing `events` gate, implement Discord scheduled-event fetch/PATCH/DELETE helpers, and cover the new behavior in Discord action tests plus CLI/docs/tool-display updates.
