## Summary

Keep open. Current main has limited bundle-backed LSP support, but it does not implement the requested real-time diagnostics/dashboard/editor feedback. The same author also has the paired open LSP auto-detection issue #65251, so this should not be auto-closed independently without maintainer judgment on the combined LSP product boundary.

## Best Possible Solution

Keep this issue open and have maintainers decide the LSP product boundary together with #65251. A good implementation path is to first define a generic LSP diagnostics contract: capture publishDiagnostics/server lifecycle/errors/latency in a runtime or plugin-owned service, expose it through a documented gateway/plugin API, then decide whether the UI is core Control UI, a plugin-served dashboard, or an external editor integration published through ClawHub.
