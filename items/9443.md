## Summary

Keep open. Current main still does not provide official prebuilt Android APK release assets. The Android docs explicitly say the app is not publicly released, the Android README still marks release hardening incomplete, CI only builds a scoped Play debug APK for validation, the release helper builds signed AAB bundles rather than APK release assets, and the latest public release has no APK asset. Community workarounds linked in the discussion, including bighamx/openclaw-android-node-apk, yuga-hashimoto/openclaw-assistant, and AndroidForClaw, are useful signals but do not replace an official OpenClaw APK release path.

## Best Possible Solution

Keep #9443 open as the canonical official Android APK release request. A complete solution likely belongs in release infrastructure: define signed APK/flavor policy, build on tag or release, upload APK assets to GitHub releases, update Android install docs, and add release checks that match the Android app readiness path. Newer duplicate requests such as #35515 should point back to #9443 rather than replacing it.
