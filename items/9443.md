## Summary

Keep #9443 open. Current main still does not provide official prebuilt Android APK release downloads: public docs say the Android app has not been released, the Android release helper builds signed AAB bundles rather than APK assets, CI only validates a debug Play APK build, and v2026.4.24 release assets are macOS-only. Related #35515 was closed as a duplicate of this issue, #53972 was closed unmerged as duplicate debug-CI work, and community APK/client links in the discussion remain unofficial workarounds.

## Best Possible Solution

Keep #9443 open as the canonical official Android APK release request. A complete fix should define the supported signed APK/flavor and distribution policy, add maintainer-approved release automation or an official distribution channel, publish official Android install assets where users are told to find them, and update https://docs.openclaw.ai/platforms/android with the install path. Community APK links should remain treated as unofficial workarounds unless maintainers explicitly adopt or document them.
