## Summary

Current main still does not satisfy #9443. The request is for official prebuilt Android APK downloads attached to GitHub releases, but the Android docs still say the app is not publicly released, the Android README remains local-build/release-hardening oriented, CI only validates a Play debug APK build, and the release helper builds signed AAB bundles rather than APK release assets. The discussion links useful community workarounds and clients, but they do not replace an official OpenClaw APK release path.

## Best Possible Solution

Keep #9443 open as the canonical official Android APK release request. A complete solution should define the signed APK/flavor policy, add release infrastructure that builds on tag or release and uploads APK assets to GitHub releases, update Android install docs, and add release checks that match the Android app readiness path. New duplicate requests should point back to #9443.
