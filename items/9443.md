## Summary

Keep #9443 open. Current main still does not publish official prebuilt Android APKs in GitHub releases: Android docs say the app is not publicly released, the Android README remains local-build/release-hardening oriented, CI only builds a Play debug APK for validation, and the release helper emits signed AAB bundles rather than APK release assets. Duplicate/report context points back to this issue as the canonical tracker, while community APK/client links are useful workarounds but not official OpenClaw artifacts.

## Best Possible Solution

Keep #9443 open as the canonical official Android APK release request. A complete fix should define the signed APK/flavor policy, add maintainer-approved release automation that uploads official APK assets to GitHub releases, update https://docs.openclaw.ai/platforms/android with the official install path, and continue redirecting duplicate APK-release requests here until that ships or maintainers explicitly decide against official APK distribution.
