## Summary

Keep #8176 open. Current main has implemented related system-prompt and context-prepend hook behavior, but it still does not expose or consume a `promptOverride`/user-prompt replacement field for `before_prompt_build` or legacy `before_agent_start`. The related PR #8178 was closed unmerged, while #8804/#22653/#22705 covered the adjacent `systemPrompt` problem rather than the current user prompt replacement requested here.

## Best Possible Solution

Keep #8176 open until OpenClaw either implements and documents an explicit prompt replacement field for `before_prompt_build` plus legacy `before_agent_start`, or maintainers explicitly reject that API. A complete fix should update public hook types, prompt-mutation filtering, hook merge precedence, embedded runner, CLI/Codex harness paths, docs, and focused regression tests for replacement behavior and `prependContext` ordering.

## Risks / Open Questions

- The existing Active Memory `config.promptOverride` is plugin-internal prompt customization for that plugin's memory sub-agent, not a core hook result field for replacing the user's current prompt.
- A complete implementation still needs maintainer API decisions on multi-plugin precedence, whether later hooks see the original or effective prompt, and how `allowPromptInjection=false` should filter a new replacement field.
