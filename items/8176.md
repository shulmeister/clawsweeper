## Summary

Current main still does not implement a hook-owned user prompt replacement. The public hook result types, prompt-mutation field allowlist, hook merger, prompt-build helper, embedded runner, CLI runner, docs, and existing tests all cover system/context mutation only. The directly related PR #8178 was closed unmerged, and the systemPrompt-related follow-ups do not satisfy the requested promptOverride capability.

## Best Possible Solution

Keep #8176 open until OpenClaw either implements and documents an explicit user-prompt replacement field for before_prompt_build and legacy before_agent_start, or a maintainer rejects that API. A complete fix should update public hook result types, the prompt-mutation field list, hook merge precedence, resolvePromptBuildHookResult, embedded and CLI runner application before prependContext/submission, allowPromptInjection filtering, docs, and focused regression tests for merge order and submitted prompt text.
