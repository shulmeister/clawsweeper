## Summary

Current main still does not implement the requested hook-owned user prompt replacement. The related systemPrompt side is now wired in the embedded runner, but public hook result types, merging, embedded/CLI/harness prompt assembly, docs, and tests still cover only system/context mutations and prepend behavior, not a `promptOverride` that can replace the current user prompt.

## Best Possible Solution

Keep #8176 open until OpenClaw either implements and documents an explicit user prompt replacement field for `before_prompt_build` and legacy `before_agent_start`, or a maintainer explicitly rejects that API. A complete implementation should update the public hook result types, prompt mutation allowlist, hook merge precedence, `resolvePromptBuildHookResult`, embedded runner, CLI runner, agent harness helper, `allowPromptInjection` filtering, public hooks docs, and regression tests proving submitted prompt text plus override/prepend ordering.
