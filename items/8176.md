## Summary

Current main still does not implement the requested user-prompt replacement hook result. The public hook result types, allowed prompt-mutation field list, hook merger, prompt-build helper, runner prompt assembly, docs, and tests all cover system/context mutation only. The directly related PR #8178 was closed unmerged, and the related systemPrompt work does not satisfy the promptOverride request.

## Best Possible Solution

Keep #8176 open until OpenClaw either implements and documents an explicit user-prompt replacement field for before_prompt_build and legacy before_agent_start, or a maintainer rejects that API. A complete fix should update the public hook result type and prompt-mutation field list, hook merge precedence, resolvePromptBuildHookResult, embedded runner application before prependContext/submission, allowPromptInjection behavior, docs, and focused regression tests for merge order and submitted prompt text.
