## Summary

Keep #39000 open. Current main still supports plugin gateway method registration and external gateway dispatch, but it does not expose `api.callGatewayMethod()` or another generic plugin-facing request/response seam for calling plugin-registered gateway methods from another plugin.

## Best Possible Solution

Keep #39000 open for maintainer product/API design at the plugin SDK/runtime boundary. The best fix is an explicit inter-plugin request/response seam, with contract tests proving plugin isolation, caller identity/scope propagation, plugin-registered handler dispatch, error propagation, startup/deferred-load behavior, and plugin author documentation.
