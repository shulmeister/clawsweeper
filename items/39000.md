## Summary

Keep #39000 open. Current main still supports plugin gateway method registration, but does not expose a plugin-facing `api.callGatewayMethod()` or equivalent generic inter-plugin request/response API. The linked metabolism/stability use case is concrete and points to a missing OpenClaw plugin SDK/runtime seam, not cleanup that can be moved to ClawHub alone.

## Best Possible Solution

Keep #39000 open for maintainer/product design. The likely implementation belongs in the plugin SDK/runtime boundary: add a deliberate inter-plugin request/response seam, either `api.callGatewayMethod()` or a more explicit typed service/RPC abstraction, with tests for plugin isolation, scope propagation, missing methods, handler failures, and startup timing.

## Risks / Open Questions

- Design still needs maintainer judgment around caller identity, authorization/scope propagation, startup availability, recursion, error mapping, and whether gateway methods are the right abstraction versus a typed service/RPC registry.
