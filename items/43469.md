## Summary

Keep open. Current main still does not scan Markdown skill definitions in the shared install/audit scanner: `.md` is not in `SCANNABLE_EXTENSIONS`, the current tests explicitly expect `readme.md` to be skipped, and the skill install/security audit paths rely on that scanner for dangerous-code findings. The PR is not implemented on main and is not superseded by the separate Skill Workshop scanner, though its own review comments should be addressed before merge.

## Best Possible Solution

Keep the PR open and ask for a focused maintainer/security review. The best path is to add `.md` coverage to `src/security/skill-scanner.ts`, add regression tests for `SKILL.md` scan inclusion and rule isolation, and harden the download-and-exec rule for the bypasses already noted in review before merging. If possible, align or share rule logic with the Skill Workshop scanner so generated-skill and installed-skill safety checks do not drift.
