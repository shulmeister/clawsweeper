## Summary

Keep open. Current main still has a real remaining adapter gap: Discord and Slack now have sendPayload support, but Google Chat, Mattermost, MS Teams, and Synology Chat still do not. The PR diff is stale against current shared outbound helpers and needs rework, but closing it would drop still-unimplemented behavior for four bundled channel plugins.

## Best Possible Solution

Keep #30142 open or replace it with a narrower follow-up that adds sendPayload for the still-missing bundled adapters: Google Chat, Mattermost, MS Teams, and Synology Chat. Reuse the current shared sendPayload/text-media helper patterns where possible, leave the already-shipped Discord and Slack implementations alone, and add focused adapter tests for text-only, media-only, mixed mediaUrls, rich/channelData payloads, and empty unsendable payload behavior.
