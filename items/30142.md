## Summary

Keep #30142 open. Current main at 82262387651ab270b565f71533ce4937c9c00dc6 only partially supersedes the PR: Discord and Slack already have shipped `sendPayload` implementations, but Google Chat, Mattermost, MS Teams, and Synology Chat still do not expose `sendPayload` on their outbound adapters. The PR should not merge as-is because its Discord/Slack pieces overlap newer code, but closing it would drop real remaining adapter work.

## Best Possible Solution

Keep #30142 open for maintainer action, or replace it with a narrower PR targeting only Google Chat, Mattermost, MS Teams, and Synology Chat. Drop the already-shipped Discord/Slack changes, reuse the current shared payload helpers where they fit, and add focused tests for text-only, media-only, mixed `mediaUrls`, empty/channelData-only payloads, configured chunk behavior, and any adapter-specific delivery semantics before landing.
