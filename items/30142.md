## Summary

Keep #30142 open. Current main still has a real remaining adapter gap: Discord and Slack now expose sendPayload, but Google Chat, Mattermost, MS Teams, and Synology Chat still do not. The PR branch is stale against current shared outbound structure, but the central requested behavior is not fully implemented on main.

## Best Possible Solution

Keep #30142 open, or replace it with a narrower follow-up that adds sendPayload for the still-missing bundled adapters: Google Chat, Mattermost, MS Teams, and Synology Chat. Reuse the current plugin-sdk reply-payload/shared outbound helper patterns where possible, leave the already-shipped Discord and Slack implementations alone, and add focused adapter or contract tests for text-only, media-only, mixed mediaUrls, structured/channelData payloads, chunking, and empty unsendable payload behavior.
