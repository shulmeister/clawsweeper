## Summary

Keep #30142 open. Current main still only partially supersedes the PR: Discord and Slack already have shipped sendPayload support, but Google Chat, Mattermost, MS Teams, and Synology Chat still lack sendPayload on their bundled outbound adapters. The PR should be narrowed or rebased before merge, but the remaining adapter work is real and not fully tracked by a narrower canonical item in the provided context.

## Best Possible Solution

Keep #30142 open for maintainer action, or replace it with a narrower PR that targets only Google Chat, Mattermost, MS Teams, and Synology Chat. Reuse the current shared reply-payload helpers where they fit, leave the shipped Discord/Slack implementations alone, and add adapter-level payload contract coverage before landing.
