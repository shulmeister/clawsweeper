## Summary

Keep open. Current main appears to have addressed the token/signature side for remote node-host auth, but the requested first-class SSH transport for `openclaw node run` is still missing. The node CLI and runner still build a direct `ws`/`wss` URL from host/port/TLS settings, while the docs continue to tell users to manage an external SSH tunnel for remote node hosts.

## Evidence

- **Node CLI exposes only direct connection options:** `openclaw node run` registers `--host`, `--port`, `--tls`, `--tls-fingerprint`, `--node-id`, and `--display-name`; there is no `--ssh`, `--ssh-identity`, or transport option for node host startup.
