## Summary

Keep #23926 open. Current main has partial blocking support through the existing `before_install` plugin hook for gateway-managed skill dependency installers, but native ClawHub skill install/update and the Control UI ClawHub install path still go through `installSkillFromClawHub` and `installPackageDir` without a pre-copy policy hook. The plugin hook surface also still has no post-install observational lifecycle hook with installedAt/targetDir/success/failure audit semantics.

## Best Possible Solution

Keep #23926 open as the tracker for completing skill install lifecycle hook/API coverage. Extend the existing `before_install` contract or add equivalent skill lifecycle hooks so native ClawHub skill install/update/gateway/UI paths run a blocking policy point before copying files, then add a documented post-install observational hook carrying installedAt, targetDir, source, outcome, and audit metadata. Clarify separately whether external `clawhub sync` can integrate with the same API or needs ClawHub-owned support.
