## Summary

Keep open. Current main still does not implement a target auth-profile flag for `models auth login`, and this PR's proposed `--profile` spelling still conflicts with OpenClaw's existing global `--profile` parser. The related #40402 issue and #49315 PR confirm this is a real auth-profile UX gap, but they do not make this PR safely closeable as implemented or superseded without maintainer judgment.

## Best Possible Solution

Keep the PR open for maintainer/contributor action. The likely acceptable path is to use a non-conflicting flag such as `--profile-id` or another explicit auth-profile name, validate provider prefixes and collision behavior, plumb it through a typed provider-auth contract, update docs/tests, and handle provider-owned flows such as GitHub Copilot without duplicate auth/config state.
