## Summary

Current `main` already routes bare plugin ids like `acpx` to bundled plugins before any ClawHub fallback, and the same implementation is present in released tag `v2026.4.22`, so the reported CLI behavior is already fixed on supported code.

## Close Comment

Closing as implemented on current `main` after Codex review. The reported `openclaw plugins install acpx` path is already handled before any ClawHub fallback: `src/cli/plugins-install-command.ts:484` calls `resolveBundledInstallPlanBeforeNpm`, and `src/cli/plugin-install-plan.ts:48` resolves bare specs by bundled `pluginId` directly. There is also direct coverage in `src/cli/plugin-install-plan.test.ts:11`. The same implementation is already present in the released `v2026.4.22` tree as well, so this no longer needs a separate maintainer action.
