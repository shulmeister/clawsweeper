## Summary

Keep #38691 open. Current main and latest release v2026.4.24 still hard-stop the Android Voice Tab mic path when `SpeechRecognizer.isRecognitionAvailable(context)` is false, with no OEM `voice_recognition_service` fallback. The PR addresses a real missing in-repo Android behavior, but its current patch needs maintainer review/fixes because the discussion identifies a credible retry-loop regression when no usable recognizer backend exists.

## Best Possible Solution

Keep the PR open as a plausible Android bug fix, but require an updated patch against current `apps/android/app/src/main/java/ai/openclaw/app/voice/MicCaptureManager.kt` that uses the configured OEM recognizer when available and preserves graceful disable behavior when no usable recognizer backend exists. Add focused coverage where feasible, or document concrete OEM/manual validation if Android framework testing is impractical.
