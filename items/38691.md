## Summary

Keep #38691 open. Current main still disables the Android Voice Tab manual mic path whenever `SpeechRecognizer.isRecognitionAvailable(context)` is false, and repository search found no OEM `voice_recognition_service` lookup or targeted recognizer creation. The PR is a focused, plausible fix with real Xiaomi device verification, but it needs maintainer review/rebase and should address the retry-loop concern before merge.

## Best Possible Solution

Keep the PR open for maintainer review or author revision. The likely fix belongs in `apps/android/app/src/main/java/ai/openclaw/app/voice/MicCaptureManager.kt`: add a generic configured OEM speech recognizer fallback using the secure `voice_recognition_service` component and `SpeechRecognizer.createSpeechRecognizer(context, component)`, while retaining a clean disable path when no backend is usable. Add focused Robolectric coverage if feasible, plus manual verification on a Chinese OEM device and a no-recognizer safety check.
