## Summary

Keep #38691 open. Current main still hard-disables the Android Voice Tab manual mic path when `SpeechRecognizer.isRecognitionAvailable(context)` is false, and there is still no `voice_recognition_service` or targeted OEM recognizer fallback. The PR is stale against the current package path and needs the retry-loop concern addressed, but the core bug remains unsolved on main.

## Best Possible Solution

Keep the PR open for revision rather than close it. The best path is to rebase the change onto `apps/android/app/src/main/java/ai/openclaw/app/voice/MicCaptureManager.kt`, add a generic configured OEM recognizer fallback using the secure `voice_recognition_service` component and `SpeechRecognizer.createSpeechRecognizer(context, component)`, and preserve a clean disable path when no recognizer backend is usable. Focused Android/Robolectric coverage would be useful if feasible, with the existing Xiaomi device verification retained as manual evidence.
