## Summary

Keep #38691 open. Current main and latest release still hard-stop the Android Voice Tab manual mic path when `SpeechRecognizer.isRecognitionAvailable(context)` is false, and there is no OEM recognizer lookup or `Settings.Secure.VOICE_RECOGNITION_SERVICE` fallback on the shipped path. The PR also has a credible unresolved review concern: removing the hard stop without tightening `ERROR_CLIENT` handling could turn unsupported devices into a retry loop.

## Best Possible Solution

Keep the PR open for maintainer review or revision. The likely product path is to add a generic OEM speech-recognition fallback in `apps/android/app/src/main/java/ai/openclaw/app/voice/MicCaptureManager.kt`, preferably using `Settings.Secure.VOICE_RECOGNITION_SERVICE` rather than a raw setting string, while preserving graceful disable behavior for devices with no usable recognizer and adding focused Android regression coverage or clearly documented manual-device verification.
