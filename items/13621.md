## Summary

Current main has superseded the legacy Chrome extension relay path with Chrome MCP existing-session attach and raw-CDP/managed Playwright paths. The observable problem, browser automation being blocked by iframe boundaries, is covered by current Chrome MCP whole-page snapshot/ref actions and by Playwright frame-scoped snapshots/actions for CDP profiles.

## Risks / Open Questions

- This was a read-only review; I did not run a live Salesforce Embedded Messaging reproduction against Chrome.
- Existing-session Chrome MCP still has documented route limits: selector/frame-scoped snapshots are not supported there, so the supported workflow is whole-page snapshot refs; managed/raw CDP profiles provide explicit --frame scoping.
