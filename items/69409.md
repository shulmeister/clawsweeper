## Summary

Keep #69409 open. Current main has reload modes, restart-required planning, active-work deferral, and separate Crestodian approval for explicit operator restarts, but it does not implement a human approval gate for restart-required direct config-file edits. In default `hybrid` mode, changes like `plugins.allow` still route to an automatic gateway restart once the drain/deferral path allows it.

## Best Possible Solution

Keep the issue open as a core gateway/config-reload product request. The likely implementation belongs under `gateway.reload`: add `auto`/`notify`/`manual` semantics or equivalent, persist pending restart metadata with changed paths and file-watch audit context, send configured admin notifications, expose CLI/RPC approve/deny commands, and add tests/docs covering direct file-watch reloads plus `config.patch`/`config.apply` behavior. Existing Crestodian explicit restart approval can inform the UX, but it does not currently intercept config-triggered restarts.
