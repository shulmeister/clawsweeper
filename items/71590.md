## Summary

Keep #71590 open. Current main still implements only the existing exponential temporal-decay path, and the strict runtime config, generated schema, docs, and tests do not contain the proposed `model: "weibull"` or `weibullShape` options. The PR is a recent draft with a concrete implementation for the memory-ranking request in #57307, but it has unresolved review feedback about inverted public JSDoc, so it is not obsolete or closeable cleanup.

## Best Possible Solution

Keep #71590 open as a focused draft implementation candidate. The contributor should fix the public JSDoc direction, complete the PR evidence/template, update public memory config docs if the new config is user-facing, run the memory-core/config changed gates, and let maintainers decide whether the Weibull decay model belongs in bundled `memory-core` or in the broader #57307 memory-ranking design.
