## Summary

Current main and the v2026.4.24 release implement the missing-result-context fix: exec completion events are queued with their output, heartbeat exec-event prompts embed those completion details directly instead of referring to missing "system messages above", and empty-output completion events use a quiet HEARTBEAT_OK fallback. This also matches the related #69626 cleanup for empty/mismatched exec-event heartbeat sessions.

## Best Possible Solution

Close #66982 as implemented. Keep the shipped v2026.4.24 exec-event prompt and scoped wake behavior; if someone still reproduces on v2026.4.24 or newer, ask for a fresh issue with the session transcript, queued system-event text, heartbeat config, channel/plugin route, and exact OpenClaw version because that would be a new regression against the current implementation.
