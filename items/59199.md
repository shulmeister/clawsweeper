## Summary

Current main already exposes the requested post-turn completion signal via the public plugin hook `agent_end`, with documented completion semantics and runtime wiring in both embedded and CLI agent paths. The exact internal hook name proposed in the issue does not exist, but the capability itself is already implemented well enough to support post-response notifications or sound playback.

## Evidence

- **Public docs describe a completion hook:** The agent loop docs list `agent_end` as a plugin hook that runs after completion and is intended for inspecting final messages and run metadata.
  - file: [docs/concepts/agent-loop.md:97](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/docs/concepts/agent-loop.md#L97)
- **Hook is part of the public plugin contract:** `agent_end` is included in the exported plugin hook names, with a typed event payload carrying `messages`, `success`, `error`, and `durationMs`.
  - file: [src/plugins/hook-types.ts:57](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/plugins/hook-types.ts#L57)
- **Embedded agent runtime emits the completion hook:** The embedded runner calls `runAgentEnd(...)` at the end of a run with success/error state and duration, matching the requested 'turn complete' lifecycle signal.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:2655](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/pi-embedded-runner/run/attempt.ts#L2655)
- **CLI agent runtime emits the same hook:** The CLI runner emits `runAgentHarnessAgentEndHook(...)` on both success and failure paths, so the completion seam is available across agent runtimes.
  - file: [src/agents/cli-runner.ts:299](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/cli-runner.ts#L299)
- **Internal hooks docs already redirect users to plugin hooks for deeper lifecycle automation:** The internal hooks page explicitly separates gateway hooks from plugin hooks and points readers to the plugin hook surface for richer lifecycle integration.
  - file: [docs/automation/hooks.md:206](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/docs/automation/hooks.md#L206)
- **Bundled plugins already use `agent_end` for post-turn automation:** Bundled plugins consume `agent_end` today for after-turn workflows, which shows this is the intended seam rather than a missing core capability.
  - file: [extensions/skill-workshop/index.ts:48](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/skill-workshop/index.ts#L48)

## Risks / Open Questions

- Discoverability is still a little rough: users reading only the internal hooks page may miss that post-turn completion is exposed through plugin hooks as `agent_end`, not as a literal internal `agent:turn:complete` event.
