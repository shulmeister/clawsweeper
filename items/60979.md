## Summary

Keep open. Current main does not implement the requested `sessions_spawn(runtime="acp")` delivery option for binding ACP run output to an arbitrary channel/topic. The existing ACP spawn surface exposes only `streamTo: "parent"`, rejects several direct channel-delivery params, and intentionally keeps background run-mode ACP output internal so the parent/task lifecycle handles completion.
