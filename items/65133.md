## Summary

Current main already routes managed loopback CDP health checks outside the browser SSRF policy, which matches the root cause identified in the follow-up comment on #65133. The same behavior is present in the shipped v2026.4.23 release tag, with regression tests and docs covering strict-SSRF loopback CDP behavior.

## Evidence

- **Managed loopback CDP control is exempted from browser SSRF gating:** `resolveCdpReachabilityPolicy()` returns `undefined` for non-remote loopback `openclaw` profiles, so OpenClaw's own local managed CDP control plane is not blocked by strict browser SSRF policy.
