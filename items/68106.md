## Summary

Keep open. Current main still swallows `resolveWorkflowFile` errors and falls back to inline pipeline execution, so the linked Lobster workflow-path bug is not implemented on main. The PR is a plausible focused fix, but the existing review feedback identifies a real edge case: extension-only detection can misclassify valid inline pipelines ending in file-like tokens, so it needs tightening rather than automatic cleanup closure.

## Best Possible Solution

Keep the item open or consolidate it with the active Lobster workflow-path fix work. The best code path is to update `extensions/lobster/src/lobster-runner.ts` so missing workflow-file path errors are rethrown only for inputs that are clearly workflow file references, while preserving valid inline pipelines with file-like arguments; add tests for both the missing relative workflow path and the inline-pipeline edge case called out in review.
