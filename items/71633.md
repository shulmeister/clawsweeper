## Summary

Keep open. Current main still bundles the default-enabled Bonjour plugin through unpatched @homebridge/ciao@1.3.6, and the published ciao 1.3.6 Windows interface path still calls `arp -a | findstr /C:"---"` without an options object. This PR remains a focused fix for the linked Windows terminal-flicker bug rather than conservative cleanup.

## Best Possible Solution

Keep #71633 open for maintainer review. The best path is to land this narrow `@homebridge/ciao@1.3.6` pnpm patch, or replace it with an upstream ciao version that includes the same `windowsHide: true` behavior, with lockfile consistency and either a focused regression check or Windows manual verification.
