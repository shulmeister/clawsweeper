## Summary

Keep open. Current main still sends BlueBubbles inbound text through raw `BodyForAgent`, so the hardening gap is not implemented on main. The PR as submitted is also not a safe cleanup close or merge-as-is: it wraps only the legacy `formatInboundEnvelope` body while the model-facing path still prefers `BodyForAgent`, and existing review discussion already asks for follow-up changes.

## Best Possible Solution

Keep the PR open, but require changes before merge. The fix should target the model-facing path (`BodyForAgent` semantics or structured `UntrustedContext`) rather than only the deprecated envelope `Body`, preserve empty-body behavior, and add focused BlueBubbles coverage showing wrapped untrusted context reaches the prompt without regressing command parsing or attachment-only messages.
