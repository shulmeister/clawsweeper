## Summary

Keep #38235 open. Current main still resolves Slack agent identity for normal delivery, but the draft preview stream has no identity parameter and its initial send omits the resolved username/icon. The native Slack streaming path also starts `client.chatStream` without any identity fields. Related PR #38237 remains the focused open fix in the provided context, while #46432 was closed unmerged.

## Best Possible Solution

Keep this issue open. The best maintainer path is to land or adapt the focused draft-preview fix from PR #38237 by adding an optional Slack send identity to `createSlackDraftStream`, passing `slackIdentity` from dispatch, and adding a regression test for the initial draft `chat.postMessage`. Then verify the pinned Slack SDK/native streaming API; if native stream authorship can be passed safely, thread it through `startSlackStream`, otherwise document that per-agent Slack authorship requires normal or draft-preview delivery.
