## Summary

Current main still has the reported Slack streaming identity gap. Normal Slack delivery resolves and passes agent identity, but the draft preview stream has no identity parameter and its first `sendMessageSlack` call omits identity; native Slack streaming also starts `client.chatStream` without an identity surface. The linked fix PR #38237 is still open and PR #46432 was closed unmerged, so this is not implemented or superseded on main.

## Best Possible Solution

Keep this issue open. The best maintainer path is to land or adapt the focused draft-preview fix from #38237: add an optional Slack send identity to `createSlackDraftStream`, pass `slackIdentity` from dispatch, assert the initial draft send receives it, and document or explicitly handle the native `chatStream` limitation after verifying Slack's current API surface. Do not close this in favor of #37763 unless maintainers intentionally choose that broader issue as the canonical Slack identity tracker.
