## Summary

Keep open. Current main still accepts `cron add/edit --system-event ... --session main` without the requested shell-command warning, and the runtime still treats main-session `systemEvent` payload text as queued heartbeat/reminder content rather than a shell command. The linked fix PR #63112 directly tracks this warning but is open and unmerged, so the issue remains actionable.

## Best Possible Solution

Keep #63107 open and land or rework PR #63112, or implement the same narrow fix on main: shared shell-command heuristic, non-blocking stderr warning in both `src/cli/cron-cli/register.cron-add.ts` and `src/cli/cron-cli/register.cron-edit.ts` before the RPC call, and focused CLI tests covering warning/no-warning cases. A small docs clarification on `docs.openclaw.ai/automation/cron-jobs` would also help distinguish main-session system events from isolated agent-turn script execution.
