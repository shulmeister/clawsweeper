## Summary

Keep #66934 open. Current main has fixes for the related optimistic-message/history-reload flicker, but the sender-label part is not fully resolved: chat history sanitization still promotes inbound metadata `senderInfo.label` into `senderLabel`, and the Control UI/WebChat client id is still `openclaw-control-ui`, which the UI will render ahead of the local user fallback label.

## Best Possible Solution

Keep the issue open and fix the sender-label path directly: preserve real inbound sender labels for non-direct/group/channel surfaces, but suppress or map internal direct WebChat client ids such as `openclaw-control-ui` to the local user fallback before `senderLabel` reaches `renderMessageGroup`. Add focused regression coverage for a user history entry containing `Sender (untrusted metadata)` with `{"label":"openclaw-control-ui"}` so it renders as the local user (`You`) while existing group sender-label tests continue to pass. The optimistic-message/history-reload fixes already on main should be kept as-is.
