## Summary

Keep open. Current `main` still has an unguarded startup path where `start()` awaits `runMissedJobs()` before `armTimer()`, so store/persist failures can still abort startup and leave the scheduler unarmed. The PR body's specific example about isolated-agent execution failures is overstated on current `main` because those execution errors are already converted into error outcomes, but the broader startup-resilience gap is still real and the PR discussion shows substantive follow-up fixes rather than a stale or incoherent change.

There is no surrounding recovery path for startup catch-up failures here.
