## Summary

Keep open. Current `main` still strips Telegram formatting entities from inbound agent context: `resolveTelegramInboundBody()` only runs `expandTextLinks()` before building `rawBody`, and `expandTextLinks()` only rewrites `text_link` entities. The Telegram inbound-context payload then forwards that plain `bodyText`/`rawBody` into `BodyForAgent` and `RawBody`. Existing tests cover `text_link` expansion only and explicitly expect non-`text_link` entities like `bold` to leave the text unchanged, so there is no evidence this regression has been fixed on `main`.
