## Summary

Current `main` already covers the private-network node-install flow this PR is trying to unblock: `openclaw node install` passes the install process environment into node service environment construction, `buildNodeServiceEnvironment` persists `OPENCLAW_ALLOW_INSECURE_PRIVATE_WS`, tests cover that passthrough, and docs now document the exact `OPENCLAW_ALLOW_INSECURE_PRIVATE_WS=1 openclaw node install ...` workflow. The requested `--daemon-env` flag would be redundant for the reported use case rather than fixing a missing behavior on `main`.
