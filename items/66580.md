## Summary

Current main already implements the core behavior from draft PR #66580 and the linked #66487 bug: heartbeat exec-event prompts are built from the actual queued exec completion text, empty completions no longer ask for user relay, duplicate/noisy exec completion paths have producer-side safeguards, and tests plus the v2026.4.24 changelog cover the shipped behavior.

## Best Possible Solution

Close draft PR #66580 as implemented and keep the shipped mainline implementation from v2026.4.24. If a new poll-vs-notify edge case remains, track it as a fresh narrow issue with current logs, config, affected channel/plugin details, and reproduction steps rather than keeping this duplicate draft PR open.
