## Summary

Current main already implements the useful behavior from draft PR #66580: heartbeat exec-event prompts now receive the actual pending exec completion payload, avoid the old “shown in system messages above” wording, suppress empty-output relays, and include tests for the payload path. The PR’s secondary replay-invalid classification change is also present on main.

## Best Possible Solution

Close this stale draft PR as implemented and keep the shipped mainline implementation, which handles the linked #66487 user problem with payload-bearing exec prompts plus empty-output and duplicate-relay safeguards. Any newly observed poll-vs-notify edge case should be filed as a fresh, narrow issue with current reproduction logs.
