## Summary

Keep PR #70432 open. Current main still has the macOS installer behavior the PR targets: `install_node` calls `run_quiet_step` for `brew install node@${NODE_DEFAULT_MAJOR}` as a bare command, and `ensure_macos_default_node_active` still emits the "was installed" PATH-ordering message even when the Homebrew node binary is not present on disk. The linked bug #70411 has concrete reproduction evidence, and the PR discussion shows review feedback was addressed rather than superseded.

## Evidence
