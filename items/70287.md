## Summary

Keep PR #70287 open. Current main still uses the unsupported `$search` query for `msteams:search`, and the PR remains a focused, relevant fix rather than an implemented, duplicate, or non-actionable item.

## Best Possible Solution

Keep this PR open and review it as the active fix for #70127. The best implementation direction is the PR's MS Teams plugin-local list-and-local-filter path: remove `$search` from chat/channel message-list requests, keep sender filtering where Graph supports it, cap `$top` at 50 with pagination for the search window, strip Teams HTML before local matching, and keep focused regression tests proving `$search` is absent.
