## Summary

Keep open. Current main still has a fixed TUI footer and no public `registerStatusProvider` or equivalent plugin status-contribution API. There is overlapping open work and a broader related open footer-customization issue, so this should remain open for maintainer review or consolidation rather than be auto-closed.

## Best Possible Solution

Keep this issue open. The best path is a narrow core/plugin-API design for TUI status contributions: define a typed `PluginStatusProvider`, expose `registerStatusProvider` through `OpenClawPluginApi`, store providers in the plugin registry, wire the TUI/backend to read and render them safely, and add SDK docs plus contract/TUI tests. Coordinate with #53774 as the broader footer-customization tracker, and split or rework the status-provider pieces from #64127 if that PR becomes the implementation vehicle.
