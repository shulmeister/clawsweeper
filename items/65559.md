## Summary

Current `main` already fixes the reported webhook-mention drop: Discord preflight now hydrates mention metadata from REST when message text contains mention syntax but Discord left mention arrays empty, and a regression test covers that exact `allowBots: "mentions"` case. The separate per-guild/per-channel `allowBots` point is not part of the current typed Discord config contract, so it does not keep this specific issue open.

## Evidence

- **Mention-like text now triggers REST hydration:** `shouldHydrateDiscordMessage` returns true when inbound Discord text contains `<@...>`, `<@&...>`, `@everyone`, or `@here` but mention metadata is empty, which addresses webhook-authored messages whose `mentionedUsers` array is blank.
