## Summary

Keep open. Current main still retries Telegram `getFile`, but the actual Bot API file download is a single `fetchRemoteMedia` call with no 2-3 attempt backoff. Existing tests still assert that `fetchRemoteMedia` errors are not caught/retried, and the related PR #46013 was closed unmerged in the provided GitHub context.
