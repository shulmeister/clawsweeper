## Summary

Keep #11960 open. Current main still fixes only the secondary hook-config schema mismatch; it does not persist bundled session-memory on automatic daily/idle rollovers because the stale rollover path archives the old transcript and fires plugin session_end/session_start hooks without emitting the internal command reset hook or before_reset path that session-memory listens to.

## Best Possible Solution

Keep #11960 open until main emits the existing reset hook path for automatic daily/idle stale rollovers, with previousSessionEntry available to session-memory and before_reset consumers. The narrow maintainer path is to land or replace PR #71792/#61675 with regression coverage for daily and idle rollovers, then update session-memory metadata/docs to describe the shipped auto-reset behavior.
