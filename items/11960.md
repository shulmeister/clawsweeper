## Summary

Keep open. Current main partially addresses the surrounding lifecycle work with plugin `session_end` hooks for daily/idle rollovers and fixes the hook-config schema passthrough, but the requested bundled `session-memory` behavior still is not implemented: it is still registered only for `command:new` and `command:reset`, and its handler exits unless the internal event is a `/new` or `/reset` command. The newer `before_reset` plugin hook is also typed for explicit `new`/`reset` reasons only, not daily/idle auto-resets.
