## Summary

Keep #11960 open. Current main still fixes only the secondary hook-config schema mismatch; the primary auto daily/idle rollover memory persistence gap remains. `session-memory` still listens only to internal `command:new` / `command:reset`, while stale rollover in `initSessionState()` archives the prior transcript and fires typed `session_end` / `session_start` plugin hooks without calling the reset command hook path. The provided related PRs #71792 and #61675 target this exact missing behavior and are still open, so this issue should remain available as the tracker until a fix lands or maintainers choose a replacement.

## Best Possible Solution

Keep #11960 open until #71792, #61675, or a replacement lands a narrow fix that fires the existing reset hook behavior, or an equivalent documented session reset surface, during automatic daily and idle stale rollover. The fix should pass the previous session entry/transcript to session-memory, preserve typed `session_end` / `session_start` behavior, decide whether `before_reset` is part of the intended plugin contract for auto rollover, and add regression coverage for both daily and idle rollover.
