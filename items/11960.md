## Summary

Keep open. Current main fixes the secondary hook-config schema passthrough and has adjacent plugin lifecycle support for daily/idle rollovers, but it still does not implement the requested internal `session:reset`/`session:before-reset` event or wire the bundled `session-memory` hook to automatic daily/idle resets. The existing bundled hook remains command-scoped, while auto rollovers emit plugin `session_end` lifecycle hooks instead.

## Best Possible Solution

Keep this issue open until OpenClaw either adds a supported internal `session:reset`/`session:before-reset` event and wires `session-memory` to daily/idle rollovers, or maintainers explicitly decide that plugin `session_end` is the intended replacement and update the docs/issue with a migration path. The secondary hook-config schema issue can be treated as already fixed.
