## Summary

Keep open. Current main still does not implement the requested internal `session:reset`/`session:before-reset` hook event, and the bundled `session-memory` hook still only persists on explicit `/new` or `/reset` command events. The secondary schema mismatch is fixed, and there is useful adjacent plugin lifecycle work plus open related trackers/PRs, but the requested auto-reset memory persistence path is not shipped on main.

## Best Possible Solution

Keep #11960 open until OpenClaw either ships automatic daily/idle reset persistence for `session-memory` with a supported internal reset event/contract, or maintainers explicitly decide that plugin `session_end`/`before_reset` plus a documented migration path is the intended replacement. The secondary hook-config schema issue can be treated as fixed.
