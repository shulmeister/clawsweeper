## Summary

Keep #39248 open. The report is concrete, not protected by maintainer association/labels, and current main still lacks source, test, docs, release, or live evidence strong enough to prove that `sessions_spawn` subagents initialize successfully when `agents.defaults.sandbox.mode` is `"non-main"`. The current code has relevant sandbox plumbing, but `sessions_spawn` can still return `accepted` before the child run has emitted activity, which matches the reported failure shape.

## Best Possible Solution

Keep the issue open until maintainers add or identify a targeted regression proof for `sessions_spawn` under `agents.defaults.sandbox.mode: "non-main"`. The ideal fix path is an integration or focused mocked test that exercises a spawned child session through sandbox context creation and verifies child run start/transcript/message state, plus an implementation change if the Docker-backed initialization path still stalls silently.
