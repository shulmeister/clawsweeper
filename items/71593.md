## Summary

Keep this issue open. Current main still has a closed SecretRef source/provider model (`env`, `file`, `exec`) and no plugin-defined SecretRef provider contract. The provided GitHub context also shows open PR #72548 with closing syntax for this issue, so the issue should remain open until that PR is reviewed and merged or closed. The engram authToken comment remains a concrete data point for the unsolved plugin config coverage problem.

## Best Possible Solution

Keep this open as a core/plugin API design issue. Maintainers should review PR #72548 as the current implementation candidate for pluggable SecretRef sources, then decide whether to land it, revise it, or explicitly document `exec` as the supported universal provider extension point. If the plugin provider seam lands, separately track any remaining schema coverage gaps such as `skills.entries.*.env` and plugin config fields like the engram `agentAccessHttp.authToken` example.
