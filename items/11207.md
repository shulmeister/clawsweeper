## Summary

Keep open. Current `main` has meaningful hardening for untrusted file/web/hook content, but the implementation is still a prompt-level mitigation rather than a conclusive fix for this report. The code now wraps attached file text in `EXTERNAL_UNTRUSTED_CONTENT` markers and warning text, yet the threat model still records indirect prompt injection as a current risk with "Residual Risk: High," and I did not find an enforcement path that temporarily disables mutating tools or otherwise guarantees a user’s explicit read-only directive wins over model behavior.
