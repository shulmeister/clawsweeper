## Summary

Current `main` no longer schedules managed `memory-core` dreaming on the `main` session. It now reconciles managed jobs to `sessionTarget: "isolated"`, and gateway cron tests verify those jobs run on dedicated `cron:<jobId>` sessions instead of `main`, which removes the `operator.admin` cleanup path described in #68074.

## Evidence

- **Managed dreaming cron now uses isolated sessions:** `buildManagedDreamingCronJob` now returns `sessionTarget: "isolated"` with an `agentTurn` payload and `delivery: { mode: "none" }`, instead of the old `main`/`systemEvent` shape reported in the issue.
