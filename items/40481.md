## Summary

Keep this PR open. Current main has better native/interactive approval buttons for Telegram and channels that render shared interactive replies, but the generic forwarded fallback still emits the old placeholder text (`/approve <id> ...`). The PR's core request remains useful for plain-text or non-interactive channel notifications, although it should be rebased onto the current approval-renderer architecture.

## Best Possible Solution

Keep the item open and rework or rebase the PR so the generic fallback text in `src/infra/exec-approval-forwarder.ts` emits concrete per-decision `/approve <actual-id> ...` commands while preserving existing native Telegram and shared interactive button behavior. Use existing approval action helpers where possible, and keep explicit coverage for markdown-capable and plain/non-markdown channel text.
