## Summary

Keep open. Current main still has the exact Windows coverage gap described by the issue: the Windows CI workflow delegates to `pnpm test:windows:ci`, and that package script still omits `extensions/lobster/src/lobster-runner.test.ts`. The pnpm runner regression is covered, but the Lobster-side packaged-runtime regression is not enforced by the Windows lane on main. The related PR #67395 is open/unmerged in the provided context, so it does not make the issue implemented on main.

## Best Possible Solution

Keep the issue open until main either adds `extensions/lobster/src/lobster-runner.test.ts` to `test:windows:ci` or replaces it with an equivalent Windows install/build smoke that exercises the packaged Lobster runtime resolution path. If #67395 remains the active fix vehicle, port or merge its Lobster fallback regression coverage and update the Windows lane before closing this issue.
