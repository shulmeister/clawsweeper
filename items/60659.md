## Summary

Keep open. Current main still only resolves session reset policy from global session config, chat type, and channel override. I did not find a general per-agent or per-route-binding reset policy surface; the existing ACP persistent/oneshot mode is narrower and does not cover the reported embedded/route-agent session reset use case.
