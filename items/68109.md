## Summary

Keep open. Current `main` does not provide strong evidence that #68109 is fixed: the Linux/systemd restart path still just runs `systemctl restart` and returns a completed outcome, the CLI command still only awaits that restart path without any explicit process-exit normalization, and existing tests only assert the success message, not the process exit status. The only directly related fix I found is PR #68302, which was closed rather than landed.
