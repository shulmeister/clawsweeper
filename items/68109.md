## Summary

Keep open. Current main still lacks source or regression-test evidence that `openclaw gateway restart` on Linux/systemd always reports process exit code 0 on success. The systemd helper returns a completed restart outcome after `systemctl restart`, the CLI action only awaits that path, and the related PR #68302 was closed unmerged.

## Best Possible Solution

Keep the issue open until maintainers either reproduce the current systemd behavior or add a focused regression harness for `openclaw gateway restart` that asserts child-process `code === 0` and `signal === null` on successful service-manager restarts. If the bug still reproduces, fix it at the CLI/lifecycle boundary for all service managers rather than with a narrow systemd-only helper side effect.
