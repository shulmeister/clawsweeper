## Summary

Keep #70372 open. Current main has phase-aware delivery for explicit `commentary`/`final_answer` metadata and OpenAI Responses pending-phase handling, but it does not contain this PR's `hadThinkingInMessage` state or an equivalent Anthropic/Bedrock unphased-thinking fallback. The related #62030 work covers explicitly phased commentary, and #70344 is a broader still-open candidate, but neither proves this specific PR obsolete on current main.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to either land a scoped version that only applies the thinking-derived final-answer fallback to unphased Anthropic/Bedrock-style streams, or fold that exact regression into the broader #70344/#62030 delivery-suppression work with a focused test that emits native thinking events followed by unphased text and verifies no narration reaches block/partial channel delivery when reasoning visibility is off.
