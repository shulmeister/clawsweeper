## Summary

Keep this PR open. Current main still lacks a general embedded-runner egress guard that strips empty `tools` arrays and orphaned `tool_choice`, while the related `toolsAllow` forwarding bug is already fixed. The PR remains a plausible focused fix for the active-memory / strict OpenAI-compatible provider failure, though it still needs maintainer review and stronger regression tests.

## Best Possible Solution

Keep this PR open for maintainer review or a successor patch. The best fix is a narrow embedded-runner egress sanitizer that strips empty `tools` arrays and removes `tool_choice` when no non-empty tools remain, covers session HTTP fallback, boundary-aware, and custom stream paths, preserves non-empty tool payloads, composes with existing payload wrappers, and adds direct regression tests. Avoid redoing the already-landed `toolsAllow` forwarding work.
