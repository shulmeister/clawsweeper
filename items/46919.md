## Summary

Keep open. Current main has adjacent auth controls, including tokenless Tailscale Serve for Control UI/WebSocket and explicit no-auth loopback mode, but it does not implement the requested broader config behavior: raw non-loopback/tailnet no-auth is still rejected, HTTP APIs do not use Tailscale header auth, and `openclaw dashboard` only suppresses tokenized URLs for SecretRef-managed tokens rather than a general `urlToken`-style config flag. This is core Gateway auth/security behavior, not a ClawHub plugin cleanup item.
