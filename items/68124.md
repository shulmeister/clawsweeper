## Summary

Keep open. Current `main` still lacks the observability fix discussed on the issue, and it still contains unguarded override normalization paths that can plausibly produce the reported non-fatal `toLowerCase`/type errors during bootstrap-related session resolution. The issue may be coming from custom or managed `agent:bootstrap` hook code rather than the bundled hook itself, but the current codebase still does not provide enough stack/context to close this confidently.

## Evidence

- **Internal hook errors still log message-only:** `triggerInternalHook` still catches handler exceptions and logs only `Hook error [type:action]: <message>`. There is no redacted stack trace or event context on `main`, so the debugging improvement described in the issue comment is not implemented in the checked-out code.
