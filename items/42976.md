## Summary

Keep open. Current main does not include the PR's two core fixes: safe-regex tokenization still handles generic escapes before character-class state, and session persistence still only applies input provenance before writing messages, with no configured redaction pass for persisted `text` or `thinking` blocks. The linked bug #42982 remains a valid product/code issue, though this PR also needs rework before merge because the discussion identifies an unaddressed `toolCall.arguments` redaction gap and there is no focused regression coverage for the reported session-log path.

## Best Possible Solution

Keep the PR open for maintainer handling or supersede it with a focused maintainer PR that fixes safe-regex character-class escape handling, applies configured redaction consistently before session JSONL persistence, explicitly decides how to handle `toolCall.arguments`, and adds regression tests tied to #42982.
