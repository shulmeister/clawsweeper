## Summary

Keep #36687 open. Current main has guardrails and tests for the WebChat/UI stale external-route leakage symptom, but the specific 2026.3.2 update/reset claim for session.dmScope is still unresolved. Runtime routing, outbound key construction, doctor checks, schema, and docs still treat omitted session.dmScope as main; onboarding only writes per-channel-peer for unset local setup and preserves explicit values.

## Best Possible Solution

Keep the issue open until maintainers verify the 2026.3.2 update/reset path. If an update can drop or ignore a previously isolated session.dmScope, add a targeted migration/doctor repair plus regression test. If omitted dmScope is intentionally still main, close only with an explicit maintainer product decision and keep the existing WebChat stale-route guardrails.
