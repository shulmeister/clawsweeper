## Summary

Keep #36687 open. Current main has guardrails for the WebChat/TUI stale external-route leakage symptom, but the issue's stronger update regression claim remains unresolved: there is still no evidence that the 2026.3.2 path cannot drop or ignore an existing isolated `session.dmScope`, and current code intentionally treats omitted `session.dmScope` as `main`.

## Best Possible Solution

Keep the issue open for maintainer verification of the 2026.3.2 update/reset path. If an update can drop or ignore an existing isolated `session.dmScope`, add a targeted migration or doctor repair plus regression coverage around config preservation. If omitted `dmScope` intentionally remains `main` and no reset-path reproduction appears, close only after explicit maintainer product judgment while keeping the shipped WebChat/UI stale-route guardrails.
