## Summary

Current `main` no longer matches the reported Control UI bleed-through path: startup context and system events are separated from transcript-visible user history, and Control UI chat history renders the clean transcript path.

## Evidence

- **Control UI transcript writes raw user input:** `chat.send` emits the transcript update with `buildChatSendTranscriptMessage({ message: parsedMessage, ... })`, and that helper stores `content: params.message` for the user turn. That means the persisted/history-visible text is the raw typed message, not runtime-injected prompt text.
