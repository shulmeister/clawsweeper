## Summary

Keep #8892 open. Current main still lacks a native `openclaw tui --agent <id>` startup selector; it only supports full agent-prefixed session keys, CWD workspace inference, and in-TUI `/agent` switching. The duplicate/similar #19247 is closed, while this issue remains the canonical open request, and the latest maintainer comment also explicitly kept it open.

## Best Possible Solution

Keep the issue open until OpenClaw adds a native TUI startup agent selector: register `--agent <id>` in `src/cli/tui-cli.ts`, add an `agentId`/`agent` field to `TuiOptions`, validate the configured agent, define precedence against full `--session agent:<id>:...` and CWD workspace inference, and update `docs.openclaw.ai/cli/tui`, `docs.openclaw.ai/web/tui`, plus focused CLI/TUI routing tests.
