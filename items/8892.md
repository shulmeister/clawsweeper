## Summary

Keep #8892 open. Current main still does not implement a native `openclaw tui --agent <id>` startup selector. The TUI can infer an agent from the current workspace, accept a full `agent:<id>:<session>` session key, and switch agents after launch with `/agent <id>`, but those do not satisfy the requested generic-directory startup routing flag.

## Best Possible Solution

Keep #8892 open as the canonical user-facing request. The best implementation is a native `--agent <id>` selector for `openclaw tui`: register and validate the flag in `src/cli/tui-cli.ts`, extend `TuiOptions`/`runTui` with a typed startup agent id, define precedence against full `--session agent:<id>:...` and cwd workspace inference, then update focused TUI tests plus `docs/cli/tui.md` and `docs/web/tui.md`.
