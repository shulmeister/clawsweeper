## Summary

Keep #8892 open. Current main still does not provide a native `openclaw tui --agent <id>` startup selector. The TUI can infer an agent from CWD, accept a full `agent:<id>:<session>` key, or switch after launch with `/agent`, but those do not satisfy the requested CLI flag. The similar #19247 is closed, and the latest maintainer comment in the provided issue context explicitly kept this issue open as the canonical request.

## Best Possible Solution

Keep the issue open until OpenClaw adds a native TUI startup agent selector. The likely implementation path is to register `--agent <id>` in `src/cli/tui-cli.ts`, add an `agentId` or `agent` field to `TuiOptions`, validate it against configured agents, define precedence against full `--session agent:<id>:...` and CWD workspace inference, then update `docs/cli/tui.md`, `docs/web/tui.md`, and focused CLI/TUI routing tests.
