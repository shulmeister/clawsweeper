## Summary

Keep open. Current main still does not implement a native `openclaw tui --agent <id>` startup selector. The TUI can infer an agent from CWD, accept full `agent:<id>:<session>` keys, and switch inside the running TUI with `/agent`, but the requested CLI flag and the reported `--session agent:ollama` shorthand remain unsupported.

## Best Possible Solution

Keep the issue open until OpenClaw adds a native TUI startup agent selector: register `--agent <id>` in `src/cli/tui-cli.ts`, carry it through `TuiOptions`/`runTui`, validate the configured agent, define precedence against full `--session agent:<id>:...` and CWD auto-discovery, then update `docs/cli/tui.md`, `docs/web/tui.md`, and focused CLI/TUI routing tests.
