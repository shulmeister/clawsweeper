## Summary

Keep #8892 open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still lacks a native `openclaw tui --agent <id>` startup selector; the shipped alternatives are explicit full `agent:<id>:<session>` keys through `--session`, CWD workspace inference, and post-launch `/agent`, which do not satisfy the generic-directory startup use case.

## Best Possible Solution

Keep the issue open until OpenClaw adds a native TUI startup selector. A likely implementation is to add `--agent <id>` in `src/cli/tui-cli.ts`, add a typed `agentId` or `agent` field to `TuiOptions`, validate it against configured agents, define precedence versus `--session agent:<id>:...` and CWD workspace inference, and update focused TUI routing tests plus `docs/cli/tui.md` and `docs/web/tui.md`.
