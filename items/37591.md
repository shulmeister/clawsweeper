## Summary

Keep #37591 open. Current main still has the reported contract mismatch for the native macOS companion node: agent `exec host=node` unconditionally calls `system.run.prepare`, the gateway rejects node commands that were not declared by the node, and the Swift macOS node still advertises and dispatches `system.run` without defining or handling `system.run.prepare`. The shipped #46843-era node-service startup fix improves the CLI node-host path, but it does not make the native macOS app node itself satisfy the prepare/run contract.

## Best Possible Solution

Keep #37591 open as the canonical native macOS companion-node bug until maintainers ship a reviewed fix. The likely product path is either to add a secure `system.run.prepare` implementation to the Swift macOS node, including advertised command support and regression coverage, or to land a carefully reviewed gateway/agent compatibility fallback that preserves the approval-plan security contract for already-deployed nodes that only advertise `system.run`.
