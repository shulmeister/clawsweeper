## Summary

Current main still has the reported macOS companion-app node contract mismatch. The gateway `exec host=node` path invokes `system.run.prepare` before `system.run`, gateway command policy rejects commands not advertised by the node, and the macOS companion app still advertises/dispatches `system.run` without defining, advertising, or handling `system.run.prepare`. The headless node host does implement prepare, so this remains a macOS companion-node gap. PR #43330 was a direct attempted fix but is closed unmerged, while PR #69725 is an open fallback direction rather than a shipped fix.

## Best Possible Solution

Keep #37591 open as the canonical tracker until maintainers either implement and ship secure `system.run.prepare` support in the macOS companion-app node or land a reviewed fallback such as #69725. A direct macOS fix should add the shared Swift command/types, advertise `system.run.prepare` from `MacNodeModeCoordinator`, dispatch it in `MacNodeRuntime`, return the canonical `systemRunPlan` shape expected by the gateway, and add focused Swift plus agent/gateway regression coverage.
