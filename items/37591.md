## Summary

Keep #37591 open. Current main still has the reported macOS companion-app node contract mismatch: the gateway `exec host=node` path unconditionally invokes `system.run.prepare`, but the macOS Swift node command enum, advertised command list, and invoke dispatcher still expose `system.run` without `system.run.prepare`. The discussion and related PR context support this: PR #43330 attempted the direct macOS prepare implementation but was closed unmerged, and the newer #49031-style reports are duplicate symptoms rather than a clear canonical replacement for the original tracker.

## Best Possible Solution

Keep #37591 open as the primary tracker until maintainers either implement secure `system.run.prepare` support in the macOS companion-app node or explicitly choose a gateway-side fallback. The likely code path is to add a Swift command enum case, advertise it from `MacNodeModeCoordinator`, implement a `MacNodeRuntime` prepare handler that returns the same canonical `systemRunPlan` contract used by the headless node host, and add macOS runtime tests for valid/invalid prepare payloads plus docs updates for macOS node commands. PR #43330 is useful prior art, but any revival should address the security/TOCTOU concern using the current approval-plan hardening model.
