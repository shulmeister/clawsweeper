## Summary

Keep open. Current main has a partial legacy image cleanup path, but it still does not implement the issue's current-turn-only vision scope. Hook prepended context is merged into the prompt before image detection, `detectImageReferences()` scans that whole prompt without excluding `<relevant-memories>` or active-memory `Untrusted context`, and the history pruning contract intentionally preserves the three most recent completed turns, including media markers and image blocks.

## Best Possible Solution

Implement the current-turn vision boundary in the embedded runner: when the current turn has explicit media, hydrate only current-turn attachments and current-turn trailing offloaded refs unless the user explicitly asks to compare earlier images. Strip or ignore `<relevant-memories>` and active-memory `Untrusted context` blocks before prompt image-reference detection, and add regression tests covering memory-lancedb, active-memory, and recent-history image turns in a Telegram-style session.
