## Summary

Keep open. Current `main` still does not implement Slack App Home handling: the Slack manifest/setup text enables the App Home messages tab, but the plugin has no `app_home_opened` event registration, no `views.publish` call, and no App Home tests or changelog entry showing this feature landed.

## Evidence

- **Manifest enables App Home only:** The Slack manifest builder declares `features.app_home.messages_tab_enabled: true`, and setup text tells users to enable the Messages tab, but this file contains no App Home rendering or publish logic.
  - file: [extensions/slack/src/shared.ts:38](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/extensions/slack/src/shared.ts#L38)
  - command: `nl -ba extensions/slack/src/shared.ts | sed -n '1,140p'`
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Slack monitor registers no App Home handler:** `registerSlackMonitorEvents` wires only message, reaction, member, channel, pin, and interaction handlers. There is no registration for `app_home_opened` or any home-specific module.
  - file: [extensions/slack/src/monitor/events.ts:3](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/extensions/slack/src/monitor/events.ts#L3)
