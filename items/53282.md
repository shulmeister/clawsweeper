## Summary

Current main already covers the requested setup behavior well enough: interactive onboarding installs the gateway service by default in QuickStart, manual onboarding prompts to install it with the default set to yes, doctor can prompt to install the service when the gateway is unhealthy and no service is installed, and the docs now prominently direct users to `openclaw onboard --install-daemon` or `openclaw gateway install`.

## Close Comment

Closing as implemented on current `main` after Codex review. The setup path now installs the managed gateway service during onboarding: QuickStart enables it automatically and manual onboarding prompts `Install Gateway service (recommended)` with the default set to yes in `src/wizard/setup.finalize.ts:105`. If setup runs without the service, the wizard now prints explicit follow-up commands in `src/wizard/setup.finalize.ts:282`, including `openclaw gateway run` and `openclaw onboard --install-daemon`. The doctor repair flow also offers to install the service when it detects that the gateway is unhealthy and no managed service is installed in `src/commands/doctor-gateway-daemon-flow.ts:163`. The docs also now point users to the managed install path in `docs/platforms/index.md:45`, `docs/cli/onboard.md:116`, and `docs/platforms/macos.md:27`. Because the requested setup/install behavior is already present on `main`, this issue no longer needs to stay open.
