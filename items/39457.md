## Summary

Keep #39457 open. Current main still does not provide a post-update lifecycle script/hook that runs after a successful update and before gateway restart, and the loaded-service update path still refreshes the gateway service by reinstalling from generated install-plan data rather than preserving custom LaunchAgent ProgramArguments wrappers.

## Best Possible Solution

Keep the issue open and implement a core updater-owned lifecycle hook or equivalent durable service customization mechanism. The practical fix should run after a successful update and after any generated service refresh that can rewrite the plist, but before the restart script or daemon restart, with tests covering a loaded macOS LaunchAgent whose custom ProgramArguments wrapper is restored before relaunch.
