## Summary

Keep #39457 open. Current main still refreshes a loaded gateway service with `gateway install --force` and then immediately runs the restart path, with no executable user lifecycle hook between the LaunchAgent plist rewrite and restart. Existing plugin/internal hook surfaces cover plugin install, Gateway startup/stop, sessions, messages, and tools, but not the updater-owned post-refresh/pre-restart slot requested here.

## Best Possible Solution

Keep the issue open. The best maintainer path is to add a core updater-owned post-update hook or an equivalent durable service customization mechanism after `refreshGatewayServiceEnv` succeeds and before `runRestartScript`/`runDaemonRestart` in `src/cli/update-cli/update-command.ts`, with focused tests for the loaded macOS LaunchAgent update path and user-facing docs for the supported contract.
