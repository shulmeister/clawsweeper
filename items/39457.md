## Summary

Keep #39457 open. Current main still refreshes an installed gateway service by forcing `gateway install`, then immediately restarts the service; there is no updater-owned post-update hook or durable LaunchAgent `ProgramArguments` customization point in the required slot. The existing hook/plugin surfaces cover gateway startup, install scanning, and plugin/hook-pack updates, but not a core update lifecycle hook that can repair the plist after refresh and before relaunch.

## Best Possible Solution

Keep the issue open. The best product/code direction is a core updater-owned lifecycle hook or an equivalent durable service customization mechanism, likely in `src/cli/update-cli/update-command.ts` after `refreshGatewayServiceEnv` succeeds and before `runRestartScript` or `runDaemonRestart`. Add focused tests for the loaded macOS LaunchAgent update path proving hook-applied plist/wrapper changes are in place before relaunch, and document the supported hook or configuration surface.
