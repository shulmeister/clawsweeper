## Summary

Current main still delegates `openclaw docs` to the live docs MCP/Mintlify search path, and the public search endpoint still returns HTTP 500 for both reporter queries. The docs index and pages contain relevant Browser/Skills content, so the no-results/wrong-results behavior is not obsolete. The linked PR #39632 was closed unmerged and only proposed better missing-mcporter messaging, not a fix for the live search regression.

## Best Possible Solution

Keep #39587 open. The best fix is to restore the live docs search endpoint or move `openclaw docs` to a stable searchable docs index/API, then add focused coverage or a smoke check for successful Browser/Skills queries and useful error reporting when the backend fails.
