## Summary

Current main still leaves #39587 unresolved. `openclaw docs <query>` still delegates to the live `https://docs.openclaw.ai/mcp.SearchOpenClaw` path through `mcporter`, and the public Mintlify search endpoint still returns HTTP 500 for both reporter queries (`browser extension` and `Skills`). The relevant Browser and Skills docs exist locally and on the public docs site, so this remains a real docs search regression rather than missing content. Related PR #39632 was closed unmerged and addressed missing-`mcporter` messaging, not the failing live search backend.

## Best Possible Solution

Keep #39587 open. The best fix is to restore the live docs search backend or move `openclaw docs` to a stable searchable docs index/API, then add focused coverage or a smoke check for successful Browser/Skills queries and useful error reporting when the backend fails.
