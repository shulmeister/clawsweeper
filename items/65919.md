## Summary

Keep #65919 open. Current main has pieces of the lifecycle cleanup, but it still does not implement the requested active-run presence lease for long quiet phases like compaction/memory flush through run completion plus dispatch idle. The linked PR #66365 is the active candidate fix and is not merged in the provided context.

## Best Possible Solution

Keep #65919 open until #66365 or an equivalent maintainer-approved implementation lands. The best path is an internal, rate-limit-aware active-run presence lease that refreshes already-enabled typing/reaction/preview surfaces during known long quiet phases, releases only after run completion/failure/abort plus dispatch idle, preserves stale-presence cleanup caps, and keeps compaction notices/final-only delivery semantics unchanged.
