## Summary

Keep open. Current main still restricts memory-flush write access to one exact daily memory path, and there is no implemented `SESSION_HANDOFF.md` or generic additional-write-path plumbing. The linked fix PR #66765 directly addressed this issue but is closed and unmerged in the provided GitHub context, and its proposed `additionalWritePaths` shape is absent from current main.

## Best Possible Solution

Keep the issue open and fix it in the memory-flush contract: extend `MemoryFlushPlan` with validated additional write targets, thread them through `runMemoryFlushIfNeeded`, `RunEmbeddedPiAgentParams`, and `createOpenClawCodingTools`, then allow `SESSION_HANDOFF.md` as a full overwrite target while keeping the daily memory file append-only. Add focused tests for custom prompts/config that permit `SESSION_HANDOFF.md` and still reject unrelated paths or path escapes.

## Risks / Open Questions

- The issue asks for allowing any path explicitly mentioned in a configured prompt; implementing that by parsing free-form prompt text would broaden the write sandbox and needs a constrained design, likely explicit resolved-plan paths or validated config rather than arbitrary text extraction.
- The bundled default prompt on current main only targets `memory/YYYY-MM-DD.md`, so the remaining bug is specifically for custom/recommended prompts that add `SESSION_HANDOFF.md`, not the current built-in default prompt alone.
