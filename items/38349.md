## Summary

Keep #38349 open. Current main still does not implement LLM-driven contextual group activation: the activation contract remains `mention | always`, config/session/gateway schemas do not accept `contextualActivation`, and WhatsApp/Telegram still drop unmentioned group messages before any model-driven peeking/engaged state machine can decide to run. The related implementation PR #38350 is closed unmerged in the provided GitHub context, and local history does not show an equivalent shipped implementation. This is also not a clean ClawHub close today because the bundled channel pre-dispatch gating path does not expose a sufficient plugin seam to replace skipped-message activation decisions externally.

## Best Possible Solution

Keep #38349 open as the canonical product/design issue. The best path is either to implement a first-class `contextual` activation contract across config schemas, sessions, gateway protocol, WhatsApp and Telegram pre-dispatch gating, observability, CLI/docs, and tests, or first add a typed plugin/channel activation seam that lets a ClawHub plugin make the peeking/engaged decision before bundled channel runtimes drop unmentioned group messages. PR #38350 and #1721 remain useful prior design context, but neither is shipped on current main.
