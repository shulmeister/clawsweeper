## Summary

Keep #38349 open. Current main still only supports group activation modes `mention` and `always`; there is no `contextualActivation` config/session/protocol shape, no peeking/engaged state machine, no decision-model call path, and bundled WhatsApp/Telegram gating still skips unmentioned group messages before an LLM-driven activation decision could run. The prior implementation PR #38350 is useful context but is closed unmerged, and #1721 is an older superseded request rather than a shipped fix. This is not a clean ClawHub close today because the current plugin/channel surface exposes require-mention resolution and observational hooks, not a pre-dispatch activation hook that can override bundled channel skips.

## Best Possible Solution

Keep #38349 open as the canonical product/API issue. The best path is either to implement a first-class `contextual` activation contract across config schemas, session persistence, Gateway protocol, WhatsApp and Telegram pre-dispatch gating, observability/CLI/docs/tests, or first add a typed plugin/channel activation seam that lets a ClawHub plugin make the peeking/engaged decision before bundled channel runtimes drop unmentioned group messages. PR #38350 and #1721 remain useful design context, but neither is shipped on current main.
