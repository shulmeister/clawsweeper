## Summary

Current main still does not implement contextual group activation. The shipped activation contract remains `mention`/`always`, WhatsApp and Telegram runtime paths only gate on those modes, docs and tests enforce those values, and PR #38350 was closed unmerged rather than shipped.

## Best Possible Solution

Keep #38349 open for maintainer product/design judgment. A complete solution should either add an explicit `contextual` activation contract across config, sessions, gateway protocol, WhatsApp and Telegram gating, observability, docs, and tests, or first introduce a typed plugin/channel activation seam that lets this behavior live outside core without patching bundled channel internals. PR #38350 can be used as prior implementation discussion, but it is closed unmerged and should not be treated as shipped.
