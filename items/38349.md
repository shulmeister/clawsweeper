## Summary

Keep open. Current main still only supports `mention` and `always` group activation, and the requested contextual peeking/engaged state machine is not implemented. The related PR #38350 is closed unmerged per the provided GitHub context, so it does not supersede this issue as shipped work.

## Best Possible Solution

Keep tracking the issue until a maintainer chooses the product direction. The best implementation path is to add a conservative channel activation contract or plugin SDK hook that can make async LLM activation decisions before mention gating for allowed group messages, then wire Telegram/WhatsApp with schema, docs, tests, observability, and safe defaults. If maintainers want this outside core, expose the missing plugin seam first instead of closing it as ClawHub now.
