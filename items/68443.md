## Summary

Keep #68443 open. Current main does not implement the proposed Feishu chat-history `list` message action: the Feishu channel advertises `read`, pins, member info, and channel list/info actions, but no generic `list`; the shared message-action registry also has no `list` action. Existing Feishu history code only fetches topic-thread history internally with `container_id_type="thread"`, which does not satisfy the PR's chat/group `container_id_type="chat"` request. The PR is still potentially useful, but the provided review discussion identifies a real bug in the PR head where `threadId` can be passed as a chat container id, so it needs changes rather than cleanup closure.

## Best Possible Solution

Keep #68443 open for maintainer/author follow-up. The best path is to revise the PR so `list` accepts only real chat ids for `container_id_type="chat"` or routes thread ids to the existing thread-list path, add focused tests for pagination/limit/result mapping and action dispatch, and coordinate with #41306 if OpenClaw wants both the message action surface and the `feishu_chat` tool surface.
