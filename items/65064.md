## Summary

Current `main` already scopes provider normalization/runtime loading to provider plugin IDs, keeps provider discovery manifest-first, and has loader guards/tests that prevent unrelated plugins from importing or re-running `register()` on the provider path. The shipped `v2026.4.23` release commit already contains this implementation.

## Evidence

- **Catalog-hook normalization is explicitly scoped:** `resolveProviderPluginsForCatalogHooks()` now derives `onlyPluginIds` from `resolveCatalogHookProviderPluginIds(...)` and passes that scope into `resolveProviderPluginsForHooks(...)`, instead of broad plugin loading.
  - file: [src/plugins/provider-runtime.ts:107](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/plugins/provider-runtime.ts#L107)

  - file: [src/plugins/providers.runtime.ts:76](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/plugins/providers.runtime.ts#L76)
