## Summary

Keep #38781 open. Current main still has the macOS Swift node `system.run.prepare` contract gap: the macOS app advertises and dispatches `system.run`, `system.which`, `system.notify`, and exec approval commands, but not `system.run.prepare`, while the agent `host=node` exec path still calls `system.run.prepare` before `system.run`. The related #69725 fallback PR is open/unmerged and is an alternate fix path, not a shipped superseding implementation.

## Best Possible Solution

Keep #38781 open until maintainers either merge a direct Swift macOS `system.run.prepare` implementation or explicitly choose and merge an alternate fallback such as #69725. The direct fix should add the command enum/types, advertise it from the macOS node, dispatch it in `MacNodeRuntime`, return the plan shape expected by `executeNodeHostCommand`, and cover advertisement, dispatch, shell-wrapper/rawCommand, env, and path-canonicalization behavior with focused tests.
