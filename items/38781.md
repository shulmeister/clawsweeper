## Summary

Keep #38781 open. Current main at d54d2d6b9b8aa28fb222d895c0f01f43933f4409 still has the reported macOS node contract gap: the agent exec path invokes `system.run.prepare`, while the Swift macOS node does not define, advertise, or dispatch that command. The PR is active, author-authored, and addresses a real unmerged bug; related #69725 is an open alternate fallback, not a merged superseding fix.

## Best Possible Solution

Keep #38781 open until maintainers merge a direct macOS `system.run.prepare` implementation or intentionally supersede it with a merged fallback such as #69725. The strongest direct fix is to align the Swift macOS node with the TypeScript node-host contract: add shared prepare payload/plan types, advertise `system.run.prepare`, dispatch it in `MacNodeRuntime`, return the canonical approval plan shape expected by `executeNodeHostCommand`, and add focused macOS/agent tests for declared commands, prepare payload shape, shell-wrapper/rawCommand consistency, env behavior, and path hardening.
