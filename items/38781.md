## Summary

Keep #38781 open. Current main still has the reported macOS companion-node contract gap: Swift macOS node mode advertises and handles system.run/system.which/system.notify/exec approvals, but not system.run.prepare, while exec host=node still unconditionally calls system.run.prepare before system.run. The related #69725 is an open alternate fallback PR, not a merged superseding fix, and the PR discussion has positive user signal that locally merging this PR restored Mac-node exec from an Ubuntu gateway.

## Best Possible Solution

Keep #38781 open until maintainers either merge a direct Swift macOS system.run.prepare implementation or explicitly choose and merge a fallback such as #69725. The best direct fix is to align macOS node mode with the TypeScript node-host contract: add prepare command/types, advertise it, dispatch it in MacNodeRuntime, return the plan shape expected by executeNodeHostCommand, and cover command advertisement, dispatch, shell-wrapper/rawCommand, env, and path-canonicalization behavior with focused tests.
