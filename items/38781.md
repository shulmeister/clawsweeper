## Summary

Keep open. Current main at 04e08cea still has the reported macOS node contract mismatch: the Swift companion node advertises and handles system.run but not system.run.prepare, while the node exec path still invokes system.run.prepare before system.run. The related fallback PR #69725 is relevant, but it is open and not shipped, so it is not a strong superseding close reason.

## Best Possible Solution

Keep this PR open until maintainers either finish/review the direct macOS system.run.prepare implementation with tests, or land a canonical fallback/superseding PR such as #69725 and then close this as superseded with that link.
