## Summary

Keep #38781 open. Current main still has the reported macOS node contract gap: the Swift macOS node advertises and handles system.run, system.which, and system.notify, but not system.run.prepare, while the agent node exec path still unconditionally calls system.run.prepare before system.run. The related #69725 is an open alternate fallback PR, not a merged superseding fix, and #71939 is a closed duplicate/rebased copy of this same work. The discussion also contains positive user signal: umatz reported that locally merging this PR restored Mac node use from an Ubuntu gateway, and the author has actively rebased/fixed lint.

## Best Possible Solution

Keep #38781 open until maintainers either merge a direct macOS system.run.prepare implementation or explicitly choose and merge a fallback path such as #69725. The best direct fix is to align the Swift macOS node with the TypeScript node-host contract: add shared prepare params/plan types, advertise system.run.prepare, dispatch it in MacNodeRuntime, return the approval plan shape expected by executeNodeHostCommand, and verify shell-wrapper/rawCommand, env, and path canonicalization behavior.
