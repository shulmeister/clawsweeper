## Summary

Keep open. Current main has partial interruption protection via prompt-level abort racing and abort-signal propagation into tools, but the specific tool execution design request is not fully implemented: `process poll` still ignores the signal, `sessions_send` does not receive or pass a signal, and the gateway request layer exposes timeout-only cancellation. The related contributor PRs treat this as future design work, not as an implemented duplicate.

## Best Possible Solution

Keep this issue open as a design/runtime follow-up. A good maintainer path would define the intended tool-interruption contract first, then add focused implementation and tests: a tool-level safety net or documented prompt-level equivalent, abort-aware `process poll` that stops waiting without killing the background process, a cancellable gateway call path for `sessions_send` if desired, and explicit tests for old-run cleanup, partial-result/transcript behavior, and background tool continuation.
