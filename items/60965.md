## Summary

Keep open. Current main at 02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54 still has the reported mismatch: the sessions_spawn schema exposes streamTo in the unified parameter object, while execute returns an error for streamTo when runtime is subagent. Open PR #68397 appears to target this exact behavior, but it is still open/unmerged, so the issue is not implemented on main yet.
