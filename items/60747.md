## Summary

Current main no longer has the observable stale-repair-request blocker described by #60747: normal already-paired reconnects do not create a repair request, access upgrades remain approval-gated, and stale pending device requests are pruned/managed on the current pairing path.

## Evidence

- **Normal paired reconnect path bypasses repair creation:** The WebSocket handler loads the paired device, verifies the public key, then checks approved roles and scopes before falling through to metadata update/token issuance. `requestDevicePairing` is only called inside `requirePairing`, which is reached for not-paired, metadata, role, or scope upgrade paths, not for an already-paired reconnect with approved access.
  - file: [src/gateway/server/ws-connection/message-handler.ts:970](https://github.com/openclaw/openclaw/blob/b40df76c186ee51f3f590e949d928f46059d2ffa/src/gateway/server/ws-connection/message-handler.ts#L970)
  - command: `nl -ba src/gateway/server/ws-connection/message-handler.ts | sed -n '900,1198p'`
  - sha: [b40df76c186e](https://github.com/openclaw/openclaw/commit/b40df76c186ee51f3f590e949d928f46059d2ffa)
- **Role and scope upgrades remain approval-gated:** For paired devices, current main compares the requested role against effective paired roles and requested scopes against approved scopes; mismatches call `requirePairing` and return before RPC access proceeds.
  - file: [src/gateway/server/ws-connection/message-handler.ts:1146](https://github.com/openclaw/openclaw/blob/b40df76c186ee51f3f590e949d928f46059d2ffa/src/gateway/server/ws-connection/message-handler.ts#L1146)

- **Token issuance is bounded to approved pairing state:** `ensureDeviceToken` refuses to issue a token unless the requested role is in the approved paired-device roles and the requested scopes fit the approved device baseline, preventing the unsafe broad `already-paired` shortcut discussed on the related PR.
  - file: [src/infra/device-pairing.ts:884](https://github.com/openclaw/openclaw/blob/b40df76c186ee51f3f590e949d928f46059d2ffa/src/infra/device-pairing.ts#L884)
