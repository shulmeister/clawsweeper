## Summary

Keep #46315 open. Current main still intentionally keeps WebChat replies on the internal/WebChat surface and documents deterministic same-channel routing, so the requested Feishu <-> WebChat bidirectional reply/mirroring behavior remains a real product/routing feature request rather than an implemented cleanup item.
