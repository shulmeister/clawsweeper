## Summary

Keep open. Current `main` does not surface the requested compact run-progress model in Control UI chat. The UI shows queued messages, BTW side results, fallback/compaction toasts, and tool cards, but it does not expose run phases like waiting on subagent/tool, finalizing reply, last activity, elapsed time, or reconnect-during-run state.

## Evidence

- **Control UI docs still describe only tool streaming, not run-progress UI:** The current Control UI docs list chat capabilities as streaming tool calls and live tool output cards, plus basic non-blocking chat send/abort behavior. They do not describe a compact run-state/progress strip, elapsed timer, last-activity indicator, or reconnect-during-run signal.
  - file: [docs/web/control-ui.md:105](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/web/control-ui.md#L105)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Backend has richer agent-event types than the chat UI consumes:** `AgentEventStream` includes `item`, `plan`, `approval`, `command_output`, `patch`, and lifecycle/tool streams, and item events carry `progressText`, timestamps, status, and summaries. That is the sort of structured progress data the issue asks to surface.
  - file: [src/infra/agent-events.ts:5](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/infra/agent-events.ts#L5)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Control UI agent-event handler drops most progress-oriented streams:** `handleAgentEvent` only handles `compaction`, lifecycle fallback/compaction cleanup, `fallback`, and `tool`. It returns early for every other agent-event stream, so `item`/`plan`/`approval`/`command_output` progress never reaches the chat UI.
  - file: [ui/src/ui/app-tool-stream.ts:450](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/ui/src/ui/app-tool-stream.ts#L450)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Gateway chat wiring does not add any separate run-progress model:** Gateway events route `agent` into the limited `handleAgentEvent` path and `chat.side_result` into a BTW-only side card. There is no additional current-run state object for active phase, waiting-on target, reconnect count, or reply-finalization state.
  - file: [ui/src/ui/app-gateway.ts:503](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/ui/src/ui/app-gateway.ts#L503)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Chat view has no compact run-status strip:** The chat view renders queued items, BTW side results, fallback indicator, compaction indicator, and context notice above the composer. There is no UI element here for run state, latest meaningful action, last activity, elapsed time, or websocket reconnect status.
  - file: [ui/src/ui/views/chat.ts:1134](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/ui/src/ui/views/chat.ts#L1134)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
