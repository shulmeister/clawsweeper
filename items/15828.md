## Summary

Keep #15828 open. Current main still implements bundled session-memory as a command-only hook for /new and /reset, while the requested idle, shutdown, periodic/message-count triggers and LLM summary output remain unimplemented. The same-day maintainer/Codex issue comment also explicitly kept this open and treated PRs #61675 and #71792 as narrower idle/daily candidates, not complete replacements.

## Best Possible Solution

Keep #15828 open unless maintainers intentionally split it. The implementation path is to extend bundled session-memory with explicit configurable idle, shutdown, command, and message-count/periodic triggers; define dedupe and summary-generation behavior; wire the selected lifecycle path through source, config schema, docs, and focused tests; and treat #61675/#71792 as narrower idle/daily candidates rather than complete replacements.
