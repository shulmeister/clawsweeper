## Summary

Keep open. Current main still implements bundled session-memory as a manual /new and /reset internal hook only. The requested idle, shutdown, periodic message-count checkpoints, and optional LLM summary output are not implemented, and the related trackers/PRs cover only part of the request and are not shipped on main.

## Best Possible Solution

Keep #15828 open until maintainers either implement configurable auto-save behavior for bundled session-memory or explicitly split/redirect the feature. A complete path should cover idle/daily rollovers, gateway shutdown semantics, periodic message-count checkpointing and dedupe, summary generation policy/costs, config/schema/docs updates, and focused tests. If maintainers want this plugin-owned instead, first create/link the canonical plugin/API design tracker and close this only after the bundled-hook product decision is explicit.
