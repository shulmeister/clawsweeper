## Summary

Keep open. Current main still implements bundled `session-memory` as a `/new` and `/reset` internal command hook only. The requested idle/shutdown/message-count auto checkpoints and optional LLM summary output are not implemented, and the related open trackers/PRs cover only part of the remaining work and are not shipped on main.

## Best Possible Solution

Keep #15828 open until maintainers either implement configurable auto-save behavior for the bundled `session-memory` hook or explicitly split/redirect the feature. A robust path would cover idle/daily rollovers, gateway shutdown semantics, periodic message-count dedupe, summary generation policy/costs, config/schema/docs updates, and focused tests. If maintainers prefer plugin-owned memory automation, first define the missing product contract and close this only after linking a canonical plugin/API tracker.
