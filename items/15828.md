## Summary

Keep open. Current main still implements the bundled session-memory hook as a `/new` and `/reset` command hook only. The requested idle, shutdown, periodic message-count checkpoints, and optional LLM summary output are not implemented, and the related context does not provide a canonical merged/shipped replacement.

## Best Possible Solution

Keep #15828 open until maintainers either implement configurable auto-save behavior for the bundled `session-memory` hook or explicitly redirect the feature to a plugin. A robust fix would add supported trigger configuration, wire idle/daily session-end and shutdown lifecycle paths safely, add periodic/message-count checkpointing with dedupe, decide how LLM summarization should run and be billed, then update `HOOK.md`, public docs, config/schema help if needed, and focused tests for each trigger.
