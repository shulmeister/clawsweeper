## Summary

Keep open. Current main still renders the Agents-page picker as a native `<select>` driven by `change`, with no source, test, changelog, or related-item evidence that the simulated-click browser automation behavior has been fixed. Related #57067 covers broader Agents-page dropdown friction, but it is not a precise canonical tracker for this automation-specific report.

## Best Possible Solution

Keep this issue open until current main is verified with a supported browser automation path. If pointer-based automation still cannot operate the Agents selector, replace it with an accessible automation-friendly combobox/listbox or visible agent list and add focused browser/UI coverage. If only untrusted `element.click()` is unsupported by the browser/tool, document the supported automation path, such as assigning the select value and dispatching `change`, and close with explicit maintainer judgment.
