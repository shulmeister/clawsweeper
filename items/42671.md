## Summary

WhatsApp group reply-to-bot triggering is already implemented on current main: quoted replies to bot messages satisfy mention gating without a prefix, and quoted reply metadata/body are passed into inbound context. The behavior is covered by tests, documented, and present in the inspected v2026.4.22 release source.

## Close Comment

Closing as implemented. This was a Codex review against current `main`, and the requested behavior is already present for WhatsApp groups: replying/quoting one of the bot's prior messages satisfies mention gating without needing a prefix in `extensions/whatsapp/src/auto-reply/monitor/group-gating.ts:163`, there is an explicit regression test for that in `extensions/whatsapp/src/auto-reply/web-auto-reply-monitor.test.ts:106`, and the quoted message context is forwarded via `ReplyToId` / `ReplyToBody` / `ReplyToSender` in `extensions/whatsapp/src/auto-reply/monitor/inbound-dispatch.ts:121` and documented in `docs/channels/whatsapp.md:200` and `docs/channels/whatsapp.md:237`. I also checked the shipped `v2026.4.22` release source (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), which contains the same implementation. Given that the requested no-prefix reply-to-bot flow already exists, there is nothing left to track on this issue.
