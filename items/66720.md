## Summary

Keep #66720 open. Current main has a partial adjacent fix: `sessions_spawn` hides ACP-only affordances when ACP is unavailable, and docs/tests cover that behavior. The central PR/request is still not implemented on main: there is no `simplifiedSchemaForGateway` option or tool-specific property stripping for strict OpenAI-compatible gateways, and the nested `attachments` / `attachAs` subagent schema remains advertised. The linked #66719 remains the canonical bug, but this PR is a plausible focused fix that needs rework and maintainer review rather than cleanup closure.

## Best Possible Solution

Keep the linked #66719 problem open and rework this PR or replace it with a core implementation that lets provider/model compatibility request a simplified `sessions_spawn` schema from the real runtime tool assembly path. The implementation should preserve current ACP availability hiding, strip only the intended gateway-incompatible properties when explicitly activated, and add regression tests for the strict-gateway schema shape and activation path.
