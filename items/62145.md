## Summary

Keep open. Current main still routes any disconnected Control UI state through the full login gate, so the reported workflow problem remains: prior dashboard/chat content is not preserved in a disabled/read-only state with a reconnect affordance.

- **Disconnect forces disconnected state:** The Control UI gateway close handler sets `host.connected = false` for the active client before recording the disconnect error.
