## Summary

Keep open. Current `main` still lacks the Telegram-side `isSilentReplyText` preflight guard that the PR adds, while Slack already has the parallel suppression path. The existing Telegram dispatch logic/tests only show cleanup of a stale preview after a `NO_REPLY` turn, which is consistent with the bug report rather than proof of a fix. I found no evidence that this was already implemented on `main` or shipped in `v2026.4.23`.
