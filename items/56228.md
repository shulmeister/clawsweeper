## Summary

Keep open. Current `main` still appears to have the reported gap: reply-context text for a referenced Discord message is derived through `resolveDiscordMessageText()`, and that helper still only falls back through `content`, media placeholders, `embeds[0]`, and optional fallback text. There is no component-tree extraction path for components-v2 messages, so a referenced v2-only message can still produce an empty reply body and be dropped from reply context.

## Evidence

- **Message text resolver still ignores component payload text:** `resolveDiscordMessageText()` builds `rawText` from `message.content`, attachment/sticker placeholders, `embedText`, and `fallbackText`, then returns that result (plus forwarded text when requested). There is no `message.components` or v2-flag handling in this path.
  - file: [extensions/discord/src/monitor/message-utils.ts:650](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/discord/src/monitor/message-utils.ts#L650)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Reply context is dropped when extracted referenced text is empty:** `resolveReplyContext()` calls `resolveDiscordMessageText(referenced, { includeForwarded: true })` and returns `null` when that string is empty, which matches the reported symptom of missing `ReplyToBody` for referenced messages whose visible text is not in `content` or embeds.
  - file: [extensions/discord/src/monitor/reply-context.ts:17](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/discord/src/monitor/reply-context.ts#L17)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Referenced-message hydration is already present:** `mergeFetchedDiscordMessage()` copies `fetched.referenced_message` into `referencedMessage`, so the remaining gap is downstream text extraction rather than failure to hydrate the referenced Discord payload.
  - file: [extensions/discord/src/monitor/message-handler.preflight.ts:282](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/discord/src/monitor/message-handler.preflight.ts#L282)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Current tests cover embed/sticker fallbacks but not components-v2 extraction:** The `resolveDiscordMessageText()` tests exercise forwarded messages, mention resolution, sticker placeholders, and embed fallback behavior. There is no test here for components-v2 text extraction from a referenced message.
  - file: [extensions/discord/src/monitor/message-utils.test.ts:894](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/discord/src/monitor/message-utils.test.ts#L894)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
