## Summary

Keep open. Current `main` still handles long contexts with in-place auto-compaction and retry, plus manual/daily/idle session resets. I found no built-in auto session split or `session.autoSplit`/`carryOver` configuration on `main`, so the request is not implemented and is still actionable as a feature request.

## Evidence

- **Auto-compaction is documented as in-place compaction/retry, not session rollover:** The compaction docs say OpenClaw compacts when a session nears the context limit or hits overflow, keeps the full conversation on disk, and suggests `/new` only when a clean slate is needed.
