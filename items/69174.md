## Summary

FriendliAI is not implemented on current main, but this PR is an optional third-party model-provider integration that already fits the existing external provider-plugin path. The PR and linked issue do not identify a missing generic OpenClaw core or plugin-SDK seam; the best maintainer outcome is to route it to ClawHub/community plugin publishing rather than keep a bundled-core provider PR open.

## Best Possible Solution

Close this core PR as ClawHub/provider-plugin scope. The FriendliAI author or community should publish an external FriendliAI provider plugin through ClawHub or npm using the existing provider SDK, manifest auth/setup metadata, catalog hooks, onboarding preset helpers, and dynamic model resolution; only open a new OpenClaw core PR if that work exposes a missing generic plugin-SDK seam or maintainers explicitly choose to promote FriendliAI as an official bundled provider.
