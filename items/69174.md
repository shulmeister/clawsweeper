## Summary

FriendliAI is not implemented on current main, but the PR is a provider-specific optional integration that already fits the existing external provider-plugin model. The current SDK/docs cover API-key auth, provider catalogs, onboarding metadata, dynamic model resolution, and ClawHub/npm publishing, and the PR/linked issue do not identify a missing core seam that would require bundling this in OpenClaw core.

## Best Possible Solution

Close this core PR as ClawHub/provider-plugin scope. The best path is for the author or FriendliAI community to publish an external FriendliAI provider plugin using the existing provider SDK, manifest auth/setup metadata, and ClawHub/npm install flow; only open a core PR if that work exposes a missing generic plugin SDK seam.
