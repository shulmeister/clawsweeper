## Summary

Current `main` already contains two relevant duplicate-suppression paths for webchat/control-ui replies: transcript-level delivery-mirror dedupe and streaming/block-reply dedupe for providers that resend full content on `text_end`. Those same code paths are already present in the latest shipped release `v2026.4.23`.

## Evidence

- **Delivery-mirror transcript dedupe on append:** `appendExactAssistantMessageToSessionTranscript` checks for a redundant delivery-mirror and reuses the latest matching assistant message instead of appending a duplicate mirror entry.
  - file: [src/config/sessions/transcript.ts:204](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/config/sessions/transcript.ts#L204)
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Latest-message-only matching logic:** `findLatestEquivalentAssistantMessageId` only suppresses the mirror append when the latest assistant message text matches, which is the guard added for duplicate visible replies across turns.
  - file: [src/config/sessions/transcript.ts:286](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/config/sessions/transcript.ts#L286)
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Regression test for duplicate delivery mirrors:** Test covers the exact case where a delivery mirror should not be appended when the latest assistant message already matches.
  - file: [src/config/sessions/transcript.test.ts:149](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/config/sessions/transcript.test.ts#L149)
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Provider full-content resend guard in embedded webchat stream handling:** `resolveAssistantTextChunk` explicitly handles providers that resend full content on `text_end`, appending only the unseen suffix to keep output monotonic.
  - file: [src/agents/pi-embedded-subscribe.handlers.messages.ts:139](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/agents/pi-embedded-subscribe.handlers.messages.ts#L139)
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Message-end duplicate block-reply guard:** For `text_end` channels, `handleMessageEnd` skips the message_end safety send when content was already delivered, preventing duplicate visible replies from mixed `text_end`/`message_end` delivery.
