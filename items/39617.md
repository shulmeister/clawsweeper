## Summary

Keep PR #39617 open. Current main still has the unresolved Slack and Discord portions of the reported bug: both native/slash routing paths use the provider startup config object, while `resolveAgentRoute` derives DM session keys from `cfg.session.dmScope`. Telegram has since moved to a fresh runtime config pattern, so the PR is partially superseded but not obsolete.

## Best Possible Solution

Keep PR #39617 or the linked #39605 bug active. The maintainer path is to rebase the remaining fix onto `extensions/discord/src/monitor/native-command.ts` and `extensions/slack/src/monitor/slash.ts`, load the current runtime config only for native/slash route resolution and configured binding readiness, preserve Telegram's existing fresh-runtime-config behavior, and add focused Slack/Discord regression tests for `session.dmScope` changes without provider restart.
