## Summary

Keep PR #39617 open. Current main still does not fully implement the requested fix: Slack slash commands and Discord native commands continue to resolve routes with provider-startup config snapshots, while `resolveAgentRoute` still derives DM session keys from `session.dmScope`. Telegram is already on a fresh-runtime-config path with regression coverage, so the PR is partially superseded and needs a rebase onto the current `extensions/*` plugin paths rather than closure.

## Best Possible Solution

Keep PR #39617 open, or keep #39605 as the canonical bug if maintainers prefer a fresh replacement PR. Rebase the remaining fix onto `extensions/discord/src/monitor/native-command.ts`, `extensions/discord/src/monitor/native-command-route.ts`, and `extensions/slack/src/monitor/slash.ts`; load current runtime config for native/slash route resolution and configured binding readiness; preserve Telegram's existing fresh-runtime-config behavior; and add focused Slack/Discord regression tests that simulate startup config followed by changed `session.dmScope` or bindings without provider restart.
