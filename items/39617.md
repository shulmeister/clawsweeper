## Summary

Keep open. Current main still has the PR's reported stale-config routing behavior for Slack slash commands and Discord native slash commands. Telegram has since implemented the fresh-config pattern with a regression test, so this PR is partially superseded, but the Slack and Discord fixes remain useful work rather than something that should be closed.

## Best Possible Solution

Keep this PR or its linked bug active. The best current path is to rebase the fix onto the `extensions/discord` and `extensions/slack` code paths, reload current config only for slash/native route resolution, and add Slack/Discord regression tests proving `session.dmScope` or route bindings are re-read between invocations without restarting the provider. Leave the already-fixed Telegram path intact.
