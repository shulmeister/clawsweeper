## Summary

Keep open. Current main has partial CLI JSONL event plumbing, mainly Claude stream-json-style deltas, but it still does not implement the requested configurable CLI backend incremental streaming mode, capability metadata, or diagnostics, and the channel reply path still treats CLI runs as final-text-oriented. Related items in the provided context do not supersede this: #33957 is about ACP attach/recover and #38255 is Telegram preview tuning, not backend-level CLI incremental streaming.

## Best Possible Solution

Keep the issue open and implement a first-class CLI backend streaming contract: add an opt-in config such as `agents.defaults.cliBackends.<id>.streamingMode`, typed plugin capability metadata, parser/dialect hooks for backend-owned incremental events, channel preview forwarding through the existing `onPartialReply`/draft preview path with debounce controls, and diagnostics that distinguish no backend incremental output from output buffered or dropped before channel dispatch.

The only JSONL dialect currently typed is `claude-stream-json`.
