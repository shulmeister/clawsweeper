## Summary

Current main does not contain PR #66749's requested guard. The exec runtime still calls `requestHeartbeatNow(scopedHeartbeatWakeOptions(..., { reason: "exec-event" }))` unconditionally after queuing exec notifications, while the session utilities and heartbeat runner still identify subagent session keys and fall them back to the main session. The linked issue #66748 remains a plausible focused bug report, and the PR is a narrow fix rather than cleanup-close material.

## Best Possible Solution

Keep PR #66749 open for maintainer review. The best path is to preserve exec system-event enqueueing for subagent sessions, suppress only the redundant `exec-event` heartbeat wake at the two exec-runtime call sites, and land the added regression test or an equivalent test that proves subagent exec completion does not wake the parent heartbeat while normal agent/global wake behavior remains unchanged.
