## Summary

Current main still does not implement the requested Dreaming tab agent selector. The Dreaming view has no selected-agent props or selection callback, the Dreams header only renders refresh/on-off controls, and the controller/backend calls remain default-agent scoped. Related #63829 and #68959 overlap with multi-agent memory/UI concerns, but neither supersedes this Dreaming-specific UI and `doctor.memory.*` scoping work.

## Best Possible Solution

Keep this issue open. Add Dreams/Dreaming selected-agent state and a selector backed by `agents.list`, pass the selected `agentId` through `loadDreamingStatus`, `loadDreamDiary`, and Dream Diary actions, update `doctor.memory.*` handlers to accept an optional `agentId` with default-agent fallback, and add focused UI/controller/gateway tests. Keep the memory-wiki vault isolation portion linked to #63829 rather than treating it as resolved here.
