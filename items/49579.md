## Summary

Keep open. Current `main` does not already implement this change, and the proposed blanket `WhatsApp gateway disconnected...` filter still appears unsafe because that banner carries critical close-state information into prompt assembly.

## Evidence

- **Prompt assembly only consumes queued system events:** `get-reply-run.ts` prepends the result of `drainFormattedSystemEvents(...)` into the reply prompt body. There is no alternate path here that injects WhatsApp `runtime.error(...)` messages into prompt assembly.
  - file: [src/auto-reply/reply/get-reply-run.ts:468](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/auto-reply/reply/get-reply-run.ts#L468)
  - command: `nl -ba src/auto-reply/reply/get-reply-run.ts | sed -n '430,500p'`
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Current compaction filter does not already suppress WhatsApp banners:** On current `main`, `compactSystemEvent` filters heartbeat noise and scrubs `Node:` suffixes, but has no WhatsApp-specific connected/disconnected suppression. So this PR is not already implemented on `main`.
  - file: [src/auto-reply/reply/session-system-events.ts:22](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/auto-reply/reply/session-system-events.ts#L22)

- **WhatsApp monitor enqueues a disconnect banner for every close:** The monitor queues `WhatsApp gateway disconnected (status ${decision.normalized.statusLabel})` immediately after resolving a close decision, before branching into logged-out/conflict/max-attempt handling.
  - file: [extensions/whatsapp/src/auto-reply/monitor.ts:445](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/extensions/whatsapp/src/auto-reply/monitor.ts#L445)

Since the queued disconnect banner includes the normalized status label for all of these outcomes, suppressing every `WhatsApp gateway disconnected...` line would also hide non-routine outages from prompt-visible system events.
