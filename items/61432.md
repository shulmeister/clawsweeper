## Summary

Keep open. The current message:received path still creates the internal hook event inline, fires it through fireAndForgetHook, and never retains or inspects the event.messages array after handlers run. The public hook docs and internal type contract still say hooks can push messages to send to the user, and the existing reset-command path shows the expected consumption pattern. The linked PR #61657 positively identifies the same root cause and proposed a regression test, but it was closed unmerged, so it does not supersede or fix this issue on main.
