## Summary

Current main already covers the reported Windows status/health stall: `status` and `health` route before full Commander/plugin startup, their startup policy disables plugin preloading, and one-shot gateway RPC clients now wait for bounded WebSocket teardown instead of leaving the CLI process alive until an external kill.

## Best Possible Solution

Close this issue as implemented on current main and keep any remaining general CLI startup/plugin-discovery performance work consolidated under the broader existing tracking issue (#67040).
