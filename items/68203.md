## Summary

Current `main` already fixes the RFC 2544 / fake-IP VPN compatibility problem from #68192 through an operator-controlled `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange` path with targeted test coverage. This PR proposes a different `stealthMode` tool argument, and its latest patch still leaves `src/agents/tools/web-fetch.ts` syntactically unterminated.

## Evidence

- **Current main passes operator config into web_fetch runtime:** `createWebFetchTool()` reads configured fetch options and forwards `fetch?.ssrfPolicy` into `runWebFetch`, so the SSRF exception is controlled by config rather than tool-call arguments.
