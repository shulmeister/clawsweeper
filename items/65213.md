## Summary

Keep this PR open. Current main still lacks `openclaw sessions explain <key>` or an equivalent focused stored-session diagnostic surface. The existing sessions command lists/cleans stores, while `sandbox explain` only covers sandbox/tool policy. The PR is a plausible implementation candidate, but the review discussion shows it still needs maintainer review and likely rework around canonical session/store/model/tool resolution before merge.

## Best Possible Solution

Keep this PR open as a candidate implementation. The best path is maintainer review or author follow-up: rebase onto current main, make `sessions explain` use the canonical target/store/model/tool helpers consistently, cover the reviewed edge cases in `src/commands/sessions.test.ts` and CLI registration tests, then run the normal changed gate before merge.
