## Summary

Keep #40205 open. Current main still uses Ollama's deprecated `/api/embeddings` request contract in the active bundled Ollama memory embedding provider, and tests still assert the old endpoint and `prompt`/`embedding` shape. The PR is stale because the implementation moved to `extensions/ollama`, but the requested fix remains valid.

## Best Possible Solution

Keep this PR open and ask for a rebase against `extensions/ollama/src/embedding-provider.ts` and `extensions/ollama/src/embedding-provider.test.ts`: use `POST /api/embed`, send `{ model, input }`, parse `embeddings[0]`, update tests away from `prompt` and `embedding`, and preserve serialized local Ollama batch behavior before running the relevant extension changed checks.
