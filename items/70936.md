## Summary

Keep open. The PR is aimed at a real PDF.js standard-font warning, but its patch targets the old core PDF extractor shape. Current main routes PDF fallback extraction through the bundled document-extract plugin, and that active plugin-owned PDF.js call still omits standardFontDataUrl, so the underlying bug is not implemented on main and the PR should be retargeted rather than closed.

## Best Possible Solution

Keep the item open, but retarget the fix to extensions/document-extract/document-extractor.ts: resolve pdfjs-dist/package.json from the plugin package root, pass a trailing-slash standardFontDataUrl into getDocument, update the local PdfJsModule type, and add a focused regression test in extensions/document-extract/document-extractor.test.ts. That plugin-owned patch should be the canonical fix for #51455.
