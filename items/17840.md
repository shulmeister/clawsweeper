## Summary

Keep #17840 open. Current main still only records inbound reactions as passive system events, and there is no opt-in `reactionTriggersTurn`, `awaitReaction`, or equivalent channel config that wakes an agent turn. The issue discussion is actionable and includes concrete Telegram, Slack, and Discord workflows plus a recent maintainer comment explicitly keeping this item open.

## Best Possible Solution

Keep tracking this issue as the canonical product request. The likely implementation path is an opt-in channel config such as `reactionTriggersTurn` with scopes like `off`, `own`, `all`, and possibly allowlist/per-channel overrides, wired after existing channel authorization/routing so accepted reaction system events can request a heartbeat/agent turn while preserving current passive defaults.
