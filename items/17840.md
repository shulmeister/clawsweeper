## Summary

Keep open. Current main still treats inbound reactions as passive system events, lacks an opt-in `reactionTriggersTurn`/`awaitReaction` style config, and the issue discussion includes concrete Telegram, Slack, and Discord workflows plus a recent maintainer comment explicitly keeping the item open.

## Best Possible Solution

Keep tracking this issue. The best implementation path is an opt-in reaction-turn setting, likely in channel config and docs, wired through bundled channel reaction handlers after authorization/routing so reactions can enqueue context and then request a turn only for configured scopes such as `own`, while preserving today's passive default.
