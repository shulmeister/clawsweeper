## Summary

Keep #17840 open. Current `main` still has inbound reaction notifications as passive system events, with no opt-in `reactionTriggersTurn`, `awaitReaction`, or equivalent wake-on-reaction config/API. The issue has concrete Slack, Telegram, and Discord workflows, plus a recent maintainer comment explicitly keeping it open.

## Best Possible Solution

Keep #17840 open as the canonical product request unless maintainers explicitly select another canonical issue. A likely implementation is an opt-in channel/plugin config such as `reactionTriggersTurn: "off" | "own" | "all" | "allowlist"`, wired after existing reaction authorization/routing so accepted reaction system events can request an agent turn without changing default passive behavior.
