## Summary

Keep PR #38789 open. Current main still reuses a saved skills snapshot prompt before considering live sandbox workspace entries, and resolved skill snapshots still skip live entry loading. The sandbox code does mirror eligible skills into non-rw sandbox workspaces, but the embedded attempt and compaction paths still pass the original snapshot through unchanged, so the PR’s central sandbox prompt-path problem is not implemented on main. The PR should not be landed as-is because its review history flags unresolved correctness gaps around empty sandbox skill sets, saved skill filters, remote eligibility, and env alignment.

## Best Possible Solution

Keep PR #38789 open or supersede it with a focused agents fix that rebuilds or remaps skills prompts for non-rw sandbox workspaces while preserving the saved session skill filter, remote eligibility context, and skill env/runtime contracts. Add regression coverage for snapshot-present sandbox runs, empty sandbox skill sets, filtered skill snapshots, and attempt/compaction parity.
