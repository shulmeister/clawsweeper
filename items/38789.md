## Summary

Keep PR #38789 open. Current main still has the sandbox skill prompt path problem the PR targets: non-rw sandbox runs switch to the sandbox workspace and mirror skills there, but the embedded attempt and compaction paths still pass the original skills snapshot through unchanged. Since `resolveSkillsPromptForRun` returns the snapshot prompt before considering live entries, and `resolveEmbeddedRunSkillEntries` skips live loading when `resolvedSkills` exists, current main does not implement this PR’s behavior. The PR remains useful, but its review comments identify correctness gaps that should be addressed before landing or superseding it.

## Best Possible Solution

Keep this PR open or replace it with a focused agents fix that rebuilds or remaps skills prompts for non-rw sandbox workspaces using the same sandbox-aware skill set for prompt construction, snapshot skill filtering, remote eligibility, and env/runtime setup. Add regression coverage for snapshot-present sandbox runs, empty sandbox skill sets, filtered snapshots, and attempt/compaction parity.
