## Summary

Keep PR #38789 open. Current main still does not implement the PR’s sandbox-specific prompt rebuild: embedded non-rw sandbox runs can receive an existing host-built skills snapshot prompt, while live entries from the sandbox workspace are skipped when the snapshot already has resolved skills. The PR addresses a plausible core agents bug, but its own review history still has unresolved correctness concerns around empty sandbox skill sets, preserved skillFilter, and env alignment.

## Best Possible Solution

Keep the PR open or supersede it with a focused core agents fix that rebuilds or remaps the skills prompt for non-rw sandbox workspaces while preserving the saved session skillFilter, remote eligibility context, and env/runtime contracts. Add regression coverage for snapshot-present sandbox runs, empty sandbox skill sets, filtered skill snapshots, and compaction/attempt parity.
