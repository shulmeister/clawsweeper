## Summary

Keep PR #38789 open. Current main still has the snapshot-first sandbox skill prompt behavior the PR is trying to fix: non-rw sandbox runs switch to the sandbox workspace, but resolved skill snapshots skip live sandbox skill loading and `resolveSkillsPromptForRun` returns the persisted snapshot prompt before considering sandbox-local entries. The PR is still relevant, though it should be refined before landing because review comments identify gaps around empty sandbox skill sets, preserving `skillsSnapshot.skillFilter`, remote eligibility, and env override alignment.

## Best Possible Solution

Keep this PR open or replace it with a focused agents fix that rebuilds non-rw sandbox skill prompts from the same sandbox-aware, filter-preserving skill set used for runtime setup in both attempt and compaction paths. The final implementation should avoid returning host snapshot prompts for empty sandbox skill sets, preserve remote eligibility and saved skill filters, keep env overrides aligned with the advertised skill set, and add regression tests for snapshot-present non-rw sandbox attempt and compaction behavior.

## Risks / Open Questions

- Landing PR #38789 as-is could re-expose filtered-out skills in sandboxed runs unless the saved `skillsSnapshot.skillFilter` is applied to rebuilt live entries.
- Landing PR #38789 as-is could still surface stale host paths if a non-rw sandbox has no live skill entries and the code falls back to the snapshot prompt.
- Prompt contents and skill env overrides can diverge if the sandbox path advertises live entries while env setup still comes from the persisted snapshot.
