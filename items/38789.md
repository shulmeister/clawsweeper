## Summary

Keep open. Current main still does not implement this PR’s sandbox-specific prompt rebuild: embedded attempt and compact runs still pass the saved skills snapshot through unchanged, and prompt resolution still returns `skillsSnapshot.prompt` before considering live entries from the effective sandbox workspace. The newer sandbox skill-sync path is useful, but it does not by itself rewrite an already-built host-workspace snapshot prompt. The PR also still has unresolved review concerns around empty sandbox skill sets, preserved skill filters, and env override alignment.

## Best Possible Solution

Keep this open unless a maintainer explicitly picks another canonical sandbox-skill-path fix. The best implementation is a shared sandbox-aware helper used by both attempt and compact flows when `workspaceAccess !== "rw"`: force-load entries from the effective sandbox workspace, rebuild the prompt from those entries, preserve `skillsSnapshot.skillFilter` and remote eligibility, return an empty prompt when the sandbox has no visible entries, and align env overrides with the same advertised skill set. Add focused regression tests for the helper, attempt/compact wiring, empty sandbox entries, filter preservation, and env override behavior.
