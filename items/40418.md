## Summary

Keep open. Current main has partial `/new`/`/reset` session-memory preservation and startup hydration, but it does not implement the requested synthesis/anchor workflow: the bundled hook still writes the last N raw user/assistant messages to dated slug files, startup context reads recent memory files back as bounded untrusted notes, and related synthesis work remains open/unmerged in PR #50584. The linked raw-memory poisoning bug #69943 also means this should not be closed as a harmless completed feature.

## Best Possible Solution

Keep #40418 open until maintainers either land a scoped implementation or explicitly redirect the feature to a ClawHub/context-engine plugin. A strong implementation would coordinate with #50584 and #69943: opt-in synthesis, sanitizer/filtering before persistence, canonical daily summary append or startup package, receipt links to full transcripts, timezone-aware filenames, and regression tests for `/new`/`/reset` hydration.
