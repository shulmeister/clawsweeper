## Summary

Keep open. Current main has moved Google thinking sanitization into the plugin SDK shared helper and added native Google transport handling, but the PR's remaining sanitizer behavior is not implemented: the wrapper still sanitizes from `model.id`, not `model.reasoning`, and a negative `thinkingBudget` for a reasoning-capable non-Gemini-3 model still only has the budget deleted without a replacement `thinkingLevel`. The PR needs a rebase and policy review, but it is still plausible useful work rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer review, but require a rebase onto current main. The best path is a current-code patch in `src/plugin-sdk/provider-stream-shared.ts` and the Google wrapper that uses resolved model capability only where the Google API policy supports it, plus a targeted regression test for a reasoning-capable Gemini 2.5 wrapper payload with invalid negative budget and a non-reasoning negative test.
