## Summary

Keep this PR open. Current main still has the gap the PR targets: web chat per-message metadata reads only top-level message.cost, while OpenClaw docs and usage aggregation treat message.usage.cost as the normalized transcript shape. The PR is not ready to close as implemented or superseded; it should be rebased onto the current sanitizer location and covered by focused gateway/UI tests.

## Best Possible Solution

Keep the PR open and ask for a small rebase/update: port the sanitizer change to src/gateway/chat-display-projection.ts, make grouped-render read m.cost ?? m.usage.cost, define the savingsPct aggregation behavior, and add focused tests for nested usage.cost preservation plus inline cost/savings rendering. The ClawRouter example in the PR body is useful prior art, but the core gap is generic webchat handling of OpenClaw’s existing usage.cost transcript metadata.
