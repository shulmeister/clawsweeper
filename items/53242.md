## Summary

Keep open. Current `main` still truncates `chat.history` text fields before the byte-budget checks, and the default cap is now `8_000` chars, which is even lower than the `12_000` chars described in the report.

## Evidence

- **Current default cap is still a hard character limit:** `DEFAULT_CHAT_HISTORY_TEXT_MAX_CHARS` is `8_000`, while the per-message byte cap remains `128 * 1024` bytes.
  - file: [src/gateway/server-methods/chat.ts:157](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/server-methods/chat.ts#L157)

- **Truncation happens by character count with explicit marker:** `truncateChatHistoryText()` slices text at `maxChars` and appends `\n...(truncated)...`. `sanitizeChatHistoryContentBlock()` applies that to `text`, `content`, `partialJson`, `arguments`, and `thinking`.
  - file: [src/gateway/server-methods/chat.ts:835](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/server-methods/chat.ts#L835)
