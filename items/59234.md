## Summary

Keep open. Current `main` still has the exact OpenClaw-side behavior the issue asks to change: QMD vector availability is probed via `qmd status` unconditionally, and boot sync still forces `qmd embed` even when `memory.qmd.searchMode` is `"search"`. I did not find evidence on `main` that search-mode startup now skips the vector probe or degrades to BM25-only when vector readiness fails.

## Evidence
