## Summary

Keep open. The current QMD manager still starts boot maintenance for QMD and schedules embed maintenance independently of `memory.qmd.searchMode`, while the public docs still describe `qmd update` + `qmd embed` as managed boot/periodic behavior. The issue is recent, has multiple independent confirmations, and the requested OpenClaw-side BM25-only degradation/skip behavior is not clearly implemented on main.

## Evidence

- **Boot QMD maintenance still starts from full manager initialization:** `QmdMemoryManager.initialize` calls `this.runUpdate("boot", true)` when `this.qmd.update.onBoot` is enabled, with no visible `searchMode === "search"` guard in that boot path.

  - sha: [7bbd47349ec3](https://github.com/openclaw/openclaw/commit/7bbd47349ec37f01a8f2b1dc219bf950615f361b)
