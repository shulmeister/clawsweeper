## Summary

Keep open. Current main still routes macOS Talk playback through direct ElevenLabs, local MLX, or local system voice; gateway `talk.speak` is only retried after direct ElevenLabs playback fails. Configured providers such as `openai` are parsed from `talk.config` but still land on the unsupported/system-voice path, so the reported macOS routing gap remains.
