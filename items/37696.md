## Summary

Keep #37696 open. Current main still has the reported strict-mode behavior: unclosed reasoning tags leave the parser inside a thinking block, so strict mode drops the trailing assistant text. The checked-in tests still assert that empty/drop behavior, and shared delivery/channel paths still use strict stripping.

## Best Possible Solution

Keep the issue open until a focused fix lands. The likely path is to update `stripReasoningTagsFromText` so malformed/unclosed reasoning tags cannot silently erase all deliverable assistant text, then update the strict-mode unit tests and affected delivery/channel expectations. PR #51915 is the current open candidate to review or supersede; the earlier linked PRs are useful prior attempts but are closed and unmerged.
