## Summary

Keep #72249 open. Current main still archives the planned state/config/workspace paths with a single `tar.c(...)` call and has no built-in volatile-file filter, EOF-class retry loop, enriched final EOF error, or `skippedVolatileCount` output. The linked #72251 PR is the active unmerged fix for this exact issue, so the issue should remain open until that or an equivalent fix lands.

## Best Possible Solution

Keep #72249 open and let #72251 or an equivalent core backup change land. The best fix belongs in `src/infra/backup-create.ts`/backup planning: skip known volatile state paths before archiving, retry only EOF-class tar races with temp-archive cleanup and bounded backoff, enrich final errors with the offending path and attempt count, and add tests plus JSON/stdout observability for skipped volatile files.
