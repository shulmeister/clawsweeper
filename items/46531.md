## Summary

Keep #46531 open. Current main has useful safeguards for gateway RPC config writes and documentation for daemon-visible env, but the central reported failure mode is still plausible: an enabled `models.providers.<id>.apiKey` SecretRef remains an active startup secret, startup still fails closed when it cannot resolve, and the direct `openclaw config set` path that a model-assisted shell workflow may use does not show the same daemon-environment preflight before writing.

## Evidence

- **Issue discussion supports real setup/runtime mismatch:** The existing discussion explicitly identifies this as a product gap: setup can validate/store env-backed refs against an interactive shell while daemon startup later resolves against the service environment. Comment: https://github.com/openclaw/openclaw/issues/46531#issuecomment-4061423264
