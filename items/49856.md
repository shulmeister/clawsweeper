## Summary

Current `main` already implements the reported web UI model-picker fix through refactored chat-model selection code, plus matching gateway fallback logic and regression tests. The same implementation is present in the shipped `v2026.4.22` release, so this PR is redundant on top of current `main`.

## Evidence

- **Dropdown now resolves from the active session, not the global default:** `resolveChatModelOverrideValue` reads the active session row and only falls back to the local override cache; it no longer derives the picker value from global defaults.
  - file: [ui/src/ui/chat-model-select-state.ts:34](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/ui/src/ui/chat-model-select-state.ts#L34)

  - file: [ui/src/ui/chat-model-ref.ts:106](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/ui/src/ui/chat-model-ref.ts#L106)
