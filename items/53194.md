## Summary

Issue #53194 proposes an opt-in, scheduled agent workflow for observing friction and submitting/voting on feature ideas. On current `main`, that is better scoped as a ClawHub skill/plugin workflow, not OpenClaw core. The repo vision explicitly says optional capability should usually ship as plugins and that new skills should go to ClawHub first, while existing docs show core already provides the extension and scheduling surfaces such a workflow would need.

## Close Comment

Codex review: closing this as `clawhub` rather than a core OpenClaw item. The current project vision says optional capability should usually ship as plugins (`VISION.md:54`) and that new skills should be published to ClawHub first, not added to core by default (`VISION.md:66`). The existing product already exposes the building blocks this proposal needs through the plugin/skill system and ClawHub install path (`docs/tools/plugin.md:11`, `docs/tools/plugin.md:48`) plus built-in cron and skill management surfaces (`docs/web/control-ui.md:118`, `docs/web/control-ui.md:119`). Given that, a scheduled “observe friction -> submit idea -> vote/refine” workflow is better treated as a ClawHub skill/plugin or adjacent service than as OpenClaw core behavior.
