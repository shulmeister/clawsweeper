## Summary

Keep open. Current main still lets the exec tool call's model-supplied `security` argument participate in `minSecurity(configuredSecurity, requestedSecurity)`, so the behavior this PR targets is still present and the PR is not obsolete.

## Best Possible Solution

Keep this PR open and review it as a focused bug fix. The best landing shape is a minimal change in `src/agents/bash-tools.exec.ts` that makes configured/session exec security authoritative for model tool calls, preserves the explicit elevated-full override, adds regression coverage for configured `full` with model `allowlist` and configured `allowlist` with model `deny`/`full`, and either removes or documents the now-ignored model-call `security` parameter.

## Evidence
