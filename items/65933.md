## Summary

Keep open. Current main still lets the exec tool call's model-supplied `security` argument participate in `minSecurity(configuredSecurity, requestedSecurity)`, so the PR is not obsolete or implemented on main.

## Best Possible Solution

Keep this PR open and review it as a focused bug fix. The best implementation is a minimal change in `src/agents/bash-tools.exec.ts` that treats configured/session exec security as authoritative for model tool calls, preserves the explicit elevated full override, and includes regression tests for configured `full` with model `allowlist`, configured `allowlist` with model `deny` or `full`, and elevated full mode. Also align or document the now-ignored model-call `security` parameter before landing.
