## Summary

Keep #38721 open. Current main still has the gateway shutdown watchdog and adjacent bounded cleanup for websockets/MCP/HTTP, but I did not find a shutdown contract that enumerates, cancels, or bounded-drains active ProcessSupervisor child/PTY runs. The maintainer discussion explicitly says #61565 fixed a different websocket shutdown blocker and should not supersede this child-process-handle report.

## Best Possible Solution

Keep #38721 as the canonical tracker for gateway shutdown blocked by active child-process or pipe handles. The best fix is to add an explicit shutdown contract for supervisor-managed exec and CLI child processes: stop accepting new process work, expose active-run enumeration plus cancel-all or bounded drain APIs on ProcessSupervisor, call that path from gateway shutdown before the watchdog expires, and add regression coverage proving active child handles cannot trip the shutdown watchdog. Treat #61565 and newer websocket/MCP/systemd hardening as adjacent mitigations, not closure evidence for this report.
