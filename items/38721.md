## Summary

Keep #38721 open. Current main has several adjacent shutdown improvements, including the #61565 websocket close bound, bounded bundled MCP runtime disposal, process-tree killing when managed runs are explicitly cancelled, and lane reset after restart. I did not find high-confidence evidence that gateway shutdown now cancels or detaches every active exec/process child handle before the 25s shutdown watchdog, and a maintainer comment explicitly separated #61565 from this child-process/pipe-handle report.

## Best Possible Solution

Keep the issue open and treat it as the canonical tracker for gateway shutdown blocked by active child-process/pipe handles. The best fix would add an explicit shutdown-phase cancellation/drain policy for supervisor-managed exec/process sessions and any other gateway-owned child-process surfaces, with a bounded grace window and regression coverage that proves active child handles do not outlive gateway close or trip the shutdown watchdog. The existing #61565 websocket fix and MCP disposal improvements should remain, but they are adjacent mitigations rather than proof this report is resolved.
