## Summary

Keep open. Current `main` still has the shutdown-timeout path, and the gateway shutdown code does not appear to cancel or drain active managed child-process runs/background exec sessions before closing sockets and HTTP servers. The only clearly shipped related fix on `main` is the websocket close bound from #61565, which the issue already called out as a different root cause.
