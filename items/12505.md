## Summary

Keep #12505 open. Current main has useful opt-in tool sandboxing through Docker/SSH/OpenShell backends and Docker deployment bootstrap, but it still does not implement the issue's central request: secure-by-default tiered sandbox presets plus plugin/skill process isolation, IPC-proxied SDK calls, and manifest-declared env/fs/net/resource capability enforcement. The existing issue discussion already called this out and treated merged PR #29974 as partial progress, not closure.

## Best Possible Solution

Keep #12505 open as the canonical tracking item unless maintainers split it into narrower issues. The best path is to design core support for secure-by-default sandbox presets, plugin/skill process isolation or a safer bundle-first execution path, manifest-declared env/fs/net/resource policy, credential scoping, and an IPC/proxy SDK boundary. Treat PR #29974 and the current Docker/SSH/OpenShell sandbox backend work as partial opt-in tool-sandbox progress, not closure.
