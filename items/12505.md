## Summary

Keep open. Current main has meaningful opt-in agent/tool sandboxing and Docker deployment support, including work related to #29974, but #12505 asks for a broader unified built-in sandbox: plugin/skill process isolation, tiered default presets, credential/env scoping, manifest-declared capability enforcement, IPC proxying, and multi-platform native isolation. Those core requests are still not implemented on main.

## Evidence

- **Current sandbox config is narrower than requested presets:** The agent sandbox config only exposes mode off/non-main/all, backend, workspaceAccess, sessionToolsVisibility, scope, and backend-specific settings. There is no sandbox_level or minimal/standard/strict/paranoid preset contract.
