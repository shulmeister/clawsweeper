## Summary

Keep open. Current main has meaningful optional agent tool sandboxing (Docker/SSH/OpenShell backends) and opt-in Docker deployment bootstrap, but it still does not implement #12505's central requests: unified built-in sandbox presets, secure-by-default sandbox level, plugin/skill process isolation, IPC-proxied plugin SDK calls, manifest-declared env/fs/net/resource capability enforcement, or cross-platform native sandboxing.

## Best Possible Solution

Keep #12505 open as the canonical tracking item unless maintainers split it into narrower sandbox architecture issues. The best path is to design and implement core support for secure-by-default sandbox presets, plugin/skill process isolation or a bundle-first safer execution path, manifest-declared capability policy for env/fs/net/resources, credential scoping, and an IPC/proxy SDK boundary; treat #29974 as useful opt-in Docker sandbox progress, not as closure.
