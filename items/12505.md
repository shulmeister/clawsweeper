## Summary

Keep #12505 open. Current main has useful opt-in tool sandbox backends and hardening, but it still does not implement the central request: secure-by-default tiered sandbox presets, plugin/skill process isolation, IPC-proxied SDK calls, manifest-declared env/fs/net/resource policies, or credential scoping. The existing issue discussion also explicitly treats PR #29974 as partial Docker sandbox progress, not closure for this broader architecture request.

## Best Possible Solution

Keep #12505 open as the canonical sandbox/isolation architecture tracker unless maintainers split it into focused follow-up issues. The best path is a core design for secure-by-default tiered sandbox presets, plugin/skill process isolation or an equivalent safer execution boundary, IPC/proxy SDK enforcement, manifest-declared env/fs/net/resource policies, credential scoping, and migration guidance that treats PR #29974 and current sandbox backends as partial opt-in tool-sandbox groundwork.
