## Summary

Keep #12505 open. Current main has useful opt-in sandbox backends for agent tool execution, including Docker, SSH, and OpenShell plus a plugin SDK seam for sandbox backends, but it still does not implement the issue’s central request: secure-by-default tiered sandbox presets, native plugin process isolation, IPC-proxied plugin SDK calls, or per-plugin manifest policies for env/fs/net/resources. The existing issue discussion also already treats PR #29974 as partial Docker sandbox progress rather than a full resolution.

## Best Possible Solution

Keep #12505 open as the canonical sandbox/isolation architecture tracker unless maintainers split it into narrower issues. The best product path is a core design for secure-by-default tiered presets, plugin/skill isolation or an equivalent stronger execution boundary, IPC/proxy SDK enforcement, manifest-declared env/fs/net/resource policies, credential scoping, resource controls, and migration guidance that treats current Docker/SSH/OpenShell tool sandbox backends as partial groundwork.
