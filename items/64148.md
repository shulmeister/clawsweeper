## Summary

Keep this PR open. Current main still creates backup archives from the full resolved state asset list without a tar filter, and current tests/docs do not show the proposed default exclusion for `extensions/*/node_modules`. The provided PR context is a plausible implementation candidate: it targets the right archive creation path, fixes the review-noted Windows separator and canonical state-path issues in later commits, and closes the narrower user report #64144, even though that issue is now closed.

## Best Possible Solution

Keep this PR open for maintainer review. Either land or recreate the narrow tar filter with the PR's Windows separator and canonical state-path fixes plus regression tests, or explicitly fold the same default `extensions/*/node_modules` behavior into the broader backup exclude-pattern work before closing this PR.
