## Summary

Keep open. Current main does not implement this PR's requested Anthropic Opus 4.6 xhigh advertisement: the shipped/provider profile deliberately exposes `adaptive` for Claude 4.6 and reserves `xhigh`/`max` for Claude Opus 4.7. The PR is also stale against the current provider-thinking contract because it edits deprecated `supportsXHighThinking`, but there is not a conservative auto-close reason: the transport still maps forced Opus 4.6 `xhigh` to `max`, so the feature idea remains plausible pending maintainer product/API direction.

## Best Possible Solution

Keep the PR open for maintainer judgment. If OpenClaw should advertise Opus 4.6 `xhigh` as `max`, rework this onto `resolveThinkingProfile`, update the Anthropic profile tests and provider docs, and confirm the UI/gateway thinking-level surfaces accept the new level. If maintainers intentionally want Claude 4.6 limited to `adaptive`, close manually as a product rejection rather than as an automated cleanup item.
