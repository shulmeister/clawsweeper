## Summary

Current main still does not implement the PR's requested Anthropic Opus 4.6 xhigh/max advertisement, and the PR's one-line hook is stale against the current provider-thinking API. The underlying request remains plausible because the Anthropic transport can map a forced Opus 4.6 xhigh request to max effort, so this needs maintainer product judgment or a reworked patch rather than an automated close.

## Best Possible Solution

Keep the PR open for maintainer judgment, but require a rework before merge. If OpenClaw should expose Opus 4.6 max effort, implement it in extensions/anthropic/register.runtime.ts through resolveThinkingProfile, update the Anthropic profile tests and thinking docs, and keep Sonnet 4.6 excluded unless transport/runtime behavior truly supports a distinct xhigh/max level. If the intended policy is Opus 4.7-only xhigh/max, a maintainer should close this PR with that rationale or link a canonical policy issue.
