## Summary

Keep PR #38729 open. Current main still does not advertise `xhigh`/`max` for Anthropic Claude Opus 4.6, so the requested capability is not implemented. The patch itself is stale because Anthropic now uses `resolveThinkingProfile`, and legacy `supportsXHighThinking` is bypassed once a non-empty provider profile is returned.

## Best Possible Solution

Keep the PR open, but require a rework against the current provider-profile path. If OpenClaw should expose Claude Opus 4.6 max effort, add the level through `extensions/anthropic/register.runtime.ts` `resolveThinkingProfile`, update `extensions/anthropic/index.test.ts`, and update `docs/tools/thinking.md`. Keep Sonnet 4.6 excluded unless the transport/runtime can provide distinct max/xhigh behavior. If the Opus 4.7-only boundary is intentional policy, a maintainer should close with that rationale or link a canonical policy issue.
