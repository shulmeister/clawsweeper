## Summary

Keep PR #38729 open. Current main still does not expose `xhigh`/`max` for Anthropic Claude Opus 4.6, and the PR's exact `supportsXHighThinking` hook is now stale against the active `resolveThinkingProfile` API. The underlying request remains plausible because the Anthropic transport can map a forced Opus 4.6 `xhigh` run to `output_config.effort: "max"`, but deciding whether to advertise that level needs maintainer product/API judgment.

## Best Possible Solution

Keep the PR open, but require a rework before merge. If OpenClaw should expose Claude Opus 4.6 max effort, implement it in `extensions/anthropic/register.runtime.ts` through `resolveThinkingProfile`, update the Anthropic provider tests and `docs/tools/thinking.md`, and keep Sonnet 4.6 excluded unless runtime behavior supports a distinct max/xhigh level. If Opus 4.7-only advertisement is intentional product policy, a maintainer should close the PR with that rationale or link a canonical policy issue.
