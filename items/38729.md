## Summary

Keep PR #38729 open. Current main still does not advertise `xhigh`/`max` for Anthropic Claude Opus 4.6: the Anthropic provider profile exposes those levels only for Opus 4.7, while tests and docs explicitly preserve the Opus 4.6 adaptive-only boundary. The PR discussion and latest diff narrowed the idea to Opus 4.6, and that request remains plausible because the transport can map an explicit Opus 4.6 `xhigh` request to Anthropic `max`; however, the submitted `supportsXHighThinking` hook is stale against current main because `resolveThinkingProfile` wins before legacy hooks are consulted.

## Best Possible Solution

Keep the PR open and ask for a rework against the current provider-profile path. If OpenClaw should expose Claude Opus 4.6 max effort, add `xhigh`/`max` through `extensions/anthropic/register.runtime.ts` `resolveThinkingProfile`, update `extensions/anthropic/index.test.ts`, and update `docs/tools/thinking.md` / public docs wording. Keep Sonnet 4.6 excluded unless runtime behavior can provide a distinct max/xhigh path. If Opus 4.7-only advertisement is intentional, a maintainer should close with that explicit product rationale or link a canonical policy item.
