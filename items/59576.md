## Summary

Keep open. The report is recent, has a concrete Mattermost WebSocket/file_ids race diagnosis, and a second user corroborated a similar attachment-size failure. Current main does not appear to implement the proposed REST post re-fetch before resolving inbound media, so this is not safe to close as implemented or unreproducible.
