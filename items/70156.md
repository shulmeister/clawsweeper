## Summary

Keep this PR open. Current main still lacks the proposed Swift .gitignore generalization, large-file CI guard, unjustified-exec CI guard, package scripts, and current boundary-runner wiring. The PR remains actionable repo hygiene, but it needs a rebase and fixes for the review-commented parser/allowlist gaps before it should become a required CI gate.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase it onto current main, wire the new checks through scripts/run-additional-boundary-checks.mjs, update allowlists for the current shell/exec surface, add regression coverage for the parser gaps noted in review, and then run the targeted script tests plus the relevant changed CI checks.
