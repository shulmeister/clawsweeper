## Summary

Keep #13676 open. Current main still lacks the requested first-class Project object, Projects dashboard workflow, project-to-home-session binding, project workspace list, and project-scoped tool/skill/runtime constraints. Existing agents, sessions, per-agent workspaces, skills, and sandbox policies are adjacent primitives, and the provided discussion keeps this issue as the canonical broader tracker after #12333 and #13535 were closed.

## Best Possible Solution

Continue tracking #13676 as the canonical core/product dashboard request. A complete implementation should add an explicit Project storage and Gateway protocol model, expose Projects in Control UI, bind each project to a home session plus optional workstreams, associate one or more workspace paths, and surface project-scoped tool/skill/runtime constraints while reusing existing agent, session, workspace, skill-policy, and sandbox primitives where they fit.
