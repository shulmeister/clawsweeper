## Summary

Keep #13676 open. Current main has adjacent agents, sessions, per-agent workspaces, skill/tool controls, and sandbox/runtime settings, but it still does not implement the requested first-class Project object, Projects dashboard workflow, home-session binding, project workspace list, or project-scoped constraints. The issue discussion also treats #13676 as the canonical tracker after #12333 and #13535 were closed as superseded.

## Best Possible Solution

Continue tracking #13676 as the canonical product/core dashboard request. A complete fix should add an explicit Project model and Gateway protocol/storage contract, surface Projects in Control UI, bind each project to a home session plus optional workstreams, associate one or more workspace paths, and expose project-scoped skill/tool/runtime constraints while reusing existing agents, sessions, sandbox, and skill policy primitives where appropriate.
