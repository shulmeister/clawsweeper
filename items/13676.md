## Summary

Keep #13676 open. Current main still lacks a first-class Project object and the requested Gateway protocol, Control UI navigation, session binding, workspace list, and project-scoped constraint surfaces. Existing agents, sessions, workspaces, skill allowlists, tool policy, and sandbox settings are useful adjacent primitives, but they do not satisfy the issue's durable Project model.

## Best Possible Solution

Continue tracking #13676 as the canonical core/product dashboard request. A complete solution should add explicit Project storage and Gateway protocol methods, expose a Projects route/list in Control UI, bind projects to a home session plus optional workstreams, associate one or more workspace paths, and surface project-scoped tool/skill/runtime constraints while reusing existing agent, session, workspace, tool-policy, and sandbox primitives where they fit.
