## Summary

Keep open. Current main still has adjacent agents, sessions, per-agent workspaces/skills, and sandbox/runtime controls, but it does not implement the requested first-class Project object or dashboard Projects workflow. The narrower related issues #12333 and #13535 are closed as superseded by this issue, so #13676 remains the canonical tracker.

## Best Possible Solution

Continue tracking #13676 as the canonical product/core dashboard request. A complete solution should add an explicit Project model and Gateway protocol/storage contract, expose a Projects list or tab in Control UI, bind projects to home/additional sessions and workspace paths, and surface project-scoped skill/tool/runtime constraints. Existing agents and sessions can be reused underneath, but they do not currently satisfy the user-facing Project abstraction.
