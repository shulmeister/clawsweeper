## Summary

Keep open. Current `main` does not contain the PR's claimed fix: `sessions.send` still fails missing sessions with `INVALID_REQUEST`, the protocol still lacks structured session/agent error codes, and there is no shipped `sessions.start` gateway method or `sessions_start` agent tool.

## Evidence

- **Missing-session send still errors generically:** `handleSessionSend` returns `ErrorCodes.INVALID_REQUEST` with `session not found: ${key}` when the target session has no `sessionId`, so the reported failure mode is still present on `main`.
  - file: [src/gateway/server-methods/sessions.ts:479](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/gateway/server-methods/sessions.ts#L479)
  - command: `nl -ba src/gateway/server-methods/sessions.ts | sed -n '470,500p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Protocol has no structured session error codes:** The gateway error-code schema on `main` still exports only `NOT_LINKED`, `NOT_PAIRED`, `AGENT_TIMEOUT`, `INVALID_REQUEST`, `APPROVAL_NOT_FOUND`, and `UNAVAILABLE`; there is no `SESSION_NOT_FOUND`, `AGENT_NOT_FOUND`, or `SESSION_AUTO_STARTED`.
  - file: [src/gateway/protocol/schema/error-codes.ts:3](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/gateway/protocol/schema/error-codes.ts#L3)

  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Session resolution still reports INVALID_REQUEST on misses:** `sessions.resolve` still maps missing key/sessionId/label lookups to `INVALID_REQUEST`, not a dedicated missing-session code, so structured callers still do not get the PR's proposed signal.
  - file: [src/gateway/sessions-resolve.ts:30](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/gateway/sessions-resolve.ts#L30)
  - command: `nl -ba src/gateway/sessions-resolve.ts | sed -n '20,210p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **No gateway sessions.start method on main:** The registered gateway method list includes `sessions.create` and `sessions.send` but no `sessions.start`, so the explicit pre-warm/start API from the PR is not shipped.
  - file: [src/gateway/server-methods-list.ts:87](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/gateway/server-methods-list.ts#L87)
