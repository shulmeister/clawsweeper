## Summary

Keep open. Current `main` already covers the JSON `api_error` / MiniMax `"unknown error, 520"` message path, but it still does not treat HTTP status `520` as a transient status in the explicit status-based classifiers. The PR's requested behavior is only partially implemented on `main`.
