## Summary

Keep #11623 open. Current main still has adjacent macOS menu-bar status, HoverHUD, WebChat, Canvas, Notify, and Talk overlay surfaces, but none implements the requested Clawi feature: multiple configurable floating per-agent desktop bubbles with persisted positions, mouse-follow/anchor behavior, and per-agent chat routing. The existing issue discussion already reached the same keep-open conclusion, and current main still matches that assessment.

## Best Possible Solution

Keep #11623 open. The best path is either a native macOS implementation under `apps/macos/Sources/OpenClaw` that derives bubble state from agent/session activity, persists positions, supports drag/anchor or mouse-follow behavior, and routes chat to the selected agent, or a maintainer-designed native macOS UI/window plugin API that would make an external ClawHub implementation feasible.
