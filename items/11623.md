## Summary

Keep open. Current main still has adjacent macOS surfaces (menu bar activity, Hover HUD, WebChat panel/window, Canvas panel, and a single Talk Mode orb overlay), but not the requested per-agent floating Clawi bubbles with configurable names, drag/reposition persistence, mouse-follow/anchor behavior, and single-click inline chat. The plugin/ClawHub path is also not currently sufficient because the documented plugin API has no native macOS SwiftUI/AppKit window/UI extension point.

## Best Possible Solution

Keep #11623 open. The best path is either a native macOS implementation under `apps/macos/Sources/OpenClaw/` that connects bubble state to agent/session activity, persisted positions, and chat routing, or an explicit maintainer decision to design a native macOS UI extension API before moving this kind of feature to plugins/ClawHub.
