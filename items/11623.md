## Summary

Keep open. Current `main` has macOS overlay infrastructure and activity/presence UI, but it does not implement the requested per-agent floating bubbles (`Clawi`) feature. The idea is still in scope for the macOS app, not a ClawHub/plugin-only request, and there is no strong evidence it has already landed.

## Evidence

- **macOS app scope stays in core:** The macOS app is documented as the menu-bar companion that owns native UI, notifications, permissions, and node capabilities, so this request fits OpenClaw core app scope rather than ClawHub.
  - file: [docs/platforms/macos.md:9](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/docs/platforms/macos.md#L9)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Existing floating orb is single talk overlay only:** `TalkOverlayController` manages one overlay window with a single model (`isVisible`, `phase`, `isPaused`, `level`) and one `NSPanel`; there is no per-agent collection, naming, chat entry, or multi-bubble behavior.
  - file: [apps/macos/Sources/OpenClaw/TalkOverlay.swift:17](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/apps/macos/Sources/OpenClaw/TalkOverlay.swift#L17)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Current agent activity collapses to one current item:** `WorkActivityStore` tracks multiple jobs/tools internally, but the exposed UI state is a single `current` activity plus one `iconState`, which does not match the requested 'each running agent gets a visible orb' model.
  - file: [apps/macos/Sources/OpenClaw/WorkActivityStore.swift:21](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/apps/macos/Sources/OpenClaw/WorkActivityStore.swift#L21)

- **Current macOS shell is single menu-bar item plus anchored HUD/chat:** The app entry point renders one `MenuBarExtra` critter and wires hover/click behavior to a status-item HUD and web chat panel, not floating per-agent desktop bubbles.
  - file: [apps/macos/Sources/OpenClaw/MenuBar.swift:42](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/apps/macos/Sources/OpenClaw/MenuBar.swift#L42)
