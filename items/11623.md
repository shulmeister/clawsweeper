## Summary

Keep open. Current main still does not implement per-agent floating macOS bubbles with names, drag/reposition, mouse-follow, and inline chat. The existing macOS surfaces are menu-bar status, menu/window WebChat, Canvas, and single voice/talk overlays, and the documented plugin API does not expose native macOS window/UI extension points that would make this a clean ClawHub/plugin close.

## Best Possible Solution

Keep #11623 open until maintainers either implement the macOS app feature or explicitly decide against it. A concrete implementation would likely belong under `apps/macos/Sources/OpenClaw/` as a native SwiftUI/AppKit bubble controller tied to agent/session state, persisted positions, and WebChat/session routing. If the desired direction is external plugins, first design and document a native macOS UI extension API rather than closing this as ClawHub-ready today.
