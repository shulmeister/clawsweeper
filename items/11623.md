## Summary

Keep #11623 open. Current main has adjacent macOS HUD, chat, talk, and activity surfaces, but it still does not implement multiple configurable floating Clawi agent bubbles with persisted positions, drag/anchor or mouse-follow behavior, or bubble-launched per-agent chat routing. Closing as ClawHub is also premature because the current plugin API documents provider/channel/tool/service style registrations, not a native macOS SwiftUI/AppKit window extension point.

## Best Possible Solution

Keep #11623 open. The best path is either a native macOS implementation under apps/macos/Sources/OpenClaw that connects bubble state to agent/session activity, persists positions, supports drag/anchor or mouse-follow behavior, and routes chat to the selected agent, or a maintainer-led native macOS UI/window plugin API design before moving this to ClawHub/plugins.
