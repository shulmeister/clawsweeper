## Summary

Keep #39497 open. Current main still maps sandbox file-tool paths only through `containerWorkdir`, while the read-only workspace mount is `/agent`; the read-tool wiring does not pass an `/agent` mapping or mount table into the workspace-root guard, and current regression coverage still expects `/agent/...` reads to be rejected under `tools.fs.workspaceOnly`.

## Best Possible Solution

Keep the issue open. The likely fix belongs in the sandboxed file-tool guard/wiring: make `wrapToolWorkspaceRootGuardWithOptions` mount-aware via the sandbox mount table, or pass an explicit `/agent` mapping only for `workspaceAccess: "ro"`, then add regression coverage proving `/agent/...` reads resolve while writes/edit/apply_patch remain disabled. Treat `/tmp` reads as a separate policy decision.
