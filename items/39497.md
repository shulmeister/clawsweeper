## Summary

Keep #39497 open. Current main still rejects the reported `/agent/...` read path in the relevant `workspaceAccess: "ro"` + `tools.fs.workspaceOnly` path: the read-tool guard only remaps `containerWorkdir` paths, the sandboxed read-tool wiring passes only `sandbox.containerWorkdir`, and current regression coverage still expects `/agent/secret.txt` to throw `Path escapes sandbox root`.

## Best Possible Solution

Keep the issue open. The best fix is in the sandboxed file-tool guard/wiring: make `wrapToolWorkspaceRootGuardWithOptions` mount-aware through the sandbox mount table, or pass an explicit `/agent` mapping only for `workspaceAccess: "ro"`; then add regression coverage proving `/agent/...` reads resolve while write/edit/apply_patch remain disabled. Treat `/tmp` reads as a separate policy decision.
