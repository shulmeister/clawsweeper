## Summary

Keep PR #39853 open. Current main still does not expose structured non-self Feishu @mentions in inbound conversation metadata, and the PR addresses a real bundled Feishu multi-bot routing gap. The submitted patch still needs revision before merge because it only preserves `open_id` mentions, has no focused regression coverage, and overlaps with broader open Feishu arbitration work in #38939 rather than being cleanly superseded by it.

## Best Possible Solution

Keep #39853 open. The best maintainer path is to revise it into a tested Feishu mention extraction path that excludes the receiving bot, preserves a stable identity across `open_id`/`user_id`/`union_id` where available, exposes a typed structured mention list in inbound conversation metadata, and reconciles that contract with the broader arbitration policy in #38939.
