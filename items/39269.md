## Summary

Keep PR #39269 open as a rework/split candidate. Current main has already covered or superseded several pieces, but at least two central bug fixes from the batch are still not fully present on main: shared outbound Discord Unknown Channel/10003 permanent-error handling, and a queued webchat subagent announce regression for stale external routes. The PR should not merge as-is because other pieces now conflict with current main's chat-history and plugin-duplicate policy.

## Best Possible Solution

Keep #39269 open, but do not merge the batch directly. Split or reimplement the still-valid pieces: add shared outbound Discord Unknown Channel/code 10003 permanent-error classification with policy tests, and add a queued webchat-plus-stale-external-route regression around resolveAnnounceOrigin/sendAnnounce. Drop or rework the already-covered provider error sanitizer, align chat-history behavior with current visible-history projection and transcript de-dupe semantics, and leave duplicate plugin warning suppression to the current documented precedence/diagnostic policy or a separate maintainer decision.
