## Summary

Keep #39269 open. Current main still only partially overlaps this batch PR: Codex-prefixed provider error JSON is covered, but the shared outbound queue still lacks the Discord Unknown Channel/10003 permanent-error policy, chat.history still has no delivery-mirror display filter, resolveAnnounceOrigin still strips webchat before falling back to persisted external routes, and the plugin duplicate-warning behavior intentionally differs from the PR. The PR also has unresolved P1/P2 review comments, so it is not a clean merge candidate as-is.

## Best Possible Solution

Keep this PR open as a source for focused follow-up, not as a batch merge. Split or reimplement the still-valid pieces: add a generic outbound queue Unknown Channel/10003 permanent-error rule with policy tests if maintainers want it; add delivery-mirror filtering in the shared chat display projection before limiting/pagination; fix webchat stale-route handling in resolveAnnounceOrigin while preserving explicit completion routes; and leave the current duplicate-plugin warning policy in place unless a maintainer explicitly chooses quieter cross-origin diagnostics.
