## Summary

Keep open. Current main only partially overlaps this batch PR: Codex-prefixed error sanitization and safer completion-route handling exist, but the Discord Unknown Channel permanent-error rule is still missing, chat.history still has no general delivery-mirror filter, resolveAnnounceOrigin still strips webchat before falling back to persisted external route state, and the plugin duplicate-warning behavior on main intentionally differs from the PR. The PR also still has unresolved review regressions, so it is not a clean merge candidate as-is.

## Best Possible Solution

Keep this PR open only as a source of focused follow-up work, not as a clean batch merge. The best path is to split or reimplement the still-valid pieces: add a Discord `Unknown Channel`/10003 permanent-error rule with a policy test if maintainers still want that behavior, put any delivery-mirror filtering in the shared chat display projection before limiting, add a focused webchat-plus-stale-external-route subagent regression test before changing announce origin behavior, and leave the plugin duplicate-warning policy unchanged unless a maintainer explicitly chooses quieter cross-origin diagnostics.
