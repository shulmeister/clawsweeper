## Summary

Current main still has the reported behavior: bundled provider plugin catalog hooks can add providers to `models.json` independently of explicit `models.providers` entries, and Codex still has a synthetic/fallback catalog path. The related #68965 tracks the narrower `models.mode: "replace"` aspect, but this issue's broader question about whether implicit bundled catalogs belong in `models.json` still needs maintainer product/API judgment.

## Best Possible Solution

Keep this open for maintainer decision on the `models.json` contract. A concrete fix would likely gate write-time implicit provider catalogs on explicit provider auth/config intent, keep unauthenticated static rows limited to display surfaces like `models list --all`, and ensure `models.mode: "replace"` excludes implicit bundled providers or explicitly link that narrower work to #68965 if maintainers decide replace-mode is the supported opt-out.
