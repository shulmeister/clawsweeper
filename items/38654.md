## Summary

Keep open. Current `main` supports multiple Telegram bot accounts in one OpenClaw gateway, but it does not implement bot-to-bot communication inside the same Telegram group. The Telegram channel code explicitly treats this as a platform limitation for groups and only adds a `channel_post` path for Telegram channels, not group chats.

## Evidence

- **Telegram docs only cover privacy/admin visibility for group messages:** The Telegram docs say bots can see all group messages by disabling privacy mode or making the bot an admin, but they do not claim support for seeing other bots' messages in groups.
  - file: [docs/channels/telegram.md:77](https://github.com/openclaw/openclaw/blob/ac063568d354c14e0a0a43bcffb9725bb3fca651/docs/channels/telegram.md#L77)

- **Multi-agent docs cover multiple Telegram bot accounts, not cross-bot group collaboration:** The multi-agent guide shows separate Telegram `accountId`s bound to different agents in one gateway. That is multi-account routing, not multiple OpenClaw bots observing each other in one Telegram group.
  - file: [docs/concepts/multi-agent.md:336](https://github.com/openclaw/openclaw/blob/ac063568d354c14e0a0a43bcffb9725bb3fca651/docs/concepts/multi-agent.md#L336)

- Telegram's platform rules may make true bot-to-bot visibility in group chats impossible without an out-of-band relay/API design.
- The issue proposes multiple different scopes (platform config, relay API, docs); any future fix needs product clarification before closing.
