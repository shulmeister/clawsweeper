## Summary

Current main already uses a non-bootout macOS restart path for `openclaw gateway restart`: it clears disabled state, tries `launchctl kickstart -k`, and if the service was previously unloaded it re-registers the plist with `launchctl bootstrap` before retrying. The latest release `v2026.4.22` contains that implementation and regression coverage.
