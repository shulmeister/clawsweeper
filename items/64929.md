## Summary

Keep open. Current `main` has a partial mitigation: Playwright CDP connections are now cached per `cdpUrl`, so the exact "reconnect every action" theory is no longer fully true. But the local-managed profile path still opts out of persistent Playwright tab operations, and action routing still re-resolves target ids with fresh per-page CDP sessions plus `/json/list` fallback. In this read-only Linux checkout I could not run a macOS Brave reproduction, so there is not enough evidence to say the reported slowdown is fixed or obsolete.
