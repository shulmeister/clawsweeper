## Summary

Current main still does not implement the requested safe fallback for binary document formats in the generic `read` tool. The OpenClaw wrapper delegates to the pinned upstream read implementation before any document MIME/extension guard, the sandbox MIME hook keeps only images, and the existing PDF tool is separate from `read` and does not cover Office documents. Related fix PRs are not landed, with #57600 still the closest active implementation vehicle.

## Best Possible Solution

Keep #35406 open. The best implementation is a small core guard in `src/agents/pi-tools.read.ts` before `executeReadWithAdaptivePaging`: detect known binary document extensions/MIMEs, return a concise structured fallback, point PDFs to the existing `pdf` tool when available, and add host plus sandbox regression tests for at least `.docx`, `.xlsx`, `.pptx`, and `.pdf`. PR #57600 remains the closest active implementation vehicle if maintainers want to revive it.
