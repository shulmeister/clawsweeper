## Summary

Current main already implements auto-compaction near the effective context limit, includes the optional pre-compaction memory flush requested here, and exposes threshold-equivalent tuning through `agents.defaults.compaction.reserveTokensFloor`, `agents.defaults.compaction.memoryFlush.softThresholdTokens`, and runtime `contextTokens` caps.

## Close Comment

Closing from a Codex review because current `main` already implements this well enough. Auto-compaction is documented as default-on in `docs/concepts/compaction.md`, the threshold is already configurable through the effective context budget plus compaction headroom (`models.providers.*.models[].contextTokens` / `agents.defaults.contextTokens` and `agents.defaults.compaction.reserveTokensFloor`), and the optional pre-compaction memory flush is already implemented/configurable under `agents.defaults.compaction.memoryFlush` (`docs/reference/session-management-compaction.md`, `docs/gateway/configuration-reference.md`). The current code also applies the effective runtime context cap to Pi’s auto-compaction threshold in `src/agents/pi-embedded-runner/compact.ts` and computes the preflight threshold in `src/auto-reply/reply/agent-runner-memory.ts`. The core feature was already called out in the `v2026.1.11` changelog, with later fixes for small-context/runtime-cap edge cases landing afterward. If you still see a concrete failure on a current build, please open a fresh bug with the exact provider/model, `contextWindow`/`contextTokens` settings, `/status` output, and logs.
