## Summary

Keep #26037 open. Current main still has Qwen Coding Plan setup and generic OpenAI-compatible `extra_body` pass-through, but the bundled qwen provider does not yet officially map Bailian/Qwen thinking for the requested Coding Plan models such as `qwen3.5-plus`, `glm-5`, and `kimi-k2.5`.

## Best Possible Solution

Keep the issue open and implement this in the bundled `qwen` provider: verify current Bailian Coding Plan thinking semantics for `qwen3.5-plus`, `glm-5`, `kimi-k2.5`, and `MiniMax-M2.5`; add Qwen-owned reasoning metadata and a provider thinking profile; map OpenClaw `/think` levels to the correct `enable_thinking` and `thinking_budget` payload fields; document the supported setup on docs.openclaw.ai/providers/qwen; and add Qwen-owned tests for catalog discovery, payload construction, config validation, and the reported WebUI/gateway restart path.
