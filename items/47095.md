## Summary

Keep open. Current main still has an asymmetric Discord path: text message ingress resolves the per-channel Discord config and carries `GroupSystemPrompt` into the reply pipeline, but Discord voice segment ingress calls `agentCommandFromIngress` directly without resolving/passing the channel `systemPrompt` as `GroupSystemPrompt` or `extraSystemPrompt`. The related Slack issue #59238 is useful supporting context for per-channel prompt behavior, but it does not supersede this Discord voice-specific bug.
