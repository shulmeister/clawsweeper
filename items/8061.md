## Summary

Keep #8061 open. Current main still has only a global subagent concurrency cap at `agents.defaults.subagents.maxConcurrent`; `agents.list[]` entries do not have a typed or strict-schema per-agent cap, and native plus ACP `sessions_spawn` both enqueue child runs on the shared `subagent` lane. The issue has a concrete TTS-local-model use case, a second user confirmed the same problem, and the latest maintainer/Codex discussion also identified this as an unmet core config/runtime gap.

## Best Possible Solution

Keep #8061 open. Add a validated per-target-agent subagent concurrency setting, likely `agents.list[].subagents.maxConcurrent` unless maintainers choose the issue's top-level shape; update types, strict Zod schema, generated schema/help, and docs; then enforce it in the subagent queue/runtime for native and ACP spawns so a capped target serializes while uncapped targets can still run concurrently under the existing global lane cap. Add focused tests for capped target serialization and uncapped parallelism.
