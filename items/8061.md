## Summary

Current main still does not implement per-target-agent subagent concurrency. `agents.defaults.subagents.maxConcurrent` remains the only configured subagent concurrency cap, `agents.list[]` remains strict without `maxConcurrent`, and native plus ACP subagent spawns still dispatch through the shared `subagent` lane. The existing 2026-04-26 Codex review comment keeping #8061 open still matches current `7a85c1a8226e`.

## Best Possible Solution

Keep #8061 open. The best core path is to add a validated and documented target-agent concurrency setting under `agents.list[]`, define whether it caps all runs for an agent id or only subagent target runs, update generated schema/help/docs, and enforce it in native and ACP `sessions_spawn` scheduling alongside the existing global `CommandLane.Subagent` cap. Add focused tests proving a capped target such as `tts-agent` serializes while uncapped targets can still run concurrently.
