## Summary

Keep #8061 open. Current main still only supports global subagent concurrency via `agents.defaults.subagents.maxConcurrent`; it does not support an `agents.list[]` per-target-agent concurrency cap for cases like limiting one TTS agent while allowing other subagents to run in parallel. The issue is concrete, non-maintainer-authored, has no protected labels, and the sole comment reports the same problem rather than pointing to an implementation or duplicate.
