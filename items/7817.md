## Summary

Current main still does not implement #7817's broad Control UI feature. OpenClaw has a grouped session picker, parent/child/spawned session metadata, checkpoint branching, slash-command steering, and display-only reply pills, but it still lacks project tree metadata/navigation, composer-level threaded target routing or fanout, arbitrary message split/move triage, and seed-to-branch workflows. The related #55349 request is narrower quote-reply/threaded-reply work and does not supersede this issue.

## Best Possible Solution

Keep #7817 open as the broad Control UI product feature. A complete implementation likely needs explicit project/session-tree metadata in the session contract, additive Gateway protocol methods for split/move/seed and route/fanout targeting, a project tree navigator in the Control UI, composer target controls, docs, and UI/protocol tests. Keep #55349 open separately as the narrower quote-reply/threaded-reply request.
