## Summary

Keep #7817 open. Current main has useful session primitives, including grouped session selects, parent/child metadata, display-only reply pills, and checkpoint branching, but it still does not implement the requested Control UI project tree navigator, composer-level thread routing or fanout, arbitrary message split/move triage, or seed-elsewhere workflow. The related #55349 is narrower quote-reply work and does not supersede this broader feature.

## Best Possible Solution

Keep #7817 open as the broad Control UI product feature. A complete implementation should add additive Gateway protocol/session support for project/session-tree metadata, composer route targets or fanout, split/move selected transcript messages, and seed/copy branch workflows, then wire those into a project tree navigator with focused UI/protocol tests and docs. Reuse existing parentSessionKey, childSessions, checkpoint branching, and reply-pill primitives where they fit; keep #55349 separate for the narrower quote-reply/threaded-reply request.
