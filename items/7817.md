## Summary

Current main still does not implement the requested Control UI project tree, composer-level threaded entry routing/fanout, arbitrary message split/triage, or seed-to-branch workflow. The shipped surface has an agent-grouped session select, a sessions table with per-session overrides, parent/checkpoint metadata, and checkpoint branching, plus display-only reply metadata for channel replies. The related #55349 remains a narrower quote-reply request and does not supersede this broader feature.

## Best Possible Solution

Keep #7817 open as the broad Control UI product feature. A complete implementation likely belongs across the session contract, Gateway protocol/server methods, and Control UI: explicit project/session-tree metadata, a project/tree navigator, composer target/thread routing semantics, transcript-level split/move/seed operations, docs, and UI/protocol tests. Keep #55349 as a narrower quote-reply follow-up rather than treating it as the canonical replacement.
