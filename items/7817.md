## Summary

Keep #7817 open. Current main has session parent/child metadata, checkpoint branching, local delete/pin/display affordances, and display-only reply pills, but it still does not implement the requested Control UI project tree navigator, composer routing/fanout, selected-message split/move triage, or seed-elsewhere workflow. The related #55349 quote-reply request is narrower and does not supersede this broader product feature.

## Best Possible Solution

Keep #7817 open as the broad Control UI product feature. A complete implementation should add additive Gateway/session metadata for projects and tree relationships, composer route targets or fanout, selected-message split/move workflows, and seed/copy branch behavior, then wire those into a Control UI project tree navigator with focused protocol/UI tests and docs. Reuse parentSessionKey, childSessions, checkpoint branching, slash-command steering, and reply-pill primitives where they fit; keep #55349 separate for quote-reply UX.
