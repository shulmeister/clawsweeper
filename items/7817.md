## Summary

Current main does not implement #7817’s broad Control UI feature. OpenClaw has session listing, parent/child metadata, checkpoint branching, and display-only reply pills, but it still lacks project tree metadata/navigation, composer-level threaded target routing or fanout, arbitrary message split/move triage, and seed-to-branch workflows. The existing related #55349 is a narrower quote-reply request and does not supersede this issue.

## Best Possible Solution

Keep #7817 open as the broad Control UI product feature. A complete solution likely needs explicit project/session-tree metadata in the session contract, additive Gateway protocol methods for split/move/seed and routing, a project tree navigator in the Control UI, composer target/fanout semantics, docs, and UI/protocol tests. Keep #55349 as a narrower quote-reply follow-up rather than treating it as the canonical replacement.
