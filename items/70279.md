## Summary

Keep open. Current main has several partial fixes for #70279: fallback skip decisions are now logged, bundled claude-cli defaults use stdin/jsonl, inherited Claude/Anthropic env overrides are cleared, and fallback runtime model metadata is separate from explicit session model overrides. However the root/systemd-specific request is still not implemented well enough to close: the CLI backend docs do not mention running claude-cli as root, `IS_SANDBOX=1`, or a non-root service-user recommendation, and the source still derives `--permission-mode bypassPermissions` from OpenClaw's default YOLO exec policy without any UID/root guard or first-class allowRoot handling.

## Best Possible Solution

Keep #70279 open and narrow it to the remaining maintainer work: add a root/systemd Claude CLI troubleshooting section to the public CLI backend docs, document the supported non-root service-user or IS_SANDBOX workaround if maintainers accept it, and consider a root-aware diagnostic or first-class config guard for `bypassPermissions` on UID 0. If model-fallback/decision is considered insufficient for operators, add the requested CLI-backend-scoped skip log as a smaller follow-up.
