## Summary

Current main still matches the actionable Signal group-drop path. Signal group admission is checked against the sender allowlist before group-id config is consulted, and the shared mention helper still falls back to require mentions when a group entry exists without an explicit requireMention override.

## Best Possible Solution

Keep this issue open and fix the bundled Signal plugin. The safest direction is to add Signal group-id admission using channels.signal.groups under groupPolicy="allowlist" before sender authorization blocks the message, while preserving groupAllowFrom as sender authorization when configured. Then make an explicit maintainer decision on whether Signal-specific group entries should default to requireMention:false or whether docs/config should continue requiring an explicit override, and add regression tests for a specific Signal group id under allowlist policy.
