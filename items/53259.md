## Summary

Current `main` does not implement the documented Discord elevated `allowFrom` fallback or the matching `sandbox explain` diagnostics, so PR #53259 still addresses a live bug and should remain open.

## Evidence

- **Docs promise a Discord fallback today:** The docs explicitly say `tools.elevated.allowFrom.discord` falls back to `channels.discord.allowFrom` when omitted.
  - file: [docs/tools/elevated.md:90](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/tools/elevated.md#L90)
  - command: `nl -ba docs/tools/elevated.md | sed -n '84,96p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Runtime path does not implement the documented fallback:** `resolveElevatedPermissions` only asks the channel plugin for `elevated.allowFromFallback`. A repo-wide search shows the only `allowFromFallback` references on `main` are this call site and the adapter type, so no Discord implementation exists and `channels.discord.allowFrom` is never consulted for elevated auth.
  - file: [src/auto-reply/reply/reply-elevated.ts:202](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/auto-reply/reply/reply-elevated.ts#L202)

  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Diagnostics still ignore the Discord fallback:** `sandbox explain` reads only `tools.elevated.allowFrom.<channel>` into `globalAllow`/`agentAllow` and computes `allowedByConfig` from those tokens. It does not read `channels.discord.allowFrom`, so the diagnostic remains inconsistent with the documented behavior.
  - file: [src/commands/sandbox-explain.ts:183](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/commands/sandbox-explain.ts#L183)
