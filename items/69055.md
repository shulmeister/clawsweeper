## Summary

Keep open. Current main does not implement the requested structured `cron.delivery.sent` log/event. The cron dispatch path has the needed job, agent, run session, and resolved delivery context, but it only passes normal outbound delivery fields into `deliverOutboundPayloads`. The shared outbound diagnostics emit generic `message.delivery.*` events with channel/kind/session/result counts or error category, not cron job/run attribution, target, account, or platform message IDs. The companion #69054 remains a separate lint/validation request, not a canonical duplicate for this logging feature.

## Best Possible Solution

Keep the issue open and implement a narrow cron-owned telemetry path: thread `{ cronJobId, cronRunId, agentId }` from `dispatchCronDelivery` into the outbound delivery attempt, include resolved `accountId`, `channel`, `to`, and returned platform `messageId`, emit exactly one structured success/error event per actual send attempt without message content or secrets, and add focused tests for success, failure, best-effort partial failure, and multi-payload delivery. #69054 should remain a companion lint issue rather than superseding this forensic logging request.
