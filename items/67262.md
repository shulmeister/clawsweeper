## Summary

Current main already implements an equivalent and stronger fix than PR #67262: exec completion payloads are embedded into the heartbeat prompt, empty exec completion prompts return HEARTBEAT_OK only, successful empty exec.finished events are suppressed at the gateway producer, and regression tests cover the relevant prompt and producer paths. The linked issue #66595 is already closed as implemented, and the fix shipped in v2026.4.24.

## Best Possible Solution

Close PR #67262 as implemented on main. Keep the shipped implementation that fixes the issue at both layers: the gateway producer suppresses successful empty exec completions and carries compact output when present, while the heartbeat prompt embeds exec event text and uses HEARTBEAT_OK for empty payloads.
