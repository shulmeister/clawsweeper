## Summary

Current main already implements the behavior PR #67262 was trying to add, and does so more completely: heartbeat exec prompts embed the actual completion text, empty completion prompts return HEARTBEAT_OK only, gateway exec.finished production suppresses successful empty-output events, and tests cover the prompt and producer paths. The linked issue #66595 is already closed as implemented, and the fix shipped in v2026.4.24.

## Best Possible Solution

Close PR #67262 as implemented on main. Keep the shipped implementation from v2026.4.24: gateway exec.finished events now suppress successful empty-output completions and carry compact output when present, while heartbeat prompt construction embeds exec event text directly and uses HEARTBEAT_OK for truly empty payloads. The linked bug #66595 is already closed as completed, so this PR is now duplicate obsolete work rather than a change that should be merged.
