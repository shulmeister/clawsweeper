## Summary

Current main and the latest release no longer match the reported missing-price-table failure. OpenClaw pins @mariozechner/pi-ai 0.70.2, whose catalog includes an Anthropic `claude-opus-4-7` entry with per-million input/output/cache pricing, and the Anthropic transport uses that model cost table when writing `usage.cost`. The gateway pricing cache also converts OpenRouter/LiteLLM per-token pricing into OpenClaw's per-million format before cost estimation, with regression coverage for the exact 0.000005 -> 5 conversion that would otherwise cause the observed millionfold undercount.

## Best Possible Solution

Close #70416 as implemented on main and in v2026.4.24. Keep the current catalog/dependency-backed pricing behavior; if users still need exact 1h-vs-5m Anthropic cache-write accounting, track that as a new focused TTL-aware pricing enhancement rather than keeping this missing-entry bug open.
