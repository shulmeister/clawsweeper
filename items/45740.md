## Summary

Keep open. Current main still has the gh-issues skill constructing sub-agent prompts with raw issue bodies and actionable review comment JSON, and the runtime does not provide the requested per-spawn tool scoping/sanitization path in that workflow. The discussion supports treating this as unattended-automation hardening rather than a confirmed standalone vulnerability under SECURITY.md, but it is concrete, actionable in this repo, and none of the allowed close reasons apply.
