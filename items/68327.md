## Summary

Keep open. Current main partially overlaps this PR, but it does not fully implement the central auth-profile startup latency change, and the PR as submitted has unresolved review-blocking regressions in the provided discussion.

## Best Possible Solution

Do not merge this PR as-is. The best path is a narrower revision or replacement PR that keeps the existing external-auth behavior for OAuth callers, moves any secrets-only no-overlay optimization into prepareSecretsRuntimeSnapshot or a dedicated helper, adds regression tests for SecretRef startup plus external OAuth recovery, removes profiling instrumentation, and splits the Windows path/mDNS/async-lock changes into independently reviewed fixes with tests.
