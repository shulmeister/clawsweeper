## Summary

Keep open. Current main still lacks WhatsApp per-group `allowFrom`: the strict WhatsApp group schema/type/generated metadata only admit `requireMention`, `tools`, `toolsBySender`, and `systemPrompt`, while the inbound access path computes one account/channel-level group sender allowlist from `groupAllowFrom` or `allowFrom` fallback. The issue is also not superseded by the linked `denyFrom` PR or the maintainer-owned multi-account inheritance issue; those cover different layers.

## Best Possible Solution

Keep the issue open and implement this in the bundled WhatsApp plugin: add `allowFrom` to `WhatsAppGroupConfig`, the strict Zod schema, generated metadata, docs, and tests; resolve the active group config before access control; use `groups[<jid>].allowFrom` as an override for group sender authorization and command authorization; preserve existing fallback to account/channel `groupAllowFrom` when the per-group field is undefined.
