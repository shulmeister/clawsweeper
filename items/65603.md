## Summary

The Azure AI Foundry GPT request-shaping regression described in #65603 appears implemented on current main: Foundry models now carry explicit compat flags that suppress unsupported `store` payloads and pick the correct token field, with regression tests covering the exact shaping path. The original reporter also confirmed on 2026-04-14 that upgrading to 2026.4.12 fixed their Azure Foundry setup.

## Evidence

- **Foundry models get explicit compat shaping:** `buildFoundryModelCompat()` assigns `supportsStore: false` for GPT-5 Foundry responses routes and selects `max_completion_tokens` vs `max_tokens` per Foundry model needs, matching the failure mode discussed in the issue.
