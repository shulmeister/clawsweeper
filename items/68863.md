## Summary

PR #68863 should close as superseded by the open, unmerged normalization-level fix in #68750 and the canonical bug report #68470. This PR targets the same MiniMax premature-compaction symptom but changes the runner fallback; the linked discussion shows that dropping the fallback trades inflated prompt counts for lost timeout-compaction signal and aggregate session fallback behavior. #68750 directly matches #68470's stated root cause by adding MiniMax `prompt_cache_hit_tokens` handling in usage normalization.

## Best Possible Solution

Close #68863 as superseded, keep #68470 and #68750 as the canonical MiniMax token-accounting thread, and land a normalization-level fix with regression coverage. If maintainers want the broader no-assistant-usage fallback problem fixed later, it should carry a true latest-call usage snapshot while preserving timeout-compaction signal instead of dropping the fallback outright.
