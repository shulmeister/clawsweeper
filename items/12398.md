## Summary

Keep #12398 open. Current main still lacks cron postconditions or an equivalent deterministic post-run verifier: the cron job contract, gateway schema, run state/logging, scheduler execution path, CLI/UI/tool schemas, and plugin hooks all stop at existing execution status, delivery status, transient retry, and failure-alert handling. The latest maintainer-side Codex review comment also explicitly kept the issue open, and current main has not added the requested field or behavior since then.

## Best Possible Solution

Keep #12398 open for maintainer/product implementation. The likely core path is an additive cron contract field plus gateway schema, CLI/UI/tool schema support, deterministic checker execution after isolated agent completion, persisted postcondition state/run-log evidence, retry prompt injection, and escalation/failure-alert integration. If maintainers prefer a plugin route, the missing prerequisite is a first-class cron post-run hook/API that can fail a run and schedule cron-owned retry/escalation; current plugin hooks are not sufficient.
