## Summary

Keep #12398 open. Current main still has no cron postconditions field, deterministic post-run checker, checker result state, retry prompt injection from checker output, or cron-owned post-run hook/API that can fail a successful isolated agent run and drive scheduler retry/escalation. The existing implementation covers execution status, delivery status, transient error backoff, and failure alerts only.

## Best Possible Solution

Keep #12398 open for maintainer/product implementation. The likely core path is an additive cron contract and gateway schema field, deterministic checker execution after isolated agent completion and before applyJobResult, persisted checker status/output in cron state and run logs, retry prompt injection with checker output, and integration with existing retry/failure-alert delivery. If maintainers prefer a plugin route, the missing prerequisite is a first-class cron post-run hook/API that can fail a cron run and schedule cron-owned retry/escalation.
