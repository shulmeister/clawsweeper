## Summary

Current main still lacks a WebChat setting to disable, dismiss, customize, or replace the “New messages” banner. The issue is concrete, not protected by maintainer-only metadata, and remains actionable core WebChat/UI configuration work rather than a cleanup close.

## Best Possible Solution

Keep #9546 open. A narrow fix should add a persisted WebChat new-message indicator mode, feed it into scheduleChatScroll and renderChat, update focused UI tests, and document the chosen surface on the public WebChat/config docs.
