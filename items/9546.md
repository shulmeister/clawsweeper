## Summary

Keep #9546 open. Current main still shows the Control UI/WebChat `New messages` indicator when the user is scrolled away from the bottom, and there is still no browser-local setting or `gateway.webchat` configuration option to disable, customize, or replace it with forced auto-scroll. The existing Codex review comment on the issue reached the same conclusion, and the current checkout at 7a85c1a8226eb7d3a96906d2721a189768d3034a still matches that behavior.

## Best Possible Solution

Keep the issue open until maintainers choose the product surface. A narrow fix should add either a persisted browser-local Control UI preference or a documented `gateway.webchat` config option, gate `chatNewMessagesBelow`/`showNewMessages` behavior through it, and update focused UI tests plus WebChat/config docs.
