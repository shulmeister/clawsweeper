## Summary

Keep #9546 open. Current main still shows the Control UI/WebChat “New messages” indicator when the user is scrolled away from the bottom, and neither the persisted UI settings nor the strict `gateway.webchat` config schema exposes a disable/customize/autoscroll option. The prior Codex review reached the same conclusion, and the commits since that review only touched an unrelated chat loading skeleton and diagnostics config.

## Best Possible Solution

Keep the issue open until maintainers choose the product surface. The narrow implementation should add either a browser-local Control UI preference or a documented `gateway.webchat` config option, gate `chatNewMessagesBelow`/`showNewMessages` behavior through it, and update focused UI tests plus WebChat/config docs.
