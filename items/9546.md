## Summary

Keep #9546 open. Current main still drives the WebChat `New messages` indicator directly from scroll state, renders it whenever that state is true, and has no persisted UI or `gateway.webchat` option to disable, customize, or replace it with forced autoscroll.

## Best Possible Solution

Keep #9546 open. A narrow fix should add a persisted WebChat/Control UI preference or documented `gateway.webchat` option for the new-message indicator mode, feed that setting into both `scheduleChatScroll` and `renderChat`, and update focused UI tests plus public WebChat/config docs.
