## Summary

Keep this issue open. Current main still lacks user-level targeted overrides or a trust-level setting for the cited Safety, Tool Call Style, and self-update permission sections. The existing paths are whole-prompt replacement, a narrow GPT-5 personality overlay, and provider-owned section overrides for only selected sections; those do not satisfy the requested user config surface.

## Best Possible Solution

Keep this open for core agent configuration design. A conservative path is a documented, schema-backed user-level and per-agent prompt policy surface for selected sections or an explicit trust policy, wired through `src/agents/system-prompt.ts`, generated config schema/help/docs, and tests proving Safety, Tool Call Style, and self-update permission text can be selectively replaced or relaxed without requiring full `systemPromptOverride`. Coordinate with the broader #25493 prompt-template discussion while preserving this issue's narrower section-level safety-policy requirement.
