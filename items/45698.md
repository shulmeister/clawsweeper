## Summary

Keep #45698 open. Current main has a partial backend improvement for channel-status fan-out, but the reported Control UI stuck/sluggish behavior is not clearly implemented: the UI still issues `channels.status` with the same 8000ms server-side timeout, the browser RPC client has no per-request timeout/abort path, and the polling intervals called out by related PR #45930 are still 5s/2s/3s on main. The linked PR #45930 remains open and narrower than this issue, so it is not strong enough to supersede the broader regression report.
