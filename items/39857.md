## Summary

Keep #39857 open. Current main at e672b61417af5c45b0431df6d9109a1f4b618ef5 still lets positive context-token overrides win before known model-window lookup in the shared resolver and status summary resolver, and session/status paths still pass configured or persisted overrides into those code paths. The linked fix PR #39859 is still separate from main in the provided context and its pull ref remains distinct from origin/main.

## Best Possible Solution

Keep #39857 open until PR #39859 or an equivalent narrow maintainer-approved fix lands. The fix should make configured and persisted positive contextTokens values act as ceilings when a smaller selected-model window is known, preserve existing override behavior when the model window is unknown, align or remove the duplicate status-summary resolver, and add focused regression coverage for resolver, reply setup, /status, status summary, and session metadata persistence paths.
