## Summary

Keep open. Current main at e672b61417af5c45b0431df6d9109a1f4b618ef5 still validates `entry?.config` in non-activating manifest snapshot and CLI metadata paths, and `validatePluginConfig` still turns `undefined` into `{}`, which is the reported false-positive path for strict required plugin config schemas. The linked #70811 appears to be the right fix direction, but it is not merged into this checkout/current main.

## Best Possible Solution

Land #70811 or an equivalent narrow loader fix: defer/skip schema validation only when `entry?.config === undefined` in non-activating snapshot/CLI metadata paths, while preserving validation for real activation and for non-activating paths that do provide explicit config. Add focused regression coverage in `src/plugins/loader.test.ts` and `src/plugins/loader.cli-metadata.test.ts` for strict required schemas with absent config.
