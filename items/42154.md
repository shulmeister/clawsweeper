## Summary

PR #42154 is superseded by the current sandbox-specific tool policy design on main. The linked bug #41757 was closed after main added `tools.sandbox.tools.alsoAllow` support via #54492, with tests and docs treating sandbox tool policy as its own restrictive layer rather than automatically promoting `agents.list[].tools.alsoAllow` into sandbox allowlists.

## Best Possible Solution

Close this PR as superseded by #54492 and keep the shipped explicit sandbox policy behavior. Users who want a plugin tool in sandboxed sessions should add the tool, plugin selector, or relevant group to `tools.sandbox.tools.alsoAllow` or the per-agent sandbox tool policy; any future automatic promotion from normal `tools.alsoAllow` should be a fresh, explicit design discussion.
