## Summary

Keep #14344 open. Current main still does not expose WhatsApp message delete/revoke: WhatsApp discovery advertises only react/poll, dispatch supports only react, the runtime only handles react, the active listener/send API has no delete helper, and docs still list shared delete for Discord/Slack/Telegram/Matrix only. The closed related #38723 was a duplicate of this request, not a shipped fix.

## Best Possible Solution

Keep #14344 open as the canonical bundled WhatsApp plugin feature request. The implementation should add delete to WhatsApp message-action discovery/support, route it through the plugin-owned gateway action runtime, add an active-listener delete/revoke helper that sends the Baileys message key, enforce bot-owned/fromMe deletion with group participant handling, and cover discovery/runtime/send API behavior with focused tests plus updates to docs/cli/message.md and docs/channels/whatsapp.md.
