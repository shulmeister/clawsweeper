## Summary

Keep open. Current main still does not implement WhatsApp message delete/revoke support: the shared message action vocabulary includes `delete`, but the WhatsApp plugin only advertises `react`/`poll`, only dispatches `react`, and its runtime/listener send API has send, poll, reaction, and typing paths but no delete/revoke path. The public docs also exclude WhatsApp from `message delete` and describe WhatsApp tool support as reactions plus polls. The issue discussion contains concrete user reports, and the later #38723 appears to be a duplicate rather than a better canonical tracker.

## Best Possible Solution

Keep #14344 open as the canonical WhatsApp delete/revoke feature request. The best implementation belongs in the bundled WhatsApp plugin: advertise `delete`, allow it in `supportsAction`, route it through the gateway-owned WhatsApp action runtime, extend the active listener/send API to call Baileys with a delete key, validate bot-owned/fromMe deletion and group participant keys, then add focused runtime/action tests plus updates to `docs/cli/message.md` and `docs/channels/whatsapp.md`. Treat #38723 as the duplicate if maintainers want a single tracker.
