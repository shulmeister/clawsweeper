## Summary

Keep open. Current main still lacks WhatsApp message delete/revoke support. The shared action registry has `delete`, but the WhatsApp plugin only advertises `react`/`poll`, only accepts `react` in plugin dispatch, and falls back to the shared unsupported-action error for `message delete channel=whatsapp`. The issue and RaviTharuma follow-up give concrete use cases, and the existing maintainer/Codex comment also kept this open.

## Best Possible Solution

Keep #14344 open as the canonical WhatsApp delete/revoke feature request. The likely implementation belongs in the bundled WhatsApp plugin: advertise `delete`, add a guarded WhatsApp delete/revoke action path through the gateway-owned Baileys listener, validate bot-owned/fromMe deletion semantics and group participant keys, add focused action/runtime tests, and update the public WhatsApp and message CLI docs. The later #38723 can be handled as a duplicate separately if maintainers want one tracker.
