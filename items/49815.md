## Summary

Keep open. Current `main` has partial Browser/CDP fixes and docs for loopback SSRF and attach-only loopback CDP, but it does not fully implement the issue’s requested behavior for same-host Docker Browserless on loopback. A profile with loopback `cdpUrl` still defaults to `attachOnly: false`, is still classified as `local-managed` rather than `remote-cdp`, and still falls into the `Port ... is in use ... but not by openclaw` branch when HTTP is reachable but the websocket is not. That means the reported `attachOnly: false` failure mode remains possible on current `main`, even though the `attachOnly: true` / loopback SSRF path has been improved.
