## Summary

Keep #38716 open. Current main still lacks Slack RTS support via assistant action tokens: there is no action_token capture, no assistant_thread event handling, and no Slack `assistant.search.context` or `search` message action implementation. The related search PR context does not supersede this request because it targeted `search.messages` with a user token, not the requested assistant action-token RTS flow.

## Best Possible Solution

Keep the issue open and implement it in the Slack channel plugin: capture `action_token` from Slack assistant thread events, retain it only for the appropriate assistant-thread context, expose a Slack search message action or dedicated tool, and call Slack RTS `assistant.search.context` with tests and docs. If maintainers decide RTS belongs outside bundled Slack, move it to an explicit ClawHub/plugin tracker rather than auto-closing this missing bundled-plugin behavior.
