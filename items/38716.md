## Summary

Keep #38716 open. Current main still does not extract Slack assistant `action_token` values from `assistant_thread_*` events, does not call Slack RTS `assistant.search.context`, and does not expose Slack `search` as a message action. The related #58143 discussion is useful adjacent context, but that PR was closed unmerged and targeted `search.messages` with a user token, not the requested assistant action-token RTS flow.

## Best Possible Solution

Keep the issue open until the bundled Slack plugin either implements Slack RTS properly or maintainers choose a different product direction. A complete implementation should capture `action_token` from `assistant_thread_started` and `assistant_thread_context_changed` events, preserve it only in the appropriate thread/request scope, call Slack `assistant.search.context`, expose it through Slack message action/tool discovery, and add Slack tests plus docs for required scopes and setup.
