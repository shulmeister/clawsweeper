## Summary

Keep #38716 open. Current main still does not implement Slack RTS via assistant action tokens: the Slack plugin has assistant thread status support, but no action_token capture, no assistant_thread_started/context_changed event registration, no assistant.search.context/search.context call, and no Slack message-tool search action. The related search.messages PR context does not supersede this request because it targeted user-token Slack Web API search rather than assistant action-token RTS.

## Best Possible Solution

Keep the issue open and implement it in the Slack plugin if maintainers want bundled RTS support: subscribe to assistant_thread_started and assistant_thread_context_changed, capture and scope the action_token to the assistant thread, expose a Slack search message action or dedicated tool, call assistant.search.context with that token, and add focused tests plus Slack docs/setup-scope updates. If maintainers decide Slack RTS should live outside the bundled Slack plugin, open a specific ClawHub/plugin API tracker instead of closing this as already handled.
