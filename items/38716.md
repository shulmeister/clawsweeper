## Summary

Keep #38716 open. Current main still does not implement Slack RTS via assistant action tokens, and the related search.messages PR context is a different Slack Web API/user-token search path rather than assistant.search.context/action_token support.

## Best Possible Solution

Keep the issue open. The best implementation path is in the Slack plugin unless maintainers make a product call to move it out: subscribe to assistant_thread_started and assistant_thread_context_changed, capture and scope action_token to the assistant thread, add an assistant.search.context helper/action or dedicated tool, gate it on Slack setup/scopes, and add focused tests plus Slack setup/docs updates. If maintainers prefer this outside bundled Slack, first open a narrower plugin/API design tracker for exposing enough Slack assistant-thread context; current evidence does not show an already-available external plugin path that implements this against the bundled Slack integration.
