## Summary

Keep #38716 open. Current main still does not implement the requested Slack RTS assistant-action-token flow: the Slack manifest does not subscribe to assistant thread events, the monitor has no action_token capture path, the Slack Web API wrapper has no assistant.search.context helper, and Slack message actions do not expose search. The related search.messages work is a different user-token API path and does not supersede this issue.

## Best Possible Solution

Keep the issue open for a focused Slack plugin feature. The likely implementation belongs in the bundled Slack plugin: subscribe to assistant_thread_started and assistant_thread_context_changed, capture and scope action_token for the relevant assistant interaction, add an assistant.search.context helper, expose it as a Slack search message action or dedicated tool, and update Slack setup/docs/tests for Agents & AI Apps and required search scopes. If maintainers prefer externalization, first track the missing seam for an external plugin to observe the existing Slack app event stream and assistant action tokens without replacing the bundled Slack channel.
