## Summary

Keep open. Current `main` only partially addresses the regression: Feishu account configs now allow per-account policy fields again, but the strict schema still accepts `name` and not legacy `botName`, and there is no Feishu-specific doctor compatibility export to migrate that key before validation. The provided timeline’s later link to open issue #65177 matches the current-tree evidence that this upgrade path is still not fully repaired.
