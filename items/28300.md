## Summary

Keep open. Current main has overlapping Control UI theming work through Claw/Knot/Dash, light/dark/system modes, roundness controls, and one browser-local tweakcn-backed Custom import slot, but it still does not implement the requested six curated preset themes or an in-app Custom Theme Studio with hue generation, gradient controls, grouped color editing, and per-variable override/reset UI.

## Best Possible Solution

Keep this open for Control UI product/design handling. A good implementation would extend the existing Appearance theme model deliberately: decide whether to add accepted built-in preset families, a local generator/editor, or narrow the request around tweakcn imports; then align `ThemeName`/`resolveTheme`, CSS token generation, browser-local persistence, quick settings, i18n, docs, and focused tests.
