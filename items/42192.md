## Summary

Keep open. Current main has not implemented PR #42192's cross-origin duplicate-warning suppression: distinct duplicate plugin IDs still produce `duplicate plugin id detected` diagnostics, and current tests/docs intentionally preserve that behavior for config/global/bundled precedence. The PR is also not safely obsolete because the linked bug #42099 remains open and the PR review identified an unresolved edge-case bug in the proposed per-origin tracking.

## Best Possible Solution

Keep the PR open for maintainer judgment or author revision. A viable fix should be rebased onto the current `resolveDuplicatePrecedenceRank` behavior, clearly define which cross-origin duplicate cases should be silent, preserve any duplicate-state information consumed by validation/Doctor/startup surfaces, and add regression tests for the #42099 reproduction plus the same-root promotion edge case raised in review.
