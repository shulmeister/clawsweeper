## Summary

Keep open. Current `main` still serializes Feishu text commands through the per-chat sequential queue, so the reported queue-bypass regression is not fully implemented away. There is a partial `/stop` carveout, but `/status` and `/new` still stay on the normal chat lane despite docs promising immediate command-only handling.
