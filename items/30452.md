## Summary

Keep open. Current main implements the compaction-only model override, but the issue’s remaining memory-flush-specific model override and successful flush-then-reset mode are still absent from config, schema, docs, plugin memory-flush contracts, and runtime routing.

## Best Possible Solution

Keep this issue open, or split the remaining work into narrower maintainer-approved issues. The likely implementation path is to add `agents.defaults.compaction.memoryFlush.model` and `memoryFlush.resetAfterFlush` to config types, zod schema, generated/help/docs/UI surfaces, and tests; route the silent memory-flush turn through the configured model override; then define and test successful flush-then-reset semantics across session store updates, transcript/session-file handling, queue refresh, active run state, and channel-visible behavior.
