## Summary

Keep open. Current main implements the `compaction.model` portion, but the requested `memoryFlush.model` and `memoryFlush.resetAfterFlush` / flush-then-reset behavior are still absent from config, schema, docs, plugin memory flush contracts, and the runtime memory flush path.

## Best Possible Solution

Keep this issue open or split the remaining scope into narrower maintainer-approved issues. The concrete implementation path is to add `agents.defaults.compaction.memoryFlush.model` and `memoryFlush.resetAfterFlush` to the config types, zod schema, generated/help/docs surfaces, and runtime; route the memory flush turn through the configured override model; then define and test successful flush-then-reset semantics, including session store updates, queue refresh, transcript/session-file handling, and channel-visible behavior.
