## Summary

Keep #30452 open. Current main implements the top-level compaction model override, but the issue also asks for `memoryFlush.model` and `memoryFlush.resetAfterFlush`, and those are still missing from config, runtime, docs, and tests.

## Best Possible Solution

Keep the issue open, or split it into narrower tracked work: one item for `agents.defaults.compaction.memoryFlush.model`, and one for `agents.defaults.compaction.memoryFlush.resetAfterFlush`. The likely implementation path is to add config types, strict schema/generated schema/help/docs/UI surfaces, route the silent memory-flush turn through the configured model override, and implement flush-then-reset only after a successful flush by reusing existing reset/session lifecycle machinery with focused regression tests.
