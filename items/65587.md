## Summary

The request is a clear feature idea, but current main already provides supported personality controls via GPT-5 prompt overlays and SOUL.md, while the exact dual-model router was explicitly rejected in PR #65591 as too much core harness complexity. VISION.md pushes optional capability like this toward plugins/ClawHub rather than new one-off core orchestration.
