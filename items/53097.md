## Close Comment

Closing after Codex review: current OpenClaw already covers this use case on `main`, and it shipped in `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). The implementation is exposed through Gateway WS RPC rather than a new REST `/v1/agents` route: `agents.list` returns configured agents, and the returned rows include `workspace` and resolved `model` data (`src/gateway/server-methods/agents.ts:416`, `src/gateway/session-utils.ts:679`, `src/shared/session-types.ts:14`). The requested file-import side is also already present through `agents.files.list` and `agents.files.get`, which expose the allowed bootstrap/identity workspace files without direct filesystem access (`src/gateway/server-methods/agents.ts:655`, `src/gateway/server-methods/agents.ts:685`). The Gateway protocol docs already document these methods as part of the public surface (`docs/gateway/protocol.md:331`).
