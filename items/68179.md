## Summary

Keep PR #68179 open. Current main still has the bug the PR targets: Discord explicit targets are parsed with kind information, but `parseDiscordExplicitTarget` returns the bare `target.id`, which can strip `user:` before session/cron outbound delivery. The PR is a recent, focused bug fix with a matching regression test in the provided diff and useful production-repro context in the discussion; it is not obsolete, implemented on main, or superseded by the related ClawSweeper reports.

## Best Possible Solution

Keep the PR open and either merge it after normal maintainer validation or apply the same one-line change on main: `parseDiscordExplicitTarget` should return `target.normalized`, with the added regression tests for Discord `messaging.parseExplicitTarget` and `messaging.inferTargetChatType`. This is plugin-owned Discord behavior, not a ClawHub or external-administration cleanup item.
