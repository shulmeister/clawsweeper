## Summary

Keep #17876 open. Current main still exposes immediate model-setting, manual fallback management, OpenRouter-specific scan/probe helpers, and provider auth probes, but not the requested staged workflow to add a candidate as fallback, test an arbitrary model, promote it while moving the old primary to fallback, and roll back in one command. The issue discussion also includes an explicit maintainer comment saying to keep it open.

## Best Possible Solution

Keep #17876 open for a core models CLI workflow. Build on the existing status/probe, fallback, auth, and scan primitives: add a candidate as fallback, test an arbitrary provider/model ref, promote only after verification while moving the old primary into fallbacks, persist enough prior state for openclaw models rollback, then add focused config-transition tests and CLI docs.
