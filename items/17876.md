## Summary

Keep #17876 open. Current main still provides immediate default-model setting plus manual fallback/probe helpers, but not the requested staged migration workflow with arbitrary model testing, promotion, automatic old-primary fallback placement, or rollback. The latest maintainer comment in the issue also explicitly kept it open for this reason.

## Best Possible Solution

Keep the issue open for maintainer/product follow-through. The likely core path is a `openclaw models migrate/test/promote/rollback` workflow, or equivalent onboarding/config UX, built on existing fallback, status probe, and scan primitives: add a candidate as fallback, run real provider/tool/subagent probes for an arbitrary model ref, promote only after verification while moving the old primary into fallbacks, and persist enough previous-state metadata for rollback. Add docs and focused tests around the command surface and config transitions.
