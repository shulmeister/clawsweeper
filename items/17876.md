## Summary

Keep #17876 open. Current main still has manual model set/fallback/scan/status helpers, but not the requested staged migration workflow for arbitrary model testing, promote-with-old-primary-as-fallback, or one-command rollback. The latest maintainer discussion also explicitly says to keep this issue open.

## Best Possible Solution

Keep #17876 open for maintainer/product follow-through. The likely core path is a `openclaw models migrate/test/promote/rollback` workflow, or equivalent onboarding/config UX, built on existing fallback, status probe, and scan primitives: add a candidate as fallback, run provider/tool/subagent probes for an arbitrary model ref, promote only after verification while moving the old primary into fallbacks, and persist enough previous-state metadata for rollback. Add docs and focused tests around the command surface and config transitions.
