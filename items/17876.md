## Summary

Issue #17876 should stay open. Current main still has immediate primary model setting plus manual fallback/status/scan helpers, but not the requested staged model migration workflow with arbitrary model testing, promote, automatic old-primary fallback placement, or rollback. The latest maintainer comment also explicitly kept this issue open for that reason.

## Best Possible Solution

Keep #17876 open for maintainer/product follow-through. The likely core path is a `openclaw models migrate/test/promote/rollback` workflow, or equivalent onboarding/config UX, built on existing fallback, status probe, and scan primitives: add a candidate as fallback, run real provider/tool/subagent probes for an arbitrary model ref, promote only after verification while moving the old primary into fallbacks, and persist enough previous-state metadata for rollback. Add docs and focused tests around the command surface and config transitions.
