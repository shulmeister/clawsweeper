## Summary

Keep #17876 open. Current main still exposes immediate/default-setting and manual fallback primitives, plus OpenRouter-specific scanning and runtime failover, but it does not implement the requested staged model migration workflow with arbitrary model testing, promotion that moves the old primary to fallback, or rollback. The issue also has an explicit maintainer comment from steipete directing that it remain open.

## Best Possible Solution

Keep #17876 open for a core `openclaw models` staged migration workflow. Build on the existing status/probe, fallback, auth, OpenRouter scan, and runtime failover primitives: add a candidate as fallback, test an arbitrary provider/model ref, promote only after verification while inserting the old primary into fallbacks, persist enough previous config for `openclaw models rollback`, and cover the config transitions with focused CLI tests and docs.
