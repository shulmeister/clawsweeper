## Summary

Keep open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still does not implement this PR's central behavior: health snapshots still have no runtime snapshot input, probing remains implicit by default, gateway health still serves through the cached refresh path, and Discord still gates the startup connected-status patch on `gateway.isConnected`. The PR discussion includes a concrete live repro plus a follow-up addressing the bot review blocker, so this remains plausible maintainer backlog rather than cleanup-close material.

## Best Possible Solution

Keep this PR open or consolidate it into a maintainer-chosen canonical health-runtime fix. The product path should thread runtime channel/account state into health snapshots, make probe intent explicit, update gateway health cache/broadcast behavior without overwriting runtime fields, remove the Discord startup `gateway.isConnected` gate for the connected-status patch, and keep focused regression coverage for both the health snapshot path and the Discord race.
