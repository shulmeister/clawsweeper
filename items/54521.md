## Summary

Keep open. Current `main` appears to have fixed the Linux/systemd half of the report, but the issue discussion explicitly added a macOS LaunchAgent reproduction and current launchd code still serializes install environment values into plist `EnvironmentVariables`, with no file-source provenance for doctor to treat `OPENCLAW_GATEWAY_TOKEN` as non-embedded.

## Best Possible Solution

Keep the current systemd implementation, then either retitle/scope #54521 to the remaining macOS LaunchAgent secret inlining/doctor mismatch or explicitly close it only after maintainers choose a canonical LaunchAgent tracker. The likely code path is a launchd-specific secret strategy that stops copying state-dir `.env` values into plist `EnvironmentVariables` or adjusts install/doctor repair semantics with regression tests in `src/daemon/launchd*.test.ts` and `src/commands/doctor-gateway-services.test.ts`.
