## Summary

Keep this issue open. Current main still does not classify legacy Word/OLE MIME types as binary in the media-understanding auto-embed guard, and the provided context shows #54380 is still open and unmerged as the implementation candidate for this exact bug.

## Best Possible Solution

Review and land #54380, or an equivalent scoped patch, that marks `application/x-cfb` and `application/msword` as binary in `src/media-understanding/apply.ts` and adds focused regression tests in `src/media-understanding/apply.test.ts`. After that merges, this issue can close as implemented.
