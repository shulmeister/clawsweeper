## Summary

Keep #13271 open. Current main still has adjacent safeguards, but not the requested per-channel, per-turn tool-call budget with `warn`, `block`, and `auto-delegate` overflow behavior. The latest maintainer/Codex comment already kept this open, and inspection of current main did not find a new implementation since that review.

## Best Possible Solution

Keep this open until OpenClaw adds a typed product/API path for per-channel, per-turn tool-call budgets. A complete fix should add documented channel or agent/channel-scoped config, pass reliable channel/request context into the enforcement path, define `warn`, `block`, and `auto-delegate` semantics, and cover Discord plus at least one non-Discord channel in tests. If maintainers prefer plugin ownership, the missing hook/runtime contract should be designed first rather than closing this as ClawHub-only.
