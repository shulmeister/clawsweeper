## Summary

Keep #13271 open. Current main still does not implement a per-channel tool-call budget or the requested `warn`/`block`/`auto-delegate` overflow behavior. Adjacent features exist, including Discord per-channel allow/deny tool policy, global/per-agent loop detection, plugin `before_tool_call` blocking, and `sessions_spawn`, but they do not combine into first-class per-channel, per-turn budget enforcement.

## Best Possible Solution

Keep this open until OpenClaw adds an explicit product/API path for per-channel, per-turn tool-call budgeting. A likely implementation would add documented channel/agent config, pass reliable channel/request context into the tool-call enforcement path, and test `warn`, `block`, and `auto-delegate` behavior across at least Discord and a non-Discord channel. If maintainers prefer plugin ownership, the missing core hook/runtime contract should be designed first rather than closing this as ClawHub-only today.
