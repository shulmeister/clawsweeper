## Summary

PR #63031 should close as ClawHub/community-plugin scope. The contribution is useful Databricks provider work, but it is optional service-specific provider functionality, already implemented as a native plugin shape, and current main exposes the provider/plugin seams needed to publish it outside the core repository.

## Best Possible Solution

Keep OpenClaw core unchanged for this optional Databricks provider and move the implementation to a maintained ClawHub/npm native provider plugin. If publishing exposes a missing reusable provider SDK, onboarding, or manifest seam, open a narrower OpenClaw PR for that generic API/docs improvement rather than bundling Databricks-specific behavior in core.
