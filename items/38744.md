## Summary

Keep PR #38744 open. Current main still does not implement this PR's native iOS ElevenLabs realtime WebSocket STT path: iOS Talk Mode continuous and push-to-talk capture remain Apple SFSpeechRecognizer-backed. Main does have ElevenLabs Scribe v2 Realtime through the gateway/plugin realtime transcription surface for Voice Call, plus browser OpenAI Realtime docs, but those are adjacent surfaces and do not supersede the native iOS Talk Mode feature. Existing review comments also identify unresolved reliability blockers in the PR branch.

## Best Possible Solution

Keep the PR open for maintainer/product judgment. If this feature proceeds, rebase it onto current main and decide whether iOS Talk Mode should use a native ElevenLabs STT client or a new gateway/native realtime transcription protocol. Before merge, address the existing P1 reliability findings, keep Apple Speech fallback robust, persist/settings-test the STT controls, and add focused iOS coverage or manual device verification.
