## Summary

Keep PR #38744 open. Current main has ElevenLabs Scribe v2 Realtime as a gateway/plugin realtime transcription provider for Voice Call, but it does not implement the PR's native iOS Talk Mode WebSocket STT path or the proposed iOS STT settings. The PR is also not cleanly superseded by the current browser WebRTC realtime Talk work or related issues, and the PR discussion still contains unresolved reliability findings.

## Best Possible Solution

Keep the PR open for maintainer/product judgment. If the feature proceeds, rework it around a robust native iOS Talk Mode realtime STT design or a gateway/native seam that lets companion apps consume an appropriate realtime transcription or realtime voice surface. Address WebSocket readiness, committed-transcript waiting for PTT, fallback permissions/recovery, iOS settings, docs, and focused tests before merge.
