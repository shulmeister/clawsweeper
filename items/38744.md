## Summary

Keep #38744 open. Current main still does not implement ElevenLabs realtime WebSocket STT for iOS Talk Mode; the existing ElevenLabs Scribe v2 Realtime support is a gateway/Voice Call streaming provider, not the iOS app microphone path. The PR is still relevant, but the provided review discussion identifies unresolved reliability issues, so it is not conservative to close as implemented, superseded, or ClawHub-only.

## Best Possible Solution

Keep the PR open for maintainer/product judgment. If the feature proceeds, rework it around a robust iOS Talk Mode realtime STT design: confirmed WebSocket readiness, explicit committed-transcript waiting for PTT, Apple Speech permission/fallback recovery, and alignment with the existing gateway realtime-transcription or realtime-talk provider surfaces where possible.
