## Summary

Keep PR #38685 open. Current main documents Telegram multi-account routing and the individual Telegram group access gotchas, but it still does not include the requested same-group Telegram multi-agent walkthrough. The PR is useful docs work, and the author addressed early review feedback, but the latest sample still needs a safer explicit group scope before it should merge.

## Best Possible Solution

Keep the PR open and request a small docs revision. Preserve the dedicated same-group Telegram multi-agent walkthrough in `docs/concepts/multi-agent.md`, but update the config to scope the intended Telegram group explicitly, preferably under each `channels.telegram.accounts.<id>.groups` entry in the multi-account setup, while keeping the corrected `groupAllowFrom` sender-ID wording and account-level `groupPolicy` precedence notes.
