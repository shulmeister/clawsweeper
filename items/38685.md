## Summary

Keep PR #38685 open. Current main still lacks a dedicated Telegram same-group, multi-bot, multi-agent setup example; the existing docs cover Telegram account routing and Telegram group access rules separately. The PR is useful docs work, but should be revised before merge so each participating Telegram account scopes the intended group under `channels.telegram.accounts.<id>.groups`.

## Best Possible Solution

Keep the PR open and request a focused docs revision. Preserve the same-group Telegram multi-agent walkthrough, Privacy Mode remove/re-add guidance, `groupAllowFrom` sender-ID wording, and account-level `groupPolicy` precedence note, but update the config sample so each participating Telegram account explicitly defines the intended group under `channels.telegram.accounts.<id>.groups`. Then merge as a focused docs improvement.
