## Summary

Keep PR #38685 open. Current main documents Telegram multi-account routing and the separate Telegram group access gotchas, but it still does not include the PR's requested same-group Telegram multi-agent setup example. The PR is useful docs work and the author addressed the initial factual review issues, but the current PR sample still needs an explicit account-level Telegram group scope before it should merge.

## Best Possible Solution

Keep the PR open and request a small docs revision. Preserve the dedicated same-group Telegram multi-agent walkthrough in `docs/concepts/multi-agent.md`, but change the sample to scope the intended Telegram group explicitly under each participating `channels.telegram.accounts.<id>.groups` entry. Keep the corrected `groupAllowFrom` sender-ID wording and the account-level `groupPolicy` precedence note.
