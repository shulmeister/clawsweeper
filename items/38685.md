## Summary

Keep PR #38685 open. Current main documents Telegram multi-account routing and the underlying Telegram group access gotchas, but it still does not include the dedicated same-group multi-agent walkthrough this PR proposes. The PR also has an unresolved correctness issue in review: the example is framed around one group while using open group policy without an explicit `channels.telegram.groups` allowlist.

## Best Possible Solution

Keep the PR open for maintainer/contributor follow-up. The best path is to revise the proposed docs example to include explicit Telegram group scoping, likely via `channels.telegram.groups` or account-scoped `channels.telegram.accounts.<id>.groups`, while retaining the useful privacy-mode, re-add, `groupAllowFrom`, and per-account binding guidance.
