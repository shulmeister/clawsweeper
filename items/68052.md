## Summary

Current `main` already gates Feishu reasoning previews on the stored session `reasoningLevel`, and the Feishu dispatcher only exposes reasoning-stream callbacks when that gate resolves to `stream`. Regression tests cover the gate, and the released `v2026.4.23` tag already contains the same implementation.

## Evidence

- **Session-store reasoning gate on main:** `resolveFeishuReasoningPreviewEnabled(...)` loads the Feishu session store and returns true only when the persisted session entry has `reasoningLevel === "stream"`.
