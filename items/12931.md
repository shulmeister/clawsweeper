## Summary

Keep #12931 open. Current main partially helps with an explicit `/new <model>` workflow, but the requested configurable bare `/new`/`/reset` greeting model and suppress-greeting option are still missing. Bare resets still build and run the reset greeting prompt, and reset model override logic only runs when there is trailing reset command text to parse.
