## Summary

Current main still lacks the requested configurable or suppressible bare `/new`/`/reset` greeting behavior. Both auto-reply and gateway agent paths still turn a bare reset into a model-facing Session Startup greeting prompt, and the typed/session schema surface has no `resetGreeting`, `resetGreetingModel`, silent reset, or static acknowledgement option.

## Best Possible Solution

Keep #12931 open for a core session/reset behavior change. The best implementation would add typed and schema-validated session config for reset greeting behavior, support both a configured cheaper greeting model and a no-model silent/static acknowledgement path, wire it through both auto-reply and gateway bare reset paths, preserve `/new <model>` semantics deliberately, and update session/slash-command/config docs plus regression tests for bare `/new`, bare `/reset`, `/new <model>`, BOOTSTRAP.md/startupContext interactions, and model override persistence.
