## Summary

Keep #12931 open. Current main still turns bare `/new` and `/reset` into a model-backed Session Startup greeting prompt, existing `/new <model>` override handling only runs when reset has trailing text, and the typed/schema/docs config surface still lacks a `resetGreeting`, `resetGreetingModel`, silent reset, or static acknowledgement option. The existing maintainer/Codex comment remains accurate against current main SHA 631552c554095b0c520376db9131dac0f5427f94.

## Best Possible Solution

Keep this open for a core session/reset behavior change. The best implementation would add typed and schema-validated config for bare reset greeting behavior, support both a cheaper configured greeting model and a no-model silent/static-ack path, wire it through both `src/auto-reply/reply/get-reply-run.ts` and `src/gateway/server-methods/agent.ts`, document the options in session/slash-command/config docs, and add regression tests for bare `/new`, bare `/reset`, `/new <model>`, startup-context/bootstrap interactions, and existing model override persistence.
