## Summary

Keep #12931 open. Current main still supports explicit `/new <model>` only; it does not provide a config-backed bare `/new` or `/reset` greeting model, and it does not provide a silent/suppressed greeting path. Bare resets still build a reset prompt that asks the model to greet the user and route that prompt into the normal agent run.

## Best Possible Solution

Keep the issue open for a core session/reset implementation decision. A complete fix would add typed and schema-validated config for bare reset greeting behavior, document it, and cover both requested modes: optionally run the bare reset greeting with a configured cheaper model, or suppress the model turn entirely and return no message or a gateway-generated acknowledgement. The implementation likely belongs in the session reset/reply path around `initSessionState`, reset model selection, and `runPreparedReply`, with tests for bare `/new`, bare `/reset`, `/new <model>`, and existing startup-context behavior.
