## Summary

Keep #12931 open. Current main still has only explicit `/new <model>` handling for trailing reset text. Bare `/new` and `/reset` still build a model prompt that asks for a greeting, route that prompt into the normal agent run, and the session config type/schema/docs still expose no `resetGreeting`, `resetGreetingModel`, or silent-reset control.

## Best Possible Solution

Keep the issue open for a core session/reset decision. A complete fix should add typed and schema-validated session config for bare reset greeting behavior, document it, and test both requested modes: using a configured cheaper model for the bare reset greeting, and suppressing the model turn with either no outbound message or a gateway-generated acknowledgement. The likely implementation area is the session reset path around `initSessionState`, `applyResetModelOverride`, `resolveBareSessionResetPromptState`, `runPreparedReply`, and the gateway `agent` bare reset handler, with coverage for bare `/new`, bare `/reset`, `/new <model>`, bootstrap/startup-context behavior, and existing model override persistence.
