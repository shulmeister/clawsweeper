## Summary

Keep open. Current main still has the opposite behavior from this PR: non-Control-UI-visible external-channel runs are gated out of chat/node-session fanout, and the regression test explicitly asserts that suppression. The underlying TUI live-display gap remains plausible, but this PR’s one-line guard removal still needs maintainer/security design because it would also widen node session fanout for hidden external-channel runs.

## Best Possible Solution

Keep the PR open for maintainer review, but do not merge it as-is. The likely product direction is a narrower authorized live-view path for TUI/operator clients, or teaching TUI to subscribe to the existing `session.message`/session-event transcript surface, while preserving the privacy intent of `isControlUiVisible` for general Control UI and node-session fanout.
