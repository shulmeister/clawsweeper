## Summary

Current `main` already covers the issue well enough: legacy config keys now have explicit compatibility handling and repair guidance, `doctor` supports headless repair flags, and the Bedrock AWS SDK default-chain path now marks providers authenticated instead of failing with `No API key found`. The relevant work is on `main` after `v2026.4.23`, so there is no confirmed shipped release tag yet.

## Evidence

- **Legacy Bedrock key kept as deprecated compat alias:** `ModelsConfig` still includes `bedrockDiscovery` with an inline note that it is a deprecated legacy alias kept so doctor/runtime fallbacks can read older configs until migration completes.
  - file: [src/config/types.models.ts:143](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/config/types.models.ts#L143)
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Bedrock legacy config is migrated into plugin-owned discovery config:** `migrateAmazonBedrockLegacyConfig` removes `models.bedrockDiscovery` and moves or merges it into `plugins.entries.amazon-bedrock.config.discovery`, with tests covering both direct move and merge behavior.
  - file: [extensions/amazon-bedrock/config-compat.ts:62](https://github.com/openclaw/openclaw/blob/b13545355d42701a1b2a5838b18101777c1bd808/extensions/amazon-bedrock/config-compat.ts#L62)
  - sha: [b13545355d42](https://github.com/openclaw/openclaw/commit/b13545355d42701a1b2a5838b18101777c1bd808)
- **Slack legacy streaming and allow aliases have explicit repair rules:** The Slack doctor contract flags legacy scalar `streaming`/`streamMode` usage and `channels.<id>.allow`, and its normalizer rewrites them to `streaming.*` and `enabled`.
  - file: [extensions/slack/src/doctor-contract.ts:58](https://github.com/openclaw/openclaw/blob/b13545355d42701a1b2a5838b18101777c1bd808/extensions/slack/src/doctor-contract.ts#L58)
  - sha: [b13545355d42](https://github.com/openclaw/openclaw/commit/b13545355d42701a1b2a5838b18101777c1bd808)
- **Validation now surfaces legacy config issues before raw schema fallout:** `validateConfigObjectRaw` collects plugin/channel legacy issues first and returns those targeted messages before running schema validation, which is the path that turns legacy config into actionable guidance instead of a raw validation trace.
  - file: [src/config/validation.ts:620](https://github.com/openclaw/openclaw/blob/b13545355d42701a1b2a5838b18101777c1bd808/src/config/validation.ts#L620)
  - sha: [b13545355d42](https://github.com/openclaw/openclaw/commit/b13545355d42701a1b2a5838b18101777c1bd808)
- **Gateway startup error is now human-readable and points to doctor repair:** Startup invalid-config handling formats explicit issue lines and appends `Run "openclaw doctor --fix" to repair, then retry.` instead of surfacing a raw parser/schema stack.
  - file: [src/gateway/server-startup-config.ts:214](https://github.com/openclaw/openclaw/blob/b13545355d42701a1b2a5838b18101777c1bd808/src/gateway/server-startup-config.ts#L214)
  - sha: [b13545355d42](https://github.com/openclaw/openclaw/commit/b13545355d42701a1b2a5838b18101777c1bd808)
- **Headless doctor repair flags are wired in CLI:** The doctor command currently exposes `--yes`, `--repair`/`--fix`, and `--non-interactive`, and passes them through to the doctor command implementation for unattended repair flows.
  - file: [src/cli/program/register.maintenance.ts:20](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/cli/program/register.maintenance.ts#L20)
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **AWS SDK default-chain auth now marks Bedrock-style providers authenticated:** When auth resolves as `aws-sdk` with no explicit API key, the embedded auth controller now tries provider runtime auth and otherwise injects the `__aws_sdk_auth__` sentinel so PI auth storage treats the provider as configured; dedicated tests cover the IMDS / instance-role path.
  - file: [src/agents/pi-embedded-runner/run/auth-controller.ts:371](https://github.com/openclaw/openclaw/blob/b13545355d42701a1b2a5838b18101777c1bd808/src/agents/pi-embedded-runner/run/auth-controller.ts#L371)
  - sha: [b13545355d42](https://github.com/openclaw/openclaw/commit/b13545355d42701a1b2a5838b18101777c1bd808)
- **Issue discussion matched the Bedrock auth root cause:** The follow-up triage comment linked this regression to `#62995` and `#62009`; that diagnosis lines up with the current `auth-controller` fix on `main` that propagates runtime auth / sentinel state for AWS SDK auth.
  - command: `GitHub context comment 4274828282`
