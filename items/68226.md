## Summary

Keep open. Current main has a persisted/derived session start timestamp, but the user-facing `/status` and `session_status` footer still only renders the session key plus last-updated age. The requested duration or absolute start time is a core status-surface feature and is not implemented on main.

## Best Possible Solution

Keep the issue open and implement this in the core status path: derive the session start from `SessionEntry.sessionStartedAt` with the existing transcript-header fallback for legacy rows, pass it through `buildStatusText()` into `buildStatusMessage()`, render either `started <timestamp>` or compact `duration <age>` on the `🧵 Session:` line for both `/status` and `session_status`, and add focused renderer/tool tests plus any user-facing docs update. Coordinate with related open time/status work such as #57442 or #51683 if maintainers choose one canonical thread.
