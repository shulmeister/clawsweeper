## Summary

Keep open. Current main has a real post-execution/pre-model tool-result middleware path, but it is explicitly restricted to bundled plugins and there is still no public `tool_result_before_model`, `before_context_send`, or equivalent external plugin hook. The related closed issue #34144 and closed unmerged PR #14544 do not supersede this request, and open #62123 is overlapping coordination work rather than a clear replacement for this synchronous same-turn pre-LLM policy boundary.

## Best Possible Solution

Keep this issue open until OpenClaw either lands a documented, trust-gated public post-execution/pre-model tool-result mutation hook, or maintainers explicitly decide this capability must remain bundled-only. A complete implementation should define ordering relative to `before_tool_call`, bundled `registerAgentToolResultMiddleware`, `after_tool_call`, `tool_result_persist`, and `before_message_write`; specify sync/async and failure semantics; update public plugin SDK docs; and add runtime tests proving the model-visible result is the sanitized result.

## Risks / Open Questions

- Exposing model-visible tool-result rewriting to external plugins is a high-trust API/security decision; maintainers need to define origin/trust policy, ordering, sync versus async semantics, and failure behavior.
- There is meaningful overlap with #62123, so any implementation should coordinate the synchronous pre-model sanitizer with the proposed async transform path rather than adding competing hook surfaces.
