## Summary

Keep open. Current main still lacks the requested public `tool_result_before_model` plugin hook or equivalent public post-execution/pre-model result mutation hook. OpenClaw has a real same-turn tool-result middleware path for Pi/Codex, but it remains a trusted bundled-plugin-only seam, and the public hooks still expose only pre-call param/blocking, post-call observation, and transcript persistence mutation.

## Best Possible Solution

Keep the issue open until OpenClaw either lands a documented, trust-gated public API for post-execution/pre-model tool-result rewriting, or maintainers explicitly decide that this capability must remain bundled-only. A complete solution should define ordering relative to `before_tool_call`, `after_tool_call`, bundled tool-result middleware, and `tool_result_persist`; specify sync/async and failure semantics; and add runtime tests proving the model receives only the sanitized result.
