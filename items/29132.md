## Summary

Keep open. Current main still does not expose the requested public pre-LLM tool-result sanitization hook for plugin authors. OpenClaw has a pre-model tool-result middleware path, but it is explicitly restricted to bundled/trusted plugins; the public hooks remain persistence, observation, prompt-context, or inbound-dispatch hooks, none of which provide the requested same-turn tool-result rewrite boundary. The Agent14 comment usefully documents a wrapper-plugin workaround and production validation offer, but that workaround is not equivalent to a first-class core plugin API seam.

## Best Possible Solution

Keep the issue open until OpenClaw either adds a documented, trust-gated plugin API for same-turn pre-model tool-result rewriting or records an explicit maintainer decision that this capability remains bundled-only. A complete implementation should document ordering relative to `after_tool_call` and `tool_result_persist`, define failure policy, and add Pi/Codex coverage proving the model only sees the sanitized result.
