## Summary

Current main and v2026.4.24 already address the central failure mode in #72144: multi-agent QMD startup can no longer spawn parallel `qmd embed` model loads, periodic embeds are staggered, embed failures back off, and per-agent QMD caches reuse the shared model directory. The remaining desire for a complete embed skip/precheck overlaps the narrower open QMD embed-control reports (#63626/#67113), but the reported parallel boot GGUF memory storm is fixed.

## Best Possible Solution

Close this item and keep the shipped QMD hardening: serialized embed execution, shared model cache symlinks, staggered timers, and embed-failure backoff. Track any remaining request for a complete `qmd embed` opt-out or pre-spawn up-to-date check under the narrower existing QMD embed-control reports (#63626/#67113).
