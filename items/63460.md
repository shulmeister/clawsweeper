## Summary

Keep open. Current main does not implement the PR's broader Discord markdown-context preservation: Discord chunking still only balances fenced code blocks plus a special reasoning-italics wrapper, table conversion still hardcodes an empty blockquote prefix, and the related Discord send/draft paths still use that old table conversion. The work remains potentially useful, but the branch should be narrowed because the provided PR context shows it now contains many unrelated release, CI, plugin, model-list, and live-test changes.

## Best Possible Solution

Keep the PR open as an implementation candidate, but ask for a rebased, narrowly scoped branch that contains only the Discord/core markdown chunking changes, resolves the prior review edge cases, and adds focused tests for blockquote continuation, table header repetition, table plus blockquote conversion, and general inline-marker balancing. If maintainers prefer smaller pieces, split the Discord-specific chunker fix from any shared core chunker or table-conversion API changes.
