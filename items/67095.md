## Summary

Qiniu provider support is a useful integration, and the PR is already shaped mostly as a provider plugin. On current main, optional provider additions without maintainer ownership should live as external ClawHub/npm plugins rather than becoming default bundled OpenClaw plugins. The current SDK and docs provide the needed provider/auth/catalog seams, and the PR does not identify a missing core API.

## Best Possible Solution

Keep OpenClaw core unchanged for this PR and move the Qiniu implementation into a separately maintained provider plugin published through ClawHub or npm. Reopen core discussion only if the external plugin hits a concrete SDK/API gap or a maintainer explicitly sponsors Qiniu as an official bundled provider.
