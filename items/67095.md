## Summary

Qiniu support is a useful optional model-provider integration, but current OpenClaw architecture and docs route new unsponsored providers through external plugins published via ClawHub/npm. The PR is already shaped as a provider plugin and does not identify a missing core SDK/API seam, while current main does not ship Qiniu as a bundled provider.

## Best Possible Solution

Keep OpenClaw core unchanged for this PR. Move the Qiniu implementation into a separately maintained provider plugin published through ClawHub or npm, using the existing provider plugin SDK. Reopen core discussion only if external packaging hits a concrete SDK/API gap or a maintainer explicitly sponsors Qiniu as an official bundled provider.
