## Summary

Keep open. Current `main` has partial restart/recovery work, but the issue's core claim is still not fully resolved: OpenClaw now has service-level auto-restart guidance, channel health-monitor restarts, startup session-key migration, and post-restart subagent orphan recovery, yet current code and docs still explicitly document process-restart recovery gaps for active runs and pending subagent work.
