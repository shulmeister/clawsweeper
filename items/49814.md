## Summary

Keep open. Current main does not implement this PR’s requested sandbox-level `evaluateEnabled` support, and the proposed PR change is still incomplete because the sandbox runtime path does not carry or consume that field.

## Evidence

- **Sandbox schema still rejects the field:** `SandboxBrowserSchema` on current `main` does not define `evaluateEnabled`; the sandbox browser object only includes `enabled`, ports, `headless`, `enableNoVnc`, `allowHostControl`, `autoStart`, and `binds`. That means `agents.defaults.sandbox.browser.evaluateEnabled` is not accepted by the sandbox runtime schema on `main`.
  - file: [src/config/zod-schema.agent-runtime.ts:213](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/src/config/zod-schema.agent-runtime.ts#L213)
  - command: `nl -ba src/config/zod-schema.agent-runtime.ts | sed -n '210,228p'`
  - sha: [320d52a23e41](https://github.com/openclaw/openclaw/commit/320d52a23e41a326287826219a20f1f29f334554)
- **Sandbox config type still has no slot for it:** `SandboxBrowserConfig` on current `main` has no `evaluateEnabled` property, so even if schema acceptance were added, the resolved sandbox config shape still would not model the setting.
  - file: [src/agents/sandbox/types.ts:32](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/src/agents/sandbox/types.ts#L32)
  - command: `nl -ba src/agents/sandbox/types.ts | sed -n '24,60p'`
  - sha: [320d52a23e41](https://github.com/openclaw/openclaw/commit/320d52a23e41a326287826219a20f1f29f334554)
- **Sandbox resolver does not propagate it:** `resolveSandboxBrowserConfig()` returns `enabled`, image/network/ports, `headless`, `enableNoVnc`, `allowHostControl`, `autoStart`, timeout, and `binds`, but no `evaluateEnabled`. The PR as described would still be incomplete against current runtime behavior unless this function changes too.
  - file: [src/agents/sandbox/config.ts:129](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/src/agents/sandbox/config.ts#L129)
  - command: `nl -ba src/agents/sandbox/config.ts | sed -n '129,185p'`
  - sha: [320d52a23e41](https://github.com/openclaw/openclaw/commit/320d52a23e41a326287826219a20f1f29f334554)
- **Sandbox runtime reads only top-level browser config:** `resolveSandboxContext()` derives `evaluateEnabled` from `params.config?.browser?.evaluateEnabled` and falls back to `DEFAULT_BROWSER_EVALUATE_ENABLED`; it does not read from `resolvedCfg.browser`. So a sandbox-specific setting would still have no effect at runtime.
  - file: [src/agents/sandbox/context.ts:174](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/src/agents/sandbox/context.ts#L174)
