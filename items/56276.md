## Summary

Keep open. Current `main` still shows host context only in the TUI header via the connection URL; the footer/status bar requested in #56276 renders agent, session, model, mode flags, and token counts, but no hostname. The public TUI docs describe the same footer shape, so this feature request is not implemented on `main` and is not a conservative auto-close candidate.

## Evidence

- **TUI footer still has no hostname field:** `updateFooter()` builds the footer from agent, session, model, think/fast/verbose/reasoning, and token counts only. There is no hostname/host segment in `footerParts`.
  - file: [src/tui/tui.ts:845](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/tui/tui.ts#L845)

  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Host context is only present in the header:** `updateHeader()` renders `client.connection.url` alongside the agent and session, which means the host is visible in the header, not the footer/status bar requested by the issue.
  - file: [src/tui/tui.ts:591](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/tui/tui.ts#L591)
