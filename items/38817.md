## Summary

Keep open. Current main still strips unsupported tool-schema keywords only when the resolved model compat declares `unsupportedToolSchemaKeywords`; the bundled Volcengine/Kimi catalog does not attach that compat metadata or a provider compat hook. The linked fix PRs #38883 and #46887 were closed unmerged, so they do not supersede the issue.

## Best Possible Solution

Implement a provider-owned Volcengine/Kimi compat patch, preferably by adding `unsupportedToolSchemaKeywords` to the affected Volcengine/Kimi model definitions or by adding a Volcengine `normalizeResolvedModel`/`contributeResolvedModelCompat` hook. Add a focused regression test showing `minLength`/`maxLength`/`minItems` are stripped for `volcengine`, `volcengine-plan`/Ark, and Kimi model IDs while unaffected providers keep validation keywords.
