## Summary

Keep open. Current main still only strips unsupported tool-schema keywords when model compat metadata declares them, and the bundled Volcengine/Kimi catalog does not attach `unsupportedToolSchemaKeywords`. The two related fix PRs in the provided GitHub context, #38883 and #46887, are closed unmerged, so they do not supersede or prove this bug fixed.

## Best Possible Solution

Keep the issue open and implement the missing bundled-provider behavior, preferably by adding Volcengine/Kimi model compat metadata such as `unsupportedToolSchemaKeywords` in the Volcengine catalog path, plus a focused regression test proving bundled `volcengine` and `volcengine-plan` Kimi models strip `minLength`/`maxLength`/`minItems` before tool schemas are sent.
