## Summary

Keep #38817 open. Current main still only strips unsupported tool-schema keywords when resolved model compat declares `unsupportedToolSchemaKeywords`; the bundled Volcengine/Kimi catalog and provider do not add that compat or a provider hook, and the two linked fix PRs were closed unmerged.

## Best Possible Solution

Implement this as provider-owned compatibility rather than a hard-coded core provider branch: add `unsupportedToolSchemaKeywords` to the affected bundled Volcengine/Ark/Kimi model definitions or have the Volcengine provider contribute the compat via `normalizeResolvedModel`/`contributeResolvedModelCompat` or `normalizeToolSchemas`. Add focused regression coverage for `volcengine`, `volcengine-plan`/Ark, and Kimi model IDs while proving unaffected providers retain validation keywords.
