## Summary

Keep this issue open. Current main still has a plausible Control UI blank-screen crash path matching SnowSky1's linked analysis: global app render and cron edit/render paths still dereference cron job payloads without guarding malformed persisted rows, while the linked fix candidate #54552 was closed unmerged.

## Best Possible Solution

Keep this issue open. The best maintainer path is to reproduce against current main with either the reporter's snapshot or a synthetic malformed/stale cron row, then add a focused null-safe guard or normalization layer before Control UI render/edit paths plus a jsdom regression test. If persisted cron shape is the root cause, repair or filter malformed rows in cron service or doctor code while keeping the Control UI resilient to bad snapshots.
