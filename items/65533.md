## Summary

Keep open. Current main now parses OpenRouter/MiniMax `reasoning_details` into separate thinking blocks, but I did not find evidence that plain-text assistant replay normalizes that stored signature back to OpenRouter’s documented `reasoning` field. The replay path still depends on `@mariozechner/pi-ai` 0.70.2 using the raw `thinkingSignature` as the outbound assistant field name, and there is no repo regression test covering the plain-text MiniMax replay payload described in #65533.

## Evidence

- **Current main stores OpenRouter `reasoning_details` as the thinking signature:** `getCompletionsReasoningDeltas` maps `reasoning_details[*].type === "reasoning.text"` to a thinking block with `signature: "reasoning_details"`. Only when no such detail was used does it fall back to `reasoning_content`, `reasoning`, or `reasoning_text`.
