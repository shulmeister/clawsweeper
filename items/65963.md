## Summary

Keep #65963 open. Current main partially addresses the session-sprawl side by using a daily default cadence and deleting/scrubbing dreaming narrative sessions, but the central requested operator knob is still not implemented: memory-core does not expose a public `dreaming.model` config key and the narrative subagent path does not pass any provider/model override.

## Best Possible Solution

Keep the issue open or split it into narrower follow-ups. The most direct product/code path is to add a documented `plugins.entries.memory-core.config.dreaming.execution.defaults.model` or simpler `dreaming.model` surface, validate it in `extensions/memory-core/openclaw.plugin.json`, route it through the existing `resolveMemoryDreamingConfig` execution config, pass provider/model into `generateAndAppendDreamNarrative` subagent runs, and document the required `plugins.entries.memory-core.subagent.allowModelOverride` / `allowedModels` trust policy. Separate follow-ups can decide whether `retainDays`, `batchSize`, or a `sessions purge --kind dreaming` CLI remains necessary after the current immediate cleanup behavior.
