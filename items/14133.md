## Summary

Keep #14133 open. Current main still has only the generic Cron Jobs surface in Control UI: no Reminders tab/route, no reminder-specific render path, no cron job type/tags/display metadata in the wire or stored job shape, and no backend `cron.list` filter that can separate one-shot reminder jobs from ordinary cron jobs. The prior same-day Codex review comment correctly identified the same missing core UI/protocol work, and I found no current-main change that supersedes it.

## Best Possible Solution

Keep this open as a core Control UI plus Gateway cron protocol enhancement. The likely path is an additive cron classification/display metadata field, reminder-producing cron paths that set it, typed `cron.list` filtering for reminders versus non-reminders, Cron tab hiding/filtering for reminder jobs, and a Reminders tab showing cleaned reminder text, fire time, channel/recipient, and delete actions with protocol/service/UI tests and docs updates.
