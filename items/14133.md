## Summary

Keep #14133 open. Current main still does not implement a dedicated Reminders tab in Control UI/Mission Control, nor the cron metadata/tag/type contract needed to distinguish one-shot reminder jobs from other cron jobs. The existing issue discussion already contains a 2026-04-26 Codex review comment from steipete keeping this open, and the current checkout at 04e08cea confirms the same missing product/API surface.

## Best Possible Solution

Keep this issue open as a core Control UI and cron API enhancement. A good fix would add an additive cron classification/metadata contract, make reminder-producing paths set it, update cron listing/filtering and tests, then add a `/reminders` Control UI tab with card rendering, human-readable reminder text, fire time, channel/recipient display, delete actions, and Cron tab filtering or hiding.
