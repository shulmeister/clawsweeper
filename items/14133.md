## Summary

Keep open. Current main still does not implement a Reminders tab in Control UI, nor the cron metadata/tag/type contract needed to distinguish one-shot reminders from other cron jobs. Existing cron and Apple Reminders support is related, but it does not satisfy the requested separate card UI or filtering behavior.

## Best Possible Solution

Keep tracking this as a core Control UI/API enhancement. A good implementation would add an additive cron classification field or metadata contract, update cron creation/listing/filtering and tests, have reminder-producing paths set that field, then add a `/reminders` Control UI tab with card rendering, human-readable reminder extraction, channel/recipient display, delete actions, and Cron tab filtering/hiding.
