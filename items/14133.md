## Summary

Keep #14133 open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still lacks a Reminders tab/route, reminder-specific cron classification/display metadata, and cron.list filtering that could hide reminder one-shot jobs from the generic Cron tab. The prior maintainer-side Codex comment already kept the issue open, and the relevant files have not changed since that reviewed SHA.

## Best Possible Solution

Keep this open as a core Control UI plus cron protocol enhancement. The best implementation is an additive cron classification/display metadata contract, reminder-producing paths that set it, typed list filtering, Cron tab filtering or hiding for reminder jobs, and a Reminders tab with human-readable reminder text, fire time, channel/recipient, and delete actions, backed by UI/protocol/service tests and docs updates.
