## Summary

Keep #69978 open. Current main still has the TUI streaming watchdog as a fixed event-handler option with the 30s default, no `/watchdog` TUI command or documented command surface, and no per-run warning dedupe. Related #68596 and #69081 are active and relevant, but they do not fully supersede this issue because #69978 combines a concrete TUI runtime command with duplicate-warning suppression for the same run.

## Best Possible Solution

Keep this issue open as a focused TUI UX enhancement. The best implementation would add a TUI-owned watchdog setting that can be shown/updated by `/watchdog`, persist the local preference, pass a dynamic watchdog value into the event-handler layer, and add a `warnedRunIds`-style dedupe guard cleared on run finalization/termination/session changes. Coordinate the timeout/config piece with #68596 and the broader status-desync behavior with #69081, but do not close #69978 unless a canonical tracker explicitly covers both the command and duplicate-warning suppression.
