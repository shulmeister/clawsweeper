## Summary

Keep open. The RFC asks for a new core/plugin SDK capability, and current main does not yet expose plugin session extension registration or gateway patch routing for plugin-owned session state. The related PR #71731 is an open docs/RFC packet and explicitly does not implement the runtime hooks.

## Evidence

- **Plugin API lacks requested registration surface:** OpenClawPluginApi lists the current registration methods from registerTool/registerHook/registerGatewayMethod through memory/context/harness registrations, but there is no registerSessionExtension or equivalent session-extension API.
  - file: [src/plugins/types.ts:2051](https://github.com/openclaw/openclaw/blob/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54/src/plugins/types.ts#L2051)
  - command: `nl -ba src/plugins/types.ts | sed -n '1980,2265p'`

- **Gateway session patch schema is fixed to core fields:** SessionsPatchParamsSchema accepts a fixed set of core session fields such as label, thinkingLevel, model, sendPolicy, and groupActivation. It has no plugin envelope, plugin id, action key, or arbitrary extension payload.
  - file: [src/gateway/protocol/schema/sessions.ts:131](https://github.com/openclaw/openclaw/blob/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54/src/gateway/protocol/schema/sessions.ts#L131)
