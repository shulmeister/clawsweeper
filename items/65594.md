## Summary

Current main already resolves this via the documentation path proposed in the issue: `pnpm build` remains the runtime build, `pnpm ui:build` remains separate, and the repo plus shipped docs now explicitly tell users to run both for a complete local/packageable `dist/` including `dist/control-ui`.

## Evidence

- **Build remains split by design:** The root scripts still define `build` as `node scripts/build-all.mjs`, while `ui:build` is a separate command. That matches the issue's root-cause analysis and shows the chosen fix path is documentation rather than merging the builds.
