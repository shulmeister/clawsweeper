## Summary

Keep open. Current main still contains the exact sessions_spawn failure mode: the ACP-only `streamTo` parameter is exposed in the shared schema when ACP is available, then `runtime="subagent"` calls with `streamTo:"parent"` are rejected before reaching `spawnSubagentDirect`. Open PR #72331 is the current maintainer-owned candidate fix, but it is not merged, so the issue should not be closed as implemented or obsolete yet.

## Best Possible Solution

Keep this issue open until PR #72331 or an equivalent narrow fix lands. The fix should live in `src/agents/tools/sessions-spawn-tool.ts`: treat `streamTo` as ACP-only request data, ignore or strip it for `runtime="subagent"`, preserve ACP pass-through to `spawnAcpDirect`, and update `src/agents/tools/sessions-spawn-tool.test.ts` to replace the rejecting subagent case with tolerant regression coverage.
