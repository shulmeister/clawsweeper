## Summary

Keep #10737 open. Current main still does not implement channel-layer auto-acknowledgment or native typing for the legacy `channels.imessage` path; BlueBubbles has typing support, but that is a separate recommended iMessage channel rather than a fix for this requested `channels.imessage.autoAck` behavior.

## Best Possible Solution

Keep the issue open until maintainers either implement channel-level feedback for the bundled legacy iMessage plugin or explicitly decline it. A concrete fix likely belongs in `extensions/imessage`: add typed config/schema/docs for auto-ack, wire the inbound monitor through `createChannelReplyPipeline` with typing callbacks or a pre-dispatch ack path, feature-detect any supported `imsg` typing RPC, fall back to the requested short text acknowledgment when configured, and add regression tests for DM-only behavior plus no duplicate final replies. BlueBubbles should remain documented as the recommended iMessage channel, but it should not close this `channels.imessage` request.
