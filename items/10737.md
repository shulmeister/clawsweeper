## Summary

Keep #10737 open. Current main still lacks legacy `channels.imessage` channel-layer auto-acknowledgment or native typing feedback, and the issue now has an explicit maintainer comment saying to keep it open despite BlueBubbles typing support.

## Best Possible Solution

Keep the issue open until maintainers either implement or explicitly decline legacy `channels.imessage` feedback. A concrete fix should add typed config/schema/docs for `autoAck` or native typing with text fallback, send it after access/mention gating but before `dispatchInboundMessage`, honor `dmOnly`, and add regression tests proving the ack is not re-ingested as a final reply. BlueBubbles can remain the recommended iMessage path, but it does not close this legacy iMessage request.
