## Summary

Keep #10737 open. Current main still has no legacy iMessage auto-ack or native typing wiring: the iMessage config/schema has no requested field, the inbound monitor builds the reply pipeline without typing callbacks, and the outbound helper only calls the imsg send RPC. BlueBubbles already has automatic typing indicators, but that is a separate recommended iMessage path, not an implementation for channels.imessage.

## Best Possible Solution

Keep the issue open until the bundled legacy iMessage plugin either implements channel-level feedback or maintainers explicitly decline it. A likely fix belongs in extensions/imessage: add typed config/schema/docs, wire createChannelReplyPipeline with typingCallbacks or an auto-ack pre-dispatch path, call a supported imsg typing RPC only when available, fall back to the requested short text acknowledgment when configured, and add regression tests for DM-only behavior and no duplicate final replies. BlueBubbles can remain documented as the recommended iMessage path with existing typing support, but it should not be treated as resolving channels.imessage.
