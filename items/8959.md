## Summary

Keep #8959 open. Current main still has only a portable single-select presentation contract; it does not expose a cross-platform multi-select/batch-submit message-tool API with min/max values or selected-value-array callbacks. The related dialog PR #18286 was closed unmerged, and the existing maintainer/Codex issue comment already reached the same keep-open conclusion.

## Best Possible Solution

Keep the issue open until OpenClaw adds an additive cross-platform message presentation contract for multi-select/batch-submit, likely a new `multiselect` block or a backward-compatible `select` extension with `minValues`, `maxValues`, and structured selected-value array callbacks. The work belongs in core/shared presentation first, with updates to `src/agents/tools/message-tool.ts`, `src/interactive/payload.ts`, plugin SDK exports, fallback rendering, Discord/Slack/Telegram renderers and interaction dispatch behavior, public docs at https://docs.openclaw.ai/plugins/message-presentation, and focused contract tests. A ClawHub/plugin-only answer would not expose the requested portable message-tool API.
