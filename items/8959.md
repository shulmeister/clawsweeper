## Summary

Keep #8959 open. Current main still lacks the requested cross-platform message-tool multi-select/batch-submit contract: shared presentation supports only a single `select` block shape, docs describe one selected value, Telegram degrades selects to independent buttons, Slack renders shared selects as `static_select`, and Discord's native min/max plumbing is not exposed through the shared presentation schema. The related dialog PR #18286 was closed unmerged, and the existing maintainer/Codex comment already reached the same keep-open conclusion.

## Best Possible Solution

Keep the issue open until OpenClaw adds an additive cross-platform message presentation contract for multi-select/batch-submit, likely a new `multiselect` block or backward-compatible `select` extension with `minValues`, `maxValues`, and structured selected-value array callbacks. The work should update `src/agents/tools/message-tool.ts`, `src/interactive/payload.ts`, agent runtime types, fallback rendering, Discord/Slack/Telegram renderers and interaction dispatch behavior, public docs at https://docs.openclaw.ai/plugins/message-presentation, and focused contract tests. This belongs in core/shared presentation first; a ClawHub/plugin-only answer would not expose the requested portable message-tool API.
