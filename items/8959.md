## Summary

Keep open. Current main has related rich message presentation and Discord-native select support, but the shared message tool still lacks a cross-platform multi-select/batch-submit contract. The related dialog PR #18286 was closed unmerged and is not present on main, so it does not supersede the issue.

## Evidence

- **Shared message tool schema is single select only:** The shared `presentation` schema accepts block types `text`, `context`, `divider`, `buttons`, and `select`; a select block has only `placeholder` and `options`. There is no shared `multiSelect`, `selectMenu`, `minValues`, `maxValues`, or submit/batch callback field.
