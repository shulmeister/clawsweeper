## Summary

Keep #8959 open. Current main still lacks the requested cross-platform message-tool multi-select/selectMenu contract: the shared presentation schema and SDK export only a single-select-shaped block, docs document only that shape, and Slack/Discord/Telegram renderers do not provide portable min/max multi-selection with one structured selected-values callback. The related dialog PR #18286 was closed unmerged and its proposed dialog files are absent from current main; existing wizard/poll multiselect code is adjacent but does not satisfy the message presentation request.

## Best Possible Solution

Keep the issue open until OpenClaw adds an additive portable batch-selection contract in the shared message presentation/message tool surface, exports it through openclaw/plugin-sdk/interactive-runtime, documents it at docs.openclaw.ai/plugins/message-presentation, and implements focused render/callback handling for Slack, Discord, Telegram, and fallback text paths. Existing poll, wizard, provider-native Discord, and Slack event parsing code should be treated as adjacent implementation references, not as the closing implementation.
