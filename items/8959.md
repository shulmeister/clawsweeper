## Summary

Keep #8959 open. Current main still has only a single-select shared message presentation contract, even though Slack/Discord have some provider-native select plumbing. The existing maintainer/Codex comment reached the same conclusion, and the related dialog PR #18286 was closed unmerged rather than shipping this cross-platform message-tool feature.

## Best Possible Solution

Keep the issue open until OpenClaw adds an additive cross-platform message presentation contract for multi-select/batch-submit, likely a new `multiselect` block or a backward-compatible `select` extension with `minValues`, `maxValues`, and structured selected-value array callbacks. The work should update the shared message tool schema, `MessagePresentation` types and normalization, fallback text, Discord/Slack/Telegram renderers and dispatch behavior, docs, and focused contract tests.
