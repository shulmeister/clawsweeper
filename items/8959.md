## Summary

Keep #8959 open. Current main has a shared message presentation contract with buttons and single-select menus plus channel-native adjacent pieces, but it still does not implement the requested cross-platform message-tool multi-select with min/max values and one structured selected-values callback.

## Best Possible Solution

Keep the issue open until OpenClaw adds an additive portable batch-selection contract to the shared message presentation/message tool surface, exports it through openclaw/plugin-sdk/interactive-runtime, documents it on docs.openclaw.ai/plugins/message-presentation, and implements focused render/callback handling for Slack, Discord, Telegram, and fallback text paths. Existing provider-native components and CLI/wizard multiselects should remain implementation references, not treated as completion.
