## Summary

Keep open. Current main already includes the SearXNG provider, docs, and auto-detection, but the Tavily half of this PR is not implemented on main: generic `web_search` for Tavily still only accepts `query` and `count`, and the Tavily request path does not carry any freshness/days parameter.
