## Summary

Keep #37131 open. Current main has partial no-LLM direct invocation through HTTP `/tools/invoke`, Lobster `openclaw.invoke`, and skill `command-dispatch: tool`, but the issue asks for a broader direct tool invocation contract across coding tools, cron payloads, CLI, and generic slash-command surfaces. Those remaining slices are still missing or only tracked by open related items (#63919 for coding tools over HTTP, #18160 for direct cron exec).

## Best Possible Solution

Keep #37131 open as the umbrella product/API request until maintainers either ship a single security-reviewed direct invocation contract across HTTP, cron, CLI, and slash-command surfaces, or split and link canonical narrower trackers for each remaining surface. Treat #63919 and #18160 as active implementation slices, not replacements for the umbrella.
