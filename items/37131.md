## Summary

Keep open. Current main still has only a partial direct-invocation story: `POST /tools/invoke` can execute gateway/plugin tools without an LLM turn, but `resolveGatewayScopedTools` still builds from `createOpenClawTools`, not `createOpenClawCodingTools`, so the requested direct `exec`/`read`/`write` coding-tool path is not implemented. Cron also still accepts only `systemEvent` and `agentTurn` payloads, not a direct `toolCall` payload with explicit tool args. Related PR #63919 usefully tracks part of the missing HTTP coding-tool work, and #18160 tracks the narrower direct cron exec request, but neither is merged/shipped and neither covers the whole remaining umbrella request.

## Best Possible Solution

Keep #37131 open as the umbrella product/API request until maintainers either implement the remaining direct-invocation surfaces or explicitly split it into canonical follow-ups. The likely path is to land a security-reviewed HTTP coding-tool composition for `/tools/invoke`, decide whether slash/CLI direct invocation belongs in core, and add an additive cron payload contract for direct tool calls only if maintainers accept that product direction.
