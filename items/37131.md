## Summary

Keep #37131 open. Current main has partial direct invocation through `POST /tools/invoke`, `/bash`, and skill command dispatch, but the umbrella request is not implemented: generic Gateway direct invoke still does not compose the coding tool factory that owns `exec`/`read`/`write`, cron still has no direct `toolCall`/`exec` payload, and there is no generic `/tool` or `openclaw agent --tool` surface. Related #63919 and #18160 are useful open slices, but they do not supersede the full request.

## Best Possible Solution

Keep #37131 open as the umbrella product/API request until maintainers either ship a security-reviewed direct tool invocation contract across HTTP, cron, slash command, and CLI surfaces, or split the remaining scopes into canonical narrower issues and close this as superseded by those maintainers-approved trackers.
