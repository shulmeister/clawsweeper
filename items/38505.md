## Summary

Keep #38505 open. Current main c7a0d9b1889ca09ab5bfd84cda39a8d50d3d5768 still handles Discord inbound conversation triggers through a create-message listener only; there is no MessageUpdate listener, content-change filter, edit metadata path, or edited-message re-dispatch path. The related open #53654 covers a broader edit/delete request, but it does not supersede this focused edit-to-reprocess issue while the behavior remains missing.

## Best Possible Solution

Keep the issue open and implement the missing behavior in the bundled Discord plugin: add a Carbon MessageUpdateListener, ignore non-content edits, route qualifying edited messages through the existing Discord preflight/inbound worker path with the edited content as authoritative, include edit metadata or replay handling where useful, and add focused tests for mention-added edits, DMs or active threads, duplicate/update replay behavior, and non-content update suppression. Coordinate with #53654 so the edit portion is not implemented twice.
