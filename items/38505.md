## Summary

Keep #38505 open. Current main still wires Discord inbound conversation handling through Carbon message-create events only; there is no Discord message-update listener, registration, handler type, docs entry, or test coverage for reprocessing edited Discord messages. The related broader #53654 is useful context, but it is not an explicit canonical superseder for this narrower edit-trigger report.

## Best Possible Solution

Implement this in the bundled Discord plugin: add a Carbon MessageUpdateListener, register it in the monitor startup path, filter updates whose content did not change, reuse the existing Discord preflight/mention/DM/thread authorization path, and add focused tests for edit-to-mention and edit-to-reprocess behavior. Keep #38505 open until that ships or a maintainer explicitly consolidates it into a canonical edit/delete tracking issue.
