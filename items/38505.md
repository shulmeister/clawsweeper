## Summary

Keep open. Current main still wires Discord inbound conversation handling through Carbon message-create events only; there is no Discord messageUpdate listener, content-change filter, edited-message re-injection path, or edit metadata path. The request is concrete, author is not a maintainer, there are no protected labels, and the related broader #53654 does not make this focused edit-reprocess issue obsolete.

## Best Possible Solution

Implement the missing behavior in the bundled Discord plugin: add a Carbon MessageUpdateListener, ignore non-content edits, reuse the existing Discord preflight/routing/inbound worker path with the edited message as authoritative, and add focused tests for mention-added edits, DMs or active threads, duplicate/update replay behavior, and non-content update suppression.
