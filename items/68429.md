## Summary

Current main still appears to have the reported race: outbound delivery records session metadata and mirrors the sent message, but I found no prewarm/bootstrap step before the first Telegram callback is accepted as a normal inbound turn. The linked fix PR #68433 directly targets this issue but remains open/unmerged, so the bug should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep #68429 open until #68433 or an equivalent maintainer-reviewed fix lands. The likely product/code path is to prewarm or otherwise mark the mirrored outbound session bootstrap-ready immediately after outbound session metadata is recorded for `openclaw message send`, then add a regression test covering proactive Telegram delivery followed by the first callback turn.
