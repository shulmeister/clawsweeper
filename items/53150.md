## Summary

Current main already supports dynamic backend model switching on the Gateway OpenAI-compatible APIs, but via the documented agent-first contract: the request `model` selects an OpenClaw agent target, and `x-openclaw-model` selects the provider/model override for that agent.

## Evidence

## Risks / Open Questions

- Raw provider/model ids in the JSON body are still unsupported by design; callers must use the documented `x-openclaw-model` header for backend model switching.

## Close Comment

Closing from a Codex review. Current `main` already supports dynamic backend model switching on the Gateway OpenAI-compatible API, but the contract is now agent-first rather than raw-provider-in-body: `model` selects the OpenClaw agent target, and `x-openclaw-model` selects the backend provider/model override. The behavior is documented in `docs/gateway/openai-http-api.md:81` and `docs/gateway/openai-http-api.md:237`, implemented in `src/gateway/http-utils.ts:303` and `src/gateway/openai-http.ts:541`, and covered by tests in `src/gateway/openai-http.test.ts:261`. This is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
