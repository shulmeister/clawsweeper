## Summary

Keep open. Current main still supports Discord presence through static config, auto-presence, and the gated Discord message action `setPresence`, but it still does not expose the requested supported code-plugin runtime API `api.runtime.channel.discord.updatePresence()` or an explicitly documented alternative code-plugin seam.

## Best Possible Solution

Keep #29725 open until maintainers either add a typed, documented Discord presence runtime helper for code plugins or explicitly document a different supported code-plugin seam. A durable fix likely needs an additive SDK/runtime contract, Discord implementation wiring to the existing gateway presence path, docs, and contract tests, while preserving the core/plugin boundary.
