## Summary

Current main still lacks the requested `api.runtime.channel.discord.updatePresence()` runtime helper. Discord presence can be set from static config, auto-presence, or the gated `setPresence` message-action path, but no typed code-plugin runtime API exposes dynamic presence updates under `api.runtime.channel.discord`.

## Best Possible Solution

Keep #29725 open until maintainers either add a narrow typed Discord presence runtime helper under the Discord plugin runtime surface or document a supported code-plugin equivalent. A durable implementation likely belongs in `extensions/discord/src/runtime.ts` or a Discord-owned runtime API layer, backed by the existing gateway registry/updatePresence path and covered by action gating, account selection, docs, and tests.
