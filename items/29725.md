## Summary

Keep #29725 open. Current main still has Discord static presence, auto-presence, and a gated `setPresence` message action, but it still does not expose the requested typed `api.runtime.channel.discord.updatePresence()` code-plugin helper or a documented equivalent. The related reports in the provided context cover degraded/auto presence, self-profile work, or session-list behavior, not this remaining plugin runtime API gap.

## Best Possible Solution

Keep #29725 open until maintainers either add a narrow Discord-owned `api.runtime.channel.discord.updatePresence()` helper or document a supported code-plugin equivalent. The likely implementation belongs in the Discord plugin runtime surface, reusing the existing gateway registry and `gateway.updatePresence(...)` path with focused tests and docs for gating, account selection, validation, connection state, and auto-presence interaction.
