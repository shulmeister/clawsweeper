## Summary

Keep open. Current main still has Discord presence plumbing for static config, auto-presence, and the Discord message action `set-presence`, but the requested supported code-plugin API `api.runtime.channel.discord.updatePresence()` is not present in the public plugin runtime contract or the Discord runtime slot.

## Best Possible Solution

Keep #29725 open until maintainers either add a typed Discord runtime helper, such as `api.runtime.channel.discord.updatePresence(...)`, with docs and contract tests, or explicitly decide that plugin code should use a different documented runtime/action seam for dynamic Discord presence updates.
