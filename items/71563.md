## Summary

Keep this PR open. Current main still does not route Telegram `/new` or `/reset` to the control lane, and the PR is a focused implementation candidate. Because these are state-changing session lifecycle commands, maintainer review should decide whether the bypass is safe and should require active-run regression coverage beyond key-resolution tests.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a narrow Telegram change that routes only registered `/new` and `/reset` session lifecycle commands to the control lane, with the existing sequential-key unit tests plus an active-run Telegram middleware regression showing those commands bypass a busy lane without reviving stale reply/finalization races.
