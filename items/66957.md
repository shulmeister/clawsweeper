## Summary

Current main still has the reported bug: `models.mode: "replace"` does not short-circuit implicit provider discovery. The planner resolves implicit providers and merges them with explicit providers before it reads `cfg.models?.mode`, while the current schema help and docs still promise that `replace` uses only configured providers. This is a real product bug, not stale or implemented.

## Best Possible Solution

Keep the issue open. The likely fix belongs in `src/agents/models-config.plan.ts`: branch on `cfg.models?.mode === "replace"` before invoking implicit provider discovery, normalize/enforce secrets for only explicit providers, and add a focused regression test proving the injected `resolveImplicitProviders` callback is not called and implicit plugin providers are excluded in replace mode. Adjacent UI/catalog `models.mode` issues can remain tracked separately unless maintainers consolidate them under one canonical model-catalog contract issue.
