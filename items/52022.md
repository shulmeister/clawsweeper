## Summary

Keep open. Current `main` still compacts home-prefixed skill paths with `"~/" + remainder` and does not normalize Windows backslashes, so mixed-separator prompt paths remain possible. The prompt formatter emits `skill.filePath` verbatim, and the existing test only checks `~/` compaction generally, not Windows separator normalization. The same unfixed logic is also present in the latest release tag `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

## Evidence

- **Current compaction logic still preserves native separators after `~/`:** `compactHomePath()` returns `"~/" + filePath.slice(prefix.length)` with no Windows slash normalization.
