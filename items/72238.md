## Summary

Keep #72238 open. Current main still has the browser plugin's runtime value imports directly from `playwright-core`, and there is no browser-local runtime shim on main. The PR is a focused active fix candidate for open bug #72168, with maintainer Docker proof confirming the direction but explicitly requesting packaged-topology regression coverage before merge.

## Best Possible Solution

Keep #72238 open. The best path is to add a regression test or CI lane that models the packaged topology: no root `playwright-core`, empty `OPENCLAW_PLUGIN_STAGE_DIR`, first gateway start with the browser plugin enabled, automatic staging of `playwright-core`, and a Playwright-backed browser action loading through the staged dependency. Then merge this PR or an equivalent maintainer patch.
