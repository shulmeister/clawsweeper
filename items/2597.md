## Summary

Keep #2597 open. Current main has context visibility and mitigation surfaces, but the requested passive agent-visible `context=X%` field in the system-prompt Runtime line is still missing, and the latest maintainer-authored issue comment explicitly kept it open for that reason.

## Best Possible Solution

Keep the issue open until OpenClaw either implements bounded context-usage plumbing from session/token accounting into embedded-runner runtime params, `RuntimeInfoInput`, and `buildRuntimeLine()`, or a maintainer explicitly decides that explicit `/status`/`session_status`/`/context`, memory flush, context-engine plugins, or community backup/sync workflows are the intended product boundary.
