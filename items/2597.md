## Summary

Keep #2597 open. Current main still has useful mitigations for context loss, including `/status`, `/context`, `session_status`, and pre-compaction memory flush, but the requested passive agent-visible `context=X%` field in the system-prompt Runtime line is not implemented. The latest maintainer-authored issue comment also explicitly kept this open for that reason. The linked `openclaw-workspace-sync` community plugin is a useful recovery/backup workaround, but it does not replace the requested per-turn context-pressure primitive.

## Best Possible Solution

Keep the issue open until OpenClaw either implements bounded context-usage plumbing from session/token accounting into embedded-runner runtime params, `RuntimeInfoInput`, and `buildRuntimeLine()`, or a maintainer explicitly decides that context pressure should remain available only through `/status`, `session_status`, `/context`, memory flush, context-engine plugins, or community backup/sync workflows.
