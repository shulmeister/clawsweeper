## Summary

Keep #2597 open. Current `main` still does not implement the requested passive per-turn `context=X%` field in the system-prompt Runtime line, and the latest maintainer-facing issue comment explicitly kept it open. Existing `/status`, `/context`, memory-flush behavior, UI context metadata, and the community `openclaw-workspace-sync` plugin are useful mitigations, but they do not provide the proposed agent-visible Runtime-line primitive.

## Best Possible Solution

Keep the issue open until OpenClaw either implements bounded context-usage plumbing from the latest reliable session token snapshot into embedded-runner params, `RuntimeInfoInput`, and `buildRuntimeLine()` with focused tests/docs, or a maintainer explicitly decides that the existing `/status`, `/context`, UI warnings, memory/context plugins, and `openclaw-workspace-sync` are the intended product boundary.
