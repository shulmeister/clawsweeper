## Summary

Keep #2597 open. Current main has useful mitigations for context loss, including `/status` context reporting, `session_status`, documented context inspection, context-engine/plugin paths, and pre-compaction memory flush. The issue’s central request is still missing: passive agent-visible `context=X%` in the system prompt Runtime line on every turn. A maintainer-authored Codex review comment on the issue also explicitly kept it open for that reason.

## Best Possible Solution

Keep the issue open until OpenClaw either implements bounded context usage plumbing from session/token usage into embedded-runner runtime params, `RuntimeInfoInput`, and `buildRuntimeLine()`, or a maintainer explicitly decides that context pressure should remain available only through `/status`, `session_status`, memory flush, hooks, context-engine plugins, or community backup/sync workflows.
