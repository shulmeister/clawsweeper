## Summary

Keep #2597 open. Current main still has explicit context-visibility surfaces such as `/status`, `session_status`, `/context`, and cached token snapshots, but it does not implement the requested passive agent-visible `context=X%` field in the system-prompt Runtime line. The latest provided maintainer-facing comment also explicitly kept this issue open for that reason.

## Best Possible Solution

Keep the issue open until OpenClaw either implements bounded context-usage plumbing from the latest fresh session token snapshot into embedded-runner runtime params, `RuntimeInfoInput`, and `buildRuntimeLine()` with focused tests/docs, or a maintainer explicitly decides that `/status`, `session_status`, `/context`, UI warnings, context-engine/memory plugins, and the community `openclaw-workspace-sync` recovery plugin are the intended product boundary.
