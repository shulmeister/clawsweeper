## Summary

Keep open. Current main still has the reported asymmetry: one-shot `at` jobs use transient-error detection and retry backoff, while recurring `cron`/`every` errors compute a backoff floor but still choose the later natural schedule, so long-period transient failures can still wait until the next scheduled run. The related PR #46423 is useful and directly tracks the fix, but the provided context says it is still open and unmerged, so the issue should not be closed as implemented.

## Evidence

This branch does not call `isTransientCronError`, so transient and permanent errors are scheduled the same way.
