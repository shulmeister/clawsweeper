## Summary

Current `main` already routes Feishu reply mentions through native Feishu/Lark mention markup instead of plain `@name` text, including plain post replies, card replies, and streaming-card closeout.

## Close Comment

Closing as `implemented_on_main` from a Codex review. Current `main` at `9ece33c50506ba031df382f69bde65dfda19bea5` no longer treats outbound Feishu mentions as plain `@name` text on the reply path: `extensions/feishu/src/reply-dispatcher.ts:442` and `extensions/feishu/src/reply-dispatcher.ts:461` pass mention targets into the outbound sender, `extensions/feishu/src/send.ts:467` rewrites them into Feishu post content before send, and `extensions/feishu/src/mention.ts:79` / `extensions/feishu/src/mention.ts:93` emit native Feishu/Lark mention markup (`<at user_id="...">...</at>` for post text and `<at id=...></at>` for card/lark_md content). Streaming-card closeout also preserves those mentions in `extensions/feishu/src/reply-dispatcher.ts:309`. I could not do a live Feishu UI repro in this read-only pass, but the current outbound code path matches the requested fix, so this issue is no longer actionable against `main`. If someone still sees white-rendered mentions on a newer build, please open a fresh issue with the exact outbound message type/payload path and a screenshot.
