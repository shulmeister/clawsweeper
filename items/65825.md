## Summary

Keep PR #65825 open. Current main still does not implement the PR's central fix: discovery does not thread a shared `realpathCache` through its helper stack, and `safeRealpathSync()` still does not cache the canonical resolved path. The later shared discovery-result cache on main is related performance work, but it does not cover the first-pass repeated `fs.realpathSync()` calls reported in #65733.

## Best Possible Solution

Keep the PR open and either merge or port the narrow fix: thread one `Map<string,string>` through `discoverOpenClawPlugins` helper calls, add the canonical `resolved -> resolved` cache entry in `safeRealpathSync`, and keep the regression tests for normal package extension packs plus symlinked `extraPaths`. If maintainers believe #67940 fully resolves #65733, that should be backed by Windows startup profiling and an explicit superseding note; current main does not prove that.
