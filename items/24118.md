## Summary

Keep #24118 open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still does not expose `channels.signal.linkPreview` in Signal config types, strict schema, generated metadata, or docs, and the Signal outbound path does not pass preview fields or `updateConfiguration` through to signal-cli. Upstream signal-cli does expose link-preview-related controls, so the request remains concrete and actionable for the bundled Signal plugin. The provided steipete/Codex discussion also kept this open and distinguishes closed #29466 as Webchat preview-card work, not Signal channel support.

## Best Possible Solution

Keep #24118 open until the bundled Signal plugin adds a backward-compatible `channels.signal.linkPreview` surface, including top-level and per-account schema/types/generated metadata/docs; resolves the setting through account inheritance; threads it through outbound text/media sends; passes supported preview configuration and preview metadata to signal-cli JSON-RPC; and adds focused schema/send tests. If maintainers decide Signal link previews are out of scope, record that explicit product decision rather than auto-closing this concrete channel enhancement.
