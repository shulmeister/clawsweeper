## Summary

Keep open. Current main still has the Linux/systemd restart path followed by a hard post-restart health failure path, and the related PR #66685 is open/partial rather than a proven fix for the false non-zero restart result.

## Best Possible Solution

Keep #66675 open. The best fix belongs in the core CLI/daemon lifecycle around `runDaemonRestart`, `runServiceRestart`, and restart health inspection: prove the false-negative root cause, make the post-restart check authoritative against current status/probe health, and add a focused regression test for the Linux/systemd healthy-after-restart case. Handle expired approval followup warnings separately via #66685 or an equivalent narrow patch, but do not let that partial cleanup supersede the restart exit-code bug.
