## Summary

Keep #69364 open. Current main has useful per-agent Control UI pieces, including agent file editing, tool policy controls, skill allowlist controls, and flat session rows with subagent metadata, but it does not implement the requested cross-agent relationship view for delegation, dataflow, task/flow sequence, and live status. Related #52803 overlaps with multi-agent orchestration UX but is broader and not a clean duplicate of this configuration/workspace relationship proposal.

## Best Possible Solution

Keep the issue open as a focused first-party Control UI product/API request. A practical path is a read-only MVP that assembles current supported data first: agents.list and subagents.allowAgents for delegation, sessions.list parent/child/status fields for live relationships, and the task/Task Flow ledger for sequence. Reuse existing per-agent Files, Tools, and Skills panels for drill-in editing; if maintainers want this outside core, first design a Control UI plugin extension point.
