## Summary

Current main still has unresolved sandbox `os.networkInterfaces()` crash paths. Some related mitigations landed for Docker/Bonjour and status display, but the exact reported `uv_interface_addresses` failure can still escape through Bonjour/ciao and OpenClaw-owned LAN/tailnet helpers; PR #71950 is relevant but remains open/unmerged in the provided GitHub context.

## Best Possible Solution

Keep #71721 open. The best maintainer path is to use PR #71950 or an equivalent patch for the OpenClaw LAN/tailnet helpers, and also add a focused Bonjour/ciao regression for `SystemError: uv_interface_addresses returned Unknown system error 1` so sandboxed Nemoclaw/OpenShell gateways either disable/recover Bonjour or otherwise continue running without disconnecting the TUI.
