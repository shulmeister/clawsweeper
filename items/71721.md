## Summary

Keep #71721 open. Both the checked-out main SHA and the latest remote main still propagate `os.networkInterfaces()` failures through the gateway LAN and tailnet discovery paths, and the linked fix PR #71950 remains open/unmerged.

## Best Possible Solution

Keep #71721 open and land PR #71950 or an equivalent maintainer-reviewed fix that switches the gateway LAN and tailnet discovery paths to safe interface enumeration, then updates the regression tests to assert graceful fallback instead of thrown errors.
