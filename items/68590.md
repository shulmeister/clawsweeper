## Summary

Keep #68590 open. Current main has not implemented the PR's `index.yml` rewrite, and the PR remains a plausible focused memory-core repair change rather than a cleanup-close candidate. The PR also has unresolved review concerns that need maintainer or author changes before merge, especially preserving effective repair behavior when QMD still lists matching collection metadata.

## Best Possible Solution

Keep the PR open for author or maintainer changes. The best path is to add a safe QMD index writer using proper YAML/string quoting, preserve the repair path's effective remove/add reset for duplicate-document and null-byte failures, and add regression coverage for a repair where `qmd collection list --json` returns populated matching collections. If maintainers prefer the current brute-force repair, the index rewrite should be introduced as a narrow helper without routing repair through skip-on-match bootstrap logic.
