## Summary

Keep open. Current main has narrow retry, failover, gateway-auth, Discord, and Codex-account rate-limit handling, but it does not implement the issue's requested configurable cross-provider rate limiting, token budgets, queueing, warning thresholds, or `openclaw limits` CLI. Related PR #13686 attempted this scope but was closed unmerged, so it is not a superseding implemented fix.
