## Summary

Keep #13615 open. Current main still lacks the requested configurable outbound API rate limiting/throttling layer, token/request budget enforcement, queue/backpressure controls, warnings/status management, and `openclaw limits` CLI. Existing code covers adjacent reactive 429 handling, auth-profile cooldown/fallback, per-channel retry, inbound gateway/webhook throttles, and usage readouts only; the provided GitHub context also includes a maintainer comment explicitly keeping this issue open.

## Best Possible Solution

Keep #13615 open as the canonical core product/design item. The best implementation path is an opt-in outbound policy layer at model-provider, tool/web-search, and channel request boundaries, with config/schema/docs for provider/model/global RPM, TPM, daily/monthly budgets, queue/backpressure behavior, 429 retry/backoff, warning/status reporting, and an `openclaw limits` CLI. Treat closed PR #13686 as prior art and #14206 as a narrower Feishu channel follow-up, not a replacement.
