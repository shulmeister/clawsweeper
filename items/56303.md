## Summary

Current `main` does not contain the reported `sessions_spawn` channel-name hardcoding. The tool treats `thread` generically, session-mode binding is delegated through generic spawn hooks, and Feishu-specific handling lives in the Feishu extension rather than OpenClaw core. The remaining concern described in comments is a separate feature request for richer plugin metadata, not the bug reported here.

## Evidence

- **sessions_spawn tool has no hardcoded channel-name branch:** On current `main`, `createSessionsSpawnTool` derives `thread` only from `params.thread === true` and forwards context; there is no `agentChannel === "feishu"` or similar channel-specific check in the source.
  - file: [src/agents/tools/sessions-spawn-tool.ts:217](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/tools/sessions-spawn-tool.ts#L217)
  - command: `nl -ba src/agents/tools/sessions-spawn-tool.ts | sed -n '60,340p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **session-mode requirement is generic, hook-driven behavior:** `spawnSubagentDirect` rejects `mode="session"` without `thread=true`, then routes thread binding through the generic `subagent_spawning` hook path rather than channel-name conditionals.
  - file: [src/agents/subagent-spawn.ts:597](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/subagent-spawn.ts#L597)
  - command: `nl -ba src/agents/subagent-spawn.ts | sed -n '500,560p'; nl -ba src/agents/subagent-spawn.ts | sed -n '560,660p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Feishu-specific binding lives in the extension:** Feishu registers `subagent_spawning`/delivery hooks from the extension package, and the implementation only acts when the requester channel is Feishu and `threadRequested` is true. This is extension-owned behavior, not core hardcoding.
  - file: [extensions/feishu/src/subagent-hooks.ts:292](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/feishu/src/subagent-hooks.ts#L292)
  - command: `nl -ba extensions/feishu/subagent-hooks-api.ts | sed -n '1,120p'; nl -ba extensions/feishu/src/subagent-hooks.ts | sed -n '220,360p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Existing plugin metadata does not express the requested auto-threading contract:** Feishu advertises `capabilities.threads: true`, but the channel-entry SDK options only expose basic entry metadata and do not define a separate field for `sessions_spawn` thread-binding requirements. That supports the repo comment that the remaining gap is a feature request, not proof of the reported hardcoded-core bug.
  - file: [src/plugin-sdk/core.ts:414](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/plugin-sdk/core.ts#L414)
  - command: `nl -ba src/plugin-sdk/core.ts | sed -n '400,780p'; nl -ba extensions/feishu/src/channel.ts | sed -n '560,620p'; nl -ba src/channels/plugins/types.core.ts | sed -n '260,320p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Issue discussion already identified the mismatch:** A March 28 maintainer comment states the hardcoded `isFeishuChannel` snippet is not present on `main` and reframes the remaining concern as a feature request for plugin metadata; the reporter acknowledged that comment with "OK thanks."

## Risks / Open Questions

- There may still be a worthwhile feature request around plugin-declared subagent thread-binding requirements, but that is not the same as the bug reported in this issue.
