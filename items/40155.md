## Summary

Keep #40155 open. Current main shows Telegram and WebUI both enter the shared reply/agent dispatch path, and vLLM models are not disabled for tools by default, but there is no direct source, test, release, or history evidence that the reported Telegram + vLLM + himalaya skill mismatch is fixed. Telegram still supplies channel-specific context, skill filters, streaming/tool-progress options, and routing metadata that can diverge from WebUI behavior, so this is not safe to auto-close from source inspection alone.

## Best Possible Solution

Keep the issue open and reproduce on current main using the same agent/session, vLLM gpt-oss-20b model, and himalaya skill through both WebUI chat.send and Telegram. Capture the resolved skill snapshot, effective tool inventory, model compat, messageProvider/channel metadata, and replyOptions for both paths. If the Telegram path loses the skill/tool catalog, add a regression test around Telegram dispatch/getReplyOptions and fix the context/skill/tool-policy divergence; if it no longer reproduces with that evidence, close later as cannot_reproduce or implemented_on_main.
