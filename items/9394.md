## Summary

Keep #9394 open. Current main still does not implement the requested iMessage rich-text attributes, text sizing, or Shake/Nod/Explode/Ripple/Bloom/Jitter text-animation effects. Legacy `imsg` remains plain-text/media/reply oriented, and BlueBubbles only has partial classic `effectId` support while stripping Markdown text before send. The existing issue discussion already includes a Codex review comment by steipete keeping this open for the same gaps, and current main has not materially changed that surface.

## Best Possible Solution

Keep #9394 open for maintainer product/API direction. The likely path is a typed BlueBubbles/iMessage rich-text and text-effect send surface covering supported attributes, size, and animation/effect enums, wired through outbound delivery and message actions with tests and docs. Legacy `imsg` should either gain an upstream-supported capability or be documented as unsupported for those attributes.
