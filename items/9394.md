## Summary

Keep #9394 open. Current main only partially covers the request: BlueBubbles can send a limited set of classic iMessage effects through `sendWithEffect`, but legacy `imsg` delivery remains text/media/reply only, BlueBubbles deliberately strips Markdown formatting before sending, and there is still no rich text, text size, or requested Shake/Nod/Explode/Ripple/Bloom/Jitter text-animation support. The latest issue discussion also already kept this open for the same gap.

## Best Possible Solution

Keep #9394 open until a maintainer decides the product direction and the missing behavior is implemented or explicitly documented as unsupported. The likely path is a typed BlueBubbles/iMessage rich-text and Apple text-effect send surface, wired through message actions and outbound delivery with tests and docs; legacy imsg limitations should be documented if upstream imsg cannot carry those attributes.
