## Summary

Keep #9394 open. Current main still does not support iMessage rich text spans, text sizing, or the requested Shake/Nod/Explode/Ripple/Bloom/Jitter text effects. Legacy imsg outbound sends plain text/media/reply payloads, iMessage remains outside the shared rich Markdown IR render path, and BlueBubbles only has narrow classic effectId support while stripping Markdown before send.

## Best Possible Solution

Keep #9394 open for maintainer product/API direction. The likely fix belongs in the iMessage/BlueBubbles channel surface: typed rich-text/style-span and text-effect send support, text-size handling where the underlying transport supports it, outbound/message-action wiring, tests, and docs. Legacy imsg should either gain upstream-supported capability handling or explicitly document these attributes as unsupported.
