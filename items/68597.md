## Summary

Keep open. Current main still uses lexical path containment plus `lstat` on the requested leaf for builtin and QMD `memory_get`; that rejects direct symlink files but does not add the PR's canonical realpath containment check for intermediate directory symlink escapes. The PR is a focused security hardening change with concrete repro steps, scoped tests, and follow-up fixes for the review comments.

## Best Possible Solution

Keep PR #68597 open for maintainer review or land an equivalent fix: add a shared canonical contained-regular-file resolver, use it in both builtin `readMemoryFile` and QMD `readFile`, preserve transient missing-file behavior, and keep regression tests for workspace memory, extraPaths, QMD collection directory symlink escapes, plus symlink-permission guards in tests.

## Evidence

- **Builtin memory_get still lacks canonical containment:** `readMemoryFile` allows workspace memory paths and extra paths by lexical `path.relative` / `startsWith` checks, then calls `statRegularFile(absPath)` and `fs.readFile(absPath)` without resolving the target and allowed root to real paths. An intermediate directory symlink can therefore remain lexically under `memory/` or an extra path while resolving outside that root.
