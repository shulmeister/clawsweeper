## Summary

Keep #8582 open. Current main implements the ElevenLabs realtime STT / provider-generic Voice Call streaming portion and has improved general docs for duplicate plugin precedence and schema validation, but one central DX ask remains unresolved: distinct duplicate plugin ids still always warn and there is no per-id suppression or explicit override acknowledgement config. The requested focused docs/example for extending a bundled plugin with new config keys is also still only partially covered by general manifest docs.

## Best Possible Solution

Keep #8582 open unless a maintainer splits it. The best path is to add a tested per-id duplicate override/suppression acknowledgement in plugin config and manifest-registry diagnostics, document that behavior, and add focused plugin docs explaining that custom config keys require updating `openclaw.plugin.json` `configSchema` when extending or forking a bundled plugin. Alternatively, create canonical follow-up issues for those remaining DX asks, then close this umbrella as partially shipped plus superseded.
