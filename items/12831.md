## Summary

Keep open. Current main has partial plumbing for live tool events, inline tool cards, session.tool broadcasts, and a resizable details sidebar, but it still does not implement the requested separate activity/observer mode. Tool output remains tied to verbose=full, /verbose is still session-scoped, Control UI renders tool activity in the main chat stream, and the TUI still gates tool cards by verbose rather than showing a split activity pane.

## Best Possible Solution

Keep this issue open as the product/UX tracker. The best implementation is a first-class observer mode built on the existing tool-events/session.tool plumbing, with a dedicated Control UI activity surface, a TUI split-pane equivalent, an output policy independent of /verbose, connection-scoped observer controls or an explicit subscription endpoint, and focused tests/docs for the new behavior.
