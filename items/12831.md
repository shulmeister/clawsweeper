## Summary

Keep #12831 open. Current main has partial tool-event plumbing and live tool cards, but it still does not deliver the requested first-class observer mode: Control UI tool activity remains in the chat item stream, tool output is still gated by `verbose=full`, verbose remains session-scoped rather than per-connection, and TUI tool visibility is still controlled by `/verbose`. The prior Apr 26 Codex review comment correctly identified the same gaps, and related #46604 is narrower rather than a clear superseding canonical tracker.

## Best Possible Solution

Keep #12831 open as the broad observer-mode tracker. Build on the existing `tool-events` and `session.tool` plumbing, but add a dedicated Control UI activity panel with independent scroll/state, observer output visibility not coupled to `/verbose full`, connection-scoped controls or a documented subscription workflow, TUI split/observer support, and focused tests/docs. Treat #46604 as a narrower web split-panel follow-up unless maintainers explicitly make it canonical.
