## Summary

Keep #12831 open. Current main has useful lower-level tool-event plumbing, including `tool-events` and `session.tool`, but it still does not provide the requested first-class Activity panel / observer mode with separated UI state, per-connection verbose behavior, output visibility decoupled from `/verbose full`, an external observer stream surfaced in product/docs, or a TUI split-pane observer. The related open #46604 and draft PR #60814 overlap, but neither is merged or clearly canonical for the full #12831 scope.

## Best Possible Solution

Keep #12831 open as the broad observer-mode tracker. Build on the shipped `tool-events` and `session.tool` gateway plumbing, add a dedicated Control UI Activity panel with independent state/scrolling and expandable output, decide per-connection or per-tab verbose semantics, document or expose the observer stream for external clients, and add TUI split-pane support or split those pieces into explicit canonical follow-up issues before closing this umbrella.
