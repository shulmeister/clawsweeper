## Summary

Keep #12831 open. Current main has useful partial plumbing for live tool events and inline tool cards, but it still does not provide the requested separate activity/observer mode: Control UI tool activity is rendered in the chat stream, raw output remains tied to `verbose=full`, `/verbose` is persisted as a session override, and the TUI still gates tool visibility by verbose instead of offering a split activity pane.

## Best Possible Solution

Keep #12831 open as the product/UX tracker for a first-class observer mode. The best implementation should build on existing `tool-events` and `session.tool` plumbing, but add a dedicated Control UI activity surface, TUI split-pane equivalent, output policy independent of `/verbose`, connection-scoped observer controls or a documented subscription endpoint, and focused tests/docs for the new behavior.
