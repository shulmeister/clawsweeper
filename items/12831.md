## Summary

Keep #12831 open. Current main has useful partial plumbing for live WS/session tool events and inline tool cards, but it still does not provide the requested separate activity/observer mode: Control UI renders tool activity inside the chat thread, raw output remains tied to verbose=full, TUI tool visibility remains verbose-gated, and the related #46604 is a narrower open web split-panel request rather than a canonical replacement for this broader issue.

## Best Possible Solution

Keep #12831 open as the broad product/UX tracker for a first-class activity observer mode. Build on the existing tool-events and session.tool plumbing, but add a dedicated Control UI activity surface with independent scrolling, a TUI split/observer equivalent, an output policy independent of /verbose, connection-scoped observer controls or a documented subscription path, and focused tests/docs for the new behavior. If #46604 becomes the canonical web-only implementation, link it as a narrower follow-up rather than closing this broader tracker prematurely.
