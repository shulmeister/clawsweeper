## Summary

Current `main` still enforces one active agent run per session. The code queues each run onto a session-specific lane before the global lane, and the docs explicitly describe per-session serialization with followup/steer/interrupt queue modes as the supported behavior. The linked cross-reference is to an open PR, which reinforces that this request is not yet shipped on `main`.

## Evidence

- **Per-session lane serialization in runtime:** `runEmbeddedPiAgent()` resolves a `session:<key>` lane, enqueues into that session lane, then enqueues into the global lane. That structure prevents concurrent turns within one session on current `main`.
  - file: [src/agents/pi-embedded-runner/run.ts:252](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/pi-embedded-runner/run.ts#L252)
  - command: `nl -ba src/agents/pi-embedded-runner/run.ts | sed -n '238,290p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Session lanes are distinct lane keys:** `resolveSessionLane()` prefixes each session with `session:` and returns that lane id for embedded runs.
  - file: [src/agents/pi-embedded-runner/lanes.ts:3](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/pi-embedded-runner/lanes.ts#L3)
  - command: `nl -ba src/agents/pi-embedded-runner/lanes.ts | sed -n '1,80p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Queue implementation defaults each lane to single concurrency:** The command queue is documented in code as serializing executions, and new lanes start with `maxConcurrent: 1`, so a given session lane runs FIFO unless code changes that behavior.
  - file: [src/process/command-queue.ts:31](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/process/command-queue.ts#L31)

  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Docs explicitly say one active run per session:** The queue design doc says inbound runs are serialized, `runEmbeddedPiAgent` enqueues by session key to guarantee only one active run per session, and queue modes are steer/followup/collect/interrupt rather than concurrent same-session turns.
  - file: [docs/concepts/queue.md:8](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/concepts/queue.md#L8)
