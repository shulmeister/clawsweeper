## Summary

Keep open. Current main still lacks the PR's central Mission Control runtime surface: there is no dashboard.summary/dashboard.delta RPC/event contract, no incident/dashboard handlers, no Tool Runtime/Execution Queue/Autonomy Posture UI panels, and no global tool activity registry. The related closed reports cover underlying tool guardrails/hooks/channel visibility, but they do not supersede this dashboard aggregation work. The PR also still has unresolved review concerns and unrelated branch churn, so it is neither implemented on main nor safely closeable under a conservative cleanup rule.

## Best Possible Solution

Keep the PR open or replace it with a focused successor. The best path is to split the useful Mission Control work into scoped gateway/UI changes: add a reviewed dashboard summary/delta contract, tool activity and process snapshots, incident/autonomy data, focused Control UI panels, and regression tests; fix the tool-error and startup-race review findings; and drop unrelated workspace, Docker, script, and test-environment churn before maintainer review.
