## Summary

Keep #40652 open. Current main only partially addresses the requested recovery guidance: one gateway startup path now suggests `openclaw doctor --fix`, but it still omits the `.bak` rollback hint, and daemon start/restart preflight still suggests generic `openclaw doctor` without `--fix` or backup recovery.

## Best Possible Solution

Keep the issue open and make a narrow UX fix in the gateway/daemon invalid-config startup paths: standardize the message on `openclaw doctor --fix`, add an explicit `openclaw.json.bak` rollback hint where a backup may exist, and extend the existing startup/daemon lifecycle tests to assert both recovery steps.
