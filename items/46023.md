## Summary

Keep open. Current main still has the reported Telegram final-reply media limit gap: the reply delivery path computes/uses media loading without passing a configured mediaMaxBytes value, while the direct send path already passes maxBytes. Related PR #46425 directly proposed this threading fix but was closed unmerged, so it does not supersede the open issue.
