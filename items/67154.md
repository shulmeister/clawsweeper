## Summary

Keep open. Current main has hardened `exec-approvals.json` writes with temp-file rename, which appears to address the linked corruption class from #63707, but it has not implemented #67154's requested separation. Runtime usage fields are still part of allowlist entries, gateway/node exec paths still update those fields during exec, tests assert that behavior, and docs still document `lastUsedAt`/`lastUsedCommand` inside `~/.openclaw/exec-approvals.json`.

## Best Possible Solution

Keep #67154 open as a focused feature/architecture request. A maintainer-quality fix would add a backward-compatible `exec-runtime.json` store for usage/audit metadata, migrate reads so existing `exec-approvals.json` entries still work, stop ordinary exec usage from writing the allowlist file, update `openclaw approvals`/`exec-policy` output to merge or display runtime metadata, and update docs/tests to prove allowlist-only files can be read-only while runtime tracking remains writable.
