## Summary

Current main still lacks configurable reasoning/thinking display formatting per channel or per agent. The shared formatter hardcodes italic Markdown, the runner/subscriber/Telegram paths call it without any format option, config/docs only expose reasoning visibility and provider transport `thinkingFormat`, and the related implementation PR #25336 is closed unmerged. The issue should stay open.

## Best Possible Solution

Keep #8913 open. A good fix should define and document a reasoning display-format contract, validate it in the chosen config scope, preserve italic as the compatibility default, thread the value through embedded runner payloads, subscribe/message-end/streaming callbacks, and Telegram reasoning-lane dispatch, then add regression coverage for default italic plus configured code/spoiler/blockquote behavior.
