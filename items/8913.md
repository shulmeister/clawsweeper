## Summary

Current main still lacks configurable reasoning/thinking block formatting. Visible reasoning is still formatted by a shared italic-only helper, Telegram still uses that helper directly, config/docs expose visibility and thinking-level controls but not a rendered-format option, and the related implementation PR #25336 was closed unmerged.

## Best Possible Solution

Keep #8913 open. A good implementation should define a documented `reasoningFormat`/`thinkingFormat` contract, validate it in the selected config scope, thread it through embedded runner payloads, subscribe handlers, and Telegram reasoning split/dispatch, then add regression tests for default italic plus configured code/spoiler/blockquote behavior on visible reasoning paths.
