## Summary

Keep #8913 open. Current main still hardcodes reasoning display as `Reasoning:\n_..._` in shared agent formatting and Telegram reasoning-lane handling, and the config/docs/tests do not expose a per-channel or per-agent reasoning display-format setting. The related PR #25336 is useful prior work, and rkd-me's blockquote suggestion is relevant product input, but #25336 was closed unmerged and does not supersede the remaining request.

## Best Possible Solution

Keep #8913 open. A complete fix should define a documented reasoning display-format contract, preserve italic as the compatibility default, validate the config shape, decide override precedence, thread the selected format through embedded run payloads, subscribe/message-end/streaming callbacks, and Telegram reasoning-lane delivery, then add regression tests for default italic plus configured code/spoiler/blockquote behavior.
