## Summary

Current main still does not implement configurable reasoning/thinking block display formatting. The observable behavior remains an italic-only `Reasoning:` formatter shared by core reasoning payload paths and Telegram's reasoning split path, with config/docs/tests covering visibility and provider transport thinking formats but not a user-selectable channel or agent display format. The related implementation PR #25336 is closed unmerged, and the latest maintainer/Codex review comment already identified the same remaining gap.

## Best Possible Solution

Keep #8913 open. A good fix should define a documented `reasoningFormat`/`thinkingFormat` display contract, validate it in the chosen config scope, thread it through embedded runner payloads, subscribe handlers, reasoning streaming, and Telegram split/dispatch, preserve the stable `Reasoning:` prefix where parsers depend on it, and add regression tests for default italic plus configured code/spoiler/blockquote behavior.
