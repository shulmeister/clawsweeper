## Summary

Keep open. Current main still hardcodes visible reasoning formatting as italic Markdown and exposes reasoning visibility/default thinking controls, but no per-channel or per-agent reasoning/thinking format option. The related PR #25336 was closed unmerged in the provided GitHub context, and the latest maintainer/Codex comment on #8913 also kept the issue open.

## Best Possible Solution

Keep #8913 open until maintainers either implement the feature or choose a different product direction. A complete fix should define a documented `ReasoningFormat`/`thinkingFormat` contract, validate it in agent and/or Telegram channel config, thread it through runner payloads, subscribe handlers, and Telegram reasoning split/dispatch paths, and add integration tests proving the supported formats on visible reasoning paths.
