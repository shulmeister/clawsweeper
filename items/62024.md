## Summary

Keep open. Current main still has the reported Discord explicit-target bug: `parseDiscordExplicitTarget` returns the bare `target.id`, while the outbound normalizer still maps bare numeric recipients to `channel:<id>`. The linked PR #62052 is closed and unmerged per the provided GitHub context, so it does not supersede or implement the fix on main.
