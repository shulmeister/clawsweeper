## Summary

Keep this PR open. Current main still has the unguarded assistant `surface_error` throw path, and the PR remains a focused fix with a missing regression test for `failoverFailure=false` plus a classified `failoverReason`.

## Best Possible Solution

Keep this PR open and review or land it, or land an equivalent minimal fix in `src/agents/pi-embedded-runner/run/assistant-failover.ts` that requires `params.failoverFailure` before throwing `FailoverError` from the assistant `surface_error` branch.
