## Summary

Keep PR #70900 open. Current main still has the exact `surface_error` throw guard that the PR targets, while the surrounding failover code still allows a classified `failoverReason` without `failoverFailure` to reach that branch after profile rotation is exhausted.

## Best Possible Solution

Keep #70900 open and either merge its minimal `&& params.failoverFailure` guard in `src/agents/pi-embedded-runner/run/assistant-failover.ts` or recreate that fix on main with the focused regression test in `src/agents/pi-embedded-runner/run/assistant-failover.test.ts`. The test should lock in `surface_error` with `failoverFailure=false` and classified `failoverReason` returning `continue_normal`, while retaining coverage that #70124-style genuine provider failures still throw.
