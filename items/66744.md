## Summary

Keep open. Current main still contains the unsafe `provider?.models?.some(...)` call reported in #66744, the lookup tests do not cover non-array `models`, and the linked fix PR #66761 remains unmerged in the provided GitHub context. This is a real bug report with a concrete stack trace and should stay open until the guard and regression tests land.

## Best Possible Solution

Keep #66744 open as the canonical bug. Review and land PR #66761 or an equivalent narrow patch that changes the lookup to `Array.isArray(provider?.models) && provider.models.some(...)`, adds regression coverage for object/non-array `models`, and then close the issue with the landed SHA and release once shipped.
