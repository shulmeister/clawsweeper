## Summary

Current `main` still appears to have the reported audit false positive. `listKnownNodeCommands()` only derives names from the platform/default allowlist plus explicit `allowCommands`, while dangerous command IDs such as `camera.snap`, `screen.record`, and `sms.send` are defined separately as valid-but-gated commands. That means `gateway.nodes.denyCommands` can still be flagged as containing unknown commands for entries that are actually valid dangerous commands, so this PR should stay open.
