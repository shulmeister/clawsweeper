## Summary

Keep open. Current `main` still persists session transcript messages without applying `logging.redactPatterns` or `logging.redactSensitive`; the redaction config is documented as console-only, and the security docs still warn that session transcripts can contain secrets.

## Evidence

- **Session transcript append still writes the final message directly:** `installSessionToolResultGuard` applies optional provenance/hooks/truncation, then calls the raw session append with `finalMessage`; there is no built-in call to `redactSensitiveText` before persistence.
  - file: [src/agents/session-tool-result-guard.ts:243](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/agents/session-tool-result-guard.ts#L243)

  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Session-manager wrapper adds no redaction transform:** `guardSessionManager` only wires input provenance, optional hook transforms, and tool-result size caps. No default transcript-redaction transform is installed from logging config.
  - file: [src/agents/session-tool-result-guard-wrapper.ts:72](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/agents/session-tool-result-guard-wrapper.ts#L72)
