## Summary

Keep open. Current main has lower-level per-run provider/model override plumbing, but the requested user-facing `openclaw agent --model` CLI flag is still not registered or passed through the CLI path. The PR is stale against current file layout, but it addresses a real CLI/docs mismatch and has downstream use-case support in the discussion.

## Best Possible Solution

Keep the PR open, but rework it onto current main by adding `--model <provider/model-or-alias>` to `openclaw agent`, adding `model?: string` to `AgentCliOpts`, passing it through both Gateway and `--local` paths into the existing per-run `AgentCommandOpts.model` surface, and updating `docs/cli/agent.md` plus CLI/gateway tests. Avoid duplicating already-shipped lower-level provider/model override machinery unless a concrete persistence or auth-profile gap remains.
