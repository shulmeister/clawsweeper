## Summary

Keep open. Current `main` does not implement the PR's Feishu card-action payload changes, and the PR as described/reviewed appears to introduce a real dedupe regression by reusing the original Feishu message ID as the synthetic inbound `message_id`. This is not strong close evidence for any allowed reason.

## Evidence

- **Current parser still requires `context.chat_id`:** `parseFeishuCardActionEventPayload` only accepts `context.chat_id` and rejects payloads missing it; it does not read `open_chat_id`, `open_message_id`, or `form_value`. That means the requested behavior is not already on `main`.

No support for `open_message_id`/`open_chat_id` is present on `main`.
