## Summary

Keep open. Current main already distinguishes Gateway client identities at connect time, including browser Control UI/WebChat and native macOS, but that metadata still does not reach the agent as trusted runtime info. `chat.send` maps the connected client into sender/untrusted context and scopes; the runtime prompt line still exposes only channel/capability metadata, so the issue’s central request remains real.

## Best Possible Solution

Keep #39597 open and implement a narrow trusted Gateway client-context path: define sanitized client metadata, populate it from client.connect.client in chat.send, propagate it through auto-reply/followup run params into RuntimeInfoInput, render fields such as client=openclaw-macos, client_mode=ui, and client_platform=macos in buildRuntimeLine, then add Gateway chat and system-prompt tests plus WebChat/Control UI docs for the privacy-safe trusted fields.

## Risks / Open Questions

- Raw userAgent, modelIdentifier, and instanceId can be privacy-sensitive; expose only sanitized trusted fields such as client id, mode, platform, and device family.
- Runtime-line changes affect prompt bytes and cache stability, so the added fields should be deterministic and covered by prompt tests.
