## Summary

Current main still does not implement strict on-demand HEARTBEAT.md loading. The loader still reads HEARTBEAT.md as part of the default workspace bootstrap set, the session cache stores that full loader result before run-kind filtering, and normal default-agent non-heartbeat runs still retain HEARTBEAT.md whenever heartbeat guidance is enabled. The related #55311 lightContext path is addressed for actual heartbeat runs, but it does not solve this issue's requested non-heartbeat omission.

## Best Possible Solution

Keep #13968 open. Implement a run-kind-aware bootstrap loading and cache contract that excludes HEARTBEAT.md for default non-heartbeat runs before disk reads and session snapshot caching, while preserving HEARTBEAT.md for actual heartbeat runs and lightweight heartbeat context. Add focused tests for enabled default non-heartbeat omission, cache behavior, disabled cadence/prompt regressions, and heartbeat retention, then update the heartbeat docs to describe the final contract.
