## Summary

Keep #13968 open. Current main still does not implement strict on-demand HEARTBEAT.md loading for heartbeat turns only: non-heartbeat/default runs can still load and inject HEARTBEAT.md when heartbeat guidance is enabled. The related #55311 lightContext forwarding work is separate and does not satisfy this request.

## Best Possible Solution

Implement an explicit run-kind/isHeartbeat-aware bootstrap loading path so default non-heartbeat runs omit HEARTBEAT.md before loading/injection, while actual heartbeat runs still keep it and lightweight heartbeat mode remains heartbeat-only. Add focused tests for default non-heartbeat omission, heartbeat retention, disabled cadence/prompt-section behavior, and update heartbeat/token-use docs if the product behavior changes.
