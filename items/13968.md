## Summary

Current main still does not implement strict on-demand HEARTBEAT.md loading. HEARTBEAT.md remains part of the default bootstrap file set, is read before run-kind filtering/cache effects, and is only omitted from normal runs when heartbeat guidance is disabled. The related closed #55311 fixed heartbeat lightContext forwarding, not the default non-heartbeat omission requested here.

## Best Possible Solution

Keep #13968 open. Implement a run-kind-aware bootstrap loading path that omits HEARTBEAT.md for default non-heartbeat runs before disk reads and cache population, preserves HEARTBEAT.md for actual heartbeat runs and lightweight heartbeat context, adds focused tests for default enabled omission/cache behavior/lightContext retention, and updates heartbeat/system-prompt docs to match the new contract.
