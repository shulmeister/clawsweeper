## Summary

Keep open. Current `main` has partial groundwork: heartbeat runs now pass `bootstrapContextRunKind: "heartbeat"`, but `HEARTBEAT.md` is still included on ordinary runs whenever heartbeat guidance is enabled for the default agent. That does not match issue #13968's requested behavior of loading `HEARTBEAT.md` only for heartbeat polls.

## Evidence

- **Bootstrap loader still enumerates HEARTBEAT.md unconditionally:** `loadWorkspaceBootstrapFiles()` still includes `DEFAULT_HEARTBEAT_FILENAME` in its default bootstrap file list, so the file is part of the base load set before any later filtering.
  - file: [src/agents/workspace.ts:477](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/agents/workspace.ts#L477)
  - command: `nl -ba src/agents/workspace.ts | sed -n '477,541p'`

If guidance is enabled, ordinary runs still keep the file.
