## Summary

Keep open. Current main has heartbeat bootstrap improvements, including the related lightweight heartbeat path, but it still does not implement strict on-demand HEARTBEAT.md loading for heartbeat turns only. Default non-heartbeat runs can still load and inject HEARTBEAT.md when heartbeat guidance is enabled.

## Best Possible Solution

Implement an explicit run-kind/isHeartbeat-aware bootstrap loading path so non-heartbeat default runs omit HEARTBEAT.md before loading and injection, while heartbeat runs and lightweight heartbeat mode still keep it. Add focused tests for default non-heartbeat omission, heartbeat retention, disabled-cadence behavior, and update the heartbeat/system-prompt docs to match the chosen product behavior.
