## Summary

Keep open. Current main still contains the same undefined-to-0 timeout coercions that this PR is trying to remove, and the timeout resolver still treats explicit 0 as the no-timeout sentinel while undefined falls back to the configured default. The PR is not implemented on main or clearly superseded by another open canonical item in the provided context.
