## Summary

Keep #13225 open. Current main still lacks the requested post-model, pre-tool-call routing contract: `before_tool_call` can rewrite params, block, or require approval, but it cannot inspect preceding model output and redirect a pending tool call to another model/session. The existing `before_model_resolve` hook is a pre-session model-selection seam, not a pending-tool-call delegation seam. The related open PRs #58079 and #62701 are context-field work and do not supersede routing semantics.

## Best Possible Solution

Keep #13225 open as a plugin/runtime API design request. The best path is an additive typed pre-tool routing contract, either on `before_tool_call` or a separate hook, that defines what assistant context may be inspected and whether/how plugins can redirect a pending tool call to another provider/model/session with explicit audit, permission, and failure behavior. Treat #58079 and #62701 as related context-field work unless they merge and are extended to include routing semantics.
