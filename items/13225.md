## Summary

Keep open. Current main has a useful plugin `before_tool_call` hook that can rewrite params, block, or require approval, and that likely covers the commenter’s per-agent TTS argument injection use case because the hook context includes `agentId`. It does not implement the issue’s core request: a pre-tool-call decision that can see preceding assistant output and redirect the call to another model/session. The related #15971 was cron-tier routing and is closed stale, so it does not supersede this broader hook/API request.
