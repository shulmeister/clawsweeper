## Summary

Keep #13225 open. Current main still supports `before_tool_call` parameter rewrites, blocking, and plugin approval, and `ctx.agentId` likely covers the commenter’s TTS argument-injection example. It still does not implement the issue’s central request: inspecting preceding assistant output and redirecting a pending tool call to another model/session.

## Best Possible Solution

Keep #13225 open as a plugin/runtime API design request. Preserve the shipped `before_tool_call` params/block/approval behavior and `before_model_resolve` run-level model override hook, but define a typed pre-tool-call routing contract only if maintainers want plugins to inspect preceding assistant output and redirect a pending tool call to another model or session. PRs #62701 and #58079 may cover context plumbing, but they do not by themselves close the model/session delegation request.
