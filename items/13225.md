## Summary

Keep #13225 open. Current main still has a useful `before_tool_call` plugin hook for parameter rewrites, blocking, and approval, and it carries `agentId` for the commenter’s TTS argument-injection use case. It still does not provide the issue’s central requested capability: a pre-tool-call decision surface that can inspect preceding assistant output and redirect a specific pending tool call to another model or session.

## Best Possible Solution

Keep the issue open as a plugin/runtime API design request. The shipped `before_tool_call` params/block/approval behavior and `before_model_resolve` model override hook should remain, but a complete fix would need an explicit typed pre-tool-call routing/delegation contract or a canonical successor issue/PR that defines how preceding assistant output and model/session redirects work safely.
