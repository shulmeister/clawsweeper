## Summary

Keep #13225 open. Current main still does not provide the requested pre-tool-call model/session routing contract: `before_tool_call` can rewrite params, block, or require approval, but it does not expose preceding assistant output or return a redirect/model/session routing decision. Existing `before_model_resolve` supports run-level model override only before session messages load. The related open PRs #58079 and #62701 are useful context work for hook context, but neither is shown as merged or as adding redirect semantics.

## Best Possible Solution

Keep #13225 open as a plugin/runtime API design request. The best path is an additive typed contract that preserves existing `before_tool_call` params/block/approval behavior and existing run-level `before_model_resolve`, while either implementing or explicitly rejecting a new per-tool-call route result that can inspect the relevant preceding assistant output and redirect the pending tool call to a selected model/session. Treat #58079 and #62701 as useful context work, not canonical closure unless they merge and are extended to cover routing.
