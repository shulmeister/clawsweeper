## Summary

Keep open. Current main still lacks KaTeX/MathJax/LaTeX math rendering in the Control UI chat markdown path, and PR #67103 is a focused implementation attempt against that current markdown-it renderer with added tests. It is not mergeable as-is because review comments identify a P0 markdown-rendering regression and other delimiter/sanitization issues, and it overlaps open PR #61510, but the work is still potentially useful and should be consolidated or reworked rather than auto-closed.

## Best Possible Solution

Keep the PR open for maintainer judgment, but require author rework before merge: restore the markdown cache/link-hardening helpers, add robust delimiter rules that do not parse currency or code spans as math, narrow KaTeX style/sanitizer handling, and coordinate with open PR #61510 plus issues #42840, #56689, and #62118 so there is one reviewed implementation path for Control UI math rendering.
