## Summary

Current main already handles this case: errored or aborted assistant tool-call turns no longer synthesize orphan tool results, and the latest shipped release v2026.4.22 already contains the same guard plus regression coverage.

Since the report was against `2026.1.24-3`, this appears fixed in newer builds. If you can still reproduce it on `v2026.4.22` or later, please open a fresh issue with an updated session snippet and provider/model details.
