## Summary

Keep open. Current main has a configurable shared inbound debounce path for text-only same-sender bursts, but it does not fully cover this issue’s stated scenarios: media/attachments/voice-note follow-ups are deliberately flushed immediately, and Telegram’s forwarded-burst handling is isolated from the normal-message lane, so a normal question followed by a forwarded context message is not clearly batched into one turn.

## Evidence
