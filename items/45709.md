## Summary

Keep open. Current main still renders the shared shell header on the Usage tab and the Usage view also renders its own Usage header, so the reported duplicate header remains on desktop. The earlier linked PR #45715 targeted this bug but was closed unmerged, and the related #67290 review also found the cleanup is still not implemented.

## Evidence

- **Shared shell header still includes usage:** The shell suppresses the shared content header only for state.tab === "config"; for usage it renders page-title/page-sub from titleForTab/subtitleForTab.

  - file: [ui/src/ui/views/usage.ts:441](https://github.com/openclaw/openclaw/blob/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071/ui/src/ui/views/usage.ts#L441)
