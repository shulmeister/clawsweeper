## Summary

Current main still silently drops `agentId` and `sessionKey` for `action: "wake"` mappings. The config type accepts those fields on hook mappings, but the wake mapping path builds and dispatches only `{ text, mode }`, and `dispatchWakeHook` still enqueues to `resolveMainSessionKeyFromConfig()`. PR #70268 is an open, unmerged fix for this exact bug, so the issue should stay open until that or an equivalent patch lands. The reload-mode concern in the first issue comment is separate follow-up material, not evidence that the wake-routing bug is resolved.

## Best Possible Solution

Keep #64556 open as the canonical bug until PR #70268, or an equivalent maintainer-approved patch, lands on main. The fix should route mapped wake hooks through the same agent/session policy used by agent hooks: carry `agentId`, `sessionKey`, and session-key source through the wake action, validate templated session keys with existing guardrails, normalize agent-scoped session targets, and add focused tests for explicit `sessionKey`, `agentId` fallback, and policy rejection. The separate reload-mode observation in the first comment should be tracked independently if still relevant.
