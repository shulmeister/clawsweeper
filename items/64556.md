## Summary

Current main still drops wake mapping routing fields, and open PR #70268 explicitly targets this issue. Maintainer cleanup should keep this issue open until #70268 is reviewed and either merged or closed.

## Best Possible Solution

Keep this issue open and use #70268 as the active implementation candidate. The best fix is to thread mapped wake `agentId` and `sessionKey` through `HookAction`, `server-http`, and `dispatchWakeHook`, reuse the same session-key policy and agent normalization used by agent hooks, and add regression tests for explicit mapped `sessionKey`, mapped `agentId` fallback to that agent's main session, templated-session guardrails, and unchanged legacy fallback when no routing fields are configured.
