## Summary

Current `main` already has a WhatsApp history-length setting. `channels.whatsapp.historyLimit` is part of the config contract, the WhatsApp runtime applies it when building injected group-history context, docs describe it as the supported knob, and schema/tests cover both root and per-account overrides.

## Evidence

  - file: [src/config/types.whatsapp.ts:75](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/config/types.whatsapp.ts#L75)

## Risks / Open Questions

- The current implementation clearly supports count-based history limits; the issue body also mentioned a possible age-based limit, which is not evidenced here.

## Close Comment

Codex review: current `main` already implements a WhatsApp history-length limit. The supported config is `channels.whatsapp.historyLimit` (with per-account overrides), and the runtime applies it when building WhatsApp group history context in `extensions/whatsapp/src/auto-reply/monitor.ts:147`. The behavior is documented in `docs/channels/whatsapp.md:255` and covered by schema/tests in `src/config/types.whatsapp.ts:75` and `extensions/whatsapp/src/config-schema.test.ts:42`. This support is present on `main` from commit `37c250695b600fb12afc799ee1b32d7aae73d096`, so I’m closing this as implemented on main. If you still need an age-based cutoff in addition to the existing count-based limit, please open a new focused issue for that specific behavior.
