## Summary

Current `main` does not implement the proposed `"discovery"` registration mode or an equivalent fix. Enabled non-activating loads still use `registrationMode: "full"`, plugin `register()` still runs during snapshot loads, and the SDK/docs/tests on `main` do not expose or describe a discovery-only mode. This PR should stay open.

## Evidence

- **Registration mode type on main has no discovery variant:** `PluginRegistrationMode` is still limited to `"full" | "setup-only" | "setup-runtime" | "cli-metadata"` on current `main`. There is no `"discovery"` mode in the public plugin API contract.
  - file: [src/plugins/types.ts:1990](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/plugins/types.ts#L1990)
  - command: `rg -n "export type PluginRegistrationMode" src/plugins/types.ts`
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Enabled non-activating loads still resolve to full:** In `loadOpenClawPlugins`, enabled plugins choose `"full"` unless they are in the setup-runtime/setup-only paths. There is no branch that maps `activate: false` to a distinct lightweight registration mode.
  - file: [src/plugins/loader.ts:2162](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/plugins/loader.ts#L2162)
  - command: `nl -ba src/plugins/loader.ts | sed -n '2156,2190p'`
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Snapshot loads still execute register() with that mode:** `createApi(..., { registrationMode })` is followed by `runPluginRegisterSync(register, api)` even for non-activating loads; only after registration does `!shouldActivate` trigger rollback of process-global state. That means plugin-side `register()` side effects still run during snapshot loads on current `main`.
  - file: [src/plugins/loader.ts:2711](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/plugins/loader.ts#L2711)
  - command: `nl -ba src/plugins/loader.ts | sed -n '2708,2768p'`
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Registry registration handlers are only enabled for full:** `createApi` only passes the capability-registration handlers when `registrationMode === "full"`; otherwise `buildPluginApi` falls back to no-op handlers. This is incompatible with the PR's proposed `"discovery"` mode where registration methods stay active.
  - file: [src/plugins/registry.ts:1340](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/plugins/registry.ts#L1340)
  - command: `nl -ba src/plugins/registry.ts | sed -n '1338,1639p'`
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Docs on main still describe only the existing modes:** The plugin SDK docs list `"full"`, `"setup-only"`, `"setup-runtime"`, and `"cli-metadata"` as the registration modes. There is no documentation of a `"discovery"` mode or equivalent snapshot-load signal.
  - file: [docs/plugins/sdk-entrypoints.md:196](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/docs/plugins/sdk-entrypoints.md#L196)
