## Summary

Keep #39476 open. Current main added a narrow skip for parent-owned one-shot ACP child sessions, but normal peer A2A `sessions_send` still exposes requester session context, still runs ping-pong plus announce, and still has no per-run guard that blocks the target/reply/announce turns from calling `sessions_send` back.

## Best Possible Solution

Keep the issue open and fix the core A2A `sessions_send` path. The maintainer-grade fix should add a framework-level guard that prevents A2A target/reply/announce turns from calling `sessions_send` back to the requester, and should narrow or remove requester-session prompt exposure where feasible. Preserve the current parent-owned ACP child skip as a separate narrow protection, keep #44309 for broader dispatch-only semantics, and add regression coverage for normal peer A2A duplicate visible delivery.
