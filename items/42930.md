## Summary

Current `main` already handles zero-text assistant turns by converting them into an explicit incomplete-turn error or retry path instead of letting WebChat end with an empty assistant response, and the vLLM docs now point OpenAI-compatible local servers at `openai-completions`/`/v1/chat/completions`.

## Evidence

- **Empty-response turns are now detected and surfaced as an explicit user-visible error:** `resolveIncompleteTurnPayloadText()` treats zero-payload, zero-text assistant turns as `emptyResponseAssistant` and returns `⚠️ Agent couldn't generate a response. Please try again.` instead of silently succeeding.
