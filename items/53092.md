## Summary

Keep open. Current main still does not validate `tools.web.search.provider` against the set of extension/plugin-registered web-search providers during config validation; unknown config values are accepted and later downgraded to a warning plus auto-detect fallback.

## Evidence

- **Schema still accepts any string:** `ToolsWebSearchSchema` defines `provider` as `z.string().optional()` with no enum/superRefine against registered providers.
  - file: [src/config/zod-schema.agent-runtime.ts:320](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/config/zod-schema.agent-runtime.ts#L320)
  - command: `nl -ba src/config/zod-schema.agent-runtime.ts | sed -n '316,333p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Provider registry exists from manifest contracts:** Web-search providers are resolved from manifest-declared `webSearchProviders`, so the repo already has a registered-provider source of truth available outside the config schema.
  - file: [src/plugins/web-search-providers.runtime.ts:37](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/plugins/web-search-providers.runtime.ts#L37)
  - command: `nl -ba src/plugins/web-search-providers.runtime.ts | sed -n '28,40p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Unknown configured provider only warns and falls back:** Runtime web-tool selection emits `WEB_SEARCH_PROVIDER_INVALID_AUTODETECT` with message `tools.web.search.provider is "...". Falling back to auto-detect precedence.` instead of rejecting config.
  - file: [src/secrets/runtime-web-tools.shared.ts:266](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/secrets/runtime-web-tools.shared.ts#L266)
  - command: `nl -ba src/secrets/runtime-web-tools.shared.ts | sed -n '264,278p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Tests codify current non-validating behavior:** `src/web-search/runtime.test.ts` explicitly expects a config typo (`provider: "missing-id"`) to succeed by falling back to `duckduckgo`, and `src/secrets/runtime-web-tools.test.ts` expects only a warning diagnostic on `tools.web.search.provider`.
  - file: [src/web-search/runtime.test.ts:400](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/web-search/runtime.test.ts#L400)
  - command: `nl -ba src/web-search/runtime.test.ts | sed -n '394,424p' && nl -ba src/secrets/runtime-web-tools.test.ts | sed -n '728,746p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)

## Risks / Open Questions

- Typos in `tools.web.search.provider` are still accepted at config-parse time and can silently change runtime behavior via auto-detect fallback.
- Current behavior is inconsistent: explicit runtime `providerId` fails fast for unknown providers, but the persisted config path does not.
