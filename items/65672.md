## Summary

Keep open. Current `main` partially reapplies Pi compaction settings after `resourceLoader.reload()`, but it does not preserve the `ownsCompaction` auto-compaction guard after reload and does not include the shared reload helper or regression coverage described in this PR.

## Evidence

- **`runEmbeddedAttempt` only reapplies compaction settings after reload:** `applyPiAutoCompactionGuard(...)` runs before `resourceLoader.reload()`, but after reload current main only calls `applyPiCompactionSettingsFromConfig(...)`. There is no second guard application in this path.
