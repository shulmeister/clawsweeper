## Summary

Keep #47264 open. The linked fix PR #47528 was closed unmerged, and current main still has the reported LINE multi-account failure mode: `accounts.default` is not used for the default LINE credentials, and named LINE accounts remain disabled unless each account explicitly sets `enabled: true`. Gateway startup then skips disabled accounts and marks missing default credentials as unconfigured, so the webhook-route 404 report is still actionable.

## Evidence

- **Default LINE account does not read accounts.default:** `resolveLineAccount` resolves the default account ID, but sets `accountConfig` to `undefined` when `accountId === DEFAULT_ACCOUNT_ID`; token and secret resolution therefore cannot read `channels.line.accounts.default.channelAccessToken` or `channelSecret`.

  - sha: [28497515fe1e](https://github.com/openclaw/openclaw/commit/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071)
- **Named LINE accounts default disabled:** The current enabled expression falls back to `false` for non-default accounts when `accountConfig.enabled` is omitted, so a configured account like the issue's `twgreen` example will not start unless it redundantly sets `enabled: true`.
