## Summary

Keep #68252 open. Current main still has the reported memory-core dreaming cleanup failure mode: memory-core calls `subagent.deleteSession()` in the narrative `finally` block and logs cleanup failures, while the gateway plugin runtime dispatches `sessions.delete` without admin scope and tests explicitly preserve the `missing scope: operator.admin` behavior for fallback deletion. The linked PR #70464 appears to be the active fix path, but it is open/unmerged in the provided context, so the bug is not shipped as fixed on main.

## Best Possible Solution

Keep #68252 open as the canonical current bug until PR #70464 or an equivalent maintainer-approved patch lands. The likely fix should allow bounded cleanup of memory-core-owned `dreaming-narrative-*` sessions without broadly granting admin to plugin fallback calls, and should keep regression coverage for the least-privilege gateway behavior plus the memory-core cleanup path.
