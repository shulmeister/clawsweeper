## Summary

Keep open. Current `main` still routes `memory-core` dreaming cleanup through plugin `subagent.deleteSession` -> gateway `sessions.delete`, and that gateway method is still admin-gated. The repo also contains a current test asserting the exact non-admin failure (`missing scope: operator.admin`) for plugin runtime session deletion. `v2026.4.12` shipped stale-artifact scrubbing to reduce orphan buildup, but that is a mitigation for leftovers, not a fix for the reported scope failure or warning path.
