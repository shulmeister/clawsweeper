## Summary

Keep #11040 open. Current main has partial direct-lineage support (`spawnedBy`, `parentSessionKey`, `spawnDepth`, `childSessions`), but it still does not implement the requested first-class parent/root/trace/depth session-chain contract. The exact implementation PR #11043 was closed unmerged, and the existing issue discussion already includes a Codex keep-open review with the same core gap.

## Best Possible Solution

Keep #11040 open until core implements the additive Phase 1 session-chain contract across `SessionEntry`, native and ACP spawn write paths, gateway protocol schemas/list rows/events, agent-facing `sessions_list` output, shared/UI row types where needed, docs, and multi-hop regression tests. If maintainers decide the existing direct-child `spawnedBy`/`childSessions` model is the intended boundary, close it explicitly with that product rationale instead of conservative cleanup.
