## Summary

Keep #11040 open. Current main has stronger partial lineage than the earlier review captured: `SessionEntry`, gateway rows, events, and spawn paths now persist/expose `spawnedBy`, `parentSessionKey`, and `spawnDepth`. But the requested first-class Phase 1 chain contract is still not implemented because there is no persisted `rootSessionKey` or session-chain `traceId` in the session model, patch protocol, gateway row type, sessions_list tool output, UI session row type, native spawn path, or ACP spawn path. The related implementation PR #11043 is closed unmerged in the provided GitHub context, and the latest maintainer comment on the issue already kept it open for the same remaining root/trace gaps.

## Best Possible Solution

Keep the issue open until core either implements the additive Phase 1 contract across `SessionEntry`, gateway schema/patch/list rows, native and ACP spawn write paths, `sessions_list` tool rows, UI/shared row types, and regression tests for multi-hop root/trace lineage, or a maintainer explicitly narrows the product direction to the current `spawnedBy`/`parentSessionKey`/`spawnDepth` model. A ClawHub observability plugin can consume the existing partial lineage, but it cannot create canonical persisted `rootSessionKey` and `traceId` fields for all spawned sessions without a core contract.
