## Summary

Current `main` only partially covers this request. OpenClaw now persists and exposes `spawnedBy`, `parentSessionKey`, `spawnDepth`, and `childSessions`, but it does not define or expose the proposed `rootSessionKey` or stable `traceId` on `SessionEntry`/`sessions.list`, so the requested first-class end-to-end chain metadata is not fully implemented.

## Evidence

- **SessionEntry has parent/depth metadata, but not root/trace:** `SessionEntry` currently defines `spawnedBy`, `parentSessionKey`, and `spawnDepth`; there is no `rootSessionKey` or session-chain `traceId` field in this type.
  - file: [src/config/sessions/types.ts:132](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/config/sessions/types.ts#L132)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Gateway session rows expose partial lineage only:** Gateway session list rows include `spawnedBy`, `spawnDepth`, `parentSessionKey`, and `childSessions`, which helps reconstruct some relationships, but there is still no `rootSessionKey` or `traceId` on the row shape.
  - file: [src/gateway/session-utils.types.ts:18](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/gateway/session-utils.types.ts#L18)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **sessions.list returns only the partial lineage fields:** `buildGatewaySessionRow()` copies `spawnedBy`, `spawnDepth`, `parentSessionKey`, and `childSessions` into `sessions.list` output. No root-chain or trace identifier is added here.
  - file: [src/gateway/session-utils.ts:1381](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/gateway/session-utils.ts#L1381)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **sessions_spawn writes spawnedBy and spawnDepth, not root/trace:** Subagent spawn patches the child session with `spawnDepth` first, then `spawnedBy`, and emits a lifecycle event carrying `parentSessionKey`; it does not persist `rootSessionKey` or `traceId` on the session entry.
  - file: [src/agents/subagent-spawn.ts:743](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/agents/subagent-spawn.ts#L743)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Protocol/docs do not define the requested root/trace fields:** `SessionsListParamsSchema`/`SessionsPatchParamsSchema` cover `spawnedBy`, `parentSessionKey`, and `spawnDepth`, and `docs/concepts/session-tool.md` documents `sessions_list` at a high level without any root/trace chain fields. A repository search for `rootSessionKey`, `traceId`, or `sessions.chain` under session/gateway protocol surfaces returned no matches.
  - file: [src/gateway/protocol/schema/sessions.ts:54](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/gateway/protocol/schema/sessions.ts#L54)

  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **There is internal descendant tracking, but it lives in the subagent registry, not session rows:** The subagent registry can walk descendants from a `rootSessionKey`, but that lineage is derived from `requesterSessionKey`/`childSessionKey` in registry records rather than first-class session metadata exposed by `sessions.list`.
  - file: [src/agents/subagent-registry-queries.ts:200](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/agents/subagent-registry-queries.ts#L200)
