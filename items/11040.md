## Summary

Keep #11040 open. Current main has useful partial lineage support (`spawnedBy`, `parentSessionKey`, `spawnDepth`) in gateway/session surfaces, but it still does not implement the requested first-class chain model: `SessionEntry` and gateway rows have no `rootSessionKey` or session-chain `traceId`, the patch schema cannot persist those fields, and the `sessions_list` tool output still omits `spawnDepth` along with root/trace metadata. The related PR #11043 claimed to implement the request but is closed unmerged in the provided GitHub context, and the latest maintainer review comment already kept the item open for the same remaining gaps.

## Best Possible Solution

Keep the issue open until OpenClaw either implements the additive Phase 1 contract in core (`SessionEntry`, gateway schema/patch/listing, native and ACP spawn write paths, `sessions_list` tool rows, and regression tests for multi-hop lineage) or a maintainer explicitly decides that stable root/trace chain identifiers are out of core scope. A ClawHub observability plugin can build on existing partial lineage, but it cannot create canonical persisted `rootSessionKey`/`traceId` fields for all spawned sessions without a core contract.
