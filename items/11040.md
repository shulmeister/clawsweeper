## Summary

Keep #11040 open. Current main has partial lineage support (`spawnedBy`, `parentSessionKey`, `spawnDepth`) but does not implement the requested first-class chain model: there is no `rootSessionKey` or `traceId`, and the `sessions_list` tool output still does not surface `spawnDepth`. The related PR #11043 claimed to implement the full request but was closed unmerged in the provided GitHub context, so it does not supersede the issue.
