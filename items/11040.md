## Summary

Keep #11040 open. Current main still has only partial lineage support (`spawnedBy`, `parentSessionKey`, `spawnDepth`) and does not implement the requested first-class `rootSessionKey` plus stable session-chain `traceId` contract across persisted sessions, gateway protocol/list rows, agent-facing `sessions_list`, or native/ACP spawn write paths. The related PR #11043 was closed unmerged, and the latest issue discussion already contains a maintainer Codex review saying to keep it open.

## Best Possible Solution

Keep #11040 open until core implements the additive Phase 1 chain contract across `SessionEntry`, gateway schemas/list rows/events, native and ACP spawn write paths, agent-facing `sessions_list`, UI/shared row types where needed, and multi-hop regression tests. If maintainers decide the current `spawnedBy`/`parentSessionKey`/`spawnDepth` model is the intended boundary, narrow or close the issue explicitly with that product rationale.
