## Summary

Current `main` intentionally runs `BOOT.md` on every `gateway:startup`, and the docs clearly separate that from the one-time `BOOTSTRAP.md` flow. That makes this read more like a product-change request than a conservative cleanup close candidate, so it should stay open.
