## Summary

Current main still does not implement remote http(s) markdown image rendering in Control UI chat. The markdown renderer still emits inline images only for base64 data-image sources, tests still assert remote images are flattened to alt text, and the Control UI CSP/docs still block remote image loads. The provided GitHub context also shows open PR #41025 references this issue with closing syntax and remains unmerged, so this issue should stay open while maintainers review, revise, or close that implementation candidate.

## Best Possible Solution

Keep this issue open and review PR #41025 as the implementation candidate. If maintainers want remote images in Control UI chat, the best fix should address both the markdown renderer and the image-loading policy, likely through an explicit allowlist, managed-media/proxy path, or documented opt-in rather than only permitting http(s) URLs in the renderer.

## Risks / Open Questions

- A renderer-only fix would not solve the visible behavior because current Control UI CSP still blocks remote http(s) image loads.
- Allowing remote markdown images would make the operator browser fetch model-supplied third-party URLs, which has privacy and tracking implications and conflicts with the current documented policy unless maintainers intentionally change it.
