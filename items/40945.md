## Summary

Current main still does not render remote http/https markdown images in Control UI chat. The markdown renderer only emits an `<img>` for base64 `data:image/...` sources and the tests assert that remote images flatten to alt text. A newer Control UI CSP and docs also explicitly reject remote image fetches, so the linked open PR #41025 is not enough by itself and this needs maintainer product/security judgment rather than cleanup closure.

## Best Possible Solution

Keep #40945 open as the canonical Control UI remote-image product/security decision. If maintainers want remote inline images, implement a safe design such as an authenticated same-origin managed-image proxy or explicit allowlist/config, then update the markdown renderer, CSP, tests, and Control UI docs together. If maintainers want the current hardening to stand, close this with an explicit maintainer rationale and close or retarget PR #41025 accordingly.
