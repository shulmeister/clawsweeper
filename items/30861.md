## Summary

Keep #30861 open. Current main still has a chat session selector grouped by agent and avatar refresh/sync on session switches, but it does not implement the requested persistent active-agent plus workspace indicator, workspace tooltip, switch feedback, or single-agent minimal badge behavior. The related PR #31602 was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep #30861 open as the tracker for a Control UI chat-header context indicator. A good implementation would resolve the active agent from the current session key, surface the agent id/name and workspace path from the agents/session data, keep the session selector and slash-command state synchronized after reload/reconnect, and add focused UI helper tests for multi-agent and single-agent behavior.
