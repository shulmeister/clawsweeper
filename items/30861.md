## Summary

Keep #30861 open. Current main has a chat session selector whose options are grouped by agent, and the gateway/UI already carry agent workspace data, but the requested persistent chat-header active-agent plus workspace indicator, workspace tooltip, switch confirmation feedback, and single-agent minimal behavior are still absent. The related PR #31602 was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep #30861 open as the canonical tracker for a Control UI chat-header context indicator. A good implementation should derive the active agent from the current session key, show agent id/name and workspace path from `agents.list`, keep `switchChatSession` and persisted `lastActiveSessionKey` synchronized after reload/reconnect, add switch feedback, keep the single-agent case visually minimal, and add focused UI helper/render tests for multi-agent and single-agent behavior. The closed PR #31602 may be useful prior art, but it did not land.
