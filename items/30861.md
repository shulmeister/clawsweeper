## Summary

Keep #30861 open. Current main still has adjacent session-selection and agent-list plumbing, but it does not implement the requested persistent Dashboard chat-header active-agent/workspace indicator, full workspace tooltip, stable visual agent badge, or agent-switch confirmation feedback. The backend already exposes agent workspace metadata, and docs describe multi-agent workspaces as a real product concept, so this remains a focused Control UI feature gap rather than a missing backend contract. Related PR #31602 was closed unmerged and does not supersede the issue.

## Best Possible Solution

Keep #30861 open as the Control UI tracker. Implement a minimal chat-header context indicator derived from the active `sessionKey` plus `agents.list`, showing agent id/name and workspace path with a full-path tooltip, updating through `switchChatSession` and slash-command/session changes, surviving reload via persisted session state, adding clear switch feedback, staying quiet for single-agent setups, and adding focused render/helper tests. Closed PR #31602 can inform a fresh patch but should not be treated as canonical.
