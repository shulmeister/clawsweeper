## Summary

Keep open. Current `main` still unconditionally emits the embedded-runner `llm_output` hook even when the current attempt produced no assistant output, so the PR is not obsolete. The PR's core direction addresses a real gap, but its submitted diff also bundles an unrelated `chat-content` parsing change with unresolved review concerns, so it should not be auto-closed or treated as cleanly superseded.

## Best Possible Solution

Keep the PR open for maintainer review, but ask for a focused fix: guard embedded-runner `llm_output` on actual current-attempt assistant output, add a regression test for timeout/no-output behavior, and split or drop the unrelated `chat-content` JSON parsing change unless it is fixed and justified separately.
