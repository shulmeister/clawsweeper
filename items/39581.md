## Summary

Keep PR #39581 open. Current main still emits the embedded-runner `llm_output` hook whenever the hook is registered, even when the current attempt produced no assistant output, and it passes a `lastAssistant` value that is full-history scoped rather than current-attempt scoped. The PR addresses a real remaining issue, but should be narrowed because its branch also includes unrelated `chat-content` JSON parsing changes with unresolved review feedback.

## Best Possible Solution

Keep the PR open, but request a focused patch: guard embedded-runner `llm_output` on actual current-attempt assistant output, avoid stale `lastAssistant`, preserve normal-output hook emission, add regression coverage for timeout/no-output and normal-output cases, and split or drop the unrelated `chat-content` JSON parsing change unless it is fixed and justified separately.
