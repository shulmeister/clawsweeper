## Summary

Keep PR #39581 open. Current main still emits the embedded-runner `llm_output` hook whenever hooks are registered, even when the current model attempt produced no assistant output, so the central timeout/no-output bug is not implemented or superseded. The PR remains useful, but it should be narrowed before merge because it also includes an unrelated `chat-content` JSON parsing change that review comments flagged as risky.

## Best Possible Solution

Keep the PR open, but ask for a focused patch: guard embedded-runner `llm_output` on actual current-attempt assistant output, avoid reusing stale `lastAssistant`, add regression coverage for timeout/no-output and normal-output cases, and split or drop the unrelated `chat-content` JSON parsing change unless it is fixed and justified separately.
