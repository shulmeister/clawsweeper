## Summary

Current `main` already expands Linux `openclaw doctor` gateway diagnostics beyond the thin systemd-exists check described in #14084. The doctor flow now audits Linux systemd service config, reads both `Environment=` and `EnvironmentFile=` values, detects entrypoint/token/PATH/runtime drift, checks key unit settings, and offers repair/reinstall flows.
