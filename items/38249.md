## Summary

Keep #38249 open. Current main has stricter active-schema validation and doctor can prune unknown keys, but it still does not implement `openclaw doctor --config-diff`, version-to-version schema comparison from `meta.lastTouchedVersion`, categorized upgrade reporting, migration suggestions, or `--config-diff --fix`. Related #39780 is a Control UI before/after apply flow, and closed #40015 does not supersede this CLI migration-diff request.

## Best Possible Solution

Keep #38249 open until maintainers implement or explicitly reject a CLI upgrade-diff workflow. A complete solution should add `openclaw doctor --config-diff` plumbing, docs, and tests; compare the active config against the current schema and a trustworthy previous-version baseline from `meta.lastTouchedVersion`; report unknown keys, invalid enum values, removed/renamed/new keys, and migration suggestions; and reserve `--fix` for explicit safe migrations with backups. Keep #39780 separate for Control UI before/after apply UX.
