## Summary

Keep open. Current main has useful config validation and doctor repair behavior, including unknown-key stripping and `meta.lastTouchedVersion` tracking, but it still does not implement the requested `openclaw doctor --config-diff` CLI surface, version-to-version schema diff output, new-key report, or migration guidance. The related open #39780 is UI-specific before/after config application work, and closed #40015 is about safe agent-driven config writes, so neither supersedes this CLI doctor migration request.

## Best Possible Solution

Keep this issue open until OpenClaw either implements a `doctor --config-diff` workflow that compares the config's `meta.lastTouchedVersion` against the current schema and reports removed/renamed/new keys plus changed enum values and safe fixes, or a maintainer explicitly decides that versioned schema migration guidance should be handled by a narrower canonical issue. #39780 should remain the UI-specific before/after config-apply track; #40015 does not replace this CLI doctor surface.
