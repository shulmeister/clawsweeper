## Summary

Keep #38249 open. Current main has current-schema validation, `meta.lastTouchedVersion` stamping, legacy migration rules, and generic `doctor --fix` unknown-key pruning, but it still does not provide the requested `openclaw doctor --config-diff` CLI flag, version-to-version schema comparison, categorized new/removed/renamed key reporting, or actionable upgrade migration guidance.

## Best Possible Solution

Keep #38249 open until maintainers either implement or explicitly reject a real CLI upgrade-diff workflow. A complete fix should add `openclaw doctor --config-diff` plumbing, tests, and docs; compare the active config against current schema plus a trustworthy previous-version baseline from `meta.lastTouchedVersion`; report unrecognized keys, invalid enum values, new/removed/renamed keys, and migration suggestions; and reserve `--fix` for safe migrations with backups. #39780 can remain the separate Control UI before/after apply-flow item.
