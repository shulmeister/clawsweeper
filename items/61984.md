## Summary

Keep #61984 open. Current main has the valid API adapter enum in code/generated schema and some nearby custom-provider docs, but the user-facing docs/help still do not comprehensively document the allowed `models.providers.<name>.api` values, which API format/baseUrl suffix to use for common endpoints, or the requested provider-prefix stripping behavior. The related prefix bugs (#50647/#50711) appear addressed separately but do not supersede this docs request.
