## Summary

Keep open. Current `main` still exposes TTS as provider-specific config plus per-reply `[[tts:...]]` overrides; it does not have a provider-agnostic persona layer, persona selection surface, or the proposed speech-core rewrite/default-tags flow. The issue also already has an open linked PR (#70748), so conservative cleanup should not close it.

## Evidence

- **TTS config schema has no persona fields:** `TtsConfig` on current `main` only defines `auto`, `enabled`, `mode`, `provider`, `summaryModel`, `modelOverrides`, `providers`, `prefsPath`, `maxTextLength`, and `timeoutMs`. There is no `persona`, `personas`, `personaPrompt`, `defaultTags`, or rewrite sub-config in the shipped schema.
