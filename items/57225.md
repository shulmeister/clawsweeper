## Summary

Keep open. Current `main` still matches the reported behavior: Discord does not force block streaming on, and when block streaming is disabled the reply delivery path explicitly buffers text-only block replies until final delivery. The issue comment's workaround also matches current code: tool-start flush hooks exist, but they do not make text blocks reach the channel unless block streaming is enabled.
