## Summary

Keep open. Current `main` still matches the reported bug: the queued announce path in `src/agents/subagent-announce-delivery.ts` sends `deliver: true` for any non-subagent requester even when `origin.channel`/`origin.to` are missing, while the direct path still uses best-effort target resolution that disables delivery when no concrete external target exists. In the gateway handler, a delivery request that resolves to the internal channel still falls through to `resolveMessageChannelSelection`, which throws the same multi-channel error described in the issue. The only linked fix work is PR #70585, and the provided timeline still shows it as open, so this is not implemented on `main` yet.

## Evidence

- **Queued announce path still forces delivery without validating origin target:** `sendAnnounce` passes `channel`, `accountId`, `to`, and `threadId` from `item.origin`, but it still sets `deliver: !requesterIsSubagent` unconditionally for depth-0/non-subagent sessions. There is no `origin?.channel && origin?.to` guard and no queue-path `bestEffortDeliver` fallback here.
  - file: [src/agents/subagent-announce-delivery.ts:299](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/subagent-announce-delivery.ts#L299)
  - command: `nl -ba src/agents/subagent-announce-delivery.ts | sed -n '299,340p'`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Direct path still uses guarded best-effort delivery instead:** `sendSubagentAnnounceDirectly` still routes non-subagent delivery through `resolveExternalBestEffortDeliveryTarget(...)`, so the direct path and queued path remain inconsistent on current `main`. The helper returns `deliver: true` only when both a deliverable `channel` and `to` are present.
  - file: [src/agents/subagent-announce-delivery.ts:475](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/subagent-announce-delivery.ts#L475)

## Risks / Open Questions

- Originless depth-0 announce sessions can still exhaust retries or fail outright on multi-channel gateways because the queued path requests external delivery without a concrete target.
- Current tests demonstrate direct-path delivery behavior, but there is still no merged proof on `main` that the queued announce path downgrades to session-only or best-effort delivery when origin data is missing.
