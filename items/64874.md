## Summary

Keep open. Current main has adjacent bootstrap diagnostics through `/context` and `doctor`, and it persists the data needed for `/status`, but it does not implement this PR's central changes: `/status` still lacks per-file bootstrap health, edit mismatch errors do not mention bootstrap truncation, and the injected bootstrap warning still lacks the proposed edit-failure guidance. The linked feature issue #45415 remains open, while related PR #60288 was closed unmerged, so this draft PR is not obsolete or superseded.

## Best Possible Solution

Keep the PR open as a plausible implementation candidate, but require a rebase/update before review: move the `/status` change to `src/status/status-message.ts`, derive thresholds from current bootstrap config/report data, avoid hardcoded 20,000-char messaging, and add focused tests for bootstrap warning text, edit mismatch diagnostics, and `/status` rendering from `sessionEntry.systemPromptReport`. Maintainers should then decide whether `/status` should duplicate the per-file `/context` summary or link users to `/context list` with a concise health warning.
