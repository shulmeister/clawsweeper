## Summary

Current main already supports the requested cross-channel continuity for direct-message sessions: DMs share the agent main session by default, operators can opt into cross-channel peer linking with `session.identityLinks`, WebChat uses the same gateway session/routing model, and the feature was documented as shipped in release `2026.1.16-1`.

## Evidence

  - file: [docs/channels/channel-routing.md:24](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/docs/channels/channel-routing.md#L24)

  - file: [docs/gateway/config-agents.md:1103](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/docs/gateway/config-agents.md#L1103)

## Risks / Open Questions

- Operators can still intentionally disable this continuity with `session.dmScope: "per-channel-peer"` or `per-account-channel-peer`, so the reported behavior can still appear as a configuration outcome even though core support exists.
