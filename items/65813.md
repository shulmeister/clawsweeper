## Summary

Current `main` still treats rate-limit auth-profile rotation as a capped same-provider retry budget, then escalates to model fallback without waiting for every profile to be independently cooldowned. The exact 'retry back onto the exhausted primary in the same request chain' path appears guarded now, but the core behavior reported in #65813 is still present and is explicitly documented and tested.

## Evidence
