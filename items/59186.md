## Summary

Close as cannot reproduce: current `main` has explicit direct StepFun support, but the reported config mixes the standard `stepfun` provider with the Step Plan endpoint and uses an unsupported bare env-var placeholder (`STEP_API_KEY`), which current auth handling intentionally treats as a literal string unless you use the documented `STEPFUN_API_KEY` marker or a SecretRef form.

## Evidence

- **Current StepFun contract:** Bundled StepFun plugin defines two separate providers: `stepfun` and `stepfun-plan`, and both advertise `STEPFUN_API_KEY` as the provider auth env var. The Step Plan surface is a distinct provider, not a `stepfun` base URL override.
  - file: [extensions/stepfun/openclaw.plugin.json:4](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/stepfun/openclaw.plugin.json#L4)
  - command: `nl -ba extensions/stepfun/openclaw.plugin.json | sed -n '1,120p'`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Docs show supported StepFun setup:** The user-facing docs say `stepfun` is the standard endpoint, `stepfun-plan` is the Step Plan endpoint, the auth env var is `STEPFUN_API_KEY`, and advanced config examples use `apiKey: "${STEPFUN_API_KEY}"` rather than an arbitrary bare env name.
  - file: [docs/providers/stepfun.md:9](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/docs/providers/stepfun.md#L9)
  - command: `nl -ba docs/providers/stepfun.md | sed -n '1,210p'`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Auth code does not treat arbitrary all-caps strings as env refs:** `resolveUsableCustomProviderApiKey()` returns unknown string values as literal `models.json` API keys, and only switches to env lookup for known env-marker names. This means a config value like `STEP_API_KEY` is sent literally unless it is encoded as a SecretRef / `${VAR}` form.
  - file: [src/agents/model-auth.ts:153](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/model-auth.ts#L153)
  - command: `nl -ba src/agents/model-auth.ts | sed -n '153,175p'`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Marker guardrail is intentional and tested:** Tests explicitly assert that known markers such as `OPENAI_API_KEY` are recognized, but arbitrary all-caps keys are not. Separate tests cover unknown env IDs only through SecretRef objects, not bare strings.
  - file: [src/agents/model-auth-markers.test.ts:92](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/model-auth-markers.test.ts#L92)
