## Summary

Current main already preserves locally optimistic Control UI chat messages across stale `chat.history` reloads, with regression coverage and docs. The open PR's temp-id map approach is obsolete and had unresolved review feedback about stale pending entries, content-only dedupe, and cross-session leakage.

## Best Possible Solution

Close this PR and keep the current main implementation: it solves the observable Control UI history-reload race with a smaller tail-preservation helper, docs, and regression tests, while avoiding the PR's unresolved pending-map review issues. Any future edge case should be handled as a focused follow-up against the shipped helper rather than by merging this stale PR.
