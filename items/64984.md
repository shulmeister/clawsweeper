## Summary

Current `main` appears to have fixed the session-reset/session-delete half of the MCP runtime leak, but not the gateway shutdown/restart half that #64984 explicitly reports. The code now retires a session-scoped MCP runtime after embedded-run shutdown during reset, and gateway tests cover that path, yet the gateway close path still does not call `disposeAllSessionMcpRuntimes()`. An open PR (#67567) is still tracking exactly that missing server-close wiring. Because the report is about reconnect/restart leaks and the restart path remains unproven on `main`, this is not strong enough to auto-close.
