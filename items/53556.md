## Summary

Current main still only partially covers the reported Feishu backward-compatibility cases. Legacy `groupPolicy: "allowall"` and `topicSessionMode` are handled, but old `botId` credentials are neither accepted nor migrated to `appId`, and there is no Feishu doctor compatibility rule or documented upgrade path for that remaining migration.

## Best Possible Solution

Keep this issue open and scope it to a Feishu-owned compatibility path: add explicit `botId` to `appId` handling, preferably through Feishu doctor `legacyConfigRules` or `normalizeCompatibilityConfig` plus focused schema/doctor tests, keep the existing `topicSessionMode` and `allowall` compatibility covered, and document the supported upgrade path on the Feishu docs page.
