## Summary

Keep PR #33914 open. Current main at e672b61417af5c45b0431df6d9109a1f4b618ef5 still does not implement the central plugin-hook identity context requested by the PR: `PluginHookAgentContext` lacks `sourceProvider`, sender identity, group, and parent-session fields, and the main agent plus compaction hook contexts do not pass them. The PR is stale and broad, but the remaining product/API gap is real and supported by plugin-author discussion.

## Best Possible Solution

Keep this PR open for maintainer review or a narrowed rebase. The best path is an additive plugin SDK/context update exposing `sourceProvider`, `senderId`, `senderName`, `senderIsOwner`, `groupId`, and `spawnedBy` on `PluginHookAgentContext`; deriving normalized origin identity consistently; preserving null semantics; threading the fields through early run, attempt, queued/memory, preflight/direct compaction, and reset hook contexts; and either completing or splitting the outbound `sessionKey` message-hook correlation work.
