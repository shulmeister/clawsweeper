## Summary

Keep PR #33914 open. Current main still does not implement the PR’s central plugin API change: `PluginHookAgentContext` lacks `sourceProvider`, `senderId`, `senderName`, `senderIsOwner`, `groupId`, and `spawnedBy`, and the main agent/compaction hook contexts are built without those fields. The already-landed `PluginHookMessageContext.sessionKey` work only covers the message-hook slice, so the trust-aware agent hook gap remains real core SDK work rather than cleanup, ClawHub, or a duplicate close.

## Best Possible Solution

Keep this open for maintainer review or a narrowed rebase. The best implementation path is an additive plugin SDK/context update that exposes `sourceProvider`, `senderId`, `senderName`, `senderIsOwner`, `groupId`, and `spawnedBy` on `PluginHookAgentContext`; derives origin/provider identity consistently; preserves null-vs-undefined semantics; threads the fields through normal runs, queued runs, memory flush, compaction/reset, and relevant agent hook emission paths; and treats the already-landed message-hook `sessionKey` work as baseline rather than reintroducing stale changes.
