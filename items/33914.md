## Summary

Keep PR #33914 open. Current main only implements the message-hook sessionKey slice; it still does not expose sourceProvider, sender identity, groupId, or spawnedBy on PluginHookAgentContext, and the main agent/compaction hook emission paths still build minimal contexts. This is a real plugin API gap, not ClawHub-only work, because external plugins cannot add missing core hook context fields themselves.

## Best Possible Solution

Keep the item open for maintainer review. The best path is an additive plugin SDK/context update that exposes the requested trust fields on PluginHookAgentContext, derives sourceProvider from normalized origin metadata, preserves null semantics for confirmed-absent sender/group/parent context, threads the fields through all agent and compaction hook emission paths, and keeps the already-landed message-hook sessionKey work as the baseline.

## Risks / Open Questions

- PR #33914 is partially stale because main independently landed message-hook sessionKey plumbing and related hook refactors; it needs a careful rebase or narrower cherry-pick rather than a blind merge.

- This changes the public plugin SDK contract, so docs, generated API/baseline checks, and focused propagation tests should accompany the runtime changes.
