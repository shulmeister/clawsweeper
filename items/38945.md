## Summary

Keep #38945 open. Current main still lacks the PR's broader Unicode, combining-mark, and NFC tokenizer fixes after memory moved to `extensions/memory-core`; later CJK work does not cover accented Latin, Cyrillic, Arabic/Hebrew, Thai/Hindi combining marks, or NFC/NFD equivalence for MMR and FTS tokenization.

## Best Possible Solution

Keep the PR open and port the narrow tokenizer fix to current memory-core: update MMR tokenization, `buildFtsQuery()`, and shared FTS search token planning to use NFC-normalized Unicode letter/mark/number tokens while preserving shipped CJK behavior. Add focused tests for accented Latin/Finnish, Cyrillic, Arabic/Hebrew, Thai/Hindi combining marks, and NFC/NFD normalization.
