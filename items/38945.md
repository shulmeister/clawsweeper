## Summary

Keep #38945 open. Current main at 73e21511075d7ed5651faae305f439d65b6ef47c still lacks the PR’s broader Unicode, combining-mark, and NFC tokenizer fix in the moved memory-core implementation. Later shipped CJK work improves Chinese/Japanese/Korean paths, but it does not supersede the non-CJK Unicode and combining-mark behavior this PR targets.

## Best Possible Solution

Keep the PR open and port the narrow tokenizer fix to current memory-core: update `extensions/memory-core/src/memory/mmr.ts`, `extensions/memory-core/src/memory/hybrid.ts`, and the shared keyword tokenization in `extensions/memory-core/src/memory/manager-search.ts` as needed. Add regression tests for combining-mark scripts and NFC/NFD normalization while preserving the shipped CJK-specific tokenization and trigram fallback behavior.

## Evidence

The author also addressed the review comment about normalizing before lowercasing in commit 7398a23fc53e791f9b4d6440962e69881b8f6028. This is a plausible focused bug fix, not low-signal cleanup.
- **Current MMR tokenizer is still ASCII plus CJK-specific:** `tokenize()` lowercases through `normalizeLowercaseStringOrEmpty`, extracts ASCII with `/[a-z0-9_]+/g`, then adds CJK unigrams/bigrams.
