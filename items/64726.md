## Summary

Keep open. Current main partially addresses the request with batch embedding, exact-hash embedding cache reuse, cache entry counts, session-start syncing, and max-entry pruning, but the central requested cache improvements remain missing: fuzzy reuse for minor edits, hit/miss/latency metrics, gradual model migration, and vector compression. Related items in the provided context cover narrower timeout/retry behavior and do not supersede this broader feature request.

## Best Possible Solution

Keep this issue open but narrow it around the missing cache behavior. Batch embedding and max-entry pruning already exist in memory-core, so the best implementation path is to add explicit cache metrics and decide whether fuzzy reuse, model migration, and vector compression are safe enough for the built-in memory cache or should become separate, well-scoped follow-up issues.
