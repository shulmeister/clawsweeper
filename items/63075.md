## Summary

Keep #63075 open. Current main still promises and attempts to clear Discord native commands when `commands.native=false`, but the same clear-on-disable path already existed in the reported v2026.4.8 build, and there is no focused regression coverage proving the stale-command UI failure is fixed. This is distinct from the broader empty `Done` native-command issue (#53416).

## Best Possible Solution

Keep the issue open. The best fix is a narrow Discord provider change with regression tests around `channels.discord.commands.native=false`: prove startup sends the expected empty command overwrite, surface clear failures in logs/status, and, if historical or guild-scoped registrations are possible, clear every scope OpenClaw may have registered. Then live-verify with a Discord bot that disabling native commands removes previously visible slash commands after Discord's normal propagation window.
