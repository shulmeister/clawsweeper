## Summary

Current `main` already allows plugin-owned `plugins.entries.<id>.config` payloads at the gateway schema layer and then validates them against each discoverable plugin's manifest `configSchema`, so the reported `must NOT have additional properties` startup failure no longer matches shipped behavior.
