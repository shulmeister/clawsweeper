## Summary

Keep open. Current main still has strict-by-default browser SSRF blocking plus hostname exceptions, but not the requested confirmation mode, per-request approval timeout/rate limiting, CIDR/port allowlist/blocklist, non-overridable metadata blocking under broad private-network allow, or temporary `openclaw ssrf` CLI controls.

## Best Possible Solution

Keep this issue open as the canonical product/security enhancement. A complete implementation should extend the browser SSRF config/type/schema and navigation guard with a maintainer-approved confirm mode, explicit allowlist/blocklist semantics with metadata endpoints blocked even in broad allow mode, UI or channel approval plumbing with timeout/rate limiting, tests for redirects/DNS rebinding/proxy cases, and docs for the final config/CLI surface if CLI toggles are accepted.
