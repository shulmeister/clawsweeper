## Summary

Keep open. Current main has subagent spawn guardrails for depth, active children, agent allowlists, timeouts, and provider-level cooldown/fallback behavior, but it does not implement the requested native subagent failure circuit breaker: no config schema, no persisted consecutive-failure/cooldown state, no spawn-time breaker check, and no CLI/UI status surface. Existing plugin hooks can support workarounds, but the issue asks for first-class core reliability behavior. The related #65817 is the same topic but closed, while #20341 is a closed-unmerged heartbeat PR and #60266 addresses model fallback switching, not subagent cascade blocking.

## Best Possible Solution

Keep #66010 open as the canonical subagent cascade circuit-breaker request. The likely implementation belongs in the core subagent spawn/registry path: add a typed `agents.defaults.subagents.circuitBreaker` config contract, persist failure streak and cooldown state keyed by target agent and/or task signature, check the breaker before `sessions_spawn` starts work, reset on successful completion, expose forced override/manual clear, and surface state in `openclaw agents status` and the Control UI. Provider-level cooldown/cost controls should be tracked separately.

## Risks / Open Questions

- The requested feature overlaps provider cooldown/cost-cap concerns; implementation should keep provider failure policy separate from subagent spawn/result failure policy.
- A native breaker could block legitimate recovery if keyed too broadly, so the design needs explicit reset-on-success, cooldown expiry, manual override, and clear operator visibility.
