## Summary

Keep open. Current main still emits a generic `Run: openclaw doctor --non-interactive` restart-sentinel hint from config patch/apply, update, and restart paths, then delivers the same message through the heartbeat/system-event wake path without context-aware actionability guidance for approval-constrained surfaces.

## Best Possible Solution

Keep the issue open and implement a narrow restart-sentinel follow-up renderer. It should distinguish a recommended doctor check from a command that is actionable in the current surface, and for heartbeat/session-only or approval-constrained delivery say to run it in a terminal or approvals-capable Control UI, or mark the check as deferred.
