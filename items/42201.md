## Summary

Keep open. Current main still rejects unknown `openclaw.json` keys through strict schema validation and still aborts startup on invalid config snapshots, so the PR's core behavior is not implemented. The PR discussion also shows the contributor iterated on real review concerns rather than leaving an obviously obsolete or low-signal change.

## Best Possible Solution

Keep the PR open for maintainer judgment. The best path is to either rebase/refit this implementation or explicitly consolidate it with the current canonical unknown-config-key tracker/PR, preserving strict validation for known invalid fields, legacy keys, and config writes while adding a tested tolerant startup/read mode for unknown keys.
