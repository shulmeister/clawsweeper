## Close Comment

Closing as implemented on main after a Codex review. The current Podman path no longer uses the old build-as-one-user then save/load into a second `openclaw` user store flow that matches the reported `/var/tmp/openclaw-podman-image.../openclaw-image.tar` permission failure. `scripts/podman/setup.sh:2` now keeps setup/build in the current non-root user and directly `podman build`s or `podman pull`s the image (`scripts/podman/setup.sh:369`), while `scripts/run-openclaw-podman.sh:502` launches with `--userns=keep-id` plus the caller's uid/gid. The docs were updated to the same model in `docs/install/podman.md:8`. This implementation is present in the shipped `v2026.4.22` release (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). If you still hit a WSL-specific Podman permission error on `v2026.4.22` or newer, please open a fresh issue with the exact command, full current script output, and `podman info` details.
