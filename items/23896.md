## Summary

Keep #23896 open. Current main still lacks Discord per-sender mention gating: `requireMentionFromIds` / `require_mention_from_ids` is absent from config types, strict schema, generated metadata, docs, tests, and the Discord preflight path. Runtime still resolves one guild/channel `requireMention` boolean and then applies sender allowlists separately, so the reported bridge-bot-versus-human policy cannot be represented.

## Best Possible Solution

Keep #23896 open. Best path is an additive Discord plugin/config change: add a typed sender-scoped mention policy such as `requireMentionFromIds` at guild/channel level, extend strict zod schema plus generated metadata/docs, make the Discord mention resolver sender-aware after sender/PluralKit identity resolution, and add preflight tests proving listed human senders must mention while unlisted allowed bridge bots can trigger without mentions.
