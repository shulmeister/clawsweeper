## Summary

Current `main` has partial context-cost mitigations and plugin seams, but it still rebuilds the system prompt each run and defaults to the legacy pass-through context engine. I did not find a shipped core change that provides the requested general lazy-load/lightweight mode for ordinary replies, so this should stay open.

## Evidence

- **System prompt still rebuilt every run:** The context docs say OpenClaw rebuilds its system prompt on every run, and that prompt includes tools, skills metadata, runtime data, and injected workspace files. The example `/context list` also shows large baseline overhead from tool schemas and project context.

  - file: [docs/concepts/context-engine.md:102](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/docs/concepts/context-engine.md#L102)
