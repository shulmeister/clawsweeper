## Summary

Current main implements the observable receive-only/auto-reply suppression need through the shipped `session.sendPolicy` mechanism. WhatsApp inbound messages still build normal WhatsApp context and run the agent, while `sendPolicy: "deny"` suppresses reply delivery paths; explicit outbound message actions remain on the separate message-action runner path. Close as implemented on main rather than keeping the older dedicated `autoReply` proposal open.
