## Summary

Current main already supports the requested outcome: individual tools can be excluded from the effective tool set and prompt via `tools.allow`/`tools.deny`, with additional per-provider/model restriction via `tools.byProvider`.

## Evidence

- **Config surface documents per-tool restriction:** The gateway config reference documents `tools.allow` / `tools.deny` for global per-tool policy and `tools.byProvider` for provider/model-specific restriction, including examples disabling `browser` and `canvas`.
  - file: [docs/gateway/configuration-reference.md:2234](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/docs/gateway/configuration-reference.md#L2234)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Policy pipeline filters tool inventory:** `applyToolPolicyPipeline()` iterates configured policy steps and filters the tool list through allow/deny policy before tools are exposed further.
  - file: [src/agents/tool-policy-pipeline.ts:92](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/tool-policy-pipeline.ts#L92)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)

  - file: [src/agents/pi-tools.ts:671](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/pi-tools.ts#L671)

## Close Comment

Closing this as implemented. Codex review against current `main` and the shipped `v2026.4.22` tag found equivalent support already in place via `tools.allow` / `tools.deny` and `tools.byProvider` (`docs/gateway/configuration-reference.md:2234`). The tool inventory is filtered before prompt assembly in `src/agents/pi-tools.ts:671`, and the embedded runner builds prompt/report output from that filtered `effectiveTools` set (`src/agents/pi-embedded-runner/run/attempt.ts:893`, `src/agents/pi-embedded-runner/run/attempt.ts:940`). There is also regression coverage showing `tools: { deny: ["browser"] }` removes `browser` from the effective tool list (`src/agents/pi-tools-agent-config.test.ts:536`). So the requested outcome is already available, even though the shipped config uses policy allow/deny keys rather than `tools.entries.<name>.enabled`.
