## Summary

Keep open. Current main clearly documents and tests Matrix DM isolation knobs (`session.dmScope` plus `channels.matrix.dm.sessionScope`), so the expectation of an automatic separate Matrix session under default shared-DM routing is not enough to close this by itself. But the reported core symptom, that Matrix inbound traffic reached routing/session metadata and still did not show up in WebChat, remains plausible and was not disproved; the maintainer asked for a fresh isolated-DM repro plus logs, and that follow-up never arrived. The issue is also only 8 days old, so `stale_insufficient_info` does not apply.
