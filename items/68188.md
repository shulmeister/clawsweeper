## Summary

Keep open. Current main already supports the maintainer-suggested Matrix DM isolation path, but that feature shipped before the reported 2026.4.15 version and does not disprove the distinct symptom: Matrix inbound metadata was recorded while no WebChat-visible transcript/session update appeared. The issue is recent, has a maintainer request for a fresh isolated-DM repro plus logs, and lacks the follow-up evidence needed for a conservative close.

## Best Possible Solution

Keep the issue open. Maintainers should ask for or wait on the requested fresh reproduction with isolated DM config, gateway logs, and session JSONL. If reproduced, add a Matrix monitor/dispatch regression around the path where recordInboundSession succeeds but dispatchReplyFromConfig or transcript/session discovery fails to make the inbound turn visible in WebChat, with clearer logging for queued, dropped, and appended states.
