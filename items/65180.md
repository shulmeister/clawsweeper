## Summary

Keep this PR open. Current main has not absorbed the two central changes proposed here: local `model run` still calls `agentCommand` without an explicit per-run session id/key, and session transcript fallback still uses the agent default transcript path instead of the profile-local sessions directory derived from the store path. The PR is a draft implementation candidate for a concrete, still-plausible behavior fix rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open as a focused draft implementation candidate. The best path is to review or refine the patch so local one-shot `model run` either generates an explicit per-run session id/key or uses an equivalent generic stateless session mode, while transcript fallback prefers the profile-local `sessionsDir` derived from `storePath`; keep the focused regression tests for both behaviors and decide separately whether the canary baseline docs should land.
