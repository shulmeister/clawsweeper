## Summary

Current main already addresses the observable WhatsApp contact-message bug shown in #68165: the screenshot is a WhatsApp pairing challenge, and main now suppresses historical catch-up pairing replies, skips outbound/fromMe DMs, and has setup/tests that keep personal WhatsApp accounts on owner allowlist/self-chat mode. Telegram setup has also been tightened to numeric sender IDs only, with docs and tests covering that setup path.

## Best Possible Solution

Close this combined report against the current implementation. Keep the shipped WhatsApp pairing safeguards and Telegram numeric-ID setup path; if Telegram setup still fails on current releases, it should be reported as a focused Telegram issue with status output, logs, token/setup mode, and the exact error.
