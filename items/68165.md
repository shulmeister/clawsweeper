## Summary

Keep open. Current `main` contains safeguards that should keep Telegram allowlist setup separate from WhatsApp owner-number setup, and WhatsApp pairing replies are implemented as replies to the inbound chat, but I did not find a shipped fix or other high-confidence evidence that the specific regression in #68165 has been resolved. The onboarding docs still contain confusing QuickStart copy that says Telegram and WhatsApp both prompt for a phone number, which makes the report plausible rather than obsolete.
