## Summary

Current main already fixes the reported Windows update restart hang. The update helper no longer generates the v2026.4.24 `netstat | findstr` batch loop; it now writes a hidden `.cmd` wrapper that runs embedded PowerShell, uses `Get-NetTCPConnection` first, falls back to direct `netstat.exe` parsing, force-stops stale listener PIDs, then reruns the Scheduled Task. Current Scheduled Task scripts also execute the gateway command directly, and normal stop/restart paths explicitly terminate listener-backed gateway processes instead of relying only on `schtasks /End`.

## Best Possible Solution

Close this issue as implemented on current main. Keep the current PowerShell-backed Windows update restart helper and listener cleanup behavior; if a user can reproduce the hang on a post-fix build, they should open a fresh issue with the generated restart script, `gateway.cmd`, gateway-restart log, process list, OpenClaw version, and Windows build.
