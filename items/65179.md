## Summary

Keep open. Current main has not implemented the proposed allowlist bypass; it still falls back when cron `payload.model` is not allowed. The PR remains a plausible implementation candidate, but it conflicts with current docs/tests and has unresolved review feedback, so it should get maintainer review rather than an automated close.

## Best Possible Solution

Keep this PR open for maintainer product review. If explicit cron `payload.model` should bypass `agents.defaults.models`, implement that on current main with exact provider/model catalog validation, add tests for allowlisted, catalog-only, and wrong-provider collision cases, update the cron docs, and drop or split the unrelated SSRF change. If the allowlist fallback is intentional, close the PR after a maintainer states that contract.
