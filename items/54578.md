## Summary

Keep this issue open. Current main still lacks a top-level `openclaw tree` command or an `agents tree` / `sessions tree` subcommand, and the existing CLI/UI/API surfaces remain flat listings or current-session subagent helpers rather than a hierarchical active-agent tree.

## Best Possible Solution

Keep this open for maintainer product judgment. A narrow first implementation should add a core inspection command such as `openclaw sessions tree` or `openclaw agents tree`, build hierarchy from existing `spawnedBy`, `parentSessionKey`, `childSessions`, status, runtime, task, token, and cost metadata, and include focused tests for text and JSON output. Treat `--watch` and per-process resources as follow-on work unless maintainers want a larger first slice.
