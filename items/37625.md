## Summary

Keep #37625 open. Current main still does not provide the requested write-time Markdown code-fence lint/fix utility, config mode, per-write skip, or integration into the actual workspace Markdown write paths; the two related implementation PRs were closed unmerged.

## Best Possible Solution

Keep the issue open until maintainers either implement a central Markdown code-fence lint/fix utility with sh/ts/python/json inference, config mode, per-write skip, and focused tests across the canonical workspace write paths, or explicitly decide that OpenClaw should not mutate Markdown at write time.
