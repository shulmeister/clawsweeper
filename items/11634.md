## Summary

Current `main` does not implement xAI OAuth/device-code auth. The bundled xAI plugin still exposes only API-key auth, and the docs explicitly say xAI auth is API-key only today, so this enhancement request is not implemented and should stay open.

## Evidence

- **xAI manifest only declares API-key auth choice:** `providerAuthChoices` for the bundled xAI plugin contains a single entry with `method: "api-key"` and `choiceId: "xai-api-key"`; there is no OAuth or device-code entry.
  - file: [extensions/xai/openclaw.plugin.json:15](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/extensions/xai/openclaw.plugin.json#L15)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **xAI provider runtime only registers API-key auth:** The xAI provider's `auth` array contains only one method, `methodId: "api-key"`, wired to `XAI_API_KEY` / `--xai-api-key`.
  - file: [extensions/xai/index.ts:160](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/extensions/xai/index.ts#L160)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **CLI auth-choice coverage includes only `xai-api-key`:** The auth-choice options test expects xAI to appear only as `methodId: "api-key"` with `choiceId: "xai-api-key"`.
  - file: [src/commands/auth-choice-options.test.ts:185](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/commands/auth-choice-options.test.ts#L185)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Docs explicitly say xAI OAuth is unavailable:** The xAI provider docs state: auth is API-key only today and there is no xAI OAuth or device-code flow in OpenClaw yet.
  - file: [docs/providers/xai.md:405](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/docs/providers/xai.md#L405)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Other providers do declare OAuth/device-code choices when supported:** The OpenAI bundled plugin manifest includes explicit `openai-codex` OAuth and `openai-codex-device-code` auth choices, showing the expected implementation pattern that xAI lacks.
  - file: [extensions/openai/openclaw.plugin.json:12](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/extensions/openai/openclaw.plugin.json#L12)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
