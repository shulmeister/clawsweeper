## Summary

Keep #39608 open. Current main still has the same WhatsApp linked-device history-sync gap identified in the issue discussion: the plugin passes `syncFullHistory: false` to Baileys without an explicit `shouldSyncHistoryMessage`, while Baileys 7.0.0-rc.9 defaults the omitted predicate from `syncFullHistory`. Recent comments report the same DM-only/group-silent behavior, and the older #14069 report was closed stale rather than replaced by a shipped fix.

## Best Possible Solution

Keep this issue open until the WhatsApp plugin deliberately fixes or documents linked-device group inbound behavior. The likely code path is `extensions/whatsapp/src/session.ts`: either pass an explicit Baileys `shouldSyncHistoryMessage` predicate, expose a reviewed config/product choice, or document with maintainer approval that relinked WhatsApp group inbound is unsupported. Add a focused regression test in `extensions/whatsapp/src/session.test.ts` for the options passed to `makeWASocket`, and prefer a live linked-device group smoke before closing as fixed.
