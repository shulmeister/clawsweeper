## Summary

Keep #39608 open. Current main still creates the WhatsApp Baileys socket with `syncFullHistory: false` and no `shouldSyncHistoryMessage` override, while Baileys 7.0.0-rc.9 defaults that predicate to `() => !!syncFullHistory`. The issue discussion has fresh confirmations on 2026.3.28 and 2026.4.20 after #14069 was closed stale, so the reported linked-device group inbound loss is recent, plausible, and not implemented on main.

## Best Possible Solution

Keep this issue open until the WhatsApp plugin either supplies an explicit Baileys history-sync predicate, exposes a deliberate config/product choice for the linked-device sync behavior, or a maintainer documents that group inbound on relinked WhatsApp Web devices is unsupported. The likely implementation path is in `extensions/whatsapp/src/session.ts`, with a focused regression test in `extensions/whatsapp/src/session.test.ts` asserting the socket options passed to `makeWASocket`.
