## Summary

Keep #39608 open. Current main still creates the WhatsApp Baileys socket with `syncFullHistory: false` and no `shouldSyncHistoryMessage`, while upstream Baileys 7.0.0-rc.9 derives the omitted predicate from `syncFullHistory`. The recent issue comments provide concrete linked-device repro evidence, and OpenClaw docs/tests still present WhatsApp group inbound as supported once Baileys emits the upsert.

## Best Possible Solution

Keep the issue open until the WhatsApp plugin intentionally fixes or documents linked-device group inbound behavior. The likely maintainer path is to review the Baileys history-sync contract, update `extensions/whatsapp/src/session.ts` to pass an explicit `shouldSyncHistoryMessage` predicate or product-backed config option, add a regression assertion in `extensions/whatsapp/src/session.test.ts` for the `makeWASocket` options, and validate with a live linked-device group smoke. If maintainers decide this mode is unsupported, document that clearly in the WhatsApp channel docs instead.
