## Summary

Keep open. Current main has adjacent liveness features, including typing keepalive, preview streaming, tool-progress preview edits, and lifecycle status reactions, but it does not implement the requested configurable fixed-interval chat progress heartbeat with min duration, per-channel cadence/enablement, templates, and explicit rate-limit fallback behavior. Related #52640 tracks a different persistent editable task-status surface and explicitly treats fixed-interval heartbeats as adjacent rather than the same request.

## Evidence

- **Issue scope:** The provided issue body asks for a built-in fixed-interval progress heartbeat during one long-running turn, with lifecycle messages started/running every N seconds/done/failed plus config keys such as progressHeartbeat.enabled, intervalMs, minTaskDurationMs, channels, and templates.
- **No requested config surface:** Exact search for the requested config names found no implementation or docs entries for progressHeartbeat or minTaskDurationMs in current main.
  - command: `rg -n "progressHeartbeat|minTaskDurationMs" src extensions docs package.json`
  - sha: [28497515fe1e](https://github.com/openclaw/openclaw/commit/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071)
- **Current streaming is preview/edit based:** Streaming docs show current channel controls are channels.<channel>.streaming modes; Telegram and Discord map progress to partial preview edits, while Slack has a progress preview mode. This is not a fixed interval message heartbeat contract.
  - file: [docs/concepts/streaming.md:121](https://github.com/openclaw/openclaw/blob/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071/docs/concepts/streaming.md#L121)
  - sha: [28497515fe1e](https://github.com/openclaw/openclaw/commit/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071)
- **Tool progress is event driven:** Tool-progress preview updates are documented as short status lines that appear in the same preview message while tools are running and follow active preview streaming mode; the documented controls only enable/disable toolProgress, not interval/min-duration/template cadence.
  - file: [docs/concepts/streaming.md:184](https://github.com/openclaw/openclaw/blob/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071/docs/concepts/streaming.md#L184)
  - sha: [28497515fe1e](https://github.com/openclaw/openclaw/commit/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071)
- **Telegram implementation lacks timer heartbeat:** Telegram pushPreviewToolProgress appends distinct tool/progress lines to the preview stream when invoked, and callbacks feed it from onToolStart/onItemEvent/onPlanUpdate. There is no periodic timer that emits running updates every configured interval.
  - file: [extensions/telegram/src/bot-message-dispatch.ts:405](https://github.com/openclaw/openclaw/blob/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071/extensions/telegram/src/bot-message-dispatch.ts#L405)
  - sha: [28497515fe1e](https://github.com/openclaw/openclaw/commit/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071)
- **Typing cadence is not progress messages:** Typing docs and implementation expose typingIntervalSeconds as a refresh cadence for typing indicators, backed by a keepalive interval that calls the typing start callback. This is separate from user-visible started/running/done/failed progress messages.
  - file: [docs/concepts/typing-indicators.md:8](https://github.com/openclaw/openclaw/blob/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071/docs/concepts/typing-indicators.md#L8)
  - sha: [28497515fe1e](https://github.com/openclaw/openclaw/commit/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071)
