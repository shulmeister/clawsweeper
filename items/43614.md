## Summary

Keep PR #43614 open. Current main still emits the misleading `no bootstrap files` wording for the normal status overview when `bootstrapPendingCount` is zero, and the linked issue's root cause still matches current code. The PR needs to be ported because the status summary builder now lives in `src/commands/status.command-sections.ts`, not the original `src/commands/status.command.ts` hunk from the PR.

## Best Possible Solution

Keep the PR open and ask for a small refresh: change `buildStatusAgentsValue` in `src/commands/status.command-sections.ts` to use wording such as `no workspaces bootstrapping`, update the existing status output assertion, and add the PR's regression that confirms mature workspaces no longer print `no bootstrap files`. That would directly resolve linked issue #43527 without changing bootstrap behavior.
