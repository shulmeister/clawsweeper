## Summary

Current main still does not implement the full reported behavior. `logging.file` appears to be read by the logger path, but there is still no root `--log-file` option and no `OPENCLAW_LOG_FILE` handling. Related PRs #61824 and #66110 are open and directly relevant, but #61295 is the original tracked issue those PRs reference rather than a clearly superseded duplicate.
