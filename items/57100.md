## Summary

Keep open. Current `main` at `95b7a85f0607784d8e33b4ce7e42617174196490` still applies provider `defaultModel` as `agentModelOverride` during `openclaw agents add`, and the current tests still codify that behavior. That directly contradicts the PR's target fix and the later comment claiming the core fix already landed upstream.

## Evidence

- **Agent add still bakes returned override into the new agent config:** `openclaw agents add` calls `applyAuthChoice(... setDefaultModel: false ...)` and then immediately writes `authResult.agentModelOverride` into the agent entry. If provider auth still returns an override, the agent will not inherit `agents.defaults.model`.
