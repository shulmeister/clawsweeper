## Summary

Current main already implements the Windows UI runner fix requested by PR #67346. Instead of dropping the resolved pnpm path to a basename, main now quotes Windows shell-launcher paths before spawning, preserving the exact path found by `which()` and avoiding the `C:\Program` truncation. The behavior is covered by a Windows-focused UI script test and documented in the active changelog with credit to this PR author.

## Best Possible Solution

Close PR #67346 as already implemented on main by the maintainer fix in `4c7a94aac41e4537fdfcd95285f2326f4fdef851`; keep the shipped approach because it fixes the Windows space-in-path crash while preserving the exact resolved pnpm launcher path and adding regression coverage.
