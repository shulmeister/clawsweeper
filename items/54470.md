## Summary

Current main still has the Windows command-resolution mismatch reported here. Gmail setup verifies dependencies using PATH/PATHEXT-aware checks, but then executes bare `gcloud`, `gog`, and `tailscale` commands through paths that do not resolve `gcloud.cmd` or `gog.cmd` first. This is a concrete core CLI bug with a clear reproduction and no evidence of a shipped fix on current main.

## Best Possible Solution

Keep this issue open. Fix Gmail webhook setup by resolving `gcloud`, `gog`, and `tailscale` through a Windows-aware executable resolver before spawning, while preserving the existing no-shell hardening and `.cmd` wrapper behavior. Add win32 regression tests proving a PATH-only `gcloud.cmd` is executed successfully by Gmail setup, and cover the `gog` serve/start paths as well.
