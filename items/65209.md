## Summary

Current main only partially addresses the report. The docs now explain that `openclaw mcp list/show/set/unset` manage OpenClaw-owned `mcp.servers`, but the CLI output and help still use generic wording and do not warn that mcporter config is excluded. The related PR #69726 proposed the right focused fix, but it is closed and unmerged, so this issue remains actionable.

## Best Possible Solution

Keep this issue open and land a focused CLI/docs/test change like the closed PR attempted: label non-JSON `openclaw mcp list` output as OpenClaw-managed, add a short note that mcporter `config/mcporter.json` servers are not included, tighten `mcp list` help text, and add a regression test in `src/cli/mcp-cli.test.ts` while leaving JSON output unchanged.
