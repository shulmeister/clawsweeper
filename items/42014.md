## Summary

Keep open. Current main still does not expose the requested session sync status in memory manager status or `openclaw memory status`; the PR addresses a real remaining observability gap, though it now needs to be ported from the old `src/memory`/`src/cli` paths into the current `extensions/memory-core` plugin layout. The related reports in the provided context cover different session/status features and do not supersede this PR.

## Best Possible Solution

Keep the PR open, ask for a port onto current `extensions/memory-core` code, and land the feature only after adding session sync status to the current manager status payload, rendering it in `extensions/memory-core/src/cli.runtime.ts`, and covering it in focused memory manager and CLI tests. If a maintainer prefers a different status contract, capture that in review before closing or replacing this PR.
