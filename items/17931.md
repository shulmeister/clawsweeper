## Summary

Keep open. Current main still does not implement automatic read-only skill-directory mounts for sandbox containers: non-rw sandbox modes still copy eligible skills into the sandbox workspace, and rw mode still exposes workspace skills through the writable workspace mount. The closed #22669 bind-ordering work enables a manual custom-bind workaround, but it does not provide the requested automatic per-skill immutability.

## Best Possible Solution

Keep #17931 open until OpenClaw implements or explicitly rejects automatic read-only per-skill mounts for sandboxed runs. A robust fix should resolve eligible skill entries deterministically, add read-only Docker bind overlays for each source skill directory, ensure those overlays shadow writable workspace paths in rw mode, stop relying on writable copied skill directories for Docker non-rw modes, update config hashing/recreate behavior, extend fs-bridge mount awareness, and update tests/docs. The #22669 custom-bind ordering fix is useful but should be treated only as a manual workaround, not the product fix.
