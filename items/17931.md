## Summary

Keep open. Current main still does not implement automatic read-only per-skill bind mounts. Non-rw sandbox modes still stage skills by copying them into the sandbox workspace, and rw mode still exposes workspace skills through the writable workspace mount. The related #22669 bind-ordering issue is addressed by appending custom binds after workspace mounts, but that only enables a manual workaround; it does not provide the requested automatic skill immutability.

## Best Possible Solution

Keep #17931 open until OpenClaw either implements automatic read-only per-skill mounts for sandboxed Docker runs or maintainers explicitly choose a different security model. A robust fix likely belongs in the sandbox setup path: resolve the eligible skill entries, add deterministic read-only bind mounts for their source directories, ensure those mounts shadow writable workspace paths in rw mode, update config hashing/recreate behavior, and replace the copy-based non-rw expectations with tests/docs for the new mount contract.
