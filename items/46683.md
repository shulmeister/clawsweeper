## Summary

Keep open. Current main does recognize Slack reply button/select action IDs now, but the remaining user problem is not fully fixed: block-action system events are still routed with only channel/user scope, while normal Slack thread messages use thread-specific session keys. That means clicks from threaded interactive replies can still miss the originating thread session described in #46683.
