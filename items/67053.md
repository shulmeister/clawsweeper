## Summary

Keep open. Current main mitigates stale TUI streaming with a 30s silence watchdog, but it still does not meet the issue's expected behavior of going idle promptly after the last visible content token. The TUI remains tied to terminal chat/lifecycle events, and gateway lifecycle finalization can still wait for async flush/finalization work.

## Best Possible Solution

Keep #67053 open as the canonical product/UX tracker. The best fix is to split visible content completion from run lifecycle completion: either emit a gateway/TUI content-complete or finalizing phase, or have the TUI transition from `streaming` to a non-streaming finalizing state after the assistant delta stream ends while preserving correct run ownership. Add regression coverage for delayed lifecycle final events and the named worker agent path noted in the discussion.
