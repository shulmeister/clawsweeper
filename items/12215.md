## Summary

Current `main` already has a partial core answer via `openclaw models status --probe`, including live probe status, latency, and configurable probe timeout/concurrency, but it does not implement the full request for per-model health checks across all configured models or any dashboard-style surface. That is not strong enough evidence to close this feature request as implemented or to move it to ClawHub-only scope.
