## Summary

Keep open. Current main still lacks the requested activity-based sub-agent stall recovery: config and `sessions_spawn` expose only wall-clock `runTimeoutSeconds`, run records do not track last tool activity or nudge state, and the registry/sweeper paths handle gateway wait completion, orphan/lost-context cleanup, and archive TTLs rather than nudge-then-kill recovery. PR #43017 is useful prior art but the provided GitHub context shows it was closed unmerged; #39127/#39141 are broader open activity/watchdog proposals, not shipped substitutes.

## Best Possible Solution

Keep #39305 open until maintainers either implement activity-based sub-agent recovery or explicitly choose a different product direction. A likely implementation should add strict config and `sessions_spawn` override fields for nudge and kill thresholds, track deterministic child activity such as tool calls or run progress, inject an auditable nudge into the child session, terminate through the existing sub-agent kill/completion flow, and cover the behavior with focused runtime/config tests plus sub-agent docs. PR #43017 can be used as prior art; #39127/#39141 may provide infrastructure but do not currently replace this issue.
