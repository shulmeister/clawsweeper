## Summary

Keep #39305 open. Current main still supports wall-clock subagent run timeouts and manual subagent list/kill/steer controls, but it does not implement activity-based last-tool-call tracking, a nudge stage, automatic stall kill escalation, config fields, or sessions_spawn overrides. The related implementation PR #43017 was closed unmerged, and the broader #39127/#39141 activity/watchdog issues do not currently supersede this concrete recovery behavior.

## Best Possible Solution

Keep #39305 open until maintainers either implement activity-based subagent stall recovery or explicitly consolidate it into a canonical session activity/watchdog design. A good core implementation would add strict config and sessions_spawn override fields, track child tool/run activity, inject auditable nudges into child sessions, terminate through the existing subagent kill/cancellation path after the kill threshold, document the behavior, and cover config/runtime/docs with focused tests.
