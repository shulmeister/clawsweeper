## Summary

Keep open. Current main still lacks the requested activity-based sub-agent stall recovery: sub-agent config and `sessions_spawn` expose only wall-clock `runTimeoutSeconds`, the runtime records no last-tool-call/nudge/kill state, and the related PR #43017 was closed unmerged rather than shipped. The broader #39127/#39141 activity/watchdog issues do not fully supersede this concrete nudge-to-kill sub-agent behavior.

## Best Possible Solution

Keep the issue open until OpenClaw implements an activity-based sub-agent stall recovery path or maintainers explicitly choose a different product direction. The implementation likely belongs in the sub-agent runtime/registry: record last meaningful child activity such as tool calls or deterministic run progress, add strict config and `sessions_spawn` overrides for nudge and kill thresholds, deliver the nudge through an auditable system/session path, terminate through the existing sub-agent kill/completion flow, and update tests plus docs. PR #43017 can be treated as prior art, while #39127/#39141 remain broader activity/watchdog infrastructure rather than a complete substitute.
