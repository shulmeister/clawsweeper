## Summary

Keep open. Current main still does not implement activity-based sub-agent stall recovery. The implementation and docs expose wall-clock `runTimeoutSeconds`, stale/orphan cleanup, and manual `/subagents kill`, but there is no last-tool-call activity tracking, nudge timer, post-nudge kill timer, or per-spawn stall override. PR #43017 is useful prior art, but the provided GitHub context shows it was closed unmerged; #39127 and #39141 are broader open activity/watchdog proposals rather than a shipped substitute for this concrete sub-agent nudge-to-kill behavior.

## Best Possible Solution

Keep #39305 open until maintainers either implement activity-based sub-agent recovery or explicitly choose a different product direction. The likely path is to add strict config and `sessions_spawn` override fields for nudge and kill thresholds, track deterministic child activity such as tool calls or run progress, inject an auditable nudge into the child session, terminate through the existing sub-agent kill/completion flow, and cover the behavior with focused runtime/config tests plus sub-agent docs. PR #43017 can be treated as prior art; #39127/#39141 may provide infrastructure but do not currently replace this issue.
