## Summary

Keep #14206 open. Current main has Feishu mitigations such as streaming cards, text chunk limits, inbound sequential handling, and API-call reduction flags, but it still does not provide the requested configurable outbound minimum interval/rate limiter between Feishu messages. Non-streaming chunked replies and Feishu comment replies still send chunks sequentially without a pacing hook.

## Best Possible Solution

Keep the issue open and implement this in the Feishu plugin surface: add a documented top-level and per-account outbound pacing setting, apply it to reply-dispatcher chunk sends, comment-thread chunk sends, and message-tool/outbound text/media sequences where multiple Feishu API calls can be emitted, and add focused fake-timer tests plus config-schema/docs coverage. The broader #13615 external API rate-limiting item does not supersede this Feishu-specific channel behavior.
