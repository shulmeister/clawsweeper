## Summary

Keep #14206 open. Current main still lacks a Feishu outbound minimum-interval/rate-limit setting, and the main Feishu reply/comment/message-tool send paths still call Feishu send APIs directly rather than through a shared per-account pacing layer. The related #13615 remains broader external API throttling work and does not supersede this Feishu-specific channel request.

## Best Possible Solution

Keep the issue open. Implement Feishu outbound pacing inside the bundled Feishu plugin: add documented top-level and per-account minimum-interval/rate-limit config, expose it through schema/manifest/generated metadata/docs, route reply chunks, comment replies, message-tool sends, pairing/bot notifications, media/card fallbacks, and streaming card create/update/close calls through one per-account pacing helper, and add fake-timer tests. If #13615 later provides a generic limiter contract, use it as the implementation mechanism, but close #14206 only once Feishu exposes and applies the setting.
