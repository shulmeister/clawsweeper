## Summary

Keep #14206 open. Current main still has Feishu mitigations such as streaming cards, fixed streaming update throttling, typing API backoff handling, inbound sequential handling, and quota-reduction flags, but it does not implement the requested user-configurable outbound minimum interval/rate limiter between Feishu message sends.

## Best Possible Solution

Keep the issue open and implement the pacing inside the Feishu plugin surface: add a documented top-level and per-account outbound minimum-interval/rate-limit setting, apply it consistently to reply-dispatcher chunk sends, comment-thread chunk sends, message-tool text/card sends, and text-plus-media sequences, then add focused fake-timer tests plus config-schema/manifest/docs coverage. The broader open #13615 external API rate-limiting request should not supersede this Feishu-specific channel behavior.
