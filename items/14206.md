## Summary

Keep #14206 open. Current main still lacks a Feishu outbound minimum-interval/rate-limit setting, and the Feishu reply, comment, message action, media, and outbound adapter paths still call Feishu send APIs directly without shared per-account pacing. The related #13615 remains broader external API throttling work and does not supersede this Feishu-specific channel request.

## Best Possible Solution

Keep the issue open. Implement this in the Feishu plugin surface: add documented top-level and per-account outbound minimum-interval/rate-limit config, expose it through schema/manifest/generated metadata/docs, route reply chunks, comment replies, message-tool text/card/media sends, text-plus-media sends, fallback sends, and streaming final sends through one per-account pacing helper, and cover it with fake-timer tests. Treat #13615 as broader rate-limiting work, not as the canonical replacement for this Feishu-specific behavior.
