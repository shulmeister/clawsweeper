## Summary

Keep open. Current `main` does not carry the PR’s fix: `extensions/discord/package.json` still does not declare `@snazzah/davey`, the global updater still retries installs with `--omit=optional`, and bundled-runtime reinstall logic only discovers direct plugin deps from extension manifests. The report is not strong enough for `cannot_reproduce` either, because `pnpm-lock.yaml` shows `@snazzah/davey` only as a transitive dependency of `@discordjs/voice`, so installs that omit optional deps can still plausibly lose it.
