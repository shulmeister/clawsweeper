## Summary

Keep open. Current `main` still lacks the reported capability-path fix for `models.providers.*.models`, and this PR has an unresolved P1 correctness issue in its fallback logic, so there is not strong evidence to close it conservatively.

## Evidence

- **Current main still does exact provider/model matching:** `resolveGatewayModelSupportsImages` on current `main` still uses `entry.id === params.model` and `entry.provider === params.provider`, so the case-insensitive lookup proposed by this PR is not implemented on `main`.
