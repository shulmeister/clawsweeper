## Summary

Keep this PR open. Current main still uses an exact, case-sensitive gateway image-support lookup, and the related #65211 was closed unmerged rather than shipping a replacement. The PR targets a real remaining bug, but it needs revision because its model-only fallback can weaken the existing fail-closed provider-mismatch behavior.

## Best Possible Solution

Keep this PR open for maintainer or contributor follow-up. The best patch is a narrower version: use `findModelInCatalog` for provider-scoped lookups, run model-only fallback only when no provider was supplied, preserve the explicit-provider fail-closed behavior, and add regression tests for mixed-case provider/model matches plus provider-mismatch cases.
