## Summary

Keep open. Current `main` still exposes ACP-only `streamTo`/`resumeSessionId` in the unified `sessions_spawn` schema, while the executor explicitly rejects those fields for `runtime="subagent"`. The docs still describe `streamTo` only on the ACP path, and the current test suite codifies the rejection behavior rather than a fix, so the report still matches `main` at `78a431ed224b6f5b9d58672346551a5b411ad200`.

## Evidence

- **Unified schema still advertises ACP-only field:** `SessionsSpawnToolSchema` on current `main` includes `streamTo` unconditionally in the shared `sessions_spawn` parameters object, so subagent callers still see that field in the tool schema.
  - file: [src/agents/tools/sessions-spawn-tool.ts:100](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/tools/sessions-spawn-tool.ts#L100)
  - command: `nl -ba src/agents/tools/sessions-spawn-tool.ts | sed -n '80,270p'`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Executor still rejects subagent calls that include streamTo:** The runtime path parses `streamTo`, then returns `status: "error"` when `streamTo` is present and `runtime !== "acp"`; the same guard exists for `resumeSessionId`. That is the failure mode described in the issue, and it is still present on `main`.
