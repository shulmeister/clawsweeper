## Summary

Keep open. Current main has documented and tested safe paths for `/model` and `session_status(model=...)`, including pending live-switch handling and bounded retry protection, but #31396 still lacks the key reproduction details needed to close as implemented or cannot-reproduce. The report could also be about provider/model connection hangs, and provider-level connect timeout work is not shipped on main.

## Best Possible Solution

Keep #31396 open until the reporter or maintainer supplies a concrete reproduction: exact command or prompt, provider/model, channel or UI, current/default model, whether a run was active, logs around the freeze, and OpenClaw version/config. If it reproduces on `/model` or `session_status(model=...)`, add a targeted regression around the implicated live-switch path. If it is provider connection latency/hang, fold the remaining work into a scoped provider/request timeout design, likely informed by PR #41936 but without global dispatcher side effects.
