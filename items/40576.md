## Summary

Current main already contains shipped fixes for the reported OpenAI/custom-endpoint regression: OpenClaw now classifies provider endpoints before choosing the native OpenAI websocket path, keeps non-public/custom `baseUrl` values on HTTP, and has regression coverage for local/custom base URLs. The changelog records those fixes in v2026.4.22.

## Close Comment

Closing from a Codex review because this regression is already implemented on current `main` and shipped in `v2026.4.22`. The current code now classifies provider endpoints before choosing the native OpenAI websocket path (`src/agents/provider-attribution.ts:259`, `src/agents/openai-ws-stream.ts:373`) and explicitly keeps local/custom `baseUrl` endpoints on HTTP in the embedded runner (`src/agents/pi-embedded-runner/run/attempt.thread-helpers.ts:41`). The release changelog for `v2026.4.22` records the shipped fix for this custom-endpoint/OpenAI transport path (`CHANGELOG.md:185`, `CHANGELOG.md:239`). If this still reproduces on `v2026.4.22` or newer, please open a fresh issue with the exact provider config/env path in use and the current logs.
