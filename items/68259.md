## Summary

Keep open. Current `main` still documents and wires `openclaw sessions` as a listing command plus `cleanup` only; there is no CLI implementation of `sessions_send`, even though cross-session messaging exists as an agent tool. The request is coherent, recent, and in core CLI scope, so none of the conservative auto-close reasons apply.
