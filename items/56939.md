## Summary

Keep open. Current `main` still performs ambient Ollama discovery against `http://127.0.0.1:11434` even without an explicit Ollama provider config, and the bundled tests explicitly lock that behavior in. The docs still describe implicit discovery as gated by `OLLAMA_API_KEY` or auth profile, so the reported surprise/docs mismatch remains plausible rather than obsolete.

## Evidence

In normal runtime it calls `buildProvider(...)` even when there is no real Ollama key and no meaningful explicit config, then only returns `null` after probing if zero models were found.
