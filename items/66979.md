## Summary

Keep #66979 open. The report is recent and sparse, but it describes a concrete, salvageable host filesystem/skill visibility problem with sandbox mode off. Current main documents and implements sandbox-off host execution and multi-root skill loading, but I did not find evidence that this exact intermittent behavior is already fixed, duplicated, superseded, or outside the repo. The provided title-search related item #55068 is about a hardcoded media staging size limit and does not cover this host-file/skill recognition report.

## Best Possible Solution

Keep the issue open and ask for a focused repro. Maintainers should request the affected OpenClaw version, sanitized agent/tool/sandbox/skill config, whether the session is main or channel/group, the exact file and skill locations, output from `openclaw sandbox explain` and `openclaw skills check/status`, and logs around a turn where visibility flips. A fix likely belongs in sandbox runtime resolution, filesystem policy diagnostics, skill snapshot refresh, or docs depending on that evidence.
