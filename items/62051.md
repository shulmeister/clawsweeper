## Summary

Current main no longer matches the v2026.4.5 regression: cron CLI paths skip plugin bootstrap, gateway startup loads a scoped plugin set, and runtime plugin resolution reuses the active gateway registry before doing any fresh load.

## Evidence

- **Cron CLI skips plugin preload:** The cron command policy is covered as `bypassConfigGuard: true` and `loadPlugins: "never"`, so the CLI bootstrap returns before `ensureCliPluginRegistryLoaded`; `cron run` then calls the Gateway RPC method rather than loading plugins locally.

  - file: [src/gateway/server-plugins.ts:477](https://github.com/openclaw/openclaw/blob/20c3177281d292d3db208b1e875e4b51a14fec94/src/gateway/server-plugins.ts#L477)
