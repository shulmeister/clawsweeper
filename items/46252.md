## Summary

Keep open. Current main appears to fix the broad `usage.cost` daily aggregate by counting `.jsonl.reset.*` and `.jsonl.deleted.*`, but the issue's latest `sessions.usage` undercount for an active `.jsonl` plus same-session `.jsonl.reset.*` still matches current code: per-session usage resolves and scans only one transcript file, preferring the active file when present.
