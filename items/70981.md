## Summary

Keep open. Current main still builds gateway session rows with the model-only `resolveThinkingDefaultForModel()` path, while the UI uses `GatewaySessionRow.thinkingDefault` for the `Default (...)` label. PR #70981 remains a focused fix for the reported mismatch; #71779 may be a broader follow-up, but the provided discussion does not clearly establish it as the maintainer-chosen canonical replacement for #70981.

## Best Possible Solution

Keep #70981 open unless maintainers explicitly choose #71779 as the canonical broader fix. The best product path is to land a gateway-side session/default resolver that matches the documented runtime precedence, with regression coverage for configured global defaults and, ideally, per-agent defaults/current supported option lists.
