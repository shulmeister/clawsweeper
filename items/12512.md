## Summary

Keep #12512 open. Current main still routes eligible skills into model-facing prompt selection and tells the model to read/follow selected SKILL.md files, while current skill metadata/config only provides visibility, load, install, and allowlist controls. I found plugin prompt-hook controls, but they do not isolate or permission SKILL.md content itself.

## Best Possible Solution

Keep #12512 open as core security hardening. The best path is to define and document a SKILL.md instruction-isolation model, add enforceable per-skill capability metadata for role/memory/tool influence, integrate it with prompt assembly/tool policy/sandbox behavior, and add malicious-skill regression tests. ClawHub scanners or plugins can supplement this, but OpenClaw core must provide the boundary first.
