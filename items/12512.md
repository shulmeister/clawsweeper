## Summary

Keep #12512 open. Current main still exposes eligible skills through prompt-facing skill catalogs and tells the model to read and follow selected SKILL.md files. I found visibility/allowlist/size/env/install/sandbox controls, but not the requested instruction isolation, role locking, per-skill role/memory/tool capability metadata, or behavioral monitoring boundary.

## Best Possible Solution

Keep #12512 open as core security hardening. Define and document a SKILL.md instruction precedence/isolation model, add enforceable per-skill capability metadata for role, memory, and tool influence, apply enforcement through prompt assembly/read flow/tool policy/memory/sandbox behavior, and add malicious-skill regression tests. Treat marketplace moderation and scanners as supplemental controls, not substitutes for the core boundary.
