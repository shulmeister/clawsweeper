## Summary

Keep open. Current main narrows startup exposure to an escaped available-skills catalog, but it still instructs the agent to read and follow a selected SKILL.md on demand. I found no core instruction-isolation layer, role locking, behavioral monitoring, or skill capability permission model like the issue requests. The existing April 26 Codex review comment reached the same conclusion, and the newer main SHA does not change this skill/prompt surface.

## Best Possible Solution

Keep #12512 open as core security hardening. A real solution should add a documented skill instruction-isolation model around SKILL.md consumption, define enforceable skill permissions/capabilities if skills can affect role, memory, or tools, and add tests/docs showing how the boundary behaves for malicious skill text. If maintainers choose a narrower product scope, capture that explicitly rather than closing this as already implemented.
