## Summary

Current main still contains the exact Control UI `activeMinutes: 0` session refresh callsites targeted by PR #72225, and `loadSessions` drops non-positive values before calling `sessions.list`. The PR is not obsolete or implemented on main.

## Best Possible Solution

Keep PR #72225 open for maintainer review or merge an equivalent scoped fix: replace the three remaining Control UI `activeMinutes: 0` chat-session refresh callsites with `CHAT_SESSIONS_ACTIVE_MINUTES`, and keep regression coverage for initial chat refresh and session-switch refresh so the shared active-session window does not drift again.
