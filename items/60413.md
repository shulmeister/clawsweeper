## Summary

Do not close PR #60413. It is not safely implemented on current main: the current Ollama stream function still forwards only `options?.signal` to the request and has no `timeoutSeconds`/default timeout plumbing. The PR also has useful but unresolved review context around stream-duration timeout behavior and config schema wiring, so this remains a potentially useful PR rather than a conservative cleanup close.
