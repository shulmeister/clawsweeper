## Summary

Keep #28965 open. Current main has partial model-config normalization and one-session override clearing through `sessions.patch`, plus some `models status` source reporting, but it still lacks the requested core operational surfaces: `openclaw models reconcile` and bulk `openclaw sessions model-reset` with scope, dry-run, match, and JSON options. The related ClawSweeper context also points to broader still-open model/config hierarchy work rather than a shipped or canonical replacement for this specific proposal.

## Best Possible Solution

Keep this issue open until OpenClaw either implements first-class core CLI support for model config reconciliation and bulk session override reset, or a maintainer explicitly consolidates it into a canonical model/config hierarchy issue. The likely implementation belongs in the core CLI and config/session-store code, reusing existing model-ref normalization, allowlist upsert, fallback dedupe, and session patch/reset semantics with dry-run and JSON reporting.
