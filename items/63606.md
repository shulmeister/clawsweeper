## Summary

This PR is superseded by the merged Bedrock prompt-caching implementation in #69953. Current main no longer relies on a configured display-name fallback for opaque application inference profile ARNs; it resolves the profile through Bedrock `GetInferenceProfile`, caches successful eligibility results, injects Bedrock cache points only when all routed targets support them, and preserves explicit cache-retention settings for opaque app-profile ARNs.

## Best Possible Solution

Close this PR as superseded by #69953 and keep the shipped GetInferenceProfile-based implementation. It is more reliable than a display-name fallback because it validates the actual routed Bedrock foundation model before injecting cache points; any remaining cache-ttl-only improvement should be tracked separately.
