## Summary

Keep #39585 open. Current main has Telegram-native exec approval delivery and generic elevated full mode that can skip exec approvals, but it still lacks the requested Telegram-scoped trusted-owner/group/tool-class approval bypass with audit labeling and destructive-operation fallback behavior.

## Best Possible Solution

Keep the issue open. A complete implementation should live in the Telegram plugin/config surface with an explicit opt-in policy requiring resolved owner sender ID plus allowlisted group/topic, configured tool classes, audit reason such as trusted-owner-bypass, destructive-operation fallback behavior, schema/docs updates, and focused Telegram approval-path tests.
