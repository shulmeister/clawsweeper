## Summary

Keep #39585 open. Current main has Telegram-native exec approval delivery and broader session/host-level ways to skip approvals, but it still does not implement the requested Telegram-scoped trusted owner x allowlisted group x tool-class approval bypass with audit labeling.

## Best Possible Solution

Keep the issue open. A complete solution would add an explicit Telegram-scoped policy, probably under the Telegram plugin config, that requires an allowlisted owner sender and allowlisted group/topic, scopes bypass to configured tool classes, records a trusted-owner-bypass audit marker, preserves destructive-op fallback behavior, and adds focused schema/docs/tests around the exec approval path.
