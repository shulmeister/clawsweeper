## Summary

Keep #11172 open. Current main still does not inject OPENCLAW_AGENT_ID, CLAWDBOT_AGENT_ID, or OPENCLAW_SESSION_KEY into exec environments for spawned subagent sessions. Agent/session identity exists as tool metadata and approval context, while the child process environment only gets the existing OPENCLAW_SHELL marker plus inherited/request/sandbox env.

## Best Possible Solution

Keep the issue open until OpenClaw either implements or explicitly rejects a documented core exec identity contract. The likely fix belongs in shared exec environment construction: derive identity from the resolved session context, inject OPENCLAW_AGENT_ID and OPENCLAW_SESSION_KEY consistently, decide whether CLAWDBOT_AGENT_ID compatibility is supported, define whether caller env can override identity, and add focused tests/docs for all exec hosts and PTY/sandbox/background paths.
