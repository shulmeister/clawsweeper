## Summary

Keep #11172 open. Current main still does not inject `OPENCLAW_AGENT_ID`, `CLAWDBOT_AGENT_ID`, or `OPENCLAW_SESSION_KEY` into exec child-process environments for subagent sessions. Agent identity exists in session/tool context, but the exec environment builders only merge inherited, request, sandbox, and runtime marker env values. The related implementation PR #11497 was closed unmerged, and open PR #53262 is about requiring `agentId` in the `sessions_spawn` schema, not exposing identity to spawned commands.

## Best Possible Solution

Keep the issue open until OpenClaw ships or explicitly rejects a documented core exec identity contract. The likely implementation belongs in shared exec environment construction after session/agent context is resolved: inject `OPENCLAW_AGENT_ID` and `OPENCLAW_SESSION_KEY`, decide whether `CLAWDBOT_AGENT_ID` compatibility is supported, define caller override behavior, and add focused tests plus docs for subagent exec modes.
