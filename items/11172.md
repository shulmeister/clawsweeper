## Summary

Keep open. Current main still does not inject OPENCLAW_AGENT_ID, CLAWDBOT_AGENT_ID, or OPENCLAW_SESSION_KEY into subagent exec environments. The issue is concrete, the author is not protected, and it is not superseded: the related implementation PR #11497 was closed unmerged, while PR #53262 only concerns requiring agentId in the sessions_spawn schema.

## Best Possible Solution

Keep tracking #11172 until OpenClaw either implements an explicit, documented exec environment identity contract for subagent sessions or a maintainer makes a product/security decision against exposing that identity. A likely implementation would inject OPENCLAW_AGENT_ID and OPENCLAW_SESSION_KEY, decide whether legacy CLAWDBOT_AGENT_ID is supported, define override/spoofing semantics for params.env and sandbox env, and cover gateway/node/sandbox exec paths with focused tests and docs.
