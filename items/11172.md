## Summary

Keep open. Current main still does not inject OPENCLAW_AGENT_ID, CLAWDBOT_AGENT_ID, or OPENCLAW_SESSION_KEY into exec/subagent command environments, and the related PR that attempted this was closed unmerged. The request is concrete and still actionable, so it should not be closed as stale or duplicate.

## Evidence

- **No identity env vars on main:** `rg -n "OPENCLAW_AGENT_ID|CLAWDBOT_AGENT_ID|OPENCLAW_SESSION_KEY" src packages extensions docs test ui .github` returned no matches on current main.
  - command: `rg -n "OPENCLAW_AGENT_ID|CLAWDBOT_AGENT_ID|OPENCLAW_SESSION_KEY" src packages extensions docs test ui .github`
  - sha: [1dfa52d0711d](https://github.com/openclaw/openclaw/commit/1dfa52d0711dfc86e1c04538fe46e3d43961766f)
- **Exec env construction lacks identity injection:** `createExecTool` builds the execution environment from sanitized `process.env`, requested env overrides, or sandbox env, with no insertion of agent/session identity variables before execution.
  - file: [src/agents/bash-tools.exec.ts:1536](https://github.com/openclaw/openclaw/blob/1dfa52d0711dfc86e1c04538fe46e3d43961766f/src/agents/bash-tools.exec.ts#L1536)

  - file: [src/agents/bash-tools.exec-runtime.ts:563](https://github.com/openclaw/openclaw/blob/1dfa52d0711dfc86e1c04538fe46e3d43961766f/src/agents/bash-tools.exec-runtime.ts#L563)

  - file: [src/agents/bash-tools.shared.ts:30](https://github.com/openclaw/openclaw/blob/1dfa52d0711dfc86e1c04538fe46e3d43961766f/src/agents/bash-tools.shared.ts#L30)
