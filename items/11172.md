## Summary

Keep open. Current main still does not inject OPENCLAW_AGENT_ID, CLAWDBOT_AGENT_ID, or OPENCLAW_SESSION_KEY into subagent exec environments, and the related implementation PR was closed unmerged. The request is concrete, not protected by labels/author association, and not superseded by the separate agentId schema PR.

## Best Possible Solution

Keep #11172 open until OpenClaw implements or explicitly rejects a documented exec environment identity contract for subagent sessions. A likely fix belongs in the exec environment construction shared by gateway, node, and sandbox paths: inject OPENCLAW_AGENT_ID and OPENCLAW_SESSION_KEY, decide whether legacy CLAWDBOT_AGENT_ID is supported, define whether tool-call env can override these values, and add focused tests plus docs at docs.openclaw.ai/help/environment.
