## Summary

Keep #69413 open. Current main still gates Signal group messages through hydrated text plus configured mention regexes; it does not compare `dataMessage.mentions[]` UUID/phone metadata against the bot account. The PR is a scoped Signal plugin bug fix, and its follow-up commit addresses the review concern about validating mention spans before accepting native metadata.

## Best Possible Solution

Keep the PR open and review it as a scoped Signal plugin bug fix. The best path is to merge or revise the native metadata detection in `extensions/signal/src/monitor/mentions.ts` and `extensions/signal/src/monitor/event-handler.ts`, keep the span-validation guard from the follow-up commit, keep/add UUID and phone mention-gating tests, and decide whether the no-pattern `canDetectMention` case belongs in this PR or a follow-up.
