## Summary

Keep open. Current main has strong MCP tool registration/execution support and a later visible-text sanitizer for XML/downgraded tool-call leaks, but I did not find a current fix for this issue's specific LM Studio/plain-text bracket block format (`[mempalace_...] ... [END_TOOL_REQUEST]`) or for converting that text back into an executable tool round-trip. The OpenAI-compatible stream path still treats ordinary `delta.content` as assistant text and only creates tool calls from structured `delta.tool_calls`.

## Best Possible Solution

Keep the issue open and reproduce on current main with either the reported LM Studio/MemPalace setup or a deterministic OpenAI-compatible stream fixture that emits the reported bracket block. A good fix should either safely convert recognized plain-text tool blocks into normal tool-call events before final text delivery, or explicitly document/guard unsupported local-model text tool formats and prevent hangs. The MemPalace `predicate` argument error should be checked separately against the MCP tool schema so schema/tool-argument repair is not conflated with visible raw tool-block leakage.
