## Summary

The issue should be closed as incoherent. Its examples mix intentional CLI/stdout-stderr UX, test-only fixtures, and outdated code snippets, while current main already documents console capture and structured subsystem logging. There is not a single concrete, current, user-facing bug here to track as filed.

## Evidence

- **ACP client is explicitly a debug CLI surface:** The public docs describe `openclaw acp client` as a built-in debug client used to sanity-check the bridge interactively, so terminal `console.log/error` output in that command is expected UI, not stray production debugging.
  - file: [docs/cli/acp.md:100](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/docs/cli/acp.md#L100)

  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **Reported ACP client lines are interactive terminal output:** `src/acp/client.ts` prints tool updates, available commands, session banners, prompt results, and exit status to stdout/stderr for the interactive ACP client. These are user-facing CLI messages, not hidden debug traces.
  - file: [src/acp/client.ts:94](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/acp/client.ts#L94)
  - command: `nl -ba src/acp/client.ts | sed -n '80,250p'`
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **Reported ACP helper lines are permission prompts/status on stderr:** `src/acp/client-helpers.ts` uses stderr for permission denials, timeouts, and approval results around interactive prompts. That is expected command-line UX for a permission gate, not evidence of an unstructured logging bug.
  - file: [src/acp/client-helpers.ts:63](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/acp/client-helpers.ts#L63)
  - command: `nl -ba src/acp/client-helpers.ts | sed -n '55,120p'`
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **Reported ACP server lines are standard CLI help/warnings/errors:** `src/acp/server.ts` prints help text to stdout and prints secret-exposure warnings / fatal startup errors to stderr. The issue body mischaracterizes these as debug statements, and even its cited `console.error` usage example does not match current main.
  - file: [src/acp/server.ts:215](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/acp/server.ts#L215)

That undercuts the issue's premise that there is no proper logging/control in place.
  - file: [docs/gateway/logging.md:43](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/docs/gateway/logging.md#L43)

The issue claims there is a `console.error("[agent] runtime error: ...")` line, but current main instead has a single `stopReason` console line in a lifecycle fallback path, so the report does not accurately describe the code it asks to change.
