## Summary

Keep open. Current main has the stale-unended subagent run liveness/reconciliation work mentioned in the issue comments, and #71756 now tracks broader lifecycle finalization, but the specific sessions.json cleanup/finalization requested here is not implemented yet. The restore path prunes stale run-registry rows; it does not update or remove the direct session-store row that external monitors read.
