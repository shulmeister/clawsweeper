## Summary

Current `main` already implements both parts of this request: sandbox backends are pluggable (`docker`, `ssh`, and plugin-registered backends such as `openshell`), and exec defaults/routing can be overridden per agent via `agents.list[].tools.exec.*`. The feature is documented, covered by tests, and shipped by `v2026.3.31`.

## Evidence

- **Pluggable sandbox backend registry exists in core:** Core keeps a backend registry and registers built-in `docker` and `ssh` backends; unknown backend ids fail with a message telling users to load the providing plugin or use Docker.
  - file: [src/agents/sandbox/backend.ts:26](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/agents/sandbox/backend.ts#L26)
- **Plugin SDK exposes sandbox backend registration:** The public plugin SDK exports `registerSandboxBackend`, `getSandboxBackendFactory`, and related sandbox backend types, so sandbox runtimes are not Docker-only anymore.
  - file: [src/plugin-sdk/sandbox.ts:24](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/plugin-sdk/sandbox.ts#L24)
- **Bundled OpenShell plugin registers its own backend:** The OpenShell plugin calls `registerSandboxBackend("openshell", ...)`, proving non-core backends can be plugged into the sandbox layer.
  - file: [extensions/openshell/index.ts:14](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/extensions/openshell/index.ts#L14)
- **Docs and config describe multiple sandbox backends and per-agent sandbox overrides:** Docs list `docker`, `ssh`, and `openshell` under `agents.defaults.sandbox.backend`, and the agent config type allows per-agent `sandbox.backend` overrides.
  - file: [docs/gateway/sandboxing.md:57](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/docs/gateway/sandboxing.md#L57)
- **Per-agent exec routing/defaults are resolved over global defaults:** Exec defaults resolve in order `session -> agent -> global`, including `host`, `security`, `ask`, and `node`, so agents no longer share one global exec target.
  - file: [src/agents/exec-defaults.ts:33](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/agents/exec-defaults.ts#L33)
- **Per-agent exec behavior is covered by tests:** Tests explicitly verify that agent-specific `tools.exec` defaults override global defaults and affect exec host behavior.
  - file: [src/agents/pi-tools-agent-config.exec.test.ts:113](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/agents/pi-tools-agent-config.exec.test.ts#L113)
- **Release notes show both pieces shipped:** `v2026.3.22` added pluggable sandbox backends plus SSH/OpenShell support, and `v2026.3.31` added per-agent `tools.exec` defaults.
  - file: [CHANGELOG.md:2164](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/CHANGELOG.md#L2164)

## Risks / Open Questions

- A later comment on the issue asks for broader per-agent filesystem/process isolation via gateway+isolated nodes. That is not the same as the original request and is not established by the evidence above.

## Close Comment

Closing as implemented after a Codex review of current `main`. The original request is already in the codebase: sandbox backends are pluggable (`src/agents/sandbox/backend.ts:26`, `src/plugin-sdk/sandbox.ts:24`, `extensions/openshell/index.ts:14`), docs list multiple backends under `agents.defaults.sandbox.backend` (`docs/gateway/sandboxing.md:57`), and per-agent exec routing/defaults now resolve from `agents.list[].tools.exec.*` ahead of global defaults (`src/agents/exec-defaults.ts:33`). There is direct regression coverage for agent-specific exec defaults in `src/agents/pi-tools-agent-config.exec.test.ts:113`. The changelog shows this shipped in `v2026.3.22` for pluggable sandbox backends and in `v2026.3.31` for per-agent `tools.exec` defaults (`CHANGELOG.md:2198`, `CHANGELOG.md:1668`). If the broader gateway-plus-isolated-node filesystem-isolation idea from later comments is still wanted, that should be tracked separately from this now-implemented feature request.
Codex evidence: Reviewed against main [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091). Fix evidence: release [v2026.3.31](https://github.com/openclaw/openclaw/releases/tag/v2026.3.31).
