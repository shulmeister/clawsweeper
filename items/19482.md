## Summary

Keep #19482 open. Current main still recreates `HEARTBEAT.md` unconditionally during workspace bootstrap, while runtime/docs/tests preserve a meaningful missing-vs-empty heartbeat distinction. The PR remains a narrow fix for a real current-main behavior gap; its negative regression test should be tightened before landing.

## Best Possible Solution

Keep the PR open and either land it after tightening the negative regression fixture, or apply the same fix directly on main: only seed `HEARTBEAT.md` for brand-new workspaces while preserving missing-file heartbeat semantics for existing workspaces.
