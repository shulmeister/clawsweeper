## Summary

Keep PR #38780 open. Current main has restart/session-delivery continuation and related compaction/context plumbing, but it still does not implement the PR's central agent-elected continuation surface: continue_work, continue_delegate, request_compaction, fallback tokens, continuation config, delegate runtime/store, or gateway continuation trigger/drain fields. The PR is active, large, and still needs explicit maintainer product/code review rather than cleanup closure.

## Best Possible Solution

Keep PR #38780 open for explicit maintainer review. If maintainers want this capability in core, the best path is a smaller opt-in continuation runtime that reuses the shipped restart/session-delivery, heartbeat, subagent, compaction, and TaskFlow substrate; updates config, protocol, docs, and clients coherently; and resolves the branch review comments before merge. If maintainers decide the product direction is wrong or belongs outside core, close it deliberately as a maintainer decision rather than conservative cleanup.
