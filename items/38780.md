## Summary

Keep PR #38780 open. Current main still does not implement the PR's central self-elected continuation surface: no continuation tools, fallback tokens, continuation config block, delegate store/runtime, context-pressure threshold contract, or gateway continuation trigger fields. Main has restart/session-delivery continuation and related context/compaction plumbing, but that is narrower than the proposed agent-controlled continue_work/continue_delegate/request_compaction runtime. The PR is active, size XL, and has unresolved review surfaces, so cleanup closure would be premature.

## Best Possible Solution

Keep PR #38780 open for explicit maintainer review. If maintainers want this in core, the best path is a smaller, well-tested opt-in continuation runtime that reuses the shipped restart/session-delivery, heartbeat, subagent, compaction, and TaskFlow substrate, updates config/protocol/docs/client surfaces coherently, and resolves the existing review comments before merge. If maintainers decide the product direction is wrong, close it deliberately as a maintainer decision rather than via conservative cleanup.
