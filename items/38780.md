## Summary

Keep PR #38780 open. Current main still lacks the proposed self-elected continuation runtime: no continuation tools, response-token fallback parser, `agents.defaults.continuation` config contract, gateway continuation trigger/drain fields, delegate-return modes, or context-pressure warning surface. Main has restart-continuation/session-delivery plumbing, but that is narrower recovery work and does not supersede this PR. The PR discussion is active and contains unresolved correctness review comments, so this needs explicit maintainer product/code review rather than conservative cleanup.

## Best Possible Solution

Keep PR #38780 open for explicit maintainer review. If maintainers want this in core, the best path is a smaller, well-tested continuation runtime that reuses the merged session-delivery, heartbeat, subagent, and TaskFlow substrate, keeps the feature disabled by default, updates config/protocol/docs/client surfaces coherently, and resolves the existing review comments before merge. If maintainers decide the product direction is wrong, close it deliberately as a maintainer decision rather than via conservative cleanup.
