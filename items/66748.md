## Summary

Current main still has the reported code path: subagent exec completion can enqueue an exec system event, call requestHeartbeatNow with the subagent session key, and heartbeat resolution intentionally falls back that key to the main session. The linked PR #66749 and broader #69492 are relevant, but neither is merged/shipped in this checkout, so this remains a real open bug rather than a conservative cleanup close.

## Best Possible Solution

Keep #66748 open until #66749 or an equivalent main-branch change stops exec-event heartbeat wakes for subagent session keys. The fix should cover both maybeNotifyOnExit and emitExecSystemEvent, keep enqueueSystemEvent behavior intact for internal consumption, and add regression coverage for subagent session keys; if #69492 lands first, close this only after confirming that the broader audience-classification path prevents the same main-session wake.
