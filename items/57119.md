## Summary

Keep open. Current `main` still treats `skills.entries.<skill>.enabled: false` as a skill-eligibility/prompt filter, not a filesystem or exec boundary. The code and docs still assume skill folders are trusted code, and host `read` remains unrestricted unless the separate optional `tools.fs.workspaceOnly` guard is enabled.
