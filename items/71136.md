## Summary

Keep this issue open. Current main still has strict-priority token resolution in several live paths, and the partial context-window guard does not provide the requested shared minimum resolver for contextTokens and maxTokens.

## Best Possible Solution

Implement a shared token-limit resolver that computes the minimum of valid agent defaults, per-agent overrides, provider model metadata, discovered model/cache metadata, and safe defaults for both contextTokens and maxTokens. Wire it through agent run setup, auto-reply, gateway session/status defaults, session persistence, and provider transport request construction, then add focused regression tests for high configured caps and lower model/discovery limits.
