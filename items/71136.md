## Summary

Keep #71136 open. Current main has partial protection for the embedded runner when `agents.defaults.contextTokens` is lower than model metadata, but it still does not implement the requested all-source minimum for `contextTokens` or `maxTokens`. Several active code paths preserve priority/override behavior: overrides can short-circuit lookup, model config can replace discovered metadata, and tests assert configured `contextWindow`/`maxTokens` winning rather than being clamped to discovered limits.

## Best Possible Solution

Keep the issue open and implement a shared conservative token-limit resolver. It should collect positive limits from agent default/per-agent config, provider model config, runtime/discovered model metadata, persisted session state where appropriate, and the default, then use the minimum for `contextTokens`. For `maxTokens`, clamp agent/model stream params and configured model metadata to the selected model's known output limit at the model-resolution or transport boundary. Add focused tests for high agent defaults over smaller model config, high model config over smaller discovery metadata, per-agent context caps, and oversized `maxTokens`.
