## Summary

Current main no longer matches the reported v2026.3.3 behavior: the hardcoded Anthropic default is gone, Ollama setup/discovery supports local-only flows without preselecting a cloud model, and the current UI config helpers write agent overrides under `agents.list` instead of a top-level `main` key.

## Evidence

- **Default model baseline is no longer Anthropic:** `DEFAULT_PROVIDER` and `DEFAULT_MODEL` now resolve to `openai` / `gpt-5.5`, so the reported `anthropic/claude-opus-4-6` startup default is obsolete on current main.
  - file: [src/agents/defaults.ts:3](https://github.com/openclaw/openclaw/blob/ac063568d354c14e0a0a43bcffb9725bb3fca651/src/agents/defaults.ts#L3)
  - sha: [ac063568d354](https://github.com/openclaw/openclaw/commit/ac063568d354c14e0a0a43bcffb9725bb3fca651)
- **Bundled Ollama setup does not preselect a cloud default:** Current Ollama provider auth setup returns a local provider config patch with `baseUrl`, `api: "ollama"`, and `models: []`, and explicitly expects `defaultModel` to be undefined.
  - file: [extensions/ollama/index.test.ts:125](https://github.com/openclaw/openclaw/blob/ac063568d354c14e0a0a43bcffb9725bb3fca651/extensions/ollama/index.test.ts#L125)
  - sha: [ac063568d354](https://github.com/openclaw/openclaw/commit/ac063568d354c14e0a0a43bcffb9725bb3fca651)
- **Local-only Ollama discovery is covered on current main:** The Ollama discovery tests assert that when local models are discovered without ambient Ollama credentials, OpenClaw auto-registers the provider with a synthetic local auth marker and discovered models.
  - file: [extensions/ollama/provider-discovery.test.ts:194](https://github.com/openclaw/openclaw/blob/ac063568d354c14e0a0a43bcffb9725bb3fca651/extensions/ollama/provider-discovery.test.ts#L194)
  - sha: [ac063568d354](https://github.com/openclaw/openclaw/commit/ac063568d354c14e0a0a43bcffb9725bb3fca651)
- **Current UI config helper writes agent entries under `agents.list`:** `ensureAgentConfigEntry()` finds/creates entries in `agents.list` and patches `['agents','list',nextIndex,'id']`; it does not write a top-level `main` key.
  - file: [ui/src/ui/controllers/config.ts:235](https://github.com/openclaw/openclaw/blob/ac063568d354c14e0a0a43bcffb9725bb3fca651/ui/src/ui/controllers/config.ts#L235)
  - sha: [ac063568d354](https://github.com/openclaw/openclaw/commit/ac063568d354c14e0a0a43bcffb9725bb3fca651)
- **UI tests assert the `agents.list` shape for `main`:** The controller tests expect editing the inherited `main` agent to produce `{ agents: { defaults: ..., list: [{ id: "main" }] } }`, which is incompatible with the old `Unrecognized key: "main"` report shape.
  - file: [ui/src/ui/controllers/config.test.ts:225](https://github.com/openclaw/openclaw/blob/ac063568d354c14e0a0a43bcffb9725bb3fca651/ui/src/ui/controllers/config.test.ts#L225)
  - sha: [ac063568d354](https://github.com/openclaw/openclaw/commit/ac063568d354c14e0a0a43bcffb9725bb3fca651)
- **Release notes mention the Ollama minimal-config validation improvement:** The changelog for `2026.4.20` includes an Ollama fix adding provider-policy defaults for `baseUrl` and `models` so local discovery can proceed instead of failing minimal-provider validation.
  - file: [CHANGELOG.md:370](https://github.com/openclaw/openclaw/blob/ac063568d354c14e0a0a43bcffb9725bb3fca651/CHANGELOG.md#L370)
