## Summary

Keep PR #63074 open. Current main still short-circuits Windows-style bind sources as non_absolute before blocked credential-path classification, and the requested sandbox security capability remains missing on main. The PR discussion is useful: Greptile confirms the intended root cause/fix, while Codex review comments flag portability work still needed for the added test.

## Evidence

- **Current classifier short-circuits non-POSIX sources first:** On current main, getBlockedBindReason returns { kind: "non_absolute" } for any parsed source that does not start with '/', before normalizeHostPath and getBlockedReasonForSourcePath can classify blocked credential paths.
