## Summary

Keep #45359 open. Current main has meaningful OAuth hardening and a configured-fallback path for Codex `server_error`, but I did not find source-backed proof that the reported Telegram/direct `openai-codex/gpt-5.4` failure after OAuth refresh is fixed without the user's manual model switch. The related #48153 stale-token issue is useful context, but it is an Anthropic stale auth-profile overwrite report and does not fully supersede this OpenAI Codex model-specific `server_error` report.
