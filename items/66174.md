## Summary

Keep PR #66174 open. Current main does not implement save-time canonicalization of persisted transcript paths, and the current resolver still preserves some structurally valid absolute `agents/<id>/sessions/*.jsonl` paths from outside the active store root. The PR is aimed at a real remaining session-store problem, but the submitted approach likely needs revision because review discussion points out it uses `resolveSessionFilePath`, which can preserve the exact cross-root paths the PR intends to rewrite.

## Best Possible Solution

Keep the PR open and revise it. The best fix is a save-time canonicalization step in `src/config/sessions/store.ts` that derives the persisted target from the active store directory and `sessionId` directly, covers both `sessionFile` and the legacy `transcriptPath` alias if that alias remains part of the contract, and adds regression coverage for workspace-style and structurally valid external `agents/<id>/sessions/*.jsonl` paths without breaking existing read-time cross-agent transcript candidate behavior.

## Risks / Open Questions

- The PR as submitted may not fix the cross-root structural-path case unless it derives the save target directly from the active `sessionsDir` and `sessionId` instead of reusing `resolveSessionFilePath`.
- A broad rewrite of every custom `sessionFile` could conflict with intentional cross-agent/custom transcript read behavior, so the final fix should preserve read-time compatibility while canonicalizing the persisted store-owned save path.
