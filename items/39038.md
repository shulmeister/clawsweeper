## Summary

Keep open. Current main removes the old `node host PATH` debug print and includes adjacent Windows Scheduled Task, PATH, status, and node-host retry fixes, but I found no direct source, test, release, or reporter-confirmation proof that the reported native Windows 11 24H2 `openclaw node run` hang/no Gateway connection is fixed.

## Best Possible Solution

Keep tracking this as a native Windows node-host startup bug until maintainers either verify a focused Windows 11 24H2 + Node 24 smoke where `openclaw node run` reaches the Gateway connection, ideally including a non-ASCII profile path, or explicitly document that this install/runtime combination is unsupported and should use a clean install without `--ignore-scripts` or WSL2.
