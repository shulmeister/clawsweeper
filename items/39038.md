Keep open. Current main removed the separate cosmetic `node host PATH` debug line, but #39038 reports a broader native Windows 11 24H2 node-host hang/no Gateway connection with `--ignore-scripts`, Node 24, and a non-ASCII Windows username. I found no direct source, test, changelog, release, related-item, or reporter-confirmation proof that this specific startup/connect hang is fixed on current main.

Keep tracking this as a native Windows node-host startup bug until maintainers either add/verify a focused Windows 11 24H2 Node 24 smoke for `openclaw node run` reaching Gateway connection, preferably including a non-ASCII profile path, or confirm the only supported resolution is reinstalling without `--ignore-scripts`/using WSL2 and document that boundary clearly.
