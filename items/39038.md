## Summary

Keep #39038 open. Current main has adjacent Windows PATH/service hardening and the old `node host PATH` debug print is gone, but I did not find source, test, docs, release, duplicate, or reporter-confirmation evidence that native Windows 11 24H2 foreground `openclaw node run` now reaches the Gateway connection path after the reported hang point. The workaround comments about reinstalling without `--ignore-scripts` and using WSL2 are useful, but they do not close the reported native Windows startup bug.

## Best Possible Solution

Keep tracking this as a native Windows node-host startup bug. The best maintainer path is a focused Windows 11 24H2 smoke or regression test proving `openclaw node run --host 127.0.0.1 --port 18789` reaches Gateway connection with Node 24 and a non-ASCII profile path, plus clearer docs if native Windows plus `--ignore-scripts` is considered unsupported.
