## Summary

Keep open. Current `main` does not implement configurable custom-term autocomplete or dictation-aware term suggestions in WebChat, and the request fits the repo's stated web frontend ergonomics scope rather than a clear ClawHub-only closure.

## Evidence

- **Chat input is a plain textarea:** The WebChat compose UI renders a bare `<textarea>` with keyboard/input/paste handlers only. There is no `list`, datalist hookup, or custom suggestion source attached to the input.
