## Summary

Current `main` still matches the reported failure mode. The media-capability resolver returns the already-active provider list for every non-memory capability as soon as that list is non-empty, so compat-only bundled media providers are still dropped instead of merged. Discord voice transcription rebuilds media provider state through that path on each segment, and the latest release `v2026.4.23` still carries the same logic.

## Evidence

- **Non-memory capability resolution still short-circuits on any active providers:** `resolvePluginCapabilityProviders` returns `activeProviders` immediately for every key except `memoryEmbeddingProviders`, so it never unions compat-loaded providers when the active registry is only partially populated.
