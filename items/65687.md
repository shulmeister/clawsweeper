## Summary

Current main still leaves the reported bug path open. The media provider resolver returns the active non-speech registry early when there is no explicit `plugins` config, so a partial active registry can still suppress the compat load that would add the configured Deepgram provider. Discord voice uses the shared `tools.media.audio` transcription path, so the issue is still actionable in core/plugin resolution.

## Best Possible Solution

Fix `src/plugins/capability-provider-runtime.ts` so configured media-understanding providers are merged from the compat registry when the active registry is partial. Add a regression test where active `mediaUnderstandingProviders` contains OpenAI/MiniMax but `cfg.tools.media.audio.models` requests Deepgram, and verify the returned provider set includes Deepgram without pulling unrelated bundled media providers.
