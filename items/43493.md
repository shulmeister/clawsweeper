## Summary

Keep #43493 open. Current main still does not implement an interactive metadata step for custom provider setup: the custom path delegates to the existing custom provider wizard, that wizard only collects endpoint/auth/model/alias details, and the config writer still fills generic metadata defaults unless an existing model already has explicit values. The related #48338 is a broader open issue, not a merged replacement or canonical PR that supersedes this one.

## Best Possible Solution

Keep the PR open for maintainer review or ask the author to rebase and align it with the current custom provider setup path. If maintainers prefer solving the broader custom-provider context UX through #48338 or another replacement PR, explicitly mark this PR superseded then; current main does not yet solve the central interactive metadata setup request.
