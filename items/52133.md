## Summary

Keep open. Current `main` does not implement the PR’s rolling transcript-pruning feature for ordinary persisted sessions: on-disk transcript rewriting is still limited to post-compaction truncation, the new `agents.defaults.compaction.transcriptPruning.*` config surface is absent, and the docs still describe session pruning as in-memory only. The PR also appears stale against current `main` because one of its touched files, `src/plugin-sdk/subpaths.test.ts`, no longer exists.

## Evidence

- **Compaction is still the only persisted transcript rewrite on main:** `truncateSessionAfterCompaction()` returns `reason: "no compaction entry found"` when a session has not compacted, so ordinary persisted sessions still have no rolling prune path on `main`. The only caller is the post-compaction hook in `compact.ts` guarded by `agents.defaults.compaction.truncateAfterCompaction`.

  - file: [src/agents/pi-embedded-runner/compact.ts:1139](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/agents/pi-embedded-runner/compact.ts#L1139)
