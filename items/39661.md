## Summary

Keep #39661 open. Current main still only supports Discord `requireMention` as a guild/channel boolean, while `users` and `roles` are sender allowlists. The runtime mention decision is not sender-aware, and the strict Discord config schema does not accept the requested per-user policy map.

## Best Possible Solution

Keep this open for bundled Discord plugin work. Add a typed sender-specific mention policy to the Discord guild/channel config contract, validate it in zod and generated metadata, define precedence across guild defaults, guild sender overrides, channel defaults, and channel sender overrides, pass sender identity into the mention-resolution path, and cover the behavior in Discord docs and regression tests with two users in the same channel requiring different trigger behavior.
