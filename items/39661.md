## Summary

Keep #39661 open. Current main still only supports Discord `requireMention` as one guild/channel boolean, while `users` and `roles` remain sender allowlists. The existing ClawSweeper keep-open comment remains accurate on current main at 7e13f3f5140168e31027b39a9855ade3fadf24d5, and the provided related items do not supersede this per-user mention-policy request.

## Best Possible Solution

Keep the issue open for bundled Discord plugin work. A good implementation would add a typed sender-specific mention policy to the Discord guild/channel config contract, validate it in the zod schema and generated config metadata, define precedence between guild default, guild sender override, channel default, and channel sender override, pass sender identity into the Discord mention resolver, and add docs plus regression tests showing two users in the same channel with different trigger behavior.
