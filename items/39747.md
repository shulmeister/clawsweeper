## Summary

Keep #39747 open. Current main still resolves an explicit heartbeat target through deliverable-channel normalization before any cfg-backed plugin channel bootstrap can register plugin-only channel ids. Telegram is now bundled, so the example is less representative, but the documented/configured contract still allows plugin heartbeat targets such as BlueBubbles/custom channel plugins, and the remaining load-order bug is not implemented on main.

## Best Possible Solution

Keep #39747 open, or replace it with a focused successor, until heartbeat target resolution bootstraps or reads plugin channel metadata before deliverable-channel recognition, retries normalization after registry initialization, and adds regression coverage for `agents.defaults.heartbeat.target` pointing at a plugin-only channel with an initially empty active registry.
