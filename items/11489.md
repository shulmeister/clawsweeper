## Summary

Keep #11489 open. Current main still has DM/node pairing only; group access is controlled through open/disabled/allowlist policies, with no groupPolicy="pairing", no owner approve/reject CLI for group IDs, no cross-channel group-join owner notification, and no pending read-only group-agent path. The existing maintainer/Codex comment also explicitly kept this issue open, and related #8495 and PR #42400 are narrower rather than canonical replacements.

## Best Possible Solution

Keep #11489 open for maintainer/product design. The likely best path is an additive core/channel contract: add groupPolicy="pairing" to config/schema/docs, persist pending group approvals keyed by channel/account/group ID, add CLI approve/reject semantics, emit owner-visible group-join or first-seen-unconfigured-group notifications, teach Telegram/WhatsApp and other group-capable channels to honor that state, and decide whether pending read-only observation belongs in the same change or a narrower follow-up coordinated with #8495 and PR #42400.
