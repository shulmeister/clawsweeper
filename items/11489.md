## Summary

Keep open. Current `main` still treats group access as explicit allowlist/open/disabled policy, not pairing: there is no `groupPolicy: "pairing"`, no pending-group reader agent/config, no group-specific pairing approve/reject CLI, and the docs explicitly say DM pairing does not grant group access.
