## Summary

Keep open. Current main still lacks the requested group pairing workflow: `groupPolicy` only supports `open`, `disabled`, and `allowlist`; Telegram group handling drops unauthorized group messages instead of creating owner approval notifications; and the `openclaw pairing` CLI is DM-code oriented with no group approve/reject target. The related #42400 context is useful but remains an open, partial PR for passive `neverReply` suppression, not owner-controlled group pairing.
