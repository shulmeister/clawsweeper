## Summary

Keep this PR open. Current main still lacks an iMessage default/named watcher dedupe, and the linked bug #65141 remains a real current-main issue. The PR is not ready to close as implemented or duplicate cleanup because it is an implementation candidate, but it should be revised before merge: the current diff wires startup dedupe through the plugin-wide enabled predicate and includes unrelated outbound target changes with unresolved review concerns.

## Best Possible Solution

Keep this PR open for maintainer review, but ask for a narrower revision: drop the unrelated outbound-target changes, implement iMessage-owned watcher dedupe on the gateway startup path or an explicit startup eligibility seam rather than plugin-wide `isEnabled`, and keep tests for #65141 cases covering duplicate default/named backends, configured versus placeholder named accounts, disabled named accounts, and distinct `cliPath`/`dbPath` backends.
