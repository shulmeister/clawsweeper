## Summary

Keep open. Current `main` has SecretRef-based workarounds for avoiding plaintext credentials, but it does not implement a native macOS Keychain backend for OpenClaw API keys/auth profiles. Provider/auth secrets still support inline plaintext fields and are still persisted to `auth-profiles.json`/`openclaw.json` unless the operator manually switches to SecretRefs.

- Closing as implemented would discard a real remaining gap: there is still no native macOS Keychain backend for OpenClaw-managed API keys/auth profiles.
- SecretRefs plus exec/file providers are a workaround, but they do not satisfy the issue's request for built-in Keychain support.
