## Summary

Keep #40747 open. Current main still exposes `feishu_doc(action: "insert")` with required `after_block_id` and no `index` parameter, so the PR’s central user problem remains. The PR is focused and useful, but should be updated to validate `index` as a non-negative integer and add targeted coverage before merge.

## Best Possible Solution

Keep the PR open or replace it with an equivalent maintainer patch in `extensions/feishu`: make `after_block_id` optional for `action: "insert"`, add `index: Type.Optional(Type.Integer({ minimum: 0, ... }))`, plumb `index` through `insertDoc` for root-child insertion when `after_block_id` is omitted, preserve existing after-block behavior, and add focused tests plus Feishu doc skill/docs updates for the new insert mode.
