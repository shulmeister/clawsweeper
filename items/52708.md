## Summary

Keep open. Current `main` has clear work toward both halves of the report: ACP dispatch now emits lifecycle end/error events, ACP background runs now flow through the detached-task completion system, and ACP oneshot sessions are explicitly closed after a turn completes. But the orphan-process part still depends on the external `acpx` package implementation, which is not present in this checkout, and ACP run-mode spawns still are not registered in the subagent registry. That is not strong enough evidence to conservatively close this issue as implemented or obsolete.

## Evidence

- **ACP oneshot turns now call runtime close on completion:** `AcpSessionManager.runTurn()` explicitly calls `runtime.close({ reason: "oneshot-complete" })` in the post-turn cleanup path for `meta.mode === "oneshot"`. That is relevant to the orphan-process report, but it only proves OpenClaw now asks the runtime to close; it does not prove the external ACPX runtime actually kills the full child process tree.
  - file: [src/acp/control-plane/manager.core.ts:951](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/acp/control-plane/manager.core.ts#L951)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **ACP background tasks now mark terminal completion/failure:** The ACP manager resolves parent-owned background task context and calls `completeTaskRunByRunId()` / `failTaskRunByRunId()` when the turn ends. This is evidence that parent completion delivery was addressed in current code.
  - file: [src/acp/control-plane/manager.core.ts:2111](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/acp/control-plane/manager.core.ts#L2111)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Detached-task runtime queues terminal updates back to the owner session:** `maybeDeliverTaskTerminalUpdate()` formats a background-task terminal message and either sends it to the requester origin or queues a session system event. That supports the missing-completion-event part of the report.
  - file: [src/tasks/task-registry.ts:1018](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/tasks/task-registry.ts#L1018)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **ACP dispatch now emits lifecycle end/error events:** After `acpManager.runTurn()` returns, `tryDispatchAcpReply()` emits `stream: "lifecycle"` with `phase: "end"` or `phase: "error"` for the current `runId`, and there are matching tests for both paths.
  - file: [src/auto-reply/reply/dispatch-acp.ts:456](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/auto-reply/reply/dispatch-acp.ts#L456)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **ACP run-mode spawns still are not registered in subagent registry:** A repo-wide search shows `registerSubagentRun(...)` is used for normal subagents and session tools, but not in `src/agents/acp-spawn.ts`. So the original complaint about ACP runs being absent from subagent-style tracking is not clearly obsolete from current main alone.
  - command: `rg -n "registerSubagentRun\(" src/agents/acp-spawn.ts src/agents`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Process cleanup is delegated to external ACPX runtime code:** The bundled extension depends on external package `acpx@0.5.3`, and `extensions/acpx/src/runtime.ts` forwards `close()` to `BaseAcpxRuntime.close(...)`. This checkout does not include ACPX internals, so the exact child-process teardown behavior for `mode:"run"` cannot be verified here with strong confidence.
  - file: [extensions/acpx/package.json:7](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/extensions/acpx/package.json#L7)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)

## Risks / Open Questions

- Closing as implemented would rely on inferred behavior from an external `acpx` dependency rather than repo-local proof that `claude-agent-acp`/`claude` descendants are terminated after normal ACP run completion.
- The completion-event half looks substantially improved on `main`, but ACP run-mode tracking still differs from subagent registry handling, so the report is not cleanly obsolete.
- There are still overlapping ACP lifecycle/orphan-process reports in the tracker, so closing this one would be aggressive for conservative maintainer cleanup.
