## Summary

Current main does not implement #70976. The public plugin hook catalog still lacks `model_failover` and `model_failure_terminal`, the failover paths still expose the relevant facts through structured logs and terminal error handling rather than plugin events, and the linked implementation PR #70990 is still described in the provided GitHub context as open/unmerged. This is a real core/plugin API feature request, not a ClawHub-only item or an implemented-on-main cleanup candidate.

## Best Possible Solution

Keep #70976 open until PR #70990 or an equivalent additive core/plugin SDK change lands. The implementation should add documented typed plugin hooks for model failover decisions and terminal before-reply all-models-failed outcomes, emit them from the existing failover/terminal seams, cover fail-open hook behavior and payload shape with focused tests, and update the public plugin hook docs and any plugin-surface metadata required by the repo rules.
