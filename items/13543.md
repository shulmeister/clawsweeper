## Summary

Keep #13543 open. Current main supports sandboxing at session/agent level and sandbox-time tool allow/deny filtering, but it does not implement the requested per-tool execution policy where selected tools such as web_fetch/browser are forced into a sandbox while other tools such as exec/message remain on host in the same session. The related Docker sandbox PR #29974 is useful context but explicitly only partially addressed or unblocked this issue, so it is not a duplicate or implemented-on-main close.
