## Summary

Keep #13543 open. Current main still lacks the requested per-tool sandbox execution routing. The implementation supports session/agent-level sandbox modes plus sandbox-time tool allow/deny filtering, and the browser tool can choose a sandbox target only when the session already has sandbox browser context. That does not satisfy selective routing such as always sandboxing `web_fetch` or `browser` while keeping `exec`, `message`, and `cron` on host in the same session. The existing 2026-04-26 Codex/steipete comment also explicitly kept this open and described PR #29974 as only partial/unblocking work.

## Best Possible Solution

Continue tracking this as a core sandbox/tool-routing enhancement. A good fix would add a documented, schema-backed per-tool execution policy in the sandbox/tool runtime path, with tests showing selected tools are forced into a sandbox while other tools remain on host in the same session, and with docs preserving the distinction from `tools.sandbox.tools` allow/deny filtering.
