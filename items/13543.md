## Summary

Keep #13543 open. Current main still supports sandboxing at the agent/session level plus sandbox-time tool allow/deny filtering, but it does not implement the requested per-tool execution routing where selected tools such as `web_fetch` or `browser` are forced into a sandbox while other tools stay on host in the same session. The existing maintainer/Codex discussion also explicitly kept this open, and PR #29974 only partially addressed/unblocked it.

## Best Possible Solution

Continue tracking this as a core sandbox/tool-routing enhancement unless maintainers explicitly choose a different product direction. A real fix would add a documented, schema-backed per-tool execution policy in the sandbox/tool runtime path, preserving the existing session-level modes and the current sandbox tool allow/deny policy as availability filtering rather than confusing it with backend selection.
