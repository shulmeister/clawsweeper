## Summary

Keep open. Current main does not implement the PR's proposed session-store recovery behavior: invalid JSON is still swallowed as an empty store, session-store writes do not create a .prev backup, and sessions.list / sessions.resolve still read the combined store directly without resilient transcript fallback. The PR also has unresolved review findings, so the right maintainer action is review or author revision, not automated closure.

## Best Possible Solution

Keep this PR open for maintainer review or author revision. The best path is to fill in the bug evidence, address the unresolved recovery-path review comments, decide whether corrupt session stores should throw or remain best-effort, and add focused tests for sessions.list / sessions.resolve recovery before landing. If maintainers decide the behavior is not desired, close it manually after that product judgment rather than as automated cleanup.
