## Summary

Keep open. Current main still routes WhatsApp inbound user text to the model through raw `BodyForAgent`, while the PR only wraps the legacy `formatInboundEnvelope`/`Body` text. The active reply path prefers `BodyForAgent` over `Body`, and the docs describe plaintext inbound envelopes as deprecated in favor of `BodyForAgent` plus structured context. The existing PR review comments also identify a real landing risk: wrapping `combinedBody` with random marker IDs would make WhatsApp combined echo keys nondeterministic unless the echo-key and model-body paths are separated.

## Best Possible Solution

Keep the item open, but require a rework: protect the actual WhatsApp model-facing `BodyForAgent` or structured `UntrustedContext` path, keep command/raw body handling deterministic, keep configured `messagePrefix` outside untrusted user content, and compute echo keys from an unwrapped stable body. Add focused regression coverage for direct WhatsApp messages, group/history context, prefixes/reply context, and duplicate echo suppression.
