## Summary

Keep #69943 open. The report describes a real current-main bug: the bundled session-memory hook still extracts user/assistant text and writes it into `memory/YYYY-MM-DD-slug.md` without sanitizing leaked chat-template tokens, raw tool-call XML, role markers, or `NO_REPLY`. The related PR #69961 directly tracks the fix, but it is open/unmerged in the provided GitHub context, so the issue remains the canonical bug tracker rather than something already implemented on main.

## Best Possible Solution

Keep #69943 open as the canonical bug until a focused fix lands. The best code path is to merge #69961 or equivalent: add a pure session-memory sanitizer, apply it at `src/hooks/bundled/session-memory/transcript.ts` before messages are assembled and again in `handler.ts` before writing, elide malformed/NO_REPLY-only turns visibly, and add unit plus handler wiring tests for leaked chat-template tokens, raw `<tool_call>` blocks, orphaned role markers, and preservation of legitimate prose.
