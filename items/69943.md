## Summary

Keep #69943 open. Current main still has the reported session-memory persistence gap: the hook reads assistant/user text from the session transcript without sanitizing leaked model control tokens or raw tool-call XML, then writes the resulting `sessionContent` verbatim under `## Conversation Summary`. The related PR #69961 is a directly relevant fix attempt and should remain the implementation path, but it is open/unmerged in the provided context, so the canonical issue should not be closed yet.

## Best Possible Solution

Keep #69943 as the canonical bug until #69961 or an equivalent patch lands. The best implementation is to sanitize assistant turns before session-memory persistence, preferably reusing or adapting the existing assistant-visible-text sanitizer for model control tokens/tool-call XML, adding explicit handling for `NO_REPLY`/artifact-dominated turns, applying defense-in-depth before writing `## Conversation Summary`, and adding regression tests for transcript extraction and handler output.
