## Summary

Keep this issue open. Current main still advertises `/status` and `/compact` in TUI slash autocomplete, but the local TUI command switch has no handlers for those names and still falls through to `sendMessage(raw)`, which uses `EmbeddedTuiBackend.sendChat()` in local mode.

## Best Possible Solution

Fix the TUI local embedded command boundary. Known shared commands advertised in TUI autocomplete, especially `/status` and `/compact`, should either execute through a shared local adapter or render an explicit local unsupported response in `opts.local` mode, and they should not fall through as ordinary model text. Add focused local-mode regression tests proving both commands do not reach `sendChat()` / `agentCommandFromIngress()` while preserving the existing gateway-backed shared command path.
