## Summary

Keep open. Current `main` still allows model pricing entries with omitted `cacheRead`/`cacheWrite`, `estimateUsageCost()` still multiplies those possibly-undefined fields directly, and the OTel plugin still only exports a cost metric when `costUsd` is truthy. The requested default-to-zero behavior is not implemented on current `main`, and the same `estimateUsageCost()` logic is present in the latest release tag `v2026.4.23`.

## Evidence
