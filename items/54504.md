## Summary

Keep this issue open. Current main still does not implement lightweight label-scoped subagent instruction files, workspace/user precedence, YAML frontmatter overrides, or an equivalent spawn-time plugin API for prompt/model/tool-policy mutation. The prior ClawSweeper comment reached the same conclusion, and the current checkout has not added this surface since then.

## Best Possible Solution

Keep this open for maintainer product/API design. The strongest path is either a narrow core resolver in the sessions_spawn/native subagent path for workspace/user-scoped role markdown files with documented precedence and frontmatter schema, or a new explicit spawn-time plugin hook that can mutate the pending child prompt, model/thinking, and tool policy before the child run is launched. The implementation would need tests and docs around precedence between explicit spawn params, per-agent config, global defaults, role-file frontmatter, and tool-policy ordering.
