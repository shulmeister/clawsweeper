## Summary

Current main already dispatches `gateway:startup` with the hook context `boot-md` needs (`cfg`, `deps`, `workspaceDir`) after channels start, and dedicated regression tests cover both the startup dispatch and `boot-md` execution path. The same implementation is present in release `v2026.4.22`.

## Risks / Open Questions

- This was a code-and-history review only; I did not perform a live macOS LaunchAgent restart reproduction in the read-only checkout.
- `boot-md` still returns early on malformed/non-startup events, so future regressions would depend on some other caller emitting an incomplete `gateway:startup` event.

## Close Comment

Closing from a Codex review because this appears to already be implemented on current `main` and shipped in `v2026.4.22`. The gateway startup path now emits `gateway:startup` after channels start with the context `boot-md` needs in `src/gateway/server-startup-post-attach.ts:268`, and there are regression tests for both the dispatch path (`src/gateway/server-startup-post-attach.test.ts:222`) and the `boot-md` execution path (`src/hooks/bundled/boot-md/handler.gateway-startup.integration.test.ts:35`). I also verified the same implementation is present in release commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5` (`v2026.4.22`). If you still hit this on `v2026.4.22` or newer, please open a fresh issue with current restart logs so maintainers can check for a new LaunchAgent-specific regression.
