## Risks / Open Questions

- The implemented event name differs from the issue text (`session:compact:after` instead of `session:compaction` / `session:post-compaction`).
- The current internal hook context does not appear to expose the exact `trigger` or `compactionCount` fields requested, so this closure relies on the existing capability being close enough rather than identical.
- The implementation commit I found is on `main` as of 2026-04-24, but it does not appear to be in the latest tagged release `v2026.4.22`.

## Close Comment

Closing from a Codex review because current `main` already has this capability in practice. OpenClaw now fires a post-compaction internal hook as `session:compact:after` from the shared compaction path (`src/agents/pi-embedded-runner/compaction-hooks.ts:249`, wired from `src/agents/pi-embedded-runner/compact.ts:976` / `:1122`), and the tests in `src/agents/pi-embedded-runner/compact.hooks.test.ts:308` verify that the after-compaction hook runs with compaction metrics. For the concrete re-bootstrap use case in this issue, current `main` also reinjects post-compaction AGENTS.md guidance via `readPostCompactionContext()` and queues that refresh before the next turn (`src/auto-reply/reply/post-compaction-context.ts:71`, `src/auto-reply/reply/agent-runner.ts:1572`). The naming/payload is not identical to the issue text, but the requested post-compaction hook + recovery path is present on `main`. I could not confirm a released tag containing the implementation yet, so `fixedRelease` is left unset.
