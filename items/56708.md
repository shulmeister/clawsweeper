## Summary

Current `main` does not implement reaction-triggered agent turns. Slack reactions still enqueue a system event only, system events are still defined as input for the next prompt/heartbeat, and the runtime wake API remains a separate explicit call with no reaction-specific config knob found.

## Evidence

- **Slack reaction handler still only enqueues a system event:** `registerSlackReactionEvents` formats the reaction text and calls `enqueueSystemEvent(...)`; there is no adjacent wake/turn trigger call.
  - file: [extensions/slack/src/monitor/events/reactions.ts:45](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/slack/src/monitor/events/reactions.ts#L45)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **System events are explicitly queued for the next prompt:** The system event queue comment says these events are prefixed to the next prompt, and `enqueueSystemEvent` only appends to the in-memory queue.
  - file: [src/infra/system-events.ts:1](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/infra/system-events.ts#L1)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Docs still describe system events as next-heartbeat input unless explicitly woken:** The CLI docs say `system event` enqueues a system event on the main session and that the next heartbeat injects it; `--mode now` is required to trigger the heartbeat immediately.
  - file: [docs/cli/system.md:34](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/cli/system.md#L34)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Wake capability exists, but as a separate explicit runtime API:** Plugin runtime docs show `enqueueSystemEvent(event)` and `requestHeartbeatNow()` as separate calls, which matches the current codepath: reaction handlers enqueue, but do not request an immediate heartbeat.
  - file: [docs/plugins/sdk-runtime.md:309](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/plugins/sdk-runtime.md#L309)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **No reaction-specific wake config found on current main:** Repository search did not find a current config or codepath such as `reactionTriggersTurn`, `reactionWakeMode`, or a reaction handler calling `requestHeartbeatNow`.
  - command: `rg -n "reactionTriggersTurn|reactionWakeMode|requestHeartbeatNow" src extensions docs`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
