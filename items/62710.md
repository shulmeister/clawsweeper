## Summary

Keep #62710 open. Current main partially clears auto-sourced auth overrides during explicit /new, /reset, and gateway reset via the shared reset-preserved-selection helper, but the PR’s resolver-level fix is still not on main: a new session with an existing auto auth override still rotates from the stale current profile with pickNextAvailable() instead of discarding it and picking the first available profile. The targeted resolver tests from the PR are also not present on main, so meaningful requested behavior remains missing.
