## Summary

Keep open. Current main has a replay-time mitigation that drops `provider:"openclaw", model:"delivery-mirror"` assistant rows before model replay, so the Anthropic rejection risk is reduced. But the requested write-side behavior is still missing: outbound message sends still create a delivery mirror, successful delivery still appends it as an assistant transcript row, there is no config toggle, and tests explicitly preserve non-duplicate consecutive assistant mirror writes.

## Best Possible Solution

Keep the issue open and fix the remaining core transcript behavior deliberately. Add a narrow outbound mirror policy around `appendAssistantMessageToSessionTranscript` or the mirror construction sites: either expose a documented `session.deliveryMirror` opt-out, or skip/degrade mirror writes that would create adjacent assistant-only transcript turns, especially for cron, isolated-session, and subagent-origin sends. Keep the replay filter, add regression coverage for message-tool outbound delivery into a session whose latest row is assistant, and document the transcript-only `delivery-mirror` contract for LCM/plugin consumers.
