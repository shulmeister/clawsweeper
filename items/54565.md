## Summary

Current main still does not implement per-group WhatsApp debounce. The config type/schema/metadata accept debounce only at channel/account/global inbound scopes, and the WhatsApp runtime still passes one scalar debounce value to the listener instead of resolving a window from each inbound group message.

## Best Possible Solution

Keep this issue open. Implement the WhatsApp case first by adding `debounceMs?: number` to the WhatsApp group config type, strict Zod schema, generated metadata, and public docs; resolve exact group and wildcard group values per inbound group message; pass that through the existing `createInboundDebouncer.resolveDebounceMs` seam; and add regression tests proving group-specific values override account/channel/global debounce only for the targeted group. Cross-channel support should be separate per-plugin design.

## Risks / Open Questions

- The request also mentions Telegram, Discord, and other channels; those group config models differ and should be handled as explicit per-plugin follow-up design, not assumed from WhatsApp semantics.
- The reported WhatsApp image/image/text pattern will not be fully solved by only adding a group-level scalar if media messages continue to bypass inbound debouncing.
