## Summary

Keep PR #72069 open. Current main still does not implement the PR's central behavior: a fallback retry after a failed claude-cli attempt only gets OpenClaw session-history awareness, while Claude CLI JSONL history is only imported for chat-history display surfaces. The PR is recent, open, contributor-authored but not protected, and the supplied discussion shows active review follow-up addressing source-provider gating and compact-summary pairing rather than obsolete work.

## Best Possible Solution

Keep #72069 open for maintainer review. The best path is to land or adapt a core fallback-runner fix that, only when the fallback chain started on `claude-cli` and retries on another runtime/provider, reads the bound Claude CLI session JSONL, extracts a bounded summary plus recent-turn prelude, prepends it to the retry prompt, and covers the behavior in `src/agents/command` and `src/gateway` tests.
