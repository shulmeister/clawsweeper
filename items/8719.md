## Summary

Keep #8719 open. Current main has related hardening, including workspace-only filesystem guards, apply_patch workspace containment, config mutation guards, web_fetch SSRF checks, and plugin install scanning, and the ClawMoat comment is useful external prior art. It still does not implement the requested native data-centric Security Profile v1.1: no data security levels, default protected OS folders, allowed/denied path policy, cross-tool data policy, network egress policy by data level, sensitive-action audit log, TTL elevation model, or native secure-by-default profile.

## Best Possible Solution

Keep #8719 open as a product/security umbrella until maintainers either define and implement a native Security Profile or split it into canonical narrower issues. Track filesystem PathGuard separately through #52621/#60981, acknowledge ClawMoat as complementary external prior art, and create or keep follow-ups for data classification, web_fetch/network egress policy, sensitive-action audit logging, TTL elevation, session isolation, config integrity, skill/plugin verification, and cross-tool policy enforcement.
