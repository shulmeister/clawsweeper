## Summary

Keep #8719 open. Current main has security hardening primitives, audit checks, sandbox defaults, exec approvals, plugin allowlists, and an optional `tools.fs.workspaceOnly` guard, but it still does not implement the requested native data-centric Security Profile v1.1. The issue discussion also has an explicit maintainer keep-open comment and useful external prior art via ClawMoat; neither the current code nor the open PathGuard PR supersedes the broader umbrella.

## Best Possible Solution

Keep #8719 open as the product/security umbrella until maintainers either define a native OpenClaw Security Profile or split it into canonical narrower issues. Track filesystem PathGuard work separately, use ClawMoat as complementary external prior art, and create/keep follow-ups for data classification/security levels, protected OS presets, cross-tool FS/exec/export/network policy, sensitive-action audit logs, approval/TTL elevation, session isolation, config integrity, and plugin/skill verification.
