## Summary

Keep #8719 open. Current main has several useful hardening primitives, but it still does not implement the requested native data-centric Security Profile v1.1: no data levels, OS protected-folder presets, cross-tool filesystem/exec/export/network policy, append-only sensitive-action audit log, approval TTL elevation, full session isolation model, or signature-style plugin verification. The issue discussion also contains an explicit maintainer keep-open review, and the ClawMoat comment is useful external prior art rather than proof that core OpenClaw has shipped the requested profile.

## Best Possible Solution

Keep #8719 open as the native product/security umbrella until maintainers either define OpenClaw Security Profile v1.1 or split it into canonical narrower issues. Use ClawMoat as external prior art, keep filesystem sandboxing tracked separately through the PathGuard/filesystem-access work, and create focused follow-ups for data levels, protected OS presets, cross-tool filesystem/exec/export/network policy, append-only sensitive-action audit logs, approval/TTL elevation, session isolation, config integrity, and plugin/skill verification.
