## Summary

Keep this PR open. Current main still normalizes Signal shared allowlist and DM-policy entries through phone-number E.164 normalization, which strips `uuid:` values instead of preserving them. The PR remains a focused candidate fix, even though current main has separate UUID-aware Signal approval-auth and monitor paths.

## Best Possible Solution

Review and rebase this PR against current main, preferably by sharing one Signal allow-entry normalizer between shared config formatting, DM-policy normalization, and the existing approval-auth path; add or keep targeted regression tests for UUID allowlist entries before landing.
