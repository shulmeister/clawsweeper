## Summary

Current main already fixes the PR's central user problem, but at the source: memory-core dreaming narrative sub-sessions are transient, deleted after use, and stale leaked dreaming artifacts are scrubbed. The exact Control UI "Hide system" toggle from PR #67155 is not on main, but the rows it was intended to hide should no longer accumulate in the Sessions tab.

## Best Possible Solution

Keep the shipped memory-core cleanup as the fix for #67058's observable clutter. Do not merge the UI-only toggle from PR #67155 unless maintainers later decide they want a broader product-level Sessions filter for a newly identified remaining system-session family.
