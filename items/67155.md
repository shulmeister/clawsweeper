## Summary

PR #67155 can close as implemented on current main. The exact Control UI Hide system checkbox is not present, but the user-visible problem it worked around, accumulated memory-core dreaming narrative sessions, is fixed at the source by deleting transient narrative sessions, scrubbing stale leaked artifacts, and covering the cleanup path with regression tests.

## Best Possible Solution

Keep the shipped memory-core cleanup as the fix for the observable Sessions-tab clutter. Do not merge the UI-only hide-system toggle from #67155 unless a new accumulating system-session family appears or maintainers explicitly want a broader non-destructive session visibility feature.
