## Summary

Keep #69199 open. Current main and the latest release tag still send missing node:sqlite failures through the generic embedding/provider unavailable message, while the PR is the active narrow respin for that exact UX bug. The related #59451 is a longer-term fallback feature, and closed #59637 is the noisy predecessor superseded by this PR, not a reason to close it.

## Best Possible Solution

Keep #69199 open for maintainer review and land this narrow fix or an equivalent patch on main: add node:sqlite-missing detection in buildMemorySearchUnavailableResult, preferably normalize the checked string as the review comment suggests, and keep the regression test in extensions/memory-core/src/tools.test.ts. Leave fallback keyword search tracked separately in #59451.
