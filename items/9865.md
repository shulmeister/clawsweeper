## Summary

Keep open. The request is clear, non-maintainer-authored, and has no protected labels, but current main does not implement first-class provider Batch API routing for background LLM turns. Existing batch support is scoped to embeddings/media-style paths, while cron, sub-agent, and heartbeat model execution still expose normal model/timeout/light-context controls rather than a deferred batch mode. This also should not be closed as ClawHub because current provider/plugin surfaces are StreamFn-based and do not expose a durable provider batch-turn contract for multi-hour deferred agent completions.
