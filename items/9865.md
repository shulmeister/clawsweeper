## Summary

Keep #9865 open. Current main still lacks first-class OpenAI/Anthropic Batch API routing for delayed background LLM turns in cron, heartbeat-style work, sessions_send, and sessions_spawn/subagent execution. The existing batch surfaces are for memory embeddings, media/STT, inbound message coalescing, browser actions, or config writes, not provider batch turn submission, polling, cancellation, usage reconciliation, and transcript delivery. The prior issue discussion also explicitly treats #9865 as the canonical tracker while #49326, #70606, and #56126 are closed related duplicate/follow-up context.

## Best Possible Solution

Keep #9865 open as the canonical product/API tracker. The best path is an additive provider/runtime batch-turn contract with submit, poll, cancel, timeout, failure, usage, and transcript reconciliation semantics, followed by explicit opt-in surfaces for cron agentTurn jobs, heartbeat/background runs, sessions_send, and sessions_spawn/subagents, plus plugin SDK docs and focused regression coverage. Treat #49326, #70606, and #56126 as closed related duplicate/follow-up context rather than replacements.
