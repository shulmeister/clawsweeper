## Summary

Keep #9865 open. Current main still lacks a first-class provider Batch API contract for delayed/background LLM turns; existing batch support is for embeddings, media, config edits, or message coalescing, not cron/subagent/heartbeat model-turn routing. The latest issue discussion also explicitly treats #9865 as the canonical tracker rather than a duplicate or ClawHub-only request.

## Best Possible Solution

Keep #9865 open as the canonical product/API tracker. The best path is an additive provider/runtime batch-turn contract with submit, poll, cancel, timeout, failure, usage, and transcript reconciliation semantics, then explicit opt-in fields for cron agentTurn jobs, heartbeat/background runs, sessions_send, and sessions_spawn/subagents, with OpenAI/Anthropic provider implementations, docs, and focused regression coverage.
