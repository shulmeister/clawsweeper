## Summary

Keep #9865 open. Current main still lacks first-class provider Batch API routing for background LLM turns in cron, heartbeat, or subagent execution. Existing batch support is scoped to embeddings/media-style work, and the provider/plugin seam remains StreamFn-based rather than a durable submit/poll/reconcile batch-turn contract. The provided discussion also already treats #9865 as the canonical tracker, with #49326 proposed as a duplicate rather than a replacement.

## Best Possible Solution

Keep #9865 open as the canonical product/API tracker. The likely path is maintainer-approved provider/runtime batch-turn support with submit, poll, cancel, reconcile, and delayed delivery semantics, then explicit config/tool surfaces for cron jobs, heartbeat runs, and sessions_spawn/subagent work, plus docs and regression coverage. Until that contract exists, this should not be closed as implemented or ClawHub-only.
