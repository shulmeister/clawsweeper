## Summary

Keep #9865 open. Current main still does not provide first-class provider Batch API routing for background LLM turns in cron, heartbeat, or subagent execution; the batch support present today is scoped to embeddings/media/channel mechanics, and the provider/plugin seam remains immediate StreamFn-based rather than a durable submit/poll/reconcile contract.

## Best Possible Solution

Keep #9865 open as the canonical product/API tracker. The likely maintainer path is a provider/runtime batch-turn contract with submit, poll, cancel, reconcile, timeout, and delayed delivery semantics, followed by explicit opt-in config/tool surfaces for cron jobs, heartbeat runs, and sessions_spawn/subagent work, plus docs and regression coverage. Related #49326 can be treated as a duplicate/follow-up reference rather than a replacement for this tracker.
