## Summary

Keep this PR open. Current main still builds WhatsApp ack-reaction options from `sender.jid` only, while the current inbound identity path can normalize a group participant as `sender.lid` with no `sender.jid`. The PR is a narrow implementation candidate for that remaining bug and adds the missing regression case.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. Rebase or adapt the patch to current main by setting the ack reaction participant from `sender.jid ?? sender.lid` in `extensions/whatsapp/src/auto-reply/monitor/ack-reaction.ts`, preserving direct-message behavior and JID precedence, then keep the LID-only group regression in `extensions/whatsapp/src/auto-reply/monitor/ack-reaction.test.ts` and run the targeted WhatsApp extension format, lint, type, and test lanes before merge.
