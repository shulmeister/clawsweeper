## Summary

Keep #68037 open. Current main still implements the reported bug: the HTTP MCP `tools/call` path normalizes every returned array block down to `{ type, text }`, so non-text MCP content fields such as `data`, `mimeType`, `resource`, `uri`, and `name` are still lost. The provided related PR context shows an open matching fix in #68046 and a closed unmerged alternative in #68043, but the fix is not present on current main.

## Best Possible Solution

Keep the issue open until the HTTP MCP gateway preserves typed MCP content blocks. The likely maintainer path is to adapt or land the focused #68046-style fix in `src/gateway/mcp-http.handlers.ts`, widen the result content type beyond text-only blocks, and add regression tests in `src/gateway/mcp-http.test.ts` for `image`, `audio`, `resource`, and `resource_link` blocks plus existing text/fallback behavior.
