## Summary

Keep PR #64512 open. Current main still does not implement Bedrock `service_tier`: the Amazon Bedrock plugin does not read `extraParams`, the shared Bedrock stream wrapper only exposes cache behavior, the SDK barrel does not export Bedrock service-tier helpers, and Bedrock docs do not document the feature. The PR is not ready as-is because its current diff includes broad unrelated generated/formatting churn, but the core Bedrock feature remains useful and not implemented on main.

## Best Possible Solution

Keep the PR open, but ask for a focused revision: implement Bedrock `serviceTier`/`service_tier` resolution and top-level `{ serviceTier: { type } }` injection, guard injection to `model.api === "bedrock-converse-stream"`, compose it in `extensions/amazon-bedrock/register.sync.runtime.ts` via `extraParams`, add focused tests for valid, invalid, snake_case/camelCase, caller-preserved, and non-Converse cases, update `docs/providers/bedrock.md`, and remove unrelated generated/formatting churn from the branch.
