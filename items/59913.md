## Summary

Keep open. The report is concrete, non-stale, and current main still has the core unsanitized early-return path for `existingImages` when the prompt contains no detected image references. The issue also has a linked open PR (#62070), so this is active fix work rather than conservative cleanup material.

## Evidence

- **Current main still returns existing images before sanitization:** `detectAndLoadPromptImages()` computes `allRefs = detectImageReferences(params.prompt)` and, when `allRefs.length === 0`, returns `images: params.existingImages ?? []` before the later `sanitizeImagesWithLog(..., "prompt:images", ...)` path. That matches the reported OpenAI-compatible gateway path where images arrive as existing images and prompt text has no file/media reference.
