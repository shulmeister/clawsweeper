## Summary

Keep #38714 open. Current main still detects Discord reaction add/remove events, but routes them as session-scoped system events, not as internal Hooks events or typed plugin hook events. The related #52658 report is closed, but it describes the same missing behavior and does not supersede this item with shipped implementation.

## Best Possible Solution

Keep this item open until maintainers choose and implement the contract: either add dedicated typed/internal reaction hook events, or intentionally forward Discord reaction notifications as structured synthetic message_received events with documented metadata and regression tests for Discord reaction add/remove dispatch. Because current plugin surfaces cannot observe these reactions, this should remain an OpenClaw core/plugin-API tracking issue rather than ClawHub-only work.
