## Summary

Keep open. Current `main` still does not expose Discord reaction add/remove through either the internal Hooks system or plugin hook registry. Discord reaction handling enqueues human-readable system events instead, while documented/internal hook events remain limited to message lifecycle events such as `message:received`, `message:preprocessed`, and `message:sent`.

## Evidence

- **Discord reactions become system events, not hook events:** The Discord reaction listener formats reaction text and calls `enqueueSystemEvent(...)`; there is no `triggerInternalHook(...)` or plugin hook dispatch in this path.
  - file: [extensions/discord/src/monitor/listeners.ts:631](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/extensions/discord/src/monitor/listeners.ts#L631)
