## Summary

Keep #38714 open. Current main still receives Discord reaction add/remove events, but the Discord path only enqueues routed session system events. It does not expose reactions through typed plugin hooks, internal Hooks events, or synthetic `message_received` delivery. The closed related items #52658 and #60805 consolidate into #38714 rather than shipping or superseding it.

## Best Possible Solution

Keep #38714 open until maintainers choose the contract. The likely fix should add either dedicated typed/internal reaction events or structured synthetic `message_received` delivery for Discord reaction add/remove events, with payload metadata for emoji, reactor, target message, action/removal, route/session context, Discord listener wiring, plugin/internal hook tests, and docs. Keep broader reaction-triggered wake or auto-response policy coordinated with #11747.
