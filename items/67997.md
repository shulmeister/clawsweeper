## Summary

This PR should close because it optimizes the Mintlify source page for GitHub-rendered browsing, but the repo's docs policy requires internal links in `docs/**/*.md` to remain root-relative. Current `main` keeps the affected `docs/tools/plugin.md` links root-relative, and the docs tooling depends on that form for localized-link rewriting and internal route auditing.

## Best Possible Solution

Do not merge this PR as-is. Keep internal links in `docs/**/*.md` root-relative so Mintlify, docs i18n, and internal link auditing continue to work; use the published docs site for user-facing navigation and reserve absolute `docs.openclaw.ai` links for README or other GitHub-rendered docs surfaces.
