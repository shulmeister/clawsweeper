## Summary

Keep PR #39715 open. Current main still lacks a stdout/stderr drain between child wait resolution and exec/session output snapshotting, and the PR's focused regression coverage is not present. The merged sibling stdin/write fix #43876 does not supersede this stdout/background-output issue.

## Best Possible Solution

Keep PR #39715 open. The best path is to land or adapt a minimal stdout/stderr drain before RunExit and session aggregation are snapshotted, preserve the settled/cancel/runId-reuse guards already discussed on the PR, and add focused regression coverage for backgrounded stdout/stderr flushed at process exit.
