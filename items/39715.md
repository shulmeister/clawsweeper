## Summary

Keep PR #39715 open. Current main still has the stdout/stderr completion path that the PR targets: the child adapter settles on `close`, the supervisor snapshots stdout/stderr immediately after `adapter.wait()`, and background exec completion copies the current aggregate into the finished-session record without a drain barrier. The related sandbox stdin/write fix does not supersede this stdout/background-output loss path.

## Best Possible Solution

Keep PR #39715 open. The best path is to land or adapt a minimal stdout/stderr drain before `RunExit` and finished-session aggregation are snapshotted, preserve the settled/cancel/registry-finalization safeguards from the PR review discussion, and add focused regression coverage for backgrounded stdout/stderr flushed at process exit.

## Evidence

There is no `setImmediate` or equivalent stdout/stderr drain before the snapshot.
