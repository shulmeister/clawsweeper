## Summary

Keep open. Current main still lacks first-class per-message `author` support on `chat.send`: the protocol schema has no `author` field and rejects extra fields, the handler uses gateway client info as sender identity, the persisted user transcript row contains only role/content/timestamp/media, and history/session message paths only surface the existing `senderLabel` sanitizer result. This is gateway wire-contract work, not a ClawHub-only plugin request.

## Best Possible Solution

Implement this as an additive gateway protocol change: add a validated optional `author` object to `ChatSendParamsSchema`, preserve install-class `client.id` semantics, persist normalized author metadata with the user transcript row, project `senderLabel`/`authorId`/`authorName` through `chat.history` and `session.message`, and add focused tests for schema validation, history/event output, spoofing/attestation policy boundaries, and the timestamp-prefixed sanitizer edge case.

## Risks / Open Questions

- Author identity must not become an unauthenticated security signal by accident; any attestation/roster check needs explicit session-kind policy.
- This is a public gateway protocol addition, so schema, validators, generated protocol artifacts, docs, UI/client types, and regression tests need to move together.
