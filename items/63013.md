## Summary

Current main already implements the PR's central fix: unresolved explicit provider/model refs remain an explicit empty plugin scope instead of widening into an unrestricted provider load, preventing unrelated plugins such as memory-lancedb from being loaded and config-validated under cropped config. The implementation is covered by provider and runtime-registry regression tests and is present in v2026.4.24.

## Best Possible Solution

Close PR #63013 as redundant with the shipped explicit-empty provider runtime scope implementation on main. Keep the current main behavior; if maintainers still want more targeted hook-alias owner activation, open a fresh narrow PR against the current manifest activation/provider-owner contract instead of reviving this stale mixed branch.
