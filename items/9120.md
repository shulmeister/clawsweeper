## Summary

Keep open. Current main still only provides timing-based inbound debouncing and active-run queue collection; it does not implement the requested user-controlled continuation marker, marker stripping, marker timeout, or continuation-specific typing mode. The existing issue discussion already reached the same conclusion, and rechecking current main at d74f897c1c6b2de756c98eb3bc7de45f411d6a5c found no newer implementation.

## Best Possible Solution

Keep the issue open for maintainer/product judgment. A complete implementation would likely extend the shared inbound message config/types/schema with `messages.continuationMarker` and timeout semantics, buffer marked messages before any agent invocation, strip the marker from prompt text, release the batch on the first non-marker message or timeout, define media/control-command behavior, add channel-safe tests, document interaction with `messages.inbound` and `messages.queue.collect`, and decide whether typing needs a dedicated continuation suppression mode.
