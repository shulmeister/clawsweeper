## Summary

Keep #9120 open. Current main has timer-based inbound debouncing and active-run followup collection, but it still lacks the requested user-controlled continuation marker workflow, config fields, marker stripping, timeout release, and continuation-aware typing mode.

## Best Possible Solution

Keep the issue open for maintainer/product judgment. A complete path likely belongs in shared inbound handling or first in a documented plugin API seam: add typed messages.continuationMarker and messages.continuationTimeout config/schema/docs, buffer marked text-only inbound messages before agent invocation, strip the marker from model-visible text, release on the first unmarked message or timeout, define media/control-command behavior, and add focused unit/integration coverage across shared channel inbound helpers and typing policy.
