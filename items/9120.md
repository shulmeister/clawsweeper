## Summary

Current main has timing-based inbound debouncing that partially addresses the cost concern, but it does not implement the requested semantic continuation marker, marker stripping, or continuation timeout. Because the explicit marker-based capability remains meaningfully missing, this should stay open for maintainer/product judgment.

## Evidence

- **Existing partial solution:** `messages.inbound` supports batching rapid same-sender messages by debounce window, with per-channel overrides.
