## Summary

Keep open. Current main still provides timing-based inbound debouncing and active-run followup collection, but not the requested user-controlled continuation marker, marker stripping, marker timeout, or continuation-specific typing behavior.

## Best Possible Solution

Keep the issue open for maintainer/product judgment. A complete implementation would likely add typed and documented `messages.continuationMarker` and timeout semantics, validate them in the strict config schema, buffer marked text-only messages before agent invocation, strip the marker from prompt text, release on the first non-marker message or timeout, define media/control-command behavior, update shared channel helpers and bundled channel tests, and decide whether typing needs a dedicated continuation suppression mode.
