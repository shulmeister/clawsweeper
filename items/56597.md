## Summary

Current `main` still persists `/exec` session overrides and resolves them ahead of agent/global config, so the reported Discord behavior remains present rather than fixed. The docs now describe `/exec` as a per-session setting, and the reset path/tests explicitly preserve those exec fields, so there is not strong evidence for an auto-close reason.
