## Summary

Current `main` still exposes `openclaw cron list` as a global job listing. The CLI has no `--agent` flag, the gateway protocol for `cron.list` has no `agentId` field, and the cron service list paths do not scope results by agent. This issue remains valid on `95b7a85f0607784d8e33b4ce7e42617174196490`.

## Evidence

- **CLI list command has no agent filter:** `registerCronListCommand` defines only `--all` and `--json`, then calls `cron.list` with `{ includeDisabled }` only.
  - file: [src/cli/cron-cli/register.cron-add.ts:40](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/cli/cron-cli/register.cron-add.ts#L40)

- **Gateway schema lacks agentId for cron.list:** `CronListParamsSchema` allows `includeDisabled`, paging, query, enabled, and sorting fields, but no `agentId`. `CronListPageOptions` matches that shape.
  - file: [src/gateway/protocol/schema/cron.ts:272](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/gateway/protocol/schema/cron.ts#L272)
