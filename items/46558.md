## Summary

Keep #46558 open. Current main still implements `agents.defaults.memorySearch.extraPaths` as per-agent indexing scope with per-agent sqlite stores, so the reported N x M embedding/storage duplication remains for the builtin memory backend. The linked #46542 is relevant and directly addresses the gap with a proposed shared store, but the provided context shows it is still open and current main does not contain the proposed `sharedPaths`/`_shared.sqlite`/`--shared` surface.

## Evidence

- **Defaults extraPaths are copied into each resolved agent config:** `mergeConfig` builds `extraPaths` from defaults plus per-agent overrides, deduped only within the resolved agent config. The same resolver defaults `store.path` through `resolveStorePath(agentId, ...)`, whose fallback is `memory/${agentId}.sqlite`.
