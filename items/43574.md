## Summary

Keep open. Current main still has the Discord `send` action accepting `media`/`path`/`filePath`, but `thread-reply` still only reads `media` and does not pass `mediaLocalRoots` into `handleDiscordAction`, so the PR remains useful rather than obsolete.

## Best Possible Solution

Keep the PR open and update it against current main: in `extensions/discord/src/actions/handle-action.guild-admin.ts`, make `thread-reply` use the same `media ?? path ?? filePath` precedence as `send` and pass `{ mediaLocalRoots: ctx.mediaLocalRoots }` into `handleDiscordAction`; add focused handler tests for `filePath`, `path`, precedence, and mediaLocalRoots passthrough under the current Discord action test file.
