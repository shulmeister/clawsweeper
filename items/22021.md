## Summary

Keep #22021 open. Current main still does not expose `X-Actual-Model` or an equivalent per-response actual runtime model field on the OpenAI-compatible HTTP endpoints. The prior maintainer/Codex comment remains accurate, and the adjacent session metadata fixes do not satisfy this transport-layer request.

## Best Possible Solution

Keep the issue open. The best path is to define a Gateway HTTP contract for actual provider/model metadata, thread resolved model data from `onModelSelected`, `agentMeta`, or a new run-result metadata seam into `src/gateway/openai-http.ts` and `src/gateway/openresponses-http.ts`, decide non-streaming versus SSE behavior, document it on the public OpenAI-compatible Gateway docs, and add focused tests for both HTTP endpoint files.
