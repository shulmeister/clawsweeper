## Summary

Current `main` does not implement the requested file-based fallback when `node:sqlite` is unavailable. The bundled memory engine still opens a SQLite database unconditionally, converts manager-construction failures into `manager: null`, and `memory_search` responds with a structured unavailable payload instead of searching `MEMORY.md` / `memory/*.md`. The issue discussion also explicitly split the narrower UX/error-message work into separate follow-up items, leaving this fallback feature as the remaining scope.
