## Summary

Keep #13241 open. Current main has supporting issue automation, stale handling, candidate labels, duplicate tooling, issue templates, and broad roadmap language, but it still does not implement the requested lightweight issue triage and priority framework with explicit states, P0-P3 priority taxonomy, cadence, current-cycle priority surface, and stale-state transitions.

## Best Possible Solution

Keep the issue open for maintainer process judgment. A complete solution should define the smallest repo-owned issue state/label taxonomy, clarify who assigns priority and how, establish the triage cadence and stale-state transitions, and create or document a visible current-cycle priorities surface. Existing stale automation, Barnacle candidate labels, duplicate tooling, issue templates, and broad roadmap wording should be treated as supporting infrastructure, not the completed framework.
