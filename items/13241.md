## Summary

Keep #13241 open. Current main has stale cleanup, auto-response triage candidate labels, path/size/beta labels, structured issue intake, duplicate-PR cleanup, and broad roadmap priorities, but it still does not implement the requested lightweight issue triage and priority framework: lifecycle states, P0-P3 priority taxonomy, assignment rules, recurring cadence, visible current-cycle priorities, and documented stale-state transitions.

## Best Possible Solution

Keep the issue open for maintainer process design. The best path is a small repo-owned triage playbook plus matching GitHub labels/project surface: define lifecycle states, P0-P3 semantics, who can assign priority, cadence, stale-state transitions, and a visible current-cycle priorities board or pinned issue, while reusing the existing stale workflow, Barnacle cleanup labels, issue templates, duplicate cleanup workflow, and roadmap docs as supporting infrastructure.
