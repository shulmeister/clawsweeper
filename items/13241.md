## Summary

Keep #13241 open. Current main has useful partial triage automation, issue intake templates, stale handling, and PR candidate labels, but it still does not implement the proposed lightweight issue triage and priority framework: no repo-owned P0-P3 priority taxonomy, no needs-repro/confirmed/blocked state policy, no regular triage cadence, and no visible current-cycle priority board or pinned issue. The existing issue thread also contains a recent Codex review from steipete explicitly reaching the same keep-open conclusion.

## Best Possible Solution

Keep the issue open for maintainer product/process judgment. A complete solution would define a minimal repo-owned label/state taxonomy, document who assigns priority and how, establish the triage cadence and stale-state transitions, and create or document a visible current-cycle priorities surface. Existing Barnacle, labeler, stale automation, and issue templates can remain supporting infrastructure but do not close this proposal.
