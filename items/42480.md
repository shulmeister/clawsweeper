## Summary

Keep PR #42480 open. Current main still has the Discord debounce synthetic-message gap: the multi-entry flush spreads the last Carbon message into a plain object and does not materialize `mentionedUsers`, `mentionedRoles`, or `mentionedEveryone`, while Discord preflight mention gating still relies on those fields. The PR discussion also shows the initial reviewer gap for roles/everyone was addressed in follow-up commit `ebcfeb5`, so the remaining maintainer path is to port/rebase the fix onto the current `extensions/discord` tree and add a focused regression test.

## Best Possible Solution

Keep the PR open and ask for a current-main port: materialize `mentionedUsers` and `mentionedRoles` from all debounced entries with de-duplication, set `mentionedEveryone` if any entry has it, and add a focused `extensions/discord/src/monitor/message-handler.queue.test.ts` regression proving preflight receives those fields for a synthetic batched message.
