## Summary

Keep open. Current main still routes verbose-off edit failures through the generic mutating-tool warning path, so the user-visible payload can remain a non-actionable “Edit failed” warning rather than explaining exact oldText/whitespace/newline mismatch. The underlying edit wrapper adds context for tool/model recovery, and existing tool policy can disable edit, but the specific chat-surfacing bug is not shipped. The linked PR #67276 is the active implementation candidate, not a landed fix.

## Best Possible Solution

Keep #66915 open until #67276 or an equivalent patch lands. The best fix is a narrow payload-builder change that detects edit exact-match mismatch errors and surfaces a concise sanitized verbose-off message mentioning exact text, whitespace, and newlines, with regression coverage; document existing `tools.deny: ["edit"]` for write-only deployments rather than adding overlapping config unless maintainers want a dedicated alias.
