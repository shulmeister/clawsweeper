## Summary

Close as ClawHub/plugin-scope. Outlook/M365 triage is a useful optional Microsoft 365 service integration, but current OpenClaw direction keeps optional/commercial service integrations out of core unless a missing plugin API is demonstrated. Current main has the plugin and approval surfaces needed for an external implementation, and this PR also has unresolved safety/regression findings in the provided review discussion.

## Best Possible Solution

Move the Outlook triage assistant to an external ClawHub/npm plugin using the current plugin APIs and existing plugin approval flow. If a generic Teams adaptive-card plugin callback gap remains after that, handle it as a small core/Microsoft Teams plugin API PR separate from the M365 integration.
