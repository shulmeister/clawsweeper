## Summary

Close as implemented on `main`: the OpenAI-compatible streaming request builder now always sends `stream_options.include_usage`, with targeted Qwen/DashScope coverage and a changelog entry confirming the fix; the latest tagged release `v2026.4.23` already contains the implementation.

## Evidence

- **Current main forwards streaming usage request:** `buildOpenAICompletionsParams()` now unconditionally includes `stream_options: { include_usage: true }` in the `openai-completions` streaming payload, which directly addresses the reported missing field.
  - file: [src/agents/openai-transport-stream.ts:1719](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/openai-transport-stream.ts#L1719)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Regression tests match the report's affected providers:** Tests explicitly assert `stream_options.include_usage` for native Qwen/DashScope endpoints, generic DashScope endpoints, explicit custom OpenAI-compatible providers, and local backends like `llama-cpp`.
  - file: [src/agents/openai-transport-stream.test.ts:1422](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/openai-transport-stream.test.ts#L1422)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Changelog documents the exact bugfix:** The changelog entry says OpenAI completions now always send `stream_options.include_usage` on streaming requests so OpenAI-compatible backends stop reporting 0% context usage.
  - file: [CHANGELOG.md:472](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/CHANGELOG.md#L472)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Provider docs reflect DashScope/Qwen streaming-usage support:** The Qwen provider docs now describe native Model Studio / DashScope endpoints as advertising streaming-usage compatibility on the shared `openai-completions` transport, consistent with the issue report.
  - file: [docs/providers/qwen.md:265](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/docs/providers/qwen.md#L265)
