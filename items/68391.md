## Summary

Keep open. Current main still matches the reported bug path: `openclaw models` registers a parent `--agent` flag, but `models set` and `models set-image` ignore command options and unconditionally call global default write commands. Those commands still write `agents.defaults.model.primary` / `agents.defaults.imageModel.primary`, not `agents.list[]`. The related PR #68399 proposed the conservative rejection fix, but the provided GitHub context shows it was closed unmerged, and the current checked-out source does not contain its guard or regression tests.

## Best Possible Solution

Implement the conservative fix from the closed PR #68399 unless maintainers explicitly want full per-agent model override support now: have `models set` and `models set-image` inspect inherited `--agent` via `resolveOptionFromCommand` and fail with a clear error before invoking the write path. Add focused CLI regression tests proving parent-level `--agent` is rejected and the downstream global write commands are not called. If product direction is to support per-agent model overrides instead, wire the command to update the selected `agents.list[]` entry and document that behavior.
