## Summary

Keep #72205 open. Current main still has the reported Discord preview tool-progress duplication path: the helper only dedupes an exact match against the immediately previous line, while Discord pushes both `tool: <name>` and bare `<name>` from different callbacks. The linked PR #72213 is the right focused follow-up, but it is still open in the provided context and its head commit is not in this checkout.

## Best Possible Solution

Keep #72205 open and review or land PR #72213, or an equivalent Discord plugin patch that canonicalizes tool-progress labels and dedupes against existing preview entries, with a regression test in `extensions/discord/src/monitor/message-handler.process.test.ts`. Let the merged fix close the issue.

## Evidence

- **Current helper only dedupes last exact line:** `pushPreviewToolProgress` normalizes whitespace, then compares only `previewToolProgressLines.at(-1)` to the new normalized string. It does not compare against prior entries or canonicalize the `tool: ` prefix, so `tool: session_status` and `session_status` remain distinct.
