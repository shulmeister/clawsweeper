## Summary

Keep #7057 open. Current main still exercises the reported WSL-sensitive `pi-tools.workspace-paths` and `pi-tools.safe-bins` test paths without WSL guards or temp-dir cleanup hardening, while both related mitigation PRs were closed unmerged. The issue has concrete reproduction details and targets WSL2, which current docs describe as a supported/recommended Windows path.

## Best Possible Solution

Keep #7057 open until a focused test-only fix lands for `src/agents/pi-tools.workspace-paths.test.ts` and `src/agents/pi-tools.safe-bins.test.ts`. The maintainer path should either harden temp-dir creation/cleanup around cwd-sensitive tests or apply a narrowly justified WSL skip using the existing WSL detector, with WSL proof or diagnostics and no unrelated changes.
