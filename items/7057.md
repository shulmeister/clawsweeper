## Summary

Keep #7057 open. Current main still does not address the reported WSL timeout/ENOENT paths in the two named test files, and the directly related mitigation PRs were closed unmerged. The issue remains concrete and actionable.

## Best Possible Solution

Keep #7057 open until a focused test-only fix lands. The likely maintainer path is to update `src/agents/pi-tools.workspace-paths.test.ts` and `src/agents/pi-tools.safe-bins.test.ts` with targeted WSL handling using the existing WSL detector where justified, plus robust temp-dir creation/cleanup around cwd-sensitive tests and enough diagnostics or WSL proof to show the reported timeout/ENOENT path is addressed. Avoid broad unrelated mitigation PRs.
