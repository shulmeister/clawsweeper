## Summary

Close #64184 as implemented on current main. The exact proposed `MediaFileIds` field is not present, but current Slack ingestion now exposes native Slack file IDs in agent-visible file placeholders, carries those placeholders into `RawBody`/`BodyForAgent`, and documents that `download-file` uses those IDs. The same behavior is present in release `v2026.4.24`.

## Best Possible Solution

Close this PR as obsolete on current main. Keep the shipped Slack placeholder-based workflow for `download-file`; open a narrower follow-up only if maintainers want a tested, cross-channel structured media-ID API beyond Slackâ€™s documented placeholders.
