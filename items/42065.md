## Summary

Keep #42065 open. Current main has since added broader Control UI locale infrastructure, generated zh-CN coverage, structural locale tests, and a `pnpm ui:i18n:check` workflow, so much of the PR is now overlapped. The specific global/topbar locale switch requested by this PR is still not present, and several key shell/topbar strings remain hardcoded, so there is no conservative auto-close reason.

## Best Possible Solution

Keep the PR open only as a product/UI follow-up for a real global locale control. The best path is to rebase or replace it with a small current-main patch that adds a topbar or otherwise globally reachable locale switch, awaits successful `i18n.setLocale` before persisting settings, and uses the existing `scripts/control-ui-i18n.ts` / `pnpm ui:i18n:sync` generated-locale workflow instead of the older hand-maintained zh-CN/report-script approach.
