## Summary

Keep open. This is a recent non-maintainer regression report for the bundled Telegram native `/restart` path. Current `main` still has a Telegram native pre-dispatch auth gate and a separate owner-only restart gate that can both produce the same unauthorized reply, and I found no current source, test, docs, or history evidence that the reported owner-allowlisted Telegram restart failure is fixed or superseded.

## Best Possible Solution

Keep this issue open. The maintainer path is to request the exact Telegram config and sender/chat IDs, then add a focused regression test in `extensions/telegram/src` that drives native `/restart` through Telegram authorization into `handleRestartCommand` with `channels.telegram.allowFrom`, `channels.telegram.groupAllowFrom`, `commands.allowFrom.telegram`, and `commands.ownerAllowFrom`. If reproduced, fix the Telegram native preauth/owner handoff or fresh config resolution so a correctly owner-allowlisted Telegram sender can restart without weakening owner-only command gating.
