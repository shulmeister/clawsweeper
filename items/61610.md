## Summary

Keep #61610 open. Current main has partial task-maintenance and CLI-cancel fixes, but the issue's core recovery path is still incomplete: stock `openclaw tasks cancel` still does not cancel cron-runtime task records, maintenance intentionally keeps active cron-owned stale tasks running, and TaskFlow cancellation still remains blocked when active child tasks cannot be cancelled. The raw34 follow-up about flow-level deadlock is therefore still relevant rather than fully superseded by #60299/#62419/#62506.

## Evidence
