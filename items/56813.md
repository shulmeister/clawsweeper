## Summary

Keep open. Current `main` still bundles the unscoped `openshell` npm package inside the OpenShell sandbox plugin and explicitly prefers that bundled executable over the `openshell` command on `PATH`, so the reported collision is not fixed on current code.

## Evidence

- **Bundled dependency still present on main:** `extensions/openshell/package.json` still declares `"openshell": "0.1.0"` as a runtime dependency, so the published OpenShell sandbox plugin continues to bundle that npm package.
  - file: [extensions/openshell/package.json:7](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/openshell/package.json#L7)
  - command: `nl -ba extensions/openshell/package.json | sed -n '1,80p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Runtime still prefers bundled module over PATH:** `resolveBundledOpenShellCommand()` does `require.resolve("openshell/package.json")`, derives the bundled binary path, and `resolveOpenShellCommand("openshell")` returns that bundled command before falling back to plain `openshell` from `PATH`. That matches the issue's failure mode rather than avoiding it.
  - file: [extensions/openshell/src/cli.ts:31](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/openshell/src/cli.ts#L31)
  - command: `nl -ba extensions/openshell/src/cli.ts | sed -n '1,120p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Tests currently lock in the problematic preference:** The unit test explicitly expects OpenClaw to prefer a bundled OpenShell command when available, and only use the `PATH` command when no bundled copy exists.
  - file: [extensions/openshell/src/openshell-core.test.ts:43](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/openshell/src/openshell-core.test.ts#L43)
  - command: `nl -ba extensions/openshell/src/openshell-core.test.ts | sed -n '35,65p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **User-facing docs expect PATH-installed CLI:** The docs say the prerequisite is 'The `openshell` CLI installed and on `PATH` (or set a custom path via `plugins.entries.openshell.config.command`)', which conflicts with the current implementation preferring a bundled npm package.
  - file: [docs/gateway/openshell.md:21](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/gateway/openshell.md#L21)
  - command: `nl -ba docs/gateway/openshell.md | sed -n '18,28p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Reporter's root-cause analysis matches current code:** The issue comment points to an npm name collision with unscoped `openshell`; current main still depends on that exact unscoped package name, so the discussion materially supports keeping the bug open.
