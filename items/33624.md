## Summary

Keep #33624 open. Current main documents and implements Gateway WebSocket/HTTP APIs, plus a TypeBox-to-Draft-07 JSON Schema generator for the WebSocket protocol, but it still does not publish or serve a Swagger/OpenAPI schema for custom client generation. The prior ClawSweeper comment also correctly kept this open, and the provided related items do not supersede this specific request.

## Best Possible Solution

Keep the issue open for maintainer/product follow-through. A good implementation would define the intended contract boundary first, then add a generated and committed OpenAPI document or a served `/openapi.json` endpoint for the Gateway HTTP APIs, with tests proving the route/spec stays in sync. If maintainers want the WebSocket protocol covered too, that likely needs separate JSON-Schema-to-client guidance rather than treating the existing Draft-07 protocol schema as a Swagger replacement.
