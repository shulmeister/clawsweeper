## Summary

Keep #33624 open. Current main still has Gateway HTTP and WebSocket API docs plus a generated Draft-07 JSON Schema for the WebSocket protocol, but it does not publish a Swagger/OpenAPI document or serve an `/openapi.json`/Swagger route for custom client generation. The prior ClawSweeper comment reached the same conclusion, and the provided related items are about other API/plugin surfaces rather than an OpenAPI schema for OpenClaw.

## Best Possible Solution

Keep the issue open for maintainer/product follow-through. The likely path is to define the intended public contract boundary first, then add either a committed generated OpenAPI document or a served `/openapi.json` endpoint for the Gateway HTTP APIs, with tests that keep the route/spec in sync. If maintainers also want WebSocket client generation, treat the existing TypeBox/Draft-07 protocol schema as a separate contract rather than a Swagger replacement.
