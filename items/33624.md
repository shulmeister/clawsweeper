## Summary

Keep #33624 open. Current main still has documented Gateway HTTP endpoints and a separate WebSocket protocol JSON Schema pipeline, but no OpenAPI/Swagger document, generator, dependency, or /openapi.json-style route for HTTP client generation. The prior ClawSweeper keep-open comment remains accurate against current main e672b61417af5c45b0431df6d9109a1f4b618ef5.

## Best Possible Solution

Keep the issue open. Define the supported public Gateway HTTP API boundary, then add either a committed/generated OpenAPI document or a served /openapi.json endpoint, plus docs and a check or test that keeps it synchronized. Treat the existing WebSocket protocol JSON Schema as a separate contract unless maintainers explicitly decide to model part of that surface in OpenAPI.
