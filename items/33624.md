## Summary

Keep #33624 open. Current main documents and serves specific Gateway HTTP endpoints, and separately generates a WebSocket protocol JSON Schema, but it still does not provide an OpenAPI/Swagger document, generator, dependency, or `/openapi.json`-style discovery route for HTTP client generation.

## Best Possible Solution

Keep the issue open. Maintainers should define the supported public Gateway HTTP API boundary, then add either a committed/generated OpenAPI document or a served `/openapi.json` endpoint, plus docs and a synchronization check or regression test. Treat the existing WebSocket protocol JSON Schema as a separate contract unless maintainers explicitly decide to model part of that surface in OpenAPI.
