## Summary

Keep #33624 open. The issue asks for an OpenAPI/Swagger schema so users can generate clients for OpenClaw programmatic access. Current main documents and implements Gateway HTTP endpoints, and it has TypeBox/Draft-07 schema machinery for the WebSocket protocol, but there is still no committed OpenAPI/Swagger document, no `/openapi.json`/Swagger route, and no OpenAPI generation/check script. The prior ClawSweeper comment and the implementation-options comment remain consistent with the current code state.

## Best Possible Solution

Keep the issue open for maintainer/product follow-through. Define the supported public HTTP API boundary, then add either a committed/generated OpenAPI document or a served `/openapi.json` endpoint for Gateway HTTP APIs, plus a check/test that keeps the spec synchronized. Treat the existing TypeBox/Draft-07 WebSocket protocol schema as a separate contract unless maintainers intentionally decide to model that surface in OpenAPI too.
