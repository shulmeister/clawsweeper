## Summary

Current `main` already addresses this report: gateway service entrypoint resolution now prefers `dist/index.js` over legacy `dist/entry.js`, `openclaw update` refreshes a loaded gateway service with `gateway install --force` before restart, and doctor has update-mode repair coverage for `entry.js -> index.js` service mismatches. The fix is present on `main` after the latest tagged release, so it is implemented but not yet shipped in `v2026.4.23`.

## Evidence

- **Service entrypoint resolution prefers `index.js`:** `resolveCliEntrypointPathForService()` builds candidate dist entrypoints and returns the first accessible one; the shared candidate order is `index.js`, `index.mjs`, `entry.js`, `entry.mjs`, which directly avoids treating `dist/entry.js` as the preferred service target when `dist/index.js` exists.
