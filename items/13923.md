## Summary

Current `main` already implements queryable provider usage/quota surfaces: agents can read usage through `session_status`, gateway clients can call structured `usage.status`, and Anthropic usage parsing includes week reset plus Sonnet/Opus breakdown.

## Risks / Open Questions

- The current implementation normalizes provider usage into window summaries; it does not expose every provider-specific raw field from the issue text as a dedicated top-level property.

## Close Comment

Closing from a Codex review because current `main` already implements the core request. Agents can query provider usage through `session_status` (`docs/concepts/session-tool.md:81`, `src/status/status-text.ts:192`), and gateway clients have a structured `usage.status` method for provider quota summaries (`src/gateway/server-methods/usage.ts:393`, `docs/gateway/protocol.md:281`). I also checked the latest shipped release commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5` and it already contains these surfaces, so this feature is implemented on released code.
