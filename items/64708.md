## Summary

Keep this PR open. Current main still has the mixed Telegram bare-target/topic-session failure mode: `resolveDeliveryTarget()` only preserves a session-derived `threadId` when `resolved.to === resolved.lastTo`, so `-100...` does not match `telegram:-100...:topic:2996`. The PR is pointed at a real bug, but the submitted diff is not merge-ready because it imports a non-existent module and its added test stub would not exercise Telegram target normalization.

## Best Possible Solution

Keep this PR open for rework or replace it with an equivalent maintainer patch. The best fix is to import `normalizeTargetForProvider` from `src/infra/outbound/target-normalization.ts`, compare explicit and session routes through provider-aware normalization before preserving session-derived `threadId`, add a focused regression where a loaded Telegram test plugin normalizes `-100...` and `telegram:-100...:topic:<id>` to the same route, and keep the existing stale-thread guard for genuinely different recipients.
