## Summary

Keep #71717 open. The report is fresh, non-maintainer-authored, and describes a concrete Windows 11 host-exec failure mode (`spawn EPERM`) that is not covered by the related Windows exec items. Current main still has a plausible all-command failure path: gateway exec builds a PowerShell argv and sends it through the child process supervisor, while the spawn wrapper only retries `EBADF` by default and has no Windows `EPERM` fallback or alternate shell selection.

## Best Possible Solution

Keep the issue open and ask for a Windows 11 reproduction with the exact exec command, full redacted tool error, OpenClaw/Gateway logs around the spawn, `where pwsh`, `where powershell`, and whether PowerShell can be started by the same user/service account. If reproduced, the likely code path to fix is Windows shell selection and child spawn recovery: probe candidate PowerShell executables or fall back to the next shell candidate on `EPERM`, add bounded diagnostics for the selected shell path/error code, and add a regression test around `runExecProcess`/`createChildAdapter` for child-mode Windows `EPERM`.
