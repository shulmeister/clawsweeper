## Summary

Current main already solves the reported Control UI/WebChat noise: heartbeat user prompts and OK-only acknowledgments are filtered from visible chat history and live display projections while the underlying transcript remains intact. The implementation does not add a public `showPrompt` flag, but it implements the requested default behavior directly. The related #49374 context covered OK/poll suppression; this review verified the prompt-specific filtering path too.

## Best Possible Solution

Keep the current main behavior: heartbeat prompts and OK-only acknowledgments remain in the transcript for audit/replay but are hidden from Control UI/WebChat display paths by the shared chat projection. A separate `showPrompt` knob is not needed for the reported UX problem unless maintainers later want an explicit debugging/audit display mode.
