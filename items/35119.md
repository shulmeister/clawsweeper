## Summary

Keep #35119 open. Current main fixes the original structured HTTP 500 / JSON `server_error` cases, but the later April 20 raw OpenAI Codex wording is still absent from the classifier, and the assistant fallback policy still requires a failover signal, classified reason, or timeout before it advances to configured model fallbacks.

## Best Possible Solution

Keep the issue open. Add a focused regression for an OpenAI Codex assistant turn with `stopReason: "error"`, the raw text `The server had an error processing your request...`, and configured model fallbacks, then classify that wording as a transient server failure so the assistant path reaches the fallback model instead of surfacing the raw error.

## Evidence
