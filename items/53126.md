## Summary

Keep open. Current `main` exposes the voice-call CLI and gateway RPC, but the CLI still starts its own local voice-call runtime and webhook server on `serve.port` (default `3334`) instead of delegating to a running gateway instance. That leaves the reported CLI port-conflict behavior plausible on current `main`. The RPC method exists and has unit coverage, but without a live repro there is not enough evidence to close the issue as fixed or obsolete.

## Evidence
