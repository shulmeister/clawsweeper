Keep #40392 open. Current main still implements numbered config backup rotation and tests assert `.bak.1`-through-`.bak.4` behavior, so the datetime-suffix request is not implemented. The PR is coherent and targets the live config-write path, but it has an unresolved migration/data-retention concern and overlapping open work without a clear maintainer-chosen canonical replacement.

## Best Possible Solution
