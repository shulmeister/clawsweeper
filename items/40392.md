## Summary

Keep #40392 open. Current main still uses numbered config backups, while this PR is a focused, coherent implementation of the still-open datetime-backup request. It is not implemented on main and is not clearly superseded; however, it is not merge-ready because review discussion identifies migration-time deletion of legacy .bak.N backups.

## Best Possible Solution

Keep the PR open unless a maintainer deliberately chooses another canonical item. The best implementation should preserve or migrate existing .bak.N backups during rollout, keep the bounded retention contract, handle timestamp collisions without clobbering or pruning newer rollback points, and update the config backup tests around those migration and collision cases.
