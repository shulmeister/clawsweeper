## Summary

Issue #11919 proposes expanding the core skill format into a dependency/interface/composition system, but current project direction keeps skills intentionally simple and pushes optional ecosystem capability toward plugins and ClawHub rather than new core orchestration layers.

## Close Comment

Closing this as a ClawHub-scope idea from a Codex review; if a future plugin/ClawHub prototype proves the core loader is the blocker, please open a fresh issue with that concrete evidence.
