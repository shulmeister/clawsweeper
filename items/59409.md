## Summary

Keep open. Current `main` still lacks the Latin-1-to-UTF-8 recovery described in the report for Feishu inbound attachment filenames, and the linked candidate fix was closed without merging.

## Evidence

- **Current Feishu decoder does not recover mojibake `filename=` values:** `decodeDispositionFileName` only decodes RFC5987 `filename*=` and otherwise returns the plain `filename=` token trimmed. There is no Latin-1-to-UTF-8 repair path for headers like the reported `å...` mojibake.
  - file: [extensions/feishu/src/media.ts:127](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/feishu/src/media.ts#L127)
  - command: `nl -ba extensions/feishu/src/media.ts | sed -n '127,139p'`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Inbound save path uses the decoded filename directly:** Feishu inbound message processing passes `result.fileName || mediaKeys.fileName` into `saveMediaBuffer(...)`. If the decoded name is garbled, that garbled stem is what gets persisted to disk.
  - file: [extensions/feishu/src/bot-content.ts:427](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/feishu/src/bot-content.ts#L427)
  - command: `nl -ba extensions/feishu/src/bot-content.ts | sed -n '427,442p'`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Media store preserves provided filename stem:** `buildSavedMediaId` derives the saved media ID from `originalFilename` after sanitization only; it does not do any encoding repair. A mojibake filename therefore remains mojibake in the saved path.
  - file: [src/media/store.ts:271](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/media/store.ts#L271)
  - command: `nl -ba src/media/store.ts | sed -n '271,285p'`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Mainline tests do not cover the reported mojibake case:** The current Feishu media tests cover basic header metadata extraction, but the branch checked out on `main` has no tests for Chinese/mojibake `filename=` recovery or `filename*=UTF-8` Chinese attachment names.
  - file: [extensions/feishu/src/media.test.ts:591](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/feishu/src/media.test.ts#L591)
