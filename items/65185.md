## Summary

Keep this PR open. Current main still forces manual iOS gateway connections to TLS for every non-loopback host, and both Swift and TypeScript gateway clients still choose a bootstrap token before an explicit password in mixed-auth payloads. The PR is a plausible implementation candidate for the linked #47887 regression, but its submitted test diff has review-identified compile and assertion problems that need author changes before merge.

## Best Possible Solution

Keep this PR open as an implementation candidate for #47887, but request changes: move the new Swift tests inside `GatewayConnectionSecurityTests`, update the stale `openclaw.local` assertion, wire iOS manual TLS resolution to the shared local-network host policy with explicit coverage for LAN and Tailscale cases, and make password-over-bootstrap precedence consistent in Swift and TypeScript or split the TypeScript half into a clearly linked follow-up.
