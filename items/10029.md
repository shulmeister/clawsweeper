## Summary

Keep #10029 open. Current main still renders WebChat code blocks as escaped monospace content with language labels, copy-button chrome, and JSON collapse handling, but it does not tokenize code, call a syntax highlighter, include WebChat highlighter styles, or test highlighted output. The existing maintainer/Codex issue comment correctly separated the already-shipped copy-button work from this still-missing syntax-highlighting request.

## Best Possible Solution

Keep #10029 open until Control UI/WebChat adds sanitization-compatible syntax highlighting in `ui/src/ui/markdown.ts` or an adjacent renderer layer, includes theme CSS for highlighted tokens, and adds focused tests proving highlighted fenced code while preserving copy, language label, JSON collapse, cache, and fallback behavior.
