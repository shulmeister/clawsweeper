## Summary

Keep #10029 open. Current main still renders WebChat code blocks as escaped monospace text with language labels, copy-button chrome, and JSON collapse behavior, but no syntax highlighter is wired into the WebChat markdown path. The related #30390 copy-button work is adjacent and already present, but it does not satisfy syntax highlighting.

## Best Possible Solution

Keep #10029 open until WebChat implements real syntax highlighting in `ui/src/ui/markdown.ts`, with an approved highlighter or markdown-it highlight integration, theme CSS matching the Control UI, and focused tests proving highlighted fenced code while preserving the current copy button, language label, and JSON collapse behavior.
