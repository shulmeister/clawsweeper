## Summary

Keep #10029 open. Current main still renders WebChat fenced and indented code blocks as escaped plain text with language labels, copy controls, and JSON collapse support, but no syntax highlighter/tokenizer is wired into the WebChat markdown path. The cross-referenced #30390 copy-button work is adjacent and already represented in current code, but it does not satisfy the requested syntax highlighting.

## Best Possible Solution

Keep #10029 open until WebChat implements real syntax highlighting in the markdown rendering path, with an approved highlighter or markdown-it highlight integration, theme CSS that matches the UI, and focused tests proving highlighted output for fenced code while preserving the existing copy-button, language-label, and JSON-collapse behavior.
