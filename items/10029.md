## Summary

Keep #10029 open. Current main has WebChat code-block rendering, language labels, JSON collapse, and copy-button support, but it does not implement syntax highlighting: fenced code is HTML-escaped and emitted as plain text inside <code class="language-..."> without a highlighter/tokenizer, and the chat CSS only styles the block container. The related #30390 discussion is adjacent and appears reflected by the copy-button implementation, but it does not supersede the syntax-highlighting request.
