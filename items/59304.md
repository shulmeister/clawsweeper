## Summary

Keep open. Current `main` has related update-hint plumbing for gateway startup and `openclaw status`, but it does not implement the requested zero-network `openclaw update hint` subcommand or the proposed skill-preamble surfacing.

## Evidence

- **No `update hint` command on main:** The `update` CLI currently wires `status` as its only documented subcommand in this area, and the public update docs list `openclaw update`, `openclaw update status`, and `openclaw update wizard`, but not `openclaw update hint`.
  - file: [src/cli/update-cli.ts:128](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/cli/update-cli.ts#L128)
  - command: `nl -ba src/cli/update-cli.ts | sed -n '120,180p'; nl -ba docs/cli/update.md | sed -n '15,60p'`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Main already has only partial overlap:** OpenClaw already persists cached update availability from `update-check.json`, logs a startup hint when a newer version is available, and `status` can format an `Update available ... Run: openclaw update` message. That is adjacent functionality, but it is not the requested silent local subcommand for skill preambles.
  - file: [src/infra/update-startup.ts:157](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/infra/update-startup.ts#L157)
