## Summary

Current main still lacks the requested runtime `web_fetch` / `browser` fallback guidance in missing web_search API-key messages. The PR remains useful, but it targets the old core wrapper while current execution and missing-key payloads are provider-owned, so it needs a refresh rather than closure.

## Best Possible Solution

Keep the PR open and ask for a refresh against current main: update the provider-owned web-search runtime missing-key payloads for Brave, Perplexity, Grok/xAI, Gemini, and Kimi, preserve the real env var names/config paths, avoid baseline/generated churn, and add focused message coverage where practical.
