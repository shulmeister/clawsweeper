## Summary

Keep PR #39984 open. The requested user-facing improvement is still not implemented in current main: `web_search` now delegates execution to provider-owned runtime tools, and the current Brave, Perplexity, Gemini, Grok/xAI, Kimi/Moonshot, and MiniMax missing-key messages still only describe API-key configuration, without the requested `web_fetch` or `browser` fallback guidance. The PR itself needs a refresh because it targets the old core wrapper and prior review found invalid env-var-name edits plus unrelated `.secrets.baseline` churn.

## Best Possible Solution

Keep the PR open and ask for a refresh against current provider-owned web search runtimes. Update the missing-key payloads in the relevant bundled web-search providers, preserve valid env var names and current config paths, avoid baseline/generated churn, and add focused message coverage where practical. Include newer API-key-backed search providers such as MiniMax if the goal is consistent UX across `web_search`.
