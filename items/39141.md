## Summary

Keep #39141 open. Current main still lacks the prerequisite `sessions.activity` gateway API / WS activity statechange surface and does not implement the requested opt-in session activity watchdog/alert engine. Existing channel health checks, task audits, diagnostic `session.stuck` logging, heartbeat, and CLI no-output watchdogs are adjacent but do not satisfy the session-scoped activity, waiting-state, dedupe/cooldown, delivery/webhook, and optional summary requirements.

## Best Possible Solution

Keep #39141 open until #39127 or an equivalent session-scoped activity API lands. Then implement or explicitly expose a plugin-accessible opt-in watchdog that consumes that state, supports no-token structured alerts, configurable thresholds, dedupe/cooldowns, scoped operator/channel/webhook delivery, optional tokenized summaries, and focused docs/tests.
