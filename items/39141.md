## Summary

Keep #39141 open. Current main still does not provide the prerequisite session-scoped activity API or WebSocket statechange event from #39127, and it does not implement the optional gateway watchdog/alert engine requested here. Existing channel health, cron failure alerts, CLI no-output watchdogs, and subagent orphan recovery are adjacent but scoped to different surfaces.

## Best Possible Solution

Keep #39141 open until #39127, or an equivalent session-scoped activity/attention API, lands. Then implement the opt-in gateway watchdog in core or expose a plugin-accessible watchdog surface that consumes reliable session activity state, supports no-token structured alerts, configurable thresholds, dedupe/cooldowns, scoped operator/channel/webhook delivery, optional tokenized summaries, docs, and focused tests. Reuse channel health, cron failure alert, and subagent recovery pieces only where they fit; they are not substitutes for session-scoped activity state.
