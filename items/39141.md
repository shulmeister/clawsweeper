## Summary

Keep #39141 open. Current main does not implement the requested optional gateway-side session activity watchdog and alerts, and the issue's prerequisite per-session activity API (#39127) is still open in the provided GitHub context. Existing code has channel-level busy/stale handling, cron failure alerts, heartbeat alerts, plugin lifecycle hooks, and session rows with subagent status/timing, but not the requested session-scoped activity state, attention state, WS statechange event, deduped alert engine, or session watchdog configuration.

## Best Possible Solution

Keep #39141 open until #39127 lands or maintainers make a product-scope decision. The practical path is to implement the per-session activity API first, then add an opt-in watchdog that consumes that state, supports no-token alerts, dedupe/cooldown thresholds, scoped delivery/webhooks, optional tokenized summaries, and focused tests/docs. If the stable session activity API becomes plugin-accessible, the alert delivery and policy layer could later move to a ClawHub plugin without losing the core tracking item.
