## Summary

Keep #70876 open. Current main still appears to have the reported `/tools` inventory gap: the slash command renders only `resolveEffectiveToolInventory()`, that inventory only models core/plugin/channel sources, while the actual embedded run path separately materializes and merges configured MCP server tools. The PR is not obsolete, but it should be cleaned up before merge because the provided latest diff still includes unrelated `install-package-dir` changes and a test/implementation mismatch flagged in review context.

## Best Possible Solution

Keep the PR open or replace it with a focused maintainer patch that adds MCP server tools to `/tools` using the existing session runtime only, passes the real `sessionId`, preserves `reservedToolNames`, filters MCP tools through `applyFinalEffectiveToolPolicy()`, extends inventory/status tests for an MCP group, and drops or splits the unrelated `install-package-dir` peer-dependency changes.
