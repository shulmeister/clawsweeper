## Summary

Current main still omits MCP server tools from the /tools inventory even though the embedded run path materializes and exposes them. This PR targets a real gap, but it should stay open for a focused revision because the branch includes unrelated installer changes and the inventory fix needs to match MCP runtime lifecycle, naming, and final tool-policy behavior.

## Best Possible Solution

Keep this PR open, but request a focused revision or supersede it with a narrow PR for the linked /tools MCP inventory bug. The fix should drop unrelated installer changes, add an MCP-aware inventory path that is lifecycle-safe, materialize tools with the same reserved-name rules as the run path, apply the same final bundled-tool policy, render a clear MCP servers group, and add compact plus verbose regression tests.
