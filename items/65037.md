## Summary

Keep this PR open. Current main still does not implement the PR's central durable pending-final-delivery payload state: subagent retry cleanup still rebuilds announce attempts from the live run record, and the run record type has no `pendingFinalDelivery` fields. The PR is concrete, reviewed, and overlaps active durable completion-delivery work in #67777/#68464, so it should be reviewed, rebased, or consolidated rather than cleanup-closed.

## Best Possible Solution

Keep this PR open for maintainer review or consolidation into the active durable subagent completion delivery work. The likely product path is to add typed durable pending-final-delivery payload state to the subagent run record, snapshot it at the first failed final-delivery boundary, reuse it across restart/retry while allowing safe live outcome/frozen-text refreshes, and clear it on success, give-up, and steer replacement. Split or drop unrelated auto-reply abort queue changes if the rebased branch no longer needs them.
