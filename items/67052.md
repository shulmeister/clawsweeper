## Summary

Keep open. Current main mitigates stale TUI streaming with a 30s client-side watchdog, but it still does not implement the requested prompt transition after the last visible content token. The footer remains driven by chat `delta`/`final` and lifecycle events, so delayed lifecycle/chat-final delivery can still show `streaming • Xm Ys` until `final` arrives or the watchdog fires.

## Best Possible Solution

Keep #67052 open until TUI streaming status is decoupled from full lifecycle/chat-final completion. A good fix would either add a lightweight content-complete/finalizing signal from the run pipeline or have the TUI transition from `streaming` to a distinct `finalizing`/idle state based on the end of visible deltas, with tests preserving concurrent-run and delayed-final behavior.
