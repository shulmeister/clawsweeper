## Summary

Keep open. Current `main` has narrow embedded-runner cleanup hardening, but it does not implement the requested cross-cutting async lifecycle standardization or a shared shutdown/resource manager across the cited subsystems.

## Evidence

- **No shared lifecycle manager on main:** A repo-wide search found no `ResourceManager` or equivalent shared shutdown abstraction, which is the core request in #49804.
  - command: `rg -n "ResourceManager|shutdown protocol|drain gate|cancellation token" src extensions docs test VISION.md`
  - sha: [320d52a23e41](https://github.com/openclaw/openclaw/commit/320d52a23e41a326287826219a20f1f29f334554)
