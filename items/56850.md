## Summary

Current `main` still implements cron as fixed `at`/`every`/`cron` schedules only. The request is recent and coherent, and the repo docs position flexible periodic checking under heartbeat/standing orders rather than a conditional next-run scheduler, so there is not strong evidence to auto-close this item.

## Evidence
