## Summary

Keep open. Current `main` still allows hook-injected bootstrap files to accumulate because the per-session bootstrap snapshot is cached by reference, exposed to hooks as a mutable array, and then used without any path-based deduplication before prompt assembly.

## Evidence

- **Per-session bootstrap snapshot is cached by reference:** `getOrLoadBootstrapFiles()` returns the existing cached `WorkspaceBootstrapFile[]` directly and stores the initial array directly in the cache; there is no cloning on read or write, so later hook mutations persist for that session.
  - file: [src/agents/bootstrap-cache.ts:3](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/bootstrap-cache.ts#L3)
  - command: `nl -ba src/agents/bootstrap-cache.ts | sed -n '1,120p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Hooks receive and can mutate the same bootstrap array:** `applyBootstrapHookOverrides()` passes `params.files` into `context.bootstrapFiles` and returns the mutated `event.context.bootstrapFiles` afterward. That means a hook calling `context.bootstrapFiles.push(...)` mutates the shared array in place.
  - file: [src/agents/bootstrap-hooks.ts:19](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/bootstrap-hooks.ts#L19)

- **Docs explicitly describe bootstrapFiles as mutable:** The hook documentation says `agent:bootstrap` exposes `context.bootstrapFiles` as a mutable array, so push-style hook implementations are part of the supported contract.
  - file: [docs/automation/hooks.md:123](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/automation/hooks.md#L123)
