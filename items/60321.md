## Summary

Current main preserves existing Windows Task Scheduler settings by updating an existing OpenClaw Gateway task with `schtasks /Change /TR` before falling back to `/Create /F`, which addresses the reported reset-after-start/update workflow. The fix landed in PR #59335 and shipped in v2026.4.5.
