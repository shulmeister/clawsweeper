## Summary

Keep open. Current main still has the threshold-only chat auto-follow behavior that this PR targets, with no manual-scroll release lock or wheel-intent path wired into the chat view. The related #70105 is debug event-log work, not a canonical replacement for this chat fix.

## Best Possible Solution

Keep the PR open for maintainer review, or land an equivalent focused fix on main: add explicit chat manual-scroll release/reacquire state, wire wheel-intent handling through the chat view, preserve nested scrollable behavior, and keep the regression tests in `ui/src/ui/app-scroll.test.ts`. Treat #70105 as separate debug event-log work, not a replacement for this chat PR.
