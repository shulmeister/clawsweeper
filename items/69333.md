## Summary

Close #69333 as ClawHub/plugin-scope. The remaining Hubitat installed-app discovery workflow is a service-specific integration that can be shipped as a Hubitat skill/native plugin/MCP bundle using current plugin surfaces, while current main already has generic browser readiness/status/doctor paths and clearer browser-unavailable diagnostics.

## Best Possible Solution

Keep OpenClaw core focused on the generic browser readiness/diagnostic surfaces already present on main. Move the Hubitat-specific installed-app discovery guidance into a ClawHub bundle/native plugin/skill that documents Maker API first and rendered-browser fallback second, and open a new core issue only if implementing that plugin exposes a concrete missing browser or plugin API.
