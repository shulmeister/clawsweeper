## Summary

Keep #60275 open. Current main has useful heartbeat session routing, but not native per-session heartbeat entries: config exposes one heartbeat object per default/agent, the scheduler tracks one heartbeat state per agent, and operator surfaces expose only agent-level heartbeat status plus one global last-heartbeat event. The requested capability for multiple persisted session-bound heartbeat schedules under one agent remains meaningfully missing.

## Evidence
