## Summary

Current main still has the reported CLI text mismatch: `openclaw channels status` registers only `--probe`, `--timeout`, and `--json`, while its help and both status footers still refer to bare `status --deep`. The public channels docs now say `openclaw status --deep`, and #71887 appears to be the active fix, but it is still open/unmerged, so #69341 should remain open.

## Best Possible Solution

Keep #69341 open and land/review #71887 or an equivalent patch that changes `src/cli/channels-cli.ts`, `src/commands/channels/status.ts`, and `src/commands/channels/status-config-format.ts` to name `openclaw status --deep` or remove the guidance, with regression coverage for the status footer/help text.
