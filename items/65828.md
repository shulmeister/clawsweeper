## Summary

Keep this PR open. Current main still has the MiniMax web-search metadata/onboarding gap: runtime can read MINIMAX_API_KEY, but provider metadata, contract metadata, setup visibility, tests, and docs have not been aligned. The PR addresses real remaining work, but the existing review concern about false-positive auto-detection for generic MiniMax API keys still needs author or maintainer resolution before merge.

## Best Possible Solution

Keep this PR open for maintainer review or author revision. The best path is to preserve the useful metadata/docs/onboarding alignment and add focused tests for MiniMax Search setup, while resolving the auto-detection tradeoff explicitly, for example by separating explicit/configured MINIMAX_API_KEY support from blind auto-detection, adding a capability check, changing precedence, or getting maintainer product approval for the Token Plan behavior.
