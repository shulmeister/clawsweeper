## Summary

Keep open. Current main still has the MiniMax web-search metadata gap: the runtime can read MINIMAX_API_KEY, but the provider metadata, contract artifact, tests, and docs still treat it only as a fallback and do not expose MiniMax Search in onboarding. This PR addresses real remaining work, but its current approach has an unresolved false-positive auto-detection concern that needs author/maintainer resolution rather than automatic closure.

## Best Possible Solution

Keep this PR open and ask for a targeted revision: keep MiniMax Search visible in onboarding, align docs/contract/runtime metadata, and add tests for MINIMAX_API_KEY behavior, but address the auto-detection false-positive risk before merge. A good fix would separate explicit/configured MiniMax API key support from blind auto-detection, add a capability check or safer precedence if feasible, or otherwise require maintainer product judgment on the Token Plan tradeoff.
