## Summary

Keep open. Current `main` still has the metadata/runtime mismatch the PR is trying to fix, so this is not implemented already. But the PR also has a substantive unresolved regression risk from review: OpenClaw auto-detect treats listed env vars as sufficient credentials, so adding bare `MINIMAX_API_KEY` to MiniMax Search metadata would auto-select MiniMax Search for users who have a general MiniMax model key but no Token Plan search entitlement.

## Risks / Open Questions

- Leaving `main` as-is preserves the current UX bug: MiniMax Search is still hidden from setup and not auto-detected from `MINIMAX_API_KEY`.
- Merging the PR as written risks routing web search to MiniMax based on a generic `MINIMAX_API_KEY` alone, which can turn a previously working provider selection into runtime MiniMax search failures for users without Token Plan entitlement.
