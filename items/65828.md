## Summary

Keep open. Current main still has the MiniMax web-search metadata mismatch the PR targets: runtime search can read `MINIMAX_API_KEY`, but provider metadata used by setup and auto-detection still advertises only the Coding Plan env vars and lacks `onboardingScopes`. The PR is a focused bundled-plugin fix, but the existing review concern about false-positive auto-detection for non-Token-Plan MiniMax keys still needs maintainer/author resolution before landing.

## Best Possible Solution

Keep the PR open. The best path is to update the MiniMax bundled plugin metadata and docs so setup and auto-detection can see MiniMax Search, while resolving the review concern by either gating `MINIMAX_API_KEY` auto-detection to search-capable/Token-Plan credentials, adjusting precedence/UX, or documenting an intentional fallback with a focused regression test around metadata/runtime key parity.
