## Summary

Keep #38260 open. The report is a native SIGILL crash in the Linux npm-global webchat image path, and current main still has Linux/Node image flows that load sharp/libvips. I found input-size guards and plugin-staged dependency changes, but not source, tests, docs, release notes, or related-item evidence proving the reported invalid-opcode crash is fixed or superseded.

## Best Possible Solution

Keep the issue open. A maintainer should reproduce or collect a core/backtrace on Linux npm-global Node 22, identify whether bundled sharp/libvips, system libvips, CPU feature selection, or image optimization is the root cause, then either stage a known-safe native dependency, add a Linux-safe fallback/kill switch, or isolate native image work out of the gateway process with a regression/smoke test for webchat image flows.
