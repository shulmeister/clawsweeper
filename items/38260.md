## Summary

Keep #38260 open. Current main still has Linux/Node image paths that load the media-understanding-core sharp/libvips runtime, and I did not find source, test, docs, changelog, or related-item evidence proving the reported libvips-cpp SIGILL crash is fixed or superseded.

## Best Possible Solution

Keep the issue open. A maintainer should reproduce or collect a core/backtrace on Linux npm-global Node 22, identify whether bundled sharp/libvips, system libvips, CPU feature selection, or image optimization is the root cause, then ship a known-safe dependency/runtime fallback, a kill switch, or isolate native image processing out of the gateway process with a WebChat image-flow regression smoke.
