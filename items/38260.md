## Summary

Keep #38260 open. The report is a native Gateway SIGILL/core-dump during WebChat image flow, and current main still has sharp/libvips-backed image processing reachable from the WebChat managed-image path. I found image-size guard work, but not source, test, history, or release evidence that the reported invalid-opcode class is fixed or no longer reachable.

## Best Possible Solution

Keep the issue open and ask for/collect a focused repro: affected CPU/features, installer/runtime env, the triggering image or safe equivalent, core dump backtrace, and Gateway logs. A likely fix belongs around src/media/image-ops.ts and extensions/media-understanding-core/image-ops.ts, such as a Linux-safe backend fallback, sharp/libvips CPU/runtime guard, or documented kill switch, plus a regression test that exercises the WebChat managed-image path without letting a native crash take down the Gateway.
