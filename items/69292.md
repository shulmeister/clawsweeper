## Summary

Keep open. Current main still does not implement this PR's proposed doctor skip for installed manifest-channel plugins, and the related PRs in the provided context are closed/unmerged rather than superseding it. The PR is a plausible core doctor/activation fix, but it should be revised before merge because the workspace-installed branch conflicts with the current trust model: workspace plugins are disabled unless explicitly allowed or entry-enabled.

## Best Possible Solution

Keep the PR open, but ask for a revision that narrows or proves the skip against the current activation contract. The likely path is to handle globally installed plugins only when they truly auto-load under the active trust policy, keep workspace plugins explicitly enabled or allowlisted, and add regression coverage for restrictive `plugins.allow`, global, workspace, and bundled cases before maintainer merge review.
