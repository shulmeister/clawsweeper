## Summary

Keep open. Current main implemented the linked Feishu/bundled-channel duplicate-warning path, but it has not implemented this PR's broader global/workspace external-channel behavior. The PR also has a real review concern: treating workspace plugins as already enabled conflicts with current activation policy, where workspace plugins stay disabled unless explicitly allowed or enabled.

## Best Possible Solution

Keep the PR open for maintainer review, but require revision before merge: narrow or prove the global-installed duplicate case with loader/doctor coverage, do not treat workspace-origin plugins as already enabled unless the trust/allowlist contract is preserved, and decide explicitly whether external channel plugins should be activated through plugins.entries or a new manifest-channel activation path.
