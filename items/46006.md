## Summary

Current main no longer has the reported hardcoded 15-second embedded/subagent LLM timeout behavior. Spawned subagent runs now pass an explicit run timeout through the gateway agent path, default subagent run timeout is configurable and otherwise no-timeout, and the embedded LLM idle watchdog is configurable/falls back through the current timeout helpers instead of a fixed 15 seconds.

## Evidence
