## Summary

Keep open. Current `main` and the latest release still implement `plugins.allow_phantom_entries` using only installed plugin dirs plus `listChannelPlugins()`, which reflects loaded channel plugins rather than the full bundled-plugin inventory. The code and docs still support the reported false positive for bundled plugins such as `openai`, `browser`, `acpx`, and `memory-core`, and the current remediation text conflicts with docs that say `plugins.allow` can gate bundled plugin loading.

## Evidence

- **Audit still keys phantom detection off loaded channel plugins:** `collectPluginsTrustFindings` builds `bundledPluginIds` from `listChannelPlugins()` and uses that set to suppress `plugins.allow_phantom_entries`. It does not consult the bundled plugin manifest inventory, so bundled non-channel plugins are still eligible to be reported as phantom.
