## Summary

Keep open. Current main still contains the reported audit inventory mismatch: `plugins.allow_phantom_entries` compares allowlist entries against installed extension dirs plus loaded channel plugins, not the full bundled plugin manifest inventory. The issue also has explicit maintainer direction to keep #68154 as the canonical tracker while #67925/#68786 are duplicates.

## Best Possible Solution

Keep #68154 open. Fix `collectPluginsTrustFindings` to validate `plugins.allow` against installed plugins plus the full bundled manifest inventory, normalize aliases consistently, preserve true phantom-entry warnings, soften the remediation text, and add regression coverage for bundled non-channel IDs such as `browser`, `openai`, `acpx`, and `memory-core`. Keep #71540 separate unless the reload/restore symptom is proven to be the same root cause.
