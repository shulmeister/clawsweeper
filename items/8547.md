## Summary

Current main still lacks user-configurable Web UI font size or font scale in browser-local settings, the Appearance UI, gateway `ui` schema/types, chat CSS, and public docs. The related #3027 is closed and not a canonical open tracker; the prior #8547 review also kept this open.

## Best Possible Solution

Keep #8547 open for maintainer product/design decision and implementation in core Control UI. A concrete path is to add a browser-local text-size/font-scale setting, expose it in the Appearance panel, apply it through CSS custom properties for chat body, message text, tool cards, and compose input, and add gateway schema/types/docs/tests only if a server-side `ui` config knob is intentionally part of the product contract. Treat #3027 as historical context because it is closed.
