## Summary

Keep #8547 open. Current main still lacks a user-configurable Web UI font size/font scale in browser-local settings, the Appearance UI, gateway `ui` config schema/types, chat CSS, or public configuration docs. The related #3027 is closed and does not provide an open canonical replacement; the latest issue discussion also includes a maintainer Codex review explicitly keeping this item open.

## Best Possible Solution

Keep #8547 open for core Control UI accessibility/customization work. A concrete path is to add a browser-local text-size/font-scale setting in Appearance, apply it via CSS custom properties across chat messages, tool cards, sidebars, and compose inputs, and add gateway schema/types/docs/tests only if maintainers want server-side `ui` config as a public contract.
