## Summary

Keep #8547 open. Current main still lacks a user-configurable Control UI/WebChat font-size or font-scale setting in browser-local settings, Appearance/Quick Settings, gateway `ui` config, docs, CSS variables, or focused tests. The latest issue discussion already kept this open, and the related #3027 tracker is closed rather than an active canonical follow-up.

## Best Possible Solution

Implement a bounded Control UI text-size or font-scale setting in `UiSettings`, expose it in Appearance and Quick Settings, apply it through CSS custom properties across WebChat and surrounding Control UI surfaces, and add focused storage/rendering tests plus public docs. Extend the gateway `ui` schema/types/docs only if maintainers want server-side font sizing as a public config contract.
