## Summary

Keep #8547 open. Current main still does not provide user-configurable Web UI font size or font scale in browser-local settings, the Appearance UI, gateway `ui` config schema/types, chat CSS variables, or public docs. The related #3027 is closed, and the recent #8547 Codex review comment also kept this open rather than identifying a canonical shipped fix.

## Best Possible Solution

Keep #8547 open for core Control UI product/design work. A concrete path is to add a browser-local text-size/font-scale setting, expose it in Appearance, apply it through CSS custom properties across chat body, messages, tool cards, and compose input, and add gateway schema/types/docs/tests only if maintainers want a server-side `ui` config knob as part of the public contract. Treat #3027 as historical context because it is closed, not canonical remaining work.
