## Summary

Current main still lacks a user-configurable Control UI/WebChat font size or font scale. Browser-local settings, Appearance/Quick Settings UI, gateway `ui` config schema/types/docs, and chat CSS all still expose fixed theme/roundness/identity settings rather than a text-size control. The related #3027 is closed and does not supersede the remaining work; the latest issue discussion also explicitly kept #8547 open.

## Best Possible Solution

Keep #8547 open for core Control UI accessibility/customization. A practical implementation is to add a bounded text-size/font-scale setting to browser-local `UiSettings`, expose it in Appearance/Quick Settings, apply it through CSS custom properties across chat messages, sidebars, tool cards, and compose inputs, and extend the gateway `ui` schema/types/docs only if maintainers want server-side font sizing as a public config contract.
