## Summary

Keep #29707 open. Current main still treats direct Amazon Nova Bedrock model IDs as non-Anthropic Bedrock models: explicit cacheRetention is dropped before stream options, the bundled amazon-bedrock plugin wraps direct Nova streams with cacheRetention: "none", and tests/docs still lock in that no-cache behavior. Official Bedrock prompt-caching docs now list Amazon Nova models as prompt-cache-capable and describe Converse cachePoint usage, so this remains a real bundled Bedrock plugin/runtime gap rather than an obsolete feature request. citeturn0search0

## Best Possible Solution

Keep #29707 open for the bundled amazon-bedrock plugin. The best fix is to add explicit Nova-family cache eligibility for supported direct Bedrock model IDs, preserve/pass explicit cacheRetention for those models, stop forcing Nova to cacheRetention: "none", inject or preserve Bedrock Converse cachePoint metadata according to AWS's current Nova contract, keep unsupported non-Anthropic Bedrock models protected, and update Bedrock regression tests plus https://docs.openclaw.ai/reference/prompt-caching.
