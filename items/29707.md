## Summary

Keep #29707 open. Current main still does not provide native prompt caching for Amazon Nova on Bedrock: known Nova model IDs are treated as non-Anthropic Bedrock models, explicit cacheRetention is ignored before the stream call, and the Amazon Bedrock plugin wrapper forces cacheRetention: "none" at runtime. The prior #20815/#20866/#22303 path fixed Nova cachePoint crashes by disabling caching for non-Anthropic Bedrock models, not by adding Nova-native cache support.

## Best Possible Solution

Keep the issue open for the Amazon Bedrock provider owner path. The likely fix belongs in the bundled amazon-bedrock provider and shared cache-retention eligibility code: verify the current Bedrock Nova prompt-caching contract, add Nova model-family eligibility, avoid Anthropic-only cachePoint injection when Nova requires a different payload shape, preserve the #20815 crash fix with regression tests, and update docs once the supported Nova behavior is proven.
