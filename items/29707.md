## Summary

Keep #29707 open. Current main still does not implement native Amazon Nova prompt caching on Bedrock: known Nova model IDs are classified as non-Anthropic Bedrock models, the Bedrock provider wrapper forces cacheRetention to "none", the shared resolver drops explicit cacheRetention for Nova, and tests/docs still lock in that no-cache behavior. The related #20815 path fixed Nova cachePoint crashes by disabling non-Anthropic Bedrock caching, not by adding a Nova-native cache payload path.

## Best Possible Solution

Keep the issue open for the Amazon Bedrock provider owner path. The likely fix belongs in the bundled amazon-bedrock provider plus shared cache-retention eligibility code: verify the current AWS Bedrock Nova prompt-caching contract, add Nova model-family eligibility, pass through or translate cacheRetention without Anthropic-only cachePoint injection, preserve the #20815 crash fix with regression tests, and update the prompt-caching docs when supported behavior is proven.
