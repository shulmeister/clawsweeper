## Summary

Keep #69961 open. Current main still writes session-memory transcript text verbatim, so the bug targeted by this PR and linked issue #69943 is not fixed on main. The related discussion shows real affected users and a confirmed root cause. There is overlapping open work in #71257, but it is not merged or clearly canonical enough to make #69961 obsolete.

## Best Possible Solution

Keep #69961 open as a plausible focused fix for the unresolved bundled session-memory poisoning bug. Maintainers should choose or consolidate one canonical implementation, address the current review edge cases, use existing shared text-sanitizer helpers where practical, and land regression coverage for both transcript extraction and handler persistence so poisoned memory cannot be re-injected across `/new` or `/reset`.
