## Summary

PR #70888 should stay open. Current main still has the reported no-cfg depth lookup bug: persisted explicit requester session keys cannot reach the default on-disk session store, so they fall back to segment counting and resolve as depth 0. The PR appears to address a real agents/announce-delivery behavior gap, though it should be narrowed or cleaned up before landing because the current branch also contains unrelated drift/test changes and one bot-raised test hygiene issue.

## Best Possible Solution

Keep the PR open, but ask for a narrow rebase/cherry-pick that keeps the src/agents/subagent-depth.ts fix plus focused regression tests for explicit acct-suffixed requester sessions and announce classification, while dropping unrelated drift files and addressing the test temp-directory cleanup comment.
