## Summary

Keep #13616 open. Current main has a partial native backup surface for local archive creation and verification, but the requested first-party restore/import workflow, selective cron/session restore or merge, cloud backup listing/restore, encryption, and disaster-recovery flow remain missing. The latest maintainer comment in the provided issue context also explicitly kept this umbrella open for those gaps.

## Best Possible Solution

Keep #13616 open as the native umbrella for first-party backup restore/import and disaster-recovery behavior. The best product path is to extend the existing `openclaw backup create` and `openclaw backup verify` implementation with a safe restore/import surface, component-aware cron and session semantics, cloud/encryption decisions if maintainers want them in core, and matching docs/tests, or split narrower remaining pieces into canonical follow-up issues before closing this umbrella.
