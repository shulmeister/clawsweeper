## Summary

Keep #13616 open. Current main partially addresses the backup side with local `openclaw backup create`/`verify`, shipped in 2026.3.8, but it does not implement the requested native import/restore workflow, selective cron/session restore, cloud storage, encryption, or scheduled backup commands. The linked ClawHub/community options are useful and should be acknowledged, but the discussion itself notes remaining native/session-history gaps, so this is not a high-confidence auto-close.
