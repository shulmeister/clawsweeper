## Summary

Keep #13616 open. Current main has a partial native backup surface (`openclaw backup create` and `openclaw backup verify`) and useful community/plugin workarounds are linked in the issue, but the requested first-party restore/import workflow, selective cron/session restore or merge, cloud storage, encryption, scheduled backup commands, and disaster-recovery flow are not implemented on main. The latest maintainer comment also explicitly keeps this item open for those remaining gaps.

## Best Possible Solution

Keep #13616 open as the native umbrella for first-party backup restore/import and disaster-recovery behavior. The best product path is to add a safe restore/import surface beside the existing create/verify commands, with component-aware cron and session semantics, or split any narrower remaining pieces into canonical issues before closing this umbrella.
