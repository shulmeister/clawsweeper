## Summary

Keep this PR open. Current main still has the mutable module-global `workspacePackagePaths` and `collectWorkspacePackagePaths()` still appends extension package paths into that shared array before `buildMissingPackages()` consumes it, so the PR is not implemented or superseded on main. The PR is a narrow scripts fix for a real idempotency hazard; it should be reviewed, likely with a small regression test added.

## Best Possible Solution

Keep the PR open and review it as a focused scripts bug fix. The best path is to land a version that makes `collectWorkspacePackagePaths()` return a fresh deterministic package-path list, threads that list into `buildMissingPackages()`, removes or justifies the redundant defensive sort, and adds a small `test/scripts/audit-seams.test.ts` regression that calls `main()` twice in one Node process and asserts stable, non-duplicated `missingPackages` output.
