## Summary

Keep open. Current main still does not extract array-shaped tool-result text in the Control UI tool-card path, and this PR remains a plausible narrow fix for the reported Web UI "No output" behavior. The related duplicate PRs do not make this one obsolete because this PR handles more shapes and incorporates the review feedback to concatenate fragments.

## Best Possible Solution

Keep this PR open or use it as the canonical basis for the fix. The best maintainer path is to add a small `extractToolText` helper that joins text fragments from string fields and array content blocks, add regression coverage in `ui/src/ui/chat/tool-cards.node.test.ts`, then close the weaker duplicate PRs once the behavior is merged.
