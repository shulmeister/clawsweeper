## Summary

Keep open. Current main at c7a0d9b1889ca09ab5bfd84cda39a8d50d3d5768 still does not extract array-shaped `text` or `content` from embedded `toolresult` / `tool_result` blocks in the Control UI tool-card path. The linked issue #38223 was closed based on the standalone tool-result fallback, but this PR targets the narrower inline block path, which remains missing. Related PR #38228 was closed as superseded by this broader PR, and there is no merged current-main implementation to close against.

## Best Possible Solution

Keep #38876 open as the canonical fix path, or land an equivalent maintainer patch in `ui/src/ui/chat/tool-cards.ts` that extracts text from array-shaped `text` and `content` fields for embedded `toolresult` / `tool_result` blocks. Add regression tests in `ui/src/ui/chat/tool-cards.node.test.ts` for array `content`, array `text`, typed text blocks, non-string fragments, and multi-fragment joining before merge.
