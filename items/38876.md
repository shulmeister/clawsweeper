## Summary

Keep open. Current main at 46b9044c3f9d8519a56024f032b4b6e0b42aaf2b still drops array-shaped tool-result text in the Web UI tool-card path, latest v2026.4.24 has the same behavior, and PR #38876 remains a plausible narrow fix for the linked #38223 bug. Related open PR #38228 tracks the same bug but is less complete, so #38876 should not be closed as obsolete or duplicate without maintainer choice of canonical implementation.

## Best Possible Solution

Keep this PR open and use it, or an equivalent maintainer patch, to implement array-shaped tool-result extraction in `ui/src/ui/chat/tool-cards.ts` with regression tests for `content: [{ text: ... }]`, `content: [{ type: "text", text: ... }]`, array `text`, and multi-fragment joining. After choosing a canonical fix, close the weaker duplicate PR with a link to the selected implementation.
