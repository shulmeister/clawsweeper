## Summary

Keep PR #38876 open. Current main still does not extract text from array-shaped `text` or `content` fields for embedded Web UI tool-result cards, so the reported `No output` behavior remains possible. The closed related PR #38228 was not merged and does not supersede this PR.

## Best Possible Solution

Keep #38876 open as the focused fix path, or land an equivalent maintainer patch in `ui/src/ui/chat/tool-cards.ts` that extracts and joins text from array-shaped `text` and `content` fields for embedded tool-result blocks. Add regression coverage in `ui/src/ui/chat/tool-cards.node.test.ts` for array `content`, array `text`, typed text blocks, ignored non-string fragments, and multi-fragment joining.
