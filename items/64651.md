## Summary

Keep open. Current main still has Control UI chat and form input CSS below the 16px iOS Safari focus threshold, and the related PR is still open rather than merged.

## Best Possible Solution

Keep this issue open and either review/land PR #64673 after rebasing it onto current main, or replace it with a narrower maintainer patch that guarantees all focusable Control UI text inputs compute to at least 16px on iOS/touch-primary devices. The patch should cover the chat composer, agent chat textarea, generic `.field` controls, and config textareas, with a small CSS regression test so future mobile overrides do not reintroduce sub-16px input sizes.
