## Summary

Keep open. Current main still contains the cross-agent account rewrite that #70607 tries to change, so the PR is not implemented or obsolete. The PR also has unresolved correctness feedback because the one-character guard flip would conflict with existing same-agent account-preservation behavior, so this needs maintainer review rather than conservative cleanup.

## Best Possible Solution

Keep #70607 open, but require a revised maintainer-reviewed fix. The best path is targeted regression coverage for Agent A spawning Agent B where B has a matching route binding, plus coverage that same-agent spawns keep the active inbound account; then adjust requester-origin routing so parent completion delivery preserves the requester session/account without breaking target-agent child startup or thread-bound session routing.
