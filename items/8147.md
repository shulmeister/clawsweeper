## Summary

Keep #8147 open. Current main still hardcodes plain Enter-to-send in the browser Control UI/WebChat composer, exposes no Cmd/Ctrl+Enter send mode or persisted send-shortcut preference, and documents/configures only WebChat history truncation. Related issue context also treats #8147 as the canonical remaining tracker rather than a duplicate to close.

## Best Possible Solution

Keep #8147 open as the canonical WebChat/Control UI send-hotkey tracker. A complete fix should add a typed send-shortcut preference or product decision, wire it through the browser composer and any native WebChat composer that shares the behavior, persist/configure it if intended, update visible composer hints and WebChat docs/config schema, and add focused tests for plain Enter, Shift+Enter, IME composition, and Cmd/Ctrl+Enter.
