Keep #8147 open. Current main still hardcodes Control UI/WebChat send-on-plain-Enter behavior, only exempts Shift+Enter and IME composition, and has no persisted UI setting or gateway WebChat config option for Cmd/Ctrl+Enter sending. The provided related-item context also makes #8147 the canonical tracker: #39821 was closed as a duplicate of this issue, and PR #38363 was closed unmerged.

## Best Possible Solution

Keep #8147 open as the canonical tracker. A complete fix should update the Control UI/WebChat composer key handling, add and persist a typed send-shortcut preference or deliberately change the default to Cmd/Ctrl+Enter, update placeholder/button/help copy, and add focused tests for plain Enter, Shift+Enter, IME composition, and Cmd/Ctrl+Enter behavior.
