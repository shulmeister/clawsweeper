## Summary

Keep #8147 open. Current main still implements plain Enter-to-send in Control UI/WebChat, with only Shift+Enter and IME-composition exemptions. There is no Cmd/Ctrl+Enter send mode, no chat shortcut prop, no persisted UI preference, and no `gateway.webchat` config/docs option. The GitHub discussion also already treats #8147 as the canonical tracker while related items are duplicate/overlapping or unmerged.

## Best Possible Solution

Keep #8147 open as the canonical tracker. A complete fix should add a typed Control UI/WebChat send-shortcut preference or explicit product decision, persist it in UI settings, update composer placeholder and send-button shortcut hints, document any new setting or config surface, and add focused tests for plain Enter, Shift+Enter, IME composition, and Cmd/Ctrl+Enter behavior.
