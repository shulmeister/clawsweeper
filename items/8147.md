## Summary

Keep #8147 open. Current main still hardcodes WebChat/Control UI chat submission to plain Enter, with Shift+Enter and IME-composition exceptions but no Cmd/Ctrl+Enter send mode or configurable shortcut. The existing maintainer-posted Codex review comment also says to keep this open, PR #38363 was closed unmerged, and related issue #39821 remains open for the same remaining shortcut behavior.

## Best Possible Solution

Keep #8147 open until maintainers either implement or explicitly decline a core WebChat/Control UI send-shortcut setting. A complete fix should update the chat composer key handling, expose/persist a typed setting or config surface, adjust placeholder/help text, and add focused coverage for plain Enter, Shift+Enter, IME composition, and Cmd/Ctrl+Enter behavior.
