## Summary

Keep open. Current `main` has already removed the old hardcoded-channel shape by routing thread-bound subagent behavior through generic plugin lifecycle hooks and per-channel thread-binding config, but it still does not implement the requested automatic `sessions_spawn` parameter injection. `sessions_spawn(mode="session")` still explicitly requires `thread: true`, and docs/system prompt guidance still tell agents to pass those params manually.

## Evidence
