## Summary

Current main still has the LiteLLM docs gap described by this issue. The manual setup docs still recommend an unpinned `pip install 'litellm[proxy]'`, and targeted searches found no v1.82.7/v1.82.8 advisory, downgrade, key-rotation, or server-audit remediation text. OpenClaw's bundled LiteLLM plugin is network/proxy based and does not vendor the PyPI package, so this remains a user-facing docs advisory task rather than a direct dependency remediation.

## Best Possible Solution

Keep this issue open and land or adapt the contributor's LiteLLM docs advisory in `docs/providers/litellm.md`, which publishes to `https://docs.openclaw.ai/providers/litellm`: replace the unpinned install example with current safe guidance, add a dated notice for the v1.82.7/v1.82.8 LiteLLM PyPI compromise, and include downgrade, key-rotation, and server-audit steps for users who ran an affected proxy.
