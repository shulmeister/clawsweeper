## Summary

Keep #66785 open. Current main has good evidence that OpenRouter is available in onboarding and the model picker is searchable/filterable, but a central reported packaged-setup problem still appears present: config-present install/setup paths can still launch onboarding. The issue is recent and labeled as a regression, so it should not be closed as stale or already implemented.

## Best Possible Solution

Keep the issue open. Fix packaged setup so a valid existing config starts the gateway/dashboard or exits cleanly instead of launching onboarding, cover that with an installer/Docker regression test using a preseeded config, and then separately verify the Umbrel model-selection surface can select OpenRouter without a viewport trap.
