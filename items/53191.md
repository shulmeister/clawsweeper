## Summary

Keep open. Current `main` does not implement native `playbooks` support in config, docs, or executable workflow surfaces, and the closest adjacent mechanisms are reusable skills/plugins plus planned-but-unshipped OpenProse pipeline syntax. The request is coherent and actionable enough that none of the conservative close reasons fit.

## Evidence

- **No native playbooks config surface on main:** A targeted search found no `playbook`/`playbooks` support under config or gateway docs, and the current skills config type only exposes `allowBundled`, `load`, `install`, `limits`, and `entries` under `skills`, with no analogous `playbooks` key.
  - file: [src/config/types.skills.ts:40](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/config/types.skills.ts#L40)
