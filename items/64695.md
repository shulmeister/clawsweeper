## Summary

Keep this issue open. The attempted fix in #64722 was closed unmerged, and current main still wires both task registries directly to the SQLite stores. Those stores call `requireNodeSqlite()`, which still throws the exact missing-`node:sqlite` error when the runtime lacks the builtin, so the reported maintenance/update path remains actionable.

## Best Possible Solution

Implement a runtime-safe storage selection for the task registry and task-flow registry: detect unavailable `node:sqlite` before default store use, fall back to a durable JSON/file store or other supported store, and add regression tests that simulate `require("node:sqlite")` throwing `ERR_UNKNOWN_BUILTIN_MODULE` through restore and maintenance/update paths. Update the task automation docs/changelog once the fallback ships.
