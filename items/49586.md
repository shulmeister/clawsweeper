## Summary

Current `main` already fixes the reported hot-reload model visibility gap. On model-related config changes, gateway hot reload now invalidates the cached model catalog, and the next catalog read regenerates `models.json` before rebuilding the catalog. Current tests cover both cache invalidation and newly visible models after hot reload, so the PR is redundant on `main` and in the latest release.
