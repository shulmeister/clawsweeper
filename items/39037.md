## Summary

Keep #39037 open. Current main still has no CTWA/referral extraction in the bundled WhatsApp plugin, even though the Baileys version it depends on exposes `ContextInfo.externalAdReply.ctwaClid` and related ad fields. The previous ClawSweeper keep-open comment remains accurate against c7a0d9b1889ca09ab5bfd84cda39a8d50d3d5768.

## Best Possible Solution

Keep the issue open and implement the feature in the bundled WhatsApp plugin. Extract CTWA attribution from Baileys `contextInfo.externalAdReply` and any Cloud-style referral shape OpenClaw receives, normalize it as untrusted structured context with fields such as `ctwaClid`, `sourceUrl`, `sourceType`, `sourceId`, and `headline/title`, pass it through the agent prompt and any hook/session metadata surface maintainers choose, and add focused tests for extraction plus propagation.
