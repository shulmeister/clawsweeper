## Summary

Keep open. Current main still does not expose Click-to-WhatsApp attribution data from WhatsApp/Baileys contextInfo. The inbound path can find contextInfo for mentions and quoted replies, but it never maps externalAdReply/ctwaClid/referral fields into WebInboundMessage, MsgContext, prompt metadata, plugin hook metadata, or session metadata.

## Best Possible Solution

Keep the issue open and implement it in the bundled WhatsApp plugin: extract CTWA ad metadata from Baileys contextInfo.externalAdReply (and any Cloud-style referral shape if present), carry it as untrusted structured context and/or a typed session metadata attribute, render it in inbound prompt/session metadata where agents can access it, and add tests covering extraction from common WhatsApp message wrappers plus prompt/session propagation.
