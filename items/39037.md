## Summary

Keep open. Current main still does not expose Click-to-WhatsApp referral or `ctwaClid` data from WhatsApp inbound messages into session metadata, agent context, or plugin hook metadata. The request is not a conservative ClawHub close because the missing data is dropped inside the bundled WhatsApp inbound normalization path before external plugins/hooks can observe it.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin: extract CTWA referral fields from the first inbound message's Baileys `contextInfo`/referral shape, preserve them as untrusted structured context for the agent, and decide whether a sanitized subset belongs in persisted session metadata for later conversion reporting. Add focused tests around the inbound extractor, monitor normalization, `buildWhatsAppInboundContext`, and session metadata behavior so future changes do not drop attribution fields again.
