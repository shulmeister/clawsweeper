## Summary

Keep #42122 open. Current main still unconditionally runs the Control UI build in the root Dockerfile image used by install-smoke, and the install-smoke workflow does not pass the proposed smoke-only skip arg. The PR is now dirty against main and needs a rebase/adaptation, but the central optimization is not implemented or clearly superseded.

## Best Possible Solution

Keep the PR open but require a rebase onto current main. The best implementation is to add a build-stage-scoped `OPENCLAW_SKIP_UI_BUILD` arg in `Dockerfile`, keep normal/release Docker builds unchanged, pass the arg only from the install-smoke Dockerfile build steps that do not exercise the Control UI, and validate the updated Docker smoke path.
