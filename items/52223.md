## Summary

Current `main` does not implement the requested task/context-aware "intelligent model routing" in OpenClaw core. The documented and implemented selection path is still static `primary` plus ordered `fallbacks`, with provider auth failover inside each candidate. There are adjacent alternatives on `main` such as per-agent/sub-agent model routing and provider-owned auto-routing (`openrouter/auto`, LiteLLM, `kilocode/kilo/auto`), but those are not the same as a built-in core router that chooses models by task type, context length, complexity, or time. The issue is recent and coherent, so there is not enough evidence to close it.
