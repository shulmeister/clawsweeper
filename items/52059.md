## Summary

Current `main` does not implement `gateway.auth.scopes`, and the WebSocket gateway path still intentionally strips scopes from device-less shared token/password sessions except for a narrow direct-local backend self-pairing carve-out. This PR is not implemented on `main`, and the reported behavior is still reflected in current tests, so there is not strong evidence to close it.
