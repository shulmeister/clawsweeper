## Summary

Keep open. Current main has partial mitigations, but it does not implement the requested configurable cron outbound payload policy. The linked PR #63650 is closed unmerged, `cron.deliveryPayloads` is absent from the config/schema surface, and the isolated cron resolver still preserves all successful deliverable payloads unless a channel plugin opts into final-assistant-text collapsing.

## Best Possible Solution

Keep this issue open and implement a scoped isolated-cron outbound payload policy, likely in the cron-owned resolver/dispatch path. Add typed config/schema/docs plus tests for default full behavior, last-only payload selection, and none/suppressed automatic delivery with no synthesized-text fallback. The implementation should account for the existing per-job `delivery.mode="none"` behavior and channel plugin final-text policy rather than replacing them accidentally.
