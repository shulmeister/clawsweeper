## Summary

Keep open. Current `main` still matches the report: announce target resolution only hydrates `accountId` from `sessions.list` for plugins that opt into `preferSessionLookupForAnnounceTarget`, and the Discord plugin still does not opt in, so Discord announce routing can still drop the session-scoped `accountId` and fall back downstream to default-account behavior.

## Evidence

- **Announce resolution short-circuits unless the channel opts into session lookup:** `resolveAnnounceTarget()` returns the parsed fallback target immediately when `plugin.meta.preferSessionLookupForAnnounceTarget` is absent, so `sessions.list` is never queried for Discord on that path.
  - file: [src/agents/tools/sessions-announce-target.ts:26](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/agents/tools/sessions-announce-target.ts#L26)
