## Summary

Keep open. Current main still lacks a durable macOS desktop launch-at-login intent or post-Sparkle-update reconciliation for the `ai.openclaw.mac` LaunchAgent, and the related fixes only cover gateway LaunchAgent repair or the narrower KeepAlive/manual-quit behavior.

## Best Possible Solution

Keep #39588 open and fix the desktop macOS launch-at-login contract directly: persist the user's intended launch-at-login preference separately from live launchd loaded status, distinguish disabled from stale/unloaded/replaced LaunchAgent states, reconcile or re-bootstrap `ai.openclaw.mac` on app startup and after Sparkle updates, and add focused tests for stale/unloaded plist state so update-time launchd drift cannot silently flip the UI to disabled.
