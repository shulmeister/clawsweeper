## Summary

Keep #39588 open. Current main still derives the macOS desktop app Launch at login toggle from live LaunchAgent state and writes directly to launchd, with no separate persisted user intent or Sparkle/post-update reconciliation for `ai.openclaw.mac`. The merged #40213 fix is related but narrower: it removed `KeepAlive` to stop relaunch-after-quit behavior, not update-time persistence or UI/launchd drift.

## Best Possible Solution

Keep #39588 open and fix the desktop macOS Launch at login contract directly: persist the user's intended launch-at-login preference separately from live launchd state, distinguish disabled from stale/unloaded/repair-needed LaunchAgent states, reconcile or re-bootstrap `ai.openclaw.mac` on app startup and after Sparkle updates, and add focused tests for stale plist, unloaded job, and UI reconciliation behavior. Treat #40213 as the separate relaunch-after-quit fix, not the canonical resolution for this issue.
