## Summary

Keep #9876 open. Current main still carries `sessionKey` to macOS exec approval prompt data, but the native `ExecApprovalsPromptPresenter.buildAccessoryView` does not render a Session row, while the Control UI does. The richer requester/channel context also remains TypeScript-side metadata only; the Swift prompt model does not decode or display `turnSource*` or sender/requester fields. The prior Codex review comment in the provided issue context kept this open for the same gap, and current main still matches that finding.

## Best Possible Solution

Keep #9876 open. The narrow fix should add a trimmed, non-empty `Session` detail row in `ExecApprovalsPromptPresenter.buildAccessoryView`. The fuller fix should extend the Swift prompt request model to decode safe gateway turn-source/requester metadata, render readable Channel/Sender/Thread context where available, and add focused macOS coverage for the socket, gateway, and node-host prompt paths.
