## Summary

Keep #9876 open. Current main still has the macOS exec approval request carrying `sessionKey`, but `ExecApprovalsPromptPresenter.buildAccessoryView` does not render it, and the Swift prompt model still does not decode requester/channel fields even though the gateway exec approval payload carries turn-source metadata. The existing maintainer-posted Codex review comment also explicitly kept this issue open for the same current product gap.

## Best Possible Solution

Keep #9876 open. The narrow fix belongs in `ExecApprovalsPromptPresenter.buildAccessoryView`: add a trimmed, non-empty `Session` detail row for `request.sessionKey` and cover both socket and gateway prompt paths. The fuller fix should extend the Swift prompt request model to decode safe requester metadata already present in gateway exec approval payloads, derive user-readable labels for channel/requester context, and add focused macOS prompt/model tests.
