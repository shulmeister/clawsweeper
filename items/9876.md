## Summary

Keep #9876 open. Current main still propagates `sessionKey` to macOS exec approval prompt requests, but the macOS popup does not render a Session row and the Swift prompt model still drops requester/channel turn-source metadata. The Control UI still renders Session, so the parity gap remains.

## Best Possible Solution

Keep #9876 open. The narrow fix belongs in `apps/macos/Sources/OpenClaw/ExecApprovalsSocket.swift`: trim `request.sessionKey` and render a non-empty `Session` detail row in `ExecApprovalsPromptPresenter.buildAccessoryView`. The fuller fix should extend the native prompt request model to decode safe turn-source/requester metadata already carried by gateway approval requests and display readable Channel/Sender/Thread context where available, with focused macOS coverage for gateway and node prompt paths.
