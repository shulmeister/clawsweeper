## Summary

Keep #9876 open. Current main still carries `sessionKey` in the macOS exec approval request model, but the AppKit approval popup does not render it, and the Swift request model still has no requester/channel fields even though the gateway exec approval payload already carries turn-source metadata.

## Best Possible Solution

Keep #9876 open. The narrow fix should add a trimmed non-empty Session row to `ExecApprovalsPromptPresenter.buildAccessoryView` and cover both socket and gateway prompt paths. The fuller fix should extend the Swift prompt request model to decode safe requester context already present in the gateway payload, such as `turnSourceChannel`, `turnSourceTo`, `turnSourceAccountId`, and `turnSourceThreadId`, derive user-readable labels, and add focused macOS prompt/model tests.
