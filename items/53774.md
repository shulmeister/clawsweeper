## Summary

Current main still does not implement configurable TUI footer fields for cost, balance, or rate-limit status. There is partial usage/cost visibility through per-response `/usage full`, `/status`, and `usage-cost`, but that is separate from the persistent TUI footer requested here. The issue should stay open, especially because an open README-only PR claims to close it and a separate open API proposal only covers part of the needed plugin seam.

## Best Possible Solution

Keep this issue open. The best path is to close or rewrite #72483, then either implement a real TUI footer configuration surface in `src/tui/tui.ts`, `TuiOptions`, CLI/docs, and tests, or deliberately split it into a canonical footer-template issue plus #64086/#72557 for the plugin status-provider API. Existing `/usage full`, `/status`, and `usage-cost` behavior can be reused as data sources for estimated cost, but balance and cooldown/rate-limit fields still need a supported core or plugin-facing source.
