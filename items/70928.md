## Summary

Current main still appears to bypass the hook-aware outbound seams for the WebChat/Gateway TUI `chat.send` path. The related umbrella #50126 is relevant, but this issue remains a concrete default-surface bug because WebChat and `openclaw-tui` coverage is still missing, and the related implementation PR #12584 was closed unmerged in the provided context.

## Best Possible Solution

Keep this issue open and fix the Gateway `chat.send` reply path used by WebChat and `openclaw-tui`. The best implementation is to route this path through the shared hook-aware dispatcher/observer seam, or add equivalent `message_sending`, `message_sent`, and `before_message_write` calls around the existing reply delivery and transcript append paths, with regression coverage for WebChat `chat.send` and Gateway-backed TUI replies.
