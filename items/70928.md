## Summary

Current main does not have enough implemented coverage to close #70928. Webchat and openclaw-tui both enter the gateway `chat.send` path, and that path still builds a direct reply dispatcher plus local transcript/broadcast handling without the outbound `message_sent` emitter used by deliverable channel paths. Related PR #12584 is still open in the provided context, so the remaining bug is not shipped on main.
