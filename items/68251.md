## Summary

Current main already implements the requested behavior: Gemini memory embeddings default to `gemini-embedding-001`, `memorySearch.model` is configurable/documented, and memory search now degrades more safely on embedding failures/timeouts. The same implementation is present in the released `v2026.4.23` tag.

## Evidence

- **Gemini memory adapter uses the Gemini embedding default:** The bundled Gemini memory embedding adapter registers `defaultModel: DEFAULT_GEMINI_EMBEDDING_MODEL` for `provider: "gemini"`, so implicit Gemini memory-search config resolves through the Gemini embedding default rather than an inference/chat model.
  - file: [extensions/google/memory-embedding-adapter.ts:23](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/extensions/google/memory-embedding-adapter.ts#L23)

  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Default Gemini embedding model is `gemini-embedding-001`:** The provider constant is `DEFAULT_GEMINI_EMBEDDING_MODEL = "gemini-embedding-001"`, and `normalizeGeminiModel("")` falls back to that exact value when no explicit model is provided.
  - file: [extensions/google/embedding-provider.ts:32](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/extensions/google/embedding-provider.ts#L32)

  - file: [src/agents/memory-search.ts:193](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/agents/memory-search.ts#L193)
