## Summary

Keep open. Current main has a narrower shipped CJK-aware estimator for context-pruning message text, but it does not fully implement this PR’s broader request: soft-trim eligibility still uses raw UTF-16 length, and assistant tool-call arguments are still estimated with raw `JSON.stringify(...).length`. The PR is stale and needs rebase, but it still points at real remaining context-pruner gaps.

## Best Possible Solution

Keep this PR open or replace it with a rebased maintainer patch that uses the shared CJK estimator for the remaining context-pruner estimation gaps, especially assistant tool-call arguments and soft-trim eligibility. If soft-trim thresholds move to weighted units, update the head/tail slicing logic and regression tests at the same time to avoid overlapping or growing trimmed CJK output.
