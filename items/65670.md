## Summary

Keep this PR open. Current main still only preserves the selected Node executable's own bin directory for daemon service PATH construction, and there is no `vite-plus`, `.vite-plus`, or `js_runtime/node` handling or regression coverage. The PR discussion also supports this as a narrow implementation candidate rather than obsolete work.

## Best Possible Solution

Review, rebase if needed, and land this PR or an equivalent narrow fix in `src/commands/daemon-install-plan.shared.ts`, with regression coverage in `src/commands/daemon-install-plan.shared.test.ts`. Keep the PATH addition limited to the matching `.vite-plus` managed toolchain root rather than broadening daemon PATH generically.
