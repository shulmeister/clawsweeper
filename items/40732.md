## Summary

Keep open. Current main still uses a native textarea for the raw JSON/JSON5 config editor, has no CodeMirror/raw-editor component or status-bar breadcrumb implementation, and the related context shows the same raw-editor gap remains tracked elsewhere rather than shipped or clearly superseded.

## Best Possible Solution

Keep this PR open or explicitly consolidate it with the canonical raw config editor effort. The likely product path is a maintainer-selected core UI implementation under `ui/src/ui/views/config.ts` and `ui/src/ui/components`, with tests for textarea replacement, status text, breadcrumbs, selection state, and JSON5 edge cases.
