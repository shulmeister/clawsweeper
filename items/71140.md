## Summary

Keep this issue open. Current main still has MEDIA/GIF delivery and native Telegram reaction primitives, but the normal outbound reply path does not parse `[[react...]]` / `[[react_to_current:...]]`, does not create `channelData.telegram.reaction`, and Telegram delivery/adapter code does not execute reaction-only replies. Draft PR #71141 remains open, unmerged, authored by the same reporter, and uses closing syntax for this issue, so the right cleanup path is review/land or replace that PR rather than close this issue early.

## Best Possible Solution

Keep this issue open and treat draft PR #71141, or an equivalent maintainer-owned patch, as the active implementation path. The fix should extend shared directive parsing for `[[react:...]]` and `[[react_to_current:...]]`, preserve structured Telegram reaction data through outbound normalization, allow reaction-only Telegram replies when a valid target exists, execute native Telegram reactions in both reply delivery and outbound adapter paths, and add parser, normalization, delivery, and adapter tests.
