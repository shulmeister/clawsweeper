## Summary

Keep #71140 open. Current main still parses MEDIA and reply/audio directives, and Telegram has native reaction primitives, but the reported outbound reaction directive path is not implemented: `[[react_to_current:...]]` is not parsed, no structured `channelData.telegram.reaction` is produced, Telegram reply delivery still rejects textless/media-less replies, and `telegramOutbound.sendPayload` only consumes Telegram buttons/quote metadata. The linked #71141 is an open draft implementation PR, not merged/shipped, and the existing maintainer comment explicitly says the issue is still valid.

## Best Possible Solution

Keep #71140 open until #71141 or an equivalent fix lands. The right implementation path is the one already outlined by the issue and maintainer comment: extend the shared reply directive parser to emit structured reaction metadata, preserve it through outbound normalization, allow reaction-only Telegram replies when a reply target exists, call `reactMessageTelegram` from Telegram delivery/sendPayload, and add parser, normalization, delivery, and outbound-adapter tests for reaction-only and text/media-plus-reaction replies.
