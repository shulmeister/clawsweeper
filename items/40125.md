## Summary

Keep #40125 open. Current main has several related pieces: overflow recovery now compacts/truncates and then returns reset/new guidance after failure, model fallback deliberately does not chase context-overflow errors, `/context` can report cached usage, and the Control UI has a high-context notice. But I did not find the requested persistent-chat proactive warning path for Telegram/auto-reply sessions, nor per-session warning state for one-time warning/urgent notices before failure.

## Best Possible Solution

Keep the issue open as a focused UX enhancement. The best path is to add auto-reply/session-level context-pressure evaluation for persistent channels, persist per-session warning/urgent flags, send concise `/reset` and `/new` guidance once per threshold, and reuse existing `contextTokens`, fresh `totalTokens`, `/context`, and compaction metadata rather than creating a separate estimator.
