## Summary

Keep #8122 open. Current main still supports automatic session reset policy at global, chat-type, and channel scope only; Telegram group/topic config has other per-group controls but no `session.reset` field, and the runtime reset path does not consult Telegram group/topic config. Related PR #8134 was closed unmerged, and closed issue #20888 does not supersede this per-Telegram-group request.

## Best Possible Solution

Keep #8122 open. Implement an additive, account-aware reset override for Telegram group/topic config, or add a generic channel plugin reset resolver if maintainers want this across plugins. Update Telegram types, strict Zod schema, generated config metadata/help, public docs, and focused precedence tests for topic/group over channel over type/global, including multi-account cases.
