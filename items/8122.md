## Summary

Keep #8122 open. Current main still does not implement per-Telegram-group session reset configuration: reset policy is still global/type/channel scoped, Telegram group/topic config has no session reset field, the strict Telegram schema would reject the proposed shape, and session initialization does not read Telegram group/topic config for reset policy. The related PR #8134 was closed unmerged, and the broader #20888 lifecycle issue does not supersede this specific per-group request.

## Best Possible Solution

Keep the issue open. Implement an additive, account-aware reset policy override for Telegram group/topic config, or introduce a generic channel group/topic reset resolver if maintainers want this across plugins. Update Telegram types, Zod schema, generated config metadata/help, public docs, and focused tests for precedence such as topic/group over channel over type/global, including multi-account behavior.
