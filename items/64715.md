## Summary

Keep this issue open. Current main still exposes Telegram inline keyboards through the agent/message send surface, but I found no implementation, schema, docs, or tests for Telegram native reply keyboards, ReplyKeyboardRemove, resize_keyboard, or one_time_keyboard. The related items in the provided context are inline-button/callback work and do not cover this separate reply-keyboard capability.

## Best Possible Solution

Keep the issue open and implement it in the Telegram plugin send path with tests and docs. A good maintainer path is to add a typed Telegram reply-keyboard/remove-keyboard option to the channel-owned message action schema, parse it in `extensions/telegram/src/action-runtime.ts`, pass a reply-markup union through `extensions/telegram/src/send.ts`, and document when to use inline buttons versus native reply keyboards at `https://docs.openclaw.ai/channels/telegram` and the message CLI docs. If maintainers want this to be portable beyond Telegram, first add a generic presentation/quick-reply contract rather than overloading existing callback buttons.
