## Summary

Keep this issue open. Current main still intentionally turns billing and rate-limit provider failures into user-visible channel replies, and WhatsApp has no suppress/custom/owner-only error policy surface. Telegram has adjacent delivery-error throttling/silencing controls, but the billing failure path returns a normal final text payload before channel delivery, so the requested cross-channel provider-error policy remains missing.

## Best Possible Solution

Keep this open and implement a shared outbound provider/runtime error-reply policy before channel delivery. It should support normal reply, silent, custom message, and owner-only routing with per-channel/account/group/topic overrides for WhatsApp and Telegram, while preserving detailed diagnostics in logs/control UI and adding focused tests for WhatsApp group suppression/custom text, Telegram group suppression, owner DM visibility, and unchanged diagnostics.
