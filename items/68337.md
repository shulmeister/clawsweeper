## Summary

Keep PR #68337 open. Current main documents that ACP runs on the host runtime and tells users to install/prewarm missing adapters on the Gateway host, but it does not explicitly warn Docker users to install ACP harness CLIs in the gateway container rather than a sandbox container. The PR is a focused, recent draft docs change with no protected labels, no maintainer-author exemption, and no canonical duplicate in the provided related context.

## Best Possible Solution

Keep the PR open and land a polished docs note that says ACP harness CLI adapters must be available in the gateway runtime/container PATH, not only in sandbox images or sandbox containers. The likely home is `docs/tools/acp-agents.md` or `docs/tools/acp-agents-setup.md`, with an optional cross-reference from `docs/install/docker.md`; no core code change is needed for this cleanup item.
