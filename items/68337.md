## Summary

Current main already covers the substance of PR #68337. The ACP docs state that ACP sessions run on the host runtime, not inside the OpenClaw sandbox, require the harness command to start on the Gateway host, and tell operators to install/prewarm missing harness adapters on the Gateway host. The Docker VM runtime docs also tell Docker operators to bake external binaries into the image and verify them inside the `openclaw-gateway` container. The runtime code and tests enforce the same boundary by forbidding ACP spawn from sandboxed requester sessions and rejecting `sandbox="require"` for ACP.

## Best Possible Solution

Close the draft PR as already implemented. Keep the current ACP and Docker runtime documentation as the source of truth; any future wording tweak should be a small docs follow-up against those existing sections, not this redundant PR.
