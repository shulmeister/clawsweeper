## Summary

Keep #70221 open. Current main still does not implement a `[embed]` width parameter: the shortcode/parser contract only normalizes height, the UI preview type has no width field, the iframe renderer only applies inline height, and the CSS still forces the preview iframe to `width: 100%`. This is a focused Control UI feature request, not a stale or duplicate cleanup candidate.

## Best Possible Solution

Keep the issue open and implement this in the Control UI embed pipeline: add a validated width/preferredWidth field to the shortcode and stored canvas preview contract, preserve it through message normalization/types, apply it in renderPreviewFrame with responsive bounds, and update rich-output docs plus focused parser/render tests.
