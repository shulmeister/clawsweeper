## Summary

Keep PR #62740 open. Current main at 20c3177281d292d3db208b1e875e4b51a14fec94 still calls `response.json()` in `callZaloApi` without checking `response.ok`, so the PR’s requested Zalo HTTP-error handling is not implemented on main. The PR discussion is also useful: bot review caught raw body size/logging and HTTP 408 ambiguity, and the provided PR context says the contributor addressed those in the head commit by truncating non-JSON body text, preserving structured Zalo JSON errors, and adding `httpStatus` disambiguation.
