## Summary

Keep #10960 open. Current main still does not implement the requested true soft-steer behavior: inbound steer mode only attempts injection when the active run is streaming, rejects active embedded runs that are not streaming, and otherwise falls back to queued follow-up behavior. The docs also describe steering as delivered at the next model/tool boundary rather than cleanly pausing or aborting a mid-generation or long-tool wait and resuming with the new input. Related duplicates point back to #10960, and the linked implementation PR #64758 is still unmerged in the provided GitHub context, so this remains the canonical tracker.

## Best Possible Solution

Keep #10960 open as the canonical product/runtime tracker. A complete fix likely belongs in the auto-reply queue and embedded-run control path: when steer-mode input arrives during active non-streaming work or mid-generation, abort or pause cleanly, preserve usable context/partial output, append the user's steering message, and resume in the same session with focused regression coverage for streaming, non-streaming/tool-wait, and fallback paths. PR #64758 or a successor should be reviewed against that behavior before closing the issue.
