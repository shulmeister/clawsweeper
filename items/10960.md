## Summary

Keep #10960 open. Current main still does not implement true mid-stream soft steer for messaging-channel queue mode: inbound steer is attempted only while the active run is streaming, non-streaming active runs reject queued injection, and active steer-mode messages fall back to follow-up queue behavior. The docs still describe delivery at the next model/tool boundary rather than clean pause/abort/resume with the new user input. Related duplicates point back here, and PR #64758 remains open and unmerged.

## Best Possible Solution

Keep #10960 open as the canonical product/runtime tracker. A complete fix should live in the auto-reply queue plus embedded/reply-run control path: when steer-mode input arrives during active non-streaming work, tool waits, or mid-generation, cleanly pause or abort, preserve usable partial context, append the user's steering message, and resume in the same session. Regression coverage should include streaming steer success, non-streaming/tool-wait fallback, steer-backlog/followup behavior, and the distinction from explicit gateway sessions.steer.
