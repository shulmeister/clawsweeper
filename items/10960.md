## Summary

Keep #10960 open. Current main still gates same-turn steering on streaming state, rejects active-but-not-streaming queue injection, and documents boundary/followup behavior rather than the requested pause/abort, inject, and resume soft-steer lifecycle. Related PRs #64758 and #65010 are useful evidence that the gap is known, but they are unmerged in the provided context and current main does not include their proposed lifecycle changes.

## Best Possible Solution

Keep #10960 open as the core/runtime tracker. The best fix should define soft-steer lifecycle semantics for streaming generation, tool execution, backend failure, and fallback; replace streaming-only acceptance gates with a safe active agent-loop predicate; keep `sessions.steer` hard-interrupt semantics distinct; update queue/agent/gateway docs; and add regression coverage for streaming steer, active non-streaming/tool-wait steering, followup fallback, and abort/resume behavior.
