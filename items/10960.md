## Summary

Keep #10960 open. Current main still only attempts inbound steer injection when the active run is streaming, still rejects active-but-not-streaming embedded/reply backends, and still documents follow-up/model-boundary fallback rather than true mid-generation pause/abort, inject, and resume behavior. Related PRs #64758 and #65010 are useful but remain unmerged in the provided context and are separate from origin/main.

## Best Possible Solution

Keep #10960 open as the runtime/product tracker. The likely fix should replace the streaming-only gates with an active agent-loop lifecycle predicate for embedded and reply-run backends, preserve safe follow-up fallback when same-turn steering cannot be accepted, and add regression coverage for streaming steer success, active-but-not-streaming/tool-wait behavior, explicit sessions.steer behavior, and backend failure paths. Re-review after #64758, #65010, or a successor PR lands.
