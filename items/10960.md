## Summary

Keep open. Current main still does not implement true mid-stream prompt injection. Steering only succeeds while a run is actively streaming, and the injected message is deferred until the next model/tool boundary rather than interrupting generation immediately or during long tool execution.

## Evidence

- **Docs describe next-boundary steering, not immediate interruption:** The agent docs say queued steering is delivered only after the current assistant turn finishes executing its tool calls, before the next LLM call.
  - file: [docs/concepts/agent.md:82](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/docs/concepts/agent.md#L82)
  - command: `nl -ba docs/concepts/agent.md | sed -n '82,90p'`
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Queue docs say steer falls back when not streaming:** Queue mode docs define `steer` as cancelling pending tool calls after the next tool boundary and explicitly say that if the run is not streaming it falls back to followup.
  - file: [docs/concepts/queue.md:29](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/docs/concepts/queue.md#L29)
  - command: `nl -ba docs/concepts/queue.md | sed -n '25,33p'`
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Runtime only attempts steer when `isStreaming` is true:** `runReplyAgent` only calls `queueEmbeddedPiMessage(...)` on the `shouldSteer && isStreaming` path. Otherwise the active-run queue policy continues down the followup/enqueue path.
  - file: [src/auto-reply/reply/agent-runner.ts:969](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/auto-reply/reply/agent-runner.ts#L969)

  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Steer API rejects non-streaming runs:** `queueEmbeddedPiMessage` returns false when there is no active handle or when `handle.isStreaming()` is false, with an explicit `reason=not_streaming` branch.
