## Summary

Current main and v2026.4.24 already resolve Brave as a bundled web_search provider through the manifest-backed lazy web-provider path. That path applies bundled-provider compatibility enablement even when plugins.enabled is false, then createWebSearchTool builds the web_search tool from the resolved provider set, so the 2026.4.15 unavailable-tool symptom is obsolete. The linked #68290 identified the same area, but the shipped implementation now covers the observable issue without requiring Brave to appear in the gateway boot plugin count.

## Best Possible Solution

Close this issue as fixed by the shipped lazy bundled web-provider compatibility path. Keep the current manifest-backed web_search resolution behavior; if #68290 is handled separately, it should be treated as superseded or reduced to any still-useful regression coverage rather than merged as the original runtime fix.
