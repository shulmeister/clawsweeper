## Summary

Current main implements the relevant fixes for the 2026.3.12/2026.3.13 `openclaw devices list` loopback gateway-close regression: the WebSocket handshake budget is now shared and longer, devices list has a local loopback fallback for pairing-scope failures, and gateway startup/channel sidecars no longer make normal RPC unavailable while plugin work settles. The Feishu workaround in the issue comments is consistent with plugin/channel startup pressure rather than a remaining core devices-list bug.
