## Summary

Keep open. Current `main` still implements session maintenance as a global `session.maintenance` setting only; there is no per-agent `agents.list[].maintenance` type/schema support or agent-aware resolution/plumbing in the cleanup/save paths.

- **Agent config has no per-agent maintenance field:** `src/config/types.agents.ts:70` defines `AgentConfig` with per-agent overrides like `model`, `skills`, `heartbeat`, `sandbox`, `tools`, and `runtime`, but no `maintenance` property.
