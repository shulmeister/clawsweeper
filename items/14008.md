## Summary

WeCom is already scoped on current `main` as an external/community channel plugin, not a bundled OpenClaw core extension, so this request belongs in the ClawHub/third-party plugin path rather than this repo's core backlog.

## Close Comment

Codex review: current `main` treats WeCom as a third-party/community plugin, not a bundled OpenClaw core extension. `VISION.md:54` says optional capability should usually ship as plugins maintained in their own repo, and `docs/plugins/building-plugins.md:16` says you do not need to add a plugin to this repository and should publish to ClawHub or npm instead. WeCom is already listed on the community plugins page at `docs/plugins/community.md:115`, and the channel catalog contract expects the external package `@wecom/wecom-openclaw-plugin` in `src/channels/plugins/contracts/channel-catalog.contract.test.ts:40`. Closing this as `clawhub` scope: the useful path here is maintaining/publishing the WeCom plugin externally and, if needed, improving its community-plugin docs/catalog surface rather than adding it to OpenClaw core.
