## Summary

Issue #34474 asks OpenClaw maintainers to install or enable Slack's GitHub app for the `openclaw/openclaw` repository so Slack can subscribe to release notifications. That is repository/third-party service administration, not an OpenClaw source, docs, plugin API, or bundled Slack plugin change. The related Slack Canvas report is a separate plugin feature and does not supersede this request.

## Best Possible Solution

Close this source-repository issue as not actionable in `openclaw/openclaw`. If maintainers want Slack release subscriptions enabled, handle it through GitHub/Slack organization administration rather than code, docs, or plugin changes. Keep #40882 separate because it concerns Slack Canvas event support inside the bundled Slack plugin.
