## Summary

Current `main` already suppresses the startup config-watcher reread for self-auth writes, preventing the immediate restart cycle that could reprocess a startup-generated or startup-persisted `gateway.auth` token and cause token drift. A focused regression test covers this startup-write path and asserts no restart on the first watcher event.

## Close Comment

Closing as `implemented_on_main` based on a Codex review of current `main`. The gateway now records the final startup config hash after startup writes such as auth-token persistence (`src/gateway/server.impl.ts:333`), passes that hash into the config reloader (`src/gateway/server.impl.ts:859`), and the reloader skips the first watcher reread when it matches that startup self-write (`src/gateway/config-reload.ts:351`). There is also a focused regression test for this exact startup-write path that uses `gateway.auth.mode: "token"` and asserts the first watcher event does not trigger a restart (`src/gateway/config-reload.test.ts:1003`). The checked-out fix is in commit `a29233c9af0c03e906b5a1e3dceb927119eafe04`. I could not tie that commit to a released version from the current checkout, so I’m marking the release as unknown/not yet confirmed.
