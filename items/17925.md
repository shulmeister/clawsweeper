## Summary

Keep #17925 open. Current main implements the Gemini/Google Search grounding side as a bundled `gemini` web_search provider, but the ZAI/GLM side requested by the issue is still absent: the ZAI plugin has no web-search manifest contract, no web-search registration, no ZAI search provider files, and no ZAI search docs/tests. The issue discussion also already called this out and kept the tracker open for ZAI/Gemini native search passthrough, with the latest review noting Gemini is done but ZAI remains missing.

## Best Possible Solution

Keep #17925 open until the bundled ZAI plugin either registers a ZAI/GLM `web_search` provider through the existing plugin web-search contract with manifest metadata, config/docs, and tests, or implements provider-owned native `tools: [{ type: "web_search" }]` passthrough for ZAI model calls with equivalent coverage. The Gemini side is already implemented, so the remaining work can be narrowed to ZAI/GLM and any desired automatic-fallback semantics.
