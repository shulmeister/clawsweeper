## Summary

Keep open. Current main implements the Gemini/Google side as a bundled `gemini` web_search provider using Google Search grounding, but I found no ZAI/GLM `web_search` provider or native passthrough. The issue remains a partial-implementation tracker, and the latest maintainer/Codex discussion already preserved the ZAI scope.

## Best Possible Solution

Keep tracking this issue until OpenClaw either adds ZAI/GLM native `web_search` passthrough or a bundled ZAI web_search provider with docs/tests, and until maintainers decide whether the existing managed Gemini provider fully satisfies the Google fallback semantics requested here.
