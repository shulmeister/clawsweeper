## Summary

Keep #17925 open. Current main partially satisfies the request: Gemini/Google now has a bundled `gemini` web_search provider using Google Search grounding, but ZAI/GLM still has no bundled web_search provider, no `webSearchProviders` manifest contract, and no native `web_search` passthrough in the ZAI plugin runtime. The issue discussion already preserved the ZAI/Gemini native-passthrough scope after related Kimi/Gemini progress, and the referenced Zhipu/ZAI PRs #10454 and #13814 are closed and unmerged, so they do not supersede the remaining work.

## Best Possible Solution

Keep tracking this issue until OpenClaw either adds ZAI/GLM native `web_search` passthrough or ships a bundled ZAI web_search provider with manifest/docs/tests. Maintainers should then decide whether the existing managed Gemini provider fully satisfies the requested Google fallback behavior or whether a narrower follow-up should track automatic provider-native passthrough semantics.
