## Summary

Keep #17925 open. Current main partially satisfies the Google/Gemini side through a bundled `gemini` web_search provider using Google Search grounding, but the ZAI/GLM side is still missing: the ZAI plugin has no `webSearchProviders` manifest contract, no web-search config surface, no `registerWebSearchProvider` registration, and no native `tools: [{ type: "web_search" }]` passthrough path. The related Zhipu/ZAI PRs #10454 and #13814 are closed and unmerged, and the latest maintainer discussion already preserved this issue for the remaining ZAI/Gemini native-passthrough scope.

## Best Possible Solution

Keep this issue open until OpenClaw either adds provider-owned ZAI/GLM native `web_search` passthrough in the bundled ZAI plugin or ships a bundled ZAI web_search provider with manifest ownership, config/docs, and tests. Maintainers can separately decide whether the existing managed Gemini provider fully satisfies the Google fallback behavior or whether a narrower follow-up should track automatic native passthrough semantics.
