## Summary

Keep #17925 open. Current main implements the Gemini side as a bundled `gemini` web_search provider using Google Search grounding, but the requested ZAI/GLM native `web_search` passthrough or bundled ZAI search provider is still missing. The related ZAI PRs in the provided context are closed and unmerged, and the latest maintainer/Codex issue comment already keeps this tracker open for the remaining ZAI work.

## Best Possible Solution

Keep #17925 open until the bundled ZAI plugin either registers a ZAI/GLM `web_search` provider through the existing plugin web-search contract with manifest ownership, config/docs, and tests, or adds provider-owned native `tools: [{ type: "web_search" }]` passthrough for ZAI model calls. Gemini's bundled provider appears implemented, so maintainers may split any remaining automatic-fallback semantics into a narrower follow-up if desired.
