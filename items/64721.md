## Summary

Keep open. Current main partially covers the report because `payload.model` and `payload.timeoutSeconds` are now present in the cron tool and gateway cron schemas, but the remaining requested `payload.contextTokens` and `maxSpendUsd` fields are still not part of the cron job contract and would not survive gateway validation. I also found no current `maxSpendUsd` implementation in the checkout, so that part needs either implementation or explicit product clarification rather than closure.

## Best Possible Solution

Keep this issue open and implement the remaining cron job contract deliberately: add `contextTokens` to cron agentTurn create/update payload schemas, normalization, types, tool schema and flat-params recovery, then thread it into isolated cron execution as a per-job context cap with regression tests. For `maxSpendUsd`, first confirm intended semantics because current main does not contain that field; either add a real per-run spend cap path and tests or split that request into a separate product/API issue. Preserve the already-present `model` and `timeoutSeconds` behavior.
