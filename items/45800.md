## Summary

Keep open. Current main still renders the WebChat New session plus button in the composer control cluster and wires it directly to `handleSendChat("/new")` without a confirmation step. The related PR #45863 would address this, but it is open/unmerged in the provided GitHub context, so the requested UX fix is not implemented on main.
