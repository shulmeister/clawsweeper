## Summary

Current `main` already contains the Slack dispatch/streaming cleanup this issue asked for: draft/native/block streaming are made mutually exclusive, draft preview is not initialized when streaming is off or native streaming is active, native streams use the planned reply thread, and stream finalization now guards/stops/falls back without duplicating delivery. The targeted Slack tests for these cases were added in `fc5920fb51346a682b269fdbeeec0971ae4ae723`, which is on `main` but not in the latest tagged release yet.
