## Summary

Keep open. Current main still has Feishu account config on `name` with strict validation, while the doctor legacy channel migrations contain only the thread-bindings migration and no Feishu `botName` -> `name` repair. The linked PR #65202 was closed unmerged, and related context shows PR #65384 is still the focused implementation candidate for this gap.

## Best Possible Solution

Keep this issue open and review the focused Feishu doctor migration PR (#65384), or land an equivalent owner-scoped fix in `src/commands/doctor/shared/legacy-config-migrations.channels.ts` with regression coverage that migrates both top-level and per-account `botName` to `name` while preserving an existing `name`. Once that PR or equivalent lands on main, close this issue as implemented with the landed commit.
