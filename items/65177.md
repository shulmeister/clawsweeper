## Summary

Keep open. Current `main` still uses a strict Feishu account schema with `name` and no `botName`, while the doctor migration pipeline does not contain a Feishu `botName -> name` repair. The referenced PR #65202 contains the missing migration as a patch, but that change is not present in local `main` (`ae5c6573670b3742b156899bd2b9c36b8677f191`) or the latest release snapshot checked (`v2026.4.23`, `a9797214338ba31c52c796adbb75afb16e0684a9`).

## Evidence

- **Feishu schema is strict and only accepts `name`:** `FeishuAccountConfigSchema` defines `name` as the display-name field and ends with `.strict()`, so legacy `botName` remains an extra property and will fail validation.
  - file: [extensions/feishu/src/config-schema.ts:191](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/extensions/feishu/src/config-schema.ts#L191)
  - command: `nl -ba extensions/feishu/src/config-schema.ts | sed -n '188,206p'`
  - sha: [ae5c6573670b](https://github.com/openclaw/openclaw/commit/ae5c6573670b3742b156899bd2b9c36b8677f191)
- **Doctor repair path uses the legacy migration registry:** `applyLegacyDoctorMigrations` clones the raw config, applies every entry in `LEGACY_CONFIG_MIGRATIONS`, then applies channel compatibility migrations. If the Feishu rename is not registered there, `openclaw doctor --fix` will not rewrite it.
  - file: [src/commands/doctor/shared/legacy-config-compat.ts:4](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/src/commands/doctor/shared/legacy-config-compat.ts#L4)
