## Summary

Keep open. Current main has not implemented the PR's central security change: `tools.fs.workspaceOnly` still defaults to unrestricted behavior, tests lock that default, and there is no agent filesystem sensitive-path denylist on the read/write/edit paths. The PR is incomplete as submitted because review comments correctly identify that the read path would remain unguarded when `workspaceOnly=false`, but that is a reason to request changes or supersede with a maintainer patch, not to close this focused security-hardening PR.

## Best Possible Solution

Keep this PR open or replace it with a maintainer-owned superseding patch. The best path is to decide the default policy explicitly, update `src/agents/tool-fs-policy.ts`, config types/help/docs, and tests together, and if OpenClaw adds a sensitive-path denylist, enforce it consistently for read/write/edit with symlink-safe realpath handling and regression coverage. Coordinate with the related open proposal #58947 rather than closing this implementation attempt as obsolete.
