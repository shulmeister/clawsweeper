## Summary

Current main still does not implement configurable bootstrap dynamic-file classification or configurable bootstrap file injection order. The prompt renderer hardcodes both the ordering and the dynamic set, config schemas and docs expose only existing injection/size controls, and related context treats this as an unresolved prompt-cache follow-up rather than shipped or superseded.

## Best Possible Solution

Keep this issue open. A conservative implementation should add documented `agents.defaults` and likely per-agent config for `bootstrapDynamicFiles` and possibly `bootstrapFileOrder`, thread those values into prompt assembly before `sortContextFilesForPrompt()`, preserve deterministic fallback ordering, update generated config schema/help/docs, and add tests proving selected files move below `OPENCLAW_CACHE_BOUNDARY` without destabilizing the stable prefix.
