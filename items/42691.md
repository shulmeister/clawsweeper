## Summary

Keep open. Current `main` does not contain the PR‚Äôs core fix: cron-embedded agent work still routes onto the shared `nested` lane, and gateway lane concurrency still only configures `cron`, `main`, and `subagent`. That means the requested dedicated `cron-nested` lane/concurrency alignment is not implemented on `main`. One side topic from the PR has already drifted: `GatewayCredentialPlan` no longer has a `remoteEnabled` field, so that cleanup is partly obsolete, but the main gateway/cron lane change remains unresolved.
