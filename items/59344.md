## Summary

Keep open. Current `main` at `78a431ed224b6f5b9d58672346551a5b411ad200` still does not dedupe hook-adjusted bootstrap files by path, so the underlying bug is not implemented on main. The PR is directionally useful, but the review thread raises a real correctness gap: its dedupe key uses `path.resolve(pathValue)`, which is anchored to process CWD rather than `workspaceDir`, so relative hook paths can still miss deduplication.

## Evidence
