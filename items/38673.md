## Summary

Current main implements the requested per-agent TTS configuration. Agents can now set `agents.list[].tts`, the schema accepts it, runtime resolution deep-merges it over global `messages.tts` by active agent id, auto-reply paths pass that id into TTS, and docs/tests cover the behavior. The earlier related PR #44583 was closed unmerged, but maintainer commit 0ca952cdd58ac0491c282e466b49d90f10d2a96a implemented the observable request on main.

## Best Possible Solution

Close #38673 as implemented on main. Keep the shipped `agents.list[].tts` deep-merge design as the core solution for per-agent voices, and leave separate per-channel/topic policy requests such as #39614 to track channel-based voice selection if maintainers want that additional behavior.
