## Summary

Current `main` does not implement per-agent TTS config. TTS is still resolved from global `messages.tts`, and the per-agent schema/docs do not allow an `agents.list[].messages.tts` override, so there is not strong evidence to close this issue.

## Evidence

- **Agent config type has no per-agent messages/TTS field:** `AgentConfig` includes fields like `identity`, `groupChat`, `tools`, and `runtime`, but no `messages` or `tts` field, so there is no typed per-agent TTS surface on `main`.
  - file: [src/config/types.agents.ts:70](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/src/config/types.agents.ts#L70)
  - command: `nl -ba src/config/types.agents.ts | sed -n '55,125p'`
  - sha: [51186d272543](https://github.com/openclaw/openclaw/commit/51186d2725439646f7fea0e59bb466f957404c46)
- **Agent schema is strict and omits messages/TTS:** `AgentEntrySchema` is a strict object and enumerates allowed per-agent keys. It has no `messages` or `tts` property, which means `agents.list[].messages.tts` is not part of the current config schema.
  - file: [src/config/zod-schema.agent-runtime.ts:812](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/src/config/zod-schema.agent-runtime.ts#L812)

  - sha: [51186d272543](https://github.com/openclaw/openclaw/commit/51186d2725439646f7fea0e59bb466f957404c46)
- **TTS runtime resolves only global messages.tts:** `resolveTtsConfig(cfg)` reads `const raw: TtsConfig = cfg.messages?.tts ?? {};`, so the effective TTS provider/config is sourced from the top-level message config rather than agent-specific config.
  - file: [extensions/speech-core/src/tts.ts:313](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/extensions/speech-core/src/tts.ts#L313)

  - sha: [51186d272543](https://github.com/openclaw/openclaw/commit/51186d2725439646f7fea0e59bb466f957404c46)
- **Docs still describe TTS as global messages.tts:** The user-facing TTS docs state: `TTS config lives under messages.tts in openclaw.json`, with examples only under the global `messages` block.
  - file: [docs/tools/tts.md:72](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/docs/tools/tts.md#L72)

- **Per-agent docs do not document TTS overrides:** The `agents.list` documentation enumerates supported per-agent overrides such as `model`, `identity`, `groupChat`, `sandbox`, `runtime`, and `tools`, but does not include any per-agent TTS or `messages` override.
