## Summary

Keep #70309 open. Current main has MCP config editing/counting and session-scoped effective tool inventory, but it does not provide the requested Control UI MCP connectivity panel: no per-server connected/error state, no tools grouped by configured MCP server outside a live session, no one-click reconnect/refresh, and no UI-surfaced connection errors.

## Best Possible Solution

Keep the issue open and implement a generic core/Gateway diagnostic path for configured `mcp.servers`, then add a Control UI Infrastructure or MCP panel that shows per-server status, last checked time, redacted error details, discovered tool names/counts, and a scoped refresh/reconnect action. This likely belongs in Gateway protocol/server-methods plus Control UI state/rendering and docs, not ClawHub, because it is generic status for OpenClaw-owned MCP config rather than a service-specific plugin.
