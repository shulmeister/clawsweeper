## Summary

Keep this issue open. Current main still supports MCP server configuration and session-time MCP tool materialization, but it does not provide the requested Control UI MCP connectivity panel with per-server connected/error state, per-server tool names, manual reconnect/refresh, or surfaced redacted error details.

## Best Possible Solution

Keep this issue open. The best implementation is an additive Gateway operator diagnostics API backed by the MCP config/runtime loader, returning each configured server's name, transport summary, connected/error state, listed tool names/count, and redacted last error, plus a scoped refresh/reconnect action. Then render it in Control UI under Infrastructure > MCP and document it on the public MCP/configuration docs pages.
