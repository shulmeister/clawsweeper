## Summary

Current main already covers the reported Windows Node 25 drive-letter ESM loader failure for both affected surfaces. The browser plugin entry is loaded through the plugin loader path that now converts Windows absolute import specifiers to file URLs, and bundled/channel plugin loading now disables native Jiti on Windows so Telegram no longer hands raw `C:` paths to Node's ESM loader. This also matches the already-closed related report #72221 for the same root cause.

## Best Possible Solution

Close this as fixed on current main. Keep the shared Windows-safe plugin/channel loader behavior, and ask for a fresh issue only if the next release still reproduces with full Windows logs, OpenClaw version, Node version, and affected plugin/channel details.
