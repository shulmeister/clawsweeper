## Summary

Keep open. Current main has partial bootstrap truncation visibility, including truncation markers, prompt warnings on actual truncation, `/context` reporting, and doctor near-limit warnings. It still does not implement the requested Control UI warning/limit enforcement for MEMORY.md: the file editor only shows a raw byte count in the preview metadata, Save is not disabled by any bootstrap budget, and the backend writes the submitted content without a size-limit rejection. The directly related PR #64874 remains open/draft, so the remaining work is not shipped or a conservative superseding close.
