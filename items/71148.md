## Summary

Keep open: current OpenClaw main still ships the bundled Lobster plugin against @clawdbot/lobster 2026.4.6, and the upstream Lobster source still has the reported per-call Ajv compile sites without a cache/release path.

## Best Possible Solution

Keep #71148 open. Fix the Ajv validator lifecycle in openclaw/lobster, preferably by memoizing validators by stable schema content and reusing compiled workflow input validators, then release @clawdbot/lobster and update extensions/lobster/package.json plus pnpm-lock.yaml in OpenClaw. Add a focused regression/perf test that repeated identical response schemas do not repeatedly call Ajv.compile().
