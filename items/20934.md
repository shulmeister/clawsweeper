## Summary

Keep #20934 open. Current main still lacks REST-native session reset/status/compact/model/reasoning endpoints for HTTP chat-completions integrations. The underlying Gateway RPC/session operations exist, and there are partial tool/plugin surfaces, but they do not provide the standardized authenticated /v1/sessions REST API requested here. A maintainer/Codex review comment on the issue also explicitly kept this item open as the canonical REST tracker.

## Best Possible Solution

Keep #20934 open as the canonical REST session-management tracker. The best implementation is an authenticated Gateway HTTP API for reset, status/resolve-or-list, compact, model, and reasoning updates that reuses existing sessions.reset, sessions.list/sessions.resolve, sessions.compact, and sessions.patch semantics, with route/auth/scope tests and public docs under the OpenAI-compatible HTTP API.
