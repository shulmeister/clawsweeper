## Summary

Keep #13387 open. Current main still does not auto-prune consecutive assistant `NO_REPLY` transcript messages before model replay. The linked implementation PR #13388 was closed unmerged, and the code only has adjacent mitigations: heartbeat-pair filtering, display/history suppression of silent rows, image/tool-result context pruning, and silent-reply delivery policy.

## Best Possible Solution

Keep the issue open until core implements transcript replay pruning, likely in the embedded-runner replay pipeline near `sanitizeSessionHistory`/`validateReplayTurns`/`limitHistoryTurns`, with focused tests for consecutive assistant `NO_REPLY` turns and docs/config updates if the behavior is configurable.
