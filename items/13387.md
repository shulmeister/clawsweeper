## Summary

Current main still does not implement the requested consecutive assistant `NO_REPLY` pruning. The replay path has adjacent cleanup for provider replay, heartbeat pairs, image history, tool-result context pruning, and UI/history suppression, but no pass that collapses repeated silent assistant turns before model replay. The linked PR #13388 was closed unmerged, so the issue remains actionable.

## Best Possible Solution

Keep #13387 open. Implement a core replay-context pruning pass near `sanitizeSessionHistory` / `validateReplayTurns` / `limitHistoryTurns` that collapses consecutive assistant silent replies while preserving isolated legitimate `NO_REPLY` turns. Add focused tests for text-block `NO_REPLY`, legacy string content, and empty-content silent stop turns, plus docs/config updates if maintainers want this configurable.
