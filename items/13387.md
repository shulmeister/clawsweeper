## Summary

Current `main` still does not implement replay-context pruning for consecutive assistant `NO_REPLY` / silent-reply turns. The embedded runner sanitizes, validates, heartbeat-filters, limits, and repairs transcript history before assigning it to model state, but there is no pass that detects repeated silent assistant replies and collapses them. The linked implementation PR #13388 remains closed/unmerged in the provided GitHub context, and current source does not contain its proposed helper or config surface.

## Best Possible Solution

Keep #13387 open. Implement a focused replay-context pruning pass in the embedded runner history pipeline, likely after validation and history limiting or inside a clearly documented transcript hygiene helper, that collapses consecutive assistant silent replies while preserving isolated legitimate silent turns. Reuse shared silent detection such as isSilentReplyPayloadText and cover string content, text blocks, JSON silent envelopes, and empty-content silent stop turns with targeted replay-context tests.
