## Summary

Current main still does not implement inter-session progress streaming for `sessions_send`. The tool remains limited to fire-and-forget or synchronous `agent.wait`; wait expiry still returns `status: "timeout"`; A2A announce remains completion-only; and gateway event delivery is still WebSocket/session-subscriber or target-session oriented rather than a caller-visible internal session event stream. Related ACP parent streaming exists, but it is scoped to `sessions_spawn({ runtime: "acp", streamTo: "parent" })`, not arbitrary `sessions_send` delegation.

## Best Possible Solution

Keep this issue open and implement an additive core session-event observation path. A good direction is a permission-checked `agent.events` long-poll/read API backed by bounded per-run buffering, plus either a `sessions_subscribe` tool or a `sessions_send` progress option. Tests should cover long-running `sessions_send`, timeout-vs-accepted semantics, cross-session event visibility, target-session routing isolation, and Telegram/topic-bound progress delivery.
