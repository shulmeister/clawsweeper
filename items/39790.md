## Summary

Keep PR #39790 open. Current main at e672b61417af5c45b0431df6d9109a1f4b618ef5 still lacks both behaviors this PR targets: transcript title extraction does not strip inbound metadata, and derived session titles cannot prefer the computed outer session-row displayName. The linked #39722 TUI /sessions bug remains actionable, and the prior Greptile review supports the PR direction while noting missing metadata-strip test coverage.

## Best Possible Solution

Keep PR #39790 or linked issue #39722 open until main implements the narrow gateway fix: strip inbound metadata inside session title text extraction before directive stripping, pass the computed row displayName into deriveSessionTitle or equivalent title logic, and add focused regression tests in the session-utils filesystem/title coverage.
