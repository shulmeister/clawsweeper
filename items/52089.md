## Summary

The reported slash-command menu clipping bug is already fixed on `main` and was shipped after `2026.3.13`. Current Control UI still renders the menu above the composer, but the clipping `overflow: hidden` on `.agent-chat__input` is gone.

## Close Comment

Closing from a Codex review because this is already fixed on `main` and appears to have shipped in `v2026.3.24`. The original broken release (`v2026.3.13-1`) had `overflow: hidden` on `ui/src/styles/chat/layout.css:376` for `.agent-chat__input`, which would clip the absolutely-positioned slash menu. On current `main`, that clipping rule is gone at `ui/src/styles/chat/layout.css:476`, while the menu still renders above the input via `.slash-menu { bottom: 100%; }` at `ui/src/styles/chat/layout.css:719` and is still mounted inside the composer in `ui/src/ui/views/chat.ts:1201`. The fix landed in commit `21ac4b9a8a847e2895c9101515c8972f213d3ac9`, which is present by `v2026.3.24`.
