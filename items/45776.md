## Summary

Current main already addresses the reported repeated Feishu registration behavior. Feishu still has several expected per-surface registration log lines during one full load, but current startup/runtime loading scopes plugin activation, reuses cached/active registries, and treats same-plugin channel re-registration idempotently so the full Feishu registration set should not repeat 4-5 times on CLI startup or inbound gateway turns.

## Evidence

- **Issue discussion narrowed the bug:** The comments distinguish expected Feishu sub-surface log lines from the actual bug: repeated full registration sets within one CLI/gateway turn. The later Docker repro on 2026.3.23 reported the full feishu_doc/chat/wiki/drive/bitable set repeating 4-5x per inbound agent turn, with no functional failure but about 500ms overhead.
- **Feishu full registration is centralized:** The bundled Feishu entry registers all runtime tool surfaces only from registerFull, so repeated full log sets indicate repeated full plugin activation, not separate production entrypoints independently registering the same tools.
