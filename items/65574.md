## Summary

Keep open. Current main still does not propagate ambient proxy env into stdio MCP child launches: `src/agents/mcp-stdio.ts:36` only forwards user-configured `raw.env`, and `src/agents/mcp-stdio-transport.ts:50` still builds the child env from the MCP SDK default inherited env plus that explicit config. The report is also reinforced by prior shipped proxy work in `CHANGELOG.md:1236` and by production confirmation in the PR discussion. One maintainer-level nuance remains: the PR’s inherited `NODE_OPTIONS` behavior conflicts with the current host env safety policy, so this needs review, not auto-closure.
