## Summary

Keep open. Current main still does not merge ambient gateway proxy env vars into MCP stdio child processes, and the provided discussion contains a concrete production proxy-only reproduction plus PR-side fixes for earlier review findings. The implementation likely needs rework against current main’s stdio env safety policy, but it is a plausible focused fix for an unresolved core MCP launch behavior.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best path is a current-main patch in the MCP stdio launch/transport path that propagates safe ambient proxy-related host env to child processes with explicit precedence tests, while reconciling any NODE_OPTIONS handling with the documented stdio startup safety policy.
