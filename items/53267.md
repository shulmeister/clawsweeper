## Summary

The requested sub-agent pattern-promotion registry is not an OpenClaw core concept on current `main`; skill evolution/procedural-memory workflows are intentionally routed to plugins/ClawHub, and the repo contains no `semantic-patterns.json`/`promoted=True`/pattern-registry surface to extend.

## Close Comment

Closing from this Codex review as `clawhub`. On current `main`, OpenClaw keeps optional skill workflow features out of core: `VISION.md:57-70` routes new skill capability to plugins/ClawHub first, and `VISION.md:101-107` explicitly pushes back on new core skills and heavy orchestration layers. The shipped skills surface is also scoped that way today: `docs/tools/skills.md:91-129` puts reusable-procedure capture in the optional Skill Workshop plugin and publish/sync flows in ClawHub, while `src/cli/skills-cli.ts:52-170` only exposes `search/install/update/list/info/check` for skills. I also checked current `main` for the issue-specific concepts (`semantic-patterns.json`, `promoted=True`, `pattern registry`, `python_execute`) and found no matches, so this looks like external skill/evolution tooling rather than an OpenClaw core gap. If you want to pursue it, the better fit is a ClawHub skill/plugin or Skill Workshop-style extension, not core OpenClaw.
