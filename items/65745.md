## Summary

Current `main` still appears to misclassify valid dangerous node command denylist entries as ineffective. The audit derives `knownCommands` from platform allowlists plus explicit `allowCommands`, while the dangerous commands reported in the issue live in `DEFAULT_DANGEROUS_NODE_COMMANDS` and are intentionally excluded from default allowlists. That means entries like `camera.snap` and `screen.record` can still fall into the audit's `unknownExact` bucket. The quickstart wizard also seeds `gateway.nodes.denyCommands` from that dangerous-command list, which makes the warning especially misleading for a normal generated config.

## Risks / Open Questions

- Operators can be warned away from keeping explicit deny entries for high-impact commands that OpenClaw itself seeds in new gateway configs.
- Because the repo lacks a regression test for dangerous-but-valid deny entries, the misleading audit behavior can persist unnoticed.
