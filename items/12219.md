## Summary

Keep #12219 open. The current main branch has useful adjacent work: plugin `before_tool_call` and `before_install` hooks, skill install scanning for gateway-backed dependency installs, and ClawHub skill install/update plumbing. It still does not implement the requested `skill.yaml`/`skill.yml` permission manifest standard, install-time permission review for `openclaw skills install`, `openclaw skills audit`, or signing/attestation/trust-grade support. The latest maintainer discussion also explicitly kept this issue open and treated @ellistevo/openclaw-secure, ClawHub shell-security-ultimate, and globalcaos/tinkerclaw as references rather than shipped replacements.

## Best Possible Solution

Keep #12219 open for maintainer product/security design. Define a versioned skill permission manifest, validate it during ClawHub and direct skill install/update flows, show declared permissions before `openclaw skills install`, add an audit/trust report or equivalent, and connect declarations to `before_install` and `before_tool_call` enforcement where appropriate. Treat @ellistevo/openclaw-secure, ClawHub shell-security-ultimate, and globalcaos/tinkerclaw as useful references, not shipped replacements.
