## Summary

Keep open. Current main has adjacent safety work for plugin install hooks, tool-call hooks, and security-audit skill scanning, but it still does not implement a `skill.yaml` permission manifest standard, `openclaw skills audit`, install-time permission review for `openclaw skills install`, signing/attestation, or trust-score presentation. The issue discussion also includes a maintainer-side Codex review explicitly keeping this open, while the linked external implementations are useful ecosystem evidence rather than a shipped core solution.

## Best Possible Solution

Keep #12219 open for maintainer product/security design. A solid path is to define a versioned skill permission manifest, validate it during ClawHub/direct skill install and update, show permissions before `openclaw skills install`, add a real `openclaw skills audit` or equivalent user-facing trust report, and connect declarations to existing `before_install` and `before_tool_call` enforcement hooks where the API is sufficient. External projects like `@ellistevo/openclaw-secure`, `shell-security-ultimate`, and globalcaos/tinkerclaw should be treated as useful reference implementations, not a reason to close the core issue.
