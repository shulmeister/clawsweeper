## Summary

Keep open. Current main has adjacent skill loading, ClawHub install, install-scan, and plugin hook infrastructure, but it still does not implement a `skill.yaml` permission manifest standard, install-time permission review for `openclaw skills install`, `openclaw skills audit`, signing/attestation, or trust-score display. The issue also has an explicit 2026-04-26 maintainer-side Codex review keeping it open, and the external implementations discussed there are useful reference work rather than shipped OpenClaw behavior.

## Best Possible Solution

Keep #12219 open for maintainer product/security design. The best path is to define a versioned skill permission manifest, validate it during ClawHub/direct skill install and update, show permissions before `openclaw skills install`, add a real `openclaw skills audit` or equivalent trust report, and connect declarations to existing `before_install` and `before_tool_call` enforcement hooks where the API is sufficient. Treat `@ellistevo/openclaw-secure`, `shell-security-ultimate`, and globalcaos/tinkerclaw as useful reference implementations rather than reasons to close the OpenClaw core issue.
