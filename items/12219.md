## Summary

Keep open. Current main has adjacent safety hooks, install scanning, and skill gating metadata, but it still does not implement the requested `skill.yaml` permission manifest standard, direct `openclaw skills install` permission review, `openclaw skills audit`, signing/attestation, or trust-score flow. A maintainer review comment on the issue also explicitly kept it open, and the external implementations linked in discussion are useful ecosystem evidence rather than a core implementation.

## Best Possible Solution

Keep #12219 open until maintainers choose the product direction or implement a concrete standard. A strong path would define a versioned skill permission manifest, validate it in ClawHub/direct skill installs, show permissions before `openclaw skills install`, add `openclaw skills audit`, and connect manifest declarations with existing `before_install`/`before_tool_call` enforcement hooks where appropriate.
