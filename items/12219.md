## Summary

Keep #12219 open. Current `main` still lacks a `skill.yaml`/`skill.yml` permission manifest standard, install-time skill permission review for `openclaw skills install`, `openclaw skills audit`, trust grades, signing, or attestation. The issue also has explicit maintainer discussion from steipete on April 26, 2026 saying to keep it open for product/security design.

## Best Possible Solution

Keep #12219 open for explicit maintainer product/security design. The likely path is a versioned skill permission manifest with validation during ClawHub and direct `openclaw skills install`/update flows, visible install-time permission review, an audit/trust report or equivalent, and integration with existing `before_install` and `before_tool_call` enforcement hooks. Use `@ellistevo/openclaw-secure`, ClawHub `shell-security-ultimate`, and `globalcaos/tinkerclaw` as positive reference implementations while coordinating with broader supply-chain work such as #10827 and #50090.
