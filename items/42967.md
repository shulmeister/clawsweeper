## Summary

Keep open. Current `main` does not implement this PR's agent-tool change: the `tts` tool schema still only accepts `text`, `channel`, and `timeoutMs`, and its execute path never reads or forwards a `voiceId` override.

## Evidence

- **Agent tts tool lacks voiceId parameter:** `src/agents/tools/tts-tool.ts` defines `TtsToolSchema` with only `text`, `channel`, and `timeoutMs`; there is no `voiceId` field in the current agent tool contract.
  - file: [src/agents/tools/tts-tool.ts:10](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/agents/tools/tts-tool.ts#L10)
  - command: `nl -ba src/agents/tools/tts-tool.ts | sed -n '1,220p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Agent tts execute path does not forward overrides:** The current `createTtsTool` execute path reads `text`, `channel`, and `timeoutMs`, then calls `textToSpeech({ text, cfg, channel, timeoutMs })` without `overrides`, `voiceId`, or `disableFallback` handling.
  - file: [src/agents/tools/tts-tool.ts:67](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/agents/tools/tts-tool.ts#L67)
