## Summary

Keep open. Current main still treats Feishu `requireMention` as a reply gate, not a context-isolation policy: unmentioned group messages are buffered into pending history and injected into the next mentioned turn. The Feishu schema/docs expose `requireMention`, `historyLimit`, and group/session scopes, but I found no `contextIsolation`, `contextPolicy`, `excludeOtherBots`, or mention-only context mode. The related #40782 mention-identity work is relevant but does not implement this context-policy request.

## Best Possible Solution

Keep this issue open. Implement a backwards-compatible Feishu plugin policy, likely top-level and per-group, such as `contextPolicy: "all" | "mention-only"` or `contextIsolation: true`, defaulting to today's pending-history behavior. When enabled, `extensions/feishu/src/bot.ts` should skip recording unmentioned group messages into pending history and should exclude messages addressed to other bots once Feishu multi-bot identity matching is reliable. Update `extensions/feishu/src/config-schema.ts`, plugin metadata/docs, and Feishu tests for multi-bot pending-history behavior.
