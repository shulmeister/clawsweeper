## Summary

Current `main` matches the documented intent for new DM routing when `session.dmScope=main`, but it still lists persisted direct-session keys and only contains migration logic for the opposite transition (`main` -> per-channel/per-peer). That leaves a credible path for an old per-user DM session to remain visible, so this is not safe to auto-close.

## Evidence

- **Docs still say `dmScope=main` should collapse Discord DMs into the main session:** The public docs explicitly state that, by default, direct chats share `agent:main:main` when `session.dmScope=main`, matching the issue's expected behavior.
