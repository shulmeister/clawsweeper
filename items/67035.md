## Summary

Keep #67035 open. Current main implements important parts of the reported Control UI live-rendering failure, especially stale history reload/optimistic-message preservation and active-memory timeout containment, and the narrower #70702 was closed as implemented for the assistant-reply-until-refresh symptom. But #67035 is broader: the reporter provided concrete textarea/IME draft-state anchors, multiple users reported input swallowing across platforms, and current main still renders the chat textarea directly from `props.draft` with no focused/composing DOM-value preservation handlers. That remaining input/IME surface makes an implemented-on-main close too broad.

## Best Possible Solution

Keep #67035 open or split it into narrower tracked work: keep the shipped Control UI stale-reload fixes, link #70702/#71878 as the solved assistant-reply-until-refresh subset, and add a focused Control UI textarea/IME fix in `ui/src/ui/views/chat.ts` with browser/unit coverage for focused composition draft preservation. If maintainers decide the remaining active-memory or explicit accepted/processing UX work should be separate, create narrower follow-ups before closing this umbrella.
