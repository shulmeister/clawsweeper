## Summary

Do not close. The same WebChat action-button overlap path is not cleanly fixed on current main: the CSS reserves right padding only for `.chat-bubble.has-copy`, but the current grouped message renderer still builds `bubbleClasses` from `chat-bubble`, `streaming`, and `fade-in` only, while `hasActions` is computed separately afterward. A later related issue/PR discussion (#61514/#61872) is useful evidence, but current main still does not show the class application needed for the original report.
