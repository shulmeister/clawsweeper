## Summary

Keep open. Current main partially addresses the related hook bypass by adding an optional beforeDeliver seam and inbound message_sending wiring, but it does not implement the issue's requested universal fail-closed structured-output validation gate at the native reply-dispatch handoff. The dispatcher still only runs beforeDeliver when supplied, and the auto-wired inbound hook is conditional on registered message_sending hooks and only rewrites/cancels text content. Structured ReplyPayload.presentation normalization/degradation exists later in the outbound delivery path, not as a canonical pre-send validator in reply-dispatcher.
