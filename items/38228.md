## Summary

Keep open. Current main still does not extract array-shaped `tool_result.content` in the embedded tool-card path, while OpenClaw tool results and docs still use `content: [{ type: "text", text: ... }]` / `content[0].text`. The PR is addressing a real remaining bug, but the submitted patch needs revision because its `String(firstItem.text)` behavior can render junk values and it only reads the first array element.

## Best Possible Solution

Keep the PR open or supersede it with a small guarded main-branch fix in `ui/src/ui/chat/tool-cards.ts` that extracts only string text from array-shaped tool-result content, ideally by reusing shared text-block extraction or by joining all string text blocks. Add regression coverage in `ui/src/ui/chat/tool-cards.node.test.ts` for embedded `tool_result` blocks with `content: [{ type: "text", text: "..." }]`, and consider standalone `toolResult` coverage if the render path depends on it.
