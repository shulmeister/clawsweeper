## Summary

Keep open. Current main still does not extract array-shaped tool-result content in Control UI tool cards, and the PR discussion identifies defects in the submitted patch rather than evidence that the work is obsolete or already shipped.

## Best Possible Solution

Keep this PR or a canonical replacement open until main gets a guarded tool-card extraction fix, preferably reusing or aligning with the shared text-block extraction contract, plus regression coverage in `ui/src/ui/chat/tool-cards.node.test.ts` for `content: [{ text: ... }]` and non-string/nullish text values. If a maintainer lands that directly on main, close this stale PR as superseded by that commit.
