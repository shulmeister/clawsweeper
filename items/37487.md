## Summary

Current main still does not implement the requested first-class per-profile browser proxy configuration or per-agent default browser profile binding. The issue discussion includes a useful external workaround, m13v/browser-lock, and PR #42806 proposed the right kind of core changes, but that PR was closed unmerged and current main only has global `browser.extraArgs` proxy support.

## Best Possible Solution

Keep #37487 open until OpenClaw implements or explicitly rejects the core product surface: per-profile proxy config with secret/redaction handling, managed-browser launch wiring, agent/subagent default browser profile resolution, clear profile/proxy diagnostics, and docs/tests. The m13v/browser-lock workaround is useful prior art, but the remaining work belongs in the bundled browser/agent config path rather than being considered solved by current main.
