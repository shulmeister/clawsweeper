## Summary

Current `main` still does not implement the requested first-class per-profile browser proxy configuration or per-agent browser profile binding. The only managed-browser proxy path is the existing global `browser.extraArgs` Chrome flag workaround, and PR #42806, which proposed the requested bundled browser/agent config surface, is closed unmerged. The `m13v/browser-lock` discussion is useful prior art for isolation, ports, launch locking, and per-instance proxy flags, but it is not a shipped OpenClaw core/browser-plugin solution.

## Best Possible Solution

Keep #37487 open. The best path is a core/bundled browser-plugin implementation: add optional `browser.profiles.<name>.proxy` with secret/redaction handling, apply it to managed local browser launches, expose safe `proxyConfigured`/diagnostic status, add `agents.defaults.browser.profile` and `agents.list[].browser.profile` with clear sub-agent inheritance semantics, and document/test concurrent isolated multi-profile usage. Treat PR #42806 as a closed unmerged implementation attempt and `m13v/browser-lock` as prior art, not as the shipped fix.
