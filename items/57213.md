## Summary

Current `main` already narrows persisted-model restarts to explicit `liveModelSwitchPending` requests, so a session that is merely pinned to the primary model no longer redirects or cancels fallback candidates. The PR's extra `ignorePersistedLiveModelSelection` threading is redundant against the active code path.

## Evidence
