## Summary

Keep PR #39975 open. Current main still serializes `message_reference` in `sendWebhookMessageDiscord` when `replyTo` is present, and the bound-thread webhook path can pass `replyToId` into that helper. Discord's current Execute Webhook parameter list still omits `message_reference`, while Create Message documents it for replies/forwards, so the PR's core fix remains valid even though the branch targets the old `src/discord/send.outbound.ts` path.

## Best Possible Solution

Keep the PR open or replace it with an equivalent refreshed patch against `extensions/discord/src/send.outbound.ts`: remove `message_reference` only from `sendWebhookMessageDiscord`, preserve bot-token/channel reply behavior in `sendMessageDiscord` and media/component send paths, and add a focused test proving `replyTo` does not serialize into Execute Webhook payloads.
