## Summary

Current `main` still rejects workspace bootstrap files when the file path resolves through a symlink outside the workspace root, and the bootstrap loader converts that rejection into `missing: true`. I did not find a merged fix or test coverage for the reported file-symlink case on `main`.

## Evidence

- **Bootstrap loader marks boundary-read failures as missing:** `loadWorkspaceBootstrapFiles()` calls `readWorkspaceFileWithGuards()` for `AGENTS.md`/`SOUL.md`/etc., and any non-OK result becomes `{ missing: true }`, which is what later renders as `[MISSING] Expected at: ...`.
  - file: [src/agents/workspace.ts:526](https://github.com/openclaw/openclaw/blob/ac063568d354c14e0a0a43bcffb9725bb3fca651/src/agents/workspace.ts#L526)
  - command: `nl -ba src/agents/workspace.ts | sed -n '477,540p'`
  - sha: [ac063568d354](https://github.com/openclaw/openclaw/commit/ac063568d354c14e0a0a43bcffb9725bb3fca651)
- **Guarded read still goes through strict boundary resolution:** `readWorkspaceFileWithGuards()` uses `openBoundaryFile({ rootPath: workspaceDir, boundaryLabel: "workspace root" })` and immediately returns that failure if boundary resolution rejects the path. There is no symlink-following fallback here.
  - file: [src/agents/workspace.ts:64](https://github.com/openclaw/openclaw/blob/ac063568d354c14e0a0a43bcffb9725bb3fca651/src/agents/workspace.ts#L64)
  - command: `sed -n '1,120p' src/agents/workspace.ts`
  - sha: [ac063568d354](https://github.com/openclaw/openclaw/commit/ac063568d354c14e0a0a43bcffb9725bb3fca651)
- **Boundary resolver rejects symlinks whose canonical target leaves the workspace root:** `applyResolvedSymlinkHop()` throws `Symlink escapes ...` whenever a symlink hop resolves outside `rootCanonicalPath`. That matches the reported `SOUL.md -> ../workspace-shared/SOUL.md` setup.
  - file: [src/infra/boundary-path.ts:311](https://github.com/openclaw/openclaw/blob/ac063568d354c14e0a0a43bcffb9725bb3fca651/src/infra/boundary-path.ts#L311)
  - command: `nl -ba src/infra/boundary-path.ts | sed -n '305,320p'`
  - sha: [ac063568d354](https://github.com/openclaw/openclaw/commit/ac063568d354c14e0a0a43bcffb9725bb3fca651)
- **Current tests cover symlinked workspace roots, not symlinked bootstrap files escaping the workspace:** There is a test proving a symlinked workspace directory works, but no corresponding test for a bootstrap file symlink whose target lives outside the workspace. That gap aligns with the still-unhandled case in the code.
  - file: [src/agents/workspace.load-extra-bootstrap-files.test.ts:54](https://github.com/openclaw/openclaw/blob/ac063568d354c14e0a0a43bcffb9725bb3fca651/src/agents/workspace.load-extra-bootstrap-files.test.ts#L54)
