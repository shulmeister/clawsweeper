## Summary

Keep open. Current main still loads workspace bootstrap files through the boundary-safe open path and marks any failed guarded read as missing, while the boundary resolver still rejects symlink targets that resolve outside the workspace root. The issue has concrete reproduction steps and linked PR work, but the related fixes are not merged on main.

## Best Possible Solution

Implement a narrowly scoped bootstrap-file symlink path in src/agents/workspace.ts, or an explicitly versioned boundary-file option if maintainers prefer that seam. It should allow user-created AGENTS.md/SOUL.md/etc. symlinks to shared files while enforcing the existing max size and regular-file constraints, preserving hardlink rejection, and adding regression coverage for relative symlinks, absolute symlinks, dangling symlinks, oversized targets, and hardlinks. Keep #38622 open until that lands or maintainers make an explicit product decision; use the existing linked PRs as reference but do not close the issue as shipped today.
