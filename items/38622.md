## Summary

Keep #38622 open. Current main still routes workspace bootstrap files through a workspace-root boundary open, rejects symlink targets that resolve outside the workspace, and converts that failure into the injected `[MISSING] Expected at: ...` marker. Related fix PRs are not merged, so the reported behavior is still actionable in core.

## Best Possible Solution

Keep the issue open and land a focused core fix in `src/agents/workspace.ts`: for explicitly present standard workspace bootstrap files, follow symlink targets for injection while preserving `MAX_WORKSPACE_BOOTSTRAP_FILE_BYTES`, dangling-symlink missing behavior, hardlink rejection, and the global boundary-file read semantics for other surfaces. Add regression tests for relative/absolute symlinks, dangling symlinks, and hardlinks.
