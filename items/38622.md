## Summary

Keep #38622 open. Current main still marks workspace bootstrap files as missing when the file inside the workspace is a symlink whose target resolves outside the workspace root. The related PRs/comments describe the missing fallback, but that fallback is not present in current main and the remaining work is still a concrete OpenClaw core behavior issue.

## Best Possible Solution

Keep the issue open and land a focused core fix for workspace bootstrap-file injection: allow user-created symlinked bootstrap files such as AGENTS.md/SOUL.md to be read when explicitly present in the workspace, while preserving size limits and avoiding broad relaxation of boundary-file security. Add regression tests for relative and absolute symlinks to shared workspace files, plus dangling symlink and hardlink behavior.
