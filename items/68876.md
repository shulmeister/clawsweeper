## Summary

Current main already implements the PR's corpus-noise fix for memory-core dreaming. Cron-run transcripts are classified from the session store and skipped by session ingestion, direct generated cron/heartbeat/silent-reply scaffolding is stripped before corpus export, and regression tests cover the fresh session corpus behavior. The implementation is present in the v2026.4.24 release tag.

## Evidence
