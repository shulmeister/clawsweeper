## Summary

Current `main` already contains a targeted memory-core dreaming narrative cleanup implementation: it logs `deleteSession` failures, scrubs stale dreaming session-store entries and orphan transcripts across agent session directories, and has focused tests for timeout/cleanup failure, orphan scrubbing, and per-workspace session isolation. The fixing code is on `main` in commit `6d271762ab50fb4376b8859bcf55eefb2d3814ea`, but it is not in the latest tagged release `v2026.4.23`.

## Evidence

- **Narrative cleanup now logs delete failures and runs an orphan scrubber:** `generateAndAppendDreamNarrative()` no longer swallows cleanup failures silently: its `finally` block warns on `deleteSession` failure and then calls `scrubDreamingNarrativeArtifacts()` to clean stale narrative session artifacts.
