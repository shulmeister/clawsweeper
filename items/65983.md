## Summary

Keep open. Current main still has the lifecycle gap described in #65983: background exec PTY runs are tracked only by in-memory supervisor/session maps, `reconcileOrphans()` remains a deliberate no-op, and the linked fix PR #66001 is still open/unmerged and explicitly covers only the in-process/session-loss half of the issue, not durable restart orphan reaping.

## Best Possible Solution

Keep #65983 open until OpenClaw either lands a scoped fix such as #66001 plus an explicit follow-up for durable restart orphan cleanup, or implements a complete active-exec ownership model: persisted PTY/background exec metadata, startup reconciliation that kills or re-registers stale runs, cleanup on normal finalize, and regression tests covering both in-process session-loss and gateway-restart/cold-start behavior.
