## Summary

Current main still has the reported macOS browser proxy auth gap: local gateway token/password resolution only accepts plain strings, so a SecretRef dictionary resolves to nil before MacNodeBrowserProxy builds auth headers. The PR discussion identifies the same real root cause, and the latest PR update appears to be useful work rather than obsolete, duplicate, or already implemented on main.
