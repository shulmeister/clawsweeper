## Summary

Keep open. Current main has not implemented this PR: generated gateway install-token persistence still defaults a missing `gateway.auth.mode` to `"token"`, and the existing test asserts that behavior. The docs also state gateway auth is required by default and reserve `auth.mode="none"` for explicit trusted loopback setups, so this remains a product/security-policy change rather than a cleanup close.

## Best Possible Solution

Keep this PR open for explicit maintainer product/security judgment, but do not merge it as-is. If local friction should be reduced, implement a narrower design that only permits `auth.mode="none"` when the effective exposure is proven local-only, preserves token auth for daemon/non-loopback/private-ingress and Tailscale Serve risk paths, updates gateway docs/security guidance, and adds regression coverage for loopback, lan/tailnet/custom binds, Tailscale Serve, startup auth, and generated-token persistence.
