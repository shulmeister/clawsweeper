## Summary

Keep #69849 open. Current main has upgraded to @agentclientprotocol/sdk 0.20.0 and acpx 0.6.1, but the pinned ACP SDK still contains optional array schemas without default [] values, and OpenClaw does not appear to add a local Anthropic/Claude schema-default normalization layer for this path. The issue is recent, unprotected, and still actionable in this repo via an upstream SDK bump or a narrow OpenClaw compatibility adapter once the desired contract is clear.

## Best Possible Solution

Keep the issue open. The best path is to either upgrade to an upstream @agentclientprotocol/sdk release that adds default [] metadata for the affected optional arrays, or add a narrow OpenClaw compatibility normalization layer at the ACP/Anthropic tool-schema boundary with a regression test that exercises the Claude/API Pro schema registration failure. Avoid a dependency patch unless a maintainer explicitly approves it.
