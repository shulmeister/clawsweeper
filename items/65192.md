## Summary

Current `main` already prevents dreaming-generated sessions from re-entering session ingestion and adds explicit dedupe/repair recovery paths for accumulated Dream Diary artifacts, so this issue is implemented on main but not yet in a tagged release.

## Evidence

- **Unreleased changelog explicitly calls out the fix:** The unreleased `2026.4.24` changelog says dreaming now stops re-ingesting its own narrative transcripts and adds explicit repair/dedupe recovery flows in CLI, doctor, and the Dreams UI.
