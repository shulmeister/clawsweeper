## Summary

Close PR #70742 as implemented on current main. The shipped cleanup path already waits for idle, clears pending tool-call state, and returns before normal synthetic-result flushing when teardown timeout cleanup is requested. All production embedded-runner cleanup call sites pass `clearPendingOnTimeout: true`, and regression coverage plus the 2026.3.8 changelog cover the clear-vs-flush behavior. The PR's remaining broader change to skip flushing on every timeout is not needed for the cleanup bug and would change the existing default helper contract.

## Best Possible Solution

Close #70742 as already handled on main by the shipped cleanup-specific timeout hardening. Keep the current helper contract: teardown cleanup paths use `clearPendingOnTimeout: true` to clear and return, while non-cleanup callers may still flush after a best-effort idle wait timeout unless maintainers open a separate issue to change that behavior.
