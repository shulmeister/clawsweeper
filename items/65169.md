## Summary

Keep this PR open. Current main still has the pre-approval command exposure path the PR targets: unpaired node reconnects return the node's declared commands, those commands are stored on the live node session, and `node.invoke` authorizes against that live command list plus the platform allowlist. The linked contributor issue #65168 is also still open, so this paired PR should remain available for maintainer review rather than being closed by cleanup.

## Best Possible Solution

Keep the PR open and review it as an implementation candidate for #65168. The likely product fix is to fail closed for unapproved node reconnects by publishing an empty command set until `node.pair.approve` succeeds, with the existing allowlist test inverted and the `node.invoke` approval-bypass fixture adjusted to use legitimately node-paired test nodes. Maintainers may also choose to add a direct node-pair approval check in `node.invoke`, but current main does not yet provide the documented protection.
