## Summary

Keep #10659 open. Current main has useful SecretRef storage/resolution, marker persistence, host exec env sanitization, and plugin hook seams that make the linked `jmkritt/openclaw-secrets-hardening` and `openclaw-credential-vault` mitigations credible. It still does not implement the native masked-secrets feature requested here: no `{{secret:NAME}}` substitution, no `secrets.mask`/`maskedSecrets` config, no native configured redaction of arbitrary secret values from exec/process output before the model sees them, and no generic block on reading readable `.env` files or arbitrary secret env vars.

## Best Possible Solution

Keep #10659 open until OpenClaw ships a native masked-secret or credential-provider/broker path for agent-executed commands and process output, or maintainers explicitly supersede it with a canonical design such as #59165. Continue acknowledging `jmkritt/openclaw-secrets-hardening` and `openclaw-credential-vault` as useful mitigations, not closure evidence for the core request.
