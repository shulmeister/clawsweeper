## Summary

Keep #10659 open. Current main has substantial SecretRef support and plugin hooks that can power third-party vault plugins, but it still does not implement the requested native masked-secrets exec flow: `{{secret:NAME}}` substitution, `secrets.mask` or `security.maskedSecrets` config, command blocking for secret reads, and output redaction before exec/process results reach the agent. The linked core PR #22744 was closed unmerged, and the issue discussion now includes an explicit Codex review comment keeping the item open.

## Best Possible Solution

Keep #10659 open until OpenClaw either ships a native masked-secrets/credential-broker flow for exec and process output, or maintainers explicitly supersede it with a canonical credential-provider/plugin design such as #59165. Third-party workarounds should be acknowledged, but they do not yet replace the native security/product request.
