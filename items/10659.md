## Summary

#10659 should stay open. Current main has useful SecretRef support for supported credential fields and plugin hooks that can power third-party workarounds, but it does not implement the requested native masked-secrets exec flow: `{{secret:NAME}}` substitution, command blocking for secret reads, and redaction of exec/process output before the agent sees it. The linked masked-secrets PR #22744 was closed unmerged, and broader credential-provider work is still tracked in open related items.

## Evidence
