## Summary

Keep #10659 open. Current main has meaningful SecretRef support and plugin hooks, but it still does not implement the requested native masked-secrets flow: `{{secret:NAME}}` substitution in agent-executed commands, a `secrets.mask`/`maskedSecrets` config surface, command blocking for direct secret reads, or guaranteed redaction of exec/process output before it reaches the agent. The provided discussion also includes a same-day maintainer-associated Codex review explicitly keeping the issue open, plus linked core PRs that were closed unmerged rather than shipped.

## Best Possible Solution

Keep #10659 open until OpenClaw ships a native masked-secrets or credential-broker flow for exec/process output, or maintainers explicitly supersede it with a canonical design such as #59165. The existing third-party workaround and plugin/vault approaches should be acknowledged as useful, but they do not close the native security/product request.
