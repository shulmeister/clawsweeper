## Summary

Current main and the v2026.4.24 release already solve the reported status-card context-window bug through the newer runtime-aware status path. The PR's broad empty-string normalization is now obsolete and would conflict with the intentionally preserved provider-less runtime sentinel called out in review.

## Best Possible Solution

Close this PR without merging its exact patch. Keep the shipped runtime-aware status implementation and sentinel-preserving session-status behavior; any additional coverage should be a narrow regression around `providerOverride: ""` plus a known `modelProvider`, not broad normalization of empty provider overrides.

## Evidence

- **Status tool preserves the sentinel only for provider-less runtime models:** `session_status` computes `hasExplicitModelOverride`, reads the runtime provider/model, and only injects `providerOverride: ""` when there is a runtime model but no runtime provider. Known runtime providers flow through to the status card instead of being rewritten as empty overrides.
