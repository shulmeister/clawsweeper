## Summary

Keep open. Current main still embeds the Slack manifest JSON inside the Slack setup intro note, so the underlying copy/paste bug from #65751 is not implemented on main. PR #65809 targets the right bug, but its proposed `console.log`/`prepare()` approach needs revision because gateway/UI wizard flows only deliver `WizardPrompter` session steps, not arbitrary stdout.

## Best Possible Solution

Keep this PR or the canonical bug #65751 open and request a revised fix that makes the Slack manifest copyable through a WizardPrompter/session-capable surface. The likely direction is to separate the manifest from the framed instructions while still emitting it through a transport that both CLI and gateway/UI wizard clients can render, preserving env-shortcut suppression behavior and adding regression tests for the Slack setup wizard path.
