## Summary

Keep #62276 open. Current main has better port-conflict diagnostics, service stop hints, systemd restart limits, and deep service discovery, but the remaining reported class is not fully resolved: the Linux comment describes a user-systemd service installed by onboarding colliding with an existing system-level gateway service on the same port, and current `installSystemdService` still writes/enables/restarts the user unit without first scanning or warning about `/etc/systemd/system` listeners. The related #28191/#70664 items do not fully supersede this Windows/Linux report.
