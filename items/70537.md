## Summary

Keep #70537 open. Current main has safer dependency behavior than the reporter's f788c88 build, but the core gateway/OpenResponses serialization path is materially the same as the reported version and main does not contain the exact array-with-special-characters regression test proposed by the related fork PR. The issue is recent, has a concrete reproduction, and the linked fix attempts are still external/open rather than landed in openclaw/openclaw.

## Best Possible Solution

Keep the issue open and land a focused OpenClaw regression for tool calls whose arguments include array fields with quotes/newlines on the affected gateway/SSE path. If the dependency upgrade is the intended fix, document that with a targeted test proving f788c88-style payloads now parse deterministically; otherwise normalize the function_call arguments boundary in src/gateway/openresponses-http.ts and cover both streaming and non-streaming responses.
