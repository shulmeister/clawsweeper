## Summary

Keep open. Current main still does not implement quoted/replied Feishu media resolution: it resolves media only from the triggering message, builds the agent media payload before fetching the quoted message, and fetched Feishu messages still expose only parsed placeholder content without the raw JSON needed to extract image_key/file_key. The PR discussion and related #41675 describe a real Feishu group-chat workflow gap, so the useful fix should be rebased/adapted rather than auto-closed.

## Best Possible Solution

Keep the PR open, but require a current-main rebase that preserves newer Feishu routing, ACP conversation binding, thread bootstrap, broadcast, and outbound identity flows. The focused implementation should add rawContent to FeishuMessageInfo, resolve quoted/replied media with the same resolver used for current-message media, build the final agent media payload after all media is collected, and add a regression test proving quoted image/file media reaches MediaPaths/MediaTypes.
