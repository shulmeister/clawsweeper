## Summary

Keep open. Current `main` still matches the reported surrogate-pair bug: the coarse chunking pass can split an emoji across UTF-16 code units, the existing test only covers the fine-split CJK path, and embedding inputs are still forwarded without surrogate sanitization.

## Evidence

- **Coarse chunking still slices on raw UTF-16 boundaries:** `chunkMarkdown` still advances the outer pass with `start += maxChars` and slices `line.slice(start, start + maxChars)` before any surrogate-pair check. The only boundary guard is inside the nested fine-split path, so Latin-heavy lines with a single emoji can still be cut mid-pair.
  - file: [src/memory-host-sdk/host/internal.ts:427](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/memory-host-sdk/host/internal.ts#L427)
  - command: `nl -ba src/memory-host-sdk/host/internal.ts | sed -n '423,446p'`
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **Current test coverage misses the reported path:** The regression test in `internal.test.ts` exercises a long all-surrogate CJK line, which routes through the fine-split logic. There is no exact-boundary test for a mostly-ASCII line with a single emoji at the coarse-split boundary.
  - file: [src/memory-host-sdk/host/internal.test.ts:452](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/memory-host-sdk/host/internal.test.ts#L452)
  - command: `nl -ba src/memory-host-sdk/host/internal.test.ts | sed -n '452,490p'`
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **Embedding text is still passed through unsanitized:** `buildTextEmbeddingInput` returns `{ text }` unchanged, and the remote embedding provider forwards string arrays directly into the JSON request body as `input` without replacing unpaired surrogates.
  - file: [src/memory-host-sdk/host/embedding-inputs.ts:19](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/memory-host-sdk/host/embedding-inputs.ts#L19)
  - command: `nl -ba src/memory-host-sdk/host/embedding-inputs.ts | sed -n '14,24p'`
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **Direct reproduction against the current algorithm still breaks the emoji:** A Node reproduction using the current coarse-split logic on `"a".repeat(39) + "🌸"` with `maxChars=40` produced `"...\ud83c"` and `"\udf38"`, confirming both output segments contain broken surrogate halves.
  - command: `node -e 'const line="a".repeat(39)+"🌸"; const maxChars=40; const coarse=line.slice(0,maxChars); const next=line.slice(maxChars,maxChars*2); ...'`
