## Summary

Keep #65782 open. Current main at c3833f7729163a8f6cdc47d3b0be86fd5fcab160 still has the reported coarse chunker surrogate-pair gap and still sends embedding text/inputs to providers without an unpaired-surrogate sanitizer. The linked PR #65783 appears to target the right bug, but it is open and unmerged, so the issue is still actionable.

## Best Possible Solution

Keep the issue open and review/land #65783 or an equivalent fix: add a surrogate-boundary guard to the outer `chunkMarkdown` coarse split, sanitize unpaired surrogates at the embedding boundary for both text and structured inputs, and add the coarse-boundary regression test before closing the issue via the fixing PR.
