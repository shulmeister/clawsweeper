## Summary

Current main still does not discover, diagnose, or repair the specific `~/.openclaw/moltbot.json` layout reported here. The adjacent gateway crash-loop messaging has improved, and #54201 is closed, but that only addresses part of the symptom; this issue's migration/repair gap remains open.

## Best Possible Solution

Keep this issue open for a narrow maintainer fix. Avoid restoring broad runtime legacy auto-detection, but add an explicit Doctor/startup diagnostic for `~/.openclaw/moltbot.json` when canonical `openclaw.json` is missing, empty, or skeletal; offer a safe `doctor --fix` copy or merge path when feasible; and update the public migration/changelog wording so affected users know how to recover.
