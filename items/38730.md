## Summary

Keep open. Current main has real cron-session retention mitigation, but the requested guardrails are still incomplete: high-frequency `every` cron jobs can still be created through CLI, Gateway RPC, Control UI, and the agent cron tool without a warning, and the linked warning PR #38731 is still open/unmerged in the provided GitHub context.

## Best Possible Solution

Keep #38730 open until PR #38731 or an equivalent fix lands. The best product/code path is a shared high-frequency `every` schedule warning used by CLI, Gateway RPC, Control UI, and agent-tool-created cron jobs, plus a maintainer decision on whether manual `openclaw sessions cleanup` should invoke cron run-session retention or whether cron cleanup remains scheduler-owned and clearly documented.
