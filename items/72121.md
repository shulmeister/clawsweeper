## Summary

Keep open. Current main still rejects `sk-ant-api03-...` on the reported `models auth paste-token --provider anthropic` path, and open PR #72124 is an unmerged implementation candidate that explicitly links this issue.

## Best Possible Solution

Review and land PR #72124 or an equivalent patch that either accepts both `sk-ant-oat01-` setup tokens and `sk-ant-api03-` API keys on the paste-token path with focused validator coverage, or deliberately preserves separate auth methods while updating CLI help/errors and docs to direct API-key users to `models auth login --provider anthropic --method api-key`.
