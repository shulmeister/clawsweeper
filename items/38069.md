## Summary

Keep #38069 open. Current main still has global one-shot transient retry and a recurring error-backoff floor, but it does not implement configurable delayed re-execution for failed recurring cron jobs. The related implementation PR #69252 is closed unmerged, and the public config docs explicitly say `cron.retry` applies only to one-shot jobs.

## Best Possible Solution

Keep #38069 open until recurring cron retry is deliberately designed and shipped. The likely path is an additive retry policy in cron domain types and gateway schemas, CLI add/edit flags, explicit retry-attempt state distinct from natural schedule state, transient-error classification for recurring executions, scheduling at endedAt plus configured delay before the next daily/weekly slot, and regression docs/tests that distinguish one-shot retry, recurring backoff floor, and recurring execution retry.
