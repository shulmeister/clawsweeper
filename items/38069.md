## Summary

Keep #38069 open. Current main still does not implement configurable delayed re-execution for failed recurring cron jobs; the scheduler only retries one-shot transient failures and applies a backoff floor to recurring jobs, which still lets daily/weekly cron jobs wait for their natural next schedule.

## Best Possible Solution

Keep #38069 open until recurring cron retry is designed and shipped. The likely path is an additive retry policy in the cron job/config contract, CLI/API edit support, explicit retry attempt state separate from natural schedule state, transient-error classification for recurring runs, scheduling retries before the next natural daily/weekly slot without refire loops, and regression tests/docs that distinguish one-shot retry from recurring execution retry.
