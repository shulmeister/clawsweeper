## Summary

Keep #38069 open. Current main still does not implement configurable delayed re-execution for failed recurring cron jobs: `cron.retry` is wired for one-shot jobs, while recurring jobs use an error-backoff floor against the next natural schedule. For daily/weekly jobs, that still means waiting for the next scheduled slot. The provided related context also shows PR #69252 tried to address this exact gap but was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep #38069 open until recurring cron retry is designed and shipped. The likely implementation is an additive retry policy in the cron job/config contract, gateway protocol/schema support, CLI add/edit flags, explicit retry attempt state separate from natural schedule state, transient-error classification for recurring executions, scheduling retries at `endedAt + configured delay` before the next natural daily/weekly slot, and regression docs/tests that distinguish one-shot retry, recurring backoff floor, and recurring execution retry.
