## Summary

Keep open. Current main still lacks configurable delayed re-execution for failed recurring cron jobs; it only retries one-shot jobs and applies a lower-bound backoff to recurring jobs before their natural next scheduled run.

## Best Possible Solution

Keep #38069 open until recurring cron execution retry is designed and implemented. The likely maintainer path is to add an explicit recurring retry policy to the cron job/config contract, wire CLI/API editing, track retry attempts separately from normal schedule state, schedule transient failed recurring runs before the next natural daily/weekly slot, and add regression tests plus docs clarifying one-shot versus recurring retry behavior.
