## Summary

Current `main` already ships BM25-backed hybrid `memory_search` for the builtin memory engine, so the requested feature is implemented well enough on OpenClaw core. The current implementation uses weighted hybrid merging rather than the exact RRF formula proposed in the issue, but the requested BM25+embedding hybrid retrieval itself is already present and documented.

## Risks / Open Questions

- The shipped implementation uses weighted score merging, not the exact Reciprocal Rank Fusion formula proposed in the issue.
- If maintainers want RRF as an additional ranking mode, that would be a narrower follow-up enhancement rather than this feature request.

## Close Comment

Closing as implemented on `main` after a Codex review. OpenClaw already ships BM25-backed hybrid `memory_search`: the current code runs keyword and vector retrieval together in `extensions/memory-core/src/memory/manager.ts:391`, the user docs describe BM25 + vector hybrid retrieval in `docs/concepts/memory-search.md:48`, and the feature was recorded in the `v2026.1.20` changelog at `CHANGELOG.md:5957` as `Memory: add hybrid BM25 + vector search (FTS5) with weighted merging and fallback.` This means the core request here is already present. The one mismatch is that current OpenClaw uses weighted hybrid merging rather than the exact RRF formula suggested in the issue; if an alternate fusion mode is still desired, please open a new issue scoped specifically to RRF support.
