## Summary

Keep #69051 open. Current main has the broader TTS persona foundation, but it still does not implement this PR-A's source-text metadata, speech-provider capability declaration, ElevenLabs expressive-tag allowlist, or per-provider fallback text routing. The PR is active, recently narrowed after maintainer feedback, and still has unresolved review discussion that needs maintainer judgment rather than automated cleanup closure.

## Best Possible Solution

Keep #69051 open as the active narrow PR-A proposal. The best path is for maintainers to review the source-text metadata and provider capability seam directly, require fixes for the latest fallback-routing review comments and any public SDK type exposure gaps, then either land this narrow implementation or replace it with a clearly linked superseding PR that implements the same provider-aware source/plain text routing contract.
