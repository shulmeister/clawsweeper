## Summary

Current main already solves the reported subagent completion cleanup failure by routing registry cleanup through `runSubagentAnnounceFlow`, which deletes the child session directly. The private `cleanupProvisionalSession` helper is only used inside `subagent-spawn.ts`, so this export-only PR is obsolete and would not be the right fix for the original runtime symptom.

## Best Possible Solution

Close this export-only PR and keep the shipped announce-flow cleanup implementation. If #68398 has a current reproduction on v2026.4.24 or later, it should be handled as a targeted bug against the current `runSubagentAnnounceFlow` or delivery path, not by exporting the private spawn cleanup helper.
