## Summary

Current main and the v2026.4.24 release already avoid the reported cross-chunk `cleanupProvisionalSession` failure: registry cleanup delegates to `runSubagentAnnounceFlow`, and that module deletes the child session directly via `sessions.delete`. The helper in `subagent-spawn.ts` remains private and is only used inside that file, so this PR's export-only change is no longer needed and would not be the right fix for the original runtime symptom.

## Best Possible Solution

Close this export-only PR and keep the shipped announce-flow cleanup implementation. If #68398 still has a current reproduction on v2026.4.24 or later, it should be handled with a new targeted bug report against the current `runSubagentAnnounceFlow`/delivery path, not by exporting the private spawn cleanup helper.
