## Summary

Keep #16387 open. Current main still treats configured provider headers as static strings, the Fireworks provider does not add `x-session-affinity`, and the available provider-plugin hooks do not provide the requested one-line config templating for `{{session.key}}`, `{{agent.id}}`, or `{{run.id}}`.

## Best Possible Solution

Keep the issue open. The best product path is a narrow, documented dynamic-header implementation for `models.providers.*.headers` and probably `models.providers.*.request.headers`, resolved at stream/transport dispatch with explicit variables such as `session.key`, `session.id`, `agent.id`, and `run.id`. Cover main turns and compaction, OpenAI completions/responses transports, and tests for Fireworks `x-session-affinity` plus LiteLLM/Langfuse-style headers. If maintainers prefer provider-owned behavior instead, extend the provider hook context to expose the needed session key/agent/run identifiers and document that config headers remain static.
