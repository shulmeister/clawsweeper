## Summary

Keep #16387 open. Current main still preserves configured model-provider headers as literal strings, documents them as static headers, and the bundled Fireworks provider does not add `x-session-affinity`. Newer provider-owned turn-state hooks can attach per-turn headers for provider plugins, but they do not implement the requested `models.providers.*.headers` templating for custom config providers or expose the requested `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` config variables.

## Best Possible Solution

Keep the issue open. The best maintainer path is either a narrow, documented dynamic-header resolver for `models.providers.*.headers` and `models.providers.*.request.headers` at request dispatch, or an explicit product decision to solve this provider-by-provider with `resolveTransportTurnState`. If using config templating, cover at least `session.key` or `session.id`, `agent.id`, and `run.id`/turn id, and add tests for Fireworks `x-session-affinity`, LiteLLM/Langfuse-style headers, main turns, compaction, and OpenAI-compatible transports.
