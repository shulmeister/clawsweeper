## Summary

Keep #16387 open. Current main still does not implement dynamic interpolation for provider config headers such as `{{session.key}}`, `{{agent.id}}`, or `{{run.id}}`. The repo now has a bundled Fireworks provider and provider-plugin hooks for custom headers, but those do not satisfy the requested one-line config templating behavior, and the Fireworks plugin does not add `x-session-affinity` itself.

## Best Possible Solution

Keep the issue open and implement a narrow, documented dynamic-header path if maintainers want this product behavior: resolve `models.providers.*.headers` and likely `models.providers.*.request.headers` at stream/transport dispatch with explicit allowed variables such as session key/id, agent id, and per-turn run/turn id; add tests for Fireworks affinity and LiteLLM/Langfuse-style headers. If maintainers prefer provider-owned behavior, use `resolveTransportTurnState` in the relevant provider plugin and document that config headers remain static, but that would be a product decision rather than an automatic cleanup close.
