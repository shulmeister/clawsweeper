## Summary

Keep #16387 open. Current main still treats configured provider headers as static strings for both `models.providers.*.headers` and `models.providers.*.request.headers`; there is no generic interpolation for `{{session.key}}`, `{{agent.id}}`, or per-run values. The bundled Fireworks plugin also does not add `x-session-affinity`, and the provider-owned hook surface is plugin code, not a dynamic custom-provider config mechanism. The Fireworks prompt-cache and LiteLLM/Langfuse use cases from the issue discussion remain unimplemented; related PRs #16448 and #18213 were closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep the issue open. The best maintainer path is a narrow, documented dynamic header resolver for `models.providers.*.headers` and `models.providers.*.request.headers`, evaluated at request dispatch with explicit supported placeholders for session key, agent identity, and per-turn/run identity. Add regression tests for Fireworks `x-session-affinity`, LiteLLM/Langfuse-style headers, main turns, compaction, retries, and OpenAI-compatible transports; if maintainers prefer provider-owned behavior only, document how custom providers can achieve the same result without a missing core API.
