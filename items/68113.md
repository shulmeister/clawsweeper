## Summary

Keep open. Current main and latest tagged release still use a module-local Mattermost slash-command state map, while the registered HTTP route returns the reported 503 whenever that map is empty. The linked PR #69038 is the plausible fix and reporter-verified workaround, but the provided GitHub context says it is still open/unmerged; #70145 is a duplicate of this original issue, not a better canonical tracker.

## Best Possible Solution

Keep #68113 open as the canonical Mattermost slash-command 503 regression. Land PR #69038 or an equivalent fix that shares slash-command account state across the bundled sidecar route loader and monitor activation path, then add regression coverage for split loader/module-instance behavior and verify the fix in the next npm release before closing this issue. #70145 can be treated as a duplicate of this tracker.
