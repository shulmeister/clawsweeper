## Summary

Keep open. Current main still does not render the explicit agent identity emoji with the assistant name in the web chat message header/footer path. The gateway can resolve an emoji, but the Control UI bootstrap and UI AssistantIdentity shape drop it, and renderMessageGroup only renders assistantName as the sender label. The linked PR #45898 was closed unmerged and targeted the selector dropdown, not the chat message header described in #45755.
