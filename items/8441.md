## Summary

Keep #8441 open. Current main still does not support top-level `skills.entries.<skill>.thinking` or `skills.entries.<skill>.model`: the public config type and strict schema omit those keys, existing tests preserve rejection of unknown top-level skill entry fields, skill command specs have no override fields, and skill invocation only rewrites the prompt or dispatches a tool. The related implementation PR #8455 is closed unmerged in the provided context, and the latest issue discussion already kept this open.

## Best Possible Solution

Keep #8441 open. A good implementation should extend the public `SkillConfig` type and strict Zod schema with validated optional `model` and `thinking` defaults, carry those values through `SkillCommandSpec` construction, apply them during skill invocation as defaults that yield to explicit per-turn/session directives, and update config/runtime tests plus the public skills config and thinking precedence docs.
