## Summary

Keep #8441 open. Current main still does not implement top-level `skills.entries.<skill>.thinking` or `.model` defaults: the public skill config type and strict schema omit those fields, the command spec has no place to carry them, skill command construction does not resolve them, and invocation rewrites the prompt without applying model or thinking directives. The related implementation PR #8455 was closed unmerged, and the latest issue discussion explicitly says the request remains active.

## Best Possible Solution

Keep #8441 open. A good implementation would extend the public `SkillConfig` type and strict Zod schema with validated optional `model` and `thinking` defaults, carry those values through `SkillCommandSpec` generation, apply them during skill invocation only as defaults that respect explicit user/session directives, and update config/runtime tests plus `docs/tools/skills-config.md` and thinking precedence docs.

## Risks / Open Questions

- The main product decision is precedence: per-skill defaults should likely apply only when the user has not provided an explicit per-turn or session directive.
- The closed PR #8455 appears to have broadened directive parsing and made skill overrides take precedence over explicit user directives, so a future implementation should be narrower and covered by focused tests.
