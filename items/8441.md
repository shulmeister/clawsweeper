## Summary

Keep #8441 open. Current main still does not implement first-class `skills.entries.<skill>.thinking` or `skills.entries.<skill>.model`: the public `SkillConfig` type and strict Zod schema omit those fields, tests still assert unknown top-level skill-entry keys are rejected, docs direct custom keys into `config`, and skill slash-command invocation only rewrites the prompt body without carrying per-skill model/thinking defaults into model selection or directive handling. The related implementation PR #8455 is closed and unmerged, and the latest issue discussion already kept this open.

## Best Possible Solution

Keep #8441 open. Implement this in core skills config and invocation plumbing: add validated optional `model` and `thinking` fields to `SkillConfig` and `SkillEntrySchema`, carry resolved per-skill defaults through `SkillCommandSpec` or an equivalent invocation context, apply them during skill command handling as per-turn defaults with clear precedence against explicit directives/session overrides and agent/global defaults, add schema plus invocation/model-selection tests, and update `docs/tools/skills-config.md` and the public skills config docs.
