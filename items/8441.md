## Summary

Keep #8441 open. Current main still does not implement top-level `skills.entries.<skill>.thinking` or `skills.entries.<skill>.model`: the skill config type and strict schema omit those fields, the schema tests still preserve rejection of unknown top-level skill entry fields, command specs do not carry per-skill model/thinking defaults, and skill invocation rewrites or dispatches the skill without changing model/thinking selection. The related PR #8455 was closed unmerged, and the latest maintainer comment in the issue already reached the same keep-open conclusion.

## Best Possible Solution

Keep #8441 open. Implement this in core skill config and invocation plumbing: extend `SkillConfig` and `SkillEntrySchema` with validated optional `model` and `thinking`, carry defaults through `SkillCommandSpec` or an equivalent invocation context, apply them only as per-skill defaults below explicit message/session/user overrides, add focused schema and invocation tests, and update the public skills config and thinking/model docs.
