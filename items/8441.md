## Summary

Keep #8441 open. Current main still does not support `skills.entries.<skill>.thinking` or `skills.entries.<skill>.model`: the public config type and strict schema omit those keys, command specs do not carry them, skill invocation only rewrites the prompt, and docs still describe per-skill entries as `enabled`/`env`/`apiKey` plus custom data under `config`. The related implementation PR #8455 is closed unmerged in the provided GitHub context, and the latest issue discussion already says the feature remains active.

## Best Possible Solution

Keep #8441 open. A good implementation should extend the public `SkillConfig` type and strict Zod schema with validated optional `model` and `thinking` defaults, carry them through `SkillCommandSpec` construction, apply them during skill invocation as defaults that do not override explicit per-turn/session directives, and update config/runtime tests plus the public skills config and thinking precedence docs.
