## Summary

Keep open. Current `main` does not implement the PR’s requested OpenRouter audio transcription path. On `7130e56f872aea96577ac9fcbda0c7555a9ecc69`, the bundled OpenRouter media provider is still image-only, and the audio pipeline only accepts providers that expose `transcribeAudio`, so `provider: "openrouter"` is not a working audio-transcription backend on current `main`.

## Evidence

- **OpenRouter media provider is image-only on main:** `openrouterMediaUnderstandingProvider` declares `capabilities: ["image"]`, `defaultModels: { image: "auto" }`, and only `describeImage`/`describeImages`; there is no `transcribeAudio` implementation.
  - file: [extensions/openrouter/media-understanding-provider.ts:7](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/extensions/openrouter/media-understanding-provider.ts#L7)
  - command: `nl -ba extensions/openrouter/media-understanding-provider.ts | sed -n '1,260p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Audio pipeline rejects providers without transcribeAudio:** Media-understanding capability checks treat audio support as `Boolean(provider.transcribeAudio)`, and runtime key resolution returns `null` for audio when a provider lacks `transcribeAudio`. That means the current OpenRouter provider cannot satisfy `tools.media.audio.models` entries for audio transcription.
  - file: [src/media-understanding/provider-supports.ts:10](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/media-understanding/provider-supports.ts#L10)
  - command: `nl -ba src/media-understanding/provider-supports.ts | sed -n '1,120p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Current audio docs still exclude OpenRouter fallback/support:** The current audio docs list bundled provider auth fallback order as `OpenAI → Groq → Deepgram → Google → Mistral` and do not document an OpenRouter audio-transcription path or `provider: "openrouter"` example.
  - file: [docs/nodes/audio.md:34](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/docs/nodes/audio.md#L34)
