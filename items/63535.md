## Summary

Keep open. Current main still only performs retention-based cron run-session pruning; it does not implement stale `status: "running"` recovery. The submitted PR is a plausible fix direction, but the provided review discussion identifies unresolved correctness issues, and current main’s cron session persistence shape means the PR should be rebased/redesigned rather than closed as implemented or obsolete.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up. The best path is a rebased cron lifecycle recovery patch that runs independently from retention pruning, repairs the session row shape current main actually persists plus any legacy `:run:` aliases, and adds regression tests for `sessionRetention: false`, stale versus recent running sessions, and one-sweep deferral when retention would otherwise prune immediately. The related lifecycle bug in #56121 is useful context but does not supersede this PR before a corrected implementation exists.
