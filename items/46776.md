## Summary

Keep open. Current main partially addresses Feishu group fan-out with @all recognition, per-account dedupe, and broadcast-agent dispatch, but the issue also asks for the same agent to carry conversational context between DM and group sessions. Current routing and docs still keep direct and group sessions distinct, and the provided related-item context does not identify a narrower canonical follow-up for that remaining behavior.

## Evidence

- **Issue scope includes cross-session memory:** The issue body reports two behaviors: group @all delivery to more than three Feishu agents, and the same agent not knowing prior conversation when moving between a single-agent session and a group session. The expected behavior explicitly asks that messages to the same agent be understood in any session.
