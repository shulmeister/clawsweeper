## Summary

Keep #42606 open. The hardening request is still valid on current main: the live noVNC bootstrap route has moved to `extensions/browser/src/browser/bridge-server.ts` and still sends only cache/referrer headers before returning inline-script HTML. The PR's approach is useful, and Greptile's review supported the nonce/CSP design, but maintainer comments correctly note that the branch patches the old `src/browser/**` path and needs a narrow port/rebase before it can merge.

## Best Possible Solution

Port the same narrow hardening to `extensions/browser/src/browser/bridge-server.ts` and `extensions/browser/src/browser/bridge-server.auth.test.ts`: build the bootstrap response with a per-response script nonce, set a restrictive document CSP plus `X-Content-Type-Options: nosniff` and `X-Frame-Options: DENY`, and keep the existing one-time token and password-in-fragment behavior covered by tests.
