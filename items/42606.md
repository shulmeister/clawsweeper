## Summary

Current `main` does not include the hardening from PR #42606. The `/sandbox/novnc` handler in `extensions/browser/src/browser/bridge-server.ts` still returns inline bootstrap HTML without a CSP nonce, `Content-Security-Policy`, `X-Content-Type-Options`, or `X-Frame-Options`, and the corresponding test still only checks `Cache-Control` and `Referrer-Policy`. This is not implemented on `main`, so there is no basis to close the PR as already fixed or obsolete.

## Evidence

- **Route still lacks hardening headers on main:** The `/sandbox/novnc` handler sets `Cache-Control`, `Pragma`, `Expires`, and `Referrer-Policy`, then sends `buildNoVncBootstrapHtml(resolved)` directly. There is no `Content-Security-Policy`, `X-Content-Type-Options`, or `X-Frame-Options` header, and the bootstrap HTML still uses a plain inline `<script>` without a nonce.
