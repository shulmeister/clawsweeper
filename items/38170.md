## Summary

Keep PR #38170 open. Current main at e672b61417af5c45b0431df6d9109a1f4b618ef5 still lacks Telegram `download-file` support and still does not preserve Telegram `file_id` in parser-safe media placeholders/history. The PR’s workflow remains useful, including the body’s `tg-media-resolve` workaround context, but the branch must be refreshed because it edits removed pre-plugin Telegram paths.

## Best Possible Solution

Keep this PR or a refreshed follow-up open, but port the idea to `extensions/telegram`: expose `download-file` in `channel-actions.ts`, implement `downloadFile` in `action-runtime.ts` using the existing Telegram media transport/download helpers, embed or otherwise preserve `file_id` in a parser-safe inbound history/body contract, and add focused tests plus Telegram docs updates.
