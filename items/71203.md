## Summary

Keep this PR open. Current checked-out main still refreshes `models.json` only through `prewarmConfiguredPrimaryModel`, which returns before any cache write when no explicit default primary model is configured and otherwise refreshes only `resolveOpenClawAgentDir()`. The PR remains a focused, active implementation candidate for the startup refresh gap tracked by #71192, and the author already pushed a follow-up split in `ad393b8e4afe27c35f09c24c6177f6c83ff488e2` after review feedback.

## Best Possible Solution

Keep this PR open for review. The best path is to land or revise a startup helper that refreshes the unique configured agent dirs independently of primary-model prewarm, with regression coverage for non-default agents and the no-default-primary case while preserving default and override agent-dir semantics.
