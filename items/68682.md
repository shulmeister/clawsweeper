## Summary

Current main already implements the useful part of PR #68682 with a safer policy-based design: Anthropic's bundled Claude CLI backend no longer hardcodes `--permission-mode bypassPermissions` in its raw defaults, while normalization derives bypass only from OpenClaw's effective YOLO exec policy, preserves explicit Claude permission-mode overrides, strips legacy skip-permissions flags, and drops malformed permission-mode values. This also addresses the PR review concern that removing noninteractive permission mode outright could make default Claude CLI runs block on prompts.

## Best Possible Solution

Close PR #68682 and keep the shipped policy-based implementation from main/#70723. It removes the hardcoded Anthropic backend default, preserves explicit user overrides, strips unsafe legacy or malformed args, and aligns Claude CLI noninteractive behavior with OpenClaw's existing exec approval policy. If maintainers want to change OpenClaw's default exec posture itself, that should be a separate maintainer product/security discussion rather than merging this older raw-removal PR.
