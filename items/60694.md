## Summary

Keep open. Current main documents the safe configuration pattern, but the requested enforcement is still not implemented: `write`/`edit`/`apply_patch` denial does not constrain what an allowed `exec` command can do. The remaining capability is command-level exec write restriction, or an explicit maintainer decision that documentation-only mitigation is sufficient.

## Evidence

- **Tool policy is name-based and does not map write denial to exec:** Policy matching checks the normalized called tool name against deny/allow patterns. It only special-cases `apply_patch` against `write`; there is no equivalent rule making `deny: ["write"]` block `exec`.
  - file: [src/agents/tool-policy-match.ts:14](https://github.com/openclaw/openclaw/blob/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54/src/agents/tool-policy-match.ts#L14)
  - command: `nl -ba src/agents/tool-policy-match.ts | sed -n '1,120p'`
  - sha: [02f3e9cfa25f](https://github.com/openclaw/openclaw/commit/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54)
- **Exec is a separate runtime tool from filesystem tools:** The core catalog defines `write`, `edit`, and `apply_patch` as filesystem tools, while `exec` and `process` are runtime tools. Current grouping keeps these surfaces separate.
  - file: [src/agents/tool-catalog.ts:53](https://github.com/openclaw/openclaw/blob/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54/src/agents/tool-catalog.ts#L53)
  - command: `nl -ba src/agents/tool-catalog.ts | sed -n '1,120p'`
  - sha: [02f3e9cfa25f](https://github.com/openclaw/openclaw/commit/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54)
- **Existing tests prove exec can remain allowed beside filesystem policy entries:** The sandbox policy test expands `group:fs` separately from the `bash` alias and asserts `exec` is allowed while `apply_patch` is denied, confirming the policy model is per tool, not per semantic write capability.
  - file: [src/agents/tool-policy.test.ts:194](https://github.com/openclaw/openclaw/blob/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54/src/agents/tool-policy.test.ts#L194)
  - command: `nl -ba src/agents/tool-policy.test.ts | sed -n '160,205p'`
  - sha: [02f3e9cfa25f](https://github.com/openclaw/openclaw/commit/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54)
- **Exec still executes arbitrary shell text once allowed:** `runExecProcess` receives the command string and constructs either a sandbox `docker exec ... /bin/sh -lc` invocation or a host shell argv. There is no write-pattern inspection tied to `tools.deny: ["write"]` in this execution path.
  - file: [src/agents/bash-tools.exec-runtime.ts:537](https://github.com/openclaw/openclaw/blob/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54/src/agents/bash-tools.exec-runtime.ts#L537)
