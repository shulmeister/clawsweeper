## Summary

Keep #60805 open. Current main receives Discord reaction add/remove events and queues them as session system events, but I found no generic plugin reaction hook and no reaction-path wake/agent-turn trigger that would let a plugin respond in real time. The linked emoji-confirm plugin use case is therefore still blocked by a missing core/plugin API surface, so this is not a ClawHub-only cleanup close.
