## Summary

Close PR #63375 as cannot_reproduce. The PR’s stated reason is that `console.log` in a plugin `register()` can corrupt `openclaw config schema` JSON, but current `main` builds that schema from plugin manifest metadata and regression coverage explicitly guards against booting or replacing the plugin runtime registry during schema loads. The SDK docs already expose `api.logger`; any broader logging-best-practices doc should be tied to a currently reproducible JSON-output path.

## Best Possible Solution

Do not merge this PR as written. Keep the current manifest-only config schema path and regression tests; if maintainers want broader plugin logging guidance, add a smaller docs note to `docs/plugins/building-plugins.md` or `docs/plugins/sdk-runtime.md` that references `api.logger` and names only a verified current stdout-corruption surface.
