## Summary

Keep open. Current main has partial fixes for some bullets, but the core diagnostics contradiction is still present: `status --deep --json` includes the gateway health payload, while health builds channel summaries from a minimal snapshot that lacks runtime state and credential source metadata. For Discord, that can still render `running: false` and `tokenSource: "none"` even when `channels.status` uses the runtime snapshot/account projection path and can report the live configured account correctly. The gateway SecretRef warning also still treats a SecretRef token as unavailable before attempting the same async SecretRef resolution path used by gateway probe/audit flows.

## Best Possible Solution

Keep this issue open and fix the shared diagnostics surfaces: make `health`/`status --deep` reuse the same account snapshot construction or runtime/account projection contract as `channels.status`, preserving `running`, `connected`, `tokenSource`, and probe metadata. Then update doctor gateway auth to resolve applicable SecretRefs through the async SecretRef credential path, or consume the same resolvability result as secrets audit, before emitting an unavailable warning. Add focused regression tests for a configured running Discord account and for a resolvable `gateway.auth.token` SecretRef.
