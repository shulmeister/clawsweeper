## Summary

Current `main` no longer appends bundled MCP/LSP tools directly. `runEmbeddedAttempt` now routes them through a dedicated `applyFinalEffectiveToolPolicy(...)` pass before they are merged into `effectiveTools`, and that helper applies the same allow/deny pipeline used for core tools. Focused tests cover bundled-tool allowlist and explicit deny behavior, so the config-path request in this issue is implemented on `main` (but not yet in the latest tagged release).

## Evidence

- **Bundled MCP/LSP tools are filtered before merge:** `runEmbeddedAttempt` materializes bundled MCP and LSP tools, passes them through `applyFinalEffectiveToolPolicy(...)`, and only then spreads the filtered result into `effectiveTools`. This is the exact bypass described in the issue, now closed on `main`.
