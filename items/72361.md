## Summary

Keep this PR open. Current main still has the reported `/allowlist --store` policy gap: store-targeted edits skip the only `resolveConfigWriteDeniedText` check and call `updatePairingStoreAllowlist` directly, while docs say `/allowlist add|remove` honors `configWrites`. The PR is a focused implementation candidate paired with open issue #72360 by the same contributor, so it should be reviewed/merged or superseded by an equivalent narrow fix rather than closed.

## Best Possible Solution

Keep this PR open and review/merge it, or supersede it with an equivalent narrow fix on main: apply `resolveConfigWriteDeniedText` to store-targeted `/allowlist add|remove --store` mutations before `updatePairingStoreAllowlist`, using the explicit channel/account config-write target, and retain regression tests for channel-level false, account-level false, and allowed store edits. Close the paired issue #72360 only after this implementation or an equivalent maintainer-approved fix lands.
