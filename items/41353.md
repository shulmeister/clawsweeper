## Summary

Current `main` already implements macOS LaunchAgent auto-bootstrap for installed-but-unloaded gateway services, covers it with regression tests, and the changelog shows it shipped in `v2026.4.5`.

## Evidence

- **Recovery helper on main:** `recoverInstalledLaunchAgent()` checks whether the LaunchAgent plist still exists and then calls `repairLaunchAgentBootstrap()` before returning a loaded recovery result.
  - file: [src/cli/daemon-cli/launchd-recovery.ts:14](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/cli/daemon-cli/launchd-recovery.ts#L14)
- **Restart path uses recovery:** `runDaemonRestart()` prefers unmanaged-process recovery first, then falls back to `recoverInstalledLaunchAgent({ result: "restarted" })` when the macOS service is installed but not loaded.
  - file: [src/cli/daemon-cli/lifecycle.ts:186](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/cli/daemon-cli/lifecycle.ts#L186)
- **Start path uses recovery:** `runDaemonStart()` wires the same macOS recovery helper for not-loaded services, so start and restart both attempt re-bootstrap instead of only printing not-loaded hints.

## Close Comment

Closing as implemented on `main` after a Codex review. The current macOS gateway CLI now explicitly recovers an installed-but-unloaded LaunchAgent via `recoverInstalledLaunchAgent()` (`src/cli/daemon-cli/launchd-recovery.ts:14`) and wires that recovery into both `openclaw gateway start` and `openclaw gateway restart` (`src/cli/daemon-cli/lifecycle.ts:152` and `src/cli/daemon-cli/lifecycle.ts:186`). Regression coverage exists for both recovery paths in `src/cli/daemon-cli/lifecycle.test.ts:210`. This behavior is also called out in the `2026.4.5` changelog (`CHANGELOG.md:1251`), and the merged implementation came from PR #43766 / commit `fd968bfb2d16abf5232ad29ad7397068e6bc5b03`. As a result, this issue’s requested behavior is already shipped.
