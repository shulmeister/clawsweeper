## Summary

Keep #70418 open. Current main has prompt-cache retention, cache-TTL pruning metadata, and heartbeat keep-warm behavior, but it does not implement the proposed independent per-agent cache TTL warmer. The related heartbeat work (#70417 / #70602) is adjacent and only addresses heartbeat cache alignment; it does not supersede this separate design request. The CausaMem link in the comments is useful adjacent memory-system context, but it is not an OpenClaw implementation or canonical replacement for the requested core behavior.

## Best Possible Solution

Leave the design issue open for maintainer product judgment. If accepted, implement it as a core agent-runner/cache subsystem or first add a documented plugin API that can safely build the exact next-turn prefix, schedule per-agent refreshes, cap tokens, consume/discard the stream, and avoid transcript/session delivery side effects. Related heartbeat cache-alignment work should remain a separate narrower fix path.
