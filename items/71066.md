Keep #71066 open. The report is recent, concrete, and maintainer comments explicitly distinguish it from the generic Telegram stall/watchdog class. Current main still contains the sticky IPv4 fallback path and the related #71502 lifecycle fix only provides a retest baseline; it does not prove the India-host dispatcher loop is fixed or unreproducible.

## Best Possible Solution
