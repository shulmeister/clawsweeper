## Summary

Keep this PR open. Current main still has an unbounded `Promise.allSettled(pendingToolTasks)` drain in the reply pipeline, while tool-result callbacks are tracked as pending tasks and removed only when their delivery promise settles. The PR is a plausible implementation candidate for the open #53889 deadlock family, but it should not be merged blindly because review already identified a real ordering risk from a fixed 30s ceiling on healthy, slow tool-result delivery.

## Best Possible Solution

Keep this PR open as a focused implementation candidate for the #53889 class of session/tool-result deadlocks. The best path is to revise it into a progress-aware or idle-aware bounded drain for `pendingToolTasks`, add a regression test around a never-settling tool-result delivery task, and explicitly address the ordering risk from late healthy callbacks before merge.
