## Summary

Keep PR #69328 open. Current main still does not append `session.message` payloads into the active Control UI transcript, and there is no current-main ACP worktree diff verifier matching the PR's proposed terminal-status behavior. The PR is not ready to close as implemented or superseded; it should be updated to address the review concern that the proposed `git diff --stat HEAD` gate misses untracked files and unborn-HEAD repositories.

## Best Possible Solution

Keep the item open and ask for a focused update: implement active-run `session.message` appending in the Control UI using the existing gateway `message`/`messageId`/`messageSeq` payloads with robust dedupe and optimistic echo replacement, and either revise or split the ACP diff verifier so it handles untracked files and unborn-HEAD repositories or explicitly scopes the gate to tracked diffs with tests for persistent and oneshot modes.
