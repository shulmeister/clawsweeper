## Summary

Keep open. Current main does not contain the requested native Windows companion app: there is no `apps/windows` project or Tauri/Rust app surface, and the current docs explicitly say Windows companion apps are planned and not available yet. Existing native Windows support covers CLI install and managed Gateway startup through Scheduled Tasks or a Startup-folder fallback, which partially addresses setup friction but does not provide the requested tray app, native watchdog UI, or companion-app packaging.

## Best Possible Solution

Keep the issue open for maintainer product/design review. A viable path would be a focused RFC or PR that adds an actual `apps/windows` companion app, defines how it wraps or supervises the existing Gateway service without weakening security prompts, includes Windows installer/tray/watchdog behavior, updates public platform docs, and adds the appropriate app/platform CI or manual validation plan.
