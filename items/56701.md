## Summary

Keep open. Current `main` still does not support per-agent TTS voice overrides: `agents.list[]` has no `tts` field in the strict config schema, resolved agent config has no TTS slot, and the speech runtime still resolves TTS only from global `messages.tts`. The issue timeline’s cross-referenced PRs show related work exists, but that work is not present on `main` at `95b7a85f0607784d8e33b4ce7e42617174196490`.

## Evidence

- **Agent config schema has no per-agent TTS field:** `AgentEntrySchema` is strict and lists the allowed `agents.list[]` keys. It includes entries like `model`, `heartbeat`, `subagents`, `sandbox`, `tools`, and `runtime`, but no `tts` or `talk`, so the exact `agents.list[].tts` shape requested in the issue would not validate on current `main`.
