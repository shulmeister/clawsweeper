## Summary

Keep this PR open. Current main still does not implement the requested skill/minimal-mode pruning cap: context-pruning settings keep the default `keepLastAssistants: 3`, the embedded extension factory has no `promptMode` input, and the run/compaction paths do not pass prompt mode into that factory. The PR discussion is also relevant: the initial `skillsSnapshot` signal problem was called out, and the PR’s second commit switched the signal to `promptMode`, so the remaining work is a plausible focused implementation candidate rather than an obsolete or duplicate cleanup item.

## Best Possible Solution

Review the PR as an active implementation candidate. The likely maintainer path is to land or recreate the prompt-mode plumbing: add `promptMode` to the embedded extension factory inputs, cap `settings.keepLastAssistants` with `Math.min(settings.keepLastAssistants, 1)` only when the effective prompt mode is `minimal`, pass the effective run and compaction prompt modes into the factory, and keep focused tests for minimal versus full prompt behavior. Do not close it as implemented or superseded unless a separate merged change adds that behavior on main.
