## Summary

Keep open. Current main does not implement `channels.telegram.silentVerbose` or an equivalent config option: the Telegram config schema/type exposes streaming controls and `silentErrorReplies`, but no verbose-silent field, and delivery only passes `silent` for explicit silent sends or error replies. Existing `streaming.preview.toolProgress: false` can hide progress updates, but it does not satisfy the requested behavior of keeping verbose/streaming content visible while suppressing Telegram notifications.
