## Summary

Keep open. Current main has an iOS `AVSpeechSynthesizer` fallback, but it does not provide a documented or fully supported iOS Talk provider alternative to ElevenLabs. The iOS app still presents Talk as ElevenLabs-configured, and the runtime explicitly logs that non-ElevenLabs Talk providers are "not yet supported on iOS".

## Evidence

- **Talk docs still describe ElevenLabs-centric Talk config:** `docs/nodes/talk.md` still documents Talk mode as ElevenLabs TTS, with config examples and defaults only for `voiceId`, `modelId`, `outputFormat`, and `apiKey`, and only mentions Android local-system fallback.
  - file: [docs/nodes/talk.md:2](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/docs/nodes/talk.md#L2)

- **iOS settings still hardcode ElevenLabs as the Talk provider:** The iOS Settings UI shows `Provider: ElevenLabs` and says Talk is configured via `talk.apiKey`, `talk.modelId`, and `talk.voiceId`, with no surfaced system/native provider option.
  - file: [apps/ios/Sources/Settings/SettingsTab.swift:337](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/apps/ios/Sources/Settings/SettingsTab.swift#L337)
