## Summary

Keep #40016 open. Current main has useful backend session label plumbing through `sessions.patch`, but it still does not expose the requested first-class `openclaw sessions rename` / `set-subject` CLI command, and the `openclaw sessions` list output remains key-oriented rather than showing a user-set label/subject/display name.

## Best Possible Solution

Keep the issue open. Implement a focused `openclaw sessions rename <session-key> <label>` command, with `set-subject` or `set-label` as an alias only if maintainers want that wording. Reuse the existing `sessions.patch` label validation/RPC path, preserve channel-owned `subject` and `displayName` semantics, update `openclaw sessions` text and JSON output to surface an effective human-readable label before or beside the truncated key, and add targeted CLI tests plus `docs/cli/sessions.md` coverage.
