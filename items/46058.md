## Summary

Keep open. The issue is a contributor discussion about an official chat-first Android UX and possible focused upstreaming from external Android forks. Current main has a real Android Chat tab and related connection/media work, but it is not clearly the requested chat-first mode: the post-onboarding default is still Connect, the docs still instruct users to connect first, and the Android app is marked extremely alpha with release hardening incomplete. This is also in OpenClaw companion-app scope, not a ClawHub/plugin cleanup.
