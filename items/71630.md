## Summary

Keep this issue open. Current main still lacks a compact Web UI composer runtime status indicator for In progress, Interrupted, and Done; the existing UI only exposes local action controls plus separate Talk/compaction/fallback indicators. The related per-session activity API issue #39127 and channel status-surface issue #52640 remain open and do not supersede this focused Control UI request.

## Best Possible Solution

Keep this issue open. Implement a compact, accessible Control UI composer status chip near the attachment, voice, and Talk controls in ui/src/ui/views/chat.ts, with stable desktop/mobile styles and render tests. Use #39127's session-scoped activity API when available; an interim version may derive from existing chatRunId/chatSending/chat event lifecycle, but it should fail closed after disconnect/reconnect uncertainty and must not write status messages into the transcript.
