## Summary

Keep open. Current main still downloads media from `msg.reply_to_message` without checking whether that replied-to message was authored by the bot, and the existing bot-authored update guard only covers top-level Telegram message echoes.

## Best Possible Solution

Keep the PR open. The best path is to land a narrow Telegram fix in `resolveReplyMediaForMessage` that skips media download when `replyMessage.from.id` matches `ctx.me.id`, while preserving normal user-authored reply-media ingestion, plus a regression test in `extensions/telegram/src/bot.test.ts`.

## Evidence

- **Current reply-media path lacks the guard:** `resolveReplyMediaForMessage` reads `msg.reply_to_message`, checks `hasInboundMedia(replyMessage)`, then immediately resolves the file id and calls `resolveMedia`; there is no `replyMessage.from.id === ctx.me.id` check before download.
