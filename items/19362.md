## Summary

Keep open. Current main still does not implement Discord CHANNEL_DELETE session cleanup or an equivalent deleted-channel session removal path. The PR targets a real missing Discord behavior linked to #19333, and the existing Codex review comment already found the same gap; the patch likely needs porting/review rather than closure.

## Best Possible Solution

Keep the PR open and refresh it against the current extensions/discord layout. The best fix is to add a Discord channel-delete listener in the Discord plugin startup path, robustly extract the deleted channel ID, remove/archive matching agent:...:discord:channel:<channelId> session entries using the current session-store locking and transcript cleanup patterns, and add focused regression tests for multi-agent stores and transcript retention behavior.
