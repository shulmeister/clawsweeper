## Summary

Current main still lacks Discord CHANNEL_DELETE session cleanup. The PR addresses a real remaining Discord plugin gap linked to #19333, but its patch needs porting to the current extensions/discord layout and review for the transcript archival issues already raised in PR comments.

## Best Possible Solution

Keep the PR open and refresh it against current extensions/discord. The best fix is to add a Discord channel-delete listener in the Discord plugin startup path, robustly extract the deleted channel ID, delete/archive matching agent:<agentId>:discord:channel:<channelId> entries across the relevant agent session stores using the lock-aware session-store helpers, run transcript archive cleanup, and add focused regression tests for multi-agent stores and transcript retention behavior.
