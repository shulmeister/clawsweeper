## Summary

Keep #19362 open. Current main still does not implement Discord CHANNEL_DELETE session cleanup: Discord registers message, reaction, thread-update, and optional presence listeners only, while channel sessions are still persisted as isolated `agent:<agentId>:discord:channel:<channelId>` keys. The PR remains useful but needs a refresh for the current `extensions/discord` plugin layout and the transcript archival/cleanup issues already raised in review.

## Best Possible Solution

Keep the PR open and refresh it against current main. Implement deleted-channel cleanup in `extensions/discord/src/monitor` by adding and registering a Carbon `ChannelDeleteListener`, validating the deleted channel ID, scanning the relevant agent session stores for `agent:<agentId>:discord:channel:<channelId>` entries through an approved SDK/runtime seam, deleting matching entries under `updateSessionStore`'s lock, archiving and cleaning transcripts with current `sessions.delete` semantics, and adding regression tests for registration, cleanup, and transcript handling.
