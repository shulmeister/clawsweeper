## Summary

Current main still does not implement Discord guild channel deletion cleanup. Discord monitor startup registers message, reaction add/remove, thread-update, and optional presence listeners, but no ChannelDelete listener. The only cleanup-adjacent current behavior handles archived threads by marking matching thread sessions stale via updatedAt=0, not by deleting deleted guild-channel sessions or archiving their transcripts. PR #19362 remains useful, but it needs to be refreshed from the old src/discord paths to the current extensions/discord layout and should address the transcript archival issues already raised in review.

## Best Possible Solution

Keep #19362 open and refresh it against current main. Add and register a Discord channel-delete listener under extensions/discord/src/monitor, validate the deleted channel id, remove matching agent:<agentId>:discord:channel:<channelId> session rows through the current plugin-safe session-store/runtime seam, archive exactly the deleted entries' transcripts consistently with sessions.delete/store maintenance, run archived transcript cleanup, and add focused tests for registration, invalid ids, multi-agent/per-agent matching, deletion behavior, and transcript handling.
