## Summary

Keep #36630 open. Current main at 82262387651ab270b565f71533ce4937c9c00dc6 still does not implement the PR's central bidirectional Signal quote-reply support: Signal outbound sends do not emit signal-cli quote metadata, Signal adapters and monitor delivery do not forward quote/reply author metadata, and inbound Signal quote context still omits ReplyToId plus the richer quote payload model. The PR should not be closed as implemented, duplicate, ClawHub, or not-actionable; it remains a real missing bundled Signal behavior, though the supplied review context shows the current PR version still needs hardening before merge.

## Best Possible Solution

Keep #36630 open. The best maintainer path is to land a reviewed, hardened version of this Signal quote-reply work or reimplement it narrowly in the bundled Signal plugin: safe decimal quote timestamp/id validation, quote-author resolution from the referenced message for group replies, quote-aware Signal outbound delivery through formatted/plugin/monitor/queue paths, inbound ReplyToId plus richer signal-cli quote metadata, extension-boundary-compliant imports, and focused regression coverage for delivery consumption, debounce, queued/recovered delivery, and plugin-boundary behavior.
