## Summary

Current main still does not implement the PR's central Signal bidirectional quote-reply behavior. Outbound Signal sends have no quote timestamp/author plumbing, inbound quote context remains partial, and the open PR is still the active implementation candidate with recent review/security concerns that need maintainer handling rather than cleanup closure.

## Best Possible Solution

Keep #36630 open. The best path is to either land a reviewed, hardened version of the PR or reimplement the same core behavior narrowly: add strict Signal quote timestamp/id validation, pass `replyToId` plus resolved `quoteAuthor` through Signal outbound delivery, surface inbound `ReplyToId` and richer quote metadata, and cover the delivery/consumption edge cases in Signal and shared outbound tests.
