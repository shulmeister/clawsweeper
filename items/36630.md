## Summary

Keep PR #36630 open. Current main still does not implement the PR's central outbound Signal quote-reply behavior, and the inbound Signal quote context remains partial. The PR is active and materially relevant, but it needs maintainer review for scope, boundary, and security concerns rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open as the active implementation candidate. The best path is to narrow the diff to Signal quote-reply support, keep Signal-specific behavior inside the plugin or a documented SDK seam, address the security/bot review comments, and add focused tests for outbound `quote-timestamp`/`quote-author`, inbound `ReplyToId`, group author resolution, null suppression, and multi-payload reply consumption.

That means main cannot translate `[[reply_to:*]]` into native Signal quote metadata.
