## Summary

Keep #36630 open. Current main still lacks the PR's central behavior: native outbound Signal quote metadata is not emitted, Signal outbound paths do not forward quote/reply author metadata, and inbound Signal quote handling remains body/sender-only without ReplyToId or richer signal-cli quote payload support.

## Best Possible Solution

Keep #36630 open. The best maintainer path is to land a reviewed, hardened version of the Signal quote-reply implementation or reimplement it narrowly: strict decimal and safe Signal quote timestamp/id validation, quote-author resolution for group replies, quote-aware Signal outbound delivery, inbound ReplyToId plus richer quote metadata, and focused regression coverage for delivery consumption, debounce, and plugin boundary cases.
