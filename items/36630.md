## Summary

Keep PR #36630 open. Current main still does not implement the PR's core outbound Signal quote-reply behavior, and inbound quote handling remains partial. The PR is not obsolete or implemented-on-main; it needs maintainer review, especially because the PR discussion includes security/review concerns around quote-author caching and quoted-text prompt handling.

## Best Possible Solution

Keep the PR open. The best path is a maintainer-reviewed Signal implementation that adds quote-timestamp/quote-author delivery through the proper plugin/outbound contracts, surfaces structured inbound `ReplyToId` quote metadata, preserves existing channel threading behavior, and includes focused regression coverage for Signal group, DM, multi-payload, and malformed quote-id cases.
