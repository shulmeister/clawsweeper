## Summary

Current `main` already hardens cron job loading against the missing/normalized-away `sessionTarget` shape behind this report, and it adds regression coverage for the manual-run crash path. The issue comment’s `sessionTarget: "default"` diagnosis matches the current normalization logic: unsupported values are dropped during load, and `main` now backfills a safe target before downstream `.startsWith()` calls.

## Evidence
