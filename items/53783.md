## Summary

Current main does not have enough evidence to close this Telegram cross-agent visibility bug. Telegram still records agent-scoped group session keys, and the session tools still apply generic cross-agent visibility and agent-to-agent gates with no same-Telegram-group bidirectional guarantee or regression test covering two agents in one group.

## Best Possible Solution

Keep this issue open. Reproduce with two configured agents in the same Telegram group under explicit `tools.sessions.visibility: "all"` and enabled, allowlisted `tools.agentToAgent`. If the asymmetry reproduces, add regression coverage that both agents can see both `agent:<agent>:telegram:group:<id>` keys in `sessions_list` and resolve/send through `sessions_send` both directions, then fix the gateway/session-store/visibility path. If cross-agent same-group sends are intentionally opt-in, improve the tool error and docs so users get clear config guidance instead of a one-way `No session found`. Track #59933 separately because it is same-agent group listing, not this cross-agent mismatch.
