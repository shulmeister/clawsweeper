## Summary

Keep this issue open. Current main still gates `/status` and `/usage full` cost display on API-key-like auth, so an `aws-sdk` Bedrock provider with explicit model `cost` metadata will still show tokens without estimated dollars. The related same-author PR #53303 was closed unmerged, so it is not a shipped fix candidate.

## Best Possible Solution

Keep this issue open. Implement the core usage-display change by resolving explicit, nonzero model cost metadata independently of auth mode, then show estimated cost in both `/status` and `/usage full` for `aws-sdk`, OAuth, token, mixed, and API-key providers when pricing is configured. Add regression coverage for a Bedrock-style `aws-sdk` provider with explicit costs and update the public usage-cost/token-use docs, while avoiding misleading `$0` estimates for unknown/default pricing.
