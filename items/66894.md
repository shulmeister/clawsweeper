## Summary

Keep open. Current main still has the reported `/compact` cancellation gap: the command handler calls `compactEmbeddedPiSession` without an `abortSignal`, does not register a compaction handle in `ACTIVE_EMBEDDED_RUNS`, and the linked bug #66535 remains open. The PR is not obsolete, though it should be rebased and cleaned up because the provided PR context shows unresolved review points and an unrelated `OPENCLAW_LAUNCHD_LABEL` profile change bundled into the branch.

## Best Possible Solution

Keep the PR open, but ask the contributor or maintainer to rebase and narrow it to #66535: register manual compaction as an active embedded run at the queued execution point, wire a real AbortController through `abortSignal`, cascade abort to any active reply operation for the same session, clear the handle reliably, and add focused regression tests. Split the unrelated `OPENCLAW_LAUNCHD_LABEL` profile change into its own PR or remove it from this branch.
