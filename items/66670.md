## Summary

Keep open. Current main still has the observable gap: sessions_spawn creates task_runs rows with child_session_key containing the target agent, but the task creation path does not pass agentId, and the task registry only persists agent_id when it is explicitly provided.

## Best Possible Solution

Populate agentId at task creation time for sessions_spawn by passing the resolved targetAgentId into createRunningTaskRun/registerSubagentRun, and add a regression test that verifies task_runs/listTasksForAgentId attribution for cross-agent subagent spawns. Keep childSessionKey parsing as a compatibility fallback only, not the primary plugin contract.
