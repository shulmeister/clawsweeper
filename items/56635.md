## Summary

Keep open. Current `main` still omits `--exclude-labels` when building Gmail watcher serve args, the watcher/runtime uses those args directly, the config schema still exposes no `hooks.gmail.excludeLabels` escape hatch, and the current tests/latest release both match the regressed behavior rather than a fix.

## Evidence

- **Current main omits the flag:** `buildGogWatchServeArgs()` on current `main` builds `gog gmail watch serve` args with `--account`, `--bind`, `--port`, `--path`, `--token`, `--hook-url`, `--hook-token`, optional `--include-body`, and optional `--max-bytes`, but no `--exclude-labels`.
