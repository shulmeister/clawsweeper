## Summary

Keep #69374 open. Current main still drops WhatsApp forwarding state before plugin hooks can observe it: Baileys `contextInfo` is extracted for mentions and quoted replies, but `forwardingScore` / `isForwarded` are not carried into the WhatsApp inbound message type, finalized inbound context, canonical hook context, or `inbound_claim` / `message_received` metadata. This should not be closed as ClawHub because an external plugin cannot recover data that the bundled WhatsApp normalization path has already discarded.

## Best Possible Solution

Add an additive WhatsApp-owned pass-through: extract `contextInfo.forwardingScore` and `contextInfo.isForwarded` during WhatsApp inbound normalization, derive a boolean `isForwarded` consistently, carry both values through the normalized inbound/finalized/canonical hook context, and include them in `inbound_claim` and `message_received` metadata. Cover it with WhatsApp extraction or monitor tests plus hook-mapper tests, and update plugin docs/changelog if the metadata becomes public behavior.
