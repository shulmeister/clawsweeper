## Summary

Keep #69374 open. Current main still drops WhatsApp forwarding state before plugin hooks: Baileys exposes `forwardingScore` and `isForwarded`, but the WhatsApp normalized inbound type, inbound context builder, canonical hook context, and plugin hook event metadata do not carry either field.

## Best Possible Solution

Keep the issue open and implement this as an additive bundled WhatsApp/plugin-hook contract change: read `contextInfo.forwardingScore` and `contextInfo.isForwarded` during WhatsApp inbound normalization, carry them through `WebInboundMessage`/`FinalizedMsgContext` or a narrow canonical hook field, expose them in `inbound_claim` and `message_received` metadata, and add WhatsApp extraction plus hook-mapper regression tests and plugin-hook docs.
