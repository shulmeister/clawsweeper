## Summary

Keep PR #65802 open. Current `main` still makes Feishu WebSocket inherit ambient proxy env by default, so the linked bug #65799 remains real on the checked-out code. The PR is a focused fix for that behavior, but the submitted tests need the Greptile-noted `OPENCLAW_FEISHU_WS_USE_PROXY` env cleanup before it is merge-ready.

## Best Possible Solution

Keep the PR open and ask for a small follow-up patch: export/use the opt-in env var as proposed, add the default-bypass and opt-in regression tests, and restore `OPENCLAW_FEISHU_WS_USE_PROXY` in the test `beforeEach`/`afterEach` cleanup before landing. The linked issue #65799 can close through the PR once the cleaned-up implementation merges.
