## Summary

Current main has not absorbed the central CLI diagnostic fix. The source and docs show `commands` is a built-in runtime chat slash command, but `resolveMissingPluginCommandMessage` still falls through to the generic `plugins.allow` error for unknown primaries such as `openclaw commands`. The PR remains a plausible implementation candidate for the closed linked bug #65083, but it should be cleaned up before merge because the provided review context still flags brittle string-based control flow and the diff includes unrelated ACP Matrix assertion changes.

## Best Possible Solution

Keep this PR open, but require a focused update before merge: implement structured core runtime-slash-command classification for `openclaw commands` (for example using the built-in chat command registry or a typed diagnostic result), preserve plugin allow/disable diagnostics for real plugin command aliases, add focused `run-main` and exit-path regression coverage for `commands` with and without `plugins.allow`, and drop or split the unrelated ACP Matrix assertion changes.
