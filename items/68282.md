## Summary

Current main already generalizes the reported fix: doctor-contract loading still uses the shared plugin Jiti loader, and that loader now prefers native loading for bundled `dist/extensions/*.js|.mjs|.cjs` artifacts on supported runtimes. The changelog records this as PR #69925, but the checkout does not show a newer shipped tag than `v2026.4.23`, so this looks fixed on `main` without a confirmed released version yet.

## Evidence

- **Config validation still reaches doctor legacy rules through the same seam:** `validateConfigObject` collects doctor plugin ids, calls `listPluginDoctorLegacyConfigRules`, then feeds those rules into `findLegacyConfigIssues`, so the reported slowdown path still flows through the doctor-contract registry on current main.
