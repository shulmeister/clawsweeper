## Summary

Keep open. Current main still has only whole-tool allow/deny policy, while `gateway` remains one owner-only tool whose `action` parameter multiplexes `restart`, `config.get`, `config.schema.lookup`, `config.apply`, `config.patch`, and `update.run`. Main has useful safeguards for gateway config writes, and `commands.restart=false` can block restart, but it still does not implement the requested policy shape: deny `gateway:config.patch`/`gateway:config.apply` while keeping read-only gateway actions available.

## Evidence

- **Gateway actions are still multiplexed behind one tool:** `GATEWAY_ACTIONS` includes both read-only and write/restart actions, and `createGatewayTool()` registers a single tool named `gateway`. The action is read from params at execution time, so `gateway:config.patch` is not a distinct tool in the current catalog.
