## Summary

Keep open. Current main still has the unsupported-platform throw in `resolveGatewayService()`, and both `openclaw status` and `openclaw gateway status` still reach that resolver before their service-status fallback handling can run. The directly related PR #25690 proposed the requested unsupported-platform stub behavior, but the provided GitHub context shows it was closed unmerged.

## Best Possible Solution

Keep #25621 open until OpenClaw either adds a graceful unsupported-platform `GatewayService` adapter/fallback, guards status and lifecycle resolver calls so diagnostic commands keep working on OpenBSD, or explicitly documents unsupported service-manager platforms as out of scope. Reviving the substance of closed PR #25690 remains the most direct implementation path.
