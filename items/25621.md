## Summary

Keep open. Current main still throws for unsupported gateway service platforms such as OpenBSD, and both `openclaw status` and `openclaw gateway status` still resolve the gateway service before their fallback/error-handling paths can produce a useful status. The directly related PR #25690 proposed the requested unsupported-platform stub behavior, but the provided GitHub context shows it was closed unmerged.

## Best Possible Solution

Keep #25621 open until OpenClaw either adds a graceful unsupported-platform `GatewayService` adapter/fallback, guards status and lifecycle resolver calls so diagnostic commands keep working on OpenBSD, or explicitly documents unsupported service-manager platforms and manual-supervisor setup as out of scope. Reviving the substance of closed PR #25690 remains the most direct implementation path, with updated tests that assert status commands do not crash on unsupported platforms.
