## Summary

Keep open. Current `main` has partial native-subagent hardening, but the requested persistent/configurable retention feature is not implemented end-to-end. `agents.defaults.subagents` still exposes only `archiveAfterMinutes`, native subagent session-mode cleanup is a hardcoded 5-minute TTL rather than a `sessionRetention` setting, the docs still say restart loses pending auto-archive timers, and successful ACP spawns still do not schedule any automatic session cleanup.

## Evidence

  - file: [src/config/types.agent-defaults.ts:388](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/config/types.agent-defaults.ts#L388)

## Risks / Open Questions

- Closing as implemented would overstate current behavior: the core lacks a documented/operator-configurable retention knob for subagent sessions and still lacks an automatic retention path for successful ACP spawns.
- The native subagent code has partial restart-safe recovery, so any follow-up likely needs issue scoping narrowed between native subagents, ACP sessions, and docs drift rather than assuming a single fully solved path.
