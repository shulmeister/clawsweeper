## Summary

Keep #38302 open. Current main still lacks an account-scoped Slack native slash command prefix or nativeNames mapping. Slack config/schema expose only the existing slashCommand knobs, native command generation and lookup remain provider-scoped, and Slack registration dispatches the provider-wide command names without threading account config. The related implementation attempts #38326 and #38391 were closed unmerged, and the issue discussion includes another user confirming the same multi-app Slack command collision.

## Best Possible Solution

Keep #38302 open and implement the feature in the bundled Slack plugin. A maintainer-grade fix should add a validated account-level command naming contract, likely slashCommand.nativeNames or a nativePrefix-derived mapping; thread the resolved account config through Slack native spec generation, registration, argument menu labels, and findCommandByNativeName dispatch; preserve the existing Slack /status -> /agentstatus provider override; update Slack setup/docs; and add focused multi-account tests proving prefixed commands route to the intended internal commands.
