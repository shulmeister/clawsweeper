## Summary

Keep open. Current main still lacks the requested per-account Slack native slash command prefix or account-level nativeNames mapping. The Slack config type/schema only supports enabled/name/sessionPrefix/ephemeral, the native command registry resolves by provider only, and Slack native command registration/lookup still registers and dispatches unprefixed provider-resolved names. The related PRs #38326 and #38391 attempted this work but closed unmerged, and a second user confirmed the same multi-app Slack problem in the issue discussion.

## Best Possible Solution

Keep #38302 open and implement an account-scoped Slack native command naming contract. The likely product direction is to add an account-level `slashCommand.nativeNames?: Record<string,string>` or equivalent prefix-derived mapping, validate it in Slack config schema/types, thread it through native spec generation and lookup for the Slack account, update Slack registration/interactive dispatch, and document the required Slack app manifest setup with focused tests for multi-account command collision handling.
