## Summary

Keep #38302 open. Current main still does not provide account-scoped Slack native command names: Slack config/schema expose only the existing slashCommand fields, native command generation and lookup remain provider-scoped, and Slack registration/dispatch still registers provider-level names like /agentstatus without threading an account prefix or nativeNames mapping. The issue also has a second user confirming the multi-app Slack collision, while the two related implementation attempts (#38326 and #38391) are closed and unmerged.

## Best Possible Solution

Keep #38302 open and implement the feature in the bundled Slack plugin. A maintainer-grade fix should add a validated account-level naming contract, likely slashCommand.nativeNames or a nativePrefix-derived mapping; thread the resolved account config through native command spec generation, Slack registration, argument menu values, and findCommandByNativeName dispatch; preserve /agentstatus; update Slack setup/docs; and add focused multi-account tests proving prefixed native commands route to the intended internal commands.
