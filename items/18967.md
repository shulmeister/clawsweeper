## Summary

Current main still does not provide the requested narrow parent-scoped `sessions_send` mechanism for native sub-agents. It has broad configurable opt-in for the existing `sessions_send`, native sub-agent announce/completion flow, and ACP-specific parent streaming/A2A loop handling, but no `sessions_send_parent` tool, no literal `parent` target alias, and no direct-parent-only/no-cross-agent/no-chaining enforcement path.

## Best Possible Solution

Keep #18967 open. The best product/code path is a narrow native `sessions_send_parent` tool or a literal `parent` target that resolves from the sub-agent session's stored direct-parent metadata, enforces same agent tree and direct-parent-only access, avoids automatic forwarding beyond that parent, and includes focused policy, resolver, visibility, docs, and regression tests. Existing broad `sessions_send` opt-in can remain a workaround but should not be treated as the safe feature requested here.
