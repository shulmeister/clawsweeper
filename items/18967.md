## Summary

Keep #18967 open. Current main still lacks a native parent-scoped child-to-direct-parent send path: default sub-agent policy denies `sessions_send`, the send tool has no `sessions_send_parent` or `sessionKey: "parent"` mode, and existing visibility/ACP streaming features do not satisfy the issue's parent-only/no-cross-agent/no-chaining constraints.

## Best Possible Solution

Keep #18967 open. The best implementation is a narrow native parent-send path, either a dedicated `sessions_send_parent` tool or `sessions_send` with literal `sessionKey: "parent"`, resolved from direct parent metadata (`spawnedBy`/`parentSessionKey`) and guarded to the same agent tree with no automatic forwarding to grandparents. Add resolver, policy, visibility, docs, and regression tests that prove default sub-agents cannot target arbitrary sessions while the parent-only path works.

- ACP `streamTo: "parent"` progress streaming may look similar, but it is ACP-specific and does not let a child ask its direct parent a question through a constrained send API.
