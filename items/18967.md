## Summary

Current main still lacks a parent-scoped `sessions_send` path for native sub-agents. The available implementation is either the existing broad `sessions_send` tool, which can be opt-enabled for sub-agents by config, or ACP-specific parent streaming/completion behavior. Neither provides the requested direct-parent-only, no-cross-agent, no-chaining sub-agent send surface.

## Best Possible Solution

Keep #18967 open. The likely product/code path is a narrow `sessions_send_parent` tool or a literal `parent` target that resolves from the sub-agent session's stored parent/requester metadata, enforces same agent tree and direct-parent-only access, does not auto-forward further up the tree, and has focused policy, visibility, resolver, docs, and tests. The existing broad opt-in `sessions_send` config can remain as a workaround but should not be treated as the requested safe parent-scoped feature.
