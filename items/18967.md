## Summary

Current main still does not implement the requested narrow parent-scoped native sub-agent send path. It has the existing broad `sessions_send` tool, configurable sub-agent opt-in for that broad tool, downward/tree session visibility, sub-agent announce delivery, and ACP-specific parent-to-child loop suppression, but no `sessions_send_parent`, no literal `parent` target alias, and no direct-parent-only/no-cross-agent/no-chaining enforcement for native sub-agents. The OWNER comment on #18967 also explicitly kept this open, and the current checkout remains consistent with that assessment.

## Best Possible Solution

Keep #18967 open. The best path is a narrow native `sessions_send_parent` tool or literal `parent` target that resolves from stored direct-parent metadata, enforces same agent tree and direct-parent-only access, does not auto-forward further up the tree, and adds focused policy, resolver, visibility, docs, and regression tests. The existing broad `sessions_send` opt-in can remain as a workaround but should not be treated as the safe feature requested here.
