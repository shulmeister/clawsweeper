## Summary

Keep #18967 open. Current main still has broad `sessions_send` plus configurable subagent allowlist/visibility escape hatches, but it does not provide the requested native parent-scoped send path with direct-parent-only, same-tree, no-chaining enforcement. A prior maintainer/Codex comment on the issue also explicitly kept this open for that distinction.

## Best Possible Solution

Keep #18967 open. The best path is a narrow native `sessions_send_parent` tool, or a literal `parent` target on `sessions_send`, resolved from stored direct-parent metadata. It should enforce direct-parent-only access within the same OpenClaw agent tree, avoid automatic forwarding further up the tree, preserve the existing broad opt-in as a separate escape hatch, and add resolver/policy/visibility/docs coverage.
