## Summary

Keep open. Current `main` still does not expose a gateway runtime/service version in `openclaw gateway status` output, and the read-probe path already calls the `status` RPC but drops the returned payload instead of surfacing its `runtimeVersion`.

## Evidence

- **Gateway status model has no version field:** `GatewayStatusSummary` only includes bind host/mode, port, probe URL, and note fields; `DaemonStatus` exposes `gateway`, `rpc`, `health`, and `extraServices`, but no gateway/service version field is defined for the status payload.
  - file: [src/cli/daemon-cli/status.gather.ts:42](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/cli/daemon-cli/status.gather.ts#L42)
  - command: `nl -ba src/cli/daemon-cli/status.gather.ts | sed -n '42,185p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Current JSON output is just the gathered daemon status object:** `gatherDaemonStatus()` returns the final status object with `service`, `config`, `gateway`, `port`, `rpc`, `health`, and `extraServices`; there is still no version field added before JSON output.
  - file: [src/cli/daemon-cli/status.gather.ts:472](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/cli/daemon-cli/status.gather.ts#L472)
  - command: `nl -ba src/cli/daemon-cli/status.gather.ts | sed -n '472,510p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Require-RPC path calls `status` but discards the response:** When `--require-rpc` is used, `probeGatewayStatus()` performs `callGateway({ method: "status" ... })` only as a health/readability check, then returns `{ ok: true, authProbe }` without preserving any returned version data.
  - file: [src/cli/daemon-cli/probe.ts:56](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/cli/daemon-cli/probe.ts#L56)
  - command: `nl -ba src/cli/daemon-cli/probe.ts | sed -n '56,85p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Version data already exists on the gateway `status` RPC surface:** `getStatusSummary()` populates `runtimeVersion` from `resolveRuntimeServiceVersion(process.env)`, so the underlying gateway status summary already has the version information the issue is asking to surface.
  - file: [src/commands/status.summary.ts:268](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/commands/status.summary.ts#L268)
  - command: `nl -ba src/commands/status.summary.ts | sed -n '268,270p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Docs still position gateway status as the operator-facing runtime check:** The CLI docs describe `gateway status` as showing service state plus connectivity/auth capability and recommend `--require-rpc` for scripts and automation, which matches the issue's claim that this is the operator-facing status surface.
  - file: [docs/cli/gateway.md:167](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/cli/gateway.md#L167)
