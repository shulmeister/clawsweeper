## Summary

Keep this PR open. Current main fixed the original Claude CLI session-reset symptom with stable `mcpResumeHash` handling, but it still does not implement the PR's `gateway.mcpLoopback.port` config surface or any operator-facing way to pin the loopback MCP port. The remaining capability is plausible core gateway/config work, so this is not a conservative cleanup close.

## Best Possible Solution

Keep this PR open, but rework it against current main: add `gateway.mcpLoopback.port` to gateway config types, Zod/schema/help/generated metadata and docs, then thread it into the current lazy `ensureMcpLoopbackServer(configuredPort)` path with targeted tests for configured-port binding and bind-failure diagnostics. If maintainers decide the stable `mcpResumeHash` solution fully eliminates the need for a fixed operator port, close only after that explicit product decision or a canonical follow-up is recorded.
