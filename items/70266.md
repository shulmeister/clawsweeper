## Summary

Keep #70266 open. Current main still renders the macOS Talk Mode overlay as the generic SwiftUI orb and only carries `ui.seamColor` into that native Talk path. Existing assistant avatar support is implemented for the Control UI `/avatar` surface, but there is no corresponding macOS Talk Mode avatar transport, parser field, image fetch/load path, or `TalkOverlayView` rendering hook. The reporter’s follow-up correctly distinguishes this issue from the closed web/chat identity work in #22199.

## Best Possible Solution

Keep the issue open and implement the feature in the macOS Talk Mode path: extend a suitable gateway/native contract to expose the resolved assistant avatar for the active agent, parse/cache it in the macOS app, render it as a circular image inside `TalkOverlayView` while preserving existing rings/pulse/thinking state visuals, fall back to the current orb when unresolved, and add macOS parser/overlay tests plus any protocol tests for the new field.
