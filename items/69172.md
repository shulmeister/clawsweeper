Current main has not implemented the central Gemini custom endpoint / OpenAI-compatible override behavior, and the linked #61714 is closed unmerged rather than a canonical tracker. The PR is not merge-ready as written, but it contains a plausible repo-actionable fix area for bundled Google plugin behavior, so none of the allowed close reasons fit.

## Best Possible Solution

Keep the item open, but ask for a scoped revision or replacement PR limited to the Google plugin. The best path is a shared Google override resolver with tests for web search, embeddings, image generation, and media understanding; preserve native Gemini as the default for official Google endpoints; keep GOOGLE_API_KEY credential detection correct; avoid unrelated pnpm/memory-sdk/channel changes; remove accidental root files; and explicitly test OAuth auth plus image-edit/audio/video OpenAI-compatible semantics.
