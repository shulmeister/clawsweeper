## Summary

Keep open. I did not find repo-local evidence that #59228 is fixed on current `main`. The documented contract still says `command-dispatch: tool` should receive raw slash-command args, the generic inline dispatch path still forwards them, but the Telegram native-command path still rebuilds the command exclusively from `ctx.match` and there is no Telegram-specific regression coverage for skill `command-dispatch` arg forwarding. Recent issue comments also confirm the bug remained present on 2026.4.9.
