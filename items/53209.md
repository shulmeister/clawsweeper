## Summary

Keep open. Current `main` does not implement the PR’s sandbox env allowlist/sync behavior: Docker sandbox env sanitization still blocks secret-like keys unconditionally, sandboxes are created before skill env overrides run, and sandbox exec env still comes only from `cfg.docker.env` rather than the active skill overrides.

## Evidence

- **Secret-like env names are still unconditionally blocked in sandbox sanitization:** `sanitizeEnvVars()` blocks keys like `OPENAI_API_KEY`, `GITHUB_TOKEN`, and `OPENCLAW_GATEWAY_TOKEN`, and the options type has no `allowedSensitiveKeys` escape hatch on `main`. That means skill-declared secret-like names still cannot pass container env sanitization.
  - file: [src/agents/sandbox/sanitize-env-vars.ts:1](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/sandbox/sanitize-env-vars.ts#L1)
  - command: `nl -ba src/agents/sandbox/sanitize-env-vars.ts | sed -n '1,102p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Docker sandbox creation still sanitizes only static sandbox config env:** `buildSandboxCreateArgs()` sanitizes `params.cfg.env` and `createSandboxContainer()` does not thread any skill-derived allowlist into that call. The create path therefore still drops secret-like names during container creation.
  - file: [src/agents/sandbox/docker.ts:403](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/sandbox/docker.ts#L403)
  - command: `nl -ba src/agents/sandbox/docker.ts | sed -n '348,489p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Sandbox is created before skill env overrides are applied:** In the embedded runner, `resolveSandboxContext()` runs first, then `applySkillEnvOverrides*()` runs later inside the try block. So any skill-configured `primaryEnv`/`requires.env` values are not available when the Docker sandbox is created.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:468](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/pi-embedded-runner/run/attempt.ts#L468)
  - command: `nl -ba src/agents/pi-embedded-runner/run/attempt.ts | sed -n '453,518p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Sandbox exec env still comes from sandbox config, not active skill overrides:** Tool execs use `sandbox.backend?.env ?? sandbox.docker.env`, and the Docker backend populates that from `params.cfg.docker.env`. There is no current-main helper that syncs the post-override `process.env` skill values into the sandbox exec env.
  - file: [src/agents/pi-tools.ts:533](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/pi-tools.ts#L533)
