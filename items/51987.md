## Summary

The reported 2026.3.13-era 3s local gateway handshake timeout is no longer present on current `main`. Gateway pre-auth and client connect-challenge timeouts now default to 10_000ms, and the changelog records that fix as shipped in `v2026.3.22`.

## Close Comment

Closing as implemented on `main` based on this Codex review. The current gateway handshake path no longer uses the 3s default described in the report: `src/gateway/handshake-timeouts.ts:1` now sets the shared pre-auth timeout to `10_000ms`, `src/gateway/server/ws-connection.ts:366` uses that value for the server-side handshake timer, and `src/gateway/client.ts:167` / `src/gateway/client.ts:820` route the client connect-challenge timeout through the same shared limit.
