## Summary

Keep #24196 open. Current main still does not implement native multi-file Discord message uploads for the normal message action/outbound reply path. The bundled Discord plugin accepts and forwards one scalar `mediaUrl`, the REST upload helper constructs a `files` array with one item, and multi-media payloads are still sent as repeated single-media messages. The provided issue discussion also includes a same-day maintainer-side Codex comment explicitly keeping this issue open for the same reason.

## Best Possible Solution

Keep the issue open and implement native Discord multi-file sending in the bundled Discord plugin. Add a backward-compatible ordered multi-attachment input shape for message actions and outbound payloads, preserve existing single `media`/`path`/`filePath` behavior, validate and split by Discord count and size limits, send supported batches as one Discord `files` array, and update focused Discord tests and docs.
