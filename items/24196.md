## Summary

Keep #24196 open. Current main still does not support native multi-file Discord uploads for ordinary message actions or outbound reply delivery: the message action input is scalar, the send API accepts one `mediaUrl`/`filename`, `sendDiscordMedia` constructs exactly one file entry, and shared reply-payload fanout sends multiple media URLs as separate messages. The existing 2026-04-26 maintainer comment also explicitly kept this issue open, and the relevant current-main code/docs still match that finding.

## Best Possible Solution

Keep #24196 open and implement native multi-file sending in the bundled Discord plugin: add a backward-compatible ordered multi-attachment input shape for message actions and outbound payloads, preserve existing single-file parameters, validate and split by Discord count/size limits, send eligible batches as one Discord `files` array, and update focused Discord tests plus the public Discord docs.
