## Summary

Keep open. Current `main` still implements `sessions_send` as an announce-capable agent-to-agent flow, not a strict internal-only handoff, and the session routing layer still lets `webchat` own sessions that have no established external route. The docs are clearer about the announce path now, but I did not find a code change on `main` that would resolve the behavior reported in #64917.
