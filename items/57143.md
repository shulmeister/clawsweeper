## Summary

Keep open. Current `main` still ships the bundled `gh-issues` skill with multiple hardcoded `/data/.clawdbot/...` paths, while OpenClaw's canonical state/config location is `~/.openclaw` (or `OPENCLAW_STATE_DIR` / `OPENCLAW_CONFIG_PATH`). The only linked fixes, PRs #57146 and #57224, are still open and unmerged, so this is not implemented on `main` yet.

## Evidence

Those literals are present in the shipped skill text on current `main`.

## Risks / Open Questions

- This remains user-facing because `skills/gh-issues/SKILL.md` is bundled guidance; standard installs using `~/.openclaw` will follow invalid `/data/.clawdbot` paths for token fallback, claims, and cursor state.
- I did not execute the skill in this read-only review, but the shipped prompt text itself is sufficient evidence that the documented command paths are still wrong on current `main`.
- There is a small test-gap risk: current env-path guidance coverage does not prevent this exact legacy-path regression from shipping again.
