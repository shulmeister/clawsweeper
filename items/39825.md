## Summary

Keep #39825 open. Current main still has the reported Discord voice readiness-abort shape: `/vc join` waits for a fixed 15s `Ready` transition, destroys the connection on failure, and only records a session after that wait succeeds, so `/vc status` can still report no active sessions after an abort. The newer #63098 remains open with the same symptom, but there is no maintainer signal making it the canonical replacement.

## Best Possible Solution

Keep #39825 open as the Discord native voice readiness-abort tracker unless maintainers intentionally consolidate it into #63098. A concrete fix likely belongs in `extensions/discord/src/voice/manager.ts`: add regression coverage for `entersState` aborts/timeouts before session activation, improve diagnostics around readiness failure, and decide whether to add bounded retry/backoff, a longer default, or a documented `channels.discord.voice.connectTimeoutMs` setting.
