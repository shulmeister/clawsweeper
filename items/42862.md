## Summary

Keep open. Current `main` does not implement auto-rendering plain image URLs in chat. The chat text path renders message text through markdown/linkify, which turns `https://...` into links, while the markdown image renderer explicitly refuses remote image sources and only allows base64 data-URI images. Separate chat image rendering exists, but only for structured image blocks, attachments, or transcript media paths, not plain text URLs.

## Evidence
