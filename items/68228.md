## Summary

Current `main` no longer silently drops pasted images when the gateway thinks the active model is text-only, and `anthropic/claude-opus-4.7` is now a first-class Anthropic model on `main`, so the specific 2026.4.15 webchat failure path described here is already covered well enough.

## Evidence

- **Current gateway path offloads instead of dropping text-only attachments:** On current `main`, `parseMessageWithAttachments()` sets `shouldForceOffload` when `supportsImages === false` and saves the image as `media://inbound/...`, appending a `[media attached: ...]` marker instead of returning an empty attachment result.
  - file: [src/gateway/chat-attachments.ts:309](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/gateway/chat-attachments.ts#L309)
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Regression tests cover the text-only-model image path:** Current tests assert that text-only models offload pasted images instead of dropping them, preserve `offloadedRefs`, and cap the number of forced offloads.
  - file: [src/gateway/chat-attachments.test.ts:146](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/gateway/chat-attachments.test.ts#L146)
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Media-only turns no longer fall into the empty-text gate:** Reply-path tests assert that media-only turns are allowed and rewritten with `[User sent media without caption]`, which avoids the old `I didn't receive any text...` outcome for image-only turns.
  - file: [src/auto-reply/reply/get-reply-run.media-only.test.ts:269](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/auto-reply/reply/get-reply-run.media-only.test.ts#L269)
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Changelog records the exact gateway/webchat fix on unreleased main:** The unreleased `2026.4.24` changelog says Gateway/WebChat now preserves image attachments for text-only primary models by offloading them as media refs instead of dropping them; the same section also records the related `imageModel` routing fix tied to cross-reference #69171.
  - file: [CHANGELOG.md:266](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/CHANGELOG.md#L266)
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Claude Opus 4.7 is first-class on current main:** Current Anthropic docs state that `anthropic/claude-opus-4.7` is supported directly on `main`, so the pre-catalog custom `models.json` path described in the report is no longer the normal path for this model.
  - file: [docs/providers/anthropic.md:269](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/docs/providers/anthropic.md#L269)
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Latest release still had the old drop-on-text-only behavior:** The `v2026.4.23` tag still returned `{ message, images: [], imageOrder: [], offloadedRefs: [] }` immediately when `supportsImages === false`, which shows the attachment-preserving change is newer than the latest shipped release and currently lives on `main` only.
  - file: [src/gateway/chat-attachments.ts:307](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/gateway/chat-attachments.ts#L307)

## Risks / Open Questions

- The current fix preserves pasted images by offloading them through media refs; if custom model capability metadata still fails to load, unknown custom vision models may use the media-understanding/offload path rather than native inline image input.
- This was a read-only code/history review, not a live reproduction against a running gateway with a failing `models.json` file.
