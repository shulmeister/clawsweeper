## Summary

The reported AbortError happens in the external `@tencent-weixin/openclaw-weixin` channel plugin's QR-login path, not in OpenClaw core. Current main keeps WeChat login and Tencent iLink API handling plugin-owned, and the issue timeline already points to Tencent/openclaw-weixin#15 as the right tracker.

## Evidence

- **WeChat is documented as external-plugin-owned:** The WeChat docs say the code does not live in the OpenClaw core repo, and explicitly assign WeChat login and Tencent iLink API calls to the external plugin.
  - file: [docs/channels/wechat.md:27](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/channels/wechat.md#L27)
  - command: `nl -ba docs/channels/wechat.md | sed -n '1,170p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Core login path only dispatches to plugin auth.login:** `runChannelLogin` resolves the selected channel plugin, then calls `plugin.auth.login(...)`. There is no Weixin-specific QR-fetch or iLink request implementation in core on this path.
  - file: [src/cli/channel-auth.ts:89](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/cli/channel-auth.ts#L89)

- **Explicit channel auth loads a scoped plugin snapshot:** `resolveInstallableChannelPlugin` uses `loadChannelSetupPluginRegistrySnapshotForChannel(...)` for the selected channel, and the snapshot loader narrows setup loading with `onlyPluginIds` when it has a trusted channel-to-plugin mapping.
  - file: [src/commands/channel-setup/channel-plugin-resolution.ts:214](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/commands/channel-setup/channel-plugin-resolution.ts#L214)

- **Tests enforce no broad setup-only preload without targeting:** Loader coverage verifies that a disabled setup-only channel plugin is not loaded by a broad setup registry pass, and only materializes when explicitly targeted with `onlyPluginIds`.
  - file: [src/plugins/loader.test.ts:5018](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/plugins/loader.test.ts#L5018)
