## Summary

Keep open. Current `main` has partial source attribution in memory search results and a separate provenance-rich `memory-wiki` companion, but the active durable-memory path (`memory-core` promotion into `MEMORY.md`) does not implement the provenance model this issue asks for: promoted entries carry only a single inline `source=path:lines` string, promotion gating does not require provenance, and the core durable-memory data model does not track multiple supporting sources or provenance history across edits/merges.

## Evidence

  - file: [docs/concepts/memory.md:43](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/docs/concepts/memory.md#L43)
