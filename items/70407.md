## Summary

Current main does not yet solve the reported upgrade break. `memory-core` still declares `dreaming.storage` as an object-only plugin config schema, config loading still validates plugin entries against that schema and throws `INVALID_CONFIG` on failure, and there is no current doctor/legacy migration for the old scalar `dreaming.storage: "both"` shape.

## Best Possible Solution

Keep the issue open and add a narrow backward-compatibility repair for `plugins.entries.memory-core.config.dreaming.storage` when it is one of the legacy scalar modes (`inline`, `separate`, `both`). The safest path is a config/doctor migration before plugin schema validation that rewrites it to `{ mode: <legacy>, separateReports: false }`, plus a regression test covering `loadConfig`/snapshot validation and `openclaw doctor --fix` behavior for the exact reported shape.
