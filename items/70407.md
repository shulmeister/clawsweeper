## Summary

Keep #70407 open. Current main still validates `plugins.entries.memory-core.config.dreaming.storage` through the memory-core manifest schema before the runtime resolver can normalize it, and that schema only accepts an object. I did not find a raw config migration or doctor repair path that converts the legacy string form (`"both"`, `"inline"`, `"separate"`) into `{ mode, separateReports }`.

## Best Possible Solution

Keep the issue open and fix it in the memory-core/config repair path. The conservative path is to add a raw config migration or `doctor --fix` repair for `plugins.entries.<memory-plugin>.config.dreaming.storage` when it is one of the legacy string modes, rewrite it to `{ mode: <value>, separateReports: false }`, add regression coverage for config validation/doctor repair, and include a targeted error hint if validation still fails.
