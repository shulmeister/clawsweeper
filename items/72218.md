## Summary

Keep #72218 open. Current main still returns raw heartbeat final payloads from `buildReplyPayloads`, so `HEARTBEAT_OK` token sanitation is not applied at that shared final-payload seam. The dedicated heartbeat runners do their own ack normalization, which narrows the end-to-end leak path, but it does not make this PR obsolete because the helper still violates the documented heartbeat reply contract and lacks the focused regression coverage proposed by the PR.

## Best Possible Solution

Keep #72218 open for maintainer review or land an equivalent narrow change in `src/auto-reply/reply/agent-runner-payloads.ts`: apply `HEARTBEAT_OK` stripping to heartbeat final payloads, preserve media-bearing payloads after stripping, keep Bun socket error formatting scoped to non-heartbeat runs, and add the focused `isHeartbeat: true` regression tests in `src/auto-reply/reply/agent-runner-payloads.test.ts`. Treat #72253/#69492 as separate broader exec/system-event leakage work.
