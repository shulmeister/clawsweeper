## Summary

Current `main` no longer carries resolved provider secrets through the model catalog or agent system prompt, and the remaining config-read surfaces redact provider `apiKey` values.

## Evidence

- **Model catalog shape is metadata-only:** `ModelCatalogEntry` contains only `id`, `name`, `provider`, `alias`, `contextWindow`, `reasoning`, and `input`, and `loadModelCatalog()` copies only those fields into the runtime catalog array.
  - file: [src/agents/model-catalog.types.ts:3](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/agents/model-catalog.types.ts#L3)
- **Catalog loader strips provider secret fields:** The catalog loader iterates discovered entries and pushes only `{ id, name, provider, contextWindow, reasoning, input }`, so provider `apiKey`/header data is not retained in the prompt-visible catalog object.
  - file: [src/agents/model-catalog.ts:141](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/agents/model-catalog.ts#L141)
- **System prompt gets aliases, not provider config:** Embedded runs pass `buildModelAliasLines(params.config)` into prompt construction; that helper emits only `- alias: provider/model`, and the system prompt renders only that alias section.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:914](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/agents/pi-embedded-runner/run/attempt.ts#L914)
- **models.json persistence now normalizes secrets to markers:** Provider normalization reverse-looks up resolved env-backed `apiKey` values and replaces them before persistence, with docs explicitly stating regenerated `models.json` stores env var names or SecretRef markers instead of resolved plaintext.
  - file: [src/agents/models-config.providers.normalize.ts:77](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/agents/models-config.providers.normalize.ts#L77)
- **Config-read surfaces redact apiKey values:** `config.get` responds with `redactConfigSnapshot(...)`, and redaction tests explicitly cover `models.providers.openai.apiKey` so gateway config inspection no longer returns plaintext keys.
  - file: [src/gateway/server-methods/config.ts:397](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/gateway/server-methods/config.ts#L397)
- **Status-path review found no current key-printing formatter:** A repo search across `src/commands/status*` found no API-key masking/printing path; the remaining auth label helper returns only source labels like `api-key (models.json)`, not secret fragments.
  - file: [src/agents/model-auth-label.ts:60](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/agents/model-auth-label.ts#L60)
  - command: `rg -n "resolveModelAuthLabel|authLabel|maskApiKey|models.json\)|api-key \(" src/commands/status* src/commands -g '!**/*.test.ts'`
- **Release notes document shipped models.json secret hardening:** The changelog for `2026.3.7` includes `Secrets/models.json persistence hardening` and `Models/provider apiKey persistence hardening`, matching the leak surface described in this issue.
  - file: [CHANGELOG.md:3006](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/CHANGELOG.md#L3006)

## Close Comment

Closing from a Codex review of current `main`. The leak path described here is not present anymore: `src/agents/model-catalog.types.ts:3` and `src/agents/model-catalog.ts:141` limit the runtime model catalog to safe metadata fields, `src/agents/pi-embedded-runner/run/attempt.ts:914`, `src/agents/model-alias-lines.ts:4`, and `src/agents/system-prompt.ts:774` show the system prompt only receives model alias lines rather than provider configs, and `src/agents/models-config.providers.normalize.ts:77` plus `docs/cli/agent.md:56` show regenerated `models.json` persists env-var/SecretRef markers instead of resolved plaintext secrets. Gateway config reads are also redacted via `src/gateway/server-methods/config.ts:397`, with explicit coverage for `models.providers.*.apiKey` in `src/config/redact-snapshot.test.ts:35`. Based on that current-main inspection, this is implemented on main. If someone can still reproduce on a current build, please open a new issue with the exact version, provider mix, and a sanitized captured request payload.
