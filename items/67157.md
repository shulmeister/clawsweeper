## Risks / Open Questions

- Merging the current PR shape without updating the npm release metadata guard would make release preflight reject the changed bin path.
- A launcher that imports `runCli` directly can bypass `src/entry.ts` bootstrap behavior unless respawn, warning filtering, env normalization, and compatibility setup are preserved.
- Changing the npm bin target without updating gateway argv detection can affect gateway lock/liveness classification in shim environments that expose the resolved script path.
- The PR discussion includes unresolved review findings in the added live smoke and tarball script, including async error handling, output parsing, tarball path resolution, and verifying the intended installed binary.
