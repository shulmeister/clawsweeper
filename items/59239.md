## Summary

Current main already ships the requested architectural memory-retrieval pass via the optional `active-memory` plugin: it runs a bounded blocking memory sub-agent before eligible conversational replies, injects recalled context automatically, and shipped in release `v2026.4.12`.

## Evidence

- **Shipped in changelog:** The `2026.4.12` release notes say OpenClaw added an optional Active Memory plugin that runs a dedicated memory sub-agent right before the main reply so users do not have to manually say "remember this" or "search memory" first.
  - file: [CHANGELOG.md:756](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/CHANGELOG.md#L756)

- **Docs match the issue's motivation:** The Active Memory docs explicitly say normal memory is too reactive, that models often fail to decide to search memory on their own, and that Active Memory gives the system a bounded chance to surface relevant memory before the main reply is generated.
  - file: [docs/concepts/active-memory.md:10](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/docs/concepts/active-memory.md#L10)

  - file: [extensions/active-memory/openclaw.plugin.json:2](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/active-memory/openclaw.plugin.json#L2)

- **Main implementation hooks before prompt build:** The plugin registers `before_prompt_build`, checks eligibility, builds a recall query from the current prompt plus recent turns, runs recall, and returns `prependContext` so memory is injected before the main model reply.
  - file: [extensions/active-memory/index.ts:1964](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/active-memory/index.ts#L1964)

## Risks / Open Questions

- The shipped implementation is scoped to eligible conversational sessions, not a generic per-tool trigger DSL; if maintainers want mandatory recall for automation/headless tool paths too, that would be a separate follow-up rather than evidence this issue is still unimplemented.
