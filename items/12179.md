## Summary

Current main already has an optional bundled `active-memory` feature that performs a pre-reply memory recall pass and injects the result into prompt context before the main agent answers.

## Risks / Open Questions

- The shipped feature is intentionally scoped to targeted agents and eligible interactive persistent chat sessions, so it is not a blanket hook for every possible turn type.

## Close Comment

Closing as implemented on current `main`. This was a Codex review against the checked-out repository at `86f69ba5a06ae256906c3a26308801e0b84e0091`. OpenClaw already ships an optional bundled `active-memory` feature that runs on `before_prompt_build`, performs a bounded memory recall pass from the latest user message/recent context, and prepends recalled context before the main reply (`extensions/active-memory/index.ts:1964`). The feature is configurable under `plugins.entries.active-memory` (`extensions/active-memory/openclaw.plugin.json:4`) and documented as running before the main reply to surface relevant memory (`docs/concepts/active-memory.md:10`). The changelog shows this shipped in `v2026.4.12` (`CHANGELOG.md:701`).
Codex evidence: Reviewed against main [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091). Fix evidence: release [v2026.4.12](https://github.com/openclaw/openclaw/releases/tag/v2026.4.12).
