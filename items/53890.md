## Summary

Current main still does not implement config-level default outbound Telegram topic/thread binding per agent. Telegram topic config supports inbound/topic routing with `agentId`, explicit outbound `threadId` and `:topic:` targets work, and current-context auto-threading works, but there is no accepted `outbound` topic flag or reverse per-agent default topic lookup for proactive sends. The prior ClawSweeper comment remains accurate against current main, and the related items in the provided context do not supersede this feature request.

## Best Possible Solution

Keep this issue open. If maintainers want the feature, implement it in the Telegram plugin/config surface with typed config, schema/help/generated docs, and tests. A conservative design should define precedence explicitly: explicit `threadId` or `:topic:` target first, current conversation/tool context next, then configured per-agent outbound default. The wiring should stay Telegram-owned and avoid adding Telegram-specific policy to generic outbound core beyond any necessary generic seam.
