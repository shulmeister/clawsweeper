## Summary

Keep open. Current main still generates the Windows Startup-folder login item as a `.cmd` launcher that runs `start "" /min cmd.exe /d /c ...`, and the startup-fallback test explicitly asserts that content. The related closed #57682/update-restart path has separate PowerShell/netstat coverage and does not fix the Startup-folder login-item flash reported here.

## Best Possible Solution

Implement a narrow Windows daemon fix in `src/daemon/schtasks.ts`: keep the existing Startup-folder `.cmd` path for detection, generate a companion silent launcher or another no-window handoff that avoids the login flash, remove companion artifacts on uninstall/repair, and update `schtasks.startup-fallback.test.ts` plus Windows docs if the user-visible startup fallback behavior changes.
