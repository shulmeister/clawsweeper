## Summary

Keep open. Current main still does not implement interactive custom Feishu account IDs during onboarding, and the PR discussion identifies blocking bugs in the proposed patch rather than evidence that the work is obsolete or superseded.

## Best Possible Solution

Keep the PR open for revision. The maintainer path is to thread the resolved `accountId` through Feishu `finalize`, `runEditFlow`, and `runNewAppFlow`, write credentials and policy through `patchFeishuConfig` for that selected account, prompt only when a real Feishu account already exists, and add focused tests for first-run setup plus second named-account onboarding.

## Evidence

- **Feishu wizard disables account prompting:** `resolveAccountIdForConfigure` chooses an explicit override or the current Feishu default, while `resolveShouldPromptAccountIds` always returns `false`; current main therefore does not prompt for a new Feishu account id during onboarding.
