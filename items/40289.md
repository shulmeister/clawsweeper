## Summary

Keep #40289 open. The PR is stale against current file layout, but the underlying Discord button-context bug still appears present on current main: component card text is rendered into Discord components, but it is not persisted with the component entry or included in the ordinary agent fallback event text when a button/select is clicked. The related reports in the provided context cover broader interaction metadata and referenced-message extraction gaps, but they do not supersede this focused fix.

## Best Possible Solution

Keep the PR open, but require a rebase/rework against `extensions/discord`: add a structured card-context field to the component entry type, collect `spec.text` and relevant text/section blocks in `buildDiscordComponentMessage`, pass that through the registry and monitor fallback path, append it to the agent event text, and update `extensions/discord/src/components.test.ts` plus `extensions/discord/src/monitor/monitor.test.ts`. The Discord-specific message-tool prompt guidance from the PR should also be reconsidered against the current channel `messageToolHints` and shared presentation docs.
