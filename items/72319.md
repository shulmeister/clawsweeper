## Summary

Keep PR #72319 open. Current main still only truncates text tool-result blocks before emitting them, while the agent-event path can forward full tool results to the Control UI and the UI renders `partialResult`/`result` text. The linked security issue remains a real current-main gap. The PR is focused on the right chokepoint, but the existing review comment about preserving user-configured `logging.redactPatterns` should be addressed before merge.

## Best Possible Solution

Keep the PR open and land a small core fix at the existing `sanitizeToolResult()` chokepoint, redacting text blocks before truncation for both partial and final tool events. The implementation should preserve existing image stripping/truncation behavior, apply default secret patterns, include user-configured `logging.redactPatterns` unless maintainers explicitly choose otherwise, and add regression tests for JSON/env/bearer/common-prefix secrets plus a custom-pattern case.
