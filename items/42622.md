## Summary

Current main already scopes Discord ack/status reactions to the active account, respects `messages.statusReactions.enabled=false`, and documents/tests account-scoped routing so non-default Discord accounts do not inherit channel-only bindings unless `accountId: "*"` is configured.

## Close Comment

Closing from a Codex review because current `main` already implements the requested behavior. Discord ack/status reactions are scoped to the active account context in `extensions/discord/src/monitor/message-handler.process.ts:229` and `extensions/discord/src/monitor/ack-reactions.ts:13`, and the current regression tests cover both `messages.statusReactions.enabled=false` (`extensions/discord/src/monitor/message-handler.process.test.ts:537`) and named-account reaction routing (`extensions/discord/src/monitor/message-handler.process.test.ts:390`). Routing is also account-scoped: `src/routing/resolve-route.ts:443` resolves bindings per `channel + accountId`, `src/routing/resolve-route.test.ts:451` verifies that bindings without `accountId` match only the default account, and `docs/cli/agents.md:62` documents that `accountId: "*"` is the only all-accounts fallback. This implementation is already present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
