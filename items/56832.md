## Summary

Current `main` already guards the `/new`/reset overlap described here: reset cleanup aborts and waits for the prior embedded run, refuses the reset if the old run is still active, and the reply path now turns overlap/shutdown failures into user-facing text instead of leaking raw backend payloads. The same implementation is present in release `v2026.4.23`.

## Evidence

- **Reset cleanup now aborts and waits before rotating the session:** `ensureSessionRuntimeCleanup()` clears queues, aborts the prior embedded run, waits up to 15s for it to end, and returns `UNAVAILABLE` instead of rotating into a bad state when the old run is still active.
  - file: [src/gateway/session-reset-service.ts:194](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/gateway/session-reset-service.ts#L194)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Reply path waits for prior active run and returns a clean overlap message:** `resolvePreparedReplyQueueState()` waits for the previous active run to finish; if it is still active afterward, it returns `⚠️ Previous run is still shutting down. Please try again in a moment.` rather than continuing into a reset race.
  - file: [src/auto-reply/reply/get-reply-run-queue.ts:12](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/auto-reply/reply/get-reply-run-queue.ts#L12)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **User-facing error normalization prevents raw backend payload leakage:** `buildExternalRunFailureText()` maps backend failures to friendly fallback text (`Something went wrong... use /new...`) instead of surfacing raw internal payloads in chat.
  - file: [src/auto-reply/reply/agent-runner-execution.ts:372](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/auto-reply/reply/agent-runner-execution.ts#L372)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Regression tests cover the active-run reset race:** Tests assert that `sessions.reset` aborts active runs and clears queues, and that it returns `UNAVAILABLE` without rotating the session when the active run does not stop.
  - file: [src/gateway/server.sessions.gateway-server-sessions-a.test.ts:2689](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/gateway/server.sessions.gateway-server-sessions-a.test.ts#L2689)
