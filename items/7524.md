## Summary

Keep #7524 open. Current main still has no `groupScope`, per-group main-session routing, or general shared-session mapping for group/channel peers. The focused BlueBubbles/Signal use case remains valid, and the existing discussion includes both a user +1 for Signal groups and a maintainer Codex review explicitly keeping this item open rather than treating #19929 as canonical.

## Best Possible Solution

Keep #7524 open as the focused group-to-main/shared-session tracker unless maintainers explicitly mark #19929 canonical. The likely implementation is an additive routing/config contract for selective group/channel/thread sessions to use main or named shared sessions, wired through `buildAgentPeerSessionKey()`, `buildAgentSessionKey()`, `resolveAgentRoute()`, strict config schemas/types/help/generated metadata, relevant channel group configs, public docs, and regression tests for BlueBubbles, Signal, Telegram, Matrix, Discord, and Slack routing.
