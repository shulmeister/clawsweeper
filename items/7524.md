## Summary

Keep #7524 open. Current main still has a clear missing capability: group/channel session keys are hard-coded to isolated `agent:<agentId>:<channel>:<peerKind>:<peerId>` buckets, while only direct-message `dmScope` can collapse to main. The related #7493 appears to be the same feature class but is already closed, so #7524 is the useful open tracker; #27355 is a related group-session contamination report, not an implementation of group-to-main scoping. This is not a ClawHub cleanup candidate because the missing behavior sits in OpenClaw core routing/config contracts, not an optional plugin surface.
