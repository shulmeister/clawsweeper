## Summary

Keep #7524 open. Current main still lacks a declarative `groupScope` or equivalent per-group/group-to-main session routing contract; group/channel peers remain isolated in routing, config schemas only expose `dmScope`, and BlueBubbles/Signal group paths do not have a per-group main-session option. The open broader issue #19929 overlaps, but the provided maintainer comment explicitly kept #7524 open unless maintainers fold it into a broader design.

## Best Possible Solution

Keep #7524 open as the focused tracker unless a maintainer explicitly marks #19929 as the canonical shared-session design. The best implementation is a core routing/config contract for selective group/channel-to-main behavior, propagated through inbound and outbound session-key construction, route bindings, generated config metadata, docs, and regression tests for at least BlueBubbles and Signal group routing.
