## Summary

Keep #7524 open. Current main still lacks a selective `groupScope` or equivalent group-to-main routing contract; group/channel peers still route to isolated peer session keys, config schemas and docs expose only DM scoping, and the broader #19929 has not been marked canonical over this focused request.

## Best Possible Solution

Keep #7524 open as the focused tracker unless a maintainer explicitly marks #19929 as the canonical shared-session design. The best fix is a core routing/config contract for selective group/channel-to-main or named shared-session behavior, wired through inbound and outbound session-key construction, route bindings, generated config metadata, docs, and regression coverage for at least BlueBubbles and Signal group routing.
