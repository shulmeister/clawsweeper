## Summary

Keep #7524 open. Current main still only collapses direct-message peers through `dmScope`; group and channel peers continue to resolve to channel-scoped session keys, the strict config schemas expose no `groupScope`/shared-group option, and bundled BlueBubbles/Signal group paths feed `peer.kind: "group"` into the same isolated core route. The broader #19929 is open but has not been marked canonical over this focused request, and the latest maintainer comment explicitly kept #7524 open.

## Best Possible Solution

Keep #7524 open as the focused tracker unless a maintainer explicitly makes #19929 canonical. The best implementation is a core routing/config contract for selective group/channel-to-main or named shared sessions, wired through inbound and outbound session-key construction, route binding overrides, generated config metadata, public docs, and regression coverage for at least BlueBubbles and Signal group routing.
