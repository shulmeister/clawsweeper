## Close Comment

Closing this as a `clawhub` request after Codex review. On current `main`, OpenClaw's scope is to keep core lean and push optional integrations to plugins/ClawHub rather than merge them into core: `VISION.md:54-70` says optional capability should usually ship as plugins, memory is a plugin slot, and new skills should go to ClawHub first; `VISION.md:101-107` also lists core skill additions that can live on ClawHub as out of scope for now. The plugin docs already describe the intended external path in `docs/tools/plugin.md:11-16`, `docs/tools/plugin.md:26-69`, and `docs/tools/plugin.md:109-111`. I also found no `ITIR` or `statibaker` implementation on current `main`. If you want to pursue this, the right path looks like an external OpenClaw plugin/skill or memory plugin published and maintained in its own repo, then distributed through ClawHub/npm rather than as an OpenClaw core issue.
