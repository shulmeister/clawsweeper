## Summary

Keep open. Current `main` has substantial review/edit/conflict infrastructure for memory via `memory-wiki` and the Dreams UI, but it does not cleanly satisfy the issue's full MVP scope because there is no first-class non-destructive forget/tombstone flow, and one shipped forget path still hard-deletes data.

## Evidence

- **Review/listing surfaces exist:** `extensions/memory-wiki/src/gateway.ts` registers a read-only `wiki.palace` gateway method, and the Dreams UI describes that surface as a place to inspect memory pages, claims, open questions, and contradictions.
  - file: [extensions/memory-wiki/src/gateway.ts:147](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/extensions/memory-wiki/src/gateway.ts#L147)
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Dreams UI exposes the review surface:** The Dreams UI renders a Memory Palace view for compiled wiki pages with counts for pages, claims, questions, and contradictions, which covers part of the requested review/listing workflow.
  - file: [ui/src/ui/views/dreaming.ts:1111](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/ui/src/ui/views/dreaming.ts#L1111)
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Edit tooling exists without DB access:** `openclaw wiki apply` is documented as a safe mutation path for syntheses and metadata, including questions, contradictions, confidence/status, and structured claims.
  - file: [docs/cli/wiki.md:146](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/docs/cli/wiki.md#L146)
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Conflict representation and resolution primitives exist:** `memory-wiki` treats `contested`, `contradicted`, `refuted`, and `superseded` as contested claim states, and compile/lint generate contradiction and claim-health reports.
  - file: [extensions/memory-wiki/src/claim-health.ts:9](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/extensions/memory-wiki/src/claim-health.ts#L9)
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Issue scope is only partially implemented:** The documented wiki tooling lists review/edit/conflict-oriented flows, but there is no documented first-class `forget`/tombstone command in `docs/cli/wiki.md` or `docs/plugins/memory-wiki.md`; the supported `wiki apply` flows stop at metadata/status updates.
  - file: [docs/plugins/memory-wiki.md:223](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/docs/plugins/memory-wiki.md#L223)

  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Existing forget path is destructive, not tombstoned:** The shipped `memory_forget` tool in `extensions/memory-lancedb/index.ts` calls `db.delete(...)` directly for both `memoryId` and high-confidence query matches, which conflicts with the issue's non-destructive forget acceptance criterion.
  - file: [extensions/memory-lancedb/index.ts:462](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/extensions/memory-lancedb/index.ts#L462)
