## Summary

Current main still allows subagent completion delivery to proceed with no usable result text. The related PR #68464 appears to be the right candidate direction, but it is open/unmerged in the provided context and the shipped code still has the reported `(no output)` path, so this should not be closed yet.

## Best Possible Solution

Keep this issue open and link it to PR #68464 as the current candidate fix. The maintainer path should be to land or replace that work with a source-level change that persists a user-safe completion payload/claim, defers completion delivery when no usable payload exists, and updates the `(no output)` regression coverage to assert retry/defer or a sanitized real fallback instead of announcing early.
