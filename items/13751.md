## Summary

Keep open. Current main has partial mitigations for Feishu sender-name lookup noise, but it still performs sender-name resolution through the Feishu contact API by default. The issue discussion reports multiple affected organizations and links #19980 for false permission prompts even when the scope is granted, so the central request remains unresolved rather than obsolete or superseded.

## Best Possible Solution

Keep #13751 open. The best product/code path is to make basic Feishu bot operation avoid contact-directory/profile APIs by default: derive sender attribution from webhook data when available, otherwise use open_id/user_id without calling contact.user.get unless an explicit opt-in enables profile lookup. The live directory peer listing should also be explicit/opt-in or static-first, with docs and tests updated around the new default behavior.
