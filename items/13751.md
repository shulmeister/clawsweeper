## Summary

Keep open. Current main still performs Feishu inbound sender-name resolution through the contact user API by default, and the live directory peer path still uses contact user listing. The issue discussion remains applicable: users object to broad contact access and the related false-permission report does not supersede this request.

## Best Possible Solution

Keep #13751 open. Make basic Feishu inbound bot operation contact-free by default: use webhook sender fields when available, otherwise attribute with `open_id` or `user_id` without calling `contact.user.get`; make profile lookup an explicit opt-in. The live directory peer path should also be static-first or explicitly opt-in for contact-directory access, with Feishu docs and regression tests updated around the new default.
