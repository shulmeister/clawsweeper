## Summary

Keep #13751 open. Current main has a partial mitigation for stale `contact:contact.base:readonly` permission noise, but basic Feishu inbound sender-name resolution is still contact-API backed by default, and live peer listing still uses the contact directory path. The central request, contact-free basic bot operation by default, is not implemented.

## Best Possible Solution

Keep #13751 open. Make normal Feishu inbound bot operation contact-free by default: parse sender identity from webhook fields when available, otherwise use open_id/user_id without calling `contact.user.get`; move profile lookup behind an explicit opt-in. For directory peers, prefer configured static peers unless live contact-directory lookup is explicitly requested. Update Feishu config metadata, docs, and regression tests for the new default.
