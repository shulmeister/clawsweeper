## Summary

Keep open. Current main still resolves inbound Feishu sender names through `client.contact.user.get()` by default, and live peer listing still uses `client.contact.user.list()`. The repo has added opt-out and error-suppression mitigations, but the central request to remove the default dependency on contact/profile APIs for basic bot operation remains unresolved.

## Best Possible Solution

Keep #13751 open. The best product/code path is to make basic Feishu inbound bot operation avoid contact-directory/profile APIs by default: use webhook sender data when available, otherwise attribute with `open_id`/`user_id` without calling `contact.user.get`; make profile lookup an explicit opt-in. The live directory peer listing should also be static-first or explicitly opt-in for contact-directory access, with docs and tests updated around the new default behavior.
