## Summary

Keep open. Current main still routes subagent/ACP completion handoff through the requester-session announce agent call, and the available configuration only covers child subagent model/light bootstrap or announce timeout. The related ClawSweeper items cover delivery correctness and reliability, not the requested cheaper announce model or light announce context controls.

## Best Possible Solution

Add an explicit, schema-backed announce policy for task-completion handoff, likely under `agents.defaults.subagents.announce`, with `model`/fallbacks and a light-context or no-history mode. Wire it only into the internal announce `agent` call, preserve existing delivery/idempotency/nested-session behavior, document it on the subagents and ACP delivery pages, and add focused tests proving the announce run uses the configured cheaper model/light context while child subagent model selection remains unchanged.
