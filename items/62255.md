## Summary

Keep open. Current main does not implement a configurable global or provider-level model-call rate limiter across agents. Existing behavior is reactive rate-limit cooldown/profile rotation/fallback after provider errors, plus unrelated ingress/webhook rate limiters. The requested capability also does not cleanly fit a ClawHub-only close because the current model call path lacks an awaited pre-call plugin hook that could enforce a global queue before provider submission.
