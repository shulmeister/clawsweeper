## Summary

The request is clear and useful, but current main shows Perplexity support in OpenClaw is already plugin-scoped, and the missing LLM/model-provider capability can be built with the existing provider plugin API rather than tracked as an OpenClaw core issue. The existing bundled Perplexity plugin is web-search only; a first-class Perplexity model provider should live as a ClawHub/OpenClaw provider plugin unless maintainers explicitly decide to bundle it.

## Risks / Open Questions

- Maintainers may still want an official bundled Perplexity model provider because VISION.md also lists major model-provider support as a future priority.
- Perplexity's chat-completions behavior may need provider-specific compatibility hooks, but the current provider SDK exposes hooks for catalogs, dynamic model resolution, transport normalization, stream wrapping, and replay/tool compatibility.
