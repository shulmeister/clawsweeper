## Summary

Keep #69519 open. Current main still renders the Sessions Limit control as a plain string-backed input, while the gateway schema expects an integer limit. The PR targets a real UI validation gap and is recent, but the provided review context shows it still needs cleanup around state typing and empty/invalid value handling before it should be merged.

## Best Possible Solution

Keep the PR open and ask for a coherent Sessions limit implementation: either keep string UI state and parse only at the controller boundary, or migrate the whole Control UI Sessions limit state to `number | undefined`; in either case, use a numeric input with `min=1`/`step=1`, preserve a clear blank/invalid state, and add focused view/controller tests for the input and resulting `sessions.list` params.
