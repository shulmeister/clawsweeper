## Summary

Keep #39565 open. Current main at 73e21511075d7ed5651faae305f439d65b6ef47c still uses the pre-PR TUI copy-sensitive-token classifier, so punctuation-wrapped entity IDs and long dotted IDs without underscores can still be split by sanitizeRenderableText. The linked bug #39505 remains open, and the provided PR context shows a focused fix with regression tests and follow-up commits addressing prior bot review concerns.

## Best Possible Solution

Keep this PR or the linked #39505 work open. The best path is a focused TUI formatter change in src/tui/tui-formatters.ts that classifies punctuation-stripped candidates for copy-sensitive checks, strips markdown asterisks at token edges, preserves dotted identifiers and stronger hyphenated ID forms, and keeps punctuation-only or natural-language hyphenated long tokens chunkable, with regression coverage in src/tui/tui-formatters.test.ts.
