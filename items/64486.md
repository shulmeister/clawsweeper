## Summary

Current main still implements the existing transparency-note behavior, not the requested forced pre-send choice. The reply text is detected after the model turn, a note is appended when no successful cron add or existing enabled session cron is found, and tests explicitly assert that hollow commitment text is still delivered with the disclaimer.

## Best Possible Solution

Keep this issue open. The best path is a focused core or hook-backed implementation in the agent finalization/reply pipeline that reuses the existing commitment detection plus `successfulCronAdds`/existing-cron checks, then requests one revision before dispatch: either remove commitment language or perform a concrete scheduling action. If maintainers prefer this as plugin policy, expose any missing per-turn scheduling/task/heartbeat evidence through the finalization hook instead of adding a one-off hidden path.
