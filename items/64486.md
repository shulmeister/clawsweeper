## Summary

Current main still implements the after-the-fact transparency note, not the requested pre-send forced-choice gate. The request is a concrete core auto-reply behavior change, recent, unprotected by labels/author association, and not satisfied by the existing cron reminder guard or docs.

## Best Possible Solution

Keep the issue open and implement this in the core auto-reply/agent-runner reminder-commitment path, likely reusing hasUnbackedReminderCommitment plus successful cron tracking, with an explicit opt-in config if maintainers want a gradual rollout. The implementation should add tests that prove unbacked commitment text is not delivered unchanged with only an appended disclaimer when the gate is enabled.
