## Summary

Current main already addresses the reported post-output CLI hang class. Root help now uses a fast path before full program construction, `status` and `gateway status` route before full plugin registration and skip plugin loading, and one-shot Gateway RPC calls wait for bounded WebSocket teardown before resolving. The cross-referenced #64216 and related Windows report #64853 corroborate the same CLI/bootstrap hang family, and the shipped changelog records the Gateway RPC teardown fix in v2026.4.22.

## Best Possible Solution

Close this issue as implemented on main. Keep the shipped route-first/no-plugin startup policy for read-only CLI commands and the bounded Gateway RPC teardown behavior; if a current release still reproduces a platform-specific hang, track it as a fresh issue with exact version, shell, command timings, active handle data, and affected plugin/profile details.
