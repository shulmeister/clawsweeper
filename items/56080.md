## Summary

PR #56080 should close as duplicate/superseded. The node systemd unit-name bug is still present on current main, but open PR #39580 already tracks the same remaining systemd fix in a narrower one-file branch. #56080 carries that same `src/daemon/systemd.ts` change plus unrelated `tools.exec.nodeCwd` config/runtime changes, so the best maintainer cleanup is to follow #39580 or recreate the narrow fix with tests.

## Best Possible Solution

Close #56080 as duplicate/superseded by open PR #39580, then land or recreate the narrow `src/daemon/systemd.ts` resolver fix with a regression test that asserts `OPENCLAW_SYSTEMD_UNIT=openclaw-node` drives install activation and uninstall disable targets. Split the unrelated `tools.exec.nodeCwd` work into a separate PR if maintainers want that feature.
