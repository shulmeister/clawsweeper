## Summary

Keep PR #43789 open. Current main still compacts prompt-facing skill paths only by home-directory prefix (`~/...`) and does not implement the PR’s workspace-relative `skills/.../SKILL.md` prompt locations for workspace-local skills. The related sandbox host-path problem is still tracked by open #43383/#43526, so this PR is not safely obsolete or implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review or explicitly supersede it only after choosing a canonical sandbox skill-path fix. The best implementation path is to make prompt locations sandbox-readable for workspace-local skills, preserve prompt-budget behavior using the same rendered path strings, and add regression coverage for `skills/.../SKILL.md` plus the embedded-run snapshot path that currently returns `skillsSnapshot.prompt` unchanged.
