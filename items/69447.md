## Summary

Keep open. Current main still has the reported end-to-end limitation: the Control UI picker and handlers are image-only, the UI request serializer still emits every attachment as an image, and the Gateway parser still drops sniffed non-image payloads. The linked fix PR #70947 is open and not merged, so this remains a real current-main product gap rather than cleanup-close material.

## Best Possible Solution

Keep #69447 open until an end-to-end generic attachment fix lands on main. The likely path is to review and land #70947 or a successor that updates the Control UI accept/filter/serialization and Gateway attachment parsing/offload behavior, with regression tests for PDFs or generic files reaching transcript/media context while preserving existing image behavior. Split the feedback-channel meta into a separate issue if maintainers want it tracked independently.
