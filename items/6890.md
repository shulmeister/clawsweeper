## Summary

Keep open. Current main still does not implement a Ralph Loop feature or a per-agent max Ralph iteration/default loop setting. The issue discussion already contains a recent maintainer-visible Codex keep-open review, and the relevant current source still only supports agent runtime fields such as `embeddedPi.executionContract`, tool-loop detection guardrails, and fixed retry paths, not a configurable STOP-driven repeated same-prompt loop.

## Best Possible Solution

Keep #6890 open. If accepted, implement an OpenClaw-native per-agent loop/default surface in config types, strict zod schemas, generated config help/labels/reference docs, and `resolveAgentConfig()`, then wire the embedded runner with explicit precedence against higher-priority runtime/CLI overrides. Add tests for global default, per-agent override, override precedence, disabled, finite, and unlimited semantics, and keep it clearly distinct from strict-agentic planning-only retries and `tools.loopDetection`.
