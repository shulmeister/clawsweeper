## Summary

Keep open. Current main still does not implement Ralph Loop or an agent-scoped max Ralph iteration/default setting. The request is detailed and actionable, has only an enhancement label, and the latest issue discussion already includes a maintainer-side Codex review recommending the same keep-open outcome.

## Best Possible Solution

Keep the issue open for maintainer product judgment. If accepted, implement an OpenClaw-native per-agent loop configuration surface in the agent config types, strict schemas, schema help/labels/generated reference, docs, and config resolver; then wire it into the embedded runner with explicit precedence against global/runtime overrides. Keep it separate from existing tool-loop detection and strict-agentic planning retries unless maintainers intentionally merge those concepts, and add tests for default, per-agent override, higher-precedence override, disabled, finite, and unlimited semantics.
