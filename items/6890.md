## Summary

Keep open. Current main still does not implement a Ralph Loop feature or the requested per-agent `loop.max_ralph_iterations` configuration surface. The only current loop-related config I found is tool-loop safety detection, which prevents repetitive tool calls; it is not a STOP-driven repeated same-prompt agent loop with CLI/per-agent/global precedence.

## Best Possible Solution

Keep #6890 open. If maintainers accept the feature, add an OpenClaw-native loop/default config surface to `AgentConfig`, strict zod schemas, generated config/help docs, and `resolveAgentConfig`, then wire the embedded runner with explicit precedence against higher-priority runtime or CLI overrides. Add focused tests for global default, per-agent override, override precedence, disabled, finite, and unlimited behavior. If maintainers decide this belongs outside core, first define the plugin/runtime API needed to express the loop safely.
