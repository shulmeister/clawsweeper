## Summary

Keep open. Current main still does not implement Ralph Loop or an agent-scoped max Ralph iteration/default setting. OpenClaw has adjacent strict-agentic retry behavior for GPT-5 plan-only stalls, but that is fixed-cap retry handling, not a STOP-driven repeated same-prompt loop with configurable per-agent iteration precedence. The issue is coherent, author association is NONE, labels are not protected, and the discussion includes a supporting +1 plus a maintainer Codex review that already recommended keeping it open.

## Best Possible Solution

Keep the issue open for maintainer product judgment. If accepted, implement an OpenClaw-native loop configuration surface in the agent config types, strict schemas, schema help/docs, and agent config resolver; then wire it into the embedded agent runner with explicit precedence against any global or CLI runtime controls that exist, while keeping strict-agentic plan-only retry behavior as a separate contract unless maintainers intentionally merge the designs. Add tests for default, per-agent override, higher-precedence override, disabled, finite, and unlimited semantics.
