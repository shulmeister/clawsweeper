## Summary

Keep open. Current main still does not implement Ralph Loop or an agent-scoped max Ralph iteration/default setting. The issue is detailed and coherent, has no protected labels, and an existing maintainer-side Codex review already recommended keeping it open; this re-check against current main reaches the same conclusion.

## Best Possible Solution

Keep the issue open for maintainer product judgment. If accepted, implement an OpenClaw-native per-agent loop configuration surface in the agent config types, strict schemas, schema help/docs, and agent config resolver; then wire it into the embedded runner with explicit precedence against any global or CLI runtime controls that exist. Keep strict-agentic plan-only retry behavior and tool-loop detection separate unless maintainers intentionally merge the designs, and add tests for default, per-agent override, higher-precedence override, disabled, finite, and unlimited semantics.
