## Summary

Current `main` already surfaces Chrome MCP existing-session attach failures explicitly, preserves the underlying attach error during readiness checks, and includes a regression test for the `DevToolsActivePort` case. The fix appears to be on `main` after the latest tagged release, so it is implemented but not yet tied to a shipped release tag.

## Evidence
