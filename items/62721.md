## Summary

Keep open. Current main still retains the Discord gateway supervisor error listener after dispose, so the leak concern is not implemented on main. However, the proposed one-line removal is not safe to auto-close or merge as-is because current code and tests intentionally keep handling late EventEmitter "error" events after dispose to avoid uncaught errors.
