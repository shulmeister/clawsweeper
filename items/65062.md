## Summary

Current `main` already addresses both reported failure modes: QMD startup now rebinds malformed managed collections instead of preserving stale `memory-root-* -> **/*.md` state, and provider/runtime plugin resolution is scoped so disabled or out-of-allowlist provider-owner plugins are not activated during normalization. The remaining `plugins.allow`-empty auto-load warning is intentional policy, not the reported runtime bypass.

## Evidence

- **QMD startup now rebinds stale managed collections:** `ensureCollections()` removes and recreates a managed collection when the listed path/pattern no longer matches, and `shouldRebindCollection()` explicitly treats mismatched patterns as a rebind condition. That covers the reported malformed `memory-root-*` broad-glob state.

## Risks / Open Questions

- The fix is present on `main`; I did not find evidence in the checked-out repo that it shipped in the latest release tag, so `fixedRelease` remains unknown.
- If an operator leaves `plugins.allow` empty, discoverable non-bundled plugins may still auto-load by design; hard lock-down still requires an explicit allowlist.
