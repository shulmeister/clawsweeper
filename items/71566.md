## Summary

Keep open. Current main still does not expose Gateway RPC handlers, method discovery entries, or operator-scope mappings for directory.self, directory.peers.list, and directory.groups.list. This PR is a plausible in-repo Gateway protocol addition over the existing channel directory adapter contract, not obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to finish it as a proper Gateway protocol addition: add typed params/result schemas and validators for directory.self, directory.peers.list, and directory.groups.list; add focused tests for handlers, method discovery, read-scope authorization, unsupported capabilities, runtime UNAVAILABLE errors, and cache-first lookup behavior; update public Gateway protocol docs and generated client artifacts if this is intended to be a stable WS RPC surface; then land this PR or an equivalent implementation.
