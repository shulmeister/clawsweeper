## Summary

Current main does not implement the PR’s gateway RPC surface. OpenClaw already has channel directory adapters and an `openclaw directory` CLI, but `directory.self`, `directory.peers.list`, and `directory.groups.list` are not registered, scoped, or wired into `coreGatewayHandlers`, so the PR is plausible unfinished gateway work rather than conservative cleanup material.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it on current main, add targeted tests for `directory.self`, `directory.peers.list`, and `directory.groups.list`, register the methods and read scopes, and verify invalid params, unsupported capabilities, cache-first lookup, account resolution, and runtime error-code behavior.
