## Summary

Keep #37656 open. Current main still does not implement per-agent, exec-time workspace `.env` loading for `exec`; the PR remains the plausible implementation path for the closed-but-linked #37401 request rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open as the canonical implementation path for per-agent workspace `.env` loading. The best code path is a reviewed core `exec` implementation that loads the resolved gateway workspace `.env` at request time, preserves non-override semantics, filters dangerous host env overrides, skips node/sandbox hosts where appropriate, and lands focused regression tests that do not write to the repo root.
