## Summary

Keep open. Current main does not implement per-agent, exec-time workspace `.env` loading. The exec tool still builds host execution env from `process.env` plus request `params.env`, while dotenv loading remains startup/CWD/global behavior. The PR is a plausible core exec change with substantive security and reliability review history, not an obsolete cleanup candidate.

## Best Possible Solution

Keep the PR open and continue it as the canonical implementation path for #37401-style per-agent workspace `.env` loading. The best product/code direction is to land a carefully reviewed core exec implementation that loads the resolved gateway workspace `.env` at request time, preserves non-override semantics, filters dangerous host env overrides, skips node/sandbox hosts where appropriate, and adds focused regression tests without writing to the repo root.
