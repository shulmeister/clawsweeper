## Summary

Keep #37656 open. Current main still does not load a resolved per-agent workspace `.env` into the `exec` tool at request time. The existing implementation passes the agent workspace as `cwd`, then builds exec env from `process.env` plus explicit request overrides. The PR is still a focused implementation path, and the supplied PR discussion shows active review work on real edge cases rather than an obsolete or duplicate change.

## Best Possible Solution

Keep #37656 open as the canonical implementation path for the remaining #37401 behavior. The best maintainer path is a reviewed core `exec` change that loads the resolved gateway workspace `.env` at request time, preserves non-override semantics including Windows casing, filters dangerous host env overrides, skips node/sandbox hosts where workspace env is not executed, adds isolated regression tests, and updates environment docs once shipped.
