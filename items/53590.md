## Summary

Keep this issue open. Current main still only has a partial configured-primary warmup warning and hook-specific doctor validation; the reported OpenRouter invalid/retired slug path remains unhandled because OpenRouter dynamic resolution still creates a runtime model for uncatalogued IDs instead of treating the live-catalog miss as a diagnostic.

## Best Possible Solution

Keep this open and add a non-fatal diagnostic path for configured primary and fallback model refs. Gateway startup and openclaw doctor should warn when a provider with a successfully fetched live catalog can prove the configured ref is absent, while preserving OpenRouter’s dynamic-model escape hatch when the catalog cannot be reached or the provider intentionally accepts custom IDs. Changelog notes should call out known retired/renamed slugs when OpenClaw changes bundled defaults or compatibility behavior.
