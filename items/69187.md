## Summary

Current main does not yet fully fix #69187. The memory-core dreaming narrative path still depends on `subagent.deleteSession()` for cleanup, gateway fallback session deletion still requires `operator.admin`, and the existing scrubber only removes missing-file stale entries or unreferenced orphan transcripts. The linked #70464 PR is an open, unmerged fix for this exact failure mode, so the issue should remain open until that or an equivalent fix lands.

## Best Possible Solution

Keep #69187 open and land or replace #70464 with a tested fix: bounded/stable memory-core dreaming session keys or separate idempotency, a least-privilege lifecycle-independent way for memory-core to delete only its own `dreaming-narrative-*` transient sessions, and regression coverage proving cron fallback cleanup removes the referenced session-store entry and transcript without granting broad plugin admin rights.
