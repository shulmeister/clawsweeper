## Summary

Keep #39719 open. Current main still reuses persisted session skill snapshots across workspace changes: the snapshot schema has no workspace/source provenance, `agent-command` refreshes only on missing snapshot/version/filter changes, a fresh workspace version remains `0`, and the embedded runner/prompt path trusts cached `resolvedSkills`/prompt instead of re-resolving against the current workspace. Related sandbox skill sync work addresses sandbox host-path remapping, not ordinary iCloud-to-local workspace migration in `sessions.json`.

## Best Possible Solution

Keep the issue open and implement workspace-aware skill snapshot invalidation. Add normalized workspace/source provenance to `SkillSnapshot` and `SessionSkillSnapshot`, compare it with the current resolved `workspaceDir` before reusing a cached snapshot, invalidate snapshots on `agents.defaults.workspace` and per-agent workspace changes, and add a regression test with an old absolute `SKILL.md` path in `sessions.json` plus current config pointing at the new workspace.
