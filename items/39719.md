## Summary

Keep #39719 open. Current main still reuses persisted skill snapshots by version and skill filter only, while the snapshot schema stores absolute skill paths without the workspace/source-root provenance needed to detect an iCloud-to-local workspace move. The prior ClawSweeper keep-open comment remains valid on e672b61417af5c45b0431df6d9109a1f4b618ef5.

## Best Possible Solution

Keep the issue open and implement workspace-aware skill snapshot invalidation. Add normalized workspace/source-root provenance to SkillSnapshot and SessionSkillSnapshot, compare it with the current resolved workspace before reuse, invalidate snapshots when agents.defaults.workspace or per-agent workspace changes, update agent command and cron/embedded/CLI consumers consistently, and add a regression test with a stale sessions.json absolute SKILL.md path from the old iCloud workspace.
