## Summary

Keep open. Current main still contains the Telegram no-visible-response branch that synthesizes a `NO_REPLY` fallback and sends it through normal Telegram delivery, and the outbound silent-reply defaults still rewrite direct-chat `NO_REPLY` into visible filler text. The current Telegram regression test also still asserts the buggy DM rewrite behavior, while the linked fix PR #70630 remains open rather than merged.

## Best Possible Solution

Land or otherwise implement the narrow Telegram adapter fix from #70630: for successful no-visible-response Telegram direct turns, do not synthesize a transport-visible `NO_REPLY`; treat the turn as a handled silent no-op for status reaction cleanup, while preserving the existing real-error and non-silent delivery-failure fallback paths. Flip the current Telegram dispatch regression test to assert no chat message is delivered for this DM case, and add/keep coverage for group behavior and visible block delivery.
