## Summary

Current main still contains the Telegram DM no-visible-response path that synthesizes a `NO_REPLY` fallback and routes it through outbound delivery, where direct-message silent-reply policy rewrites it into visible filler text. The issue also has open PR #70630 using closing syntax and the same author, so it should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review or land PR #70630, or apply an equivalent narrow Telegram adapter fix. The fix should treat successful no-visible-response Telegram direct turns as handled silent no-ops, avoid synthesizing transport-visible `NO_REPLY`, preserve the real dispatch-error and non-silent-delivery-failure fallback path, and update the Telegram dispatch regression test to assert no DM delivery for this successful silent case.
