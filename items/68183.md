## Summary

Keep #68183 open. Current main has an optional modern Docker CLI install path, but official Docker release builds still do not enable `OPENCLAW_INSTALL_DOCKER_CLI`, and the setup script only injects that build arg for local `openclaw:local` builds. For a pulled `ghcr.io/openclaw/openclaw:latest` image, sandbox setup verifies `docker --version` inside the running image and disables sandbox when the CLI is missing. The Docker sandbox runtime still shells out to `docker`, so the reporter's Windows Docker Desktop + official image sandbox path remains unresolved. Related ClawSweeper items cover adjacent Docker sandbox mount/parameter issues, not this CLI packaging/API mismatch.

## Best Possible Solution

Keep the issue open and fix the official Docker gateway sandbox story deliberately: either build/publish a sandbox-capable image variant or enable a current Docker CLI in the official image when sandbox support is advertised, update `scripts/docker/setup.sh` and docs to select that path for `OPENCLAW_SANDBOX=1`, and add release-workflow/test coverage that proves the shipped image contains a compatible CLI instead of silently disabling sandbox.
