## Summary

Keep #70805 open. Current main still has the watch-mode/runtime-postbuild gap the PR is trying to fix: `gateway:watch` launches through `watch-node`, sets `OPENCLAW_WATCH_MODE=1`, and `run-node` skips clean runtime-postbuild sync in watch mode before checking for missing staged alias wrappers, plugin metadata, static assets, skill outputs, or bundled runtime dependency trees. The PR discussion includes a concrete Telegram failure log and follow-up review fixes, but those changes are not present on main.

## Best Possible Solution

Keep the PR open for maintainer review or extraction. The desired fix belongs in the run-node/runtime-postbuild staging seam: validate required staged outputs before watch-mode launch, preserve built package metadata when watch-mode package entries point at unbuilt outputs, recover missing frozen metadata, and keep regression coverage for alias wrappers, overlays, skills, static assets, runtime deps, and lock serialization.
