## Summary

Keep #70805 open. Current main still skips runtime-postbuild entirely for clean watch-mode launches, and its runtime-postbuild freshness check can return clean from stamp/head/git-status state without proving the required staged outputs still exist. The PR is a focused bug-fix attempt for a real remaining gap; the related items in the provided context are title-search overlaps, not superseding canonical work.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land or rework a scoped fix that removes the clean-watch-mode blind spot, validates required runtime-postbuild outputs before launching, preserves/rebuilds bundled plugin package metadata correctly, and keeps targeted regression coverage for alias wrappers, dist-runtime overlays, static assets, skills, and staged runtime dependency drift.
