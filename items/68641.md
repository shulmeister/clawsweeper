## Summary

Current main already implements the linked Control UI/webchat image-history fix. The shipped web UI renderer reconstructs images from structured image blocks and transcript MediaPath/MediaPaths, gates local preview paths through the existing media route, and gateway tests cover persisted MediaPath/MediaPaths for sent images. PR #68641’s Swift OpenClawKit approach is therefore no longer needed for #68605.

## Best Possible Solution

Close PR #68641 as implemented for #68605. Keep the shipped web Control UI/gateway implementation and tests from main; open a separate scoped native OpenClawKit issue only if maintainers want image-history rendering in that shared Swift chat UI.
