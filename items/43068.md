## Summary

Keep open. Current main has script-file preflight for shell-bleed/model-output mistakes, but it does not implement the PR’s sandbox-only Python/Node import-template validation. The PR is a plausible command-execution hardening change, but its own review context identifies material bypasses that need author/maintainer follow-up before it can land.

## Best Possible Solution

Keep the PR open for maintainer/security review. The best path is to rebase it onto the current `validateScriptFileForShellBleed` implementation, preserve the newer shell-bleed/fail-closed behavior on main, implement sandbox-only import validation with parser-grade or multiline-aware handling for Node and Python edge cases, and add focused regression tests for whitespace import calls, multiline imports/re-exports, Python bare-relative denied names, sandbox-only enforcement, and host-exec preservation.
