## Summary

Keep #14874 open. Current main still has no optional SIE verification, signed-envelope lookup, warn/strict config, or loader-time rejection path for skills. The related implementation PR #14893 was closed unmerged, and the remaining request is a core skill-loader security/policy capability or a missing plugin API hook, not a ClawHub-only item.

## Best Possible Solution

Keep #14874 open. The best path is either a core skill-loader verification policy with backward-compatible disabled/default behavior and deterministic warn/strict outcomes, or a first-class plugin/API hook that can enforce signed-skill decisions before skills are exposed to prompts, commands, snapshots, sandbox sync, or runtime use. Existing ClawHub/plugin install hooks are not enough for the requested loader-time trust boundary.
