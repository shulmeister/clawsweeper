## Summary

Keep #14874 open. Current main still does not implement optional SIE verification at the skill-loader boundary: skills are loaded from `SKILL.md`, merged, filtered, exposed to prompts/snapshots, and synced without signed-envelope lookup, verifier calls, warn/strict config, or strict rejection behavior. The related PR #14893 was closed unmerged, and the existing ClawHub/install-hook integrity surfaces are adjacent but do not satisfy this loader-time signed-instruction policy request.

## Best Possible Solution

Keep #14874 open. The best path is a disabled-by-default loader-time verification policy with deterministic warn/strict outcomes, or a new documented loader verification hook/API that can reject or warn before skills are visible to prompts, slash-command discovery, snapshots, sandbox sync, or runtime use. Existing ClawHub install flows, package archive checks, and `before_install` hooks should be treated as adjacent hardening, not as the implementation of this request.
