## Summary

Keep #14874 open. Current main still does not implement optional SIE warn/strict verification at the skill loader boundary. The existing protections are adjacent: path containment, size caps, bundled allowlists, install-time dangerous-code scanning, ClawHub metadata/lockfile/fingerprint helpers, and plugin before_install hooks. None of them warn or reject signed/unsigned/tampered skills before they become prompt/runtime-visible. The directly related PR #14893 was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep #14874 open. The best path is a disabled-by-default core loader verification policy, or a documented loader-time verification hook/API, that can produce deterministic warn/strict decisions for unsigned, valid-signed, and invalid-signed skills before they are exposed to prompts, slash-command discovery, snapshots, sandbox sync, or runtime use. Existing ClawHub metadata, content hashes, install scans, and before_install hooks should remain adjacent protections, not treated as this feature.
