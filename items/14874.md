## Summary

Keep #14874 open. Current main still lacks optional SIE verification at the skill-loader boundary: no signed-envelope lookup, no verifier call, no warn/strict config, and no loader-time rejection path before skills are exposed to prompts, snapshots, commands, or sandbox sync. The related implementation PR #14893 was closed unmerged, and the existing ClawHub/install-hook/archive-integrity surfaces do not satisfy this loader-time signed-instruction policy request.

## Best Possible Solution

Keep #14874 open. The best path is either a core skill-loader verification policy with disabled-by-default backward compatibility and deterministic warn/strict outcomes, or a first-class plugin/API hook that can enforce signed-skill decisions before skills are visible to prompts, slash-command discovery, snapshots, sandbox sync, and runtime use. Existing ClawHub install flows, package archive verification, and `before_install` hooks are adjacent hardening but do not implement the requested loader-time trust boundary.
