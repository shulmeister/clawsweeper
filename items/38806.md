## Summary

Keep #38806 open. Current main still gates Feishu thread-aware reply delivery on group chats, so P2P thread messages still lose the reply target before the send layer and can fall back to a standalone message. The provided related PRs (#38808 and #38996) are useful targeted fixes, but they are not shipped on current main.

## Best Possible Solution

Keep #38806 open and land a targeted Feishu fix that detects direct-thread messages with `isDirect && Boolean(ctx.threadId)`, preserves quote-only P2P behavior, passes a real reply target with `replyInThread`/`threadReply` enabled for P2P thread replies, and adds regression coverage for both P2P thread messages and P2P plain quote replies. PR #38808 or #38996 appear to be the relevant implementation candidates to review rather than closing the issue.
