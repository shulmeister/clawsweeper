## Summary

Keep open. Current main still finalizes direct `codex-cli` runs from stdout and does not request `--output-last-message`; the PR targets a real, still-open enhancement (#65074). The PR is not merge-ready as-is because the provided review discussion identifies unresolved correctness and cleanup concerns, but those are change requests rather than a reason to close the item.

## Best Possible Solution

Keep the PR open for maintainer/contributor follow-up. The best path is to rebase onto current main, inject `--output-last-message` before positional prompt arguments for fresh and resume Codex runs, read the artifact inside the existing cleanup-protected flow, mask the artifact path in verbose logs, scope any Codex JSONL fallback narrowly to `codex-cli`, and keep tests for artifact precedence, empty-artifact stdout fallback, cleanup, and argv ordering. Close #65074 only after that behavior lands on main.
