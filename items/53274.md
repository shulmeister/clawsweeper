## Summary

Keep open. Current `main` still requires a browser secure context for Control UI device identity, so the issue title remains unresolved on `8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf`. The issue’s secondary scope-cleanup points appear partially addressed, but the HTTP reverse-proxy/device-identity problem is still present and documented as current behavior.
