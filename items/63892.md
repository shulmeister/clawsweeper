## Best Possible Solution

Review and land or explicitly supersede #64384, preferably with a focused change in `src/auto-reply/reply/agent-runner-memory.ts` that only skips transcript fallback when fresh persisted tokens are below the preflight threshold, plus a regression test in `src/auto-reply/reply/agent-runner-memory.test.ts` proving `totalTokensFresh: true` with an above-threshold total triggers proactive compaction. ## Evidence

- **Current main still short-circuits fresh token totals before threshold evaluation:** `runPreflightCompactionIfNeeded` computes `shouldUseTranscriptFallback` from `totalTokensFresh` and persisted totals, then returns immediately when the total is fresh. That return happens before the threshold calculation and before `shouldRunPreflightCompaction` is called, so a fresh cached total above the threshold cannot trigger the proactive path on current main.
