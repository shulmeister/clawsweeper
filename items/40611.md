## Summary

Keep open. Current main still uses the 1s heartbeat wake retry for `requests-in-flight` skips, the tests assert that 1s behavior, and there is no `heartbeat.retryDelayMs` config/schema/docs support. The related PR #40629 proposed the exact 60s/configurable fix for #40611 but is closed unmerged in the provided GitHub context, while #13744 and #14051 are broader related items rather than a canonical replacement for this regression.

## Best Possible Solution

Implement this as a core heartbeat scheduling fix, not as stale cleanup: add a separate requests-in-flight retry delay in `src/infra/heartbeat-wake.ts` with a conservative default such as 60s, expose it through `agents.defaults.heartbeat` and per-agent heartbeat config if maintainers want the requested knob, wire it through runner config updates, and add focused wake/runner tests plus docs/config schema updates. Keep #13744 for broader session lock scope/configuration and #14051 for activity-based heartbeat behavior.
