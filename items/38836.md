## Summary

Current main has partial restart resilience, including OpenClaw-managed restart intent/draining, restart lifecycle fallback text, restart sentinels, and main-session recovery. It still does not implement the requested native Feishu-visible notification/fan-out for sessions interrupted by a direct supervisor/systemctl restart: plain SIGTERM remains a stop unless an OpenClaw restart intent is present, aborted main-session recovery resumes with `deliver: false`, and the restart sentinel is one request/session scoped rather than a scan of all interrupted Feishu sessions.

## Best Possible Solution

Keep the issue open until OpenClaw either implements a native cross-channel restart-interruption path or explicitly narrows the supported behavior. A robust fix should persist the affected external delivery contexts before shutdown or during restart abort, then replay a Feishu-visible restart/interruption notice or continuation after startup. It should cover direct supervisor/systemctl restarts or document that only `openclaw gateway restart` is supported, and include regression coverage for Feishu/WebSocket or a generic channel-delivery contract.
