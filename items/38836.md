## Summary

Keep open. Current main still does not implement the requested Feishu-visible notification/fan-out for in-flight sessions interrupted by a direct supervisor/systemctl restart. Managed OpenClaw restarts have better drain/restart-intent behavior, and restart sentinel handling can notify one scoped session, but plain SIGTERM remains a stop path without restart drain, main-session restart recovery resumes with `deliver: false`, and there is no post-boot scan over all interrupted Feishu delivery contexts.

## Best Possible Solution

Keep the issue open until OpenClaw either implements a generic restart-interruption path for external channel turns or explicitly documents that direct supervisor/systemctl restarts are outside the supported no-loss boundary. The best fix should persist affected delivery contexts before shutdown or during abort/stale-lock marking, then on startup send a Feishu-visible interruption notice or resume-and-deliver through the original route, with regression coverage for Feishu or a generic channel delivery contract.
