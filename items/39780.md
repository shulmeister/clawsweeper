## Summary

Keep #39780 open. Current main still has the reported Control UI gap: the Overview insecure-HTTP recommendation tells users to set `gateway.controlUi.allowInsecureAuth: true` manually and links docs, while the existing base-hash guarded config diff/apply machinery is scoped to Config/Quick Settings and is not wired into Overview recommendation callouts.

## Best Possible Solution

Keep #39780 open until Control UI recommendation callouts can stage narrow base-hash guarded config changes, show before/after values, offer accept/dismiss controls, apply through the existing config write path, and confirm the saved value. The work likely belongs in a shared recommendation/apply component reused by the insecure HTTP Overview path and future config-hardening suggestions, with focused UI/controller tests and public Control UI docs updated.
