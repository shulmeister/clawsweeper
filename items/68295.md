## Summary

Current `main` already closes the gap described in #68295: session-backed routed replies now default to transcript mirroring, outbound sends populate mirror context from the resolved session route, and regression tests cover direct delivery, plugin-handled sends, and `routeReply` mirroring. The fix appears to be on `main` after `v2026.4.23`, so `fixedRelease` is not yet known from a tagged release.

## Evidence

- **Routed replies now mirror by default for session-backed chats:** `routeReply` passes a `mirror` object whenever a `sessionKey` is present, unless explicitly disabled. That removes the need for each provider/model path to opt into transcript mirroring manually.
  - file: [src/auto-reply/reply/route-reply.ts:238](https://github.com/openclaw/openclaw/blob/b13545355d42701a1b2a5838b18101777c1bd808/src/auto-reply/reply/route-reply.ts#L238)

  - sha: [b13545355d42](https://github.com/openclaw/openclaw/commit/b13545355d42701a1b2a5838b18101777c1bd808)
- **Telegram/final reply path supplies the active session key to routeReply:** The auto-reply dispatch path calls `routeReply` with `ctx.SessionKey`, so Telegram-originated final replies flow through the default mirroring path on current `main`.
