## Summary

Keep #69426 open. Current main still gates the 1M context override to `anthropic` and `claude-cli`, so `agents.defaults.models["github-copilot/<claude-opus-or-sonnet>"].params.context1m = true` remains ineffective for the proxied-provider case described by #69353. A later main-branch change shipped Claude CLI support, but not the provider-independent model-prefix behavior this PR is trying to add.

## Best Possible Solution

Keep the PR open and ask for a rebase over current main. The best maintainer path is to preserve existing Anthropic/Claude CLI behavior, make explicit `context1m: true` use the Claude Opus/Sonnet model-family gate independent of provider for proxied providers, and add focused regression tests for GitHub Copilot plus any maintainer-approved Bedrock/LiteLLM normalization case.
