## Summary

Current main still has exact-only `skills info` lookup, so the PR's case-insensitive and separator-normalized resolver is not implemented or obsolete. The related closed PR #38701 was not merged, and #38713's head still exists as the active follow-up for the reported #38546 CLI mismatch.

## Best Possible Solution

Keep this PR open. The best path is to refresh it against current main and land a focused resolver in `src/cli/skills-cli.format.ts` that preserves exact lookup first, then allows only unique case-insensitive and separator-normalized matches with ambiguity guards, trims/sanitizes not-found output, and adds regression tests in `src/cli/skills-cli.test.ts`.
