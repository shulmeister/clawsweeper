## Summary

Keep open. Current `main` does not contain this fix: `openclaw skills info <name>` still passes the raw argument through unchanged and `formatSkillInfo` still does exact `name`/`skillKey` equality only, so the case-insensitive/separator-normalized lookup proposed in #38713 is not implemented on `main`. The existing tests on `main` also cover only exact-match `skills info` behavior, not the new lookup variants.
