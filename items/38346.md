## Summary

Keep open. Current main still does not implement the requested Telegram mode: observe or ingest all group messages so non-allowlisted users can be identified later, while only allowlisted users can trigger replies. The prior ClawSweeper comment remains accurate on current `main`: `groupPolicy: "open"` allows all group senders past policy, while `groupPolicy: "allowlist"` drops unauthorized senders before mention-skip history or silent ingest can run.

## Best Possible Solution

Keep this issue open for maintainer product/API judgment or implementation in the Telegram plugin. A good fix would add an explicit observe-only or trigger-authorization policy that lets Telegram receive and optionally ingest non-triggering group messages after transport visibility is enabled, while ensuring only configured senders can start agent turns. The implementation should update Telegram/group docs and add tests covering unauthorized sender ingest without reply plus allowlisted sender reply.
