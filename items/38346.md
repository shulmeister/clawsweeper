## Summary

Keep #38346 open. Current main still does not provide a Telegram mode that observes or ingests all visible group messages while separately limiting agent-triggering/replies to allowlisted senders. `groupPolicy: "open"` allows the sender through the group access gate, while `groupPolicy: "allowlist"` drops unauthorized senders before message processing and before the existing `ingest` hook can run.

## Best Possible Solution

Keep #38346 open for bundled Telegram plugin work. Add an explicit observe/ingest path for configured groups/topics that can record or hook all visible messages, then apply a separate sender trigger allowlist before agent turns, native/control commands, tool use, and replies. Update Telegram config schema/types/docs and add regression coverage for unauthorized unmentioned ingest without reply, unauthorized mentioned no reply, and allowlisted mentioned reply.
