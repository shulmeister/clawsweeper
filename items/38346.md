## Summary

Keep #38346 open. Current main has Telegram mention-skip `ingest: true`, but it has not split observe/ingest authorization from reply-trigger authorization: `groupPolicy: "allowlist"` still blocks unauthorized senders before ingestion, while `groupPolicy: "open"` still lets non-allowlisted senders trigger normal replies when mention gating passes.

## Best Possible Solution

Keep #38346 open for bundled Telegram plugin work. The best path is to separate observation/ingest authorization from reply-trigger authorization: allow configured groups to receive and optionally ingest non-triggering messages, then apply a sender trigger allowlist before starting agent turns or replies. Update Telegram group docs, config schema/types, and tests for unauthorized unmentioned ingest without reply, unauthorized mentioned no reply, and allowlisted mentioned reply.

- Observe-only ingestion has retention and privacy implications, so the feature should require explicit configuration and tests that unauthorized observed messages do not trigger replies.
