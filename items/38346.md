## Summary

Keep #38346 open. Current main still lacks a separate Telegram observe/ingest authorization layer and trigger-only sender allowlist: `groupPolicy: "allowlist"` blocks unauthorized senders before ingest, while `groupPolicy: "open"` allows random group senders to trigger replies when mention gating passes.

## Best Possible Solution

Keep #38346 open for bundled Telegram plugin work. The likely fix belongs in Telegram message preflight: allow configured groups to observe and optionally ingest non-triggering messages, then apply a separate sender trigger allowlist before starting agent turns or replies. Update Telegram config schema/types/docs and add tests for unauthorized unmentioned ingest without reply, unauthorized mentioned no reply, and allowlisted mentioned reply.

## Risks / Open Questions

- Telegram Bot API privacy mode can prevent receiving all group messages at the transport level unless privacy is disabled or the bot is an admin; any implementation should document this prerequisite.
- Observe-only ingestion has retention and privacy implications, so it should be explicit opt-in and tested to ensure unauthorized observed messages do not trigger replies.
