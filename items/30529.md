## Summary

Keep #30529 open. Current main already has automatic Discord typing for inbound replies and heartbeat runs, but the requested message-tool typing action / explicit start-stop control is still not exposed in the shared message action contract, Discord action discovery, Discord action runtime, or CLI docs. The issue discussion specifically called out explicit `action="typing"` control as useful, so closing this would drop meaningful requested scope.

## Best Possible Solution

Keep tracking #30529 until OpenClaw either implements an explicit message-tool typing action or a maintainer decides the product direction. A concrete implementation would add a shared `typing`/`typing-indicator` action name and target-mode/schema support, advertise it from Discord `describeMessageTool`, route it through `handleDiscordMessageAction` to `sendTypingDiscord` with the existing Discord target/account resolution, and update CLI/public docs plus focused tests. If maintainers want this cross-provider, design the shared action contract first so Telegram/Slack/Discord can opt in without channel-specific ad hoc parameters.
