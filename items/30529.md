## Summary

Keep #30529 open. Current main has Discord typing infrastructure for automatic inbound replies and heartbeat delivery, but it still does not expose the requested agent-callable Discord message-tool action such as `message(action="typing", channel="discord", ...)`. The issue discussion and related bug links point to broader typing lifecycle work, not a canonical duplicate or shipped message-tool typing contract.

## Best Possible Solution

Keep #30529 open until maintainers either add a shared `typing` or `typing-indicator` message-tool action, or explicitly close the product question by deciding automatic typing is sufficient. A concrete implementation should add the shared action name and target/schema fields, advertise it from Discord `describeMessageTool`, route it through `handleDiscordMessageAction` to `sendTypingDiscord` with account and target resolution, and update the Discord and typing-indicator docs. If cross-provider parity is desired, design the action so Telegram, Slack, Discord, and other typing-capable plugins can opt in cleanly.
