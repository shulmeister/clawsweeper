## Summary

Keep #30529 open. Current main has Discord native typing helpers and automatic typing for inbound replies/heartbeat delivery, but it still does not expose an agent-callable `typing` or `typing-indicator` message-tool action. The issue discussion explicitly preserved that manual/shared action scope, and the current shared action list, Discord discovery, Discord action handler, CLI docs, and Discord skill docs still omit it.

## Best Possible Solution

Keep tracking #30529 until a maintainer either implements the explicit message-tool typing action or makes a product decision that automatic typing is sufficient. A concrete implementation should add a shared `typing` or `typing-indicator` action contract, advertise it from Discord `describeMessageTool`, route it through `handleDiscordMessageAction` to `sendTypingDiscord` with existing target/account resolution, and update CLI/docs/skill guidance plus focused tests. If maintainers want cross-provider parity, design the shared contract first so Telegram, Slack, Discord, and similar plugins can opt in cleanly.
