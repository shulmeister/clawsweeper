## Summary

Keep this PR open for maintainer review rather than auto-closing. Current main has not implemented the proposed one-line change, and the PR is not obsolete, but the provided discussion and current code show that it should not be treated as a proven safe fix yet: production startup paths pass an explicit config, the linked bloat report says the observed bloat is actually plugin-provider discovery with empty providers, and the proposed fallback change may persist resolved provider request SecretRefs unless it is broadened and tested.

## Best Possible Solution

Keep this PR open for maintainer review, but do not land it as-is. The best path is to either revise it into a tested fallback-path fix that preserves source markers for all SecretInput-backed provider fields, including `request.*`, or close it manually in favor of a separate fix for the actual `models.providers: {}` plugin-discovery bloat described in #65020. A useful regression test should fail on current main, prove the lean/source-vs-runtime distinction, and cover provider request SecretRefs so `models.json` never persists plaintext resolved transport secrets.
