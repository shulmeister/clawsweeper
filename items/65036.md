## Summary

Keep open. Current `main` still has the original behavior, so the PR is not implemented on `main`, and the proposed one-line swap is not yet high-confidence safe: `loadConfig()` returns the resolved runtime snapshot, while `models-config` only source-normalizes provider `apiKey` and top-level `headers`. Provider `request.auth`/`request.proxy`/`request.tls` secret surfaces are valid and sensitive, so switching the omitted-config path from `runtimeSource` to `loaded` can persist resolved request secrets into `models.json`. The PR discussion also already identified that the added regression test does not distinguish the lean-loaded-vs-bloated-runtimeSource case in the current harness.

## Risks / Open Questions

- Merging the PR as written can persist resolved provider request secrets into `models.json` when `ensureOpenClawModelsJson()` is called without an explicit config.
- Closing this PR would be premature because current `main` still has the reported omitted-config/runtimeSource behavior and the underlying bloat report appears plausible.
- A safe fix likely needs a narrower change that removes provider-list bloat without replacing source-backed secret surfaces with resolved runtime request values, plus a regression test that can actually distinguish those two inputs.
