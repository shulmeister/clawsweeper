## Summary

Current `main` already routes heartbeat/session state through the new home-directory resolution helpers, which prefer `OPENCLAW_HOME` and `$HOME` over `os.homedir()`. That removes the path-resolution behavior that could send a native macOS install toward `/home/node` during heartbeat/session writes.

## Evidence

- **Home directory resolution now prefers env-driven paths:** `resolveRequiredHomeDir()` and `resolveEffectiveHomeDir()` prefer `OPENCLAW_HOME`, then `HOME`, then `USERPROFILE`, and only then fall back to `os.homedir()`. This directly addresses wrong-home resolution for internal state paths.
  - file: [src/infra/home-dir.ts:32](https://github.com/openclaw/openclaw/blob/6d271762ab50fb4376b8859bcf55eefb2d3814ea/src/infra/home-dir.ts#L32)
