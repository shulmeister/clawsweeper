## Summary

Keep open. Current main does not implement the requested `models.providers.<id>.timeoutMs` provider-scoped timeout. The old hardcoded-30s framing appears obsolete because main now has global agent/run and global LLM idle timeout controls, but the issue discussion explicitly reframed the remaining request as a provider-specific override so slow providers can be tuned without raising the global timeout.
