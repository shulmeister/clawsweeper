## Summary

Current main still does not implement #69748. Discord guild access remains `requireMention?: boolean` plus `users`/`roles` sender allowlists, not the proposed `deny`/`bypass`/`mention` tiers, permission aliases, additive channel inheritance, or sender-specific no-mention bypass behavior.

## Best Possible Solution

Keep #69748 open for bundled Discord plugin work. A good implementation would add typed `deny`, `bypass`, and `mention` access lists at guild and channel scope, define merge versus `inherit: false` behavior, support role IDs and a small audited set of Discord permission aliases, preserve legacy `requireMention`/`users` through compatibility migration or doctor warnings, and add schema, docs, and preflight tests for precedence and per-channel overrides.
