## Summary

Keep #8972 open. Current main still resolves Discord guild channel allowlists through exact channel id/slug/name matching, thread parent fallback, and the literal `*` wildcard only. A configured key like `ws-*` remains a literal key and will not match `ws-task-1`; the category alternative is also not implemented and is only separately tracked by open #60582.

## Best Possible Solution

Keep the issue open until Discord guild channel allowlists support a documented pattern form, such as glob or prefix entries, with matcher precedence, config schema/types/help, docs, status/audit behavior, and regression tests proving `ws-task-1` matches `ws-*` without a per-channel config edit. Treat category inheritance as the separate #60582 path unless maintainers intentionally combine the designs.
