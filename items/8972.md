## Summary

Keep open. Current main supports exact Discord channel keys and a literal catch-all `*` fallback, but it does not implement glob/prefix patterns such as `ws-*`, and the alternative category-based allowlist is not present.

## Evidence

- **Discord docs describe exact channel allowlist behavior:** The Discord docs say that when a guild has a `channels` map, non-listed channels are denied, with examples using exact keys like `general` and `help`; they do not document glob patterns.
  - file: [docs/channels/discord.md:422](https://github.com/openclaw/openclaw/blob/1dfa52d0711dfc86e1c04538fe46e3d43961766f/docs/channels/discord.md#L422)
  - command: `nl -ba docs/channels/discord.md | sed -n '408,446p'`
  - sha: [1dfa52d0711d](https://github.com/openclaw/openclaw/commit/1dfa52d0711dfc86e1c04538fe46e3d43961766f)
- **Shared channel matcher only checks exact keys and literal wildcard:** `resolveChannelEntryMatch` checks configured entries with `hasOwnProperty(entries, key)` for each candidate and separately recognizes only the configured literal `wildcardKey`; there is no glob or prefix matching path.
  - file: [src/channels/channel-config.ts:60](https://github.com/openclaw/openclaw/blob/1dfa52d0711dfc86e1c04538fe46e3d43961766f/src/channels/channel-config.ts#L60)
  - command: `nl -ba src/channels/channel-config.ts | sed -n '1,260p'`
  - sha: [1dfa52d0711d](https://github.com/openclaw/openclaw/commit/1dfa52d0711dfc86e1c04538fe46e3d43961766f)
- **Discord channel matching passes only `*` as wildcard:** Discord builds candidates from channel id/name/slug and calls `resolveChannelEntryMatchWithFallback` with `wildcardKey: "*"`, so a key like `ws-*` is treated as an exact key rather than a pattern.
  - file: [extensions/discord/src/monitor/allow-list.ts:374](https://github.com/openclaw/openclaw/blob/1dfa52d0711dfc86e1c04538fe46e3d43961766f/extensions/discord/src/monitor/allow-list.ts#L374)
  - command: `nl -ba extensions/discord/src/monitor/allow-list.ts | sed -n '260,430p'`
  - sha: [1dfa52d0711d](https://github.com/openclaw/openclaw/commit/1dfa52d0711dfc86e1c04538fe46e3d43961766f)
- **Tests cover catch-all wildcard, not glob patterns:** The current Discord monitor tests assert that `*` applies to unknown channels, but there is no corresponding test for `ws-*` matching `ws-task-1`.
  - file: [extensions/discord/src/monitor.test.ts:407](https://github.com/openclaw/openclaw/blob/1dfa52d0711dfc86e1c04538fe46e3d43961766f/extensions/discord/src/monitor.test.ts#L407)
  - command: `nl -ba extensions/discord/src/monitor.test.ts | sed -n '340,560p'`
  - sha: [1dfa52d0711d](https://github.com/openclaw/openclaw/commit/1dfa52d0711dfc86e1c04538fe46e3d43961766f)
- **No category allowlist schema:** `DiscordGuildSchema` exposes `channels` but no `categories` field and is strict, so the issue's category-based alternative is not accepted by the documented schema shape.
  - file: [src/config/zod-schema.providers-core.ts:476](https://github.com/openclaw/openclaw/blob/1dfa52d0711dfc86e1c04538fe46e3d43961766f/src/config/zod-schema.providers-core.ts#L476)
