## Summary

Keep #8972 open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still resolves Discord guild channel allowlists by exact id/slug/name, normalized exact match, thread-parent fallback, and the literal `*` wildcard only. There is no glob branch that would make `ws-*` match `ws-task-1`, and the category alternative remains separately tracked by open issue #60582. The existing maintainer/Codex review comment on #8972 already reached the same keep-open conclusion, and the relevant files have not changed since that review.

## Best Possible Solution

Keep #8972 open until Discord guild channel allowlists support a documented and tested pattern form such as `ws-*`. The likely implementation belongs in the shared channel config matcher or Discord resolver with explicit opt-in pattern handling, exact-match precedence, existing thread-parent behavior preserved, schema/types/docs/status/audit updates, and regression coverage proving dynamically created matching Discord channels are allowed without per-channel config edits. Treat category inheritance as #60582 unless maintainers intentionally combine the designs.
