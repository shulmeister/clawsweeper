## Summary

Keep #8972 open. Current main still treats Discord guild channel allowlist keys as exact channel id/name/slug entries, thread-parent fallbacks, or the literal `*` wildcard. A key like `ws-*` is not evaluated as a glob, and the category-based alternative is tracked separately in open issue #60582 rather than superseding the glob request.

## Best Possible Solution

Keep #8972 open. Implement a documented Discord guild channel pattern form such as `ws-*`, preserve exact-match and thread-parent precedence, keep literal `*` as the broad fallback, update Discord types/Zod schema/docs/status/audit behavior, and add regression tests proving dynamically created matching Discord channels are accepted without per-channel config edits. Track category inheritance through #60582 unless maintainers intentionally merge both designs.
