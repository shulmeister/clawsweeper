## Summary

Keep #8972 open. Current main still resolves Discord guild channel allowlists by exact channel id/slug/name, thread parent fallback, and the literal `*` wildcard only; a key such as `ws-*` is treated as a normal literal channel key, not a glob or prefix pattern. The category-based alternative is also not implemented and is only separately tracked by open issue #60582, which does not supersede the main wildcard request.

## Best Possible Solution

Keep the issue open until Discord guild channel allowlists support a documented pattern form such as glob or prefix entries, with matcher, config schema/types/help, docs, status/audit handling, and regression tests proving `ws-task-1` matches `ws-*` without a per-channel config edit. Treat category-level inheritance as the separate #60582 path unless maintainers intentionally combine both designs.
