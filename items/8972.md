## Summary

Keep #8972 open. Current main still supports exact Discord guild channel keys plus a literal `*` fallback and parent-channel fallback for threads, but not glob/prefix entries like `ws-*`. The category alternative is not implemented either and is only partially covered by separate open issue #60582, which does not supersede the main glob/wildcard request.

## Best Possible Solution

Keep the issue open until the Discord channel allowlist supports a documented pattern form such as glob or prefix entries, with updates to the Discord matcher, config/schema help, docs, and regression tests proving that a dynamic channel like `ws-task-1` matches `ws-*` without a per-channel config entry. Treat category-level inheritance as the separate #60582 path unless maintainers intentionally merge both designs.

## Risks / Open Questions

- Closing this now would drop an actionable Discord bundled-plugin feature request; current main still requires exact per-channel entries or a broad `*`.
- A future implementation should preserve precedence and avoid surprising access widening: exact channel match, thread parent fallback, then explicit wildcard/pattern fallback.
- The category-based alternative overlaps open #60582, so maintainers may want to split that path while keeping this issue focused on glob/prefix channel patterns.
