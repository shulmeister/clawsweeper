## Summary

Current `main` contains a shared reply-delivery fix for this bug: commentary-phase assistant text is suppressed until a final answer exists, and `text_end` channels no longer re-send the same reply again at `message_end`. Regression tests on `main` cover both the premature/hallucinated first message and the duplicate final-delivery paths. The fix appears to have landed after `v2026.4.23`, so it is implemented on `main` but not in the latest tagged release.
