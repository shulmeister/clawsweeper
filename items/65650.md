## Summary

Keep open. Current `main` does not appear to implement the requested SIGTERM-resilient inbound Slack message persistence/replay. The issue’s concrete file references are stale, but the underlying gap remains: Slack Socket Mode events are handled in-memory and startup recovery only covers outbound deliveries/restart sentinels, not inbound message requeueing.

## Evidence

- **Inbound Slack events dispatch directly from memory:** `createSlackMessageHandler()` debounces inbound Slack events and then calls `prepareSlackMessage()` and `dispatchPreparedSlackMessage()` directly. There is no persisted inbound queue or recovery handoff in this path.
