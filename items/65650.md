## Summary

Keep open. Current main has partial restart mitigations (active-work drain, restart-continuation queues, and main-session transcript-lock recovery), but it still does not have the durable inbound/SQLite message-store wiring requested here. The open related PR #49431 appears to be the right implementation candidate, but it is still unmerged and should be reviewed or superseded rather than using it as a close reason.

## Best Possible Solution

Keep this issue open and use it to validate or supersede #49431. The best fix is a durable pending-inbound store wired at the channel/gateway ingress boundary before Slack Socket Mode ACKed work can be lost, plus startup re-dispatch/recovery, shutdown/restart handling, and regression tests that simulate an inbound Slack turn interrupted after acceptance but before final reply delivery.
