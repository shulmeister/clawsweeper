## Summary

The issue is a useful local Linux browser-routing runbook, but it does not identify a missing OpenClaw core API or bundled-browser regression to fix in this item. Current main already keeps the core browser surface documented and ships a bundled `browser-automation` skill for OpenClaw's own browser tool, while VISION.md directs new optional skills and bundle-style capabilities to ClawHub. The linked browser-use draft PR #23252 is a useful related implementation pointer, but bundling that optional browser stack into core was closed unmerged; this routing playbook is better published as a ClawHub skill/plugin or external runbook, with actual core browser bugs left to #43654/#44431.
