## Summary

Keep open. Current `main` already fixes two parts of the issue narrative: startup catch-up is capped/staggered, and expired isolated cron run sessions are pruned. But the default gateway service still does not add any heap tuning, and isolated cron runs still persist a `skillsSnapshot` payload under both the base cron session key and the per-run `:run:` key. That leaves the reported memory-pressure/OOM problem plausible on current `main`, so there is not strong evidence to close this conservatively.

## Evidence
