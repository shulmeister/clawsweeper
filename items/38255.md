## Summary

Keep open. Current main still does not expose `channels.telegram.streamThrottleMs` or `channels.telegram.minInitialChars`, while PR #38255 is a plausible implementation path for open related issue #38066. The PR discussion also still has a real timer-bound review concern to address before merge.

## Best Possible Solution

Keep PR #38255 open as the active implementation path for #38066. The best fix is to add the Telegram config keys to schema/types/generated metadata/docs, thread them through `dispatchTelegramMessage` into `createTelegramDraftStream`, preserve safe draft/message fallback timing, and validate `streamThrottleMs` with both lower and timer-safe upper bounds plus focused tests.
