## Summary

Keep PR #38255 open. Current main at c7a0d9b1889ca09ab5bfd84cda39a8d50d3d5768 still lacks the requested Telegram config surface, still hardcodes the Telegram draft preview minimum initial character threshold, and still uses the fixed 1000ms draft-stream throttle unless a caller manually passes `throttleMs`. The related issue #38066 remains open and this PR is the active implementation path, although it still needs maintainer review for the timer-safe upper bound concern and likely a current-main rebase/regeneration pass.

## Best Possible Solution

Keep PR #38255 open as the active implementation path for #38066. The best path is to add the Telegram config keys to schema, exported types, generated config metadata, and docs; thread them through `dispatchTelegramMessage` into `createTelegramDraftStream`; preserve separate draft-vs-message fallback cadence semantics; add a timer-safe upper bound for `streamThrottleMs`; and keep focused Telegram/config/shared-loop tests around fallback, final flush, and validation behavior.
