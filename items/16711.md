## Summary

Keep #16711 open. Current main still does not implement the requested chat composer Expand button or large prompt modal, and the issue is a clear Control UI ergonomics request rather than ClawHub scope or an obsolete/duplicate item.

## Best Possible Solution

Keep the issue open for maintainer/product UI handling. The best path is to add a first-class long-prompt composer affordance in the Control UI chat composer: an Expand control near the existing composer actions, modal state that preserves the current draft, Cancel/Apply/Send actions, and modal-specific keyboard handling where Enter inserts a newline and Ctrl/Cmd+Enter sends. If maintainers prefer the alternative, a larger auto-growing composer should be chosen explicitly and tracked here or in a narrower canonical issue.
