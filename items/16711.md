## Summary

Current main still does not implement the requested chat composer Expand button or large prompt modal. The Control UI chat composer remains a single inline textarea with capped auto-grow behavior, normal Enter-to-send handling, and run controls limited to New session, Export, Send/Queue, and Stop. The issue is clear, user-facing, not protected by close labels, and belongs to core Control UI ergonomics rather than ClawHub/plugin scope.

## Best Possible Solution

Keep #16711 open until the Control UI implements a first-class long-prompt composer affordance or a maintainer explicitly chooses a different UX. The likely implementation belongs in `ui/src/ui/views/chat.ts` plus a small focused component/style/test path: add an Expand control in the composer toolbar, preserve the existing draft, provide modal actions for Cancel/Apply/Send, and adjust modal key handling so Enter inserts a newline while Ctrl/Cmd+Enter sends.
