## Summary

Current main already provides the requested outbound interception surface: internal `message:sent` hooks are documented and emitted, and plugin `message_sending` hooks can cancel outbound delivery before it is sent. A bundled plugin already uses that path to log and suppress sends, so the issue’s requested hook-based solution is implemented.

## Evidence

  - file: [docs/automation/hooks.md:47](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/docs/automation/hooks.md#L47)

## Close Comment

Closing from a Codex review because current `main` already has the hook-based outbound suppression surface requested here. Internal `message:sent` hooks are documented in `docs/automation/hooks.md:47-50,117`, and outbound plugin hooks can cancel delivery before send in `src/infra/outbound/deliver.ts:834-863` with the documented `message_sending` `{ cancel: true }` semantics in `docs/plugins/sdk-overview.md:212-215`. Operator-side logging is available through `api.logger` (`docs/plugins/sdk-overview.md:231`), and a bundled plugin already uses that exact pattern to log and suppress outbound sends in `extensions/thread-ownership/index.ts:144-197`. This is also covered by tests in `src/infra/outbound/deliver.test.ts:1078-1128` and `extensions/telegram/src/bot/delivery.test.ts:249-402`. The latest shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`) already contains these paths.
