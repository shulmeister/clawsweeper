## Summary

Keep open. Current main still contains the reported no-real-conversation guard in the direct compaction path, and the hook-side empty-preparation behavior now writes a fallback boundary rather than performing a real summarization. That may avoid a cancel loop, but it does not resolve the reported user problem where real persisted history is not available to compaction and the result is a stub/no-op compaction. Related #71130 is fixed separately for empty embedded provider submissions, but that changelog entry does not cover this compaction guard failure.
