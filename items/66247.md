## Summary

Keep open. Current main still sequentializes Telegram updates directly by chat/topic key, so ordinary follow-up messages in a busy topic cannot reach core immediate queue policy until the topic lane drains. The PR is a focused Telegram fix, but its review discussion identifies remaining implementation issues that should be addressed before merge rather than auto-closing it.

## Best Possible Solution

Keep the PR open or replace it with an updated patch that implements the Telegram-owned sequential-key wrapper against current main, uses the bot instance config, matches canonical queue-mode normalization, treats reply and embedded active runs as busy, covers media-only updates, and preserves existing `/stop`, status, `/btw`, and approval lanes with focused Telegram tests.
