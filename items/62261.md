## Summary

Keep open. Current main still has updater-specific evidence matching the report: failed dev preflight lint attempts are retained in the shared step list, and the final git update result treats any earlier nonzero step as a blocking failure even after a later candidate passes. The related #63627 discussion is useful for the OOM angle, but it does not fully supersede this issue's `openclaw update --channel dev` result/accounting bug.
