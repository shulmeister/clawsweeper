## Summary

Keep open. Current main still does not implement the proposed `google-genai` provider, `google-genai-sdk` auth mode, or `models.providers.google-genai.vertexai` config contract. The PR remains a plausible implementation candidate for unified Google AI Studio / Vertex AI auth, but it changes shared provider/auth surfaces and the review context still has substantive correctness gaps to resolve before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to revise it so unified Google GenAI auth fits the current bundled Google plugin/provider architecture, covers all keyless-auth call sites, forwards tools/system/options/finish metadata correctly, and adds focused tests/docs; alternatively, supersede it with a narrower PR that fixes the remaining `google-vertex` ADC/proxy behavior in the existing provider path.
