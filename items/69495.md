## Summary

Keep PR #69495 open. Current main still has the original heartbeat fallback gap: `agents.*.heartbeat.model` is string-only in schema/types, heartbeat runtime only passes a primary `heartbeatModelOverride`, and the reply pipeline has no per-run fallback override for heartbeat ticks. The PR is therefore not obsolete, but it should address the unresolved review concern that explicit `fallbacks: []` must be preserved if the intended semantics are to disable heartbeat fallback chains.

## Best Possible Solution

Keep the PR open and ask for a focused revision: change heartbeat schema/types to use `AgentModelConfig`, resolve primary and fallback values in `runHeartbeatOnce`, pass an explicit `heartbeatModelFallbacks` array through `GetReplyOptions` and `FollowupRun.run.modelFallbacksOverride` into `runWithModelFallback`, preserve empty arrays when they intentionally disable fallback chains, update pricing ref collection/docs/generated schema, and add regression tests for object-form, inherited defaults, and explicit empty fallbacks.
