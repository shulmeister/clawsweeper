## Summary

Keep #7433 open. Current main still does not implement the requested Baileys reliability hooks in the bundled WhatsApp plugin: no socket-level getMessage or cachedGroupMetadata, no group metadata event maintenance, and no lid-mapping.update persistence path. The prior maintainer-side Codex comment remains consistent with current main; the related #66917 is a broader closed WhatsApp stability report and does not supersede this narrower implementation request.

## Best Possible Solution

Keep the issue open and implement the reliability work inside the bundled @openclaw/whatsapp plugin: add bounded full-message retry storage and pass it through Baileys getMessage, expose a shared group metadata cache through cachedGroupMetadata, seed and maintain that cache from connect hydration plus groups.upsert/groups.update/group-participants.update events, and decide whether Baileys signalRepository.lidMapping is sufficient or whether lid-mapping.update should persist reverse mappings. Add focused WhatsApp plugin tests for socket options, retry lookup, group cache updates, participant updates, and LID mapping behavior.
