## Summary

Keep open. Current main filters several Control UI control-token cases, but it does not currently hide the timestamped `System:` / `System (untrusted): ... Exec completed` rows or sender-envelope leak shapes that #67036 targets. The related PRtags group does not provide a merged canonical fix: #68518 remains open and narrower, while #69366 is closed and unmerged.

## Best Possible Solution

Keep this PR or a canonical replacement open. Rebase against current main, put a shared browser-safe predicate on the current Control UI paths (`loadChatHistory`, `buildChatItems`/rendering, export, and live final-event handling), keep it narrowly anchored to timestamped internal exec/system-event and known sender-envelope/session-reset leak shapes, and add regression tests that also preserve legitimate `System:` chat text.
