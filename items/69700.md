## Summary

Keep open. Current main still treats every top-level `sources/*.md` file as a memory-wiki source page and requires wiki frontmatter, while the public plugin docs still describe `sources/` as imported raw material. The CLI docs partially clarify that imported source pages keep provenance in frontmatter, and the ingest/bridge code does wrap raw content inside generated source pages, but that does not fully resolve the reported contradiction for users placing raw markdown directly under `sources/`. Related #64696 is open but covers different bridge import/lint durability bugs, so it does not supersede this docs/API question.

## Best Possible Solution

Keep this open for a small maintainer product/docs decision. The conservative fix is either to document clearly that `sources/` contains wiki source pages that preserve raw content inside a wrapper with frontmatter, or to add an explicit raw-source location/mode that compile and lint intentionally ignore or handle differently. If the wrapper model is intended, update the memory-wiki docs and lint/CLI messaging so users know to use `openclaw wiki ingest` or bridge import rather than dropping raw markdown directly into `sources/`.
