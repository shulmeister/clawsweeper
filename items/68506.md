## Summary

Keep PR #68506 open. Current main still does not implement the PR's central `status --all` degraded-state recovery guidance: invalid config, gateway log failures, unreachable/query-failed health, bind failures, and OAuth re-auth cases are still mostly surfaced as raw diagnostics or handled only in adjacent doctor/docs paths. The PR remains useful, but it needs rebase/review cleanup before merge.

## Best Possible Solution

Keep the PR open. The best maintainer path is to ask the contributor to rebase onto current main, preserve the useful `status --all` degraded-state `Cause:` / `Fix:` guidance, reuse the existing OAuth refresh helper where it fits, and add tests that prevent duplicate unreachable-gateway output and false-positive `401`/`403` auth classifications.
