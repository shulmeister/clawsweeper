## Summary

Keep open. Current main still lacks the requested `audio/webm`, `video/webm`, and `video/x-matroska` mappings in the shared MIME-to-extension map, while the media save and outbound filename paths still depend on `extensionForMime(...)`. The PR remains a plausible, narrow bug fix rather than a conservative cleanup close.

## Best Possible Solution

Keep the PR open. The best path is to update or rebase the patch onto current main, add `audio/webm` and `video/webm` to `.webm` plus `video/x-matroska` to `.mkv` in `src/media/mime.ts`, and add focused regression coverage in `src/media/mime.test.ts` and ideally `src/media/store.test.ts` for MIME-only buffers saved without an original filename.
