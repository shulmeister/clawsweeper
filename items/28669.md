## Summary

Keep #28669 open. Current main at 82262387651ab270b565f71533ce4937c9c00dc6 still lacks the requested `audio/webm`, `video/webm`, and `video/x-matroska` MIME-to-extension mappings, while shared media-save and outbound filename paths still depend on `extensionForMime`. The PR discussion is also materially relevant: the author reports using the patch successfully, Greptile identified the patch as narrow and standard, and the prior ClawSweeper review found the same gap on main.

## Best Possible Solution

Keep the PR open. The best maintainer path is to rebase or recreate the narrow patch on current main, add `audio/webm -> .webm`, `video/webm -> .webm`, and `video/x-matroska -> .mkv` in `src/media/mime.ts`, and add focused regression coverage in `src/media/mime.test.ts`; a saved-media or outbound attachment filename test would directly protect the reported user-visible behavior.
