## Summary

Keep #69605 open. Current main still renders assistant message-level copy/expand actions absolutely inside `.chat-bubble`, has CSS intended to reserve space via `.chat-bubble.has-copy`, but never adds `has-copy` to the bubble classes. That leaves the reported overlap/mis-click path plausible and unfixed. The related open PR #69634 appears to target the right root cause, while closed PR #69689 described the same fix but was not merged.

## Best Possible Solution

Keep the issue open and land PR #69634 or an equivalent focused fix that adds `has-copy` to assistant bubbles when copy/expand actions render. Add a regression test in `ui/src/ui/chat/grouped-render.test.ts` asserting assistant bubbles with markdown actions get the spacing class, while user bubbles do not. Then close #69605 from the merged fix.
