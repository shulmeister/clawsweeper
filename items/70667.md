## Summary

Keep #70667 open. Current main still has the stale one-step `openclaw devices approve --latest` guidance in both files this PR targets, while the CLI implementation, tests, and `docs/cli/devices.md` already confirm that implicit or `--latest` approval is preview-only and requires a second explicit `requestId` command. The Greptile dirty-scope comment appears superseded by the force-push/current two-file PR context, so it is not a reason to close this docs fix.

## Best Possible Solution

Keep the PR open and land the narrow docs correction after normal maintainer review: update `apps/android/README.md` and `skills/node-connect/SKILL.md` to describe `approve --latest` as preview-only and add the required `openclaw devices approve <requestId>` step. No core behavior change is needed because `src/cli/devices-cli.ts`, tests, and `docs/cli/devices.md` already enforce the preview-only contract.
