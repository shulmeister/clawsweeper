## Summary

Keep #17424 open. Current main has partial `outputDimensionality` config plumbing and provider API support, but OpenAI/OpenAI-compatible memory embedding requests still do not send a `dimensions` field on either the inline `/embeddings` path or the OpenAI batch JSONL path. The recent maintainer/Codex issue comment that kept this open still matches current main.

## Best Possible Solution

Keep the issue open. Implement OpenAI-compatible memory embedding dimensions by deciding whether to support the issue's `dimensions` alias or document `outputDimensionality` as the supported key, carrying the resolved value into the OpenAI embedding client/runtime cache identity, adding `dimensions` to both inline and batch `/embeddings` request bodies where supported, and adding tests for defaults, per-agent overrides, inline embedding, batch indexing, and reindex/cache identity.
