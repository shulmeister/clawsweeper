## Summary

Keep #17424 open. Current main has generic `outputDimensionality` config plumbing and provider API support, but the OpenAI/OpenAI-compatible memory embedding request bodies still omit the `dimensions` parameter for both inline and batch embedding paths. The issue's requested user-visible capability remains missing.

## Best Possible Solution

Keep the issue open. Implement the missing OpenAI-compatible memory-search behavior by carrying the resolved dimension setting into the OpenAI remote client, adding `dimensions` to both inline and batch `/embeddings` request bodies, including it in cache/reindex identity where needed, and adding tests for defaults, per-agent overrides, inline embedding, and batch indexing. Maintainers should also decide whether to accept the issue's `dimensions` config alias or document `outputDimensionality` as the supported key.
