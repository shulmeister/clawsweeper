## Summary

Keep #17424 open. Current main has provider-specific `outputDimensionality` plumbing for Gemini/Bedrock-style providers, but it still does not support the requested `agents.defaults.memorySearch.dimensions` public config key and does not pass OpenAI/OpenAI-compatible `dimensions` in either inline or batch embedding requests.

## Best Possible Solution

Keep #17424 open. Maintainers should decide whether the public config should be `dimensions`, `outputDimensionality`, or an alias pair; then thread the resolved value through defaults and per-agent overrides into OpenAI/OpenAI-compatible inline and batch embedding requests, include the value in runtime cache/reindex identity, and add focused schema, resolver, request-body, batch, and cache/reindex tests.

## Risks / Open Questions

- Some OpenAI-compatible providers may reject a `dimensions` field, so implementation should gate or validate it by provider/model capability or explicit config.
- Changing embedding vector size must participate in cache and reindex identity so indexes do not mix vectors with different dimensions.
