## Summary

Keep open. Current main still routes Slack user targets through `conversations.open` before `chat.postMessage`, including plain text proactive sends, so the reported failure mode remains plausible when direct `chat.postMessage(channel=<USER_ID>)` succeeds but the conversations-open scope/path fails.

## Evidence

- **Slack send resolves user targets through conversations.open:** `resolveChannelId` treats explicit `user:` targets and bare U-prefixed IDs as user IDs, then calls `client.conversations.open({ users: recipient.id })` before returning the DM channel ID used for subsequent send operations.
