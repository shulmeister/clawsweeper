## Summary

Keep open. Current `main` still allows Discord startup to continue with `botUserId` unset when bot identity fetch fails, so the reported mention-gating/self-filter/reply-detection bug is not implemented on `main`. The PR is also stale against the current tree layout because Discord monitor code now lives under `extensions/discord/src/monitor/` rather than `src/discord/monitor/`.
