## Summary

Current `main` contains explicit ACPX fixes and test coverage for the failure modes described here: MCP injection on ACP bridge paths, Codex ACP bind/spawn flows with MCP bridges enabled, and startup ACP identity reconciliation. The fix appears to be on unreleased `main`, not in the latest tagged release `v2026.4.23`.

## Evidence

- **Unreleased changelog entry for the MCP-injection failure:** The unreleased `2026.4.24` changelog says ACPX now routes OpenClaw ACP bridge commands through an MCP-free runtime path so documented setups no longer fail on per-session MCP injection (`#68741`).
