## Summary

Keep #68285 open. Current main has Matrix and CLI-backend changes around this area, but the reporter’s concrete workaround was Gemini CLI tool auto-approval (`--yolo`), and main still does not configure Gemini CLI approval mode or mark the generated OpenClaw MCP server trusted. The 100% CPU Matrix symptom also remains unverified and was not covered by the reporter’s workaround comment.

## Best Possible Solution

Keep the issue open and split the fix path: first decide the intended Gemini CLI approval contract for OpenClaw-owned MCP tools, preferably by marking only the generated OpenClaw loopback MCP server trusted or otherwise passing the appropriate Gemini approval mode with tests and docs. Then request or capture Matrix logs/CPU profiling for the remaining 100% CPU symptom on a small Ubuntu VDS, because that part is not proven fixed by the current code or the reporter’s `--yolo` workaround.
