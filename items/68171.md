## Summary

Close #68171 as a duplicate of the older open doctor memory-probe false-warning issue #44426. Current main still has the same underlying 3s non-interactive `doctor.memory.status` timeout path, so this is not implemented; the best cleanup is to consolidate the remaining fix under the canonical issue instead of keeping a newer duplicate open.

## Best Possible Solution

Keep #44426 as the canonical issue for the remaining fix. The likely product/code fix belongs in the doctor gateway-memory probe path: make transient `doctor.memory.status` timeouts inconclusive or retry/back off for cold-start local embeddings, and add focused regression coverage for non-interactive doctor with local embeddings where `memory status --deep` would become ready after the short timeout.
