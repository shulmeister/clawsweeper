## Summary

Keep open. Current `main` still hardcodes a `3000ms` timeout for non-interactive `openclaw doctor` gateway health and memory probes, which matches the reported false warning path after restarts; `doctor` also exposes no timeout override, while adjacent health/status commands default to `10000ms`.

## Evidence

- **Non-interactive doctor still uses 3s for gateway health and memory probe:** `runGatewayHealthChecks()` passes `3000` ms when `ctx.options.nonInteractive === true` both to `checkGatewayHealth()` and `probeGatewayMemoryStatus()`. That is the exact path the issue describes for automation/headless doctor runs.
