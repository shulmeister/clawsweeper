## Summary

Keep open. Current `main` does not implement a cron `delivery.mode="session"` path. Core still only supports `none`, `announce`, and `webhook`, while `session:...` is a separate session-targeting mechanism, not a framework-managed post-run `sessions_send` delivery mode.

## Evidence

- **Cron delivery type excludes session mode:** `CronDeliveryMode` is still defined as only `"none" | "announce" | "webhook"`.
  - file: [src/cron/types.ts:22](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/cron/types.ts#L22)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Cron tool schema/docs expose only three modes:** The cron tool constants and user-facing guidance list only `none|announce|webhook`; there is no `session` delivery mode in the tool contract.
  - file: [src/agents/tools/cron-tool.ts:29](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/tools/cron-tool.ts#L29)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Docs separate session targets from delivery modes:** Docs describe `session:custom-id` as where the job runs, then document delivery/output as only `announce`, `webhook`, and `none`.
  - file: [docs/automation/cron-jobs.md:82](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/automation/cron-jobs.md#L82)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Delivery planner recognizes no session mode:** `resolveCronDeliveryPlan()` maps only `announce`, `webhook`, `none`, and legacy `deliver`; isolated/current/session targets still default to `announce`, not a session handoff.
  - file: [src/cron/delivery-plan.ts:29](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/cron/delivery-plan.ts#L29)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Existing session support is an alternate execution target, not delivery:** Cron accepts `session:...` as a `sessionTarget` and resolves `current` into `session:<sessionKey>`, which changes the run session rather than adding a post-run delivery channel.
  - file: [src/cron/service/jobs.ts:156](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/cron/service/jobs.ts#L156)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)

## Risks / Open Questions

- `session:...` targets may cover some adjacent workflows, but they are not the requested framework-managed delivery of one cron run's output into another session.
