## Summary

Keep PR #39891 open. Current main still creates a real Google Chat typing message before dispatch and only clears it inside the delivery path; an allowed group/channel exact `NO_REPLY` final can skip delivery entirely, leaving the typing message orphaned. The PR targets the right behavior, but it likely needs a refresh because current `monitor.ts` no longer imports `deleteGoogleChatMessage` directly, and it still needs a focused regression test.

## Best Possible Solution

Keep PR #39891 and linked issue #39843 active. The best maintainer path is to refresh the Google Chat fix against current main, delete the leftover `typingMessageName` after dispatch when delivery never ran, catch and log cleanup failures, avoid double-deleting after normal delivery, and add a focused regression test for Google Chat space reactions followed by exact `NO_REPLY`.
