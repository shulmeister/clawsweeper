## Summary

PR #71740 adds AssemblyAI as a bundled, service-specific media-understanding provider. Current main already exposes the media-understanding provider API, provider HTTP helpers, request override contract, plugin manifest contract, and ClawHub/npm install path needed for AssemblyAI to live as an external provider plugin, so the bundled-core PR can close as ClawHub/community plugin work.

## Best Possible Solution

Close the bundled-core PR and direct the AssemblyAI work to an external ClawHub/npm provider plugin using the existing media-understanding plugin API. Reopen core work only if the external implementation proves a concrete missing SDK or runtime seam.
