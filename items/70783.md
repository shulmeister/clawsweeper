## Summary

Current main does not implement the requested tabbed multi-chat workflow. The Control UI has one active chat view with a grouped session dropdown, deep links, persisted last-active session selection, and a Sessions table, but switching replaces the single active `sessionKey` and clears/reloads chat state rather than maintaining multiple open tabs with close/reorder/persisted tab state. The issue is recent, author is not a maintainer, has no protected label, and the provided related items are about unrelated session/system-message behavior, so there is no conservative close reason.

## Best Possible Solution

Keep #70783 open as an actionable Control UI feature. The likely product path is a first-class chat tab model in `ui/`: persist an ordered per-gateway open-session list, render a tab strip above the chat view, integrate tab switching with existing `switchChatSession`/`sessions.list`/URL behavior, add close/create/reorder flows, preserve per-tab local draft/attachment/sidebar state where intended, and cover desktop/mobile rendering plus session-switch persistence with focused UI tests.
