## Summary

Current main has added meaningful context-cost controls since the report, but the specific selective/lazy workspace-file injection request is not fully implemented. There is no general per-file `inject` policy, frontmatter-driven loading, TOC-only bootstrap mode, or budget-aware summarization for workspace bootstrap files.

## Evidence

- **Current behavior still injects bootstrap files broadly:** Docs state that AGENTS.md, SOUL.md, TOOLS.md, IDENTITY.md, USER.md, HEARTBEAT.md, BOOTSTRAP.md, and MEMORY.md are injected into the context window on every turn unless a file-specific gate applies; the documented file-specific gate only calls out HEARTBEAT.md omission when heartbeat guidance is disabled.
