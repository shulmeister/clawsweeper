## Summary

Keep open. Current `main` still routes Telegram native `/commands` through a group-policy check that ignores topic/group overrides, while normal Telegram message handling uses the topic-aware path. The reported forum-topic bypass remains credible on `95b7a85f0607784d8e33b4ce7e42617174196490`, and the issue timeline already points to an open fix PR rather than a landed change.

## Evidence

- **Native command auth ignores topic/group policy overrides:** `resolveTelegramCommandAuth()` resolves `topicConfig` and `groupConfig`, but its native-command policy check calls `evaluateTelegramGroupPolicyAccess()` with `useTopicAndGroupOverrides: false`, so forum-topic `groupPolicy` overrides are not applied for `/commands`.
  - file: [extensions/telegram/src/bot-native-commands.ts:450](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/telegram/src/bot-native-commands.ts#L450)
  - command: `nl -ba extensions/telegram/src/bot-native-commands.ts | sed -n '428,472p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Policy helper only honors topic overrides behind the flag:** `evaluateTelegramGroupPolicyAccess()` reads `topicConfig?.groupPolicy` and `groupConfig?.groupPolicy` only when `useTopicAndGroupOverrides` is true; otherwise it falls back to account/default policy only.
  - file: [extensions/telegram/src/group-access.ts:146](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/telegram/src/group-access.ts#L146)
  - command: `nl -ba extensions/telegram/src/group-access.ts | sed -n '138,160p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Regular message path does use topic-aware policy evaluation:** Normal Telegram inbound message handling calls the same policy helper with `useTopicAndGroupOverrides: true`, so current `main` has a real behavior split between messages and native commands.
  - file: [extensions/telegram/src/bot-handlers.runtime.ts:594](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/telegram/src/bot-handlers.runtime.ts#L594)
  - command: `nl -ba extensions/telegram/src/bot-handlers.runtime.ts | sed -n '594,610p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Native-command tests only cover account-level disabled policy:** The existing regression test asserts only top-level `channels.telegram.groupPolicy: "disabled"` for native commands. There is no native-command test here for a forum-topic override such as `topics.<id>.groupPolicy: "disabled"`.
  - file: [extensions/telegram/src/bot-native-commands.group-auth.test.ts:99](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/telegram/src/bot-native-commands.group-auth.test.ts#L99)
