## Summary

Keep #61034 open. Current main exposes reasoning visibility/effort controls, but I found no implemented option or config schema for choosing the language of visible reasoning output while leaving final responses unchanged. This is not a clean ClawHub close because the requested behavior sits in core agent config, prompt/runtime handling, and channel reasoning delivery rather than an optional standalone plugin with an already sufficient documented surface.
