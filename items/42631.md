## Summary

Keep open. Current `main` still limits job-level model overrides to isolated `agentTurn` cron jobs; `sessionTarget="main"` jobs must use `payload.kind="systemEvent"`, and that path neither accepts nor carries a per-job model/provider override.

## Evidence

- **Cron payload type only gives `model` to `agentTurn` jobs:** `CronPayload` is a union of `{ kind: "systemEvent"; text: string }` and `CronAgentTurnPayload`; the `model` field exists only on `CronAgentTurnPayload`. That means main-session `systemEvent` jobs have no typed per-job model field on current `main`.
  - file: [src/cron/types.ts:112](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/cron/types.ts#L112)
  - command: `nl -ba src/cron/types.ts | sed -n '70,130p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Gateway schema does not allow `model` on `systemEvent` payloads:** The cron protocol schema accepts `systemEvent` payloads with only `kind` and `text`; `model` is only present in the `agentTurn` payload schema. So API/CLI payload validation still rejects or omits a job-level model on `main + systemEvent` jobs.
  - file: [src/gateway/protocol/schema/cron.ts:130](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/gateway/protocol/schema/cron.ts#L130)
  - command: `nl -ba src/gateway/protocol/schema/cron.ts | sed -n '90,170p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Normalization strips model-related fields from `systemEvent` payloads:** When cron payload normalization sees `kind === "systemEvent"`, it explicitly deletes `model`, `fallbacks`, `thinking`, `timeoutSeconds`, `lightContext`, `allowUnsafeExternalContent`, and `toolsAllow`. Even if supplied, a per-job model override is removed on the main-session path.
  - file: [src/cron/normalize.ts:227](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/cron/normalize.ts#L227)
  - command: `nl -ba src/cron/normalize.ts | sed -n '180,245p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Service guard still forces `main` jobs to be `systemEvent` only:** `assertSupportedJobSpec` throws unless `sessionTarget === "main"` uses `payload.kind="systemEvent"`, while isolated/current/session jobs must use `agentTurn`. That prevents the existing per-job `payload.model` support from being used on main-session cron jobs.
  - file: [src/cron/service/jobs.ts:156](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/cron/service/jobs.ts#L156)
  - command: `nl -ba src/cron/service/jobs.ts | sed -n '150,230p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Main-session execution path only enqueues text and heartbeat:** `executeMainSessionCronJob` resolves only the system-event text, enqueues that event, and triggers heartbeat. It does not resolve or forward any per-job model/provider override for the target session.
  - file: [src/cron/service/timer.ts:1194](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/cron/service/timer.ts#L1194)
  - command: `nl -ba src/cron/service/timer.ts | sed -n '1190,1265p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Current docs still scope `--model` to isolated cron jobs:** The user-facing docs describe `--model` / `--thinking` under 'Payload options for isolated jobs' and state isolated cron model precedence explicitly, with no corresponding main-session support documented.
  - file: [docs/automation/cron-jobs.md:99](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/docs/automation/cron-jobs.md#L99)
