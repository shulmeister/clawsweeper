## Summary

Keep open. Current main still preserves JSON-stringified content-block arrays as plain strings, so the reported vLLM double-serialization path is not fixed. The PR remains useful, but its latest patch still needs a narrower detector before it is safe to merge or cleanup-close.

## Best Possible Solution

Keep this PR open and update it against current main. The best fix is a shared conservative parser that only treats a stringified JSON array as chat content when it has a recognizable chat-content-block shape, extracts text consistently with the array path, preserves arbitrary JSON strings, and is covered by focused regression tests for vLLM text blocks, mixed text/image blocks, invalid JSON, and typed non-chat arrays.
