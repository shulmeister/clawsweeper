## Summary

Keep #70046 open. Current main still only applies `--tz` to offset-less ISO datetimes, has no `HH:MM`/`HH:MM:SS` time-only branch in `parseAt`, and has no regression coverage for the PR's time-only cases. The PR remains the active clean respin for the open #59441 bug after the older #59444 branch was closed as noisy/unmerged.

## Best Possible Solution

Keep #70046 open for maintainer review. The best path is to land a focused cron parser fix that adds explicit `HH:MM`/`HH:MM:SS` handling in `src/cli/cron-cli/shared.ts`, resolves time-only input against today's date in the requested IANA timezone, returns `null` for invalid zones/ranges, and adds focused regression tests for #59441. The older #59444 should remain closed as superseded by this cleaner PR.
