## Summary

Current main already avoids the reported starvation path: interactive turns run on a per-session lane, while exec/heartbeat/stall notifications are queued as session-scoped system events and heartbeat wakes defer when the main or target session lane is busy. Deferred maintenance also runs on its own lane and waits for the session lane to go idle.

## Evidence

- **Interactive turns use a dedicated session lane:** Embedded agent runs resolve a session lane from the session key and enqueue work on that lane before the global lane, so inbound interactive work is no longer sharing one flat FIFO with unrelated background work.
  - file: [src/agents/pi-embedded-runner/run.ts:249](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/pi-embedded-runner/run.ts#L249)
  - command: `nl -ba src/agents/pi-embedded-runner/run.ts | sed -n '244,268p'`
- **System events are stored separately from the command lane queue:** System events are kept in a lightweight in-memory per-session queue and appended via enqueueSystemEvent(), rather than being inserted into the command lane queue ahead of user turns.
  - file: [src/infra/system-events.ts:1](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/infra/system-events.ts#L1)
  - command: `nl -ba src/infra/system-events.ts | sed -n '1,120p'`
- **Exec and stall notifications enqueue system events plus a heartbeat wake:** Node exec lifecycle events and CLI stall notices both call enqueueSystemEvent(...) and requestHeartbeatNow(...), so they do not directly occupy the interactive session lane.
  - file: [src/gateway/server-node-events.ts:687](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/gateway/server-node-events.ts#L687)
  - command: `nl -ba src/gateway/server-node-events.ts | sed -n '663,706p'`
- **Heartbeat explicitly defers when main/session lanes are busy:** runHeartbeatOnce() returns skipped/requests-in-flight when either the main lane or the resolved session lane is busy, with comments stating this avoids interrupting an active streaming turn and lets the wake layer reschedule later.
  - file: [src/infra/heartbeat-runner.ts:759](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/infra/heartbeat-runner.ts#L759)
  - command: `nl -ba src/infra/heartbeat-runner.ts | sed -n '754,792p'`
- **Regression test covers queued exec event plus busy session lane:** The test seeds a queued exec-completed system event, reports the session lane as busy, and asserts the heartbeat is skipped with requests-in-flight and no reply call, which is the reported starvation case handled conservatively.
  - file: [src/infra/heartbeat-runner.skips-busy-session-lane.test.ts:64](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/infra/heartbeat-runner.skips-busy-session-lane.test.ts#L64)
  - command: `nl -ba src/infra/heartbeat-runner.skips-busy-session-lane.test.ts | sed -n '60,120p'`
- **Deferred maintenance moved off the interactive lane:** Deferred turn maintenance has its own lane prefix, waits until the session lane reaches zero, and is enqueued on the separate maintenance lane, so maintenance work no longer sits ahead of user turns in the session lane.
  - file: [src/agents/pi-embedded-runner/context-engine-maintenance.ts:103](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/pi-embedded-runner/context-engine-maintenance.ts#L103)
  - command: `nl -ba src/agents/pi-embedded-runner/context-engine-maintenance.ts | sed -n '100,110p;376,404p;532,550p'`

- The implementation achieves the requested behavior via separate system-event queues, deferred wakes, and separate maintenance lanes rather than a literal named priority-lane API.

## Close Comment

Closing from a Codex review because current `main` already prevents queued system events from blocking interactive turns. Interactive runs are enqueued on a per-session lane in `src/agents/pi-embedded-runner/run.ts:249`. Exec/stall notifications are queued as session-scoped system events instead of entering that lane directly in `src/gateway/server-node-events.ts:687` and `src/agents/cli-runner/execute.ts:486`, and heartbeat wakes now explicitly defer when the main lane or target session lane is busy in `src/infra/heartbeat-runner.ts:759`. There is also regression coverage for the busy-session-lane case in `src/infra/heartbeat-runner.skips-busy-session-lane.test.ts:64`. Deferred maintenance now runs on its own lane and waits for the session lane to go idle in `src/agents/pi-embedded-runner/context-engine-maintenance.ts:103` and `src/agents/pi-embedded-runner/context-engine-maintenance.ts:545`. This addresses the starvation described here, even though the implementation uses deferral and separate queues/lanes rather than a literal "priority lanes" feature.
