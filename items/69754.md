## Summary

Keep #69754 open. Current main still routes Feishu approval forwarding through generic text-command fallback instead of Feishu-specific approval cards or explicit degraded-UX guidance. The adjacent #58298 is about agent-authored cards, while the remaining problem here is built-in approval fallback clarity; related native Feishu exec-approval work also remains unshipped/open rather than implemented on main.

## Best Possible Solution

Keep the issue open. The best path is to add Feishu-specific approval fallback copy and regression coverage now, and coordinate with the separate Feishu native exec approval card work so built-in approvals either render Feishu action cards or clearly explain why the current surface is using manual `/approve` text and where the user can complete it.
