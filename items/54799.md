## Summary

Keep this issue open. Current main has generic cron jobs, plugin lifecycle hooks, and skill metadata/status surfaces, but it still does not implement declarative SKILL.md schedule/event trigger metadata, automatic skill-to-cron reconciliation, last-triggered skill status, or a skills audit command. The existing plugin cron hook surface is useful context, but it is not a complete shipped replacement for the requested core skill metadata/status contract.

## Best Possible Solution

Keep this open for maintainer product/API design. The strongest implementation path is a versioned `metadata.openclaw.triggers` contract in the skills metadata parser, validation/tests, scheduler reconciliation on gateway startup and skill refresh, event-trigger mapping through the lifecycle hook taxonomy, persisted last-triggered state, and `openclaw skills list/info/audit` plus docs. If maintainers prefer ClawHub, first add a public plugin API that lets a plugin inspect normalized skill metadata/status and reconcile Gateway cron jobs without importing private `src/agents/**` internals.
