## Summary

Keep open. Current `main` still appears to expose the reported heartbeat user prompt on the Control UI path: the UI still loads history through `chat.history`, the UI-side history filter does not hide heartbeat user prompts, and the gateway `chat.history` sanitizer still only drops assistant-side silent/commentary entries. Current `main` does contain heartbeat prompt filtering in the newer session-history snapshot path, but that code is wired to `/sessions/:sessionKey/history` HTTP/SSE rather than the Control UI's existing `chat.history` request flow.
