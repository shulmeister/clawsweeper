## Summary

Keep #10213 open. Current main still does not implement Anthropic server-side compaction: there is no config/schema option for an Anthropic-native opt-in, the Anthropic plugin wrapper only handles existing beta/service-tier/fast-mode behavior, and the native Anthropic transport does not send `context_management.edits` or preserve streamed `compaction` blocks. The related implementation PR #10214 is closed unmerged, while #65287 is broader follow-up work rather than a maintainer-designated replacement; the latest issue discussion also explicitly kept this focused tracker open.

## Best Possible Solution

Keep #10213 open as the focused Anthropic server-side compaction tracker. A maintainer-quality fix should add an explicit opt-in model/provider parameter or config, validate and document it, inject the current Anthropic compaction beta plus `context_management.edits` only for eligible direct Anthropic requests, preserve returned `compaction` blocks for replay, handle streaming and `compaction` stop behavior, account for usage iterations, and add focused tests for both the bundled Anthropic plugin wrapper and native Anthropic transport paths. Treat #65287 as related broader design work unless maintainers explicitly mark it canonical.
