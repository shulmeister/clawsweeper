## Summary

Keep #10213 open. Current main at 04e08cea62983b2c766d15f61f15b3ffc90f9baa still does not implement Anthropic server-side compaction: the config surface is absent, the strict schema would reject the proposed `agents.defaults.compaction.serverSide` shape, the bundled Anthropic provider wrapper only handles existing beta/header, service-tier, and fast-mode behavior, and the only shipped `context_management` server-side compaction path is OpenAI Responses-specific. The linked implementation PR #10214 is closed and unmerged, and the prior issue discussion already recorded a keep-open review for the same focused tracker.

## Best Possible Solution

Keep #10213 open as the focused tracker for Anthropic-native server-side compaction. A good fix should add an explicit opt-in config or per-model parameter, validate and document it, update the bundled Anthropic provider wrapper/payload policy to add the correct beta header and `context_management.edits` payload without clobbering existing headers or payload hooks, handle/log Anthropic compaction response events, and add targeted tests comparable to the existing OpenAI Responses server-side compaction coverage. Coordinate with broader related issue #65287 only if maintainers decide that newer issue should become the canonical tracker.
