## Summary

Keep #10213 open. Current main still has local/OpenClaw compaction and OpenAI Responses server-side compaction, but not Anthropic-native server-side compaction: no accepted config/schema surface, no Anthropic request injection for `context_management.edits`, no dedicated beta-header handling for `compact-2026-01-12`, and no transport/content model support for Anthropic compaction blocks or a `compaction` stop reason. The linked implementation PR #10214 was closed unmerged, and the open #65287 is broader cache/TTL work rather than an explicit canonical replacement for this focused request.

## Best Possible Solution

Keep #10213 open as the focused Anthropic server-side compaction tracker. A complete implementation should add an explicit opt-in config or per-model/provider params, validate and document it, inject `compact-2026-01-12` plus `context_management.edits` only for direct Anthropic-compatible requests, preserve/replay Anthropic compaction blocks in streamed and stored assistant history, handle Anthropic compaction/pause stop behavior, add logging, and cover the provider wrapper plus transport paths with focused tests. Treat #65287 as related broader prompt-cache/native-compaction design unless maintainers explicitly mark it canonical.
