## Summary

Keep open. Current main at 2f6615d2ee9d0d9db0f4ad1e13f71d1c0df8166d still does not implement Anthropic server-side compaction: the requested compaction config surface is absent, the strict schema would not accept `agents.defaults.compaction.serverSide`, the Anthropic provider wrapper only handles existing beta/header and service-tier behavior, and the only `context_management` server-side compaction implementation is OpenAI Responses-specific. The linked implementation PR #10214 was closed unmerged, and the related reports do not provide a shipped or canonical superseding fix.

## Best Possible Solution

Keep #10213 open as the focused tracker for Anthropic-native server-side compaction. A good implementation should add an explicit opt-in config or per-model parameter, validate and document it, update the bundled Anthropic provider wrapper to add the correct beta header and `context_management.edits` payload without clobbering existing headers/payload hooks, handle/log Anthropic compaction response events, and add targeted tests comparable to the existing OpenAI Responses server-side compaction coverage.
