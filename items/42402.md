## Summary

Keep open. Current main still lets truthy non-string runtime values from `env.vars` pass through `collectConfigEnvVarsByTarget` and into `containsEnvVarReference`, which still assumes a string and calls `.includes()`. The PR is stale against the current file split because `src/config/env-vars.ts` now re-exports from `src/config/config-env-vars.ts`, but the underlying guard and regression test are still useful.

## Best Possible Solution

Keep the PR open and adapt it to current main: add a `typeof value !== "string" || !value.trim()` guard in `src/config/config-env-vars.ts` for `envConfig.vars`, add the regression test to the existing `src/config/config.env-vars.test.ts`, and decide separately whether non-string values should warn or simply be ignored.
