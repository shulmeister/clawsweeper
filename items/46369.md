## Summary

Keep open. Current main still does not implement the requested cleanup on `cron.remove`: the RPC path reaches `CronService.remove`, which delegates to `ops.remove`, and that implementation only removes the job from the cron store, persists, rearms the timer, and emits a removed event. The existing session reaper only prunes expired run-session keys and explicitly keeps the base cron session, so this does not satisfy the issue's sessions.json cleanup request. The related PRs #46475 and #46506 are open fix attempts, not merged superseding work.
