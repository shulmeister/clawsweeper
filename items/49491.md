## Summary

Current main already covers the report’s bundled behaviors: fallback success is surfaced as fallback status instead of a user-facing lifecycle error, ChatGPT usage-limit text is preserved instead of being collapsed to the generic API-rate-limit copy, and Control UI auth failures now expose actionable token-mismatch/rate-limit states with reconnect suppression for non-recoverable auth errors.

## Evidence

- **Control UI ignores lifecycle error events and only renders fallback/compaction state:** `handleLifecycleFallbackEvent` returns unless the lifecycle phase is `fallback` or `fallback_cleared`, and `handleAgentEvent` only routes lifecycle events into compaction/fallback handlers. That means a failed primary attempt no longer becomes a generic UI lifecycle error banner in the fallback-success path.
