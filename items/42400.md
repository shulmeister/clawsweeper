## Summary

Keep open. Current main does not implement the PR's `neverReply` config or equivalent no-LLM passive group observation behavior. The newer `sendPolicy: deny` path only suppresses outbound delivery while still running inbound agent processing, and the plugin `inbound_claim` hook exists but is not globally broadcast from dispatch. Because the related plugin route still has a concrete missing core/API issue (#48434) and this PR contains useful cross-channel work and review discussion, none of the conservative close reasons apply.
