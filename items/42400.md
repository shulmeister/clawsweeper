## Summary

Keep open. Current main does not implement `channels.*.neverReply`, and the ClawHub/plugin direction is not yet a clean close because unbound `inbound_claim` broadcast is still missing on main. The PR discussion usefully points to `flowolforg/openclaw-listen-only` and #48434 as a possible architectural path, but that means remaining core/plugin API work still exists rather than this being ready to auto-close.

## Evidence
