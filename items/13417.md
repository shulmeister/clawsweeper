## Summary

Current main still does not implement outbound WhatsApp Newsletter/Channel targets for the message tool. `@newsletter` targets are rejected during WhatsApp target normalization, outbound target resolution still advertises only E.164 or group JID support, metadata/session routing still models only direct/group WhatsApp targets, and the related PR #13424 was closed unmerged. The adjacent #62697 is inbound newsletter handling, not a superseding outbound tracker.

## Best Possible Solution

Keep #13417 open as the outbound WhatsApp Newsletter/Channel message-tool tracker. The fix likely belongs in the bundled WhatsApp plugin: add a newsletter JID recognizer/normalizer, define outbound allowlist/default-target semantics for newsletter channels, model newsletter/channel routing separately from direct/group sessions, route sends through the active listener, add focused normalization/resolver/session/send tests, and update the message and WhatsApp docs. Treat #62697's inbound newsletter handling and optional newsletter metadata/follow/react operations as separate follow-up work unless maintainers choose one broader WhatsApp Channels project.
