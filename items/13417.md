## Summary

Current main still does not implement WhatsApp Newsletter/Channel JID support for the message tool. `@newsletter` targets are rejected by WhatsApp target normalization before they can reach Baileys, the outbound/message metadata still advertises E.164 or group JIDs only, and the related PR #13424 was closed unmerged. The prior issue discussion also explicitly kept this open with the same current-product boundary.

## Best Possible Solution

Keep #13417 open until the bundled WhatsApp plugin supports Newsletter/Channel targets end to end: add newsletter JID recognition/normalization, decide the outbound allowlist and chat/session model, route sends consistently through the active listener, add focused tests, and update the message/WhatsApp docs. Newsletter metadata/follow/react actions can be separate follow-up work after send-target support is correct.
