## Summary

Keep open. Current main still does not accept WhatsApp Newsletter/Channel JIDs: `@newsletter` is absent from the WhatsApp code/docs, target normalization rejects non-user/non-group JIDs containing `@`, and the message tool still documents/advertises WhatsApp targets as E.164 or group JIDs only. The related PR #13424 was closed unmerged and does not supersede the issue.

## Best Possible Solution

Keep this issue open until the WhatsApp plugin implements first-class Newsletter/Channel target support. A complete fix likely belongs in `extensions/whatsapp`: add newsletter JID recognition/normalization, update outbound allow/route/chat-type handling, decide the session peer model, add focused tests, and update message/WhatsApp docs. Newsletter-specific metadata/react/follow actions can be tracked separately after send-target support is correct.
