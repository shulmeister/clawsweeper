## Summary

Keep open. Current main still has direct embedded-runner helper paths, cron still checks CLI-ness before resolving canonical-provider CLI runtime aliases, and open PR #72499 from the same author uses closing syntax for this exact work. Per the cleanup rule, the issue should stay open while that implementation candidate is reviewed.

## Best Possible Solution

Keep this issue open and review/land #72499, or an equivalent patch, that applies the existing resolveCliRuntimeExecutionProvider(...) ?? provider plus isCliProvider(...) dispatch pattern to queued followups, models status --probe, LLM slug generation, and cron. Add focused regression coverage for those paths, then let the merged PR close the issue.
