## Summary

Keep open. Current main has a concrete cron `failureAlert` path and Telegram/outbound delivery already supports silent sends, but the cron failure-alert contract does not expose `silent`, the resolver and `sendCronFailureAlert` callback do not carry it, and the Gateway delivery call does not pass `silent` into `deliverOutboundPayloads`. The provided related items are broader cron requests and do not supersede this focused feature request.

## Best Possible Solution

Keep the issue open and implement an additive cron contract change: add optional `silent?: boolean` to `CronFailureAlert` and its TypeBox/Zod/config surfaces as appropriate, merge and resolve it in `src/cron/service/jobs.ts` and `src/cron/service/timer.ts`, pass it through `sendCronFailureAlert`, and forward it to `deliverOutboundPayloads`. Add focused cron failure-alert tests plus a docs update for the public cron configuration surface.
