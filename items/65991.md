## Summary

Keep open. Current main still does not accept or propagate a `silent` option for cron `failureAlert`; the request is a concrete cron/outbound integration gap, not a duplicate, ClawHub-only plugin idea, or stale report. Telegram and generic outbound delivery already support silent sends, so the remaining work is to thread the option through the cron failure-alert contract and delivery path.

## Best Possible Solution

Implement a narrow cron failure-alert contract change on main: add `silent?: boolean` to per-job `CronFailureAlert` and any intended global/default `cron.failureAlert` config shape, update TypeBox/Zod schemas, docs, UI/CLI form and protocol types, then thread it through `resolveFailureAlert`, `emitFailureAlert`, `sendCronFailureAlert`, and `sendFailureNotificationAnnounce` into `deliverOutboundPayloads({ silent })`. Add regression coverage proving Telegram cron failure alerts set `disable_notification` when `silent: true` while preserving the default noisy behavior.
