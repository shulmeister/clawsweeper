## Summary

Current main has related plumbing, but the requested interactive per-message model override is still missing. `/model` remains a persistent session directive, `/btw` is one-shot but uses the current model, and per-run provider/model overrides are only exposed through trusted gateway/agent-call paths rather than a user-facing `/ask` or `/model --once` flow.
