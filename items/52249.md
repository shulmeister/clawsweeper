## Summary

Keep open. Current `main` no longer uses the old direct-resume path described in the report, but ACP parent-stream completion still only enqueues a system event and requests a heartbeat wake. On current `main`, the wake layer returns immediately when no heartbeat handler is registered, and heartbeat-disabled runs short-circuit as `disabled`, so there is still no self-sufficient parent wake path here.

## Evidence

- **ACP parent relay still depends on heartbeat wake:** `startAcpSpawnParentStreamRelay()` emits parent updates by calling `enqueueSystemEvent(...)` and then `requestHeartbeatNow(...)`. I found no alternate resume or delivery path in this relay.
