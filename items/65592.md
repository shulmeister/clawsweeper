## Summary

Keep this PR open. Current main documents the SearXNG JSON API prerequisite, but it still does not show that provider-specific guidance during `openclaw onboard` or `openclaw configure --section web`. The branch remains a focused implementation candidate because the PR discussion shows the author moved away from the initially late `runSetup` approach toward a pre-credential `credentialNote` path.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. The likely best path is a generic, plugin-owned pre-credential note field if maintainers want that public setup contract, plus setup-flow tests for normal text credentials, SecretRef mode, and quickstart behavior. If maintainers do not want a new public field, the narrower alternative is a SearXNG-owned setup hook that still displays before collecting the base URL.
