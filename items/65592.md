## Summary

Keep this PR open. Current main documents the SearXNG JSON API prerequisite, but it still does not show that provider-specific note during `openclaw onboard` or `openclaw configure --section web`. The PR is a focused implementation candidate, and the provided PR history shows the author moved from the initially problematic `runSetup` approach to a `credentialNote` field displayed before the credential and secret-ref paths.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for showing SearXNG JSON API guidance during onboarding/configure. The likely best path is to land a generic, plugin-owned pre-credential note field only if maintainers want that public setup contract, with tests covering normal credential prompts, secret-ref mode, and quickstart skip behavior; otherwise the narrower fallback is a SearXNG-specific setup-flow hook that still displays before collecting the base URL.
