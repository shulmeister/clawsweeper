## Summary

Current `main` contains a targeted fix for this Discord thread-binding failure: Discord thread-binding managers now live in a `globalThis`-backed shared registry so ESM and Jiti/plugin-hook loader paths see the same manager, and subagent spawn routing now preserves the caller account for same-agent spawns instead of drifting to a default/bound account. Regression tests were added for both behaviors, which matches the issue’s suspected root causes.

## Evidence
