## Summary

Keep open. Current `main` still does not defensively normalize `approvals.exec.mode` inside the approval forwarder, so any invalid mode that reaches runtime produces no approval targets and returns `false` instead of falling back to `"session"`. The config schema does restrict file-based values to `"session" | "targets" | "both"`, but that only proves validation at the config boundary, not the runtime behavior claimed fixed in the issue comment. I also did not find approval-specific subagent-routing coverage that would justify closing the broader reliability part of the report.
