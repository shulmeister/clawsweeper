## Summary

Current `main` still treats a second resolve on the same approval ID as `unknown or expired approval id`, and the Telegram callback path has no idempotent duplicate-resolution handling. The issue has a fresh non-restart repro in the thread and I found no shipped main-branch fix that would justify closing it.

## Evidence
