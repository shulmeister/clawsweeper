## Summary

Keep #40252 open. Current main still implements the reported iMessage behavior: blank media-only sends resolve the attachment MIME type, synthesize `<media:image>` / `<media:${kind}>`, and pass that synthesized value as the outbound JSON-RPC `text` field alongside `file`. The related fix PRs in the provided GitHub context (#40341 and #46343) are closed unmerged, and current tests still encode placeholder-based media-only echo/cache behavior.

## Best Possible Solution

Keep the issue open and fix the iMessage plugin send path in `extensions/imessage/src/send.ts` so media-only sends pass an empty text field plus `file`, while preserving text+media captions, reply IDs, directive stripping for real text, and the empty-payload guard. Add focused regression coverage for `sendMessageIMessage()` and media-only reply delivery asserting `params.text`/cached visible text does not become `<media:image>` for image/audio/video media-only sends.
