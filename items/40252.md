## Summary

Keep #40252 open. Current main still synthesizes `<media:image>` / `<media:${kind}>` for blank iMessage media sends and passes that value as the JSON-RPC `text` field alongside the file attachment. The linked fix PRs #40341 and #46343 were closed unmerged, and the current tests still encode the placeholder behavior for media-only iMessage replies.

## Best Possible Solution

Keep the issue open and fix the iMessage send path in `extensions/imessage/src/send.ts` so media-only sends pass an empty text field plus `file`, while preserving text+media captions, the empty-payload guard, reply IDs, and echo-cache behavior. Add focused regression coverage for `sendMessageIMessage()` or the outbound media path asserting media-only image/audio sends do not put `<media:...>` in `params.text`.
