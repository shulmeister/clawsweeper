## Summary

Keep this PR open. Current main has already shipped the related inbound NUL hardening for the named crash, but the PR still overlaps unshipped Gemini 3.1 template behavior and Sonnet 4.6 context-window behavior. It also has an unresolved review concern on the Sonnet fallback, so the conservative path is maintainer review/rebase rather than auto-closing.

## Best Possible Solution

Keep the PR open for maintainer review or ask the contributor to rebase/split it: retain only any still-needed generic NUL sanitizer change, verify whether Gemini CLI should prefer 3.1 template IDs under current Google catalog behavior, and address the Sonnet 4.6 review comment by limiting any native 1M fallback to explicit Anthropic/Claude CLI provider contexts while preserving `models.providers` overrides.
