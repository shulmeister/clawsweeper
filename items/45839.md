## Summary

Keep open. Current main supports Telegram inline buttons and callback routing for other flows, including exec/plugin/native callbacks, but the DM pairing workflow still emits a text-only pairing code with CLI approval instructions and has no pairing approve/deny callback path. The related closed generic inline-button items do not cover this system-generated pairing request workflow.
