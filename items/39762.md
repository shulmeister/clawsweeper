## Summary

Keep #39762 open. Current main at e672b61417af5c45b0431df6d9109a1f4b618ef5 still does not implement Feishu read-only group mode: the config/docs surface is absent, accepted group messages still create real reply dispatchers, and outbound Feishu target resolution has no group-reply guard. The PR discussion remains actionable because the author addressed earlier broadcast and whitespace review feedback, while the later raw `oc_*` chat-id bypass and unrelated `.secrets.baseline` churn still need maintainer review before merge.

## Best Possible Solution

Keep this PR open for Feishu plugin maintainer review. The best path is a focused bundled Feishu plugin change: add `disableGroupReplies` to Feishu config/schema/docs, enforce a no-op dispatcher for accepted group messages in both broadcast active-agent and single-agent paths, block explicit and bare Feishu `chat_id` outbound targets when enabled, add targeted regression tests, and remove or separately justify the `.secrets.baseline` churn before merge.
