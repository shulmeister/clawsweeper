## Summary

Keep open. Current main has partial hygiene: raw validation details are logged and returned as tool results for the agent, and details are hidden from normal user payloads unless verbose is enabled. But the requested UX is broader than raw-detail redaction: current payload policy still intentionally sends generic mutating-tool failure warnings such as `Write failed` to chat surfaces, including the same class of missing-parameter write failures described here.

## Best Possible Solution

Keep this issue open and implement a typed/classified tool-input validation path. Required-parameter failures from `write`, `edit`, `exec`, and similar tools should still be returned to the agent as tool errors and logged, but final chat payloads should suppress those validation-only warnings by default unless verbose diagnostics are explicitly requested. Add regression coverage around `buildEmbeddedRunPayloads` for mutating validation errors and an integration-style reply path for a chat channel such as Feishu/DM.
