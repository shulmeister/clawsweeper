## Summary

Current main implements the observable request through the existing responsePrefix cascade with model-aware template variables. Slack replies are wired through the shared reply pipeline, the selected provider/model is captured before delivery, and outbound normalization prepends the resolved prefix to Slack reply text. This supports per-channel/account Slack prefixes such as model/provider labels or Slack custom emoji shortcodes without adding the exact modelPrefixes/prefixTemplate shape proposed in the issue.
