## Summary

Keep open. Current main supports Slack/global `responsePrefix` strings with model/provider template variables, but it does not implement the requested per-model emoji mapping (`modelPrefixes`) or a `{emoji}`-style prefix template. The feature is only partially approximable today.

## Evidence

- **Slack config surface only exposes a single string prefix:** `SlackAccountConfig`/channel config has `responsePrefix?: string` plus ack/typing reactions; there is no Slack config field for `modelPrefixes` or `prefixTemplate`.
  - file: [src/config/types.slack.ts:196](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/config/types.slack.ts#L196)
