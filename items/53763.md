## Summary

Keep this issue open. Current main has a first-class bundled browser plugin, headless controls, remote CDP support, and optional sandbox/browser Docker paths, but the central request is not fully shipped: ordinary managed browser launch still depends on a configured or system-installed Chromium-family executable, and container/browser binary installation remains optional/manual rather than a built-in runtime that works without external browser setup.

## Best Possible Solution

Keep this issue open and scope it to the remaining product decision: whether OpenClaw should ship a pinned Chromium runtime by default, publish an official prebuilt Docker/browser variant, or add installer/doctor flows that automatically install and persist Playwright Chromium and wire it into executable discovery. Reuse the existing browser plugin, headless config, remote CDP support, custom launch args, and sandbox-browser container rather than rebuilding the already-shipped control surface.
