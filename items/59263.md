## Summary

Current `main` already implements the WhatsApp QR-login recovery and auth-persistence hardening described here. The code now explicitly handles the Baileys post-scan `515` restart path, clears/relinks on `401 loggedOut`, waits for queued creds writes before reopening sockets, and has targeted regression coverage. The changelog shows this shipped in the `2026.4.18` release line.

## Evidence
