## Summary

Current main already resolves the llm-slug-generator model through the central default-model resolver instead of hook-local parseModelRef, so bare aliases from agents.defaults.models are resolved before default-provider fallback. The PR is redundant with main; the change is not in the latest tagged release v2026.4.24, so no fixed release is claimed.

## Best Possible Solution

Close this PR as redundant and keep the current main implementation that routes llm-slug-generator through resolveDefaultModelForAgent and the shared alias-aware model resolver. A future narrow test-only PR could add an explicit llm-slug-generator alias fixture, but the requested behavior is already implemented on main.
