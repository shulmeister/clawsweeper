## Summary

Keep #71335 open. Current main still defaults `memorySearch.sync.watch` to `true` without considering gateway mode, and both builtin and QMD memory managers create chokidar watchers when that resolved flag is true. The related #64549 is a separate open auto-index correctness bug, not a canonical tracker for this gateway default/FD-pressure policy change.

## Best Possible Solution

Keep the issue open and implement a gateway-aware memory sync default in the config-resolution path used by memory-core managers. The fix should make the implicit watch default false for headless gateway mode, keep explicit `sync.watch: true` authoritative, keep CLI/desktop behavior unchanged, and add focused tests in `src/agents/memory-search.test.ts` plus memory-core watcher coverage for implicit gateway default, explicit true, and explicit false. Update schema help and public memory docs if the default changes.

## Risks / Open Questions

- Changing this default affects freshness for gateway users who rely on external file edits being indexed before an explicit search, session-start sync, interval sync, or manual `openclaw memory index`.
- Implementation needs to preserve explicit `sync.watch: true` and the current CLI/desktop default while defining precisely whether the trigger is `gateway.mode`, the running gateway process, or both.
