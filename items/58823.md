## Summary

Keep this PR open. Current main still implements and tests the opposite precedence in both the shared subagent model helper and the isolated cron path, while the public sub-agent docs support the reporter's global subagent-default expectation. This PR is also paired with open issue #58822 by the same contributor and has an unresolved allowlist-validation concern that should be reviewed before landing.

## Best Possible Solution

Keep this PR open for maintainer review. Maintainers should decide whether `agents.defaults.subagents.model` is intended to outrank an agent's own `model.primary`; if yes, land a narrow fix in `src/agents/model-selection.ts` and `src/cron/isolated-agent/model-selection.ts` with regression coverage and subagent-spawn allowlist validation. If #58003's current precedence is the intended contract, update `docs/tools/subagents.md` and related config docs, then resolve the paired issue accordingly.
