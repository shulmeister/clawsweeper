## Summary

Keep PR #58823 open. Current main still implements and tests the opposite precedence from the PR in both the shared subagent model helper and the isolated cron path. The PR remains a focused candidate for open issue #58822, but it reverses the behavior intentionally shipped by #58003 and now has an allowlist/security concern in the PR discussion, so this needs maintainer product/security judgment rather than conservative cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. Decide the intended subagent model precedence contract, then either land a narrow fix in src/agents/model-selection.ts and src/cron/isolated-agent/model-selection.ts with regression coverage for explicit spawn override, per-agent subagents.model, global subagent default, target-agent primary fallback, and allowlist validation, or keep #58003's precedence and update docs plus close #58822/#58823 with that rationale.

## Risks / Open Questions

- Merging as-is would reverse #58003's named-agent precedence contract, so maintainers need to decide whether global subagent defaults or target-agent primary models should win when both are configured.
- The subagent spawn path builds a direct initial session patch for the resolved model; the allowlist concern raised in the PR discussion should be resolved before changing precedence.
- Docs and current tests disagree on the intended contract, so closing this PR as obsolete would hide a real unresolved product/API decision.
