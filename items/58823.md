## Summary

Keep PR #58823 open. Current main still implements the opposite precedence from the PR in both the shared subagent model helper and isolated cron path, and checked-in tests assert that behavior. The PR is a focused candidate for open issue #58822, but it would reverse the behavior merged in #58003 and now has model-allowlist concerns in the latest PR discussion, so this needs maintainer product/security judgment rather than conservative cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. Decide the intended subagent model precedence contract first; if global `agents.defaults.subagents.model` should be a true subagent default, land a narrow fix in `src/agents/model-selection.ts` and `src/cron/isolated-agent/model-selection.ts` with regression coverage for explicit spawn override, per-agent `subagents.model`, global subagent default, fallback to agent primary/default model, and the allowlist behavior raised in the PR discussion. If #58003's agent-primary precedence is intended, update the subagents docs and close #58822/#58823 with that rationale.
