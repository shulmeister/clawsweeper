## Summary

Keep open. Current main still implements the precedence that PR #58823 is trying to change: per-agent subagent model, then the agent's own model, then `agents.defaults.subagents.model`. Checked-in tests also assert that current behavior, while the user-facing subagents docs support the PR's reported expectation that `agents.defaults.subagents.model` should override caller inheritance. This is a real product-contract conflict with merged PR #58003, not something already implemented or safely auto-closeable.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to settle the intended subagent model precedence contract, then land a narrow update across `src/agents/model-selection.ts` and `src/cron/isolated-agent/model-selection.ts` with tests covering explicit spawn/job override, per-agent `subagents.model`, global `agents.defaults.subagents.model`, and fallback to the agent primary model. If #58003's named-agent behavior should remain, implement that distinction explicitly and update docs/tests to match.
