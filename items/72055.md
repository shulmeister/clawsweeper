## Summary

Keep PR #72055 open. Current main still does not expose a Bonjour plugin config key for the mDNS service instance name: the bundled bonjour entry always formats `ctx.machineDisplayName`, and its manifest schema has no config properties. The existing advertiser and plugin API already provide the needed seams, so this PR remains a focused implementation path rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open for review, or land an equivalent maintainer-owned patch: add `instanceName` to `extensions/bonjour/openclaw.plugin.json`, resolve it from `api.pluginConfig` in `extensions/bonjour/index.ts` before falling back to `ctx.machineDisplayName`, cover unset/empty/suffixed/non-string cases in bonjour plugin-entry tests, and document `plugins.entries.bonjour.config.instanceName` on the public Bonjour docs page.
