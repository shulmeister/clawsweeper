## Summary

Keep open. Current main still materializes inherited Discord thread configured ACP bindings with the parent channel ID, so sibling Discord threads under the same parent can still share one configured ACP session key. The broader open #50955 work is related but complementary, not a clear superseding implementation for this narrow bug fix.

## Best Possible Solution

Keep this PR open as a focused implementation candidate for the Discord configured ACP thread-session collision. The best path is a surgical Discord parent-match change that materializes the live thread ID while retaining `parentConversationId`, plus regression coverage proving two Discord thread IDs under one parent produce distinct configured ACP session keys and explicit handling or documentation for session-key-only reverse lookup. Coordinate with #64199 as the matching bug report and #50955 as broader ACP thread binding/routing work, but do not treat either as a merged supersession yet.
