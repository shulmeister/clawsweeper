## Summary

Keep open. Current main does not inject `OPENCLAW_SESSION_KEY` or `OPENCLAW_AGENT_ID` into exec child process environments. The exec tool has access to both session and agent context, but the environment path only marks children with `OPENCLAW_CLI=1` and `OPENCLAW_SHELL=exec`, and repo-wide searches found no implementation or docs for the requested env var names.

## Best Possible Solution

Keep #66705 open for a focused core change. The likely fix belongs in the exec environment construction boundary: inject `OPENCLAW_SESSION_KEY` from trusted `defaults.sessionKey` and `OPENCLAW_AGENT_ID` from trusted `agentId` after normal env merging/sanitization, document the new runtime markers, and add regression tests proving host and sandbox/PTY child processes receive the values without weakening existing env filtering.
