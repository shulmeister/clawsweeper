## Summary

Keep #71330 open. Current main still hard-codes short-term/deep promotion output to workspace `MEMORY.md`; there is no `promotionTarget` or equivalent memory-core config surface, docs still describe `MEMORY.md` as the only durable promotion destination, and tests assert the same behavior. The issue is recent, authored by a non-maintainer, has no protected labels, and no related item in the provided context tracks this exact remaining configurable promotion-target work.

## Best Possible Solution

Keep the issue open and implement a memory-core-owned configurable promotion target. The likely path is a strict manifest/schema addition under `plugins.entries["memory-core"].config.dreaming` or the deep-promotion subsection, defaulting to `MEMORY.md`, with safe workspace-relative date templating, shared use by manual `openclaw memory promote --apply` and managed dreaming, automatic creation/header behavior, docs updates, and tests proving the target remains indexed/searchable for both built-in and QMD memory backends.
