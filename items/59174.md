## Summary

Keep open. Current `main` still treats a quoted WhatsApp image as reply metadata text only, not as downloadable quoted-media input, so the reported regression remains plausible and is not implemented on `main`.

## Evidence

- **Quoted reply extraction falls back to a media placeholder:** `describeReplyContext` unwraps `contextInfo.quotedMessage`, extracts text/location, and otherwise substitutes `extractMediaPlaceholder(quoted)`. That produces strings like `<media:image>` for quoted media instead of attaching the referenced image itself.
  - file: [extensions/whatsapp/src/inbound/extract.ts:412](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/whatsapp/src/inbound/extract.ts#L412)
  - command: `nl -ba extensions/whatsapp/src/inbound/extract.ts | sed -n '408,440p'`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Quoted media placeholder is string-only:** `extractMediaPlaceholder` returns fixed tokens such as `<media:image>` for image messages.
  - file: [extensions/whatsapp/src/inbound/extract.ts:271](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/whatsapp/src/inbound/extract.ts#L271)
  - command: `nl -ba extensions/whatsapp/src/inbound/extract.ts | sed -n '271,287p'`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Only the current inbound message downloads media:** `enrichInboundMessage` computes `replyContext` first, then calls `downloadInboundMedia(msg, sock)` only for the outer inbound message and returns top-level `mediaPath/mediaType/mediaFileName`. There is no corresponding download/save path for quoted media.
  - file: [extensions/whatsapp/src/inbound/monitor.ts:483](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/whatsapp/src/inbound/monitor.ts#L483)
  - command: `nl -ba extensions/whatsapp/src/inbound/monitor.ts | sed -n '470,545p'`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Inbound contract has no quoted-media attachment field:** `WebInboundMessage` carries `replyToBody`/`replyToSender` plus top-level `mediaPath/mediaType/mediaFileName`, but no quoted-media path/url field that would let the model inspect a referenced image.
  - file: [extensions/whatsapp/src/inbound/types.ts:69](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/whatsapp/src/inbound/types.ts#L69)

I did not find a current-main test proving quoted-image bytes are surfaced to the model.
