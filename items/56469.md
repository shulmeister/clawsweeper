## Summary

Keep open. Current `main` supports account-level `typingIndicator` overrides for Feishu, but it does not implement the requested per-device or per-platform behavior, and the inbound Feishu message contract/parser do not carry device metadata that would drive desktop-vs-mobile selection.

## Evidence

- **Feishu config only models `typingIndicator` as a boolean, including account overrides:** The shared Feishu config shape defines `typingIndicator` as `z.boolean().optional()`, and the top-level schema defaults it to `true`. There is no object shape for device/platform-specific typing settings.
  - file: [extensions/feishu/src/config-schema.ts:184](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/feishu/src/config-schema.ts#L184)
  - command: `nl -ba extensions/feishu/src/config-schema.ts | sed -n '156,236p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Runtime typing behavior is a single boolean gate per resolved account:** `createFeishuReplyDispatcher` resolves the Feishu account once and skips typing only when `account.config.typingIndicator` is false. There is no branch on message device, client platform, or similar inbound metadata.
  - file: [extensions/feishu/src/reply-dispatcher.ts:143](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/feishu/src/reply-dispatcher.ts#L143)
  - command: `nl -ba extensions/feishu/src/reply-dispatcher.ts | sed -n '143,181p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Current tests cover account-level typingIndicator, not per-device behavior:** Feishu schema tests accept account-level optimization flags, and reply-dispatcher tests verify typing is skipped when an account-level `typingIndicator` is disabled. That is the only granularity currently exercised.
  - file: [extensions/feishu/src/config-schema.test.ts:202](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/feishu/src/config-schema.test.ts#L202)
  - command: `nl -ba extensions/feishu/src/config-schema.test.ts | sed -n '202,220p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Inbound Feishu event/model does not expose device or platform fields:** `FeishuMessageEvent` includes sender ids, chat ids, chat type, message type, content, timestamps, and mentions, but no device/platform field. `parseFeishuMessageEvent` maps that payload into `FeishuMessageContext` without any device metadata, so current main has no input to distinguish desktop from mobile requests.
  - file: [extensions/feishu/src/event-types.ts:1](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/feishu/src/event-types.ts#L1)
  - command: `nl -ba extensions/feishu/src/event-types.ts | sed -n '1,32p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **User-facing docs describe only a boolean typingIndicator plus multiple accounts:** The Feishu docs document `typingIndicator` as a simple true/false quota-optimization flag and separately document multiple accounts. They do not describe per-device typing settings or automatic device-based routing.
  - file: [docs/channels/feishu.md:204](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/channels/feishu.md#L204)
  - command: `nl -ba docs/channels/feishu.md | sed -n '204,269p'`

## Risks / Open Questions

- The issue body explicitly mentions per-account configuration as an alternative, and current main does support account-level `typingIndicator` overrides. A maintainer could still decide that existing multi-account support is an acceptable workaround, but it does not satisfy the stated per-device behavior.
