## Summary

Keep open. Current `main` does not implement this PR’s behavior change: WhatsApp still defaults to `maxAttempts: 12`, the connection controller still returns `action: "stop"` when that limit is reached, and the monitor still shuts down the loop with a "Stopping web monitoring" message. The stale-status/silent-failure aspect appears improved because status surfaces now report `healthState: "stopped"`, but the reconnect-loop change itself is not on `main`.
