## Summary

Keep open. Current main still does not expose a `tools` parameter on `sessions_spawn`; the current behavior remains config/session-policy based, matching the prior ClawSweeper comment rather than superseding it.

## Best Possible Solution

Keep tracking this issue until maintainers either implement a spawn-time `tools` policy through the `sessions_spawn` schema, `SpawnSubagentParams`, child session metadata, and tool-policy pipeline, or explicitly decide that dynamic per-spawn tool policies should remain unsupported in favor of predefined agent/config profiles.
