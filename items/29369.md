## Summary

Keep #29369 open. Current main still does not expose or honor a spawn-time `tools` allowlist on `sessions_spawn`; the schema, executor, native spawn params, ACP spawn params, docs, and policy pipeline all continue to route tool policy through configured/session-scoped policy rather than a per-call parameter.

## Best Possible Solution

Keep #29369 open until maintainers design and implement spawn-time tool policy for `sessions_spawn`. The concrete path is to add schema/type support for a narrow `tools` policy object, validate it, propagate it through both native and ACP spawn paths, merge it into the effective policy pipeline as a final narrowing step, document precedence against existing `tools.allow`, `agents.list[].tools`, group, sandbox, and `tools.subagents.tools` policy, and add tests for deny-wins behavior and both runtimes.
