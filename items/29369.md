## Summary

Keep #29369 open. Current main still does not expose or honor a spawn-time `tools` allowlist on `sessions_spawn`; the existing behavior only supports configured tool policy through profiles/global/agent/group/sandbox/subagent config, not per-call tool policy.

## Best Possible Solution

Keep #29369 open until maintainers design and implement a spawn-time tool policy through the `sessions_spawn` schema, native and ACP spawn parameter types, child-session/run metadata as needed, and the effective tool-policy pipeline. The design should document precedence against `tools.allow`, `agents.list[].tools`, provider/group/sandbox policy, `tools.subagents.tools`, and deny-wins behavior, with tests for both native and ACP-backed spawns.
