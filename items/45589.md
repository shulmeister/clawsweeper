## Summary

Current main addresses the reported compound outage well enough to close: Discord message intake no longer waits on the agent turn, LLM no-token hangs now have a 120s idle watchdog, Discord inbound runs have their own worker timeout path, and model fallback iterates configured fallback candidates/providers instead of stopping at the first failed candidate. Remaining narrower edge cases are already tracked separately in the linked follow-ups.
