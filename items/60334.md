## Summary

Keep #60334 open. Current main has a doctor path for dropping unknown config keys and normalizing legacy cron job shapes, but the reported cron failure is not fixed: malformed `cron/jobs.json` still throws `Failed to parse cron store` and there is no automatic quarantine or backup restore on load. The related #60799 remains open and only partially overlaps because it tracks valid plain-array cron stores being ignored/clobbered, not the malformed-store recovery plus strict config-upgrade report here.
