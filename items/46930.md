## Summary

Keep open. Current main still emits the exact multiple-gateway advisory through `defaultRuntime.error(...)` in the daemon status human-output path, including the recommendation text linked to `/gateway#multiple-gateways-same-host`. The docs also describe multiple gateways as supported when isolated, so the requested severity downgrade remains a plausible product/UX bug rather than something already implemented or superseded. The only provided timeline cross-reference is to #46938, which is about memory/session timeout behavior and does not duplicate or supersede this issue.
