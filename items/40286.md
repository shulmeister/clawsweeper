## Summary

Keep open. Current main handles normal Slack `message` events, including `file_share` subtypes and file-download fallback placeholders, but it does not register or handle Slack `file_shared` fallback events. The PR is stale and needs rebasing into `extensions/slack`, tests, and docs updates, but the remaining capability is not implemented or superseded on main.

## Best Possible Solution

Keep the PR open for maintainer follow-up. The best path is to rebase the concept into the current Slack plugin, register `file_shared` alongside existing interaction/message handlers, hydrate the matching message safely, feed it through the normal message pipeline with explicit dedupe semantics, add order/race tests for `message` versus `file_shared`, and update Slack setup docs if the bot event subscription must include `file_shared`.
