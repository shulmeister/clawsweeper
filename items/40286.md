## Summary

Keep open. Current main still lacks the PR's central Slack `file_shared` fallback for oversized uploads, so the work is not implemented or superseded. The PR needs a careful rebase from old `src/slack` paths into `extensions/slack`, but the requested behavior remains missing.

## Best Possible Solution

Keep the PR open for maintainer follow-up. Rebase the concept into `extensions/slack`, register `file_shared` alongside the existing Slack event handlers, hydrate the matching message safely through Slack history or file APIs, feed it through the normal message pipeline with explicit source typing and dedupe semantics, preserve interaction and command handling, and update Slack setup docs/manifests plus tests.
