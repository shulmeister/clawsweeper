## Summary

Keep PR #71950 open. Current main still has the sandbox crash path this PR targets: gateway LAN discovery and tailnet discovery call the throwing `readNetworkInterfaces()` helper, and the current regression tests still assert that `uv_interface_addresses` failures propagate. The PR is also tied to open bug #71721 and has explicit maintainer feedback saying the fix direction is right after a follow-up adjustment to preserve status warnings.

## Best Possible Solution

Keep #71950 open and continue review toward landing this PR or an equivalent fix on main. The best implementation is the PR's current direction: route gateway LAN and tailnet discovery through safe/inspection-based network-interface helpers, preserve warning text for status/display surfaces, and keep regression coverage for `pickPrimaryLanIPv4()` and `listTailnetAddresses()` when `os.networkInterfaces()` throws.
