## Summary

Current `main` still does not implement a configurable exec approval timeout. It ships a fixed 30-minute default and documents that default, but the public config types, strict schemas, labels/help, and agent-side runtime plumbing still lack `tools.exec.approvalTimeoutMs`, `approvals.timeoutMs`, or an env-backed override. The prior maintainer/Codex comment on PR #25789 also explicitly said to keep this open, and the related feature issue #51287 remains open while competing PR #51292 was closed unmerged.

## Best Possible Solution

Keep PR #25789 open for maintainer review or deliberately supersede it with canonical work under open issue #51287. A complete implementation should choose the public config key, update exported config types, strict Zod schemas, labels/help/generated schema and user docs, thread the value through exec defaults, gateway/node approval registration, wait/expiry behavior and CLI transport timeout floors, and add focused regression coverage.
