## Summary

Keep open. Current main still has the reported source-of-truth split: `nodes status` reads `node.list`, while unfiltered `nodes list` reads only `node.pair.list`. The gateway’s `node.list` is the effective known-node catalog, and the docs explicitly say `node.pair.*` is a separate store from WS device pairing. PR #65772 is still a plausible focused fix, and its discussion shows the author addressed the earlier review concerns.

## Best Possible Solution

Keep #65772 open and either merge it after maintainer review or adapt the same approach on main: make `nodes list` prefer the effective `node.list` paired-node view, keep pending data from `node.pair.list`, preserve existing JSON pairing metadata, and retain a narrow compatibility fallback for older gateways. After a fix lands, close #65706 and duplicate PRs/issues against the landed implementation.
