## Summary

Keep open. Current `main` still has the source-of-truth mismatch described in the PR: `nodes status` reads effective node state from `node.list`, but unfiltered `nodes list` still renders paired output from `node.pair.list`, so the reported disagreement remains plausible on shipped code. Local test coverage on `main` does not lock the unfiltered `nodes list`/`nodes status` alignment or the `--json` compatibility path, and the PR discussion shows the author responding to substantive review concerns rather than an obsolete or incoherent change.
