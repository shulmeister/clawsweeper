## Summary

Keep open. Current main still has the source-of-truth split this PR is trying to fix: `nodes status` uses the effective `node.list` catalog, while unfiltered `nodes list` renders only `node.pair.list`. The PR remains a plausible implementation candidate and its review comments were addressed in the provided discussion, so this should go through maintainer review rather than be closed by cleanup.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a focused CLI fix that makes unfiltered `openclaw nodes list` prefer the effective `node.list` paired-node view, preserves pending and legacy pairing metadata from `node.pair.list`, keeps filtered modes strict, and adds command-level regression coverage for the mismatch and JSON contract.
