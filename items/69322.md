## Summary

Keep #69322 open. Current main still exposes `doctor.memory.status` only for the default agent and does not document or test a `perAgent` payload. The PR is a focused implementation for the still-missing RPC parity request, but it should address the unresolved review note about secondary-agent manager acquisition failures before merge.

## Best Possible Solution

Keep the PR open and ask for the small follow-up: catch `getActiveMemorySearchManager` failures inside `collectMemoryAgentStatus`, return an `{ agentId, dirty: false, files: 0, chunks: 0, error }` entry for that agent, fix the docs indentation, then rerun the focused doctor test and changed gateway/type lanes before merge.
