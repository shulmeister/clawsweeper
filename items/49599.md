## Summary

Keep open. Current main does not implement custom HTTP headers for CLI gateway WebSocket connections via CLI flags, env vars, or `gateway.remote.headers`.

## Evidence

- **CLI surface lacks header flag:** `GatewayRpcOpts` only includes `url`, `token`, `password`, `timeout`, `expectFinal`, and `json`, and `gatewayCallOpts()` only registers `--url`, `--token`, `--password`, `--timeout`, `--expect-final`, and `--json`.
  - file: [src/cli/gateway-cli/call.ts:7](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/cli/gateway-cli/call.ts#L7)
  - command: `nl -ba src/cli/gateway-cli/call.ts | sed -n '1,120p'`
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Gateway call path has no header plumbing:** `CallGatewayBaseOptions` and `GatewayRemoteSettings` do not define a headers field. `resolveGatewayCallContext()` reads only `OPENCLAW_GATEWAY_URL` for URL override, and `executeGatewayRequestWithScopes()` constructs `GatewayClient` with url/token/password/tlsFingerprint but no headers.
  - file: [src/gateway/call.ts:42](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/gateway/call.ts#L42)

  - file: [src/gateway/client.ts:101](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/gateway/client.ts#L101)
