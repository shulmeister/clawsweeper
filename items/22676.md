## Summary

Keep #22676 open. Current main still has the reported Signal daemon restart race: the Signal daemon handle exposes a synchronous `stop(): void`, the stop implementation only sends SIGTERM, monitor teardown does not await process exit, and config reload still restarts the Signal channel immediately after `stopChannel`. The focused fix is still represented by PR #71863, whose head is separate from current `main`.

## Best Possible Solution

Keep #22676 open until PR #71863 or an equivalent focused fix lands on main. The fix should make Signal daemon shutdown asynchronous, send SIGTERM, wait for `signal-cli` exit, escalate to SIGKILL on timeout, and await that shutdown through monitor/channel reload teardown before starting a replacement Signal channel. Add regression coverage for daemon stop wait/escalation and abort or reload-driven monitor shutdown.
