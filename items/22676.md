## Summary

Current main still contains the Signal daemon restart race described in #22676. The Signal daemon stop handle is synchronous and only sends SIGTERM, Signal monitor teardown does not await daemon exit, and hot reload still stops then immediately starts the Signal channel. The issue should stay open while PR #71863 or an equivalent fix is reviewed and landed.

## Best Possible Solution

Keep #22676 open until Signal daemon shutdown is made asynchronous and awaited through monitor/channel reload teardown: send SIGTERM, wait for process exit, escalate to SIGKILL on timeout, and only then allow restart. PR #71863 appears to be the active candidate and should be reviewed or replaced with an equivalent focused fix plus regression coverage for daemon stop semantics and aborted/reload shutdown.
