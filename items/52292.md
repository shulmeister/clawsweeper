## Close Comment

Closing as implemented on main after a Codex review. Current OpenClaw no longer ships the bundled `nano-banana-pro` wrapper: `CHANGELOG.md:2212`-`2213` records that the stock path moved to the core `image_generate` tool in `v2026.3.22`, `docs/tools/skills-config.md:41` now directs built-in image generation to `agents.defaults.imageGenerationModel` + `image_generate`, and the current tool/docs explicitly say generated images are delivered automatically (`src/agents/tools/image-generate-tool.ts:549`, `docs/tools/image-generation.md:10`). Current compatibility coverage also migrates and removes legacy `skills.entries.nano-banana-pro` config (`src/commands/doctor-legacy-config.migrations.test.ts:190`). If someone is still running a separately installed copy of the old skill, that would need to be fixed in that third-party skill rather than in current OpenClaw main.
