## Summary

Keep #70680 open. Current main still classifies any non-archived `.jsonl` file that is not `sessions.json` or a compaction checkpoint as a primary session transcript, while trajectory capture currently writes `<session>.trajectory.jsonl` beside session transcripts. `doctor` builds orphan candidates from that primary-transcript helper, so trajectory sidecars still enter the orphan scan and are not cross-checked through trajectory pointer files. The linked PR #70812 is the right narrow fix direction but is still open and not present in the checked-out main SHA, so the original issue remains the canonical tracker.

## Best Possible Solution

Keep #70680 open as the canonical bug until PR #70812 or an equivalent fix lands. The best implementation is to make session artifact classification distinguish primary transcripts from structured sidecars such as `*.trajectory.jsonl`, add regression coverage for the doctor orphan scan, and review other cleanup paths that consume `isPrimarySessionTranscriptFileName()`.
