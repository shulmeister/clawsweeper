## Summary

Keep #70680 open. Current main still matches the reported failure mode: runtime trajectory sidecars are named `*.trajectory.jsonl`, the primary-transcript classifier still accepts any non-archived `.jsonl` except compaction checkpoints, and doctor still feeds that classifier into orphan-transcript cleanup. The related fix PR #70812 exists and is still open, so the issue should remain as the canonical tracker until that fix or an equivalent lands.

## Best Possible Solution

Keep the issue open as the canonical bug. Land #70812 or an equivalent narrow fix that excludes `*.trajectory.jsonl` from primary transcript classification, adds direct helper coverage, and adds a doctor regression test proving a real orphan transcript is still reported while a neighboring trajectory sidecar is ignored.
