## Summary

Keep PR #68197 open. Current main still has the exact LINE behavior the PR targets: group mention gating only treats text messages as mention-detectable, the strict LINE group config schema does not accept `requireMentionForNonText`, and the current test suite explicitly preserves non-text group messages bypassing `requireMention`. The PR is a focused bundled LINE plugin change, not a ClawHub cleanup candidate.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a backward-compatible LINE plugin option that preserves the current default, adds targeted tests for non-text group messages under `requireMention`, and updates any public/documented LINE config surfaces that should expose `requireMentionForNonText`.
