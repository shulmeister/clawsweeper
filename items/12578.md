## Summary

Keep open. Current main still only supports session-scoped `responseUsage`; there is no `agents.defaults.usageDefault`, `responseUsageDefault`, or equivalent inherited usage-footer default in config, docs, or the reply/TUI paths. Related PRs #48635 and #69618 explicitly track possible fixes, but neither behavior is present in the reviewed main SHA.

## Best Possible Solution

Keep the issue open until a maintainer accepts and lands an inherited usage-footer default. A good fix would add the chosen config key to agent defaults schema/types/docs, apply it in the reply runner when the session has no explicit `responseUsage`, and preserve `/usage off|tokens|full` as a session-level override.
