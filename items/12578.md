## Summary

Keep #12578 open. Current main still does not implement a persistent default for per-response usage footers: `agents.defaults` has no `usageDefault`, `responseUsageDefault`, or `responseUsage` config key; reply formatting reads only session-local `responseUsage`; `/usage`, gateway patching, and TUI controls still set or clear the per-session field; and user docs still describe `/usage` as per-session persistence. The two related PRs are useful context, but both are open/unmerged in the provided GitHub context, and the maintainer comment already correctly kept this issue open.

## Best Possible Solution

Keep #12578 open until a focused core config change lands. Maintainers should choose one public key under `agents.defaults` such as `responseUsage` or `responseUsageDefault`, add schema/types/generated help/docs, resolve usage mode from explicit session override first and inherited default second, preserve explicit `off` versus clear-to-inherit semantics, and add regression coverage for config parsing, reply footer inheritance, `/usage` overrides, and gateway/TUI patch behavior.
