## Summary

Current main still does not implement a persistent default for per-response usage footers. The reply runner resolves footer mode only from session `responseUsage`, `/usage` writes/deletes only that session field, session construction preserves only an existing session value, and the agent defaults schema/types/docs still lack `usageDefault`, `responseUsageDefault`, or `agents.defaults.responseUsage`. Related PRs #48635 and #69618 are relevant possible implementations, but the provided context says they remain open and the behavior is not present on current main.

## Best Possible Solution

Keep #12578 open until a maintainer chooses and lands an inherited usage-footer default. A focused fix should add the chosen key, likely `agents.defaults.responseUsage` or `agents.defaults.responseUsageDefault`, to agent defaults schema/types/generated help/docs; apply it in the reply runner only when the session has no explicit `responseUsage`; preserve `/usage off|tokens|full` as the session-level override; and add config/runner/slash-command tests for inheritance, explicit off, and clear-to-inherit semantics.
