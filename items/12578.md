## Summary

Keep #12578 open. Current main still has no persistent usage-footer default under `agents.defaults`; response usage remains a session-scoped field controlled by `/usage`, and the related implementation PRs are open/unmerged per the provided GitHub context.

## Best Possible Solution

Keep #12578 open until a focused core config implementation lands. The maintainer path should choose one public key, add schema/types/generated help/docs, resolve response usage from session override first and `agents.defaults` second, preserve clear-to-inherit semantics, and add regression tests for inherited default, explicit off, and session override behavior.
