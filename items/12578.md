## Summary

Current main still does not implement a persistent default for per-response usage footers. The implementation resolves usage footer mode from session `responseUsage` only, `/usage` still mutates only the session entry, session construction preserves only an existing session value, and the agent defaults schema/types/docs do not expose `usageDefault`, `responseUsageDefault`, or `agents.defaults.responseUsage`. Related PRs #48635 and #69618 are relevant possible implementations, but they remain unmerged per the provided context and are not current-main behavior.

## Best Possible Solution

Keep #12578 open until a focused implementation lands. The likely product path is to add a chosen inherited key such as `agents.defaults.responseUsage` or `agents.defaults.responseUsageDefault` to config schema/types/generated help/docs, apply it in the reply runner only when a session has no explicit `responseUsage`, preserve `/usage off|tokens|full` as a session override, and add tests for inheritance, explicit off, and clear-to-inherit behavior.
