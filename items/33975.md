## Summary

Keep #33975 open. Current main has a usable model-attribution workaround through `messages.responsePrefix` template variables such as `{modelFull}`, but the issue also requests an opt-in approval gate before switching to a fallback model. I found no config/schema field, runtime type, fallback-loop hook, or test coverage for that remaining approval behavior.

## Best Possible Solution

Keep the issue open until OpenClaw either implements an opt-in fallback approval mode in the core model fallback path, with schema/types/docs/tests, or a maintainer explicitly splits the shipped response-prefix attribution work from the still-missing approval behavior into a narrower follow-up.
