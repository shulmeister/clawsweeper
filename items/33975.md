## Summary

Keep this issue open. Current main covers the model-attribution half through `messages.responsePrefix` templates, but the requested opt-in approval gate before model fallback execution is still not implemented, and the latest discussion confirms that remaining scope.

## Best Possible Solution

Keep this open, or split it so this issue becomes the canonical fallback-approval tracker. Treat `messages.responsePrefix` with `{modelFull}` and `{provider}` as the shipped attribution solution, then implement the remaining feature as an opt-in per-agent fallback approval policy in the core model fallback path with schema/types, docs, and tests for approval, timeout, fail-fast/proceed, cancellation, and channel delivery behavior.
