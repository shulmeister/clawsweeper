## Summary

Keep #33975 open. Current main covers the model-attribution half through `messages.responsePrefix` template variables, but it still has no opt-in approval gate before automatically advancing to configured fallback models.

## Best Possible Solution

Keep this issue open until OpenClaw either implements an opt-in fallback approval mode in the core model fallback path, including config schema/types/docs/tests and channel-appropriate approval UX, or a maintainer explicitly splits that remaining behavior into a narrower canonical tracker. Keep using `messages.responsePrefix` with `{modelFull}`/`{provider}` as the current model-attribution answer unless maintainers choose a dedicated config surface.
