## Summary

Current main already contains the Control UI/webchat protections that address disappearing assistant replies: oversized markdown now falls back to sanitized plain-text rendering instead of disappearing, and chat final-message handling preserves assistant replies even when the final payload is sparse or text-only. The shipped `v2026.4.22` tag contains these code paths and regression coverage.

## Close Comment

Closing from a Codex review: current `main` already includes the webchat/control-ui protections needed for this report, and the shipped `v2026.4.22` release contains them as well. Long replies no longer depend on the markdown parser succeeding for the whole payload: `ui/src/ui/markdown.ts:489` switches oversized content to a sanitized plain-text fallback, with regression coverage in `ui/src/ui/markdown.test.ts:470`. The chat controller also preserves assistant replies on final events by accepting text-only assistant payloads and falling back to streamed text when needed (`ui/src/ui/controllers/chat.ts:252` and `ui/src/ui/controllers/chat.ts:432`). That means the March 2026 beta behavior described here is already fixed on current OpenClaw. If you still see disappearing replies on a newer build, please open a fresh issue with the exact version, a sample failing reply, and browser console/network logs.
