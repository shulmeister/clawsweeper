## Summary

Keep open. Current main still has no generic user-visible feedback hook for messages that enter the active-session `enqueue-followup` path. Telegram, Discord, and Slack have initial status-reaction behavior on receipt, and Control UI has its own queued-message UI, but Feishu/Lark still routes feedback through reply-start typing callbacks and the shared enqueue branch returns silently. The prior PR #39676 is closed unmerged; PR #39117 appears to be a related open fix vehicle, but it is not merged into main.

## Best Possible Solution

Keep the issue open. The best fix is a generic, non-blocking queued-message feedback seam on the active-run `enqueue-followup` path, likely an `onQueued` callback in `GetReplyOptions` or equivalent typed channel callback. Invoke it before typing cleanup, swallow/log callback failures so enqueueing remains reliable, wire Feishu/Lark and other shared-dispatcher channels to typing or queue reactions, and add focused tests proving the callback fires only for queued followups and does not disturb existing Telegram/Discord/Slack status reactions or normal reply-start typing.
