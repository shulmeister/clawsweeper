## Summary

Current `main` already covers the reported origin-allowlist problem. The gateway now accepts `gateway.controlUi.allowedOrigins: ["*"]` as an intentional allow-all policy, preserves existing configured origins instead of reseeding over them, documents the exact non-loopback Control UI requirements, and ships tests for both wildcard acceptance and no-overwrite migration behavior. The current released runtime (`v2026.4.22`) matches `main` for the relevant gateway code paths.
