## Summary

Keep PR #62449 open. Current main still writes through the originally resolved `settings.file` inside the file transport, while the default path is date-dependent, so the stale date-rolled log-file bug described by #62381 remains plausibly unfixed on main. The PR discussion and bot review also indicate the PR targets a real, narrowly scoped logging rollover bug rather than obsolete work.

## Evidence
