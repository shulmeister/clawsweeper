## Summary

Current OpenClaw main already supports cron `sessionTarget` values of `current` and `session:*`, preserves custom session ids during normalization/load, does not rewrite the cron store on load, and executes manual runs against the requested session key.

## Close Comment

Closing as `implemented_on_main` based on a Codex review. Current `main` already accepts `sessionTarget: "current"` and `sessionTarget: "session:..."` in the cron schema (`src/gateway/protocol/schema/cron.ts:21`), preserves custom `session:*` ids during normalization (`src/cron/normalize.ts:319`), and the cron store load path reads jobs without rewriting the persisted store on startup (`src/cron/service/store.ts:36`; covered by `src/cron/service/store.test.ts:44`). Manual runs also pass the custom session key through to the isolated cron runner (`src/gateway/server-cron.ts:333`, covered by `src/gateway/server-cron.test.ts:292`). In other words, the reported reset to `"isolated"` does not match current OpenClaw behavior. If this still reproduces on a current build, please open a new issue with the exact `jobs.json` entry and gateway logs from that run.
