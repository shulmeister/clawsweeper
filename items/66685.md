## Summary

Keep PR #66685 open. Current `main` still logs every `sendExecApprovalFollowupResult` dispatch error after deduplication, and the PR’s specific approval-not-found suppression plus regression test are not present. The PR is focused, aligns with the existing `isApprovalNotFoundError` helper, and the provided discussion indicates the red CI/parity signals were unrelated to this two-file change rather than evidence that the fix is obsolete.

## Best Possible Solution

Keep the PR open and either merge or rebase/adopt its narrow implementation: import `isApprovalNotFoundError` in `src/agents/bash-tools.exec-host-shared.ts`, return silently for approval-not-found followup dispatch failures, and keep warning logs for real dispatch failures with the added regression test in `src/agents/bash-tools.exec-host-shared.test.ts`.
