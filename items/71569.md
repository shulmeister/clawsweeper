## Summary

Keep open. Current main still has a real Mattermost docs/config/runtime mismatch: the docs advertise `channels.mattermost.streaming` modes, but the Mattermost schema, public config type, and generated config metadata only expose the legacy `blockStreaming` fields. The normal Mattermost inbound path also creates and updates a draft preview unconditionally, so `streaming: "off"` / `progress` are not implemented as documented. Related issue #70253 overlaps on the ignored streaming config symptom, but it is broader and does not supersede this focused Mattermost streaming/notification UX report.

## Best Possible Solution

Implement the documented Mattermost preview streaming contract in the bundled Mattermost plugin: add structured `streaming` support to the Mattermost schema/types/generated metadata and any doctor compatibility path, resolve `off | partial | block | progress` in the Mattermost runtime, skip draft preview creation when mode is `off`, implement `progress` so the final answer is delivered as a normal notifying post, and add focused config/runtime tests for top-level and per-account Mattermost streaming settings. Keep #71569 open as the focused tracker; reference #70253 only for the overlapping ignored-config symptom.
