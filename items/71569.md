## Summary

Keep open. Current main still has the documented Mattermost preview-streaming contract mismatch: docs advertise `channels.mattermost.streaming` with `off | partial | block | progress`, but the Mattermost schema/type/generated metadata do not expose that key, and the inbound runtime still creates a draft preview stream without resolving preview modes. Current main does appear to improve the visual-spam part by using same-post draft updates/finalization, but the config key and notification-oriented `off`/`progress` behavior remain unimplemented.

## Best Possible Solution

Keep this issue open as the focused tracker. Implement the documented Mattermost preview-streaming contract in the bundled Mattermost plugin: add `streaming` schema/types/generated metadata and doctor/compat handling for root and per-account config; resolve `off | partial | block | progress` in the Mattermost runtime; skip draft preview creation for `off`; make `progress` deliver the final answer as a normal notifying post; and add focused config/runtime tests. Reference #70253 only for the overlapping ignored-config symptom.
