## Summary

Keep #71569 open. Current main still has a real Mattermost streaming config/runtime mismatch: docs describe `channels.<channel>.streaming` modes for Mattermost, but the Mattermost config schema/types/generated metadata do not accept a `streaming` key, and the inbound monitor creates draft-preview streaming unconditionally instead of honoring `streaming.mode` or a final-notification/silent-draft option. The provided #71569 body/comment and the closed #71565 context remain relevant rather than obsolete.

## Best Possible Solution

Keep the issue open and fix it in the Mattermost plugin surface: add the documented `streaming` config object/scalar migration support to `extensions/mattermost/src/config-schema-core.ts`, `extensions/mattermost/src/types.ts`, generated metadata/docs as needed, and the Mattermost account/runtime resolver; then make `monitor.ts` honor `streaming.mode` (`off`, `partial`, `block`, `progress`) and add focused schema/runtime tests covering accepted config, preview disabled behavior, and whichever final-notification or final-post strategy maintainers choose.
