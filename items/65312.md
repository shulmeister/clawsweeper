## Summary

Keep open. Current main enforces and tests the 2 MB local avatar cap, but the user-facing CLI and gateway config docs still do not mention the limit. There are also open PRs (#65675 and #72172) that explicitly propose the narrow docs fix for this issue, so the best maintainer path is to review/land one of them rather than close the issue first.

## Best Possible Solution

Keep the issue open while maintainers review the active docs PRs. The narrow fix should document that local workspace avatar image files must be at most 2 MB in `docs/cli/agents.md` and `docs/gateway/config-agents.md`, matching `AVATAR_MAX_BYTES`; then duplicate or stale PR attempts can be closed after one implementation lands.
