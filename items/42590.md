## Summary

Current `main` already supersedes this PR: the `nodes run` path it patches no longer exists, `system.run` is intentionally blocked from `nodes invoke`, and node shell execution now goes through `exec host=node`, which derives and applies a longer gateway timeout for `node.invoke` calls.

## Close Comment

Closing after a Codex review because current `main` already supersedes this PR. The `nodes run` wrapper this patch targets is no longer present (`src/agents/tools/nodes-tool.ts:19`, `src/cli/nodes-cli.coverage.test.ts:110`), and `system.run` is explicitly blocked from `nodes invoke` in favor of `exec host=node` (`src/agents/tools/nodes-tool-commands.ts:120`, `docs/cli/nodes.md:62`). The active node shell path now lives in `src/agents/bash-tools.exec-host-node.ts:202` and `src/agents/bash-tools.exec-host-node.ts:423`, where the gateway timeout is derived from the requested exec timeout and applied to the `node.invoke` call. I also checked the `v2026.4.22` release tree and it already has the same structure, so there is no remaining `main`-branch change for this PR to land.
