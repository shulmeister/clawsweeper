## Summary

Keep this issue open. Current main still lacks a global, per-surface, or per-channel incomplete-turn delivery policy for genuine empty turns; direct/TTS-capable surfaces can still receive the visible "Agent couldn't generate a response" fallback.

## Best Possible Solution

Keep this issue open and implement a first-class incomplete-turn delivery policy in the embedded-runner/reply-run configuration path. It should default to current visible error behavior, support global and per-surface or per-channel silent overrides for direct/TTS surfaces such as Signal, preserve WARN/server logging, and add regression coverage for direct chats, groups, genuine empty turns, reasoning-only turns, message-tool side effects, and existing `NO_REPLY` semantics.
