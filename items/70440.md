## Summary

Keep #70440 open. Current main has related mitigations for #70396/#70623 and existing silent-reply policy, but it still lacks the requested global/per-channel server-side policy to silently swallow genuine empty-turn fallback responses, especially for direct Signal/TTS delivery. A maintainer comment on the issue explicitly says this remains a separate valid feature request after the fallback fix.

## Best Possible Solution

Keep the issue open and implement an explicit empty/incomplete-turn delivery policy, likely under agent defaults plus per-surface/channel override. The default should remain user-visible error delivery, while an opt-in silent mode should warn in server logs, return no channel payload, and include focused tests for direct Signal/TTS, normal text channels, committed-message suppression, and existing NO_REPLY behavior.
