## Summary

Keep #38829 open. Current main still has the reported TUI cross-surface realtime gap: the gateway and Control UI have a `session.message` transcript-event path, but `src/tui` still does not subscribe to or handle it. The related #54605 context reinforces that #38829 is the canonical remaining tracker, not something to close as stale or implemented.

## Best Possible Solution

Keep #38829 open. The likely maintainer path is a focused TUI fix: add gateway-client methods for `sessions.messages.subscribe`/unsubscribe, subscribe to the active session on connect and session changes, handle `session.message` for the current session by appending or reloading transcript history while preserving active local stream state, and cover cross-surface delivery plus session filtering in TUI tests. Coordinate with the related #41964 discussion so the fix uses the transcript subscription path instead of widening hidden external-channel chat fanout without an authorization decision.
