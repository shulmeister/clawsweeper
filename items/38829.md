## Summary

Keep #38829 open. Current `main` still has the same product gap described by the issue and the prior ClawSweeper review: the TUI receives live `chat`, `chat.side_result`, and `agent` websocket events, but it does not subscribe to or handle the gateway transcript update path that other surfaces use for cross-surface realtime messages.

## Best Possible Solution

Implement the missing TUI transcript subscription path: subscribe to the active session on connect and when the selected session changes, unsubscribe old sessions as needed, handle `session.message` by reloading or appending history for the current session while preserving active local stream/optimistic-message behavior, and add focused TUI tests for cross-surface message delivery and session filtering.
