## Summary

Keep #38829 open. Current main still lacks a TUI path to subscribe to or handle gateway transcript events, so messages appended from another surface can remain invisible until the TUI reloads history. The related #41964 is still open and narrower; it does not add the missing TUI transcript subscription/update path.

## Best Possible Solution

Keep #38829 open. A focused fix should add subscribe/unsubscribe methods to the TUI backend and gateway client, subscribe to transcript events for the active session after connect and session switches, handle `session.message` for the active session by reloading or appending history without disrupting an active local stream, ignore other sessions, clean up stale subscriptions on switch/stop, and add TUI regression tests for cross-surface delivery, active-run deferral, session switching/unsubscribe, and session filtering.
