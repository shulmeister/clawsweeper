## Summary

Keep #38829 open. Current main still does not implement the TUI side of live cross-surface transcript updates: the Gateway has `sessions.messages.subscribe` and emits `session.message`, and Control UI consumes that path, but `src/tui` still only routes `chat`, `chat.side_result`, and `agent` events and has no transcript-message subscription or handler.

## Best Possible Solution

Keep #38829 open. The best fix is a focused TUI change: add `TuiBackend`/`GatewayChatClient` subscribe and unsubscribe methods for `sessions.messages.subscribe`, subscribe to the active session on connect and session changes, handle `session.message` for the current session by appending or reloading history while preserving active local stream state, ignore other-session events, and add TUI tests for cross-surface delivery and session filtering. Coordinate with #41964 so any chat-event fanout change complements, rather than replaces, the transcript subscription path.
