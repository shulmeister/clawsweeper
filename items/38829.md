## Summary

Keep #38829 open. Current main still has the gateway transcript-event contract and Control UI handling, but the TUI does not subscribe to or handle `session.message` / `sessions.changed`, so cross-surface messages can still require a history reload/re-entry to appear. The related #41964 remains a narrower unresolved external-channel chat fanout PR and does not supersede this transcript subscription gap.

## Best Possible Solution

Keep #38829 open. The best fix is a focused TUI implementation: add `TuiBackend` / `GatewayChatClient` subscribe and unsubscribe methods for `sessions.messages.subscribe` or `sessions.subscribe`, subscribe after gateway connect and when the active session changes, handle `session.message` for the active session by reloading or appending history without disrupting an active local stream, ignore other sessions, clean up stale subscriptions on session switch/stop, and add TUI regression tests for cross-surface delivery, active-run deferral, session switching/unsubscribe, and session filtering. Treat #41964 as related but narrower chat-event fanout work, not the canonical fix for this transcript subscription gap.
