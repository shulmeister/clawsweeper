## Summary

Keep open. Current main still renders Control UI tool-output URLs as escaped/plain text in expanded, collapsed, inline, and raw-details paths, and native WebChat still displays tool results with plain SwiftUI Text. The directly related PR #8880 was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep tracking #8812. The best implementation is a safe URL-aware rendering path for Control UI expanded tool output, raw details, collapsed previews, and inline short cards, with tests for anchors, escaping, unsafe schemes, whitespace preservation, and trailing punctuation. If WebChat remains in scope, add equivalent SwiftUI URL-aware rendering using AttributedString or Link-style handling. PR #8880 can be used as prior art, but it was not merged and should not be treated as shipped.
