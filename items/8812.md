## Summary

Keep open. Current main still does not make URLs clickable in Control UI tool output cards or native WebChat tool-result cards, and the related PR #8880 was closed unmerged rather than shipped.

## Best Possible Solution

Keep #8812 open. Implement a safe URL-aware rendering path for Control UI expanded tool output, raw details/sidebar output, collapsed previews, and inline short cards, with focused tests for anchors, escaping, unsafe schemes, punctuation, and whitespace preservation. If WebChat remains in scope, add equivalent SwiftUI URL-aware rendering using AttributedString or Link handling. PR #8880 is useful prior art but was closed unmerged and should not be treated as shipped.
