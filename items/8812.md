## Summary

Keep open. Current main still does not auto-linkify URLs in Control UI tool output cards or native WebChat tool result cards. PR #8880 was the directly related attempt, but it was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep tracking #8812. The best implementation is a safe URL rendering path for Control UI expanded tool output, collapsed previews, inline short cards, and raw details, with tests for anchors, XSS escaping, unsafe schemes, whitespace preservation, and common URL punctuation. If WebChat remains in scope, add equivalent SwiftUI URL-aware rendering using AttributedString/Link-style handling instead of plain Text. PR #8880 is useful prior art but should not be treated as shipped.
