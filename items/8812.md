## Summary

Keep #8812 open. Current main still renders URL-bearing tool output as escaped/plain text in Control UI tool-card paths and as plain SwiftUI Text in native WebChat. PR #8880 is useful prior art, but the provided GitHub context shows it was closed unmerged, so the feature is not shipped.

## Best Possible Solution

Keep #8812 open. Implement a safe URL-aware renderer for Control UI tool output across expanded output, raw details, collapsed previews, inline short cards, and sidebar/raw views, with focused tests for anchors, escaping, unsafe schemes, punctuation, and whitespace. If native WebChat remains in scope, add equivalent SwiftUI URL rendering via AttributedString/Link handling rather than plain Text. PR #8880 is useful prior art but was not merged.
