## Summary

Current main still leaves the reported Control UI crash path plausible: cron jobs from `cron.list` are accepted into UI state without payload-shape validation, `renderApp` computes cron suggestions on every render before the Chat/Cron tab branch, and that code still dereferences `job.payload.kind` directly. The linked focused fixes #54550 and #54552 were closed unmerged, and no equivalent malformed-cron-payload app-render regression test is present.

## Best Possible Solution

Keep this issue open. The best maintainer path is to revive or reimplement the focused #54552 approach on current main: validate or filter malformed `cron.list` rows at the UI boundary, guard shared Control UI cron render helpers before reading `payload.kind`, and add a jsdom regression test proving malformed cron entries do not prevent switching back to Chat.
