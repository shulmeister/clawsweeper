## Summary

Current `main` already implements automatic Markdown-to-Slack-mrkdwn conversion for Slack outbound text, including normal send and slash-reply delivery paths, with regression coverage and a changelog entry showing it shipped in the `2026.1.13` release.

## Evidence

- **Slack formatter converts Markdown to mrkdwn:** `markdownToSlackMrkdwn` maps shared Markdown IR to Slack markers (`*`, `_`, `~`, code fences, `<url|label>` links) and `normalizeSlackOutboundText` reuses that conversion.
  - file: [extensions/slack/src/format.ts:121](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/extensions/slack/src/format.ts#L121)
  - command: `nl -ba extensions/slack/src/format.ts | sed -n '100,170p'`
- **Outbound Slack sends auto-convert message text:** `sendMessageSlack` resolves Slack chunking/table settings, then runs each outbound Markdown chunk through `markdownToSlackMrkdwnChunks(...)` before posting to Slack.
  - file: [extensions/slack/src/send.ts:359](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/extensions/slack/src/send.ts#L359)
  - command: `nl -ba extensions/slack/src/send.ts | sed -n '356,382p'`
- **Slash replies also auto-convert:** `deliverSlackSlashReplies` applies the same Markdown-to-mrkdwn conversion to reply chunks before responding to Slack slash commands.
  - file: [extensions/slack/src/monitor/replies.ts:193](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/extensions/slack/src/monitor/replies.ts#L193)
  - command: `nl -ba extensions/slack/src/monitor/replies.ts | sed -n '185,216p'`

  - file: [extensions/slack/src/format.test.ts:9](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/extensions/slack/src/format.test.ts#L9)
