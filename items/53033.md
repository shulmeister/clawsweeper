## Summary

Current main already addresses the reported Control UI breakage. The CSP path now hashes inline scripts instead of requiring `unsafe-inline`, and the `resizable-divider` code no longer uses the decorator form that existed in `v2026.3.22` and matched the reported minified-syntax failure area.

## Evidence

## Close Comment

Closing as implemented on main after a Codex review. The two reported causes are already addressed in the shipped code path: the gateway now computes SHA-256 hashes for inline Control UI scripts and applies them to the CSP instead of requiring `unsafe-inline` (`src/gateway/control-ui-csp.ts`, `src/gateway/control-ui.ts`), and the `resizable-divider` component no longer uses the older decorator-based registration that existed in `v2026.3.22` and matched the reported broken minified bundle area (`ui/src/ui/components/resizable-divider.ts`). The released `v2026.4.22` line also includes coverage asserting hashed CSP for `index.html` without `unsafe-inline` (`src/gateway/control-ui.http.test.ts`).
