## Summary

Keep #39923 open. Current main still implements and tests the numbered config backup ring (`.bak`, `.bak.1` through `.bak.4`), while datetime-suffixed config backups have not shipped. The related implementation PRs in the provided context are open or closed unmerged, so they are useful prior art but not a reason to close the issue.

## Best Possible Solution

Keep the issue open until a datetime-suffix implementation lands or maintainers explicitly decide to keep the numbered backup ring. A correct implementation should update `src/config/backup-rotation.ts` and `src/config/config.backup-rotation.test.ts`, with coverage for bounded pruning, chmod discovery, collision handling, and legacy `.bak.N` migration. The open PRs #39982 and #40392 can remain implementation candidates; closed-unmerged PR #46353 is only prior art.
