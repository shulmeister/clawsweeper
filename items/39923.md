## Summary

Keep #39923 open. Current main at e672b61417af5c45b0431df6d9109a1f4b618ef5 still implements the numbered `.bak` / `.bak.1` through `.bak.4` config backup ring, still prunes non-numbered `.bak.*` files, and existing tests assert that behavior. The related PRs #39982 and #40392 are implementation candidates but are not shipped on main; #46353 was closed unmerged.

## Best Possible Solution

Keep #39923 open until a datetime-suffix config backup implementation lands on main or maintainers explicitly decide to retain the numbered backup ring. A complete fix likely belongs in `src/config/backup-rotation.ts` with matching `src/config/config.backup-rotation.test.ts` coverage for bounded datetime pruning, permission discovery, collision handling, and legacy `.bak.N` migration; open PRs #39982 and #40392 can remain implementation candidates.
