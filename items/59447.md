## Summary

Current `main` already avoids the reported embedded-runner failure mode by repairing malformed session transcript lines before reopening the session, so the bad-control-character JSON parse path this PR targets is already handled without adding a new `session_parse_error` surface.

## Evidence

- **Embedded runner now repairs the transcript before reopening it:** `runEmbeddedAttempt` calls `repairSessionFileIfNeeded(...)` immediately before `SessionManager.open(params.sessionFile)`, so malformed JSONL lines are handled before the session reload path that originally threw raw `JSON.parse` errors.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:1039](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/pi-embedded-runner/run/attempt.ts#L1039)
  - command: `nl -ba src/agents/pi-embedded-runner/run/attempt.ts | sed -n '1036,1064p'`
- **Repair logic drops malformed JSONL lines and rewrites a clean backup-backed transcript:** `repairSessionFileIfNeeded` reads the session file, `JSON.parse`s each line, counts malformed lines, and when needed rewrites the file from only valid entries after saving a backup. That directly addresses the PR's reported case of bad control characters or other malformed transcript lines breaking a later reopen.
  - file: [src/agents/session-file-repair.ts:19](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/session-file-repair.ts#L19)
  - command: `nl -ba src/agents/session-file-repair.ts | sed -n '1,140p'`

  - file: [src/agents/session-file-repair.test.ts:37](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/session-file-repair.test.ts#L37)
  - command: `nl -ba src/agents/session-file-repair.test.ts | sed -n '1,140p'`

  - file: [src/agents/pi-embedded-runner/types.ts:95](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/pi-embedded-runner/types.ts#L95)
