## Summary

Keep open. Current main still does not implement the requested configurable session-level circuit breaker for consecutive model errors. Main has useful provider cooldowns, heartbeat duplicate suppression, and timeout-triggered compaction, but there is no persisted per-session breaker state, no `agents.defaults.circuitBreaker` schema/config surface, and no pause/reset/alert integration across heartbeat, cron, and auto-reply paths. The PR is a concrete implementation candidate with tests and follow-up commits, but review comments show it still needs maintainer-quality rework rather than cleanup closure.

## Best Possible Solution

Keep #42933 open for maintainer judgment or rework. The best product path is a structured session-level breaker owned by the agent/session runtime: exported config/schema/docs under `agents.defaults` and per-agent config, persisted session breaker state, structured error metadata from auto-reply/heartbeat/cron instead of text-prefix detection, correct isolated-session ownership, tests for heartbeat/cron/auto-reply paths, and clear behavior for pause/reset/alert. If maintainers do not want this PR's approach, close it only after linking a canonical replacement issue/PR that covers the remaining session-level breaker work.
