## Summary

Keep PR #71141 open. Current main still lacks the requested Telegram reaction-directive path: reaction tags are not parsed into outbound payload metadata, Telegram sendPayload ignores channelData.telegram.reaction, and Telegram delivery still drops empty text/no-media replies before any reaction handling. The PR is a focused, recent bug-fix attempt tied to still-open issue #71140, so it is not obsolete or superseded.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a cleaned, tested version of this patch: parse [[react:...]]/[[react_to_current:...]] in shared reply normalization, preserve the intent as channelData.telegram.reaction, allow reaction-only Telegram replies when a reply target exists, execute reactMessageTelegram before optional text/media delivery, and add normalization plus Telegram adapter/delivery regression coverage. Consider documenting the reaction directive if it is intended as a supported rich-output directive.
