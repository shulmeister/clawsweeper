## Summary

Keep open. Current main still has the main-process plugin text-transform gap this PR targets: runtime text transforms are resolved through an indirect createRequire-based helper, not a direct active-registry lookup. The PR discussion supports the diagnosis, but the patch likely needs revision for current main and cleanup of the dead fallback noted in review.

## Best Possible Solution

Keep this PR open for maintainer review or a revised patch. The best fix is to update the plugin runtime registry/text-transform path on current main to use a direct active-registry accessor that works in the bundled main process, remove or justify the unreachable fallback, and add a regression test proving plugin-registered text transforms are applied through the main runtime prompt/provider path. Related #64921 should remain the bug-tracking anchor until that behavior is fixed and tested.
