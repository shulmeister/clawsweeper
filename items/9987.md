## Summary

Keep #9987 open. Current main has backend support to enumerate pending exec approvals via `exec.approval.list`, plus MCP/channel replay plumbing, but the requested user-facing `openclaw approvals pending` CLI command is still absent from command registration, CLI docs, and CLI tests. The two related implementation PRs, #10013 and #10341, were closed unmerged rather than shipped.

## Best Possible Solution

Keep #9987 open until OpenClaw adds a real `openclaw approvals pending` CLI command in `src/cli/exec-approvals-cli.ts` that calls the existing `exec.approval.list` gateway method, renders a useful table with command details, supports `--json`, and adds focused CLI docs/tests. `--watch` can ship in the same change or be split into a narrower follow-up if maintainers want an incremental implementation.
