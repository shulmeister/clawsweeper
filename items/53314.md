## Summary

Keep open. Current `main` still lacks the PR's distinct pending-descendant cleanup path, so a descendant-gated announce still returns plain `false` and is routed through the normal retry/give-up cleanup logic. That means the reported race can still consume announce retry budget if descendants clear between the inner announce gate and the later cleanup decision.

## Evidence

- **Pending descendants still return plain false:** `runSubagentAnnounceFlow` is still typed as `Promise<boolean>`, and when `countPendingDescendantRuns(...) > 0` it sets `shouldDeleteChildSession = false` and returns `false` rather than a distinct deferred signal.
