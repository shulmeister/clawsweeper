## Summary

Keep open. Current `main` already exposes non-main sessions in Control UI well enough to browse them individually, so the issue body is partly outdated, but the requested feature is broader: chat state, history loading, and sends are still bound to a single `sessionKey`, and there is no merged multi-channel timeline/toggle in the current UI.

## Evidence

- **Control UI exposes sessions as a first-class surface:** The public Control UI docs list both chat and session management as current features, which supports that channel-backed sessions are meant to be visible in the UI rather than hidden outside it.
  - file: [docs/web/control-ui.md:107](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/web/control-ui.md#L107)
  - command: `nl -ba docs/web/control-ui.md | sed -n '104,121p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Gateway lists all stored sessions for Control UI:** `sessions.list` reads the combined gateway session store, and `listSessionsFromStore` maps store entries into UI rows, so Control UI can enumerate non-main sessions instead of being hard-wired to only `main`.
  - file: [src/gateway/server-methods/sessions.ts:579](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/gateway/server-methods/sessions.ts#L579)

- **Session rows carry channel-derived labels:** Gateway session rows derive `channel`, `subject`, `groupChannel`, and `displayName` from stored session metadata, which is the plumbing needed to show channel conversations distinctly in the UI.
  - file: [src/gateway/session-utils.ts:1231](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/gateway/session-utils.ts#L1231)
