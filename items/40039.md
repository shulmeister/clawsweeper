## Summary

Keep #40039 open. Current main still lacks the proposed symptom-first limited-tools troubleshooting section, and the docs gap is real. ## Best Possible Solution

Keep #40039 open and ask for a small revision: include `minimal`, align the status-check wording with current `openclaw status` behavior or explicitly sequence it after #39996, and link detailed profile definitions to the canonical tools/config docs so the troubleshooting page stays compact and accurate.
