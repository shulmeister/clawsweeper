## Summary

Keep #40039 open. Current main still lacks the proposed symptom-first troubleshooting section, so the PR is not obsolete. ## Best Possible Solution

Keep #40039 open as salvageable docs work. Revise the troubleshooting section to include `minimal`, align the diagnosis steps with status behavior currently shipped on main or explicitly wait for #39996, and keep detailed profile definitions linked to the canonical tools/config docs so the troubleshooting page stays compact and accurate.
