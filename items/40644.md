## Summary

Keep #40644 open. Current main still renders the Control UI cron surface as a summary strip, jobs list, run-history list, and edit form; it does not include the requested day/week calendar, 24-hour marker timeline, heatmap, click-detail popup, drag-to-reschedule, or cron-to-UI timeline helper. The linked PR #41892 explicitly tracks this issue, but it is still open/unmerged in the provided GitHub context, so the issue remains useful as the canonical product request until that work lands or is replaced by a narrower maintainer-approved plan.

## Best Possible Solution

Keep #40644 open as the canonical Control UI feature request. Review, shrink, or replace PR #41892 as the implementation path; close the issue only after a merged mainline change ships a maintainer-approved calendar/timeline view, or after maintainers create a narrower canonical tracking issue for the remaining work.
