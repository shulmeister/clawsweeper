## Summary

Keep #70898 open. Current main still rotates silent auto-reply sessions to a new transcript without hydrating recent DM context, so the linked user-visible continuity bug is not implemented on main. The PR is relevant and focused, but it also has later review feedback that should be addressed before merge, especially strict role alternation in replayed tails and bounded transcript reading.

## Best Possible Solution

Keep the PR open and finish it on the PR branch: seed silent `resetReplyRunSession` rotations with a small recent user/assistant transcript tail, keep explicit `/new` and `/reset` fresh, normalize or collapse replayed tails so strict providers see a user-led alternating history, avoid full-file reads with a bounded tail reader or safe size cap, and add regression tests for compaction failure, role-ordering reset, assistant-only/consecutive same-role tails, and pre-existing targets.
