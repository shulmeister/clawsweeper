## Summary

Keep PR #70898 open. Current main still rotates silent auto-reply sessions to a new session id/file without seeding recent transcript context, and the linked #70853 bug remains a concrete Telegram DM continuity report. The PR is relevant and active, but it still needs maintainer review/changes for the late role-ordering and bounded-read concerns before it is a clean landing candidate.

## Best Possible Solution

Keep the PR open and continue it as the implementation vehicle for #70853. The best path is to preserve recent direct-chat continuity during silent compaction/role-ordering resets while normalizing the retained tail through the existing transcript hygiene/turn-validation expectations, avoiding unbounded transcript reads on the reset path, and adding regression coverage for both compaction-failure and role-ordering reset cases.
