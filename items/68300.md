## Summary

Keep open. Current main still logs internal hook handler failures with only the formatted message, so the PR's requested stack trace and structured event context are not implemented. The related #69997 appears to target a likely root cause for #68124, but it is separate open work and does not supersede this diagnostics improvement.

## Best Possible Solution

Keep #68300 open for maintainer review. The best path is to either land this narrow diagnostics change with a focused regression test for `triggerInternalHook` logging metadata, or explicitly fold the observability piece into the canonical #68124/#69997 fix if maintainers prefer one combined change.
