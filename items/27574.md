## Summary

Keep #27574 open. Current main still does not implement a live browser preview panel in the Dashboard/Control UI, and this should not be closed as ClawHub/plugin work yet because the documented and typed plugin API still lacks a Control UI view/panel registration seam. Existing browser support covers CLI/tool/gateway routes for screenshots, snapshots, console/errors, tabs, and actions, but not a real-time dashboard panel showing what the browser sees.

## Best Possible Solution

Continue tracking #27574 until maintainers either implement a core Control UI browser preview backed by the existing browser.request/routes, or first add a supported plugin API for Control UI panel/view contributions and then move the browser-preview implementation into a browser plugin. A good implementation should show current tab URL/title, refreshed screenshots or stream, console/page errors, and clearly scoped operator-only controls with rate limits and auth/origin protections.
