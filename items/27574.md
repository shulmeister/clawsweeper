## Summary

Keep #27574 open. Current main has browser-control backend primitives, screenshots, console/error/request inspection, tab URL/title metadata, and an admin `browser.request` gateway method, but it still lacks the requested Control UI/Dashboard browser preview panel with live screenshots, URL/title display, console errors, and optional interaction. The existing maintainer discussion also explicitly kept this open and treats #63926 as narrower login-handoff/browser-management work, not a full duplicate.

## Best Possible Solution

Keep #27574 open as the broad remote browser-preview/dashboard request. The best maintainer path is either a first-class Control UI browser panel backed by existing `browser.request` and browser-control primitives, or first adding a supported Control UI panel contribution API and then implementing the preview in the browser plugin. Coordinate with open #63926 for login handoff/browser management, but do not treat #63926 as a complete replacement for live remote-debugging preview.

## Risks / Open Questions

- A browser preview/input panel would expose page pixels, URLs/titles, console/page errors, and possibly remote-control actions, so implementation needs explicit operator-admin authorization, origin protections, rate limiting, and clear handoff semantics.
- A plugin-based implementation likely needs a supported Control UI panel/tab contribution API first; current plugin surfaces can expose backend endpoints but do not cleanly add a dashboard view.
