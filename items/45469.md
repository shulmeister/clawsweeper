## Summary

Keep open. Current main still does not implement the requested max retry limit for `GatewayClient.scheduleReconnect()`: reconnects are delayed with exponential backoff capped at 30s, but there is no reconnect counter, max-retry option, or terminal give-up path for ordinary unreachable-gateway close paths. The related PR #46487 proposed this exact cap but was closed unmerged, while merged PR #45689 explicitly scoped itself to request timeouts and left reconnect policy as follow-up work.

## Evidence
