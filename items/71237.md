## Summary

Keep PR #71237 open. Current main has related cleanup paths, but they are partial: stale/capped session entries archive their own removed transcripts, doctor can interactively rename top-level orphan transcripts to `.deleted.*`, and disk-budget cleanup can remove unreferenced transcript files only when a configured budget is exceeded. Current main does not provide the PR's standalone `pruneOrphanedTranscripts` utility or a normal `sessions cleanup` path that directly reclaims arbitrary orphan `.jsonl` files. The PR is recent, focused, and the author addressed review feedback, so it needs maintainer review rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to either merge/iterate the standalone primitive with clear preserved-path requirements, then wire it into `openclaw sessions cleanup` using a union of all live transcript and checkpoint paths, or supersede it with an equivalent mainline implementation that directly reclaims orphan transcripts without relying only on doctor archival or disk-budget pressure.
