## Summary

Keep this PR open. Current main has related session maintenance, doctor orphan detection, and pressure-gated disk-budget cleanup, but it still does not provide the proposed reusable orphan-transcript pruning primitive or an ordinary on-demand `sessions cleanup` path that scans for already-unindexed `.jsonl` files and unlinks them. The PR is recent, scoped, and has active author follow-up on review feedback, so conservative cleanup should leave it for maintainer review.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is either to merge the tested primitive as a staged building block and follow with explicit `openclaw sessions cleanup` wiring, or supersede it with a narrower PR that implements the full cleanup command directly while collecting preserved paths from all relevant stores, `resolveSessionTranscriptCandidates`, topic transcripts, and compaction checkpoint metadata.
