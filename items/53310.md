## Summary

Current main already uses the fixed Feishu schema 2.0 card config for markdown/announcement cards, and the shipped `v2026.4.22` release contains that implementation. The old legacy config that matches the reported `230099 parse card json err` path is no longer present in the bundled Feishu channel.

## Risks / Open Questions

- The report mentions a custom out-of-tree Feishu extension at `~/.openclaw/extensions/feishu/`; if that fork still carries the old card config, the failure can persist there even though `openclaw/openclaw` main is fixed.

## Close Comment

Closing from a Codex review because current `main` already has this fixed in the bundled Feishu channel. The Feishu markdown-card builder now sends schema 2.0 cards with `config.width_mode: "fill"` in `extensions/feishu/src/send.ts:595`, and the regression tests in `extensions/feishu/src/send.test.ts:447` explicitly verify that the legacy `wide_screen_mode` / `enable_forward` fields are absent. The announce path still routes Feishu card-worthy text through that sender in `extensions/feishu/src/outbound.ts:145`, and the released `v2026.4.22` tag contains the same implementation. If this is still reproducible, it is more likely coming from the custom `~/.openclaw/extensions/feishu/` plugin copy mentioned in the report rather than current OpenClaw `main`; please open a fresh issue with that extension revision and a current repro if needed.
