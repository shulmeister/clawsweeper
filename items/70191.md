## Summary

Keep this issue open. Current main still exposes only shallow HTTP liveness/readiness probes and separate CLI/WS/model-list surfaces; it does not provide the proposed auth-gated, no-inference HTTP endpoint that reports the active provider/model/fallback contract plus recent routing/error telemetry. The issue is recent, contributor-authored, concrete, and asks for maintainer API/security direction before a PR.

## Best Possible Solution

Keep this issue open for maintainer product/API direction. The likely implementation belongs in the Gateway as an additive authenticated HTTP probe, either /health/deep or an authenticated extension of readiness details, backed by existing configured model/fallback resolution and new bounded routing/error telemetry hooks. It should include tests for local versus remote auth, unauthorized no-body/no-leak behavior, no-inference operation, error/path redaction, status semantics, and public Gateway health/model documentation.
