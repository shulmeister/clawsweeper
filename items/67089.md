## Summary

Keep open. Current main already enforces `operator.admin` for `config.*` gateway methods through the reserved admin namespace policy and tests, so the PR’s explicit scope-listing portion is mostly obsolete. The central HMAC config-integrity feature is not implemented on main, and repo security docs still list config integrity verification as a recommended hardening item. The PR also has unresolved review blockers, including a reported CI-breaking syntax error and design risks around bootstrapping missing signatures, so it needs maintainer/security rework rather than automated closure.

## Best Possible Solution

Keep the PR open or convert it into a maintainer-owned security hardening track. Preserve the existing reserved `config.*` admin policy on main, but rework the HMAC proposal before merge with an upgrade/bootstrap path for existing configs, non-silent signature write failure semantics, injected homedir/env dependencies, focused tests for load/write/reload/audit behavior, and docs that explain manual-edit recovery.
