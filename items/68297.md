## Summary

Current `main` does not implement a repeatable `--daemon-env` flag for `openclaw node install`. The node install path still only persists `OPENCLAW_ALLOW_INSECURE_PRIVATE_WS` when that variable is already present in the installer process environment, so the feature request remains valid and should stay open.
