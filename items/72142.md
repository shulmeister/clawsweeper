## Summary

Current main already implements the Windows bundled-plugin loader fix behind this Telegram crash. The report is from 2026.4.24; main disables native Jiti loading for bundled plugin dist paths on win32, has direct regression coverage, and the changelog records the Telegram/channel-plugin `ERR_UNSUPPORTED_ESM_URL_SCHEME` fix under 2026.4.25.

## Best Possible Solution

Close this issue as fixed on main and keep the generic bundled-plugin Windows loader behavior. The user-facing path is to pick up a build/release after 2026.4.24; reopen or file a new report only if the same Telegram crash reproduces on a build containing 9089e6b595ba33b4afc27b99c56ace80c4fbbc7b or newer.

## Evidence

  - file: [src/channels/plugins/bundled.ts:187](https://github.com/openclaw/openclaw/blob/9089e6b595ba33b4afc27b99c56ace80c4fbbc7b/src/channels/plugins/bundled.ts#L187)
