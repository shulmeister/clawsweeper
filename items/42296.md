## Summary

Current main already prevents Feishu media API JSON from reaching the agent body. The implementation parses inbound Feishu media messages into placeholders before `BodyForAgent`, `RawBody`, and `CommandBody` are built, while media downloads still use the original Feishu JSON payload. Main also avoids the review concern on this PR because rich-text `post` text is preserved by `parsePostContent()` instead of being overwritten with only media placeholders.

## Best Possible Solution

Close this PR as obsolete and keep the current main implementation, which normalizes Feishu media text at parse time while preserving rich-text post content and still resolving media attachments from the original Feishu payload.
