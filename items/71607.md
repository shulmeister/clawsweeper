## Summary

Current main still matches the reported failure mode: the timer force-reloads the cron store with recomputation skipped, maintenance preserves existing scheduled next-run timestamps, and the reload path does not invalidate nextRunAtMs when a persisted job's schedule changes. PR #71651 directly tracks a proposed fix and regression test, but it is still open and not present in current main, so the issue should stay open until that work lands or maintainers choose a different canonical tracker.
