## Summary

Keep PR #69906 open. Current main still has the WhatsApp shallow merge that lets root `channels.whatsapp.groups` flow into every account, while Telegram already has the `isMultiAccount` guard. The PR is a focused implementation for the still-open maintainer issue #69874 and addresses the Aisle/Greptile review discussion with docs and tests rather than being obsolete or already shipped.

## Best Possible Solution

Keep the PR open for maintainer review or merge after normal validation. The best path is to land the narrow WhatsApp `groups` inheritance guard with the PR's regression tests and migration docs, while leaving any broader fail-closed group-policy change as a separate Telegram-plus-WhatsApp product/security decision.
