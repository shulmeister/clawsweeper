## Summary

Current main already preserves the intended cycle-0 memory-flush dedup behavior, and the PR's final head is behaviorally equivalent to that shipped implementation. The repo intentionally treats missing `compactionCount` as cycle 0 when `memoryFlushCompactionCount` is present, persists that marker only after a flush, and starts fresh sessions with `memoryFlushCompactionCount` unset.

## Best Possible Solution

Close the PR as already implemented on main. Keep the existing cycle-0 dedup semantics, because the current source, tests, and persistence path show that `memoryFlushCompactionCount=0` with a missing `compactionCount` is a valid already-flushed marker rather than a missed-flush signal.
