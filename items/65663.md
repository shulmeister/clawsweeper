## Summary

Keep open. Current `main` still preserves OpenAI/Azure Responses reasoning blocks on same-model tool-call follow-up turns, and the shipped tests/docs only cover orphaned reasoning or model-route-switch cleanup, not the Azure-specific `reasoning -> tool_call` replay failure described in #65663.

## Evidence

- **Current replay path keeps Azure in the shared OpenAI Responses branch:** `sanitizeSessionHistory` treats `azure-openai-responses` the same as `openai-responses`/`openai-codex-responses`. It only asks `downgradeOpenAIReasoningBlocks` to drop replayable reasoning when `modelChanged` is true, so same-model follow-up turns keep the reasoning block.
  - file: [src/agents/pi-embedded-runner/replay-history.ts:460](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/agents/pi-embedded-runner/replay-history.ts#L460)
  - command: `nl -ba src/agents/pi-embedded-runner/replay-history.ts | sed -n '456,515p'`
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **Reasoning followed by a tool call is still considered replay-safe:** `hasFollowingNonThinkingBlock` returns true for any later non-`thinking` block, and `downgradeOpenAIReasoningBlocks` keeps the reasoning block when that check passes. A following `toolCall` therefore preserves the reasoning block instead of dropping it.
