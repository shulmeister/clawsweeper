## Summary

Current main still contains the unsafe support-capture path that this PR targets: `buildSkillsCapture` maps `resolvedSkills` without filtering missing names/paths, then sorts with `left.name.localeCompare`, and `redactPathForSupport` still calls `file.startsWith` on a required string. The PR is recent, focused, contributor-authored, and not protected by a maintainer-only label; it should remain open for maintainer review or an equivalent fix.

## Best Possible Solution

Keep the PR open and either merge it after normal maintainer validation or land an equivalent defensive fix in `src/trajectory/metadata.ts` and `src/logging/diagnostic-support-redaction.ts`, with regression tests for partial `resolvedSkills` entries and non-string path redaction input. Treat any producer-side normalization in `src/agents/skills/workspace.ts` as a separate follow-up, not a reason to close this focused PR.
