## Summary

Keep open. Current `main` still lacks the legacy delivery-queue canonicalization this PR is trying to add, so it is not `implemented_on_main` or `cannot_reproduce`. The PR also appears to be entangled with unrelated gateway changes that still have review-blocking concerns in the provided GitHub context, which argues for refresh/splitting rather than closure.
