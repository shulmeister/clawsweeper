## Summary

Keep this PR open. Current main still has the MCP channel bridge leak pattern this PR targets: pending Claude permissions and pending approvals are raw in-memory Maps, are not cleared by close(), and are only deleted on matched reply/resolved-event paths. The PR is also paired with still-open same-author issue #71646 and remains a focused implementation candidate rather than something conservative cleanup should close.

## Best Possible Solution

Keep this PR open for maintainer review and normal validation. The best path is to land or request revisions on a focused MCP bridge fix that wraps pending entries with stable timestamps, adds a lazy self-terminating TTL sweeper, clears both pending Maps on close, guards post-close writes, and includes regression tests; reconcile the nearby #56420 session-binding work during review/rebase.
