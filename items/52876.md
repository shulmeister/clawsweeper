## Summary

The report matches the 2026.3.22 packaging regression that was already shipped as fixed. Current main still contains the packaging guards for `dist/control-ui` and the public/plugin-loader `openclaw/plugin-sdk` resolution paths needed for installed plugins.

## Evidence

- **v2026.3.23 shipped packaging fixes:** The changelog entry for v2026.3.23 records both the packaged plugin runtime-barrel fix and the npm release guard that blocks publishing when `dist/control-ui/index.html` is missing (`Plugins/runtime barrels` at line 2498 and `Release/npm publish` at line 2513).
  - file: [CHANGELOG.md:2498](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/CHANGELOG.md#L2498)
- **Current prepack blocks missing Control UI artifacts:** `openclaw-prepack` requires `dist/control-ui/index.html`, runs `pnpm build` and `pnpm ui:build`, and exits non-zero if the prepared Control UI bundle is missing.
  - file: [scripts/openclaw-prepack.ts:8](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/scripts/openclaw-prepack.ts#L8)
- **Current release check validates packed tarball contents:** The npm release check requires `dist/control-ui/index.html` in the packed artifact and tracks the Control UI asset prefix, preventing a repeat of the broken 2026.3.22 package.
  - file: [scripts/openclaw-npm-release-check.ts:62](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/scripts/openclaw-npm-release-check.ts#L62)
- **Public plugin SDK root export exists on main:** The package exports `./plugin-sdk` from `dist/plugin-sdk/index.js`, which is the import path from the reported `Cannot find module 'openclaw/plugin-sdk'` failure.
  - file: [package.json:54](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/package.json#L54)
- **User-installed plugin resolution is covered by tests:** Current tests explicitly cover resolving plugin-sdk aliases for user-installed plugins via the running `openclaw` argv hint and `moduleUrl` hint.
  - file: [src/plugins/sdk-alias.test.ts:751](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/plugins/sdk-alias.test.ts#L751)
- **Control UI asset preservation is covered by tests:** Current tests verify rebuild flows preserve `dist/control-ui/index.html` instead of deleting Control UI assets during packaging/build steps.
  - file: [src/infra/run-node.test.ts:267](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/infra/run-node.test.ts#L267)
