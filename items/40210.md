## Summary

Current `main` still appears to reject workspace bootstrap files when the file inside the workspace is a symlink resolving outside the workspace root. The issue has a direct user reproduction plus an additional confirmation comment, and the exact fix is only represented by still-open PR #40230, not shipped on `main`.

## Best Possible Solution

Keep the issue open and use #40230 as the concrete implementation starting point. Maintainers should decide whether session bootstrap context should intentionally allow trusted out-of-workspace symlink targets; if yes, land or rework a targeted fallback in `src/agents/workspace.ts` with regression coverage for out-of-workspace symlinked bootstrap files while preserving hardlink rejection, max-size limits, and the broader boundary-path security behavior.
