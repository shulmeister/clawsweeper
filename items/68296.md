## Summary

Current main no longer matches this PR's premise. The PR only suppresses the read-start diagnostic for `file`/`filePath`, but read execution on main requires canonical `path` before calling the underlying read tool, and committed tests explicitly reject legacy alias parameters. Landing this patch as-is would hide a real invalid-call warning rather than fix a false positive.

## Best Possible Solution

Close this PR as obsolete against current main. Keep any remaining alias-support discussion on the linked #60008-style product/API question, and if accepted, update the read tool schema, REQUIRED_PARAM_GROUPS.read, workspace guard behavior, diagnostics, and regression tests together.
