## Summary

Current main no longer matches the PR premise. The PR only suppresses the read-start diagnostic for `file` and `filePath`, but current runtime validation requires canonical `path` before the underlying read tool executes, and current tests explicitly reject legacy alias parameters. Landing this PR as-is would hide an invalid-call diagnostic rather than fix a false positive.

## Best Possible Solution

Close this PR as obsolete against current main. Keep any remaining alias-support decision on the linked #60008 product/API question, and if accepted, implement it across the read tool schema, `REQUIRED_PARAM_GROUPS.read`, workspace/root guard handling, diagnostics, and regression tests in one coherent change.
