## Summary

Current main has not absorbed PR #66865’s central behavior. `update.run` still treats skipped results as ok, the Control UI only reports failures when `ok === false`, the gateway restart loop still has no update-specific version assertion/status polling path, and there is no `update.status` RPC on main. The PR discussion also includes a real Linux systemd stale-chunk reproduction, so this is not conservative-close material.

## Best Possible Solution

Keep the PR open for maintainer review/rework. The best path is a scoped update contract fix on main: `ok` only for `status: "ok"`, stable reason codes and UI banners for skipped/error outcomes, safe process replacement or explicit non-ok when restart is unavailable, and a post-restart running-version assertion. The author/maintainer should address or split the unresolved review findings before merge.
