## Summary

Keep this PR open. Current main and the latest release still lack the central update invariant: skipped update results are still exposed as successful RPC payloads, the Control UI still only renders feedback for explicit `ok: false`, and update-triggered restarts still use the generic restart path without the PR's update-specific respawn/status verification. The PR is also still an active implementation candidate with recent author follow-up and a live pull ref.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to review, possibly narrow, and land a core fix that makes skipped/error `update.run` outcomes non-ok, surfaces stable reason codes in the Control UI, blocks unsafe global package self-updates before replacing files, and performs update-specific process replacement with health/version verification. After a fix lands, #62492 can close as implemented and the remaining self-update/restart reliability scope in #64892 and #65668 should be reassessed separately.
