## Summary

The request is for an optional enhanced memory plugin and already points to an external plugin repo. OpenClaw main explicitly keeps optional capability in plugins, prefers external plugin distribution/maintenance, and uses ClawHub/npm as the discovery and install path for community plugins. This belongs in the plugin ecosystem rather than as a new OpenClaw core issue.

## Close Comment

Closing as `clawhub` after a Codex review. This request is for an optional enhanced memory plugin and the issue itself points to an external plugin repository rather than a concrete OpenClaw core defect. Current main is explicit that optional capability should usually ship as plugins and be hosted/maintained in the plugin’s own repository (`VISION.md:52`). The supported path for that is ClawHub/npm distribution (`docs/plugins/community.md:9`, `docs/cli/plugins.md:56`), and OpenClaw already treats memory backends as plugin-scoped, including install-on-demand `memory-lancedb` (`docs/tools/plugin.md:109`, `extensions/memory-lancedb/package.json:14`). If you want this LanceDB variant adopted, the right path is publishing/maintaining it as a ClawHub or npm plugin rather than tracking it as OpenClaw core work.
