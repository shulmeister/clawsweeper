## Summary

Keep #31379 open. Current main still renders the Channels page from `channelMeta` or `channelOrder` exclusively, and the backend still builds those arrays from loaded channel plugins only. A custom-only `channels.status` snapshot can therefore still hide the built-in channel cards. The related fix PR #44165 is open/unmerged in the provided GitHub context, while #31350 was closed unmerged.

## Best Possible Solution

Keep #31379 open and resolve it through #44165 or an equivalent focused UI change in `ui/src/ui/views/channels.ts` that merges snapshot-provided custom channel ids with the built-in channel id list, preserves snapshot/custom order, de-dupes ids, and adds regression coverage for custom-only `channelMeta` and `channelOrder`.

A custom-only snapshot still produces custom-only cards.
