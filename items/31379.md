## Summary

Keep #31379 open. Current main still renders the Channels settings cards from `channelMeta` or `channelOrder` exclusively, and the backend still builds those arrays from loaded channel plugins. A custom-only channel snapshot can therefore still hide the built-in channel cards. The provided related context shows #44165 as the focused open fix and #31350 as closed/unmerged, so the issue is not implemented or safely superseded on main yet.

## Best Possible Solution

Keep #31379 open until #44165 or an equivalent focused Control UI fix lands. The best fix belongs in `ui/src/ui/views/channels.ts`: merge snapshot-provided custom channel ids with the built-in channel id list, preserve snapshot/custom order, de-dupe ids, keep configured/active channels promoted by the existing sort, and add regression tests for custom-only `channelMeta` and `channelOrder`.
