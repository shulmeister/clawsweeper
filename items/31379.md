## Summary

Current `main` still has the reported Channels settings behavior. The Control UI renders only the ids supplied by `channelMeta` or `channelOrder`; the backend still builds those fields from loaded channel plugins; and there is no landed regression test for appending built-in channel cards when a custom-only channel snapshot is present. The focused fix PR #44165 remains the right implementation path, while the earlier PR #31350 was closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep #31379 open until PR #44165 or an equivalent focused Control UI fix lands. The implementation should update `ui/src/ui/views/channels.ts` to merge snapshot-provided custom channel ids with the built-in channel id list, de-duplicate while preserving snapshot/custom order first, retain the enabled-channel promotion sort, and add regression coverage in `ui/src/ui/views/channels.test.ts` for custom-only `channelMeta`, custom-only `channelOrder`, and the no-snapshot fallback.
