## Summary

Keep #31379 open. Current main still has the reported Control UI behavior: if `channels.status` returns `channelMeta` or `channelOrder` for only a custom channel plugin, `resolveChannelOrder` returns only those ids and never appends the built-in channel cards. The related fix PR #44165 remains open/unmerged in the provided context, and the earlier PR #31350 was closed unmerged.

## Best Possible Solution

Keep the issue open and resolve it through #44165 or an equivalent focused UI change that merges snapshot-provided custom channel ids with the built-in channel id list, preserving snapshot order and deduping, with regression coverage for custom-only `channelMeta` and `channelOrder`.
