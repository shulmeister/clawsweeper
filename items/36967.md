## Summary

Keep PR #36967 open. Current main has superseded some supporting pieces, but the central Feishu work is still not implemented: Feishu does not expose a config-backed /allowlist adapter, and it is still not marked markdown-capable in plugin metadata. The PR also has unresolved review concerns around default-account and inherited allowlist semantics, so it should be narrowed and reworked rather than closed.

## Best Possible Solution

Keep PR #36967 open for a narrower rework: wire a Feishu-owned allowlist adapter using buildDmGroupAccountAllowlistAdapter or an equivalent that reads effective merged Feishu account config and writes without dropping inherited allowlist entries; set Feishu markdownCapable in both package/channel metadata as needed; add focused Feishu /allowlist channel=feishu, markdown-capability, and envelope regression tests; remove the security-fix and hook-schema chunks already superseded on main; address the existing review comments before merge.
