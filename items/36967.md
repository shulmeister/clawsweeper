## Summary

Keep PR #36967 open. Current main has superseded some broad chunks of the PR, including generic hook-channel handling and generic security groupPolicy fixing, but it still does not solve the central Feishu behavior: explicit `/allowlist channel=feishu` is not plugin-aware, Feishu has no config-backed allowlist adapter, Feishu is not marked markdown-capable, and Feishu envelope headers are not recognized. Existing review comments also identify account inheritance/read-write risks in the old branch, so the maintainer path is a narrower rework rather than closure.

## Best Possible Solution

Keep PR #36967 open for maintainer/contributor follow-up, but narrow it before merge: use the existing plugin SDK allowlist adapter for Feishu with merged-account semantics, make `/allowlist channel=feishu` resolve plugin channel IDs safely, add `markdownCapable` to Feishu runtime and package metadata, add Feishu envelope recognition, and add focused regression tests. Drop or separate the security-fix and hook-schema chunks already handled generically on main.
