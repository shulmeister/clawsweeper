## Summary

Keep open. Current main still lacks the central Feishu behavior this PR targets: Feishu is not resolved as an explicit /allowlist target from shared command paths, the Feishu channel plugin still has no config-backed allowlist adapter, Feishu is not marked markdown-capable in channel metadata, and Feishu envelope headers are still not recognized. Some PR chunks are now superseded on main, so the best path is a narrower update rather than conservative closure.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up, but narrow it before merge: add a Feishu allowlist adapter through the existing plugin SDK helper with correct merged-account semantics, mark Feishu markdown-capable in both runtime/package metadata, add Feishu envelope recognition plus regression tests, and drop the now-obsolete hook-schema/security-fix chunks already handled generically on main.
