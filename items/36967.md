## Summary

Keep open. Current main still does not implement the main Feishu behavior this PR targets: Feishu config-backed `/allowlist` support, Feishu markdown capability metadata, and Feishu envelope stripping are still missing. Some PR chunks are now superseded by main, notably open-ended hook channel validation and generic security fix handling, and the PR review discussion identifies account-resolution risks that should be addressed before this can land.

## Best Possible Solution

Keep this open, but narrow the path before merge: add a Feishu-owned allowlist adapter through the existing plugin SDK helper, make `/allowlist channel=feishu` resolve via the plugin-aware channel path if cross-channel targeting is desired, mark Feishu markdown capability in plugin metadata, add Feishu envelope recognition or a generic plugin metadata source for envelope labels, and drop the already-superseded hook/security-list edits. Address the PR review comments around defaultAccount and inherited account allowlists with targeted regression tests.
