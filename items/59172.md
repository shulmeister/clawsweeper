## Summary

Keep open. Current `main` still limits the floating-holiday CLI tagline tables to 2025-2027, so this PR addresses a real unmerged gap and does not meet any conservative close condition.

## Evidence

- **Current main still ends all floating-holiday tables at 2027:** `HOLIDAY_RULES` on current `main` has Lunar New Year, Eid, Diwali, Easter, and Hanukkah entries only for 2025-2027; there are no 2028-2030 rows in the checked-out code.
  - file: [src/cli/tagline.ts:179](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/cli/tagline.ts#L179)
  - command: `nl -ba src/cli/tagline.ts | sed -n '175,245p'`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Behavior would silently stop after 2027 for table-driven holidays:** `isTaglineActive` only returns true when a rule matches the current date, and `pickTagline` filters the tagline pool through that check. With no future entries in `HOLIDAY_RULES`, those holiday taglines drop out entirely after 2027.
  - file: [src/cli/tagline.ts:240](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/cli/tagline.ts#L240)

  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Current tests do not cover the reported future-date regression:** The checked-in tagline tests only verify `off`, `default`, and `OPENCLAW_TAGLINE_INDEX` behavior; they do not exercise holiday activation dates past 2027.
  - file: [src/cli/tagline.test.ts:1](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/cli/tagline.test.ts#L1)
