## Summary

Keep open. Current main still lacks the PR’s root-to-basePath redirect: with a configured Control UI basePath, `pathname === "/"` falls through as `not-control-ui`, and the HTTP server then returns the same plain `Not Found` response described by the linked bug. The PR remains a focused, relevant fix, and its review-thread query-string test gap was addressed in the provided discussion.

## Best Possible Solution

Keep the PR open and have a maintainer either merge it or reapply the same narrowly scoped change on main: teach `classifyControlUiRequest` to redirect read-method requests for `/` to `${basePath}/` while preserving the query string, keep non-read root requests falling through for plugin/webhook safety, and retain the added regression tests for GET root, query-string preservation, and POST fall-through.
