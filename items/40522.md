## Summary

Keep open. Current main still lacks the PR’s requested root-to-Control-UI-basePath redirect: with a non-empty `gateway.controlUi.basePath`, GET or HEAD `/` is classified as `not-control-ui`, the Control UI handler declines it, and the gateway falls through to `Not Found`. The PR remains a focused fix for the linked #34298 behavior, and the provided discussion shows the review-requested query-string preservation test was addressed.

## Best Possible Solution

Keep the PR open. A maintainer should merge it or reapply the same narrow change on main: in `classifyControlUiRequest`, redirect read-method requests for `/` to `${basePath}/` while preserving the query string, keep non-read root requests falling through for plugin/webhook safety, and retain regression tests for GET root redirect, query-string preservation, and POST fall-through.
