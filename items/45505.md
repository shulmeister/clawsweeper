## Summary

Keep open. Current main includes the narrow late-success counter reconciliation from #45493, but the broader follow-up request is still not implemented: the compaction success hook only reconciles `sessions.json.compactionCount`/`updatedAt`, while token/session-id/UI-facing run state still converges through separate finalization paths. The issue is core session-state architecture work, not ClawHub/plugin scope, and no canonical narrower issue appears to supersede the remaining generalized reconciliation request.

## Evidence

That is evidence of the merged narrow repair path, not a generalized session-state reconciliation event.
