## Summary

Current main has partial reminder telemetry, but it does not implement the Telegram outbound sanitization described by #66732. The unscheduled-reminder guard note still exists and is appended upstream when `successfulCronAdds` is zero, while Telegram direct text, media caption, and payload delivery paths forward text to `sendMessageTelegram` without a Telegram-specific filter for that note.

## Best Possible Solution

Keep #66732 open. The best fix is a narrowly scoped Telegram outbound sanitizer, preferably sharing the existing `UNSCHEDULED_REMINDER_NOTE` constant or a small exported helper, applied to direct text sends, media captions, and `sendPayload` fallback text before `sendMessageTelegram`. Add regression tests in `extensions/telegram/src/outbound-adapter.test.ts` for the exact note and reported variants, while preserving dismiss/interactive payload data. If maintainers prefer an upstream-only fix, extend telemetry so every successful reminder-creation path increments `successfulCronAdds`, then prove Telegram no longer receives the contradictory note.
