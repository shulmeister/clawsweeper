## Summary

Keep open. Current main still has the reported Discord explicit-target bug: `parseDiscordExplicitTarget` parses `user:<id>` correctly but returns the bare `target.id`, and the downstream delivery path then treats a bare Discord snowflake as a channel target. The PR's proposed behavior and regression tests are not present on main, and the related #72247 is closed/unmerged rather than a shipped replacement.

## Best Possible Solution

Keep this PR open for maintainer review or land an equivalent Discord-owned fix. The best patch is the narrow change from `target.id` to `target.normalized` in `extensions/discord/src/channel.ts`, with focused regression tests in `extensions/discord/src/channel.test.ts` for `user:<id>`, `channel:<id>`, and ideally Discord user mentions before closing the PR as implemented.
