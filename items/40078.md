## Summary

Keep #40078 open. Current main still does not implement a Discord guild/channel `preflightTranscribe` option or equivalent silent history ingestion mode. Discord can preflight-transcribe some audio for DMs and spoken mention detection, but unmentioned `requireMention` guild audio still follows the no-mention skip path and records the original media-placeholder history entry for later context.

## Best Possible Solution

Keep #40078 open. Add an explicit Discord guild/channel option such as `preflightTranscribe` across config types, strict schema, generated metadata, docs, and focused tests. In Discord preflight, after authorization and before the no-mention return, transcribe eligible unmentioned audio when enabled, store transcript text in pending guild history instead of `<media:audio>`, suppress agent dispatch, and prove a later @mention receives the transcript in history context.

## Risks / Open Questions

- Silent background transcription has cost and privacy implications, so it should remain explicit opt-in and run only after guild/channel/member access checks.
- An implementation should avoid a second STT call if the same audio is later processed during a mentioned turn.
