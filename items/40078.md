## Summary

Keep #40078 open. Current main still lacks a Discord guild/channel opt-in that silently transcribes unmentioned audio in `requireMention` channels and stores the transcript for later history context. Existing Discord audio preflight can support DMs and spoken mention detection, but the no-mention guild path still records the original `<media:audio>` history entry and returns without carrying the transcript forward.

## Best Possible Solution

Keep #40078 open. Add an explicit Discord guild/channel option such as `preflightTranscribe` in config types, strict schema, generated metadata, docs, and focused tests. In the Discord preflight path, after authorization but before the no-mention return, transcribe eligible unmentioned audio, replace or supplement the pending history entry with transcript text, suppress agent dispatch, and add a regression proving a later @mention sees transcript text rather than `<media:audio>` placeholders.

## Risks / Open Questions

- Silent background transcription has cost and privacy implications, so any implementation should stay explicit opt-in and run only after guild/channel/member access checks.
- The implementation should avoid duplicate transcription when a later mentioned turn processes the same original audio attachment.
