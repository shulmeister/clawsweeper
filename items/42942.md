## Summary

Current `main` already contains the OpenAI Codex transport and error-handling changes that address this March 2026 `openai-codex/gpt-5.4` timeout regression. The transport was corrected to the Codex-specific backend path, stale `openai-codex` metadata is normalized back to the canonical Codex transport, and Cloudflare/HTML 403 failures are surfaced explicitly instead of only collapsing into a generic timeout.

## Evidence

- **Shipped transport fix:** The changelog for release `2026.4.20` says OpenClaw now routes ChatGPT/Codex OAuth Responses through `/backend-api/codex`, specifically so `openai-codex/gpt-5.4` no longer hits the removed legacy route.
  - file: [CHANGELOG.md:376](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/CHANGELOG.md#L376)
- **Shipped stale-metadata normalization:** The changelog for release `2026.4.15` says legacy `openai-codex` rows with missing `api` or `https://chatgpt.com/backend-api/v1` are normalized back to the canonical Codex transport instead of broken HTML/Cloudflare paths.
  - file: [CHANGELOG.md:573](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/CHANGELOG.md#L573)
- **Current code normalizes broken Codex transport metadata:** `normalizeTransport(...)` rewrites native `openai-codex` rows with unset/legacy transport metadata to `api: "openai-codex-responses"` and `baseUrl: "https://chatgpt.com/backend-api"`, which is the post-fix canonical transport.
  - file: [src/agents/pi-embedded-runner/model.provider-runtime.test-support.ts:125](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/agents/pi-embedded-runner/model.provider-runtime.test-support.ts#L125)
- **Regression test locks in the normalization:** The test `normalizes stale discovered openai-codex /backend-api/v1 metadata` asserts that a stale `gpt-5.4` Codex row is repaired to `api: "openai-codex-responses"` and `baseUrl: "https://chatgpt.com/backend-api"`.
  - file: [src/agents/pi-embedded-runner/model.test.ts:1236](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/agents/pi-embedded-runner/model.test.ts#L1236)
- **Current code surfaces Cloudflare/HTML failures explicitly:** `formatAssistantErrorText` now returns explicit provider HTML/CDN error copy for `auth_html_403` and `upstream_html`, instead of only falling through to `LLM request timed out.`.
  - file: [src/agents/pi-embedded-helpers/errors.ts:1003](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/agents/pi-embedded-helpers/errors.ts#L1003)
- **No current provider route to `backend-api/conversation` found in agent/provider code:** A repo search over `src/agents`, `src/plugins`, and `src/infra` found no current ChatGPT backend provider route using `backend-api/conversation`, which matches the later transport cleanup.
  - command: `rg -n 'backend-api/conversation|/conversation\\b' src/agents src/plugins src/infra`
