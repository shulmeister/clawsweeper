## Summary

Keep open. Current main still has no `channels.slack.ignoreSubtypes` config key or equivalent monitor-level subtype suppression. The prior issue comment remains accurate for `message_changed` / `message_deleted`: those subtypes resolve to Slack system-event handlers, authorize a session context, enqueue a system event, and return before normal message handling. The related `thread_broadcast` work appears handled, but that does not solve this issue's requested edit/delete suppression knob.
