## Summary

Current main does not yet implement the proposed IMDS suppression. The Amazon Bedrock memory embedding adapter still calls hasAwsCredentials during provider creation, and hasAwsCredentials still loads @aws-sdk/credential-provider-node and invokes defaultProvider({ timeout: 1000, maxRetries: 0 }) after only static key/token checks. There is no AWS_EC2_METADATA_DISABLED guard or temporary IMDS disable in the current source, and the existing tests do not cover the reported IMDS-probe regression. The related fix PR #64944 was closed unmerged, so it is not a reason to close this issue.

## Best Possible Solution

Review and land a maintained variant of #64944 in the amazon-bedrock plugin: make credential auto-detection avoid IMDS probing when no explicit AWS credential marker is present, honor AWS_EC2_METADATA_DISABLED as an early false result, and add focused tests in extensions/amazon-bedrock/embedding-provider.test.ts that prove IMDS is disabled/restored during the defaultProvider probe. Keep the actual Bedrock runtime client on the normal AWS SDK credential chain so EC2 instance-role auth still works for configured Bedrock usage.
