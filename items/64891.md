## Summary

Keep open. Current `main` still routes Bedrock memory credential auto-detection through AWS SDK `defaultProvider()`, and AWS documents that this default chain can probe IMDS at `169.254.169.254` with IMDSv2 falling back to IMDSv1 on 403. Memory auto-selection still reaches the Bedrock adapter in `provider: "auto"`, so the reported startup probes remain plausible on current `main`; the related Bedrock memory cleanup now on `main` does not remove that probe path.
