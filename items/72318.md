## Summary

Current main contains a post-v2026.4.24 Windows plugin-loader fix for this exact Telegram `ERR_UNSUPPORTED_ESM_URL_SCHEME` failure. The shipped v2026.4.24 code still allowed native Jiti on Windows, while current main disables native Jiti on `win32`, has a Telegram-specific bundled-entry regression test asserting `tryNative: false`, and the changelog records the fix as addressing channel plugins such as Telegram crashing on `C:\...` paths.

## Best Possible Solution

Close this issue as implemented on current main. Keep the main-branch Windows loader policy that disables Jiti native import on `win32`; users affected on 26.4.24 should pick up the next release containing the #71749 fix rather than tracking a separate duplicate issue.
