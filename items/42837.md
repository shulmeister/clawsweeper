## Summary

Current `main` still appears to have the reported Feishu named-account routing bug. `dynamicAgentCreation` creates a Feishu DM binding without `match.accountId`, but Feishu inbound routing re-resolves with `account.accountId`, and the routing contract explicitly treats account-less bindings as matching only the default account. That means named Feishu accounts can still fall through to the default `agent:main` route even after the dynamic agent and binding are created.

## Evidence

- **Dynamic binding omits accountId:** `maybeCreateDynamicAgent()` writes Feishu direct-message bindings with only `channel` and `peer`, both when backfilling a missing binding for an existing agent and when creating a new agent. No `match.accountId` is recorded.
  - file: [extensions/feishu/src/dynamic-agent.ts:61](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/extensions/feishu/src/dynamic-agent.ts#L61)

  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Inbound Feishu routing uses named accountId:** The main Feishu message handler resolves the route with `accountId: account.accountId`, then after dynamic-agent creation re-resolves using the same named account id.
  - file: [extensions/feishu/src/bot.ts:590](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/extensions/feishu/src/bot.ts#L590)

  - file: [extensions/feishu/src/comment-handler.ts:136](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/extensions/feishu/src/comment-handler.ts#L136)
