## Summary

Keep open. The report is recent, detailed, and targets core gateway-host exec behavior. Current main still routes non-sandbox gateway exec through a fresh shell command invocation and explicitly skips the Python/Node script preflight in `security=full` + `ask=off` YOLO mode, matching the code path called out by the issue. I found no current-main fix for direct argv execution, cross-exec file coherence, or macOS/APFS stale-read mitigation, and the supplied related items are only title-overlap stale/context issues rather than a canonical duplicate.

## Best Possible Solution

Keep #71326 open for maintainer investigation. Reproduce on macOS/APFS with gateway-host `security=full` and `ask=off`, add the narrowest regression harness possible around atomic rename followed by cross-exec reads, then fix in the core exec runtime if confirmed. A likely direction is preserving shell behavior for complex commands while adding a direct argv/interpreter path or an explicit filesystem-coherence barrier for simple command/file-read cases, with security review against the existing preflight and approval contracts.
