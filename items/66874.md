## Summary

Keep open. Current main still does not satisfy either requested path: the official Dockerfile does not preinstall `@anthropic-ai/claude-code` or a `claude` binary, and the Docker docs only provide generic persistence/binary-baking guidance rather than Claude CLI-specific install, `~/.claude` persistence, and `cliBackends.claude-cli.command` examples. The linked #2440/community-plugin context does not supersede this Docker image/docs request.

## Best Possible Solution

Keep #66874 open as a concrete enhancement. The best near-term path is to add Docker/Anthropic documentation for a persistent Claude CLI install, `~/.claude` persistence, and an explicit `agents.defaults.cliBackends.claude-cli.command` example; separately decide whether the official image should pin and preinstall `@anthropic-ai/claude-code`.

## Risks / Open Questions

- This remains a product/packaging choice: preinstalling Claude Code would add third-party binary pinning, image-size, and update-policy tradeoffs, while a docs-only fix must be explicit enough for Docker upgrades and credential persistence.
