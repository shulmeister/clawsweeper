## Summary

Keep PR #33845 open. Current main still keeps Slack thread participation in a process-local dedupe cache, while Slack mention gating still relies on that cache for non-mention thread continuation. The related PRs in the provided context are open/unmerged and do not prove a shipped canonical replacement.

## Best Possible Solution

Keep #33845 open until a canonical Slack thread-participation persistence fix lands on main. The likely product/code path is to migrate Slack participation tracking to an accepted persistent helper such as `openclaw/plugin-sdk/persistent-dedupe` or the proposed persistent keyed store if #70626 becomes the chosen SDK contract, with regression tests for restart round-trip, TTL expiry, missing/corrupt state, clear semantics, max-entry/LRU behavior, and split module instances.
