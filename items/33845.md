## Summary

Keep PR #33845 open. Current main still uses only an in-memory Slack thread participation cache, so the restart-survival behavior proposed by the PR is not implemented. The related persistent keyed store PR is still open/draft and explicitly does not migrate consumers, so it does not supersede this Slack fix.

## Best Possible Solution

Keep the PR open and have maintainers choose the integration path: land a reviewed Slack-owned persistence fix, or migrate the Slack thread participation cache to a shared persistent store/helper once that primitive is accepted. The remaining product bug is real on current main because Slack bot-participated thread state is still lost on gateway restart.
