## Summary

Keep PR #33845 open. Current main still has only an in-memory Slack thread participation cache, so the restart-survival behavior proposed by the PR is not implemented. The related persistent-keyed-store PR is a possible future primitive, but it explicitly has no Slack consumer migration and does not supersede this focused Slack fix.

## Best Possible Solution

Keep the PR open and have maintainers decide the integration path: either land a reviewed Slack-local persistence fix, or migrate the Slack thread participation cache to the shared persistent keyed store once that SDK helper exists. The remaining product problem belongs in the Slack plugin code path, because current main still drops bot-participated thread state on gateway restart.
