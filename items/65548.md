## Summary

Current main already routes `/status` and `session_status` through the shared `src/status/*` module with local runtime shims, removing the bundler-opaque lazy import path described in the report. The shipped fix is documented in the changelog, and the issue thread confirms the behavior was working again in release v2026.4.14.
