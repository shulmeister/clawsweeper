## Summary

Keep this PR open. Current main already has the Feishu schema-generation part, but the PR is not fully obsolete: gateway health snapshots still use the pinned `loadConfig()` path, the provider-plan normalization hunk is not on main and needs review against the current manifest-owned auth alias/catalog model, and the skill symlink change conflicts with current root-escape rejection behavior and tests. The existing Greptile review also flagged the PR's health hunk as needing rework, so the best maintainer path is a rebase/split with targeted tests rather than auto-closing.

## Best Possible Solution

Ask the contributor or a maintainer to rebase and split this into focused PRs: drop the already-shipped Feishu schema hunk; wire `getHealthSnapshot()` to the current fresh best-effort config path with a regression test; verify whether plan providers should remain distinct for catalog/runtime while using manifest auth aliases; and treat the skill symlink fallback as a separate security-sensitive change with tests for pnpm and macOS path-canonicalization cases.
