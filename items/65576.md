## Summary

Current main already treats a cron job's own `payload.timeoutSeconds` as an explicit run timeout for LLM idle-timeout resolution, so the reported `timeoutSeconds: 180` repro does not silently disable the watchdog or block failover on current OpenClaw. The changelog also records this behavior as shipped in `v2026.4.9`, and the latest tagged release `v2026.4.23` still contains the same handoff path.
