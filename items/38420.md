## Summary

Keep open. Current main has implemented several pieces of the UX audit, but multiple concrete requests from #38420 remain visible in the Control UI source. The related #39780 covers only a narrower config apply/hardening subset, and the older #13142 was a Config/Models subset, so neither supersedes this full-dashboard audit.

## Best Possible Solution

Keep #38420 open as the umbrella Control UI UX audit unless a maintainer deliberately splits it into narrower tracked issues. Continue using #39780 for the config apply/hardening subset, but leave this issue to track remaining full-dashboard work such as Instances per-row actions and reason/chip affordances, Sessions key copy/truncation and less repetitive override controls, clearer Config action semantics, keyboard shortcut/help support, and broader operator feedback/toast or activity-log UX.
