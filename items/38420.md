## Summary

Keep #38420 open. Current main has addressed several bullets, including breadcrumbs, masked gateway token entry, config invalid-state warning, usage loading/empty/error states, and session delete confirmation, but the full Control UI audit is still not implemented. Meaningful requested work remains in Instances, Sessions, Config action semantics, and dashboard-wide shortcut/help/feedback UX. The linked #39780 is still only a narrower Config hardening issue, not a superseding tracker for the whole dashboard audit.

## Best Possible Solution

Keep #38420 open unless a maintainer intentionally splits it. The best path is to either implement or split focused follow-ups for Instances per-instance actions and clearer presence affordances, Sessions key copy/truncation and less repetitive advanced controls, Config action semantics, dashboard shortcut/help UX, and save/delete/apply feedback or activity history. Treat #39780 as only the narrower Config hardening follow-up, not a superseding tracker for this full Control UI audit.
