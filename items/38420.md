## Summary

Keep #38420 open. Current main has addressed some pieces of the UX audit, including masked gateway-token entry, but multiple concrete dashboard requests remain implemented as the older behavior: Instances has no per-instance actions or reason/chip affordances, Sessions still repeats advanced selects per row and lacks key copy/desktop truncation, and Config still presents adjacent Save/Apply/Update buttons. The related #39780 is a narrower open Config hardening request, not a superseding tracker for the full dashboard audit.

## Best Possible Solution

Keep #38420 open as the umbrella Control UI UX audit unless a maintainer intentionally splits it. Track #39780 as the narrower Config hardening item, and either implement or split the remaining dashboard work: Instances per-row actions and clearer reason/chip affordances, Sessions key copy/truncation and less repetitive override controls, clearer Config action semantics, broader operator shortcuts/help, and feedback/activity UX.
