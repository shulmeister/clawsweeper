## Summary

Current `main` does not implement this PR, and the new command-auth path explicitly keeps pairing-store identities DM-only for group command authorization. The PR also targets a fallback path that is not the primary authorization path for Telegram/Discord, matching the existing review concern, so it should stay open for maintainer judgment rather than be auto-closed.
