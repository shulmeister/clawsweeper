## Summary

Keep open. Current `main` has partial fixes for web/session thinking displays, but the issue is not resolved end-to-end: `/status` still prefers raw configured `thinkingDefault` over model-aware fallback/remapping, the TUI footer still renders only the persisted session override instead of the effective default, and GitHub Copilot `xhigh` support still does not include the models named in the report.

## Evidence

- **`/status` still surfaces unsupported configured defaults:** `src/status/status-text.ts:294-323` computes `explicitThinkingDefault` from config and passes `resolvedThink: resolvedThinkLevel ?? explicitThinkingDefault ?? (await resolveDefaultThinkingLevel())`. That means a configured `thinkingDefault: "xhigh"` still wins even when the selected model would need model-aware fallback/remapping.
  - file: [src/status/status-text.ts:294](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/status/status-text.ts#L294)
  - command: `nl -ba src/status/status-text.ts | sed -n '285,323p'`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **TUI footer still ignores effective default thinking:** `src/tui/tui.ts:828` renders `sessionInfo.thinkingLevel ?? "off"`. `src/tui/tui-session-actions.ts:168-170,303` only copies persisted `entry.thinkingLevel`/history `thinkingLevel` into `sessionInfo`, with no `thinkingDefault` or model-default resolution path. That still leaves the footer inconsistent when no session override is stored.
  - file: [src/tui/tui.ts:828](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/tui/tui.ts#L828)
