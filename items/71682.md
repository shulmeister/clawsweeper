## Summary

Keep #71682 open. Current main has a real CLI/config path for increasing or disabling the LLM idle timeout, and model-selection docs cover `openclaw models set`, so part of the report is already addressed. The remaining actionable gap is still present: the agent-facing owner-only `gateway config.patch/apply` tool allowlist does not permit `agents.defaults.llm.idleTimeoutSeconds`, so the reporter's "LLM can change the timeouts" workflow is still blocked; timeout-driven fallback/cooldown behavior is also still tracked by open #51057.

## Best Possible Solution

Keep this open for maintainer triage. The likely fix is to document the concrete CLI workaround (`openclaw config set agents.defaults.llm.idleTimeoutSeconds <seconds>` or `0` to disable), add the idle-timeout key to the relevant config docs, and decide whether the agent-facing gateway config tool should allow bounded edits to `agents.defaults.llm.idleTimeoutSeconds`. Leave broader timeout-profile cooldown/faster fallback policy to #51057 unless this issue adds a distinct local-Ollama repro.
