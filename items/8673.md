## Summary

Keep #8673 open. Current main still lacks the requested general retry/backoff around OAuth token refresh; the only retry-like path I found is a targeted refresh_token_reused recovery, not transient network/API retry. The linked implementation PR #45975 remains open/unmerged in the provided GitHub context, so the issue is still a valid tracker rather than implemented on main.
