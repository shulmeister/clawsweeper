## Summary

Current main still does not implement bounded retry/backoff around OAuth token refresh. The built-in refresh path calls the provider token refresh once under timeout/locks, and the only retry-like branch is the narrow `refresh_token_reused` recovery path. The related fix PR #45975 is open/unmerged in the provided context, so #8673 remains the right tracker.

## Best Possible Solution

Keep #8673 open and implement a bounded, tested retry policy around the actual OAuth refresh operation on the current auth path. Prefer reusing `src/infra/retry.ts`, retry only transient network/timeout/rate-limit/5xx failures, preserve immediate handling for permanent OAuth failures such as `invalid_grant`, keep the existing `refresh_token_reused` adoption path intact, and add focused tests under `src/agents/auth-profiles/` for success after transient failure, retry exhaustion preserving the final error, and non-retryable auth failures.
