## Summary

Keep open. Current main still single-shots the OAuth refresh operation: the built-in path awaits `getOAuthApiKey(...)` directly, the manager only adds timeout/locking, and the only retry-like branch is gated to `refresh_token_reused`. The related fix PR #45975 is still open/unmerged in the provided context, while earlier PR #8677 was closed unmerged.

## Best Possible Solution

Keep #8673 open and implement a bounded, tested retry policy around the actual OAuth refresh operation on the current auth path. Prefer reusing `src/infra/retry.ts`, retry only transient network, timeout, rate-limit, and 5xx failures, preserve immediate handling for permanent OAuth failures, keep the existing `refresh_token_reused` adoption path intact, and add focused tests under `src/agents/auth-profiles/` for success after transient failure, retry exhaustion preserving the final error, and non-retryable auth failures.
