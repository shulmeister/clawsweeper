## Summary

Keep open. Current main still does not support `openclaw cron add --id`: the CLI does not expose or forward an `id`, the create type excludes `id`, the Gateway add schema rejects extra properties, and the service still generates a UUID. The issue also has two open unmerged PRs with closing syntax, so the issue should remain open until maintainers land or reject the implementation candidate(s).

## Best Possible Solution

Keep this issue open and review the existing implementation candidates, likely preferring the smaller focused PR if it still applies cleanly. The merged solution should add `--id` to `cron add`, carry the id through the Gateway protocol and `CronJobCreate`, validate a safe slug-like format, reject collisions before persistence, preserve UUID generation when omitted, and add focused CLI/protocol/service tests.
