## Summary

Keep #69076 open. Current main still treats manual-cancel exec exits and successful process kill/remove actions as failed in the relevant code paths, and the Telegram sent-message cache still stores timestamp-only records. The PR’s central fix is not implemented on main.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land or adapt the core cancellation semantics on main: classify explicit `manual-cancel` exits as `killed`, make successful process `kill`/`remove` control actions return completed tool status, preserve killed status through poll/log, and suppress notify-on-exit for killed sessions. Maintainers should also decide whether the additive Telegram sent-message metadata belongs in this PR or should be split, but that is not a reason to close the item.
