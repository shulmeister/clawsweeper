## Summary

Keep this PR open. Current main still rejects `messages.groupChat.historyLimit: 0` even though docs and provider/channel schemas treat zero as the disable value, so the core config fix is still useful. The PR is not ready to close or land as-is because it also includes unrelated memory-core search changes and an unresolved review concern about vector-only search regression.

## Best Possible Solution

Keep this PR open and either ask the contributor to split/drop the unrelated memory-core changes or have a maintainer cherry-pick the scoped config fix: change `GroupChatSchema.historyLimit` to `min(0)`, regenerate `src/config/schema.base.generated.ts`, and add the regression test. The memory-core fallback work should move to a separate PR after resolving the vector-only search review concern.
