## Summary

Keep PR #39954 open. Current main still documents `tools.profile` as a mechanical profile table, but it does not include the PR's practical warning that `tools.profile: "messaging"` is intentionally narrow or the direct `tools.profile: "full"` guidance/example for users who want the broadest command/control surface.

## Best Possible Solution

Keep the PR open and port the useful wording into the current `docs/tools/index.md` profile section. The best maintainer path is a compact note explaining that `tools.profile: "messaging"` is intentionally narrow, `tools.profile: "full"` is the unrestricted baseline for broader command/control, and the examples should be reconciled with the evolved profile table and adjacent open help/status/troubleshooting PRs.
