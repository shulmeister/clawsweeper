## Summary

Keep #22358 open. Current main still has only adjacent subagent lifecycle hooks and delivery plumbing; it does not expose the requested post-subagent-completion, pre-announcement plugin hook with completion-output inspection, delivery-result visibility, or announcement enrichment/suppression semantics. The issue discussion also explicitly keeps this narrower hook request alive after related delivery fixes and after the broader #8354 observability request was closed.

## Best Possible Solution

Keep #22358 open until OpenClaw adds a public typed plugin hook at the post-subagent-completion/pre-announcement point, or explicitly folds that exact scope into a canonical lifecycle/delivery design. The contract should include child/session/run metadata, completion output or child-history access, outcome/timing/usage where available, delivery path/status/error visibility, and explicit return semantics for announcement enrichment, suppression, or filtering, with docs and ordering regression tests.
