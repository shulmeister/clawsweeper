## Summary

Keep open. Current main has subagent spawn/routing/spawned/ended hooks, plus generic message/delivery hooks, but no typed post-subagent-completion/pre-announcement hook that exposes child output/history, delivery result, usage, or return semantics to enrich, suppress, or filter the completion announcement.

## Best Possible Solution

Keep #22358 open until OpenClaw adds a public typed plugin hook at the post-subagent-completion/pre-announcement point, or explicitly folds that exact scope into a canonical lifecycle/delivery design. The contract should expose child/session/run metadata, completion output or child-history access, timing/usage where available, delivery status/error visibility, and return semantics for announcement enrichment, suppression, or filtering, with public docs and ordering regression tests.
