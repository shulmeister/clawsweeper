## Summary

Keep open. Current main improves WhatsApp outbound audio by sending PTT voice notes and transcoding non-Ogg audio to Ogg/Opus, but it still does not compute or thread Baileys `seconds` and `waveform` metadata. The related Discord voice-message utilities exist on main, matching the useful prior context from PR #7253, but they remain Discord-local and are not used by the WhatsApp send path.

## Evidence

- **WhatsApp audio payload omits duration/waveform:** The WhatsApp send API audio branch constructs only `{ audio: mediaBuffer, ptt: true, mimetype: mediaType }`; there is no `seconds` or `waveform` field in the Baileys payload.
