## Summary

Keep open. Current `main` still requires a browser secure context for Control UI device identity, so the PR's primary change is not implemented. Main has separately absorbed some secondary pieces of the PR (`dangerouslyDisableDeviceAuth` scope preservation and the `scope-errors` follow-up), but the core HTTP reverse-proxy/device-identity request remains outstanding.
