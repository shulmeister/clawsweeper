## Summary

Keep open. Current `main` appears to have fixed the specific WhatsApp `dmPolicy: "allowlist"` pairing-reply leak that motivated the report, but the requested operator-facing surfaces are still not present: there is no documented `message:unauthorized`/`access:denied` hook event and no documented `suppressPairingReply` config flag.

## Evidence

- **WhatsApp access control now silently blocks unauthorized allowlist DMs:** `checkInboundAccessControl()` only sends a pairing reply when access resolves to `pairing`; otherwise unauthorized DMs fall through to a silent block. The local `suppressPairingReply` variable only suppresses historical reconnect-time replies and is not a config field.
