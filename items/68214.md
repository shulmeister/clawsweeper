## Summary

Keep open. Current main appears to fix the reported WhatsApp allowlist pairing-reply leak, but the actual feature request still asks for a configurable suppression flag and an access-denied hook/suppress-default contract, and those surfaces are not implemented.

## Best Possible Solution

Keep this open but scope it to the missing operator control surface: either add a typed, cross-channel access-denied hook with deny reason/default action/suppress-default semantics, or add and document a per-channel/account suppression flag if maintainers want a smaller config-only solution. Preserve the current shipped behavior that `dmPolicy: "allowlist"` silently blocks unauthorized WhatsApp DMs.
