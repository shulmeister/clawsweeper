## Summary

Keep open. Current `main` still exposes the full `sessions_spawn` parameter schema to agents, rejects some runtime-incompatible fields only at execution time, and does not implement any documented or validated per-agent `allowedParams`/`deniedParams` filtering. The only linked follow-up in the issue timeline is still an open PR, so there is no strong evidence this request is already resolved on `main`.

## Evidence

- **sessions_spawn still publishes the broad schema:** `createSessionsSpawnTool()` advertises one unconditional schema containing many optional fields, including `resumeSessionId`, `model`, `thinking`, `cwd`, `cleanup`, `sandbox`, `context`, `streamTo`, `lightContext`, `attachments`, and `attachAs`. There is no per-agent pruning step in this tool definition.
  - file: [src/agents/tools/sessions-spawn-tool.ts:100](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/tools/sessions-spawn-tool.ts#L100)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Unsupported runtime fields are rejected after exposure, not hidden from the model:** Current execution logic reads `streamTo` and `resumeSessionId` from the call and returns runtime errors when they are used with `runtime=subagent`, which matches the report that the model can still see and emit those parameters.
  - file: [src/agents/tools/sessions-spawn-tool.ts:229](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/tools/sessions-spawn-tool.ts#L229)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Per-agent docs do not expose parameter filtering:** The documented `agents.list[].tools` override surface covers profile/allow/deny/elevated policy, plus subagent allowlists, but does not document any per-tool parameter allowlist/denylist such as `allowedParams` or `deniedParams`.
  - file: [docs/gateway/config-agents.md:928](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/docs/gateway/config-agents.md#L928)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Validated tool config on main lacks allowedParams/deniedParams:** The runtime config schema only recognizes `tools.sessions_spawn.attachments` under `tools.sessions_spawn`; there is no validated config field for parameter filtering on this tool.
  - file: [src/config/zod-schema.agent-runtime.ts:935](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/config/zod-schema.agent-runtime.ts#L935)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Related fix is still pending, not landed:** The issue timeline links PR #68397 (`fix(sessions_spawn): silently strip ACP-only fields for runtime=subagent`) and issue #56193, and both are marked open in the provided GitHub context. That points to ongoing work rather than an implemented fix on `main`.
