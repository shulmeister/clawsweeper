## Summary

#66155 is the same remaining Control UI/WebChat bubble-action overlap bug already tracked by open issue #58702. Current main still shows the shared root cause: assistant action icons render inside the bubble, the touch-device CSS makes them visible, and the renderer still never applies the existing `.has-copy` padding hook.

## Best Possible Solution

Close #66155 as a duplicate of #58702 and fix the shared Control UI chat bubble/action layout there, carrying over the useful iPad Safari portrait-mode reproduction detail from this issue.
