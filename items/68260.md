## Summary

Current `main` no longer has the reported Ollama plugin schema/runtime mismatch. Plugin config is limited to discovery settings, while remote/custom Ollama hosts are configured through `models.providers.ollama.baseUrl`; that behavior is documented, covered by tests, and already present in release `v2026.4.23`.

## Evidence

- **Plugin manifest scope:** The Ollama plugin manifest on current `main` only allows plugin-level `discovery.enabled` under `configSchema`; `baseUrl` is not part of plugin config anymore.
  - file: [extensions/ollama/openclaw.plugin.json:27](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/extensions/ollama/openclaw.plugin.json#L27)
  - command: `sed -n '1,220p' extensions/ollama/openclaw.plugin.json`
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Runtime now reads provider config:** Ollama discovery reads `params.ctx.config.models?.providers?.ollama` as the explicit config source, normalizes `explicit.baseUrl`, and passes `explicit?.baseUrl` into provider discovery. This matches `models.providers.ollama.baseUrl`, not `plugins.entries.ollama.config.baseUrl`.
  - file: [extensions/ollama/src/discovery-shared.ts:102](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/extensions/ollama/src/discovery-shared.ts#L102)
  - command: `nl -ba extensions/ollama/src/discovery-shared.ts | sed -n '94,157p'`
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Stream path uses provider-level baseUrl:** The stream/runtime side resolves provider config from `config.models.providers` by provider id, then uses that provider-level `baseUrl` for Ollama runs.
  - file: [extensions/ollama/src/stream.ts:58](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/extensions/ollama/src/stream.ts#L58)
  - command: `nl -ba extensions/ollama/src/stream.ts | sed -n '43,81p'`
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Tests cover remote host config:** Current tests explicitly preserve and use remote/custom Ollama hosts supplied via `models.providers.ollama.baseUrl`, including normalization of `/v1` to the native base URL.
  - file: [extensions/ollama/provider-discovery.test.ts:129](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/extensions/ollama/provider-discovery.test.ts#L129)

  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Docs match the supported config surface:** The Ollama docs document remote/custom-host configuration via `models.providers.ollama.baseUrl` and warn not to use the `/v1` path.
