## Summary

Keep #39777 open. Current main still silently returns from Debug snapshot refresh when disconnected, and the Snapshots card still has no inline loading/progress or disconnected callout. The related PR #39806 remains the active unmerged fix attempt rather than evidence that the issue is resolved.

## Best Possible Solution

Keep the issue open. The maintainable fix belongs in the Control UI Debug surface: pass connection state or a refresh-disabled reason into `renderDebug()`, show an inline loading/progress state in the Snapshots card while `debugLoading` is true, show a clear disconnected/disabled callout when refresh cannot run, and add focused tests for `loadDebug()` and `renderDebug()`. Review or refresh PR #39806 as the existing implementation attempt.
