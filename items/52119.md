## Summary

Current main does not support the issue’s specific claim that OpenClaw parses natural-language chat text as JSON, but the report still points to a plausible Jan/local OpenAI-compatible backend interoperability problem. The code path stores user chat input as plain text and only JSON-parses gateway protocol frames, while current docs describe backend 400s from string-only content or tool-schema incompatibility as local-model compatibility issues rather than chat parsing bugs. Because I did not prove the Jan failure is fixed on main, this should stay open.

- **Gateway only JSON-parses transport frames:** `src/gateway/server/ws-connection/message-handler.ts:335-369` converts raw websocket frames to text and `JSON.parse`s them only to validate request/connect frames. This is gateway protocol parsing, not parsing user chat text into structured JSON.
