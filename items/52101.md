## Summary

Keep open. Current `main` has not implemented this PR: the bare `openclaw/plugin-sdk` barrel is still intentionally limited to five runtime exports, and `src/plugin-sdk/compat.ts` still does not re-export the compat symbols named in PR #52101. Main does provide documented narrow subpaths or `openclaw/plugin-sdk/core` for several of these helpers, so this is an unmerged SDK-surface expansion proposal, not cleanup-ready stale work.

## Risks / Open Questions

- Closing as `implemented_on_main` would be incorrect: current `main` still omits the requested exports from both `src/plugin-sdk/index.ts` and `src/plugin-sdk/compat.ts`.
- Closing as `cannot_reproduce` would also be weak: code inspection shows the bare-root legacy path still resolves through compat, so a plugin that depends on these missing root/compat exports would still fail on current `main`.
- This PR also conflicts with current SDK-boundary policy that keeps the root barrel intentionally tiny, so it needs maintainer product/architecture judgment rather than conservative auto-closure.
