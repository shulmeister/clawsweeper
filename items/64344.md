## Summary

Keep open. Current main still lacks the PR's requested audit distinction: `openclaw secrets audit` has no `MIGRATION_RECOMMENDED` finding code or summary count, and the config audit path classifies the resolved post-substitution config value as plaintext without comparing it to the authored `${ENV_VAR}` template. The supplied PR discussion supports keeping this as a focused implementation candidate: the author narrowed out broader `config.env`/dotenv attribution follow-ups, and the bot review described the final mechanism as scoped to exact templates on SecretRef-capable fields.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or revise a focused fix that compares the authored config after `$include` resolution with the resolved audit target, emits `MIGRATION_RECOMMENDED` only for exact `${ENV_VAR}` templates on SecretRef-capable fields, preserves mixed-string plaintext findings, and updates the CLI JSON/text summary plus docs/tests. Broader `$VAR`, auth-profile, and dotenv attribution issues should remain separate follow-ups unless maintainers explicitly rescope this PR.
