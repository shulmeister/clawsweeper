## Summary

Current `main` does not contain the PR’s timeout-start deferral. The cron wall-clock timeout is still armed immediately in `src/cron/service/timer.ts` before isolated work is handed to `runIsolatedAgentJob`, and the embedded runner still waits on session/global lane acquisition with no `onExecutionStart`/`onLaneAcquired` hook in the live code. This means the reported enqueue-time timeout behavior is not implemented on `main`, so the PR should remain open.
