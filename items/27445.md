## Summary

Keep #27445 open. Current main has adjacent completion-routing improvements for active or nested requesters, but it still does not expose the requested native `announceTarget` config default or `sessions_spawn` per-call override. Docs and tests still describe and assert top-level requester completion delivery through the external channel path.

## Best Possible Solution

Keep #27445 open as a core subagent announce policy request. The likely path is to add a public parent/channel announce policy to native `sessions_spawn` and `agents.defaults.subagents`, update the strict config schema and docs, and cover top-level parent-only routing, current channel delivery, active requester steering, nested subagent delivery, and interactions with #33478 callback/thread hardening.
