## Summary

Keep #27445 open. Current main still has no public `announceTarget` parameter or `agents.defaults.subagents.announceTarget` config default for native `sessions_spawn`. The existing implementation routes top-level completion announces through direct/external delivery, while only nested requester subagent sessions get the internal `deliver=false` parent-session injection requested for orchestration. Related #33478 and #8299 are complementary handoff/suppression requests, not canonical replacements.

## Best Possible Solution

Keep #27445 open as a core subagent announce-routing policy request. The best implementation remains a backwards-compatible `announceTarget` policy with default `"channel"` and opt-in `"parent"`, exposed on native `sessions_spawn` and `agents.defaults.subagents`, with schema/docs updates and regression coverage for current channel delivery, top-level parent-only routing, nested subagent injection, ACP `streamTo` separation, and interactions with #33478/#8299.
