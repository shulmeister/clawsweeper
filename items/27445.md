## Summary

Keep #27445 open. Current main has improved sub-agent completion handling, including parent-session steering for active requesters and internal delivery for nested subagents, but it still does not implement the requested `announceTarget` config default or public `sessions_spawn` override for top-level parent-only routing. Docs still describe top-level requester completions as a follow-up agent call with external delivery (`deliver=true`), while nested subagent requesters get `deliver=false`. Related #33478 is complementary, not a full replacement.

## Best Possible Solution

Keep tracking this as a core sessions_spawn/subagent announce policy request until maintainers either implement a public routing option or explicitly decide the current parent follow-up behavior is the intended product direction. A complete implementation would likely add an `announceTarget` or broader announce policy to config types/schema/docs and the `sessions_spawn` schema, then cover top-level parent-only routing, external channel routing, nested subagent routing, and interaction with #33478 thread/callback hardening.
