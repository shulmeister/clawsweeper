## Summary

Current main still does not honor the requested custom session.mainKey routing path: config defaults overwrite custom values to "main", route builders still hardcode DEFAULT_MAIN_KEY, and several channel/outbound paths do not pass a configured main key. The PR discussion is useful and addressed earlier review feedback, but a later P1 review concern about custom main-key guards is still reflected in current main, so this PR should be refreshed rather than closed.

## Best Possible Solution

Keep the PR open and refresh it against current main. The best path is to preserve and normalize session.mainKey in applySessionDefaults(), thread normalizeMainKey(cfg.session?.mainKey) through resolveAgentRoute/buildAgentSessionKey, outbound base-session routing, WhatsApp broadcast, and Telegram conversation routing, then update main-session guard logic such as session-delivery.ts so custom main keys are classified as main. Add focused regression tests for custom mainKey with dmScope: "per-channel-peer", outbound/channel route consistency, and internal delivery guard behavior, then align the conflicting docs around whether session.mainKey is supported or legacy.
