## Summary

Keep open. Current `main` still injects thread history/starter text into the user-role prompt and queued body, so this PR is not implemented on `main`. The proposed move into `extraSystemPrompt` also conflicts with the current security contract that attacker-controlled strings stay out of system-role metadata.
