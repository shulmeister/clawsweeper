## Summary

Current main already implements the Control UI image paste/upload path end to end, and the latest release tag v2026.4.24 contains the same core path. The UI captures pasted, selected, and dropped images, converts them into chat.send attachments, and the gateway normalizes/parses image attachments before dispatch; regression tests cover nested dashboard image payloads and image-only chat.send requests.

## Best Possible Solution

Keep the shipped Control UI and gateway attachment implementation. If image paste/drop still fails on a current release, track it as a new focused bug with current-version reproduction data, browser console output, WebSocket payload evidence, and provider/model details.
