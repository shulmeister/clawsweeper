## Summary

Keep open. Current `main` still lacks macOS app localization plumbing and shipped Simplified Chinese resources, so issue #52028 is not implemented on `60f7a59f5ed581839cfd1d52583449c5aa1f03a4`.

## Evidence

- **macOS app still declares English development region:** `apps/macos/Sources/OpenClaw/Resources/Info.plist` keeps `CFBundleDevelopmentRegion` as `en`, and the bundled permission usage strings are still hard-coded English text.
  - file: [apps/macos/Sources/OpenClaw/Resources/Info.plist:5](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/apps/macos/Sources/OpenClaw/Resources/Info.plist#L5)
- **No packaged localization assets for the macOS target:** The macOS SwiftPM target only copies `Resources/OpenClaw.icns` and `Resources/DeviceModels`; there is no `defaultLocalization`, no `.process`ed string catalog, and no localized resource entry.
  - file: [apps/macos/Package.swift:42](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/apps/macos/Package.swift#L42)
- **Repository has no macOS `.lproj` or string catalog files:** A read-only search for `*.lproj`, `Localizable.strings`, `InfoPlist.strings`, and `*.xcstrings` under `apps/macos` returned no results.
  - command: `find apps/macos \( -name '*.lproj' -o -name 'Localizable.strings' -o -name 'InfoPlist.strings' -o -name '*.xcstrings' \) | sort`
- **Representative macOS UI remains hard-coded English:** Settings tabs are declared with English literals such as `Label("General")`, `Label("Channels")`, and `Label("About")`; onboarding pages also use English `Text(...)` literals such as `"Welcome to OpenClaw"` and `"Meet your agent"`. A repo search found no `NSLocalizedString`, `String(localized:)`, `LocalizedStringKey`, or string-catalog usage in `apps/macos`.
  - file: [apps/macos/Sources/OpenClaw/SettingsRootView.swift:26](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/apps/macos/Sources/OpenClaw/SettingsRootView.swift#L26)
- **Simplified Chinese exists for web UI, not for macOS app:** The Control UI i18n registry already ships `zh-CN`, which shows the feature request is still specifically unresolved for the mac app rather than already solved product-wide.
  - file: [ui/src/i18n/lib/registry.ts:11](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/ui/src/i18n/lib/registry.ts#L11)
