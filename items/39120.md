## Summary

Keep #39120 open. Current main at 82262387651ab270b565f71533ce4937c9c00dc6 still rejects arbitrary `bootstrap-extra-files` basenames, has no `extraBootstrapFiles`/custom context-file config key, and docs/tests continue to document and assert the recognized-bootstrap-basename limitation. The related implementation PR #39174 was closed unmerged, and #39635 was closed as a duplicate of this broader tracker rather than superseding it.

## Best Possible Solution

Keep #39120 open for maintainer/product decision. A direct fix should either widen the bundled `bootstrap-extra-files` contract or add a typed custom bootstrap-file config key, while preserving workspace realpath checks, deterministic ordering, per-file and total bootstrap budgets, subagent/lightweight-context filtering, docs, schema/help artifacts, and focused tests for arbitrary basenames, missing files, traversal, ordering, and truncation. Since #39635 was closed as duplicate, continue using #39120 as the canonical tracker until a design ships or maintainers explicitly move the work to ClawHub/plugin guidance.
