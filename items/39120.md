## Summary

Keep #39120 open. Current main still rejects arbitrary `bootstrap-extra-files` basenames, has no first-class custom bootstrap-file config key, and docs/tests still preserve the recognized-bootstrap-name restriction. The related implementation PR #39174 was closed unmerged, and #39635 is a narrower config request rather than a full replacement for this hook/config arbitrary-file tracker.

## Best Possible Solution

Keep #39120 open for maintainer/product decision. A direct fix should widen the bootstrap file name contract or add a typed custom-file wrapper, relax or replace `VALID_BOOTSTRAP_NAMES` inside the extra-file loader, preserve workspace realpath and size/budget guards, define subagent/light-context filtering for arbitrary files, and update the bundled hook docs plus tests. If maintainers prefer a first-class config route, coordinate it with #39635 and close one issue only after a canonical design is accepted.
