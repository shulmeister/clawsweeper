## Summary

Keep #39120 open. Current main still restricts `bootstrap-extra-files` to recognized bootstrap basenames, has no first-class `extraBootstrapFiles`/`contextFiles` config for arbitrary workspace files, and docs/tests still document and assert the restriction. The related PR #39174 was closed unmerged, while #39635 is a related narrower config proposal rather than a shipped replacement for this broader hook/config request.

## Best Possible Solution

Keep #39120 open for maintainer/product decision. A direct fix should either widen the bundled `bootstrap-extra-files` contract or add a typed first-class custom bootstrap-file config key, preserve workspace realpath and size/budget guards, define subagent/lightweight-context filtering for arbitrary files, and update docs plus tests. Coordinate with #39635 if maintainers choose the config route, then close one item only after a canonical design ships or is explicitly superseded.
