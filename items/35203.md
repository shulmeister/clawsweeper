## Summary

Keep #35203 open. Current main still has isolated hierarchical subagents, plugin lifecycle/runtime seams, per-agent memory/search, and token/cost reporting, but it does not implement the RFC's central capabilities: learned capability profiling/routing, a shared blackboard/pub-sub layer, layered cross-agent memory visibility/promotion, or multi-agent token budget enforcement. The Octo/teleconference and KinthAI discussion is useful prior art, but it is not shipped superseding behavior on current main.

## Best Possible Solution

Keep #35203 open as an umbrella RFC until a maintainer scopes it. The practical path is to split it into focused issues for capability metrics/routing, blackboard or artifact search/query, memory visibility and promotion gates, and token budget aggregation/enforcement. Plugin-first prototypes should use runtime.subagent, taskFlow, context-engine, lifecycle hooks, and memory plugin seams where sufficient; any missing pre-spawn data, usage telemetry, visibility, or enforcement hook should become a narrow plugin API/core issue.
