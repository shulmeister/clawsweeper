## Summary

Keep open. Current main still provides isolated/nested subagents, session visibility controls, usage/cost reporting, and plugin runtime building blocks, but it does not implement the RFC's core capabilities: learned capability profiling, shared blackboard, layered memory visibility/promotion, or token budget enforcement. The issue is active and the related Octo/teleconference/KinthAI comments describe partial or external approaches, not a shipped superseding implementation.

## Best Possible Solution

Keep #35203 open as an RFC until a maintainer scopes it. The best path is to split it into narrower work: capability metrics/routing, blackboard or meeting-artifact search, memory visibility/promotion, and token budget enforcement. Plugin-first prototypes should use runtime.subagent, taskFlow, and lifecycle hooks where sufficient; missing hooks or runtime data needed for these layers should become focused plugin API issues.
