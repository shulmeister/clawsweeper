## Summary

Keep open. The provided GitHub context shows #63096 is still open and unmerged with closing syntax for this issue, so this issue should not be closed before that implementation candidate is reviewed and either merged or closed. Current main also does not prove byte-identical chat, heartbeat, and ACP completion-wake prompts: the normal inbound path builds session-derived Inbound/Group context, while the gateway agent path still forwards only caller-supplied extraSystemPrompt.

## Best Possible Solution

Keep this issue open while maintainers review #63096 or a current-main replacement. Before closing, add or verify diagnostics.cacheTrace coverage proving the intended same-session behavior across normal chat, heartbeat, and ACP completion wakes; if #63096 only fixes completion wakes, split the remaining heartbeat/chat and image-pruning cache churn into narrower follow-ups or extend the core prompt/session-context path accordingly.
