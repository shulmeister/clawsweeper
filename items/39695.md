## Summary

Keep open. Current main has better fresh-onboarding defaults and Discord session-routing plumbing, but it still does not fully close the reported legacy Discord exec/no-session-log failure: an explicit or upgraded `tools.profile: "messaging"` still excludes `exec`, explicit profiles are preserved, and the related legacy migration / silent-tool surfacing work is not present as a complete merged fix on main.

## Best Possible Solution

Keep #39695 open until maintainers ship a targeted fix or diagnostic path. The likely path is a narrow legacy local-onboard migration or doctor/startup warning when SOUL/profile expectations require `exec` but the effective tool policy excludes it, plus regression coverage for a Discord-bound turn that attempts `exec` and persists a JSONL transcript.
