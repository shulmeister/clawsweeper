## Summary

Keep #39695 open. Current main has general session-file plumbing and some silent/incomplete-turn guardrails, but the reported Discord-bound exec gap is still plausible for explicit or legacy `tools.profile: "messaging"` configs: that profile still excludes `exec`, onboarding preserves explicit profiles, and I found no current merged migration or Discord regression proving inbound Discord turns can execute `exec` and persist a JSONL transcript for this scenario.

## Best Possible Solution

Keep #39695 open until maintainers either ship a narrow migration or doctor diagnostic for legacy/explicit messaging-profile configs that expect runtime tools, or add a targeted Discord-bound regression showing an inbound Discord turn can execute `exec` and persist the JSONL session transcript. If messaging profiles are intentionally chat-only, add a clear user-facing diagnostic pointing users to `tools.profile: "coding"`, `tools.alsoAllow`, or `tools.exec`.
