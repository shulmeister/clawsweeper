## Summary

Keep open. Current main still has the reported Linux fnm PATH resolver gap: Linux only adds `current/bin`, while Darwin adds `aliases/default/bin`. The issue remains a real code/docs/testable bug; the open PR #68180 appears to be the current implementation candidate rather than a reason to close the canonical issue.

## Best Possible Solution

Keep #68169 open and resolve it by landing a focused fix that mirrors the Darwin fnm behavior in `resolveLinuxUserBinDirs`: add `$FNM_DIR/aliases/default/bin` and home-relative Linux fnm alias defaults while preserving `current/bin`, plus regression tests for the modern `fnm alias <version> default` case. #68180 is the obvious implementation candidate if it still applies cleanly.
