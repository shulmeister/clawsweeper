## Summary

Current main partially mitigates the adjacent empty-payload symptom, but it does not implement the reporter’s requested routing change. Background exec completion still enqueues a system event and wakes the heartbeat path, and with `heartbeat.isolatedSession: true` the model turn still runs under the `:heartbeat` session key rather than the initiating main session.

## Best Possible Solution

Keep #68386 open. The best maintainer path is to decide the intended contract for async exec completion delivery under `heartbeat.isolatedSession: true`: either route the completion processing back through the initiating session, or explicitly document heartbeat-based processing as expected and adjust issue framing. If changing behavior, add a regression covering a main-session background exec with isolated heartbeat enabled so the completion is not treated as an unrelated heartbeat-only turn.
