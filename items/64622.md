## Summary

Keep open. Current main does not already implement the PR's claimed gateway/A2A routing fixes, and the PR is not cleanly merge-obsolete: it has unresolved P1 review comments plus an unrelated docs-template commit. The related #63723 review also keeps the underlying sessions_send timeout semantics open and explicitly calls for reconciling or replacing this work with a narrower tested fix.

## Best Possible Solution

Keep this PR open for changes rather than closing it. The best path is a narrower PR that fixes true A2A timeout behavior without changing same-agent/session-local timeout semantics, sanitizes stale webchat-backed main-session metadata with focused regression coverage, and handles outbound mirror continuity without overwriting resolved channel-specific routes before persistence. Split the pending-review template docs into a separate docs proposal, and reconcile the sessions_send timeout semantics with the still-open #63723 delivery-receipt versus reply-timeout work.
