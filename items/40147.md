## Summary

Keep open. Current main and v2026.4.24 still force Feishu thread replies from inbound thread_id/root_id even when replyInThread is disabled, so the PR is not implemented or obsolete. It also is not a safe duplicate close: related PR #39507 covers the same routing family but proposes different root_id behavior, while #40147 still has an unresolved review concern about root_id forcing affecting DM quote replies.

## Best Possible Solution

Keep #40147 open for maintainer review, or explicitly consolidate it with #39507 after deciding the intended root_id semantics. A current-main fix should update extensions/feishu/src/bot-content.ts and extensions/feishu/src/reply-dispatcher.ts together, add regression coverage for thread_id-only disabled groups, root_id in normal groups, topic-group/topic-sender routing, and p2p root_id quote replies, then retire the conflicting current test expectation.
