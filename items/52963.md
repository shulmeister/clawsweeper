## Summary

Keep open. Current `main` still does not implement the requested Slack user-account behavior end to end. The Slack docs and runtime both say `userToken` is read-oriented and that writes still prefer the bot token when one is configured, so replies will not switch to the regular Slack user in the reported setup. Inbound message handling is generic over Slack `message` events, but the repo does not document or explicitly add support for Slack's user-level event-subscription path, and the requested no-APP-badge reply path is still absent.
