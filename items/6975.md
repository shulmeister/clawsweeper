## Summary

Keep open. Current main has adjacent Telegram native reply and manual `quoteText` support, and the discussion workaround `channels.telegram.replyToMode: "all"` is useful for enabling native replies, but it still does not implement the requested configurable `channels.telegram.replyContextLength`. The bot auto-reply path still drops quote text before the Telegram API call, and related PRs #43010 and #43734 were closed unmerged.

## Best Possible Solution

Keep #6975 open. The best fix is to add a typed and validated `channels.telegram.replyContextLength` setting, update generated config metadata and Telegram docs, apply the configured limit when deriving outgoing reply quote text for Telegram reply tags, and make the bot delivery path send `reply_parameters: { message_id, quote, allow_sending_without_reply }` when quote text is available. Keep `replyToMode: "all"` documented as a native reply-threading workaround, but not as a replacement for configurable quote excerpt length.
