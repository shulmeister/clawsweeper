## Summary

Keep open. Current main still lacks `channels.telegram.replyContextLength`; Telegram config/schema/docs expose `replyToMode` and `textChunkLimit`, but no quote-length knob. The discussion workaround (`replyToMode: "all"`) is useful for native replies, but it is not equivalent to configurable quote excerpt length. Related PRs #43010 and #43734 were closed unmerged, so they are prior art rather than a shipped fix.

## Best Possible Solution

Keep the issue open. The best fix is to add a typed and validated `channels.telegram.replyContextLength` setting, apply it when deriving Telegram reply quote text, send that text through `reply_parameters.quote` where Telegram supports it, and update Telegram docs/config metadata plus regression tests. Preserve the useful documented `replyToMode: "all"` workaround as an adjacent native-reply control, not as a replacement for configurable quote excerpt length.
