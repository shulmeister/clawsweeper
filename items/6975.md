## Summary

Keep open. Current main does not implement `channels.telegram.replyContextLength`: the Telegram config type, Zod schema, generated metadata/docs search, and user-facing config docs expose `replyToMode` and `textChunkLimit` but no reply-context-length setting. The related PRs #43010 and #43734 were closed unmerged, and the later workaround discussion only covers `replyToMode: "all"`, not the requested configurable quote excerpt length.
