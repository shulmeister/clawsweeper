## Summary

Keep #38775 open. Current main still implements ordered/last-used auth-profile rotation plus reactive cooldown/failover, while Anthropic usage snapshots are wired to usage/reporting surfaces rather than pre-request profile selection. I found no `auth.balancing` config, usage-aware selector, proactive utilization circuit breaker, or async usage cache/polling path.

## Best Possible Solution

Keep #38775 open for maintainer/product decision. A good implementation would add first-class usage-aware auth-profile routing across Gateway request paths: provider-owned usage parsing for Anthropic, core-owned cached per-profile headroom state, proactive utilization thresholds/circuit breakers, shadow accounting for concurrent requests, fallback to current reactive rotation for profiles without usage data, `auth.balancing` schema/help/docs, and focused regression tests for utilization-based profile choice plus cooldown interaction. If maintainers prefer externalization, first add a documented async plugin API for pre-request profile selection and usage-cache integration.
