## Summary

Keep #38775 open. Current main still has ordered/round-robin auth-profile selection plus reactive cooldown/failover, and Anthropic usage snapshots exist only for usage/reporting surfaces. I did not find the requested usage-aware routing strategy, proactive utilization circuit breaker, async usage polling/cache, or `auth.balancing` config surface. The prior ClawSweeper comment correctly identified the same gap, and the related closed issues cover existing multi-key/failover behavior rather than this headroom-aware routing request.

## Best Possible Solution

Keep #38775 open for maintainer/product decision. A good implementation would add first-class usage-aware auth-profile routing across Gateway request paths, likely using provider-owned usage parsing for Anthropic, a core-owned cached headroom selector, proactive thresholds/circuit-breaker behavior, fallback to current reactive rotation for profiles without usage data, `auth.balancing` schema/help/docs, and focused regression tests for utilization-based profile choice and cooldown interaction.
