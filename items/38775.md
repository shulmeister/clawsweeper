## Summary

Keep open. Current main still has provider auth-profile ordering, reactive cooldown/failover, and Anthropic usage reporting, but not the requested usage-aware multi-account routing strategy or `auth.balancing` config surface.

## Best Possible Solution

Keep #38775 open for maintainer/product decision and implementation. A good solution would add a first-class, documented usage-aware auth-profile routing path, likely provider-owned for Anthropic usage parsing but core-owned for Gateway-wide pre-request profile selection, cached usage state, config/schema/docs, and regression tests covering profile choice by utilization/headroom plus fallback to current reactive behavior for profiles without usage data.
