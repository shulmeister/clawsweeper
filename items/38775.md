## Summary

Keep open. Current main still has reactive auth-profile rotation and usage reporting, but it does not implement usage-aware, headroom-based routing or an `auth.balancing` configuration surface for multi-account Anthropic OAuth profiles.

## Best Possible Solution

Keep this issue open for maintainer product/implementation work. A good implementation would likely add a native auth-profile balancing strategy in the Gateway auth-selection path, persist or cache per-profile usage windows safely, keep current reactive cooldown fallback for API-key/non-usage providers, and document any new `auth.balancing` config alongside model-failover/auth docs. A provider-specific hook could help, but bundled Anthropic still lacks usage-aware profile selection on current main.
