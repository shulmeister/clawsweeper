## Summary

Keep open. Current `main` still uses explicit-order or round-robin auth-profile selection with reactive cooldown/failover. Anthropic `/api/oauth/usage` support exists for provider-usage/status reporting, but I found no code path that feeds those utilization windows into request routing or multi-account load balancing.
