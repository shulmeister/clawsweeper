## Summary

Keep PR #66305 open. Current main still has the exact empty-catalog retry behavior reported by #66094: `loadModelCatalog()` clears `modelCatalogPromise` when discovery succeeds with zero models, so the next non-read-only caller can repeat the slow discovery path. I found no negative-cache TTL/token implementation or regression coverage on main; the PR remains a plausible focused bug fix, though it should be rebased onto the current loader shape.

## Best Possible Solution

Keep the PR or an equivalent follow-up open, rebase it onto current main, and implement a bounded empty-catalog negative cache in `src/agents/model-catalog.ts` while preserving import-error retry, `useCache:false`, read-only non-cached loads, config hot-reload cache reset, and stale concurrent load protection. Add focused tests in `src/agents/model-catalog.test.ts` for immediate empty-cache reuse, retry after the window, `useCache:false` bypass/reset, stale concurrent loads, and existing error retry behavior.
