## Summary

Keep #8299 open. Current main still lacks a documented config or per-spawn framework control to suppress sub-agent announce delivery; suppression remains token/model-driven (`ANNOUNCE_SKIP`/`NO_REPLY`) plus internal lifecycle-only flags. The related implementation PRs in the provided context are closed unmerged, and broader routing items do not ship the specific requested skip policy.

## Best Possible Solution

Keep #8299 open until OpenClaw adds a documented config-backed sub-agent announce policy and/or a public `sessions_spawn` parameter. A complete fix should update the TypeBox tool schema, spawn parsing, registry/announce flow contract, config types/zod/generated schema/help, docs, and regression tests for skip behavior plus cleanup/retry bookkeeping. Coordinate with broader announce routing work only if maintainers choose that as the canonical design.
