## Summary

Keep #8299 open. Current main still does not expose a documented config option or public `sessions_spawn` parameter that deterministically suppresses sub-agent completion announcements. The shipped behavior still relies on exact `ANNOUNCE_SKIP` / `NO_REPLY` output or internal completion-message plumbing, and the adjacent routing issues do not supersede this narrower suppress/skip request.

## Best Possible Solution

Keep #8299 open until OpenClaw adds a documented framework-level sub-agent announce suppression control, either as `agents.defaults.subagents` config and/or a public `sessions_spawn` parameter. A complete fix should update TypeScript config types, zod/generated schema/help, the public tool schema/parser, registry and delivery contracts, docs such as https://docs.openclaw.ai/concepts/session-tool, and regression tests proving deterministic skip behavior without relying on `ANNOUNCE_SKIP` or `NO_REPLY`. Coordinate naming with #27445, but do not treat routing-to-parent work as a substitute for a skip/suppress control.
