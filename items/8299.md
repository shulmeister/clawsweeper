## Summary

Current main still does not provide the requested public framework control to suppress sub-agent announcements. `sessions_spawn` has no documented `announce`/`silent`/`announcePolicy` parameter, `agents.defaults.subagents` has no config-backed suppression policy, and the announce flow still suppresses only via model output tokens or internal lifecycle-only flags.

## Best Possible Solution

Keep #8299 open until OpenClaw adds a documented config-backed sub-agent announce policy and/or a public `sessions_spawn` parameter, or maintainers explicitly supersede it with a broader announce routing design. The implementation should update the tool schema, config types/zod/generated schema/help, public docs, and regression tests for cleanup/retry behavior.
