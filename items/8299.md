## Summary

Keep #8299 open. Current main still does not provide a documented config option or public `sessions_spawn` parameter to suppress sub-agent completion announcements at the framework level. Suppression remains driven by child output tokens (`ANNOUNCE_SKIP` / `NO_REPLY`) plus internal lifecycle flags, and the related implementation PRs were closed unmerged.

## Best Possible Solution

Keep #8299 open until OpenClaw adds a documented framework-level sub-agent announce suppression control, either as `agents.defaults.subagents` config and/or a public `sessions_spawn` parameter. A complete fix should update the tool schema and parser, spawn/registry/announce flow contracts, config types/zod/generated schema/help, docs, and regression tests for skip behavior and cleanup bookkeeping. Coordinate with broader announce routing work only if maintainers choose that as the canonical design.
