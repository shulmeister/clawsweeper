## Summary

Keep PR #69690 open. Current main has adjacent gateway-bindable registry reuse, but it still does not implement this PR's specific omitted-coreGatewayHandlers path: the loader cache key includes gateway method names, gateway startup supplies them, later helper/runtime loads can request gateway binding without supplying them, and current active registry state has no recorded method-name set to reconstruct compatibility. The PR discussion includes concrete repro logs, root-cause analysis, tests, and follow-up commits addressing the bot-flagged default-mode inheritance risk.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it or apply an equivalent patch on current main that persists the active gateway method-name set, uses it only for explicit gateway-bindable compatibility checks, and adds regression coverage for omitted handlers, mismatched method sets, and default-mode non-inheritance.
