## Summary

Keep open. Current main still generates Feishu synthetic reaction message IDs with a `:reaction:EMOJI:uuid` suffix, preserves that suffixed ID as the inbound message context, and passes raw message IDs into multiple Feishu API calls. PR #39558 remains relevant to linked issue #34528, but it needs a refresh for current main because the Feishu reply surface has moved and newer streaming-card reply paths are still uncovered.

## Best Possible Solution

Refresh this PR or replace it with a current-main patch that centralizes Feishu reaction-suffix stripping in the Feishu plugin and applies it at every Feishu message-ID API sink: send/reply/get/edit/update, media resource and media replies, reactions, typing indicators, thread-reply/action paths, and streaming cards. Add regression tests for the helper and at least one reply/API boundary using a suffixed `om_xxx:reaction:EMOJI:uuid` ID, then close #34528 when the fix lands.
