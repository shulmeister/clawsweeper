## Summary

Keep #39558 open. Current main still creates Feishu synthetic reaction message IDs and still passes raw message IDs into multiple Feishu API boundaries. The PR is useful and linked to the still-open canonical bug #34528, but it needs a current-main refresh because its helper is not present, `send-message.ts` no longer exists, and newer API sinks such as streaming-card and pin paths are not covered.

## Best Possible Solution

Refresh #39558 or replace it with an equivalent Feishu plugin patch that centralizes a reaction-message-ID normalizer and applies it at every Feishu API-facing message ID boundary, while preserving suffixed IDs for internal dedupe/session identity. Add helper tests plus at least one boundary regression test that passes a `:reaction:<emoji>:<uuid>` ID into an API call and verifies the clean `om_...` ID is sent.
