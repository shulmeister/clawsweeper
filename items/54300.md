## Summary

Keep open. Current main still has the two concrete hardening gaps: Docker label names are interpolated into a Docker Go template without validation, and the thread-ownership plugin still builds the ownership URL with raw channel/thread values. The TLS/private-WS and Control UI dangerous flags are documented, tested break-glass behavior, so that part needs maintainer security/product judgment rather than automated closure.

## Best Possible Solution

Keep this issue open for maintainer security triage. Land a narrow hardening change that validates Docker inspect label names in readDockerContainerLabel and encodes or strictly validates thread-ownership channel/thread path segments with regression tests. Separately decide whether the private-WS and Control UI break-glass policy or error wording should be changed, since that behavior is currently documented, tested, and audited as intentional.
