## Summary

Current main already fixes the reported iMessage final-reply failure. The monitor resolves the iMessage account once in scope, the reply dispatcher passes only the account id into the delivery helper, and the delivery/send helpers do not depend on an out-of-scope accountInfo symbol. The same implementation is present in the v2026.4.24 release tag.

## Best Possible Solution

Close this issue as already implemented on main and shipped in v2026.4.24. If a user can still reproduce a final-reply failure on a current release, it should be filed as a new issue with fresh gateway logs, imsg version, config, and whether the failing target is a handle, chat_id, or media reply.
