## Summary

Keep open. Current main still lacks multi-gateway cron coordination: cron config/docs expose only local scheduler settings, the scheduler uses process-local locks plus persisted running markers, and timer execution is performed by each gateway instance without a leader-election or backend-backed lock step. The related #13582 was a closed local scheduler reliability bug, not a canonical tracker for HA coordination.

## Best Possible Solution

Keep #13613 open until maintainers either implement explicit cron coordination (leader election or distributed per-job locks with config, docs, health/CLI visibility, and multi-process tests) or add a public scheduler/lock-provider plugin seam that makes a ClawHub plugin feasible. Treat #13582 as a separate closed scheduler-firing reliability issue, not the canonical tracker for this HA request.
