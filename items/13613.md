## Summary

Keep #13613 open. Current main still has gateway-local cron scheduling with local config/schema/status, a module-local lock chain, and no distributed leader election, lock-provider config, or cross-gateway due-job ownership surface. The existing steipete Codex review comment already kept this open and correctly distinguishes related #13582 as local scheduler reliability work, not HA cron coordination.

## Best Possible Solution

Keep #13613 open until OpenClaw either implements HA cron coordination in the core cron scheduler or adds a documented scheduler/lock-provider plugin API. The fix should cover config/schema/docs, due-job ownership semantics across gateways, leader or lock health visibility in status/CLI, and multi-process or multi-node regression coverage. Treat #13582 as shipped/local scheduler reliability work, not as the canonical tracker for this HA request.
