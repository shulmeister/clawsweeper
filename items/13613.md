## Summary

Keep open. Current main still has only local cron scheduling/concurrency controls, not distributed leader election or backend-backed locks for HA/multi-gateway cron execution. The related #13582 cron reliability report is a separate closed scheduler-firing issue, not a canonical tracker for the requested multi-gateway coordination work.

## Evidence

- **Cron config has no coordination backend:** CronConfig exposes enabled, store, maxConcurrentRuns, retry, webhook, retention, runLog, and failure alert/destination settings, but no coordination/leader-election/Redis/etc. configuration surface.
