## Summary

Keep #13613 open. Current main still has local cron scheduling with process-local/shared-file coordination only; it does not provide multi-gateway leader election, Redis/etcd/DynamoDB locks, a coordination config surface, health/CLI leader visibility, or a public scheduler lock-provider plugin seam. The existing maintainer/Codex issue comment also kept this open, and related #13582 was a closed local scheduler reliability bug rather than a canonical HA-coordination tracker.

## Best Possible Solution

Keep #13613 open until maintainers either implement explicit HA cron coordination in core (leader election or distributed per-job locks with config schema, docs, health/CLI visibility, and multi-process or multi-node tests) or first add a public scheduler/lock-provider plugin API that makes a ClawHub implementation feasible. Treat #13582 as a separate closed local scheduler reliability bug, not the canonical tracker for this HA request.
