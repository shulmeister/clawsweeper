## Summary

Keep #13613 open. Current main still implements cron as a Gateway-local scheduler with process-local serialization and shared file persistence, but no distributed lease, leader election, coordination backend, lock TTL, HA status, or CLI leader surface. The existing discussion already distinguishes related #13582 as local scheduler reliability work rather than HA cron coordination, and current source/docs still match that assessment.

## Best Possible Solution

Keep #13613 open until OpenClaw either implements distributed cron ownership in the core scheduler or adds a documented scheduler/lock-provider extension API. A complete fix should define config/schema/docs, due-job ownership semantics across Gateways, leader or lock health in status/CLI, and multi-process or multi-node regression coverage. Treat #13582 as shipped local scheduler reliability work, not the canonical tracker for HA coordination.
