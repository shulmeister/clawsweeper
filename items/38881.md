## Summary

Keep PR #38881 open. Current main still does not auto-switch the final agent reply run to `agents.defaults.imageModel` when image input reaches a text-only primary/default model. The current implementation uses `imageModel` for media-understanding/image tooling, while `runPreparedReply` still receives the selected/default `provider` and `model`. The PR targets a real gap, but it needs maintainer/author follow-up before merge because the review discussion and current webchat offload path require a more complete implementation.

## Best Possible Solution

Keep #38881 open for a focused production fix in the reply/model-selection path. The fix should detect real inbound image inputs after current normalization, including inline webchat images, offloaded media refs, `MediaPath(s)`, `MediaUrl(s)`, and image MIME/type data; resolve `agents.defaults.imageModel.primary` and fallbacks through the real alias index and model catalog; switch the final `runPreparedReply` model only when the active model is positively known not to support images; respect explicit heartbeat/session/channel choices; coordinate with media understanding to avoid duplicate analysis; and add regression coverage proving the final agent run receives the configured vision model.
