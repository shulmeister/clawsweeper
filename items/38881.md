## Summary

Keep open. Current main has related `agents.defaults.imageModel` media-understanding routing, but it still does not implement this PR's specific main-agent model auto-switch before `runPreparedReply`. The PR discussion remains materially relevant, and there is no high-confidence close reason or canonical superseding item in the provided context.

## Best Possible Solution

Keep the PR open for maintainer judgment. Either land a corrected implementation that switches the main reply model only when that is truly the desired behavior, before media understanding and with robust image/fallback/override handling, or explicitly supersede it with docs/tests clarifying that `agents.defaults.imageModel` is a preprocessing/media-understanding route rather than a main-run model switch.
