## Summary

Keep PR #38881 open. Current main still lacks the proposed auto-switch for the final agent reply run: `agents.defaults.imageModel` is used for media-understanding/image-tool routing, while `runPreparedReply` still receives the selected/default `provider` and `model`. The PR targets a real gap, but the existing review discussion identifies remaining design issues that should be addressed before merge.

## Best Possible Solution

Keep #38881 open for a focused production fix around `src/auto-reply/reply/get-reply.ts`: detect real inbound image attachments after normalization, resolve `agents.defaults.imageModel` primary/fallbacks through the real alias index and catalog, switch the final run before or in coordination with media understanding to avoid duplicate analysis, respect explicit heartbeat/session/channel choices, and add a regression test proving `runPreparedReply` receives the configured image model.
