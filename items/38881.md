## Summary

Current main still does not implement this PR's central behavior: inbound image messages do not auto-switch the main agent run from a text-only selected reply model to `agents.defaults.imageModel`. Main only uses `imageModel` in media-understanding/tool paths, so this PR remains materially relevant, though its implementation still needs maintainer review for the bot-raised design concerns.

## Best Possible Solution

Keep this PR open for maintainer review. A production fix should live around `src/auto-reply/reply/get-reply.ts` and switch the main run only when normalized inbound media contains actual images and the current selected model is positively known to lack vision. It should run early enough to avoid duplicate media-understanding work, thread the real alias index, respect heartbeat/session/channel overrides, handle `imageModel` fallbacks and URL-only media, and add focused `get-reply` regression tests proving `runPreparedReply` receives the image model.
