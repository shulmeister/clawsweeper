## Summary

Keep PR #38881 open. Current main still does not implement the requested final agent-run auto-switch from a text-only active/default model to `agents.defaults.imageModel` when image input is present. Main has related media-understanding and WebChat offload work, but that does not replace the final `runPreparedReply` model selection path, and the PR still needs follow-up for the remaining review concerns before it would be mergeable.

## Best Possible Solution

Keep #38881 open for a focused production fix in the reply/model-selection path. The best implementation should detect real normalized inbound image inputs, resolve `agents.defaults.imageModel.primary` and `fallbacks` through the same alias index/catalog used by normal model selection, switch the final `runPreparedReply` model only when the active model is positively known not to support image input, respect heartbeat/session/channel overrides, avoid duplicate media-understanding work, and add regression coverage proving the final agent run receives the configured vision model.
