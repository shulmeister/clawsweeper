## Summary

Keep open. Current main has useful adjacent strict-agentic safeguards for GPT-5-family OpenAI/OpenAI-Codex plan-only turns, but it still does not implement the issue's requested atomic run.begin/run.exec/run.commit-or-run.abort contract, mandatory terminal transaction state, or machine-readable abort blocker across agent run surfaces.

## Best Possible Solution

Keep #17213 open as a core reliability/product-contract item. The best path is a maintainer-approved design that extends the shared agent event contract and all run emitters with a backward-compatible transaction lifecycle: begin intent, execution evidence, mandatory commit or abort terminal state, and structured abort/blocker details, plus gateway/client/docs/tests that prevent user-visible completion from masquerading as committed execution.

## Risks / Open Questions

- The issue body still lacks a concrete reproduction trace, so maintainers need to define the exact product contract and migration path before implementation.
- Adding mandatory transaction lifecycle semantics would affect shared agent events, gateway chat/session finalization, embedded Pi, ACP/command emitters, clients, docs, and tests.
- Current strict-agentic behavior is intentionally model-lane specific and has an explicit default opt-out, so treating it as the full fix would close a broader reliability/API design request prematurely.
