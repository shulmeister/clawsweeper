## Summary

Keep open. Current main has adjacent reliability behavior, but it still does not provide the requested atomic run.begin/run.exec/run.commit/run.abort contract, mandatory commit-or-abort terminal state, or machine-readable abort blocker for announced pre-action intent. The existing maintainer-side Codex review comment on the issue also explicitly kept it open, and the related #40710 PR is an open, narrower false-start gate rather than a shipped replacement.

## Best Possible Solution

Keep #17213 open as a core reliability/product-contract item. The best path is a maintainer-approved design that extends the shared agent event contract, gateway session/chat finalization, embedded and ACP run emitters, clients, docs, and tests so announced intent must reach a mandatory commit or abort terminal state with structured abort/blocker details. Treat #40710 as related input, not as the canonical replacement unless it lands with the broader contract or a new canonical issue supersedes this one.
