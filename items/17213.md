## Summary

Keep #17213 open. Current main has narrower safeguards for lifecycle end/error fallback, stale run-context cleanup, and strict-agentic plan-only recovery for supported GPT-5 OpenAI/Codex runs, but it still does not implement the requested atomic run.begin -> run.exec -> mandatory run.commit/run.abort contract with structured abort blockers across the shared agent event and Gateway protocol surfaces. The existing maintainer discussion also explicitly kept the issue open, and related PR #40710 was closed unmerged rather than shipped.

## Best Possible Solution

Keep #17213 open as a core reliability and product-contract item. The best path is a maintainer-approved, backward-compatible lifecycle design that defines explicit begin intent, execution evidence, mandatory commit or abort terminal state, and structured abort/blocker details across AgentEvent types, Gateway protocol/schema, emitters, plugin or harness-facing surfaces, clients, docs, and regression tests. Treat #40710 and #55721 as useful prior art and narrower fixes, not superseding closure evidence.
