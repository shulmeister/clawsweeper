## Summary

Keep open. Current main has useful adjacent reliability work, including strict-agentic blocked exits, structured tool/item events, chat abort state, and terminal lifecycle end/error handling, but it still does not implement the requested atomic run.begin/run.exec/run.commit/run.abort contract, mandatory commit/abort terminal state, or machine-readable abort blocker for announced pre-action intent. The prior Codex review comment on the issue also kept it open for the same product gap, and the related #40710 PR is an unresolved narrower execution-gate proposal rather than a shipped or canonical replacement.

## Best Possible Solution

Keep #17213 open as a core reliability/product-contract item. The best path is a maintainer-approved lifecycle design that updates the shared agent event contract, gateway session state, chat abort/finalization behavior, embedded and CLI run emitters, and client/docs/tests so every announced intent reaches a mandatory commit or abort terminal state with a structured blocker when aborted. Related #40710 can inform false-start detection, but it should not be treated as the canonical replacement for this broader transaction contract.
