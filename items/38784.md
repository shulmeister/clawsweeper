## Summary

Keep open. Current main no longer force-overwrites an explicit `compat.supportsUsageInStreaming` override, but it still defaults Azure OpenAI `openai-completions` endpoints to `supportsUsageInStreaming: false`. That means the issue's reported default behavior remains plausible for manual or legacy Azure chat-completions configs, and the current tests do not prove the Azure default was fixed.
