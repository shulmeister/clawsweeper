## Summary

Keep #12198 open. Current main still does not provide the requested shared Markdown validation contract for prompt-bearing Markdown. Existing safeguards cover path containment, size limits, frontmatter parsing, installer code scanning, Skill Workshop-generated content, and external web/content wrapping, but bootstrap Markdown, extra bootstrap Markdown, SKILL.md discovery/read paths, and native ClawHub skill installs still accept or expose Markdown without validation such as HTML-comment rejection.

## Best Possible Solution

Keep the issue open until OpenClaw has a shared prompt-bearing Markdown validation contract with tests and docs. Wire it into workspace bootstrap ingestion, extra bootstrap files, SKILL.md discovery/model-visible read paths, and native ClawHub skill install/update handling; coordinate any separate ClawHub publish-side policy as a follow-up boundary decision.
