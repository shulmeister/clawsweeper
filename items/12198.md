## Summary

Keep #12198 open. Current main still has adjacent prompt-injection and install-safety work, but it does not provide mandatory Markdown validation for prompt files. Workspace bootstrap Markdown is injected after boundary/size guards and truncation only, SKILL.md discovery/loading checks path and size/frontmatter only, the built-in install scanner excludes Markdown, ClawHub skill install checks only for a SKILL.md root before copying, and Skill Workshop's generated-skill scanner is local to that plugin with a limited ruleset that does not cover HTML comments.

## Best Possible Solution

Keep this open until OpenClaw has a shared prompt-Markdown validation contract, with tests and docs, wired into workspace bootstrap ingestion, skill discovery/loading, skill install/ClawHub install or publish policy, and any plugin hook needed for third-party enforcement before prompt Markdown reaches the model. The initial policy should cover high-confidence cases such as HTML comments and should define warn/block behavior explicitly.
