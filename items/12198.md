## Summary

Current `main` no longer auto-appends `SKILL.md` bodies into the system prompt; it only publishes a catalog of skill name/description/location and tells the agent to explicitly read one selected skill file. There is also partial protection for workspace-generated skills via `skill-workshop` content scanning. But I did not find a repo-wide markdown-validation pipeline for prompt-bearing markdown files, nor a ClawHub publish-time gate. Because the issue’s main skill-file premise is outdated while the broader feature request is still not implemented, there is not enough evidence to close it conservatively.
