## Summary

Keep #12198 open. Current main still has adjacent safety work, but it does not implement mandatory Markdown validation for prompt files: SKILL.md and workspace bootstrap Markdown are read with path/size/frontmatter guards only, the built-in scanner excludes Markdown files, ClawHub skill install copies archives after checking for SKILL.md, and Skill Workshop's scanner covers only generated content with a limited rule set.

## Best Possible Solution

Keep the issue open until OpenClaw has a shared prompt-Markdown validation contract and wires it into skill loading/install, workspace bootstrap/context-file ingestion, and ClawHub publish/install policy, or exposes a documented plugin hook that can enforce the same behavior before prompt Markdown reaches the model.
