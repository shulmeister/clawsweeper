## Summary

Keep #12198 open. Current main has adjacent prompt-safety and skill-install controls, but it still does not implement the requested shared prompt-bearing Markdown validation contract, such as rejecting HTML comments before workspace bootstrap files, extra bootstrap files, or SKILL.md content can reach prompt/read paths. The existing code safety scanner is scoped to JavaScript/TypeScript-like skill code and ClawHub native skill install still only verifies that a SKILL.md-like root exists before copying the folder.

## Best Possible Solution

Keep the issue open until OpenClaw has a shared prompt-Markdown validation contract with tests and docs, wired into workspace bootstrap ingestion, extra bootstrap hooks, SKILL.md discovery/read paths, and native ClawHub skill install or publish policy. The existing realpath/size guards, external-content wrappers, and code-oriented skill scanner can inform the implementation, but they do not replace a validator for Markdown that is used as prompt material.
