## Summary

Keep open. Current main still does not implement the PR's requested Feishu `mediaLocalRoots` config surface or account-config fallback. Core now injects scoped runtime roots for normal outbound media sends, which helps the linked local-file failure path, but that is separate from this PR's explicit Feishu channel/account config request, and the submitted patch is not merge-ready against the current `mediaAccess`/`readFile` safety contract.

## Best Possible Solution

Keep #39523 open, but require a revised Feishu-owned fix or a superseding PR. The fix should add the top-level/account `mediaLocalRoots` field only if that remains the desired config contract, update generated channel metadata and public Feishu docs, define merge/override behavior with injected outbound roots, route any supported `"any"` mode through the current `mediaAccess`/`mediaReadFile` safety path, and add regression tests for configured arrays, `"any"`, and normal outbound sends.
