## Summary

Keep open. Current main still does not implement Feishu `mediaLocalRoots` as a channel/account config field, and the exact PR is not ready against current media-access behavior because regular sends inject explicit roots and `localRoots: "any"` now requires a readFile-backed path.

## Best Possible Solution

Keep the PR open, but require a revised Feishu-owned fix or a superseding PR. The fix should add the config field to Feishu schema, generated metadata, and docs; define whether configured roots merge with or override agent-scoped outbound roots; route Feishu media through the current `mediaAccess`/`mediaReadFile` contract or restrict config to explicit directory arrays; and add regression tests for account-config roots, `"any"` only if still supported, and the normal outbound message-send path.
