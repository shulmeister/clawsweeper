## Summary

Keep #71350 open. Current main already implements the narrow per-agent `thinkingDefault` schema/runtime path and flat per-agent inference params through `agents.list[].params`, including `temperature` and `maxTokens`. The broader requested surface is still not complete: there is no nested `agents.list[].models` per-agent per-model merge layer, and no `thinkingStrategy` rules/escalation/AGENTS.md-driven adaptive thinking implementation. The related #64792 was closed into this issue, while PR #44126 remains an open related implementation discussion, not shipped replacement behavior.

## Best Possible Solution

Keep #71350 open as the canonical product/design tracker. The best path is to acknowledge that `agents.list[].thinkingDefault` and flat `agents.list[].params` already ship, decide whether OpenClaw should add a true `agents.list[].models` merge layer or just improve docs around the existing flat params, and design adaptive thinking as an explicit core agent-loop policy or approved tool/API surface with schema, generated config help, docs, and tests for rule matching, provider-supported thinking levels, and failure escalation.
