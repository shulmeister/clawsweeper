## Summary

Keep this issue open. Current main partially satisfies it with per-agent `thinkingDefault` and flat `agents.list[].params` overrides, but the requested per-agent per-model `agents.list[].models` map and `thinkingStrategy` rules/escalation/AGENTS.md policy surfaces are still absent. The related #32496 and #44126 items cover adjacent parameter-forwarding and adaptive-thinking work, but neither supersedes the full remaining request.

## Best Possible Solution

Keep this issue open as the canonical product/API discussion, or deliberately split it into narrower follow-ups. Preserve the shipped `agents.list[].thinkingDefault` and flat `agents.list[].params` behavior, then decide whether core should add a per-agent per-model `models` override map and a `thinkingStrategy` API for message rules, failure escalation, and AGENTS.md-sourced policies. Coordinate broad parameter forwarding with #32496 and adaptive-thinking controls with #44126.
