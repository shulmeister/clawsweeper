## Summary

Current main still has the reported schema gap: Slack exposes and dispatches `download-file`, and the Slack dispatch/runtime paths require `fileId`, but the shared message tool schema does not include `fileId` and Slack's message-tool discovery contributes no Slack-specific schema property for it. The same gap is present in the latest release tag `v2026.4.24`. The linked PR #45626 appears to target this exact issue, but it is open/unmerged, so this is not a conservative auto-close.
