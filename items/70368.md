## Summary

Keep this PR open. Current main still has the reported loopback warning behavior, so the PR is not obsolete or implemented. The PR remains a plausible implementation candidate for #70357, but its latest effective-bind classification still needs maintainer review because `auto` and `custom` can resolve or fall back to non-loopback listeners.

## Best Possible Solution

Keep this PR open and request a corrected, maintainer-reviewed implementation. The safest path is to downgrade only definitely local-only exposure, fail closed for ambiguous or fallback-prone `auto`/`custom` binds, keep `trusted_proxies_missing`, `bind_no_auth`, `loopback_no_auth`, and Control UI origin checks semantically coherent, add regression coverage for loopback, lan, custom loopback, custom exposed, and auto/container or fallback behavior, and update the public audit-check docs if the severity taxonomy changes.
