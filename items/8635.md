## Summary

Keep #8635 open. Current main still does not expose structured raw token usage through the agent-facing `session_status` tool; it only returns a status-card text payload. Separate Gateway usage APIs and transcript parsers exist and could be reused, but they do not satisfy the requested `session_status` contract.

## Best Possible Solution

Keep the issue open until maintainers either add a structured `usage` payload to `session_status` or intentionally define and document a different supported agent-facing usage API. A conservative implementation should reuse the existing transcript/session usage parsers and Gateway usage result types, preserve current session visibility checks, define stable fields for session totals, last turn, cost, and provider budget/window data, then add focused `session_status` tests plus docs updates for the public session tools and token-use pages.
