## Summary

Keep #8635 open. Current main still does not expose structured per-session token usage through the agent-facing `session_status` tool. It returns a `/status`-style text card plus minimal metadata, while raw session usage exists on separate Gateway usage APIs and transcript parsers that could be reused for a future implementation.

## Best Possible Solution

Keep the issue open until maintainers either add a structured `usage` payload to `session_status` or define a different supported agent-facing usage API. A conservative implementation should reuse the existing transcript/session usage parsers and Gateway usage types, define the stable response contract for session totals, last turn, cost, and quota/budget fields, then add focused tests for `session_status` plus docs updates on https://docs.openclaw.ai/concepts/session-tool and https://docs.openclaw.ai/reference/token-use.
