## Summary

Keep #8635 open. Current main still exposes `session_status` as a rendered `/status`-style card with minimal metadata, not a structured token/cost usage API for agents. The related #39400 is adjacent usage-window instability, not a canonical replacement for this broader raw session/last-turn usage request.

## Best Possible Solution

Keep the issue open until OpenClaw adds a documented structured agent-facing usage surface. The likely path is to extend `session_status.details` or add an equivalent supported tool that reuses `sessions.usage` and transcript usage parsing, with `usage.session`, `usage.lastTurn`, cost, budget/window availability/source metadata, and regression tests/docs while preserving the current visibility checks.
