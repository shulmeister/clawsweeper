## Summary

Keep #8635 open. Current main still does not expose raw structured token/cost usage through the agent-facing `session_status` tool. It shows a `/status`-style text card, with transcript-backed fallback improving display accuracy, while structured per-session usage remains on separate Gateway methods rather than the minimal/default tool surface requested by the issue.

## Best Possible Solution

Keep the issue open until OpenClaw adds a documented structured agent-facing usage API. The likely path is either extending `session_status` with stable `usage.session`, `usage.lastTurn`, cost, provider-window availability/source fields, and tests, or adding an equivalent supported tool that reuses the existing Gateway `sessions.usage` and transcript usage parsers while preserving session visibility checks.
