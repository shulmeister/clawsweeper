## Summary

Keep #23585 open. Current main still requires the configured trusted-proxy identity header and returns `trusted_proxy_user_missing` when it is absent; there is no accepted `defaultUser` or `fallbackUser` field in the public config type, zod schema, generated schema, docs, or tests. The issue discussion also distinguishes this from #25293 and #17761, so there is no shipped fix or canonical duplicate to close against.

## Best Possible Solution

Keep #23585 open. If maintainers accept the feature, implement an explicit trusted-proxy fallback/default user option across `GatewayTrustedProxyConfig`, zod and generated schemas, config help/docs/security audit guidance, and auth tests; only apply it after trusted proxy source and required-header checks pass, and make `allowUsers` behavior explicit. If maintainers reject fallback identity on security grounds, document Cloudflare Zero Trust IP-whitelist/no-identity WebSocket setups as unsupported instead of closing this as implemented.
