## Summary

Keep open. Current `main` already enforces auth cooldown windows, but it still emits warn-level `model-fallback/decision`, `embedded run failover decision`, and `auth profile failure state updated` logs on each repeated failure/update with no throttling or coalescing, so the reported log-spam behavior is not implemented away.

## Evidence

- **Cooldown windows exist, but they are only used to keep retry windows immutable:** `computeNextProfileUsageStats` preserves active cooldown/disable windows instead of extending them, which addresses repeated backoff growth but does not suppress repeated observations.
