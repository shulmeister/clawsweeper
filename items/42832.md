## Summary

Keep open. Current main still has at least two core Nostr DM delivery failures the PR is trying to fix: the Nostr bus still passes an array to `SimplePool.subscribeMany()` even though nostr-tools 2.23.3 expects a single `Filter`, and `startNostrGatewayAccount()` still resolves immediately, which the channel manager treats as a stopped/crashed account and auto-restarts. The missing-`sendMedia` part is partly superseded by current generic text fallback behavior, and the PR's media URL fallback needs revision because Nostr is documented as no-media and the PR review identified a local-path leak risk.

## Best Possible Solution

Keep the PR open, rebase it onto current main, retain the singular `subscribeMany()` filter fix and the long-lived `startAccount()` lifecycle fix, add current-file regression tests for both, and either drop the Nostr `sendMedia` fallback or implement a public-URL-only/text-safe fallback aligned with the documented no-media Nostr limitation.
