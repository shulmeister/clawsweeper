## Summary

Current main has structured cron APIs (`cron.list`, `cron.status`, `cron.runs`) and the agent-facing `cron` tool surface, but the tool is still marked owner-only and embedded isolated cron agent runs are explicitly started with `senderIsOwner=false`. That means the reported autonomous-agent introspection gap is still real for the default embedded cron-run path.

## Risks / Open Questions

- CLI-provider cron runs pass owner auth differently, so some environments may already reach the cron tool. The unresolved gap is the embedded/default autonomous cron-run path.
- If maintainers consider non-owner access to cron control-plane data too broad, the likely fix is a narrower read-only introspection surface rather than exposing full `cron` management to all agents.
