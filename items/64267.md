## Summary

Keep this open. Current main has several relevant mitigations, including commentary-phase suppression, canonical final-message reasoning sanitization, and MiniMax thinking disablement, but the specific streaming regex gap called out in the issue discussion is still present: the streaming scanner used for partial/block delivery does not accept thinking tags with attributes, while the canonical sanitizer does.

## Best Possible Solution

Keep the issue open and land a narrow core fix that makes the streaming `THINKING_TAG_SCAN_RE` share the canonical reasoning-tag grammar, including antml prefixes and attributes. Add focused streaming regression tests for attributed `<think>`/`<thinking>`/`<antml:thinking>` chunks across boundaries, while keeping the existing commentary-phase and MiniMax mitigations.

## Evidence

- **Issue discussion identified a streaming regex gap:** A commenter traced the leak to the streaming path using a stale thinking-tag scanner that missed antml-prefixed tags and tags with attributes, and the reporter later said the production build still had the old regex after upgrading to 2026.4.12. That is directly relevant to the remaining code path on main.
