## Summary

Keep #20230 open. Current main covers only part of the request: Control UI resolves and renders an active assistant name/avatar, and user bubbles have distinct styling. The remaining requested behavior is still missing: there is no per-agent bubble color contract, and rendered assistant chat groups still use one active assistant identity rather than per-message/per-agent attribution.

## Best Possible Solution

Keep the issue open until maintainers implement or explicitly reject per-agent chat styling and assistant attribution. A complete implementation likely belongs across Control UI and the shared chat/session metadata contract: persist assistant agent identity per message or group, add a validated optional color/styling field or render hook, render assistant-specific CSS variables/classes, and add focused tests/docs for mixed-agent and historical-session attribution.
