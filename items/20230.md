## Summary

Keep #20230 open. Current main already has active assistant name/avatar display and distinct user-vs-assistant bubble styling, and related issues #13206, #22199, and #53438 closed around that identity/name/avatar work. The central remaining request here is still missing: no per-agent chat bubble color contract, no assistant identity/style stored per message or group, and mixed-agent assistant messages still render through one active assistant identity. The existing April 26 Codex review comment by steipete also explicitly kept this open for the same remaining gap.

## Best Possible Solution

Keep the issue open until maintainers either implement or explicitly reject per-agent chat styling and per-message assistant attribution. A complete fix likely belongs in the Control UI plus shared chat/session contracts: persist assistant identity on messages or message groups, add a validated optional color/styling field or render hook, render assistant-specific CSS variables/classes, and add focused tests for mixed-agent and historical-session attribution.
