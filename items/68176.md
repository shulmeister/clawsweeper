## Summary

Current main still has the reported behavior: CLI version commit resolution prefers live git metadata before baked build-info, and there is no build-time commit define on main. PR #68176 is still a plausible focused fix for #68148 rather than obsolete, duplicated, or out of scope.

## Best Possible Solution

Keep PR #68176 open. The best path is to land or update a narrow implementation that bakes the build commit into the bundled CLI, reads it after env overrides but before live git, preserves dev-from-source live git fallback, and keeps focused regression coverage for injected-vs-live priority, env override, fallthrough, and cache behavior. Close #68148 only after that behavior is on main.
