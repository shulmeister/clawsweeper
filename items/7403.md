## Summary

Keep open. Current main does not appear to implement the requested Private/Secret/Incognito mode: there is no per-session/channel persistence-off flag in the gateway session protocol or session config, transcript and session metadata paths still write to disk, and workspace bootstrap files are still loaded/injected by default. The related #12242 discussion reinforces that the remaining request is a program-level guarantee, not just prompt guidance or after-the-fact pruning.

## Evidence

The comment by yipu3 adds a concrete per-session/per-channel `ephemeral: true` requirement to prevent JSONL session history from being written and cites #12242 as the same concern.
