## Summary

Current main still does not implement the requested Private/Secret/Incognito mode. #7403 remains the canonical open tracker after #12242 was closed as the duplicate privacy/persistence request, and the current gateway protocol, session config/store, transcript helpers, workspace bootstrap, docs, and UI session controls still have no per-session or per-channel private/ephemeral flag that prevents disk persistence and suppresses workspace/memory context reads.

## Best Possible Solution

Keep #7403 open as the canonical product request. A complete fix should add an explicit private/ephemeral session mode across the Gateway protocol, session config/store, transcript writing, workspace bootstrap and memory loading, Mission Control/channel controls, and tests proving that no session history or workspace memory files are read or written while the mode is active.
