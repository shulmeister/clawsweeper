## Summary

Current main still does not implement the requested Private/Secret/Incognito mode. #7403 remains the canonical tracker after #12242 was closed as the duplicate privacy/persistence request, and the related #52478 private-mode scaffolding PR is still not present on main. The current protocol, config, UI, session persistence, transcript writing, and workspace bootstrap paths still lack a per-session or per-channel private/ephemeral control that suppresses workspace/memory reads and prevents disk persistence.

## Best Possible Solution

Keep #7403 open as the canonical product request. A complete implementation should add an explicit private/ephemeral mode across the Gateway session protocol, session config/store, transcript creation/appending, workspace bootstrap and memory loading/writing, Mission Control and channel controls, plus regression tests proving that private sessions do not read workspace memory/bootstrap files and do not write session history or memory files.
