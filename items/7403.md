## Summary

Keep open. Current main still does not implement an enforceable Private/Secret/Incognito mode for sessions: there is no Gateway protocol field, session config key, persisted session state, transcript no-persist branch, or UI/channel control that would suppress workspace context and disk history. The provided related context also makes #7403 the canonical tracker because #12242 was closed as a duplicate and #52478 was closed unmerged after deferring runtime filesystem/session-history enforcement.

## Best Possible Solution

Keep #7403 open as the canonical product request. A complete implementation should add an explicit private/ephemeral session mode through the Gateway protocol, session config/store state, transcript creation and append paths, workspace bootstrap and memory read/write paths, Mission Control UI, and channel controls, with regression tests proving private sessions do not read workspace/memory context and do not write session history or memory files.
