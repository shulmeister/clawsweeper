## Summary

Current `main` still leaves the reported Bonjour/mDNS DNS-label crash path actionable. The gateway can pass a long Linux/container hostname into the enabled-by-default Bonjour plugin, the plugin appends ` (OpenClaw)`, and `safeServiceName()` only trims instead of enforcing the 63-byte DNS label limit before calling `@homebridge/ciao`. The related fix PRs #37712 and #37714 were closed unmerged, so they do not supersede the issue.

## Best Possible Solution

Keep #37705 open. Implement byte-aware DNS label sanitization in the bundled `bonjour` plugin before calling `@homebridge/ciao`, covering both the service instance name and configured mDNS hostname label, preserving UTF-8 boundaries, and adding regression tests for the reported Kubernetes-style 64-byte label and multibyte names. Also clarify the supported workaround: current code/docs use `OPENCLAW_DISABLE_BONJOUR=1`, not `CLAWDBOT_DISABLE_BONJOUR=1`.
