## Summary

Current main still does not enforce the 63-byte DNS label limit on the Bonjour service instance name before passing it to @homebridge/ciao. The issue should remain open; the related fix PRs in the provided GitHub context were closed unmerged and do not supersede the report.

## Best Possible Solution

Keep #37705 open until the bundled Bonjour path clamps mDNS/DNS labels before calling @homebridge/ciao. The likely fix is a shared UTF-8 byte-aware helper that limits service instance names and mDNS host labels to 63 bytes, applied in the Bonjour plugin/advertiser path, with regression tests for long Kubernetes-style hostnames and multibyte names. OPENCLAW_DISABLE_BONJOUR remains a workaround, but it does not satisfy the requested default crash prevention.
