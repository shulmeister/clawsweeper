## Summary

Current main still leaves #37705 actionable: a Kubernetes-style hostname becomes a 64-byte Bonjour service instance label, and current code passes it to @homebridge/ciao without a byte-aware DNS-label clamp. The related fix PRs were closed unmerged in the provided discussion, and the later landed Bonjour crash-suppression commit does not add truncation.

## Best Possible Solution

Keep #37705 open. Add a shared byte-aware DNS-label sanitizer in the bundled `bonjour` plugin before calling `@homebridge/ciao`, apply it to both the service instance `name` and configured mDNS `hostname`, preserve UTF-8 boundaries, and keep a non-empty fallback. Add regression tests for the reported 64-byte instance label, longer `OPENCLAW_MDNS_HOSTNAME`, exact-boundary labels, and multibyte names. A maintainer reply should also clarify that the supported workaround is `OPENCLAW_DISABLE_BONJOUR=1`, not `CLAWDBOT_DISABLE_BONJOUR=1`.
