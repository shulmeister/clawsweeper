## Summary

Current main still has the idle/daily session-memory hook gap, so this is not implemented. The same remaining bug is now tracked by open issue #50891 plus active implementation PRs #61675 and #71792, making #37955 a duplicate/superseded tracker.

## Best Possible Solution

Close #37955 as a duplicate/superseded tracker and keep the remaining product/code work consolidated under #50891 and the active fix PRs, especially #61675 or the narrower #71792 if maintainers prefer the `command:reset` plus `auto:*` source contract.

## Evidence

- **Current main still gates fallback reset hooks to explicit commands:** `maybeEmitMissingResetHooks` immediately returns unless `resetTriggered` is true, the sender is authorized, and no reset hook was already triggered; it then only matches `/new` or `/reset` before calling `emitResetCommandHooks`. Automatic stale rollovers have `resetTriggered=false`, so this path does not satisfy #37955 on current main.
