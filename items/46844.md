## Summary

Keep open. Current main still lacks the requested configurable Talk Mode idle timeout/auto-deactivation after Voice Wake-triggered Talk Mode. The macOS runtime can activate Talk Mode from Voice Wake, but Talk Mode only has intra-turn silence handling and resumes listening after assistant replies instead of disabling itself after an idle period.

## Evidence

  - file: [apps/macos/Sources/OpenClaw/VoiceWakeRuntime.swift:533](https://github.com/openclaw/openclaw/blob/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071/apps/macos/Sources/OpenClaw/VoiceWakeRuntime.swift#L533)
