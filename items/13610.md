## Summary

Current main has the external-secrets baseline, but it still only supports SecretRef sources `env`, `file`, and `exec`. The requested native AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, and Google Cloud Secret Manager providers are not implemented, and the issue discussion includes explicit maintainer/Codex-review context keeping this as the umbrella for that remaining provider work.

## Best Possible Solution

Keep #13610 open as the umbrella for native secrets-manager provider support. The best implementation path is either to add first-class provider sources with types, schema, resolver dispatch, CLI/config docs, and tests for AWS Secrets Manager, Vault, Azure Key Vault, and GCP Secret Manager, or for maintainers to make an explicit product/API decision that exec wrappers or a plugin/ClawHub provider API are the supported path and then close against that canonical decision.
