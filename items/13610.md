## Summary

Keep #13610 open. Current main has the shipped SecretRef baseline, but it is still limited to env/file/exec providers and does not include native AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, or Google Cloud Secret Manager source/provider integrations. The issue discussion also explicitly preserves #13610 as the native-provider umbrella after #26155 shipped the baseline.

## Best Possible Solution

Keep #13610 open as the native secrets-manager provider umbrella. The best path is either to implement first-class AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, and Google Cloud Secret Manager support across the SecretRef types, schema, resolver dispatch, CLI/configure flow, docs, and tests, or make an explicit maintainer product decision that exec wrappers or a future plugin/ClawHub secret-provider API are the supported route.
