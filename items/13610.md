## Summary

Keep #13610 open. Current main has the external SecretRef baseline, but it still only supports `env`, `file`, and `exec` sources; the requested native AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, and Google Cloud Secret Manager providers are not implemented. The issue discussion also explicitly keeps this as the umbrella for remaining first-class provider work after #26155.

## Best Possible Solution

Keep #13610 open as the umbrella for native secrets-manager provider support. The best product path is either to implement first-class AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, and GCP Secret Manager provider sources across types, schema, resolver dispatch, CLI/config docs, and tests, or for maintainers to make an explicit canonical decision that exec wrappers or a future plugin/ClawHub provider API are the supported route.
