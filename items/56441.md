## Summary

Current `main` no longer shows the reported config-backed leak path: provider SecretRefs are resolved into the active runtime snapshot and projected back to markers for `models.json`, rather than exported into inherited `process.env`, so the `exec` child-process reproduction in this issue no longer matches current behavior.

## Evidence

- **Secrets resolve into runtime snapshot, not env export:** The secrets runtime resolves SecretRefs, applies them onto the resolved config object, and activates that config via `setRuntimeConfigSnapshot(...)`. This path stores resolved provider auth in the runtime snapshot; there is no corresponding write of resolved provider API keys into `process.env` here.
  - file: [src/secrets/runtime.ts:337](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/secrets/runtime.ts#L337)
  - command: `nl -ba src/secrets/runtime.ts | sed -n '330,390p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Source-managed provider secrets are rewritten to markers:** When a provider `apiKey` came from a SecretRef, OpenClaw rewrites it to the env var name for env-backed refs or to a non-env marker for file/exec-backed refs, instead of keeping the resolved secret value in downstream provider config artifacts.
  - file: [src/agents/models-config.providers.source-managed.ts:30](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/models-config.providers.source-managed.ts#L30)
  - command: `nl -ba src/agents/models-config.providers.source-managed.ts | sed -n '25,110p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Regression test covers marker projection:** The current test suite asserts that a runtime-resolved OpenAI provider secret is projected back to `OPENAI_API_KEY`, while a file-backed provider secret becomes `secretref-managed`, and that generated `models.json` keeps the marker rather than a resolved plaintext secret.
  - file: [src/agents/models-config.runtime-source-snapshot.test.ts:250](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/models-config.runtime-source-snapshot.test.ts#L250)
  - command: `nl -ba src/agents/models-config.runtime-source-snapshot.test.ts | sed -n '248,280p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Model auth reads config/runtime snapshot directly:** Provider auth resolution now treats env SecretRefs as env-var references and non-env SecretRefs as non-env markers, then resolves usable auth from config/runtime state rather than requiring a startup export of resolved provider keys into `process.env`.
  - file: [src/agents/model-auth.ts:73](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/model-auth.ts#L73)
  - command: `nl -ba src/agents/model-auth.ts | sed -n '68,155p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Exec still inherits sanitized process env only:** `tools.exec` still builds child env from sanitized `process.env`, which means the original report only reproduces if the sensitive provider key is actually present in the parent process environment. With current main's provider-secret handling, the config-backed `models.providers.<id>.apiKey` path no longer puts it there.
  - file: [src/agents/bash-tools.exec.ts:1536](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/bash-tools.exec.ts#L1536)

  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Changelog shows intentional hardening in this area:** The changelog documents deliberate hardening to keep SecretRef-managed provider auth out of generated `models.json` and to persist markers instead of resolved plaintext secrets, which matches the current code paths above.
  - file: [CHANGELOG.md:3016](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/CHANGELOG.md#L3016)
