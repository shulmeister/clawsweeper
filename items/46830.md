## Summary

Keep open. Current main has a Microsoft Teams-specific delivery-failure system event, but the broader request for channel delivery failure notification is not implemented across the channels named in the issue. Feishu and Telegram still primarily log or attempt fallback behavior rather than injecting a delivery-failure event into the agent context or returning structured delivery status to the agent.
