## Summary

Keep #66875 open. Current main contains partial Control UI mitigations, including final-state cleanup, active-run `session.message` suppression, and optimistic-message preservation, but the central eager reload behavior is still present: same-session `chat` finals still trigger `chat.history`, and `session.message` events after `chatRunId` is cleared still reload immediately. The supplied discussion also shows multiple user reproductions and an open, unmerged PR #67037 that addresses only part of this race.

## Best Possible Solution

Keep the issue open until the Control UI event pipeline narrows or removes same-session final `chat.history` reloads, adds a post-final `session.message` suppression/cooldown or run-id reconciliation, and keeps the optimistic-history merge tests. PR #67037 can be used as partial input, but the maintainer path should cover final-event reloads and post-final `session.message` events explicitly before closing #66875.
