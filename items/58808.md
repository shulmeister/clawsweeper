## Summary

Keep this PR open. Current main still has the reported SDK seam gap: channel agent-tool factories are typed and invoked with only `cfg`, while the adjacent core tool path already receives trusted sender context. The PR remains a narrow additive implementation candidate for the sender-aware channel plugin use case described in the PR and the linked closed feature issue #58806.

## Best Possible Solution

Keep this PR open or land an equivalent core SDK change. The best path is to extend `ChannelAgentToolFactory` params with optional `requesterSenderId` and `senderIsOwner`, thread `options.senderId` and `options.senderIsOwner` through `listChannelAgentTools`, and add a narrow regression test around a fake channel plugin `agentTools` factory.
