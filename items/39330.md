## Summary

Keep #39330 open. Current main still appends the internal unscheduled-reminder note into final reply payloads, and the current regression tests still assert that user-visible behavior. The PR remains a focused fix for the linked leak report, though the review concern about stripping only OpenClaw-injected text still needs maintainer resolution before merge.

## Best Possible Solution

Keep #39330 open for maintainer review. The best path is a narrow fix in src/auto-reply/reply/agent-runner.ts and src/auto-reply/reply/agent-runner-reminder-guard.ts that prevents internally injected reminder-guard notes from reaching final channel-visible payloads while preserving legitimate model output that happens to contain the same sentence. Update the existing reminder-guard tests to assert clean delivery behavior across the current cron.add, existing-cron, unrelated-cron, disabled-cron, missing-sessionKey, and cron-store-failure cases.
