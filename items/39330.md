## Summary

Keep #39330 open. Current main still appends the internal unscheduled-reminder note to reply payloads and returns those guarded payloads through final delivery; the submitted PR addresses a real linked bug, but the outstanding review concern about over-stripping legitimate model output remains actionable.

## Best Possible Solution

Keep the PR open for maintainer review. The best fix is to prevent internally injected reminder-guard notes from reaching final delivery while preserving legitimate model output containing the same sentence, either by tagging/gating payloads modified by appendUnscheduledReminderNote or by restructuring the guard so it records the condition without appending user-visible text. Update the existing reminder-guard tests for no cron.add, existing active cron coverage, unrelated cron jobs, disabled cron jobs, missing sessionKey, and cron-store read failure.
