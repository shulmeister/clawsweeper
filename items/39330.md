## Summary

Keep open. Current main still appends the unscheduled reminder note into final reply payloads, and the current tests assert that behavior. PR #39330 is a focused bug-fix candidate for a still-live behavior change, though it needs maintainer review because one review concern about stripping legitimate text remains unresolved.

## Best Possible Solution

Keep #39330 open for maintainer review. The best path is a narrowly scoped fix in src/auto-reply/reply/agent-runner.ts and src/auto-reply/reply/agent-runner-reminder-guard.ts that prevents the internally injected reminder guard from reaching channel-visible final payloads, while preserving legitimate model output that merely contains the same text; update the reminder-guard tests to assert the desired clean delivery behavior.
