## Summary

Keep #39330 open. Current main still appends the internal unscheduled-reminder note into final reply payloads, current tests assert that visible behavior, and the latest release evidence still contains the same append path. The PR is a focused candidate for linked issue #37239, though the remaining review concern about stripping only OpenClaw-injected text still needs maintainer resolution before merge.

## Best Possible Solution

Keep #39330 open for maintainer review. The best path is a narrow fix in src/auto-reply/reply/agent-runner.ts and src/auto-reply/reply/agent-runner-reminder-guard.ts that prevents internally injected reminder-guard notes from reaching final channel-visible payloads, while preserving legitimate model output that happens to contain the same sentence. Update the reminder-guard tests to assert clean delivery behavior for the existing cron.add, existing-cron, unrelated-cron, disabled-cron, missing-sessionKey, and cron-store-failure cases.
