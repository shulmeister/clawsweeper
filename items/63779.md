## Summary

Current main has part of the proposed lifecycle-event plumbing for ACP turns, but the reported wake problem is not fully solved. For requester sessions with a normal channel route, ACP task completion is still delivered as an outbound message with a transcript mirror, and the existing regression test asserts that no parent-session system event is queued on that path. That matches the reported delivery-mirror assistant-message symptom closely enough that this issue should stay open.

## Best Possible Solution

Keep this issue open and implement an explicit ACP completion resume path for yielded parents. The fix should route ACP lifecycle completion into the requester session as an internal/user-turn activation, or through the same completion announce machinery used for subagent orchestration, even when a deliverable channel route exists. Add a regression test for the deliverable-origin case showing the parent is woken/resumed rather than only receiving a mirrored assistant transcript row.
