## Summary

Keep open. Current main only resets a stale saved session when a tokenized URL is supplied without an explicit session; a normal startup/reload still hydrates the persisted scoped session selection and treats non-main session keys as explicit user-selected sessions. That leaves the reported routed historical-session restore path unresolved.

## Evidence

- **Persisted scoped session is restored on load:** loadSettings() reads sessionsByGateway for the current gateway scope and assigns scopedSessionSelection.sessionKey and lastActiveSessionKey into settings without filtering routed keys such as agent:*:*:direct:* or agent:*:*:group:*.
