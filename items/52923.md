## Summary

Keep open. Current main still drops flat numeric `usage.cost` values in `src/infra/session-cost-usage.ts`, so the PR addresses a real unfixed bug and is not implemented on main.

## Evidence

- **Current extractor only accepts object-shaped cost:** `extractCostBreakdown` casts `record.cost` to `Record<string, unknown>` and immediately reads `cost.total`; a flat numeric `usage.cost` is rejected and returns `undefined`.
  - file: [src/infra/session-cost-usage.ts:79](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/infra/session-cost-usage.ts#L79)

  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Session logs still miss flat numeric cost:** `loadSessionLogs` only sets `cost` from `extractCostBreakdown(usageRaw)` and otherwise falls back to configured model pricing. If OpenRouter supplies only a flat numeric `usage.cost`, current main does not preserve that upstream total.
  - file: [src/infra/session-cost-usage.ts:1043](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/infra/session-cost-usage.ts#L1043)
