## Summary

Keep open. Current `main` does not implement `openclaw models status --probe --all`: the status CLI has no `--all` flag, the routed parser does not preserve a `probeAll` option, and probe target construction still chooses a single model per provider via `selectProbeModel()`. The PR is a focused active implementation for open issue #63145, with recent author follow-up and no protected-label or maintainer-author auto-close guard.

## Best Possible Solution

Keep PR #63259 open as the active implementation for #63145. The best path is to review and refine the PR: wire `probeAll` through both Commander and routed CLI paths, expand `buildProbeTargets` across all configured candidates while preserving the existing one-per-provider default, add/keep focused regression coverage, document the new flag, and either separate or explicitly accept the unrelated CI-drift test fix.
