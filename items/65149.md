## Summary

Keep this PR open. Current main still lacks `final_reply_payloads` or an equivalent final-reply hook with resolved provider/model context, and the existing `message_sending` delivery hook still does not carry runtime model identity. The related #69110 issue is relevant, but it is open and does not supersede this concrete implementation candidate.

## Best Possible Solution

Keep the PR open for maintainer/API review. The best path is to rebase and finish a supported core plugin API change that gives plugins stable, truthful final-delivery provider/model context, caches the global hook runner at the call site, updates hook types/runner exports/docs, and adds focused tests for final reply assembly, usage footer ordering, and fallback-selected model versus actual runtime model. Coordinate the final shape with #69110 rather than closing this PR before a replacement lands.
