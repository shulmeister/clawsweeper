## Summary

Keep PR #71086 open. Current main still always surfaces mutating-tool warnings even when assistant output exists, so the PR is not obsolete. However, the existing review concern is valid: the proposed patch gates suppression only on user-facing reply presence, which can include partial/error turns and could hide unresolved mutating-tool failures.

## Best Possible Solution

Keep the PR open and ask for a narrower fix: suppress stale mutating-tool warnings only after a verified successful terminal turn or confirmed same-action recovery, while preserving warnings for unresolved mutating failures after error/interrupted turns. Add regression coverage for both the recovered-success case and the partial/error case, and update the stale `payloads.ts` comment noted in review.
