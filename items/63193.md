## Summary

Keep PR #63193 open. Current main does not contain the Android release handoff work proposed by the PR: there is no `.github/workflows/android-release.yml`, no `apps/android/scripts/export-upload-cert.sh`, no `android:export-upload-cert` package script, and the Android README still lacks the release handoff/Play upload certificate flow. The PR is draft and ops-sensitive, so it needs maintainer review rather than automated cleanup.

## Best Possible Solution

Keep the PR open for explicit Android/release maintainer handling. The useful path is to either land a reviewed, current-main version of the workflow/docs/certificate-export helper with release-secrets validation, or have maintainers close it manually if Android release automation is no longer desired.
