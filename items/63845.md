## Summary

Keep this issue open. It is a recent, concrete maintainer/design-docs question about operating many skills/tools, not a stale bug or a clear ClawHub feature request. Current main has relevant building blocks and partial documentation, but I did not find a canonical answer for tested tool-count ceilings, high-count SKILL.md routing guidance, low-end model reference configurations, or whether smart aggregation tools are recommended vs anti-patterns.

## Best Possible Solution

Keep the issue open for maintainer guidance or convert it into a focused docs task. The likely best product path is a docs section under Context, Compaction, or Building Plugins covering high-count tool/skill setups: no fixed tested ceiling unless maintainers can provide one, how to inspect prompt cost with /context detail, how to use tools.profile/allow/deny and skills.limits, when to mark skills out of model invocation, recommended compaction/pruning knobs for tool-heavy turns, and guidance that a hybrid aggregated domain tool is acceptable as a plugin when it preserves traceability and bounded output. If maintainers consider this support-only, they should close it explicitly with the support policy rather than via this automated cleanup.
