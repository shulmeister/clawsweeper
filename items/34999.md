## Summary

Close as `clawhub`: current `main` intentionally defines multi-agent as isolated agents with separate session stores, and the repo vision pushes optional orchestration layers into plugins/ClawHub rather than new core default architecture. Existing plugin hooks and subagent runtime APIs are the intended extension seam, and the issue thread already includes a report of this working in an external OpenClaw-based implementation.
