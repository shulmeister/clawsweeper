## Summary

The reported Windows-native exec regression is already fixed on current `main`, and the shipped `v2026.4.22` release contains the same PowerShell call-operator fix. Windows exec still uses PowerShell `-Command`, but the enforced Windows shell command builder now prepends `&` so quoted executable paths are invoked instead of parsed as string literals.
