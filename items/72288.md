## Summary

Current main has removed the post-turn `rewriteSubmittedPromptTranscript` path reported in 2026.4.24 and now submits the transcript-safe prompt directly, while preserving hidden runtime context as non-display runtime context. This matches the earlier closed #72012 report for the same prompt-rewrite branch duplication family.

## Best Possible Solution

Close this issue as fixed on main, keep the shipped runtime-context submission model, and rely on `openclaw doctor --fix` for users with existing 2026.4.24 transcripts affected by duplicated prompt-rewrite branches. Leave any new duplicate-response reports to open with fresh reproduction data if they still occur after the next release/main build.
