## Summary

Keep #71018 open. The current Feishu/Lark plugin on main has only a transient streaming-card tool-start status, while the report asks for a full tool-execution step area with tool name, args, result, and duration. Core now exposes enough structured callbacks to build that UI, but the Feishu plugin does not appear to consume them yet, so there is no high-confidence close reason.

## Best Possible Solution

Keep the issue open and implement the Feishu/Lark card tool-execution area in `extensions/feishu`: consume the existing structured callbacks (`onItemEvent`, `onCommandOutput`, `onPatchSummary`, `onApprovalEvent`, and/or `onToolResult`) alongside `onToolStart`, update the streaming/static card builder to render persistent tool steps with name, args/summary, result/status, and duration, and add focused Feishu tests covering start, result, error, and empty-state behavior. If `openclaw-lark` is a separate legacy package, maintainers should also decide whether to map it onto `@openclaw/feishu` or document the supported migration path.
