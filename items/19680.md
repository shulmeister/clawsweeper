## Summary

Keep #19680 open. Current main has useful VISION.md scope guidance, plugin/lean-core direction, memory notes, security guidance, and PR review mechanics elsewhere, but it still has not been restructured into the requested human/AI decision framework. The issue thread also already contains a recent Codex review keeping it open, with no provided superseding PR or maintainer decision.

## Best Possible Solution

Keep the issue open until a maintainer either accepts and implements a docs-only VISION.md restructure or explicitly decides that current VISION.md plus CONTRIBUTING.md and the PR template are sufficient. A good implementation would add a concise product thesis, three or four PR gut-check questions, and rationale-grounded examples for security, memory, lean core, plugins, and won't-merge categories so humans and AI review tools can reason from principles.
