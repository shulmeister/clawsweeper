## Summary

Keep PR #63104 open. Current main has Discord inbound replay dedupe, but it is still scoped to each createDiscordMessageHandler instance, so the PR's central cross-handler duplicate-delivery fix is not implemented or clearly superseded. The linked bug #63027 is closed, but the provided context does not identify a canonical merged replacement for this PR.

## Best Possible Solution

Do not close this PR as implemented or obsolete. Maintainers should review or adapt the PR into current main by adding accountId/messageId dedupe that spans Discord handler instances, preferably claimed at the active inbound-worker boundary, with regression coverage for duplicate delivery across distinct handlers and for a deactivated handler seeing the event before an active handler.
