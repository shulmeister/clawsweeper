## Summary

Keep open. Current main still does not implement the PR's inline/legacy daily-memory read filtering: `memory_get` reaches `QmdMemoryManager.readFile()`, which returns raw full or partial QMD file content, while inline/both dreaming storage can still write managed Light/REM blocks into `memory/YYYY-MM-DD.md`. The linked issue #68367 appears closed for the default separate-storage path, but this PR targets remaining opt-in inline/both and old-file cases, and the PR discussion still has substantive correctness/API-baseline concerns to resolve before merge.

## Best Possible Solution

Keep the PR open or replace it with an equivalent core fix. The best implementation should strip managed Light/REM dreaming blocks on daily memory reads after resolving/normalizing the actual path, preserve ordinary file formatting when no managed block is removed, cover full reads, paginated reads, and normalized-equivalent paths with regression tests, and avoid exporting a new Plugin SDK helper unless there is a real third-party need; if it is exported, regenerate the Plugin SDK API baseline.
