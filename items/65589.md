## Summary

Close this PR as superseded. The linked runaway-dreaming incident was real and the PR discussion added useful production evidence, but this branch is not merge-ready because its enforcer is only sketched and never wired into the live dreaming pipeline. Current main has since shipped the production-path fixes for the observed runaway/data-corruption behavior: single managed dreaming cron execution, live queued-event gating, separate dream output storage, self-ingestion filters, generated transcript skipping, and repair/dedupe recovery.

## Best Possible Solution

Close this PR as superseded by the shipped memory-core dreaming fixes on main. Keep the current main implementation, and handle any remaining desire for a configurable cost budget as a new focused design/implementation that is wired into the live dreaming pipeline, config schema, docs, and current tests.
