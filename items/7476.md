## Summary

Keep open. Current main still lacks outbound WhatsApp sticker sending: WebP media is still routed through the generic image payload, no WhatsApp send option exposes sticker/asSticker, WebP images are not preserved as stickers by the shared media loader, and WhatsApp message actions still do not support sticker. The related implementation PR #7502 was closed unmerged, and the latest issue discussion already reached a keep-open conclusion.

## Best Possible Solution

Keep the issue open and implement this in the bundled WhatsApp plugin. Add an explicit outbound sticker contract such as action=sticker or asSticker, preserve valid WebP sticker media before JPEG optimization, route outbound sends through Baileys { sticker: buffer }, and update the message/gateway option surfaces, WhatsApp docs, and focused send-api/media-loader tests.
