## Summary

Keep open. Current main still does not implement outbound WhatsApp sticker sends: WebP media is still routed as generic image media, the WhatsApp send options have no sticker/asSticker contract, shared media loading still optimizes non-GIF images by default, and WhatsApp message actions still advertise only react/poll. The prior implementation PR #7502 was closed unmerged, and the latest issue discussion already reached the same keep-open conclusion.

## Best Possible Solution

Keep the issue open and implement this in the bundled WhatsApp plugin: add an explicit outbound sticker contract such as action=sticker or asSticker, preserve valid WebP sticker media before JPEG optimization, route outbound sticker sends through Baileys { sticker: buffer }, and cover send-api/media-loader behavior plus WhatsApp docs. Do not close as ClawHub while the existing bundled WhatsApp channel owns the affected send path and the latest maintainer/Codex discussion already recommended keeping it open.
