## Summary

Keep #7476 open. Current main still has inbound WhatsApp sticker recognition, but outbound WhatsApp sends route WebP/image media through the generic image path, expose no `asSticker` or sticker send option, and do not advertise or dispatch `action=sticker` for WhatsApp. The prior implementation PR #7502 is closed unmerged, and the latest issue discussion already includes a maintainer Codex review explicitly keeping this open.

## Best Possible Solution

Keep the issue open and implement this in the bundled WhatsApp plugin. Add an explicit outbound sticker contract, preserve valid WebP sticker media before image optimization, route outbound sends through Baileys' native `{ sticker: ... }` payload, advertise/dispatch the WhatsApp sticker action or option consistently, and add focused send-api/media-loader tests plus WhatsApp docs.
