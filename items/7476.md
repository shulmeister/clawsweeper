## Summary

Keep #7476 open. Current main still lacks outbound WhatsApp sticker sending: WebP media follows the generic image path, no WhatsApp send option or message action exposes sticker/asSticker, shared media loading still optimizes non-GIF images by default, and the related implementation PR #7502 was closed unmerged. The latest issue discussion also already reached a keep-open conclusion.

## Best Possible Solution

Keep the issue open and implement this in the bundled WhatsApp plugin. Add an explicit outbound sticker contract such as action=sticker or asSticker, preserve valid WebP sticker media before JPEG/PNG optimization, route outbound sends through Baileys { sticker: buffer }, and update WhatsApp action discovery, gateway/message option surfaces, docs, and focused send-api/media-loader tests.
