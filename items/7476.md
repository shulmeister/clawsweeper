## Summary

Keep #7476 open. Current main still lacks outbound WhatsApp sticker send support: WebP media goes through the generic image path, no sticker/asSticker option reaches the active listener, WhatsApp message actions do not advertise or dispatch sticker, and the related implementation PR #7502 was closed unmerged.

## Best Possible Solution

Keep the issue open and implement this in the bundled WhatsApp plugin. Add an explicit outbound sticker contract, preferably `action=sticker` or `asSticker`; preserve qualifying WebP sticker media before generic image optimization; route outbound sends through Baileys `{ sticker: buffer }`; advertise and dispatch the WhatsApp sticker action or option consistently; and add focused send-api, media-loading, message-action, docs, and regression coverage. The closed #7502 patch and reporter comments are useful implementation evidence, but the API shape should be deliberate rather than only empty-caption WebP detection.
