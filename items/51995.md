## Summary

Close: current `main` already fixes the reported behavior. The send-path poll guard still exists, but `hasPollCreationParams` now treats empty strings/arrays, zero-valued numeric poll fields, and false boolean poll flags as non-intent, and there is a regression test proving `action: "send"` with inert poll defaults no longer gets misclassified as a poll.
