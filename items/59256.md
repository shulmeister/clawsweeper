## Summary

Current `main` already has the sandbox session-visibility override, `sessions_send` access checks, docs, and regression coverage needed for sandboxed cross-agent messaging. The old timeout behavior described in #59256 is superseded by explicit config support (`agents.defaults.sandbox.sessionToolsVisibility: "all"`) plus fail-fast permission checks when that access is not enabled.

## Evidence

- **Sandbox override exists on main:** Sandboxed sessions are no longer unconditionally clamped to tree visibility; `agents.defaults.sandbox.sessionToolsVisibility` can be set to `"all"`, and the runtime preserves `tools.sessions.visibility` in that case.
