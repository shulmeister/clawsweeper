## Summary

Keep open. Current main does not implement the requested visible WSL2 headed Chrome behavior by synthesizing DISPLAY=:0. The browser launch env is inherited from process.env plus HOME, current Linux no-display handling falls back to headless only for default local managed launches, and explicit headed mode still produces a missing-display error. The provided issue context also includes a maintainer comment explicitly saying the related landed fallback does not close this WSL2 headed DISPLAY request; related DISPLAY PRs are closed/unmerged, so there is no canonical merged duplicate to close against.
