## Summary

Current `main` has Matrix preview/live-marker support for clients and some edit filtering, but it still does not expose streaming/edit state (`is_streaming`, `is_final`, raw `m.relates_to`, or equivalent) in the agent-facing inbound context. The request remains unimplemented, so this item should stay open.

## Evidence

- **Agent-facing Matrix context only exposes body text plus existing routing fields:** `finalizeInboundContext(...)` passes `BodyForAgent`, `BodyForCommands`, `WasMentioned`, reply/thread metadata, and related fields, but no streaming/edit-status field and no raw Matrix relation payload for agents to inspect.

- **Matrix live/streaming marker exists only on outbound event content:** The Matrix plugin defines `org.matrix.msc4357.live` as an outbound live-marker key for in-progress messages, intended for Matrix clients to render a streaming animation until a later edit clears it.
