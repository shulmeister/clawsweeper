## Summary

Keep open. Current main has related Matrix outbound streaming work: partial previews are marked with MSC4357 live markers and finalized via Matrix edits. But the requested agent-facing metadata is still missing: Matrix replacement/edit events are dropped on inbound, and the context sent to agents exposes `WasMentioned`/message IDs but not `is_streaming`, `is_final`, `m.relates_to`, or the MSC4357 live marker. The provided related items do not supersede this Matrix-specific request.

## Best Possible Solution

Keep #64980 open. The best product/code path is an additive Matrix inbound contract: either suppress/defer OpenClaw-originated live preview events until the finalized edit, or expose bounded structured metadata to agents/plugins (`is_streaming`, `is_final`, Matrix relation target/type, and/or MSC4357 live/finalized-preview markers). Add Matrix monitor tests for configured bot senders in a shared room with `streaming: "partial"` and `blockStreaming: true`, proving agents do not act on truncated preview text unless explicitly configured to do so.
