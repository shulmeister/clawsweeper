## Summary

Keep open. Current `main` still hard-blocks dependency-denylist findings even when `dangerouslyForceUnsafeInstall` is set, so PR #65857 is not already implemented. The PR also changes security-policy semantics beyond current docs, so this needs explicit maintainer review rather than conservative cleanup closure.
