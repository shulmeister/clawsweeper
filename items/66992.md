## Summary

Keep open. Current main still generates and rewrites the macOS gateway LaunchAgent plist without `ProcessType: Interactive`; the heartbeat scheduler still depends on an unref'd `setTimeout`, so the specific App Nap/process-suspension concern is not implemented or otherwise clearly mitigated. The related context also points to an open launchd ProcessType PR/report rather than shipped behavior.

## Best Possible Solution

Keep this open until the macOS gateway LaunchAgent generator and rewrite paths add `ProcessType: Interactive`, with focused tests covering fresh install and regenerated/restart-fallback plists. If maintainers prefer one canonical tracker, link and close only after confirming the canonical issue or PR remains open and explicitly owns the App Nap heartbeat-stall behavior.
