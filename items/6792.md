## Summary

Keep open. Current main still does not implement manifest-level install-time `configPatch`; the only shipped config patch path remains provider-auth scoped, plugin install persistence has no patch input, docs still describe manual plugin config, and the linked implementation PR #43421 was closed unmerged.

## Best Possible Solution

Keep this issue open for a core plugin install/API decision. A complete fix should add a typed manifest-level patch contract, parse and sanitize it from plugin metadata, propagate it through every install source, apply a validated merge in the config persistence path with explicit array semantics, show the applied diff or skip reason, and cover local path/link/archive/npm/ClawHub/marketplace/bundled plus invalid-patch cases in tests and docs.
