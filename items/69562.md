## Summary

Keep #69562 open. Current main still validates only the cron announce channel, not `delivery.accountId` or `delivery.failureDestination.accountId`, while the open bug #69513 reports that invalid cron account IDs are accepted and persisted. The PR is a plausible focused fix with regression coverage, and the earlier related PR #69518 is closed unmerged rather than a shipped superseding fix.

## Best Possible Solution

Keep the PR open for maintainer review and rebase. The best path is to land a focused gateway-layer fix for #69513 that validates explicit cron announce `delivery.accountId` and `delivery.failureDestination.accountId` against the resolved configured channel accounts, while preserving valid default/unlisted/default-normalized account behavior and adding the focused rejection/acceptance tests shown in #69562.
