## Summary

Current `main` already has embedded `--local` auth-profile resolution and runtime API-key injection for provider-backed runs, which covers the reported OpenRouter 401 path after onboarding. The fix is present on `main` but I could not tie it to a tagged release from this shallow checkout.

## Evidence

- **Embedded runner now auto-selects auth profiles from the agent auth store:** `runEmbeddedPiAgent` resolves `preferredProfileId`, validates any user-locked profile, computes `resolveAuthProfileOrder(...)`, and builds `profileCandidates` for the active provider before the run starts. This is the missing selection step the issue described for fresh `--local` runs.
