## Summary

Keep open. Current main does not implement the outbound metadata routing this PR is trying to add: gateway `send` rejects extra `metadata`, `sendMessage` has no metadata/Hermes parameter, direct delivery and channel outbound context do not carry opaque metadata, and the delivery queue does not persist/replay it. The PR is not superseded, but it needs maintainer review and rework before merge because the branch includes Hermes-specific hardcoded identifiers, an unrelated auth commit, and unresolved review comments.

## Best Possible Solution

Keep the PR open for maintainer judgment, but ask for a focused rework: split out the unrelated auth commit, decide whether OpenClaw wants a generic opaque outbound metadata API, route that metadata through gateway send, direct delivery, channel context, queue storage, and recovery with tests, and move Hermes-specific builders/defaults or deployment policy out of core unless maintainers explicitly want them bundled.
