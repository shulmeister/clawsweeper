## Summary

Keep this issue open. Current main has adjacent post-v2026.4.24 subagent work, including longer child-agent startup request timeouts and expanded completion-delivery coverage, but I did not find a targeted fix or regression test proving that a spawned/completed subagent is visible to `/subagents list` for the same requester. The reported issue is recent, has concrete reproduction evidence, and the reporter confirmed it still reproduced on v2026.4.24 with a related parent-completion symptom.

## Best Possible Solution

Keep this issue open. The best maintainer path is a focused regression test or live verification for the exact flow: spawn through `sessions_spawn` or the standard subagent path, confirm `registerSubagentRun` writes the expected controller/requester key, call `/subagents list` from the same requester and assert active/recent entries are non-empty, then verify parent completion delivery consumes the same registered run. If the current child-agent timeout change is the real fix, land that proof and close as implemented; otherwise patch the registry write/read or requester-key mismatch directly.
