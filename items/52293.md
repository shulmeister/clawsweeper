## Summary

Keep open. Current `main` still does not recognize `"transport endpoint is not connected"` as a systemd user-bus-unavailable condition, so the PR's requested matcher and regression coverage are not already implemented on `main`.

## Evidence

- **Bus-unavailable matcher still lacks the new string:** `isSystemdUserBusUnavailableDetail()` on current `main` matches `failed to connect to bus`, `failed to connect to user scope bus`, `DBUS_SESSION_BUS_ADDRESS`, `XDG_RUNTIME_DIR`, and `no medium found`, but not `transport endpoint is not connected`.
  - file: [src/daemon/systemd-unavailable.ts:23](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/daemon/systemd-unavailable.ts#L23)
  - command: `nl -ba src/daemon/systemd-unavailable.ts | sed -n '1,120p'`
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Machine-scope fallback still depends on that matcher:** `shouldFallbackToMachineUserScope()` immediately returns false unless `isSystemdUserBusUnavailableDetail(detail)` is true, so a transport-endpoint error would still miss the fallback path on current `main`.
  - file: [src/daemon/systemd.ts:357](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/daemon/systemd.ts#L357)

- **Requested regression tests are not present on main:** Current daemon tests cover existing bus-unavailable and permission-denied cases plus existing non-fatal probe errors, but there is no test for `transport endpoint is not connected` in the availability/fallback/probe sections.
  - file: [src/daemon/systemd.test.ts:154](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/daemon/systemd.test.ts#L154)
