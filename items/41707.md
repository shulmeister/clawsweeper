## Summary

Current `main` already contains a specific fix for mid-turn rate-limit failures after tool use: the runner now converts those 429/tool-call failures into an immediate final error payload instead of falling through and continuing the run, and there is a regression test covering that path. The same implementation is present in the released `v2026.4.22` tree.
