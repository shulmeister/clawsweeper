## Summary

Current `main` and the shipped `v2026.4.23` release already document and implement the isolated-cron `payload.model` behavior this PR was trying to cover, so the temporary workaround section is now obsolete.

## Evidence

- **Shipped docs already say cron model overrides use the selected model and do not silently append the agent primary:** `docs/cli/cron.md` in `v2026.4.23` states that `openclaw cron add|edit --model ...` uses the selected allowed model, warns on disallowed models, and that a plain model override no longer appends the agent primary as a hidden extra retry target.
