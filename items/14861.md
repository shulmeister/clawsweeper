## Summary

Keep open. Current main still has partial diagnostics, but it does not implement the requested gateway boot summary, and the specific Gmail watcher missing-account startup case remains silent in the gateway boot path.

## Best Possible Solution

Keep #14861 open and implement a gateway post-sidecar startup summary, or a maintainer-approved equivalent diagnostics surface, in the gateway startup logging path. It should capture configured subsystem outcomes and skipped/failure reasons for Gmail watcher, hooks, channels, plugin services, memory sidecars, cron/heartbeat, and similar boot work, with regression coverage for the missing `hooks.gmail.account` case and the aggregate summary.
