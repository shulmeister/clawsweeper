## Summary

Keep #14861 open. Current main has partial startup diagnostics, but it still does not provide the requested gateway boot summary, and the specific Gmail watcher missing-account startup path remains silent.

## Best Possible Solution

Keep #14861 open. The best fix is a maintainer-approved gateway post-sidecar startup summary or equivalent diagnostics surface in the startup path, likely by returning structured outcomes from `startGatewaySidecars` and logging configured subsystem success/skipped/failure reasons after sidecars settle. It should cover Gmail watcher skipped reasons such as missing `hooks.gmail.account`, hooks/internal hooks, channel startup, plugin services, memory sidecars, and scheduled services, with regression tests for the aggregate summary and the Gmail missing-account case.
