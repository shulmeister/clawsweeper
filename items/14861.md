## Summary

Keep open. Current main adds some Gmail watcher lifecycle logging, but the requested boot observability is still incomplete: there is no post-initialization subsystem startup summary, and expected Gmail non-start states such as missing `hooks.gmail.account` are still not surfaced at gateway boot.

## Evidence

- **Current startup log is only a compact ready line:** `logGatewayStartup` logs agent model, `ready (...)`, and log file, with plugin ids and duration only. It does not report per-subsystem success/failure after sidecars/hooks finish.
  - file: [src/gateway/server-startup-log.ts:25](https://github.com/openclaw/openclaw/blob/1dfa52d0711dfc86e1c04538fe46e3d43961766f/src/gateway/server-startup-log.ts#L25)
  - command: `nl -ba src/gateway/server-startup-log.ts | sed -n '1,140p'`
  - sha: [1dfa52d0711d](https://github.com/openclaw/openclaw/commit/1dfa52d0711dfc86e1c04538fe46e3d43961766f)
- **Sidecars start after the ready log and are not summarized:** `startGatewayPostAttachRuntime` logs startup readiness first, then asynchronously starts sidecars. The sidecar path emits individual messages for some failures, but no aggregate boot summary after all subsystem startup work completes.
  - file: [src/gateway/server-startup-post-attach.ts:441](https://github.com/openclaw/openclaw/blob/1dfa52d0711dfc86e1c04538fe46e3d43961766f/src/gateway/server-startup-post-attach.ts#L441)
