## Summary

Keep #14861 open. Current `main` still lacks the requested aggregate gateway boot summary and still does not surface the specific `hooks.enabled=true` plus missing `hooks.gmail.account` startup outcome. Existing targeted startup diagnostics, phase tracing, plugin diagnostics, and Gmail watcher wrapper logs do not cover the central observability request; the existing issue discussion already kept this open for the same gap, and the relevant current source still matches that analysis.

## Best Possible Solution

Keep #14861 open. The best maintainer path is to add a structured gateway startup outcome collector in `startGatewaySidecars` or `startGatewayPostAttachRuntime`, record started/skipped/failed results for Gmail watcher, internal hooks, cron, memory, heartbeat, channels, plugin services, scheduled services, and related sidecars, then emit a concise boot summary after startup settles. Add regression coverage for the aggregate summary and for `hooks.enabled=true` with missing `hooks.gmail.account`.
