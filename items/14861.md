## Summary

Keep open. Current `main` improved startup observability for some gateway sidecars, but it does not fully implement the reported request: gmail-watcher is still silent when it is skipped because `hooks.gmail.account` is unset or hooks are disabled, and there is still no boot summary/status output covering subsystem start/skips/failures.

## Evidence

- **Gmail watcher startup is only attempted when hooks are enabled and an account is configured:** `startGatewaySidecars` gates gmail-watcher startup behind `params.cfg.hooks?.enabled && params.cfg.hooks.gmail?.account`, so the `no gmail account configured` case from the report still produces no log at gateway boot.
  - file: [src/gateway/server-startup-post-attach.ts:173](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/gateway/server-startup-post-attach.ts#L173)
  - command: `nl -ba src/gateway/server-startup-post-attach.ts | sed -n '160,320p'`
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **Lifecycle helper still suppresses the exact skipped reasons called out in the issue:** `startGmailWatcherWithLogs` warns for actionable failures, but explicitly suppresses `hooks not enabled` and `no gmail account configured`, so those startup outcomes remain non-observable in logs.
  - file: [src/hooks/gmail-watcher-lifecycle.ts:27](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/hooks/gmail-watcher-lifecycle.ts#L27)

- **Gateway startup logging still does not emit a subsystem boot summary:** `logGatewayStartup` logs agent model, a generic `ready (...)` line, and log-file/security notices, but nothing like the requested per-subsystem initialized/skipped/failed summary.
  - file: [src/gateway/server-startup-log.ts:25](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/gateway/server-startup-log.ts#L25)
