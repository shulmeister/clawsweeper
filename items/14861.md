## Summary

Keep open. Current main has partial readiness/probe observability, but it still does not implement the requested post-initialization gateway boot summary, and the issue's specific Gmail missing-account startup case remains silent.

## Best Possible Solution

Keep the issue open and implement a gateway post-sidecar startup summary, or an explicitly maintainer-approved equivalent diagnostics surface, in `src/gateway/server-startup-post-attach.ts`/startup logging. It should report configured subsystem outcomes and skipped/failure reasons for Gmail watcher, hooks, channels, plugin services, memory sidecars, cron/heartbeat, and similar boot work, with regression tests for the missing `hooks.gmail.account` case and the aggregate summary.
