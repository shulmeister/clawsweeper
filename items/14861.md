## Summary

Keep #14861 open. Current main has partial observability improvements, including startup trace timings, readiness probes, plugin initialization failure summaries, and narrow startup-task skipped/failed logging, but it still does not emit the requested aggregate Gateway boot summary with per-subsystem started/skipped/failed outcomes. The original Gmail missing-account path is still skipped before the lifecycle logger runs, and the lifecycle wrapper still suppresses the "no gmail account configured" reason when called directly.

## Best Possible Solution

Keep #14861 open. The maintainer path is to add a structured startup outcome collector around startGatewaySidecars/startGatewayPostAttachRuntime, record started/skipped/failed outcomes for Gmail watcher, internal hooks, channels, plugin services, memory, heartbeat/cron/recovery sidecars, and emit a concise normal boot summary after sidecars settle. Add regression coverage for the aggregate summary and for hooks.enabled=true with missing hooks.gmail.account.
