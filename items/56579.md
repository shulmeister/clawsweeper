## Summary

Keep open. Current `main` still has the reported parser bug: `parseClawHubPluginSpec` treats a trailing `@` as part of the package name, so `clawhub:demo@` is accepted and later maps to `package_not_found` instead of an invalid-spec failure. The reporter-linked fix discussion exists, but that behavior is not present on current `main` or the latest release tag inspected.

## Evidence

- **Parser still accepts trailing @ on current main:** `parseClawHubPluginSpec` returns `{ name: spec }` whenever `atIndex >= spec.length - 1`, so `clawhub:demo@` is parsed as name `demo@` instead of being rejected.
