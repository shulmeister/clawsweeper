## Summary

Keep open. Current `main` does not implement Feishu `ignoreAtAll`, and the PR discussion already identifies a concrete logic bug in the proposed change that would suppress direct `@bot` mentions when `@_all` is also present.

## Evidence

- **Current main still treats `@_all` as a bot mention unconditionally:** `checkBotMentioned()` returns `true` as soon as `@_all` appears in message content, with no `ignoreAtAll` option or fallback path.
  - file: [extensions/feishu/src/bot-content.ts:233](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/feishu/src/bot-content.ts#L233)
  - command: `nl -ba extensions/feishu/src/bot-content.ts | sed -n '228,245p'`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Main Feishu message parsing has no `ignoreAtAll` plumbing:** `parseFeishuMessageEvent()` only accepts `(event, botOpenId, _botName)` and calls `checkBotMentioned(event, botOpenId)`. Group gating later still depends on `ctx.mentionedBot` when `requireMention` is enabled.
  - file: [extensions/feishu/src/bot.ts:100](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/feishu/src/bot.ts#L100)
