## Summary

Close as `clawhub`: current `main` treats WeChat as an external channel plugin, and both WeChat-specific outbound/context-token behavior and isolated-session context requirements are documented as plugin-owned rather than OpenClaw core-owned.

## Evidence

- **WeChat is documented as an external plugin, with context tokens owned by that plugin:** Current main docs say OpenClaw connects to WeChat through Tencent's external `@tencent-weixin/openclaw-weixin` plugin, and explicitly state that WeChat login, Tencent iLink API calls, media handling, context tokens, and account monitoring are owned by the external plugin.
  - file: [docs/channels/wechat.md:27](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/channels/wechat.md#L27)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Channel-specific send execution belongs to the channel plugin, not core:** Plugin architecture docs state that core only hosts the shared `message` tool and dispatch, while channel plugins own scoped action discovery and execute the final action through their adapter.
  - file: [docs/plugins/architecture.md:181](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/plugins/architecture.md#L181)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Isolated cron sessions intentionally do not inherit ambient conversation context:** Cron docs define `sessionTarget: "isolated"` as a fresh session and explicitly say isolated jobs do not inherit ambient conversation context such as channel/group routing or send/queue policy.
  - file: [docs/automation/cron-jobs.md:87](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/automation/cron-jobs.md#L87)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Project scope prefers optional channel capability to live as plugins maintained outside core:** VISION.md says core should stay lean, optional capability should usually ship as plugins, plugin distribution should be via npm/local extension loading, and plugin promotion belongs in ClawHub rather than core.
  - file: [VISION.md:52](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/VISION.md#L52)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Issue discussion already pointed to external plugin ownership:** The existing comment from `w-sss` identified the root cause as a WeChat plugin design limitation around token availability in isolated sessions and recommended follow-up in the WeChat plugin rather than OpenClaw core.

## Risks / Open Questions

- If the real defect is broader than WeChat and core is marking failed plugin sends as delivered across multiple channels, that would warrant a new core bug with a cross-channel reproduction.
