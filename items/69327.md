## Summary

Keep #69327 open. Current main has related sandbox improvements for workspace handoff, Docker env application, config-hash recreation, and `--agent` recreate filtering, but it does not close the specific subagent inheritance gap described in the issue comment: native `sessions_spawn(..., sandbox="inherit")` still does not carry the parent’s resolved sandbox Docker env/config into the child spawn context.

## Best Possible Solution

Keep the issue open. Add a regression test for `sessions_spawn(runtime="subagent", sandbox="inherit")` from a sandboxed parent with `sandbox.docker.env` and a changed `setupCommand`, then either thread the parent resolved `SandboxContext.docker`/env into native subagent sandbox resolution or explicitly define and document target-agent-only semantics. Preserve the existing real-workspace handoff and config-hash recreation behavior while narrowing any remaining stale-workspace/bootstrap reports with a concrete reproduction.
