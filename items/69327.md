## Summary

Keep #69327 open. The current code has strong evidence for the workspace-inheritance and recreate pieces, but I do not see a current-main implementation that threads a parent sandbox Docker environment/context through `sessions_spawn(sandbox="inherit")`. The commenter’s root-cause analysis still matches the source: spawned tool context carries workspace/group metadata only, `spawnSubagentDirect` treats `inherit` as a sandbox requirement mode, and `resolveSandboxContext` rebuilds the child sandbox from the child session/agent config rather than a parent sandbox context.

## Best Possible Solution

Keep the issue open and add a focused regression around `sessions_spawn` from a sandboxed parent with `sandbox.docker.env` configured. If intended semantics are parent inheritance, thread parent `SandboxContext.docker.env` or a narrowed parent sandbox context through the spawned-run path and merge it before target-agent overrides. If intended semantics are target-agent-only, update docs/tool text for `sandbox="inherit"` and add diagnostics that show which agent config supplied the child sandbox env.
