## Summary

Keep open. Current `main` still has the mismatch the issue reports: the docs show `pidsLimit: 256` as an example value, but the sandbox config resolver does not apply any default `pidsLimit`, and Docker only receives `--pids-limit` when the user explicitly configures a positive number.
