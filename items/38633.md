## Summary

Keep open. Current main still shows `pidsLimit: 256` in the user-facing sandbox config example, while the resolver leaves `pidsLimit` unset unless explicitly configured and Docker create args only emit `--pids-limit` for a positive configured value. The related docs PR #38914 would have clarified this but is closed unmerged, and #38604 tracks the broader security/default question rather than fully superseding this docs mismatch.

## Best Possible Solution

Keep this issue open until maintainers choose and land one of the two concrete fixes: either add an intentional default `pidsLimit` in `src/agents/sandbox/config.ts` with regression coverage and docs updates, or update `docs/gateway/config-agents.md` and related public docs to state that `pidsLimit: 256` is only an example and no Docker process limit is applied unless the user configures it. Coordinate with the broader open security issue #38604 rather than treating the closed, unmerged PR #38914 as sufficient.
