## Summary

Keep #38633 open. Current main still shows `pidsLimit: 256` in the user-facing sandbox Docker config example while the implementation leaves `pidsLimit` unset unless explicitly configured, and the related fix/clarification PRs in the supplied context are closed unmerged.

## Best Possible Solution

Keep #38633 open until either the docs explicitly say `pidsLimit` is unset unless configured, or the sandbox resolver adds and documents a real default. The code-level security default question should stay coordinated with #38604; this issue can be resolved by a narrow docs clarification if maintainers do not want a runtime default.
