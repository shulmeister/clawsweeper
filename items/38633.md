## Summary

Keep #38633 open. Current main still has the reported docs/runtime mismatch: user-facing sandbox config docs show `pidsLimit: 256`, while runtime resolution leaves `pidsLimit` unset unless configured and Docker args only emit `--pids-limit` for a positive configured value. Related PRs #38621 and #38914 are closed unmerged, and #38604 remains the broader open security/default discussion.

## Best Possible Solution

Keep #38633 open until maintainers either land a narrow docs clarification that `pidsLimit` is unset unless explicitly configured, or add a real default in `resolveSandboxDockerConfig` with matching tests and docs, coordinated with #38604.
