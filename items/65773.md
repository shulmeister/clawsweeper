## Summary

Keep open. Current main still embeds the Slack app manifest JSON inside the Slack setup intro note, so the copy/paste bug tracked by #65751 is not fixed on main. This PR targets the right behavior, but its proposed `console.log`/`prepare()` approach needs revision because review comments correctly identify env-shortcut and gateway/session delivery regressions.

## Best Possible Solution

Keep the Slack manifest bug and implementation candidate open, but request a revised fix. The likely maintainer path is to make the manifest copyable outside the framed clack note while still using a wizard/session-capable transport, preserving env-shortcut suppression, and adding focused regression coverage for CLI output and gateway/session delivery.
