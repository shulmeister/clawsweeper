## Summary

Keep this PR open. Current main still resolves exec approvals file and socket paths through the home-relative `~/.openclaw` defaults, while `OPENCLAW_STATE_DIR` is documented and implemented as the mutable state root elsewhere. The PR discussion is useful: Greptile agreed the path-resolution fix was focused, and the Codex review identified the important legacy fallback boundary that the author addressed in commit `09ef697995f09852041748c51d0cbd63bb04baf3`. Related #62938 appears to cover the same remaining bug, but it is also open, so this PR is still a valid implementation candidate until maintainers choose and land a canonical fix.

## Best Possible Solution

Keep this PR open while maintainers review the state-dir bug and choose a canonical implementation, likely coordinated with #62938. The final patch should make exec approvals file and socket resolution respect an explicit `OPENCLAW_STATE_DIR` via the existing state-dir resolver, preserve the current `~/.openclaw` default and legacy fallback behavior when no override is set, update effective-policy reporting for the overridden path, and add focused regression tests for path/socket resolution and store writes.
