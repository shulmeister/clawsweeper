## Summary

Keep PR #65736 open. Current main still ignores `OPENCLAW_STATE_DIR` for exec approvals: the resolver expands hardcoded `~/.openclaw/exec-approvals.*` paths, and all read/write/ensure paths flow through those helpers. The PR is a focused core bug fix with regression coverage for a behavior that docs already promise for mutable state roots; related issue #29736 tracks the same user problem but does not supersede this useful implementation.

## Best Possible Solution

Keep #65736 open. The best maintainer path is to land a focused fix that makes exec approvals use `resolveStateDir()` only for explicit `OPENCLAW_STATE_DIR` overrides, preserves existing default `~/.openclaw` behavior and legacy fallback boundaries, updates effective-policy path reporting, and adds the proposed regression tests. Cross-link #29736 as the problem report if needed.
