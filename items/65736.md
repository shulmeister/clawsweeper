## Summary

Keep open. Current `main` still ignores `OPENCLAW_STATE_DIR` for exec approvals file/socket resolution and still reports the default `~/.openclaw` approvals path in effective-policy summaries, while the rest of core path resolution and deployment docs treat `OPENCLAW_STATE_DIR` as the mutable state root. The PR addresses a live mismatch, not an obsolete item, though the review discussion shows it still needs maintainer judgment on the exact implementation shape.

## Evidence

- **Current main hardcodes exec approvals to ~/.openclaw:** `resolveExecApprovalsPath()` and `resolveExecApprovalsSocketPath()` unconditionally expand `~/.openclaw/exec-approvals.{json,sock}`. Read/write call sites (`readExecApprovalsSnapshot`, `loadExecApprovals`, `saveExecApprovals`) all use those resolvers, so current main still reads and writes approvals under the home-relative default rather than the configured state dir.

## Risks / Open Questions
