## Summary

Keep #72058 open. Current main has adjacent bundled runtime-deps hardening for doctor/startup staging and mirror reuse, but the npm global postinstall path still treats an existing dependency package.json sentinel as installed without validating the installed version. The issue is fresh, has same-day confirmations/workarounds, and the provided context shows open follow-up PR #72084 plus open related issue #72114 rather than a shipped fix or canonical closed replacement.

## Best Possible Solution

Keep #72058 open until PR #72084 or an equivalent patch lands with a regression test proving stale bundled runtime dependency versions are reinstalled during npm global postinstall/update. Then use #72114, or a maintainer-selected replacement, as the canonical tracker for any remaining cold-cache staging timeout/shutdown-race behavior before closing the broad user regression.
