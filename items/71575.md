## Summary

Keep open. Current main has tightened some related streaming paths, but it still intentionally shows Telegram tool-progress preview text by default when preview streaming is active. That means the central requested behavior, final-only Telegram delivery during tool use, is not implemented as the default/current behavior.

## Best Possible Solution

Keep #71575 open for maintainer product judgment. If final-only Telegram delivery should be the default, change the Telegram preview/tool-progress defaults, likely by making `streaming.preview.toolProgress` default false for Telegram or disabling preview streaming by default, then update Telegram docs and the default-progress tests. If the current progress-preview UX is intentional, close manually with that rationale and point users to `channels.telegram.streaming.preview.toolProgress: false` or `channels.telegram.streaming.mode: "off"`.
