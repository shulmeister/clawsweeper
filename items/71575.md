## Summary

Current main still intentionally shows Telegram tool-progress preview updates by default when preview streaming is active. The issue is recent, unprotected, and not superseded: the related #71663 was closed as a duplicate of this broader Telegram final-only/default-hidden behavior request.

## Best Possible Solution

Keep this issue open for a Telegram-owned behavior change. The likely fix is to make Telegram hide tool/progress previews by default, or add a final-only default mode while preserving an explicit opt-in for users who want live tool progress, then update extensions/telegram/src/bot-message-dispatch.ts, Telegram dispatch tests, docs/channels/telegram.md, and docs/concepts/streaming.md together.
