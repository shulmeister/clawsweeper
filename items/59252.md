## Summary

Current `main` already addresses the reported HTTP failover/live-switch race: user-only live-switch gating prevents fallback rotation from being mistaken for a pending session model change, and fallback handling now normalizes `LiveSessionModelSwitchError` into the failover path instead of bouncing back into an outer retry loop. The changelog shows the broader live-switch fix shipped in `v2026.4.5`, and current code/tests/docs cover the exact stale-session/fallback scenario described here.
