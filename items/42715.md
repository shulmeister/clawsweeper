## Summary

Keep open. Current `main` no longer appears to send unauthenticated requests for a remote custom provider with no `apiKey`; it now throws `No API key found for provider ...` before request-time auth resolution. But the broader reported behavior is still intentionally present: auth and missing-key failures are documented and tested as model-fallback triggers, and the fallback notice is only surfaced in verbose mode. That is not strong evidence for a conservative close.
