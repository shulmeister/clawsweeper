## Summary

Keep PR #71612 open. Current main still retains `toolCallHistory` across a stable cron session key and has no idle-gap or run-boundary reset in `recordToolCall`; the PR is a focused bug fix with regression tests for that remaining behavior.

## Best Possible Solution

Keep the PR open for maintainer review. The best fix is to scope tool-loop history to an actual isolated cron run boundary, either by adopting the PR's idle-gap reset with regression coverage or by passing/resetting a per-run diagnostic key for cron tool execution.
