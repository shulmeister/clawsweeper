## Summary

Current main still matches the reported Control UI Logs bug: the Logs view renders the `.log-stream` scroll container, and active desktop/mobile CSS still caps it at fixed pixel heights. The linked implementation candidate, #53916, remains the right path to review or replace, and this issue should stay open while that PR is open and unmerged.

## Best Possible Solution

Keep this issue open and review, refresh, and land #53916 if its CSS approach is acceptable. An equivalent fix should make the Control UI Logs `.log-stream` height responsive to the available viewport while preserving usable behavior on shorter screens; after that lands, this issue can close as implemented.
