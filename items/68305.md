## Summary

Keep open. Current main still emits the exact misleading `/reasoning stream` acknowledgement reported in #68305, while the Feishu plugin already has reasoning-preview support wired and tested. The linked PR #68311 appears to be the right small fix direction, but it is still open/unmerged in the supplied GitHub context, so the issue is not implemented on main.

## Best Possible Solution

Keep #68305 open until the directive acknowledgement is changed to channel-neutral wording or explicitly includes Feishu/Lark, and update the public reasoning docs so they no longer describe `/reasoning stream` as Telegram-only. The existing linked PR #68311 is a plausible small implementation path if maintainers choose that wording.
