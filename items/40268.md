## Summary

Current main still uses strict config validation on the gateway/read path, so the requested tolerant read behavior is not implemented. Startup and reload now have last-known-good recovery paths and doctor has a helper that can strip unknown keys, but those do not satisfy the issue's request to tolerate and warn on unknown keys during normal config reads.

## Best Possible Solution

Keep this issue open as the high-signal read-path bug. A proper fix would add an explicit tolerant read mode for gateway startup/hot-reload that strips or quarantines unknown keys, logs the exact paths, preserves strict validation for `config.set`/`config.patch`/`config.apply` writes, and adds regression tests for unknown root keys plus nested examples such as `gateway.channelHealthCheckMinutes` and misplaced hook/plugin config.
