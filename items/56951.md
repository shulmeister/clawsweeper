## Summary

Keep open. Current `main` still does not enable Slack inline/button capabilities by default at runtime; Slack remains config-gated, and the issue-linked PR head is not reflected in the checked-out `main` behavior.

## Evidence

- **Embedded runner still derives capabilities only from config/plugin hooks:** Both runner paths build `runtimeCapabilities` from `resolveChannelCapabilities(...)` and optional plugin `messageToolCapabilities`, but there is no Slack-specific default injection for `inlineButtons` or similar.

  - file: [extensions/slack/src/shared.ts:105](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/slack/src/shared.ts#L105)
