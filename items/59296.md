## Summary

Current `main` already preserves the pre-compaction session identity for plugin-facing lifecycle handling: compaction-triggered rotation emits `session_end` for the old session with `nextSessionId`, then `session_start` for the new session with `resumedFrom`. That covers the old/new ID handoff the report asked for, and the same implementation is present in the shipped `v2026.4.23` tree.

## Evidence

## Risks / Open Questions

- Plugins that still rely only on `before_compaction`/`after_compaction` will not see the old session id; the supported seam on current main for external sync during rotation is the compaction-triggered `session_end`/`session_start` lifecycle pair.
