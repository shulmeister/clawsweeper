## Summary

Current main already addresses the reported multi-agent OAuth drift behavior by merging peer auth state with the main agent store at runtime, remapping stale legacy `:default` OAuth references to fresher account-scoped profiles, and mirroring successful OAuth refreshes back into the main store. The same implementation is already present in release `v2026.4.23`.

## Evidence

- **Runtime uses main store as canonical peer source:** `loadAuthProfileStoreForRuntime()` merges the main agent auth store with the requested agent store for non-main agents, and subagents with no local auth store inherit secret-bearing profiles from main.

  - file: [src/agents/auth-profiles/persisted.ts:216](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/auth-profiles/persisted.ts#L216)
