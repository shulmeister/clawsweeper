## Summary

Keep PR #68444 open. Current main still has hardwired Feishu mention-forward behavior and no `mentionForwardMode` config or policy seam. The PR is a focused Feishu plugin fix for linked open issue #68357, with schema/policy/bot/docs coverage in the provided diff and only a non-blocking Greptile log-ordering nit in discussion.

## Best Possible Solution

Keep the PR open and land this fix or an equivalent Feishu-local implementation: add `mentionForwardMode: "auto" | "none"` to Feishu schema at top-level/account/group scope, resolve it in `resolveFeishuReplyPolicy`, pass effective mention targets into `buildFeishuAgentBody` and both dispatcher call sites, document the opt-out in Feishu channel docs, and keep schema/policy/bot regression tests covering default auto behavior plus disabled forwarding.
