## Summary

Keep open. Current main still does not visibly show the active config file path in Control UI Overview or prominently in Config/Settings. The backend already exposes the resolved config path for admin-scoped sessions and low-privilege tests verify it is withheld, but the Overview render path ignores it and the Config view only uses the path as an Open button tooltip. Related PR #54025 was closed unmerged, so it did not resolve the issue.

## Best Possible Solution

Add a visible, copyable active config path field in the Control UI Overview snapshot card or Config/Settings header, sourced from the existing admin-scoped hello snapshot or config.get snapshot path. Add focused UI coverage proving the path is rendered for authorized Control UI sessions and not exposed through low-privilege snapshots.
