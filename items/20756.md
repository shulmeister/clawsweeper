## Summary

Keep #20756 open. Current main still does not auto-select the only enabled account when `message` sends omit `accountId`; the shared action path only forwards explicit/tool/agent-binding account IDs, and WhatsApp still falls back to list-based default account resolution that ignores enabled state. The related fix PR #20801 was closed unmerged.

## Best Possible Solution

Keep #20756 open. Implement shared outbound account resolution after channel selection in the message action path: preserve explicit/tool/agent-binding account IDs, respect a usable configured/default account when no explicit account was provided, auto-select exactly one enabled account when no usable default exists, and return clear errors for zero or multiple enabled accounts. Add focused regression tests for `runMessageAction` and the WhatsApp disabled-default/single-enabled-account case, then update the public multi-account/channel routing docs.
