## Summary

Keep #43190 open. Current main does not include the proposed Teams channel archive plugin or the `channel_deleted` lifecycle hook, and the PR is not a clean ClawHub close because it exposes missing core/plugin API needed for an external archive plugin to work well.

## Best Possible Solution

Keep the item open, but steer it toward a smaller maintainer-reviewable path: split generic core/plugin API changes first, such as attachment metadata on `message_received` or a documented deleted-conversation lifecycle hook with tests/docs, then ship the archive implementation as a ClawHub/npm plugin unless maintainers explicitly want it bundled. Any bundled plugin path should address the remaining review comments and update labeler/docs/contracts.
