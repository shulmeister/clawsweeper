## Summary

Keep open. Current main still has a pre-delivery `message_sending` hook that can rewrite or cancel outbound content, but the shared outbound delivery path does not populate `ctx.sessionKey` or `ctx.runId`, and the hook event does not expose a bounded tool-call/result history needed for deterministic verification.

## Best Possible Solution

Keep #18571 open. The best fix belongs in the core plugin hook contract and shared outbound delivery path: propagate `sessionKey` and a real `runId` into `message_sending`/`message_sent`, expose a bounded documented transcript or tool-call/result history view for deterministic pre-send verification, and add focused regression tests for correlation plus cancel/rewrite behavior.
