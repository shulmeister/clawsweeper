## Summary

Keep #18571 open. Current main still has a `message_sending` hook that can rewrite or cancel outbound content, but it does not yet provide the requested deterministic verification contract: no consistently propagated `sessionKey`, no `runId`, and no bounded transcript/tool-call evidence view on the pre-send outbound paths. The existing maintainer-thread Codex review reached the same conclusion, and the current HEAD changes are unrelated to this hook surface.

## Best Possible Solution

Keep the issue open. Implement an additive plugin hook contract that propagates `sessionKey` and a real `runId` into `message_sending` and `message_sent` wherever OpenClaw sends outbound agent messages, exposes a bounded documented transcript/tool-call/result evidence view for verifier plugins, and adds regression tests for correlation plus cancel/rewrite behavior across shared outbound delivery and direct reply paths.
