## Summary

Keep #18571 open. Current main has `message_sending` cancellation/rewrite support and documented message hook correlation fields, but the shared outbound pre-send hook still is not invoked with `sessionKey` or `runId`, the outbound session carrier cannot propagate `runId`, and the hook event still has no bounded tool-call/result history or transcript handle for deterministic verification.

## Best Possible Solution

Keep the issue open and implement an additive plugin hook contract: propagate `sessionKey` and a real `runId` into `message_sending` and `message_sent` across shared outbound delivery and direct reply paths, expose a bounded documented transcript or tool-call/result-history view suitable for deterministic pre-send verification, and add regression tests for correlation plus cancel/rewrite behavior.
