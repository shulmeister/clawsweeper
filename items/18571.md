## Summary

Keep open. Current main still has a pre-delivery `message_sending` hook that can rewrite or cancel outbound content, but the shared delivery path still does not pass `sessionKey` or `runId` into that hook, and there is no first-class tool-call/result history payload for deterministic pre-send verification. Newer `before_agent_finalize` support helps some final-answer checks, especially for Codex Stop relay, but it does not replace the requested outbound-message correlation and delivery-gating surface.

## Best Possible Solution

Keep #18571 open. The likely fix belongs in core/plugin hook contracts: propagate session correlation through the shared outbound delivery path, add or derive `runId` for agent-originated sends, pass `ctx.sessionKey`/`ctx.runId` into `message_sending` and `message_sent`, and expose a bounded, documented tool-call/history view or transcript handle for deterministic pre-delivery verification. Add focused outbound hook tests covering session/run correlation and cancellation/rewrite behavior.
