## Summary

Current `main` appears to have fixed the active-model part and supports storing a label on the main session entry, but the TUI header still uses the parsed session key instead of a configurable label/display name. The issue is therefore only partially implemented and should stay open.

## Evidence

- **TUI header still ignores session display name/label:** `updateHeader()` formats the header from `formatSessionKey(currentSessionKey)` only, so the top TUI header still shows the session key (`main`, etc.) rather than `sessionInfo.displayName` or `label`.
  - file: [src/tui/tui.ts:575](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/tui/tui.ts#L575)

  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **TUI footer now shows active model and display name:** `updateFooter()` uses `sessionInfo.displayName` for the session label and `sessionInfo.modelProvider/sessionInfo.model` for the model, so the footer reflects human-friendly names and active model state when available.
  - file: [src/tui/tui.ts:816](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/tui/tui.ts#L816)
