## Summary

Keep this PR open. Current main still creates `.openclaw-config-doc-cache` under the supplied repo root and only removes the per-run isolated child directory, so the PR addresses a real remaining repository side effect. The existing regression test covers that the isolated retry succeeds with repo-root dependency resolution, but current main does not assert cleanup of the parent cache directory.

## Best Possible Solution

Keep the PR open for maintainer review and merge after validation. The best fix is the PR's narrow cleanup: preserve the repo-root retry parent for dependency resolution, remove the empty `.openclaw-config-doc-cache` parent after the isolated copy is deleted, tolerate `ENOENT`/`ENOTEMPTY`, and keep the regression assertion that the retry succeeds without leaving repo-root residue.
