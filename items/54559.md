## Summary

Keep this issue open. Current main still matches the reported behavior: `generic_repeat` is warn-only, warnings do not block tool execution, `process(action="list")` is not treated as a known poll tool, and tests still preserve the warn-only behavior until the global circuit breaker.

## Best Possible Solution

Keep this issue open until main changes. The best fix is a narrow runtime update in `src/agents/tool-loop-detection.ts` that checks generic repeats against `criticalThreshold` before the warning branch, plus focused updates to `src/agents/tool-loop-detection.test.ts`, `src/agents/pi-tools.before-tool-call.e2e.test.ts`, and the loop-detection docs/config help so shipped semantics match the public contract. Maintainers should separately decide whether `process(action="list")` belongs in known poll/no-progress detection and review or replace open PR #60248 rather than closing this issue before a merge.
