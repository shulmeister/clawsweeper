## Summary

Current main does not satisfy the requested dedicated Dreaming LLM model behavior. There is a partial `dreaming.execution.*.model` config shape in the resolver, but the actual Dream Diary subagent call does not pass a provider/model override, and the public docs still state that Dreaming uses the default runtime model. The existing `agents.defaults.memorySearch` embedding model configuration addresses the commenter’s embedding-provider point, but it is a different path from the requested chat/subagent model used for Dreaming narrative work.

## Best Possible Solution

Keep the issue open and implement a documented Dreaming model override in `memory-core`: resolve a global and/or per-phase Dreaming model from `plugins.entries.memory-core.config.dreaming`, pass it to the plugin subagent runtime when generating Dream Diary entries, preserve current default-model inheritance when unset, add regression tests around the `subagent.run` payload, and update `docs/concepts/dreaming.md` plus `docs/reference/memory-config.md`.
