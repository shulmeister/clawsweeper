## Summary

Keep open. Current main still has only partial session-level send suppression, not the requested per-account channel read-only mode or hard gateway-level no-outbound guarantee. The open WhatsApp PR #71382 is a relevant implementation candidate for one bundled channel, but it is not merged and does not cover the broader generic channel/account policy requested here.

## Best Possible Solution

Keep this issue open. The best path is a generic channel/account outbound delivery policy, exposed through shared channel/account config and schema, enforced before gateway/direct outbound delivery and before message-action/plugin handoff. It should cover text, media, polls/actions, typing, reactions, read receipts, and channel-specific transport side effects where supported, with docs/config metadata/UI hints and regression coverage for auto-reply, direct gateway sends, message-tool sends, and at least one bundled account-based channel such as WhatsApp. Coordinate with #71382 for the WhatsApp transport work.
