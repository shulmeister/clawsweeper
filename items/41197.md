## Summary

Current `main` already restarts the gateway by default after a successful `openclaw update`, and the macOS update path now uses a detached launchd restart helper with bootstrap fallback plus regression coverage. The reported behavior appears implemented/fixed on current `main`.

## Evidence

- **CLI default is restart-on-success:** `openclaw update` only exposes `--no-restart`; restart is the default behavior.
  - file: [src/cli/update-cli.ts:37](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/cli/update-cli.ts#L37)
- **Update command schedules restart unless explicitly disabled:** `updateCommand` sets `shouldRestart = opts.restart !== false`, prepares restart follow-ups for loaded services, and calls `maybeRestartService(...)` after a successful update.
  - file: [src/cli/update-cli/update-command.ts:972](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/cli/update-cli/update-command.ts#L972)
- **Successful update runs restart and verifies health:** `maybeRestartService` refreshes the service env, runs the detached restart script or daemon restart, and waits for a healthy restarted gateway.
  - file: [src/cli/update-cli/update-command.ts:729](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/cli/update-cli/update-command.ts#L729)
- **macOS update restart helper re-bootstrap path exists:** The macOS restart helper tries `launchctl kickstart -k`; if that fails it enables the service, bootstraps the LaunchAgent again, and kickstarts it, which addresses unloaded LaunchAgent cases.
  - file: [src/cli/update-cli/restart-helper.ts:110](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/cli/update-cli/restart-helper.ts#L110)
- **Regression coverage for update restart path:** Tests assert that a normal update refreshes the installed service and invokes the restart script, with fallback to `runDaemonRestart()` when needed.
  - file: [src/cli/update-cli.test.ts:1440](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/cli/update-cli.test.ts#L1440)
- **Release notes identify shipped macOS restart recovery:** The changelog for `2026.4.5` includes macOS gateway restart recovery entries: let launchd `KeepAlive` own restarts again, re-bootstrap LaunchAgents if `kickstart` unloads them, and recover installed-but-unloaded LaunchAgents on start/restart.
  - file: [CHANGELOG.md:1249](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/CHANGELOG.md#L1249)

## Close Comment

Closing as implemented on current `main` after Codex review. The current update path restarts the gateway by default unless `--no-restart` is passed (`src/cli/update-cli.ts:37`), performs restart follow-up after successful updates (`src/cli/update-cli/update-command.ts:729`, `src/cli/update-cli/update-command.ts:972`), and the macOS-specific helper now re-runs `launchctl kickstart`, then falls back to `launchctl enable` + `launchctl bootstrap` if the LaunchAgent was unloaded (`src/cli/update-cli/restart-helper.ts:110`). There is regression coverage for both the normal update restart path and the macOS helper (`src/cli/update-cli.test.ts:1440`, `src/cli/update-cli/restart-helper.test.ts:137`). The changelog also shows the LaunchAgent restart recovery shipped in the `2026.4.5` line (`CHANGELOG.md:1249`). If this is still reproducible on a current release, please open a fresh issue with exact version, `openclaw gateway status --deep`, and the gateway restart log so it can be triaged as a new regression.
