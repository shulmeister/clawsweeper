## Summary

Keep open. Current `main` still does not implement `openclaw doctor --dry-run` or a doctor-wide no-write validation mode. The issue is recent, author is not a maintainer, has no protected labels, and the related global agent dry-run proposal #41418 is a separate layer rather than a duplicate of this doctor/config-repair request.

## Best Possible Solution

Keep #13362 open until OpenClaw either implements a doctor-specific `--dry-run` mode or a maintainer explicitly rejects the product direction. The likely implementation belongs in the doctor CLI registration, `DoctorOptions`, config repair/finalization flow, write-config contribution, exit-code behavior, docs, and focused tests proving validation and fix recommendations still run while no config/state writes occur.
