## Summary

Keep #13362 open. Current main at 82262387651ab270b565f71533ce4937c9c00dc6 still does not expose `openclaw doctor --dry-run` or an equivalent doctor-wide no-write validation mode, and the recent maintainer/Codex issue comment explicitly kept this open. The related global agent dry-run issue #41418 is complementary, not a replacement for doctor/config repair preview.

## Best Possible Solution

Keep #13362 open until OpenClaw either implements or explicitly rejects a doctor-specific dry-run mode. The best implementation is to add `openclaw doctor --dry-run`, thread `dryRun` through `DoctorOptions`, repair-mode resolution, prompting, config flow, and every doctor contribution, make all write/repair paths emit planned changes without mutating config, state, services, runtime dependencies, or plugin/channel-owned repairs, define exit-code behavior for findings, update `docs/cli/doctor.md`, and add focused regression tests proving dry-run is no-write across the doctor surfaces.
