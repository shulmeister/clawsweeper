## Summary

Keep #13362 open. Current main still does not expose `openclaw doctor --dry-run` or an equivalent doctor-wide no-write validation mode, and the existing doctor flow can still write config, cron state, service config, and bundled plugin runtime dependency state through repair or confirmed preview paths. The maintainer/Codex comment already posted on 2026-04-26 also explicitly keeps this issue open, and the related global agent dry-run issue #41418 is complementary rather than a replacement.

## Best Possible Solution

Keep #13362 open until OpenClaw implements or explicitly rejects a doctor-specific dry-run mode. The likely product/code path is to add `openclaw doctor --dry-run`, thread `dryRun` through `DoctorOptions`, make every doctor repair/write contribution produce recommendations without mutating config/state/services/runtime deps, define exit-code behavior for detected issues, update https://docs.openclaw.ai/cli/doctor, and add focused tests proving dry-run does not mutate config or service state.
