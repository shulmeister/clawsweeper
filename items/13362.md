## Summary

Keep open. Current `main` still does not expose `openclaw doctor --dry-run`, does not carry a doctor-wide `dryRun` option through the command flow, and the existing non-repair preview path can still prompt to apply config repairs rather than guaranteeing no writes. Related #41418 remains a separate global agent-tool dry-run proposal and does not supersede this doctor/config-validation request.

## Best Possible Solution

Keep #13362 open until OpenClaw either implements a doctor-specific `--dry-run` mode or a maintainer explicitly rejects the product direction. The likely implementation belongs in the doctor CLI registration, `DoctorOptions`, config repair/finalization flow, write-config contribution, exit-code behavior, docs, and focused tests proving no writes occur while validation and fix recommendations still run.
