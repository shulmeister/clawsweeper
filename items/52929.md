## Summary

Current `main` already implements stale session-lock cleanup for `.jsonl.lock` files during gateway startup and in the shared session-lock utility, with regression coverage. The fix appears to have landed after the latest tagged release, so the issue is fixed on `main` but not yet tied to a shipped release from the evidence available here.
