## Summary

Keep open. Current main has model fallback and overflow compaction, but it does not implement the requested per-candidate retry policy or configurable fallback context mode. A related contributor PR, #60984, was closed unmerged, so it is not a shipped superseding fix.

## Best Possible Solution

Keep this issue open and design a small agents fallback policy API in the existing agent model config surface. The implementation should add bounded per-candidate retries only for classified transient failures, thread retry metadata through model-fallback observations/tests, and define how fallback context safe/strict mode compacts or refuses smaller-context fallbacks without replaying side effects. The closed unmerged #60984 can be used as prior discussion, but the final shape should align with current runEmbeddedPiAgent overflow recovery and config-schema conventions.

## Risks / Open Questions

- Retrying a failed model after a partial or side-effecting attempt needs careful replay/side-effect gating, especially across tool calls and messaging-tool sends.
- Fallback context compaction must preserve current transcript and plugin/context-engine contracts; a safe/strict config surface needs schema, docs, and focused regression tests.
