## Summary

Keep this PR open. Current main still gates Slack lifecycle status reactions through the ack-reaction decision, so `messages.statusReactions.enabled: true` alone does not activate the controller. The PR is a narrow implementation candidate, with one small test assertion cleanup already noted in review.

## Best Possible Solution

Keep this PR open and review or adapt its narrow Slack prepare change: let `messages.statusReactions.enabled === true` activate lifecycle reactions independently of the ack-reaction gate, preserve `enabled: false` as an explicit disable, keep the existing ack-reaction fallback behavior, and address the existing review note in the explicit-disable test.
