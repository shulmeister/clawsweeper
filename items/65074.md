## Summary

Keep open. Current `main` still builds the bundled `codex-cli` backend around `--json`/resume text output and finalizes replies from stdout-parsed CLI output; there is no `--output-last-message` support or artifact-preferred finalization path in the checked-out code.

## Evidence

- **Bundled codex-cli backend still uses stdout/JSONL modes only:** The bundled OpenAI Codex CLI backend passes `exec --json` for fresh runs and plain-text resume output. Its config exposes `output: "jsonl"` and `resumeOutput: "text"`, but does not add `--output-last-message` or any artifact path flag.
  - file: [extensions/openai/cli-backend.ts:23](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/extensions/openai/cli-backend.ts#L23)
  - command: `nl -ba extensions/openai/cli-backend.ts | sed -n '1,120p'`
  - sha: [ae5c6573670b](https://github.com/openclaw/openclaw/commit/ae5c6573670b3742b156899bd2b9c36b8677f191)
- **CLI runner builds argv from backend config only:** `executePreparedCliRun()` constructs `args` from backend args, model/session/system prompt/image inputs, then spawns the child with `[backend.command, ...args]`. There is no temp artifact allocation, no extra result-file argument injection, and no post-run artifact cleanup path here.
  - file: [src/agents/cli-runner/execute.ts:179](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/src/agents/cli-runner/execute.ts#L179)
  - command: `nl -ba src/agents/cli-runner/execute.ts | sed -n '179,419p'`
  - sha: [ae5c6573670b](https://github.com/openclaw/openclaw/commit/ae5c6573670b3742b156899bd2b9c36b8677f191)
- **Final reply is derived from stdout parsing after process exit:** After the managed run exits, the runner trims `result.stdout`, feeds that raw stdout into `parseCliOutput(...)`, and returns the parsed text as the final CLI output. No file read happens after exit.
  - file: [src/agents/cli-runner/execute.ts:455](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/src/agents/cli-runner/execute.ts#L455)
  - command: `nl -ba src/agents/cli-runner/execute.ts | sed -n '455,538p'`
  - sha: [ae5c6573670b](https://github.com/openclaw/openclaw/commit/ae5c6573670b3742b156899bd2b9c36b8677f191)
- **Output parser contract is stdout-based:** `parseCliOutput()` handles only `text`, `json`, and `jsonl` modes from the provided raw output string. In text mode it returns `params.raw.trim()` directly; JSON/JSONL modes also fall back to trimmed raw output.
  - file: [src/agents/cli-output.ts:467](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/src/agents/cli-output.ts#L467)
  - command: `nl -ba src/agents/cli-output.ts | sed -n '467,492p'`
  - sha: [ae5c6573670b](https://github.com/openclaw/openclaw/commit/ae5c6573670b3742b156899bd2b9c36b8677f191)
- **Docs still describe CLI finalization as parsing stdout output:** The public CLI-backends docs say OpenClaw 'Parses output (JSON or plain text) and returns the final text' and define `output: "text"` as treating stdout as the final response. That matches current code, not the proposed artifact-preferred contract.
  - file: [docs/gateway/cli-backends.md:139](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/docs/gateway/cli-backends.md#L139)
