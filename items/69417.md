## Summary

Keep open. Current main still does not support SecretRef/SecretInput values on the core `mcp.servers.*.env.*` or `mcp.servers.*.headers.*` paths. The existing ACPX/plugin SecretRef support is related but distinct: it covers `plugins.entries.acpx.config.mcpServers.*.env.*`, not OpenClaw's core `mcp.servers` config surface. The PR addresses a real remaining core gap and its review-thread edge cases were acknowledged and followed up on in the PR discussion.

## Best Possible Solution

Keep the PR open and either merge it after review or adapt its core changes into main: add `SecretInput` support to MCP env/header schema and types, register `mcp.servers.*.env.*` and `mcp.servers.*.headers.*` as SecretRef targets, resolve only active MCP transport surfaces at runtime, teach `secrets configure`/`secrets audit` the MCP sensitivity heuristics, and update the MCP and SecretRef docs/tests.
