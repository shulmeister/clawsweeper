## Summary

Keep open. Current main still has transcript/session-log based usage summaries and provider quota snapshots, but not the requested native append-only per-call usage ledger with event ids, source/session/run attribution, and ledger-first readers. The recent issue discussion explicitly distinguishes related transcript-backed work from the ledger design, and current main has not changed the relevant usage files since that keep-open review.

## Best Possible Solution

Keep the issue open until OpenClaw adds a typed append-only usage ledger written from the normalized model-call usage/cost seam, with `agentId`, `sessionKey`, `sessionId`, `runId`, source, provider/model, token/cache/cost fields, idempotency, and no prompt/completion content. Then make `usage.cost`, session usage readers, CLI/UI summaries, and any backfill path ledger-first with transcript scanning as compatibility fallback, while keeping provider quota snapshots separate.
