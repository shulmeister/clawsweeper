## Summary

Keep open. Current main still provides usage visibility through provider quota snapshots and transcript/session-log scanning, not the requested native append-only per-call usage ledger with stable event ids, model/source/session/run attribution, and ledger-first readers. Related #49064 and #70205 are still transcript-backed/narrower work, and the latest maintainer discussion explicitly preserves this issue for the ledger design.

## Best Possible Solution

Keep the issue open until OpenClaw has a typed `UsageLedgerEntry` and append-only storage path written from the normalized model-call usage/cost seam, with `agentId`, `sessionKey`, `sessionId`, `runId`, source, provider/model, token/cache/cost fields, idempotency, and no prompt/completion content. Then make `usage.cost`/session usage readers ledger-first with transcript fallback/backfill and document the privacy and compatibility behavior.
