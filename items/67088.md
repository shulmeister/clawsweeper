## Summary

Keep open. Current main still has the reported control-flow bug: `resolveBrowserOpenCommand()` checks `SSH_*` plus missing `DISPLAY` before the macOS branch, so a normal macOS shell with lingering SSH variables still returns `ssh-no-display` and makes `openclaw dashboard` print the SSH/no-GUI hint. The linked fix PR #67115 is still open, and #67110 was closed unmerged, so the issue is not implemented or safely superseded yet.

## Best Possible Solution

Keep #67088 open until a targeted fix lands. The likely fix belongs in `src/infra/browser-open.ts`: restrict the `ssh-no-display` short-circuit to Linux, or otherwise exempt `darwin` before returning `ssh-no-display`, then add focused regression coverage for `darwin` with lingering `SSH_*` vars and no `DISPLAY`, plus a preserved Linux SSH/no-display case. PR #67115 is the canonical open candidate to review or replace with an equivalent patch.
