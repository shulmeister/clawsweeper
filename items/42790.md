## Summary

Current `main` already skips the off-mode Tailscale hostname probe that would have called `tailscale status --json`, so the PR's original ENOENT motivation looks obsolete. But the PR also changes the `status --all` diagnosis to treat `tailscaleMode: "off"` as healthy, and that behavior is still not present on `main`, so there is still user-visible work here and the PR is not safely closable as implemented-on-main.

## Evidence

- **Off-mode probe is already gated on main:** `createStatusScanCoreBootstrap()` sets `tailscaleDnsPromise` to `Promise.resolve(null)` when `gateway.tailscale.mode` is `"off"`; only the non-off branch calls `getTailnetHostname(...)`.
