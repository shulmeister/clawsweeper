## Summary

Current main already implements PR #42790's core fix: `status --all` now skips the Tailscale hostname/CLI probe when `gateway.tailscale.mode` is `off`, so the disabled-mode ENOENT warning path is obsolete.

## Best Possible Solution

Close PR #42790 as obsolete against current main. Keep the shipped shared status scan gate for the ENOENT/no-probe fix, and handle any remaining off-mode daemon visibility work in the separate focused status follow-up rather than reviving this old patch.
