## Summary

Keep #69541 open. Current main still has adjacent prompt-injection hooks and several point sanitizers, but it does not implement the requested manifest-level `contracts.injectedContextTags`, a runtime injected-context registry/getter, or registry-backed consumer migration. The issue is recent, contributor-authored, unlabelled, and the related items are adjacent rather than a canonical replacement.

## Best Possible Solution

Keep the RFC open as a core/plugin-contract design item. A good implementation would add a backwards-compatible `contracts.injectedContextTags` manifest type with loader validation, expose an aggregated runtime helper, and migrate consumers incrementally starting with display/session sanitizers and memory-core dreaming ingestion while retaining existing hardcoded defaults as fallback.

## Risks / Open Questions

- Closing this would lose a still-unimplemented public plugin/core contract RFC while current consumers continue to maintain independent hardcoded strip lists.
- The right implementation needs maintainer design review because it extends the public manifest/SDK contract and can affect prompt persistence, display, compaction, and memory ingestion behavior.
- Consumer migration must avoid over-stripping user-authored XML examples and should keep compatibility fallback defaults while plugins adopt the new declaration.
