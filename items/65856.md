## Summary

Keep open. The PR identifies a real metadata mismatch on current `main`, but the proposed fix is not conservative: current onboarding/auth-choice surfaces still hardcode `scope: "text-inference"`, so changing Runway and Alibaba to `onboardingScopes: ["image-generation"]` would hide them from interactive setup and CLI auth-choice help rather than moving them to an existing image/video wizard.

## Risks / Open Questions

- Merging the PR as written would regress discoverability/setup for Runway and Alibaba by removing them from the only shipped interactive auth-choice surface.
- There is still a product gap on current main: video-only providers inherit the text-inference default until a dedicated media/image onboarding surface exists or the auth-choice UX is reworked.
