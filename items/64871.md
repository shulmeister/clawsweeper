## Summary

Keep open. Current `main` still caches workspace bootstrap files for the full lifetime of a `sessionKey` without reloading on later turns, and the existing tests explicitly lock in that behavior. The PR addresses a real, still-unfixed gap; the in-thread review only raises a performance trade-off, not a correctness objection.

## Evidence
