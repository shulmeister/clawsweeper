## Summary

Keep #27482 open. Current main still supports video understanding through media-understanding and `openclaw infer video describe`, but the requested direct video upload path through the chat/message attachment surface remains image-only in the Control UI and Gateway chat pipeline.

## Best Possible Solution

Keep #27482 open until core supports first-class video chat attachments: widen Control UI accept/type guards and previews, extend Gateway attachment parsing/transcript metadata beyond `ChatImageContent`, route accepted video files through provider-native video input or the existing media-understanding video path with capability checks and limits, and add focused UI/Gateway/tests/docs coverage for MP4, WebM, and MOV.
