## Summary

Keep #27482 open. Current main still excludes direct video uploads from the Control UI chat picker, intentionally filters MP4/WebM/MOV-style files in tests, drops `video/*` attachments in Gateway `chat.send` parsing, and dispatches only image attachments to the agent/model path. Adjacent media-understanding and video-generation surfaces exist, but they do not implement the requested direct video-file upload-to-LLM chat workflow.

## Best Possible Solution

Keep #27482 open until OpenClaw adds first-class direct video chat attachments across the Control UI and Gateway path: accept and preview MP4/WebM/MOV, define a video-capable attachment/content model, route provider-native video input or media-understanding summaries by model/provider capability and size policy, persist transcript metadata, and cover UI, Gateway parsing, provider dispatch, and docs with focused tests.
