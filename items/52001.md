## Summary

Current `main` already uses provider-qualified model values throughout the Control UI model picker flow, and the selected value is forwarded unchanged to `sessions.patch`. The current tests also cover duplicate/shared model cases and verify distinct qualified values, so the reported wrong-prefix behavior is implemented/fixed on `main` even though it was likely real in older releases such as v2026.3.13.
