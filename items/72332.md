## Summary

Keep PR #72332 open. Current main has already added some Bonjour/ciao cancellation handling, but it has not shipped the remaining Windows flicker and dependency-hardening work this PR targets: the TUI relaunch spawn still lacks `windowsHide`, the bundled Bonjour plugin still depends on unpatched `@homebridge/ciao@1.3.6`, and the upstream package still contains exec calls without `windowsHide` plus hard `process.exit(1)` paths. The PR is recent, linked to active user bug reports, and contains plausible useful work, so it should get maintainer review/refinement rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review, but ask for a focused follow-up revision: preserve the useful Windows flicker and Bonjour hardening fixes, add tests/manual verification for the exact Windows and mDNS conflict paths, avoid unrelated formatting churn, and get explicit approval for the `@homebridge/ciao` patch or replace it with an OpenClaw-owned wrapper/upstreamed dependency fix.
