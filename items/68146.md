## Summary

Keep open. Current `main` still fail-fast returns `UNAVAILABLE` for startup-gated `chat.history`/`models.list` until post-attach clears the gate, and only the Control UI chat path retries client-side. The PR's server-side barrier is not already implemented on `main`, so this is not closable as implemented or obsolete.
