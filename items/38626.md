## Summary

Keep #38626 open. Current main has coarse subagent/task records, `/subagents` list/info/log controls, task list/show/cancel, kill/steer, and internal orphan recovery, but it still does not implement the requested durable per-subagent lifecycle timeline, persisted tool/warning/error events, artifact-pointer completion payloads, first-failure/recommended-action diagnostics, or idempotent audit-logged cancel/steer/resume supervision surface. Related #62478 is closed background-spawn work and remains complementary, not a canonical replacement.

## Best Possible Solution

Keep the issue open unless a maintainer intentionally narrows or supersedes it. The best product/code path is a durable per-run event ledger for subagent/task spawn, queued/running, tool start/end, warning/error, completion, and supervision actions; persist it beside task/subagent records; expose it in `/subagents info/log`, `openclaw tasks show/list --json`, gateway/control UI surfaces, and docs; include artifact pointers plus first-failure/recommended-action diagnostics; and make cancel, steer, and operator resume idempotent with auditable action records. Treat #62478 as shipped complementary background-spawn work, not the canonical tracking item for this observability scope.
