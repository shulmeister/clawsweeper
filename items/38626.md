## Summary

Current `main` has partial subagent observability/control via the background task ledger and `subagents` controls, but it does not fully implement the requested lifecycle timeline, persisted artifact pointers, failure-signature/next-step payloads, or a public resume/audit-logged supervision surface.

## Evidence

- **Background tasks already cover detached subagent runs, listing, show, cancel, and state-change notifications:** Docs describe subagent spawns as tracked background tasks with `openclaw tasks list`, `show`, `cancel`, and optional `state_changes` notifications.
  - file: [docs/automation/tasks.md:15](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/docs/automation/tasks.md#L15)
  - sha: [51186d272543](https://github.com/openclaw/openclaw/commit/51186d2725439646f7fea0e59bb466f957404c46)
- **Public subagent control surface is limited to list/steer/kill:** The session-tool docs and tool implementation expose `list`, `steer`, and `kill`, but not resume or richer supervision/audit controls.
  - file: [src/agents/tools/subagents-tool.ts:23](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/src/agents/tools/subagents-tool.ts#L23)
  - sha: [51186d272543](https://github.com/openclaw/openclaw/commit/51186d2725439646f7fea0e59bb466f957404c46)
- **Task records persist only coarse status/timing/summary fields:** `TaskRecord` stores created/started/ended/lastEvent timestamps plus `error`, `progressSummary`, and `terminalSummary`, but no persisted lifecycle event timeline or artifact-pointer fields.
  - file: [src/tasks/task-registry.types.ts:53](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/src/tasks/task-registry.types.ts#L53)

## Risks / Open Questions

- Existing task-ledger and subagent-control work partially overlaps this issue, so future implementation should extend those surfaces instead of adding a second observability/control path.
- The repo documents active/recent task inspection and coarse timestamps, which could make the issue look partially done unless acceptance criteria are checked against the missing timeline/artifact/failure-signature requirements.
