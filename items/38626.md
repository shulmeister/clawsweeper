## Summary

Keep open. Current main has partial subagent/task observability and controls: active/recent listing, run metadata, transcript paths, terminal summaries, task cancellation, and subagent kill/steer/send controls. It still does not satisfy the central request for a persisted per-subagent lifecycle timeline with tool start/end and warning/error events, artifact-pointer summaries, first-failure signature plus recommended next action, or idempotent/audit-logged cancel/steer/resume supervision controls. The related #62478 is closed background-spawn work and remains complementary, not a canonical replacement for this observability/control issue.

## Best Possible Solution

Keep tracking this item until OpenClaw either implements a durable subagent lifecycle/supervision record or a maintainer deliberately narrows the scope. The likely product path is to add a persisted per-run event timeline for subagents/tasks, surface it through `/subagents info` or `openclaw tasks show` and runtime APIs, attach artifact pointers and terminal diagnostics, and make cancel/steer/resume responses idempotent with an auditable action log.
