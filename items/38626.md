## Summary

Keep open. Current main still has useful coarse subagent and background-task tracking, but it does not meet the issue’s acceptance criteria for a persisted per-subagent lifecycle timeline, tool start/end and warning/error history, artifact-pointer run summaries, first-failure signature with recommended next action, or idempotent audit-logged cancel/steer/resume supervision controls. The closed #62478 background-spawn item is complementary, not a canonical replacement for this observability/control issue.

## Best Possible Solution

Keep this issue open until OpenClaw adds a durable subagent/task lifecycle event model and operator API/UI that records spawn/queued/running/tool/warning/error/completion transitions, includes artifact pointers and first-failure guidance in terminal payloads, and makes cancel/steer/resume idempotent with an inspectable audit trail. The existing active/recent lists and task summaries should be treated as partial groundwork, not the finished product.
