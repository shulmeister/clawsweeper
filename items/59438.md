## Summary

Current `main` already contains a targeted Control UI fix for stale agent-avatar swaps during session changes, with regression coverage that matches this report; the issue timeline also cross-references the matching fix PR.

## Evidence

- **Timeline cross-reference:** The issue timeline already links closed PR `#59463`, titled `fix(ui): refresh chat avatar when switching agents`, which matches this report’s symptom and scope.
- **Per-session avatar guard on main:** `refreshChatAvatar` tracks request versions per host, captures the active `sessionKey`, and only applies fetched avatar results when the same host/session is still active. That directly prevents the cross-session avatar swap described in the issue.
  - file: [ui/src/ui/app-chat.ts:550](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/ui/src/ui/app-chat.ts#L550)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Regression test for agent switching:** The test `ignores stale avatar responses after switching sessions` starts an avatar fetch for `agent:main`, switches to `agent:ops` before the first response resolves, and asserts the stale `main` avatar is discarded while `/avatar/ops` wins.
  - file: [ui/src/ui/app-chat.test.ts:263](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/ui/src/ui/app-chat.test.ts#L263)
