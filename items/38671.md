## Summary

Keep open. Current main still lacks direct shell-command cron jobs and the proposed command+message hybrid path: cron payloads are still only systemEvent or agentTurn, CLI creation/editing exposes only --system-event/--message payloads, and execution still routes through heartbeat/system events or an isolated agent turn. The issue discussion adds concrete use cases and edge cases, and related PR #62789 remains an open implementation path rather than shipped or superseding evidence.

## Best Possible Solution

Keep tracking this issue until core cron supports a command payload end to end, or until maintainers explicitly reject the product direction. The likely implementation belongs in core cron/protocol/CLI/runtime docs, reusing existing exec security semantics, with PR #62789 considered as a possible starting point rather than a reason to close the issue now.
