## Summary

Keep open. Current `main` still models cron payloads as only `systemEvent` or `agentTurn`, the CLI/docs only expose `--system-event` and `--message`, and isolated cron execution still routes through agent runners with a prompt/model. `lightContext` only reduces bootstrap context; it does not add a direct shell-command mode or bypass inference.

## Evidence

- **Cron payload types only support system events or agent turns:** `CronPayload` is a union of `{ kind: "systemEvent" }` and `CronAgentTurnPayload`; the agent-turn payload has `message`, `model`, `thinking`, `lightContext`, and `toolsAllow`, but no `command` field or direct-exec mode.
  - file: [src/cron/types.ts:112](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/src/cron/types.ts#L112)

  - sha: [51186d272543](https://github.com/openclaw/openclaw/commit/51186d2725439646f7fea0e59bb466f957404c46)
- **CLI creation path still requires message/system-event payloads:** `openclaw cron add` exposes `--system-event` and `--message`, then errors unless exactly one is provided. It also enforces that isolated/current/custom-session jobs use `--message (agentTurn)` rather than any direct command payload.
  - file: [src/cli/cron-cli/register.cron-add.ts:90](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/src/cli/cron-cli/register.cron-add.ts#L90)

  - file: [src/cron/service/jobs.ts:156](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/src/cron/service/jobs.ts#L156)

## Risks / Open Questions

- This is a static code/docs review rather than a live runtime reproduction, but the feature request is about a missing product surface and the current type/CLI/execution paths leave no implemented direct-command mode.
- Current cron already offers mitigations like `--light-context` and `--tools exec`, so maintainers may want to frame the issue as an enhancement request rather than a bug.
