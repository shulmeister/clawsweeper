## Summary

Keep #14785 open. Current main has partial mitigations, including removal of duplicate in-band tool inventory and configurable tool policy workarounds, but the central request is still not implemented: OpenClaw still forwards full structured tool parameter schemas, the message tool still carries broad cross-channel fields, and the issue discussion explicitly records remaining P0/P1/P2 work after partial fixes.

## Best Possible Solution

Keep #14785 open as the parent performance tracker. The best path is incremental, measured PRs: finish removing remaining duplicated tool text where appropriate, skip schemas when the effective tool set is empty, prune message parameters by active channel/capability without breaking cross-channel agents, move long tutorial text out of tool descriptions, and evaluate a compatible lazy or slim schema mode for large plugin/declarative-tool setups.
