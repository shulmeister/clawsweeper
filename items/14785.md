## Summary

Keep #14785 open. Current main still sends full effective tool parameter schemas into embedded sessions, the message tool only scopes actions/capabilities while retaining broad cross-channel parameters, and the cron tool still carries tutorial-style guidance in its always-sent description. The issue discussion also has a fresh maintainer Codex review explicitly keeping this parent tracker open.

## Best Possible Solution

Keep #14785 open as the canonical parent performance tracker. The best path is incremental core work: avoid sending schema payloads when no effective tools remain, prune message parameters by active channel/capability while preserving cross-channel jobs, move cron tutorial text out of the always-sent tool description, and design a compatible lazy or slim schema mode for plugin and declarative tool growth. Narrower work such as #14619 can remain linked follow-up, but it does not supersede the broader issue.
