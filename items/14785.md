## Summary

Keep #14785 open. Current main has some related token-accounting and policy surfaces, but the central request is still unresolved: the effective tool set is still converted into full custom tool definitions, `message` still builds a broad cross-channel parameter schema, the system prompt still emits tool-name/summary lines, and the issue discussion explicitly says previous work was partial while measurable gaps remain.

## Best Possible Solution

Keep #14785 open as the canonical tool-schema overhead tracker. The best maintainer path is incremental core work: avoid duplicated tool-list text where safe, omit schema payloads when no effective tools remain, prune `message` parameters by active channel/capability without breaking cross-channel jobs, move long cron/tool guidance out of always-sent schemas, and design a compatible lazy or slim schema mode for plugin and declarative tools. Keep narrower items such as #14619 linked as follow-ups rather than superseding this broader issue.
