## Summary

Keep #14785 open. Current main still sends full structured tool parameter schemas for the effective tool set, and the message tool still exposes broad cross-channel parameters even when a current channel is known. The issue discussion also explicitly treats #30359 and tool policy workarounds as partial mitigations, not completion of the parent request.

## Best Possible Solution

Keep #14785 open as the parent performance tracker. The best path is incremental, measured PRs: skip tool definitions when the effective list is empty, prune message parameters by active channel/capability, move long cron tutorial text out of tool descriptions, and evaluate compatible lazy or slim schema modes for large plugin/declarative-tool setups.
