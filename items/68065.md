## Summary

Keep open. Current main still returns `sessions_send.status = "timeout"` immediately when the first `agent.wait` call times out, without post-timeout compensation to distinguish an accepted still-running target run from delivery failure. The linked PR #68196 tracks a plausible fix, but it is still open and not shipped.

## Best Possible Solution

Keep #68065 open until the result contract is fixed on main. The best path is to refine/land PR #68196 or an equivalent patch that adds post-timeout reconciliation without treating a mere readable history as proof of run acceptance, preserves compatibility for real hard timeouts, and adds focused tests in `src/agents/run-wait.test.ts` and `src/agents/tools/sessions.test.ts`.
