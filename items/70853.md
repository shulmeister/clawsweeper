## Summary

Keep #70853 open. Current main still rotates silent auto-reply sessions onto a new sessionId/transcript without replaying or injecting prior DM turns, and the related fix PR #70898 is still open/unmerged rather than shipped on main.

## Best Possible Solution

Keep the issue open and land PR #70898 or an equivalent core auto-reply fix: in resetReplyRunSession, best-effort seed a bounded tail of prior user/assistant transcript records or inject equivalent logical context for silent compaction/role-ordering rotations, with regression tests for both reset triggers. Explicit gateway /new and /reset should remain fresh-session paths.
