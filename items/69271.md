## Summary

Keep #69271 open. Current main still matches the reported UX: the Control UI Sessions table exposes checkbox selection plus a bulk delete bar, but no per-row delete/archive action. The backend supports `sessions.delete` and archives transcript files as part of deletion, but there is no separate archive action, archive filter/view, or protocol method for preserving a session row in cold storage. The provided related reports are title-search false positives around Telegram/Discord actions, canvas automation, and usage defaults rather than a canonical duplicate for this Control UI Sessions request.

## Best Possible Solution

Keep the issue open. The narrow path is to add an action cell or row menu in `ui/src/ui/views/sessions.ts` wired through `deleteSessionsAndRefresh` for single-session delete, with focused UI/controller tests. If maintainers want true archive rather than delete-with-transcript-archive, add an explicit protocol/storage contract first, including archived-session listing/filtering, restore semantics if supported, docs, and gateway tests.
