## Summary

Keep #9132 open. Current main exposes lifecycle status fields for sessions, but the Control UI Sessions table still does not render an active/completed/timed-out status indicator or status sorting/filtering. The related #46792 is closed, so it does not supersede this item.

## Best Possible Solution

Keep #9132 open and implement the visible lifecycle state in the Control UI Sessions view. A focused fix should add a compact Status column or badge in ui/src/ui/views/sessions.ts using GatewaySessionRow.status plus subagentRunState, hasActiveSubagentRun, startedAt, endedAt, and runtimeMs; consider status sorting/filtering if in scope; and add rendering coverage in ui/src/ui/views/sessions.test.ts for running, done, timeout, killed, and failed rows.
