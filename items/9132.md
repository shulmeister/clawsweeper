## Summary

Keep #9132 open. Current main already carries session lifecycle status and timing through the gateway/session data model, but the Control UI sessions table still renders only Key, Label, Kind, Updated, Tokens, Compaction, Thinking, Fast, Verbose, and Reasoning, and the chat session selector labels also omit lifecycle status. The prior issue comment correctly identified this same gap, and current main still has not added the requested active/completed/timed-out visual indicator.

## Best Possible Solution

Keep the issue open until the Control UI renders the existing lifecycle fields. The likely product/code path is a Status column or badge in ui/src/ui/views/sessions.ts that maps running/done/failed/killed/timeout to clear labels and uses updatedAt/startedAt/endedAt/runtimeMs for last activity or runtime detail, with focused tests in ui/src/ui/views/sessions.test.ts. If maintainers want status visible while switching chats, extend ui/src/ui/chat/session-controls.ts option labels or titles as a second scoped UI change.
