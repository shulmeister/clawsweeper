## Summary

Keep #9132 open. Current main carries session lifecycle status and timing through the gateway/session data model, subscriptions, and UI state updates, but the Control UI sessions table still renders Key, Label, Kind, Updated, Tokens, Compaction, Thinking, Fast, Verbose, and Reasoning without any active/completed/timed-out status indicator. The prior issue comment reached the same conclusion, and current main still matches that gap.

## Best Possible Solution

Keep the issue open until the Control UI renders the existing lifecycle fields. The likely implementation is a Status column or badge in ui/src/ui/views/sessions.ts mapping running/done/failed/killed/timeout to clear labels, with last activity or runtime detail from updatedAt/startedAt/endedAt/runtimeMs and focused coverage in ui/src/ui/views/sessions.test.ts. If maintainers want status visible during chat switching too, extend ui/src/ui/chat/session-controls.ts option labels or titles as a second scoped UI change.
