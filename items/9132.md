## Summary

Keep #9132 open. Current main still has session lifecycle data (`running`, `done`, `failed`, `killed`, `timeout`, timing, and subagent run state), but the Control UI Sessions table does not render a lifecycle status indicator, and its search/sort state is still limited to key/label/kind/display name, updated time, and tokens. The prior steipete/Codex issue comment also explicitly kept this item open, and related #46792 is closed rather than an active canonical replacement.

## Best Possible Solution

Implement #9132 in the Control UI Sessions view. Add a compact Status column or badge in ui/src/ui/views/sessions.ts that maps `running`, `done`, `failed`, `killed`, `timeout`, `subagentRunState`, and `hasActiveSubagentRun` to clear labels; surface last activity/runtime from `updatedAt`, `startedAt`, `endedAt`, and `runtimeMs`; and add focused rendering tests in ui/src/ui/views/sessions.test.ts. Lifecycle sorting/filtering can be included in the same patch or tracked as a narrower follow-up if maintainers want a smaller first change.
