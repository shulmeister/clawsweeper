## Summary

Current main already treats Feishu DM traffic as direct-message context before shared chat-type normalization, so this PR's core `p2p -> direct` change is redundant on current OpenClaw. The added payload regression also does not hit a live Feishu path on current main because `buildEmbeddedRunPayloads` does not accept `messageProvider` or `chatType` inputs.

## Evidence

## Close Comment

Closing this PR after Codex review because current `main` already implements the requested behavior through the Feishu channel itself. In `extensions/feishu/src/bot.ts:303` and `extensions/feishu/src/bot.ts:995`, Feishu only treats `group` as group and writes all other inbound Feishu chats as `ChatType: "direct"`, so `chat_type: "p2p"` is already handled as a DM before shared `normalizeChatType()` runs. That same behavior is present in the shipped `v2026.4.22` release (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). The proposed shared-normalizer change is therefore redundant on current `main`, and the added payload regression would not exercise a live Feishu path on current code because `buildEmbeddedRunPayloads` does not take `messageProvider`/`chatType` inputs (`src/agents/pi-embedded-runner/run/payloads.ts:121`).
