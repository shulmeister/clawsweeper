## Summary

Keep #69453 open. Current main contains partial hardening for post-run persistence, but it still does not implement the issue's requested consistent 1M-class GPT-5.4 context budget across session/status/catalog surfaces. The OpenAI Codex provider now explicitly distinguishes native `contextWindow` 1,050,000 from effective `contextTokens` 272,000, and core status/default read-only paths still use synchronous lookup plus 200,000 fallback behavior. The related merged PR #70743 also explicitly called #69453 an adjacent context-window/catalog mismatch that it did not fix.

## Best Possible Solution

Keep the issue open for a focused maintainer/product decision and targeted fix. If 1,050,000 is intended as the effective GPT-5.4 budget, remove or override the OpenAI Codex 272k `contextTokens` cap and add regression coverage for fresh session creation, status/Control UI display, model catalog, and follow-up/session persistence. If 272k is the intended effective cap, update the issue with that decision, document/display native `contextWindow` versus effective `contextTokens` clearly, and still harden cold read-only paths so fresh sessions do not fall back to 200k unexpectedly.
