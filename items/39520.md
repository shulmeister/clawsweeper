## Summary

Keep PR #39520 open. Current main still lacks the Feishu-owned `channels.feishu.localRoots` / account-scoped `localRoots` config requested by #39506, and Feishu media upload still only forwards core-provided `mediaLocalRoots`. The PR is not obsolete or superseded by the narrower shipped #27884 behavior.

## Best Possible Solution

Keep the PR open for maintainer review or implementation. The best path is to land a Feishu-owned `localRoots: "any" | string[]` config that resolves through the merged account config, honors empty arrays intentionally, injects raw `readFile` only for the explicit `"any"` bypass, and adds docs/tests. If maintainers decide Feishu should only use core `mediaLocalRoots`, close with that product/security rationale instead of treating this PR as obsolete.
