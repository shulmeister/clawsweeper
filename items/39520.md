## Summary

Keep PR #39520 open. Current main still does not implement the requested Feishu-owned `channels.feishu.localRoots` / account-scoped `localRoots` config, and the Feishu media send path still only forwards caller-provided `mediaLocalRoots` arrays to `loadWebMedia`. The linked #39506 behavior remains distinct from the older #27884 context-roots fix and from the separate Feishu reply-path media plumbing tracked in #51375.

## Best Possible Solution

Keep the PR open for maintainer review or land an equivalent Feishu plugin change. The best path is to add `localRoots: "any" | string[]` to the shared Feishu top-level/account config schema, resolve it from the merged runtime account so account overrides work, intentionally define empty-array behavior, inject raw `readFile` only for explicit `"any"`, and add Feishu docs plus tests for top-level, account override, `"any"`, empty array, and no `readFile` on array roots.
