## Summary

Current `main` already reconciles orphaned running task records during task-registry maintenance, wires that maintenance into gateway startup, and the same implementation is present in the shipped `v2026.4.23` release.

## Evidence

- **Runtime-aware orphan detection:** `hasBackingSession()` and `shouldMarkLost()` treat active tasks as lost after the 5-minute grace window when their backing ACP session, subagent session, cron job, or CLI run context no longer exists.
