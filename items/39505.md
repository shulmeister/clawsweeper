## Summary

Keep #39505 open. Current main still has the copy-sensitive token gap for backtick/punctuation-wrapped Home Assistant entity IDs: the raw token fails the file-like check, then the long-token sanitizer inserts literal spaces matching the reported examples. The related focused fix remains in open PR #39565; PR #39553 was closed unmerged.

## Best Possible Solution

Keep #39505 open until PR #39565 or an equivalent focused fix lands. The fix should classify the punctuation-stripped core token for file-like/dotted entity IDs, preserve backtick- and quote-wrapped Home Assistant IDs exactly, and add regression tests for bare and markdown-wrapped long dotted IDs without weakening the existing narrow-terminal long-token wrapping guard.
