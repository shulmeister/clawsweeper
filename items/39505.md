## Summary

Current main still has the reported copy-safety gap for punctuation-wrapped long entity IDs. The sanitizer can still insert spaces into backtick/quote-wrapped Home Assistant entity IDs, and the direct fix remains in open PR #39565 rather than on main.

## Best Possible Solution

Keep #39505 open until PR #39565 or an equivalent focused fix lands. The likely implementation should classify the stripped core token, preserve dotted/underscore Home Assistant entity IDs inside surrounding markdown punctuation, and add regression tests for backtick-wrapped and bare entity IDs without weakening the existing narrow-terminal wrapping guard.
