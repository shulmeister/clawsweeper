## Summary

Keep #39317 open. The reported false-success path still exists on current main: `codex-cli` runs use JSONL stdout, `executePreparedCliRun` only enters the failure branch for non-zero or abnormal process exits, and the exit-0 path parses stdout into a normal `CliOutput` that is later recorded as `result: "success"`. The useful prior fix proposal in #39460 matches the missing behavior, but that PR was closed unmerged; the related #66081 output-last-message PR is explicitly scoped to final reply selection and does not implement failure classification.

## Best Possible Solution

Keep the issue open and implement the missing Codex CLI failure classification in the CLI runner path: parse structured JSONL for sandbox-denied command/action/error signals, surface a non-success `FailoverError` before `buildCliRunResult`, add a typed failover reason/status policy if needed, exclude local sandbox-denial from auth-profile health tracking, and cover both detection and false-positive cases in `src/agents/cli-output.test.ts` plus a runner-level regression test.
