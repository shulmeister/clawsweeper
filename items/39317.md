## Summary

Keep #39317 open. Current main still has no exit-0 Codex JSONL sandbox-denial classification: the CLI runner treats exit-0 runs as normal output, the JSONL parser returns only text/session/usage, and the final CLI attempt is recorded as success. The directly relevant fix PR #39460 was closed unmerged, while related PR #66081 is scoped to final-message artifacts and explicitly does not change failure classification.

## Best Possible Solution

Keep the issue open. The best fix is to revive the substance of #39460: parse structured Codex JSONL records for sandbox-denied command/tool/action/error/status signals after exit 0, make `executePreparedCliRun` throw a non-success `FailoverError` before `buildCliRunResult`, add an explicit reason/status/auth-profile policy if maintainers want one, avoid broad assistant-message keyword matching, and add parser plus runner regression tests for true denial and false-positive discussion cases.
