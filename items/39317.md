## Summary

Keep #39317 open. Current main still treats exit-0 CLI runs as success before checking Codex JSONL for semantic sandbox/write-denied failures. The relevant parser only returns text/session/usage, the runner only throws on nonzero/non-exit subprocess status, and successful `executePreparedCliRun` output is recorded as `result: "success"`. The related fix PR #39460 described the right structured JSONL approach but is closed unmerged; #66081 is related final-message artifact work and does not address failure classification.

## Best Possible Solution

Keep the issue open. Revive the substance of #39460 on current main: parse structured Codex JSONL records for sandbox-denied command/tool/action/error/status signals after exit 0, make `executePreparedCliRun` throw a `FailoverError` before `buildCliRunResult`, add an explicit local sandbox-denial reason/status/auth-profile policy if maintainers want that contract, avoid broad assistant-message keyword matching, and add parser plus runner regression tests for true denials and false-positive discussion cases.
