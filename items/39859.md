## Summary

Keep open. Current main still does not implement PR #39859's central behavior: positive contextTokensOverride values are returned before the resolver looks up known model/config/discovery limits, so normal status/session/runtime paths can still over-report the effective model context window.

## Best Possible Solution

Keep #39859 and linked #39857 open. Land a narrow product fix on current main: resolve the known model context window before honoring contextTokensOverride, return Math.min(contextTokensOverride, modelTokens) when both are known, preserve override behavior when the model window is unknown, keep normalized provider lookup for configured windows, align status/session persistence helpers, and add focused resolver/status/session regression tests. Avoid carrying over the stale PR branch's unrelated CI and secrets-baseline churn.
