## Summary

Keep #29195 open. Current main has partial improvements around ACP error delivery, task tracking, runtime timeouts, and permission documentation, but the central ACPX/Codex non-interactive reliability issue is still present: ACPX still defaults to approve-reads/fail, the embedded runtime receives those values directly, parent agents still cannot inspect cross-agent ACP children under default session visibility, and the docs still describe stale ACP process cleanup as manual. Related #72064 also reports the same default-permission failure mode on v2026.4.24 with explicit ACPX permission config as the workaround.

## Best Possible Solution

Keep #29195 open as the canonical ACPX/Codex non-interactive reliability umbrella, or split and link narrower follow-ups before closing it. The concrete path is a safe permission preflight or explicit non-interactive ACPX profile/default decision for Codex sessions, requester-facing remediation for permission/config failures, parent-owned cross-agent ACP child status/history without requiring global tools.sessions.visibility=all, and automatic stale harness cleanup/reporting instead of manual ps/kill guidance.
