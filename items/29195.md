## Summary

Keep #29195 open. Current main has improved ACPX/Codex docs, task records, timeout plumbing, and terminal task notifications, but the central reliability request is still not fully implemented: ACPX still defaults to the non-interactive permission footgun, parent agents still cannot inspect cross-agent ACP children under default session visibility, and stale ACP process cleanup is still documented as manual.

## Best Possible Solution

Keep #29195 open as the canonical ACPX/Codex non-interactive reliability umbrella until narrower follow-ups exist. Best path: decide a safe default or preflight for ACPX permissions, surface permission/config failures to the requester with remediation, allow a spawning agent to inspect its own cross-agent ACP child without global visibility=all, and replace manual stale-process cleanup with automatic timeout/reporting semantics while preserving the shipped task/status and docs improvements.
