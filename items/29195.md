## Summary

Keep open. Current main has meaningful ACP cleanup, including background-task tracking, `/acp status`, run timeout plumbing, and merged permission docs, but #29195 is not fully resolved. The ACPX plugin still defaults to `permissionMode=approve-reads` with `nonInteractivePermissions=fail`, docs still warn that non-interactive writes/exec can fail with the reported `AcpRuntimeError`, and the troubleshooting docs still describe indefinitely stalled ACP sessions as requiring manual process inspection and killing.

## Best Possible Solution

Keep #29195 open or narrow it to the remaining core work: decide the ACPX non-interactive permission default/preflight policy, surface permission/config failures reliably to the spawning session, provide an agent-callable ACP status/progress path for owned children, and replace the documented manual stall cleanup path with automatic timeout/reporting semantics. The shipped documentation and background-task improvements should stay, but they do not fully satisfy the issue yet.
