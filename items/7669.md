## Summary

Keep #7669 open. Current main still does not implement an npm `dev` dist-tag publish path: the public OpenClaw NPM Release workflow only accepts `beta` and `latest`, the publish helper maps every non-`latest` request to `beta`, the release-plan types exclude `dev`, and the live npm registry still has no `openclaw@dev` dist-tag. The existing maintainer comment from steipete reached the same conclusion, and current main still matches that evidence.

## Best Possible Solution

Keep the issue open until release managers either implement a supported main-snapshot npm publish path for `openclaw@dev` with validation/postpublish checks, or intentionally retire the npm dev tag and update `https://docs.openclaw.ai/install/development-channels`, `https://docs.openclaw.ai/cli/update`, and `https://docs.openclaw.ai/reference/RELEASING` to say dev is git-only.
