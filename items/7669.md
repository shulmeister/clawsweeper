## Summary

#7669 should remain open. Current main still has no supported npm `dev` dist-tag publishing path: the public npm release workflow only accepts `beta` or `latest`, the publish helper coerces every non-`latest` request to `beta`, release planner types model only `stable`/`beta` and `latest`/`beta`, tests cover only beta/latest release plans, and the live npm registry currently has no `openclaw@dev` dist-tag. The April 26, 2026 steipete/Codex review comment in the issue reached the same conclusion, and current main still matches that evidence.

## Best Possible Solution

Keep #7669 open until release maintainers either implement a supported main-snapshot openclaw@dev publish path with workflow input/trigger support, release-plan validation, tests, and postpublish verification, or intentionally retire npm dev and update the public release-channel, update, and release docs to make dev git-only.
