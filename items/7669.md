## Summary

Keep #7669 open. Current main still does not implement a supported npm `dev` dist-tag publish path: the public release workflow only allows `beta`/`latest`, real publish requires release tags, the publish helper maps non-`latest` requests to `beta`, release-plan contracts only model stable/beta, docs still describe dev as git-first with npm `dev` optional, and the live npm registry currently has no `dev` dist-tag.

## Best Possible Solution

Keep #7669 open until maintainers either implement a first-class main-snapshot npm `dev` publish path, including workflow input/trigger support, release-plan validation, publish-helper handling, and post-publish verification; or explicitly retire npm dev snapshots and update public release-channel/update/release docs so dev is documented as git-only.
