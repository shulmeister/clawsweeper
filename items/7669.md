## Summary

Keep #7669 open. Current main still does not implement a dev npm dist-tag publish path: the public release workflow only permits beta/latest, the publish helper normalizes any non-latest request to beta, and the live npm registry has no openclaw@dev tag. The docs still describe dev as a git-main channel with an optional dev npm dist-tag when published, so the requested CI/CD and registry acceptance criteria remain unmet.

## Evidence
