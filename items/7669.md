## Summary

Keep #7669 open. Current main still does not provide a first-class npm `dev` dist-tag publish path: the public npm release workflow is manual/tag-oriented, exposes only `beta` and `latest`, the publish helper and release-plan contract cannot emit `dev`, tests cover only stable/beta behavior, live npm dist-tags still lack `dev`, and the implemented `dev` update path switches package installs to a git checkout rather than installing `openclaw@dev`.

## Best Possible Solution

Keep #7669 open until maintainers either implement a first-class npm `dev` snapshot channel for main builds, including workflow trigger/input support, release-plan and publish-helper support, snapshot/version semantics, tests, post-publish verification, and docs; or explicitly retire npm dev snapshots and update the release-channel, updater, installer, and release docs so `dev` is documented as git-only for npm users.
