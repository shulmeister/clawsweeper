## Summary

Keep open. Current main still allows QMD memory in direct and channel sessions when `memory.backend` is `qmd` and `memory.qmd.scope` is unset, so PR #70297 is not obsolete or implemented on main. The PR is a focused two-file behavior/test change, but it likely needs maintainer review plus docs/changelog alignment because the behavior is user-visible and current docs are internally inconsistent.

## Best Possible Solution

Keep PR #70297 open for maintainer review. If maintainers agree QMD should be direct-only by default, merge or reimplement the same source/test change, then align `docs/concepts/memory-qmd.md`, `docs/reference/memory-config.md`, generated config help if needed, and the changelog/migration note for operators who need explicit channel opt-in.
