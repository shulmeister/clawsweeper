## Summary

Current main does not make this PR obsolete. The active `src/memory-host-sdk` implementation still defaults QMD scope to direct plus channel sessions, and its regression test still expects channel access. A direct-only copy exists under `packages/memory-host-sdk`, but the exported runtime path re-exports from `src`, so the product behavior still matches the bug described by the PR.

## Best Possible Solution

Keep the PR open. The maintainer path should update the active `src/memory-host-sdk/host/backend-config.ts` default to direct-only, keep or adjust the `src` regression test to deny channel sessions by default, and reconcile the package mirror and public docs so QMD scope behavior is consistent across code and documentation.
