## Summary

Keep #35735 open. Current main still does not provide `openclaw sessions rm <key>` or `openclaw sessions clear`; it only lists sessions and exposes policy-based `sessions cleanup`. The existing gateway/UI `sessions.delete` path is useful but does not cover the PR's offline CLI workflow with store/agent targeting, bulk filtering, dry-run, and JSON output.

## Best Possible Solution

Keep #35735 open for maintainer judgment. The best path is a rebased, focused implementation of `openclaw sessions rm <key>` and `openclaw sessions clear` that uses explicit safe store mutation semantics, addresses the existing transcript-reporting/JSON-schema/maintenance-pruning review comments, documents the public CLI at `https://docs.openclaw.ai/cli/sessions`, and adds targeted registration plus command tests. If maintainers do not want this command shape, replace it with a narrower canonical design issue rather than auto-closing the PR as stale.
