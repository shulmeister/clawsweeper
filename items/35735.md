## Summary

Keep open. Current main still does not ship `openclaw sessions rm` or `openclaw sessions clear`; only listing and `sessions cleanup` are registered and documented. The PR covers a real CLI gap, but the proposed mutation path still needs maintainer review and updates against current session-store maintenance semantics before it can safely merge or be superseded.

## Best Possible Solution

Keep the item open. The best path is a rebased implementation of `openclaw sessions rm <key>` and `openclaw sessions clear` that explicitly defines safe deletion semantics, dry-run parity, transcript archival/removal behavior, stable JSON output, and interaction with `session.maintenance`; alternatively, a maintainer can manually close this PR only after deciding to track all remaining work solely in #35682.
