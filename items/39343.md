## Summary

Keep #39343 open. Current main partially solves the Telegram album case via `media_group_id` buffering, but the requested LINE same-sender/time-proximity media batching and a documented/configurable media batch delay are still missing. The issue is concrete, author association is `NONE`, and there are no protected labels in the provided GitHub context.

## Best Possible Solution

Keep this issue open and implement a focused channel-owned media batching feature: preserve Telegram `media_group_id` buffering, add LINE sender/chat/time-proximity media buffering, expose a documented config knob or media-specific inbound debounce setting, flush on timer or text arrival, and add LINE plus Telegram regression coverage and docs.
