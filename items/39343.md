## Summary

Keep #39343 open. Current main already handles Telegram albums by `media_group_id`, but the requested LINE same-sender/time-window media batching and a media-specific/documented delay remain missing. The prior ClawSweeper comment reached the same result, and the current checkout at `631552c554095b0c520376db9131dac0f5427f94` still supports that assessment.

## Best Possible Solution

Keep the issue open and implement the remaining bundled-channel behavior: preserve the existing Telegram `media_group_id` album buffer, add LINE same-sender/chat/time-proximity media batching, flush on timer or text arrival, expose and document a media-specific delay or equivalent inbound debounce setting, and add LINE plus Telegram regression coverage.
