## Summary

Keep #39343 open. Current main already buffers Telegram albums by `media_group_id`, but the requested LINE same-sender/time-window image batching and documented/configurable media batch delay remain missing. The prior ClawSweeper comment reached the same conclusion, and current main at 7e13f3f5140168e31027b39a9855ade3fadf24d5 still matches that assessment.

## Best Possible Solution

Keep the issue open and implement a focused bundled-channel media batching feature: preserve the existing Telegram `media_group_id` buffer, add LINE same-sender/chat/time-proximity media batching, flush on timer or text arrival, expose and document a media-specific delay or equivalent inbound debounce setting, and add LINE plus Telegram regression coverage.

## Risks / Open Questions

- Maintainers still need to choose the public config shape: a LINE/Telegram `mediaBatchDelayMs`, an extension of `messages.inbound.byChannel`, or a more general media-specific inbound debounce contract.
- LINE batching needs careful handling for group mention policy, sender/chat keys, replay cache semantics, reply token timing, and flushing when text arrives.
