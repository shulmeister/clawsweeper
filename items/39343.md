## Summary

Keep #39343 open. Current main still has only the Telegram album/media_group_id part implemented; LINE inbound media events are still handled one event at a time, and the shared inbound debounce contract remains text-only for normal media-bearing messages.

## Best Possible Solution

Keep #39343 open. Preserve the existing Telegram media_group_id album buffering, then implement the missing bundled LINE behavior: same-sender/chat/time-window media batching with clear timer and text-flush semantics, a deliberate config surface or documented use of messages.inbound, docs updates, and LINE plus Telegram regression coverage for batching keys, timer flush, text flush, replay-cache timing, and reply-token behavior.
