## Summary

Keep open. Current main has exact `NO_REPLY` suppression and several assistant-visible sanitizers for XML/model-control scaffolding, but it does not implement this PR's channel-agnostic guard for plain route-marker plus tool-JSON orchestration leakage. The related #45005 was a narrower exact-token channel fix and does not supersede this PR.

## Best Possible Solution

Keep this PR open or replace it with an equivalent narrow shared guard in `src/auto-reply/reply/normalize-reply.ts`: detect structured route-marker/tool-JSON leakage, strip suspicious text while preserving media/channelData, honor custom silent tokens, and keep the regression tests that prove explanatory prose is not dropped.
