## Summary

Keep #37878 open. Current main still handles model-usage costs only when they are native Python int/float values, so numeric-string costs are dropped, booleans are accepted as numbers, and NaN/Infinity can flow through aggregation/current/latest-day paths. The linked PR #37877 is closed and unmerged in the provided context, and its parse_cost helper/tests are not present on main.

## Best Possible Solution

Keep the issue open and implement the narrow fix from PR #37877 on current main: add a shared finite cost parser in skills/model-usage/scripts/model_usage.py that accepts finite numeric strings, rejects booleans, rejects NaN/Infinity, and is used by aggregate_costs, pick_current_model, and latest_day_cost. Add focused tests in skills/model-usage/scripts/test_model_usage.py covering numeric-string aggregation/current/latest-day behavior and ignored bool/non-finite costs.
