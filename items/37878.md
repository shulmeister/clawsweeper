## Summary

Keep #37878 open. Current main still drops numeric-string model costs, still allows bool/NaN/Infinity through the numeric paths, and the linked fix PR #37877 is closed unmerged rather than shipped.

## Best Possible Solution

Keep #37878 open and land the narrow parser fix: add a shared finite-cost parser in skills/model-usage/scripts/model_usage.py that accepts finite numeric strings and finite numeric values, rejects bool and non-finite values, and use it from aggregate_costs, pick_current_model, and latest_day_cost. Add focused tests in skills/model-usage/scripts/test_model_usage.py for aggregation, current-model selection, latest-day cost, bool rejection, and NaN/Infinity rejection.
