## Summary

Current main still has the reported model-usage cost parsing bug: numeric-string costs are ignored, booleans are accepted as numbers, and NaN/Infinity are not filtered. The linked PR #37877 is closed unmerged, so there is no shipped fix to close against.

## Best Possible Solution

Keep #37878 open and land a narrow fix in skills/model-usage/scripts/model_usage.py: add a shared parse_cost helper that accepts finite numeric strings and finite numeric values, rejects bool and non-finite values, and use it from aggregate_costs, pick_current_model, and latest_day_cost. Add focused tests in skills/model-usage/scripts/test_model_usage.py for numeric-string aggregation, bool/non-finite rejection, current-model selection, and latest-day cost.
