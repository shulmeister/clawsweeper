## Summary

Current main does not implement the requested behavior. The model-usage helper still only accepts native int/float costs, so numeric strings are ignored; because Python bool is an int subclass, booleans are still accepted; and there is no finite-number check for NaN/Infinity. The linked PR #37877 was closed unmerged, and the current tests do not cover the requested edge cases.

## Best Possible Solution

Keep the issue open and land a narrow fix in skills/model-usage/scripts/model_usage.py: add a shared parse_cost helper that accepts finite numeric strings and finite numeric values, rejects bool and non-finite values, and route aggregate_costs, pick_current_model, and latest_day_cost through it. Add focused tests in skills/model-usage/scripts/test_model_usage.py for numeric-string aggregation, bool/non-finite rejection, current-model selection, and latest-day cost.
