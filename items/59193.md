## Summary

Current main already gives Telegram users a browseable/tappable skill list through native Telegram skill commands plus a paginated `/commands` UI, which covers the issue's stated minimum ask.

## Evidence

- **Telegram plugin auto-enables native skill command UI:** The Telegram channel plugin advertises native commands and sets `nativeSkillsAutoEnabled: true`, so Telegram skill commands are part of the supported built-in command/menu surface by default.
  - file: [extensions/telegram/src/shared.ts:149](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/telegram/src/shared.ts#L149)

  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Installed skills are registered into Telegram's native menu:** Native command registration resolves the bound Telegram agent and feeds its skill catalog into the Telegram command list, so users can select skill commands from Telegram instead of memorizing names.
  - file: [extensions/telegram/src/bot-native-commands.ts:489](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/telegram/src/bot-native-commands.ts#L489)

- **Telegram `/commands` already lists commands with inline-button pagination:** `/commands` builds a paginated command list for channel surfaces, and Telegram callback handling paginates that list with inline buttons while sourcing the same agent-scoped skill command catalog.
  - file: [src/auto-reply/reply/commands-info.ts:43](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/auto-reply/reply/commands-info.ts#L43)

## Risks / Open Questions

- This is not a dedicated model-picker-style skill chooser; it is implemented through Telegram native command menus plus `/commands` pagination. If product wants stronger parity with `/models`, that would be a separate enhancement rather than an unimplemented gap.
