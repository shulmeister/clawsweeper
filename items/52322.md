## Summary

Keep open. Current `main` does not show a shipped fix for GitHub Copilot PAT auth: the Copilot token exchange still hard-codes `Authorization: Bearer <githubToken>`, the test suite locks that behavior in place, and the provider/docs still center only the interactive device-login path rather than a first-class PAT flow.

## Evidence

- **Copilot token exchange still uses a fixed Bearer header:** `resolveCopilotApiToken()` sends `Authorization: Bearer ${params.githubToken}` to `https://api.github.com/copilot_internal/v2/token`, with no PAT-specific branch or alternate auth scheme.
