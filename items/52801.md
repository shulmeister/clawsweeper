## Summary

Keep open. Current `main` does not have the PR's outbound `viewOnce` feature: the agent/tool and gateway schemas do not accept it, the outbound action path does not thread it, and the WhatsApp and Signal senders on `main` still lack the platform-specific view-once handling described in the PR.

## Evidence

- **Tool schema still lacks `viewOnce`:** `buildSendSchema()` exposes `gifPlayback` and `forceDocument`, but no `viewOnce` field on current `main`. That means agent/tool callers cannot request the feature through the supported schema.
  - file: [src/agents/tools/message-tool.ts:124](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/tools/message-tool.ts#L124)

- **Gateway send schema still rejects `viewOnce`:** `SendParamsSchema` includes `mediaUrl`, `mediaUrls`, and `gifPlayback`, but no `viewOnce`, with `additionalProperties: false`. The gateway contract on `main` therefore does not implement the PR's new flag.
  - file: [src/gateway/protocol/schema/agent.ts:87](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/gateway/protocol/schema/agent.ts#L87)
