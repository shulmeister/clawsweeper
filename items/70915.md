## Summary

Keep PR #70915 open. Current main still has the reported ZAI image-routing gap: the bundled ZAI media-understanding provider delegates directly to the generic image helpers, while ZAI onboarding can persist Coding Plan base URLs and the GLM vision models are advertised as image-capable. The PR is a focused fix for open issue #70886, though its exact implementation still needs maintainer review because a review comment flags a possible request-scoped config/models.json race.

## Best Possible Solution

Keep the PR open or supersede it with a maintainer-owned fix. The best implementation is a ZAI-owned media-understanding route that sends known ZAI vision models from Coding Plan configs to the matching standard ZAI endpoint, with regression tests for global/CN coding URLs and version-agnostic `/api/coding/paas/vN` rewriting. Once that lands, close linked issue #70886 from the merged fix.
