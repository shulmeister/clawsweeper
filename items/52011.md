## Summary

Keep open. Current `main` at `60f7a59f5ed581839cfd1d52583449c5aa1f03a4` still does not implement compaction-specific model fallbacks: `AgentCompactionConfig` and its zod schema expose only a single `compaction.model` string, the compaction target resolver reads only that single override, and the embedded compaction path fails immediately when `resolveModelAsync(...)` cannot resolve the chosen model. I found no use of the shared `runWithModelFallback` logic anywhere in the compaction paths.
