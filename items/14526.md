## Summary

Keep #14526 open. Current main still has only partial operator-safety pieces: manual backup create/verify, update doctor checks, restart health diagnostics, and background auto-update timing. It does not implement the requested opt-in safe-update workflow with automatic pre-update backup, retention policy, channel/agent smoke verification, or automatic rollback/restore. The linked `openclaw-auto-update-kit` is useful operator evidence/workaround, not a core implementation or superseding item.

## Best Possible Solution

Keep #14526 open until maintainers either implement an opt-in safe-update mode in the core update/auto-update flow or explicitly decide to leave it external. A complete core solution would likely add update CLI/config flags for pre-update backup, retention limits, post-update gateway/status/channel/agent smoke checks, and controlled rollback/restore behavior built on the existing backup primitives.
