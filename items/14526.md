## Summary

Keep open. Current main has partial operator-safety pieces, including manual `openclaw backup create/verify`, update doctor checks, gateway restart health reporting, and background auto-update timing. It still does not implement the requested opt-in safe-update workflow with automatic pre-update state backup, retention policy, channel/agent smoke verification, or automatic rollback/restore. The issue discussion and linked `https://github.com/Bortlesboat/openclaw-auto-update-kit` are useful operator evidence/workaround, not a core implementation or superseding item.

## Best Possible Solution

Keep #14526 open until maintainers either implement an opt-in safe-update mode in the core update/auto-update flow or make an explicit product decision to leave it external. A complete core solution would likely add update configuration/CLI flags for pre-update backup, retention limits, post-update gateway/status/channel/agent smoke checks, and controlled rollback/restore behavior using the existing backup primitives.
