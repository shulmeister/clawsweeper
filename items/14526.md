## Summary

Keep open. Current `main` has some adjacent pieces already shipped: a standalone `openclaw backup` command, `openclaw update`, doctor/health checks, and manual rollback guidance. But it does not implement the requested opt-in safe self-update workflow that takes a pre-update snapshot, performs richer post-update validation, and automatically restores or rolls back on failure.

## Evidence

- **Backup surface is standalone and has no restore action:** The CLI registers `openclaw backup create` and `openclaw backup verify` only. There is no `backup restore` subcommand or integrated safe-update workflow entrypoint.
  - file: [src/cli/program/register.backup.ts:20](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/cli/program/register.backup.ts#L20)
  - command: `nl -ba src/cli/program/register.backup.ts | sed -n '1,180p'`
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **Updater's final check is doctor, not snapshot+rollback:** The git update flow ends by running `openclaw doctor --non-interactive --fix`. This is a post-update check/repair step, but there is no pre-update state backup or rollback branch here.
  - file: [src/infra/update-runner.ts:1187](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/infra/update-runner.ts#L1187)
  - command: `nl -ba src/infra/update-runner.ts | sed -n '1120,1225p'`

- **Post-restart verification stops at health, without revert:** After restart, the CLI waits for the gateway to become healthy and prints diagnostics if it does not. It does not restore a previous snapshot or downgrade automatically.
  - file: [src/cli/update-cli/update-command.ts:783](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/cli/update-cli/update-command.ts#L783)

- **Rollback remains manual in user docs:** The update docs describe rollback as manually pinning an npm version or checking out an older git commit. That is materially different from the issue's proposed automatic rollback/restore workflow.
  - file: [docs/install/updating.md:126](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/docs/install/updating.md#L126)
