## Summary

Keep #14526 open. Current main has adjacent pieces: manual backup create/verify, update doctor checks, plugin post-update sync, restart health/version/plugin diagnostics, timing-only auto-update policy, and manual rollback docs. It still does not implement the proposed opt-in safe-update workflow with automatic pre-update state backup, retention, channel or real-agent smoke verification, or automatic rollback/restore. The linked openclaw-auto-update-kit is useful operator prior art/workaround, not a merged core implementation or superseding tracker.

## Best Possible Solution

Keep #14526 open until maintainers either implement an opt-in safe-update mode in the core `openclaw update` and `update.auto` paths or explicitly decide this operator workflow should remain external. A complete core path would likely build on existing backup and health primitives, adding pre-update backup creation, retention controls, post-update gateway/status/channel/real-agent smoke checks, and controlled rollback/version pin plus state restore, while treating openclaw-auto-update-kit as useful prior art.
