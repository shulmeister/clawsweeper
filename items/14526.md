## Summary

Keep open. Current main has useful pieces, including `openclaw backup create/verify`, `openclaw update` with doctor checks, gateway restart health reporting, auto-update configuration, and manual rollback docs. It does not implement the requested opt-in safe-update workflow with automatic pre-update state backup, backup retention policy, channel/agent smoke verification, or automatic rollback/restore. The linked `openclaw-auto-update-kit` is useful operator evidence/workaround, but it does not make the OpenClaw core request implemented or superseded.
