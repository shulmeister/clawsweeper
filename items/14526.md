## Summary

Keep #14526 open. Current main has useful primitives: manual `openclaw backup create/verify`, package update doctor checks, post-update plugin sync, restart/version/plugin diagnostics, auto-update timing policy, and manual rollback docs. It still does not implement the proposed opt-in safe-update workflow: automatic pre-update backup, backup retention controls, channel or real-agent smoke verification, and automatic rollback/restore. The Bortlesboat `openclaw-auto-update-kit` link is useful operator prior art/workaround, and the latest maintainer-side Codex comment explicitly kept this item open.

## Best Possible Solution

Keep #14526 open until maintainers either implement an opt-in safe-update mode in the core `openclaw update` and `update.auto` paths or explicitly decide the workflow belongs outside core. A complete core path should build on existing backup and health primitives, add pre-update backup creation, retention controls, gateway/status/channel/real-agent smoke checks, and controlled rollback/version pin plus state restore, using openclaw-auto-update-kit as prior art.
