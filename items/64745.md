## Summary

Current main and the shipped v2026.4.24 macOS app already include the safeguards that address the reported observable failure: second OpenClaw app instances terminate immediately, the Launch at Login plist does not keep the app alive or respawn it after quit/crash, and the release appcast ships those macOS bits. The report has no logs or crash artifacts to reconstruct the exact 2026.4.8 trigger, but the current product behavior no longer matches an unbounded app self-launch/self-replication loop.

## Best Possible Solution

Close as already addressed by the current macOS app behavior. Keep the duplicate-instance guard and non-KeepAlive LaunchAgent contract, and ask for a new issue with macOS logs/crash reports if a current release can still reproduce repeated OpenClaw launches.

## Evidence

- **Issue discussion reviewed:** The report describes the 2026.4.8 macOS app repeatedly creating OpenClaw icons until the machine froze; the useful follow-up comment identified it as a likely macOS launch loop rather than malware. No crash log, screenshot, or install-method details were provided.
