## Summary

Current main still does not honor `openclaw node run --node-id` as the node identity. The CLI/node host path forwards the value as `GatewayClient.instanceId`, but gateway pairing reconciliation and `NodeRegistry.register()` still resolve the node id from `connect.device?.id ?? connect.client.id`. The related PRs #61587 and #61719 were closed unmerged, so there is no canonical merged fix to close this issue against.
