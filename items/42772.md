## Summary

Current main already canonicalizes Control UI main-session aliases to the gateway-provided canonical session key before chat events are filtered, so the alias-vs-canonical mismatch described by PR #42772 does not reproduce on current main. The proposed `matchesSessionKey()` change is therefore obsolete for the current code path.
