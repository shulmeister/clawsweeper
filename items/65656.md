## Summary

Keep this issue open. Current main still matches the reported failure path: Markdown tables are extracted into LINE Flex messages, the text chunk consumes the reply token, and the rich/table message is then sent with `allowReplyToken=false`, which forces the Push API. The related fix PR was closed unmerged, and current tests still assert the push-after-reply behavior rather than the requested bundled reply behavior.

## Best Possible Solution

Keep this issue open and fix it in the LINE plugin delivery path. The likely maintainer path is to extend `sendLineReplyChunks` or `deliverLineAutoReply` so text plus extracted table Flex messages can be sent in the same Reply API call when the reply token is still usable and the total message count fits LINE's five-message limit, with overflow falling back to push. Add a regression test for a text response containing a Markdown table that verifies the text and generated Flex message are bundled through `replyMessageLine` instead of `pushMessagesLine`, while preserving the existing quick-reply ordering behavior.
