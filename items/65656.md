## Summary

Current `main` still matches the reported bug: in the LINE text+table/rich-message path, text is sent through `replyMessageLine`, then rich/flex/media messages are still sent through push-only code, so free-plan push-quota exhaustion can still drop the table with 429. The linked PR #65673 was closed unmerged after review identified it as a no-op.

## Evidence

- **Current mixed text+rich flow still pushes rich messages after text:** In `deliverLineAutoReply`, when text chunks exist, OpenClaw first calls `sendLineReplyChunks(...)`, then in the same branch still sends `richMessages` and `mediaMessages` with `allowReplyToken = false`. That preserves the reported push-only behavior for text+table replies.
  - file: [extensions/line/src/auto-reply-delivery.ts:146](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/extensions/line/src/auto-reply-delivery.ts#L146)
  - command: `nl -ba extensions/line/src/auto-reply-delivery.ts | sed -n '137,165p'`
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **Reply helper consumes the reply token before rich messages are considered:** `sendLineReplyChunks` uses `replyMessageLine` for up to five text chunks and sets `replyTokenUsed = true`; its error path also marks the token used before falling back. By the time post-text rich messages run, the reply token is no longer available.
  - file: [extensions/line/src/reply-chunks.ts:44](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/extensions/line/src/reply-chunks.ts#L44)

- **Current test suite encodes the buggy behavior:** `auto-reply-delivery.test.ts` has a test named `uses reply token for text before sending rich messages` that explicitly expects the text to go via `replyMessageLine` and the flex message to go via `pushMessagesLine`, which is the same split delivery the issue reports.
  - file: [extensions/line/src/auto-reply-delivery.test.ts:78](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/extensions/line/src/auto-reply-delivery.test.ts#L78)
