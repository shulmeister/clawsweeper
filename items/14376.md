## Summary

Keep open. Current main has partial supporting work: cron records `lastErrorReason`, applies generic consecutive-error backoff, retries transient one-shot jobs, and can send repeated-failure alerts. It does not implement the requested reason-aware guard policy for recurring cron jobs: billing/auth/format do not circuit-break by default, rate limits do not use provider `Retry-After`, and there is no provider/global circuit breaker for the retry-storm scenarios described in the issue comments.
