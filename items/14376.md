## Summary

Keep #14376 open. Current main has useful partial work, including `lastErrorReason`, one-shot transient retry, failure alerts, and auth-profile cooldowns, but recurring cron jobs still do not apply the requested reason-aware guard policy or circuit breaker.

## Best Possible Solution

Keep #14376 open until recurring cron scheduling has an explicit reason-aware guard policy. The likely product/code path is in `src/cron/service/timer.ts`, cron state/config types, protocol/schema exposure, and docs/tests: classify the terminal run error once, persist the reason, respect provider Retry-After where available for rate limits, auto-pause or circuit-break recurring jobs for billing/auth/format with safe configurable defaults and one alert, keep timeout/network/rate-limit jobs enabled with bounded backoff, and coordinate with existing auth-profile cooldowns plus any provider/global rate-limit breaker.
