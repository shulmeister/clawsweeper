## Summary

Keep #14376 open. Current main records `lastErrorReason`, has one-shot transient retry, generic recurring backoff, failure alerts, and auth-profile cooldowns, but recurring cron still does not apply the requested reason-aware guard policy: no billing/auth/format circuit-breaker, no job-level cooldown/disabled state, and no Retry-After-aware recurring scheduling.

## Best Possible Solution

Keep #14376 open until recurring cron scheduling has an explicit reason-aware guard layer. The likely fix belongs in `src/cron/service/timer.ts`, cron state/config types, gateway protocol/schema, UI/docs, and regression tests: classify terminal run errors once, persist/display reason and guard state, respect Retry-After where available, auto-pause or circuit-break recurring jobs for billing/auth/format with safe configurable defaults and one alert, and keep timeout/network/rate-limit jobs enabled with bounded backoff coordinated with existing auth-profile cooldowns.
