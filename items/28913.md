## Summary

Keep open. Current main still does not implement source-based filtering for memory-lancedb autoCapture: the runtime config and manifest reject any captureFilter/excludeWrapped option, shouldCapture remains trigger-based and source-unaware, and the agent_end capture path stores matching user-message text without checking EXTERNAL_UNTRUSTED_CONTENT boundaries first. The prior ClawSweeper comment reached the same conclusion; current main still matches that behavior.

## Best Possible Solution

Keep the issue open and implement a narrow, backwards-compatible memory-lancedb capture filter: add typed config and manifest support for captureFilter.excludeWrapped, validate it in runtime parsing and UI hints, check configured external-content boundary markers before shouldCapture in the agent_end path, and add regression tests for wrapped EXTERNAL_UNTRUSTED_CONTENT containing normal trigger words such as 'always' while preserving direct user preference capture.
