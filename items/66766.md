## Summary

Current main does not implement the requested behavior. `resolveContextTokensForModel()` still returns the 1M Anthropic window for Opus/Sonnet 4 only when `params.context1m === true`, the regression test suite explicitly expects `anthropic/claude-opus-4-6` without that flag to resolve to 200k, and the Anthropic stream wrapper only injects the `context-1m` beta header when the flag is enabled. The linked #66862 PR appears to track the requested fix, while #66790 was closed unmerged, so the issue should stay open until that or an equivalent maintainer-approved change lands.

## Best Possible Solution

Keep #66766 open and resolve it through #66862 or an equivalent maintainer-reviewed change. If OpenClaw should treat eligible Anthropic Opus/Sonnet 4 models as 1M by default, update `src/agents/context.ts`, the request-header/beta behavior, tests, and docs together. If explicit `context1m` remains intended, align `models list`, `/status`, and Control UI semantics so users do not see conflicting effective context limits.
