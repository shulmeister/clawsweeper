## Summary

Current main no longer reaches the synthetic compatibility-config path reported in #68088 from memory-search doctor config resolution. The memory-search resolver now uses the local registered-provider helper, doctor reports the intended inactive-memory note when no memory plugin is active, and v2026.4.24 contains the same implementation.

## Best Possible Solution

Close #68088 as implemented on main. Keep the current config-only memory-search behavior; the still-open #68110 patch can be treated by maintainers as obsolete/superseded because the original doctor false positives are avoided by no longer cold-loading memory capability providers from this path.

## Evidence

- **Memory-search uses local provider registry, not runtime capability loading:** `src/agents/memory-search.ts` imports `getMemoryEmbeddingProvider` from `../plugins/memory-embedding-providers.js`; its primary, fallback, and multimodal lookups use that local helper. The file also documents that config resolution only validates adapters already registered by the active runtime instead of cold-loading plugins.
  - file: [src/agents/memory-search.ts:11](https://github.com/openclaw/openclaw/blob/d251932fcfd60e205e80bf7e9abdb4af63582e6f/src/agents/memory-search.ts#L11)
