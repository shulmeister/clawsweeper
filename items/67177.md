## Summary

Keep open. Current main still has the exact missing rewrite reported in #67177: `file.download.info` attachments return the raw `content.downloadUrl`, while the Graph shares rewrite is only applied to the separate `contentUrl` branch. The added logging work reduces the silent-failure aspect, but the central DM file download bug is still present and untested.

## Best Possible Solution

Keep #67177 open. The narrow fix belongs in `extensions/msteams/src/attachments/download.ts`: apply `tryBuildGraphSharesUrlForSharedLink(downloadUrl)` in the `file.download.info` branch, keep the direct URL fallback for non-shared URLs, and add a regression test proving SharePoint/OneDrive `content.downloadUrl` routes through the Graph shares endpoint with the Graph-scoped auth retry.
