## Summary

Keep #28025 open. Current main still does not implement Slack tool-use-gated reply threading or the proposed `channels.slack.thread.replyOnToolUse` option; Slack routing is still selected from configured reply mode plus existing thread context before tool callbacks run.

## Best Possible Solution

Keep #28025 open. If accepted, add a default-false typed Slack plugin config option, carry a reliable per-turn tool-use signal into Slack reply planning before streaming/final delivery commits to a target, preserve explicit thread replies and existing reply-mode/thread inheritance behavior, and add Slack tests for DMs, channels, existing threads, streaming, and tool/progress callbacks.
