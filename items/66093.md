## Summary

Keep #66093 open. Current main still has the core behavior described in the report: fallback attempts deliberately drop a session auth profile when the selected fallback provider does not auth-alias to the primary provider, while CLI session reuse still treats any stored/current auth-profile difference as an auth-profile invalidation. The issue is recent, has concrete logs and a follow-up stale-binding finding, and I did not find a current regression test or implementation that resolves the fallback/stale-binding mismatch for google-gemini-cli.

## Best Possible Solution

Keep the issue open and fix the CLI auth/session contract. The likely maintainer path is to make fallback CLI providers resolve a provider-appropriate auth profile without reintroducing #1156 cross-provider leakage, and add regression coverage for google-gemini-cli fallback reuse plus stale `cliSessionBindings` where stored/current authProfileId is missing on one side. The fix should live around `resolveProviderScopedAuthProfile`/CLI run auth resolution or `resolveCliSessionReuse`, with tests proving both primary-to-fallback and fallback-to-primary binding upgrade behavior.

## Evidence

- **Issue report has concrete reproduction and logs:** The provided issue body describes google-gemini-cli as a fallback provider, OAuth auth profile setup, repeated `cli session reset: provider=google-gemini-cli reason=auth-profile` logs, and tiny promptChars across 21 resets. The sole follow-up comment adds a stale `cliSessionBindings` path where a binding without authProfileId continues to invalidate after the provider becomes primary.
- **Provider-scoped auth still drops cross-provider fallback authProfileId:** `resolveProviderScopedAuthProfile` only forwards `params.authProfileId` when `resolveProviderIdForAuth(params.provider)` equals `resolveProviderIdForAuth(params.primaryProvider)`; otherwise it returns undefined authProfileId/authProfileIdSource. For a zai primary falling back to google-gemini-cli, this still drops the CLI provider auth profile.
