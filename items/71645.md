## Summary

PR #71645 should stay open. Current main still has the Mattermost tool-start handler update the active draft preview with `buildMattermostToolStatusText(payload)` without first flushing streamed assistant text or rotating to a new post, and the draft stream loop keeps only one pending text slot. The PR is a focused Mattermost plugin bug fix, not implemented on main or superseded by the related general streaming fixes.

## Best Possible Solution

Keep PR #71645 open for maintainer review. The best path is to preserve Mattermost streamed assistant text across tool-start events using the existing draft stream lifecycle, add a focused regression test for partial text followed by a tool-start status, and update the Mattermost preview-streaming docs if the accepted behavior intentionally creates a separate status post at tool boundaries.
