## Summary

Current main still does not implement the requested bundled Vapi provider for the Voice Call plugin. The issue is concrete, not protected by labels/author association, but recent discussion already identified that the earlier implementation-complete comment did not land on main, and local source/docs checks confirm that remains true at 2f6615d2ee9d0d9db0f4ad1e13f71d1c0df8166d.

## Best Possible Solution

Keep the issue open for maintainer/product judgment or an implementation PR that adds Vapi to the bundled Voice Call provider surface. A complete fix would add a Vapi provider implementation, config/schema and manifest metadata, runtime registration, tests, and user-facing docs; alternatively, maintainers could explicitly redirect this to ClawHub if they decide the external Clawdia/Vapi bridge is the intended path.
