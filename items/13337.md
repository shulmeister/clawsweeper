## Summary

Keep open. Current main still does not include a Vapi phone provider in the bundled Voice Call plugin, and the current plugin API does not expose a public drop-in phone-provider registration seam that would make this exact request safely closable as ClawHub-only. The showcased Clawdia/Vapi bridge is useful related work, but it is an external bridge rather than the requested `extensions/voice-call` provider.

## Best Possible Solution

Keep #13337 open for maintainer/product judgment or an implementation PR. A complete core/plugin fix would add a Vapi Voice Call provider implementation, config schema and env handling, manifest/UI hints, runtime registration, webhook parsing/security behavior, unit tests, and docs. If maintainers prefer ClawHub, the better path is to either document the external Clawdia/Vapi bridge as the intended solution or add a public Voice Call phone-provider extension API before redirecting this exact provider request out of core.
