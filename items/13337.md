## Summary

Keep #13337 open. Current main contains the Voice Call plugin, but the phone-provider set is still closed to `telnyx`, `twilio`, `plivo`, and `mock`; there is no `VapiProvider`, Vapi config/env metadata, runtime branch, docs, or tests. The documented Clawdia phone bridge is useful adjacent Vapi bridge work, but it does not implement `provider: "vapi"` inside `@openclaw/voice-call`, and the public plugin API does not currently expose a telephony-provider registration seam that would make this request safely external-only.

## Best Possible Solution

Keep #13337 open for maintainer/product judgment or an implementation PR. A bundled implementation should add a Vapi `VoiceCallProvider`, Vapi config schema/env resolution/validation, runtime provider selection, manifest UI/env metadata, focused provider/runtime tests, and Voice Call docs. If maintainers prefer external ownership, first add and document a public telephony-provider capability seam or explicitly endorse the Clawdia Vapi bridge as the supported path.
