## Summary

Keep open. Current main still only partially overlaps the request: gateway runtime version is available over the WebSocket `hello-ok` handshake and `update.run` exists as an admin-scoped WS RPC, but the requested HTTP contract is still missing. `GET /health` and `/healthz` return only shallow liveness JSON without `version`, and there is no `POST /admin/update` HTTP route or `target_version` parameter.

## Best Possible Solution

Keep the issue open until maintainers either implement the requested HTTP API or explicitly choose a WS-only product direction. A conservative implementation would add a source-backed `version` field to `/health` and `/healthz` with tests/docs, then decide whether `POST /admin/update` should be a thin authenticated HTTP wrapper around `update.run`. If version pinning is required, extend the typed `UpdateRunParamsSchema` and update runner contract rather than adding ad hoc host-package-manager behavior.
