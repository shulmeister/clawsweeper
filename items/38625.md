## Summary

Keep open. Current main still has only partial overlap: the gateway exposes a runtime version in the WebSocket `hello-ok` handshake and has an authenticated `update.run` WebSocket RPC, but `GET /health` still returns only `{ ok, status }` and there is no `POST /admin/update` HTTP endpoint or `target_version` update parameter. The Crab fleet-monitoring use case in #38625 therefore still has real remaining product/API work.

## Best Possible Solution

Keep #38625 open until a maintainer decides and implements the API direction. The likely best implementation is additive: expose the runtime version on the HTTP health surface if that is accepted, and either extend the existing gateway update RPC/HTTP bridge to support a target version or add `POST /admin/update` by reusing `runGatewayUpdate`, existing gateway-token authorization, admin scoping, rate limiting, restart-sentinel handling, and docs/tests.
