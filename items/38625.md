## Summary

Keep open. Current main still only partially overlaps the request: OpenClaw exposes the runtime version on the WebSocket hello response and has an authenticated `update.run` WebSocket RPC, but `GET /health`/`/healthz` still returns only `{ ok, status }`, and there is no core `POST /admin/update` HTTP endpoint or `target_version` update parameter.

## Best Possible Solution

Keep the issue open until maintainers either implement the requested HTTP contract or explicitly choose a different product direction. A conservative implementation would add a source-backed runtime `version` field to the shallow probe response and tests, then decide whether `POST /admin/update` should be a thin authenticated HTTP wrapper around the existing `update.run` updater. If `target_version` is required, extend the typed update params and update runner contract rather than adding an ad hoc host-package path.
