## Summary

Keep open. Current main still starts outbound delivery recovery on gateway startup and replays pending `delivery-queue/*.json` entries based on retry count/backoff only, with no configurable TTL or expiry action. The concrete production reproduction in the issue discussion remains relevant, and the related SQLite/outbox PRs and umbrella were closed without landing this behavior on main.

## Best Possible Solution

Keep this issue open until outbound delivery recovery has a bounded, documented expiry policy. The core path likely needs optional `messages.delivery` configuration, an age check in startup recovery and reconnect drains before delivery, a safe default terminal action such as moving expired entries to `failed/`, and regression tests for old `enqueuedAt` entries, default backward compatibility, and configured expiry behavior.
