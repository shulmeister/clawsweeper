## Summary

Keep open. Current main still has no configurable TTL/expiry for outbound `delivery-queue` recovery: pending `.json` entries are loaded by age order and replayed based on retry count/backoff/max retries only. The shipped two-phase ack marker cleanup fixes a narrower already-acked replay window, but it does not expire old pending queue files, and the related SQLite/outbox PRs were closed unmerged while #32063 closed stale.

## Best Possible Solution

Keep this issue open until outbound delivery recovery has a bounded, documented expiry policy. The likely core fix is optional `messages.delivery` config, strict schema/help/docs updates, an age check in both startup recovery and reconnect drains before delivery, a conservative terminal action such as moving expired entries to `failed/`, and regression tests for old `enqueuedAt` entries, default backward compatibility, configured expiry behavior, and interaction with existing `.delivered` marker cleanup.
