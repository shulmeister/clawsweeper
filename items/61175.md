## Summary

Keep open. Current main only suppresses hook-pack fallback after security failures when `--dangerously-force-unsafe-install` is set; the ordinary plugin-install failure path can still try hook-pack installation and append `Also not a valid hook pack`, which is the reported confusing behavior.

## Evidence

- **Security fallback guard is force-only:** `shouldExitOnForcedUnsafeInstall` exits early on `security_scan_blocked`/`security_scan_failed` only when `forceUnsafeInstall` is true.
