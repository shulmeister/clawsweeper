## Summary

Current main still does not implement a config-backed default for `openclaw tui --deliver`. The TUI CLI only exposes an explicit `--deliver` flag defaulting to false, the runtime derives `deliverDefault` only from options, sends use that constant, and the config type/schema/docs have no `tui.deliver` surface. Related PRs #33198 and #33208 were closed unmerged, while #66991 tracks a related stuck-spinner failure mode rather than this persisted-default request.

## Best Possible Solution

Keep #33102 open until a maintainer-approved implementation adds a documented config key such as `tui.deliver`, wires it through `OpenClawConfig`, zod schema, schema help/generated docs, CLI precedence, and TUI runtime delivery, and adds focused tests for config default, explicit flag override, and false fallback. Track #66991 separately unless one PR intentionally fixes both the persisted default and the stuck-spinner UX path.
