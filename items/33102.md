## Summary

Keep open. Current `main` still has no config-backed default for TUI reply delivery: the CLI only reads the explicit `--deliver` flag, the TUI runtime defaults delivery from that option, and the config schema/docs do not define `tui.deliver`. The linked config PRs were closed unmerged, and the later stuck-spinner watchdog mitigation is separate from the requested persisted default.

## Best Possible Solution

Keep #33102 open until a maintainer-approved implementation adds a documented config key, likely `tui.deliver`, wires it through config types/schema/help/docs and CLI precedence, and adds focused tests for config default, explicit flag override, and fallback behavior. Track the stuck-spinner behavior in #66991 separately unless a single PR intentionally fixes both.
