## Summary

Keep #33102 open. Current main at c7a0d9b1889ca09ab5bfd84cda39a8d50d3d5768 still has no config-backed default for `openclaw tui --deliver`: the CLI flag is explicit-only, TUI delivery is derived from startup options, message sends forward that fixed value, and the config type/schema/help surfaces expose no `tui.deliver` key. The related implementation PRs #33198 and #33208 were closed unmerged in the provided GitHub context, and the linked #66991 stuck-spinner report is related but does not replace this persisted-default request.

## Best Possible Solution

Keep #33102 open until a maintainer-approved change adds a documented config key such as `tui.deliver`, wires it through `OpenClawConfig`, zod/generated schema help, CLI precedence, and TUI send behavior, and adds focused coverage for config default, explicit CLI override, and false fallback. Treat #33198/#33208 and the arcabot workaround as useful prior art; treat #66991 as related stuck-spinner context unless one PR intentionally fixes both surfaces.
