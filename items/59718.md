- **Current sandbox docs distinguish workspaceAccess from custom binds but do not document the requested file-tool write-through behavior:** The public sandboxing docs say workspaceAccess "none" makes tools see a sandbox workspace, "ro" disables write/edit/apply_patch, and "rw" mounts the agent workspace read/write. That overlaps with #59718's least-privilege goal and weighs against closing as implemented.
