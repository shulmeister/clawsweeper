## Summary

Keep #68494 open. The report is recent, concrete, and still tracks real remaining work: current main has partial mitigations for bootstrap prompt pressure and Telegram polling transport recovery, but it does not fully implement the reported session-level recovery for unrecoverable context overflow or the CLI-backed compaction/session-rotation path discussed in the issue. The two directly related fixes, PR #68388 and PR #68575, are still open/unmerged per the provided context and their head commits are not present in this checkout.

## Best Possible Solution

Keep the issue open until the maintainer chooses and lands the canonical fix path: rework or land the CLI-backed compaction/session-rotation work from #68388 so it applies to the affected CLI-style providers such as `cliproxy/*`, add the Telegram/session reset behavior from #68575 or an equivalent core recovery path for unrecoverable context overflow, and keep the existing Telegram transport rebuild/bootstrap-pressure mitigations as supporting hardening with focused regression tests.
