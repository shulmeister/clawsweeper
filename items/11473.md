## Summary

Keep open. Current `main` does not implement a token-free static cron payload. Cron still only accepts `systemEvent` and `agentTurn`, and execution still routes jobs through either heartbeat/system-event handling or an isolated agent run.

## Evidence

- **Cron payload types remain limited to two kinds:** `CronPayload` is still a union of only `systemEvent` and `agentTurn`; there is no `staticMessage` payload type.
  - file: [src/cron/types.ts:112](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/cron/types.ts#L112)
  - command: `nl -ba src/cron/types.ts | sed -n '90,135p'`
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Agent cron tool schema exposes only `systemEvent` and `agentTurn`:** The cron tool hard-codes `CRON_PAYLOAD_KINDS = ["systemEvent", "agentTurn"]` and its payload schema only describes `text` for system events or `message` for agent turns.
  - file: [src/agents/tools/cron-tool.ts:28](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/agents/tools/cron-tool.ts#L28)
