## Summary

Keep #11473 open. Current main still has no token-free `staticMessage` cron payload: cron payload contracts, Gateway validation, runtime execution, CLI, Control UI, agent tool schema, and public docs are still built around `systemEvent` and `agentTurn`. The provided issue discussion also includes a recent Codex review comment keeping this open and noting that #18160 is broader direct-exec cron work, not a shipped replacement for fixed text-to-channel delivery.

## Best Possible Solution

Keep #11473 open until OpenClaw either implements an additive token-free static message cron payload or explicitly supersedes it with a broader non-agent cron payload design that covers fixed text-to-channel delivery. A complete path should update `CronPayload`, Gateway TypeBox schemas and validators, normalization/patch logic, runtime direct outbound delivery with run-log/audit/failure reporting, CLI, Control UI, agent cron tool guidance, public docs, and focused tests. Cross-reference #18160 for broader direct-exec design, but do not close this narrower request yet.
