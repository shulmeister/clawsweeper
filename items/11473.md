## Summary

Keep #11473 open. Current main still has no token-free `staticMessage` cron payload: the cron type contract, Gateway schema, validation, runtime execution, agent cron tool surface, Control UI, and docs remain centered on `systemEvent` and `agentTurn`. Related #18160 is broader direct-exec work, remains open, and does not currently supersede this narrower fixed text-to-channel request.

## Best Possible Solution

Keep #11473 open until OpenClaw implements an additive token-free static message cron payload, or maintainers explicitly supersede it with a broader non-agent cron payload design that covers fixed text-to-channel delivery. A complete solution should update the cron payload type, Gateway TypeBox schema, normalization/validation, runtime direct outbound delivery with run-log/audit/failure reporting, CLI and Control UI create/edit flows, docs, and focused tests. Cross-reference #18160 for broader direct-exec design, but do not close this narrower request yet.
