## Summary

Keep #11473 open. Current main still does not implement a no-AI static cron message payload: cron payloads are limited to `systemEvent` and `agentTurn`, protocol/CLI/UI/docs expose only those modes, main jobs enqueue system events into the heartbeat path, and detached jobs still require `runIsolatedAgentJob`. The adjacent direct-exec cron items do not supersede this exact static channel-send request, and the latest maintainer discussion already points to keeping it open.

## Best Possible Solution

Keep #11473 open until maintainers either implement a first-class no-AI static channel-send cron payload or explicitly fold it into a broader non-agent cron payload design. The likely implementation belongs in the core cron contract and Gateway protocol, with CLI/UI/docs updates and tests for validation, scheduling, direct channel delivery, failure reporting, and token-free execution.
