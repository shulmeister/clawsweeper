## Summary

Keep #11473 open. Current main still lacks a token-free static cron message payload: cron payloads are still only `systemEvent` and `agentTurn`, Gateway validation rejects the proposed `staticMessage` shape, main jobs still enqueue system events into heartbeat processing, and detached jobs still run through `runIsolatedAgentJob`. The related exec/shell cron items do not cover this exact static channel-send request, and the latest maintainer discussion already says to keep it open.

## Best Possible Solution

Keep #11473 open until maintainers either implement a first-class no-AI static channel-send cron payload or explicitly fold it into a broader non-agent cron payload design. The likely implementation belongs in the core cron contract and Gateway protocol, with CLI/UI/docs updates and tests for validation, scheduling, direct channel delivery, failure reporting, and token-free execution.
