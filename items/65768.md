## Summary

Keep open. Current main still excludes `MINIMAX_OAUTH_TOKEN` from the MiniMax `web_search` credential path, and #65785 is an open implementation candidate linked with closing syntax, so the issue should not be closed before that PR is resolved.

## Best Possible Solution

Keep this issue open and review #65785 or an equivalent patch. The fix should either add `MINIMAX_OAUTH_TOKEN` consistently to the MiniMax web search runtime resolver, web search provider metadata/public artifact, setup/autodetect expectations, tests, and docs, or explicitly document that `web_search` requires a separate Coding Plan credential despite MiniMax portal OAuth.
