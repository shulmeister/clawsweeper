## Summary

Current `main` already provides programmatic cumulative session usage and cost access through Gateway RPCs and CLI JSON surfaces, including per-session totals, per-model/provider aggregates, and per-session cumulative timeseries.

## Risks / Open Questions

- There is not a dedicated top-level `openclaw usage --session` shortcut; session access is exposed through Gateway RPC/CLI surfaces instead.
- Per-log rows expose tokens/cost, while model information is exposed in aggregated `modelUsage`/`byModel` summaries rather than on each individual log row.

## Close Comment

Closing from a Codex review because current OpenClaw already implements the requested programmatic session-usage surface. `src/gateway/server-methods/usage.ts:397` exposes `usage.cost`, `sessions.usage`, `sessions.usage.timeseries`, and `sessions.usage.logs`; `src/infra/session-cost-usage.ts:554` aggregates cumulative per-session token/cost totals plus per-model/provider breakdowns from transcript usage entries; and `src/infra/session-cost-usage.ts:826` returns cumulative token/cost timeseries data for a session. The public Gateway docs list these RPCs in `docs/gateway/protocol.md:281`, and the CLI exposes JSON-friendly access through `src/cli/gateway-cli/register.ts:420` and `src/cli/gateway-cli/register.ts:443`. The `v2026.4.22` release already contains these RPCs. If you still want a narrower follow-up like a dedicated top-level CLI shortcut or per-log model fields, please open a focused new issue for that gap.
