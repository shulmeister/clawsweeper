## Summary

Keep open. Current main still forwards bundle MCP tool arguments without schema-based coercion, so a call like `{ "params": "{}" }` would remain a string when sent to the MCP SDK. The relevant fix exists only in open, unmerged PR #70882, which is still dirty and has review comments, so the issue should remain open until a maintainer lands a fix.

## Best Possible Solution

Keep #70872 open and use PR #70882, or a maintainer-authored successor, as the implementation path. The fix should live at the bundle MCP materialization/runtime boundary, coerce schema-declared object/array properties from valid JSON strings before `client.callTool`, and add regression tests covering stringified object and array params plus already-correct object params. Maintainers should address the open review concerns on #70882 before landing.
