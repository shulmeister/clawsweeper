## Summary

Keep open. Current main has partial keyless LM Studio handling for non-interactive setup and runtime, but the reported interactive `openclaw onboard --install-daemon` path still requires a non-empty LM Studio API-key prompt when `LM_API_TOKEN` is absent.

## Best Possible Solution

Keep #66937 open. The best fix is to make LM Studio interactive onboarding accept a blank API-key prompt as unauthenticated local mode, synthesize the existing non-secret local marker instead of writing a real auth profile, keep real key/header auth behavior intact, add an interactive regression test for the reported flow, and update the LM Studio provider docs to say the API key is optional for unauthenticated local servers.
