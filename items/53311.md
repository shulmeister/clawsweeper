## Summary

Keep open. The issue is paired with an open same-author PR that explicitly closes it, and current main still turns an unknown configured channel id into a validation issue that blocks gateway startup instead of warning and skipping the missing channel/plugin.

## Best Possible Solution

Keep this issue open while maintainers review PR #53315 or a replacement patch. The likely product fix belongs in `src/config/validation.ts` and gateway startup coverage should prove a configured but undiscoverable plugin-owned channel warns and is skipped while other channels still start, with docs updated if the validation contract changes.
