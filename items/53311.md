## Summary

Keep open. Current `main` still fails gateway startup when `channels.<id>` refers to a missing/uninstalled channel plugin. The config validator emits a fatal `unknown channel id` issue, startup throws on invalid config snapshots, and the only recovery on `main` is a narrow `openclaw plugins install` Matrix reinstall path rather than graceful runtime skipping.

## Evidence

- **Missing channel ids are still fatal config issues:** `validateConfigObjectWithPlugins` still pushes `channels.<id>: unknown channel id: <id>` into `issues`, not `warnings`, when a configured channel id is absent from bundled/plugin channel registries.
  - file: [src/config/validation.ts:920](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/config/validation.ts#L920)

- **Gateway startup still aborts on invalid config:** Gateway startup reads the config snapshot and immediately calls `assertValidGatewayStartupConfigSnapshot(...)`; that helper throws when `snapshot.valid` is false, so a fatal `unknown channel id` still stops startup rather than skipping the missing channel.
  - file: [src/gateway/server-startup-config.ts:71](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/server-startup-config.ts#L71)
