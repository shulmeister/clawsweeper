## Summary

Keep #70807 open. Current main does not contain the PR’s compat timeout propagation, media timeout widening, or Ollama helper scripts, and the core timeout issue remains plausible. The PR should not be cleaned up as obsolete, but it is also not merge-ready as-is because the review discussion identifies concrete defects in the runtime-config threading and Ollama scripts.

## Best Possible Solution

Keep the PR open for maintainer or author follow-up. The best path is to revise the core fix so the runtime agent timeout is carried through a runtime-only option independent of the user config object, add regression coverage for the no-endpoint-config/CLI-enabled compat path, keep the config type aligned with the Zod schema, and handle invalid timeout values consistently with `agent-command.ts`. The media timeout and Ollama helper scripts should be split or explicitly scoped; if kept, the scripts need fd cleanup, configurable listen/target addresses, and defaults aligned with the documented Ollama `11434` base URL.
