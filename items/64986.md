## Summary

Keep open. Current main still has a plausible built-in-test bug: the macOS Voice Wake test UI tells users to say only a trigger word, but the tester code only reports success when there is non-empty post-trigger command text. The runtime path separately supports trigger-only wake detection, so tester and runtime remain inconsistent and this report is not safely closable.

## Evidence

- **Test UI promises trigger-only detection:** The built-in macOS test card instructs users to "say a trigger word" / "say your trigger word," which matches the reporter's expectation that the test should succeed on wake-word detection alone.
  - file: [apps/macos/Sources/OpenClaw/VoiceWakeTestCard.swift:74](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/apps/macos/Sources/OpenClaw/VoiceWakeTestCard.swift#L74)
  - sha: [ae5c6573670b](https://github.com/openclaw/openclaw/commit/ae5c6573670b3742b156899bd2b9c36b8677f191)
- **Tester requires non-empty command after trigger:** `VoiceWakeTester.handleResult` only marks detection when `match.command` is non-empty; otherwise final results fall through to failure states like `No trigger heard`. The tester's silence fallback also depends on `textOnlyFallbackMatch`, which returns nil when the stripped command is empty.
  - file: [apps/macos/Sources/OpenClaw/VoiceWakeTester.swift:233](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/apps/macos/Sources/OpenClaw/VoiceWakeTester.swift#L233)
  - sha: [ae5c6573670b](https://github.com/openclaw/openclaw/commit/ae5c6573670b3742b156899bd2b9c36b8677f191)
- **Text-only fallback still rejects trigger-only utterances:** `WakeWordGateConfig` defaults `minCommandLength` to `1`, and `VoiceWakeTextUtils.textOnlyCommand` rejects transcripts whose post-trigger text is shorter than that. A transcript like just `openclaw` therefore cannot pass the tester fallback path.
  - file: [Swabble/Sources/SwabbleKit/WakeWordGate.swift:24](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/Swabble/Sources/SwabbleKit/WakeWordGate.swift#L24)
  - sha: [ae5c6573670b](https://github.com/openclaw/openclaw/commit/ae5c6573670b3742b156899bd2b9c36b8677f191)
- **Runtime supports trigger-only wake, unlike tester:** The main macOS runtime has explicit trigger-only pause handling: when only the trigger is heard, it schedules `triggerOnlyPauseCheck` and can begin capture with an empty command. That makes the built-in tester behavior materially different from the real runtime.
  - file: [apps/macos/Sources/OpenClaw/VoiceWakeRuntime.swift:371](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/apps/macos/Sources/OpenClaw/VoiceWakeRuntime.swift#L371)
  - sha: [ae5c6573670b](https://github.com/openclaw/openclaw/commit/ae5c6573670b3742b156899bd2b9c36b8677f191)
