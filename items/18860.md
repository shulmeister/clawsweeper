## Summary

Keep PR #18860 open. Current main at b3ac316e0b0d7e9ba0990eff9b8c575c4ba21b69 still does not expose an `after_tools_resolved` plugin hook, a hook-runner dispatch method, or an equivalent plugin-facing API that returns the finalized post-policy tool list with JSON Schema parameters. The linked feature issue #21330 is closed and points back to this PR rather than superseding it; the recent maintainer/Codex comment also explicitly kept this PR open.

## Best Possible Solution

Keep the PR open as useful design and implementation context, but land only a narrow maintainer-approved plugin/runtime seam. The likely implementation belongs in `src/plugins/hook-types.ts`, `src/plugins/hooks.ts`, `src/plugins/types.ts`, and `src/agents/pi-embedded-runner/run/attempt.ts`, with focused contract tests and docs for plugin authors. If maintainers prefer the issue body's pull-based alternative, create a canonical follow-up and close/supersede this PR only after that path is explicitly tracked.
