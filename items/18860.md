## Summary

Keep this PR open. Current main still lacks the requested `after_tools_resolved` hook or an equivalent plugin-facing API for resolved per-attempt tool/schema discovery, and the latest maintainer-side Codex review in the provided discussion explicitly said to keep this implementation/design path open.

## Best Possible Solution

Keep this PR open as the active implementation/design path for plugin-facing resolved tool/schema discovery. A maintainer-quality path should either finish and review this additive hook across `src/plugins/hook-types.ts`, `src/plugins/hooks.ts`, `src/plugins/types.ts`, `src/agents/pi-embedded-runner/run/attempt.ts`, docs, and focused contract tests, or explicitly supersede it with a chosen pull-based `getResolvedTools` API before closing this PR.
