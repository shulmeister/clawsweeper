## Summary

Keep #18860 open. Current main still does not expose the requested `after_tools_resolved` plugin hook, a hook-runner method for it, or an equivalent plugin-facing API that returns the resolved post-policy tool list with JSON schemas. The linked feature issue #21330 is closed and points back to this PR, so it does not supersede the remaining work.

## Best Possible Solution

Keep the PR open as useful implementation and design context, but land only a narrow core plugin/runtime seam if maintainers approve the feature. The likely path is to add a typed hook or equivalent resolved-tools API in `src/plugins/hook-types.ts`, `src/plugins/hooks.ts`, and the embedded runner after `effectiveTools` is finalized, with focused contract tests and docs for plugin authors.
