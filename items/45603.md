## Summary

Current main no longer has the stale Feishu duplicate-warning path reported here. Feishu is modeled as a bundled channel, built-in channel auto-enable materializes `channels.<id>.enabled` without creating a `plugins.entries.<id>` entry, and manifest discovery suppresses false duplicate diagnostics when multiple candidates resolve to the same physical plugin. The same relevant implementation is present in the latest release tag `v2026.4.24`.
