## Summary

Keep open. Current main still does not honor `--custom-base-url` for `--auth-choice litellm-api-key` non-interactive onboarding: LiteLLM remains on the generic single-provider API-key path, whose non-interactive handler never reads `ctx.opts.customBaseUrl`. The PR is not obsolete, though it still needs maintainer review of the discovery/logging tradeoffs raised in review comments.

## Best Possible Solution

Keep the PR open and either land a focused LiteLLM non-interactive handler that reads `ctx.opts.customBaseUrl` while preserving existing config, or ask the contributor to narrow the patch and add regression coverage for `litellm-api-key` plus `--custom-base-url`. Maintainers should also decide the model-discovery defaults before merge.
