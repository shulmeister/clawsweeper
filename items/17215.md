## Summary

Keep #17215 open. Current main has useful upgrade safety pieces, including `openclaw update --dry-run`, `--json`, post-update doctor/plugin sync, config migration/repair, rollback docs, and plugin compatibility diagnostics. It still does not implement the requested native pre/post upgrade compatibility scanner with risk scoring, smoke recommendations, rollback recommendation hints, or a `doctor --upgrade-scan`-style command. The prior issue discussion also already reached the same keep-open conclusion, and current main has not changed that outcome.

## Best Possible Solution

Continue tracking this as a core update/doctor safety enhancement. The strongest product path is a native upgrade scan, likely under `openclaw update` and/or `openclaw doctor`, that emits both human and JSON reports covering pre-update config/schema/deprecation checks, post-update health/plugin/runtime checks, smoke-test recommendations, risk classification, and rollback guidance, with docs and tests for the new report contract.
