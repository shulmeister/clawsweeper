## Summary

Keep #17215 open. Current main has update dry-run, post-update plugin sync, doctor health checks, plugin compatibility metadata, and manual rollback docs, but it still does not implement the requested native pre/post-update compatibility scanner with `doctor --upgrade-scan`, risk-scored JSON and human reports, smoke recommendations, or rollback hints. Related PR #54354 remains narrower plugin engine-compatibility work and is not a shipped substitute.

## Best Possible Solution

Keep the issue open as a core update/doctor safety enhancement. The best path is a native `openclaw update` and/or `openclaw doctor --upgrade-scan` mode that produces human and JSON reports for pre-update config/schema/deprecation checks, post-update gateway/plugin/runtime health checks, smoke-test recommendations, risk classification, and rollback guidance, with docs and focused tests for the report contract. Treat PR #54354 as related plugin-specific compatibility work if it lands, not as a full superseding fix.
