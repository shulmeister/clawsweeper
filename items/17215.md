## Summary

Keep #17215 open. Current main has adjacent update safety pieces (`openclaw update --dry-run`, final `openclaw doctor`, plugin sync/integrity drift reporting, plugin doctor compatibility notices), but it still does not provide the requested native pre/post-update compatibility scanner with risk scoring, smoke recommendations, rollback hints, or a `doctor --upgrade-scan`-style command. The existing maintainer discussion also explicitly kept this open, and current main still matches that assessment.

## Best Possible Solution

Continue tracking this as a core update/doctor safety enhancement. The best product path is a native upgrade scan under `openclaw update` and/or `openclaw doctor` that emits both human and JSON reports covering pre-update config/schema/deprecation checks, post-update health/plugin/runtime checks, smoke-test recommendations, risk classification, and rollback guidance, with docs and tests for the report contract. It should reuse existing `openclaw update --dry-run`, final doctor repair, `status --deep`, `plugins doctor`, plugin compatibility diagnostics, and rollback/update docs rather than creating an isolated checker.
