## Summary

Keep #17215 open. Current main has adjacent update-safety pieces, including `openclaw update --dry-run`, post-update plugin sync, and non-interactive doctor repair during updates, but it still does not implement the requested native pre/post-update compatibility scanner with risk-scored JSON/human reports, smoke recommendations, rollback hints, or a `doctor --upgrade-scan` surface. The only GitHub comment in the provided context is a maintainer Codex review that also kept this issue open on 2026-04-26.

## Best Possible Solution

Continue tracking this as a core update/doctor safety enhancement. The best path is a native `openclaw update` and/or `openclaw doctor` upgrade-scan mode that emits human and JSON reports for pre-update config/schema/deprecation checks, post-update health/plugin/runtime checks, smoke-test recommendations, risk classification, and rollback guidance, with docs and focused tests for the report contract.
