## Summary

Keep #17215 open. Current main still has useful upgrade safety pieces, but it does not provide the requested native pre/post-update compatibility scanner with risk scoring, smoke recommendations, rollback recommendation hints, or a `doctor --upgrade-scan`-style command. The existing maintainer-side Codex review comment reached the same keep-open conclusion, and current main still matches that assessment.

## Best Possible Solution

Continue tracking this as a core update/doctor safety enhancement. The best product path is a native upgrade scan under `openclaw update` and/or `openclaw doctor` that emits both human and JSON reports covering pre-update config/schema/deprecation checks, post-update health/plugin/runtime checks, smoke-test recommendations, risk classification, and rollback guidance, with docs and tests for the report contract.
