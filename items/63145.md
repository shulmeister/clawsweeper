## Summary

Keep open. Current main still does not implement `openclaw models status --probe --all`: the status CLI has no `--all` flag, the command options do not carry a `probeAll`/equivalent switch, and probe target construction still chooses one model per provider via `selectProbeModel`. The open PR #63259 directly tracks the requested implementation, but it has not merged; a maintainer also closed broader issue #12215 in favor of #63145 as the canonical remaining OpenClaw-core tracker.

## Best Possible Solution

Keep #63145 open as the canonical issue. Finish maintainer review of PR #63259 or an equivalent patch that adds a status-level `--all` flag, threads a `probeAll` option through the status command and probe summary, expands probe targets across every configured primary/fallback/image model while preserving existing `--probe` behavior, updates docs, and covers JSON/provider-filter behavior with tests. Close #63145 when that implementation is merged and shipped.
