## Summary

Keep open. Current main still has the inherited `OPENCLAW_LAUNCHD_LABEL` precedence that #66000 tries to address, so the PR is not obsolete or implemented elsewhere. But the PR as written needs maintainer follow-up because it would clear non-matching custom launchd labels even though current code and tests support explicit `OPENCLAW_LAUNCHD_LABEL` overrides.

## Best Possible Solution

Keep the PR open for maintainer revision rather than closing it. The best fix is a narrower reconciliation in the `--profile` path: clear or replace only stale inherited stock OpenClaw gateway labels from another profile, while preserving explicit custom `OPENCLAW_LAUNCHD_LABEL` overrides. Add regression coverage for inherited stock-label conflict, matching stock-label preservation, custom-label preservation, and absent-label no-op behavior.
