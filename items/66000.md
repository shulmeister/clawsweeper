## Summary

Keep this PR open. Current main still has the linked profile/launchd precedence bug: `--profile` updates profile state, but an inherited `OPENCLAW_LAUNCHD_LABEL` still wins plist resolution. The PR is a plausible implementation candidate, but its current clearing rule still needs maintainer review because current code and tests protect explicit custom launchd-label overrides.

## Best Possible Solution

Keep this PR open and request a narrower fix: when `--profile` is explicitly supplied, ignore or clear stale inherited OpenClaw-managed labels such as `ai.openclaw.<other-profile>`, while preserving explicit custom `OPENCLAW_LAUNCHD_LABEL` overrides. Add regression tests for stale managed labels, matching managed labels, custom labels, absent labels, and the gateway status/plist-resolution path for #65643.
