## Summary

Keep #8190 open. Current main still has per-session reset, per-session soft reset, and CLI auth-identity reuse invalidation, but no global auth-switch/session-refresh action across active channel sessions with preserved context. The 2026-04-26 maintainer/Codex issue comment reached the same conclusion, and the relevant surface has not gained a global reset protocol since then.

## Best Possible Solution

Keep #8190 open as the canonical tracker. The best product path is an explicit additive gateway capability, such as a scoped `sessions.reset` mode or new auth-refresh/reset-all method, that refreshes auth-backed runtime state for all active sessions while preserving transcripts/context where possible. It should reuse the existing per-session soft reset and CLI auth-epoch invalidation pieces, but add multi-session protocol, server implementation, channel/session-store tests, and user-facing docs before closure.
