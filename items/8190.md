## Summary

Keep #8190 open. Current main has useful partial mechanisms: `sessions.reset` exists, `/reset soft` can preserve the current transcript while clearing CLI backend session ids, and CLI session reuse now considers auth profile/auth epoch. But the requested core capability is a global auth-switch/session-refresh action across active sessions with context preservation, and current protocol/implementation still only exposes per-session reset keyed by a single `key`.

## Evidence
