## Summary

Keep #8190 open. Current main still has single-session hard reset and current-session soft reset paths, but no global all-session auth refresh/reset that preserves conversation context across active channel sessions. The closed #7533 auth-order/account-switching work is related but does not supersede this cross-session lifecycle request, and the latest issue discussion already explicitly kept #8190 open.

## Best Possible Solution

Keep #8190 as the canonical tracker. The best implementation is an additive gateway/session capability such as `sessions.refreshAuth` or an explicit all-sessions auth-refresh mode that enumerates active session stores, invalidates auth-backed runtime and CLI caches, preserves transcripts/session context where possible, and adds protocol, gateway, channel/session-store tests plus user-facing docs. Treat #7533 as shipped account/auth-order switching, not a replacement for this global session refresh behavior.
