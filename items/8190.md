## Summary

Keep #8190 open. Current main still lacks a global auth-switch/session-refresh action that updates all active sessions while preserving context. The implemented pieces are narrower: `sessions.reset` is single-session, hard reset rotates transcript/session state, `/reset soft` clears CLI reuse only for the current session, and CLI auth-identity invalidation is only a per-session mitigation.

## Best Possible Solution

Keep #8190 open as the canonical tracker. The best path is an additive gateway capability, such as `sessions.reset` with an explicit all-sessions/auth-refresh scope or a new `sessions.refreshAuth` method, that enumerates active sessions, invalidates auth-backed runtime/CLI caches, preserves transcripts and per-channel context where possible, and adds protocol, server, channel/session-store tests, and docs.
