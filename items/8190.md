## Summary

Keep #8190 open. Current main still has single-session reset, current-session soft reset, and CLI auth-identity reuse invalidation, but it does not implement a global auth-switch/session-refresh action that updates all active channel sessions while preserving context. A maintainer/Codex comment on the issue already reached the same conclusion, and current main has not added the missing protocol or server behavior.

## Best Possible Solution

Keep #8190 open as the canonical tracker. The best path is an additive gateway capability, such as a scoped `sessions.reset` mode or new auth-refresh/reset-all method, that refreshes auth-backed runtime state for all active sessions while preserving transcripts/context where possible. It should reuse the existing per-session soft reset and CLI auth-epoch invalidation pieces, then add multi-session protocol, implementation, tests, and user-facing docs.
