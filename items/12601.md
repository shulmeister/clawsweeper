## Summary

Keep #12601 open. Current main still lacks Slack Canvas content read support: there is no Slack `read-canvas` message action, no runtime dispatch to Slack `canvases.sections.lookup`, no `canvases:read` scope in Slack setup/docs, and the related PR #18752 was closed without merge.

## Best Possible Solution

Keep this issue open for the bundled Slack plugin unless a maintainer explicitly redirects it to ClawHub. The likely implementation belongs in `extensions/slack`: add a gated Canvas read action, add `canvases:read` to Slack setup/docs, call Slack's Canvas read API with the existing token/account model, return markdown safely, and add unit/contract coverage plus live verification when credentials are available.
