## Summary

Keep #12601 open. Current main still lacks Slack Canvas content read support: Slack exposes no `read-canvas` action, dispatches no Slack Canvas API call, has no `canvases:read` setup/docs/config gate, and the existing `download-file` path only handles Slack file metadata/private downloads. The related PR #18752 was closed unmerged as not merge-safe, so it does not supersede the issue.

## Best Possible Solution

Keep this open for the bundled Slack plugin. A clean implementation should live in `extensions/slack`: add a gated Canvas read action, add `canvases:read` to Slack setup/docs, call the appropriate Slack Canvas read API through the existing account/token model, return bounded markdown/section data safely, and add action discovery, runtime dispatch, config/docs, and live-verification coverage.
