## Summary

Keep #12601 open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still does not expose Slack Canvas content reading: the bundled Slack plugin advertises and dispatches existing message/file/pin/member/emoji actions only, setup/docs do not include `canvases:read`, and the linked implementation PR #18752 was closed unmerged.

## Best Possible Solution

Keep #12601 open for the bundled Slack plugin. A complete fix should add a Canvas read action or dedicated Canvas tool under `extensions/slack`, expose it through Slack message action discovery and dispatch, add `canvases:read` to setup/docs/config as needed, call Slack Canvas APIs through the existing account/token model, return bounded markdown or section data, and cover discovery, dispatch, runtime behavior, docs, and live Slack validation. The closed PR #18752 is useful prior art but not a canonical merged fix.
