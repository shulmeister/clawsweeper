## Summary

Keep #12601 open. Current main still does not expose Slack Canvas content reading in the bundled Slack plugin, and the latest maintainer issue discussion already kept it open as Slack plugin work. The related PR #18752 is useful prior art for a `read-canvas` action, but it was closed unmerged and is not present on main.

## Best Possible Solution

Keep #12601 open for the bundled Slack plugin. A complete fix should add a Slack Canvas read action or dedicated Canvas tool under `extensions/slack`, add `canvases:read` setup and docs guidance, call Slack Canvas APIs through the existing account/token model, return bounded markdown or section data, and cover discovery, dispatch, runtime behavior, docs, tests, and ideally live Slack validation. PR #18752 remains useful prior art but was closed unmerged.
