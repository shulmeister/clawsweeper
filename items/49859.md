## Summary

Current `main` does not contain the PR's exact UI fallback change: `parseSessionKey()` still renders group sessions as `${channelLabel} Group` with no identifier, and `resolveSessionDisplayName()` only avoids that fallback when a row already has `label` or `displayName`. Telegram inbound paths now do record richer per-group metadata, so many live rows likely show unique names already, but that is indirect and does not prove the report is obsolete for every session row. With that ambiguity, and because the PR also bundles unrelated gateway/scripts changes not on `main`, there is not strong enough evidence to close this conservatively.

## Evidence
