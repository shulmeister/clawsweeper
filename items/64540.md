## Summary

Keep open. Current main still does not implement the empty-heartbeat-object behavior, and this PR is a plausible implementation candidate for the linked bug, but the patch shown in the provided PR context is not complete enough to close as obsolete or already implemented.

## Best Possible Solution

Keep this PR open for maintainer review or contributor rework. If OpenClaw should accept `heartbeat: {}` as an explicit disable signal, implement that rule once and apply it consistently across `src/infra/heartbeat-summary.ts`, `src/infra/heartbeat-runner.ts`, and `src/agents/heartbeat-system-prompt.ts` before default merging, then add regression tests for defaults and per-agent empty heartbeat configs with and without inherited defaults. If maintainers want `0m` to remain the only disable contract, document that decision clearly and close the linked bug rather than merging a partial behavior change.
