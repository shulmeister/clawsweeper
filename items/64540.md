## Summary

Do not close PR #64540 as conservative cleanup. Current main still treats an empty heartbeat object as a configured/default heartbeat path, and the linked bug #64293 remains a real current behavior question rather than an obsolete PR. The PR is plausible, but it needs maintainer review and likely changes beyond `src/infra/heartbeat-summary.ts` so runtime scheduling, prompt guidance, docs, and tests stay aligned.

## Best Possible Solution

Keep the PR open for maintainer review. If OpenClaw wants `heartbeat: {}` to mean disabled, update the patch across the shared heartbeat policy surface: summary helpers, runner explicit-agent selection/scheduling, `runHeartbeatOnce`, system-prompt guidance, and bootstrap gating. Add focused regression tests for defaults `{}`, per-agent `{}`, interaction with defaults plus non-default agents, and prompt omission; then update `docs/gateway/heartbeat.md` and `docs/gateway/config-agents.md` to document the accepted disable forms. If maintainers prefer the existing documented contract, close or redirect the PR after explicitly saying `every: "0m"` is the supported disable mechanism.
