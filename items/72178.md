## Summary

Keep open. Current main still lacks cursor-based `chat.history` pagination and the Web UI load-older path proposed by PR #72178. The PR is recent and concrete, but the provided review discussion identifies a real cursor-overlap bug that should be fixed or folded into a canonical pagination PR rather than auto-closed.

## Best Possible Solution

Keep #72178 open for maintainer review or author revision. The best path is a single canonical `chat.history` pagination implementation covering the TypeBox schema, gateway handler, docs, generated clients, and Web UI load-older state/rendering, with regression tests for byte-budget trimming, sanitized mid-slice drops, duplicate prevention, and existing gateway `NO_REPLY` lead-fragment behavior.
