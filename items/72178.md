## Summary

Keep open. Current main still lacks cursor pagination for `chat.history`, the web UI has no load-older chat flow, and partial `NO_REPLY` stream prefixes are still not suppressed. The PR also has unresolved review feedback about a cursor duplication bug, so it needs maintainer review or changes rather than cleanup closure.

## Best Possible Solution

Keep the PR open, but require changes before merge: fix cursor derivation so it is based on the raw index of the first retained message after sanitization and byte capping, add a regression test for sanitized mid-page `NO_REPLY`/commentary rows, and either coordinate the pagination work with PR #56383 or split the partial `NO_REPLY` fix into a narrower PR if maintainers prefer.
