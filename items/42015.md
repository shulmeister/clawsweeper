## Summary

Close as `clawhub`. The PR is a large optional integration of the external AgentOrientedTUI/AOTUI runtime into OpenClaw core, while current main already has plugin surfaces for gateway services, dynamic per-session tools, prompt/context injection, lifecycle hooks, compaction hooks, CLI registration, and ClawHub/npm distribution. No protected labels or maintainer author signal require keeping it open for automatic cleanup.

## Best Possible Solution

Move the AOTUI/Agent Apps host into an external OpenClaw plugin published through ClawHub/npm using `registerService`, dynamic `registerTool` factories, `before_prompt_build`, compaction/lifecycle hooks, plugin config, and plugin CLI registration. If that implementation proves a specific SDK gap, open a small API-focused PR or design discussion for that seam.
