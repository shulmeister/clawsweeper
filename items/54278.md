## Summary

Keep this issue open. Current main still has useful partial support for Codex OAuth reliability and usage visibility, but it does not implement the requested product contract: explicit quota/auth diagnostic states, Control UI quota-wait vs reauth guidance/actions, or a brokered host-side reauth execution API. The prior ClawSweeper keep-open comment remains consistent with current main.

## Best Possible Solution

Keep this open as a core/provider plus Control UI product issue. Extend the existing models.authStatus/provider-usage path with an explicit diagnostic object such as ok, expiring_soon, quota_wait, and reauth_required; preserve WHAM-aware cooldown behavior; surface 5-hour and weekly Codex renewal details in local time; and add a narrow authorized broker/adapter execution API that is only available when diagnosis is reauth_required. Broader reason-aware cron guardrails can remain tracked separately in #14376.
