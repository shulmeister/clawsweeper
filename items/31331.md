## Summary

Current main still appears to have the Docker-outside-of-Docker sandbox bind-source mismatch. The Docker backend builds managed workspace mounts from gateway-visible paths without remapping them to host Docker paths, and the Docker install flow still mounts host storage at `/home/node/.openclaw/...` while onboarding runs inside the container. The related fix PRs #31355 and #31457 were closed unmerged, and the issue discussion still has recent same-issue reports.

## Best Possible Solution

Keep #31331 open. Best fix is to make Docker sandbox container creation host-path aware for Docker-hosted gateways: remap managed workspace, sandbox workspace, and agent workspace bind sources through the gateway container mount table before validation/hash/create args, and cover `workspaceAccess` `rw`, `ro`, `none`, and sandbox browser containers with regression tests. If maintainers prefer the host-path parity contract instead, update Docker setup/onboarding so it writes host paths and mounts identical paths by default, with docs/tests proving the Docker install path works.
