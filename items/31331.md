## Summary

Keep #31331 open. Current main still mounts Docker sandbox workspaces from gateway-visible paths and does not remap those paths to host Docker paths for Docker-outside-of-Docker. The Docker install path still maps host workspace storage to `/home/node/.openclaw/workspace`, while sandbox create args use the resolved gateway paths literally. The issue is also canonical (`dedupe:parent`) with recent same-issue reports, and the related fix PRs #31355 and #31457 were closed unmerged in the provided context.

## Best Possible Solution

Keep #31331 open as the canonical Docker-install/DooD sandbox workspace bind issue. Best fix is to make Docker sandbox and sandbox-browser container creation host-path aware by remapping managed workspace, sandbox workspace, and agent workspace bind sources through the gateway container mount table before validation, hashing, and `docker create`; add regression coverage for `workspaceAccess` `rw`, `ro`, and `none`. If maintainers prefer host-path parity as the product contract instead, update Docker setup/compose/onboarding so it writes and mounts identical host paths by default, with docs/tests proving that the Docker install sandbox path works.
