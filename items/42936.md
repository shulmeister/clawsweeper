## Summary

Keep open. Current main already covers Feishu Card Kit streaming and auto-mode markdown/card routing, but it still only accepts `channels.feishu.streaming` as a boolean. The PR’s remaining object-config request (`enabled`, `throttleMs`, `title`) is not implemented in current source, plugin manifest, generated config metadata, or docs, so this is not safely closeable as implemented or superseded.

## Best Possible Solution

Keep the PR open, but ask for a narrowed rebase against current main. The best path is to implement only the remaining Feishu object config support across `extensions/feishu/src/config-schema.ts`, `extensions/feishu/openclaw.plugin.json`, generated config metadata, docs, and runtime wiring, with regression coverage for omitted/default streaming, `streaming: false`, `{ enabled: false }`, custom `throttleMs`, and custom title behavior.
