## Summary

Keep open. Current main has partial cleanup for heartbeat history and queued System: events, but the broader Control UI request is not fully implemented because the pre-compaction memory-flush prompt still travels through the normal embedded prompt path and the chat-history display projection has no memory-flush/internal-turn filter.

## Best Possible Solution

Keep the issue open. The best maintainer path is to make internal/runtime turns share one display contract: heartbeat prompt/OK rows remain hidden, queued restart/system events stay out of the transcript prompt, and memory-flush runs should pass a hidden transcript prompt or stable internal provenance so `chat.history` and Control UI can suppress them while raw audit/replay data remains available. Add focused tests for `chat.history` covering heartbeat, restart/system-event notices, and pre-compaction memory flush prompts.
