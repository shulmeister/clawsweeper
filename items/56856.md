## Summary

Keep open. Current `main` has partial status-state work in the TUI (`waiting`, `streaming`, `running`, disconnect handling, and a streaming watchdog), but it does not implement the issue's requested operator-facing label model. The status line still renders raw internal states plus connection text, waiting uses whimsical phrases instead of a stable `Waiting | Connected` label, and there is no distinct `Generating` label or explicit yielded/subagent-wait presentation that satisfies the acceptance criteria.

## Evidence

- **Status line still renders raw activity/connection strings:** Busy states render as `${activityStatus} • ${elapsed} | ${connectionStatus}` and idle renders as `${connectionStatus} | ${activityStatus}`. There is no mapping layer for `Idle`, `Generating`, `Working`, `Waiting`, `Reconnecting`, and `Disconnected` labels as requested.
