## Summary

Keep PR #69340 open. Current main still has the underscore-only copy-sensitive guard in the TUI formatter, so long hyphen-separated package/file-like tokens still fall through to 32-character splitting with inserted spaces. The PR remains a focused fix for still-open bug #48432, and the bot review comment supports the same narrow root cause rather than showing the work is obsolete.

## Best Possible Solution

Keep #69340 open or supersede it only with an equivalent current-main formatter fix. The best landing path is a narrow `src/tui/tui-formatters.ts` change that preserves file-like hyphen-separated long tokens, plus a focused `sanitizeRenderableText` regression test for a #48432-style package token such as `ubuntu-budgie-themes-long-package-name`, while retaining the existing narrow-terminal splitting behavior for ordinary long prose tokens. Coordinate with related TUI copy-safety work such as #39565 if maintainers prefer a broader formatter policy.
