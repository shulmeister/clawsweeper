## Summary

Keep #39117 open. Current main still does not include the PR’s queued-message busy receipt, per-session stuck-lane watchdog recovery, aborted-run reply suppression, or richer sessions.reset audit metadata. The PR also has an unresolved P1 review concern around clearing ACTIVE_EMBEDDED_RUNS before the original timed-out run has actually stopped, so this needs maintainer review or a revised implementation rather than conservative auto-close.

## Best Possible Solution

Keep the PR open for maintainer judgment or author revision. The safest path is to split or revise the implementation: add a user-visible queued receipt or shared onQueued seam, add stuck-lane recovery that does not mark a session idle before the original run has actually stopped, suppress aborted-run stale assistant fallbacks with focused tests, and decide whether sessions.reset should expose persisted audit metadata.
