## Summary

Keep #13620 open. Current main still lacks outbound Telegram album/media-group support: multi-media reply payloads are routed through per-item sends, direct Telegram message actions accept one media URL, outbound payload delivery fans out media sequentially, docs only advertise a single `mediaUrl`, and related PRs were closed unmerged with their referenced commits absent from this checkout.

## Best Possible Solution

Keep the issue open and implement album support in the bundled Telegram plugin: add a guarded `sendMediaGroup` path for 2-10 groupable photos/videos across reply delivery, direct message actions, and outbound payload delivery, while preserving current single-item fallback for GIF/audio/document/media-note cases, caption overflow, buttons, reply/thread/silent/pin behavior, and non-Telegram channels. Use #21309/#51449/#51457 as prior art, then add focused Telegram tests and docs for the resulting `mediaUrls` or album semantics.
