## Summary

Keep #13620 open. Current main still lacks outbound Telegram album/media-group sending: Telegram reply delivery loops over media one item at a time, direct Telegram send accepts one mediaUrl, the outbound payload adapter sequences mediaUrls through repeated sendMessageTelegram calls, and tests/docs reflect sequential media behavior rather than sendMediaGroup albums. The prior implementation attempts linked in the issue discussion (#21309, #51449, #51457) are useful prior art but were closed unmerged in the provided GitHub context, so they do not supersede the open issue.

## Best Possible Solution

Keep #13620 open and implement outbound album support in the bundled Telegram plugin. The likely path is a sendMediaGroup flow for 2-10 groupable photos/videos across reply delivery, direct Telegram sends, and outbound payload delivery, with existing single-item behavior retained as fallback for one item, GIFs, audio, documents, caption overflow, buttons, reply/thread/silent/pin behavior, and non-Telegram channels. Add focused Telegram tests and user-facing docs for mediaUrls/mediaGroup/album semantics.
