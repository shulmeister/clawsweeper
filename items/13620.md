## Summary

Keep #13620 open. Current main still does not implement outbound Telegram media groups/albums: the Telegram reply path iterates media URLs one by one, the direct Telegram send/action surfaces accept only a single `mediaUrl`, the outbound adapter sequences `mediaUrls` through repeated single sends, tests lock in sequential behavior, and docs expose only single-media sending. The related album issues/PRs in the provided GitHub context are closed or unmerged, so there is no shipped or canonical successor that supersedes this issue.

## Best Possible Solution

Keep #13620 open and implement outbound album support in the bundled Telegram plugin. The likely product path is a `sendMediaGroup` implementation for 2-10 groupable photo/video items across reply delivery and outbound payload/direct message paths, with single-item and non-groupable fallbacks, first-item caption handling, caption overflow follow-up, inline-button follow-up behavior, and preservation of thread/reply/silent/pin behavior. Add focused Telegram tests and update `docs/channels/telegram.md` plus `docs/cli/message.md` if the user-facing message tool gains a multi-media parameter.
