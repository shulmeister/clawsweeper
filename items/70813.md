## Summary

Keep this PR open. Current main at 9626ef274ae2346b3d519f99c31856be98aab570 still does not implement WhatsApp edit or unsend actions: WhatsApp discovery advertises only react/poll, the channel action adapter supports only react, runtime rejects non-react WhatsApp actions, the active Web listener/send API has no edit/unsend methods, and WhatsApp docs list only react plus reaction/poll gates. The PR remains a plausible WhatsApp-plugin-owned implementation candidate because the shared message action contract already includes edit and unsend.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or revise a WhatsApp-plugin-owned implementation that advertises edit/unsend only when message sends are enabled, dispatches both actions through the existing gateway action surface, reuses outbound target authorization, constructs own-message mutation keys with fromMe: true and no caller-controlled participant, and adds focused unit/seam tests plus user-facing WhatsApp docs for the new actions and gates.
