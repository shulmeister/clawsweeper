## Summary

Keep #64752 open. Current main still receives Telegram `message_reaction` updates and records them as queued system events, but it does not trigger an immediate agent/heartbeat turn from a reaction-only update. The latest issue comment reports the same behavior on OpenClaw 2026.4.21, and the related #47677/#47687 work remains open rather than shipped.

## Best Possible Solution

Keep #64752 open. Implement the narrow Telegram-owned fix by adding a documented, backward-compatible reaction wake/trigger setting, then after an authorized reaction event is enqueued, request a session-scoped agent/heartbeat turn for the resolved route. Add regression tests proving a reaction-only update wakes the agent when configured and preserves the existing passive system-event behavior by default; leave richer reaction semantics/custom emoji handling to #47677/#47687 or land that broader design if maintainers approve it.
