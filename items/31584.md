## Summary

Keep open. Current main still has only partial runtime Copilot auth from environment variables; it does not implement the requested non-interactive onboarding/login flow or a Copilot token flag. The prior ClawSweeper comment matches the current state: Copilot setup remains device-login/TTY based, and the generic non-interactive onboarding path rejects provider methods without `runNonInteractive`.

## Best Possible Solution

Keep tracking this issue. The likely fix belongs in the bundled GitHub Copilot provider: add a plugin-owned non-interactive auth method that accepts a GitHub token via a manifest-backed CLI flag/env/ref flow, persists or references the auth profile consistently with other provider onboarding paths, and updates Copilot provider docs plus focused tests for `openclaw onboard --non-interactive --auth-choice github-copilot ...`.
