## Summary

Current main still does not implement non-interactive GitHub Copilot onboarding. The bundled GitHub Copilot provider is present, but its only auth method is an interactive device-code flow, non-TTY login is explicitly rejected, and the generic non-interactive provider path requires a provider-owned runNonInteractive implementation that Copilot does not expose. Runtime token discovery from COPILOT_GITHUB_TOKEN, GH_TOKEN, or GITHUB_TOKEN exists, but that is runtime auth discovery, not the requested openclaw onboard setup flow. Related Copilot issues #43658 and #46031 are adjacent auth-profile/runtime bugs and do not supersede this automation request.

## Best Possible Solution

Keep this issue open as bundled GitHub Copilot provider work. Add a provider-owned non-interactive auth method for github-copilot that accepts a GitHub token through manifest-backed CLI flag/env/ref metadata, persists the token via the auth-profile secret model, wires into openclaw onboard --non-interactive, applies an appropriate default model when requested, and updates focused provider/onboarding tests plus the public GitHub Copilot docs.
