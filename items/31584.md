## Summary

Keep open. Current main still has only partial GitHub-token runtime support for Copilot via environment variables/auth profiles; it does not provide the requested non-interactive onboarding/login path or a Copilot-specific onboarding flag. The prior ClawSweeper comment remains accurate against current main e23d17da79dc2345ca55902e632d3c7622accacc.

## Best Possible Solution

Keep tracking this issue. The likely fix belongs in the bundled GitHub Copilot provider: add a plugin-owned non-interactive auth method that accepts a GitHub token via a manifest-backed CLI flag/env/ref flow, persists or references the Copilot auth profile consistently with other provider onboarding paths, sets the expected default model when appropriate, and updates the GitHub Copilot provider docs plus focused non-interactive onboarding tests.
