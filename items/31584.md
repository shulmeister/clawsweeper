## Summary

Keep open. Current main still lacks a non-interactive GitHub Copilot onboarding/login path: Copilot only registers an interactive device-code auth method, direct login rejects non-TTY execution, and generic non-interactive onboarding errors unless the provider method implements runNonInteractive. The prior Codex review comment reached the same conclusion, and current main at c7a0d9b1889ca09ab5bfd84cda39a8d50d3d5768 still matches that behavior.

## Best Possible Solution

Keep tracking this as bundled GitHub Copilot provider work. The likely implementation is a plugin-owned non-interactive auth method or token method that accepts a GitHub token through manifest-backed CLI flag/env/ref handling, stores or references it as a github-copilot auth profile, applies the expected default model when requested, and adds focused non-interactive onboarding tests plus updates to the public GitHub Copilot provider docs.
