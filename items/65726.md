## Summary

Keep open. Current `main` still lacks any ACP binding/runtime `notifyPolicy` setting, ACP background tasks still default to visible `done_only` completion messages, and the current A2A/session-send logic explicitly keeps follow-up delivery enabled for unrelated sessions talking to an ACP target instead of deduplicating bound-thread replies.

## Evidence

- **ACP binding/runtime config has no notifyPolicy field:** `AgentRuntimeAcpConfig` only allows `agent`, `backend`, `mode`, and `cwd`, and `AgentAcpBinding.acp` only allows `mode`, `label`, `cwd`, and `backend`. There is no binding-level or runtime-level `notifyPolicy` option on current `main`.
