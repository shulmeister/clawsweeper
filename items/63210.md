## Summary

Keep open. Current main has related hardening for empty responses, malformed tool-call argument repair, and stop-reason propagation, but it does not implement the requested main-session `stopReason:"length"` recovery or tool-execution guard. The linked discussion also confirms this is separate from the subagent issue #63188 and only partially overlapped by #63689/#66146.

## Best Possible Solution

Keep #63210 open and implement a main embedded Pi runtime recovery path: preserve provider `length`, detect it before tool dispatch completion, refuse or mark unsafe truncated tool calls, inject a concise self-correction/continuation prompt with a bounded retry budget, and add regression tests for parsed-but-incomplete tool arguments plus text-only length truncation. Track #63188 separately for `mode:"run"` subagents and treat #66146/#63689 as partial mitigations, not canonical coverage of this request.
