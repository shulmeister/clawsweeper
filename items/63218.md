## Summary

Keep PR #63218 open. Current main still lacks the PR's core multi-instance Ollama routing change and the dedicated-agent fallback scoping change. The existing code can still register a single stream function per shared `api` value, Ollama streams are configured with a base URL at creation time, and Ollama is not included in the boundary-aware transport path. The PR discussion also contains unresolved bot feedback, so the best maintainer path is revision/review rather than cleanup closure.

## Best Possible Solution

Keep the PR open and revise it into a focused fix: add boundary-aware Ollama routing that resolves the active model/provider `baseUrl` at request time, add regression coverage for two custom Ollama providers sharing `api: "ollama"`, preserve dedicated child-agent fallback isolation with canonical model-ref comparison, and update the Ollama/failover/config docs once the behavior is final.
