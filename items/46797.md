## Summary

Current main still appears to emit the misleading config.patch restart sentinel wording. The config.patch path builds a restart sentinel with kind `config-patch`, status `ok`, and mode `config.patch`; the shared formatter renders that as `Gateway restart config-patch ok (config.patch)`, not `Gateway restart required` or equivalent.

## Evidence

- **Generic restart sentinel formatter still uses completed-sounding wording:** `summarizeRestartSentinel` returns `Gateway restart ${kind} ${status}${mode}` for all non-auto-recovery restart sentinels, so a config-patch/ok/config.patch payload formats as the reported `Gateway restart config-patch ok (config.patch)`.
  - file: [src/infra/restart-sentinel.ts:173](https://github.com/openclaw/openclaw/blob/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071/src/infra/restart-sentinel.ts#L173)
  - command: `nl -ba src/infra/restart-sentinel.ts | sed -n '130,190p'`
  - sha: [28497515fe1e](https://github.com/openclaw/openclaw/commit/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071)
- **config.patch still creates that payload:** The config.patch handler calls `buildConfigRestartSentinelPayload` with `kind: "config-patch"` and `mode: "config.patch"`; the builder hard-codes `status: "ok"` and stores the mode in `stats`, producing the formatter inputs that match the issue report.
  - file: [src/gateway/server-methods/config.ts:583](https://github.com/openclaw/openclaw/blob/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071/src/gateway/server-methods/config.ts#L583)

- **Tests still assert the old message shape:** Restart sentinel tests still expect strings like `Gateway restart config-patch error (patch)` and `Gateway restart restart ok`, with no test coverage for a `restart required` notice.
