## Summary

Keep #29100 open. Current main still does not support a user-configurable cost metadata overlay for auto-discovered Amazon Bedrock models. The Bedrock plugin discovery schema and TS/zod config types omit any `costs` map, discovery assigns zero `DEFAULT_COST`, inference profiles only inherit that zero/default cost, and the public docs still document pricing through explicit `models.providers.<provider>.models[].cost` entries. The related PR #31160 was closed unmerged in the provided GitHub context, so it does not supersede the issue.

## Best Possible Solution

Keep #29100 open. The best path is a backward-compatible pricing overlay for discovered Bedrock models, preferably owned by the Amazon Bedrock plugin unless maintainers choose a general discovered-model cost overlay. It should update the plugin manifest schema, TS/zod config types or plugin config typing, UI/help metadata, deterministic discovery cache key behavior, foundation-model and inference-profile cost resolution, focused discovery tests, and docs for the public Bedrock and token-use pages. It should also verify how `/status`, `/usage full`, `/usage cost`, diagnostics, and transcript-backed summaries display cost for Bedrock `aws-sdk`/IAM auth.

## Risks / Open Questions

- A general gateway pricing cache exists for selected configured model refs, but it is not a user-configurable Bedrock discovery metadata overlay and does not satisfy the requested config contract.
- The remaining product choice is whether to implement this as plugin-owned `plugins.entries.amazon-bedrock.config.discovery.costs` or as a broader `models.costs` overlay for discovered models.
