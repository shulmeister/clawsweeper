## Summary

Keep #29100 open. Current main still does not support a pricing/cost metadata overlay for auto-discovered Amazon Bedrock models. Bedrock discovery accepts only discovery controls, discovered foundation models are assigned `DEFAULT_COST`, inference profiles only inherit that base cost, and the docs still describe cost estimation as coming from explicit `models.providers.<provider>.models[].cost` entries. The related PR #31160 attempted exactly this feature but was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open until OpenClaw adds a supported cost overlay for Bedrock auto-discovery, ideally in the plugin-owned discovery config or a deliberate general model-pricing overlay. The implementation should update config types/schema/manifest UI hints, apply pricing during discovery for both foundation models and inference profiles, include pricing in cache-key semantics, add focused Bedrock discovery tests, and update the public Bedrock/token-use docs.
