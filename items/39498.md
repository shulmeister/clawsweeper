## Summary

Keep #39498 open. The underlying ByteString crash class is real on current main, and Comfy Cloud still has a header-bound API-key path that uses trim-only SecretInput normalization before constructing Headers. But the PR's proposed global change to normalizeSecretInputString is too broad: that helper is also used by non-header credentials, including Matrix passwords sent in JSON bodies, where stripping non-Latin1 characters would silently corrupt valid secrets.

## Best Possible Solution

Keep the PR open but request a scoped fix. Preserve trim-only semantics for normalizeSecretInputString, hasConfiguredSecretInput, and non-header password/body credentials; apply normalizeSecretInput or normalizeOptionalSecretInput only at header-bound API-key/token call sites such as Comfy Cloud X-API-Key; add regression coverage for affected plugin/provider header paths and a guard showing Matrix-style password/body credentials keep non-Latin1 characters.
