## Summary

Keep #39498 open. Current main still has at least one header-bound pasted-Unicode crash path, but the PR's global `SecretInput` normalizer change is too broad and can corrupt valid non-header secrets such as Matrix passwords.

## Best Possible Solution

Keep the PR open but request a scoped fix. Preserve trim-only semantics for generic `SecretInput`, `hasConfiguredSecretInput`, and non-header password/body credentials, then apply `normalizeSecretInput` or a configurable resolver normalizer only at header-bound secret callsites such as Comfy Cloud `X-API-Key`, with regression coverage for those affected plugin/provider header paths.
