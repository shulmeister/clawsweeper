## Summary

Keep open. Current main still derives `lastMessagePreview` from the newest user/assistant transcript entry without filtering `type: "diagnostic.heartbeat"`, and the existing session preview tests do not cover heartbeat diagnostic tail entries. The related #66544 PR addresses a different display-name fallback path, so it does not supersede this preview-specific fix.

## Best Possible Solution

Keep this PR open or fold it into a single maintainer-approved fix with #66544. The best code direction is to protect all session-list preview/title consumers from internal heartbeat/diagnostic transcript entries, likely by filtering `diagnostic.*` or known heartbeat sentinel entries in the transcript preview helper and adding the regression test from this PR.
