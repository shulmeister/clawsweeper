## Summary

Keep #69132 open. Current main no longer calls `/api/experimental/web_search`, but it still makes a single request to the configured/default Ollama host at `/api/web_search` and throws on a 404. The reporter and linked discussion explicitly observed that both local `/api/experimental/web_search` and `/api/web_search` return 404 on Ollama 0.16+/0.17.x, and the broader cloud fallback/token-scoping fix remains represented by open PR #69222 rather than shipped on main.

## Best Possible Solution

Keep the issue open until maintainers land #69222 or an equivalent fix. The likely product/code path is to preserve `/api/web_search`, add tested 404 compatibility behavior for local Ollama hosts, retry hosted `https://ollama.com/api/web_search` only when an env-sourced `OLLAMA_API_KEY` is available, keep local/cloud bearer-token scoping explicit, and update the Ollama Web Search docs to match the chosen contract.
