## Summary

Keep PR #72177 open. It is a recent contributor PR for a real LiteLLM/image-generation surface, and current main still has the pre-hardening behavior: LiteLLM edits are JSON data URLs through `postJsonRequest`, no LiteLLM image timeout default is applied, the agent tool still accepts model-supplied `timeoutMs`, and there is no chat-native LiteLLM image routing or response-image dedupe on main.

## Best Possible Solution

Keep #72177 open for maintainer review. The best path is to land or split the LiteLLM hardening after resolving the active review comments, with tests for multipart edits, 180s LiteLLM defaults, model-supplied timeout suppression, chat-native LiteLLM image routing, dedupe, and parsing limits; update the image-generation docs/config help if per-call `timeoutMs` is removed from the image tool.
