## Summary

Current main already fixes the observable stuck-spinner/no-active-run failure. The TUI still sends with `deliverDefault` and enters `waiting`, but Gateway `chat.send` now registers the client run before acknowledging it, maps lifecycle events back to that run, and broadcasts final/error chat frames that let the TUI clear its busy state. The same implementation is present in the latest release tag `v2026.4.24`. The separate default/config ergonomics remain tracked by #33102.

## Best Possible Solution

Close #66991 as implemented on main and in `v2026.4.24`; keep the shipped gateway run-registry/lifecycle fix, leave #33102 as the canonical follow-up for default/config ergonomics, and do not revive the closed unmerged #67017 default-flip PR for this bug.
