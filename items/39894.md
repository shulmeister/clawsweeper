## Summary

Keep PR #39894 open. Current main still builds the outbound payload plan by parsing raw reply text, so echoed inbound metadata can remain in user-visible outbound text while MEDIA directives are extracted. The linked canonical bug #39847 remains open, and the PR contains a relevant narrow fix plus regression coverage, though its unrelated CI/macOS/test-runner changes should be split before landing.

## Best Possible Solution

Keep PR #39894 or canonical issue #39847 open until main lands a narrow core fix in src/infra/outbound/payloads.ts: strip copied inbound metadata before parseReplyDirectives() inside createOutboundPayloadPlanEntry(), add a regression in the outbound payload tests for Conversation info/Sender metadata followed by visible text plus MEDIA, and split the unrelated CI/macOS/test-runner edits into separate work if still needed.
