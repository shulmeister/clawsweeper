## Summary

Keep open. Current `main` still lacks the selective heartbeat-drain implementation this PR proposes, and the PR discussion shows at least one acknowledged changed-path edge case still unresolved rather than a landed/obsolete fix.

## Evidence

- **Current main still drains all system events on every reply run:** `drainFormattedSystemEvents` unconditionally calls `drainSystemEventEntries(params.sessionKey)` with no heartbeat-mode branching, so queued system events are fully drained regardless of whether the run is a periodic heartbeat or a normal turn.

That means the PR's core mechanism is not already implemented on `main`.
