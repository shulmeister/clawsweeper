## Summary

Current main still appears to have the reported Control UI streaming layout bug. The active turn can still contain multiple `kind: "stream"` items, and each such item is rendered as its own full assistant `chat-group`. The linked fix PR was closed unmerged, and the current code does not contain the proposed `stream-run` style grouping.

## Best Possible Solution

Keep this issue open and fix it in the Control UI rendering path. The likely product direction is to build one in-flight assistant item/group for a run, interleaving prior streamed segments, visible tool cards, and the live tail inside that single assistant column; when tool calls are hidden, merge segment text and live text into one updating bubble. Add focused coverage in `ui/src/ui/chat/build-chat-items.test.ts` or an unmocked rendering test that asserts segmented streaming produces one outer `.chat-group.assistant`. The closed PR #63986 and the still-relevant duplicate-bubble PR context are useful implementation references, but current main has not shipped the fix.
