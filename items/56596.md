## Summary

Current `main` contains targeted fixes for the reported post-update no-op behavior: Google Gemini base URLs are normalized back to the required `/v1beta` form, custom OpenAI `api.openai.com/v1` providers are normalized onto the correct transport, and invalid configs now fail closed with a `doctor --fix` prompt instead of letting chat continue in a broken state.

## Evidence

- **Gemini base URL normalization added on main:** `normalizeGoogleApiBaseUrl()` now rewrites bare `https://generativelanguage.googleapis.com` origins to `https://generativelanguage.googleapis.com/v1beta`, matching the workaround comment's root-cause hypothesis about broken Google base URLs after update.

  - sha: [fc5920fb5134](https://github.com/openclaw/openclaw/commit/fc5920fb51346a682b269fdbeeec0971ae4ae723)
- **Regression coverage for bare Gemini host:** Unit coverage explicitly asserts that both `https://generativelanguage.googleapis.com` and the slash-suffixed variant normalize to the v1beta endpoint.
  - file: [src/infra/google-api-base-url.test.ts:9](https://github.com/openclaw/openclaw/blob/fc5920fb51346a682b269fdbeeec0971ae4ae723/src/infra/google-api-base-url.test.ts#L9)
  - command: `nl -ba src/infra/google-api-base-url.test.ts | sed -n '1,80p'`

  - file: [src/agents/pi-embedded-runner/model.test.ts:167](https://github.com/openclaw/openclaw/blob/fc5920fb51346a682b269fdbeeec0971ae4ae723/src/agents/pi-embedded-runner/model.test.ts#L167)
