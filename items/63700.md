## Summary

Keep open. Current main still does not implement repeatable `--file` support on `openclaw infer model run`: the command is registered with only prompt/model/transport/json flags, and both local and gateway execution pass only the prompt message with no image/audio attachment payload. The dedicated `infer image describe` path has improved since the report and now uses media understanding providers, but it is still a separate command, processes `describe-many` files independently, and hardcodes the prompt for explicit model overrides. Audio support is currently an `audio transcribe` capability, not a general multimodal model-run attachment path. The cross-referenced #69030 is related to `model run` semantics, but it tracks raw mode/context bypass, not multimodal file handling.

## Best Possible Solution

Keep this issue open and implement the missing core CLI capability. A good path is to add repeatable `--file` handling to `src/cli/capability-cli.ts`, validate and MIME-detect supported media, pass image attachments through the existing local `agentCommand` image path and gateway `attachments` protocol, and decide explicitly whether audio should become model content or remain a transcription-only capability until provider contracts support multimodal audio. Add CLI tests for single image, multiple images, unsupported/providerless cases, and docs under `docs/cli/infer.md`. Coordinate the UX with #69030 because raw mode and multimodal attachments both affect what users expect from `infer model run`.
