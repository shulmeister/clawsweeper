## Summary

Keep PR #39112 open. Current main still persists native Ollama `/api/chat` responses with omitted `prompt_eval_count` / `eval_count` as zero usage, and the dashboard aggregation path consumes those persisted transcript usage totals without adding a later estimate. The PR is still solving a real gap, but it needs to be ported from the old `src/agents/ollama-stream.*` layout to the current bundled Ollama plugin files.

## Best Possible Solution

Keep this PR open or land an equivalent current-main fix in `extensions/ollama/src/stream.ts`: estimate native Ollama input/output usage only when counters are absent, preserve exact nonnegative counters including explicit zero, include message text, image payloads, tool calls, tool results, and tool schemas in prompt estimates, and add focused regression coverage in the current Ollama plugin tests for missing counters, explicit zero counters, streamed responses, and multimodal/tool payload estimates.
