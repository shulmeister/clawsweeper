## Summary

Keep PR #39112 open. Current main still zero-fills native Ollama `/api/chat` usage when `prompt_eval_count` / `eval_count` are omitted, and no downstream dashboard aggregation path backfills a character-based estimate. The PR still addresses a real unimplemented behavior, though it needs rebase/adaptation from the older `src/agents/ollama-stream.*` path to the current `extensions/ollama` plugin path.

## Best Possible Solution

Keep this PR open or have a maintainer land an equivalent current-main fix in `extensions/ollama/src/stream.ts`: estimate native Ollama input/output usage only when counters are absent, preserve exact nonnegative counters including explicit zero, include images/tools in prompt estimates, and add focused regression coverage in the Ollama stream tests. If the existing PR is used, rebase it from the old `src/agents/ollama-stream.*` layout onto the current bundled Ollama plugin files.
