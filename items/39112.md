## Summary

Keep PR #39112 open. Current main still writes native Ollama `/api/chat` responses with missing `prompt_eval_count` / `eval_count` as zero usage, and the dashboard aggregation path consumes those transcript usage values without adding a later character-based estimate. The PR addresses a real remaining gap, though it needs rebase/porting from the old `src/agents/ollama-stream.*` layout to the current Ollama plugin files.

## Best Possible Solution

Keep this PR open or land an equivalent current-main fix in `extensions/ollama/src/stream.ts`: estimate native Ollama input/output usage only when counters are absent, preserve exact nonnegative counters including explicit zero, include image/tool payloads in prompt estimates, and add focused regression tests under `extensions/ollama/src/stream-runtime.test.ts` or `extensions/ollama/src/stream.test.ts` for missing counters, explicit zero counters, and multimodal/tool payload estimates.
