## Summary

Keep #70471 open. Current main still lacks the PR's central Feishu comment hardening: recipient gating, duplicate final-reply suppression after tool delivery, angle-bracket escaping, account-scoped typing reaction state, and force-fresh credential-aware bot identity refresh. The related #66427 is closed/unmerged and appears to be the older related attempt, not a canonical shipped replacement.

## Best Possible Solution

Keep the PR open and review it as the active focused fix for Feishu comment delivery hardening. Maintainers should either land a reviewed version of #70471 with its regression tests or split/port the five behavior fixes into smaller Feishu plugin PRs; #66427 should remain historical related context rather than the canonical replacement.
