## Summary

Keep open. Current main has a partial mitigation for tier-priced models by fetching/merging tiered pricing metadata and recomputing some estimated-cost views from tokens, but the gateway transport still writes assistant/transcript usage cost from `calculateCost(model, usage)` at stream time. The code explicitly acknowledges that the transcript may contain stale flat-rate cost and only fixes it later during aggregation, so the issue’s core claim about `message.usage.cost.total` in stored JSONL remains true.

## Risks / Open Questions

- Closing as implemented would hide that raw transcript consumers still receive the flat streamed cost in `message.usage.cost.total`.
- Current main appears to fix summary/estimation paths, but not the underlying stored-field correctness the issue reports.
- Any downstream tooling that trusts transcript `usage.cost.total` instead of recomputing from tokens+tiered pricing can still undercount OpenRouter spend.
