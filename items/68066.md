## Summary

Keep open. Current main still does not implement the issue's core request: there is no OpenRouter post-hoc `/generation?id=<responseId>` reconciliation and no persisted billed-cost field. The existing tiered-pricing cache helps gateway-side cost estimation, but the OpenAI-compatible stream path still records assistant usage from streamed token usage and local `calculateCost`, while transcript summaries later trust `message.usage.cost.total`.

## Best Possible Solution

Keep the issue open. The best product/code path is a narrow OpenRouter-specific billed-cost reconciliation in the OpenAI-compatible transport or an adjacent helper: after stream completion, use the captured `output.responseId` to fetch OpenRouter's generation record with the same authenticated base route, then persist the authoritative billed total by overwriting `message.usage.cost.total` or by adding a documented billed-cost field. Add mocked transport tests covering a streamed base-tier undercount plus an authoritative `/generation` total, and add transcript/usage-summary coverage if a new field is introduced.
