## Summary

Keep #16670 open. Current main still has memory-search configuration, docs, doctor diagnostics, and `openclaw memory status --deep`, but the setup/onboarding and configure flows still do not include the requested mandatory memory embedding setup step for provider, model, API key or SecretRef, and verification.

## Best Possible Solution

Keep #16670 open until OpenClaw adds a dedicated onboarding/configure memory setup path or a maintainer explicitly decides memory embedding setup should remain optional. The likely implementation belongs in setup/configure as a memory or embeddings section that prompts for provider, model, API key or SecretRef under `agents.defaults.memorySearch`, accounts for daemon/LaunchAgent credential visibility, verifies readiness with an embedding probe or `openclaw memory status --deep`, and updates the public onboarding/configure docs.
