## Summary

Keep #16670 open. Current main still has memory-search docs and doctor/readiness diagnostics, but neither `openclaw onboard` nor `openclaw configure` includes the requested mandatory memory/embedding setup step for provider, model, API key or SecretRef, and endpoint verification. The latest issue discussion also includes a maintainer/Codex review explicitly keeping this item open for the same remaining gap.

## Best Possible Solution

Keep #16670 open until OpenClaw adds a dedicated memory/embedding setup path or a maintainer explicitly decides against making it mandatory. The likely path is an onboarding/configure section that uses the memory embedding provider registry, writes `agents.defaults.memorySearch` provider/model/API-key or SecretRef settings, verifies readiness with a small embedding probe or `openclaw memory status --deep`, and updates user docs to clarify Codex OAuth vs OpenAI API keys plus daemon/LaunchAgent environment visibility.

## Risks / Open Questions

- Existing automatic detection covers some users with already-visible credentials, but it does not satisfy the issue's mandatory onboarding/API-key/verification request.
- Making the step mandatory is a product decision, especially for local-only or keyless setups, so the issue should remain open for maintainer design rather than conservative closure.
