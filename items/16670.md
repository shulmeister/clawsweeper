## Summary

Keep open. Current main still has memory-search docs, embedding provider auto-detection, and doctor readiness warnings, but the requested mandatory onboarding/setup step for memorySearch provider/model/API-key configuration and endpoint verification is still missing. The latest maintainer-thread context also explicitly kept this item open after the same gap was reviewed.

## Best Possible Solution

Keep #16670 open until OpenClaw either adds a dedicated memory/embedding setup path or a maintainer explicitly decides against making it mandatory. The strongest implementation would add a memory section to onboarding and configure, reuse the memory embedding provider registry, collect provider/model/API-key or SecretRef settings for agents.defaults.memorySearch, verify the selected embedding endpoint with a small probe or memory status check, and update docs to distinguish OpenAI API keys from Codex OAuth and daemon/LaunchAgent environment visibility.
