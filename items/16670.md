## Summary

Keep #16670 open. Current main still documents and diagnoses memory-search embedding setup, but neither `openclaw onboard` nor `openclaw configure` has the requested mandatory memory/embedding provider, model, API-key, and endpoint verification step. The latest issue discussion also already contains a maintainer/Codex review keeping this item open for the same remaining gap.

## Best Possible Solution

Keep #16670 open until OpenClaw either adds a dedicated memory/embedding setup path or a maintainer explicitly decides against making it mandatory. The likely implementation belongs in onboarding/configure, should reuse the memory embedding provider registry, collect provider/model/API-key or SecretRef settings for `agents.defaults.memorySearch`, verify readiness with a small embedding probe or `memory status --deep` path, and update user docs to clarify Codex OAuth versus OpenAI API keys and daemon/LaunchAgent environment visibility.
