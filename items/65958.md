## Summary

Keep open. Current main has adjacent hot-refresh support for skill file/config changes, but the reported GitHub CLI dependency path is still not clearly handled: the bundled GitHub skill only tells users to run `gh auth login`/`gh auth status`, the Skills UI reports a generic `Installed` success, and the documented snapshot refresh triggers do not include newly installed local binaries such as `gh` appearing on PATH during an existing session.

## Best Possible Solution

Keep this issue open and fix it in the skills dependency UX/docs path: add explicit post-install guidance for binary-backed skills such as GitHub CLI, likely in `skills.install` completion messages, the Skills UI callout, and `docs/tools/skills.md` or `docs/cli/skills.md`. The product decision should also clarify whether users need a new session, a skill snapshot invalidation, or `openclaw gateway restart` after external CLI installation/authentication, then cover that behavior with a targeted test.
