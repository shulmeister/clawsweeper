## Summary

Keep #70443 open. Current main does not implement `--offline` or an equivalent Docker setup path: `scripts/docker/setup.sh` still builds `openclaw:local`, pulls any non-default `OPENCLAW_IMAGE`, and builds the sandbox image when sandbox setup is enabled. The Docker docs also describe only local build or registry image use, not a preloaded airgapped rerun. The related platform-build items #69143/#69314 do not supersede this separate offline rerun request.

## Best Possible Solution

Implement a narrow Bash 3.2-compatible `--offline` flag in `scripts/docker/setup.sh`: parse it early, skip only the main build/pull and sandbox build steps, require `docker image inspect "$IMAGE_NAME"` for the main image with a clear `docker load` message on failure, warn rather than fail for missing `openclaw-sandbox:bookworm-slim`, and leave env validation, `.env` upserts, permission fix-up, onboarding, gateway config sync, compose startup, and sandbox overlay handling unchanged. Add focused coverage in `src/docker-setup.e2e.test.ts` and document the workflow in `docs/install/docker.md`.
