## Summary

Keep open. Current main still has the reported provider/model-specific gap: Qwen/legacy ModelStudio exposes the affected Alibaba-hosted models through the generic OpenAI-compatible transport, and that path only creates tool-call blocks from structured `delta.tool_calls` while preserving `reasoning_content`/reasoning fields as thinking. The existing Kimi tagged-tool parser is scoped to the separate Kimi provider and only rewrites standalone text blocks, and the MiniMax mitigation is scoped to native MiniMax Anthropic-message routes, not `qwen/MiniMax-M2.5`.

## Best Possible Solution

Keep this issue open and fix it in provider-owned stream normalization. The best path is a Qwen/ModelStudio OpenAI-compatible wrapper or shared OpenAI-compatible seam that detects validated tagged Kimi-style tool-call sections in reasoning/thinking content, converts them to structured tool-call blocks, and changes `stopReason` only after successful extraction. Add focused regression tests for `qwen/kimi-k2.5`, `qwen/MiniMax-M2.5`, and the existing Kimi wrapper so text-block and thinking-block behavior are explicit.
