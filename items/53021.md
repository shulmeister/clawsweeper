## Summary

Keep open. Current `main` still computes LAN Control UI display URLs for several user-facing surfaces when `gateway.bind="lan"`, while the default/non-loopback Control UI origin allowlist still seeds only `localhost` and `127.0.0.1`. That means the PR's reported mismatch is not implemented on `main` and remains plausible. The PR branch also bundles unrelated agent-error and skill changes, so it is not a conservative cleanup close candidate.
