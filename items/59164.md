## Summary

Current `main` still appears to miss `subagent_ended` lifecycle wiring for plugin SDK runs started via `api.runtime.subagent.run()`. The gateway-backed plugin runtime dispatches the `agent` method, but the `agent` handler path does not register a `SubagentRunRecord`, while the hook emission path still depends on that registry entry.

## Evidence

- **Plugin SDK runtime exposes background subagent runs:** The public plugin runtime docs still present `api.runtime.subagent.run(...)` as the way to start background subagent runs.
  - file: [docs/plugins/sdk-runtime.md:87](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/docs/plugins/sdk-runtime.md#L87)
