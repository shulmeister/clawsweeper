## Summary

Keep open. Current main still matches the core report: normal TUI submit clears the editor immediately, then sends chat input without an active-run guard. Escape remains the separate abort path, and the provided related PR #45586 that proposed the missing guard was closed unmerged. Adjacent fixes for optimistic/history reconciliation do not close this active-generation submit/draft-preservation bug.
