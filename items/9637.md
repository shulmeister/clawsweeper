## Summary

Current main still does not implement a TUI plain-text accessibility option. The config contract has no `ui.accessibility.plainText` or equivalent key, the TUI/tool-display paths still render emoji and Unicode symbols unconditionally, and the related PR #9685 is closed unmerged per the provided GitHub context.

## Best Possible Solution

Keep this issue open. The best path is to add a real `ui.accessibility.plainText` or equivalent config contract in config types, zod schema, generated schema/help/docs, then thread that resolved option through the TUI/tool-display paths so tool summaries, tool execution headers, selectors, separators, cursors, and tests can switch to ASCII/plain text. PR #9685 should be treated as useful prior-art context, but it needs to be refreshed against current main and the current config/TUI architecture.
