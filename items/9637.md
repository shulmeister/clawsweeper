## Summary

Current main still does not implement a TUI plain-text/accessibility mode. The `ui` config accepts only `seamColor` and `assistant`, generated help/docs omit any `ui.accessibility.plainText` key, and TUI/tool rendering still emits emoji and Unicode glyphs unconditionally. Related PR #9685 was closed unmerged, and the latest maintainer comment already explicitly kept #9637 open as actionable core TUI/config work.

## Best Possible Solution

Keep #9637 open and implement it as core TUI/config work. Add a documented `ui.accessibility.plainText` or equivalent option to `OpenClawConfig`, zod validation, generated schema/help, and public docs; load it once during TUI startup; thread it through tool display, ChatLog/tool cards, selectors/themes, waiting/status rendering, and settings surfaces; provide ASCII alternatives for emojis and Unicode glyphs; and add focused config validation plus TUI rendering tests. Treat closed PR #9685 as useful prior art, not completed work.
