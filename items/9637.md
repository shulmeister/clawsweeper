## Summary

Current main still lacks a TUI plain-text/accessibility config option. The request remains actionable core TUI/config work; the related PR #9685 is useful prior art but was closed unmerged and does not supersede the issue.

## Best Possible Solution

Keep #9637 open and implement it as core TUI/config work. Add a documented ui.accessibility.plainText or equivalent config option to OpenClawConfig, zod schema, generated schema/help, and public docs; load it during TUI startup; thread it through chat log, tool cards, selectors, settings/theme rendering, and tool summary formatting; provide ASCII alternatives for emoji and Unicode glyphs; and add focused config validation plus rendering tests. Treat closed PR #9685 as prior art, not as a completed fix.
