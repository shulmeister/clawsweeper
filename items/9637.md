## Summary

Current main still lacks a TUI plain-text accessibility option. The config contract has no `ui.accessibility.plainText` or equivalent, the strict zod schema would reject that key, TUI/tool-display paths still render emoji and Unicode symbols unconditionally, and the linked implementation PR #9685 is closed unmerged.

## Best Possible Solution

Keep this issue open. The best path is to add a real `ui.accessibility.plainText` or equivalent config contract across `src/config/types.openclaw.ts`, `src/config/zod-schema.ts`, generated schema/help/labels, and docs, then load/thread that resolved option through TUI startup, `ChatLog`, `ToolExecutionComponent`, selector themes/components, and `resolveToolDisplay`/summary formatting. Use #9685 as prior-art context, but refresh it against current config/TUI architecture and add focused tests for config validation plus ASCII/plain-text rendering.
