## Summary

Keep PR #40744 open. Current main still appears to preserve malformed `required: null` tool-schema values on the generic/OpenAI-compatible path, and the PR's central bug remains useful even though the exact patch needs rebasing onto the newer `normalizeToolParameterSchema` structure.

## Best Possible Solution

Keep the PR open and ask for a rebase/adaptation onto current main. The best fix is to strip own non-array `required` fields in the shared tool-parameter normalization path before provider-specific cleaning, preserve valid `required` arrays, and add regression tests for the Bailian/OpenAI-compatible top-level path plus the nested Gemini path. Treat the Aisle prototype-pollution note as a hardening follow-up or include a narrow safe-object/key-filter change if the Gemini loop is touched.
