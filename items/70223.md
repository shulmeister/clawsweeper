## Summary

Keep open. Current main still does not provide a WorkingDirectory for packaged gateway service installs: the launchd plist can emit it, but the normal non-dev program-args paths return only programArguments, and the install plan passes that undefined value through. Current docs/source no longer show a fresh default relative ./agents/main config, but an existing configured relative agentDir still resolves through path.resolve against process.cwd(), so the reported launchd cwd=/ failure class is not fully fixed.

## Best Possible Solution

Keep the issue open and fix both sides conservatively: make packaged gateway install/update always supply a stable WorkingDirectory for launchd, then add a focused regression test proving the rendered LaunchAgent includes it. For defense in depth, either migrate/doctor relative agents.list[].agentDir values or resolve them relative to the loaded openclaw.json directory, with tests covering cwd=/ and legacy relative config.
