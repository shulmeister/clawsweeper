## Summary

Keep open. Current main appears to route recovery through the shared outbound delivery/chunking path, but the requested permanent-error fallback is still missing: `isPermanentDeliveryError()` does not recognize Telegram `message is too long`, so that failure would still be retried/backed off instead of moved to `failed/` immediately.
