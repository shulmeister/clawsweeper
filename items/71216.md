## Summary

Keep this issue open. Current main still does not provide the requested first-class config/runtime surface: no root `sandbox`, no `routing.rules`, no `instances` config, and no `gateway.nodes.denyPaths`. The existing ClawSweeper keep-open comment remains consistent with current main at 9626ef274ae2346b3d519f99c31856be98aab570; the relevant changes since that comment are unrelated compaction/provider-timeout work.

## Best Possible Solution

Keep this issue open and split the work deliberately: decide whether root sandbox policy should alias or wrap `agents.defaults.sandbox`; design tag-aware provider/model routing in the agent/model-selection layer with explicit metadata propagation; decide whether multi-instance declarations belong in config or in the existing profile/gateway-management model; and implement `gateway.nodes.denyPaths` only with schema/types/docs plus runtime path-enforcement, doctor, and security tests.

## Risks / Open Questions

- The issue bundles four separable product/API surfaces; implementing them as one schema-only change would be hard to review safely.
- `gateway.nodes.denyPaths` would need actual runtime enforcement across node/tool/file access chokepoints, not just schema/types/docs.
- `routing.rules` and `instances.*` need maintainer design because they affect model-selection metadata, gateway/profile management, and trust boundaries.
