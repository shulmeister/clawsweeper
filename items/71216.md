## Summary

Keep #71216 open. Current main has adjacent sandbox and multi-gateway configuration, but it does not implement the requested `routing.rules`, `instances`, or `gateway.nodes.denyPaths` schema/enforcement, and the sandbox support lives under `agents.defaults.sandbox` / `agents.list[].sandbox` rather than a root `sandbox.mode` key. This is a recent, concrete core config/security design request, not a cleanup-close candidate.

## Best Possible Solution

Keep the issue open for maintainer design. The best path is to acknowledge that `agents.defaults.sandbox.mode` already covers part of the sandbox ask, then split or refine the remaining work: decide whether root `sandbox.mode` is a compat alias or not needed, design `routing.rules` against model resolution/plugin hook semantics, decide whether multi-instance validation belongs in config or profile/service tooling, and implement any accepted `gateway.nodes.denyPaths` with schema, docs, doctor/audit coverage, and runtime enforcement across node and host filesystem/tool chokepoints.
