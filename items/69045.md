## Summary

Third-party Zoho channel planning belongs in the ClawHub/community-plugin path. Current `main` already documents the supported external channel-plugin contract, recommended file structure/imports, and install/discovery flow, so there is no OpenClaw core change to track in this issue.

## Evidence

- **ClawHub is the canonical discovery surface for community plugins:** The community-plugins docs say third-party plugins are published on ClawHub or npm, installable with `openclaw plugins install <package-name>`, and explicitly call ClawHub the canonical discovery surface for community plugins.
  - file: [docs/plugins/community.md:14](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/plugins/community.md#L14)

  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Vision keeps plugin promotion/discovery out of core:** `VISION.md` says plugins should be hosted and maintained in their own repositories, that optional capability should usually ship as plugins, and that plugin discovery/promotion belongs in ClawHub rather than OpenClaw core.
  - file: [VISION.md:68](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/VISION.md#L68)
  - command: `nl -ba VISION.md | sed -n '56,76p'`

- **Current docs already define the third-party plugin publishing path:** The getting-started plugin docs say you do not need to add a plugin to the OpenClaw repository; publish to ClawHub or npm and install with `openclaw plugins install`. They also show the external-plugin publish/install path.
  - file: [docs/plugins/building-plugins.md:16](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/plugins/building-plugins.md#L16)

- **Canonical channel entry helper is already documented:** The SDK docs say new channel plugins should import `defineChannelPluginEntry` from `openclaw/plugin-sdk/channel-core`; `openclaw/plugin-sdk/core` is kept as a broader umbrella surface. The entry-point reference also describes `defineChannelPluginEntry` as the channel-specific wrapper that auto-registers the channel capability.
  - file: [docs/plugins/sdk-overview.md:25](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/plugins/sdk-overview.md#L25)

This does not match the issue's expectation that a raw `npm install -g` / `node_modules` placement is the discovery contract.
