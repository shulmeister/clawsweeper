## Summary

Keep open. Current `main` still treats the canonical internal model key as `provider/id`, documents allowlist/config keys as full `provider/model` IDs, uses `name` as display metadata rather than identity, and keeps `reasoning` boolean-only. The proposed `provider/name` decoupling is not implemented.
