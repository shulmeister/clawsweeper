## Summary

Keep open. Current main still does not implement a per-agent model allowlist. OpenClaw has a global model catalog/allowlist at `agents.defaults.models`, and per-agent model config supports primary/fallback selection, but there is no agent-scoped allowlist field or enforcement path. Related #9010 was closed as a duplicate of #7424, so #7424 remains the canonical tracker.

## Best Possible Solution

Keep #7424 open until maintainers implement or explicitly reject an agent-scoped model allowlist. A likely path is to add a config shape such as `agents.list[].models` or `agents.list[].model.allowlist`, update TS types, zod schema, generated config docs/labels, CLI and Control UI surfaces, then thread enforcement through `buildAllowedModelSet`, `/model` list/set, session override validation, and tests for agent-specific visibility and rejection.

## Risks / Open Questions

- Per-agent fallback overrides are partially related and are included in the allowed set, but they do not hide or reject other globally allowlisted models for that agent.
- A future implementation needs product judgment on whether an agent allowlist intersects with or overrides the global catalog, and how `/model`, `/models`, session overrides, and Control UI pickers should present that policy.
