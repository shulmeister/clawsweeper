## Summary

Keep #7424 open. Current main still has only a global model catalog/allowlist under `agents.defaults.models`; per-agent config supports `model` primary/fallbacks but has no agent-scoped allowlist field, schema support, model-list filtering, or `/model`/session override enforcement. Related #9010 was closed as a duplicate of #7424, so this remains the canonical tracker.

## Best Possible Solution

Keep #7424 open as the canonical feature request. Implement an agent-scoped model allowlist in core config, likely under `agents.list[].model.allowlist` or `agents.list[].models`, then enforce it across `buildAllowedModelSet`, `models.list`, `/model`, session patch overrides, OpenAI-compatible model overrides, docs, generated config metadata, and regression tests for per-agent visibility and rejection.

## Risks / Open Questions

- There are adjacent features for per-agent primary/fallback model defaults and plugin subagent `allowedModels`, but they do not solve ordinary agent-scoped `/models` visibility or `/model` rejection for this issue.
- A proper implementation needs product judgment on whether agent allowlists intersect with or override `agents.defaults.models`, and then must update config types, zod schema, docs/generated labels, gateway model listing, session override validation, slash commands, and Control UI pickers.
