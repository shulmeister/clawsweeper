## Summary

Keep #7424 open. Current main still has only a global model catalog/allowlist under `agents.defaults.models`; ordinary per-agent config supports `model` primary/fallback defaults but has no agent-scoped model allowlist field, schema support, model-list filtering, or complete `/model`/session override enforcement for a per-agent policy. Related #9010 was closed as a duplicate of #7424, so this remains the canonical tracker.

## Best Possible Solution

Keep #7424 open as the canonical feature request. Implement an agent-scoped model allowlist in core config, likely under `agents.list[].model.allowlist` or `agents.list[].models`, and enforce it consistently across allowed-model resolution, `models.list`, `/model`, `sessions.patch`, OpenAI-compatible model overrides, current-session model status, Control UI model pickers, docs, generated config metadata, and regression tests for per-agent visibility and rejection.

## Risks / Open Questions

- Adjacent features such as per-agent primary/fallback model defaults and plugin subagent `allowedModels` do not satisfy ordinary agent-scoped `/models` visibility or `/model` rejection for routed family/shared agents.
- A real implementation needs product judgment on whether per-agent allowlists intersect with, replace, or inherit from `agents.defaults.models`, then must update config schema, generated metadata, Gateway RPCs, chat slash commands, session patches, OpenAI-compatible overrides, Control UI, docs, and tests.
