## Summary

Keep #33838 open. Current main still supports node command declaration as command-name strings only; it does not provide a node-declared input/output schema mechanism for gateway validation, routing metadata, or API discovery. The existing ClawSweeper comment is still consistent with current main at a35d259719cc7aecd1854fdc830ad6390b15f02a, and the related title-search items do not supersede this protocol/API gap.

## Best Possible Solution

Keep #33838 open for maintainer/product design. The likely implementation is a backwards-compatible gateway node protocol/API change: add command metadata/schema declarations to the connect and/or node-pairing surfaces, persist approved schemas with the node surface, expose them through node discovery/describe APIs, and validate `node.invoke` params/results where appropriate. Because the missing piece is gateway understanding of node command contracts, this should not be closed as ClawHub-only plugin work unless a separate maintainer decision first creates a plugin/API schema declaration surface.
