## Summary

Keep #33838 open. Current main still treats node command declarations as command-name strings only; it does not provide a node-declared input/output schema mechanism for gateway validation, routing metadata, or API discovery. The existing ClawSweeper keep-open comment remains accurate for current main at e23d17da79dc2345ca55902e632d3c7622accacc, and the related title-search items do not supersede this protocol/API gap.

## Best Possible Solution

Keep #33838 open for maintainer/product design. The likely path is an additive gateway protocol/API change: add per-command metadata with inputSchema/outputSchema to node connect and/or node.pair.request, persist approved schema metadata with the node surface, expose it through node.list/node.describe or a dedicated discovery method, and validate node.invoke params/results where appropriate while preserving existing string-only command declarations for compatibility.
