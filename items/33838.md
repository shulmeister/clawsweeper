## Summary

Keep #33838 open. Current main still has no node-declared input/output schema mechanism: node commands are modeled as string allowlists through connect, pairing, storage, discovery, and invocation, and current validation only checks protocol shape, allowlist membership, and a few hard-coded sanitizers. The prior ClawSweeper keep-open comment remains accurate for current main c7a0d9b1889ca09ab5bfd84cda39a8d50d3d5768; the provided related items cover adjacent node-command work and do not supersede this protocol/API gap.

## Best Possible Solution

Keep #33838 open for an additive gateway protocol design. A good fix would preserve existing `commands: string[]` behavior while adding optional per-command metadata such as input/output JSON Schema, carrying it through connect/pairing approval/storage, exposing it via `node.list`/`node.describe` or a dedicated discovery method, and validating `node.invoke` params/results where maintainers decide it is safe and compatible.
