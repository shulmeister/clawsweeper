## Summary

Keep #33838 open. Current main still models node commands as command-name strings, not declared per-command input/output schemas. The gateway validates only the RPC envelope, persists and discovers only string command lists, and forwards invoke params/results as unknown payloads. The related ClawSweeper items cover different node file-transfer, rule-checking, and denyCommands work, so none supersedes this feature request.

## Best Possible Solution

Keep the issue open for a maintainer-designed additive gateway protocol/API change. A strong design would preserve commands: string[] compatibility while adding optional per-command metadata, carrying it through connect, node.pair.request/persistence, node registry/session state, node.list/node.describe discovery, generated clients, node-host/plugin runtime APIs, docs, and focused tests. Validation policy for node.invoke params/results should be explicit.
