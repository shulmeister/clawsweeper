## Summary

Keep PR #43730 open. Current main still documents Telegram reaction notifications without the Bot API admin requirement or the resulting direct-message limitation, while the Telegram handler and tests still treat private-chat reaction updates as routable if such an update is injected. The PR remains a focused docs/config/test-comment clarification after the author removed the unrelated FILIN-TODO scratch file noted in review.

## Best Possible Solution

Keep the PR open and either merge or maintainer-adapt its scoped clarification: add the Telegram Bot API administrator requirement and DM reaction limitation to docs/channels/telegram.md, mirror the caveat in the Telegram config type/help surface as appropriate, and add the test comment so private-chat reaction unit tests are not interpreted as a production guarantee. No core routing change is indicated unless Telegram changes Bot API delivery semantics.
