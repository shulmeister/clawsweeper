## Summary

Current `main` already implements streamed token accounting for `vllm` and other supported OpenAI-compatible local backends, and the same implementation is present in the shipped `v2026.4.22` release. The issue’s reported regression no longer matches current code/docs/tests.

## Risks / Open Questions

- If a self-hosted backend does not actually emit a final usage chunk, OpenClaw will still show `tokens ?`; that would be a backend-specific or new regression, not the old default-compat bug described here.
- Generic custom `openai-completions` providers still default `supportsUsageInStreaming` off unless explicitly overridden; this close only fits the reported `vllm` path, which is now explicitly supported.
