## Summary

Keep open. Current main has adjacent Bedrock Mantle support for Bearer auth, but the requested feature is specifically Bearer API-key auth for the native Bedrock Converse streaming provider/proxy path. The public docs still describe native Bedrock as AWS SDK credential-chain auth, while Mantle is a separate OpenAI-compatible surface.

## Evidence

- **Issue requests native Bedrock Converse Bearer auth:** Issue #59514 asks for `api: "bedrock-converse-stream"` with `auth: "api-key"` to send `Authorization: Bearer <api-key>` and skip SigV4 signing for Bedrock-format gateway proxies.
  - command: `GitHub issue #59514 body, lines 205-253 in fetched page`
- **Native Bedrock remains AWS SDK auth:** The Amazon Bedrock provider docs say OpenClaw uses pi-ai's Bedrock Converse streaming provider and that Bedrock auth uses the AWS SDK default credential chain, not an API key. The example config still uses `api: "bedrock-converse-stream"` with `auth: "aws-sdk"`.
  - file: [docs/providers/bedrock.md:100](https://github.com/openclaw/openclaw/blob/7bbd47349ec37f01a8f2b1dc219bf950615f361b/docs/providers/bedrock.md#L100)
  - command: `Opened https://docs.openclaw.ai/providers/bedrock`
  - sha: [7bbd47349ec3](https://github.com/openclaw/openclaw/commit/7bbd47349ec37f01a8f2b1dc219bf950615f361b)
- **Mantle is separate and OpenAI-compatible:** The Mantle docs say `amazon-bedrock-mantle` connects to Mantle's OpenAI-compatible `/v1/chat/completions` surface, uses `api: "openai-completions"`, and is separate from the standard Amazon Bedrock provider, which uses the native Bedrock API.
  - file: [docs/providers/bedrock-mantle.md:100](https://github.com/openclaw/openclaw/blob/7bbd47349ec37f01a8f2b1dc219bf950615f361b/docs/providers/bedrock-mantle.md#L100)

- **Related PR added adjacent Mantle support:** PR #61296 added the `amazon-bedrock-mantle` provider with bearer-token auth and `api: "openai-completions"`; it merged as dbac5fa on 2026-04-05. This supports the maintainer comment's workaround direction but does not prove the requested Converse-format proxy auth path is implemented.
