## Summary

Keep open. The issue is recent, has a current +1, and the timeline links an open implementation PR (#59170). Current public configuration docs still describe `tools.agentToAgent` and `tools.sessions.visibility` as top-level tool configuration, while per-agent docs list tool profile/allow/deny/elevated and `subagents.allowAgents`, not the requested per-agent session visibility or agent-to-agent allow/receive scoping. This is a plausible core access-control feature request, not a ClawHub/plugin cleanup candidate.
