## Summary

Keep open. Current main now honors OPENCLAW_HOME for the shared effective-home resolver, default state/config paths, config IO, and profile-derived state/config paths, but the report’s explicit OPENCLAW_GIT_DIR/default checkout and installer onboarding path concerns are still present. The existing Codex review comment reached the same conclusion, and current main still matches that remaining gap.

## Best Possible Solution

Keep this issue open and narrow the remaining work to the OPENCLAW_HOME contract. A maintainer should decide whether OPENCLAW_HOME should seed the default OPENCLAW_GIT_DIR and installer onboarding/workspace paths; if yes, update src/cli/update-cli/shared.ts, scripts/install.sh, docs/cli/update.md, docs/help/environment.md, and focused tests. Review #65536 separately because it addresses a related sandbox tilde-expansion bug, not the broader home-derived OpenClaw path defaults.
