## Summary

Keep open. Current main still preserves OpenAI-compatible `reasoning_content` as replayable thinking metadata for generic/custom `openai-completions` providers, and the replay policy does not drop those thinking blocks for an oMLX-style provider. The linked fix PRs for #46637 were closed unmerged, and the only current `reasoning_content` stripping I found is scoped to DeepSeek V4 disabled-thinking payload patching, not this oMLX/Qwen history replay bug.
