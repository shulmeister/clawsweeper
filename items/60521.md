## Summary

Keep open. Current main still appears to drop `allowUnsafeExternalContent` from direct `POST /hooks/agent` requests before dispatch, matching the reporter's narrowed root cause. The downstream cron agent runner can honor the flag when it is present on an `agentTurn` payload, but the `/hooks/agent` normalizer does not preserve it, so the reported request-body flag cannot reach that path.

## Evidence

- **Normalizer drops request flag:** `normalizeAgentPayload` builds the accepted `/hooks/agent` payload fields and returns `message`, `name`, `agentId`, `idempotencyKey`, `wakeMode`, `sessionKey`, `deliver`, `channel`, `to`, `model`, `thinking`, and `timeoutSeconds`, but not `allowUnsafeExternalContent`.
  - file: [src/gateway/hooks.ts:399](https://github.com/openclaw/openclaw/blob/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54/src/gateway/hooks.ts#L399)
