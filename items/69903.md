## Summary

Keep PR #69903 open for maintainer judgment. Current main has since added a broader, plugin-based diagnostics/OTel path, and it explicitly avoids exposing a mutable logger transport registry, so this PR should not be merged as-is. But the exact Sentry-style DSN/captureException/crash-flush behavior is not implemented on main, and the current public plugin API does not fully expose internal log records or a crash-flush hook for a third-party Sentry plugin. That makes auto-closing as implemented or ClawHub too aggressive.

## Best Possible Solution

Keep the PR open, but do not merge it as-is. The best path is a maintainer decision between: closing manually as superseded by the current `diagnostics-otel` direction, or asking for a rework that fits the diagnostics/plugin architecture. A rework should avoid a core `@sentry/node` dependency and mutable logger transport registry, use redacted diagnostic events, wire the packaged `openclaw.mjs`/`src/entry.ts` path, add a narrow public sink or fatal-flush API if Sentry should live in ClawHub, and cover crash-flush plus redaction behavior in tests.
