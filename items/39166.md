## Summary

Keep open. Current main partially supersedes this PR by forwarding explicit `runTimeoutSeconds` for ACP spawns, but it still does not implement the PR's central config-default fallback: `acp.defaultRunTimeoutSeconds` is absent from config types/schema/help/labels/generated schema, `spawnAcpDirect` has no resolver that applies config when the tool omits `runTimeoutSeconds`, and docs only describe explicit ACP `runTimeoutSeconds` plus native subagent defaults.

## Best Possible Solution

Keep the PR open, but ask for a rebase against current main and scope it to the still-missing behavior: add `acp.defaultRunTimeoutSeconds` to the config type/schema/help/labels/generated schema and docs, resolve ACP spawn timeout as explicit `runTimeoutSeconds` -> `acp.defaultRunTimeoutSeconds` -> `0` or the maintainer-approved default, and cover omitted, explicit override, and explicit `0` cases in ACP spawn tests.
