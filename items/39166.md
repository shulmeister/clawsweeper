## Summary

Keep PR #39166 open. Current main has the explicit ACP `runTimeoutSeconds` plumbing, but it still does not implement the PR's central config-level fallback `acp.defaultRunTimeoutSeconds` for omitted `sessions_spawn(runtime="acp")` calls. The linked request #38419 was closed stale rather than shipped, so this PR remains the actionable place for the remaining behavior after a focused rebase.

## Best Possible Solution

Keep this PR open and ask for a narrow rebase against current main. Preserve the existing explicit ACP timeout plumbing, add `acp.defaultRunTimeoutSeconds` to the ACP config type/schema/help/labels/generated metadata and docs, resolve ACP spawn timeout as explicit per-call value -> `acp.defaultRunTimeoutSeconds` -> `0`, and keep focused tests for omitted/default, explicit nonzero override, and explicit `0` overriding a nonzero default.
