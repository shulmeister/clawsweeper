## Summary

Current main still does not implement the PR's central `acp.defaultRunTimeoutSeconds` fallback. It has explicit ACP `runTimeoutSeconds` plumbing, but omitted ACP timeouts still remain `undefined` with no config-level default, so the PR remains actionable after a rebase.

## Best Possible Solution

Keep this PR open and ask for a focused rebase against current main. The best implementation is to preserve the already-landed explicit ACP timeout behavior, add `acp.defaultRunTimeoutSeconds` to the config contract and user-facing docs, resolve ACP timeout as explicit per-call value -> config default -> `0`, apply that resolved value to ACP runtime initialization and gateway dispatch, and retain tests for config default, explicit override, and explicit `0` over a non-zero default.
