## Summary

Keep open. Current main still intercepts native Chat UI slash commands locally before `chat.send`, while the gateway already has `chat.send` slash-command behavior. This PR is an active, plausible implementation candidate, but it has grown into broader iOS/macOS chat UI work and has unresolved review concerns, plus an open same-author companion PR for offline APNs alerts.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either land a focused native Chat UI change that routes slash commands through `chat.send` while preserving reset/compact history semantics and regression tests, or split the broader UI/APNs/model-picker changes into smaller reviewable PRs coordinated with #67493.
