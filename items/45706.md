## Summary

Keep open. The issue has concrete logs/config and is not stale; the related #38902 discussion is relevant, but current main still does not show a complete Mindie/OpenAI-compatible parameter fix. The suggested `supportsStreamOptions`/`supportsToolChoice` keys remain unsupported, and the current OpenAI-completions request builder still emits `stream_options.include_usage` unconditionally, with proxy-style `tool_choice: "auto"` behavior still present when tools are enabled.
