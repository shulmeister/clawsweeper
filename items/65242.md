## Summary

Keep open. Current main does not implement the PR's central CompletionDeliveryGate behavior: there is no gate module or env flag, ACP task completions still visibly deliver or queue background-task messages, and the subagent announce/farewell paths still have no shared ownership claim. The PR is also not merge-ready as-is because later review comments identify unresolved P1 risks around claiming before successful delivery, bypassing cleanup, and Discord child thread binding. The earlier Greptile runtime-key finding was addressed in PR commit 5271ff28, but that does not make the current PR obsolete or safe to close.

## Best Possible Solution

Keep the PR open but require revision before merge: implement a cleanup-safe completion ownership seam that only records delivery after a path has actually delivered or intentionally taken over, add regression coverage for task-registry banner vs announce vs farewell on ACP child completions, and split the Discord thread-binding/requester-origin routing changes into a separately reviewed fix. Reconcile overlap with the still-open related completion-delivery PR #64416 rather than closing this work as implemented.

## Risks / Open Questions

- Applying the PR as-is could suppress fallback farewell delivery after an announce path fails or defers, leaving users with no completion message.

- The PR bundles Discord thread-binding and requester-origin routing changes with the completion gate, so it should be split or validated separately before merge.
