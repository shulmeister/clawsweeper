## Summary

Keep open. Current main still lacks the PR's core Docker/DooD fix: sandboxed agents receive only the host-local browser bridge URL, sandbox browser CDP readiness is still loopback-only, and the active plugin SDK/browser runtime bridge contract has no advertised/container-reachable URL. The draft should not merge as-is, but the underlying product bug remains real and is not implemented or superseded on main.

## Best Possible Solution

Keep this open or replace it with a current-path PR. The best fix is to add an explicit advertised/container-reachable browser bridge URL through src/plugin-sdk/browser-bridge.ts and the bundled browser runtime, keep host-local bridgeUrl for gateway/browser-tool calls, inject the advertised URL into embedded sandbox info, resolve sandbox browser CDP host/port separately for Dockerized gateways, handle attach-only non-loopback tab creation without the persistent Playwright path where needed, and cover Dockerized gateway plus non-Docker host cases with focused tests and docs.
