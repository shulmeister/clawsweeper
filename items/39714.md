## Summary

Keep open. Current main still lacks the advertised/container-reachable sandbox browser bridge URL and Dockerized CDP host handling that this PR is trying to fix. The draft PR is not mergeable as-is because it targets obsolete browser paths and includes unrelated churn, but the underlying bug remains real and the PR still contains useful reproduction and partial implementation context.

## Best Possible Solution

Keep this PR open as useful bug context, or replace it with a current-path PR. The best implementation is to add an explicit advertised/container-reachable browser bridge URL through `src/plugin-sdk/browser-bridge.ts` and the browser plugin runtime, keep the host-local bridge URL for gateway-side browser tools, inject the advertised URL into `buildEmbeddedSandboxInfo()`, resolve sandbox browser CDP host/port separately for Dockerized gateways, handle attach-only non-loopback tab creation without the persistent Playwright path where needed, and add focused Dockerized gateway plus non-Docker regression coverage and docs.
