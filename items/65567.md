## Summary

Keep this PR open. Current main still has the unguarded `hashText(event.text)` and `emitSystemStatus(event.text)` paths described in the PR, while the PR discussion shows the author updated the branch to cover both crash sites after bot review.

## Best Possible Solution

Keep this PR open and review it as a real fix candidate. The best path is to land this branch or a superseding patch that guards both `hashText(event.text)` and `emitSystemStatus(event.text)`, with a focused regression test for a `usage_update` status event missing `text` and, if maintainers agree, an explicit contract update or boundary normalization for optional status text.
