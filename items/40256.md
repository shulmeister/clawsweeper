## Summary

Current main still leaves channel/session-dependent prompt text before the large Project Context block, so the reported prefix-cache invalidation is not implemented on main. The discussion also has recent user reproductions on llama.cpp/lemonade-server and an open PR (#40296) attempting the original ordering fix, but that PR is not merged and does not prove current main is fixed.

## Best Possible Solution

Keep #40256 open. The maintainer path should either land a focused version of #40296 or a fresh patch that moves volatile channel/session sections below the cache boundary or into a separate late message, adds an ordering regression test for Project Context versus Messaging/Group Chat Context/Reactions, and separately evaluates the current-channel message tool description churn reported in the later comments.
