## Summary

Current `main` already normalizes inbound Slack mention tokens in the Slack message preparation path, and the normalized `rawBody` is what gets copied into both `BodyForAgent` and `RawBody`. Regression tests cover the exact behavior reported, including unresolved-name fallback and lookup caps. The implementing change appears to be on `main` after the latest tagged release, so the fix is present but not yet tied to a released version.

## Evidence

- **Mention normalization now happens before `rawBody` is finalized:** `resolveSlackMessageContent` collects unique Slack mention ids from inbound text, resolves user names, rewrites mentions to annotated human-readable form, and then builds `rawBody` from the rendered text.
