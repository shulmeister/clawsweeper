## Summary

Keep open. Current main still only exposes a markdown copy action for assistant messages; the issue’s requested plain-text copy option/dropdown/preference is not implemented. The existing maintainer-side Codex comment from 2026-04-26 also correctly matches current main.

## Best Possible Solution

Keep #7909 open until the Control UI adds a real plain-text copy path for assistant messages. The likely implementation belongs near ui/src/ui/chat/copy-as-markdown.ts and ui/src/ui/chat/grouped-render.ts: expose a dropdown or adjacent action with Copy as Markdown and Copy as Plain Text, wire a tested Markdown-to-plain-text conversion helper, and add focused UI/helper tests for the new clipboard behavior.
