## Summary

Current main still does not implement #7909. Control UI assistant-message copy remains a single Markdown-only clipboard action, existing plain-text/markdown-stripping helpers are not wired into that action, and the relevant docs/changelog/searches do not show a shipped Markdown/plain-text copy selector. The related Android issue/PR are a different mobile long-press copy/selection surface and do not supersede this Control UI request.

## Best Possible Solution

Keep #7909 open. The best implementation belongs in the Control UI assistant-message action path: add a compact Markdown vs plain-text copy menu or user preference near `ui/src/ui/chat/copy-as-markdown.ts` and `ui/src/ui/chat/grouped-render.ts`, reuse or adapt an existing markdown-to-text helper, and add focused UI/clipboard tests proving both formats write the expected clipboard text.
