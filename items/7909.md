## Summary

Keep #7909 open. Current main still exposes only a single Control UI assistant-message copy action labeled “Copy as markdown” and writes the markdown string directly to the clipboard. I found no Markdown/plain-text selector, no default copy-format preference, no assistant-message plain-text copy wiring, and no focused tests or docs showing the requested feature has shipped.

## Best Possible Solution

Keep #7909 open. The best implementation belongs in the Control UI assistant-message copy path around ui/src/ui/chat/copy-as-markdown.ts and ui/src/ui/chat/grouped-render.ts: add a compact Markdown vs plain-text copy menu or a persisted user preference, reuse or adapt the existing markdown-to-text helper, and add focused UI/clipboard tests proving both formats write the expected clipboard text.
