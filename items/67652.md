## Summary

Keep open. Current main still only normalizes and resolves `mediaUrls` and `mediaUrl`; it does not accept the loose outbound `media` alias that PR #67652 targets. The PR is therefore not obsolete or implemented on main, and its review thread still has a behavior concern around empty `mediaUrl` fallback that should be resolved before merge.

## Best Possible Solution

Keep PR #67652 open and have the author rebase/update it against current main, preserve the narrow shared `src/plugin-sdk/reply-payload.ts` fix, add/keep alias regression tests, and resolve the empty `mediaUrl` fallback review comment before maintainer merge consideration.
