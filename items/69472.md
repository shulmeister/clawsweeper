## Summary

Keep PR #69472 open. Current main still lacks the PR's channel-agnostic suppression for plain-text reasoning-prefaced `NO_REPLY` payloads, and no searched source, docs, changelog, or history evidence shows the fix already shipped. The PR is also not safely closeable as-is because its own review thread flags a plausible false-positive risk that should be addressed before merge rather than discarding the work.

## Best Possible Solution

Keep #69472 open as the canonical implementation candidate for #69470. The best path is to revise the PR so `isReasoningPrefacedSilentReply` suppresses only truly reasoning-only preamble payloads ending in single or doubled `NO_REPLY`, preserves #19537 and normal `Analysis:`-headed substantive replies, and lands with the proposed focused tests plus a regression for the review-thread false positive.
