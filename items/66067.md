Keep this PR open. Current main still has the reported TUI thinking-spacing behavior: thinking blocks are joined with single newlines, the stream assembler displays that formatter output directly, and the current tests still expect newline-joined thinking. The PR remains a focused implementation candidate for a real readability bug linked to closed issue #66008, with an actionable CRLF review follow-up rather than evidence of obsolescence.

## Best Possible Solution
