## Summary

Keep #66067 open. Current main still has the behavior the PR is trying to fix: thinking blocks are joined with single newlines, the stream assembler feeds that text into the assistant display path, and the existing formatter test still expects `alpha\nbeta`. The PR is a focused bug-fix candidate rather than obsolete cleanup, though the unresolved Copilot review comments about CRLF handling should be addressed before merge.

## Best Possible Solution

Keep #66067 open and ask for a small revision: normalize thinking whitespace in the TUI formatter while first canonicalizing CRLF/bare CR to `\n`, preserve explicit paragraph breaks, and keep the formatter plus `TuiStreamAssembler` regression tests. After that, this remains a narrow TUI bug fix candidate for maintainer review.
