## Summary

Issue #68050 proposes an integration that already fits OpenClaw's intended external plugin model. Current main documents the needed hook surfaces and directs optional capabilities like this to third-party plugins distributed via ClawHub/npm rather than OpenClaw core.

## Evidence

- **Vision scope puts optional capability in plugins, not core:** OpenClaw's vision says optional capability should usually ship as plugins, code plugins are the right path for runtime hooks, plugin authors should host and maintain plugins in their own repository, and plugin promotion belongs in ClawHub rather than core.

- **Current docs already expose the exact hook types AgentShield uses:** The plugin hook catalog on main documents `before_prompt_build` for prompt mutation plus `before_tool_call` and `after_tool_call` for tool-call policy and observation, matching the integration described in the issue.
  - file: [docs/plugins/hooks.md:63](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/docs/plugins/hooks.md#L63)
