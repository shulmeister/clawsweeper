## Summary

Keep open. Current main does not implement `openclaw models status --probe --rate-limits`, does not add `rateLimit` data to probe results, and existing provider usage/quota windows are not the same per-probe rate-limit-header surface. The linked issue #42401 was closed as superseded by this PR, so this PR remains the canonical unmerged thread rather than something conservative cleanup can close.

## Best Possible Solution

Keep the PR open. The best path is for maintainers to ask for the review-comment fixes or replace it with a maintainer-owned implementation that adds an opt-in `--rate-limits` probe path, a typed `rateLimit` payload in JSON, RPM/TPM table columns, tests for failure/base URL cases, and docs for the new CLI surface.
