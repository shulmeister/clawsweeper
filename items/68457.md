## Summary

Keep #68457 open. Current main still lacks a WhatsApp `silentErrorReplies`/error-policy config surface and does not wire WhatsApp auto-reply delivery through an error-suppression policy. The linked PR #68465 is relevant and directly tracks the request, but it is open/unmerged per the provided GitHub context, so the issue remains useful canonical tracking until that work lands or is explicitly superseded.

## Best Possible Solution

Keep the issue open and land/review #68465 or a maintainer-owned replacement. The best fix belongs in the WhatsApp plugin config/runtime surface: add a documented WhatsApp error-reply policy, update schema/types/generated metadata/UI hints/docs, wire `dispatchWhatsAppBufferedReply` to suppress configured gateway-synthesized error payloads while preserving normal agent text, and add focused inbound-dispatch regression tests.
