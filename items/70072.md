## Summary

Current main has useful partial work around dreaming promotion, including selector-level `memory promote-explain` and fixes that let daily/grounded signals satisfy deep gates, but it does not implement the issue's central request: scheduled deep-dream artifacts still do not expose every ranked/rejected candidate with gate reasons, verbose scoring is still logger-only, and there is no `openclaw memory dreaming doctor` command.

## Best Possible Solution

Keep this issue open and implement the introspection surface in `memory-core`: add reusable gate-reason/explanation helpers around `rankShortTermPromotionCandidates`, include ranked and rejected candidate details plus active thresholds in the scheduled deep report, optionally write structured verbose JSONL under `memory/.dreams/deep/`, and add an aggregate `openclaw memory ... doctor` diagnostic only if maintainers want a CLI surface beyond `promote-explain`.
