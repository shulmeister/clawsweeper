## Summary

Keep open. Current `main` still resolves Telegram exec approval callbacks by clearing inline buttons, not deleting the approval message, and the Telegram tests assert that behavior. The linked PR #56319 still exists as an open remote PR ref rather than a merged fix.

## Evidence

- **Current main clears buttons after approval instead of deleting the message:** In the Telegram callback handler, the exec-approval path calls `clearCallbackButtons()` after `resolveExecApproval(...)` and then returns. The local `deleteCallbackMessage()` helper exists earlier in the same handler, but it is not used for the approval-success path.
