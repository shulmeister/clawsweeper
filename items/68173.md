## Summary

Current main already covers the central request: model selection and fallback editing are available in the Control UI, advanced settings can add/manage provider model entries, Ollama local models are discovered automatically, and the old chat-based `/models add` path has been deprecated. The shipped implementation is a generic model/config workflow rather than a bespoke local-model manager, but it solves the reported dashboard/config-panel workflow well enough to close this broad feature request.

## Best Possible Solution

Keep the shipped dashboard/config implementation for model selection, fallback editing, manual provider/model registration, Ollama auto-discovery, and deprecated chat-based add behavior. If more polish is desired, track it as a narrower UX enhancement for a dedicated local-model wizard or toggle view rather than keeping this broad request open.
