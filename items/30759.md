## Summary

Keep open. Current main still supports WebChat uploads only for image attachments, while this issue asks for PDF, Word, and TXT document upload/extraction. The related #70438 is useful but remains open and does not fully cover Word/doc/docx extraction.

## Best Possible Solution

Keep this issue open unless maintainers choose a canonical broader tracker. A complete fix belongs in the WebChat/UI plus gateway attachment path: broaden composer accept/filter/preview behavior for documents, add explicit size and MIME validation, route TXT/PDF/Word content through safe extraction or offload, surface errors instead of silently dropping files, and add tests covering PDF, TXT, and doc/docx behavior. If maintainers consolidate with #70438, that canonical issue should explicitly include Word/doc/docx extraction, not only configurable PDF/text handling.
