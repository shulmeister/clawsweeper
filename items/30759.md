## Summary

Keep #30759 open. Current main partially satisfies the request after PR #72299/commit 6b6dcafc: WebChat now accepts and serializes non-video files, Gateway persists them as inbound media, and the media pipeline extracts PDF and text content. The remaining requested Word/DOC/DOCX behavior is still missing: Office-style application/vnd attachments are treated as binary and skipped, the bundled document extractor only registers PDF, and provider native document input metadata is currently limited to PDF.

## Best Possible Solution

Keep #30759 open, or manually consolidate it into a canonical WebChat document-upload issue that explicitly tracks Word/DOC/DOCX extraction and user-facing unsupported-file behavior. The best implementation path is to build on #72299/6b6dcafc, add a safe DOCX/Word extractor or a clear unsupported-document policy, align MIME/size/docs behavior, and cover PDF, TXT, DOC/DOCX, and unsupported Office cases with focused UI, Gateway, media-understanding, and extractor tests.
