## Summary

Keep open. Current main still implements WebChat attachments as image-only: the UI accept/filter path rejects PDF/Word/TXT before send, accepted attachments are serialized as images, and the gateway chat.send attachment parser drops non-image payloads. The related #70438 is a useful design thread for configurable non-image handling, but it is still open and does not fully cover the Word/doc/docx extraction requested here.

## Best Possible Solution

Keep this open unless a maintainer manually chooses a canonical tracking issue. The implementation should add a document-aware WebChat upload path: broaden the UI accept/filter/preview behavior, validate MIME and size limits, route documents through a safe gateway extraction/offload flow, support text/PDF and Word/doc/docx behavior explicitly, and avoid silently dropping non-image chat.send attachments.
