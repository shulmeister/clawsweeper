## Summary

Keep open. Current main still does not expose inbound media on `PluginHookAgentContext`, and the `before_agent_reply` call site still passes only agent/session/provider metadata. Because a handled hook returns before `stageSandboxMedia`, the motivating gap for plugins that claim media-bearing turns remains. The PR is a plausible API addition, but it needs rebase onto the current hook-type split plus tests/docs/API review before it can land.

## Best Possible Solution

Keep the PR open and ask for a rebase onto current main. The best path is to add `mediaPaths`, `mediaUrls`, and `mediaTypes` to `src/plugins/hook-types.ts`, populate defensive copies in the `before_agent_reply` context in `src/auto-reply/reply/get-reply.ts`, preserve URL-only attachments, document the fields in `docs/plugins/hooks.md`, and add focused tests for text-only, local media, URL-only media, and mutation safety before maintainer API review.
