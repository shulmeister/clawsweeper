## Summary

Current `main` still does not implement the requested one-click Anthropic browser OAuth onboarding flow. The Anthropic plugin remains wired around Claude CLI reuse, setup-token, and API-key authentication; existing `sk-ant-oat` handling is compatibility for already-present tokens at request time, not credential acquisition or account-type onboarding.

## Best Possible Solution

Keep this open for Anthropic-provider product work. If maintainers accept the feature, implement a provider-owned browser OAuth method in `extensions/anthropic`, expose it through the manifest/auth choices used by onboarding/configure, use the existing provider auth `openUrl`/OAuth helper seams, persist the returned auth profile, select safe Pro-vs-API defaults, update provider docs, and add focused regression coverage. If Anthropic cannot support a sanctioned browser OAuth flow, maintainers should explicitly narrow or close the issue around the supported Claude CLI/setup-token/API-key UX.
