## Summary

Keep #32515 open. Current main still lacks the requested one-click Anthropic browser OAuth flow from onboarding/configuration. Anthropic has useful related support now: Claude CLI reuse, setup-token paste, API key auth, env-backed `ANTHROPIC_OAUTH_TOKEN`, and runtime handling for `sk-ant-oat*` credentials. But none of that implements `openclaw onboard` opening a browser for “Sign in with Anthropic” and configuring the account automatically.

## Best Possible Solution

Keep the issue open for maintainer/product handling. If accepted, implement this inside the bundled Anthropic plugin as a provider-owned OAuth auth method and manifest auth choice, using the existing provider auth `openUrl`/OAuth handler seams, persisting the resulting profile through auth profiles, updating onboarding/configure/docs, and adding regression coverage for browser OAuth selection plus API-key-versus-OAuth token handling. If Anthropic does not provide a supported public OAuth flow, maintainers should explicitly retitle or close the issue around the narrower setup-token/Claude CLI UX direction.
