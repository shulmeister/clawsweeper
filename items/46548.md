## Summary

Keep open. Current main partially addresses the retry-confusion case by clearing an unresolved mutating tool failure when the same action later succeeds, so users should not see a stale failure for that retry path. But the core failure-reason request remains unimplemented for normal non-verbose mutating tool failures: the warning is still emitted as `failed` without `lastToolError.error`, and tests explicitly assert that the detail is absent. The cross-referenced PR #46592 was closed unmerged and, per the provided discussion, only attempted the failure-reason half while explicitly leaving retry confirmation out of scope.
