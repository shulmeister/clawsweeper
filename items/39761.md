## Summary

Keep PR #39761 open. Current main still does not expose agent reasoning metadata to the `message_sending` plugin hook, and the PR is a plausible backward-compatible plugin API addition rather than obsolete cleanup. The related #69110 model-identity bug is adjacent but does not supersede this broader tool/token/session context request.

## Best Possible Solution

Keep PR #39761 open for maintainer review or adaptation. The best path is a backward-compatible `agentContext` or equivalent typed field on `PluginHookMessageSendingEvent`, threaded through outbound delivery and populated from embedded-runner metadata for agent-originated sends, with docs in `docs/plugins/hooks.md` and focused hook contract tests. Coordinate provider/model identity with #69110 instead of closing this PR as obsolete.
