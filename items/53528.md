## Summary

Keep open. Current `main` does not implement the PR’s core behavior: external `jobs.json` writes are still invisible while the cron service is idle, so there is not strong evidence to close this as already implemented or obsolete.

## Evidence

- **Store cache has no external invalidation path:** `ensureLoaded()` returns immediately when `state.store` is already populated unless callers force a reload, and `store.ts` contains no watcher or other invalidation hook for external `jobs.json` edits.
  - file: [src/cron/service/store.ts:30](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/cron/service/store.ts#L30)
  - command: `nl -ba src/cron/service/store.ts | sed -n '19,104p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Cron startup does not register a file watch:** `start()` loads the store, recomputes schedules, and arms the timer, but there is no `watchStore` registration or cleanup in the current service startup/shutdown path.
  - file: [src/cron/service/ops.ts:87](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/cron/service/ops.ts#L87)
  - command: `nl -ba src/cron/service/ops.ts | sed -n '87,139p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Idle scheduler has no polling fallback when there are zero enabled jobs:** `armTimer()` returns without scheduling any timer when `nextWakeAtMs` is absent and `enabledCount` is `0`, so a later external creation of `jobs.json` after empty startup has no wake path on current `main`.
  - file: [src/cron/service/timer.ts:628](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/cron/service/timer.ts#L628)

  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Existing seam tests do not cover external file-change pickup:** Current `store` seam tests exercise load/persist and legacy normalization, but there is no regression coverage for detecting direct external `jobs.json` writes after startup.
  - file: [src/cron/service/store.test.ts:43](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/cron/service/store.test.ts#L43)

## Risks / Open Questions

- Closing this PR would drop a still-unimplemented cron recovery path for degraded scenarios where direct `jobs.json` writes are the only control surface.
- The PR likely still needs design changes before merge because the reviewed approach invalidates cache state but, by itself, does not wake an idle scheduler that has no timer armed.
