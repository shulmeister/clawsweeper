## Summary

Keep open. Current main still does not expose `stopReason` or `hasToolCalls` to `message_sending` hook handlers. The hook event type, shared outbound hook invocation, generic inbound dispatcher hook path, docs, and nearby tests all still show only delivery/routing metadata. Related PR #39761 is relevant overlap, but it remains open feature work rather than shipped or clearly canonical closure evidence for this issue.

## Best Possible Solution

Keep the issue open and implement a small public plugin-hook contract change: thread assistant completion metadata from the agent/block-reply path into delivery payload metadata, expose optional `stopReason` and `hasToolCalls` fields to `message_sending` handlers, and add focused type, outbound, inbound dispatcher, and docs coverage. If PR #39761 becomes the chosen broader solution, link this issue to that PR and close it only when the merged code demonstrably exposes the requested fields.
