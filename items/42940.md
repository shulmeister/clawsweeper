## Summary

Keep open. Current `main` still closes the Feishu streaming card on every `final` reply, so the reported multi-final duplication path is not implemented away, and the PR’s proposed move to `onIdle` has a real lifecycle risk because dispatcher idleness does not await async `onIdle` work.

## Evidence

- **Current main still closes on each final:** `extensions/feishu/src/reply-dispatcher.ts:416-418` merges the final text into `streamText` and immediately `await closeStreaming()`, which resets the active streaming session before any later final in the same turn.

## Risks / Open Questions

- Closing this PR would drop a still-plausible Feishu bug report that current main and the latest release both still reflect.
- Merging the PR as-is would move card finalization onto async `onIdle` work that `waitForIdle()` does not cover, risking stuck `[Generating...]` cards during restart or shutdown races.
- There is no current Feishu test covering the intended single-card multi-final behavior plus restart-safe finalization, so the proposed fix is not yet proved on main.
