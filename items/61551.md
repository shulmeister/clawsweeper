## Summary

Keep open. Current main contains a scoped guard for later default-mode loads after a gateway-bindable registry, but the global hook runner initializer still overwrites unconditionally and the ensureRuntimePluginsLoaded path can still reach a fresh plugin load when the active registry cache key is not compatible. That is not strong enough evidence to close #61551 as fixed.

## Evidence

- **Global runner initializer still overwrites:** initializeGlobalHookRunner assigns state.registry and state.hookRunner to a newly created runner on every call; there is no idempotency or merge guard in this function.

- **Loader guard is scoped, not general:** activatePluginRegistry only skips initializeGlobalHookRunner when a later load is runtimeSubagentMode === "default", the active mode is "gateway-bindable", and a global runner already exists. Other activating loads still call initializeGlobalHookRunner.
