## Summary

Keep open. Current main does not implement the PR's requested Claude Code flat-argument compatibility: the read/write/edit tool validation still requires canonical `path`, `content`, and `edits`, current tests explicitly assert alias rejection, and the production `toToolDefinitions` adapter still rebuilds tools without forwarding a `prepareArguments` hook. The PR may still target a real compatibility gap, but the current patch is not safely obsolete or merge-ready because the reviewed adapter path would likely bypass the new hook.

## Best Possible Solution

Keep the PR open for maintainer/product review and request changes rather than auto-closing it. A maintained fix should normalize Claude-style flat read/write/edit arguments before schema/runtime validation in the actual embedded runner path, likely by preserving or invoking `prepareArguments` in `toToolDefinitions` or by adding a central pre-validation argument normalizer. It should include regression tests for direct read/write/edit tools and the `splitSdkTools`/`toToolDefinitions` path, and should either drop or narrowly justify the generic `file` -> `path` alias.
