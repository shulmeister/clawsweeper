## Summary

Keep open. Current main still preserves the reported core behavior: stale CLI/agent sessions without an explicit sessionId are still treated as new, that flag still forces skillsSnapshot rebuild/persistence, and sessions.json serialization still writes the full SessionEntry including skillsSnapshot and systemPromptReport. The linked PR #48425 is relevant but remains open/unmerged in the provided GitHub context, so it does not prove the issue is fixed on main.
