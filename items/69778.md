## Summary

Keep open. Current main still has the reported product gap: gateway/task restore and sub-agent orphan recovery have short grace/stale mechanisms, but restart-aborted sub-agent sessions marked `abortedLastRun: true` are still recoverable without an age-bound `agents.defaults.taskRecovery.maxAgeMinutes`-style cutoff. Existing docs and tests actually prove the current behavior preserves stale aborted runs for recovery rather than marking them lost.

## Best Possible Solution

Keep the issue open and implement an age-bounded recovery policy in the task/subagent restart path. Add a documented config default or hard default for stale interrupted task recovery, mark over-age interrupted CLI/subagent tasks/runs lost instead of dispatching synthetic resume prompts, add startup logging for skipped stale recoveries, and cover both task registry maintenance and subagent orphan recovery with regression tests.
