## Summary

Keep open. Current main still preserves and resumes restart-aborted subagent work without the requested age bound. There is a 2-hour stale-run prune for ordinary unended subagent rows, but `abortedLastRun: true` is explicitly exempted and current tests assert a 3-hour-old restored aborted run remains recoverable. The requested `agents.defaults.taskRecovery.maxAgeMinutes`-style config and startup skip-to-lost behavior are not present.

## Best Possible Solution

Keep the issue open and fix it in core recovery logic. Add an explicit max-age policy for restart-aborted task/subagent recovery, apply it before `scheduleSubagentOrphanRecovery` or inside `recoverOrphanedSubagentSessions`, mark too-old active task records/runs `lost` or terminal instead of re-dispatching, log a concise startup skip summary, and add regression tests covering old `abortedLastRun: true` restored rows plus task-list/status behavior.
