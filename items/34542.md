## Summary

Keep #34542 open. Current main has the runtime diagnostics/fallback errors, but the PR's caller-facing guidance is still not fully implemented in prompts, tool description/schema, or subagent docs. The PR should be refreshed against current main rather than closed as implemented.

## Best Possible Solution

Keep the PR open or replace it with a focused refresh against current main: drop or reconcile the superseded runtime-error changes, then add explicit unsupported-channel fallback guidance to buildAgentSystemPrompt, buildSubagentSystemPrompt, describeSessionsSpawnTool/createSessionsSpawnToolSchema, and the current public subagent/ACP docs. Add prompt/tool-copy assertions so agents do not regress to claiming thread-bound ACP/subagent sessions on unsupported channels.
