## Summary

Current main does not include the PR's requested structured Slack media file ID propagation. `SlackMediaResult` still only carries `path`, `contentType`, and `placeholder`; `resolveSlackMedia()` does not return `fileId`; Slack message preparation only maps media path/type/url fields into `FinalizedMsgContext`; and the shared message context type has no `MediaFileId` or `MediaFileIds`. The PR remains a plausible implementation candidate, though it should be reviewed together with the related open Slack file-ID/schema work so the final agent-facing contract is consistent.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to settle the agent-facing Slack file ID contract, then either land this PR with targeted Slack media/prepare tests and any necessary schema follow-through, or replace it with a combined fix that covers both inbound context propagation and the Slack `download-file` tool schema gap.
