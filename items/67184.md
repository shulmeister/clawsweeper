## Summary

Keep open. Current main has not implemented the requested non-image WebChat upload support: the UI still advertises and filters only `image/*`, and the downstream gateway path still treats attachments as image-oriented and drops sniffed non-image payloads. The PR addresses a real remaining capability, but it likely needs broader handling/tests before it can be considered complete.
