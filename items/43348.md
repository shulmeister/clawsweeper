## Summary

Keep open. Current main does not implement `acp_send`, `sessions_cancel`, or a `sessions.cancel` gateway RPC. Existing `sessions_send` can fire-and-forget, but it still starts the A2A announce path except for a narrow owned-child ACP skip, so it is not equivalent to the PR's ACP-callback-only send. ACP cancellation primitives exist for slash/task/admin paths, but the requested agent tool and gateway surface are still missing. The PR also has unresolved correctness review comments, so it needs maintainer review and updates rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer/product review. If this feature is still desired, rebase it onto current main, wire display metadata through `src/agents/tool-display-config.ts`, add `sessions.cancel` through the typed gateway protocol and scope map, add `sessions_cancel` with the same visibility/A2A policy checks used by other session tools, validate `acp_send` targets as ACP-only, and cover the new behavior with focused tool/protocol/gateway tests plus docs updates.
