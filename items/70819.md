## Summary

Keep open. Current main still rejects any `node_modules` symlink whose real target resolves outside the install root, and it has no trusted-host allowlist for `node_modules/openclaw` or `node_modules/.bin/openclaw`. The PR is a plausible focused fix for a still-live package-install scanner edge case, not superseded by the related discovery/profile symlink PRs.

## Best Possible Solution

Keep #70819 open for maintainer review. The best code path is to land a narrow scanner change that allows only exact `node_modules/openclaw` and `node_modules/.bin/openclaw` symlinks when their real target is inside the trusted host OpenClaw package root, while retaining fail-closed coverage for spoofed, nested, arbitrary, and non-exact symlink names.
