## Summary

Current main still has the requested coupling: explicit unthreaded `mode="session"` is rejected in both native subagent and ACP spawn paths, tests assert the native rejection, and public docs still document `thread: true` as required. The issue discussion describes a real remaining product gap, not an obsolete or shipped behavior.

## Best Possible Solution

Keep this open. The best implementation is to decouple session persistence from thread binding in `src/agents/subagent-spawn.ts` and `src/agents/acp-spawn.ts`: allow `mode="session"` without `thread`, create persistent native and ACP child sessions, route initial/completion output through the parent or announce path when there is no thread binding, return stable `childSessionKey` and label metadata for `sessions_send`, preserve existing `thread=true` behavior, and update regression tests plus the public ACP/subagents docs.
