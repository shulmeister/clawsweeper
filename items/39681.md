## Summary

Keep #39681 open. Current main still does not add the requested Advanced Features / Next Steps bridge to `docs/tools/creating-skills.md`; it only has a small metadata table and still omits most named advanced fields. The full `docs/tools/skills.md` reference and implementation/tests show those features exist, so the issue remains a valid docs discoverability gap. Provided GitHub context shows PR #59650 tracks the exact fix but remains open and unmerged.

## Best Possible Solution

Keep #39681 open until PR #59650 or an equivalent docs-only change lands. The best fix is a concise Advanced Features / Next Steps section in `docs/tools/creating-skills.md` that links to the public Skills reference for conditional activation, env/apiKey injection, command dispatch, template variables, invocation controls, testing guidance, and the full frontmatter/config reference.
