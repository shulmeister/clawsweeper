## Summary

Keep #70630 open. Current main still has the Telegram no-visible-response DM behavior the PR is fixing: it synthesizes a `NO_REPLY` fallback, runs it through outbound silent-reply projection, and counts only delivered/queued fallback as a final response. The existing regression test still explicitly expects that fallback to be delivered, while the linked bug #70628 and the PR discussion identify this as the unwanted user-visible chatter path.

## Best Possible Solution

Keep the PR open for maintainer review and CI. The best path is to merge or adapt this narrow Telegram dispatch fix so no-visible direct-message turns are treated as handled silence, while preserving real error fallback and group fallback behavior, and updating the existing regression test accordingly.
