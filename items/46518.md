## Summary

Keep open. This is a concrete bundled WhatsApp plugin login regression report with strong reporter evidence, including a same-phone side-by-side where Chromium/whatsapp-web.js paired while OpenClaw's Baileys QR flow timed out. Current main still ships the WhatsApp plugin on @whiskeysockets/baileys 7.0.0-rc.9, and the only repo patch for that dependency is a media-upload patch, not a QR/login handshake fix or a switch to the reported working Chromium-backed approach. The related issues in the provided context are closed stale or unrelated, so there is no canonical open replacement to close this against.
