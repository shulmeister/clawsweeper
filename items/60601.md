## Summary

Keep open. Current main has improved OpenAI-compatible proxy behavior by sending `tool_choice: "auto"` when tools are present, and it has an `extra_body` escape hatch that can override request payload fields. That is not enough to close this issue because the report specifically says Qwen 2.5 Coder 32B via llama.cpp still emits plain text when `tool_choice` is unset or `auto`, and current main still only promotes structured `delta.tool_calls`; plain JSON in `content` is appended as text.

## Evidence

- **Current main sends auto, not required, for proxy-like openai-completions endpoints:** `buildOpenAICompletionsParams` honors explicit `options.toolChoice`; otherwise proxy-like configured endpoints with tools get `params.tool_choice = "auto"`. The reporter states `auto` does not fix this Qwen/llama.cpp case.
