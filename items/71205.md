## Summary

Keep #71205 open. Current main at 9089e6b595ba33b4afc27b99c56ace80c4fbbc7b still hardens a matching config/state directory to 0o700 with no OPENCLAW_SKIP_STATE_DIR_HARDEN or equivalent opt-out, and the current test coverage only asserts the default hardening behavior. The PR is a focused core config/runtime bug fix for a concrete multi-UID sidecar deployment case; the provided discussion also shows the bot observability suggestion was addressed in d500b555e.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land or adapt the opt-in env guard in src/config/io.ts with the proposed regression test, while deciding whether docs/security audit wording should mention that multi-UID operators who set the opt-out must manage state-dir permissions themselves.
