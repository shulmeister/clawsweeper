## Summary

Keep open. Current `main` still lacks the requested `metadata.openclaw.activates.plugins` contract and per-turn skill-driven plugin tool visibility. Also, an open unmerged PR explicitly uses closing syntax for this issue, so this issue should remain open until that PR is reviewed, landed, or closed.

## Best Possible Solution

Keep this issue open for maintainer design and PR handling. The likely implementation path is to add a typed `metadata.openclaw.activates.plugins` skill metadata contract, preserve it through frontmatter parsing and skill snapshots, document it, and add a request-time tool visibility plan before tool construction that activates plugin tools from matched skills while preserving plugin disablement and `tools.deny` precedence. In parallel, maintainers should review PR #72533 and either close it as insufficient or replace it with a real implementation.
