## Summary

PR #67221 is superseded. Current main intentionally keeps Google Chat add-on `appPrincipal` bound to the JWT `sub`/numeric principal, while merged PR #71145 shipped warnings for email-shaped `appPrincipal` misconfiguration in v2026.4.24. The PR's secondary status change is also superseded by the current Google Chat-specific status snapshot extras, while the generic safe snapshot continues to omit `audience` and `audienceType` as requested by review feedback.

## Best Possible Solution

Close #67221 as superseded. Keep the shipped numeric `sub`/OAuth client ID principal contract, keep Google Chat status fields in channel-specific snapshot extras, and track any remaining documentation or config-help work in #53888 or a focused follow-up.
