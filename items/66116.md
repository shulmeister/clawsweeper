## Summary

Close as not actionable in this repository. Current main already sends Gemini Search through Google's documented Grounding with Google Search request shape, and the reported quota bucket mismatch points to Google AI Studio project/API-key billing or tier routing rather than an OpenClaw code path. The related ClawSweeper items are baseUrl/proxy gaps, not this quota-path claim.

## Best Possible Solution

Keep the existing Gemini Search implementation on Google's documented `generateContent` + `google_search` grounding path. Users who still see `generate_content_free_tier_requests` on a supposedly paid AI Studio key should resolve the Google project/API-key tier or billing/quota routing with Google, or open a new OpenClaw issue only if a sanitized raw payload proves OpenClaw sends a different request than Google's documented grounding request.

## Evidence
