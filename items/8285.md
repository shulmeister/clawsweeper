## Summary

Keep #8285 open. Current main still lacks a supported configurable pre-processing text acknowledgement surface: message config/schema/docs expose reaction acknowledgements, status reactions, and typing indicators, but no `messages.ackText`/`ackTextScope`. The existing plugin hooks do not close the gap because handled `before_dispatch`/`reply_dispatch` paths are terminal and skip the default model path.

## Best Possible Solution

Keep #8285 open as a core/product or plugin-API feature request. The best path is to add a supported non-terminal pre-processing acknowledgement contract, likely `messages.ackText` with scope and channel/account overrides or a documented plugin hook/delegate that can send interim text and then continue normal model dispatch. Update config types, zod schema, generated schema/help/labels, public docs, and channel tests for affected messaging surfaces.
