## Summary

Current main still does not provide a supported configurable pre-processing text acknowledgement. The issue is concrete across Slack, WhatsApp, webchat, and Feishu, and current code still only exposes ack reactions/status reactions/typing indicators plus terminal hook paths.

## Best Possible Solution

Keep #8285 open as a core/product or plugin-API feature request. The best path is to add a supported non-terminal pre-processing text acknowledgement contract, such as `messages.ackText` with scope and channel/account overrides or a documented hook with a safe send delegate, then update config types, zod schema, generated config docs, public docs, and channel tests for Slack, WhatsApp, webchat, and Feishu so the acknowledgement sends immediately while normal processing and final replies continue.
