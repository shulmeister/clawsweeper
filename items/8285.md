## Summary

Keep #8285 open. Current main still does not provide a supported configurable pre-processing text acknowledgement, and the existing plugin hooks do not offer a non-terminal safe-send contract that could implement `messages.ackText` while continuing normal agent processing. The issue discussion is also still actionable: Slack/WhatsApp/webchat were reported in the body, Feishu was confirmed in a comment, and a maintainer-side Codex review on 2026-04-26 already kept it open.

## Best Possible Solution

Keep #8285 open as a core/product or plugin-API feature request. The best path is to add a supported non-terminal pre-processing acknowledgement contract, such as `messages.ackText` with scope and per-channel/account overrides or a documented plugin hook with a safe send delegate, then update config types, zod schema, generated config docs/help, public docs, and channel tests for Slack, WhatsApp, webchat, and Feishu so the acknowledgement sends immediately while normal processing and final replies continue.
