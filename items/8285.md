## Summary

Keep #8285 open. Current main still has ack reactions, status reactions, and typing indicators, but no supported configurable pre-processing visible text acknowledgement that sends immediately and then allows normal agent processing to continue across Slack, WhatsApp, webchat, or Feishu. The latest issue discussion is concrete and already includes a same-day maintainer/Codex comment reaching the same keep-open conclusion.

## Best Possible Solution

Keep tracking this as a core/product or plugin-API feature request. The best path is to add a supported pre-processing text acknowledgement contract, either `messages.ackText`/scope with channel/account overrides or a documented non-terminal plugin hook, then update config types, zod schema, generated config docs, public docs, and channel tests so Slack, WhatsApp, webchat, and Feishu send the acknowledgement immediately while preserving normal processing and final replies.
