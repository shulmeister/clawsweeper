## Summary

Keep #8285 open. Current main still does not provide a supported configurable text acknowledgement before agent processing. The implemented surfaces cover emoji/status reactions, typing indicators, WebChat request acks, and terminal/handled plugin hooks, but not a cross-channel `messages.ackText`-style send-and-continue contract for Slack, WhatsApp, WebChat, Feishu, or other messaging channels.

## Best Possible Solution

Keep #8285 open as a core/product or plugin-API feature request. The best path is a supported non-terminal pre-processing acknowledgement contract, likely `messages.ackText` with scope and per-channel/account overrides, or a documented plugin/channel hook that can send interim text and then continue normal model dispatch. Implementation should update config types, zod schema, generated labels/help, public docs, and channel tests for the affected messaging surfaces.
