## Summary

Current main already fixes the proxy-only Gemini memory embedding failure through the shared memory-host HTTP helper shipped by #71506. Gemini direct and batch embedding requests now route through that helper, which opts into trusted env-proxy mode only when Undici would proxy the target, with tests and changelog coverage. This PR's Gemini-specific fetchImpl propagation is stale against the current plugin-based Google embedding surface.

## Best Possible Solution

Close #64095 as implemented by the merged #71506 shared memory-host SDK proxy-mode fix. Keep the helper-based implementation on current main and do not merge the stale Gemini-specific fetchImpl propagation PR; any future custom fetchImpl API drift should be handled as a separate targeted contract issue.
