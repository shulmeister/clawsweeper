## Summary

Keep open. Current main still only extracts simple markdown/div text from flat `elements` or `body.elements` for Feishu interactive cards, so the PR's quoted-card variants are not implemented away. The related #39339 was closed unmerged, and the current PR still has unresolved review feedback that needs maintainer/change review rather than cleanup.

## Best Possible Solution

Keep this PR open until a Feishu quote/read parser fix is merged. The best path is to update this PR or a canonical successor so `parseInteractiveCardContent` handles header titles, non-empty fallback across `elements`/`body.elements`/`i18n_elements`, primitive template variables, and nested card elements, with focused tests for each variant and the empty-array fallback cases.
