## Summary

Keep open. Current main still does not implement the PR's Feishu quoted interactive-card variant extraction: it only parses flat `elements` and `body.elements`, and the proposed PR still has unresolved fallback edge-case review comments that need maintainer/code review rather than conservative cleanup.

## Best Possible Solution

Keep PR #38776 open or replace it with a maintainer-owned Feishu parser fix that implements robust quoted interactive-card extraction for headers, `body.elements`, non-empty locale fallback, template primitive values, and nested card elements, with regression tests for each variant and the empty-array fallback cases raised in review.
