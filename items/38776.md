## Summary

Keep #38776 open. Current main still only partially handles quoted Feishu interactive cards: it parses top-level `elements` and `body.elements` for `markdown`/`div` text, but the PR's header, `i18n_elements`, `template_variable`, and nested `column_set`/generic element extraction remain unimplemented. The linked regression #32712 is relevant, and the PR discussion contains useful review feedback rather than evidence that the work is obsolete.

## Best Possible Solution

Keep #38776 open unless maintainers explicitly pick a broader canonical successor. The best path is a focused Feishu plugin parser update in `extensions/feishu/src/send.ts` with tests for header titles, non-empty fallback across `elements`/`body.elements`/`i18n_elements`, primitive template variables, nested card elements, and Feishu read API raw/post/card-reference shapes where applicable.
