## Summary

Keep open. Current `main` still uses a narrow Feishu interactive-card parser, so this PR's requested quoting support for additional card variants is not implemented on `main` and there is not strong evidence to close it under any allowed reason.

## Evidence

- **Current parser only handles flat elements and body.elements:** `parseInteractiveCardContent` in current `main` only reads `candidate.elements` or `candidate.body?.elements`, then extracts text from `div.text.content` and `markdown.content`. There is no handling for headers, `i18n_elements`, `template_variable`, `column_set`, or recursive nested `elements`.
