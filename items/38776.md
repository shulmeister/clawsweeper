## Summary

Keep #38776 open. Current main still only has partial Feishu interactive-card parsing for quoted messages: it handles top-level `elements` and `body.elements` with `div`/`markdown`, but the header, i18n, template-variable, column/nested, and raw/card-reference variants targeted by this PR are not implemented on main. Related broader parser attempts are not merged, so there is no shipped canonical replacement yet.

## Best Possible Solution

Keep #38776 open unless maintainers explicitly choose a canonical successor. The best path is a focused Feishu plugin parser update in `extensions/feishu/src/send.ts` with regression tests for header titles, non-empty fallback across `elements`/`body.elements`/`i18n_elements`, primitive template variables, nested card elements, and any raw/post fallback shapes that current Feishu APIs return.
