## Summary

Keep #38776 open. Current main still has only partial Feishu quoted interactive-card parsing: it handles flat `elements` and `body.elements` with `div`/`markdown` text, but it still lacks the header, `i18n_elements`, template-variable, `column_set`, and recursive nested-element variants this PR targets. The related broader parser attempts in the provided context are either closed unmerged or still open, so there is no shipped replacement on main.

## Best Possible Solution

Keep #38776 open until a Feishu quote/read parser fix lands, or until maintainers explicitly choose a canonical successor. The best implementation path is a focused Feishu plugin parser update in `extensions/feishu/src/send.ts` with regression tests for header titles, non-empty fallback across `elements`/`body.elements`/`i18n_elements`, primitive template variables, nested card elements, and the raw/post fallback shapes covered by newer related reports.
