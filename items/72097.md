## Summary

Keep this issue open. Current main still resolves plugin tools into a concrete run/session tool array and passes that fixed array into the embedded agent session; plugin install/registry refresh does not notify active sessions or re-resolve the tool list before the next model turn. The prior ClawSweeper keep-open comment remains consistent with current main, and related #44401 is adjacent GatewayRestart broadcast work rather than a fix for live toolset injection.

## Best Possible Solution

Implement this in the core plugin-runtime and embedded agent-session lifecycle: after successful plugin install or registry refresh, either broadcast a registry/tool change to active sessions or re-resolve plugin tools at the next safe model-turn boundary. Update effectiveTools, customTools, sessionToolAllowlist/setActiveToolsByName, prompt/tool inventory, and context-engine availableTools together, with regression coverage for install -> immediate use in an existing session and for registry reuse without duplicate plugin runtime bindings.
