## Summary

Keep open. Current `main` still sends Slack outbound text through mrkdwn normalization that escapes `&` to `&amp;`, and the Slack formatter tests explicitly assert that behavior. The relevant Slack send/format files are unchanged between release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`) and current `main` (`8866544ffefb42ca234c66c337134fb60743d8d0`), so there is no evidence this issue was implemented on `main`.
