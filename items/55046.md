## Summary

Keep open. Current main still has the reported ACP media-understanding gap: ACP dispatch invokes media understanding for inbound media but does not pass an agent directory or active model context, while provider-backed image understanding requires an agent directory and the normal reply path supplies one.

## Best Possible Solution

Keep this issue open. Fix ACP dispatch by resolving the effective ACP agent directory, likely from `resolvedAcpAgent`/`canonicalSessionKey` via the existing agent-scope helpers, and pass it into `applyMediaUnderstanding`; also pass active model context when ACP runtime options or configured agent model can provide it so native-vision skip behavior matches the normal reply path. Add a focused `dispatch-acp.test.ts` regression for an inbound image ACP turn with configured `tools.media.image`, asserting media understanding receives the agent directory and enriches the prompt before `runTurn`. Treat the Windows `OPENCLAW_AGENT_DIR` doctor expectation as a separate follow-up if it still reproduces after the ACP fix.
