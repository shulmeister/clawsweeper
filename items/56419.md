## Summary

Keep open. Current `main` still hard-codes a `30_000` ms timeout for explicit `image` tool requests in `src/agents/tools/image-tool.ts`, and `src/media-understanding/image.ts` still turns that into an OpenClaw-initiated abort via `AbortController`. I did not find a matching timeout override on the `image` tool surface or in `agents.defaults.*`, while the separate `tools.media.image.timeoutSeconds` config is only consumed by the media-understanding runner path. The linked open PR #56454 in the issue timeline matches the remaining gap rather than proving it is fixed.

## Evidence
