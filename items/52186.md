## Summary

Current `main` still treats non-explicit TTS as a fallback chain, so the remaining behavior request in #52186 is not implemented strongly enough to close. The active auto-TTS path still selects the configured provider first and then falls through to other registered providers on failure, which means an ElevenLabs-configured reply can still end up playing a fallback provider voice instead of hard-sticking to ElevenLabs.

## Evidence
