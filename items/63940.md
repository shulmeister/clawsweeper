## Summary

Keep this PR open. Current main still emits and tests the stricter skill-selection prompt that this PR changes: the prompt says to read exactly one skill only after selecting it, and the tests assert that wording. The PR remains a focused implementation candidate rather than an obsolete or already-shipped change.

## Best Possible Solution

Keep the PR open for maintainer review. If the relaxed skill-selection behavior is still desired, land or rebase the focused prompt/test change on current main, ideally while coordinating with the separate prompt-tool-selection work so the final Skills guidance is coherent.
