## Summary

Keep open. Current main still rethrows direct web_fetch network errors without URL or cause context after provider fallback misses, so the central bug fixed by this PR is not implemented. The PR is useful but should not land as-is because it bundles unrelated changes and has unresolved review blockers.

## Best Possible Solution

Keep the item open, but do not land this branch as-is. The best maintainer path is a focused web_fetch PR that wraps non-SSRF direct-fetch transport errors after provider fallback misses, preserves the original error as cause, includes the requested URL and root reason, and adds a regression test for a no-provider network failure. Split the unrelated Feishu, Telegram, config/model, package override, and tool-result fixes into separate PRs with their own review and validation.
