## Summary

Keep #39138 open. Current main still starts outbound delivery recovery on gateway activation and replays pending queue JSON entries without a stale-age cutoff, maximum replay count, or documented recovery policy. The source still permits a 15-hour-old first-replay entry with retryCount=0 and no lastAttemptAt to be delivered after restart.

## Best Possible Solution

Implement a conservative outbound delivery recovery policy in src/infra/outbound: expire or move stale pending deliveries to failed before replay, preserve recovery for recent crash-window sends, and add focused tests for old retryCount=0 entries, maxRecoveryMs interaction, and restart recovery. If maintainers want operator control, add documented config/schema for max age and possibly max replay count.
