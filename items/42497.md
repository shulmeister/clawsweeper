## Summary

Keep open. Current main still routes agent gateway tools through the WebSocket/RPC client path and does not contain this PR's local in-process dispatch helper. The linked bug tracker #40237 was already closed as superseded by this PR, so this PR is the canonical remaining item rather than obsolete cleanup. It likely needs rebase/rework for current scope and option semantics before merge.

## Best Possible Solution

Keep the PR open, but require a current-main rebase or adapted implementation: add an in-process dispatch path for embedded gateway-native tool calls, wire it through `dispatchGatewayMethod` or a successor API, and preserve current option/auth/scope/timeout/final-response semantics with regression tests for cron calls and the reviewed edge cases.
