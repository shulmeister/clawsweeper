## Summary

Current `main` already hardens macOS LaunchAgent restarts against self-triggered gateway restarts from within the managed process tree, matching the issue's reported failure mode.

## Evidence

## Close Comment

Closing from a Codex review: current `main` already implements the macOS LaunchAgent self-restart hardening this report asks for. In `src/daemon/launchd.ts:635`, `restartLaunchAgent()` detects when `openclaw gateway restart` is being invoked from the managed LaunchAgent process tree and hands the restart off to a detached helper instead of unloading the service in-process. That helper, implemented in `src/daemon/launchd-restart-handoff.ts:78`, waits for the caller PID to exit and then runs `launchctl enable` plus `kickstart -k` with bootstrap fallback outside the dying gateway tree. Regression coverage exists in `src/daemon/launchd.test.ts:649` and `src/daemon/launchd.test.ts:753`, and the shipped changelog documents this under `CHANGELOG.md:2672` / `CHANGELOG.md:2705` for `2026.3.11`. If you still hit a similar failure on a newer build, please open a fresh issue with the exact command path and restart logs.
