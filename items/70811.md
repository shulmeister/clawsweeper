## Summary

Keep PR #70811 open. Current main still validates `entry?.config` as `{}` in non-activating plugin snapshot and CLI metadata paths, so the linked false-positive invalid-config bug is not implemented on main. The PR is a recent, focused bug-fix attempt with regression coverage and active follow-up on review feedback, not obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. The likely best path is to land a scoped loader change that skips or defers validation only for non-activating loads with truly undefined plugin config, preserves normalized/defaulted `pluginConfig` for metadata registration, and keeps activation plus explicit-config paths validating, with the PR's focused loader regression tests retained.
