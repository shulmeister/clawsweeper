## Summary

Keep this PR open. Current main still has the behavior this PR targets: plugin config validation runs with `entry?.config` even in non-activating manifest-only and CLI metadata paths, and `validatePluginConfig` still converts `undefined` to `{}` before JSON Schema validation. The provided PR discussion shows recent focused follow-up work, no merged replacement, and #64946 appears related but not superseding because it describes wrapped/unwrapped schema disagreement rather than omitted config during preflight.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent loader change that skips schema validation only for non-activating loads when plugin config is actually `undefined`, preserves normalized/defaulted `api.pluginConfig` for non-activating registration, keeps activation and explicit-config validation strict, and retains focused regression coverage in `src/plugins/loader.test.ts` and `src/plugins/loader.cli-metadata.test.ts`.
