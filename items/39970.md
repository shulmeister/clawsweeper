## Summary

Keep this PR open. Current main has generic outbound `audioAsVoice` plumbing and the legacy iMessage plugin now declares text chunking, but the actual legacy iMessage outbound path still does not expose `sendPayload`, does not propagate `audioAsVoice` through `sendIMessageOutbound`, and still maps empty audio media to the generic `<media:audio>` placeholder in the low-level sender. The PR is stale against the `extensions/imessage` layout, but the underlying bug fix remains useful.

## Best Possible Solution

Keep the PR open and ask for a narrowed rebase onto `extensions/imessage`. The best fix is an iMessage-owned raw outbound adapter or explicit `sendPayload` path that carries `audioAsVoice` through `extensions/imessage/src/channel.ts`, `extensions/imessage/src/channel.runtime.ts`, and `extensions/imessage/src/send.ts`; suppresses the generic `<media:audio>` placeholder for voice-intent audio; sends visible caption text separately when needed; preserves chunking, reply-to handling, and echo-cache behavior; and adds iMessage-specific regression tests.
