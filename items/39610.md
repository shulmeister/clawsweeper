## Summary

Keep #39610 open. Current main still only supports `agents.defaults.userTimezone`; it does not expose or resolve `agents.list[].userTimezone` for agent config, schema validation, prompt/current-time paths, heartbeat active hours, envelope/system-event timestamps, or gateway timestamp injection. The linked issue #39579 is closed because this PR references it, but the PR is still open and unmerged, and no superseding implementation is present on main.

## Best Possible Solution

Keep #39610 open as the canonical implementation path for per-agent user timezone support. The maintainable direction is to add `agents.list[].userTimezone` to config types, zod/generated schema, and docs; centralize agent-aware timezone resolution; thread a config-aware `agentId` through every `"user"` timezone path; and refresh tests for system prompts, session status, heartbeat active hours and body injection, envelopes/system events, gateway/chat timestamp injection, memory flush, and post-compaction context.
