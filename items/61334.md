## Summary

Keep open. The original 4x/second report was corrected by the reporter, but the reopened/refined request is still a real behavior change request: current main deliberately clamps cron timer sleeps to 60 seconds, so far-future or daily schedules still re-arm once per minute instead of sleeping until the next scheduled job.

## Evidence

- **Current scheduler clamp:** Current main defines MAX_TIMER_DELAY_MS as 60_000 and computes clampedDelay = Math.min(flooredDelay, MAX_TIMER_DELAY_MS), then logs "cron: timer armed" with that clamped delay.
