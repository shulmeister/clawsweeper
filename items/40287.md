## Summary

Keep PR #40287 open. Current main still has no Slack `file_shared` event handler or setup/docs subscription for `file_shared`, so the central large-file fallback remains missing. Some adjacent work is already implemented on main, including Slack attachment types/content handling and app_mention race handling, but that does not replace the PR's main fallback path. The PR itself is stale against the current plugin layout because it targets `src/slack`, while Slack now lives under `extensions/slack`.

## Best Possible Solution

Keep the PR open, but ask for a rebase/port into `extensions/slack`. The best implementation is a corrected `file_shared` fallback in the Slack plugin that registers the event, hydrates the matching Slack message via Slack APIs, feeds it through the existing message pipeline, includes race-safe deduplication and bounded history lookup, updates generated/setup docs for the `file_shared` subscription, and adds focused unit tests for normal message/file_shared dedup and large-file fallback behavior.
