## Summary

Keep open. Current main still has the provider-agnostic Anthropic adaptive-thinking path that maps Opus 4.7 `high`/`xhigh` to `output_config.effort` and emits it without a GitHub Copilot provider gate. The bundled Copilot model metadata now mitigates the default catalog path by setting Copilot Claude models to `reasoning: false`, but configured reasoning-capable `github-copilot/claude-opus-4.7` metadata can still hit the reported payload shape. The directly linked fix PRs #69937 and #69944 were closed unmerged, and related #70376 was also closed unmerged.

## Best Possible Solution

Keep #69928 open and land a focused provider/model capability fix in the Anthropic Messages request-shaping path: for `github-copilot/claude-opus-4.7`, either omit `output_config.effort` or clamp unsupported efforts such as `high` and `xhigh` to `medium`, with regression tests for Copilot Opus 4.7 high/xhigh, direct Anthropic Opus 4.7 preservation, and an out-of-scope Copilot Claude guard.
