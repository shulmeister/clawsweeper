## Summary

Keep open. Current main still has the provider-agnostic Anthropic Messages adaptive-effort emission path: Opus 4.7 maps high/xhigh to unsupported non-medium efforts and `buildAnthropicParams()` emits `output_config.effort` without checking `github-copilot`. Current Copilot defaults appear to avoid some default sessions by marking bundled Copilot Claude models as `reasoning: false`, but that is a mitigation, not the requested provider/model compatibility fix, and no Copilot Opus 4.7 regression coverage has landed. The directly related fix PRs #69937/#69944/#70376 were closed unmerged, while #70322/#70584 are narrower low/minimal follow-ups and do not supersede the high/xhigh report.

## Best Possible Solution

Keep the issue open until OpenClaw lands a provider/model-aware adaptive-effort compatibility fix. The likely implementation belongs in `src/agents/anthropic-transport-stream.ts` or the GitHub Copilot provider wrapper: for `github-copilot` + `claude-opus-4.7`, omit `output_config.effort` or clamp unsupported values to `medium`, while preserving direct Anthropic Opus 4.7 behavior. Add regression tests for high, xhigh, low, and minimal on Copilot Opus 4.7 plus scope guards for direct Anthropic and Opus 4.6.
