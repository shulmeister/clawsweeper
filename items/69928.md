## Summary

Keep open. Current main partially mitigates the common built-in GitHub Copilot path by defining Copilot Claude models with `reasoning: false`, but it does not implement the reported provider/model effort compatibility fix. The Anthropic Messages transport still treats Opus 4.7 adaptive effort as model-only behavior and emits `output_config.effort` without a GitHub Copilot gate whenever the model is reasoning-enabled. The exact fix PRs for #69928 were closed unmerged, and related Copilot Opus 4.7 effort work remains open/unshipped.

## Best Possible Solution

Keep #69928 open until a maintainer either lands a scoped Copilot Anthropic Messages compatibility fix or explicitly decides that Copilot Claude reasoning should remain disabled. The best code path is a provider/model effort capability rule or helper that omits/clamps `output_config.effort` for `github-copilot/claude-opus-4.7`, with regression tests proving Copilot Opus 4.7 behavior and direct Anthropic Opus 4.7 scope guards.
