## Summary

Current main already implements the requested safeguard: pruning/truncation paths re-run tool-use/result repair, the repair helper drops orphaned tool results, and regression tests cover the Anthropic replay and history-pruning cases. The same implementation is present in the shipped `v2026.4.22` tree.

## Close Comment

Closing as `implemented_on_main` after a Codex review. Current main already hardens this exact failure mode in the active pruning/replay paths: history pruning re-runs tool-use/result repair (`src/agents/compaction.ts:542`), embedded-runner truncation re-runs pairing repair after `limitHistoryTurns` (`src/agents/pi-embedded-runner/compact.ts:958`, `src/agents/pi-embedded-runner/run/attempt.ts:1693`), and the shared repair helper drops orphaned/free-floating tool results (`src/agents/session-transcript-repair.ts:456`). I also verified the same fix surface is present in the shipped `v2026.4.22` release tree, so this issue appears already fixed.
