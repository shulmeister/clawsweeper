## Summary

Keep open. Current main still hard-codes discovery of the personal `~/.agents/skills` source and has no source-level disable option in the config type or strict Zod schema. Existing agent skill allowlists and prompt-size limits reduce prompt exposure, but they do not stop the personal source from being scanned, which is the central request in this issue. The sole discussion comment also supports the short-term config-toggle direction, and the related ClawSweeper items are about skill source classification or symlink handling rather than this missing source-disable control.

## Best Possible Solution

Keep this issue open and implement a small core skills-loader configuration surface, such as `skills.load.disableSources: ["agents-skills-personal"]` or an explicit personal-scan boolean. The change should be applied before each source is loaded, update config types/Zod schema/generated docs, and add regression tests proving the personal source is skipped while existing agent allowlists and precedence behavior still work.

## Risks / Open Questions

- A source-level toggle needs careful naming and docs because current precedence and tests treat personal/project `.agents/skills` roots as built-in behavior.
- Existing `agents.defaults.skills` allowlists may be an acceptable workaround for prompt bloat, but they do not satisfy the request to stop scanning a specific source.
