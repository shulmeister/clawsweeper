## Summary

Keep PR #42330 open for maintainer follow-up or explicit superseding. Current main already has a stronger generic inbound dedupe path that covers Google Chat message IDs, but the PR's other requested behavior is not fully implemented on main: Google Chat still defaults reply threading to off, lifecycle add-on payload handling is still covered by open focused work, and DM/space classification still has a related open gap. The submitted PR also is not safe to merge as-is given the review comments, so no conservative close reason fully applies.

## Best Possible Solution

Do not merge #42330 as-is. Keep the already-shipped shared inbound dedupe path on main, use focused follow-ups such as #58993 for spaceType classification and #65058 for add-on lifecycle payload variants, and make a separate maintainer-backed decision on whether Google Chat should default replyToMode to all. Once those narrower paths are accepted or rejected, this broad PR can be explicitly superseded.
