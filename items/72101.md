## Summary

Keep #72101 open. Current main at e672b614 still has the two behaviors this PR fixes: `sessions_spawn` rejects `runtime="subagent"` calls that include ACP-only `streamTo`, and it forwards `model:"default"` as a literal subagent model override. The related #68397 is closed unmerged and only overlaps the ACP-only-field half, so it does not supersede this PR's model-normalization fix.

## Best Possible Solution

Keep #72101 open, or land an equivalent agents-tool patch that ignores ACP-only `streamTo` for `runtime="subagent"`, normalizes empty/`default` `sessions_spawn.model` input to no override via shared helper logic, preserves ACP behavior, and updates the sessions_spawn/session_status regression coverage. Consider whether to include the related `resumeSessionId` behavior from #68397 before merging.
