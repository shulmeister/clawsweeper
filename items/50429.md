## Summary

Keep open. Current main still sends non-streaming Feishu card replies without a plain-text fallback when `sendStructuredCardFeishu()` rejects, so this PR is not obsolete. The PR is also not cleanly closeable as duplicate/superseded because it is the active implementation attempt for #43690, and the latest PR commit addressed the Greptile double-conversion finding. Remaining review concerns about multi-chunk duplication and block payload fallback mean the best path is a revised version of this PR or equivalent maintainer patch, not auto-close.

## Best Possible Solution

Keep the PR open and ask for a small revision: preserve the single table conversion fix already made, add structured-card failure fallback only for safe final payloads, and handle partial multi-chunk card delivery without resending chunks that already succeeded. Then add focused tests for first-chunk failure, later-chunk failure, block payload suppression, and the single `convertMarkdownTables()` call.

- The PR's intended fallback is still useful, but landing it as-is could duplicate already-delivered card chunks if chunk 2+ fails and the fallback resends the entire reply as text.
