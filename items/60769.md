## Summary

Current main does not implement the reported fix. The configured-model fallback path still returns the config unchanged for a single selected model before it can detect that the single selection is the current primary, so stale fallbacks remain possible. The related PR #60809 directly targeted this issue but was closed unmerged, and its referenced commits are not present in this checkout.

## Evidence
