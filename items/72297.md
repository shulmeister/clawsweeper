Keep this PR open. Current main still has the QMD name-mismatch gap from the linked bug, and the PR is an active focused implementation candidate. The latest PR update also addresses the earlier stderr-only removal concern by requiring matching refreshed collection metadata before rebinding.

## Best Possible Solution

Keep this PR open and review the latest `c6c4e17ce95ffdcc9be5dd028ed695973b109ca6` patch as the implementation candidate for the linked QMD collection name-mismatch bug. The best path is to land a fix that rebinds only when QMD provides authoritative matching path/pattern metadata for the stale collection name, keeps refusing stderr-only destructive removal, and retains regression coverage for both the verified-rebind case and the name-only/manual-repair case.
