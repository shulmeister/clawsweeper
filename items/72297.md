## Summary

Keep PR #72297 open. Current main still cannot rebind a QMD path-pattern conflict when `collection list` only returns names and the stale collection name is only present in the `qmd collection add` error. The PR is a focused draft bug fix with regression coverage, and the existing Codex bot comment reported no major issues. Related QMD compatibility work does not supersede this narrower case.

## Best Possible Solution

Keep the PR open for normal review. If the exact QMD error format is accepted, merge the focused parser plus regression test so memory-core can rebind stale path-pattern conflicts even when `collection list` lacks metadata.
