## Summary

Current main no longer treats `agents.defaults.model.primary` as the thing that narrows the chat picker. `models.list` only narrows when `agents.defaults.models` is explicitly populated, and configure now exposes a provider-scoped multi-select for that allowlist so Ollama users can persist multiple discovered local models without hand-editing JSON.

## Best Possible Solution

Keep the current main behavior: `agents.defaults.models` remains the explicit picker allowlist, `agents.defaults.model.primary` remains just the default model, and configure is the supported way to select multiple Ollama models without hand-editing JSON. Users with an old single-entry allowlist should rerun `openclaw configure --section model` and select the desired Ollama entries, or clear the allowlist to show the full discovered catalog.

## Evidence

- **Primary model alone does not collapse the catalog:** The allowed-model resolver reads `agents.defaults.models`; when that map is empty it sets `allowAny` and returns the full catalog. This means `agents.defaults.model.primary` by itself does not restrict the chat picker on current main.
