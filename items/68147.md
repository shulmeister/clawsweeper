## Summary

Keep open. I did not find a strong basis to close this conservatively: current main still treats `agents.defaults.models` as the effective chat/model allowlist, the default-model setter still writes the selected primary into that map, and `models.list` still feeds the chat picker from that filtered catalog. I also did not find a main-vs-release change that clearly fixes the reported single-model behavior for qualified Ollama refs like `ollama/gemma4:e4b`.
