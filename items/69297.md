## Summary

Keep #69297 open. Current main still does not implement WhatsApp per-group admin privacy controls: the config type/schema reject `channels.whatsapp.groups.*.admin`, runtime policy has no configured-admin path, group ingress still blocks allowlist misses before command handling, and docs describe only the existing allowlist behavior. The linked issue #69233 was closed in favor of this PR as the canonical implementation thread, not because the behavior shipped.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a focused WhatsApp plugin change that adds `groups.<jid>.admin` to the config type, schema, generated metadata, UI hints, and docs; applies the admin exception consistently in mention gating, command authorization, real group allowlist ingress, activation, and ack reactions; preserves `groupPolicy: "disabled"`; covers wildcard and exact-group fallback semantics; and splits unrelated test/runtime cleanup unless maintainers intentionally want it bundled.
