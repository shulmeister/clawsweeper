## Summary

Keep open. Current main already has partial installer hardening for npm failures, but the issue's requested cleanup is not implemented end-to-end: Linux `install.sh` extracts npm error metadata, auto-fixes some permission/build-tool cases, and retries once, while `install.ps1` still only reports a raw exit code and `scripts/docker/setup.sh` still masks `config get` failures with `|| true`. That is not strong enough evidence to close as implemented or obsolete.

## Evidence

- **Structured npm diagnostics exist in the Bash installer:** `scripts/install.sh` parses npm error code/syscall/errno, prints installer/debug-log paths, and adds special handling for build-tool, `ENOTEMPTY`, and `EEXIST` failures.
  - file: [scripts/install.sh:729](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/scripts/install.sh#L729)

  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Linux installer already auto-fixes one permissions case:** `fix_npm_permissions()` switches npm to `~/.npm-global` when the global prefix is not writable, which covers part of the issue's `EACCES` concern.
  - file: [scripts/install.sh:1562](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/scripts/install.sh#L1562)
