## Summary

Current main still has the reported UX bug: approval-not-found errors are mapped back to the generic `❌ Failed to submit approval: unknown or expired approval id` reply, and Telegram callback resolution failures are rethrown without stale/restart-specific guidance. The related narrow fix PR #67982 was closed unmerged, so this issue remains actionable.

## Best Possible Solution

Keep this issue open. The best narrow fix is to keep approval invalidation semantics unchanged for now, but translate `isApprovalNotFoundError(...)` at the user-facing approval surfaces into guidance that the request is no longer available, may have expired or been invalidated by a gateway restart, and should be re-requested. Add focused coverage in `src/auto-reply/reply/commands-approve.test.ts` and the Telegram callback tests. Durable approval persistence can remain a separate product decision if maintainers want restart-surviving approvals later.
