## Summary

Keep #38364 open. Current main still has no structured reply-to metadata path from Control UI or `chat.send`/`sessions.send` into the agent `MsgContext`. The no-output side is partially mitigated by `NO_REPLY` filtering in Control UI and Gateway history, but the central reply metadata request remains unimplemented.

## Best Possible Solution

Keep #38364 open. Add an additive reply metadata object to `chat.send` and likely `sessions.send`, capture selected parent/reply-target state in Control UI, validate/map it in Gateway into existing `MsgContext.ReplyTo*` fields plus an optional parent-content summary, and cover protocol validation, Gateway mapping, and Control UI request behavior with focused tests.
