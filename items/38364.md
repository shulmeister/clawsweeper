## Summary

Keep open. Current main partially addresses the confusing no-output path by suppressing empty, whitespace-only, and NO_REPLY-only assistant output in Control UI/history, but the main reply-to metadata request remains unimplemented: chat.send has no reply/parent field, the Control UI send path does not collect or forward one, and the gateway agent context is still built from plain message text plus routing/thread fields rather than a replied-to message id or parent summary.

## Best Possible Solution

Keep #38364 open. The best implementation path is an additive Gateway protocol change for chat.send reply metadata, Control UI state/actions that capture the selected parent/replied-to message, gateway mapping into existing MsgContext reply fields plus an optional parent-content summary, and focused tests/docs covering both browser send payloads and agent-visible context. The existing silent-output filtering should be kept as the partial fix, but it does not close the reply metadata request.
