## Summary

Current main still only partially addresses #38364. Silent NO_REPLY/no-output messages are filtered in the Control UI and gateway history/event paths, but the central reply-to metadata request remains unimplemented for Control UI chat sends: the protocol has no reply/parent field, the UI send path has no reply-target state or parameter, and the gateway-created MsgContext does not populate the existing reply fields for chat.send messages.

## Best Possible Solution

Keep #38364 open. The best product/code path is an additive chat.send protocol field for reply metadata, Control UI composer/thread state that captures the selected parent message, gateway mapping into the existing MsgContext reply fields plus an optional parent-content summary, and focused protocol/UI/server tests. Keep the current silent-output filtering as a partial fix.
