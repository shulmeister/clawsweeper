## Summary

Keep open. Current main still uses a plain `systemctl --user restart <unit>` for Linux/systemd gateway restarts and still ships `KillMode=control-group`, so the PR's reported self-restart SIGTERM failure is not implemented or superseded on main. The PR is recent, focused, and the discussion shows active author follow-up on the bot-review concerns rather than an obsolete or low-signal cleanup candidate.

## Best Possible Solution

Keep the PR open for maintainer review. The product path is to land a Linux/systemd-specific self-restart handoff in `src/daemon/systemd.ts`, pair it with bounded SIGTERM/restart-marker handling in `src/cli/gateway-cli/run-loop.ts`, and keep the targeted daemon/run-loop regression tests so external restarts remain synchronous while in-unit restarts return `scheduled`.
