## Summary

Keep open. Current `main` still has the reported Discord DM inbound routing bug: `ctx.To` is derived from a channel-form `replyTarget` before DM semantic correction, while downstream code still consumes `ctx.To`. The linked PR #68144 contained a targeted fix and regression test, but it was closed unmerged for queue hygiene rather than because the report was invalid or already fixed.

## Evidence

- **Current main still assigns DM `ctx.To` from channel-form reply target:** `processDiscordMessage` still computes `effectiveTo = autoThreadContext?.To ?? replyTarget` and later writes `To: effectiveTo` into the inbound context. On DMs, that leaves `ctx.To` channel-addressed unless some earlier thread context overrides it.
