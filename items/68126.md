## Summary

Keep open. Current main still computes Discord DM inbound `ctx.To` from the channel-form reply target before the DM user target is resolved, then writes that value into the inbound context. The linked PR #68144 proposed the narrow fix and regression test, but it was closed unmerged for queue hygiene, not because the bug was invalid or already fixed.

## Best Possible Solution

Keep #68126 open. The best fix is the narrow change from PR #68144: compute `dmConversationTarget` before `effectiveTo`, set `effectiveTo = autoThreadContext?.To ?? dmConversationTarget ?? replyTarget`, keep `deliverTarget` channel-addressed for Discord API delivery, and add a regression test asserting DM inbound `ChatType` is `direct` and `ctx.To` is `user:<authorId>`.
