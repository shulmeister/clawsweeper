## Summary

Keep open. Current main still contains the exact `sessions_send` mixed-selector rejection that PR #39551 removes, the latest release tag also contains that guard, and no related item in the provided context is merged or clearly selected as the canonical replacement.

## Best Possible Solution

Keep #39551 open until maintainers land one canonical fix for the `sessions_send` mixed-selector bug. The best code path is a narrow executor-side precedence rule in `src/agents/tools/sessions-send-tool.ts`: normalize a non-empty `sessionKey` as authoritative, ignore redundant `label` and `agentId` for target lookup, preserve existing label-only behavior and downstream visibility checks, and keep or add the regression test proving mixed inputs send to the `sessionKey` target without calling `sessions.resolve`.
