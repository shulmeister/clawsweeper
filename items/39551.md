## Summary

Keep open. Current main still hard-rejects `sessions_send` calls that provide both `sessionKey` and `label`, while this PR is a narrow fix for that live behavior with focused regression coverage. Related PRs overlap, but the provided context does not identify a merged fix or maintainer-selected canonical replacement.

## Best Possible Solution

Keep #39551 open until maintainers choose and land one canonical fix for the live `sessions_send` mixed-selector bug. The best implementation is the narrow executor-side precedence rule proposed here: normalize non-empty `sessionKey` as authoritative, ignore redundant `label` and `agentId` for target lookup, preserve label-only behavior and downstream visibility/A2A checks, update the tool description, and keep regression coverage proving mixed inputs send to the `sessionKey` target without calling `sessions.resolve`.
