## Summary

Current `main` already routes `openclaw agent --agent <id>` through the target agent session, resolves `agents.list[].tools` from that session key, and applies deny-first tool filtering across agent, group, sandbox, and subagent policy layers. The reported `tools.deny` gap for daemon/CLI agents is not reflected in the current codebase.

## Evidence

- **CLI `--agent` resolves to the agent session key:** `openclaw agent` computes a `sessionKey` from `--agent`, `--to`, or `--session-id` before calling the gateway agent method, so daemon/CLI turns are scoped to a concrete agent session.
