## Summary

Keep PR #40443 open. Current `main` does not contain the PR's explicit reload/discard semantics: `loadConfig` has no options parameter, `applyConfigSnapshot` still preserves dirty form state unconditionally, and all visible Reload Config call sites still call `loadConfig(state)` without a discard path. The linked bug #40352 remains open, and the PR discussion shows a focused bug fix with review feedback addressed rather than a stale low-signal drive-by.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it onto current `main`, preserve the explicit reload vs passive refresh split in `ui/src/ui/controllers/config.ts`, keep the Agents/Config/Channels reload call sites wired to discard pending edits, and retain focused regression tests for form-mode dirty state, raw-mode dirty state, config hash-conflict recovery, and the Agents model dropdown rerender.
