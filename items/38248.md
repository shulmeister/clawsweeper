## Summary

Keep #38248 open. Current main still has model pricing metadata, usage/cost reporting, provider spend-limit error classification, and billing/quota cooldowns, but it does not implement the requested native request-level rolling spend ceiling for fallback chains. The discussion's external mitigations such as vibe-billing, Kalibr routing, and LiteLLM virtual keys remain workarounds, and PR #72335 was closed unmerged as a README-only submission in the provided context.

## Best Possible Solution

Keep the issue open until OpenClaw either ships a native request-level cost guard in the model/fallback path or explicitly designs a documented blocking plugin/API seam for cross-provider spend enforcement. A strong implementation would add config, hourly/daily accounting, pre-attempt estimated-cost checks, post-response reconciliation from usage/cost data, pause/reject behavior, owner notification, and fallback-chain regression coverage.
