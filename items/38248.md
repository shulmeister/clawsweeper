## Summary

Keep #38248 open. Current main still has cost estimation, usage summaries, provider spend-limit error classification, cooldown probes, and external LiteLLM spend-limit guidance, but it does not implement a native request-level rolling spend ceiling such as `agents.defaults.costGuard.maxSpendPerHour` or `maxSpendPerDay`. The issue is concrete, not protected by maintainer labels, and the prior comments identify useful external mitigations (`npx vibe-billing setup`, Kalibr routing), but those do not supersede the requested OpenClaw failover safety guard.

## Best Possible Solution

Keep the issue open until OpenClaw either adds a native cost guard in the model request/failover path or maintainers intentionally define and document a plugin/API seam that can enforce rolling spend ceilings before provider calls. The likely product path is a strict config surface under agent/model defaults, shared cost accounting over recent windows, pre-attempt estimated-cost checks, post-response reconciliation when providers return usage/cost, and clear pause/reject plus owner notification behavior.
