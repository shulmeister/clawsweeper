## Summary

Keep open. Current main still lacks a native request-level rolling spend ceiling for model fallback. Existing code can estimate and aggregate costs after usage is recorded, and docs point to external LiteLLM spend limits, but there is no `agents.defaults.costGuard` config or pre-call pause/reject gate in the fallback path.

## Best Possible Solution

Keep this open for a maintainer-approved core safety design. A strong fix would add an `agents.defaults.costGuard`-style config surface or equivalent, maintain rolling per-hour/day spend accounting in the request path, check the ceiling before each primary/fallback model call, support pause/reject behavior and owner notification, and add config docs plus fallback-chain regression tests.
