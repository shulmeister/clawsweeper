## Summary

PR #10356 should remain open. Current main still has no Typecast TTS implementation, and the PR discussion includes an explicit maintainer-posted Codex review choosing to keep it open. The current speech-provider plugin API makes a Typecast plugin technically plausible, but that product direction does not make this active PR safe to close through cleanup automation, especially with unresolved voice-note and telephony review risks.

## Best Possible Solution

Keep PR #10356 open for maintainer product and runtime review. The best path is either to continue it as a bundled Typecast speech-provider plugin after fixing the voice-note compatibility and telephony sample-rate concerns, or to ask the author to extract it into a ClawHub/plugin implementation using registerSpeechProvider.
