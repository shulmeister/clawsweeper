## Summary

Keep open. Current main still default-enables the bundled Bonjour gateway discovery plugin, still resolves @homebridge/ciao to 1.3.6, has no local ciao patch, and ciao 1.3.6 still runs the reported Windows `arp -a | findstr` command every 15 seconds without a `windowsHide` option. Existing docs/tests cover disablement workarounds, not a fix for the default Windows console flash.

## Best Possible Solution

Keep #70197 open and fix the bundled Bonjour/ciao path. The best maintainer path is to consume or upstream a `@homebridge/ciao` fix that passes `{ windowsHide: true }` for the Windows ARP probe; if that cannot land quickly, add a narrow dependency patch for the locked ciao version and verify on Windows that gateway Bonjour advertising no longer spawns visible `cmd.exe`/`conhost.exe` windows. Keep `OPENCLAW_DISABLE_BONJOUR=1`, `openclaw plugins disable bonjour`, and `discovery.mdns.mode="off"` documented as workarounds, not as the final fix.
