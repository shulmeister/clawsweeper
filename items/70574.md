Keep #70574 open. Current main still contains the reported cross-agent accountId overwrite path, and the linked fix PR #70607 is still open/unmerged in the provided GitHub context rather than shipped on main.

## Best Possible Solution

Keep the issue open until a fix lands on main. The likely fix belongs in `src/agents/spawn-requester-origin.ts`: preserve the parent/requester accountId for cross-agent spawns, keep any intended same-agent/thread-binding behavior explicit, and add regression coverage that Agent A spawning Agent B preserves Agent A's accountId while nested `streamTo="parent"` and self-spawn cases still route correctly.
