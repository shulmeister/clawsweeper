## Summary

Keep PR #62682 open. Current main still lacks the terminal-abort contract proposed by the PR: model fallback does not inspect AbortSignal.reason, HTTP disconnect aborts are untagged, and the main runWithModelFallback callers do not pass an abort signal into the fallback layer. Related PR #52365 remains open and only covers the cron budget pre-attempt angle, while merged PR #54388 propagated HTTP disconnect aborts but did not add the ClientDisconnectError reason handling from this PR.
