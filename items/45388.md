## Summary

Keep open. Current main still does not implement the reported live-tail behavior for `openclaw tui --session ...` against external Telegram sessions: the TUI does not subscribe to transcript `session.message` events, its chat event type has no inbound user-message state, and gateway tests intentionally preserve suppression of chat/node session events for non-Control-UI-visible external-channel runs.

## Evidence

- **TUI event wiring lacks transcript subscription handling:** The TUI websocket event handler only dispatches `chat`, `chat.side_result`, and `agent` events. It does not handle `session.message` or `sessions.changed`, which are the current gateway transcript-event surfaces for live session message appends.
  - file: [src/tui/tui.ts:1054](https://github.com/openclaw/openclaw/blob/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071/src/tui/tui.ts#L1054)
  - command: `nl -ba src/tui/tui.ts | sed -n '900,1160p'`
  - sha: [28497515fe1e](https://github.com/openclaw/openclaw/commit/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071)
- **ChatEvent still has no inbound user-message state:** The TUI `ChatEvent.state` union is still limited to `delta | final | aborted | error`, so current main does not contain the `user-message` event shape proposed in the linked PR #45417 for live inbound external messages.
  - file: [src/tui/tui-types.ts:21](https://github.com/openclaw/openclaw/blob/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071/src/tui/tui-types.ts#L21)
  - command: `nl -ba src/tui/tui-types.ts | sed -n '1,120p'`
  - sha: [28497515fe1e](https://github.com/openclaw/openclaw/commit/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071)
- **Current TUI workaround reloads history only after non-local final events:** Current TUI code calls `maybeRefreshHistoryForRun()` when a non-local `chat` final arrives, and the regression test asserts history reload after a non-local chat final. That can resync after a run final, but it is not live inbound message streaming.
  - file: [src/tui/tui-event-handlers.ts:355](https://github.com/openclaw/openclaw/blob/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071/src/tui/tui-event-handlers.ts#L355)

- **Gateway intentionally suppresses chat events for external/non-Control-UI-visible runs:** The gateway agent-event regression test explicitly expects zero chat broadcasts and no node session sends when a run is registered with `isControlUiVisible: false`, which matches Telegram/external-channel runs and leaves the issue's live stream path unresolved.
  - file: [src/gateway/server-chat.agent-events.test.ts:1367](https://github.com/openclaw/openclaw/blob/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071/src/gateway/server-chat.agent-events.test.ts#L1367)

- **Gateway has a session.message event surface, but TUI is not using it:** Gateway transcript updates can broadcast `session.message` to `sessions.messages.subscribe` subscribers. That supports a likely fix path, but current TUI code does not subscribe to or render this event.
