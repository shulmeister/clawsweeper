## Summary

Current main still does not implement configurable auto-cleanup or archival for hook/webhook sessions. Hooks are dispatched as isolated cron runs, but the hook path does not set a cleanup policy, the hooks config/schema has no cleanup surface, subagent archival remains scoped to subagents, and `sessions.list` still treats recent sessions by `updatedAt` rather than hook completion state.

## Best Possible Solution

Keep this issue open. The product fix belongs in the core hook/cron session lifecycle: add a maintainer-approved hook session cleanup policy surface, wire it through hook dispatch and isolated cron finalization, decide transcript-retention semantics, update config schema/help/docs, and add focused tests for Gmail/per-hook session keys, delayed cleanup, and `sessions.list` behavior.
