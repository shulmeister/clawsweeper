## Summary

Keep open. Current main has generic session-store maintenance, but it does not implement a hook/webhook-specific completion or delayed archive policy, does not make hook sessions respect `agents.defaults.subagents.archiveAfterMinutes`, and `sessions.list` still treats recent sessions by `updatedAt` rather than hook run completion state.

## Evidence

- **Hook config has no cleanup policy surface:** The public hooks config defines ingress/session routing fields (`enabled`, `path`, `token`, `defaultSessionKey`, `allowRequestSessionKey`, `allowedSessionKeyPrefixes`, `mappings`, `gmail`, `internal`) but no `sessionCleanup`, mode, delay, or transcript-retention setting.
  - file: [src/config/types.hooks.ts:93](https://github.com/openclaw/openclaw/blob/1dfa52d0711dfc86e1c04538fe46e3d43961766f/src/config/types.hooks.ts#L93)
  - command: `nl -ba src/config/types.hooks.ts | sed -n '1,240p'`
  - sha: [1dfa52d0711d](https://github.com/openclaw/openclaw/commit/1dfa52d0711dfc86e1c04538fe46e3d43961766f)
- **Config schema rejects unknown hook cleanup fields:** The root hooks zod schema is strict and lists the accepted hook fields; there is no cleanup/archive field under `hooks`. A repository search for `sessionCleanup` found no implementation.
  - file: [src/config/zod-schema.ts:644](https://github.com/openclaw/openclaw/blob/1dfa52d0711dfc86e1c04538fe46e3d43961766f/src/config/zod-schema.ts#L644)
  - command: `nl -ba src/config/zod-schema.ts | sed -n '620,670p'; rg -n "sessionCleanup|hook session|hook sessions|webhook sessions" . --glob '!extensions/diffs/assets/viewer-runtime.js'`
  - sha: [1dfa52d0711d](https://github.com/openclaw/openclaw/commit/1dfa52d0711dfc86e1c04538fe46e3d43961766f)
- **Hook dispatch creates isolated cron sessions without post-run archival:** Webhook agent hooks are converted into an isolated cron job and dispatched through `runCronIsolatedAgentTurn`; after the awaited result, the handler only summarizes/delivers status and does not archive, delete, or mark the hook session complete.
  - file: [src/gateway/server/hooks.ts:43](https://github.com/openclaw/openclaw/blob/1dfa52d0711dfc86e1c04538fe46e3d43961766f/src/gateway/server/hooks.ts#L43)

- **Isolated hook/cron run persists sessions instead of cleaning them on completion:** The isolated run path forces a new cron session for `sessionTarget === "isolated"` and persists the session entry after finalization; no completion cleanup policy is applied in this path.
  - file: [src/cron/isolated-agent/run.ts:480](https://github.com/openclaw/openclaw/blob/1dfa52d0711dfc86e1c04538fe46e3d43961766f/src/cron/isolated-agent/run.ts#L480)
