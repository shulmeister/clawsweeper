## Summary

Keep open. Current `main` does not implement this PR’s internal-tool SSE/output-item behavior, and the fixes discussed in review remain on the PR branch rather than the checked-out `main` commit.

## Evidence

- **Streaming path still ignores tool events:** The `/v1/responses` SSE handler on current `main` subscribes to agent events but only handles `assistant` and `lifecycle` streams. There is no `tool`-stream branch that could emit internal `function_call` or `function_call_output` output items.

- Closing this PR as implemented would be incorrect: current `main` still cannot stream internal tool execution as OpenResponses output items.
