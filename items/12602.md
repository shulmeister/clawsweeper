## Summary

Keep #12602 open. Current main has shipped major Slack rich-output slices: raw authored `channelData.slack.blocks`, Slack Block Kit sending, shared `presentation` rendering for sections/context/dividers/buttons/selects, and message-tool block payloads. The issue discussion explicitly kept this as an umbrella after those slices, and current main still has no native Slack table-block conversion path: `markdown.tables: "block"` is still coerced to `code`, with tests asserting that behavior.

## Best Possible Solution

Keep #12602 open as the umbrella for the remaining Slack rich-output work unless a maintainer explicitly supersedes it with a narrower canonical issue. The likely implementation path is in the Slack plugin: wire the existing markdown table seam into native Slack table/rich Block Kit rendering end to end, add focused Slack outbound tests, and update the public message-presentation or markdown-formatting docs. Do not treat raw `channelData.slack.blocks` or shared `presentation` support as missing; those parts are already shipped.
