## Summary

Keep #12602 open. Current main has shipped major Slack Block Kit slices: authored `channelData.slack.blocks`, Block Kit delivery through `sendMessageSlack`, shared `presentation` rendering, and interactive reply directives. The remaining umbrella gap is still real: `markdown.tables: "block"` is still coerced to `code`, tests assert that behavior, and I found no Slack table-block sender/renderer on current main. The latest provided maintainer discussion also explicitly kept this issue open until that richer/table rendering work is implemented or superseded.

## Best Possible Solution

Keep #12602 open as the umbrella for the remaining Slack rich-output work unless a maintainer explicitly supersedes it with a narrower canonical issue. The likely path is in the Slack plugin: wire the existing markdown table metadata seam into native Slack table/rich Block Kit rendering end to end, add focused outbound tests, and update the public message-presentation or markdown-formatting docs. Treat raw `channelData.slack.blocks`, shared `presentation`, and interactive reply directives as already shipped.
