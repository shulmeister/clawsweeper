## Summary

Keep #12602 open. Current main implements raw Slack Block Kit payload delivery plus shared presentation and interactive button/select rendering, but the remaining native Slack markdown-table/rich Markdown delivery path is still not wired: `markdown.tables: "block"` is accepted by the shared markdown seam and then coerced to `code` before Slack outbound rendering. The latest provided issue discussion also includes a maintainer/Codex review explicitly keeping this umbrella open until that remaining work lands or is superseded by a narrower canonical issue.

## Best Possible Solution

Keep #12602 open unless maintainers explicitly move the remaining scope to #57708 or another canonical follow-up. The remaining implementation belongs in the bundled Slack plugin: carry `markdownToIRWithMeta` table metadata through outbound Slack delivery when `markdown.tables: "block"` is configured, render supported markdown tables/rich Markdown as Slack Block Kit table or rich blocks, add focused Slack outbound tests and public docs, and avoid duplicating the already-shipped raw `channelData.slack.blocks`, shared `presentation`, and interactive directive support.
