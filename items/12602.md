## Summary

Keep open. Current main has implemented major slices of Slack rich output: authored `channelData.slack.blocks`, shared presentation rendering to Slack Block Kit, and interactive reply buttons/selects. But the issue discussion explicitly kept this as an umbrella pending table/richer rendering work, and current main still coerces Slack `markdown.tables: "block"` to code rather than sending native Slack table blocks.
