## Summary

Keep #12602 open. Current main implements authored Slack Block Kit payloads, shared presentation rendering, and interactive buttons/selects, but the remaining Slack native table/rich markdown delivery path is still missing: `markdown.tables: "block"` is accepted in the schema/IR layer but coerced back to text/code rendering before Slack delivery.

## Best Possible Solution

Keep #12602 open unless a maintainer explicitly supersedes it with a narrower canonical Slack rich markdown/table issue. The concrete implementation belongs in the bundled Slack plugin: carry `markdownToIRWithMeta` table metadata through outbound Slack delivery when `markdown.tables: "block"` is configured, render supported markdown tables as Slack Block Kit table/rich blocks, add focused Slack outbound tests and public docs, and avoid duplicating the already-shipped raw `channelData.slack.blocks`, shared `presentation`, and interactive directive support.
