## Summary

Keep open. Current main has backend abort support, including `chat.abort` without a runId for active agent runs, so part of the original "no API" claim is now partially addressed. The central UI regression is still not implemented, though: the Control UI only shows Stop when local `chatRunId` is set, and the reconnect/hello path explicitly clears `chatRunId`, which can remove the Stop button after a gateway restart/reconnect.

## Evidence
