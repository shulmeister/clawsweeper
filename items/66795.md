## Summary

Keep PR #66795 open. Current main still has the exact Telegram model-picker bug described by the PR: callback session reconstruction routes to the correct agent, but when there is no stored model override it falls back to the global `agents.defaults.model`, so the picker can fail to mark the routed agent's `agents.list[].model` default as current. The PR discussion and Greptile summary describe a focused fix plus regression test, and current main does not contain that regression coverage.

## Best Possible Solution

Keep the PR open or land an equivalent maintainer patch: compute the routed agent's resolved default once inside `resolveTelegramSessionState()`, use it for both stored-override default-provider resolution and the final no-override fallback, and keep/add the regression test that opens a Telegram provider page for an agent whose `agents.list[].model` differs from `agents.defaults.model`. The broader related routed-session reports do not supersede this picker-specific gap because the current callback fallback is still wrong on main.
