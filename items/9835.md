## Summary

Keep #9835 open. Current main still supports Telegram inbound reaction events and outbound reaction prompt guidance, but it does not implement the requested configurable two-way prompt guidance: there is no user-facing `reactionGuidance` config, schema validation only accepts `reactionNotifications` and `reactionLevel`, and the `## Reactions` prompt still only covers outbound reaction etiquette rather than incoming reaction soft-CTA interpretation.

## Best Possible Solution

Keep the issue open. If maintainers want this feature, implement it in the Telegram/plugin prompt path by adding a documented `reactionGuidance` config option, deciding its relationship to `reactionLevel`, gating prompt injection on enabled inbound `reactionNotifications` plus guidance mode, adding minimal/full guidance for both sparse sending and incoming reaction interpretation, updating schema/generated metadata/docs, and adding focused tests for off/minimal/full behavior and notification gating.
