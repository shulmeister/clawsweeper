## Summary

Keep open. Current main partially covers the underlying Telegram reaction plumbing and adds a Telegram reaction prompt section for sending guidance, but the issue's requested behavior remains meaningfully incomplete: there is no user-facing `reactionGuidance` config option, prompt injection is tied to `reactionLevel` rather than `reactionNotifications`, and the prompt does not include the requested receiving-reaction soft CTA guidance.

## Evidence

- **No requested config option:** A targeted search for `reactionGuidance` under config, Telegram, and docs found only the plugin prompt hook, not a config schema/type/docs option. Telegram config exposes `reactionNotifications` and `reactionLevel`, not `reactionGuidance` with `off|minimal|full`.
  - command: `rg -n "reactionGuidance" src/config extensions/telegram docs/channels/telegram.md`
  - sha: [1dfa52d0711d](https://github.com/openclaw/openclaw/commit/1dfa52d0711dfc86e1c04538fe46e3d43961766f)
- **Existing Telegram config differs from requested shape:** Telegram config defines `reactionNotifications?: "off" | "own" | "all"` and `reactionLevel?: "off" | "ack" | "minimal" | "extensive"`; it does not define `reactionGuidance` or `full`.
  - file: [src/config/types.telegram.ts:176](https://github.com/openclaw/openclaw/blob/1dfa52d0711dfc86e1c04538fe46e3d43961766f/src/config/types.telegram.ts#L176)
