## Summary

Keep #9835 open. Current main still has partial Telegram reaction support, but not the requested product behavior: there is no public `reactionGuidance: "off" | "minimal" | "full"` config, no strict-schema/generated metadata/docs support for it, and the injected `## Reactions` prompt only covers outbound reaction guidance rather than incoming reaction soft-CTA interpretation. The existing issue discussion already reached the same conclusion, and there are no protected author/label rules forcing maintainer-only closure.

## Best Possible Solution

Keep #9835 open. If maintainers want this behavior, implement it in the bundled Telegram plugin and agent prompt composition path: add and document `reactionGuidance` with `off|minimal|full` semantics, define how it composes with existing `reactionLevel` and `reactionNotifications`, gate incoming soft-CTA prompt guidance on enabled reaction notifications, update strict schema/generated config metadata/docs, and add focused tests for off/minimal/full plus notification gating.
