## Summary

Keep #9835 open. Current main still has Telegram inbound reaction events and outbound reaction prompt guidance, but it does not implement the requested configurable two-way prompt guidance: there is no `reactionGuidance` config, prompt injection still follows `reactionLevel`, and the `## Reactions` prompt only covers sending reactions, not incoming reaction soft-CTA interpretation.

## Best Possible Solution

Keep the issue open. If maintainers still want this feature, implement it in the Telegram/plugin prompt path by adding a documented user-facing `reactionGuidance` setting, deciding whether it should be independent from or layered on top of `reactionLevel`, gating prompt injection on enabled inbound `reactionNotifications` plus guidance mode, adding minimal/full prompt text for both sparse sending and incoming soft-CTA interpretation, updating config schema/generated metadata/docs, and adding focused tests for off/minimal/full behavior and notification gating.
