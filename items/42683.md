## Summary

Keep open. Current main still does not implement the PR's core behavior: the legacy iMessage plugin only forwards resolved reply_to_* fields when imsg already supplies them, and there is no chat.db fallback lookup before inbound dispatch. The PR is stale against the old src/imessage layout, but it remains a focused, relevant bug fix with review feedback already addressed in follow-up commits.

## Best Possible Solution

Keep the PR open, but refresh it against current main by moving the lookup and tests into extensions/imessage/src/monitor, rechecking the current imsg payload contract, and validating local-only chat.db lookup behavior including the SSH-wrapper remoteHost guard. If maintainers decide legacy imsg should not gain this behavior, close should be an explicit product decision in favor of BlueBubbles or a canonical follow-up, not automated cleanup.
