## Summary

Keep open. Current main has related cron reliability work, backoff, failure alerts, persisted run state, and generic gateway probes, but #13593's central observability requests are still missing: structured per-attempt success/skipped/failure logging, cron-specific health reporting, and `openclaw cron list` warnings for overdue or repeatedly failing jobs.

## Best Possible Solution

Keep #13593 open. The best product path is to add structured per-attempt cron execution logging for scheduled, startup catch-up, and manual execution paths; warn on skipped outcomes; add cron health semantics either through `/health/cron` or an explicit extension of the existing health/readiness surface; and teach `openclaw cron list` to annotate overdue and repeated-failure jobs with focused service, CLI, and probe tests. PR #46501 remains useful prior art, but it was closed unmerged and is not present on current main.
