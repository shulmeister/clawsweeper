## Summary

Keep open. Current main has related cron reliability fixes, persisted run state, backoff, and failure-alert plumbing, but #13593's observability acceptance criteria remain unmet: no per-attempt success/skipped execution logging, no cron-specific health endpoint/component, and no `openclaw cron list` warning annotations for overdue or repeatedly failing jobs. The related PR #46501 was closed unmerged, and its claimed log strings and CLI warnings are absent from current main.

## Best Possible Solution

Keep #13593 open. The best path is to implement structured per-attempt cron execution logging across scheduled/startup/manual execution paths, add a cron health signal or explicit maintainer decision about whether `/health/cron` belongs in core, and teach `openclaw cron list` to surface overdue and repeated-failure warnings with focused service/CLI/probe tests. PR #46501 can be used as prior art, but it was not merged and is not present on current main.
