## Summary

Keep #13593 open. Current main has partial cron observability through last status/error/duration fields, consecutive-error tracking, failure alerts, and durable JSONL run history, but it still lacks the issue's central requested surfaces: structured logging for every cron execution attempt, warnings for skipped/non-throwing failures, a cron-specific health endpoint/component, and `openclaw cron list` annotations for overdue or repeatedly failing jobs. The prior related PR #46501 is useful prior art, but the provided GitHub context shows it was closed unmerged.

## Best Possible Solution

Keep #13593 open. The maintainer path is to add structured per-attempt cron execution logging for scheduled, catch-up, and manual runs; warn on skipped and non-throwing error outcomes; expose cron health either at `/health/cron` or as a cron component in the existing readiness/health model; and add `openclaw cron list` annotations for overdue and repeatedly failing jobs. PR #46501 can be mined as prior art but should not be treated as shipped.
