## Summary

Keep #13593 open. Current main has cron state, durable run history, failure alerts, backoff, and generic gateway probes, but it still does not implement the requested per-attempt cron execution logging, skipped-run warnings, cron-specific health endpoint/component, or `openclaw cron list` health annotations. The prior PR #46501 is relevant prior art, but it is closed/unmerged and its changes are not on current `main`.

## Best Possible Solution

Keep #13593 open. The best product path is to add structured per-attempt cron execution logging across scheduled timer runs, startup catch-up, and manual runs; warn on skipped outcomes; add cron health semantics through `/health/cron` or an explicit cron component in the existing health/readiness model; and teach `openclaw cron list` to annotate overdue and repeatedly failing jobs, with service, CLI, probe, docs, and regression-test coverage. PR #46501 remains useful prior art, but it was closed unmerged and is not present on current main.
