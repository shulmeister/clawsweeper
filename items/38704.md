## Close Comment

Codex review: closing as already implemented on current `main`. OpenClaw already ships a built-in `exec` tool for running terminal commands (`src/agents/tool-catalog.ts:83`, `src/agents/bash-tools.exec.ts:1366`), and the requested security controls are also present: `security=allowlist|full`, `ask=off|on-miss|always`, and per-agent allowlists in `~/.openclaw/exec-approvals.json` are documented in `docs/tools/exec.md` and `docs/tools/exec-approvals.md`. I also verified that the latest shipped release, `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), already contains those docs and the core `exec` tool. The request is framed as a new plugin, but the underlying capability already exists in OpenClaw core, so there is no separate missing feature to track here.
