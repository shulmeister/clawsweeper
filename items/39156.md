## Summary

Current main removes the split module graph described in #39156: core entrypoints, plugin-sdk subpaths, and bundled root plugins are built in one dist graph, and the plugin loader/SDK alias logic keeps built plugins on the canonical dist module graph so logger singletons are shared instead of duplicated.

## Close Comment

Closing from a Codex review. Current `main` no longer has the split core/plugin-sdk build layout described in this report: `tsdown.config.ts:292` builds core entrypoints, plugin-sdk subpaths, and bundled root plugins in one dist graph specifically so runtime singletons are emitted once, and `src/infra/tsdown-config.test.ts:55` locks that in. The loader and SDK alias path also keep built bundled plugins on the canonical dist module graph instead of mixing graphs (`src/plugins/loader.ts:452`, `src/plugins/sdk-alias.ts:165`, `src/plugin-sdk/root-alias.cjs:15`). Since `extensions/diagnostics-otel/package.json:22` is a bundled root plugin, it now follows that unified path rather than the isolated module path from the original report. I did not do a live OTLP end-to-end run in this read-only pass, but the current build/runtime topology directly addresses the module-isolation bug reported here.
