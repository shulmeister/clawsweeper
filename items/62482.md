## Summary

Keep PR #62482 open. Current main still has the stale subsystem child file-logger cache that the PR is trying to fix, and the rollover regression test is not present on main. The linked bug #62381 therefore still appears to be actionable rather than implemented, duplicate, or stale-cleanup material.

## Evidence

- **Current main still caches one child file logger per subsystem logger:** `createSubsystemLogger` initializes `fileLogger` once, then `emitLog` only calls `getChildLogger({ subsystem: resolvedSubsystem })` when `fileLogger` is null. There is no check for parent logger replacement or calendar-day rollover before file writes.
  - file: [src/logging/subsystem.ts:325](https://github.com/openclaw/openclaw/blob/4069c81b159087cb71f6b64676d533cf0d897d68/src/logging/subsystem.ts#L325)
  - command: `nl -ba src/logging/subsystem.ts | sed -n '280,450p'`
  - sha: [4069c81b1590](https://github.com/openclaw/openclaw/commit/4069c81b159087cb71f6b64676d533cf0d897d68)
- **Raw file writes use the same stale cache path:** `raw(message)` also reuses the closure-level `fileLogger` after first initialization, so the stale-binding behavior is not limited to structured log levels.
  - file: [src/logging/subsystem.ts:414](https://github.com/openclaw/openclaw/blob/4069c81b159087cb71f6b64676d533cf0d897d68/src/logging/subsystem.ts#L414)
  - command: `nl -ba src/logging/subsystem.ts | sed -n '280,450p'`
  - sha: [4069c81b1590](https://github.com/openclaw/openclaw/commit/4069c81b159087cb71f6b64676d533cf0d897d68)
- **Parent logger rolls by current local date:** The default file path is computed from `new Date()` via `defaultRollingPathForToday()`, and `getLogger()` rebuilds `loggingState.cachedLogger` when resolved settings change. `getChildLogger()` binds a sublogger to the current base logger, which is why a subsystem-level cached child can remain attached to the old file transport after the parent changes.
  - file: [src/logging/logger.ts:571](https://github.com/openclaw/openclaw/blob/4069c81b159087cb71f6b64676d533cf0d897d68/src/logging/logger.ts#L571)

- **Main lacks the PR's rollover regression test:** The logging test directory on current main only has `src/logging/subsystem.test.ts`; `src/logging/subsystem.rollover.test.ts` from the PR is absent.
  - file: [src/logging/subsystem.test.ts:1](https://github.com/openclaw/openclaw/blob/4069c81b159087cb71f6b64676d533cf0d897d68/src/logging/subsystem.test.ts#L1)
