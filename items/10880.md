## Summary

Current `main` and the shipped `v2026.4.22` release already default the macOS desktop app packaging to universal binaries (`arm64` + `x86_64`), which implements the requested Intel Mac support at the app packaging level.

## Risks / Open Questions

- This closes the packaging/support request for the desktop app itself. Separate Intel-specific limitations may still exist in individual native plugins or runtimes, but that is narrower than the app-binary support requested here.

## Close Comment

Closing as implemented. Codex review checked current `main` and the shipped `v2026.4.22` tag: `scripts/package-mac-app.sh:24` now defaults release builds to universal output and expands that to `arm64 x86_64`, and `scripts/package-mac-dist.sh:17` explicitly sets distribution packaging to `BUILD_ARCHS=all` with a note that it supports both Apple Silicon and Intel Macs. The shipped changelog in `CHANGELOG.md` also records the macOS universal-binary packaging change. On that basis, the requested Intel macOS desktop-app support is already implemented in the current release line. If an Intel Mac still cannot install or launch `v2026.4.22` or newer, please open a new bug with the exact OpenClaw version, macOS version, and whether the failure is during install, launch, update, or a specific plugin/runtime path.
