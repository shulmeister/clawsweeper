## Summary

Keep open. Current main has partial noise controls for heartbeat/cron alerts and exec lifecycle events, but it does not implement the broader requested policy to silence or group auto-generated action confirmations across chat surfaces. Slash-command skill/tool dispatch still returns explicit success text such as `✅ Done.`, skill installation paths still surface installed messages, and cron/heartbeat delivery remains alert-on by default unless heartbeat-specific visibility is disabled.

## Best Possible Solution

Keep the issue open and implement a coherent notification/confirmation policy, likely under `messages` or `commands`, that covers command acknowledgements, direct skill/tool dispatch replies, cron/system-event delivery, and install/update confirmations with per-action defaults. Reuse existing heartbeat visibility and `tools.exec.notifyOnExit` semantics where possible, then add targeted tests for Telegram/Discord outbound replies, cron heartbeat delivery, skill command dispatch, and Control UI feedback.
