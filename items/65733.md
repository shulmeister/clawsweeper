## Summary

Keep open. Current main still has plugin discovery calling `safeRealpathSync()` without a shared per-discovery realpath cache, and the linked implementation PR #65825 is still open and unmerged with closing syntax for this issue. That PR is a candidate fix, not a reason to close the issue early.

## Best Possible Solution

Keep this issue open and review/land or supersede PR #65825 with a focused fix that threads a shared realpath cache through plugin discovery helpers and includes regression coverage for repeated `safeRealpathSync()` calls during a discovery pass. After that PR or an equivalent fix merges, close this issue as implemented with the landed commit and release evidence.
