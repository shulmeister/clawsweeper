## Summary

Keep #65733 open. Current main still performs uncached `safeRealpathSync` calls inside a single plugin discovery pass, and the gateway startup/auto-enable path can still derive plugin registry data through discovery when the persisted registry is missing or stale. The linked fix PR #65825 remains open in the provided context; the shipped #67940-era change only reuses completed shared discovery results across workspace cache misses, not per-entry realpath work.

## Best Possible Solution

Keep #65733 open. The best path is to land or supersede #65825 on current main with a focused patch that threads a shared realpath cache through discovery helpers, caches canonicalized paths where safe, and adds regression coverage for `fs.realpathSync` call count and symlink/package-root behavior. Do not close solely for #67940 because it caches completed shared discovery results rather than eliminating repeated realpath calls within one discovery pass.
