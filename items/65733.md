## Summary

Keep open. In checked-out `main` at `c150110e02550eaf6c56db0c851dc62fc0ca88e1`, plugin discovery still calls `safeRealpathSync` repeatedly on hot paths without passing a shared cache, while the only shared `realpathCache` is created later inside manifest-registry duplicate handling. That matches the issue’s reported failure mode, so there is not strong evidence to close this as fixed or obsolete.

## Evidence

- **Discovery hot paths still call realpath without cache:** `safeRealpathSync` accepts an optional cache, but discovery uses it without the cache parameter in multiple places that run during candidate scanning and safety checks (`checkSourceEscapesRoot`, `resolvesToSameDirectory`, `addCandidate`, `discoverInDirectory`).
  - file: [src/plugins/discovery.ts:157](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/plugins/discovery.ts#L157)
