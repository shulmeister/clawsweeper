## Summary

Keep this PR open. Current main still lacks the proposed dedicated hook-pack publishing guide and cross-links; the existing docs cover hook structure plus install/update behavior, not packaging and publishing a reusable hook pack. The topic is valid because current source supports package-based hook packs, but the PR should be rebased and scoped before merge because the branch mixes docs work with unrelated Discord gateway runtime/test changes and its marketplace language needs to match current ClawHub-first docs.

## Best Possible Solution

Keep this PR open, ask for a scoped rebase, and preserve the hook-pack publishing docs if still desired. The best path is to add the guide, navigation entry, and cross-links updated for current ClawHub-first or npm distribution behavior, while splitting or dropping the unrelated Discord gateway changes into a separate bug-fix PR with its own evidence and tests.
