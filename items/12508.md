## Summary

Keep #12508 open. Current main still has host-owned plugin hook runners that allow sequential mutation of prompts, tool params, outbound messages, and persisted transcript-adjacent messages, but there is no per-handler checksum/signature verification, tamper-detection handoff, modification audit trail, or per-hook manifest allow-list. The existing maintainer-side Codex comment keeping this open still matches current main.

## Best Possible Solution

Keep this open for core plugin-runtime security design. The likely path is a host-owned change across the hook runner, registry/config policy, manifest/docs contract, and focused tests for sequential mutation, per-hook permissions, deterministic modification auditing, and integrity verification. This should not move to ClawHub unless core first exposes an API that lets an external plugin enforce these guarantees across the full hook chain.
