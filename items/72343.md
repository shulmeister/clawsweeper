## Summary

Keep open. Current main still routes bare `openclaw` directly to Crestodian, and the docs/tests still describe fresh configless setup through Crestodian. The PR is a focused active change for a behavior that is not implemented on main; the provided PR discussion also shows the author addressed the bot's e2e coverage concern in head commit 5b857830ce58ec5d5628c065dccab97f998bdf39. The related ClawSweeper title-search items are unrelated fresh-install/install issues and do not supersede this PR.

## Best Possible Solution

Keep the PR open for maintainer review. If accepted, the right implementation belongs in `src/cli/run-main.ts` before the bare-root Crestodian branch: use `readConfigFileSnapshot()` to route missing or valid empty authored config to `setupWizardCommand({})`, while preserving Crestodian for configured installs and invalid configs that need repair. Keep the associated CLI/docs/e2e updates so the first-run contract is explicit.
