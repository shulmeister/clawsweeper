## Summary

Keep PR #39245 open. Current main has related canonical Kimi ID preservation and provider-prefixed tool-name normalization for dispatch/replay, but it still does not implement the PR's central mixed-ID behavior: matching or sanitizing `functions.exec:0` and `functions exec:0` as the same provider-emitted tool call. The linked earlier PR #39091 is closed, while #39245 remains the active branch and has an unresolved collision concern that needs maintainer review before merge.

## Best Possible Solution

Keep #39245 open for maintainer review. The best path is a narrow agent-side fix that preserves current name normalization, adds collision-safe handling for leading `functions ` versus `functions.` IDs only where provider interop requires it, binds normalized pairing to additional context, preserves replay-safe raw IDs, adds regression coverage for the mixed dot/space ID case, and separates unrelated logging cleanup.
