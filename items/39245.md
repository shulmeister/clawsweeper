## Summary

Keep PR #39245 open. Current main still pairs tool-call results by exact raw ID and does not implement the PR's central mixed `functions.exec:0` / `functions exec:0` matching behavior. The linked #39091 is closed unmerged, while this PR remains an active focused bug-fix attempt with recent author updates and an unresolved collision-safety concern.

## Best Possible Solution

Keep #39245 open for maintainer review. The best path is a narrow agent-side fix that preserves current canonical Kimi ID behavior and replay-safe raw IDs, normalizes only the intended leading `functions ` versus `functions.` provider mangling where interoperability requires it, binds normalized pairing to additional context, adds regression coverage for mixed dot/space IDs and `functions exec` names, and keeps prefixed `sessions_spawn` redaction covered.
