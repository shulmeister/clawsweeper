## Summary

Keep this PR open. Current main still lacks the PR's central normalization behavior for mixed `functions.exec:0` / `functions exec:0` tool-call IDs and `functions.` / `functions ` prefixed tool names. The PR is active, recently rebased in the provided discussion, and its predecessor #39091 is closed unmerged rather than a shipped canonical fix.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a narrow agent-side fix that normalizes only the intended leading `functions ` versus `functions.` provider mangling where pairing requires it, strips `functions.` / `functions ` prefixes before allowed-name checks and `sessions_spawn` redaction, preserves replay-safe raw IDs, and includes regression tests for mixed dot/space IDs plus prefixed tool names. Split or explicitly justify the node workdir changes if they are only CI-follow-up work.
