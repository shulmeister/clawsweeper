## Summary

Keep #39245 open. Current main has newer run-boundary normalization for provider-prefixed tool names, but it still does not implement the PR's core ID-normalization/pairing behavior for canonical `functions.exec:0` versus space-mangled `functions exec:0`. The PR also has recent author activity and an unresolved security review about normalization-induced collisions, so it needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep #39245 open for maintainer review. The best path is to land a narrowly scoped fix that preserves the already-shipped run-boundary tool-name normalization, adds safe regression-tested handling for `functions exec:0` versus `functions.exec:0` where matching is actually required, and addresses the collision concern by binding normalized IDs to additional context such as tool name or adjacency. Once this PR or a narrower replacement lands, #39091 can be closed as the older duplicate.
