## Summary

Keep open. Current `main` still does not implement this PR's behavior: Discord embed extraction only reads `title`/`description` from the first embed, both for normal message fallback and forwarded/thread-starter paths, and the current tests do not cover fields/footer or multi-embed extraction. None of the allowed close reasons apply.
