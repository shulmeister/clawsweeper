## Summary

Keep open. Current `main` still has the two gaps described in the report: Gateway capability checks only do an exact provider+model match, and the Gateway model catalog still does not merge user-configured `models.providers.*.models[]` entries. That means a configured vision model can still be invisible to `resolveGatewayModelSupportsImages`, so this is not safe to auto-close.

## Evidence

- **Gateway catalog still ignores user-configured provider models:** `loadGatewayModelCatalog()` on current `main` just returns `loadModelCatalog(...)` and does not append entries from `cfg.models.providers.*.models[]`, so configured models with `input: ["text", "image"]` are not exposed to Gateway capability checks.
