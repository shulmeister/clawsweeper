## Summary

Keep open. Current main still uses exact provider/model matching in the gateway image-support resolver and does not implement the requested generic fallback to configured provider model metadata. The linked PR #65178 directly attempts this fix, but it is open and unmerged, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open. The best maintainer path is to review and either land #65178 or replace it with an equivalent gateway fix that uses normalized provider/model matching and falls back to configured provider model metadata when the gateway catalog misses. Add focused regression tests for case-insensitive slash-containing model ids, provider aliases, missing provider/catalog entries, and explicit input: ["text", "image"] custom provider rows.
