## Summary

Keep open. Current main still intentionally keeps one-off memory signals out of default `memory-core` dreaming promotion. Source, docs, and tests all show the deep phase requires threshold gates for score, accumulated signals/recalls, and unique contexts; manual threshold overrides and grounded backfill exist, but they do not implement automatic promotion for single critical events like the issue describes.

## Best Possible Solution

Keep the issue open for a maintainer/product decision on `memory-core` dreaming. A good implementation would add a bounded single-event durability path, such as an explicit high-priority/importance signal in the short-term store or a conservative classifier for direct critical feedback/explicit memory directives, while preserving the existing repeated-signal gates for ordinary material. Tests should cover a one-off important event that promotes and an ordinary one-off note that still does not.

## Evidence

- **Default promotion thresholds still require repeated/context-diverse signals:** `memory-core` defines default promotion gates of `minScore=0.75`, `minRecallCount=3`, and `minUniqueQueries=2`. Ranking then skips candidates whose total signal count is below `minRecallCount` or whose query/day diversity is below `minUniqueQueries`. A single occurrence with one signal/context therefore still fails default ranking.
