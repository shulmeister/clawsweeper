## Summary

Keep #22774 open. Current main still does not provide durable human/main-agent/sub-agent sender attribution across the Gateway send schemas, sessions.send/sessions.steer forwarding path, persisted provenance, chat.history-normalized messages, and Control UI rendering. The existing senderLabel support is display-only for user-role groups, while InputProvenance is trace metadata rather than a renderable sender identity contract. The related PRs #23336 and #36006 were closed unmerged, and #58110 is a broader multi-agent chat UX request rather than a canonical replacement for this narrower attribution issue.

## Best Possible Solution

Keep #22774 open as the specific tracker for durable multi-agent sender attribution. The likely product/code path is an additive sender identity contract carrying sender id, sender kind, display name, source session/agent, and renderable UI identity through chat.send, sessions.send, sessions.steer, sub-agent announce delivery, transcript persistence, chat.history/live chat events, and Control UI rendering. Coordinate with #58110 for broader multi-agent chat UX, but do not treat it as a replacement for this attribution contract.
