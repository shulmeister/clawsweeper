## Summary

Keep #22774 open. Current main has partial user `senderLabel` rendering and inter-session provenance, but it still does not provide a durable sender attribution contract or Chat UI rendering that distinguishes human, main-agent, and sub-agent identities across `chat.send`, `sessions.send`/`sessions.steer`, transcript persistence, history, and sub-agent announce flows.

## Best Possible Solution

Keep #22774 open. The best path is an additive core sender attribution contract carrying durable sender id, sender kind, display name, source session/agent, and renderable identity through `chat.send`, `sessions.send`, `sessions.steer`, agent input provenance, and sub-agent announce delivery; persist it in transcript/history events; expose it through `chat.history` and live chat events; then update the Chat UI to render distinct human/main-agent/sub-agent identities with tests for main-to-sub-agent steer and sub-agent-to-main announce flows.
