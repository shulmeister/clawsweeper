## Summary

Current main still contains the exact generic transport-stream throws this PR targets. OpenAI-compatible streams can populate `output.errorMessage` from provider `finish_reason`, but the finalize/error path still throws `An unknown error occurred` and the catch block overwrites the provider message. This PR remains a focused, plausible bug fix rather than cleanup-close material.

## Best Possible Solution

Keep the PR open. The best path is to merge or recreate the narrow fix on current main, preserve provider `output.errorMessage` at the shared helper and OpenAI Responses throw sites, and add regression tests for both message propagation and fallback behavior, with a small maintainer check of model-failover classification impact.
