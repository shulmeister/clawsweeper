## Summary

Current main has partial improvements, but it does not fully implement the requested CLI behavior. `openclaw message send` still has no `--agent` selector, the CLI always passes `resolveDefaultAgentId(cfg)` as the outbound agent, and `--account` is handled only as an account id rather than selecting the bound agent/workspace. Non-default agent workspaces can therefore still be missed when media access is workspace-scoped, so this should stay open.

## Evidence
