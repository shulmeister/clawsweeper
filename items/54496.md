## Summary

Current main still does not implement a silent reasoning persistence mode. OpenClaw can produce thinking and can hide or stream reasoning in user-facing delivery, but stored session transcripts and replay surfaces still keep thinking blocks in important paths.

## Best Possible Solution

Keep this issue open and implement a core transcript-persistence policy independent of thinking production and reasoning visibility. A maintainer-quality path would keep `thinkingLevel` for model reasoning effort, keep `reasoningLevel` for user/channel visibility, add a documented persistence/discard mode, strip `type: "thinking"` and `redacted_thinking` before durable transcript writes or split replay-safe opaque provider state outside user/history/memory transcript APIs, and add coverage for `sessions.get`, ACP `loadSession`, compaction/replay, built-in memory/session indexing, plugin hooks, and provider replay compatibility.

## Risks / Open Questions

- A fix must preserve provider replay compatibility for signed thinking blocks, especially Anthropic, Bedrock, Google, OpenAI Responses, and Codex-style continuations.
- Changing transcript persistence affects `sessions.get`, ACP `loadSession`, compaction/replay, session-history tools, memory indexing, transcript events, and plugin `before_message_write` behavior.
- A plugin can currently mutate message writes, but treating this as only a plugin workaround would leave core config, UI/session controls, provider replay policy, and documented transcript semantics unresolved.
