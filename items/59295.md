## Summary

Current `main` already handles the reported stale-baseline/double-fire case: it re-resolves the live session/default model from disk, treats stale runtime model fields as lower priority than persisted overrides, and clears `liveModelSwitchPending` once the current run already matches the persisted selection. The same implementation is present in release `v2026.4.23`.

## Evidence

- **Live selection is recomputed from current session store, not a load-time snapshot:** `resolveLiveSessionModelSelection` reloads the session store with `skipCache: true`, resolves the current default model for the agent, and prefers normalized persisted overrides over stale runtime `modelProvider`/`model` fields.
