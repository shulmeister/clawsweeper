## Summary

Current main already contains the cron scheduler changes that separate restart catch-up from steady-state timer execution and add regression coverage for the skip/advance paths that could cause unrelated jobs to fire on a wake. The same scheduler files are unchanged between current main and release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

## Evidence

- **Restart catch-up is startup-only:** `start()` clears stale running markers, then calls `runMissedJobs(state)` during service startup before arming the normal timer loop. This keeps missed-slot replay scoped to restart/bootstrap instead of ordinary timer wakes.
  - file: [src/cron/service/ops.ts:87](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/cron/service/ops.ts#L87)
  - command: `nl -ba src/cron/service/ops.ts | sed -n '87,130p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Steady-state timer only runs actually due jobs:** `onTimer()` calls `collectRunnableJobs(state, dueCheckNow)` with no `allowCronMissedRunByLastRun` option. In `isRunnableJob()`, the normal path returns runnable only when `nextRunAtMs <= now`; the missed-slot replay branch is gated behind `allowCronMissedRunByLastRun` and therefore is not active for ordinary timer wakes.
  - file: [src/cron/service/timer.ts:707](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/cron/service/timer.ts#L707)

  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Missed-slot replay is explicitly gated:** The replay path based on `computeJobPreviousRunAtMs()` only runs when `allowCronMissedRunByLastRun` is true. Regular timer ticks do not pass that flag, so a normal 03:00 wake cannot dump future jobs through that branch.
  - file: [src/cron/service/timer.ts:903](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/cron/service/timer.ts#L903)

## Close Comment

Closing from a Codex review because current `main` already has the scheduler changes that cover this report, and the same code shipped in `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). The restart catch-up path is now limited to service startup in `src/cron/service/ops.ts:87`, while ordinary timer wakes only run jobs whose stored `nextRunAtMs` is actually due and do not enable the missed-slot replay branch used for restart recovery (`src/cron/service/timer.ts:707`, `src/cron/service/timer.ts:903`). The steady-state timer also uses `recomputeNextRunsForMaintenance()` instead of the old full recompute path when no jobs are due and after processing completed runs, specifically to avoid silently advancing or replaying unrelated cron jobs on a wake (`src/cron/service/timer.ts:712`, `src/cron/service/timer.ts:806`). There is now regression coverage for restart catch-up and for the non-startup skip/advance paths in `src/cron/service.restart-catchup.test.ts:79`, `src/cron/service.issue-16156-list-skips-cron.test.ts:31`, and `src/cron/service.issue-17852-daily-skip.test.ts:6`.
