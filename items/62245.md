## Summary

The reported behavior is WeChat-plugin-specific rather than an OpenClaw core gap. Current main documents WeChat as an external `@tencent-weixin/openclaw-weixin` plugin, while core already routes plugin inbound messages through the normal agent pipeline where skills are resolved and injected per run.

## Evidence

- **WeChat is external plugin-owned:** The WeChat docs state OpenClaw connects through Tencent's external `@tencent-weixin/openclaw-weixin` plugin, that `openclaw-weixin` is the channel id/package, and that WeChat login, media upload/download, context tokens, and monitoring are owned by the external plugin.
