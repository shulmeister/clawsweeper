## Summary

Keep PR #69197 open. Current `main` does not implement the proposed poll-active suppression, and the PR still contains useful focused work for a real exec notification race. It should not be auto-closed as implemented, duplicate, ClawHub, or stale cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. The best code path is to land a robust variant of the poll suppression fix, preferably replacing the boolean with an in-flight poll counter/token, keeping the kill-path `exitNotified` placement on confirmed success paths, and retaining focused regression tests for active poll suppression, normal notify-on-exit behavior, poll timeout, and overlapping polls.
