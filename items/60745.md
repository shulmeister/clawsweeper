## Summary

Keep open. Current main does not implement completion-based ephemeral session disposal for cron jobs or hook mappings. Cron cleanup is still retention/timer based, there is no cron or hook `ephemeral` contract/CLI surface, and hook agent runs do not delete their session entry after `runCronIsolatedAgentTurn` completes.

## Evidence

- **Cron reaper is retention based:** `sweepCronRunSessions` resolves `cron.sessionRetention`, computes a cutoff, and deletes only cron run session entries whose `updatedAt` is older than that cutoff. There is no per-job ephemeral bypass or completion-triggered deletion path here.

- **Cron job schema has no ephemeral field:** The strict cron job/add/patch protocol schemas list `deleteAfterRun`, `sessionTarget`, `wakeMode`, payload, delivery, failure alert, and state fields, but no `ephemeral` field.
  - file: [src/gateway/protocol/schema/cron.ts:250](https://github.com/openclaw/openclaw/blob/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54/src/gateway/protocol/schema/cron.ts#L250)
