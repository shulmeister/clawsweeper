## Summary

Keep PR #66146 open. Current main still only forwards OpenAI-compatible max token fields when `options.maxTokens` is explicitly present, so the PR's central transport bug is not implemented on main. The PR should not be auto-closed as obsolete, but it also should not land as-is because the branch bundles unrelated TUI changes that drew blocking review comments.

## Best Possible Solution

Keep the PR open, but ask for a focused transport-only update: set an `effectiveMaxTokens` from `options?.maxTokens ?? model.maxTokens` in both OpenAI Responses and Completions builders, preserve the existing `compat.maxTokensField` behavior, add regression tests for omitted `options.maxTokens`, and split or drop the unrelated TUI refactor before merge.
