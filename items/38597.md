## Summary

Keep open. The checked-out main at 04e08cea still uses the whitespace-only Discord split path and hard-splits when no whitespace is found, so the CJK boundary bug is not implemented on main. The active follow-up PR #71384 is the right place to resolve this, and #38597 appears to be the canonical issue it targets rather than a cleanup duplicate.

## Best Possible Solution

Keep #38597 open until PR #71384, or an equivalent patch, lands in `extensions/discord/src/chunk.ts` with CJK punctuation and complete-character fallback behavior plus regression tests in `extensions/discord/src/chunk.test.ts`. After that lands, close #38597 as implemented and handle #38607 as the duplicate follow-up if it remains open.
