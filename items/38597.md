## Summary

Keep open. Current main still uses whitespace-only Discord long-line splitting and falls back to a hard 2000-character split for CJK text with no whitespace. The focused fix is tracked in open PR #71384, but the behavior has not landed on main.

## Best Possible Solution

Keep #38597 open and use PR #71384 as the active fix path. The implementation should update extensions/discord/src/chunk.ts to preserve whitespace priority, add CJK punctuation and safe CJK character-boundary fallbacks with a minimum-progress guard, and add regression coverage for Chinese, Japanese, Korean, punctuation, mixed-script, long-token, and surrogate-pair cases.
