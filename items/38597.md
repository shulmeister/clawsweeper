## Summary

Keep #38597 open. Current main at 82262387651ab270b565f71533ce4937c9c00dc6 still has the reported Discord CJK chunking bug: long-line splitting only searches for whitespace and otherwise hard-slices at the Discord limit. The prior discussion is useful and active: @openperf provided the root-cause analysis and patch direction, and #71384 is the current open PR carrying that fix, but it has not landed on main.

## Best Possible Solution

Keep #38597 open and use #71384, or a successor PR, as the active fix path. The fix belongs in extensions/discord/src/chunk.ts with focused tests in extensions/discord/src/chunk.test.ts: preserve whitespace priority, add CJK terminal/closing punctuation fallback, add a Unicode-safe CJK character/code-point boundary fallback with a minimum-progress guard, and cover length mode, newline mode, mixed script, long-token fallback, preserveWhitespace/fence behavior, and supplementary-plane CJK cases.
