## Summary

Current main still contains the exact `buildReplyPayloads` heartbeat sanitation bypass described in #72217. The issue is recent, actionable, not maintainer-authored, has no protected labels, and the remaining work is a narrow code/test fix rather than cleanup-closeable stale or duplicate work.

## Best Possible Solution

Keep #72217 open and fix the helper directly: add focused coverage in `src/auto-reply/reply/agent-runner-payloads.test.ts` for `isHeartbeat: true` with exact `HEARTBEAT_OK` producing no reply payloads, plus alert/media cases that remain deliverable. Then update `buildReplyPayloads` so heartbeat final payloads also run through heartbeat-token stripping, ideally using heartbeat-mode semantics and the configured ack limit where that context is available.
