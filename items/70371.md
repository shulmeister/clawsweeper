## Summary

Keep open. Current main still treats plugin-local schema failures as invalid global config and as strict plugin-load errors in key startup/registry paths, so the requested graceful per-plugin quarantine is not implemented. The linked PR #70394 is a relevant open fix attempt, not a shipped/superseding resolution.

## Best Possible Solution

Keep #70371 open and finish a core fix. The best path is to quarantine plugin-entry schema failures per plugin: preserve clear diagnostics, mark the plugin error/disabled in the runtime and metadata registry, skip its registration and tools, continue booting unaffected plugins/channels/jobs, and extend doctor/status repair output for `plugins.entries.<id>.config` schema failures. PR #70394 is a plausible starting point but needs maintainer review against the config-load, runtime-loader, metadata-loader, and doctor acceptance criteria.
