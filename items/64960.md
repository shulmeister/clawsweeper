## Summary

Keep open. Current main mitigates the unbounded overflow loop with capped compaction attempts and a terminal context-overflow error, but it does not implement the requested compaction model fallback chain. Compaction still resolves to a single configured override or the session model, retries summarization with that same model, and the generic model-fallback path intentionally does not handle context-overflow/compaction failures.

## Best Possible Solution

Keep this issue open as an actionable reliability bug. The likely fix belongs in the compaction orchestration path: add a validated `agents.defaults.compaction.fallbacks` or equivalent candidate list, try compaction candidates ephemerally per attempt without persisting a user-visible session model switch, add regression coverage for a primary compaction model returning an Azure-style HTTP 400 content-filter error, and coordinate any final hard escape with the related open timeout/escalation work in #64962.
