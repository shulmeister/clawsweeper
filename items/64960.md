## Summary

Current `main` still has the compaction gap described in #64960: compaction resolves to one effective model (session model or a single `agents.defaults.compaction.model` override), retries summarization on that same model, and overflow recovery re-invokes compaction without any compaction-specific model fallback chain. A content-filtered Azure summarization failure can therefore still strand overflow recovery.

## Evidence
