## Summary

Keep #70796 open. Current main provides a chat-header session picker and a separate Sessions page, but the left sidebar Chat section is still only a static Chat navigation item, so the requested expandable/dropdown conversation list under Chat remains a concrete Control UI enhancement.

## Best Possible Solution

Keep the issue open and implement this as a Control UI navigation enhancement: add an expandable recent-conversations list under the Chat sidebar item, backed by the existing `sessions.list`/`sessionsResult` data and `switchChatSession` path, with active-session highlighting, display labels/timestamps/previews where available, and a New Chat action that reuses the existing `/new` flow.
