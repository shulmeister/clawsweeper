## Summary

Keep open. The issue reports duplicate plugin register/load cycles on gateway restart, and current main still has a plausible duplicate-load path: startup loads use the full startup plugin id set, then deferred gateway reloads reuse that same plugin id set after listen when any deferred configured channel plugin exists. Existing tests cover diagnostic suppression and scope reuse, not a guarantee that non-deferred plugins are only registered once.

## Evidence

When deferred plugins exist it only changes preferSetupRuntimeForChannelPlugins/suppressPluginInfoLogs, not the scope.
