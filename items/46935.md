## Summary

Keep open. Current main still shows `/model` Available values from `ModelCatalogEntry.id` without provider qualification, while `/model <value>` sends the typed string through gateway model resolution. Bare allowlisted ids are now partially improved for unique configured models, but slash-containing provider-native catalog ids can still be displayed in a non-canonical form that is not round-trippable. The linked PR #47249 appears to target this exact UX bug but is open/unmerged, so it is not strong evidence to close the issue.
