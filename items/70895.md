## Summary

Keep this PR open. Current main still routes webhook agent runs through the isolated cron runner and passes `forceNew: true` whenever `sessionTarget` is `"isolated"`, so the session resolver cannot reuse an existing fresh session for stable hook `sessionKey` values. The PR is also paired with open bug #70894 by the same author and uses closing syntax, so the best maintainer action is review/land this PR or a narrow equivalent rather than closing it as obsolete.

## Best Possible Solution

Review and land this PR or a narrow equivalent on main: compute `forceNew` from true scheduled isolated cron semantics rather than every isolated agent turn, allow hook/webhook-sourced isolated turns with explicit hook provenance or stable hook session keys to reuse fresh existing sessions, and add regression tests while preserving fresh-per-run behavior for ordinary isolated cron runs.
