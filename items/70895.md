## Summary

Keep #70895 open. Current main still routes /hooks/agent through an isolated CronJob and still passes forceNew: true for every isolated run, so stable hook sessionKey values cannot reuse an existing session. The PR is a small, relevant fix for #70894 with a targeted regression test; it is not implemented, superseded, or a ClawHub/out-of-repo item.

## Best Possible Solution

Keep the PR open and review/land the narrow fix, or adapt it into an equivalent patch: compute hook-sourced runs before resolveCronSession, pass forceNew: false for webhook/hook sessions with stable sessionKey or externalContentSource, keep ordinary isolated cron runs fresh, and add the regression test from the PR. Link it to #70894 and the broader #11665 discussion rather than closing this useful focused fix.
