## Summary

Keep open. Current main has only a generic all-media-failed warning; it still drops individual failed media items in mixed payloads and still strips all media in the agent-runner normalization catch without carrying user-visible dropped-media details. The related #70109 candidate is closed unmerged in the provided context, so it does not supersede this PR.

## Best Possible Solution

Keep the PR open and ask for an updated, focused implementation against current main: either preserve per-item dropped-media diagnostics through normalization, routing, outbound JSON/delivery, transcript mirror, and followup paths, or replace it with an equivalent smaller mainline fix that covers partial drops and catastrophic normalization failures with sanitized user-visible notices.
