## Summary

Keep open. Current main only has a narrow generic `⚠️ Media failed.` fallback when every media item in the reply-media normalizer is dropped. It does not implement the PR's full dropped-media pipeline, and meaningful silent-loss cases remain: mixed payloads still send surviving media without warning about dropped items, and the agent-runner normalization catch still strips media with only verbose logging. The related #70109 is closed unmerged, so there is no canonical open replacement tracking the remaining work.

## Best Possible Solution

Keep the PR open but ask for a rebase/scoped update against current main. The best product path is either to land a smaller version that surfaces every dropped `MEDIA:` item, including mixed-success and normalization-catch cases, with sanitized display names and focused tests, or to make an explicit maintainer decision that the generic all-dropped warning is sufficient and close only with that rationale.
