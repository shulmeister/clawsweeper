## Summary

Keep this PR open. Current main only implements a narrow generic warning when every reply media item fails normalization; it does not implement this PR's dropped-media contract, per-item sanitized user notices, catastrophic normalization visibility, outbound JSON/transcript propagation, or dropped-media-only routing/followup preservation. The related #70109 PR is closed and unmerged, so it does not supersede this active PR.

## Best Possible Solution

Keep this PR open. The best path is a maintainer-reviewed, possibly scoped implementation that preserves sanitized per-item media drop details through reply media normalization, catastrophic fallback, route/followup filtering, outbound JSON/delivery, and transcript mirroring, while retaining the privacy and delivery-safety fixes already discussed in the PR review thread.
