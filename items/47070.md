## Summary

Keep open. Current main still starts `sessions_spawn` child runs without attaching `inputProvenance`, and there is no `sourceTool: "sessions_spawn"` plumbing on the spawn path. The linked draft PR #47087 appears to be the proposed fix, but it has not landed in the checked-out main SHA.

## Evidence

- **Spawn dispatch lacks inputProvenance:** `spawnSubagentDirect()` dispatches the child with `callSubagentGateway({ method: "agent", params: { ... } })`, passing message, session, delivery fields, lane, prompt, timeout, label, and metadata, but no `inputProvenance` object.
  - file: [src/agents/subagent-spawn.ts:990](https://github.com/openclaw/openclaw/blob/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071/src/agents/subagent-spawn.ts#L990)
