## Summary

Keep #68483 open. Current main still redacts `sessions_history` content and reports `contentRedacted`, but it does not add a `[OPENCLAW-REDACTED]` marker, a `notice` field, or a shared `redactSensitiveTextForAgent` helper. The PR is still useful prior work for linked issue #68425 and the related #68423 incident, but it likely needs maintainer review/fixes before merge because the PR discussion includes a remaining false-positive notice concern and the changelog patch lacks the required Thanks attribution.

## Best Possible Solution

Keep the PR open and ask for a focused update: gate the notice on redaction actually visible in the final returned payload or adjust the wording, add/keep regression coverage for clean, redacted, capped, and truncation edge cases, and fix the changelog attribution. If accepted, the implementation belongs in `src/logging/redact.ts` plus `src/agents/tools/sessions-history-tool.ts`, with tests in the existing redaction and sessions tool suites.
