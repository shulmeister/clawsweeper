## Summary

Current `main` still runs memory flush compaction through the normal embedded runner without assigning a separate lane, and the embedded runner still defaults unspecified runs to the `main` global lane. That means the reported main-lane starvation path remains plausible on current code; there is not strong evidence to close this issue as fixed or obsolete.
