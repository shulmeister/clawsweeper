## Summary

Keep PR #40230 open. Current main still has no symlink-following path for workspace bootstrap files: bootstrap reads go through `openBoundaryFile`, failed boundary validation is returned directly, and the loader records those files as `missing: true`. The linked bug #40210 remains a concrete user problem, and the PR discussion includes user confirmations plus follow-up commits addressing review concerns, so this is not a conservative cleanup close.

## Best Possible Solution

Keep the PR open for explicit maintainer review or replacement. The likely best implementation belongs in `src/agents/workspace.ts`: allow symlinked fixed workspace bootstrap files only when intentionally supported, preserve the existing max-size and regular-file checks, reject hardlinked targets, read through a verified file descriptor with same-file/race protection, and add regression tests for absolute/relative external symlinks, dangling symlinks, oversized targets, hardlinks, and extra-bootstrap traversal boundaries. Update workspace docs if maintainers choose to make this supported behavior.
