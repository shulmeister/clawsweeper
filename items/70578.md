## Summary

Keep #70578 open. Current main has a partial fix for the projection path: source keys missing from the runtime shape are now copied instead of dropped. But the issue's central requested behavior is not fully implemented: the root config schema is still strict, tests still assert unknown root fields are invalid, and bundled/plugin CLI surfaces are still gated by a restrictive `plugins.allow` path that returns the exact style of error described in the report.

## Best Possible Solution

Keep the issue open and narrow it into a maintainer-owned config contract fix: decide whether unknown top-level `openclaw.json` keys should be preserved inertly or fail fast, then add regression coverage around `plugins install`, gateway startup writes, and suspicious size-drop recovery. Separately decide whether bundled OpenClaw CLI surfaces should bypass or auto-seed `plugins.allow`, because current source and docs still require explicit allowlist membership.
