## Summary

Keep this PR open. Current main partially covers the sticky auto-failover and overflow-loop problems, but it does not implement the PR's full behavior: there is no dedicated failoverState, no memoryConversationKey, and context-engine lifecycle calls still pass the stable sessionKey rather than a /new-rotated, /reset-preserved LCM identity.

## Best Possible Solution

Keep this PR open, but require a rebase and likely split/narrowing: preserve the auto-override cleanup already on main, decide whether a dedicated failoverState is still needed, implement the missing LCM conversation identity seam consistently across context-engine lifecycle calls, and keep only overflow diagnostics or circuit-breaker changes that are not already covered by the current three-attempt overflow guard.
