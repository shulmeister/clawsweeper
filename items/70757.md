## Summary

Keep this PR open. Current main still uses the BlueBubbles API password as the inbound webhook authentication token, while the PR is the active implementation candidate for the same-author linked bug report and has follow-up commits addressing review comments.

## Best Possible Solution

Keep this PR open, or land an equivalent maintainer-approved fix. The best path is to add a dedicated BlueBubbles `webhookSecret`, authenticate inbound BlueBubbles webhooks only against that secret, enforce that it differs from the API password across inherited account configs, expose it through setup/CLI metadata, and update docs, tests, and SecretRef metadata consistently.
