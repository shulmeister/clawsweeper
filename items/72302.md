## Summary

Current main already fixes the Windows Node v22/v24 raw drive-letter ESM loader failure reported against v2026.4.24. Main now disables native Jiti loading on win32, honors that non-native path in the shared plugin loader, converts Windows absolute plugin import paths to file URLs, and has focused regression coverage for the channel/plugin loader seams. The changelog records this under the Windows/native bundled plugin fix, but this checkout has no tag newer than v2026.4.24, so no shipped fixed release is assigned.

## Best Possible Solution

Close as implemented on current main. Keep the shared Windows-safe plugin loader policy and file-URL normalization in core; if the same crash reproduces on a build containing e672b61417af5c45b0431df6d9109a1f4b618ef5 or a newer release, ask for a fresh issue with exact version, full logs/diagnostics, Node version, install method, and affected channel/plugin IDs.
