## Summary

This issue should stay open. The reported Bedrock large tool-argument failure is recent, provider-specific, and current main does not show a Bedrock live-stream guard, retry, chunking path, or argument repair that would close it. Existing fixes cover adjacent malformed/replayed tool-call cases, but not this Bedrock `write`/`exec` empty-arguments path.

## Best Possible Solution

Keep the issue open and implement a Bedrock-specific regression fix in the provider stream/runtime boundary: detect `toolCall` blocks for required-argument tools that end with empty `{}` or malformed/truncated `partialJson`, surface a clear non-retrying provider/runtime error or trigger a bounded retry, and add unit coverage that simulates Bedrock `contentBlockStart` plus missing/truncated `toolUse.input` for large `write`/`exec` arguments. If product wants automatic large-content handling, add a separate design for chunked file writes or temp-file guidance rather than closing this bug.
