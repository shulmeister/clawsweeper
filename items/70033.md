## Summary

Keep #70033 open. The report is recent, provider-specific, and not covered by the currently shipped malformed tool-call argument repair: Amazon Bedrock models are registered as `bedrock-converse-stream`, while the repair wrapper is only enabled for `openai-completions` and Kimi on `anthropic-messages`. Related closed reports about empty/malformed tool arguments are useful context, but they do not prove this Bedrock large-argument failure is fixed on current `main`.

## Best Possible Solution

Keep the issue open and implement a focused Bedrock fix: add a `bedrock-converse-stream` regression fixture or live-gated probe for large `write`/`exec` arguments, verify whether Bedrock emits partial input, empty input, or truncation, then either enable/adapt the existing malformed tool-call argument repair for Bedrock or add Bedrock-specific retry/diagnostic handling. Coverage should include Amazon Bedrock tool calls with large string arguments and confirm OpenClaw does not repeatedly dispatch `{}` into required-parameter validation failures.
