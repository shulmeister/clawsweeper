## Summary

Keep #11057 open. Current main has adjacent long-context work, but it still does not implement native Claude Max/browser-session 1M support for `anthropic/claude-opus-4-6`: the 1M beta remains `params.context1m` opt-in, OAuth/subscription tokens strip the `context-1m` beta, and docs/tests explicitly preserve the standard-window fallback for that path. The custom-provider workaround in the discussion is useful, but it does not close the built-in Anthropic provider request.

## Best Possible Solution

Keep this issue open until OpenClaw either implements native Claude Max/browser-session 1M support for Opus 4.6, documents a maintainer-approved decision that Opus/Sonnet 4.6 1M remains API-key/custom-provider only, or supersedes it with a narrower canonical Anthropic OAuth long-context issue. The likely code surface is `extensions/anthropic/stream-wrappers.ts`, provider/model context accounting in `src/agents/context.ts` and model listing/status paths, plus Anthropic provider docs and regression tests.
