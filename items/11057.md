## Summary

Keep #11057 open. Current main has adjacent long-context support, including Opus 4.7 defaults and an explicit Opus/Sonnet 4.x `context1m` API-key path, but it still does not implement the reported Claude Max/browser-session Opus 4.6 1M behavior. The provided discussion also includes a custom-provider workaround and a prior Codex review keeping this open, but that workaround does not close the native Anthropic provider request.

## Best Possible Solution

Continue tracking this issue until OpenClaw either implements native Claude Max/browser-session 1M support for Opus 4.6, documents a maintainer-approved product decision that this will remain API-key/custom-provider only, or supersedes it with a narrower canonical Anthropic OAuth long-context issue. The likely code surface is the Anthropic provider stream wrapper plus context-window accounting and provider docs/tests.
