## Summary

Keep #11057 open. Current main still does not provide native 1M Opus 4.6 support for Claude Max/browser-session auth in the built-in Anthropic provider: 1M remains opt-in via `params.context1m`, `sk-ant-oat` token auth strips the 1M beta header, and context accounting/tests/docs preserve the standard-window behavior unless configured. The custom-provider/community-proxy workaround is useful but does not resolve the core provider request.

## Best Possible Solution

Keep this issue open until OpenClaw either implements native Claude Max/browser-session 1M support for Opus/Sonnet 4.6 in the built-in Anthropic provider, documents a maintainer-approved API-key/custom-provider-only decision, or designates a narrower canonical follow-up. The likely path spans `extensions/anthropic/stream-wrappers.ts`, `src/agents/context.ts`, model listing/status reporting, Anthropic provider docs, and OAuth/API-key regression tests.
