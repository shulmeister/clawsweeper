## Summary

Keep #11057 open. Current main still does not provide native 1M Opus 4.6 support for Claude Max/browser-session auth: 1M remains config-gated by `params.context1m`, and OAuth/setup-token style credentials still strip the `context-1m` beta header. The issue discussion’s custom-provider workaround is useful, and related #66766/#66862 track adjacent 4.6 1M behavior, but neither resolves this built-in Claude Max request on main.

## Best Possible Solution

Keep this issue open until OpenClaw either implements native Claude Max/browser-session 1M support for Opus 4.6 in the built-in Anthropic provider, documents a maintainer-approved API-key/custom-provider-only boundary, or links a narrower canonical follow-up. The likely fix needs to align Anthropic stream beta handling, OAuth/setup-token behavior, context-window reporting/model listing, docs, and tests; coordinate with #66766/#66862 if that work lands first.
