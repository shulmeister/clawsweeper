## Summary

Keep #66862 open. Current main does not implement the PR’s central 4.6/4.x behavior: `anthropic/claude-opus-4-6` without `context1m` is still expected to resolve to 200k, while the shipped no-flag 1M path is limited to Opus 4.7. The duplicate candidate #66790 is closed unmerged and the underlying bug #66766 remains open, so there is no merged or active canonical replacement. The PR also has a real unresolved integration concern: the Anthropic request path still only sends the 1M beta header when `context1m: true` is present and strips it for OAuth tokens, so any accepted fix needs to keep session planning and request capability aligned.

## Best Possible Solution

Keep the PR open for maintainer review or request changes. The best fix should update the context resolver and Anthropic provider request path together so `/status`, compaction/session planning, beta-header injection, and OAuth/legacy-token fallback all agree. If a newer PR lands the #66766 fix, close #66862 as superseded by that canonical merged PR.
