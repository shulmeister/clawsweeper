## Summary

Keep open. Current `main` does not implement the requested Control UI orchestration view. The Sessions tab is still a flat paginated table with search/sort and one bulk action (`Delete`), and its columns do not expose hierarchy, run status, model, elapsed time, or bulk kill/steer/retry/cleanup controls. The backend already exposes subagent relationship/status fields, but the browser-side session row type drops `parentSessionKey`, and the UI does not render an orchestration tree or dedicated task board. Existing subagent/task management is documented as slash-command driven (`/tasks`, `/subagents`, `/kill`, `/steer`), so the issue remains a valid frontend feature request rather than something already shipped. It is also not a `clawhub` issue: `VISION.md` explicitly lists web-frontend ergonomics as a core project priority.
