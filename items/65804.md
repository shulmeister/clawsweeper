## Summary

Keep open. Current `main` still has the stale cached subsystem file logger behavior, so the underlying bug is not implemented on `main`, and this PR as written carries a hot-path regression risk called out in review.

## Evidence

- **Current main still caches the subsystem file child once and never refreshes it:** `createSubsystemLogger()` keeps a single `fileLogger` and only initializes it on first file write in both `emitLog()` and `raw()`. There is no parent/root identity check or invalidation path, so long-lived subsystem loggers can stay bound to an old child logger.
  - file: [src/logging/subsystem.ts:327](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/logging/subsystem.ts#L327)
  - command: `nl -ba src/logging/subsystem.ts | sed -n '320,430p'`
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **Root logger is rebuilt when the rolling log path changes:** `getLogger()` re-resolves settings on each call and replaces `loggingState.cachedLogger` whenever `settingsChanged()` detects a different `file` path. The default file path comes from `defaultRollingPathForToday()`, so day rollover can rebuild the root logger while existing subsystem children still point at the old base.
  - file: [src/logging/logger.ts:397](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/logging/logger.ts#L397)
  - command: `nl -ba src/logging/logger.ts | sed -n '392,460p'; nl -ba src/logging/logger.ts | sed -n '500,590p'`
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **Current main therefore still matches the PR's reported root cause:** Because subsystem file children are cached independently while the root logger cache can rotate to a new file, the code path described in the PR body is still consistent with current `main`. I did not find a replacement fix in the checked-out code or visible local history.
  - file: [src/logging/subsystem.ts:349](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/logging/subsystem.ts#L349)
