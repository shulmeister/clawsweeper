## Summary

PR #65804 is a focused fix for the stale subsystem file-logger cache, but the same remaining work is already tracked by the older canonical PR #37549, with #67206 covering the related logging.file reconfiguration variant. Current main still has the stale cache, so this should close as a duplicate/superseded PR, not as implemented.

## Best Possible Solution

Close #65804 as a duplicate/superseded PR and concentrate maintainer review on #37549 for the midnight rollover fix, with #67206/#67168 covering the related logging.file reconfiguration invalidation path. The preferred implementation should use generation/base invalidation with a regression test and avoid adding per-log config resolution to the hot path.
