## Summary

PR #69822 should stay open. Current main does not implement the requested `socket.drain` teardown path: teardown mutations still emit only `sessions.changed` to `sessions.subscribe` listeners, message-only subscribers remain limited to transcript events, and unregistered `socket.drain` events would be rejected by the gateway event-scope guard. The prior broad PR #65539 is closed/unmerged and is superseded by this narrower PR, not the other way around; the linked issue #63374 being closed is not enough to close the PR because the observable gateway behavior is not present on main.

## Best Possible Solution

Keep #69822 open as the focused remaining implementation candidate for #63374. The best path is maintainer review of the gateway contract, then either land a scoped version that registers `socket.drain`, exposes a message-subscriber lookup seam, emits teardown drains to event and message subscribers, and preserves focused regression tests, or close only after an explicit product decision rejecting that behavior.
