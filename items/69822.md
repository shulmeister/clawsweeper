## Summary

Keep #69822 open. Current main does not implement the PR's central behavior: message-only session subscribers still do not receive a teardown `socket.drain` path, `socket.drain` is not registered as a gateway event/scope-guarded broadcast, and teardown `sessions.changed` broadcasts still use event subscribers only with `dropIfSlow: true`. The older #65539 is closed unmerged and does not supersede this focused replacement PR.

## Best Possible Solution

Keep #69822 open for human gateway maintainer review. The best path is to review or adapt the focused core change that registers `socket.drain`, exposes message-subscriber connection lookup, unions teardown recipients, protects teardown broadcasts from slow-buffer dropping, and keeps the regression tests. Do not close as implemented, duplicate, or superseded unless a separate maintained PR lands the same gateway behavior on main.
