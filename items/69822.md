## Summary

Keep open. Current main does not implement the PR's requested gateway behavior: `socket.drain` is not registered as a gateway event/scope-guarded broadcast, `emitSessionsChanged` still only targets `sessions.subscribe` listeners, and teardown `sessions.changed` broadcasts still use `dropIfSlow: true`. The related prior PR #65539 is closed unmerged, and the linked feature issue #63374 remains open, so there is no strong duplicate/superseded or implemented-on-main basis to close #69822.
