## Summary

Keep open. Current main still does not add `cron` to the internal hook event contract or bridge CronService lifecycle events into the internal `HOOK.md` system. The cron service emits lifecycle events internally and the gateway broadcasts them to WebSocket clients, but `server-cron` does not call `createInternalHookEvent`, `triggerInternalHook`, or `fireAndForgetHook` for cron events. Related implementation PRs in the provided GitHub context are closed unmerged, and the newer plugin `ctx.jobId` surface is only a partial plugin-hook aid, not the requested internal hook event support.

## Best Possible Solution

Keep #37855 open as the canonical issue for cron lifecycle events in the internal `HOOK.md` system. The best implementation is an additive core change: add a typed `cron` internal hook event based on `CronEvent`, emit it from `src/gateway/server-cron.ts` with `createInternalHookEvent` plus `fireAndForgetHook` or equivalent non-blocking error handling, and cover `added`, `updated`, `removed`, `started`, and `finished` in focused hook and gateway tests. If maintainers instead want this only in plugin hooks, that should be an explicit product/API decision with docs explaining why `HOOK.md` cron events are out of scope.
