## Summary

Keep #37855 open. Current main still has cron lifecycle events inside CronService and gateway WebSocket/webhook handling, but it does not expose those events through the internal HOOK.md hook contract. The same-scope implementation PRs #37881 and #37885 are closed unmerged in the provided GitHub context, and the later same-scope issue #47903 is also closed, so this issue remains the best canonical tracker.

## Best Possible Solution

Keep #37855 open as the canonical internal-hooks cron lifecycle request. The best implementation is an additive core change: add a typed cron internal hook event based on CronEvent, dispatch CronService onEvent data from src/gateway/server-cron.ts through createInternalHookEvent plus non-blocking fireAndForgetHook/triggerInternalHook handling, document cron, cron:added, cron:updated, cron:removed, cron:started, and cron:finished in docs/automation/hooks.md, and add focused hook/gateway tests covering mutation and execution events.
