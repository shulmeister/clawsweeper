## Summary

Keep #37855 open. Current main at e672b61417af5c45b0431df6d9109a1f4b618ef5 still does not expose cron lifecycle events through Internal Hooks. The cron service has the needed source events, but the public hook type contract, gateway bridge, docs, and tests still omit cron. Related implementation PRs #37881 and #37885 were closed unmerged, and #47903 is closed, so this issue remains the best focused tracker rather than a close candidate.

## Best Possible Solution

Keep #37855 open as the canonical Internal Hooks cron lifecycle request. The maintainer path is to add cron to InternalHookEventType, define a typed CronEvent-based hook context/type guard, dispatch CronService onEvent data from src/gateway/server-cron.ts through createInternalHookEvent plus non-blocking hook delivery, document cron, cron:added, cron:updated, cron:removed, cron:started, and cron:finished on the public hooks page, and add focused hook/gateway tests for mutation and execution events. The older closed PRs can be mined for approach details, but they did not ship.
