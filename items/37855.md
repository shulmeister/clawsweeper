## Summary

Keep #37855 open. Current main still produces cron lifecycle events internally and broadcasts them over the Gateway cron event channel, but it does not expose cron lifecycle events through the Internal Hooks contract, factory, Gateway bridge, docs, or hook tests. The related implementation PRs/discussions in the provided GitHub context were closed unmerged, so this issue remains the focused tracker.

## Best Possible Solution

Keep #37855 open as the canonical Internal Hooks cron lifecycle request. The maintainer path is to add cron to InternalHookEventType, define a CronEvent-based hook context/type guard, dispatch CronService onEvent data from src/gateway/server-cron.ts through createInternalHookEvent plus non-blocking hook delivery, document cron, cron:added, cron:updated, cron:removed, cron:started, and cron:finished on https://docs.openclaw.ai/automation/hooks, and add focused hook/gateway tests for mutation and execution events. The closed PRs #37881 and #37885 can be mined for implementation details, but they did not ship.
