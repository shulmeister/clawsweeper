## Summary

Current `main` no longer requires a Feishu `card` argument in the shared `message` tool schema, and the Feishu send path accepts plain-text sends without a card.

## Close Comment

Closing as implemented on `main` after a Codex review. The current shared `message` tool schema no longer requires a Feishu `card` payload (`src/agents/tools/message-tool.ts:90`), and the Feishu channel is not contributing any extra schema that would make `card` required (`extensions/feishu/src/channel.ts:182`). The current Feishu send path accepts plain text/media sends and only rejects the call when all of text/message, media, and card/presentation are missing (`extensions/feishu/src/channel.ts:695`), with regression coverage in `extensions/feishu/src/channel.test.ts:803`. The changelog also records the schema fix in `v2026.3.24-beta.1` (`CHANGELOG.md:2117`). If this still reproduces on a current build, please open a new issue with the exact tool payload and the runtime-exposed tool schema from that build.
