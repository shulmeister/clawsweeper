## Summary

Keep this PR open. Current main at 9626ef274ae2346b3d519f99c31856be98aab570 still calls OpenAI Responses, Azure OpenAI Responses, and OpenAI chat completions directly, and no `LLM_MAX_CONCURRENT` or provider-level limiter surface exists. The PR is a plausible implementation candidate, but its current diff is incomplete and merge-blocked by the existing review findings.

## Best Possible Solution

Keep this PR open for author or maintainer revision. A mergeable solution should add a tested provider-request limiter covering OpenAI Responses, Azure OpenAI Responses, and OpenAI chat completions; hold limiter slots for the full stream lifetime; reject queued work promptly on `AbortSignal`; safely parse invalid or empty concurrency configuration; and document the chosen config surface. If maintainers implement that separately, this PR can then be closed as superseded.
