## Summary

Current `main` no longer matches the reported root cause: logging config is loaded through the same active-config path resolver as the rest of config IO, and the logger uses `logging.file` when present instead of the `/tmp/openclaw/openclaw-YYYY-MM-DD.log` fallback.

## Evidence
