## Summary

Keep #46056 open. Current main still has no explicit shell-builtin allowlist path: each top-level chain segment must be satisfied by an allowlist match, safeBins, skill auto-allow, or a skill prelude. A builtin such as `cd` still resolves through the normal executable-path machinery and will miss normal path/bare-name allowlist entries when there is no resolved executable path. The docs now state that chained commands need every top-level segment to match, but they do not implement or clearly document the requested builtin handling/workaround. The issue discussion also contains a maintainer acknowledgement that the behavior is annoying, so this should remain for explicit product/security judgment.
