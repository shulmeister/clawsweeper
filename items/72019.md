## Summary

Keep open. Current main still duplicates the native subagent task in both the subagent system prompt and the first child user message. The ClawHub skill linked in discussion is only a workaround, and open PRs #72028, #72053, and #72570 are implementation candidates with closing syntax, so this issue should remain open until one canonical fix merges.

## Best Possible Solution

Keep this issue open and review one canonical fix, likely #72053, #72028, or #72570. The fix should single-source the subagent task text, preserve multiline task formatting, and add a regression test that inspects the child system prompt and first gateway user message together.

## Risks / Open Questions

- Maintainers still need to choose the intended prompt contract: keep the full task only in the system prompt with a short user kickoff, or keep it only in the first user message.
- Changing task placement can affect providers or models that weight system and user roles differently, so the selected PR should include focused regression coverage for both prompt surfaces.
- The ClawHub skill linked in discussion may help users reduce orchestration cost, but it does not fix the duplicated native sessions_spawn payload in OpenClaw core.
