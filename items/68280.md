## Summary

Keep open. Current main still lacks the PR's fail-fast behavior for missing local probe auth, while the open PR remains a plausible implementation candidate for the related loopback `device-required` log-spam path after maintainer/security review.

## Best Possible Solution

Keep this PR open as the implementation candidate. Revise/review it so fail-fast applies only to local loopback probes with explicitly required but unresolved auth, preserves explicit URL/RPC override and unset/none/trusted-proxy auth behavior, redacts normal user-facing failure reasons, sanitizes every rendered probe-error surface, and then run the focused gateway probe, gateway status, shared status, and daemon status regression tests.
