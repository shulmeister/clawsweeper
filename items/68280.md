## Summary

Keep open. Current main still attempts local gateway/status probes with empty auth when local auth cannot be resolved, so the PR's central fail-fast behavior is not implemented. The PR is a plausible focused gateway/CLI fix, but its current head has unresolved review concerns around false-positive fail-fast behavior for unset auth mode and explicit override targets, plus safe rendering of propagated failure text.

## Best Possible Solution

Keep the PR open for author or maintainer rework. The best path is to implement fail-fast only for actual local daemon loopback probes when local auth mode requires credentials and no local credential resolves, while preserving explicit URL/RPC override behavior, adding regression tests for unset/none/trusted-proxy auth modes and explicit override targets, and sanitizing any surfaced failure text at CLI rendering boundaries.
