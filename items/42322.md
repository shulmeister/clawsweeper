## Summary

Keep PR #42322 open. Current main does not implement Feishu `replyToMode`: the Feishu config schema, generated metadata, bot dispatch path, and reply dispatcher still only expose `replyInThread` and always pass the selected `replyToMessageId` for normal group replies. The PR is a focused channel improvement, but its current branch also needs maintainer review because the PR discussion flags a remaining `replyToMode="first"` streaming/media edge case.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to add Feishu `replyToMode` deliberately across schema/types, generated config metadata/docs, bot dispatch, and reply-dispatcher behavior, with focused tests for `off`, `first`, and `all`, including streaming-card and media sends. The implementation should preserve current main's `replyInThread` and normal-group #32980 behavior unless maintainers intentionally change that product contract.
