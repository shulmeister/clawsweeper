## Summary

Keep this issue open. Current main still hardcodes `SESSIONS_HISTORY_TEXT_MAX_CHARS` to 4000 in `sessions_history`, while the env-backed `readEnvInt` pattern exists for comparable bash output limits. The latest release tag in this checkout has the same hardcoded sessions-history code, and the related fix PR #65029 was closed unmerged.

## Best Possible Solution

Keep the issue open and implement the narrow fix in `src/agents/tools/sessions-history-tool.ts`: read `SESSIONS_HISTORY_TEXT_MAX_CHARS` through the existing `readEnvInt` helper with a sensible default and bounds, then add a focused regression test in `src/agents/openclaw-tools.sessions.test.ts` proving that an env override above 4000 changes sessions-history text truncation. If the env var is intended to be public, document it alongside the sessions-history bounded-recall docs.
