## Summary

Keep #20935 open. Current main has a narrow memory event journal for recall, promotion, and dreaming plus an unrelated config audit log, but it does not provide an append-only audit log for every MEMORY.md or memory/*.md mutation with what/when/why/session/user context or tamper-evidence. The Pinchy comment is useful external prior art, and the latest issue discussion also points to remaining OpenClaw memory-core work rather than a completed feature.

## Best Possible Solution

Keep #20935 open. The best implementation is a first-class memory-change audit layer in memory-core plus shared host/tool write hooks, emitting append-only records for every MEMORY.md and memory/*.md mutation with timestamp, operation, relative path, changed-content summary or diff/hash, trigger/why, session/run/user context, and optional integrity signing. Cover promotion, memory flush, dreaming, and ordinary write/edit paths with regression tests, then document the audit surface in the public memory docs.
