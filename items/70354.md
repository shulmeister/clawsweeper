## Summary

Keep #70354 open. Current main mitigates fresh canonical Linux installs with `Restart=always`, but the reported `update.run`/SIGUSR1 supervised restart path still exits 0, so existing or manual systemd units using `Restart=on-failure` can still be left dead. Open PR #70466 appears to be the direct fix, but it is not merged in the provided state.

## Best Possible Solution

Keep #70354 open until main lands #70466 or an equivalent fix: Linux/systemd supervised SIGUSR1/update restarts should exit non-zero when relying on `Restart=on-failure`, while preserving launchd’s clean-exit behavior and spawned restart exit 0. Add a Linux/systemd supervised restart regression test, and consider auditing or repairing existing `Restart=on-failure` gateway units.
