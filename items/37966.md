## Summary

Keep #37966 open. Current main at e672b61417af5c45b0431df6d9109a1f4b618ef5 still does not honor explicit `cacheRetention` for the bundled LiteLLM Anthropic proxy route: LiteLLM is configured as `openai-completions`, `resolveCacheRetention` only accepts custom non-Bedrock providers on `api: "anthropic-messages"`, and the OpenAI-compatible `cache_control` payload wrapper is still OpenRouter-gated. The related fix PR #38221 was closed unmerged, and the reporter reconfirmed the issue on 2026-04-21.

## Best Possible Solution

Keep #37966 open until OpenClaw either implements a conservative LiteLLM Anthropic prompt-cache pass-through or explicitly decides and documents that this route is unsupported. A likely fix belongs in the LiteLLM/provider proxy wrapper path or a shared proxy-cache wrapper: when explicit `cacheRetention` is configured for a LiteLLM Anthropic-backed model on `openai-completions`, inject Anthropic `cache_control` markers into the OpenAI-compatible payload, then add regression coverage for `litellm/claude-opus-4-6` and update the LiteLLM/prompt-caching docs.
