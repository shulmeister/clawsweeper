## Summary

Keep #37966 open. Current main still does not honor explicit `cacheRetention` for the bundled LiteLLM `openai-completions` route: the default LiteLLM model is configured for OpenAI-compatible transport, cache-retention eligibility only admits custom providers when `modelApi === "anthropic-messages"`, and Anthropic `cache_control` payload injection remains OpenRouter-gated. The linked fix PR #38221 was closed unmerged, and the reporter reconfirmed the issue on 2026-04-21.

## Best Possible Solution

Keep #37966 open until OpenClaw either implements conservative LiteLLM Anthropic prompt-cache pass-through or records an explicit maintainer product decision that this route is unsupported. A likely implementation belongs in the bundled LiteLLM/provider wrapper path: when explicit `cacheRetention` is configured for an Anthropic-backed LiteLLM `openai-completions` model, inject Anthropic `cache_control` markers into the OpenAI-compatible payload, add regression coverage for `litellm/claude-opus-4-6`, and update the public LiteLLM/prompt-caching docs.
