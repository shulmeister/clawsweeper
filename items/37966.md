## Summary

Keep issue #37966 open. Current main still does not implement the reported LiteLLM OpenAI-compatible Anthropic cache-control path: the bundled LiteLLM provider uses `openai-completions`, cache retention remains eligible only for direct Anthropic/Bedrock/custom `anthropic-messages` paths, and the only generic `cache_control` payload wrapper is scoped to verified OpenRouter routes. The related PR #38221 was closed unmerged per the provided GitHub context, and current tests only cover LiteLLM when configured as `anthropic-messages`, not the documented LiteLLM `/v1` path in the report.

## Best Possible Solution

Keep the issue open until OpenClaw either implements a conservative LiteLLM Anthropic pass-through path or maintainers explicitly decide and document that this setup is unsupported. A likely fix belongs in the LiteLLM/provider wrapper path or shared proxy wrapper logic: detect explicitly configured `cacheRetention` for LiteLLM Anthropic model refs on `openai-completions`, inject Anthropic `cache_control` blocks into the OpenAI-compatible payload, and add regression coverage for `litellm/claude-opus-4-6` on the documented LiteLLM route.
