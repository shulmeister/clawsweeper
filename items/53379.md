## Summary

Current main already rejects ambiguous cron announce delivery at create/update time, which addresses the reported runtime failure when multiple channels are configured.

## Evidence

  - file: [src/gateway/server-methods/cron.ts:40](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/server-methods/cron.ts#L40)
