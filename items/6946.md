## Summary

Current main still lacks the requested configurable Telegram processing-indicator message lifecycle. Telegram has adjacent stream-preview, native-command progress-placeholder, ack-reaction, and status-reaction behavior, but none implements “send a standalone ⌛️ on inbound receipt, then delete it on final reply or edit it on failure.” The issue discussion and related closed PRs remain useful prior art rather than a shipped/superseding fix.

## Best Possible Solution

Keep the issue open until the bundled Telegram plugin implements an opt-in processing-indicator lifecycle: add config schema/types/docs/tests, send a standalone placeholder immediately after an eligible inbound message is accepted, delete it after successful final delivery, and edit or replace it on dispatch failure. Use closed PRs #43021 and #43720 as prior art, but implement fresh against current main.
