## Summary

Current main still does not implement the requested configurable Telegram processing-indicator message lifecycle. It has adjacent feedback paths, including typing/status reactions, preview tool-progress edits, default verbose tool-progress messages, and native-command progress placeholders, but none is the issue's standalone 'send an immediate visible message on inbound receipt, then delete on final reply or edit on failure' feature. The existing issue discussion also already treated typing TTL/tool-progress work as complementary, and the two PRs that attempted this exact feature were closed unmerged.

## Best Possible Solution

Keep the issue open. The best path is an opt-in implementation in the bundled Telegram plugin: add schema/types/docs/generated metadata, send a standalone placeholder immediately after an eligible inbound Telegram message is accepted, carry the sent message ID through dispatch, delete it after successful final reply delivery, and edit or replace it on dispatch failure. Use the native-command progress placeholder code and the closed PRs #43021/#43720 as prior art, but implement fresh against current main with Telegram dispatch tests for success, error, preview-streaming, and group/topic behavior.
