## Summary

Keep open. Current main has adjacent Telegram visibility features, but it does not implement the requested configurable processing indicator message that is sent immediately, then deleted or edited on reply/failure. The related PRs #43021 and #43720 proposed this exact feature but were closed unmerged, so they do not supersede the open issue.

## Evidence

- **No matching processing-indicator surface on main:** A repo-wide search for `processingIndicator`, `processing indicator`, `messages.processing`, `channels.telegram.processing`, `deleteOnReply`, and `showErrorOnFailure` returned no hits on current main.
  - command: `rg -n "processingIndicator|processing indicator|messages\.processing|channels\.telegram\.processing|deleteOnReply|showErrorOnFailure" . --glob '!*node_modules*' --glob '!pnpm-lock.yaml' --glob '!bun.lock'`
  - sha: [0bcb4c95c185](https://github.com/openclaw/openclaw/commit/0bcb4c95c185909b12d463b04d49b16ead3459d5)
- **Existing Telegram preview is not the requested lifecycle:** Telegram preview streaming uses an edited preview message and documents final text replies as editing that same preview in place, rather than sending a standalone processing placeholder and deleting it before a separate final reply.
  - file: [docs/channels/telegram.md:275](https://github.com/openclaw/openclaw/blob/0bcb4c95c185909b12d463b04d49b16ead3459d5/docs/channels/telegram.md#L275)

  - file: [extensions/telegram/src/bot-message-dispatch.ts:405](https://github.com/openclaw/openclaw/blob/0bcb4c95c185909b12d463b04d49b16ead3459d5/extensions/telegram/src/bot-message-dispatch.ts#L405)
