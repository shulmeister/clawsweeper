## Summary

Keep open. Current main still does not implement the requested Telegram standalone processing-indicator message lifecycle. Telegram has adjacent feedback mechanisms (ack/status reactions, preview-stream tool progress, and native-command progress placeholders), but there is no configurable normal inbound-turn placeholder message sent immediately, deleted after final reply, and edited on failure. The related implementation PRs #43021 and #43720 were closed unmerged, so this issue remains the canonical place for the remaining work.

## Best Possible Solution

Keep #6946 open. The best path is an opt-in implementation in the bundled Telegram plugin: add typed config/schema/generated metadata/docs, send a standalone placeholder after an eligible inbound Telegram message is accepted, carry its message id through dispatch, delete it after successful final delivery, and edit or replace it on dispatch failure. Reuse native-command progress placeholders and preview cleanup as prior art, with focused Telegram dispatch tests for success, error, preview streaming, group/topic, and cleanup behavior.
