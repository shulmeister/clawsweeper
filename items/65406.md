## Summary

Close as clawhub. PR #65406 is a docs-only guide for a specific community billing proxy, not a missing OpenClaw core/provider API. Current project guidance says optional third-party capabilities and plugin promotion belong in ClawHub/npm, while core docs should document extension points and supported core behavior. Current docs already cover Anthropic API-key/Claude CLI routes and a general Claude Max community proxy page, and the existing config/plugin surfaces are sufficient for a proxy or bundle-style/plugin-owned guide outside core.

## Best Possible Solution

Do not merge this third-party proxy guide into core docs. Keep core docs focused on Anthropic/Claude CLI support and extension points; the contributor should host the guide with the linked `openclaw-billing-proxy` project or package it as a ClawHub/npm plugin or bundle. Reopen core work only if a concrete missing provider/plugin API is identified or a maintainer explicitly wants this proxy curated in OpenClaw docs.
