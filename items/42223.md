## Summary

Keep PR #42223 open. Current main still preserves the reported failure mode: a collapsed sidebar group that contains the active tab is forced visible, and the linked bug issue #42167 was closed as superseded by this PR, not as fixed. The PR patch is stale against the current `nav-section` refactor, but the underlying fix is still useful.

## Best Possible Solution

Keep #42223 open as the canonical fix path for #42167. Update the PR against current main by changing the `showItems` logic so `isGroupCollapsed` hides child items even when the active tab is inside the group, and add a focused Control UI regression test for collapsing an active group.
