## Summary

Keep PR #65992 open. Current main has the related Unicode slugification fix, but `ingestMemoryWikiSource` still uses only the readable title slug for both `pageId` and `sources/<slug>.md`, so source pages with identical or equivalent slugs can still silently overwrite each other. The PR is a focused, plausible bug fix with regression coverage and active author follow-up, not a conservative cleanup close.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to either land a rebased version of the focused source identity fix with its regression tests, or ask for targeted changes if maintainers want a different identity/index strategy. Closing as implemented would be premature because current main still uses slug-only source page identities.
