## Summary

Keep open. Current main still uses the normalized title slug as the entire memory-wiki local ingest source identity, so two files with the same title/slug can still target the same `source.<slug>` and `sources/<slug>.md`. This PR remains a focused implementation candidate for the collision bug linked from #65965.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is still a memory-wiki-owned fix that preserves readable source page names while adding a stable collision-resistant source identity suffix and regression tests for legacy reuse, symlink aliases, same-slug titles, and same-title different source paths.
