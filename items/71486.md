## Summary

Keep open. Current `main` still has the exact parsing gap the PR describes: provider-side model resolution strips everything after the first `@`, and the shared auth-profile splitter only treats `q*`/`Nbit` suffixes as quant markers, not `iq*`. This is not implemented on `main`, not stale, and the PR is addressing a real unresolved bug.

## Evidence

- **Current main still truncates model ids at the first @:** `stripModelProfileSuffix()` trims the value, finds the first `@`, and slices everything after it away. For `lmstudio/qwen3.6-27b@iq3_xxs`, that would reduce the model id to `qwen3.6-27b`.
