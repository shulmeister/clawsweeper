## Summary

Issue #60919 requests an opt-in to derive text model fallback candidates from agents.defaults.models. Current main does not implement that behavior, but the exact remaining work is already tracked by the linked open PR #60922, so the issue can close as superseded by that canonical implementation thread.

## Evidence

- **Current fallback resolver uses only explicit fallback sources:** On current main, resolveFallbackCandidates chooses params.fallbacksOverride when defined, otherwise reads agents.defaults.model via resolveAgentModelFallbackValues, then iterates only that fallback list. There is no branch deriving candidates from agents.defaults.models.
