## Summary

Current `main` does not match the PR’s premise. Memory capability registration is an exclusive memory-slot surface, `active-memory` is not a memory-slot plugin and does not call `registerMemoryCapability`, and `memory-core` remains the plugin that exports `publicArtifacts` for wiki bridge use. The proposed merge behavior is therefore not justified by current code or docs.

## Evidence

- **Memory capability is documented as exclusive:** The Plugin SDK docs describe `registerMemoryCapability` under `Exclusive slots` and call it the preferred exclusive memory-plugin API, which means current contracts expect one active memory capability rather than additive merges.
  - file: [docs/plugins/sdk-overview.md:204](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/docs/plugins/sdk-overview.md#L204)
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Current slot policy disables extra single-kind memory plugins:** `resolveMemorySlotDecisionShared` enables only the selected memory plugin for single-kind memory plugins; non-selected single-kind memory plugins are disabled once a slot is chosen or already filled.
  - file: [src/plugins/config-activation-shared.ts:355](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/plugins/config-activation-shared.ts#L355)
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Only memory plugins may register the memory capability:** The registry rejects `registerMemoryCapability` from non-memory plugins and also skips dual-kind plugins that are not selected for the memory slot.
  - file: [src/plugins/registry.ts:1563](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/plugins/registry.ts#L1563)
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **`active-memory` is not a memory-slot plugin:** The `active-memory` manifest declares no `kind`, so it is not a `kind: "memory"` plugin eligible for exclusive memory capability registration.
  - file: [extensions/active-memory/openclaw.plugin.json:1](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/extensions/active-memory/openclaw.plugin.json#L1)
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **`active-memory` does not register memory capability on current main:** The plugin entry defines commands and runtime behavior but no `kind: "memory"`, and repository search shows no `registerMemoryCapability(...)` call in `extensions/active-memory/index.ts`.
  - file: [extensions/active-memory/index.ts:1872](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/extensions/active-memory/index.ts#L1872)
  - command: `rg -n "registerMemoryCapability\\(" extensions/active-memory/index.ts`
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **`memory-core` is the current public-artifact provider:** `memory-core` is the memory plugin on current main that registers `publicArtifacts`, which is what wiki bridge reads through `listActiveMemoryPublicArtifacts()` from `memory-state`.
  - file: [extensions/memory-core/index.ts:24](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/extensions/memory-core/index.ts#L24)
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
