## Summary

Keep PR #70466 open. Current main still treats `spawned` and `supervised` respawn modes together and exits with status 0 after a supervised restart, so the PR’s central systemd `Restart=on-failure` fix is not implemented on main. The linked issue #70354 provides a concrete reproduction and journal evidence, and the PR is a narrow, relevant fix; it should not be auto-closed as obsolete.

## Best Possible Solution

Keep the PR open and have the author or maintainer finish it by splitting the spawned and supervised branches in `src/cli/gateway-cli/run-loop.ts`, preserving spawned exit 0 and launchd delay, making supervised systemd exit non-zero, and adding a focused non-launchd/systemd regression test in `src/cli/gateway-cli/run-loop.test.ts`.
