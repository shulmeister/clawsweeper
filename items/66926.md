## Summary

Keep #66926 open. Current main still enables tagged reasoning for direct Google/Gemini provider paths, still injects the exact <think>/<final> prompt language the report identifies, and the Google transport only turns native Gemini functionCall parts into executable tool calls. Main has useful thinkingConfig/Gemma 4 normalization, but that is only a partial adjacent fix and does not remove the tagged prompt/tool-call conflict.

## Best Possible Solution

Keep the issue open and land a narrow Google-provider fix: remove or conditionally disable tagged reasoning mode for direct google-generative-ai models, or revise the tagged prompt so native function calls are explicitly outside the <think>/<final> text contract and must be emitted through the provider's function-call mechanism. Add regression coverage for Google reasoningTagHint/system prompt behavior with tools, plus a focused transport test that tool schemas and native functionCall handling remain intact. Track the remaining Gemma 4/thinkingConfig reliability question separately if live verification still shows text-only tool calls after the prompt fix.
