## Summary

Keep #17683 open. Current main has useful partial controls for one-shot cron runs, lightweight context, timeouts, tool allowlists, exec approvals, and post-run usage/cost telemetry, but it still lacks the requested first-class scoped/script-limited automation mode with native per-job token caps, daily spend ceilings, budget aborts, dry-run/cost previews, and admin-defined script-only execution. The existing issue discussion also explicitly kept this open for the same missing behavior.

## Best Possible Solution

Keep #17683 open as the product/API tracker. The best implementation would add a native scoped automation mode across cron, hooks, gateway protocol, CLI, and docs: per-job token caps, daily spend ceilings, graceful budget aborts based on runtime usage accounting, admin-defined script-only execution, dry-run/plan cost preview, and structured planned-vs-actual cost logs. Smaller maintainable follow-ups could split budget enforcement, script-only execution, and dry-run/cost preview.
