## Summary

Current main still has partial building blocks for scoped cron automation, but it does not implement the requested first-class scoped/script-limited mode with native token/spend budgets, budget aborts, script-only execution, dry-run execution, or cost preview.

## Best Possible Solution

Keep #17683 open as the product/API tracker. The best implementation would add a native scoped automation mode across cron/hooks/gateway protocol and docs: per-job token caps, daily spend ceilings, graceful budget aborts using runtime usage accounting, admin-defined script-only execution with dry-run/plan preview, and structured logs that show planned and actual cost. If maintainers prefer incremental work, split it into narrower issues for budget enforcement, script-only execution, and dry-run/cost preview.
