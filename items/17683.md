## Summary

Keep #17683 open. Current main has partial building blocks for scoped automation, including isolated cron sessions, lightweight context, tool allowlists, timeouts, and post-run usage/cost telemetry. It still does not implement the requested first-class scoped/script-limited mode with native per-job token caps, daily spend ceilings, graceful budget aborts, dry-run/plan cost preview, or admin-defined script-only execution. The existing 2026-04-26 review comment already reached the same keep-open conclusion, and the relevant commits since then do not add the missing budget/script-mode behavior.

## Best Possible Solution

Keep #17683 open as the product/API tracker. The best implementation would add a native scoped automation mode across cron, hooks, gateway protocol, CLI, and docs: per-job token caps, daily spend ceilings, graceful budget aborts using runtime usage accounting, admin-defined script-only execution, dry-run/plan cost preview, and structured logs for planned and actual cost. If maintainers want smaller steps, split follow-ups for budget enforcement, script-only execution, and dry-run/cost preview.
