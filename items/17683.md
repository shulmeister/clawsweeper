## Summary

Keep #17683 open. Current main has partial scoped-automation controls such as isolated cron sessions, light context, timeouts, tool allow-lists, run logs, and usage-cost reporting, but it still does not implement the requested first-class scoped/script-limited mode with per-job or daily spend ceilings, hard token caps, dry-run cost preview, graceful budget aborts, or an admin-defined script-only execution registry. The existing maintainer/Codex issue comment reached the same keep-open conclusion on April 26, 2026, and the current source still supports that conclusion.

## Best Possible Solution

Keep #17683 open as the broader product/API tracker. The best path is a native scoped automation mode spanning cron payloads, gateway protocol/schema, `/hooks/agent`, CLI flags, embedded runner accounting/enforcement, docs, and tests: per-job token caps, daily or hourly spend ceilings, graceful budget aborts, admin-defined script-only execution, dry-run cost previews, and structured planned-vs-actual run logs. Treat open #38248 as a narrower cost-guard component, not as a complete substitute for this issue.
