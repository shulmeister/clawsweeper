## Summary

Keep #17683 open. Current main still has only partial scope controls for cron/hooks/agent turns: isolated sessions, light context, timeouts, tool allow-lists, and post-run usage/cost telemetry. It does not provide the requested first-class scoped/script-limited mode with native token/spend ceilings, budget aborts, admin-defined script-only execution, dry-run cost previews, or planned-vs-actual run logs. The existing maintainer/Codex comment already reached this conclusion, and the relevant code paths have not changed materially since that review.

## Best Possible Solution

Keep #17683 open as the product/API tracker. The best path is a native scoped automation mode covering cron, hooks, gateway protocol, CLI, embedded runner accounting, docs, and tests: per-job token caps, daily spend ceilings, graceful budget aborts, admin-defined script-only execution, dry-run/plan cost preview, and structured planned-vs-actual run logs.
