## Summary

Current main still has the reported restart/resume gap. Exec approvals are process-local in a fresh in-memory manager, stale IDs still return INVALID_REQUEST, replay sanitization strips runtime details but does not rewrite approval-pending transcript content or re-register old approvals, and restart recovery is generic rather than approval-aware.

## Best Possible Solution

Keep this issue open. The likely fix belongs in the agent replay/session repair path or in an approval-manager persistence/re-registration contract: on resume after restart, detect exec tool results with `status: "approval-pending"` whose ID is absent from the live manager and rewrite them to an expired/error tool result so the agent re-issues exec, or persist and re-register pending approval records. Add a focused regression test that simulates a gateway restart with an approval-pending transcript tail and proves the resumed turn does not keep resolving the stale ID.
