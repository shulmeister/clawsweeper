## Summary

Keep #43765 open. Current main only covers part of the heartbeat event-consumption problem; several central behaviors from the PR remain absent, especially Feishu websocket health reporting and the exec non-zero exit semantics. The PR branch also has recorded follow-up review concerns, so the best path is not auto-close but maintainer/author follow-up with a rebased, split, validated patch.

## Best Possible Solution

Keep the PR open for maintainer judgment. The best path is to preserve the heartbeat event-consumption behavior already present on main, then split/rebase the remaining useful pieces into focused changes: Feishu runtime health snapshots/statusSink wiring, explicit exec/process terminal-status semantics with regression tests, and any first-visible diagnostics work as a separate design/runtime PR. The PR author should also resolve the recorded P0 branch regressions before review can continue.
