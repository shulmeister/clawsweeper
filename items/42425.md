## Summary

Current main still loads internal hooks only from the default workspace during gateway startup, so PR #42425 is not obsolete or implemented. The linked issue #42072 was closed because this PR is the canonical tracker, but the PR itself still carries the remaining fix work and should stay open for maintainer review.

## Best Possible Solution

Keep PR #42425 open as the canonical fix path. The best implementation is a focused startup hook-loading path that enumerates `listAgentWorkspaceDirs(cfg)`, loads shared/bundled/managed/plugin hooks once, loads workspace-local hooks per configured workspace with event/session/agent scoping, preserves legacy handlers relative to the default workspace, and keeps regression coverage for non-default workspace discovery, scope isolation, failure isolation, and shared/local override behavior.
