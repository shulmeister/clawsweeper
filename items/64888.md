## Summary

Current `main` does not implement either requested change from #64888. The embedded runner still has no catch path that guarantees terminal lifecycle metadata on an uncaught `runEmbeddedAttempt` failure, and the liveness union/resolver still do not support a distinct `"cancelled"` state.

## Evidence

- **Liveness type still lacks `cancelled`:** `EmbeddedRunLivenessState` is still defined as only `"working" | "paused" | "blocked" | "abandoned"`, so the requested new terminal state is not present on current `main`.
  - file: [src/agents/pi-embedded-runner/types.ts:98](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/src/agents/pi-embedded-runner/types.ts#L98)
  - command: `nl -ba src/agents/pi-embedded-runner/types.ts | sed -n '88,118p'`
  - sha: [ae5c6573670b](https://github.com/openclaw/openclaw/commit/ae5c6573670b3742b156899bd2b9c36b8677f191)
- **Aborts still collapse into `blocked`:** `resolveRunLivenessState()` still returns `"blocked"` when `(aborted || timedOut) && payloadCount === 0`; there is no branch that maps user aborts to a distinct `"cancelled"` state.
  - file: [src/agents/pi-embedded-runner/run/incomplete-turn.ts:254](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/src/agents/pi-embedded-runner/run/incomplete-turn.ts#L254)

  - sha: [ae5c6573670b](https://github.com/openclaw/openclaw/commit/ae5c6573670b3742b156899bd2b9c36b8677f191)
- **No error boundary around embedded attempt call:** `runEmbeddedPiAgent()` still directly awaits `runEmbeddedAttemptWithBackend(...)` inside the main loop. The surrounding structure has a `finally` for disposal/cleanup, but no `catch` that sets fallback terminal lifecycle metadata before rethrowing unexpected attempt errors.
  - file: [src/agents/pi-embedded-runner/run.ts:834](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/src/agents/pi-embedded-runner/run.ts#L834)

- **Lifecycle state defaults to `working` unless terminal meta is set:** The subscribe state initializes `livenessState` to `"working"`, and `setTerminalLifecycleMeta()` is the path that mutates it. If an uncaught attempt exception bypasses those terminal-meta calls, there is still no explicit fallback state emitted from this runner path.
