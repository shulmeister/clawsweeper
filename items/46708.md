## Summary

Current main does not give enough evidence to close #46708. The backend has pending approval listing/replay primitives, but the Control UI still only reacts to live `exec.approval.requested` events and prunes local state on reconnect rather than hydrating pending approvals. The TUI has no exec/plugin approval RPC wiring, Discord native approvals require explicit `enabled: "auto"`/`true` plus approvers despite docs claiming unset auto-enable, and a newer open regression (#71804) reports the same Discord approval path still broken on v2026.4.24.
