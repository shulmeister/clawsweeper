## Summary

Current main addresses the report’s observable failure: pairing-required/scope-upgrade errors now surface actionable details instead of opaque closes, `openclaw devices list` and `approve` have a loopback-only local fallback so operators can recover pairing, and the gateway/client handshake path has been hardened for busy local startup cases. The remaining reverse-proxy discussion is configuration guidance, now covered in docs.

## Evidence
