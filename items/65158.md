## Summary

Cross-channel message sync/forwarding is a useful idea, but current main treats this as plugin territory rather than OpenClaw core. The plugin SDK already exposes channel-specific routing, shared message dispatch, and inbound/outbound message hooks that are sufficient for a ClawHub plugin to mirror or forward messages between channels such as WeChat and WebChat.

## Risks / Open Questions

- No existing ClawHub package was linked on the issue, so this close relies on the current plugin boundary being sufficient rather than on an already-published off-the-shelf implementation.
- If maintainers later want a first-party unified inbox/history UX across multiple channels, that broader product direction could justify reopening as a core feature request.
