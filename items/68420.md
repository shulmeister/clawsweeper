## Summary

Current main implements the PR's central behavior through the runtime outcome classifier and shared model-fallback loop. Incomplete terminal results, including non-GPT `Agent couldn't generate a response` payloads matching the Gemini-style gap in the PR, are classified as fallback-eligible and converted into `FailoverError`s so configured fallback candidates run. The exact PR taxonomy (`incomplete_response` reason / `incomplete_turn` trace stage) is not present, but the user-visible fallback problem is solved on main.

## Best Possible Solution

Close this PR and keep the current main implementation, which centralizes unusable terminal-result handling in the runtime outcome classifier and shared model-fallback loop. If maintainers still want the exact `incomplete_response` taxonomy, `incomplete_turn` trace label, or no-fallback payload wording from this PR, that should be tracked as a narrower follow-up rather than keeping this obsolete PR open.
