## Summary

Keep open. Current main contains the narrower #70678 quiet-session watchdog mitigation, but the broader WhatsApp regression reported here is still not resolved: the issue has fresh cross-platform corroboration, outbound sends still fail when the active listener is absent, reconnect retry still creates an active-listener gap, the shared health monitor can restart a reconnecting/disconnected account, and group hydration failure is still a one-shot log-and-continue path with no retry or recovery coverage.

## Best Possible Solution

Keep this issue open as a real WhatsApp plugin regression. Preserve the #70678 transport-activity watchdog fix, then add focused fixes and tests for retryable 408/428 reconnect churn, health-monitor coordination while WhatsApp is already reconnecting, outbound delivery behavior during short listener gaps, and recovery from `groupFetchAllParticipating` 1006 plus Baileys/libsignal Bad MAC failures so group events and sends do not silently disappear.
