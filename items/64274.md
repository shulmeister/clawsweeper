## Summary

Current main does not yet implement the reported fix. Runtime auth snapshots are still keyed by the resolved auth-store path, while the default-agent path resolution still honors OPENCLAW_AGENT_DIR. That means a process whose OPENCLAW_AGENT_DIR points at the active agent can collapse the default lookup and the explicit agent lookup to the same auth-profiles.json path. The related PR #64320 proposed the right fix shape and regression coverage, but it is closed and unmerged, and equivalent coverage is not present on main.

## Best Possible Solution

Keep this issue open. The best fix is to revive the #64320 approach or equivalent: key runtime auth snapshots by a stable original identity such as raw agentDir plus a separate default-agent sentinel, update get/has/set/replace callers consistently, and add a regression test where OPENCLAW_AGENT_DIR points at the requested agent while main and that agent have distinct MiniMax portal profiles.
