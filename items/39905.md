## Summary

Keep PR #39905 open. Current main still lacks the PR's central Telegram per-conversation reply and preview-streaming override surface: scoped direct/group/topic config types and strict schemas do not expose `replyToMode` or scoped `streaming`, normal message dispatch still uses account-level startup values, and native command delivery still uses the captured account-level `replyToMode`. The related quoted-DM streaming subfix is present on main, but that only covers one overlapping bug and does not supersede the broader conversation-scoped configuration feature.

## Best Possible Solution

Keep this PR, or a maintainer-owned follow-up, open until the feature is ported or explicitly rejected. The best implementation is to add typed and schema-validated scoped Telegram overrides under `direct`, `groups`, and `topics`; resolve effective `replyToMode` and preview streaming with topic > direct/group > account precedence before normal dispatch; apply the same reply-threading resolution to native command replies; use the current nested `ChannelPreviewStreamingConfig` shape; and update Telegram/config docs plus focused tests in the current `extensions/telegram` layout.
