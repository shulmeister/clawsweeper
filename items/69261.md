## Summary

Keep PR #69261 open. Current main still resolves Ollama embedding auth by precedence only, then attaches the selected bearer to whatever embedding base URL was resolved. Existing tests also assert the unfixed env-to-local and placeholder-token behaviors, so the PR is not implemented or superseded on main.

## Best Possible Solution

Keep PR #69261 open. The best path is to rebase it onto current main, fix or document the provider opt-out/env fallback behavior with a regression test, and land the host-scoped auth selection plus placeholder filtering in `extensions/ollama/src/embedding-provider.ts`. The older Ollama `/api/embeddings` to `/api/embed` migration is separate work and should remain tracked separately.
