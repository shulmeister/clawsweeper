## Summary

Keep open. Current main still lacks the /v1/chat/completions tool protocol changes proposed by this PR: the adapter does not parse tools/tool_choice, does not pass clientTools into the agent command, and always returns normal assistant content with finish_reason "stop". The adjacent /v1/responses path and agent runtime already have clientTools plumbing, so the PR targets a real missing gateway adapter surface rather than obsolete or out-of-scope work.

## Best Possible Solution

Keep the PR open. The best path is to finish/review the Chat Completions adapter work against the existing clientTools runtime seam, resolve the stream finalization review concern, and land with focused tests for tools/tool_choice parsing, non-stream tool_calls, streaming delta.tool_calls, and role:"tool" follow-up handling.
