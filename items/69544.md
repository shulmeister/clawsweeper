## Summary

Keep #69544 open. Current main still derives mention patterns from `agents.list[].identity.emoji`, Matrix room mention gating uses those shared regexes to decide whether to reply, and I found no Matrix, global, or per-agent config flag that disables emoji-derived mentions without replacing `mentionPatterns` entirely. The related context does not supersede this Matrix-specific request.

## Best Possible Solution

Add a backward-compatible emoji-mention policy that defaults to current behavior, with a Matrix channel-level default and a per-agent override if maintainers want the requested scope. The implementation should make `deriveMentionPatterns` skip `identity.emoji` when the policy disables emoji mentions, update config schema/types/docs, and add regression tests for Matrix require-mention rooms where a message containing only the agent emoji no longer triggers when disabled.
