## Summary

Keep open. The PR is not implemented on current `main`, and the proposed fix in the PR still appears incomplete: it adds a config fallback for PDF model lookup, but the review discussion correctly identifies an alias-normalization gap for inline provider configs such as `aws-bedrock` vs `amazon-bedrock`. Current `main` already handles that normalized inline-model matching in the embedded runner, which is evidence the PR's narrower helper would duplicate resolver logic but miss an existing contract.
