## Summary

Keep open. Current main still does not implement `sessionSaveRedirectPath` or the supporting `postHookActions` contract, and the PR remains tied to open security-surface work rather than a cleanup-close candidate.

## Best Possible Solution

Keep #38162 open. If maintainers accept the direction, resolve or land the foundational #38161 hook contract first, then review the redirect implementation with explicit workspace containment, symlink/retraction semantics, tests for absolute and relative escapes, docs for the new hook context fields, and a clear decision on whether `blockSessionSave` means strict never-persist or best-effort rollback.
