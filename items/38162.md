## Summary

Keep PR #38162 open. Current main still does not implement `sessionSaveRedirectPath`, the supporting `postHookActions` coordination mechanism, or the related session-memory policy fields. The PR is not obsolete cleanup; it remains a concrete, security-sensitive hook/session-memory capability that needs maintainer review rather than automatic closure.

## Best Possible Solution

Keep #38162 open. Resolve or supersede #38161 first, then review #38162 as the redirect-specific follow-up with explicit workspace-containment semantics, symlink/retraction tests, late block/unblock behavior, docs for the new hook context fields, and a maintainer-approved security posture for session-memory persistence controls.
