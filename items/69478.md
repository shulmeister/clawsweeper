## Summary

Keep open. Current main still lacks the requested run-intent deduplication: system events only suppress consecutive duplicate text, exec approval requests mint fresh IDs/context keys, and the gateway/native approval paths dedupe only exact approval IDs. The linked attempted fixes #69731 and #69893 were closed unmerged, and current main has no #69478 implementation or changelog evidence.

## Best Possible Solution

Keep #69478 open and fix the approval-request layer, not just text-only system-event suppression. Add a stable dedup/idempotency key derived from the exec run intent, for example agent/session plus normalized command/systemRunPlan/cwd/env binding and failover context, then have gateway registration coalesce or cancel already-pending equivalent approval requests. Cover it with regression tests proving fresh approval IDs for the same pending exec prompt do not create multiple native Telegram/channel prompts while distinct commands still prompt independently.
