## Summary

Keep open. Current `main` still appears to misclassify `gateway` action `config.schema.lookup` as mutating, which means its failures can still produce user-facing warning payloads instead of staying silent.

## Evidence

- **`config.schema.lookup` is documented and implemented as a read-only inspection action:** Docs describe `config.schema.lookup` as fetching one path-scoped schema node for drill-down/inspection, and the gateway tool exposes it as a dedicated action.
  - file: [docs/cli/config.md:65](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/docs/cli/config.md#L65)
  - command: `nl -ba docs/cli/config.md | sed -n '56,76p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Gateway tool forwards the compound action name unchanged:** `createGatewayTool()` accepts `action: "config.schema.lookup"` and calls `callGatewayTool("config.schema.lookup", ...)` directly, so mutation classification sees the full dotted action name.
  - file: [src/agents/tools/gateway-tool.ts:461](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/agents/tools/gateway-tool.ts#L461)
  - command: `nl -ba src/agents/tools/gateway-tool.ts | sed -n '280,520p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Mutation classifier still treats dotted gateway actions as mutating unless they exactly match a short allowlist verb:** `READ_ONLY_ACTIONS` contains bare verbs like `get`, `list`, `inspect`, `check`, `probe`, but not `lookup` or `config.schema.lookup`. For `gateway`, the code returns mutating when `action` is not in that exact set.
  - file: [src/agents/tool-mutation.ts:24](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/agents/tool-mutation.ts#L24)

- **Mutating tool errors are intentionally surfaced to users:** `resolveToolErrorWarningPolicy()` always sets `showWarning: true` for mutating tool errors, and tests confirm mutating tool errors are still shown even when assistant output exists.
  - file: [src/agents/pi-embedded-runner/run/payloads.ts:95](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/agents/pi-embedded-runner/run/payloads.ts#L95)
