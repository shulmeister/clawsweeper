## Summary

Keep open. Current `main` still has the silent text-only webchat/control-ui attachment failure in the normal `chat.send` path. The gateway decides attachment handling from the active session model, force-offloads images for text-only sessions, dispatches no inline `images`, and does not populate `MediaPath`/`MediaPaths` on the live turn context. Because media understanding only runs when those inbound media fields exist, the newer `agents.defaults.imageModel` auto-resolution work does not rescue this webchat path, and there is still no user-facing warning here.

## Evidence

- **Gateway still gates webchat attachments on the active session model:** `chat.send` resolves the session model, computes `supportsImages`, and passes that flag into `parseMessageWithAttachments`. For ordinary webchat sessions there are no plugin-bound media fields added to the live `ctx`, so the dispatched turn only carries parsed inline images when the session model itself supports images.
