## Summary

Keep this PR open. Current main still routes the CLI default picker, CLI allowlist picker, and gateway model catalog through `loadModelCatalog` without a `models.mode === "replace"` branch, while the same-author linked bug #64950 is still open. The PR remains a plausible fix candidate, but the review-thread question about implicit/plugin-discovered models and focused regression coverage still needs maintainer resolution.

## Best Possible Solution

Keep this PR and #64950 open, or close both only after a maintainer-chosen successor lands. The best fix is a shared mode-aware catalog path for the CLI picker and gateway `models.list` that honors `models.mode: "replace"`, explicitly defines implicit/plugin-discovery semantics, preserves configured metadata and aliases, and adds focused regression tests for the default picker, allowlist picker, and gateway catalog output.
