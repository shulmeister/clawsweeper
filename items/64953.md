## Summary

Keep open. Current `main` does not implement this PR, and the proposed `buildConfiguredModelCatalog` shortcut would bypass the existing replace-mode catalog pipeline that already rewrites `models.json`, merges implicit providers, and applies provider-plugin catalog hooks.

## Evidence

The current source supports those concerns, so this is not a conservative cleanup close.

## Risks / Open Questions

- The PR's direct `buildConfiguredModelCatalog` branch would omit implicit/discovered providers that current replace-mode `models.json` generation still includes.
- The PR would also omit provider-plugin supplemental catalog entries and can change empty-catalog setup/onboarding behavior in the picker.
