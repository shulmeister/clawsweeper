## Summary

Keep open. Current main still has the strict plugin-load and plugin-local invalid-config behavior this PR targets, so it is not implemented. The PR should not land as-is because it bundles unrelated fixes and the review discussion flags real regressions in the media-delivery and fail-open runtime registry changes.

## Best Possible Solution

Keep the item open for maintainer review, but request a focused replacement. The fix should address plugin-local config invalidity at the config-validation and gateway-startup boundary, mark only the bad plugin error/disabled with Doctor diagnostics, keep other plugins/channels running, and avoid caching or activating failed runtime registries unless a deliberate degraded-runtime contract is added. Unrelated qqbot, media, and Vertex changes should be split out or dropped.
