## Summary

Current `main` already implements RAM-backed caching for workspace bootstrap/personality files and reuses those results across turns, so the core request in #14386 is now satisfied.

## Risks / Open Questions

- This does not add a filesystem watcher; live edits are handled by cache invalidation on reload and by clearing per-session snapshots on reset/rollover, so hot-reload semantics are narrower than the issue's proposed design.
