## Summary

Current main still does not implement the requested agent-level hard `maxTurns` / `maxToolCalls` / `toolCallTimeout` budget or forced-final-response behavior. The repo has useful mitigations through `tools.loopDetection`, unknown-tool guarding, LLM idle timeout, and overall run timeout, but those are pattern/timeout safeguards rather than a total model-tool iteration budget. The existing issue comment and related #13271 both point to keeping this as a distinct global/per-agent request.

## Best Possible Solution

Keep #9912 open until OpenClaw adds a typed global/per-agent hard budget around the model-tool loop, or a maintainer explicitly narrows the product direction to the existing loop-detection and timeout controls. A complete implementation should update config schema/types/docs, enforce total turns/tool calls before additional tool execution or model continuation, define the last-tool-result/error-summary final response behavior, and keep #13271 as the separate per-channel overflow/delegation track.
