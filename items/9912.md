## Summary

Keep open. Current main has configurable tool-loop detection that mitigates repeated/no-progress loops, but it does not implement the requested hard agent-level maxTurns/maxToolCalls/toolCallTimeout budget or the requested force-final-response behavior. Related #13271 is still open and is per-channel, not a complete superseding tracker for this global request.

## Evidence

- **Requested options absent:** A targeted search finds no maxToolCalls or toolCallTimeout config surface in source/docs/packages/extensions. maxTurns occurrences are xAI tool-specific, web/code-execution examples, or sessions-send ping-pong helpers, not a general agent iteration budget.
  - command: `rg -n "maxToolCalls|toolCallTimeout" src docs packages extensions --glob '!**/dist/**' --glob '!**/*.map'; rg -n "\bmaxTurns\b" src docs packages extensions --glob '!**/dist/**' --glob '!**/*.map'`

- **Existing config is loop detection:** The exposed config is ToolLoopDetectionConfig with enabled, historySize, warningThreshold, unknownToolThreshold, criticalThreshold, globalCircuitBreakerThreshold, and detector toggles. It is available globally and per-agent via tools.loopDetection, not as a hard maxTurns/maxToolCalls budget.

When enabled, it blocks repeated unknown tools, identical no-progress outcomes, known poll no-progress loops, and no-progress ping-pong loops; generic identical calls are warning-first.
