## Summary

Keep #9912 open. Current main still lacks the requested global/per-agent hard `maxTurns`, `maxToolCalls`, or `toolCallTimeout` budget that forces a final response after model-tool iterations. Existing safeguards cover overall run timeout, LLM idle timeout, unknown-tool protection, and optional pattern-based loop detection, but they are not an equivalent total turn/tool-call budget and do not implement the requested last-tool-result finalization behavior.

## Best Possible Solution

Keep #9912 open until OpenClaw adds a typed global/per-agent hard budget around the model-tool loop, or a maintainer explicitly closes the scope as covered by existing loop detection and timeout controls. A complete implementation should update config schema/types/docs, enforce total turns/tool calls before further tool execution or model continuation, define timeout behavior after the first tool call, and specify final-response behavior using the last tool result or an explicit iteration-limit message. Keep #13271 as the separate per-channel overflow/delegation track.
