## Summary

Keep #9912 open. Current main still lacks a global or per-agent hard budget for model-tool turns, total tool calls, or timeout-after-first-tool-call behavior. Existing `tools.loopDetection`, run timeouts, and accepted-but-ignored OpenResponses `max_tool_calls` are related but do not implement the requested config contract or forced final-response behavior.

## Best Possible Solution

Keep #9912 open as the canonical global/per-agent hard iteration budget request. A complete fix should add typed config, schema/help, generated config docs, and public docs for max model-tool turns and/or total tool calls at `agents.defaults` and per-agent scope; define timeout-after-first-tool-call semantics; enforce the limits in the embedded/tool-call runtime; add focused tests for limit exhaustion and final-response behavior; and keep #13271 separate for per-channel budgets and auto-delegation.
