## Summary

Keep #9912 open. Current main still does not provide a global or per-agent hard iteration budget such as `maxTurns`, `maxToolCalls`, or `toolCallTimeout`. The current config schemas/types expose adjacent settings like whole-run `timeoutSeconds`, `maxConcurrent`, and optional `tools.loopDetection`, but not the requested hard cap and forced-response behavior. The existing maintainer review comment also explicitly kept this issue open and distinguished it from related per-channel or narrower duplicate requests.

## Best Possible Solution

Keep #9912 open as the canonical global/per-agent hard iteration budget request. A complete fix should add typed `agents.defaults` and per-agent config, update schema/help/docs, enforce max model-tool turns and total tool invocations in the embedded/tool-call runtime before the next tool executes, define timeout-after-first-tool-call behavior, and add focused tests for limit exhaustion and final-response behavior. Keep #13271 separate for per-channel budgets and auto-delegation.
