## Summary

Keep open. Current main still ignores top-level `ref` when `browser action=act` also includes a `request` object, so the reported compatibility bug remains. The misleading reachability wrapping has been improved for service validation errors, but that only fixes part of the user-visible symptom. Related PR #38790 targeted the right behavior but was closed unmerged.

## Best Possible Solution

Keep the issue open and implement a narrow normalizer for `action: "act"`: when `request` is an object, clone it and fill only missing `request.ref`, `request.targetId`, and finite `request.timeoutMs` from top-level params while preserving nested request precedence. Add focused regression tests for `ref="e146", request={ kind: "click" }`, nested-value precedence, and unchanged fully flattened behavior.
