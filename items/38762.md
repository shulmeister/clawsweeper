## Summary

Keep open. Current main still accepts fully flattened legacy act params only when `request` is absent; when `browser action="act"` receives `ref` at the top level and `request: { kind: "click" }`, `readActRequestParam` returns the nested request unchanged and drops the top-level `ref`. The misleading reachability wrapping has been improved, but the compatibility bug reported in #38762 remains. Related PR #38790 targeted the right behavior but is closed and its head SHA is not on current `main`.

## Best Possible Solution

Keep #38762 open and implement a narrow compatibility normalizer in `extensions/browser/src/browser-tool.ts`: when `action: "act"` receives an object `request`, clone it and fill only missing `request.ref`, `request.targetId`, and finite `request.timeoutMs` from top-level params. Add focused tests in `extensions/browser/src/browser-tool.test.ts` for the reported mixed shape, nested-value precedence, and unchanged fully flattened legacy behavior.
