## Summary

Keep #38762 open. Current main still does not support the reported mixed browser act shape: when `request` is an object, top-level `ref`/`targetId`/`timeoutMs` are not merged into it before `/act`. The browser-service error wrapping has improved, but the central compatibility issue remains. The related PR #38790 targeted this fix but was closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep the issue open and land a narrow compatibility normalizer in `extensions/browser/src/browser-tool.ts`: when `action: "act"` receives an object `request`, clone it and fill only missing `request.ref`, `request.targetId`, and finite `request.timeoutMs` from top-level params. Preserve nested values, keep fully flattened legacy behavior unchanged, and add focused tests in `extensions/browser/src/browser-tool.test.ts` for the mixed shape, nested precedence, and targetId/timeoutMs fill behavior.
