## Summary

Keep open. Current main still does not accept the reported mixed browser act shape `action=act ref=... request={kind: ...}`. The tool parser returns an existing `request` object unchanged before considering top-level legacy fields, so top-level `ref` is ignored when `request.ref` is absent. The related PR #38790 described the right compatibility fix but was closed unmerged, and current tests still cover only fully flattened params plus nested-request precedence, not the mixed fallback shape.

## Best Possible Solution

Keep the issue open and implement a narrow browser tool compatibility normalizer for `action: "act"`: when `request` exists, fill missing `request.ref`, `request.targetId`, and valid `request.timeoutMs` from top-level fields while preserving nested request precedence. Add a focused regression test for `ref="e146", request={kind:"click"}` and keep the improved non-reachability error behavior intact.
