## Summary

Keep #38762 open. Current main still ignores a top-level `ref` when `browser action="act"` also receives an object `request`, so the reported mixed parameter shape remains unsupported. The linked fix PR #38790 targeted the right behavior but was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open and land a narrow compatibility normalizer in `extensions/browser/src/browser-tool.ts`: when `action: "act"` receives an object `request`, clone it and fill only missing `request.ref`, `request.targetId`, and finite `request.timeoutMs` from top-level params. Preserve nested request precedence and add focused tests in `extensions/browser/src/browser-tool.test.ts` for the mixed shape, nested-value precedence, and unchanged fully flattened legacy behavior.
