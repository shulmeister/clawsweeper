## Summary

Keep open. Current main still ignores a top-level `ref` when `action="act"` also supplies a `request` object, so the exact mixed shape from #38762 remains unsupported. Main has partially improved the misleading reachability wrapping for browser service/application errors, but the compatibility behavior requested by the issue and by the closed-unmerged PR #38790 has not landed.

## Best Possible Solution

Keep #38762 open until the browser tool either normalizes top-level act fields into an existing `request` object when missing, with nested request values taking precedence, or a maintainer explicitly decides that `ref` must only be accepted inside `request` and updates the error message/tests accordingly.
