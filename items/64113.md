## Summary

PR #64113 adds a new optional third-party Yep web-search provider as a bundled plugin. Current main does not ship Yep, but OpenClaw already has the plugin, manifest, SDK, web-search provider, and ClawHub/npm distribution surfaces needed for this to live outside core. The PR author is not a maintainer and the labels are not protected, so this is a ClawHub/community-plugin scope close rather than an implementation-on-main close.

## Best Possible Solution

Do not merge Yep as another bundled OpenClaw core plugin by default. Move the PR's provider implementation into an externally maintained OpenClaw plugin, declare its manifest/config/SecretRef contracts, and publish it through ClawHub or npm so users can install it with `openclaw plugins install`.
