## Summary

Keep open. Current main still lacks a Google Chat user OAuth credential/config surface, and the requested reaction, native attachment upload, and email-alias DM paths still run through the same service-account `chat.bot` access-token path. The external Google Drive file-upload PR is useful workaround context, but it does not supersede the missing bundled Google Chat user OAuth support.

## Best Possible Solution

Keep the issue open until the bundled Google Chat plugin has an explicit user OAuth credential model, schema/config/docs, safe token resolution and secret handling, token selection for reactions/uploads/proactive DMs, service-account behavior preserved for webhook/bot operations, and tests for graceful fallback when user OAuth is not configured. If maintainers want this moved out of the bundled plugin, make that product decision explicit and identify any plugin API gap first.
