## Summary

Keep #9764 open. Current main still has Google Chat wired around service-account `chat.bot` auth, with no `userOAuth` config/schema/secret surface and no per-operation user-token selection for reactions, uploads, reaction typing, or proactive DM lookup. The issue discussion already includes a same-day maintainer-side Codex review keeping this open; the cross-referenced Google Chat work is in an external repo, not shipped in openclaw/openclaw.

## Best Possible Solution

Keep the issue open. The bundled Google Chat plugin should add a strict user OAuth credential model with SecretRef/redaction support and docs, resolve refresh-token credentials safely, construct a scoped user OAuth client, use that token for reactions, attachment uploads, `typingIndicator="reaction"`, and proactive DM/email-alias lookup, preserve the existing service-account path for webhook verification and bot-level sends, and warn or fall back cleanly when user OAuth is absent.
