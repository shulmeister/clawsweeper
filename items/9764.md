## Summary

Keep #9764 open. Current main still lacks Google Chat user OAuth support: the bundled Google Chat plugin exposes only service-account credentials, schema validation is strict and has no `userOAuth`/refresh-token config, credential resolution only handles service-account inputs, auth uses only the `chat.bot` service-account scope, and reactions, uploads, and DM lookup all share that same bot-token path. The latest issue discussion also contains a maintainer-side Codex review explicitly keeping the issue open.

## Best Possible Solution

Keep the issue open. The bundled Google Chat plugin should add a strict, secret-aware user OAuth credential model, resolve refresh-token credentials safely, construct scoped user OAuth clients, route reactions, media uploads, reaction typing, and proactive DM/email-alias lookup through user auth, preserve service-account auth for webhook verification and bot-level sends, and document fallback behavior when user OAuth is absent.
