## Summary

Issue #11940 is best closed as a ClawHub/community plugin request, not kept open for OpenClaw core. Current project scope explicitly keeps optional capability out of core, prefers externally hosted plugins, and uses ClawHub as the canonical discovery surface for community plugins. The requested feature is already framed as an external UI plugin in its own repository, and current main already exposes model/fallback/auth surfaces that such a plugin can build on.

## Close Comment

Closing this as a Codex review under the `clawhub` bucket. The request is for optional model-management UI, and current project scope says that kind of capability should usually live as a plugin rather than in OpenClaw core: see `VISION.md:54`, `VISION.md:55`, `VISION.md:58`, and `VISION.md:59`. The community-plugin docs also make ClawHub the canonical discovery surface for third-party plugins (`docs/plugins/community.md:9` and `docs/plugins/community.md:14`). Current main already exposes the underlying model-management surfaces a plugin can build on, including session model overrides and model snapshots in the Control UI (`docs/web/control-ui.md:111` and `docs/web/control-ui.md:129`) plus documented fallback/priority configuration in `docs/concepts/models.md:18`, `docs/concepts/models.md:19`, and `docs/concepts/models.md:174`. Given that this feature is already presented as an external plugin in its own repository, the conservative maintainer path is to keep it in the plugin ecosystem rather than track it as an OpenClaw core issue.
