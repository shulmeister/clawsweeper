## Summary

Keep #68279 open. Current main still does not export `stringEnum` or `optionalStringEnum` from the bare `openclaw/plugin-sdk` entrypoint, while the helpers remain available from `openclaw/plugin-sdk/core` and other narrower subpaths. The linked PR #66214 remains the relevant unmerged fix path, and a maintainer comment explicitly left this issue open as the current concrete tracker.

## Best Possible Solution

Keep #68279 open as the active tracker. The best code path is to review or replace PR #66214 with a small backward-compatible export change in `src/plugin-sdk/index.ts`, plus the generated Plugin SDK API baseline update and a root-entry contract test covering `stringEnum` and `optionalStringEnum`. If maintainers decide the deprecated root entrypoint intentionally should not regain these exports, that should be an explicit product/API decision documented in the issue rather than an automated cleanup close.
