## Summary

Keep #40423 open. Current main still only injects `OPENCLAW_SHELL` for the exec, ACP client, and TUI local-shell child-process paths; it does not expose `OPENCLAW_AGENT_ID` or `OPENCLAW_SESSION_KEY`. The related `OPENCLAW_MCP_*` runtime context is a separate CLI-harness/MCP surface and does not supersede this PR's spawned-shell contract. The PR remains a plausible core runtime change, but it needs rebase/review against current main and the earlier gateway approval/safeBin propagation concern should be rechecked.

## Best Possible Solution

Keep the PR open or ask for a rebase. The best implementation path is a core runtime change that adds a shared resolver for deterministic `OPENCLAW_AGENT_ID` and optional `OPENCLAW_SESSION_KEY`, threads explicit `agentId` through exec direct, PTY, sandbox, gateway approval/safeBin, ACP client, and TUI local-shell paths, and updates the runtime env docs plus focused tests for explicit-agent-id precedence and session-key fallback.
