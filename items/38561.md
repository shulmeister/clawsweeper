## Summary

Keep open. Current `main` does not implement the PR’s session-store sync after `openclaw models set`, and the read-only status/session surfaces still prefer persisted runtime model fields over config defaults when no explicit override exists. The PR is not obsolete, although its claimed impact is somewhat broader than the current code path evidence: execution of new turns already re-resolves the configured default model unless a session override is present.
