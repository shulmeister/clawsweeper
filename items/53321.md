## Summary

Current `main` does not implement proactive auto session splitting near the context window. OpenClaw has auto-compaction and a pre-compaction memory flush, but session rollover is still documented/configured as daily, idle, or manual `/new`/`/reset`, and overflow handling still falls back to telling the user to start fresh or doing a visible reset only after failure.

## Evidence
