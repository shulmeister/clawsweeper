## Summary

Keep open. Current main has important mitigations for long sessions, including preflight compaction, silent memory flush, and opt-in transcript rotation after compaction, plus a new byte-size trigger for oversized transcripts. It still does not implement the requested proactive auto session split policy with a configurable token/context threshold, default invisible rollover behavior, active tool-call boundary semantics, or explicit continuity payloads carrying active-task and pending-confirmation state. The provided related context shows #53330 was closed in favor of this issue, so this remains the canonical tracker.

## Best Possible Solution

Keep this issue open as the canonical product/design tracker. Build on the existing memory-flush, preflight compaction, queued-followup refresh, and truncateAfterCompaction machinery, then add a supported auto-split policy with threshold config, documented defaults/notifications, active tool-call boundaries, session-key routing tests, and continuity payload handling for summary plus task or pending-confirmation state.
