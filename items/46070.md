## Summary

Keep #46070 open. Current main has not restored the requested behavior for a non-Control-UI Paperclip-style WebSocket client that omits device identity: the gateway still clears self-declared scopes for device-less shared token/password auth outside the narrow direct-local gateway-client exception, tests assert that policy, and the `agent` method still requires `operator.write`. The existing issue comment saying this may be Paperclip-side is useful context, but it is not enough to close a plausible core WebSocket auth regression.
