## Summary

Keep this PR open. Current main still has the documentation gap: the runtime clears self-declared WebSocket scopes when trusted-proxy/device-less sessions are not bound to device identity, while the trusted-proxy and protocol docs still describe the connection exceptions without explaining the resulting missing-scope behavior or clearly limiting `x-openclaw-scopes` guidance to HTTP endpoints.

## Best Possible Solution

Keep this PR open and review or land it, or land an equivalent docs update in `docs/gateway/trusted-proxy-auth.md` and `docs/gateway/protocol.md` that separates HTTP `x-openclaw-scopes` behavior from WebSocket handshake scopes, documents scope clearing without device identity, preserves the local `allowInsecureAuth` token/password exception, and frames `dangerouslyDisableDeviceAuth` as break-glass only.
