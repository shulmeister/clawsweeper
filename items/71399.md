## Summary

Keep PR #71399 open. Current main still does not implement the requested ACP `agent:turn:end` internal hook, and this PR is the current focused implementation attempt rather than obsolete cleanup. The branch still needs maintainer/code review follow-up because the proposed payload omits the advertised failure `errorCode` and uses unbounded fire-and-forget dispatch even though a bounded helper exists.

## Best Possible Solution

Keep the PR open for maintainer review or a successor patch. The best path is to emit `agent:turn:end` from `AcpSessionManager.recordTurnCompletion()` on success and failure, include `sessionKey`, `success`, `durationMs`, and optional failure `errorCode`, use bounded non-blocking dispatch or document why this event intentionally differs, and cover the type guard plus ACP manager success/failure emission in tests. If the event becomes operator-facing, update the internal hooks docs at `docs/automation/hooks.md`.
