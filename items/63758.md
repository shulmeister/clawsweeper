## Summary

Keep open. Current main still creates and finalizes cron task records without setting `progressSummary`, while `/tasks` and `session_status` only show live detail for running tasks when that field is populated. The task registry already supports progress summaries and other producers use them, so this remains a cron producer gap rather than an implemented fix or storage limitation.

## Best Possible Solution

Fix this in the cron producer paths, not in the generic task registry. Add coarse `progressSummary` values when cron task records are created and update them during meaningful phases such as queued/running execution, handoff/delivery, and cleanup. Cover at least scheduled managed cron and manual cron with regression tests that assert an active cron task exposes nonempty live detail through the task status formatter or `/tasks`/`session_status` path.
