## Summary

Keep this issue open. Current main still routes `/v1/chat/completions` through the normal ingress path with full agent prompt assembly, has no HTTP light-context/voice-mode contract, and still only supports `x-openclaw-session-key` plus the OpenAI `user` field for stable session routing. Open PR #69060 remains only a partial implementation candidate for the stable-session/QMD warm-cache subproblem.

## Best Possible Solution

Keep this issue open. Review and either land or close #69060 separately for the stable session/QMD warm-cache portion. For the remaining work, add an HTTP-specific light/voice mode in `src/gateway/openai-http.ts` and `src/gateway/http-utils.ts`, document the request contract at https://docs.openclaw.ai/gateway/openai-http-api, and thread an explicit prompt policy through `buildAgentCommandInput` and `agentCommandFromIngress` so `/v1/chat/completions` can intentionally reduce bootstrap context, tools, skills, and memory sections with regression coverage in `src/gateway/openai-http.test.ts`.
