## Summary

Keep #68920 open. Current main still routes `/v1/chat/completions` through the normal ingress agent run and does not implement `x-openclaw-session`, `x-openclaw-light-context`, or an HTTP-specific slim prompt mode. The existing code/docs only support `x-openclaw-session-key` and the OpenAI `user` field for session stability, while the related PR #69060 is explicitly only a partial session-warm fix and remains unmerged in the provided context.

## Best Possible Solution

Keep #68920 open. Land or supersede the narrow stable-session work from #69060, then design the larger HTTP low-latency mode with an explicit header/config contract, tests for session-key precedence and context reduction, and docs for voice/realtime callers. The implementation likely belongs in `src/gateway/openai-http.ts`/`src/gateway/http-utils.ts` plus the agent command prompt-mode/bootstrap/tool-selection contract, not as a cleanup-only close.
