## Summary

Current main does not implement the requested light-context or voice-mode HTTP path. The `/v1/chat/completions` handler still builds a normal ingress agent command and calls `agentCommandFromIngress`, session routing still supports only `x-openclaw-session-key`, OpenAI `user`, or a per-call random UUID, and the HTTP chat-completions config has no light-context/system-prompt override field. The linked PR #69060 remains relevant as a partial session-warm fix, but it does not resolve the main context-assembly request.
