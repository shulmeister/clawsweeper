## Summary

Current `main` still drops Feishu card callback fields outside `action.value`. The reported gap remains in the parser, handler types, and fallback extraction path, so this item should stay open.

## Evidence

- **Action event type still excludes the reported fields:** `FeishuCardActionEvent.action` only declares `value` and `tag`; there is no `option`, `options`, or `form_value` field on current `main`.
  - file: [extensions/feishu/src/card-action.ts:21](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/extensions/feishu/src/card-action.ts#L21)
  - command: `nl -ba extensions/feishu/src/card-action.ts | sed -n '14,30p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Fallback extraction still reads only `action.value`:** `buildFeishuCardActionTextFallback` checks `action.value.text`, then `action.value.command`, then JSON-stringifies `action.value`. Sibling callback fields on `action` are never read or merged.
  - file: [extensions/feishu/src/card-interaction.ts:81](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/extensions/feishu/src/card-interaction.ts#L81)

  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Webhook payload parser discards extra action fields before the handler sees them:** `parseFeishuCardActionEventPayload` reads `action.tag` and `action.value` only, then constructs a `FeishuCardActionEvent` without preserving any other `action.*` properties.
  - file: [extensions/feishu/src/monitor.account.ts:243](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/extensions/feishu/src/monitor.account.ts#L243)

- **Current tests cover text/JSON payloads, not select or form callback fields:** The card-action tests exercise `action.value.text`, generic JSON payloads, and structured quick-action envelopes. There are no tests for `action.option`, `action.options`, or `action.form_value` on current `main`.
