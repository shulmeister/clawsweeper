## Summary

Avian is a reasonable optional OpenAI-compatible provider, but current main already has the provider-plugin, auth, onboarding, catalog, install, and ClawHub/npm distribution surfaces needed to ship it outside OpenClaw core. There is no protected label or maintainer-author block in the provided PR context, and the PR discussion shows the contributor had already adapted the work into a provider-plugin shape.

## Best Possible Solution

Close the OpenClaw core PR as a ClawHub/community provider-plugin fit. The Avian provider should be published and maintained as an external plugin using the existing provider SDK and ClawHub/npm install path; bundled status should only be revisited through explicit maintainer promotion.
