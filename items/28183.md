## Summary

Avian is a reasonable optional LLM-provider integration, but current OpenClaw has the public provider-plugin, onboarding, catalog, install, and ClawHub publishing surfaces needed to ship it outside core. The PR discussion also shows the contributor already migrated the work toward that plugin shape, so the best maintainer cleanup path is to close the core PR and point the work to ClawHub/npm packaging rather than bundling another service-specific provider by default.

## Best Possible Solution

Close the OpenClaw core PR as a ClawHub/plugin-distribution fit. The Avian work should live as an externally maintained provider plugin published on ClawHub or npm using the existing provider SDK; if maintainers later want official bundled status, that should happen through an explicit maintainer-owned promotion decision.
