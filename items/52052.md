## Summary

Keep open. Current `main` does not implement this PR's `sticker_file_id` prompt projection, so it is not closable as already shipped, and the remaining work is still useful. There is also a live contract mismatch on `main`: Telegram sticker sending still accepts `fileId`/`stickerId`, not `sticker_file_id`, so the PR as described would still need follow-up before it fully delivers sticker replay.
