## Summary

Keep #60388 open. Current main still lacks the terminal abort/reason plumbing needed to stop model fallback after the run's own timeout budget, and the linked fix PR #62682 remains open rather than merged into main.

## Evidence

- **Fallback abort handling still treats timeout aborts as retryable:** `shouldRethrowAbort` only rethrows AbortError when `!isTimeoutError(err)`. A run-budget timeout from the embedded runner is wrapped as AbortError with TimeoutError cause, so this path returns the failure to the fallback loop instead of short-circuiting it.
  - file: [src/agents/model-fallback.ts:91](https://github.com/openclaw/openclaw/blob/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54/src/agents/model-fallback.ts#L91)
  - command: `nl -ba src/agents/model-fallback.ts | sed -n '70,230p'`
  - sha: [02f3e9cfa25f](https://github.com/openclaw/openclaw/commit/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54)
- **Fallback runner has no AbortSignal/reason contract on main:** `runWithModelFallback` accepts cfg/provider/model/run/onError/classifyResult, but no `abortSignal` or terminal-abort reason parameter. That means main cannot implement the PR-described `signal.reason` guard in the fallback layer.
  - file: [src/agents/model-fallback.ts:715](https://github.com/openclaw/openclaw/blob/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54/src/agents/model-fallback.ts#L715)
