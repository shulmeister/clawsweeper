## Summary

Keep open. Current main has partial mitigations for the reported symptoms, including daily-memory startup context on bare /new or /reset, Telegram streaming config migration, and installer-only duplicate install warnings. It does not fully address the 2026.4.10 memory/session upgrade regression or the stale pinned CLI path scenario, and the same-author related memory/session issue #65358 remains open. The sole issue comment is promotional and does not provide a maintainer resolution or real workaround.

## Best Possible Solution

Keep this open or split it deliberately. Use #65358 as the canonical tracker for the 2026.4.10 memory/session continuity regression, then track the stale CLI-path work as an update/doctor/install improvement: enumerate known and PATH-resolved `openclaw` binaries, compare versions, warn on pinned LaunchAgent/cron paths that point at older binaries, and safely repair or guide removal of stale `~/.local/bin/openclaw` wrappers when switching install methods.
