## Summary

Keep #8209 open. Current main has partial cron auditability through internal started/finished events, task ledger records, completion-only run logs, completion webhook delivery, and failure destinations, but it still lacks the requested native cron lifecycle webhook/telemetry surface or typed plugin cron start/complete hook for external Mission Control-style integrations.

## Best Possible Solution

Keep #8209 open for explicit maintainer product direction. The conservative implementation path is a generic typed cron lifecycle event surface, preferably a plugin hook or subscription API that emits start and finish events for both agentTurn and systemEvent jobs, plus docs/examples for posting to external systems. If core webhooks are chosen instead, keep them generic, schema-validated, SSRF-guarded, and covered by focused tests for start, success, failure, timeout, disabled, and misconfigured destinations.
