## Summary

Current `main` does not implement `tools.allow`/`tools.deny` on `bindings[]`. Both the route and ACP binding schemas remain strict and only accept the existing binding fields, so the requested binding-level feature is still absent. OpenClaw does have separate channel/group tool policy controls for some surfaces such as Telegram, which may cover the reporter's immediate use case, but that is not the generic per-binding capability requested here.

## Evidence

- **Binding schemas still omit `tools`:** `RouteBindingSchema` only accepts `type`, `agentId`, `comment`, and `match`; `AcpBindingSchema` adds only `acp`. Both end with `.strict()`, so extra top-level fields like `tools` are rejected.
