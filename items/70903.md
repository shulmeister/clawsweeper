## Summary

Keep open. Current main still has the core failure mode: billing failures are persisted as long `disabledUntil` windows, same-provider fallback chains count as fallback availability and therefore do not get the single-provider billing probe path until near expiry, and `openclaw configure --section model` still re-authenticates through `applyAuthChoice` without clearing stale usage-state lockouts. Direct `models auth login` cleanup and active-window immutability are useful partial fixes, but they do not fully resolve the issue body or the follow-up GitHub Copilot configure re-auth comment.

## Best Possible Solution

Keep this issue open and finish the shared provider auth recovery work: allow throttled billing recovery probes for same-provider-only fallback chains that share disabled profiles, and make successful provider re-auth through `openclaw configure --section model` clear stale cooldown/disabled usage state for the affected provider/profile. Add regression coverage for Anthropic same-profile Opus fallback billing recovery and GitHub Copilot configure re-auth clearing a `format` cooldown.
