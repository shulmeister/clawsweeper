## Summary

Keep open. Current `main` still does not ship a bundled/app-contained Node or Gateway runtime for macOS: the docs explicitly say `OpenClaw.app` expects an external CLI install, the npm package entrypoint remains a Node shebang, and the macOS app/LaunchAgent resolution still selects a Node/OpenClaw command from managed/user/system paths. There is a useful local-prefix installer workaround that can install Node under `~/.openclaw`, but that is optional and does not fully solve the issue's requested default signed app or dedicated-runtime packaging path.

## Best Possible Solution

Keep the issue open. The best maintainer path is to either ship a signed app-contained runtime or a first-class dedicated OpenClaw-owned Node path for macOS, then document how Full Disk Access, Accessibility, Automation, and iMessage/Mail process contexts map to that runtime. Cross-link #7227 for the related Accessibility/TCC security class, but preserve this issue for the dedicated-runtime and Full Disk Access packaging work.
