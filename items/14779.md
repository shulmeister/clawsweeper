## Summary

Current main already provides equivalent cost-management seams: plugin model override (`before_model_resolve`), prompt/context mutation (`before_prompt_build` plus plugin-bundled `agent:bootstrap` hooks), bounded context budgets, and heartbeat short-circuiting/cost controls.

## Close Comment

Closing from a Codex review because current `main` already covers this request with shipped equivalents. Plugins can override provider/model selection through `before_model_resolve` (`src/plugins/hook-before-agent-start.types.ts:1`, wired in `src/agents/pi-embedded-runner/run/setup.ts:58`), mutate prompt/system context through `before_prompt_build` (`src/plugins/hook-before-agent-start.types.ts:21`), and plugin-bundled `agent:bootstrap` hooks can mutate injected bootstrap files before prompt assembly (`docs/automation/hooks.md:16`, `docs/automation/hooks.md:45`, `docs/automation/hooks.md:123`). Heartbeat also already skips empty/comment-only `HEARTBEAT.md` and task-mode runs with nothing due instead of making a model call (`src/infra/heartbeat-runner.ts:615`, `src/infra/heartbeat-runner.ts:713`, `src/infra/heartbeat-runner.ts:855`), and main documents bounded context budgets plus heartbeat cost controls like `lightContext` and `isolatedSession` (`docs/gateway/configuration-reference.md:989`, `docs/gateway/configuration-reference.md:1372`). If there is still a concrete gap beyond these shipped seams, please open a fresh focused issue against that specific missing hook or behavior.
