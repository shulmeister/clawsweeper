## Summary

Keep open. Current `main` still routes Volcengine through the generic OpenAI-completions transport, the Volcengine docs still present raw `VOLCANO_ENGINE_API_KEY` setup and DeepSeek V3.2 as a supported surface, and the bundled tests do not cover an actual Volcengine completion request with a Volcano-style UUID key. I did not find strong evidence that #56199 was fixed or made obsolete on `main`.

## Evidence

- **Volcengine still passes the stored key through unchanged:** The plugin resolves `VOLCANO_ENGINE_API_KEY` and injects that value directly into both the `volcengine` and `volcengine-plan` provider configs. There is no Volcengine-specific key-format conversion or alternate auth branch in this registration path.
