## Summary

Keep open. Current main updates ACP session repair/retry behavior and pins ACPX 0.6.0, but the requested process-tree/orphan cleanup is not implemented. The pinned ACPX runtime still spawns agent subprocesses without a detached process group, terminates only the direct child with child.kill(SIGTERM/SIGKILL), and still unrefs a child that does not exit. Existing OpenClaw tests cover retrying missing persistent sessions, not reaping old orphaned claude-agent-acp/plugin-tools process trees. Related orphan-process issues #52708 and #68916 remain open and do not clearly supersede this persistent-session repair report.
