## Summary

Keep open. Current main implements the per-agent TTS half, but the requested per-agent STT/language/provider override is still missing. The issue discussion already reached the same boundary: a maintainer comment says `agents.list[].tts` is implemented and the issue should remain open for STT. Local source confirms agent config exposes `tts` only, while audio transcription still resolves from global `tools.media.audio`, per-entry settings, or explicit request overrides rather than an active agent `stt` block.

## Best Possible Solution

Keep this issue open and narrow the remaining work to per-agent STT: define an `agents.list[].stt` or equivalent media-audio override contract, wire it into active-agent transcription paths such as media understanding and Discord voice, add schema/type/docs coverage, and preserve the already-shipped `agents.list[].tts` behavior.
