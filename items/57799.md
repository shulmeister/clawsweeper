## Summary

Close #57799 as `clawhub`. The PR is a large optional standalone Mission Control dashboard app, while current main already ships a Gateway-served Control UI and the project scope directs optional capabilities to ClawHub/npm plugins or external apps. Current plugin HTTP routes and gateway methods provide a path for secure data access without adding this app to core, and the PR discussion also identified serious merge blockers.

## Best Possible Solution

Do not merge this standalone dashboard into OpenClaw core. Keep the shipped Gateway Control UI as the first-party admin surface, and move Mission Control to a ClawHub/npm plugin or separately maintained external app using authenticated plugin HTTP routes or gateway methods. Open a narrower plugin/API proposal only if that external implementation path exposes a concrete missing OpenClaw extension point.
