## Summary

Current main supports a configurable single ackReaction plus lifecycle status reactions, but it does not implement the requested configured array or agent-selected/contextual ack reaction for each inbound message. The issue is clear, non-protected, and not a duplicate from the provided context, so it should stay open.

## Evidence

- **Global config type only allows one ack reaction string:** messages.ackReaction is typed as an optional string, while statusReactions is a separate lifecycle configuration. There is no ackReactions array or dynamic selector in the global messages contract.
  - file: [src/config/types.messages.ts:113](https://github.com/openclaw/openclaw/blob/cbe5515b70abaeba98550b58d739ca0896907c43/src/config/types.messages.ts#L113)

- **Schema confirms fixed shape:** The zod schema accepts ackReaction as z.string().optional() and statusReactions.emojis as fixed lifecycle keys such as thinking/tool/done/error, not arbitrary per-message choices from an array.
  - file: [src/config/zod-schema.session.ts:165](https://github.com/openclaw/openclaw/blob/cbe5515b70abaeba98550b58d739ca0896907c43/src/config/zod-schema.session.ts#L165)
