## Summary

Keep PR #67156 open. Current main still ships `skills/canvas/SKILL.md` without YAML frontmatter, while the active skill loader requires a parsed `description` and the docs require `name` plus `description`. The PR’s final patch directly addresses the still-present defect and incorporated the review feedback to avoid a hard Darwin OS gate.

## Best Possible Solution

Keep the PR open and either land it or land a maintainer-equivalent patch that adds minimal `name` and `description` YAML frontmatter to `skills/canvas/SKILL.md`, with optional non-gating metadata such as the emoji. Best follow-up is to extend the bundled skill frontmatter regression test to include canvas or all bundled skills.

- Landing only the metadata change fixes discovery for canvas, but broadening the bundled-frontmatter test would reduce the chance of another bundled skill silently missing required metadata.
