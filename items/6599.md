## Summary

Keep open. Current main has runtime model fallback support and `models status --probe`, but it still does not expose `/models test-fallback` or an equivalent command that intentionally skips the primary model and verifies the configured fallback chain. The related implementation PR #43342 was closed unmerged, and the latest maintainer discussion already points to keeping #6599 open.

## Best Possible Solution

Continue tracking #6599 until maintainers implement or explicitly decline a first-class fallback-chain probe. A strong implementation would add `/models test-fallback` and likely a CLI equivalent, reuse the real fallback resolution/run path with an intentional primary skip, support first-fallback and all-fallback modes, add focused slash/CLI tests, and document how it differs from `models status --probe`.
