## Summary

Keep #6599 open. Current main at 82262387651ab270b565f71533ce4937c9c00dc6 still has model fallback runtime and `models status --probe`, but no `/models test-fallback` or `openclaw models test-fallback` command that intentionally skips the primary model and exercises the configured fallback chain. The provided issue context also includes a same-day maintainer/Codex keep-open comment and notes that PR #43342, the direct implementation attempt, is closed unmerged.

## Best Possible Solution

Keep #6599 open until maintainers implement or explicitly decline a first-class fallback-chain probe. A strong solution would add `/models test-fallback` and likely `openclaw models test-fallback`, reuse the real fallback resolution/run path with an intentional primary skip, support first-fallback and all-fallback modes, respect active agent/session auth context, add focused slash/CLI tests, and document how it differs from `models status --probe`.
