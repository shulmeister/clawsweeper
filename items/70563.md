## Summary

Keep #70563 open. Current main has generic replay cleanup for thinking blocks, but I could not verify a Kimi Coding fix for the specific `reasoning_content` contract in this report. `sessions_spawn` still routes subagent runs through the child agent path with model/thinking overrides, Kimi enables upstream thinking on request payloads, and `extensions/kimi-coding` has no `reasoning_content` replay or placeholder handling. The existing Kimi tests cover thinking toggles and tagged tool-call conversion, not `sessions_spawn` with Kimi thinking replay.

## Best Possible Solution

Keep the issue open and add a focused Kimi Coding fix: either capture/replay the provider-required `reasoning_content` for assistant tool-call turns, synthesize a provider-acceptable placeholder as DeepSeek/Venice do where appropriate, or explicitly disable Kimi thinking for replay shapes that cannot satisfy the upstream contract. The regression test should cover `sessions_spawn runtime="subagent"` with Kimi thinking enabled and a tool-call history replay.
