## Summary

Keep this issue open. Current main still has the reported Kimi Coding shape: native subagent spawning can run with a thinking override, but the Kimi provider only toggles top-level `thinking` and still lacks a replay repair that preserves or fills `reasoning_content` on assistant tool-call messages. The related DeepSeek, Venice, and Z.AI fixes do not cover Kimi Coding, and the prior bot review remains consistent with current main.

## Best Possible Solution

Keep this issue open and fix the provider-owned replay path in `extensions/kimi-coding`. When Kimi thinking is enabled, the provider should preserve existing `reasoning_content` or add a Kimi-accepted placeholder for replayed assistant tool-call messages, with focused regression coverage for thinking-enabled `sessions_spawn` plus Kimi tool-call continuation. Cross-check #50178 for any reusable provider seam, but do not close this Kimi-specific report as implemented or duplicated yet.
