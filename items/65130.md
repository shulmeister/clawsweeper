## Summary

Keep open. Current `main` still routes package-manager updates through plain `npm i -g openclaw@...` without any `EEXIST`-specific recovery or `--force` fallback, while the installer has separate logic for that exact failure mode. The direct npm-global container update smoke is also skipped in CI, so there is no strong evidence that the reported container update path is fixed on `main`.

## Evidence

- **Package-manager update path still uses plain npm global install:** `globalInstallArgs()` returns `npm i -g <spec> --no-fund --no-audit --loglevel=error` for npm, and the only retry path is `--omit=optional`; there is no `--force` or `EEXIST` handling in this updater path.
  - file: [src/infra/update-global.ts:561](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/src/infra/update-global.ts#L561)

- **`openclaw update` delegates global package installs to that argv:** The update command resolves the global install target and executes `runUpdateStep({ name: "global update", argv: globalInstallArgs(...) })`, so container/package-manager updates still hit the plain npm install path above.
  - file: [src/cli/update-cli/update-command.ts:347](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/src/cli/update-cli/update-command.ts#L347)
