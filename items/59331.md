## Summary

Close as implemented on main. Current Telegram polling code persists per-account `update_id` watermarks to disk, reloads and confirms them before restarting polling, and adds safe watermark/dedupe guards so already-handled updates are skipped instead of replayed after a gateway restart.

## Evidence

- **Persistent Telegram offset store:** Current main has a dedicated Telegram offset store that resolves a per-account state file under the OpenClaw state dir, reads a saved `lastUpdateId`, writes updated values atomically, and rejects stale records when the bot token changes.
  - file: [extensions/telegram/src/update-offset-store.ts:27](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/telegram/src/update-offset-store.ts#L27)
  - command: `nl -ba extensions/telegram/src/update-offset-store.ts | sed -n '1,160p'`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Polling startup reads and persists offsets:** The Telegram monitor now loads `readTelegramUpdateOffset`/`writeTelegramUpdateOffset`, reads the persisted offset before startup, normalizes it, and persists newer update ids during polling before handing control to `TelegramPollingSession`.
  - file: [extensions/telegram/src/monitor.ts:161](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/telegram/src/monitor.ts#L161)
  - command: `nl -ba extensions/telegram/src/monitor.ts | sed -n '155,235p'`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Replay guard in bot runtime:** The bot runtime tracks pending and failed update ids, persists only a safe watermark below unfinished work, and skips updates at or below the persisted cutoff, which directly addresses restart replay/duplicate processing risk.
  - file: [extensions/telegram/src/bot-core.ts:287](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/telegram/src/bot-core.ts#L287)

  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Regression test for startup confirmation:** There is an explicit test asserting that a persisted offset is confirmed with Telegram via `getUpdates({ offset: last+1, limit: 1, timeout: 0 })` before the polling runner starts.
  - file: [extensions/telegram/src/monitor.test.ts:766](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/telegram/src/monitor.test.ts#L766)
