## Summary

Keep open. Current main still has a Feishu tool-send path that bypasses the shared `message:sent` hook emitter, so there is not strong evidence that issue #52144 is fixed.

## Evidence

- **Shared outbound core is where `message:sent` is emitted:** `src/infra/outbound/deliver.ts:507` defines `createMessageSentEmitter(...)`, and `src/infra/outbound/deliver.ts:549` triggers the internal `message:sent` hook from that shared delivery path.
  - file: [src/infra/outbound/deliver.ts:507](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/infra/outbound/deliver.ts#L507)
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Tool sends try plugin dispatch before core delivery:** `src/infra/outbound/outbound-send-service.ts:71` dispatches channel plugin actions first, and `src/infra/outbound/outbound-send-service.ts:164` returns early when a plugin handled the send, so the shared `sendMessage`/`deliverOutboundPayloads` path is skipped.
  - file: [src/infra/outbound/outbound-send-service.ts:71](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/infra/outbound/outbound-send-service.ts#L71)
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Feishu `send` is plugin-handled on current main:** `extensions/feishu/src/channel.ts:685` handles `ctx.action === "send" || ctx.action === "thread-reply"` inside the Feishu plugin and calls `runtime.sendMessageFeishu`, `runtime.sendCardFeishu`, or `runtime.feishuOutbound.sendMedia` directly.
  - file: [extensions/feishu/src/channel.ts:685](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/extensions/feishu/src/channel.ts#L685)
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Feishu send implementation has no hook emission:** `extensions/feishu/src/send.ts:454` sends directly through Feishu API helpers and returns a `FeishuSendResult`; there is no `triggerInternalHook`, `createInternalHookEvent`, or `runMessageSent` call in this send path.
  - file: [extensions/feishu/src/send.ts:454](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/extensions/feishu/src/send.ts#L454)
  - command: `rg -n "message:sent|runMessageSent|triggerInternalHook|createInternalHookEvent" extensions/feishu/src`
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Docs still promise the event for outbound delivery:** `docs/automation/hooks.md:50` documents `message:sent` as firing when an outbound message is delivered, which matches the reporter's expectation and makes the missing Feishu tool-send hook a real behavioral gap rather than an unsupported use case.
  - file: [docs/automation/hooks.md:50](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/docs/automation/hooks.md#L50)
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
