## Summary

Keep this issue open. Current main still has the reported timeout-compaction fall-through: high-context timeouts call `contextEngine.compact()`, but only `compacted: true` retries the prompt. `compacted: false` still logs the same warning and falls through to normal timeout/error handling, and focused tests assert that behavior.

## Best Possible Solution

Keep the issue open. The fix likely belongs in `src/agents/pi-embedded-runner/run.ts`: after timeout-triggered compaction returns `compacted: false` or otherwise fails to reduce context, route to an explicit recovery ladder instead of normal timeout handling. Reuse existing pieces where safe: compaction fallback candidates tracked by #64960 if/when implemented, `compact_then_truncate` or tool-result truncation when the context shape supports it, and finally a clear blocked-session or fresh-session recovery path. Add regression coverage that replaces the current fall-through expectations in `run.timeout-triggered-compaction.test.ts`.
