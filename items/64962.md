## Summary

Current `main` still sends failed timeout-triggered compaction down the timeout/failover path rather than escalating into the overflow-recovery path (`compact_then_truncate`/tool-result truncation/session reset). The repository also has explicit tests that lock in that behavior, so this issue is not implemented on `main` and should stay open.

## Evidence

- **Timeout compaction still falls through on `compacted:false`:** `run.ts` increments timeout-compaction attempts, calls `contextEngine.compact()`, and when `timeoutCompactResult.compacted` is false it logs `compaction did not reduce context ...; falling through to normal handling` with no overflow-specific escalation.

- **Mainline test codifies timeout error after failed timeout compaction:** `run.timeout-triggered-compaction.test.ts` explicitly asserts that when timeout compaction returns `{ compacted: false }`, the runner returns an error payload containing `timed out` and leaves the run blocked.
