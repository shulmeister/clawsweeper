## Summary

Keep open. Current main still interpolates the memory schema table identifiers directly, so the PR is addressing a real current code path. It should not be closed as implemented or duplicate, but it also is not ready to land unchanged because the PR has no regression tests and an unresolved review note about the new dot-splitting table quoter regressing pre-quoted identifiers.

## Best Possible Solution

Keep this PR open for maintainer review and iteration. The best path is a revised memory-schema fix that defines the SQLite identifier contract, quotes or validates dynamic identifiers consistently in both `src/` and `packages/`, preserves intended schema-qualified behavior, adds regression tests for malicious and edge-case identifiers, and then lands if the security model holds.

## Risks / Open Questions

- Landing the PR unchanged could regress valid pre-quoted or dotted SQLite identifiers because `quoteTable` blindly splits on every dot.
- The PR currently lacks focused regression tests for malicious identifier input, schema-qualified table names, and the chosen contract for pre-quoted identifiers.
- The default bundled caller uses constants today, so maintainers should confirm whether the exported SDK helper is intended to accept arbitrary table names or should be narrowed.
