## Summary

Keep open. Current main still does not show model-run errors inline in web chat: live error events only set lastError, assistant error-only history has no text/render path, and the UI lacks the error bubble styling/tests this PR adds. The PR is still a plausible focused fix rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open or fold its narrow behavior into a maintainer-owned follow-up. The implementation should append structured assistant error messages for live chat error events, preserve/render persisted assistant stopReason: "error" entries including empty-detail fallbacks, style them distinctly in the web chat, and update controller/message extraction/rendering tests against current main's normalizeMessage/grouped-render path.
