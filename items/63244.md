## Summary

Keep open. Current main does not implement PR #63244's central behavior: chat delta/final/error broadcasts and non-tool agent broadcasts still include only sessionKey, while spawnedBy is only present on session snapshot/tool/session event paths. The PR is still a focused, potentially useful additive contract change, and the provided discussion shows active review of schema-sync and hot-path lookup concerns rather than an obsolete or duplicate item.

## Best Possible Solution

Keep PR #63244 open for maintainer review. The best path is to finish the additive gateway contract change on the PR branch: enrich subagent/ACP chat delta, flushed delta, final, error, seq-gap, and non-tool agent broadcasts with spawnedBy; keep non-lineage sessions unchanged; update the chat/agent schemas; and retain targeted tests for all affected broadcast paths. Any broader optimization for session-store clone/read cost should be tracked separately at the store-cache layer if maintainers agree it is needed.
