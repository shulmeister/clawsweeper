## Summary

Keep open. Current `main` already has the persistent-session retry path for `runTurn`, but it still does not implement the other two behaviors this PR targets: retrying `setSessionConfigOption()` after stale persistent-session resume failures, and treating `ACP_TURN_FAILED` resume-loss errors as stale-binding cleanup signals. That means the PR is not implemented on `main`, not obsolete, and not safe to auto-close.

## Evidence

- **`setSessionConfigOption()` still has no retry path on `main`:** `setSessionConfigOption()` resolves the session, calls `ensureRuntimeHandle()`, then does a single `runtime.setConfigOption()` call inside `withAcpRuntimeErrorBoundary()`. There is no retry loop and no call to `prepareFreshHandleRetry()` in this path.
