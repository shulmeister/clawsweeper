## Summary

Keep open. Current main still displays `openclaw logs` timestamps as UTC by default: `--local-time` remains opt-in, the runtime only enables local time when that flag or a valid `TZ` is present, and the formatter falls back to `toISOString()`/UTC when local time is disabled. The parser now prefers top-level `time` over the UTC metadata fallback, but that local-offset value is still reparsed and rendered as UTC unless local-time mode is active. Related PRs #46782 and #47365 propose the requested behavior but are still open; #47240 was closed unmerged.
