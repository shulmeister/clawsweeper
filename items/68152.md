## Summary

Keep open. Current main still renders the exec approvals host select with a hardcoded `Node` option, while `nodes.binding.node` already exists in the locale bundles and is used elsewhere in the Nodes UI. The PR is still relevant, though it should likely address the adjacent hardcoded `Gateway` option before merge so non-English locales do not get a mixed-language select.

## Best Possible Solution

Keep the PR open and ask for a small follow-up: translate the exec approvals target select consistently, using `t("nodes.binding.node")` for `Node` and adding/using an appropriate `Gateway` translation key across locale bundles before merge. A focused render test for the zh-CN option labels would be useful if the UI test harness can cover this view cheaply.
