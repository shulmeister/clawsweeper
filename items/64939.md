## Summary

Current `main` already contains the OpenAI-compatible audio transcription fixes that match this Groq report: the shared Groq STT path uses the fixed multipart upload helper, guarded runtime fetch now normalizes cross-implementation `FormData` bodies and clears stale multipart headers, and current tests cover the exact `model`/file preservation case. The changelog shows the relevant fixes shipped across `2026.4.11` and `2026.4.12`, so this no longer needs an open core bug on `main`.
