## Summary

Keep #68101 open. Current main still swallows workflow-file resolution failures in the Lobster wrapper and then falls back to `runToolRequest({ pipeline, ctx })`, so a missing relative `.lobster` path can still be treated as an inline pipeline. The linked PR #68106 is the right canonical fix path, but it is still open/unmerged in the provided GitHub context; PR #68132 is closed and unmerged.

## Best Possible Solution

Keep the issue open until #68106 or an equivalent maintainer-reviewed fix lands. The fix should preserve workflow-file resolution errors for file-like relative inputs, keep normal inline pipelines working, and add focused regression tests for missing relative workflow files plus slash-containing pipeline arguments.
