## Summary

Current `main` still drops workflow-file resolution errors in the Lobster runner and falls back to `runToolRequest({ pipeline })`, which matches the reported misleading behavior rather than a fix. Docs still say `pipeline` may be a workflow file path, and the existing tests only cover successful file detection, not the failing relative-path case.

## Evidence
