## Summary

Keep #60713 open. Current main still has the reported Windows-sensitive behavior: `resolvePreferredOpenClawTmpDir()` probes and returns `/tmp/openclaw` before falling back to `os.tmpdir()`, with no `win32` guard, and the existing tests assert that POSIX path preference.

## Evidence

- **Current implementation still prefers POSIX temp first:** `resolvePreferredOpenClawTmpDir()` unconditionally checks `POSIX_OPENCLAW_TMP_DIR` and returns it when available before considering the fallback path.
