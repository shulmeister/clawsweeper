## Summary

Keep PR #40472 open. Current main still does not refresh the TUI on an external session reset: the gateway reset path emits only `sessions.changed` to session-event subscribers, while the TUI gateway client does not subscribe and the TUI event loop handles only `chat`, `chat.side_result`, and `agent`. The linked issue #38966 remains the canonical user bug, and this PR is a focused attempt to fix that behavior, though it likely needs rebasing/adaptation to the current `sessions.changed` subscription model.

## Best Possible Solution

Keep the PR open or supersede it with a maintainer-owned patch that makes TUI subscribe to the current session lifecycle signal and refresh only when the changed/reset event matches `state.currentSessionKey`. The implementation should clear stale active run/tool tracking, reload history/session info, ignore unrelated sessions, and add focused TUI handler tests plus a gateway/TUI contract check for the advertised event path.
