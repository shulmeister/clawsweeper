## Summary

Keep #70605 open. Current main does not implement the PR’s requested behavior: bootstrap/context file content is trimmed and injected into the system prompt, but `sanitizeContextFileContentForPrompt()` only removes the default heartbeat prompt quote and collapses extra blank lines. The PR is recent, non-maintainer-authored, has no protected labels, and contains a plausible focused fix plus tests, though maintainers should review the regex safety concerns from the existing bot review before merging.

## Best Possible Solution

Keep the PR open for maintainer review. If OpenClaw wants struck-through bootstrap text to mean deleted instructions, land a safe sanitizer in `src/agents/system-prompt.ts` with focused tests for inline markdown, multiline blocks, HTML tags with attributes, and tilde-fenced code preservation. If maintainers prefer a broader context-management solution, mark this PR superseded only after linking a canonical implementation or follow-up that actually covers the strikethrough behavior.
