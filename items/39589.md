## Summary

Keep #39589 open. Current main still rejects the reporter's `models.providers.openai` provider entry when it omits `models`, so the custom OpenAI-compatible embedding base URL cannot be accepted through that config path. The memory embedding code would honor the provider `baseUrl` if validation allowed the entry, and docs provide `memorySearch.remote.baseUrl` as a workaround, but the reported provider-only configuration remains an actionable config-contract bug. The related PRs #46977 and #46980 proposed the same fix but are closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep #39589 open until a focused config-contract fix lands. The likely fix is to make `ModelProviderSchema.models` optional with a default empty array, regenerate generated config schema artifacts, add a regression test for a provider entry containing only `baseUrl`/`apiKey`/`api`, and add or keep a memory embedding test proving `models.providers.openai.baseUrl` is used when `memorySearch.remote.baseUrl` is absent.
