## Summary

Keep open. Current main still requires `models.providers.*.models`, so the reporter's provider-only OpenAI config remains invalid. The memory embedding client would honor `models.providers.openai.baseUrl` after config validation, but the schema still blocks the exact configuration in #39589. The related #46977 fix was closed unmerged and #46980 remains the relevant open PR from the provided context, so this is not implemented on main or safely superseded by shipped behavior.

## Best Possible Solution

Keep #39589 open until a focused fix lands. The best implementation is to make `ModelProviderSchema.models` optional with a default empty array, regenerate the config schema artifacts, add a schema regression test for a provider entry containing only `baseUrl`/`apiKey`/`api`, and add or retain a memory embedding test proving `models.providers.openai.baseUrl` is used when no `memorySearch.remote.baseUrl` override is set. #46980 appears to be the current candidate PR to review or revive.
