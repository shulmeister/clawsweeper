## Summary

Keep #39589 open. Current main still rejects provider-only `models.providers.openai` entries because the core Zod schema, generated config schema, and TypeScript config contract all require `models`. The memory embedding runtime would honor `models.providers.openai.baseUrl` if validation accepted the config, so the remaining problem is still the config contract. Related PRs #46977 and #46980 attempted this same optional-models fix but are closed unmerged in the provided context.

## Best Possible Solution

Keep #39589 open until a focused config-contract fix lands: make `ModelProviderSchema.models` optional with a default empty array, align `ModelProviderConfig`, regenerate generated schema/help artifacts, add a regression test proving `models.providers.openai` with only `baseUrl`/`apiKey`/`api` validates, and keep or add an embedding test proving provider `baseUrl` is used when `memorySearch.remote.baseUrl` is absent.
