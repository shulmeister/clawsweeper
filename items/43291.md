## Summary

Keep open. Current main still does not implement the PR’s CLI-backend fix: the CLI prepare path resolves and passes a heartbeat prompt, but the CLI system-prompt builder has no sessionKey/promptMode input, so buildAgentSystemPrompt falls back to full mode and can still include ## Heartbeats/HEARTBEAT_OK for default-agent cron or subagent CLI sessions. The embedded runner has the minimal-mode helper, but that behavior is not wired into the CLI-backend prompt path. The PR discussion and Greptile review support the root cause and the later shared-helper direction, so this remains useful work rather than a cleanup close.

## Best Possible Solution

Keep the PR open or land an equivalent scoped fix: add sessionKey to the CLI buildSystemPrompt path, derive minimal/full mode from the same cron/subagent helper used by the embedded runner, pass promptMode into buildAgentSystemPrompt, and add focused CLI regression tests proving cron/subagent CLI prompts omit ## Heartbeats/HEARTBEAT_OK while normal CLI sessions keep full guidance.
