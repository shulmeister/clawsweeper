## Summary

Current `main` now converts timeout-with-no-output runs into explicit user-visible errors, reactivates completed subagent sessions before follow-up sends, and documents both the LLM idle-timeout knob and subagent run-timeout knob. That matches the issue's requested recovery behavior well enough to close as implemented on `main`, although the fix does not appear in the latest tagged release yet.

## Evidence

- **Silent timeout path now emits a user-visible error:** `runEmbeddedPiAgent` no longer silently completes when a timed-out run has no assistant payloads; it emits explicit timeout text and, for idle watchdog cases, points users to `agents.defaults.llm.idleTimeoutSeconds`.
