## Summary

Keep open: current `main` still assembles MCP stdio child env as `getDefaultEnvironment()` plus user-configured `env`, and the adjacent tests/docs still treat `NODE_OPTIONS` as blocked config rather than ambiently propagated from the gateway host. The upstream MCP SDK safe inherited env list omits proxy vars and `NODE_OPTIONS`, so the requested host-level proxy propagation is not implemented on `main`. 

## Evidence

- **Current stdio child env assembly:** `OpenClawStdioClientTransport.start()` builds `baseEnv` from `getDefaultEnvironment()` merged with `this.serverParams.env`; it does not merge ambient `process.env` proxy variables into MCP stdio children.
