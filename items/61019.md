## Summary

Keep open. Current main still lacks a session-model refresh or migration path after `openclaw models set`: the CLI changes the global config only, while session list/UI surfaces continue to prefer persisted runtime `model`/`modelProvider` fields for existing sessions. The next agent run appears to use the new configured default when there is no explicit session override, but the reported active-session status/picker confusion remains.

## Evidence

- **models set only updates config:** `modelsSetCommand` calls `updateConfig` with `applyDefaultModelPrimaryUpdate`, logs the new default, and does not touch session stores or active session metadata.
  - file: [src/commands/models/set.ts:6](https://github.com/openclaw/openclaw/blob/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54/src/commands/models/set.ts#L6)
