## Summary

Keep PR #40366 open. Current `main` does not implement the PR's two central behaviors: normalized-empty agent command outputs still do not get a fallback reply, and Slack preview/progress turns that end without a final reply still clear the draft and return silently. The PR discussion is also materially useful because the author iterated through concrete bot-review correctness issues and added targeted regressions on the branch.

## Best Possible Solution

Keep #40366 open for maintainer review or contributor rebase. The best path is to adapt the branch's normalized-empty delivery fallback and Slack preview-only no-final fallback to the current `src/agents/command` and `extensions/slack` code, preserving the reviewed guardrails for intentional `NO_REPLY`, successful tool-sent empty runs, block-stream success, same-thread status delivery, and JSON output behavior.
