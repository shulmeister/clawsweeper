## Summary

Keep this PR open. Current main still has the Node DEP0190 pattern in `scripts/ui.js`, and the PR remains a plausible implementation candidate, but the submitted `args.join(" ")` approach risks losing Windows argument boundaries and has no regression coverage. The Greptile comment found no P0/P1 issue, while the later ClawSweeper comment identified the same quoting and coverage gaps; current-main inspection still supports keeping it open for revision.

## Best Possible Solution

Keep the PR open and ask for a current-main revision that routes `scripts/ui.js` through `scripts/pnpm-runner.mjs` or an equivalent shared Windows spawn resolver. The fix should avoid `shell: true` plus a separate args array, preserve argument boundaries with tested command-line quoting, and add regression coverage for `.cmd` launchers, paths containing spaces, args containing spaces, safe forwarded args, and unsafe metacharacter rejection.
