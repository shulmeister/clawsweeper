## Summary

Keep open. Current main still has the duplicated subagent task path reported in #72019: the task is embedded in the subagent system prompt and also sent as `[Subagent Task]: ${task}` in the initial child user message. PR #72026 is an active, focused fix attempt rather than obsolete main behavior; related PR #72053 is a stronger overlapping candidate, but it is also open and unmerged, so this should remain a maintainer choice rather than an automatic cleanup close.

## Best Possible Solution

Keep #72026 open for maintainer review while consolidating the overlapping #72019 fixes. The best fix should keep the full delegated task in exactly one model-visible payload location, preserve multiline/structured task text, cover the final `message` plus `extraSystemPrompt` gateway payload with regression tests, and either remove or explicitly deprecate any unused `task` parameter.
