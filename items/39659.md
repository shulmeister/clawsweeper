## Summary

Keep #39659 open. Current main still has Docker/env/profile/container primitives and ClawDock convenience helpers, but no first-class named Docker instance lifecycle or equivalent managed create/list/update/delete workflow.

## Best Possible Solution

Keep #39659 open until maintainers either implement a minimal Docker multi-instance lifecycle or explicitly choose another product direction. A strong solution would add a documented command or supported helper that stores per-instance metadata, validates or allocates ports and paths, sets isolated compose project/container names, and provides safe create/list/update/delete cleanup with tests and Docker install docs. The related profile-management PR context is not a canonical duplicate because profiles do not cover Docker compose/container/port lifecycle.
