## Summary

Keep #39659 open. Current main has profile-based local gateway isolation, Docker Compose setup, ClawDock single-project helper commands, and `--container` execution targeting, but it still does not provide the requested first-class Docker multi-instance lifecycle with named create/list/delete/update, per-instance metadata, managed port/path allocation, and compose/container isolation. Related PR #54126 was closed unmerged and is not a shipped replacement.

## Best Possible Solution

Keep this issue open until maintainers implement or explicitly decline a supported Docker multi-instance lifecycle. A good implementation would add a documented `openclaw instance ...` or Docker/ClawDock equivalent that records per-instance metadata, validates or allocates unique host ports and paths, isolates compose project/container names, and provides safe create/list/update/delete cleanup with focused tests and Docker install docs.
