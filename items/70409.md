## Summary

Current main does not implement #70409's stage-only Dreaming maintenance loop. Deep dreaming still ranks candidates and calls the direct MEMORY.md promotion path, /dreaming exposes only status/on/off, docs still describe deep as writing MEMORY.md, and the memory-core config schema/resolver has no dailySignalFiles or maintenance surface. The PR is repo-relevant and potentially useful, but the provided review context flags unresolved P1 issues and unrelated Bedrock Mantle changes, so it needs maintainer cleanup rather than conservative closure.

## Best Possible Solution

Keep the PR open for maintainer/product review, but require a narrower memory-core-only branch that addresses the apply/rollback crash-safety and daily-signal review comments, then aligns the manifest, SDK config resolver, docs, slash command behavior, and focused tests. If maintainers prefer not to take the XL branch, supersede it with smaller canonical PRs for stage-only maintenance and dailySignalFiles.
