## Summary

Keep open. Current main still models cron payloads as only systemEvent or agentTurn and the Gateway cron runtime still has no command-job dependency or execution branch, so the backend/runtime/server capability requested by #62789 remains missing. The related CLI split was acknowledged in the PR discussion, but it does not supersede this backend PR.
