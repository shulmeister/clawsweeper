## Summary

Current main already removed the Feishu message-tool requirement that forced a `card` schema and now supports media-only sends through the outbound media path, with a regression test covering the exact scenario.

## Evidence

- **Feishu discovery no longer contributes a required card schema:** `describeFeishuMessageTool` returns only `actions` and `capabilities`; there is no `schema` contribution here anymore, so Feishu no longer makes `card` required at tool-schema discovery time.
  - file: [extensions/feishu/src/channel.ts:182](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/channel.ts#L182)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Media-only sends are handled without card:** The send/thread-reply path reads `media`, allows `!card && mediaUrl`, and routes media sends through `feishuOutbound.sendMedia`; the only rejected combination is still `card && mediaUrl`.
  - file: [extensions/feishu/src/channel.ts:695](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/channel.ts#L695)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Regression test matches the reported bug:** There is an explicit test named `sends media-only messages without requiring card` that sends Feishu `media` with no card and expects the outbound media path to be called successfully.
  - file: [extensions/feishu/src/channel.test.ts:803](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/channel.test.ts#L803)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Changelog says the fix shipped in 2026.3.24:** The 2026.3.24 changelog entry says Feishu `message(..., media=...)` sends were routed through the outbound media path so file/image attachments actually send.
  - file: [CHANGELOG.md:2181](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/CHANGELOG.md#L2181)
