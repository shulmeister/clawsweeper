## Summary

Keep open. The PR is a focused bug fix with a linked reproduction (#68044) and a positive review comment, but current main does not implement the direct `anthropic` name-based image-support fallback that the PR adds. Current main has newer attachment offload and Anthropic dynamic-model paths that may mitigate the missing-catalog case, but the stale text-only catalog path is not proven fixed and the helper/tests still only cover `claude-cli`, not direct `anthropic`.

## Best Possible Solution

Keep the PR open until there is maintainer proof for both direct Anthropic missing-catalog and stale text-only catalog behavior. The best path is either to merge/adapt the narrow fallback with tests, or add a provider-owned model-capability normalization/contract test that proves pasted images reach `anthropic/claude-opus-4-7` even when catalog metadata lags.
