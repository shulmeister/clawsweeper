## Summary

Close as `clawhub`: the issue is a research spike for an optional Tesla Powerwall/Solar integration, and current `main` explicitly pushes optional integrations into external plugins/ClawHub rather than OpenClaw core.

## Evidence

- **Issue is scoped as external research, not a concrete core bug:** The issue body asks to research `jasonacox/pypowerwall` and summarize structure, auth/session patterns, polling strategy, and unsafe/outdated parts to copy. It does not identify a current OpenClaw regression or a missing SDK seam.
- **Vision says optional capability should usually ship as plugins:** Current `main` says OpenClaw should stay lean, optional capability should usually ship as plugins, plugin promotion belongs in ClawHub, and the bar for adding optional plugins to core is intentionally high.

A Tesla Powerwall/Solar integration fits that direction better as an external plugin than as core work.
  - file: [VISION.md:106](https://github.com/openclaw/openclaw/blob/6d271762/VISION.md#L106)
