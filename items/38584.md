## Summary

Current main’s Control UI streaming path maintains one live stream item per run and updates its text in place; the reported per-chunk bubble duplication does not match the code on `main` or the latest release tag.

## Close Comment

Thanks for the report. I reviewed this with Codex against current `main` (`51186d27`), and I can’t reproduce the reported “new bubble per chunk” behavior from the current Control UI code path. A chat run sets `chatStreamStartedAt` once at send start in `ui/src/ui/controllers/chat.ts:338`, delta events then replace `state.chatStream` in place in `ui/src/ui/controllers/chat.ts:451`, and the renderer builds a single live `stream` item in `ui/src/ui/chat/build-chat-items.ts:291` that is keyed once via `repeat(..., (item) => item.key, ...)` in `ui/src/ui/views/chat.ts:870`. The current release `v2026.4.22` (`00bd2cf7`) contains the same logic, so this report no longer matches current behavior. If you still see multiple bubbles on `v2026.4.22` or newer, please open a new issue with a current repro, screenshot, and the exact model/provider or tool activity involved.
