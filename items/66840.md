## Summary

Issue #66840 is a real current-main bug, but it is already superseded by open PR #67050, which explicitly tracks the same install-scanner test-file false positive and is the active fix path. Current main still has the env-plus-network critical rule and scans package directories without a test-file exclusion, so this should not close as implemented.

## Best Possible Solution

Use PR #67050 as the canonical review and implementation path: add a test-file/test-directory exclusion in `src/security/skill-scanner.ts` and regression coverage for install-time package scanning, while closing #66840 as the duplicate tracker.
