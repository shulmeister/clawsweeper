## Summary

Keep open. Current main still does not implement configurable TUI status-line command rendering: `ui.statusLine` is absent from config types and strict Zod validation, `runTui()` only mounts the existing built-in status/footer/editor layout, and there is no `tui-statusline`/`createStatusLine` implementation. This is also not a clean ClawHub close because the current plugin API documents providers, channels, tools, hooks, commands, routes, and similar capabilities, but no TUI render/status-area extension point.

## Best Possible Solution

Keep this PR open for maintainer/product review. If accepted, the best path is a core TUI/config change with type, Zod schema, generated schema/docs alignment, focused validation and lifecycle tests, and explicit documentation of the trusted local-command model. If maintainers prefer this outside core, first create a documented TUI status/render plugin seam, then move command rendering into a plugin.
