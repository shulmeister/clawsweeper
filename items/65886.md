## Summary

Keep open. Current main does not implement configurable TUI status-line command rendering: `ui.statusLine` is absent from the config type and strict Zod schema, `runTui()` only mounts the existing built-in status/footer/editor layout, and there is no `tui-statusline`/`createStatusLine` implementation on main. This also is not a clean ClawHub close because current plugin docs/API cover channels, providers, tools, hooks, commands, routes, etc., but not a TUI render/status-area extension point, so an external plugin cannot satisfy the request without a new core seam.

## Best Possible Solution

Keep the PR open for maintainer/product review. If accepted, the best path is a core TUI/config change with schema/generated-doc alignment, focused tests for config validation and command-runner lifecycle, and explicit documentation of the security/trust model. If maintainers prefer this to live outside core, first create a documented TUI status/render plugin seam, then move the command-rendering implementation to a plugin.
