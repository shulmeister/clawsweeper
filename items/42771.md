## Summary

Keep open. Current main still installs and documents Playwright browser downloads under `/home/node/.cache/ms-playwright`; the requested `/opt/playwright` Docker behavior and PATH symlink are not implemented or superseded. The PR is a focused Docker/docs change for the still-open linked bug #42819, though its patch needs review fixes before merge.

## Best Possible Solution

Keep the item open or fold it into the canonical Docker browser-path bug #42819. The best implementation path is a Dockerfile change that sets a stable browser path such as `/opt/playwright`, makes it writable for the runtime `node` user both with and without preinstalled browsers, preserves install failure propagation, avoids deleting BuildKit cache mounts for image-size reasons, adds targeted Dockerfile assertions, and updates the Docker/browser docs together.
