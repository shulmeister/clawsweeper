## Summary

Keep open. Current main still lacks the fallback/resilience requested in #40088: the default sync interval remains disabled, the memory watcher only wires add/change/unlink handlers, and interval sync still does not mark memory dirty before calling sync, so a dead watcher can still leave memory files unsynced unless the user manually configures intervalMinutes or forces an index.

## Best Possible Solution

Keep #40088 open and implement the focused memory-core fix: choose a nonzero default or otherwise provide a documented safety-net interval, make interval sync actually scan memory files without forcing a full reindex, add watcher error/ready logging or health handling, and add regression tests in src/agents/memory-search.test.ts and extensions/memory-core/src/memory/manager.watcher-config.test.ts. The closed unmerged #42042 is useful prior art but is not a canonical open tracker or shipped fix.
