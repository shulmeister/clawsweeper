## Summary

Keep PR #27771 open. Current main still does not implement `channels.signal.configPath`, the Signal schema remains strict without that key, and daemon autostart cannot pass `--config <path>` to `signal-cli`. The prior PR discussion also supports maintainer takeover after the contributor said they cannot continue.

## Best Possible Solution

Keep the PR open for maintainer takeover or replacement. Implement optional `channels.signal.configPath` in the current Signal plugin layout: add it to the exported config type, strict schema, generated metadata/UI hints, docs, and account configured detection; trim and omit blank values; propagate it through `monitorSignalProvider` into `spawnSignalDaemon`; emit `--config <path>` before daemon startup args; and add focused schema and autostart tests for set, blank, and unset cases under `extensions/signal`.
