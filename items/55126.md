## Summary

Current main still does not pass `memorySearch.outputDimensionality` into the bundled OpenAI embeddings request path. The generic config/provider plumbing preserves the field, but the OpenAI client, inline remote request body, batch request body, and OpenAI cache-key data still omit the corresponding OpenAI `dimensions` parameter. #55507 is an open, unmerged implementation candidate with closing syntax for this issue, so this issue should remain open until that PR is reviewed and either landed or closed.

## Best Possible Solution

Keep this issue open and review #55507 as the implementation candidate. The fix should preserve `outputDimensionality` on the OpenAI embedding client, send it as OpenAI `dimensions` for inline and batch embedding requests when supported, include dimensions in OpenAI memory embedding runtime cache-key data so dimension changes trigger reindexing, and add regression tests for request bodies and cache-key behavior.
