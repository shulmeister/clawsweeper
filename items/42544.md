## Summary

Keep open. Current main still runs service-mode stale gateway cleanup before startup solely when `OPENCLAW_SERVICE_MARKER` is set, and the cleanup helper remains kill-based rather than health-aware. The PR is not safe to merge as-is because review comments identify real flaws, but the underlying requested behavior has not been implemented or superseded on main.

## Best Possible Solution

Keep the PR open as a valid gateway-service behavior gap, but require a new current-main implementation before merge. The fix should live around `src/cli/gateway-cli/run.ts` service-marker cleanup and should use an in-process Node probe or existing gateway reachability helper, respect effective bind host/auth constraints, return early when an already-running healthy gateway is intentionally left alone, preserve explicit restart cleanup semantics, and add focused tests for healthy loopback and non-loopback/custom bind cases.
