## Summary

Keep #20321 open. Current main still does not implement Telegram `groupPolicy: "members"`; the shared config contract, Telegram schema, runtime authorization, docs, and tests only support `open`, `disabled`, and `allowlist`. The prior closed PRs and the `openclaw-chat-only-trigger` plugin are useful related workarounds/prior art, but they do not ship the requested bundled Telegram all-members allowlist policy.

## Best Possible Solution

Keep #20321 open for explicit bundled Telegram plugin and shared config-contract work. The likely path is to add `groupPolicy: "members"` to the exported type/schema/docs, define fail-closed all-member verification semantics, implement Telegram Bot API membership/count checks with caching and join/leave invalidation in the Telegram plugin, update security/doctor guidance, and add focused runtime/schema tests. Treat the closed PRs and `openclaw-chat-only-trigger` as useful prior art, not as a close basis.
