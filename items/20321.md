## Summary

Current main still does not implement Telegram `groupPolicy: "members"`. The linked plugin workaround and closed PRs are useful prior art, but the public config contract, strict schema, generated metadata, Telegram runtime, docs, and tests still model only `open`, `disabled`, and `allowlist`; the remaining requested bundled Telegram behavior should stay open.

## Best Possible Solution

Keep #20321 open for explicit bundled Telegram plugin/config-contract work. A complete fix should add `members` to exported config types, schema, generated metadata/help, docs, and examples; implement fail-closed Telegram all-members authorization in inbound and native-command paths with bounded Bot API use, caching, and membership-change invalidation; update security/audit behavior; and add focused config, runtime, docs, and Telegram tests. Treat #12045, #28351, #28392, #33551, and `openclaw-chat-only-trigger` as prior art rather than closure evidence.
