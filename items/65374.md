## Summary

Keep this issue open. Current main has adjacent dreaming fixes, including isolated managed cron execution, source tags in session-corpus lines, and separate-workspace fan-out, but the narrowed shared-workspace isolation bug still exists: dreaming remains scoped to workspace directories, and session ingestion intentionally gathers every configured agent attached to the same workspace into the same dreaming corpus and recall signal.

## Best Possible Solution

Keep this open as a narrowed memory-core bug. The best path is to decide whether dreaming is agent-scoped or workspace-scoped when agents share a workspace; if agent-scoped, carry the active agent/session target into dreaming sweeps, store/read session corpus and recall signals with an agent scope, filter `__dreaming_sessions__` ranking by that scope, and add regression coverage with two active agents on the same workspace proving Agent B's dream cannot ingest Agent A's transcript. If workspace-level pooling is intended, document that behavior and provide a config escape hatch before closing.
