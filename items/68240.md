## Summary

Keep open. Current `main` does not implement the proposed `openclaw onboard --daemon-env` flag, and the existing code/docs show only the broader durable env paths (`~/.openclaw/.env` and config `env`) rather than this CLI surface. The PR also appears to merge arbitrary env entries after `buildGatewayInstallPlan()`, which would bypass the current service-env filtering contract and needs maintainer review rather than auto-close cleanup.
