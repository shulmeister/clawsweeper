## Summary

Keep PR #68240 open. Current main still does not expose `openclaw onboard --install-daemon --daemon-env`, and the onboard daemon install paths pass only the existing `buildGatewayInstallPlan()` environment to `service.install()`. The requested CLI capability remains real, but the PR should not merge as-is without validating/denying unsafe env keys before merging extra values into the daemon environment.

## Best Possible Solution

Keep the PR or its linked issue open, but require a revised implementation that parses repeatable `--daemon-env KEY=VALUE` in a shared helper, rejects malformed/empty keys, normalizes keys, blocks dangerous keys using the existing host-env security checks before any merge, covers both non-interactive and wizard install paths, and adds focused CLI/install-plan tests plus docs for the supported daemon env injection path.
