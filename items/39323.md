## Summary

Keep PR #39323 open. Current main still lacks owner notifications for new DM pairing requests, lacks a `pairing:request` hook/event, and has no `pairing.notify` config surface. The closed related issue #38998 does not supersede the PR because the requested capability remains missing on main.

## Best Possible Solution

Keep the PR open as the active implementation candidate, or replace it with a maintainer-owned narrower design. The best path is to add a documented, versioned `pairing:request` event or plugin hook emitted only for newly created DM pairing requests, then implement owner notification through a plugin or minimal core config once that event exists. Before merge, rebase and resolve the remaining review concerns around Discord target normalization, sanitization, PII logging, docs, and focused tests.
