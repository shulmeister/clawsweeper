## Summary

Keep PR #39323 open. Current main still lacks owner notifications for new DM pairing requests, lacks the proposed pairing hook surface, and only supports requester-side approval notifications. The closed related issue #38998 does not supersede the PR; this PR is still the concrete implementation vehicle for that remaining capability.

## Best Possible Solution

Keep the PR open as the active implementation candidate or replace it with a maintainer-owned narrower design: add a documented, versioned `pairing:request` hook or equivalent core event for DM pairing requests, then implement owner notification either in core with clear config/docs or through a plugin once the missing event API exists. Address the remaining Discord target normalization concern before merge.
