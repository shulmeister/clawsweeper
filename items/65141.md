## Summary

Keep open. Current `main` still starts one iMessage gateway task per configured account key, and the iMessage plugin still exposes both `accounts.default` and named accounts as separate startup targets. Because each monitor falls back to the same `imsg`/`dbPath` backend when no distinct runtime config is present, the duplicate-watcher report remains credible. The linked PR #65194 is still open and its patch does not land on the current startup path.
