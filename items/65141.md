## Summary

Keep this issue open. The provided GitHub context shows an open, unmerged same-author PR (#65194) with closing syntax for this issue, and current main still appears to start one iMessage runtime per configured account id returned by the plugin, including both `default` and a named account in the reported shape.

## Best Possible Solution

Keep the issue open and review/land or close PR #65194.
