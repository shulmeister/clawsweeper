## Summary

Keep #39231 open. Current main does not implement the proposed opt-in CaMeL config, P-LLM/Q-LLM orchestration, taint-tracking value system, side-effect policy enforcement, gateway approval source, tests, or docs. The linked RFC #39160 is closed and #47492 tracks a distinct verified-human provenance layer, so neither supersedes this PR’s core structural prompt-injection defense request.

## Best Possible Solution

Keep #39231 open for explicit maintainer/security product judgment. If OpenClaw wants this direction, the best path is a redesigned core implementation with accepted config/schema, isolated planner and quarantined extraction sessions, provenance-preserving taint through tool results and hook rewrites, replay-safe side-effect handling, gateway approval UX, docs, and focused regression tests. If maintainers choose a different architecture, close only after linking a canonical replacement for the remaining core work.
