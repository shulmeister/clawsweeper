## Summary

Keep open. Current `main` still snapshots `~/.openclaw/.env` into the gateway service environment, and the LaunchAgent installer still writes that environment into plist `EnvironmentVariables`. Because runtime `.env` loading skips keys already present in `process.env`, the stale-value behavior described in #53387 is still consistent with the code on `main`.
