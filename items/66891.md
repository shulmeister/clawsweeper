## Summary

Keep open. Current main already moved Discord reply delivery onto the shared outbound adapter and preserves basic `interactive` / `channelData.discord.components` replies, but this PR is not fully obsolete: the review-backed forum fallback and bound-thread webhook/component behavior still appear missing from current main, and the PR discussion contains useful cases that should be ported or split into focused follow-ups.

## Best Possible Solution

Keep the PR open for maintainer handling. The best path is to port any still-valid behavior into the current shared outbound adapter: add a component-send fallback for forum/media channels in `extensions/discord/src/outbound-payload.ts`, preserve text normalization parity for component text where needed, resolve or explicitly scope webhook persona behavior for bound-thread component replies, and cover those cases in `extensions/discord/src/outbound-adapter.test.ts` or a focused outbound component test.
