## Summary

Keep open. Current main still matches the prior ClawSweeper keep-open conclusion: same-session agent turns are serialized through the session lane, lane wait data remains diagnostic-only, the gateway accepts the request before the queued run starts, and `openclaw agent` waits for a final gateway result without a typed session-lane busy or wait-timeout error to print.

## Best Possible Solution

Keep this issue open and fix the gateway/CLI contract. A maintainer-quality fix should add a typed session-lane wait/busy result or error from the queue/agent dispatch path, propagate `lane`, `queueAhead`, and `waitedMs` through the gateway final/error frame, make `openclaw agent` print an actionable message, and add focused regression coverage for a blocked same-session CLI dispatch. If queueing should remain the default, add and document an explicit queue policy option rather than silently timing out or falling back.
