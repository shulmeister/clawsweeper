## Summary

Keep this PR open. Current main still has the QMD status-probe timeout bug: `probeVectorAvailability()` caps `qmd status` at 5 seconds instead of honoring `memory.qmd.limits.timeoutMs`. The checkpoint-export part is now partly superseded by current session artifact handling, and the PR’s broad `.checkpoint.` substring filter matches the unresolved review concern, so the PR should be revised rather than closed.

## Best Possible Solution

Revise this PR instead of closing it: keep the `probeVectorAvailability()` change to pass `this.qmd.limits.timeoutMs` directly and add the focused timeout regression test; drop the export-side `.checkpoint.` substring filter because current main already excludes exact compaction checkpoint transcripts through `listSessionFilesForAgent()`, or replace it with `isCompactionCheckpointTranscriptFileName()` only if maintainers want an extra export-layer guard.
