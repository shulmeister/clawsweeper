## Summary

Keep open. Current main still has the QMD status timeout bug this PR fixes, so the PR is not obsolete. The checkpoint-export hunk is the part that needs revision: current main already excludes real compaction checkpoint transcripts through shared session artifact filtering, and the PR's broad `.checkpoint.` substring filter has a valid regression risk for ordinary session IDs.

## Best Possible Solution

Keep the PR open and ask for a narrow revision: retain the `probeVectorAvailability()` timeout change and its regression test, but drop the QMD export substring filter because current main already excludes real checkpoint transcripts through `listSessionFilesForAgent()` and `isUsageCountedSessionTranscriptFileName()`. If maintainers still want explicit QMD-layer defense, use the existing exact compaction checkpoint helper rather than `basename.includes(".checkpoint.")`.
