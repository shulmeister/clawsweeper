## Summary

Keep #68248 open. Current main has substantial avatar identity plumbing that addresses part of the request, including IDENTITY.md/config avatar resolution, authenticated Control UI avatar routes, chat avatar rendering, and tests. The remaining requested Control UI product behavior is still not fully implemented: favicon customization is still static, chat/agent selectors render text-only native selects rather than avatars next to agent/session names, and the quick settings UI intentionally exposes user avatar controls without a display-name field.

## Best Possible Solution

Keep the issue open, but narrow implementation to the missing Control UI surfaces: reuse the existing agent.identity.get and /avatar/<agentId> plumbing to add avatar-aware agent/session selector UI, update the browser tab favicon when the active agent/session identity changes, and expose a deliberate display-name override path in settings or config. Avoid rebuilding the already-shipped avatar backend.
