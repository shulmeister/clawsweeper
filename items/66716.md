## Summary

Keep open. Current main still has the fresh-token early return in `runPreflightCompactionIfNeeded`, so sessions with `totalTokensFresh: true` and a persisted total return before `shouldRunPreflightCompaction` can evaluate the threshold. The PR is a plausible focused fix, and the follow-up commit addressed the reviewer’s legacy `totalTokensFresh: undefined` concern; related PRs are not merged or clearly superseding this one.

## Best Possible Solution

Keep the PR open for maintainer review or consolidation with the related preflight-compaction PRs. The best fix is to update current main so fresh persisted totals are checked against the preflight threshold, stale or legacy-unknown totals fall back to transcript estimation, and focused regression tests cover fresh-above-threshold, fresh-below-threshold, stale fallback, legacy undefined freshness, heartbeat, and CLI bypass behavior.
