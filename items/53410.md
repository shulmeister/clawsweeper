## Summary

Keep open. Current `main` still uses prerelease-only semver comparison for update checks, so numeric `-N` suffixes are not treated as OpenClaw correction revisions. The PR’s behavior is not implemented on `main`.

## Evidence

- **`compareSemverStrings` is still wired to the generic semver helper:** `src/infra/update-check.ts:371` still delegates to `parseComparableSemver(..., { normalizeLegacyDotBeta: true })` and `compareComparableSemver(...)`, not to the revision-aware `compareOpenClawVersions()` logic in `src/config/version.ts`.
