## Summary

Current `main` already supersedes this PR with a supported gateway auth-recovery flow that turns OAuth refresh failures into user-facing re-auth guidance using the documented `openclaw models auth login` path. The remaining `/connect_ai_subscription` command in this PR is both directionally out of step with OpenClaw’s terminal-first setup model and called out by prior review comments for concrete correctness issues.
