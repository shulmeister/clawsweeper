## Summary

Keep #39635 open. Current main still lacks `agents.defaults.extraBootstrapFiles` or a per-agent equivalent, and the bundled `bootstrap-extra-files` hook still only loads recognized bootstrap basenames, so arbitrary files such as `CONTEXT_BRIEF.md` are not supported through the requested first-class config path.

## Best Possible Solution

Keep #39635 open as the canonical tracker for first-class custom bootstrap file config. A complete fix should add a strict schema/type field for `agents.defaults` plus per-agent override semantics, safely resolve workspace-relative custom files, inject them after standard bootstrap files in order, apply existing per-file and total bootstrap budgets, document the config, update generated schema/help as needed, and add focused tests for missing files, invalid paths, ordering, custom basenames, and default-vs-agent override behavior.
