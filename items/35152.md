## Summary

Keep #35152 open. Current main still does not implement an on-demand `openclaw doctor` duplicate-installation diagnostic; it has source-workspace install hygiene, gateway service drift/extra-service checks, split-brain guardrails, and an installer-time duplicate global-install warning, but no doctor binary/install-root inventory with versions and remediation as proposed by the PR.

## Best Possible Solution

Keep #35152 open until it is refreshed or superseded by a current doctor-health implementation that safely detects duplicate OpenClaw binaries/install roots, deduplicates by realpath, reports versions and relevant service metadata, respects the current `doctor --deep` service-scan boundary, uses safe process APIs, derives npm package/bin locations correctly, and includes focused regression tests.
