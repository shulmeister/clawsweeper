## Summary

Current main still does not implement this PR's requested duplicate-installation doctor check. Main has source-install hygiene notes, extra gateway-service scans, and a runtime mitigation for gateway lock conflicts, but no doctor path that scans multiple openclaw binaries, deduplicates by realpath, resolves versions, or reports duplicate npm/system installs. The PR is stale and needs rebase/review, but it is still salvageable unlanded behavior rather than closeable maintainer cleanup.

## Best Possible Solution

Keep this PR open until a maintainer either refreshes it against the current doctor-health contribution flow or explicitly decides the duplicate-installation detector is no longer desired. A refreshed implementation should add a focused doctor contribution for duplicate OpenClaw binary installs, use shell-free execution for version probing, include systemd/launchd service context without false positives, and cover the PR review findings with tests.
