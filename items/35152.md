## Summary

Keep open. Current main has adjacent mitigations, including an installer-time duplicate global install warning and gateway service entrypoint/config guards, but it still does not implement this PR's `openclaw doctor` duplicate-installation check. The PR remains useful but needs a refresh before merge.

## Best Possible Solution

Keep #35152 open until a maintainer refreshes or replaces it with a current doctor health contribution that safely scans duplicate OpenClaw installs, reports versions and relevant service metadata, respects existing `--deep` service-scan semantics, and includes focused tests. If maintainers decide the installer warning plus gateway service hardening is sufficient, close it explicitly as a product decision rather than as already implemented.
