## Summary

Keep #35152 open. Current main has adjacent installer-time duplicate global npm install warnings and doctor gateway-service drift checks, but it still does not implement this PR's on-demand `openclaw doctor` duplicate binary/version/systemd diagnostic. The PR remains a plausible contribution, but it needs a refresh against the current doctor health flow and the prior review concerns.

## Best Possible Solution

Keep #35152 open until it is refreshed or replaced with a current doctor health contribution that safely detects duplicate OpenClaw installs, reports versions and relevant service metadata, respects current `--deep` service-scan semantics, and includes focused tests. A refreshed implementation should use safe process APIs such as `execFile`/`execFileSync`, derive npm bin/package locations correctly, avoid false positives through realpath dedupe, and integrate with the existing gateway service drift checks. If maintainers decide the installer warning plus service-entrypoint repair is sufficient, close it explicitly as a product decision.
