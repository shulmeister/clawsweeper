## Summary

Keep open. Current main has improved sandbox infrastructure and plugin-backed sandbox backends, but it does not implement the requested Sysbox/sysbox-runc architecture, socketless Docker-in-Docker gateway mode, or a docker runtime option. The March comment that #10361/#29974 only covered sandbox-common/DooD remains accurate for the Sysbox-specific scope.

## Evidence

- **Docker sandbox config has no runtime field:** The strict SandboxDockerSchema lists image, containerPrefix, workdir, readOnlyRoot, tmpfs, network, user, capDrop, env, setupCommand, limits, seccomp/AppArmor, DNS/hosts, binds, and dangerous overrides; there is no sandbox.docker.runtime/sysbox-runc setting accepted here.
