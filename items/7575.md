## Summary

Keep #7575 open. Current main has Docker/SSH/OpenShell sandbox backends and a public sandbox backend registration seam, but it still does not ship Sysbox/sysbox-runc, socketless Docker-in-Docker gateway isolation, or a Docker runtime selector. The latest maintainer discussion also explicitly keeps this issue open for that distinction.

## Best Possible Solution

Keep #7575 open as the canonical tracker for Sysbox/sysbox-runc socketless sandbox architecture until a maintainer decides the product direction. A good path is either a conservative Docker runtime option with config schema, types, docs, tests, and host setup guidance, or an endorsed Sysbox sandbox backend plugin if the current sandbox plugin SDK is sufficient.
