## Summary

Keep #7575 open. Current main still lacks Sysbox/sysbox-runc support and lacks a supported `agents.defaults.sandbox.docker.runtime` selector; the maintained Docker sandbox path remains Docker-socket based, while the plugin-backed sandbox backend seam could support a future Sysbox backend but does not itself resolve this issue. The latest provided issue discussion also explicitly keeps this as the canonical Sysbox/runtime-architecture tracker requiring maintainer judgment.

## Best Possible Solution

Keep #7575 open as the canonical Sysbox/runtime-architecture tracker. A good resolution would either add a conservative Docker/OCI runtime selector with schema, types, Docker create wiring, docs, and tests, or explicitly endorse and document a Sysbox sandbox backend plugin built on `registerSandboxBackend` if maintainers decide the capability belongs outside core.
