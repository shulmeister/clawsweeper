## Summary

Keep #7575 open. Current main still does not ship Sysbox/sysbox-runc integration, socketless Docker-in-Docker gateway isolation, or a `sandbox.docker.runtime` selector. Existing sandbox support is Docker/SSH/OpenShell, and the Docker path remains socket-based; the latest issue discussion also explicitly keeps this as the canonical Sysbox/runtime-architecture tracker.

## Best Possible Solution

Keep #7575 open as the canonical tracker until maintainers choose the product direction. A good implementation would either add a conservative `sandbox.docker.runtime` config path with schema, docs, tests, and host setup guidance, or ship/endorse a Sysbox sandbox backend plugin if the current plugin SDK is sufficient.
