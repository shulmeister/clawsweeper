## Summary

Keep #7575 open. Current main still does not implement Sysbox/sysbox-runc or `agents.defaults.sandbox.docker.runtime`, and the issue discussion explicitly keeps this as the canonical Sysbox/runtime-architecture tracker rather than treating Docker socket sandboxing or the sandbox backend plugin seam as completion.

## Best Possible Solution

Keep #7575 open as the canonical Sysbox/runtime-architecture tracker. A good resolution is either a conservative built-in Docker OCI runtime selector with types, strict schema, config docs, Docker create `--runtime` wiring, and tests, or an explicit maintainer decision to move Sysbox to a plugin-backed sandbox backend with public docs/ClawHub tracking and any missing SDK seams filed separately.

## Risks / Open Questions

- Closing as implemented would conflate current Docker socket sandbox support with the requested socketless Sysbox/sysbox-runc architecture.
- Closing as clawhub would be premature without an already-posted Sysbox plugin/ClawHub item and would contradict the current issue discussion that keeps #7575 canonical.
- A plugin-backed Sysbox backend may be the eventual best direction, but maintainers still need to decide whether to add a Docker runtime selector, document a Sysbox plugin path, or add any missing SDK support.
