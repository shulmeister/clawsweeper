## Summary

Keep open. Current `main` still uses a single global agent lane for normal runs (`main`) with one shared `agents.defaults.maxConcurrent` cap, and there is no documented or schema-backed per-agent lane setting such as `agents.defaults.lane`, `agents.list[].lane`, or `agents.lanes`.

## Evidence

- **Docs still describe one shared global agent lane:** The queue docs say each session run is queued into a global `main` lane and overall parallelism is capped by `agents.defaults.maxConcurrent`. The agent config docs also document only one `maxConcurrent` knob for parallel agent runs across sessions.
