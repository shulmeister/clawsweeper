## Summary

Keep PR #69399 open. Current main still gates user-visible model fallback and fallback-cleared notices behind verbose mode, has no `agents.defaults.notifyOnModelFallback` config/type/schema/doc surface, and tests explicitly assert verbose-off replies do not include those notices. The PR is a focused implementation for open issue #67197, and the provided PR context shows it also addressed the bot's requested once-per-state and fallback-cleared coverage in its second commit.

## Best Possible Solution

Keep the PR open for maintainer review. If the product direction is accepted, land this narrow config-backed implementation or an equivalent one that adds `agents.defaults.notifyOnModelFallback`, wires it into the fallback and fallback-cleared notice guards, and keeps the generated schema/help/docs plus e2e coverage aligned.
