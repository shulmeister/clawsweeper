## Summary

Keep PR #39412 open. Current main already covers the daemon gateway-token persistence goal with the newer SecretRef-aware install flow, but the provider `*_FILE` env-var behavior proposed by the PR is still absent from current main. The branch should not be merged as-is because its gateway-token changes overlap newer code and the PR discussion includes unresolved P1/P2 review findings.

## Best Possible Solution

Keep the PR open as useful stale work. Rebase or rewrite it against current main, discard the obsolete gateway-token persistence edits, and implement only the still-missing provider `*_FILE` env-var convention if maintainers want Docker/Kubernetes-style ambient file env support. The implementation should integrate with the current provider auth split (`model-auth-env.ts`, `live-auth-keys.ts`, provider usage paths), preserve list separators for `*_API_KEYS_FILE`, report file-read misconfiguration clearly, and add focused tests for direct env precedence, missing/unreadable files, single-key vars, and multi-key list vars.
