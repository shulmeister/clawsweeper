## Summary

Keep PR #39412 open. Current main already covers the daemon gateway-token persistence half through the newer SecretRef-aware install flow, but the provider `*_FILE` environment variable behavior proposed by this PR is still absent from main. The branch also has unresolved review comments that make a straight merge unsafe, so the useful path is a targeted rebase/rewrite rather than closure.

## Best Possible Solution

Keep the PR open as useful but stale work. Rebase or rewrite it against current main, preserve only the still-missing provider `*_FILE` behavior if maintainers still want that env convention, and adapt it to the current SecretRef/gateway install architecture. The gateway-token persistence portion should not be reintroduced wholesale because main already has the safer SecretRef-aware flow and tests.
