## Summary

Keep open. Current main still does not expose an opt-in that lets `web_fetch` reach localhost/LAN/private-network targets. The code only exposes and forwards the narrower `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange` fake-IP exception, while tests continue to assert localhost, private IP literals, and private DNS answers are blocked. The related implementation PR #39630 is still open in the provided context, so the issue remains actionable rather than implemented or stale.

## Best Possible Solution

Keep the issue open until a maintainer lands a web_fetch-specific private-network opt-in or explicitly rejects the product direction. The likely implementation belongs in `src/config/zod-schema.agent-runtime.ts`, `src/config/types.tools.ts`, web_fetch schema/help/docs, and `src/agents/tools/web-fetch.ts`, with tests covering default blocking and explicit opt-in for localhost/private DNS. PR #39630 is the current related implementation attempt to evaluate or supersede.
