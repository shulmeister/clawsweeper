## Summary

Keep open. Current `main` still has the approval-routing gap described by the PR, but the code moved since the PR was opened. On `main`, `hasExecApprovalClients()` lives in `src/gateway/server-request-context.ts` and only recognizes connected clients by retained operator scopes. The plain-HTTP/browser-container path still allows a shared-auth operator to connect without device identity while clearing its scopes first, so the underlying `no-approval-route` failure remains plausible and is not already implemented on `main`.
