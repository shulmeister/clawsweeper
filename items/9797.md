## Summary

Keep #9797 open. Current main at 82262387651ab270b565f71533ce4937c9c00dc6 still lacks the requested agent-facing `queue_status` tool or equivalent stable queue snapshot surface for intelligent task dispatch. Existing queue metrics/logs and `/status` queue details are not a per-lane/per-agent snapshot consumable by agents, and related PR #43618 is explicitly scoped to operator diagnostics rather than an in-agent tool.

## Best Possible Solution

Implement one shared read-only queue snapshot provider in core, then expose it through an agent-facing `queue_status` tool with catalog/profile/group-policy registration, permissions, docs, and focused tests. If operator diagnostics lands first, reuse that backend and keep this issue focused on the in-agent dispatch surface.
