## Summary

Keep #9797 open. Current main still lacks the requested agent-facing `queue_status` tool and still lacks a stable queue snapshot RPC/CLI surface for per-lane queue depth, active runs, max concurrency, and per-agent workload. The related #43618 is relevant but remains a separate open PR in the provided context and explicitly does not add the in-agent tool requested here.

## Best Possible Solution

Keep this issue open and implement a shared read-only queue snapshot provider in core. Expose it through the requested agent-facing `queue_status` tool, and preferably through the operator CLI/RPC surfaces discussed in the comments, with tool catalog, permissions, docs, and tests updated together. If #43618 becomes the canonical snapshot backend, use #9797 to track the remaining agent-tool consumer until it ships.
