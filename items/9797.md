## Summary

Keep #9797 open. Current main still lacks an agent-facing `queue_status` tool and does not expose a stable per-lane queue snapshot RPC/CLI backend. The issue discussion also contains a same-day maintainer/Codex comment explicitly keeping it open, and related PR #43618 is only an open operator-diagnostics proposal that says it does not add the in-agent tool.

## Best Possible Solution

Implement a shared read-only queue snapshot provider in core, then expose it through an agent-facing `queue_status` tool with tool catalog/profile/policy registration, permissions, docs, and focused tests. If PR #43618 or equivalent operator diagnostics lands first, reuse its snapshot provider and let #9797 track the remaining in-agent consumer.
