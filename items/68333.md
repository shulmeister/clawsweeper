## Summary

Current `main` already supports the reported opt-out: `channels.telegram.commands.native: false` disables native Telegram menu sync, clears built-in commands, and allows `customCommands` to register by themselves. The same behavior is already present in the released `v2026.4.23` tree.

## Evidence

- **Telegram menu builder respects `commands.native: false`:** `registerTelegramNativeCommands` only populates `nativeCommands` when `nativeEnabled` is true; otherwise it builds the menu from custom/plugin commands without native entries.
  - file: [extensions/telegram/src/bot-native-commands.ts:505](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/extensions/telegram/src/bot-native-commands.ts#L505)
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Explicit disable clears built-in Telegram commands:** When native commands are explicitly disabled, startup calls `bot.api.setMyCommands([])` to remove the built-in Telegram menu.
  - file: [extensions/telegram/src/bot-native-commands.ts:1168](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/extensions/telegram/src/bot-native-commands.ts#L1168)
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Regression test covers custom-only registration:** Test verifies that with `commands: { native: false }`, Telegram registers only the configured `customCommands` and no reserved native commands.
  - file: [extensions/telegram/src/bot.command-menu.test.ts:177](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/extensions/telegram/src/bot.command-menu.test.ts#L177)
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Regression test covers clearing native menu on startup:** `createTelegramBot` test asserts `setMyCommands([])` is called when native commands are disabled.
  - file: [extensions/telegram/src/bot.create-telegram-bot.test.ts:2156](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/extensions/telegram/src/bot.create-telegram-bot.test.ts#L2156)
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Config/docs now advertise the opt-out:** Schema accepts boolean `commands.native`; Telegram UI hints and channel docs explicitly document disabling native commands and note that custom/plugin commands may still register.
  - file: [src/config/zod-schema.core.ts:839](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/config/zod-schema.core.ts#L839)
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Latest release already contains the same behavior:** Diffing `v2026.4.23` (`a9797214338ba31c52c796adbb75afb16e0684a9`) against current `main` for the Telegram command-menu files shows only non-behavior changes on this path, so the released tree already includes the opt-out behavior.
  - command: `git diff --unified=0 a9797214338ba31c52c796adbb75afb16e0684a9..c2139635ff82b367b734ea052487258b1b33f060 -- extensions/telegram/src/bot-native-commands.ts extensions/telegram/src/bot.command-menu.test.ts`
