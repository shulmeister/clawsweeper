## Summary

Current main already contains Control UI and gateway dedupe guards that address the reported dashboard duplicate-message / repeated-send regression, and those guards are covered by focused tests.

## Evidence

- **Control UI defers transcript reloads during active runs:** `handleSessionMessageGatewayEvent` skips `loadChatHistory` while `chatRunId` is active and defers the reload instead of reloading history mid-run, which is the exact UI race that would duplicate optimistic chat state.
  - file: [ui/src/ui/app-gateway.ts:490](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/ui/src/ui/app-gateway.ts#L490)
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Regression tests cover the active-run suppression path:** The UI test asserts that `session.message` does not reload history while a chat run is active.
  - file: [ui/src/ui/app-gateway.sessions.node.test.ts:146](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/ui/src/ui/app-gateway.sessions.node.test.ts#L146)
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Regression tests cover the deferred single reload after terminal events:** The UI test asserts deferred `session.message` reloads replay once after an `aborted` or `error` terminal chat event clears the active run.
  - file: [ui/src/ui/app-gateway.node.test.ts:725](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/ui/src/ui/app-gateway.node.test.ts#L725)
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Browser send path is single-shot and idempotent:** `sendChatMessage` appends one optimistic user message and makes one `chat.send` request with a generated `runId` used as `idempotencyKey`; it does not fan out repeated sends on its own.
  - file: [ui/src/ui/controllers/chat.ts:325](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/ui/src/ui/controllers/chat.ts#L325)
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Gateway rejects duplicate or repeated in-flight chat sends:** The gateway returns cached or `in_flight` responses for repeated `chat.send` calls with the same idempotency key, and transcript injection also short-circuits when that key already exists.
  - file: [src/gateway/server-methods/chat.ts:2185](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/gateway/server-methods/chat.ts#L2185)
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
