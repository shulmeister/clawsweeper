## Summary

Keep #40536 open. Current main still has no first-class `skills.entries.<skill>.model` override, and the existing skill flow is prompt/catalog driven after normal model selection rather than a core preselected skill invocation that can temporarily switch models. Related model-override work covers other surfaces such as cron, compaction, sessions, channels, and plugin hooks, but not per-skill model routing.

## Best Possible Solution

Keep the issue open for maintainer/product design. A real fix would need a first-class `skills.entries.<skillKey>.model` contract in config types, zod schema, docs, UI/API surfaces, and tests, plus a runtime design for when the override applies. Because current skills are selected by the active model from the prompt catalog, OpenClaw likely needs either a pre-model skill router or a documented skill-command/sub-call path that can resolve the skill first, run that work on the configured model, and restore the normal session/default model afterward.
