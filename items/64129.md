## Summary

Keep open. Current main has adjacent mitigations for empty/default heartbeat scaffolds and optional cost controls, but the specific reported path still exists for existing actionable HEARTBEAT.md files: heartbeat runs still inherit the normal default model unless an explicit heartbeat model override is configured, and Google onboarding can still make Gemini the primary model.

## Best Possible Solution

Add explicit protection at the provider-auth/default-model and heartbeat boundary: preserve an existing default unless the user confirms, require confirmation before heartbeat inherits a newly configured paid provider, or make heartbeat use an explicit safe override/disabled state for paid-provider transitions. Keep the existing empty-template skip and isolated-session/light-context controls as complementary mitigations, not the full fix.
