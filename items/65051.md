## Summary

Keep open. Current main still contains the bundled-first grouping logic the PR is trying to fix, the UI already treats `bundled` as a secondary badge when `source` says otherwise, and existing UI coverage would not catch the reported workspace-skill regression.

## Evidence

- **Grouping still prefers `bundled` over `source`:** `groupSkills` sends any `skill.bundled` entry to the built-in group before checking `skill.source`, so a `source: "openclaw-workspace"` skill with `bundled: true` would still be grouped incorrectly on current main.
  - file: [ui/src/ui/views/skills-grouping.ts:24](https://github.com/openclaw/openclaw/blob/c018e7347542aa963095de1ca704b943476856fe/ui/src/ui/views/skills-grouping.ts#L24)
  - command: `nl -ba ui/src/ui/views/skills-grouping.ts | sed -n '1,40p'`
  - sha: [c018e7347542](https://github.com/openclaw/openclaw/commit/c018e7347542aa963095de1ca704b943476856fe)
- **UI already models `bundled` as a secondary badge:** The detail view only shows the bundled badge when `skill.bundled` is true and `skill.source !== "openclaw-bundled"`, which matches the PR's premise that source and bundled can legitimately diverge.
  - file: [ui/src/ui/views/skills.ts:423](https://github.com/openclaw/openclaw/blob/c018e7347542aa963095de1ca704b943476856fe/ui/src/ui/views/skills.ts#L423)
  - command: `nl -ba ui/src/ui/views/skills.ts | sed -n '418,432p'`
  - sha: [c018e7347542](https://github.com/openclaw/openclaw/commit/c018e7347542aa963095de1ca704b943476856fe)
- **Gateway status payload carries both fields:** `SkillStatusEntry` exposes both `source` and `bundled`, so the grouping helper is the place that resolves conflicting combinations for the Control Panel.
  - file: [src/agents/skills-status.ts:31](https://github.com/openclaw/openclaw/blob/c018e7347542aa963095de1ca704b943476856fe/src/agents/skills-status.ts#L31)
  - command: `nl -ba src/agents/skills-status.ts | sed -n '26,45p'`
  - sha: [c018e7347542](https://github.com/openclaw/openclaw/commit/c018e7347542aa963095de1ca704b943476856fe)
- **Current UI test fixture is not aligned to canonical skill sources:** `skills.test.ts` builds its default fixture with `source: "workspace"` instead of `openclaw-workspace`, and the test only exercises dialog/ClawHub behavior, so it would not catch the grouping bug or the mixed workspace/built-in case described in the PR.
  - file: [ui/src/ui/views/skills.test.ts:18](https://github.com/openclaw/openclaw/blob/c018e7347542aa963095de1ca704b943476856fe/ui/src/ui/views/skills.test.ts#L18)
