## Summary

Current main already fixes the linked workspace-skill grouping bug at the status payload layer. Workspace skill entries are loaded as `openclaw-workspace`, workspace wins name-collision precedence, and `buildWorkspaceSkillStatus` now only marks actual bundled or legacy-unknown bundled-name entries as `bundled`. The Control Panel already maps `openclaw-workspace` to Workspace Skills, so PR #65051 is now a defensive UI fallback for a mixed payload shape that main no longer emits.

## Best Possible Solution

Close PR #65051 as implemented on main. Keep the shipped status-layer fix and regression coverage from v2026.4.22; if maintainers want extra defensive UI hardening for malformed or legacy status payloads, that should be a narrow follow-up rather than this now-redundant bug-fix PR.
