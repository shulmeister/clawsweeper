## Summary

Keep #69732 open. Current main still exposes `source` on exec approval snapshots while `exec.approvals.set` validates allowlist entries with a schema that rejects `source`. The PR is recent, linked to open bug #69698, and the latest PR discussion moved away from stripping `source` because durable approvals depend on it. This needs maintainer review or a revised fix, not cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review and ask for a revised narrow fix for #69698: make `exec.approvals.set` safe for GET→edit→SET round-trips without losing durable `allow-always` metadata. The likely path is to either accept/preserve `source` in the exec approvals protocol schema for both gateway and node set payloads, or rehydrate internal metadata from the current snapshot during save while keeping socket tokens redacted. Add a regression covering UI-style GET→save round-trips with durable allowlist entries.
