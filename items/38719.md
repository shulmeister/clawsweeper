## Summary

Current `main` already contains an explicit live-model-switch coordination path for mid-session `/model` changes, with regression coverage and a changelog entry showing the session/live-switch alignment fix shipped after the reported `2026.3.1` version.

## Evidence

- **`/model` persists a pending live switch instead of interrupting the active turn:** Directive handling applies the new model selection with `markLiveSwitchPending: true` and retargets queued followups, with an inline comment that `/model` should not interrupt the active run.
  - file: [src/auto-reply/reply/directive-handling.impl.ts:415](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/src/auto-reply/reply/directive-handling.impl.ts#L415)
- **Regression test: active-session `/model` does not request a live restart:** The test asserts that mutating an active session with `/model openai/gpt-4o` sets `liveModelSwitchPending` and does not call `requestLiveSessionModelSwitch`, matching the intended fix for mid-session switches.
  - file: [src/auto-reply/reply/directive-handling.model.test.ts:715](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/src/auto-reply/reply/directive-handling.model.test.ts#L715)
- **Runner reconciles pending live switches from persisted session state:** `shouldSwitchToLiveModel(...)` reads `liveModelSwitchPending`, resolves the persisted selected model, and clears stale flags when the current model already matches, replacing the older in-memory-only approach.
  - file: [src/agents/live-model-switch.ts:124](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/src/agents/live-model-switch.ts#L124)
- **Embedded runner restarts cleanly onto the requested model:** During an active attempt, the runner checks `shouldSwitchToLiveModel(...)`, clears the pending flag, and throws `LiveSessionModelSwitchError` to restart on the requested provider/model instead of continuing with conflicting state.
  - file: [src/agents/pi-embedded-runner/run.ts:913](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/src/agents/pi-embedded-runner/run.ts#L913)
- **Regression test and shipped release note:** There is a dedicated test that the active prompt restarts on a live model switch, and the changelog for `2026.4.7` says: `Sessions/model selection: resolve the explicitly selected session model separately from runtime fallback resolution so session status and live model switching stay aligned with the chosen model.`
  - file: [CHANGELOG.md:1104](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/CHANGELOG.md#L1104)

## Close Comment

Closing as implemented on current `main`. Codex review found that mid-session model changes now go through the persisted live-switch path instead of forcing the old conflicting state: `/model` writes `liveModelSwitchPending` without interrupting the active turn (`src/auto-reply/reply/directive-handling.impl.ts:415`), live-switch reconciliation reads the persisted selection and clears stale flags (`src/agents/live-model-switch.ts:124`), and the embedded runner restarts onto the requested model when that pending switch is detected (`src/agents/pi-embedded-runner/run.ts:913`). Regression coverage also now asserts that active-session `/model` changes do not request an immediate restart and that the active prompt can restart cleanly on the new model (`src/auto-reply/reply/directive-handling.model.test.ts:715`, `src/auto-reply/reply/agent-runner-execution.test.ts:1867`). The changelog entry for release `2026.4.7` explicitly calls out the session/live-model-switch alignment fix (`CHANGELOG.md:1104`). If this still reproduces on a current build, please open a new issue with updated repro steps and logs.
