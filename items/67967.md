## Summary

Keep open. Current main has not implemented the PR’s requested MiniMax Anthropic-compatible tool-call ID passthrough: MiniMax still uses the shared hybrid replay hooks directly, and current regression tests still expect `sanitizeToolCallIds: true` for `minimax` on `anthropic-messages`. The PR is a focused provider-plugin bug fix tied to an open canonical bug report (#65380) and should remain available for maintainer review/update rather than being closed by cleanup.

## Best Possible Solution

Keep the PR open and update it against current main. The best fix is still provider-owned: make MiniMax disable tool-call ID sanitization only for `modelApi === "anthropic-messages"`, while preserving strict sanitization for `openai-completions`. Add or update assertions in `extensions/minimax/index.test.ts` and preferably the transcript-policy coverage in `src/agents/transcript-policy.test.ts` so the runtime-resolved policy is locked to `sanitizeToolCallIds: false` for MiniMax Anthropic-compatible runs.
