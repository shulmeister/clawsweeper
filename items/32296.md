## Summary

Keep #32296 open. Current main has several adjacent hardenings around per-request chat.send fields, session queueing, orphaned active-turn text, and empty/reasoning provider responses, but I did not find source, tests, release notes, or issue discussion showing the broad cross-channel “agent replies to the previous message” symptom is fixed. The issue discussion also includes a recent 2026-04-18 report that it still reproduced on origin/main after the v2026.4.12 queue/transcript fix shipped.

Ask affected users for current-main session JSONL, gateway logs, channel/provider, queue mode, and transcript shape; then split fixed sub-variants such as #31573 webchat session routing, #65388 active-turn text repair, and provider empty-output fixes into narrower closed references while adding regression coverage around the shared reply/session-runner path for queued, steered, compacted, and fallback turns.
