## Summary

Keep open. Current main still contains the reported `totalTokensFresh` preflight short-circuit: `runPreflightCompactionIfNeeded()` returns before `shouldRunPreflightCompaction()` whenever a positive persisted total exists and `totalTokensFresh` is not false, so fresh cached totals still do not reach the threshold check. The linked fix PR #65622 is closed unmerged, while related PRs #64384 and #66716 remain open implementation candidates rather than shipped fixes.

## Best Possible Solution

Keep this issue open until a focused fix lands on main. The best code path is to restructure `runPreflightCompactionIfNeeded()` so fresh persisted totals are used for the threshold gate instead of causing an early return, while preserving transcript fallback for stale or missing totals. Add a regression test in `src/auto-reply/reply/agent-runner-memory.test.ts` for `totalTokensFresh: true` with cached tokens above threshold triggering `compactEmbeddedPiSession()`, then close this issue after the merged PR ships or explicitly mark it duplicate of a maintainer-chosen canonical issue such as #63892.
