## Summary

Keep open. Current `main` still short-circuits `runPreflightCompactionIfNeeded()` whenever a persisted token count is marked fresh, while normal session usage persistence keeps `totalTokensFresh` set to `true` after successful turns and session reset is the only obvious reset path. The latest release `v2026.4.23` matches current `main` on this code path, and the test suite does not show a landed regression test for the fresh-persisted preflight case.

## Evidence

- **Current preflight fast-path still returns early on fresh persisted totals:** `runPreflightCompactionIfNeeded()` computes `hasPersistedTotalTokens`, then returns immediately when `entry.totalTokensFresh !== false` and a persisted total exists. That bypasses `shouldRunPreflightCompaction()` for the exact state reported in the issue.
