## Summary

Keep #13744 open. Current main still has no public openclaw.json/env surface for session write-lock timeoutMs/staleMs, and the embedded attempt still acquires the session file lock before session setup/streaming and releases it only during final teardown. Main has partial error-handling improvements for SessionWriteLockTimeoutError, but the core configurability and lock-scope requests remain unresolved.

## Best Possible Solution

Keep #13744 open until OpenClaw adds a supported openclaw.json and/or OPENCLAW_* configuration surface for session write-lock timeoutMs and staleMs, wires it into embedded run and transcript mutation paths, and either narrows the lock critical section to actual transcript read/repair/prewarm/append/rewrite work or explicitly documents the broad per-session lifecycle lock as intentional. Preserve the existing SessionWriteLockTimeoutError failover suppression and add focused contention/config tests for concurrent same-session callers.
