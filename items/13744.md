## Summary

Keep #13744 open. Current main partially addresses the error-classification part with a dedicated session write-lock timeout error, but it still does not expose configurable session write-lock timeout/stale settings and the embedded run still holds the lock across the broad session lifecycle including model execution and cleanup.

## Best Possible Solution

Keep #13744 open until OpenClaw adds a supported configuration surface for session write-lock timeoutMs and staleMs, wires it into embedded run and transcript mutation paths, and either narrows the lock to actual transcript critical sections or documents the broad lifecycle lock as intentional. Preserve the current SessionWriteLockTimeoutError failover suppression and add focused contention/config coverage for concurrent same-session callers.
