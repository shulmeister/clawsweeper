## Summary

Keep open. Current `main` still holds the session transcript lock across the embedded agent run, still uses the same built-in 10s/30min lock defaults when callers do not override them, and does not expose session write-lock timeout/stale settings in the config schema. The requested configurability, narrower lock scope, and clearer lock-timeout handling are not implemented on `main`.
