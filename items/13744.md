## Summary

Keep #13744 open. Current main has the failover-classification part of the request, but the main lock behavior remains: session write lock timeout/stale values are not exposed through config/env, and the embedded attempt still acquires the file lock early and releases it only during final cleanup rather than narrowing it to transcript mutation critical sections.

## Best Possible Solution

Keep #13744 open until OpenClaw adds a supported config/env surface for session write lock timeoutMs and staleMs, and either narrows the embedded-run file-lock critical section to transcript mutation phases or explicitly documents broad per-session file locking as intentional product behavior. Preserve the existing SessionWriteLockTimeoutError/failover suppression behavior and add focused concurrency or cross-process lock contention tests for the final design.
