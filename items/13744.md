## Summary

Keep open. Current main partially addresses the error-classification part, but the core requested behavior is still missing: the embedded run still acquires the session write lock near the start of the attempt and releases it during end-of-attempt cleanup, and the session write lock defaults remain hardcoded rather than exposed via openclaw.json or env.
