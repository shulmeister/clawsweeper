## Summary

Issue #13744 should stay open. Current main includes the failover-classification fix from PR #68700, but the central requested work remains: session write lock timeout/stale values are still not exposed through config/env, and the embedded run still holds the session file lock broadly across the attempt rather than only around transcript write critical sections.

## Best Possible Solution

Keep #13744 open until OpenClaw adds a supported config/env surface for session write lock timeout/stale values and either narrows the embedded-run file-lock critical section or explicitly documents that broad per-session transcript locking is the intended product behavior. Preserve the PR #68700 failover-classification fix and add focused tests for same-session concurrency and cross-process lock contention.
