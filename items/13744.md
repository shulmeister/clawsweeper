## Summary

Keep #13744 open. Current main has fixed the model-failover misclassification part via #68700, and the in-process per-session queue reduces same-process run collisions, but the main requested work is still not implemented: session write lock timeout/stale defaults remain hardcoded, there is no public config/env surface for them, and the embedded attempt still holds the file lock from early session setup through end-of-attempt cleanup.

## Best Possible Solution

Keep tracking #13744 until OpenClaw either adds a supported config/env surface for session write lock timeout/stale values and narrows the embedded-run file-lock scope, or a maintainer explicitly decides that broad transcript locking is the intended product boundary and documents/configures the remaining contention behavior. The implementation should preserve the current per-session queue and #68700 failover-classification fix while adding targeted tests for concurrent same-session and cross-process lock contention.
