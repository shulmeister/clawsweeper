## Summary

Current main still does not implement a configurable WebChat Enter/send behavior. The browser composer hard-codes unmodified Enter to send, persisted UI settings and gateway webchat config/docs expose no send shortcut preference, and the native macOS composer still documents and implements fixed Return-to-send behavior. Related #8147 overlaps but remains open, and the latest maintainer discussion explicitly kept #19859 open pending possible consolidation.

## Best Possible Solution

Keep #19859 open unless maintainers explicitly consolidate it into #8147. A complete fix should add a typed chat send-shortcut/send-on-enter preference, wire it through the browser Control UI/WebChat composer and native Swift WebChat composer, support Ctrl/Cmd+Enter to send when plain Enter inserts a newline, and update config schema/types/help, settings UI, public docs, and focused composer tests.
