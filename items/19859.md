## Summary

Keep #19859 open. Current main still hardcodes WebChat/Control UI Enter-to-send behavior and does not expose the requested send-on-enter preference, Ctrl/Cmd+Enter send mode, config schema entry, UI setting, native composer behavior, or WebChat docs update.

## Best Possible Solution

Keep the issue open for a focused WebChat/Control UI enhancement unless maintainers explicitly consolidate it into #8147. A complete fix should decide the product surface, add a typed send-on-enter or send-shortcut preference, plumb it through the browser composer and native Swift composer, support Ctrl/Cmd+Enter to send when plain Enter inserts a newline, and update config schema/types/help, docs, UI copy, and focused composer tests.
