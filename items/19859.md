## Summary

Keep #19859 open. Current main still hardcodes WebChat/Control UI Enter to send and Shift+Enter for newline, with no send-on-enter preference or Ctrl/Cmd+Enter send mode in the browser composer, native composer, config schema/types/help, or WebChat docs.

## Best Possible Solution

Keep the issue open for a focused WebChat/Control UI enhancement. A good fix should decide the product surface, then plumb a send-on-enter preference through the browser composer and native Swift composer, support Ctrl/Cmd+Enter to send when plain Enter inserts a newline, and update config schema/types/help, docs, UI copy, and focused composer tests.
