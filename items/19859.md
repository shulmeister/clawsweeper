## Summary

Keep #19859 open. Current `main` still lacks a configurable WebChat send shortcut: browser WebChat and native macOS WebChat hard-code plain Enter/Return to send, while persisted UI settings, gateway config/schema help, and public WebChat docs expose no `sendOnEnter` or equivalent preference. #8147 is overlapping and open, but the latest issue discussion explicitly kept #19859 open unless maintainers choose to consolidate it.

## Best Possible Solution

Keep #19859 open unless maintainers explicitly consolidate it into #8147. A complete fix should add a typed WebChat send-shortcut preference, support plain Enter inserting newlines with Ctrl/Cmd+Enter sending when configured, wire it through the browser Control UI/WebChat composer and native WebChat composer, persist or schema-type the setting in the chosen settings surface, and update settings UI, config/schema help, public WebChat docs, and focused composer tests.
