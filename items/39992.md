## Summary

Current main still does not implement the requested `openclaw doctor` warning for runtime catalog models that are available but missing from an active `agents.defaults.models` allowlist. The model catalog/listing surfaces now include provider-owned supplemental rows, but doctor only validates a single configured `hooks.gmail.model`; the direct PR #46235 is closed unmerged in the provided context.

## Best Possible Solution

Keep #39992 open. Implement a read-only core doctor health contribution that loads the same runtime catalog used by model list/gateway surfaces, identifies provider-owned forward-compatible or supplemental catalog rows through a stable marker/contract, compares them with a non-empty `agents.defaults.models` allowlist, and warns with concrete model ids plus `openclaw models allowlist add <model-id>` guidance. It should skip when no allowlist is configured, never auto-modify config, and include focused tests for no allowlist, empty allowlist, no gaps, detected gaps, provider marker behavior, and warning wording. Treat the later API-vs-subscription cost-profile warning as a separate doctor/product decision.
