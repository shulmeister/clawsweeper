## Summary

Current main still does not implement the requested `openclaw doctor` warning for synthetic/provider-owned runtime catalog models that are available but missing from an active `agents.defaults.models` allowlist. The model-listing surface has improved since the original report, but the proactive doctor diagnostic remains absent, and the direct implementation PR #46235 is closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep this issue open and implement a read-only core doctor health contribution. It should load the same runtime catalog used by model-list/gateway surfaces, identify provider-owned synthetic or forward-compatible rows through a stable marker or documented provider-owned contract, compare them with a non-empty `agents.defaults.models` allowlist, and warn with concrete model ids plus `openclaw models allowlist add <model-id>` guidance. It should skip when no allowlist is configured, never auto-modify config, and include focused tests for no allowlist, empty allowlist, no gaps, gap detection, provider filtering/marker behavior, and warning wording. Treat the later cost-profile/API-vs-subscription warning from the comments as a separate doctor check or maintainer product decision.
