## Summary

Keep open. Current main still treats only a single exact silent token or JSON action envelope as silent; a doubled `NO_REPLY\n\nNO_REPLY` payload is not suppressed by the shared predicate and can still be normalized into visible `NO_REPLY` text before delivery. The Quiubo plugin-side guard is a useful workaround, but the shared OpenClaw behavior remains missing.

## Best Possible Solution

Keep this issue open until main adds shared repeated silent-token detection in `src/auto-reply/tokens.ts` or an equivalent normalization path. The narrow fix should make token-only sequences such as `NO_REPLY\n\nNO_REPLY` silent, apply through `isSilentReplyPayloadText` and delivery normalization, add regression tests for doubled `NO_REPLY` and custom tokens such as `HEARTBEAT_OK`, and preserve the existing #19537 substantive-reply guard. Maintainers can review #69472 as a possible consolidation path, but this issue should not close as implemented before the relevant behavior is merged.
