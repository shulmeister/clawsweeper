## Summary

Keep open. Current `main` has partial conversation-management primitives in Control UI, but it does not implement the requested native chat sidebar/session workspace. The chat surface still uses a header `<select>` for session switching, a separate `+` button for `/new`, and a separate Sessions page for renaming; the only in-chat sidebar is for tool-output viewing, not conversation history/session navigation.
