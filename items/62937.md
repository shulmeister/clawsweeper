## Summary

Keep this draft PR open. Current main has backend-based sandboxing and exec approval hardening, but it does not include this PR's macOS `sandbox-exec` wrapper or `sandboxProfile` exec parameter. The PR is also explicitly paired with the same-author open feature issue #58730, so conservative cleanup should leave both for maintainer product/security review.

## Best Possible Solution

Keep this PR open for maintainer review alongside #58730. If OpenClaw wants native macOS process isolation, design it as a sandbox backend or backend capability behind the existing `agents.defaults.sandbox` / `BashSandboxConfig.buildExecSpec` model, with config docs and regression coverage. If maintainers decide macOS Seatbelt support should not be carried in core, close this PR and the paired issue together after that explicit decision.
