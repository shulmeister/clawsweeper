## Summary

Keep open. The issue body points at an outdated bundled file, but the current `main` code still has the same unresolved shape: ACP runtime handles live in a process-long singleton cache, the cache reuse check does not consider any OpenClaw exec-policy/config fingerprint, and the SIGUSR1 restart path resets command lanes without resetting ACP runtime cache state. I did not find a regression test proving Discord/ACP sessions pick up `tools.exec.security` or `tools.exec.safeBins` changes after a gateway restart.

## Evidence

  - file: [src/acp/control-plane/manager.core.ts:1382](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/acp/control-plane/manager.core.ts#L1382)
