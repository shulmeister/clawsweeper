## Summary

Keep PR #68307 open. Current main still calls `resolveBootstrapContextForRun` from `resolveCommandsSystemPromptBundle` without a `warn` callback, while the bootstrap truncation warning is only emitted by `buildBootstrapContextFiles` when such a callback is supplied. The compact, attempt-runner, and CLI prepare paths already pass `makeBootstrapWarn`, so the PR addresses a real remaining gap in the commands/system-prompt path. The existing PR discussion also supports the change as a minimal additive fix.

## Best Possible Solution

Keep the PR open and land the focused fix after updating it for current main: import `makeBootstrapWarn`, create a `commands-system-prompt` subsystem logger, pass the warning callback into `resolveBootstrapContextForRun`, and add a small regression test in `src/auto-reply/reply/commands-system-prompt.test.ts` that verifies the callback is supplied or that a truncation warning reaches the logger.
