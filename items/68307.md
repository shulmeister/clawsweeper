## Summary

Keep open. Current `main` still drops bootstrap truncation warnings in the commands/system-prompt path: `resolveCommandsSystemPromptBundle` calls `resolveBootstrapContextForRun` without a `warn` callback, while the bootstrap helpers only emit truncation warnings when that callback is supplied. The PR matches an existing pattern already used in other runners, and there is no strong evidence the request is obsolete or already implemented on `main`.

## Evidence

- **Commands system-prompt path still omits `warn`:** On current `main`, `resolveCommandsSystemPromptBundle` calls `resolveBootstrapContextForRun` with `workspaceDir`, `config`, `sessionKey`, and `sessionId`, but no `warn` handler or subsystem logger.
