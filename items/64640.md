## Summary

Keep open. The webhooks half has been superseded on main by synchronous registration and request-time secret resolution, but current main still does not forward the active OpenClawConfig from resolveMemorySearchConfig into memory embedding provider lookup for provider, fallback, or multimodal validation. A narrowed memory-search fix from this PR remains useful after rebasing.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up, but require it to be narrowed or split: drop the obsolete webhooks changes in favor of merged #67941, keep the memory-search change that passes cfg into provider/fallback/multimodal lookup, add the runtime-config forwarding regression test, then run the changed agent/plugin gates.
