## Summary

Keep open. The merged #67941 work supersedes the webhooks/synchronous-registration half of this draft PR, but current main still does not pass the active OpenClawConfig into memory-search provider, fallback, or multimodal provider lookup. That leaves a meaningful unresolved part of #64640 that is not covered by the provided canonical PR evidence.

## Best Possible Solution

Keep #64640 open or replace it with a narrower maintainer-owned PR that forwards cfg through resolveMemorySearchConfig provider, fallback, and multimodal provider lookups with regression coverage. Treat the webhooks changes as superseded by merged #67941 and discard that portion during any rebase.
