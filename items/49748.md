## Summary

Keep open. Current `main` still does not broadcast `inbound_claim` for regular non-plugin-bound messages: `dispatch-from-config.ts` only calls the targeted `runInboundClaimForPluginOutcome(...)` path for plugin-owned bindings, then goes straight from `message_received` hooks to `markProcessing()` and agent dispatch. The test suite now explicitly asserts that non-plugin-bound messages must not call `runInboundClaim`, so the reported gap is still present rather than fixed.
