## Summary

Keep open. Current `main` still appears to implement the reporter's bug: the dashboard `Set Default` action writes `agents.defaultId`, but the config schema only accepts `agents.defaults` and `agents.list`, while default-agent resolution still comes from `agents.list[].default` or the first list entry.

## Evidence

- **Dashboard writes unsupported key:** The Agents UI `onSetDefault` handler updates the config form at `['agents', 'defaultId']`, matching the reporter's failing key.
  - file: [ui/src/ui/app-render.ts:2103](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/ui/src/ui/app-render.ts#L2103)
  - command: `nl -ba ui/src/ui/app-render.ts | sed -n '2098,2106p'`
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **Agents schema has no `defaultId` field:** `AgentsSchema` only allows `defaults` and `list` under `agents`, so `agents.defaultId` is rejected by strict validation.
  - file: [src/config/zod-schema.agents.ts:7](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/config/zod-schema.agents.ts#L7)

  - file: [src/config/zod-schema.agent-runtime.ts:812](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/config/zod-schema.agent-runtime.ts#L812)

- **Default-agent fallback matches reporter's observation:** `resolveDefaultAgentId()` chooses the first `agents.list[]` entry with `default: true`, otherwise the first list entry, so list order still controls the fallback default.
  - file: [src/agents/agent-scope-config.ts:78](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/agents/agent-scope-config.ts#L78)

## Risks / Open Questions

- The dashboard still appears able to stage an invalid config that the gateway will reject on save.
- If users rely on the UI instead of editing `agents.list[].default`, the effective default agent can still fall back to list order and produce confusing status/routing behavior.
