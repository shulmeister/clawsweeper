## Summary

Keep open. Current main still labels the composer plus button as "New session" and routes it to `/new`; the requested UI wording/distinction from reset context is not implemented. The docs also still describe `/new`/`/reset` as manual reset behavior that creates a new `sessionId` for the same `sessionKey`, matching the confusion described in the issue rather than resolving it.
