## Summary

Keep open. Current main does not implement optional marquee/auto-scrolling display for long status information. The Control UI topbar and nearby compact text surfaces still use static hiding/ellipsis or omit long runtime status text entirely, and the TUI header/status/footer are plain `Text`/`Loader` messages without overflow measurement or scrolling state. Related reports show some underlying model/token/session visibility work is separate, but they do not supersede this broader UI overflow behavior request.

## Best Possible Solution

Keep this issue open as a product/UI enhancement. The best implementation path is an accessible reusable overflow-status component or helper, then explicit opt-in use in high-value surfaces such as the TUI footer/status line and selected Control UI status/topbar/plugin-status regions. It should preserve static rendering when content fits, pause or allow manual interaction on focus/hover, respect reduced-motion, and coordinate with #13670 and #42788 where model/token/session label visibility is still being tracked separately.
