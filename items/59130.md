## Summary

Current `main` still matches the reported bug: `memory-lancedb` recall is plain semantic similarity, injects historical memories into prompt context without timestamp/provenance fields, and has no regression coverage for freshness-sensitive queries like "latest".

## Evidence

- **Memory schema lacks provenance or source-time fields:** `MemoryEntry` only stores `id`, `text`, `vector`, `importance`, `category`, and `createdAt`; there is no `sourceTimestamp`, `sourceType`, `sourceUrl`, thread/channel provenance, or similar metadata to support freshness-aware recall.
  - file: [extensions/memory-lancedb/index.ts:30](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/memory-lancedb/index.ts#L30)
  - command: `nl -ba extensions/memory-lancedb/index.ts | sed -n '20,140p'`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Search path is similarity-only:** `MemoryDB.search()` runs `vectorSearch(...).limit(limit).toArray()`, converts `_distance` to a similarity score, and filters by `minScore`; it does not rank or filter by recency or timestamps beyond carrying `createdAt` through the result.
  - file: [extensions/memory-lancedb/index.ts:120](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/memory-lancedb/index.ts#L120)
  - command: `nl -ba extensions/memory-lancedb/index.ts | sed -n '20,140p'`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Auto-recall still injects top semantic matches for every prompt:** The `before_prompt_build` hook embeds `event.prompt`, calls `db.search(vector, 3, 0.3)`, and prepends the returned memories directly into context. There is no freshness-intent detection, live lookup fallback, or stale-memory warning on this path.
  - file: [extensions/memory-lancedb/index.ts:581](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/memory-lancedb/index.ts#L581)

- **Injected context omits provenance details:** `formatRelevantMemoriesContext()` renders each recalled item as `[category] text` under `<relevant-memories>`. The prompt framing says the memories are untrusted historical data, but it still provides no timestamp/source receipt the assistant can surface to distinguish recall from a live query.
