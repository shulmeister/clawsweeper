## Summary

Keep open. Current main preserves explicit empty plugin scopes semantically, but it has not implemented the PR’s requested O(1 loader fast-path: `loadOpenClawPlugins({ onlyPluginIds: [] })` still checks the registry cache, enters the normal load path, creates a registry, runs discovery, and loads the manifest registry before filtering every candidate out. Related merged work (#65120 and #65298) covers activation planning and empty-scope correctness, not this performance shortcut.

## Best Possible Solution

Keep the PR open, but require a focused loader regression test that spies or mocks discovery/manifest loading for `onlyPluginIds: []`, plus coverage that the fast-path still distinguishes `undefined` from `[]` and preserves the reentry/activation side effects. The likely implementation belongs in `src/plugins/loader.ts` near the existing cache/reentry setup, using `createEmptyPluginRegistry` without widening empty scopes.
