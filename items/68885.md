## Summary

PR #68885 should close as ClawHub/plugin scope. It adds repo-wide static ffmpeg/ffprobe dependencies and rewrites the bundled video-frames skill, but current main intentionally keeps video-frames as a host-ffmpeg-gated bundled skill and exposes plugin/bundle/ClawHub paths that can carry dependency-owning media helpers outside core. The PR discussion also shows review concerns around the dependency/runtime behavior, and no concrete missing core plugin API or maintainer-owned product/security reason is present.

## Best Possible Solution

Do not merge static ffmpeg/ffprobe dependency ownership into OpenClaw core. Keep the shipped host-binary video-frames skill gating and system-binary media runtime; publish a no-host-ffmpeg workflow as a ClawHub skill bundle or native plugin that owns ffmpeg-static/@ffprobe-installer dependencies and any QQ Bot-specific behavior. Revisit core only for a focused missing plugin API, security fix, or explicit maintainer product decision.
