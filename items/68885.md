## Summary

Close as ClawHub/plugin scope. PR #68885 tries to move the bundled video-frames skill and shared media ffmpeg execution onto ffmpeg-static/@ffprobe-installer. Current main keeps this as a host-binary contract with explicit skill gating, and the project direction says optional skills/dependency bundles should ship through ClawHub or plugins unless there is a clear core product/security reason. Current plugin and bundle docs already provide the surfaces needed to ship a dependency-owning video-frame helper outside core.

## Best Possible Solution

Do not merge the static ffmpeg/ffprobe dependency bundle into OpenClaw core. Keep the current host-binary skill gating and shared system-binary resolver in core; if users want a no-host-ffmpeg video-frame workflow, publish it as a ClawHub skill bundle or native plugin that owns ffmpeg-static/@ffprobe-installer dependencies and any QQ Bot-specific behavior. Revisit core only if that external implementation exposes a concrete missing plugin API.
