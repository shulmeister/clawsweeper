## Summary

Keep #67990 open. Current main has shipped important backup pieces after the earlier backup umbrella work: `openclaw backup create`, `.tar.gz` output, manifest verification, `--only-config`, `--no-include-workspace`, missing top-level path skips, and canonical top-level path dedupe that addresses part of the symlink story. But the proposal’s central remaining requests are not implemented: configurable exclusion rules, service-aware gateway quiescence or another explicit SQLite/session-race strategy, restore/full-mode semantics, and fine-grained archive filtering such as plugin `node_modules` exclusion. The linked standalone `JIRBOY/openclaw-backup` is useful prior art, not a merged core implementation or a ClawHub substitute for core backup command behavior.

## Best Possible Solution

Keep the issue open, but narrow it into a maintainer-readable backup design: add a core backup exclusion policy surface or CLI/config option, decide whether the ENOENT/SQLite problem should be solved by resilient per-file archiving or explicit gateway quiescence, keep the already-shipped tar.gz/create/verify/config-only behavior, and cross-link #13616, #64144, and #67417 so overlapping pieces can close as they land.
