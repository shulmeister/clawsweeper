## Summary

Current main still handles Feishu bot added/removed subscription events only by parsing and logging them. I found no emitted internal hook event, no typed plugin hook for these Feishu lifecycle events, and the only related PR explicitly scoped out Feishu hook work. The requested capability remains meaningful work, so the issue should stay open.
