## Summary

Keep #34129 open. Current main still preserves a blank tool-call name when both the model-provided name and id are blank, then assigns a call_auto_* fallback id. That matches the reported NVIDIA/Kimi failure shape, and the related sentinel fix is tracked by open PR #56391 but is not on main.

## Best Possible Solution

Keep #34129 open until PR #56391 or an equivalent maintainer-approved fix lands. The likely fix belongs in the embedded runner tool-call normalization path, with regression coverage that blank or whitespace-only model tool names without an inferable id are converted to a dispatch-safe sentinel or otherwise stop the Tool not found retry loop.
