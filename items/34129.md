## Summary

Keep #34129 open. Current main still allows the reported blank/non-inferable tool-call name to remain blank while assigning a `call_auto_*` fallback id, and the current regression test explicitly preserves that behavior. The linked sentinel fix remains represented by open PR #56391 rather than shipped main code.

## Best Possible Solution

Keep #34129 open until PR #56391 or an equivalent maintainer-approved fix lands. The fix should prevent blank or whitespace-only model tool names with blank/non-inferable ids from reaching dispatch as raw blank names with `call_auto_*` ids, and should update regression coverage so the reported loop shape cannot return.
