## Summary

Keep #34129 open. Current main still preserves a blank tool-call name when the model emits both a blank name and blank id, then assigns a call_auto_* fallback id. The related sentinel fix remains tracked by PR #56391 rather than implemented on main.

## Best Possible Solution

Keep #34129 open until PR #56391 or an equivalent maintainer-approved fix lands. The fix should change the embedded-runner tool-call normalization path so blank or whitespace-only model tool names without an inferable id cannot enter dispatch as raw blank names with call_auto_* ids, and should update regression coverage to prove that behavior.
