## Summary

Keep open. Current main still does not persist the resolved gateway port in the gateway lock payload, and the PR’s focused change is not implemented or superseded by the related stale-lock work. The PR likely needs adaptation to current main because lock payload parsing now goes through `LockPayloadSchema`.

## Best Possible Solution

Keep the PR open for maintainer review. The likely path is to adapt the change to current main by adding optional `port` support to `LockPayload`, `LockPayloadSchema`, and the lock writer, keeping the contributor’s range guard and regression tests. If lock-file discovery is intentionally unsupported, close only after a maintainer states that product boundary explicitly and points external tools at the supported discovery path.
