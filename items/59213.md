## Summary

Current `main` already covers both reported failure modes: live model switching is gated to explicit user-driven switches instead of fallback rotations, and Anthropic "extra usage" responses are classified as billing rather than retryable rate limits.

## Evidence

- **Session flag now distinguishes user switches from fallback rotation:** `liveModelSwitchPending` is documented as only for explicit user-driven model changes, and system-initiated fallbacks are explicitly excluded.
  - file: [src/config/sessions/types.ts:188](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/config/sessions/types.ts#L188)
- **Live-switch reconciliation no longer conflates fallback rotations:** `shouldSwitchToLiveModel` replaced the older in-memory approach because it could not distinguish user-initiated switches from system fallback rotations.
  - file: [src/agents/live-model-switch.ts:123](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/live-model-switch.ts#L123)
- **Runner still checks pending live switches only through the explicit flag path:** The embedded runner consults `shouldSwitchToLiveModel(...)` before throwing `LiveSessionModelSwitchError`, so the reconciliation path is now mediated by the explicit pending-switch state.
  - file: [src/agents/pi-embedded-runner/run.ts:974](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/pi-embedded-runner/run.ts#L974)
- **Fallback loop no longer rethrows live-switch conflicts into an infinite restart loop:** `model-fallback.ts` now treats `LiveSessionModelSwitchError` as a failover condition and continues instead of rethrowing into the outer retry loop; the paired test explicitly calls out the prior indefinite restart behavior.
  - file: [src/agents/model-fallback.ts:921](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/model-fallback.ts#L921)
- **Anthropic extra-usage message is classified as billing:** The exact `Extra usage is required for long context requests.` family is matched in billing classification, with regression coverage asserting billing classification for the reported Anthropic variants.
  - file: [src/agents/pi-embedded-helpers/failover-matches.ts:143](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/pi-embedded-helpers/failover-matches.ts#L143)
- **Regression test covers exact Anthropic message from this report:** The billing-classification test includes `Extra usage is required for long context requests.` and asserts `classifyFailoverReason(..., { provider: "anthropic" }) === "billing"`.
  - file: [src/agents/pi-embedded-helpers.isbillingerrormessage.test.ts:231](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/pi-embedded-helpers.isbillingerrormessage.test.ts#L231)
