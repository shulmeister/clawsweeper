## Summary

Keep #60582 open. Current main has parent-channel fallback for Discord threads, but normal guild messages do not pass the fetched channel parent/category ID into channel allowlist resolution, and the documented config still describes only guild/channel entries rather than category-level inheritance.

## Evidence

- **Discord preflight fetches channel metadata:** The preflight path fetches channelInfo for every message, and DiscordChannelInfo includes parentId, so category metadata can exist at runtime.
  - file: [extensions/discord/src/monitor/message-handler.preflight.ts:443](https://github.com/openclaw/openclaw/blob/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54/extensions/discord/src/monitor/message-handler.preflight.ts#L443)
  - command: `nl -ba extensions/discord/src/monitor/message-handler.preflight.ts | sed -n '420,630p'`
  - sha: [02f3e9cfa25f](https://github.com/openclaw/openclaw/commit/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54)
- **Only thread parent is passed to config fallback:** resolveDiscordChannelConfigWithFallback receives parentId from threadParentId, which is only assigned when earlyThreadChannel is present. A normal guild channel's category parentId from channelInfo is not passed, so category-level entries cannot cover ordinary child channels.
  - file: [extensions/discord/src/monitor/message-handler.preflight.ts:774](https://github.com/openclaw/openclaw/blob/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54/extensions/discord/src/monitor/message-handler.preflight.ts#L774)
  - command: `nl -ba extensions/discord/src/monitor/message-handler.preflight.ts | sed -n '700,920p'`
  - sha: [02f3e9cfa25f](https://github.com/openclaw/openclaw/commit/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54)
- **Allowlist resolver supports parent fallback only when parent keys are provided:** The resolver can match parentKeys before wildcard, but it depends on caller-provided parent keys. This supports thread-parent inheritance but does not by itself implement Discord category inheritance for normal channels.
  - file: [extensions/discord/src/monitor/allow-list.ts:435](https://github.com/openclaw/openclaw/blob/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54/extensions/discord/src/monitor/allow-list.ts#L435)
  - command: `nl -ba extensions/discord/src/monitor/allow-list.ts | sed -n '260,520p'`
  - sha: [02f3e9cfa25f](https://github.com/openclaw/openclaw/commit/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54)
- **Docs still describe per-channel allowlist behavior:** The Discord docs say that if a guild has channels configured, non-listed channels are denied, and examples show individual channel keys. They do not document category-level channel config or a category: prefix.
  - file: [docs/channels/discord.md:422](https://github.com/openclaw/openclaw/blob/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54/docs/channels/discord.md#L422)
