## Summary

Keep #37589 open. Current main still rebuilds array-content messages and text-part objects in `stripInlineDirectiveTagsFromMessageForDisplay` even when no inline directive tag changes the text; the closed related PRs in the provided context were not merged and do not supersede the issue.

## Best Possible Solution

Keep the issue open and implement the narrow identity-preserving optimization in `src/utils/directive-tags.ts`: call `stripInlineDirectiveTagsForDisplay` once per text part, reuse unchanged text-part objects, track whether any part changed, and return the original message object when nothing changed. Add focused `toBe` regression tests in `src/utils/directive-tags.test.ts` for unchanged array messages and mixed changed/unchanged parts.
