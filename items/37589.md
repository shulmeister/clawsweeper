## Summary

Keep #37589 open. Current main still recreates every array-backed message object and every text part in `stripInlineDirectiveTagsFromMessageForDisplay`, even when `stripInlineDirectiveTagsForDisplay` reports no change. The prior implementation PRs referenced by the issue are closed and unmerged, and current tests still do not assert referential identity preservation.

## Best Possible Solution

Implement the narrow identity-preservation fix in `src/utils/directive-tags.ts`: reuse unchanged text-part objects, track whether any text block changed, and return the original message object when nothing was stripped. Add focused Vitest coverage with `toBe` assertions for unchanged array-backed messages and mixed changed/unchanged text parts. If maintainers consider the helper obsolete, remove or de-export it deliberately from the plugin text runtime and update tests/API expectations accordingly.
