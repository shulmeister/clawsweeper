## Summary

Keep #37589 open. Current main still leaves `stripInlineDirectiveTagsFromMessageForDisplay` rebuilding every text part and returning a new message object for array-backed content even when no inline directive tag was stripped. The newer chat history projection path is change-aware, but it does not fix this exported utility; related PRs in the provided context are closed and unmerged.

## Best Possible Solution

Implement the narrow optimization in `src/utils/directive-tags.ts`: reuse unchanged text parts, track whether any text part actually changed, return the original message object when nothing was stripped, and add focused `toBe` regression tests for unchanged array messages plus mixed changed/unchanged parts. If maintainers consider the helper obsolete, remove or de-export it deliberately with tests updated to the current projection API instead of silently leaving the stale behavior.
