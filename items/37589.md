## Summary

Keep #37589 open. Current main still recreates array-backed message objects and text-part objects in `stripInlineDirectiveTagsFromMessageForDisplay` even when no inline directive tag was stripped, and the linked attempted fixes are closed unmerged.

## Best Possible Solution

Implement the focused identity-preservation fix in `src/utils/directive-tags.ts`: reuse unchanged text-part objects, track whether any text part changed, and return the original message object when nothing was stripped. Add colocated Vitest coverage with `toBe` assertions for unchanged array-backed messages and mixed changed/unchanged text parts. If maintainers do not want this behavior as a plugin-facing helper contract, handle that deliberately through SDK surface/docs changes.
