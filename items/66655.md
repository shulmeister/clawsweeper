## Summary

#66655 is superseded. It patched an older inline Control UI helper, but current main delegates the thinking dropdown to `ui/src/ui/chat/session-controls.ts`, where the UI already consumes `thinkingDefault` from the session/default payload. The remaining configured-default mismatch is now the gateway payload bug tracked by open issue #70967 and open PR #70981, with maintainer discussion already selecting #70981 over a duplicate gateway PR.

## Best Possible Solution

Close #66655 as superseded and keep review focused on #70981/#70967. The best implementation path is a gateway-side, config-aware `thinkingDefault` payload resolver with regression coverage, while keeping the current UI behavior that simply consumes `thinkingDefault` from session/default payloads.
