## Summary

Current `main` does not support the PR's stated failure mode. In production ACP server flow, gateway events are dispatched fire-and-forget, so changing one inner `void this.finishPrompt(...)` to `await` would not change live behavior, and `finishPrompt()` already guards the snapshot/update path that the PR claims can leave prompts hanging.
