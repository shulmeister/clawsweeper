## Summary

Current `main` already keeps TUI busy-state rendering separate from the input editor, so the reported spinner-loss-on-typing behavior appears resolved.

## Evidence

- **Dedicated status container and loader:** The TUI layout gives status its own `statusContainer`, and busy states render through a `Loader` that is only torn down when `activityStatus` leaves `sending|waiting|streaming|running`. That matches the expected 'keep showing progress while a run is pending' behavior.
  - file: [src/tui/tui.ts:523](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/tui/tui.ts#L523)
  - command: `nl -ba src/tui/tui.ts | sed -n '520,760p'`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Pending indicator is owned by run lifecycle:** Sending a message sets `activityStatus` to `sending`, then `waiting` after the chat request is accepted. Incoming chat events move the same run to `streaming` and only clear status on final/error/abort.
  - file: [src/tui/tui-command-handlers.ts:573](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/tui/tui-command-handlers.ts#L573)
  - command: `nl -ba src/tui/tui-command-handlers.ts | sed -n '560,660p'`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Typing path does not clear status:** Normal editor typing falls through to `super.handleInput(data)` with no status mutation. Only explicit control shortcuts such as Ctrl+C/Ctrl+O/etc. change activity text.
  - file: [src/tui/components/custom-editor.ts:16](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/tui/components/custom-editor.ts#L16)
  - command: `nl -ba src/tui/components/custom-editor.ts | sed -n '1,120p'`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Streaming path keeps the busy state alive:** Chat deltas call `setActivityStatus("streaming")` for the active run and finals/errors are what transition the status back out of the busy state.
  - file: [src/tui/tui-event-handlers.ts:313](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/tui/tui-event-handlers.ts#L313)
