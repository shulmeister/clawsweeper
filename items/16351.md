## Summary

Keep #16351 open. Current main covers `openclaw config validate` and protects existing env/SecretRef-style secrets during config write-back, but the first requested behavior is still missing: bare `openclaw config` intentionally launches the configure wizard instead of showing help on an already configured system.

## Best Possible Solution

Keep this issue open until a maintainer either implements the remaining CLI behavior or decides the current wizard default is the intended product direction. A likely code path is `src/cli/config-cli.ts` root `config` action: show help, at least for already configured systems, while preserving the shipped `config validate` and secret-preserving write behavior.
