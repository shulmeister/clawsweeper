## Summary

Keep #16351 open. Current main covers two requested pieces (`openclaw config validate` and preserving env/SecretRef-backed values during config writes), but the remaining requested root behavior is still not implemented: bare `openclaw config` still dispatches to the configure wizard instead of showing help on an already configured system.

## Best Possible Solution

Keep the issue open until a maintainer either implements the remaining root `openclaw config` behavior or explicitly decides the current wizard default is intentional. The likely implementation point is `src/cli/config-cli.ts` in the root `config` action: detect an already configured system and display command help, while preserving the shipped `config validate` and secret-preserving write behavior.
