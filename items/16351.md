## Summary

Keep #16351 open. Current main implements `openclaw config validate` and preserves existing env/SecretRef-backed secret markers during config write-back, but the remaining requested behavior is still missing: bare `openclaw config` still starts guided setup instead of showing help on an already configured system.

## Best Possible Solution

Keep the issue open until a maintainer either implements the remaining root `openclaw config` behavior or explicitly decides the wizard default is the intended product direction. The likely implementation point is `src/cli/config-cli.ts` root `config` action: detect an already configured system and show command help, while preserving the shipped `config validate` and secret-preserving write behavior.
