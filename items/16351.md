## Summary

Keep open. Current main implements `openclaw config validate` and config write protections for invalid/secret-resolved payloads, but bare `openclaw config` is still wired and documented as a guided setup entry point. That leaves one explicit requested behavior unresolved, matching the existing maintainer-posted Codex review on the issue.

## Best Possible Solution

Keep #16351 open until a maintainer either implements the remaining root `openclaw config` behavior or explicitly decides the wizard default is intentional. The likely code path is `src/cli/config-cli.ts` and `src/commands/configure.commands.ts`: when no subcommand and no `--section` are supplied on an already configured system, print config help instead of entering guided setup, while preserving the shipped `config validate` and SecretRef/env-preserving write behavior and updating docs/tests accordingly.
