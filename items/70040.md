## Summary

Keep PR #70040 open. Current main still has no Chinese/CJK trigger support in `memory-lancedb`, so the request is not implemented-on-main. PRtags correctly grouped it with earlier Chinese/CJK attempts (#16392, #16672, #20241), but the provided context shows those PRs are closed and unmerged, and the current checkout confirms their behavior is not present. The open PR is also not ready as-is because the existing length gate would still reject common short Chinese memory statements and there is no Chinese regression coverage on main.

## Best Possible Solution

Keep the PR open or replace it with a focused memory-lancedb CJK support patch that adds high-signal Chinese/CJK trigger coverage, adjusts the short-input gate for CJK memory statements, adds category detection where appropriate, and includes regression tests for `shouldCapture`, `detectCategory`, and the auto-capture path. If maintainers prefer one of the earlier attempts, reopen/supersede with a current-main branch rather than closing this as already shipped.
