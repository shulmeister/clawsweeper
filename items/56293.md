## Summary

Keep open. Current `main` still lacks a TUI message-timestamp display or toggle. The chat history pipeline preserves timestamps internally, but the TUI renderer and documented command/option surface do not expose them in the chat view, so the feature request is not implemented and does not fit any conservative auto-close reason.

## Evidence

- **TUI history loader ignores message timestamps when rendering:** `loadHistory()` iterates history messages and calls `chatLog.addUser(text)` / `chatLog.finalizeAssistant(text)` with only extracted text. It never passes a timestamp or any display option into the chat-log renderer.
  - file: [src/tui/tui-session-actions.ts:288](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/tui/tui-session-actions.ts#L288)
  - command: `nl -ba src/tui/tui-session-actions.ts | sed -n '260,380p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Chat-log components do not have any timestamp-rendering API:** `ChatLog.addUser()` constructs `UserMessageComponent(text)` and `finalizeAssistant()` constructs `AssistantMessageComponent(text)`. The message components only accept text, so there is no existing hook for a `[HH:MM:SS]` prefix or optional per-message timestamp display.
  - file: [src/tui/components/chat-log.ts:105](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/tui/components/chat-log.ts#L105)

- **User and assistant message components render text only:** `UserMessageComponent` forwards only `text` to `MarkdownMessageComponent`, and `AssistantMessageComponent` stores only a markdown body and `setText(text)`. Neither component formats or displays timestamps.
  - file: [src/tui/components/user-message.ts:4](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/tui/components/user-message.ts#L4)
