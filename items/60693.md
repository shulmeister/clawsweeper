## Summary

Keep open. Current main appears to address the Discord REST proxy path and documents model precedence better, but the issue bundles remaining actionable requests: the ACPX Codex adapter still goes through npx rather than a bundled codex-acp repair path, `openclaw models set` only updates the default model rather than all per-agent/cron overrides, and `acp.allowedAgents` is still treated as a raw allowlist/probe source without checking whether a target is actually usable or disabled.
