## Summary

Keep this PR open. Current main still supports streamable-http MCP servers, but the bundled session runtime caches the original MCP client and forwards tool calls directly; it does not detect lost streamable-http sessions, rebuild the client/transport, refresh tool metadata, or retry. The PR is a focused implementation candidate for a real supported surface, and the discussion shows maintainer follow-up hardening rather than a reason to close it.

## Best Possible Solution

Keep this PR open for review. The best path is to validate and land a hardened streamable-http reconnect implementation in `src/agents/pi-bundle-mcp-runtime.ts`, with regression coverage in `src/agents/pi-bundle-mcp-runtime.test.ts` and `src/agents/pi-bundle-mcp-test-harness.ts`, preserving the custom schema validator, single in-flight reconnect, narrow lost-session detection, metadata refresh, one retry, and retryability after failed reconnect attempts.
