## Summary

Keep open. Current main still appears to have the reported bug: the setup flow only includes web search providers whose `onboardingScopes` contain `"text-inference"`, while the DuckDuckGo provider base used by both runtime and contract public artifacts still does not set `onboardingScopes`. Existing tests do not prove the real bundled DuckDuckGo provider appears in onboarding because the relevant setup tests use a mocked DuckDuckGo entry that already includes the missing field.

## Best Possible Solution

Keep the issue open until the actual bundled DuckDuckGo provider metadata is fixed. The narrow implementation is to add `onboardingScopes: ["text-inference"]` to `createDuckDuckGoWebSearchProviderBase()` so both runtime and public contract artifacts inherit it, then add a regression test that resolves the real bundled DuckDuckGo provider through the setup/public-artifact path and proves `resolveSearchProviderOptions()` includes `duckduckgo`. PR #65940 or an equivalent maintainer-approved patch should close the issue after merge.
