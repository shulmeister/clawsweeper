## Summary

Keep open. Current main still has the behavior the PR is trying to change: a non-default Discord account without its own token inherits the channel-level token. The PR is not obsolete, but its current revision should not be merged as-is until it handles internal implicit account scans/action discovery separately from explicitly targeted cron/outbound delivery.

## Best Possible Solution

Keep the PR open and ask for a focused revision: preserve strict token resolution for explicitly targeted non-default Discord sends, but make implicit default-account resolution and internal account enumeration/action discovery continue to inherit `channels.discord.token`. Add regression coverage for `resolveDiscordToken`, `resolveDiscordAccount` with omitted accountId/defaultAccount, `listEnabledDiscordAccounts` or Discord action discovery, and the explicit cron/outbound accountId path.
