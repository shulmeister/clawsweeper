## Summary

Current `main` still has the Discord token fallback this PR is trying to change, and the PR discussion shows follow-up review concerns rather than an obsolete fix. This remains live, useful work and should stay open for maintainer review.

## Evidence

- **Current main still falls back to the channel token for named accounts:** `resolveDiscordToken` checks for a per-account token and then falls through to `channels.discord.token`; there is no explicit-account guard on `main`, so a non-default `accountId` without its own token still inherits the top-level token.
