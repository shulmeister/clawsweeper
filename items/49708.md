## Summary

Keep open. Current `main` still canonicalizes `sessionFile` through the real session directory path and persists that value, while `openclaw doctor` builds orphan-candidate paths from the caller-visible `sessionsDir`. With `~/.openclaw` as a symlink, that leaves a realpath-vs-logical-path mismatch exactly in the area the report describes, so the false-positive orphan scenario remains plausible on current `main`. I did not find a merged fix in the checked-out history.

That preserves the canonical target path, not the logical symlink alias.
