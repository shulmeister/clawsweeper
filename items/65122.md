## Summary

Current main already includes a shipped ACPX probe-agent fix for non-`codex` deployments, with matching docs and regression coverage. That addresses the reported `runtime="acp"` failure mode from `2026.4.5`, where the embedded backend could stay unavailable because readiness was probed against the wrong harness agent.

## Evidence

- **Shipped fix note:** `v2026.4.23` changelog explicitly says ACPX now exposes `probeAgent` and stops marking the runtime backend unavailable when the default probe agent is not installed or authenticated.
