## Summary

Keep open. Current main has automatic bootstrap/context handling for spawned subagents, but it still does not expose this PR's requested `contextFiles` / `inheritContext` `sessions_spawn` API, and current cross-agent workspace behavior intentionally uses the target agent workspace rather than the requester workspace promised by the PR. The submitted PR also still has unresolved review blockers and an unrelated plugin logger type change.

## Best Possible Solution

Keep this PR open for maintainer product judgment. If OpenClaw wants explicit file-selection APIs, rework the PR against current bootstrap plumbing with documented requester-vs-target workspace semantics, bounded `contextFiles`, regression tests for same-agent and cross-agent behavior, docs, and removal of the unrelated plugin logger type change. If maintainers decide automatic bootstrap plus `lightContext` is the supported direction, close this manually with that rationale and leave the current target-agent workspace behavior documented.
