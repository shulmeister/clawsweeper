## Summary

Keep open. Current main has automatic workspace bootstrap context for spawned subagents plus a `lightContext` option, but it still does not expose or honor this PR's explicit `sessions_spawn.contextFiles` / `inheritContext` API. The PR is also not merge-ready as submitted because its requester-workspace semantics conflict with current target-workspace behavior and prior review found an unbounded `contextFiles` input plus an unrelated plugin logger type change.

## Best Possible Solution

Keep this PR open for maintainer product judgment. If OpenClaw wants explicit `sessions_spawn` file-selection, rework it on top of current bootstrap plumbing with documented requester-vs-target workspace semantics, bounded `contextFiles`, regression tests for same-agent and cross-agent spawns, docs, and removal of the unrelated plugin logger type change. If automatic bootstrap plus `lightContext` is the intended direction, maintainers should close it manually with that rationale.
