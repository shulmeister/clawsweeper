## Summary

Keep open. Current main has automatic workspace bootstrap injection for spawned subagents and a `lightContext` opt-out, but it still does not expose or honor this PR's explicit `sessions_spawn.contextFiles` or `inheritContext` API, and current cross-agent workspace behavior intentionally uses the target agent workspace rather than the requester workspace promised by the PR.

## Best Possible Solution

Keep the PR open for maintainer product judgment. If explicit spawn-time file selection is desired, rework the PR on top of current bootstrap plumbing with documented requester-vs-target workspace semantics, bounded `contextFiles`, regression tests for same-agent and cross-agent spawns, docs updates, and removal of the unrelated plugin logger type change. If automatic bootstrap plus `lightContext` is the intended product direction, maintainers should close it manually with that rationale.
