## Summary

Current main already has a per-session label/displayName path that makes WebChat session choices human-readable: users can set a session label from the Sessions UI, Gateway persists it via sessions.patch, and the Chat session selector renders label/displayName before falling back to raw keys. The implementation is also present in the latest release tag v2026.4.24.
