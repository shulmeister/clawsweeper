## Summary

Keep open. Current `main` still matches the reported behavior: memory-core falls back when `subagent.run()` throws `RequestScopedSubagentRuntimeError`, returns `null`, and then the `finally` block still calls `subagent.deleteSession({ sessionKey })`, which can throw the same request-scoped runtime error and log the cleanup warning. The current tests encode that behavior rather than proving it fixed, and the issue discussion correctly identifies it as a lifecycle/scope mismatch.

## Best Possible Solution

Keep the issue open and fix the memory-core dreaming narrative lifecycle directly: either run Dream Diary subagent work through a gateway-safe/background subagent runtime, or return a structured fallback result and skip request-scoped `deleteSession` after request-scoped fallback. Update the narrative and sweep tests so request-scoped fallback completes without a cleanup warning.
