## Summary

Keep #46015 open. Current main has implemented fallback observability for `/tts status`, but the remaining behavior explicitly called out by the maintainer comment is still not fully implemented: auto TTS still uses a provider fallback chain, and I did not find a per-agent Discord voice assignment config/API.
