## Summary

Keep #46661 open. Current main has adjacent transcription support for media attachments and the Voice Call plugin, but it does not implement the requested custom ASR endpoint for native macOS/iOS/Android voice input. The native app voice paths still use Apple/Android platform speech recognizers directly, and settings/config expose mic/locale/TTS-related options rather than an ASR provider URL, auth header, model, or language endpoint configuration.
