## Summary

Keep #13175 open. Current main still does not implement generic Feishu inline/native card buttons for shared presentation payloads: Feishu message actions render presentation buttons as markdown fallback text, and the Feishu outbound adapter still lacks the native presentation/sendPayload hooks that core uses for rich outbound controls. The existing Feishu card-action plumbing is useful, but it is currently wired to specific quick-action/approval flows rather than generic inline button rendering.

## Best Possible Solution

Keep #13175 open. Implement Feishu-owned native rendering for shared `MessagePresentation` buttons: add Feishu outbound `presentationCapabilities` plus `renderPresentation` or equivalent delivery, render Schema 2.0 `action`/`button` elements for value and URL buttons, wrap callback values in structured Feishu interaction envelopes, route clicks through `handleFeishuCardAction`, and add tests for clickable buttons, URL fallback, unsupported fallbacks, and callback dispatch. Treat #47891 as related narrower `sendPayload`/native-card payload work unless maintainers broaden it explicitly.
