## Summary

Keep #13175 open. Current main has Feishu interactive card infrastructure and built-in quick-action/approval card callbacks, but the remaining requested capability is broader: arbitrary Feishu inline button rendering from the message/presentation path, equivalent to Telegram-style inline buttons. Current Feishu presentation buttons still degrade to markdown fallback text instead of native action buttons.
