## Summary

Keep #13175 open. Current main has Feishu interactive card callback infrastructure and built-in quick-action/approval cards, but it still does not implement arbitrary Feishu inline button rendering from the shared message/presentation path; presentation buttons are explicitly tested as markdown fallback text.

## Best Possible Solution

Implement native Feishu rendering for shared `MessagePresentation` button blocks, likely in the Feishu presentation/card rendering path, by mapping button values into structured Feishu card interaction envelopes and adding tests that assert `action`/`button` card elements plus callback routing. If PR #47891 is intended to be the canonical implementation, keep #13175 open until that PR merges or the maintainer explicitly links it as the tracking item.
