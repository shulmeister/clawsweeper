## Summary

Keep #13175 open. Current main has Feishu interactive card infrastructure, structured callback handling, and built-in quick-action/approval cards, but it still does not implement native rendering/routing for arbitrary shared presentation button blocks or the requested `[label](btn:action)` inline-button flow. The related open PR #47891 tracks Feishu `sendPayload` interactive-card delivery for channelData cards, not this full generic inline-button feature.

## Best Possible Solution

Implement native Feishu rendering for shared `MessagePresentation` button blocks in the Feishu plugin, probably by adding `presentationCapabilities` and `renderPresentation` to the Feishu outbound adapter and mapping button values into structured Feishu card interaction envelopes. Add tests that assert Feishu `action`/`button` card elements are generated and that clicked values route through `handleFeishuCardAction`. Keep #13175 open until that implementation lands or maintainers explicitly choose a narrower canonical tracking item.
