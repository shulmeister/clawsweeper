## Summary

Keep #13175 open. Current main still does not implement generic Feishu inline/native interactive card buttons for shared message presentations: Feishu presentation button blocks render as markdown fallback text, Feishu outbound exposes only text/media delivery, and existing card-action handling consumes Feishu-specific envelopes rather than generating generic presentation button callbacks. Related PR #47891 is narrower unmerged sendPayload/native-card payload work, not a shipped fix for this feature.

## Best Possible Solution

Keep #13175 open. Implement Feishu-owned native rendering for shared presentation buttons: add Feishu outbound presentationCapabilities plus renderPresentation or an equivalent sendPayload path, render Schema 2.0 action/button elements for value and URL buttons, wrap callback values in structured Feishu interaction envelopes, route clicks through handleFeishuCardAction, preserve fallback text for unsupported surfaces, and add tests for clickable buttons, URL fallback, unsupported fallback, and callback dispatch. Treat #47891 as related narrower sendPayload/native-card payload work unless maintainers explicitly broaden it.
