## Summary

Keep #69505 open. Current main does not implement an `assistantPrefix`/assistant-prefill contract for cron `agentTurn`, agent defaults, the agent RPC, or CLI flags. The existing `responsePrefix` feature is only a post-generation outbound reply transform, so it does not address the reported failure mode where streamed preamble reaches WhatsApp before the intended final reminder.

## Best Possible Solution

Keep the issue open for maintainer/product design. A proper fix should add an additive assistant-prefill field to cron payload/schema/storage and agent defaults/agent RPC as appropriate, expose CLI/docs for cron and agent runs, and implement/test provider handling for at least Anthropic plus one OpenAI-compatible path while leaving existing `messages.responsePrefix` as a separate outbound formatting feature.
