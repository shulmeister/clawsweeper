## Summary

Keep open. Current main has an idle watchdog that resets on every streamed chunk and tool-call loop detection for repeated/no-progress tool patterns, but I found no semantic n-gram/text repetition breaker for active model text or thinking streams. Current docs/tests also show aborted partial assistant text is intentionally persisted, so the requested context rollback/sanitization behavior is not implemented. Related #39961 is the closest implementation candidate for a streaming text repetition guard, but the provided context says it is still open/unmerged, and #64500 tracks a separate tool ping-pong breaker issue.

## Best Possible Solution

Keep this issue open. The likely maintainer path is to review/adapt #39961 or implement equivalent core agent-stream protection near the stream wrapper/subscription layer: detect repeated text/thinking n-grams while a stream is active, abort with clear liveness/replay metadata, and either omit or quarantine loop-aborted partial assistant content so the next turn is not poisoned. Track #64500 separately because it is about tool-call ping-pong breaker semantics, not streaming text repetition.
