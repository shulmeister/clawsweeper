## Summary

Keep #61694 open. The requested global opt-out is still meaningful core auth-store work on the target main SHA: external CLI auth is still overlaid at runtime, including Codex CLI bootstrap behavior, and there is no `auth.externalCliSync` or `OPENCLAW_AUTH_EXTERNAL_CLI_SYNC` gate in the current code. This is not a ClawHub/plugin close because the issue asks for a central auth-store policy knob, not an optional integration.
