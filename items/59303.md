## Summary

Current `main` explicitly treats rate-limit failures as model-fallback triggers, caps same-provider profile rotations at one by default, and bypasses long SDK `Retry-After` waits so 429s can escape to the fallback path instead of looping on the same provider.

## Evidence

- **Rate-limit escalation in runner:** The embedded runner increments a rate-limit profile-rotation counter and, once the configured cap is exceeded, throws `FailoverError` with `reason: "rate_limit"` to force model fallback.
  - file: [src/agents/pi-embedded-runner/run.ts:608](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/pi-embedded-runner/run.ts#L608)
  - command: `nl -ba src/agents/pi-embedded-runner/run.ts | sed -n '608,632p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Default cap is one same-provider rotation:** `DEFAULT_MAX_RATE_LIMIT_PROFILE_ROTATIONS` is `1`, and `resolveRateLimitProfileRotationLimit(...)` reads that as the default when config does not override it.
  - file: [src/agents/pi-embedded-runner/run/helpers.ts:34](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/pi-embedded-runner/run/helpers.ts#L34)
  - command: `nl -ba src/agents/pi-embedded-runner/run/helpers.ts | sed -n '32,47p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Long Retry-After waits are bypassed:** Provider transport parses `retry-after-ms` / `retry-after` and marks retryable 429/5xx responses as `x-should-retry: false` when the wait exceeds the default 60-second cap, preventing long same-provider SDK retry sleeps from blocking fallback.
  - file: [src/agents/provider-transport-fetch.ts:11](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/provider-transport-fetch.ts#L11)
  - command: `nl -ba src/agents/provider-transport-fetch.ts | sed -n '10,75p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Regression coverage for 429 fallback:** `runWithModelFallback` test coverage includes a documented 429 rate-limit case and asserts the run falls through to the fallback model instead of stopping on the first provider.
  - file: [src/agents/model-fallback.test.ts:1389](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/model-fallback.test.ts#L1389)
