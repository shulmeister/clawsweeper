## Summary

Keep #13583 open. Current main has adjacent guardrail surfaces for pre-tool blocking, one-pass final-answer revision, and outbound message cancellation, but it still lacks the requested fail-closed pre-response responseGate with mandatory tool-call accounting, typed validators, retry/fallback policy, and audit records. The issue discussion also already records complementary external/plugin work while calling out this remaining core/plugin API gap.

## Best Possible Solution

Keep #13583 open as the canonical response-gate policy-as-code design item. The best maintainer path is a versioned core/plugin API for mandatory tool-call accounting, typed response validators, fail-closed final-response gating, bounded retry/fallback behavior, and audit records; domain-specific policy packs can then live in plugins or ClawHub.

## Risks / Open Questions
