## Summary

Keep open. Current main has added a partial `before_agent_finalize` revision hook and existing tool/message hooks, but it still does not implement the requested policy-as-code response gate: no two-phase tool-only planning protocol, no `responseGate`/`require_tools` validator config, no mandatory-tool accounting with retry/fallback semantics, and no general fail-closed pre-response enforcement layer.

## Best Possible Solution

Keep this open as the canonical design item for response-gate policy-as-code. Build on the shipped `before_agent_finalize`, `before_tool_call`, and message hooks, but add a versioned validator contract for mandatory tool-call accounting, retry/fallback semantics, audit output, and tests. Policy packs and service-specific rules can then live in ClawHub plugins once the core/plugin API can enforce them mechanically.
