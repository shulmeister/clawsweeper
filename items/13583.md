## Summary

Keep open. Current main still has only partial final-answer and outbound-delivery hooks, not the requested policy-as-code response gate: there is no two-phase tool-only planning protocol, no `responseGate`/`require_tools` validator config, no mandatory-tool accounting, and no fail-closed pre-response validator layer. The latest issue discussion, including the posted Codex review, also points to this as a remaining core/plugin-API gap rather than something already implemented or suitable for ClawHub alone.

## Best Possible Solution

Keep this open as the canonical response-gate policy-as-code design item. Build on the shipped tool/finalize/message hooks, but add a versioned core/plugin API for mandatory tool-call accounting, typed validators, retry/fallback semantics, fail-closed behavior, audit output, and regression tests. Once that mechanical API exists, service-specific policies and policy packs can live in ClawHub plugins.
