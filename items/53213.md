## Summary

Keep open. Current main does not show a repo-side fix for shifted-space input in the TUI: the composer still delegates ordinary keys to upstream `pi-tui`, there is no shift+space regression coverage, and the only observable change on main is an upstream Pi package bump that is not enough evidence to close this issue as fixed or obsolete.

## Evidence

- **Composer key handling still has no shift+space path:** `CustomEditor.handleInput` only special-cases alt+enter, alt+up, ctrl+l/o/p/g/t/c/d, shift+tab, and escape, then falls through to `super.handleInput(data)` for everything else. There is no explicit handling for shifted-space or tmux/iTerm escape sequences.
  - file: [src/tui/components/custom-editor.ts:16](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/tui/components/custom-editor.ts#L16)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **No regression test for shifted-space:** The current `CustomEditor` tests only assert `alt+enter` and `alt+up` routing. There is no test covering plain space, shifted-space, or tmux/iTerm-specific key sequences.
  - file: [src/tui/components/custom-editor.test.ts:11](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/tui/components/custom-editor.test.ts#L11)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **No repo-side input-handler change since latest release:** Diffing the latest release against current main for the TUI input files shows no changes in `src/tui/components/custom-editor.ts` or `src/tui/tui.ts`; the command `git diff --name-only v2026.4.22..8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf -- src/tui/components/custom-editor.ts src/tui/tui.ts` returned no paths.
  - command: `git diff --name-only v2026.4.22..8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf -- src/tui/components/custom-editor.ts src/tui/tui.ts`
- **Only clear main-vs-release change is upstream Pi dependency bump:** Current main uses `@mariozechner/pi-tui` `0.70.0`, while tag `v2026.4.22` used `0.69.0`. That may matter, but from the checked-out repo alone it is not strong evidence that this specific shifted-space bug is fixed.

## Risks / Open Questions

- `@mariozechner/pi-tui` may have fixed this upstream between 0.69.0 and 0.70.0, but that is not provable from the repository contents reviewed here.
- The report is environment-specific (`macOS`, `iTerm2`, `tmux -CC`), and this read-only review did not reproduce that terminal path directly.
