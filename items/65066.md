## Summary

Keep this PR open. Current main still has no memory runtime/capability registration in `memory-lancedb`, while doctor/status still resolve memory health through the active memory runtime. The PR is a focused implementation candidate for the linked doctor/status false-negative, and the review-thread concern about a lightweight embedding probe was addressed in the PR discussion rather than making the PR obsolete.

## Best Possible Solution

Keep the PR open as the implementation candidate. Rebase it onto current main, review the runtime adapter against the current memory SDK contract, keep the regression coverage in `extensions/memory-lancedb/index.test.ts`, then land or request targeted changes. Closing now would drop a still-missing plugin-owned fix for doctor/status visibility.
