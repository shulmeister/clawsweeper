## Summary

Keep open. Current main still has the reported mismatch: `openclaw memory status` scans session totals by counting every `*.jsonl`, while session indexing uses the usage-counted transcript helper that excludes compaction checkpoint snapshots. The reporter's suggested boundary matches the current indexing contract and the confirming comment's design rationale.

## Best Possible Solution

Keep the issue open and fix the memory CLI status scan to share the same session transcript eligibility contract as indexing, preferably by using `isUsageCountedSessionTranscriptFileName` in `scanSessionFiles`. Add a focused regression test for a sessions directory containing one normal session transcript plus one `.checkpoint.<uuid>.jsonl`, asserting the status total matches the indexed session count.
