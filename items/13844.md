## Summary

Current `main` does not appear to implement a channel-level pre-routing message drop/filter feature for Telegram or other channels. Telegram config currently exposes access-control, mention-gating, and `ingest` hooks, but no `inputFilter`/`ignorePatterns`-style field, and the inbound Telegram body path still forwards ordinary text like `/start` unless it is blocked as an unauthorized control command or skipped by mention gating.

## Evidence

- **Telegram config schema has no message-pattern drop option:** `TelegramTopicSchema`, `TelegramGroupSchema`, and the top-level Telegram channel schema define fields such as `requireMention`, `ingest`, `disableAudioPreflight`, `allowFrom`, `groupPolicy`, and `customCommands`, but no `inputFilter` or `ignorePatterns` field.
  - file: [src/config/zod-schema.providers-core.ts:115](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/config/zod-schema.providers-core.ts#L115)
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **Public Telegram config types only describe mention-skip ingest, not inbound drop rules:** The exported Telegram config types document `ingest` as emitting internal hooks for mention-skipped messages and include no config surface for silently dropping matching inbound text patterns before routing.
  - file: [src/config/types.telegram.ts:223](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/config/types.telegram.ts#L223)
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **Telegram inbound body path still forwards normal text:** `resolveTelegramInboundBody()` builds `rawBody` from inbound message text, returns `null` only for unauthorized control commands or mention-skipped group messages, and otherwise returns `{ bodyText, rawBody, ... }` for downstream processing.
  - file: [extensions/telegram/src/bot-message-context.body.ts:148](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/extensions/telegram/src/bot-message-context.body.ts#L148)
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **Body result is consumed by the Telegram message context pipeline:** `buildTelegramMessageContext` calls `resolveTelegramInboundBody()`, aborts only when it returns null, and otherwise carries `rawBody`/`bodyText` forward into the Telegram message context used for reply handling.
  - file: [extensions/telegram/src/bot-message-context.ts:421](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/extensions/telegram/src/bot-message-context.ts#L421)
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **No built-in `/start` command handling found:** Control-command detection only recognizes registered chat command aliases. The builtin command registry includes `/help`, `/commands`, `/tools`, `/skill`, and `/status`, and a repo search for `nativeName: "start"` or `textAlias: "/start"` returned no matches in `src/auto-reply`, `extensions/telegram`, `docs/channels/telegram.md`, or `docs/help/faq.md`.
  - file: [src/auto-reply/commands-registry.shared.ts:128](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/auto-reply/commands-registry.shared.ts#L128)

  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **No existing `ignorePatterns` or `inputFilter` implementation found:** A repo search for `ignorePatterns` or `inputFilter` returned no matches under `src`, `docs`, or `extensions`.
