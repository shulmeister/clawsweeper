## Summary

Keep open. Current main does not implement `smartExtraction` or an LLM-backed atomic memory extractor for `memory-lancedb`, and the PR is a plausible feature rather than obsolete. The PR also has substantive unresolved review blockers around manifest schema coverage, capture safety filters, untrusted memory prompt input, and silent extraction failures, so it should be revised rather than auto-closed.

## Best Possible Solution

Keep the PR open, but require a rebase and a maintainer-quality revision before merge: add manifest schema/UI hint/docs coverage for `smartExtraction`, preserve `shouldCapture` and `captureMaxChars` before any LLM call, treat existing memories as untrusted prompt data, make extraction failures observable or fall back safely, and add targeted tests for config validation, safety filtering, prompt-injection handling, and failure behavior.
