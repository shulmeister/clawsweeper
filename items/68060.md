## Risks / Open Questions

- Merging this PR as-is would expose a new `smartExtraction` config that the current manifest schema does not allow, causing plugin-load failure.
- The smart-extraction path appears to bypass existing capture guardrails (`captureMaxChars` and prompt-injection filtering), which is a behavioral regression from current `main`.
- The extractor prompt appears to interpolate existing stored memories verbatim, creating a new prompt-injection surface from user-controlled memory content.
- Extraction failures appear to degrade to empty output, which would make auto-capture silently stop working and be hard to diagnose.
