## Summary

Current `main` already implements the requested behavior: usage normalization accepts OpenAI-style `prompt_tokens` / `completion_tokens`, session usage/status paths consume that normalized shape, and `/status`/`session_status` explicitly fall back to transcript-derived usage when the live session snapshot is sparse.
