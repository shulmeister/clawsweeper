## Summary

Keep open. Current `main` still treats Feishu `@所有人` / `@_all` as satisfying mention gating, and the Feishu config/docs only expose `requireMention`, not an `ignoreAtAll` or `ignoreBroadcast` override. The timeline’s linked PR #59422 is useful evidence that this is a real implementation gap, but it is not implemented on `main` yet.

## Evidence

- **`@_all` still counts as a bot mention on `main`:** `checkBotMentioned` returns `true` whenever the raw message content contains `@_all`, before checking whether the bot itself was mentioned. That matches the reported behavior where `requireMention: true` still responds to `@所有人` broadcasts.
  - file: [extensions/feishu/src/bot-content.ts:233](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/feishu/src/bot-content.ts#L233)
  - command: `nl -ba extensions/feishu/src/bot-content.ts | sed -n '229,240p'`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Inbound gating still hinges on `requireMention` plus `ctx.mentionedBot`:** The group-message path resolves `requireMention` and drops only when `requireMention && !ctx.mentionedBot`. There is no separate branch to ignore broadcast mentions while still honoring direct bot mentions.
  - file: [extensions/feishu/src/bot.ts:466](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/feishu/src/bot.ts#L466)

- **Schema has no `ignoreAtAll` / `ignoreBroadcast` setting:** The Feishu group schema and shared Feishu config expose `requireMention` but no broadcast-specific ignore flag at either channel or group scope.
  - file: [extensions/feishu/src/config-schema.ts:141](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/feishu/src/config-schema.ts#L141)
