## Summary

Keep #61484 open. Current main still leaves the agent-facing `sessions_send` tool text-only: its schema accepts only session target fields, `message`, and timeout, and the execution path forwards only `message` into the nested run. There is related media plumbing elsewhere (`openclaw message send --media` and `sessions.send` RPC attachments), but that does not satisfy the requested `sessions_send` media/path parity.
