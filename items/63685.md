## Summary

Keep open. Current main documents and tests Gemma 4 support for the corrected Google model id `gemma-4-26b-a4b-it`, but the reported failing id `gemma-4-26b` is still accepted as a forward-compatible Gemma id and sent through unchanged. The related #68996 PR is open and adjacent; it is not a shipped fix for this setup/model-id failure.

## Best Possible Solution

Keep this issue open and fix the setup/model-id path directly: confirm whether terminal onboarding or model selection can offer `gemma-4-26b`, remove or correct that choice to `gemma-4-26b-a4b-it`, or add provider-side validation with a friendly error before dispatching to Google. Add regression coverage for the reported `gemma-4-26b` selection path, separate from the already-covered corrected Gemma 4 id.
