## Summary

Keep #29552 open. Current main has adjacent update/service-token diagnostics and device-token retry behavior, but it still preserves the user-visible newer-config warning for truly newer versions and ordinary local gateway call paths still let OPENCLAW_GATEWAY_TOKEN override gateway.auth.token with no retry using local config credentials after a shared-token mismatch. The issue discussion also has a recent 2026-04-22 report that the bug persisted, and the directly linked fallback-retry PR #29676 was closed unmerged.

## Best Possible Solution

Keep the issue open until maintainers either implement the missing local auth fallback/recovery for token mismatch, decide that env-first local call precedence is the intended product boundary and document the manual repair path, or split the remaining config-version warning/update mismatch symptoms into a narrower canonical issue. The most direct code path is to add a guarded local-only retry in src/gateway/call.ts that falls back to resolved gateway.auth token/password after AUTH_TOKEN_MISMATCH, with tests covering stale OPENCLAW_GATEWAY_TOKEN and no behavior change for remote URL or explicit auth overrides.
