## Summary

Keep #29552 open. Current main has related diagnostics and a trusted device-token retry path, but ordinary local gateway calls still resolve credentials once with env-first precedence and do not retry with `gateway.auth` credentials after `AUTH_TOKEN_MISMATCH`. The issue thread also has recent confirmations that the mismatch persisted, and the directly related fallback PR #29676 was closed unmerged.

## Best Possible Solution

Keep this open. The best product path is either a guarded local-only retry in `src/gateway/call.ts` that falls back to resolved `gateway.auth` token/password after `AUTH_TOKEN_MISMATCH` when no remote URL or explicit auth override is involved, with regression coverage for stale `OPENCLAW_GATEWAY_TOKEN`, or an explicit maintainer decision that env-first local call precedence is intended plus clearer repair docs/doctor guidance. If the version-stamp/update mismatch is now considered separate from auth recovery, split that into a narrower canonical issue instead of closing this one.
