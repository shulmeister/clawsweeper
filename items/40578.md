## Summary

PR #40578 should stay open. Current main still has the memory-lancedb cross-agent global table behavior the PR is trying to address, and the PR is a focused attempt against a bundled memory plugin. It is not implemented on main, not cleanly superseded by the closed related issues, and not a ClawHub-only request. The right path is to fix the PR’s review-blocking correctness issues and land a maintainer-approved implementation.

## Best Possible Solution

Keep the PR open as a focused bundled memory-lancedb feature. The best path is to update the PR to use existing hook/tool `agentId` context consistently for store/search/delete, add an opt-in backward-compatible config and schema/migration, address the bot review findings, and add regression coverage for two-agent isolation plus legacy global memory visibility. No new core API appears necessary from current main.
