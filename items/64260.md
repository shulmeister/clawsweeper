## Summary

Keep this issue open. Current main still does not thread context window size, max output tokens, or utilization into the `llm_output` hook payload; the available model metadata and resolved context budget are present elsewhere but not exposed on this event. Related #64327 and #64268 remain open and are complementary, not shipped replacements.

## Best Possible Solution

Keep the issue open and implement this as a plugin hook contract change: extend `PluginHookLlmOutputEvent` with stable model metadata such as native `contextWindow`, `maxTokens`, and the resolved effective context budget, then compute utilization from usage totals where reliable. Populate the fields in the embedded runner, CLI harness, and Codex harness, add focused hook contract/emission tests, and update the public plugin hooks docs. Coordinate names with #64327 so native context window and effective `contextTokens` do not drift into two incompatible API shapes.
