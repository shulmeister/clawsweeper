## Summary

Close as ClawHub/plugin-surface work. The PR adds parallel internal tool hooks, but current main already supports the useful behavior through the public plugin hook API: plugins can rewrite, block, require approval before tool calls, and observe results afterward. The proposed PR also has a real design bug noted in review: internal hooks return void, so its deny/block path cannot work as written.

## Best Possible Solution

Close this PR and keep tool-call policy, approval, rewriting, and observation on the supported plugin hook surface. A tool preflight/audit plugin can use before_tool_call and after_tool_call today and be published through ClawHub; any remaining gap should be filed as a focused plugin API issue rather than adding parallel internal hooks.
