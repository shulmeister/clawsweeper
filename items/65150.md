## Summary

Keep open. Current `main` does not implement `tool:beforeExecute` / `tool:afterExecute` internal hooks, and the PR head still has confirmed correctness bugs in the proposed implementation, so there is no conservative close path here.

## Evidence

- **Current main still lacks a `tool` internal hook event type:** `InternalHookEventType` on `main` is still limited to `command | session | agent | gateway | message`, so the requested surface is not already implemented.
  - file: [src/hooks/internal-hook-types.ts:1](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/src/hooks/internal-hook-types.ts#L1)
  - command: `nl -ba src/hooks/internal-hook-types.ts | sed -n '1,220p'`
  - sha: [ae5c6573670b](https://github.com/openclaw/openclaw/commit/ae5c6573670b3742b156899bd2b9c36b8677f191)
- **Current hook API cannot return deny decisions:** `InternalHookHandler` returns `Promise<void> | void`, and `triggerInternalHook` returns `Promise<void>`, so there is no existing result channel for a hook to return `{ permissionDecision: "deny" }`. That matches the review finding that the PR's deny branch is unreachable.
  - file: [src/hooks/internal-hook-types.ts:18](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/src/hooks/internal-hook-types.ts#L18)
  - command: `nl -ba src/hooks/internal-hook-types.ts | sed -n '1,220p'`
  - sha: [ae5c6573670b](https://github.com/openclaw/openclaw/commit/ae5c6573670b3742b156899bd2b9c36b8677f191)
- **Current event scheduler depends on returned promises to preserve ordering:** `createEmbeddedPiSessionEventHandler()` only serializes a handler when it returns a promise; otherwise it returns immediately. The PR review correctly notes that making `handleToolExecutionStart` async work but dropping its promise would let later tool events run before start bookkeeping finishes.
  - file: [src/agents/pi-embedded-subscribe.handlers.ts:26](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/src/agents/pi-embedded-subscribe.handlers.ts#L26)
  - command: `nl -ba src/agents/pi-embedded-subscribe.handlers.ts | sed -n '1,180p'`

- **Current main start handler is synchronous bookkeeping with no tool hook wiring:** `handleToolExecutionStart()` on `main` performs tool-start tracking and emits events, but it does not call `createInternalHookEvent()` or `triggerInternalHook()` for tool execution. That rules out `implemented_on_main`.
  - file: [src/agents/pi-embedded-subscribe.handlers.tools.ts:559](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/src/agents/pi-embedded-subscribe.handlers.tools.ts#L559)
