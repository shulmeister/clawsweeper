## Summary

Keep PR #42824 open. The linked IRC truncation bug #42121 is closed, but current `origin/main` still has the exact behavior this PR changes: IRC passes `disableBlockStreaming: undefined` when `channels.irc.blockStreaming` is unset, and the shared reply directive code then allows `agents.defaults.blockStreamingDefault: "on"` to enable block streaming. The PR remains a focused, plausible fix with missing regression coverage on main.

## Best Possible Solution

Keep the PR open and either merge its scoped IRC helper/test approach after normal maintainer validation, or land an equivalent current-main fix that makes IRC block streaming require explicit `channels.irc.blockStreaming: true` while preserving per-account opt-in behavior.
