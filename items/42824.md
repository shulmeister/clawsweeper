## Summary

Keep open. Current `main` still falls back to the global agent block-streaming default when `channels.irc.blockStreaming` is unset, so this PR's behavior is not implemented on `main` and the report remains plausible and aligned with current docs.

## Evidence

  - file: [extensions/irc/src/inbound.ts:361](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/extensions/irc/src/inbound.ts#L361)

- **Global reply directives treat undefined as fallback to agent default:** When `opts.disableBlockStreaming` is undefined, reply directives resolve block streaming from `agentCfg.blockStreamingDefault`, exactly the inheritance path described in the PR.
  - file: [src/auto-reply/reply/get-reply-directives.ts:437](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/auto-reply/reply/get-reply-directives.ts#L437)
