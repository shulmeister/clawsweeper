## Summary

Current `main` no longer uses the temp-tar `podman save ... -o` then `podman load` flow from the report. The Podman setup path was replaced with a rootless-user model that builds or pulls the image directly into the current user's Podman store, so the reported image-load bottleneck is already gone on `main`.
