## Summary

Keep open. Current main has useful adjacent cleanup controls, including `session.maintenance`, `openclaw sessions cleanup`, and a `sessions_spawn.cleanup` option, but the reported ACPX default remains: one-shot ACP runs close the runtime handle without discarding persistent ACPX state, and `sessions_spawn` still defaults cleanup to `keep`. The linked #59555/#33069 work fixed pairing and routing context, not retention; #62436 is related storage pressure but tracks a different HTTP passthrough/stateless mode request.
