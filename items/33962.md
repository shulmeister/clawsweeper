## Summary

Keep open. Current main still uses the default agent's effective primary model for session-memory slug generation, with only the general `agents.defaults.timeoutSeconds` mitigation present. There is no slug-specific lightweight model config, no fast 3-second heuristic fallback, and no debounce/concurrency change covering the central request. The provided discussion remains relevant: #34644/#66237 address timeout behavior, PR #65492 was timeout-only and unmerged, and #71709 tracks a separate slug-generator failure-classification problem.

## Best Possible Solution

Keep #33962 open until maintainers either implement a dedicated slug-generation model surface, such as `agents.defaults.slugGenerator.model`/timeout docs and tests, or explicitly choose another product direction. The likely implementation path is `src/hooks/llm-slug-generator.ts` plus config schema/help/generated docs and regression tests covering model selection, timeout/fallback behavior, and the existing `llmSlug` disable path.
