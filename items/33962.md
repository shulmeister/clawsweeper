## Summary

Keep open. Current main still routes session-memory slug generation through the default agent's resolved primary/default model and the normal embedded-agent queue path. Main has a timeout mitigation, but it does not provide a lightweight slug-specific model, fast heuristic timeout, or debounce/non-main lane behavior requested by #33962.

## Best Possible Solution

Keep #33962 open. A good maintainer path is to add a slug-specific configuration such as agents.defaults.slugGenerator.model (plus optional timeoutSeconds if desired), resolve it through the existing model-ref/config schema machinery, pass it into generateSlugViaLLM, and add tests for default-model fallback, lightweight override, invalid/missing config behavior, and timestamp fallback. Consider also moving slug generation off the main global lane or adding a short heuristic fallback/debounce so simultaneous resets cannot block user-facing turns.
