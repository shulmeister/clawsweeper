## Summary

Keep #33962 open. Current main still routes session-memory slug generation through the default agent model and the embedded runner's session/global lanes. The only current mitigation I found is timeout handling: no dedicated slug model, no early heuristic fallback before awaiting the LLM helper, and no debounce or lane decoupling for concurrent slug requests.

## Best Possible Solution

Keep #33962 open for a focused core fix. Add a slug-specific lightweight model override to the config contract, wire it through src/hooks/llm-slug-generator.ts, preserve the existing hooks.internal.entries.session-memory.llmSlug opt-out, and add regression coverage for model selection and fallback behavior. A short early heuristic fallback or debounce can be considered as an additional mitigation, but the central fix is decoupling slug generation from the user-facing primary model path and normal lane congestion.
