## Summary

Keep #33962 open. Current main still uses the default agent's effective primary model for session-memory slug generation, queues the run through the normal embedded runner lanes with `sessionKey: "temp:slug-generator"`, and only falls back to a timestamp after awaiting the LLM call. The generic `agents.defaults.timeoutSeconds` mitigation exists, but there is still no slug-specific lightweight model config, fast heuristic timeout/fallback, or debounce/concurrency behavior that addresses the central lane-congestion request.

## Best Possible Solution

Keep #33962 open until maintainers add an explicit product path for slug generation. The likely fix belongs around `src/hooks/llm-slug-generator.ts`, `src/hooks/bundled/session-memory/handler.ts`, and agent-default config/schema/help/docs: add a slug-specific lightweight model override such as `agents.defaults.slugGenerator.model`, consider a short slug-specific fallback deadline, and add regression tests for model selection, fallback timing, lane behavior, and the existing `llmSlug: false` opt-out.
