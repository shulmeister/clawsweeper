## Summary

Current `main` has partial discoverability for sub-agent targeting via `agents_list`, but it does not implement the requested sibling-agent roster in the session prompt or a tool that exposes peer role/theme plus agent-to-agent messaging eligibility. The issue should stay open.

## Evidence

- **`agents_list` is limited to sub-agent spawn discovery:** The tool description and implementation are scoped to `sessions_spawn` allowlists and only return `id`, optional `name`, and `configured` for allowed agents.
  - file: [src/agents/tools/agents-list-tool.ts:15](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/agents/tools/agents-list-tool.ts#L15)
  - command: `nl -ba src/agents/tools/agents-list-tool.ts | sed -n '1,140p'`

- **System prompt only advertises generic tools, not a sibling roster:** The session system prompt lists generic summaries for `agents_list`, `sessions_list`, `sessions_send`, and related tools, but it does not inject configured sibling-agent metadata.
  - file: [src/agents/system-prompt.ts:497](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/agents/system-prompt.ts#L497)
