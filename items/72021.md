## Summary

Keep open. Current main still enforces the deep promotion `minRecallCount` gate with aggregate `signalCount`, and that aggregate still includes recall, daily, and grounded signals. Tests and docs also encode dreaming/daily/grounded evidence as promotion evidence, so this remains an active product semantics issue rather than a cleanup-close candidate.

## Best Possible Solution

Keep this issue open for maintainer product judgment. If accepted, split the deep promotion semantics in `extensions/memory-core/src/short-term-promotion.ts`: make `minRecallCount` check real `recallCount` only, keep daily/grounded/session evidence under a clearly named aggregate signal metric, recalibrate scoring/default thresholds so recall-only candidates can pass, and update `extensions/memory-core/src/short-term-promotion.test.ts`, dreaming diagnostics, and `docs/concepts/dreaming.md` to match the chosen behavior.
