## Summary

Keep open. Current main has diagnostics stability/export and OpenTelemetry queue metrics, but it still does not provide the PR's central capability: a live queue diagnostics snapshot exposed as `diagnostics.queue` and `openclaw gateway diagnostics queue`. The related #9797 remains a separate agent-facing `queue_status` request, and this PR explicitly scoped that out rather than duplicating it.

## Best Possible Solution

Keep this open or ask for an updated PR that adds a minimal core-owned, read-only queue snapshot seam plus `diagnostics.queue` and `openclaw gateway diagnostics queue`, aligned with current diagnostics stability/export behavior. Track the in-agent `queue_status` tool separately under #9797.
