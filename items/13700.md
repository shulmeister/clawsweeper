## Summary

Keep #13700 open. Current main still has adjacent compaction checkpoint branch/restore and native subagent transcript forking, but it does not implement the requested arbitrary named session snapshot surface: no `/session save`, `/session load`, `/session restore`, named snapshot Gateway API, reusable session template flow, or restore path for `.reset` / `.deleted` transcript archives. The latest issue discussion also contains a maintainer-facing Codex review explicitly keeping this open.

## Best Possible Solution

Keep #13700 open until OpenClaw either implements a deliberate native session snapshot surface or splits it into canonical scoped issues. A strong implementation would add typed Gateway snapshot save/list/load/delete methods, slash/TUI/CLI controls, deterministic transcript/model/session metadata preservation where feasible, spawn-from-snapshot support for reusable batch workflows, and a safe restore path for `.reset` / `.deleted` transcript archives.
