## Summary

Keep #13700 open. Current main has adjacent partial work, including `sessions_spawn` with `context:"fork"` and Gateway compaction checkpoint branch/restore, but the requested user-facing native snapshot surface is still missing: no `/session save`, `/session load`, named snapshot API, reusable session template, or restore path for `.reset`/`.deleted` archives.

## Best Possible Solution

Keep #13700 open until OpenClaw either implements a native session snapshot surface or splits the work into canonical issues. A strong path is typed Gateway methods for named snapshot save/list/load/delete, slash/TUI controls, model/session metadata preservation where deterministic, and a deliberate restore path for `.reset`/`.deleted` transcript archives with safety checks. Keep the existing `sessions_spawn context:"fork"` and compaction checkpoint branch/restore as related partial capabilities, not as a closure of this issue.
