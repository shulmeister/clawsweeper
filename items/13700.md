## Summary

Keep #13700 open. Current main has partial related work: native `sessions_spawn` can fork the requester transcript, and the gateway can branch/restore automatically-created compaction checkpoints. That positively addresses the sub-agent fork/cache discussion in the issue, but the original user-facing request remains meaningfully missing: there is no `/session save <name>` or `/session load <name>` command, no general named session snapshot API, and no native restore path for existing `.reset`/`.deleted` transcript archives.
