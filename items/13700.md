## Summary

Keep #13700 open. Current main still lacks the requested native session snapshot surface: no `/session save`, `/session load`, named arbitrary snapshot API, reusable session template, or restore path for `.reset`/`.deleted` transcript archives. The adjacent compaction checkpoint branch/restore APIs and `sessions_spawn` `context:"fork"` are narrower and do not cover the central request; the latest maintainer comment in the issue also explicitly kept it open.

## Best Possible Solution

Keep #13700 open until OpenClaw implements a deliberate native session snapshot surface or splits it into canonical scoped issues. A strong path is typed Gateway save/list/load/delete snapshot methods, slash/TUI controls, deterministic preservation of transcript/model/session metadata where feasible, spawn-from-snapshot support for reusable batch work, and a safe restore path for `.reset`/`.deleted` transcript archives.
