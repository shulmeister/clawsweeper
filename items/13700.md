## Summary

Keep open. Current `main` has partial adjacent capabilities, but it does not implement the requested native OpenClaw feature set of explicit session snapshots with `/session save|load` or restore-from-archived-session files. What exists today is narrower: compaction checkpoints (`sessions.compaction.branch|restore`), subagent transcript forking, and ACP-only `resumeSessionId`.

## Evidence

- **No `/session save|load|restore` user command on current main:** The current built-in slash-command list includes `/new`, `/reset`, `/compact`, and `/session idle|max-age`, but no `/session save`, `/session load`, or `/session restore`. The TUI docs likewise document `/session <key>` only as a session switcher.
  - file: [docs/tools/slash-commands.md:91](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/docs/tools/slash-commands.md#L91)
  - command: `nl -ba docs/tools/slash-commands.md | sed -n '85,125p'`
- **`/session` is only session selection in TUI:** The TUI command list documents `/session <key>` (or `/sessions`) for switching sessions, not snapshot save/load semantics.
  - file: [docs/web/tui.md:105](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/docs/web/tui.md#L105)
  - command: `nl -ba docs/web/tui.md | sed -n '100,120p'`
- **Gateway API has compaction checkpoint methods, not generic session snapshot save/load:** The registered session methods are `sessions.compaction.list|get|branch|restore`, plus `sessions.create|send|abort|patch|reset|delete|compact`. There is no `sessions.save`, `sessions.load`, or general native session-restore method.
  - file: [src/gateway/server-methods-list.ts:82](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/gateway/server-methods-list.ts#L82)
  - command: `nl -ba src/gateway/server-methods-list.ts | sed -n '76,94p'`
- **Session schema confirms restore is limited to compaction checkpoints:** The protocol schema defines `SessionsCompactionBranchParamsSchema` and `SessionsCompactionRestoreParamsSchema`, both keyed by `checkpointId`. There is no schema for named snapshot save/load or restore of `.reset`/`.deleted` archives.
  - file: [src/gateway/protocol/schema/sessions.ts:201](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/gateway/protocol/schema/sessions.ts#L201)
  - command: `nl -ba src/gateway/protocol/schema/sessions.ts | sed -n '150,240p'`
- **Implementation is specifically pre-compaction checkpoint branch/restore:** `sessions.compaction.branch` and `sessions.compaction.restore` require `checkpoint.preCompaction.sessionFile` and fork from that pre-compaction transcript. This is a narrow recovery path for compaction checkpoints, not arbitrary explicit checkpoints at any moment or named save/load snapshots.
  - file: [src/gateway/server-methods/sessions.ts:976](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/gateway/server-methods/sessions.ts#L976)
  - command: `nl -ba src/gateway/server-methods/sessions.ts | sed -n '976,1198p'`
- **Native transcript forking exists only for subagent children:** Current docs describe `sessions_spawn` with `context: "fork"` as branching the requester's current transcript into a child subagent session. That is useful, but it is not a save/load checkpoint for the current session.
  - file: [docs/tools/subagents.md:75](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/docs/tools/subagents.md#L75)
  - command: `nl -ba docs/tools/subagents.md | sed -n '70,125p'`
- **Resume/load exists only for ACP runtime sessions, not native OpenClaw sessions:** ACP sessions support `resumeSessionId`, which replays history via `session/load`, and the docs explicitly say it requires `runtime: "acp"`. That leaves the requested native OpenClaw session snapshot/load workflow unimplemented.
  - file: [docs/tools/acp-agents.md:306](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/docs/tools/acp-agents.md#L306)
  - command: `nl -ba docs/tools/acp-agents.md | sed -n '300,375p'`
