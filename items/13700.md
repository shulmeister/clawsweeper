## Summary

Keep #13700 open. Current main has related partial work: native `sessions_spawn` can fork the requester transcript into a child session, and Gateway RPC can branch/restore compaction checkpoints. But the original user-facing request remains missing: no `/session save`, `/session load`, named snapshot API, reusable session template, or restore path for `.reset`/`.deleted` transcript archives.

## Best Possible Solution

Keep #13700 open until OpenClaw implements a native session snapshot surface or maintainers split it into canonical follow-ups. A good implementation would add typed Gateway methods for named checkpoint save/list/load/delete, wire user-facing slash/TUI controls, preserve model/session metadata where deterministic, and add a deliberate restore path for `.reset`/`.deleted` transcript archives with clear safety checks.
