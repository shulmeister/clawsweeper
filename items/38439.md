## Summary

Keep open. Current main partially addresses this area by resolving avatars from workspace IDENTITY.md and surfacing size/reason metadata, which matches the reporter's later <2 MB finding, but it still lacks source/test proof for the reported non-default Windows webchat flow. The related open PR #38923 explicitly tracks the remaining IDENTITY.md-only/backtick/non-default-agent work, so closing this issue as fixed would be premature.

## Best Possible Solution

Keep #38439 open until #38923 or an equivalent maintainer-reviewed fix lands. The best fix is narrow: normalize Windows namespaced and drive-letter paths before avatar workspace containment checks, strip single backtick wrappers from IDENTITY.md avatar values, add regression coverage for a non-default agent such as gaia resolving a workspace-relative IDENTITY.md avatar through the Control UI/webchat route, and keep the existing 2 MB size-limit metadata/UI explanation.
