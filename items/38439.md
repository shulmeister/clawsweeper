## Summary

Keep open. Current main has the avatar route, IDENTITY.md fallback, metadata, and 2 MB avatar-size reporting, but it still contains the Windows path-containment behavior called out in the discussion and lacks the non-default-agent/backtick regression coverage tracked by open PR #38923.

## Best Possible Solution

Keep #38439 open until #38923 or an equivalent maintainer-reviewed fix lands. The best fix is to normalize Windows namespaced and drive-letter paths before avatar containment checks, strip single-backtick wrappers from IDENTITY.md avatar values, add a non-default-agent Control UI/webchat regression for a workspace-relative IDENTITY.md avatar, and preserve the existing `too_large` metadata/UI explanation.
