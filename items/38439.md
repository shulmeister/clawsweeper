## Summary

Keep open. Current main has some avatar handling improvements, including IDENTITY.md fallback, metadata, and a 2 MB local avatar limit, but it still does not fully cover the reported/non-default Windows webchat path or the later oversized-avatar UX concern. The linked open PR #38923 still explicitly tracks this issue, and current tests do not cover its stated backtick parsing or gaia/non-default flow.

## Best Possible Solution

Keep #38439 open until maintainers either merge/adapt the relevant parts of PR #38923 or explicitly split/close the remaining work. The practical fix path is to add regression coverage for non-default agent IDENTITY.md avatars, Windows namespaced path containment, and backtick-wrapped Avatar values, then improve the oversized-avatar path so users get a clear size-limit error instead of a broken avatar/404 symptom.
