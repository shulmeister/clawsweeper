## Summary

Current main already ships the effective fix: gateway/node host exec now defaults to `security=full` with `ask=off`, so the Windows allowlist execution-plan path that produced this report is no longer the default behavior. The reporter also confirmed the issue was resolved by switching the session to `security=full`.

## Evidence
