## Summary

Keep open. Current main still has the reported behavior: provider catalog discovery only gets a timeout in live mode, returns null in normal runtime, and then awaits the catalog run without a client-side deadline. The PR appears to target a real remaining agents/provider-discovery gap, and the latest discussion includes security-review concerns on the PR implementation that need maintainer judgment rather than cleanup closure.
