## Summary

Keep PR #39644 open. Current main still advertises PowerShell completion but does not install it, still emits POSIX `source` syntax for the completion profile/reload hint, and still hard-codes `powershell` for Windows time-format detection instead of using the existing PowerShell resolver that prefers PS7.

## Best Possible Solution

Keep PR #39644 open, or supersede it only with a landed port. The fix should update `src/cli/completion-runtime.ts` to install PowerShell completion via `. "..."`, emit a PowerShell-safe reload hint, update `src/agents/date-time.ts` to call `resolvePowerShellPath()` with non-interactive flags, and add targeted regression tests for the Windows/PowerShell paths.
