## Summary

Keep PR #39644 open. Current main still has the two Windows behaviors the PR fixes: PowerShell is listed as a supported completion shell, but `installCompletion()` only installs zsh/bash/fish and still returns the unsupported message for `powershell`; Windows time-format detection still calls a hard-coded `powershell` executable instead of the existing resolver that prefers PowerShell 7. The PR discussion also surfaced a valid reload-hint issue, and the contributor added a follow-up commit for it, so the useful path is to port/rebase the patch onto the current `completion-runtime.ts` layout rather than close it.

## Best Possible Solution

Keep the PR active and port the useful changes to current main: add a PowerShell branch in `src/cli/completion-runtime.ts`, write PowerShell dot-source syntax and a PowerShell-safe reload hint, switch `src/agents/date-time.ts` to `resolvePowerShellPath()` with non-interactive flags, and add focused regression tests for PowerShell install/idempotence plus the Windows time-format executable path. Once that lands, this PR can close as merged or superseded by the landed port.
