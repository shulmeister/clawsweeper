## Summary

Current main implements the bundled plugin runtime-dependency staging fix that addresses the reported 2026.4.24 ENOTEMPTY failures under ~/node_modules. The issue discussion already pointed at the runtime-deps localization work, and current source, tests, docs, and changelog entries confirm the dependency install path is now OpenClaw-managed rather than the user/global node_modules tree.

## Best Possible Solution

Close this issue as implemented on current main. Keep the existing managed bundled-runtime-deps staging path and regression tests; if ENOTEMPTY still reproduces after the next stable release, ask for a new report with fresh startup logs, install method, OPENCLAW_PLUGIN_STAGE_DIR or systemd STATE_DIRECTORY details, and the affected channel/plugin list.
