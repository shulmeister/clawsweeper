## Summary

Keep open. Current main still does not implement `tools.exec.nsSandbox` or a bubblewrap wrapper path for safeBins-approved gateway exec commands. Main has stronger safeBins path/profile hardening than the PR originally described, but it does not provide the PR's requested kernel-level filesystem isolation, and this is plausible core exec/security behavior rather than conservative cleanup fodder.

## Best Possible Solution

Keep the PR open for maintainer/security judgment. The best path is a narrowed, rebased core implementation of optional Linux bubblewrap isolation for safeBins-approved gateway exec, wired through config types, strict schema, docs/help labels, gateway allowlist evaluation, runtime spawning, and tests. If maintainers decide the current safeBins hardening plus full sandbox backends are sufficient, that should be an explicit product/security decision rather than conservative cleanup closure.
