## Summary

Keep open. Current main still does not implement the PR's bubblewrap namespace sandbox or the proposed `tools.exec.nsSandbox` config surface. Main does have stronger safeBins path hardening now, which weakens the original `/usr/bin/rm` bypass motivation, but it does not provide kernel-level filesystem isolation. This remains plausible core exec/security hardening that needs maintainer product/security judgment, not a conservative auto-close.

## Best Possible Solution

Keep the PR open for maintainer judgment. The best path is either a narrowed/rebased core implementation of optional bwrap isolation for safeBins-approved gateway exec, with config types/schema/docs/tests and resolved review issues, or an explicit maintainer decision that current safeBins hardening is sufficient and namespace isolation should not be added.
