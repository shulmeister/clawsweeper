## Summary

Current `main` still routes `/hooks/agent` through an isolated cron-style agent run. The requested existing-session delivery mode is not implemented in code, types, or docs, so there is not strong evidence to close this issue.

## Evidence

- **Hook agent dispatch is still hardcoded isolated:** `dispatchAgentHook()` builds a `CronJob` with `sessionTarget: "isolated"` for webhook agent requests.
  - file: [src/gateway/server/hooks.ts:64](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/gateway/server/hooks.ts#L64)
  - command: `nl -ba src/gateway/server/hooks.ts | sed -n '35,95p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Isolated runs force a fresh session even when a session key is supplied:** `runCronIsolatedAgentTurn()` passes `forceNew: input.job.sessionTarget === "isolated"` into `resolveCronSession()`, so a supplied webhook session key does not reuse the existing session when the job is isolated.
  - file: [src/cron/isolated-agent/run.ts:486](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/cron/isolated-agent/run.ts#L486)
  - command: `nl -ba src/cron/isolated-agent/run.ts | sed -n '455,495p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Forced-new cron sessions create a new session id and clear prior routing state:** `resolveCronSession()` creates a new session whenever `forceNew` is true and clears delivery/session-file state for the fresh session entry.
  - file: [src/cron/isolated-agent/session.ts:31](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/cron/isolated-agent/session.ts#L31)
  - command: `nl -ba src/cron/isolated-agent/session.ts | sed -n '1,120p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Public docs still describe `/hooks/agent` as isolated:** The user docs explicitly say `POST /hooks/agent` runs an isolated agent turn, and the config reference exposes `sessionKey` but no `sessionTarget` or `targetExistingSession` option.
  - file: [docs/automation/cron-jobs.md:232](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/docs/automation/cron-jobs.md#L232)

  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)

  - file: [src/config/types.hooks.ts:11](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/config/types.hooks.ts#L11)
