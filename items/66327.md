## Summary

Keep open. Current main still does not implement MS Teams `sendPayload`: the bundled Teams outbound adapter and channel wiring expose text/media/poll only, while the core delivery path only uses rich interactive payload delivery when `handler.sendPayload` exists. Approval replies already generate interactive `/approve` buttons, and Teams already has Adaptive Card send infrastructure, so the PR addresses a real, unshipped bundled-plugin gap rather than obsolete work. The provided PR discussion also shows useful review activity, with the latest dep-aware media routing concern still needing author or maintainer resolution before merge.

## Best Possible Solution

Keep the PR open for maintainer review. The desired product direction is to add `sendPayload` to the bundled MS Teams channel using the existing channel outbound contract, Adaptive Card sender, media sequencing helpers, and dep-aware send path, with focused tests for approval buttons, prebuilt cards, fallback text/media behavior, and injected outbound deps.
