## Summary

Keep open. Current `main` still hard-returns `sessions_send` timeouts with no post-timeout reconciliation, so the reported behavior is not already implemented away. The PR is directionally useful, but one review comment raises a real correctness risk in the proposed compensation logic, so this is not a cleanup close.

## Evidence

- **Current main still hard-returns timeout:** `sessions_send` still returns `{ status: "timeout" }` immediately when `waitForAgentRunAndReadUpdatedAssistantReply()` times out; there is no post-timeout compensation path on current `main`.
