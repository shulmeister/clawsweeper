## Summary

Current `main` already implements the requested Feishu multi-account routing behavior: explicit tool `accountId` wins, otherwise the active message/agent account is used, and only then does Feishu fall back to the configured `defaultAccount`. The behavior is covered by targeted regression tests and was shipped in release `v2026.2.26`.
