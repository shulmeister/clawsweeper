## Summary

Current OpenClaw startup code loads and validates the config snapshot, resolves startup secrets, and only then bootstraps plugins; the report's claimed config-before-plugin race does not match current main or the latest shipped release. The concrete examples also point to an external WeChat plugin and a `memory-lancedb-pro` plugin that is not present in this repo, so there is no strong in-repo core repro left to keep open.

## Evidence

- **Gateway startup orders config before plugin bootstrap:** `startGatewayServer()` measures `config.snapshot`, then `config.auth`, and only after that runs `plugins.bootstrap`, so plugin bootstrap is downstream of config loading/preparation.

- **Startup config is validated and secrets-resolved before use:** `prepareGatewayStartupConfig()` first asserts a valid config snapshot, applies overrides, resolves startup auth SecretRefs, then activates runtime secrets and returns the prepared `cfg` used for startup.
  - file: [src/gateway/server-startup-config.ts:215](https://github.com/openclaw/openclaw/blob/713807b55d0beb17c0b96d0490a8abae695bb960/src/gateway/server-startup-config.ts#L215)
