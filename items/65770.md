## Summary

Keep open. Current main does not implement the PR’s run-scoped announce tracking: announce delivery still carries only source session/channel/tool metadata, run records do not persist pending or delivered announce IDs, and restore does not clear stale pending announce state. The PR is a plausible focused implementation candidate, but its existing review thread identifies a concrete redundant-persist issue that should be fixed before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate. Resolve the existing review comment by making stale-pending cleanup either caller-persisted or self-persisted, rebase on current main, then rerun the targeted subagent announce/registry tests and the appropriate changed-agent validation before maintainer merge review.
