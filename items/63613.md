## Summary

Close as ClawHub/community plugin work. Current main does not bundle a LocalAI provider, but the PR is a self-contained provider plugin using already-public provider registration and self-hosted OpenAI-compatible setup seams. OpenClaw’s current architecture and docs explicitly steer optional provider integrations to external plugins published through ClawHub/npm unless there is a missing generic plugin API, and this PR does not demonstrate such a gap.

## Best Possible Solution

Move the LocalAI provider draft into an external LocalAI provider plugin published through ClawHub/npm, reusing `openclaw/plugin-sdk/provider-setup`, the manifest provider metadata, and the onboarding shape already sketched in this PR. Keep OpenClaw core focused on generic provider SDK seams; open a focused SDK issue or PR only if LocalAI needs a missing generic hook.
