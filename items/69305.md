## Summary

Keep PR #69305 open. Current main still emits zsh leaf-command completions from options only, so the linked positional completion bug is not implemented on main. The PR is a focused, plausible fix with tests, but it should address the remaining review concern around broad `<path>` heuristics before merge.

## Best Possible Solution

Keep the PR open and ask for a small revision rather than closing it. The best fix is still in `src/cli/completion-cli.ts`: emit zsh positional specs from Commander registered arguments for leaf commands, preserve optional/variadic semantics, and add regression tests in `src/cli/completion-cli.test.ts`; refine completion actions with an explicit allowlist or metadata so filesystem completion applies to real file/directory arguments like `wiki ingest <path>` without regressing config-key positionals.
