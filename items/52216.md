## Summary

Current `main` has newer SDK docs that distinguish `api.registerHook(...)` from `api.on(...)`, but the user-facing plugin page still groups them together as one "Lifecycle hooks" entry. That means the PR's clarification is not clearly implemented on `main`, so there is not strong enough evidence to close it as redundant.

## Evidence

- **Current plugin docs still conflate the two APIs:** `docs/tools/plugin.md:334-365` still lists ``registerHook / on(...)`` in one table row as `Lifecycle hooks`, even though the surrounding section discusses typed lifecycle hook guard behavior separately.
  - file: [docs/tools/plugin.md:341](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/docs/tools/plugin.md#L341)

  - file: [docs/plugins/sdk-overview.md:94](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/docs/plugins/sdk-overview.md#L94)
