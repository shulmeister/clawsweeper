## Summary

Current main already covers this request: OpenClaw treats `~/.openclaw/media` as an allowed local media root, the image tool uses those roots for host-local reads, and sandboxed image-tool runs now rewrite absolute inbound paths like `~/.openclaw/media/inbound/photo.png` to `media/inbound/<filename>` inside the sandbox with regression coverage.
