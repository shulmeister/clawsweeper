## Summary

Keep open. Current main still contains the exact cron `payload.model` allowlist fallback that this PR is trying to change, and existing tests still lock in that fallback behavior. The PR is not implemented on main, not superseded by the provided related items, and it is a focused recent bug-fix PR rather than a conservative cleanup close candidate.

## Best Possible Solution

Keep #68555 open. If the intended product behavior is to reject explicitly configured but disallowed cron `payload.model` values, update `src/cron/isolated-agent/model-selection.ts` to return `ok: false` for the real lowercase `model not allowed:` resolver error, avoid ad-hoc `console.error()` in the helper, and update the cron model-selection/run tests to assert the hard-error behavior. If maintainers instead want fallback to remain intentional, close only after an explicit maintainer decision and doc/test wording that explains the fallback contract.
