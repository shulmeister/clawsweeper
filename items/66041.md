## Summary

Keep this PR open. Current main still has the Telegram ack-reaction mismatch described by the PR: Telegram group messages with `requireMention: false` can continue into processing, but the shared `group-mentions` ack gate still rejects them because `requireMention` is false. The PR is a focused implementation candidate with a regression test and docs note rather than obsolete work.

## Best Possible Solution

Review and land this PR or an equivalent Telegram-owned fix that treats the default `group-mentions` ack scope as `group-all` only for Telegram groups already configured with `requireMention: false`, keeping the regression test in `extensions/telegram/src/bot.create-telegram-bot.test.ts` and the Telegram docs note. Closing should wait until that implementation is merged into main.
