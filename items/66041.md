## Summary

Keep #66041 open. Current main still passes the default `group-mentions` scope directly into the shared ack-reaction gate, and that gate still returns false when `requireMention` is false. The PR's Telegram-local effective scope promotion, regression test, and docs note are not present on main. The linked issue #65808 is closed, but this open PR still appears to be the focused implementation for the remaining behavior.

## Best Possible Solution

Keep the PR open and land this Telegram-owned fix, or an equivalent patch, after normal maintainer review. The best implementation is to keep the shared `shouldAckReaction` semantics unchanged and translate the effective Telegram scope locally when `ackReactionScope` is `group-mentions`, the chat is a group, and resolved `requireMention` is false, with the focused Telegram regression test and docs note included.
