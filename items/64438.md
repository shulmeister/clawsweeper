## Summary

Keep open. Current main supports remote embedding endpoints, local MMR re-ranking in the builtin memory engine, and QMD's own local/sidecar reranking path, but it does not expose or call a Cohere-style remote reranker endpoint. The request is concrete and recent, and the current documented plugin/provider surface only exposes memory embedding adapters, not a reranker adapter that would make this a clean ClawHub close.

## Best Possible Solution

Keep this issue open for a focused memory-search design. The likely path is an additive reranker seam in `memorySearch` plus schema/docs/tests, with runtime support in the builtin backend and a clear QMD integration story. If maintainers prefer plugin ownership, first add a documented reranker provider API so Qwen/Cohere/Jina-style rerankers can live in provider plugins without replacing the whole memory backend.
