## Summary

Keep open. Current main still has the reported bug: trajectory sidecars are documented and generated as `<session>.trajectory.jsonl`, but `isPrimarySessionTranscriptFileName()` still accepts any non-archived `.jsonl`, and doctor orphan detection still feeds that broad predicate directly before offering archival repair. The PR is a recent, focused fix for open issue #70680 with tests and follow-up handling for `.trajectory` session-id concerns; the related #72047 is closed unmerged, so it does not supersede this PR.

## Best Possible Solution

Keep #70812 open as an active fix candidate for #70680. The best path is to finish review of the branch's sidecar classification approach, ensure valid `.trajectory` session IDs and disk-budget behavior remain correct, then land this or an equivalent fix with the helper and doctor regression coverage.
