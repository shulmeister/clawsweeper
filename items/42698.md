## Summary

Keep open. Current main partially implements the Bonjour/CIAO crash mitigation for the IPv4-loss assertion, but it does not cover the reverse `IPv4 address changed from undefined to defined` assertion that this PR explicitly reports and tests. The PR needs rebasing onto the current bundled `bonjour` plugin files, not closure as implemented.

## Best Possible Solution

Keep the PR open and port the remaining useful behavior onto current main: update `extensions/bonjour/src/ciao.ts` so the CIAO interface-assertion classifier covers both `changed from undefined to defined` and `change from defined to undefined`, add the missing regression in `extensions/bonjour/src/ciao.test.ts`, and keep the advertiser logging behavior in `extensions/bonjour/src/advertiser.ts` unchanged. Current main already shipped the #38628 IPv4-loss case, so the rebased change should be narrowly about the reverse ciao assertion variant.
