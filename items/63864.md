## Summary

Keep open. Current main has not implemented configurable per-session lane concurrency: session lanes are still created with `maxConcurrent: 1`, and gateway config application only updates the cron/main/subagent lanes. This PR is a plausible implementation candidate for the first part of open #53023, but the current diff has material review-raised blockers and should be revised or superseded rather than auto-closed.

## Best Possible Solution

Keep this PR open for changes or supersede it with a clean implementation. The product path is to add a real `agents.defaults` session-lane concurrency option on current main, register it in types/schema/defaults/generated metadata/docs, add resolver and queue behavior tests, store the default in shared queue state, update already-created `session:*` lanes on config reload, and preserve existing public exports/types. Leave #53023 open for the broader auto-yield and lane-wait-threshold follow-up unless those are also implemented.
