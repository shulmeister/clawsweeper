## Summary

Keep this PR open. Current main still lacks the proposed passive session recovery prompt discipline, Recovery Bundle / Task Ledger state files, and checkpoint writes for update_plan, sessions_yield, and sessions_spawn. Existing main-session and subagent restart recovery is active synthetic-resume recovery, not the passive informational checkpoint layer proposed here.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase on current main, address the redaction and synchronous-I/O concerns, keep recovery strictly passive and informational, prove approvals/auth/permissions are never inherited, add focused tests for update_plan, sessions_yield, sessions_spawn, prompt composition, and persisted recovery state, and decide whether to consolidate with the adjacent open recovery-log work.
