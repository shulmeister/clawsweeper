## Summary

Keep #69975 open. Current main still has the misleading `--tz` help text in both `cron add` and `cron edit`, and the related issue #59456 was closed specifically as superseded by this active PR. The PR is focused and useful, but it should not be closed as implemented or duplicate because main has not changed; it needs a small wording revision to remove the incorrect universal `default: UTC` claim for cron schedules.

## Best Possible Solution

Keep #69975 open as the canonical narrow help-text PR for #59456. Revise both `--tz` help strings to mention `--cron` plus offset-less `--at` datetimes without implying cron defaults to UTC, align `cron edit --at` with `cron add`, then validate the rendered help output and land the focused two-file change.
