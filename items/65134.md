## Summary

Keep this PR open. Current main still routes session-memory slug generation through `runEmbeddedPiAgent`, creates a temp session file, and resolves the normal agent primary model. The PR addresses a real remaining cost/congestion problem also tracked by #33962, but it should be narrowed before merge because the submitted branch includes unrelated timeout-failover logging changes.

## Best Possible Solution

Keep this PR open as a plausible implementation candidate, but ask the contributor to narrow it to the slug generator change, drop or split the timeout logging commit, and add regression coverage showing `generateSlugViaLLM` uses `completeSimple`, avoids temp session files/embedded-runner lanes, prefers `agents.defaults.heartbeat.model` when configured, and falls back to the primary model when not configured. Cross-link #33962 as the canonical user-facing issue for the remaining work.
