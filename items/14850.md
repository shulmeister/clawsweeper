## Summary

Keep #14850 open. Current main has the narrower route-binding `bindings[].session.dmScope` support for direct peers, but it still has no binding-level `sessionKey`, `sessionAlias`, or `sessionScope` that can intentionally route Discord guild/channel peers into an agent main or alias session. Non-direct peers still resolve to isolated channel/group keys, and the public docs continue to describe Discord guild channels as isolated.

## Best Possible Solution

Keep the issue open until maintainers either add an explicit, documented, opt-in binding-level session target for normal route bindings on non-direct peers, or make a product decision to reject channel-to-main session unification. The likely code path is `AgentRouteBinding.session`, `RouteBindingSchema`, `resolveAgentRoute`, and `buildAgentPeerSessionKey`, with tests proving Discord guild/channel bindings can intentionally share an agent main or aliased session while preserving isolated defaults.
