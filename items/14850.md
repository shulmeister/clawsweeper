## Summary

Keep #14850 open. Current main has the narrower `bindings[].session.dmScope` route-binding override for direct peers, but it still does not provide a binding-level `sessionKey`, `sessionAlias`, or `sessionScope`, and non-direct Discord guild/channel peers remain isolated. The related discussion does not supersede the request: #50875 is still open and scoped away from group/channel peers, and #39049 was closed unmerged with groups/channels off by default.

## Best Possible Solution

Keep the issue open until maintainers either add an explicit, documented, opt-in binding-level session target for non-direct route bindings, or make a product decision to reject channel-to-main session unification. The likely implementation path is `AgentRouteBinding`, `RouteBindingSchema`, `resolveAgentRoute`, and `buildAgentPeerSessionKey`, with tests proving Discord guild/channel bindings can intentionally share an agent main or aliased session while preserving isolated defaults.
