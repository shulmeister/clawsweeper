## Summary

Keep open. Current `main` has added live fallback observability for connected clients, but it still does not persist the fallback cause into the session transcript or durable session state. The reported forensic gap remains: offline history can show a model switch without a durable record of the triggering failure.

## Evidence

- **Live fallback events now carry reason data, but only as runtime lifecycle events:** `agent-runner` emits a `lifecycle` event with `phase: "fallback"` plus `reasonSummary`, `attemptSummaries`, and `attempts`, which shows related work from the linked UX/fallback discussions landed for live subscribers.
  - file: [src/auto-reply/reply/agent-runner.ts:1508](https://github.com/openclaw/openclaw/blob/713807b55d0beb17c0b96d0490a8abae695bb960/src/auto-reply/reply/agent-runner.ts#L1508)
  - sha: [713807b55d0b](https://github.com/openclaw/openclaw/commit/713807b55d0beb17c0b96d0490a8abae695bb960)
- **Gateway persistence layer ignores fallback lifecycle phases:** The durable session lifecycle code only recognizes `start`, `end`, and `error`. `server-chat` persists `start` immediately and finalizes only `end`/`error`, so emitted `phase: "fallback"` events are not written to session state.
  - file: [src/gateway/session-lifecycle-state.ts:6](https://github.com/openclaw/openclaw/blob/713807b55d0beb17c0b96d0490a8abae695bb960/src/gateway/session-lifecycle-state.ts#L6)
  - sha: [713807b55d0b](https://github.com/openclaw/openclaw/commit/713807b55d0beb17c0b96d0490a8abae695bb960)
- **Session event handler does not persist non-start/end/error lifecycle events:** In the agent event handler, the persistence path runs for `error`, `end`, and `start` branches only. There is no persistence branch for `fallback` even though fallback lifecycle events are broadcast to clients.
  - file: [src/gateway/server-chat.ts:988](https://github.com/openclaw/openclaw/blob/713807b55d0beb17c0b96d0490a8abae695bb960/src/gateway/server-chat.ts#L988)
  - sha: [713807b55d0b](https://github.com/openclaw/openclaw/commit/713807b55d0beb17c0b96d0490a8abae695bb960)
- **Transcript custom persistence still only covers model snapshots here:** Replay-history support reads and appends `customType: "model-snapshot"`. There is no adjacent transcript persistence for a fallback-cause record before or alongside the snapshot.
  - file: [src/agents/pi-embedded-runner/replay-history.ts:389](https://github.com/openclaw/openclaw/blob/713807b55d0beb17c0b96d0490a8abae695bb960/src/agents/pi-embedded-runner/replay-history.ts#L389)
  - sha: [713807b55d0b](https://github.com/openclaw/openclaw/commit/713807b55d0beb17c0b96d0490a8abae695bb960)
- **Durable session-store updates only capture the resulting active model:** After a run, session-store persistence records `providerUsed`/`modelUsed` from the fallback result, but does not store fallback reason, status, or failed-attempt details.
  - file: [src/agents/command/session-store.ts:66](https://github.com/openclaw/openclaw/blob/713807b55d0beb17c0b96d0490a8abae695bb960/src/agents/command/session-store.ts#L66)
  - sha: [713807b55d0b](https://github.com/openclaw/openclaw/commit/713807b55d0beb17c0b96d0490a8abae695bb960)
- **No transcript-level fallback trigger record exists on main:** Repo-wide search for a transcript marker like `fallback-trigger` returned no matches.
