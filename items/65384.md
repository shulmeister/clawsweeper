## Summary

Keep this PR open. Current main still lacks a Feishu `botName`-to-`name` doctor migration: Feishu account config supports `name` and is strict, but the Feishu plugin does not expose a doctor contract or runtime doctor adapter, and the current doctor migration tests only cover the generic/bundled-contract path plus Mattermost. The provided PR is a focused implementation candidate for the still-open related bug #65177, with review discussion already addressing the arrow-message polish and blank-name conflict case.

## Best Possible Solution

Keep this PR open and review or land it, or land an equivalent Feishu-owned doctor migration. The implementation should stay in the Feishu plugin boundary, expose `legacyConfigRules` and `normalizeCompatibilityConfig` through the bundled contract surface, wire the Feishu channel doctor adapter, and keep regression coverage for `botName` to `name`, including conflict and blank-name cases. After merge, the related bug #65177 can close as implemented.
