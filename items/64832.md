## Summary

Keep this PR open. Current main has the isolated-session transcript rotation fix, but it still does not archive the prior transcript after heartbeat or cron rotation. The PR carries unique remaining cleanup work, though its implementation should address the unresolved review concern around historical cron run keys before merge.

## Best Possible Solution

Keep the PR open for maintainer review and revision. The right product/code direction is to preserve the shipped `sessionFile` rotation behavior, then add a narrowly scoped archive-on-rotation path for heartbeat and isolated cron that snapshots the prior entry before overwrite, archives after successful persistence with `reason: "reset"`, and treats historical cron run rows separately from active session references.
