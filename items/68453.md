## Summary

Keep open. Current main still does not implement the PR‚Äôs specific cron path where the latest successful tool result is exact `NO_REPLY` but the final assistant message is empty, and this PR still has an acknowledged review concern around scanning full session history for stale tool results.

## Best Possible Solution

Keep the PR open for maintainer review or consolidation with the active #68452/#70599 work. The best fix is a narrow current-attempt guard for cron runs: recognize the latest successful current-turn tool result whose text is exact `NO_REPLY`, avoid suppressing timeout/abort/tool/provider-error paths, and add regression coverage for both the intended empty-final cron success and the stale-history/no-current-tool case.
