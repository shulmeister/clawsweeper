## Summary

Keep open. Current `main` shipped a different solution in #58900: `chat.history` truncation is configurable via `gateway.webchat.chatHistoryMaxChars` and RPC `maxChars`, but the default remains 8,000 chars, the Web UI still does not request a higher limit, and truncation is still character-count based rather than the PR's proposed byte-aware fallback. That is not strong enough evidence that this PR's request is already implemented on `main`.
