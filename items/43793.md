## Summary

Keep open. Current main still lacks the PR's central TUI reconnect UX: generic gateway disconnects still fall back to idle, there is no Ctrl+R editor hook, and the TUI backend/GatewayChatClient do not expose a manual reconnect method. The PR discussion also records useful follow-up fixes on the branch, so the work is not obsolete, but it should be rebased and narrowed.

## Best Possible Solution

Keep this PR open for maintainer review, but ask the contributor to rebase and narrow it. The best path is a focused TUI/gateway change that exposes a safe manual reconnect API, wires a visible retry shortcut or command, shows retry/restart guidance only for reconnectable disconnects, preserves auth/pairing-specific guidance using the current onReconnectPaused detailCode seam, resets activity status on reconnect, and includes regression tests. The macOS remote-probe work should be split or reviewed as a separate surface.
