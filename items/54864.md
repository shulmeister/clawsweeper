## Summary

Keep this issue open. Current main has the lower-level `--session-id` isolation behavior needed by callers that already generate IDs, but it still does not expose `openclaw agent --new-session`, generate a fresh CLI session ID, or return that generated ID as a top-level JSON field. The related implementation PR is closed and unmerged, so it is useful prior art rather than a superseding fix.

## Best Possible Solution

Keep this issue open and implement a narrow core CLI feature in the existing `openclaw agent` path: add a `--new-session` boolean option, reject it together with `--session-id`, generate one `crypto.randomUUID()` in `agentCliCommand` before choosing gateway versus `--local`, pass that ID as the effective `sessionId` to both paths, surface it as a top-level `sessionId` in `--json` output, and update CLI docs plus parser/session-resolution tests while preserving the default agent main-session reuse behavior.
