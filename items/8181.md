## Summary

Keep #8181 open. Current main has partial progress for explicit local embeddings: `provider: "local"` can use the default downloadable GGUF model. The requested zero-config setup is still not implemented because `node-llama-cpp` is still an operator-installed optional runtime, setup/help text still sends users to install/approve/rebuild it manually, and `auto` mode still skips local embeddings on a clean config unless an existing local model file path is configured.

## Best Possible Solution

Keep #8181 open until maintainers either implement true turnkey local embeddings or explicitly decide and document that local embeddings remain a manual optional runtime. A complete fix should cover native runtime install/build or prebuilt binaries, first-use model download with clear progress/failure messaging, `auto` versus explicit `local` semantics, and matching docs/tests.
