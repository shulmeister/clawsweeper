## Summary

Keep #8181 open. Current main only partially addresses the request: explicit `provider: "local"` can use the default downloadable GGUF model, but OpenClaw still does not provide a true zero-config local embeddings path because `node-llama-cpp` remains optional/manual, docs still require native build steps, and `auto` mode skips the default downloadable model unless a real local file path already exists.

## Best Possible Solution

Keep #8181 open until maintainers either implement true turnkey local embeddings or explicitly decide local embeddings should remain an optional manual runtime. A complete fix should handle `node-llama-cpp` install/build or prebuilt binaries, first-use model download with clear progress/failure messaging, and docs/tests where `memorySearch.provider: "local"` is the only required user config.
