## Summary

Keep #8181 open. Current main still only partially satisfies zero-config local memory embeddings: explicit `provider: "local"` can use the default downloadable GGUF model, but the native `node-llama-cpp` runtime remains operator-installed/manual, and `provider: "auto"` still skips local unless a configured local model file already exists.

## Best Possible Solution

Keep #8181 open until maintainers either implement true turnkey local embeddings or explicitly decide/document that local embeddings remain a manual optional runtime. A complete fix would cover native runtime install/build or prebuilt binaries, first-use model download with clear progress/failure messaging, `auto` versus explicit `local` selection semantics, and matching docs/tests.
