## Summary

Keep #8181 open. Current main has partial explicit-local support: `provider: "local"` falls back to the default EmbeddingGemma GGUF URI and uses `node-llama-cpp` model resolution. The requested zero-config setup is still missing because the native runtime remains optional/manual, docs and runtime errors still tell users to install/approve/rebuild `node-llama-cpp`, and clean `provider: "auto"` does not select the default downloadable local model.

## Best Possible Solution

Keep #8181 open until maintainers either implement true turnkey local embeddings or explicitly decide that local embeddings remain a manual optional runtime. A complete fix should handle node-llama-cpp install/build or prebuilt binary availability, first-use GGUF download UX and progress/failure messaging, clear provider:auto versus provider:local semantics, platform fallback messaging, and matching docs plus tests.
