## Summary

Keep #8181 open. Current main at c3833f7729163a8f6cdc47d3b0be86fd5fcab160 still only partially satisfies the request: explicit `provider: "local"` can use a default downloadable GGUF model, but local embeddings are still operator-installed/manual, and `auto` does not choose the default downloadable local model without an existing local file path.

## Best Possible Solution

Keep #8181 open until maintainers either implement true turnkey local embeddings or explicitly choose/document local embeddings as an optional manual runtime. A complete fix should cover native runtime install/build or prebuilt binaries, first-use model download with clear progress/failure messaging, `auto`/`local` selection semantics, and matching docs/tests.
