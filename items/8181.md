## Summary

Keep #8181 open. Current main partially addresses the request by adding an explicit local provider default GGUF model that is passed to node-llama-cpp for resolution/download, matching the helpful comment on the issue in part. However, the requested zero-config setup is still not implemented: current source/docs still require an optional node-llama-cpp runtime install/native build path, `node-llama-cpp` is not declared as an OpenClaw package dependency, and auto mode deliberately does not select the default downloadable local model unless a real local model path already exists.
