## Summary

Keep open. Current main has ACP lifecycle cleanup paths, and the bundled ACPX plugin is now on acpx 0.6.0, but I did not find a high-confidence fix for the reported Windows process-tree leak. OpenClaw still delegates ACP cancel/close to the ACPX runtime, and the inspected acpx 0.6.0 package still uses direct process termination rather than a Windows process-tree kill. The related minions PR #68718 remains open and is not present on this main checkout.
