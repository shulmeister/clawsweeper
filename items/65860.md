## Summary

Current main still reproduces the reported bad guidance: the fallback plugin override error links to `https://docs.openclaw.ai/tools/plugin#runtime-helpers`, and the Gateway test suite explicitly expects that same dead anchor. The relevant `allowModelOverride` docs exist elsewhere, so the issue should stay open until the source string and test expectation are corrected.

## Best Possible Solution

Keep the issue open. The narrow fix belongs in `src/gateway/server-plugins.ts`: replace the dead `tools/plugin#runtime-helpers` URL with a valid public docs target such as `https://docs.openclaw.ai/plugins/sdk-runtime` near the `api.runtime.subagent` guidance, or add a real runtime-helpers section to `docs/tools/plugin` if maintainers prefer that URL. Update the existing Gateway test expectation so this does not regress.
