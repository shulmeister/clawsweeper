## Summary

Keep this PR open. Current main still has the reported exec allowlist behavior: backslash-newline commands are rejected before allowlist analysis, and parse/analysis failures still surface as allowlist misses rather than a distinct unsupported-shell-syntax denial. The PR remains a focused implementation candidate for the linked user-visible bug, with maintainer review still needed around durable-approval behavior and changelog attribution.

## Best Possible Solution

Keep this PR open and review it as the active implementation candidate. The best path is to land an adapted version that normalizes POSIX backslash-newline before allowlist analysis with quote/heredoc/comment-aware parsing, emits a distinct unsupported-shell-syntax denial through gateway and node-host paths, defines durable-approval behavior for unparseable commands, and includes focused regression coverage plus a properly attributed changelog entry.
