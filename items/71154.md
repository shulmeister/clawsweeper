## Summary

Keep open. Current main still has the old exec allowlist behavior: any command containing POSIX backslash-newline is rejected before allowlist analysis, and both gateway and node-host policy still report analysis failures as allowlist misses rather than a distinct unsupported-shell-syntax reason. The related issue #71153 remains open, and the only related PR #71225 is closed/unmerged, so #71154 is not obsolete or implemented on main.

## Best Possible Solution

Keep #71154 open as the active, focused fix path for #71153. The likely maintainer path is to review and harden the PR rather than close it: splice POSIX `\<newline>` before allowlist analysis, add the distinct `unsupported-shell-syntax` denial path across gateway/node-host/user messaging, keep the regression tests, and address the durable-approval/security and changelog-review notes before merge.
