## Summary

Keep open. The PR’s requested behavior is still missing on current main: broad channel status scope still resolves every channel-capable plugin without checking effective activation, and the runtime loader still intentionally widens an empty `scope: "channels"` load instead of forwarding `onlyPluginIds: []`. The PR discussion and review comments identify real edge cases, and current tests still encode one of the old behaviors.
