## Summary

Current `main` already supports customizing the webchat/control-ui user identity with a browser-local name and avatar, and the chat renderer uses that identity for user messages. The requested outcome is implemented, though as local UI settings rather than a `ui.user.*` config key.

## Risks / Open Questions

- The implementation is browser-local UI state, not a workspace-configured `ui.user.avatar` / `ui.user.name` schema entry.
- Local user avatars are restricted to local-safe sources (`data:image/...` or same-origin/root-relative routes) rather than arbitrary remote URLs.
