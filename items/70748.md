## Summary

Keep open. Current main still does not implement first-class provider-aware TTS personas: no `messages.tts.persona(s)` config, no persona resolver, no provider `prepareSynthesis` hook, and no `/tts persona` or `tts.setPersona` control surface. Main has shipped adjacent per-agent TTS overrides, but that does not satisfy the reusable provider-agnostic persona feature tracked by #68323 and this active PR.

## Best Possible Solution

Keep #70748 open as the active implementation candidate for #68323. The best path is to rebase it onto current main, reconcile it with shipped `agents.list[].tts` per-agent overrides, and review the final persona design across config schema, speech-core resolution, provider `prepareSynthesis`, gateway/CLI/chat controls, docs, generated baselines, and targeted provider tests.
