## Summary

Keep open. Current `main` still computes cron run status only from structured `payload.isError` flags and does not inspect narrated denial text in `summary`/`outputText`/`synthesizedText`, so the reported `status:"ok"` false-success path remains plausible and is still shipped in the latest tagged release available in this checkout.

## Evidence

- **Classifier ignores denial text:** `resolveCronPayloadOutcome` derives `hasFatalErrorPayload` from `payload?.isError === true` and later builds `summary`, `outputText`, and `synthesizedText` without any token scan for denial phrases or gateway/system denial markers.
  - file: [src/cron/isolated-agent/helpers.ts:150](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/src/cron/isolated-agent/helpers.ts#L150)
  - command: `nl -ba src/cron/isolated-agent/helpers.ts | sed -n '137,205p'`
  - sha: [7875092f4ddc](https://github.com/openclaw/openclaw/commit/7875092f4ddc6f2dbe218bea8506c9ac1da87c39)
- **Run status still keyed only off fatal payload flag:** `finalizeCronRun` sets run `status` to `"error"` only when `hasFatalErrorPayload` is true; otherwise it records `"ok"`. No additional summary/output denial classification happens at the finalization point.
  - file: [src/cron/isolated-agent/run.ts:822](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/src/cron/isolated-agent/run.ts#L822)
