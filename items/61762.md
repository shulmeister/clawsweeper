## Summary

Issue #61762 is the same loopback cron/device-scope-upgrade failure already handled in #57688. Current main does not use `openclaw pairing` for this; it provides `openclaw devices list` / `openclaw devices approve <requestId>` recovery, with local loopback fallback when direct gateway access is blocked by `pairing required`.
