## Summary

Keep PR #72174 open. Current main still has the attach-only bug: the launch flag path and Debug Settings toggle write the disable marker, then call the launchd disable/uninstall path. Related daemon launchd lifecycle fixes do not supersede this PR because the remaining bug is app-side attach-only code invoking uninstall.

## Best Possible Solution

Keep PR #72174 open for maintainer review or landing. The best product path is the PR's marker-only attach-only helper used by both the launch flag path and Debug Settings toggle, plus the focused regression test proving attach-only writes the marker without issuing daemon uninstall commands.
