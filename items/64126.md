## Summary

Keep this PR open. Current main still uses a direct `meta.token !== token` comparison for diff viewer artifact access, while the shared `safeEqualSecret` helper is already available through the documented SDK runtime subpath and used on comparable auth/token surfaces. The PR is a focused implementation candidate for the `/plugins/diffs/view/{artifactId}/{token}` surface, not something already implemented or superseded on main.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for the remaining diffs artifact token comparison. The likely best product/code path is to land the narrow `DiffArtifactStore.getArtifact` change using `safeEqualSecret` from `openclaw/plugin-sdk/browser-security-runtime`, then run the normal changed extension validation before merge. A small regression assertion is optional because existing tests already cover invalid-token rejection, but validation should at least confirm the diffs store/http tests and package-boundary imports remain green.
