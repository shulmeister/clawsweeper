## Summary

Current main still has no Feishu/Lark option to hide or customize the runtime footer. The bundled Feishu reply dispatcher always builds an `Agent`/`Model`/`Provider` card note for card replies, and the Feishu config schema, plugin manifest, tests, and docs do not expose a footer-specific setting.

## Best Possible Solution

Keep this issue open and implement a conservative Feishu-owned config option. The likely path is a typed per-channel/per-account setting such as `channels.feishu.showRuntimeFooter` or a small `replyFooter` mode/template, wired through `resolveCardNote`, the Feishu schema, plugin manifest metadata, generated config/docs surfaces, and reply dispatcher tests for both streaming and non-streaming card replies while preserving existing default footer behavior.
