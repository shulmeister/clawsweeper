## Summary

Current `main` already preserves custom LaunchAgent environment variables during both service refresh/reinstall flows and plist rewrites on restart fallback, which addresses the issue’s reported env-loss behavior.

## Evidence

- **Install plan preserves custom existing service env:** `collectPreservedExistingServiceEnvVars` keeps safe non-`OPENCLAW_*` values from the existing LaunchAgent environment, then merges them into the new install environment before writing the service definition.
  - file: [src/commands/daemon-install-helpers.ts:178](https://github.com/openclaw/openclaw/blob/fc5920fb51346a682b269fdbeeec0971ae4ae723/src/commands/daemon-install-helpers.ts#L178)
  - command: `nl -ba src/commands/daemon-install-helpers.ts | sed -n '178,252p'`
  - sha: [fc5920fb5134](https://github.com/openclaw/openclaw/commit/fc5920fb51346a682b269fdbeeec0971ae4ae723)
- **Gateway install path reads existing service env:** `openclaw gateway install` passes `existingServiceEnv` into `buildGatewayInstallPlan`, so forced reinstalls/service refreshes no longer drop preserved custom environment variables.
  - file: [src/cli/daemon-cli/install.ts:157](https://github.com/openclaw/openclaw/blob/fc5920fb51346a682b269fdbeeec0971ae4ae723/src/cli/daemon-cli/install.ts#L157)
  - command: `nl -ba src/cli/daemon-cli/install.ts | sed -n '150,205p'`
  - sha: [fc5920fb5134](https://github.com/openclaw/openclaw/commit/fc5920fb51346a682b269fdbeeec0971ae4ae723)
- **Update flow now refreshes service env via forced install:** After package updates, the update command explicitly runs `gateway install --force` to refresh the managed service definition, which now uses the preservation logic above.
  - file: [src/cli/update-cli/update-command.ts:258](https://github.com/openclaw/openclaw/blob/fc5920fb51346a682b269fdbeeec0971ae4ae723/src/cli/update-cli/update-command.ts#L258)
  - command: `nl -ba src/cli/update-cli/update-command.ts | sed -n '258,283p'`
  - sha: [fc5920fb5134](https://github.com/openclaw/openclaw/commit/fc5920fb51346a682b269fdbeeec0971ae4ae723)
- **Restart fallback rewrite preserves existing plist env:** When restart falls back to rewriting the LaunchAgent plist, it reads the current plist environment and writes that same environment back into the regenerated plist.
  - file: [src/daemon/launchd.ts:613](https://github.com/openclaw/openclaw/blob/fc5920fb51346a682b269fdbeeec0971ae4ae723/src/daemon/launchd.ts#L613)
  - command: `nl -ba src/daemon/launchd.ts | sed -n '609,633p'`
  - sha: [fc5920fb5134](https://github.com/openclaw/openclaw/commit/fc5920fb51346a682b269fdbeeec0971ae4ae723)
- **Regression test covers preserved custom vars:** The test suite now asserts that safe custom vars from an existing service environment survive rebuilds and that PATH is merged instead of discarded.
  - file: [src/commands/daemon-install-helpers.test.ts:381](https://github.com/openclaw/openclaw/blob/fc5920fb51346a682b269fdbeeec0971ae4ae723/src/commands/daemon-install-helpers.test.ts#L381)
