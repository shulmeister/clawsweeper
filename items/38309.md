## Summary

Keep #38309 open. Current main still does not support SecretRef objects in `sandbox.docker.env`: the config schema and public settings type remain string-only, the secrets runtime resolves sandbox SSH SecretRefs but not Docker env entries, and the Docker sandbox path treats `docker.env` as already-resolved `Record<string, string>`. The linked PR #38310 directly attempted this work, but the provided GitHub context shows it was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open until core implements the requested surface end to end: make `sandbox.docker.env` accept `SecretInput` in schema/types, resolve defaults and per-agent Docker env SecretRefs through the secrets runtime before sandbox launch, add target-registry/docs coverage, and keep the resolved Docker runtime config as `Record<string, string>` with regression tests for plain strings plus SecretRef objects.
