## Summary

Keep #38309 open. Current main at 7e13f3f5140168e31027b39a9855ade3fadf24d5 still does not support SecretRef values in `sandbox.docker.env`: the schema and public type remain string-only, Docker env values are merged and consumed as plain strings, and the secrets collector/target registry do not cover this path. PR #38310 is relevant prior work, but the provided GitHub context shows it was closed unmerged.

## Best Possible Solution

Keep the issue open until OpenClaw implements SecretRef support end-to-end for Docker sandbox env values: accept `SecretInput` in `SandboxDockerSchema` and `SandboxDockerSettings.env`, collect and resolve default and per-agent `sandbox.docker.env` refs through the existing secrets runtime, add target registry coverage for audit/plan/configure, keep resolved launch config as `Record<string, string>`, and add focused schema/runtime regression tests. PR #38310 remains useful design context but is not shipped or canonical.
