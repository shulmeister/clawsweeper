## Summary

Keep #38309 open. Current main at 46b9044c3f9d8519a56024f032b4b6e0b42aaf2b still does not implement SecretRef support for `sandbox.docker.env`: schema validation, public types, runtime config merging, secret collection, and target registry coverage remain string-only or absent for this surface. The linked PR #38310 is useful design context but was closed unmerged, and #33073 covered other SecretRef credential fields rather than Docker sandbox env values.

## Best Possible Solution

Keep the issue open until OpenClaw implements SecretRef support end-to-end for sandbox Docker env values: accept `SecretInput` in `SandboxDockerSchema` and `SandboxDockerSettings.env`, collect and resolve default and per-agent `sandbox.docker.env` refs through the existing secrets runtime, add target registry coverage for audit/plan/configure, keep the resolved launch config as `Record<string, string>`, and add focused schema/runtime tests. The closed PR #38310 remains useful context but is not a shipped or canonical replacement.
