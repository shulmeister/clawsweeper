## Summary

Keep #38309 open. Current main at 82262387651ab270b565f71533ce4937c9c00dc6 still does not support SecretRef objects in `sandbox.docker.env`: the Zod schema, public Docker sandbox type, generated config schema, runtime Docker env merge, and secrets collector/registry paths all remain string-only or omit Docker env. The related implementation PR #38310 is closed unmerged, so it remains useful context but does not supersede or implement the issue.

## Best Possible Solution

Implement SecretRef support end-to-end for Docker sandbox env values: accept `SecretInputSchema` in `SandboxDockerSchema.env`, type source settings as `Record<string, SecretInput>` while keeping the resolved Docker config/env handed to Docker string-only, add defaults and per-agent Docker env secret collectors plus target-registry entries for audit/plan/configure, update generated schema and public docs, and add focused validation/runtime/Docker-env regression tests. PR #38310 should be treated as useful prior design context, not as a closing artifact unless equivalent changes land on main.
