## Summary

Keep #38309 open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still does not support SecretRef values in `sandbox.docker.env`: the Zod schema, generated config schema, public config type, secret target registry, runtime collector, and Docker launch path remain string-only/plain-string paths. The related PR #38310 is relevant prior work, but the provided GitHub context shows it was closed unmerged.

## Best Possible Solution

Keep #38309 open until OpenClaw implements SecretRef support end-to-end for Docker sandbox env values: accept `SecretInput` in `SandboxDockerSchema` and `SandboxDockerSettings.env`, keep resolved `SandboxDockerConfig.env` as `Record<string,string>`, collect/resolve default and per-agent `sandbox.docker.env` refs through the secrets runtime, add target-registry coverage for audit/plan/configure, update docs and generated schema, and add focused schema/runtime/launch-path regression tests. PR #38310 remains useful prior design context but is not canonical because it closed unmerged.
