## Summary

Keep #14619 open. Current main still renders an in-band Tooling section with per-tool names and summaries, still sends tool definitions with descriptions/parameter schemas through the model tool path, and has no scoped config option to omit only the human-readable tool list. Related #14785 is broader token-overhead work, not a canonical duplicate for this specific skip-tool-list-text request.

## Best Possible Solution

Keep #14619 open until OpenClaw either adds a scoped agent/default config such as skipToolListText that omits only the human-readable Tooling list while preserving tool schemas, or a maintainer explicitly decides that the full systemPromptOverride workaround is the intended product boundary. If implemented, update the prompt builder, config schema/types/help/docs, and add prompt/report tests proving the list is omitted while tools remain callable.
