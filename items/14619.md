## Summary

Keep #14619 open. Current main still renders the human-readable Tooling/tool summary list in the system prompt and still sends structured tool definitions with descriptions and parameter schemas. The only nearby config surface is the full `systemPromptOverride`, not the requested scoped option to omit just the tool list text. The provided discussion is also aligned: jondecker76 identified the duplication, and the April 26 Codex review comment from steipete kept this exact item open; current main still matches that finding.

## Best Possible Solution

Keep #14619 open until OpenClaw adds a scoped agent/default config such as `skipToolListText` that omits only the human-readable Tooling list while preserving structured callable tool schemas, or until a maintainer explicitly decides that full `systemPromptOverride` is the intended boundary. A complete fix should update the prompt builder, config types/schema/help/docs, `/context` reporting if needed, and prompt/runtime tests proving tool schemas remain callable while the in-band list is omitted.
