## Summary

Keep #14619 open. Current main still emits the human-readable `## Tooling` section from active tool names while separately serializing callable tool definitions with descriptions and parameter schemas. I found no scoped `skipToolListText` or equivalent config; the existing `systemPromptOverride` replaces the whole generated prompt, which is broader than the requested optimization. The April 26 Codex/maintainer-side review and related open #14785 both support leaving this narrower request open.

## Best Possible Solution

Keep the issue open until OpenClaw adds a scoped agent/default config such as `skipToolListText` that suppresses only the human-readable Tooling section while preserving structured tool schemas. The implementation should update prompt builder/run plumbing, config types/zod/generated schema/help/docs, `/context` reporting, and focused tests proving both prompt omission and continued tool schema availability.
