## Summary

Current `main` still builds a human-readable per-tool list into the `## Tooling` system-prompt section, tests still assert those tool-description lines are present, and the agent-default config surface does not expose a `skipToolListText`-style option. Docs also still describe both tool-list text and tool schemas as separate context costs. This does not meet the bar for closing as already implemented or obsolete.

## Evidence

- **System prompt still renders tool names plus descriptions:** `buildAgentSystemPrompt()` builds `toolLines` as `- <name>: <summary>` entries and injects them into the `## Tooling` section.
  - file: [src/agents/system-prompt.ts:570](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/agents/system-prompt.ts#L570)
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **Prompt tests still require the human-readable tool list:** The current test suite expects the prompt to contain `- Read: Read file contents` and `- Exec: Run shell commands`, confirming the text list remains intentional behavior on `main`.
  - file: [src/agents/system-prompt.test.ts:374](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/agents/system-prompt.test.ts#L374)

## Risks / Open Questions

- `src/agents/system-prompt-report.ts:124` now hardcodes `tools.listChars` to `0`, so `/context` underreports this overhead even though `src/agents/system-prompt.ts` still emits the tool list. That makes the issue easier to misread as already solved.
- A full `systemPromptOverride` exists, but it is not a conservative substitute for the requested targeted opt-out because it replaces the whole OpenClaw-owned prompt rather than just skipping tool-list text.
