## Summary

Keep #14619 open. Current main still builds a human-readable `## Tooling` section with active tool names/descriptions in the system prompt, while the same effective tools are also converted to callable tool definitions with descriptions and schemas. I found no scoped `skipToolListText` or equivalent agent/default config; the only documented prompt control is full `systemPromptOverride`, which is broader than the requested tool-list-only suppression. The existing same-day maintainer-side Codex comment also explicitly kept the issue open, and the relevant files are unchanged from that reviewed SHA through current main.

## Best Possible Solution

Keep #14619 open until OpenClaw adds a scoped agent/default option, such as `agents.defaults.skipToolListText`, that suppresses only the human-readable Tooling list while preserving callable tool schemas. The implementation should update prompt construction, config types/Zod/generated schema/help/docs, `/context` reporting, and focused tests proving the prompt text is omitted while tools remain available.
