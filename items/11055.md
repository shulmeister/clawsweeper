## Summary

Current main has an opt-in user-facing compaction notice and a silent pre-compaction memory flush, but it does not implement the requested agent-visible pre-compaction warning (`memoryFlush.visibleWarning` / `warningPrompt`). Existing plugin hooks and memory flush surfaces are not enough to inject a visible agent turn before compaction, so meaningful requested work remains.

## Evidence
