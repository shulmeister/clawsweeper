## Summary

Current main still does not implement the requested agent-visible pre-compaction warning (`memoryFlush.visibleWarning` / `warningPrompt`). The shipped related behavior is silent pre-compaction memory flushing plus optional user-channel `notifyUser` notices, which do not give the agent a visible warning turn before compaction.

## Best Possible Solution

Keep #11055 open until OpenClaw either implements an explicit agent-visible pre-compaction warning path or a maintainer decides against that product direction. A concrete implementation would add validated config for the warning prompt, inject a visible agent-facing turn before the soft threshold, and preserve the existing silent memory flush plus separate `notifyUser` user-channel notices.
