## Summary

Keep open. Current `main` implements adjacent user-facing compaction notices (`compaction.notifyUser`) and a silent pre-compaction memory flush, but it does not implement the requested agent-visible pre-compaction warning/config surface (`memoryFlush.visibleWarning` / `warningPrompt`).

## Evidence

- **Config surface lacks requested fields:** `AgentCompactionConfig` exposes `notifyUser`, and `AgentCompactionMemoryFlushConfig` only exposes `enabled`, `softThresholdTokens`, `forceFlushTranscriptBytes`, `prompt`, and `systemPrompt`. There is no `visibleWarning` or `warningPrompt` setting.
  - file: [src/config/types.agent-defaults.ts:475](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/config/types.agent-defaults.ts#L475)

  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
