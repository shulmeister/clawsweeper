## Summary

Keep #11055 open. Current main still has silent pre-compaction memory flush and optional user-channel compaction notices, but it does not implement the requested `memoryFlush.visibleWarning` / `warningPrompt` config or an agent-visible pre-compaction warning turn.

## Best Possible Solution

Keep #11055 open until OpenClaw either implements an explicit agent-visible pre-compaction warning path or maintainers reject that product direction. A concrete implementation would add validated `agents.defaults.compaction.memoryFlush.visibleWarning` and `warningPrompt` config, inject a visible agent-facing warning before auto-compaction at the soft threshold, preserve the existing silent memory flush and separate `notifyUser` user notices, and add config/runtime/docs coverage for the distinction.

## Risks / Open Questions

- A visible agent warning needs product judgment because it could duplicate or interfere with the silent memory flush and user-channel `notifyUser` notices.
- Implementation should dedupe warnings carefully around the soft threshold so the warning does not repeat or consume enough context to worsen compaction pressure.
