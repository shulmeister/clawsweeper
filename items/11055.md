## Summary

Current main still does not implement the requested agent-visible pre-compaction warning (`memoryFlush.visibleWarning` / `warningPrompt`). It has silent pre-compaction memory flushes and opt-in user-channel compaction notices, but neither gives the agent a visible pre-compaction turn as requested in #11055.

## Best Possible Solution

Keep #11055 open until OpenClaw either implements an explicit agent-visible pre-compaction warning path or a maintainer decides against the product direction. A likely implementation would add validated config for the warning prompt and inject a visible agent-facing turn before the soft threshold while preserving the existing silent memory flush and separate `notifyUser` user-channel notices.
