## Summary

Keep #11055 open. Current main still implements silent pre-compaction memory flush plus optional user-channel compaction notices, but it does not implement `memoryFlush.visibleWarning`, `warningPrompt`, or an agent-visible warning turn before compaction. The latest issue discussion already distinguished this from #14790, which is closed unmerged and covered user-facing channel notices rather than this agent-facing warning request.

## Best Possible Solution

Keep #11055 open until maintainers either implement or explicitly reject an agent-visible pre-compaction warning. A concrete path would add validated `agents.defaults.compaction.memoryFlush.visibleWarning` and `warningPrompt` config, document it separately from `notifyUser`, inject a visible agent-facing warning before auto-compaction at the soft threshold, preserve the existing silent memory flush, and add config/runtime/docs tests for the distinction. If maintainers want plugin ownership, first add a supported plugin/runtime API for safe pre-threshold agent-visible injection.

- Implementation should dedupe threshold crossings carefully so the warning does not repeat during the same compaction cycle.
