## Summary

Current `main` now has an explicit authenticated dashboard launch flow (`openclaw dashboard`), clearer Control UI auth guidance, and dashboard/docs text that matches the reported `unauthorized` symptom, so this issue is no longer actionable as a core bug cleanup item.

## Evidence

- **Dedicated dashboard command now bootstraps authenticated UI access:** `dashboardCommand` reads the current gateway auth token, builds the Control UI URL, includes `#token=...` when safe, copies the URL, and opens the browser. That is the supported path for opening an authenticated dashboard session on current `main`.
