## Summary

Keep open. Current `main` does not contain the PR’s Synology webhook behavior: `createWebhookHandler` still rejects all non-`POST` methods, and successful webhook ACKs still return `204` rather than `200 {"success":true}`. The test suite in-tree also still asserts `204`, so there is no evidence this was already implemented on `main`.
