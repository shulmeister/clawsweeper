## Summary

Keep this issue open. Current main implements the global `agents.defaults.thinkingDefault` part, but the issue also requests global `agents.defaults.reasoningDefault`, and that remains absent from the defaults type/schema, generated help/schema, runtime resolution, status surfaces, and docs. The linked open PRs are implementation candidates or partial channel fixes, not shipped main behavior.

## Best Possible Solution

Keep this issue open until maintainers either implement and document `agents.defaults.reasoningDefault` across the config type, strict schema, generated schema/help, directive resolution, status/session surfaces, channel behavior where needed, and regression tests, or explicitly decide reasoning visibility should remain per-agent/session-only and remove the unsupported global key from agent tool/config surfaces. Review #39115 as the likely canonical implementation candidate and treat #71817 as a partial per-channel fix only.
