## Summary

Keep open. Current `main` does not implement this PR’s image-runtime fix: the image path still does a raw `modelRegistry.find(...)` lookup and throws on miss, while the shared helper added later on `main` exists specifically to recover provider-prefixed custom model IDs. The PR’s original `getAvailable()/getAll()` approach also drew valid review concerns, so this likely needs a rebase onto current `main` rather than conservative auto-close.

## Evidence
