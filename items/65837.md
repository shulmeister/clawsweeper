## Summary

Current main already solves the image-tool custom provider resolution problem through the newer media-understanding path. The image runtime now resolves through `resolveModelWithRegistry`, retries after dynamic provider preparation, and has regression coverage for configured image models when discovery has not registered the provider. The PR discussion correctly identified a gap in the PR's first fallback attempt, but current main supersedes that patch path.

## Best Possible Solution

Close this PR as implemented on current main. Keep the shipped `resolveModelWithRegistry` image-runtime path and its tests; do not merge the older `resolveModelFromRegistry` fallback patch unless a separate current-main regression proves that shared helper still needs a broader change.
