## Summary

Current main still does not respect OPENCLAW_WORKSPACE_DIR for the default agent workspace fallback that feeds the system prompt. The linked fix PR #66831 directly addressed this bug but was closed unmerged, and the described resolver/test changes are absent from the checked-out main branch.

## Best Possible Solution

Keep #66786 open and implement the narrow resolver fix from the closed PR: make resolveDefaultAgentWorkspaceDir() prefer a nonblank OPENCLAW_WORKSPACE_DIR before OPENCLAW_PROFILE/home fallback, while preserving explicit per-agent and agents.defaults.workspace precedence. Add regression coverage in src/agents/workspace.test.ts or workspace.defaults.test.ts plus a default-agent consumer test in src/agents/agent-scope.test.ts or workspace-run.test.ts, and update docs only if the intended env precedence needs clarification.
