## Summary

Keep #40067 open. Current main still has no Playwright dialog-race suppression, and the PR addresses a plausible gateway crash, but the submitted shape hard-codes Playwright-specific behavior in generic core infra. The maintainer discussion and current plugin boundaries point to a browser-plugin-owned rewrite using the existing unhandled-rejection handler seam.

## Best Possible Solution

Keep the PR open for a rewrite. Move the Playwright dialog-race predicate into the browser plugin, register and unregister it from the browser control service lifecycle with `registerUnhandledRejectionHandler`, and add browser-plugin tests for the exact `Protocol error (Page.handleJavaScriptDialog): No dialog is showing` message plus wrapped or nested error forms. Core unhandled-rejection infra should stay generic.
