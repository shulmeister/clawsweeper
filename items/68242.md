## Summary

Keep open. Current `main` does not already contain the WhatsApp dispatcher `onError` logging from this PR, so there is no `implemented_on_main` close path. The PR also bundles a second, unrelated CLI memory-slot change, and the existing review discussion raises a credible correctness gap there: current config normalization defaults an unset memory slot to `memory-core`, but the proposed helper reads only `config.plugins?.slots?.memory`. Conservative maintainer cleanup should not auto-close a PR that still contains useful WhatsApp work and an unresolved review finding on an unrelated commit.
