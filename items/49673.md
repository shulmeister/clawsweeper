## Summary

Current `main` does not already implement this installer behavior, and the proposed PR still has an unresolved Linux-path regression risk. Conservative cleanup says keep it open rather than close as implemented/obsolete.

## Evidence

- **Current main still checks Node before any nvm load:** At `320d52a23e41a326287826219a20f1f29f334554`, `main()` goes straight from `install_homebrew` to `check_node()` / `install_node()` with no `try_load_nvm`-style step ahead of the version check.
  - file: [scripts/install.sh:2327](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/scripts/install.sh#L2327)
  - command: `nl -ba scripts/install.sh | sed -n '2320,2345p'`
  - sha: [320d52a23e41](https://github.com/openclaw/openclaw/commit/320d52a23e41a326287826219a20f1f29f334554)
- **Main only handles nvm after failure:** The only nvm-specific logic on `main` is the fallback messaging in `ensure_default_node_active_shell()`. It detects nvm and tells the user to run `nvm install/use/alias`, but it does not proactively source nvm before `check_node()`.
  - file: [scripts/install.sh:1353](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/scripts/install.sh#L1353)
  - command: `nl -ba scripts/install.sh | sed -n '1340,1415p'`
  - sha: [320d52a23e41](https://github.com/openclaw/openclaw/commit/320d52a23e41a326287826219a20f1f29f334554)
- **Linux install path does not switch PATH to a newly installed system Node:** `install_node()` updates PATH only in the macOS Homebrew branch. The Linux branch installs Node and returns without prepending the new binary, so if an old nvm-managed Node were activated before `check_node()`, the shell could still be on that old Node when `ensure_default_node_active_shell()` runs.
  - file: [scripts/install.sh:1408](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/scripts/install.sh#L1408)
  - command: `nl -ba scripts/install.sh | sed -n '1398,1485p'`
  - sha: [320d52a23e41](https://github.com/openclaw/openclaw/commit/320d52a23e41a326287826219a20f1f29f334554)
- **PR already has a concrete unresolved regression concern:** The provided PR review context flags that commit `f2c58ff0916ce2af96830f6dfd6990fdd36210ba` may break Linux installs when nvm is installed but its `default` alias still points to Node < 22.14, because the new helper would activate that old nvm Node before `check_node()`.
  - sha: [f2c58ff0916c](https://github.com/openclaw/openclaw/commit/f2c58ff0916ce2af96830f6dfd6990fdd36210ba)
