## Summary

Keep open. Current `main` still pins `sqlite-vec` 0.1.9 and still treats a successful extension load as sufficient without probing that vec functions or `vec0` actually registered, so the reported load-without-functions failure mode is not clearly fixed. Current code and docs do show a better fallback than the issue body claimed: when sqlite-vec is unavailable, search falls back to in-process cosine similarity over stored embeddings rather than keyword-only FTS.
