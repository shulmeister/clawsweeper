## Summary

Keep #43588 open. Current main still does not implement Slack `allowBots: "mentions"`: Slack config types/schema/metadata remain boolean-only, the Slack runtime only has a boolean bot-message gate, and there are no Slack mention-mode tests. The PR’s linked discussion is useful and its final commit addressed the Greptile cache-cleanup review, but the requested behavior still needs to be ported to the current `extensions/slack` plugin layout.

## Best Possible Solution

Keep #43588 open and port the focused implementation to the current bundled Slack plugin layout: update Slack config types/schema/metadata/help, resolve a tri-state `allowBots` mode in `extensions/slack/src/monitor/message-handler/prepare.ts`, apply the mention-only bot gate after normal mention resolution, and add Slack tests for no mention, explicit mention, DM, and implicit thread participation with cache cleanup. The linked feature issue #43587 being closed should not close this PR because current main still lacks the requested Slack behavior.
