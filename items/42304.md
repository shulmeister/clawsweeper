## Summary

Keep PR #42304 open. Current main does not implement Android inline Adaptive Card parsing or rendering: Android chat still renders every text part through Markdown, has no AdaptiveCard parser/composable files, and the Android message model/history parser only preserves text or base64 attachments. The PR discussion is concrete and tied to the existing third-party `openclaw-adaptive-cards` plugin, and there is no merged Android replacement or canonical open item that fully supersedes this PR.

## Best Possible Solution

Keep the PR open and ask for a rebase/design pass that fits current Android chat state. The likely path is either a carefully scoped Android renderer with parser tests and Compose safety guards, or a broader Android client contract for generic message presentation/A2UI-style rich content if maintainers want to avoid Adaptive Card-specific UI in the app.
