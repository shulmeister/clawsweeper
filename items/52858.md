## Summary

Keep open. Current `main` still lacks the legacy `systemEvent.message -> text` compatibility path that PR #52858 adds, so there is not strong evidence the bug is already fixed on `main`. The checked-out tree stayed clean during review.

## Evidence

- **Runtime path still assumes `systemEvent.text` exists:** `normalizePayloadToSystemText` on current `main` still does a direct `payload.text.trim()` for `systemEvent` payloads, with no fallback to a legacy `message` field.
  - file: [src/cron/service/normalize.ts:64](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/cron/service/normalize.ts#L64)
  - command: `nl -ba src/cron/service/normalize.ts | sed -n '56,74p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Normalizer deletes legacy `message` instead of promoting it to `text`:** In `coercePayload`, current `main` trims `message`/`text`, but when `kind === "systemEvent"` it only deletes `message` and other agent-turn fields; it does not migrate `message` into `text`. That means a persisted legacy payload can normalize to a `systemEvent` without usable text.
  - file: [src/cron/normalize.ts:147](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/cron/normalize.ts#L147)
  - command: `nl -ba src/cron/normalize.ts | sed -n '147,238p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Store load path has no dedicated migration/tracking layer from the PR:** Current `ensureLoaded` reads jobs from disk and passes each raw job through `normalizeCronJobInput`, but there is no `store-migration.ts` path or issue tracking for `legacySystemEventMessage` on `main`. The PR's store-migration files are absent from HEAD.
  - file: [src/cron/service/store.ts:37](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/cron/service/store.ts#L37)
  - command: `nl -ba src/cron/service/store.ts | sed -n '19,81p'; git ls-tree --name-only HEAD src/cron | rg 'store-migration'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Main-job execution still depends on normalized text being present:** `resolveJobPayloadTextForMain` calls `normalizePayloadToSystemText(job.payload)` and then checks whether the trimmed text is non-empty. With the current direct `payload.text.trim()` implementation, a legacy loaded `systemEvent` payload missing `text` is still unsafe.
  - file: [src/cron/service/jobs.ts:887](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/cron/service/jobs.ts#L887)
  - command: `nl -ba src/cron/service/jobs.ts | sed -n '887,893p'; nl -ba src/cron/service/timer.ts | sed -n '1194,1204p'`

- **Regression coverage from the PR is not present on current `main`:** The existing store-load regression test only covers invalid main jobs with `agentTurn` payloads. There is no current test for legacy persisted `systemEvent` payloads using `message` instead of `text`.
  - file: [src/cron/service.store-load-invalid-main-job.test.ts:35](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/cron/service.store-load-invalid-main-job.test.ts#L35)
