## Summary

Keep open. Current main still documents and implements the canonical config file as JSON5 at `openclaw.json`, with no `openclaw.yaml` or `openclaw.yml` discovery/parsing path. The later discussion's safer config-editing direction is partly implemented through `config.patch` and `openclaw config set`, but that does not implement the requested alternative YAML config file format.
