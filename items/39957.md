## Summary

Keep PR #39957 open. Current main still lacks the PR’s narrow suppression for unphased `message_end` block replies when an assistant turn has `stopReason === "toolUse"` plus structured tool-call blocks, so the reported Telegram-visible intermediate text leak remains plausible.

## Best Possible Solution

Keep the PR open and land either this narrow `message_end && stopReason === "toolUse" && structured tool-call block` suppression with regression coverage, or an equivalent display-phase fix that covers unphased tool-use assistant turns. Preserve the PR’s non-goals: normal `stop` turns, `toolUse` turns without tool-call blocks, and `text_end` block-reply delivery should continue to emit.
