## Summary

Keep PR #6457 open. Current main has the auth-reply threading portion, but the central Telegram `all_group_chats` command registration/cleanup is still absent, and the plugin-command `Command not found.` path still sends without topic thread params.

## Best Possible Solution

Keep this PR or a focused successor open until Telegram native command menu sync registers and clears both default and `all_group_chats` scopes, and until the plugin-command `Command not found.` fallback uses the same `resolveTelegramThreadSpec`/`buildTelegramThreadParams` path as other native command replies. The likely code changes belong in `extensions/telegram/src/bot-native-command-menu.ts` and `extensions/telegram/src/bot-native-commands.ts`, with focused tests for dual registration, dual cleanup, cache behavior, and threaded fallback replies.
