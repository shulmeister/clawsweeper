## Summary

PR #6457 should remain open. Current main includes part of the thread-routing work for native auth replies, but the central group-scope Telegram command registration/cleanup is still absent, and the plugin-command "Command not found." fallback still sends without topic thread params.

## Best Possible Solution

Keep this PR, or a focused successor, open until the Telegram plugin registers native command menus for both default and all_group_chats scopes, clears both scopes when native commands are explicitly disabled, threads the plugin-command "Command not found." fallback through the existing Telegram thread helpers, and adds focused tests for dual registration, dual cleanup, cache/error behavior, and threaded fallback replies.
