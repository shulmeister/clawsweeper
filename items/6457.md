## Summary

Keep PR #6457 open. Current main has implemented part of the topic-thread/auth-reply plumbing, but the central group-scope Telegram command registration and cleanup remain absent, and the plugin-command "Command not found." fallback still does not pass thread params. Related thread-handling PRs only supersede part of this PR, not the all-group command scope work.

## Best Possible Solution

Keep #6457 open or replace it with a focused Telegram plugin patch against current main. Implement dual default plus all_group_chats command registration in syncTelegramMenuCommands, clear both scopes when native commands are disabled, preserve topic params for the plugin-command not-found fallback via resolveTelegramThreadSpec/buildTelegramThreadParams, and add regression tests for dual registration, dual cleanup, retry/cache behavior, and threaded not-found replies.
