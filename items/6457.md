## Summary

Keep PR #6457 open. Current main has absorbed the auth-reply thread-preservation part, but the central Telegram group-scope command registration/cleanup is still missing, and the plugin-command "Command not found." fallback still sends without topic thread params. The latest maintainer-side review said to keep it open for the same reasons, and current main at d74f897c still matches those gaps.

## Best Possible Solution

Keep this PR or a canonical follow-up open until Telegram native command sync registers commands for both the default and `all_group_chats` scopes, clears both scopes when native commands are explicitly disabled, and threads the plugin-command `Command not found.` reply with the same `resolveTelegramThreadSpec`/`buildTelegramThreadParams` path used by other native command replies. The likely patch belongs in `extensions/telegram/src/bot-native-command-menu.ts` and `extensions/telegram/src/bot-native-commands.ts`, with focused tests for both command scopes, disabled cleanup, and threaded not-found behavior.
