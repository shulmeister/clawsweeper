## Summary

PR #6457 should stay open. Current main has the native-command auth/diagnostic threading portion, but the central Telegram all_group_chats command registration/cleanup is still absent, and the plugin-command "Command not found." fallback still sends without topic thread params. The prior March triage and April 26 Codex review in the PR discussion are still consistent with current main at d54d2d6b9b8aa28fb222d895c0f01f43933f4409.

## Best Possible Solution

Keep this PR or a focused successor open until Telegram native command menu sync registers both default and all_group_chats scopes, explicit native-command disable clears both scopes, and the plugin-command "Command not found." fallback uses the same topic-thread helper path as other native command replies. The likely touched files are extensions/telegram/src/bot-native-command-menu.ts, extensions/telegram/src/bot-native-commands.ts, and focused Telegram tests for dual registration, dual cleanup, cache/error behavior, and threaded fallback replies.
