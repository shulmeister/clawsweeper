## Summary

Keep PR #6457 open. Current main has the thread-helper foundation and thread-aware native auth replies, but it still lacks the PR's central Telegram group-scope command registration/cleanup and still has an unthreaded plugin-command "Command not found." fallback. The prior community and maintainer comments accurately match the current code at 9089e6b595ba33b4afc27b99c56ace80c4fbbc7b.

## Best Possible Solution

Keep #6457 open, or replace it with a focused Telegram plugin patch, until OpenClaw registers Telegram native command menus for both default and all_group_chats scopes, clears both scopes when native commands are disabled, applies the existing thread helpers to the plugin-command not-found fallback, and adds regression coverage for dual registration, dual cleanup, cache/retry semantics, and threaded fallback replies.
