## Summary

Keep open. Current `main` still restricts Web UI chat uploads to images and silently ignores unsupported file types, which matches the confusion reported in #56375 rather than proving the issue obsolete or fixed.

## Evidence

- **Web UI file picker is image-only:** `CHAT_ATTACHMENT_ACCEPT` is hard-coded to `"image/*"`, and supported upload MIME types are limited to values starting with `image/`. Current `main` therefore does not implement general file upload in the Web UI.
  - file: [ui/src/ui/chat/attachment-support.ts:1](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/ui/src/ui/chat/attachment-support.ts#L1)
  - command: `nl -ba ui/src/ui/chat/attachment-support.ts | sed -n '1,20p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Unsupported selections are ignored with no feedback:** The Web UI `handleFileSelect` path skips any file whose MIME type fails `isSupportedChatAttachmentMimeType(...)` and then clears the input. That explains a user-visible `no effect` outcome when selecting a non-image file.
  - file: [ui/src/ui/views/chat.ts:244](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/ui/src/ui/views/chat.ts#L244)
  - command: `nl -ba ui/src/ui/views/chat.ts | sed -n '244,295p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Chat send path still serializes uploads as images only:** When attachments are sent to the backend, `buildApiAttachments(...)` emits `{ type: "image", ... }` for every attachment. The current Web UI attachment contract is still image-specific end-to-end.
  - file: [ui/src/ui/controllers/chat.ts:441](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/ui/src/ui/controllers/chat.ts#L441)

- **Issue discussion already identified the likely limitation:** In the existing issue comment, `martinfrancois` noted that image uploads worked for them on newer builds, explicitly stated that the Web UI currently only supports image uploads, and linked #57707 as the broader all-file-types follow-up. That discussion supports keeping this open rather than auto-closing it.
