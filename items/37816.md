## Summary

Keep #37816 open. Current main still provides Telegram preview streaming, edited-in-place tool/progress lines, /verbose tool bubbles, and /reasoning preview visibility, but it does not implement the requested opt-in mode that emits each relevant agent activity step as a persistent separate Telegram message. The related Telegram reports in the supplied context are narrower closed issues and do not supersede this feature request.

## Best Possible Solution

Keep this open as Telegram plugin work. Implement an opt-in mode in extensions/telegram, with config schema, docs, and tests, that emits durable separate messages for relevant agent activity events while preserving the current preview-edit behavior by default. If current replyOptions hooks are not enough for memory or other activity surfaces, add a narrow additive channel/plugin event contract rather than hard-coding around it.

## Risks / Open Questions

- A full persistent Telegram activity mode could be noisy and may hit Telegram rate limits, especially in groups.
- Reasoning, memory, command output, and tool result summaries can expose sensitive data, so the mode should be explicit opt-in with filtering.
- The implementation needs deliberate NO_REPLY/internal-turn suppression so housekeeping turns do not leak into user-facing chats.
