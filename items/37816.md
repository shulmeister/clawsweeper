## Summary

Keep #37816 open. Current main has Telegram preview streaming, tool-progress preview lines, /verbose tool bubbles, and /reasoning visibility, but it still does not provide the requested opt-in Telegram mode that mirrors full Web UI agent activity as persistent per-step messages. The related issues in the body are narrower Telegram verbose/streaming/NO_REPLY bugs and do not supersede this feature request.

## Best Possible Solution

Keep this issue open as the canonical Telegram full-activity visibility feature request. The best implementation path is an explicit opt-in Telegram activity mode in the bundled Telegram plugin, backed by source/tests/docs, that emits durable per-step messages from agent event hooks while excluding NO_REPLY housekeeping and respecting reasoning, verbose, redaction, group privacy, and output truncation controls.

## Risks / Open Questions

- Any eventual full-activity Telegram mode should be opt-in and privacy-aware because it can expose reasoning, tool output, file paths, memory details, and command output in chats.
- The existing event hooks cover many tool/progress events, but maintainers should audit whether Web UI-only memory or lifecycle events need an additive event contract before promising exact parity.
