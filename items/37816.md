## Summary

Keep #37816 open. Current main has Telegram preview streaming, edited tool-progress lines, /verbose tool bubbles, and /reasoning visibility, but it still does not provide the requested opt-in Telegram mode that emits every agent activity step as persistent separate messages. The related issues in the provided context are narrower Telegram verbose/streaming/NO_REPLY bugs and do not supersede this feature request.

## Best Possible Solution

Keep tracking this as a Telegram plugin feature. The best implementation path is an opt-in config mode, for example channels.telegram.streaming.mode:"full" or channels.telegram.activityMessages:true, implemented in extensions/telegram with config schema/docs/tests. It should emit durable separate messages for relevant agent activity events while preserving the current preview-edit behavior as the default and explicitly suppressing internal NO_REPLY housekeeping turns.

## Risks / Open Questions

- A full persistent Telegram activity mode would increase message volume and may hit Telegram rate limits or make noisy chats harder to read.
- Reasoning, memory, tool output, and planning visibility have privacy and safety implications, so this should be opt-in with clear filtering for NO_REPLY/internal housekeeping.
