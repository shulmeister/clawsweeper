## Summary

Keep #37816 open. Current main still lacks the requested opt-in Telegram mode that emits each agent activity step as a durable, separate message. Telegram currently supports preview streaming modes, edited-in-place tool/progress lines, separate/streamed reasoning controls, and verbose tool bubbles, but those do not add up to Web UI parity for persistent per-activity visibility. The related Telegram issues in the provided context are narrower closed reports and do not supersede this feature request.

## Best Possible Solution

Keep this open as Telegram plugin work. Implement an opt-in mode in extensions/telegram with config schema, docs, and tests that emits durable separate messages for relevant agent activity events while preserving current preview-edit behavior by default. If the current replyOptions hooks are not enough for memory or other activity surfaces, add a narrow additive channel/plugin event contract rather than hard-coding around it.

## Risks / Open Questions

- A full persistent Telegram activity mode could be noisy and may hit Telegram rate limits, especially in groups.
- Reasoning, memory, command output, and tool result summaries can expose sensitive data, so the mode should be explicit opt-in with filtering.
- The implementation needs deliberate NO_REPLY/internal-turn suppression so housekeeping turns do not leak into user-facing chats.
