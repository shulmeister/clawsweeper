## Summary

Current main still does not implement the central request for automatic conditional tool/schema injection or default on-demand/summary workspace bootstrap injection. It has useful adjacent mitigations, but they are explicit policy controls, lightweight subagent/bootstrap modes, truncation, and compaction rather than automatic per-turn relevance routing.

## Best Possible Solution

Keep this issue open as the broad per-turn prompt overhead tracker. Use #14785 for the narrower tool-schema/schema-footprint work, but solve this issue with an automatic, policy-preserving tool relevance/schema-routing design plus a default or opt-in summary/index/on-demand mode for recurring workspace bootstrap files. Keep the shipped skill compacting, explicit `toolsAllow`, lightweight context, truncation, and compaction behavior as partial mitigations.

## Risks / Open Questions

- Automatic tool relevance routing still needs policy-preserving semantics for explicit allow/deny lists, safety/recovery tools, plugin tools, provider schema limits, and channel-specific capabilities.
- A default workspace bootstrap summary/index mode could hide standing instructions unless it has deterministic ordering, clear cache behavior, and a reliable path to read full files when needed.
- Closing this now would lose the broader tracker while #14785 only covers the narrower tool-schema footprint and not workspace bootstrap summary/index behavior.
