## Summary

Keep #12554 open. Current main has lower-level scoped exec-process machinery and can abort active chat/subagent runs, but it still lacks the requested session-wide operator surface to enumerate all exec processes for a session and terminate them as a unit. The existing HTTP kill endpoint is subagent-run control, not exec-process kill-all semantics.

## Best Possible Solution

Keep the issue open and implement an additive operator surface: for example sessions.exec-list plus sessions.kill with a killExec/terminateChildren option, or sessions.abort with an explicit killExec option. It should resolve the canonical session key, enumerate process registry/supervisor records for that scopeKey, terminate all matching exec sessions through supervisor cancellation plus process-tree fallback, return per-process status, add CLI/Gateway docs, and cover same-session kill, unrelated-session isolation, background exec, HTTP/RPC/CLI validation, and subagent cancellation interactions.

## Risks / Open Questions

- Do not close this as implemented based on POST /sessions/:sessionKey/kill; that endpoint controls subagent runs, not session-scoped exec process enumeration and termination.
- A correct implementation needs explicit authorization and isolation so one operator/session cannot inspect or terminate unrelated sessions' exec processes.
- The fix should preserve current background exec behavior unless the caller explicitly asks for stronger kill-exec semantics.
