## Summary

Keep open. Current main has stronger per-exec process management than the issue-era behavior, including process-tree killing for individual background exec sessions, but it still does not provide a session-level `sessions_kill`/`sessions.abort --kill-exec` path that enumerates and terminates all exec processes spawned by a session. The issue is actionable and not superseded by the existing `process` tool because that tool requires individual process session IDs and is scoped to the agent/tool context rather than exposed as a session-wide kill/list operator action.
