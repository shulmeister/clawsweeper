## Summary

Keep #12554 open. Current main still lacks a first-class session-level operator action that enumerates and terminates all exec/background process sessions spawned by a conversation or subagent session. Existing paths either abort chat runs, kill subagent runs, or manage one background exec session at a time through the scoped `process` tool.

## Best Possible Solution

Keep #12554 open. The best path is to add a first-class session-level exec management surface backed by the existing session/scope metadata: list active background exec/process records for a target OpenClaw session key, cancel them as a unit through the process supervisor/process registry, expose it through an operator RPC/CLI such as `sessions.exec-list` and `sessions.kill --kill-exec` or `sessions.abort --kill-exec`, and add tests proving it terminates only the target session's execs.
