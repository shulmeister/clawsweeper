## Summary

Keep open. Current main has individual background exec management and newer subagent kill paths, but it still does not expose a session-level operator action that lists and terminates all exec processes spawned by a conversation/subagent session. The existing `sessions.abort`/`sessions.delete`/`/sessions/:key/kill` paths abort active runs or subagent runs, while backgrounded exec processes intentionally survive tool aborts and must be managed one process session at a time through the `process` tool.

## Best Possible Solution

Keep #12554 open. The best implementation is a first-class session-level exec management path: add process-supervisor registry support to list and cancel active runs by OpenClaw session key/scope, wire it through an operator RPC/CLI such as `sessions.exec-list` and `sessions.kill --kill-exec` or an explicit `sessions.abort --kill-exec`, and add tests proving background execs spawned by that session are enumerated and terminated without exposing unrelated sessions.
