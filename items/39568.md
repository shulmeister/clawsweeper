## Summary

Keep #39568 open. Current main still matches the reported Feishu reply-mention bug: mention detection returns false when Feishu supplies a non-empty `mentions` array that omits the bot, there is no orphaned `@_user_N` placeholder fallback, and reply `root_id` still flows into thread-reply routing. The linked fix PR #39569 is still open and unmerged, and its diff covers the missing behavior with proposed regression tests.

## Best Possible Solution

Keep #39568 open until #39569 or an equivalent Feishu plugin fix lands. The maintainer path is to update `checkBotMentioned()` to detect orphaned reply mention placeholders, clear or suppress root/thread routing for explicit bot @mentions in reply messages, and add regression tests covering both mention detection and non-thread reply behavior.
