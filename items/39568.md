## Summary

Keep #39568 open. Current main still has the reported Feishu reply-mention failure path: bot mention detection returns from the structured `mentions` array and does not detect orphaned `@_user_N` placeholders, while `root_id`/`threadReply` still force Feishu thread replies. The linked PR #39569 remains relevant but is open and unmerged in the provided GitHub context.

## Best Possible Solution

Keep #39568 open until #39569 or an equivalent Feishu plugin fix lands. The maintainer path is to make `checkBotMentioned()` detect Feishu reply events where the bot mention placeholder survives but the structured mention is dropped, suppress `rootId`/`threadReply` only for normal group reply @mentions that should stay in the main chat, and add regression tests for both mention detection and non-thread reply dispatch.
