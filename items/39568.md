## Summary

Current main still appears to contain the reported Feishu bug: bot mention detection returns from the `mentions` array without checking orphaned text placeholders, and `parseFeishuMessageEvent()` still preserves `root_id`. The linked PR #39569 directly tracks the requested fix but is open and unmerged in the provided GitHub context, so the issue should stay open.

## Best Possible Solution

Keep #39568 open until the Feishu plugin fix lands or a maintainer supersedes it. The likely path is to review/land PR #39569 or an equivalent patch that detects orphaned reply @mention placeholders, avoids unwanted root thread routing for explicit reply mentions, and adds regression tests for both behaviors.

## Evidence
