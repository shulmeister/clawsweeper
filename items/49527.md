## Summary

## Close Comment

Closing from a Codex review because this regression is already implemented on current `main` and was shipped in `2026.3.22`. The changelog entry at `CHANGELOG.md:2448` under `CHANGELOG.md:2206` matches this report directly: OpenAI-compatible providers were fixed to stop reporting all-zero usage. Current `main` also treats `vllm` as a local streaming-usage-compatible provider (`src/agents/openai-completions-compat.ts:33`), keeps `supportsUsageInStreaming` enabled for it (`src/agents/openai-completions-compat.ts:104`), always requests `stream_options.include_usage` on streaming OpenAI-compatible calls (`src/agents/openai-transport-stream.ts:1725`), and the usage view aggregates transcript `message.usage` into daily/session `totalTokens` (`src/infra/session-cost-usage.ts:134`, `src/infra/session-cost-usage.ts:185`). There is also vLLM/local-provider regression coverage in `src/agents/openai-completions-compat.test.ts:40`. If this still reproduces on a current build, please open a fresh issue with a transcript sample and the raw provider usage payload from the affected vLLM response.
