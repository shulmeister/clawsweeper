## Summary

Keep open. Current main still does not implement the two requested reliability fixes: Discord voice autoJoin is wired only to READY, not RESUMED, and the join/teardown path still calls @discordjs/voice connections without an idempotent destroyed-state guard or scoped stale-connection cleanup. The linked PR #40676 tracks the right bug but remains open and has unresolved review concerns, so it is not a safe supersession for cleanup.

## Best Possible Solution

Keep #40665 open as the canonical bug until a maintainer-reviewed fix lands. The best implementation is a focused Discord plugin change that listens for RESUMED as well as READY, makes voice teardown idempotent when @discordjs/voice reports Destroyed, and scopes any stale connection cleanup to the owning account/manager, likely by using an account-scoped @discordjs/voice group or another explicit ownership check. Add regression coverage for RESUMED autoJoin, already-destroyed cleanup, and multi-account same-guild safety; then link or revise #40676 rather than closing this issue now.
