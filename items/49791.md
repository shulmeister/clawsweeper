## Summary

Keep open. The thread now points to two separate things: the original `/think` failure was at least partly explained by Feishu group command authorization, but current `main` still has a silent-drop path for unauthorized standalone commands in groups, so there is not strong enough evidence to close this as fixed or obsolete.

## Evidence

- **Feishu group commands are still authorized from sender allowlists, not just groupPolicy:** Current Feishu inbound handling computes `commandAuthorized` for group messages from per-group `allowFrom` or top-level `channels.feishu.allowFrom`. `groupPolicy` alone does not authorize `/think` or other standalone commands.
  - file: [extensions/feishu/src/bot.ts:554](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/extensions/feishu/src/bot.ts#L554)
  - command: `nl -ba extensions/feishu/src/bot.ts | sed -n '497,570p'`
  - sha: [320d52a23e41](https://github.com/openclaw/openclaw/commit/320d52a23e41a326287826219a20f1f29f334554)
- **Current tests codify that behavior:** Feishu tests explicitly cover group command authorization from per-group `allowFrom` and fallback to top-level `allowFrom`, confirming that command authorization is expected to depend on those lists.
  - file: [extensions/feishu/src/bot.test.ts:958](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/extensions/feishu/src/bot.test.ts#L958)

- **Unauthorized standalone commands still return no reply:** The reply path still returns `undefined` for unauthorized whole-message control commands, which matches the reporter's 'queuedFinal=false, replies=0' symptom and means the UX problem described in follow-up comments is still present on current main.
  - file: [src/auto-reply/reply/get-reply-run.ts:338](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/src/auto-reply/reply/get-reply-run.ts#L338)

That aligns with the reporter's later comment that adding group command authorization made `/think` work, but it does not resolve the silent-failure UX concern.
  - file: [docs/tools/slash-commands.md:18](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/docs/tools/slash-commands.md#L18)
