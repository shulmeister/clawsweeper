## Summary

Current `main` already supports workspace-relative avatar paths through gateway-side normalization, so this PR's frontend-only change is redundant/obsolete.

## Close Comment

Closing after Codex review: current `main` already implements workspace-relative avatar support, but the implementation lives on the gateway path instead of this PR's UI-only approach. `src/gateway/session-utils.ts:118` resolves workspace-relative `identity.avatar` files into renderable `data:` URLs, `listAgentsForGateway` exposes that as `identity.avatarUrl` at `src/gateway/session-utils.ts:696`, and `agents.list` returns it from `src/gateway/server-methods/agents.ts:429`. Assistant bootstrap avatars are also normalized through `src/gateway/control-ui-shared.ts:35` and served from `src/gateway/control-ui.ts:738`, with coverage in `src/gateway/control-ui.http.test.ts:563` and `:644` for both plain and base-path-mounted dashboards. This behavior is already present in the shipped `v2026.4.22` release (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), so this PR is obsolete relative to current `main`.
