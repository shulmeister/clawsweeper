## Summary

Keep #23451 open. Current main has exec-specific approval controls and plugin-level `before_tool_call.requireApproval`, but it still lacks the requested built-in `tools.confirmation` risk-tier policy and assistant-text-aware pause before the first same-response tool call executes.

## Best Possible Solution

Keep #23451 open for core agent tool-execution work. The likely path is a documented `tools.confirmation` config/schema/help surface, deterministic tool risk classification, access to same-response assistant text before first tool execution, user-visible text flush before holding tools, approval/timeout execute-or-skip semantics, and transcript handling that preserves existing plugin `before_tool_call.requireApproval` as a lower-level custom policy hook.
