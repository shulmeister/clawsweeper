## Summary

Keep open. Current main fixes the gateway.auth.token SecretRef drift/persistence sub-symptom, but the central macOS LaunchAgent plaintext projection remains: install planning still collects durable .env/config/auth-profile environment values, and the LaunchAgent renderer still writes environment entries inline under plist EnvironmentVariables.

## Best Possible Solution

Keep this issue open and finish the macOS install-path fix: stop putting secret-bearing durable env/auth-profile/provider/channel values into LaunchAgent plist EnvironmentVariables, load state-dir .env or supported SecretRefs at gateway startup through a launchd-safe runtime entrypoint or wrapper, scrub stale plist entries, and add coverage for install --force, update/reinstall, ENV/FILE/structured SecretRefs, auth profiles, provider/channel secrets, and gateway.auth.token.
