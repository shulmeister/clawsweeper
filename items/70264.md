## Summary

Current main still matches the reported heartbeat-enabled ACP parent-wake gap. The ACP stream relay queues `acp:spawn:stream` wakes, but that reason is still not action-priority, heartbeat prompt routing still only special-cases exec and cron events, and the direct terminal mirror path does not request a parent wake after successful delivery. #56442 is a relevant open candidate fix direction, but it has not landed, so this valid bug should stay open.

## Best Possible Solution

Keep #70264 open. The best path is a core fix that separates lightweight `streamTo: "parent"` progress relay from opt-in terminal completion routing, likely by refreshing/adapting #56442 or an equivalent `parentUpdates`/`wakeOwnerOnDone` design, plus regression coverage for heartbeat-enabled ACP completion prompts and the heartbeat-disabled/yield-adjacent path tracked in #52249.
