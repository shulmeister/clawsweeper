## Summary

Keep PR #69638 open. Current main still does not accept a token-only `connectAuth` payload when `gateway.auth.mode` is `password`: the password branch reads only `connectAuth.password`, while the WebSocket handshake preserves `token` and `password` as distinct auth fields. The linked issue #51953 provides a concrete Rabbit R1 + Tailscale Funnel reproduction, and the PRâ€™s provided diff includes the requested fallback plus the edge tests Greptile asked for, so the work remains plausibly useful rather than obsolete.

## Best Possible Solution

Keep the PR open and have a maintainer review/merge a narrow implementation: in password mode, prefer `connectAuth.password` and fall back to `connectAuth.token`, with regression tests for token-only success, token mismatch, and password-over-token precedence. That directly addresses #51953 without broad auth-surface changes.
