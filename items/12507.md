## Summary

Keep open. Current main still lacks skill signature/hash verification and unsigned-skill capability gating. Recent ClawHub archive-integrity work applies to code/bundle plugin installs and explicitly excludes skill-family packages; native skill install and runtime loading still expose SKILL.md based on path/frontmatter/invocation checks rather than trusted integrity metadata.

## Best Possible Solution

Keep #12507 open as the canonical product/security tracking issue. A solid implementation would add a skill integrity/signing contract to ClawHub skill version metadata, persist expected hash/signature/publisher trust in `.clawhub` origin/lock state, verify downloads during `openclaw skills install/update`, reverify SKILL.md or package contents at runtime load/snapshot time, expose verification status on SkillEntry, and wire config/capability policy so unsigned or unverified skills can be blocked or run with restricted tool/network capability according to maintainer-approved migration rules.
