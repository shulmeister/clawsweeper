## Summary

Current main still lacks native SKILL.md signature or trusted hash verification. The ClawHub skill API type has no digest/signature/publisher-key contract, the native skill installer downloads and extracts archives without comparing them to trusted version metadata, origin/lock metadata persists only registry/slug/version/time, and runtime skill loading reads local SKILL.md files into SkillEntry/SkillSnapshot without verification status or capability gating. The existing ClawHub plugin archive verification is adjacent but explicitly excludes skill-family packages.

## Best Possible Solution

Keep #12507 open as the canonical native skill integrity work. A complete fix should add trusted skill-version digest/signature/publisher metadata to the ClawHub skill API, verify native skill archives before extraction, persist expected integrity/signature state in .clawhub origin/lock metadata, reverify SKILL.md or full skill contents during skill snapshot/load, expose verification state on SkillEntry/status surfaces, and define maintainer-approved policy, docs, and tests for unsigned or unverified skills and capability restrictions.
