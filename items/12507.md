## Summary

Keep open. Current main still lacks SKILL.md signature/hash verification and unsigned-skill capability gating. ClawHub package archive integrity exists for plugin installs, but the code explicitly routes skill-family packages away from that verifier into the native skill install path, which persists no trusted hash/signature metadata and runtime-loads local SKILL.md files without a trust status.

## Best Possible Solution

Keep #12507 open as the canonical skill integrity/signing product issue. A complete fix should add trusted hash/signature/publisher metadata to ClawHub skill versions, persist it in .clawhub origin/lock state, verify native skill install/update downloads before extraction, reverify SKILL.md or full skill contents at runtime snapshot/load time, expose verification status on SkillEntry, and wire maintainer-approved config/capability policy for unsigned or unverified skills with docs and regression tests.
