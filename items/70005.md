## Summary

Keep #70005 open. Current main has partial noise mitigations for dreaming, including transcript filtering and a small REM tag blacklist, but it does not implement the requested semantic-importance material selection. REM reflections still rank concept tags by occurrence strength, candidate truths still score frequency/recall-style signals, and session ingestion still records transcript lines after length/time/dedupe filters without a decision/preference/unusual-event filter.

## Best Possible Solution

Keep the issue open for memory-core product work. The likely fix belongs in `extensions/memory-core/src/dreaming-phases.ts` and/or the short-term recall pipeline: add a semantic importance or grounded event extraction pass before REM/narrative inputs, prioritize decisions/preferences/conclusions/unusual events over repeated role/scaffolding tokens, and add regression tests where a low-frequency meaningful user decision beats high-frequency generic transcript scaffolding. Only add a public `dreaming.promptStyle` or strategy config if maintainers want this behavior tunable rather than as the default.
