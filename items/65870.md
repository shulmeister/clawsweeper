## Summary

Keep open. Current main still has the reported Brave Search legacy docs URL in both Brave provider surfaces, still has the `/brave-search` to `/tools/brave-search` redirect, and still keeps a separate legacy `docs/brave-search.md` alongside the canonical `docs/tools/brave-search.md`. The related PR #65892 is an open, unmerged fix attempt, so the issue remains actionable rather than implemented on main.

## Best Possible Solution

Keep #65870 open until a scoped fix lands on main: update both Brave `docsUrl` fields to `https://docs.openclaw.ai/tools/brave-search`, then either remove or convert `docs/brave-search.md` into an intentional redirect/stub so the canonical docs source is `docs/tools/brave-search.md`. PR #65892 is the active implementation attempt but should not close the issue until merged or superseded by an equivalent main-branch fix.
