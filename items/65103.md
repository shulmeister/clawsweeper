## Summary

Current `main` no longer matches the exact Slack DM reproduction path described in the report, because Slack runs tool-result output in markdown mode and fenced `MEDIA:` lines are ignored by the reply-directive parser. But the underlying dual-shape implementation the report identified is still present on `main`: generation tools still emit both legacy `MEDIA:` lines and structured `details.media`, and the embedded tool-result handler still only does explicit dedupe work for `toolResultFormat === "plain"`. That is not strong enough evidence to auto-close conservatively.
