## Summary

Keep open. Current main still has the bug this PR targets: the Anthropic Messages transport preserves and serializes caller-supplied temperature for non-thinking Claude Opus 4.7 requests. The PR is a plausible follow-up to #72123 and #70582, but it still needs formatting cleanup and focused regression coverage before merge.

## Best Possible Solution

Keep this PR open or land an equivalent focused fix on main: strip temperature for claude-opus-4-7 and claude-opus-4.7 in the Anthropic Messages option and payload paths, preserve temperature for supported Anthropic models, add regression tests for omit/pass-through behavior, and explicitly decide whether the Anthropic Vertex plugin path is in scope.
