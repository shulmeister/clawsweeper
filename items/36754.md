## Summary

Current main still skips symlinked plugin directories during automatic plugin root scanning, so #36754 remains a real bug rather than a close candidate.

## Best Possible Solution

Keep #36754 open and implement symlink-aware root scanning in `discoverInDirectory`: resolve `entry.isSymbolicLink()` targets with a following stat/realpath step, skip or diagnose broken symlinks, preserve existing package containment checks, and add regression tests for symlinked global/workspace plugin directories plus broken symlink handling.
