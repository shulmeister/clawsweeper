## Summary

Current `main` still matches the reported bug: extension-root auto-discovery only treats `Dirent.isDirectory()` entries as package directories, so symlinked plugin directories under `~/.openclaw/extensions` or workspace `.openclaw/extensions` still fall through. The explicit `plugins.load.paths` workaround remains a separate `fs.statSync()` path, and there is no focused regression test for discovering a symlinked plugin directory. Related fix PRs in the provided context were closed unmerged, so this issue should stay open.

## Best Possible Solution

Keep #36754 open until `discoverInDirectory` safely resolves symlink dirents for plugin package directories, skips or diagnoses broken symlinks, preserves existing package-boundary and hardlink checks, and adds a focused regression test proving a symlinked plugin directory under the global/workspace extensions root is discovered.
