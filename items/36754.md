## Summary

Keep #36754 open. Current main still skips symlinked plugin directories during automatic workspace/global plugin discovery; the explicit `plugins.load.paths` workaround follows symlinks through a separate code path and does not fix extension-root scanning.

## Best Possible Solution

Keep #36754 open. The best fix is a narrow change in `src/plugins/discovery.ts`: when `discoverInDirectory` encounters a symlinked entry, resolve the target type before the file/directory branch, skip or diagnose broken symlinks, and then reuse the existing manifest, bundle, index, candidate-validation, and path-safety gates. Add regression tests for symlinked plugin directories under both global and workspace extension roots while keeping the existing symlink escape tests intact.

## Evidence
