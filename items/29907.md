## Summary

Keep #29907 open. Current main still lacks a non-LLM cron payload for direct exec/Lobster execution, condition evaluation, and conditional agent handoff. Related #18160 and PR #47628 cover the direct exec/script subset, but that work is still open/unshipped and does not fully cover this issue's condition/onCondition Lobster handoff. This is not a ClawHub close because current plugin hooks can inspect/update cron jobs but cannot register a new cron payload executor, and the core schema/runtime still reject new payload kinds.

## Best Possible Solution

Keep #29907 open unless maintainers explicitly consolidate it into a broader canonical issue. The best implementation path is an additive core cron/protocol feature: a deterministic exec/script/Lobster payload runner with timeout/abort, run-log/state, delivery/failure semantics, security/approval policy, docs, CLI/UI/agent-tool schema support, and a condition DSL that can silently no-op or enqueue an agentTurn with captured output when the condition passes.
