## Summary

Keep #29907 open. Current main still only supports `systemEvent` and `agentTurn` cron payloads, so there is no non-LLM cron executor for `exec`/`lobster`, no condition DSL, and no conditional handoff into an agent turn. Related #18160 and PR #47628 cover part of the direct script/exec idea, but they remain open/unshipped and do not cover this issue's Lobster pipeline plus `condition`/`onCondition` behavior.

## Best Possible Solution

Keep #29907 open unless maintainers explicitly consolidate it into a broader canonical cron-execution issue. The best product path is an additive core cron/protocol feature: deterministic exec/script/Lobster payload execution with timeout/abort, run logs/state, security and approval policy, condition evaluation, silent no-op behavior, and conditional enqueue/delivery of an `agentTurn`, with CLI, Control UI, docs, and protocol tests updated together.
