## Summary

Keep #29907 open. Current main still only supports cron payloads that enqueue a main-session system event or run an isolated/current/session agent turn. There is no core or documented plugin-facing cron executor for deterministic `exec` or Lobster pipeline runs, no `condition`/`onCondition` contract, and related #18160 is only the narrower direct-exec subset.

## Best Possible Solution

Keep #29907 open unless maintainers create a canonical broader cron-executor tracker. The best path is either a core cron payload implementation or a stable documented plugin-facing cron executor seam covering deterministic `exec` and Lobster pipeline execution, timeout/sandbox/output-cap policy, condition DSL (`exitCode`, stdout, JSON path, always), silent no-op runs, conditional `onCondition` agentTurn enqueue/delivery, run-log/task-state visibility, CLI and Control UI support, public docs, and protocol/runtime tests. Keep #18160 as the narrower direct-exec subset if it remains separate.
