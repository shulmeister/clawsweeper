## Summary

Keep open. Current main still installs fish completion by sourcing the cached fish script from `.config/fish/config.fish`; it does not install `openclaw.fish` into fish's auto-loaded `.config/fish/completions/` directory. The directly related PR #27135 is useful prior art, but it is closed unmerged in the provided GitHub context, so it does not supersede or resolve this issue.

## Best Possible Solution

Continue tracking this issue. The likely fix belongs in the CLI completion runtime and its doctor/setup/update callers: install fish completions to `.config/fish/completions/<binName>.fish` under the user's home directory, detect that installed state there, clean up legacy `config.fish` source blocks safely, update reload messaging and docs, and add focused regression tests. PR #27135 is useful prior art but cannot close the issue because it did not land.
