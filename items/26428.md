## Summary

Keep open. Current main still installs fish completions by adding a source block to `~/.config/fish/config.fish`; it does not install `openclaw.fish` into fish's auto-loaded `~/.config/fish/completions/` directory. The directly related PR #27135 addressed this behavior but is closed unmerged, so it is useful prior art rather than a superseding fix.

## Best Possible Solution

Continue tracking the issue. The likely fix belongs in the CLI completion runtime and its callers: install fish completions to `~/.config/fish/completions/<binName>.fish`, update installed-state/doctor/setup checks and messaging for that target, safely clean legacy `config.fish` source blocks, and add focused regression tests plus docs. PR #27135 is useful prior art but cannot close the issue because it did not land.
