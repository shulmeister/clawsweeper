## Summary

Current main still does not implement #26428. Fish completion installation, detection, doctor/update repair, and docs remain shell-profile/source-block oriented, while the requested auto-loaded fish completions directory remains unimplemented. PR #27135 is useful prior art but was closed unmerged.

## Best Possible Solution

Keep #26428 open. The best fix is a fish-specific path in `src/cli/completion-runtime.ts` that writes or copies the cached fish completion to `$XDG_CONFIG_HOME/fish/completions/<binName>.fish` or `~/.config/fish/completions/<binName>.fish`, detects that file as installed, removes legacy OpenClaw source blocks from `config.fish`, updates doctor/update messaging and https://docs.openclaw.ai/cli/completion, and adds focused regression tests. PR #27135 is the relevant closed prior-art implementation to review or rebase.
