## Summary

Keep open. Current main appears to have strengthened bundled-plugin discovery for config validation, so the original startup warning may be partly or fully addressed, but the issue discussion still matches a live misleading-warning path: the security/doctor audit only treats loaded channel plugins as bundled when checking `plugins.allow`, which excludes bundled provider/browser/memory plugin ids like the commenter reported. With no direct reproduction proof on current main and no regression coverage for bundled non-channel allowlist entries, closure would be premature.
