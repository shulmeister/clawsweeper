## Summary

Current `main` already contains a Feishu-specific fix for reasoning-stream previews: preview callbacks are gated to `reasoningLevel === "stream"`, the streamed reasoning text is flattened into a single blockquoted preview instead of nested raw reasoning frames, and regression tests cover the exact before-answer/reasoning-only cases described in the issue. The fixing commit is on `main` after the latest tagged release, so this is implemented but not yet tied to a release tag.

## Evidence

- **Feishu reasoning preview is now explicitly gated by session mode:** `resolveFeishuReasoningPreviewEnabled` only returns true when the Feishu session store entry has `reasoningLevel === "stream"`, which matches the issue's "Reasoning Stream" setting and avoids wiring the preview path outside that mode.
