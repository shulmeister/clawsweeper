## Summary

Keep open. Current `main` still rejects `streamTo` and `resumeSessionId` for `runtime="subagent"` instead of stripping them or returning stronger corrective behavior, so the feature request is not implemented on `main` and remains actionable.

## Evidence

- **Current tool code still hard-rejects ACP-only fields on subagent runtime:** `createSessionsSpawnTool()` reads `streamTo` and `resumeSessionId`, then returns `status: "error"` when either is present and `runtime !== "acp"`. There is no normalization/auto-strip path for `runtime="subagent"`.
  - file: [src/agents/tools/sessions-spawn-tool.ts:229](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/tools/sessions-spawn-tool.ts#L229)
  - command: `nl -ba src/agents/tools/sessions-spawn-tool.ts | sed -n '150,280p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Tests lock in the rejection behavior:** Current tests explicitly assert that `resumeSessionId` without ACP is rejected, and that `streamTo` with `runtime: "subagent"` returns an error and does not spawn either runtime.
  - file: [src/agents/tools/sessions-spawn-tool.test.ts:354](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/tools/sessions-spawn-tool.test.ts#L354)
  - command: `nl -ba src/agents/tools/sessions-spawn-tool.test.ts | sed -n '330,430p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Public docs still describe these fields as ACP-only:** ACP docs document `resumeSessionId` and `streamTo` under the ACP `sessions_spawn` interface, and the subagent docs list subagent params without either field. That matches the current hard rejection rather than the requested forgiving behavior.
  - file: [docs/tools/acp-agents.md:319](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/tools/acp-agents.md#L319)
  - command: `nl -ba docs/tools/acp-agents.md | sed -n '285,335p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Subagent docs do not claim auto-normalization:** The subagent tool-params section documents the supported subagent fields and omits `streamTo`/`resumeSessionId`, so there is no user-facing evidence that current `main` already strips ACP-only fields for subagent spawns.
  - file: [docs/tools/subagents.md:88](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/tools/subagents.md#L88)
