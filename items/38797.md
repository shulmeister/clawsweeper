## Summary

Current `main` does not implement per-agent isolation for `memory-lancedb`, and there is still an open linked PR for this feature. The issue remains actionable and should stay open.

## Evidence

- **Shared memory schema has no agent scope:** `MemoryEntry` and the table bootstrap only include `id`, `text`, `vector`, `importance`, `category`, and `createdAt`; there is no `agentId` or similar scope key in stored rows.
  - file: [extensions/memory-lancedb/index.ts:30](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/extensions/memory-lancedb/index.ts#L30)
  - sha: [51186d272543](https://github.com/openclaw/openclaw/commit/51186d2725439646f7fea0e59bb466f957404c46)
- **Auto-recall and auto-capture stay global:** `before_prompt_build` calls `db.search(...)` with no agent filter, and `agent_end` writes memories via `db.store(...)` with no agent identifier. The hook handlers also ignore the hook context parameter entirely.
  - file: [extensions/memory-lancedb/index.ts:582](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/extensions/memory-lancedb/index.ts#L582)
  - sha: [51186d272543](https://github.com/openclaw/openclaw/commit/51186d2725439646f7fea0e59bb466f957404c46)
- **No config surface for isolation:** The config parser only allows `embedding`, `dreaming`, `dbPath`, `autoCapture`, `autoRecall`, `captureMaxChars`, and `storageOptions`. The shipped manifest schema matches that and exposes no `agentScoping`, memory group, or per-agent override setting.
  - file: [extensions/memory-lancedb/config.ts:94](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/extensions/memory-lancedb/config.ts#L94)
  - sha: [51186d272543](https://github.com/openclaw/openclaw/commit/51186d2725439646f7fea0e59bb466f957404c46)
- **Core hook context already carries agent identity:** Plugin hook agent context already exposes optional `agentId`, so the missing behavior is in `memory-lancedb` rather than an incoherent request for a nonexistent seam.
  - file: [src/plugins/hook-types.ts:154](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/src/plugins/hook-types.ts#L154)
  - sha: [51186d272543](https://github.com/openclaw/openclaw/commit/51186d2725439646f7fea0e59bb466f957404c46)
- **Related implementation work is still open:** Remote refs still include `refs/pull/40578/head` and `refs/pull/40578/merge`, matching the linked PR from the issue timeline, so this is not a stale no-path-forward report.
  - command: `git ls-remote origin 'pull/40578/head' 'pull/40578/merge'`
