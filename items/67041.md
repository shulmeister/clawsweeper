## Summary

Keep open. Current main still does not add the PR's requested default `logs.tail` redaction coverage for standalone JWTs, Basic auth headers, or the named security headers. The PR is a plausible focused security hardening change, but it needs the existing review nits addressed before merge.

## Best Possible Solution

Keep the PR open and ask for a small fixup: add the requested patterns to the shared default redaction set used by `logs.tail`, remove the Basic auth trailing word-boundary issue, add focused tests for standalone JWTs, Basic auth with padding, and the named headers, and ideally include a `logs.tail` regression test proving returned lines are redacted.
