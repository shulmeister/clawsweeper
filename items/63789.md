## Summary

Keep #63789 open. The PR has explicit maintainer signal that it remains useful as the lower-level transport/acknowledgement primitive, and current main does not implement the requested handoff_id, structured delivered/accepted/queued/rejected acknowledgement contract, control-result visibility, or append-only handoff ledger.
