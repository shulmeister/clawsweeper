## Summary

Keep open. Current main fixes the concrete active-memory example by passing an explicit `toolsAllow: ["memory_search", "memory_get"]` and filtering the embedded run tool list, and that partial fix is present in `v2026.4.24`. But the broader issue asks for disabled-by-default / explicit per-agent least-privilege behavior, and current main still treats unrestricted agents with no tool policy as unrestricted rather than disabled by default.

## Best Possible Solution

Keep the shipped `toolsAllow` path for active-memory and other one-shot embedded agents. For #66781, maintainers should make an explicit product decision on global/no-profile tool defaults: either keep the current configurable model and document it as intentional, or introduce a narrower default baseline such as `minimal`/explicit allowlists for new agents and add migration/docs/tests around that behavior.
