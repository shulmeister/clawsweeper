## Summary

Keep open. Current main still only applies embedded `toolsAllow` after `createOpenClawCodingTools()` has already applied config/tool-policy filtering, and the proposed PR #60842 remains open/unmerged in the provided GitHub context. The observable bug described in #60841 is therefore not implemented on main.

## Evidence

- **Embedded run still post-filters toolsAllow:** `runEmbeddedAttempt` builds `allTools` with `createOpenClawCodingTools(...)` and then returns `applyEmbeddedAttemptToolsAllow(allTools, params.toolsAllow)`. The runtime allowlist is not forwarded into tool creation, so tools removed earlier cannot be re-added here.

  - file: [src/agents/pi-tools.ts:259](https://github.com/openclaw/openclaw/blob/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54/src/agents/pi-tools.ts#L259)
