## Summary

Current main already covers the reported Discord media-loss case: empty native component objects are ignored, simple/trivial component specs with media downgrade to the classic send path, component sends with media auto-inject a Discord file block when needed, and the scoped `message` tool no longer requires Discord-native `components`.

## Close Comment

Closing from a Codex review because current `main` already covers this report. The Discord send path now ignores empty native component objects (`extensions/discord/src/actions/runtime.messaging.ts:91`), downgrades classic/trivial component specs with media back to `sendMessageDiscord` (`extensions/discord/src/send.components.ts:249`), and auto-adds the matching Discord `file` block when a component message includes media (`extensions/discord/src/send.components.ts:49`). The scoped `message` tool also no longer requires Discord-native `components`; it exposes the shared optional `presentation` payload instead (`src/agents/tools/message-tool.ts:90`, `src/agents/tools/message-tool.test.ts:446`). These behaviors are present in the shipped `v2026.4.22` release (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), so there is not a remaining maintainer action on this issue.
