## Summary

Keep this issue open. Current main has some backend thinking default resolution in `chat.history`, but the Control UI session picker still builds its default label from `sessions.list` rows/defaults that use model-only fallback defaults, and reasoning visibility still reads only the persisted session `reasoningLevel` with an `off` fallback. The related #66098 PR is an open partial candidate, not a merged or closing fix.

## Best Possible Solution

Review and either land #66098 or a successor that updates the gateway/UI contract to expose effective fresh-session thinking and reasoning defaults, updates the Control UI to render those effective values, adds regression coverage for fresh sessions with configured defaults, and explicitly resolves whether `agents.defaults.reasoningDefault` should be a supported global config key.
