## Summary

Keep open. The issue is recent, reporter-authored, has no protected maintainer label, and the observable bug is not proven fixed on current main. Current code still carries Telegram display names into channel context/history, but the suggested root cause remains unverified: current OpenAI Responses serialization and pi-ai 0.70.2 Chat Completions serialization do not show user `senderName` becoming a model API message `name` field. The linked PR #62441 is draft/unmerged and its follow-up comment explicitly requested an outbound payload to identify the actual offending field.
