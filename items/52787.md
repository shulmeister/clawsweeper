## Summary

Keep open. I did not find strong evidence that current `main` has fixed this Control UI image-upload bug. The file-selection and attachment-serialization paths in `ui/src/ui/views/chat.ts` and `ui/src/ui/controllers/chat.ts` are still present on `main`, and the file-upload handler in `ui/src/ui/views/chat.ts` is unchanged from the latest shipped release `v2026.4.22`. I also did not find a targeted regression test that exercises the reported path end-to-end (attach image in webchat, preview persists, `chat.send` carries attachments).
