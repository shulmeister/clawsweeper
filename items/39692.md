## Summary

Keep PR #39692 open. Current main has the Control UI i18n framework and some independently localized channel, gateway URL, and instances strings, but the PR’s central work is still not implemented across Agents, agent panels, Logs, Debug, config form/channel config, and exec approval dialogs. The prior Greptile review remains useful rebase guidance because current main still preserves the monospace security-audit command, the "Cron Jobs" label, and the "never" refresh fallback semantics.

## Best Possible Solution

Keep the PR open and ask for a focused rebase against current main. The best implementation is to move the remaining user-facing Control UI literals into the English source locale map using the current key structure, regenerate zh-CN and other generated locale bundles through the Control UI i18n pipeline, preserve the Greptile-reviewed UX details, and add targeted regression coverage where practical.
