## Summary

Keep open. Current main and the latest release still contain the bootstrap path behavior that can produce the reported doubled `__openclaw__` request, and the two linked fix PRs were closed without merge.

## Best Possible Solution

Keep the issue open. The best path is a focused maintainer fix that either supports `/__openclaw__/` or configured base paths without producing a doubled bootstrap URL, or explicitly redirects/rejects that route if it is unsupported. The change should cover `src/gateway/control-ui-contract.ts`, `ui/src/ui/controllers/control-ui-bootstrap.ts`, server route compatibility in `src/gateway/control-ui.ts`, and regression tests for the reported URL/basePath behavior.
