## Summary

Keep open. Current main has adjacent strict-agentic and single-action-narrative handling, but it does not yet catch the specific dummy shell/no-op tool-call evasion described here. The implemented guard retries one safe read/search-style tool call plus planning prose; shell tools such as `bash`/`exec` are treated as mutating/side-effectful and remain excluded, and a terminal `toolUse` stop reason still bypasses the planning-only retry path.

## Best Possible Solution

Keep this issue open and implement a narrower structural progress classifier for shell/exec-style tools. The fix should distinguish no-op or no-output dummy commands such as `sleep 0.1` from real shell progress, thread enough tool args/result metadata into the planning-only decision, preserve side-effect safety for mutating commands, and add a regression test where a single dummy shell call plus plan-only prose triggers strict-agentic retry and then the blocked exit.
