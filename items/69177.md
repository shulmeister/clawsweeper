## Summary

Current main already fixes the observable bug from PR #69177. The tool still does not pass the PR's exact `contextTokens` parameter, but it now passes the resolved `sessionEntry` into the shared status renderer, and that renderer reads `sessionEntry.contextTokens` directly before falling back to agent/model defaults. The linked bug #69151 is closed, and the behavior is present in release `v2026.4.24`.

## Best Possible Solution

Close PR #69177 as redundant. Keep the current main implementation that reads persisted `sessionEntry.contextTokens` in the shared status renderer; if maintainers want more guard coverage, add a behavior-level `session_status` output test rather than asserting the internal `buildStatusText` parameter shape.
