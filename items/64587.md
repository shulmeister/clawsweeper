## Summary

PR #64587 adds another bundled web_search provider for Z.AI, but current main already has the plugin, manifest, runtime, and public SDK surfaces needed to ship this as an external ClawHub/npm provider. VISION.md sets a high bar for adding optional integrations to core, and the provided PR review thread still has unresolved implementation concerns around regional Z.AI endpoint routing and timeout enforcement. This should close as ClawHub/community plugin work rather than remain open as a core PR.

## Best Possible Solution

Close this core PR and invite the contributor to publish Z.AI web search as an external ClawHub/npm plugin using api.registerWebSearchProvider, openclaw/plugin-sdk/provider-web-search, contracts.webSearchProviders, and the exported Z.AI endpoint helper. If OpenClaw maintainers later want an official bundled Z.AI search provider, it should come as a maintainer-owned product decision with regional endpoint routing and timeout enforcement fixed before merge.
