## Summary

PR #64587 adds Z.AI Web Search as another bundled `web_search` provider, but current `main` already has the plugin/API surfaces needed to ship this as an external provider plugin through ClawHub or npm. The current project direction explicitly keeps optional capabilities out of core unless there is a strong product/security/maintainer reason, and the provided PR review thread still has unresolved implementation concerns around regional endpoint routing and timeout enforcement. This should close as ClawHub rather than stay open as a core PR.

## Best Possible Solution

Close this core PR and ask the contributor to publish Z.AI web search as an external provider plugin through ClawHub or npm, using `api.registerWebSearchProvider`, `openclaw/plugin-sdk/provider-web-search`, and the exported Z.AI endpoint helper. If an official bundled Z.AI search provider is desired later, it should come as a maintainer-owned product decision with the regional endpoint and timeout issues fixed.
