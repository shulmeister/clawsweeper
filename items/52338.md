## Summary

Current main already covers the reported loopback CLI/operator scope-loss path: CLI gateway calls request `operator.read` by default, local authenticated calls/probes stay device-bound so shared-auth hardening does not strip read access, repaired/admin-superset pairings are accepted for `operator.read`, and loopback `openclaw devices` has a local fallback for pairing deadlocks. The same implementation is present in release `v2026.4.22`.
