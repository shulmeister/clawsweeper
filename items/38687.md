## Summary

Keep #38687 open. Current main still does not implement the PR's target-aware media backend proxy bypass: the media runner still creates an env-proxy fetch before resolving the backend baseUrl, and current tests still cover only the older “proxy env means provider gets fetchFn” behavior. The PR discussion also shows substantive review feedback was addressed, so the branch remains useful rather than obsolete.

## Best Possible Solution

Keep #38687 open and adapt it to current main: resolve the media backend baseUrl before choosing `fetchFn`, reuse or extend `shouldUseEnvHttpProxyForUrl`/`matchesNoProxy` deliberately, and add focused runner/proxy tests proving NO_PROXY-covered audio backends receive no proxy fetch while non-excluded backends still do.
