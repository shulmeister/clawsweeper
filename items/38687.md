## Summary

Keep #38687 open. Current main still does not implement the PR's target-aware media backend proxy bypass: the runner creates an env-proxy fetch before resolving the backend baseUrl, the proxy-fetch helper has no target/baseUrl argument, and current runner tests still assert the old HTTPS_PROXY behavior rather than the requested NO_PROXY-covered audio-backend bypass. The PR discussion also shows substantive review feedback was addressed, so the branch remains useful rather than obsolete.

## Best Possible Solution

Keep #38687 open and adapt it to current main: resolve the media backend baseUrl before choosing fetchFn, reuse or intentionally extend shouldUseEnvHttpProxyForUrl/matchesNoProxy rather than duplicating NO_PROXY parsing, and add focused runner/proxy tests proving NO_PROXY-covered audio backends receive no proxy fetch while non-excluded backends still do.
