## Summary

Keep PR #69696 open. Current main still has case-sensitive OpenRouter base URL normalization in the OpenRouter plugin, Arcee plugin, and embedded runner test-support path, and current regression tests cover lowercase stale `/v1` normalization but not uppercase host variants or the no-op/custom-proxy cases this PR adds. The merged #68574 fixed the lowercase stale-route problem but does not supersede this tighter edge-case fix.

## Best Possible Solution

Keep the PR open. The best maintainer path is to validate or refresh #69696 against current main, then either merge that focused change or apply the same minimal fix: lowercase only the normalized comparison value in the OpenRouter, Arcee, and embedded-runner helpers, while preserving the canonical returned URL and the custom-proxy no-op behavior in tests.
