## Summary

Keep open. Current main still passes a hook-mutated bootstrap file's runtime `file.name` through as the context-list display label, while `agent:bootstrap` exposes `context.bootstrapFiles` as a mutable hook surface and only sanitizes `path`. A hook-injected virtual file with `{ path, content, virtual: true }` and no `name` can still render as `undefined` in `/context list`; the session-memory side may have partial sanitizer work now, but the display bug alone is still a valid current issue.

## Best Possible Solution

Keep the issue open and land a focused fix in the bootstrap context reporting path: derive a stable display label from `file.name`, then normalized `file.path` basename/path, then a safe fallback; add regression coverage for path-only virtual hook files and confirm session transcript indexing excludes generated hook/bootstrap content from recall where applicable.
