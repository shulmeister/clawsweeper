## Summary

Keep open. Current main still builds injected bootstrap file stats from `file.name` directly and `/context list` renders that same field, while hook-mutated bootstrap files are only sanitized for a usable `path`. A path-only virtual bootstrap file can still show as `undefined`, so this issue is not cleanup-ready even if the session-memory/accumulation half may need separate verification.

## Best Possible Solution

Keep this issue open. The narrow first fix should resolve a stable display label in `buildBootstrapInjectionStats` using `file.name`, then `basename(file.path)`, then a safe fallback, with regression coverage for path-only virtual hook files. Separately verify whether session-memory indexing still ingests hook/bootstrap content; if it does, exclude injected/system bootstrap content from session-memory sources or split that work into a focused follow-up.
