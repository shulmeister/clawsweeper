## Summary

Keep #39013 open. Current main still has the path-only virtual bootstrap display bug: hook-mutated bootstrap files are sanitized for `path`, but `buildBootstrapInjectionStats` still emits `name: file.name`, and `/context list` renders that field directly. A hook that pushes `{ path, content, virtual: true }` can still show `undefined`, so the issue is not cleanup-ready even if the session-memory/accumulation half needs a separate fresh repro.

## Best Possible Solution

Keep the issue open. First fix the deterministic reporting bug by deriving a safe display label in `buildBootstrapInjectionStats` from `file.name`, then normalized `file.path`/basename, then an explicit placeholder, and add regression coverage for path-only virtual bootstrap files injected by `agent:bootstrap` hooks. Then separately recheck the session-memory/accumulation behavior on current main and either exclude injected system/bootstrap content from session-memory indexing or split that remaining behavior into a focused follow-up.
