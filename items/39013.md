## Summary

Keep #39013 open. Current main still has the path-only virtual bootstrap display bug: hook-mutated bootstrap files are sanitized for `path`, but `buildBootstrapInjectionStats` still copies `file.name` into the report and `/context list` renders that name directly. A hook-injected object shaped like `{ path, content, virtual: true }` can still display as `undefined`. The session-memory/accumulation part may need a fresh focused repro, but the issue is not cleanup-ready while the display half remains unfixed.

## Best Possible Solution

Keep the issue open. First fix the stable reporting bug by deriving a safe display label in `buildBootstrapInjectionStats`, for example `file.name`, then basename or normalized `file.path`, then an explicit placeholder, and add regression coverage for path-only virtual files injected by `agent:bootstrap` hooks. After that, separately reproduce the session-memory/accumulation path on current main and either exclude injected system/bootstrap content from session-memory indexing or split that remaining behavior into a focused follow-up.
