## Summary

Keep #39013 open. Current main still has the deterministic `/context list` display bug for hook-injected path-only virtual bootstrap files: hooks can mutate `bootstrapFiles`, sanitization preserves `path` but does not derive `name`, `buildBootstrapInjectionStats` returns `name: file.name`, and `/context list` renders that value directly. The session-memory amplification claim likely needs a fresh focused reproduction because current session transcript indexing now filters to user/assistant message records and skips custom/non-message records, but the unresolved display bug alone is enough to keep the issue open.

## Best Possible Solution

Keep the issue open. First fix `buildBootstrapInjectionStats` to derive a stable display label from `file.name`, then `path.basename(file.path)`, then `file.path`, then an explicit fallback, and add a regression test for a hook-injected virtual bootstrap file with `path` and no `name`. After that, retest the session-memory/accumulation behavior and either exclude any remaining injected bootstrap/system content from recall or split hook-specific dedupe behavior into a narrower follow-up.
