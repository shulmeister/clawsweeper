## Summary

Current main no longer has the reported iMessage self-chat echo path: inbound `is_from_me` messages are dropped before echo-cache lookup, and the current sent-message cache no longer short-circuits text fallback after an ID miss.

## Evidence

- **iMessage self messages dropped before echo-cache matching:** `resolveIMessageInboundDecision()` records the self-chat lookup and immediately returns `{ kind: "drop", reason: "from me" }` when `params.message.is_from_me` is true, so self-chat echoes no longer dispatch as inbound messages even if IDs differ.
  - file: [src/imessage/monitor/inbound-processing.ts:1548](https://github.com/openclaw/openclaw/blob/7bbd47349ec37f01a8f2b1dc219bf950615f361b/src/imessage/monitor/inbound-processing.ts#L1548)
  - command: `web open https://github.com/openclaw/openclaw/blob/main/src/imessage/monitor/inbound-processing.ts`
  - sha: [7bbd47349ec3](https://github.com/openclaw/openclaw/commit/7bbd47349ec37f01a8f2b1dc219bf950615f361b)
- **Message-ID miss falls through to text matching:** `DefaultSentMessageCache.has()` checks the message ID cache first, but after an ID miss it continues to normalize and check the text cache; there is no `skipIdShortCircuit` parameter or early false return in the current implementation.
  - file: [src/imessage/monitor/echo-cache.ts:492](https://github.com/openclaw/openclaw/blob/7bbd47349ec37f01a8f2b1dc219bf950615f361b/src/imessage/monitor/echo-cache.ts#L492)
  - command: `web open https://github.com/openclaw/openclaw/blob/main/src/imessage/monitor/echo-cache.ts`
  - sha: [7bbd47349ec3](https://github.com/openclaw/openclaw/commit/7bbd47349ec37f01a8f2b1dc219bf950615f361b)
- **Related discussion acknowledged:** Related issue #59363 and PR #59386 documented the same iMessage echo-loop/control-character family. PR #59386 noted that the echo-cache normalization fix was sound, while current main also contains the stronger `is_from_me` drop path.
  - command: `web open https://github.com/openclaw/openclaw/issues/59363 and https://github.com/openclaw/openclaw/pull/59386`
