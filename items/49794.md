## Summary

Keep open. Current `main` does not implement the PR’s hard `memory_search` wall-clock timeout. The memory tool still awaits manager init and `manager.search()` directly with no outer `Promise.race`/tool-boundary timeout, and the tests still cover only quota/generic failures plus QMD debug, not a stalled search/init path.

## Evidence
