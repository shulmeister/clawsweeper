## Summary

Keep open. Current main still appears to allow the reported double-delivery path for `sessions_spawn(runtime: "acp", thread: true, mode: "session")`: ACP session spawns inline-deliver to the bound thread, while `sessions_spawn` still registers the accepted ACP run for completion announcement by default when `streamTo` is not `parent`. The native subagent path has an explicit suppression guard for this inline-thread case; the ACP path does not show the same guard.

## Evidence

- **ACP session spawns inline-deliver to the bound thread:** `resolveAcpSpawnBootstrapDeliveryPlan` sets `useInlineDelivery` when there is a delivery target, `effectiveStreamToParent` is false, and `spawnMode === "session"`; the subsequent `agent` gateway call passes `deliver: deliveryPlan.useInlineDelivery`.
