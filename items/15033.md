## Summary

Current `main` already supports content-based automatic model/provider selection through the new `before_model_resolve` plugin hook, which runs before model resolution, receives the incoming prompt plus attachment metadata, and can override the provider/model deterministically.

## Evidence

- **Public hook contract for content-based model routing:** `before_model_resolve` is a first-class plugin hook. Its event includes the incoming `prompt` and optional attachment metadata, and its result can return `modelOverride` and `providerOverride`. That is the core API needed for automatic message-complexity or task-type routing.

## Close Comment

Closing from a Codex review: current `main` already implements the requested capability through the new `before_model_resolve` plugin hook. The public hook contract accepts the incoming prompt plus attachment metadata and can return `providerOverride` / `modelOverride` (`src/plugins/hook-before-agent-start.types.ts:1`). The agent runtime now runs that hook before model resolution and applies the override to the active run (`src/agents/pi-embedded-runner/run/setup.ts:44`). This is also documented in `docs/concepts/agent-loop.md:93` and covered by regression tests in `src/plugins/hooks.model-override-wiring.test.ts:1`. The implementation was introduced by commit `382952a9e1e033d9d9ba2383c7b1bb24299ee001`, which is on current `main`. I did not confirm a released tag for it yet, so I’m closing this as implemented on `main`, not as released in a specific version.
