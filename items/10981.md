## Summary

Keep open. Current `main` has gateway RPC methods for `sessions.reset` and `sessions.compact`, but it still does not expose an agent-facing tool or documented session-tool path for autonomous agents to invoke reset/compact programmatically.

## Evidence

- **Agent tool registration omits any reset/compact session tool:** `createOpenClawTools()` registers `sessions_list`, `sessions_history`, `sessions_send`, `sessions_spawn`, `sessions_yield`, `subagents`, and `session_status`. There is no `sessions_manage` or equivalent reset/compact tool in the shipped agent tool set.
  - file: [src/agents/openclaw-tools.ts:273](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/agents/openclaw-tools.ts#L273)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **User-facing session-tool docs list no programmatic compact/reset tool:** The current session-tool docs enumerate only `sessions_list`, `sessions_history`, `sessions_send`, `sessions_spawn`, `sessions_yield`, `subagents`, and `session_status`; reset/compact are absent from the documented agent tool surface.
  - file: [docs/concepts/session-tool.md:13](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/docs/concepts/session-tool.md#L13)

- **Tool catalog for the sessions section still lacks reset/compact:** The central tool catalog lists only `sessions_list`, `sessions_history`, `sessions_send`, `sessions_spawn`, `sessions_yield`, and `subagents` in the sessions section. No session-maintenance tool is present.
  - file: [src/agents/tool-catalog.ts:145](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/agents/tool-catalog.ts#L145)
