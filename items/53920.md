## Summary

Keep PR #53920 open. Current main still has the reported bug: `scripts/setup-auth-system.sh` edits the tracked auth-monitor systemd template in place before copying it, and the tracked template still contains the machine-specific `/home/admin/openclaw/scripts/auth-monitor.sh` path. The latest PR head is a focused fix for that exact scripts surface and appears to address the earlier review concerns about missing placeholders, python3 availability, quoted `ExecStart`, and previously mutated notification environment lines. I did not find a merged/superseding main change or duplicate that resolves this behavior.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a narrow fix in `scripts/setup-auth-system.sh` that copies the service template before rendering, updates only the installed user unit, writes `ExecStart` for the current checkout, preserves idempotency for templates already mutated by the old script, and adds either a focused renderer regression test or clear manual Linux/systemd verification. If this PR stalls, implement the same behavior directly on main rather than closing the item.
