## Summary

Current `main` does not already carry the PR’s WhatsApp remote/headless caveat, and the underlying QR-only CLI login behavior still exists. The PR is not strong evidence for closure; if anything, it still addresses a real docs gap, though it likely needs revision to acknowledge the existing Control UI / `web.login.start` QR-image path and the docs i18n glossary guard noted in review.
