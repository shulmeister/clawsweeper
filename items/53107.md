## Summary

Current main still does not implement cron job tags/categories. The prior ClawSweeper comment remains accurate against 9626ef274ae2346b3d519f99c31856be98aab570: tags are absent from the persisted cron model, Gateway contract/list filtering, CLI, Control UI form/list filters, agent cron tool schema, and cron docs.

## Best Possible Solution

Keep this issue open. The best implementation is an additive core cron contract change: add normalized optional tags?: string[] to the cron job model, persistence/normalization, Gateway add/update/list schemas and tag filtering, agent cron tool schema/recovery, CLI add/edit/list/show and tag filter support, Control UI create/edit/display/filter controls, docs, and regression tests. Tag-based bulk enable/disable should either ship on top of the same list/update contract or be split into a narrower follow-up after tags are persisted and filterable. This belongs in core, not ClawHub, because the requested behavior is organization and filtering for OpenClaw's built-in cron scheduler state across core APIs and first-party UI/CLI surfaces.
