## Summary

Current `main` does not implement cron job tags/categories. The cron job types, RPC schemas, server-side list API, CLI output, and Control UI filters all lack a `tags` field or tag-based filtering, so the feature request remains open work rather than maintainer cleanup.

## Evidence

- **Cron job model has no tags field:** `CronJobBase` only defines id/agentId/sessionKey/name/description/enabled/deleteAfterRun/createdAtMs/updatedAtMs/schedule/sessionTarget/wakeMode/payload/delivery/failureAlert, and `CronJob` only adds `state`. There is no `tags` property in the persisted cron model.
  - file: [src/cron/types-shared.ts:1](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/cron/types-shared.ts#L1)
  - command: `nl -ba src/cron/types-shared.ts | sed -n '1,80p'; nl -ba src/cron/types.ts | sed -n '130,220p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **RPC schemas do not accept tags or tag filters:** `CronJobSchema`, `CronAddParamsSchema`, and `CronJobPatchSchema` enumerate allowed cron fields with `additionalProperties: false`, and `CronListParamsSchema` only supports `includeDisabled`, `limit`, `offset`, `query`, `enabled`, `sortBy`, and `sortDir`. A `tags` field or `tag` filter is not part of the API contract.
  - file: [src/gateway/protocol/schema/cron.ts:250](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/gateway/protocol/schema/cron.ts#L250)

  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Server-side cron listing cannot filter by tag:** `listPage()` filters only by enabled/disabled and a free-text query over `job.name`, `job.description`, and `job.agentId`, then sorts by next run, update time, or name. There is no tag-aware filtering or grouping in the service layer.
  - file: [src/cron/service/ops.ts:208](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/cron/service/ops.ts#L208)

No tag/category field is shown for jobs or used for filtering.
