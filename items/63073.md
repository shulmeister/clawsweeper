## Summary

Current main still does not implement the PR's remaining MiniMax usage-host behavior. The original GET-method problem is fixed on main, but usage polling still hardcodes the CN host and ignores configured global/CN baseUrl context; the overlapping #65059/#65054 work is still open and unmerged, so it is a consolidation path rather than a reason to close this PR.

## Best Possible Solution

Keep this PR open for maintainer review or consolidation with #65054/#65059. The best final fix is a MiniMax-owned, region-aware usage host resolver that derives from the same configured provider/auth source as the usage credential, preserves `GET /v1/api/openplatform/coding_plan/remains`, supports global and CN baseUrls, handles invalid/custom baseUrl safely, and adds regression tests for URL, method, global, CN, and mixed `minimax`/`minimax-portal` configurations.
