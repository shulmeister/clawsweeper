## Summary

Keep open. Current main still has the behavior this PR is trying to change: Ollama synthetic auth is returned for any meaningful explicit Ollama config without checking whether the baseUrl is local/LAN. The PR is useful but not implemented on main, and it needs to be rebased against current main because the lightweight provider-discovery entry now duplicates the same synthetic-auth hook.

## Best Possible Solution

Keep the PR open, but require a rebase/update before merge: put the locality classifier in shared Ollama code, gate both extensions/ollama/index.ts and extensions/ollama/provider-discovery.ts, update any discovery/default-api-key path that can still stamp ollama-local onto remote configs, and add regression tests for Cloud https://ollama.com, public IPs, LAN/mDNS/bare hostnames, loopback aliases, and IPv6 local ranges.
