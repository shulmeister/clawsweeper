## Summary

Keep open. Current main still does not implement the requested Control UI topbar display for current model plus token usage. The existing code has adjacent model/usage surfaces, but they remain in the chat header, chat notices/message metadata, debug tab, usage/status commands, or docs-listed usage surfaces rather than an always-visible topbar status component.

## Best Possible Solution

Keep #13670 open until a maintainer either accepts and implements a focused Control UI topbar status component, or explicitly declines that placement. A good implementation would live in the topbar render path, derive active model from current session/default model state, derive token/context usage from current session usage data where possible, avoid debug-tab-only polling for normal UI state, and add focused UI tests for the new topbar surface.
