## Summary

Keep #13670 open. Current main still does not implement the requested always-visible Control UI topbar display for active model plus token/context usage. The current topbar status area only renders the chat mobile controls and theme toggle; model/usage information exists only in adjacent scoped surfaces such as chat controls, slash-command output, message metadata, the Usage page, Debug tab snapshots, and high-context composer notices.

## Best Possible Solution

Keep #13670 open until maintainers either implement a focused Control UI topbar status component or explicitly decline the placement. A good implementation should live in the topbar render path, derive the active model from current session/default model state, derive token/context usage from fresh session usage or session rows rather than promoting Debug-tab polling into global UI state, and add focused UI tests for visibility, stale/fresh handling, and responsive layout.
