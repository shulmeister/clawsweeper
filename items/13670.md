## Summary

Keep #13670 open. Current main still does not implement the requested always-visible Control UI topbar display for the active model plus token/context usage. The model and usage data exist in Debug, Overview, Usage, and Chat surfaces, but the topbar render path only exposes navigation/search plus chat mobile and theme controls. The existing maintainer/Codex review comment also explicitly kept this open, and the newer current-main source still matches that conclusion.

## Best Possible Solution

Keep #13670 open until maintainers either implement a focused Control UI topbar status component or explicitly decline that placement. A good implementation should live in the topbar/header render path, derive the active model from current session/default model state, derive fresh token/context usage from session usage data, handle stale or unavailable usage states, and add responsive UI coverage.
