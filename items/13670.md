## Summary

Keep open. Current main has adjacent model and token/usage surfaces, but it still does not implement the requested Control UI topbar display for current model plus token usage. The `.topbar-status` area only renders mobile chat controls and the color-mode toggle, while debug/status and usage data remain scoped to the debug tab, usage page, chat commands, or chat-local notices.

## Best Possible Solution

Keep #13670 open until a maintainer either accepts the enhancement and implements a real topbar status component, or explicitly declines the topbar placement. A likely implementation would live in the Control UI topbar render path, derive the active model from existing session/model state, derive token/context usage from current session usage data rather than debug-only polling where possible, and add focused UI tests plus docs if the surface becomes user-facing.
