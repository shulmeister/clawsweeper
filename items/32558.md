## Summary

Keep open. Current main still uploads Teams OneDrive and SharePoint files through single Graph `/content` PUT requests, with no upload-session fallback or upload-session tests. The PR is a focused, unmerged slice from earlier large Teams work, and the related prior PRs are closed unmerged rather than canonical replacements.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to re-scope the fallback threshold against current Microsoft Graph limits, fix the stall-count off-by-one, and land focused upload-session coverage if Teams should support resumable SharePoint/OneDrive uploads beyond the current direct `/content` path.
