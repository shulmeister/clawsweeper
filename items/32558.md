## Summary

Keep PR #32558 open. Current main still lacks any Microsoft Graph upload-session implementation for the active Teams OneDrive/SharePoint upload helpers, and the PR remains a focused Teams reliability change rather than conservative cleanup. Related PRs #26668 and #26712 were closed unmerged, so they do not supersede this focused PR.

## Best Possible Solution

Keep this PR open for Microsoft Teams maintainer review. The best path is a focused `extensions/msteams/src/graph-upload.ts` change that adds a current Graph upload-session path for Teams OneDrive and SharePoint uploads, uses valid 320 KiB-multiple chunks, handles `202 nextExpectedRanges` plus final `200/201`, retries/resumes transient upload-session failures, updates the threshold/comment to match current Graph guidance, and lands with focused regression tests for OneDrive and SharePoint.

## Risks / Open Questions

- The proposed PR uses an old 4 MB cutoff. Current Microsoft Graph docs say direct driveItem content upload supports files up to 250 MB, while upload sessions remain the documented resumable/large-file path; maintainers should choose the threshold deliberately before merging. 
- Greptile's PR review found an off-by-one in `UPLOAD_SESSION_MAX_STALLS`; that should be fixed in the PR before approval.
- This read-only review did not run live OneDrive/SharePoint tenant validation.
