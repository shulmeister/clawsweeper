## Summary

Current main makes #67037 obsolete. The Control UI now skips active-run `session.message` history reloads, records a deferred reload instead of dropping the event, and preserves optimistic user/assistant messages across stale or empty `chat.history` refreshes. That addresses the flicker/collapse/duplicate-bubble problem described in the PR without taking the PR's broader suppression implementation.

## Best Possible Solution

Close #67037 as implemented on current main. Keep the current main approach because it preserves visible local chat state and avoids dropping deferred transcript updates; if #66875 still reproduces, ask for a new narrow repro against current main and fix that remaining path directly.
