## Summary

Current main already implements the useful part of PR #67037: same-session `session.message` reloads are skipped while a chat run is active, the skipped update is deferred and replayed when the active run reaches a terminal event, and history reloads preserve optimistic local messages so stale/empty snapshots no longer flicker or erase visible chat state. The PR review discussion also raised dropped-reload risks in the PR implementation; current main addresses that risk with a deferred reload rather than a plain suppression.

## Best Possible Solution

Close PR #67037 as implemented on current main. Keep the shipped current-main approach: defer same-session reloads only while an actual chat run is active, replay the deferred reload after terminal events, and preserve optimistic transcript entries during stale or empty history refreshes. Use #66875 or a new narrow issue for any remaining flicker path that still reproduces on current main.
