## Summary

Keep open: current main fixes the exec-approvals atomic-write half, but the transcript memory-pressure half remains in several core paths. The original report is still actionable because idempotency checks, session transcript helpers, chat history snapshots, and the manual max-lines compaction fallback still load whole transcript files before scanning or slicing them.

## Best Possible Solution

Keep this issue open. Treat the exec-approvals persistence concern as already fixed, then land a focused core follow-up that introduces streaming or bounded transcript scan helpers for idempotency, latest assistant lookup, delivery-mirror equivalence, chat history snapshots, and max-lines compaction, with regression coverage using large transcript fixtures.
