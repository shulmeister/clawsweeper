## Summary

Keep open. Current main still has the central bootstrap issue: oversized bootstrap files are truncated by fixed head/tail slices, so relevant middle content can still be omitted. Context-engine, compaction-provider, hook, and community plugin surfaces cover adjacent/plugin-side work, but they do not replace the bundled bootstrap injection path or fully ship the bundled core architecture request. The existing issue discussion already includes a maintainer Codex review keeping this open for the same reason.

## Best Possible Solution

Keep #18548 open until the core bootstrap truncation behavior is fixed or explicitly descoped. A likely maintainer path is to implement relevance-aware or structure-aware bootstrap selection in `src/agents/pi-embedded-helpers/bootstrap.ts`, align incremental compaction work with the existing context-engine and compaction-provider contracts, and treat observation extraction as a plugin/hook/context-engine capability unless a concrete missing SDK API is identified.
