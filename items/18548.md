## Summary

Keep open. Current main has adjacent compaction and plugin/context-engine improvements, but the central bundled request is not fully implemented: bootstrap file injection still uses fixed head/tail positional truncation and drops the middle regardless of relevance. The issue also asks for core bootstrap and compaction architecture changes, so a ClawHub-only close is not supported.

## Best Possible Solution

Keep #18548 open until the missing core behavior is implemented or explicitly descoped. A good maintainer path would split the work: add relevance-aware or structured bootstrap file selection in `src/agents/pi-embedded-helpers/bootstrap.ts`; align incremental compaction with the existing context-engine and compaction-provider APIs; and treat observation extraction as a plugin/context-engine capability unless it exposes a concrete missing SDK hook.
