## Summary

Keep #18548 open. Current main still trims oversized workspace bootstrap files by fixed head/tail character slices, and the active run/compaction prompt paths still feed bootstrap files through that trimmer. Context-engine, compaction-provider, hook, and Lossless Claw style plugin surfaces are relevant to the compaction and observation portions, but they do not resolve the core bootstrap relevance request. A maintainer/Codex review already reached the same conclusion on April 26, 2026.

## Best Possible Solution

Keep #18548 open, preferably with bootstrap relevance as the core tracker. The likely core change belongs in src/agents/pi-embedded-helpers/bootstrap.ts as a relevance-aware or structure-aware selector for oversized bootstrap files, with tests replacing the current fixed head/tail expectations and docs/config updated for the new selection behavior. Track incremental compaction against #58398/context-engine/Lossless Claw work, and treat observation extraction as plugin/hook/context-engine work unless a concrete missing SDK API is identified.
