## Summary

Keep open. Current main still has the central bootstrap problem: oversized workspace bootstrap files are truncated by fixed head/tail slices, so middle content is omitted without relevance or section selection. The compaction and context-engine/plugin surfaces have improved, but they do not fully implement the bundled request, and the latest issue discussion already reached the same keep-open conclusion.

## Best Possible Solution

Keep #18548 open until the core bootstrap behavior is implemented or explicitly descoped. The likely maintainer path is to split the item: add relevance-aware or structure-aware bootstrap selection in src/agents/pi-embedded-helpers/bootstrap.ts, align incremental compaction work with the existing context-engine and compaction-provider contracts, and treat observation extraction as a plugin/hook/context-engine capability unless a concrete missing SDK API is identified.
