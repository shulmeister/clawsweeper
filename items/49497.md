## Summary

Keep open. Current `main` has generic OpenAI-compatible self-hosted/provider plumbing and even a small SiliconFlow-specific request-compat shim, but it does not implement the specific request in #49497: there is no SiliconFlow guide on `main`, no bundled SiliconFlow provider/plugin, and no `remoteId`/`idOverride`-style model-mapping field in the current model schema. This is a coherent enhancement/docs request, not a cleanup candidate for conservative closure.

## Evidence

- **Model schema still has no logical-to-remote model mapping field:** `ModelDefinitionSchema` is a strict object with fields such as `id`, `name`, `api`, `baseUrl`, `reasoning`, `input`, `cost`, `contextWindow`, `maxTokens`, `headers`, `compat`, and `metadataSource`; there is no `remoteId` or `idOverride` field.
