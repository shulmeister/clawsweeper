## Summary

Keep #9465 open. Current main still lacks the requested per-job cron `hooks.pre`/`post`/`onError` contract, gateway schema/API surface, CLI flags, and timer pre-hook execution path. The shipped cron-aware `before_agent_reply` plugin seam can short-circuit some cron agent runs, but it is not job-level cron hook configuration and does not satisfy the shell/webhook lifecycle design. Related PR #45129 remains open/unmerged in the provided context, and PR #9466 was closed unmerged as a docs-only proposal.

## Best Possible Solution

Keep #9465 open until maintainers either land a per-job cron hook contract through cron types, gateway schemas/API, timer/manual-run execution paths, CLI/UI/docs, and tests, or explicitly supersede this broad request with a narrower canonical tracker such as the shell pre-hook gating work in PR #45129.
