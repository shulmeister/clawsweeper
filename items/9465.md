## Summary

Keep #9465 open. Current main still does not implement the requested persisted per-job cron hook lifecycle (`hooks.pre`, `hooks.post`, `hooks.onError`) across cron types, Gateway protocol, scheduler/manual/startup execution paths, CLI, or Control UI. The existing `before_agent_reply` plugin seam can short-circuit some cron-driven agent turns with `ctx.jobId`, but it is a global plugin workaround, not a job-defined scheduler contract. The issue discussion also has concrete recent demand for deterministic pre-LLM gating, and the related narrower PR #45129 is open/unmerged while PR #9466 was closed unmerged as proposal/docs-only.

## Best Possible Solution

Keep #9465 open until maintainers either land an additive first-class per-job cron hook lifecycle across persisted cron types, Gateway schemas/RPC, scheduler due/startup/manual paths, CLI, Control UI, docs, and tests, or explicitly supersede it with a canonical narrower tracker. Document the existing plugin `before_agent_reply` path as a workaround, not as the replacement for the requested cron scheduler contract.
