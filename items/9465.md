## Summary

Keep #9465 open. Current main still lacks a per-job cron hooks contract and execution path: cron job types, Gateway schemas, CLI add/edit surfaces, automatic timer execution, startup catchup, and manual cron.run all proceed without job-level pre/post/onError hook configuration. The existing cron-aware before_agent_reply plugin hook can short-circuit some agent runs, but it is global plugin behavior rather than the requested CronJob hooks lifecycle. Related PR #45129 remains the narrower open implementation path in the provided context; PR #9466 was closed unmerged as docs-only.

## Best Possible Solution

Keep #9465 open until maintainers either land an additive per-job cron hook contract across cron types, Gateway schemas/RPC, timer/startup/manual execution paths, CLI/UI/docs, and tests, or explicitly supersede it with a narrower canonical tracker such as the shell pre-hook gating work in PR #45129.
