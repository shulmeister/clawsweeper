## Summary

Keep open. Current `main` added best-effort system-level discovery to `gateway/daemon status --deep`, but the lifecycle commands still manage only the per-user macOS LaunchAgent. The requested LaunchDaemon-aware stop/uninstall flow and `--system` support are not implemented.

## Evidence

- **Darwin service resolution is still LaunchAgent-only:** The macOS service registry entry is labeled `LaunchAgent` and wires stop/uninstall/load/read operations only to the LaunchAgent helpers.
  - file: [src/daemon/service.ts:172](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/daemon/service.ts#L172)
  - command: `nl -ba src/daemon/service.ts | sed -n '170,185p'`
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **launchd path/domain are user-scope only:** `resolveLaunchAgentPlistPath` always builds `~/Library/LaunchAgents/<label>.plist`, and `isLaunchAgentLoaded` checks only `gui/$UID/<label>`.
  - file: [src/daemon/launchd.ts:56](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/daemon/launchd.ts#L56)
