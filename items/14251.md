## Summary

Keep #14251 open. Current main still manages macOS gateway lifecycle through the per-user LaunchAgent path/domain. `gateway status --deep` adds a diagnostic scan for extra services, but stop/uninstall do not expose system scope handling or manage `/Library/LaunchDaemons/ai.openclaw.gateway.plist`.

## Best Possible Solution

Keep the issue open until macOS gateway lifecycle management is made scope-aware. A good fix would add a user/system launchd scope model, detect `/Library/LaunchDaemons/ai.openclaw.gateway.plist`, provide clear sudo guidance or an explicit `--system` path for stop/uninstall, ensure status reports the canonical system daemon accurately, and update tests/docs for both user LaunchAgents and system LaunchDaemons.
