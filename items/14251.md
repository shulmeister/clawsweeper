## Summary

Keep #14251 open. Current main still treats the macOS managed gateway as a per-user LaunchAgent for stop/uninstall. System LaunchDaemons are only discovered by deep status scans, and the issue discussion already has maintainer/Codex signal that #14315 was narrower and did not supersede this lifecycle request.

## Best Possible Solution

Keep the issue open until macOS gateway lifecycle management becomes launchd-scope-aware: detect `/Library/LaunchDaemons/ai.openclaw.gateway.plist` during stop/uninstall, avoid success responses while a KeepAlive system daemon can remain active, provide exact elevated bootout/remove guidance or an explicit `--system` path, and add focused tests/docs for user LaunchAgents versus system LaunchDaemons.

## Risks / Open Questions

- A system LaunchDaemon with KeepAlive can remain active or restart after the current stop fallback sends SIGTERM to a listener process, so the CLI may still report success without owning the actual launchd service lifecycle.
- The current cleanup hints are generic user LaunchAgent hints, not exact `/Library/LaunchDaemons` `sudo launchctl bootout system ...` guidance for this report.
