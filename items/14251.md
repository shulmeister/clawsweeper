## Summary

Keep #14251 open. Current main still models macOS gateway lifecycle as a per-user LaunchAgent in the gui/<uid> launchd domain; system-level LaunchDaemon awareness is limited to best-effort status/doctor discovery, and the canonical /Library/LaunchDaemons/ai.openclaw.gateway.plist is not managed by stop/uninstall. The supplied April 26 discussion also explicitly kept this open and distinguished closed #14315 as a narrower orphan LaunchAgent cleanup issue, not a replacement for system LaunchDaemon lifecycle support.

## Best Possible Solution

Keep #14251 open until macOS gateway lifecycle management becomes launchd-scope-aware. A good fix should model user vs system scope, detect /Library/LaunchDaemons/ai.openclaw.gateway.plist, avoid reporting stop/uninstall success while a KeepAlive system daemon remains active, provide exact sudo guidance or explicit --system handling for stop/uninstall, and add focused tests/docs for both user LaunchAgents and system LaunchDaemons.
