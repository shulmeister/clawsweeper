## Summary

Keep #14251 open. Current main still manages macOS gateway lifecycle through per-user LaunchAgent paths and the `gui/<uid>` domain. `gateway status --deep` has diagnostic scanning for extra system-level services, but stop/uninstall do not expose a system scope, do not operate on `/Library/LaunchDaemons/ai.openclaw.gateway.plist`, and do not provide the requested sudo guidance path.

## Best Possible Solution

Keep the issue open until macOS gateway lifecycle management is launchd-scope-aware. A good fix would add a user/system scope model, detect the canonical `/Library/LaunchDaemons/ai.openclaw.gateway.plist`, provide clear sudo guidance or explicit `--system` handling for stop/uninstall, make status accurately report canonical system daemons, and add tests/docs for both user LaunchAgents and system LaunchDaemons.
