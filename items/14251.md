## Summary

Keep open. Current main has added `gateway status --deep` as a diagnostic scan, but the requested stop/uninstall lifecycle support for system-level macOS LaunchDaemons is still missing.

## Evidence

- **Lifecycle CLI has no system option:** `status` accepts `--deep`, but `uninstall`, `stop`, and the other lifecycle commands only expose `--json`; there is no `--system` or equivalent lifecycle scope selection.
  - file: [src/cli/daemon-cli/register-service-commands.ts:49](https://github.com/openclaw/openclaw/blob/cbe5515b70abaeba98550b58d739ca0896907c43/src/cli/daemon-cli/register-service-commands.ts#L49)
  - command: `nl -ba src/cli/daemon-cli/register-service-commands.ts | sed -n '38,105p'`
  - sha: [cbe5515b70ab](https://github.com/openclaw/openclaw/commit/cbe5515b70abaeba98550b58d739ca0896907c43)
- **macOS launchd management is user-domain only:** The launchd service path resolves to `~/Library/LaunchAgents`, the domain resolves to `gui/<uid>`, and `isLaunchAgentLoaded`, `uninstallLaunchAgent`, and `stopLaunchAgent` use that user domain/path rather than `system` and `/Library/LaunchDaemons`.
  - file: [src/daemon/launchd.ts:56](https://github.com/openclaw/openclaw/blob/cbe5515b70abaeba98550b58d739ca0896907c43/src/daemon/launchd.ts#L56)
  - command: `nl -ba src/daemon/launchd.ts | sed -n '50,75p;150,170p;258,285p;348,360p;466,506p'`
  - sha: [cbe5515b70ab](https://github.com/openclaw/openclaw/commit/cbe5515b70abaeba98550b58d739ca0896907c43)
- **Deep scan does not cover the reported canonical LaunchDaemon well enough:** `findExtraGatewayServices` scans `/Library/LaunchDaemons` only in deep mode, but `collectServiceFiles` skips names matching `isIgnoredLaunchdLabel`; the default reported file `ai.openclaw.gateway.plist` matches that ignored label, so `gateway status --deep` is not a lifecycle fix and may not even report the canonical system daemon.
  - file: [src/daemon/inspect.ts:136](https://github.com/openclaw/openclaw/blob/cbe5515b70abaeba98550b58d739ca0896907c43/src/daemon/inspect.ts#L136)
  - command: `nl -ba src/daemon/inspect.ts | sed -n '29,45p;135,145p;160,215p;336,365p'`
  - sha: [cbe5515b70ab](https://github.com/openclaw/openclaw/commit/cbe5515b70abaeba98550b58d739ca0896907c43)
- **Docs confirm current scope is diagnostics only:** The daemon docs list lifecycle options for `uninstall|start|stop|restart` as `--json` only, and describe `status --deep` as a best-effort system-level service scan rather than management support.
  - file: [docs/cli/daemon.md:38](https://github.com/openclaw/openclaw/blob/cbe5515b70abaeba98550b58d739ca0896907c43/docs/cli/daemon.md#L38)

It is related context, but it does not supersede this feature request for detecting/managing system-level LaunchDaemons from stop/uninstall.
