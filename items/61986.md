## Summary

Current main still wakes the heartbeat runner immediately for each queued Android `notifications.changed` event, so the requested batching to the scheduled heartbeat interval remains missing. There are mitigations on main, including Android-side filtering/quiet hours/rate limiting and server-side dedupe behavior, but they do not solve the issue’s core complaint. The issue has no protected label and the author is not a maintainer, but it should remain open because the bug is still represented in source and tests. The provided timeline also points to an open fork PR, `https://github.com/KChow-ctrl/openclaw/pull/2`, which looks like related attempted work rather than a merged or canonical superseding OpenClaw item.
