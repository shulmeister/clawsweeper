## Summary

Current main still has the reported sanitizer gap. `sanitizeUserFacingText()` strips final tags textually and only collapses exact adjacent duplicate blocks separated by blank lines, so same-line, single-newline, partial-prefix, and `</final>` boundary duplicate cleanup remain unimplemented.

## Best Possible Solution

Keep #14629 open. Implement a narrow sanitizer-level fix in `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts` for same-line/single-newline repeats and `</final>` boundary duplicate cleanup, with focused tests in `src/agents/pi-embedded-helpers.sanitizeuserfacingtext.test.ts` that preserve normal repeated wording and ordinary single-newline formatting.
