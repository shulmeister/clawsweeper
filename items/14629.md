## Summary

Current main still has the sanitizer gap described in #14629. Final tags are stripped textually, duplicate collapse only handles exact adjacent blank-line paragraph blocks, and the relevant tests do not cover the reported same-line, single-newline, tag-boundary, or partial-prefix duplicate cases. The existing maintainer-side Codex comment also explicitly kept this issue open for the same path.

## Best Possible Solution

Keep #14629 open. The best path is a narrow change in src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts that handles same-line and single-newline repeats, the reported </final> boundary duplicate shape, and conservative partial-prefix duplicates, with focused tests in src/agents/pi-embedded-helpers.sanitizeuserfacingtext.test.ts preserving legitimate repetition and ordinary formatting.
