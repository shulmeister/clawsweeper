## Summary

Current main still matches the reported limitation: the sanitizer only collapses consecutive duplicate paragraph blocks separated by blank lines, strips final tags as a simple textual replacement, and has tests for paragraph dedupe plus preservation of single-newline content. The requested same-line/single-newline duplicate handling, partial duplicate handling, and tag-boundary cleanup remain unimplemented, so this should stay open.

## Best Possible Solution

Keep the issue open until `sanitizeUserFacingText()` gains conservative duplicate cleanup for same-line/single-newline repeats, safe partial-prefix duplicate handling, and explicit tests for the reported `</final>` boundary shape. The implementation should stay narrowly scoped to the sanitizer/reply normalization path and include regression cases for false positives on ordinary single-newline content.
