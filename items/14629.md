## Summary

Current main still has the reported sanitizer gap: duplicate collapse is paragraph-only, final tags are removed as plain text without boundary cleanup, and existing tests preserve single-newline content rather than covering same-line, single-newline, or partial-prefix duplicate cleanup. The existing maintainer/Codex comment on the issue also reached the same keep-open conclusion, and the provided related items do not supersede this sanitizer-specific request.

## Best Possible Solution

Keep #14629 open. The best path is a narrow sanitizer change in `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts` that conservatively handles same-line/single-newline repeats, the reported `</final>` boundary shape, and safe partial-prefix duplicates, with focused tests in `src/agents/pi-embedded-helpers.sanitizeuserfacingtext.test.ts` that also preserve legitimate single-newline and intentionally repeated content.

## Evidence

- **Current sanitizer only collapses blank-line-separated duplicate blocks:** `collapseConsecutiveDuplicateBlocks()` trims the input, splits only on `/\n{2,}/`, returns the original text when fewer than two blocks exist, and only removes adjacent blocks whose normalized text is exactly equal. Same-line, single-newline, and partial duplicates are not handled here.
