## Summary

Keep open. Current `main` still generates the Windows update restart batch with the same `for /f ('netstat -ano ^| findstr ...')` pattern the report identifies, and the test suite explicitly asserts that pattern. There is also no diff in `src/cli/update-cli/restart-helper.ts` or its tests between the latest release commit `a9797214338ba31c52c796adbb75afb16e0684a9` (`v2026.4.23`) and current `main` `78a431ed224b6f5b9d58672346551a5b411ad200`, so there is no evidence this regression was fixed on main.
