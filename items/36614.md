## Summary

Keep #36614 open. Current main still has the write-side route contamination pattern: `recordInboundSession` writes `deliveryContext`, `lastChannel`, `lastTo`, and related fields to the caller-supplied last-route target, and several direct-message channel/plugin paths still pass the agent main session key even when `dmScope=per-channel-peer` resolves inbound messages to isolated sessions. Some read-side/migration logic can reduce visible misrouting, but it does not remove the underlying write-path bug across the affected inbound surfaces.

## Best Possible Solution

Keep #36614 open and finish the write-side fix. The maintainer path should centralize the last-route target decision or update affected direct-message callers to use `resolveInboundLastRouteSessionKey`, so routes with `lastRoutePolicy: session` write delivery metadata only to the scoped inbound session while `dmScope=main` continues updating the main session. Add regression coverage for iMessage plus representative Slack, Signal, LINE, Matrix, Mattermost, and Discord component paths, preserving the existing Telegram and WhatsApp guarded behavior.
