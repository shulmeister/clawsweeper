## Summary

Keep #36614 open. Current main still has the reported write-side contamination path: per-channel-peer DMs resolve to isolated session keys, but iMessage and several other direct-message handlers still send main-session last-route updates, and the central recorder/store still persists deliveryContext/lastChannel/lastTo to the requested target session.

## Best Possible Solution

Keep #36614 open and finish the write-side fix. The best maintainer path is to make direct-message last-route updates use resolveInboundLastRouteSessionKey or an equivalent central guard so route fields are written only to the scoped inbound session when lastRoutePolicy is session, while preserving main-session writes for dmScope=main. Add regression coverage for iMessage plus representative Slack/Signal/LINE/Matrix/Mattermost paths and keep the existing WhatsApp guarded behavior intact.
