## Summary

Keep #36614 open. Current main has partial read-side and some channel-side improvements, but the reported write-side contamination still exists: several DM channel handlers still write inbound last-route delivery data to the agent main session while per-channel-peer routing resolves the actual inbound DM to a non-main session.

## Best Possible Solution

Keep #36614 open and finish the write-side fix. The likely maintainer path is to make all inbound DM last-route updates use `resolveInboundLastRouteSessionKey` or an equivalent central guard, including direct `updateLastRoute` callers, then add regression coverage for iMessage plus representative `recordInboundSession` and direct-update channel paths under `session.dmScope: "per-channel-peer"`.
