## Summary

Keep this PR open. Current main still lacks the proposed server-side dual-bind behavior for tailnet/custom bind hosts, while local gateway callers still construct loopback URLs. The open #50751 client-side alternative is overlapping, but it is not merged or shipped and does not make this focused PR obsolete.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either land a current-main version of the server-side dual-bind change with the PR's tailnet/custom regression tests and docs updates, or explicitly choose the #50751 client-side host-resolution approach and close this PR only after the shipped path demonstrably fixes internal calls, CLI probes, and `sessions_spawn` for `gateway.bind=tailnet` and `gateway.bind=custom`.
