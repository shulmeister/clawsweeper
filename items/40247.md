## Summary

Keep #40247 open. Current main still does not bind loopback alongside tailnet/custom specific-IP gateway binds, while local gateway callers still default to `127.0.0.1`. The PR remains a focused, plausible server-side fix with targeted regression tests, and the related open client-side PR #50751 has not shipped or clearly superseded this approach.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is either to land a current-main version of this server-side dual-bind fix with the PR's tailnet/custom regression tests, or explicitly choose the client-side host-resolution approach in #50751 and close #40247 only after the shipped path demonstrably fixes internal calls, CLI probes, and `sessions_spawn` for `gateway.bind=tailnet` and `gateway.bind=custom`.
