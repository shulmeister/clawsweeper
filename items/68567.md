## Summary

Keep open. Current main has not implemented this PR's gateway startup cache bypass: startup still calls `loadGatewayPlugins` without a cache option, `loadGatewayPlugins` still calls `loadOpenClawPlugins` without forwarding cache state, and the loader only bypasses registry reuse when `options.cache === false`. The PR head appears to address the early bot-review concern by adding/forwarding `cache`, and its checks are green. No canonical duplicate or shipped replacement was found.

## Best Possible Solution

Keep the PR open for maintainer review rather than auto-closing it. The best path is to narrow or split the branch, keep the gateway change that explicitly passes `cache: false` through `prepareGatewayPluginLoad` -> `loadGatewayPlugins` -> `loadOpenClawPlugins`, and add a gateway bootstrap regression test proving startup/reload bypasses the registry cache when intended. Separately route the memory-wiki runtime-loading pieces through the existing wiki/runtime PRs if those remain needed.
