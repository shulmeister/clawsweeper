## Summary

Keep open. Current `main` does treat `OPENCLAW_RAW_STREAM=0` as disabled inside the raw-stream writer, but the reported LaunchAgent/operator problem is not fully closed: docs still only describe enabling raw stream logging, not supported disable values or durable env precedence, and the LaunchAgent/service env paths can still source `OPENCLAW_RAW_STREAM` from state-dir `.env`/config independently of `launchctl` service args. The related #18740/#18753 context covered enabling via CLI flags, not this disable/LaunchAgent semantics report.

## Best Possible Solution

Keep this issue open and resolve it with a narrow docs-and-regression pass: document that raw stream logging is enabled only by truthy values (`1`, `true`, `yes`, `on`) and disabled by unset or any other value such as `0`, `false`, or `off`, while calling out durable `~/.openclaw/.env` / `$OPENCLAW_STATE_DIR/.env` and config `env` precedence for managed services. Add focused tests for `appendRawStream` disabled values and for LaunchAgent install/restart service-env planning so operators have a verified disable path. If current behavior still allows appends with process env `OPENCLAW_RAW_STREAM=0`, fix the runtime path rather than closing as documentation-only.
