## Summary

Keep PR #43416 open. Current main does not implement the native Swift OpenClawChatUI assistant-message copy affordance this PR targets: the shared chat bubble renders assistant text, tool cards, attachments, and selectable text, but no copy control. The related Control UI/WebChat copy action is a separate web surface and does not cover the macOS/iOS shared Swift chat UI. The PR remains useful, but it should be revised before merge so it copies the same visible assistant text that the renderer shows and has a touch-device affordance on iOS.

## Best Possible Solution

Keep the PR open or supersede it with a focused OpenClawKit change that adds an assistant-only copy action in apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMessageViews.swift, copies only the rendered visible assistant text from AssistantTextParser, supports both pointer and touch devices, and includes focused Swift tests for the visible-text helper. The existing Control UI copy-as-markdown implementation can remain separate.
