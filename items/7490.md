## Summary

Keep #7490 open. Current main at 82262387651ab270b565f71533ce4937c9c00dc6 still lacks the requested optional per-agent `description` config field and still does not expose specialization metadata from `agents_list`. The related PR #7530 was closed unmerged, and the existing issue discussion already identified this same remaining gap.

## Best Possible Solution

Keep #7490 open until maintainers either implement or explicitly reject a core discovery field. A complete implementation should add optional `description` support to the agent config type, strict Zod runtime schema, generated schema/help metadata, config resolver path, and `agents_list` response, then update multi-agent/sub-agent docs and focused tests for description inclusion, omission, and continued rejection of unrelated unknown keys.
