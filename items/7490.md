## Summary

Keep #7490 open. Current main still does not implement the requested optional per-agent `description` field: the TypeScript config type omits it, the strict Zod and generated config schemas reject unknown agent-entry fields, and `agents_list` still returns only id/name/configured/model/embedded runtime metadata. The related PR #7530 attempted this exact feature but was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep #7490 open until OpenClaw either implements the additive core config/tool behavior or a maintainer explicitly rejects the product surface. A complete fix should add `description?: string` to `AgentConfig`, accept it in `AgentEntrySchema` and generated config schema/help/docs, include a trimmed optional `description` in `agents_list` responses for configured agents, and add focused tests for inclusion and omission.
