## Summary

Keep #7490 open. Current main still does not implement the requested optional per-agent `description` config field, the strict config schema still rejects undeclared per-agent keys, and `agents_list` still returns only id/name/configured/model/embedded runtime metadata. The directly related PR #7530 attempted this feature but the provided GitHub context shows it was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open until OpenClaw either implements the additive core config/tool behavior or a maintainer explicitly rejects the product surface. A complete fix should add `description?: string` to `AgentConfig`, accept it in `AgentEntrySchema` and generated config schema/help/docs, include a trimmed optional `description` in `agents_list` responses for configured agents, and add tests for both inclusion and omission.
