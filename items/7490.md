## Summary

Keep #7490 open. Current main still does not implement the requested optional per-agent `description` field for dynamic agent discovery: the config type, strict runtime schema, generated schema, resolver surface, `agents_list` response, tests, and docs all expose agent ids/name/model/runtime metadata without a description. The related implementation PR #7530 was closed unmerged, so it does not supersede or ship this request.

## Best Possible Solution

Keep #7490 open until a maintainer accepts, implements, or explicitly rejects the additive core discovery surface. A complete implementation should add `description?: string` to `AgentConfig`, accept and preserve a trimmed optional `description` in `AgentEntrySchema`, regenerate config schema/help metadata and labels, include description in `agents_list` for configured agents, update public docs such as `https://docs.openclaw.ai/tools/subagents`, and add focused tests for inclusion, omission, and strict rejection of unrelated unknown keys.
