## Summary

Keep #7490 open. Current main still lacks the requested optional per-agent `description` config field, the strict agent schema and generated JSON schema do not accept it, and `agents_list` still returns only id/name/configured/model/embedded runtime metadata. The related PR #7530 attempted this feature but was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open until OpenClaw either implements the additive core config/tool behavior or a maintainer explicitly rejects the product surface. A complete fix should add `description?: string` to `AgentConfig`, accept it in `AgentEntrySchema` and the generated config schema/help/docs, include a trimmed optional `description` in `agents_list` responses for configured agents, and add tests for inclusion and omission.
