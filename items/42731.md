## Summary

PR #42731 is a docs-only discoverability/setup page for the third-party cursor-agent-api-proxy project. Current OpenClaw already supports the underlying OpenAI-compatible custom-provider setup through `models.providers` and onboarding, while current scope docs put third-party plugin/provider discovery, provenance, and promotion in ClawHub rather than core docs. The patch also adds hand-authored zh-CN docs, which current docs policy explicitly disallows in this repo.

## Best Possible Solution

Close this PR and route cursor-agent-api-proxy discoverability to ClawHub or the upstream cursor-agent-api-proxy repository. If tighter OpenClaw integration is desired, publish a small external provider plugin or setup bundle through ClawHub using the existing provider plugin/custom-provider surfaces, while keeping core docs focused on generic provider extension points.
