## Summary

Keep open. Current `main` still restores cached memory state with the destructive `restoreMemoryPluginState` path, so the reported cache-hit clobber is not implemented away on `main`; at the same time, the PR discussion surfaces a credible unresolved correctness risk in the revised additive-merge approach, so this is not a high-confidence auto-close candidate.
