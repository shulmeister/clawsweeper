## Summary

Close #68334. Current main already fixes the observable memory-wiki public-artifact loss: warm plugin-registry cache hits restore and persist memory capability, and the later activate:false snapshot clobbering path was fixed by #69219 and shipped in v2026.4.20. The proposed additive merge split is no longer the best direction because current main keeps destructive full-state restore semantics for cache activation and rollback.

## Best Possible Solution

Close #68334 as already implemented on main. Keep the shipped loader behavior: cache entries persist and restore memoryCapability, activate:false snapshot restores include previousMemoryCapability, and restoreMemoryPluginState remains a destructive full-state swap for rollback/cache correctness. Future work should be a new focused issue only if a fresh reproduction shows public artifacts still disappearing on current main.
