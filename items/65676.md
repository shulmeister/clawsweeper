## Summary

Current `main` already separates explicit user-driven live model switches from system fallback rotation and also treats any `LiveSessionModelSwitchError` during fallback as failover instead of re-entering the retry loop, which matches the reported bug.

## Evidence

- **Runner only switches on explicit pending live-switch flag:** `run.ts` now calls `shouldSwitchToLiveModel(...)` before throwing `LiveSessionModelSwitchError`, rather than unconditionally comparing against persisted selection every loop.
  - file: [src/agents/pi-embedded-runner/run.ts:1015](https://github.com/openclaw/openclaw/blob/6d271762ab50fb4376b8859bcf55eefb2d3814ea/src/agents/pi-embedded-runner/run.ts#L1015)
  - command: `nl -ba src/agents/pi-embedded-runner/run.ts | sed -n '1002,1035p'`
  - sha: [6d271762ab50](https://github.com/openclaw/openclaw/commit/6d271762ab50fb4376b8859bcf55eefb2d3814ea)
- **Session contract excludes fallback from live-switch detection:** The session entry type documents `liveModelSwitchPending` as set only for explicit user-driven model changes, and explicitly says system-initiated fallbacks never set it, so fallback selection is not mistaken for a live model switch.
  - file: [src/config/sessions/types.ts:190](https://github.com/openclaw/openclaw/blob/6d271762ab50fb4376b8859bcf55eefb2d3814ea/src/config/sessions/types.ts#L190)
  - command: `nl -ba src/config/sessions/types.ts | sed -n '186,205p'`
  - sha: [6d271762ab50](https://github.com/openclaw/openclaw/commit/6d271762ab50fb4376b8859bcf55eefb2d3814ea)
- **Fallback path no longer rethrows switch error into outer retry loop:** `runWithModelFallback` catches `LiveSessionModelSwitchError`, normalizes it into failover state, records the failed candidate, and continues to the next fallback candidate instead of rethrowing and looping.
  - file: [src/agents/model-fallback.ts:921](https://github.com/openclaw/openclaw/blob/6d271762ab50fb4376b8859bcf55eefb2d3814ea/src/agents/model-fallback.ts#L921)
  - command: `nl -ba src/agents/model-fallback.ts | sed -n '880,970p'`
  - sha: [6d271762ab50](https://github.com/openclaw/openclaw/commit/6d271762ab50fb4376b8859bcf55eefb2d3814ea)
- **Regression tests cover the exact loop shape:** `model-fallback.test.ts` includes tests asserting that `LiveSessionModelSwitchError` on the last candidate does not escape as a live-switch retry, and that fallback continues to the next candidate when such an error occurs.
  - file: [src/agents/model-fallback.test.ts:644](https://github.com/openclaw/openclaw/blob/6d271762ab50fb4376b8859bcf55eefb2d3814ea/src/agents/model-fallback.test.ts#L644)
