## Summary

Keep open. Current main still appears to have the bug this PR targets: `executeNodeHostCommand` checks only for `system.run` and then unconditionally invokes `system.run.prepare`, while the macOS node command enum/advertisement path on main still lacks `system.run.prepare`. The related #38781 is relevant, but it is not a clear superseding close target for this PR because it only adds the missing macOS command advertisement and does not cover already-installed or legacy nodes that advertise `system.run` without `system.run.prepare`.
