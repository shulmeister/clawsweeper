## Summary

Keep open. Current main still has the behavior this PR documents: cron-tagged/main-session scheduled events bypass HEARTBEAT.md file-gate parsing and use the specialized cron-event prompt, while the cron docs and AGENTS.md template still do not explain that those cron prompts do not automatically include the default “Read HEARTBEAT.md” instruction. The PR discussion also supports the documentation as accurate, with only style/template wording feedback remaining.

## Best Possible Solution

Keep this PR open or replace it with an equivalent docs-only patch. The maintainer path is to document in docs/automation/cron-jobs.md that main-session cron system events use a self-contained cron-event prompt and do not automatically include the default HEARTBEAT.md instruction, then add direct agent-facing cron handling guidance to docs/reference/templates/AGENTS.md. Any runtime change that automatically merges HEARTBEAT.md into cron-event prompts should be tracked separately with prompt-selection and file-gate regression tests.
