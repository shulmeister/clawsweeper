## Summary

Keep open. Current main still has the behavior this docs PR explains: cron/system-event heartbeat runs bypass normal heartbeat file gates, choose `buildCronEventPrompt(...)`, and that prompt still does not include `Read HEARTBEAT.md`. Current cron docs and the AGENTS template still lack the proposed clarification/workaround, while existing tests preserve the cron prompt separation.

## Best Possible Solution

Keep this PR open or replace it with an equivalent docs-only patch. The best maintainer path is to document that main-session cron system events use a self-contained cron-event prompt and do not include the default `Read HEARTBEAT.md` instruction, add a clear workaround in `docs/automation/cron-jobs.md`, and make the `docs/reference/templates/AGENTS.md` guidance directly operational for agents. Any runtime behavior change should be tracked separately with updated prompt tests.
