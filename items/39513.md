## Summary

Keep open. Current main still routes cron-triggered main-session heartbeat turns through the specialized cron-event prompt, that prompt still omits the default `Read HEARTBEAT.md` instruction, tests explicitly lock in the omission, and the current cron docs / AGENTS template do not contain the PR’s proposed clarification or workaround. The PR remains relevant documentation work, though the existing review comments suggest small doc-structure/template wording fixes before merge.

## Best Possible Solution

Keep tracking this PR or an equivalent docs patch. The best path is to document that main-session cron system events use a specialized self-contained prompt without the default `HEARTBEAT.md` instruction, add clear workaround guidance to `docs/automation/cron-jobs.md`, and make the AGENTS template guidance operational rather than meta-instructions. If maintainers want the product behavior changed instead, that should be handled as a separate runtime change with updated tests.
