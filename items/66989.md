## Summary

Current main, and the shipped v2026.4.24 tag, implement the Slack Socket Mode inbound path reported here: socket startup uses the account app token, channel/group subscriptions are handled through Bolt's actual `message` event, `app_mention` is registered separately, and tests cover channel/group delivery. The newer #70804 related item describes a different remaining problem after inbound reaches OpenClaw: no outbound reply/slash response.

## Best Possible Solution

Keep the shipped Slack Socket Mode inbound implementation. Use #70804 and its linked reply/slash-command trackers for the separate remaining case where inbound reaches OpenClaw but no outbound reply is sent. If true zero-inbound behavior still reproduces on v2026.4.24 or newer, it should be reopened or filed as a fresh report with current logs, config, channel policy, and status timestamps.
