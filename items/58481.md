## Summary

Keep #58481 open. Current main still creates the WhatsApp Baileys socket without setting TCP keepalive on the underlying socket. Existing WhatsApp keepalive/reconnect code covers application-level Baileys pings, status heartbeats, watchdog reconnects, and outbound retry across reconnects, but it does not implement the issue’s requested OS-level TCP keepalive for WSL2 NAT stability. The issue is recent, has a confirming comment, and the related #61788 remains open rather than superseding it.

## Best Possible Solution

Keep the issue open and implement a narrow WhatsApp plugin fix in `extensions/whatsapp/src/session.ts` or the existing Baileys patch so every WhatsApp WebSocket enables OS-level TCP keepalive, for example `setKeepAlive(true, 15000)` once the underlying `ws` raw socket exists. Add a focused regression test with a mocked Baileys/WebSocket client proving the raw socket receives TCP keepalive, while preserving the existing Baileys WS ping interval, proxy-agent behavior, reconnect watchdog, and outbound reconnect retry behavior. Docs/changelog only need updates if maintainers want to advertise the WSL2 stability fix.
