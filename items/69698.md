## Summary

Keep #69698 open. Current main still has the reported mismatch: persisted exec allowlist entries can contain internal `source` metadata, gateway/node snapshot transport only redacts socket data, the Control UI clones and resubmits the snapshot file unchanged, and `exec.approvals.set` still rejects `source` via the protocol schema. The linked PR #69732 is an open, unmerged candidate fix rather than shipped behavior on main.

## Best Possible Solution

Keep the issue open and review/land a focused fix like #69732: preserve `source` as internal persisted metadata, but strip internal allowlist fields from gateway and node-host exec approvals transport payloads before the Control UI receives or resubmits them. Add regression coverage for an allow-always entry with `source` round-tripping through `exec.approvals.get` and `exec.approvals.set` without schema rejection.
