## Summary

Keep #71049 open. Current main still hardcodes the Microsoft Teams streaminfo/informative status path for personal DMs and does not expose a server-side config option to disable it. The existing `typingIndicator` flag only controls the native typing keepalive path, and the test suite explicitly preserves that `typingIndicator=false` still sends the informative update.

## Best Possible Solution

Keep this open as a bundled Microsoft Teams plugin feature request. A good fix would add a typed `channels.msteams` option, likely under a Teams-specific streaming/DM status config, validate it in the strict schema and generated metadata, document it on the Teams channel docs page, pass it into `createTeamsReplyStreamController`, and add tests proving personal DMs can use native typing plus final-message delivery without the streaminfo informative status.
