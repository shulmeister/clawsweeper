## Summary

Keep #36314 open. Current main has several related robustness pieces, including supervisor restart policies, global unhandled-rejection/exception handling, channel health monitoring with restart caps, config last-known-good recovery, and health diagnostics. The issue asks for a broader Gateway-level watchdog/error-recovery/health-degradation feature set, and meaningful requested behavior remains missing: no configurable Gateway process watchdog with maxRestarts/restartWindow/autoStop notification semantics, no configurable global recovery strategy map, and no /health payload with memory/connection/error counters.

## Best Possible Solution

Keep the issue open as the umbrella for Gateway robustness until maintainers either implement a first-class Gateway process watchdog/error-strategy/health-degradation design or split it into narrower tracked issues. A conservative implementation would extend the Gateway config/schema/docs with explicit watchdog settings, add restart-loop detection and notification semantics above platform supervisor defaults, expose richer health metrics deliberately, and keep existing channel-health/config-recovery behavior as partial building blocks.
