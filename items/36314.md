## Summary

Keep #36314 open. Current main has several robustness pieces, including channel health restarts, supervisor restart limits, global unhandled rejection/exception handling, readiness probes, stability bundles, and last-known-good config recovery. The issue asks for a cohesive Gateway-level process watchdog, configurable recovery strategy map, richer health counters, and deliberate degraded startup mode; those remain only partially covered.

## Best Possible Solution

Keep the umbrella issue open or split it into focused maintainer-owned work: a first-class gateway.watchdog config/schema/docs surface, restart-loop detection and notification semantics above platform supervisors, a deliberate configurable global recovery policy if maintainers want user control there, richer privacy-safe Gateway health metrics, and a degraded startup design that composes with last-known-good config and channel health recovery.
