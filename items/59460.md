## Summary

Current `main` already implements the requested same-agent session visibility via `tools.sessions.visibility: "agent"`, which is functionally the issue's proposed `self-sessions` MVP.

## Evidence

- **Visibility enum already includes same-agent scope:** `SessionToolsVisibility` accepts `"self" | "tree" | "agent" | "all"`, and config typing documents `agent` as same-agent access.
  - file: [src/agents/tools/sessions-access.ts:13](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/tools/sessions-access.ts#L13)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Guard logic allows same-agent sessions without `all`:** Cross-agent targets are blocked unless visibility is `all`; same-agent targets only hit the narrower `self`/`tree` restrictions, so `agent` provides the requested cross-user same-agent visibility.
  - file: [src/agents/tools/sessions-access.ts:206](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/tools/sessions-access.ts#L206)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **User-facing docs describe the exact scope:** The config docs say `agent` means any session belonging to the current agent id, including other users under the same agent, which matches the issue's `self-sessions` use case.
  - file: [docs/gateway/config-tools.md:263](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/docs/gateway/config-tools.md#L263)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Concept docs expose the level publicly:** Session tools docs list `agent` as 'All sessions for this agent'.
  - file: [docs/concepts/session-tool.md:132](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/docs/concepts/session-tool.md#L132)
