## Summary

Current main already preserves configured custom-provider headers for image requests through the resolved model and OpenAI-compatible transport path. PR #67326's direct `getApiKeyAndHeaders` approach is no longer needed for the linked user problem and its current diff would regress caller-supplied `authStore` handling.

## Best Possible Solution

Close this PR as implemented on current main. Keep the shared model-resolution/header propagation path; if maintainers want extra coverage, add a small image-runtime test that verifies configured provider headers survive via `model.headers` without adding an image-only `getApiKeyAndHeaders` auth path.
