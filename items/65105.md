## Summary

Close. Current main and the shipped `v2026.4.23` release already cover the reported config-clobber symptoms: suspicious update-only config roots are auto-restored from backup, destructive partial writes are rejected before replace, backups are taken from the pre-overwrite file, doctor now warns about orphaned agent dirs, and the stale gateway service entrypoint drift was separately fixed.

## Evidence

- **Release notes shipped the config-clobber recovery:** `v2026.4.23` explicitly says OpenClaw now restores the last-known-good config on critical clobber signatures such as missing metadata, missing `gateway.mode`, or sharp size drops, and also recovers configs accidentally prefixed with non-JSON output during gateway startup or `openclaw doctor --fix`. That matches the reported post-update `openclaw.json` collapse.
