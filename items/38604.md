## Summary

Keep open. Current `main` still does not provide a default Docker sandbox `pidsLimit`: config resolution only preserves agent/global values, Docker args only emit `--pids-limit` for an explicit positive number, and the latest release tag has the same behavior. The open PR #44098 tracks the right fix direction, but it is still an unmerged related PR, so the issue should remain open until a focused fix lands.

## Best Possible Solution

Keep #38604 open until a focused core fix lands. The best path is to add a documented `DEFAULT_SANDBOX_PIDS_LIMIT`, apply it in `resolveSandboxDockerConfig` while preserving explicit agent/global overrides, add regression coverage for default and override behavior, ensure Docker creation emits the resolved default, and update the public sandbox/config security docs. #44098 can be the canonical implementation path if maintainers want to continue that PR; otherwise land an equivalent focused patch.
