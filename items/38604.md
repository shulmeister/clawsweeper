## Summary

Keep #38604 open. Current main still resolves Docker sandbox `pidsLimit` only from explicit agent/global config, and Docker create args only include `--pids-limit` when that resolved value is already a positive number. The related PR #44098 is a plausible implementation path but is still open/unmerged in the provided GitHub context; older linked PR attempts were closed unmerged. The WASM-runtime comment is useful broader security discussion, but it does not fix the missing Docker sandbox PID default reported here.

## Best Possible Solution

Keep #38604 open until a focused core fix lands. The likely path is to add a maintainer-approved default PID limit in `resolveSandboxDockerConfig`, preserve explicit agent/global overrides, add regression tests for default and override behavior, confirm `buildSandboxCreateArgs` emits the resolved default, and update the public sandbox/config/security docs. PR #44098 can remain the canonical implementation path if maintainers want to continue it.
