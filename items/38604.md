## Summary

Keep open. Current main still does not provide a default Docker sandbox pidsLimit: config resolution preserves only explicit agent/global values, Docker create args emit --pids-limit only when a positive value is already present, and tests/docs cover explicit configuration rather than a runtime default. The related PR #44098 is the right implementation direction but remains unmerged in the provided GitHub context.

## Best Possible Solution

Keep #38604 open until a focused core fix lands. The best path is to add a documented DEFAULT_SANDBOX_PIDS_LIMIT or maintainer-approved equivalent, apply it in resolveSandboxDockerConfig while preserving explicit agent/global overrides, add regression coverage for default and override behavior, ensure buildSandboxCreateArgs emits the resolved default, and update the public sandbox/config docs. PR #44098 can be the canonical implementation path if maintainers want to continue it; otherwise land an equivalent focused patch.
