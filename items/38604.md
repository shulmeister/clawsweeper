## Summary

Keep open. Current main still resolves sandbox Docker pidsLimit only from agent/global config, with no fallback default, and Docker create args only add --pids-limit when a positive value is present. The broader WASM runtime suggestion in the issue discussion may be useful future sandbox direction, but it does not resolve this Docker hardening gap on main. Provided related context shows PR #44098 is still the focused open fix, while the other related PRs were closed unmerged.

## Best Possible Solution

Keep #38604 open until the Docker sandbox resolver gets an explicit default pidsLimit, preferably via a named constant, with regression coverage proving omitted config produces --pids-limit while explicit agent/global values still take precedence. Review or supersede PR #44098 as the focused implementation path, and update the public sandbox/config docs to explain the default and any operator override guidance.
