## Summary

Keep open. Current main still resolves Docker sandbox pidsLimit only from agent/global config, and Docker container creation only emits --pids-limit when that resolved value is a positive number. The open PR #44098 tracks the same fix, while the related closed PRs in the provided timeline were not merged, so the original issue remains the canonical open report.

## Best Possible Solution

Keep #38604 open until a focused core fix lands. The best path is a small maintainer-reviewed change that adds a documented DEFAULT_SANDBOX_PIDS_LIMIT, applies it in resolveSandboxDockerConfig while preserving explicit agent/global overrides, adds regression coverage for the default and override behavior, and updates the public sandbox/config security docs. #44098 is the existing open PR tracking that direction.
