## Summary

Keep this issue open. Current main still has the strict QMD vector-count parser that can return null for non-colon `qmd status` formats, and there is an open unmerged implementation candidate, #63678, that explicitly targets this bug with `Fixes #63652`. Per the review rule, the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open while maintainers review and either land #63678 after resolving its dirty/stale state, or apply an equivalent scoped memory-core fix on main: broaden the QMD vector-count parser with line-anchored handling for supported `qmd status` variants and add regression tests for those formats, without unrelated lockfile or dependency churn.
