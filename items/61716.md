## Summary

Current main does not implement the requested OpenAI-compatible custom-provider onboarding prompts. The classic onboarding flow collects base URL, API key, endpoint compatibility, model ID, endpoint ID, and optional alias, then writes default model metadata. The config/schema already supports `input`, `contextWindow`, and `maxTokens`, and docs still direct users to config/model catalog entries for these fields. The linked #64213 is related and useful, but it tracks runtime image-capability fallback when configured metadata exists; it does not supersede this onboarding UX request.
