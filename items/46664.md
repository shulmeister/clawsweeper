## Summary

Keep #46664 open. Current main still has only narrow, mostly in-memory mobile queuing paths: Android queues recognized voice turns in memory and flushes them on gateway reconnect, iOS queues watch quick replies in memory, and normal text/chat or iOS push-to-talk still fail or return offline when the gateway is unavailable. The issue discussion accurately narrows the remaining work to a persistent, general mobile command outbox with replay semantics; that capability is not implemented on main.
