## Summary

Current `main` no longer sends `spawnedBy` or `workspaceDir` through the public `agent` RPC during `sessions_spawn`; it persists lineage/workspace on the child session and strips those fields before the validated `agent` call. The gateway schema still rejects those fields on public `agent` requests, and regression tests cover both behaviors. The same implementation is present in release `v2026.4.22`.

## Evidence

- **Subagent spawn stores lineage on session and strips internal fields before `agent` RPC:** `spawnedBy` and inherited workspace are first persisted via `patchChildSession({ spawnedBy, spawnedWorkspaceDir })`, then destructured out of `spawnedMetadata` so only `publicSpawnedMetadata` is passed to `callSubagentGateway({ method: "agent" ... })`. That directly avoids the reported validation error path.
  - file: [src/agents/subagent-spawn.ts:905](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/subagent-spawn.ts#L905)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Gateway `agent` schema still forbids those extra properties:** `AgentParamsSchema` does not include `spawnedBy` or `workspaceDir` and uses `additionalProperties: false`; the handler returns `invalid agent params` when validation fails. That means the current fix is to keep those fields out of the public `agent` request, not to accept them there.
  - file: [src/gateway/protocol/schema/agent.ts:131](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/gateway/protocol/schema/agent.ts#L131)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Regression tests cover both rejection and correct inheritance path:** Tests assert that public `agent` calls carrying `spawnedBy`/`workspaceDir` are rejected as invalid, and separately assert that a spawned session still receives inherited workspace only from stored session metadata.
  - file: [src/gateway/server-methods/agent.test.ts:798](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/gateway/server-methods/agent.test.ts#L798)

## Close Comment

Closing after Codex review. Current `main` already avoids this failure: `src/agents/subagent-spawn.ts:917` stores `spawnedBy`/`spawnedWorkspaceDir` on the child session, and `src/agents/subagent-spawn.ts:964` strips `spawnedBy` and `workspaceDir` before calling the public `agent` RPC. The public `agent` schema in `src/gateway/protocol/schema/agent.ts:131` still rejects those extra properties by design, and `src/gateway/server-methods/agent.test.ts:798` plus `src/gateway/server-methods/agent.test.ts:903` cover both rejecting public injection and forwarding inherited workspace only from stored session metadata. This implementation is also present in released tag `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
