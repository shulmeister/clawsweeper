## Summary

Keep #14747 open. Current main still has no user-configurable lane wait diagnostic threshold; the queue falls back to a hardcoded 2000 ms unless each enqueue passes warnAfterMs, and the diagnostics config schema/types do not expose laneWaitWarnMs. The related PR #16366 addressed the request in discussion but was closed unmerged, so it does not supersede the issue.
