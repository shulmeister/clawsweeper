## Summary

Keep #14747 open. Current main still hardcodes the lane wait diagnostic fallback and does not expose a user-configurable diagnostics lane wait threshold in the config schema, types, labels, UI config section, or docs. The related PR #16366 targeted this issue but was closed unmerged, and its head commit is not present in the local checkout.

## Best Possible Solution

Keep the issue open until OpenClaw adds a maintainer-approved configurable lane wait diagnostic threshold. A focused implementation should update config schema/types/labels/docs/UI config surfaces, wire the command queue fallback to that config while preserving per-call `warnAfterMs`, and add regression coverage for the configured default and cron enqueue behavior.
