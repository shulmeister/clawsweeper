## Summary

Current main already implements the requested Slack post-then-edit UX through Slack draft preview streaming and native single-message streaming; the feature is documented and regression-tested, and the changelog shows it shipped in 2026.2.17.

## Evidence

- **Release note:** CHANGELOG entry for 2026.2.17 says Slack gained native single-message text streaming plus configurable draft preview streaming modes.
  - file: [CHANGELOG.md:4595](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/CHANGELOG.md#L4595)
  - command: `nl -ba CHANGELOG.md | sed -n '4582,4600p'`
- **User docs:** Slack docs say `channels.slack.streaming` controls live preview behavior, with `partial` as the default and `progress` status text mode available; docs also describe draft preview fallback when native transport is unavailable.
  - file: [docs/channels/slack.md:553](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/docs/channels/slack.md#L553)
  - command: `nl -ba docs/channels/slack.md | sed -n '551,579p'`
- **Post then edit implementation:** `createSlackDraftStream` sends the first preview message, stores channel/message ids, and edits that same message on subsequent updates.
  - file: [extensions/slack/src/draft-stream.ts:48](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/extensions/slack/src/draft-stream.ts#L48)
  - command: `nl -ba extensions/slack/src/draft-stream.ts | sed -n '45,90p'`

  - file: [extensions/slack/src/draft-stream.test.ts:45](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/extensions/slack/src/draft-stream.test.ts#L45)

## Risks / Open Questions

- The exact custom placeholder-message rotation config proposed in the issue is not present as-is; current main covers the underlying UX with built-in preview/native streaming modes rather than a user-supplied placeholder message list.

## Close Comment

Closing from a Codex review: current `main` already has this behavior. The feature shipped in `2026.2.17` (`CHANGELOG.md:4595-4596`), and the current implementation posts the first Slack preview message and then edits that same message on later updates (`extensions/slack/src/draft-stream.ts:48-79`). Finalization also edits the preview in place when safe (`extensions/slack/src/monitor/message-handler/dispatch.ts:678-723`). The user docs describe Slack live preview streaming as built in, with `partial` mode updating the preview in place and `progress` mode showing status text while generating (`docs/channels/slack.md:553-568`), and there is regression coverage for the send-then-edit flow in `extensions/slack/src/draft-stream.test.ts:45-55`.

If the remaining ask is specifically custom placeholder text rotation or a messages-array config surface, that would need a new scoped follow-up issue, but the core post-then-edit Slack UX requested here is already implemented on `main`.
