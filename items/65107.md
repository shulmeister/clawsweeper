## Summary

The reported 2026.4.11 browser regression is already fixed on current main and was shipped in v2026.4.14. Current code allows normal public hostname navigation by default, while the IP-literal/hostname-allowlist restriction only applies when operators explicitly opt into strict browser SSRF mode.

## Evidence

## Risks / Open Questions

- Installs that explicitly set `browser.ssrfPolicy.dangerouslyAllowPrivateNetwork: false` without an allowlist will still reproduce the IP-literal restriction; that is intentional current behavior, not the default-path bug reported here.
