## Summary

Keep #37842 open. Current main still lacks graph-aware sessions_send cycle detection: inter-session provenance is source-only, sessions_send does not read or extend an incoming chain, and the existing A2A limiter still bounds only the direct requester-target ping-pong flow.

## Best Possible Solution

Keep #37842 open for a core sessions_send fix. Add an explicit chain or trace field to InputProvenance and the gateway schema, propagate it through createSessionsSendTool, runAgentStep, and A2A follow-up runs, reject or warn when the target agent/session already appears in the chain, and add regression coverage for A->B->C->A while preserving existing maxPingPongTurns behavior for direct A<->B exchanges.
