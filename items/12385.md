## Close Comment

Closing as `clawhub` after a Codex review. Current `main` already exposes plugin-owned policy seams for the main enforcement points this proposal is targeting: prompt/session hooks in `docs/concepts/agent-loop.md:93-100`, tool-call block/approval hooks in `docs/plugins/building-plugins.md:182-194` and `src/plugins/hook-types.ts:324-336`, install-time policy hooks in `src/plugins/hooks.ts:1064-1091`, and fail-closed handling for `before_tool_call` in `src/plugins/hook-runner-global.ts:42-45`. Repo scope also points optional capability out of core: `VISION.md:54-70` says optional capability should usually ship as plugins and new skills should go to ClawHub first. Since SHIELD is proposed as a reusable, tool-agnostic security policy layer that could work across agents, this fits better as a ClawHub plugin/guardrail package on top of the existing hook surface than as new OpenClaw core behavior. If a concrete SHIELD plugin hits a missing runtime seam on current `main`, please open a focused issue for that specific core gap.
