## Summary

Keep PR #43097 open. Current main does not contain the proposed Pi tool-definition adapter conformance harness or the adapter-level already-aborted-signal guard, and the PR still targets a real Pi adapter boundary with useful coverage. It is not protected by author association or label, but none of the conservative close reasons apply because the work is neither implemented nor clearly superseded.

## Best Possible Solution

Keep the PR open, rebase it onto current main, and revise it rather than close it. The best path is to preserve the machine-readable Pi adapter conformance harness, add focused current/legacy execute-layout coverage for both agent and client tool adapters, keep the already-aborted behavior where it is semantically safe, remove or move the post-`onClientToolCall` abort check, and clarify the bash-to-exec normalization assertion in the test.
