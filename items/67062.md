## Summary

Keep #67062 open. Current main still hardcodes QQ Bot media SSRF hostnames and does not expose or consume `channels.qqbot.mediaAllowHosts`. The reporter's example host `multimedia.nt.qq.com.cn` is likely already covered by the existing `*.qq.com.cn` wildcard, but the requested configurable host override remains unimplemented and actionable.

## Best Possible Solution

Keep the issue open and implement a plugin-owned QQ Bot config surface, likely `channels.qqbot.mediaAllowHosts` with account-level support if QQ Bot account config needs parity. The implementation should normalize/validate host suffixes, merge them with the existing Tencent defaults, pass the resulting SSRF policy through `downloadFile()`/media fetch paths, add focused tests, and document the public option on the QQ Bot docs page.
