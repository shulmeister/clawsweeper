## Summary

Keep open. Current `main` only supports static `headers` on remote MCP server definitions and passes that fixed map into SSE/streamable HTTP transports when the MCP session is created. I found no MCP-specific request-time header hook or runtime header provider, so the requested per-request/session header support is not implemented on `main`, and this is not a stale or incoherent report.

## Evidence
