## Summary

Keep open. Current main still supports only configured/static remote MCP HTTP headers. The config/schema, HTTP resolver, transport construction, runtime client reuse, docs, and plugin surfaces do not provide a per-request header resolver, headersProvider, or session/runtime-context templating path. Related #65590 is static streamable-http header forwarding, and #62776/#62134 cover OAuth auth flow work, not this broader dynamic header capability.

## Best Possible Solution

Keep this issue open for maintainer design. The best path is either a core MCP runtime capability or an explicit public plugin API extension: add a typed request-scoped header resolver for remote MCP transports, thread session/runtime context into MCP request handling, merge dynamic headers after static config with clear precedence and redaction rules, cover both streamable-http and SSE, and update docs/tests for tenant/user/token-refresh behavior.
