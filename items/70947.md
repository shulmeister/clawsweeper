## Summary

Keep open. Current main still treats Control UI/webchat attachments as image-only through the picker, local payload serializer, gateway parser, transcript rendering, and tests. The PR is open and appears to be the current focused attempt to fix #69447/#57707-style work, not superseded by a landed implementation, though it needs review fixes before merge.

## Best Possible Solution

Keep the PR open and require targeted fixes before merge: add pre-decode base64/decoded-size validation for generic files, preserve the existing image sniff fallback for omitted or mislabeled image MIME values, keep/expand the proposed UI/gateway/renderer/media-store regression coverage, and fix the changelog attribution. If maintainers choose not to take this exact branch, open a canonical replacement PR that implements the same end-to-end Control UI + gateway generic attachment behavior because current main still lacks it.
