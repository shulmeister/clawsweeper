## Summary

Current main still does not implement the requested exec-approvals state-dir fix. The default exec approvals file and socket still resolve through hardcoded `~/.openclaw` paths, while `resolveStateDir()` already has the `OPENCLAW_STATE_DIR` precedence contract. This PR is therefore not obsolete; it should stay open for maintainer consolidation with the overlapping open #65736 and the linked open #62917.

## Best Possible Solution

Keep this PR open for maintainer handling. The best path is to consolidate it with #65736 or explicitly choose one surviving branch, then land a focused fix that derives exec approvals file/socket defaults from `resolveStateDir()`, preserves existing fallback behavior, adds `OPENCLAW_STATE_DIR` precedence and file-creation coverage, and either includes or deliberately splits the adjacent hardcoded-path cleanups noted in review.
