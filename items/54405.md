## Summary

Keep this issue open. Current main still does not implement client-provided tool passthrough for `/v1/chat/completions`, while PR #66278 is an open, unmerged implementation candidate that uses closing syntax for this issue. Under the review rules, the issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and treat PR #66278 as the active implementation candidate. The maintainer path is to review, revise, and land that PR or an equivalent gateway fix that parses Chat Completions `tools` and `tool_choice`, forwards client tools into agent runs, emits OpenAI-compatible `tool_calls` for non-stream and streaming responses, handles `role: "tool"` follow-up messages, and adds focused contract tests. After that merges, close this issue as implemented.
