## Summary

Keep open. Current `main` still lacks the requested malformed-`SKILL.md` visibility/validation path: the workspace skill loader drops invalid or incomplete skill entries without surfacing diagnostics, and the CLI/docs expose `list`, `info`, and `check` but no `skills validate` command. One detail in the report is outdated: missing `name` no longer blocks loading because the loader falls back to the skill directory name.

## Evidence

- **Malformed or missing frontmatter does not raise a user-visible error:** `parseFrontmatterBlock()` returns an empty object when the opening/closing `---` block is missing, instead of surfacing a parse failure.
  - file: [src/markdown/frontmatter.ts:195](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/markdown/frontmatter.ts#L195)
