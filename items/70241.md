## Summary

Keep open. Current main still documents a ClawHub plugin publish surface that the issue discussion says is absent from the shipped `clawhub@latest` client, and the repo does not contain a current-main fix for the reported 120s `clawhub package publish` timeout. The OpenClaw release workflow also depends on a pinned external `openclaw/clawhub` checkout, which supports the docs/client mismatch rather than resolving it for external plugin authors.

## Best Possible Solution

Keep the issue open and reconcile the publish surface at the source of truth: either publish a newer `clawhub` CLI that supports the documented plugin publish source forms, `--dry-run`, and `--json`, then diagnose the 120s publish timeout with client/server timeout ownership; or update OpenClaw docs and release helpers to match the currently shipped `clawhub@latest` surface. If the timeout fix belongs in `openclaw/clawhub`, link a canonical ClawHub-side tracker while keeping this issue open for the OpenClaw docs/client mismatch.
