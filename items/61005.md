## Summary

Current main still has the reported Android onboarding blocker: the final onboarding gate requires both operator connectivity and node connectivity, so `Connected (operator offline)` remains unable to finish. The linked PR #61269 is relevant and positive prior work, but it is open/unmerged and the current main code/test expectations still encode the blocking behavior.
