## Summary

Keep #70438 open. Current main still has image-only WebChat attachment handling and no `gateway.webchat.attachments` policy surface for PDF offload, text inlining, or inline text caps. Related #30759 and #60226 cover broader document upload/context gaps, and open PR #70947 may address generic non-image file persistence, but it explicitly does not add configurable PDF/text handling, so this narrower config request is not implemented or fully superseded.

## Best Possible Solution

Keep #70438 open unless maintainers intentionally fold it into a canonical WebChat attachment policy tracker. The best product path is a typed `gateway.webchat.attachments` config with documented defaults, Control UI accept/filter behavior driven by that policy, Gateway handling for PDF `media://inbound/<id>` refs, capped UTF-8 inline handling for text/JSON/XML/YAML/CSV/code MIME types, unsupported-file feedback, and focused UI/Gateway tests. Coordinate with #30759, #60226, and PR #70947 so generic file persistence and configurable PDF/text behavior do not split into conflicting implementations.
