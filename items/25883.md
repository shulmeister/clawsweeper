## Summary

Keep #25883 open. Current main still does not implement the requested Redis/BullMQ-style distributed job orchestration or horizontal worker coordination; cron, task, and command queue behavior remain Gateway-local, in-process, local-file, or local-registry backed. The existing issue discussion also already preserves the linked ScalyClaw prototype as prior art and explicitly kept this item open for architecture review.

## Best Possible Solution

Keep #25883 open for maintainer architecture review. The concrete path is to decide whether OpenClaw should add a pluggable distributed scheduler/job backend with shared queue/state, worker claiming, retry/backoff, dead-letter or durable failure state, cron support, and observability; otherwise document Redis/BullMQ-style orchestration as an external deployment pattern. Preserve the linked ScalyClaw prototype as prior art.
