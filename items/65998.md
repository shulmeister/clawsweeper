## Summary

Current main and v2026.4.24 solve the reported Active Memory failure. Active Memory still uses a memory-only `toolsAllow`, the embedded runner now forwards that allowlist into the attempt path, and the attempt path skips bundled MCP startup when the runtime allowlist cannot reach bundled MCP tool names, so large globally configured MCP schemas are no longer sent to the Active Memory sub-agent. The related #66581/#69892 forwarding reports describe the older missing-forwarding path; current main includes the forwarding and release notes include the bundle-MCP allowlist startup fix.

## Best Possible Solution

Close #65998 as implemented on main and shipped in v2026.4.24. Keep the current shipped behavior: Active Memory uses the memory-only allowlist, the embedded runner forwards it, and the attempt path avoids bundle-MCP startup when that allowlist cannot reach bundled MCP tools. Any remaining non-Active-Memory scheduled-run MCP/LSP allowlist work should stay tracked separately, such as the related #70971 surface noted in the provided context.
