## Summary

Keep #38709 open. Current main still has the earlier view-local WebChat input history implementation, but it does not contain the PR's state-layer hardening for loading/reload invalidation, bidirectional history-mode navigation, draft restoration, modifier/IME/selection guards, or caret/textarea restoration. The maintainer discussion also explicitly says this is not superseded by #71524 and should be reviewed as a standalone input-history hardening change after rebase and green checks.

## Best Possible Solution

Keep #38709 open as the standalone WebChat input-history hardening track. The best path is to rebase it onto current main, drop unrelated churn, port the state-layer key handling/reset/loading/caret behavior into the current UI structure, and get the UI/check lanes green; if needed, split the remaining work into narrower PRs linked to #38702.
