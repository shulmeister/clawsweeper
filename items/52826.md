## Summary

Keep open. Current `main` does not implement a session-level `process.*` / `step.*` event family, and the existing Gateway/session streams are not reconnect-safe source-of-truth process updates with stable step identity.

## Evidence

- **Gateway event catalog has no process event family:** `GATEWAY_EVENTS` only lists `agent`, `chat`, `session.message`, `session.tool`, `sessions.changed`, and other unrelated families; there is no `process.*` or `step.*` event surface.
  - file: [src/gateway/server-methods-list.ts:145](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/gateway/server-methods-list.ts#L145)
  - command: `nl -ba src/gateway/server-methods-list.ts | sed -n '140,156p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Broadcast allowlist only exposes session.message/session.tool/sessions.changed:** The session-related broadcast guard entries are `sessions.changed`, `session.message`, and `session.tool` only. No process-step broadcast channel exists on current `main`.
  - file: [src/gateway/server-broadcast.ts:22](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/gateway/server-broadcast.ts#L22)
  - command: `nl -ba src/gateway/server-broadcast.ts | sed -n '1,60p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Agent lifecycle stream is still coarse start/end/error only:** The embedded agent lifecycle emitter sends `stream: "lifecycle"` with `phase: "start"`, `phase: "error"`, or `phase: "end"`; there are no structured process or step lifecycle payloads with stable step ids.
  - file: [src/agents/pi-embedded-subscribe.handlers.lifecycle.ts:23](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/pi-embedded-subscribe.handlers.lifecycle.ts#L23)
  - command: `nl -ba src/agents/pi-embedded-subscribe.handlers.lifecycle.ts | sed -n '18,150p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Docs describe only assistant/tool/lifecycle streams:** The current agent-loop documentation says `subscribeEmbeddedPiSession` bridges only `tool`, `assistant`, and `lifecycle` streams, and defines lifecycle as `start | end | error`.
  - file: [docs/concepts/agent-loop.md:38](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/concepts/agent-loop.md#L38)
  - command: `nl -ba docs/concepts/agent-loop.md | sed -n '30,60p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Session broadcasts are transcript/metadata updates, not stable process state:** `createTranscriptUpdateBroadcastHandler` emits `session.message` and `sessions.changed`; `createLifecycleEventBroadcastHandler` emits `sessions.changed`. These payloads carry transcript/session metadata and message sequence, not stable `stepId` / `step title` / process-state records.
  - file: [src/gateway/server-session-events.ts:83](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/gateway/server-session-events.ts#L83)

- **Current protocol does not meet reconnect-safe sequence/replay requirement:** Protocol docs state per-client socket sequence numbers are connection-local, and gateway docs say events are not replayed on gaps. That does not satisfy the issue's request for a session-level source of truth that survives reconnect/reload.
  - file: [docs/gateway/protocol.md:259](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/gateway/protocol.md#L259)
