## Summary

Keep #60864 open. Current main has meaningful restart-continuation work: a restart sentinel can carry a one-shot continuation, startup moves it into a durable session-delivery queue, and queued continuation work is retried. That addresses part of the RFC and the related merged PR #63406/#70780 discussion positively. It does not yet implement the requested first-class task checkpoint model: there is no `paused_for_restart`/`resuming`/`resumed`/`failed_resume` task state, and the restart sentinel captures route/message-level continuation data rather than user goal, plan cursor, intermediate artifacts, or a new checkpointed run seeded from that state.

It does not include userGoal, plan progress, artifacts, checkpoint status, or original/resumed run linkage requested by the RFC.
