## Summary

Keep open. The original auto-topic-label feature is present on current main, but the reported regression is not proven fixed: current dispatch still skips the rename whenever the DM topic session-store entry already has systemSent set, which is exactly the suspected failure mode in #62303.

## Evidence

- **Current first-turn gate still depends on systemSent:** DM-topic dispatch loads the session store and sets isFirstTurnInSession from !entry?.systemSent. A newly-created topic session that already has systemSent true will not enter the auto-label path.
