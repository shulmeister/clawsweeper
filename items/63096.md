## Summary

Keep this PR open. Current main still forwards gateway `agent` requests with `extraSystemPrompt: request.extraSystemPrompt`, so inter-session `task_completion` wakes that omit it still do not reconstruct the session-derived Inbound Context / Group Chat Context that normal chat turns build. The PR remains a plausible implementation candidate for the open cache-stability bug tracked by #63030.

## Best Possible Solution

Keep this PR open as the implementation candidate for #63030. The best path is to rebase or refresh it onto current main, retain the regression tests for synthesized persisted session context and explicit prompt pass-through, and have gateway maintainers review prompt parity with the normal chat path before merging.
