## Summary

Keep this PR open. Current main still lacks the central optimization: normal non-live models.json provider discovery returns an unscoped filter and does not use explicit configured providers to narrow plugin discovery. The required ownership and onlyPluginIds seams exist, and the PR discussion shows earlier review blockers were addressed rather than proving the work is obsolete. Related #72352 overlaps, but it is also open and broader, so it is not a high-confidence superseding close reason by itself.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either land or adapt its narrow implementation: pass explicit providers into the discovery-filter path, resolve their owning plugin ids with the existing ownership helper, fall back to broad discovery when any explicit provider has no owner, preserve live all and mixed live-filter semantics, and keep focused regression tests. If maintainers prefer #72352 as the canonical broader implementation, close one PR only after explicitly selecting that path or after the chosen fix merges.
