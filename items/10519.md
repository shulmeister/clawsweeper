## Summary

Keep #10519 open. Current main still renders Telegram fenced code blocks as `<pre><code>` without preserving the fence language, and existing Telegram tests assert that missing behavior. The related WebChat copy-button work is a different UI surface and does not satisfy Telegram Bot API HTML formatting support for `<pre><code class="language-*">`.

## Best Possible Solution

Keep the issue open and implement Telegram fenced-code language preservation in the Markdown IR/render path or a Telegram-specific fence renderer, then update Telegram formatting tests to expect `class="language-js"` while preserving HTML escaping, chunking, file-reference wrapping, and plain-text fallback behavior.
