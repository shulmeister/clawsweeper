## Summary

Keep #10519 open. Current main still renders Telegram fenced code blocks as `<pre><code>` without preserving the fence language, and the Telegram format test still asserts that missing behavior. The related WebChat copy-button work is a separate UI renderer and does not change Telegram Bot API HTML output.

## Best Possible Solution

Keep the issue open and implement Telegram fenced-code language preservation in the Markdown IR/render path or a Telegram-specific fence renderer. Update Telegram format and chunking tests to expect `class="language-js"` while preserving HTML escaping, file-reference wrapping, chunking, and plain-text fallback behavior.
