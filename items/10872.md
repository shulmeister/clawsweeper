## Summary

Keep open. Current main still only includes iMessage reply context when resolved `reply_to_*` fields are already present; it does not model or resolve the `thread_originator_guid` signal that the issue discussion identifies as the remaining imsg v0.5.0 path.

## Best Possible Solution

Keep the issue open until the bundled legacy `extensions/imessage` monitor either resolves `thread_originator_guid` into `ReplyToId`/`ReplyToBody`/`ReplyToSender` using a documented imsg RPC or carefully-scoped chat.db lookup, or a maintainer explicitly decides to close the imsg-specific gap in favor of BlueBubbles. The implementation should model the GUID field, add parser/inbound-processing regression tests, and preserve existing quote visibility filtering.
