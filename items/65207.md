## Summary

Current `main` still appears to match the report: explicit Ollama models are resolved correctly by the Ollama plugin, but default `openclaw models list --provider ollama` only enumerates agent-default/allowlist entries and does not include `models.providers.ollama.models` unless `--all` is used. The issue remains plausible and should stay open.

## Evidence

- **Ollama plugin accepts explicit provider models:** `resolveOllamaDiscoveryResult()` returns the explicit `models.providers.ollama` config directly when `models` are present, normalizing `baseUrl`, defaulting `api`, and preserving/synthesizing auth.
  - file: [extensions/ollama/src/discovery-shared.ts:102](https://github.com/openclaw/openclaw/blob/a5db42862d76fce18e5d7fa57f3907a543e61137/extensions/ollama/src/discovery-shared.ts#L102)

  - sha: [a5db42862d76](https://github.com/openclaw/openclaw/commit/a5db42862d76fce18e5d7fa57f3907a543e61137)
- **Regression test covers explicit Ollama config path:** `provider-discovery.test.ts` asserts that when explicit Ollama models are configured, no `/api/tags` or `/api/show` discovery fetch happens and the configured model list is returned as-is.
  - file: [extensions/ollama/provider-discovery.test.ts:351](https://github.com/openclaw/openclaw/blob/a5db42862d76fce18e5d7fa57f3907a543e61137/extensions/ollama/provider-discovery.test.ts#L351)

  - file: [src/commands/models/list.list-command.ts:80](https://github.com/openclaw/openclaw/blob/a5db42862d76fce18e5d7fa57f3907a543e61137/src/commands/models/list.list-command.ts#L80)
