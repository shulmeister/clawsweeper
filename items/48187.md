## Summary

Keep PR #48187 open. Current main has Telegram topic routing and binding primitives, but `/status` still does not receive Telegram message context or render topic/binding/debug lines, so the PR is not obsolete or implemented on main.

## Best Possible Solution

Keep the PR open and review the feature slice. The best direction is to expose a generic channel-owned status hook or status-line extension point, then implement Telegram topic routing/binding status in the Telegram plugin using existing conversation and session-binding APIs. Preserve the PR's useful tests for configured binding, live binding, drift, ACP metadata, and non-topic no-op behavior.
