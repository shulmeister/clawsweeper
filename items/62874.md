## Summary

Current main still matches the reported failure mode closely enough that this should stay open: local vLLM/OpenAI-compatible completions with tools still default to `tool_choice: "auto"`, and only an explicit option or `extra_body` override can force `required`. The current vLLM docs also support treating `required` as the stronger compatibility path when schema/tool-call extraction matters. ([docs.vllm.ai](https://docs.vllm.ai/usage/tool_calling/))

## Best Possible Solution

Keep this open for maintainer decision. The likely path is a vLLM/Qwen-scoped compatibility option, per-model `params.toolChoice` support, or explicit vLLM docs showing `agents.defaults.models["vllm/<model>"].params.extra_body: { tool_choice: "required" }`, with regression coverage proving the override reaches the local vLLM OpenAI-compatible payload without changing native OpenAI behavior globally.
