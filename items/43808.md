## Summary

Keep open. Current main already carries `delivery.threadId` through the core cron announce path, but this PR still contains meaningful remaining work: gateway cron add/update/list schema round-trip support for `delivery.threadId`, plus failure-destination deduplication that distinguishes same-chat/different-thread Telegram targets.

## Best Possible Solution

Keep the PR open, or have a maintainer extract the still-relevant pieces onto current main: add `delivery.threadId` to the gateway cron protocol schema and regression tests, and update failure-destination deduplication so explicit primary thread context does not collapse distinct failure routes.
