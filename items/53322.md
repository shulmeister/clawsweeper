## Summary

Current main already includes generic tool-finalization and tool-start ordering safeguards that match this browser-tool regression, and those same safeguards are present in the released `v2026.4.22` code.

## Close Comment

Closing from a Codex review because current `main` already has the turn/tool-ordering safeguards this report asks for. `src/agents/pi-embedded-runner/run/attempt.ts:2901` and `src/agents/pi-embedded-runner/wait-for-idle-before-flush.ts:12` now wait for the agent to become idle before flushing pending tool results, which prevents the runner from finalizing while tool execution is still in flight. `src/agents/pi-embedded-subscribe.handlers.tools.ts:696` also flushes pending block replies before `tool_execution_start`, so assistant-visible text does not get emitted ahead of tool start.
