## Summary

Keep PR #72023 open. Current main still performs ACP startup identity reconciliation for persisted pending identities with stable runtime ids without excluding `acp.mode === "oneshot"`, so the warning path reported in #72013 is still present and the PR's focused one-shot guard remains useful.

## Best Possible Solution

Keep #72023 open and land it, or land an equivalent maintainer patch, that skips `session.acp.mode === "oneshot"` before startup pending-identity reconciliation and includes the focused regression test. Do not close it as implemented unless main gains that explicit startup skip or another source-level change demonstrably suppresses historical pending one-shot records from the warning path.
