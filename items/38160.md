## Summary

Keep open. Current main covers part of the local-fallback ask through the normal model fallback chain, but it does not implement this PR's dedicated `localModelFallback` subsystem or semantic response cache. The PR also has unresolved correctness review comments, so there is no conservative auto-close reason.

## Best Possible Solution

Keep the PR open for maintainer product/architecture review. The best path is to split it: use or document existing `agents.defaults.model.fallbacks` for hosted-to-local fallback unless a concrete missing API is identified, and redesign any semantic response cache against the current memory/embedding architecture with fail-open cache errors, correct vector persistence/decoding, telemetry-preserving fallback attempts, docs, and focused tests.
