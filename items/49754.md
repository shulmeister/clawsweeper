On `main` at `320d52a23e41a326287826219a20f1f29f334554`, a repo-wide grep finds no `qclaw`, `qclaw-cos`, or `/proxy/qclaw-cos` references, and the Gateway HTTP stack routes extension-owned HTTP behavior through plugin registries (`src/gateway/server/plugins-http.ts`) while the project vision says optional capability should usually ship as plugins and new extension work should go through the plugin/ClawHub path (`VISION.md:54-70`, `docs/tools/plugin.md`).
