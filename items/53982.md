## Summary

Keep this issue open. Current main still documents and displays a Nextcloud Talk bot install command without `--feature response`, and outbound 401 send failures still only tell users to check the bot secret. Upstream Nextcloud Talk docs confirm `response` is the feature that lets bots post messages/reactions. citeturn1view0turn1view1

## Best Possible Solution

Apply a narrow Nextcloud Talk plugin/docs fix: update the English setup command and setup wizard intro to include `--feature webhook --feature response --feature reaction`, add a short troubleshooting note for HTTP 401 replies when the bot was registered without `response`, and improve the send-path 401 error to mention re-registering with `--feature response` while preserving the bot-secret mismatch possibility. Add focused coverage for the setup text or send-error behavior.
