## Summary

Keep open. Current `main` still hard-codes MiniMax music support to `music-2.5+`, `music-2.5`, and `music-2.0`, and the public MiniMax docs in-repo match that older catalog. The issue is therefore not implemented on `main`. However, the current `music_generate` path does forward a `model` override through the tool/runtime/provider stack, so the missing `music-2.6*` support is substantiated while the claim that OpenClaw always ignores `model` is not supported by the current code/tests.
