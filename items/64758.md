## Summary

Keep open. Current main still only attempts steer injection while the active run is streaming; an active but non-streaming run falls back into the followup queue, which is the behavior this PR is trying to change. The PR remains a plausible implementation candidate for the open soft-steer feature request, but it should not be closed as obsolete or implemented on main, and it needs the unresolved heartbeat-regression review addressed before landing.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to revise it so non-streaming active `steer` user messages abort/unblock the current run without aborting heartbeat turns that queue policy would drop, preserve the streaming fast-path, add regression coverage for both non-streaming steer fallback and heartbeat non-abort behavior, address the minor review cleanup around the steer-backlog test name and unused `isStreaming` contract, then rebase and run the targeted changed tests.
