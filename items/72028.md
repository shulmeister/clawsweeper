## Summary

Keep open. Current main still duplicates the subagent task in both the subagent system prompt and the first child user message, so the underlying #72019 behavior is not implemented on main. PR #72028 is an active fix attempt for a real remaining bug, but its "first user message" approach needs maintainer review because forked subagent sessions can inherit earlier parent transcript messages before the new task turn. Related PR #72053 appears to be a stronger competing fix, but it is still open and unmerged, so this should be consolidated by maintainer review rather than automatic cleanup close.

## Best Possible Solution

Keep this open for maintainer consolidation. The best product/code path is to land one implementation that stores the authoritative subagent task exactly once in a non-ambiguous place, adds regression coverage for normal and `context="fork"` spawns plus multiline tasks, then close #72019 and any duplicate PRs in favor of the landed fix.
