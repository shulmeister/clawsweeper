## Summary

Keep open. Current main still has the Agents/Control UI model dropdown path rendering configured models from aliases/raw refs and catalog models from raw id/provider labels, rather than the catalog display name. That leaves the reported central problem unresolved for Claude Sonnet 4.6, and Opus 4.7 still needs either catalog metadata or a UI fallback that can render the intended friendly name when the catalog row is synthetic or missing upstream.

## Best Possible Solution

Update the Agents/Control UI model dropdown to share the catalog display-name logic already used by the chat selector, while preserving dedupe and configured aliases. Add focused UI/helper regression coverage for `anthropic/claude-sonnet-4-6` and `anthropic/claude-opus-4-7`, including the case where Opus 4.7 is synthetic or missing from the upstream pi-ai catalog. If Opus 4.7 should always display as `Claude Opus 4.7`, add provider/catalog metadata in the appropriate model catalog layer rather than relying on the short alias.
