## Summary

Close. Current OpenClaw no longer has the reported stale `.jsonl.lock` -> silent empty `sessions_history` path: `sessions_history` delegates to `chat.history`, `chat.history` reads transcript JSONL directly rather than waiting on the lock file, stale lock reclamation is implemented for write contention, and stale lock cleanup is run at gateway startup. The released `v2026.4.23` tag already contains the relevant history-read and lock-cleanup code.
