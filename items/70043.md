## Summary

Keep open. Current `main` still derives Telegram reaction guidance from an unset `channels.telegram.reactionLevel` as `minimal`, wires that into the agent prompt for Telegram sessions, and injects the exact `MINIMAL` wording into the live system prompt. The latest release `v2026.4.23` shipped the same behavior, so there is no evidence the reported silent effective switch has been fixed on `main`.

## Evidence

- **Prompt guidance is resolved on each agent attempt from the active runtime channel:** `resolveChannelReactionGuidance(...)` is called during prompt assembly for each run when a runtime channel and config are present, so prompt text can change from effective config/runtime behavior without any explicit persisted session setting.
