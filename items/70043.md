## Summary

Current main still matches the reported behavior: unset Telegram `reactionLevel` resolves to `minimal`, the Telegram plugin forwards that as prompt reaction guidance, and the agent system prompt emits the exact `Reactions are enabled for Telegram in MINIMAL mode.` text. The provided related items are loose Telegram/session overlaps, not a canonical fix for this defaulting and prompt-stability bug.

## Best Possible Solution

Keep this issue open. Maintainers should decide the product contract for unset Telegram reaction levels in existing sessions, then update the Telegram plugin resolver or prompt adapter so prompt-affecting reaction guidance is stable and intentional. Likely fixes are to inject guidance only when `channels.telegram.reactionLevel` is explicit, add a migration/doctor notice for existing configs, or otherwise align implementation, schema comments, docs, and tests around the chosen default.
