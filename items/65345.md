## Summary

Keep open. Current main still contains confirmed docs/code drift from this audit: `--strict-json` is documented as JSON5 but implemented and tested as strict JSON, `/tools/invoke` can return an undocumented 403 via `before_tool_call`, `$include` array docs say later wins while arrays concatenate, and OpenResponses `previous_response_id` reuse is documented as user-scoped while tests show reuse across different `user` values.

## Best Possible Solution

Keep this issue open and split or resolve the confirmed docs/code drift by area. The lowest-risk path is docs-first for behavior that appears intentional and tested: update `docs.openclaw.ai/cli/config` to say `--strict-json` means strict JSON, document `/tools/invoke` 403 plus `before_tool_call` rewrite/block behavior, clarify `$include` array concatenation and security checks, and align OpenResponses docs with the auth-subject/agent/requested-session reuse scope unless maintainers decide the implementation should instead change.
