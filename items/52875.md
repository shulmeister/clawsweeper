## Summary

Keep open. Current `main` still does not implement the reported fix path: `sessions_send` still fails with `No session found` when label resolution misses, there is no agent-visible `sessions_start`/`sessions.start` tool on `main`, and the gateway still classifies missing sessions as `INVALID_REQUEST` rather than a dedicated `SESSION_NOT_FOUND` code.

## Evidence

- **`sessions_send` still returns missing-session errors instead of auto-starting:** `createSessionsSendTool` resolves `label` through `sessions.resolve` and returns `status: "error"` with `No session found with label: ...` when resolution fails. There is no fallback that creates or bootstraps a session.
  - file: [src/agents/tools/sessions-send-tool.ts:117](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/tools/sessions-send-tool.ts#L117)

  - file: [src/agents/tools/sessions-send-tool.ts:338](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/tools/sessions-send-tool.ts#L338)
