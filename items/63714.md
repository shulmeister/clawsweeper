## Summary

This unmerged PR adds a large optional Slack/Telegram quick-agent work-intent orchestration layer. Current main does not ship the PR's task-os core path, but it already exposes the generic plugin surfaces needed to build this as a ClawHub plugin/skill: synthetic reply hooks, message hooks, gateway-bound subagent runtime, and Task Flow. The PR also has unresolved dispatcher lifecycle review blockers, so the conservative maintainer cleanup is to close it as ClawHub scope rather than merge it into core.

## Best Possible Solution

Close this core PR and move the idea to a ClawHub plugin/skill. Build it with before_agent_reply or appropriate inbound/message hooks for work-intent detection and acknowledgement, api.runtime.taskFlow for durable stage tracking, and api.runtime.subagent for research/spec/builder/verifier runs. If that external implementation proves a generic OpenClaw plugin API is missing, open a narrow core PR for that seam.
