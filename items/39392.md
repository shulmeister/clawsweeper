## Summary

Current main still does not contain the focused Control UI fix for the reported stale/wrong Agents Overview primary model selection after switching agents. The issue discussion and related PRs identify the right surface, but the current source still renders the overview/model select without a selected-agent remount boundary or the later selected-value stabilization described in PR #70633.

## Best Possible Solution

Keep #39392 open until a focused Control UI fix lands. The likely fix belongs in the Agents Overview render path: key/remount the overview by selectedAgent.id or make the select/options share one explicit selected value, then add a regression test that switches main -> another agent -> main and verifies the displayed Primary model matches the selected agent's resolved config. PR #39401 remains the closest open candidate; PR #70633's selectedPrimary approach is useful context but is not present on current main.
