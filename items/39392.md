## Summary

Keep #39392 open. Current main still does not contain the narrow UI isolation fix described by the issue and the two linked PRs: the Agents overview is rendered directly, agent switching resets files/skills/tools state but not the shared config form state, and there is no regression coverage for the main -> another agent -> main model selector path.

## Best Possible Solution

Keep the issue open and choose one canonical fix path: land or reimplement the focused Control UI change that isolates the Agents overview model controls per selected agent, preferably with a regression test for switching main -> another agent -> main and verifying the displayed model value. The existing PRs #39401 and #39495 are useful starting points but should be reconciled against current main before landing.
