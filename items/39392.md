## Summary

Current main still lacks the focused Control UI isolation fix for #39392. The issue discussion and related PRs identify a plausible stale Agents Overview model-control problem, and the current source still renders the overview/model select without a selected-agent remount boundary or agent-switch reset of the shared config/model form state.

## Best Possible Solution

Keep #39392 open until a maintainer lands a focused Control UI fix, likely by updating/landing #39401, choosing the broader #54724 path, or applying an equivalent fresh patch. The fix should isolate/remount or explicitly synchronize the Agents Overview model/fallback controls by selected agent and add a regression test that switches main -> another agent -> main and verifies the displayed Primary model matches the selected agent's resolved config.
