## Summary

Current `main` supports the underlying capability through `agents.list[].params` and validates/forwards `temperature` and `maxTokens`, but the requested `agents.list[].model.{temperature,maxTokens}` schema is still not implemented. `model` remains strict `{ primary, fallbacks }`, so closing as implemented would be too aggressive for conservative cleanup.

## Evidence

- **`model` schema still only allows primary/fallbacks:** `AgentModelConfig` is still `string | { primary?: string; fallbacks?: string[] }`, and the zod schema is strict with only those keys.
  - file: [src/config/types.agents-shared.ts:8](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/config/types.agents-shared.ts#L8)
  - command: `nl -ba src/config/types.agents-shared.ts | sed -n '1,20p'; nl -ba src/config/zod-schema.agent-model.ts | sed -n '1,15p'`

  - file: [docs/gateway/configuration-reference.md:1815](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/docs/gateway/configuration-reference.md#L1815)
