## Summary

Current main still does not implement optional human-readable `name`/`label` metadata on group config entries. Account-level `name` fields exist, but Telegram, WhatsApp, Signal, IRC, BlueBubbles, Feishu, LINE, Mattermost, and other group-entry schemas remain strict and omit `name`, `label`, or `description`; generated config metadata also still rejects extra Telegram group fields.

## Best Possible Solution

Keep the issue open until OpenClaw chooses and implements a canonical optional metadata field, likely `name`, on group-entry config types and Zod schemas for the bundled/core channel surfaces and relevant plugin schemas, regenerates config metadata, updates docs, and adds focused schema tests proving the metadata is accepted and ignored functionally.
