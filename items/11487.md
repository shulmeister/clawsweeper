## Summary

Keep #11487 open. Current main still does not implement optional human-readable `name`/`label`/`description` metadata for group config entries. Account-level `name` fields exist, but strict group-entry schemas and generated config metadata for core and bundled plugin channels still omit the requested field and reject extra properties. The existing maintainer-side Codex comment also explicitly kept this item open for the same reason, and the current checkout recheck confirms that remains true at `04e08cea62983b2c766d15f61f15b3ffc90f9baa`.

## Best Possible Solution

Keep the issue open until OpenClaw chooses a canonical optional metadata field, likely `name`, and adds it to group-entry config types and Zod schemas for core/bundled channel surfaces and relevant plugin schemas. Regenerate config metadata, update the public group/channel docs, and add focused schema tests proving the metadata is accepted and ignored functionally.
