## Summary

Current `main` does not implement optional `name`/`label` metadata for group config entries. Multiple channel schemas still use strict group-entry objects without such a field, and docs/examples still show unlabeled ID-keyed group maps.

## Evidence

- **Telegram group entries remain strict and unlabeled:** `TelegramGroupSchema` includes fields like `requireMention`, `allowFrom`, `systemPrompt`, and `topics`, then calls `.strict()`. The enclosing account schema still maps `groups` to `TelegramGroupSchema`, so extra metadata keys like `name`/`label` are rejected.
