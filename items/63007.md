## Summary

Keep this PR open. Current main still builds outbound session identity for gateway send, but it does not pass that identity into the `message_sending` hook context, the public message hook context still has no `agentId`, and guarded hook cancellation still collapses to the generic `No delivery result` gateway error.

## Best Possible Solution

Keep this PR open, or replace it with an equivalent focused maintainer patch: add optional `agentId` to the public message hook context, pass outbound `session.agentId` and `session.key` into `message_sending`, preserve cancellation reason through a narrow delivery callback/result, surface an explicit guarded gateway-send denial, and cover the gateway/outbound path plus docs.
