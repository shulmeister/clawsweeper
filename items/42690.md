## Summary

Keep open. Current `main` still does not implement the reported `session_status` behavior: the status card only exposes a second model line as `↪️ Fallback` when persisted fallback state matches, and the `session_status` path does not read `model-snapshot` custom transcript entries to show an in-run routed active model.

## Evidence

- **Status card compares selected vs active, but only renders a fallback line:** `buildStatusMessage` resolves selected and active model refs, then later emits only `↪️ Fallback: ...` gated by `fallbackState.active`; there is no generic `Active:` line for non-fallback runtime routing.
  - file: [src/status/status-message.ts:484](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/status/status-message.ts#L484)
  - command: `nl -ba src/status/status-message.ts | sed -n '484,520p' ; nl -ba src/status/status-message.ts | sed -n '861,870p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Second line requires persisted fallback notice state:** `resolveActiveFallbackState` returns active only when `fallbackNoticeSelectedModel` and `fallbackNoticeActiveModel` already match the selected and active refs. If runtime routing changes the active model without that persisted fallback state, the card will not show the second line.
  - file: [src/status/fallback-notice-state.ts:9](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/status/fallback-notice-state.ts#L9)
  - command: `nl -ba src/status/fallback-notice-state.ts | sed -n '1,40p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Current session_status tests still collapse runtime model into the primary card model:** The test suite explicitly expects `session_status` to use the runtime session model as the card's primary model when no override is set, which is the opposite of the issue's requested selected-vs-active distinction.
  - file: [src/agents/openclaw-tools.session-status.test.ts:796](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/agents/openclaw-tools.session-status.test.ts#L796)
  - command: `nl -ba src/agents/openclaw-tools.session-status.test.ts | sed -n '796,840p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Model-snapshot exists, but session_status path does not consume it:** Replay history defines a separate `model-snapshot` custom entry type. A search of `src/agents/tools` and `src/status` found no `model-snapshot` handling; the status path's transcript backfill is usage-oriented, not custom-entry-oriented, so the issue's suggested source is not implemented on current `main`.
  - file: [src/agents/pi-embedded-runner/replay-history.ts:46](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/agents/pi-embedded-runner/replay-history.ts#L46)
