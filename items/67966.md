## Summary

Keep #67966 open. Current main still keeps local-managed browser profiles on the raw CDP tab-open path, while only remote CDP profiles use persistent Playwright for tab creation and redirect-hop inspection. The related sandbox SSRF propagation bugs are fixed, but this specific local-managed Playwright interception request remains unimplemented.

## Best Possible Solution

Keep the issue open and implement a focused browser-plugin change: make local-managed openclaw profiles use the existing Playwright-backed createPageViaPlaywright/gotoPageWithNavigationGuard path for navigations that need redirect-hop inspection, while preserving Chrome MCP existing-session behavior and raw-CDP fallbacks where Playwright is unavailable. Add regression tests for local-managed strict SSRF mode, including hostname and IP-literal URLs, and update browser docs if the profile capability matrix changes.
