## Summary

Current main still has the requested gap. Media-understanding outputs and decisions carry provider/model telemetry, and status output can show it, but the prompt-visible inbound media body generated for BodyForAgent still renders only the media header, user text, and transcript/description. The reporter's validated local workaround remains aligned with the missing upstream formatter change.

## Best Possible Solution

Keep this issue open. Implement a narrow formatter change in src/media-understanding/format.ts that renders current-turn analysis provider/model for each successful media output when known, without guessing when unavailable. Add focused coverage in src/media-understanding/format.test.ts and an apply-path regression in src/media-understanding/apply.test.ts proving BodyForAgent receives the telemetry.
