## Summary

Keep open. Current `main` still contains the reported raw-mode regression paths: snapshot redaction compares restored raw content against the materialized runtime config, `normalizeExecSafeBinProfilesInConfig()` still writes explicit `undefined` properties, and raw-string redaction still does global value replacement that can corrupt key names such as `profiles` when a secret value is `file`.

## Evidence

- **Runtime snapshot still materializes exec safe-bin fields to explicit undefined:** `normalizeExecSafeBinProfilesInConfig()` still assigns `undefined` instead of deleting empty `safeBinProfiles`/`safeBinTrustedDirs`, so the runtime object can retain keys that disappear after JSON round-trip.
  - file: [src/config/normalize-exec-safe-bin.ts:25](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/config/normalize-exec-safe-bin.ts#L25)

That matches the issue's claim that materialized defaults/normalization can permanently disable raw mode.
