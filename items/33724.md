## Summary

Keep open. Current main still hard-codes OpenClaw config backups to `${configPath}.bak` plus four numbered backups, runs that maintenance from config write paths, and has no typed config/schema/UI option for backup directory, retention count, or trigger policy. The related implementation PRs #33896 and #34478 were closed unmerged in the provided GitHub context, and the existing `openclaw backup` archive command is a separate manual archive workflow, not a way to reconfigure built-in `.bak` files.

## Best Possible Solution

Keep #33724 open until a maintainer either implements configurable built-in config backup behavior or explicitly rejects the product direction. A complete implementation likely belongs in typed config/schema/help/docs/UI wiring plus plumbing through `src/config/backup-rotation.ts`, `src/config/io.ts`, `src/config/mutate.ts`, `src/config/io.observe-recovery.ts`, and audit/recovery code that currently assumes `${configPath}.bak`, with tests for default compatibility, custom directory, custom retention, and any supported trigger policy.
