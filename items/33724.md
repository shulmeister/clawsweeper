## Summary

Keep open. Current main still hard-codes config backup rotation to `${configPath}.bak` plus four numbered backups, triggers that maintenance from both config write paths, and has no `OpenClawConfig` or schema/UI surface for backup path, retention count, or trigger policy. The related implementation PRs #33896 and #34478 were closed unmerged per the provided GitHub context, and the cron/archive backup alternatives do not reconfigure OpenClaw's own config-write `.bak` files.

## Best Possible Solution

Keep #33724 open until a maintainer either implements configurable built-in config backup behavior or explicitly rejects the product direction. A complete path likely needs typed config/schema/help/docs/UI wiring for backup path, retention count, and trigger policy, plumbing through `src/config/backup-rotation.ts`, `src/config/io.ts`, `src/config/mutate.ts`, and recovery/audit code that currently assumes `${configPath}.bak`, plus tests covering default compatibility and custom directory/retention behavior. Related PRs #33896 and #34478 are useful prior art but were closed unmerged.
