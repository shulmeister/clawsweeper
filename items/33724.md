## Summary

Keep #33724 open. Current main still hard-codes automatic config-write backups to an adjacent five-file `.bak` ring, with no typed config/schema/UI surface for backup directory, retention count, or trigger policy. The related implementation PRs #33896 and #34478 were closed unmerged, and `openclaw backup` is a separate archive command rather than configuration for the automatic `.json.bak` rotation.

## Best Possible Solution

Keep #33724 open until maintainers either implement configurable built-in config backup behavior or explicitly reject that product direction. A complete implementation should add a typed config/schema/help/docs/UI surface for backup directory, retention, and trigger policy; thread that policy through `src/config/backup-rotation.ts`, `src/config/io.ts`, `src/config/mutate.ts`, and `src/config/io.observe-recovery.ts`; preserve current defaults; and update rotation/recovery/audit tests. The cron workaround and `openclaw backup` archive command can remain documented alternatives but do not replace configurable automatic config-write rotation.
