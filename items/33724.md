## Summary

Keep open. Current main still hard-codes the built-in config backup location, five-file retention ring, and write-triggered behavior. The related implementation PRs were closed unmerged, and the discussed cron/backup alternatives do not reconfigure or suppress OpenClaw’s own `.bak` files.

## Best Possible Solution

Keep #33724 open until a maintainer either implements configurable built-in config backup behavior or explicitly rejects the product direction. A complete implementation likely needs a typed config surface for backup path/count/trigger policy, schema/help/docs/UI wiring, option plumbing through `src/config/io.ts`, `src/config/mutate.ts`, and recovery code that reads `${configPath}.bak`, plus tests covering default compatibility and custom directory/retention behavior.
