## Summary

Keep #46590 open. Current main still does not implement declarative `agents.list[].cron`: the agent config type and strict agent schema have no cron field, the gateway cron service is still initialized from the central cron store, and the user-facing docs still describe jobs as persisted in `~/.openclaw/cron/jobs.json` and managed with `openclaw cron ... --agent`. The related items are adjacent cron/agent UX requests, not a canonical duplicate or merged implementation. This also should not be closed as ClawHub-only because the requested behavior is a core config schema and scheduler startup contract, not just an optional external plugin/package.
