## Summary

Keep #72153 open. Current main still constructs the Windows `config.openFile` command with `Start-Process -LiteralPath`, and the existing regression test still locks in that unsupported form. Local PowerShell parameter introspection and a binding check confirm `Start-Process` supports `-FilePath` but not `-LiteralPath`, matching the linked #69933 reproduction and both bot/QC review comments.

## Best Possible Solution

Keep the PR open and merge or manually apply its minimal production/test change: replace `Start-Process -LiteralPath` with `Start-Process -FilePath` in `src/gateway/server-methods/config.ts`, update `src/gateway/server-methods/config.test.ts`, then run the changed gateway validation in an environment with `pnpm` available.
