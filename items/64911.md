## Summary

Current `main` contains a gateway-startup sequencing fix and regression coverage for the reported websocket handshake race after restart. The fix appears to be on `main` after the latest tagged release, so it is implemented but not yet tied to a shipped release tag.

## Evidence

- **WebSocket handlers attach before the port starts listening:** `startGatewayServer` now calls `attachGatewayWsHandlers(...)` before `await startListening()`, so clients cannot connect into an uninitialized websocket control plane.
  - file: [src/gateway/server.impl.ts:789](https://github.com/openclaw/openclaw/blob/c018e7347542aa963095de1ca704b943476856fe/src/gateway/server.impl.ts#L789)
  - command: `nl -ba src/gateway/server.impl.ts | sed -n '680,870p'`
  - sha: [c018e7347542](https://github.com/openclaw/openclaw/commit/c018e7347542aa963095de1ca704b943476856fe)
- **Startup waits for sidecars by default:** `deferStartupSidecars` is documented as defaulting to `false`, and `startGatewayPostAttachRuntime` awaits `sidecarsPromise` unless that explicit test-only override is set.
  - file: [src/gateway/server.impl.ts:232](https://github.com/openclaw/openclaw/blob/c018e7347542aa963095de1ca704b943476856fe/src/gateway/server.impl.ts#L232)
