## Summary

Keep #38076 open. Current main still validates `init_skill --resources` case-sensitively: allowed resource names are lowercase, `parse_resources()` validates stripped raw tokens before any normalization, and `main()` passes `args.resources` directly into that parser. A read-only parser invocation accepts lowercase input but rejects `Scripts,References,ASSETS`, and the related fix PRs in the supplied issue context are closed and unmerged.

## Best Possible Solution

Land the narrow skill-creator fix on main: normalize each nonempty `--resources` token to lowercase before validation and order-preserving deduplication, keep lowercase resource directory names and existing unknown-resource error behavior, and add focused `skills/skill-creator/scripts/test_init_skill.py` coverage for mixed-case acceptance, lowercase deduplication, and invalid-resource rejection.
