## Summary

Keep #38076 open. Current main still rejects mixed-case `init_skill --resources` values: resources are stripped but not lowercased before validation against a lowercase-only allowlist, and the CLI passes the raw argument into that parser. The related fix PRs in the provided context are closed and unmerged, so they do not supersede the issue.

## Best Possible Solution

Implement the narrow parser fix in `skills/skill-creator/scripts/init_skill.py`: lowercase each nonempty `--resources` token before validation and deduplication, while keeping the current unknown-resource error path. Add focused `skills/skill-creator/scripts/test_init_skill.py` coverage for mixed-case accepted values, lowercase deduped output, and unknown-resource rejection.
