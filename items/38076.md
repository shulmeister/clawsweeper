## Summary

Keep #38076 open. Current main still implements `init_skill --resources` as case-sensitive parsing: resource tokens are stripped but not lowercased before validation against a lowercase-only allowlist, so inputs such as `Scripts` or `ASSETS` still fail. The linked fix PRs in the supplied GitHub context are closed and unmerged, and current main has no `test_init_skill.py` regression coverage for this parser behavior.

## Best Possible Solution

Keep the issue open and implement the narrow parser fix in `skills/skill-creator/scripts/init_skill.py`: normalize each nonempty `--resources` token to lowercase before validation and order-preserving deduplication, preserving existing invalid-resource handling. Add a focused `skills/skill-creator/scripts/test_init_skill.py` regression test for mixed-case accepted values, lowercase output/deduplication, and unknown-resource rejection.
