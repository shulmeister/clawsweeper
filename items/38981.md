## Summary

Keep PR #38981 open. Current main still has no `workspaceConfig.allowedExternalPaths` config surface, still roots bootstrap/context loading and `agents.files.get/set` to the agent workspace, and current docs/tests continue to describe or enforce outside-workspace symlink rejection. The PR remains substantive unmerged work rather than obsolete cleanup.

## Best Possible Solution

Continue tracking this PR or a maintainer-owned successor until OpenClaw either implements an opt-in external symlink allowlist across config, bootstrap/context loading, gateway file reads/writes, docs, and tests, or explicitly decides that shared context should remain workspace-local only.
