## Summary

Current `main` has partial updater UX in Control UI: a generic config-backed update section, an `Update` button, and a top-level update-available banner. It does not yet implement the dedicated Update Manager requested here with richer operator-facing state such as last check/result history, dry-run controls, health-safe rollout controls, changelog/release-note surfacing, or post-update health summaries. The issue is recent and coherent, and the reporter's companion docs kit is a useful workaround rather than a core replacement.

## Evidence

There is no documented dedicated Control UI Update Manager surface here.
