## Summary

Keep this issue open. Current main has meaningful partial session/topic management in the Web Chat selector, Control UI Sessions table, TUI session picker, and Gateway session RPCs, but it still does not implement the full requested topic-management product surface: first-class Control UI create/import/export/archive-restore workflow, TUI create/rename/delete/archive actions, session-row message count, or description/tags/createdAt topic metadata contract.

## Best Possible Solution

Keep this issue open, but narrow the remaining work into concrete maintainer-owned pieces: add a first-class Control UI create/manage workflow around sessions.create, sessions.patch, and sessions.delete; decide whether description, tags, createdAt, and messageCount belong in the Gateway session-row contract; add TUI rename/delete/archive actions only if maintainers want management beyond the existing picker; and define deleted-archive restore plus import/export semantics with docs and focused tests.
