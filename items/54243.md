## Summary

Keep open. Current main still treats `--no-include-workspace` as a planner/discovery option, not an archive filter: it can remove separate workspace assets from the plan, but the state directory remains an included asset and is archived as a whole. The default workspace is under the state directory, CLI/docs copy still implies workspace contents are excluded or archive size is reduced, and related exclude-pattern work is still open rather than shipped.

## Best Possible Solution

Keep this issue open until backup either makes `--no-include-workspace` materially exclude nested workspace directories from the archived state tree, or deliberately changes the flag/docs/CLI copy to say it only skips separate workspace discovery and does not filter the state payload. If maintainers prefer the exclude-pattern route, review and land a narrowed version of #44288 or equivalent core work, with regression coverage proving a file under `.openclaw/workspace` is absent from the actual archive when workspace exclusion or an explicit exclude is requested.
