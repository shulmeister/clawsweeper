## Summary

Issue #32188 should stay open. Current main has related pieces, including TUI footer status, chat session/model controls, per-message metadata, context warnings, usage commands, overview auth cards, and macOS menu usage, but the requested persistent in-interface status area for active agent/model plus live token/quota/subscription/auth context is still not implemented in the main chat/application chrome.

## Best Possible Solution

Keep the enhancement open and implement a first-class persistent status surface in the chat/application chrome. It should show the active agent/session/model/runtime and live operation state, then add privacy-aware details for context usage, token/cache/cost data, provider quota windows, auth/subscription state, and relevant fallbacks using existing sessions, usage, model-auth, and status APIs.
