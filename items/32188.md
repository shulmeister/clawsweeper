## Summary

Issue #32188 remains a valid enhancement. Current main has model/session selectors, transient fallback and context indicators, per-message usage metadata, overview auth/usage cards, and documented slash-command/CLI usage surfaces, but it still lacks the requested persistent in-interface status area showing the active agent/model plus live token/quota/subscription context during operations.

## Best Possible Solution

Keep the enhancement open and implement a first-class persistent status surface in the chat/application chrome. It should show active agent/session/run status and selected provider/model, then add privacy-aware context usage, token/cache/cost, provider quota windows, auth/subscription state, and fallback information using existing sessions.list, usage.status, models.authStatus, model/auth, and session event data where possible.
