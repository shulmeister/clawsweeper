## Summary

Keep #32188 open. Current main has adjacent status, usage, and model-selection surfaces, and the TUI footer now partially shows agent/model/token context, but the requested persistent in-interface status area for active agent/model plus live token, quota, and subscription context is still not implemented in the Control UI/application chrome.

## Best Possible Solution

Keep the enhancement open. The best product path is a persistent Control UI status surface in chat/application chrome that shows active agent/session/run state and selected provider/model, then layers in privacy-aware context usage, token/cache/cost, provider quota windows, auth/subscription state, and fallback state using existing session, model auth, usage/cost, and run event data where available.
