## Summary

Issue #32188 should stay open. Current main has several related surfaces, including chat session/model controls, per-message usage metadata, high-context warnings, fallback/compaction indicators, overview auth usage cards, local /usage, and agent-routed /status reporting. Those are still partial and do not implement the requested persistent interface/status area showing the active agent/model plus token, quota, subscription, or auth context during operations.

## Best Possible Solution

Keep tracking this enhancement. The likely implementation should add a first-class status surface in the chat/application chrome that stays visible during normal operation and shows the active agent/session/model plus relevant live usage, context, quota/auth/subscription state, with privacy-aware detail disclosure. Existing session rows, usage.status/sessions.usage APIs, model selectors, /usage, /status, context notices, and overview auth cards can provide inputs, but they do not replace the requested persistent interface affordance.
