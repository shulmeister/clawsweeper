## Summary

Keep open. Current main does not yet implement the PR's requested cron timeout-cause surfacing: alerts still show only the raw last error, cron run logs and the gateway protocol do not expose errorReason for run entries, and CLI JSON output remains a pass-through. The PR discussion also contains useful review findings and follow-up cron reliability context, so this needs maintainer review rather than cleanup closure.
