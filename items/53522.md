## Summary

Current `main` already contains the `tools.web.fetch.maxResponseBytes` schema/help/type coverage and a dual-stack loopback heuristic, but it does not contain the PR's Synology webhook `HEAD`/`200 {"success":true}` behavior, and it also lacks the PR's `gatewayReachable` guard around the benign dual-stack diagnosis. This PR is only partially implemented on `main`, so conservative cleanup should keep it open.
