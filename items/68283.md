## Summary

Keep #68283 open. Current main still has the native approval handler bootstrap retry behavior this PR targets: failed registered-context startup is retried with a fixed 1,000 ms timer, existing pre-registered context startup still bypasses the guarded retry path, and there is no pairing-required special backoff or operator.approvals diagnostic in the bootstrap code. Related #70641/#70649 cover adjacent startup coordination and current main has since tightened chat approval auto-enable semantics, but that does not supersede this PR’s retry-after-failure backoff fix.

## Best Possible Solution

Keep #68283 open and either merge a rebased version of its scoped bootstrap retry fix or supersede it with a narrow PR that adds bounded retry state, a long pairing-required backoff with `operator.approvals` remediation logging, retry-state reset on success/context cleanup, and tests for registered and pre-existing runtime contexts.
