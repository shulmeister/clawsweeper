## Summary

Keep open. Current main still does not surface model-fallback attempts in `openclaw infer model run --json`, so the reporter's fallback-visibility expectation remains unmet. The OpenRouter auto fix path also needs a focused current-main regression test: OpenClaw docs/config still use `openrouter/auto`, OpenRouter's own docs also describe `openrouter/auto` as the Auto Router API model id, and related PR #69580 was closed unmerged rather than shipped. citeturn0search1

## Best Possible Solution

Keep this issue open. The best maintainer path is a provider-owned regression fix and test: verify current OpenRouter `openrouter/auto` and `openrouter/@preset/...` behavior against the real API, avoid applying the closed PR's bare-`auto` assumption unless live evidence proves it, and plumb `runWithModelFallback` attempt details into the `openclaw infer model run --json` envelope for both local and gateway transports.
