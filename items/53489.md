## Summary

Current `main` does not contain this PR's behavior change. Thread/session initialization still forks from the parent transcript when `ParentSessionKey` is present, and current tests/docs on `main` explicitly expect and document that inheritance, so this PR is not implemented on `main` and should remain open.
