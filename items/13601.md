## Summary

Keep open. Current main partially implements persistent JSONL-backed cron run history plus Gateway/UI filtering, but meaningful parts of #13601 remain missing: CLI status/time filters, time-range filtering in the protocol, success-rate/list statistics, and a manual history prune command. The related #16799/#16831 context is status UX work and does not supersede this item.
