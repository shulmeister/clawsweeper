## Summary

Keep #13601 open. Current main has persistent JSONL-backed cron run history plus Gateway/UI status, delivery, query, and sort filters, but the issue's core CLI workflow and several acceptance criteria remain incomplete: no `cron logs`/`cron history` command, no CLI status or time-range filters, no protocol/run-log time-window filter, no success-rate/list aggregates, and no manual prune/vacuum or retention-days policy.

## Best Possible Solution

Keep #13601 open until core cron history is completed or maintainers deliberately narrow the scope. The likely path is to extend `cron.runs` and `readCronRunLogEntriesPage` with time-window filters, add user-facing CLI flags or a `cron logs/history` command, add success-rate/list aggregation, and either implement manual history pruning/retention-days or document a deliberate config-only retention decision with tests and docs.
