## Summary

Keep #13601 open. Current main has useful partial cron run history via per-job JSONL files, `cron.runs`, and UI/RPC filters for status, delivery, query, and sort order, but the requested CLI log/history surface, time-window filters, success-rate/list aggregates, and manual retention/prune workflow are still materially incomplete. The existing April 26 review comment in the issue reached the same keep-open conclusion.

## Best Possible Solution

Keep #13601 open until maintainers either implement or deliberately narrow the remaining history surface. The likely path is to add timestamp filters through `ReadCronRunLogPageOptions`, `cron.runs`, protocol schemas, UI state, CLI options, and docs; expose a clearer `openclaw cron logs` or `openclaw cron history` command with status/time/follow/export behavior; add success-rate and failure aggregate calculations to `cron list` or a dedicated summary; and decide whether manual prune/vacuum/history-retention-days belongs in core or should be explicitly documented out of scope.
