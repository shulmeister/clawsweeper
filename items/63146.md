## Summary

Keep open. Current main does not include the PR's requested corpus supplement ordering change or its regression tests. The PR is a focused bug fix: `listMemoryCorpusSupplements()` still returns insertion order on main, while current call sites consume that order for first-hit lookup and plugin loader cache snapshots. The existing prompt supplement path already sorts by `pluginId`, so the proposed corpus-side fix matches an established local determinism pattern.

## Best Possible Solution

Keep the PR open or land an equivalent minimal fix: sort `listMemoryCorpusSupplements()` by `pluginId`, add the two regression tests for reverse registration and re-registration stability, then run the changed memory/plugin test lane in a writable validation context.
