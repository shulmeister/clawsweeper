## Summary

Current `main` and the latest shipped release both intentionally preserve Feishu thread/topic affinity for inbound threaded messages, even when `replyInThread` is configured as `"disabled"`. The PR's proposed behavior conflicts with current code and an explicit regression test on `main`, so this cleanup item is obsolete rather than mergeable.

## Evidence

- **Current main still treats inbound thread context as authoritative:** `resolveFeishuGroupSession()` computes `threadReply` from inbound `thread_id`/`root_id` and then sets `replyInThread` to `(config === "enabled") || threadReply`, so an inbound threaded message still replies in-thread even when config is `"disabled"`.
  - file: [extensions/feishu/src/bot-content.ts:71](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/extensions/feishu/src/bot-content.ts#L71)
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Dispatcher explicitly forces thread-mode replies for threaded inbound messages:** `createFeishuReplyDispatcher()` documents `threadReply` as "inbound message is already inside a thread/topic context" and applies `effectiveReplyInThread = threadReplyMode ? true : replyInThread`.
  - file: [extensions/feishu/src/reply-dispatcher.ts:85](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/extensions/feishu/src/reply-dispatcher.ts#L85)
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Main now has a regression test for the exact scenario in this PR:** `bot.test.ts` covers `replyInThread: "disabled"` with an inbound `thread_id` and expects the dispatcher to receive `replyInThread: true` and `threadReply: true`. That is the opposite of this PR's intended change.
  - file: [extensions/feishu/src/bot.test.ts:2396](https://github.com/openclaw/openclaw/blob/a29233c9af0c03e906b5a1e3dceb927119eafe04/extensions/feishu/src/bot.test.ts#L2396)

- **Latest release still ships the same behavior:** The release tag `v2026.4.22` contains the same `replyInThread = ... || threadReply` logic in `extensions/feishu/src/bot-content.ts`, so this is not an unlanded fix waiting to merge.

## Close Comment

Closing after Codex review. Current `main` does not reproduce the PR's claimed bug under the code OpenClaw now ships: inbound Feishu messages that already belong to a topic/thread still keep thread affinity regardless of `replyInThread: "disabled"`. That behavior is present in `extensions/feishu/src/bot-content.ts:71`, enforced again in `extensions/feishu/src/reply-dispatcher.ts:112`, and covered by an explicit regression test in `extensions/feishu/src/bot.test.ts:2396`. The same logic is also present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). Based on that evidence, this PR no longer matches current OpenClaw behavior and is being closed as obsolete.
