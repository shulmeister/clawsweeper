## Summary

Keep #69060 open. Current main still resolves OpenAI-compatible HTTP sessions from `x-openclaw-session-key`, then the OpenAI `user` field, then a random UUID; it does not implement the proposed `x-openclaw-session` agent-scoped stable hint. The PR is a focused implementation for an open subproblem in #68920, with a bot review noting only a non-blocking validation hardening concern, so it is not obsolete or safely closeable.

## Best Possible Solution

Keep the PR open and either finish it in-place or replace it with an equivalent main-branch patch: add `x-openclaw-session` handling inside `resolveSessionKey`, preserve `x-openclaw-session-key` and `user` priority, add focused tests in the gateway HTTP/request-context coverage, and decide whether to validate or sanitize the hint before merge.
