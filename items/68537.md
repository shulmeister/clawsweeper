## Summary

Keep PR #68537 open. Current main still has the Telegram configured-ACP-binding limitation this PR targets: top-level Telegram ACP bindings only compile and match topic-shaped IDs whose chat id starts with `-`, so a plain positive direct-chat id is still ignored. The PR is a focused, plausible fix with matching tests/docs, and the related #62769 remains the canonical open product bug rather than a reason to close the implementation PR.

## Best Possible Solution

Keep the PR open for maintainer review against the related open bug #62769. The best product path is to land or adapt the focused Telegram adapter change: compile positive direct-chat ids as self-parented ACP binding targets, match direct inbound conversations before topic parsing, keep bare negative group ids rejected, and include focused adapter/message-context coverage plus user-facing docs updates.
