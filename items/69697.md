## Summary

Keep PR #69697 open. Current main still has the undefined-params crash path the PR targets: `hashToolCall()` passes `params` into `digestStable()`, `stableStringify(undefined)` returns the JavaScript value `undefined`, and `createHash().update(serialized)` throws `ERR_INVALID_ARG_TYPE`. The existing hash tests cover string, number, and null params but not undefined. The PR is a focused fix for a real current-main bug, though it should ideally add a regression test before landing.

## Best Possible Solution

Keep the PR open and either land this focused stable-stringify fallback fix with regression coverage or apply an equivalent fix on main. The implementation belongs in `src/agents/tool-loop-detection.ts`; current tests should be extended in `src/agents/tool-loop-detection.test.ts` to lock undefined params and undefined error serialization behavior.
