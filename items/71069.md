## Summary

Current main still contains the exact generic reset path reported by the issue. The code supports Gemma 4 through provider-qualified configured/catalog models, but a persisted override outside the configured agent model allowlist is cleared with only “Model override not allowed for this agent; reverted to …”. The issue is recent and remains a concrete request for clearer rejection behavior or allowlist handling, so it should not be auto-closed.

## Best Possible Solution

Keep the issue open. The best fix is likely to improve the stored session override rejection path so it reports the rejected normalized provider/model, points users to /models or configured allowed models, and clarifies that local GGUF-backed models need a provider-qualified configured ref such as ollama/gemma4, lmstudio/<model>, or a custom OpenAI-compatible provider entry.
