## Summary

Keep this issue open. Current main still clears stored session model overrides that are outside the configured allowlist and emits the same generic reset event without naming the rejected model. The exact reported `Gemma4-26b-a4-it-gguf` ref is not recognized or documented in-tree; current Gemma examples are provider-specific, so maintainers still need the reporter's intended backend/config before deciding whether this is unsupported spelling, missing docs, a diagnostic gap, or an alias/normalizer bug.

## Best Possible Solution

Keep this issue open. Ask the reporter for the exact model/provider configuration and intended backend, then either improve the reset diagnostic to include the rejected ref and allowed/provider-specific alternatives, document the required Gemma/GGUF setup more directly, or add a canonical alias only if `Gemma4-26b-a4-it-gguf` is intentionally supported for a provider.
