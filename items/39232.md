## Summary

Keep #39232 open. Current main still has no typed plugin hook equivalent to `on_raw_inbound` / `before_channel_filter`, and the existing message hooks fire only after channel access, routing, and group mention gating. The related WhatsApp PR #39240 was closed unmerged, so it does not supersede or implement the requested core/plugin API surface.

## Best Possible Solution

Keep this issue open for maintainer/product design. A good fix would add an explicit, observe-only, privacy-aware pre-filter inbound plugin hook or channel contract, wire it through bundled channels such as WhatsApp and Telegram before access/mention gating, document the payload and guardrails, and cover it with contract plus channel tests. If maintainers prefer not to expose raw inbound content globally, close only after documenting that product/security decision and any narrower supported alternative.
