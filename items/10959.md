## Summary

## Close Comment

Thanks for the suggestion. This was reviewed by Codex against current `main`, and the evidence points to LinkedIn support belonging in the plugin/ClawHub path rather than OpenClaw core.

Concrete repo evidence:
- `VISION.md:69-70` says new skills should go to ClawHub first.
- `VISION.md:103-105` says OpenClaw is not merging commercial service integrations outside the model-provider category, and prefers MCP/plugin-style integration paths over adding more first-class core runtime.
- `docs/plugins/building-plugins.md:11-19` and `docs/plugins/community.md:9-22` document plugins/ClawHub as the supported way to add new channels, tools, and similar capabilities.
- Current `main` (`86f69ba5a06ae256906c3a26308801e0b84e0091`) does not include a built-in LinkedIn integration in `src/`, `extensions/`, or `docs`.

Because of that, I’m closing this as a ClawHub/plugin-scoped request rather than a core roadmap item. If someone wants to pursue LinkedIn support, the best fit is an external OpenClaw plugin or MCP integration published through ClawHub.
