## Summary

Keep open. Current `main` still hard-codes non-batch memory indexing concurrency to 4 and exposes only `agents.defaults.memorySearch.remote.batch.concurrency`, which applies to batch-API jobs rather than the normal per-file indexing path requested in #14207.

## Evidence

- **Non-batch indexing concurrency is still hard-coded:** `EMBEDDING_INDEX_CONCURRENCY` is set to `4`, and `getIndexConcurrency()` returns that constant whenever batch mode is off.
  - file: [extensions/memory-core/src/memory/manager-embedding-ops.ts:46](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/extensions/memory-core/src/memory/manager-embedding-ops.ts#L46)
  - command: `nl -ba extensions/memory-core/src/memory/manager-embedding-ops.ts | sed -n '1,120p'`
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **File discovery and indexing both use that same fixed concurrency:** `syncMemoryFiles()` builds file entries and then runs indexing tasks with `runWithConcurrency(..., this.getIndexConcurrency())`, so the normal memory indexing path remains capped by the fixed value unless batch mode is enabled.
  - file: [extensions/memory-core/src/memory/manager-sync-ops.ts:688](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/extensions/memory-core/src/memory/manager-sync-ops.ts#L688)
  - command: `nl -ba extensions/memory-core/src/memory/manager-sync-ops.ts | sed -n '660,860p'`
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **Current config schema has batch concurrency only:** `MemorySearchConfig` defines `remote.batch.concurrency` but has no `memorySearch.indexing.concurrency` or equivalent knob for non-batch indexing workers.
  - file: [src/config/types.tools.ts:364](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/config/types.tools.ts#L364)
  - command: `nl -ba src/config/types.tools.ts | sed -n '336,470p'`
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **Docs still describe only batch-job concurrency:** The memory config reference documents `remote.batch.concurrency` under `Batch indexing` and does not document a separate indexing-worker concurrency setting.
  - file: [docs/reference/memory-config.md:325](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/docs/reference/memory-config.md#L325)
  - command: `nl -ba docs/reference/memory-config.md | sed -n '300,360p'`
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
