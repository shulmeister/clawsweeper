## Summary

Keep #14207 open. Current main still hard-codes non-batch memory indexing concurrency at 4 and only exposes `remote.batch.concurrency` for provider batch APIs, which does not cover the custom OpenAI-compatible inline embedding proxy path described in the issue. The related #68812/#68822 context is open follow-up work, not a shipped or clearly canonical replacement.

## Best Possible Solution

Keep #14207 open until OpenClaw adds an agent-scoped non-batch memory indexing concurrency setting, wires it through `MemorySearchConfig`, `ResolvedMemorySearchConfig`, Zod schema/help/generated metadata, docs, and focused tests, and uses it in `getIndexConcurrency()` without changing the existing provider Batch API meaning of `remote.batch.concurrency`.
