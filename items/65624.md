## Summary

Keep open. Current `main` still derives Mattermost native slash-command callbacks as `http://...` when `channels.mattermost.commands.callbackUrl` is omitted, and the active slash-command state/HTTP handler still authorize by membership in a token set rather than by token-to-trigger binding. The latest release commit provided (`v2026.4.23`, `a9797214338ba31c52c796adbb75afb16e0684a9`) shows the same behavior, so this is not implemented or shipped yet.

## Evidence
