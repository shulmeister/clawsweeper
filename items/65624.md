## Summary

Keep this issue open. Current main still derives HTTP Mattermost slash callback URLs and validates slash callbacks by token membership only, and the provided GitHub context shows an open, unmerged linked PR with closing syntax for this issue. That PR is the implementation candidate, so the issue should remain open until the PR is reviewed, landed, or closed by maintainers.

## Best Possible Solution

Review and either land or close the linked Mattermost hardening PR. The best code path is to reject non-HTTPS resolved slash callback URLs before native command registration and preserve token-to-trigger bindings through slash activation and HTTP dispatch, with focused regression tests in the Mattermost plugin. After that PR or an equivalent maintainer-approved fix merges, close this issue as implemented with the merge SHA and shipped release.
