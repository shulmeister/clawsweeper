## Summary

Keep open. Current main does not have the proposed `session-working-context` module, a generic per-session `workingContext`/capsule field, or `/status`, compaction, `/new`, exec/git, or spawn wiring for active repo, cwd, branch, temp clones, and push target. Existing main stores related but narrower session metadata such as `sessionFile`, `spawnedWorkspaceDir`, token/compaction state, `systemPromptReport.workspaceDir`, sandbox status, and ACP cwd. The linked #67511 remains open, and this draft PR is a plausible MVP storage-shape discussion rather than obsolete or safely closeable.

## Best Possible Solution

Keep #71135 open for maintainer review against #67511. The best path is to decide the working-context storage shape, then wire it into the session/transcript lifecycle, exec/git context capture, `/status`, compaction/new-session flows, subagent spawn inheritance, and docs. If maintainers prefer a different design, supersede this PR from #67511 rather than auto-closing it as stale or implemented.
