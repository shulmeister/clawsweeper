## Summary

Current main does not implement heartbeat-specific model fallback configuration. `heartbeat.model` is still schema/type constrained to a string, the heartbeat runner only forwards a single `heartbeatModelOverride`, docs describe only `provider/model`, and the linked PR #69495 is still an unmerged proposed fix rather than shipped behavior.

## Best Possible Solution

Keep #69434 open as the user-facing tracker until #69495 or an equivalent fix lands. The right product/code path is to make `agents.*.heartbeat.model` use `AgentModelConfig`, resolve primary and fallback values with the existing model-input helpers, pass heartbeat fallbacks through the reply/followup/runWithModelFallback path, update pricing-cache collection, docs/schema help/generated schema, and add regression tests for defaults and per-agent heartbeat object form.
