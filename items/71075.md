## Summary

Close as ClawHub/plugin-scoped. The PR adds a large Jarvis Desktop-specific helper layer to core, but current main already exposes the needed gateway/plugin seams for an external Jarvis-owned plugin or package: public plugin gateway runtime exports, CLI/gateway-method/service registration, documented operator RPC methods, and documented gateway event families. The thread also links the dependent Jarvis Desktop PR, so the integration has an external home; OpenClaw core should only take a later narrow API/security fix if a concrete missing generic seam is identified.

## Best Possible Solution

Keep OpenClaw core unchanged for this PR and move the Jarvis Desktop helper layer into a Jarvis-owned ClawHub/native plugin or external package using `openclaw/plugin-sdk/gateway-runtime`, plugin CLI registration, plugin gateway methods, and the documented Gateway protocol. If that work exposes a missing generic OpenClaw API, open a narrow follow-up PR for that API only.
