## Summary

Keep open. Current main still lets subagent completion output fall back to raw `toolResult`/`tool` text without checking `isError` or `is_error`, so the leak reported in #39032 is not resolved well enough to close. The direct fix linked in the issue discussion (#39367) was closed unmerged; the later merged hardening work (#59649) covered adjacent task/status/followup surfaces but does not add the missing error-tool-result guard in this subagent completion selector.

## Best Possible Solution

Implement the missing subagent completion selector guard: skip failed tool results marked by message-level `isError: true` or content/block-level `is_error: true` when deriving completion output, while preserving successful tool-output fallback. Add focused regression coverage for failed `toolResult` and `tool` messages, plus a direct-fallback assertion so failed tool text cannot bypass the requester-agent rewrite path.
