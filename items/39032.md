## Summary

Keep open. Current main still allows `toolResult` / `tool` message text to become subagent completion fallback output without checking failed-tool markers such as `isError`, `is_error`, or status-based tool-result errors. The direct fix in #39367 was closed unmerged, and the merged #59649 hardened adjacent internal-context surfaces but did not add this extractor guard.

## Best Possible Solution

Keep the issue open and implement the narrow extractor fix in `src/agents/subagent-announce-output.ts`: skip failed tool results before populating `latestRawText`, covering top-level `isError`, content/block-level `is_error`, and status-based tool-result errors such as `details.status: "error"` or `"timeout"`. Add focused regression tests proving failed tool results are ignored while successful tool-only completion fallback still works.
