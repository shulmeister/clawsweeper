## Summary

Keep open. Current main still allows subagent completion output extraction to treat tool-result/tool text as announcement fallback without checking `isError` or `is_error`, so the reported leak path is not fixed. The issue discussion’s direct fix, #39367, was closed unmerged; #59649 was adjacent sanitization for task/status/followup surfaces and does not close this specific extraction path.

## Best Possible Solution

Keep the issue open and implement the narrow extraction fix in `src/agents/subagent-announce-output.ts`: detect failed tool results (`isError`, `is_error`, and the existing tool-result error status shape where applicable) before populating `latestRawText`, with regression tests that prove failed tool results are skipped while successful tool-only completion fallback still works. A secondary hardening path can validate or guide `message` tool targets in subagent contexts, but the closeable bug is the announce extraction leak.
