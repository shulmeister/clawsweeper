## Summary

Keep open. Current main still lets `toolResult` / `tool` message text become subagent completion fallback output without checking failed-tool markers, so the reported leak path remains present. The related narrow fix in #39367 was closed unmerged; the merged #59649 hardens adjacent task/status/followup surfaces but does not add this extractor guard.

## Best Possible Solution

Keep the issue open and implement the narrow extractor fix in `src/agents/subagent-announce-output.ts`: skip failed tool results before populating `latestRawText`, covering top-level `isError`, content/block-level `is_error`, and existing tool-result status shapes such as `details.status: "error"` or `"timeout"`. Add focused regression tests proving failed tool results are ignored while successful tool-only completion fallback still works. Optional secondary hardening is stricter `message` tool target validation in subagent contexts, but that does not replace fixing the announce extraction leak.
