## Summary

Keep open. Current main still contains the reported Mattermost HMAC derivation from the bot token with the public hardcoded key, and the linked PR is a plausible security fix but has an unresolved P1 regression risk: its random per-process fallback can break CLI-signed button payloads verified by the gateway unless a persistent/configured interaction secret is present.

## Best Possible Solution

Keep #64546 open and revise it rather than auto-closing it. The best fix is a persistent or explicitly configured per-account Mattermost interaction secret, SecretRef-aware where appropriate, that is available to both message-sending and gateway-verification processes; update schema/types/secret contract/docs and add regression tests covering cross-process stability plus rejection of bot-token-derived forgery.
