## Summary

Keep this PR open. Current main still derives Mattermost interaction signing secrets from the bot token with the public hardcoded HMAC key, and the docs/tests still encode that behavior. The PR targets the right owner surface and is paired with the open linked bug #64545, but it needs iteration because the current patch’s generated in-memory fallback can break CLI/gateway cross-process verification when no persistent interaction secret is configured.

## Best Possible Solution

Keep this PR and the linked bug #64545 open. The right fix belongs in the Mattermost plugin: add a persistent per-account interaction signing secret shared by send and monitor paths, support SecretRef/config or credential-store storage, preserve or intentionally migrate existing deployments, then update schema/types/docs and regression tests for both forgery resistance and cross-process button verification.
