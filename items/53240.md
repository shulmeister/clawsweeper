## Summary

Current `main` already resolves Telegram approval callbacks through the shared operator-approvals gateway client and a global pending-approval lookup, not a default-account-scoped session. The current code also binds Telegram approval authorization/config by account, so the reported non-default-account resolution failure is not how `main` behaves now.
