## Summary

Keep this issue open. Current main partially improves the compaction side by estimating transcript tokens for preflight compaction and adding a transcript-size compaction trigger, but the central requested behavior is still missing: provider-missing/all-zero usage is not converted into an explicit local estimated context snapshot for /status or /usage, and no estimated marker is rendered.

## Best Possible Solution

Keep this open and implement a shared estimated context snapshot path for sessions whose provider usage is missing or all-zero. Reuse readSessionMessages plus estimateMessagesTokens or a shared chars-per-token helper, mark the result explicitly as estimated, render it in /status and /usage with an estimated indicator, and let compaction gates consume the same estimate without replacing fresh provider snapshots. Add regression coverage for a MiniMax-style all-zero usage transcript and for transcripts with messages but no usage metadata.

## Risks / Open Questions

- A fix should not overwrite fresh provider-reported context snapshots with heuristic estimates; the estimated state needs to be explicit and stale-aware.
- Related closed issues such as #62768 and #56460 cover broader local usage reporting/dashboard surfaces, but they do not cover the narrower missing/all-zero provider usage fallback requested here.
- Preflight compaction now has local transcript estimation, so the best implementation should reuse or centralize that estimator rather than adding a second heuristic with different semantics.
