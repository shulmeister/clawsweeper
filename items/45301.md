## Summary

Keep open. Current main improves/nudges the Feishu wiki path by typing `space_id` fields as strings and showing quoted examples, but I did not find a robust fix for the reported precision-loss path when a provider emits a 19+ digit ID as a bare JSON number. The execution path forwards `p.space_id` directly, and local reproduction confirms plain `JSON.parse` rounds the sample unsafe integer shape before plugin code can recover it.
