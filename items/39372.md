## Summary

Keep PR #39372 open. Current main still rejects the exact headless `mode="session"` without `thread=true` behavior for both native subagents and ACP, docs/tests still encode the thread requirement, and the related discussion has not been superseded by a shipped implementation. The PR also still needs maintainer judgment or follow-up for the ACP inline-delivery concern raised in review.

## Best Possible Solution

Keep #39372 open as the canonical maintainer decision point. If OpenClaw should support headless persistent sessions, update both native and ACP spawn guards, make ACP inline delivery require an actual binding, keep failed-start cleanup correct, add distinct headless accepted notes, and update tests/docs. If maintainers prefer current thread-bound session semantics, close the PR with that explicit product rationale.
