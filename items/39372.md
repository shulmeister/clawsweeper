## Summary

Keep open. Current main still rejects the exact headless `mode="session"` behavior this PR proposes for both native subagents and ACP. The related issue #65396 was closed as a duplicate of this PR, making #39372 the canonical tracker, and the PR discussion contains a still-relevant ACP inline-delivery safety concern that needs maintainer-quality resolution before merge or close.

## Best Possible Solution

Keep #39372 open as the canonical maintainer decision point for headless persistent sessions. If the feature should land, update both native and ACP spawn paths, make ACP inline delivery require an actual binding, adjust model-facing notes/docs/tests, and preserve current run-mode `sessions_yield` orchestration behavior. If maintainers choose the existing `mode="run"` plus retained-session workflow instead, close this PR with that product rationale.
