## Summary

Current main already contains the fix path for this report: Feishu tool registration still lives in `extensions/feishu/index.ts`, but gateway config-schema reads now use manifest metadata instead of activating plugin runtime, and there is a regression test calling out the prior Feishu re-registration behavior directly. The same code and changelog entry are present in release `v2026.4.22`, so this appears shipped.

## Close Comment

Closing from a Codex review: this is already implemented on current `main`, and the same fix is present in `v2026.4.22`. The Feishu tools still register through `extensions/feishu/index.ts:73`, but gateway schema reads now use manifest metadata in `src/config/runtime-schema.ts:21` instead of activating plugin runtime, and `src/config/runtime-schema.test.ts:227` adds a regression guard that explicitly calls out the old Feishu re-registration behavior. The shipped changelog entry at `CHANGELOG.md:922` also notes that schema checks no longer re-activate plugin registry state. If you still see repeated Feishu tool-registration logs on `v2026.4.22` or newer, please open a new issue with fresh logs and the exact request path still triggering runtime activation.
