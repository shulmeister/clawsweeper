## Summary

Keep open. Current main still preserves bare Telegram `group:<numeric>` targets instead of normalizing them, and the PR's own bot review correctly identified that the patch is incomplete because `parseTelegramTarget` is not updated. The work is stale against the extension refactor, but the underlying behavior is not implemented on main.

## Best Possible Solution

Keep this open or replace it with a current narrow Telegram fix in `extensions/telegram/src/targets.ts`: normalize bare `group:<numeric>` before topic/colon parsing, preserve explicit `telegram:group:` handling, and add regression coverage for `normalizeTelegramChatId`, `normalizeTelegramLookupTarget`, `parseTelegramTarget`, `resolveTelegramTargetChatType`, and the send path.
