## Summary

Keep open. Current main still defines the message tool `channel` parameter as a bare optional string with no schema description or enum, while the existing prompt/docs guidance and runtime fallback only partially mitigate agents passing Slack-style channel IDs. The linked PRs #10364 and #13584 were closed unmerged, so they do not supersede the issue.

## Best Possible Solution

Keep the issue open and implement a focused core change in `src/agents/tools/message-tool.ts`: add a clear `description` to `buildRoutingSchema().channel` explaining that it expects a messaging provider/channel plugin ID such as `slack` or `discord`, not the conversation target. Consider an enum only if it can safely reflect dynamic plugin channels; otherwise add a focused schema test proving the description is exposed.
