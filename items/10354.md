## Summary

Keep #10354 open. Current main still exposes the message tool channel parameter as a bare optional string, and the linked fix PRs were closed unmerged. Runtime fallback and prompt guidance reduce some failures, but they do not implement the requested schema guidance.

## Best Possible Solution

Keep the issue open and land a focused core change in src/agents/tools/message-tool.ts that adds a schema description to buildRoutingSchema().channel explaining that it expects a messaging provider or channel plugin ID such as slack or discord, not the conversation target. Add a narrow schema test proving the description is exposed; defer an enum unless maintainers decide dynamic plugin channels can be represented safely.
