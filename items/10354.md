## Summary

Keep open. Current main partially mitigates the reported Slack/current-context failure by falling back to the current provider when `channel` is an ID, but the requested message tool schema guidance is still missing: `channel` remains a bare optional string with no description or enum. The linked replacement PR #13584 and original PR #10364 were closed unmerged, so they do not supersede the issue.
