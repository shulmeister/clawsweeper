## Close Comment

Closing as implemented on current `main`. This Codex review checked current `main` and the shipped `v2026.4.22` tag (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). The cron deadlock path described here is addressed by remapping nested cron-triggered agent work off the `cron` lane in `src/agents/lanes.ts:8` and using that remap from `src/cron/isolated-agent/run-executor.ts:179`. Manual `cron run` preflight also clears stale `runningAtMs` markers before the already-running check in `src/cron/service/ops.ts:510`, and cron execution is wrapped with a timeout safety net in `src/cron/service/timer.ts:79` so a hung run does not wedge the lane permanently. The current regression coverage in `src/cron/isolated-agent.lane.test.ts:85` and `src/cron/service/ops.regression.test.ts:241` matches the reported failure mode. If you still see a cron stall on `v2026.4.22` or newer, please open a fresh issue with current logs and the affected job JSON, because that would be a new regression surface rather than this already-implemented one.
