## Summary

Current `main` already supports viewing configured temperature with `openclaw config get ...` and setting temperature through agent/model `params`; the runtime forwards that value and tests cover the config shape. The shipped `v2026.4.22` release already contained this support.

## Close Comment

Closing from a Codex review because this is already implemented in current OpenClaw and was already shipped in `v2026.4.22`. You can inspect current configured values with `openclaw config get ...` (`docs/cli/config.md:107`), and the agent config docs already support `temperature` in both model params and per-agent params (`docs/gateway/config-agents.md:366`, `docs/gateway/config-agents.md:944`; also shipped in `docs/gateway/configuration-reference.md:1240` and `:1816` at `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). The runtime wiring is present too: `src/agents/pi-embedded-runner/extra-params.ts:290` forwards `extraParams.temperature`, and `src/config/config-misc.test.ts:597` covers the supported config shape. If the remaining ask is specifically for a dedicated Control UI control rather than config/CLI support, that would need a narrower follow-up request.
