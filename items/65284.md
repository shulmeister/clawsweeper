## Summary

Keep this PR open. Current main still preserves orphaned providers during generated models.json merge and still has provider-unaware context-window lookup paths, so the central bug is not implemented or superseded. The PR remains potentially useful, but it needs focused maintainer review because it changes documented merge semantics and bundles unrelated embedded-runner/cron error-handling work.

## Best Possible Solution

Keep the PR open and request a focused revision: split unrelated embedded-runner/cron error handling, make an explicit maintainer decision on merge-mode orphaned-provider semantics, update docs/schema/tests if orphaned providers should be dropped, and finish provider-aware context-window resolution in gateway/status/sessions paths with persisted contextTokens used only as fallback when provider+model metadata is available.
