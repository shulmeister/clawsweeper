## Summary

Keep open. Current main has useful generic Linux/VPS, Docker, Kubernetes, and Bedrock-on-AWS provider documentation, but it does not yet provide the requested comprehensive AWS deployment guide for EC2/ECS/Lambda. The issue discussion also contains concrete deployment requirements and an active AWS sample repo that would be useful input for a docs PR rather than grounds for closure.

## Evidence

- **Issue request remains materially broader than current docs:** #13597 asks for AWS deployment coverage across EC2 setup, systemd, security groups/firewall, IAM, ECS task/service/networking, Lambda applicability, logging/monitoring, backups, troubleshooting, performance tuning, and hardening. Comments add concrete production needs: Docker Compose as the primary AWS path, persistent storage/EFS, gateway bind/reverse-proxy behavior, AWS Secrets Manager, multi-agent topology, and the active sample repo https://github.com/aws-samples/sample-OpenClaw-on-AWS-with-Bedrock.
