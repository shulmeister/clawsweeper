## Summary

Keep #13597 open. Current main still has generic VPS/Docker/Kubernetes and cloud-provider VM docs plus Amazon Bedrock provider setup, but it does not include the requested AWS deployment guide for EC2, ECS/Fargate, or Lambda applicability. The issue discussion adds concrete production requirements and an AWS sample repo that should feed a docs PR rather than close the request.

## Best Possible Solution

Keep #13597 open until OpenClaw ships an AWS deployment page, likely `docs/install/aws.md`, linked from the install hub and Linux server page. The guide should make Docker Compose the primary EC2/Lightsail path; cover durable state/EFS or equivalent storage, gateway bind/reverse proxy/ALB/TLS/origin settings, security groups, IAM roles, Secrets Manager, logging/monitoring, backups, troubleshooting, performance and hardening; add ECS/Fargate task/service/networking guidance; document whether Lambda is unsupported or only useful for narrow adjunct workflows; cover multi-agent topology; and link the AWS sample repo from the issue discussion where appropriate.
