## Summary

Current `main` gates ACP prompt completion on terminal chat states, preserves the terminal snapshot text before resolving, and has regression coverage for delta-then-final flows. That directly addresses the reported early-finish-on-progress-text behavior for ACP one-shot runs.

## Evidence

- **ACP prompt completion now waits for terminal chat state:** `AcpGatewayAgent.handleChatEvent()` processes `delta` snapshots for streaming text but only resolves the ACP prompt when the gateway event state is `final`, `aborted`, or `error`. A `delta` progress/update message returns early and does not finish the run.
  - file: [src/acp/translator.ts:946](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/acp/translator.ts#L946)
  - command: `nl -ba src/acp/translator.ts | sed -n '938,965p'`
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **Regression tests cover final snapshot winning over earlier progress:** Current tests verify that a final chat snapshot is emitted before resolving `end_turn`, that repeating final text is not duplicated after earlier deltas, and that an extended final snapshot appends only the missing tail. This is the exact class of bug where early visible progress text could otherwise be treated as the completion payload.
  - file: [src/acp/translator.session-rate-limit.test.ts:1064](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/acp/translator.session-rate-limit.test.ts#L1064)

- **ACP control-plane treats text deltas as progress, not terminal completion:** `AcpSessionManager.runTurn()` accumulates `text_delta` into `taskProgressSummary` while the turn stream is still running, then marks the task terminal only after `awaitTurnWithTimeout()` completes the full `runtime.runTurn()` iterator. Early assistant text does not end the turn here.
  - file: [src/acp/control-plane/manager.core.ts:783](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/acp/control-plane/manager.core.ts#L783)

That matches the fixed terminal-result delivery model.
