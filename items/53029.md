## Summary

Keep open. Current `main` still hardcodes the reply-layer typing TTL to 2 minutes and does not expose any `typingTtlMs` config in the agent/session schema or docs. Recent typing work keeps indicators alive on tool activity, but that is not the requested configurable TTL feature.

## Risks / Open Questions

- The report's original symptom is partly mitigated by later typing keepalive work, so any eventual implementation should be scoped against current behavior rather than the older bundled-JS patch description.
- Long-running tool calls that do not emit text or `tool` update events can still hit the hardcoded 2-minute reply-layer TTL, so closing this as obsolete would be unsafe.
