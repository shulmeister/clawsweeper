## Summary

#70666 is still a real current-main bug, but the same remaining work is now tracked by the narrower open PR #71170. Current main still appends the reminder-guard note whenever a reminder-like reply has `successfulCronAdds === 0` and no same-session cron job, and core only increments that counter for the native `cron` add tool path. PR #71170 directly targets this issue with an additive `agents.defaults.reminderGuard` and per-agent opt-out for external schedulers, matching option (b) from the issue while leaving SDK-level scheduling signals for later design review.

## Best Possible Solution

Use #71170 as the canonical tracker and review or land that PR, or an equivalent additive per-agent/global reminder-guard opt-out. Keep a richer plugin SDK scheduling signal as a separate API proposal if maintainers want external reminder tools to report successful scheduling directly.
