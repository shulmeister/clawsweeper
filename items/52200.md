## Summary

Keep open. Current `main` still appears to have the Windows hybrid-path bug this PR fixes: home-prefix compaction rewrites `C:\...` to `~/...` but does not normalize remaining backslashes, and the available-skills prompt emits that path verbatim.

## Evidence

- **Current compaction logic still leaves Windows separators intact:** `compactHomePath()` returns `"~/" + filePath.slice(prefix.length)` when a skill path is under the home directory, with no subsequent `\\ -> /` normalization in the current implementation.
  - file: [src/agents/skills/workspace.ts:77](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/agents/skills/workspace.ts#L77)
  - command: `nl -ba src/agents/skills/workspace.ts | sed -n '56,95p'`
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Prompt output uses the compacted path as-is:** The available-skills prompt writes `<location>${skill.filePath}</location>` directly, so any hybrid Windows path produced by compaction is surfaced to the model unchanged.
  - file: [src/agents/skills/workspace.ts:657](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/agents/skills/workspace.ts#L657)
  - command: `nl -ba src/agents/skills/workspace.ts | sed -n '648,666p'`
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Current tests do not cover Windows backslash normalization:** `src/agents/skills.compact-skill-paths.test.ts` only checks `~` compaction and preserving paths outside home; the Windows-specific assertion proposed in PR #52200 is not present on `main`.
  - file: [src/agents/skills.compact-skill-paths.test.ts:37](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/agents/skills.compact-skill-paths.test.ts#L37)
  - command: `nl -ba src/agents/skills.compact-skill-paths.test.ts | sed -n '1,120p'`

- **String-level reproduction still yields a hybrid path:** A direct reproduction of the current compaction logic with Windows-style inputs produced `~/\AppData\\Roaming\\npm\\skills\\foo\\SKILL.md`, and the output still contained backslashes (`true`).
  - command: `node -e 'const filePath=String.raw`C:\\Users\\alice\\AppData\\Roaming\\npm\\skills\\foo\\SKILL.md`; const home=String.raw`C:\\Users\\alice`; const path={sep:"\\"}; const prefix=home.endsWith(path.sep)?home:home+path.sep; const out=filePath.startsWith(prefix)?"~/"+filePath.slice(prefix.length):filePath; console.log(out); console.log(out.includes("\\"));'`
