## Summary

Keep open. Current `main` does have concurrency caps now, so the report's "fires all batches in parallel" explanation is at least partly outdated, but the core failure mode is not clearly fixed: batch mode is still intentionally disabled after repeated failures, fallback goes to the non-batch embedding path, and that non-batch remote fetch path still surfaces raw `fetch failed` transport errors without its own retry wrapper. The issue discussion also narrowed one follow-up into distinct watch-sync issue #62069, while the timeline shows a newer similar open report (#71522), so this does not look obsolete or safe to auto-close.
