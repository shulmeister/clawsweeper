## Summary

Current `main` already ships a broader host-exec approval system that covers this request: optional pre-execution approval prompts, configurable allowlist/ask policy, UI/chat approval flows, and persisted approval metadata. The feature was shipped in release `v2026.4.22` via commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`.

## Risks / Open Questions

This shipped in `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). The implementation is broader than the original `rm`/`mv`/`cp` proposal, but it covers the same confirmation-hook safety goal.
