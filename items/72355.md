## Summary

Keep this issue open. Current main appears to mitigate the fatal Bonjour/CIAO crash-loop path, and related #72481 was closed for that narrower crash wiring, but the hostname behavior reported here remains: the bundled Bonjour plugin still defaults the advertised hostname and TXT lanHost to openclaw/openclaw.local unless OPENCLAW_MDNS_HOSTNAME is set, with no discovery.mdns.hostname config field or plugin config field.

## Best Possible Solution

Keep this issue open and narrow it to Bonjour hostname selection/configuration. Maintainers should decide whether the bundled Bonjour plugin should default to a sanitized system hostname or add a supported hostname override, then update extensions/bonjour, MdnsDiscoveryConfig, the strict zod schema, generated schema/help/docs, the bonjour plugin manifest if plugin-owned config is chosen, docs.openclaw.ai gateway docs, and Bonjour tests while preserving the current CIAO cancellation suppression and bounded advertiser recovery.
