## Summary

Keep open. Current main fixes the reported `CIAO PROBING CANCELLED` gateway crash-loop family, but it still defaults the Bonjour advertised hostname/TXT `lanHost` to `openclaw.local` unless `OPENCLAW_MDNS_HOSTNAME` is set, and there is no `discovery.mdns.hostname` config key. The remaining hostname-selection/configuration request is concrete and not fully implemented or covered by a narrower canonical item.

## Best Possible Solution

Keep the issue open but narrow it: either change Bonjour's default host label to a sanitized system hostname, or add a plugin-owned/config-backed hostname override and align the schema, docs, and tests. Preserve the current ciao crash suppression and bounded advertiser recovery already on main.
