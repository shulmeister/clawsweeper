## Summary

Keep open. Current main does not implement a user-configurable Android talk-mode language override: there is no persisted `languageOverride` setting, no Settings UI control, no STT language extras in Android recognizer intents, and gateway/device locale reporting still uses `Locale.getDefault()`. The PR discussion is concrete and includes follow-up commits addressing review comments, and the provided related items do not appear to supersede this Android feature work.

## Best Possible Solution

Keep #40764 open. The best path is to land a reviewed version of the Android app feature that persists a language override, exposes it in Settings, threads the selected locale into Android STT/TTS and gateway/device locale metadata, preserves `auto` as the default, and adds targeted tests where feasible plus manual Android STT/TTS verification.
