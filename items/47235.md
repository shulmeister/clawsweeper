## Summary

Keep open. Current main still implements `memory status --deep` as a fresh live embedding probe and still reports the probe result as `Embeddings: ready` or `Embeddings: unavailable`; the underlying built-in manager still probes by embedding `["ping"]`. Status-only managers also bypass the built-in manager cache, so repeated status invocations can still create fresh probe paths. The related proxy issue/PR in the report is a separate routing problem, and the related ClawSweeper items track adjacent doctor/status symptoms rather than fully superseding this CLI status flapping/misleading-output report.
