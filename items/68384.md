## Summary

Keep #68384 open. Current main still accepts any trimmed non-JSON stdout for a single `jsonOnly: false` exec secret ref, and the existing tests only cover the valid single-line fallback, not the noisy multi-line case this PR adds. The PR is a focused security-hardening fix with a relevant linked bug report, not an obsolete or duplicate cleanup candidate.

## Best Possible Solution

Keep the PR open for maintainer/security review. The right product direction is to fail closed in `parseExecValues()` for ambiguous multi-line non-JSON stdout, preserve valid JSON and valid one-line plaintext behavior, and land the focused regression test or an equivalent implementation on main.
