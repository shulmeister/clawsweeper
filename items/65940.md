## Summary

Keep this PR open. Current main still filters setup web-search providers by `onboardingScopes`, and DuckDuckGo still does not expose that metadata through its shared provider base, runtime provider, or contract artifact. The PR is also paired with the same author's still-open bug report #65862, while the related alternative PR #70170 is closed unmerged, so there is no implemented or canonical superseding fix on main.

## Best Possible Solution

Review or rebase this PR together with #65862. On current main, the narrow fix should add `onboardingScopes: ["text-inference"]` to DuckDuckGo's shared web-search provider metadata in `extensions/duckduckgo/src/ddg-search-provider.shared.ts`, preserving both runtime and public contract paths, and add or update a focused contract/setup assertion so DuckDuckGo remains visible in the web-search setup picker.
