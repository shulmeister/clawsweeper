## Summary

Keep open. Current main still filters setup web-search providers by `onboardingScopes`, and DuckDuckGo’s current shared provider metadata still omits `onboardingScopes: ["text-inference"]`, so the user-visible onboarding bug remains. The PR’s exact two-file patch is now stale because DuckDuckGo metadata was factored into a shared base, and #70170 is a later duplicate candidate rather than a merged/superseding fix.

## Best Possible Solution

Keep #65940 open for maintainer review, update the patch to add `onboardingScopes: ["text-inference"]` in `extensions/duckduckgo/src/ddg-search-provider.shared.ts`, and add a focused regression test against the real DuckDuckGo provider/setup option. If only one duplicate should remain, #70170 is the later duplicate candidate; #65940 is the earlier linked fix for #65862.
