## Summary

Keep this PR open. Current `main` still only transforms Google Workspace Add-on `chat.messagePayload`, so `addedToSpacePayload` and `removedFromSpacePayload` still fall through to the generic event validation and are rejected as `invalid payload`. The PR remains a focused implementation candidate for the add-on lifecycle-payload part of #65007, though it should add regression tests and address the review comment before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to revise the parser with explicit branches for `messagePayload`, `addedToSpacePayload`, and `removedFromSpacePayload`, preserve add-on token handling for each branch, add focused tests for the added/removed lifecycle variants, and leave #65007 open for the remaining group allowlist and reply-thread issues.
