## Summary

Keep open. Current `main` still only normalizes Google Workspace add-on payloads when `chat.messagePayload` is present, so `addedToSpacePayload` and `removedFromSpacePayload` remain unhandled and would still fall through to `400 invalid payload`. The PR appears to address a real gap, but the review thread also identified a logic issue in the proposed implementation and there is no corresponding regression test for the lifecycle variants on `main`.

## Evidence

- **Current parser only accepts add-on `messagePayload`:** `parseGoogleChatInboundPayload` transforms add-on envelopes only under `rawObj.commonEventObject?.hostApp === "CHAT" && rawObj.chat?.messagePayload`. The scoped add-on shape declared above it does not include `addedToSpacePayload` or `removedFromSpacePayload` on current `main`.
