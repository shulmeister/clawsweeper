## Summary

Keep #8813 open. Current main still hardcodes read-based Skills guidance and `buildSkillsSection` cannot inspect registered tools, even though runtime/plugin tool names already reach prompt construction. Tests and docs still assert the existing `read SKILL.md` behavior, and the related items do not supersede this specific `Skill`-tool preference request.

## Best Possible Solution

Keep #8813 open. The best fix is in `src/agents/system-prompt.ts`: pass the normalized available-tool set and casing resolver into `buildSkillsSection`, prefer invoking a registered `Skill` tool when present, preserve the `read` fallback otherwise, then update `src/agents/system-prompt.test.ts` and public Skills/system-prompt docs for both behaviors.
