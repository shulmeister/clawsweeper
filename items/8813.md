## Summary

Current main still has the behavior reported in #8813. The skills system prompt unconditionally tells the model to read SKILL.md with the read tool, buildSkillsSection does not receive availableTools, and tests/docs still assert and describe the read-based behavior. This is a meaningful remaining core prompt behavior request, not safe to close.
