## Summary

Keep #8813 open. Current main still emits unconditional read-based Skills guidance, even though runtime tool names are available to prompt construction and plugin registration can expose a tool named Skill. The prior April 26 Codex review in the issue reached the same conclusion; current main at 82262387651ab270b565f71533ce4937c9c00dc6 still matches that behavior.

## Best Possible Solution

Keep #8813 open. The best fix likely belongs in src/agents/system-prompt.ts: pass normalized available-tool information and caller casing into buildSkillsSection, prefer invoking the registered Skill tool when present, preserve the current read fallback otherwise, then add tests in src/agents/system-prompt.test.ts for Skill-tool preference and read fallback. Update public Skills/system-prompt/context docs to describe both behaviors.
