## Summary

Current `main` already supports per-job cron `payload.model` overrides end-to-end. The cron payload and patch types include `model`, the Gateway protocol accepts it for `cron.update`, the cron tool exposes/recovers it, runtime model selection applies it with allowlist handling, and regression tests cover a model-only `cron.update` patch. The implementation was already present in release `v2026.4.22`.
