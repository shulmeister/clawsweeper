## Summary

Keep #45508 open. Current main has added a separate Control UI Talk/Realtimes path, but the original WebChat mic and per-message Read aloud controls still use browser SpeechRecognition and speechSynthesis. Gateway TTS methods exist, and Talk has backend speech support, but WebChat Read aloud is not wired to them; there is also no apparent gateway STT RPC for the browser mic path. The prior comment’s split is still accurate: TTS is implementation-ready through existing gateway methods, while STT needs a transport/API decision.
