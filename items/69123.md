## Summary

Keep open. Current main still requires `agents.defaults.cliBackends.<id>.command` in the TypeScript config type, Zod schema, and generated config schema, so the partial `modelAliases` override described by PR #69123 is not implemented. The PR is directionally useful, but the existing review comment about regenerating `src/config/schema.base.generated.ts` is a real blocker before this should be considered complete.

## Best Possible Solution

Keep the PR open and ask for a small revision: make `CliBackendConfig.command` and `CliBackendSchema.command` optional, introduce or otherwise preserve a resolved-backend type with `command: string`, regenerate `src/config/schema.base.generated.ts`, and add regression coverage that validates a `modelAliases`-only `agents.defaults.cliBackends.claude-cli` override through both runtime config parsing and backend resolution.
