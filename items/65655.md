## Summary

Keep open. Current main still has the Mattermost slash-command auth gap this PR targets: omitted callback URLs derive plain HTTP URLs and only warn in the loopback case, while callback token validation is still account-token-set based and not bound to the slash trigger. The PR is also paired with the same author's open bug #65624 via closing syntax, so it should remain open as an implementation candidate rather than be auto-closed.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for #65624. The best path is to update the patch so trigger binding is required or fails closed, decide how to handle explicit non-HTTPS callbackUrl values, keep the Mattermost docs/tests aligned, then land the PR or a maintainer-equivalent patch.
