## Summary

Keep PR #27061 open. Current main still accepts empty or whitespace-only skill `name` and `description` values because `quick_validate.py` strips the fields and then skips validation under truthy guards. The PR's `.skill` filename rationale is inaccurate, but the underlying metadata validation bug is real and not superseded by the related closed PRs.

## Best Possible Solution

Keep PR #27061 open, or land an equivalent narrow fix, until `skills/skill-creator/scripts/quick_validate.py` explicitly rejects empty and whitespace-only `name` and `description` values after stripping and `skills/skill-creator/scripts/test_quick_validate.py` includes focused regression tests. Before merge, update the rationale to focus on invalid skill metadata and skill triggering/discovery behavior rather than archive filename generation.
