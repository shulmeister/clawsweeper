## Summary

Keep open. Current main has several trace-like surfaces, but they remain fragmented: trajectory capture is per-session/runtime, cache trace has its own cache-stage schema, diagnostic events have separate low-level event types, agent event streams have their own payload shape, and task/Task Flow records track status/progress without emitting a canonical run trace schema. Related PR #43145 is open and unmerged; it appears to target an agent timeline/run-trace feature, but that work is not present on main and does not prove the ACP/task-flow adoption requested here.

## Best Possible Solution

Keep #68204 open as an architecture/observability tracking issue. The best path is to design a small versioned run-trace contract, decide how it relates to existing trajectory and diagnostic-event surfaces, then map agent events, ACP relay events, subagent/task lifecycle updates, and Task Flow state changes into that contract with tests. PR #43145 should be treated as related prior art, not as a shipped or complete replacement.
