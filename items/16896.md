## Summary

Keep #16896 open. Current main still lacks the requested Dashboard/WebChat right-click Reply workflow: there is no message-level context menu or reply selection state, outgoing `chat.send` remains a plain message/attachment payload, and the existing reply metadata support only displays assistant-side `[[reply_to_*]]` directives. The April 26 maintainer Codex review already treated this as the canonical open tracker, and related #29732 was closed as a duplicate of this item rather than as shipped work.

## Best Possible Solution

Keep #16896 open as the canonical Dashboard/WebChat reply UX request. The likely product path is to add an accessible message-level Reply action/context menu in the Control UI chat surface, store selected reply target/quote state in the composer, focus the textarea, and send either a designed quoted-message representation or an additive `chat.send` reply-target field with matching protocol, rendering, docs, and UI tests.
