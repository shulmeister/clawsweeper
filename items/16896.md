## Summary

Keep #16896 open. Current main still lacks the requested Dashboard/WebChat message-level Reply workflow: there is no right-click/context-menu handler, no composer reply target state, no Reply action in message actions, and `chat.send` has no WebChat reply or quote field. The related #29732 is already closed and cross-referenced back to #16896, so this remains the canonical tracker.

## Best Possible Solution

Keep #16896 open as the canonical Dashboard/WebChat reply UX tracker. A complete implementation should add an accessible message-level Reply action or context menu, store the selected message/quote in composer state, focus the input, send or render a designed quote representation, and add protocol/controller/UI regression tests if reply metadata is persisted.
