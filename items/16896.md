## Summary

Keep #16896 open. Current main still has only inbound/display support for reply directive metadata; the requested Dashboard/WebChat UX, right-clicking a prior message, choosing Reply, quoting/referencing it, and focusing the composer, is not implemented. The related #29732 review treats #16896 as the canonical open tracker for this remaining feature.

## Best Possible Solution

Keep #16896 open as the canonical Dashboard/WebChat reply UX request. The likely implementation belongs in the Control UI chat surface: add a message-level context menu or equivalent accessible action, carry selected reply target/quote state into the composer, focus the textarea, and send either a structured reply target through an expanded `chat.send` contract or an intentionally designed quoted-message representation, with UI tests for the interaction.
