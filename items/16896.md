## Summary

Keep #16896 open. Current main still lacks a Dashboard/WebChat right-click or message-level Reply workflow: there is no context-menu handler, no selected reply-target composer state, no Reply action in bubble controls, and `chat.send` has no reply/quote field. Existing reply support is only assistant directive metadata display (`[[reply_to_*]]`), not a user-initiated WebChat reply UX.

## Best Possible Solution

Keep #16896 open as the canonical Dashboard/WebChat reply UX tracker. A complete fix should add an accessible message-level Reply action and/or right-click context menu, store the selected quoted message in composer state, focus the input, render the selected quote clearly, and decide whether sending plain quoted text is sufficient or whether an additive `chat.send`/message metadata field is needed with UI and gateway regression tests.
