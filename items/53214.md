## Summary

Current main already implements TUI tool-call visibility: `/verbose on|full|off` controls whether tool cards appear, `Ctrl+O` toggles collapsed vs expanded tool output, and non-local final chat events trigger a history reload so externally delivered replies become visible in the session view.

## Evidence

- **Docs advertise the requested controls:** The TUI docs list `Ctrl+O` as the shortcut to toggle tool output expansion, document `/verbose <on|full|off>`, and state that tool calls show as cards with args and results.
  - file: [docs/web/tui.md:95](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/web/tui.md#L95)
- **Keyboard shortcut is wired in the TUI:** `editor.onCtrlO` flips `toolsExpanded`, updates all tool cards, and renders the new collapsed/expanded state.
  - file: [src/tui/tui.ts:999](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/tui/tui.ts#L999)
- **Tool events are rendered when verbose is enabled:** The TUI agent-event handler accepts `stream === "tool"`, suppresses them only when verbose is `off`, and shows tool summaries/results for `on` and full output for `full`.
  - file: [src/tui/tui-event-handlers.ts:414](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/tui/tui-event-handlers.ts#L414)
- **Tool cards render summary + expandable output:** `ToolExecutionComponent` builds a titled tool card, shows formatted args, and collapses long output unless expanded.
  - file: [src/tui/components/tool-execution.ts:116](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/tui/components/tool-execution.ts#L116)
- **Tests cover the behavior:** Tests verify tool events are suppressed when verbose is off, summarized without output when verbose is on, and that non-local final chat events refresh history.
  - file: [src/tui/tui-event-handlers.test.ts:401](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/tui/tui-event-handlers.test.ts#L401)
- **Shipped release notes already mention hidden tool cards:** The changelog section for release `2026.1.9` includes `Control UI/TUI: persist per-session verbose off + hide tool cards`, indicating this capability was already shipped before the issue was filed.
  - file: [CHANGELOG.md:6706](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/CHANGELOG.md#L6706)

The related visibility concern for externally delivered replies is also covered by the non-local history refresh path in `src/tui/tui-event-handlers.ts:355` with test coverage in `src/tui/tui-event-handlers.test.ts:458`.
