## Summary

Keep #13304 open. Current main still implements only static binding-based agent routing and does not support content/prefix/regex matching, prefix stripping, or a documented pre-routing plugin decision that can choose a different existing agent for the normal inbound turn. The prior issue comment reached the same conclusion, and the re-check against current HEAD 04e08cea still matches that finding.

## Best Possible Solution

Keep the issue open. Implement either a first-class bindings[].match.content contract or a documented pre-routing plugin hook that can return a target agentId plus transformed BodyForAgent. The design should define safe prefix/regex semantics, deterministic precedence with existing peer/account/guild/team tiers, stripMatch behavior, config validation/schema/docs, and resolver/channel regression tests.
