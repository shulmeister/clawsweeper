## Summary

Keep #13304 open. Current main still supports only static binding-based agent routing; it does not provide a `bindings[].match.content` matcher, prefix/regex stripping, or a documented pre-routing plugin decision that can reroute an inbound channel message to another existing agent.

## Best Possible Solution

Keep the issue open and implement either a first-class `bindings[].match.content` contract or a documented pre-routing plugin hook that can return a target `agentId` plus transformed `BodyForAgent`. The implementation should define safe prefix/regex semantics, deterministic precedence with existing peer/account/guild/team tiers, prefix stripping behavior, config validation/schema/docs, and resolver/channel regression tests.
