## Summary

Current `main` already ships the requested pre-model-selection plugin seam under the explicit `before_model_resolve` hook, with `before_prompt_build` as the companion post-session hook. The runtime invokes `before_model_resolve` before `resolveModelAsync()`, the SDK/types/docs expose it as the preferred path for model/provider override work, and the behavior is covered by wiring and priority-merge tests. This was already present in release `v2026.4.22`.

## Risks / Open Questions

- The shipped hook name and payload are not identical to the issue's proposed `before_model_select` interface; callers needing richer turn-history fields would need a follow-up enhancement, but the core pre-model-routing hook itself already exists.
