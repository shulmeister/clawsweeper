## Summary

Keep open. Current main has partial browser tab cleanup for tracked session lifecycle events and background idle/excess sweeps, but it still does not implement the requested default of closing temporary browser tabs/windows when an ordinary primary-agent browser task completes.

## Best Possible Solution

Keep #40694 open for a focused browser-plugin/core lifecycle design: distinguish ephemeral automation-opened tabs from reusable, user-owned, labeled, pinned, or explicitly preserved tabs; add a primary-agent task/turn completion cleanup path; and keep the existing idle/excess sweep as a fallback/debug-friendly policy.

## Evidence

That is materially different from closing temporary tabs at task completion.
