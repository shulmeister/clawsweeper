## Summary

Keep PR #42290 open. Current main still does not contain the streaming tar implementation: backup creation now lives in src/infra/backup-create.ts and still calls tar.c with a file destination. The PR is stale against the refactored code and has review issues to fix, but the core behavior it targets is not implemented or superseded on main.

## Best Possible Solution

Keep the PR open as potentially useful work, but require a rebase/refactor into src/infra/backup-create.ts. The best implementation is to preserve the existing temp archive and atomic publish behavior, write the tar output through a real stream pipeline with proper error propagation and stream destruction, add focused tests for stream errors, tmp cleanup, and OOM messaging, and align any user guidance with the CLI flags that actually exist on main.
