## Summary

Keep PR #66783 open. Current main already avoids the original hard block on microphone by sending `microphone=(self)`, but it still does not implement the PR's title-level feature: configurable `gateway.http.securityHeaders.permissionsPolicy` with override and explicit-disable behavior. The PR also has review feedback about unrelated bundled changes, so it needs maintainer review or a cleaned-up revision rather than automated closure.

## Best Possible Solution

Keep the PR open, but ask for a cleaned-up revision that preserves the focused Permissions-Policy config work while dropping or separately explaining unrelated schema metadata and auth-resolution changes. If maintainers decide the same-origin microphone default is sufficient, close only after opening or documenting a clear product decision that OpenClaw will not expose a configurable Permissions-Policy header.
