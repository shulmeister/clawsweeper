## Summary

Keep #70508 open. Current main still has the Mattermost draft-preview behavior the PR targets: partial assistant text updates one draft post, assistant/reasoning boundaries only reset local text tracking, and tool starts write status text into that same draft stream. The PR is not already implemented or superseded on main, and the provided review discussion shows useful but still-unresolved race/test concerns rather than a reason to close it.

## Best Possible Solution

Keep the PR open and ask for a tightened Mattermost-local fix: preserve prior assistant preview text across tool and assistant boundaries, account for draft flush/async callback ordering, and add focused regression tests around boundary rotation and tool-start/item-event sequencing. The current main behavior is not implemented well enough to close this PR.
