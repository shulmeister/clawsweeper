## Summary

Current main still does not provide pluggable session-store backends. The session store remains configured as a file path, loaded/saved through filesystem JSON helpers, and serialized through a lock-file based write path. The related PostgreSQL/MongoDB discussions cited in the issue context are memory-backend work, not the gateway session-store backend API requested here, so they do not supersede this issue.
