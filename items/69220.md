## Summary

Current main still matches the reported conflict: Google Gemini is routed to tagged reasoning output while the Gemini transport sends native `thinkingConfig` with `includeThoughts`. The issue is recent, has corroborating comments, and the remaining fix belongs in the bundled Google provider/core reasoning-mode policy rather than cleanup closure.

## Best Possible Solution

Keep the issue open and fix the bundled Google/Gemini reasoning-output policy. The fix should make direct Gemini/Vertex use native reasoning output when native `thinkingConfig`/thought parts are active, cover both the built-in fallback and the `google-gemini` provider hook path, and add regression tests proving Google Gemini no longer receives the `<think>/<final>` system prompt while native `thinkingConfig` is preserved.
