## Summary

Keep #72210 open. Current main at 6b6dcafcee9ca86385b7c1ee2442752314d50257 still emits `delivery.resolved.ok:false` with `error:"delivery is disabled"` for a bare `delivery.mode:"none"` isolated cron run. The older delivery status bug is fixed, and related PR #72219 is the right narrow fix, but it is not merged into current main per the provided GitHub context.

## Best Possible Solution

Keep #72210 open until PR #72219 or an equivalent change lands. The narrow fix should change `buildCronDeliveryTrace()` or the no-deliver context path so bare `delivery.mode:"none"` run records omit `delivery.resolved` or emit neutral skipped metadata, while preserving `deliveryStatus:"not-requested"` and explicit no-deliver target context for agent-initiated message-tool sends. Add a focused regression test that inspects the emitted run-result/run-log `delivery` object.
