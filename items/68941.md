## Summary

Keep open. Current `main` still uses the old non-Windows guard for the Homebrew prompt, so FreeBSD and other non-`darwin`/`linux` POSIX platforms still hit the reported path. This PR addresses a real, still-unfixed behavior bug rather than something already implemented or obsolete.

## Evidence

That means FreeBSD qualifies for the Homebrew recommendation whenever a selected skill has a `brew` install option and `brew` is missing.
