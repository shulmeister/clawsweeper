## Summary

Keep open. Current main does have pre-consumption tool-result mutation seams, but they are bundled-plugin-only extension factories, not a general public typed hook like the requested `before_tool_result_emit`. That means the issue’s requested external/plugin-author surface is not implemented on main well enough to close.

## Evidence

- **Public hook list still lacks the requested hook:** `PluginHookName` and `PLUGIN_HOOK_NAMES` include `before_tool_call`, `after_tool_call`, `tool_result_persist`, and `before_message_write`, but no `before_tool_result_emit` or equivalent public typed hook.
  - file: [src/plugins/hook-types.ts:72](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/plugins/hook-types.ts#L72)

  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Existing live-result seams are explicitly bundled-only:** The SDK exposes `registerEmbeddedExtensionFactory` and `registerCodexAppServerExtensionFactory`, and both comments state only bundled plugins may use them.
  - file: [src/plugins/types.ts:2122](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/plugins/types.ts#L2122)

## Risks / Open Questions

- This may be partially addressed for bundled plugins already, so maintainers could decide to resolve it with documentation if the intended scope was only bundled/trusted plugins.
- If the desired outcome is a third-party/self-hoster-safe hook, closing now would hide a real remaining gap between bundled-only middleware and the public plugin hook surface.
