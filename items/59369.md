## Summary

Keep open. Current `main` mitigates the reporter's service-file workaround by installing managed gateways with the foreground `openclaw gateway --port ...` entrypoint, but `openclaw gateway start` itself still fails closed when `systemctl --user is-enabled` cannot reach the user bus, so the reported CLI problem is not fully fixed on `main`.

## Evidence

- **`gateway start` still aborts on any `isLoaded()` error:** `runServiceStart()` resolves service-loaded state first, and `resolveServiceLoadedOrFail()` converts any thrown `service.isLoaded()` error into `"Gateway service check failed"` and exits without a fallback path.
  - file: [src/cli/daemon-cli/lifecycle-core.ts:97](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/cli/daemon-cli/lifecycle-core.ts#L97)
  - command: `nl -ba src/cli/daemon-cli/lifecycle-core.ts | sed -n '96,108p'; nl -ba src/cli/daemon-cli/lifecycle-core.ts | sed -n '185,200p'`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Linux systemd probe still throws on user-bus failures:** `isSystemdServiceEnabled()` still calls `systemctl --user is-enabled <unit>` and throws `systemctl is-enabled unavailable: ...` for bus-unavailable errors instead of degrading to a non-fatal status.
  - file: [src/daemon/systemd.ts:659](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/daemon/systemd.ts#L659)
