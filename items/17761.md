## Summary

Current main still does not implement the requested shared-secret fallback for `gateway.auth.mode: "trusted-proxy"`. `authorizeGatewayConnectCore` returns trusted-proxy failures before token/password/Tailscale auth, startup validation still rejects trusted-proxy plus shared-token config, and tests/docs continue to codify fail-closed loopback behavior. The related fix PR #17746 is closed unmerged, while #64122 and #67524 remain open rather than shipped.

## Best Possible Solution

Keep #17761 open until a maintainer either lands an intentional trusted-proxy shared-secret fallback design or explicitly decides the fail-closed trusted-proxy contract is final. A complete fix would need coordinated changes in `src/gateway/auth.ts`, `src/gateway/auth-resolve.ts`, startup validation, regression tests, and public docs; otherwise leave the open follow-up PRs/issues to track the remaining design.

## Evidence
