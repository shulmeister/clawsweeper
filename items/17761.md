## Summary

Keep open. Current main still does not implement the requested shared-secret fallback for `gateway.auth.mode: "trusted-proxy"`; the dispatcher returns trusted-proxy failures before token/password/Tailscale auth, startup validation rejects mixed trusted-proxy plus shared-token config, and tests/docs intentionally codify fail-closed loopback behavior. The discussion also includes a same-day maintainer review explicitly keeping the item open, and the related open PR/issue context is not a merged or canonical shipped replacement.

## Best Possible Solution

Keep #17761 open until a maintainer either lands an intentional trusted-proxy fallback design or explicitly decides the fail-closed contract is the final product direction. A real fix would need coordinated changes in `src/gateway/auth.ts`, `src/gateway/auth-resolve.ts`, startup validation, regression tests, and public docs; otherwise close only after a canonical merged PR or issue clearly supersedes this one.
