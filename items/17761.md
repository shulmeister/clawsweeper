## Summary

Current main still does not implement the requested trusted-proxy shared-secret fallback. The gateway auth dispatcher returns trusted-proxy failures before token/password/Tailscale branches, startup validation rejects trusted-proxy plus shared token, tests assert valid local direct tokens are rejected, and docs describe the current fail-closed contract. The provided GitHub context also shows production impact reports, #17746 closed unmerged, and #64122 still related open work rather than a shipped fix.

## Best Possible Solution

Keep #17761 open until maintainers either land an intentional trusted-proxy shared-secret fallback or explicitly decide the current fail-closed contract is final. A complete fix would need coordinated changes in `src/gateway/auth.ts`, `src/gateway/auth-resolve.ts`, startup validation, regression/e2e tests, and `docs/gateway/trusted-proxy-auth.md`, likely by continuing or superseding #64122.
