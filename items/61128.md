## Summary

Keep open. The issue discussion points to PR #59547 as the implementation, but that PR is closed and unmerged in the provided GitHub context, and current main still appears to scan/read/parse skills per load without the requested in-memory cache, metadata validation, pruning, TTL, or LRU-style eviction.

## Evidence

- **Current local loader has no cache state:** `loadSkillsFromDirSafe` resolves the root, calls `loadSingleSkillDirectory`, enumerates candidate directories, and builds a fresh result map each call. `readSkillFileSync` opens and reads `SKILL.md` directly before parsing frontmatter; there is no cache key, metadata check, TTL, or eviction path in this module.
  - file: [src/agents/skills/local-loader.ts:25](https://github.com/openclaw/openclaw/blob/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54/src/agents/skills/local-loader.ts#L25)
