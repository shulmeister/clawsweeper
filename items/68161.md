## Summary

Keep open. Current main still has the stale optional `hello-ok.auth` schema and optional generated Swift models, while the gateway handler already sends `auth` unconditionally. Related PR #68227 covers the same remaining work and appears broader, but it is also open and not a merged or maintainer-declared canonical replacement, so this useful focused PR should not be auto-closed.

## Best Possible Solution

Keep the PR open for maintainer review or consolidate it with #68227. The final landed change should require `HelloOkSchema.auth`, regenerate both Swift protocol models, update Swift call sites/tests, and make protocol docs/examples explicit that `hello-ok.auth.role` and `hello-ok.auth.scopes` are always present while `deviceToken` remains conditional.
