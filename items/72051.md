## Summary

Keep PR #72051 open. Current main still lacks both substantive fixes from the PR: mirrored TaskFlow terminal records can still store `endedAt < updatedAt`, and the packaged build still does not emit `task-registry-control.runtime` as a stable root dist entry. The PR is recent, focused, and addresses remaining work also reflected by open issue #68997; the existing review comment is an actionable revision request, not a cleanup-close signal.

## Best Possible Solution

Keep the PR open and request a small revision: preserve historical terminal `endedAt` when no newer event timestamp exists, clamp only cases that would otherwise violate `endedAt >= updatedAt`, keep the stable `task-registry-control.runtime` dist entry, and retain/add regression tests for both the `lastEventAt > endedAt` mirror case and the legacy/no-`lastEventAt` retention case. This also remains a plausible fix path for the related open packaging issue #68997.
