## Summary

Keep open. Current main still lacks the core memory-wiki durability fixes requested here: lint still does raw wikilink extraction and path-only target checks, stale report links are emitted as vault-root-relative native Markdown, bridge render fingerprints lack a renderer-version token, and imported-source skip logic still trusts file existence rather than validating source-page frontmatter. The related #65197 PR contained the proposed fix set but was closed unmerged; #70986 covers a broader lint parser/resolver subset but does not cover the bridge cache/render invalidation criteria in this issue.

## Best Possible Solution

Keep this issue open and implement the bounded memory-wiki fixes in the plugin: make wikilink extraction markdown-aware, normalize and resolve native Markdown links relative to the containing page, generate report links relative to reports/stale-pages.md in native mode, sanitize or otherwise prevent reply directive tags in bridged source content from becoming lint targets, add a renderer-version token to bridge render fingerprints, and validate existing imported source pages before skip. Add regression tests for each acceptance criterion and coordinate the parser/resolver portion with #70986 rather than closing this issue prematurely.
