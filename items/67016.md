## Summary

Keep #67016 open. Current main exposes some Dreaming status, including counts, next scheduled run, and Dream Diary content, but it still does not surface last run timestamp, run result/partial/failure details, duration/error, or diary update/workspace status in the Dreaming UI.

## Best Possible Solution

Extend `doctor.memory.status` and the Control UI `DreamingStatus` model to include the managed sweep cron state (`runningAtMs`, `lastRunAtMs`, `lastRunStatus`, `lastError`, `lastDurationMs`, and next run), plus diary metadata such as workspace/path and `updatedAtMs`; then render a compact Last run / Result / Diary updated / Next run status block in `ui/src/ui/views/dreaming.ts` with focused regression tests.
