## Summary

Keep PR #40209 open. Current main still has the case-sensitive system-event queue that the PR is trying to fix: events are stored and looked up by the raw trimmed session key, while targeted heartbeat resolution canonicalizes agent session keys to lowercase before peeking pending events. The linked issue #34338 was closed stale, but the source-level bug shape remains present on main.

## Best Possible Solution

Keep PR #40209 open for maintainer review or landing. The best code direction is to make system-event session-key canonicalization match heartbeat/session routing and add focused regression tests for mixed-case channel session lookup and cross-case deduplication. If maintainers are concerned about arbitrary non-agent keys, adjust the implementation to canonicalize only agent-scoped/session-store keys rather than closing the PR as stale.
