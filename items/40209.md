## Summary

Keep open. Current main still has the case-sensitive system-event queue described by PR #40209: `requireSessionKey()` returns the trimmed session key without lowercasing, while heartbeat/session routing canonicalizes agent session keys to lowercase before peeking pending events. The linked issue #34338 was closed stale, but the underlying code shape remains present and the PR's targeted mixed-case tests are not on main.

## Best Possible Solution

Keep PR #40209 open for maintainer review or landing. The best code direction is the scoped fix proposed here: normalize `requireSessionKey()` to the same lowercase canonical form used by session routing, with focused regression tests for mixed-case channel session lookup and deduplication. If maintainers prefer a broader heartbeat session-key cleanup, this PR should be folded into that work rather than closed as stale.
