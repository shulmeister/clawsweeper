## Summary

Keep this PR open. Current main still keeps system-event queues keyed by the raw trimmed session key, while heartbeat session resolution canonicalizes forced agent session keys to lowercase before peeking/consuming system events. The PR remains a focused implementation candidate for the linked channel-session heartbeat failure.

## Best Possible Solution

Keep this PR open for maintainer review or landing. The best path is to align system-event session-key canonicalization with heartbeat/session routing and keep the mixed-case channel lookup plus cross-case dedupe regression tests; if broad lowercasing is considered too wide, adjust the patch to canonicalize only recognized agent-scoped session keys.
