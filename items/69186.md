## Summary

Keep #69186 open. Current main has a public `agent_end` hook and generic Control UI Web Push subscription/test plumbing, but it does not implement the requested user-facing Control UI completion sound/notification setting, does not wire `chat` terminal events to Web Audio or browser notifications, and exposes no `gateway.webchat` or browser-local setting for completion sound or custom sound URL. The related #59199 context positively confirms the lower-level hook exists, but that does not satisfy this issue's central Control UI UX request.

## Best Possible Solution

Keep this open for a focused Control UI UX implementation: add a browser-local completion notification setting, fire it from terminal `chat` events for the active run, respect browser audio/notification permission rules, optionally reuse existing Web Push only if product wants background/PWA notifications, and add targeted UI/controller tests plus docs. If maintainers want this outside core, first add an explicit browser/UI extension API or supported push-notification SDK contract; the existing `agent_end` hook alone is not enough to close this request.
