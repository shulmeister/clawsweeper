## Summary

Keep open. The issue body is outdated about Markdown vs HTML, but the requested feature is still not implemented on current `main`: Telegram outbound delivery is still hardcoded to `parse_mode: "HTML"`, the public Telegram config schema/types do not expose a `parseMode` option, and the docs still describe HTML as the fixed outbound mode.

## Evidence
