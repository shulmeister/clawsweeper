## Summary

Current main still does not implement #10944. Telegram outbound formatting remains hard-coded to Bot API HTML mode across text sends, captions, edits, and stream previews; the Telegram config schema/types/docs expose markdown table and delivery knobs but no `parseMode`, `MarkdownV2`, or plain-text setting. The issue discussion correctly updates the original Markdown claim to HTML while preserving the actionable request, and related #10758 is a closed custom-emoji/raw-HTML request rather than a channel/account parse-mode config.

## Best Possible Solution

Keep #10944 open until Telegram has a typed parse-mode config surface, ideally top-level plus per-account override if maintainers want that granularity. The implementation should wire the setting through normal text sends, captions, edits, draft/preview delivery, and plain fallback behavior, with tests covering HTML, MarkdownV2 or Markdown if supported, and plain-text/no-parse-mode sends.
