## Summary

Keep open. Current main still hard-codes Telegram formatted outbound delivery to Bot API `parse_mode: "HTML"`, docs describe HTML as the active behavior, and the Telegram config schema/types do not expose a `parseMode`/`MarkdownV2`/plain-text setting. The issue discussion correctly notes the original Markdown claim is outdated, but the remaining request for configurable parse mode is still actionable and not superseded by the related custom-emoji issue.

The maintainer/comment discussion says the original `Markdown` claim is outdated because current OpenClaw uses HTML, but explicitly says the request is still valid and adds a MarkdownV2/list-spacing use case.
