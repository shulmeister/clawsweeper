## Summary

Keep open. Current main still does not implement #10944: Telegram outbound formatting is still wired to Bot API HTML mode, while the Telegram config schema/types/docs expose related delivery knobs but no channel or per-account `parseMode`, `MarkdownV2`, or plain-text setting. The issue discussion remains actionable because it already corrected the original Markdown claim to current HTML behavior, and related #10758 is a separate closed custom-emoji/raw-HTML request rather than a parse-mode config tracker.

## Best Possible Solution

Keep #10944 open and implement this in the bundled Telegram plugin/config surface. The best path is a typed `channels.telegram.parseMode` setting, plus per-account override if maintainers want that granularity, wired through normal sends, media captions, edits, draft/stream preview materialization, and Telegram action delivery. Tests should cover HTML, MarkdownV2 or supported Markdown mode, and plain/no-parse-mode sends, with docs and generated config metadata updated.
