## Summary

Keep #70512 open. Current main still routes Feishu/Lark DMs by sender id only, even though inbound events carry root/thread ids and group-topic routing already has separate session-key logic. The related #10242 ClawSweeper close covers Feishu DM "new thread" send capability, not automatic per-topic DM session isolation, so it is not a canonical duplicate or implemented replacement.

## Best Possible Solution

Keep the issue open and implement it in the bundled Feishu plugin rather than closing as cleanup. A conservative path is to add an opt-in Feishu DM session-scope config, keep today's per-user behavior as the default, and when enabled derive the direct peer/session id from sender plus the canonical DM topic/root/thread id. Cover p2p events with root_id, thread_id, both fields, and no-topic fallback; update Feishu docs/config metadata and any session/conversation binding helpers affected by the new direct conversation id shape.
