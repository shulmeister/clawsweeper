## Summary

Keep this issue open. Current main still routes Feishu/Lark direct messages by sender open ID only; inbound root/thread fields are parsed, but topic-aware session routing, thread context, config, docs, and tests are only implemented for group/topic-group sessions.

## Best Possible Solution

Keep this issue open. Implement the feature in the bundled Feishu plugin with an explicit opt-in DM topic/session-scope config, derive direct-message peer/conversation ids from sender plus the canonical DM topic/root/thread id, preserve the current sender-scoped default, propagate DM topic context consistently into routing/bindings/replies, and add focused p2p/private tests plus config schema and Feishu docs updates.
