## Summary

Current `main` no longer boots the runtime plugin registry for `config.get` / `config.schema` requests, and the gateway startup loader reuses cached registries for the same scope. That removes the repeated `register(api)` path described in #56522. The schema-path fix and regression test both trace to commit `fc5920fb51346a682b269fdbeeec0971ae4ae723` on `main`; I could not verify a released tag containing it from this checkout, so `fixedRelease` is left null.

## Evidence

- **Config requests still hit schema loading:** `config.get`, `config.schema`, and `config.schema.lookup` all call `loadSchemaWithPlugins()`, so this surface was the important repeated-call path to check for duplicate plugin registration.
