## Summary

Current `main` still matches the reported inconsistency: the shared `sessions_spawn` tool schema exposes ACP-only fields, the native subagent runtime rejects those fields, and `/subagents spawn` still takes a separate direct path through `spawnSubagentDirect()`. This does not meet the bar for a conservative close.

## Evidence

  - file: [src/agents/tools/sessions-spawn-tool.ts:105](https://github.com/openclaw/openclaw/blob/a5db42862d76fce18e5d7fa57f3907a543e61137/src/agents/tools/sessions-spawn-tool.ts#L105)
