## Summary

Keep open. Current main and v2026.4.24 still route wake hooks to the configured main session only, while PR #70268 targets the still-open #64556 bug with focused gateway changes and tests. The PR should be reviewed/finished, not auto-closed.

## Best Possible Solution

Keep #70268 open and continue review. The right implementation belongs in the gateway hook path: carry `sessionKey` and `agentId` through wake mapping normalization, validate/canonicalize them with the existing hook session policy, wake the same canonical session that receives the event, cover direct and mapped wake routes in tests, and resolve the global-scope review concern before landing.
