## Summary

Keep PR #43611 open. Current main still does not implement the core request across the exec paths: `runExec` still requests UTF-8 strings, `runCommandWithTimeout` and the supervisor child adapter still use default `Buffer.toString()`, and docs still publish a Windows Chinese mojibake workaround. The PR is a focused bug-fix attempt with useful discussion and a user confirmation, but it has unresolved review findings and overlaps with newer open PR #64661 rather than being safely obsolete or implemented on main.

## Best Possible Solution

Keep this PR open for maintainer judgment. The best path is to consolidate the Windows encoding work with #64661 or rework #43611 so it extracts a shared Windows codepage decoder, applies it to `runExec`, `runCommandWithTimeout`, supervisor child stdout/stderr, and node-host without regressing Windows command spawning, and adds focused GBK/GB18030 split-chunk regression tests.
