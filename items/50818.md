## Summary

Keep open. Current main has the narrower #70258 reply-route fallback, but it still does not implement this PR's remaining session-key targeting work: no `resolveEventSessionKey`, no cron-run event-key remapping with `session.mainKey`, and ACP/CLI/node/bash enqueue paths still write events under the original session key. The latest PR update narrowed away the product-sensitive `heartbeat.target: "none"` override, so the remaining patch is still useful and not implemented on main.

## Best Possible Solution

Keep #50818 open. Review the latest slimmed PR against current main and validate the remaining #52305 scenarios: cron-run enqueue/wake key alignment, custom `session.mainKey`, bash exec, gateway node exec events, ACP stream events, and CLI watchdog events. Leave `heartbeat.target: "none"` behavior internal-only unless maintainers explicitly reopen that product decision in a separate focused change.
