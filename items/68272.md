## Summary

Current `main` does not match the reported bug path. MiniMax M2.7 is still a text-only chat model in OpenClaw, image understanding is routed through the separate `MiniMax-VL-01` media provider, and Gateway text-only attachment handling now offloads images instead of dropping them.

## Evidence

- **MiniMax chat catalog is text-only:** The bundled MiniMax provider catalog builds `MiniMax-M2.7` / `MiniMax-M2.7-highspeed` with `input: ["text"]`, not native image input.
  - file: [extensions/minimax/provider-catalog.ts:49](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/extensions/minimax/provider-catalog.ts#L49)
  - command: `nl -ba extensions/minimax/provider-catalog.ts | sed -n '49,86p'`
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Docs align with separate MiniMax vision model:** The MiniMax docs state that image understanding is registered separately from the text catalog and uses `MiniMax-VL-01` for both `minimax` and `minimax-portal`.
  - file: [docs/providers/minimax.md:326](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/docs/providers/minimax.md#L326)
  - command: `nl -ba docs/providers/minimax.md | sed -n '326,338p'`
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Gateway offloads text-only image attachments:** Current Gateway handling appends `[media attached: media://inbound/... ]` and logs `Offloaded image for text-only model` when `supportsImages === false`; it does not drop the attachment on this path.
  - file: [src/gateway/chat-attachments.ts:423](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/gateway/chat-attachments.ts#L423)
  - command: `nl -ba src/gateway/chat-attachments.ts | sed -n '392,458p'`
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Tests assert offload rather than drop:** `parseMessageWithAttachments` is covered by a test that explicitly expects text-only models to offload images instead of dropping them.
  - file: [src/gateway/chat-attachments.test.ts:146](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/gateway/chat-attachments.test.ts#L146)
  - command: `nl -ba src/gateway/chat-attachments.test.ts | sed -n '146,176p'`
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **MiniMax primary still auto-pairs to vision provider:** Image-tool and media-default resolution code pair a MiniMax primary with `MiniMax-VL-01`, and they also honor configured provider models whose `input` includes `image`.
  - file: [src/agents/tools/image-tool.ts:124](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/agents/tools/image-tool.ts#L124)
