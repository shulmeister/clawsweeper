## Summary

Keep this issue open. Current main still does not implement the documented Discord elevated fallback from `channels.discord.allowFrom`, `sandbox explain` still bypasses that fallback path, and PR #53259 remains an open implementation candidate linked to this issue.

## Best Possible Solution

Review and either land, revise, or close PR #53259. The fix should add the Discord-owned `channels.discord.allowFrom` fallback to the elevated authorization path and reuse the same resolution in `sandbox explain`, with regression coverage for omitted `tools.elevated.allowFrom.discord`, explicit empty override behavior, and diagnostics consistency.
