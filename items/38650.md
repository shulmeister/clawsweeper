## Summary

Keep PR #38650 open. Current main still does not implement the requested behavior for symlinked workspace bootstrap files, especially the linked reproduction where SOUL.md points to a shared file outside the workspace. A newer narrow PR (#40230) overlaps, but it is still open and not an accepted maintainer-canonical replacement, so this stale PR should not be conservatively closed yet.

## Best Possible Solution

Keep this open until a maintainer lands a current-main fix or explicitly closes it in favor of a canonical successor such as #40230. The best implementation path is to define the intended symlink policy for workspace bootstrap files, then add targeted coverage for in-workspace symlink aliases, outside-workspace shared bootstrap files, symlink loops, hardlink rejection, and Gateway agents.files list/get behavior without weakening fs-safe defaults globally.
