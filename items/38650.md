## Summary

Keep PR #38650 open. Current main still does not consistently allow symlinked workspace files: bootstrap loading now uses a separate boundary-file reader, but Gateway workspace file list/get paths still treat symlinked allowlisted files as missing or unsafe. The PR also remains useful discussion because the author addressed the earlier dirty-branch review by removing the accidental log and unrelated systemd change, but the remaining fix needs rebasing/narrowing against the newer symlink policy.

## Best Possible Solution

Keep this open until a maintainer either lands a narrower current-main fix or closes it in favor of a canonical successor such as #40230. The best implementation path is to define the intended workspace-file symlink policy, then update the actual workspace bootstrap and Gateway list/get surfaces with focused tests instead of broadly weakening default fs-safe behavior.
