## Summary

Keep open. Current main has internal effective context-token plumbing, including an agents.defaults.contextTokens cap used for embedded runtime behavior, but the public plugin hook contract still does not expose a native or effective context limit on llm_output or model_call_* events. The related #64260 is still an open adjacent request rather than a shipped or superseding implementation.

## Best Possible Solution

Keep this issue open. The best fix is an additive plugin hook contract update that exposes both native model context metadata and the resolved effective context limit, likely sourced from the existing ctxInfo.tokens/effectiveModel path, on llm_output and the sanitized model_call_started/model_call_ended telemetry. Update src/plugins/hook-types.ts, embedded/CLI/Codex hook emitters, docs/plugins/hooks.md, and focused hook tests. Coordinate with #64260 so one hook-telemetry change covers native context window plus effective contextTokens rather than landing two divergent field names.
