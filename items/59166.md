## Summary

Closing as `cannot_reproduce`: current `main` does not append full subagent completion output to chat `/status`. The current implementation only adds compact task/subagent summary lines, hides stale completed tasks, sanitizes internal completion markers, and truncates long task text. A related but different behavior still exists in CLI `openclaw status`, which can show queued system events.

## Evidence

- **`/status` stays on the chat agent path:** The UI test confirms `/status` is not a local slash command override; it is handled by the agent status flow, so the relevant behavior is the current `/status` status-card implementation.
  - file: [ui/src/ui/chat/slash-commands.node.test.ts:44](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/ui/src/ui/chat/slash-commands.node.test.ts#L44)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Current status builder only adds compact task/subagent summary lines:** `buildStatusText()` derives a single `taskLine` and a single `subagentsLine` for the session status card; it does not append queued completion payloads or full subagent results.
  - file: [src/status/status-text.ts:248](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/status/status-text.ts#L248)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Subagent line is a summary, not full completion output:** The subagent status formatter returns either `🤖 Subagents: N active` or a verbose summary like `· X done`; there is no path here that emits full completion bodies into `/status`.
  - file: [src/auto-reply/reply/commands-status-subagents.ts:4](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/auto-reply/reply/commands-status-subagents.ts#L4)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Task status intentionally suppresses stale completions and internal completion details:** Task status only keeps recent terminal tasks within a 5-minute window, truncates titles/details, and strips internal completion-context markers such as `[Internal task completion event]`.
  - file: [src/tasks/task-status.ts:11](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/tasks/task-status.ts#L11)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Tests cover the exact reported mismatch:** `/status` tests assert that stale completed tasks are omitted, recent failures are summarized, and long task text is truncated rather than dumped verbatim into the status output.
  - file: [src/auto-reply/reply/commands-status.test.ts:274](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/auto-reply/reply/commands-status.test.ts#L274)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Related but separate CLI behavior exists:** The TUI gateway status summary can include a `Queued system events` line, which matches the maintainer comment's narrower follow-up area, but that is distinct from chat `/status`.
  - file: [src/tui/tui-status-summary.ts:84](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/tui/tui-status-summary.ts#L84)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Issue discussion already identified the mismatch:** The April 1, 2026 maintainer comment by `sahilsatralkar` said the report no longer matched current main and suggested any remaining UX request should be reframed around a narrower chat-minimal or CLI-system-events follow-up.
