## Summary

Keep #38991 open. Current main still mitigates the original macOS `cron.list` decode failure by skipping malformed jobs on the app side, but it does not implement this PR's gateway-side repair of missing legacy `createdAtMs` and `updatedAtMs` fields during cron store load, so the PR remains behaviorally useful after cleanup/rebase.

## Best Possible Solution

Keep this PR open or replace it with an equivalent scoped patch: repair missing/non-finite `createdAtMs` and `updatedAtMs` during cron store/service load, persist the repaired store only when mutation occurs, and extend the regression to prove both `cron.list` and the persisted store expose numeric timestamps. Drop unrelated `.secrets.baseline` and Fastfile churn before merge.
