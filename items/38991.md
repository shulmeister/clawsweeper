## Summary

Keep #38991 open. Current main has a macOS-side lossy decode workaround for malformed cron.list jobs and the base store loader backfills only updatedAtMs in some paths, but it still does not implement this PR's main behavior: repairing missing createdAtMs and updatedAtMs during cron service load so legacy jobs remain visible and decodable instead of being skipped.

## Best Possible Solution

Keep the PR open, but require a cleaned and rebased implementation focused on cron timestamp repair only. The best code path is to backfill finite non-negative createdAtMs and updatedAtMs during cron store/service load before cron.list returns jobs, persist the repaired shape according to the current jobs.json/jobs-state.json split, and extend the regression tests so both the in-memory cron.list result and persisted store/sidecar carry numeric timestamps. Remove the unrelated .secrets.baseline and apps/ios/fastlane/Fastfile churn before review or merge.
