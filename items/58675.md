## Summary

Keep open. Current main still does not auto-discover GitHub Copilot chat models; it only returns an empty implicit catalog plus manual forward-compatible model resolution. The PR remains actionable, but should be narrowed/rebased because the branch also carries unrelated agent/session and live fixture/test changes.

## Best Possible Solution

Keep the item open, but ask for a focused rebase/split. The best path is a narrow GitHub Copilot provider change that adds `/models` catalog discovery inside `extensions/github-copilot`, reuses `resolveCopilotTransportApi`, forwards IDE and configured headers, respects configured baseUrl and discovery disable flags, skips explicit model lists, sorts deterministically, and covers the behavior in provider discovery/discovery unit tests. Unrelated agent/session and live fixture changes should move to separate PRs.
