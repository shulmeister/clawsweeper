## Summary

Keep this PR open. Current main still has the stale legacy-settings fallback described by the PR: when the page-derived scoped key is absent, `loadSettings()` can read the shared legacy key and preserve a sibling base-path gateway URL. The PR is a focused attempt at a real web UI storage bug and is not implemented or superseded on main.

## Best Possible Solution

Keep the PR open and ask for a maintainer-quality revision or equivalent main patch: add targeted storage tests for `/a` to `/b` legacy fallback, preserve intentional same-host different-path gateway overrides such as `/ws`, keep endpoint changes persistent after reload, and avoid regressing per-gateway session token isolation. If the branch is not accepted as-is, land the same behavior in `ui/src/ui/storage.ts` with those regression tests.
