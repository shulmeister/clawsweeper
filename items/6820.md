## Summary

Keep #6820 open. Current main still does not expose `xhigh` for `openai-codex/gpt-5.2`: the OpenAI Codex provider recognizes base `gpt-5.2` as a modern model, but the xhigh allowlist only includes newer/base 5.4+ ids plus codex-suffixed 5.3/5.2/5.1 ids. The previous fix PRs linked from the issue are closed unmerged, and the latest maintainer review comment already kept the item open.

## Best Possible Solution

Keep #6820 open until the OpenAI Codex provider either adds base `gpt-5.2` to `OPENAI_CODEX_XHIGH_MODEL_IDS` with focused regression coverage for `listThinkingLevels("openai-codex", "gpt-5.2")` or `resolveThinkingProfile`, or maintainers explicitly document that base OpenAI Codex GPT-5.2 is high-only while `gpt-5.2-codex` is xhigh-capable. The broader RFC #37804 can track systemic model-capability drift, but it does not supersede this concrete missing provider/model case.
