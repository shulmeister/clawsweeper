## Summary

Keep #6820 open. Current main still recognizes `openai-codex/gpt-5.2` as a modern OpenAI Codex model, but the OpenAI Codex provider xhigh allowlist omits base `gpt-5.2`, so `resolveThinkingProfile` will not expose `xhigh` for that provider/model pair. The one-off fix PRs referenced in the issue were closed unmerged, and the broader RFC #37804 is related but does not supersede this concrete missing-model request.

## Best Possible Solution

Keep #6820 open until the OpenAI Codex provider either adds base `gpt-5.2` to `OPENAI_CODEX_XHIGH_MODEL_IDS` with focused regression coverage for `listThinkingLevels("openai-codex", "gpt-5.2")` or `resolveThinkingProfile`, or maintainers explicitly document that base OpenAI Codex GPT-5.2 is high-only while `gpt-5.2-codex` is xhigh-capable. Treat #37804 as the broader design discussion, not as the canonical replacement for this concrete missing model case.
