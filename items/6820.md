## Summary

Keep #6820 open. Current main still does not expose `xhigh` for base `openai-codex/gpt-5.2`: the OpenAI Codex provider advertises `xhigh` through an explicit model-id list that includes `gpt-5.2-codex` but not `gpt-5.2`, while the same provider still recognizes base `gpt-5.2` as a modern Codex model elsewhere. The related PRs that proposed the exact allowlist/test fix (#6013 and #43054) were closed unmerged, and the current test coverage still lacks the requested `openai-codex/gpt-5.2` xhigh case.

## Best Possible Solution

Keep #6820 open until the OpenAI Codex provider either explicitly adds base `gpt-5.2` to `OPENAI_CODEX_XHIGH_MODEL_IDS` with focused regression coverage for `openai-codex/gpt-5.2`, or maintainers document that base Codex GPT-5.2 should remain high-only while `gpt-5.2-codex` is xhigh-capable.
