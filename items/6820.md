## Summary

Keep #6820 open. Current main still does not expose `xhigh` for base `openai-codex/gpt-5.2`: the OpenAI Codex provider xhigh allowlist includes nearby Codex models like `gpt-5.2-codex`, but omits base `gpt-5.2`; the profile gate depends directly on that allowlist; and the prefix helper does not make `gpt-5.2` match `gpt-5.2-codex`. The related fix PRs #6013 and #43054 were closed unmerged, and current tests still lack the requested `openai-codex/gpt-5.2` xhigh case.

## Best Possible Solution

Keep #6820 open until the OpenAI Codex provider either adds base `gpt-5.2` to `OPENAI_CODEX_XHIGH_MODEL_IDS` with focused regression tests for `listThinkingLevels("openai-codex", "gpt-5.2")` and directive/hint behavior, or maintainers explicitly document that base Codex GPT-5.2 should remain high-only while `gpt-5.2-codex` is xhigh-capable.
