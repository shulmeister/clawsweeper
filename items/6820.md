## Summary

Keep #6820 open. Current main still recognizes `openai-codex/gpt-5.2` as a modern OpenAI Codex model, but the OpenAI Codex provider's xhigh profile omits the base `gpt-5.2` id and only includes `gpt-5.2-codex` for the 5.2 family. The active thinking profile gate uses exact-or-prefix matching, so `gpt-5.2` will not match `gpt-5.2-codex`. Related PRs #6013 and #43054 proposed the exact allowlist/test fix but were closed unmerged, so there is no shipped or canonical superseding item to close against.

## Best Possible Solution

Keep #6820 open until the OpenAI Codex provider either explicitly adds `gpt-5.2` to `OPENAI_CODEX_XHIGH_MODEL_IDS` with focused regression coverage for `openai-codex/gpt-5.2`, or maintainers document that the base Codex GPT-5.2 route should remain high-only while `gpt-5.2-codex` is xhigh-capable.
