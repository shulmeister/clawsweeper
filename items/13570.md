## Summary

Keep #13570 open. Current main at 04e08cea62983b2c766d15f61f15b3ffc90f9baa still lacks `contextPruning.forcePruneRatio` or an equivalent high-usage TTL bypass: the config types/schema omit the key, effective context-pruning settings have no parsed bypass threshold, and the runtime extension returns during a fresh TTL window before context-usage pruning runs. The related PR #13572 directly matched the request but was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open until a maintainer either implements or explicitly rejects the product direction. The likely implementation is to add `agents.defaults.contextPruning.forcePruneRatio` to config types, Zod/generated schema, effective settings, and docs; estimate current context usage before the fresh-TTL return in `src/agents/pi-hooks/context-pruning/extension.ts`; bypass TTL only when the configured ratio is met; and add regression coverage proving fresh TTL still prunes at high usage while normal TTL behavior remains unchanged below the threshold.
