## Summary

Keep #13570 open. Current main still does not implement `contextPruning.forcePruneRatio` or any equivalent high-usage TTL bypass: the config type/schema/docs omit the key, runtime settings do not store a bypass threshold, and the context-pruning extension still exits before pruning whenever the cache TTL is fresh. The related PR #13572 is closed unmerged, and the referenced PR #13512 is also closed unmerged and addressed a different `stripThinking` feature.

## Best Possible Solution

Keep #13570 open until a maintainer implements or explicitly declines the feature. The likely fix belongs in the existing context-pruning path: add `agents.defaults.contextPruning.forcePruneRatio` to config types, strict schemas, generated schema, and docs; carry it through `computeEffectiveSettings()`; estimate context usage before the fresh-TTL early return; bypass TTL only when the configured threshold is met; preserve the current TTL gate below threshold; and add tests for above-threshold bypass and below-threshold preservation.
