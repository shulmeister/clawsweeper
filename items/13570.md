## Summary

Keep #13570 open. Current main still has no `contextPruning.forcePruneRatio` config field or equivalent high-usage TTL bypass, and the context-pruning extension still returns during a fresh TTL window before pruning runs. The related PR #13572 was closed unmerged in the provided GitHub context, so it remains useful reference material but not a shipped fix.

## Best Possible Solution

Keep #13570 open until a maintainer implements or explicitly declines the feature. A likely implementation belongs in `agents.defaults.contextPruning`: add `forcePruneRatio` to the public type, strict schema, generated schema/help, and docs; carry it through effective runtime settings; estimate context usage before the fresh-TTL early return; bypass TTL only at or above the configured threshold; and add focused regression tests. PR #13572 can be used as reference, but it was closed unmerged and is not current main behavior.
