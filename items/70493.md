## Summary

Keep #70493 open. Current main still has the browser route paths that can return the reported Playwright-unavailable failure for managed browser navigation/snapshot/act, and I did not find the isolated-session regression coverage that the maintainer comment explicitly required before closure.

## Best Possible Solution

Keep the issue open. The best fix is to make isolated sub-agent browser calls use the same capable browser runtime as the main session, either by ensuring bundled browser plugin runtime deps are available in the isolated gateway or by explicitly proxying browser operations to the host/node browser owner, then add an isolated-session regression test for `browser` `navigate` or `snapshot` that proves the Playwright-unavailable error is gone.
