## Summary

Keep this issue open. Current main still contains the Playwright-unavailable response path and the browser tool still routes sandboxed/sub-agent contexts through distinct sandbox or host/node browser targets. I did not find isolated sub-agent navigate/snapshot/act regression coverage, and the maintainer discussion explicitly says not to close until that coverage exists.

## Best Possible Solution

Keep this issue open. The best fix is to make isolated sub-agent browser actions use a Playwright-capable runtime consistently, either by packaging/loading the browser plugin runtime in the isolated gateway or by explicitly delegating browser actions to the host/node browser owner when that is the intended architecture. Add a regression test that exercises navigate or snapshot through the real isolated sub-agent path and proves it does not return the Playwright-unavailable error when the main gateway or browser node supports Playwright.
