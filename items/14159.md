## Summary

Current `main` already has platform-enforced pre-execution hook boundaries for tool calls, approvals, subagent spawning, and reply interception, which covers the enforcement primitive requested here even though the exact `preToolHooks` JSON shape is not present.

## Close Comment

Closing from a Codex review because current `main` already provides the enforced pre-execution hook boundary this request was asking for. Agent tools are wrapped through `wrapToolWithBeforeToolCallHook(...)` before execution (`src/agents/pi-tools.ts:704`), the hook can block or require approval before the underlying tool runs (`src/agents/pi-tools.before-tool-call.ts:187`, `src/plugins/hooks.ts:777`, `docs/plugins/building-plugins.md:184`), and the MCP/plugin tool bridge goes through the same boundary (`src/mcp/plugin-tools-handlers.ts:21`, `src/mcp/plugin-tools-serve.test.ts:92`). For the related use cases mentioned here, `subagent_spawning` can fail a subagent launch before binding proceeds (`src/agents/subagent-spawn.ts:494`) and `before_agent_reply` can intercept replies before normal output continues (`src/plugins/hooks.ts:566`). The exact `preToolHooks` JSON shape from the issue is not present, but the non-voluntary platform-level enforcement primitive is now on `main`, so this request is implemented well enough to close.
