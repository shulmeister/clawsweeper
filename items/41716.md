## Summary

Keep #41716 open. Current main still does not implement the requested gateway ↔ node file-transfer feature: there are no `file_pull`/`file_push` actions in the `nodes` agent tool, no `file.read`/`file.write` command constants or gateway policy entries, no CLI file helper, and the headless node host still only handles plugin commands plus `system.run`/`system.run.prepare`. The linked implementation PR #67495 is open and unmerged in the provided GitHub context, and the reporter explicitly asked to keep the issue open until that PR lands.

## Best Possible Solution

Keep #41716 open as the tracking issue while maintainers review PR #67495. The best path is to land that PR or an equivalent mainline implementation that adds the `file.read`/`file.write` node-command contract, gateway policy/security gating, `nodes` tool `file_pull`/`file_push` actions, node-side handling where required, and focused tests/docs; then close the issue from the merged change.
