## Summary

Keep #23906 open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still has no per-call sessions_spawn tool selector for tools:"none" or a named subset; the spawn schema, native spawn params, and gateway agent RPC do not expose or transport disableTools/toolsAllow. The related #15032 is real overlap but remains open and does not fully cover this issue's local-model tool-less spawn plus plugin/MCP inheritance details; the provided same-day maintainer-side Codex review also explicitly kept #23906 open.

## Best Possible Solution

Keep #23906 open until maintainers implement or explicitly reject a backward-compatible per-spawn tool selection contract. A concrete implementation should add a sessions_spawn tools:"none"|string[] field or explicit disableTools/toolsAllow fields, validate and document it, carry it through SpawnSubagentParams and the gateway agent RPC, map tool-less spawns to runner disableTools:true, map subsets to toolsAllow, and add tests for core tools, plugin/bundle MCP tools, global subagent policy precedence, sandbox/security behavior, and small local-model tool-less spawns. Keep #15032 related, but do not close #23906 as duplicate unless maintainers choose one canonical tracking issue for both security isolation and local-model/plugin inheritance requirements.
