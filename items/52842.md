## Summary

Keep open. Current `main` still does not expose `stringEnum`/`optionalStringEnum` from the root `@openclaw/plugin-sdk` surface, while `core` still exports them. The current docs and changelog also do not document this specific root-surface migration, so the reported compatibility/documentation gap is not clearly resolved on `main`.
