## Summary

Keep #69835 open. Current main has not implemented the PR’s two central behavior changes: explicit gateway `workspaceDir` forwarding is still rejected by protocol validation, and missing bootstrap files are still surfaced as `[MISSING]` context. The PR is focused enough to remain useful, but it needs revision before merge because review comments identified an unguarded workspace path override and a stale test expectation.

## Best Possible Solution

Keep the PR open for revision. The gateway half should either keep explicit `workspaceDir` rejected or accept it only through a clearly trusted internal/admin path with tests proving unauthorized callers cannot use it. The bootstrap-noise half should update all missing-file expectations consistently, including system prompt stability and cache tests, or be split into a smaller PR if maintainers want to review that behavior separately.
