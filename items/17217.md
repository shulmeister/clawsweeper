## Summary

Keep #17217 open. Current main documents session lifecycle and has manual reset/compaction surfaces, but the central requested behavior is still missing: `contextPruning.mode: "cache-ttl"` does not wall-clock expire all conversation messages older than the TTL, and there is no explicit `prune-context` session-control API.

## Best Possible Solution

Keep the issue open for maintainer product and implementation work. A complete resolution should decide whether `contextPruning.ttl` is meant to expire full conversation context or only trim tool-result cache bloat. If full TTL pruning is desired, implement it in the Pi/context-engine/session-control surface with explicit manual prune/reset controls, tests for mid-session TTL expiry, and docs for the chosen semantics. If not, close only after an explicit maintainer decision and docs that say TTL does not expire normal conversation messages.
