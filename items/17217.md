## Summary

Current main still does not implement the requested wall-clock TTL pruning of normal conversation history or a manual prune-context/session-prune API. Session boundaries are documented around daily, idle, and manual `/new`/`/reset`, while `contextPruning.ttl` remains a cache-touch cadence gate for trimming old tool results. A prior steipete maintainer review comment also explicitly kept this issue open, and the relevant docs/source paths are unchanged since that review.

## Best Possible Solution

Keep the issue open for maintainer product work. Decide whether `contextPruning.ttl` should expire or summarize full conversation history, or whether the current tool-result-only behavior is intentional. If full TTL pruning is desired, add an explicit gateway/protocol session prune surface and implement age-based removal or summarization in the Pi/context-engine path with tests and docs. If it is intentionally out of scope, update user-facing docs to make that limitation unambiguous.
