## Summary

Keep #17217 open. Current main documents session lifecycle and provides manual reset/compaction surfaces, but the central request is still not implemented: `contextPruning.mode: "cache-ttl"` only gates in-memory pruning of old tool results during context assembly, and there is no explicit `prune-context`/`sessions.prune` API that expires or summarizes normal conversation messages by wall-clock TTL mid-session. A maintainer-thread Codex review already reached the same keep-open conclusion on 2026-04-26, and current main still matches that assessment.

## Best Possible Solution

Keep the issue open for maintainer product and implementation work. A complete resolution should decide whether `contextPruning.ttl` is intended to expire full conversation history or only gate tool-result cache pruning. If full TTL pruning is desired, implement an explicit session-control prune surface, add Pi/context-engine behavior that removes or summarizes messages older than the TTL mid-session, and cover it with tests and docs. If it is not desired, update user-facing docs to state clearly that TTL does not expire normal conversation messages and that reset/compaction are the supported controls.
