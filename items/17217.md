## Summary

Keep #17217 open. Current main documents session lifecycle and supports manual reset/compaction, but the central requested behavior is still missing: cache-TTL pruning is opportunistic, in-memory tool-result trimming, not wall-clock removal or summarization of all messages older than the TTL, and there is no `sessions.prune` or `/api/session/prune-context` session-control endpoint.

## Best Possible Solution

Keep the issue open for maintainer product/implementation work. A complete fix would clarify whether TTL is intended to age-prune full conversation context or only tool output, then add the chosen behavior in the Pi/context-engine/session-control surface with docs and tests. If the product direction remains tool-result-only pruning, the issue should be resolved by an explicit maintainer decision and docs explaining that TTL is not a wall-clock full-history expiration policy.
