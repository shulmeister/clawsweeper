## Close Comment

Closing as implemented on current `main`. This Codex review found that the TUI now explicitly matches canonical session-key aliases in `src/tui/tui-event-handlers.ts:265`, so chat events for aliases like `main` and `agent:<id>:main` are treated as the same session instead of being ignored. There is also a regression test for that exact case in `src/tui/tui-event-handlers.test.ts:234`, and `CHANGELOG.md:2938` records the fix as shipped in `v2026.3.7`. Since this report was against `2026.3.3`, the issue appears to have been fixed in a later release. If you still see the problem on a current build, please open a new issue with the exact version, active session key, and a fresh repro/log capture.
