## Summary

Keep #71619 open. Current main still treats Linux version-manager directories as part of the daemon minimal service PATH, and current tests still assert that behavior. The PR remains a focused, live fix for #71578's doctor/minimal-PATH contradiction rather than obsolete, implemented, or superseded work.

## Best Possible Solution

Keep #71619 open for maintainer review or land an equivalent patch. The best fix is to make Linux minimal service PATH include stable user bins, pnpm/NPM prefix, Nix profiles, and system dirs while excluding Linux version-manager roots, preserving current macOS behavior, and updating service-env/service-audit regression coverage for the #71578 hardened-PATH expectation.
