## Summary

Keep this PR open. Current main still implements the Linux daemon minimal PATH policy that this PR is trying to harden: Linux version-manager directories are still treated as expected service PATH entries, tests still assert that behavior, and the public FAQ still documents it. The PR remains a focused implementation candidate for the PATH-audit contradiction described in the linked bug report.

## Best Possible Solution

Review and land this PR or an equivalent patch. The best implementation path is to keep Linux daemon minimal PATH limited to stable user bins, PNPM/NPM prefix, Nix profiles, and system dirs; exclude Linux version-manager directories from required minimal PATH; preserve macOS version-manager behavior; and update daemon service-env/service-audit tests plus the public first-run FAQ to match the hardened Linux policy.
