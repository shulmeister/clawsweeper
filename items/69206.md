## Summary

Keep #69206 open. Current main still generates and advertises dock commands, and Telegram native command dispatch now preserves `/dock_discord` as a native command body, but I found no runtime handler that turns `dock:discord` or `/dock-discord` into a channel handoff. The remaining behavior can still fall through to the normal command/agent path, which is the reported user problem.

## Best Possible Solution

Keep the issue open and either implement a real dock command handler for generated `dock:<channel>` commands or revise/remove the advertised dock commands if the intended product behavior is only command discovery. A fix likely belongs in the shared command-handling layer so Telegram, Discord, Slack, and UI-invoked runtime commands all resolve the same channel handoff semantics, with regression coverage for `/dock_discord` from Telegram native command dispatch.
