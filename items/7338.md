## Summary

Keep #7338 open. Current main still lacks opt-in, domain-scoped OpenClaw agent attestation headers for external HTTP requests. The implemented header work is provider/request-governance plumbing, not the requested shared agent-origin attestation contract across web_fetch, skills, generic fetch, or arbitrary tool HTTP.

## Best Possible Solution

Keep #7338 open as the canonical product/API design tracker. A complete solution should decide whether attestation belongs in core or behind a new plugin/API seam, define opt-in domain-scoped configuration, specify static versus cryptographic header semantics, wire supported outbound HTTP paths consistently, document privacy defaults, and add tests for allowed-domain injection plus non-leakage to other domains.
