## Summary

Keep #7338 open. Current main still lacks an opt-in, domain-scoped OpenClaw agent attestation contract for arbitrary external HTTP requests from web_fetch, skills, or tools. Existing headers are limited to web_fetch's configurable User-Agent or provider/model traffic attribution, and the provided issue discussion already identifies #7338 as the canonical tracker after #11137 was closed as related/duplicate work.

## Best Possible Solution

Keep #7338 open as the canonical tracker. A complete solution should define whether attestation is core behavior or a new plugin/API seam, add opt-in domain-scoped config and validation, specify signed/static header semantics, wire the supported external HTTP paths consistently, document privacy defaults, and add tests for allowed-domain injection plus non-leakage to other domains.

## Risks / Open Questions

- Cryptographic attestation has privacy, tracking, trust-root, expiry, and domain-scoping implications; implementation needs explicit maintainer product/security design rather than conservative cleanup.
