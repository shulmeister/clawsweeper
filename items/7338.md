## Summary

Keep #7338 open. Current main still lacks an opt-in OpenClaw-wide agent attestation contract for arbitrary external HTTP requests, skills, or tools. Existing header mechanisms are scoped to provider/model traffic or fixed web_fetch User-Agent configuration, and the provided discussion identifies #7338 as the canonical tracker after #11137 was closed as a duplicate.

## Best Possible Solution

Keep #7338 open as the canonical tracker. A complete solution should define an opt-in, domain-scoped attestation contract, update config/types/zod schema/docs, decide whether this is core behavior or a new plugin API surface, and add tests for allowed-domain injection, non-leakage to other domains, privacy defaults, and the relevant external HTTP paths.

## Risks / Open Questions

- A service-specific plugin can sign requests it owns, but that does not satisfy the issue's requested OpenClaw-wide attestation across web_fetch, skill execution, and tool calls.
- Cryptographic attestation has privacy, tracking, trust-root, and domain-scoping implications, so the eventual design needs explicit maintainer product/security direction.
