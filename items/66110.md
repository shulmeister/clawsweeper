## Summary

Keep open. Current main still lacks the requested root `--log-file` option and `OPENCLAW_LOG_FILE` logger override, while PR #66110 is a focused fix attempt for open issue #61295. It is not safely closeable as implemented or superseded; the existing PR review comments also identify concrete follow-up needed before this exact patch should land.

## Best Possible Solution

Keep PR #66110 open and ask for a revision: add root `--log-file <path>`, promote it before the route-first fast path as well as Commander preaction, add an `OPENCLAW_LOG_FILE` resolver that can accept an explicit env object, use the daemon service env when reporting status, and keep the docs/tests listed in the PR updated. The linked issue #61295 remains the right product problem to solve; the unrelated log rotation PRs are separate follow-ups.
