## Summary

Keep this PR open. Current main still does not implement the root `--log-file` flag or `OPENCLAW_LOG_FILE` file-sink override; it only has the narrower shipped/configured `logging.file` path behavior. The PR is still a useful implementation candidate, but the provided review context also identifies branch-specific gaps around route-first commands and daemon status resolving the service environment.

## Best Possible Solution

Keep this PR open as the implementation candidate, but require a revision before merge: add root `--log-file` to the root option parser/help, promote it before both Commander and route-first execution paths, add an explicit `OPENCLAW_LOG_FILE` resolver with precedence over `logging.file`, make daemon status resolve against the inspected service environment when available, and keep focused tests/docs for those boundaries.
