## Summary

Current main still has the reported Telegram `/models` callback bug. Provider buttons and pagination emit raw provider IDs such as `hf.co`, but `parseModelCallbackData()` still rejects `.` in `mdl_list_{provider}_{page}` callbacks, so the list handler is bypassed for `mdl_list_hf.co_1`. The related PRs #40251 and #46035 described the same narrow fix, but both are closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep #38745 open and land a narrow Telegram plugin fix: either allow `.` in the `mdl_list` provider parser or parse by `mdl_list_` prefix plus the final page separator. Add regression coverage for `parseModelCallbackData('mdl_list_hf.co_1')`, provider keyboard generation with `hf.co`, and dotted-provider pagination/list callback handling.
