## Summary

Current main still has the reported Telegram `/models` callback bug. Provider buttons and pagination emit raw provider IDs such as `hf.co`, but `parseModelCallbackData()` still rejects `.` in `mdl_list_{provider}_{page}` callbacks, so `mdl_list_hf.co_1` is not handled as a model-list callback. The related PRs #40251 and #46035 targeted the same fix, but the provided GitHub context shows both are closed unmerged.

## Best Possible Solution

Keep #38745 open and land a narrow Telegram plugin fix: parse `mdl_list_` callbacks in a way that supports dotted provider IDs, either by allowing `.` in the provider regex or by splitting after the `mdl_list_` prefix at the final page separator. Add regression coverage for `parseModelCallbackData('mdl_list_hf.co_1')`, provider keyboard generation with `hf.co`, and dotted-provider pagination/list callbacks.
