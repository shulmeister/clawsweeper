## Summary

Current main still has the reported Telegram model-list callback bug. Provider buttons and pagination preserve the raw provider id, but the list callback parser rejects dots, so `mdl_list_hf.co_1` still does not reach the model-list handler. The two related PRs in the provided GitHub context identified the same narrow fix, but both are closed unmerged.

## Best Possible Solution

Keep #38745 open and land a narrow Telegram plugin fix: allow dotted provider ids in `mdl_list` parsing, or parse the callback by prefix plus the final page separator, then add regression coverage for `parseModelCallbackData('mdl_list_hf.co_1')`, provider keyboard generation with `hf.co`, and dotted-provider pagination/callback handling.
