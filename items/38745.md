## Summary

Keep #38745 open. Current main still emits Telegram model-list callback data with raw provider IDs, but `parseModelCallbackData()` still rejects dotted provider IDs such as `hf.co`, so `mdl_list_hf.co_1` cannot reach the Telegram model-list handler. The related fix PRs #40251 and #46035 are closed and unmerged in the provided GitHub context, and their referenced commits are not present in this checkout.

## Best Possible Solution

Keep #38745 open and land a narrow Telegram plugin fix in `extensions/telegram/src/model-buttons.ts`: parse `mdl_list_` callbacks in a way that supports dotted provider IDs, preferably by splitting after the `mdl_list_` prefix at the final page separator or by expanding the provider regex to include `.`. Add regression coverage for `parseModelCallbackData('mdl_list_hf.co_1')`, provider keyboard generation with `hf.co`, and pagination/model-list callback behavior for dotted providers.
