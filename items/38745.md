## Summary

Keep #38745 open. Current main still emits raw Telegram `mdl_list_${provider.id}_1` callbacks but parses list callbacks with a provider regex that excludes dots, so a provider ID such as `hf.co` still cannot reach the model-list button branch. Existing tests cover non-dotted providers only, and the related fix PRs are closed/unmerged in the provided context with their referenced commits absent from this checkout.

## Best Possible Solution

Keep #38745 open and land a narrow Telegram plugin fix in `extensions/telegram/src/model-buttons.ts`. The most robust fix is to parse `mdl_list_` by removing the prefix and splitting at the final `_<page>` separator, or at minimum allow `.` in the provider character class. Add regression coverage for `parseModelCallbackData("mdl_list_hf.co_1")`, provider keyboard generation for `{ id: "hf.co" }`, and paginated model-list callbacks for dotted providers.
