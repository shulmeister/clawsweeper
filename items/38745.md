## Summary

Current main still appears to have the reported Telegram model-list bug. The parser still rejects dots in `mdl_list_*` provider IDs, while provider and pagination buttons still emit the raw provider ID, so `mdl_list_hf.co_1` remains unparseable. Related PRs #40251 and #46035 identified the same fix direction but are closed unmerged in the provided context.

## Best Possible Solution

Keep #38745 open until the Telegram plugin accepts dotted provider IDs in `mdl_list` callback parsing and adds focused regression coverage for `mdl_list_hf.co_1`, provider keyboard generation, and paginated model-list callbacks.
