## Summary

Keep #65799 open. Current main still gives Feishu WebSocket an ambient proxy agent whenever HTTP(S) proxy env is detected, and the current regression tests still assert that behavior. The linked PR #65802 is a plausible fix path, but it is open/unmerged and review discussion already noted a test cleanup issue, so the underlying user-visible bug is not resolved on main.

## Best Possible Solution

Keep the issue open until #65802 or an equivalent patch lands on main: gate Feishu WebSocket proxy use behind an explicit `OPENCLAW_FEISHU_WS_USE_PROXY=1` opt-in, add/adjust regression tests so ambient proxy env is ignored by default and honored only when opted in, restore the new env var in test cleanup, and document the opt-in if it remains a supported user-facing escape hatch.
