## Summary

Keep this issue open. Current main has partial mitigations the reporter’s follow-up noted: compact skills formatting, character/count budgets, and per-agent skill allowlists. But the requested behavior is not implemented: the prompt still injects every visible skill on each run, and compact mode still emits per-skill name plus SKILL.md location rather than a names-only routing list or an automatically selected subset. I found no user-facing `skills.promptMode`, `names-only`, or `auto` prompt visibility setting.

## Best Possible Solution

Keep this issue open and scope it as a skills prompt optimization. The best path is to add a deliberate core prompt visibility mode, likely `names-only` or `auto`, in the skills prompt builder and config/schema docs, with tests proving full, compact, names-only, truncation, and per-agent override behavior. Coordinate with broader prompt-size/cache work such as #14619 and #45062, but do not treat those as superseding this skills-specific request unless a canonical maintainer issue is opened.

## Risks / Open Questions

- A naive keyword matcher could hide relevant skills and regress skill discovery, so an auto mode needs clear fallback behavior.
- Changing skill prompt shape can affect prompt-cache stability and existing tests that assume XML skill entries.
- Per-agent allowlists mitigate prompt size for configured agents, but they are manual policy controls and do not solve the automatic lazy-loading request.
