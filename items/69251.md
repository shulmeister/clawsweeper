## Summary

Current main still appears to have the reported prepare-time denial path for POSIX `exec host=node` commands whose shell payload starts with an existing file path. The related #69308 shell fallback work addressed `/usr/bin/false`/`nologin` shells, but did not remove the `/bin/sh -lc` node transport wrapper or change the stable-binding denial described here.

## Best Possible Solution

Keep this open. The likely fix belongs in the POSIX `exec host=node` prepare/binding path: either stop using an opaque `/bin/sh -lc` transport for file-backed commands, or teach node-side preparation to bind the resolved inner script/interpreter file safely before approval. Add targeted regression tests for `exec host=node` direct script paths and interpreter file operands under `security: "full"` plus `ask: "off"`, while preserving the existing fail-closed behavior for truly unbindable shell payloads.
