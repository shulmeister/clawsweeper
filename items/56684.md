## Summary

Current `main` exposes generic memory-plugin and hook surfaces, but it does not contain any `claude-mem` integration or worker `/api/sessions/init` code. The reported duplicate init paths and observation persistence behavior belong to the external memory pluginâ€™s hook usage and session-mapping logic, so this should close from the OpenClaw core tracker and be handled in the plugin/ClawHub package instead.

## Evidence

- **Vision keeps optional memory capability in plugins, not core:** `VISION.md` says core should stay lean, optional capability should usually ship as plugins, plugin promotion belongs in ClawHub, and plugin authors should host and maintain plugins in their own repository.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/VISION.md#L54)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Hook surfaces are intentionally distinct:** The public hook docs describe `before_agent_start` as a legacy compatibility hook, while `message_received`, `session_start`, and `before_compaction`/`after_compaction` are separate observation/lifecycle hooks. Core is not promising a single canonical init callback across those surfaces.
  - file: [docs/plugins/hooks.md:63](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/plugins/hooks.md#L63)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **No claude-mem or worker session-init code exists on main:** A repo-wide search on current `main` found no matches for `claude-mem`, `/api/sessions/init`, or `contentSessionId` under `src`, `docs`, `packages`, or `extensions`, so the reported worker/session-init implementation is not present in this repository.
  - command: `rg -n "claude-mem|/api/sessions/init|contentSessionId" src docs packages extensions`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **tool_result_persist is a generic hook seam:** `tool_result_persist` only receives generic OpenClaw hook context (`agentId`, `sessionKey`, `toolName`, `toolCallId`). It does not encode any claude-mem-specific worker API behavior, reinforcing that downstream persistence/session wiring is plugin-owned.
  - file: [src/plugins/hook-types.ts:367](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/plugins/hook-types.ts#L367)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Plugins can derive the real workspace from core runtime helpers:** The plugin runtime exposes `resolveAgentWorkspaceDir`, and its implementation resolves a workspace from `(cfg, agentId)`. Combined with hook `ctx.agentId`, an external memory plugin can compute the real workspace path without requiring a core-owned claude-mem integration.
  - file: [src/agents/agent-scope-config.ts:152](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/agent-scope-config.ts#L152)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
