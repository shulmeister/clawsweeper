## Summary

Current `main` still hard-disables native Jiti loading on Windows and still routes source plugin loading through synchronous Jiti paths, so this Windows performance bug does not appear implemented or obsolete.

## Evidence

- **Windows native Jiti is still explicitly disabled:** `supportsNativeJitiRuntime()` returns false on `win32`, so the loader still refuses the native Jiti path on Windows.
  - file: [src/plugins/sdk-alias.ts:696](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/plugins/sdk-alias.ts#L696)
