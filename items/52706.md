## Summary

Current `main` still uses the `"last"` sentinel as the default cron delivery channel in the Control UI, intentionally persists implicit-`last` behavior for new jobs, and still has a covered first-run failure path when there is no prior session target. The issue is not implemented-on-main and should stay open.
