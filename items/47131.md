## Summary

Keep open. Current main still does not implement the requested inbound plain-text `reply in thread` / `rit` directive. The code supports outbound assistant tags and configured reply threading modes, but there is no inbound preprocessing that strips those user trigger phrases and forces `replyToId` for the current inbound message. The related PR #47297 was closed unmerged, so it does not supersede the issue.
