## Summary

Keep open. Current main still lacks a user-facing `sessions_spawn` announcement policy for top-level channel-bound sessions. Existing fixes handle narrower related problems, such as suppressing child in-flight delivery and steering active parent sessions, but normal run-mode subagents still auto-announce completion back to the requester channel by default.

## Best Possible Solution

Add an explicit, documented `sessions_spawn` completion delivery policy for native and ACP run-mode spawns, preserving current channel delivery as the default. A small enum such as `completionDelivery: "channel" | "parent" | "none"` should thread through the tool schema, spawn registration, announce dispatch, task delivery status, docs, and regression tests for top-level channel-bound sessions like iMessage/BlueBubbles and WhatsApp. Reuse the existing nested-subagent internal injection path for parent-only delivery where possible.
