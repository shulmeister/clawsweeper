## Summary

Keep #39406 open. Current main partially mitigates same-action mutating retry noise, but it still lacks the requested normal-chat config policy for suppressing transient mutating tool warnings after a successful final reply.

## Best Possible Solution

Keep #39406 open for maintainer product judgment or implementation. The best path is a documented normal-chat tool-error delivery policy, preserving the current safety default, optionally suppressing mutating warnings only when configured or after verified final success, with focused payload/state tests plus updates to config types, schema/help, generated schema, and public docs. Coordinate with related mutating-warning work such as #39631/#51091 rather than treating the existing retry cleanup as the full request.

## Risks / Open Questions

- A future fix should preserve the current default of surfacing unresolved mutating failures unless the deployment explicitly opts into suppression.
- The exact same-action retry case is already mitigated, so implementation should avoid regressing that cleanup while adding the broader policy.
- The current `messages.suppressToolErrors` help text reads broader than the implementation for mutating tools; docs and behavior should be aligned when this is addressed.
