## Summary

Current main still does not implement the requested configurable suppression policy for transient mutating tool error warnings. `messages.suppressToolErrors` exists, but mutating tool warnings are still intentionally emitted even when an assistant reply exists, while `suppressToolErrorWarnings` is a per-run option wired through heartbeat/follow-up paths rather than a normal global message policy. The related PR #39636 was closed unmerged in the provided GitHub context, so it does not supersede this issue.

## Best Possible Solution

Keep #39406 open for maintainer product judgment or implementation. A good fix would add a documented normal-chat tool-error delivery policy that covers mutating tool warnings while preserving the current safety default, update config schema/help/docs, and adjust payload tests for assistant-success and retry-success cases.
