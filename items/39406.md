## Summary

Keep #39406 open. Current main now partially mitigates the exact edit retry-noise case by clearing `lastToolError` when the same mutating action later succeeds, but it still does not provide the requested normal-chat config policy for suppressing transient mutating tool warnings after a successful final reply. Existing config remains partial: `messages.suppressToolErrors` skips mutating failures, and `suppressToolErrorWarnings` is only a run/heartbeat option.

## Best Possible Solution

Keep #39406 open for maintainer product judgment or implementation. A good fix would add a documented normal-chat tool-error delivery policy, preserve the current safety default, optionally suppress mutating warnings only when configured or after verified final success, cover mutating retry-success and final-success cases in payload tests, and update config types, schema/help, generated schema, and public docs. Coordinate with the related open mutating-warning work such as #39631/#51091 rather than treating the current partial retry cleanup as the full request.
