## Summary

Current `main` still performs health-monitor stale-socket recovery as `stopChannel` plus `startChannel`, with no channel-level restart mode or reconnect/preserve-cache path. The docs only expose interval/threshold/restart-cap settings plus per-channel opt-out, so the feature request is not implemented on `main` and remains actionable.

## Evidence
