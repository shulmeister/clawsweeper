## Summary

Keep open. Current main still lacks the proposed `model_failover` and `model_failure_terminal` plugin hooks, and this PR remains a plausible narrow core plugin API change rather than ClawHub-only work because external plugins cannot observe these failover seams until core exposes typed hook names, runner methods, and emission points.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to decide the exact `model_failover` and `model_failure_terminal` event contract, address the remaining review comments, keep focused hook-runner and failover-emission tests, document the new hooks in the plugin hook catalog, and then land it if maintainers accept this narrow core plugin API surface for failover observability. Broader fallback notices, workspace-context re-injection, and circuit-breaker policy should remain under #65824 or separate follow-ups.
