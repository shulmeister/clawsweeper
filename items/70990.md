## Summary

Keep PR #70990 open. Current main does not implement the requested `model_failover` or `model_failure_terminal` plugin hooks, and the PR targets a real core plugin API gap rather than optional ClawHub work. Existing `model_call_started`/`model_call_ended` hooks provide sanitized per-call telemetry, but they do not expose failover decisions or terminal all-models-failed summaries. The linked issue #70976 remains the canonical feature request, and the provided Greptile review treated the PR as safe to merge with only minor P2 cleanup comments.

## Best Possible Solution

Keep the PR open as a plausible core plugin API addition. The best path is to review #70990 against #70976, require the small review-cleanup items plus public hook docs/API contract updates if missing, and then land or revise it through normal maintainer review rather than closing it as implemented, ClawHub, or duplicate work.
