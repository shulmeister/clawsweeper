## Summary

Current main no longer recommends the broken coding-agent completion path. The bundled coding-agent skill now explicitly forbids `openclaw system event`, `tools.exec.notifyOnExit`, and heartbeat delivery for completion, and instead requires direct `openclaw message send` routing plus process monitoring. The requested `sessions_spawn` path remains documented and tested for OpenClaw sub-agents/ACP harness sessions, but the specific work-loss bug in the bundled skill is addressed without making all coding-agent launches use `sessions_spawn`.

## Best Possible Solution

Close this issue as implemented by the merged coding-agent skill rewrite. Keep the current split: coding-agent background CLI runs use explicit `openclaw message send` completion routing, while OpenClaw sub-agent and ACP harness workflows use `sessions_spawn`/`sessions_yield` when that runtime is the right fit. Track deeper system-event audience semantics under the existing canonical follow-up rather than reopening this skill-doc item.
