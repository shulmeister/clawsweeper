## Summary

Keep open. Current main has a useful default-off package auto-updater under `update.auto.*`, but it only supports stable delay/jitter and beta cadence auto-apply. The issue’s core requested workflow still has meaningful gaps: configurable `mode`, confirmation or notify-only behavior, HH:MM/cron scheduling with timezone, and post-update channel notification.

## Best Possible Solution

Keep #12855 open as the product/implementation tracker for the missing first-class update workflow. The likely path is to either add a deliberate `update.auto` mode/schedule/notification design in core, wired through config schema, runtime updater, channel delivery, docs, and tests, or have a maintainer explicitly narrow the product scope and open/link narrower follow-ups. #65011 can remain the UI/discoverability follow-up for the updater that already ships, but it does not replace the confirmation, notify-only, timezone scheduling, or post-update channel notification work requested here.
