## Summary

Keep #12855 open. Current main has a useful default-off package auto-updater under `update.auto`, and Bortlesboat's linked guide confirms operators can use that partial capability today, but the requested first-class workflow is still missing: wall-clock or cron scheduling, timezone selection, confirmation and notify-only modes, configured channel notifications, and post-restart version/changelog/health messaging. #12896 was a closed unmerged attempt, and #65011 is an open UI/discoverability follow-up rather than a replacement for this implementation request.

## Best Possible Solution

Keep #12855 open as the canonical tracker for the first-class auto-update workflow. Build on the existing `update.auto` surface by adding schema/types/help/docs for `mode`, `schedule`, `timezone`, and notification settings; implement HH:MM or cron timezone scheduling; support `notify-only`, `confirm`, and `silent` flows through configured channels; persist post-restart update state; and report previous/new version, changelog link or summary, and health after restart. Treat #65011 as the separate UI/discoverability layer and #12896 as historical closed/unmerged work.
