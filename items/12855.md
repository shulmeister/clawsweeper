## Summary

Issue #12855 should remain open. Current main has a default-off package auto-updater under update.auto, but it is the narrower rollout updater with delay/jitter/check cadence, not the requested first-class scheduled workflow with confirmation/notify-only modes, timezone-aware wall-clock or cron scheduling, configured channel notifications, or post-restart version/changelog/health reporting. The maintainer-authored latest comment also explicitly kept this issue open as the canonical engine/workflow tracker, with #65011 left as the separate UI/discoverability follow-up.

## Best Possible Solution

Keep #12855 open as the canonical engine/workflow issue. Extend the existing core update.auto package updater with config schema, exported types, docs, and tests for mode, schedule, timezone, notifyAfterUpdate, and notifyChannel; add HH:MM or cron timezone scheduling; implement notify-only and confirm flows through configured channel targets; persist update state across restart; and report old/new version, changelog link or summary, and health after restart. Keep #65011 as the separate Control UI/onboarding/discoverability follow-up.
