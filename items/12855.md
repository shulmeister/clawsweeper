## Summary

Keep #12855 open. Current main has a real default-off package auto-updater, but it only supports stable delay/jitter and beta cadence auto-apply. The issue’s requested first-class workflow still lacks configurable modes, HH:MM/cron timezone scheduling, confirmation/notify-only behavior, and configured post-update channel notification. The closed PR #12896 did not merge, and #65011 is an open UI/discoverability follow-up rather than a replacement for the remaining workflow work.

## Best Possible Solution

Keep #12855 open as the product/implementation tracker for a deliberate first-class update workflow. The likely path is to extend `update.auto` through config schema/types/help, runtime updater scheduling, confirmation or notify-only channel delivery, post-update restart notification, docs, and tests, or have a maintainer explicitly narrow the scope and link canonical follow-ups. #65011 should remain the UI/discoverability follow-up for the updater that already ships, not the replacement for this workflow request.
