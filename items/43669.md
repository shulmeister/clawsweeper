## Summary

Keep #43669 open. Current main has adjacent cleanup hardening for pending tool results, but it still does not implement this PR's central post-prompt idle wait before session snapshot and final-reply selection. The PR discussion also records that the initial review feedback was addressed in the PR head, so the remaining work is plausibly useful rather than obsolete.

## Best Possible Solution

Keep the PR open. The best path is to land a current-main version that waits abortably for agent idle immediately after activeSession.prompt() returns when the session is still streaming, distinguishes idle timeout in logs, and adds focused regression coverage proving the final snapshot/reply waits for the retry tool loop to complete. If maintainers prefer a different shape, supersede this PR with an equivalent mainline fix that covers the same post-prompt final-response race.
