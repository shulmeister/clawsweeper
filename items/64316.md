## Summary

Keep open. The PR's narrow reset-path behavior is not implemented on current main: `resetReplyRunSession` still rotates the session and cleans transcripts without disposing the previous bundle MCP runtime. Current main has broader MCP lifecycle mitigations, including idle TTL and one-shot cleanup on other surfaces, but those are backstops rather than the mid-run reset disposal requested here. Related PR #68450 remains open, so it is useful context but not a landed superseding fix.
