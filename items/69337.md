## Summary

Keep #69337 open. The PR identifies a plausible TUI visibility problem, but current main has not implemented the proposed commentary fallback and the proposed shared-helper change conflicts with existing shared, session preview, web UI, and live-chat contracts that intentionally hide commentary-only assistant text.

## Best Possible Solution

Keep the PR open for maintainer/contributor revision. The best path is a scoped TUI solution, such as a caller-specific option or fallback in the TUI formatter/stream assembler, plus updated TUI-focused tests, while preserving the shared helper, session preview, web UI, and live-chat contracts that hide commentary-only assistant text.

## Risks / Open Questions

- Merging the PR as-is would likely break existing tests that require commentary-only assistant content to stay hidden.
