## Summary

Current `main` already implements the substantive request: ACP bindings now validate for arbitrary channel ids, and plugin channels consume configured-binding routing through the public `openclaw/plugin-sdk/conversation-runtime` surface rather than `PluginRuntime.channel.routing`. This shipped in `v2026.4.22`, so the PR is superseded.

## Close Comment

Closing as implemented on `main` after Codex review. The underlying ACP-any-channel support is already shipped in `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`): `src/config/zod-schema.agents.ts:64` no longer has the three-channel ACP allowlist, `src/config/config.acp-binding-cutover.test.ts:112` explicitly accepts arbitrary channel ids, and plugin authors already get the configured-binding helpers from the supported public SDK surface at `src/plugin-sdk/conversation-runtime.ts:13` (also documented in `docs/plugins/sdk-subpaths.md:57`). Current plugin channels such as LINE are already using that public path for configured ACP bindings in `extensions/line/src/bot-message-context.ts:10`. Given that, this PR is superseded by the implementation already on `main`.
