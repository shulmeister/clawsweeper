## Summary

Current `main` already contains the Windows PowerShell allowlist/command-rebuild handling that matches this report: PowerShell wrappers are unwrapped for `-Command`/`-c` forms, Windows allowlist hits execute the parsed inner argv instead of replaying the wrapper argv, and durable exact-command allow-always matching is present for reruns.

## Evidence

- **PowerShell wrapper parsing covers the failing forms:** `stripWindowsShellWrapperOnce` explicitly handles `powershell`/`pwsh`, `-Command`, `-c`, `--command`, and quoted flag values such as `-WorkingDirectory "C:\Users\Jane Doe\proj"`, which is the class of wrapper parsing failure described in the issue.
  - file: [src/infra/exec-approvals-analysis.ts:573](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/infra/exec-approvals-analysis.ts#L573)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Regression tests for quoted PowerShell flags and aliases:** Tests assert that Windows analysis unwraps PowerShell commands with value-taking flags, quoted working-directory values, and `-c`/`--command`, with comments noting the old behavior broke allow-always matching for the inner command.
  - file: [src/infra/exec-approvals-analysis.test.ts:510](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/infra/exec-approvals-analysis.test.ts#L510)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Windows allowlist execution now uses inner argv:** In allowlist mode on Windows, when shell analysis succeeds and matches the allowlist, execution switches from the original wrapper argv to `segments[0].argv`, avoiding the broken wrapper replay path that produced invalid PowerShell invocations.
  - file: [src/node-host/invoke-system-run-allowlist.ts:119](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/node-host/invoke-system-run-allowlist.ts#L119)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Durable exact-command trust exists for reruns:** Exact-command allow-always entries are recognized via `=command:<hash>`, so commands without a stable executable path match can still reuse durable approval on rerun instead of falling back to an allowlist miss.
  - file: [src/infra/exec-approvals.ts:792](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/infra/exec-approvals.ts#L792)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
