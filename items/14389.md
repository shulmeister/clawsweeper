## Summary

Current `main` has partial safety nets for config writes and invalid config recovery, but it does not implement the requested post-change health-check timeout with automatic rollback after a failed restart. The issue should stay open.

## Evidence

- **`config.patch` writes then only schedules restart:** The RPC handler persists the new config with `writeConfigFile(...)`, writes a restart sentinel, schedules `scheduleGatewaySigusr1Restart(...)`, and returns success. There is no code here that waits for the gateway to come back healthy or restores the previous config if restart health fails.
  - file: [src/gateway/server-methods/config.ts:579](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/gateway/server-methods/config.ts#L579)
  - command: `nl -ba src/gateway/server-methods/config.ts | sed -n '520,625p'`
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **Startup recovery is limited to invalid config snapshots:** On boot, OpenClaw only calls `recoverConfigFromLastKnownGood(...)` when `readConfigFileSnapshot()` reports the config is invalid. That is narrower than the issue request for rollback when a config change causes the gateway restart itself to fail health checks.
  - file: [src/gateway/server-startup-config.ts:71](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/gateway/server-startup-config.ts#L71)
  - command: `nl -ba src/gateway/server-startup-config.ts | sed -n '55,110p'`
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **Docs describe last-known-good for invalid/clobbered configs, not restart-time health rollback:** The docs say the gateway keeps a last-known-good copy and restores it when `openclaw.json` fails validation or looks clobbered. They do not describe an automatic backup-before-write plus restart-time health-check rollback flow for `config.patch`/`openclaw config` changes.
  - file: [docs/gateway/configuration.md:87](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/docs/gateway/configuration.md#L87)
  - command: `nl -ba docs/gateway/configuration.md | sed -n '70,110p'`
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **CLI write-safety docs stop at pre-commit rejection and invalid-edit recovery:** `openclaw config set` validates the full post-change config before commit and saves rejected payloads as `.rejected.*`; direct invalid edits may later be restored from last-known-good. This is helpful, but still not the requested auto-rollback on restart health failure after an accepted change.
  - file: [docs/cli/config.md:357](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/docs/cli/config.md#L357)
