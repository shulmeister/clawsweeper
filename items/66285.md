## Summary

Keep open. Current main still does not support `"explicit"` for Active Memory `allowedChatTypes`: the extension types, runtime normalization filter, and manifest schema only allow `direct`, `group`, and `channel`, and the docs/examples omit explicit sessions. The PR is a focused fix for linked issue #65775, but its current diff also has a real review-comment edge case: explicit session keys are built as `agent:<agent>:explicit:<raw sessionId>`, while the proposed detector checks broad `:group:`/`:channel:` markers before `:explicit:`, so some explicit session IDs could still be misclassified.

## Best Possible Solution

Keep the PR open and have the author or maintainer update the Active Memory fix to treat `agent:<agent>:explicit:<id>` as `explicit` before scanning for broad group/channel/direct markers, while preserving the schema/config/test coverage added by the PR and ideally documenting `allowedChatTypes: ["explicit"]` in the Active Memory docs.
