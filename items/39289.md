Closing from a Codex review. Current `main` already has the Control UI/WebChat guards that prevent duplicate rendering around tool-heavy runs and history reloads: `ui/src/ui/app-gateway.ts:422` coordinates final chat events with deferred `session.message` reloads, and `ui/src/ui/controllers/chat.ts:179` clears transient stream/tool state when applying persisted `chat.history` so the same reply is not rendered twice. If this reappears on a newer build, please open a fresh issue with the exact OpenClaw version, whether tools were involved, and a short screen recording or event log.
