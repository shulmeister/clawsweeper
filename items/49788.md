## Summary

Current `main` does not implement an "Accessible" Control UI theme preset. The theme model only recognizes `claw`, `knot`, and `dash`; both theme pickers expose only those three options; and the CSS theme-family overrides in `ui/src/styles/base.css` define only the existing Knot and Dash variants beyond the default Claw dark/light themes.

## Evidence

- **Theme model only supports three families:** `ThemeName` is limited to `claw | knot | dash`, `VALID_THEME_NAMES` contains only those three values, and `resolveTheme()` maps only to `dark/light/openknot/openknot-light/dash/dash-light`. There is no accessible-theme identifier or resolver path.
  - file: [ui/src/ui/theme.ts:1](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/ui/src/ui/theme.ts#L1)
  - command: `nl -ba ui/src/ui/theme.ts | sed -n '1,120p'`
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Full settings theme picker has only Claw/Knot/Dash:** The Control UI appearance section defines exactly three theme cards: Claw, Knot, and Dash. No Accessible preset is present.
  - file: [ui/src/ui/views/config.ts:571](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/ui/src/ui/views/config.ts#L571)
