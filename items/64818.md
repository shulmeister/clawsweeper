## Summary

Keep open. Current main still has the reported core failure shape: `src/canvas-host/a2ui/.bundle.hash` is tracked while ignored, the A2UI build path still writes that hash file, and the dev update preflight walkback still does plain detached checkouts between candidates without resetting the disposable worktree. The related PR #64822 proposed the minimal untrack fix but is closed unmerged, and #67547 is a related A2UI reproducibility issue rather than a canonical replacement for this updater walkback bug.

## Best Possible Solution

Fix this in OpenClaw core. The safest path is to make the dev preflight worktree resilient by resetting/cleaning the disposable worktree before each fallback checkout, and also either stop tracking `src/canvas-host/a2ui/.bundle.hash` or make the A2UI bundler avoid unnecessary writes. Add an updater regression test that simulates a successful build dirtying a tracked generated file, a later lint/build failure, and a successful walkback checkout to an older candidate. Coordinate with #67547 for the separate A2UI bundle reproducibility problem, but do not treat it as a replacement for this updater fix.
