## Summary

Current main still uses the plain integer/decimal regex in parseConfigValue, so unquoted scientific notation is still parsed as a string on the slash-command config/MCP path. The PR remains a plausible focused fix rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open for normal maintainer review. The likely best path is to land a focused parseConfigValue change with a colocated unit test for exponent notation and existing booleans/null/plain numbers/JSON/quoted strings, while marking the earlier heartbeat review comments as outdated because that accidental test was removed from the final diff.
