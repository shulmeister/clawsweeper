## Summary

Keep open. Current main still has the reported bundled `gh-issues` skill problems: the top-level `user-invocable: true` key remains, the skill is still a single 34,800-byte file, and the bundled skill-creator validator still rejects that key. The related PR #54360 was closed unmerged, so there is no landed fix to close against.

## Best Possible Solution

Keep this open and resolve it with a focused bundled-skills cleanup: decide whether `user-invocable` is a supported public frontmatter field, then align `skills/gh-issues/SKILL.md`, `skills/skill-creator/scripts/quick_validate.py`, runtime docs, and tests accordingly. If the key remains redundant because runtime defaults to true, remove it from the bundled skill; separately split the large gh-issues orchestration material into `references/` files and tighten the description to match the skill description guidance.
