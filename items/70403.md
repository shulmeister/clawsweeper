## Summary

Keep #70403 open. Current main still appends Dream Diary narratives from light and REM phases, while the deep phase also writes a narrative, so the PR's central one-DREAMS.md-entry-per-sweep behavior is not implemented. Related PRs #65138, #70332, and #70523 address adjacent dreaming cleanup/fallback/dedupe problems, but they do not currently supersede this exact phase-level behavior change.

## Best Possible Solution

Keep #70403 open or fold its narrow change into the canonical Dream Diary cleanup branch: remove light/REM calls to generateAndAppendDreamNarrative, keep deep-phase diary generation as the single full-sweep narrative path, update the current light/REM tests to assert no subagent/DREAMS.md append, and change the public dreaming docs from per-phase to per-sweep wording.
