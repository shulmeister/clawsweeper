## Summary

Keep PR #59787 open. The PR is a plausible bug fix but its current head still appears to have the exact media-only suppression bug: `sendTarget.hasText` is set unconditionally whenever a target exists, so media-only sends can still suppress the following text reply. For PRs, flawed or incomplete work should remain open rather than be closed under the allowed cleanup reasons.

- **PR head still sets hasText unconditionally:** The current PR patch assigns `sendTarget.hasText = true` inside `if (sendTarget)` without deriving it from the `content`/`message` text check, so a media-only tool call with a target is still marked as text-bearing.
