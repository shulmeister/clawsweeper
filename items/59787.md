## Summary

Keep open. Current main still has the reported behavior: messaging sent targets do not carry a text-content marker, and reply suppression still suppresses based only on a matching target. The PR discussion also identifies a remaining bug in the PR head: `hasText` was being set unconditionally, so the proposed branch still needed a small follow-up rather than cleanup closure.
