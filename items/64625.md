## Summary

Keep this issue open. Current main still has the Slack custom invoke path that forwards the raw tool context without adding `ctx.mediaLocalRoots` or `ctx.mediaReadFile`, while the fallback path does add them. The linked fix PR #64632 was closed unmerged, and the current source still matches the reported failure mode.

## Best Possible Solution

Review or recreate the narrow fix from #64632 in `extensions/slack/src/channel-actions.ts`: build an augmented Slack action context containing the existing `toolContext` plus `ctx.mediaLocalRoots` and `ctx.mediaReadFile`, pass that to both custom and fallback invoke paths, and add a regression test that exercises Slack `upload-file` through the custom invoke path with agent-scoped media roots.
