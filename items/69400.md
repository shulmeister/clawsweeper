## Summary

Current main does not implement the requested wrapper support. The daemon argument resolver still emits runtime-plus-entrypoint ProgramArguments, the install CLI has no --wrapper option, docs list no wrapper support, and the linked fixing PR #69425 remains open/unmerged. The issue is concrete and recent, so it should stay open until #69425 or an equivalent core daemon change lands.

## Best Possible Solution

Keep #69400 open as the canonical issue until PR #69425 or an equivalent core daemon implementation lands. The implementation should live in the daemon install/program-args path, add regression coverage for OPENCLAW_WRAPPER and any --wrapper persistence semantics, preserve safe wrapper metadata across reinstalls, and update the gateway CLI/macOS service docs if a user-facing option or env var is shipped.
