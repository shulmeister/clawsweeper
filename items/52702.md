## Summary

Current main already covers the reported context-growth problem with built-in in-memory tool-result pruning, oversized tool-result truncation, and an optional tool-result compaction plugin, so this request is implemented well enough via a different design.

## Close Comment

Closing as implemented on current main after Codex review. OpenClaw already trims old tool results out of the in-memory prompt (`docs/concepts/session-pruning.md:9`, `docs/gateway/config-agents.md:570`, `src/agents/pi-hooks/context-pruning/pruner.ts:220`), caps oversized live tool results and uses them in preemptive overflow handling (`src/agents/pi-embedded-runner/tool-result-truncation.ts:14`, `src/agents/pi-embedded-runner/run/preemptive-compaction.ts:41`), and now also ships the opt-in `tokenjuice` plugin to compact noisy `exec`/`bash` outputs before they re-enter session context (`docs/tools/tokenjuice.md:10`). This was already present in the released `v2026.4.22` tree at `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`. The implementation differs from the exact proposal here, but the underlying request to prevent tool-result context blow-up is already covered on main.
