## Summary

Current main already handles browser tab/CDP failures as normal browser-tool errors, and `config.patch` now avoids restart on validation/no-op cases that matched the reported loop. The same guards are present in the shipped `v2026.4.23` tag.

## Evidence

  - file: [extensions/browser/src/node-host/invoke-browser.ts:275](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/browser/src/node-host/invoke-browser.ts#L275)

## Risks / Open Questions

- This review was read-only and did not run a live end-to-end reproduction; the conclusion relies on current code and regression tests.
- A model can still intentionally submit a real config change that requires restart; this review only closes the report because browser failures themselves now stay in the error path and no-op/invalid `config.patch` calls no longer restart the gateway.
