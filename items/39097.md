## Summary

Keep open. Current main still provides browser SSRF/protocol/private-network navigation guards and hostname allowlist support, but it does not implement the requested operator-configurable browser URL/domain/path denylist or warnlist for known bot-blocked sites. The prior ClawSweeper comment kept this open, and a fresh read against 73e21511 did not find a shipped implementation or canonical superseding item.

## Best Possible Solution

Keep the issue open until the bundled browser plugin has an explicit product decision or implementation. A complete fix would add browser denylist/blockedUrls policy, and optionally warnlist, through config types, zod/generated schema/help, config resolution, managed and existing-session navigation guards, user-facing browser errors, docs, and focused tests for domain wildcard and path-pattern behavior. Any default blocked-domain list should be maintainer-reviewed because it is product policy, not just plumbing.
