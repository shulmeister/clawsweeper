## Summary

Current main still lacks an operator-configurable browser URL/domain/path denylist or warnlist. The browser policy surface remains SSRF/private-network oriented, with allowlist exceptions only, and navigation paths wire only `ssrfPolicy` through managed Playwright, Chrome MCP existing-session, and CDP fallback guards.

## Best Possible Solution

Keep #39097 open. The best implementation is a maintainer-approved browser URL policy surface, likely `browser.denylist` or `browser.blockedUrls` plus optional `warnlist`, wired through typed config, strict zod/generated schema/help, browser config resolution, shared navigation policy options, managed Playwright, existing-session Chrome MCP, raw CDP fallback, clear user-facing errors/warnings, docs, and focused tests for exact host, wildcard domain, and path-pattern behavior.
