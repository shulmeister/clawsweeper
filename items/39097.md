## Summary

Current main still lacks an operator-configurable browser URL/domain/path denylist or warnlist. The browser navigation guard remains an SSRF/private-network policy with hostname allowlist exceptions, and all browser navigation routes pass only that SSRF policy into managed Playwright, Chrome MCP existing-session, and raw CDP paths. The related items in the provided context do not supersede this request.

## Best Possible Solution

Keep #39097 open. A complete fix should add a maintainer-approved browser URL policy surface, likely browser.denylist/browser.blockedUrls plus optional warnlist, wire it through typed config, strict zod/generated schema/help, config resolution, BrowserNavigationPolicyOptions, managed Playwright, Chrome MCP existing-session, raw CDP fallback paths, clear user-facing errors or warnings, docs, and focused tests for exact host, wildcard domain, and path-pattern behavior.
