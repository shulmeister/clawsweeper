## Summary

Keep open. Current main has browser SSRF/protocol/private-network navigation guards and hostname allowlist support, but it still does not implement the requested operator-configurable URL/domain/path denylist or warnlist for known bot-blocked sites. The issue is recent, actionable, and has no protected author/label blocker, so no close reason fits.

## Best Possible Solution

Keep the issue open until the bundled browser plugin has an explicit product decision or implementation. A complete fix would add a browser denylist/blockedUrls policy, and optionally warnlist, through config types, zod/generated schema/help, config resolution, managed and existing-session navigation guards, user-facing browser errors, docs, and focused tests for domain wildcard and path-pattern behavior. Any default blocked-domain list should be maintainer-reviewed because it is product policy, not just plumbing.
