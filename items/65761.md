## Summary

Keep open. Current `main` still uses a fire-and-forget `sendBlockReply` contract and the same-channel dispatch path still does not await block delivery, so this PR is not implemented on `main` and the underlying behavior remains plausible. The PR also appears to need refresh/scope cleanup because its diff snapshot includes unrelated stale hunks and it touches a public SDK surface that must stay source-compatible for plugins.
