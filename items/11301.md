## Summary

Keep #11301 open. Current main still does not provide a runtime-container `/workspace` alias for the Docker/Podman gateway or CLI containers. The shipped container paths still mount the workspace at `/home/node/.openclaw/workspace`, while `/workspace` remains sandbox-container behavior. PR #12764 was closed unmerged, so it is useful prior art but not a shipped or superseding fix.

## Best Possible Solution

Keep #11301 open. Maintainers should either implement a safe runtime compatibility alias or second bind mount for `/workspace -> /home/node/.openclaw/workspace` across Dockerfile, Compose, Docker setup, Podman/Quadlet, and relevant smoke/docs coverage, or explicitly decide that `/workspace` is sandbox-only and update runtime/docs wording to remove the ambiguity. Treat PR #12764 as prior art, not a canonical fix.
