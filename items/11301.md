## Summary

Keep open. Current main still does not add the requested `/workspace` symlink or top-level runtime-container alias: the OpenClaw Docker/Compose/Podman paths continue to mount the persisted workspace only at `/home/node/.openclaw/workspace`. Current sandbox containers do use `/workspace`, and current AGENTS/SOUL templates no longer appear to contain the original broad `/workspace` wording, but that does not implement the Dockerfile/entrypoint compatibility request in #11301. The exact related PR #12764 was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep #11301 open until a maintainer either implements the runtime-container compatibility path, likely by adding a safe `/workspace -> /home/node/.openclaw/workspace` alias in the Docker image/entrypoint and aligning Docker/Podman docs or smokes, or explicitly decides that only sandbox containers should expose `/workspace`. The already-closed PR #12764 can be used as implementation context but is not canonical remaining work because it was not merged.
