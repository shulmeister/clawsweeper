## Summary

Keep #11301 open. Current main still mounts Docker/Podman runtime container workspaces only at `/home/node/.openclaw/workspace`; it does not create a `/workspace` symlink or second bind mount for the gateway/CLI containers. Existing `/workspace` behavior is still scoped to sandbox containers, and the related PR #12764 was closed unmerged according to the provided GitHub context.

## Best Possible Solution

Keep #11301 open until a maintainer either implements a safe Docker/Podman runtime compatibility alias or bind mount for `/workspace -> /home/node/.openclaw/workspace` and aligns docs/smokes, or explicitly decides `/workspace` is sandbox-only and updates Docker/runtime documentation plus issue expectations. Treat PR #12764 as prior art, not a superseding fix, because it was closed unmerged.
