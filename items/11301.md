## Summary

Keep open. Current main still does not add the requested `/workspace` symlink or top-level runtime-container alias: Docker Compose, Docker setup helpers, and Podman continue to mount the persisted workspace only at `/home/node/.openclaw/workspace`. The existing `/workspace` support is for agent sandbox containers, not the main OpenClaw runtime container, and the related implementation PR #12764 was closed unmerged.

## Best Possible Solution

Keep #11301 open until a maintainer either implements a safe `/workspace -> /home/node/.openclaw/workspace` compatibility alias in the runtime Docker image or entrypoint and aligns Docker/Podman docs or smokes, or explicitly decides that `/workspace` is sandbox-only and updates the issue/docs expectations accordingly. The closed PR #12764 remains useful implementation context but does not supersede the issue because it was not merged.
