## Summary

Keep open. Current `main` does not add a `/workspace` symlink to the Docker gateway/container configuration. The original report’s AGENTS/SOUL documentation premise appears obsolete on current `main`, because Docker docs now distinguish `/home/node/.openclaw/workspace` for the gateway container from `/workspace` for sandbox containers, but the requested ergonomics enhancement itself is still not implemented.

## Evidence

- **Gateway and CLI containers still mount the workspace only at the long path:** `docker-compose.yml` mounts `OPENCLAW_WORKSPACE_DIR` to `/home/node/.openclaw/workspace` for both `openclaw-gateway` and `openclaw-cli`; there is no `/workspace` mount or alias in the compose config.
  - file: [docker-compose.yml:15](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/docker-compose.yml#L15)

- **Current Docker docs explicitly document the long path for the gateway container:** `docs/install/docker.md` says Docker Compose bind-mounts `OPENCLAW_WORKSPACE_DIR` to `/home/node/.openclaw/workspace`.
  - file: [docs/install/docker.md:170](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/docs/install/docker.md#L170)
