## Summary

Keep #7540 open. Current main still does not subscribe the WhatsApp/Baileys socket to `call`, and there is no WhatsApp call-tracking config, plugin hook, docs, or test coverage. The pinned Baileys version exposes the requested `call: WACallEvent[]` event, so this remains an actionable bundled WhatsApp plugin enhancement rather than a closeable cleanup item.

## Best Possible Solution

Keep the issue open and implement the feature in the bundled WhatsApp plugin: add a global/per-account opt-in such as `channels.whatsapp.trackCalls` if maintainers want the default off, subscribe to Baileys `call`, apply the same account and access-control boundaries as inbound messages, expose sanitized contact/call activity through a documented session-event or tool surface, and add focused schema, docs, and WhatsApp plugin tests.
