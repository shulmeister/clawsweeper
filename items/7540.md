## Summary

Keep #7540 open. Current main still does not subscribe the bundled WhatsApp plugin to Baileys `call` events, does not define WhatsApp call-tracking config, and does not expose sanitized WhatsApp call activity through the existing message/session/plugin-hook surfaces. The issue discussion also already kept this open and framed the fix as WhatsApp-plugin-owned rather than PSTN voice-call-plugin work.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin under `extensions/whatsapp`: subscribe to Baileys `call`, normalize and sanitize `WACallEvent` payloads, apply the same account/access-control boundaries as inbound messages, add an opt-in channel/account setting if maintainers want one, expose contact activity through a documented session-event, plugin-hook, or tool surface, and cover the behavior with focused tests and docs.
