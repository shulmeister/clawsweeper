## Summary

Keep open. Current main has useful partial hardening for specific external-content paths, inbound metadata/history, and ownership-based tool policy, but it still does not implement the central request: generic trust-boundary marking for non-owner current user messages and open-world tool results at the shared agent/provider boundary, plus a stable system-prompt anchor for those wrappers. The linked implementation PR #62973 was closed unmerged.

## Best Possible Solution

Keep this issue open and implement a core-owned, tested trust-boundary strategy: add stable system-prompt guidance, wrap or otherwise label non-owner current messages at prompt assembly time, classify and wrap open-world tool results in the shared transport or persistence path, and add regression coverage for the existing external-content marker sanitizer plus provider payload behavior.

## Risks / Open Questions

- Future implementation should preserve the stronger existing `EXTERNAL_UNTRUSTED_CONTENT` marker sanitizer and special-token stripping instead of replacing it with weaker XML-only wrapping.
- A shared transport-level wrapper needs tests for double-wrapping, provider payload shape, prompt-cache stability, and owner vs non-owner behavior across web_fetch, web_search, x_search, external MCP, and channel ingress.
