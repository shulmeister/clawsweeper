## Summary

Keep open. Current `main` still matches the reported bug: the macOS node advertises `screen` support and declares `screen.record`, the macOS docs still list `screen.record` as a node capability, but gateway policy continues to classify `screen.record` as dangerous and excludes it from the default macOS allowlist. The gateway test suite explicitly asserts that gated behavior, so this is not implemented on `main` and the issue remains actionable.
