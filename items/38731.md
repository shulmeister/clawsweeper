## Summary

Keep PR #38731 open. Current main still has no high-frequency cron advisory for `cron add`, `cron edit`, gateway `cron.add`, or gateway `cron.update`; the linked issue #38730 remains open. The PR is a plausible focused fix, but it needs rework before merge because review identified still-relevant gaps: update paths are uncovered, CLI warning output uses the error channel, and the gateway warning call in the PR uses the wrong logger argument order.

## Best Possible Solution

Keep the PR open or replace it with a revised implementation that centralizes the high-frequency cron advisory and applies it consistently to create and update paths. CLI `cron add` and `cron edit --every` should warn non-fatally for intervals under 30 minutes; gateway `cron.add` and `cron.update` should log the same condition with `warn(message, meta)`; focused regression tests should cover both add and update without rejecting valid short schedules.
