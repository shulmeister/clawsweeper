## Summary

Keep #9656 open. Current main still normalizes `envelope.editMessage?.dataMessage` as an ordinary Signal inbound message and does not model or act on `editMessage.targetSentTimestamp`. The requested edit marker/metadata, original-message deduplication or transcript replacement, and edit-specific reply suppression are still absent. The related PR #9701 is closed and unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open until the bundled Signal plugin explicitly handles edit envelopes: type and validate `editMessage.targetSentTimestamp`, mark edited content in agent/session context, use the target timestamp for deduplication or transcript replacement, and suppress edit-triggered replies unless the edit explicitly mentions the agent. Add focused Signal plugin tests for edit envelopes, dedup/messageId behavior, and group mention gating.
