## Summary

Keep #9656 open. Current main still treats Signal `editMessage` envelopes as ordinary inbound messages, does not model the edit `targetSentTimestamp`, uses the new envelope timestamp as `MessageSid`, and has no edit-aware reply suppression or focused edit-envelope tests. Related PR #9701 is useful prior work, but the provided context says it was closed unmerged, and current source still lacks its advertised behavior.

## Best Possible Solution

Implement this in the bundled Signal plugin under `extensions/signal`: type `editMessage.targetSentTimestamp`, preserve edit metadata or an `[edited]` marker in agent/session context, use the target timestamp for dedupe or transcript replacement, suppress automatic replies for edits unless the edited text explicitly mentions or commands the agent, and add focused Signal edit-envelope tests. Treat PR #9701 as useful history, not shipped code.
