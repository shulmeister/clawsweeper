## Summary

Keep #9656 open. Current main partially recognizes Signal edit envelopes by reading `envelope.editMessage?.dataMessage`, but it still treats the edited content as a normal inbound message: no `[edited]` marker/metadata, no `targetSentTimestamp` field on the edit type, no use of the target timestamp for deduplication, and no edit-specific suppression of replies. The linked PR #9701 was closed unmerged, so it does not supersede the issue.
