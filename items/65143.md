## Summary

Keep this PR open. Current main still does not expose `models.providers.*.timeoutSeconds`, so the requested provider-level stream timeout is not implemented. The PR is a relevant implementation candidate, but it should be revised before merge because the current approach mutates Undici process-global timeout state and the generated runtime schema surface still needs to be kept in sync.

## Best Possible Solution

Keep the PR open and adapt it rather than close it. The best path is to add `models.providers.*.timeoutSeconds` consistently across the config type, Zod schema, help/labels, generated schema baseline, and docs, then wire it through the provider request/transport fetch path using per-request timeout dispatchers instead of mutating global Undici timeout state per embedded attempt. Add focused tests for schema metadata and provider fetch timeout wiring before maintainer review.
