## Summary

Current `main` does not implement per-provider `models.providers.*.timeoutSeconds`, and the existing timeout controls are still run-wide / agent-wide rather than provider-scoped. The PR author already addressed the early help/label feedback in follow-up commits, so this is not a stale or incoherent PR; it remains a substantive feature change that needs maintainer judgment instead of conservative auto-close.
