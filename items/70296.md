## Summary

Keep #70296 open. Current main appears to read back configured agent/default models and has standardized BOOT silent behavior on `NO_REPLY`, but the sentinel-notice suppression request is still not implemented: restart sentinel messages are still formatted with the restart summary plus doctor hint and are still enqueued/delivered to resolved channel routes without a suppress policy. Related open issue #72043 tracks a broader recovery-notice knob, but it does not fully supersede this bundled report without maintainer cleanup/splitting.

## Best Possible Solution

Keep the remaining work open or explicitly close #70296 as a duplicate only after maintainers choose #72043 as the canonical recovery-notice tracker. The code path should add a typed/schema-backed gateway notice policy, apply it to restart-sentinel prompt events and outbound channel delivery while preserving continuations, and cover config-patch/config-apply restart notices with tests and docs. The model display and BOOT token sub-items should be split or marked obsolete with current-main evidence if maintainers want a cleaner issue.
