## Summary

Keep #69768 open. The report is still actionable on current main: the Gemini schema cleaner removes many incompatible JSON Schema keywords but still does not remove `if`, `then`, or `else`, so the validated root cause from the issue discussion remains possible. Current main has improved no-body HTTP 400/422 failover classification, but that only affects error handling and does not fix Gemini rejecting conditional tool schemas. The linked PR #69964 proposed the right narrow fix but was closed unmerged according to the provided GitHub context.

## Best Possible Solution

Keep the issue open and land a narrow agent-schema fix: add `if`, `then`, and `else` to Gemini's unsupported schema keyword cleanup, then add a focused regression test showing conditional keywords are stripped from nested tool schemas while ordinary properties and metadata remain. The no-body 400/422 failover behavior can remain as a separate already-improved error-handling path; it does not replace the schema serialization fix.
