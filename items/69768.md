## Summary

Keep #69768 open. Current main still does not strip JSON Schema `if`/`then`/`else` from Gemini tool schemas, and the related fix PR #69964 was closed unmerged. The cron failure-visibility docs now cover `failureDestination`, but the reported Gemini 400 schema-cleaning bug remains actionable in core.

## Best Possible Solution

Implement the narrow core fix from the unmerged PR: add `if`, `then`, and `else` to `GEMINI_UNSUPPORTED_SCHEMA_KEYWORDS`, add a regression test proving nested conditionals are stripped while valid properties remain, then run the changed agent/schema test lane. Optionally follow with docs guidance for production cron users to configure `cron.failureDestination`, but that should not replace the schema-cleaning fix.
