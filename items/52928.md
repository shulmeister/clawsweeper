## Summary

Keep open. Current main still lacks the requested `agents.setDefault` Gateway RPC, `agents.update` still cannot change default-agent state, the advertised method/scopes/docs list only the existing agent CRUD and file helpers, and the Control UI still stages the invalid `agents.defaultId` config key tied to related #59070.

## Best Possible Solution

Implement an additive core Gateway method. Add typed `agents.setDefault` params/result schemas, validators, method registry entry, ADMIN scope entry, and a server handler that rejects unknown configured agent ids, sets only the selected `agents.list[]` entry to `default: true`, clears the flag from others, writes config, and returns `{ ok: true, defaultId }`. Update protocol docs and switch the Control UI Set Default path away from `agents.defaultId`; add focused server-method tests plus UI/controller coverage. This should also address the related #59070 invalid-config symptom.
