## Summary

Keep #66198 open. Current `main` still sends Signal inbound text to the model-facing `BodyForAgent` field as raw `entry.bodyText`, while `formatInboundEnvelope(...)` is now a legacy/plaintext envelope path. The PR is useful as security hardening direction, but the submitted patch only wraps the formatted envelope `Body` and group-history envelope text, so it does not address the prompt path that current reply code prefers. The existing P1 review comment on the PR correctly identifies this bypass.

## Best Possible Solution

Keep the PR open and request a targeted revision: harden the actual Signal prompt-fed surface, likely `BodyForAgent` or a structured `UntrustedContext` entry appended by the reply prompt layer, and add focused Signal tests proving inbound DM and group message bodies reach the model with external-content boundaries. Trusted metadata such as message IDs should stay outside the untrusted block, and history wrapping should be deterministic or applied once at ingest.
