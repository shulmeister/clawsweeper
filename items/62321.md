## Summary

Current main implements the requested bounded, queued subagent execution: `sessions_spawn` runs child agents on the `subagent` lane, the gateway configures that lane from `agents.defaults.subagents.maxConcurrent`, the queue only starts work while active tasks are below the cap, and docs describe the subagent concurrency safety valve. The same implementation is present in the `v2026.4.24` tag.
