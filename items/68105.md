## Summary

Keep open. Current `main` still applies RTL bidi isolation only in the TUI formatter path; the outbound reply path used for non-TUI channels still sanitizes via `sanitizeUserFacingText`/`normalizeReplyPayload` without any matching RTL isolation step, and the only regression coverage for RLI/PDI remains in TUI tests.

## Evidence

  - file: [src/tui/tui-formatters.ts:113](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/tui/tui-formatters.ts#L113)

  - file: [src/tui/tui-formatters.test.ts:339](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/tui/tui-formatters.test.ts#L339)
