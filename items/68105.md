## Summary

Keep open. The current main branch still only applies RTL bidi isolation in the TUI rendering sanitizer. Non-TUI outbound reply paths call `sanitizeUserFacingText` and gateway chat delta/final code, but those paths do not import or apply the TUI `RLI/PDI` wrapping helper. The related #21936 work is present only as TUI behavior/tests, and #32852 was a Telegram draft-preview issue rather than a general outbound gateway fix.

## Best Possible Solution

Keep the issue open and implement a shared text-direction isolation helper outside `src/tui`, then apply it at the outbound user-visible text boundary used by channel delivery and gateway chat payloads. Add focused regression tests for Hebrew/Arabic punctuation, mixed LTR/RTL lines, multiline behavior, and already-isolated text.
