## Summary

Keep #46965 open. Current main has a partial global `messages.suppressToolErrors` setting, but it is not equivalent to the requested global/per-agent `suppressToolErrorWarnings`: mutating tool failures such as `edit`, `write`, and `exec` still surface warnings outside heartbeat runs, and the config schema still exposes `suppressToolErrorWarnings` only under heartbeat config.

## Evidence

- **Issue request remains specific and actionable:** The issue asks for a global `agents.defaults.suppressToolErrorWarnings` or per-agent `agents.list[].suppressToolErrorWarnings`, or category-based filtering, because the existing heartbeat-only setting does not cover normal Telegram runs.
- **Config metadata only exposes heartbeat-scoped suppressToolErrorWarnings:** Generated config help includes `agents.defaults.heartbeat.suppressToolErrorWarnings`, `agents.list.*.heartbeat.suppressToolErrorWarnings`, and `agents.list[].heartbeat.suppressToolErrorWarnings`; it does not expose a global `agents.defaults.suppressToolErrorWarnings` or agent-level non-heartbeat key.
