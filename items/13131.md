## Summary

Current main does not implement gateway-level tool-call audit logging, but the same remaining work is now tracked by open PR #63557, which explicitly closes #13131. This issue can close as superseded by that canonical PR, not as shipped.

## Evidence

- **Current main direct tool invoke path has no audit sink:** `/tools/invoke` resolves the tool, runs `runBeforeToolCallHook`, executes `gatewayTool.execute`, and returns JSON. There is no gateway audit JSONL write in this path on current main.
  - file: [src/gateway/tools-invoke-http.ts:267](https://github.com/openclaw/openclaw/blob/1dfa52d0711dfc86e1c04538fe46e3d43961766f/src/gateway/tools-invoke-http.ts#L267)

- **Current main OpenResponses path has no gateway audit context:** `/v1/responses` calls `runResponsesAgentCommand` with session/run/channel/model inputs, but current main does not thread a gateway tool-audit context or write a centralized tool-call audit record there.
  - file: [src/gateway/openresponses-http.ts:684](https://github.com/openclaw/openclaw/blob/1dfa52d0711dfc86e1c04538fe46e3d43961766f/src/gateway/openresponses-http.ts#L684)
