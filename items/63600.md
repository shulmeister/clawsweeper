## Summary

Close as ClawHub/community plugin work. Current main supports explicit custom OpenAI-compatible provider configuration that can point at LocalAI-style `/v1` endpoints, and it already recognizes `localai` as a local streaming-usage-compatible family, but there is no bundled LocalAI provider or docs page. The provider plugin SDK exposes the needed auth/catalog/discovery/setup seams, while VISION.md directs optional provider integrations and plugin promotion to ClawHub/npm. The PR discussion also found factual docs risks around unsupported `LOCALAI_API_KEY` discovery and later quick-start/auth/model-id mismatches, so this should not be kept as a core docs PR.

## Best Possible Solution

Close this core docs PR and direct LocalAI-specific discovery/onboarding/docs to a ClawHub/npm provider plugin using the existing provider SDK. Keep core docs focused on the generic custom OpenAI-compatible `models.providers` path unless a maintainer explicitly wants a small vetted LocalAI mention in the local models guide.
