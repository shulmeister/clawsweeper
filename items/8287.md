## Summary

Keep #8287 open. Current main still has node command advertisement and plugin-registered agent tools, including `api.runtime.nodes` for plugins, but it does not implement the requested node-advertised agent-tool protocol/lifecycle. A maintainer Codex review on the issue already kept it open on 2026-04-26, and the current main inspection still matches that conclusion.

## Best Possible Solution

Keep #8287 open for a maintainer-approved additive gateway protocol/runtime design: define a validated node tool-advertisement schema, store it in `NodeRegistry` with connect/disconnect lifecycle, synthesize agent tools and catalog/effective-inventory entries from connected nodes, route calls through `node.invoke`, and document naming plus policy behavior. The existing `api.runtime.nodes` plugin path and openclaw-tescmd workaround should be acknowledged as current workarounds, not replacements for the requested zero-plugin node tool registration model.

## Risks / Open Questions

- Design needs explicit security review so node-advertised tools do not bypass existing node pairing scopes, command allowlists, plugin policy, or `tools.allow`/`tools.deny` behavior.
- Naming/versioning for dynamic node tools still needs a stable contract for reconnects, conflicts, catalog display, and policy matching.
