## Summary

Keep #8287 open. Current main still does not implement node-registered agent tools: nodes can declare caps/commands/permissions and node-host plugin commands, and plugins can now invoke nodes through `api.runtime.nodes`, but the Gateway still does not accept node-advertised tool definitions, store them with node lifecycle, synthesize per-node agent tools, or expose them through tool catalog/effective inventory. The linked openclaw-tescmd workaround and related hook/tool PRs are useful context, but they do not supersede the requested core protocol/runtime seam.

## Best Possible Solution

Keep #8287 open for a maintainer-approved additive gateway protocol/runtime design: define a validated node tool-advertisement schema, persist it in `NodeRegistry` with connect/disconnect lifecycle, expose it through agent tool construction and tool catalog/effective inventory, route calls through `node.invoke`, and document naming plus `tools.allow`/policy behavior. The existing plugin workaround should be acknowledged but not treated as a full replacement for the requested zero-plugin node tool registration model.
