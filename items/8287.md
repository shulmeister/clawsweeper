## Summary

Keep #8287 open. Current main still exposes nodes as declared command targets plus one static `nodes` agent tool, not as node-advertised, dynamically registered LLM tools. The current plugin runtime now provides a cleaner `api.runtime.nodes` helper for plugin-owned node integrations, but that only addresses part of the workaround and does not satisfy the requested zero-plugin node tool registration model. The issue also has a recent maintainer-authored comment explicitly keeping it open for protocol/runtime design.

## Best Possible Solution

Keep #8287 open for maintainer-approved Gateway protocol and runtime work: add an additive node tool-advertisement schema, validate and store advertised tools on connected node sessions, synthesize node-backed agent tools and catalog/effective inventory entries with node provenance, route execution through existing node invocation authorization, and document naming, reconnect, pairing, and tool policy behavior. Treat the existing plugin `api.runtime.nodes` surface as useful interim plumbing, not as the final resolution.
