## Summary

Keep PR #64661 open. Current main still has the Windows CJK/mojibake failure mode in the shared exec paths, while the PR is a focused bug-fix attempt tied to concrete open reports (#50519 and #56462). It is not implemented on main, not superseded, and not out of core scope; it likely needs review changes before merge, especially around stateful decoding for spawn chunks.

## Best Possible Solution

Keep the PR open and ask for a maintainer review/rework pass: extract the existing node-host Windows codepage decoder into a shared infra module, update `runExec`, `runCommandWithTimeout`, QMD process execution, and TUI local shell to decode captured bytes safely, and add focused GBK/Shift_JIS regression tests including a split-multibyte-chunk case. The linked issues should remain canonical bug context until that fix lands.
