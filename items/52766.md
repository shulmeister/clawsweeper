## Summary

Current `main` already implements the reported gateway handshake-timeout fix: the pre-auth WebSocket timeout is 10 seconds instead of 3 seconds, it is configurable via `OPENCLAW_HANDSHAKE_TIMEOUT_MS`, the gateway server uses that shared timeout for pre-auth sockets, and the changelog records this exact fix as shipped in release `2026.3.22`.
