## Summary

Keep open. Current main has not implemented this PR's requested Slack media dispatcher stripping: the Slack fetch wrapper still forwards `dispatcher` into `fetchWithRuntimeDispatcher`, and the current regression test explicitly expects that forwarding. The provided discussion also links an open reproduced Slack attachment bug, while the merged #64766 covers a different FormData transcription path.

## Best Possible Solution

Keep this PR open as an active Slack bug-fix candidate. The maintainer path is to review/rebase it against current main, reconcile it with #63401, validate the SSRF/runtime-fetch contract, and land the canonical fix with the test changed to assert that Slack media does not forward the upstream dispatcher when that is the intended transport boundary.
