## Summary

PR #70602 is superseded by current main's channel-context fix for the linked heartbeat cache divergence. Normal heartbeat polls now preserve the real originating channel for downstream prompt/tool assembly and do not broadly force `senderIsOwner=false`; owner-only tool filtering remains authorization-based. The linked issue #70417 has already been closed as implemented, while #70418 remains the separate cache-warmer design follow-up.

## Best Possible Solution

Close this PR as superseded by the shipped/current main heartbeat context behavior for #70417. Keep the current auth-based owner-only tool policy, and use #70418 for any future dedicated cache-warmer design rather than adding a heartbeat-specific `keepOwnerTools` path.
