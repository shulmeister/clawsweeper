## Summary

Keep #14079 open. Current main still lacks Linux desktop notification delivery for the headless node host: the node host does not advertise `system.notify`, the invoke dispatcher has no Linux `system.notify`/`notify-send`/libnotify/D-Bus implementation, and docs/CLI still present this notification surface as macOS-node-only. Plugins also cannot transparently fill the existing `system.notify` command because that command is reserved by core. The related #24616 is closed error-message work and explicitly distinguishes itself from this Linux notification feature request.

## Best Possible Solution

Keep this issue open. The product fix belongs in the node-host notification path: add Linux `system.notify` support using `notify-send`/libnotify/D-Bus with graceful headless no-op behavior, then update node-host command declaration, tests, and the public nodes docs. If maintainers want this outside core instead, first expose an explicit plugin API that can safely own or extend this core notification command.
