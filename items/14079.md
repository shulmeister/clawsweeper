## Summary

Keep #14079 open. Current main still does not implement Linux desktop notifications for the headless node host: the node host does not declare `system.notify`, there is no Linux `system.notify` handler, and plugins cannot currently supply that exact command because it is reserved by core. The related #24616 addressed/concerned the error experience and explicitly distinguishes itself from this feature request, so it does not supersede the remaining Linux notification support work.

## Evidence

- **Linux/headless node host does not declare system.notify:** `runNodeHost` advertises the host platform via `process.platform`, but its declared commands are `NODE_SYSTEM_RUN_COMMANDS`, `NODE_EXEC_APPROVALS_COMMANDS`, and plugin node-host commands only. `system.notify` is not in that list, so a Linux node host still will not expose the command requested by `openclaw nodes notify`.
  - file: [src/node-host/runner.ts:187](https://github.com/openclaw/openclaw/blob/1dfa52d0711dfc86e1c04538fe46e3d43961766f/src/node-host/runner.ts#L187)
  - command: `nl -ba src/node-host/runner.ts | sed -n '160,200p'`
  - sha: [1dfa52d0711d](https://github.com/openclaw/openclaw/commit/1dfa52d0711dfc86e1c04538fe46e3d43961766f)
- **Core node-host command constants keep notify separate from run commands:** `NODE_SYSTEM_RUN_COMMANDS` contains `system.run.prepare`, `system.run`, and `system.which`; `NODE_SYSTEM_NOTIFY_COMMAND` is defined separately as `system.notify` and is not included in the node-host advertised run-command list.
  - file: [src/infra/node-commands.ts:1](https://github.com/openclaw/openclaw/blob/1dfa52d0711dfc86e1c04538fe46e3d43961766f/src/infra/node-commands.ts#L1)

- **Node-host invoke path has no system.notify implementation:** The node-host invoke dispatcher handles `system.which` and then plugin-registered commands before continuing into run handling. There is no Linux/headless `system.notify` branch here, and the repo search found no `notify-send`, `libnotify`, or `dunstify` implementation for this path.
  - file: [src/node-host/invoke.ts:472](https://github.com/openclaw/openclaw/blob/1dfa52d0711dfc86e1c04538fe46e3d43961766f/src/node-host/invoke.ts#L472)

The `system.notify` note references macOS app notification permission state, not Linux desktop delivery.
