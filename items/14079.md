## Summary

Keep #14079 open. Current main still does not implement Linux desktop notification delivery for the existing `system.notify` / `openclaw nodes notify` surface. The related closed #24616 covered the misleading Linux error path, not the requested `notify-send`/libnotify/D-Bus support, and the existing maintainer/Codex comment already reached the same keep-open conclusion.

## Best Possible Solution

Keep #14079 open. The best maintainer path is a focused core/node-host implementation for Linux `system.notify`: detect a usable desktop session (`DISPLAY` or `WAYLAND_DISPLAY`), advertise `system.notify` only when delivery is available, use `notify-send` first with a Freedesktop D-Bus/libnotify fallback, degrade cleanly on headless hosts, add node-host/gateway/CLI tests, and update the public nodes docs and CLI help. If maintainers prefer external ownership, core first needs a documented way for plugins to implement or augment the reserved `system.notify` command.
