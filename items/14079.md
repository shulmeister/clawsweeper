## Summary

Current main still does not implement Linux desktop delivery for the existing `system.notify` / `openclaw nodes notify` surface. The related #24616 only addressed misleading error UX; it does not supersede the Linux notification feature request.

## Best Possible Solution

Keep #14079 open. The best path is to add Linux support in the headless/node-host notification path: detect desktop-session availability, advertise `system.notify` only when delivery is available, implement `notify-send` with a D-Bus/libnotify fallback, degrade cleanly on headless hosts, cover node-host/Gateway/CLI behavior with focused tests, and update the nodes/CLI docs. If maintainers prefer this outside core, first add a documented plugin API that can safely own or extend the reserved `system.notify` surface.

## Evidence
