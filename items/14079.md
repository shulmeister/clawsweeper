## Summary

Keep #14079 open. Current main still does not provide Linux desktop notification delivery for the existing `system.notify` / `openclaw nodes notify` surface: the headless node host does not advertise `system.notify`, the node-host invoke path has no notify handler, no `notify-send`/libnotify/D-Bus implementation is present, docs and CLI still describe the surface as macOS-node-only, and plugins cannot replace this exact command because core reserves it.

## Best Possible Solution

Keep the issue open. The best product fix is to add Linux `system.notify` support in the node-host notification path: advertise the command when appropriate, implement `notify-send` with libnotify/D-Bus fallback and graceful headless no-op behavior, add focused tests for command advertisement/dispatch/error behavior, and update the nodes CLI/docs. If maintainers prefer this outside core, first add an explicit plugin API that can safely extend or own this core notification surface.
