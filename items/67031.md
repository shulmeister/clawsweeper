## Summary

Keep #67031 open. Current main has partially improved the OpenAI-compatible `/v1/chat/completions` image limits, but the broader request remains real: tool/transcript image sanitization still only exposes `agents.defaults.imageMaxDimensionPx`, the sanitizer byte target remains the 5 MB default, the shared image pixel guard remains a hardcoded 25 MP cap, and top-level `media` has no `media.image.*` schema. Related #40950 is still open and explicitly scoped to sandbox staging only, so it does not supersede this broader config gap.

## Best Possible Solution

Keep the issue open and implement a conservative config contract that threads image byte, input-pixel, dimension, and per-request total limits through the core image sanitization and media-loading layers. Reuse the existing `agents.defaults.imageMaxDimensionPx` as a backward-compatible alias, preserve the new `gateway.http.endpoints.chatCompletions.*` controls for endpoint-specific overrides, and treat #40950/#40880 as the narrower sandbox-media staging follow-up rather than a replacement for this issue.
