## Summary

Keep PR #38640 open. Current main still does not implement its prompt-level LLM retry wrapper, and the PR still has useful but unresolved retry-boundary work. Adjacent main behavior mitigates some rate-limit pain through SDK retry-wait caps, auth-profile rotation, cooldowns, and model fallback, but that is not the same as the PR's exponential retry around `activeSession.prompt(...)`.

## Best Possible Solution

Keep #38640 open for maintainer judgment. The best product/code path is to either consolidate it explicitly into the canonical configurable LLM/provider retry work (#49376/#61443) or rework this PR at the intended provider/runner boundary with abortable backoff, shared failover status parsing, clear interaction with SDK retries, and focused unit plus runner/failover coverage.
