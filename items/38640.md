## Summary

Keep PR #38640 open. Current main has adjacent provider SDK retry caps, auth-profile rotation, cooldown, and fallback behavior, but it still does not implement the PR's prompt-level exponential-backoff retry wrapper around `activeSession.prompt(...)`. The PR discussion also contains unresolved correctness concerns, so this is not a conservative auto-close candidate.

## Best Possible Solution

Keep the PR open for maintainer judgment or supersede it explicitly with a canonical LLM retry design. A best implementation would add prompt/provider retry only at the intended boundary, honor abort signals during backoff, parse numeric and string status codes consistently with `failover-error.ts`, respect provider SDK behavior, and add focused unit plus runner/failover coverage.
