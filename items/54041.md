## Summary

Keep open. Current main still has only partial Anthropic request shaping: users can add raw `anthropicBeta` values and enable the 1M context beta, but there is no native Anthropic `context_management` payload support, no compaction/context-editing stream semantics, and no provider-defined `memory_20250818` tool path. The issue discussion is concrete and recent, including a production +1 asking specifically for Anthropic server-side compaction and post-compaction reinjection behavior.

## Best Possible Solution

Keep this open for maintainer/product design in the bundled Anthropic provider. A good fix would add documented Anthropic model params for beta/header handling, validated `context_management` body shaping, request/stream/replay tests for compaction and context editing, handling for compaction blocks and `pause_after_compaction`, and a clear decision on whether `memory_20250818` can wrap existing OpenClaw memory tools or needs a provider-defined-tool SDK seam.
