## Summary

Current main has related macOS LaunchAgent restart hardening, but it does not implement the requested two-phase `bootout`/wait/`bootstrap + kickstart` ordering. The normal `restartLaunchAgent` path still performs stale-port cleanup, enables the service, then runs `launchctl kickstart -k`; the current unit test explicitly asserts no `bootout` or `bootstrap` occurs on the successful restart path. The detached handoff path also still starts with `kickstart -k`.

## Best Possible Solution

Keep #70801 open. The best path is a focused macOS LaunchAgent lifecycle change that makes `restartLaunchAgent` stop the old launchd job, verify the old runtime/port is gone, then start the LaunchAgent cleanly, with explicit unit coverage for ordering and separate coverage for the detached self-restart handoff so the existing recovery paths do not regress.

## Evidence

There is no normal-path `bootout`, old PID wait, or `bootstrap` before start.
