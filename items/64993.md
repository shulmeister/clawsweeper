## Summary

Keep open. Current `main` partially fixes the narrow `senderIsOwner` header spoof by deriving owner/non-owner from two bearer tokens, but it still trusts mutable `x-session-key`, `x-openclaw-account-id`, and `x-openclaw-message-channel` headers for loopback MCP scope and cache keys. The broader scope-binding issue reported in #64993 is therefore not implemented away on `main`.
