## Summary

Current main has not fully implemented this PR's requested Chat-page localization. Some chat header controls already use `t("chat.*")`, but the main Chat view and newer chat subcomponents still render hard-coded English strings. The PR remains directionally useful, though it needs a rebase/update for the current file split, all 14 supported locales, generated i18n metadata, and the already-reviewed no-op `fallback` parameter issue.

## Best Possible Solution

Keep the PR or its work open, but update it for current main: add the missing Chat keys to `ui/src/i18n/locales/en.ts`, replace hard-coded Chat strings across `ui/src/ui/views/chat.ts` and `ui/src/ui/chat/*` components with `t()` calls, regenerate all supported locale bundles and `.i18n` metadata through the Control UI i18n pipeline, and either remove `{ fallback: ... }` params or implement real fallback support in `t()` with tests.
