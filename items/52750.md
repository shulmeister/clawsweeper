## Summary

Current `main` already preserves Telegram topic/thread routing for ACP parent-stream system events. ACP parent-stream announcements enqueue the parent session's `deliveryContext`, heartbeat preflight resolves that queued context, heartbeat delivery merges it ahead of stale session route data, and the Telegram plugin explicitly preserves heartbeat thread IDs for group routes. A regression test added on `main` covers the exact drift case from the issue and expects delivery back to the originating topic.
