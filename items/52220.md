## Summary

Current `main` already keeps restart-sentinel wake state alive across early outbound failures, so the reported Telegram-startup race no longer matches the code. The wake is enqueued before outbound delivery, outbound notice delivery is retried with backoff, and targeted tests cover transient and exhausted-not-ready cases.
