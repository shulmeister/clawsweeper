## Summary

Keep open. Current main still lacks the requested Telegram send-level NO_REPLY guard in sendMessageTelegram, while Slack has the parity guard and regression tests. Higher-level Telegram/agent suppression exists for exact or prefix silent replies, but it does not implement the defensive adapter guard requested here, and open PR #68304 still tracks that exact fix.

## Best Possible Solution

Keep this issue open until a focused Telegram adapter fix lands, or a maintainer explicitly decides the send-level guard is unnecessary. The best code path is to add an early isSilentReplyText check in extensions/telegram/src/send.ts before Telegram API calls, bypassing only exact NO_REPLY text when there is no media or inline keyboard/buttons, and add send.test coverage proving no Telegram API call for exact/whitespace NO_REPLY plus bypass coverage for media/buttons.
