## Summary

Keep open. Current `main` still lacks the Telegram-side exact-`NO_REPLY` pre-send guard described in the report, and the direct Telegram message action path still calls `sendMessageTelegram` without any helper-level suppression. There is partial mitigation in higher-level reply delivery and streaming code, but not a complete implementation of this issue’s requested fix.

## Evidence
