## Summary

Current `main` already contains a targeted fix for the cron double-announce path that matches this report: `dispatchCronDelivery()` now marks `deliveryAttempted = true` before both early returns in text finalization, which prevents the heartbeat fallback from enqueueing a second announce. The repo also has a dedicated regression test file documenting this exact bug and covering both early-return cases plus replay idempotency.
