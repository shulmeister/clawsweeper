## Summary

Keep open. Current main has useful mitigations, especially opt-in `isolatedSession` and `lightContext`, but non-isolated heartbeat runs still intentionally reuse the existing heartbeat/main session and there is no heartbeat `sessionRotation` policy with turn, age, token, or consecutive-error thresholds. The broader #45390 session TTL tracker remains open, but this heartbeat-specific request is not fully implemented or clearly superseded.

## Best Possible Solution

Keep this open or explicitly consolidate it into #45390 after maintainer review. A complete fix should add a documented heartbeat lifecycle policy under `agents.defaults.heartbeat` or intentionally change the default to isolated heartbeat sessions, with schema validation, archive/summary behavior, health/error thresholds, tests for non-isolated heartbeat rollover, and docs explaining interaction with `isolatedSession`, `lightContext`, daily/idle reset, and the related session-memory poisoning fix.
