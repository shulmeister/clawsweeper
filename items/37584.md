## Summary

Keep open. Current main still does not implement per-agent `agents.list[].tools.web.search.*` overrides for web-search provider, API key, timeout, or related provider settings. The current contract keeps per-agent `tools` limited to policy/sandbox/exec/fs controls, while web-search settings are resolved from global `tools.web.search` plus plugin-owned `plugins.entries.<plugin>.config.webSearch.*` config. The reporter also reaffirmed the issue on 2026-04-20, and the related reports cover adjacent web-search/provider or tool-policy work, not this exact agent-scoped web-search settings gap.

## Best Possible Solution

Keep the issue open. The best implementation path is to add a documented agent-scoped web-search settings contract, validate it in the agent schema/types, merge global/plugin/default/agent settings when constructing the effective agent runtime, thread either the effective search config or current agent id through `createOpenClawTools()` -> `createWebSearchTool()` -> `runWebSearch()`/provider context, and add regression tests plus docs. If maintainers want this to stay provider-plugin-owned, first add a generic SDK/core seam that lets web-search providers resolve per-agent credentials and settings safely.
