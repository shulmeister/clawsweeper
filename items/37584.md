## Summary

Keep open. Current main still supports global/plugin-scoped web search settings, but not per-agent `agents.list[].tools.web.search.*` overrides. The strict per-agent tools schema only covers tool policy/sandbox/exec controls, and the `web_search` runtime resolves `cfg.tools.web.search` without an agent id or agent-scoped config. The reporter also reaffirmed the issue on 2026-04-20, and the provided related items do not identify a canonical superseding tracker for this exact missing capability.

## Best Possible Solution

Keep the issue open. The likely implementation path is to add a documented agent-scoped web-search settings contract, merge global/default/agent config at agent runtime construction, thread the effective search config or agent id through `createOpenClawTools()` -> `createWebSearchTool()` -> web-search runtime/provider context, and update schema help, docs, and regression tests. If maintainers prefer this as plugin-owned behavior, first add a generic core/SDK seam for per-agent provider credentials.
