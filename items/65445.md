## Summary

Keep this issue open. Current main partially mitigates session cruft: maintenance archives pruned/capped transcripts as `.deleted`, expires old `.deleted` and `.reset` archives, caps `sessions.json.bak.*` rotations, and optional disk-budget cleanup can remove archive artifacts under pressure. The requested default garbage collection is still not implemented: age-based cleanup does not sweep `.jsonl.bak.*` transcript archives, there is no recycle/soft-delete config, and `sessions_saved.json` is not recognized in current source/docs/tests.

## Best Possible Solution

Keep this issue open and implement a focused session-maintenance enhancement with tests for orphaned `.jsonl.deleted.*`, `.jsonl.reset.*`, `.jsonl.bak.*`, and any legacy `sessions_saved.json` behavior maintainers still want to support. Align `src/config/sessions/*`, gateway archive helpers, config schema/help, and public docs around either the proposed recycle/retention model or a clearly documented replacement using existing maintenance controls.
