## Summary

Keep open. Current main still drops any target for the cron wake action: the agent tool sends only mode/text, the gateway wake handler forwards only mode/text, and the CronService wake path enqueues/request-wakes without agentId or sessionKey. Normal cron job execution does pass job.agentId/sessionKey, so the reported gap remains specific and actionable.
