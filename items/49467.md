## Summary

Current main already supports Feishu multi-account routing by `accountId`, and Feishu group mention handling is evaluated per bot account, so the requested per-bot routing outcome is already achievable without adding `channels.feishu.accounts.*.defaultAgent`.

## Risks / Open Questions

- The exact requested config shape (`channels.feishu.accounts.<id>.defaultAgent`) does not exist; the supported mechanism is account-scoped `bindings[].match.accountId`.
- To get one-bot-per-mention behavior in shared groups, Feishu accounts still need normal mention gating (`requireMention`) and correct per-account bindings.

## Close Comment

Closing this as already implemented. Codex review verified that current OpenClaw routes Feishu inbound traffic with the active Feishu `accountId` (`extensions/feishu/src/bot.ts:590`), evaluates mentions against that account’s own bot Open ID (`extensions/feishu/src/bot-content.ts:229`), and gives account-scoped bindings their own routing tier (`src/routing/resolve-route.ts:777`). The supported configuration is `bindings[].match.accountId`, documented in `docs/gateway/config-channels.md:739` and `docs/concepts/multi-agent.md:245`, rather than a per-account `defaultAgent` field under `channels.feishu.accounts.*`. In practice, that already lets separate Feishu bot apps in the same group route to different agents by binding each Feishu account to the desired agent. This is present in the latest shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). If you still see misrouting on that release or newer, please open a new issue with the exact `bindings`/`channels.feishu.accounts` config and a concrete repro.
