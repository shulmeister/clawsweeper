## Summary

Keep open. Current main still rejects symlink hops and outside-lexical paths unless they resolve inside the configured workspace root, so the linked workspace-bootstrap symlink bug is not implemented. The PR is a plausible implementation candidate for #64472, but it should be revised before merge because review comments identify real boundary-widening and coverage gaps.

## Best Possible Solution

Keep the PR and #64472 open, but require a safer revision before merge: scope any exception to workspace bootstrap reads whose workspace root is within the canonical OpenClaw state root, canonicalize that state root before comparison, avoid allowing direct absolute paths to sibling ~/.openclaw directories, and add positive and negative regression tests for the intended kb symlink case, direct credential/session paths, and non-OpenClaw boundary roots.
