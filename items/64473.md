## Summary

Keep open. Current main still enforces workspace-root containment for symlink hops and outside-lexical paths, so the linked bug is not implemented. The PR is a plausible focused fix for #64472, but the submitted patch should not be closed as shipped or merged as-is because review comments identify a real boundary-widening risk and missing regression coverage.

## Best Possible Solution

Keep the PR/open bug path active, but require changes before merge: scope any exception to workspace bootstrap reads whose boundary root is itself within the canonical OpenClaw state root, canonicalize the OpenClaw state root before comparison, avoid allowing direct absolute paths to sibling `~/.openclaw` directories, and add regression coverage for both the intended `workspace/AGENTS.md -> ../kb/AGENTS.md` case and the security-negative cases raised in review. The linked issue #64472 can remain the canonical user problem while this PR is revised or replaced.
