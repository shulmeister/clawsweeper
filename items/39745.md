## Summary

Keep PR #39745 open. Current main has already implemented the Twitch timeout cleanup, but Synology Chat and Zalouser still create setTimeout-backed Promise.race timeouts without storing or clearing the timer handle when the primary operation wins, so the PR remains partially useful rather than obsolete.

## Best Possible Solution

Keep this PR open. Ask the author to rebase on current main, drop or resolve the obsolete Twitch hunk, retain the Synology Chat and Zalouser timer cleanup, and add focused fake-timer success-path coverage where feasible. Maintainers should separately decide whether Synology Chat should keep the 120s plugin-local timeout with cleanup or remove it in favor of the core agent timeout.
