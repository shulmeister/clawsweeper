## Summary

Current main no longer has a distinct subagent serialization path that would drop token usage. Subagents are launched through the normal Gateway `agent` pipeline with the child session key, the embedded runner carries provider usage into `agentMeta`, provider transports populate usage for OpenAI-compatible and Anthropic streams, and session usage readers/events consume assistant `message.usage` from JSONL for all sessions.

## Best Possible Solution

Close as implemented and keep the current unified subagent/agent execution path. If a new zero-usage case appears, it should be filed with a current transcript excerpt and provider/model details, because the remaining likely failure would be provider-specific usage omission rather than a subagent JSONL serialization path.
