## Summary

Keep #38430 open. Current main still has no implementation of the requested default path expansion or the proposed OPENCLAW_EXPAND_PATHS control, and several current prompt/tool display paths still intentionally compact home-directory paths to `~/`. The related PR #38495 is closed unmerged, so it does not supersede or implement the issue.

## Best Possible Solution

Keep the issue open for maintainer/product direction. The best fix is a narrow, tested user-visible path policy: either add prompt guidance requiring full absolute local paths in final replies, or add a conservative reply/tool-result normalizer that expands `~/` only for local file paths before user delivery. It should not blindly remove internal prompt compaction unless maintainers accept the token-cost tradeoff, and it should cover skill locations, edit success messages, tool display metadata, and any channel/media delivery surfaces that expose local paths.
