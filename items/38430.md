## Summary

Keep #38430 open. Current main still does not implement the requested default of full absolute file paths in agent replies, nor the related PR’s proposed `OPENCLAW_EXPAND_PATHS` control. Main still intentionally compacts some prompt-visible and user-visible home paths to `~/...`, and the related PR #38495 is closed unmerged rather than a canonical shipped fix.

## Best Possible Solution

Keep the issue open for maintainer product direction. A robust fix would decide the exact surface to change, then add either OpenClaw-owned prompt guidance for user-visible final replies, a delivery-time formatter for user-facing path text, or a documented config/env switch, with regression coverage for final replies and tool-result summaries. It should also explicitly preserve or revise the existing prompt-token skill path compaction and sandbox relative-path guidance rather than changing those accidentally.
