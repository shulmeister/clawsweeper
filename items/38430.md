## Summary

Keep #38430 open. Current main still does not implement a user-visible policy that expands `~/...` to full absolute paths in agent replies. The active prompt still prefers relative paths in sandboxed contexts, skill prompt construction intentionally compacts home paths to `~/...`, host edit recovery can still emit tool text containing the original tilde path, and the related PR #38495 was closed unmerged.

## Best Possible Solution

Keep the issue open for maintainer product direction. The likely fix belongs either in prompt guidance for final user-visible replies, a delivery-time formatter for file-path text, or a documented config/env switch, with regression coverage that distinguishes final replies from prompt-token-saving skill path compaction and tool input expansion.
