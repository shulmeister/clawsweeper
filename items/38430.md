## Summary

Keep #38430 open. Current main still lacks a user-visible policy or formatter that expands `~/...` to full absolute paths in agent replies. The code still intentionally compacts home paths to `~/...` in skill prompt text, tests assert that behavior, host-edit recovery can echo the original tilde path in tool output, and no `OPENCLAW_EXPAND_PATHS` or equivalent path-expansion option exists on main. The related PR #38495 remains closed/unmerged in the provided issue context, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open. The best path is a maintainer-chosen product fix that distinguishes internal prompt/tool path handling from final user-visible replies, such as explicit final-reply prompt guidance, a delivery-time formatter, or a documented config switch, with regression coverage for final replies and preserved tests for intentional skill-prompt compaction and tool input expansion.
