## Summary

Keep open. Current `main` still generates auto-TTS audio in the OpenClaw temp directory, then forwards that temp file path straight into QQ Bot outbound delivery. QQ Bot outbound still restricts local media to `~/.openclaw/media/**`, and the QQ Bot channel adapter does not consume the generic `mediaLocalRoots`/`mediaAccess` contract that would allow temp-root files. That matches the reported failure mode rather than proving a fix on `main`.
