## Summary

Keep #8724 open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still has no typed global or per-model text-generation timeout under `agents.defaults.models`, and the existing timeout controls are an overall agent run timeout, an idle/no-chunk LLM watchdog, or image/tool-specific request timeouts. Related first-token items are adjacent but do not solve the reported case where a model keeps streaming non-useful output past a model-specific total budget.

## Best Possible Solution

Keep #8724 open until OpenClaw either implements and documents a typed global/per-model text-generation timeout contract or a maintainer explicitly rejects the product direction. A complete fix should add schema/type/generated-doc support for the config shape, resolve the effective timeout per fallback candidate, keep total generation timeout distinct from idle/first-token behavior, abort and log timeout events, integrate with fallback handling, and add regression coverage for a stream that keeps emitting chunks past the configured budget.
