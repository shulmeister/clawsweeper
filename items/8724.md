## Summary

Keep #8724 open. Current main still lacks the requested global/per-model generation-timeout contract under `agents.defaults.models`, and the existing timeout behavior is a global run timeout plus a global LLM idle watchdog, not a model-specific total generation budget for streams that keep emitting non-useful text.

## Best Possible Solution

Keep #8724 open until OpenClaw either implements and documents a typed global/per-model generation-timeout contract or a maintainer explicitly rejects the product direction. A complete implementation should validate a chosen config shape, resolve the effective timeout per fallback candidate, keep it distinct from idle/no-token behavior, abort and log timeout events, and add regression coverage for a stream that keeps emitting chunks past the configured budget.
