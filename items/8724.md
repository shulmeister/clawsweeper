## Summary

Keep #8724 open. Current main still lacks a typed global or per-model total generation timeout for active streaming model loops. The implemented surfaces remain global run timeout and LLM idle/no-chunk timeout, which do not enforce the requested per-model generation budget while chunks continue to arrive.

## Best Possible Solution

Keep #8724 open until OpenClaw implements and documents a typed global/per-model text-generation timeout. A complete fix should add config types, zod/generated schema/help/docs support; resolve the effective timeout per fallback candidate; abort and log total generation-budget expirations even while chunks keep streaming; classify that timeout for fallback handling; and add regression coverage for a stream that emits chunks past the configured budget.
