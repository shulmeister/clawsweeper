## Summary

Keep #8724 open. Current main still does not implement the requested global/per-model generation timeout contract. It has a global agent run timeout and a global LLM idle watchdog, but no accepted `agents.defaults.models[...].generationTimeout`/per-model timeout config and no total generation budget that stops a model which keeps streaming looping planning text.

## Best Possible Solution

Keep #8724 open until OpenClaw either implements and documents a typed global/per-model generation-timeout contract or a maintainer explicitly rejects the product direction. A complete implementation should validate config under `agents.defaults` and `agents.defaults.models`, plumb the resolved budget into each embedded model attempt and fallback candidate, keep it distinct from idle/first-token timeout behavior, log timeout events, and add regression coverage for a model that keeps streaming non-useful planning chunks past the configured generation budget.
