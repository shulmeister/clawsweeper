## Summary

Keep #8724 open. Current main still has only a global agent run timeout plus a global LLM idle watchdog; it does not accept or resolve the requested global/per-model generation timeout under `agents.defaults.models`, and it does not provide a total generation budget that stops a model that keeps streaming non-useful planning text.

## Best Possible Solution

Keep #8724 open until OpenClaw either implements and documents a typed global/per-model generation-timeout contract or a maintainer explicitly rejects the product direction. A complete solution should validate `agents.defaults.generationTimeout` and/or `agents.defaults.models[model].generationTimeout`, resolve the effective budget per fallback candidate, keep it distinct from idle/first-token timeout behavior, log timeout events, and add regression coverage for a model that keeps streaming non-useful chunks past the configured budget.
