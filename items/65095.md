## Summary

Keep this issue open. Current main does not implement a gateway/client compatibility grace period or an offline/skip version-check mode; it still rejects Gateway WebSocket clients whose advertised protocol range excludes the server protocol. The related #65081 only covers broader release-channel cadence, so closing this would lose the narrower gateway protocol/offline compatibility request.

## Best Possible Solution

Keep this open and narrow it into a gateway protocol compatibility policy. A good fix would define supported protocol ranges, keep incompatible protocol changes explicitly versioned, add tests for old/new client negotiation, document remote/offline behavior, and decide whether a carefully scoped compatibility mode is acceptable. Release-channel documentation can link to #65081, but this issue should remain the tracker for Gateway/client version-skew behavior.
