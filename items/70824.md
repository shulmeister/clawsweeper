## Summary

Do not close PR #70824 as implemented or superseded. Current main has changed the text-only attachment path so images are offloaded instead of silently dropped, but it still does not implement the PR's core merge fix: `mergeProviderModels` continues to overwrite explicit `input` with implicit catalog metadata, and the existing unit test still asserts that behavior. The PR also contains a cross-provider capability fallback that remains product/behavior-sensitive rather than obviously obsolete, because current gateway and runner code are still provider/model-metadata scoped.

## Best Possible Solution

Keep the PR open for maintainer review. The best code direction is to land or rework the narrow `mergeProviderModels` explicit-`input` preservation with regression tests, because current main still overwrites user-declared image capability. The gateway fallback should be reviewed separately and made provider/routing-aware, or dropped, because capability detection should not globally borrow image support from a different provider unless routing will actually select that provider.
