## Summary

Issue #10480 should stay open. Current main still supports Cloudflare AI Gateway onboarding only as the bundled `cloudflare-ai-gateway` provider for Anthropic Messages through the `/anthropic` AI Gateway endpoint, with `cloudflare-ai-gateway/claude-sonnet-4-6` as the default model. The requested Workers AI provider-type or `@cf/...` model-selection path for `openclaw onboard --non-interactive --auth-choice cloudflare-ai-gateway-api-key` is not implemented.

## Best Possible Solution

Keep #10480 open until the Cloudflare AI Gateway plugin gets an explicit Workers AI onboarding path or maintainers make a product-scope decision. The likely implementation would extend the Cloudflare plugin auth-choice/CLI contract to accept either a Workers AI provider type or explicit `@cf/...` model refs, generate the `workers-ai/v1` OpenAI-compatible Gateway base URL/config, update docs at `docs/providers/cloudflare-ai-gateway.md` and non-interactive wizard docs, and add focused tests for the generated provider/model config.
