## Summary

#10480 should stay open. Current main still does not provide a Cloudflare Workers AI model/provider selector for `openclaw onboard --non-interactive --auth-choice cloudflare-ai-gateway-api-key`; the bundled Cloudflare AI Gateway onboarding path remains Anthropic-oriented and sets the Claude Sonnet default, so the user-visible gap from the issue remains real.

## Best Possible Solution

Keep #10480 open. The best fix likely belongs in `extensions/cloudflare-ai-gateway`: add an explicit non-interactive Workers AI mode or model selector, generate OpenAI-compatible provider config for `https://gateway.ai.cloudflare.com/v1/<account>/<gateway>/workers-ai/v1` with `@cf/...` model IDs, update Cloudflare onboarding docs, and add focused tests for the generated provider/model config. If maintainers decide the generic custom-provider path is the intended support boundary, document the exact Workers AI command/config before closing.
