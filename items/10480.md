## Summary

Issue #10480 should stay open. Current main still configures the bundled Cloudflare AI Gateway onboarding path for Anthropic through the `/anthropic` endpoint and does not expose a Workers AI provider type, `@cf/...` model selection, or `/workers-ai/v1` config generation for `openclaw onboard --non-interactive --auth-choice cloudflare-ai-gateway-api-key`.

## Best Possible Solution

Keep #10480 open until the bundled Cloudflare AI Gateway plugin gets an explicit Workers AI onboarding path or maintainers make a product-scope decision. The likely fix belongs in the Cloudflare provider plugin: accept either an endpoint/provider type or explicit `@cf/...` model refs, generate the `/workers-ai/v1` OpenAI-compatible Gateway provider config, update `docs/providers/cloudflare-ai-gateway.md` and non-interactive onboarding docs, and add focused tests for the generated provider/model config.
