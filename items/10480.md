## Summary

Current `main` still does not implement non-interactive Workers AI model selection for Cloudflare AI Gateway onboarding. The bundled Cloudflare AI Gateway path remains Anthropic-oriented: onboarding exposes Cloudflare account/gateway/API-key flags but no generic onboard `--model`, Workers AI model flag, or provider-type selector; the Cloudflare provider non-interactive flow only resolves account ID, gateway ID, and API key; generated config and catalog discovery still target the `/anthropic` endpoint with `api: "anthropic-messages"`. The issue discussion also includes a 2026-04-26 maintainer/Codex comment explicitly keeping this item open for the same missing Workers AI path.

## Best Possible Solution

Keep #10480 open. The best implementation belongs in the bundled `extensions/cloudflare-ai-gateway` provider plugin: add an explicit Workers AI mode or non-interactive model selector, generate an OpenAI-compatible provider config using `https://gateway.ai.cloudflare.com/v1/<account>/<gateway>/workers-ai/v1` and `@cf/...` model IDs, update the Cloudflare AI Gateway provider docs and onboarding CLI docs, and add focused tests for the generated provider/model config. If maintainers decide the generic `custom-api-key` workaround is the intended answer, document the exact Workers AI command/config as the product-supported path before closing.
