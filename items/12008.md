## Summary

Keep #12008 open. Current main still does not expose a typed or schema-validated Gemini safety-settings config field, and the direct Google Generative AI request builder still omits `safetySettings`. The provided discussion also identifies #49529 as the open implementation attempt and includes a latest Codex/maintainer review comment keeping this issue open.

## Best Possible Solution

Keep #12008 open until PR #49529 or an equivalent implementation lands. The best path is a typed and schema-validated `models.providers.*.safetySettings` surface, generated schema/help/docs updates, runtime mapping into direct `google-generative-ai` requests for Google and compatible provider entries, explicit request-level precedence, and regression coverage for validation and payload emission.
