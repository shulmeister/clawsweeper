## Summary

Keep open: current `main` does not expose or forward Gemini safety settings, so issue #12008 is not implemented on main.

## Evidence

- **Direct Gemini text payload has no safety-settings field:** `buildGoogleGenerativeAiParams` only populates `temperature`, `maxOutputTokens`, `thinkingConfig`, `cachedContent`, `systemInstruction`, `tools`, and `toolConfig`. There is no mapping for Gemini `safetySettings` or harm-threshold categories.
  - file: [extensions/google/transport-stream.ts:441](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/extensions/google/transport-stream.ts#L441)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Provider config type has no Google safety block:** `ModelProviderConfig` allows `baseUrl`, `apiKey`, `auth`, `api`, `injectNumCtxForOpenAICompat`, `headers`, `request`, and `models`. There is no provider-level safety-settings schema here.
  - file: [src/config/types.models.ts:114](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/config/types.models.ts#L114)

  - file: [docs/providers/google.md:286](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/docs/providers/google.md#L286)
