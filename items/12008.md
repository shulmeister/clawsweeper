## Summary

Keep open. Current main still does not expose Gemini safety settings in the model provider config surface or map them into direct Google Generative AI requests. Related PR #49529 is relevant implementation work, but it remains open/not merged in the provided GitHub context, so #12008 still tracks a missing user-facing capability.

## Best Possible Solution

Keep #12008 open until PR #49529 or an equivalent implementation lands. The likely maintainer path is to add a typed, schema-validated models.providers.*.safetySettings config surface, document it, inject it only on the google-generative-ai request path, preserve explicit request-level safety settings if supported, and add config validation plus payload tests for direct Google Gemini and custom provider keys using the Google adapter.
