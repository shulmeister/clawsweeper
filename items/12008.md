## Summary

Keep open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still lacks a schema-validated Gemini safety-settings config surface and does not map safetySettings into direct google-generative-ai payloads. The provided GitHub context also identifies PR #49529 as the relevant implementation path, but it is open/unmerged there and the latest maintainer review comment already kept #12008 open.

## Best Possible Solution

Keep #12008 open until PR #49529 or an equivalent implementation lands. The maintainer-ready path is a schema-validated models.providers.*.safetySettings config field, generated schema/docs updates, runtime mapping into direct google-generative-ai payloads while preserving explicit request-level settings, and regression tests for both bundled Google and custom provider keys using the Google adapter.
