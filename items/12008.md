## Summary

Keep open. Current main still does not expose Gemini safety settings in provider config/schema or map them into outgoing Google Generative AI requests. The linked PR #49529 is relevant implementation work, but it is not on current main, so the requested capability remains missing.

## Best Possible Solution

Keep #12008 open until PR #49529 or an equivalent implementation lands. The likely path is to add a typed and schema-validated models.providers.*.safetySettings config surface, document it, map it into Google Generative AI payloads in the bundled Google provider without overriding explicit request-level settings, and cover config validation plus transport payload tests.
