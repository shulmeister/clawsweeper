## Summary

Keep PR #70596 open. Current main has not implemented explicit builtin-local memory prewarming: post-attach startup only schedules the memory startup path for QMD, and `startGatewayMemoryBackend` still skips every non-QMD backend. The PR is a focused attempt at a real performance issue, but it is not safe to close as implemented or obsolete; it also needs revision because the current PR file list does not touch the QMD-only post-attach gate and the existing review comment identifies a supported risk around poisoning the cached builtin manager after a failed startup probe.

## Best Possible Solution

Keep the PR open and ask for a revision: schedule the memory startup path for explicit builtin local configs, make failed provider initialization retry-safe or prewarm through a disposable path that cannot poison the default manager, and add regression coverage at both `startGatewayPostAttachRuntime` and `startGatewayMemoryBackend` for explicit-local, non-local skip, and failed-prewarm retry behavior.
