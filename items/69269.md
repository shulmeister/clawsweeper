## Summary

Keep #69269 open. Current main still has the coordinated invariant gap described by the issue: several reset/compaction hook contexts still omit or pass through raw `messageProvider`, and the auto-reply compaction accounting helper still ignores successful zero-token compaction results. The linked PR #69270 appears to be the right canonical implementation track, but it is open and its head commit is not present on current main.

## Best Possible Solution

Keep #69269 open and use #69270, or an equivalent maintainer-reviewed patch, as the implementation path. The fix should introduce one shared hook `messageProvider` resolver, apply it to subscribe-time compaction, engine-owned compaction, command reset, and gateway reset, and update compaction accounting so finite zero-token outcomes reset cached `totalTokens`, freshness, input/output/cache breakdowns, and cost state consistently. Add focused regression coverage for mixed-case explicit providers, channel-scoped direct keys, non-channel keys that must remain unresolved, gateway/manual compaction bookkeeping, and `tokensAfter === 0`.
