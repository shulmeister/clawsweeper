## Summary

Keep #14248 open. Current main at 82262387651ab270b565f71533ce4937c9c00dc6 still does not expose Slack Socket Mode ping/pong tuning in config, schema, runtime, tests, or docs, while recent issue comments report production impact and PR #41473 remains open/unmerged.

## Best Possible Solution

Keep #14248 open until PR #41473 or an equivalent patch lands on main. The implementation should add a supported channels.slack.socketMode config object for clientPingTimeout, serverPingTimeout, and pingPongLoggingEnabled; validate it in the Slack config schema/types/UI hints; preserve account inheritance; pass values directly to SocketModeReceiver before start; add focused config/runtime tests; update the public Slack and configuration docs; and then close duplicate #15170 as already covered.
