## Summary

Current main still does not expose Slack Socket Mode ping/pong tuning. The Slack runtime constructs SocketModeReceiver without clientPingTimeout/serverPingTimeout/pingPongLoggingEnabled, the Slack config type/schema has no socketMode surface and is strict, docs/changelog do not document a shipped fix, and the related PR #41473 remains only an implementation path in the provided context rather than behavior on main.

## Best Possible Solution

Keep #14248 open until PR #41473 or an equivalent patch lands on main. The product fix should add a supported channels.slack.socketMode config object for clientPingTimeout, serverPingTimeout, and pingPongLoggingEnabled; validate it in Slack types/schema/metadata; preserve account inheritance; pass it into SocketModeReceiver; add focused Slack config/runtime tests; and update the public Slack/config docs.
