## Summary

Keep open. Current `main` still does not expose Slack Socket Mode ping/pong timeout settings: the Slack config schema has no `socketMode`, `clientPingTimeout`, `serverPingTimeout`, or `pingPongLoggingEnabled` fields and is strict, while the Slack Socket Mode receiver is still constructed without passing any of those options.

## Evidence

- **Slack config schema omits Socket Mode timeout fields and is strict:** `SlackAccountSchema` lists Slack channel fields through `typingReaction`, then calls `.strict()`. There is no `socketMode`, `clientPingTimeout`, `serverPingTimeout`, or `pingPongLoggingEnabled` field, so the proposed config shape is not accepted on current `main`.
  - file: [src/config/zod-schema.providers-core.ts:930](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/config/zod-schema.providers-core.ts#L930)
