## Summary

Current main still does not expose or pass through Slack Socket Mode ping/pong timeout options. The Slack plugin constructs `SocketModeReceiver` with only `appToken`, `autoReconnectEnabled`, and shared `clientOptions`; the Slack config type/schema/docs/UI hints contain no `clientPingTimeout`, `serverPingTimeout`, or `pingPongLoggingEnabled` surface. The provided issue discussion includes concrete production reports, and the related PR #41473 was open/unmerged in the supplied context, so this remains actionable OpenClaw Slack plugin work rather than a closeable duplicate or stale report.

## Best Possible Solution

Keep this issue open until the Slack plugin exposes a supported `channels.slack.socketMode` config surface and passes `clientPingTimeout`, `serverPingTimeout`, and `pingPongLoggingEnabled` into `SocketModeReceiver` at construction time. A good fix should update `src/config/types.slack.ts`, `src/config/zod-schema.providers-core.ts`, config help/labels/generated metadata, Slack docs, and the `createSlackBoltApp` tests so the constructor pass-through is covered. PR #41473 is the related starting point, but it needs to land or be superseded by an equivalent current-main patch before this issue can close.
