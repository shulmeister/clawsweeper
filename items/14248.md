## Summary

Keep #14248 open. Current main still does not expose or pass through Slack Socket Mode ping/pong tuning options, and the related PR #41473 is open/unmerged in the provided context, so this remains actionable Slack plugin work rather than a closeable item.

## Best Possible Solution

Keep the issue open until the Slack plugin adds a supported `channels.slack.socketMode` config surface and passes `clientPingTimeout`, `serverPingTimeout`, and `pingPongLoggingEnabled` into `SocketModeReceiver` construction. The fix should update Slack config types, strict zod schema, UI/help metadata, Slack docs, and `createSlackBoltApp` tests. PR #41473 is the related starting point, but it needs to land or be superseded by an equivalent current-main patch before #14248 can close.
