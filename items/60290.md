## Summary

Keep open. Current main still has the session-store behavior this PR is trying to change: `updateSessionStoreAfterAgentRun` merges the full `next` entry, so a stale in-memory `status` can still be written over the persisted terminal status. The PR also has an unresolved review finding on one of its bundled MiniMax changes, so it needs maintainer/author follow-up rather than conservative cleanup closure.

- **Current main still merges full next entry:** At current main SHA 7bbd47349ec37f01a8f2b1dc219bf950615f361b, `updateSessionStoreAfterAgentRun` calls `mergeSessionEntry(store[sessionKey], next)` without omitting `status`, so the specific session-status fix from PR #60290 is not implemented on main.
