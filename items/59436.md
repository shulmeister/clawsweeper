## Summary

Keep open. Current `main` still builds every candidate session row before applying `search`, `activeMinutes`, and `limit`, and it still resolves child sessions with a per-row full-store scan, so this optimization is not already implemented.

## Evidence

- **`listSessionsFromStore()` still does row build before filters/limit:** `Object.entries(store)` is filtered, then immediately mapped through `buildGatewaySessionRow()`, then sorted, and only afterward does the function apply `search`, `activeMinutes`, and `limit`.
  - file: [src/gateway/session-utils.ts:1483](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/gateway/session-utils.ts#L1483)
  - command: `nl -ba src/gateway/session-utils.ts | sed -n '1461,1568p'`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Child-session lookup is still per-row store scanning:** `buildGatewaySessionRow()` still calls `resolveChildSessionKeys(key, store)`, and `resolveChildSessionKeys()` still loops `Object.entries(store)` for each row instead of using a prebuilt index.
  - file: [src/gateway/session-utils.ts:291](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/gateway/session-utils.ts#L291)
  - command: `nl -ba src/gateway/session-utils.ts | sed -n '285,340p' && nl -ba src/gateway/session-utils.ts | sed -n '1330,1340p'`
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Independent PR validation supports the fix shape:** In the PR discussion, `anyech` reported a stage repro on a live-like store where the same optimization shape reduced `rowBuildCount` from 381 to 2 and total wall time from about 1313 ms to about 216 ms for the filtered hot path. ([github.com](https://github.com/openclaw/openclaw/pull/59436))
- **Recent discussion says the live perf lane still exists:** A follow-up PR comment on April 19, 2026 reported `sessions.list` still taking about 10.9 s on a deployment with roughly 700 sessions across 3 stores, which argues against closing this as obsolete or already fixed. ([github.com](https://github.com/openclaw/openclaw/pull/59436))
