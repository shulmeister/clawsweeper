## Summary

Keep this PR open. Current main does not implement the PR’s bundle-MCP lenient tool-listing change: the bundle MCP runtime still calls the SDK’s strict `client.listTools()`, and the pinned MCP SDK still validates `inputSchema.type` as literal `"object"`. However, this PR should not be merged as-is because its raw `client.request()` path bypasses SDK `listTools()` metadata caching, and its explicit non-object schema pass-through can still leak invalid tool schemas into provider payloads after bundle MCP tools are appended.

## Best Possible Solution

Keep this PR open as a useful implementation candidate, but require rework before merge: fix the bundle MCP `tools/list` path in `src/agents/pi-bundle-mcp-runtime.ts` without losing SDK metadata behavior, filter or reject explicit non-object root input schemas before cataloging, and add a harness regression for a configured MCP server whose `inputSchema` omits `type`. The linked bug being closed for the separate loopback normalization path is not enough to close this PR as obsolete.
