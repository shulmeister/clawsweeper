## Summary

Keep #70202 open. Current main has multi-model configuration surfaces after onboarding (`openclaw configure`, model allowlists, fallback chains, and model picker/session switching), but the actual `openclaw onboard` path still prompts for a single default model and applies only that primary model. The requested onboarding-time flow for selecting multiple models plus one default remains missing.

## Best Possible Solution

Keep the issue open. The focused product/code path is to extend `src/wizard/setup.ts` after the default model prompt so onboarding can reuse `promptModelAllowlist`, `applyModelFallbacksFromSelection`, and `applyModelAllowlist`, preserving one primary default while letting users select additional models for `/model` and fallback/cost management. Add focused onboarding tests and update `docs/start/wizard-cli-reference.md`; consider matching non-interactive flags if automated onboarding should express the same policy.
