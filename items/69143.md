## Summary

The PR's premise that cross-platform local builds require editing `scripts/docker/setup.sh` does not hold on current `main`. The setup script delegates both local image builds directly to Docker's `docker build`, and Docker already provides `DOCKER_DEFAULT_PLATFORM` for commands that accept `--platform`; the PR adds a convenience alias, not a missing core capability.

## Best Possible Solution

Close this PR as obsolete for core. Keep the current Docker setup script and, if maintainers want better UX for this workflow, add a docs note showing `DOCKER_DEFAULT_PLATFORM=linux/amd64 ./scripts/docker/setup.sh` instead of adding a second OpenClaw-specific alias for Docker's existing platform selector.
