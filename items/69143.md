## Summary

The PR's stated blocker is not reproducible on current main: `scripts/docker/setup.sh` delegates both local image builds to Docker's normal `docker build` command without scrubbing Docker CLI environment variables, and Docker already supports `DOCKER_DEFAULT_PLATFORM` for commands that accept `--platform`. The PR's `OPENCLAW_DOCKER_PLATFORM` alias is a convenience/discoverability layer, not required for cross-platform local builds. The separate same-scope issue #69314 was already closed for the same reason, and the only PR review feedback is a minor style suggestion.

## Best Possible Solution

Close this PR as obsolete for core. Keep the current Docker setup path and, if maintainers want to improve UX, add a docs note showing `DOCKER_DEFAULT_PLATFORM=linux/amd64 ./scripts/docker/setup.sh` rather than adding a second OpenClaw-specific alias for Docker's existing platform selector.
