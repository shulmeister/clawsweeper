## Summary

Keep #43043 open. Current main still has `openclaw channels logs` reading the resolved current-day log path directly, while the shared rolling-log tail helper is only used by `openclaw logs`/Gateway log tail paths. The PR’s exact March patch is also stale and only partial because it would not handle a stale current-day log file that already exists.

## Best Possible Solution

Keep the item open and rework it against current main: fix active log selection once in the shared logging/tail path or logger rotation layer, then have `openclaw channels logs` consume that shared behavior while preserving manifest-registry channel filtering and the no-broad-channel-plugin-load constraint. Add deterministic tests for cross-date selection, stale current-day files, and channel filtering; if #47700 becomes the canonical root fix, fold this command coverage there before closing #43043 as superseded.
