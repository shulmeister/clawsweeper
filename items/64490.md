## Summary

Keep open. Current main still emits zsh option descriptions inside double-quoted `_arguments` specs without escaping `$` or backticks, and existing CLI descriptions contain both. The provided PR remains a plausible focused fix, not an obsolete or duplicate item.

## Best Possible Solution

Keep this PR open as the implementation candidate. A maintainer should review the small escaping helper, keep the regression assertions for `$` and backticks, and run the targeted completion test plus a zsh `openclaw models <TAB>` repro before merging or requesting final changes.
