## Summary

Keep open. Current main still has the vulnerable zsh option-description escaping path, and the reported `openclaw models <TAB>` failure is supported by current source: the `models` command has backtick-bearing option descriptions that are emitted into double-quoted zsh `_arguments` strings without escaping backticks or `$`. The related completion PRs in the provided context cover different completion issues and do not supersede this fix.

## Best Possible Solution

Keep PR #64490 open for maintainer review. The appropriate fix is still in the CLI completion generator: escape zsh double-quoted option descriptions for `$` and backticks, keep single quotes literal in that context, and land focused regression tests in `src/cli/completion-cli.test.ts` for `models`-style descriptions.

## Evidence
