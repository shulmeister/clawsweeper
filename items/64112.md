## Summary

Keep this PR open. Current main still has the broad missing-command fallback that can turn an arbitrary unknown primary command into a plugin allowlist error, and I did not find an equivalent shipped fix in the current CLI code or tests.

## Best Possible Solution

Keep this PR open and either review/land it after confirming the alias, denylist, and detection-error review comments are handled, or apply an equivalent fix on main: only emit plugin allowlist/disablement diagnostics after resolving the missing primary to a real bundled plugin CLI root or manifest command alias, and add a regression for `openclaw run` with restrictive `plugins.allow` falling through to the normal unknown-command path.
