## Summary

Keep open. Current main has partial fixes for the original report, including configured fallback handling, forward-compatible xAI/Google model resolution, and fallback status visibility. The remaining security-relevant gap is still present: fallback attempts build tool policy from the active fallback provider/model, not the originally requested model, so a deny list on the requested xAI/Grok model would not follow a successful Anthropic fallback attempt.

## Best Possible Solution

Keep the issue open and implement the missing requested-model tool-policy inheritance. The likely fix is to carry both selected/requested provider+model and active fallback provider+model through runWithModelFallback, runAgentAttempt, runEmbeddedPiAgent, createOpenClawCodingTools, bundled tool filtering, and explicit allowlist source collection, then apply the requested-model policy or a fail-closed intersection of requested and active policies. Add a regression test where an xai/grok requested model denies exec/write/edit, the run falls back to anthropic/claude-sonnet, and those tools remain unavailable.
