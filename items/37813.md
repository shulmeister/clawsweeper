## Summary

Keep open. Current main has partial fixes for #37813, including xAI forward-compatible model IDs, configured fallback-chain wiring for session overrides, and fallback status visibility. The requested-model tool-permission part still appears unimplemented: fallback attempts resolve tool policy from the active fallback provider/model, not from the originally requested model whose `tools.byProvider` deny list the issue expects to preserve.

## Best Possible Solution

Keep #37813 open until OpenClaw either carries the selected/requested model reference through fallback attempts for tool-policy resolution, or maintainers explicitly decide that fallback attempts should use only the active fallback model’s tool policy. The implementation should add focused regression coverage showing that `tools.byProvider[requestedProvider/requestedModel].deny` remains enforced when the run falls back to another provider/model, plus coverage for status visibility and configured fallback-chain ordering.
