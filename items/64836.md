## Summary

Current main already addresses the MiniMax auth/config failure mode reported after the 2026.4.10 upgrade: MiniMax OAuth setup writes the required `minimax-portal` provider API/auth-header config, catalog discovery recognizes stored portal auth profiles, and the fix is recorded as shipped in 2026.4.11.

## Best Possible Solution

Keep the shipped MiniMax provider/auth behavior and tests. If a user can still reproduce repeated auth loss on current v2026.4.24 or main after a one-time re-auth, the follow-up should capture `openclaw models status --json`, redacted `auth-profiles.json` profile IDs/modes, cooldown state, and the relevant `models.providers.minimax*` config so maintainers can investigate a new current-regression path.
