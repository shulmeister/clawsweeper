## Summary

Keep PR #71354 open. Current `main` still does not implement the PR's core behavior: status bootstrap returns `null` without calling `getTailnetHostname` whenever `gateway.tailscale.mode` is `off`. The PR is recent, focused, contributor-authored, and the author addressed the cold-start regression review by adding the `skipColdStartNetworkChecks` guard in follow-up commit `65e2d7ffe4ba231d30e266b5d0b53721a105a99c`. Current main does contain related label/docs work that clarifies `Tailscale exposure: off` is OpenClaw Serve/Funnel config rather than daemon state, but that does not supersede the PR's runtime hostname detection request.

## Best Possible Solution

Keep the PR open for maintainer/product review. The likely best path is to add focused tests around `createStatusScanCoreBootstrap` for off-mode runtime detection and cold-start skipping, then decide explicitly whether status should query and surface local Tailscale runtime data when OpenClaw exposure is off, possibly with safer binary resolution or redaction if maintainers accept the behavior.
