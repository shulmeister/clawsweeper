## Summary

Keep #11492 open. Current main still lacks agent-facing Slack `channel-info` and `channel-list` actions: the Slack message action discovery, generic dispatch, runtime handler, runtime exports, docs, and contract tests all omit them. The existing Slack `channelInfo` config flag is present but unused for the message-tool surface, while Discord already wires that gate to both requested actions. Related PR #11491 is closed unmerged, and #53199 remains only a partial overlapping proposal unless it also adds `channel-info`.

## Best Possible Solution

Keep #11492 open until the bundled Slack plugin exposes both `channel-info` and `channel-list` through the message tool. The likely implementation belongs in `extensions/slack`: add discovery behind the existing `channelInfo` gate, map the generic actions in `message-action-dispatch.ts`, add runtime helpers using Slack `conversations.info` and paginated `conversations.list`, then update Slack docs plus focused discovery, dispatch, runtime, and contract tests. PR #53199 can help with the `channel-list` portion, but it should not close this issue unless `channel-info` is implemented or a narrower canonical follow-up is opened.
