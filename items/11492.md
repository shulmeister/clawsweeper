## Summary

Keep #11492 open. Current main still does not expose Slack `channel-info` or `channel-list` as message-tool actions. Slack has internal channel resolution code and a `channelInfo` config flag, and Discord exposes the requested actions, but Slack discovery, dispatch, runtime exports, tests, and docs still omit them. The exact earlier PR #11491 was closed unmerged, and the related PR #53199 is still open rather than shipped on main.

## Best Possible Solution

Keep the issue open until Slack exposes the requested message-tool actions on main. The likely fix belongs in `extensions/slack`: add `channel-info` and `channel-list` to action discovery behind the existing `channelInfo` gate, dispatch them through `message-action-dispatch`, implement runtime helpers around Slack `conversations.info` and paginated `conversations.list` using existing token/account patterns, then update Slack docs and focused discovery/dispatch/runtime tests. If PR #53199 becomes canonical, it should explicitly cover both requested actions or document a maintainer-approved scope split.
