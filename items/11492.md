## Summary

Keep #11492 open. Current main still does not expose Slack `channel-info` or `channel-list` as agent-facing message actions. Slack has a `channelInfo` config/schema flag and internal Slack channel lookup paths, but discovery, dispatch, runtime exports, runtime handling, tests, and docs still omit the requested actions. Related PR #11491 is closed unmerged, and PR #53199 remains only a partial/open path in the provided context because it covers `channel-list` plus `search`, not the full `channel-info`/`channel-list` request.

## Best Possible Solution

Keep #11492 open until the bundled Slack plugin exposes both `channel-info` and `channel-list` through the message tool. The likely implementation belongs in `extensions/slack`: add discovery behind the existing `channelInfo` gate, add dispatcher branches, add runtime/action helpers around Slack `conversations.info` and paginated `conversations.list`, then update Slack docs plus focused contract/runtime tests. PR #53199 can help with the `channel-list` portion, but should not close this issue unless `channel-info` is also implemented or tracked separately.
