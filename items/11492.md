## Summary

Keep open. Current main still does not expose Slack `channel-info` or `channel-list` message actions. Slack has internal channel name/list resolution paths, and the generic channel action enum includes these action names, but the Slack plugin's discovery, dispatch, runtime action surface, tests, and docs only expose send/read/file/pin/member/emoji actions. The related exact PR #11491 was closed unmerged, and the later PR #53199 is still described as an open partial/superseding attempt rather than shipped main behavior.

## Best Possible Solution

Keep the issue open until Slack exposes both requested message actions on main. The likely implementation belongs in `extensions/slack`: add `channel-info` and `channel-list` to Slack action discovery behind the existing `channelInfo` gate, dispatch them to runtime actions, implement `conversations.info` and paginated `conversations.list` helpers with the existing read-token/account model, and update Slack docs plus focused action discovery/dispatch/runtime tests. If PR #53199 is revived as the canonical fix, it should either add direct `channel-info` as well or a maintainer should explicitly narrow/supersede this issue.
