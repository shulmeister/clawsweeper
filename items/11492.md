## Summary

Keep open. Current `main` still does not expose Slack `channel-info` or `channel-list` through the agent message tool. Slack has an `actions.channelInfo` config flag and internal channel lookup helpers, but the Slack message-tool discovery, dispatch, and runtime paths do not implement these actions, while Discord already does.

## Evidence

- **Slack message-tool discovery omits channel actions:** `listSlackMessageActions()` only adds `send`, reaction, message, pin, member, and emoji actions. It never adds `channel-info` or `channel-list`, so agents cannot discover them for Slack.
  - file: [extensions/slack/src/message-actions.ts:30](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/extensions/slack/src/message-actions.ts#L30)
  - command: `nl -ba extensions/slack/src/message-actions.ts | sed -n '1,220p'`
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Slack dispatcher has no channel-info/channel-list handling:** `handleSlackMessageAction()` maps Slack actions like `send`, `react`, `read`, `edit`, `delete`, `pin`, `member-info`, `emoji-list`, `download-file`, and `upload-file`. Any other action falls through to `Action ... is not supported`, so generic `channel-info`/`channel-list` requests are unsupported.
