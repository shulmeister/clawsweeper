## Summary

Keep #34004 open. Current main still only exposes Discord `set-presence` behind `channels.discord.actions.presence`; it does not implement the requested `self-profile`/`selfProfile` action, nickname/avatar mutation, strict self-only target checks, multi-account ambiguity handling for profile updates, or `bestEffort` partial-result behavior. The related implementation PR #34010 is closed unmerged, and the related broader issue #33270 is closed rather than an open canonical tracker for the remaining self-profile work.

## Best Possible Solution

Keep #34004 open. If maintainers want this in the bundled Discord plugin, implement a scoped `self-profile` message action behind `channels.discord.actions.selfProfile` with nickname/avatar/status/activity support, strict bot-self target validation, explicit `accountId` behavior for multi-account ambiguity, avatar MIME/base64/10MB validation, gateway preflight for presence fields, focused tests, and Discord docs. If maintainers prefer ClawHub, make that product call explicit and either link a plugin that can safely own the Discord bot account or document any missing plugin API needed to extend bundled Discord actions.
