## Summary

Keep open. Current main still has only the narrower Discord `set-presence` action behind `channels.discord.actions.presence`; it does not implement the requested Discord `self-profile`/`selfProfile` action, bot nickname/avatar mutation, self-only target guards, multi-account profile routing requirement, or `bestEffort` partial profile results. The related PR #34010 was closed unmerged in the provided GitHub context, so it does not supersede this issue.

## Best Possible Solution

Keep #34004 open. If maintainers want this in the bundled Discord plugin, implement a scoped `self-profile` message action behind `channels.discord.actions.selfProfile` with self-only target checks, explicit account routing for multi-account configs, strict avatar validation, gateway preflight for presence updates, tests, and docs. If maintainers prefer an external plugin, first document or extend the plugin/runtime API needed to operate safely on the relevant Discord bot account.
