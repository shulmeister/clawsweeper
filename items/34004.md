## Summary

Keep open. Current main only implements the narrower Discord `set-presence` path behind `channels.discord.actions.presence`; it does not implement the requested `self-profile`/`selfProfile` action, nickname or avatar mutation, self-only target guards, multi-account accountId requirement for profile mutation, or best-effort partial profile updates. Related PR #34010 is closed unmerged per the provided GitHub context, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open. If maintainers want this in core, implement it in the bundled Discord plugin as a scoped `self-profile` message action behind `channels.discord.actions.selfProfile`, with self-only target checks, explicit account routing for multi-account configs, strict avatar validation, gateway preflight for presence updates, tests, and docs. If maintainers prefer an external capability, first record the plugin/API direction explicitly rather than closing this as already handled.
