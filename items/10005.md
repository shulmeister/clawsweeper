## Summary

Keep open. Current main still renders chat `/status` and `session_status` as a status-card text response, and I did not find a config option or tool parameter equivalent to `session.statusCard` / `agents.defaults.statusCard`. The related closed issue #13450 is useful context for CLI automation because `openclaw status --json` exists, but that is a separate CLI path and does not suppress the chat/session status card requested here.
