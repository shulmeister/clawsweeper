## Summary

Keep open. Current main still does not provide a config option or tool parameter to hide the rendered chat/session status card. The related `openclaw status --json` work is a separate CLI surface and does not affect chat `/status` or the `session_status` agent tool requested here.

## Best Possible Solution

Keep #10005 open until OpenClaw adds an explicit status-card visibility control for chat `/status` and `session_status`, or a maintainer makes a product decision not to support it. A complete implementation should add the config schema/types/help/docs, thread the setting through `buildStatusReply`, `buildStatusText`, `buildStatusMessage`, and `session_status`, preserve current rendered-card behavior by default, and cover both chat and agent-tool paths in targeted tests.
