## Summary

Keep #10005 open. Current main still always renders chat `/status` and `session_status` as status-card text, and there is no `session.statusCard`, `agents.defaults.statusCard`, JSON-only, or equivalent suppression setting in the config types/schema/docs. The related CLI `openclaw status --json` path is implemented, but it is a separate diagnostics command and does not solve the chat/tool status-card clutter request.

## Best Possible Solution

Keep #10005 open until OpenClaw adds an explicit, documented status-card visibility control for chat `/status` and `session_status`, or a maintainer rejects the product direction. A complete fix should preserve current rendering by default, add schema/types/help/docs for the chosen key, thread the setting through `buildStatusReply`, `buildStatusText`, `buildStatusMessage`, and `session_status`, and add focused tests for both chat-command and agent-tool behavior. The existing CLI `openclaw status --json` path should remain documented as a separate diagnostics surface.
