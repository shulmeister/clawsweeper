## Summary

Keep #10005 open. Current main still renders chat `/status` and the `session_status` tool as status-card text with no `session.statusCard`, `agents.defaults.statusCard`, JSON-only, or equivalent suppression setting. The related closed #13450 CLI `openclaw status --json` work is real, but it covers a different CLI diagnostics surface and does not satisfy chat `/status` or agent-tool output.

## Best Possible Solution

Keep #10005 open until OpenClaw either adds an explicit status-card visibility control for chat `/status` and `session_status`, or a maintainer decides not to support the feature. A complete implementation should preserve current card rendering by default, add schema/types/help/docs for the chosen config key, thread the setting through `buildStatusReply`, `buildStatusText`, `buildStatusMessage`, and `session_status`, and add focused tests for both chat-command and agent-tool paths while keeping CLI `openclaw status --json` documented as a separate surface.
