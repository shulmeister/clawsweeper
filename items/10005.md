## Summary

Keep open. Current main still does not provide a config or tool parameter to hide the chat/session status card. The related closed CLI JSON request is implemented on the `openclaw status --json` command path, but that is separate from chat `/status` and the `session_status` agent tool requested here.

## Best Possible Solution

Keep #10005 open until OpenClaw adds an explicit status-card visibility control for the chat `/status` and `session_status` paths, or a maintainer decides the product direction. A complete fix should add the config schema/type/docs entry, thread it through `buildStatusReply`/`buildStatusText`/`session_status`, preserve existing default card behavior, and add targeted tests for both chat `/status` and the agent tool.
