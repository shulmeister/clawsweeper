## Summary

Keep #33478 open. Current main preserves some thread/topic routing for the A2A announce path, but the central requested structured `sessions_send` skip/action contract is still missing and the A2A lifecycle still depends on exact `REPLY_SKIP`/`ANNOUNCE_SKIP` text. A maintainer comment also explicitly asked to keep this issue open as acceptance criteria for the broader A2A handoff work in #63789.

## Best Possible Solution

Keep #33478 open as acceptance criteria for A2A handoff work. The best implementation path is to add a structured `sessions_send` outcome/action contract, or the equivalent first-party handoff protocol from #63789, that handles skip/no-reply/announce outcomes without exact model text, preserves channel/thread context, keeps legacy tokens as compatibility behavior, and adds regression coverage for both chatter around `REPLY_SKIP` and threaded announce routing.
