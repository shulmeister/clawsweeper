## Summary

Keep #33478 open. Current main at 82262387651ab270b565f71533ce4937c9c00 still lacks the requested structured `sessions_send` action/override contract, and the A2A handoff loop still depends on exact `REPLY_SKIP` / `ANNOUNCE_SKIP` assistant text. Thread announce routing is partially fixed and tested, but the remaining structured outcome API is explicitly called out by a maintainer as acceptance criteria for the broader A2A handoff work.

## Best Possible Solution

Keep #33478 open as acceptance criteria for first-party A2A handoff work, likely coordinated with #63789, #27445, and #44309. The best implementation is a structured `sessions_send` outcome/action contract covering skip, no-reply, announce suppression, explicit routing overrides, legacy token compatibility, and regression tests for token chatter plus threaded announce routing.
