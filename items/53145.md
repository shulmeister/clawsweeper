## Summary

Keep open. Current `main` still routes generic `sessions_send` follow-ups through the announce flow with the literal `"Agent-to-agent announce step."` prompt, and that flow only stops on `REPLY_SKIP` / `ANNOUNCE_SKIP`, not `NO_REPLY`. `main` does contain a newer narrow fix for parent-owned background ACP children, but that guard is explicitly limited to that subtype and does not broadly implement the issue's requested suppression/filtering.

## Evidence

- **Generic announce step still injects the placeholder text and forwards any non-ANNOUNCE_SKIP reply:** `runSessionsSendA2AFlow` calls `runAgentStep` with `message: "Agent-to-agent announce step."` and then sends any trimmed reply unless `isAnnounceSkip(...)` is true. There is no `NO_REPLY` check here.

  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Ping-pong step still re-injects follow-up text unless it is REPLY_SKIP:** The reply loop breaks only on empty reply or `isReplySkip(replyText)`. A literal `NO_REPLY` is not filtered before becoming the next inter-session message.
