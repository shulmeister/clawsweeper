## Summary

Keep open. Current main still leaves the reported `sessions_send` A2A announce path active: fire-and-forget sends start an announce flow, ping-pong forwards peer replies unless they are exactly `REPLY_SKIP`, the announce placeholder is still submitted as an inter-session agent input, and `NO_REPLY` is not recognized by the A2A skip-token helper. Related reports cover adjacent raw-output surfaces, but they do not supersede this specific peer `sessions_send` issue.

## Best Possible Solution

Keep this issue open. The likely fix belongs in the `sessions_send` A2A plumbing: treat exact silent/control replies such as `NO_REPLY` before ping-pong or announce delivery, avoid persisting or displaying the announce placeholder as a normal inter-session user turn, prevent acknowledgement follow-up text from being reinjected as peer input, and add focused regression coverage for `sessions_send({ timeoutSeconds: 0 })` between peer sessions.
