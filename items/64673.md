## Summary

Keep open. Current main still has the iOS Safari auto-zoom trigger this PR targets: chat and form text controls remain below 16px, the latest release tag has the same CSS, and the linked issue #64651 remains unresolved in the provided context. The PR is not ready as-is because the provided bot review caught a misplaced stylelint suppression and the patch appears narrower than the current UI surface, but it is still a plausible focused fix rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open, but ask for a refresh against current main. The best fix is a focused Control UI CSS change that prevents iOS/iPadOS Safari focus zoom for all interactive text-entry controls, ideally scoped to touch-primary devices if desktop layout should stay dense, with coverage for chat compose, agent chat, shared `.field` controls, config inputs/selects/textareas, and existing mobile overrides. Add a small CSS regression test or document real-device/iOS Simulator verification before landing.
