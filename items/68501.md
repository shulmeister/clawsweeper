## Summary

Keep open. Current main still lacks the requested configurable inline dreaming target: explicit inline mode writes to `memory/YYYY-MM-DD.md`, the memory-core config schema/resolver expose only `storage.mode` and `separateReports`, and docs do not describe an Obsidian `daily-memory/` target or `daily-notes.json` interaction. The default storage mode has shifted to separate reports, which reduces default pollution, but it does not solve the issue for users who opt into inline storage. The related #48294 context tracks pre-compaction memoryFlush path configuration, not this dreaming inline writer.

## Best Possible Solution

Keep the issue open and implement this in the bundled `memory-core` plugin rather than closing it as shipped. A good fix would add a validated `plugins.entries["memory-core"].config.dreaming.storage.inlineRelativePath` or narrow `inlineTarget` option, resolve it safely under the workspace, preserve the managed light/REM markers, update the manifest schema, SDK config type/resolver, docs at `/concepts/dreaming` and `/reference/memory-config`, and add tests proving `memory/daily-memory/{date}.md` works without breaking the current `memory/{date}.md` default.
