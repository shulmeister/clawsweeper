## Summary

Keep open. Current main has a static `params.extra_body`/`extraBody` passthrough for OpenAI-compatible completions, plus OpenTelemetry `traceparent` headers for diagnostic tracing, but it does not implement the issue’s requested dynamic provider request metadata: templated `trace`, automatic `user = sessionKey`, or a provider hook context that exposes session/task identity for request-body attribution. Related open PRs are implementation candidates, not shipped behavior.

## Best Possible Solution

Keep this issue open and use it to review a coherent provider-attribution design: either land a safe session-aware provider hook such as #54725 and document how plugins inject `user`/`trace`, or add a narrowly scoped built-in OpenAI-compatible/OpenRouter request-body attribution option with tests covering `sessionKey`, agent id, and static `extra_body` precedence.
