## Summary

PR #42199 is obsolete. Current main keeps strict SSRF DNS pinning by default, but the proxy-required memory embeddings problem has been fixed by merged PR #71506 using the established TRUSTED_ENV_PROXY path and a safer EnvHttpProxyAgent gate.

## Best Possible Solution

Close #42199 as superseded by the merged #71506/current TRUSTED_ENV_PROXY design. Keep strict fetchWithSsrFGuard DNS pinning as the default, and handle any remaining proxy-required call sites with scoped trusted-mode opt-ins rather than a broad lower-level ProxyAgent branch.
