## Summary

Keep open. Current main has a partial, later history-filtering fix for heartbeat/runtime-context leaks, but it has not implemented this PR’s source-tagging approach: heartbeat/cron/exec turns still do not set `InputProvenance.kind = "internal_system"`, `chat.history` does not drop system-origin user messages, and the Control UI still loads `chat.history` with a client filter that only hides assistant acks, silent replies, synthetic repair output, and empty user messages.

## Best Possible Solution

Keep the PR open, but rebase/reshape it onto current main: set `InputProvenance.kind = "internal_system"` at the heartbeat/cron/exec origin, add one shared gateway-side system-origin user-message filter used by both `chat.history` and session-history/SSE paths, keep the Control UI provenance check as defense in depth, and add regressions for `chat.history`, `/sessions/:id/history`, and inline updates so source tags replace fragile prompt-text matching.
