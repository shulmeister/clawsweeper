## Summary

Keep open. Current `main` still matches the maintainer’s issue comment: the one-shot `at` guard exists, but only the startup catch-up path passes it. The normal scheduler tick paths still collect due jobs without `skipAtIfAlreadyRan: true`, so the reported re-trigger condition remains plausible on current code.

## Evidence

- **Normal timer path still omits the one-shot completed-job guard:** `onTimer` loads jobs and calls `collectRunnableJobs(state, dueCheckNow)` with no options. That means the normal scheduler tick does not activate `isRunnableJob(... skipAtIfAlreadyRan ...)` for completed `kind: "at"` jobs.
  - file: [src/cron/service/timer.ts:707](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/cron/service/timer.ts#L707)
  - command: `nl -ba src/cron/service/timer.ts | sed -n '687,712p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **The guard exists, but only when explicitly requested:** `isRunnableJob` has a `skipAtIfAlreadyRan` branch that returns `false` for completed one-shot jobs unless they are in a transient-error retry state. The bug report is specifically about normal timer paths not enabling this branch.
  - file: [src/cron/service/timer.ts:857](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/cron/service/timer.ts#L857)

  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Startup catch-up uses the guard; normal tick paths do not:** `runMissedJobs` passes `skipAtIfAlreadyRan: true` into `collectRunnableJobs`, but the live paths do not. This matches the code-level diagnosis left on the issue by `martingarramon`.
  - file: [src/cron/service/timer.ts:989](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/cron/service/timer.ts#L989)

- **Secondary due-job entrypoint also omits the guard:** `runDueJobs` also calls `collectRunnableJobs(state, now)` without `skipAtIfAlreadyRan: true`, so the missing protection is not limited to one call site.
  - file: [src/cron/service/timer.ts:1122](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/cron/service/timer.ts#L1122)
