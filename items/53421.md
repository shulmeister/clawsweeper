## Close Comment

Thanks for the contribution. This was a Codex review against current `main` (`8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf`). I’m closing this as `clawhub` rather than a core-repo change.

The reason is scope, not usefulness: current project policy says optional capability should usually ship as plugins outside the core repo (`VISION.md:54-61`), and the plugin docs explicitly say new plugins do not need to be added to `openclaw/openclaw` and should be published to ClawHub or npm instead (`docs/plugins/building-plugins.md:16-19`, `docs/plugins/community.md:9-22`, `docs/plugins/community.md:133-149`).

This smart-router also fits the existing external plugin model cleanly because `main` already exposes the needed `before_model_resolve` hook for provider/model overrides (`src/plugins/hook-before-agent-start.types.ts:1-18`). In other words, this is a good plugin idea, but it belongs as a community plugin package rather than a new bundled plugin in core. If there is later a product-level decision that smart routing must ship bundled by default, it can be reconsidered with that explicit rationale.
