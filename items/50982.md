## Summary

Keep #50982 open. Current main still does not expose `--thread-id` on `openclaw cron add/create` or `openclaw cron edit`; the CLI only accepts `--channel`, `--to`, and account/best-effort delivery options and sends those through to `cron.add`/`cron.update`. Main does support Telegram topic delivery when users manually encode the target as `chatId:topic:topicId`, but that is the workaround the issue is asking to replace with a first-class flag. The related PR #60890 is an open, unmerged implementation track, so it should not make the original issue closeable yet.

## Best Possible Solution

Keep the issue open and land PR #60890 or an equivalent focused fix that adds `--thread-id` to `cron add/create` and `cron edit`, composes it into the existing Telegram `:topic:` target format, adds validation for invalid Telegram topic target combinations, and covers the behavior in `src/cli/cron-cli.test.ts`. The follow-up cascade noted in the April 21 comment should be handled either in the same fix if small or in a separate issue/PR for create-time target validation and safer failure-notification routing.
