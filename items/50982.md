## Summary

Keep this issue open. Current main still does not expose `--thread-id` on `openclaw cron add/create` or `openclaw cron edit`, while PR #60890 is open and unmerged as the implementation candidate. The existing manual `:topic:` target path works, but it is not the requested cron CLI UX.

## Best Possible Solution

Keep this issue open until PR #60890 is reviewed and either landed or closed with maintainer judgment. The best implementation path is a focused cron CLI change that adds `--thread-id` to `cron add/create` and `cron edit`, composes Telegram topic targets as `<chatId>:topic:<threadId>`, preserves existing manual topic targets, rejects invalid combinations early, and adds regression coverage in `src/cli/cron-cli.test.ts`.
