I found no source, docs, tests, release, or related-item evidence that scrollbar-thumb dragging has been fixed or superseded since the prior review.

## Best Possible Solution

Keep this issue open and investigate the macOS native WebChat scroll path with a real macOS app build and a long chat history. The likely fix belongs around `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatView.swift`: avoid fighting user-driven scrollbar interaction by gating bottom-pinning and animated `scrollPosition` updates while the user is actively scrolling, or move the macOS message list to an NSScrollView-backed implementation if SwiftUI `scrollPosition` remains unstable. Add focused regression coverage where feasible and include manual verification for dragging the scrollbar thumb.
