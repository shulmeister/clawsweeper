## Summary

Current main implements the requested behavior: `sessions.list` loads the combined session store, canonicalizes stored keys, and does not filter out `agent:main:main`. Existing regression coverage writes a store with `main` plus other sessions and asserts the RPC list includes the canonical `agent:main:main` row. The reporter also confirmed the issue disappeared after updating; #8609 is unrelated UI rendering work, and #46133 was useful discussion but is not needed as remaining work.
