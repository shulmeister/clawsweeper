## Summary

Keep open. Current main still exposes a `filename` argument on the message tool, but the send pipeline does not carry that value into `MessageSendParams`, normalized outbound payloads, channel outbound contexts, or Feishu `sendMediaFeishu`. The PR is not mergeable as submitted because it bundles unrelated local docs/scripts/skills/CI/CODEOWNERS changes and has unresolved review comments for Feishu remote-file and local-root regressions, but the core filename-preservation bug remains real.

## Best Possible Solution

Keep the PR open for maintainer/contributor follow-up, but require a clean narrow replacement patch: thread `filename` through `ReplyPayload`, normalized outbound payloads, `MessageSendParams`, `ChannelOutboundContext`, and Feishu `sendMedia`; optionally derive a basename for local `filePath` sends when no override is provided; preserve `loadWebMedia`/`mediaLocalRoots` and `sanitizeFileNameForUpload`; add regression tests for message-tool passthrough, core delivery context, Feishu explicit Unicode filenames, remote non-image URLs, and blocked local paths; drop unrelated BuildDocument, BuildTools, skills, workflow, and CODEOWNERS changes.
