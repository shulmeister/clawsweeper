## Summary

Current `main` and the shipped `v2026.4.23` release already carry Telegram inbound sender metadata through session history sanitization into the Control UI, and the session picker labels Telegram direct sessions as `Telegram · <id>`. The reported `openclaw-control-ui` label now only matches Control UI-authored turns inside a Telegram-routed session, which is a different generic UX gap rather than this Telegram-specific bug.

## Evidence
