## Summary

Current main still appears to have the reported QMD dirty-status gap. The QMD manager maintains a private dirty flag for watcher/search sync, but its public status always returns dirty false, and the CLI renders the Dirty line directly from that status field.

## Evidence

- **QMD status hardcodes clean state:** QmdMemoryManager.status() returns dirty: false for the QMD backend instead of exposing the manager's dirty state or checking index/file drift.

- **Watcher only updates private dirty flag:** The QMD watcher marks this.dirty true on add/change/unlink and schedules sync, but that private flag is not surfaced by status().
