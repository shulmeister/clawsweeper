## Summary

Keep PR #68417 open. Current main still resolves exec approvals through `~/.openclaw`, still validates that path against the home root, and still rejects symlinked path components before writing the approvals file. The PR’s reported server-deployment failure is therefore not implemented or obsolete, and the branch includes focused regression coverage plus review iteration.

## Best Possible Solution

Keep #68417 open as a still-relevant focused bug-fix PR, or explicitly fold its exact symlink regression coverage and trust-root safety into the broader #62938/#29736 state-dir fix. The maintainer path should settle whether exec approvals should resolve directly under `resolveStateDir()` or continue supporting `~/.openclaw` symlinks when they exactly target `OPENCLAW_STATE_DIR`, then update tests and docs accordingly.
