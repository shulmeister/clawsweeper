## Summary

Current `main` now keeps runtime system-event content out of transcript-facing user turns and strips/hides remaining internal-only history rows in WebChat. The latest tagged release does not appear to include the full fix yet, so this is implemented on `main` but not tied to a shipped release here.

## Evidence

- **Transcript-facing prompt is now separate from runtime/system-event prompt:** `buildReplyPromptBodies()` now accepts a separate `transcriptBody` and returns `transcriptCommandBody`, so `systemEventBlocks` are prepended to model/queue bodies but not to the transcript-facing body that should be persisted.
  - file: [src/auto-reply/reply/prompt-prelude.ts:8](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/auto-reply/reply/prompt-prelude.ts#L8)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Heartbeat/cron reply path passes visible transcript text separately:** `get-reply-run` drains formatted system events, then calls `buildReplyPromptBodies()` with `transcriptBodyBase` alongside `systemEventBlocks`, producing a distinct `transcriptCommandBody` for storage instead of persisting the runtime-enriched prompt.
  - file: [src/auto-reply/reply/get-reply-run.ts:519](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/auto-reply/reply/get-reply-run.ts#L519)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Persisted user turn is rewritten back to the visible transcript prompt:** The embedded runner rewrites the just-appended submitted user prompt from `submittedPrompt` to `transcriptPrompt`, replacing runtime-enriched/internal text in session history after execution starts.
  - file: [src/agents/pi-embedded-runner/run/transcript-prompt-rewrite.ts:56](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/pi-embedded-runner/run/transcript-prompt-rewrite.ts#L56)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Regression test covers removal of internal runtime context from stored user history:** The test creates a user message containing `<<<BEGIN_OPENCLAW_INTERNAL_CONTEXT>>>...<<<END_OPENCLAW_INTERNAL_CONTEXT>>>` and verifies the reopened session transcript keeps only the visible `transcriptPrompt` (`visible ask`).
  - file: [src/agents/pi-embedded-runner/run/transcript-prompt-rewrite.test.ts:55](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/pi-embedded-runner/run/transcript-prompt-rewrite.test.ts#L55)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Gateway/WebChat history still sanitizes and hides residual internal-only rows:** `chat.history` first runs `stripEnvelopeFromMessages()` to remove internal runtime context/inbound metadata from returned rows, and the Control UI loader drops heartbeat-only and empty internal-only user rows; the targeted UI test covers the hidden internal-only user-message case.
  - file: [src/gateway/server-methods/chat.ts:1908](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/gateway/server-methods/chat.ts#L1908)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Latest tagged release did not yet have the separate transcript body path:** In `v2026.4.23`, `buildReplyPromptBodies()` had no `transcriptBody`/`transcriptCommandBody`, so I could not tie this fix to a shipped release tag.
  - file: [src/auto-reply/reply/prompt-prelude.ts:8](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/auto-reply/reply/prompt-prelude.ts#L8)
