## Summary

Keep open. Current main still only trims cron payload `model` and `fallbacks` values; it does not strip internal whitespace from provider/model refs. The PR is still addressing a real cron/model-normalization surface, but the submitted approach should not be closed as implemented or merged unchanged because model and fallback values can also be aliases, and alias resolution preserves internal spaces.

## Best Possible Solution

Rework the fix so cron handles accidental whitespace in syntactic provider/model refs without rewriting alias strings. A conservative path is to normalize only values that are clearly provider/model refs or to resolve aliases before repairing provider/model ids, then add regression tests for payload-level and top-level `model`/`fallbacks` plus negative tests proving spaced aliases still resolve.
