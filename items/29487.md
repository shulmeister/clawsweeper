## Summary

Keep open. Current main implements Feishu reply context for `message.parent_id`, but the issue discussion identifies the remaining Feishu UI quote path as `quote.message_id`, and current code still does not model or fetch that ID.

## Best Possible Solution

Keep #29487 open and implement the remaining Feishu plugin behavior after verifying the event contract: extend the event/context types to carry the quote message id, parse `quote.message_id`, prefer it for explicit quote context while preserving `parent_id` fallback, and add regression tests for both Feishu UI quote replies and existing parent_id replies under the current group context visibility rules.
