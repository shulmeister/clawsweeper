## Summary

Keep open. Current main still fetches Feishu quoted/replied context only from `message.parent_id`; it does not model or parse the Feishu UI quote identifier `quote.message_id` called out in the issue discussion, so a quote-only event would still omit the quoted content.

## Best Possible Solution

Keep #29487 open. Implement this in the bundled Feishu plugin by verifying the Feishu UI quote payload shape, extending event/context types to carry the quoted message ID, parsing `quote.message_id` or the verified equivalent, preferring that quote ID when fetching quoted content while keeping `parent_id` fallback, and adding regression tests for quote-only UI replies plus existing parent-id replies under group context visibility rules.
