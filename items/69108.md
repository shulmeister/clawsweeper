## Summary

Keep open. Current main already fixes the `--agent` plus `--session-id` collapse, but the PR's `--new-session` CLI feature is still absent from code and docs, and #54864 remains the open canonical request for that missing behavior.

## Best Possible Solution

Keep this PR or the canonical #54864 path open. The best next maintainer direction is to preserve the shipped session-id precedence fix, then rebase/narrow this PR to the missing `--new-session` feature: add the Commander option, enforce mutual exclusion with `--session-id`, generate one UUID before gateway/local branching, pass that effective session id through both paths, return it for JSON/stdout callers, and add focused CLI/session tests plus docs/changelog attribution.
