## Summary

Keep #39555 open. Current main still has the provider-backed hybrid memory-search behavior the PR targets: it sends the full cleaned multi-word query into a single keyword lookup, while the FTS query builder AND-joins tokens. The PR is stale against the current `extensions/memory-core` layout and needs the bot review's fan-out bound addressed, but the underlying bug remains actionable. Related #57711 overlaps query expansion, but it is not merged or maintainer-designated as the canonical replacement and does not clearly cover this PR's OR-style cross-chunk recall case.

## Best Possible Solution

Keep the PR open as an actionable memory-core bug-fix candidate unless maintainers explicitly replace it with a canonical PR. Port the OR-style per-keyword keyword recall to `extensions/memory-core/src/memory/manager.ts`, preserve source/model filtering, bound or rank extracted terms, and add an active regression test for provider-backed hybrid search where relevant multi-word query terms appear in separate chunks.
