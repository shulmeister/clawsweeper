## Summary

Keep open. Current main still has the hybrid memory-search gap described by the PR: provider-backed hybrid search performs one full-query keyword lookup, while the FTS query builder AND-joins every token. The PR is stale against the moved memory implementation and has an unresolved fan-out review concern, but the underlying bug is not implemented away or clearly superseded.

## Best Possible Solution

Keep the PR open as an actionable bug-fix candidate, but require a port onto extensions/memory-core, a bounded per-keyword hybrid lookup strategy, and focused regression coverage for multi-word cross-chunk queries before maintainer merge review.
