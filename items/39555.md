## Summary

Keep open. Current main still routes provider-backed hybrid memory search through a single full-query FTS lookup, while the FTS query builder still AND-joins tokens. The PR is stale because memory code now lives under `extensions/memory-core`, and its proposed per-keyword fan-out needs bounding, but the underlying multi-word hybrid recall bug is not implemented away or clearly superseded.

## Best Possible Solution

Keep #39555 open as an actionable bug-fix candidate. The best path is to port the fix to `extensions/memory-core`, bound or rank extracted query terms before FTS fan-out, and add focused active regression coverage for multi-word hybrid queries whose terms span chunks. If maintainers prefer #57711’s narrower keyword-extraction strategy, they should explicitly mark that PR as the canonical replacement before closing this one.
