## Summary

Keep open. Current main still does not pass Feishu card action `option`, `options`, or `form_value` through to the synthetic message path, and PR #42908 is still useful but incomplete against the current implementation because the production webhook parser rebuilds `action` with only `value` and `tag`. The linked issue #42754 and broader follow-up #43202 remain open, so this is not implemented or safely obsolete.

## Best Possible Solution

Keep #42908 open but require a current-main update: pass `option`, `options`, `form_value` and likely related `input_value`/`name` fields through `parseFeishuCardActionEventPayload`, update `buildFeishuCardActionTextFallback` or the legacy dispatch path to include those fields without breaking `value.text`/`value.command`, and add an ingress-path regression test that starts from a raw `card.action.trigger` payload instead of only constructing `FeishuCardActionEvent` directly. Coordinate with #42754/#43202/#43953 so one Feishu card-action pass-through fix lands cleanly.
