## Summary

Keep open. The PR is a recent, focused draft for a real core auto-reply interruption gap; current main still lacks the proposed run-generation registry and stale-output fence, and the related umbrella issue #70319 remains open rather than superseded.

## Best Possible Solution

Keep #70363 open for maintainer review under #70319. If maintainers want this direction, finish or split the core auto-reply run-invalidation work so the generation/fence contract covers block replies, partial replies, typing/progress, followups, tool dispatch, new-message takeover, and subprocess cancellation without changing plugin boundaries.
