## Summary

Keep this PR open. Current main partially mitigates the original exec-only MCP leak by skipping bundled MCP startup when `toolsAllow` cannot name bundled MCP tools, but runtime `toolsAllow` still is not authoritative over the final bundled MCP/LSP tool list.

## Best Possible Solution

Keep this PR open or supersede it with a maintainer patch that applies runtime `toolsAllow` after bundled MCP/LSP materialization, preserves intended `bundle-mcp` plugin/group semantics, supports exact MCP and LSP tool names, and adds focused regression coverage for `toolsAllow: ["exec"]`, one exact MCP tool, `toolsAllow: ["bundle-mcp"]`, and configured bundled LSP.
