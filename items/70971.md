## Summary

Keep PR #70971 open. Current main has a partial mitigation that skips bundle MCP creation when `toolsAllow` has only non-bundled names such as `exec`, but it still does not make runtime `toolsAllow` authoritative over the final combined tool list. Bundled LSP tools are still materialized whenever tools are enabled, and bundled MCP is still materialized when the allowlist contains any namespaced bundled tool, after which `effectiveTools` appends all policy-surviving bundled tools without reapplying `toolsAllow`.

## Best Possible Solution

Do not auto-close this PR. The best maintainer path is to keep it open, rebase/update the explanation for current main's partial MCP gate, and add a regression test proving runtime `toolsAllow` is applied to the final combined tool list for bundled MCP/LSP tools before `effectiveTools` is serialized.
