## Summary

Keep PR #41419 open. Current main still constructs Discord upload Blobs without a MIME type and still sends WebP images through the shared optimizer, so the PR is neither implemented nor obsolete. The PR discussion also contains useful follow-up and unresolved security/review concerns, so it should be revised or intentionally superseded rather than auto-closed.

## Best Possible Solution

Keep this PR open for maintainer review or supersede it intentionally with a focused fix for #41256: preserve Discord upload Blob MIME types only from verified media types, preserve static and animated WebP without destroying frames after safe validation and metadata handling, update both standard and component Discord upload tests, and reconcile the shared media behavior change with WhatsApp expectations.
