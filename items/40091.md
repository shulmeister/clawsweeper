## Summary

PR #40091 is a stale one-file fix for an older reset path. Current main has a shared reset-selection policy used by both chat `/new`/`/reset` and gateway `sessions.reset`, and the remaining product question about clearing explicit `/model` selections is already tracked by open issue #55063.

## Best Possible Solution

Close this stale PR as superseded and keep the remaining behavior discussion on #55063. If maintainers decide bare `/new` and `/reset` should clear explicit user model pins, implement that centrally in `resolveResetPreservedSelection()`, keep `/new <model>` behavior via `applyResetModelOverride()`, and update both auto-reply and gateway reset tests/docs around the chosen contract.
