## Summary

PR #63039 adds Bocha as a new bundled commercial web-search provider. Current main does not ship Bocha, but the requested capability fits the existing external plugin path: OpenClaw already exposes web-search provider registration, docs say web-search plugins can be published through ClawHub/npm without adding them to this repo, and the PR discussion already identified branch policy problems with hardcoded core migration code and hand-authored localized docs. Close as ClawHub/third-party plugin scope rather than merge as a bundled core plugin.

## Best Possible Solution

Close this bundled-provider PR as ClawHub/third-party plugin scope. The best path is for Bocha to live as a provider-owned external OpenClaw plugin using `api.registerWebSearchProvider(...)`, published through ClawHub or npm with clear setup and security metadata; a new core issue should be opened only if that external implementation exposes a concrete missing SDK/API extension point.
