## Summary

PR #63039 adds Bocha as a new bundled commercial web-search provider. Current main does not ship Bocha, but this is optional third-party provider work that fits the existing plugin/web-search provider API and the current OpenClaw scope guidance points it to ClawHub or an external plugin package rather than core. The PR discussion also identified core-policy problems in the branch, reinforcing that the right cleanup outcome is an external, provider-owned package instead of merging a bundled core plugin.

## Best Possible Solution

Close this bundled-provider PR as ClawHub/third-party extension scope. The best path is for the author or community to maintain Bocha as an external OpenClaw code plugin using `api.registerWebSearchProvider(...)`, publish it through ClawHub or npm with clear setup/security metadata, and open a narrow SDK/API issue only if that external implementation hits a concrete missing extension point.
