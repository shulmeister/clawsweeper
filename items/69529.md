## Summary

Keep #69529 open. Current main still contains the two central bugs this PR targets: outbound message sends still copy input.defaultAccountId into the target channel context before plugin dispatch, and the gateway exec-approvals schema still rejects allowlist entries with persisted source metadata. The PR is useful, but it bundles unrelated skill-description cleanup and should be split or narrowed before maintainer merge.

## Best Possible Solution

Keep the PR open, but ask for a narrower maintainer-ready shape: land the #69525 outbound-runner fix with a focused cross-provider regression test and stronger assertion; land the #69482 gateway schema fix with protocol/CLI coverage; move the partial #69475 skill-description cleanup to its own PR or leave it to the canonical issue.
