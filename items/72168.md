## Summary

Keep #72168 open. The report is fresh, concrete, independently confirmed in the provided discussion, and current main does not contain the linked #72238 fix commit or any relevant changes since the earlier review point. The browser plugin still has the same failure shape: `pw-ai` is soft-loaded, Playwright-backed actions still emit the reported 501 when that load fails, and the Playwright implementation still uses bare ESM imports of `playwright-core` while the root package does not declare `playwright-core`.

## Best Possible Solution

Keep #72168 open and use #72238 or an equivalent maintainer patch as the fix path. The fix should make `playwright-core` resolvable for packaged browser Playwright chunks after `openclaw doctor --fix` and gateway restart, and should include regression coverage for packaged global/Homebrew-style installs where bundled plugin deps are staged outside the OpenClaw package tree.
