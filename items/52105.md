## Summary

Keep open. Current `main` still has the same Control UI swallow seam for local slash commands: `handleSendChat` clears the draft before dispatch, and `dispatchSlashCommand` still returns silently when `host.client` is missing and still awaits `executeSlashCommand(...)` without a protective `try/catch`. The referenced fixing commit for this issue (`a5f88924a7759d911a17141b1009ad20e9d38811`) added exactly those guards, but that patch is not present on current `main`.

## Evidence

- **Draft is cleared before local slash-command dispatch:** `handleSendChat` clears `host.chatMessage` / attachments, then awaits `dispatchSlashCommand(...)` for local commands. If dispatch throws or returns early without injecting a result, the command disappears from the input exactly as reported.
