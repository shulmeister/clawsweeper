## Summary

Keep PR #39365 open. Current main has the low-level `openclaw system event` CLI reference, `/hooks/wake` docs, and runtime/test coverage for wake delivery, but it still does not include the PR's dedicated script/webhook/CI/monitoring recipes page or automation nav entry. The PR remains useful as the canonical docs work, especially with #48869 closed in favor of it, but it needs revision before merge because the review context identifies an unsafe webhook shell example and misleading troubleshooting claims.

## Best Possible Solution

Keep #39365 open as the canonical docs PR, or replace it with a focused maintainer-authored docs change. The best version is a safe recipes page linked from `docs/cli/system.md`, `docs/gateway/heartbeat.md`, and webhook docs; it should recommend `openclaw system event --mode now` for external scripts, explain that writing inbox files alone does not wake an agent, include sanitized webhook examples, and correct the `openclaw agent --message` and channel-trigger troubleshooting language.
