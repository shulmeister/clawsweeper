## Summary

Keep PR #39365 open. Current main has the `openclaw system event` CLI/runtime surface and adjacent wake docs, but it still lacks the PR's dedicated external-script recipes page and automation nav entry. The PR remains useful as the canonical docs work, but it needs revision before merge because review comments identify unsafe and misleading recipe text.

## Best Possible Solution

Keep #39365 open as the canonical docs PR, or replace it with an equivalent maintainer-authored docs change. The best version should add a safe `docs/automation/system-events.md` recipes page, add the nav entry, cross-link from `docs/cli/system.md`, heartbeat, and webhook docs, explain that writing files alone does not wake the agent, recommend `openclaw system event --mode now` for external scripts, and fix the unsafe/misleading recipe text called out in review.
