## Summary

Keep open. Current `main` still rejects wildcard plugin API ranges like `*`, so the reported install failure remains plausible and the linked fix PR is still open rather than implemented on `main`.

## Evidence

- **Current parser still rejects `*`:** `src/infra/clawhub.ts` routes plugin API range checks through `satisfiesSemverRange()`, but `satisfiesComparator()` only handles empty tokens, caret ranges, and explicit comparison operators. There is no wildcard branch for `*`, `x`, or `X`, so `*` falls through to semver parsing and returns false.
  - file: [src/infra/clawhub.ts:302](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/infra/clawhub.ts#L302)
  - command: `nl -ba src/infra/clawhub.ts | sed -n '276,360p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Installer still throws the reported error path:** `validateClawHubPluginPackage()` rejects a package whenever `compatibility.pluginApiRange` does not satisfy the runtime version and emits the same error shape reported in the issue: `requires plugin API ... but this OpenClaw runtime exposes ...`.
  - file: [src/plugins/clawhub.ts:688](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/plugins/clawhub.ts#L688)
  - command: `nl -ba src/plugins/clawhub.ts | sed -n '640,725p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Tests still do not cover wildcard compatibility:** The current `satisfiesPluginApiRange` test covers `^`, bounded ranges, exact gateway calver comparisons, and invalid input, but no `*`/`x` wildcard case exists on `main`.
  - file: [src/infra/clawhub.test.ts:80](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/infra/clawhub.test.ts#L80)
