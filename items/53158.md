## Summary

Current `main` already uses provider-agnostic subagent session keys and stores the actual model provider separately, so the reported `:openai:` key format is no longer how subagent sessions are created. The same behavior is present in the shipped `v2026.4.22` release.

## Close Comment

Closing this as implemented on current OpenClaw. Codex review of `main` found that subagent session keys are now created with a provider-agnostic `:subagent:` prefix in `src/agents/subagent-spawn.ts:686`, not `:openai:`. The actual provider/model are stored separately on the session entry via `modelProvider` and `model` in `src/agents/subagent-spawn.ts:204`. There is also direct regression coverage in `src/agents/subagent-spawn.test.ts:121` and `src/agents/subagent-spawn.model-session.test.ts:87` asserting the `agent:<id>:subagent:<uuid>` key shape and separate provider persistence. This behavior is already present in the shipped `v2026.4.22` release (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
