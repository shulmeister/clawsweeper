## Summary

Keep open. Current main still skips symlinked plugin directories during automatic plugin-root scanning, and the PR is a focused fix with a regression test for a real reported setup. It is not implemented on main, not superseded, and not a ClawHub-only request because it affects core plugin discovery for documented load roots.

## Best Possible Solution

Keep PR #63206 open and rebase or adapt its approach on current main: classify symlinked scanned entries with a stat-following helper, continue to run the existing package manifest/bundle/index discovery path, and keep the regression test for a symlinked plugin directory under the global extensions root. Do not broaden this into arbitrary workspace recursion.

## Evidence
