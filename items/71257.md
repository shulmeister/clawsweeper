## Summary

Keep #71257 open. Current main still persists session-memory transcript text without sanitizing assistant artifacts, and those memory files are loaded back into startup context on later `/new` or `/reset` sessions. The PR is a relevant core fix attempt, but it overlaps #69961/#69943 and needs review follow-up before merge.

## Best Possible Solution

Keep #71257 open, but consolidate it with #69961/#69943 into one scoped session-memory sanitizer fix. The final patch should sanitize assistant transcript content before memory persistence, preferably by reusing or aligning with existing assistant-visible text, directive-tag, media, and silent-token helpers; cover chat-template tokens, raw and truncated tool-call XML, `NO_REPLY`, `[[audio_as_voice]]`, `[[reply_to_*]]`, and `MEDIA:` lines; and add transcript/handler regression tests.
