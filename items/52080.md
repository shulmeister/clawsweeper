## Summary

Current main already rebuilds workspace bootstrap context on post-compaction retries, so the AGENTS.md reinjection behavior targeted by PR #52080 is already covered and the PR is redundant for its stated bug.

## Evidence

- **Compaction clears continuation-skip state:** `hasCompletedBootstrapTurn()` returns false when a `type: "compaction"` record appears after the last full-bootstrap marker, which forces the next attempt to treat the session as needing fresh bootstrap context again.
  - file: [src/agents/bootstrap-files.ts:57](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/agents/bootstrap-files.ts#L57)
- **Behavior is covered by an explicit test:** There is a direct test asserting `hasCompletedBootstrapTurn()` returns false after compaction, proving post-compaction retries are expected to re-bootstrap rather than continue with stripped workspace context.
  - file: [src/agents/bootstrap-files.test.ts:296](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/agents/bootstrap-files.test.ts#L296)
- **Each retry recomputes bootstrap context:** Every embedded attempt calls `resolveAttemptBootstrapContext()`, and that helper only strips bootstrap files when `hasCompletedBootstrapTurn(sessionFile)` is true. After compaction, that condition flips to false, so AGENTS.md and other bootstrap files are resolved again for the retry.
  - file: [src/agents/pi-embedded-runner/run/attempt.context-engine-helpers.ts:37](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/agents/pi-embedded-runner/run/attempt.context-engine-helpers.ts#L37)
- **System prompt is rebuilt per attempt from resolved context:** `runEmbeddedAttempt()` rebuilds the system prompt on each attempt using freshly resolved `contextFiles`, so overflow retries do not reuse a single frozen pre-compaction system prompt.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:652](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/agents/pi-embedded-runner/run/attempt.ts#L652)
- **Overflow paths retry via a fresh attempt:** Both the SDK-managed compaction path and the explicit overflow compaction path `continue` the outer loop after compaction, which causes a new attempt and therefore a fresh bootstrap-context evaluation before the next model call.
  - file: [src/agents/pi-embedded-runner/run.ts:1132](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/agents/pi-embedded-runner/run.ts#L1132)
- **Docs describe the same contract:** The config docs now explicitly state that with `contextInjection: "continuation-skip"`, post-compaction retries still rebuild context.
  - file: [docs/gateway/config-agents.md:74](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/docs/gateway/config-agents.md#L74)

## Risks / Open Questions

- The PR also proposes broader compaction-summary wording for preserving standing instructions stated in conversation text, and current main does not include that prompt tweak. I did not find strong evidence that this extra change is needed to fix the PR's titled AGENTS.md reinjection bug.

## Close Comment

Closing from a Codex review: current `main` already rebuilds workspace bootstrap after compaction, so the stale-AGENTS retry condition described here is already covered. `hasCompletedBootstrapTurn()` treats a `type: "compaction"` entry as clearing continuation-skip (`src/agents/bootstrap-files.ts:57`), that behavior is explicitly tested (`src/agents/bootstrap-files.test.ts:296`), and each retry goes back through `resolveAttemptBootstrapContext()` before rebuilding the system prompt (`src/agents/pi-embedded-runner/run/attempt.context-engine-helpers.ts:37`, `src/agents/pi-embedded-runner/run/attempt.ts:652`, `src/agents/pi-embedded-runner/run.ts:1132` and `src/agents/pi-embedded-runner/run.ts:1231`). The docs also now say post-compaction retries rebuild context (`docs/gateway/config-agents.md:74`). Based on that, `main` already implements the AGENTS.md reinjection behavior this PR is aiming for, so I’m closing it out.
