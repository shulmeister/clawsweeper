## Summary

Keep #38601 open. Current main still turns an explicit `pty: true` request into `false` whenever the exec target is sandboxed, and I found no user-facing warning, error, docs note, or regression test covering the requested sandbox PTY warning. The related fix PR #39349 is still open and unmerged in the provided GitHub context, while #46392 was closed unmerged, so the remaining work has not shipped on main.

## Best Possible Solution

Keep the issue open and land a small source-backed fix. The likely path is to emit a warning, or intentionally throw, when `params.pty === true && sandbox` before `usePty` is suppressed; add a focused regression test for sandboxed exec with `pty:true`; and update `docs/tools/exec.md` / public docs to state the sandbox limitation. PR #39349 already tracks the proposed warning and can be used or superseded by an equivalent maintained fix.
