## Summary

Keep #38601 open. Current main at 04e08cea62983b2c766d15f61f15b3ffc90f9baa still disables PTY whenever the exec target is sandboxed, and I found no warning/error path or user-facing docs update for an explicit `pty:true` request. The related fix PR #39349 is still open, while #46392 was closed unmerged, so the issue is not implemented or superseded on main.

## Best Possible Solution

Keep the issue open and land a narrow exec fix, likely via the existing #39349 direction: when `params.pty === true` and sandbox execution is active, add a user-visible warning or explicit error, cover it with a regression test, and update `docs/tools/exec.md` / tool schema wording to state the sandbox limitation.
