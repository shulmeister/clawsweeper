## Summary

Current `main` still drops `pty:true` whenever a sandbox runtime is active, and the user-facing exec docs still do not clearly warn about that limitation. The report remains valid and should stay open.

## Evidence

- **PTY is still forcibly disabled in sandbox:** `usePty` is derived as `params.pty === true && !sandbox`, so an explicit `pty:true` request is turned off whenever the session has a sandbox runtime.
