## Summary

Keep #38601 open. Current main still suppresses an explicit `pty: true` request whenever exec resolves to a sandbox runtime, and I found no current warning/error path, user-facing docs caveat, schema caveat, or regression test for sandboxed `pty:true`. The provided GitHub context shows the focused fix PR #39349 is still open and #46392 was closed unmerged, so the issue is not implemented or superseded.

## Best Possible Solution

Keep the issue open and land a narrow fix in the exec tool routing layer: when `params.pty === true && sandbox`, emit a clear warning or deliberately reject the request before `usePty` is suppressed. Add a focused regression test for sandboxed exec with `pty:true`, and update the exec docs plus tool schema/help text to state the sandbox PTY limitation.
