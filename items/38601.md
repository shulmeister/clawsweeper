## Summary

Keep #38601 open. Current main still suppresses explicit `pty:true` whenever exec resolves to a sandbox runtime, and I found no warning/error path, user-facing sandbox caveat, or regression test covering that feedback. The related fix PR #39349 is still open and unmerged, while #46392 was closed unmerged, so the issue is not implemented or superseded.

## Best Possible Solution

Keep #38601 open and land a narrow exec-tool fix. Maintainers should choose the intended contract: either honor `pty:true` in sandbox backends that support it, or intentionally suppress it with a clear warning/rejection before `usePty` is set false. Add a focused regression test for sandboxed `pty:true`, and align `docs/tools/exec.md`, `src/agents/bash-tools.schemas.ts`, and `src/agents/bash-tools.descriptions.ts` so users and agents see the same behavior.
