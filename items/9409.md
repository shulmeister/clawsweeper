## Summary

Keep #9409 open. Current main still surfaces generic context-overflow text to users; it logs some diagnostics internally, but the requested user-visible session/channel label, context size, model limit, and overage are not implemented. The linked PR #15749 was closed unmerged, and the adjacent model-switch items do not supersede this request.

## Best Possible Solution

Keep the issue open. The best implementation is a shared context-overflow diagnostic formatter that accepts a safe session/channel label, provider/model identity, effective context budget, observed prompt/session token count, and computed overage, then wires that formatter through runEmbeddedPiAgent, formatAssistantErrorText, sanitize/fallback delivery paths, and focused regression tests.
