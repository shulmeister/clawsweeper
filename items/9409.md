## Summary

Keep #9409 open. Current main still surfaces generic user-visible context-overflow messages; diagnostic details are logged or stored in metadata but not shown to the user. The direct implementation PR #15749 is closed unmerged, and the related model-switch/context-hardening items do not supersede this user-facing diagnostic request.

## Best Possible Solution

Keep #9409 open. The best maintainer path is a shared context-overflow diagnostic formatter that accepts a safe session/channel label, provider/model identity, effective context budget or model limit, observed or estimated token count, computed overage, and concise recovery actions, then wires it through `run.ts`, precheck/attempt handling, `formatAssistantErrorText`, `sanitizeUserFacingText`, and auto-reply fallbacks with focused regression tests.
