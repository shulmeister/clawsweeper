## Summary

Keep #9409 open. Current main still returns generic user-visible context-overflow messages in the embedded runner, shared formatter, sanitizer, precheck path, and auto-reply fallbacks. The code now logs and stores useful provider/model/context-token diagnostics, and docs describe context snapshots, but those details are not surfaced in the overflow reply requested by the issue. The related diagnostic PR #15749 was closed unmerged, and the broad #52788 PR is also closed unmerged and does not supersede this request.

## Best Possible Solution

Keep the issue open. The best path is a shared context-overflow diagnostic formatter that accepts a safe session/channel label, provider/model identity, effective context budget, observed prompt/session token count, computed overage, and concise recovery actions, then wires it through `run.ts`, precheck/attempt handling, `formatAssistantErrorText`, sanitizer/normalization, and auto-reply fallback paths with focused regression tests.
