## Summary

Keep #9409 open. Current main still surfaces generic context-overflow guidance and does not show the requested session/channel, context size, model limit, or overage diagnostics. The related PR #15749 was closed unmerged, so it does not supersede this issue.

## Best Possible Solution

Keep the issue open. The best implementation is a shared context-overflow formatter that accepts safe session/channel labeling, provider/model identity, effective context budget, observed prompt/session token count, and overage, then wires it through `runEmbeddedPiAgent`, `formatAssistantErrorText`, sanitizer/fallback delivery paths, and regression tests.
