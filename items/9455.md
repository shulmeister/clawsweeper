## Summary

Keep #9455 open. Current main has the Cron Jobs filter bar and partial filtering from merged #9510, but the issue's explicit remaining behaviors are still missing: Jobs search does not match job IDs, and the Jobs last-run-status filter cannot select absent/unknown status jobs.

## Best Possible Solution

Keep #9455 open, but narrow it to the remaining delta on top of merged #9510: include job.id in the cron.list search haystack or equivalent client-side search path, add an unknown Jobs last-status filter option/type, normalize absent lastStatus/lastRunStatus into that unknown bucket, and add focused controller/service/view tests for job-id search and unknown-status filtering.
