## Summary

Keep #9455 open. Current main has most of the Cron Jobs filter bar, but it still misses two explicit requested behaviors: text search does not match the job id, and the Jobs last-run-status filter has no explicit unknown option for jobs with no last status. The related #16799/#16831 context remains separate CLI/status readability work, not a canonical duplicate for this dashboard filter request.

## Best Possible Solution

Keep #9455 open and narrow it: include job.id in cron.list search matching, add an explicit unknown last-run-status filter that matches jobs with no lastStatus/lastRunStatus, and update the UI/controller/service tests for those behaviors. Keep #16799/#16831 separate because they target CLI/status readability, not this dashboard filter bar.
