## Summary

Keep #9455 open. Current main has the Cron Jobs filter bar, but it still misses two explicit requested behaviors: search does not match the cron job id, and the Jobs last-run-status filter has no explicit `unknown` option for jobs without status. The latest maintainer-side issue comment also already kept this open for those same gaps.

## Best Possible Solution

Keep #9455 open and narrow the remaining work: include `job.id` in the `cron.list` query haystack, add an explicit `unknown` value to the Cron Jobs last-run-status filter that matches jobs without `lastRunStatus`/`lastStatus`, expose that option in the UI, and add controller/service/view tests for both behaviors. Keep the related CLI/status readability items separate.
