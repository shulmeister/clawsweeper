## Summary

Keep open. Current main has a Cron Jobs filter bar with search, enabled, schedule-kind, last-run-status, sorting, reset, and tests, but it does not fully satisfy the issue’s requested behavior: job search does not match job id, and the last-run status filter has no explicit unknown option. The related #16799/#16831 context is about CLI/status readability for cron-heavy setups, not a canonical duplicate or superseder for this dashboard filter request.
