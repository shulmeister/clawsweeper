## Summary

Keep #9455 open. Current main includes the Cron Jobs filter bar from merged #9510, but it still misses two explicit requested paths: search by jobId and filtering jobs with absent/unknown last-run status.

## Best Possible Solution

Keep #9455 open and narrow it to the remaining gaps on top of #9510: include job.id in the cron.list search haystack, add an unknown Jobs last-status filter option, normalize missing status into that unknown bucket, consider lastRunStatus fallback where relevant, and add focused controller/service/view tests for job-id search plus unknown-status filtering.
