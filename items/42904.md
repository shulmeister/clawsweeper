## Summary

Keep PR #42904 open. Current main and v2026.4.24 already implement the cross-date fallback for the shared `openclaw logs` / `logs.tail` path, but the PR’s `openclaw channels logs` surface still reads only `getResolvedLoggerSettings().file` directly and has no active rolling-log fallback. The PR also bundles unrelated cron warning work that is still not on main and is separately tracked by #42196/#42245, so this should be split or reworked rather than auto-closed as implemented.

## Best Possible Solution

Keep the item open but rework it before merge: split the cron warning into the existing #42196/#42245 track, then update `openclaw channels logs` to reuse or export the shared rolling-log resolver from `src/logging/log-tail.ts`. Add focused channel-log tests for missing today's rolling file, newest-mtime selection, and custom non-rolling `logging.file` behavior so the earlier review concerns stay covered.
