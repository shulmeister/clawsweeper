## Summary

Keep open. Current `main` does not implement this fix: `openclaw channels logs` still reads only the currently resolved dated log path, so the midnight-crossing case described in the PR remains plausible on `main`.

## Evidence

- **channels logs still reads one resolved file only:** `channelsLogsCommand` reads `getResolvedLoggerSettings().file` directly and passes it to `readTailLines`; if that file is missing, `readTailLines` returns an empty array and the command prints `No matching log lines.` There is no fallback to the active previous-day file.
  - file: [src/commands/channels/logs.ts:88](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/commands/channels/logs.ts#L88)
  - command: `nl -ba src/commands/channels/logs.ts | sed -n '77,106p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **default logger path is date-based for today:** The default resolved log file is `openclaw-YYYY-MM-DD.log` for the current local date, so after midnight a fresh CLI invocation targets today's file even if a long-running gateway is still appending to yesterday's file.
  - file: [src/logging/logger.ts:337](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/logging/logger.ts#L337)
  - command: `nl -ba src/logging/logger.ts | sed -n '330,340p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **similar fallback already exists elsewhere:** `readConfiguredLogTail` resolves a missing rolling log path by scanning the directory and picking the newest matching dated log file. That fallback exists for `openclaw logs`, but `openclaw channels logs` does not use it.
  - file: [src/logging/log-tail.ts:26](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/logging/log-tail.ts#L26)
  - command: `nl -ba src/logging/log-tail.ts | sed -n '22,53p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **CLI path still routes to the unfixed command:** `openclaw channels logs` still dispatches to `channelsLogsCommand`, so the missing fallback in that command remains user-visible on current `main`.
  - file: [src/cli/channels-cli.ts:158](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/cli/channels-cli.ts#L158)
