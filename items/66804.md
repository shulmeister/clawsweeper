## Summary

Keep open. Current main has adjacent mitigations for Active Memory timeout behavior, including a hard deadline and `thinking: "off"` by default, but it still does not make MiniMax-M2.7 Active Memory complete successfully or fall back to another memory model after the selected model times out. The issue has multiple recent confirmations, including a 2026.4.22 MiniMax-M2.7 timeout at about 15 seconds, and the broader related Active Memory timeout reports remain open rather than superseding this MiniMax-specific report conclusively.

## Best Possible Solution

Keep this open unless a maintainer intentionally folds it into a broader canonical Active Memory timeout issue such as #65309 or #70021. A real fix should either make MiniMax-M2.7 viable for the Active Memory subagent through safer defaults/tuning, or add a tested fallback-on-timeout path for Active Memory, such as retrying `config.modelFallback` or falling back to a local memory-search summary path without blocking the user turn beyond the configured budget.
