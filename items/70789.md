## Summary

Keep PR #70789 open. Current main still has the reverse newest-first tool-result pairing logic in the Control UI, so the PR is not obsolete, implemented elsewhere, or superseded. The fix targets the implicated code path for the linked #70746 frontend rendering bug, although it still deserves maintainer review and a stronger regression test for the true call-call-result-result ordering case.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land the pairing fix, preferably with a focused `extractToolCards` regression test that uses two same-name tool calls before their two results and also preserves exact-id precedence for mixed id/name cases.
