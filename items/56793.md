## Summary

Current `main` does not implement the requested one-shot import/clone flow from an existing-session browser profile into a managed OpenClaw profile. The shipped browser surface still treats managed `openclaw` and `existing-session`/`user` as separate modes: docs describe them as separate, the CLI exposes only list/create/delete/reset profile operations, the HTTP routes expose only `/profiles`, `/profiles/create`, `/profiles/:name`, and `/reset-profile`, and the profile service only creates attach configs or managed CDP profiles. This remains an unimplemented feature request, so it should stay open.
