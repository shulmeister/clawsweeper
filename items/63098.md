## Summary

Keep open. This is a recent, concrete Discord voice regression report with a second user confirmation, and current main still matches the reported failure mode: `/vc join` waits for voice readiness, uses a fixed 15s initial ready timeout, destroys the connection on timeout, and has no documented or schema-backed `channels.discord.voice.connectTimeoutMs` override. Related #63125 is broader and still open; this issue remains the narrower Discord voice tracking item.

## Best Possible Solution

Keep #63098 open as the focused Discord voice join reliability bug. A maintainer fix should likely stay in the bundled Discord plugin: add a validated `channels.discord.voice.connectTimeoutMs` or otherwise increase the initial ready deadline, consider a bounded retry/rejoin path for abort-like readiness failures, update Discord config docs/metadata, and add regression tests around high-latency `entersState(...Ready...)` behavior and `/vc join` interaction handling.
