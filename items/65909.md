## Summary

Keep #65909 open. Current main still contains the concrete failure mode described in the issue discussion: `applyExtraParamsToAgent` has `agentDir`/`workspaceDir`, and provider hook types support them, but the `wrapProviderStreamFn` call does not forward them. Since Codex native search activation checks persisted `openai-codex` auth through `agentDir`, valid auth can still be missed as `codex_auth_missing`. The linked PR #67843 is a relevant focused fix, but it is not present in this checkout and was described as partial rather than a full close for the broader live native-search behavior.

## Best Possible Solution

Keep #65909 open as the user-visible tracker. The next maintainer path is to land or recreate the focused #67843 fix by forwarding `agentDir` and `workspaceDir` into `wrapProviderStreamFn` context, add regression coverage that persisted Codex auth can activate native search through the provider wrapper path, then separately verify or document whether `mode: "live"` emits observable native search/tool events end-to-end.
