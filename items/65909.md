## Summary

Keep open. Current main has the Codex-native web search feature, but the specific failure mode described in the issue and linked PR still appears present: `applyExtraParamsToAgent` has `agentDir`/`workspaceDir` but does not pass them into the provider wrapper context, while the OpenAI/Codex wrapper relies on `ctx.agentDir` to detect persisted `openai-codex` auth. The related PR #67843 is an open partial fix, not a merged/superseding resolution.

## Evidence

- **Wrapper context still omits directories:** `applyExtraParamsToAgent` builds `wrapperContext` with `agentDir` and `workspaceDir`, but the later `providerRuntimeDeps.wrapProviderStreamFn({ context })` payload only includes config/provider/model/extra params/thinking/model/streamFn. This matches the concrete failure path described in #67843.
  - file: [src/agents/pi-embedded-runner/extra-params.ts:624](https://github.com/openclaw/openclaw/blob/0bcb4c95c185909b12d463b04d49b16ead3459d5/src/agents/pi-embedded-runner/extra-params.ts#L624)

  - sha: [0bcb4c95c185](https://github.com/openclaw/openclaw/commit/0bcb4c95c185909b12d463b04d49b16ead3459d5)
- **Codex wrapper consumes ctx.agentDir:** The shared OpenAI Responses stream family creates the Codex native web search wrapper with `agentDir: ctx.agentDir`; if the provider wrapper context omitted `agentDir`, the wrapper receives `undefined`.
  - file: [src/plugin-sdk/provider-stream.ts:119](https://github.com/openclaw/openclaw/blob/0bcb4c95c185909b12d463b04d49b16ead3459d5/src/plugin-sdk/provider-stream.ts#L119)
