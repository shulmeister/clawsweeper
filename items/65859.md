## Summary

Keep #65859 open. Current main has already implemented assistant-side inline audio playback for `MEDIA:`/structured audio attachments, but the WebChat upload path remains image-only and gateway chat attachment parsing still drops non-image uploads. The central request includes audio upload for ASR/transcription workflows, so the issue is only partially satisfied.

## Best Possible Solution

Keep the issue open but narrow the remaining work: preserve the existing assistant audio player, then add WebChat outbound audio support end to end by expanding the attachment MIME contract, rendering selected audio files with an audio preview/filename UI, serializing audio metadata rather than hard-coded `type: "image"`, teaching the gateway chat attachment parser/media pipeline to accept audio and route it to media understanding/ASR, and covering picker/drop/send plus assistant playback in focused UI and gateway tests.
