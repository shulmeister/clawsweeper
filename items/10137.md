## Summary

Keep #10137 open. Current main only partially satisfies the report: generic `api: "openai-responses"` requests already send array `input` and `stream: true`, but there is still no stable compatibility option to add fallback top-level `instructions` for strict Responses-compatible proxies or omit `max_output_tokens`. The related #10140 is the closed duplicate, leaving #10137 as the canonical tracker.

## Best Possible Solution

Keep #10137 open as the canonical tracker. Add opt-in Responses proxy compatibility controls in the transport/payload policy and stable provider/model config, especially fallback/default top-level `instructions` and optional omission of `max_output_tokens`, with focused regression tests and docs. Treat #10140 as the closed duplicate copy.
