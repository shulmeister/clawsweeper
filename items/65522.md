## Summary

Keep open. Current main still leaves the `Refresh chat data` button enabled during active send/run/stream states, and open PR #66395 explicitly targets this issue with closing syntax. Per the maintainer cleanup rule, the issue should remain open until that PR is reviewed and merged or closed.

## Best Possible Solution

Keep this issue open and review PR #66395 as the implementation candidate. The likely product fix belongs in `ui/src/ui/app-render.helpers.ts` by disabling refresh while `chatSending`, `chatRunId`, or `chatStream` indicate an active turn; maintainers should either land a narrowed version of that PR, request tests or scope cleanup, or close the PR and keep the issue open until an equivalent main-branch fix exists.
