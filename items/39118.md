## Summary

Keep open. Current main still lacks the running-gateway ExecStart protection and active-service filtering this PR proposes, and the PR discussion identifies real regressions in the proposed patch that need revision rather than cleanup closure.

## Best Possible Solution

Keep this PR open, but require a revised implementation: companion-service filtering should avoid dependency-only false positives without hiding custom gateway units; any active-service filter must be scope-aware for user and system units; and the running-gateway ExecStart guard must run after all ExecStart-level issues, including entrypoint mismatch, have been computed. Add focused tests for those paths before merge.
