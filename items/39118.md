## Summary

Keep #39118 open. Current main still does not implement the PR's core doctor safety behavior: dependency-only companion systemd units can still be reported as gateway-like, extra services are warned without active-state filtering, and service repair can still rewrite ExecStart without checking whether the gateway is currently running. The PR is not obsolete, but its review comments show it needs revision before merge.

## Best Possible Solution

Keep this PR open and require a revised implementation. The best path is to filter dependency-only companion references without suppressing real custom gateway units, make active-service filtering scope-aware for user versus system services, and apply the running-gateway ExecStart guard after all ExecStart-level audit issues, including entrypoint mismatch, have been computed. Add focused tests for companion dependencies, custom-named gateways, system-scope deep scans, inactive extras, and active-running custom launchers before merge.
