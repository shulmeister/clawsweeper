## Summary

Keep PR #39118 open. Current main still lacks the PR's core doctor safety behavior for dependency-only companion units, active-state filtering of extra services, and running-gateway protection before ExecStart rewrites. The PR is not obsolete, but the existing review comments show it needs revision before merge.

## Best Possible Solution

Keep the PR open and request a revised implementation. The best path is to filter dependency-only companion references without suppressing real custom gateway units, make active-service filtering scope-aware for user versus system services, apply the running-gateway ExecStart guard after all ExecStart-level audit issues have been computed, and add focused regression tests for companion dependencies, custom-named gateways, system-scope deep scans, inactive extras, and active-running custom launchers.
