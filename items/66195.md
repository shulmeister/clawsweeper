## Summary

#66195 is a plugin-specific reproduction of the same root `openclaw/plugin-sdk` `stringEnum` compatibility break already tracked by the broader SDK compatibility issue #55080 and the linked active fix PR #66214. Current main still does not restore `stringEnum` on the root SDK surface, while the helper remains available from focused SDK subpaths, so this should close as duplicate/superseded rather than as implemented.

## Best Possible Solution

Close #66195 as a duplicate/superseded plugin-specific symptom, and keep the remaining maintainer decision centralized on #55080/#66214: either restore a tested temporary root/compat export for `stringEnum`/`optionalStringEnum`, or explicitly document the migration-only policy for affected external plugins.
