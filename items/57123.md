## Summary

Keep open. Current `main` still exposes only fixed inbound debounce settings and a single resettable timer per key; the adaptive/incremental debounce requested in #57123 is not implemented.

## Evidence

- **Config schema only exposes fixed inbound debounce fields:** `InboundDebounceSchema` accepts only `debounceMs` and `byChannel`. There are no config fields for incremental growth, step size, or max cap.
  - file: [src/config/zod-schema.core.ts:688](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/config/zod-schema.core.ts#L688)

- **Runtime resolves one static debounce value:** `resolveInboundDebounceMs` picks a single number from override, per-channel override, base config, or `0`. It does not track message count or compute an adaptive window.
  - file: [src/auto-reply/inbound-debounce.ts:21](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/auto-reply/inbound-debounce.ts#L21)
