## Summary

Keep open. Current main does not honor `===DONE_OK===` or any equivalent cron success sentinel, and the PR describes a concrete cron false-positive error-classification case that is not duplicated or superseded by the provided related items. This needs maintainer product/code review rather than conservative cleanup closure.

## Best Possible Solution

Keep PR #72135 open for maintainer review. If accepted, implement the sentinel in `resolveCronPayloadOutcome`, preserve `runLevelError` precedence, add focused tests in the current cron payload outcome suite, and document or prompt-standardize the supported token if it becomes part of the cron contract.
