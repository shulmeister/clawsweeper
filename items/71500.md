## Summary

Keep open. Current main still resolves image models with a direct registry lookup and throws `Unknown model` on a miss, while the existing async model resolver has a provider dynamic-model preparation path and the LM Studio plugin implements that hook. The PR is recent, contributor-authored, has no protected labels in the provided context, and appears to address a plausible still-open bug rather than cleanup that can be closed.
