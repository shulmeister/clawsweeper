## Summary

Keep open. Current `main` still resolves logging config through the same `require("../config/config.js")` seam in the logging subsystem, still falls back to the default rolling `/tmp/openclaw/...` path when no config is read, and still lacks a packaged-dist regression test for this path. I did not reproduce a packaged install in this read-only review, but the source strongly indicates the issue is not fixed on `main`.
