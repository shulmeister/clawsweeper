## Summary

Current main still lacks first-class `frequency_penalty` and `presence_penalty` support. The only shipped path is the narrower `params.extra_body` workaround for `openai-completions` proxy request bodies, which does not satisfy the requested top-level model params behavior across the OpenAI/OpenAI-compatible transport paths.

## Best Possible Solution

Keep this issue open until OpenClaw either implements first-class sampling penalty params or a maintainer explicitly decides that `params.extra_body` is the intended product answer. A good implementation would collect snake_case and possibly camelCase sampling params from resolved model/agent params, inject them only into compatible payload builders or provider-owned wrappers, preserve existing `onPayload`/`extra_body` behavior, and add focused tests plus docs for the supported fields and transports.

## Risks / Open Questions

- `params.extra_body` already solves a subset of OpenAI-compatible completions proxy cases, so any implementation should define merge order and collision behavior with that existing escape hatch.
- Some providers reject unsupported sampling fields; a fix should be scoped or capability-gated rather than blindly injecting penalties into every provider transport.
