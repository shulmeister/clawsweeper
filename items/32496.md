## Summary

Keep #32496 open. Current main still lacks first-class frequency_penalty, presence_penalty, or repetition_penalty passthrough from model params. The shipped workaround is params.extra_body/extraBody for openai-completions proxy request bodies only, which does not satisfy the requested top-level model-param behavior across OpenAI/OpenAI-compatible transports. Related PRs #32540, #32582, and #33620 were closed unmerged; #71350 is adjacent per-agent params/thinking work, not a canonical replacement for this specific sampling-penalty request.

## Best Possible Solution

Keep this issue open as the canonical sampling-penalty passthrough request unless a maintainer explicitly folds it into another provider-params effort. The best implementation is core/provider-transport work: collect supported snake_case and camelCase sampling params from resolved model/agent params, inject them only into compatible OpenAI/OpenAI-compatible payload paths, preserve existing onPayload and extra_body behavior, and add focused tests plus docs for supported fields and transports.
