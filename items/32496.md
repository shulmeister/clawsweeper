## Summary

Keep #32496 open. Current main still does not provide first-class `frequency_penalty`, `presence_penalty`, or `repetition_penalty` passthrough from resolved agent/model params. The config surface can carry arbitrary params and the newer `extra_body` escape hatch helps a subset of `api: "openai-completions"` proxy payloads, but the core extra-param wrapper and OpenAI Responses, Completions, and WebSocket builders do not forward the requested sampling-penalty fields. Related PRs #32540, #32582, and #33620 were closed unmerged, and #71350 is adjacent per-agent params/thinking work rather than a canonical replacement for this request.

## Best Possible Solution

Keep #32496 open as the canonical sampling-penalty passthrough request. The best implementation is targeted transport work: collect supported numeric snake_case and camelCase sampling params from resolved global, per-model, per-agent, and override params; inject them only into compatible OpenAI/OpenAI-compatible request paths, including Responses, Completions, and WebSocket where supported; preserve existing `onPayload`, `extra_body`, provider-wrapper, and proxy-store behavior; and add focused tests plus docs for supported fields and transport scope.
