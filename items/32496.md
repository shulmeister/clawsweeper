## Summary

Keep #32496 open. Current main still stores arbitrary model params but does not first-class forward frequency_penalty, presence_penalty, repetition_penalty, top_p/top_k, or camelCase aliases from config into OpenAI/OpenAI-compatible stream payloads. The existing extra_body path is only a documented OpenAI-compatible Completions proxy workaround, and the related PRs discussed in the issue were closed unmerged; #71350 is adjacent per-agent config work, not a replacement for this transport passthrough request.

## Best Possible Solution

Keep #32496 open as the canonical sampling-penalty passthrough request. The best fix is targeted core transport work: resolve numeric snake_case and camelCase sampling params from merged global, per-model, per-agent, and override params; inject them only into compatible OpenAI/OpenAI-compatible Responses, Chat Completions, Azure, and WebSocket payload paths; preserve extra_body/onPayload/provider hook behavior; then add focused transport tests and docs for the supported fields and scope.
