## Summary

Current main still lacks the self-approval guard that this PR proposes. The gateway records the requesting connection, but `exec.approval.resolve` only validates allowed decisions before calling the shared resolver, and the manager itself does not enforce requester/resolver separation. The PR is not obsolete or superseded; it should remain open for maintainer review and validation.

## Best Possible Solution

Keep this PR open. The best path is to review and land a focused fix that adds the requester/resolver connection check inside `exec.approval.resolve` validation, preserves self-deny and different-connection approval behavior, updates the existing bypass test to use a separate resolver connection, adds focused regression tests, and runs the changed gateway checks before merge.
