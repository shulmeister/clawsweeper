## Summary

Keep #14151 open. Current main still lacks any configurable force-prepend `<think>` option, and the existing path only prompts for, parses, streams, or strips reasoning/final tags that are already present. The related implementation PR #14323 is closed unmerged, and the existing issue discussion already recorded a maintainer-side Codex review keeping this open.

## Best Possible Solution

Keep #14151 open. A complete fix should add a typed global and/or per-model configuration option, validate and document it, thread it from agent/model config through the embedded runner into `SubscribeEmbeddedPiSessionParams`, inject exactly once before streaming and final-message parsing, guard existing tags and native thinking events, and add focused tests for streaming, non-streaming, existing tags, native thinking, and `enforceFinalTag`.
