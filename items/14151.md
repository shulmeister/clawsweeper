## Summary

Keep #14151 open. Current main at 2f6615d2ee9d0d9db0f4ad1e13f71d1c0df8166d still does not implement a user-configurable force-prepend `<think>` option. The existing implementation strips already-present `<think>/<thinking>` blocks, handles native thinking stream events, and can enforce `<final>` output, but it does not synthesize an opening `<think>` tag for streaming or non-streaming assistant output. The related PR #14323 directly attempted this feature but is closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep the issue open until a maintainer either approves the product direction or implements the feature in the core embedded runner/subscriber path. A complete fix should add a typed global/per-model config option, validate and document it, thread the resolved value through runner params into `subscribeEmbeddedPiSession`, inject before stream parsing for both streaming and final-message paths, guard against double tags, and add focused regression coverage for streaming, non-streaming, and final-tag enforcement interactions.
