## Summary

Keep open. Current `main` does not implement a user-configurable option to force-prepend an opening `<think>` tag for local reasoning models; the runtime still relies on provider-level `native` vs `tagged` handling, and the referenced PR was closed without an evident merge ref.

## Evidence

- **No user-facing force-prepend option in codebase:** Repo-wide search for `forcePrependThinkTag`, `force_prepend_think_opening_tag`, and `prependThink` returned no matches in `src`, `extensions`, `docs`, or `ui`.
  - command: `rg -n "forcePrependThinkTag|force_prepend_think_opening_tag|prependThink" src extensions docs ui`
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **Reasoning mode contract is provider-owned, not a per-user prepend setting:** The current plugin contract exposes only `resolveReasoningOutputMode` for provider-owned `native` vs `tagged` behavior; there is no config surface here for forcing an opening `<think>` tag.
  - file: [src/plugins/types.ts:1251](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/plugins/types.ts#L1251)
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **Core defaults providers to native reasoning unless a provider opts into tagged mode:** `resolveReasoningOutputMode` falls back to `native`, and `isReasoningTagProvider` only turns on tag-based handling when a provider/plugin explicitly declares it.
  - file: [src/utils/provider-utils.ts:17](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/utils/provider-utils.ts#L17)
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **MiniMax explicitly stays on native reasoning mode:** The MiniMax provider hooks set `resolveReasoningOutputMode: () => "native"`, which is the opposite of adding frontend-enforced `<think>` tag injection for that provider family.
  - file: [extensions/minimax/provider-registration.ts:45](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/extensions/minimax/provider-registration.ts#L45)
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **Current local-model guidance avoids tag injection and recommends native separation:** The local-model docs say to use Responses API with LM Studio to keep reasoning separate from final text, indicating the current direction is native reasoning separation rather than forced `<think>` prefixing.
  - file: [docs/gateway/local-models.md:14](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/docs/gateway/local-models.md#L14)
