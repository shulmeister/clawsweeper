## Summary

Current main at 04e08cea62983b2c766d15f61f15b3ffc90f9baa still does not implement a configurable force-prepend `<think>` option. The embedded subscription API exposes reasoning visibility and final-tag enforcement, the runner passes only those existing controls, and the parser strips existing think/final tags rather than synthesizing missing opening tags. Related PR #14323 attempted this feature but is closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep #14151 open until a maintainer decides whether OpenClaw should own this normalization. A complete implementation should add a typed global and/or per-model config option, validate and document it, thread the resolved value through runner params into `subscribeEmbeddedPiSession`, inject before streaming and final-message parsing, guard against double-prepending, and add focused regression tests for streaming, non-streaming, native reasoning events, and `enforceFinalTag` interactions.
