## Summary

Current main already contains the two pieces this issue asked for: macOS LaunchAgent restart recovery that re-enables/re-bootstraps unloaded services, and restart-sentinel/session-continuation handling so chat-initiated restarts surface a restart message and resume/acknowledge in the original session instead of appearing crashed.

## Close Comment

Closing as implemented on main after Codex review. The regression described here is covered by shipped restart work in current OpenClaw: `CHANGELOG.md:1252`-`1254` documents the macOS LaunchAgent restart recovery that avoids leaving the gateway unloaded, and `CHANGELOG.md:281` and `CHANGELOG.md:290` document the restart-sentinel continuation path for chat-initiated restarts. The current code matches that: `src/daemon/launchd.ts:625` re-enables and re-bootstraps unloaded LaunchAgents during restart, `src/gateway/server-restart-sentinel.ts:307` restores the original session route and dispatches the post-restart continuation, and `ui/src/ui/app-gateway.ts:343` keeps restart closes surfaced as a restart message instead of an unexplained hang. This behavior is present in the shipped `v2026.4.22` release (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
