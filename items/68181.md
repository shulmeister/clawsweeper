## Summary

Keep open. Current main still matches the reported macOS screen.snapshot bug: malformed params are decoded with try? and fall back to default snapshot params, then the snapshot service is invoked; the returned image data is base64-encoded into JSON with no payload-size guard. The linked fix PR #68186 is open and unmerged, so it does not make the issue cleanup-closeable yet.

## Best Possible Solution

Keep the issue open and review or replace PR #68186 with an equivalent macOS runtime fix: treat nil params as defaults but malformed params as INVALID_REQUEST before capture, reject over-budget snapshot payloads before base64 JSON emission using the gateway payload budget, sanitize capture failures as intended, and add focused MacNodeRuntimeTests for malformed params, oversized data, and the valid snapshot path.
