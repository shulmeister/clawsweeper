## Summary

Keep open. Current `main` still swallows malformed `screen.snapshot` params, still base64-encodes snapshot bytes into JSON without a size guard, and the same code is present in the latest release `v2026.4.23`, so this is neither fixed nor obsolete.

## Evidence

- **Malformed params still fall back to defaults on current main:** `handleScreenSnapshotInvoke` still parses `paramsJSON` with `try?` and falls back to `MacNodeScreenSnapshotParams()`, so malformed JSON will not fail closed with `INVALID_REQUEST` before capture starts.

  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
