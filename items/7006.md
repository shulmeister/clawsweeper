## Summary

Keep open. Current main still does not expose the concrete model selected by OpenRouter for `openrouter/auto`, and it still computes response footer cost from local model pricing rather than OpenRouter-reported `usage.cost`. The related ClawRouter comment describes a separate router, #9016 is only a narrower cost-tracking request, and PR #43026 is closed unmerged and not present in this checkout.

## Best Possible Solution

Keep #7006 open. The best fix is to capture OpenRouter's provider-reported routed `model` and numeric usage cost in the OpenAI-compatible transport, preserve selected versus resolved model identity in assistant usage/session metadata, and thread that through `/usage` response footers and relevant status/transcript surfaces with focused regression tests. Treat #9016 as related cost-only work, not a replacement for the resolved-model transparency request.
