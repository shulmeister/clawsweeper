## Summary

Keep #7006 open. Current main still carries `openrouter/auto` as the selected OpenRouter model through the OpenAI-compatible stream path and formats response usage from local token pricing, not OpenRouter's resolved response model plus provider-reported `usage.cost` / `cost_details`. The ClawRouter comment is useful prior discussion, but the reporter correctly distinguished it from OpenRouter's own auto route, and the related PR #43026 was closed unmerged.

## Best Possible Solution

Keep #7006 open. The best fix is to capture OpenRouter's response `model` and provider-reported `usage.cost` / `cost_details` in the OpenAI-compatible transport, preserve selected-versus-resolved model metadata in assistant/session records, and thread that through response footers, `/usage`, transcripts, status/session surfaces, and regression tests for `openrouter/auto`. Treat #9016 as related cost-only work, not a replacement for the resolved-model transparency request.
