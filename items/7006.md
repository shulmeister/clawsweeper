## Summary

Keep open. Current main still does not expose the concrete model selected by OpenRouter for `openrouter/auto`, and the response footer still derives cost from local pricing/estimation rather than OpenRouter's provider-reported per-response cost. The ClawRouter comment is a useful related workaround, but the reporter correctly noted it is a separate routing service; #9016 is cost-only, and PR #43026 was closed unmerged.

## Best Possible Solution

Keep #7006 open. The best fix is to capture OpenRouter's routed `model` and numeric per-response cost (`usage.cost` and any relevant `usage.cost_details`) in the OpenAI-compatible transport, preserve selected versus resolved model identity in assistant/session metadata, and thread that through response footers, `/usage`, transcripts, and status surfaces with focused regression tests for `openrouter/auto`. Treat #9016 as related cost-only work, not a full replacement for the resolved-model transparency request.
