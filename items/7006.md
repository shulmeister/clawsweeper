## Summary

Keep open. Current main still does not expose the concrete model chosen by OpenRouter for `openrouter/auto` or provider-reported OpenRouter cost in the response footer or agent-visible metadata. Existing usage plumbing shows token counts plus locally estimated cost from the selected model config, while `openrouter/auto` still has zero bundled pricing. The provided discussion also includes a maintainer comment explicitly keeping this open, and the related PR #43026 is closed unmerged/not present in this checkout.

## Best Possible Solution

Keep #7006 open. The best implementation is to teach the OpenRouter/OpenAI-compatible completions transport to capture provider-reported routed model and cost metadata for `openrouter/auto`, preserve selected versus resolved model identity, thread actual cost through assistant usage/session metadata and transcript-backed status surfaces, and append the resolved model plus cost in the response usage footer with focused regression tests. The open cost-only issue #9016 can remain a related narrower work item, but it does not supersede the resolved-model transparency request.
