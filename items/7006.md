## Summary

Keep #7006 open. Current main still does not expose OpenRouter's resolved `openrouter/auto` model or provider-reported `usage.cost` / `cost_details` to the agent/footer path. The ClawRouter comment is useful prior art for a separate router, but the requester correctly scoped this to OpenRouter's own auto route.

## Best Possible Solution

Keep #7006 open. The best implementation path is to capture OpenRouter response `model` and provider-reported `usage.cost` / `cost_details` in the OpenAI-compatible transport, preserve selected-versus-resolved model metadata in assistant/session records, and thread that through response footers, usage/session/status surfaces, transcripts, and targeted regression tests for `openrouter/auto`. Treat #9016 as related cost-only work and #43026/ClawRouter as useful prior art, not replacements.

- A fix must preserve both selected model (`openrouter/auto`) and resolved routed model so fallback state, transcripts, status, and context-window lookup do not regress.
