## Summary

Keep open. Current main does not implement the requested OpenRouter `openrouter/auto` transparency in the response footer or agent-visible usage metadata. The existing footer shows tokens/cache plus estimated cost, the OpenRouter auto catalog still has zero configured cost, and the shared OpenAI-compatible transport keeps the requested model id rather than consuming a routed model from OpenRouter responses. The linked ClawRouter plugin is useful prior discussion, but the reporter correctly noted it solves a separate routing-service problem. Related #9016 tracks cost exposure only, and PR #43026 was closed unmerged and is not present in this checkout.
