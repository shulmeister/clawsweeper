## Summary

Keep #71112 open. Current main does not implement the proposed non-vector `ltm query` path for `memory-lancedb`; the bundled plugin currently exposes only `list`, `search`, and `stats`, and `MemoryDB` does not expose the underlying LanceDB table. The PR is recent, focused, and extension-owned, with bot review discussion indicating the earlier correctness/security findings were addressed and only a remaining performance-style concern is left.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to finish the `memory-lancedb` CLI query feature in the bundled plugin with focused tests and bounded/safe query behavior; if maintainers instead want this outside core, first define or document a memory-lancedb inspection API because the current public plugin surface does not clearly support an external implementation.
