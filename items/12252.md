## Summary

The issue asks for a public HTTP/HTTPS multi-user chatroom/account system. Current OpenClaw core intentionally keeps the built-in web UI as an operator/admin surface, while the plugin API already supports channel integrations and plugin-owned HTTP routes. A lighter public-web chatroom belongs as a ClawHub/plugin project; the existing Mattermost plugin remains the relevant bundled self-hosted chat option mentioned by the reporter.
