## Summary

Keep open. Current main has canonical skill roots and a symlink/realpath escape guard for skills discovered through those roots, but it does not implement the requested orphan detection for standalone `SKILL.md` directories outside configured skill locations. The bundled skill scaffolding and packaging scripts still accept arbitrary paths without a canonical-location warning.

## Evidence

- **Loader only enumerates configured skill roots:** Skill discovery is limited to bundled, configured extra/plugin dirs, managed `~/.openclaw/skills`, `~/.agents/skills`, workspace `.agents/skills`, and `<workspace>/skills`; there is no scan of `~/my-skill`-style standalone directories.
