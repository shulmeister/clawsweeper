## Summary

Keep open. Current main has the existing Windows cmd wrapper hardening, but it has not absorbed this PR's proposed validator/export, and the PR's own review thread identifies unresolved regressions that are grounded in current Windows paths, scoped package specs, and npm flags.

## Best Possible Solution

Keep the PR open for maintainer/security review. A viable fix should be narrower than this patch: preserve normal Windows paths, switches, scoped package names, and '=' options; avoid echoing full argv values in errors; and add focused regression tests around .cmd/.bat wrapping, scoped installs, update flags, and any new null/control-character validation. If maintainers decide the existing cmd-wrapper hardening is sufficient, that should be an explicit product/security decision rather than an automated close.
