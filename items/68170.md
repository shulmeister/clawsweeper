## Summary

Keep open. Current main still has the reported behavior: `findOtherStateDirs` enumerates `/home` on Linux and `/Users` on macOS, skips only dot-prefixed directory names, adds any sibling `.openclaw` directory it can stat, and doctor prints those paths in the “Multiple state directories detected” warning. I found no env/config opt-out, backup-name filtering, current-user scoping, or redaction for non-current-home paths, and the docs still describe cross-home-directory detection as intended behavior.

## Best Possible Solution

Keep the issue open and patch the Doctor state integrity check. The conservative fix should either constrain discovery to the current effective home/service user or remove broad sibling-home enumeration, add an explicit skip/config escape hatch if maintainers want the diagnostic to remain optional, redact or summarize non-current-home paths, and add focused tests for cross-user `/home/alice`/`/home/bob` and backup-sibling directory cases. Update `docs/gateway/doctor.md` and public docs wording if the scan semantics change.
