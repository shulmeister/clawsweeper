## Summary

Keep open. Current main still has the default daily reset policy and does not implement the main requested safeguards: an off mode, pre-reset warning/cancel/postpone, or restore of reset archives into the active session. The current archive behavior reduces the data-loss framing, but it does not solve the observable user problem described in #61238.
