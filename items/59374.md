## Summary

Keep open. Current `main` does not implement multiple Tavily API keys or automatic failover: the bundled Tavily plugin documents a single `TAVILY_API_KEY`, its config schema accepts one secret value at `plugins.entries.tavily.config.webSearch.apiKey`, the resolver returns one key, and the client sends one `apiKey` per request with no 429/quota rotation path.

## Evidence

- **User docs still describe one Tavily key:** The Tavily docs tell users to set `TAVILY_API_KEY` and show `webSearch.apiKey` as a single string in the config example.
  - file: [docs/tools/tavily.md:25](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/docs/tools/tavily.md#L25)

- **Bundled plugin schema is single-key shaped:** The plugin metadata exposes one auth env var (`TAVILY_API_KEY`) and defines `webSearch.apiKey` as `string|object`, not an array.
  - file: [extensions/tavily/openclaw.plugin.json:4](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/tavily/openclaw.plugin.json#L4)
