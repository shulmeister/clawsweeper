## Summary

Keep #39673 open. Current main has partial Linux user-systemd recovery, including machine-scope fallback and classification for `Failed to connect to bus: No medium found`, but the real `gateway install` path still has unrecovered systemd-user failure exits and still reports raw install errors without the remediation hints that status/lifecycle paths know how to render. Related #63561 is adjacent systemd-user detection work, not a complete duplicate of this original install-fails regression.

## Best Possible Solution

Keep #39673 open and fix the Linux gateway install path directly. The best maintainer path is to share the existing systemd-unavailable hint rendering with `gateway install` failures and add a focused regression test that drives the real `service.install` / `writeSystemdUnit` / activation path when `assertSystemdAvailable`, `daemon-reload`, `enable`, or `restart` returns `Failed to connect to bus` / no-medium-style output. Keep #63561 cross-linked as adjacent systemd-user detection work unless a maintainer chooses one canonical tracker.
