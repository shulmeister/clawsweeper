## Summary

Keep open. Current main still lacks create/update-time Feishu/Lark announce target rejection, and #39599 remains open. This PR is a focused candidate fix, but the raw `delivery.to` requirement is too broad because current delivery resolution can derive targets from session/plugin routing and the update path can block remediation of legacy jobs.

## Best Possible Solution

Keep the PR open, but request a rework. The best fix should validate unresolved Feishu/Lark announce targets through the current delivery-resolution seam, allow explicit `delivery.to` and deterministic session/plugin resolution, avoid blocking unrelated updates on legacy jobs, keep provider-specific target hints in the Feishu plugin surface, and add focused create/update regression coverage.
