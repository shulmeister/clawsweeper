## Summary

Current `main` already contains a WhatsApp reconnect controller, bounded backoff/max-attempt handling, per-reconnect listener/socket cleanup, and regression coverage for reconnect-loop/watchdog behavior, which matches the reported crash path closely enough to treat this as implemented on `main`.

## Evidence

- **Bounded reconnect policy now exists:** WhatsApp reconnects now default to exponential backoff with jitter and a bounded `maxAttempts` cap instead of an endless tight retry loop.
  - file: [extensions/whatsapp/src/reconnect.ts:15](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/whatsapp/src/reconnect.ts#L15)
  - command: `nl -ba extensions/whatsapp/src/reconnect.ts | sed -n '1,260p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Reconnect controller stops and cleans up connections:** The controller normalizes close reasons, stops retrying after max attempts, and on every reconnect teardown clears timers, unregisters handlers, waits background tasks, closes the listener, and closes the socket.
  - file: [extensions/whatsapp/src/connection-controller.ts:427](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/whatsapp/src/connection-controller.ts#L427)
  - command: `nl -ba extensions/whatsapp/src/connection-controller.ts | sed -n '260,680p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Inbound monitor detaches listeners on shutdown:** The WhatsApp inbox listener now detaches `messages.upsert` and `connection.update` handlers and closes the socket during listener shutdown, which is the main leak-prevention path for repeated reconnect cycles.
  - file: [extensions/whatsapp/src/inbound/monitor.ts:700](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/whatsapp/src/inbound/monitor.ts#L700)
  - command: `nl -ba extensions/whatsapp/src/inbound/monitor.ts | sed -n '620,860p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Regression tests cover reconnect-loop behavior:** Tests now cover reconnect progress/max-attempt stop behavior and verify that a reconnected listener gets a fresh watchdog window instead of immediately re-entering the stale/reconnect path.
  - file: [extensions/whatsapp/src/auto-reply.web-auto-reply.connection-and-logging.e2e.test.ts:80](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/whatsapp/src/auto-reply.web-auto-reply.connection-and-logging.e2e.test.ts#L80)
