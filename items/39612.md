## Summary

Keep open. Current main has partial Anthropic thinking-block recovery and transcript hygiene, but it still deliberately forwards sanitized provider/LLM exception detail to external chat users. That leaves the central request unresolved: framework-level handling that never exposes raw LLM/API errors on WhatsApp, Telegram, or other user messaging channels.

## Best Possible Solution

Keep this issue open until the auto-reply framework changes the external-channel LLM/provider failure policy: retry only when safe, otherwise send a generic friendly fallback or NO_REPLY without provider exception detail. Preserve the existing targeted thinking-block recovery, add WhatsApp/Telegram regression coverage for hidden raw errors, and separately decide whether consecutive failures should notify the default agent.
