## Summary

Current `main` already reworked the `memory-core` managed dreaming cron so it no longer depends on the old heartbeat-only wake path: startup defers cleanly when cron is not attached yet, runtime reconciliation still retries/warns if cron stays unavailable, and direct cron-triggered dreaming is now handled explicitly. The report matches older released behavior, not the code on `main`.

## Evidence

- **Current main creates an isolated cron-driven dreaming job:** `buildManagedDreamingCronJob` now creates the managed dreaming job as `sessionTarget: "isolated"` with an `agentTurn` payload and `delivery.mode: "none"`, replacing the older main-session system-event shape that could short-circuit through the heartbeat-only path.
