## Summary

Current main already strips `[[reply_to:...]]` / `[[reply_to_current]]` from iMessage outbound text before send, and that implementation is also present in shipped release `v2026.4.22`, so the reported literal-tag bug is already fixed.

## Close Comment

Closing from a Codex review: current `main` already strips inline reply directives before iMessage delivery. The outbound send path calls `stripInlineDirectiveTagsForDelivery` in `extensions/imessage/src/send.ts:147`, and that helper removes `[[reply_to_current]]` / `[[reply_to:<id>]]` tags in `src/utils/directive-tags.ts:123`. The same code is present in shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), with regression coverage in `src/utils/directive-tags.test.ts:32`, so the literal-tag behavior reported against `2026.3.23-1` is already fixed on supported main/release. If this still reproduces on `v2026.4.22` or newer, please open a fresh issue with the exact current version and a new reproduction trace.
