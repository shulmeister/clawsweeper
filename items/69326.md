## Summary

Keep #69326 open. Current main does not implement the requested behavior for the browser WebChat path: the Control UI sends `chat.send` with `deliver: false`, `chat.send` resolves that as an internal WebChat turn, and tests explicitly assert that WebChat clients do not inherit an existing external `deliveryContext` even when one is present.

## Best Possible Solution

Maintainer should decide the intended product behavior for WebChat-on-channel sessions. If replies from the browser WebChat UI should optionally go back to the session `deliveryContext`, the change likely belongs in the generic `chat.send` routing path and UI delivery controls, with regression coverage for `openclaw-weixin` or a generic plugin-channel session. If WebChat must remain internal-only, the issue should be answered as an intentional behavior decision rather than auto-closed as fixed.
