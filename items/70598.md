## Summary

Keep PR #70598 open. Current main already shipped the central exec heartbeat prompt fix via #71213, but the PR still contains useful follow-up work for a current-main mixed-queue gap: exec heartbeat prompts consume only exec events, while completion cleanup drains the entire inspected queue, dropping deferred cron/non-exec events. The later review comment about preserving duplicate deferred reminders remains relevant to current main.

## Best Possible Solution

Keep the PR open, but ask for a rebase/narrowing against current main: retain the shipped #71213 exec prompt behavior, then land only the remaining queue-consumption fix and regression tests that preserve non-exec and duplicate deferred events after exec-event heartbeat runs.
