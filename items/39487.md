## Summary

Keep PR #39487 open. Current main still does not wire Feishu lifecycle or inbound liveness status into the gateway health-monitor snapshot, and the overlapping #41940 PR is open/not merged rather than a clear maintainer-selected superseder. The PR should be updated before merge because existing review comments identify correctness gaps, but the underlying Feishu status-reporting work remains real and unimplemented on main.

## Best Possible Solution

Keep the PR open or ask the author to update it. The best fix belongs in the Feishu plugin path: pass ctx.setStatus from extensions/feishu/src/channel.ts into monitor.ts, monitor.account.ts, and monitor.transport.ts; publish connected only after WebSocket start or webhook listen succeeds; publish disconnected/error status on abort and startup/runtime failures including early-abort paths; update inbound event liveness; publish lastTransportActivityAt when Feishu transport activity proves liveness; and add focused lifecycle/inbound regression tests. If maintainers later select #41940 as the canonical implementation, close #39487 as duplicate at that point.
