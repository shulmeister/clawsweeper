## Summary

Keep open. Current main still does not implement Feishu status/liveness reporting into gateway runtime state, and the submitted PR has unresolved review comments around premature connected-state reporting and early-abort status gaps. The related #41940 context is a parallel open attempt, not a merged or clearly canonical replacement.

## Best Possible Solution

Keep the PR open until a maintainer accepts a corrected Feishu implementation or explicitly chooses a canonical replacement. The best fix should wire a status sink from `extensions/feishu/src/channel.ts` through `monitor.ts`, `monitor.account.ts`, and `monitor.transport.ts`; publish connection state only after WebSocket start or webhook listen succeeds; publish disconnected/error state on abort/error including early-abort paths; use `lastTransportActivityAt` for actual transport liveness if health-monitor stale-socket recovery is the target; and add focused regression coverage for inbound activity plus transport lifecycle behavior. If #41940 becomes the maintained canonical PR, close this one as duplicate after that is clear.
