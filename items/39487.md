## Summary

Keep PR #39487 open. Current main still does not wire Feishu runtime lifecycle or liveness status into the gateway health-monitor snapshot, and the parallel #41940 PR is open/not merged rather than a clear maintainer-selected superseder. The PR itself still needs updates for the existing review comments and current transport-liveness semantics, but the underlying Feishu bug path remains real and unimplemented on main.

## Best Possible Solution

Keep the PR open or ask the author to update it. The best fix belongs in the Feishu plugin path: bind ctx.setStatus in extensions/feishu/src/channel.ts, thread a typed status sink through monitor.ts, monitor.account.ts, and monitor.transport.ts, publish connected only after WebSocket start or webhook listen succeeds, publish disconnected/error state on abort and failures including early-abort paths, update inbound event liveness, publish lastTransportActivityAt for actual transport liveness, and add focused lifecycle/inbound regression tests. If maintainers later select #41940 as the canonical implementation, then close #39487 as duplicate at that point.
