## Summary

Keep open. Current main has a bundled, enabled-by-default `microsoft-foundry` provider but still lacks a dedicated Microsoft Foundry provider docs page and docs navigation entry. The PR is not obsolete or implemented on main, and the earlier Greptile Codex-table comment appears addressed in the latest diff, but the docs still need maintainer/docs review to match the provider's current API-selection behavior and to coordinate with related docs PRs #72158 and #68502.

## Best Possible Solution

Keep the PR open and ask for a docs revision: add the Microsoft Foundry page and navigation/index coverage, coordinate or supersede #72158/#68502 as appropriate, and align the guide with `extensions/microsoft-foundry/shared.ts` for API auto-selection, auth choices, endpoint normalization, and onboarding flow.
