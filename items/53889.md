## Summary

Keep this issue open. Current main still has the live deadlock shape: tool-result delivery promises are added to `pendingToolTasks`, and the reply runner drains that set with an unbounded `Promise.allSettled()` before reaching `replyOperation.complete()`. Existing transcript repair and synthetic tool-result guards help replay/cleanup cases, but they do not bound this active reply-run drain. The related open PR #64733 remains an implementation candidate, not evidence that this issue is fixed on main.

## Best Possible Solution

Keep this issue open and resolve it by reviewing, landing, or superseding PR #64733 with an equivalent main-branch fix: add a bounded timeout or cancellation path around the `pendingToolTasks` drain, log or surface the timeout, guarantee `replyOperation.complete()` is reached so the session lane can recover, and add a focused regression test with a never-settling tool-result delivery task.
