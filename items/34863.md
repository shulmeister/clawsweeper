## Close Comment

Closing as `implemented_on_main` after a Codex review. Current `main` now handles terminal ACP final-message delivery by emitting the assistant snapshot before resolving `end_turn` (`src/acp/translator.ts:946`), and the interactive client renders `agent_message_chunk` text to stdout (`src/acp/client.ts:86`). There is also a regression test that asserts final snapshot text is emitted before `end_turn` resolves (`src/acp/translator.session-rate-limit.test.ts:1064`). The changelog records this fix in `v2026.3.12` (`CHANGELOG.md:2646`), so the `[end_turn]`-only behavior reported against `2026.3.2` is already fixed on current `main`. If this still reproduces on a newer release, please open a new issue with the current version, exact repro, and fresh logs.
