## Summary

Current main still does not implement configurable Amazon Bedrock Converse `anthropic_beta` or `additionalModelRequestFields` passthrough. The original 1M-preview motivation may be obsolete for Claude 4.6 GA, per the issue discussion, but the broader requested Bedrock-specific beta/request-field config remains absent and is concrete enough to keep open.

## Best Possible Solution

Keep this open for a Bedrock-plugin-owned implementation or explicit maintainer product decision. A good fix would add a validated per-model or plugin config surface for Bedrock Converse `anthropic_beta`/allowed `additionalModelRequestFields`, read it from `ctx.extraParams` or `plugins.entries.amazon-bedrock.config`, merge it in `extensions/amazon-bedrock/register.sync.runtime.ts`, and cover it with Bedrock plugin tests plus `docs/providers/bedrock.md`.

## Risks / Open Questions

- The original `context-1m-2025-08-07` use case may be obsolete for current Claude 4.6 GA models on Bedrock, so any fix should focus on a validated general Bedrock Converse beta/additional-request-field surface rather than hardcoding that one flag.
- A broad arbitrary JSON passthrough could create provider-side request failures or unsafe config drift; a conservative implementation should validate or narrowly allow supported Bedrock Converse additional fields.
