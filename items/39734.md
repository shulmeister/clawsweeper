## Summary

Keep open. Current main covers the narrower Claude 4.6 Bedrock 1M context metadata path and direct Anthropic beta headers, but it still does not provide a configurable Bedrock `anthropic_beta` / `additionalModelRequestFields` passthrough in the bundled `amazon-bedrock` provider.

## Best Possible Solution

Keep the issue open for maintainer product judgment or a Bedrock-owned implementation. A concrete fix should add a validated Bedrock config or agent-model param surface for allowed `anthropic_beta` / Converse additional request fields, merge it in the `amazon-bedrock` stream wrapper, and cover it with Bedrock plugin tests plus `docs/providers/bedrock.md`. If maintainers decide arbitrary Bedrock beta passthrough is out of scope now that the 1M use case is GA, that should be an explicit product decision rather than automated cleanup.
