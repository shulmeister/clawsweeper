## Summary

Keep open. Current main has several exact-duplicate protections for stream/final events, WebChat stale-history reloads, chat.send idempotency, and messaging-tool echoes, but I did not find source, tests, docs, related-item, or recent-history evidence that it fixes the broader WebChat/Qwen/Omlx repetition described here: repeated assistant preambles and intent statements across tool/approval steps in one interaction.

## Best Possible Solution

Keep this issue open and ask for a current-main WebChat reproduction with the raw session transcript, `chat` and `agent` gateway events, provider/model config, and whether tools or approvals were involved. If the duplicate is UI/history rendering, add a regression around `ui/src/ui/controllers/chat.ts` or `ui/src/ui/app-gateway.ts`; if it is duplicate event delivery, fix the embedded subscriber/gateway idempotency path; if it is model-generated repetition for Qwen/Omlx, tune the WebChat/system prompt or provider-specific guidance and cover it with a focused regression.
