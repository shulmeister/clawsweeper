## Summary

Current main does not satisfy the requested behavior for the bundled session-memory hook. The hook formats memory filenames and the `# Session` header from the host/process local timezone (`TZ` or Intl default), not from `agents.defaults.userTimezone`; the existing regression test only covers `TZ=America/New_York`, not a configured user timezone such as `Asia/Jakarta`. The linked PR #63583 proposed the right direction but is closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep this issue open and implement the session-memory hook side explicitly: resolve `agents.defaults.userTimezone` with the existing `resolveUserTimezone` helper, format the filename date, fallback HHMM slug, and `# Session` header in that resolved IANA timezone, and add a regression test using a fixed timestamp plus `agents.defaults.userTimezone: "Asia/Jakarta"` without relying on `TZ`. The docs for session-memory should then say it uses the configured user timezone with host fallback, while preserving the existing memory-core compaction behavior.
