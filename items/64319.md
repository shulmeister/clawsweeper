## Summary

This issue is a newer report of the same GitHub Copilot device-flow hang already tracked by open PR #46698. Current main still has the old polling behavior in the GitHub Copilot plugin, so this is not implemented-on-main; the right maintainer path is to review/refresh #46698 or apply the same fix to the current plugin-owned files.

## Best Possible Solution

Close this issue as superseded by #46698, then review, refresh, and land #46698 or implement the equivalent provider-local fix in `extensions/github-copilot/login.ts`, with CLI/docs/tests updated for any `--wait` or polling-contract changes.
