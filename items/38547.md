## Summary

Keep #38547 open. Current main still does not implement native remote delegation for `sessions_spawn` or `sessions_send`: neither tool exposes `gatewayUrl`/`gatewayToken`, both dispatch paths still call the current gateway's local `agent` RPC, and the existing safe remote override support remains limited to separate gateway-style helpers. The prior ClawSweeper keep-open comment remains accurate on current `main`.

## Best Possible Solution

Keep #38547 open until maintainers either implement or explicitly reject native remote session delegation. A concrete implementation would add documented `gatewayUrl`/`gatewayToken` support to `sessions_spawn` and `sessions_send`, reuse the existing safe gateway override validation and credential resolution, thread resolved remote call options through native subagent and ACP dispatch, and add tests proving model calls, tool execution, transcript storage, and credential use happen on the remote gateway without leaking local credentials.
