## Summary

Keep #38547 open. Current main still does not implement native per-call remote gateway delegation for `sessions_spawn` or `sessions_send`: the tool schemas do not expose `gatewayUrl`/`gatewayToken`, the execution paths do not thread a gateway override into subagent/session dispatch, and the existing remote-gateway override helper is only used by the separate `gateway` tool path.

## Best Possible Solution

Keep the issue open until maintainers either implement a native remote-delegation contract or explicitly reject the product direction. A likely implementation would add documented `gatewayUrl`/`gatewayToken` support to `sessions_spawn` and `sessions_send`, reuse the existing safe gateway override validation/credential rules, thread the resolved remote call options through subagent lifecycle/session send paths, and add tests proving model/tool execution is dispatched to the remote gateway without leaking local credentials.
