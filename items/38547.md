## Summary

Keep #38547 open. Current `main` still does not provide native remote gateway delegation for `sessions_spawn` or `sessions_send`: the public tool schemas omit `gatewayUrl`/`gatewayToken`, the execution paths do not read those fields, and both native subagent and ACP dispatch still call the current gateway. The prior ClawSweeper keep-open comment remains accurate against `82262387651ab270b565f71533ce4937c9c00dc6`.

## Best Possible Solution

Keep #38547 open until maintainers implement or explicitly reject native remote session delegation. A concrete implementation should add documented `gatewayUrl`/`gatewayToken` or configured remote-target support to `sessions_spawn` and `sessions_send`, reuse the existing safe gateway override and credential-resolution helper, thread the resolved remote call options through native subagent, ACP spawn, send, wait/history, cleanup, and registry paths, and add tests proving model calls, tools, transcripts, filesystem effects, and auth/account usage occur on the remote gateway.
