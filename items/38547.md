## Summary

Keep open. Current main still does not implement native per-call remote gateway delegation for `sessions_spawn` or `sessions_send`: the session tool schemas and execution paths do not accept or thread `gatewayUrl`/`gatewayToken`, while other gateway-backed tools do have that override surface. The issue remains a specific, actionable core/gateway feature request.

## Best Possible Solution

Keep the issue open for maintainer/product work. A core implementation would need to add a validated remote gateway target to `sessions_spawn` and `sessions_send`, likely reusing the existing gateway-tool URL/token validation, then thread that target through subagent/ACP/session-send gateway calls so model auth, filesystem/tool execution, transcripts, and lifecycle events run on the selected remote gateway. It should include protocol/security docs and tests covering local behavior, explicit remote URL/token behavior, scope/auth failures, and remote execution ownership.
