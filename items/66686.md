## Summary

Keep #66686 open. Current main still contains the broad memory-recall prompt and tool description the report calls out, and I did not find a current-state/configuration exception or classifier that would make the requested behavior already implemented. The issue is recent, unaffiliated, has no protected labels, and no canonical duplicate in the provided related context covers this narrower over-generalization bug.

## Best Possible Solution

Refine the memory-core prompt and `memory_search` tool description to distinguish historical recall from current system/configuration verification, then add targeted prompt-section tests that keep historical memory recall mandatory while allowing direct verification for current state questions. The likely code owner surface is `extensions/memory-core/src/prompt-section.ts`, `extensions/memory-core/src/tools.ts`, and colocated memory-core tests; a broader product decision is not needed unless maintainers want a general question-classification policy across all tools.
