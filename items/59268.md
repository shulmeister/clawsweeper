## Summary

Keep open. Current `main` still points the macOS chat UI at the gateway-wide main session key from gateway config/snapshot, not a per-device or per-node session, so the reported shared-session behavior remains plausible. The static `openclaw-macos` client id is still present, but current code also sends a unique `instanceId`, so the issue is not clearly obsolete and the root-cause hypothesis is not proven away.

## Evidence
