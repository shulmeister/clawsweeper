## Summary

Keep open. Current `main` appears to have fixed the original startup-schema rejection by validating `channels.<id>` from manifest-owned channel metadata with origin precedence, but the doctor preview path still reports disabled bundled plugins as blockers even when an external plugin owns the same channel id. That remaining warning path matches the April 23 follow-up comment and there is no evidence of a current-main fix for it.

## Evidence

- **Validation now prefers manifest-owned channel metadata over bundled fallback:** `collectChannelSchemaMetadata()` ranks plugin origins `config > workspace > global > bundled` and keeps higher-precedence `channelConfigs.<id>.schema` metadata for a shared channel id, which is then overlaid onto the bundled schema map during config validation.
  - file: [src/config/channel-config-metadata.ts:9](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/config/channel-config-metadata.ts#L9)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Config validation uses the collected manifest channel schemas, not only the legacy bundled schema map:** `ensureChannelSchemas()` seeds from generated bundled metadata but then replaces entries with `collectChannelSchemaMetadata(info.registry)` before validating `channels.<id>`. That explains why the startup `additional properties` crash described in the original report no longer looks like the active behavior on `main`.
  - file: [src/config/validation.ts:896](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/config/validation.ts#L896)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Doctor blocker scan still flags any disabled plugin whose declared channel id is configured:** `scanConfiguredChannelPluginBlockers()` iterates every manifest-registry plugin, computes activation state per plugin id, and emits a hit for each configured `channelId` on a disabled plugin. It does not check whether another active plugin also owns that same channel id, so a disabled bundled `feishu` plugin can still trigger a warning while an external Feishu/Lark plugin is intended to take over.
  - file: [src/commands/doctor/shared/channel-plugin-blockers.ts:35](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/commands/doctor/shared/channel-plugin-blockers.ts#L35)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Doctor preview always surfaces those blocker hits:** `collectDoctorPreviewWarnings()` imports `channel-plugin-blockers` when plugin disablement is present and unconditionally appends the formatted blocker warnings to doctor output.
  - file: [src/commands/doctor/shared/preview-warnings.ts:78](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/commands/doctor/shared/preview-warnings.ts#L78)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **No current-main change was found for the doctor blocker path after the latest release:** `git diff --stat a9797214338ba31c52c796adbb75afb16e0684a9..95b7a85f0607784d8e33b4ce7e42617174196490 -- src/commands/doctor/shared/channel-plugin-blockers.ts src/commands/doctor/shared/preview-warnings.ts` produced no output, so the April 23 report about `openclaw doctor` still maps to the same code on current `main`.
  - command: `git diff --stat a9797214338ba31c52c796adbb75afb16e0684a9..95b7a85f0607784d8e33b4ce7e42617174196490 -- src/commands/doctor/shared/channel-plugin-blockers.ts src/commands/doctor/shared/preview-warnings.ts`
- **Issue discussion already narrowed the remaining bug:** The April 23, 2026 follow-up comment reports that OpenClaw boots successfully on 2026.4.21 but `openclaw doctor` still warns `channel "feishu" configured but plugin disabled` for the bundled-disabled/external-plugin takeover setup. That aligns with the current blocker-scan code.
