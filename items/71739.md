## Summary

Current main still matches the reported problem. `agents list` has no command-policy override, so it inherits `loadPlugins: "always"` from the parent `agents` command; JSON mode only suppresses loading for `text-only` policies. The command body also still builds provider status before emitting JSON, so the proposed fast JSON path has not landed. The related issues are broader/open performance work, not a clear canonical replacement for this surgical `agents list --json` item.
