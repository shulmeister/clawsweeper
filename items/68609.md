## Summary

Keep #68609 open. Current main still does not implement the requested always-on preemptive-compaction debug path: when the precheck returns `route: "fits"`, the call site falls through to prompt submission without logging the estimated tokens, reserve-adjusted budget, overflow, or route. The cache-ttl pruning concern is still plausible enough to merit maintainer work rather than closure.

## Best Possible Solution

Keep the issue open and implement a low-noise debug diagnostic immediately after `shouldPreemptivelyCompactBeforePrompt()` is evaluated in `src/agents/pi-embedded-runner/run/attempt.ts`, covering every route including `fits`. Include `route`, `estimatedPromptTokens`, `promptBudgetBeforeReserve`, `overflowTokens`, reserve/effectiveReserve, model, session key, and session file, and add a focused regression test for the `fits` branch. Separately, add coverage or diagnostics that compare pruned versus unpruned estimates for cache-ttl large-context sessions before changing compaction thresholds.
