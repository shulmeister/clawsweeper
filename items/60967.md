## Summary

Keep #60967 open. Current main still does not expose or pass a per-tool MCP request timeout for bundled `mcp.servers`; it only resolves `connectionTimeoutMs` for connect/list-tools startup, while `callTool` invokes the MCP SDK without request options. The issue is recent, has no protected labels, and the related closed items do not supersede this exact remaining bug.
