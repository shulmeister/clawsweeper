## Close Comment

Closing from a Codex review because current `main` already covers this behavior, and the same implementation is shipped in `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). The connect view builds its logo URL from `state.basePath` in `ui/src/ui/views/login-gate.ts:10`, `agentLogoUrl()` emits a mounted path like `/openclaw/favicon.svg` in `ui/src/ui/views/agents-utils.ts:243`, and mounted-path behavior is locked by tests in `ui/src/ui/views/agents-utils.test.ts:106` and `ui/src/ui/navigation.test.ts:168`. The gateway also redirects the bare mount path to a normalized trailing-slash URL in `src/gateway/control-ui-routing.ts:47`. If this still reproduces on a newer build, please open a fresh issue with the exact served URL and any reverse-proxy or `gateway.controlUi.basePath` setup.
