## Summary

Keep open. Current main has useful partial hardening for QA gateway cleanup and transport readiness, but it does not fully satisfy the issue: host suite runs still do not register interruption cleanup for the detached gateway child, `qa-channel` config is written before the QA transport is confirmed ready, and fixed sleeps still remain in the restart/readiness path. The linked PR was closed unmerged, so it is not a shipped implementation for this issue.

## Best Possible Solution

Keep this issue open and land a focused QA harness fix: add explicit ownership/signal cleanup for host suite gateway children and temp roots, add an orphan-reap strategy for stale QA suite roots where feasible, replace post-restart sleeps with a concrete gateway/transport readiness handshake, and gate `qa-channel` config emission or activation on a healthy QA bus. Add regression tests for interrupted suite cleanup, temp-root removal, restart readiness, and config generation when the QA transport is unavailable.
