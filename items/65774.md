## Summary

Current main partially addresses the reported stale-running startup replay, but the broader safety problem is not fully implemented. Cron removal still only removes persisted job records, running cron executions do not have a service-level abort registry, and there is no cron/gateway emergency stop surface that aborts in-flight outbound delivery.

## Best Possible Solution

Keep this open and fix it in cron/gateway core. The best path is a focused cron safety change: retain the current interrupted-running startup protection, add a registry for active cron run AbortControllers, make cron.remove/disable and gateway shutdown able to abort matching in-flight runs, add an explicit emergency stop or pause-all outbound path, and add regression coverage for America/Guatemala out-of-hours reconnect plus deleting/aborting an active WhatsApp-style isolated delivery. If #55160 becomes the canonical maintenance-window/deferred-replay design, link this issue there but do not close it before the relevant safety behavior lands.
