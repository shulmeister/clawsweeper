## Summary

Keep open. Current main still hardcodes the MiniMax usage polling host to the CN endpoint, so the linked user-visible bug is not implemented on main. This PR is a plausible implementation candidate for #65054, but it needs maintainer review or revision because the provided diff chooses a baseUrl by static provider-config priority while current usage auth can resolve the token from a different MiniMax credential source. The related open #63073 tracks the same endpoint area, but it is not a clean supersession because it hardcodes the global host rather than preserving region/config behavior.

## Best Possible Solution

Keep this PR open for maintainer review or author revision. The best fix is to consolidate with #63073/#65054 and implement source-aligned MiniMax usage host selection: derive the host from the same provider config or auth source that supplied the usage token, preserve CN/global fallback behavior intentionally, and add focused tests that capture the requested URL for global, CN, missing/invalid baseUrl, and both-provider-config cases.
