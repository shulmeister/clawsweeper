## Summary

Keep open. Current `main` raises the default agent timeout to 48 hours, which masks the reporter's original default-600-second reproduction, but the underlying classification path still appears unchanged: the embedded run timer is still run-wide, only `timedOutDuringCompaction` is exempted, and assistant failover still treats any other timeout as an LLM timeout/fallback candidate.

## Evidence

- **Run timeout is still a whole-run timer with only a compaction exemption:** `runEmbeddedAttempt` initializes `timedOutDuringCompaction = false`, schedules a single abort timer for `params.timeoutMs`, and on expiry only checks compaction before calling `abortRun(true)`. There is no parallel `timedOutDuringToolExecution` or phase/cause enum in the attempt result.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:490](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/agents/pi-embedded-runner/run/attempt.ts#L490)

  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Failover policy still escalates any timeout that was not during compaction:** `shouldRotateAssistant` returns true for `(params.timedOut && !params.timedOutDuringCompaction)`, and the unit test still expects assistant timeouts to surface only as a special case for external aborts. No tool-execution carveout exists.
