## Summary

Keep #40438 open. Current main still has no configurable directory exclusion for memory indexing: extraPaths are normalized and walked recursively, the walker only skips symlinks, .openclaw-repair/root-memory auxiliary paths, and non-memory file types, and config/schema/docs/status expose extraPaths but not excludeDirs or exclude patterns. The PR addresses a real user problem from #6109, but it needs rebase/scope cleanup before merge because current main moved this surface into memory-core/host SDK files and the PR contains an unrelated ipv4-fix.cjs file plus an unresolved root-extraPaths bypass review note.

## Best Possible Solution

Keep the item open and salvage the focused feature: rebase onto current main, remove ipv4-fix.cjs, implement the exclusion contract in src/memory-host-sdk/host/internal.ts and its package mirror/exported surfaces, thread it through src/agents/memory-search.ts and extensions/memory-core/src/memory/manager-sync-ops.ts/status, add schema/help/docs/generated-baseline updates, and cover default exclusions, user-added exclusions, default memory dir exclusions, and excluded extraPaths roots in tests.
