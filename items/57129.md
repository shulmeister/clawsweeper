## Summary

Current `main` does not implement fresh isolated runs for unrelated top-level Slack channel questions. The routing code and regression tests explicitly keep top-level channel messages on one per-channel session, so this item should stay open.

## Evidence

- **Current routing keeps top-level channel turns on the channel session:** `resolveSlackRoutingContext()` explicitly says top-level channel messages must stay on the per-channel session and only uses a thread-scoped key for real thread replies. That directly conflicts with the issue's requested fresh top-level isolation.
  - file: [extensions/slack/src/monitor/message-handler/prepare-routing.ts:68](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/slack/src/monitor/message-handler/prepare-routing.ts#L68)
  - command: `nl -ba extensions/slack/src/monitor/message-handler/prepare-routing.ts | sed -n '1,220p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Regression test locks in shared top-level session behavior:** Current tests assert that top-level channel messages reuse the same session key regardless of `replyToMode`, and explicitly assert no `:thread:` suffix for those turns.
  - file: [extensions/slack/src/monitor/message-handler/prepare.thread-session-key.test.ts:100](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/slack/src/monitor/message-handler/prepare.thread-session-key.test.ts#L100)

  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **User-facing docs describe channel-root sessions, not fresh top-level isolation:** Slack docs say channel sessions are `agent:<agentId>:slack:channel:<channelId>` and that only thread replies add `:thread:<threadTs>`, which matches the current code path rather than the requested behavior.
  - file: [docs/channels/slack.md:516](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/channels/slack.md#L516)
  - command: `nl -ba docs/channels/slack.md | sed -n '512,530p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **There is a manual workaround, but it is not automatic isolation:** Slack docs expose `/new` as 'Start a new session', and current tests cover Slack mention-prefixed `/new`. That is a manual reset path, not automatic fresh context for every unrelated top-level message.
  - file: [docs/channels/slack.md:245](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/channels/slack.md#L245)
