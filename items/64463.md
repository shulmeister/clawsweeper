## Summary

Keep this issue open. Current main has session reset, session-store disk maintenance, context-window compaction, and token/cost reporting, but it does not implement the requested opt-in hard caps for wall-clock session duration or cumulative per-session token spend. The follow-up comment on this issue is a plausible implementation proposal rather than a duplicate or shipped fix, and the related reports cover adjacent cron/context issues rather than this general session budget guard.

## Best Possible Solution

Keep the issue open and implement this as a core session-budget feature, not a ClawHub plugin. A good path is to add optional schema/types/docs for session budget limits, persist enough per-session usage/lifecycle state to evaluate them reliably, enforce before costly LLM/tool work and on existing lifecycle ticks, emit deterministic user/operator feedback, and cover embedded Pi, cron/heartbeat, and CLI/runtime edge cases with focused tests.
