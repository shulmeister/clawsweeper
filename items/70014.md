## Summary

Keep this issue open. Current main still builds memory-core dreaming workspaces from configured agents first and only adds the runtime primary workspace when no configured workspaces resolve, so a setup with AGI sub-agent entries can still skip `/root/.openclaw/workspace`. The prior ClawSweeper comment identified the same root cause, and the current checkout still matches it.

## Best Possible Solution

Keep this issue open and fix memory-core dreaming workspace enumeration so scheduled dreaming processes the runtime/primary workspace plus configured agent workspaces, deduped by normalized path. The likely implementation belongs in the runtime dreaming call sites that already have `ctx.workspaceDir` or the memory manager status workspace, with regression coverage for a non-empty `agents.list` containing sub-agents while the primary workspace only comes from runtime/status context.
