## Summary

Keep #70014 open. Current main still builds the dreaming workspace list from `agents.list` when that list exists, and only falls back to the default agent when no configured agent IDs are found. The scheduled dreaming promotion path then uses that nonempty list and only considers the triggering workspace as a fallback when the list is empty, so a config with four AGI sub-agents and no explicit `main` entry can still skip the primary/default workspace. The related #65653 session-directory cleanup is separate from this workspace-enumeration bug.

## Best Possible Solution

Update `resolveMemoryDreamingWorkspaces` so dreaming always includes the resolved default/primary agent workspace in addition to configured `agents.list` entries, with existing workspace dedupe preserved. Add regression coverage for a config with `agents.defaults.workspace` plus several `agents.list` sub-agents that do not include `main`, and verify scheduled promotion, light/REM phase resolution, and doctor/status surfaces all report/process the combined workspace set.
