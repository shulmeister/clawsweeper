## Summary

Keep open. Current main has Slack per-target send serialization, but it does not implement the requested configurable minimum delay between outbound Slack messages, and the related PR #26732 was closed unmerged rather than becoming a canonical fix.

## Evidence

- **Current Slack send path only serializes sends:** `sendMessageSlack` builds a per-account/token/recipient/thread queue key and calls `runQueuedSlackSend`; that helper chains promises and releases the next send immediately after the prior task finishes. It does not enforce a 1000ms or configurable inter-message delay.

  - sha: [1dfa52d0711d](https://github.com/openclaw/openclaw/commit/1dfa52d0711dfc86e1c04538fe46e3d43961766f)
