## Summary

Keep open. Current `main` does not implement a Slack outbound send throttle or corresponding config/docs. Slack sends are still issued sequentially without any enforced inter-message delay, so the requested timestamp-ordering mitigation is not present.

## Evidence

- **Slack config schema has no throttle field:** `SlackAccountSchema`/`SlackConfigSchema` define delivery fields such as `textChunkLimit`, `streaming`, `mediaMaxMb`, and `replyToMode`, but no `sendThrottleMs` or similar outbound-delay option.
  - file: [src/config/zod-schema.providers-core.ts:930](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/config/zod-schema.providers-core.ts#L930)
  - command: `nl -ba src/config/zod-schema.providers-core.ts | sed -n '930,1045p'`
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **Slack docs list delivery options without throttle:** The Slack channel docs list outbound delivery controls as `textChunkLimit`, `chunkMode`, `mediaMaxMb`, and streaming settings only. The high-signal config summary also omits any outbound throttle setting.
  - file: [docs/channels/slack.md:615](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/docs/channels/slack.md#L615)
  - command: `nl -ba docs/channels/slack.md | sed -n '608,805p'`
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **Slack send path has no enforced delay:** `sendMessageSlack` posts each chunk with awaited `postSlackMessageBestEffort(...)` calls inside tight loops, with no sleep, timer, rate limiter, or timestamp-spacing logic between messages.
  - file: [extensions/slack/src/send.ts:310](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/extensions/slack/src/send.ts#L310)
  - command: `nl -ba extensions/slack/src/send.ts | sed -n '300,430p'`
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **Core outbound delivery serializes but does not throttle:** `deliverOutboundPayloadsCore` sends payloads/chunks in order, but the loops call `handler.sendText(...)` immediately for each chunk/payload and do not add channel-specific pacing. That preserves sequencing, not distinct Slack-second timestamps.
  - file: [src/infra/outbound/deliver.ts:779](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/infra/outbound/deliver.ts#L779)
