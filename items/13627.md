## Summary

Keep #13627 open. Current main still lacks the requested configurable outbound minimum delay for Slack. It serializes concurrent sends per Slack target/thread, but the next send runs immediately after the previous promise resolves, chunk/media follow-ups are posted back-to-back, and Slack config/schema/docs expose no `sendThrottleMs`, `outbound.interMessageMs`, or equivalent setting. The related PR #26732 was closed and unmerged in the provided GitHub context, so it is not a shipped fix.

## Best Possible Solution

Keep #13627 open and implement a focused Slack outbound minimum-delay feature in the bundled Slack plugin send path. Add typed config, strict schema, UI metadata, docs, and fake-timer coverage for normal sends, chunks, media follow-ups, DMs, channel messages, thread replies, Slack action sends, and slash/respond follow-ups if those remain in scope. If maintainers want a generic channel throttle instead, track that broader abstraction separately rather than treating closed unmerged PR #26732 as the fix.
