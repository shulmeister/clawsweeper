## Summary

Current `main` already implements the core of this request: peer-session discovery by agent/session, opt-in cross-agent `sessions_send`, inter-session provenance/identity markers, and visible channel/thread announce delivery for agent-to-agent flows.

## Evidence

- **Per-agent session model already exists:** Multi-agent docs define each agent as an isolated workspace/state/session scope with its own session store under `~/.openclaw/agents/<agentId>/sessions`.
  - file: [docs/concepts/multi-agent.md:8](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/docs/concepts/multi-agent.md#L8)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Official session discovery surface exists:** The CLI already exposes cross-agent session discovery via `openclaw sessions --agent <id>` and `openclaw sessions --all-agents`.
  - file: [docs/cli/sessions.md:12](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/docs/cli/sessions.md#L12)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Agent tool supports exact agent-scoped lookup:** `sessions_list` accepts an exact `agentId` filter and forwards it to the gateway `sessions.list` call, which covers the requested session-discovery mechanism.
  - file: [src/agents/tools/sessions-list-tool.ts:35](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/tools/sessions-list-tool.ts#L35)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Cross-agent access is implemented behind explicit safety gates:** Session tools can target `all` sessions, and cross-agent use is then controlled by `tools.agentToAgent`; the guard code enforces that policy for listing/sending/status.
  - file: [src/agents/tools/sessions-access.ts:171](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/tools/sessions-access.ts#L171)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Inter-session identity/provenance is implemented:** OpenClaw persists `message.provenance.kind = "inter_session"` for `sessions_send`, and replay prepends `[Inter-session message]` plus source metadata so routed agent traffic is distinguishable from end-user input.
  - file: [docs/reference/transcript-hygiene.md:80](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/docs/reference/transcript-hygiene.md#L80)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Agent-to-agent delivery back to the visible user surface exists:** `sessions_send` starts with inter-session provenance and the A2A flow then waits for the target reply, generates an announce, and sends that announce to the resolved visible channel/thread.
  - file: [src/agents/tools/sessions-send-tool.a2a.ts:64](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/agents/tools/sessions-send-tool.a2a.ts#L64)
  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)

## Risks / Open Questions

- Cross-agent messaging is intentionally opt-in: operators still need `tools.sessions.visibility="all"` and `tools.agentToAgent.enabled=true` for these flows to be available.
- I could not determine a specific shipped release tag for these exact surfaces from the available checkout history, so `fixedRelease` is left null.
