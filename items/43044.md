## Summary

PR #43044 should close as superseded by the current session-usage/freshness contract. Current main and the latest release intentionally preserve unknown totals as `?` unless `sessions.list` can supply a fresh or transcript-derived `totalTokens`; rendering missing totals as `0` would contradict that contract and the existing regression coverage. Any remaining TUI symptom from #43009 should stay tracked there and be fixed by producing a real session total, not by changing the formatter to guess zero.

## Best Possible Solution

Close this formatter-only PR. Keep the current unknown-vs-zero contract, and handle any remaining #43009 behavior through the canonical issue by ensuring `sessions.list`/session storage supplies a fresh or transcript-derived `totalTokens` when real usage is available.
