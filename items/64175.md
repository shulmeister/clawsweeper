## Summary

Keep open. Current main has an OpenRouter dynamic-model capability cache, but it does not preserve OpenRouter `supported_parameters: ["tools"]` as `compat.supportsTools`, so `supportsModelTools()` still defaults OpenRouter models to tool-capable unless a model config explicitly sets `compat.supportsTools: false`. The reported `sonar-deep-research` failure remains plausible on the current runtime path, and the related #71888 deep-research feature request does not supersede this provider/model compatibility bug.

## Best Possible Solution

Keep this issue open and fix it in the OpenRouter provider/runtime boundary: carry OpenRouter `supported_parameters.includes("tools")` into the dynamic model capability shape, set `compat.supportsTools: false` for known non-tool-capable OpenRouter models, and add regression coverage proving agent/runtime model resolution strips tools or fails early with a clear compatibility error for `openrouter/perplexity/sonar-deep-research`.
