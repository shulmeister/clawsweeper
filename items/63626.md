## Summary

Keep open. Current main still matches the reported behavior: `embedInterval` only suppresses the separate periodic embed timer, while first-run and forced QMD update paths can still spawn `qmd embed`. The related cleanup context also treats this issue as the canonical tracker: #62460 was closed as superseded by this broader embed-disable request, and #72144 left the remaining OpenClaw-side embed-disable/preflight behavior here.

## Best Possible Solution

Keep this issue open and implement a documented QMD embed-disable path in `extensions/memory-core/src/memory/qmd-manager.ts` and the public config schema. The best fix should prevent `qmd embed` from spawning for boot, first-run, manual/forced sync, session-start/post-compaction, and periodic maintenance when disabled, while preserving `qmd update`/BM25 refresh behavior. Add focused QMD manager tests and update memory config docs/help.
