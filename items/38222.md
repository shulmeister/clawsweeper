## Summary

Keep open. Current main still does not enforce length limits on plugin hook prompt mutations, and PR #38222 remains a plausible core hardening change rather than a duplicate, superseded PR, or ClawHub-only item. The supplied PR context shows the author addressed the early review comments in later commits, but that implementation is not present on main and the current shared resolver, CLI, and harness paths still compose hook text without a cap.

## Best Possible Solution

Keep the PR open and refresh it against current main. The best implementation is a shared post-merge clamp for all prompt-visible hook result fields, applied after phase hooks, legacy hooks, and phase-plus-legacy joins, then reused by embedded, CLI, and harness prompt preparation with warning logs, public plugin-hook documentation for the limit, and regression tests for oversized, combined, legacy, and normal-length cases.
