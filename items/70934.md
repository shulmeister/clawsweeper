## Summary

Current main still does not implement the requested value-first first-run onboarding. QuickStart and macOS onboarding reduce setup friction, but the primary flows still configure auth/model, workspace, gateway, channels/search/skills, permissions, and bootstrap/persona handoff before a new user can choose and run a useful example task/template.

## Best Possible Solution

Keep this open for maintainer UX/product triage. A good implementation would add a first-run value-first path with example task templates and one quick useful runnable task before optional identity, persona, reply-style, channel, search, skill, and other customization steps, likely touching the CLI wizard, wizard finalization, macOS onboarding chat/page order, public onboarding docs, and focused wizard/onboarding tests.
