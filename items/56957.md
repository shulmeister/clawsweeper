## Summary

Keep open. Current `main` still implements the reported behavior: WhatsApp group sender auth falls back from `groupAllowFrom` to DM `allowFrom`, and the core group-policy path explicitly admits groups when sender filtering exists but no `groups` map is configured. That behavior is reinforced by tests and docs, and the issue discussion shows an active follow-up fix PR (#68446), so there is not strong evidence for auto-closing.
