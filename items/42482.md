## Summary

Keep #42482 open. Current main still lacks both central pieces of the PR: the cron timeout is still armed before `executeJobCore` starts isolated execution, and the isolated fallback runner still passes the same parent abort signal into every embedded fallback attempt. The PR discussion and related reports describe real cron timeout/fallback failures with field confirmation, and I did not find an implemented equivalent or a merged canonical replacement on main.

## Best Possible Solution

Keep the PR or a canonical replacement open. The best path is to rebase/narrow the fix onto current main: add per-attempt abort-controller isolation in `src/cron/isolated-agent/run-executor.ts`, defer or split cron execution timeout accounting in `src/cron/service/timer.ts`, and add regression tests for queued isolated execution plus fallback attempts receiving fresh, non-aborted signals.
