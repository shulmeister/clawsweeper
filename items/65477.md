## Summary

Keep open. Current main still formats per-response `/usage` footers as token/cache/cost/session text and does not prepend or otherwise include the active provider/model. Existing `{model}`/`{modelFull}` response-prefix support and `/status` model display are separate surfaces, and related usage-footer PRs/issues remain open or unshipped.

## Best Possible Solution

Keep this issue open unless maintainers explicitly consolidate it into the broader usage-footer tracker. The likely fix is to pass the resolved `providerUsed`/`modelUsed` label into `formatResponseUsageLine`, format it with the existing `formatProviderModelRef` helper, prepend it before `Usage:` for the intended `/usage` modes, preserve current token/full cost and session-key behavior, and add focused formatter and runner tests.
