## Summary

Keep #14317 open. Current main still does not implement Slack `groupPolicy: "private"`: config types/schema/metadata only accept `open`, `disabled`, and `allowlist`; Slack message and slash routing evaluate only those generic policies and do not use Slack channel privacy; docs/tests/security guidance still describe the same three-value policy model. The existing maintainer-visible Codex comment by steipete keeping the issue open remains accurate on current main.

## Best Possible Solution

Keep this issue open. Implement `groupPolicy: "private"` in the bundled Slack plugin and its aligned config/security contract: accept the Slack-scoped value in schema/types/metadata, carry or fetch Slack `is_private` channel metadata for message and slash routing, allow private channels where the bot is present, preserve explicit public-channel allowlist/deny behavior, update security audit/fix guidance so `private` is not treated like `open`, and add schema, runtime, docs, and tests for private channels, public allowlist fallback, slash commands, and Slack Connect caveats.
