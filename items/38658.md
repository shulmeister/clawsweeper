## Summary

Keep PR #38658 open. Current main still implements Slack group/channel history as pending-only: skipped mention-gated messages are recorded, but processed room messages are cleared after dispatch, including when `requireMention=false`. The PR’s requested behavior is therefore not implemented or superseded on main, and the provided discussion shows the branch contains substantive Slack-specific behavior work plus follow-up fixes for the no-reply path.

## Best Possible Solution

Keep the PR open for maintainer review. If the desired product behavior is richer always-on Slack channel context, implement it in `extensions/slack/src/monitor/message-handler` by carrying `requireMention` into dispatch, recording processed room messages when `requireMention=false` in both reply and no-reply paths, updating the Slack/group history tests, and updating public docs to clarify that Slack always-on channels retain a bounded recent-message window rather than pending-only history.
