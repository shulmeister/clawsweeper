## Summary

Keep PR #38658 open. Current main still uses the pending-only Slack channel-history model: processed room messages are not retained when `requireMention=false`, dispatch clears room history after processed turns, tests assert no processed-room-history injection, and public docs still describe pending-only group history. The PR remains a focused plausible fix, but it changes Slack/channel prompt-context semantics and still needs maintainer judgment plus resolution of the ordering concern raised in review.

## Best Possible Solution

Keep the PR open for maintainer review. If always-on Slack channel context is the desired product behavior, update `extensions/slack/src/monitor/message-handler/*` to carry resolved `requireMention` into dispatch, retain processed room messages for `requireMention=false` in both reply and no-reply paths, preserve chronological order, add prompt/history regression coverage, and update the public group-message/history docs. If maintainers prefer pending-only history as the contract, close the PR with that explicit rationale.
