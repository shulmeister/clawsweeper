## Summary

Keep PR #38658 open. Current main still clears Slack room history after processed room messages instead of retaining a bounded sliding window when `requireMention=false`, and existing tests/docs still encode the pending-only history model. The PR is a focused plausible fix, but it is not merged, not implemented on main, and still needs maintainer review for the intended Slack history semantics and the review-thread ordering concern.

## Best Possible Solution

Keep the PR open for maintainer review. If always-on Slack channel context is desired, update the Slack plugin to carry the resolved `requireMention` policy into dispatch, retain processed room messages for `requireMention=false` in both reply and no-reply paths while preserving chronological order, add regression coverage for the prompt body and history window, and update the public message/group history docs. If maintainers prefer pending-only history as the product contract, close or redirect the PR with that explicit rationale.
