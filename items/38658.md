## Summary

Keep open. Current `main` still clears Slack room history after each processed room turn, and the existing Slack test suite explicitly expects `requireMention=false` channel messages to arrive without prior channel context. The PR addresses behavior that is not implemented on `main`.

## Evidence

- **Current dispatch still wipes room history after every turn:** `extensions/slack/src/monitor/message-handler/dispatch.ts:1003` clears `ctx.channelHistories` when no reply is delivered, and `extensions/slack/src/monitor/message-handler/dispatch.ts:1048` clears it again after a delivered reply. There is no `requireMention` branch or history-recording path on `main`.
  - file: [extensions/slack/src/monitor/message-handler/dispatch.ts:1003](https://github.com/openclaw/openclaw/blob/ac063568d354c14e0a0a43bcffb9725bb3fca651/extensions/slack/src/monitor/message-handler/dispatch.ts#L1003)

  - sha: [ac063568d354](https://github.com/openclaw/openclaw/commit/ac063568d354c14e0a0a43bcffb9725bb3fca651)
- **Prepared Slack messages still build context from the history map, so clearing it removes prior channel context:** `extensions/slack/src/monitor/message-handler/prepare.ts:595` builds `Body` from `ctx.channelHistories`, and `extensions/slack/src/monitor/message-handler/prepare.ts:652` exposes the same map as `InboundHistory`. The only pre-dispatch history recording in this flow is the no-mention skip path at `extensions/slack/src/monitor/message-handler/prepare.ts:455`, which does not help `requireMention=false` replies if dispatch clears history afterward.
