## Summary

Keep open. Current main has generic event timestamps and opt-in raw stream timestamps, but it does not expose a supported TTFT metric alongside model usage/token accounting, nor does the embedded-run callback contract carry a first-chunk timestamp as requested.

## Best Possible Solution

Keep the issue open and implement this as an additive telemetry/metadata contract, likely in the embedded runner/subscription path: capture model-request start and first model stream chunk time, expose a TTFT ms field in run/session usage metadata or a documented agent event, and add focused tests around streaming, non-streaming, retry/fallback, and reasoning-only first chunks.
