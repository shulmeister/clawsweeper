## Summary

Current main already implements the requested TTFT-style model inference timing. The pi-agent stream function is wrapped for every embedded attempt, records elapsed time to the first streamed response chunk as `timeToFirstByteMs`, and exposes that value through diagnostic events, plugin model-call hooks, diagnostic snapshots, and the `diagnostics-otel` metric `openclaw.model_call.time_to_first_byte_ms`.

## Best Possible Solution

Close as implemented on main. Keep the shipped `timeToFirstByteMs`/`openclaw.model_call.time_to_first_byte_ms` diagnostic surface as the canonical TTFT-equivalent model-call timing, instead of adding a second parallel TTFT field unless maintainers later want a naming alias.
