## Summary

Keep this issue open. Current main still has no `memorySearch.ignorePaths` or equivalent exclusion contract, and both builtin and QMD memory search still include `memory/dreaming/**` through their default memory-tree indexing/watching paths. The related dreaming fixes address promotion/self-ingestion contamination, not the search index or watcher scope requested here.

## Best Possible Solution

Keep the issue open and implement a unified memory-search exclusion contract, likely `memorySearch.ignorePaths` or a clearly named equivalent. It should be added to schema, types, generated config help/docs, config resolution, builtin `listMemoryFiles`, builtin chokidar watching, QMD collection/update/watch scope, and reindex scope hashing, with focused tests for `memory/dreaming/**`, dependency directories, `extraPaths`, and builtin/QMD parity.
