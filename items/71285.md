## Summary

Keep #71285 open. Current main still has no `memorySearch.ignorePaths` config, the builtin indexer recursively indexes Markdown under `memory/` including `memory/dreaming/`, and both builtin and QMD watcher ignore lists only skip dependency/cache-style directories, not `dreaming`. The related merged #66852 addresses dreaming promotion/self-ingestion, while the unmerged #6362 and open #68774 do not ship the requested search-index/watcher exclusion behavior.

## Best Possible Solution

Keep the issue open and implement a first-class memory-search exclusion path. The strongest repo-local path is to add `memorySearch.ignorePaths` to config schema/types/help/docs, resolve it per agent, and route the same normalized matcher through builtin `listMemoryFiles`, builtin chokidar `ignored`, QMD collection/watch handling, and tests that prove `memory/dreaming/**` remains searchable by default but is excluded from indexing and watcher activity when configured.

## Risks / Open Questions

- Implementing config glob ignores needs one shared matcher across builtin indexing and chokidar/QMD watch behavior; the prior #6362 discussion explicitly identified watcher/indexer semantic drift as a concern.
- There is a product choice between hardcoding `memory/dreaming/` as non-searchable and adding a general opt-in `ignorePaths`; the issue argues for opt-in config, but maintainers should decide the default because docs currently present `memory/dreaming/` as human-readable output.
