## Summary

Current main addresses the reported QA-lab abort-catalog test failure: the test now waits for the marker file to contain the expected `OPENCLAW_CODEX_DISCOVERY_LIVE=0` value instead of treating file existence or an immediate read as complete content, and the lab stop path aborts and awaits the in-flight runner model catalog. The linked #70669 discussion correctly identified this as a marker-content race; current source now covers the observable CI failure from #70659.

## Best Possible Solution

Close #70659 as implemented on current main. Keep the current QA-lab test behavior that waits for complete marker content and aborts the catalog through the lab stop path; handle any future platform-specific Windows termination flake as a narrower follow-up if it reappears on supported lanes.
