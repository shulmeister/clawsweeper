## Summary

Keep open. Current main still does not pre-create `/home/node/.openclaw` in the runtime image before switching to `USER node`, so the direct Docker Compose named-volume startup crash remains plausible. The setup script has a root-container `chown` workaround for its managed flow, but the reported repro uses `ghcr.io/openclaw/openclaw:latest` with a plain named volume, and the linked fix PR #63959 is still open/unmerged in the provided context.

## Best Possible Solution

Keep #61279 open until a narrow Dockerfile fix lands and ships: pre-create `/home/node/.openclaw` with `node:node` ownership before `USER node`, add a Dockerfile/source assertion or focused regression check, and update Docker docs if existing root-owned volumes still need manual recovery. PR #63959 appears to be the relevant starting point.
