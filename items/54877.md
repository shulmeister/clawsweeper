## Summary

Current `main` still has the reported doctor/cleanup mismatch: the doctor warning suggests a missing-transcript cleanup preview without `--fix-missing`, while the cleanup preview only counts missing transcripts when that flag is set. The linked fix PRs in the provided context are closed unmerged, so there is no implemented-on-main basis for closing.

## Best Possible Solution

Keep this issue open and land a narrow commands fix: update `src/commands/doctor-state-integrity.ts` so the missing-transcript `Preview cleanup impact` command includes `--dry-run --fix-missing`, then strengthen `src/commands/doctor-state-integrity.test.ts` to assert that exact preview hint. The existing `sessions cleanup` flag-gated behavior can remain unchanged because the CLI already documents the correct preview mode.
