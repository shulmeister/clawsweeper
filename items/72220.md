## Summary

Keep #72220 open. Current main still has the unguarded build-time runtime-deps temp cleanup path this PR targets: temp dirs are created without owner metadata, and `removeStaleRuntimeDepsTempDirs` deletes every `.openclaw-runtime-deps-*` entry before staging. The existing runtime install lock is adjacent but does not cover this postbuild staging cleanup race.

## Best Possible Solution

Keep PR #72220 open for maintainer review. The best path is to land this PR or an equivalent fix in `scripts/stage-bundled-plugin-runtime-deps.mjs` that marks active build/postbuild staging temp directories with owner metadata, skips cleanup for live owners, adds the concurrent-cleanup regression test, and either fixes or explicitly accepts the backup-directory owner gap.
