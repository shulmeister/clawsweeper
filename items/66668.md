## Summary

Current main does not yet solve the reported uninstall gap. Local path installs are still persisted with source: "path" and a separate installPath, but uninstall treats every source: "path" install as linked and skips directory deletion. Manually copied plugin directories are also explicitly rejected by the CLI when they have no config/install record. The related PRs in the provided issue timeline were closed unmerged, and the later main commit only fixed managed non-path install targets, not the path-install distinction this issue reports.

## Best Possible Solution

Keep the issue open. The likely fix is to distinguish true linked installs from copied path installs: preserve source directories only when `source === "path"` and `installPath === sourcePath`, but delete a tracked `installPath` when it differs from `sourcePath` and resolves safely under the active state-dir extensions root. Add regression tests for `source: "path"` with `installPath != sourcePath`, CLI preview behavior, and the manually copied/unmanaged case; either support safe unmanaged directory removal explicitly or document that manual copies must be removed manually.
