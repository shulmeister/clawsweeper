## Summary

Keep open. Current `main` still hardcodes OpenRouter Anthropic system-prompt cache markers to short-lived `ephemeral` cache control and does not route `cacheRetention` into that wrapper, so the PR’s reported `none`/`long` bug is still present in the checked-out code.

## Evidence

- **Cache-retention resolution still excludes OpenRouter Anthropic:** `resolveCacheRetention(...)` only returns the default `"short"` for `family === "anthropic-direct"`, and `resolveAnthropicCacheRetentionFamily(...)` only classifies direct Anthropic, Bedrock, and custom `anthropic-messages` providers. OpenRouter Anthropic is not included, so explicit/default OpenRouter retention is not resolved here.
  - file: [src/agents/pi-embedded-runner/prompt-cache-retention.ts:17](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/agents/pi-embedded-runner/prompt-cache-retention.ts#L17)
  - command: `nl -ba src/agents/pi-embedded-runner/prompt-cache-retention.ts | sed -n '1,120p'; nl -ba src/agents/pi-embedded-runner/anthropic-family-cache-semantics.ts | sed -n '69,106p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **OpenRouter wrapper still forces short ephemeral cache markers:** `createOpenRouterSystemCacheWrapper(...)` takes only `baseStreamFn` and unconditionally calls `applyAnthropicEphemeralCacheControlMarkers(...)`. That helper writes `{ type: "ephemeral" }` and never accepts `cacheRetention`, never emits `ttl: "1h"`, and never disables cache markers for `"none"`.
  - file: [src/agents/pi-embedded-runner/proxy-stream-wrappers.ts:49](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/agents/pi-embedded-runner/proxy-stream-wrappers.ts#L49)

- **Extra-params pipeline does not thread retention into the OpenRouter cache wrapper:** `applyPostPluginStreamWrappers(...)` still calls `createOpenRouterSystemCacheWrapper(ctx.agent.streamFn)` with no retention argument, so even if `cacheRetention` is resolved earlier it cannot affect OpenRouter Anthropic payload patching.
  - file: [src/agents/pi-embedded-runner/extra-params.ts:391](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/agents/pi-embedded-runner/extra-params.ts#L391)
