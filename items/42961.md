## Summary

Keep PR #42961 open. Current main still does not respect `cacheRetention` for OpenRouter Anthropic cache markers: the wrapper is always installed for verified OpenRouter Anthropic routes and the marker helper always writes the short `{ type: "ephemeral" }` form. The PR addresses a real remaining gap, though it likely needs rebasing onto the current shared prompt-cache policy code.

## Best Possible Solution

Keep the PR or a canonical successor open and adapt it to current main: resolve `cacheRetention` for verified `openrouter/anthropic/*` routes, preserve the existing default short marker, suppress markers for `none`, emit `{ type: "ephemeral", ttl: "1h" }` for `long`, and add focused OpenRouter tests plus docs for the `cacheRetention` behavior.
