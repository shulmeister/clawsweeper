## Summary

Current `main` does not implement the proposed intermediate task states. Task status types, status formatting, summaries, and runtime projection still only model `queued` and `running` as active states, and the task listener does not consume approval events. The PR remains relevant, but against current `main` it would also need broader active-state plumbing updates across executor, audit, and maintenance paths before a maintainer could confidently land it.
