## Summary

Keep #40534 open. Current main has a partial later mitigation for Telegram/Discord text-only cron announces via `finalAssistantVisibleText`, but the PR's core helper-level `isReasoning` pipeline is still absent: reasoning payloads are created, then the payload map drops the flag, and cron selection/error-recovery helpers still only special-case `isError`. The PR discussion already identified and addressed these exact gaps on the PR branch, so the work is not obsolete; it needs rebasing/review against current main.

## Best Possible Solution

Keep the PR open for maintainer review or ask the author to rebase it. The best fix is still to propagate `isReasoning` from `buildEmbeddedRunPayloads`, include it in cron delivery payload typing, filter it in summary/text/deliverable selection and error-recovery detection, and add focused regression tests for mixed reasoning/final text, reasoning-only, and post-error reasoning-only payloads while preserving the current `finalAssistantVisibleText` policy for Telegram/Discord.
