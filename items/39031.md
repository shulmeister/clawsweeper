## Summary

Keep #39031 open. Current main still does not forward legacy iMessage tapback reactions to the agent: the iMessage monitor only subscribes to ordinary message notifications, the parsed payload and config schema do not model reaction metadata, and the dispatch path only builds text-message inbound context. The related GitHub context shows #39322 is still the open implementation PR, #60677 is only a filtering fix, and #60446 was closed in favor of this canonical issue.

## Best Possible Solution

Keep #39031 open as the canonical legacy iMessage tapback-to-agent feature request. The best path is to review, rebase, or supersede #39322 with a typed implementation that subscribes to reaction notifications, parses target/reaction metadata, routes tapbacks as reaction/system events rather than ordinary text, defines access-control and dedupe behavior, and updates config, docs, and tests. If legacy imsg reaction forwarding is intentionally out of scope, document that explicitly and point users to BlueBubbles for richer iMessage reaction support.
