## Summary

Keep open. Current main still does not forward legacy iMessage `imsg rpc` tapback reactions to the agent. BlueBubbles already supports reaction system events and is the recommended iMessage path, but #39031 specifically describes the legacy `imsg` channel/plugin gap, and the related #60274/#60677 work only covers filtering tapbacks that are misclassified as normal messages.

## Best Possible Solution

Keep #39031 open until maintainers either implement legacy iMessage `imsg rpc` tapback forwarding as a reaction/system event, close it through a canonical merged PR such as an eventual tapback-event implementation, or explicitly document that tapback event support is BlueBubbles-only and legacy `imsg` will not gain parity.
