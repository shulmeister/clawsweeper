## Summary

Keep #39031 open. Current main still does not forward legacy iMessage `imsg rpc` tapback reactions to the agent: the iMessage payload, parser, monitor subscription, inbound decision shape, config schema, docs, and tests remain normal-message/attachment focused. BlueBubbles already has reaction system-event support, but that does not implement the requested legacy `imsg` behavior. The provided related context shows #60446 was closed back to this issue and #39322 remains the relevant open implementation candidate.

## Best Possible Solution

Keep #39031 open as the canonical legacy iMessage `imsg rpc` tapback-to-agent feature request. Maintainers should review or supersede #39322 with a typed reaction/system-event implementation covering subscription, payload parsing, access control, debounce/dedupe semantics, config/docs, and regression tests, or make an explicit product decision that reaction event support is BlueBubbles-only and document legacy `imsg` parity as out of scope.
