## Summary

Keep #39031 open. Current main still does not forward legacy iMessage `imsg rpc` tapback reactions to the agent. BlueBubbles does support reaction system events and is the recommended iMessage path, and #60446 was closed as a duplicate of this older canonical issue, but the requested legacy `imsg` channel behavior remains missing. Related #60274/#60677 only cover filtering tapbacks misclassified as normal messages, and #39322 remains a potentially relevant open implementation PR.

## Best Possible Solution

Keep #39031 open as the canonical legacy iMessage tapback-to-agent feature request. The best maintainer path is to either implement `imsg rpc` reaction forwarding as a reaction/system event with parser, config, access-control, debounce, and tests, or explicitly decide and document that tapback event support is BlueBubbles-only and close the legacy parity request with that product decision. The open #39322 PR is the relevant implementation candidate to review or supersede.
