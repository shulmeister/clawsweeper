## Summary

Keep open. Current main mitigates sqlite-vec failures with degraded in-process cosine search, but it still relies on node:sqlite loadable-extension support for native sqlite-vec acceleration and does not show a macOS-specific fix or documented Homebrew workaround. The issue comment’s `/opt/homebrew/bin` PATH workaround is useful evidence for QMD/Bun environments, but it is not proof that OpenClaw core now loads sqlite-vec on macOS.

## Best Possible Solution

Keep the issue open until maintainers validate current main on macOS. If node:sqlite with allowExtension now loads sqlite-vec on supported macOS installs, add source/test or doctor/status evidence and close as implemented. If it still fails, fix the macOS vector-load path or document the Homebrew/QMD PATH workaround in the memory docs while keeping the in-process cosine fallback as degraded behavior.
