## Summary

Keep #47311 open. Current main has browser SpeechRecognition dictation and a separate OpenAI Realtime Talk WebRTC path, but it does not implement the requested WebChat MediaRecorder/server-side STT path through Gateway and `tools.media.audio`. The issue comment is directionally accurate: shared audio transcription exists for inbound channel audio/voice notes, but WebChat still cannot send audio blobs through that pipeline.
