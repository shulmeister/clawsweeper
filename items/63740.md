## Summary

Keep open. Current main does not contain the corrupted `tail -n 200` text in the CLI source, and later packaging work prunes/verifies stale or missing `dist` files, but the reported failure is an in-place corruption of an installed `dist/run-main-*.js` chunk. The current inventory checks are path-based and do not prove that expected chunk contents are protected, repaired, or that the root cause cannot recur.

## Best Possible Solution

Keep this open as a real packaging/runtime integrity investigation. A maintainer-quality fix should reproduce or rule out the 2026.4.5 WSL/global-install path, audit update/postinstall/logging/restart flows for writes under the installed package root, and either prove OpenClaw never writes into core `dist` at runtime or add a shipped-artifact content-integrity/startup-repair mechanism that catches corrupted expected chunks with clear reinstall guidance.

## Evidence

- **Reporter evidence is concrete:** The provided issue body reports OpenClaw 2026.4.5 crashing during ESM compilation because an installed global `dist/run-main-8bQWSeWd.js` chunk contained `tail -n 200 ... logawaitprogram.parseAsync(parseArgv);`. The follow-up comment explicitly framed this as possible self-corruption and asked maintainers to investigate runtime writes to `dist`/`node_modules` and unsafe shell redirection.
