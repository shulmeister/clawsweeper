## Summary

Current `main` does not appear to fix this regression. `message send` now routes through plugin actions first, but the Feishu `send` action only turns `presentation` into a card; a raw JSON string in `message` still goes down the plain-text Feishu post path.

## Evidence

- **`message send` prefers channel plugin actions on current main:** `executeSendAction()` calls `tryHandleWithPluginAction()` before any core fallback, so the old 'core bypasses plugin action entirely' theory no longer matches current main.
  - file: [src/infra/outbound/outbound-send-service.ts:143](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/infra/outbound/outbound-send-service.ts#L143)
  - command: `nl -ba src/infra/outbound/outbound-send-service.ts | sed -n '71,202p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Feishu plugin `send` action only builds cards from `presentation`:** In the Feishu action handler, `presentation` is normalized into a card and sent with `sendCardFeishu()`. Otherwise, plain `message`/`text` is sent with `sendMessageFeishu()`. There is no raw-card-JSON detection on `message`.
  - file: [extensions/feishu/src/channel.ts:695](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/channel.ts#L695)

  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Plain Feishu send path always emits `msgType: "post"`:** `sendMessageFeishu()` converts the provided text into a Feishu post payload, so a JSON string passed as `message` would be wrapped as text rather than sent as `interactive`.
  - file: [extensions/feishu/src/send.ts:432](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/send.ts#L432)
  - command: `nl -ba extensions/feishu/src/send.ts | sed -n '432,510p'`

- **`message` is not JSON-parsed by the shared message tool:** `runMessageAction()` parses JSON for `presentation`, `delivery`, and `interactive`, but not for `message`, so the raw card JSON remains an ordinary string unless a channel action adds its own detection.
