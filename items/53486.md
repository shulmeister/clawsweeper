## Summary

Keep open. Current main still sends a raw Feishu interactive-card JSON string supplied via the `message` parameter down the Feishu text path, not the interactive-card path. Main now supports semantic `presentation` cards and proactive sends do reach the Feishu plugin, but the reported compatibility behavior is not implemented and the adjacent closed Feishu reports do not canonically cover this exact regression.

## Best Possible Solution

Keep this issue open for Feishu-owned maintainer handling. The narrow code fix would be a conservative compatibility detector in `extensions/feishu/src/channel.ts` that recognizes valid Feishu interactive-card JSON in the `message` string and routes it to `sendCardFeishu`, with a regression test for proactive send. The alternative best resolution is an explicit maintainer decision and docs update that raw provider-native card JSON is unsupported and `presentation` is the required proactive-card path.
