## Summary

Keep #39543 open. Current main has partial PDF capability through the `pdf` tool and file/PDF extraction paths, but the requested local PDF upload-and-summarize workflow is still not implemented in the Control UI/WebChat attachment path.

## Best Possible Solution

Keep the issue open and scope it to implementing local PDF upload in the Control UI/WebChat path: allow PDF selection/drop, preserve document metadata in the UI attachment model, teach Gateway chat attachments to validate and offload PDFs as managed media refs, and ensure the agent can route those refs to the existing `pdf` tool or PDF extraction path for summarization.
