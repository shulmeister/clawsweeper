## Summary

Keep #39543 open. Current main has a capable PDF analysis tool for paths, URLs, and managed inbound refs, but the requested local PDF upload-and-summarize workflow is still missing from the Control UI and Gateway chat attachment path.

## Best Possible Solution

Keep the issue open. The best implementation is to add first-class document/PDF attachments to the Control UI and Gateway chat pipeline: accept `application/pdf`, preserve filename/MIME/document metadata, render a pending document attachment state, serialize/offload PDFs as managed inbound media refs, route those refs to the existing `pdf` tool or PDF-capable media understanding path, and add focused UI, Gateway, PDF-tool, and docs coverage.
