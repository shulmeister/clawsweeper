## Summary

Keep #39543 open. Current main has a backend `pdf` tool for path/URL/managed-media PDF refs, but the user-facing local PDF upload workflow is still not implemented: Control UI only accepts image attachments, and Gateway chat attachment parsing drops non-image/PDF payloads before they can become PDF refs for summarization.

## Best Possible Solution

Keep the issue open and scope the fix to the Control UI/WebChat local PDF path: allow PDF selection/drop, carry file name/MIME/document metadata in `ChatAttachment`, render a document attachment state instead of image thumbnails, serialize PDFs distinctly, update Gateway validation/offload to preserve `application/pdf` as managed inbound media refs, and route those refs to the existing `pdf` tool with focused UI, Gateway, and PDF-tool integration coverage.
