## Summary

Keep open. Current main has session-memory and QMD session-export infrastructure, but the reporter's config still does not enable QMD transcript export, and same-agent gateway-to-DM session hits remain hidden by default session visibility.

## Best Possible Solution

Keep this issue open. The conservative maintainer path is to either bridge `agents.defaults.memorySearch.experimental.sessionMemory` plus `sources: ["sessions"]` into QMD session export when `memory.backend="qmd"`, or document and diagnose that QMD transcript recall requires `memory.qmd.sessions.enabled: true` and `tools.sessions.visibility: "agent"` or `"all"` for same-agent cross-channel gateway/DM recall. Add a regression test with a gateway-style `agent:<id>:paperclip:issue:<uuid>` session and a DM requester such as `agent:<id>:whatsapp:direct:*`.
