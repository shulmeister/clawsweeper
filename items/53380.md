## Summary

Current `main` only partially overlaps this request. Control UI has browser-local user avatar/name support, and the gateway exposes agent identity `emoji`/`avatar`, but the Control UI still does not render agent emoji/text avatars in chat bubbles and does not render any avatar/emoji in the sessions list. The request is not implemented well enough to close.
