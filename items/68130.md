## Summary

Keep open. Current `main` does not implement the requested log-level change, and the submitted PR has a blocker: it calls `params.logger.debug(...)` even though the `dreaming-narrative` logger contract and its tests only require `info`/`warn`/`error`.

- The submitted patch is unsafe as written because `params.logger.debug(...)` is not guaranteed by the local logger contract.
- The proposed suppression matches the literal string `missing scope: operator.admin`, which is brittle if gateway error text changes.
- The PR bundles unrelated fixes outside memory-core, so even a valid cleanup change would likely need splitting or maintainer rework before merge.
