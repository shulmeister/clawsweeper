## Summary

Keep open. Current main still matches the reported cleanup bug: request-scoped subagent startup falls back and returns null, but the finally block still calls deleteSession and logs every cleanup error. Existing tests prove the fallback path still attempts cleanup, and the current request-scoped cleanup test only verifies the sweep does not throw, not that the spurious warning is suppressed.

## Best Possible Solution

Patch extensions/memory-core/src/dreaming-narrative.ts so cleanup is skipped when startNarrativeRunOrFallback returned null, and also suppress RequestScopedSubagentRuntimeError in the cleanup catch for symmetry with startup fallback. Add a focused regression test that a request-scoped fallback writes the local diary entry without calling deleteSession or emitting the cleanup warning. Keep the related operator.admin cleanup reports separate unless the chosen fix also resolves that gateway-scoped deletion path.
