## Summary

Keep #39722 open. Current main still has the reported `/sessions` title precedence and sanitization gap: the TUI prefers `derivedTitle` over `displayName`, gateway session rows compute an outer `displayName` but derive titles without passing it, and transcript title extraction still strips only inline directive tags rather than gateway inbound metadata blocks. Related PR #39790 is relevant proposed work, but the provided GitHub context shows it is open and unmerged.

## Best Possible Solution

Keep #39722 open until #39790 or an equivalent maintainer-reviewed fix lands. The fix should pass the computed outer session `displayName` into derived-title construction, strip inbound metadata before first-user-message and last-message-preview extraction, preserve subject/message/session-id fallbacks, and add regression tests for external display-name precedence plus metadata stripping in session-title extraction.
