## Summary

Keep #39722 open. Current main still matches the reported TUI /sessions failure path: title extraction can return the gateway-injected inbound metadata block, the computed outer row displayName is not passed into deriveSessionTitle, and the TUI renders derivedTitle before displayName. The linked fix PR #39790 is still only a proposed fix in the provided context, not shipped on main.

## Best Possible Solution

Keep #39722 open until PR #39790 or an equivalent maintainer-approved fix lands. The fix should strip inbound metadata before session title and last-message preview extraction, make the computed gateway row displayName take precedence during title derivation, preserve subject/message/session-id fallbacks, and add regression coverage for both metadata-envelope stripping and external displayName precedence.
