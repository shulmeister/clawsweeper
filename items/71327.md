## Summary

Keep PR #71327 open. Current main still has the LINE schema gap described by the PR: `dmPolicy: "open"` is accepted without an explicit `allowFrom: ["*"]`, while the LINE runtime treats open DMs as broadly allowed. The PR is a focused, recent bug/security-hardening change and the provided latest patch includes targeted schema tests, including the Greptile-noted missing cases.

## Best Possible Solution

Keep the PR open for maintainer review and normal CI. The best code direction is to add the shared `requireOpenAllowFrom` guard to both top-level and account-level LINE config schemas, keep the focused regression tests from the PR, and add release-note/docs coverage if maintainers consider the validation hardening user-visible.
