## Summary

PR #69391 is a docs-only change to the bundled `coding-agent` skill that promotes a self-maintained third-party rollback tool, cowback, as the concrete recovery example. Current main keeps optional skills/plugins and plugin promotion outside core via ClawHub, and the existing skill/plugin surfaces are sufficient for distributing a cowback workflow without a core docs change. The provided review discussion also raised a correctness concern that the sample may not guarantee a baseline snapshot before the first destructive write.

## Best Possible Solution

Close this PR as a ClawHub/external-skill candidate. The cowback workflow can be published as a ClawHub skill or plugin-owned skill, where the tool-specific setup, disclosure, and safety details can live. If maintainers want bundled `coding-agent` safety guidance, open a separate vendor-neutral core docs change that documents baseline-before-spawn snapshot semantics without promoting a self-maintained third-party tool.
