## Summary

Keep #72181 open. Current `main` still does not implement BlueBubbles trainer/supervised reply mode: the BlueBubbles config schema and account type have no `trainerMode` or `trainerNotifyNumber`, and allowed inbound DMs still flow into the normal reply pipeline that sends generated chunks to the original BlueBubbles target. The existing ClawSweeper discussion already identified the same gap, and the current checkout still matches that behavior.

## Best Possible Solution

Keep #72181 open for BlueBubbles owner/product review. A good implementation would stay under `extensions/bluebubbles`, add typed/schema-backed `trainerMode` and `trainerNotifyNumber` config, intercept allowed inbound DMs before `createChannelReplyPipeline` dispatches to the original BlueBubbles target, route owner approval/custom-reply commands through the existing BlueBubbles send path, enforce pending-message TTLs, and add focused monitor/config tests plus user-facing BlueBubbles docs.
