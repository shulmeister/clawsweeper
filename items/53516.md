## Summary

The requested smart-routing feature is better handled as a plugin/ClawHub policy layer, not OpenClaw core: current main already exposes a public pre-model-resolution hook for deterministic model overrides, and project vision says optional capability should usually ship as plugins.

## Close Comment

Closing as a `clawhub` item after Codex review. Current `main` already exposes the public seam needed for this: `before_model_resolve` lets a plugin deterministically override provider/model before model resolution (`src/plugins/hooks.ts:514`, `docs/concepts/agent-loop.md:93`, `src/plugins/hooks.model-override-wiring.test.ts:5`). OpenClaw’s project direction is to keep optional capability out of core and ship it as plugins when possible (`VISION.md:54`), and the plugin docs explicitly point new extensions to ClawHub/npm rather than adding them to this repo (`docs/plugins/building-plugins.md:16`). Because this request is a routing policy layer with provider/model override behavior, it fits the existing plugin hook surface better than a new core `smartRouting` config block.
