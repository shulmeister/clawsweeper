## Summary

Keep open. Current main has not implemented this PR's requested auth-profile cooldown behavior, but the PR as written also conflicts with current HTTP 400 classification semantics and should not be auto-closed as implemented or merged without maintainer review.

## Best Possible Solution

Keep the PR open for maintainer review, but request a narrower fix: skip persistent auth-profile cooldown only for confidently classified `format` payload/schema errors, avoid a blanket raw HTTP 400 bypass, and add regression coverage for the runner path and assistant-side call sites if raw error text/status propagation remains part of the design. The changelog entry would also need normal placement/attribution cleanup before merge.
