## Summary

Keep open. Current `main` still contains the explicit `gateway.bind="loopback"` fallback to `0.0.0.0` when the preflight bind probe fails, and the runtime still throws the exact non-loopback startup error reported in the issue. The discussion-linked fix exists in PR `#65635` / commit `cc676bc`, but that change is not present in current `main` or the latest release tag `v2026.4.23`.
