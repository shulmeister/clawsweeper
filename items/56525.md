## Summary

Keep open: current `main` does not expose a first-class persistent browser viewport config. The accepted `browser` config keys stop at `extraArgs`, while the user-facing docs still treat viewport changes as runtime `resize`/`set viewport` actions and note that temporary emulation overrides are cleared when the control session stops.

## Evidence
