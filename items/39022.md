## Summary

Keep #39022 open. Current main still treats `/model` as a persistent session-scoped model override and does not implement `/model <name> --turns N`, per-model turn-limit config, warning thresholds, remaining-turn state, or automatic reversion to the default model. The existing ClawSweeper comment on the issue reached the same conclusion, and the current checkout still matches that behavior.

## Best Possible Solution

Keep the issue open for maintainer product judgment or implementation. A complete implementation likely belongs in core `/model` session behavior: parse `--turns`, persist per-session temporary model budget and revert target, optionally add per-model config schema/docs, decrement only on completed intended turns, warn near expiry, clear the user model override back to the configured default, and cover parser, persistence, selection, warning, and auto-revert paths with focused tests.
