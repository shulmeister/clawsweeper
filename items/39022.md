## Summary

Keep open. Current main supports persistent session-scoped `/model` overrides and transient auto-failover reset behavior, but it does not implement per-model turn budgets, `/model <name> --turns N`, warning thresholds, or automatic reversion to the default model.

## Best Possible Solution

Keep tracking this as a core model-selection/session-state feature unless a maintainer rejects the product direction. A complete implementation would add `/model <ref> --turns N` parsing, config/schema/docs for per-agent or per-model limits, persisted session budget state, warning and revert events, decrement semantics tied to actual user turns, and focused tests around persistence, resets, live model switching, and fallback interaction.
