## Summary

Keep #39022 open. Current main still supports persistent session-scoped `/model` overrides, but does not implement `/model <name> --turns N`, per-model turn budget config, warning thresholds, or automatic reversion to the default model. The prior ClawSweeper comment already reached the same keep-open conclusion, and the newer current main inspection did not find a shipped implementation or canonical superseding item.

## Best Possible Solution

Keep the issue open for maintainer product judgment or implementation. A complete fix likely belongs in core `/model` session behavior: extend directive/native command parsing for `--turns`, add durable session budget state, add config schema/docs for per-model limits, decrement only on the intended completed turns, warn near expiry, auto-clear the user model override back to default, and cover it with focused parser/session/model-selection tests.
