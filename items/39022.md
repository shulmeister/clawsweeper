## Summary

Keep #39022 open. Current main still implements persistent `/model` session overrides, but there is no `/model <name> --turns N` parser/command surface, no turn countdown/warning/revert session state, and no `models.limits` or per-model `maxTurns` config contract for model selection.

## Best Possible Solution

Keep the issue open for core product design and implementation. A complete solution should extend `/model` parsing and command metadata for `--turns`, persist remaining-turn/revert/warning state in session storage, decrement after completed turns, send warning and auto-revert notifications, clear the override through the existing model-override helper, add a validated per-agent/per-model config contract if accepted, update docs, and cover parser, persistence, warning, auto-revert, and config behavior with focused tests. If maintainers prefer a plugin route, first define a stable plugin API for safe model-selection interception and per-turn session accounting.
