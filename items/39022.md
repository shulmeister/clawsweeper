## Summary

Keep #39022 open. Current main still implements `/model` as a persistent session model override and has no parser, command metadata, session state, config schema, docs, or tests for `--turns`, warning thresholds, remaining-turn budgets, or automatic reversion to the default model.

## Best Possible Solution

Keep the issue open for maintainer product judgment or implementation in core model/session handling. A complete fix likely needs `/model --turns` parsing, persistent temporary-budget session fields, decrement-on-completed-turn semantics, warning and auto-revert notifications, config schema/docs for per-model limits if accepted, and focused tests for parsing, persistence, selection, warning, and reversion behavior.
