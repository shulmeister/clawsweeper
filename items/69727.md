## Summary

Keep #69727 open. Current main only exposes the existing Control UI assistant-message Copy as markdown action, plus the optional Expand action, and I found no print button, print handler, window.print path, or print-specific styling for individual assistant messages. The related copy-button item is a different surface and does not supersede this Control UI print request.

## Best Possible Solution

Keep the issue open and implement a focused Control UI feature: add an assistant-only print action beside the existing copy/expand controls in `ui/src/ui/chat/grouped-render.ts`, likely with a new helper next to `copy-as-markdown.ts` that renders the target message into an isolated printable document or window, triggers `window.print()`, and applies white-background, black-text, table/code-friendly print CSS. Add focused UI tests for action visibility and print invocation without disturbing the existing copy behavior.
