## Summary

Keep #69727 open. Current main still has only assistant-message copy and canvas-expand actions in the Control UI; I found no per-message Print button, print-only rendering path, `window.print()` handler, or print-friendly Control UI message stylesheet. The related supplied items are about copy support or other Control UI bugs and do not supersede this feature request.

## Best Possible Solution

Implement this in the Control UI assistant-message action surface: add a print icon/button next to the existing copy/expand controls, render the selected message markdown into an isolated print document or print-only container with clean table/code styling, call `window.print()`, and add focused UI tests for action rendering and print invocation. This belongs in `ui/src/ui/chat/grouped-render.ts` with a small helper next to `copy-as-markdown.ts` plus matching styles in `ui/src/styles/chat/grouped.css`.
