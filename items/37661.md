## Summary

Keep #37661 open. Current main still has tool-call loop detection and exact duplicate delivery suppression, but I did not find an assistant-text repetition guard, sentence/ngram detector, or outbound per-turn duplicate/message cap that would stop the reported pure text generation flood.

## Best Possible Solution

Keep this issue open and implement an output-level guard. The likely maintainer path is a focused assistant-text repetition detector in the streaming/final reply pipeline, plus an outbound per-turn duplicate/message cap where channel sends are planned. Add tests for repeated deltas, repeated sentence fragments in one assistant message, and repeated channel sends. Keep #34574 separate because it covers tool-call loops, while #37661 is about pure text generation/output flooding.
