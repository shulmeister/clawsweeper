## Summary

Keep open. Current main still has tool-call loop detection and some narrow duplicate-delivery fixes, but I did not find a guard for repeated assistant text generation or a generic outbound duplicate-message circuit breaker. The April 11 issue comment reports the same pure text-output loop on 2026.4.10 during normal main-session use, not just subagent timeout or tool-call loops.

## Best Possible Solution

Keep #37661 open and implement a generic output-level guard. The likely fix belongs near assistant text streaming and/or outbound delivery: detect repeated generated fragments or identical consecutive send payloads per session/turn, stop the turn before channel spam, and add focused tests for pure text repetition separate from tool-call loopDetection and heartbeat-specific duplicate suppression.
