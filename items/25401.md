## Summary

Keep #25401 open. Current main still does not implement the requested provider-level `models.providers.<id>.enabled` contract: the type/schema/docs do not expose it, models.json planning does not filter disabled providers, fallback candidate selection does not consult provider config, and `openclaw models status` has no disabled-provider output. The related implementation PR #25415 was closed unmerged, and the existing issue comment also concluded this should remain open.

## Best Possible Solution

Keep the issue open until core adds a documented `models.providers.<provider>.enabled?: boolean` contract, filters disabled providers from models.json generation and fallback candidate selection, and exposes disabled state in `openclaw models status` JSON and rich output with focused regression tests. PR #25415 may still be useful as a reference, but it was closed unmerged and current main does not satisfy the request.
