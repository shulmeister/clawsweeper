## Summary

Keep #25401 open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still does not define or honor a provider-level `models.providers.<id>.enabled` flag. The related implementation PR #25415 is closed unmerged per the provided GitHub context, and the prior issue comment already identified the same gap against an earlier main SHA; I rechecked the current checkout and the gap remains.

## Best Possible Solution

Keep the issue open until core adds a documented `models.providers.<provider>.enabled?: boolean` contract that defaults omitted values to enabled, preserves disabled provider config blocks, excludes disabled providers from models.json generation and model/fallback candidate selection, and reports disabled state in `openclaw models status` JSON and rich output. PR #25415 remains useful prior art but was closed unmerged.
