## Summary

Keep #25401 open. Current main at 82262387651ab270b565f71533ce4937c9c00dc6 still does not implement a provider-level `models.providers.<id>.enabled` contract, and the prior implementation PR #25415 was closed unmerged.

## Best Possible Solution

Keep the issue open until OpenClaw adds a documented `models.providers.<provider>.enabled?: boolean` contract that defaults to enabled, preserves disabled config blocks, excludes disabled providers from generated `models.json`, configured model catalogs, fallback candidate selection, status/probe paths, and displays disabled state in `openclaw models status` JSON and rich output. Reuse PR #25415 as implementation context if useful, but rebase/reimplement it because it was closed unmerged.
