## Summary

Keep #25401 open. Current main at f4e6322649a1ef0a9b47bd9d8b681c143850d669 still does not provide a documented provider-level `models.providers.<id>.enabled` contract, does not filter disabled providers from models.json planning or fallback candidates, and does not expose disabled provider state in `openclaw models status`. The related implementation PR #25415 was closed unmerged, and the existing issue discussion already reached the same keep-open conclusion.

## Best Possible Solution

Keep the issue open until core adds a documented `models.providers.<provider>.enabled?: boolean` contract, defaults missing values to enabled, filters disabled providers out of models.json generation and fallback candidate selection, and exposes disabled state in `openclaw models status` JSON and rich output. PR #25415 is a useful reference, but it was closed unmerged and current main does not satisfy the request.
