## Summary

Keep open. Current main does not implement the PR’s requested `[context-overflow-hint]` log or compaction troubleshooting docs, and the PR is a focused recent bug/DX fix with tests. It should not be closed as obsolete, but the unresolved review comment is substantive: the proposed `overflowCompactionAttempts > 0` gate should be tightened to an immediate post-compaction retry signal before merge.

## Best Possible Solution

Keep the PR open and ask for a small revision: implement the hint on main’s overflow handler, but gate it on an immediate compact-succeeded-then-retry-overflow condition rather than the run-scoped cumulative counter. Add the two proposed regression tests plus one negative case for a later independent overflow after an intervening successful/non-overflow loop, and keep the docs/changelog update if the runtime change lands.
