## Summary

Keep PR #66712 open. Current main has only execution-level cron direct-delivery idempotency keyed by `createCronExecutionId(jobId, runStartedAt)`, so a replay of the same scheduled slot with a different `runStartedAt` can still bypass the cache. The PR is a focused, recently updated fix for that remaining scheduled-slot gap, and the existing discussion explicitly says the slot-level guard is complementary rather than redundant.

## Best Possible Solution

Keep the PR open for normal maintainer review. The best path is to review or land a narrow slot-level guard in `src/cron/isolated-agent/delivery-dispatch.ts`, with regression coverage for same scheduled slot plus different `runStartedAt`, and include the `deliveryAttempted` propagation fix in `runStartupCatchupCandidate` if it is still desired.
