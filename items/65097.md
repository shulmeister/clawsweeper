## Summary

Keep open. Current main at 9626ef274ae2346b3d519f99c31856be98aab570 still lacks the requested Dreaming diary language/locale control: the narrative prompt path is English-only, the runtime parameter/config contract has no language or locale field, and the strict memory-core plugin schema/docs expose only existing dreaming settings. The provided discussion also shows related #69354 was closed as a duplicate of this remaining language-instability problem, not as an implementation.

## Best Possible Solution

Keep this issue open and implement the feature in `memory-core`: add a documented `dreaming.language` or `dreaming.locale` config contract, decide whether `auto` is supported, thread the resolved hint into both the narrative system prompt and user prompt path, preserve request-scoped fallback behavior, update `extensions/memory-core/openclaw.plugin.json`, public memory/Dreaming docs, and focused config/narrative tests.
