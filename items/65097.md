## Summary

Keep open. Current `origin/main` still hardcodes the Dream Diary narrative prompt in English, does not thread any language/locale setting into the narrative subagent run, and the `memory-core` dreaming config schema exposes no `language`, `locale`, `auto`, or custom prompt override field. Related Dreaming config items are adjacent but do not implement this requested language-control surface.

## Best Possible Solution

Keep the issue open and implement this in `memory-core` by adding a documented `dreaming.language` or `dreaming.locale` contract, deciding whether `auto` inference is supported, threading the resolved hint into `buildNarrativePrompt`/`NARRATIVE_SYSTEM_PROMPT`, updating the plugin manifest schema, memory config docs, and focused narrative/config tests. If custom prompt templates are accepted, keep them narrowly scoped and schema-validated rather than allowing an unbounded hidden prompt override.
