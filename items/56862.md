## Summary

Current `main` still intentionally runs search-triggered builtin memory sync asynchronously instead of awaiting it when the index is dirty, so this issue does not have strong enough evidence for closure. The repo now has better dirty-state visibility and manual reindex commands, but those are mitigations/workarounds rather than a confirmed fix for empty `memory_search` results against stale builtin state.

## Evidence

- **Search path still does not await dirty sync:** Builtin memory search calls `startAsyncSearchSync(...)` when `dirty` or `sessionsDirty` is set, then continues into provider initialization and query execution without awaiting the sync. That means searches can still run against stale indexed state.
