## Summary

Keep open. Current main still does not export `stringEnum` or `optionalStringEnum` from the root `openclaw/plugin-sdk` entry, and related reports describe a real external-plugin compatibility break. The PR is incomplete as-is because widening the root SDK surface also requires updating the exact root-export contract and SDK API baseline, but that means it needs maintainer/API-surface follow-up rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review. If OpenClaw wants to preserve backward compatibility for existing external plugins, amend this PR to deliberately promote `stringEnum` and `optionalStringEnum` on `openclaw/plugin-sdk`, update `src/plugins/contracts/plugin-sdk-index.test.ts`, regenerate the plugin SDK API baseline hash, and add a focused regression check for the root import. If maintainers decide root imports should stay minimal, close only after #66195/#68279 are resolved with a clear migration path for affected plugins to `openclaw/plugin-sdk/core` or `openclaw/plugin-sdk/channel-actions`.
