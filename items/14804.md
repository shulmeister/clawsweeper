## Summary

Keep #14804 open. Current main still has sandbox scope controls, workspace access controls, Docker tmpfs mounts for temp directories, and prune/recreate lifecycle, but it does not provide the requested explicit ephemeral or tmpfs-backed workspace for isolated sessions/runs.

## Best Possible Solution

Keep #14804 open until core sandbox lifecycle/config adds an explicit ephemeral workspace mode, likely as an additive agents.defaults.sandbox setting. The implementation should create a fresh per-run workspace where supported, clean it at run completion, define behavior across Docker/SSH/OpenShell or document backend limits, and update sandbox docs plus config/lifecycle tests.
