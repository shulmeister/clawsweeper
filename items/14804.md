## Summary

Keep #14804 open. Current main still has persistent per-scope sandbox workspaces plus manual/idle prune/recreate, but no first-class ephemeral, disposable, or tmpfs-backed workspace lifecycle for isolated sessions. The existing issue comment already kept this open for the same core sandbox gap, and the relevant sandbox config/schema/workspace/docs paths have not materially changed since that review.

## Best Possible Solution

Keep the issue open for a core sandbox lifecycle/config feature. Add documented AgentSandboxConfig/schema fields for disposable or ephemeral workspaces, create a fresh workspace per isolated run where supported, clean it on completion, define Docker/SSH/OpenShell behavior including whether tmpfs is required or best-effort, add lifecycle tests, and update the public sandbox/config docs to distinguish session/container isolation from disposable filesystem state.
