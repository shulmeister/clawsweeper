## Summary

Keep open. Current main still has sandbox isolation with agent/session/shared scope and explicit recreate/prune lifecycle, but it does not provide the requested per-run ephemeral or tmpfs-backed workspace option. The prior Codex review comment on #14804 remains accurate against current main SHA 2f6615d2ee9d0d9db0f4ad1e13f71d1c0df8166d.

## Best Possible Solution

Keep #14804 open until core sandbox lifecycle supports an explicit ephemeral workspace mode, likely as an additive agents.defaults.sandbox setting that creates a fresh per-run workspace, mounts it consistently across Docker/SSH/OpenShell or documents backend limits, cleans it at run completion, and updates sandbox docs plus lifecycle/config tests. A custom sandbox backend can be plugin-owned, but the first-class per-run workspace option requested here is still a core sandbox configuration and lifecycle gap.
