## Summary

Keep #14804 open. Current main still exposes persistent per-scope sandbox workspaces and manual prune/recreate behavior, but it does not implement a first-class ephemeral/tmpfs-backed workspace lifecycle for isolated sessions. The issue is contributor-authored, has only the `enhancement` label, and has no canonical duplicate or superseding item in the provided related context.

## Best Possible Solution

Keep the issue open for a core sandbox lifecycle/config feature. The likely product path is to add documented AgentSandboxConfig/schema fields for disposable or ephemeral workspaces, create a fresh per-run workspace where supported, clean it on completion, define Docker/SSH/OpenShell behavior including whether tmpfs is required or best-effort, add lifecycle tests, and update public sandbox/config docs.

## Risks / Open Questions

- A future implementation needs explicit cross-backend semantics: Docker could use disposable bind roots or tmpfs, while SSH/OpenShell need remote/local workspace creation and cleanup rules.
- The term isolated session can currently mean transcript/process isolation, so docs should distinguish that from disposable filesystem state.
