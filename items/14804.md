## Summary

Keep #14804 open. Current `main` still has sandbox modes, scope isolation, workspace access controls, Docker tmpfs mounts for temp directories, SSH/OpenShell remote workspace flows, and stale runtime pruning, but it does not provide the requested first-class ephemeral or tmpfs-backed workspace mode for isolated sessions/runs. The existing issue comment keeping this open remains accurate against `f4e6322649a1ef0a9b47bd9d8b681c143850d669`.

## Best Possible Solution

Keep #14804 open until core sandbox lifecycle/config adds a documented disposable workspace mode. A good implementation would add an explicit sandbox setting, create a fresh per-run workspace where supported, clean it on completion, define backend behavior for Docker/SSH/OpenShell, decide whether tmpfs is required or best-effort by platform, and update config schema, sandbox lifecycle tests, and public sandbox docs.
