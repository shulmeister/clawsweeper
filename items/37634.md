## Summary

Keep #37634 open. Current main still does not implement the requested writable isolated workspace for `workspaceAccess: "none"`: `none` uses the per-sandbox workspace, but Docker/browser mounts and both local and remote filesystem write gates still treat it as read-only. Related fix PRs #37276 and #38154 are closed unmerged, and #46300 remains an open implementation candidate rather than shipped behavior.

## Best Possible Solution

Keep #37634 open as the focused product/bug tracker unless maintainers explicitly decide that `workspaceAccess: "none"` should remain read-only. A complete implementation would make the isolated `none` sandbox workspace writable consistently across Docker/browser mounts, local and remote filesystem bridges, mutation gates, memory/tool write paths, tests, and docs, or else update the issue with the explicit by-design decision and documented workaround.
