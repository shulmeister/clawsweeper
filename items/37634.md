## Summary

Keep #37634 open. Current main still does not implement the requested isolated-but-writable `workspaceAccess: "none"` behavior: `none` selects the per-scope sandbox workspace, but Docker/browser mounts, local and remote filesystem bridges, memory flush gating, tests, and docs still treat that workspace as read-only or write-restricted. Related fix PRs #37276 and #38154 were closed unmerged, and #46300 remains an open implementation path rather than a shipped fix.

## Best Possible Solution

Keep #37634 open as the focused tracker. The best fix is to preserve `workspaceAccess: "none"` isolation from the agent workspace while making the isolated sandbox `/workspace` writable consistently across Docker/browser mounts, local and remote filesystem bridges, write gating, memory-flush behavior if applicable, regression tests, and public docs; alternatively, make an explicit maintainer-approved product decision and then close or supersede the related writable-none items accordingly.
