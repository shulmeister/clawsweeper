## Risks / Open Questions

- The PR should not be merged until the default-agent mismatch is fixed: gateway/session paths commonly resolve omitted agent ids through `resolveDefaultAgentId`, while the proposed command can leave `agentId` undefined.
- Session explain mode needs a clear contract for positional model/`--provider` combined with `--session`: reject those combinations or include them in the resolver input before reporting override fields.
- The branch is based on an older main and should be rebased before another maintainer review.
