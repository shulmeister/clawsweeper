## Summary

Keep open. Current `main` still fail-opens unpaired node reconnects: when no approved node-pair record exists, the gateway keeps the node's declared commands, stores them on the live node session, and `node.invoke` authorizes from that live command list plus the global allowlist without consulting pairing state. That contradicts the published pairing docs, and a current test still asserts the pre-approval command availability.
