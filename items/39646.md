## Summary

Keep PR #39646 open. Current main still has the stale raw default-model behavior the PR targets: `models status --json`, probe candidate construction, rich `models status` output, and `openclaw status` summary can still use a parseable raw `provider/model` value before validating that the provider remains configured. The PR is still useful directionally, but it should not merge as-is because the provided review context flags incomplete handling and unrelated bundled changes.

## Best Possible Solution

Keep this open or replace it with a narrower maintainer patch. Centralize provider-validating default-model resolution for `models status --json`, rich `models status`, auth-probe candidates, and `openclaw status`, then add regression tests for a configured default that references a provider removed from `models.providers`. Split unrelated changes out before merge.
