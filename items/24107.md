## Summary

Keep #24107 open. Current main has improved cooldown behavior since the issue was filed, including shorter stepped transient cooldowns, model-scoped rate-limit cooldowns, same-provider probe handling, and OpenAI Codex WHAM-specific cooldown probing. It still does not implement the requested generic provider-level RPM/RPD contract, quota-window counters, or pre-request quota skip for arbitrary providers. The existing maintainer/Codex comment on the issue also explicitly kept it open for the same gap.

## Best Possible Solution

Keep #24107 open as a core model fallback/auth-profile enhancement. The best path is to add a validated provider quota contract such as rpm/rpd or an equivalent explicit shape, update generated schema/help/docs, persist minute/day quota state with reset timestamps, add a pre-request quota skip that marks cooldownUntil before provider calls, and cover RPM/RPD reset behavior with focused tests alongside the existing cooldown/probe tests.
