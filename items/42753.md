## Summary

Current `main` no longer uses a bundled Node/Gateway runtime inside the macOS app for local mode, and the new macOS packaging/signing flow adds an explicit Team ID audit for every Mach-O in the app bundle. That makes this specific `sharp` Team ID mismatch report obsolete on `main`.

## Close Comment

Closing this as implemented on current `main` after a Codex review. The macOS app no longer uses a bundled Node/Gateway runtime for local mode; it now requires an external `openclaw` CLI install and launches the gateway from that runtime instead (`apps/macos/Sources/OpenClaw/GatewayEnvironment.swift:130`, `apps/macos/Sources/OpenClaw/CLIInstaller.swift:37`, `docs/platforms/mac/bundled-gateway.md:10`). Current `main` also adds a Team ID audit in the mac app signing flow that scans every Mach-O in the app bundle and fails packaging on mismatches (`scripts/codesign-mac-app.sh:205`, `apps/macos/README.md:29`). That means the original AutoClaw-specific failure mode of loading an unsigned bundled `sharp` binary into the signed app process no longer matches the current implementation.
