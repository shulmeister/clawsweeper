## Summary

Close as a duplicate of the older open macOS gateway exec reachability tracker #61713. The report describes the same product boundary: OpenClaw exec launched through the macOS LaunchAgent/gateway path cannot reach a LAN host even though the user's normal shell can. Current main still routes gateway exec through the same host exec environment, gateway allowlist path, and process supervisor; I did not find a shipped fix marker for the LAN/no-route symptom. The HTTPS/CalDAV details are useful additional reproduction evidence, but the remaining maintainer work should stay centralized in #61713.

## Best Possible Solution

Close this duplicate and keep #61713 as the canonical open tracker. The best maintainer path is to add this LAN HTTPS/CalDAV reproduction shape to that investigation, reproduce on current main/latest release under the macOS LaunchAgent gateway path, then fix the shared gateway exec/runtime or service environment/network context there.
