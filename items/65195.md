## Summary

Current `main` still routes default exec on macOS through the launchd-managed gateway, and the gateway-host exec path only compensates for `PATH`, not broader network-context differences. I did not find a main-branch fix or doc change that would make this specific launchd-vs-interactive-shell LAN reachability report obsolete, so the issue should stay open.

## Evidence

- **macOS gateway still runs as a LaunchAgent, not as a child of the app:** The macOS lifecycle doc says the app manages the Gateway via `launchd` by default, does not use child-process mode today, and tells users to run the Gateway manually in a terminal if they need tighter coupling to the UI.
  - file: [docs/platforms/mac/child-process.md:10](https://github.com/openclaw/openclaw/blob/6abab7555e1f23ef1b3aa0edced3592037e029cf/docs/platforms/mac/child-process.md#L10)
  - command: `nl -ba docs/platforms/mac/child-process.md | sed -n '8,32p'`
  - sha: [6abab7555e1f](https://github.com/openclaw/openclaw/commit/6abab7555e1f23ef1b3aa0edced3592037e029cf)
- **Default exec routing still lands on the gateway host:** The exec docs still say `tools.exec.host` defaults to `auto`, which resolves to `gateway` when no sandbox runtime is active. They also document that `host=gateway` only merges login-shell `PATH` while the daemon itself keeps a minimal PATH.
  - file: [docs/tools/exec.md:97](https://github.com/openclaw/openclaw/blob/6abab7555e1f23ef1b3aa0edced3592037e029cf/docs/tools/exec.md#L97)
  - command: `nl -ba docs/tools/exec.md | sed -n '97,131p'`
  - sha: [6abab7555e1f](https://github.com/openclaw/openclaw/commit/6abab7555e1f23ef1b3aa0edced3592037e029cf)
- **Gateway exec implementation only patches PATH from the login shell:** The gateway exec path builds host env from `process.env`, sanitizes overrides, rejects request-scoped `PATH`, and only calls `getShellPathFromLoginShell`/`applyShellPath` for the gateway host. I did not find any macOS-specific network-context reconciliation here.
  - file: [src/agents/bash-tools.exec.ts:1536](https://github.com/openclaw/openclaw/blob/6abab7555e1f23ef1b3aa0edced3592037e029cf/src/agents/bash-tools.exec.ts#L1536)

- **macOS app-context execution exists, but only on the node/system.run path:** The macOS platform docs distinguish `system.run` from gateway exec: `system.run` executes in the macOS app over local IPC, with its own approval/env behavior.
  - file: [docs/platforms/macos.md:63](https://github.com/openclaw/openclaw/blob/6abab7555e1f23ef1b3aa0edced3592037e029cf/docs/platforms/macos.md#L63)
