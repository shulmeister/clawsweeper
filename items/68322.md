## Summary

Keep open. Current main still has the exact extension discovery gap this PR targets: the Kilocode provider discovery fetch sends only Accept, with no Authorization or organization header, and the existing tests still assert that unauthenticated shape.

## Best Possible Solution

Keep the PR open and land this or an equivalent authenticated Kilocode discovery fix in extensions/kilocode/provider-models.ts, with regression tests for set and unset KILOCODE_API_KEY and any supported organization header. If standard OpenClaw discovery is still profile-backed rather than env-backed, add a follow-up or broaden the fix to resolve Kilocode credentials from the same auth storage used at runtime.
