## Summary

The reported DingTalk Stream-mode failure is tied to an external npm plugin (`@dingtalk-real-ai/dingtalk-connector`), while current `main` keeps channel integrations plugin-owned and does not ship a bundled DingTalk implementation. OpenClaw docs and vision point users to ClawHub/npm plugins for this surface, including an already-listed DingTalk community plugin.

## Evidence

- **Vision scopes optional integrations to plugins, not core:** OpenClaw's vision says optional capability should usually ship as plugins, hosted and maintained in their own repositories, with plugin discovery and review living in ClawHub.
