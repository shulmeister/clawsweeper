## Summary

Keep open. Current main still intentionally treats ended subagent runs as active when they have pending descendants, and that same predicate contributes to the per-session active-child quota. The related #71252/#71264 stale-unended work addresses the no-endedAt orphan path, but not this endedAt-with-descendants report.

## Best Possible Solution

Keep #66187 open and decide the intended lifecycle semantics. If ended orchestrators should not consume active-child quota, update `isActiveSubagentRun`, `countActiveRunsForSessionFromRuns`, and pending-descendant/cleanup handling so ended parent rows move to recent or a separate cleanup/pending state without blocking `sessions_spawn`; then revise the current opposite regression tests and docs. If the current orchestrator behavior is intended, maintainers should close manually with a clear works-as-designed explanation and possibly add docs for the quota impact.
