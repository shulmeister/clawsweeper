## Summary

Current main already persists provider OAuth results into the selected agent auth store, inherits main-agent auth into new agent stores when needed, and repairs stale `openai-codex:default` profiles that could shadow newer identity-scoped Codex OAuth profiles. The fix is documented as shipped in `v2026.4.22`.

## Close Comment

Closing from a Codex review because current `main` already covers this. The provider auth path writes OAuth results into the selected agent store (`src/plugins/provider-auth-choice.ts:125`, `src/agents/auth-profiles/profiles.ts:46`), empty agent stores can inherit auth from the main agent (`src/agents/auth-profiles/store.ts:202`), and stale `openai-codex:default` entries are repaired so they no longer shadow the newer identity-scoped Codex OAuth profile (`src/agents/auth-profiles/persisted.ts:344`). There is matching regression coverage in `src/agents/auth-profiles.ensureauthprofilestore.test.ts:234`, and the fix is called out in the shipped `v2026.4.22` changelog (`CHANGELOG.md:149`). If this still reproduces on `v2026.4.22` or newer, please open a fresh issue with the exact agent id, the profile ids present in that agent's `auth-profiles.json`, and whether the agent reused an existing `agentDir`.
