## Summary

Keep #13870 open. Current main displays and refreshes a client-supplied paired-device `displayName`, but it still does not implement the requested operator-owned rename/label workflow for paired devices in the CLI, gateway protocol/method surface, server handlers, Control UI, or devices docs. Related #23916 is broader device metadata display work, and closed #57424 overlaps the identification problem but does not supersede this issue’s specific label-management request.

## Best Possible Solution

Keep #13870 open until paired-device label management is implemented end to end: add an additive gateway method and protocol schema such as `device.rename`, authorize and persist an operator-owned display-name override with a clear precedence rule over reconnect/client metadata, add `openclaw devices rename --device <id> --name <displayName>`, add a Control UI rename action, cover storage/protocol/CLI/UI behavior with focused tests, and update the public `openclaw devices` docs. Keep the existing `node.rename` surface separate from paired-device labels, and let #23916 continue tracking broader metadata display if needed.
