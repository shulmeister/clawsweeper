## Summary

Keep #13870 open. Current main partially addresses identification by displaying client-supplied `displayName` metadata in `openclaw devices list` and the Control UI, but it still lacks the requested operator-owned paired-device rename/label workflow. The related #23916 is broader metadata display work, while #13870 remains the focused request for user-assigned paired-device names.

## Best Possible Solution

Keep #13870 open until paired-device label management is implemented end to end: add an additive gateway protocol method/schema/handler for paired-device rename, persist an operator-owned label with precedence over reconnect metadata, expose `openclaw devices rename --device <id> --name <displayName>`, add a Control UI rename action, cover storage/protocol/CLI/UI behavior with focused tests, and update the devices/protocol docs. Keep #23916 focused on broader metadata display and keep `node.rename` separate.
