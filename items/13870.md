## Summary

Keep #13870 open. Current main still supports only partial display of an existing paired-device `displayName`; it does not provide the requested user-facing paired-device rename workflow in the CLI, gateway protocol/schema, Control UI, or docs. Node rename is implemented, but it is a separate node-pairing surface and does not label paired device records.

## Best Possible Solution

Keep the issue open until OpenClaw adds paired-device label management end to end: a gateway `device.rename` method and schema, storage update path for paired device records, `openclaw devices rename --device <id> --name <displayName>`, Control UI rename action, docs at `docs/cli/devices.md` / public `https://docs.openclaw.ai/cli/devices`, and focused CLI/protocol/UI tests. Keep node rename as a separate related surface, not a substitute.
