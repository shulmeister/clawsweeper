## Summary

Keep #13870 open. Current main still only partially solves the request: paired device records can carry and display an existing `displayName`, and node rename exists for the related node-collision comment, but there is still no user-facing paired-device rename workflow (`openclaw devices rename`, `device.rename` RPC/schema, or Control UI rename action).

## Best Possible Solution

Keep the issue open until OpenClaw adds a paired-device label management path, likely a `device.rename` protocol method plus `openclaw devices rename --device <id> --name <displayName>`, Control UI rename action, docs, and focused tests. Node rename should remain separate and can be referenced as related but not sufficient.
