## Summary

Keep open. Current main partially addresses the problem by storing and rendering `displayName` for paired devices, and it adds a separate `openclaw nodes rename` path for node names. The specific requested paired-device label management surface remains missing: there is no `openclaw devices rename`, no `device.rename` RPC/schema entry, and no Control UI device rename action. The issue comment about node-name collisions is positively addressed by the node rename work, but it does not fully cover the original paired-devices request.
