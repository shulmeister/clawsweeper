## Summary

Current `main` still enforces Telegram DM pairing as DM-only and does not use the pairing allowlist store for group command authorization, so this report is neither fixed nor obsolete. Docs, runtime code, and tests all align on that behavior, and the linked open PR suggests active unresolved work rather than something safe to auto-close.

## Evidence

- **Group auth explicitly excludes pairing store:** `resolveTelegramGroupAllowFromContext` reads the channel allowlist store but then documents and enforces that DM pairing-store entries are not a group authorization source; group sender auth is derived only from `groupAllowFrom` or per-group/topic `allowFrom`.
