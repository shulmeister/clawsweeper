## Summary

Current `main` already supports spawning external harness/CLI agents through `sessions_spawn` via the ACP runtime, including Codex and other supported CLIs, with agent config for ACP defaults and background task tracking.

## Close Comment

Closing as implemented on `main` after Codex review. OpenClaw now supports external harness/CLI agents through the ACP runtime: `sessions_spawn` accepts `runtime: "acp"` and routes to the ACP spawn path (`src/agents/tools/sessions-spawn-tool.ts:26`, `src/agents/tools/sessions-spawn-tool.ts:245`). The docs explicitly cover Codex, Claude Code, Cursor, and Gemini CLI, and show `sessions_spawn` with `runtime: "acp"` and `agentId: "codex"` (`docs/tools/acp-agents.md:13`, `docs/tools/acp-agents.md:274`). Per-agent config also supports ACP-backed defaults via `agents.list[].runtime.type = "acp"` (`docs/gateway/configuration-reference.md:1790`). This shipped in `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
