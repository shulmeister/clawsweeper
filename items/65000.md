## Summary

Keep open. Current `main` has ACP background-task notifications and a recent thread-bound fallback fix, but the code still does not provide strong evidence that parent-facing ACP/subagent completion delivery is durable across retry/reconnect gaps. In particular, the `streamTo="parent"` ACP path bypasses subagent-registry completion tracking, terminal delivery only auto-runs while status stays `pending`, and the session fallback uses explicitly ephemeral in-memory system events.

## Evidence

- **ACP `streamTo="parent"` skips subagent-registry tracking:** `sessions_spawn` only registers accepted ACP runs with `registerSubagentRun(...)` when `streamTo !== "parent"`, so parent-streamed ACP runs do not use the durable subagent completion/cleanup path.
  - file: [src/agents/tools/sessions-spawn-tool.ts:282](https://github.com/openclaw/openclaw/blob/a5db42862d76fce18e5d7fa57f3907a543e61137/src/agents/tools/sessions-spawn-tool.ts#L282)
  - sha: [a5db42862d76](https://github.com/openclaw/openclaw/commit/a5db42862d76fce18e5d7fa57f3907a543e61137)
- **Some ACP child runs enter parent-stream mode implicitly:** `spawnAcpDirect` implicitly enables `effectiveStreamToParent` for run-mode ACP spawns from subagent requester sessions when heartbeat relay routing is usable, so this is not limited to an explicit operator opt-in.
  - file: [src/agents/acp-spawn.ts:793](https://github.com/openclaw/openclaw/blob/a5db42862d76fce18e5d7fa57f3907a543e61137/src/agents/acp-spawn.ts#L793)
  - sha: [a5db42862d76](https://github.com/openclaw/openclaw/commit/a5db42862d76fce18e5d7fa57f3907a543e61137)
- **Terminal auto-delivery only runs while delivery status is `pending`:** `shouldAutoDeliverTaskTerminalUpdate` returns true only when the task is terminal and `deliveryStatus === "pending"`. Once task delivery is rewritten to another state, the automatic delivery path stops.
  - file: [src/tasks/task-executor-policy.ts:92](https://github.com/openclaw/openclaw/blob/a5db42862d76fce18e5d7fa57f3907a543e61137/src/tasks/task-executor-policy.ts#L92)
  - sha: [a5db42862d76](https://github.com/openclaw/openclaw/commit/a5db42862d76fce18e5d7fa57f3907a543e61137)
- **Fallback delivery rewrites terminal ACP tasks away from `pending`:** When direct delivery cannot use the requester origin, OpenClaw queues a session system event and marks the task `session_queued`; when direct send throws, it queues a fallback system event and marks the task `failed`. Neither path keeps the task in a retryable `pending` state.
  - file: [src/tasks/task-registry.ts:1053](https://github.com/openclaw/openclaw/blob/a5db42862d76fce18e5d7fa57f3907a543e61137/src/tasks/task-registry.ts#L1053)
  - sha: [a5db42862d76](https://github.com/openclaw/openclaw/commit/a5db42862d76fce18e5d7fa57f3907a543e61137)
- **Session fallback queue is explicitly ephemeral:** `system-events.ts` documents the queue as a lightweight in-memory queue and explicitly says it avoids persistence to keep events ephemeral, which does not match a durable reconnect/restart guarantee.
  - file: [src/infra/system-events.ts:1](https://github.com/openclaw/openclaw/blob/a5db42862d76fce18e5d7fa57f3907a543e61137/src/infra/system-events.ts#L1)
  - sha: [a5db42862d76](https://github.com/openclaw/openclaw/commit/a5db42862d76fce18e5d7fa57f3907a543e61137)
- **Current tests still model missed direct delivery as non-durable session fallback:** The task-registry tests assert that outward delivery misses become `failed` plus queued system events, or `session_queued` plus queued system events, rather than a persisted retry obligation.
  - file: [src/tasks/task-registry.test.ts:598](https://github.com/openclaw/openclaw/blob/a5db42862d76fce18e5d7fa57f3907a543e61137/src/tasks/task-registry.test.ts#L598)
  - sha: [a5db42862d76](https://github.com/openclaw/openclaw/commit/a5db42862d76fce18e5d7fa57f3907a543e61137)
- **Latest related main-branch fix is narrower:** The only closely related recent change found on `main` is commit `325e5e921fd64e3b4d8d56e396270cb2552a3d67` (`fix: preserve thread-bound subagent completion fallback`), which addresses thread-bound fallback routing rather than durable retry/reconnect delivery for ACP/subagent completion.
  - command: `git show --stat --summary 325e5e92`
  - sha: [325e5e921fd6](https://github.com/openclaw/openclaw/commit/325e5e921fd64e3b4d8d56e396270cb2552a3d67)

## Risks / Open Questions

- There may be other runtime paths that soften the impact in practice, but the inspected code does not show a durable end-to-end retry/reconnect guarantee for the reported ACP completion path.
- Implicit `streamTo="parent"` ACP runs look especially exposed because they bypass subagent-registry completion tracking and rely on terminal task delivery plus ephemeral session-event fallback.
