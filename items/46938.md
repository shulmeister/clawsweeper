## Summary

Keep open. The report is a concrete core memory/session resiliency request, not protected or maintainer-authored, and it is not stale. Current main has automatic pre-compaction memory flush and timeout-triggered compaction/failover paths, but the flush still requires a successful model turn and there is no implemented pending-session memory recovery mechanism like the issue requests.
