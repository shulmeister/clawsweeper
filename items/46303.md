## Summary

Keep open. Current main still does not implement this PR's central restart-time drain behavior: inbound debounce buffers remain closure-local, there is no all-followup-queue drain waiter, and the SIGUSR1 restart path only drains active command/embedded work before closing the server. The related restart deferral/orphan-recovery work is narrower and does not make this PR obsolete.

## Best Possible Solution

Keep this PR open as an implementation candidate, but ask for a narrower maintainer-reviewed revision: add a controlled lifecycle-safe drain hook or registry for active inbound debouncers, keep the public plugin SDK backward-compatible, flush debounce buffers at the correct restart phase without losing late ingress, wait for active turns and then all followup queues including dropped-summary state, and cover restart ordering plus channel teardown races with focused tests.
