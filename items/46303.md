## Summary

Keep open. Current main still does not implement the PR's core behavior: there is no process-wide inbound debouncer registry, no all-debouncer flush hook, no all-followup-queue drain waiter, and the SIGUSR1 restart path only drains active command/embedded work before closing the server. The merged #47719 restart work is related but narrower, covering restart deferral and subagent orphan recovery rather than closure-local debounce buffers or pending followup queues.

## Best Possible Solution

Keep the PR open. The best path is a smaller additive core fix that registers active inbound debouncers without breaking the public plugin SDK, performs a bounded flush before restart teardown, waits for followup queues using the same pending predicate as the drain loop, and adds targeted tests for SIGUSR1 ordering plus channel teardown races. Do not close as implemented or duplicate until current main has equivalent behavior and coverage.
