## Summary

Keep PR #56538 open. Current main still leaves the process exec surfaces decoding Windows output as UTF-8, while the linked canonical bug #50519 remains open and the PR contains a focused implementation attempt. The PR is not merge-ready as-is, but it is not obsolete or safely closeable.

## Best Possible Solution

Keep the PR open, but ask for a focused rework: implement Windows code-page-aware decoding in src/process/exec.ts for runExec and runCommandWithTimeout, use a streaming decoder where chunks are emitted incrementally, avoid hardcoded GBK for non-Chinese Windows code pages, add regression coverage for GBK and UTF-8 Windows outputs, and remove unrelated ACPX/CLAUDE.md changes. Coordinate the remaining product bug with #50519.
