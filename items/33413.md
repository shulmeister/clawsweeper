## Summary

Keep #33413 open. Current main still exposes tool-start events and Slack's assistant thread status setter, but Slack dispatch does not map tool names to `assistant.threads.setStatus`; it only sets the static `"is typing..."` status and clears it when done. The closed PR #33420 was not merged, and #67129 is adjacent customization work rather than a canonical replacement.

## Best Possible Solution

Keep #33413 open until Slack dispatch maps `onToolStart` events to `ctx.setSlackThreadStatus` for assistant thread targets, with a small tool-name-to-label helper and focused Slack tests that simulate a tool start and assert an additional `assistant.threads.setStatus` call. If maintainainers decide draft-preview progress replaces assistant status labels, close only with an explicit supersession comment linking that chosen behavior or tracking issue.
