## Summary

Keep open. Current main still does not implement tool-level Slack `assistant.threads.setStatus` updates. The core runner emits `onToolStart`, and Slack consumes it for reactions and draft-preview progress, but the assistant thread status setter is still only used for the static start/clear typing status. The linked fix PR #33420 was closed unmerged, and #67129 is an adjacent customization request rather than a canonical replacement for this issue.

## Best Possible Solution

Keep #33413 open until Slack dispatch maps tool-start events to `ctx.setSlackThreadStatus` for assistant thread targets, with focused tests that simulate `onToolStart` and assert an additional Slack `assistant.threads.setStatus` call. If maintainers decide draft-preview progress intentionally replaces assistant status labels, close only with an explicit supersession comment linking the chosen behavior or tracking issue.
