## Summary

Keep #33413 open. Current main still sets Slack assistant thread status only to the static "is typing..." value at reply start and clears it at reply end. Tool-start events are available and Slack dispatch handles them, but only for status reactions and draft-preview progress, not per-tool `assistant.threads.setStatus` labels. Related PR #33420 is closed unmerged, and related issue #67129 is an open customization request rather than a shipped replacement.

## Best Possible Solution

Keep the issue open until Slack dispatch maps `onToolStart` events to `ctx.setSlackThreadStatus` for assistant-thread targets, with a small tool-name-to-label helper and focused Slack coverage asserting the extra `assistant.threads.setStatus` call. If maintainers decide draft-preview progress, reactions, or configurable Slack loading messages are the intended replacement UX, document that product decision and link a canonical superseding issue or PR before closing #33413.
