## Summary

Keep this PR open. Current main still lacks the hex-suffixed and underscore-stripped reverse-lookup behavior the PR targets, and the PR is a plausible focused agent-runtime fix rather than obsolete cleanup. It should not be merged as-is without regression tests and maintainer review of the heuristic/fail-closed behavior.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rework or review it against current main in `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts`, keep the allowlist-backed mapping fail-closed on ambiguity, and add focused tests for `read5f378... -> read`, `functionsread5f... -> read`, underscore-stripped variants, dropped-leading-character collisions, `blacklist1a2b3c`, `readfeed`, and `foo_bar`/`foobar` collisions before landing.
