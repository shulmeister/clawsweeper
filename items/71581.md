## Summary

Keep open. Current main still preserves explicit same-agent session keys for exec-event and cron-event heartbeat wakes, while the single-owner session.dmScope="main" helper is only used by inbound channel/plugin routing. The closed related PR #72160 was not merged and is unrelated to heartbeat/session routing.

## Best Possible Solution

Keep this issue open and implement a narrow core routing helper for internal system-event wakes. When session.dmScope="main" and the same single-owner allowlist semantics used by inbound DM routing apply, normalize same-agent direct-message session keys to resolveAgentMainSessionKey before exec and cron system-event enqueue, heartbeat wake targeting, preflight peek, and event consumption. Add regression coverage for exec notify-on-exit, explicit exec system events, and cron next-heartbeat/main-session events, plus negative cases for multi-owner/wildcard allowlists, non-main scopes, subagents, isolated heartbeats, and non-DM explicit sessions.
