## Summary

Keep PR #43462 open. The original touched files have moved on current main, but the requested behavior is still not implemented: workspace memory path authorization still checks only a case-sensitive `memory/` prefix, temporal decay still parses and classifies only lowercase `memory/` paths, and current tests do not cover case-variant memory directories. The PR's follow-up commit addressed the review feedback in the supplied discussion, so the remaining best path is to port the fix to the current memory-host-sdk and memory-core files.

## Best Possible Solution

Port the PR's directory-prefix behavior to current main by lowercasing only the normalized `memory/` directory prefix checks in `src/memory-host-sdk/host/internal.ts` and `extensions/memory-core/src/memory/temporal-decay.ts`, applying the lowercased value to the dated-memory regex, and adding focused regression tests for `MEMORY/notes.md`, `Memory/daily.md`, and `MEMORY/YYYY-MM-DD.md`. Keep current canonical root `MEMORY.md` semantics intact and update the package copy if it is still intended to mirror the host SDK.
