## Summary

Keep open. Current `main` has adjacent primitives, but not the requested feature: there is no dedicated `openclaw notify` CLI for UI-session notifications, `chat.inject` is admin-scoped but requires a single `sessionKey` and writes a transcript message, and current chat UIs ignore `chat` events for non-active sessions.

## Evidence

- **`chat.inject` is admin-scoped:** Gateway method scopes place `chat.inject` under `operator.admin`, so the relevant primitive exists but is gated as an admin RPC rather than exposed as a simple top-level notify command.
  - file: [src/gateway/method-scopes.ts:171](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/method-scopes.ts#L171)

  - file: [src/gateway/protocol/schema/logs-chat.ts:62](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/protocol/schema/logs-chat.ts#L62)
