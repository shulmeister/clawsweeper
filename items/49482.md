## Summary

Close as implemented on main. The reported `2026.3.12` freeze pattern matches a Control UI history-reload regression that was explicitly fixed in `2026.3.13`, and current main adds multiple guardrails that bound session-history payload size and avoid in-run reload storms.

## Evidence

(#45541)` The issue reports `OpenClaw version 2026.3.12` and freezes during thinking/large tasks.

## Close Comment

Codex review: closing this as implemented on current main. The report says the problem happened on `2026.3.12`, and `CHANGELOG.md:2536` records a `2026.3.13` fix for the matching regression: stopping full chat-history reloads on every live tool result so tool-heavy runs no longer freeze/re-render storm the Control UI. Current main also keeps this path bounded in code: history reload is deferred until terminal events in `ui/src/ui/app-gateway.ts:422` and skipped mid-run in `ui/src/ui/app-gateway.ts:490`; the UI only requests 200 history messages in `ui/src/ui/controllers/chat.ts:152`; the gateway caps and truncates oversized `chat.history` payloads in `src/gateway/server-methods/chat.ts:1442` and `src/gateway/server-methods/chat.ts:1902`; and the renderer only builds the last 200 messages in `ui/src/ui/chat/build-chat-items.ts:7`. On that basis, this looks fixed well enough on main and shipped after the reporter's version.
