## Summary

Keep open. Current `main` still does not let MiniMax `web_search` use `MINIMAX_OAUTH_TOKEN`, so the reported bug is still real. The submitted PR is directionally correct but stale against current `main` and not complete enough to close conservatively: current code now has separate runtime and contract surfaces, and the PR review also caught that `envVars` would still omit `MINIMAX_API_KEY` for auto-detect/UI flows.
