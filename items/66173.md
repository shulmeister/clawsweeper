## Summary

Issue #66173 reports the Umbrel Home app-store listing lagging behind OpenClaw releases. Current main documents and automates first-party npm/git/installer and GHCR Docker release paths, but this repository has no Umbrel app manifest, store metadata, or Umbrel publish workflow to change. The action belongs to the external Umbrel listing/store administration path, not OpenClaw core code or docs.

## Best Possible Solution

Leave OpenClaw core/source unchanged. Route the Umbrel Store version lag to the external Umbrel app-store/listing administration process, while users who need current OpenClaw releases use the documented installer, `openclaw update`, package-manager, or GHCR Docker image paths.
