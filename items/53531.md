## Summary

Keep open. Current main still applies auth-profile rotation to normal text runs, but the compaction path resolves one request credential set and retries only the same compaction request; it does not implement same-provider auth-profile failover for compaction failures.

## Evidence

- **Failover docs scope auth rotation to normal runs:** The model failover doc explicitly describes auth-profile rotation for "a normal text run" and does not document compaction as participating in that same retry chain.
  - file: [docs/concepts/model-failover.md:19](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/concepts/model-failover.md#L19)
  - command: `nl -ba docs/concepts/model-failover.md | sed -n '1,120p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Compaction auth lookup is single-shot:** The compaction safeguard resolves request credentials once via `ctx.modelRegistry.getApiKeyAndHeaders(model)` and returns a cancel reason on failure; there is no per-profile retry or rotation loop here.
  - file: [src/agents/pi-hooks/compaction-safeguard.ts:195](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/pi-hooks/compaction-safeguard.ts#L195)

- **Compaction summarization retries only the same credentials:** `summarizeChunks()` retries `generateSummary()` up to 3 times with the same `apiKey`/`headers` pair. It does not call any auth-profile rotation/failover helper between attempts.
  - file: [src/agents/compaction.ts:316](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/compaction.ts#L316)
