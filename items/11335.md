## Close Comment

Closing from a Codex review because current `main` already implements this request. The agent-facing schema accepts `deviceId` for camera actions in `src/agents/tools/nodes-tool.ts:98`, `camera_snap` forwards it to the node command in `src/agents/tools/nodes-tool-media.ts:86`, and there is regression coverage in `src/agents/openclaw-tools.camera.test.ts:224`. The node-side camera contract also includes `deviceId` in `apps/shared/OpenClawKit/Sources/OpenClawKit/CameraCommands.swift:23`, and both macOS and iOS camera pickers match a specific camera by device unique ID in `apps/macos/Sources/OpenClaw/CameraCaptureService.swift:195` and `apps/ios/Sources/Camera/CameraController.swift:160`. The user docs already describe `deviceId` for `camera.snap` and show `openclaw nodes camera snap --device-id <id>` in `docs/nodes/camera.md:31` and `docs/nodes/camera.md:132`. This is also present in the shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
