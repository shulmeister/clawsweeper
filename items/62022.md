## Summary

Keep open. Current main documents and tests a configurable cap for rate-limit profile rotations, but the default still escalates to model fallback after one same-provider rotation, so the requested behavior of trying all remaining auth profiles before fallback is not implemented by default. The reporter’s workaround using auth.cooldowns.rateLimitedProfileRotations is valid and now aligns with the documented tuning knob, but it does not close the default-behavior request.
