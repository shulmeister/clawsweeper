## Summary

Keep this PR open. Current main still contains the exact `options.thinkingBudgetTokens || 1024` fallback in the non-adaptive Anthropic thinking branch, and the zero-budget value the PR guards against is still reachable from `adjustMaxTokensForThinking`. The PR is a plausible focused fix, but it still needs regression coverage before merge.

## Best Possible Solution

Keep this PR open as an implementation candidate. Before merge, add focused tests in `src/agents/anthropic-transport-stream.test.ts` for a non-adaptive Anthropic reasoning model where `thinkingBudgetTokens` resolves to `0` and for the existing undefined-to-`1024` default path, then let maintainers decide whether #63016's broader default-thinking behavior needs separate product follow-up.
