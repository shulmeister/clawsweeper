## Summary

Current `main` still appears to have the ACPX-specific gap described in #53456. The bundled `acpx` plugin is pinned to `acpx` 0.5.3, current ACP docs still advertise `claude` as an ACPX harness alias, and the ACP router skill still maps `claude` to `npx -y @zed-industries/claude-agent-acp@0.21.0`. In contrast, Claude OAuth/Max reuse is documented only for the separate `claude-cli` backend, not for ACPX. I did not find any current-main code or config surface that bridges Claude CLI OAuth into ACPX sessions, so the issue should stay open.
