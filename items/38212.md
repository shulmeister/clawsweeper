## Summary

Keep #38212 open. Current main still forwards `codex-cli/gpt-5.4` through the reported session-memory slug helper into the embedded PI model resolver without a source-backed normalization to `openai-codex`, and related fixing PRs were closed unmerged while the merged GPT-5.4 hardening does not prove this exact path is fixed.

## Best Possible Solution

Keep the issue open and implement the missing compatibility in the embedded/helper path: either normalize `codex-cli` to the `openai-codex` provider family before embedded PI model resolution/auth/thinking lookup, or add an explicit product-supported helper route for CLI-backed refs. Add focused regression coverage for `resolveModel("codex-cli", "gpt-5.4")`, auth profile ordering with `openai-codex` credentials, `listThinkingLevels("codex-cli", "gpt-5.4")`, and the session-memory slug-generator path.
