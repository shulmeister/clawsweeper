## Summary

Keep #38212 open. Current main has adjacent Codex auth alias and normal CLI-routing coverage, but the reported session-memory slug helper still forwards `codex-cli/gpt-5.4` directly into the embedded runner, whose model resolution runs before auth selection and still requires a resolvable `codex-cli` model. I did not find helper-local normalization, CLI dispatch, or slug-helper regression coverage for the reported path.

## Best Possible Solution

Keep #38212 open until the session-memory slug helper has a supported path for CLI-backed Codex defaults. A maintainer-quality fix should add focused coverage for `generateSlugViaLLM` with `codex-cli/gpt-5.4`, then either route helper runs through the same CLI backend execution path used by normal replies or explicitly normalize this helper to a supported `openai-codex` embedded model/auth route.
