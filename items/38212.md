## Summary

Keep open. Current main has adjacent progress for auth aliases and legacy `codex-cli/*` config migration, but I do not find source evidence that the reported `session-memory -> llm-slug-generator -> runEmbeddedPiAgent` helper path is fixed for a `codex-cli/gpt-5.4` primary model. The slug helper still calls the embedded runner directly, while the CLI-runtime routing that can divert `codex-cli` to `runCliAgent` exists in the normal reply path, not in this helper.

## Best Possible Solution

Keep #38212 open until the session-memory slug helper has a supported path for CLI-backed Codex defaults. A maintainer-quality fix would either normalize this helper to a working `openai-codex` embedded model/auth route or route CLI-runtime primary models through the same CLI execution path used by normal replies, then add focused regression coverage for `generateSlugViaLLM` with `codex-cli/gpt-5.4` and the runtime-migrated `embeddedHarness.runtime: codex-cli` configuration.
