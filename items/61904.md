## Summary

Current main no longer matches the reported failure mode: `file`/`filePath` are not accepted read-path aliases before execution, so a `read` call using them does not both warn and succeed. The linked fix PR #61967 was relevant, but it closed unmerged and the current implementation instead enforces canonical `path` for read execution.

## Evidence

- **Read params require canonical path only:** `REQUIRED_PARAM_GROUPS.read` contains only `path`, with no `file`, `filePath`, or `file_path` alias in the current validation contract.
  - file: [src/agents/pi-tools.params.ts:75](https://github.com/openclaw/openclaw/blob/20c3177281d292d3db208b1e875e4b51a14fec94/src/agents/pi-tools.params.ts#L75)
