## Summary

Keep #26517 open. Current main still has backend support for labeled dashboard sessions and WebChat can switch among existing sessions, but the visible WebChat new-session button still sends bare `/new`, which resets the current session instead of creating and switching to a new isolated labeled session. The proposed agent-facing create/relabel API is also not exposed. The related implementation PRs #52042 and #52746 are closed unmerged, while #51793 was closed as duplicate/superseded in favor of this issue.

## Best Possible Solution

Keep #26517 open as the canonical tracker. Reuse existing `sessions.create` and `sessions.patch` backend support to add WebChat create/switch/label management for isolated labeled sessions, and add agent-controlled create/relabel semantics only after maintainer product and safety review.
