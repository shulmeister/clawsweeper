## Summary

Keep #26517 open. Current main has backend and selector groundwork for labeled sessions, but the WebChat/Control UI still cannot create or manage multiple labeled chat sessions from the chat surface, and the agent-facing create/relabel API proposed in the issue is not shipped.

## Best Possible Solution

Keep #26517 open until maintainers either ship or explicitly narrow the WebChat product scope. The likely implementation should reuse the existing `sessions.create` and `sessions.patch` backend support, add chat UI affordances for create/switch/label management, and only add agent-controlled create/relabel semantics after maintainer review of that product and safety boundary.
