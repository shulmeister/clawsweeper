## Summary

Keep #71589 open. Current main still suppresses only explicit commentary-phase assistant output in the embedded reply path; it does not suppress unphased assistant messages that end in tool use or contain tool-call blocks, which is the central bug reported in #71588. The PR is current, open, mergeable, and its follow-up commits address the review comments about `text_end` coverage and snake_case tool-call block variants.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land or refine the narrow embedded-reply fix: suppress unphased tool-use assistant continuations from user-visible replies, preserve explicit `final_answer` output, clear already-buffered suppressed text, and keep regression coverage for both `message_end` and `text_end` reply modes.
