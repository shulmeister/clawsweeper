## Summary

Current `main` now covers the remote-approval churn reported here: the bundled Codex harness defaults to no native prompt loop for trusted local runs, guarded mode routes native approvals to an auto-reviewer instead of a host-side human prompt, and the new native-hook approval bridge adds deduping/spam protection plus normal OpenClaw approval routing.

## Evidence

- **Trusted local Codex harness runs no longer stop on native approval prompts by default:** `resolveCodexAppServerRuntimeOptions()` defaults local Codex harness sessions to `approvalPolicy: "never"`, `sandbox: "danger-full-access"`, and `approvalsReviewer: "user"`. The user-facing docs state this posture is specifically so autonomous/remote runs do not stop on native approval prompts when nobody is at the host machine.
