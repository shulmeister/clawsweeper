## Summary

Keep open. Current `main` still leaves `stringEnum` undefined on the deprecated root `openclaw/plugin-sdk` alias while exporting it only from focused subpaths and `openclaw/plugin-sdk/core`, so the reported external-plugin regression is not implemented/fixed on `main` or in the latest release commit inspected.

## Evidence

- **Direct reproduction against current main root alias:** Requiring the current root alias returns `undefined` for `stringEnum` while other legacy root helpers are present: `node -e "const sdk=require('./src/plugin-sdk/root-alias.cjs'); console.log(JSON.stringify({stringEnum: typeof sdk.stringEnum, emptyPluginConfigSchema: typeof sdk.emptyPluginConfigSchema, onDiagnosticEvent: typeof sdk.onDiagnosticEvent}))"` -> `{"stringEnum":"undefined","emptyPluginConfigSchema":"function","onDiagnosticEvent":"function"}`.
  - command: `node -e "const sdk=require('./src/plugin-sdk/root-alias.cjs'); console.log(JSON.stringify({stringEnum: typeof sdk.stringEnum, emptyPluginConfigSchema: typeof sdk.emptyPluginConfigSchema, onDiagnosticEvent: typeof sdk.onDiagnosticEvent}))"`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Root plugin SDK entry does not export `stringEnum`:** `src/plugin-sdk/index.ts` keeps the shared root surface intentionally tiny and its runtime exports end with `emptyPluginConfigSchema`, `registerContextEngine`, `buildMemorySystemPromptAddition`, `delegateCompactionToRuntime`, and `onDiagnosticEvent`; there is no `stringEnum` export on the root entry.
  - file: [src/plugin-sdk/index.ts:1](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/plugin-sdk/index.ts#L1)

That means legacy consumers of `openclaw/plugin-sdk` still cannot obtain this symbol on current `main`.
