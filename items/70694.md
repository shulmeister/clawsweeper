## Summary

Keep open. Current `main` still appears to pass `sessions_spawn.task` through the tool, subagent spawn seam, and gateway agent handler without an obvious length cap, but I could not reproduce the report in this read-only review. The reported `Stats: ... in 8 ... prompt/cache 40.6k` line is ambiguous on current `main` because OpenClaw displays prompt/cache separately from non-cached input tokens, so it is not strong enough by itself to prove the task body was dropped. There is also no focused long-task regression test on `main` that would justify closing this as already fixed or non-reproducible.
