## Summary

Keep PR #71155 open. Current main still has the original stability-bundle bug: error metadata is serialized as name/code only, so message, stack, and cause are not preserved. The PR is still useful and has maintainer signal as the preferred base for #71071 over closed duplicate #71352, but it should not be merged until the noted NUL-regex and missing-test blockers are fixed.

## Best Possible Solution

Keep #71155 open as the canonical fix path for #71071. The author should replace the literal NUL byte with an escaped regex, add explicit tests for NUL stripping and stack truncation, and then the focused serializer/test change can be reviewed for merge. If the author does not follow up, a maintainer can lift the same narrow fix onto main rather than closing the item as obsolete.
