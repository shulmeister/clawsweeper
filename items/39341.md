## Summary

Keep #39341 open. Current main still omits top-level `channels.discord.allowFrom` from the Discord native slash-command audit allowlist check, even though runtime authorization, config types, tests, and docs treat top-level Discord `allowFrom` as valid. The related fix PR #39387 was closed unmerged, and #67222 is a closed duplicate/remediation-text report rather than a shipped fix.

## Best Possible Solution

Keep #39341 as the canonical tracker. Implement a focused Discord plugin fix in `extensions/discord/src/security-audit.ts` by including normalized top-level `discordCfg.allowFrom` in `ownerAllowFromConfigured` alongside legacy `dm.allowFrom` and the pairing/store allowlist, matching runtime authorization and the Slack audit pattern. Add a regression case in `extensions/discord/src/security-audit.test.ts` proving `channels.discord.commands.native.no_allowlists` is not emitted when top-level `allowFrom` contains a Discord user ID.
