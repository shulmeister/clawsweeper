## Summary

Keep #39341 open. Current main still omits top-level `channels.discord.allowFrom` from the Discord native slash-command owner allowlist audit, even though runtime authorization, schema validation, docs, and adjacent audit logic all treat that path as valid.

## Best Possible Solution

Keep #39341 as the canonical tracker. Update `extensions/discord/src/security-audit.ts` so `ownerAllowFromConfigured` includes normalized top-level `discordCfg.allowFrom` alongside legacy `dm.allowFrom` and the store/pairing allowlist, then add a focused regression in `extensions/discord/src/security-audit.test.ts` proving `channels.discord.allowFrom` suppresses `channels.discord.commands.native.no_allowlists`. PR #39387 is useful prior art but was closed unmerged.
