## Summary

Keep #39341 open. Current main still computes the Discord native slash-command owner allowlist from legacy `dm.allowFrom` plus the pairing/store allowlist, but omits top-level `channels.discord.allowFrom`; runtime authorization, config validation, and docs continue to treat top-level `allowFrom` as valid.

## Best Possible Solution

Keep #39341 as the canonical tracker. Update `extensions/discord/src/security-audit.ts` so `ownerAllowFromConfigured` includes normalized `discordCfg.allowFrom` alongside legacy `dm.allowFrom` and the pairing/store allowlist, then add a regression in `extensions/discord/src/security-audit.test.ts` proving top-level `channels.discord.allowFrom` suppresses `channels.discord.commands.native.no_allowlists`.
