## Summary

Keep #71738 open. Current main still has the Matrix gaps the PR targets: thread identity is resolved for routing/session policy, but room-history tracking and read actions remain room-flat, and outbound threaded sends/edits still add fallback reply metadata that can surface thread traffic in the main timeline. The PR is recent, focused, and not protected/maintainer-authored; it should stay open for maintainer review and fixes to the existing review comments.

## Best Possible Solution

Keep #71738 open as the canonical Matrix thread history/reply-placement fix. The best path is to revise the PR so Matrix room history, read actions, sends, and edits consistently use thread scope, add/keep focused regression tests for those paths, and address the existing watermark and pagination review comments before maintainer merge review.
