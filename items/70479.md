## Summary

Keep #70479 open. Current main still contains the compaction-failure session reset path that this PR is trying to remove: compaction failures and embedded overflow errors call `resetSessionAfterCompactionFailure`, which uses `resetReplyRunSession` to write a new session id into the active session mapping and refresh the queued followup session. The related issue #70472 remains open, and related PR #70473 explicitly covers overflow-budget extraction rather than the later session orphaning behavior. The PR discussion is useful and active: Greptile approved the shape after the dead hook cleanup, while martingarramon left concrete checks to verify before merge, not a reason to close.

## Best Possible Solution

Keep the PR open and update it against current main. The best path is a focused patch that removes compaction-failure session rotation, preserves role-ordering reset behavior, surfaces explicit `/compact` or `/new` guidance, keeps regression coverage for embedded overflow payloads and thrown compaction-failure errors, and answers the retry-loop/factory-state review checks before landing.
