## Summary

Keep open. Current `main` has adjacent capabilities for session deletion and cleanup, but the requested agent-facing `sessions_kill` / `sessions_cleanup` tool surface is still not present. The documented agent session tools remain `sessions_list`, `sessions_history`, `sessions_send`, `sessions_spawn`, `sessions_yield`, `subagents`, and `session_status`, and the only built-in kill action exposed to agents is limited to requester-owned spawned subagents.

## Evidence

- **Agent session tools do not include kill/cleanup:** The session tools docs list only `sessions_list`, `sessions_history`, `sessions_send`, `sessions_spawn`, `sessions_yield`, `subagents`, and `session_status`; there is no `sessions_kill` or `sessions_cleanup`.
