## Summary

Keep PR #39726 open. Current main still requires `tools.exec.notifyOnExitEmptySuccess` to be explicitly true, so chat-context background execs that exit 0 with empty stdout/stderr remain silent by default. The PR remains a focused behavior change, and its latest head addresses the earlier review concerns around explicit opt-out and generic provider labels.

## Best Possible Solution

Keep PR #39726 open or rework it onto current main. The best path is the narrowed PR behavior: default empty-success notifications only for real reply-capable chat channels when the config value is unset, preserve explicit `false`, keep generic provider labels like `openai` silent, and land the regression coverage with any accepted changelog/docs update.
