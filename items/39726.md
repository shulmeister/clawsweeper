## Summary

Keep PR #39726 open. Current main still only notifies empty successful background execs when `notifyOnExitEmptySuccess` is explicitly true, so the PR's chat-context default behavior has not been implemented or superseded.

## Best Possible Solution

Keep PR #39726 open or rebase/rework it onto current main. The best implementation is the PR's narrowed behavior: default `notifyOnExitEmptySuccess` from a real reply-capable chat channel only when the config value is unset, preserve explicit `false`, keep generic provider labels like `openai` and `heartbeat` silent, add the regression cases, and update changelog/docs if maintainers accept the product behavior.
