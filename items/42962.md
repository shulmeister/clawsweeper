## Summary

Keep PR #42962 open. The linked bug is still plausible on current main, and the PR is not obsolete, but the submitted fix is incomplete: current runtime marks `gateway` `config.schema.lookup` as mutating, so a payload-level suppression that only fires when `mutatingAction === false` would not hit the real error path.

## Best Possible Solution

Keep the PR or linked issue #42856 open and fix the actual classification path: make `gateway` `config.schema.lookup` non-mutating in `src/agents/tool-mutation.ts` or add an equivalent targeted warning-policy exception that matches the real runtime state. Add regression coverage for the suppress case plus inverse cases for mutating gateway actions so warning suppression does not hide real write/restart failures.
