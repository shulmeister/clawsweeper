## Summary

Current main still appears to have the reported strict-agentic planning-only retry gaps. The user-prompt gate remains allowlist-based and misses the reported imperative phrasing, while the assistant planning-only detector still misses present-continuous narration such as "I'm running it now."

## Best Possible Solution

Keep the issue open. The best fix belongs in `src/agents/pi-embedded-runner/run/incomplete-turn.ts`: add focused regression tests for the reported imperative prompt and for an accepted ack prompt followed by present-continuous zero-tool narration, then conservatively broaden actionable prompt detection and planning-only narration detection without retrying casual chatter or completed-result summaries.
