## Summary

Keep #66312 open. Current main still does not thread `sessionKey` into the system-prompt runtime metadata or render it in the `Runtime:` line. The linked issue #66292 remains a real core prompt/runtime request, and the PR is a narrow additive implementation rather than obsolete, ClawHub-scoped, or clearly superseded. Current session-tool support for `sessionKey: "current"` partially mitigates the motivation, but it does not implement the requested Runtime-line signal.

## Best Possible Solution

Keep the PR open, or have a maintainer land an equivalent current-main patch that adds optional `sessionKey` to the runtime prompt parameter types, passes `params.sessionKey` from `runEmbeddedAttempt()` into `buildSystemPromptParams()`, renders a stable `sessionKey=...` segment in `buildRuntimeLine()`, and includes focused Runtime-line regression coverage. If maintainers prefer the existing `sessionKey: "current"` tool convention instead, that should be an explicit product decision documented in the PR rather than an automated cleanup close.
