## Summary

Keep open. Current main still does not provide the requested Control UI file-explorer toggle, workspace tree/sidebar, file search/filter, context menu, or drag-to-chat flow. The closest shipped surface is the Agents "Core Files" editor, but it is intentionally limited to a fixed set of bootstrap/workspace instruction files and is not a general workspace browser. Related #41579 was also kept open for the same broader gap rather than treated as implemented or ClawHub-closeable.

## Best Possible Solution

Keep the issue open as a real Control UI/product enhancement. A good implementation would add a deliberately scoped workspace browser API with path safety, permissions, search/filter, and refresh semantics, then wire a Control UI sidebar/topbar toggle and chat insertion/open actions. If maintainers want this outside core, the prerequisite is a Control UI extension point or a canonical plugin UI integration design, not an auto-close to ClawHub today.
