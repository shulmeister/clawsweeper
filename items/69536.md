## Summary

Keep open. Current main still has the reported dreaming/backfill mismatch: the bundled session-memory hook writes `memory/YYYY-MM-DD-slug.md`, while live daily dreaming ingestion, grounded `memory rem-backfill`, and short-term promotion eligibility still recognize bare `YYYY-MM-DD.md` daily-note paths. General memory search and the #46687 startup-context fix are useful partial mitigations, but they do not make hook-generated slugged files participate as daily notes in the dreaming promotion path.

## Best Possible Solution

Keep the issue open and resolve the memory-core/session-memory boundary. The conservative fix is either to make `session-memory` append structured session sections to canonical `memory/YYYY-MM-DD.md` daily notes, or to broaden memory-core daily/backfill discovery to include date-prefixed slugged files while preserving source paths, deterministic ordering, dedupe state, and exclusions for generated dreaming artifacts. Add focused regression coverage for live light-phase ingestion and `memory rem-backfill --path ./memory --stage-short-term` with a `YYYY-MM-DD-slug.md` input, then update hook and memory/dreaming docs to state the file contract clearly.
