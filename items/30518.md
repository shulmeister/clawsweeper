## Summary

Keep #30518 open. Current main still exposes Telegram delivery-time pinning and low-level Bot API pin/unpin helpers, but it does not advertise or dispatch manual generic `pin` / `unpin` message actions for Telegram, which is the requested capability.

## Best Possible Solution

Keep the issue open and implement this in the bundled Telegram plugin: add Telegram `pin` and `unpin` to action discovery, map them in `TELEGRAM_MESSAGE_ACTION_MAP`, add action-runtime aliases/branches that call the existing `pinMessageTelegram` and `unpinMessageTelegram` helpers, add focused adapter/runtime contract tests, and update the CLI plus Telegram docs to list manual Telegram pin/unpin actions separately from delivery-time `--pin`.
