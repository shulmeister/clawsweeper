## Summary

Keep #30518 open. Current main still supports Telegram delivery-time pinning and has low-level Bot API helpers for pin/unpin, but it does not expose manual Telegram `pin` or `unpin` message actions for existing messages through the generic message action surface requested by the issue.

## Best Possible Solution

Keep the issue open and implement the capability inside the bundled Telegram plugin: add Telegram `pin` and `unpin` to action discovery, map them in `TELEGRAM_MESSAGE_ACTION_MAP`, route them in `action-runtime.ts` to the existing `pinMessageTelegram` / `unpinMessageTelegram` helpers, add focused adapter/runtime contract tests, and update the CLI and Telegram docs to list Telegram support for manual pin/unpin actions.
