## Summary

Keep #30518 open. Current main still does not expose manual Telegram `pin`/`unpin` message actions: shared action names and low-level Telegram Bot API helpers exist, and Telegram send-time `delivery.pin` is implemented, but the bundled Telegram message action discovery/map/runtime omit `pin` and `unpin`. The prior ClawSweeper comment reached the same conclusion, and current main SHA 82262387651ab270b565f71533ce4937c9c00dc6 still shows the gap.

## Best Possible Solution

Keep the issue open. Implement this in the bundled Telegram plugin by adding `pin` and `unpin` to Telegram message action discovery and `TELEGRAM_MESSAGE_ACTION_MAP`, adding runtime aliases and dispatch branches that call `pinMessageTelegram` and `unpinMessageTelegram`, deciding whether a `channels.telegram.actions.pins` gate/config-doc entry is needed, adding adapter/runtime contract tests, and updating Telegram docs to distinguish manual pin/unpin from send-time `delivery.pin`.
