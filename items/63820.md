## Summary

Current `main` already contains the channel/session-reset safeguards this issue asks for, and the same behavior is present in the tagged `v2026.4.23` release: automated heartbeat/cron/exec runs no longer advance freshness for chat sessions, channel-specific reset policy is resolved from the active message channel, and regression tests/docs cover the expected behavior.

## Evidence

- **System events no longer drive session freshness:** `initSessionState` classifies `heartbeat`, `cron-event`, and `exec-event` as system events so they do not trigger binding retargeting or implicit reset behavior for chat sessions.
