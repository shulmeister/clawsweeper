## Summary

Keep PR #40618 open. Current main still hardcodes bundled `session-memory` writes to `<workspace>/memory/`, while the PR addresses a concrete user-reported conflict where generated session snapshots pollute durable memory/search notes. The work is not implemented, not duplicated by the provided related items, and has a focused path forward, but it should address the unresolved path-safety review comment before merge.

## Best Possible Solution

Keep the PR open and request a small revision: preserve the backward-compatible default of `memory`, add `outputDir` as a workspace-relative per-hook config, validate traversal/absolute/empty values, write through `writeFileWithinRoot` rooted at the workspace with a relative `outputDir/filename`, avoid any pre-write `fs.mkdir` that can follow symlinks outside the workspace, and add regression tests for custom directories, invalid traversal fallback, and symlink alias behavior. Update the English hook docs/metadata only; localized docs should not be edited in this repo per `docs/AGENTS.md`.
