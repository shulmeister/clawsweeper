## Summary

Keep #60736 open. Current main still treats `google-vertex` mostly as a Google provider alias and does not implement the requested standalone Vertex provider, onboarding location collection, default/fallback location handling, or Vertex project/location URL construction. The related PRs #60860 and #62103 are relevant and promising, but both are still open in the provided GitHub context, so the issue should remain as the canonical user-facing tracker until an implementation lands.
