## Summary

Keep #42009 open. Current main still derives an implicit agent main-session key before considering `--to` when `--agent` is supplied, so `openclaw agent --agent <id> --to <target>` still collapses to `agent:<id>:main` instead of an agent-scoped `--to` key. The current tests cover `--agent + --session-id` scoping, but not the PR's `--agent + --to` regression.

## Best Possible Solution

Rebase or reimplement the PR on current main in `src/agents/command/session.ts`: keep explicit `--session-key` precedence, avoid deriving the implicit agent main-session key when `--to` is present, bind the normalized `--to` request key to the selected agent, and add focused regression coverage for `--agent + --to` plus the existing guardrails.
