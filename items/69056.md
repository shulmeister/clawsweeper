## Summary

Keep PR #69056 open. Current main still has a no-listener Windows `SIGUSR1` fallback that can hit Node's unsupported-signal behavior, and the PR addresses a real reported Windows restart failure. It is not ready to close as implemented, and related PR #68853 overlaps without fully superseding this PR's `emitGatewayRestart()`/unmanaged restart surface; the latest PR head also still has an unresolved token-consumption review blocker.

## Best Possible Solution

Keep #69056 open for maintainer review, but do not merge it as-is. The best product/code path is a maintainer-chosen combined gateway restart fix that covers Windows no-listener and unmanaged restart paths, preserves the in-process listener path, consumes or rolls back restart tokens on every fallback outcome, and adds targeted Windows regression tests. If #68853 becomes canonical, it should explicitly absorb this PR's remaining Windows SIGUSR1 surfaces before #69056 is closed as superseded.
