## Summary

Keep this issue open. Current main still runs `agent_end` through the generic void hook runner with no hook-level timeout, and the call sites only log rejected promises; a never-settling hook promise still has no warning path. Related memory embedding timeout work is separate and does not wrap typed hook execution.

## Best Possible Solution

Keep this issue open. Add a generic typed-hook timeout policy in `src/plugins/hooks.ts` or a narrow `agent_end` timeout wrapper first, document the default and any per-hook override on the plugin API, decide whether hooks receive an `AbortSignal`, and add regression coverage proving a never-settling `agent_end` handler produces a warning without requiring every plugin author to implement their own timeout.

## Risks / Open Questions
