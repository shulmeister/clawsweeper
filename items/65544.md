## Summary

Keep open. Current `main` still runs `agent_end` hooks without any framework timeout or abort contract, and the CLI harness only logs rejected promises. A hook handler that never resolves can therefore stall post-turn work indefinitely with no timeout-driven warning, which matches the issue's core report.

## Evidence

- **Void hooks have no timeout wrapper:** `runVoidHook` executes each handler and awaits `Promise.all(promises)` with no `AbortSignal`, deadline, or timeout path.
  - file: [src/plugins/hooks.ts:319](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/plugins/hooks.ts#L319)

  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **`agent_end` is dispatched fire-and-forget and only logs rejections:** The harness calls `void hookRunner.runAgentEnd(...).catch(...)`; a never-settling promise never reaches this `.catch`, so there is no warning path for a hung handler.
  - file: [src/agents/harness/lifecycle-hook-helpers.ts:42](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/agents/harness/lifecycle-hook-helpers.ts#L42)
