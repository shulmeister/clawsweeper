## Summary

Keep this issue open. Current main has prompt guidance and a guarded first-class gateway config tool, but the underlying exec path still permits model-issued OpenClaw config and lifecycle shell commands without confirmation under the documented default gateway/node host policy.

## Best Possible Solution

Keep this open for a maintainer/security fix. Add a hard runtime guard at the agent exec boundary that blocks or requires explicit approval for model-initiated OpenClaw config, model-selection, lifecycle, and self-management mutations such as `openclaw config set`, `openclaw models set`, `openclaw gateway restart`, and uninstall/update equivalents unless there is a user-approved config-change action. Add regression coverage for ambiguous model-switch prompts that may show example commands but must not execute them automatically, and coordinate the broader destructive-command policy with #65307.
