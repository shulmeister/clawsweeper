## Summary

Current `main` does not contain the helper change from this PR. `resolveBundledInstallPlanForCatalogEntry` still rejects a bundled plugin when the bundled source carries a scoped `npmSpec` and the catalog entry supplies the bare plugin id, and that helper is still used by onboarding/channel-setup resolution. The PR body's exact `openclaw plugins install acpx` reproduction is weaker on current `main` because bare CLI specs now short-circuit to bundled installs before ClawHub/npm, but that is not strong enough to close this PR as already implemented.
