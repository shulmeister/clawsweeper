## Summary

Keep #29736 open. Current main still resolves exec approvals file/socket through `~/.openclaw` expansion using `OPENCLAW_HOME` or OS home, not the active `OPENCLAW_STATE_DIR`/profile state root. Gateway and node-host approvals RPCs call the same store helpers, and docs/tests still describe and assert home-based storage.

## Best Possible Solution

Keep the issue open and fix the core exec approvals path contract. The likely implementation should derive the default approvals file and socket from the active state directory/profile semantics, update gateway/node/local CLI behavior consistently, preserve any needed compatibility for existing `~/.openclaw/exec-approvals.json`, and add regression tests for `OPENCLAW_STATE_DIR`, profiles, and container-style writable state roots. Docs at `docs/cli/approvals.md` and `docs/tools/exec-approvals.md` should be updated once the product direction is settled.
