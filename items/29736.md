## Summary

Keep #29736 open. Current main still resolves exec approvals file/socket defaults through home-relative `~/.openclaw` paths rather than the active `OPENCLAW_STATE_DIR` or `--profile` state root. The issue comments include concrete Linux service, profile-isolation, and container reproductions; the later “works in 2026.4.11” comment does not match the current TypeScript source, docs, and tests.

## Best Possible Solution

Keep the issue open and fix the core exec approvals path contract. Default approvals file and socket resolution should derive from the active state root/profile semantics, likely through `resolveStateDir()`, across CLI local mode, gateway RPC, headless node-host management, and runtime enforcement. Add migration/fallback behavior for existing home-backed approvals and regression coverage for `OPENCLAW_STATE_DIR`, `--profile`, container writable-state, and `OPENCLAW_HOME` compatibility.
