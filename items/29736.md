## Summary

Keep #29736 open. Current main still resolves exec approvals through `~/.openclaw` expansion using `OPENCLAW_HOME`/OS home, not the active `OPENCLAW_STATE_DIR` or profile-derived state root. The local CLI and gateway RPC paths both use the same store helpers, and current docs/tests still describe or assert home-based storage.

## Best Possible Solution

Keep the issue open and fix the core exec approvals path contract. The likely fix should derive the default approvals file and socket from the active `resolveStateDir()` / profile semantics, preserve a compatibility or migration path for existing `~/.openclaw/exec-approvals.json`, update local CLI, gateway, and node-host behavior consistently, and add regression tests for `OPENCLAW_STATE_DIR`, `--profile`, `OPENCLAW_HOME`, symlink safety, and container-style writable state roots. Update `docs/cli/approvals.md` and `docs/tools/exec-approvals.md` after the intended storage contract is settled.
