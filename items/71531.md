## Summary

Keep #71531 open. Current main at 9089e6b595ba33b4afc27b99c56ace80c4fbbc7b still has no `channels.slack.logLevel` config field, does not forward any Slack log level into Bolt, and related PR #71550 is open/unmerged rather than shipped behavior.

## Best Possible Solution

Keep the issue open and use PR #71550, or an equivalent maintainer-reviewed patch, as the implementation path. The fix should add a schema-validated and typed `channels.slack.logLevel` surface aligned with Slack's `LogLevel` values, forward it into the Bolt App construction only when configured, preserve omitted/default behavior, add focused regression tests, and update public/generated config docs as required by the config contract.
