## Summary

Keep open. Current main has not implemented the PowerShell Register-ScheduledTask path, and this PR targets a real core Windows daemon surface rather than ClawHub or external administration. However, the latest PR patch is not safely obsolete or merge-ready: it conflicts with current documented Startup-folder fallback behavior and still has source-backed review concerns in the latest diff, including invalid schtasks fallback arguments and uninstall cleanup regression.

## Best Possible Solution

Keep the PR open for maintainer review rather than closing it. The best path is a revised Windows daemon patch that preserves or deliberately updates the documented fallback contract, removes the invalid schtasks `/I 0` fallback argument, keeps uninstall cleanup after PowerShell task removal, fixes the `launchFallbackTaskScript` call shape, and adds focused tests for the PowerShell-first create/update/run/stop/uninstall behavior plus schtasks fallback behavior.

## Risks / Open Questions

- Merging as-is could regress documented non-admin Windows fallback installs and uninstall cleanup for users who previously had a Startup-folder launcher.
- The PR changes Windows daemon installation policy from best-effort per-user fallback to mandatory elevation, so the desired product behavior should be explicitly confirmed by maintainers before accepting the implementation.
