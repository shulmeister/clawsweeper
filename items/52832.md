## Summary

Keep open. Current `main` does not already implement this iOS bottom-bar redesign, and the PR still needs maintainer judgment because its switch from a reserved bottom inset to a pure overlay appears to conflict with the current web canvas inset model.

## Risks / Open Questions

- The PR's main UI request is not already shipped on `main`, so closing as implemented would be incorrect.
- There is a plausible behavioral regression in the proposed overlay approach: bottom-of-page content may render underneath the toolbar because the current stack depends on the reserved `safeAreaInset` plus safe-area-only HTML padding.
- The PR also carries an unrelated provider-registry commit in its GitHub context (`Restore deepseek provider contract registry entry`), which makes the branch less suitable for automatic maintainer cleanup without further review.
