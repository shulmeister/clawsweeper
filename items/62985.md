## Summary

Keep open. Current main still does not accept `agentId` inside `channels.telegram.accounts.<id>`, and #63380 is still open and unmerged as the implementation candidate. The issue discussion has a valid workaround using top-level `bindings`, but that documents the new route rather than restoring compatibility for configs that worked in 2026.4.5.

## Best Possible Solution

Review #63380 as the implementation candidate. If maintainers want backward compatibility for configs that worked in 2026.4.5, land a focused schema/type/runtime-compatible fix with regression coverage for `channels.telegram.accounts.<id>.agentId`. If maintainers want `bindings` to remain the only supported route, close the PR with that rationale and improve docs or doctor output so upgraded users can find the migration path.
