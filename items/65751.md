## Summary

Current main still routes the Slack setup manifest through the framed CLI note path, so the reported copy/paste bug remains valid. Two open PRs, #65773 and #65809, explicitly reference this issue with closing syntax and propose the same fix direction, so this issue should stay open until one implementation lands or is superseded by a maintainer patch.

## Best Possible Solution

Keep this issue open and review/land one canonical fix, preferably in the Slack plugin setup surface: split the manifest JSON from `introNote.lines`, emit the raw JSON through an appropriate setup `prepare`/plain-output path, and add a focused regression test that the CLI setup note does not contain framed manifest JSON. After the chosen PR or equivalent maintainer patch merges, GitHub can close this issue as implemented.
