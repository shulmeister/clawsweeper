## Summary

Current main still embeds the Slack manifest JSON in the setup wizard intro note, which is rendered by the CLI note path. The related PRs #65773 and #65809 correctly target the same bug, but neither is merged in the provided context, so the issue remains the canonical open tracker.

## Best Possible Solution

Keep #65751 open until one fix lands. The best path is to remove the manifest JSON from `introNote.lines`, print a raw parseable JSON manifest outside the framed note using the existing setup `prepare` hook or an explicit plain-output prompter API, and add a focused regression test that proves the Slack setup manifest is copyable JSON rather than note-framed text.
