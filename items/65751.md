## Summary

Keep open. Current `main` still builds the Slack manifest into the setup intro note and renders that note through the boxed terminal note UI, so the JSON remains framed with note borders instead of being emitted as raw copy-pasteable JSON.

## Evidence

- **Slack setup still embeds manifest in intro note:** `buildSlackSetupLines()` appends `"Manifest (JSON):"` and `buildSlackManifest(botName)` to the note lines shown during Slack setup, so the manifest is still part of the note payload rather than a separate raw output.
  - file: [extensions/slack/src/setup-shared.ts:85](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/extensions/slack/src/setup-shared.ts#L85)
  - command: `nl -ba extensions/slack/src/setup-shared.ts | sed -n '1,220p'`
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **Setup wizard renders intro notes via note UI:** When the Slack intro note is shown, the setup wizard calls `prompter.note(wizard.introNote.lines.join("\n"), wizard.introNote.title)`, so every Slack setup line, including the manifest JSON, is sent through the note renderer.
  - file: [src/channels/plugins/setup-wizard.ts:240](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/channels/plugins/setup-wizard.ts#L240)
