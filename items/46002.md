## Summary

Keep #46002 open. Current main has generic pre-tool block flush machinery, but the Feishu dispatcher still explicitly disables generic blockStreaming and suppresses internal block payload delivery. That means the exact reported Feishu `blockStreamingBreak: "text_end"` path is not proven fixed for Feishu, and the related PR #46019 was closed unmerged in the provided GitHub context while commenters later still reported the issue.

## Evidence

- **Feishu disables generic blockStreaming:** Feishu replyOptions unconditionally set `disableBlockStreaming: true`, so configured agent-level blockStreaming is forced off for this channel path.
