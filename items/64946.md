## Summary

Keep open. Current `main` still contains the false-positive plugin-config validation path. The reporter’s “wrapped vs top-level schema mismatch” theory is not supported by current code, but the underlying log-spam bug remains plausible and is not fixed on `main`.

## Evidence

- **Missing plugin config is coerced to `{}` before validation:** `validatePluginConfig()` still calls `validateJsonSchemaValue()` with `value: params.value ?? {}` and `applyDefaults: true`. If a snapshot/preflight path reaches validation with `entry?.config === undefined`, required-property schemas will emit false `invalid config` errors instead of deferring.
