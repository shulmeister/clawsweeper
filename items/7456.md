## Summary

Keep open. Current main does not implement general back navigation for `openclaw onboard`: the wizard prompt contract has no back sentinel/allowBack option, the Clack prompter only maps cancel to `WizardCancelledError`, and `runSetupWizard` still advances through a mostly linear sequence. The prior implementation PR #7762 was closed unmerged, so it does not supersede the issue.

## Evidence

- **Wizard prompt contract lacks back navigation API:** `WizardSelectParams`, `WizardMultiSelectParams`, `WizardTextParams`, and `WizardConfirmParams` expose initial values/search/validation only; `WizardPrompter` returns selected values directly with no back sentinel or navigation result type.
  - file: [src/wizard/prompts.ts:7](https://github.com/openclaw/openclaw/blob/0bcb4c95c185909b12d463b04d49b16ead3459d5/src/wizard/prompts.ts#L7)
