## Summary

Keep open. Current main still has no wizard-wide Back navigation for `openclaw onboard`: the interactive command enters a linear `runSetupWizard` path, the prompt contract and Clack adapter have no back sentinel or `allowBack` result, and the gateway wizard protocol exposes next/answer/cancel/status only. The issue discussion also already has an April 26, 2026 maintainer comment explicitly keeping #7456 open, while PR #7762 was closed unmerged and remains useful prior art rather than a shipped fix.

## Best Possible Solution

Implement a typed wizard navigation contract in core: add an explicit Back/previous-step result to the CLI prompter and gateway wizard APIs, refactor `runSetupWizard` into a state machine that preserves answers across backward and forward movement, and align local submenu Back behavior with the general navigation model. PR #7762 can inform the design but should not be treated as canonical because it was closed unmerged with type-safety and completeness concerns.
