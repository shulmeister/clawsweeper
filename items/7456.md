## Summary

Keep open. Current main still lacks wizard-wide Back navigation for `openclaw onboard`, and the latest maintainer discussion already explicitly kept this issue open. The related implementation PR #7762 was closed unmerged, so it is useful prior art but not a shipped fix.

## Best Possible Solution

Implement a typed wizard navigation contract instead of ad hoc menu entries: add an explicit Back result or previous-step operation to the CLI prompter and gateway wizard APIs, refactor `runSetupWizard` into a state machine that preserves answers while moving backward and forward, and keep local submenu Back behavior aligned with the new general navigation model. PR #7762 can inform the design, but it should not be treated as canonical because it was closed unmerged with type-safety and completeness concerns.
