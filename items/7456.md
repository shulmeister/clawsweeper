## Summary

Keep #7456 open. Current main still does not implement wizard-wide Back navigation for `openclaw onboard`: the CLI path enters a linear `runSetupWizard`, the prompter contract and Clack adapter expose values/cancellation but no back sentinel or navigation result, and the gateway wizard protocol supports start/next/answer/cancel/status only. The issue discussion also includes an April 26, 2026 maintainer comment explicitly keeping this open, while PR #7762 was closed unmerged and remains useful prior art rather than a shipped fix.

## Best Possible Solution

Keep #7456 open and implement a typed wizard navigation contract in core: add an explicit Back/previous-step result to the CLI prompter and gateway wizard APIs, refactor `runSetupWizard` into a state machine that preserves answers across backward and forward movement, and align the existing local submenu Back behavior with that general model. PR #7762 can inform the design, but it should not be treated as canonical because it was closed unmerged with type-safety and completeness concerns.
