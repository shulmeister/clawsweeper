## Summary

Keep #7456 open. Current main still does not implement shared Back/previous navigation for `openclaw onboard`: the CLI prompt contract, Clack adapter, setup wizard flow, WizardSession/RPC protocol, macOS wizard client, and docs remain forward/cancel oriented. The direct go-back PR #7762 was closed unmerged, #5880 addressed interrupted-session resume rather than in-session backtracking, and #18286 was a separate closed dialog-tool proposal rather than a shipped onboarding wizard fix.

## Best Possible Solution

Keep #7456 open. The best implementation is an additive typed Back/previous navigation contract across WizardPrompter, the Clack adapter, WizardSession state/history, gateway wizard schemas and handlers, generated/app clients, and CLI/macOS/Control UI rendering. It should preserve answers while moving backward and forward and add regression coverage for CLI and RPC wizard flows. PR #7762 is useful prior art but should not be treated as shipped.
