## Summary

Keep #13634 open. Current main already covers the workspace-file injection half through the shared Project Context/bootstrap pipeline, but the requested account/global Slack `channels.slack.systemPrompt` surface is still missing from Slack types, strict schema, docs/config metadata, and runtime prompt plumbing. Slack currently supports only per-room channel prompts via `channels.slack.channels.<id>.systemPrompt` plus the `"*"` wildcard fallback; DMs and MPIM/group DMs still have no global Slack prompt path.

## Best Possible Solution

Keep #13634 open. Implement an account-level `channels.slack.systemPrompt` for Slack config types, strict schema, generated metadata/UI hints, docs, and runtime prompt plumbing. Preserve existing per-channel and wildcard behavior with explicit precedence: exact channel entry, wildcard channel entry, then account/global prompt. Apply the account/global prompt consistently for Slack messages and slash commands across public channels, private channels, DMs, and MPIM/group DMs. Continue using the shared Project Context bootstrap pipeline for AGENTS.md/SOUL.md/USER.md/MEMORY.md rather than adding Slack-specific workspace-file injection.
