## Summary

Keep open. Current main covers workspace bootstrap-file injection as generic agent-runtime behavior, but the requested global Slack `channels.slack.systemPrompt` remains missing. Slack prompt configuration is still per-channel only, and direct messages/group DMs do not get a global Slack prompt path. The existing maintainer/Codex comment already reached the same conclusion, and PR #62792 is about Slack file/media ingestion, not prompt/config parity.

## Best Possible Solution

Keep #13634 open until Slack adds an accepted top-level/account-level `systemPrompt` config with documented precedence, runtime plumbing for channels, group DMs, and DMs, and focused tests. Treat the workspace bootstrap-file injection portion as already solved generically by the agent runtime; the remaining product work is the Slack global prompt surface.
