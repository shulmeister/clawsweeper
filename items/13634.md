## Summary

Keep #13634 open. Current main covers workspace bootstrap-file injection through the generic agent runtime, but Slack still only supports `systemPrompt` on per-channel entries and only applies that path to room/channel messages. The requested global `channels.slack.systemPrompt` surface remains missing; related PR #62792 is about Slack file/media ingestion, not prompt/config parity.

## Best Possible Solution

Keep the issue open and implement a Slack account/top-level `systemPrompt` surface with documented precedence against explicit channel entries and wildcard entries, then plumb it through message and slash paths for channels, MPIM/group DMs, and DMs. Do not add Slack-specific bootstrap-file injection; keep using the generic workspace bootstrap context pipeline already present in the agent runtime.
