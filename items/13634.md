## Summary

Keep #13634 open. Current main has generic Project Context workspace bootstrap injection and Slack supports per-channel/wildcard `systemPrompt` for room channels, but the requested account/global `channels.slack.systemPrompt` surface is still absent from Slack types, strict schema, docs, and runtime plumbing. DMs/MPIMs and slash-command paths still do not have a global prompt fallback. Related PR #62792 is Slack file/media ingestion scope, not prompt/config parity.

## Best Possible Solution

Keep #13634 open. Implement account-level `channels.slack.systemPrompt` in Slack config types, strict schema, generated config metadata/UI hints, docs, and runtime context. Define precedence as exact channel entry, then wildcard channel entry, then account/global prompt, and apply it consistently for Slack message and slash-command handling across channels, private channels, DMs, and MPIM/group DMs. Preserve the existing generic Project Context bootstrap pipeline rather than adding Slack-specific workspace file injection.
