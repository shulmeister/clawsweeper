## Summary

Keep issue #72265 open. The requested docs clarification is still actionable on current main: the README quick start still places daemon install immediately before a foreground gateway command, the README still links to `/logging`, and the Node/gateway troubleshooting docs do not include the proposed service-runtime Node verification checklist. The QEEK-AI comment usefully describes the intended fix, but those edits are not present in the checked-out main tree.

## Best Possible Solution

Update the README quick start to split daemon mode from foreground/debug mode, change README logging links to the canonical gateway logging URL where appropriate, and add a concise service-runtime verification section to `docs/install/node.md` plus a matching diagnostic ladder entry in `docs/gateway/troubleshooting.md` explaining shell Node versus service Node drift and restarting the gateway after Node changes.
