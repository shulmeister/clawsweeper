## Summary

Keep #25295 open. Current main still does not expose Mattermost `message action=edit` or `message action=delete`; the shipped Mattermost update/delete helpers are internal preview-streaming/model-picker primitives, not the user-facing message action surface targeted by this PR.

## Best Possible Solution

Keep #25295 open. A focused refreshed implementation should wire Mattermost `edit` and `delete` through `describeMessageTool()`, `supportsAction()`, and `handleAction()` in `extensions/mattermost/src/channel.ts`, reuse the existing Mattermost post update/delete helpers, add focused Mattermost action tests, and update the Mattermost/message docs. Split out unrelated gate-fix churn before merge.
