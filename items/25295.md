## Summary

Keep open. Current main still lacks the Mattermost user-facing `message` tool `edit` and `delete` action surface that this PR targets; existing Mattermost update/delete helpers are internal preview-streaming primitives, not advertised or dispatched message actions.

## Best Possible Solution

Keep #25295 open until Mattermost exposes `edit` and `delete` through `describeMessageTool()`, `supportsAction()`, and `handleAction()`, with focused Mattermost tests and docs. A refreshed implementation should reuse the current Mattermost account/runtime patterns and existing post update/delete helpers while dropping unrelated provider/loader/test-gate churn from the stale branch.
