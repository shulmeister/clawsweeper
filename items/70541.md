## Summary

Keep #70541 open. Current main still contains the reported auto-compaction follow-up path that reads post-compaction context from process.cwd() instead of the run workspace. The helper itself correctly treats its first argument as the AGENTS.md workspace root, and the preflight compaction path already uses followupRun.run.workspaceDir, so the issue is a focused real bug rather than an obsolete report.

## Best Possible Solution

Keep the issue open. The narrow fix should change the auto-compaction branch in src/auto-reply/reply/agent-runner.ts to call readPostCompactionContext with followupRun.run.workspaceDir, ideally with a defensive fallback only if maintainers want one. Add a regression test around the runReplyAgent auto-compaction path where process.cwd() and run.workspaceDir differ, AGENTS.md exists only in run.workspaceDir, and the queued post-compaction context includes the workspace file content.
