## Summary

Keep #70541 open. Current main still contains the reported post-compaction enqueue path using `process.cwd()` instead of the run workspace, while the context reader resolves `AGENTS.md` relative to its workspace argument and another refresh path already uses `followupRun.run.workspaceDir`.

## Best Possible Solution

Patch `src/auto-reply/reply/agent-runner.ts` so the auto-compaction completion hook passes `followupRun.run.workspaceDir` to `readPostCompactionContext` (with a defensive fallback only if maintainers want one), keep the existing agent-id resolution, and add a regression test where process cwd differs from the configured workspace and only the workspace `AGENTS.md` is injected.
