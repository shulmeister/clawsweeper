## Summary

Keep open. Current main still forwards `options.temperature` into Anthropic Messages payloads whenever thinking is disabled, with no Claude Opus 4.7 guard, while the PR is a focused attempt to fix a real request-shaping bug reported with live v2026.4.24 logs. This is not implemented on main and is not safe to auto-close as stale or non-actionable.

## Best Possible Solution

Keep this PR open and use it or a maintainer-authored equivalent as the narrow fix: omit `temperature` from Anthropic Messages request payloads for `claude-opus-4-7`, add the regression test, and decide whether to also normalize model IDs and strip the resolved option earlier for consistency with the broader #72301 discussion.
