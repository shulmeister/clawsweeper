## Summary

Keep open. Current `main` still matches the report: Active Memory excludes `explicit` at the type level, strips it during config normalization, rejects it in the plugin manifest schema, and never resolves `:explicit:` session keys.

## Evidence

- **Chat-type union excludes `explicit`:** `ActiveMemoryChatType` is still defined as only `"direct" | "group" | "channel"`, and the plugin config types use the same restricted set.
  - file: [extensions/active-memory/index.ts:171](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/extensions/active-memory/index.ts#L171)
  - command: `rg -n "type ActiveMemoryChatType|ActiveMemoryChatType =" extensions/active-memory/index.ts && nl -ba extensions/active-memory/index.ts | sed -n '55,130p'`
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **Runtime normalization silently drops `explicit`:** `normalizePluginConfig()` filters `allowedChatTypes` down to `direct/group/channel` only, then falls back to `["direct"]` when nothing survives, which reproduces the silent-strip behavior described in the issue.
  - file: [extensions/active-memory/index.ts:612](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/extensions/active-memory/index.ts#L612)
  - command: `nl -ba extensions/active-memory/index.ts | sed -n '607,630p'`
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **Session resolution still has no `:explicit:` branch:** `resolveChatType()` recognizes `:group:`, `:channel:`, `:direct:`, and `:dm:` only; there is no handling for `:explicit:` session keys.
  - file: [extensions/active-memory/index.ts:874](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/extensions/active-memory/index.ts#L874)
  - command: `nl -ba extensions/active-memory/index.ts | sed -n '874,925p'`
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **Manifest schema still rejects `explicit`:** `openclaw.plugin.json` still validates `allowedChatTypes` against `['direct', 'group', 'channel']` only, so config validation would fail before runtime even if the filter were changed.
  - file: [extensions/active-memory/openclaw.plugin.json:20](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/extensions/active-memory/openclaw.plugin.json#L20)

  - file: [extensions/active-memory/index.test.ts:501](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/extensions/active-memory/index.test.ts#L501)
