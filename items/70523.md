## Summary

Keep #70523 open. Current main reduced default daily-file pollution by defaulting dreaming storage to `separate`, but the PR’s explicit inline-storage fallback behavior is still missing: current light/REM phase code writes raw candidate/reflection blocks to `memory/YYYY-MM-DD.md` and the narrative fallback path does not report a result back to suppress them. The PR remains a plausible focused fix, though it should be cleaned up before merge because it includes unrelated MiniMax FAQ docs changes and still needs maintainer review of the bot/security feedback.

## Best Possible Solution

Keep #70523 open as the focused inline-storage fallback fix. The best path is to rebase on current main, remove unrelated MiniMax FAQ/docs churn, keep the fallback-result and inline-removal regression coverage, and either address the transient raw-write window or explicitly document why post-fallback cleanup is acceptable for this bug scope.
