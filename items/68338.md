## Summary

Keep open. Current `main` does not implement multiline `MEDIA:` continuation parsing: `MEDIA_TOKEN_RE` requires at least one same-line payload character, and the `splitMediaFromOutput` loop keeps a `MEDIA:` line as plain text when that regex finds no matches. The shipped tests and docs also still describe only same-line `MEDIA:<path-or-url>` directives. The PR remains useful, and the discussion shows the earlier greedy-continuation regression was already addressed in the follow-up commit.
