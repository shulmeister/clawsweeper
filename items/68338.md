## Summary

Keep open. Current main still parses only same-line `MEDIA:<path-or-url>` directives; a bare `MEDIA:` line followed by a path is preserved as text, so PR #68338 remains a plausible focused parser fix rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to adapt or merge the focused parser change with a regression test for `MEDIA:\n/path`, verify it consumes only the immediate continuation line, and decide whether docs should mention the accepted multiline form or continue to recommend same-line `MEDIA:<path>` syntax.
