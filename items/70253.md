## Summary

Keep #70253 open. Current main still contains the Mattermost inbound path matching the reporter's streaming root cause: it unconditionally creates a draft preview and passes `disableBlockStreaming: true`, so the account `blockStreaming` setting is not respected on that main reply path. The A2A/channel-routing part also remains unsolved: Mattermost has no `allowBots` config in its strict schema/types, mention gating still drops unmentioned group/channel traffic, and shared routing docs/code still describe one selected agent per inbound message. The related streaming reports do not fully supersede this combined Mattermost streaming plus channel A2A bug.

## Best Possible Solution

Keep the issue open. The best maintainer path is to fix the Mattermost inbound reply path so `channels.mattermost.blockStreaming`/preview streaming settings control `disableBlockStreaming` and `onPartialReply`, with a regression test, then separately define the product/API contract for channel-level A2A or Mattermost `allowBots` parity before changing bot-message routing.
