## Summary

Current main already handles CLI note contrast through the shipped note renderer, without taking this PR's forced near-white body color that review comments flagged as a light-terminal regression.

## Best Possible Solution

Close this PR as implemented on main. Keep the shipped identity note formatter, which avoids Clack's dim note-body styling without introducing a hard-coded near-white foreground. If a current release still has unreadable note bodies in a specific web terminal/theme, track that as a fresh bug with terminal details and screenshots rather than reviving this color-token patch.
