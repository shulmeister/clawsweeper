## Summary

Keep #47203 open. Current main still matches the reported browser navigate gap: the CLI/client path keeps a 20s timeout, the /navigate route reads only url/targetId, and the Playwright navigation path has no loadState/waitUntil pass-through. The commenter’s workaround using browser open or evaluate remains consistent with the source state rather than evidence of a fixed core behavior.
