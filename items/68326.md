## Summary

Current upstream OpenClaw already resolves subagent models from configured defaults/per-agent overrides, persists the resolved provider+model onto the child session before the run starts, and documents `agents.defaults.subagents.model` as the way to set a cheaper/global subagent default. The report does not match current `main` or the latest upstream release.

## Evidence

- **Subagent model selection comes from config, not a hardcoded Bedrock default:** `resolveSubagentConfiguredModelSelection()` prefers `agents.list[].subagents.model`, then the target agent primary model, then `agents.defaults.subagents.model`; `resolveSubagentSpawnModelSelection()` then falls back to `agents.defaults.model.primary` and finally the runtime default. There is no core path here that defaults subagents to `amazon-bedrock` unless config points there.
  - file: [src/agents/model-selection.ts:239](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/agents/model-selection.ts#L239)
  - command: `nl -ba src/agents/model-selection.ts | sed -n '236,262p'`
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Child session runtime model is persisted before the subagent run starts:** `persistInitialChildSessionRuntimeModel()` writes `model` and `modelProvider` into the child session store, and `spawnSubagentDirect()` calls it before starting the run. That contradicts the report's claim that new subagents should be falling back to an unrelated saved provider after restart.
  - file: [src/agents/subagent-spawn.ts:204](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/agents/subagent-spawn.ts#L204)
