## Summary

Keep open. Current main does not implement `google-vertex` memory embeddings, and this PR targets an outdated core seam while the review discussion still contains unresolved merge-blocking defects.

## Evidence

- **Memory embeddings are provider-owned on current main:** Current docs and plugin types say memory embedding behavior belongs in provider plugins, not the old core switchboard. The migration doc explicitly says concrete remote providers live in their owning plugins, and the provider hook surface exposes `createEmbeddingProvider` for that purpose.
  - file: [docs/plugins/sdk-migration.md:378](https://github.com/openclaw/openclaw/blob/a5db42862d76fce18e5d7fa57f3907a543e61137/docs/plugins/sdk-migration.md#L378)

  - sha: [a5db42862d76](https://github.com/openclaw/openclaw/commit/a5db42862d76fce18e5d7fa57f3907a543e61137)
- **Google plugin only registers the Gemini memory adapter:** The bundled Google plugin currently registers `geminiMemoryEmbeddingProviderAdapter`; there is no separate `google-vertex` memory adapter registered on main.
  - file: [extensions/google/index.ts:156](https://github.com/openclaw/openclaw/blob/a5db42862d76fce18e5d7fa57f3907a543e61137/extensions/google/index.ts#L156)
