## Summary

Close as ClawHub/plugin scope. The PR adds Google Vertex embeddings through an old hard-coded builtin memory path, but current main resolves memory embedding providers through plugin-owned adapters declared in manifests. The supplied PR/review discussion is still useful, especially the Vertex service-account details and the token-refresh/auto-selection defects, but there is no missing core extension API that requires keeping this core PR open.

## Best Possible Solution

Do not merge this core PR. Preserve the Vertex service-account implementation notes and move the work into a ClawHub/community plugin, or a maintainer-sponsored Google provider-plugin follow-up, that declares contracts.memoryEmbeddingProviders: ["google-vertex"], registers a MemoryEmbeddingProviderAdapter, and fixes the review issues before shipping.
