## Summary

Keep open. Current `main` still implements the Agents page selector as a single dropdown, so the requested visible agent list/sidebar is not implemented. The reported default-agent persistence problem also cannot be cleanly dismissed from code inspection because the toolbar action only stages `agents.defaultId` in the config draft; persistence depends on a separate Save action elsewhere in the page, which leaves the UX concern unresolved rather than closed.

## Evidence

- **Agents page still uses dropdown selector:** `renderAgents()` renders the agent chooser as `<select class="agents-select">` with all agents as `<option>` entries, matching the reported friction rather than a visible list/sidebar.
