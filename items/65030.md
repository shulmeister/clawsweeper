## Summary

Keep open. Current main includes #65952's known Bedrock context-window lookup, and the changelog shows that shipped in 2026.4.22, but the provenance behavior proposed here is not implemented. Unknown Bedrock models still materialize the fallback as a model contextWindow, and the runtime guard has no provenance input to classify that as a default. The PR also has unresolved review concerns that would need fixes before merge.

## Best Possible Solution

Keep this PR open as a still-relevant implementation candidate, but require a rebase onto current main and a narrower fix: preserve #65952's known-context-window lookup, add or replace fallback provenance only for unknown/defaulted Bedrock values, update ModelDefinitionConfig and ProviderRuntimeModel if a field is kept, thread it through every resolveContextWindowInfo call site, fix the inference-profile config scope, include explicit-default provenance in the discovery cache key, and add tests for unknown foundation models, unknown inference profiles, explicit defaultContextWindow, and runtime guard classification. If maintainers decide #65952's conservative fallback is the desired final product behavior, close this manually with that product rationale rather than as an automated duplicate.
