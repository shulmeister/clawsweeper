## Summary

Keep open. Current `main` does not implement this PR's provenance-flag behavior. Bedrock discovery now fills in known context windows for many models, but unknown Bedrock foundation models and inference profiles still fall back to `32000` as ordinary model metadata, and the context-window guard still treats any model-provided window as `source: "model"`. The discussion on the PR also explicitly says the known-window work in #65952 is complementary, not a replacement.

## Evidence

- **Current Bedrock discovery still uses numeric fallback without provenance:** `discovery.ts` resolves known model IDs from a lookup table, but when a model is unknown it still returns `defaults.contextWindow` directly. There is no `contextWindowIsLiteralDefault` or equivalent provenance field in the discovered model shape on `main`.

  - file: [extensions/amazon-bedrock/discovery.test.ts:121](https://github.com/openclaw/openclaw/blob/a5db42862d76fce18e5d7fa57f3907a543e61137/extensions/amazon-bedrock/discovery.test.ts#L121)

## Risks / Open Questions

- The original issue may be less user-visible now for known Bedrock models because `main` added lookup-table context windows, but unknown or newly added Bedrock models still silently inherit the 32000 fallback as `source=model`.
- If this PR is revived, it will need rebasing against the current lookup-table-based discovery implementation rather than the older pure-default path.
