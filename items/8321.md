## Summary

Keep #8321 open. Current main still has no per-job `disableCache` or `contextPruning` field for cron jobs, and isolated cron runs still derive context-pruning behavior from global/per-agent agent defaults. The supplied discussion also shows the direct implementation PR #8341 was closed unmerged, while #24597 tracked a broader direct-script execution idea rather than this cache/pruning override.

## Best Possible Solution

Keep #8321 open. Implement an additive per-job `contextPruning` override, likely reusing `AgentContextPruningConfig` with `mode: "off" | "cache-ttl"`, then thread it through cron create/update/list schemas, CLI/UI/agent-tool surfaces, normalization, and isolated-run config construction. Add regression tests proving a job-level `mode: "off"` overrides global `agents.defaults.contextPruning.mode: "cache-ttl"`; keep direct script execution tracked separately from this request.
