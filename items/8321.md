## Summary

Current main does not implement per-job cache or contextPruning control for cron jobs. The cron types, CLI, and gateway protocol expose per-job model/thinking/lightContext/tools controls, but no disableCache/contextPruning field, and the isolated runner still builds agent defaults only from global/per-agent config. The related PR #8341 was closed unmerged per the supplied context, and #24597 tracks a related but different direct script-exec feature, so this issue should remain open.
