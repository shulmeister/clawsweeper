## Summary

Keep #8321 open. Current main still does not expose per-job `disableCache` or `contextPruning` for cron jobs. Cron job types, Gateway protocol schemas, CLI/agent-tool/UI surfaces, and isolated-run config construction still only support the existing per-job model/thinking/timeout/lightContext/tool controls, while cache/pruning behavior is read from `agents.defaults.contextPruning`. The related PR #8341 was closed unmerged in the provided GitHub context, and the related direct-script/no-LLM request #24597 is a separate capability rather than the per-job cache/pruning override requested here.

## Best Possible Solution

Keep #8321 open. Implement an additive per-job cron `contextPruning?: AgentContextPruningConfig` override using the existing `mode: "off" | "cache-ttl"` contract, then thread it through cron create/update/list schemas, normalization/store, CLI/UI/macOS and agent-tool surfaces, isolated-run config construction, and docs. Add regression coverage proving a job-level `mode: "off"` overrides global/per-agent `agents.defaults.contextPruning.mode: "cache-ttl"`. Keep any direct script/no-LLM cron execution work tracked separately from this cache/pruning-control request.
