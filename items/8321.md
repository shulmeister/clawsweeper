## Summary

Keep #8321 open. Current main still has no per-job `disableCache` or `contextPruning` override in the cron job type, gateway protocol, CLI/UI creation paths, or isolated cron runner config. The related PR #8341 was closed unmerged in the provided GitHub context, and #24597’s direct script/no-LLM cron request is a separate cost-reduction path rather than this job-level pruning/cache override.

## Best Possible Solution

Implement an additive per-job cron `contextPruning?: AgentContextPruningConfig` override using the existing `mode: "off" | "cache-ttl"` contract. Thread it through cron create/update/list schemas, persistence normalization, CLI, Control UI/macOS protocol surfaces, docs, and agent-tool interfaces, then merge it into `cfgWithAgentDefaults` before isolated cron invokes the embedded runner. Add regression coverage proving a job-level `mode: "off"` overrides global or per-agent `agents.defaults.contextPruning.mode: "cache-ttl"`. Keep direct script/no-LLM cron execution tracked separately.
