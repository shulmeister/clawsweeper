## Summary

Keep open. Current main still treats skill OS metadata as exact Node platform strings, so `os: macos` does not match macOS/Darwin. The remaining work is either to support `macos` as an alias or clarify/reject that metadata value more explicitly.

## Evidence

- **Exact platform comparison in requirements:** Skill OS eligibility only clears missing OS requirements when the required list contains the exact local platform string. On macOS that local platform comes from `process.platform`, i.e. `darwin`, so `macos` remains missing.
  - file: [src/shared/requirements.ts:73](https://github.com/openclaw/openclaw/blob/4069c81b159087cb71f6b64676d533cf0d897d68/src/shared/requirements.ts#L73)

- **Current-platform wrapper uses Node platform directly:** The current-platform skill requirement wrapper passes `process.platform` directly into requirement evaluation with no macOS alias normalization.
  - file: [src/shared/entry-status.ts:59](https://github.com/openclaw/openclaw/blob/4069c81b159087cb71f6b64676d533cf0d897d68/src/shared/entry-status.ts#L59)
