## Summary

Keep #39322 open. Current main still lacks legacy iMessage tapback forwarding: no `includeReactions` config/schema, no reaction payload/parser, and the monitor only subscribes to attachments while handling `message`/`error` notifications. The PR remains potentially useful, especially because later PR commits addressed review feedback, but it likely needs maintainer review against the current `imsg` reaction contract and group mention-gating behavior.

## Best Possible Solution

Keep the PR open for normal maintainer review. The best path is to update the legacy iMessage implementation against the current `imsg` reaction stream, add `includeReactions` config/schema/docs, add parser and monitor tests, and explicitly define how synthetic reaction events interact with group mention gating. Close only if maintainers decide legacy `imsg` will not gain tapback forwarding and route users to BlueBubbles or a canonical follow-up instead.
