## Summary

Keep #39322 open. Current main still does not implement legacy `imsg` iMessage tapback forwarding: there is no `includeReactions` config/schema, no reaction payload/parser, and the monitor subscribes only to attachment-enriched message notifications while handling only `message` and `error` RPC methods. The PR remains a plausible implementation candidate for the still-open canonical feature request #39031, though it needs maintainer review for the current `imsg` reaction contract and group mention-gating behavior.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to update the legacy iMessage implementation against the current `imsg` reaction stream, add `includeReactions` to config/schema/docs, add parser and monitor tests, and explicitly define how synthetic reaction events interact with group mention gating. If maintainers decide legacy `imsg` should not support inbound tapback forwarding, supersede this PR with an explicit decision on #39031 rather than stale-cleanup closure.
