## Summary

Keep #39322 open. Current main still lacks iMessage tapback forwarding: no includeReactions config/schema, no reaction payload/parser, no reaction notification handler, and watch.subscribe only requests attachments. The PR is still a plausible implementation for open feature issue #39031, though it needs maintainer review for the imsg reaction contract and group mention-gating behavior.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to update the legacy iMessage plugin against the current imsg reaction stream, add includeReactions to config/schema/docs, add parser and monitor tests, and explicitly define how synthetic reaction events interact with group mention gating. If maintainers decide legacy imsg should not support inbound tapback forwarding, supersede this PR with an explicit decision on #39031 rather than stale-cleanup closure.
