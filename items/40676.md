## Summary

Keep open. Current main does not implement the two central fixes from PR #40676: Discord voice auto-join still only runs from READY handling, and the join path still calls `joinVoiceChannel` without any stale `@discordjs/voice` registry cleanup. The PR also cannot merge as-is because the Discord code has moved into `extensions/discord/**`, and review comments identify correctness risks in the branch’s extra session-key changes.

## Best Possible Solution

Keep the PR or its linked issue open for maintainer/product handling. The best implementation path is a rebased, focused Discord plugin patch that adds a `DiscordVoiceResumedListener`, registers it from `extensions/discord/src/monitor/provider.ts`, adds manager/account-safe stale connection cleanup before `joinVoiceChannel`, and includes targeted tests for RESUMED listener registration plus stale-connection cleanup. If this exact PR is abandoned, #40665 should remain the canonical bug until an updated PR lands.
