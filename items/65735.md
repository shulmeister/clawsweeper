## Summary

PR #65735 targets an older `/status` runtime-loading shape that is no longer present on current `main` or the latest shipped release. Current code uses literal dynamic imports for the `/status` runtime modules instead of the `new URL() + importRuntimeModule()` pattern described in the PR, and the referenced `commands-status-deps.runtime` module is not present on `main` at all.

## Evidence

- **Current `/status` command loader uses a literal dynamic import:** `get-reply-directives-apply.ts` now loads `./commands-status.runtime.js` directly with `import(...)`, so the PR body's `new URL() + importRuntimeModule()` reproduction no longer matches current source.
  - file: [src/auto-reply/reply/get-reply-directives-apply.ts:19](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/auto-reply/reply/get-reply-directives-apply.ts#L19)
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **Current status-message path also uses a literal dynamic import chain:** `status-text.ts` loads `status-message.runtime.js`, and that wrapper imports `../auto-reply/status.runtime.js` with a literal specifier. This is a different build shape from the one the PR is trying to patch.
  - file: [src/status/status-text.ts:46](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/status/status-text.ts#L46)
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **Status runtime wrapper is a direct import shim:** `status-message.runtime.ts` is just a thin `import("../auto-reply/status.runtime.js")` wrapper, which bundlers can statically see without adding the extra explicit entries proposed in the PR.
  - file: [src/status/status-message.runtime.ts:1](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/status/status-message.runtime.ts#L1)
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **Current pinned tsdown entries do not include the PR's added keys:** `buildCoreDistEntries()` still pins only the long-lived lazy boundaries such as `subagent-registry.runtime`; there is no `auto-reply/status.runtime` or `auto-reply/reply/commands-status-deps.runtime` entry on current `main`. That indicates `/status` is no longer handled through the explicit-entry mechanism proposed here.
  - file: [tsdown.config.ts:204](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/tsdown.config.ts#L204)
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
- **`commands-status-deps.runtime` no longer exists on `main`:** A repository search for `commands-status-deps` returned no matches under `src/`, so one of the two files this PR adds to `tsdown.config.ts` is not part of the current codebase anymore.
  - command: `rg -n 'commands-status-deps' src`
  - sha: [c150110e0255](https://github.com/openclaw/openclaw/commit/c150110e02550eaf6c56db0c851dc62fc0ca88e1)
