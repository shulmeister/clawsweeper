## Summary

Keep #36525 open. Current main still implements the existing workspaceAccess model: rw maps /workspace to the agent workspace, none/ro use the sandbox workspace but keep writes gated off, and split ro/rw paths can still expose the agent workspace at /agent. The issue is concrete, recent, unprotected, and broader than the narrower writable-none trackers.

## Best Possible Solution

Keep #36525 open for maintainer product direction and implementation. A complete fix would make /workspace consistently represent the per-scope sandbox workspace, make writeability follow resolved mount writeability, remove implicit /agent exposure unless explicitly configured through documented binds, update Docker/browser/SSH/OpenShell paths together, and revise tests plus public sandboxing docs. Keep #37634/#46300 as narrower writable-none implementation context, not as full supersession of this broader refactor.
