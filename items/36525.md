## Summary

Keep #36525 open. Current main at e672b61417af5c45b0431df6d9109a1f4b618ef5 still implements the pre-refactor sandbox workspace semantics: `rw` points `/workspace` at the agent workspace, `none`/`ro` keep workspace writes disabled, and split workspaces still expose `/agent` for non-`none` access. The related #37634 is narrower and does not supersede this broader mount-semantics request.

## Best Possible Solution

Keep #36525 open for an explicit sandbox contract decision and coordinated implementation. A complete fix should define the final `workspaceAccess` semantics, make `/workspace` consistently represent the sandbox workspace in sandbox mode if that direction is accepted, remove implicit main-workspace exposure except through documented explicit binds, align Docker/browser/remote/OpenShell filesystem bridges, update file-tool write gates, and revise tests plus public sandbox/security docs. Keep #37634 as the narrower writable-`none` issue rather than treating it as the canonical replacement for this broader refactor.
