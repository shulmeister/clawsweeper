## Summary

Keep #36525 open. Current main still implements the pre-refactor sandbox workspace contract: `rw` selects the agent workspace as the active `/workspace`, non-`rw` makes the active workspace read-only for file tools, and split non-`none` workspaces can still expose the agent workspace at `/agent`. The related #37634 is narrower and does not supersede this broader mount-semantics request.

## Best Possible Solution

Keep #36525 open for an explicit sandbox contract decision and coordinated implementation. If accepted, update workspace layout so `/workspace` consistently maps to the sandbox workspace in sandbox mode, remove implicit main-workspace exposure except through documented explicit binds, align Docker/browser/SSH/OpenShell filesystem bridges, update file-tool write gates, and revise tests plus public sandbox/security docs. Keep #37634 as the narrower writable-`none` issue rather than treating it as the canonical replacement for this broader refactor.
