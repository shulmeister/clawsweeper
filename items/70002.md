## Summary

Keep #70002 open. Current main still lacks the fork guards, and both docs workflows still consume OPENCLAW_DOCS_SYNC_TOKEN while targeting openclaw/docs. The PR is a focused two-line CI fix that matches an existing upstream-only workflow guard pattern; because it is recent and infra-labeled, it should get maintainer review rather than automated closure.

## Best Possible Solution

Keep the PR open for maintainer review. If maintainers agree with the fork policy, land the existing two job-level guards and validate with workflow syntax/CI or a manual fork dispatch where feasible; no core code or docs changes are needed.
