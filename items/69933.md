## Summary

Keep open. Current main still routes Dreaming/Diary “Open Config” to `config.openFile`, and the Windows opener still emits the same unsupported `Start-Process -LiteralPath ...` command shown in the issue log. This is a real current code bug, not a stale or already-implemented report.

## Best Possible Solution

Fix the Windows branch of `resolveConfigOpenCommand` in `src/gateway/server-methods/config.ts` to use a supported PowerShell invocation, likely `Start-Process -FilePath` with safe literal quoting or an argument-array approach that preserves paths as data. Update `src/gateway/server-methods/config.test.ts` so it asserts the supported Windows command shape and keeps the shell-injection protection for quoted paths.
