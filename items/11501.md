## Summary

Current main already supports customizing or removing outbound sender prefixes via `messages.responsePrefix` and per-channel/account overrides, including explicit empty-string disable behavior and WhatsApp-specific documented behavior.

## Close Comment

Closing as implemented on current OpenClaw. This Codex review found that outbound sender prefixes are already configurable through `messages.responsePrefix`, with more specific `channels.<channel>.responsePrefix` and `channels.<channel>.accounts.<id>.responsePrefix` overrides. Current main resolves explicit custom strings, `"auto"` for `[{identity.name}]`, and explicit `""` to hide the prefix entirely (`src/agents/identity.ts:90`, `src/agents/identity.per-channel-prefix.test.ts:16`). The user-facing docs also describe this behavior, including WhatsApp self-chat defaulting to `[{identity.name}]` or `[openclaw]` only when `messages.responsePrefix` is unset (`docs/gateway/configuration-reference.md:2083`, `docs/channels/whatsapp.md:222`). I also checked the `v2026.4.22` tag and the same implementation/docs are present there. If there is still a gap, it would need to be a more specific bug report about a channel not honoring `responsePrefix`, not a missing feature.
