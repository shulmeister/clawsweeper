## Summary

Keep #55940 open. Current main still fails `kind: "brew"` skill installs when Homebrew is missing, so the PR is not implemented or superseded; however, the PR’s automatic `apt-get` fallback remains a product/security decision because non-bundled skill installs currently only warn before executing installer metadata.

## Best Possible Solution

Keep the PR open, but require a current-main rebase and a maintainer decision on the safe scope for Linux brew fallback. The likely mergeable path is to preserve the Docker/Linux fix for #14593 while gating apt fallback to trusted bundled/managed skills, an explicit opt-in, or a narrow formula allowlist, then update the added tests and docs around the chosen contract.
