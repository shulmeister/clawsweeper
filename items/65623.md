## Summary

Keep open. Current `main` still serializes `snapshot.parsed` directly for `/config show` replies, including path-specific lookups, while the existing config redaction helper is only used on the gateway config APIs. The linked PR #65637 appears to be the still-pending fix, and the same vulnerable block is present in the latest released code snapshot provided here.

## Evidence

- **Current main still returns raw parsed config in chat replies:** `handleConfigCommand` reads the config snapshot, clones `snapshot.parsed`, and for `/config show` either JSON-stringifies the selected value or the entire object directly into the reply text. There is no call to `redactConfigSnapshot` or `redactConfigObject` on this path.
