## Summary

Keep this issue open. Current main still serializes the parsed config for `/config show` without applying the existing config redaction helpers, and there is an open, unmerged same-author PR, #65637, that is explicitly linked as the implementation candidate for this issue. Per the maintainer workflow, the issue should not be closed before that PR is reviewed and merged or intentionally replaced.

## Best Possible Solution

Keep this issue open and review/land #65637, or replace it with an equivalent maintainer-approved fix that applies schema-aware redaction to both full and path-specific `/config show` replies and adds regression coverage. After the fix is merged on main, close this issue as implemented with the fixing commit and release evidence.
