## Summary

Keep open. Current main exposes livenessState in EmbeddedPiRunResult.meta and lifecycle events, but it still does not surface a blocked/abandoned state in the user-facing reply payload contract or delivery path requested by this issue.

## Best Possible Solution

Keep this open and implement a small generic surfacing layer: when the final embedded run meta.livenessState is blocked or abandoned, ensure the final user-facing payload carries that state, either through a documented structured ReplyPayload/AgentRuntimeReplyPayload field or a stable footer such as “Run ended early: blocked.” Add focused regressions for strict blocked and incomplete abandoned exits, plus one delivery-path test proving a channel/CLI-style payload renderer receives the signal.
