## Summary

Keep #66924 open. Current main still has the targeted Google Gemma fallback gap: Gemma models are resolved by cloning the `gemini-3-flash-preview` template, and the current Gemma branch only patches `reasoning` for Gemma 4, so a 1M template `contextWindow` is still inherited. The linked #66740 report provides a concrete registry-miss reproduction for `google/gemma-4-26b-a4b-it`, and the PR’s proposed regression test covers a real missing assertion on main.

## Best Possible Solution

Keep the PR open for maintainer review. The product path is to fix Google-owned Gemma forward-compat metadata in `extensions/google/provider-models.ts`, add the missing regression test in `extensions/google/provider-models.test.ts`, and confirm whether the fallback should be a fixed conservative Gemma default or should preserve configured/provider catalog context metadata when available.
