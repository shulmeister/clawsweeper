## Summary

Keep #70779 open. It is a recent, concrete regression report against the OpenAI-compatible model transport, not a stale/support-only Feishu configuration issue. Current main still has the same risk shape: OpenAI completions requests always include `stream_options.include_usage`, while the stream processor can only emit incremental UI/channel updates if upstream chunks contain actual text/reasoning/tool deltas. The provided related ClawSweeper items cover nearby Gemini/model-id, reasoning-leak, and usage-accounting work, but none is a canonical duplicate for this visual streaming regression.

## Best Possible Solution

Keep the issue open and treat it as a model transport regression. The best next maintainer path is to add a targeted OpenAI-compatible streaming regression fixture for a Gemini-compatible backend that stops emitting incremental content when `stream_options.include_usage` is present, then either gate `include_usage` on `compat.supportsUsageInStreaming`, provide/document a per-model opt-out that is honored before request dispatch, or adapt the stream parser if the backend emits Gemini-specific delta fields that OpenClaw currently ignores. The fix should include coverage that the Control UI and Feishu receive multiple `text_delta`/card content updates before final completion.
