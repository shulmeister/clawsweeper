## Summary

Current main still lacks the bundled LSP process-tree cleanup described here, and the provided GitHub context shows two open, unmerged implementation PRs (#72367 and #72530) that explicitly link this issue with GitHub closing semantics. This issue should stay open until one of those PRs, or an equivalent fix, lands.

## Best Possible Solution

Keep this issue open and review/land PR #72367, PR #72530, or an equivalent focused fix. The implementation should make bundled stdio LSP servers disposable as process trees, use the existing process-tree cleanup pattern after bounded graceful LSP shutdown, add a Gateway shutdown sweep for active bundled LSP runtimes, and include regression coverage in bundled LSP runtime and Gateway close tests.
