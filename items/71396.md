## Summary

Keep #71396 open. Current main still implements the Feishu mention cascade that the PR is trying to remove: inbound `ctx.mentionTargets` are passed into the Feishu reply dispatcher, then attached to first outbound text/card chunks and streaming-card closes. The PR is not obsolete, though its review/security comments need maintainer handling before merge.

## Best Possible Solution

Keep the PR open and have the Feishu plugin maintainer finish the bundled plugin fix: stop unconditional automatic mention propagation for fallback/error/ack/ordinary replies, keep mention context safe for the agent, preserve intentional explicit mention delivery through allowlisted structured conversion for text/card/streaming-card paths, and land regression coverage for those paths.
