## Summary

Keep open. Current `main` has related hardening around persisted session model selection, but it still does not implement this PR's core behavior: inferring a unique configured provider for bare session `modelOverride` values inside `resolveSessionModelRef()`. The PR is therefore not already implemented on `main`, and the remaining work is still substantive enough to merit maintainer judgment rather than auto-close.

## Evidence

- **Selected-model path still returns persisted ref directly:** `resolveSessionModelRef()` resolves defaults, normalizes stored overrides, and then returns `resolvePersistedSelectedModelRef(...)` directly when a persisted selection exists. There is no `inferUniqueProviderFromConfiguredModels()` step on this selected-model path.
