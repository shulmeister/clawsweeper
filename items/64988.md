## Summary

Keep open. Current main has some nearby provider-inference work, but the specific behavior proposed by this PR is not implemented in `resolveSessionModelRef()`: bare `modelOverride` values still resolve through the persisted override resolver using the already-resolved default provider, without consulting the configured model-family map. The PR is also not ready to close as-is because review comments identify real edge cases around explicit wrapper providers, stored runtime providers, and non-OpenAI defaults.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up. The best path is to implement the inference in the selected-session model resolver or a shared helper used by it, while preserving explicit `providerOverride`, preserving wrapper providers for slash-containing model ids, handling existing sessions with stored `modelProvider`, and covering non-OpenAI default providers. Add focused tests for the PR's bare `modelOverride` plus configured unique provider case and the review-comment edge cases.
