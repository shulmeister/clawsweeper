## Summary

Keep this PR open. Current main still leaves the requested tool-loop protection default off in the runtime detector and resolver path, while tests, docs, schema help, generated schema metadata, and type comments still document or assert default-off behavior. The PR is an active focused fix attempt for a concrete repeated `web_fetch` DNS-error loop report, not an obsolete or shipped change.

## Best Possible Solution

Keep this PR open for maintainer review or consolidation with the other open loop-detection PRs. The best implementation is a minimal core change that enables tool-loop detection by default in the actual runtime path, preserves explicit false and global-to-agent inheritance semantics, updates zod schema source plus generated config/help/docs/UI-visible defaults, and adds focused coverage for no-config default-on, explicit false, global false with partial agent overrides, and schema/help consistency.
