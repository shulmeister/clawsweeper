## Summary

Keep PR #70877 open. Current main still leaves generic tool-loop detection disabled by default: the runtime default is `enabled: false`, the per-agent/global config resolver passes through undefined or explicit config without default-enabling it, tests still assert the no-config path is disabled, and docs/schema/type metadata still advertise default false. The PR remains relevant, though the provided review comments show its current implementation still needs fixes around the source default, schema/default metadata, generated baselines, and inheritance semantics before it is mergeable.

## Best Possible Solution

Keep this PR open or consolidate it with one canonical maintainer-reviewed loop-detection-default PR. The best implementation should move default-enabled behavior into the actual runtime resolution path used by `detectToolCallLoop`, preserve explicit global `enabled: false` when agent overrides omit `enabled`, update schema/help/generated baselines and public docs from source, and add regression coverage for no-config default-enabled behavior plus explicit opt-out behavior.
