## Summary

Keep open. Current `main` already supports image attachments in Control UI via file picker, paste, drag/drop, preview, and `chat.send` attachment wiring, so the issue body's "no way to upload images" claim is outdated. But there is still no screenshot-capture button or screen-capture API path in the Control UI, so the requested feature itself is not implemented on `main` and remains in scope for OpenClaw core UI.

## Evidence

- **Control UI already has image attach UI:** `ui/src/ui/views/chat.ts` renders a hidden file input plus an `Attach file` toolbar button using the paperclip icon, wired to `handleFileSelect` for `image/*` attachments.
  - file: [ui/src/ui/views/chat.ts:1205](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/ui/src/ui/views/chat.ts#L1205)
