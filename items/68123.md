## Summary

Current OpenClaw already supports cron runs that avoid posting back to the main session: isolated/current/custom-session agent-turn jobs can set `delivery.mode: "none"` (CLI `--no-deliver`), and tests verify that this suppresses the main-session relay. Main-session cron remains intentionally system-event based.

## Evidence

- **CLI exposes opt-out delivery for non-main cron jobs:** `openclaw cron add` supports `--no-deliver`, rejects it on main-session jobs, and maps isolated/current/custom-session agent-turn jobs to `delivery.mode = "none"`.
