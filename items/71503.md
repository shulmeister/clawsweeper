## Summary

Keep open. Current main still uses the older first-matching LAN IPv4 behavior in core pairing, gateway LAN selection, and the bundled device-pair plugin, so the PR's requested physical/default-route-before-bridge behavior is not implemented or superseded on main. The PR discussion also contains useful follow-ups addressing CNI prefixes and default-route preference, so there is still reviewable work rather than a cleanup-close candidate.
