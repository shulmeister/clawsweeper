## Summary

Keep this PR open. Current main still has order-dependent LAN IPv4 selection for the affected gateway/status and pairing surfaces, while this PR remains the active implementation candidate for linked #71493. The discussion shows the author already addressed the CNI-prefix review note and added default-route preference while preserving bridge/container fallback behavior.

## Best Possible Solution

Keep this PR open for maintainer review or land an equivalent fix. The desired behavior is consistent automatic LAN address selection across gateway dashboard/status, setup-code QR pairing, and device-pair: prefer the default-route or physical/default-looking LAN interface, deprioritize Docker/bridge/veth/CNI/container interfaces as fallback candidates, preserve explicit URL/config overrides, and keep regression tests for Docker bridge, CNI overlay, and default-route cases.
