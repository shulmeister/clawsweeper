## Summary

Keep open. Current `main` does not implement the PR’s inline `buffer`/`filename`/`contentType` handling for Telegram `send` actions; the equivalent code paths still only support URL-based media.

## Evidence

- **Telegram action runtime still ignores inline attachment fields:** `handleTelegramAction` reads only `mediaUrl`/`media`, treats content as optional only when `mediaUrl` exists, and forwards only `mediaUrl` into `sendMessageTelegram`. There is no read/forward path for `buffer`, `filename`, or `contentType`.
  - file: [extensions/telegram/src/action-runtime.ts:313](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/extensions/telegram/src/action-runtime.ts#L313)
  - command: `nl -ba extensions/telegram/src/action-runtime.ts | sed -n '296,380p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Telegram sender has no inline buffer code path:** `TelegramSendOpts` has `mediaUrl` but no inline attachment fields, and `sendMessageTelegram` only enters media handling when `mediaUrl` is present. The send path loads media via `loadWebMedia(mediaUrl, ...)`; there is no base64 decode/InputFile path sourced from action params.
  - file: [extensions/telegram/src/send.ts:74](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/extensions/telegram/src/send.ts#L74)
  - command: `nl -ba extensions/telegram/src/send.ts | sed -n '70,110p'; nl -ba extensions/telegram/src/send.ts | sed -n '620,900p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Shared attachment hydration still skips `send`:** `hydrateAttachmentParamsForAction` only hydrates `sendAttachment`, `setGroupIcon`, and `upload-file`. A plain `send` action still does not get base64/data-URL normalization or filename/content-type hydration.
  - file: [src/infra/outbound/message-action-params.ts:374](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/infra/outbound/message-action-params.ts#L374)
