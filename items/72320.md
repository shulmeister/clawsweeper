## Summary

#72320 reports the same missing `task-registry-control.runtime.{js,ts}` npm package failure already tracked by the older open issue #68997. Current `main` still shows the same loader/build-entry shape, so this issue should close as a duplicate while #68997 remains the canonical fix target.

## Best Possible Solution

Close #72320 as a duplicate and keep #68997 as the canonical open tracker. The fix should add a packaged, stable `task-registry-control.runtime` sidecar or make the cancellation loader bundler-traceable, plus artifact-level coverage that fails when `openclaw tasks cancel` references a non-shipped runtime module.
