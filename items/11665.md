## Summary

Keep #11665 open. Current main still routes direct `/hooks/agent` and mapped hook agent actions into an isolated cron-style job, and `runCronIsolatedAgentTurn` still passes `forceNew: true` whenever `sessionTarget` is `"isolated"`. `resolveCronSession` only reuses an existing `sessionId` when `forceNew` is false, so a stable hook `sessionKey` still cannot provide multi-turn transcript history. The related PR attempts (#15084, #27123, #27193) are closed unmerged in the provided GitHub context, and current hook config/types/docs still expose `sessionKey` but no `sessionMode`, `reuseSession`, or equivalent persistence selector.

## Best Possible Solution

Keep the issue open and implement a backward-compatible persistent hook-session option in core, likely `sessionMode: "isolated" | "persistent"` or equivalent on `/hooks/agent` payloads, hook mappings, and transform results, defaulting to isolated. Thread it through hook normalization, config types/schema/help, mapping merge, dispatch, docs, and regression tests, then pass `forceNew: false` only for the persistent hook path.
