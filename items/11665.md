## Summary

Keep #11665 open. Current main still sends `/hooks/agent` and mapped agent hook actions through an isolated cron-agent turn, which forces a fresh session id even when the hook resolves a stable `sessionKey`. The hook config, mapping, transform, and request payload surfaces still expose `sessionKey` but no `sessionMode`, `reuseSession`, or equivalent persistent hook option. Related fix PRs in the provided context were closed unmerged, and the latest maintainer/Codex issue comment also kept this open.

## Best Possible Solution

Keep this issue open until OpenClaw adds a backward-compatible persistent hook-session path, likely a `sessionMode: "isolated" | "persistent"` or equivalent option on hook mappings and request/transform payloads. Thread that option through hook normalization, schema, mapping merge, docs, and tests, and make only the persistent hook path call `resolveCronSession` without `forceNew` while preserving fresh isolated behavior for existing hooks and cron jobs.
