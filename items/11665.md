## Summary

Keep open. The requested multi-turn webhook behavior is still not implemented on current main: hook agent dispatches are still converted into isolated cron jobs, isolated jobs still force a new session, and resolveCronSession only reuses an existing sessionId when forceNew is false. The related PRs in the issue timeline (#15084, #27123, #27193) are useful evidence of the intended fix direction, but they were closed/unmerged and the current code does not contain the sessionMode/persistent hook surface described there.
