## Summary

Keep #39569 open. Current main still does not implement the Feishu reply orphan-placeholder mention fallback, and the linked bug #39568 remains a plausible open behavior gap. The PR is still useful, but its rootId-clearing strategy conflicts with current Feishu topic-session routing contracts that main preserves in code and tests.

## Best Possible Solution

Keep #39569 open as a still-relevant Feishu bugfix path. Rebase it onto the current `bot-content.ts` split, add focused dropped-bot-mention reply tests, make the fallback as bot-specific as Feishu allows, and preserve topic-session `rootId` behavior for `group_topic` / `group_topic_sender` while coordinating the final behavior with #39568.
