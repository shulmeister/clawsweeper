## Summary

Keep #39569 open. Current main still lacks the Feishu reply orphan-placeholder mention fallback, so the linked #39568 bug remains plausible. The PR has useful production verification and regression-test direction, but its broad orphan-key heuristic and blanket rootId clearing still need revision against current topic/thread routing behavior.

## Best Possible Solution

Keep #39569 open as a still-relevant Feishu bugfix. Rebase it onto the current `bot-content.ts` split, add focused regression tests for reply events where Feishu drops the bot mention, make the fallback as bot-specific as available payload data allows, and preserve intentional topic-session `rootId` behavior while coordinating final semantics with #39568.

## Risks / Open Questions

- A broad orphan-placeholder heuristic can mark the bot as mentioned when Feishu dropped a non-bot mention or when user text contains placeholder-like tokens.
- Clearing `rootId` for all mentioned replies can merge or reroute Feishu topic-scoped sessions that intentionally use root/thread metadata.
- A final fix likely needs real Feishu payload samples or the strongest available bot-specific signal before landing.
