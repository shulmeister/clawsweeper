## Summary

Keep open. Current main still does not implement the Feishu reply-mention fallback this PR targets, and the PR has not been superseded by another shipped fix. The PR is also not safe cleanup-close material because its proposed rootId clearing and orphan-placeholder heuristic touch live Feishu topic routing contracts that current tests deliberately preserve.

## Best Possible Solution

Keep #39569 open as a still-relevant Feishu bugfix path, but require a rebase onto the current `bot-content.ts` split and a narrower design: add tests for dropped bot mentions in replies, avoid broad non-bot orphan false positives where possible, and preserve `rootId` for `group_topic` / `group_topic_sender` session and reply contracts. Coordinate the final fix with the linked open bug #39568.
