## Summary

Current main already ships an OpenClaw MCP server plus opt-in MCP bridges that expose OpenClaw plugin tools and selected built-in tools to Claude/Cursor-style harnesses, so the core request is already implemented.

## Risks / Open Questions

- The implementation is not the exact standalone `@openclaw/mcp-server` package proposed in the issue, and the built-in `openclaw-tools` bridge currently exposes a selective tool set rather than every example tool listed in the proposal.

## Close Comment

Closing as `implemented_on_main` after Codex review. Current OpenClaw already ships MCP support for this use case: `openclaw mcp serve` runs OpenClaw as an MCP server (`docs/cli/mcp.md:10`, `src/cli/mcp-cli.ts:26`), and the ACPX integration already has opt-in MCP bridges for both plugin tools and selected built-in OpenClaw tools for Claude/Cursor-style harnesses (`docs/tools/acp-agents-setup.md:174`, `extensions/acpx/src/config.ts:192`, `src/mcp/openclaw-tools-serve.ts:1`). This was shipped by `v2026.4.22` for the core `openClawToolsMcpBridge` path (`CHANGELOG.md:116-140`), with the plugin-tools bridge already present in `2026.3.31` (`CHANGELOG.md:1669-1685`). The exact packaging differs from the standalone package proposed here, but the underlying OpenClaw-as-MCP / OpenClaw-tools-over-MCP capability is already present on current main.
