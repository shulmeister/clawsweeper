## Summary

Keep #66700 open. Current main still treats Nextcloud Talk webhook `object.content` as plain message text, then runs generic regex mention matching against that text. It does not parse the documented rich-object JSON payload or resolve `{mention-user1}` parameters, and the prior PRs that attempted this fix were closed unmerged rather than shipped.

## Best Possible Solution

Keep #66700 open and implement a focused Nextcloud Talk plugin fix: parse `object.content` as the documented rich-object JSON, extract and strip `{mention-*}` placeholders, match user-type parameters against the configured bot identity/API user/account id, pass cleaned text into command gating and agent context, and add webhook/inbound regression tests for `{mention-user1}` group messages with `requireMention: true`. Reuse the relevant logic ideas from closed PRs #52924/#70514, but land them as a clean scoped change on main.
