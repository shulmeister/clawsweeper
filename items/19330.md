## Summary

Keep #19330 open. Current main still saves Discord, Telegram, and Signal inbound media into the shared `media/inbound` store, and the read-side media reference path still assumes first-level inbound files. The related implementation PR #46162 is closed and unmerged in the provided GitHub context, so there is no canonical shipped fix to close against.

## Best Possible Solution

Keep this issue open as the canonical tracker for core media-pipeline isolation. A viable fix should scope inbound media writes by channel account or resolved agent, update read-side media references and sandbox staging to understand that scope with backward-compatible flat `inbound` fallback, and add regression coverage for Discord, Telegram, Signal, and nested inbound reference handling.
