## Summary

Keep open. Current `main` does not implement `chat.history({ runId })`, so `implemented_on_main` does not apply. The PR is still materially useful and the discussion shows the author addressed the earlier review findings (shared matcher, pre-sanitize filtering), but the latest matcher still appears to have a correctness gap for punctuation/symbol run IDs because `runId` comes from an unconstrained `NonEmptyString` idempotency key while the boundary regex only treats letters/numbers/marks/`_`/`-` as token characters.
