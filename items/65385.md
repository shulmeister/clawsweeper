## Summary

Current `main` already reads Comfy from the plugin-owned config path, documents that path as canonical, keeps legacy `models.providers.comfy` as a fallback when plugin config is absent, and adds plugin-owned cloud API-key handling. This PR’s core goal is therefore already implemented on `main`, albeit with a narrower compatibility contract than the branch proposed.

## Evidence
