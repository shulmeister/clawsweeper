## Summary

Keep #64549 open. Current main contains the related default-memory watcher fix, but the issue is broader: the reporter reproduced after the 2026.4.12 fix, and current main still watches directory-valued `memorySearch.extraPaths` with `**/*.md` globs while the shipped fix only explicitly moved `workspace/memory` off the failing recursive glob path. I found no source, test, or release evidence proving new markdown files in extraPaths now auto-index on current main.

## Best Possible Solution

Keep the issue open. The likely maintainer path is to update `extensions/memory-core/src/memory/manager-sync-ops.ts` so directory-valued `memorySearch.extraPaths` are watched as directory roots with the existing ignore/filter logic, not only as glob patterns, then add regression coverage that proves an `add` event for a new markdown file in both `workspace/memory` and an extra path schedules sync and results in the file being indexed. If a real watcher integration test is too heavy, add a focused unit test around watcher event handling plus a targeted indexing test for the scheduled watch path.
