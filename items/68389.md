## Summary

Keep open. Current main does not implement PR #68389's requested loader warning: populated but mismatched `plugins.allow` entries still return early from `warnWhenAllowlistIsOpen`, and no current test covers the mismatch-warning behavior. The PR remains a focused fix for the linked open issue #68352, with prior review feedback already addressed in the PR discussion.

## Best Possible Solution

Keep PR #68389 open. The best path is to rebase it onto current main, keep the full-discovered-id comparison from the author's follow-up, add/adjust focused loader tests for empty, all-mismatched, partial-match, and bundled-id allowlists, and fix the changelog attribution before maintainer review or merge.
