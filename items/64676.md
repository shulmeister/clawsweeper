## Summary

Keep open. Current main still treats agent `models.json` as a regenerated provider registry: gateway warmup can call `ensureOpenClawModelsJson`, the plan merges config/implicit providers with the existing file, and merge mode only preserves selected provider fields such as `baseUrl`/`apiKey`. The code and docs do not show the reported behavior of preserving a user-pruned provider model array across gateway restart.

## Best Possible Solution

Do not close yet. Maintainers should decide whether per-agent `models.json` model arrays are editable state or generated cache. If editable, add a gateway-restart regression around `ensureOpenClawModelsJson`/`planOpenClawModelsJsonWithDeps` and change `src/agents/models-config.merge.ts` so agent-level provider model deletions survive regeneration. If not editable, tighten docs/doctor/CLI guidance so users change `agents.defaults.models` or `models.providers` instead of hand-editing `models.json`, and explain the current merge precedence clearly.
