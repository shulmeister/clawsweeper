## Summary

Keep PR #72092 open. Current main at 637bd33e69b2117331dafe92dfaed831fb684654 still unconditionally requires a static API key in the media image, MiniMax fallback, and audio/video execution auth paths, while the PR is a focused fix for the open Bedrock `aws-sdk` credential-chain regression tracked by #72031. The earlier bot review gap around empty keys being dropped by API-key rotation is real on current main, and the latest PR context shows the contributor added an `allowEmptyKey`/`allowEmptyExecution` path plus audio/video regression coverage to address it.

## Best Possible Solution

Keep PR #72092 open for maintainer review and CI. The best path is to validate or iterate on the focused implementation that carries the `aws-sdk` missing-key allowance through media image, MiniMax fallback, and audio/video execution, including the empty-key rotation contract, then merge it to close #72031 if checks pass. PR #72066 is closed unmerged and should not be treated as superseding this work.
