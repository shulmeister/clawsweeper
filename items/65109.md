## Summary

Keep open. Current main still reproduces the reported /status undercount: the status renderer displays cache read/write tokens, but the cost estimate it renders only passes input/output tokens into the shared cost estimator. Related open PRs #52329 and #65381 are implementation candidates, but the bug is not implemented on main or in v2026.4.24.

## Best Possible Solution

Keep this issue open. Port the narrow fix to `src/status/status-message.ts`: include `cacheRead` and `cacheWrite` in `hasUsage` and in the `estimateUsageCost` usage object, then add a regression in `src/auto-reply/status.test.ts` using nonzero cache read/write pricing. Review or supersede #52329/#65381 against that current implementation path, and close this issue only after the fix lands.
