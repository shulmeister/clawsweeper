## Summary

Keep open. Current `main` still treats `plugins.*` changes as gateway-restart-required, defers restart based on global active-work counts, and then flips a global draining flag that rejects new queued work across lanes. The report's exact `300s` timeout detail looks partially stale on current `main`, but the underlying cross-channel/global-drain bug is not implemented away.

## Evidence

  - file: [docs/tools/plugin.md:154](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/docs/tools/plugin.md#L154)
