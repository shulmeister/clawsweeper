## Summary

Keep open. Current `main` still suppresses the reminder warning only when `successfulCronAdds > 0`, and that counter is incremented only for the `cron` tool's `action: "add"` path, not for `exec`/`bash` commands like `openclaw cron add --at ...` described in the issue.

## Evidence

- **Guard note still depends solely on successfulCronAdds:** `agent-runner` appends the unscheduled-reminder note whenever a reminder commitment exists and `successfulCronAdds === 0` (unless an older session cron already exists). There is no separate suppression path for shell-created reminders in the same turn.
  - file: [src/auto-reply/reply/agent-runner.ts:1389](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/auto-reply/reply/agent-runner.ts#L1389)
  - command: `nl -ba src/auto-reply/reply/agent-runner.ts | sed -n '1384,1408p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Embedded runner only counts direct cron tool calls:** `successfulCronAdds` is incremented only when `toolName === "cron"` and the tool args have `action: "add"`. `exec` and `bash` are separate tool names, so a shell command that runs `openclaw cron add --at ...` will not be counted here.
  - file: [src/agents/pi-embedded-subscribe.handlers.tools.ts:110](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/pi-embedded-subscribe.handlers.tools.ts#L110)

- **Codex app-server telemetry has the same limitation:** The Codex dynamic-tool bridge also increments `successfulCronAdds` only for `toolName === "cron"` with `action: "add"`, so the alternate agent path on `main` still misses shell-based scheduling too.
