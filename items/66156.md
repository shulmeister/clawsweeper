## Summary

Current main already fixes the user-visible leak targeted by PR #66156. Queued system-event text can still be sent to the agent as runtime context, but it is kept out of transcriptPrompt/transcriptCommandBody and hidden from visible session history; v2026.4.24 also documents this WebChat/session-history behavior.

## Best Possible Solution

Close PR #66156 as implemented on main and keep the shipped prompt/transcript split plus hidden runtime-context approach. Any remaining cleanup for linked #65994 should be evaluated against current main rather than reviving this draft branch.
