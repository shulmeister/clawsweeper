## Summary

Keep #22438 open. Current main still has fixed bootstrap filtering plus adjacent `contextInjection` and `lightweight` controls, but it does not implement the requested schema-backed `agents.defaults.bootstrapTier` option or `minimal | standard | full` tier resolver. The linked implementation PR #22439 remains open and explicitly tracks the same remaining work; the latest maintainer/reporter discussion also frames this as still missing rather than obsolete.

## Best Possible Solution

Keep #22438 open until #22439 or a successor lands, or until maintainers explicitly decide that fixed allowlists plus per-run `lightContext` are the intended product design. A complete implementation should wire `agents.defaults.bootstrapTier` through schema, TypeScript types, generated config metadata/help/docs, bootstrap resolution, hook behavior, subagent/cron defaults, and focused tests for `minimal | standard | full`.
