## Summary

Keep open. Current `main` still treats `canvas` as a node-scoped tool with snapshot-based verification, while inline Control UI rendering uses the separate `[embed ...]` path. I did not find strong evidence that the reported regression was fixed on `main` or shipped after `2026.4.15`, and the issue is too recent for `stale_insufficient_info`.
