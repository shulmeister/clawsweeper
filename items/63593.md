## Summary

Keep open. Current main still has the unresolved-tag downgrade behavior this PR targets: `--tag latest` can resolve to `targetVersion == null`, and the downgrade gate still treats any null target version as downgrade risk for registry-backed tags. The current tests cover resolved downgrade confirmation, but not the unresolved `latest` lookup regression. The PR discussion also shows the bot-raised non-latest dist-tag concern was addressed in the PR head, so this remains useful implementation work rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open, rebase it onto current main, and review the narrow `latest`-only downgradeRisk guard plus regression tests for unresolved `--tag latest` and unresolved non-latest dist-tags. The implementation belongs in `src/cli/update-cli/update-command.ts`, with coverage in `src/cli/update-cli.test.ts`.
