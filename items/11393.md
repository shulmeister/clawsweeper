## Summary

Current `main` already addresses the reported heartbeat-thread pollution: heartbeat OK replies are suppressed by default, heartbeats skip while the main or resolved session lane is busy, and an isolated `:heartbeat` session mode exists for dedicated heartbeat runs. The same behavior is already present in release `v2026.4.22`.

## Risks / Open Questions

- The issue title asks for isolated routing specifically; current behavior solves the reported pollution through default OK suppression and busy-session skipping, while full isolated-session routing remains opt-in via `heartbeat.isolatedSession`.

## Close Comment

Closing after Codex review. Current `main` already covers the reported heartbeat-thread pollution well enough: `src/infra/heartbeat-runner.ts:759` skips heartbeats when work is already in flight, `src/infra/heartbeat-runner.ts:803` supports a dedicated sibling `:heartbeat` session for isolated runs, and `src/infra/heartbeat-runner.ts:1003` plus `src/infra/heartbeat-visibility.ts:11` suppress `HEARTBEAT_OK` delivery by default unless `showOk` is explicitly enabled. The behavior is also documented in `docs/gateway/heartbeat.md:82`, and the shipped `v2026.4.22` tag already contains the same implementation. If you still hit a concrete channel-specific leak on a current build, please open a new issue with the channel/plugin, config, and a minimal reproduction so maintainers can inspect that path directly.
