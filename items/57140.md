## Summary

Current `main` does not implement the PR's normalization fix for OpenAI websocket transport. `run.ts` only trims the requested provider, while `shouldUseOpenAIWebSocketTransport` still requires exact `"openai"` and `"openai-responses"` matches, so mixed-case or padded provider values can still miss the websocket path. The PR is also stale against current `main`: stream-resolution logic has moved, and current code now explicitly keeps `openai-codex` on HTTP, so the patch would need rework rather than cleanup-closing.
