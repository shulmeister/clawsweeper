## Summary

Keep open. Current `main` already landed the inbound `threadId` capture plus revoked-context thread routing in `sendMSTeamsMessages`, but it does not subsume this PR's remaining proactive paths. `send.ts` still sends proactive text/file activities as top-level channel posts, and `replyStyle="thread"` without a live context still throws, so the PR still contains useful work beyond what is on `main`.
