## Summary

Current main already solves the PR's doctor false-positive problem in a stronger way than the PR diff: memory-search config resolution no longer calls the runtime capability-provider resolver, so it cannot fall back to a synthesized compat config that drops plugins.allow or plugins.entries. The linked #68088 root-cause analysis was valid, and the author's follow-up addressed the Greptile spy-cleanup comment, but the PR is now obsolete on main.

## Best Possible Solution

Close this PR as implemented on main. Keep the current runtime-free memory-search config-resolution design, where actual runtime capability loading stays in embedding execution paths and doctor reports the inactive-memory informational result for plugins.slots.memory="none" or absent memory backend state. The linked #68088 can be closed separately against the same shipped evidence.
