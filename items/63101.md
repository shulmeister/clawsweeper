## Summary

Keep open. Current main still validates Feishu channel config with a strict schema and I did not find a Feishu doctor/compat migration that removes or migrates the legacy keys reported here. The related open PR #63474 only tracks part of the original `installs`/`plugins` case, while later comments and related issue #65177 show additional legacy Feishu keys (`ackReaction`, `threadSession`, `mediaLocalRoots`, `botName`) that are not covered by current main.

## Best Possible Solution

Keep this issue open until maintainers either land a consolidated Feishu upgrade migration/doctor fix or split the remaining work into canonical issues. The likely fix belongs in a Feishu channel doctor/contract compatibility surface plus regression tests for top-level and account-level stale keys, with upgrade docs/changelog guidance for intentionally removed config fields.
