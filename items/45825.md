## Summary

Current main implements the Control UI compaction/context-notice recovery path described in #45825. The issue comments narrowed the vague stuck-chat report to compaction indicators, `.context-notice`, and the chat input redraw area; those surfaces are now bounded in layout, use fresh context totals, clear stale compaction state, and the same implementation is present in the shipped v2026.4.24 tree. The linked related issues #45794 and #45802 were already closed as implemented for the same Control UI render/stuck family.

## Evidence

- **Issue discussion narrows the repro:** The comments identify the concrete failure as a Control UI render/stuck state around compaction: dsantoreis linked #45794/#45802, Jettlin927 reproduced it by refreshing while `.compaction-indicator` was active, and LaurensBosscher confirmed deleting `.context-notice` or the affected `.agent-chat__input` temporarily recovered the UI.
