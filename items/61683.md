## Summary

Keep open. Current main has partial plugin-local recovery behavior, but it still fails gateway startup for plugin-entry config validation errors instead of marking only that plugin degraded.

## Evidence

- **Global plugin config validation still fails snapshots:** Plugin validation runs when a plugin is enabled or has explicit config, validates the plugin schema, and pushes failures as `plugins.entries.<id>.config...` issues. Those issues make config validation fail globally.
  - file: [src/config/validation.ts:1197](https://github.com/openclaw/openclaw/blob/20c3177281d292d3db208b1e875e4b51a14fec94/src/config/validation.ts#L1197)
  - command: `nl -ba src/config/validation.ts | sed -n '1188,1236p'`
  - sha: [20c3177281d2](https://github.com/openclaw/openclaw/commit/20c3177281d292d3db208b1e875e4b51a14fec94)
- **Gateway startup still asserts full config validity:** After limited recovery attempts, startup calls `assertValidGatewayStartupConfigSnapshot`; that helper throws `Invalid config at ...` for any invalid snapshot. `prepareGatewayStartupConfig` repeats the same assertion before auth/runtime startup.
  - file: [src/gateway/server-startup-config.ts:109](https://github.com/openclaw/openclaw/blob/20c3177281d292d3db208b1e875e4b51a14fec94/src/gateway/server-startup-config.ts#L109)

- **Existing test documents plugin-local startup abort:** The test `does not restore last-known-good for plugin-local startup invalidity` builds an invalid snapshot whose only issue is under `plugins.entries.feishu`, then expects startup config loading to reject with `Invalid config at ...`.
  - file: [src/gateway/server-startup-config.recovery.test.ts:123](https://github.com/openclaw/openclaw/blob/20c3177281d292d3db208b1e875e4b51a14fec94/src/gateway/server-startup-config.recovery.test.ts#L123)
