## Summary

This should close as ClawHub/community-plugin scope. Telegram group bot-to-bot visibility cannot be fixed by core mention routing because Telegram Bot API does not deliver other bot messages to bots in groups; current OpenClaw only normalizes bot-to-bot Telegram traffic for channel posts. For transports that do deliver such messages, the routing/turn-taking policy belongs in channel/plugin code, and the related #18869 discussion already linked a community listen-only plugin.

## Best Possible Solution

Close the core issue and route this to ClawHub/community plugin work. Keep current Telegram Bot API behavior as-is, use channel-owned bot-message controls where the platform supports them, and implement Telegram-specific group workarounds as an external plugin/bridge or documented channel-based workflow rather than a core mention-router.
