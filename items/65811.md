## Summary

Keep open. Current `main` does not include an ATOM provider, ATOM is not documented as a supported bundled provider, and the request matches an existing bundled self-hosted provider pattern already used for `vllm` and `sglang`. The issue is recent and references an active implementation PR, so there is no conservative close path.
