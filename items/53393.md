## Summary

Keep open. Current `main` does not implement the full config-derived hot-reload invalidation described by PR #53393, so `implemented_on_main` does not apply. In particular, hot reload still clears only the model catalog cache; the context-window runtime state remains sticky across reloads unless test-only reset helpers are called. The PR also appears to need refresh before merge because the existing review points out an unresolved stale in-flight context-loader race, and `models.json` cache handling on `main` has already evolved beyond the PR’s original assumptions.

## Evidence

There is no production invalidation call for context-window state or models-json ready cache in this handler.

## Risks / Open Questions

- The PR has an unresolved P1 review comment: resetting the context cache without guarding/canceling an in-flight loader can let stale model data repopulate `MODEL_CONTEXT_TOKEN_CACHE` after hot reload.
- Because `src/agents/models-config.ts` has already moved to fingerprint-based ready-cache handling on `main`, the PR’s `models.json` invalidation piece may now be redundant or conflict with current code and tests.
