## Summary

Current `main` already skips the Tailscale status probe when `gateway.tailscale.mode` is `off`, and that behavior is present in shipped release `v2026.4.22`, so this PR is obsolete.

## Evidence

- **`status --all` now uses the shared status-scan pipeline:** `statusAllCommand` delegates to `collectStatusScanOverview(...)` and no longer contains its own direct Tailscale probe path.
  - file: [src/commands/status-all.ts:13](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/commands/status-all.ts#L13)

- **Off mode short-circuits the Tailscale lookup:** `createStatusScanCoreBootstrap()` sets `tailscaleDnsPromise` to `Promise.resolve(null)` when `tailscaleMode === "off"`; only non-off modes call `getTailnetHostname(...)`.
  - file: [src/commands/status.scan.bootstrap-shared.ts:86](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/commands/status.scan.bootstrap-shared.ts#L86)
