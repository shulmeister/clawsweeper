## Summary

Keep open. Current `main` does not already ship Feishu message recall/delete, so this is not closable as implemented. The PR also still has unresolved integration problems against current `main`: it adds `tools.message` in the runtime defaults/types but not the strict Feishu config schema, registers the new tool with a Zod object instead of the TypeBox/JSON-Schema shape current tool plumbing expects, hard-wires execution to `accounts[0]` instead of the newer account-aware Feishu tool path, and is based on an outdated Feishu entrypoint shape that no longer matches current `extensions/feishu/index.ts`.
