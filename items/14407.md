## Summary

Keep #14407 open. Current main has partial web reveal controls for gateway credentials, login, and generic config secrets, but the requested cross-platform password visibility behavior is still missing from the Skills API-key input and native iOS/Android gateway credential fields. The related PR #7124 is closed unmerged in the provided GitHub context, so there is no shipped implementation to close against.

## Best Possible Solution

Keep the issue open until OpenClaw either implements a consistent sensitive-input reveal control across web Skills/config/overview/login plus the iOS and Android gateway credential fields, or a maintainer explicitly narrows the platform/product scope. The likely implementation path is to add Skills API-key reveal state in `ui/src/ui/views/skills.ts`, reusable native password reveal controls in the Swift onboarding/settings views and Android Compose gateway credential fields, and focused UI tests for the web surfaces.
