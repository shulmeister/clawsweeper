## Summary

Keep #14407 open. Current main still only has partial Control UI reveal controls; the requested cross-platform password visibility pattern remains missing from Skills/API-key UI and native macOS/iOS credential fields, and Android manual credential fields do not implement a masked/reveal toggle. The linked PR #7124 is closed and unmerged, so there is no canonical shipped fix to close against.

## Best Possible Solution

Keep the issue open and implement a reusable cross-platform sensitive-input reveal pattern: extend the existing Control UI sensitive-field toggle to Skills/API-key fields, add SwiftUI reveal wrappers for macOS and iOS SecureField credential entries, add Android Compose password fields with visualTransformation plus trailing visibility icons, and add focused UI or view-level coverage for each touched credential surface.
