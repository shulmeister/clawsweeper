## Summary

Keep #14407 open. Current main has partial web reveal controls for gateway credentials and generic config secrets, but the requested cross-platform password visibility toggle is still not implemented across all named surfaces. Skills API keys, iOS gateway password fields, and Android gateway password fields still lack the proposed masked/plain eye-toggle behavior, and the related PR #7124 was closed unmerged.

## Best Possible Solution

Keep the issue open until OpenClaw either implements a consistent sensitive-input reveal control across web Skills/config/overview/login plus iOS and Android gateway credential fields, or a maintainer narrows the product scope. The likely code path is to add Skills API key reveal state in `ui/src/ui/views/skills.ts`, reusable native password reveal controls in the iOS onboarding/settings views and Android Compose gateway credential fields, and focused UI tests for the web surfaces.
