## Summary

Keep #14407 open. Current main has partial web coverage for gateway/login/config sensitive fields, but the requested cross-platform password reveal pattern is still missing from Skills API-key entry and several macOS, iOS, and Android credential surfaces. The linked PR #7124 is closed and unmerged, and the maintainer discussion already recorded the same keep-open direction.

## Best Possible Solution

Keep the issue open and implement a reusable sensitive-input reveal pattern across platforms: extend existing Control UI reveal controls to Skills/API-key fields, add SwiftUI reveal wrappers for macOS and iOS SecureField credential entries, add Android Compose password fields with PasswordVisualTransformation and trailing visibility icons, and cover the touched credential surfaces with focused UI/view tests.
