## Summary

Keep open. Current main has partial fixes for cached plugin loads and activate:false snapshot loads, but it still does not implement the PR's central active scope-swap behavior: memory capability is still part of scope-cleared memoryPluginState, and activating a scoped load that excludes the memory plugin clears it. The PR discussion is useful evidence for OC-40, and PRtags links related #68041, but I did not find a merged implementation or a clearly canonical superseding item that covers the full remaining behavior.

## Best Possible Solution

Keep the item open, then reconcile it with #67063/#68041. The best code direction is a narrow regression-tested fix for OC-40: preserve or intentionally rehydrate the selected memory capability across active plugin scope swaps that omit the memory plugin, while still clearing stale capability state on full loader resets, config changes to plugins.slots.memory="none", failed registration rollback, cache hits, and activate:false snapshot loads. The CLI memory-slot inclusion should either land as the canonical #67063-style scoped-loader fix or be split from this PR.
