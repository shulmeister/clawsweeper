## Summary

Current `main` already does pre-send context estimation and proactive overflow recovery before the provider call, which covers the core request in #42866.

## Evidence

- **Pre-send token estimation helper:** `src/agents/pi-embedded-runner/run/preemptive-compaction.ts` estimates prompt tokens from history + system prompt + user prompt, compares them to the effective context budget, and chooses a proactive recovery route (`fits`, `compact_only`, `truncate_tool_results_only`, or `compact_then_truncate`).
  - file: [src/agents/pi-embedded-runner/run/preemptive-compaction.ts:19](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/agents/pi-embedded-runner/run/preemptive-compaction.ts#L19)

  - file: [src/agents/pi-embedded-runner/run/attempt.ts:2190](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/agents/pi-embedded-runner/run/attempt.ts#L2190)

## Close Comment

Codex review: current `main` already implements the core feature requested here. `src/agents/pi-embedded-runner/run/preemptive-compaction.ts` estimates prompt tokens against the context budget and selects a proactive recovery route; `src/agents/pi-embedded-runner/run/attempt.ts:2190` runs that check before prompt submission and sets `skipPromptSubmission = true` when the request would overflow; and `src/agents/pi-embedded-runner/run.ts:933` plus `src/agents/pi-embedded-runner/run.ts:1081` retry after preflight truncation/compaction instead of sending the oversized request first. There is matching regression coverage in `src/agents/pi-embedded-runner/run/preemptive-compaction.test.ts` and `src/agents/pi-embedded-runner/run.overflow-compaction.loop.test.ts`. The one detail that differs from the issue body is that unrecoverable overflow currently points the user to `/reset` or `/new` rather than auto-starting a fresh session, but the proactive pre-send estimation and compaction behavior itself is already on `main`, so this looks implemented.
