## Summary

Keep open. Current main still contains the same risky WhatsApp auth-dir implementation shape: `WA_WEB_AUTH_DIR` is resolved eagerly at module load, and account auth directory discovery still calls `resolveOAuthDir()` through ambient `process.env` instead of an explicit runtime env/state dir. The linked fix PR #64563 was closed unmerged, and the current docs still promise per-profile state/credential isolation.

## Best Possible Solution

Keep this issue open and revive or rework #64563 in the WhatsApp plugin: make `WA_WEB_AUTH_DIR` lazy or function-backed, route default and legacy auth-dir resolution through explicit env/state-dir inputs, add a regression test for `OPENCLAW_STATE_DIR`/`--profile` isolation after import boundaries, and then consider a doctor warning plus public docs for `OPENCLAW_OAUTH_DIR` as an operator workaround.
