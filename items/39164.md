## Summary

Current `main` already treats exec-output compaction as plugin territory, and the repo exposes plugin hooks that can rewrite tool params and tool results without adding a new core `exec-host` dependency or config surface. An `rtk` integration is plausible, but it fits the existing plugin/ClawHub path better than OpenClaw core.

## Risks / Open Questions

- This closes a core feature request on scope grounds, not because `rtk` itself is implemented on `main`.
- An `rtk` plugin may still need some packaging or distribution work outside this repo if the author wants a polished ClawHub release.
- The existing bundled tokenjuice path is Pi-oriented; a future `rtk` plugin might need additional harness-specific wiring for non-Pi runtimes.
