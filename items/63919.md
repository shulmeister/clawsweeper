## Summary

Keep open. Current main still does not wire `createOpenClawCodingTools` into `resolveGatewayScopedTools`, so this PR addresses a real gap rather than duplicating shipped behavior. The PR discussion also shows useful iteration: the initial MCP loopback exposure and duplicate agent-id concern were addressed in the PR head, but later hook/wrapper concerns still need maintainer review before landing.

## Best Possible Solution

Keep this PR open for maintainer review and iteration. The best path is to land the capability only after resolving or disproving the hook-wrapper issues: either construct an HTTP-safe unwrapped coding-tool set, or teach `/tools/invoke` to detect already-wrapped tools and consume adjusted params correctly. Preserve the `surface === "http"` guard, keep Gateway tools taking precedence, and add tests for coding-tool availability when allowlisted, loopback non-expansion, single `before_tool_call` execution, and no retained adjusted params.
