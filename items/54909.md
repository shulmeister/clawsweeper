## Summary

Current main still has the documented Telegram callback text mismatch: generic inline-button callbacks are handled, but the fallback forwards raw callback data to the agent instead of `callback_data: <value>`. Because PR #54962 is still open and explicitly targets this behavior, this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #54962 or an equivalent patch in `extensions/telegram/src/bot-handlers.runtime.ts` that forwards generic Telegram callback fallback text as `callback_data: ${data}` while preserving native command callbacks. Add or update a regression test that asserts the documented prefix, then close this issue after the fix merges.
