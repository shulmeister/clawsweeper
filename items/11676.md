## Summary

Keep #11676 open. Current main still requires a Node-hosted TypeScript Gateway/CLI runtime, and the plugin plus agent-harness surfaces are in-process Node/TypeScript extension points rather than a stable runtime-agnostic agent-core protocol for bare-metal, C/Rust/WASM, or ESP32-class ports. The linked MimiClaw project is useful external prior art, but it does not make this implemented in OpenClaw core or closeable as ClawHub work. Related #45496 is a narrower closed browser/WASM request, not a canonical tracker for this embedded/no-Node design request.

## Best Possible Solution

Keep #11676 open for explicit maintainer product/API judgment. A strong path would define a stable runtime-agnostic agent/core protocol with schemas for tool calls, memory/session state, streaming/events, auth/model selection, platform adapters, and conformance tests or a small reference implementation proving the protocol can run outside the Node-hosted OpenClaw runtime. Keep MimiClaw linked as useful prior art; do not treat the closed #45496 browser/WASM issue as the canonical tracker.
