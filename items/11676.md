## Summary

Keep #11676 open. Current main still requires a Node-hosted OpenClaw CLI/Gateway and does not expose the requested runtime-agnostic agent core/protocol for bare-metal, C/Rust/WASM, or microcontroller ports. The existing agent harness/plugin surfaces are useful but remain experimental in-process TypeScript/Node extension points, and the related closed #45496 is narrower browser/WASM work rather than a canonical embedded/no-Node plan.

## Best Possible Solution

Keep the issue open for explicit maintainer product judgment. A strong path would define a stable runtime-agnostic agent/core protocol with schemas for tool calls, memory/session state, streaming/events, auth/model selection, and platform adapters, then add conformance tests or a small reference client proving the protocol can run outside the Node-hosted OpenClaw runtime. MimiClaw should remain linked as useful external prior art.
