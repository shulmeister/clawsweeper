## Summary

Keep #11676 open. Current main still ships OpenClaw as a Node-hosted TypeScript application, and the available agent runtime/plugin harness surfaces are trusted in-process OpenClaw extension points rather than a stable runtime-agnostic agent core/protocol for no-Linux/no-Node embedded devices. The linked MimiClaw project remains useful prior art, and the existing issue discussion already records a maintainer-side Codex review choosing to keep this product-direction item open.

## Best Possible Solution

Keep the issue open until maintainers make an explicit runtime-portability decision. A strong path would define a stable runtime-agnostic agent/core protocol with schemas for tool calls, memory/session state, streaming/events, and platform adapters, then add compatibility tests or reference clients proving it can be implemented outside the Node-hosted OpenClaw runtime. MimiClaw should remain cited as useful prior art.
