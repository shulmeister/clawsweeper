## Summary

Keep #11676 open. Current main still defines OpenClaw as a Node-hosted application and the available agent/plugin surfaces are for in-process trusted OpenClaw plugins or protocol clients, not a stable runtime-agnostic core suitable for no-Linux/no-Node embedded devices. The linked MimiClaw project remains useful prior art, but it does not implement the requested portability in this repo, and the latest issue discussion already records a maintainer-side Codex review choosing to keep this product-direction item open.

## Best Possible Solution

Keep the issue open until maintainers either define a runtime-agnostic agent/core protocol or explicitly decide that embedded/no-Node ports should remain outside OpenClaw core. A useful implementation path would document a stable portable protocol boundary, separate platform adapters from agent-loop state/tool/memory contracts, and add compatibility tests or reference clients proving the interface can be implemented outside the Node runtime.
