## Summary

Keep open. Current `main` does not contain this fix, and the PR still carries unresolved regression risk outside the reported bug. `sendMessageFeishu()` still wraps outbound post messages in a `tag: "md"` payload but prefixes mentions with the rich-text formatter instead of the lark_md/card formatter, so the reported behavior is not already implemented on `main`. At the same time, the PR diff changes the shared card mention formatter used by existing card send paths, and current parser/test coverage does not prove that broader behavior change is safe.

## Evidence

- **Current main still uses the old mention formatter for post messages:** `buildFeishuPostMessagePayload()` emits post content as a single `tag: "md"` node, but `sendMessageFeishu()` still calls `buildMentionedMessage(...)` before that payload is built. That means the requested swap is not present on current `main`. See `extensions/feishu/src/send.ts:432` and `extensions/feishu/src/send.ts:454`.
