## Summary

Keep open. Current main still does not implement the Anthropic billing wording fix, and the duplicate-reply fix remains unresolved for multi-handle same-person targets. The PR also needs rework because its unconditional text-only dedupe conflicts with current cross-target delivery behavior and existing regression tests.

## Best Possible Solution

Keep this PR open for rework or split it into two focused fixes: add a precise Anthropic billing classifier for the "specified API usage limits" / "regain access" wording while preserving periodic usage-window rate_limit behavior, and solve multi-handle duplicate suppression through identity-aware target matching or a channel/plugin alias hook rather than unconditional text-only dedupe.
