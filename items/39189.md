## Summary

Keep open. Current main still does not clearly implement the Anthropic billing-classification fix, and the duplicate-reply change in this PR is not present on main because current code intentionally preserves final replies for cross-target message-tool sends. The PR remains useful as a bug report, but the patch needs rework rather than conservative closure.

## Best Possible Solution

Keep this PR open for maintainer or contributor rework. The billing half should add a narrow Anthropic-specific matcher and regression tests that keep periodic usage windows in rate_limit. The duplicate-reply half should solve multi-handle identity matching in shouldSuppressMessagingToolReplies or a channel/account alias seam, while preserving the existing cross-target same-text tests.

## Risks / Open Questions

- Anthropic "specified API usage limits" errors likely still classify as rate_limit on current main, causing retry/cooldown behavior instead of billing failover behavior.
- The multi-handle duplicate-final-reply symptom may still exist, but the PR's current fix shape can incorrectly suppress legitimate same-text replies sent to a different channel or target.
- A safe fix likely needs two narrower changes, with tests: precise Anthropic billing matching/classification order, and identity-aware same-person reply suppression that preserves cross-target delivery tests.
