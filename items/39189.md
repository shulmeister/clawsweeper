## Summary

Keep open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still lacks the Anthropic billing wording fix, and the multi-handle duplicate-reply behavior is not resolved. The PR’s current unconditional text-dedupe approach also remains unsafe because current tests intentionally preserve same-text replies sent to a different target.

## Best Possible Solution

Keep this PR open for rework or split it into two focused changes: add a narrow Anthropic billing matcher for the "specified API usage limits" / "regain access" wording while preserving periodic usage-window rate_limit behavior, and fix multi-handle duplicate suppression through identity-aware target matching or a channel/plugin alias hook rather than unconditional text-only dedupe. Add regression tests for both the Anthropic sample and the multi-handle case, plus a negative cross-target same-text test.
