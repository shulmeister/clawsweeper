## Summary

Keep #43238 open. Current main at a57d681db9b237348c5d446c8d371b2c1057f016 still only supports `contextPruning.mode: "cache-ttl"`; `rolling` is absent from the runtime settings parser, embedded-runner factory gate, config type, Zod schema, docs, and tests. The PR addresses a real linked feature request (#33448) and the PR discussion shows the original no-op runtime bug was addressed in the PR, but the feature has not shipped or been superseded on main.

## Best Possible Solution

Keep the PR open and ask for a current-main refresh rather than closing it. The best path is to rebase onto a57d681db9b237348c5d446c8d371b2c1057f016 or newer, apply the rolling-mode changes to the current `pi-hooks` paths, add `rolling` consistently to config types/Zod/generated schema and docs, keep the provider eligibility gate scoped to `cache-ttl`, and add targeted tests proving rolling mode registers the extension and prunes on each context event without the TTL gate.
