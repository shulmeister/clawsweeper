## Summary

Keep #10118 open. Current main partially covers multiline input because the pinned pi-tui editor defaults to Shift+Enter for newline and Enter for submit, and OpenClaw preserves multiline payloads after submit. The remaining requested user-facing work is still missing: TUI docs and /help do not mention Shift+Enter, there is no OpenClaw config/CLI keybinding bridge for Enter vs Ctrl/Cmd+Enter behavior, and the maintainer discussion explicitly kept this issue open with related PR #43327 still open/unmerged.

## Best Possible Solution

Keep the issue open and narrow it to the remaining user-visible work: document Shift+Enter in `docs/web/tui.md` and TUI `/help`, add focused assertions for shortcut/help visibility, and decide whether OpenClaw should expose configurable Enter versus Ctrl/Cmd+Enter send behavior through config or CLI wiring to pi-tui keybindings. Treat PR #43327 as related prior work while it remains open/unmerged, not as a shipped or superseding fix.
