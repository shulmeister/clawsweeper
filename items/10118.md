## Summary

Keep open. Current main has dependency-backed Shift+Enter newline support while Enter still submits, but the issue also asks for help/docs visibility and optional/configurable send/newline behavior. Those pieces are still missing, and the later discussion specifically asks for Ctrl+Enter/configurable and IME-friendly behavior.

## Best Possible Solution

Keep the issue open and narrow the next change: document Shift+Enter in docs/web/tui.md and /help, add focused tests for the help/docs-visible shortcut text, and make an explicit maintainer product decision on configurable Enter-vs-Ctrl/Cmd+Enter behavior using a real OpenClaw config or pi-tui keybinding surface. Treat PR #43327 as related but not canonical in its current open/unmerged form.
