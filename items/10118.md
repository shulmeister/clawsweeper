## Summary

Keep #10118 open. Current main already gets basic Shift+Enter newline and Enter submit from pinned pi-tui, and OpenClaw preserves multiline submitted text. But the issue also asks for docs/help visibility and optional configurable send/newline behavior; current TUI docs and slash help still omit Shift+Enter, and OpenClaw has no config/keybinding layer wired for Ctrl/Cmd+Enter or Enter-vs-newline preferences. The Apr 26 maintainer-posted Codex review and the related open PR #43327 both support keeping this as remaining product/docs work rather than closing it as fully implemented.

## Best Possible Solution

Keep the issue open and narrow the next maintainer change: document Shift+Enter in docs/web/tui.md and slash help, add focused tests for visible shortcut text, and make an explicit product decision on configurable Enter-vs-Ctrl/Cmd+Enter behavior using pi-tui's keybinding surface or an OpenClaw config-backed wrapper. Treat PR #43327 as related but not canonical while it remains open and unmerged.
