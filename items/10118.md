## Summary

Keep #10118 open. Current main has partial multiline handling at submit time, but it still does not expose an OpenClaw-owned Shift+Enter/Ctrl+Enter or configurable send/newline keybinding, and the TUI docs plus `/help` still only document Enter-to-send. The latest provided maintainer comment also explicitly kept the issue open, with related PR #43327 still open and unmerged.

## Best Possible Solution

Keep #10118 open and narrow it to the remaining product work: document multiline composition and terminal limitations in `docs/web/tui.md` and TUI `/help`, add focused shortcut/help assertions, and decide whether OpenClaw should expose configurable submit/newline keybindings through config/CLI wiring. Treat PR #43327 as related prior work until it is merged, updated, or explicitly superseded.
