## Summary

Keep PR #71235 open. Current main still does not add a `python` -> `python3` shim in any of the four Dockerfiles touched by the PR, and the PR is a recent, focused Docker image usability fix whose author already addressed the two review concerns in the provided discussion.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a narrowly scoped conditional shim in the affected Dockerfiles, preserving any operator-provided `python` on PATH and using `$(command -v python3)` for the symlink target; adding a small Docker smoke assertion for `python --version` would make the behavior harder to regress.
