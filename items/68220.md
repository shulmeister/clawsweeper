## Summary

Current main already blocks Telegram self-authored message updates before inbound routing, which fixes the duplicate self-trigger described in this issue for group/topic cron output. Telegram still lacks a public `allowBots` config surface, but the reported self-echo behavior is already addressed on main.

## Evidence

## Risks / Open Questions

- The specific self-echo bug is fixed on `main`, but Telegram still does not expose a general `allowBots` setting for non-self bot senders or `channel_post` traffic.
- The fix is on `main` after `v2026.4.23`, so users on the latest release tag would not have it yet.
