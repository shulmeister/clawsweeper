## Summary

Keep open. Current main still only supports the binary `tools.fs.workspaceOnly` behavior; it does not implement `tools.fs.roots`, per-root `dir`/`file` kind, `ro`/`rw` access modes, or root-scoped enforcement across FS tools and media paths. The PR is active, discusses concrete security-sensitive edge cases, and represents core config/tooling work rather than a ClawHub/plugin-only item.

## Best Possible Solution

Keep the PR open. The best path is a maintainer-reviewed core implementation of `tools.fs.roots` that updates the config schema/types/help, threads a per-agent root policy through host FS tools, `apply_patch`, prompt/media loading, and outbound delivery, and keeps the sandbox boundary explicitly separate. If the current PR is too large, split it into smaller root-policy, FS-tool enforcement, and media-propagation changes rather than closing the request.
