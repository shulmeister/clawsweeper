## Close Comment

Closing this as a `clawhub` item after Codex review. The change here is Feishu-channel special handling for one external skill (`/legion-*` -> `legion-cli`), but current `main` already has the generic path this should use instead: skill command registration in `src/agents/skills/command-specs.ts:94`, text skill-command parsing in `src/auto-reply/skill-commands-base.ts:59`, and direct tool dispatch without model fallback in `src/auto-reply/reply/get-reply-inline-actions.ts:228`. The project scope also explicitly says optional capabilities should usually ship as plugins and that new skills should go to ClawHub first (`VISION.md:54-70`, `VISION.md:99-105`). Because this PR hardcodes an external `agentlegion` workflow into bundled Feishu channel code, we should keep that behavior in the AgentLegion skill/plugin rather than merge a Feishu-specific core special case.
