## Summary

Current `main` still routes PDF-tool model execution through a registry-only lookup, while the broader agent runtime has a separate config-aware resolver for custom providers. I found no evidence on `c150110e02550eaf6c56db0c851dc62fc0ca88e1` or the latest shipped release `v2026.4.23` that this PDF-specific mismatch has been fixed, so this issue should stay open.
