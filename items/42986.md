## Summary

Keep open. Current `main` still lacks Telegram `subagent_spawning` hook wiring, `sessions_spawn(..., thread: true)` still hard-fails when no channel hook is registered, and the docs still state Discord is the only supported channel for thread-bound subagent sessions.

## Evidence

- **Spawn path still rejects missing subagent hook:** `ensureThreadBindingForSubagentSpawn` returns the same failure the issue reports when no channel plugin has a `subagent_spawning` hook: `thread=true is unavailable because no channel plugin registered subagent_spawning hooks.`
  - file: [src/agents/subagent-spawn.ts:495](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/agents/subagent-spawn.ts#L495)
  - command: `nl -ba src/agents/subagent-spawn.ts | sed -n '488,590p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Telegram entry does not register subagent hooks:** The bundled Telegram entry only exposes plugin/runtime/accountInspect metadata. Unlike Discord, it has no `registerFull(...)` callback and no subagent hook registration.
  - file: [extensions/telegram/index.ts:1](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/extensions/telegram/index.ts#L1)

  - file: [extensions/discord/index.ts:1](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/extensions/discord/index.ts#L1)
