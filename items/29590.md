## Summary

Current main still does not bridge tool execution into the internal HOOK.md handler system. The PR is not implemented or clearly superseded; it remains a focused implementation attempt, but needs rebase and maintainer judgment because current docs route tool-call interception to typed plugin hooks.

## Best Possible Solution

Keep this PR open until maintainers either merge a corrected internal tool-hook bridge or explicitly decide that operator-installed internal hooks should remain command/session/agent/gateway/message-only. If implemented, rebase onto current main, add a typed tool:after_call internal event in src/hooks/internal-hook-types.ts/internal-hooks exports, emit it once from handleToolExecutionEnd using ctx.params.sessionKey and the adjusted afterToolCallArgs payload, and add docs plus focused regression coverage.
