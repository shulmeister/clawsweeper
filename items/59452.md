## Summary

Current `main` still lacks a `--dry-run`/preview path for `openclaw cron add`. The command validates schedule syntax client-side, but it always proceeds to `cron.add` on the gateway, and the cron docs/examples do not document any preview mode. The linked discussion also points to an open, unmerged PR (#59490), which is evidence the request is still active rather than already shipped.

## Evidence
