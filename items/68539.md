## Summary

Current main does not implement the requested openai-codex voice transcription path. The bundled OpenAI media-understanding provider advertises `openai-codex` as image-only, the native Codex app-server media provider is also image-only, and the audio runner skips providers that do not implement `transcribeAudio`. This also is not a clean ClawHub close because `openai-codex` is already owned by the bundled OpenAI plugin and duplicate provider registrations are rejected.

## Best Possible Solution

Keep the issue open for maintainer/product decision. The best implementation would live in the bundled OpenAI/Codex provider layer: add `openai-codex` audio media-understanding support only after verifying a supported Codex/ChatGPT OAuth transcription path, with tests proving it uses the OpenAI Codex auth profile rather than `OPENAI_API_KEY`, advertises audio capability in code and manifest metadata, and does not silently fall back to paid OpenAI Platform STT. If upstream does not expose a durable path, document the unsupported behavior and recommend explicit `tools.media.audio` alternatives.
