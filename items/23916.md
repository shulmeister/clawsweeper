## Summary

Keep #23916 open. Current main still has the underlying paired-device metadata, but the user-facing `openclaw devices list` surface remains a compact table without client purpose/platform detail, `--full`/describe output, rename, or notes. The related #13870 and PR #55973 are narrower and do not supersede the broader request.

## Best Possible Solution

Keep #23916 open and implement a core CLI device-detail path: add a safe compact purpose/client hint to `openclaw devices list`, add `--full` or `openclaw devices describe` for client mode/id, platform/device family, timestamps, and safe host/IP details, and coordinate editable display names/notes with #13870 while updating CLI tests and docs without changing existing JSON semantics.
