## Summary

Keep PR #39630 open. Current main still does not expose the requested broad `tools.web.fetch.allowPrivateNetwork` opt-in for `web_fetch`; it only has the narrower `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange` exception and continues to block localhost/private/internal destinations. The linked feature issue #39604 remains the canonical open request, and the PR discussion has unresolved product/security policy questions about metadata endpoints and configurable exceptions.

## Best Possible Solution

Keep PR #39630 open for maintainer direction or revision. The best path is to implement a documented private-network opt-in aligned with the current `tools.web.fetch.ssrfPolicy` surface, thread it through strict schema, public types, generated config metadata, runtime guard policy, cache partitioning, tests, docs, and changelog, and explicitly resolve the metadata-endpoint policy with #39604 as the canonical feature request.
