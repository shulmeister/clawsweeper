## Summary

Keep open. Current main still does not expose the requested `tools.web.fetch.allowPrivateNetwork` opt-in for `web_fetch`; it only exposes the narrower `ssrfPolicy.allowRfc2544BenchmarkRange` escape hatch. The linked feature issue #39604 remains open, and the PR discussion still contains unresolved security/product questions around metadata endpoint handling, so this is not obsolete or safely closeable.

## Best Possible Solution

Keep the PR open for maintainer direction or revision. A strong path would add a documented `web_fetch` private-network opt-in across schema, generated config metadata, types, runtime, cache partitioning, tests, docs, and changelog, while explicitly resolving metadata endpoint policy before merge. Continue treating #39604 as the canonical feature request until the product decision is made.
