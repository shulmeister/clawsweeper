## Summary

Keep open. Current main has not implemented the PR's requested behavior: `plugins.enabled: false` is still treated as an authoritative global plugin disable, and source, tests, and docs all encode that contract. The PR is a plausible focused fix for open bug #68249, but it is not merge-ready as-is because the activation policy is now centralized and the review comments identify real policy/surface gaps that need maintainer/product resolution rather than automated closure.

## Best Possible Solution

Keep the PR open for maintainer review and require a reworked patch if the desired product behavior is that explicit per-plugin enablement can override the global plugin kill switch. The rework should change the shared activation policy, update config-state/config-policy consumers consistently, preserve or explicitly redesign the default memory-slot behavior, and update docs/tests together. If maintainers decide the documented global kill switch must remain authoritative, close the PR in favor of tracking the Brave/web_search report in #68249 with a different fix path.
