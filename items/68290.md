## Summary

Keep open. Current main still treats `plugins.enabled: false` as an authoritative global disable, and this PR proposes changing that contract. The repo’s current tests and docs still encode the existing behavior, and the PR discussion surfaced a plausible regression around the default `memory-core` slot. The idea may still be worth a maintainer decision, but there is not high-confidence evidence that it is already implemented or safe to auto-close.

## Evidence

- **Current activation logic still hard-disables before explicit enablement:** `resolvePluginActivationDecisionShared` returns `plugins-disabled` as soon as `params.config.enabled` is false, before the later `explicitSelection.explicitlyEnabled` branch. On current `main`, this preserves explicit-selection metadata without activating the plugin.
