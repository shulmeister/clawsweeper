## Summary

Current main already covers the PR’s observable image-model lookup problem without taking this PR’s exact patch. The shipped/main path preserves the `resolveProviderVisionModelFromConfig()` `provider/model` contract that review comments flagged as necessary, parses configured image-model refs before runtime, and resolves configured provider models through the shared registry/config resolver instead of a direct discovery-only lookup.

## Best Possible Solution

Close PR #68919 as implemented on main and do not merge its stale one-file runtime patch. Keep the current main approach: preserve the `provider/model` helper contract for image/PDF callers, parse configured image-model refs before invoking the runtime, and rely on the shared configured-model resolver for custom provider models.
