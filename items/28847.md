## Summary

Keep open. Current main has meaningful auth-profile rotation, per-profile cooldowns, rate-limit classification, and model fallback, but it still does not implement the requested Gemini/project quota-bucket grouping. The provided issue discussion is useful prior art: pyro0ownz linked a working sidecar, and aashari linked a fork with model-health cooldowns, but that comment explicitly says bucket/project grouping is not natively implemented there either.

## Best Possible Solution

Keep this issue open until OpenClaw either implements first-class quota-bucket metadata and bucket-level cooldown fanout in the auth-profile/provider failover path, or maintainers explicitly choose a plugin/sidecar design and add any missing extension API needed for that implementation.

## Evidence

- **Requested behavior includes bucket-aware Gemini project grouping:** The issue asks for provider key pools, 429/RESOURCE_EXHAUSTED cooldowns, and shared quota buckets so a 429 cools the whole Gemini project bucket before trying another bucket. The aashari comment is prior art for cooldowns, but it explicitly says Gemini bucket/project grouping is not native in that fork.
- **Public auth config has no quota bucket field:** AuthProfileConfig only exposes provider, mode, email, and displayName; AuthConfig exposes profiles, order, and cooldown knobs, including rateLimitedProfileRotations, but no bucket, quota group, project group, or shared cooldown key.
