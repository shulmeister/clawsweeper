## Summary

Keep open. Current main has adjacent auth-profile rotation, per-profile cooldown/backoff, rate-limit classification, and env-list API key rotation for some execution paths, but it still lacks the issue's central Gemini project/quota-bucket behavior: no first-class bucket metadata, no bucket-level health state, and no failure fanout that cools every profile/key in the same quota bucket before choosing another bucket. The issue's linked Keymanager sidecar and the aashari fork comment are useful prior work, but the fork comment explicitly says bucket/project grouping is not natively implemented.

## Best Possible Solution

Keep the issue open. The best product path is to add explicit quotaBucket/quotaGroup metadata for provider auth profiles or provider-owned external profiles, persist bucket-level health alongside profile health, update markAuthProfileFailure and auth-profile selection to cool/skip every profile in the same Gemini project bucket on 429/RESOURCE_EXHAUSTED, and add Gemini multi-project tests plus docs. If maintainers prefer a plugin route, first add a provider/plugin hook that exposes the failing auth profile identity and permits safe shared auth-health updates without log tailing or config rewrites.
