## Summary

Keep open. Current main has adjacent per-auth-profile rotation, rate-limit classification, cooldown/backoff, and model-scoped cooldowns, but it still does not implement the central request: Gemini project/quota-bucket metadata, bucket-level cooldown state, or failure fanout that cools every key/profile in the same bucket before selecting a different bucket. The linked Keymanager sidecar and the aashari fork comment remain useful prior work; the aashari comment explicitly says bucket/project grouping is not native there either.

## Best Possible Solution

Keep this issue open. The best product path is to add explicit quotaBucket/quotaGroup metadata for provider auth profiles or provider-owned profile metadata, persist bucket-level health alongside profile health, update failure handling and auth-profile selection so Gemini 429/RESOURCE_EXHAUSTED cools and skips every profile in the same project bucket, and add Gemini multi-project tests plus docs. If maintainers prefer a ClawHub/plugin route, first expose a supported provider hook/API that gives plugins the failing auth profile identity and lets them safely update shared bucket health without log tailing or rewriting config files.
