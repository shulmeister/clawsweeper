## Summary

Current macOS Talk mode on `main` already resolves the configured Talk provider and voice from `talk.config`, then calls ElevenLabs playback directly; system voice is only an explicit fallback. The issue’s `talk.speak` suspicion does not match the current macOS runtime.

## Evidence

- **macOS Talk playback chooses ElevenLabs directly:** `playAssistant` switches on the resolved Talk provider and calls `playElevenLabs(...)` when `provider == "elevenlabs"`; system voice is only the fallback path.
