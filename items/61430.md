## Summary

Current main does not implement the requested optional exec `description`/`intent` field. The model-facing exec schema, public docs, exec approval payload, and approval registration path still carry command/argv/context fields only. Existing plugin hooks can require approval with their own description, but the incoming `before_tool_call` event receives only `toolName` and `params`, so that does not provide the model-supplied intent requested by #61430.
