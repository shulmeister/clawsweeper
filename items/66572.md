## Summary

Current main already implements PR #66572's cfg-aware speech-provider resolution problem. The resolver no longer lets a partial active speech provider registry hide cfg-requested providers such as Microsoft/Edge: it detects requested speech provider ids and aliases from config, removes providers already active, resolves the cfg-compatible plugin registry with `activate: false`, filters loaded providers to the missing requested ids/aliases, and merges them with active providers. Regression tests now cover the active OpenAI plus cfg-requested Microsoft/Edge case and ensure unrelated bundled speech providers are not merged.

## Best Possible Solution

Close PR #66572 as implemented on current main and keep the shipped mainline resolver/tests. The current implementation is preferable to the PR patch because it preserves active providers while adding only cfg-requested missing speech providers from the compat registry, avoiding unrelated bundled provider expansion.
