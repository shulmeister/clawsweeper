The PR is not merged or implemented on main, and it includes concrete production-derived repro/test work. It should not be auto-closed, though the branch still needs maintainer review because bot review comments identify a real label/targetId metadata-preservation risk in the proposed cleanup algorithm.

## Best Possible Solution

Keep PR #69486 open and ask for a revision rather than auto-closing it. The best path is a focused runner-side cleanup that removes only rewrite-abandoned entries while preserving all live references, including `parentId` ancestry and non-parent metadata such as label `targetId` (and any analogous compaction/branch-summary references), plus regression tests for the production-derived lineage and the label-target case.
