## Summary

Keep this PR open. Current main has adjacent update, config validation, and restart-sentinel safeguards, but it still does not implement the PR's specific behavior: run a bounded doctor summary after `update.run` failures, attach that summary to restart sentinel messages, or write doctor-backed diagnostics for config-reload restart-check failures. The closed related issue #19992 was resolved for the broader restart/config preflight safety net, while the broader self-healing issue #55347 remains open and does not supersede this focused implementation candidate.

## Best Possible Solution

Keep this PR open for maintainer review or author refresh. The best path is to rebase it onto current main, preserve the bounded `update.run` doctor-summary behavior if maintainers want that user-facing diagnostic, and rework the config-reload portion so restart-check diagnostics are not stale or misleading. If maintainers choose a broader native self-healing design instead, fold this PR's useful failure-summary pieces into #55347 rather than closing the contribution as already implemented.
