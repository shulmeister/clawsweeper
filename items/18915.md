## Summary

Keep open. Current main still does not pass width/height to Telegram sendVideo in either direct outbound sends or reply delivery, so the requested bug fix is not implemented. The PR remains a plausible focused Telegram bug fix, but the submitted patch needs maintainer rework for the current plugin boundary and test seams before landing.

## Best Possible Solution

Keep this PR open as useful bug-fix work, but require a revised implementation that fits the current plugin boundary. The likely path is to expose any shared video-dimension probe through an approved plugin SDK/runtime seam or keep it inside the Telegram plugin, add regression tests for both direct sendMessageTelegram and reply delivery sendVideo width/height forwarding, exclude video notes, and split or drop the unrelated global MAX_VIDEO_BYTES change unless separately justified.
