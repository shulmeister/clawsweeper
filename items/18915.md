## Summary

Keep open. Current main still does not implement the PR's central Telegram portrait-video fix: both direct Telegram sends and reply delivery call `sendVideo` without probing or forwarding `width`/`height`. The PR is useful but needs rework for the current plugin boundary and tests before it can land.

## Best Possible Solution

Keep this PR open as focused bug-fix work. The best path is a revised implementation that stays inside the Telegram plugin or uses an approved `openclaw/plugin-sdk/*` media seam, forwards `width`/`height` for regular `sendVideo` in both direct sends and reply delivery, excludes video notes, adds regression tests for both send paths, and drops or splits the unrelated `MAX_VIDEO_BYTES` change.
