## Summary

This is actionable for the external `@larksuiteoapi/feishu-openclaw-plugin` package publisher, not for the OpenClaw source repo. The published package’s manifest id is `feishu-openclaw-plugin`, but its runtime export declares plugin id `feishu`; current OpenClaw intentionally rejects that mismatch. The in-repo/current Feishu path is the canonical `feishu` / `@openclaw/feishu` plugin.

## Best Possible Solution

Keep OpenClaw’s canonical plugin-id enforcement. Users should use the current `feishu` / `@openclaw/feishu` plugin path, while the external `@larksuiteoapi/feishu-openclaw-plugin` publisher should release a compatible package whose manifest id and runtime export id agree, ideally published or verified through ClawHub.
