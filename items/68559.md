## Summary

Do not close PR #68559. Current main already excludes exact compaction checkpoint transcript snapshots centrally, but memory-host-sdk session discovery still indexes reset and deleted transcript variants, so the PR's primary-session-only behavior is not implemented on main. The PR also needs revision before merge because its broad `.checkpoint.` substring guard could skip valid dotted session IDs.

## Best Possible Solution

Keep the PR open for maintainer judgment or contributor revision. A safe path would decide whether QMD memory indexing should stop indexing reset/deleted transcript variants, implement that with the central artifact helpers rather than a broad `.checkpoint.` substring check, and add explicit reset/deleted/checkpoint coverage for the affected memory-host-sdk session-file tests.
