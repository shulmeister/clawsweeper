## Summary

Keep open. Current main still only skips heartbeat execution when the main queue or resolved session lane has queued work; it does not use the active reply-run registry to cover the race window described here. There is also an open, unmerged PR (#64963) with closing syntax for this issue, so the maintainer path is to review/rebase/land or close that PR rather than auto-close the issue first.

## Best Possible Solution

Keep this issue open while maintainers review #64963 or apply an equivalent small fix: after heartbeat preflight resolves the target session key, have `runHeartbeatOnce` skip with `requests-in-flight` when the reply-run registry still has an active run for that session, and add the active-reply-run regression test described in the PR. Once that lands, GitHub can close this issue through the PR closing reference.
