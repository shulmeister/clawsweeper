## Summary

Current main still does not satisfy the issue. Bootstrap resolution loads standard files from `workspaceDir` only, there is no `agentDir` merge/override path, and the search found no warning for bootstrap-named files under `agentDir`. Docs now make the workspace/agentDir distinction clearer, but that is not the requested runtime diagnostic or per-agent bootstrap support. Related PRs #29450 and #29545 remain closed/unmerged in the provided context, and their head SHAs are absent from this checkout.

## Best Possible Solution

Keep #29387 open as the canonical tracker. The best implementation path is either a focused diagnostic like #29450, warning when recognized bootstrap filenames exist under an explicitly configured `agentDir` but are not injected, or a deliberate agentDir merge/override contract like #29545 with safeguards, regression tests across runtime callsites, and docs explaining workspace versus agentDir semantics.
