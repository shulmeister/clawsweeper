## Summary

Keep #29387 open. Current main still resolves bootstrap context from the effective workspace directory only; it neither loads bootstrap-named files from `agentDir` nor warns when those files are present there. The linked attempts (#29450 diagnostic warning and #29545 per-agent override/merge) are useful prior work, but the provided GitHub context shows both PRs closed unmerged, and current source still lacks their behavior.

## Best Possible Solution

Keep #29387 open as the canonical bug. The best maintainer path is either a narrow runtime diagnostic that threads explicit `agentDir` context into bootstrap callsites and warns when recognized bootstrap filenames are present there but ignored, or a deliberate per-agent bootstrap merge/override contract for explicitly configured `agentDir` files with allowlist filtering, hook precedence, full runtime caller coverage, regression tests, and docs that clearly distinguish workspace bootstrap files from agent state.
