## Summary

Keep #29387 open. Current main still resolves bootstrap context from the configured workspace only; there is no agentDir merge/override path and no diagnostic for bootstrap-named files placed in agentDir. Docs now steer users toward per-agent workspaces and describe agentDir as state, but that does not fix the reported silent runtime misconfiguration. The related fix PRs discussed in the issue, #29450 and #29545, are closed and unmerged in the provided GitHub context and their head/fix commits are not present in this checkout.

## Best Possible Solution

Keep #29387 open as the canonical bug. The best maintainer path is either a narrow runtime diagnostic that threads explicit agentDir/agentId context into bootstrap callsites and warns on recognized bootstrap filenames in agentDir, or a deliberate merge/override contract for explicitly configured agentDir files with allowlist filtering, hook precedence, caller coverage, regression tests, and docs that distinguish workspace bootstrap files from agentDir state.
