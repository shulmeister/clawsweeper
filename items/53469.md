## Summary

Keep this issue open. Current main has partial private-network support for SearXNG web_search and model-provider request transport, but the broader request is still missing for generic web_fetch and Gateway OpenResponses URL inputs. The implementation PR linked from the discussion was closed unmerged, and the closed provider-request issue only covers a narrower model-provider surface.

## Best Possible Solution

Keep this issue open and scope the remaining maintainer decision around a narrow, explicit private-network opt-in for generic web_fetch and Gateway OpenResponses input_file/input_image URL fetches. Treat the shipped SearXNG provider and models.providers.*.request.allowPrivateNetwork work as precedent, not full resolution. The implementation should preserve default blocking, add schema/help/docs, wire policy through the runtime guards, and cover web_fetch, redirects, Gateway URL inputs, and cache behavior with SSRF tests.
