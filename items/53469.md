## Summary

Keep open. Current `main` has related partial work: SSRF policy supports `allowPrivateNetwork`, trusted web-search providers can reach operator-trusted private endpoints, SearXNG supports private/loopback HTTP, and `models.providers.*.request.allowPrivateNetwork` exists for provider HTTP requests. But the issue is broader than that, and direct `web_fetch` still has no `allowPrivateNetwork` opt-in path in config or execution, while private-network targets remain blocked by default. I also did not find a generic local-gateway runtime path that automatically enables this for web tools; the only production `mode === "local"` auto-enable I found was plugin-specific (`extensions/comfy`).
