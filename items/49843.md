## Summary

Keep open. Current `main` does not show evidence that this was fixed: the gateway still returns one session row per stored key without collapsing rows that represent the same Telegram chat, and the Control UI still renders one option per unique session key, only renaming collisions rather than deduplicating them.

## Evidence

- **Gateway session listing has no cross-key dedupe:** `listSessionsFromStore` iterates `Object.entries(store)`, builds one row per entry, and sorts the result. There is no normalization or grouping step for duplicate Telegram identities or display names.
  - file: [src/gateway/session-utils.ts:1483](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/src/gateway/session-utils.ts#L1483)
  - command: `nl -ba src/gateway/session-utils.ts | sed -n '1461,1545p'`
  - sha: [320d52a23e41](https://github.com/openclaw/openclaw/commit/320d52a23e41a326287826219a20f1f29f334554)
- **Control UI dedupes only by exact key:** `resolveSessionOptionGroups` tracks `seenKeys`, so only identical keys are collapsed. When labels collide, it appends scope/group/key text to keep multiple options visible instead of removing duplicates.
  - file: [ui/src/ui/chat/session-controls.ts:459](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/ui/src/ui/chat/session-controls.ts#L459)
  - command: `nl -ba ui/src/ui/chat/session-controls.ts | sed -n '447,565p'`
  - sha: [320d52a23e41](https://github.com/openclaw/openclaw/commit/320d52a23e41a326287826219a20f1f29f334554)
- **Current tests expect duplicate options to remain visible:** The UI test named `disambiguates duplicate grouped labels with scoped suffixes` asserts that two duplicate-label sessions should both appear with suffixes, which matches the current implementation and not the issue's expected behavior of a single entry.
  - file: [ui/src/ui/app-render.helpers.node.test.ts:443](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/ui/src/ui/app-render.helpers.node.test.ts#L443)
