## Summary

Keep open. Current main at a35d259 implements the Discord autoPresence/runtime health portion, but it still does not implement a Discord self-profile/profile action for nickname/avatar updates, a `channels.discord.actions.selfProfile` gate, or Discord-specific self-only target guardrails. The merged PR #33277 explicitly scoped out self-profile/action-gate behavior, and the remaining capability is still meaningful.

## Best Possible Solution

Keep the issue open until maintainers either implement the missing Discord profile action in the bundled Discord plugin or explicitly decide the feature belongs elsewhere. A concrete implementation should add a default-off Discord action gate, expose the action in `extensions/discord/src/channel-actions.ts`, handle it in the Discord action runtime, update docs/config schema/tests, and enforce strict bot-self scope for nickname/avatar/status/profile changes.
