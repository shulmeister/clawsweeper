## Summary

Issue #12478 asks for a configuration-assistant skill. Current main explicitly routes new skills to ClawHub rather than OpenClaw core, and core already provides first-party config UX via `openclaw configure` and `openclaw config`. This belongs as a ClawHub skill/plugin, not a core issue.

## Close Comment

Closing this as a Codex maintainer review because the request fits ClawHub, not OpenClaw core. Current `main` explicitly says new skills should go to ClawHub first and that new core skills that can live on ClawHub are out of scope (`VISION.md:66-70`, `VISION.md:99-102`). Current OpenClaw already provides the supported core config surfaces through `openclaw configure` and `openclaw config` (`docs/cli/configure.md:8`, `docs/cli/config.md:8`), while skill discovery/install is already routed through ClawHub (`docs/tools/clawhub.md:10`, `src/cli/skills-cli.ts:62`). A configuration-assistant workflow is a good idea, but it should ship as a ClawHub skill/plugin rather than as a new bundled core skill.
