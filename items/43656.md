## Summary

Keep #43656 open. Current main still does not implement cross-gateway `sessions_send` or `sessions_spawn`: the session tool schemas expose no `gateway`, `gatewayUrl`, or `gatewayToken` params, config has no `gateway.peers` contract, and the existing gateway URL override path remains intentionally allowlisted for loopback or configured `gateway.remote.url`. The PR is unmerged, authored by a non-maintainer, has no protected labels, and still represents plausible core work tied to open feature request #43605 rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open and treat #43605 as the canonical product request. The best path is a core gateway/session implementation, not ClawHub cleanup: add a reviewed `gateway.peers` config contract, route `sessions_send` and any supported `sessions_spawn` subset through remote gateway calls with explicit auth semantics, preserve local validation parity, document the user-facing config, and add focused tests for peer resolution plus end-to-end remote routing failure/success cases.
