## Summary

Keep open. Current main still has the Windows schtasks restart stdout writes, and the stale-PID health retry can call the same restart function a second time with the same stdout, so the reported duplicate-message path is still plausible and not implemented on main. The PR is useful, but it needs a focused cleanup because the provided PR context includes unrelated browser SSRF changes and no output de-dup regression test.

## Best Possible Solution

Keep this PR open, but ask for a focused gateway-only update: remove or centralize the schtasks restart output so the stale-PID retry reports only one restart status, preserve a human success message for completed Windows restarts, add a targeted regression test around the retry/output behavior, and drop or split the unrelated browser SSRF commits before maintainer review.
