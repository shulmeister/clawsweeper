## Summary

Keep open. Current main still does not implement per-agent TTS voice configuration: `agents.list[]` is strict and has no `tts` field, `AgentConfig` has no per-agent TTS slot, docs only describe global `messages.tts`, and the speech runtime resolves TTS from `cfg.messages?.tts` plus model directives. The issue also has recent reproduction discussion and an open scoped PR #62573 in the provided GitHub context, so this remains an active product/code item rather than cleanup-closeable.

## Best Possible Solution

Keep #11483 open until a focused implementation lands. The likely product/code path is to add a documented per-agent TTS override surface in the agent config schema/types, merge or resolve it over global `messages.tts`, thread the selected agent context through automatic replies and the TTS tool path, and cover config validation plus runtime selection tests. PR #62573 appears to be the current scoped implementation to review or replace.
