## Summary

Current main implements per-agent TTS overrides: `agents.list[].tts` is typed, accepted by the strict agent schema, resolved over global `messages.tts`, threaded through automatic reply TTS, covered by tests, and documented. The implementation landed in `0ca952cdd58ac0491c282e466b49d90f10d2a96a`; it is on main but not in the latest tagged release `v2026.4.24`.

## Best Possible Solution

Close #11483 as implemented on main. Keep the `agents.list[].tts.providers.<provider>` override surface and documented deep-merge behavior; any remaining provider-specific voice/rate/pitch gaps should be filed as narrower follow-ups after this core config path ships in a release.
