## Summary

Keep open. Current `main` does not implement per-agent TTS configuration: `agents.list[]` has no `tts` field in the validated schema or config types, and the TTS runtime still resolves only global `messages.tts` settings.

## Evidence

- **Agent schema rejects extra keys like `tts`:** `AgentEntrySchema` is a strict object and its allowed keys stop at `runtime`; there is no `tts` field under `agents.list[]`. That means `agents.list[].tts` is not valid on current `main`.
  - file: [src/config/zod-schema.agent-runtime.ts:812](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/config/zod-schema.agent-runtime.ts#L812)
  - command: `nl -ba src/config/zod-schema.agent-runtime.ts | sed -n '780,920p'`
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Public agent config type also lacks `tts`:** `AgentConfig` enumerates per-agent fields such as `identity`, `sandbox`, `params`, `tools`, and `runtime`, but no `tts` property exists.
  - file: [src/config/types.agents.ts:70](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/config/types.agents.ts#L70)
