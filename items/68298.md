## Summary

Keep open. Current main still has the Discord fail-open behavior this PR targets: bot identity fetch failures are logged and converted to an undefined bot id, and guild preflight still gates the no-mention drop on `botId && mentionDecision.shouldSkip`. The PR head contains the relevant production fix and focused regression tests, so it remains useful rather than obsolete.

## Best Possible Solution

Keep the PR open and have maintainers either land #68298 or apply the same fail-closed change on main: make Discord startup throw when bot identity cannot be fetched or lacks an id, remove the redundant `botId &&` preflight guard, and keep focused regression tests for both seams. Before merge, consider changing the new `runtime.error` exact-string assertions to `expect.stringContaining(...)` per the bot review.
