## Summary

Keep #39223 open. Current main still routes `openclaw configure` and `openclaw configure --section model` through the Gateway-location Clack select before model/auth setup, and the configure command still has no non-interactive, local, auth-only, or skip-Gateway flag. The direct `openclaw models auth ...` commands and Anthropic setup-token/Claude CLI paths are useful workarounds, but they do not fix the reported configure wizard path.

## Best Possible Solution

Keep the issue open until the configure path itself is robust. A good fix would detect non-interactive or unsuitable TTY conditions before the Gateway-location select and print direct-auth guidance, or add a model/auth-only configure path that skips Gateway-location selection when the user only needs auth setup. Continue documenting `openclaw models auth login/setup-token/paste-token` as workarounds, not as the closing fix for this issue.
