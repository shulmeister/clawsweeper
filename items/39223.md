## Summary

Keep #39223 open. Current main still routes `openclaw configure` and `openclaw configure --section model` through the Gateway-location Clack select before model/auth prompts, and the configure command still has no flag or non-interactive path that bypasses that prompt. The direct `openclaw models auth ...` commands are valid workarounds already noted in the issue discussion, but they do not fix the reported configure wizard hang.

## Best Possible Solution

Keep #39223 open until the configure path itself is robust: add TTY/non-interactive detection with actionable direct-auth guidance, add a configure flag or section mode that can skip Gateway-location selection for model/auth work, or fix the Clack input hang at `Where will the Gateway run?`. Keep `openclaw models auth login/setup-token/paste-token` documented as workarounds, not as the closing fix.
