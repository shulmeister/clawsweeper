## Summary

Keep #39223 open. Current main still makes `openclaw configure` and `openclaw configure --section model` pass through the Gateway-location Clack select before model/auth setup, and the configure command still has no option to bypass that prompt. The direct `openclaw models auth ...` commands are valid documented workarounds, but they do not fix the reported configure wizard path.

## Best Possible Solution

Keep this issue open until the configure path itself is robust. A good fix would either add TTY/non-interactive detection with actionable direct-auth guidance before Clack can hang, add a configure option or section mode that skips Gateway-location selection for model/auth work, or fix the underlying Clack input behavior at `Where will the Gateway run?`. Keep `openclaw models auth login/setup-token/paste-token` documented as workarounds, not as the closing fix for this issue.
