## Summary

Keep open. Current main still routes `openclaw configure` and `openclaw configure --section model` through the same Gateway-location Clack select before any model/auth section work, with no configure flag to bypass that prompt. Direct `models auth` commands and docs provide useful Anthropic workarounds, matching the issue discussion, but they do not fix the reported configure wizard input hang or add a configure-level auth-only bypass.

## Best Possible Solution

Keep #39223 open until the configure path itself is made robust: either detect unsupported/non-interactive TTY conditions and fail with an actionable auth bypass, add a section/auth-only mode that skips the Gateway-location prompt for `--section model`, or otherwise fix the Clack input hang at `Where will the Gateway run?`. Keep the documented `models auth login/setup-token/paste-token` routes as workarounds, but do not treat them as closing the configure wizard bug.
