## Summary

Keep open. Current main does not implement either acceptable outcome from #71710: the `openclaw agent` CLI entry point does not install a SIGTERM/SIGINT cancellation path for gateway or embedded runs, and the agent/cron docs do not warn external cron users to use `timeout -k` or another SIGKILL escalation. The related open supervisor issue #66399 is adjacent but narrower: it covers exec-tool child supervision, not the parent `openclaw agent` process under cron.

## Best Possible Solution

Implement a top-level `openclaw agent` signal lifecycle: install SIGTERM/SIGINT handlers that abort the in-flight gateway request or embedded run via the existing AbortSignal plumbing, stop progress/UI and GatewayClient resources, flush final logs/transcript state where feasible, and force exit after a bounded grace window. Until that lands, add public docs for cron/supervisor wrappers to use `timeout -k <delay>` or an equivalent SIGKILL escalation. Keep #66399 separate unless maintainers intentionally broaden it to cover the parent agent CLI process as well.
