## Summary

Keep open. Current main still has the unsafe stored model override path, so the PR is not obsolete. However, the PR is not clean to close as implemented or merge as-is because it bundles unrelated heartbeat behavior changes that alter current empty-object heartbeat semantics and the model-override fix needs regression coverage/broader call-site cleanup.

## Best Possible Solution

Keep the PR open but request a focused revision: drop the heartbeat commit from this PR, change stored override normalization to accept unknown values through an existing helper such as normalizeOptionalString or a shared typed helper, update duplicate consumers such as sessions display/status resolution as needed, and add a regression test that feeds non-string providerOverride/modelOverride from a loaded session-store shape without crashing.
