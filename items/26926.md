## Summary

Keep #26926 open. Current main still does not support inbound Synology Chat image/CSV/PDF attachments: the bundled Synology Chat webhook payload, parser, delivery object, inbound context, docs, and regression tests are text-only for inbound messages. The only media path present is outbound URL-based file delivery, which does not solve user-uploaded attachments from Synology Chat.

## Best Possible Solution

Keep #26926 open and implement inbound attachment handling in the bundled Synology Chat plugin. Confirm Synology Chat's uploaded-file webhook/API shape, add typed parsing for images, CSVs, and PDFs, download/store accepted files through existing media helpers with security limits, pass media fields into the inbound context, update https://docs.openclaw.ai/channels/synology-chat, and add focused webhook plus inbound-context regression tests.
