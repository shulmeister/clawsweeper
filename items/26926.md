## Summary

Keep #26926 open. Current main still handles Synology Chat inbound webhooks as text messages only; outbound URL-based file delivery exists, but the requested inbound images, CSV files, and PDFs are not parsed, downloaded, staged, or exposed to the agent context.

## Best Possible Solution

Keep the issue open and implement inbound attachment handling in the bundled Synology Chat plugin. Confirm Synology Chat's attachment webhook/API shape, extend the typed payload/parser, download or stage accepted images/CSVs/PDFs through existing media-safe helpers, expose MediaPath(s), MediaUrl(s), MediaType(s), and filename metadata in the inbound context, update https://docs.openclaw.ai/channels/synology-chat, and add focused webhook plus inbound-context regression tests.
