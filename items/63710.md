## Summary

Keep open. Current main still intentionally ignores explicit HTTP(S) image URLs in native prompt image detection, so the mixed inline-image plus remote-image behavior this PR targets is not implemented. The PR is also paired with an open same-author bug report and has unresolved review feedback about over-fetching arbitrary URLs and SSRF-policy threading, so the best maintainer cleanup is rework/review, not closure.

## Best Possible Solution

Keep the PR and its linked bug report open for maintainer review or contributor rework. A safe fix should add remote image URL detection only for clearly intended image refs, avoid fetching arbitrary hyperlinks, thread the existing `ssrfPolicy` through `detectAndLoadPromptImages`/`loadImageFromRef`, handle URL dedupe and trailing punctuation correctly, and add regression coverage for mixed inline plus remote image prompts, non-image URL no-fetch behavior, and SSRF-policy propagation.
