## Summary

Keep open. Current main still does not implement the requested circuit breaker in `createTypingKeepaliveLoop`, and Telegram typing still uses the generic retry path unless callers pass an override. The issue discussion correctly narrowed the bug to expensive Telegram typing retries plus callers that bypass higher-level typing safeguards; both meaningful pieces still remain in current main. Related PRs #45836 and #47018 were closed unmerged, so they do not supersede the issue.
