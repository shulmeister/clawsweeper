## Summary

Current main already implements Telegram-bound inbound routing and delivery context for group/topic conversations, separate from webchat, so the requested behavior is already present.

## Close Comment

Closing as implemented on current OpenClaw. This was a Codex review of `main` and the shipped `v2026.4.22` tree. The current Telegram path already routes inbound group/topic traffic into Telegram-scoped session keys instead of webchat (`extensions/telegram/src/bot-message-context.ts:376`), persists inbound metadata with `Provider` and `OriginatingChannel` both set to `telegram` (`extensions/telegram/src/bot-message-context.session.ts:322`), and the Telegram channel contract builds message-tool context from the Telegram target/thread rather than a webchat binding (`extensions/telegram/src/threading-tool-context.ts:21`, `extensions/telegram/src/channel.ts:990`). The docs also now describe channel-isolated group routing and Telegram topic session keys explicitly (`docs/channels/channel-routing.md:10`, `docs/channels/channel-routing.md:34`). Regression coverage exists for Telegram group/topic session keys in `extensions/telegram/src/bot.create-telegram-bot.test.ts:2369` and `extensions/telegram/src/session-route.test.ts:49`.
