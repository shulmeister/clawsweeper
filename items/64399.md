## Summary

Keep open. Current main has a partial ANSI-strip implementation for the Overview `Gateway Logs` tail, but the dedicated Control UI Logs path still parses and renders log messages without stripping or safely rendering ANSI sequences. The related fix PR #64431 was closed unmerged, so it does not supersede the remaining work.

## Best Possible Solution

Keep the issue open and land a focused Control UI fix that applies one shared ANSI strategy across gateway log surfaces: strip ANSI or safely render ANSI styling for `parseLogLine()`/Logs tab display, subsystem/context parsing, raw fallback lines, exports if desired, and the Overview tail. Add regression coverage for ANSI-bearing JSON log messages and raw lines; PR #64431 is useful prior art but was not merged.
