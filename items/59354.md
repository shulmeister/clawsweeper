## Summary

Current `main` already closes the file-only iMessage echo gap by synthesizing media placeholders for attachment-only sends and inbound rows, caching those placeholders with message IDs, and testing both normal echo-drop and `is_from_me` self-chat paths.

## Evidence

- **Outbound file-only sends now produce cacheable placeholder text:** `sendMessageIMessage` converts media-only sends into synthetic body text like `<media:image>` / `<media:${kind}>`, so attachment-only outbound messages no longer look text-empty to echo detection.
  - file: [extensions/imessage/src/send.ts:86](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/extensions/imessage/src/send.ts#L86)
  - command: `nl -ba extensions/imessage/src/send.ts | sed -n '86,95p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Media replies are remembered in the echo cache with placeholder + message id:** After sending media, the iMessage deliver path records `sent.sentText` and `sent.messageId`, which preserves the synthetic placeholder for later echo matching.
  - file: [extensions/imessage/src/monitor/deliver.ts:63](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/extensions/imessage/src/monitor/deliver.ts#L63)
