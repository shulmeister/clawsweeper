## Summary

Keep open. Current `main` still lacks Voice-tab/session parity on Android, and Talk voice configuration is still modeled as one global Talk config rather than a per-agent setting. iOS has a global selected-agent path that updates Talk mode, but that is only a partial match for the request and does not cover per-agent Talk voices or the full Android/macOS scope in the issue.

## Evidence

## Risks / Open Questions

- The issue bundles two distinct scopes: Voice-tab/session UX and per-agent Talk voice config. They may need to be tracked or implemented separately.
- Platform status differs: iOS has a global selected-agent path, while Android evidence still shows no Voice-tab/session control. Any cleanup decision should avoid treating those as equivalent.
