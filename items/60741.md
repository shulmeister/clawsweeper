## Summary

Keep open. Current main has substantial session infrastructure, including `sessions.create`, `sessions.list`, `sessions.patch`, and `sessions.delete`, plus a Sessions table that can label/delete/switch rows. But the reported WebChat/Control UI workflow is not implemented end to end: the Chat plus button still sends `/new`, which resets the current session key, and local search found no Control UI call to `sessions.create` to create a separate preserved dashboard/WebChat conversation from the chat surface.
