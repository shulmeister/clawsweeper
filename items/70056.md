## Summary

Keep this PR open. Current main still has the stop-timeout stale-store behavior that the PR targets, and the linked same-author bug remains open. The PR is relevant but should be revised before merge because its current timeout cleanup can let an old timed-out task later clobber state for a replacement lifecycle.

## Best Possible Solution

Keep the PR open and ask for a revision in `src/gateway/server-channels.ts` that makes stop-timeout state honest while preventing stale timed-out lifecycles from mutating runtime or restart state for a replacement task. Add a regression test where a timed-out task eventually settles after a successful restart, then land this PR or an equivalent maintainer patch to resolve #70024.
