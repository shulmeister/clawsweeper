## Summary

Keep open. Current main still intentionally injects sender/routing metadata into the user-role prompt sent to the agent; the implemented sanitizers strip those blocks from UI/history display, not from the live AI context described by the issue.

## Evidence

- **Agent prompt still includes inbound metadata:** `getReplyRun` builds `inboundUserContext` with `buildInboundUserContextPrefix(...)` and joins it into `baseBodyForPrompt`, which is then used for the agent run.
