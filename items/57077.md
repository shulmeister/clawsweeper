## Summary

Keep open. Current `main` contains the related session-preservation fix from merged PR #59611, but it still uses the same session-picker selection pattern this PR was trying to harden, and the specific fallback-rerender regression coverage from #57077 is not present on `main`. That makes an auto-close as duplicate or implemented unsafe.
