## Summary

Keep open. Current main has some adjacent status JSON improvements, but the central issue is not fully resolved: the `tasks` CLI path still loads the shared status/health/sessions/tasks registration module, that module still statically imports unrelated command implementations, and `tasks.ts` still statically imports audit/maintenance runtimes even for list-style use. Also, the current lean `status --json` path deliberately disables `channelSummary`, which conflicts with the report's expected behavior. The related #71739 context is narrower `agents list --json` work and is not a canonical replacement for this tasks/status issue.

## Best Possible Solution

Keep #66238 open. The best fix is to split the shared status/health/sessions/tasks registrar or add route-first task JSON handlers, then lazily import task audit/maintenance and unrelated status/health/session code only for the subcommands that need them. Maintainers should also decide whether `status --json` must preserve `channelSummary` through a cold metadata fallback instead of disabling it on the fast path.
