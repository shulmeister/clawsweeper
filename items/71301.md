## Summary

Keep open. Current main partially addresses the onboarding problem by shipping `docs/`, resolving a local docs path, adding documentation guidance to the agent prompt, and exposing `config.schema.lookup` for config fields. The central ask remains unimplemented: there is no versioned local docs search/index or first-class native agent docs retrieval API such as `docs.search` / `docs.get`; the existing `openclaw docs` command searches the live docs index.

## Best Possible Solution

Keep the issue open and turn it into a concrete core design/implementation task: generate a versioned docs manifest/search index during docs or release packaging, expose local-first retrieval through a stable agent/Gateway surface such as `docs.search` and `docs.get`, reuse `config.schema.lookup` for field-level config docs, and document installed-version versus live-docs fallback behavior.
