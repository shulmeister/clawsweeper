## Summary

Keep this issue open. Current main has partial version-matched docs support through npm packaging and prompt guidance, plus `config.schema.lookup` for config-field docs, but the central request remains unimplemented: there is no first-class local docs search/get/API retrieval surface for agents, and the CLI docs command still searches the live docs service.

## Best Possible Solution

Keep this issue open and narrow it to the missing core capability: ship a release/package docs manifest plus local search/get index over bundled docs, expose a local-first docs retrieval surface to agents, keep live Mintlify search as an explicit fallback, and add tests for package-installed docs discovery plus agent/tool behavior.
