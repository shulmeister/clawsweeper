## Summary

Current `main` already implements this behavior with a stronger approach than the PR: Discord startup swaps in a proxy-aware RequestClient compat layer, so Carbon REST calls use the configured proxy without monkey-patching `globalThis.fetch`.

## Close Comment

Closing as `implemented_on_main` after a Codex review. Current `main` already routes these Discord startup Carbon REST calls through the configured proxy by replacing `client.rest` with `createDiscordRequestClient(..., { fetch: params.proxyFetch })` in `extensions/discord/src/monitor/provider.startup.ts:180`. That compat client exists specifically because Carbon removed the RequestClient fetch seam (`extensions/discord/src/proxy-request-client.ts:132`) and its request path uses `this.customFetch ?? globalThis.fetch` (`extensions/discord/src/proxy-request-client.ts:318`), so deploy and bot-identity requests no longer depend on Carbon’s default global fetch behavior. The affected startup calls still run after client construction in `extensions/discord/src/monitor/provider.ts:978` and `extensions/discord/src/monitor/provider.ts:998`, so the gap described here is already covered on `main`. Best proof commit: `665cc3d5371c275b32c2c400b54238e5d96f6761`. No tagged release contains that commit yet, so I’m leaving `fixedRelease` as null.
