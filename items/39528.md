## Summary

Keep #39528 open. Current main has meaningful hardening since the original 2026.3.2 report, including owner-only gateway tooling, admin-scoped RPCs, config path allowlists, dangerous-flag blocking, and control-plane rate limits. It still does not enforce a fresh, hard user-consent gate before the model-callable owner gateway tool can restart the gateway, write allowed config changes, or run update.run.

## Best Possible Solution

Keep this open as a product/security-behavior bug. The best fix is an enforceable, test-backed recent explicit-consent mechanism for model-initiated gateway restart, config.apply, config.patch, and update.run, while preserving the current owner-only/admin-scope, allowlist, dangerous-flag, protected-path, and rate-limit guardrails. Add regression coverage for long sessions and compaction where SOUL.md, USER.md, MEMORY.md, and user red-line instructions remain available or are re-read before sensitive control-plane actions.
