## Summary

Keep open. Current main at 82262387651ab270b565f71533ce4937c9c00dc6 still has memory-lancedb on vector-only LanceDB retrieval: no FTS/BM25 index setup, no hybrid config, no original query text passed into database search, and tests still assert vectorSearch behavior. The existing BM25+vector hybrid work is for memory-core, not LanceDB parity. PR #7636 attempted this feature but was closed unmerged, and the latest issue discussion already kept #7629 open.

## Best Possible Solution

Keep #7629 open as a focused bundled memory-lancedb parity enhancement. The best path is to implement this in extensions/memory-lancedb: add typed hybrid config plus manifest/docs entries, create or maintain a LanceDB FTS index on text, pass original query text through recall/forget/CLI/hooks and duplicate checks, use verified LanceDB hybrid search or a tested fallback/reranker when enabled, preserve vector-only fallback, and add tests for config validation, FTS setup, exact-keyword recall, fallback behavior, and query-text propagation.
