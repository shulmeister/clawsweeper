## Summary

Keep open. Current main still leaves `memory-lancedb` on pure vector retrieval: it creates no LanceDB FTS index, exposes no hybrid/BM25 config surface, and drops the original query text before search. Built-in `memory_search` has BM25 hybrid search, but that is a separate memory-core path and supports the parity concern rather than resolving this plugin request. The related implementation PR #7636 was closed unmerged; #19028 was a closed unmerged MongoDB backend proposal, not a LanceDB fix.

## Best Possible Solution

Keep the issue open. If maintainers still want parity for the bundled `memory-lancedb` plugin, implement it inside that plugin: verify the pinned LanceDB JS API, add a `hybrid` config surface to `config.ts` and `openclaw.plugin.json`, create or maintain an FTS index on the `text` column during initialization, thread the original query text through recall/forget/CLI/hooks, use LanceDB hybrid search or a verified fallback/reranker, and add focused tests for config validation, FTS setup, vector-only fallback, and exact-keyword recall.
