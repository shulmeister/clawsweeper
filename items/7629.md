## Summary

Keep open. Current main still leaves the bundled `memory-lancedb` plugin on pure vector retrieval: no LanceDB FTS index is created, no hybrid/BM25 config is accepted, and recall/search call sites pass only embeddings into `MemoryDB.search`. Built-in `memory-core` does have BM25 hybrid search, but that is a separate memory plugin and supports the parity concern rather than resolving this LanceDB request. The issue discussion already contains a Codex review saying to keep this open, and current main at `c3833f7729163a8f6cdc47d3b0be86fd5fcab160` still matches that finding.

## Best Possible Solution

Keep the issue open. If maintainers still want parity for the bundled `memory-lancedb` plugin, implement it inside that plugin: verify the pinned `@lancedb/lancedb` JS API, add a `hybrid` config surface to `config.ts` and `openclaw.plugin.json`, create or maintain an FTS index on the `text` column during initialization, pass original query text through recall/forget/CLI/hooks, use LanceDB hybrid search or a verified fallback/reranker, and add focused tests for config validation, FTS setup, vector-only fallback, exact-keyword recall, and query text propagation.
