## Summary

Keep open. Current main still implements memory-lancedb recall as pure vector search, has no hybrid/BM25 config surface, and the related implementation PR #7636 was closed unmerged. The supporting comment from olafgeibig is still relevant: LanceDB remains a useful target for flexible memory backends, but the requested plugin parity work is not present on main.

## Evidence
