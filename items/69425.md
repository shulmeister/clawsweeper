## Summary

Keep open. Current main still does not honor OPENCLAW_WRAPPER when generating daemon service ProgramArguments, and PR #69425 is a focused core daemon change for the linked #69400 problem rather than an obsolete, duplicate, ClawHub, or non-repo item. The PR still needs maintainer review/author follow-up for the posted review concern about using the installer/merged service environment, plus likely executable-bit validation, before it is merge-ready.

## Best Possible Solution

Keep PR #69425 open. The best path is to finish the core daemon implementation in src/daemon/program-args.ts with an env-aware wrapper resolver used by both gateway and node service install plans, preserve or explicitly persist the wrapper through reinstall/update, add focused tests for valid/missing/non-file/non-executable wrappers and the merged existing-service-env case, and document/changelog the behavior once the product shape is accepted.
