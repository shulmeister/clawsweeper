## Summary

Keep open. Current main has user-visible cleanup for XML-style tool-call blocks, MiniMax XML, model tokens, memory tags, and downgraded `[Tool Call: ...]` scaffolding, but I did not find source/docs/tests/changelog evidence that it handles the exact reported bare `exec{"command": ...}` reply leak on shell/Telegram surfaces.

## Best Possible Solution

Keep tracking this as a behavior bug. A good fix would add focused detection or prevention for bare JSON-ish pseudo tool calls such as `exec{"command": ...}` in the assistant-visible delivery/history and streaming reply paths, with regression coverage for shell/Telegram output. If the root cause is provider transport misconfiguration, document and surface that path without letting raw pseudo-call text reach users.
