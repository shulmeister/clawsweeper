## Summary

Keep open. Current main still does not implement a sanitizer or provider repair for the reported bare `exec{"command": ...}` visible-text leak, and the relevant delivery paths still rely on sanitizers that only cover XML-style tool-call blocks, MiniMax XML, model tokens, memory tags, and downgraded `[Tool Call: ...]` scaffolding.

## Best Possible Solution

Keep this open as a behavior bug. Preserve existing XML/downgraded-tool sanitization, then add provider-specific root-cause handling or a conservative visible-text sanitizer for actual bare tool-name-plus-JSON pseudo calls such as `exec{...}`. Cover `sanitizeAssistantVisibleText`, `pi-embedded-utils.extractAssistantVisibleText`, streaming block chunks, text_end/message_end delivery, and shell/Telegram-facing output.
