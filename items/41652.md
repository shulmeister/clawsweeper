## Summary

The reported `device-id-mismatch` does not indicate a current OpenClaw bug. The issue comments show the reporter corrected the device ID/public key relationship and the failure changed to `DEVICE_AUTH_SIGNATURE_INVALID`, which means the original `device-id-mismatch` report became obsolete. Current main still intentionally returns `device-id-mismatch` only when `device.id` does not equal the SHA-256 fingerprint of the provided public key, and the docs now describe that contract and the separate nonce/signature requirements.
