## Summary

Keep open. This is a recent Matrix regression report with a concrete version, install method, symptom, and follow-up that Synapse was unchanged and rollback restored behavior. Current main has Matrix sync/readiness and AbortError handling, but the only clearly related changelogged sync-lifecycle fix shipped in 2026.4.9, the same version the reporter says introduced the regression, so there is not strong enough evidence to close as already implemented.

## Best Possible Solution

Keep this issue open and reproduce against the current release/main with the reporter's Docker plus local Synapse shape. Capture gateway startup logs, Matrix sync state/status, `openclaw matrix verify status --verbose`, and whether room events reach `registerMatrixMonitorEvents`. If it still reproduces, the likely fix belongs in the Matrix plugin's shared-client startup/sync lifecycle or Matrix event monitor path, with a regression test around abort/readiness handling and inbound room-event delivery.
