## Summary

Keep open. Current `main` still renders Control UI chat/sidebar content through a plain `markdown-it` + DOMPurify pipeline with no MathJax/KaTeX integration, no math-related config surface, and no shipped dependency/tests proving LaTeX rendering support.

## Evidence

- **Control UI markdown renderer has no math plugin path:** `ui/src/ui/markdown.ts:153` creates a bare `new MarkdownIt(...)`; `ui/src/ui/markdown.ts:355-517` adds task-list, HTML escaping, image/code-block, and sanitization behavior only. There is no `katex`, `MathJax`, or LaTeX renderer/plugin hook in this pipeline.
  - file: [ui/src/ui/markdown.ts:153](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/ui/src/ui/markdown.ts#L153)

That means formulas currently go through the non-math markdown pipeline.
