## Summary

Keep open. Current `main` has a new context-overflow fallback reply path for empty embedded runs, which helps part of the Telegram symptom, but the cron `NO_REPLY` path still depends on model-produced final text and the gateway PID/port mismatch path still reports/restarts rather than self-healing. I did not find a post-issue mainline fix for the cron or gateway parts of this bundled reliability report.
