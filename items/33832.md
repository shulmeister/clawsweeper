## Summary

Keep open. Current main has sanitized model-call diagnostics and OTEL duration telemetry, but it still does not implement Gateway log lines for model request/response body size or time-to-first-byte. The linked implementation PR #35513 was closed unmerged, and the requested wrapper/logging symbols are absent on current main.

## Best Possible Solution

Keep #33832 open until OpenClaw core implements privacy-safe model-call request/response byte counts and TTFB/latency logging, or until maintainers explicitly decide that diagnostics/OTEL is the product surface and document that replacement. The likely implementation belongs near the existing stream wrapper and diagnostic event contract, with tests proving no raw model content is logged.

## Risks / Open Questions

- Adding request/response size logging needs a privacy-preserving byte-count implementation that does not log raw prompts, responses, headers, or secrets.
- Time-to-first-byte likely needs instrumentation at the stream boundary or provider transport boundary; the current diagnostic wrapper only knows start and terminal duration.
