## Summary

Keep #33832 open. Current main still has sanitized model-call lifecycle diagnostics and duration metrics, but it does not implement the requested Gateway logging for request body size, response body size, or time-to-first-byte. The linked implementation PR #35513 was closed unmerged, and the only commit since the prior ClawSweeper review touched WhatsApp files, not model-call diagnostics.

## Best Possible Solution

Keep #33832 open until OpenClaw core adds privacy-safe model API request/response byte counts plus first-byte and total latency logging or telemetry, likely near the existing model-call diagnostic wrapper or provider transport boundary. Add focused tests proving size/timing fields are emitted without raw model content or secrets. If maintainers decide the existing duration-only OTEL/Prometheus telemetry is the intended replacement, document that product decision before closing.

## Risks / Open Questions

- Any eventual implementation should preserve the current privacy boundary: byte counts and timing are useful, but raw prompts, responses, request bodies, headers, provider request IDs, tokens, and secrets should not be logged.
- Accurate time-to-first-byte likely belongs at the provider transport or stream-iteration boundary; the current diagnostic wrapper only records call start and terminal completion/error.
