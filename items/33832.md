## Summary

Keep #33832 open. Current main emits sanitized model-call diagnostic events and duration metrics, but it still does not implement Gateway log lines for request body size, response body size, or time-to-first-byte. The linked implementation PR #35513 attempted the requested wrapper and was closed unmerged, so the requested feature remains unshipped.

## Best Possible Solution

Keep #33832 open until OpenClaw core adds privacy-safe model API request/response byte counts plus first-byte and total latency logging, likely near the existing stream wrapper or diagnostic event contract, with tests proving no raw model content or secrets are logged. If maintainers decide OTEL/Prometheus duration telemetry is the supported replacement, document that product decision before closing.

- A future implementation must preserve the existing privacy boundary: no raw prompts, responses, headers, request bodies, provider request IDs, tokens, or secrets in Gateway logs by default.
- True time-to-first-byte likely needs instrumentation at the provider transport or stream iteration boundary; the current diagnostic wrapper only observes call start and terminal completion/error.
