## Summary

Keep #33832 open. Current main emits sanitized model-call diagnostic events and OTEL duration telemetry, but it still does not implement the requested Gateway log lines for request body size, response body size, or time-to-first-byte. The linked implementation PR #35513 was closed unmerged, and the proposed logging wrapper/file is absent on current main.

## Best Possible Solution

Keep #33832 open until OpenClaw core implements privacy-safe model API request/response byte-count and TTFB/latency logging, likely near the existing stream wrapper and diagnostic event contract, with tests proving no raw model content is logged. Alternatively, maintainers could explicitly decide that sanitized diagnostics/OTEL duration telemetry is the supported replacement and document that product decision.
