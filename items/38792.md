## Summary

Keep open. Current `main` has partial mitigation for empty `stopReason:"error"` turns and an opt-in tool-loop detector, but it still does not clearly implement the issue's requested default safety valve for long tool-only chains or immediate user-visible notification when retrying after an API error.

## Evidence

- **Silent error retries still happen before any user payload:** `runEmbeddedPiAgent` silently resubmits zero-output error turns by incrementing `emptyErrorRetries`, logging a warning, and `continue`-ing the loop. No payload is emitted at that point; the user-visible error only appears later if retries exhaust.
