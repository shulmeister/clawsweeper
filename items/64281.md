## Summary

Keep this PR open. Current main still does not provide `sessions_delegate` or `sessions_delegate_batch`; `sessions_spawn` remains a non-blocking background orchestration tool, while `sessions_send` only waits on an existing visible session. The PR is a plausible implementation candidate, but it needs maintainer review and fixes for the unresolved review risks before merge.

## Best Possible Solution

Keep the PR open as the implementation candidate for synchronous subagent delegation. The next maintainer path is to decide whether this core tool surface is wanted, then require fixes for the attachment redaction and output-capture/cleanup race review comments, add or update focused tests, and update the session-tool docs and security/tool-policy references before merge.
