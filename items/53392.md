## Summary

This request is for QClaw-specific skill/env behavior, not OpenClaw core. Current main already routes skill selection through skill descriptions and supports OS/config/bin gating for custom skills, while project policy says environment-specific skills should live in ClawHub/plugin space rather than core special cases.

## Close Comment

Closing as `clawhub` after a Codex review. The request is specifically about QClaw-local behavior (`config/skills/qclaw-openclaw/SKILL.md`, Windows/QClaw path detection, QClaw wrapper commands), but current `main` has no `QClaw` or `qclaw-openclaw` surface in this repo. OpenClaw already documents the intended extension point here: skill selection is driven by `SKILL.md` descriptions (`src/agents/system-prompt.ts:162`, `src/agents/skills/skill-contract.ts:48`), and custom skills can be gated by host OS/config/bin metadata (`docs/tools/creating-skills.md:84`, `docs/tools/skills.md:189`). Project scope also says new/env-specific skills should live on ClawHub rather than in core (`VISION.md:68`). So this belongs in the QClaw-distributed skill/plugin layer, not `openclaw/openclaw` core.
