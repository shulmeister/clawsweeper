## Summary

Keep open. Current main still treats `agents.defaults.subagents.runTimeoutSeconds` as a fallback only: explicit `sessions_spawn.runTimeoutSeconds` from the model wins, there is no `minRunTimeoutSeconds` or lock config, and tests/docs still describe and preserve the reported precedence. Related PRs #37973 and #38048 attempted this fix but are closed unmerged; #60211 is a different timeout/fallback issue.

## Best Possible Solution

Keep #37902 open until OpenClaw implements a maintainer-approved subagent timeout constraint. A narrow fix should add an explicit config contract such as `agents.defaults.subagents.minRunTimeoutSeconds` or an authoritative lock option, update the TypeScript config type, strict zod schema, generated schema/docs, and clamp model-supplied `sessions_spawn.runTimeoutSeconds` before subagent run registration. Regression tests should cover omitted timeout fallback, explicit values above the floor, explicit values below the floor, zero/no-timeout compatibility, unset compatibility, and schema acceptance/rejection. Closed PRs #37973 and #38048 are useful prior art but do not close the issue on current main.
