## Summary

Keep open. Current main still implements `agents.defaults.subagents.runTimeoutSeconds` as a fallback only, with explicit `sessions_spawn.runTimeoutSeconds` winning even when it is lower. The requested `minRunTimeoutSeconds` or locked timeout behavior is not present in config types, zod schema, generated schema, docs, or tests. The related fix PRs #37973 and #38048 were closed unmerged, so they do not supersede the issue.

## Best Possible Solution

Keep this issue open until OpenClaw either implements an explicit subagent timeout floor/lock or a maintainer rejects that product direction. The likely implementation should add a documented config key such as `agents.defaults.subagents.minRunTimeoutSeconds`, update config types, zod and generated schema, clamp `resolveConfiguredSubagentRunTimeoutSeconds`, and replace the current regression assertion with tests for omitted timeout, explicit value above the floor, explicit value below the floor, and zero/no-timeout compatibility.
