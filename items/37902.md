## Summary

Current main still lacks a subagent timeout floor or authoritative lock. `sessions_spawn.runTimeoutSeconds` is normalized and then allowed to override `agents.defaults.subagents.runTimeoutSeconds`; the config type, strict zod schema, generated schema, docs, and tests still expose fallback-only behavior. The related fix PRs #37973 and #38048 are closed unmerged in the provided GitHub context, so they do not supersede the issue.

## Best Possible Solution

Keep #37902 open until OpenClaw implements a maintainer-approved subagent timeout constraint, likely `agents.defaults.subagents.minRunTimeoutSeconds` or an explicit authoritative lock option. The fix should update config types, strict zod schema, generated schema/docs, and resolver tests so model-supplied `sessions_spawn.runTimeoutSeconds` values below the configured floor are clamped before subagent run registration and timeout handling.
