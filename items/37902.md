## Summary

Current main still lacks the requested subagent timeout floor. `sessions_spawn.runTimeoutSeconds` is normalized and accepted when present, the configured `agents.defaults.subagents.runTimeoutSeconds` remains only a fallback, and the proposed `minRunTimeoutSeconds`/lock config key is absent from types, schema, docs, and tests. The related fix PRs #37973 and #38048 were closed unmerged in the provided GitHub context, so they do not supersede the open issue.

## Best Possible Solution

Keep #37902 open until OpenClaw either implements an optional `agents.defaults.subagents.minRunTimeoutSeconds` or maintainer-approved lock/authoritative timeout setting. The change should update config types, zod/generated schema, docs, and resolver tests so model-supplied `sessions_spawn.runTimeoutSeconds` values below the configured floor are clamped before the subagent gateway/run registry paths receive them.
