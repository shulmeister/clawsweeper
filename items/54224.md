## Summary

Keep this issue open. Current main already supports Slack channel history and thread reads with user-token read preference, but the requested Slack message search capability is still missing from action discovery, dispatch/runtime exports, implementation, CLI help/options, and docs beyond a scope mention.

## Best Possible Solution

Keep this issue open and narrow the remaining work to Slack message search. The maintainer path is to add a Slack-owned `search` message action backed by Slack `search.messages` and `search:read`, require/prefer a user token without enabling writes, advertise the action only when usable credentials exist, update the generic Slack dispatch/runtime exports and response shape, make `openclaw message search` work for Slack without Discord-only `--guild-id`, and add focused tests/docs. The existing `read`/`readMessages` history and thread path can stay as the implementation for the history/thread parts of the request.
