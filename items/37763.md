## Summary

Keep open. Current main has only a partial Slack identity implementation: normal reply delivery can receive the resolved agent identity, but the draft-stream preview path still cannot receive or forward `SlackSendIdentity`, so preview/finalized preview messages can still post as the default Slack bot. The current monitor path also forwards `agents.list[].identity.emoji` as-is, while Slack docs require `:emoji_name:` syntax; the related bare-emoji and draft-stream PRs in the provided context were closed unmerged.

## Best Possible Solution

Keep #37763 open until a focused Slack plugin fix lands. The likely fix belongs in `extensions/slack`: add an optional `SlackSendIdentity` to `createSlackDraftStream`, pass the resolved `slackIdentity` from `dispatchPreparedSlackMessage`, add draft-stream/dispatch regression tests, and normalize bare `agents.list[].identity.emoji` aliases to Slack `:emoji_name:` syntax consistently.
