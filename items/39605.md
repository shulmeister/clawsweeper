## Summary

Keep #39605 open. Current main only has fresh runtime config coverage for Telegram native commands; Discord and Slack slash/native command routing still resolve their target route from the provider startup config snapshot, and resolveAgentRoute still derives direct-message session keys from cfg.session.dmScope. The open related PR #39617 remains useful but is not merged, so the issue is not implemented or superseded on main.

## Best Possible Solution

Keep the issue open and land a focused fix that reloads config inside the Discord native command dispatch path and Slack slash command handler before route resolution, matching the Telegram fresh-runtime-config pattern. Add Discord and Slack regression tests that prove dmScope or route binding changes are honored between slash/native command invocations without recreating the provider.
