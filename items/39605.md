## Summary

Keep #39605 open. Current main only partially addresses the report: Telegram native commands now reload runtime config and have regression coverage, but Discord native commands and Slack slash commands still resolve routes from the monitor startup config object while `resolveAgentRoute` still derives direct-message session keys from the supplied `cfg.session.dmScope`.

## Best Possible Solution

Keep #39605 open and land #39617 or an equivalent maintainer-reviewed fix that supplies current runtime config before Discord native command route resolution and Slack slash command route resolution. Add Discord and Slack regression tests matching the existing Telegram coverage, proving non-main DM routing or route-binding changes are honored between native/slash invocations without recreating the provider.
