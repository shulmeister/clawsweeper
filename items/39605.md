## Summary

Keep #39605 open. Current main at e672b614 still routes Discord native commands and Slack slash commands through config objects captured at monitor startup, while route/session-key resolution still derives direct-message session keys from the supplied cfg.session.dmScope. Telegram has the fresh-runtime-config pattern and regression coverage, but the Discord and Slack surfaces named in the issue remain unresolved; the provided GitHub context also shows PR #39617 is still the open canonical fix attempt.

## Best Possible Solution

Keep #39605 open and land PR #39617 or an equivalent maintainer-reviewed fix that supplies freshly loaded runtime config before Discord native command route resolution and Slack slash command route resolution. Add Discord and Slack regression tests mirroring Telegram’s fresh-config coverage, proving native/slash command routing honors updated session.dmScope or route bindings without recreating the provider.
