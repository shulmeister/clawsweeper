## Summary

Keep #68470 open. Current main does not implement the MiniMax `prompt_cache_hit_tokens` normalization described in the issue, and the timeout-compaction path still bases the retry decision on `derivePromptTokens(lastRunPromptUsage)`. The related PRs #68750 and #68863 are useful, but they are open follow-up fixes rather than shipped main behavior.

## Best Possible Solution

Keep #68470 open as the canonical MiniMax prompt-token accounting bug. The likely fix belongs in `src/agents/usage.ts` by recognizing MiniMax `prompt_cache_hit_tokens` as cache-read usage and as an OpenAI-style prompt-total breakdown so cached tokens are subtracted from raw prompt/input totals before downstream `derivePromptTokens` math. Add focused regression coverage for the MiniMax usage payload, then separately evaluate whether #68863's `attemptUsage` fallback removal is also needed for tool-loop cases.
