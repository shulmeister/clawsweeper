## Summary

Keep open. Current main still hard-errors `sessions_spawn(runtime="subagent")` when `streamTo` or `resumeSessionId` is present, and the mainline tests still assert that rejection. PR #68397 targets the implicated code and tests directly, so it is not implemented, obsolete, or low-signal cleanup. There is an older similar open PR (#56342), but #68397 is the newer, still-useful candidate and includes the schema-description adjustment noted by the bot review.

## Best Possible Solution

Keep the PR open for maintainer review as a focused bug fix. The best path is to merge a current version that silently drops `streamTo` and `resumeSessionId` only for `runtime="subagent"`, preserves ACP forwarding, updates the existing unit tests, and aligns the ACP docs/schema descriptions with the new forgiving behavior. If maintainers prefer a single canonical PR, #68397 appears to be the better candidate than older duplicate #56342 because it is newer and addresses the bot-noted schema wording issue.
