## Summary

Keep this issue open. Current main filters `models.list` and Control UI model choices only when `agents.defaults.models` provides an allowlist; a config that only has `models.providers` still falls into the `allowAny` path and returns the full catalog. The related closed #66464 is relevant background, but it does not fully supersede this narrower request about `models.providers`-only configurations.

## Best Possible Solution

Keep this issue open and make the desired semantics explicit. The most direct implementation path is to update the gateway model-list surface, probably in `src/gateway/server-methods/models.ts` or `buildAllowedModelSetWithFallbacks`, so Control UI model pickers can optionally or by default derive a configured-only view from `models.providers` when no `agents.defaults.models` allowlist exists. Add regression coverage for a config matching the report: only `models.providers.minimax.models` and `models.providers.zhipu.models`, with no `agents.defaults.models`, and assert the UI/gateway model list excludes unrelated built-in providers.
