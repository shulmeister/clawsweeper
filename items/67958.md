## Summary

PR #67958 is obsolete for the linked Ollama bug. Current main and v2026.4.24 already fix the reported qwen2.5 failure path by defaulting embedded runs to thinking off and forwarding native Ollama thinking control as top-level `think:false` for off and `think:true` only for explicit non-off thinking. The PR’s remaining heuristic-based omit behavior is not the best follow-up path and the branch also carries unrelated env-var changes.

## Best Possible Solution

Close this PR as implemented/obsolete for #67949. Keep the shipped v2026.4.24 behavior, and handle any future Ollama capability-based thinking validation in a fresh narrow PR that updates the Ollama provider model metadata/profile path without unrelated env-var changes.
