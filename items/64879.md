## Summary

Keep open. Current main has the companion mock parity workflow and providerVariant metadata, but the qa-lab mock dispatcher still does not branch scenario plans by provider, and the live QA workflows do not run a real OpenAI-vs-Anthropic parity-report lane. The reported gap remains actionable.

## Best Possible Solution

Keep this issue open. Implement Option A by deriving a provider variant inside extensions/qa-lab/src/providers/mock-openai/server.ts buildResponsesPayload() and adding provider-specific planned-tool/prose branches for at least three agentic parity scenarios, with tests asserting divergent OpenAI vs Anthropic plans. If maintainers prefer Option B, add a separate secret-gated CI lane that runs the same parity pack against real openai/gpt-5.x and anthropic/claude-opus-4-6 summaries and feeds them into openclaw qa parity-report.
