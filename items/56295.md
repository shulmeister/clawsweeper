## Summary

The reported Mattermost DM restart failure is already addressed on current main: Mattermost direct messages are now kept non-threaded, so the reply path no longer sends a stale `root_id` for the issue’s reproduction. The shipped v2026.4.23 release already contains that behavior, plus tests and docs that explicitly cover linear/non-threaded DMs.

## Evidence

- **Direct-message replies no longer carry a thread root:** `resolveMattermostEffectiveReplyToId` returns `undefined` for `kind: "direct"`, which prevents DM replies from reusing a stale Mattermost thread root after restart.
  - file: [extensions/mattermost/src/mattermost/monitor.ts:352](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/mattermost/src/mattermost/monitor.ts#L352)
  - command: `nl -ba extensions/mattermost/src/mattermost/monitor.ts | sed -n '342,364p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Outbound send only includes `root_id` when a reply target exists:** The Mattermost send path passes `opts.replyToId` into `createMattermostPost`, and the client only serializes `payload.root_id` when `params.rootId` is present. With DM `replyToId` now undefined, replies fall back to a normal post instead of a threaded reply.
  - file: [extensions/mattermost/src/mattermost/send.ts:467](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/mattermost/src/mattermost/send.ts#L467)
  - command: `nl -ba extensions/mattermost/src/mattermost/send.ts | sed -n '430,520p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Client request builder is conditional on `rootId`:** `createMattermostPost` only adds `payload.root_id` behind an `if (params.rootId)` guard, so the direct-message path does not silently emit a stale root anymore.
  - file: [extensions/mattermost/src/mattermost/client.ts:499](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/mattermost/src/mattermost/client.ts#L499)
  - command: `nl -ba extensions/mattermost/src/mattermost/client.ts | sed -n '485,512p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Regression tests cover the exact DM behavior:** Mattermost tests assert both `keeps direct messages non-threaded` and `keeps direct-message sessions linear`, matching the stale-RootId-on-DM-restart scenario from this issue.
  - file: [extensions/mattermost/src/mattermost/monitor.test.ts:478](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/mattermost/src/mattermost/monitor.test.ts#L478)
  - command: `nl -ba extensions/mattermost/src/mattermost/monitor.test.ts | sed -n '460,560p'`

## Risks / Open Questions
