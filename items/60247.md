## Summary

Keep #60247 open. Current main no longer appears to surface JSON.parse exceptions from the Control UI WebSocket client, but I did not find a source-backed fix or regression coverage proving that streaming Chinese/CJK content across UTF-8 boundaries is correctly handled end to end. The current client still parses each WebSocket message independently and drops malformed JSON rather than buffering/reassembling it, so the reported user-visible streaming loss is not high-confidence resolved.
