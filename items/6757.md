## Summary

Keep #6757 open. Current main still has manual `/compact`, admin-scoped `sessions.compact`, auto-compaction, and context-engine/plugin compaction hooks, but it does not implement the requested agent-triggered self-compaction surface through `session_status(action: "compact")`, the agent `gateway` tool, or a dedicated CLI/session tool. The closest implementation PR, #43401, was closed unmerged in the provided GitHub context, and the latest maintainer/Codex issue comment also explicitly kept this item open.

## Best Possible Solution

Keep the issue open until maintainers either implement a first-class in-session self-compaction API, likely `session_status(action: "compact")` with mutation classification, active-session deadlock avoidance, counter/token persistence, and focused tests, or explicitly decide that agents must not be able to compact their own active session. Existing manual `/compact`, admin `sessions.compact`, automatic compaction, context-engine plugin hooks, and UI/control compaction should not be treated as satisfying this request.
