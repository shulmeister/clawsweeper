## Summary

Keep #6757 open. Current main still has manual/admin compaction and documented `/compact`, but it does not expose the requested agent-triggered self-compaction path through `session_status`, the agent gateway tool, or a dedicated CLI command.

## Best Possible Solution

Keep the issue open until maintainers either implement a first-class in-session self-compaction surface, likely `session_status(action: "compact")` with mutation classification and focused tests, or explicitly decide that agents should not be allowed to compact their own active session. The existing admin `sessions.compact` RPC and manual `/compact` command should not be treated as satisfying this issue.
