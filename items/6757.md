## Summary

Keep #6757 open. Current main has adjacent manual, UI, admin RPC, auto-compaction, and plugin/context-engine compaction paths, but it still lacks the requested in-session agent-triggered self-compaction surface through `session_status(action: "compact")`, an agent `gateway` compact action, or a dedicated self-compact command. The exact implementation attempt in PR #43401 was closed unmerged, and the latest issue discussion also kept this item open.

## Best Possible Solution

Keep #6757 open until OpenClaw either implements a first-class in-session self-compaction API or maintainers explicitly decide agents must not compact their own active session. The likely implementation path remains `session_status(action: "compact")` or an equivalent agent/session tool that uses direct compaction for the active run, is classified as mutating, respects session visibility and permissions, avoids lane deadlocks, persists compaction counters/token snapshots, and has focused tests for `session_status`, mutation tracking, and active-run behavior. PR #43401 is useful prior art, but it was closed unmerged and should not be treated as shipped.
