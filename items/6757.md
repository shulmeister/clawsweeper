## Summary

Keep #6757 open. Current main has adjacent compaction support, including automatic/runtime compaction and an admin-scoped `sessions.compact` RPC, but it still does not expose the requested in-session agent-triggered self-compaction through `session_status(action="compact")`, the agent-facing `gateway` tool, or a dedicated active-session CLI/tool surface. The related PR #43401 is useful prior art, but the provided GitHub context shows it was closed unmerged.

## Best Possible Solution

Keep #6757 open until OpenClaw implements a first-class in-session self-compaction API, preferably `session_status(action="compact")` or an equivalent active-session agent tool. A correct implementation should use direct compaction when already inside a run to avoid lane deadlocks, preserve session visibility and permission semantics, mark the action as mutating, update compaction counters/token snapshots, and add focused coverage for `session_status`, mutation tracking, and active-run behavior. PR #43401 remains useful prior art but is not shipped.
