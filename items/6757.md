## Summary

Keep #6757 open. Current main still has manual `/compact`, admin/control `sessions.compact`, auto-compaction, and plugin/context-engine compaction hooks, but it does not implement the requested agent-triggered self-compaction surface through `session_status(action: "compact")`, the agent `gateway` tool, or a dedicated CLI command. The related PR #43401 proposed exactly that path but is closed and unmerged in the provided GitHub context, and a recent maintainer/Codex review comment also explicitly kept this issue open.

## Best Possible Solution

Keep the issue open until maintainers either implement a first-class in-session self-compaction API, likely `session_status(action: "compact")` with mutation classification and focused tests, or explicitly decide that agents must not be able to compact their own active session. Existing manual `/compact`, admin `sessions.compact`, context-engine plugin compaction, and UI/control compaction should not be treated as satisfying this request.
