## Summary

Keep #32438 open. Current main still writes only JSONL gateway file logs and renders formatted text only in CLI/UI consumers; it does not create the requested persistent sibling `-formatted` file for direct `tail`/`grep` when the Gateway or Control UI is unavailable. The earlier related issue #29378 is closed and PR #32442 was closed unmerged, so this remains the active tracker.

## Best Possible Solution

Keep this issue open until maintainers either implement the core logger behavior or explicitly reject the product direction. A complete fix should derive a sibling path from `logging.file`, write one human-readable redacted line whenever the JSONL file is written, preserve existing JSONL consumers, define formatted-file rotation/retention behavior, add focused logger/log-tail tests, and update the public logging docs.
