## Summary

Keep #69629 open. Current main protects Discord slash command execution with account/channel allowlists and per-provider command authorization, but it does not implement the requested Discord UI visibility/scoping. The current docs explicitly warn that commands may still be visible to unauthorized Discord users, and the Discord plugin currently builds/deploys one native command set per bot account via Carbon rather than deriving per-channel command permissions from `guilds.<id>.channels` or `bindings`.

## Best Possible Solution

Keep the issue open as a Discord plugin enhancement. The best path is to add a maintainer-reviewed design for per-account/per-guild channel visibility, preferably deriving command visibility from existing Discord guild/channel allowlists or a narrow new `channels.discord.commands` setting. If Discord supports channel permission overwrites for these commands, implement that in the Discord deploy/reconcile path and add tests around the REST payload/permission model plus interaction authorization. If Discord cannot reliably hide commands per channel, document the limitation and provide a supported text-only workaround such as `channels.discord.commands.native=false` for selected accounts, while preserving current execution authorization.
