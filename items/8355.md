## Summary

Keep #8355 open. Current main still batches the classic voice-call STT/agent/TTS path: it awaits one completed embedded-agent text result, then awaits one completed telephony TTS audio buffer, then chunks that buffer to Twilio. The newer realtime voice path is useful but separate and mutually exclusive with the classic streaming transcription path, and the related streaming TTS PR #42187 was closed unmerged.

## Best Possible Solution

Keep #8355 open as the canonical voice-call feature request. The best implementation path is a provider-generic sentence-level pipeline in the voice-call plugin: feed embedded-agent partial replies into a sentence splitter, add a streaming telephony TTS contract with a buffered fallback, stream audio chunks through Twilio and Telnyx media handlers, and preserve abort/barge-in behavior for in-flight synthesis and playback.
