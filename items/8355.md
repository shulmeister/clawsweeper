## Summary

Keep #8355 open. Current main still batches the classic voice-call STT -> embedded-agent -> TTS path: it waits for a completed text response, waits for a completed telephony audio buffer, and only then chunks audio to Twilio. The newer realtime voice path is useful but separate, and the related streaming TTS PR #42187 was closed unmerged.

## Best Possible Solution

Keep #8355 open as the canonical voice-call feature request. The best implementation path is a provider-generic sentence-level pipeline in the voice-call plugin: feed embedded-agent partial replies into a sentence splitter, add a streaming telephony TTS contract for speech providers, stream audio chunks through Twilio and Telnyx media handlers with abort/barge-in support, and preserve a buffered fallback for providers that only return complete audio buffers.
