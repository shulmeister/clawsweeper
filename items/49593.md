## Summary

Keep open. Current `main` still rejects `bindings[].tools` in route bindings, and the runtime tool-policy pipeline has no route-binding input, so the requested behavior is not implemented. The PR's schema-only approach would also leave `tools.allow`/`tools.deny` as a misleading no-op.

## Evidence

## Risks / Open Questions

- Merging the PR as written would accept `bindings[].tools` in config without enforcing it anywhere, which is especially risky for security-sensitive tool restrictions like `exec` or `sessions_spawn`.
- Closing this PR would lose a still-unimplemented feature request/bugfix; current `main` does not support the advertised per-binding tool policy behavior.
