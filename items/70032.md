## Summary

Current main does not implement the requested Nextcloud Talk typing indicator. The bundled Nextcloud Talk plugin sends messages and reactions, and its inbound path dispatches replies without any typing lifecycle. This is a concrete, recent plugin-owned enhancement rather than an obsolete, duplicate, or non-actionable report.

## Best Possible Solution

Keep #70032 open as a scoped Nextcloud Talk plugin enhancement. The likely fix belongs in `extensions/nextcloud-talk`: add a typed `sendTypingNextcloudTalk` helper for the `/ocs/v2.php/apps/spreed/api/v1/chat/{token}/typing` endpoint, add config/docs/tests for the default behavior, and wire start/stop cleanup around inbound reply dispatch. If the shared `dispatchInboundReplyWithBase` helper cannot pass plugin typing callbacks cleanly, extend that SDK helper additively or use the lower-level reply pipeline in this plugin.
