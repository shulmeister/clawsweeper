## Summary

Keep #72262 open. Current main still has only generic WhatsApp reconnect-loop troubleshooting and does not document a dedicated `status=408 Request Time-out` runbook. The QEEK-AI comment says docs were added, but the inspected current main docs and history do not contain the claimed 408 section or troubleshooting-table entry. The request is recent, docs-scoped, and complements the existing WhatsApp runtime/status diagnostics rather than duplicating or superseding them.

## Best Possible Solution

Keep the issue open and add a focused docs section in `docs/channels/whatsapp.md`, with a short cross-link or table row in `docs/channels/troubleshooting.md`. The section should explain what WhatsApp/Baileys 408 disconnects usually mean, the first diagnostic commands, how to choose wait vs relink vs DNS/proxy/connectivity checks vs plugin runtime-dependency repair, what a healthy credentials directory contains, and the safe `openclaw channels logout --channel whatsapp --account <id>` then `openclaw channels login --channel whatsapp --account <id>` recovery path with auth-dir backup cautions.
