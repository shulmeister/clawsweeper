## Summary

Current main already implements a deferred, graceful gateway restart path for config changes and adds post-restart orphan-session recovery, which addresses the reported hot-reload/session-loss behavior.

## Close Comment

Closing from a Codex review because current `main` already implements this restart path. Config-triggered gateway restarts are now deferred until active operations/replies/embedded runs/tasks drain (`src/gateway/server-reload-handlers.ts:201`), the restart loop performs a graceful shutdown with explicit restart semantics (`src/cli/gateway-cli/run-loop.ts:139`), and server shutdown now broadcasts `gateway restarting` and closes sockets with WebSocket code `1012` / `service restart` instead of a raw `1006` abnormal closure (`src/gateway/server-close.ts:161`). Current `main` also resumes restart-aborted sessions after boot via orphan recovery with retry handling (`src/agents/subagent-orphan-recovery.ts:146`), with regression coverage in `src/agents/subagent-orphan-recovery.test.ts:94`. The changelog shows the shipped restart/orphan-recovery fix in `2026.3.22` (`CHANGELOG.md:2185`, `CHANGELOG.md:2375`).

If you still hit a restart regression on a newer build, please open a fresh issue with a current stability bundle/logs and a focused reproduction. The fallback-chain note and `/workspace` alias note should be tracked separately if they still reproduce.
