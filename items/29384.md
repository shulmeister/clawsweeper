## Summary

Keep #29384 open. Current main still has no Docker sandbox support for GPU, IPC, or a raw/extra Docker create-args surface; the supported sandbox.docker fields are enumerated in types/schema/docs and buildSandboxCreateArgs only emits the existing hardening/resource/bind flags. OpenShell has a separate gpu option, but that is a different backend and does not satisfy Docker --gpus all or --ipc=host for the built-in Docker sandbox. The related #29933 is adjacent Docker deployment/socket work, not a duplicate of this Docker sandbox parameter request.

## Best Possible Solution

Keep the issue open until a maintainer-approved Docker sandbox extension lands. A concrete fix would add explicit security-reviewed fields such as sandbox.docker.gpus and sandbox.docker.ipc, or a narrowly constrained extra-create-args mechanism, then update SandboxDockerSettings, the strict Zod schema, config resolution/hash behavior, validateSandboxSecurity, buildSandboxCreateArgs, docs, and regression tests. Because --ipc=host and GPU/device access weaken isolation, the implementation should make that risk explicit rather than silently passing arbitrary Docker flags.
