## Summary

Current `main` no longer matches the report's core claim that end users have no supported path to give the main embedded agent custom tools: OpenClaw-managed `mcp.servers` now feed embedded Pi tool registration, bundle/configured MCP tools are surfaced in normal tool profiles, and the plugin path is publicly documented with install and tool-authoring guides. Skills remain prompt guidance by design, but the absence-of-any-path issue is implemented well enough to close.

## Risks / Open Questions

- Skills are still prompt-only guidance rather than first-class tool registration, so users specifically expecting SKILL.md to create callable tools may still be surprised.
- `docs/tools/creating-skills.md` still contains wording about custom tool schemas in skill frontmatter that may read broader than the reviewed supported path.
