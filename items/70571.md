## Summary

Keep open. Current main has not implemented compound `safeBins` matching, and the linked bug #59600 remains open, so this PR is not obsolete or implemented. However, the submitted patch is not safe or clean enough to merge as-is: the PR review comments correctly flag that its compound prefix branch would bypass existing `validateSafeBinArgv` profile checks, and the PR also bundles a large unrelated README rewrite.

## Best Possible Solution

Keep the PR open for maintainer handling rather than auto-closing it. The best path is to ask for a narrow rewrite that removes the README changes, preserves trusted-path checks, preserves or extends `validateSafeBinArgv` semantics for any compound entry, adds regression tests for extra-argument and denied-flag bypasses, and reconciles the behavior with the documented safeBins-versus-allowlist contract. If maintainers decide compound commands should not be supported in `safeBins`, close this PR manually in favor of updating #59600 with the supported explicit allowlist workflow.
