## Summary

Keep open: current `main` still uses the strict `startedAt < createdAt` audit check, and task creation still records `createdAt` from a later `Date.now()` than caller-supplied `startedAt`, so the reported `inconsistent_timestamps` false positives are not implemented away on `main` or the latest release.
