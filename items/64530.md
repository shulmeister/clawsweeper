## Summary

Keep open. Current main has first-party replay-time tool-call pairing repair and runtime pending-tool flushes, but it does not implement the requested reopen/startup append-only transcript compensation. Assistant-tail tool calls can still be treated as non-resumable on restart, and orphan user tails are handled by merging into the next prompt and branching/resetting the active leaf rather than appending an explicit recovery marker.

## Best Possible Solution

Keep the issue open and implement a narrow startup/reopen transcript repair seam that inspects only the active tail, appends synthetic error `toolResult` rows via `SessionManager.appendMessage(...)` for unresolved assistant tail tool calls, appends one idempotent assistant recovery marker for persisted orphan-user tails, and adds regression tests for missing tail tool results, partial parallel results, repeated reopen no-op, healthy transcript no-op, and no tool re-execution. The existing replay sanitizer can remain as provider-safety defense, but it should not be the only source of recovery truth.
