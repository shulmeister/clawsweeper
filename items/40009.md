## Summary

Keep open. Current main still implements the Control UI Raw config surface as a native textarea, with textarea-specific CSS and tests. I found no CodeMirror, Monaco, Ace, jsoneditor, folding, syntax-highlighting, formatting, or inline validation integration in the UI dependencies/source, and the issue discussion only contains an implementation recommendation, not a shipped fix or canonical superseding item.

## Best Possible Solution

Keep the issue open and implement a first-class Raw config code editor in `ui/src/ui/views/config.ts`, likely backed by CodeMirror v6 for JSON/JSON5. Preserve the current `rawAvailable`, sensitive-value reveal/redaction, `onRawChange`, Reset to saved, `config.set`, `config.apply`, base-hash, and safe round-trip semantics while adding folding, syntax highlighting, formatting, and parse/validation feedback. Update `ui/src/styles/config.css` and `ui/src/ui/views/config.browser.test.ts` so tests assert the editor contract rather than native textarea behavior.
