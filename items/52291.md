## Summary

Keep open. Current `main` appears to have picked up only the auto-install part of the PR; the Windows path-with-spaces launcher fix is not present in `scripts/ui.js`, so there is not strong evidence that the reported `pnpm ui:build` Windows failure is already implemented or obsolete.

## Evidence

- **Windows shell mode still uses the raw command path:** `createSpawnOptions` enables `shell: true` for `.cmd/.bat/.com` launchers on Windows, but `run` and `runSync` still pass the unmodified `cmd` string into `spawn`/`spawnSync`. There is no quoting helper on current `main`.
  - file: [scripts/ui.js:79](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/scripts/ui.js#L79)

- **No current test covers spaced Windows shell-launcher paths:** The Windows tests cover shell detection and unsafe arg rejection, but there is no assertion for a spaced launcher path like `C:\Program Files\...\pnpm.cmd`, which is the bug this PR addresses.
