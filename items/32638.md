## Summary

Keep open. Current main has a shipped narrow mitigation for Groq `qwen/qwen3-32b`, and the fallback loop now continues past unknown intermediate errors when more candidates remain, but the broader reported reasoning-vocabulary problem remains. Other Groq reasoning models and LM Studio/Qwen3-style `on`/`off` vocabularies are still not preserved or mapped by the current compat path, matching the April 2026 issue comments.

## Best Possible Solution

Keep tracking this until OpenClaw has a provider/model-native reasoning compatibility path. A strong fix would preserve discovered reasoning vocabularies such as LM Studio `on/off`, expose or type the compat metadata needed for provider-specific mappings, broaden Groq handling beyond a single model id if Groq still requires `none/default`, and add regression coverage around `buildOpenAICompletionsParams` for Groq and LM Studio/Qwen-style models.
