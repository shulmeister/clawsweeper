## Summary

Keep open. Current main still has the exact Linux node-service systemd unit mismatch described in #68287: node install injects the node unit override, the unit-file path resolver honors that override, but install activation and uninstall still call the gateway-only resolver before running `systemctl enable/restart/disable`. The related fix PR #68293 is still open in the provided GitHub context, so the issue remains a valid tracker rather than cleanup-closeable.

## Best Possible Solution

Keep #68287 open until #68293 or an equivalent patch lands. The fix should make `activateSystemdService()` and `uninstallSystemdService()` use `resolveSystemdServiceName(env)`, and add focused regression coverage proving node install/uninstall call `systemctl enable/restart/disable` with `openclaw-node.service` while gateway installs keep the gateway unit behavior.
