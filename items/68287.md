## Summary

Keep open. Current `main` still appears to have the reported bug: node service install/uninstall sets `OPENCLAW_SYSTEMD_UNIT=openclaw-node`, but `activateSystemdService` and `uninstallSystemdService` still hard-code the gateway resolver instead of the override-aware resolver.

## Evidence

- **Node installs set a node-specific systemd unit override:** `withNodeServiceEnv` injects `OPENCLAW_SYSTEMD_UNIT: resolveNodeSystemdServiceName()`, so node-only installs expect `openclaw-node.service`, not the gateway unit.
  - file: [src/daemon/node-service.ts:18](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/daemon/node-service.ts#L18)
  - command: `nl -ba src/daemon/node-service.ts | sed -n '1,120p'`
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Shared resolver already honors OPENCLAW_SYSTEMD_UNIT:** `resolveSystemdServiceName` trims and uses `env.OPENCLAW_SYSTEMD_UNIT` before falling back to the gateway profile-based name.
  - file: [src/daemon/systemd.ts:52](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/daemon/systemd.ts#L52)
  - command: `nl -ba src/daemon/systemd.ts | sed -n '40,70p'`
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Install path still hard-codes the gateway resolver:** `activateSystemdService` calls `resolveGatewaySystemdServiceName(params.env.OPENCLAW_PROFILE)`, so `systemctl enable/restart` targets `openclaw-gateway*.service` even when `OPENCLAW_SYSTEMD_UNIT` is set for node installs.
  - file: [src/daemon/systemd.ts:542](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/daemon/systemd.ts#L542)
  - command: `nl -ba src/daemon/systemd.ts | sed -n '520,620p'`
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Uninstall path has the same mismatch, while stop/restart use the correct helper:** `uninstallSystemdService` also hard-codes `resolveGatewaySystemdServiceName(...)` at line 591, but `runSystemdServiceAction` uses `resolveSystemdServiceName(env)` at line 612, confirming the inconsistency is still present on `main`.
  - file: [src/daemon/systemd.ts:591](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/daemon/systemd.ts#L591)
