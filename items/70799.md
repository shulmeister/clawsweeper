## Summary

PR #70799 should remain open. Current main still lacks the proposed outer guards in `markBlueBubblesChatRead` and `sendBlueBubblesTyping`; the only skip for disabled BlueBubbles Private API is inside the shared helper after account/client resolution. The runtime still invokes read receipts and typing automatically on inbound replies, and the PR includes a focused production repro plus live validation. The provided related #71470 is a different SQLite contacts fix, not a duplicate or superseding implementation.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land an equivalent BlueBubbles-owned fix in `extensions/bluebubbles/src/chat.ts`, or a deeper probe/deferral fix if maintainers prefer, with regression coverage for disabled Private API under the reported cache/account state. Coordinate with any overlapping typing-only work, but this PR still covers the read-receipt path too.
