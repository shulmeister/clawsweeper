## Summary

Keep #38683 open. Current main still enforces the explicit Telegram contract that `dmPolicy="open"` must include `allowFrom: ["*"]` during normal config validation, while the shipped repair path is doctor repair rather than startup/config-load auto-heal. The PR discussion is specific and actionable, including a security-sensitive account-level widening review comment that the author addressed in follow-up commit `792311545edfe18520724a6a448f248f3c0155ee`.

## Best Possible Solution

Keep the PR open for maintainer product judgment. The best path is to decide whether Telegram should keep requiring explicit `allowFrom: ["*"]` at startup and rely on `openclaw doctor --fix`, or accept a narrowly scoped startup/config-load normalization with account-level safeguards, warning behavior without test-order coupling, regression tests, docs, and changelog coverage.
