## Summary

Keep open. Current `main` still enforces Telegram `dmPolicy="open"` as a hard validation error unless `allowFrom` already contains `"*"`; the PR's proposed startup/schema auto-heal is not present. A separate doctor repair path exists for open-policy wildcards, but that does not make the runtime/config validation behavior described in this PR implemented on `main`.
