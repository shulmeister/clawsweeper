## Summary

Current main still rejects Telegram `dmPolicy="open"` unless the effective `allowFrom` contains `"*"`. Main has a doctor repair path that adds the wildcard, but that does not implement this PR's requested cold-start/startup auto-heal, so the PR is not obsolete enough for conservative auto-close.

## Best Possible Solution

Keep this PR open for maintainer judgment. The remaining product choice is whether Telegram should retain the explicit `allowFrom: ["*"]` startup validation contract and rely on `openclaw doctor --fix`, or add a narrowly scoped startup/config-load normalization with account-level tests, warning behavior, docs, and changelog coverage.
