## Summary

Keep #38683 open. Current main still enforces Telegram `dmPolicy="open"` as invalid unless effective `allowFrom` contains `"*"`, and gateway startup still rejects invalid config snapshots rather than applying the requested startup-time auto-heal. Main has a doctor repair path that partially covers recovery, but that does not satisfy the PR's central cold-start failure prevention. The PR discussion also contains useful security review context, and the author addressed the account-level widening concern in PR commit `792311545edfe18520724a6a448f248f3c0155ee`, so this is still a plausible focused bug-fix PR rather than stale cleanup.

## Best Possible Solution

Keep this PR open for maintainer judgment. The best path is either to implement a carefully scoped startup normalization that preserves per-account security boundaries and adds regression tests/docs, or to explicitly close the product direction by documenting that `doctor --fix` is the supported repair path and keeping startup validation strict.
