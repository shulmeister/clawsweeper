## Summary

Keep open. Current main does not provide strong evidence that the launchd-only startup hang is fixed: the generated macOS LaunchAgent plist still has stdout/stderr paths but no `StandardInPath`/`/dev/null`, and the related open #58414 explicitly narrows this same failure mode to missing launchd stdin handling. The issue is also not old enough for `stale_insufficient_info` and has no protected/maintainer author signal requiring a close/keep exception.

## Evidence

- **LaunchAgent plist still omits stdin:** `buildLaunchAgentPlist` renders `ProgramArguments`, optional `WorkingDirectory`, `StandardOutPath`, `StandardErrorPath`, and environment variables, but no `StandardInPath` entry. That leaves the specific stdin workaround requested by related #58414 unimplemented in current main.
  - file: [src/daemon/launchd-plist.ts:116](https://github.com/openclaw/openclaw/blob/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071/src/daemon/launchd-plist.ts#L116)
  - command: `nl -ba src/daemon/launchd-plist.ts | sed -n '1,180p'`
  - sha: [28497515fe1e](https://github.com/openclaw/openclaw/commit/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071)
- **Gateway install path passes only stdout/stderr log paths:** `writeLaunchAgentPlist` resolves log paths and calls `buildLaunchAgentPlist` with `stdoutPath` and `stderrPath`; there is no stdin path parameter or defaulting behavior in the install/rewrite path.
  - file: [src/daemon/launchd.ts:540](https://github.com/openclaw/openclaw/blob/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071/src/daemon/launchd.ts#L540)

  - sha: [28497515fe1e](https://github.com/openclaw/openclaw/commit/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071)
- **No current StandardInPath implementation found:** A targeted repository search for `StandardInPath` in daemon, CLI, docs, and macOS app surfaces returned no matches, so there is no source or test evidence that current main added the launchd stdin directive.
  - command: `rg -n "StandardInPath" src/daemon src/cli docs apps/macos`
  - sha: [28497515fe1e](https://github.com/openclaw/openclaw/commit/28497515fe1e6fa7ab5d51e64d2f3fe76ea39071)
- **Related issue remains open with same suspected launchd stdin cause:** Provided GitHub context shows #58414 is open and explicitly says its LaunchAgent stdin hang is related to #46153, requesting `StandardInPath=/dev/null` in `openclaw gateway install`. That related remaining work is not superseded by the current plist implementation.
