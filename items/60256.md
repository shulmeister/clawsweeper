## Summary

Keep open. Current main does not show an implemented fix for `env:` cron delivery targets: cron delivery preserves `delivery.to` as a literal string, passes it into `resolveDeliveryTarget`, and Discord outbound normalization only trims/prefixes bare numeric IDs while returning other non-empty targets unchanged. The reported `env:DISCORD_BACKUPS_CHANNEL` syntax is therefore not clearly supported or fixed on main.
