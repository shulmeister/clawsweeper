## Summary

Keep this PR open. Current main has lower-level `media://inbound` hydration and a direct image-tool test, but the normal agent tool wiring still passes `workspaceDir`; in that path the image tool rewrites `media://inbound/<id>` as a workspace-relative path before `loadWebMedia` can resolve the managed media URI. The provided PR body and Greptile review describe the same bug class and support the proposed direction, though the patch likely needs rebasing onto the newer shared media-reference helpers.

## Best Possible Solution

Keep the PR or a replacement fix open. The best implementation is to preserve `media://inbound/<id>` through image-tool resolution when `workspaceDir` is set, for example by excluding `refInfo.isMediaStoreUrl` from workspace-relative path resolution or resolving managed media refs before that branch. Add regression coverage for the agent-session shape: `createImageTool` or `createOpenClawTools` with `workspaceDir` and `fsPolicy: { workspaceOnly: true }`, covering both bare `media://inbound/<id>` and `MEDIA:media://inbound/<id>`.
