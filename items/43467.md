## Summary

Keep open. Current main still has the reported cosine-similarity length-mismatch behavior: both current memory-host SDK copies compute dot and norms only over Math.min(a.length, b.length), and memory-core fallback vector search still calls this utility. The PR is useful but stale against the current file layout, so it should be ported/rebased rather than closed.

## Best Possible Solution

Keep the PR open and ask for a small rebase/port: add the remaining-dimension norm accumulation to the current memory-host SDK implementation(s), add colocated cosineSimilarity regression tests for different-length vectors, and verify the changed memory-host/memory-core test lane.
