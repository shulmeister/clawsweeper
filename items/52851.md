## Summary

Current main already prevents the duplicate CLI/gateway startup path behind this report by separating the CLI wrapper entrypoint from the package root and guarding both with `isMainModule`; the same guarded entrypoint is present in the `v2026.4.22` tag.

## Close Comment

Closing from a Codex review because current OpenClaw already implements the fix for this duplicate-start path. The current CLI entrypoint now guards top-level side effects with `isMainModule` in `src/entry.ts:33`, and the comment there explicitly notes the old behavior could start a duplicate gateway when `entry.js` was imported as a dependency. `src/index.ts:52` also only runs the legacy CLI when `dist/index.js` is truly the main module, and `src/infra/is-main.test.ts:38` covers the `openclaw.mjs -> entry.js` wrapper case. That matches the report’s `openclaw gateway --help` repro especially well: `--help` should not need a gateway RPC at all, so a second gateway process there points to duplicate entry execution rather than missing systemd detection. The same guarded entrypoint is already present in the `v2026.4.22` release tag.
