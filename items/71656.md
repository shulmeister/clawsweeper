## Summary

Keep #71656 open. Current main has partial fixes for dreaming-corpus noise and downstream weighted/recency-aware promotion, but the issue's central remaining complaint is still present: daily and session-corpus ingestion still record fixed source scores before promotion ranking. The scan-stall concern also lacks direct budget-pressure regression coverage, and the related open items do not fully cover this issue's combined filtering, scoring, and session-corpus rotation scope.

## Best Possible Solution

Keep the issue open and narrow it into implementation work: add content-type-aware source scoring before `recordShortTermRecalls` for daily/session dreaming ingestion, extend corpus pre-filtering/deduplication for repeated blocks and low-signal status/test messages, and add regression coverage proving session-corpus scans continue across files and days under budget pressure. Link #57307 for the broader importance/time-decay policy if maintainers want to consolidate scoring design, but keep #71656 tracking the corpus filtering and scan-rotation specifics.
