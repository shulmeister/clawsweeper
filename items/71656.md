## Summary

Keep open. Current main partially improves session noise filtering and checkpointed transcript scanning, but it still does not implement the central requested fixes: daily/session ingestion scores remain fixed constants, session dedupe still includes transcript position or timestamp in the identity, and the remaining scan-rotation concern is only partially covered. Related #68876 is useful corpus-noise work, but it is open and not a complete fix for this issue.

## Best Possible Solution

Keep this open and narrow the implementation to memory-core: add content-type-aware scoring before recordShortTermRecalls for daily and session ingestion, add content-level dedupe for repeated session blocks independent of transcript timestamp/line position, and add regression coverage for budget fairness across multiple large corpus sources. Treat #68876 as a related noise-filtering improvement if it lands, not as the canonical completion of this issue.
