## Summary

Keep #37626 open. Current main still implements Feishu wiki node listing as one `client.wiki.spaceNode.list` call with only `parent_node_token`, so `feishu_wiki` `nodes` still cannot automatically walk Feishu `has_more` / `page_token` pages. The related PR #37641 targeted the right surface, but the provided GitHub context shows it was closed unmerged, and its pagination behavior is not present in current main or latest release `v2026.4.24`.

## Best Possible Solution

Keep the issue open and fix it in the Feishu plugin. The likely change belongs in `extensions/feishu/src/wiki.ts`: request a bounded `page_size`, continue while Feishu returns `has_more` with a usable `page_token`, accumulate all node items, and add a focused regression test mocking multiple `spaceNode.list` pages. Manual pagination fields are optional product design; automatic full pagination addresses the central data-loss bug without changing the user workflow.
