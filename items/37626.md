## Summary

Keep open. Current main still single-pages the bundled Feishu `feishu_wiki` `nodes` action: it calls `client.wiki.spaceNode.list` once, passes only `parent_node_token`, returns only that response's items, and exposes no pagination fields in the tool schema. The linked fix attempt, PR #37641, is closed unmerged in the provided GitHub context, so it does not supersede the issue.

## Best Possible Solution

Keep #37626 open until the bundled Feishu plugin paginates `feishu_wiki` `nodes`. The likely fix belongs in `extensions/feishu/src/wiki.ts`: request an appropriate page size, continue while Feishu returns more pages, accumulate all node items, and add a focused regression test that mocks multiple `spaceNode.list` pages. Schema additions for manual `page_size`/`page_token`/`fetch_all` should be a maintainer product choice; automatic full pagination would solve the central data-loss problem without changing the user workflow.
