## Summary

Keep open. Current `main` still implements `feishu_wiki` `nodes` as a single `spaceNode.list` call, with no `page_size`, `page_token`, `has_more` loop, or returned pagination metadata. The related PR #37641 directly addressed this, but the provided GitHub context shows it was closed unmerged, so it does not supersede the bug.

## Best Possible Solution

Keep the issue open until the bundled Feishu plugin implements wiki node pagination. The likely fix belongs in `extensions/feishu/src/wiki.ts` and `extensions/feishu/src/wiki-schema.ts`: pass `page_size` and `page_token`, loop on `has_more` with a safety bound, aggregate all nodes, return useful metadata such as `total` and `pages_fetched`, and add focused Feishu wiki tests covering single-page, multi-page, and missing-token edge cases. PR #37641 is useful prior art but should be refreshed against current `main` rather than treated as merged.
