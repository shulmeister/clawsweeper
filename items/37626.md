## Summary

Keep open. Current `main` still single-pages the bundled Feishu `feishu_wiki` `nodes` action: `listNodes` calls `client.wiki.spaceNode.list` once with only `parent_node_token`, the schema exposes no pagination controls, and the latest release tag has the same implementation. The linked fix attempt, PR #37641, is closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep #37626 open until the bundled Feishu plugin paginates `feishu_wiki` `nodes`. The likely fix belongs in `extensions/feishu/src/wiki.ts`: request an appropriate `page_size`, continue while `has_more` and `page_token` are present, accumulate all node items, and add a focused regression test that mocks multiple `spaceNode.list` pages. Schema additions for manual `page_size`/`page_token`/`fetch_all` should be a maintainer product choice; automatic full pagination would solve the central data-loss problem without changing the user workflow.
