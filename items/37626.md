## Summary

Keep open. Current main still single-pages the Feishu wiki `nodes` action: `listNodes` calls `client.wiki.spaceNode.list` once, passes only `parent_node_token`, and returns only `res.data.items`. The schema also lacks `page_size`, `page_token`, or `fetch_all`. The linked PR #37641 directly addressed this, but it was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep #37626 open until the bundled Feishu plugin implements pagination for `feishu_wiki` `nodes`. The likely fix belongs in `extensions/feishu/src/wiki.ts`: request `page_size` up to Feishu's max, continue while `has_more` and `page_token` are present, accumulate all node items, and add focused regression coverage that mocks multiple `spaceNode.list` pages. Schema additions for manual `page_size`/`page_token`/`fetch_all` should be a maintainer product choice; automatic full pagination would solve the central reported data-loss problem without requiring user workflow changes.
