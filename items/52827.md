## Summary

Close as implemented on main. Current main, and the shipped `v2026.4.22` release tree, already sanitize/rewrite replayed OpenAI Responses tool-call ids in the embedded runtime path and carry regression coverage for the `call_id|fc_*` replay case.

This Codex review checked current `main` and the shipped `v2026.4.22` tree, and the embedded OpenAI Responses replay path already rewrites replay ids before outbound payload construction: `src/agents/openai-transport-stream.ts:207` normalizes id parts and bounds them, while `src/agents/openai-transport-stream.ts:303` / `:326` split stored `call_id|fc_id` pairs so `function_call.call_id` and `function_call_output.call_id` only send the call-id portion. Based on that, this specific bug is already fixed in released OpenClaw `v2026.4.22`.
