## Summary

Keep #46656 open. Current main still lacks the requested WebChat/Control UI inline-button contract: no webchat inlineButtons config, no buttons field in chat.send/chat events, no chat.callback RPC, and no Control UI rendering/callback path for button content. The related #54720 ClawSweeper report tracks the same remaining gap, but there is no maintainer/canonical signal strong enough to close this older, detailed issue as a duplicate.
