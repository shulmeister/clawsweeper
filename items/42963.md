## Close Comment

Closing from a Codex review because this is already implemented on `main` and was shipped in `v2026.4.12`. The current replay policy preserves signed Anthropic thinking for Claude 4.5+ models instead of dropping or rewriting those blocks (`src/plugins/provider-replay-helpers.ts:73`, `src/agents/transcript-policy.ts:97`), and the replay sanitizer now drops an entire signed-thinking assistant turn rather than mutating provider-owned content when sibling tool calls would need normalization (`src/agents/session-transcript-repair.ts:281`). There is also explicit regression coverage for Claude Opus 4.5+/Sonnet 4.5+ replay preservation (`src/plugins/provider-replay-helpers.test.ts:80`, `src/agents/transcript-policy.test.ts:288`). The changelog entry for `2026.4.12` records the shipped fix as `Agents/Anthropic replay: preserve immutable signed-thinking replay safety across stored and live reruns...` (`CHANGELOG.md:725`, `CHANGELOG.md:791`). If you still hit this on `v2026.4.12` or newer, please open a fresh issue with the exact provider/model route and a minimal current repro. The separate LaunchAgent KeepAlive note here looks like a different bug and should be tracked separately.
