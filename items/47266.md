## Summary

Keep open. Current main still has the reported logging rollover problem: the default dated log path can change after midnight, but long-lived logger transports and subsystem child loggers keep writing through the logger instance created before the date changed. Related PRs #47370 and #47700 target this same remaining issue, but the fix is not present on current main.
