## Summary

Keep open. The requested `agents.defaults.compaction.memoryFlush.model` override is still not implemented on current `main`: the validated config/schema/docs expose only the existing memory-flush keys, the memory flush plan has no model field, and the runtime still runs the flush through the session run's provider/model. The linked PR #66846 is useful and directly targets this issue, but the provided context shows it remains open and unmerged, so the original issue should continue tracking the missing shipped behavior.

## Best Possible Solution

Keep #66802 open until PR #66846 or an equivalent maintainer-approved change lands. The implementation should add `agents.defaults.compaction.memoryFlush.model` to config types, zod/generated schema, labels/help/docs, carry it through the memory flush plan contract, and have `runMemoryFlushIfNeeded()` resolve provider/model/auth behavior analogously to `compaction.model`, with tests for override, fallback, and schema validation.
