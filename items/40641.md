## Summary

#40641 should stay open. Current main still contains unquoted bracket-notation config examples on the affected docs pages, including the exact command from the issue title. The CLI parser already supports bracket notation once the shell passes the argument through, so the remaining problem is the docs examples/workaround guidance rather than a missing config path parser. The related PR #40642 identified the right docs-only fix, but it is closed and unmerged, and no equivalent commit is present in the local history.

## Best Possible Solution

Keep the issue open as a concrete docs regression. Reapply the intent of #40642: quote every config CLI example that uses bracket notation, or switch those examples to dot-index form such as `agents.list.0.tools.exec.node`, and add a short note on the config docs explaining that paths containing `[]` must be quoted in shells like zsh. The likely source files are `docs/cli/config.md`, `docs/tools/exec.md`, and `docs/nodes/index.md`; localized pages should be regenerated through the docs pipeline rather than edited directly in this repo.
