## Summary

Keep open. Current main still treats `NO_PROXY` with undici-style hostname/wildcard matching only, not CIDR matching, and the shared provider HTTP helper still auto-upgrades to trusted env-proxy mode whenever that matcher returns false. A CIDR-only `NO_PROXY` entry like `192.168.x.0/24` therefore still has a plausible path where LAN provider requests stay on the proxy route.

## Evidence

- **`matchesNoProxy` does not implement CIDR parsing:** The supported cases are exact host, suffix, wildcard, optional port, and bracketed IPv6. The implementation only tokenizes entries and compares normalized host strings; it never parses CIDRs or calls the repo's CIDR helper.
  - file: [src/infra/net/proxy-env.ts:57](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/infra/net/proxy-env.ts#L57)
  - command: `nl -ba src/infra/net/proxy-env.ts | sed -n '1,220p'`
- **Provider auto-upgrade still depends on `matchesNoProxy`:** `shouldAutoUpgradeToTrustedEnvProxy` returns false only when `matchesNoProxy(url)` is true. Otherwise `fetchWithTimeoutGuarded` selects `TRUSTED_ENV_PROXY`, which routes through `EnvHttpProxyAgent` for provider HTTP helpers.
  - file: [src/media-understanding/shared.ts:245](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/media-understanding/shared.ts#L245)
  - command: `nl -ba src/media-understanding/shared.ts | sed -n '240,340p'`
- **Tests cover hostname/wildcard/port cases but no CIDR bypass case:** The current `matchesNoProxy` tests enumerate wildcard, exact host, suffix, whitespace, port, and IPv6 behavior, but there is no CIDR expectation such as `192.168.0.0/24`.
  - file: [src/infra/net/proxy-env.test.ts:98](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/infra/net/proxy-env.test.ts#L98)
  - command: `nl -ba src/infra/net/proxy-env.test.ts | sed -n '90,260p'`
- **OpenClaw already has a CIDR utility, but this path does not use it:** `isIpInCidr` exists in shared networking utilities, which makes the absence of CIDR handling in `matchesNoProxy` a real gap rather than a repo-wide limitation.
  - file: [src/shared/net/ip.ts:308](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/shared/net/ip.ts#L308)
  - command: `nl -ba src/shared/net/ip.ts | sed -n '300,345p'`
- **Recent shipped fix only covered ordinary `NO_PROXY` target matches:** The changelog entry for #52162 says provider requests now auto-upgrade only when the target is not bypassed by `NO_PROXY`, but it does not claim CIDR support or warning coverage.
  - file: [CHANGELOG.md:691](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/CHANGELOG.md#L691)
  - command: `nl -ba CHANGELOG.md | sed -n '684,696p'`
