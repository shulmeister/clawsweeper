## Summary

Keep #20837 open. Current main already carries channel metadata for ordinary agent runs, but the slash-command system-prompt rebuild path still drops the available command channel and runtime details. The linked fix PR #21271 is closed unmerged in the provided GitHub context, so the remaining core gap is still actionable.

## Best Possible Solution

Keep the issue open and implement the missing slash-command runtime metadata in src/auto-reply/reply/commands-system-prompt.ts. Mirror the embedded runner pattern for machine/shell details, channel normalization, account-aware capabilities and channel actions, then add focused regression coverage proving /context or /export-session from Telegram includes channel metadata. Treat selective cross-channel reply policy as a separate follow-up.
