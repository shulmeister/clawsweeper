## Summary

Keep #20837 open. Current main already exposes channel metadata in ordinary agent runs and has origin-based reply routing, but the slash-command system-prompt reconstruction used by `/context` and `/export-session` still omits the available channel/capabilities/shell runtime metadata. The related PR #21271 targeted this exact gap but is closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep #20837 open and scope the implementation to the remaining slash-command gap: update `src/auto-reply/reply/commands-system-prompt.ts` to mirror the embedded-runner runtime metadata path for machine/shell details, normalized channel, account-aware capabilities, message-tool capabilities, reaction guidance, and channel actions. Add focused tests proving `/context` or `/export-session` from Telegram rebuilds a prompt containing `channel=telegram` and expected capabilities. Treat more advanced prompt-driven selective reply routing as a follow-up design item, since ordinary origin-based routing already exists.
