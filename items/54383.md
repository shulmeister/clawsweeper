## Summary

Keep this issue open. Current main still forwards compaction reserveTokens into the upstream summarizer without clamping against the selected model output limit, and the provided GitHub context shows an open same-author PR #54392 with closing syntax for this issue.

## Best Possible Solution

Review and either land PR #54392 or replace it with an equivalent maintainer-approved fix that clamps compaction summarization output budgeting against the selected model's output limit before calling pi-coding-agent generateSummary, with regression coverage for direct Anthropic Claude 4.6 1M-context compaction. After that lands, close this issue as implemented.
