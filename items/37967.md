## Summary

Keep #37967 open. Current main still has Feishu per-sender/group-topic session scoping and manual subagent/thread-binding hooks, but it does not implement automatic Feishu group-chat per-user subAgents with isolated agent configuration, skills, memory, or admin hot configuration. The existing dynamic agent creation path is explicitly DM-only.

## Best Possible Solution

Keep the issue open for maintainer/product direction or implementation in the Feishu plugin. A complete fix should add an explicit opt-in group-chat mode that maps each sender or role to an isolated agent/subagent profile, provisions workspace/agentDir/skills/model/config/memory safely, routes replies back to the originating Feishu group/thread, provides bounded admin controls, and adds regression tests plus Feishu docs. If maintainers want this outside the bundled Feishu plugin, first define the missing plugin/API contract clearly rather than closing the request as already satisfied.
