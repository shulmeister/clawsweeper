## Summary

Current `main` already injects agent/session/channel context into bundled CLI agent child environments via `OPENCLAW_MCP_*` variables, so spawned shell children can attribute their origin without manual tagging.

## Evidence

- **CLI prep injects session/channel/agent env:** `prepareCliRunContext` populates `OPENCLAW_MCP_AGENT_ID`, `OPENCLAW_MCP_ACCOUNT_ID`, `OPENCLAW_MCP_SESSION_KEY`, and `OPENCLAW_MCP_MESSAGE_CHANNEL` for the agent backend environment.
  - file: [src/agents/cli-runner/prepare.ts:182](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/cli-runner/prepare.ts#L182)
  - command: `nl -ba src/agents/cli-runner/prepare.ts | sed -n '180,196p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **CLI execution forwards injected env to child process:** `executePreparedCliRun` merges `context.preparedBackend.env` into the spawned child `env`, so the CLI process and anything it spawns inherit those variables.
  - file: [src/agents/cli-runner/execute.ts:278](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/cli-runner/execute.ts#L278)

  - file: [src/gateway/mcp-http.loopback-runtime.ts:30](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/mcp-http.loopback-runtime.ts#L30)

## Risks / Open Questions

- The shipped variable names are `OPENCLAW_MCP_*`, not the exact `OC_*` names suggested in the issue.
- I did not find a separate injected channel display-name variable analogous to the proposed `OC_CHANNEL_NAME`.
