## Summary

Keep open. The issue is recent and concrete, and current main does not appear to implement the requested robust Moonshot/Kimi `reasoning_content` preservation or restoration after LCM compaction or cross-model fallback. Main has the earlier tool-call ID fix and optional `thinking.keep="all"` forwarding, but no Moonshot replay sanitizer or outgoing payload patch equivalent to the DeepSeek/Venice placeholder handling.

## Best Possible Solution

Keep the issue open. Implement a focused Moonshot replay fix in the bundled provider path, likely via provider-owned `sanitizeReplayHistory` and/or a Moonshot payload patch that preserves existing top-level OpenAI-style `reasoning_content` and safely handles missing values on assistant tool-call turns during Kimi thinking sessions. Add regression coverage for same-model long replay, LCM/context-engine compaction output, and cross-model fallback into Kimi, and coordinate the preservation side with Martian-Engineering/lossless-claw#488.
