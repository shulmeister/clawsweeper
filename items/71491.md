## Summary

Keep open. Current main still does not have a Moonshot/Kimi replay repair for missing top-level `reasoning_content` after context-engine/LCM assembly or cross-model fallback. The issue is fresh, concrete, and distinct from the already-merged native Kimi tool-call ID fix.

## Best Possible Solution

Keep this issue open. Add a Moonshot-owned replay/payload repair, likely in the Moonshot thinking stream wrapper or a provider replay hook that also runs after context-engine assembly, to preserve converted `reasoning_content` when present and synthesize a safe placeholder for assistant tool-call turns when Kimi thinking replay requires it. Cover same-provider long-history/context-engine assembly and cross-model fallback with focused tests, using the existing DeepSeek/Venice replay repair patterns as references.

## Evidence
