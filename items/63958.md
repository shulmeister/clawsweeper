## Summary

PR #63958 is superseded by merged/shipped QA packaging fixes. Current main handles the missing QA scenario pack without crashing completion-cache generation, and the PR's `qa/` packaging change conflicts with the current private QA packaging policy.

## Best Possible Solution

Any future public QA scenario-pack distribution should be opened as a new maintainer-owned product decision rather than reviving this stale PR.
