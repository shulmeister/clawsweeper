## Summary

Keep open. Current main still does not implement the requested `compaction.memoryFlush.model` override: the public config schema rejects unknown `memoryFlush` keys, generated help/docs list only the existing memoryFlush fields, and the runtime still resolves the memory flush run from the session provider/model. The linked feature issue #66802 remains a valid product request, and PR #66846 appears to be focused work toward that request rather than obsolete or superseded.

## Best Possible Solution

Keep PR #66846 open for maintainer review or reimplementation. The best path is to add a public `agents.defaults.compaction.memoryFlush.model` config field across the type, strict zod schema, generated schema/help/labels/baseline, and docs; resolve the effective flush provider/model before context-window threshold and CLI eligibility checks; pass that target into `runWithModelFallback`; and keep focused tests for provider/model overrides, model-only overrides, blank fallback behavior, and threshold calculation against the resolved flush model.
