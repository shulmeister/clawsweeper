## Summary

Current main still has the reported schema bug: `GroupChatSchema.historyLimit` uses `.positive()` while sibling history-limit schemas and docs allow `0`. There is also an open same-author PR, #65359, that references this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land #65359 or an equivalent patch that changes `GroupChatSchema.historyLimit` from `.positive()` to `.min(0)` and adds regression coverage for `0`, positive, and negative values. After that lands, GitHub can close the issue as implemented.
