## Summary

Close as `clawhub`: the proposal is a useful external ecosystem idea, but current `main` positions optional capability in plugins/ClawHub and explicitly avoids heavy orchestration, agent-hierarchy frameworks, and non-provider commercial integrations in OpenClaw core. Current docs also show core already covers local sub-agent and agent-to-agent delegation, so this RFC is a layer above existing core scope rather than a missing core primitive.

## Close Comment

Closing from a Codex review. The idea is useful, but current project scope puts this in external plugin/ClawHub territory rather than OpenClaw core: `VISION.md:54-60` says optional capability should usually ship as plugins, `VISION.md:66-70` says new skills should go to ClawHub first, and `VISION.md:99-107` explicitly avoids heavy orchestration layers, agent-hierarchy frameworks, and non-provider commercial integrations in core. Current `main` already covers local delegation primitives with `tools.agentToAgent` and sub-agents (`docs/gateway/configuration-reference.md:2429-2465`, `docs/tools/subagents.md:10-18`), so this RFC's inter-instance discovery/negotiation/payment/marketplace layer is better treated as an external ClawHub/plugin experiment. If there is a future need for a small core seam to support that kind of plugin cleanly, please open a narrower issue focused on that seam.
