## Summary

Keep #70821 open. Current main still does not expose `xhigh` in the `amazon-bedrock` provider thinking profile for Claude Opus 4.7, even though the downstream Anthropic transport already maps Opus 4.7 `xhigh` correctly. The PR is a focused, still-actionable plugin-profile fix with an appropriate targeted unit test; the related closed Opus 4.7 report covers the Anthropic provider/transport path, not this Bedrock advertisement gap.

## Best Possible Solution

Keep the PR open and land this fix or an equivalent rebased patch: add an Opus 4.7-specific Bedrock profile branch that advertises `{ id: "xhigh" }`, keep the existing Claude 4.6 `adaptive` default unchanged, and add the focused `extensions/amazon-bedrock/index.test.ts` coverage for Opus 4.7 positive and 4.6/Sonnet negative cases. No core API change appears necessary.
