## Summary

PR #65188 should close as superseded. The underlying iOS Safari input-zoom bug is still real on current main, but it is already tracked by open issue #64651 and a narrower open PR #64673. This PR is also not a clean vehicle because it omits the active agent chat textarea and bundles unrelated #47887 iOS LAN/auth changes with a review-flagged Swift test compile issue.

## Best Possible Solution

Close #65188 as a superseded duplicate PR. Keep the actual Control UI iOS Safari input-zoom work on #64651 and continue through the narrower #64673 or a clean replacement that covers chat, agent-chat, and form fields. Keep the unrelated iOS LAN/auth #47887 work in its own issue or PR.
