## Summary

Current main already covers the PR’s user-facing goal through the shipped shared queue controls plus the Telegram plugin’s fragment coalescing path. Telegram can use the generic per-channel `interrupt` queue mode to abort an active session run and run the newest inbound message, and near-limit `channel_post` fragments are already coalesced and covered by regression tests.

## Best Possible Solution

Close this PR rather than merge it. Keep the shipped shared queue `interrupt` behavior and Telegram plugin coalescing implementation; if a dedicated Telegram per-account supersede alias is still desired, track that as a new narrow config/API design item against the current plugin-based Telegram code.
