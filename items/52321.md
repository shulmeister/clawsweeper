## Summary

Keep open. Current `main` still does not implement non-interactive token input for `openclaw models auth paste-token`: the CLI only exposes `--provider`, `--profile-id`, and `--expires-in`, and the command unconditionally prompts via `clackText(...)`. The latest release `v2026.4.22` shows the same behavior, so there is no strong evidence this request was already shipped or obsoleted.

## Evidence

- **CLI surface lacks non-interactive flags:** `paste-token` registers only `--provider`, `--profile-id`, and `--expires-in`; there is no `--token`, `--stdin`, or env option path on current `main`. See `src/cli/models-cli.ts:339` through `src/cli/models-cli.ts:355`.
