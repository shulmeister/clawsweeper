## Summary

Keep this PR open. Current main still returns no results when memory search has no embedding provider and FTS is unavailable, so the central fallback request is not implemented on main. The PR contains useful memory-core work, but it is stale against current main and needs rebase/narrowing plus verification for the provider-backed vector-only regressions flagged in review.

## Best Possible Solution

Keep this PR open as a useful but not-ready implementation candidate. The best path is to rebase onto current main, keep the diff focused on `extensions/memory-core` fallback behavior and tests, add or restore regression coverage for FTS-unavailable chunk search, and explicitly preserve provider-backed vector-only and hybrid-disabled semantics before merge.
