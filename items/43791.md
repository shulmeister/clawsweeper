## Summary

Keep PR #43791 open. Current main has a narrower queued-tool-media path, but it does not implement this PR's broader embedded-run contract for preserving full tool-result ReplyPayloads in final run payloads. The PR branch also addressed several review concerns in discussion, so it still appears to contain useful work rather than being obsolete.

## Best Possible Solution

Keep the PR open and review it as a focused embedded-run contract fix. The best path is to reconcile its full ReplyPayload buffering with current main's narrower pending-tool-media implementation, preserve channelData/mediaUrl/mediaUrls/text safely, avoid replaying already-streamed results, reset buffers across compaction retries, and settle timeout behavior with targeted regression tests.
