## Summary

Keep #11517 open. Current main still has the import-heavy Pi attempt runner and Discord plugin monitor files the issue names, and the merged harness/runtimePlan work is an adjacent policy/harness seam rather than the requested RunContext/AgentContext dependency aggregation. The recent issue discussion also already reached the same keep-open conclusion.

## Best Possible Solution

Keep #11517 open as the canonical tracker. The best path is an incremental refactor that introduces a prepared RunContext/AgentContext or smaller service bundles for the Pi attempt runner and Discord plugin monitor, preserves plugin SDK boundaries, separates pure moves from behavior changes, and adds focused tests around each extracted seam.
