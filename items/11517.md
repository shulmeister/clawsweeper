## Summary

Keep open. Current `main` shows partial architectural cleanup on Discord by moving monitor code under `extensions/discord` and depending on `openclaw/plugin-sdk/*`, but the issue is not implemented well enough overall because `src/agents/pi-embedded-runner/run/attempt.ts` is still a very large orchestrator with broad direct subsystem coupling, and the Discord monitor entrypoints are still large import-heavy modules.
