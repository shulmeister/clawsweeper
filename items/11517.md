## Summary

Keep open. Current main still has the import-heavy Pi embedded attempt runner and Discord monitor files named in the issue. Recent harness/context-engine seam work is real and useful, but it is partial: it adds backend selection and plugin harness seams, not the broader RunContext/AgentContext-style dependency aggregation or import-coupling reduction requested here.

## Best Possible Solution

Keep #11517 open as the canonical tracker for reducing import coupling in the Pi attempt runner and Discord monitor. The best path is a maintainer-approved refactor that introduces a real prepared RunContext/AgentContext or narrower service bundles, moves remaining reusable harness/context-engine helpers out of PI-owned modules where appropriate, preserves plugin SDK boundaries, and adds focused tests around the extracted seams.
