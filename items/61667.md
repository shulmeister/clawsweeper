## Summary

Keep open. Current main has partial allow-always persistence for some executable paths, but the reported dynamic-command problem still remains for the issue’s Python/Windows-style case: interpreter-like commands do not get a reusable allow-always pattern by default, and the code falls back to exact command hashes when no pattern is derived. The related PR #61687 is still open and not merged.

## Evidence

- **Interpreter-like commands are not persisted by default:** `collectAllowAlwaysPatterns` returns without adding a reusable pattern for interpreter-like executables unless strict inline-eval mode is enabled and the command is not inline eval. The documented default for `tools.exec.strictInlineEval` is `false`, so a normal `python email_sender.py ...` approval still does not become broad executable/script trust by default.
