## Summary

Current `main` contains several targeted hardenings that match the reported symptoms: Codex CLI bootstrap is now `bootstrapOnly` so local `openai-codex` OAuth is kept canonical, Codex native compaction now fails closed on bound-vs-runtime auth-profile mismatches, QA coverage explicitly treats leaked Codex coordination text as a failure, and embedded-session truncation documents/fixes unbounded post-compaction growth. But this review did not produce a high-confidence end-to-end proof that switching `openai-codex/gpt-5.4` to `codex/gpt-5.4` no longer reproduces the exact runaway-context/off-task failure, and the issue is recent with a corroborating second report. Conservative maintainer cleanup should leave it open.
