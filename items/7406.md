## Summary

Keep #7406 open. Current main still has only partial Telegram forum topic-name support: Telegram can cache and expose `TopicName` in inbound prompt/hook metadata, but that value is not persisted into session `label`/`displayName`, not used by `sessions.list`, and not consumed by the Control UI session dropdown. The related implementation PRs #7407 and #43105 are closed and unmerged in the provided GitHub context, and the issue has a same-day maintainer-side keep-open comment plus a follow-up request.

## Best Possible Solution

Keep #7406 as the canonical tracker. The best fix is to map cached Telegram forum `TopicName` into a persisted session display field used by `sessions.list` and the Control UI selector, likely by setting `ThreadLabel` or a dedicated session metadata field during inbound session metadata recording. Add regression coverage for Telegram context extraction, session-store metadata, gateway row output, and Control UI label resolution, with a topic-id/key fallback only when no human-readable topic name is known.
