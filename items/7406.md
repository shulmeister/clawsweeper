## Summary

Keep #7406 open. Current main captures Telegram forum topic names into Telegram inbound context and prompt metadata, but the persisted session metadata, gateway session rows, and Control UI session-display path still have no topic-name field or label resolution. The related implementation PRs are closed unmerged, and the duplicate reports in the provided context were closed back toward #7406 as the canonical tracker.

## Best Possible Solution

Keep #7406 open as the canonical tracker. Implement this in the bundled Telegram/session metadata path by carrying the cached Telegram TopicName into persisted session display metadata, exposing it through sessions.list, and letting the Control UI dropdown resolve it while preserving explicit user labels/displayName and falling back to topic id/session key when unknown. Add regression coverage for topic extraction, metadata persistence, gateway row generation, and Control UI label resolution.
