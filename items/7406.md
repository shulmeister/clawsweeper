## Summary

Keep #7406 open. Current main has useful Telegram topic-name cache/context plumbing, but it still does not map cached `TopicName` into persisted session `label`/`displayName` or the Control UI session dropdown label path. The prior maintainer/Codex discussion also explicitly kept this issue open, and the related implementation PRs are closed unmerged.

## Best Possible Solution

Keep #7406 as the canonical tracker. The best fix is to persist the cached Telegram `TopicName` into the session display path, likely as a thread label or display-name component during inbound session metadata recording, then add regression coverage that a Telegram forum topic session row and Control UI dropdown label include the human-readable topic name.
