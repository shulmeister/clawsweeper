## Summary

Keep #7406 open. Current main captures and persists Telegram forum topic names into inbound context, but the topic name is not mapped into the persisted session label/displayName or the Control UI session dropdown display path.

## Best Possible Solution

Keep #7406 as the canonical tracker. The best fix is to map cached Telegram forum TopicName into a persisted session display field used by sessions.list and the Control UI dropdown, likely through ThreadLabel/displayName during inbound session metadata recording, with regression coverage for Telegram context, session metadata/gateway rows, and the Control UI label resolver. Include a topic-id fallback only when no human-readable topic name is known.
