## Summary

Keep #7406 open. Current main has partial Telegram topic-name plumbing for inbound prompt context, but it still does not map the known topic name into persisted session display metadata or the Control UI session dropdown path.

## Best Possible Solution

Keep #7406 open as the canonical tracker. The best fix is to map known Telegram forum `TopicName` into a persisted session display field used by `sessions.list` and the Control UI dropdown, likely by setting a generic thread/session label during Telegram inbound session metadata recording. Add regression coverage from Telegram context through session metadata/gateway row creation and the UI display resolver, with a topic-id fallback only when no human-readable topic name is known.
