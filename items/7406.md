## Summary

Keep #7406 open. Current main has useful Telegram topic-name capture/cache plumbing, matching parts of the prior PR discussion, but the requested session dropdown/display-name behavior is still not wired: session metadata display names are built from the Telegram group subject/channel/id and the UI dropdown uses only label/displayName/fallback, not the cached TopicName.
