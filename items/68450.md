## Summary

Keep open. Current main has partial MCP runtime lifecycle cleanup, including isolated cron opt-in cleanup, run-end embedded cleanup, idle TTL reaping, gateway shutdown cleanup, and session rollover cleanup. It does not fully implement this PR's stated scope: isolated heartbeat runs still create fresh sessions without passing a bundle-MCP cleanup flag, and queued follow-up run types/options still lack the one-shot cleanup plumbing proposed by the PR.

## Best Possible Solution

Keep the PR open, but ask for a rebase/narrowing against current main. The best fix is to retain current main's session runtime manager, TTL backstop, cron cleanup, gateway shutdown cleanup, and session-rollover cleanup, then add the missing deterministic one-shot cleanup for isolated heartbeat and queued follow-up/outer fallback boundaries with focused tests for heartbeat, follow-up cleanup, compaction/session-id rotation, and cleanup-import failure teardown ordering.
