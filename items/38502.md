## Summary

Keep open. Current main still shows the old Ctrl+C status text without an active-run hint, while active-run abort remains a separate Esc//abort path. The PR is a narrow one-file UX clarification and its current patch addresses the earlier review concern by advertising /abort only, not unconditional Esc.

## Best Possible Solution

Keep this PR open or land an equivalent narrow TUI change on main: append an active-run hint to the Ctrl+C clear/warn statuses when activeChatRunId is set, prefer /abort as the universal stop instruction, and add focused coverage if the status surface is testable.
