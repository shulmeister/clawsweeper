## Summary

Keep open. Current main at 7e13f3f5140168e31027b39a9855ade3fadf24d5 still lacks the active-run Ctrl+C hint, while Ctrl+C remains a local clear/warn/exit flow and `/abort` remains the actual active-run stop path. The PR is still a narrow TUI UX fix, and the current head addresses the earlier review concern by advertising `/abort` only instead of promising unconditional Escape behavior.

## Best Possible Solution

Keep this PR open or land an equivalent narrow change on main: append a `run still active (/abort to stop)` hint to the Ctrl+C clear/warn statuses when `activeChatRunId` is set, leave abort semantics unchanged, and add focused status-text coverage if the TUI status surface is testable.
