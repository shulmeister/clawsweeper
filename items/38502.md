## Summary

Current main still lacks the PR's active-run Ctrl+C hint, and the linked bug remains open. The PR is a narrow TUI UX fix rather than obsolete cleanup, with the later patch also addressing the review concern by avoiding an unconditional Esc hint.

## Best Possible Solution

Keep the PR open and either land or rework its narrow `src/tui/tui.ts` change: append an active-run hint only in the Ctrl+C clear/warn status branches, keep Ctrl+C behavior unchanged, prefer `/abort` wording because Esc can first dismiss visible BTW content, and add focused coverage if maintainers want regression proof for the displayed status.
