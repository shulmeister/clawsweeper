## Summary

Keep open. Current main still lacks the active-run Ctrl+C status hint, while Ctrl+C remains a local clear/warn/exit flow and `/abort` remains the explicit active-run abort path. The PR is still a narrow TUI UX fix, and its current diff addresses the prior review concern by advertising `/abort` only instead of unconditional Esc behavior.

## Best Possible Solution

Keep this PR open or land an equivalent narrow change on main: append `; run still active (/abort to stop)` to the Ctrl+C clear/warn statuses in `src/tui/tui.ts` only when `activeChatRunId` is set, leave abort/keybinding semantics unchanged, and add focused status-text coverage if the TUI status surface is testable. The linked issue #38501 remains the right user-problem anchor.
