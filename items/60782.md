## Summary

Current main still has the same user-visible failure path: the Web UI calls `update.run`, then surfaces `result.reason` directly, and the updater still uses `global update (omit optional)` as the failed fallback step name. The reporter's validation-theory explanation is not supported by the current schema because all `update.run` fields are optional, but the observable poor error message remains actionable.
