## Summary

Keep #12590 open. Current main still contains the reported counter synchronization path, and the tests still encode the synchronized `compactionCount` / `memoryFlushCompactionCount` state that causes the next eligible memory flush to be skipped.

## Best Possible Solution

Keep #12590 open. Fix `src/auto-reply/reply/agent-runner-memory.ts` so a flush-triggered compaction does not persist the post-increment `compactionCount` as `memoryFlushCompactionCount`; preserve the pre-increment marker or replace the equality marker with clearer state. Update the synchronized-counter expectation in `agent-runner-memory.test.ts` and add regression coverage proving consecutive eligible auto-compaction cycles run `memoryFlush` each time. PR #51421 remains the current open implementation reference.
