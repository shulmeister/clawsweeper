## Summary

Keep open. Current main still does not add the requested `moonshotai/kimi-k2.5` OpenRouter row to the onboarding/model-picker catalog. OpenRouter can resolve arbitrary model refs dynamically, so manual use may work, but the selector/catalog request remains unsatisfied unless maintainers decide K2.6 intentionally supersedes K2.5.

## Best Possible Solution

Keep the issue open until maintainers either add `moonshotai/kimi-k2.5` to the OpenRouter static catalog/docs/tests, or explicitly decide that K2.6 supersedes it and document/comment that direction. Runtime dynamic-model support and manual entry can be mentioned as a workaround, but they do not fully address the selector request.
