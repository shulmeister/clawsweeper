## Summary

Keep #14601 open. Current main still does not add `openrouter/moonshotai/kimi-k2.5` to the bundled OpenRouter onboarding/model-selector catalog. The OpenRouter plugin exposes `openrouter/auto`, Hunter/Healer Alpha, and `moonshotai/kimi-k2.6`; manual/dynamic OpenRouter refs remain only a workaround, not the requested selector row.

## Best Possible Solution

Keep the issue open. Either add `moonshotai/kimi-k2.5` to `extensions/openrouter/provider-catalog.ts` with accurate capability, context, and cost metadata, update `docs/providers/openrouter.md`, and add focused OpenRouter catalog/onboarding coverage, or have a maintainer explicitly document/comment that K2.6 supersedes K2.5 for the OpenRouter onboard selector.
