## Summary

Keep #14601 open. Current main still does not add `openrouter/moonshotai/kimi-k2.5` as a bundled OpenRouter onboarding/model-picker catalog row. OpenRouter supports manual/dynamic model refs and K2.6 is bundled, but that does not satisfy the requested K2.5 selector entry, and the existing maintainer/Codex comment already reached the same keep-open conclusion.

## Best Possible Solution

Keep the issue open. Either add `moonshotai/kimi-k2.5` to `extensions/openrouter/provider-catalog.ts` with current OpenRouter capability/context/token/cost metadata, update `docs/providers/openrouter.md`, and add focused catalog/onboarding coverage; or have a maintainer explicitly document that K2.6 supersedes K2.5 for the OpenRouter onboard selector.
