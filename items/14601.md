## Summary

Keep open. Current `main` still does not add MoonshotAI Kimi K2.5 to the OpenRouter onboarding model selector. The OpenRouter bundled catalog only includes `auto`, `openrouter/hunter-alpha`, `openrouter/healer-alpha`, and `moonshotai/kimi-k2.6`, and the onboarding model picker builds its selectable options from that catalog.

## Evidence

- **OpenRouter bundled catalog omits K2.5:** `buildOpenrouterProvider()` returns four models: `auto`, `openrouter/hunter-alpha`, `openrouter/healer-alpha`, and `moonshotai/kimi-k2.6`. There is no `moonshotai/kimi-k2.5` entry.
  - file: [extensions/openrouter/provider-catalog.ts:36](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/extensions/openrouter/provider-catalog.ts#L36)
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **Onboarding picker is catalog-driven:** `promptDefaultModel()` loads the model catalog and turns each catalog entry into a selectable option. Models not present in the catalog do not appear in the searchable picker, aside from the separate manual-entry fallback.
  - file: [src/flows/model-picker.ts:431](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/flows/model-picker.ts#L431)
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
- **Current test coverage only proves K2.6:** The OpenRouter extension test on `main` asserts that the bundled catalog includes `moonshotai/kimi-k2.6`; there is no corresponding assertion or implementation for `moonshotai/kimi-k2.5`.
  - file: [extensions/openrouter/index.test.ts:8](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/extensions/openrouter/index.test.ts#L8)
  - sha: [272bd59e7a8a](https://github.com/openclaw/openclaw/commit/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e)
