## Summary

Keep open. Current main still does not add `moonshotai/kimi-k2.5` / `openrouter/moonshotai/kimi-k2.5` to the bundled OpenRouter onboarding/model-picker catalog. Runtime dynamic OpenRouter refs and manual model entry are available workarounds, but they do not satisfy the specific selector/catalog request. The existing issue discussion also already includes a Codex review comment keeping this open for the same reason, and current main has not changed that result.

## Best Possible Solution

Keep #14601 open until maintainers either add `moonshotai/kimi-k2.5` to the bundled OpenRouter provider catalog/docs/tests so it appears in onboarding/model selection, or explicitly decide and document/comment that K2.6 supersedes K2.5. Runtime dynamic-model support and manual entry are valid workarounds but not a complete fix for the selector request.
