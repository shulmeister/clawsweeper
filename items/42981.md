## Summary

Keep open. Current main only has a partial cron allowlist mechanism (`payload.toolsAllow`) and global/per-agent subagent policy. It does not implement the requested per-session `toolPolicy` object for `cron.add` or `sessions_spawn`, and there is no per-call deny-list support for either surface.

## Evidence

## Risks / Open Questions

- Cron isolated jobs already have a hard allowlist path via `payload.toolsAllow`, so the issue is partially addressed. Closing would overstate coverage because deny lists and a first-class `toolPolicy` object are still missing.
- Subagent sessions already have global/per-agent restrictions via `tools.subagents.tools` and depth-based policy, but there is still no per-spawn/session override on `sessions_spawn`.
