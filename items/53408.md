## Summary

Keep this issue open. Current main has targeted repairs for Kimi Anthropic Messages and OpenAI-completions malformed tool-call arguments, but the reported custom/local `copilot-api` Anthropic Messages path is still outside the repair gate and still uses the plain streamed `input_json_delta` parsing path before tool validation.

## Best Possible Solution

Keep this issue open. Add a focused regression or live repro for a custom/local Anthropic Messages provider streaming large fragmented `input_json_delta` tool arguments after long context. If it reproduces, either extend argument repair or diagnostics to the custom Anthropic proxy path, or fail explicitly before missing/empty args reach tool validation. Treat #53591 and #53735 as related Kimi-specific fixes, not the canonical fix for this report.
