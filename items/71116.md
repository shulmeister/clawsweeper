## Summary

Keep #71116 open. Current main still lacks the requested secrets/credential governance documentation, and the linked #71156 is an open, unmerged docs PR that appears to cover only part of the issue. This is a concrete repo documentation/SDK-governance request, not a ClawHub-only plugin idea or stale low-information report.

## Best Possible Solution

Keep the issue open while maintainers decide scope. The best path is to review or revise PR #71156 for the `src/gateway/server-methods` secrets guidance, then add or explicitly defer a user-facing plugin credential-persistence contract in the plugin SDK docs. If the optional CI token audit or shared credential helper should proceed, split those into narrower maintainer-owned follow-up issues rather than closing this umbrella prematurely.
