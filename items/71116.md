## Summary

Keep this issue open. Current main has SecretRef runtime behavior, SDK atomic JSON/file-lock helpers, and tests for secrets reload/resolve behavior, but it still lacks the requested secrets hotspot guidance in `src/gateway/server-methods/AGENTS.md` and a central plugin credential-persistence contract. The supplied GitHub context also shows the same author has paired PR #71156 open and unmerged, so the maintainer path is to review, revise, land, or close that PR rather than auto-close this issue.

## Best Possible Solution

Keep this issue open until maintainers either merge #71156 or land an equivalent focused documentation change. The best implementation is to add secrets-surface guidance to `src/gateway/server-methods/AGENTS.md` and add or cross-link public plugin docs that define the credential-persistence contract: when to use SecretRef-backed config versus plugin-local storage, how to combine file locks with atomic writes, lifecycle/TTL expectations, and failure behavior that must not claim success before persistence or revocation guarantees hold.
