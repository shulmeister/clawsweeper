## Summary

Keep open. The report is recent and describes a real Gateway/Docker restart loop on v2026.4.24, but the provided logs stop at normal startup markers and do not include the fatal error, container exit status, health result, stability bundle, or a signature tying it to an already-fixed path. Current main has plausible related unreleased fixes for Bonjour crash loops and Control UI startup/origin handling, but the evidence is not strong enough to close #71602 as implemented, duplicate, or cannot-reproduce.

## Best Possible Solution

Keep #71602 open and ask for the full Docker/Gateway evidence around one restart: `docker compose ps`, container exit code/reason, complete logs before and after `starting HTTP server...`, `/healthz` and `/readyz` status, the latest Gateway stability bundle/export, redacted gateway config including bind/auth/controlUi origins, and whether disabling Bonjour (`OPENCLAW_DISABLE_BONJOUR=1` or `discovery.mdns.mode="off"`) stops the loop. If the added logs match the Bonjour ciao watchdog signature, close against the shipped/unreleased Bonjour fix; if they match non-loopback Control UI origin failure, close against the runtime bind/port origin seeding fix; otherwise investigate the implicated Gateway startup/post-attach path.
