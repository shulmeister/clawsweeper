## Summary

Current main has targeted post-v2026.4.24 Docker/Bonjour hardening that matches this restart-loop symptom: Docker Compose disables Bonjour by default on bridge networking, the Bonjour plugin now handles known @homebridge/ciao cancellation/assertion process errors, and stuck mDNS advertiser restarts disable Bonjour for the process instead of taking down the gateway. The v2026.4.24 release compose file did not include the Docker Bonjour disable default, so the reporter's release can still show the loop while main is fixed.

## Best Possible Solution

Keep the current main Docker/Bonjour hardening and ship it in the next release. Users on v2026.4.24 should upgrade once available or set OPENCLAW_DISABLE_BONJOUR=1 in Docker Compose; if the loop persists on a fixed build, collect full docker compose logs, restart count/exit code, and the latest gateway stability bundle for a new narrowly scoped bug.
