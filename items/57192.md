## Summary

Keep open. Current `main` still intentionally blocks `screen.record` by default on macOS, so the PR's stated behavior is not already implemented. The PR also appears incomplete as written: new-gateway wizard configs still seed `denyCommands` from `DEFAULT_DANGEROUS_NODE_COMMANDS`, which includes `screen.record`, so the proposed macOS default would be overridden for typical fresh setups. Its separate OpenAI WebSocket warm-up cleanup is partly obsolete on `main` because warm-up payload construction now already omits empty `tools` arrays in production code.

## Evidence

- **Current main still treats screen recording as dangerous:** `SCREEN_DANGEROUS_COMMANDS` still contains `"screen.record"`, and `DEFAULT_DANGEROUS_NODE_COMMANDS` is built from that list. The macOS platform defaults include `SCREEN_COMMANDS` only, which is `screen.snapshot`, not `screen.record`.
