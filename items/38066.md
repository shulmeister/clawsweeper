## Summary

Keep #38066 open. Current main still exposes `throttleMs` only as an internal `createTelegramDraftStream` parameter, not as Telegram channel/account config, and `bot-message-dispatch` still calls the draft stream without passing a configurable throttle. The related PR #38255 is the right active implementation path, but it is open and unmerged; PR #38226 was closed unmerged.

## Best Possible Solution

Keep #38066 open until PR #38255 or an equivalent maintainer-approved change lands. The fix should add Telegram `streamThrottleMs` and, if accepted, `minInitialChars` to the config type, strict schema, generated metadata, and public docs; pass the resolved top-level/per-account values through `extensions/telegram/src/bot-message-dispatch.ts` into `createTelegramDraftStream`; and cover default, clamp, fallback, and account override behavior with focused tests. Maintainers should also decide separately whether DM native draft transport should remain forced to message transport.
