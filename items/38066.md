## Summary

Keep #38066 open. Current main still does not expose a Telegram `streamThrottleMs` or `minInitialChars` config key, and the Telegram dispatcher still calls `createTelegramDraftStream` without a configurable `throttleMs`; the draft stream therefore falls back to `DEFAULT_THROTTLE_MS = 1000` whenever no parameter is provided. Related PR #38255 directly tracks the requested work but remains open and unmerged in the provided GitHub context.

## Best Possible Solution

Keep the issue open until PR #38255 or an equivalent maintainer-approved change adds the Telegram config surface, validates/docs it, wires the value through `bot-message-dispatch.ts` into `createTelegramDraftStream`, and covers fallback/default behavior with focused tests.
