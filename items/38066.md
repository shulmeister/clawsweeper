## Summary

Keep #38066 open. Current main still does not expose `channels.telegram.streamThrottleMs` or pass any configured throttle into Telegram draft streaming; `createTelegramDraftStream` still falls back to 1000ms when the caller omits `throttleMs`. The related implementation PR #38255 is open/unmerged, while earlier related PRs are closed unmerged, so the requested behavior is not shipped on main.

## Best Possible Solution

Keep #38066 open until PR #38255 or an equivalent maintainer-approved patch lands. The best implementation is to add validated Telegram account config for `streamThrottleMs` and, if accepted, `minInitialChars`; document the keys; pass the merged per-account values into `createTelegramDraftStream`; and cover default, configured, min-clamp, and DM preview transport behavior in focused tests.
