## Summary

Keep #38066 open. Current main still has the same missing user-visible config surface: Telegram draft streaming accepts an internal `throttleMs`, but Telegram channel/account config does not expose `streamThrottleMs` or `minInitialChars`, and dispatch still passes only a hardcoded `minInitialChars: 30` with no configurable throttle. The related PR #38255 remains the active open implementation path; PR #38226 was closed unmerged.

## Best Possible Solution

Keep #38066 open until PR #38255 or an equivalent maintainer-approved change lands. The fix should add Telegram preview cadence config to the exported config type, strict schema, generated metadata, and public docs; pass the resolved top-level/per-account values through `extensions/telegram/src/bot-message-dispatch.ts` into `createTelegramDraftStream`; and cover default, clamp/validation, fallback, and account override behavior with focused tests.
