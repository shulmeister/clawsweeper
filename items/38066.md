## Summary

Keep #38066 open. Current main still does not expose `channels.telegram.streamThrottleMs` or `minInitialChars` as Telegram config, schema, docs, metadata, or dispatcher wiring. `createTelegramDraftStream` has internal optional parameters, but the production dispatcher still supplies only the fixed `minInitialChars: 30` constant and no `throttleMs`, so callers fall back to the 1000ms default. The provided discussion shows PR #38255 is the active implementation path and remains open/unmerged, while the other related PRs are closed/unmerged and do not prove shipped behavior.

## Best Possible Solution

Keep #38066 open until PR #38255 or an equivalent maintainer-approved change lands. The fix should add documented top-level and per-account Telegram preview cadence settings, validate them in the strict config schema, update generated metadata/docs, merge account overrides through the existing Telegram account config path, pass the resolved values into `createTelegramDraftStream`, and cover default, validation, fallback, and account-override behavior with focused tests.
