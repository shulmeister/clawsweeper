## Summary

Current main intentionally does not provide a built-in bot-to-bot bridge in OpenClaw core, and the existing direct-send plus outbound hook/plugin seams point this Feishu multi-bot relay toward plugin/hook-pack scope rather than a core maintainer cleanup item.

## Close Comment

Codex review: closing this as plugin-scope rather than an OpenClaw core issue. Current main is explicit that OpenClaw does not ship a built-in bot-to-bot bridge (`docs/help/faq.md:932`), and the project vision keeps optional capability in plugins instead of core (`VISION.md:52`). The direct message-tool send path already goes through channel outbound adapters (`src/infra/outbound/message.ts:235`, `src/infra/outbound/deliver.ts:160`, `extensions/feishu/src/outbound.ts:152`), and outbound `message:sent` / `message_sent` hooks exist as the extension seam for custom relay behavior (`docs/automation/hooks.md:47`, `src/infra/outbound/deliver.ts:507`). For a Feishu-only multi-bot @mention relay, the supported direction is a community plugin or hook pack rather than a core OpenClaw change.
