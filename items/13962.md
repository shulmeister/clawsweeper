## Summary

Keep open. Current main only covers part of the requested context behavior via pending group-history injection. It does not implement per-mention/background model routing, does not process skipped unmentioned group messages through a background model, and does not expose per-group `mentionContextMessages`-style sizing.

## Best Possible Solution

Keep the issue open until OpenClaw implements mention-state-aware model routing and per-group mention context sizing, or exposes a clear pre-gating plugin/API path that lets a trusted plugin process unmentioned group messages with a background model before the main mention-triggered reply path.
