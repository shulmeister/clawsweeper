## Summary

Keep #13962 open. Current main still only implements pending context buffering for skipped WhatsApp group messages; it does not implement background-model processing for unmentioned messages, mention-state-aware model routing, or per-group mention context sizing.

## Best Possible Solution

Keep the issue open until OpenClaw implements mention-state-aware routing plus configurable per-group mention context sizing in the WhatsApp/group pipeline, or deliberately adds a trusted pre-gating plugin/API hook that lets unmentioned group messages be processed by a background model without bypassing group safety policy.
