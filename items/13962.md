## Summary

Keep open. Current main still only implements the group context half of the request partially: skipped unmentioned WhatsApp group messages are stored and injected into the next processed mention, but there is no per-mention/background model routing and no per-group `mentionContextMessages`-style context limit.

## Best Possible Solution

Keep the issue open until OpenClaw either implements mention-state-aware routing and per-group mention context sizing, or adds a clear plugin/API seam before group mention gating so a ClawHub plugin can process unmentioned group messages with a background model.
