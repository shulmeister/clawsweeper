## Summary

Keep #13962 open. Current main partially satisfies the context side by buffering skipped WhatsApp group messages and injecting them into the next triggered reply, and existing routing/model override features can target a whole group or chat. It still does not implement mention-state-aware model routing, background-model processing for unmentioned group chatter, or a per-group `mentionContextMessages`/context-window setting.

## Best Possible Solution

Keep the issue open until OpenClaw implements mention-state-aware routing and configurable per-group mention context sizing, or exposes a clear pre-gating plugin/API path that lets trusted plugins process unmentioned group messages with a background model. Likely touch points are WhatsApp group gating, route/model selection, WhatsApp config types/schema/docs, and tests for skipped unmentioned messages, triggered mentions, pending-history injection, and per-group sizing.
