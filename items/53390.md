## Summary

Keep open. Current `main` still treats `browser snapshot` as a structural UI-tree capture rather than a viewport-relative capture, and the implementation still forwards Playwright snapshot output without any scroll-position filtering/synchronization. I do not have a runnable `pnpm` environment here to do an end-to-end browser reproduction, so there is not strong enough evidence to close this as fixed or obsolete.

## Evidence

- **Docs describe snapshot as a stable UI tree:** The user-facing browser docs say `browser snapshot` returns "a stable UI tree (AI or ARIA)", which matches structural inspection rather than viewport pixels.
  - file: [docs/tools/browser.md:582](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/tools/browser.md#L582)

  - file: [docs/tools/browser-control.md:223](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/tools/browser-control.md#L223)
