## Summary

Keep open. Current main still routes browser text snapshots through page/root accessibility snapshot paths and Chrome MCP snapshot conversion without a viewport-scoped option or scroll-position filtering. The existing viewport-aware behavior is only for screenshot labels, not the text snapshot contract reported here.

## Best Possible Solution

Keep this issue open. Maintainers should decide the intended `browser snapshot` contract: either implement a viewport-aware text snapshot mode across Playwright, CDP, and Chrome MCP paths, or document that text snapshots are page/root accessibility trees and add a clear viewport-specific text alternative. The implementation should include a regression test with a long page, a scroll via evaluate or End key, and a follow-up snapshot that proves post-scroll visible content is represented when viewport mode is requested.
