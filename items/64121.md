## Summary

Current main still uses one combined chat session picker grouped by agent. The requested separate Agent and Agent Session controls are not implemented in the desktop header or mobile chat controls, and the related picker-cleanup context only covers narrower hidden/internal-session noise rather than this UI split.

## Best Possible Solution

Keep this issue open and implement the Control UI split directly: add an Agent selector sourced from `agentsList`, add an Agent Session selector filtered from `sessionsResult` by `parseAgentSessionKey(...).agentId`, switch agents to the most recent or main session via `buildAgentMainSessionKey`, and keep Model/Thinking patching the active `state.sessionKey`. Update both desktop `renderChatSessionSelect` and mobile `renderChatMobileToggle`, with tests for filtering, agent switching, active-session fallback, and the existing Model/Thinking session-bound behavior.
