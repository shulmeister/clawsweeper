## Summary

Keep this issue open. Current main still renders the Control UI Sessions table Updated column with `formatRelativeTimestamp`, so it shows values like `5m ago`/`3h ago` rather than the requested full `YYYY-MM-DD HH:mm` date-time. The provided GitHub context also shows open PR #53445 is an implementation candidate with closing syntax for this issue, so maintainers should review, land, or close that PR before this issue is closed.

## Best Possible Solution

Keep this issue open while #53445 remains open. Maintainers should review and land that PR if acceptable, or reimplement the same narrow Control UI change on main: add a tested full local date-time formatter, render the Sessions Updated column as `YYYY-MM-DD HH:mm`, and optionally keep relative time as hover context.
