## Summary

Keep PR #40052 open. Current main still routes default-agent primary model and fallback edits through `agents.list[index].model`, while the documented default model surface is `agents.defaults.model`. The PR addresses a real bug, but review discussion identified a stale default-agent override cleanup gap that should be handled before landing unchanged.

## Best Possible Solution

Keep this open or replace it with a maintainer-owned patch that routes default-agent primary and fallback edits to `agents.defaults.model`, keeps non-default edits on `agents.list[].model`, removes or migrates stale default-agent `agents.list[].model` overrides when saving defaults, and adds regression coverage for default-agent, non-default-agent, and `defaultId === null` path resolution.
