## Summary

Keep open. Current main still does not strip base64 image payloads from toolResult content at session JSONL write time; it only caps text/details at persistence time and prunes or sanitizes image blocks later in replay/context paths. The related #50914 work tracks the same unresolved design area, but it is not a shipped superseding fix.

## Best Possible Solution

Keep #68560 open and review it together with #50914. The best core direction is a replay-safe persistence/transcript hygiene path that removes large image payloads only after the model has consumed them, preserves MEDIA/path metadata, and has regression coverage through guardSessionManager/session persistence rather than a copied helper.
