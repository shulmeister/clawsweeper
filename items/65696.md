## Summary

Close as implemented on current main. The agent gateway handler now registers an abort controller, threads its signal into the ingress run, cleans up on both normal completion and pre-dispatch failure, preserves reconnect authorization via ownerDeviceId, and has regression tests covering the cases raised in this PR discussion.

## Evidence

- **Agent runs now register abort state with device ownership:** Current main registers agent RPC runs in `chatAbortControllers` before the accepted ack, including `ownerConnId`, `ownerDeviceId`, timeout-derived expiry, and `kind: "agent"` so `chat.abort` can target the run.

  - sha: [6d271762ab50](https://github.com/openclaw/openclaw/commit/6d271762ab50fb4376b8859bcf55eefb2d3814ea)
- **Abort signal is threaded into the agent run and completion cleanup is guarded:** `dispatchAgentRunFromGateway` now takes the controller, leaves the matching map entry only for that controller instance, and the agent ingress call receives `abortSignal`. This avoids clobbering a same-runId chat entry during cleanup.
  - file: [src/gateway/server-methods/agent.ts:231](https://github.com/openclaw/openclaw/blob/6d271762ab50fb4376b8859bcf55eefb2d3814ea/src/gateway/server-methods/agent.ts#L231)
