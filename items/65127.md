## Summary

The request is useful, but current main scopes the shared `message` surface to chat channels and delegates channel-specific actions to plugins; mailbox search for Gmail/Hotmail/IMAP fits the plugin/ClawHub path rather than OpenClaw core.

## Evidence

- **Vision: optional capability should ship as plugins, with promotion in ClawHub:** `VISION.md` says core should stay lean, optional capability should usually ship as plugins, and plugin promotion belongs in ClawHub rather than core.

  - file: [docs/cli/message.md:11](https://github.com/openclaw/openclaw/blob/713807b55d0beb17c0b96d0490a8abae695bb960/docs/cli/message.md#L11)
