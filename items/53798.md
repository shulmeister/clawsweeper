## Summary

Keep open. Current main still routes Feishu `.opus`/`.ogg` media from the message tool as native Feishu audio, but the Feishu upload call still omits the `duration` field that the plugin's own upload helper supports and documents as required for audio/video uploads. The prior review comment remains accurate on current main; adjacent Feishu media fixes cover routing/transcoding, not duration metadata.

## Best Possible Solution

Keep this open and fix it in the bundled Feishu plugin media path. Compute a finite media duration for native `.opus`/`.ogg` sends and for `audioAsVoice` transcode outputs, convert to Feishu's expected upload units, pass it to `uploadFileFeishu`, and add focused regression tests that `im.file.create` receives `duration` for `.opus` media and voice-intent transcodes. Consider covering video too because the helper marks audio/video duration as required.
