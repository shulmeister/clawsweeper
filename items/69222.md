## Summary

Keep #69222 open. Current main has a narrower Ollama web-search update that uses `/api/web_search` and honors `https://ollama.com` base URLs, but it does not implement this PR’s 404 compatibility chain, legacy endpoint retry, env-only Ollama Cloud fallback, or tightened local/cloud auth scoping. The linked bug #69132 is still open in the provided context, while the related PR #69178 was closed unmerged.

## Best Possible Solution

Keep #69222 open and ask for a rebase/update against current main. The best path is a focused patch that preserves main’s current `/api/web_search` and `https://ollama.com` base URL behavior, adds the missing 404 fallback chain and cloud retry for #69132, scopes local/provider-config auth separately from env-sourced Ollama Cloud auth, and includes regression tests for local 404 fallback, cloud fallback, no-token guidance, and the mixed-auth review case.
