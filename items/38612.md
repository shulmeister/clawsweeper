## Summary

Current main already ships the underlying capability through the newer memory-embedding adapter contract: `memorySearch.provider` is a string, providers resolve against registered adapter ids owned by the active memory plugin or declared plugin contracts, and unknown provider ids fail explicitly. This PR's plugin-slot validation approach is superseded by that shipped design.

## Evidence

- **Schema already accepts string providers:** `MemorySearchSchema` now uses `provider: z.string().optional()`, so `agents.defaults.memorySearch.provider` is no longer enum-limited on main.
  - file: [src/config/zod-schema.agent-runtime.ts:661](https://github.com/openclaw/openclaw/blob/ac063568d354c14e0a0a43bcffb9725bb3fca651/src/config/zod-schema.agent-runtime.ts#L661)

  - file: [docs/reference/memory-config.md:40](https://github.com/openclaw/openclaw/blob/ac063568d354c14e0a0a43bcffb9725bb3fca651/docs/reference/memory-config.md#L40)

## Risks / Open Questions

- Current main separates memory plugin selection (`plugins.slots.memory`) from memory embedding provider ids; if the PR author specifically wanted the plugin id itself in `memorySearch.provider`, that exact syntax is superseded rather than merged verbatim.

## Close Comment

Closing as implemented on `main` after a Codex review. Current OpenClaw already accepts `agents.defaults.memorySearch.provider` as a string (`src/config/zod-schema.agent-runtime.ts:661`), and the shipped memory-plugin contract resolves that setting against registered memory embedding adapter ids rather than the memory slot id (`docs/plugins/sdk-overview.md`, `src/plugins/capability-provider-runtime.ts:101`, `src/plugins/registry.ts:1545`). Unknown provider ids also fail explicitly in the current runtime (`extensions/memory-core/src/memory/embeddings.ts:46`). I verified this behavior is present in the released `v2026.4.22` tag (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). Because the underlying request is already covered by the shipped adapter-id architecture, this PR’s slot-validation implementation is superseded on current `main`.
