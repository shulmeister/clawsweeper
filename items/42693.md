## Summary

Current main already exposes the requested official pre-run model-routing surface via the public `before_model_resolve` plugin hook, with prompt/attachment input, per-run provider/model overrides, agent-scoped context, and published docs.

## Evidence

- **Public docs describe the exact hook:** `before_model_resolve` is documented as running pre-session to override provider/model before model resolution.
  - file: [docs/concepts/agent-loop.md:93](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/docs/concepts/agent-loop.md#L93)
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Public hook contract matches the feature request:** The hook event provides the prompt plus optional attachment metadata, and the result returns `modelOverride` and `providerOverride` for prompt-sensitive routing.
  - file: [src/plugins/hook-before-agent-start.types.ts:7](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/plugins/hook-before-agent-start.types.ts#L7)
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Runtime applies the override before resolving the model:** The embedded runner calls `runBeforeModelResolve(...)` before `resolveModel()`, then swaps local `provider`/`modelId` to the hook-provided values.
  - file: [src/agents/pi-embedded-runner/run/setup.ts:58](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/agents/pi-embedded-runner/run/setup.ts#L58)
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)

  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)

## Close Comment

Closing as implemented on main after Codex review. OpenClaw already has an official pre-run routing hook for this use case: `before_model_resolve` is documented in `docs/concepts/agent-loop.md:93`, its public contract takes the prompt plus attachment metadata and returns `providerOverride` / `modelOverride` in `src/plugins/hook-before-agent-start.types.ts:7`, and the runner applies those overrides before model resolution in `src/agents/pi-embedded-runner/run/setup.ts:58`. The hook context also includes `agentId`/`sessionKey` in `src/plugins/hook-types.ts:154`, so plugin logic can scope routing per agent. This gives a stable supported plugin path instead of patching runtime internals, which matches the issue’s requested “official routing hook or config policy” via the hook/plugin API. I also confirmed the same surface is already present in release `v2026.4.22`.
