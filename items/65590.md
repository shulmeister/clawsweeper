## Summary

Keep open. Current `main` statically looks correct for forwarding configured `headers` into the streamable HTTP client, but the evidence is not strong enough to close: there is no repo test proving those headers reach a remote streamable-HTTP server, and I could not perform the reporter's authenticated reproduction against the cited GoHighLevel endpoint from this review context.

## Risks / Open Questions

- There is no OpenClaw regression test proving `streamable-http` custom headers survive the full initialize/tools-call flow to a remote server.
- Without a live repro against the reporter's endpoint and credentials, closing as `cannot_reproduce` or `implemented_on_main` would be too speculative.
