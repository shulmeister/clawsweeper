## Summary

Current main still has the reported gap. `logs.tail` returns log lines after applying the shared default redaction, but those defaults only cover Bearer auth, key/token-like fields, CLI flags, PEMs, and known token prefixes. They do not include the reported Basic auth, generic JWT, or `x-pomerium-jwt-assertion` patterns. PR #67041 is an open unmerged candidate fix, so the issue should remain open until that or an equivalent patch lands.

## Best Possible Solution

Keep #66832 open as the canonical bug. Review and land #67041 or an equivalent patch that extends the shared log-tail redaction defaults for Basic auth, generic JWTs, `x-pomerium-jwt-assertion`, and relevant custom auth headers, with focused unit coverage in `src/logging/redact.test.ts` and a `logs.tail` regression test.
