## Summary

Keep this PR open. Current main still has the zero-output dispatch path the PR targets: an undefined resolver result becomes an empty replies array, dispatch records completion, and it returns without queuing any final reply. The PR's narrow direct-chat fallback remains a plausible implementation candidate, with the discussion already narrowing the risk around channel/group silence.

## Best Possible Solution

Keep this PR open and review/update it against current main. The best fix is a narrow dispatch-level fallback after final queued counts are known, only for direct-style user delivery with no routed origin, no suppressed delivery, no queued tool/block/final output, no block-stream output, and no intentional handled/no-text branch, plus regression tests for direct empty output and channel/group silence.
