## Summary

Keep open. Current main still redacts agent-visible text without a stable marker or warning string; `sessions_history` only exposes a partial `contentRedacted` boolean. The exact implementation work is represented by open PR #68483, but it has not landed on main.

## Best Possible Solution

Keep #68425 open and land #68483 or an equivalent core change that adds a stable exported marker/notice for agent-visible redacted outputs, updates `sessions_history` to emit it when redaction fires, and covers the behavior in `src/logging/redact.test.ts` plus `src/agents/openclaw-tools.sessions.test.ts`. If maintainers decide structured-only is sufficient, document that contract explicitly and extend it to every relevant agent-visible redacted tool-output surface.
