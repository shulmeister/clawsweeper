## Summary

Keep open. Current `main` still contains Windows `.cmd`/`.bat` spawn paths that explicitly set `shell: true`, and tests on `main` codify that behavior. The repo also has a safer Windows exec pattern elsewhere that avoids `shell`, so this issue does not look implemented or obsolete. I could not run a native Windows reproduction in this environment, so the exact user-visible trigger path is not pinned down, but there is not enough evidence to close conservatively.

## Evidence

- **Current tests assert the shell:true behavior:** `src/tui/tui.test.ts` has explicit expectations that Windows `.cmd` and `.bat` auth commands resolve to `{ shell: true }`, so current `main` is intentionally preserving this behavior rather than having already removed it.
  - file: [src/tui/tui.test.ts:374](https://github.com/openclaw/openclaw/blob/c150110e02550eaf6c56db0c851dc62fc0ca88e1/src/tui/tui.test.ts#L374)
