## Summary

Keep this PR open. Current main still reads only the first existing primary transcript candidate in `readSessionMessages` and returns `[]` when every primary candidate is missing, while reset still archives prior transcripts as `.jsonl.reset.<timestamp>`. The PR is a focused implementation candidate for that remaining `chat.history` / `sessions_history` gap, and the related open PRs are complementary rather than superseding it.

## Best Possible Solution

Keep this PR open as the focused implementation candidate for reading the latest reset archive when the primary transcript is missing. Review and land it, or fold the same behavior into a broader reset-archive recovery patch, while leaving #45003/#56131/#57139 or narrower follow-ups to track UI listing, daily-reset trigger policy, and richer archive discovery metadata.
