## Summary

Current `main` already fixes the cross-date `openclaw logs --follow` bug by resolving a missing daily log path to the newest existing rolling log file before serving `logs.tail` results.

## Evidence

- **Log tail now resolves missing daily files:** `readConfiguredLogTail()` calls `resolveLogFile()`, which detects dated rolling log paths like `openclaw-YYYY-MM-DD.log`, scans sibling rolling logs, and returns the newest existing file by mtime when the configured/today file is missing.
