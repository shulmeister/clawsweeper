## Summary

#61981 is a duplicate of the already-open canonical issue #52680, which tracks the same underlying read-tool problem: out-of-range offsets after file changes should not abort the run and should clamp or return a diagnostic. Current main still has the underlying behavior, and the prior targeted fix PR #52739 was closed unmerged, so the remaining work belongs on #52680.

## Best Possible Solution

Close #61981 as duplicate of #52680 and keep the remaining implementation discussion there. The best fix is a narrow read-tool recovery in src/agents/pi-tools.read.ts that handles both explicit-limit and adaptive-paging calls, preserves any already-aggregated read content, uses precise out-of-range offset detection, and adds regression tests for stale offsets after file length changes.

## Evidence
