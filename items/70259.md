## Summary

Keep open. Current main still ignores `agents.list[].thinkingDefault` in the `chat.history` fallback when `entry.thinkingLevel` is unset, and falls directly to the model/global resolver. The PR remains a focused bug fix with relevant tests proposed; related #70302 is an adjacent SessionRow fix, not a replacement for this chat.history path.

## Best Possible Solution

Keep #70259 open and either merge or port its narrow `chat.history` change: import/use `resolveAgentConfig`, prefer `entry.thinkingLevel`, then `resolveAgentConfig(cfg, sessionAgentId)?.thinkingDefault`, then the existing catalog-aware `resolveThinkingDefault` fallback. Add focused gateway tests for per-agent fallback, persisted session override priority, missing per-agent fallback, and normalized webchat session keys. Handle #70302 separately for the SessionRow/sessions.list surface.
