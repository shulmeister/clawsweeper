## Summary

Keep open. Current `main` partially addresses the report: plain `openclaw memory status` now uses a status-only QMD manager, and the stale `memory-alt-*` default-collection bug was shipped in `2026.4.12`. But `openclaw memory search` and `openclaw memory index` still go through the full QMD manager path, which starts boot/update work by default and then waits for pending QMD work during teardown. With current 120s QMD update/embed defaults, that remains a credible hang path on slow ARM/QMD hosts, so this is not safe to auto-close as fixed or obsolete.
