## Summary

Keep #14593 open. Current main still does not provide an official Linuxbrew/Homebrew path for the Docker onboarding flow, the reported `openai-whisper` bundled skill remains brew-only, and `installSkill` still returns `brew not installed` when no brew executable is found. The discussion also has active workaround/fix PR references and a same-day maintainer/Codex keep-open comment, so this is not safely closeable as implemented, duplicate, or stale.

## Best Possible Solution

Keep #14593 open until a focused Docker/skills fix lands. The likely path is an official opt-in Linuxbrew setup for the non-root Docker gateway image and setup script, OS-scoped non-brew installer metadata for bundled brew-only skills such as `openai-whisper`, or onboarding/status behavior that avoids offering impossible brew installs in the shipped Docker flow. Update and validate `Dockerfile`, `scripts/docker/setup.sh`, `docker-compose.yml`, `docs/install/docker.md`, bundled skill metadata, `src/agents/skills-install.ts`, and `src/commands/onboard-skills.ts`.
