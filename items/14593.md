## Summary

Keep #14593 open. Current main still does not provide an official Docker/Linuxbrew path, the Docker setup flow does not pass or persist any brew install option, `openai-whisper` is still a brew-only bundled skill, and the installer still returns `brew not installed` when brew is absent. The issue discussion also contains a same-day maintainer/Codex keep-open comment and active unresolved fix-PR references, so conservative cleanup should not close it.

## Best Possible Solution

Keep #14593 open until a focused Docker/skills fix lands. The likely maintainer path is an official opt-in Linuxbrew setup for the non-root Docker image, OS-scoped or non-brew installer metadata for bundled brew-only skills such as openai-whisper, or onboarding/status behavior that does not offer impossible brew installs in the shipped Docker flow. Validate across Dockerfile, scripts/docker/setup.sh, docs/install/docker.md, bundled skill metadata, src/agents/skills-install.ts, and src/commands/onboard-skills.ts tests, using the active fix PR discussion (#57553/#42303) as input.
