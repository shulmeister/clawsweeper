## Summary

Keep #14593 open. Current main still has no official Docker/Linuxbrew install path, the Docker setup script does not pass any brew-related build arg, Docker docs do not document a brew-based skill path, `openai-whisper` remains brew-only, and the skills installer still returns `brew not installed` when no brew executable is available. The latest issue discussion also includes a maintainer Codex review saying to keep it open while PRs such as #57553/#55940 remain implementation candidates.

## Best Possible Solution

Keep #14593 open. The best maintainer path is a focused Docker/skills fix: add a maintainer-approved opt-in Linuxbrew build/setup/docs path that works for the non-root Docker image, add non-brew installers or OS scoping for bundled brew-only skills, or change onboarding so the official Docker flow does not offer impossible brew installs. Active PRs such as #57553/#55940 can remain implementation candidates until one lands and is verified against Dockerfile/setup/docs plus onboarding and skill-install behavior.
