## Summary

Current main still does not solve #14593. The official Docker image/setup/docs have no Linuxbrew/Homebrew install option, bundled skills such as `openai-whisper` remain brew-only, onboarding still shows a Homebrew recommendation instead of provisioning brew, and the install path still returns `brew not installed` when no brew executable is available. The issue should stay open while the Docker/skill-install behavior or a maintainer-approved product direction is unresolved.

## Best Possible Solution

Keep #14593 open. The best maintainer path is a focused Docker/skills fix: either add an opt-in Linuxbrew build/setup/docs path that works for the non-root Docker image, add non-brew installers or OS scoping for bundled brew-only skills, or change onboarding so the official Docker flow does not offer impossible brew installs. Active related PRs such as #57553/#55940 can remain the implementation candidates until one lands and is verified against Dockerfile/setup/docs plus onboarding/skill-install tests.
