## Summary

Keep this issue open. Current `main` still has the reported LINE webhook behavior: both the Node HTTP handler and Express-style middleware await event processing before sending HTTP 200. The related fix PR was closed unmerged, and current tests still lock in blocking/error-before-ACK behavior rather than the immediate-ACK flow requested here.

## Best Possible Solution

Keep this issue open and implement the targeted LINE plugin fix: after signature validation/body parsing, send HTTP 200 immediately, then run `handleWebhook`/`onEvents` fire-and-forget with async error logging. Update the current blocking/error-before-ACK tests so they prove the response is sent before event processing finishes and that async processing failures are reported without changing the already-sent ACK. The closed unmerged #65397 and the reporter/commenter discussion are useful starting points, but the fix still needs to land on `main`.
