## Summary

Keep open. Current main still has the nested `payload.text` + `model` normalization bug this PR targets, and the PR discussion correctly identifies a parallel top-level/flat-param path that the submitted diff does not address. This is a useful bug-fix PR, but it should not be closed as cleanup or treated as implemented on main.

## Best Possible Solution

Keep this PR open and request a small update: apply the same `text` plus `hasAgentTurnPayloadHint()` promotion in both `coercePayload()` and `inferTopLevelPayload()`, with regression tests for nested `payload: { text, model }` and flat/top-level `{ text, model }` create/patch inputs. After that, review the targeted cron normalization tests and land if green.
