## Summary

Keep #20786 open. Current main at 82262387651ab270b565f71533ce4937c9c00dc6 still has only partial Telegram Business plumbing: the update names are allowed, but there are no business_message/business_connection handlers, no business fields in dedupe or sequential routing, and no outbound business_connection_id support. The issue discussion is actionable: dmtnikitin documented the runtime workaround and required outbound transformer, crowneglobal confirmed the same source gap, and PR #53629 is the relevant open/unmerged implementation attempt rather than shipped behavior.

## Best Possible Solution

Keep #20786 open as the canonical Telegram Business support tracker. Land PR #53629 or an equivalent Telegram plugin implementation that registers business_connection and business_message handlers before the normal message handler, routes business_message through the existing inbound pipeline, extends dedupe and sequential keys, persists/maps business_connection_id for outbound replies, and adds focused tests plus Telegram user docs.
