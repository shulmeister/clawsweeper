## Summary

Keep open. Current main still treats heartbeat work as prompt text sent through the model, and the skill metadata/config/runtime surfaces do not provide declared sandboxed function schemas or a core-enforced remote invocation path. The provided discussion and related-item context do not show a canonical duplicate, shipped fix, or plugin workaround that supersedes the requested core security/API contract.

## Best Possible Solution

Keep this issue open for maintainer product/security design. A complete fix likely needs a versioned skill metadata schema for declared sandboxed functions, parser and docs updates, runtime validation of structured remote heartbeat invocations, execution through existing tool/plugin capability boundaries without sending remote content to the model as free-form instructions, and tests covering allowed calls, rejected unknown calls, schema validation failures, and prompt-injection non-delivery. ClawHub trust badges can follow once the core contract exists.
