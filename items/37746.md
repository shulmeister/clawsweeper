## Summary

Keep open. Current main still lacks the requested declared sandboxed skill-function metadata and core-enforced remote heartbeat invocation path. Heartbeats remain prompt/model turns, and the existing plugin/tool surfaces can support tools or hooks but do not provide this skill-level security contract.

## Best Possible Solution

Keep #37746 open for maintainer/API design. The likely product path is an additive, documented skill/plugin contract for declared callable functions with schema validation, plus a heartbeat/runtime path that fetches structured invocations, validates them against that contract, executes only allowed functions, and keeps remote content out of free-form model instructions. If maintainers instead want this outside core, they should first define the missing plugin/skill API explicitly, because current main does not make the requested enforcement generally possible.
