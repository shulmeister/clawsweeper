## Summary

Keep open. Current main still only implements `tools.fs.workspaceOnly`; the proposed `tools.fs.allowedRoots` field is absent from the config contract, runtime schema, sandbox path API, and tool wiring. The PR is a focused core filesystem-policy feature rather than ClawHub/plugin work, and the linked issue/previous PR do not supersede it because #33635 was closed stale and #33708 was closed unmerged.

## Best Possible Solution

Keep the PR open as the active implementation candidate. The best path is to rebase/review it against current main, preserving pinned read/write/delete behavior, rejecting ambiguous root targets and relative roots, and threading `allowedRoots` consistently through config types/schema/help, `resolveToolFsConfig`, sandbox path matching, read/write/edit, apply_patch, and media/prompt-image consumers with targeted regression tests.
