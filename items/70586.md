## Summary

Keep this PR open. Current main still has the Slack message-event activity gap this PR targets, but the submitted patch is too broad: normal handled `message` and non-DM `app_mention` events already call `trackEvent` through `createSlackMessageHandler`, and stale-socket health is now based on transport activity rather than `lastEventAt`.

## Best Possible Solution

Keep this PR open and request a narrower revision: thread `trackEvent` into `registerSlackMessageEvents`, but call it only for accepted message-event branches that return before `handleSlackMessage` and truly represent inbound app activity. Preserve `createSlackMessageHandler` as the single tracking point for normal handled messages and app mentions, and add tests proving handled events track once, accepted early-return branches track once, and mismatched, duplicate, invalid, unauthorized, or self-generated events do not track.
