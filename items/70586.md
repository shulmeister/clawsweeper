## Summary

Keep PR #70586 open. Current main still does not thread `trackEvent` into `registerSlackMessageEvents`, so subtype system-event messages and DM-skipped `app_mention` events can miss liveness/status timestamp updates. However, the PR as written should not be closed as implemented or blindly merged: accepted message and channel `app_mention` flows already call `trackEvent` inside `createSlackMessageHandler`, so the proposed event-level call would double-update normal inbound messages.

## Best Possible Solution

Keep the PR open and revise it so message-event liveness is single-counted: either track only the `registerSlackMessageEvents` branches that return before `handleSlackMessage`, or centralize the normal-message tracking in one place while preserving the mismatch drop guard. Add focused tests for subtype system events, DM-skipped `app_mention`, mismatched events that must not track, and normal message/app_mention paths tracking exactly once.
