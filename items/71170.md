## Summary

Keep this PR open. Current main still lacks the proposed `agents.defaults.reminderGuard` and per-agent `reminderGuard` opt-out, and the shipped reminder guard still appends the note whenever a reminder-like reply has no successful core `cron.add` and no enabled same-session core cron job. The PR remains a plausible implementation candidate for the plugin-managed scheduler false-positive described in #70666.

## Best Possible Solution

Keep this PR open for maintainer review. The best near-term path is to land or adapt the config-based `reminderGuard` opt-out with the matching config/schema/docs/tests; a richer plugin SDK signal for externally scheduled reminders can remain a separate CODEOWNER-reviewed API proposal.
