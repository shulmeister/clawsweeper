## Summary

PR #42089 should close as superseded by the maintainer-owned canonical i18n/localization issue #3460. Current main has Control UI i18n for Japanese and Chinese, but the native iOS/Android localization requested by this PR is not implemented on main; the close is based on #3460's explicit maintainer policy that translation PRs are being consolidated/closed until the i18n architecture is ready. The PR branch also still has unresolved review findings and unrelated `.secrets.baseline` churn.

## Best Possible Solution

Close this PR as superseded by #3460. Future native iOS/Android localization should wait for a maintainer-requested i18n path, then come back as a focused PR that avoids unrelated baseline changes, declares all iOS localizations, preserves existing status semantics/spinners, and includes native app coverage or clear manual verification.
