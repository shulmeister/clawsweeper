## Summary

Keep open. Current `main` still routes enabled non-channel plugins through `registrationMode: "full"` during non-activating loads, still requires `activate:false` loads to be uncached, and still unconditionally runs plugin `register(api)` on that path. The reported `"discovery"`-mode/docs fix is not present on `main`.

## Evidence

- **Snapshot loads are still uncached:** `loadOpenClawPlugins` throws unless `activate:false` is paired with `cache:false`, so repeated snapshot loads still bypass the registry cache.
  - file: [src/plugins/loader.ts:1831](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/plugins/loader.ts#L1831)

  - file: [src/plugins/loader.ts:1389](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/plugins/loader.ts#L1389)
