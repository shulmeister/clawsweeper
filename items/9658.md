## Summary

Keep #9658 open. Current main still lacks a streamed, file-backed, or payload-reference argument path for the affected built-in tools. `exec`, `process` write/paste, `write`, and `apply_patch` remain string-only at their model-facing schemas/runtime validators, and the current docs still document inline payloads. The newer provider-stream code only buffers and caps streamed model tool-call arguments; it does not bypass JSON/tool-call argument serialization for large tool inputs.

## Best Possible Solution

Keep the issue open for maintainer product/API design. The likely fix belongs in core tool invocation surfaces: add a backward-compatible payload-reference or streamed/file-backed argument mechanism for exec command/stdin, process write/paste, write content, and apply_patch input, then document limits/workarounds and add regression coverage for large payloads and special-character content. If maintainers decline the API, document the supported chunking/file workflows and actual limits explicitly.
