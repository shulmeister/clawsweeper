## Summary

Keep open. Current main still does not expose streaming or file-backed tool arguments for the affected core paths: exec takes a JSON string command, process stdin/paste payloads are JSON string fields, write requires inline content, and apply_patch also takes one inline string. The existing malformed tool-call argument repair is bounded repair logic, not a transport/API that bypasses JSON serialization. The related #9000 is a closed qmd memory_search bug, not a canonical tracker for this feature request.
