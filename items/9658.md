## Summary

Keep #9658 open. Current main still requires the affected tool payloads to arrive as inline JSON-serializable strings/objects, and the existing provider-side tool streaming/argument repair code does not add streamed, file-backed, or payload-reference arguments for exec, process write/paste, write, or apply_patch. The issue discussion also already contains an April 26, 2026 keep-open review posted by steipete.

## Best Possible Solution

Keep the issue open for maintainer product/API design. A complete solution should add a backward-compatible streamed or file-backed payload/reference mechanism for exec command/stdin, process write/paste data/text, write content, and possibly apply_patch input; document supported limits and safe workarounds; and add regression coverage for large and special-character payloads. If maintainers decide not to support this, document the explicit limits and recommended chunking/file workflows instead.
