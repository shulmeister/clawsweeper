## Summary

Keep #9658 open. Current main still lacks a streaming or file-backed payload transport for the affected tool arguments: exec uses a single inline JSON string command, process write/paste use inline string fields, write requires inline content, and apply_patch takes one inline string patch. The latest issue discussion also includes a maintainer-authored Codex review comment explicitly keeping this issue open, and the mentioned #9000 is a closed qmd memory_search bug rather than a canonical tracker for this feature.

## Best Possible Solution

Keep the issue open for maintainer product/API design. A real fix likely needs a backwards-compatible core payload mechanism, such as streamed or file-backed stdin/content handles for exec, process write/paste, write, and possibly apply_patch, with schema/docs updates and regression coverage for large and special-character payloads. If maintainers decide not to support that transport, document the supported limits and safer workarounds explicitly.
