## Summary

Keep #9658 open. Current main still does not provide streaming or file-backed tool argument transport for the affected built-in tools: `exec` command, `process` write/paste payloads, `write` content, and `apply_patch` all remain inline string tool arguments. The only issue comment is a maintainer discussion note from steipete explicitly keeping this open on the same basis, and the related #9000 is a closed qmd `memory_search` bug rather than a canonical tracker for this feature.

## Best Possible Solution

Keep the issue open for maintainer product/API design. A complete fix likely belongs in the core tool transport and schemas, adding a backward-compatible streamed or file-backed payload mechanism for exec stdin/content, process write/paste, write content, and possibly apply_patch, plus docs and regression coverage for large and special-character payloads. If maintainers decide against this transport, document supported limits and recommended safe workarounds instead.
