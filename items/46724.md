## Summary

Keep open. The issue requests a core Gateway federation feature: peer Gateway authentication, mutual WebSocket links, cross-Gateway routing, and HA/scaling semantics. Current main still documents a single-Gateway control plane with isolated multi-Gateway profiles, and the protocol/auth code only recognizes operator/node roles, with no gateway peer role or gw.send-style routing method. This is not a conservative ClawHub/plugin cleanup candidate because the missing pieces are core gateway protocol, auth, routing, and product architecture.
