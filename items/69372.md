## Summary

Keep PR #69372 open. Current main has not implemented the PR's fail-closed Control UI canvas URL behavior or a safe Control UI capability refresh path. The PR is also not ready to close as obsolete because its central bug is still plausible, but the posted P1 review identifies a real blocker in the submitted approach: timed websocket reconnects can reject pending RPCs and reset in-flight chat state.

## Best Possible Solution

Keep the item open and ask for a revised core fix: preserve the fail-closed iframe URL behavior for protected canvas/a2ui paths, but refresh `hello.canvasHostUrl` without forced websocket teardown. A better implementation would add or expose an operator/Control UI-safe canvas capability refresh path, update the app state with the refreshed scoped host, and keep the regression tests for missing/unscoped host fallback, a2ui coverage, timer cleanup, and active chat preservation.
