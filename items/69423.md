## Summary

Keep #69423 open. Current main still has the core failure mode reported by the issue: Anthropic tool payload construction dereferences `tool.parameters.properties` without guarding a missing `parameters` object or accepting an `inputSchema` alias. Plugin tools are still appended into the normal agent tool set, and the shared schema normalization path returns tools unchanged when `parameters` is absent. The linked PR #69432 directly targeted this bug but is closed and unmerged in the provided GitHub context, so it does not supersede or fix the issue on main.

## Best Possible Solution

Keep the issue open and land a focused core fix: make Anthropic tool conversion tolerant of missing or malformed plugin schemas, preferably normalizing `parameters`/`inputSchema` to a safe object schema with a plugin/tool diagnostic instead of throwing. Add regression coverage in `src/agents/anthropic-transport-stream.test.ts` for both an `inputSchema`-only plugin tool and a tool with no usable schema; consider a follow-up plugin registration/status diagnostic so broken plugins are visible without taking down all Claude runs.
