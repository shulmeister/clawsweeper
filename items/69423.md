## Summary

Keep #69423 open. Current main still has the reported Anthropic crash path: the Anthropic adapter unconditionally reads `tool.parameters.properties`, plugin registration/resolution accepts and forwards plugin tools without runtime schema validation, and there is no regression test for `inputSchema` or missing-`parameters` Anthropic tools. The related PR #69432 described the right fix but is closed unmerged and its head commit is not present in this checkout.

## Best Possible Solution

Fix this in core because a malformed third-party plugin can poison the shared Anthropic request-building path. Add a small Anthropic schema resolver that accepts canonical `parameters`, optionally handles MCP/Anthropic-style `inputSchema`, and degrades missing or malformed schemas to an empty object schema or skips the bad tool with a plugin/tool diagnostic. Add regression tests in `src/agents/anthropic-transport-stream.test.ts` for `inputSchema`-only and missing-`parameters` tools, and consider a follow-up validation/status diagnostic in the plugin registration/resolution path.
