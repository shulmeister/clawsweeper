## Summary

Current main still has the reported Unix/macOS stale-gateway PID classification bug. `findGatewayPidsOnPortSync` still runs `lsof -Fpc` and trusts `parsePidsFromLsofOutput`, which only returns PIDs whose lsof command token contains `openclaw`; there is no Unix `ps`/argv verifier equivalent to the Windows path. The same parser is still used by port polling, and tests still assert that non-`openclaw` lsof command names are dropped. The linked PR #70681 appears to be the focused fix path, but it is still open and the current main SHA has not adopted it.

## Best Possible Solution

Keep #70664 open until a maintainer lands #70681 or an equivalent Unix-side fix: parse lsof as raw listener PIDs, preserve self/ancestor exclusions, verify candidate gateway PIDs with full argv via `ps`/`isGatewayArgv`, update `pollPortOnce`, and replace the comm-filter regression tests with argv-verifier coverage.
