## Summary

Keep #70664 open. Current main still has the reported Unix/macOS behavior: `findGatewayPidsOnPortSync` reads `lsof -Fpc` and `parsePidsFromLsofOutput` only accepts records whose `c` command token contains `openclaw`, with no Unix `ps`/argv verification. That means a node-based gateway reported by `lsof` as `cnode` would still be dropped. The linked PR #70681 appears to be the right canonical fix, but it is open/unmerged in the provided GitHub context, so the issue should remain open until that or an equivalent change lands.

## Best Possible Solution

Keep #70664 open and review/land PR #70681 or an equivalent mainline fix: parse raw Unix lsof listener PIDs, verify each candidate via full argv (`ps` on macOS, `/proc` or equivalent on Linux) using the existing `isGatewayArgv` contract, apply the same verification in `pollPortOnce`, and update regression tests for the `lsof cnode`/`ps openclaw-gateway` case. Once merged, close the issue as implemented with the landing commit and release evidence.
