## Summary

Keep open. Current main has partial related gateway-auth cleanup, but it still does not ship the full diagnostic behavior this PR is trying to add: `callGateway` does not capture `GatewayClient.onConnectError` and include it in the final close error. The PR is still useful, but it needs revision because the existing review flags a paired loopback/device-token regression risk.

## Best Possible Solution

Keep this PR open for revision. Preserve explicit `deviceIdentity: null` in `GatewayClient` if that API contract is desired, capture `GatewayClient.onConnectError` inside `callGateway` and include it in the final RPC close error, and retain current main's narrower backend/shared-token loopback identity omission so paired local device-token auth keeps working.
