## Summary

Current `main` already implements the reported handshake-timeout fix: the server and client now share a 10s pre-auth/connect-challenge timeout, runtime overrides exist for both sides, and the changelog says this shipped in `2026.3.22` to stop busy local gateways from dropping healthy CLI connections.

## Evidence

- **Changelog entry for the fix:** `CHANGELOG.md` documents: "Gateway/WS handshake: raise the default pre-auth handshake timeout to 10 seconds and add OPENCLAW_HANDSHAKE_TIMEOUT_MS as a runtime override so busy local gateways stop dropping healthy CLI connections at 3 seconds." under release `2026.3.22` via `#49262`.
  - file: [CHANGELOG.md:2446](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/CHANGELOG.md#L2446)

  - file: [src/gateway/handshake-timeouts.ts:1](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/src/gateway/handshake-timeouts.ts#L1)

## Close Comment

Closing from a Codex review: current `main` already contains the handshake-timeout fix described here. `src/gateway/handshake-timeouts.ts:1` raises the shared pre-auth/connect-challenge default to 10s and adds `OPENCLAW_HANDSHAKE_TIMEOUT_MS` plus `OPENCLAW_CONNECT_CHALLENGE_TIMEOUT_MS`; `src/gateway/client.ts:167` routes the client timeout through that shared resolver; and `src/gateway/server/ws-connection.ts:366` uses the shared pre-auth timeout on the server side. If you still hit this on a current release, please open a fresh issue with the exact version, gateway logs, and any local timeout overrides.
