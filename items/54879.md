## Summary

Keep open. Current main still lacks the requested CDP/IME text insertion action in the browser act schema, request union, normalizer, managed Playwright dispatch, existing-session Chrome MCP path, and user docs. The issue's direct `Input.insertText` testing is useful evidence of a real gap, and the related ClawSweeper items cover nearby CDP/session/download problems rather than this action surface.

## Best Possible Solution

Keep this issue open and implement the capability in the bundled browser plugin as an additive `browser act` kind. The likely path is to extend `BrowserActRequest`, `BROWSER_ACT_KINDS`, `ACT_KINDS`, normalization, CLI/docs, and `/act` dispatch; focus the resolved ref/selector/tab; call CDP `Input.insertText` for managed/raw CDP profiles; define supported behavior or a documented unsupported error for existing-session Chrome MCP profiles; and add focused tests for schema acceptance, normalization, dispatch, and docs.
