## Summary

Keep open. Current main has related mitigations, including read-only `talk.config`/`talk.mode` handlers and gateway-side clobber detection, but the macOS stale-write path called out in the report is still present: local macOS config saves can fall back from a failed gateway `config.set` to direct disk writes, and AppState still has a separate direct config writer. The linked #43244 is relevant but remains open and explicitly covers only gateway RPC writers, not these macOS direct writers.

## Best Possible Solution

Keep this issue open and fix the macOS-owned config mutation path. The best fix is to make macOS config saves conflict-aware end to end: do not fall back to `OpenClawConfigFile.saveDict` after a stale gateway `config.set`; route AppState gateway sync through a fresh snapshot/baseHash or restrict it to app-local fields; preserve `gateway.auth` and all `talk.providers` from a fresh source before writing; and add macOS regression coverage for stale `config.set` rejection, token-auth local gateway config, multi-provider Talk config, and no gateway restart/auth-token mutation when enabling Talk Mode. #43244 may still help gateway RPC races, but it should not be treated as the complete fix for this macOS path.
