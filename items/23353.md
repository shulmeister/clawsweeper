## Summary

Keep #23353 open. Current main still lacks the requested Anthropic native server-side tool support: no config/schema for Anthropic server tools or PTC, no Anthropic request injection for `web_search_20260209`, `web_fetch_20260209`, or `code_execution_20260120`, no `allowed_callers` propagation, and no stream/content handling for Anthropic server-tool result blocks. The related implementation PRs #23469 and #23573 are closed unmerged, and the latest maintainer/Codex issue comment also explicitly kept this tracker open.

## Best Possible Solution

Keep #23353 open as the broad Anthropic native server-tools tracker. The best path is a provider/runtime design that adds versioned Anthropic server-tool config, injects `web_search`, `web_fetch`, and `code_execution` server tools only for eligible Anthropic routes, defines coexistence or suppression behavior for managed `web_search`/`web_fetch`, preserves/renders server-tool content blocks, supports PTC `allowed_callers` and `caller` semantics, and adds focused schema, transport, replay, docs, and regression coverage. Closed PRs #23469 and #23573 remain useful reference material but do not supersede the issue.
