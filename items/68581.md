## Summary

Keep #68581 open. Current main still implements Android location as foreground-only: the Android settings model has only Off/While Using, legacy `always` is migrated down to `whileUsing`, background `location.get` returns `LOCATION_BACKGROUND_UNAVAILABLE`, and the manifest lacks background-location/foreground-service-location support. The related #71015 was closed after cross-referencing this older, more specific tracker, so it does not supersede the remaining work.

## Best Possible Solution

Keep the issue open as an Android app feature/product decision. If accepted, implement `LocationMode.Always`, settings UX, Android background permission handling, a location foreground-service path for episodic `location.get`, docs updates, and tests, with special attention to Play-safe flavor policy because background location was previously removed.
