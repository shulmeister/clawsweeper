## Summary

Current main still does not implement true progressive Feishu block delivery. Feishu reply options force core block streaming off, the Feishu delivery path drops block payloads unless they can feed the streaming-card path, and the streaming-card implementation still relies on Feishu Card Kit playback settings rather than sending each emitted block as a normal progressive reply. The existing tests assert this suppression/card-only behavior, so the related closed Feishu streaming reports do not supersede this remaining request.

## Best Possible Solution

Keep this open for a Feishu-plugin-owned behavior fix. The likely path is to add a non-card progressive block delivery mode for Feishu, wire it to the channel/global block-streaming configuration without duplicate final/card sends, and update Feishu tests plus docs/config metadata so streaming-card preview and true block replies are clearly separate.
