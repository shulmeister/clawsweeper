## Summary

Keep open. Current `main` does not already implement Telegram `allowBots`, and the PR would change an intentional `channel_post` bot-to-bot path that current code and tests still exercise. The unresolved review discussion identifies a real backward-compatibility question, so this is not a conservative auto-close candidate.

## Evidence

- **Current main intentionally processes Telegram channel posts:** `registerTelegramHandlers` documents `channel_post` as enabling bot-to-bot communication via Telegram channels, then normalizes those posts into the standard inbound pipeline. The synthetic sender is marked `is_bot: true` in the same handler, so a blanket inbound bot gate would affect this documented path.
