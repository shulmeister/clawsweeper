## Summary

Keep open. Current main has a Telegram self-echo guard, but it still does not implement the PR's requested Telegram `allowBots` config/runtime surface. The PR remains relevant, and the existing review concern about default-dropping `channel_post` traffic is a real compatibility question for maintainer review rather than a close reason.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to implement Telegram `allowBots` across config types, Zod schema, generated metadata/UI hints, docs, and runtime filtering, while preserving or explicitly documenting `channel_post` behavior and adding regression tests for both bot-authored group messages and channel posts.
