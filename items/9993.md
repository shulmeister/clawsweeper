## Summary

Current main has strong built-in mitigation for the reported unsafe config writes, but it does not implement the requested synchronous config pre-apply hook with current/pending config context. The remaining capability is meaningful enough that this should not be auto-closed.

## Evidence

- **Config RPC validation exists:** config.patch merges the patch, validates the full post-merge config, preflights active SecretRefs, and only then writes the validated config. This mitigates much of the original crash-loop risk but is not a pluggable pre-apply hook.
