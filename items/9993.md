## Summary

Keep #9993 open. Current main has built-in validation, base-hash protection, SecretRef preflight, and an agent-tool config mutation guard, but it still does not expose the requested synchronous plugin/API-level `config:pre-apply` veto before config mutations are persisted.

## Best Possible Solution

Keep #9993 open and coordinate it with the config validation/mutation RFC. The best implementation is a documented synchronous config-write interception point invoked by `config.set`, `config.patch`, `config.apply`, shared config mutation helpers, and plugin runtime config writes after pending config computation plus built-in validation/preflight, but before persistence. It should expose current config, pending config, action, raw patch/full payload, baseHash, actor/client metadata, changed paths, and a typed reject result, with regression tests proving a plugin can reject direct RPC and runtime-helper writes without changing disk or active runtime config.
