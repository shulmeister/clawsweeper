## Summary

Keep open. Current main has stronger built-in config validation, SecretRef preflight, and an agent-facing config mutation guard, but it still does not provide the requested synchronous config-write hook/plugin API that receives current and pending config and can block direct config.patch/config.apply writes. The linked config-guard remains useful prior art, not an enforced core interception point.

## Best Possible Solution

Keep #9993 open as a plugin API/product-direction issue. The strongest path is a documented synchronous config-write interception point invoked by config.patch and config.apply after the pending config is computed and validated/preflighted but before writeConfigFile, with current config, pending config, action, raw patch/baseHash/actor/session metadata, and a typed reject result. Add focused tests proving a plugin can reject direct RPC writes without changing the active config.
