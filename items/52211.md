## Summary

The request targets WeCom-specific channel behavior in an external community plugin, not OpenClaw core. Current main only documents and catalogs `@wecom/wecom-openclaw-plugin`; it does not contain an in-repo WeCom channel implementation to change.

## Evidence

- **Project scope keeps optional channel capability in plugins:** The project vision says core stays lean, optional capability should usually ship as plugins, and plugins should be hosted and maintained in their own repositories.
  - file: [VISION.md:52](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/VISION.md#L52)
- **Repo architecture assigns extension-owned behavior to the extension:** Root repo guidance says core must stay extension-agnostic and extension-owned behavior stays in the extension/plugin rather than core.
  - file: [AGENTS.md:41](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/AGENTS.md#L41)
- **WeCom is listed as an external community plugin with its own repo:** Current docs list the WeCom plugin as `@wecom/wecom-openclaw-plugin` and point users to `github.com/WecomTeam/wecom-openclaw-plugin`.
  - file: [docs/plugins/community.md:115](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/docs/plugins/community.md#L115)

  - file: [src/channels/plugins/contracts/channel-catalog.contract.test.ts:40](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/channels/plugins/contracts/channel-catalog.contract.test.ts#L40)
