## Summary

Keep this issue open. Current main has single active memory embedding provider/model configuration plus a single fallback provider and model-filtered searches, but it does not implement first-class `indexes[]`, `activeIndex`, per-index ingestion state, coverage/lag status, or runtime read failover across already-built vector spaces. The related #55590 is closed, but this issue is the more precise remaining design request rather than something current main already ships.

## Best Possible Solution

Keep the issue open and design this as a memory-core or memory-plugin-contract feature: add backward-compatible multi-index config, canonical chunk source plus per-index ingestion/backfill state, active/fallback index routing that never mixes vector spaces, per-index status/metrics in `openclaw memory status`, and tests/docs that preserve current single-index configs and existing `memorySearch.fallback` behavior.
