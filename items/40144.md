## Summary

Keep open. Current main still records loop-detection tool calls into the per-session diagnostic state without any age-gap reset, and `detectToolCallLoop` counts the full retained history for generic repeat/no-progress/ping-pong checks. The heartbeat docs still default to the main session unless `isolatedSession` is explicitly enabled, so the reported main-session heartbeat false-positive path remains plausible. Related PRs #40217 and #40798 are open/unmerged, while prior similar PRs were closed unmerged.

## Best Possible Solution

Keep #40144 open until a focused fix lands on main. The likely fix belongs in `src/agents/tool-loop-detection.ts`: clear or ignore stale `toolCallHistory` entries when the gap since the last recorded call exceeds a conservative threshold, and add regression coverage simulating repeated heartbeat cycles with matching tool calls separated by that gap. The open unmerged PRs (#40217/#40798) are useful references, but the implementation should be reviewed against current main and include tests for preserving continuous in-turn detection.
