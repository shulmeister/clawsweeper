## Summary

Keep open. Current `main` no longer matches the image part of the report because Feishu outbound media sending is implemented, but the voice-message part is still incomplete: native Feishu audio is only routed for Opus/Ogg, MP3 currently falls back to a generic file send, and the generic `audioAsVoice` flag is not handled by the Feishu outbound adapter.

## Evidence

- **Feishu outbound media send exists on main:** `sendMediaFeishu` uploads images with `im.image.create`, sends them as `msg_type: "image"`, and routes supported audio/video/file payloads through `im.file.create` plus `msg_type` selection.
  - file: [extensions/feishu/src/media.ts:320](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/extensions/feishu/src/media.ts#L320)
  - command: `nl -ba extensions/feishu/src/media.ts | sed -n '300,680p'`
- **Native Feishu audio routing is limited to Opus/Ogg:** Outbound media kind resolution only maps `.opus`/`.ogg` or `audio/ogg`/`audio/opus` to `msg_type: "audio"`; other audio formats do not get native audio treatment.
  - file: [extensions/feishu/src/media.ts:539](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/extensions/feishu/src/media.ts#L539)
  - command: `nl -ba extensions/feishu/src/media.ts | sed -n '300,680p'`
- **MP3 is still treated as unsupported audio:** The current test suite explicitly expects an MP3 upload to fall back to `file_type: "stream"` and `msg_type: "file"`, not a native audio/voice message.
  - file: [extensions/feishu/src/media.test.ts:239](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/extensions/feishu/src/media.test.ts#L239)
  - command: `nl -ba extensions/feishu/src/media.test.ts | sed -n '150,265p'`
- **Shared voice flag is not consumed by Feishu outbound:** Channel outbound context exposes `audioAsVoice`, but Feishu's `sendMedia` adapter does not destructure or pass it through, so there is no Feishu-specific voice-message path using the shared flag.
  - file: [src/channels/plugins/outbound.types.ts:14](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/src/channels/plugins/outbound.types.ts#L14)
  - command: `nl -ba src/channels/plugins/outbound.types.ts | sed -n '1,40p'`
- **Feishu docs on main show partial progress:** The channel docs now advertise outbound Images and Audio support, which confirms the issue is partially addressed, but that does not cover the missing MP3/WAV/native-voice behavior above.
  - file: [docs/channels/feishu.md:419](https://github.com/openclaw/openclaw/blob/272bd59e7a8a4d6ef6e03b9bdc1068a02986d56e/docs/channels/feishu.md#L419)
  - command: `sed -n '380,440p' docs/channels/feishu.md`
