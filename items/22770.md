## Summary

Keep #22770 open. Current main still lacks a $VISUAL/$EDITOR external-editor compose mode for the TUI, and the same issue already has a maintainer Codex review explicitly keeping it open. The adjacent #55836 tracks in-place vi/vim bindings, not the requested temp-file editor handoff.

## Best Possible Solution

Keep the issue open and implement it in the core TUI composer: add a deliberate shortcut or command, suspend the TUI, write the current composer text to a temporary markdown file, launch $VISUAL falling back to $EDITOR, read the edited content back with editor.setText without auto-submitting, surface missing-editor/non-zero-exit/file-I/O statuses, document it in the TUI guide, and add focused CustomEditor/runTui tests. Keep #55836 separate for in-place vi/vim keybinding mode.
