## Summary

Keep #22770 open. Current main still lacks an external-editor compose handoff for the TUI composer: there is no Ctrl+E/$VISUAL/$EDITOR hook, no temp-file prompt roundtrip, no slash-command alternative, and the TUI docs/tests only cover the inline editor path. The related #55836 is adjacent vi/vim in-place editing, not a duplicate or shipped substitute.

## Best Possible Solution

Implement this in the core TUI composer: add a deliberate shortcut or slash command, suspend the TUI, write current composer text to a temporary markdown file, launch $VISUAL then $EDITOR and wait, read the edited content back into CustomEditor without auto-submitting, surface clear missing-editor/non-zero-exit/file-I/O status messages, update https://docs.openclaw.ai/web/tui, and add focused CustomEditor/TUI tests. Keep #55836 as the separate in-place vi/vim input-mode request.
