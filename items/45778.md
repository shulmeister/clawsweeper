## Summary

Keep open. Current main still exposes only `systemEvent` and `agentTurn` cron payloads, and runtime validation/execution requires main jobs to be `systemEvent` and isolated/current/custom-session jobs to be `agentTurn`. There is no `directExec` no-LLM cron mode, and exact source/docs search found no `OPENCLAW_REPLY_CHANNEL` or `OPENCLAW_REPLY_TARGET` injection path. The SignalRadar ClawHub skill and webhook workaround are useful prior discussion, but this issue is identifying a missing core cron/session delivery API rather than work that can be closed as plugin-only.
