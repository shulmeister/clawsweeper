## Summary

#13911 should stay open. Current main still has no per-channel subagent announce suppression or reroute config, and the delivery path still posts completion announces to the requester channel when an external route exists. The broader #69492 is an open design proposal, not a shipped fix for this concrete channel-level behavior.

## Best Possible Solution

Keep #13911 open until OpenClaw ships a real subagent completion audience/delivery policy, likely in core config and schema/docs/tests, that can suppress or reroute announces per channel without relying on AGENTS.md behavior or silent-token conventions. If #69492 lands a broader event audience primitive that covers this exact requester-channel announce path, close #13911 only after verifying the shipped implementation and tests/docs cover Telegram/iMessage-style channel suppression.
