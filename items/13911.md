## Summary

Keep #13911 open. Current main still has no first-class per-channel or per-subagent announce suppression/reroute policy, and the current code, tests, docs, and issue discussion preserve top-level external delivery for subagent completion announcements.

## Best Possible Solution

Keep #13911 open. The best path is a core-supported subagent completion delivery/audience policy, wired through config schema, generated config docs, user docs, and regression tests, and applied before external requester-channel delivery and fallback sends. If maintainers want plugin-owned policy instead, first extend the plugin API so hooks can suppress subagent completion announcements by provenance and channel without brittle outbound text filtering.
