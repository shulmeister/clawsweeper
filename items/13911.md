## Summary

Keep #13911 open. Current main still lacks a first-class per-channel suppression or reroute policy for subagent completion announcements, and the current code/docs/tests still preserve requester-channel delivery for non-subagent requesters. The broader #69492 design may eventually subsume this, but it is not shipped and the latest maintainer discussion explicitly kept this item open.

## Best Possible Solution

Keep #13911 open. The best path is a core-supported subagent completion audience/delivery policy in config schema, types, docs, and tests, applied before external requester-channel delivery is resolved. It should cover Telegram/iMessage-style channels without relying on AGENTS.md behavior, silent-token workarounds, sanitizer proxies, or the current origin-only plugin hook; if maintainers want plugin ownership, extend the hook/API to express suppression explicitly rather than only rerouting.
