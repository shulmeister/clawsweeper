## Summary

Current main still appears to reproduce the reported leak. In `skills.update`, plaintext `apiKey` and `env` values are copied into `current`, written to config, then returned directly as `config: current` in the successful RPC response. The existing targeted test only covers CR/LF normalization and does not assert response redaction. Related PRs #66928, #66986, and #69998 are useful fix attempts, but they are open/unmerged in the provided context, so the issue remains the canonical actionable bug.

## Best Possible Solution

Keep #66769 open until a narrow gateway fix lands. The likely product/code path is to redact only the response payload in `src/gateway/server-methods/skills.ts`, preferably reusing `redactConfigObject` or the same sensitive-path rules used by `config.*` responses, while still persisting the full secret values. Add a regression test that captures the `skills.update` response and proves `apiKey` plus secret-like `env` keys are redacted while non-secret env values and on-disk config behavior remain correct.
