## Summary

Keep #30381 open. Current main still validates the OpenAI-compatible request-body `model` before chat completions dispatch and will reject a generic model such as `gpt-4` even when `x-openclaw-agent-id` selects an agent. The active issue comments describe a concrete production use case, and current tests/docs still encode the stricter agent-target model contract rather than the requested compatibility bypass.

## Best Possible Solution

Implement a header-aware compatibility path in the OpenAI-compatible model validation flow: when `x-openclaw-agent-id` resolves to a valid agent, ignore the request-body `model` for agent/model validation and let the selected agent's configured model run, while still validating any explicit `x-openclaw-model` override. Add regression coverage for `/v1/chat/completions` with `model: "gpt-4"` plus `x-openclaw-agent-id`, decide whether `/v1/responses` should share the behavior, and update `https://docs.openclaw.ai/gateway/openai-http-api` if this becomes the supported contract.
