## Summary

Current main still does not implement #30381. `x-openclaw-agent-id` can choose the agent, but `/v1/chat/completions` still passes the request body `model` into OpenClaw agent-target validation and returns a 400 for generic OpenAI SDK model ids such as `gpt-4`. The provided April 2026 issue comments also show this is still actively needed by the reporter.

## Best Possible Solution

Keep #30381 open. The likely fix belongs around `src/gateway/http-utils.ts` and `src/gateway/openai-http.ts`: when `x-openclaw-agent-id` resolves to a valid agent for chat completions, ignore or bypass OpenAI body `model` validation while still using the selected agent's configured model and still validating any explicit `x-openclaw-model` override. Add a regression test with `model: "gpt-4"` plus `x-openclaw-agent-id` proving dispatch succeeds and routes to the header-selected agent.
