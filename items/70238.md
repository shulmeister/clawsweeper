## Summary

Keep #70238 open. The Windows command-prompt flicker regression is recent, has multiple user confirmations, and current main still does not contain a source-backed fix: PR #70308 was closed unmerged, the TUI relaunch spawn still lacks `windowsHide`, and the default Bonjour/@homebridge/ciao discovery path remains unpatched.

## Best Possible Solution

Keep the issue open and do a Windows-focused fix pass. Identify the exact process creating the flashing console window, add `windowsHide: true` to any OpenClaw-owned background spawn surfaces that can run on Windows, and if the source is Bonjour/@homebridge/ciao ARP probing, upgrade or patch that dependency path. Add focused regression coverage for spawn options where mockable, and treat `OPENCLAW_DISABLE_BONJOUR=1`, `openclaw plugins disable bonjour`, or `discovery.mdns.mode="off"` only as mitigations.
