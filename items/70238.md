## Summary

Keep open. Current main mitigates the specific CLI respawn path on Windows, but the visible command-window flicker is not fully fixed: TUI relaunch still spawns without `windowsHide`, bundled Bonjour discovery remains default-enabled, and the shipped `@homebridge/ciao` path still uses Windows `arp` through `childProcess.exec` on a 15s polling path without a hidden-window option. 

## Best Possible Solution

Keep the issue open unless a maintainer manually consolidates it with #71188. The durable fix should cover all Windows-visible spawn paths: add `windowsHide` where OpenClaw owns child process creation, and address the Bonjour/@homebridge/ciao Windows interface polling path so LAN discovery can stay enabled without visible `cmd.exe` flashes. Add focused regression coverage for spawn options and the Windows discovery policy.
