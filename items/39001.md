## Summary

Keep open. Current main still expands Feishu merge_forward API items by rendering only each sub-message body; it does not prepend item.sender.id or consume item.mentions to restore @_user_N placeholders. The PR addresses a live Feishu channel behavior gap, and the current test suite still asserts the old output.

## Best Possible Solution

Continue tracking this PR or a maintainer-owned replacement until the Feishu plugin implements merge_forward sender preservation and mention restoration in extensions/feishu/src/bot-content.ts, with targeted tests for sender IDs, object/string mention IDs, union_id fallback, dollar-sign display names, and @_user_1/@_user_10 replacement collisions.
