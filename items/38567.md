## Close Comment

Closing as implemented on `main` after Codex review. The current `session_status` path resolves the persisted runtime model via `resolveSessionModelIdentityRef(...)` and feeds that into the status card when there is no explicit session override (`src/agents/tools/session-status-tool.ts:510`). The visible `🧠 Model:` line then renders that resolved selection (`src/status/status-message.ts:851`). There is also direct regression coverage asserting that `session_status` uses the runtime session model as the selected card model when no override is set (`src/agents/openclaw-tools.session-status.test.ts:796`). This behavior is documented in the changelog for `v2026.3.28` (`CHANGELOG.md:1979`). Since the report was against `2026.3.2`, this appears to have been fixed after that release.
