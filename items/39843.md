## Summary

Keep open. Current main still has the Google Chat orphaned typing-message path for silent `NO_REPLY`: the monitor creates a real typing message before dispatch, but only clears `typingMessageName` inside the delivery callback. Exact silent finals can be skipped before delivery, and Google Chat still does not wire a cleanup hook or post-dispatch deletion. The open related PR #39891 remains the concrete fix path rather than evidence that the bug is shipped.

## Best Possible Solution

Keep #39843 open until PR #39891 or an equivalent Google Chat monitor fix lands. The fix should delete any remaining `typingMessageName` after `dispatchReplyWithBufferedBlockDispatcher` completes without delivery, or wire the existing `onTypingCleanup` path, and add a focused regression test for the Google Chat emoji reaction plus exact `NO_REPLY` case.
