## Summary

Keep open. Current main has a Feishu card note/footer, but it is still fixed to agent identity plus selected model/provider. I found no `channels.feishu.cardFooter`, dynamic `[[card_footer:...]]` directive, or built-in context/compaction toggle, and the generic `/usage` footer is appended to reply text rather than exposed as configurable Feishu card footer metadata.

## Best Possible Solution

Keep the issue open as a concrete Feishu plugin feature request. The best path is to add a Feishu-owned config surface, likely under `channels.feishu` and per-account config, that can append selected safe footer fields such as context usage and compaction count to `resolveCardNote`, with tests for streaming and non-streaming cards plus Feishu docs updates. If maintainers want this across channels, first add a shared typed reply-metadata/footer hook rather than an agent text directive.

## Risks / Open Questions

- Dynamic agent-authored footer directives would need sanitization because Feishu cards render markdown/HTML-like markup.
- Context usage and compaction counts can reveal session metadata in shared chats, so any implementation should default off or be explicitly configured.
