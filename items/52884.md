## Summary

This request is an optional orchestration/memory layer that fits the repo’s plugin/ClawHub path, not OpenClaw core. Current main already provides core primitives for sub-agent orchestration and durable task flows, while the project vision explicitly rejects new default agent-hierarchy frameworks and heavy orchestration layers in core.

## Close Comment

Closing this as a Codex review because the request fits ClawHub/plugin space better than OpenClaw core. Current main already has the core primitives in place for this area: sub-agent orchestration in `docs/tools/subagents.md:10` and `docs/tools/subagents.md:160`, durable Task Flow orchestration in `docs/automation/taskflow.md:10`, and an optional structured subtask tool in `extensions/llm-task/index.ts:4`. At the same time, the project vision says optional capability should usually ship as plugins/skills (`VISION.md:54`) and explicitly says OpenClaw will not merge agent-hierarchy frameworks or heavy orchestration layers that duplicate existing agent/tool infrastructure (`VISION.md:99`). Given that, this is better pursued as a ClawHub skill or plugin built on the existing task/subagent/memory primitives rather than as a new core architecture layer.
