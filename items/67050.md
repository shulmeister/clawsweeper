## Summary

Keep open. Current main has not implemented the PR’s requested false-positive fix, and the linked bug #66840 remains a real scanner behavior gap. The submitted patch is also not safe to auto-close as implemented-on-main or merge as-is: its blanket test-path/name skip would create the scanner bypass called out in the existing P1 review comments.

## Best Possible Solution

Keep the PR open for maintainer/contributor follow-up, but do not merge the blanket skip. The best fix for #66840 is a narrower scanner change that preserves scanning for runtime-capable files, treats genuine non-runtime test findings as non-blocking or lower severity only when justified, and adds regression tests for both the reported test mock false positive and a malicious `*.test.*`/`tests/` bypass case.
