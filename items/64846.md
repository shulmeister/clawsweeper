## Summary

Keep open. Current main still hard-rejects `sessions_send` calls that include both `sessionKey` and `label`, which is the live behavior this PR attempts to fix for the linked bug report #64699. The PR head is still unmerged, and the related context shows overlapping open implementation attempts but no merged or maintainer-selected canonical replacement.

## Best Possible Solution

Keep this PR open as a valid implementation candidate for #64699, then consolidate it with the overlapping open `sessions_send` PRs. The best fix belongs in `src/agents/tools/sessions-send-tool.ts`: allow equal trimmed `sessionKey` and `label` to route by `sessionKey`, keep rejecting differing values, and add focused regression tests in `src/agents/tools/sessions.test.ts` or the gateway `sessions_send` tests before closing the linked bug.
