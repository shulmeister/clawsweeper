## Summary

Keep #38573 open. Current main still forces resolved thread/topic cron deliveries through direct outbound delivery, and the cron data model, gateway protocol, agent tool, CLI/UI, docs, and tests still do not expose an opt-in route for text-only topic/forum cron completions to pass through the main-session announce/agent-turn flow first. The related #34160-style subagent announce path exists for threaded completions, but this cron-specific direct-delivery override remains.

## Best Possible Solution

Keep this open for maintainer product/API work. A complete implementation should add a backward-compatible cron delivery route option, preserve direct delivery as the default, limit the opt-in to text-only topic/thread completions, keep structured payloads on direct delivery, define fail-closed versus direct-fallback behavior, update cron normalization/store/patch handling plus gateway protocol/tool/CLI/UI/docs surfaces, and add regression coverage proving an opted-in Telegram forum-topic cron completion reaches the main-session announce/agent-turn path before posting.
