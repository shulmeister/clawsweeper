## Summary

Keep #38573 open. Current main still forces any resolved thread/topic cron delivery through direct outbound delivery, and the cron type, gateway protocol, tool, UI, and docs surfaces still do not expose an opt-in route for text-only topic/forum completions to run through the main-session announce/agent-turn path first.

## Best Possible Solution

Keep this open for maintainer product/API work. A complete fix should add a backward-compatible cron delivery route option across CronDelivery, normalization/store handling, gateway protocol schemas, cron tool/CLI, Control UI/macOS models if exposed there, and docs; preserve direct delivery as the default; keep structured payloads on direct delivery; limit the opt-in to text-only completions; define fail-closed versus direct-fallback semantics; and add regression coverage proving a Telegram topic opt-in reaches the main-session announce/agent-turn path before posting.
