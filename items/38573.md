## Summary

Keep #38573 open. Current main still has no per-job or global option that lets text-only isolated cron deliveries to topic/forum/thread targets use a main-session announce/agent-turn route before posting. Threaded cron targets are still forced through direct outbound delivery, and the public cron type/schema/tool/docs surfaces only expose the existing delivery mode, target, account, best-effort, and failure-destination fields. Related #34160 is useful context for subagent announce behavior, but it does not supersede this cron-specific route-selection request.

## Best Possible Solution

Keep this issue open until maintainers either implement an explicit cron delivery route option or make a clear product decision not to support it. A complete implementation would likely add a backward-compatible route field to `CronDelivery`, gateway schema, cron tool schema, config/docs, and delivery dispatch; keep direct delivery as the default for structured payloads and current topic behavior; limit the opt-in to text-only completions; define fail-closed versus direct-fallback semantics; and add regression tests proving a Telegram topic opt-in reaches the main-session announce/agent-turn path before posting.
