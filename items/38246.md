## Summary

Keep #38246 open. Current main still sends visible WhatsApp auto-replies through the inbound message's reply/sendMedia closures, so an allowlisted non-owner DM can still receive a plain model reply. The main-DM pinning code only protects stored lastRoute metadata and does not block, reroute, or require explicit targets for visible replies.

## Best Possible Solution

Keep the issue open and implement a WhatsApp plugin routing guard in the visible auto-reply delivery path before deliverWebReply sends payloads. Add typed config/schema/docs for the chosen policy and regression coverage for owner DMs, allowlisted non-owner DMs where plain replies are blocked or rerouted, and explicit message-tool sends that still work intentionally.
