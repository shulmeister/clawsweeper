## Summary

Current main still routes visible WhatsApp auto-replies to the inbound DM sender through the original message reply/sendMedia functions. The owner-vs-third-party last-route pinning only protects stored route metadata, and current config/docs/tests do not provide a force-explicit-target guard for allowlisted non-owner DMs.

## Best Possible Solution

Keep #38246 open. Add a WhatsApp plugin routing guard in the auto-reply delivery path before `deliverWebReply` sends visible payloads, backed by config schema/types/docs and regression tests for owner DMs, allowlisted non-owner DMs, and explicit message-tool sends.
