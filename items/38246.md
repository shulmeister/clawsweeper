## Summary

Keep open. Current main still does not implement the requested WhatsApp DM safeguard that blocks or owner-reroutes plain assistant replies when the inbound sender is a third-party/non-owner contact. Existing protections cover related but narrower cases: lastRoute pinning, explicit target checks for the `message` tool, and broad session sendPolicy suppression, none of which provide the requested owner-aware conditional guard for WhatsApp plain auto-replies.

## Best Possible Solution

Keep tracking this as a WhatsApp/plugin-owned routing-safety feature until maintainers choose and implement the product behavior. The likely fix belongs in the WhatsApp auto-reply dispatch/delivery path or its channel config: add an owner-aware option that detects a non-owner DM sender before plain reply delivery, then either blocks visible delivery with a clear internal/logged reason or routes owner notifications through an explicit owner target. Add focused tests covering owner DM, third-party allowlisted DM, message-tool explicit sends, block/final streaming payloads, and interaction with existing lastRoute pinning and session.sendPolicy.
