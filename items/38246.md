## Summary

Keep open. Current main still routes normal WhatsApp auto-reply text back to the inbound sender, and the existing safeguards only cover lastRoute pinning or explicit message-tool sends. The requested configurable block/reroute for non-owner DM plain replies is not implemented.

## Best Possible Solution

Keep #38246 open. The best product path is to add a WhatsApp/channel-level routing guard in the auto-reply delivery path, backed by schema/config docs and tests for owner DM, non-owner allowlisted DM, and explicit message-tool sends. If maintainers prefer a different behavior, capture that explicit product decision on this issue before closing.
