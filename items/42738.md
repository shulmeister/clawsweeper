## Summary

Current `main` no longer supports the Chrome extension relay / `driver: "extension"` path that this PR patches. The reported aria-snapshot fallback case is obsolete against today’s browser architecture, which migrated to `existing-session` / Chrome MCP instead.

## Evidence

- **Release note removed the target surface:** The changelog marks the legacy Chrome extension relay path, bundled extension assets, `driver: "extension"`, and `browser.relayBindHost` as removed in release `2026.3.22`.
  - file: [CHANGELOG.md:2193](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/CHANGELOG.md#L2193)
  - command: `nl -ba CHANGELOG.md | sed -n '2178,2202p'`
- **Runtime lifecycle stub proves relay path is gone:** `ensureExtensionRelayForProfiles` is now an intentional no-op because the Chrome extension relay path has been removed.
  - file: [extensions/browser/src/browser/server-lifecycle.ts:13](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/extensions/browser/src/browser/server-lifecycle.ts#L13)
  - command: `nl -ba extensions/browser/src/browser/server-lifecycle.ts | sed -n '1,80p'`

  - file: [extensions/browser/src/browser/routes/basic.ts:202](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/extensions/browser/src/browser/routes/basic.ts#L202)

## Close Comment

Closing after Codex review. Current `main` no longer has the legacy Chrome extension relay surface this PR targets: `CHANGELOG.md:2193` records that the extension relay path and `driver: "extension"` were removed in `v2026.3.22`, `extensions/browser/src/browser/server-lifecycle.ts:13` makes the relay setup helper a no-op, and `extensions/browser/src/browser/routes/basic.ts:202` now rejects `driver: "extension"` outright. The migration path is documented in `docs/gateway/doctor.md:190` and `:210`, which move old extension configs to `existing-session` / Chrome MCP. Because the patched code path is no longer a supported surface on current `main`, this PR no longer matches a reproducible maintainer problem in OpenClaw today. If there is still an aria-snapshot failure on the current `existing-session` browser path, please open a new PR or issue against that supported flow.
