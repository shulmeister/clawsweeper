## Summary

Keep open. Current `main` still decides Feishu group-vs-DM routing purely from `chat_type === "group"`, so a topic-group event delivered as `chat_type: "p2p"` would still follow the DM path and miss group routing. I found no implementation on `main` that normalizes `oc_` topic chats into group handling, and the existing tests only cover topic routing when `chat_type` is already `"group"`.

## Evidence
