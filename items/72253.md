## Summary

Keep PR #72253 open. Current main still treats structured `Exec completed (..., code 0)` and `Exec failed (..., signal SIGTERM)` as relayable exec completion events, and the PR is a recent focused fix for linked issue #72252 rather than obsolete or implemented on main.

## Best Possible Solution

Keep #72253 open for maintainer review. The best path is a narrow core heartbeat fix that marks trusted/local structured `code 0` and cleanup `signal SIGTERM` exec completions internal, preserves relay for actionable failures like `code 1`, and coordinates with the broader #69492 audience-classification work without waiting for that umbrella design to solve the immediate #72252 chat-noise bug.
