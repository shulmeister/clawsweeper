## Summary

Current `main` and the shipped `v2026.4.23` release already implement the relevant config/runtime/docs path: `tools.exec.strictInlineEval` is a recognized exec setting, it defaults to `false`, and inline-eval approval denial is only applied when that flag is explicitly enabled. The reported `Unrecognized key` / always-blocked behavior does not match current OpenClaw.

## Evidence

- **Released docs say `security=full` + `ask=off` has no extra obfuscation prefilter, and `strictInlineEval` defaults to false:** `docs/tools/exec.md` in release `v2026.4.23` explicitly documents that host exec in `security=full` plus `ask=off` follows configured policy directly with no extra heuristic command-obfuscation or script-preflight rejection layer, and that `tools.exec.strictInlineEval` defaults to `false`.
