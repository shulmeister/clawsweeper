## Summary

Keep #39461 open. Current main still hard-codes the QQ Bot data directory under the user home directory, while diagnostics still recommends `QQBOT_DATA_DIR`; there is no implementation, docs, or test coverage for that environment variable.

## Best Possible Solution

Implement a real QQ Bot data-root override in `extensions/qqbot/src/engine/utils/platform.ts`, preferably honoring `QQBOT_DATA_DIR` in `getQQBotDataPath()` so all existing session/data/ref/download call sites inherit it. Add focused tests for the env override and update `docs/channels/qqbot.md` or the diagnostic warning to document the supported behavior.
