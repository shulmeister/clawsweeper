## Summary

Current main still ignores `QQBOT_DATA_DIR` for QQ Bot storage while diagnostics recommends it, so the reported Windows path issue remains actionable.

## Best Possible Solution

Keep the issue open. Implement `QQBOT_DATA_DIR` support in the central QQ Bot storage helper, likely `extensions/qqbot/src/engine/utils/platform.ts:getQQBotDataPath()`, preserving lazy directory creation and path normalization. Add focused tests for the env override plus diagnostics behavior, then document `QQBOT_DATA_DIR` on https://docs.openclaw.ai/channels/qqbot or remove the diagnostic recommendation if maintainers choose not to support it.
