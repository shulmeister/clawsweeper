## Summary

Keep #39461 open. Current main still does not honor `QQBOT_DATA_DIR`; QQ Bot data storage is hard-coded under the user home directory while diagnostics still recommends the unsupported variable.

## Best Possible Solution

Implement the storage-root override in the QQ Bot plugin, likely in `extensions/qqbot/src/engine/utils/platform.ts:getQQBotDataPath()`, honoring `process.env.QQBOT_DATA_DIR` after validation/normalization so all storage call sites inherit it. Add focused tests for the env override and diagnostics output, and either document `QQBOT_DATA_DIR` on https://docs.openclaw.ai/channels/qqbot or remove the diagnostic recommendation if maintainers decide not to support it.
