## Summary

Keep #39461 open. Current main still ignores `QQBOT_DATA_DIR`: QQ Bot storage paths are hard-coded under `~/.openclaw/qqbot`, while diagnostics still tells Windows users to set that unsupported variable.

## Best Possible Solution

Implement `QQBOT_DATA_DIR` support at the central QQ Bot storage helper, likely `extensions/qqbot/src/engine/utils/platform.ts:getQQBotDataPath()`, with normalization/validation and preserved lazy directory creation. Add focused tests for the env override and diagnostics output, and either document the variable on `https://docs.openclaw.ai/channels/qqbot` or remove the diagnostic recommendation if maintainers decide not to support it.
