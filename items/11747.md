## Summary

Keep #11747 open. Current main still treats Discord, Telegram, and Slack reactions as queued system events/context, not as immediate agent-turn triggers; no proposed auto-response/wake config exists. The related Slack PR fixed bound-session routing only and explicitly left reaction notification semantics unchanged, while #52658 tracks a narrower missing plugin-event surface rather than superseding this broader auto-response request.
