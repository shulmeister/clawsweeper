## Summary

Keep #11747 open. Current main still exposes only passive reaction notification modes and Discord, Telegram, and Slack reaction handlers enqueue routed system events without requesting a heartbeat or dispatching an immediate agent turn. The newer related #38714 is narrower Discord plugin-hook work, and closed #60805 cites a blocked plugin workaround but does not ship or supersede the requested cross-channel reaction wake behavior.

## Best Possible Solution

Keep #11747 open for maintainer product/API design. The likely best path is a documented cross-channel reaction wake option, such as reactionAutoRespond or reactionTriggersTurn, that preserves existing passive system-event behavior while allowing authorized reactions to trigger an immediate routed agent turn. If maintainers prefer plugin-first automation, core still needs a reaction event hook and/or wake API before this can move cleanly to ClawHub/plugin implementations; #38714 can cover the narrower Discord hook piece.
