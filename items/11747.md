## Summary

Keep #11747 open. Current main still treats Discord, Telegram, and Slack reactions as reactionNotifications-gated system events that are drained into later prompt context, not as immediate agent-turn triggers. I found no shipped reactionAutoRespond/reactionTriggersTurn/reactionWake or auto mode, and the related #52658/#34045 context does not supersede the broader cross-channel auto-response request.

## Best Possible Solution

Keep #11747 open until maintainers choose and implement a cross-channel reaction wake contract. The likely product path is a documented config option such as reactionAutoRespond or reactionTriggersTurn, with Discord/Telegram/Slack reaction handlers promoting authorized reactions into an immediate routed agent turn and tests proving the wake behavior. If maintainers want this solved by plugins instead, a broader plugin reaction event/wake API should land first, with #52658 as the narrower Discord-facing part of that work.
