## Summary

Keep #11747 open. Current main still implements Discord, Telegram, and Slack reaction notifications as routed session system events only; I found no config or channel/runtime path that promotes a reaction into an immediate agent turn. The maintainer comment on 2026-04-26 already reached the same conclusion, and current main at 04e08cea62983b2c766d15f61f15b3ffc90f9baa still matches that evidence.

## Best Possible Solution

Keep #11747 open until maintainers choose a cross-channel reaction wake contract. A likely implementation is a documented config option such as reactionAutoRespond or reactionTriggersTurn, with Discord, Telegram, and Slack reaction handlers promoting authorized reactions into an immediate routed agent turn and tests proving both the existing system-event context behavior and the new wake behavior. If maintainers prefer a plugin-first solution, the missing core work should be a plugin/channel reaction event and wake API, with #52658 or a broader canonical issue tracking that API surface.
