## Summary

Keep #11747 open. Current main still implements Discord, Telegram, and Slack reactions as authorized system-event enqueue paths only, with no reactionAutoRespond/reactionTriggersTurn-style config, no reaction-specific plugin hook, and no reaction path that dispatches an immediate agent turn. The issue discussion also shows this is broader than Discord: commenters reproduced the same passive-reaction behavior for Telegram and Slack, while related #34045 only fixed Slack system-event routing and #38714 remains the narrower open Discord hook/API tracker.

## Best Possible Solution

Keep #11747 open for maintainer product/API design. The best path is a documented cross-channel reaction wake option such as reactionAutoRespond or reactionTriggersTurn that preserves current passive system-event behavior while allowing authorized reactions to trigger an immediate routed agent turn, optionally scoped by emoji and policy. If maintainers prefer plugin-first automation, core still needs a reaction event hook and/or safe wake/dispatch API first; #38714 can continue tracking the narrower Discord hook surface.
