## Summary

Keep this issue open. Current main still has the Control UI model-switch/send ordering race: the dropdown optimistically updates local model selection and awaits `sessions.patch`, but no pending switch state is stored, the send path does not await it, and `chat.send` does not carry a model override that would make the next turn independent of the session patch completing first.

## Best Possible Solution

Implement the Control UI ordering guard proposed in the issue: store the pending model-switch operation in UI state, disable the model picker and send while it is pending, make `handleSendChat` await the pending switch before `chat.send`, roll back and do not send if `sessions.patch` fails, and add a regression test that changes the model with unresolved `sessions.patch` and immediately attempts to send.
