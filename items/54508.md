## Summary

Keep this issue open. Current main still has useful external-content wrapping, limited input provenance, plugin tool hooks, tool-result persistence hooks, and exec approval controls, but it does not implement the requested persistent taint/provenance graph, default content-derived execution prohibition, general exec risk classes, or provenance-aware secret/outbound enforcement. The linked related issues are narrower security threads, several still open, and do not fully supersede this umbrella.

## Best Possible Solution

Keep this issue open as the core security architecture umbrella unless maintainers explicitly split it into canonical implementation issues. A defensible implementation path is to carry provenance/taint metadata through external content, tool results, summaries, persistence, and future reads; enrich before_tool_call and approval paths with stable action/risk/provenance fields; and enforce content-derived execution, secret access, and outbound-action policy in core while linking narrower follow-ups such as #54373, #48519, #48503, #41140, #47492, and #53185.
