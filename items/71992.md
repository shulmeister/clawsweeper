## Summary

Current main still contains the reported Control UI duplicate-render race. In `handleChatEvent`, both `final` and `aborted` paths append to `chatMessages` before clearing `chatStream`/`chatRunId`, while the renderer still emits a live stream item whenever `chatStream` is non-null. The linked PR #72004 is the right narrow fix candidate but is still open/unmerged, so this issue should remain open as the canonical bug tracker until that fix or an equivalent lands.

## Best Possible Solution

Keep #71992 open until PR #72004 or an equivalent narrow Control UI fix lands. The likely fix belongs in `ui/src/ui/controllers/chat.ts`: snapshot any fallback stream text, clear `chatStream`/`chatRunId`/`chatStreamStartedAt` before mutating `chatMessages` in both `final` and `aborted`, and add a regression test that observes `chatStream` at the exact `chatMessages` mutation point. The broader persisted-transcript duplicate observations from the comments should remain coordinated with #69208 rather than being treated as proof this UI race is fixed.
