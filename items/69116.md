## Summary

Close as ClawHub/third-party plugin scope. Current main already ships first-party GitHub Copilot support through the `github-copilot` provider and the `copilot-proxy` local bridge, while the PR adds a second optional SDK/CLI-backed Copilot provider that the author says is already structured against public plugin SDK contracts and can be ported mechanically. The repo vision and plugin docs point optional plugin promotion/discovery to ClawHub unless a missing core API is identified, and this PR does not show such a missing API.

## Best Possible Solution

Close this core repository PR and route the `@github/copilot-sdk` backend to a ClawHub/npm provider plugin. Keep the shipped `github-copilot` provider and `copilot-proxy` bridge as the core Copilot paths; open a focused core issue only if the external SDK-backed plugin uncovers a concrete missing plugin SDK seam.
