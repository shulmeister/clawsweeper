## Summary

Close as ClawHub/third-party plugin scope. The PR adds a second optional Copilot transport backed by `@github/copilot-sdk`; current `main` already ships first-party Copilot paths through `github-copilot` and `copilot-proxy`, and the PR author says the implementation is already structured against public plugin SDK contracts and can be ported mechanically. The repo vision and plugin docs route optional provider/plugin distribution through ClawHub unless a missing core API is identified, and this review did not find such a blocker.

## Best Possible Solution

Close this core-repository PR and route the `@github/copilot-sdk` backend to a ClawHub/npm provider plugin. Keep the shipped `github-copilot` provider and `copilot-proxy` bridge as the first-party Copilot paths, and open a focused core issue only if the external port uncovers a concrete missing plugin SDK seam.
