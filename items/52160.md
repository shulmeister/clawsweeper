## Summary

Current `main` already implements the requested SDK-alias fallback for external plugins started outside the package root, with broader argv/moduleUrl handling than this PR and regression coverage in current tests.

## Close Comment

Closing from a Codex review because current `main` already covers this behavior. `buildPluginLoaderAliasMap()` now defaults to the startup argv hint and feeds it into SDK alias resolution (`src/plugins/sdk-alias.ts:599`), and `resolveLoaderPluginSdkPackageRoot()` already resolves trusted OpenClaw roots from `argv1` or `moduleUrl` (`src/plugins/sdk-alias.ts:89`, `src/plugins/sdk-alias.ts:143`). The current test suite also has explicit regressions for user-installed plugins resolving `openclaw/plugin-sdk/*` via the running OpenClaw argv hint and via `moduleUrl` (`src/plugins/sdk-alias.test.ts:751`, `src/plugins/sdk-alias.test.ts:772`). On top of that, current main already supports the launch shapes called out during review, including `dist/entry.js` and `scripts/run-node.mjs` (`src/infra/gateway-process-argv.ts:46`). This makes the PR obsolete relative to `main`.
