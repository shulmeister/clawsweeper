## Summary

Current main only partially implements #30389. WebChat/Control UI now has an attachment button and a per-session model picker, but the upload path is still image-only and there is still no visible chat-composer skill/capability picker for finance/coding/writing/image/reasoning-style routing. This remains actionable product work, not a conservative close.

## Best Possible Solution

Keep #30389 open as the umbrella WebChat UX request, or split it into narrower maintainer-owned issues. The remaining implementation should add non-image document/file upload across the picker, validation, preview, chat.send serialization, gateway parsing/offload/extraction, model/provider handling, and docs; separately define a visible composer skill/capability picker that maps cleanly to skills, tools, agents, or model/task profiles instead of relying only on typed slash commands.
