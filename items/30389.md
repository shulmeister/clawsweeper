## Summary

Current main implements two major parts of #30389: WebChat/Control UI file attachment UI and a chat-header model selector. The issue should stay open because the requested composer-visible skill/capability menu is still not implemented; current coverage is limited to slash-command completion, command palette entries, and separate Agents/Skills/Tools panels.

## Best Possible Solution

Keep #30389 open unless maintainers intentionally split it. Preserve the shipped attachment button, non-video file upload path, and model/thinking selectors; use #30759 for any remaining document-upload specifics; then define a dedicated composer-visible skill/capability picker backed by current runtime data such as commands.list, skills.status, tools.effective, agents, model profiles, or task profiles.
