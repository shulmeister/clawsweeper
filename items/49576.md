## Summary

Current `main` contains two matching fixes for this report: config reload now diffs source-authored config instead of runtime-materialized plugin/provider state, which avoids false restart triggers from plugin bookkeeping writes, and gateway restarts are now deferred until active operations/replies drain. The shipped changelog places the reload fix in `v2026.4.22`.

## Close Comment

Closing this as fixed after a Codex review of current `main`. OpenClaw `v2026.4.22` shipped a config-reload fix specifically for false gateway restarts caused by plugin/provider-derived config writes (`CHANGELOG.md:193`). Current main also defers gateway restarts until active operations, pending replies, embedded runs, and tasks finish (`src/gateway/server-reload-handlers.ts:201`, `src/gateway/server.impl.ts:314`), with regression coverage in `src/gateway/server-restart-deferral.test.ts:39`. That matches the crash pattern described here: Feishu message processing was active, `plugins.reload` fired, and the gateway then exited. If this still reproduces on `v2026.4.22` or newer, please open a fresh issue with the exact gateway log around the reload and the config paths that changed so maintainers can treat it as a new regression.
