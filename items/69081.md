## Summary

Keep open. Current main still does not implement the two concrete fixes requested for #69081: active tool events do not rearm the TUI streaming watchdog, and reconnect handling reloads history without first clearing the stale active run/watchdog state. The related PR #69125 proposed exactly those changes, but the provided GitHub context shows it was closed unmerged, and the current source still matches the reported failure mode.

## Best Possible Solution

Implement the narrow TUI fix in current main: rearm the streaming watchdog for active-run `stream: "tool"` agent events before verbose filtering can drop proof-of-life, and expose a reconnect/reset path that clears the active run id and watchdog before `loadHistory()` rehydrates the session. Add focused regression coverage for long tool-only activity and reconnect after a lost final event. Keep #69081 open until that lands; #69978 can remain a separate UX/configuration follow-up for watchdog controls and duplicate warning suppression.
