## Summary

Keep #69081 open. Current main still has the two reported TUI state-management gaps: the streaming watchdog is armed only from chat deltas, not active tool events, and reconnect handling reloads history without clearing stale active-run/watchdog state. The linked fix PR #69125 was closed unmerged in the provided GitHub context, and recent user comments report the behavior is still disruptive.

## Best Possible Solution

Keep the issue open and land a focused TUI fix: rearm the streaming watchdog for active tool events before verbosity-based early returns, add a reconnect reset path that clears `activeChatRunId` and the pending watchdog before history reload, and add fake-timer regression coverage for long active tool phases plus reconnect-after-missed-final recovery. PR #69125 is a useful prior implementation reference but should be revalidated against current main before landing.
