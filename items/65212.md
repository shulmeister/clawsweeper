## Summary

Keep open. Current main still treats every custom QMD path as the collection root with a default or user-supplied pattern, so a config entry that points directly at an existing Markdown file is not normalized into its parent directory plus exact filename. The PR is a plausible focused fix, but it still needs maintainer review for literal filename pattern handling and source/package mirror alignment.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a focused resolver change that detects existing file targets, uses the parent directory as the QMD collection root, uses a literal exact-filename pattern or mask, preserves existing directory behavior, and adds regression coverage for normal filenames and glob-metacharacter filenames. Confirm whether the `packages/memory-host-sdk` mirror should be updated in the same change.

## Risks / Open Questions

- The proposed `path.basename(resolved)` value is passed as a QMD pattern/mask; filenames containing glob metacharacters such as `[]`, `*`, or `?` may need literal escaping before the PR is safe to land.
- The mirrored private package source under `packages/memory-host-sdk/src/host/backend-config.ts` currently has the same old resolver shape, so maintainers should confirm whether this PR must update both copies or only the canonical `src/` copy.
