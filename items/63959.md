## Summary

Keep this PR open. Current main still lacks the proposed Dockerfile change: the runtime image switches to the non-root `node` user without pre-creating `/home/node/.openclaw`, while config writes still create temp files under the resolved state/config directory. The linked bug path remains plausible for named-volume deployments outside the repo setup script's bind-mount chown flow.

## Best Possible Solution

Keep the PR open as the implementation candidate for #61279. The best path is to rebase/review it, keep the image-level `install -d -o node -g node -m 0755 /home/node/.openclaw` before `USER node`, preserve the changelog entry if it lands in the active release section, and optionally document that already-created root-owned volumes still need a one-time chown.
