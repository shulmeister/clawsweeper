## Summary

Keep #32868 open. Current main still has the same-channel block-streaming delivery gap: the block pipeline can wait for the `onBlockReply` callback, but that callback only enqueues same-channel dispatcher delivery and returns before `options.deliver()` settles. The related WeCom plugin workaround and duplicate WhatsApp report support the impact, but they do not replace a core awaitable delivery contract.

## Best Possible Solution

Keep #32868 open as the canonical core bug. The maintainer path is to make same-channel block delivery awaitable, wait for that delivery before tool execution proceeds under `blockStreamingBreak: "text_end"`, update dispatcher/plugin SDK/ACP call sites and mocks consistently, and add a regression test proving actual channel delivery completes before tool execution starts.
