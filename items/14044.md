## Summary

The issue is asking about integrating OpenClaw with a Solana/on-chain program. Current main treats that as external plugin/MCP work, not core roadmap work, and already has plugin docs plus a starter example. This belongs in ClawHub/community plugin space rather than staying open as a core issue.

## Close Comment

Closing this as a ClawHub/plugin-scope request after Codex review. Current `main` treats this kind of integration as external extension work, not core roadmap work: `VISION.md:54-61` says optional capability should usually ship as plugins, `VISION.md:72-83` points integration-style extensions to the MCP bridge path, and `docs/plugins/building-plugins.md:47-170` already includes a starter plugin guide with an example `registerTool(...)` implementation. For a Solana/on-chain integration, the supported path is an external plugin or MCP-backed integration published on ClawHub/npm rather than a core OpenClaw issue. If you build this out and hit a specific SDK or MCP gap in current `main`, open a new focused issue with the exact missing seam.
