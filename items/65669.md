## Summary

Keep open. Current `main` still lacks custom cron job ID support end-to-end, so this PR addresses a real, unshipped gap rather than an obsolete request. The review discussion also surfaced concrete protocol/runtime defects and the later PR commits appear to address them, which makes this useful work even if it likely needs cleanup before merge.
