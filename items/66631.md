## Summary

Keep open. Current main still routes Feishu topic sessions by `thread_id` when `root_id` is missing, but reply delivery still falls back to the triggering `message_id` instead of resolving the topic root `om_*` message. That matches the reported direct-message-in-topic failure mode, and the existing tests cover routing/root_id-present replies but do not cover the missing-root reply target fix.

## Best Possible Solution

Keep #66631 open as the actionable Feishu bundled-plugin bug. The likely fix belongs in `extensions/feishu/src/bot.ts`/`send.ts`: when handling a topic-mode or reply-in-thread group event with `thread_id` but no `root_id`/`parent_id`, list or fetch the thread root `om_*` message, use it as `replyToMessageId`/`rootId`, and add focused tests for both normal reply and error/ACP failure reply paths while preserving normal-group behavior.
