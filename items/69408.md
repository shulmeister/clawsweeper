## Summary

Keep issue #69408 open. Current main still emits the reported `[agents/loop-detection] Loop warning: ...` at warn level for generic repeated tool calls, and the loop-detection hook does not receive isolated-cron target, run lifetime, or cron interval metadata needed to implement the requested downgrade.

## Best Possible Solution

Keep the issue open and implement a narrow loop-detection context path: propagate cron trigger/job/sessionTarget and bounded run timing into the before-tool-call hook or detector, downgrade only non-critical warning logs for bounded isolated cron runs to info, retain warning/error behavior for persistent or interactive sessions, and add focused tests proving the `[agents/loop-detection]` log level changes only for the isolated-cron case.
