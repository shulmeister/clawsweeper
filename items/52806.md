## Summary

Current `main` already covers the reported cron behavior: isolated cron runs force a fresh session, manual runs execute through the cron lane and emit finished events with session metadata, and gateway cron run logs are appended/read by `cron.runs`. The Asia/Shanghai `nextAt` complaint is also addressed by the cron schedule rollback guard and regression tests. The same implementation is present in release `v2026.4.22`.
