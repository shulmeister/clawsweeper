## Summary

Keep open. Current main still has no fresh-session CLI flag/default or resume banner for `openclaw agent --agent <id>` without `--session-id`; the CLI and Gateway resolve that request to the agent main session key and reuse a fresh stored session id. Omitted delivery channel also still normalizes through `last`, and current tests/docs encode the existing behavior rather than the requested safer operator path.

## Best Possible Solution

Keep this issue open. Add an explicit fresh-session control for `openclaw agent`, wire it through `register.agent`, `agent-via-gateway`, `AgentParamsSchema`, the Gateway `agent` handler, and the local embedded `resolveSession` path, and print a clear stderr banner when `--agent` without `--session-id` reuses an existing session. Update `docs/cli/agent.md` and `docs/tools/agent-send.md`, with focused tests for omitted, blank, explicit, reused, and fresh-session cases.
