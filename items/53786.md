## Summary

Keep this issue open. Current main has generic live session-message plumbing, but it still does not implement the reporter's requested bidirectional, exact Feishu <-> Local Dashboard sync: Control UI runs as WebChat, normal dashboard sends are `deliver: false`, WebChat clients are intentionally blocked from inheriting external Feishu routes, and dashboard refresh is keyed to the currently selected session.

## Best Possible Solution

Keep this open for maintainer triage. Reproduce on current main with Feishu DM and group sessions, recording the active Control UI session key, Feishu route/session scope, gateway logs, and whether `session.message` arrives. If true bidirectional mirroring is desired, add an explicit opt-in Control UI delivery/mirror flow for Feishu-backed sessions plus tests for Feishu inbound-to-active-session reload and explicit dashboard-to-Feishu delivery. If the intended behavior is that Control UI is a session viewer/editor rather than an external chat mirror, update the WebChat/Feishu docs to state that boundary before closing.
