## Summary

Keep this issue open. Current main has addressed adjacent symptoms, including ACP task tracking and hook delivery suppression, but the central requested capability is still missing: channel-less persistent ACP spawns are still rejected unless `thread:true`, the public docs still preserve that requirement, the `sessions_spawn(runtime:"acp")` accepted result still does not expose ACP backend/agent session identity, and `/hooks/agent` still runs through the isolated agent path rather than directly routing known ACP session keys into ACP `runTurn`.

## Best Possible Solution

Keep this issue open and narrow it to a conservative headless ACP persistence contract. The likely implementation belongs in `src/agents/acp-spawn.ts`, the `sessions_spawn`/status tool contract, and ACP docs: add an explicit headless persistent ACP mode or narrowly relax the thread guard when `streamTo:"parent"` and cleanup semantics are clear; expose the ACP backend/agent session identity or a documented status lookup path; add regression coverage for channel-less hook-triggered ACP `mode:"session"` spawns; and separately decide whether `/hooks/agent` should route known ACP session keys into `acpManager.runTurn`.
