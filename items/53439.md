## Summary

Current main still has the reported Synology Chat webhook behavior: HEAD requests are rejected with 405, successful POST acknowledgements return 204 with no body, and tests still assert that old contract. The issue also has open closing PR candidates, including the same-author focused fix, so it should remain open until one is reviewed and landed or explicitly superseded.

## Best Possible Solution

Keep this issue open. Review and either land the focused same-author PR #53441 or intentionally supersede it with PR #53522. The code change belongs in extensions/synology-chat/src/webhook-handler.ts: accept HEAD probes with 200 and change successful POST ACKs from 204/no body to 200 with {"success":true}, with updated webhook and integration tests covering both behaviors.
