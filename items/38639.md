## Summary

Keep #38639 open. Current main still does not merge arbitrary `models.providers.<id>.models[]` entries into the shared `loadModelCatalog()` output, so the later gateway/media capability failure remains valid even if Telegram has some synthetic allowlist/default-model handling now. The linked #62991 is still the active unmerged fix rather than shipped evidence.

## Best Possible Solution

Keep the issue open until maintainers either merge #62991 or implement the same behavior on main: merge validated `models.providers.<id>.models[]` entries into `loadModelCatalog()` with dedupe and suppression rules, then add regression coverage for the shared catalog and the gateway image-capability path.
