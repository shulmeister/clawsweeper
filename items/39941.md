## Summary

Keep PR #39941 open. Current main still recurses into the first object-valued `anyOf`/`oneOf` variant during config form coercion, so the discriminator-aware object variant selection remains useful unmerged work.

## Best Possible Solution

Keep PR #39941 open. The best path is to land or rework the discriminator-aware fix in `ui/src/ui/controllers/config/form-coerce.ts`, preserving primitive union and array fallback behavior, and add `anyOf`/`oneOf` object-union regression tests in `ui/src/ui/controllers/config/form-utils.node.test.ts`, including the unconstrained-variant edge case raised in review.
