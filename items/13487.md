## Summary

Keep #13487 open. Current main still lacks a Discord-wide target-selection step for multi-bot channels: each account handler uses only its own bot user ID, resolves the agent route before mention/reply target facts, and the shared resolver can still fall back to the default agent. The supplied discussion also includes an April 26 maintainer-context review explicitly keeping this open and noting that related #39428/#39485 do not implement the requested mention > reply-target > default-owner suppression matrix.

## Best Possible Solution

Keep #13487 open until the Discord plugin makes one deterministic cross-account target decision before default routing: map configured Discord bot user IDs to owning OpenClaw accounts/agents, select explicit bot mentions first, then referenced-message author, and use the per-channel default owner only for untagged non-reply messages. Suppress all non-target/default responders whenever mention or reply targeting applies, and add regression tests for explicit mention, native reply, reply-plus-mention, and untagged default-owner cases.
