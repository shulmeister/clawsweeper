## Summary

Keep #13487 open. Current main still does not implement strict Discord multi-bot precedence for mention > reply-target > default owner. Explicit mentions and reply-to-bot detection are scoped to the currently running Discord bot account, default route fallback remains available, and the current tests/docs do not cover the requested suppression matrix. The GTZhou reproduction and the prior steipete review remain relevant; #39428/#39485 overlap with account routing but do not supersede this invariant.

## Best Possible Solution

Keep #13487 open until the Discord plugin implements a deterministic target decision before default routing: use Discord mention metadata to identify eligible OpenClaw bot accounts first, referenced-message author second, and only then fall back to the per-channel/default owner. Suppress non-target/default responders whenever a targeted mention or reply applies, and add regression coverage for explicit mention, native reply, reply-plus-mention, and untagged default-owner cases. #39428/#39485 can be useful account-routing building blocks but should not close this issue unless they cover this exact precedence matrix.
