## Summary

Current main still does not implement the requested strict Discord multi-bot precedence. The Discord preflight path detects explicit mentions and reply-to-bot only for the currently running bot account, and default routing remains available when no binding/target-specific route suppresses it. The supplied reproduction comment is still relevant, and the related account-routing items do not fully supersede the requested mention > reply-target > default-owner invariant.

## Best Possible Solution

Keep #13487 open until the Discord plugin implements a deterministic multi-bot routing target decision: derive eligible bot/account targets from Discord mention metadata first, referenced-message author second, and only then fall back to the channel/default owner; suppress non-target/default responders whenever a targeted mention or reply applies; add unit/integration tests for the three acceptance cases. If #39428/#39485 land first, use them as account-routing building blocks rather than closing this issue unless they explicitly cover this precedence matrix.
