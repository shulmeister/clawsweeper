## Summary

Keep #13487 open. Current main still performs Discord routing inside each account handler before using mention/reply metadata, and that metadata only gates the current bot user ID. There is still no cross-account Discord target-selection step that maps mentioned or replied-to bot IDs to the eligible OpenClaw account/agent and suppresses the default owner when a targeted bot applies. The latest maintainer/Codex review comment on the issue reached the same conclusion, and the relevant Discord routing files have not changed since that review SHA.

## Best Possible Solution

Keep #13487 open until the Discord plugin makes one deterministic cross-account target decision before default routing: map configured Discord bot user IDs to their OpenClaw account/agent, choose explicit bot mentions first, then referenced-message author, and use the per-channel default owner only for untagged non-reply messages. Suppress all non-target/default responders whenever mention or reply targeting applies, and add regression tests for explicit mention, native reply, reply-plus-mention, multiple mentioned bots, and untagged default-owner cases.
