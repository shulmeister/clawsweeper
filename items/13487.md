## Summary

Keep open. Current main has Discord mention and reply-to-bot gating for the currently running bot account, but it does not implement the requested strict multi-bot router precedence or mandatory suppression of the per-channel default owner when a different bot is explicitly mentioned or replied to. The issue discussion also includes a relevant reproduction where native Discord replies and reply-plus-mention did not visibly trigger the intended bot.

## Evidence

- **issue discussion reproduction:** GTZhou's 2026-04-02 comment reports a multi-bot guild-channel setup where plain explicit mention works, but a native Discord reply to a bot message and a reply with explicit mention both produce no visible reply. That is directly relevant to the requested reply-target precedence behavior.
