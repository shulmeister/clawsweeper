## Summary

Keep open. Current main still does not attach session-aware metadata to the final failover wrapper error log path. The wrapper summary now includes provider/model text in the message, but the outer stderr log line still emits only a plain string and the wrapper error type itself has no session_id field.

## Evidence

- **Wrapper error type has no session metadata:** `FallbackSummaryError` stores only `attempts` and `soonestCooldownExpiry`; there is no `sessionId`, `sessionKey`, or lane field to propagate into wrapper logging.
  - file: [src/agents/model-fallback.ts:45](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/agents/model-fallback.ts#L45)
  - command: `nl -ba src/agents/model-fallback.ts | sed -n '40,70p'`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Per-attempt records are provider/model only:** `FallbackAttempt` includes `provider`, `model`, `error`, `reason`, `status`, and `code`, but no session identifier.
  - file: [src/agents/model-fallback.types.ts:7](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/agents/model-fallback.types.ts#L7)
  - command: `sed -n '1,140p' src/agents/model-fallback.types.ts`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Wrapper summary message includes attempted providers only as text:** The fallback summary is formatted as `provider/model: error (reason)` entries joined into the thrown message. That helps human-readable provider attribution, but only inside the message string.
  - file: [src/agents/model-fallback.ts:917](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/agents/model-fallback.ts#L917)

  - file: [src/auto-reply/reply/agent-runner-execution.ts:1522](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/auto-reply/reply/agent-runner-execution.ts#L1522)

## Risks / Open Questions

- Closing this would discard a still-valid logging gap: the final wrapper error path cannot emit consistent session_id metadata from its current data model.
- Provider attribution is better than before only in free-text summaries; downstream structured ingestion still has to parse or correlate across separate log lines.
