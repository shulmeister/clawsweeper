## Summary

Current main still has the reported behavior: both uncaughtException handlers log, restore terminal state, and call process.exit(1) without checking isTransientNetworkError. The linked PR #63780 appears to track the remaining fix, while #60627 was closed unmerged, so the issue is not implemented on main yet.
