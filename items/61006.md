## Summary

Current main partially addresses the old synthetic replacement by including the original prompt in fallback retries, but it does not satisfy this issue's requested transcript behavior. For existing-session fallback retries, the helper still prepends a synthetic retry prefix, the unit test asserts that behavior, and the agent-command embedded runner path forwards that effective prompt without a transcriptPrompt override, so the transcript-clean rewrite seam is not used for this path.
