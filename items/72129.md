## Summary

Keep #72129 open. Current main still only documents and registers Hugging Face for chat-completion model provider use; it does not provide a Hugging Face image-generation provider. The PR is a fresh, focused follow-up to the closed broader #63487 and the provided discussion shows the author addressed the concrete review findings on the PR head.

## Best Possible Solution

Keep the PR open for maintainer review and normal CI. If maintainers want Hugging Face text-to-image as part of the existing bundled huggingface plugin, the best path is the focused provider implementation with manifest metadata, docs, changelog, and tests.
