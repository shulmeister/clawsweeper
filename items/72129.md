## Summary

Keep PR #72129 open. Current main does not implement Hugging Face text-to-image generation: the bundled Hugging Face plugin only registers the chat provider, its manifest lacks an image-generation contract, and the public provider docs still say Hugging Face support is chat-completions only. The PR is a focused active follow-up to closed PR #63487, uses an existing plugin capability surface, and the author has already responded to review feedback, so it should remain available for maintainer review rather than be auto-closed.

## Best Possible Solution

Keep #72129 open for maintainer review as the focused path for adding Hugging Face text-to-image support to the existing bundled `huggingface` plugin. If maintainers accept the product scope, the best implementation direction is to keep it plugin-owned, use the existing provider auth and guarded provider HTTP helpers, declare `contracts.imageGenerationProviders`, and ship matching docs/tests/changelog. If maintainers decide this capability should live outside bundled plugins, that should be an explicit product decision rather than an automated cleanup close.
