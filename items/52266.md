## Summary

Keep open. Current `main` still does not implement WebChat thread-bound subagent sessions, and the Control UI chat surface still uses a header `<select>` plus the separate Sessions page rather than the requested session-switching sidebar.

## Evidence

- **Thread-bound sessions doc excludes WebChat:** The current docs still say Discord is the only supported channel for `sessions_spawn({ thread: true })` thread bindings.
  - file: [docs/tools/subagents.md:115](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/docs/tools/subagents.md#L115)
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Current spawn path still emits the reported error:** If no channel plugin registers `subagent_spawning`, `thread=true` returns the same error quoted in the issue: `thread=true is unavailable because no channel plugin registered subagent_spawning hooks.`
  - file: [src/agents/subagent-spawn.ts:495](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/agents/subagent-spawn.ts#L495)
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Control UI is WebChat mode:** The Control UI browser client still connects with `mode: "webchat"`, so the missing WebChat thread-binding support applies directly to this surface.
  - file: [ui/src/ui/app-gateway.ts:285](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/ui/src/ui/app-gateway.ts#L285)
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Current chat session switcher is a header select, not a sidebar:** The chat UI renders session switching as a `<select>` control, and the app header inserts that control at the top of the chat page.
  - file: [ui/src/ui/chat/session-controls.ts:23](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/ui/src/ui/chat/session-controls.ts#L23)
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Header uses session select in chat view:** `renderChatSessionSelect(state)` is rendered in the chat content header, not as a left session list/sidebar.
  - file: [ui/src/ui/app-render.ts:1417](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/ui/src/ui/app-render.ts#L1417)
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)

  - file: [ui/src/ui/views/sessions.ts:248](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/ui/src/ui/views/sessions.ts#L248)

- The issue title asks for a sidebar specifically; current main only offers a dropdown and the Sessions page, so closing as implemented would overstate what shipped.
- Some multi-session pieces already exist, so future triage may want to narrow this issue to WebChat thread-binding plus sidebar UX instead of generic multi-session support.
