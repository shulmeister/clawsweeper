## Summary

Keep PR #69344 open. Current main still uses the original TUI sanitizer policy: non-copy-sensitive tokens are split at 32 characters, the code-like dot/paren/bracket exemption is absent, and tests still assert the 32-character narrow-terminal guard. Related #39565/#39505 overlaps the formatter copy-safety area but does not fully supersede this PR's broader 128-character threshold and code-fragment proposal.

## Best Possible Solution

Keep #69344 open for maintainer review. The likely best path is a focused TUI formatter policy in `src/tui/tui-formatters.ts` that preserves genuinely copy-sensitive or code-like fragments without broadly disabling narrow-terminal protection, with regression tests in `src/tui/tui-formatters.test.ts` covering both code snippets and generic long-token wrapping. Coordinate it with #39565/#39505 rather than closing this PR as implemented or superseded.
