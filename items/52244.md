## Summary

Keep open. Current `main` has the generic catalog-hook and plugin-aware thinking infrastructure, but the OpenRouter plugin still does not use it for dynamic models. Dynamic OpenRouter models resolved from the capability cache can still end up with both default thinking and default reasoning effectively `off`, so this PR’s core bug is not implemented on `main` yet.

## Evidence

- **OpenRouter plugin still lacks dynamic-catalog/thinking default hooks:** `extensions/openrouter/index.ts` registers `resolveDynamicModel` and `prepareDynamicModel`, but it does not define `augmentModelCatalog`, `resolveThinkingProfile`, or `resolveDefaultThinkingLevel`. So cached dynamic model capabilities are not surfaced into the catalog/default-resolution paths.

This makes the PR’s original 'hook never runs for bundled OpenRouter' review blocker obsolete, but OpenRouter still does not provide an augmentation hook on `main`.
