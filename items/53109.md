## Summary

Current `main` already implements the requested fail-closed behavior for interrupted/incomplete streaming turns: it surfaces explicit error payloads, marks terminal lifecycle state, warns when side effects may already have happened, bounds retries, and clears stale pending tool-call state.

## Evidence

- **Explicit incomplete-turn error text:** `resolveIncompleteTurnPayloadText()` now treats interrupted `toolUse`, provider `stopReason="error"`, reasoning-only, and empty-response turns as explicit failures, returning either `"Please try again"` or `"please verify before retrying"` when side effects may already have happened.

- **Run is surfaced as failed/blocked instead of silently disappearing:** `runEmbeddedPiAgent()` now computes `incompleteTurnText`, sets terminal lifecycle metadata (`replayInvalid`, `livenessState`), logs that it is surfacing the error to the user, and returns an `isError: true` payload for incomplete turns.

## Close Comment

Codex review: closing as implemented on current `main`. The embedded Pi runner now converts interrupted/incomplete tool-call turns into explicit error results instead of leaving them ambiguous (`src/agents/pi-embedded-runner/run.ts:1924`, `src/agents/pi-embedded-runner/run/incomplete-turn.ts:160`). It marks terminal lifecycle state (`replayInvalid` / `livenessState`), warns when side effects may already have happened (`please verify before retrying`), caps silent-error retries instead of looping (`src/agents/pi-embedded-runner/run.empty-error-retry.test.ts:80`), and clears stale pending tool-call state on errored/aborted turns (`src/agents/session-tool-result-guard.ts:217`). This is present on `main` at `38caa6832d4e1d344736d635b0c5fdad37e4430b`; it has not shipped in a tagged release yet.
