## Close Comment

Thanks for the PR. Codex review against current `main` says this belongs on the plugin/ClawHub side of the boundary, not as new core surface. The current scope guardrails are explicit: `VISION.md:54` says optional capability should usually ship as plugins, and `VISION.md:101` calls out heavy orchestration layers that duplicate existing agent/tool infrastructure as out of scope for now. Current core already exposes the coordination primitives in `docs/concepts/session-tool.md:13` (`sessions_list`, `sessions_history`, `sessions_send`, `sessions_spawn`, `subagents`), while memory ownership and cross-agent sharing are documented as plugin/config concerns in `docs/concepts/memory.md:31` and `docs/concepts/multi-agent.md:145`. The plugin SDK can already register tools, commands, and gateway methods (`docs/plugins/sdk-overview.md:83`, `src/plugin-sdk/plugin-entry.ts:164`), so this is better pursued as a ClawHub plugin/skill or memory-plugin extension rather than new core/gateway machinery. Closing on scope grounds, not because the idea is without value.
