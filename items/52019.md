## Summary

Fish Audio TTS is better handled as a community plugin than a new OpenClaw core provider. Current main documents speech providers as plugin capabilities, the project vision says optional capability should usually ship as plugins, and the issue already has a community plugin implementation path.

## Evidence

- **Vision prefers plugins for optional capability:** `VISION.md:54` says OpenClaw has an extensive plugin API, `VISION.md:55` says optional capability should usually ship as plugins, and `VISION.md:59` says the bar for adding optional plugins to core is intentionally high.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/VISION.md#L54)

- **Speech providers are a first-class plugin capability:** `docs/plugins/architecture.md:35`-`42` lists `Speech` as a native plugin capability registered via `api.registerSpeechProvider(...)`, which is the intended extension seam for vendor TTS integrations.
  - file: [docs/plugins/architecture.md:35](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/docs/plugins/architecture.md#L35)
