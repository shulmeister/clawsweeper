## Summary

Keep open. Current main still uses the bundle-fragile logging config require path and has no registered runtime loader or shared flattened-build loader, while the PR targets a real open logging.file regression. Related PRs are only candidates and remain open, so this PR is not safely obsolete or implemented on main.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to choose a single canonical fix for the bundled/flattened logging config loader, likely by comparing this small registration seam against the broader shared loader in #70335, then land one implementation with regression coverage for packaged/flattened builds and close the duplicate PRs after that ships.
