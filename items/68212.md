## Summary

Keep open. Current `main` already reads `messages.tts.providers.google.voiceName`, resolves Google as the selected TTS provider, and sends that `voiceName` into the Gemini request body, but I did not find a post-report fix that would explain the reporter’s audible result as already resolved. Without a live reproduction against current `main` plus Gemini credentials/audio comparison, there is not enough evidence to conservatively close this as fixed or obsolete.
