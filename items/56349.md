## Summary

Keep open. Current `main` has partial outbound hook wiring, but it does not provide the issue’s requested invariant of a single unbypassable pre-send policy boundary across all outbound paths. Shared delivery code runs `message_sending` for some paths, while other active channel flows still send directly and bypass that hook; internal workspace hooks still only document post-send `message:sent`.

## Evidence

- **Shared outbound path applies `message_sending`, but only inside one delivery runtime:** `deliverOutboundPayloads()` runs `applyMessageSendingHook()` before channel send operations. This is a centralized path for callers that use `src/infra/outbound/deliver.ts`, but it does not prove universal coverage across all outbound surfaces.
