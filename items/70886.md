## Summary

Current main does not yet fix #70886. ZAI onboarding still applies the selected Coding Plan base URL at provider level for the whole catalog, the ZAI media-understanding provider delegates to the generic image completion path, and there is no ZAI Coding Plan vision regression test. The linked PRs are useful fix attempts, but the issue remains the canonical open bug until one lands.

## Best Possible Solution

Keep #70886 open and land a ZAI-owned fix. The likely fix is to route image-capable ZAI models such as glm-4.6v, glm-4.5v, and glm-5v-turbo from Coding Plan base URLs to the matching standard /api/paas/v4 endpoint for image completion, with regression tests for agents.defaults.imageModel.primary = "zai/glm-4.6v" under both zai-coding-global and zai-coding-cn. Reuse or supersede #70915/#70908 as implementation references, but do not close the issue until the fix is on main.
