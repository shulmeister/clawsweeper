## Summary

Keep open. Current main still lacks the requested granular Discord message action gates: `channels.discord.actions.messages` is a single boolean in config types and validation, discovery has no per-operation message gates, and the Discord runtime still checks the same `messages` gate for read, send, edit, and delete. The existing issue comment that kept this open remains accurate against the current `d54d2d6` main checkout.

## Best Possible Solution

Keep tracking this issue. The likely fix is a backward-compatible Discord gate shape, for example `channels.discord.actions.messages: boolean | { send?: boolean; read?: boolean; edit?: boolean; delete?: boolean }`, wired through config types, zod/generated schema metadata, account action-gate handling, discovery, runtime checks, docs, and focused Discord extension tests. Maintainers may choose different key names, but current main still lacks equivalent per-operation Discord message authorization.
