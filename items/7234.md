## Summary

Keep open. Current `main` at `7a85c1a8226eb7d3a96906d2721a189768d3034a` still models Discord `channels.discord.actions.messages` as one boolean gate, and the same gate still controls Discord read, send, edit, and delete message runtime paths. The prior Codex review comment on the issue remains accurate against current main; none of the provided related items supersede this Discord permission-granularity request.

## Best Possible Solution

Keep tracking this issue. The likely product/code path is a backward-compatible Discord gate shape such as `channels.discord.actions.messages: boolean | { send?: boolean; read?: boolean; edit?: boolean; delete?: boolean }`, wired through config types, zod/generated schema metadata, account action-gate handling, discovery, runtime checks, docs, and focused tests. Maintainers may choose different key names, but current main still lacks equivalent per-operation Discord message authorization.
