## Summary

Current main still models Discord `actions.messages` as a single coarse boolean and still uses that one gate for Discord read, fetch, send, edit, and delete runtime actions. The requested backward-compatible granular read/edit/delete/send gates are not present in the config type, zod schema, runtime gate checks, action discovery, docs, or contract tests, so #7234 should remain open.

## Best Possible Solution

Keep #7234 open. Implement a backward-compatible Discord gate shape such as `channels.discord.actions.messages: boolean | { send?: boolean; read?: boolean; edit?: boolean; delete?: boolean }` or equivalent separate action keys, then update config types, zod schema, generated metadata/docs, action discovery, runtime checks, and focused Discord tests while preserving current boolean behavior for existing configs.
