## Summary

Keep #7234 open. Current main still models Discord message action gating as one coarse `channels.discord.actions.messages` boolean across config types, strict validation, generated metadata, discovery, docs, and runtime enforcement. The requested ability to allow send/read/edit while disabling delete is still missing from the bundled Discord plugin surface.

## Best Possible Solution

Keep #7234 open. Implement a backward-compatible Discord gate design, such as `channels.discord.actions.messages: boolean | { send?: boolean; read?: boolean; edit?: boolean; delete?: boolean }` or equivalent top-level gates, then wire it through config types, zod/generated schema and metadata, account gate resolution, discovery, runtime checks, docs, and focused Discord tests. This belongs in the bundled Discord plugin/config surface because the official Discord integration currently exposes and enforces the coarse gate.
