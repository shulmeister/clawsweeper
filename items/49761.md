## Summary

Keep open. Current `main` at `9ece33c50506ba031df382f69bde65dfda19bea5` still has the reported Feishu behavior: `checkBotMentioned()` returns `true` for any message whose raw content contains `@_all`, and group handling still treats `ctx.mentionedBot` as the `requireMention` gate. There is also no `respondToAtAll`/similar opt-in config on current `main`, in either the schema or docs.
