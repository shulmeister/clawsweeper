## Summary

Keep open. Current main still preserves the reported DAVE-disabled pre-Ready join-abort path: Discord voice passes `daveEncryption: false` through to `@discordjs/voice`, waits 15 seconds for `Ready`, destroys the connection on failure, and only records an OpenClaw voice session after `Ready`. The existing recovery code is for post-Ready receive/decrypt failures, not a voice WebSocket close after Hello and before UDP/Ready.

## Best Possible Solution

Keep this issue open until maintainers decide the intended non-DAVE behavior. The likely code path is `extensions/discord/src/voice/manager.ts`: add explicit pre-Ready close/abort diagnostics, then either implement a bounded retry/fallback to DAVE-enabled join or reject/deprecate `channels.discord.voice.daveEncryption=false` with clear operator guidance if Discord now requires DAVE. Add regression coverage for the initial Ready failure path and update Discord voice docs/config hints. If maintainers choose #39825 as the canonical tracker, preserve this issue's Hello-before-Ready trace there before closing as duplicate.
