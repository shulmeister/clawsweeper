## Summary

Keep PR #71158 open. Current main still registers `bot-clear-storage` without `requireAuth: true`, so it remains in QQBot's pre-dispatch command map while the handler can delete files under the QQBot downloads directory. The PR is a narrow security/auth fix and is not superseded by #70706, which fixed the same class only for `bot-approve`.

## Best Possible Solution

Keep #71158 open for maintainer review and land this focused fix or an equivalent patch on main: add `requireAuth: true` to the QQBot `bot-clear-storage` command, keep the regression test that proves it routes through `getFrameworkCommands()`, and include the user-facing security changelog entry. #70706 is useful precedent, not a duplicate, because it only gated `bot-approve`.
