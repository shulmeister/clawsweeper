## Summary

Current `main` already implements restart/reconnect resume handling for session-scoped gateway restarts via the restart-sentinel path, including waking the agent, preserving routing/thread context, and dispatching a continuation turn back into the same session.

## Close Comment

Closing as implemented on `main` based on this Codex review. Current `main` now persists restart context, schedules restart-sentinel recovery on boot, injects a system wake, and immediately requests a heartbeat so the session resumes instead of staying silent (`src/gateway/server-startup-post-attach.ts:338`, `src/gateway/server-restart-sentinel.ts:45`, `src/gateway/server-restart-sentinel.ts:325`). The restart path also carries a continuation turn back into the same routed session/thread, with coverage in `src/gateway/server-restart-sentinel.test.ts:217` and `src/gateway/server-restart-sentinel.test.ts:383`. The implementation is present in commit `37c250695b600fb12afc799ee1b32d7aae73d096` on `main`; I could not tie it to a shipped release tag yet, so `fixedRelease` is left unset.
