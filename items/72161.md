## Summary

Keep #72161 open. Current main still has the Brave provider docsUrl pointing at the legacy /brave-search path in both provider descriptors, while docs/docs.json redirects /brave-search to the canonical /tools/brave-search page and the docs nav points at tools/brave-search. The linked issue #65870 and the PR comments support this as a small, focused docs-maintenance fix rather than obsolete work.

## Best Possible Solution

Keep #72161 open as the active fix for #65870. The best path is to land the focused canonicalization after normal maintainer validation: update both Brave provider descriptors to https://docs.openclaw.ai/tools/brave-search and remove or replace the duplicate legacy docs page according to the docs redirect policy.
