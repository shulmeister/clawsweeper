## Summary

Keep open. Current main still drops Discord bot-authored messages on the default `allowBots=false` path with only `logVerbose("discord: drop bot message (allowBots=false)")`; the PR’s durable `logDebug` diagnostic, actionable verbose hint, and default-off regression coverage are not present.

## Best Possible Solution

Keep the PR open and rebase it onto the current Discord plugin layout. The best maintainer path is to add the `logDebug` diagnostic and actionable verbose message in `extensions/discord/src/monitor/message-handler.preflight.ts` for the `allowBotsMode === "off"` bot-drop branch, then add a focused regression test in `extensions/discord/src/monitor/message-handler.preflight.test.ts` using the current logger/runtime mocks.
