## Summary

Keep PR #66225 open. Current main still uses the narrower final-tag regex in both affected callsites, so `<final/>` remains unstripped in the sanitizer and streaming subscriber paths the PR targets. The linked #69685 persistence work is a companion path, not a landed superseder for this PR.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a shared, well-tested final-tag stripping/scanning helper across sanitizer, streaming, and persistence paths, with regression coverage for `<final/>`, `<final />`, slash-containing attributes, code-span preservation, and the strict `enforceFinalTag` leak-prevention cases raised in review.
