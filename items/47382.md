## Summary

Keep #47382 open. Current main has related responsiveness features, especially typing indicators, preview/tool-progress streaming, and lifecycle status reactions with stall emojis, but I did not find the requested gateway-level, per-channel silence watchdog that sends configurable beacon messages based on operator-visible delivery timestamps. This is a core channel/gateway behavior request rather than a ClawHub plugin candidate.

## Evidence

- **No requested watchdog surface found:** A precise search found no `silenceWatchdog`, `maxBeacons`, `last_operator_visible`, `lastOperatorVisible`, or `visibleOutput` implementation. The only close name hit was `lastVisibleOutputTail` in ACP projection, which tracks text-tail formatting, not a delivery timestamp or beacon timer.
