## Summary

Keep open. Current `main` has rate-limit handling via auth-profile cooldowns, same-provider profile rotation, and model fallback, but it does not implement the requested per-provider request-throttling/rate-limit configuration for model providers.

## Evidence

- **Model provider config has no rate-limit field:** `ModelProviderConfig` only exposes `baseUrl`, auth/header/request overrides, and `models`; there is no `rateLimit`, `requestsPerMinute`, or `minDelayMs` field for model providers.
  - file: [src/config/types.models.ts:114](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/config/types.models.ts#L114)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Schema rejects extra provider fields:** `ModelProviderSchema` is strict and only allows `baseUrl`, `apiKey`, `auth`, `api`, `injectNumCtxForOpenAICompat`, `headers`, `authHeader`, `request`, and `models`, so the issue’s proposed `models.providers.<id>.rateLimit` shape is not supported on `main`.
  - file: [src/config/zod-schema.core.ts:349](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/config/zod-schema.core.ts#L349)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Current behavior is cooldown/failover, not throttling:** The model-failover docs say rate-limit errors put the auth profile into cooldown and move to the next profile/model, with long `Retry-After` waits surfaced early so failover can run.
  - file: [docs/concepts/model-failover.md:120](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/docs/concepts/model-failover.md#L120)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Configurable rate-limit knob is profile rotation only:** The documented auth cooldown setting for rate limits is `auth.cooldowns.rateLimitedProfileRotations`, described as the maximum same-provider auth-profile rotations before switching to model fallback. There is no per-provider request pacing setting in the model-provider config docs.
  - file: [docs/gateway/configuration-reference.md:3522](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/docs/gateway/configuration-reference.md#L3522)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Retry docs describe SDK retries plus failover:** The retry policy doc says model providers rely on provider SDK retries, and long `Retry-After` waits are short-circuited so model failover can happen; it does not describe any central throttler for model-provider requests.
  - file: [docs/concepts/retry.md:26](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/docs/concepts/retry.md#L26)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Rate-limit exhaustion is surfaced to the user as a cooldown/failure:** When embedded agent execution ends in a rate-limit-shaped failure, the runner builds a cooldown/error message and returns a final failure payload; it does not queue or delay the provider call to stay under a configured per-provider request budget.
  - file: [src/auto-reply/reply/agent-runner-execution.ts:1515](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/auto-reply/reply/agent-runner-execution.ts#L1515)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Limiter utilities are not wired into model-provider execution:** Repository search shows `createFixedWindowRateLimiter` / auth rate limiters used for gateway, webhooks, ACP, and channel/plugin ingress paths, not in model-provider runtime paths.
  - command: `rg -n "fixed-window-rate-limit|consume\(\)|createAuthRateLimiter|RateLimiter\(|rateLimitedProfileRotations" src extensions -g '!**/node_modules/**'`
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
