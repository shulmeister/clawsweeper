## Summary

Issue #12135 is a concrete per-model-provider throttling request, but the same remaining work is already tracked by open issue #13615, which covers per-provider limits, queuing, 429 backoff, visibility, and external API throttling. Current main has reactive rate-limit classification, auth-profile cooldowns, and fallback, but not the requested proactive `requestsPerMinute`/`minDelayMs` provider throttle, so this should close as duplicate/superseded rather than implemented.
