## Summary

Keep #69454 open. Current main still lacks the PR's Control UI churn and stale-index-cache fixes: chat/session refreshes remain unbounded, gateway hello still preloads node/device data, node polling still starts globally, `sessions.changed` still reloads sessions immediately, and Control UI `index.html` is still served with `Cache-Control: no-cache`. The PR discussion is actionable rather than obsolete: Greptile found the direction safe and tested, while a Codex review comment identifies a concrete tab-switch guard to fix before landing.

## Best Possible Solution

Keep the PR open and ask for an update against current main. The best path is to land this narrow Control UI performance hardening after fixing the deferred session-reload tab-switch issue, with focused tests for bounded session refreshes, Nodes-tab-scoped polling/preloads, debounced and suppressed `sessions.changed` reloads, and `index.html` `no-store` cache headers.
