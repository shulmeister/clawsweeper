## Summary

Keep #38853 open. Current main has partial mitigations: generic channel auto-restart backoff and Discord JSON 429 retry handling. The specific reported gap remains: Discord startup/allowlist REST paths still do not classify Cloudflare/Error 1015 HTML as a cooldown signal, non-JSON Discord API bodies can still be preserved in thrown errors, and the `/users/@me/guilds` allowlist path can feed those errors into logs. The prior ClawSweeper comment reached the same keep-open conclusion, and current main still supports it.

## Best Possible Solution

Keep the issue open and fix it in the Discord plugin REST startup/allowlist layer: classify Cloudflare/Error 1015 and non-JSON 429 HTML as a rate-limit cooldown signal, sanitize/summarize HTML bodies before logging, honor `Retry-After` when present, apply a conservative cooldown when absent, and add regression tests for `/users/@me/guilds` and `/oauth2/applications/@me` HTML 429 responses. Treat adjacent reconnect/full-gateway-restart issues as already handled but not canonical for this REST cooldown/log-bloat bug.
