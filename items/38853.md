## Summary

Keep #38853 open. Current main has generic channel restart backoff and limited Discord REST 429 retry handling, but the Discord startup paths still do not implement Cloudflare/Error 1015 cooldown handling or sanitize non-JSON 429 HTML before logging.

## Best Possible Solution

Keep the issue open and narrow the fix to Discord plugin startup REST paths: classify 429 HTML/Cloudflare Error 1015 as a rate-limit cooldown signal, sanitize non-JSON error bodies before logging, honor Retry-After or a conservative cooldown for allowlist and application-id requests, and add regression tests for HTML 429 responses on `/users/@me/guilds` and `/oauth2/applications/@me`.
