## Summary

Keep #38853 open. Current main has generic channel restart backoff and limited Discord REST 429 retry handling, but it still does not implement the issue's Cloudflare-specific rate-limit/cooldown behavior or log sanitization for non-JSON 429 HTML responses.

## Best Possible Solution

Keep the issue open and narrow the fix to the Discord plugin startup REST paths: classify 429 HTML/Cloudflare Error 1015 as rate-limit/cooldown signals, sanitize non-JSON error bodies before logging, honor Retry-After or a conservative cooldown for application-id and allowlist startup requests, and add regression tests for HTML 429 responses on `/users/@me/guilds` and application-id resolution.
