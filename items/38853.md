## Summary

Keep #38853 open. Current main has generic channel auto-restart backoff and Discord JSON 429 retry handling, but the Discord startup REST layer still does not classify Cloudflare/Error 1015 HTML as a cooldown signal, still preserves non-JSON response bodies in Discord API errors, and still lets `/users/@me/guilds` allowlist resolution feed those errors into logs.

## Best Possible Solution

Keep the issue open and fix it in the Discord plugin startup REST layer: classify Cloudflare/Error 1015 and non-JSON 429 HTML as a rate-limit cooldown signal, sanitize or summarize HTML bodies before logging, honor `Retry-After` when present, apply a conservative cooldown when absent, and add regression tests for `/users/@me/guilds` and `/oauth2/applications/@me` HTML 429 responses.
