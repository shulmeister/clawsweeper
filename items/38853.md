## Summary

Keep #38853 open. Current main still has only adjacent generic restart/backoff safeguards; the Discord REST startup and allowlist paths do not classify Cloudflare/Error 1015 HTML 429 responses as a cooldown condition, and non-JSON response bodies can still be embedded in Discord API errors/logs.

## Best Possible Solution

Keep the issue open and fix it in the Discord plugin REST startup/allowlist layer: classify Cloudflare/Error 1015 and other non-JSON HTML 429 responses as rate-limit cooldown signals, sanitize or summarize HTML bodies before logging, honor `Retry-After` when present, apply a conservative cooldown when absent, and add regression tests for `/users/@me/guilds` and `/oauth2/applications/@me` HTML 429 responses. Treat the generic channel restart and health-monitor caps as partial mitigations, not the canonical resolution.
