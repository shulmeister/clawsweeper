## Summary

Current main only partially overlaps this PR. The generic system-event route-preservation concern has since been handled, but the central direct-webchat relay prompt behavior is still missing: `target: "none"` still forces `canRelayToUser` false, so exec/cron heartbeat prompts remain internal-only even when a direct webchat operator route exists.

## Best Possible Solution

Keep this PR open, but ask for a rebase and narrowing against current main: preserve the direct-webchat `target: "none"` relay prompt behavior with the PR's targeted tests, drop or reconcile route-preservation pieces already covered by #66073, and remove any unrelated wizard/setup guard if it is already fixed on main.
