## Summary

Keep #38204 open. Current main has improved xAI support by using the xAI Responses API and supporting standard Grok 4.20 reasoning/non-reasoning IDs, but the requested Grok 4.20 multi-agent models are still explicitly excluded in the bundled xAI provider and covered by tests as unsupported. Later issue comments also ask for native multi-agent endpoint/model support, so the central request remains unresolved.

## Best Possible Solution

Keep the issue open for xAI-provider maintainer judgment. The likely implementation belongs in the bundled `extensions/xai` provider if maintainers want native Grok multi-agent support: add the correct upstream endpoint/request shaping, catalog handling, auth/runtime compatibility, docs, and regression tests. If maintainers decide the beta multi-agent surface should stay outside bundled xAI support, the best path is an explicit ClawHub/provider-plugin direction rather than auto-closing this request.
