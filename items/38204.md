## Summary

Keep #38204 open. Current main supports the normal xAI Grok 4.20 reasoning/non-reasoning models, but still explicitly excludes the Grok 4.20 multi-agent IDs requested here. Docs and tests preserve that as unsupported because it needs a different upstream API surface than the standard xAI Responses transport, and the issue discussion includes follow-up requests for native multi-agent endpoint support.

## Best Possible Solution

Keep the issue open for maintainer/product judgment. The likely implementation path is xAI-provider-owned support for the multi-agent endpoint, or an explicit decision to route it to a separate plugin/ClawHub package if the existing provider/runtime contracts are sufficient. Either path should update `extensions/xai`, provider docs, model catalog or dynamic resolution behavior, and regression coverage for both supported standard Grok 4.20 refs and unsupported or newly supported multi-agent refs.
