## Summary

Keep #38204 open. Current main supports the standard Grok 4.20 beta reasoning/non-reasoning xAI models and documents the multi-agent limitation, but it still does not support the requested Grok 4.20 multi-agent endpoint/model path. The issue discussion includes a useful workaround to use the standard reasoning model, but later comments ask for native multi-agent support rather than only catalog aliases or documentation.

## Best Possible Solution

Keep this open for xAI-provider maintainer judgment. If OpenClaw wants bundled support, implement it in `extensions/xai` with the correct upstream multi-agent transport, request shaping, auth/catalog behavior, model resolution, docs, and regression tests. If maintainers decide the multi-agent endpoint should live outside the bundled provider, resolve that explicitly with ClawHub/provider-plugin guidance.
