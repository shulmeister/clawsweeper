## Summary

Keep #38204 open. Current main supports standard Grok 4.20 xAI refs but still deliberately excludes Grok 4.20 multi-agent ids, and official xAI docs now describe `grok-4.20-multi-agent` as a beta Responses API model, so the remaining work needs xAI-provider product/implementation judgment rather than conservative cleanup.

## Best Possible Solution

Keep the issue open for xAI-provider maintainer judgment. A concrete solution should verify the current xAI multi-agent Responses contract, decide whether support belongs in the bundled `xai` plugin or a separate plugin, then update model resolution/catalog behavior, request shaping such as reasoning/tool handling, provider docs, and regression tests for both standard Grok 4.20 refs and multi-agent refs.
