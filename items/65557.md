## Summary

Keep open. Current main has a chat/session model picker, per-agent default model routing, and a global `agents.defaults.models` allowlist enforced for session overrides, but it does not implement the requested per-account/admin provider and model allowlists, account-scoped defaults, wildcard account model policy, or account-aware server-side rejection.

## Best Possible Solution

Keep this issue open and implement it as an OpenClaw core/admin feature: add an account/user-scoped provider/model policy to config and schema, expose it in the Control UI admin settings, filter `models.list` and the chat picker by the effective account policy, and enforce the same policy in `sessions.patch`, slash `/model`, `chat.send`/run overrides, and OpenAI-compatible model override paths. Reuse the existing global `agents.defaults.models` allowlist and multi-agent routing as building blocks, but make the server-side policy account-aware and add docs/tests for default inheritance, wildcard model patterns, and rejection of disallowed providers/models.
