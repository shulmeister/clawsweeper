## Summary

Current `main` implements only part of this request: users can switch models per session in the chat UI, and the server rejects models outside the global `agents.defaults.models` allowlist. There is no evidence of per-account provider/model allowlists, per-account defaults, or admin UI for account-scoped model policy, so the issue still tracks missing core behavior.

## Evidence

- **Per-session model picker exists in Control UI:** The chat header renders a `data-chat-model-select` picker and switches the active session model via `sessions.patch`, which covers the issue's session-level picker portion but not account-scoped policy.
