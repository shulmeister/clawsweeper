## Summary

Keep open. Current main still formats user-facing runtime failures as hardcoded English in core sanitizer/agent-runner paths, with no `messages.errorLocale`/`messages.locale` config surface or locale lookup. Existing tests assert the English copy, and the related #66055 prior-art PR was closed unmerged rather than superseding this request.

## Best Possible Solution

Keep #66056 open for maintainer product/design work. The likely implementation should add a small core error-message catalog with English default/fallback, a documented config field in `MessagesConfig` and schema/help/docs, and thread a locale or formatter through `sanitizeUserFacingText`, `formatAssistantErrorText`, and `agent-runner-execution`; tests should cover unchanged English defaults plus at least one non-English locale and missing-key fallback.
