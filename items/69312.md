## Summary

Keep #69312 open. Current main still has the reported false-positive path: `splitMediaFromOutput()` skips only fenced code spans, then trims indentation before checking for a `MEDIA:` directive, so a 4-space/tab indented code-block line can still be extracted. The PR is a focused bug-fix attempt with useful regression coverage, though its case-sensitivity cleanup needs maintainer review because current main has tests and streaming-buffer logic that still treat lowercase media directives as supported.

## Best Possible Solution

Keep the PR open and ask for a maintainer-quality revision: implement indented-code-block protection in `splitMediaFromOutput()`, preserve code-line whitespace during cleanup, add focused media and fence regression tests, and explicitly reconcile lowercase `media:` behavior with the current reply-delivery and streaming-directive tests. If maintainers want case-sensitive media directives, land that as an intentional behavior change with matching test updates; otherwise preserve the existing lowercase support while fixing the indented-code false positive.
