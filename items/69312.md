## Summary

Keep PR #69312 open. The linked bug is still present on current main: `splitMediaFromOutput()` skips fenced code blocks only, then trims leading indentation before treating `MEDIA:` as a directive, so a 4-space or tab-indented CommonMark code line can still be extracted. The PR is a focused, recently updated fix for #69311, though its final form needs maintainer review for streaming case-sensitivity alignment and the heuristic's Markdown edge cases.

## Best Possible Solution

Keep PR #69312 open. The best maintainer path is to land or adapt a focused fix in the media parser that skips `MEDIA:` extraction from 4-space/tab indented code blocks, preserves retained code indentation during cleanup, adds regression tests for media parsing and fence closing validation, and keeps streaming directive buffering semantics aligned with any case-sensitivity change.
