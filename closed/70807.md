## Summary

PR #70807 is superseded by the current main timeout work, especially the #70831 transport-timeout fix recorded in the changelog. The observable long-running Ollama/60s timeout problem is now handled in the embedded attempt/provider transport path, while this PR's extra Ollama scripts and config-threading approach have unresolved review defects and are not the best path to merge.

## Best Possible Solution

Close #70807 as superseded by the current main/#70831 timeout implementation. Keep the central embedded-attempt and provider-transport timeout wiring, and only revisit Ollama bridge/keepwarm tooling as a separate maintainer-reviewed plugin/docs/ops item if it is still wanted, with configurable ports, correct defaults, and socket cleanup.
