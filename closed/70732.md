## Summary

Current main no longer persists the raw `System (untrusted): ... Exec completed` notification as chat transcript content. Async exec completion events remain an ephemeral system-event queue, are injected only as runtime context for the heartbeat run, and heartbeat transcript prompts are now empty so the event text is not reloaded as session history.

## Best Possible Solution

Keep the current main behavior: async exec completion details may be used as runtime-only heartbeat context, but the raw system notification and heartbeat prompt stay out of the persisted transcript. Ship this in the next release; users on 2026.4.22 should update once that release is available.
