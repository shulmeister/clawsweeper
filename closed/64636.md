## Summary

Current `main` already restores env-proxy handling on the gateway/CLI startup path and on embedded LLM attempt startup, and the same bootstrap is present in the released `v2026.4.23` tree. That is sufficient evidence that the reported `2026.4.9` regression is already fixed in OpenClaw core.

## Evidence

- **issue_timeline_crossref:** The issue timeline already linked PR #64747, titled `fix(gateway): install env HTTP proxy dispatcher at startup`, which matches the reported regression and points at the same startup-level remedy now present on `main`.
