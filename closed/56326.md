## Summary

Keep open. Current main still exposes `resumeSessionId` and `streamTo` in the shared `sessions_spawn` schema when ACP is available, and still rejects those fields before reaching `spawnSubagentDirect` for `runtime="subagent"`. The report remains a real, source-backed bug rather than implemented cleanup; open PRs such as #68397 are candidate fixes, not shipped behavior.

## Best Possible Solution

Keep #56326 open until a canonical fix lands. The likely maintainer path is to merge or adapt one focused PR, preferably the broad `streamTo` plus `resumeSessionId` silent-strip approach, update `src/agents/tools/sessions-spawn-tool.test.ts` to assert subagent acceptance without forwarding ACP-only keys, and align the ACP/subagent docs if the hard-error contract changes.
