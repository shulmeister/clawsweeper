## Summary

Current `main` already implements this PR’s intent in the new cron doctor migration path, with stronger coverage than the proposed patch. The normalization logic avoids flagging already-canonical payload kinds while still rewriting legacy or whitespace-padded variants, and dedicated regression tests cover both cases.

## Evidence

- **Canonical kinds no longer count as legacy mutations:** `normalizePayloadKind` lowercases the trimmed kind for comparison, but only rewrites when the stored value differs from the canonical literal, so already-canonical `agentTurn` / `systemEvent` values return `false` and do not increment `legacyPayloadKind`.
