## Summary

Keep #61473 open. Current main still uses the older Discord model-picker confirmation path: it awaits the hidden `/model` dispatch, re-reads the originally resolved route, and emits the mismatch warning when the session store is stale. The PR's direct fallback persist, dispatch acceptance signal, and dispatch-effective route handling are not present on main, and the current test suite still contains a bound-thread case that expects the mismatch log rather than fallback persistence. Related PRs #53287 and #56578 are closed and unmerged in the provided context, so they do not supersede this open PR.

## Best Possible Solution

Do not auto-close #61473. The best path is to either land a cleaned-up version of its narrow Discord picker fallback, including accepted-dispatch gating, dispatch-effective route persistence, `applyModelOverrideToSessionEntry`, guarded I/O, post-fallback reread semantics, and regression tests, or replace it with an equivalent main-branch fix that proves stale routed-session picker selections persist correctly.
