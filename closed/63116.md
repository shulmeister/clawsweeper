## Summary

Current main fixes the reported delivery-mirror prompt-overflow path by treating OpenClaw delivery-mirror assistant rows as transcript-only artifacts and dropping them from model replay before provider submission. The mirror rows may still exist in persisted transcripts for audit/history mechanics, but they no longer contribute to the Anthropic prompt that overflowed in #63116. Existing regression coverage and docs lock the behavior, and the related #69576 cross-reference was a separate account-scoping report already closed as implemented.

## Best Possible Solution

Close #63116 as implemented on current main. Keep the shipped/current replay filtering behavior, and leave broader duplicate transcript or consumer-history cleanup on the existing maintainer-owned umbrella #69208 rather than tracking this narrow overflow report separately.
