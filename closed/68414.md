## Summary

Current `main` already contains a Telegram inbound sequencing fix that serializes same-chat debounced messages behind the active keyed run, plus a regression test that matches this report's race window. The fix landed after the latest release, so it is implemented on `main` but not yet shipped in `v2026.4.23`.

## Evidence

- **Per-key inbound debounce now reserves and chains same-key work:** `createInboundDebouncer` keeps a per-key promise chain, reserves the key before flushing a buffered item, and queues later same-key arrivals behind that chain so they cannot overtake or disappear during the flush/turn-complete window.
  - file: [src/auto-reply/inbound-debounce.ts:84](https://github.com/openclaw/openclaw/blob/972d8fc1cf3ce5c5dfd0e68a6620c51565e7898f/src/auto-reply/inbound-debounce.ts#L84)
  - sha: [972d8fc1cf3c](https://github.com/openclaw/openclaw/commit/972d8fc1cf3ce5c5dfd0e68a6620c51565e7898f)
- **Telegram inbound messages use the keyed debouncer:** Telegram handler builds a debounce key from account + conversation + sender, then awaits `inboundDebouncer.enqueue(...)` for each inbound message, so same-chat/same-sender traffic is serialized through the new keyed debounce path.
  - file: [extensions/telegram/src/bot-handlers.runtime.ts:212](https://github.com/openclaw/openclaw/blob/972d8fc1cf3ce5c5dfd0e68a6620c51565e7898f/extensions/telegram/src/bot-handlers.runtime.ts#L212)
  - sha: [972d8fc1cf3c](https://github.com/openclaw/openclaw/commit/972d8fc1cf3ce5c5dfd0e68a6620c51565e7898f)
- **Regression test matches the reported race:** The test `preserves same-chat reply order when a debounced run is still active` starts a first Telegram turn, injects a second message while that first debounced run is still active, then proves the second body starts afterward and both replies are delivered in order.
  - file: [extensions/telegram/src/bot.create-telegram-bot.test.ts:381](https://github.com/openclaw/openclaw/blob/972d8fc1cf3ce5c5dfd0e68a6620c51565e7898f/extensions/telegram/src/bot.create-telegram-bot.test.ts#L381)
