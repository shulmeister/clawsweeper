## Summary

Current `main` already preserves WebUI image uploads for text-only chat models by sending them as attachments and offloading them into `media://inbound/...` refs instead of letting the run degrade into an empty prompt.

## Evidence

- **WebUI sends image attachments even without text:** The Control UI converts composer image data URLs into `chat.send.attachments`, and `sendChatMessage` only rejects sends when both the trimmed message and attachments are empty.
  - file: [ui/src/ui/controllers/chat.ts:213](https://github.com/openclaw/openclaw/blob/6abab7555e1f23ef1b3aa0edced3592037e029cf/ui/src/ui/controllers/chat.ts#L213)
  - command: `nl -ba ui/src/ui/controllers/chat.ts | sed -n '210,360p'`
  - sha: [6abab7555e1f](https://github.com/openclaw/openclaw/commit/6abab7555e1f23ef1b3aa0edced3592037e029cf)
- **Gateway checks model image support before parsing attachments:** `chat.send` resolves the session model, computes `supportsImages`, and passes that into `parseMessageWithAttachments` so text-only sessions take the non-inline path instead of silently losing the image.
  - file: [src/gateway/server-methods/chat.ts:2204](https://github.com/openclaw/openclaw/blob/6abab7555e1f23ef1b3aa0edced3592037e029cf/src/gateway/server-methods/chat.ts#L2204)
  - command: `nl -ba src/gateway/server-methods/chat.ts | sed -n '2200,2235p'`
  - sha: [6abab7555e1f](https://github.com/openclaw/openclaw/commit/6abab7555e1f23ef1b3aa0edced3592037e029cf)
- **Text-only models now offload uploaded images instead of dropping them:** `parseMessageWithAttachments` treats `supportsImages === false` as a forced offload path and appends `[media attached: media://inbound/<id>]` to the prompt body, preserving non-empty media input for the run.
  - file: [src/gateway/chat-attachments.ts:309](https://github.com/openclaw/openclaw/blob/6abab7555e1f23ef1b3aa0edced3592037e029cf/src/gateway/chat-attachments.ts#L309)

- **Regression test covers parser behavior for text-only models:** Unit coverage asserts that a text-only model produces zero inline images, one offloaded ref, and a prompt containing a `media://inbound/...` marker.
