## Summary

Current `main` already isolates malformed skill metadata parsing and still ships bundled skills that use inline emoji metadata, so the reported crash path does not match the present code or latest release contents I could inspect.

## Evidence

- **Shared metadata parse is non-throwing:** `resolveOpenClawManifestBlock()` wraps `JSON5.parse(raw)` in `try/catch` and returns `undefined` on parse failure instead of throwing, which prevents malformed `metadata` from escaping the parser.
  - file: [src/shared/frontmatter.ts:23](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/shared/frontmatter.ts#L23)
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **Skill loader already drops bad frontmatter safely:** `loadSingleSkillDirectory()` catches frontmatter parsing errors and returns `null`, so a bad skill is skipped rather than crashing the wider load/session path.
  - file: [src/agents/skills/local-loader.ts:60](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/agents/skills/local-loader.ts#L60)
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **Inline JSON frontmatter is preserved:** The frontmatter parser explicitly prefers the original inline value for structured inline entries, and there is a test asserting inline JSON metadata is preserved verbatim.
  - file: [src/markdown/frontmatter.ts:166](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/markdown/frontmatter.ts#L166)
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **Invalid metadata already has non-throwing coverage:** Shared frontmatter consumers are tested to return `undefined` for invalid `metadata` JSON rather than throw.
  - file: [src/hooks/frontmatter.test.ts:189](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/hooks/frontmatter.test.ts#L189)
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **Bundled skills still use the reported pattern:** Current repo skills still include single-line `metadata: { "openclaw": { "emoji": ... } }` entries with emoji for bundled skills like BlueBubbles, Discord, and Slack. If this pattern still crashed skill loading on `main`, these bundled skills would be broken too.
  - file: [skills/bluebubbles/SKILL.md:4](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/skills/bluebubbles/SKILL.md#L4)
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
