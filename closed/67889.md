## Summary

Keep open. Current main still ignores the configured image-understanding timeout on the explicit `image` tool path and still hardcodes the MiniMax VLM request timeout. The linked fixes are not merged/shipped in current main.

## Best Possible Solution

Keep the issue open and land a focused fix that resolves the effective timeout from `tools.media.image.timeoutSeconds` and per-model `timeoutSeconds`, passes that value through `createImageTool`/`runImagePrompt` into `describeImage(s)`, adds optional `timeoutMs` to `minimaxUnderstandImage`, forwards it from `describeImagesWithMinimax`, and adds regression tests for default, configured, per-model, and MiniMax fallback timeout propagation.
