## Summary

Keep PR #49767 open. Current main still contains the Feishu `@_all` shortcut that marks every bot as mentioned, and the proposed PR is aimed at a live channel bug rather than obsolete work. The PR discussion also contains useful follow-up: the author addressed the spoofable raw-content detection and misleading test name, while separate review comments still raise a rich-text `post` @all detection gap that should be handled before landing.

## Best Possible Solution

Keep this PR or a canonical successor open, rebase it onto current main, remove the unconditional `@_all` mention shortcut, add a documented opt-in Feishu config path, and cover both normal text and rich-text post @all shapes with regression tests before landing.
