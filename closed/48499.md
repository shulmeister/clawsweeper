## Summary

Current main no longer matches the reported failure mode. The elevated `enabled` gate is derived only from config, not per-request runtime state; internal/webchat turns preserve the original external channel route/provider; and reply-run tests cover telegram-origin runtime-config refresh. On that basis, the report appears obsolete on current `main`.

## Evidence

- **Enabled gate is config-only:** `resolveElevatedPermissions` computes `enabled` only from `cfg.tools.elevated.enabled` and the agent-level config, then returns immediately if either is false. There is no per-request or approval-consumption state in this path.
