## Summary

Current `main` already contains the webhook-route registry fix this issue describes: gateway startup pins the HTTP-route registry, dynamic route registration writes into that pinned route registry, request dispatch re-resolves the live route registry at request time, and the LINE plugin still registers `/line/webhook`. The changelog says this shipped in version `2026.3.22`.

## Evidence

  - file: [extensions/line/src/monitor.ts:305](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/extensions/line/src/monitor.ts#L305)
