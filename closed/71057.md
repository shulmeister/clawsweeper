## Summary

The reported `delivered` false-positive concerns the external `@tencent-weixin/openclaw-weixin` channel plugin, not OpenClaw core. Current `main` documents WeChat as plugin-owned, keeps optional channel behavior on the plugin/ClawHub path, and contains no bundled WeChat runtime in `src/` or `extensions/` to fix here.

## Evidence

- **WeChat docs mark the runtime as external:** Current main says OpenClaw connects to WeChat through the external `@tencent-weixin/openclaw-weixin` plugin, and explicitly states that the WeChat code does not live in the OpenClaw core repo; Tencent API calls, media handling, context tokens, and monitoring are owned by the external plugin.
  - file: [docs/channels/wechat.md:10](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/docs/channels/wechat.md#L10)
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Channel index classifies WeChat as external:** The public channel index on current main lists WeChat as 'Tencent iLink Bot plugin via QR login; private chats only (external plugin)'.
  - file: [docs/channels/index.md:45](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/docs/channels/index.md#L45)
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
