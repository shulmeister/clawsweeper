## Summary

Current main does not expose `skill.json` as an OpenClaw skill contract; skills are `SKILL.md` instruction folders, while runtime-callable tools are already supported through the plugin API and bundle MCP surfaces. A JSON exec-tool loader would be optional tool/skill integration work that can live as an external plugin/ClawHub package, and there is no protected label, maintainer author association, or linked canonical core issue requiring this to stay open.
