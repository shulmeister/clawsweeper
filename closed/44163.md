## Summary

Current `main` does not implement this feature. `PluginHookAfterCompactionEvent` still has no `summary` field, and every current `runAfterCompaction` emitter on `main` sends only counts and `sessionFile`. The PR also appears stale against later compaction-hook refactors, so it needs a fresh rebase/update rather than closure.
