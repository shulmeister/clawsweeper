## Summary

Current main already wires MiniMax OAuth (`minimax-portal`) image understanding through the separate `MiniMax-VL-01` media provider and routes it to MiniMax's `/v1/coding_plan/vlm` endpoint. The reported `models list` `Input=text` output for `MiniMax-M2.7` is intentional in current docs and code, not evidence that the vision path is missing.

## Evidence

- **Bundled plugin metadata registers portal image understanding:** The bundled MiniMax plugin declares `minimax-portal` as a `mediaUnderstandingProvider` and sets its default image model to `MiniMax-VL-01`, so the OAuth provider has first-class image-understanding support on current main.
