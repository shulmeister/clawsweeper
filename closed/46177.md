## Summary

Keep PR #46177 open. Current main still has the misleading/underspecified cron timezone text in the LLM-facing cron tool and CLI help, while the runtime really does fall back to the gateway process host timezone when `tz` is omitted. The PR is still useful, though the existing review comment is right that the wording should say gateway host/local timezone rather than ambiguous system local timezone.

## Best Possible Solution

Keep the PR open and ask for the tiny wording adjustment from the review comment: use `gateway host timezone` or `gateway local timezone` in the cron tool description and both CLI help strings. The docs page already has the correct troubleshooting wording, but the LLM-facing tool and command help still need this clarification on main. After that, this is a small docs/help-text-only change that should be validated with the normal changed gate when the toolchain is available.
