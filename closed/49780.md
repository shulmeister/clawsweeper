## Summary

Current `main` does not show a cron serialization bug for nested payload objects. The cron payload contract is flat and validated as such; nested payload keys are rejected by schema, and store persistence uses ordinary `JSON.stringify` on the full job/state objects rather than a lossy custom serializer.

## Evidence

- **Cron payload schema is flat and rejects extra nested keys:** `agentTurn` payload only allows `message`, `model`, `fallbacks`, `thinking`, `timeoutSeconds`, `allowUnsafeExternalContent`, `lightContext`, and `toolsAllow`, with `additionalProperties: false` for both create and patch payload schemas.
  - file: [src/gateway/protocol/schema/cron.ts:4](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/gateway/protocol/schema/cron.ts#L4)
  - command: `nl -ba src/gateway/protocol/schema/cron.ts | sed -n '1,180p'`
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Gateway rejects invalid cron.add payloads after normalization:** `cron.add` normalizes input, then immediately validates it with `validateCronAddParams`; invalid payload shapes do not get stored.
  - file: [src/gateway/server-methods/cron.ts:189](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/gateway/server-methods/cron.ts#L189)
  - command: `nl -ba src/gateway/server-methods/cron.ts | sed -n '188,290p'`
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Store serialization is plain JSON of the whole job objects:** The cron store writes `configJson = JSON.stringify(stripRuntimeOnlyCronFields(store), null, 2)` and `stateJson = JSON.stringify(stateFile, null, 2)`. There is no custom nested-payload serializer that would selectively drop object properties.
  - file: [src/cron/store.ts:264](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/cron/store.ts#L264)
  - command: `nl -ba src/cron/store.ts | sed -n '40,90p;260,300p'`
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Normalization only drops typed fields when they are the wrong type:** `coercePayload()` deletes `model`, `thinking`, `timeoutSeconds`, `fallbacks`, and `toolsAllow` when they fail type coercion. That is input coercion, not JSON serialization drift.
  - file: [src/cron/normalize.ts:147](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/cron/normalize.ts#L147)
  - command: `nl -ba src/cron/normalize.ts | sed -n '120,240p'`
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Tests cover flat payload preservation and malformed-field rejection:** Normalization tests assert valid flat payload fields survive, while malformed payload arrays are rejected; patch tests also confirm payload field persistence across updates.
  - file: [src/cron/normalize.test.ts:408](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/cron/normalize.test.ts#L408)

## Risks / Open Questions
