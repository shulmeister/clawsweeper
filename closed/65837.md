## Summary

Current main already implements the image-tool custom provider resolution requested by PR #65837 through the newer media-understanding runtime path. The PR discussion correctly identified that the first fallback patch needed to search both getAvailable() and getAll(), and the PR was updated, but current main supersedes the patch by resolving image models through resolveModelWithRegistry against configured providers before throwing Unknown model.

## Best Possible Solution

Close PR #65837 as implemented on current main. Keep the shipped resolveModelWithRegistry-based image runtime and regression coverage; do not merge the older resolveModelFromRegistry fallback patch unless a separate current-main PDF/shared-helper regression proves that helper still needs broader fallback behavior.
