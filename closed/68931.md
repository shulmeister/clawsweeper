## Summary

Current `main` already defers `session.message` chat-history reloads until the active WebChat run is cleared, which prevents the optimistic user turn from being overwritten before transcript persistence completes. The same guard and its tests are present in release `v2026.4.23`, so this was already shipped.

## Evidence
