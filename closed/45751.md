## Summary

Keep #45751 open. Current main still does not reload the Control UI after a successful `update.run`: `runUpdate` only sends the RPC and clears `updateRunning`, and the 1012 service-restart close path only clears error state. The PR addresses a real missing UI behavior, though it should be revised before merge to honor the returned restart delay/coalescing and keep the regression tests for ok/failure/skipped/restart-close cases.

## Best Possible Solution

Keep the PR open and ask for a revision or maintainer patch that implements reload-after-success on current main. The best fix is a shared reload helper, reload only for `result.status === "ok"`, schedule based on the returned `restart.delayMs` plus a small buffer, keep a 1012 service-restart fallback while `updateRunning` is true, clear `updateRunning` after scheduling the fallback, and add focused tests for success, failure, skipped update, restart-delay, and service-restart close behavior.
