## Summary

Keep open. Current `main` still has the exact docs/runtime mismatch reported in issue #59476: `SECURITY.md` tells users Node.js 22.12.0 is sufficient, while the enforced runtime and package engine minimum are 22.14.0. The linked PR #59478 appears to be the proposed fix, but it is not reflected in the checked-out `main` at `78a431ed224b6f5b9d58672346551a5b411ad200`.

## Evidence

- **Security doc still says 22.12.0:** `SECURITY.md` on current `main` says “OpenClaw requires Node.js 22.12.0 or later” and `node --version` should be `v22.12.0 or later`.
  - file: [SECURITY.md:291](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/SECURITY.md#L291)

  - sha: [78a431ed224b](https://github.com/openclaw/openclaw/commit/78a431ed224b6f5b9d58672346551a5b411ad200)
- **Runtime guard enforces 22.14.0:** `assertSupportedRuntime()` emits `openclaw requires Node >=22.14.0.` on unsupported runtimes.
  - file: [src/infra/runtime-guard.ts:114](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/infra/runtime-guard.ts#L114)
  - command: `nl -ba src/infra/runtime-guard.ts | sed -n '99,120p'`

  - file: [package.json:1664](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/package.json#L1664)

  - file: [src/infra/runtime-guard.test.ts:56](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/infra/runtime-guard.test.ts#L56)

## Risks / Open Questions

- Users who follow `SECURITY.md` can install Node 22.12.0 and still fail OpenClaw startup immediately.
- Security guidance is less trustworthy when the documented minimum diverges from enforced/runtime-tested minimums.
