## Summary

Current main already addresses the reported inconsistency. OpenAI Codex GPT-5.4 now has explicit native metadata (`contextWindow: 1,050,000`) plus the intentional default runtime cap (`contextTokens: 272,000`), model-list surfaces can show both values, and embedded session persistence records the runtime-resolved context budget instead of falling back to the generic 200k value.

## Best Possible Solution

Close this as implemented. Keep the shipped split between native `contextWindow` and effective runtime `contextTokens`; users who want a larger or smaller effective budget should use the documented `models.providers.openai-codex.models[].contextTokens` override rather than treating the 1.05M native window as the default runtime cap.
