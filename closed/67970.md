## Summary

Keep #67970 open. Current main has important partial progress: memory-core can now ingest redacted session transcripts into the dreaming short-term corpus and feed promotion into MEMORY.md, and the bundled memory-lancedb plugin demonstrates auto-capture via plugin hooks. But the issue’s clarified ask is automatic capture plus intelligent extraction with reliable daily-log/short-term entries for memory-core; current memory-core writes transcript captures to internal memory/.dreams/session-corpus state, not managed memory/YYYY-MM-DD.md daily logs, and docs still describe compaction flush as a silent reminder to the agent to save memory files.

## Best Possible Solution

Keep this open as a focused memory-core product/API decision. The best path is an opt-in extraction pipeline that consumes session transcripts or turn history, applies deterministic filtering plus optional LLM classification, writes managed entries to memory/YYYY-MM-DD.md or a clearly documented user-facing short-term log, and feeds the existing .dreams ranking/promotion path into MEMORY.md. If maintainers decide this belongs outside the default memory-core path, close later with an explicit ClawHub/plugin direction and a concrete plugin link.
