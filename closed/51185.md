## Summary

Keep open. Current main still contains the reported runtime fallback: `ensureDockerImage()` checks the default sandbox image and, when missing, pulls and tags plain `debian:bookworm-slim` as `openclaw-sandbox:bookworm-slim`. The repo still has a Dockerfile and setup script for the intended sandbox image with python3/jq/ripgrep, but the runtime creation path does not use them. The provided related PR #51166 proposed this exact fix but was closed unmerged, so there is no canonical shipped fix to close against.

## Best Possible Solution

Keep the issue open and fix the runtime default-image recovery path. The conservative product/code direction is to make `ensureDockerImage()` preserve the intended default sandbox contract: pull a built OpenClaw sandbox image from the maintained registry or build from `Dockerfile.sandbox`/`scripts/sandbox-setup.sh` when available, and never silently tag plain Debian as `openclaw-sandbox:bookworm-slim`. Add a focused regression test around mocked docker commands for the missing-default-image path, plus a docs/changelog note if behavior changes.
