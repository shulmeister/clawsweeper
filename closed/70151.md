## Summary

Keep PR #70151 open. Current main still computes terminal payloads from the attempt snapshot, passes `attempt.lastAssistant` into payload construction, and can surface the `payloads=0` incomplete-turn error without re-reading the persisted session file for a late final assistant message. Existing empty-turn fixes cover retries, explicit error surfacing, and provider/parser failures, but not this stale-snapshot/late-persisted-assistant race.

## Best Possible Solution

Keep the PR open and review it as a focused embedded-runner race fix. The best implementation is a bounded, abort-aware recovery read of the persisted session before surfacing a terminal empty-turn error, followed by payload construction from the refreshed assistant message when recovery succeeds, with regression coverage for the stale snapshot plus late final assistant case.
