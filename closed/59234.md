## Summary

Keep open. Current main still calls `qmd status` for QMD vector probing without checking `memory.qmd.searchMode`, and current QMD behavior intentionally keeps embeddings active in `search` mode rather than skipping vector work.

## Best Possible Solution

Keep #59234 open. The likely product fix is an OpenClaw-side QMD mitigation: avoid `qmd status` vector probing when the configured mode is BM25-only `search`, make status reporting distinguish "vectors not needed" from "vector unavailable", and ensure failed vector/embed probes do not prevent BM25 indexing/search. Coordinate any embed-loop changes with the broader open #67113 rather than closing this status-probe report as fixed.
