## Summary

Close as implemented on current main. PR #56739's exact guard move is not present, but the reported recursion path is gone: current `readLoggingConfig()` no longer calls the full config loader, and the normal console settings fallback no longer performs a full config load. The latest release tag still had the older `loadConfig()`-backed implementation, so there is no shipped release to mark yet.

## Best Possible Solution

Close PR #56739 as obsolete/implemented by current main. Keep the shipped main implementation that makes logging config reads independent of full config loading; if maintainers want extra confidence before release, add a focused regression test that combines a stale plugin warning with patched console capture.
