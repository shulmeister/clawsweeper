## Summary

Keep open. Current main has not absorbed PR #48808: memory indexing still uses the existing `chunkMarkdown()` line/character-weighted chunker, there is no `SemanticChunker`/`src/memory/chunking` implementation, and docs/config still expose only token size plus overlap. The related #44395 is a useful open product tracker for heading-aware chunking/entity extraction, but it does not ship or fully supersede this PR’s remaining implementation work. The PR is stale and needs rebase/focus, but it still represents plausible product work rather than cleanup material.

## Best Possible Solution

Keep the item open or ask the contributor/maintainer to rebase and narrow it against the current memory-host-sdk and memory-core architecture. The best product path is a focused heading/markdown-aware chunking change with regression tests for headings, code fences, overlap, multilingual text, and indexing behavior, coordinated with open tracker #44395 for broader entity-extraction and memory-search improvements.
