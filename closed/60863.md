## Summary

This PR adds a separate bundled Bailian/DashScope provider, but current main already has Qwen as the canonical bundled Qwen Cloud / Alibaba DashScope surface. The remaining cache-control behavior is implementable with existing provider-plugin hooks, so the separate provider should live as an external/ClawHub plugin unless maintainers request a focused qwen-provider follow-up.

## Best Possible Solution

Close this PR as ClawHub/plugin scope. Keep `qwen` as the canonical bundled Qwen Cloud / Alibaba DashScope provider, encourage the Bailian implementation to be published as an external provider plugin, and consider a separate focused PR only if maintainers want DashScope cache-control behavior added to `extensions/qwen`.
