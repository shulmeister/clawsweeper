## Summary

Current main already documents and tests the intended shared-session ingress pattern: explicit session routing is supported for Gateway/API requests, `/hooks/agent` can accept caller-supplied `sessionKey` when explicitly enabled and prefix-bounded, and session docs now explain when `dmScope: "main"` is appropriate versus per-peer isolation.

## Evidence

- **Hook session override is implemented:** `resolveHookSessionKey` accepts a supplied hook `sessionKey` when `hooks.allowRequestSessionKey=true`, otherwise rejects it; it also enforces allowed prefixes.
  - file: [src/gateway/hooks.ts:319](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/gateway/hooks.ts#L319)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Hook override behavior is covered by server tests:** Gateway hook tests verify request `sessionKey` is rejected by default, accepted when enabled, routed to the configured/default key, and blocked for disallowed prefixes.
  - file: [src/gateway/server.hooks.test.ts:322](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/gateway/server.hooks.test.ts#L322)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Docs now describe hook session-key policy:** The Gateway config reference documents `POST /hooks/agent`, states that request `sessionKey` is accepted only when `hooks.allowRequestSessionKey=true`, and explains the prefix allowlist requirement.
  - file: [docs/gateway/configuration-reference.md:468](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/docs/gateway/configuration-reference.md#L468)

  - file: [docs/gateway/openai-http-api.md:87](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/docs/gateway/openai-http-api.md#L87)
