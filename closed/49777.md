## Summary

Current main addresses the reported WebChat history gap: Gateway now applies the chat history limit after display projection/filtering, the Control UI requests 200 visible messages and preserves optimistic local messages when a stale or empty history refresh arrives, and the changelog records this exact visible-history fix.

## Best Possible Solution

Close as implemented on current main. Keep the current display-normalized, bounded history path and optimistic-tail preservation; if maintainers want arbitrary-depth history browsing later, track that separately as a pagination/load-older feature.
