## Summary

Keep #50170 open. Current main still persists and emits the raw cron run status independently from delivery success, so a runner result with `status: "error"` and `delivered: true` can still produce the misleading `error`/`delivered` combination reported here. The directly linked PR #50189 is open and appears to track the right fix, but it has not landed on current main.

## Best Possible Solution

Keep #50170 open until #50189 or an equivalent fix lands. The best code path is to normalize cron persistence, events, run logs, failure counters, failure alerts, and one-shot behavior around an effective status where primary delivery success is not reported as a top-level cron failure, while preserving the raw follow-up/canvas error for diagnostics.
