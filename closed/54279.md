## Summary

Current main implements the reported Gemini memory embedding HTTP(S) proxy path. Gemini direct and batch memory embedding calls use the shared memory-host remote HTTP helper, and that helper now switches guarded fetch into trusted env-proxy mode when HTTP_PROXY/HTTPS_PROXY applies while preserving strict SSRF pinning for NO_PROXY and ALL_PROXY-only cases. The changelog records this in 2026.4.25 (Unreleased) via #71506; no shipped release tag is identifiable yet.

## Best Possible Solution

Close this issue as implemented on main. Keep the shared memory-host SDK env-proxy guard as the canonical fix for Gemini memory embeddings; if maintainers want ALL_PROXY-only behavior later, track that separately as an explicit transport/security design change.

  - file: [extensions/google/embedding-batch.ts:104](https://github.com/openclaw/openclaw/blob/be1d656514aa485073287cea50e81e1d0e4163cb/extensions/google/embedding-batch.ts#L104)

  - file: [src/memory-host-sdk/host/remote-http.ts:15](https://github.com/openclaw/openclaw/blob/be1d656514aa485073287cea50e81e1d0e4163cb/src/memory-host-sdk/host/remote-http.ts#L15)
