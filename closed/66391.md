## Summary

Keep open. Current main has stronger TUI/gateway plumbing for run-scoped tool events and native Ollama tool streaming than the reported 2026.4.12 path, but I could not verify a source/history-backed fix for this specific TUI + local Ollama session-id failure. The exact reported error string is not emitted by current OpenClaw source, and the code still exposes the TUI display name as `openclaw-tui` while process/session tools can fail when a model supplies an unknown session id.

## Best Possible Solution

Keep #66391 open as the tracking issue for TUI + local Ollama tool/session plumbing. The best next maintainer path is a focused regression that drives the Gateway-backed TUI/chat path with an Ollama-style streamed tool call for an environment/cwd prompt, plus hardening so local models cannot confuse the TUI display name `openclaw-tui` with a process/session id. If a fix already landed elsewhere, link the exact commit and release to this issue before closing.
