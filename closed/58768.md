## Summary

Current main already documents and supports the two-layer exec-approval model that the report was blocked on, ships a dedicated `openclaw exec-policy` sync path for the requested config plus host approvals file, and defaults unresolved/invalid approval policy back to no-prompt host behavior (`full` / `off` / `full`). I verified the same implementation is present in release `v2026.4.23`.

## Evidence

- **Host exec defaults are no-prompt on current main:** `src/infra/exec-approvals.ts:169-171` defines built-in approval fallbacks as `security="full"`, `ask="off"`, and `askFallback="full"`. `src/infra/exec-approvals.ts:701-713` applies those fallbacks whenever the approvals file does not provide valid values.
