## Summary

Current `main` pins `acpx` 0.5.3, and the pinned ACPX runtime no longer matches the reported `sessionDir: null` failure mode: persisted session records serialize `cwd`/`name`, and `acpx ... -s <name> prompt` resolves sessions by directory-walk lookup rather than a `sessionDir` field.

## Evidence

- **OpenClaw main pins ACPX 0.5.3:** The bundled ACPX extension depends on `acpx` `0.5.3` on current `main`.
  - file: [extensions/acpx/package.json:7](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/extensions/acpx/package.json#L7)
  - command: `nl -ba extensions/acpx/package.json | sed -n '1,20p'`
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Docs say bundled plugin uses plugin-local pinned binary:** Current docs state the bundled ACPX plugin uses its plugin-local pinned `acpx` binary and verifies/repairs that exact version, which makes stale global `acpx` installs a separate path from the bundled runtime.
  - file: [docs/tools/acp-agents-setup.md:141](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/docs/tools/acp-agents-setup.md#L141)
  - command: `nl -ba docs/tools/acp-agents-setup.md | sed -n '139,155p'`
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Pinned acpx schema does not persist sessionDir:** Inspection of the published `acpx@0.5.3` tarball (the version pinned by OpenClaw main) shows the session serializer writes `acpx_record_id`, `acp_session_id`, `agent_command`, `cwd`, and `name`; there is no `sessionDir` field in the persisted schema.
  - file: [package/dist/prompt-turn-CXMtXBl-.js:226](https://github.com/openclaw/openclaw/blob/087f8207d58aebc6efd3ea455488aad0cd07d3af/package/dist/prompt-turn-CXMtXBl-.js#L226)

  - sha: [087f8207d58a](https://github.com/openclaw/openclaw/commit/087f8207d58aebc6efd3ea455488aad0cd07d3af)
- **Pinned acpx prompt path resolves named sessions by cwd/name lookup:** In the same `acpx@0.5.3` tarball, `acpx ... prompt` calls `findSessionByDirectoryWalk(...)` and then sends the prompt using the matched record's `acpxRecordId`; the lookup is based on `cwd` and optional session name, not a `sessionDir` field.
  - file: [package/dist/cli.js:121](https://github.com/openclaw/openclaw/blob/087f8207d58aebc6efd3ea455488aad0cd07d3af/package/dist/cli.js#L121)
