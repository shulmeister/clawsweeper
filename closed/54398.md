## Summary

Current main implements the env-proxy path this issue requested. Remote memory HTTP calls now switch guarded fetch into trusted env-proxy mode only when Undici will actually proxy the target, and both OpenAI and Gemini memory embedding paths use that shared remote HTTP helper. The fix landed in PR #71506 for the same memory remote embedding env-proxy failure tracked by #52162, and the changelog records it under 2026.4.25 (Unreleased), so there is no shipped release tag yet.

## Best Possible Solution

Keep the PR #71506 implementation on main and let it ship in the next release; users still seeing memory embedding transport failures after upgrading should open a new issue with sanitized proxy env, NO_PROXY/ALL_PROXY details, provider base URL, `openclaw memory status --deep` output, and gateway logs.
