## Summary

Keep PR #54562 open for maintainer judgment. Current `main` does not implement the proposed gateway ModelRouter, and while prompt-aware model choice already belongs well on the plugin/ClawHub path via `before_model_resolve`, this PR's own discussion raises deeper unresolved questions about same-turn escalation, retry/context signal exposure, streaming behavior, and RPC parity. That makes it unsuitable for an automated close as already implemented, duplicate, or straightforward ClawHub cleanup.

## Best Possible Solution

Do not merge the current branch as a core gateway router. Maintainers should either close/redirect the prompt-complexity selection portion to a ClawHub code plugin using `before_model_resolve`, or ask for a much narrower design/PR that adds any missing generic plugin events needed for same-turn retry/context-aware escalation before implementing router policy on top.
