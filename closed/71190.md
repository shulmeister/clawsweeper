## Summary

Current main addresses the reported snowball path by separating the user-visible transcript prompt from runtime-generated prompt context. The orphan merge still preserves queued text before removing the leaf, but the generated queued marker and abort hint are now kept out of the visible user prompt for the Discord/auto-reply path, so the next user leaf no longer mutates into an ever-growing block.

## Best Possible Solution

Close as implemented on current main. Keep the transcriptPrompt/runtime-context separation and the existing orphan preservation semantics; if a new edge case appears, add a narrow regression around an already-marked orphan leaf in the current runtime-context path rather than replacing the merge with a full no-op.
