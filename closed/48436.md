## Summary

Current `main` already routes Gemini `web_search` through the proxy-aware trusted web-tools fetch path, so the reported raw-`fetch()`/`HTTP_PROXY` failure mode is already implemented away on `main`. The changelog also records this proxy fix for `web_search` provider HTTP calls as shipped in `2026.2.25`.

## Evidence

- **Gemini web search uses trusted web-tools endpoint:** The Gemini provider no longer calls raw Node `fetch` directly for search. Its request path goes through `withTrustedWebSearchEndpoint(...)`, which wraps the outbound Gemini `generateContent` call.
  - file: [extensions/google/src/gemini-web-search-provider.runtime.ts:66](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/extensions/google/src/gemini-web-search-provider.runtime.ts#L66)
  - command: `nl -ba extensions/google/src/gemini-web-search-provider.runtime.ts | sed -n '58,132p'`
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Trusted web-tools fetch explicitly enables env-proxy mode:** `withTrustedWebSearchEndpoint(...)` delegates to `withTrustedWebToolsEndpoint(...)`, and that helper sets `useEnvProxy: true` for trusted web-tool endpoints.
  - file: [src/agents/tools/web-guarded-fetch.ts:64](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/agents/tools/web-guarded-fetch.ts#L64)

  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Env-proxy mode creates an EnvHttpProxyAgent when proxy env vars are set:** In the guarded fetch implementation, trusted env-proxy mode checks for configured proxy env and then constructs `createHttp1EnvHttpProxyAgent(...)`, which is the path that makes `HTTP_PROXY`/`HTTPS_PROXY` effective.
  - file: [src/infra/net/fetch-guard.ts:357](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/infra/net/fetch-guard.ts#L357)

- **Changelog records the web_search proxy fix as shipped:** The changelog explicitly says `web_search` provider HTTP calls including Gemini were routed through a shared proxy-aware SSRF guard path so installs behind `HTTP_PROXY`/`HTTPS_PROXY` stop failing.
  - file: [CHANGELOG.md:3880](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/CHANGELOG.md#L3880)

  - file: [src/cli/run-main.ts:187](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/cli/run-main.ts#L187)
