## Summary

Current main and the v2026.4.24 release implement the practical fix for #53959: OpenAI Codex reasoning-capable runs now default to a provider-supported medium thinking level when no explicit default is set, and embedded OpenAI/OpenAI Codex GPT-5 runs keep the filtered Pi session tool allowlist active so tools do not degrade into plan-only/no-tool-call replies after the Pi session-tool behavior change. This matches the issue discussion, where multiple reporters identified `thinking: "medium"` as the working workaround and the reporter confirmed moving to the newer Codex model path worked.

## Best Possible Solution

Close #53959 as implemented on main. Keep the shipped medium thinking default, Codex GPT-5 strict-agentic retry behavior, and Pi session tool allowlist fixes; any future remaining failures should be opened as a fresh issue with current version, model, thinking level, resolved tool inventory, and sanitized run diagnostics.
