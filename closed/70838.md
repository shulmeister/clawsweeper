## Summary

Current `main` already adds a config redaction/restore pipeline that round-trips `__OPENCLAW_REDACTED__` placeholders back to the on-disk secret before `config.set`, `config.apply`, or `config.patch` can persist changes. The new tests explicitly cover `agents.defaults.memorySearch.remote.apiKey`, and QQ Bot `clientSecret` is on the shared sensitive secret-input path, so the reported corruption path is implemented away on `main`. This fix appears to be post-`v2026.4.23`, so there is no shipped release tag to cite yet.

## Evidence
