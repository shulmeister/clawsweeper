## Summary

Keep #54517 open. Current main uses the systemd-user helper for service enable/restart, so the narrow `systemctl enable` vs `systemctl --user enable` complaint is partly addressed, but the headless Ubuntu failure is not fully fixed on main: non-interactive onboarding still enables lingering only after `service.install()`, and direct `openclaw gateway install` does not enable lingering before install. The open related PR #54567 is the right canonical fix path, but it is open/unmerged, so this issue should remain open until that work lands or maintainers explicitly choose to track only the PR.

## Best Possible Solution

Land or otherwise resolve the canonical systemd linger fix in PR #54567, ideally ensuring both `openclaw onboard --install-daemon` and direct `openclaw gateway install` handle headless Ubuntu user-systemd prerequisites before attempting `systemctl --user enable`. Then close #54517 as fixed with the landed commit/release evidence.
