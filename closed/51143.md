## Summary

Current main already handles Telegram multi-account group messages through a single per-account message pipeline, respects a configured named defaultAccount, uses account-scoped group config in multi-account mode, and now guards duplicate same-token pollers/phantom default-account conflicts. The issue discussion's raw-update inference no longer indicates a missed grammY message filter because the raw logger is before the message handler on current main, and grammY's own filter implementation matches any update with a message field.
