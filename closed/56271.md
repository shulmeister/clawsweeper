## Summary

Keep open. Current main does not implement a `browser.autoClose` setting or completion-bound Control UI/browser-frame auto-close behavior. The shipped cleanup is idle/cap-based tab cleanup plus lifecycle cleanup for reset/subagent/cron paths, which is materially different from this requested opt-in task-completion behavior. Related #40694 should inform the design, but it does not make this specific Control UI/config request obsolete.

## Best Possible Solution

Keep #56271 open as a focused opt-in Control UI/browser lifecycle feature. The likely path is to design a config-backed completion-close policy, define how it interacts with `browser.tabCleanup` and reusable/labeled tabs, wire completion events to close the relevant Control UI preview and/or tracked ephemeral browser tabs, then update config schema/help, docs, and focused browser/UI tests.
