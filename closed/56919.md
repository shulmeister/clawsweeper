## Summary

Current main already fixes the reported 2026.3.28 bundled-skill warning spam. Bundled plugin skill paths that previously pointed through node_modules are now copied into a dist-owned bundled-skills directory and the generated manifest is rewritten to that in-root path; current runtime containment checks still warn only for real escapes. The issue overlaps the already-closed built-in extension warning report #43745, while the still-open managed ClawHub/WSL symlink issue #44051 is a separate openclaw-managed source case.

## Best Possible Solution

Close this bundled/install warning-spam issue as implemented. Keep the shipped bundled-skills manifest rewrite and runtime containment checks; leave the separate managed ClawHub/WSL symlink behavior tracked by #44051/#59219 if maintainers still want that policy change.
