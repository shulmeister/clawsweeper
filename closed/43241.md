## Summary

PR #43241 adds a niche new channel as a bundled core plugin, but current OpenClaw direction is to keep optional capabilities out of core and ship them as external plugins. Pilot Protocol is not present on current `main`, and the documented path for new channel integrations is ClawHub/npm plugin distribution rather than adding another bundled plugin to this repository.

## Evidence

- **Vision: optional capability stays out of core:** `VISION.md` says core should stay lean, optional capability should usually ship as plugins, the preferred path is npm/local extension loading, and the bar for adding optional plugins to core is intentionally high.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/VISION.md#L54)
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **Plugin docs: external/community plugins are first-class:** `docs/tools/plugin.md` documents that plugins can be core or external, and that external plugins are installed through `openclaw plugins install` with ClawHub/npm resolution.
  - file: [docs/tools/plugin.md:11](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/docs/tools/plugin.md#L11)

- **Building docs: new channel plugins do not need to be added to this repo:** `docs/plugins/building-plugins.md` explicitly says plugin authors do not need to add their plugin to the OpenClaw repository and should publish to ClawHub or npm instead.
  - file: [docs/plugins/building-plugins.md:16](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/docs/plugins/building-plugins.md#L16)
