## Summary

Current `main` already has a dedicated cron creation action and resets cron form state back to defaults, so the reported missing-new-button/stale-form workflow is implemented.

## Evidence

- **Dedicated New action in Cron UI:** The cron summary header renders a primary `+ New` button when quick create is available.
  - file: [ui/src/ui/views/cron.ts:422](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/ui/src/ui/views/cron.ts#L422)
- **New action initializes fresh draft state:** The render wiring opens cron quick-create and seeds a new default draft instead of reusing the prior form/job state.
  - file: [ui/src/ui/app-render.ts:1721](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/ui/src/ui/app-render.ts#L1721)
- **Default cron form starts blank for editable fields:** `DEFAULT_CRON_FORM` uses empty strings for `name`, `description`, `agentId`, `sessionKey`, `payloadText`, delivery target fields, and timeout.
  - file: [ui/src/ui/app-defaults.ts:13](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/ui/src/ui/app-defaults.ts#L13)
- **Form reset is explicit after create/cancel:** `resetCronFormToDefaults()` restores `DEFAULT_CRON_FORM`; it is called after successful `cron.add` and when canceling edit mode.
  - file: [ui/src/ui/controllers/cron.ts:389](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/ui/src/ui/controllers/cron.ts#L389)
- **Regression tests cover stale-state reset and clone/create mode:** Controller tests assert cancel resets to defaults and cloning switches back to create mode and submits `cron.add` rather than `cron.update`.
  - file: [ui/src/ui/controllers/cron.test.ts:1101](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/ui/src/ui/controllers/cron.test.ts#L1101)
- **Shipped in latest release snapshot:** The same `+ New` button and reset logic are present in the `v2026.4.22` tag via `git show v2026.4.22:ui/src/ui/views/cron.ts` and `git show v2026.4.22:ui/src/ui/controllers/cron.ts`.
  - command: `git show v2026.4.22:ui/src/ui/views/cron.ts | rg -n "onQuickCreate|\+ New|cron-summary-strip__actions"; git show v2026.4.22:ui/src/ui/controllers/cron.ts | rg -n "resetCronFormToDefaults|cancelCronEdit|startCronClone|cron.add"`
