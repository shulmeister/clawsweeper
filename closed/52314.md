## Summary

Current `main` already implements the requested pre-compaction memory flush hook/config, documents it, and covers it with tests. The feature was already listed in the `2026.1.11` changelog section and is still present in the latest released code.

## Evidence

- **Config surface matches the issue request:** `src/config/types.agent-defaults.ts:448` adds `agents.defaults.compaction.memoryFlush`, and `src/config/types.agent-defaults.ts:481` defines `enabled`, `softThresholdTokens`, `prompt`, and `systemPrompt`.
  - file: [src/config/types.agent-defaults.ts:448](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/config/types.agent-defaults.ts#L448)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Runtime performs a silent pre-compaction flush turn:** `src/auto-reply/reply/agent-runner-memory.ts:535` resolves the memory flush plan, `src/auto-reply/reply/agent-runner-memory.ts:700` gates it near the compaction threshold, and `src/auto-reply/reply/agent-runner-memory.ts:771` runs a silent `trigger: "memory"` turn with `memoryFlushWritePath` before compaction proceeds.
  - file: [src/auto-reply/reply/agent-runner-memory.ts:535](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/auto-reply/reply/agent-runner-memory.ts#L535)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Write path is constrained to the daily memory file:** `src/agents/pi-tools.ts:349` requires `memoryFlushWritePath` for memory-triggered runs, and `src/agents/pi-tools.ts:637` wraps the write tool so the flush turn can only append to the designated memory file.
  - file: [src/agents/pi-tools.ts:349](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/pi-tools.ts#L349)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Tests cover the behavior end to end:** `src/auto-reply/reply/agent-runner-memory.test.ts:97` verifies a memory flush turn runs, rotates after compaction, uses a `memory/YYYY-MM-DD.md` path, and persists `memoryFlushCompactionCount` / `memoryFlushAt`. `src/config/config.compaction-settings.test.ts:19` verifies the config values are preserved.
  - file: [src/auto-reply/reply/agent-runner-memory.test.ts:97](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/auto-reply/reply/agent-runner-memory.test.ts#L97)
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Docs and changelog explicitly say this is implemented:** `docs/reference/session-management-compaction.md:324` labels pre-compaction memory flush as implemented, `docs/concepts/memory.md:103` says it runs automatically before compaction, and `CHANGELOG.md:6521`-`6546` lists the automatic pre-compaction memory flush and its config under release `2026.1.11`.
  - file: [CHANGELOG.md:6521](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/CHANGELOG.md#L6521)
