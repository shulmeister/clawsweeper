## Summary

Keep #40446 open. Current main has adjacent strict-agentic and tool-loop protections, but it still does not implement this PR's passive after-turn execution-health monitor, `agents.defaults.executionHealth` config, or file-burst/no-effect/error-cascade signals. The PR is concrete and core-agent scoped, with useful follow-up work in the discussion, but it needs maintainer review and likely a rebase/fixes before merge.

## Best Possible Solution

Maintainers should decide whether passive execution-health monitoring belongs in core; if yes, rebase the PR onto current main and resolve the remaining review risks, preferably splitting shared parser/runtime wrapper changes from the monitor itself. If maintainers prefer the existing tool-loop/strict-agentic direction, create or link a canonical superseding issue that explicitly tracks the remaining file-burst, no-effect-loop, and error-cascade monitoring gaps.
