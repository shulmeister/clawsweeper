## Summary

Keep open. The supplied discussion reports two failures: the packaged `memory-lancedb` runtime ENOENT and a separate auto-recall embedding context-length failure. Current `main` and related ClawSweeper report #57372 support closing the packaging-only duplicate, but this issue is broader: `memory-lancedb` still embeds the full `before_prompt_build` prompt for auto-recall, and `captureMaxChars` remains capture-only rather than a recall-query limit.

## Best Possible Solution

Keep #56780 open and narrow maintainer work to the remaining recall overflow. The packaging half is already fixed by the shipped Memory/LanceDB manifest lookup work, but `memory-lancedb` still needs a bounded recall query strategy, ideally using provider/model embedding input limits or a new explicit recall query cap rather than reusing the capture-only `captureMaxChars` semantics.
