## Summary

Keep #56698 open. Current main still has no framework-level deduplication for hook-mutated bootstrap files: hooks receive a mutable `context.bootstrapFiles` array, `resolveBootstrapFilesForRun` sanitizes valid paths but preserves every entry, and `buildBootstrapContextFiles` iterates every file into injected context. The related fix PRs in the provided context are open or closed-unmerged, so they do not supersede the issue yet.

## Best Possible Solution

Keep the issue open and implement framework-level duplicate-path protection after hook overrides, preferably in `resolveBootstrapFilesForRun`/`sanitizeBootstrapFiles` or immediately before `buildBootstrapContextFiles`, with regression tests for duplicate virtual files by identical and trimmed paths. The implementation should define first-wins or last-wins semantics explicitly so hook ordering stays predictable.
