## Summary

Keep #72426 open. Current main still drops the Google Live function-call name when sending FunctionResponse payloads, while the pinned @google/genai SDK validates that live function responses contain name and response, plus id for Gemini API sessions. The PR is a narrow, relevant fix with regression coverage and is not safely obsolete: related PR #72189 is broader and still has an unresolved continuation-scope review concern.

## Best Possible Solution

Keep #72426 open for maintainer review as a focused fix for the missing Google Live function response name. Maintainers should either land this narrow call-id-to-name mapping and fail-closed behavior, or explicitly reconcile it with the broader #72189 continuation work if they choose that PR as canonical. Do not auto-close #72426 as duplicate until one of those paths is selected or shipped.
