## Summary

PR #66413 should close as superseded by the later focused secrets PR #68384. Current main still has the old exec secret parsing behavior, but this PR’s last-line truncation approach has an unresolved correctness risk for legitimate multiline text output and the branch also bundles unrelated Slack/UI fixes.

## Best Possible Solution

Close #66413 as superseded. Keep the secrets work on #64821/#68384, where maintainers can choose the conservative parser behavior, ideally rejecting ambiguous noisy non-JSON stdout or otherwise preserving/documenting multiline values. Split Slack crash handling and Control UI refresh gating into separate scoped PRs if those bugs still need work.
