## Summary

Keep #37706 open. Current main still treats raw Feishu/Lark message content containing `@_all` as `mentionedBot=true`, and group handling still trusts that flag for the `requireMention` gate. The duplicate fix PRs in the provided context are closed unmerged except #44256, which remains open, so the bug is not implemented on main.

## Best Possible Solution

Keep #37706 open as the canonical Feishu/Lark `@all` false-positive bug. The best fix is a focused Feishu plugin change that removes the raw `@_all` shortcut, relies on actual bot mention metadata and rich-text post `at` targets, and adds regression tests for broadcast-only `@all`/`@_all` plus broadcast-with-real-bot-mention cases.
