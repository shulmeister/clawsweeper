## Summary

Current `main` already degrades safely when hook modules are missing or fail to import, so the PR’s reported startup crash does not reproduce against the checked-out code.

## Evidence

- **Legacy missing-path guard already skips bad entries:** `src/hooks/loader.ts:206-211` resolves the configured legacy module path with `resolveExistingRealpath(...)`; when the file is missing it logs an error and `continue`s instead of importing or throwing.

  - file: [src/hooks/loader.ts:171](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/hooks/loader.ts#L171)
