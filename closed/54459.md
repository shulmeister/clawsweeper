## Summary

Current main has shipped restart/session durability work that addresses the reported loss mode from 2026.3.23-2: restart-triggered shutdown drains accepted turns before closing, active embedded runs hold transcript write locks through cleanup, stale transcript locks are reconciled on startup, and marked main sessions are resumed or failed visibly instead of being silently left with missing transcript state.

## Best Possible Solution

Keep the shipped gateway restart drain, transcript write-lock, and post-start recovery implementation. If a new loss case still reproduces on v2026.4.24 or newer, it should be opened as a fresh issue with exact restart mechanism, logs, session files, and whether the gateway used managed restart intent or an unmanaged stop.

## Evidence
