## Summary

Current main implements the central request through the shared `presentation` message-tool contract instead of the older `inlineButtons` runtime-capability injection proposed in #56955. Configured Slack sends expose `presentation` by default and render button/select presentation blocks into Slack Block Kit. The Slack-only `[[slack_buttons]]` directive remains a separate opt-in `interactiveReplies` feature, but default Slack Block Kit button support is already shipped.

## Best Possible Solution

Close this issue and keep the shipped shared `presentation` implementation as the default Slack button path. Any remaining cleanup around Slack-specific directive prompts or `interactiveReplies` should be handled separately, with #46647 as the related prompt-direction thread.
