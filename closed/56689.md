## Summary

Close as duplicate/superseded by the existing Control UI math-rendering work. Current main still lacks LaTeX/KaTeX/MathJax rendering, but #42840 already tracks the same feature request and #61510 is an open frontend implementation PR explicitly related to this issue.

## Best Possible Solution

Keep #42840 as the canonical feature request and consolidate implementation review in #61510 or its successor. The eventual fix should live in the Control UI markdown/chat rendering path, add a vetted math renderer such as KaTeX or MathJax, preserve existing Markdown and sanitization behavior, handle invalid LaTeX gracefully, avoid parsing ordinary dollar amounts like $100 as math, and add focused rendering tests for inline, block, and mixed Markdown/math messages.
