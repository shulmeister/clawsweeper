## Summary

Keep open. Current `main` still has the failed OpenAI Codex OAuth path returning an empty auth result while the setup wizard still prompts for default model selection afterward, so the underlying bug this PR targets is not implemented on `main`. The PR branch is also stale against current architecture because the old core-owned auth files from the PR context have been replaced by plugin-owned OpenAI auth and `src/wizard/setup.ts`.

## Evidence

- **Setup wizard still prompts for model selection after any non-retry auth result:** `runSetupWizard` applies the auth choice, only branches on `retrySelection`/`agentModelOverride`, then computes `shouldPromptModelSelection` and calls `promptDefaultModel(...)`. There is no guard for an incomplete OAuth flow.
