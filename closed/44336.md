## Summary

Current `main` already prevents exact `NO_REPLY` payloads from leaking through Telegram outbound delivery by normalizing silent replies before channel send, so this adapter-local PR is obsolete.

## Evidence

- **Central outbound normalization now runs before any channel send:** `deliverOutboundPayloadsCore` builds an outbound payload plan up front, so Telegram outbound sends receive already-normalized payloads instead of raw `NO_REPLY` text.
  - file: [src/infra/outbound/deliver.ts:729](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/infra/outbound/deliver.ts#L729)

This covers the `sendText` and no-media `sendPayload` cases the PR targets.
