## Summary

Keep open. Current main now implements the gateway-startup HTTP(S) env-proxy bootstrap, but it does not implement this PR's ALL_PROXY/SOCKS fallback, and current tests explicitly preserve ALL_PROXY-only as non-proxied. The PR should not be auto-closed as implemented unless maintainers intentionally reject the ALL_PROXY part or track it in a narrower canonical issue.

## Best Possible Solution

Keep the item open for maintainer judgment. The practical path is to rebase and narrow it to the remaining ALL_PROXY/SOCKS/global-dispatcher gap, with tests for lowercase precedence, malformed/authenticated proxy URL redaction, NO_PROXY/loopback preservation, and dispatcher replacement. If maintainers intentionally want OpenClaw to keep matching Undici's HTTP_PROXY/HTTPS_PROXY-only semantics, close this with that explicit product/security rationale and track any ALL_PROXY request separately.
