## Summary

Issue #47952 reports `openclaw logs --follow` failing on OpenClaw 2026.3.13 because the gateway/WebSocket handshake timed out after about 3 seconds on WSL2. Current `main` no longer matches that behavior: the shared gateway pre-auth handshake timeout now defaults to 10,000ms, both the server-side WebSocket pre-auth path and the client-side `connect.challenge` watchdog use that shared timeout, and the changelog records this exact fix as shipped in 2026.3.22.

## Evidence

- **Shared handshake timeout is now 10 seconds:** `DEFAULT_PREAUTH_HANDSHAKE_TIMEOUT_MS` is `10_000`, and the default connect-challenge timeout resolves to that same shared value.
  - file: [src/gateway/handshake-timeouts.ts:1](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/gateway/handshake-timeouts.ts#L1)
- **Server WebSocket pre-auth path uses the shared timeout:** The gateway WebSocket connection handler reads `getPreauthHandshakeTimeoutMsFromEnv()` and logs/closes only after that timeout expires.
  - file: [src/gateway/server/ws-connection.ts:366](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/gateway/server/ws-connection.ts#L366)
- **Client watchdog for `connect.challenge` uses the shared timeout:** The client arms its `gateway connect challenge timeout` watchdog with `resolveGatewayClientConnectChallengeTimeoutMs`, which resolves to the same 10s default unless explicitly overridden.
  - file: [src/gateway/client.ts:820](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/gateway/client.ts#L820)
- **Regression coverage asserts the shared default:** Tests explicitly assert that the connect-challenge timeout defaults to `DEFAULT_PREAUTH_HANDSHAKE_TIMEOUT_MS`.
  - file: [src/gateway/handshake-timeouts.test.ts:13](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/gateway/handshake-timeouts.test.ts#L13)
- **Changelog records the shipped fix:** `CHANGELOG.md` for `2026.3.22` says: raise the default pre-auth handshake timeout to 10 seconds and add `OPENCLAW_HANDSHAKE_TIMEOUT_MS` so busy local gateways stop dropping healthy CLI connections at 3 seconds. This directly matches the issue report from 2026.3.13.
  - file: [CHANGELOG.md:2445](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/CHANGELOG.md#L2445)
- **Checkout remained clean:** Verified with a read-only cleanliness check: `git diff --quiet && echo clean` returned `clean`.
  - command: `git diff --quiet && echo clean`
