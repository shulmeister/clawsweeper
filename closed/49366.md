## Summary

Keep open. Current main does not implement this PR's central change of excluding config-backed service env vars from the gateway install plan. Main still merges durable config/state-dir env and auth-profile env refs into service metadata, and tests explicitly expect provider-like keys to be present. Newer main has partial Linux/systemd handling that moves matching state-dir .env values into a 0600 EnvironmentFile, but macOS launchd still serializes plan environment entries into plist EnvironmentVariables, so this remains a product/security tradeoff rather than an obsolete PR.

## Best Possible Solution

Keep the PR open for maintainer review, but do not land it mechanically. The best path is to adapt the security hardening to current main: define which env sources may be persisted, preserve shell-only auth-profile refs for ref-mode installs, avoid inline secret exposure where possible across both systemd and launchd, update the current managed-env tests/docs, and link or fold the result into the related service-env lifecycle issues (#53926 and #54521) if maintainers choose a different canonical fix.
