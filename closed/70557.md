## Summary

Current main already preserves merged-model input capabilities in `models.mode: "merge"`, and the same code is present in the shipped `v2026.4.23` release. That closes the reported regression path where image capability could be lost when one merged provider lacked an `apiKey`.

## Evidence

- **Merged provider models preserve implicit input capabilities on current main:** `mergeProviderModels` copies `implicitModel.input` onto the merged model entry, so capability metadata from the valid provider is retained instead of being dropped during merge.
