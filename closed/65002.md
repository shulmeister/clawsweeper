## Summary

Current `main` already carries the WhatsApp outbound media fixes this report describes. The gateway `send` path preserves `mediaUrl`/`mediaUrls`, outbound normalization merges them into the delivery plan, WhatsApp derives `hasMedia` from that normalized media list, and the changelog shows this was shipped in `v2026.4.10` with additional WhatsApp outbound hardening in `v2026.4.12`.

## Evidence

- **Gateway send still forwards media into outbound delivery:** `send` normalizes `request.mediaUrl`/`request.mediaUrls`, rejects only truly empty requests, and passes `{ text, mediaUrl, mediaUrls }` into `deliverOutboundPayloads`.
