## Summary

Keep open. Current main does not implement the PR’s per-token Telegram poller registry or the Telegram-only hot-reload drain pause, so this is not implemented on main and there is not enough evidence to close as obsolete. Main does contain a different 409 mitigation, but it only rebuilds the transport after a conflict rather than preventing overlapping pollers.
