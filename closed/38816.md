## Summary

Keep #38816 open. Current main changed the default provider from Anthropic to OpenAI and includes adjacent fixes for provider-prefixed/configured image-capable rows, but the reported providerless `agents.defaults.imageModel` case is still not resolved from configured provider metadata. A bare `moondream`/`qwen2.5vl:7b` image model is still treated as a default-provider model rather than inferred as Ollama/LiteLLM, so the user-visible bug remains in a slightly different form.

## Best Possible Solution

Keep the issue open until OpenClaw either infers a unique configured provider for bare `agents.defaults.imageModel.primary` and fallback entries, matching the chat default model behavior, or explicitly rejects providerless image-model refs with a clear validation/runtime error and docs. Add a regression test for a bare Ollama/LiteLLM-style image model such as `moondream` or `qwen2.5vl:7b` declared under `models.providers`.
