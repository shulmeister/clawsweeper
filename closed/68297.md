## Summary

Current main partially addresses the motivating LAN `ws://` case by persisting `OPENCLAW_ALLOW_INSECURE_PRIVATE_WS` from the install command environment into node services, with docs and tests. It does not implement the requested repeatable `openclaw node install --daemon-env KEY=VALUE` flag, does not expose `daemonEnv` in the node install options, and does not provide arbitrary extra environment injection for node install. The linked implementation PR #68301 is closed unmerged, so there is no canonical landed or open replacement for the exact remaining feature.

## Best Possible Solution

Keep the issue open unless a maintainer explicitly decides the documented env-prefix workflow is sufficient. The concrete implementation path is to add a repeatable `--daemon-env <KEY=VALUE>` option to `src/cli/node-cli/register.ts`, add `daemonEnv?: string[]` to `NodeDaemonInstallOptions`, parse and validate entries in `src/cli/node-cli/daemon.ts`, merge them into the install plan environment before `service.install()`, and add focused CLI/install tests plus docs if the new flag becomes supported.
