## Summary

Current main already carries provider-returned OpenAI-compatible streaming usage through to assistant usage and session accounting. The reported 2026.3.13 drop is covered by current stream parsing, subscribe/event usage capture, guarded transcript append behavior, and focused regression tests; the same core behavior is present in the latest release tag v2026.4.24.

## Best Possible Solution

Close this specific provider-returned usage persistence bug as implemented. Keep the shipped stream usage plumbing and leave separate remaining product requests, such as providers that return no usage at all or UI/tool display enhancements, to their narrower tracking issues.
