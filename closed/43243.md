## Summary

Keep PR #43243 open. Current main still does not make queued command-lane entries abort-aware, and `runEmbeddedPiAgent()` still does not thread `params.abortSignal` into its default enqueue wrappers. The linked #43141 report has a concrete queue-timeout failure path, and this PR remains a plausible focused fix, though it needs rebase/review against current lane changes.

## Best Possible Solution

Keep the PR open and have it rebased into current main. The best path is to add an abort-aware queued-entry removal contract to `src/process/command-queue.ts`, thread `params.abortSignal` into the default `runEmbeddedPiAgent()` session/global enqueue wrappers, and add focused tests for pre-aborted, abort-while-queued, already-active, and task-not-called cases plus a cron/embedded regression that proves a timed-out queued run does not remain in the lane.
