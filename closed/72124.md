## Summary

Keep #72124 open. Current main still routes `openclaw models auth paste-token --provider anthropic` through a validator that accepts only `sk-ant-oat01-`, so the linked `sk-ant-api03-` API-key bug is not implemented on main. The PR is a focused active fix for #72121 rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer review and merge, or land an equivalent narrow fix: widen the Anthropic paste-token validator to accept both documented credential prefixes, add focused regression tests for `sk-ant-api03-`, and keep the changelog attribution. Once that ships on main, #72121 can close as fixed.
