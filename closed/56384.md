## Summary

Keep #56384 open. Current main still resolves Feishu per-group config by raw exact match, then raw case-insensitive match, then wildcard; it does not normalize `chat:`, `group:`, `feishu:`, or `lark:` prefixes before looking up `channels.feishu.groups.<chat_id>`. The PR’s central Feishu fix is therefore not implemented on main, though the branch should be narrowed/rebased because it also carries an unrelated browser test change and still has review discussion about preserving legacy precedence.

## Best Possible Solution

Keep the PR open and ask for a clean Feishu-only rebase against current main. The best code path is to add normalized prefix-aware matching inside `resolveFeishuGroupConfig`, preserve existing exact and raw case-insensitive precedence where possible, add focused regression tests for both prefixed runtime IDs and prefixed config keys, and handle the unrelated browser test fix separately.
