## Summary

Keep open. Current main contains adjacent Bedrock/Anthropic replay hardening for empty stream-error turns and malformed tool-call replay, but the specific requested diagnostic change is still missing: `embedded_run_agent_end` error logging does not include a stack trace, and prompt errors are still formatted down to message/cause text before being logged or persisted.

## Best Possible Solution

Add a bounded, redacted stack field for embedded runner prompt errors and surface it on `embedded_run_agent_end` diagnostics, while keeping the existing sanitized message/hash fields and avoiding raw provider payload leakage. The change likely belongs in the embedded runner error capture plus `handleAgentEnd`, with focused tests proving stack inclusion and redaction.
