## Summary

Current `main` contains a direct fix for agent/session-id misrouting that matches this report's symptom: child startup prompts could land in the wrong transcript and therefore appear as `role: "user"` in the parent/main session history. The fix landed in commit `934dd5b3a7389d10f2191b4674d5fa5a98f94d35` (`[codex] fix agent session-id routing (#70985)`), with coverage in both gateway and session-resolution tests. It is not in the latest tagged release `v2026.4.22` yet.

## Evidence

- **Transcript persistence writes prompts as user messages:** Agent/CLI transcript persistence appends the prompt text to the resolved session transcript as `role: "user"`, so any routing mistake would produce exactly the contamination described in the issue.
  - file: [src/agents/command/attempt-execution.ts:129](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/agents/command/attempt-execution.ts#L129)

This is the relevant routing fix on `main`.
