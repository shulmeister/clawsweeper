## Summary

Current `main` already implements the reported block-streaming dedupe fix. The latest release tag `v2026.4.23` still had exact-key-only suppression, but `main` now tracks streamed text fragments, suppresses the assembled final payload when those fragments exactly cover it, and includes targeted regression tests plus a changelog entry that explicitly references `#70921`.

## Evidence

- **Current main adds content-coverage dedupe in block reply pipeline:** `createBlockReplyPipeline` now stores streamed text fragments after successful block sends and `hasSentPayload()` returns true when the normalized concatenated streamed text exactly matches the final trimmed text, while still refusing media payloads.
