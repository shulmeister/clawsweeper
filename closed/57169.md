## Summary

Keep #57169 open. Current main still matches the reported behavior: macOS nodes advertise the screen capability and declare `screen.record`, but the gateway default node-command policy classifies `screen.record` as dangerous and excludes it from the macOS default allowlist unless `gateway.nodes.allowCommands` opts in. The linked PR #57192 is still only related open work, not a shipped fix, and the current wizard path also seeds `screen.record` into the default denylist for fresh setups.

## Best Possible Solution

Keep the issue open for maintainer/product resolution. The clean fix is either to make `screen.record` an allowed macOS default everywhere, including setup wizard denylist behavior and tests, or to explicitly keep it opt-in and update node docs/CLI status/error text so advertised macOS screen support does not imply `screen.record` is callable without `gateway.nodes.allowCommands`. PR #57192 is related but should not close this issue until the chosen policy is implemented on main.
