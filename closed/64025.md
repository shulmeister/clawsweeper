## Summary

Keep open. Current main has partially addressed plugin installs and package detail lookup, but the central discovery problem remains: ClawHub skill search is still separate from plugin/package search, package query search currently errors for the reported plugin slugs, and OpenClaw docs do not document the plugin/package inspect/explore surface that a maintainer verified exists in clawhub@0.9.0.

## Best Possible Solution

Narrow #64025 instead of closing it: fix ClawHub package search for direct plugin slug/query lookups, expose or clearly document plugin discovery/inspection through `clawhub package ...` or a type-filtered combined search, and update `docs/tools/clawhub.md` and the public ClawHub docs so plugin users can find, inspect, and install packages without falling through the skill-only search path.
