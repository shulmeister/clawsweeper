## Summary

Keep this PR open. The checked-out main still sends the concrete `sessions_spawn` task in both the subagent system prompt and the first child message, so the underlying #72019 bug is not already fixed. This PR is an active focused fix attempt, but its current design still has unresolved maintainer-level tradeoffs around making a user-message kickoff the authoritative task source for persistent and forked subagent sessions. The overlapping #72053 is also open and unmerged, so this should be consolidated by maintainer review rather than auto-closed.

## Best Possible Solution

Keep this PR open for maintainer review and consolidate it with #72019 and #72053. The best final fix should place the long delegated task in exactly one authoritative initial model-visible location, preserve multiline or structured task text, avoid ambiguous user-message task binding for forked transcripts, and add regression coverage for the actual `message` plus `extraSystemPrompt` gateway payload. If #72053 or an equivalent system-bound-task fix lands first, close this PR as superseded by that shipped implementation.
