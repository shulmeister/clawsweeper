## Summary

Keep open. Current `main` still lacks the Feishu card-to-text fallback this PR is proposing, and the PR thread already surfaced unresolved edge-case risks that need maintainer judgment rather than conservative auto-close.

## Evidence

- **Current main still has no card failure fallback:** At `84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5`, the non-streaming Feishu card path directly awaits `sendStructuredCardFeishu()` inside `sendChunkedTextReply()` with no surrounding `try/catch`, so a Feishu card creation failure still aborts delivery instead of retrying as plain text.
  - file: [extensions/feishu/src/reply-dispatcher.ts:513](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/extensions/feishu/src/reply-dispatcher.ts#L513)
  - command: `nl -ba extensions/feishu/src/reply-dispatcher.ts | sed -n '513,533p'`

That matches the PR description that card failures currently drop the reply.
