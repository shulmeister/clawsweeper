## Summary

PR #57060 should close as superseded by the linked canonical issue #56966 and current main’s explicit Telegram group-authorization boundary. The PR changes only the fallback command-auth path, while current Telegram group/native command authorization intentionally keeps DM pairing-store approvals out of group sender authorization and documents that behavior.

## Best Possible Solution

Close PR #57060 as superseded and leave #56966 as the canonical place for any maintainer product decision. If maintainers choose to allow DM-paired Telegram users to run group commands, the replacement should update Telegram group/native command authorization, security docs, and the existing tests that currently assert DM pairing is not a group authorization source.
