## Summary

Keep open. Current main still implements Feishu chat typing through message reactions, and the docs/tests explicitly describe the feature as typing reactions. The related #8722 context explains why the reaction implementation exists, but it does not supersede this newer bug because #69572 specifically reports that reactions do not produce Feishu's native typing status.

## Best Possible Solution

Keep #69572 open for Feishu plugin owners. The fix should verify the official Feishu/Lark native typing API and permissions, replace the chat-message reaction path in `extensions/feishu/src/typing.ts` when supported, update reply-dispatcher tests to assert the native call, and revise `docs/channels/feishu.md` so the config no longer promises reaction-based typing as the normal behavior. If Feishu does not expose a bot-usable native typing API, close only after documenting that product limitation explicitly.
