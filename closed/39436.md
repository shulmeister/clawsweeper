## Summary

Current main intentionally separates Anthropic API keys from legacy Anthropic setup-token auth. This PR would make the setup-token validator accept `sk-ant-api03-` API keys and persist them as `token` profiles, but OpenClaw already exposes Anthropic API keys through a separate `api_key` auth path, so I could not verify a current-main bug on the supported flow.
