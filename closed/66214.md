## Summary

Current main still does not export stringEnum or optionalStringEnum from the root openclaw/plugin-sdk entry, while the helpers are present on supported subpaths and related reports show real third-party breakage. This PR is a plausible focused compatibility fix, but it is incomplete as submitted because current contract tests intentionally lock the root runtime surface and the API baseline/docs would need to be updated if maintainers promote these helpers onto the root surface.

## Best Possible Solution

Keep this PR open for maintainer review. If OpenClaw wants to preserve root-import compatibility for existing external plugins, revise the PR into a complete SDK-surface change: add the root export, update src/plugins/contracts/plugin-sdk-index.test.ts, refresh the plugin SDK API baseline, add/adjust a regression check for the root import, and document/changelog the compatibility behavior. If maintainers choose the narrow-subpath boundary instead, use #68279 as the canonical follow-up and provide a clear migration path for affected plugins.
