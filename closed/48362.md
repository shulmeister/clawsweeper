## Summary

Current `main` no longer uses thread-suffixed ACP session aliases for the supported `/acp spawn ... --thread here` flow. Thread/topic ACP spawns create a normal ACP session key, bind the conversation to that key, and inbound session init rewrites the message to the bound target session key before session lookup. That means the reported `agent:...:acp:...:thread:...` reproduction path does not match current behavior on `main`.
