## Summary

Current main already fixes the Control UI chat-history race that can hide a just-finished assistant reply until a later refresh: terminal chat events still trigger a history reload, but `loadChatHistory` now preserves optimistic assistant/user tail messages when that reload returns an older snapshot.

## Evidence

- **Current main preserves optimistic tail messages on history reload:** `loadChatHistory` now saves the previous in-memory transcript and replaces the visible history with `preserveOptimisticTailMessages(...)`, instead of blindly overwriting `chatMessages` with the server snapshot.
