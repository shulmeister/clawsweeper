## Summary

#54246 duplicates the still-open canonical feature request #51586 for a configurable `/new`/`/reset` session reset prompt. Current `main` still lacks the requested config key, so the right maintainer cleanup is to close this newer issue as a duplicate and keep #51586 open for the remaining implementation work.

## Best Possible Solution

Close #54246 as a duplicate of #51586 and keep #51586 as the canonical implementation tracker. The eventual fix should add a documented config surface, likely `agents.defaults.sessionResetPrompt` plus optional `agents.list[].sessionResetPrompt`, wire it through config types/zod/schema docs, and have the bare reset prompt builder use it while preserving the current default and current-time line behavior.
