## Summary

Current `main` already fixes the reported missing-`qa/scenarios/index.md` crash through broader changes in the QA scenario loader and completion command flow, so this PR's narrower `discovery-eval.ts` try/catch is redundant.

## Evidence

- **Scenario loader now tolerates a missing QA pack:** In released tag `v2026.4.23`, `readTextFile()` returns an empty string when `qa/scenarios/index.md` is absent, and `readQaScenarioPack()` returns an empty scenario pack instead of throwing. The inline comment explicitly says this prevents completion/update consumers from crashing when `qa/scenarios/` is not shipped.
