## Summary

Current `main` already preserves signed Anthropic thinking blocks in the shared replay pipeline used for follow-up turns, including Claude 4.x models such as `claude-opus-4-7`. The replay policy now keeps signatures for Anthropic APIs, avoids mutable replay rewriting when provider-owned thinking replay is allowed, and regression tests cover the exact signed-thinking replay case.

## Evidence

- **Anthropic replay policy preserves signatures for Claude 4.x:** `buildStrictAnthropicReplayPolicy()` enables `preserveSignatures`, `validateAnthropicTurns`, and synthetic tool-result repair; `shouldPreserveThinkingBlocks()` returns true for `opus-4` models, so Claude 4.x does not go down the drop-thinking path.
