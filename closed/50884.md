## Summary

Close as `clawhub`. The PR adds a new Feishu operator skill/script, and current repo policy says new skills should be published to ClawHub rather than bundled into OpenClaw core unless there is a strong product or security reason. Current `main` already resolves Feishu bot identity in the channel runtime, so this is an optional operator workflow, not a missing core seam.

## Evidence

- **Vision excludes new core skills when ClawHub fits:** `VISION.md` says bundled skills are only for baseline UX, that new skills should be published to ClawHub first, and explicitly lists "New core skills when they can live on ClawHub" under "What We Will Not Merge (For Now)".
  - file: [VISION.md:68](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/VISION.md#L68)
- **Current main already fetches Feishu bot identity in runtime:** The Feishu plugin already probes bot identity from the Feishu API and returns `botName`/`botOpenId`, then uses that during monitor startup/account setup.
  - file: [extensions/feishu/src/probe.ts:51](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/extensions/feishu/src/probe.ts#L51)
- **Monitor startup/account path already applies bot identity and retries recovery:** `fetchBotIdentityForMonitor(...)` is used during account startup; `applyBotIdentityState(...)` stores the identity in runtime state, and `startBotIdentityRecovery(...)` retries in the background when needed.
  - file: [extensions/feishu/src/monitor.account.ts:626](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/extensions/feishu/src/monitor.account.ts#L626)
- **No current main consumer for the PR's cache file or slash command:** A repo-wide search on current `main` found no references to `bot-identity.json`, `refresh-bot.sh`, or `/refresh-bot-name`, so the PR's proposed shell script/cache path is not an existing core/runtime integration point.
  - command: `rg -n "bot-identity\\.json|refresh-bot\\.sh|/refresh-bot-name|刷新机器人名称|更新机器人信息" -S extensions src docs skills`
- **Docs point custom skills to local/workspace skills and ClawHub:** Docs describe bundled skills as a baseline set, direct users to ClawHub for more skills, and recommend managed/workspace skill overrides for customizations rather than changing the repo copy.
  - file: [docs/help/faq.md:170](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/docs/help/faq.md#L170)

## Risks / Open Questions

- The workflow may still be useful for Feishu operators; closing here only means it should ship via ClawHub or a local/workspace skill, not bundled into core.
- If maintainers want an officially supported in-product refresh command later, that would need a defined Feishu runtime/config seam rather than an unconsumed shell-script cache file.
