## Summary

Keep open. Current `main` does not implement vLLM `enable_thinking` / `chat_template_kwargs` request wiring. The issue’s secondary ask for per-model `maxTokens` defaults is already supported, but the primary vLLM thinking-toggle request is still missing.

## Evidence

- **vLLM still uses the generic OpenAI-completions path:** `buildVllmProvider()` returns `api: "openai-completions"`, so vLLM requests flow through the shared OpenAI-compatible completions transport rather than a vLLM-specific request builder.
  - file: [extensions/vllm/models.ts:8](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/extensions/vllm/models.ts#L8)
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Shared OpenAI-completions payload builder has no vLLM thinking-toggle fields:** `buildOpenAICompletionsParams()` only adds `max_tokens`/`max_completion_tokens`, `temperature`, tools, and reasoning via `reasoning` or `reasoning_effort`. There is no branch adding `chat_template_kwargs` or `enable_thinking`.
  - file: [src/agents/openai-transport-stream.ts:1705](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/agents/openai-transport-stream.ts#L1705)

  - file: [src/agents/models.profiles.live.test.ts:294](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/agents/models.profiles.live.test.ts#L294)
