## Summary

Keep PR #59439 open. Current main still uses only the caller shell environment for the unmanaged restart preflight probe, while nearby daemon status code already resolves probe auth from merged daemon/service environment. The PR is a plausible fix for a real CLI behavior gap, but the existing review comments identify changes still needed before merge rather than evidence that the work is obsolete.

## Best Possible Solution

Keep this PR open for maintainer review or author revision. The best implementation is to resolve unmanaged restart preflight auth from the daemon/service-command environment, preserve safe fallback behavior when config or secret resolution is unavailable, add a focused regression test for service-command auth with shell auth unset, and coordinate with the broader restart-health auth work in #57374 so only one coherent implementation lands.
