## Summary

Keep open. Current main still has the reported `sessions_spawn(runtime="subagent", streamTo="parent")` failure path, and the latest release tag still contains the same rejection. PR #68397 is an open candidate fix that explicitly addresses #61724, but it is not merged, so closing this issue now would hide an unresolved main-branch bug.

## Best Possible Solution

Keep #61724 open until main lands a fix. The narrow product path is to make `streamTo` ACP-only at read/forward time, silently ignore it for native `runtime="subagent"` calls so they reach `spawnSubagentDirect`, preserve ACP pass-through, and update `src/agents/tools/sessions-spawn-tool.test.ts` to cover both native subagent and ACP behavior. PR #68397 is the current candidate implementation, with maintainer review needed on whether to include `resumeSessionId` in the same silent-drop contract.
