## Summary

Current `main` already hardens the reported crash path: subagent completion catches task-registry write failures instead of letting them reject, and the SQLite task-registry layer now migrates/restores legacy `requester_session_key` rows so they remain writable. The same implementation is present in the shipped `v2026.4.23` tree.

## Evidence

- **Subagent completion no longer lets task-finalization errors escape:** `safeFinalizeSubagentTaskRun` wraps `completeTaskRunByRunId` / `failTaskRunByRunId` in `try/catch` and downgrades failures to a warning (`failed to finalize subagent background task state`) instead of an unhandled rejection.
  - file: [src/agents/subagent-registry-lifecycle.ts:151](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/agents/subagent-registry-lifecycle.ts#L151)
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Regression test covers the exact failure shape:** The test forces `completeTaskRunByRunId` to throw and asserts `completeSubagentRun(...)` still resolves, proving gateway completion is hardened against task-store write failures.
  - file: [src/agents/subagent-registry-lifecycle.test.ts:135](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/agents/subagent-registry-lifecycle.test.ts#L135)
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)

  - file: [src/tasks/task-registry.store.sqlite.ts:326](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/tasks/task-registry.store.sqlite.ts#L326)
