## Summary

Current main already honors `gateway.controlUi.dangerouslyDisableDeviceAuth` without a shared-auth requirement for operator Control UI sessions, and the behavior is covered by unit and gateway suite tests. The shipped release `v2026.4.22` contains this implementation.

## Evidence

- **Pairing bypass no longer depends on shared auth:** `shouldSkipControlUiPairing` no longer takes `sharedAuthOk`; after trusted-proxy and `auth.mode=none` handling, it returns `role === "operator" && policy.allowBypass`, so `dangerouslyDisableDeviceAuth` bypasses pairing for operator Control UI sessions regardless of shared auth.
