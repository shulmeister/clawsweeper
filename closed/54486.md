## Summary

Keep PR #54486 open. Current main has a manual `openclaw sessions cleanup --fix-missing` repair path, but it does not implement the PR's automatic gateway/session-store write cleanup for orphaned `sessions.json` rows, and the gateway/session listing path still builds rows from store entries even when transcripts are missing.

## Best Possible Solution

Keep the PR open. The best path is to rebase/review it against current main, share robust transcript-candidate resolution with the existing cleanup/Doctor surfaces where practical, keep metadata-only sessions safe, retain throttling/grace behavior, and split any unrelated markdownlint/doc-sentinel change if maintainers want a narrower gateway/session-store fix.
