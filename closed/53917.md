## Summary

Current main solves the Feishu local reply-image failure by normalizing and staging local reply attachments before the Feishu reply dispatcher sends them. The dispatcher still does not pass `mediaLocalRoots` directly to `sendMediaFeishu`, but the observable bug is obsolete because normal reply delivery no longer hands the original host-local path to Feishu media upload.

## Best Possible Solution

Close this issue as implemented on main and leave the current staged-reply-media design in place. Related open PRs such as #49082, #51375, and #58591 should be reviewed as superseded by the current normalizer-based fix unless they identify a remaining account-config semantics gap with a fresh reproduction.
