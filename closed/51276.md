## Summary

Keep open. Current `main` does not implement cron `payload.kind="exec"`, and the PR as submitted still has unresolved correctness gaps called out in review: the public cron protocol/tool/docs only admit `systemEvent` and `agentTurn`, and the proposed exec runner had reported process-kill and shell-argument bugs.

## Evidence

- **Current cron payload type on main excludes exec:** `CronPayload` and `CronPayloadPatch` on current `main` only include `systemEvent` and `agentTurn`; there is no `exec` variant in the shipped type contract.
  - file: [src/cron/types.ts:112](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/cron/types.ts#L112)

  - file: [src/cron/service/jobs.ts:156](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/cron/service/jobs.ts#L156)
