## Summary

The reported `openclaw-message`/`openclaw-doctor` SEA-binary OOM path no longer matches current `main` or the latest shipped release: OpenClaw now ships a single `openclaw` Node wrapper, runs doctor through `node <entry> doctor`, and lazy-loads the `message` and `doctor` command groups to keep startup narrower.

## Evidence

- **Latest release no longer ships standalone subcommand binaries:** In release `v2026.4.22`, `package.json` exposes only one bin, `openclaw`; there is no `openclaw-message`, `openclaw-doctor`, or `openclaw-onboard` binary surface anymore.
  - file: [package.json:16](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/package.json#L16)
  - command: `git show v2026.4.22:package.json | nl -ba | sed -n '1,24p'`
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **CLI entry is a Node wrapper, not a SEA sub-binary:** `openclaw.mjs` is a `#!/usr/bin/env node` bootstrap that imports `dist/entry.js`/`dist/entry.mjs`, so current installs run through Node and its normal runtime flags instead of a separate `openclaw-message` executable.
  - file: [openclaw.mjs:1](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/openclaw.mjs#L1)

- **Post-update doctor already uses `node <entry>`:** Current update flow resolves the installed entrypoint and invokes doctor as `[resolveNodeRunner(), entryPath, "doctor", "--non-interactive"]`, which is the opposite of the old standalone `openclaw-doctor` path discussed in the issue comments.
  - file: [src/cli/update-cli/update-command.ts:417](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/cli/update-cli/update-command.ts#L417)

- **`message` and `doctor` are now lazy-loaded command groups:** The core CLI registers `doctor` via `register.maintenance.js` and `message` via `register.message.js` through imported command-group specs, which keeps those heavy paths off the generic startup path until invoked.
  - file: [src/cli/program/command-registry-core.ts:80](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/cli/program/command-registry-core.ts#L80)
