## Summary

Keep open. Current main still lacks the nested Claude result unwrapping that #66819 proposes, and the PR contains a focused fix plus regression coverage for the linked #66743 behavior. The bot review concern about over-broad unwrapping was addressed in the PR context by scoping the behavior to Claude JSON output and adding a non-Claude preservation test.

## Best Possible Solution

Keep #66819 open for maintainer review or landing. The best path is to land a scoped implementation that unwraps nested Claude result envelopes only for Claude-compatible CLI result outputs, preserves non-Claude JSON payloads verbatim, and keeps the focused parser regression tests from the PR.
