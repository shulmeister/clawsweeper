## Summary

Close #70531 as a duplicate of the older open tracker #20950. Current main still lacks per-job cron definition files, but #20950 already tracks the same one-file-per-job/jobs.d storage request and #41206 is the open implementation candidate for that direction.

## Best Possible Solution

Close #70531 as a duplicate and keep #20950 as the canonical open tracker. The implementation should proceed through #41206 or a replacement patch that adds a backward-compatible directory-backed cron definition store, preserves or replaces the current jobs-state sidecar safely, updates schema/docs/doctor behavior, and includes migration and per-job write tests.
