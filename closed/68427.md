## Summary

Current `main` and the latest release already implement the requested Discord attachment total-timeout behavior with `AbortController`, per-fetch timeout enforcement, and regression tests.

## Evidence

- **Inbound Discord processing passes a dedicated attachment total timeout into both direct and forwarded media resolution:** `processDiscordMessage` builds `mediaResolveOptions` with `readIdleTimeoutMs: DISCORD_ATTACHMENT_IDLE_TIMEOUT_MS` and `totalTimeoutMs: DISCORD_ATTACHMENT_TOTAL_TIMEOUT_MS`, then uses the same options for both `resolveMediaList` and `resolveForwardedMediaList`.
