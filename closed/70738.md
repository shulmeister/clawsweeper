## Summary

Current main and the v2026.4.24 release already include a macOS Channels settings implementation that keeps channel options visible via a fallback channel list, then refreshes with the gateway-provided channel catalog/status once available.

## Best Possible Solution

Close this issue as implemented by the shipped macOS Channels settings flow. Any remaining problem should be handled as a new, narrower macOS UI report with the current OpenClaw version, install method, logs, and a fresh screenshot if channel options still disappear.
