## Summary

Keep open. Current main has operator-facing timeout controls for slow local models, including the LLM idle-timeout knob and provider-scoped Ollama request timeout, but the issue also asks for agent/LLM-side timeout adjustment and that path is still intentionally blocked by the gateway config mutation allowlist. Timeout profile cooldown/faster fallback also remains covered by the still-open related #51057.

## Best Possible Solution

Keep open for maintainer triage. The practical path is to make the supported operator recipe explicit for slow Ollama models, using openclaw config set agents.defaults.llm.idleTimeoutSeconds <seconds> and models.providers.ollama.timeoutSeconds plus keep_alive where appropriate, then decide whether owner-only gateway config.patch/apply should allow bounded edits to those timeout keys. Keep timeout auth-profile cooldown and faster fallback policy aligned with #51057.
