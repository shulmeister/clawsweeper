## Close Comment

Closing this as a `clawhub`-scope PR after Codex review. Current main is explicit that new plugins should generally live outside the core repo: `VISION.md` says plugin authors should host and maintain plugins in their own repository, `docs/plugins/building-plugins.md` says you do not need to add your plugin to the OpenClaw repository, and `docs/plugins/community.md` says community plugins should be published on ClawHub or npm instead. This PR is structured as a standalone third-party package (`@wayazi/rust-plugin`) with its own publishing/docs/release surface rather than a focused change to an existing bundled OpenClaw extension, so it fits the external plugin path better than OpenClaw core. If you want to ship it, the right path is to publish and maintain it as a community plugin on ClawHub/npm and link users to `openclaw plugins install`.
