## Summary

Current main already addresses #70717's central startup/RSS regression path. The issue discussion and #70730 correctly identified over-broad plugin startup/loading as the likely cause, but main now computes a bounded Gateway startup plugin set from the cold plugin registry, passes that set into the loader, avoids unscoped plugin loading when the set is empty, and prevents persisted channel auth alone from activating channel startup work. #70730 was closed unmerged because that mainline fix supersedes the proposed scanDir precedence reversal.

## Best Possible Solution

Close #70717 as implemented on main and keep the current cold-registry, bounded startup plugin-loading implementation. Do not revive #70730's scanDir precedence reversal; if high RSS still reproduces after these mainline changes ship, open a narrower follow-up with current version, startup plugin IDs, loaded plugin count, and fresh RSS samples.
