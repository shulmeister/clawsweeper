## Summary

Keep open. The report is recent, concrete, unprotected, and the current main code still creates chokidar-backed memory watchers on the manual `openclaw memory index --force` path for both builtin and QMD managers. Current tests cover watcher configuration and forced reindex behavior, but I did not find evidence that EMFILE during watcher setup is handled or avoided for the one-shot CLI index path.

## Best Possible Solution

Keep the issue open and fix the manual memory-index path directly: add a one-shot/manual sync mode that does not create chokidar watchers for `openclaw memory index --force` and `memory status --index`, or catch watcher EMFILE and continue the explicit reindex with a warning. Cover both builtin and QMD managers with regression tests that mock `chokidar.watch` throwing EMFILE during CLI index manager creation, and document `agents.defaults.memorySearch.sync.watch=false` as a temporary workaround only.
