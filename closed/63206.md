## Summary

Keep this PR open. Current main still skips symlinked plugin directories during automatic plugin-root scanning, and the PR is a focused implementation candidate for the same real gap tracked by #36754 and confirmed in discussion by an affected `lossless-claw` setup.

## Best Possible Solution

Keep this PR open. The best path is to refresh it onto current main, keep the narrow `discoverInDirectory` symlink target classification, retain the global-root regression test, preferably add the same workspace-root coverage, and land it after targeted plugin discovery validation. This should resolve the remaining behavior tracked by #36754.

## Evidence
