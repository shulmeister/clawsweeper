## Summary

Current main still lacks the PR's central behavior: generic inbound channel MediaPath/MediaPaths are rendered into prompt text as media notes, but are not loaded and merged into GetReplyOptions.images before the agent run. The PR discussion also corroborates the bug on Signal, so this remains a plausible focused fix rather than cleanup-close material.

## Best Possible Solution

Keep the PR open and either land an adapted version or replace it with a current-main patch that resolves image/* local MediaPath/MediaPaths into ImageContent[], merges them with existing replyOptions.images, preserves webchat behavior, and adds regression tests for Telegram/Signal-style inbound MediaPaths plus non-image and oversized attachment safeguards.
