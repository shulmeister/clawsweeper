## Summary

Keep open. Current main has a partial pnpm-stable-path fix for symlinked dist entrypoints, but it still does not implement this PR's versioned `.pnpm/.../openclaw.mjs` fallback to the stable `node_modules/openclaw/dist/...` path. The PR remains a focused gateway service install fix with a relevant regression test proposal.

## Best Possible Solution

Keep the PR open for maintainer review or reimplementation on current main. The best product/code path is to add a pnpm package-root candidate in `src/daemon/program-args.ts` that maps versioned `.pnpm/<pkg>@<version>/node_modules/<pkg>/openclaw.mjs` inputs back to stable `node_modules/<pkg>/dist/*`, with the PR's regression test in `src/daemon/program-args.test.ts`, then validate the daemon changed lane.
