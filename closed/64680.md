## Summary

Close as `clawhub`: current `main` treats memory behavior as plugin-owned policy on the existing memory slot, and it already ships bundled automatic memory flows (`memory-core` flush/dreaming). A stricter mandatory auto-write policy fits a memory plugin or plugin-API extension, not OpenClaw core.

## Risks / Open Questions

- The issue asks for an official built-in default, so a maintainer could still choose later to bless or bundle one specific memory plugin. This review only supports closing the core issue on scope grounds, not rejecting the idea itself.
