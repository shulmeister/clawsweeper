## Summary

PR #57102 is superseded. Its linked Windows ACPX queue-owner report (#56345) has since been closed as implemented, and the queue-owner recovery work was replaced by the merged #58669/runtime path plus the newer embedded `acpx/runtime` integration. The PR's Windows TTL/default patch no longer matches current main because `queueOwnerTtlSeconds` is now legacy compatibility config and is ignored by the ACPX plugin.

## Best Possible Solution

Close #57102 as superseded. Keep the shipped/current ACPX runtime integration and the merged #58669 recovery behavior; do not revive the stale Windows TTL patch. If a fresh current-main Windows queue-owner failure exists, it should be reported as a new focused issue against the current `acpx/runtime` behavior with logs and versions.
