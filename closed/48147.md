## Summary

Keep #48147 open. Current main still schedules `GatewayClient` reconnects with `.unref()`, so the linked node-host exit bug is not implemented on main. The PR remains a relevant gateway bug fix, and its latest commit addresses the review concern about clearing ref'd reconnect timers on `stop()` and manual `start()` during backoff. Related PR #49723 overlaps but does not clearly supersede this one; #48147 has the stronger process-level keepalive regression test and later bot-review follow-up.

## Best Possible Solution

Keep #48147 open or explicitly mark it as the canonical implementation after reconciling it with #49723. The desired fix belongs in `src/gateway/client.ts`: store the reconnect timeout, keep it ref'd so node-host stays alive through backoff, clear it on `stop()` and manual `start()`, and keep the process-level plus fake-timer regression coverage from this PR.
