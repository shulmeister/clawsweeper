## Summary

Current main no longer matches the reported Slack-channel allowlist failure path. The Slack monitor now matches channel IDs case-insensitively during allowlist resolution, and regression tests cover the lowercase-config/uppercase-runtime case that would otherwise silently drop allowed channel events.

## Evidence

- **Current allowlist matching is case-insensitive:** `extensions/slack/src/monitor/channel-config.ts:108` adds both lowercase and uppercase channel-ID candidates before resolving the configured channel entry, so Slack runtime IDs like `C0ABC12345` match configs written as `c0abc12345`.
