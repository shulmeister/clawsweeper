## Summary

Current `main` does not show a gateway/plugin-schema mismatch here. OpenClaw intentionally applies plugin `configSchema` to `plugins.entries.<id>.config`, and plugin runtime config is likewise read only from that `config` object. The issue’s example places `embedding` one level too high, so the reported validation error matches the documented contract rather than a core bug.

## Evidence

- **Schema merge targets `plugins.entries.<id>.config`:** `applyPluginSchemas` clones the base plugin entry schema and writes each plugin manifest's `configSchema` into the entry's `config` property (`entryObject.properties.config = nextConfigSchema`).
  - file: [src/config/schema.ts:321](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/config/schema.ts#L321)
  - command: `nl -ba src/config/schema.ts | sed -n '320,355p'`
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Runtime only reads plugin config from `entry.config`:** `resolvePluginConfigObject` looks up `config.plugins.entries[pluginId]` and returns only `entry.config`; sibling keys such as `embedding` are not exposed as plugin config.
  - file: [src/plugin-sdk/config-runtime.ts:15](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/plugin-sdk/config-runtime.ts#L15)

  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)

  - file: [docs/plugins/manifest.md:513](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/docs/plugins/manifest.md#L513)

  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)

  - file: [docs/tools/plugin.md:138](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/docs/tools/plugin.md#L138)
