## Summary

Keep this PR open. Current main still ships the bundled Bonjour plugin through unpatched @homebridge/ciao@1.3.6, and the upstream ciao Windows ARP polling exec path still lacks a windowsHide options object. The linked Windows report #71492 gives a matching process trace and Bonjour-disable workaround, while broader overlapping PR #72332 is still open and not a merged canonical replacement.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this narrow @homebridge/ciao@1.3.6 pnpm patch, or replace it with an upstream ciao version bump or reviewed broader PR that demonstrably adds windowsHide to the Windows ARP polling exec path. If #72332 becomes the accepted canonical fix and lands first, then close this PR as superseded.
