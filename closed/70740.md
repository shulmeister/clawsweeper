## Summary

Current main and the latest shipped release already document the intended split: `agents.list` is for isolated routed agent profiles, spawned subagents are the default delegation path, persistent child sessions are an explicit `sessions_spawn` mode, and manager-of-managers hierarchies are not the default OpenClaw architecture.

## Evidence

- **Multi-agent docs define `agents.list` as isolated routed profiles:** The multi-agent routing docs explicitly describe agents as isolated workspaces/state/auth/session stores hosted side-by-side in one gateway, with bindings routing inbound traffic to the right agent.
  - file: [docs/concepts/multi-agent.md:8](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/docs/concepts/multi-agent.md#L8)

  - file: [docs/tools/subagents.md:10](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/docs/tools/subagents.md#L10)
