## Summary

Current main addresses the reported false-negative for a resolvable SecretRef-backed local gateway token: `openclaw status` now resolves gateway probe auth through the async SecretInput path before probing, that resolver supports file SecretRef providers, and authenticated loopback probes reuse an existing cached operator device identity for read/presence probes.
