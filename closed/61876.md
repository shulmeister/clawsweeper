## Summary

Current main already routes Feishu audio message attachments into OpenClaw's shared media-understanding transcription path. The implementation does not export QQBot's private STT helper or add a Feishu-specific channels.feishu.stt key, but the user-visible problem is covered by the supported tools.media.audio pipeline.
