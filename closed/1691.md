## Summary

Current main already handles this request: non-native OpenAI Responses endpoints strip `prompt_cache_key`/`prompt_cache_retention` automatically, and model compat now exposes `supportsPromptCacheKey` for explicit override. The changelog shows the automatic strip shipped in `2026.3.22` and the compat override landed later in unreleased notes.

## Evidence

## Risks / Open Questions

- The original reporter also mentioned remaining vLLM 500s after locally removing `prompt_cache_key`, so there may still be a separate OpenAI-compatible request-shape bug unrelated to this specific opt-out request.
- The explicit override is model-level via `compat.supportsPromptCacheKey`, not a dedicated top-level provider flag with the exact proposed name `disablePromptCacheKey`.
