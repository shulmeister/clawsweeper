## Summary

Current main already has the requested safety layer for the startup-breaking self-modification surfaces described here: validated config writes, last-known-good config recovery on startup and hot reload, rejected/clobbered payload preservation, prompt-facing recovery warnings, and plugin install-index rollback when paired config commits fail.

## Best Possible Solution

Close this issue and keep the shipped narrow safety model: validated config writes, last-known-good recovery for startup/hot-reload failures, and plugin-index rollback around config commit conflicts. New requests for additional artifact classes should be opened as narrower follow-ups with a concrete missing path or extension API gap.
