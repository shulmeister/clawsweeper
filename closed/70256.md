## Summary

Current `main` already covers both parts of #70256: generated `models.json` entries now carry provider `apiKey` markers sourced from config/auth discovery, and media-tool lookup falls back to provider-prefixed custom model IDs. The same implementation is present in release `v2026.4.23`.

## Evidence

- **Provider catalogs now inject apiKey into generated models.json entries:** `buildSingleProviderApiKeyCatalog()` and `buildPairedProviderApiKeyCatalog()` both copy the resolved provider `apiKey` into the catalog output, which is what Pi `ModelRegistry` validates when custom models are present.
  - file: [src/plugins/provider-catalog.ts:25](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/src/plugins/provider-catalog.ts#L25)
  - command: `nl -ba src/plugins/provider-catalog.ts | sed -n '25,75p'`
  - sha: [2011de69d3dc](https://github.com/openclaw/openclaw/commit/2011de69d3dc8f1a11ebc774ba4af23f54892461)
- **models.json normalization backfills missing provider apiKey for custom-model providers:** `normalizeProviders()` detects providers with custom `models` and no `apiKey`, resolves an auth/profile/env-backed key marker, and writes it back before `models.json` is emitted.
  - file: [src/agents/models-config.providers.normalize.ts:92](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/src/agents/models-config.providers.normalize.ts#L92)
