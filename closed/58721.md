## Summary

Current `main` treats skills as documentation-only prompt inputs and routes enforceable behavior through plugins, hook packs, and compatible bundles. The linked progress-bar use case fits a ClawHub-distributed hook/bundle/plugin, not new core SKILL.md enforcement semantics.

## Risks / Open Questions

- The linked `agent-progress-bar` ClawHub skill is a valid user need, but the author would need to publish it as a hook pack, compatible bundle, or native plugin to get hard enforcement.
- `openclaw skills install` remains a skill-only content path; this close assumes maintainers want enforcement/distribution to stay on the existing plugin/hook/bundle path rather than expanding plain skill semantics.
