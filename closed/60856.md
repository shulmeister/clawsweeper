## Summary

Keep open. Current main still has a real remaining Ollama memory-index timeout path: the memory manager gives only provider id `local` the 10-minute local batch timeout, while the bundled Ollama embedding provider uses id `ollama`, so Ollama batches still take the 120-second remote timeout and emit the reported `memory embeddings batch timed out after 120s` message. Main also lacks the `embeddingBatchTimeoutSeconds` config override proposed in related PR #67616, which remains the canonical open fix candidate in the provided context.

## Evidence
