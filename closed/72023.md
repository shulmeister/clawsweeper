## Summary

Keep PR #72023 open. Current main still scans persisted ACP sessions with pending stable identities during startup without excluding `mode: "oneshot"`, so the warning reported in #72013 can still occur for historical one-shot records. The PR head remains a focused, non-obsolete fix: it adds the one-shot skip before the pending-identity reconcile path and a regression test for that case.

## Best Possible Solution

Keep #72023 open and land this PR or an equivalent maintainer patch that skips `session.acp.mode === "oneshot"` before startup pending-identity reconciliation, with the focused regression test. Do not close as implemented unless current main gains that explicit startup skip or another source-level change demonstrably prevents historical one-shot pending records from reaching the warning path.
