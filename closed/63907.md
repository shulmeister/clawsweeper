## Summary

Keep #63907 open. Current main does not restore `openclaw run`, and the same `plugins.allow excludes "run"` fallback path is still present for an unregistered primary command under a restrictive allowlist. The linked PR #64112 is relevant, but the provided context says it is still open/unmerged and it narrows the misleading fallback rather than proving the requested compatibility behavior is shipped.

## Best Possible Solution

Keep the issue open until maintainers choose and ship the compatibility path: either add `openclaw run` as a backwards-compatible alias to `openclaw agent` that is not gated by `plugins.allow`, or explicitly retire it with a doctor/docs/changelog migration and merge the fallback-message fix from #64112. Add a regression test using a restrictive `plugins.allow` configuration either way.
