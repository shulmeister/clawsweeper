## Summary

Keep #67172 open. Current main still derives isolated cron run failure status from `payload.isError`/run-level error state, not denial text in `summary`, `outputText`, `synthesizedText`, or payload text. The related fix PR #67186 proposed exactly this missing denial-token classifier, but it is closed/unmerged, and current cron tests do not cover narrated denial tokens. Reporter and later commenter discussion confirms this is a real operator-facing false-green cron problem, not just a speculative edge case.

## Best Possible Solution

Implement a narrow denial-token classifier in `src/cron/isolated-agent/helpers.ts` and regression tests in `src/cron/isolated-agent.helpers.test.ts`. It should scan the resolved summary/output/final assistant-visible text and relevant payload text for machine denial markers and carefully chosen human denial phrases, set the outcome to fatal, and surface a useful embedded error so `applyJobResult` can populate `lastErrorReason`. PR #67186 is the closest prior implementation direction, but it should be reapplied or superseded on current main with maintainer-reviewed token scope.
