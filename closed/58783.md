## Summary

Current `main` has topic-aware Telegram native exec-approval routing, suppresses the fallback forwarding path for matching Telegram approval surfaces, and finalizes expired approvals by clearing actions instead of posting a separate timeout/failure chat message. That matches the reported leak and is strong enough to close as already implemented.

## Evidence
