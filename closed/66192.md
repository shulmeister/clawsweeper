## Summary

Current main already solves the user-visible stuck TUI activity problem well enough: ordinary late terminal chat events are adopted when no active non-BTW run is tracked, and the shipped #67401 watchdog resets stale streaming state after delta silence. The remaining PR fallback would broaden global activity updates to local /btw side runs, which conflicts with the documented and tested side-result isolation contract.

## Best Possible Solution

Close this PR and keep the shipped current-main behavior: normal late terminal events already reset through the active-run path, #67401 provides the broader watchdog safety net, and any future regression test or fix for stuck activity should preserve /btw side-result isolation instead of adding a blanket `activeChatRunId == null` activity-status fallback.
