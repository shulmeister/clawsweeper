## Summary

Current main already contains the auth-profile propagation safeguards this report asked for: locked fresh-store writes, in-process cache/runtime-snapshot refresh, and regression coverage for stale-snapshot updates. The same implementation is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

## Evidence
