## Summary

Keyword-triggered model switching is already supported as plugin scope on current main via `before_model_resolve` hooks and plugin config, and project vision says optional capability should ship through plugins/ClawHub rather than new core behavior.

## Risks / Open Questions

- OpenClaw does not currently ship a built-in declarative keyword-routing feature in core; this close assumes a plugin-based solution is acceptable.
- No existing ClawHub plugin was linked in the issue discussion, so the user may still need someone to package this hook as a reusable plugin.
