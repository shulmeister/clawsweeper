## Summary

Keep open. Current main still uses the old `isStreaming` gate in the exact steer-message paths this PR targets, and the linked bug/follow-up work is not shipped or superseded by a merged PR. The PR should not land as-is because the current branch is dirty and has an unresolved P1 review about handles without stop-state support, but the underlying fix remains actionable in this repo.

## Best Possible Solution

Keep #65010 open, but require a clean rebase/narrowed diff and a fix for the stop-state fallback before merge. The implementation should make the active-but-not-stopped lifecycle predicate available for every backend that can queue steer messages, preserve safe fallback behavior for older handles, and keep targeted tests for `queueEmbeddedPiMessage`, `queueReplyRunMessage`, and `runReplyAgent` steer gating. Coordinate it with #48003, #65221, and #65222 rather than closing it as already handled.
