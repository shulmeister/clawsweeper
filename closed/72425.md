## Summary

Keep #72425 open. Current main still sends Google Live tool responses without the original function name, while Google’s Live API examples and the linked PR #72426 both point at preserving call-id-to-name state and including `name` in each `FunctionResponse`.

## Best Possible Solution

Keep the issue open. The best maintainer path is to review and land PR #72426 or an equivalent narrow fix that preserves the original Google Live function name by call id, includes `name` in every submitted `FunctionResponse`, fails closed when the name is missing, and updates regression tests. Maintainers should also check whether `src/gateway/voiceclaw-realtime/gemini-live.ts` needs the same fix before considering the bug fully resolved.

## Evidence
