## Summary

Telegram sender device type is not actionable in OpenClaw’s current Telegram channel because the bundled Telegram plugin is a Bot API/grammY integration and the Bot API update/message payload does not expose sender client application, api_id, or device class. Current main exposes the Telegram sender identity fields it actually receives, but there is no reliable upstream field to map into device_hint.

## Best Possible Solution

Close this OpenClaw core issue as not actionable with the current Telegram Bot API. Keep the shipped Telegram plugin behavior unchanged; if upstream Telegram exposes reliable sender device metadata later, implement it as an opt-in Telegram config field with schema/docs/tests. Any alternate MTProto/userbot channel should be handled as a separate plugin/product proposal with explicit maintainer review.
