## Summary

Current main no longer contains the hardcoded ACPX 0.1.16 path described in the report. The bundled ACPX plugin now pins `acpx` from package metadata (`0.5.3` on current main / latest release), and bundled runtime repair derives install specs from that package metadata instead of a separate stale literal.
