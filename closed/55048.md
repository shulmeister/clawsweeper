## Summary

Current main already routes Discord message delete through the generic channel-message delete path. The delete action is normalized as a target-based message action, mapped to Discord `deleteMessage`, executed in the messaging runtime, and the REST helper deletes by `channelId`/`messageId` without any guild requirement. The reported DM-only guard string is not present in current source.

## Best Possible Solution

Keep the shipped Discord message-action implementation. If someone can still reproduce this on `v2026.4.24` or newer, the next issue should include the exact command/tool payload, OpenClaw version, Discord account config shape, and logs showing which non-current dispatch path is returning the DM-channel guard.
