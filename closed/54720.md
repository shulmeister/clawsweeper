## Summary

Close #54720 as a duplicate of #46656. Current main still lacks WebChat/Control UI inline-button support, but #46656 is the older and more detailed open tracker for the same remaining work: button payload/config support, callback handling, and Control UI rendering.

## Best Possible Solution

The eventual fix should add a WebChat/Control UI button contract across gateway protocol/config, assistant display content assembly, Control UI message normalization/rendering, and a click-to-callback or click-to-chat.send path, with docs and regression coverage. Close #54720 to avoid splitting the same work across duplicate issues.
