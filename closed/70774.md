## Summary

Current `main` already supports the requested `claude --bare` path through the existing `agents.defaults.cliBackends.claude-cli.args` and `resumeArgs` override surface, so this does not need a new core `slim` setting to be usable.

## Evidence

- **Bundled Claude CLI backend is argv-driven:** The Anthropic plugin registers `claude-cli` with explicit default `args` and `resumeArgs`, including the Claude flags OpenClaw sends by default. This shows the backend invocation is config-driven rather than hard-coded in the runner.
  - file: [extensions/anthropic/cli-backend.ts:29](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/extensions/anthropic/cli-backend.ts#L29)
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **User config can replace Claude argv:** CLI backend resolution merges user config over the bundled backend and uses `args: override.args ?? base.args` plus `resumeArgs: override.resumeArgs ?? base.resumeArgs`, so a user can replace the default Claude argv with `--bare` on fresh and resumed sessions.
  - file: [src/agents/cli-backends.ts:135](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/agents/cli-backends.ts#L135)
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Runner preserves configured backend args:** The CLI runner starts from `baseArgs` and then appends model, system-prompt, session, prompt, and image flags. It does not rewrite the configured backend argv beyond those standard additions.
  - file: [src/agents/cli-runner/helpers.ts:367](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/agents/cli-runner/helpers.ts#L367)
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Public docs already document the override path:** The user-facing CLI backend docs explicitly say Claude-specific modes should be forced with raw backend args under `agents.defaults.cliBackends.claude-cli.args` and matching `resumeArgs`. That existing path covers passing `--bare`.
  - file: [docs/gateway/cli-backends.md:170](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/docs/gateway/cli-backends.md#L170)
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Tests cover raw Claude arg overrides:** Tests verify that explicit Claude `args` and `resumeArgs` overrides are respected and win over default permission-mode injection, which is the same override surface needed for `--bare`.
  - file: [src/agents/cli-backends.test.ts:540](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/agents/cli-backends.test.ts#L540)
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Shipped in latest release:** The latest release commit already contains the same Anthropic CLI backend defaults and the same docs text describing raw `claude-cli` arg overrides, so this capability is not just on unreleased `main`.
  - file: [extensions/anthropic/cli-backend.ts](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/anthropic/cli-backend.ts)

## Risks / Open Questions

- There is no first-class `slim`/`bare` boolean in config; users currently need to set both `args` and `resumeArgs` themselves if they want `--bare`.
