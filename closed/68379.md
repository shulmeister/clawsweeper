## Summary

Current main already addresses the reported dreaming-bloat failure mode: dreaming is opt-in/off by default, session-ingestion is bounded and skips dreaming-generated transcripts, and narrative cleanup now deletes/scrubs stale transcript artifacts. The separate session-memory behavior in the issue is a configurable hook that only runs on `/new` or `/reset`, not a background memory-core dreaming path.
