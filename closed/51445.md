## Summary

Close as `cannot_reproduce`: the PR assumes Feishu code `99991401` is a missing-scope permission error, but current OpenClaw reserves the permission-grant flow for `99991672`, and Feishu documents `99991401` as an IP-whitelist denial instead. Routing `99991401` through `extractPermissionError()` would send users to the wrong remediation path. ([feishu.apifox.cn](https://feishu.apifox.cn/doc-1939850))

## Evidence

- **Current permission extraction on main:** `extractPermissionError()` on current `main` only recognizes Feishu code `99991672` before returning a `FeishuPermissionError` with a grant URL.
  - file: [extensions/feishu/src/bot-sender-name.ts:43](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/extensions/feishu/src/bot-sender-name.ts#L43)
  - command: `nl -ba extensions/feishu/src/bot-sender-name.ts | sed -n '1,180p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **User-facing remediation path:** When `permissionErrorForAgent` is present, the Feishu agent body explicitly tells the user to have an admin authorize permissions via the grant URL. Expanding this path changes the remediation shown to users.
  - file: [extensions/feishu/src/bot.ts:183](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/extensions/feishu/src/bot.ts#L183)
  - command: `nl -ba extensions/feishu/src/bot.ts | sed -n '176,230p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **PR-cited tests do not prove a scope error:** The tests cited in the PR body use `99991401` only as a generic non-withdrawn thrown error and as a non-backoff code; neither test asserts that `99991401` is a permission-scope failure.
  - file: [extensions/feishu/src/send.reply-fallback.test.ts:255](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/extensions/feishu/src/send.reply-fallback.test.ts#L255)
  - command: `nl -ba extensions/feishu/src/send.reply-fallback.test.ts | sed -n '230,290p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **PR-cited tests continue to treat 99991401 as non-special:** `typing.test.ts` explicitly expects `99991401` to be `false` for `isFeishuBackoffError()` and `undefined` for `getBackoffCodeFromResponse()`, which shows the existing test corpus does not classify it as a scope-grant condition.
  - file: [extensions/feishu/src/typing.test.ts:36](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/extensions/feishu/src/typing.test.ts#L36)
  - command: `nl -ba extensions/feishu/src/typing.test.ts | sed -n '1,120p'`

- **Vendor error-code meaning:** Feishu's generic error-code table defines `99991401` as `ip %s is denied by app setting` and `99991672` as missing required scopes; Feishu's IP-whitelist guide separately says requests outside the whitelist are rejected with `99991401`.
