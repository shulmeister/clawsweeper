## Summary

Keep #54061 open. Current main has same-target final reply suppression, but that is the behavior the issue already called insufficient: there is still no `replyMode: "tool-only"` / explicit-send mode, and the mid-turn block/assistant-text paths are not globally suppressed when the message tool is used. The provided discussion also includes an explicit maintainer note from 2026-04-25 saying this remains open and points to still-open PR #57914 as the closest proposed fix.

## Best Possible Solution

Keep the issue open and either land/supersede PR #57914 with an `agents.defaults.replyMode: "tool-only"` style config, or implement an equivalent explicit-send mode that suppresses all auto-delivered assistant text for the relevant turn while preserving cross-target sends, media behavior, and existing final-reply dedupe tests. Add coverage for Discord/Slack-style message tool sends with nonmatching narration text and any block-reply path that can emit before the message tool completes.
