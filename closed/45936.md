## Summary

Keep #45936 open. Current main still has the Feishu quoted/read interactive-card parsing gap, and this PR contains relevant parser/test work for legacy card formats that is not shipped. It is not merge-ready as-is because the branch is dirty/unmergeable, includes unrelated generated/context-engine changes, and overlaps other open Feishu card parser PRs, but that calls for maintainer consolidation rather than cleanup closure.

## Best Possible Solution

Keep #45936 open for maintainer consolidation. The best path is a clean Feishu-only parser patch, likely shared by `send.ts` read/quote handling and inbound `bot-content.ts`, with focused tests for schema 1.0/2.0 elements, header/title text, `plain_text`/`lark_md`, notes/columns, legacy rich-text `content`, raw-card fallback where needed, sanitization, and traversal/output bounds. Remove the generated baseline and unrelated context-engine changes from this branch or move the useful parser cases into the chosen canonical PR.
