## Summary

Keep open. Current main partially improves the old config-source complaint, but the central macOS doctor problem remains: broad OpenClaw LaunchAgent content can still be reported as an extra gateway-like service, and doctor still prints static cleanup hints for the active profile gateway label whenever any extra service is found. The related open PRs are implementation candidates, not shipped fixes.

## Best Possible Solution

Keep this issue open and fix the doctor gateway-service scanner plus cleanup-hint generation. The likely maintainer path is to require an explicit gateway marker or parsed ProgramArguments that actually invoke the gateway before reporting macOS LaunchAgents, avoid treating OpenClaw ecosystem services as competing gateways, and generate cleanup hints only for detected stale/inactive extra gateway services. Review/rebase #39118 and #52515 or implement the same behavior directly, then add fixture-based macOS plist tests before closing.
