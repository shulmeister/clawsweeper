## Summary

Keep #72148 open. Current main still prepends the user-facing `Automatic session resume failed, so sending the status directly.` prefix on exec approval follow-up resume failure, and tests still assert that old behavior. The PR is a focused contributor fix for #72143 and is not superseded by #72268, which the provided context describes as a separate nested approval metadata parser fix.

## Best Possible Solution

Keep #72148 open for maintainer review and merge or adapt the focused behavior change: remove the internal session-resume-failure prefix from user-facing exec approval fallback messages, log resume failures internally with safe redaction/sanitization, and track retry/backoff separately unless maintainers decide it belongs in the same fix.
