## Summary

Current main already exposes Anthropic/Claude quota visibility through normalized provider-usage surfaces used by session_status, chat /status, CLI status --usage, and the gateway usage.status API. It does not pass through the raw anthropic-ratelimit-* header names verbatim, but the requested user-visible ability to see remaining provider quota/status inside OpenClaw is implemented.
