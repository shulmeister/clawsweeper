## Summary

Current `main` already routes `message` tool attachment aliases (`media`, `mediaUrl`, `path`, `filePath`, `fileUrl`) through the outbound media path, with regression coverage for local-file and remote-URL cases. The same outbound code is already present in the latest shipped tag, so this issue no longer matches current OpenClaw behavior.

## Evidence

- **Send path accepts media aliases:** `runMessageAction` now reads `media`, `mediaUrl`, `path`, `filePath`, and `fileUrl`, merges them into `mergedMediaUrls`, normalizes sandbox paths, and backfills `params.media` from the first resolved attachment before dispatch.
  - file: [src/infra/outbound/message-action-runner.ts:481](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/infra/outbound/message-action-runner.ts#L481)

- **Outbound media access includes the same aliases:** The outbound send service collects the same alias set when resolving per-agent media access, so local paths and remote URLs are preserved for plugin/core send handling instead of being dropped before dispatch.
