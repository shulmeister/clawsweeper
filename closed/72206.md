## Summary

Keep #72206 open. The report is recent, concrete, unprotected, and describes a live `claude-cli` fresh-session failure path that current `main` still routes through the Claude stdio live-session implementation. Existing code can still surface the observed `FailoverError` message/reason shape when a live Claude process exits without stderr, and the current tests cover adjacent behavior rather than this system-event/new-live-session regression.

## Best Possible Solution

Keep #72206 open. The best fix is likely in `src/agents/cli-runner/claude-live-session.ts` and the surrounding CLI runner path: reproduce the fresh system-event/heartbeat live-session start, preserve/emit useful startup stderr or exit diagnostics, and add a focused regression test proving a new live session succeeds without regressing warm session reuse or the #71793 buffer fix.
