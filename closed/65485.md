## Summary

Keep open. Current `main` still routes `SIGTERM` through the stop path, which skips the restart drain logic entirely. The comments narrowed the original report correctly: `agents.list` and `tools.*` no longer require a gateway restart, but direct `SIGTERM` shutdowns and plugin-scoped restart paths still leave this issue actionable.

## Risks / Open Questions

- The title/body currently mix two problems: the still-open SIGTERM drain gap, and older restart-trigger examples that are no longer all accurate on `main`.
- There is still no automated coverage for graceful draining on supervisor-driven SIGTERM (for example systemd or launchd restarts).
- Even after the SIGTERM gap is fixed, the existing deferral-timeout path can still restart with active work and may need separate product/behavior decisions.
