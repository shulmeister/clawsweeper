## Summary

Keep PR #69472 open. Current main still lacks tolerant suppression for reasoning-prefaced/doubled `NO_REPLY` payloads, and the linked issue #69470 was closed specifically because this PR is the active tracker, not because the bug is fixed. The PR is useful but should address the unresolved false-positive review concern before landing.

## Best Possible Solution

Keep #69472 as the canonical active code/test vehicle for #69470. Revise the PR so the new classifier suppresses the observed reasoning-prefaced/doubled-token silent reply without discarding substantive heading-formatted replies, keep the existing #19537 exact-token semantics, and land it with targeted `src/auto-reply/tokens.test.ts` coverage.
