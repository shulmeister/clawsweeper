## Summary

Close as `clawhub`. This PR adds a new optional, service-specific QNAIGC model provider plugin, and current main already has public provider-plugin and ClawHub/npm distribution surfaces that can support this outside the OpenClaw core repo. The PR discussion and file list are useful as a starting implementation, and later PR commits addressed the bot-reviewed labeler/auth-env snapshot gaps, but there is no maintainer/protected-label signal that this provider should be bundled in core.

## Best Possible Solution

Close the OpenClaw core PR and ask the contributor to publish QNAIGC as an external provider plugin on ClawHub or npm using the existing provider-plugin SDK. If publishing exposes a missing SDK/API seam, open a narrow follow-up issue or PR for that core extension point rather than bundling the service-specific provider by default.
