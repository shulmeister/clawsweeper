## Summary

Current `main` shows the Usage page as an aggregate per-session transcript summary, not a current-context snapshot. The reported expectation (`inputTokens` capped by the model context window) does not match the current implementation or docs, so this issue is not reproducible as a current-main bug.

## Evidence

- **Usage page backend returns aggregate per-session summaries:** `sessions.usage` loads each session with `loadSessionCostSummary(...)` and stores that aggregate under `usage` for the Usage page session rows.
  - file: [src/gateway/server-methods/usage.ts:635](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/gateway/server-methods/usage.ts#L635)
  - command: `nl -ba src/gateway/server-methods/usage.ts | sed -n '632,790p'`
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Session usage totals are cumulative across transcript entries:** `applyUsageTotals()` adds `input`, `output`, `cacheRead`, `cacheWrite`, and `totalTokens`, and `loadSessionCostSummary()` calls it while scanning the whole transcript file. That means a long session can legitimately exceed a 200k context window in total input tokens.
  - file: [src/infra/session-cost-usage.ts:185](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/infra/session-cost-usage.ts#L185)
  - command: `nl -ba src/infra/session-cost-usage.ts | sed -n '160,200p'; nl -ba src/infra/session-cost-usage.ts | sed -n '554,690p'`
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Docs separate current context from aggregate usage surfaces:** Docs say `/status` is the current-session context surface, while `sessions.usage` is a per-session usage summary surface. That matches the current code path and contradicts the issue's assumption that Usage-page `inputTokens` should be bounded by the live context window.
  - file: [docs/reference/api-usage-costs.md:19](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/docs/reference/api-usage-costs.md#L19)
  - command: `nl -ba docs/reference/api-usage-costs.md | sed -n '17,25p'; nl -ba docs/cli/status.md | sed -n '24,28p'; nl -ba docs/gateway/protocol.md | sed -n '282,288p'`
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)

## Risks / Open Questions

- If maintainers want the Usage page to show current window usage instead of cumulative session usage, that is a product/UX change rather than the bug described here.
- This does not rule out a historical 2026.3.13 double-counting defect in one reporter transcript, but the issue as filed does not provide transcript/log evidence and does not match current-main semantics.
