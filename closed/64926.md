## Summary

Current `main` and the shipped `v2026.4.23` release implement `node.list` as a cached known-state merge of pairing records plus in-memory connected sessions, not a LAN/mDNS discovery scan. The reported blocking discovery path is not present in the current RPC implementation.

## Evidence

- **`node.list` on current main is a local state merge:** The current RPC handler for `node.list` only awaits `listDevicePairing()`, `listNodePairing()`, and `context.nodeRegistry.listConnected()`, then returns merged known nodes. There is no discovery or network scan in this path.
