## Summary

Current `main` already avoids treating the reported LM Studio/Qwen Jinja prompt-rendering error as context overflow, so this PR is redundant.

## Evidence

- **Main overflow matcher does not special-case Jinja because it already only matches overflow-specific terms:** `isContextOverflowError` and `isLikelyContextOverflowError` on `main` only recognize explicit context/token/limit overflow phrases before returning true; there is no path here that would match `Error rendering prompt with jinja template: "No user query found in messages."`.

  - file: [src/agents/pi-embedded-runner/run.ts:1094](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/agents/pi-embedded-runner/run.ts#L1094)
