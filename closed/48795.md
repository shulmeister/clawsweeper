## Summary

Keep open. Current main has not absorbed this PR's main work: memory chunking remains the inline `chunkMarkdown()` implementation in the memory host SDK, there is no `src/memory/chunking` modular strategy surface, and the PR's hybrid-search and embedded-runner `Promise.all` latency changes are not present. Some related Unicode/CJK chunking safeguards already exist on main, but that is only a partial overlap, not a supersession of this PR or the related open heading-aware chunking issue #44395.

## Best Possible Solution

Keep the PR open for maintainer judgment, but ask for a rebase or split: preserve the already-shipped Unicode/CJK safeguards, move any chunking architecture into the current memory host SDK and memory-core indexing path, add focused tests for actual heading/structure-aware behavior if that remains the goal, and separate the two unrelated performance changes into their own narrow PRs. The broader product direction should stay coordinated with open issue #44395.
