## Summary

Close PR #69252 as duplicate/superseded. The requested recurring cron transient-retry behavior is real and still missing on current main, but it is already tracked by open issue #38069 and earlier open PR #46423. PR #69252 is a newer dirty duplicate that mixes the cron change with unrelated pairing/protocol/UI/Swift model changes and has unresolved review comments.

## Best Possible Solution

Close #69252 as the duplicate dirty PR, keep the recurring cron retry work tracked in #38069 and #46423, and move forward with a focused implementation that only touches the cron timer/restart-catchup logic, tests, config help, and public docs while addressing the unresolved edge cases around schedule-compute failures, retry exhaustion, non-transient backoff, and restart catch-up.
