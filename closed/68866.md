## Summary

Keep open. Current main does not implement the PR's core runtime snapshot invalidation: runtime auth snapshots are still stored as plain cloned stores with no auth-profiles.json/auth-state.json mtime fingerprint, and ensureAuthProfileStore still returns a runtime snapshot before reaching the disk-backed mtime cache. Related #53367/#53754/#53317 are closed, not merged canonical replacements on current main, and #68866 also has unresolved review concerns that need author/maintainer handling rather than cleanup closure.

## Best Possible Solution

Keep the PR open or supersede it with a corrected maintainer-owned implementation. The right fix belongs in the runtime auth snapshot layer: fingerprint auth-profiles.json and auth-state.json per agent, invalidate stale snapshots before reuse, preserve runtime-only resolved secrets, keep saveAuthProfileStore self-updates refreshing snapshots, and add regression coverage for auth-profiles.json, auth-state.json, save-refresh, and subagent fallback behavior.
