## Summary

Current main no longer has a `script` cron payload to document or time out. Cron payloads are limited to `systemEvent` and `agentTurn`; `timeoutSeconds` is accepted only for `agentTurn`, with `0` preserving no-timeout behavior. The same no-script schema is present in the latest release tag `v2026.4.24`, so the reported script-payload behavior is obsolete on current main/release.

## Evidence
