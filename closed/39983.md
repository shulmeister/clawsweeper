## Summary

Keep #39983 open. Current main still uses Ollama's deprecated `/api/embeddings` endpoint with `prompt` and parses `embedding`, while the current Ollama API documents `/api/embed` with `input` and `embeddings`. The bundled Ollama plugin exposes this path for `memorySearch.provider: "ollama"`, and the related fix PR #40205 is still open/unmerged in the provided GitHub context.

## Best Possible Solution

Keep the issue open and land a focused bundled Ollama plugin fix. Update `extensions/ollama/src/embedding-provider.ts` to call `/api/embed`, send `input` (batching when appropriate), parse `embeddings[0]` or batch `embeddings`, and preserve existing base URL normalization, auth/header handling, timeout behavior, and SSRF guard usage. Update `extensions/ollama/src/embedding-provider.test.ts` to assert the current Ollama API shape. PR #40205 can remain the candidate, but #39983 should close only after the fix lands on main.
