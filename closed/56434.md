## Summary

Current main already has a built-in scheduled memory-consolidation path: the `memory-core` Dreaming feature manages a recurring cron sweep, ranks durable candidates from daily memory/recall data, and promotes them into `MEMORY.md`. The issue’s core premise is also outdated because built-in memory indexing covers all `memory/*.md` files, not only recent daily notes.

## Risks / Open Questions

- The current implementation is opt-in (`dreaming.enabled: false` by default), so it does not exactly match a default-on interpretation of the proposal.
- I did not find evidence of an automatic `memory/archive/` rollover for old daily notes; if maintainers still want that narrower archival behavior, it would need a separate scoped issue.
