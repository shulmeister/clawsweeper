## Summary

Keep #41862 open. Current main still matches the reported bug: the export HTML template contains formatter-expanded MARKED_JS/HIGHLIGHT_JS/JS script blocks, while the generator still performs literal contiguous `{{...}}` replacements. A read-only replacement check leaves the bare placeholders behind, and the latest release tag has the same affected source state. The linked fix PR #41861 remains open/unmerged in the provided GitHub context, so there is no shipped fix to close against.

## Best Possible Solution

Keep #41862 open and use PR #41861 or a follow-up as the focused fix path: restore stable template placeholders or make `generateHtml()` validate/replace the current template shape robustly, add a regression test that reads the real template and asserts vendor/app JS are injected with no bare placeholders left, and ensure the build-copy path ships the corrected template.
