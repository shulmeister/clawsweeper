## Summary

Current `main` already handles `google/gemini-3.1-pro` by statically normalizing it to `gemini-3.1-pro-preview` before model-registry lookup, so the stale forward-compat template path this PR targeted is no longer the active implementation. The same behavior is present in the released `v2026.4.22` tag, and the PR’s original target file no longer exists on `main`.

## Evidence
