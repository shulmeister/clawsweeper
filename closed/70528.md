## Summary

Current OpenClaw already implements the requested safety behavior for config changes: agent-facing config writes are validated before commit, and invalid effective config is automatically restored from a last-known-good copy during startup/hot reload with user-visible recovery notices.

## Evidence

## Risks / Open Questions

- The implemented recovery is config-file scoped. If someone wants separate rollback for arbitrary external environment-variable mutation outside the config write surface, that would be a narrower follow-up request rather than this already-shipped config recovery behavior.
