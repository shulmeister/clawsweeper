## Summary

Keep #72402 open. Current main still implements the opposite behavior requested: all native Ollama `/think max` requests are mapped to wire `think: "high"`, with no `:cloud` exception or probe path. The related #71584 fix only made `max` accepted and mapped to Ollama's documented local high effort; it did not preserve native `max` for Ollama Cloud models.

## Best Possible Solution

Keep #72402 open and implement it in the Ollama plugin, not core: make `extensions/ollama/src/stream.ts` distinguish native Ollama Cloud model ids such as `:cloud` or add a conservative probe/fallback path, allow wire `think: "max"` only for that cloud-capable path, preserve local `max -> high`, add focused tests for both local and cloud models, and update the Ollama/thinking docs plus changelog if behavior changes.
