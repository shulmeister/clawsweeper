## Summary

Keep open. Current `main` still does not route ACP bridge diagnostics/logging away from stdout: `serveAcpGateway()` starts with `loadConfig()` and later binds ACP NDJSON to `process.stdout`, while the `openclaw acp` CLI action calls that server without setting `routeLogsToStderr()`. The PR remains a focused fix for the linked live bug #49060, though the Greptile comment's standalone-entrypoint caveat should be handled before landing or in a superseding fix.

## Best Possible Solution

Keep this PR open or replace it with a superseding ACP-specific fix that routes logs to stderr before `loadConfig()`/gateway bootstrap, accounts for the standalone server entrypoint, and adds a regression test around ACP stdout cleanliness or the early `routeLogsToStderr()` call.
