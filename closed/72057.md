## Summary

Keep #72057 open. Current main still does not automatically route Google Gemini LLM requests through ambient HTTP_PROXY/HTTPS_PROXY. The Google LLM transport uses buildGuardedModelFetch, which calls the SSRF guard without trusted env-proxy mode unless a model-provider request.proxy override is configured. Existing config/docs provide a workaround, but not the reported Docker-env-only behavior.

## Best Possible Solution

Keep the issue open and fix the model-provider LLM transport. A conservative fix would make buildGuardedModelFetch or its resolved request policy use trusted env-proxy mode for operator-controlled provider requests when shouldUseEnvHttpProxyForUrl(url) is true, while preserving DNS pinning for NO_PROXY, ALL_PROXY-only cases, and untrusted web fetches. Add regression coverage for google-generative-ai requests to generativelanguage.googleapis.com with HTTPS_PROXY set and NO_PROXY bypass coverage; document models.providers.google.request.proxy.mode = "env-proxy" as a workaround if needed.
