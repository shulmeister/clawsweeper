## Summary

Keep open. Current main still renders the Control UI New session action as a plain plus icon inside the chat input toolbar, alongside the paperclip attachment control, and it immediately sends `/new` without a hide/reposition/confirmation option. The related title-search items are adjacent Control UI button requests, not a canonical duplicate for this specific UX issue.

## Best Possible Solution

Keep the issue open for a small Control UI UX fix. The likely implementation belongs in `ui/src/ui/chat/run-controls.ts` and `ui/src/ui/views/chat.ts`: move New session out of the composer toolbar into the session controls/header, or replace the plus with a less ambiguous session/document-plus icon. If maintainers prefer a preference instead, add a typed `gateway.controlUi` config field with schema/help/docs and focused tests.
