## Summary

Current main already contains the LanceDB retry fix, in a more complete form than this PR, and that implementation is present in release v2026.4.22. The repo now resets both the module-load promise and the MemoryDB initialization promise after failure, with a regression test covering retry-after-failure. This PR is also stale against current main because its second commit removes the gateway-token clearing behavior that main still preserves and tests.
