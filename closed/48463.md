## Summary

Current `main` already implements the reported behavior: configured channel plugins only use `setupEntry` during startup when the manifest explicitly opts in with `startup.deferConfiguredChannelFullLoadUntilAfterListen: true`, and there is a direct regression test for the exact reported case.

## Evidence

- **Loader gate requires explicit startup opt-in:** `src/plugins/loader.ts:1394` returns `true` for setup-runtime on configured channel plugins only when both `preferSetupRuntimeForChannelPlugins` and `startupDeferConfiguredChannelFullLoadUntilAfterListen === true` are set; otherwise it falls back to checking whether the channel is unconfigured.
  - file: [src/plugins/loader.ts:1394](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/plugins/loader.ts#L1394)
