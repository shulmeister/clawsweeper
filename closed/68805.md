## Summary

Reporter later confirmed outbound WeChat cron delivery started working again without local changes, and current main shows the WeChat runtime is owned by the external `@tencent-weixin/openclaw-weixin` plugin rather than OpenClaw core. The report is therefore obsolete against current behavior and not supported as an open core regression.

## Evidence

- **Reporter confirmed recovery:** In the issue timeline, the reporter stated that the daily weather cron job delivered successfully on 2026-04-20 06:01 GMT+8 and that no local changes were made after filing, which directly contradicts the original 'never delivers' report.
- **WeChat runtime is external to core:** The current WeChat docs say the WeChat code does not live in the OpenClaw core repo and that Tencent iLink API calls, context tokens, and account monitoring are owned by the external plugin.
  - file: [docs/channels/wechat.md:27](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/docs/channels/wechat.md#L27)
  - command: `nl -ba docs/channels/wechat.md | sed -n '1,120p'`
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **Core only resolves plugin channel ids:** Current main keeps channel-plugin resolution lightweight and registry-based; it normalizes registered external channel ids without importing plugin runtime code, which matches the docs boundary and leaves the reported token/send path outside core.
  - file: [src/channels/registry.ts:68](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/src/channels/registry.ts#L68)

## Risks / Open Questions
