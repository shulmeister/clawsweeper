## Summary

Keep #39690 open. Current main still overwrites explicitly configured model `input` metadata with implicit/catalog metadata for matching model IDs, so `input: ["text", "image"]` can still be lost before Gateway decides whether to preserve inline image blocks.

## Best Possible Solution

Keep the issue open. The best fix is a narrow `mergeProviderModels` precedence change so an explicitly present `models.providers.*.models[].input` wins over implicit/catalog metadata, matching the existing `reasoning` override behavior. Add regression coverage for explicit image input surviving a catalog match, then verify Gateway preserves native inline image blocks for that configured model while keeping the text-only offload fallback intact.
