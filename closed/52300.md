## Summary

Current `main` already gives `/restart` user-visible feedback: it captures chat/thread routing before restart, returns explicit failure replies if restart setup fails, and after boot sends a routed restart notice plus a continuation back into the originating conversation. The shipped `v2026.4.22` changelog also explicitly says chat-initiated Gateway restarts now acknowledge successful boot automatically.
