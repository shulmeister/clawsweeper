## Summary

Current `main` does not match the reported Control UI -> session persistence path: the picker builds provider-qualified model refs, posts them unchanged to `sessions.patch`, and the gateway stores explicit overrides while clearing stale runtime model fields.

## Evidence

- **Control UI builds qualified model values:** `buildChatModelOptionFromLookup()` uses `buildQualifiedChatModelValue(entry.id, provider)`, so picker values are provider-qualified rather than bare slugs.
  - file: [ui/src/ui/chat-model-ref.ts:266](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/ui/src/ui/chat-model-ref.ts#L266)
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **OpenRouter nested-id case is covered in UI tests:** The test fixture `{ provider: "openrouter", id: "google/gemma-4-26b-a4b-it" }` is expected to produce picker value `openrouter/google/gemma-4-26b-a4b-it`.
  - file: [ui/src/ui/chat-model-ref.test.ts:38](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/ui/src/ui/chat-model-ref.test.ts#L38)
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **Chat header sends selected model unchanged:** `switchChatModel()` forwards `model: nextModel || null` to `sessions.patch`; there is no prefix-stripping step in the Control UI RPC path.
  - file: [ui/src/ui/chat/session-controls.ts:260](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/ui/src/ui/chat/session-controls.ts#L260)
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **Gateway persists model changes as resolved overrides:** `applySessionsPatchToStore()` resolves the requested model ref and passes the resolved provider/model into `applyModelOverrideToSessionEntry()` instead of writing raw request text straight into runtime model fields.
  - file: [src/gateway/sessions-patch.ts:388](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/gateway/sessions-patch.ts#L388)
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **Override write path clears stale runtime model fields:** `applyModelOverrideToSessionEntry()` deletes conflicting `entry.model` and `entry.modelProvider` fields when an explicit override is selected, preventing stale runtime identity from persisting alongside the override.
  - file: [src/sessions/model-overrides.ts:56](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/sessions/model-overrides.ts#L56)
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **Integration coverage for slash-containing provider/model ids:** Gateway integration test patches `nvidia/moonshotai/kimi-k2.5` through `sessions.patch` and asserts the session keeps `providerOverride: "nvidia"`, `modelOverride: "moonshotai/kimi-k2.5"`, with no runtime `model`/`modelProvider` fields left behind.
  - file: [src/gateway/server.sessions.gateway-server-sessions-a.test.ts:1604](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/gateway/server.sessions.gateway-server-sessions-a.test.ts#L1604)
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)

## Risks / Open Questions

- `sessions.patch` still accepts explicit bare provider/model refs from non-UI callers, so any future gateway-side hardening for wrapper providers like OpenRouter would need a separate issue or follow-up change.
