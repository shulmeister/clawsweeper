## Summary

Keep PR #53704 open. The underlying UI/backend schema mismatch still exists on current main and latest release: the Agents UI still writes `agents.defaultId`, while the config schema only accepts `agents.defaults` and `agents.list`, and runtime default resolution uses `agents.list[].default` or the first list entry. A later related PR (#59200) was closed unmerged, so it does not supersede this PR.

## Best Possible Solution

Keep this PR or a successor open and land a tested UI/config-controller fix that persists the default agent through `agents.list[].default`, strips any derived `agents.defaultId` from editable form state before save, preserves editable recovery data where possible, and adds regression tests for Set Default and save serialization.
