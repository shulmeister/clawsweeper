## Summary

Current main still has the node systemd unit-name mismatch this PR fixes. Node install/uninstall injects `OPENCLAW_SYSTEMD_UNIT=openclaw-node`, and the unit-file path resolver honors that override, but install activation and uninstall still derive the `systemctl` target from the gateway-only resolver. PR #68293 is focused and adds missing install/uninstall regression tests, so it remains useful open work despite overlap with older PR #39580.

## Best Possible Solution

Keep the item open or consolidate it with #39580 by landing the narrow resolver fix plus the regression tests from #68293. The implementation should make `activateSystemdService` and `uninstallSystemdService` derive the systemctl unit through `resolveSystemdServiceName(env)`, preserving the gateway fallback while making node install/uninstall target `openclaw-node.service`.
