## Summary

Current main still does not expose a Control UI auto-expand preference, but the same remaining feature is already tracked by open non-draft PR #54838. That PR is the canonical implementation/review thread for adding a persisted webchat expand-by-default toggle, so this newer issue can close as superseded rather than remain as a second tracker.

## Best Possible Solution

Close this issue as superseded by PR #54838, keep #54838 open as the canonical implementation/review thread, and land that PR or a successor after addressing review comments. If maintainers want the exact `ui.chat.expandToolsByDefault` config contract, extend the canonical implementation to add the config type, Zod schema, generated schema/docs metadata, and Control UI bootstrap wiring.
