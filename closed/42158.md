## Summary

Keep #42158 open. The linked fix attempts (#40055, #42182, #42520) were closed unmerged, and current main still has the safeguard outer failure path returning `{ cancel: true }` when the compaction summarization path throws before producing any summary. Some lower-level LLM failures now degrade through `summarizeWithFallback`, but the issue's requested recovery behavior is still missing for first-failure throws in the safeguard handler.

## Best Possible Solution

Keep the issue open and implement a narrow safeguard recovery path in `src/agents/pi-hooks/compaction-safeguard.ts`: when the initial built-in summarization path throws a non-user-abort error before any summary is available, build a bounded emergency summary from prior summary, preserved recent turns, split-turn context, file ops, tool failures, workspace rules, and a sanitized error note, then return a real `compaction` result. Add tests for first-call summarization failure, abort/signal passthrough, repeated fallback size bounds, and split-turn/recent-turn preservation.
