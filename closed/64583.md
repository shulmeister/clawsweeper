## Summary

Current `main` contains explicit loopback/backend pairing exemptions plus regression coverage for `sessions_send` loopback delivery, so the reported post-restart `pairing required` failure is implemented on `main` even though it does not appear to be in the latest tagged release yet.

## Evidence
