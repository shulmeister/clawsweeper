## Summary

Keep open. Current main still resolves `openclaw skills install` and `openclaw skills update` through `resolveDefaultAgentId(config)` only, so running from a configured sub-agent workspace cannot change the target workspace. The helper needed for the reported fix, `resolveAgentIdByWorkspacePath`, exists and is tested, and related PRs #56243/#56294 described the right direction, but the current checkout does not contain that change or its proposed regression coverage.

## Best Possible Solution

Revive the narrow fix from the related PR discussion: in `src/cli/skills-cli.ts`, share a resolver that loads config, tries `resolveAgentIdByWorkspacePath(config, process.cwd())`, falls back to `resolveDefaultAgentId(config)`, and uses that workspace for install/update plus list/info/check status reads. Add focused `src/cli/skills-cli.commands.test.ts` coverage for install, update, list/info/check, and unrelated-cwd fallback.
