## Summary

Keep open. Current `main` does not implement the PR's retry hardening for the sandbox image builds, and the cold-start path for `scripts/sandbox-common-setup.sh` still goes through a base Dockerfile with no apt retry logic. The review discussion also surfaced a valid gap: the submitted PR only adds retries to `upgrade`/`install`, not the preceding `apt-get update` step or the base-image build path.

## Evidence

- **Current main still has no retries in sandbox-common APT layer:** `Dockerfile.sandbox-common` on `main` still runs plain `apt-get update`, `apt-get upgrade`, and `apt-get install` with no `Acquire::Retries` handling, so the requested behavior is not already implemented on `main`.
