## Summary

Current main already solves #39107's central request: scripts and skills can reuse OpenClaw-configured model credentials through `openclaw infer model run`, plugin/Node code can use the exported simple-completion runtime, and Python or other clients can use the Gateway OpenAI-compatible chat completions endpoint. The earlier PR #39131 proposed different command/API names, but it closed unmerged and the maintained surfaces now cover the same workflow.

## Best Possible Solution

Keep the shipped maintained surfaces rather than reviving the unmerged #39131 API shape. Use `openclaw infer model run --prompt ... --json` for shell/script workflows, `openclaw/plugin-sdk/simple-completion-runtime` for plugin/Node runtime code, and Gateway `/v1/chat/completions` for Python or OpenAI-compatible clients.
