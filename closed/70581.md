## Summary

Current `main` already documents and supports the durable gateway-side secret path this report asks for. Provider `SecretRef` env auth resolves from the gateway process environment, and the supported service-safe sources are `~/.openclaw/.env`, config `env`, and optional `env.shellEnv`; an interactive-shell-only export is not a durable LaunchAgent input. The file-backed `provider: "default"` error also matches the documented SecretRef contract, which requires a configured file provider/default for `source: "file"` refs.
