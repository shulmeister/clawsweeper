## Summary

Current `main` mitigates packaged plugin runtime-dependency installs by staging them externally and by forcing an OpenClaw-owned npm cache, but it still performs runtime installs during gateway startup and still falls back to `~/.openclaw/plugin-runtime-deps` unless `OPENCLAW_PLUGIN_STAGE_DIR` or `STATE_DIRECTORY` is configured. That means the hardened-systemd class reported here is not fully fixed by default, so this should stay open.

## Risks / Open Questions

- This review did not run a live systemd reproduction; the decision is based on current code, docs, and test coverage.
- Deployments that already set `STATE_DIRECTORY` or `OPENCLAW_PLUGIN_STAGE_DIR` are likely covered by current main, but the default fallback path still targets home and leaves the original hardened-systemd default case unresolved.
