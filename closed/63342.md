## Summary

Current `main` does not have a `/agent` code path that opens an additional TUI WebSocket or resets gateway event sequencing per session, so the reported stale-session/`seq gap` mechanism does not match upstream OpenClaw anymore.

## Evidence

- **TUI creates one gateway client once:** `openclaw tui` constructs a single `GatewayChatClient` once during startup when not in local mode.
  - file: [src/tui/tui.ts:497](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/tui/tui.ts#L497)
  - command: `nl -ba src/tui/tui.ts | sed -n '488,508p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Gateway client starts once:** The TUI starts that client once after wiring handlers; there is no separate `/agent` startup path here.
  - file: [src/tui/tui.ts:1107](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/tui/tui.ts#L1107)
  - command: `nl -ba src/tui/tui.ts | sed -n '1096,1145p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **`/agent` only switches session state:** The `/agent` handler updates `currentAgentId` and delegates to `setSession("")`; it does not reconnect or create another gateway client.
  - file: [src/tui/tui-command-handlers.ts:85](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/tui/tui-command-handlers.ts#L85)
  - command: `nl -ba src/tui/tui-command-handlers.ts | sed -n '70,100p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Session switch reloads history but does not touch socket lifecycle:** `setSession()` updates TUI state, clears local run state, and reloads history. There is no WebSocket cleanup/reconnect path on agent/session switch.
  - file: [src/tui/tui-session-actions.ts:368](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/tui/tui-session-actions.ts#L368)
  - command: `nl -ba src/tui/tui-session-actions.ts | sed -n '368,384p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Gateway event sequence is per receiving client connection:** Gateway broadcast assigns `seq` from a per-client counter when frames are sent, so sequence numbers are connection-scoped rather than session-scoped.
  - file: [src/gateway/server-broadcast.ts:122](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/gateway/server-broadcast.ts#L122)
  - command: `nl -ba src/gateway/server-broadcast.ts | sed -n '122,166p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Client gap detection follows connection-wide seq:** `GatewayClient` only reports a gap when an incoming event `seq` jumps ahead of the last seen connection-level value.
  - file: [src/gateway/client.ts:752](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/gateway/client.ts#L752)
  - command: `nl -ba src/gateway/client.ts | sed -n '752,757p'`
