## Summary

Keep open. Current `main` still does not implement the gateway change this PR proposes: `resolveSessionModelRef` delegates to `resolvePersistedSelectedModelRef`, and that selected-model path still falls back to `defaultProvider` for bare runtime/override model strings instead of checking `inferUniqueProviderFromConfiguredModels` first. The existing gateway tests also still lock in that old fallback behavior, so the PR's core bugfix is not already implemented on `main`.
