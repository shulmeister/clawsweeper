## Summary

Keep PR #50885 open. Current main still does not implement the PR's Windows cmd-wrapper UTF-8 code-page prefix or `runExec` encoding option, the Windows exec tests do not cover the proposed regression cases, and the FAQ still documents garbled Windows Chinese `system.run`/`exec` output as an active workaround issue. ## Best Possible Solution

Keep #50885 open for maintainer review or explicit consolidation with the canonical Windows exec encoding work. The best implementation path is to land a tested fix in `src/process/exec.ts` that preserves non-wrapper argv behavior, handles both `runExec` and `runCommandWithTimeout`, covers UTF-8 aliasing and Windows CJK output regression tests, and coordinates with broader code-page decoder PRs rather than auto-closing useful work.
