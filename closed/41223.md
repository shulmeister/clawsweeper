## Summary

Current `main` already implements the underlying gateway self-restart recovery with a detached launchd handoff, plus tests for the in-service restart path and handoff failure path, so this PR is superseded.

## Evidence

- **Current restart path uses detached handoff when running inside the LaunchAgent:** `restartLaunchAgent()` now detects when the caller is the managed LaunchAgent and schedules a detached restart helper instead of running `launchctl kickstart -k` inline. It returns `{ outcome: "scheduled" }` and avoids direct `launchctl` calls in that path.
