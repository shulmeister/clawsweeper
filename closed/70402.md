## Summary

Current `main` already cleans up dream narrative session artifacts after runs: it always attempts session deletion, prunes stale `dreaming-narrative-*` entries from `sessions.json`, archives orphan transcript files, and has a regression test covering the exact stale-registry/orphan-file case described in #70402.

## Evidence
