## Summary

Keep #41869 open. Current main still preserves `lastError` for `status: "error"` even when delivery resolves to `delivered`, and the same path still resolves failure alerts without checking delivery success. The PR has evolved after review feedback: its latest head no longer clears `lastError`; it preserves diagnostics and retry counting while suppressing failure alerts for delivered error runs and adding a regression test. That is still plausible product work and is not implemented on main.

## Best Possible Solution

Keep the PR open, but ask the contributor to align the title/body with the latest behavior and decide the intended semantics explicitly: preserve `lastError` as execution diagnostics, keep `lastDeliveryError` delivery-specific, and add/keep a regression test for `status: "error"` with `delivered: true` proving failure alerts are suppressed while retry counting still works.
