## Summary

Current `main` no longer runs the image tool against `minimax-portal/MiniMax-M2.7`; it falls back to `MiniMax-VL-01` for MiniMax image understanding and sends those requests through the dedicated MiniMax VLM endpoint, which covers the empty-chat-content failure path described here.

## Evidence

- **MiniMax chat catalog is text-only:** The bundled MiniMax provider catalog builds `MiniMax-M2.7`/`MiniMax-M2.7-highspeed` as text-only entries via `buildMinimaxTextModel(... input: ["text"])`, so image-tool auto-resolution does not keep using the active M2.7 chat model.
  - file: [extensions/minimax/provider-catalog.ts:49](https://github.com/openclaw/openclaw/blob/4f4288e3b5be2e55abfce5213f0bcf182a252f5e/extensions/minimax/provider-catalog.ts#L49)
  - command: `nl -ba extensions/minimax/provider-catalog.ts | sed -n '45,90p'`
  - sha: [4f4288e3b5be](https://github.com/openclaw/openclaw/commit/4f4288e3b5be2e55abfce5213f0bcf182a252f5e)
- **MiniMax image default is `MiniMax-VL-01`:** Bundled MiniMax media-understanding metadata declares `defaultModels.image` as `MiniMax-VL-01` for both `minimax` and `minimax-portal`.
  - file: [extensions/minimax/openclaw.plugin.json:75](https://github.com/openclaw/openclaw/blob/4f4288e3b5be2e55abfce5213f0bcf182a252f5e/extensions/minimax/openclaw.plugin.json#L75)
  - command: `nl -ba extensions/minimax/openclaw.plugin.json | sed -n '70,95p'`
  - sha: [4f4288e3b5be](https://github.com/openclaw/openclaw/commit/4f4288e3b5be2e55abfce5213f0bcf182a252f5e)
- **Image-tool auto-resolution uses the provider image default:** `resolveKeyEntry()` calls `resolveAutoImageModelId()` for image capability on the active provider, and that helper falls back from an explicit unsupported model to the provider default image model.
  - file: [src/media-understanding/runner.ts:468](https://github.com/openclaw/openclaw/blob/4f4288e3b5be2e55abfce5213f0bcf182a252f5e/src/media-understanding/runner.ts#L468)

  - sha: [4f4288e3b5be](https://github.com/openclaw/openclaw/commit/4f4288e3b5be2e55abfce5213f0bcf182a252f5e)
- **Regression test covers active `MiniMax-M2.7` -> `MiniMax-VL-01` fallback:** There is an explicit test asserting that when the active model is `minimax-portal/MiniMax-M2.7`, auto image resolution chooses `MiniMax-VL-01` instead.
  - file: [src/media-understanding/runner.vision-skip.test.ts:215](https://github.com/openclaw/openclaw/blob/4f4288e3b5be2e55abfce5213f0bcf182a252f5e/src/media-understanding/runner.vision-skip.test.ts#L215)
