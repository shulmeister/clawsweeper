## Summary

Current `main` already implements fs-policy-aware outbound local-media sending, including non-workspace host paths when `tools.fs.workspaceOnly=false`, and the changelog shows this shipped in the 2026.3.24 release.

## Risks / Open Questions

- No live Docker reproduction was run in this read-only review, so this conclusion rests on current main code, tests, docs, and changelog evidence rather than an end-to-end container verification.
- Non-workspace local sends are still intentionally blocked when `tools.fs.workspaceOnly=true` or equivalent policy denies host reads; a user misconfiguration there would look similar to the old bug.
