## Summary

Current `main` already guards the Gmail watcher against duplicate restart loops on bind conflicts, and the same guard is present in the shipped `v2026.4.23` release commit.

## Evidence

- **Bind-conflict detection:** `spawnGogServe` watches `stderr` for `address already in use` / `EADDRINUSE` and marks the serve process as an address-in-use case instead of treating it like a generic crash.
