## Summary

Current `main` now hardens the bundled `claude-cli` path by clearing Claude/Anthropic provider-routing env and explicitly removing `CLAUDE_CODE_PROVIDER_MANAGED_BY_HOST`, with tests covering the exact host-managed/extra-usage failure shape. That matches this report well enough to treat it as fixed on `main`, but I could not tie the change to a released tag yet.

## Evidence

- **Current main removes the host-managed marker before spawning Claude CLI:** `executePreparedCliRun` now deletes `CLAUDE_CODE_PROVIDER_MANAGED_BY_HOST` with an inline note that this marker routes Claude CLI runs into Anthropic's separate host-managed usage tier instead of normal CLI subscription behavior.
  - file: [src/agents/cli-runner/execute.ts:302](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/agents/cli-runner/execute.ts#L302)

  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Bundled Anthropic backend clears inherited provider-routing/auth env:** `CLAUDE_CLI_CLEAR_ENV` now strips Anthropic/Claude auth, routing, config-root, and telemetry env before the bundled backend runs, preventing inherited shell overrides from steering OpenClaw-managed Claude CLI requests away from the local CLI login state.
  - file: [extensions/anthropic/cli-shared.ts:15](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/extensions/anthropic/cli-shared.ts#L15)

  - file: [src/agents/cli-backends.test.ts:576](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/agents/cli-backends.test.ts#L576)
