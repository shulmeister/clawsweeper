## Summary

Keep #54927 open. Current main still uses `SKILL.md` and `*/SKILL.md` chokidar watch targets while the repo depends on chokidar 5, so the PR's central bug is not implemented on main. The PR is a focused, useful fix, but it still needs the latest review concern addressed around overlapping watch roots before it is merge-ready.

## Best Possible Solution

Keep #54927 open and ask for a small update: choose the deepest matching watch root in `resolveRelativeWatchPath()`, keep the root-watch plus `depth: 1` approach, preserve the SKILL.md-only event guard, and keep/add regression tests for root-level and nested `SKILL.md` add/change/unlink while ignoring unrelated files. After that, this narrow PR is a plausible fix for the current skills watcher glob regression and can reference the open duplicate report #50468 as well as the older #27404 context.
