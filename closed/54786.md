## Summary

Keep open. Current main still does not add recursive, glob, or scan-depth support to `skills.load.extraDirs`. It supports direct skill roots and a narrow `dir/skills/<skill>/SKILL.md` unwrap, but a single extraDirs entry pointing at `plugins/` still misses `plugins/<name>/skills/<name>/SKILL.md`.

## Best Possible Solution

Keep the issue open. A maintainer-quality fix should add an explicit, bounded `skills.load` discovery mode, update the config type/schema/help/docs, extend watcher coverage safely, and add tests for `plugins/<name>/skills/<name>/SKILL.md`. If maintainers prefer the plugin-bundle path instead, document `plugins.load.paths` as the supported workaround before closing this request.
