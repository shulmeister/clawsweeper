## Summary

Close as ClawHub/plugin scope. The issue proposes a useful macOS-only integration with the linked kc CLI, but current core already has the generic SecretRef machinery needed to call external secret managers through env/file/exec providers. A kc-specific wrapper, setup helper, or import/export convenience should live as an external ClawHub/npm plugin rather than adding another core SecretRef source.

## Best Possible Solution

Keep OpenClaw core on the generic env/file/exec SecretRef backend. Publish a kc-focused ClawHub/npm plugin or wrapper that configures an exec provider, optionally adds plugin-owned CLI setup/import helpers, and links to the kc project. If that work later exposes a concrete missing generic plugin API, open a narrower API issue.
