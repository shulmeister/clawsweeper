## Summary

Current main already gives heartbeat runs the conversation context the PR was trying to recover: heartbeats run on the resolved session key as normal agent turns, the embedded runner loads/sanitizes/limits existing session history before model context assembly, and docs describe heartbeat as full main-session context by default. Main also filters no-op heartbeat prompt/HEARTBEAT_OK pairs before context assembly, addressing the feedback-loop concern without manual prompt prepending. The PR discussion also identified unresolved risks in the proposed manual transcript lookup, including prompt-size, account-mismatch, and stale-session selection issues.

## Best Possible Solution

Close this PR rather than merge the parallel prompt-prepending implementation. Keep the shipped main-session heartbeat behavior and heartbeat-pair filtering. If maintainers later want a low-cost hybrid limited-context mode for isolated heartbeats, design it separately with account-scoped newest-session resolution, a real aggregate cap, docs, and focused tests.
