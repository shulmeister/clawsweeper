## Summary

Current main covers the user-facing problem: Active Memory is available as a bundled OpenClaw plugin, not as a separate `@openclaw/active-memory` npm/ClawHub package. The linked PR #63286 context explicitly described a bundled `active-memory` plugin, current source has the bundled manifest and runtime entry, public docs show enabling it through `plugins.entries.active-memory`, and release notes list it under 2026.4.12.

## Best Possible Solution

Keep Active Memory as a bundled OpenClaw plugin and direct users to the documented `plugins.entries.active-memory` configuration path. If maintainers later want a standalone npm package or ClawHub listing, track that as a separate publishing/product task rather than reopening this core availability bug.
