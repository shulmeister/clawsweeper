## Summary

Keep open. Current `main` still contains the behavior PR #63987 is trying to relax: the bundled qwen plugin strips `qwen3.6-plus` from configured Coding Plan providers and runtime suppression runs before inline/configured model lookup. The PR is not implemented on `main`, and the discussion shows the author addressed the key review concern by adding an explicit-config suppression bypass in the second commit. There are overlapping Qwen Coding Plan PRs/issues, especially broader PR #66367, but no canonical merged fix is present in the checked-out `main`.

## Best Possible Solution

Keep this PR open unless maintainers explicitly choose a broader canonical PR such as #66367. The best fix should at minimum stop mutating explicit user-configured `qwen3.6-plus` Coding Plan models and ensure runtime suppression does not block an explicit opt-in; if maintainers decide Coding Plan support is now generally valid, also update the built-in catalog tests and public Qwen docs to advertise that behavior.
