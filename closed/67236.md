## Summary

Do not auto-close #67236. QQ Bot is a bundled OpenClaw plugin, and current main only adds a narrower bot self-echo guard; the reported replay path for the same inbound QQ message ID still appears to enqueue and process each delivered event.

## Best Possible Solution

Keep the issue open and implement QQ Bot inbound replay dedupe before Agent dispatch, preferably at gateway ingress or queue admission using account + surface/peer + messageId, aligned with the documented channel replay-dedupe contract. Add focused C2C/group/guild tests proving repeated gateway deliveries with the same QQ message ID trigger one Agent run, while keeping the existing bot self-echo behavior and quote handling intact.
