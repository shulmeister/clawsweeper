## Summary

Keep open. Current main still has gateway transcript-event plumbing, but the bundled TUI does not subscribe to or handle the `session.message` / `sessions.changed` events needed for Telegram-originated updates to appear live. #43918 fixed a related outbound route-inheritance bug, and #51691 overlaps as a broader open tracker, but the observable Telegram -> TUI real-time update problem is not implemented on current main.

## Best Possible Solution

Implement bundled TUI live transcript sync: subscribe to the active canonical session with `sessions.messages.subscribe` on connect and session changes, handle `session.message` by refreshing or merging the active transcript with the same active-run deferral semantics used by Control UI, update footer/session metadata from `sessions.changed`, and add TUI regression tests. Coordinate with #51691 as the broader tracker; if maintainers want only one issue, choose a canonical issue explicitly before closing either as duplicate.
