## Summary

Keep PR #53588 open. Current main still strips `<relevant-memories>` only through assistant-visible text paths; the WebChat/Control UI non-assistant extraction path still returns user text after envelope/inbound-metadata cleanup without removing memory blocks. The PR discussion correctly identifies a real narrow UI gap, though the patch needs rebase/test refinement against current main.

## Best Possible Solution

Keep the item open and land one canonical implementation for this bug. The likely maintainer path is to add a user-message regression test in `ui/src/ui/chat/message-extract.test.ts`, expose or introduce a neutral shared helper for stripping relevant-memory scaffolding without unwanted trimming, and apply it in `ui/src/ui/chat/message-extract.ts` after existing internal-context/envelope/inbound-metadata cleanup. Then dedupe the overlapping open PRs #51288, #53588, and #59697 around the implementation maintainers choose.
