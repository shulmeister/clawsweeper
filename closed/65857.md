## Summary

Keep open. Current main still does not propagate `--dangerously-force-unsafe-install` through dependency denylist scans: package and bundle source scans return dependency blocks unconditionally, the post-install dependency-tree scan cannot receive the override, and the installer does not pass the override into that post-install scan. Current tests also explicitly assert the opposite of this PR's requested behavior. The PR is focused and supported by the provided review context, but it needs maintainer/product confirmation because current docs describe the flag as a dangerous-code scanner override, not a dependency-denylist bypass.

## Best Possible Solution

Keep the PR open. The best maintainer path is to decide whether `--dangerously-force-unsafe-install` should intentionally bypass dependency denylist blocks. If yes, implement the propagation across package source, bundle source, post-install dependency-tree scans, and installer `afterInstall`, then update regression tests and docs to make the expanded semantics explicit. If no, close manually with product rationale and consider tightening docs/tests so contributors do not read the flag as a dependency-denylist override.
