## Summary

Current main already has the requested conservative skill-whitelist path: agents can inherit `agents.defaults.skills`, override with `agents.list[].skills`, or use an explicit empty list for no skills. That filter is applied to the skill snapshot/prompt and runtime surfaces, with prompt-size limits as a second guard against context pressure. The implementation is present in the latest release tag v2026.4.24 as well as current main.

## Best Possible Solution

Close this as implemented and keep the shipped agent-scoped allowlist model. If future work wants to optimize pre-scan I/O for very large skill directories, that should be tracked as a narrower performance issue rather than keeping this broader context-overload request open.
