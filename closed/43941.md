## Summary

Close #43941 as duplicate/superseded, not as fixed. Current main still does not remove Docker sandbox workspace directories during prune, but the same remaining bug is already tracked by #43797 and the narrower open PR #43835. #43941 also carries an unrelated zh-CN docs commit and its sandbox patch still has path/safety concerns called out in review, so it is not the best canonical PR to keep open.

## Best Possible Solution

Close #43941 as duplicate/superseded. Keep the remaining work on #43835/#43797, with a safe implementation that only removes the owning workspace after confirmed runtime removal, uses the stored scope key/workspace root correctly, guards shared workspace reuse, and adds focused prune regression coverage.
