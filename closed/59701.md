## Summary

Issue #59701 can stay closed as implemented on current main. The reported 2026.4.1 strict reset behavior no longer matches current exec-approvals defaults, normalization, runtime fallback, docs, or release notes.

## Evidence

- **Current approvals defaults are no-prompt host defaults:** Current main sets DEFAULT_SECURITY to "full", DEFAULT_ASK to "off", and DEFAULT_EXEC_APPROVAL_ASK_FALLBACK to "full", so a missing or freshly initialized approvals file no longer resolves to strict deny/on-miss defaults.

  - file: [src/infra/exec-approvals.ts:3](https://github.com/openclaw/openclaw/blob/7bbd47349ec37f01a8f2b1dc219bf950615f361b/src/infra/exec-approvals.ts#L3)
