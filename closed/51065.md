## Summary

Keep #51065 open. Current main still creates user-visible `âš ď¸Ź ... failed` reply payloads for mutating tool failures such as `edit`/`write`, and Telegram dispatch sends those final error payloads through the normal reply path. The linked PR #51091 is open and narrower than the issue's default leak complaint, while #51213 was closed unmerged, so this is not implemented or safely superseded on main.

## Best Possible Solution

Keep the issue open until maintainers deliberately choose the product contract for failed mutating tool warnings. The likely code path is `src/agents/pi-embedded-runner/run/payloads.ts`: either suppress or normalize mutating-tool warning payloads when the assistant already produced a user-facing explanation, or add a clearly documented opt-in setting for hiding mutating warnings. Update the existing payload tests and Telegram dispatch coverage to lock the chosen behavior. If maintainers want a single tracker, manually consolidate #51065 with the older open #39631 after preserving the signal-leak and streaming-off reproduction details.
