## Summary

Keep open. Current main still does not implement the requested full Feishu interactive-card parsing for quoted/fetched messages, and the PR contains useful Feishu-plugin work even though it still needs reconciliation with unresolved parser/API concerns and newer related work (#41609/#56795).

## Best Possible Solution

Keep this PR open for maintainer comparison or ask the author/maintainer to consolidate it with the canonical Feishu interactive-card work in #41609/#56795. The best product path is a shared Feishu plugin parser used by inbound events, quoted/root message fetches, and thread history, with a verified raw-card Feishu API parameter, backward-compatible `RawBody` handling, and focused tests for headers, buttons/actions, tables, code blocks, i18n/body variants, and safe recursion limits.
