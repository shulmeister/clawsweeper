## Summary

Current `main` already covers this Windows Scheduled Task failure mode. `restartScheduledTask()` now routes `/Run` through `runScheduledTaskOrThrow()`, which detects the `schtasks` "accepted but never actually started" state (`not-yet-run`) and relaunches through the installed fallback launcher instead of waiting for a fake 60s timeout. Regression tests explicitly cover both install-time and restart-time `/Run` no-op cases, and the changelog says this shipped in `v2026.4.22`.

## Evidence
