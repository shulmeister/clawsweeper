## Summary

Close PR #44114 as superseded by the canonical bug report #44077. Current main still has the sandbox write-mode bug, so this is not implemented-on-main, but this PR is not a safe or scoped landing vehicle: the remaining work is already tracked by #44077, while the PR bundles a narrow sandbox fix with unrelated Feishu changes and 1,565 generated/runtime files.

## Best Possible Solution

Close this dirty, oversized PR and keep #44077 as the canonical tracker. The best fix is a new scoped PR that changes only the sandbox mutation helper and tests: preserve permissions for existing regular destination files, avoid permission widening for symlink paths, and add regression coverage showing a 0644 file does not become 0600 after a sandbox write.

## Evidence
