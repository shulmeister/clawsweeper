## Summary

Current `main` already supports manual per-session custom titles via the existing session `label` field, persists them through `sessions.patch`, and prefers those labels in UI session selectors instead of raw internal keys. The same behavior is present in the latest shipped release `v2026.4.23`.

## Evidence

- **Control UI exposes editable session labels:** The Sessions view renders a per-session editable `Label` input and calls `onPatch(row.key, { label: value })`, which is the manual rename/custom title flow requested in the issue.
  - file: [ui/src/ui/views/sessions.ts:515](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/ui/src/ui/views/sessions.ts#L515)

  - file: [ui/src/ui/controllers/sessions.ts:391](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/ui/src/ui/controllers/sessions.ts#L391)
