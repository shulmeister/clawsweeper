## Summary

Current main already covers the requested screen-visibility workflow: macOS nodes expose screen snapshots and recordings, the agent-facing nodes tool has a first-class screen_record action, the CLI can record node screens to MEDIA files, and the bundled Peekaboo skill covers macOS screenshots, AI analysis, and live capture.

## Best Possible Solution

Keep the shipped permission-gated screen snapshot, screen recording, and Peekaboo live-capture paths. If maintainers want a separate always-on streaming UX later, track that as a narrower product/security design issue rather than keeping this broad request open.
