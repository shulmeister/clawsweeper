## Summary

Current main and v2026.4.24 address both reported failure modes: Feishu streaming-card HTTP 400 startup failures now fall back to normal card delivery with backoff, and packaged/global update paths now inventory and verify packaged dist files so missing hashed runtime chunks are detected instead of silently shipping or updating into an incomplete install.

## Best Possible Solution

Close this issue as implemented. Keep the shipped Feishu fallback/backoff behavior and packaged-dist inventory/update verification; any future Feishu API-specific 400 response-body diagnostics or live rate-limit tuning should be tracked as a new, narrower issue with fresh logs if it still reproduces on current releases.
