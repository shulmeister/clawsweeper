## Summary

Current main already implements the requested behavior through the bundled `active-memory` plugin: it runs a blocking memory-recall pass before the main reply and prepends hidden memory context for eligible interactive sessions. That makes this PR's separate `agents.defaults.memorySearch.autoPrefetch` core path redundant and based on a config surface the docs now explicitly avoid for this feature.

## Evidence

## Risks / Open Questions

- The shipped solution is not config-compatible with this PR: operators must use `plugins.entries.active-memory`, not `agents.defaults.memorySearch.autoPrefetch`.
- `active-memory` is intentionally limited to eligible interactive persistent sessions, so it is not a blanket prefetch hook for headless/background/internal flows.
