## Summary

Current main already implements the PR's intended fix: embedded Pi resource-loader reloads are followed by reapplication of OpenClaw compaction settings in both production paths, with regression coverage and release evidence. The equivalent fix landed through merged PR #67146, so #65672 is now obsolete even though its root-cause analysis was valid.

## Best Possible Solution

Close #65672 as implemented on main by the merged #67146 fix. Keep the shipped per-call post-reload reapplication; only reconsider a shared reload helper if future code adds more embedded Pi resource-loader reload call sites.
