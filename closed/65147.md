## Summary

Keep open. Current main still builds npm global update commands without `--force` on both the primary install and omit-optional fallback paths, and `openclaw update` routes global npm installs through those helpers. PR #65147 is a focused fix for the linked container EEXIST problem, though the bot review correctly flags that the fallback path should also receive `--force` before this is considered complete.

## Best Possible Solution

Keep #65147 open and ask for the small follow-up needed before merge: apply `--force` consistently to every npm global update path, including the omit-optional fallback, keep the `--` package-spec terminator and env-spec sanitization if accepted, and update `update-global` and `update-runner` tests to lock the final argv contract.
