## Summary

Keep #43671 open. The old `src/agents/ollama-stream.ts` target has moved into the Ollama plugin, but current `main` still does not implement the PR's core behavior for native Ollama `/api/chat`: there is no stream-level retry loop, an early stream end still emits an error, and a `done:true` response with no accumulated content/tool calls is accepted as an empty `stop` message rather than retried. The PR needs a rebase/narrow rewrite against `extensions/ollama/src/stream.ts`, with tests that cover the bot-review concerns around aborts and intentional empty completions.

## Best Possible Solution

Keep the PR open but request a rebase against `extensions/ollama/src/stream.ts`. The best implementation is a narrow Ollama-native retry path for the documented empty `done:true`/no-tool-call parse-failure and premature no-final-response cases, with regression tests for successful retry, retry exhaustion, abort/no-retry, whitespace/intentional-empty behavior, and no retry after any visible content or tool-call emission.
