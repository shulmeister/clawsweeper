## Summary

Current `main` already covers the user-visible ACP delivery gap from this draft PR, and the in-repo ACPX parser files the PR edited no longer exist because ACPX now delegates to the external `acpx/runtime` package.

## Evidence

- **ACPX implementation moved out of tree:** The bundled ACPX extension now imports `AcpxRuntime` from `acpx/runtime`, so the old local `extensions/acpx/src/runtime-internals/events.ts` parser targeted by this PR is no longer part of current `main`.
  - file: [extensions/acpx/src/runtime.ts:1](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/extensions/acpx/src/runtime.ts#L1)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Current ACPX package pin:** OpenClaw currently ships ACPX via the external `acpx` package at version `0.5.3`.
  - file: [extensions/acpx/package.json:6](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/extensions/acpx/package.json#L6)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)

  - file: [src/acp/translator.ts:946](https://github.com/openclaw/openclaw/blob/bff212822cf73c7a37fa7a77d9a725c2d8895b1e/src/acp/translator.ts#L946)
