## Summary

Keep #68329 open. Current main still parses CLI usage and persists CLI transcripts, but it does not run a CLI compaction lifecycle or clear/reseed external CLI resume state. The remaining work is actively tracked by open maintainer-labeled PR #71916, which says it fixes #68329 and supersedes #68388, but that work is not present in current main at d251932fcfd60e205e80bf7e9abdb4af63582e6f.

## Best Possible Solution

Keep #68329 open and let #71916, or a maintainer-approved successor, close it on merge. The best implementation direction is the focused CLI compaction lifecycle from #71916: compact the persisted OpenClaw CLI transcript after an over-budget CLI turn, clear stale external CLI resume bindings, and reseed fresh CLI sessions from compacted OpenClaw history without writing provider-private transcript files.
