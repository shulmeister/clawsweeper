## Summary

Current main already covers the reported Telegram `/help` exposure: operators can disable Telegram native command registration/menu entries and use `commands.allowFrom.telegram` as the command-only allowlist, while `/help` itself now refuses unauthorized command senders instead of returning help text. The same behavior is present in the latest release tag `v2026.4.24`.

## Evidence

- **Per-provider command allowlist is documented:** The slash-command docs describe `commands.allowFrom` as a per-provider command authorization allowlist; when configured it is the only authorization source for commands/directives, with provider-specific keys such as `telegram` overriding the global default.
  - file: [docs/tools/slash-commands.md:73](https://github.com/openclaw/openclaw/blob/8e12c24d17218dc50023aaf5543df1b7c36520aa/docs/tools/slash-commands.md#L73)
