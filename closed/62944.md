## Summary

Keep open. Current main still passes a fixed 30,000 ms timeout from the agent image tool into all image-description provider calls, while the documented media-understanding config has a timeoutSeconds contract and the separate media runner already resolves it. The provided GitHub context also shows open, unmerged PR #63483 with closing syntax for this issue, so the issue should stay open until that PR is reviewed, landed, or rejected.

## Best Possible Solution

Keep this issue open and review PR #63483 as the current implementation candidate. The likely fix belongs in src/agents/tools/image-tool.ts: resolve the configured image timeout once from the existing media-understanding config contract, pass it to every describeImage and describeImages call, preserve the intended default behavior, and add focused regression coverage for single-image, multi-image batch, and serial fallback paths.
