## Summary

Keep PR #33914 open. Current main still does not implement the central plugin hook identity context requested by the PR: `PluginHookAgentContext` lacks `sourceProvider`, sender identity, group, and parent-session fields, and the main agent/attempt/compaction hook contexts do not pass them. The PR is large and needs maintainer review or a focused rebase, but the API gap is real and supported by plugin-author discussion.

## Best Possible Solution

Keep this PR open for maintainer review or a narrowed rebase. The best path is an additive plugin SDK/context update exposing `sourceProvider`, `senderId`, `senderName`, `senderIsOwner`, `groupId`, and `spawnedBy` on `PluginHookAgentContext`; deriving normalized origin identity consistently; preserving null semantics; threading the fields through early run, attempt, queued/memory, compaction, reset, and final hook contexts; and completing outbound `sessionKey` correlation for typed message hooks. Coordinate remaining compaction/reset invariant work with open issue #69269 rather than treating this PR as obsolete.
