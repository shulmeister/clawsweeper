## Summary

Current `main` already implements the requested static/dynamic system-prompt split for Anthropic-family prompt caching, exposes cache controls via `cacheRetention`, and documents/tests the behavior.

## Evidence

- **System prompt is explicitly split at a cache boundary:** `buildProjectContextSection(..., dynamic: true)` marks frequently changing project context as below the cache boundary, and `buildAgentSystemPrompt` inserts `SYSTEM_PROMPT_CACHE_BOUNDARY` before dynamic context so stable prompt content stays above it.
