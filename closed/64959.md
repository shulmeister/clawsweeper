## Summary

Current `main` now cleans stale transcript lockfiles on gateway startup, marks the matching `agent:<id>:main` sessions as restart-aborted, and then resumes or fails those interrupted main sessions instead of leaving them stuck as `status: "running"` and recontaminating later runs.

## Evidence

- **Main-session default routing still uses the agent main key:** `resolveExplicitAgentSessionKey()` still resolves a missing explicit agent session to that agent's main session key, so the implemented fix is the new reconciliation/recovery path rather than a routing-policy removal.

  - file: [src/gateway/server-startup-post-attach.ts:160](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/src/gateway/server-startup-post-attach.ts#L160)
