## Summary

Keep #63820 open. The related discussion is credible: commenters confirmed Discord/Slack channel sessions affected, and PR #63801 identified the likely root cause, but that PR is closed/unmerged and current main still appears to advance `updatedAt` for system-event session turns. Current code forces heartbeat/cron/exec turns to reuse the existing session, then persists a new `updatedAt`, which can make the next real channel message see the session as fresh instead of daily/idle-stale.
