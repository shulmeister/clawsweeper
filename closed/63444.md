## Summary

Current `main` already routes standalone memory embedding lookups through manifest-backed capability loading, and the bundled Amazon Bedrock plugin declares and registers the `bedrock` memory embedding adapter on that path. The reported `Unknown memory embedding provider: bedrock` root cause is therefore implemented away on `main`, although I did not run a live Bedrock CLI repro in this read-only pass because that would require AWS credentials/config.

## Evidence

- **Bedrock plugin declares standalone memory-embedding contract:** The bundled Amazon Bedrock plugin manifest now declares `contracts.memoryEmbeddingProviders: ["bedrock"]`, which is the contract used to resolve embedding adapters before full runtime activation.
