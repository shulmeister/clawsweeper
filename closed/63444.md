## Summary

Current `main` already handles `memorySearch.provider: "bedrock"` on the `openclaw memory status` path: the doctor/status code maps `bedrock` to `amazon-bedrock`, the bundled Amazon Bedrock plugin registers a `bedrock` memory embedding adapter, and shipped tests assert the Bedrock path is probed via `amazon-bedrock`. The same implementation is present in the `v2026.4.23` release commit.

## Evidence
