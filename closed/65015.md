## Summary

Current `main` already exposes plugin hooks that can hard-block `message` tool calls, cancel outbound sends, and observe final delivery, so this routing-policy request fits OpenClaw's documented plugin/ClawHub extension model rather than new core behavior.

## Evidence

## Risks / Open Questions

- There is no linked existing routing-policy plugin in this issue thread, so closing assumes maintainers prefer the documented plugin path over adding a first-party bundled solution.
- A production-ready enterprise policy package would still need to be implemented, published, and maintained separately from OpenClaw core.
