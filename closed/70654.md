## Summary

Current `main` now covers the reported failure path with two protections: model-switch replay sanitization for OpenAI Responses history, and a merged GitHub Copilot follow-up that preserves encrypted reasoning item IDs during replay.

## Evidence

- **Model-switch replay sanitization on current main:** `sanitizeSessionHistory` detects the stored model snapshot, computes `modelChanged`, and passes `dropReplayableReasoning: modelChanged` into the OpenAI Responses reasoning downgrade path so stale `rs_*` replay items are stripped when switching routes/models.
