## Summary

Close as ClawHub/plugin-scope. #56823 is a service-specific Venice E2EE provider enhancement, not a core regression or missing core API. The PR and prior #52933/#53505 discussion show the implementation can stay provider-local via the existing provider `wrapStreamFn` seam with no SDK/core changes, while OpenClaw's current docs and VISION route optional provider capabilities, discovery, provenance, and security review through external plugins and ClawHub.

## Best Possible Solution

Close this OpenClaw core PR and move the Venice E2EE work to a separately maintained provider plugin published through ClawHub/npm, using the existing provider SDK and `wrapStreamFn` seams. Reopen core work only if the external implementation identifies a concrete missing, generic plugin API or maintainers explicitly decide to bundle and review Venice E2EE in core.
