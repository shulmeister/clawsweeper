## Summary

Keep open. Current main supports only top-level/account-level boolean `typingIndicator` for Feishu. It does not implement the requested per-device or per-platform desktop/mobile behavior, and the parsed inbound Feishu message model does not carry device/platform metadata to drive that decision.

## Best Possible Solution

Keep #56469 open as a Feishu plugin enhancement. The best path is to first verify whether Feishu/Lark events or a follow-up API can reliably identify client platform; if so, extend the Feishu config schema, manifest, docs, parser/context, dispatcher, and tests for a backwards-compatible `typingIndicator` union such as `boolean | { desktop?: boolean; mobile?: boolean; default?: boolean }`. If platform cannot be determined, document the account-level workaround and have a maintainer decide whether that is sufficient.

## Risks / Open Questions

- Feishu/Lark may not expose reliable client device/platform metadata in message events; implementation should verify the upstream event contract before promising desktop-vs-mobile behavior.
- Current account-level `typingIndicator` overrides may be a usable workaround for some deployments, but they do not satisfy automatic per-device behavior from one Feishu account.
