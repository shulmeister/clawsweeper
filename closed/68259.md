## Summary

Keep open. Current main has the underlying Gateway `sessions.send` RPC and a generic `openclaw gateway call` escape hatch, but it does not provide the requested first-class `openclaw sessions send` CLI wrapper. The existing `openclaw sessions` surface is list/cleanup only, and the docs likewise document only listing and cleanup. A prior ClawSweeper related item (#38352) closed a broader injection request as satisfied by `openclaw gateway call sessions.send`; this newer issue is narrower and explicitly asks for the missing command shape that external chain automation assumes.

## Best Possible Solution

Keep the issue open and implement a small core CLI wrapper under `openclaw sessions send`. It should call `sessions.resolve` when `--label` is used, call `sessions.send` with `key`, `message`, and `timeoutMs`, return non-zero on RPC/delivery failure, support stdin or `--message`, and update `docs/cli/sessions.md` plus command registration tests in `src/cli/program/register.status-health-sessions.test.ts`.
