## Summary

Current main already handles numeric Telegram cron announce destinations via `delivery.to` and a `last`/single-channel fallback; the only unsupported shape is putting the numeric chat id into `delivery.channel`, which current code now rejects during cron validation instead of failing later with `Unsupported channel`. The same implementation is present in release `v2026.4.23`, while the report was against `2026.4.15`.

## Evidence

- **CLI keeps channel and target separate:** `cron add` defaults `--channel` to `last`, so omitting `--channel` does not turn a numeric `--to` value into the channel field.
  - file: [src/cli/cron-cli/register.cron-add.ts:103](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/cli/cron-cli/register.cron-add.ts#L103)
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **CLI maps `--to` into `delivery.to`:** The cron CLI builds `delivery.to` from `opts.to` and `delivery.channel` from `opts.channel`; they are separate fields on current main.
  - file: [src/cli/cron-cli/register.cron-add.ts:230](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/cli/cron-cli/register.cron-add.ts#L230)
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **Missing channel falls back before docking explicit target:** When cron delivery has no explicit/session channel, `resolveDeliveryTarget()` calls channel selection, then reruns session target resolution with `explicitTo`, so an explicit numeric destination can still resolve against the selected channel.
  - file: [src/cron/isolated-agent/delivery-target.ts:106](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/cron/isolated-agent/delivery-target.ts#L106)
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **Single configured channel is auto-selected:** `resolveMessageChannelSelection()` returns the sole configured channel when exactly one deliverable channel is configured.
  - file: [src/infra/outbound/channel-selection.ts:197](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/infra/outbound/channel-selection.ts#L197)
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **Telegram accepts numeric chat ids:** `normalizeTelegramChatId()` accepts numeric Telegram ids (`/^-?\d+$/`), so numeric chat ids are valid Telegram targets on current main.
  - file: [extensions/telegram/src/targets.ts:32](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/extensions/telegram/src/targets.ts#L32)
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **Invalid `delivery.channel` is rejected up front:** Cron add/update validates `delivery.channel` against configured announce channel ids; non-channel values are rejected before runtime delivery.
  - file: [src/gateway/server-methods/cron.ts:40](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/gateway/server-methods/cron.ts#L40)
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Regression test covers target-id-in-channel misuse:** There is a gateway validation test asserting that a target id mistakenly supplied as `delivery.channel` is rejected before `cron.add` runs.
  - file: [src/gateway/server-methods/cron.validation.test.ts:166](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/gateway/server-methods/cron.validation.test.ts#L166)
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)

  - file: [docs/automation/cron-jobs.md:143](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/docs/automation/cron-jobs.md#L143)

## Risks / Open Questions
