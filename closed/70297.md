## Summary

Keep open. Current main still allows QMD memory in channel sessions by default in the root runtime source, and the colocated regression test still asserts that behavior. This PR is a plausible focused hardening/change request, but it is not already implemented on main and it also touches a shipped default that the changelog says was intentionally broadened, so it needs maintainer product/security judgment rather than automated cleanup.

## Best Possible Solution

Keep #70297 open for maintainer review. The best path is to decide whether QMD should be direct-only by default despite the shipped direct+channel changelog entry; if yes, land a coordinated fix that updates the root runtime source and regression test, aligns `packages/memory-host-sdk` or removes the mirror divergence, and updates QMD docs/schema help/changelog so operators get one clear default and an explicit channel opt-in path.
