## Summary

Keep #54353 open. The PR is stale against current main and likely needs to be rewritten, but the underlying Discord allowlist bug is still present in the active message preflight path, and I did not find a current regression test or a canonical open replacement item that tracks the remaining fix.

## Best Possible Solution

Keep the PR open or convert it into a maintainer-authored fix: rebase onto current main, patch the active Discord allowlist decision so configured guild channel allowlists require an explicit channel match, add a regression test for `groupPolicy: "allowlist"` plus a non-empty `guilds.<id>.channels` map where an unconfigured channel returns `null`, and audit the adjacent Discord listener/component policy callers for the same route-match semantics.
