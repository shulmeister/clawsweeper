## Summary

Current `main` no longer uses the isolated `harness-auth` `CODEX_HOME` path for Codex app-server startup. The Codex plugin now leaves start options unchanged and authenticates the app-server by sending the selected auth profile through `account/login/start`, with regression tests asserting that no `harness-auth` directory is created. This matches the issue’s reported root cause closely enough to close as already implemented on `main`, although it does not appear to be in the latest tagged release yet.
