## Summary

Current `main` already deduplicates normal inbound Discord messages before they enter the debounce/worker path, and the same implementation is present in release `v2026.4.23`.

## Evidence

- **Inbound handler claims replay key before queueing:** `createDiscordMessageHandler` now builds a replay key from account/channel/message id and returns early when `claimDiscordInboundReplay(...)` says the message was already seen, before `debouncer.enqueue(...)` runs.
