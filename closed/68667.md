## Summary

Keep PR #68667 open. Current main still has the bug the PR describes: hook completion and error events are enqueued to the default main session even when the hook request was normalized to a non-default target agent. The PR is a focused gateway security fix with relevant regression tests, and the only bot review concern in the provided discussion was addressed by the author.

## Best Possible Solution

Keep the PR open for maintainer/security review. The best path is to merge or adapt the PR’s narrow dispatcher fix: load one config snapshot in `src/gateway/server/hooks.ts`, route hook completion/error events through `resolveAgentMainSessionKey({ cfg, agentId: value.agentId })` when `agentId` is present, preserve the default/wake behavior when no target agent exists, and keep the added success/error plus integration assertions that drain the target agent’s main session.
