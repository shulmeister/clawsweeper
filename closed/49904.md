## Summary

Close as implemented on main. Current OpenClaw no longer sends bundled source `.ts` plugin/channel entries through Node native strip-only loading, so the Discord parameter-property constructors cited in the report are not on the failing path anymore. The loader now prefers native loading only for built `.js/.mjs/.cjs/.json` modules and keeps source TypeScript on Jiti's transform path, with regression tests covering that behavior. The same implementation is present in release `v2026.4.22`.
