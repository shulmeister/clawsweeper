## Summary

The reported default dark-theme contrast failure is no longer present on current `main`, and the shipped `v2026.4.22` release already carries compliant dark-theme muted text tokens.

## Evidence

- **Current default dark theme tokens:** `ui/src/styles/base.css` defines the default dark theme background as `#0e1015` and muted text as `#838387`, not the reported `#636370`.
  - file: [ui/src/styles/base.css:3](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/ui/src/styles/base.css#L3)

  - file: [ui/src/styles/base.css:27](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/ui/src/styles/base.css#L27)
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Default theme resolution:** `ui/index.html` resolves the default Claw dark-mode theme to `data-theme="dark"`, so the `:root` tokens in `ui/src/styles/base.css` are the active default dark theme for Control UI.
