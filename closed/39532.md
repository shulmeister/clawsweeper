## Summary

Current `main` already prevents the replay-poisoning scenario this PR targets. Instead of only collapsing consecutive assistant-error entries, the live transport path now drops failed assistant turns (`stopReason: "error"` or `"aborted"`) before Anthropic/OpenAI replay, and the embedded runner also retries empty error turns before they accumulate. That makes this PR's narrower `stripConsecutiveAssistantErrors()` approach obsolete on current `main`.
