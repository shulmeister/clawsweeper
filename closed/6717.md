## Summary

Current `main` already implements agent-driven model switching well enough: agents have a first-class `session_status(model=...)` tool, the system prompt explicitly advertises that override and tells agents to use tools instead of equivalent slash/CLI commands, and the runtime/test coverage handles live-switch retries for active runs.

## Evidence

- **`session_status` accepts and applies model overrides:** The tool schema includes an optional `model` field, and the execute path resolves the requested model, applies it to the session entry, persists it, and marks `liveModelSwitchPending` for runtime handoff.
  - file: [src/agents/tools/session-status-tool.ts:473](https://github.com/openclaw/openclaw/blob/1daa552d5f429f9c57daa3ed050c34239b8a22dc/src/agents/tools/session-status-tool.ts#L473)
  - sha: [1daa552d5f42](https://github.com/openclaw/openclaw/commit/1daa552d5f429f9c57daa3ed050c34239b8a22dc)
- **Agent prompt advertises model override on `session_status`:** The injected tool summary tells the agent that `session_status` can answer model-state questions and set an optional per-session model override.
  - file: [src/agents/system-prompt.ts:507](https://github.com/openclaw/openclaw/blob/1daa552d5f429f9c57daa3ed050c34239b8a22dc/src/agents/system-prompt.ts#L507)
  - sha: [1daa552d5f42](https://github.com/openclaw/openclaw/commit/1daa552d5f429f9c57daa3ed050c34239b8a22dc)

  - sha: [1daa552d5f42](https://github.com/openclaw/openclaw/commit/1daa552d5f429f9c57daa3ed050c34239b8a22dc)

## Risks / Open Questions

- This appears to be on `main` after the latest tagged release `v2026.4.22`, so I could not tie it to a shipped release yet.
- The implemented path is session-scoped live switching via `session_status`, not a separate `[model:...]` inline syntax; if maintainers still want message-scoped syntax, that would be follow-up scope rather than evidence the current request is still unimplemented.
