## Summary

No conservative auto-close reason applies. Current main does not implement `host="cloud"` or a typed cloud sandbox provider seam, and this is not safely closable as ClawHub-only because the requested transparent routing for the built-in `exec` and `browser` surfaces requires a core/plugin API extension. The PR is also not merge-ready as-is: the provided review discussion flags multiple P1/P2 correctness and security issues, and current main has evolved around `host=auto` and browser profiles since the PR was opened.

## Best Possible Solution

Keep this open for maintainer direction, but request a redesign or narrower follow-up before merge: define a current-main-compatible cloud execution plugin API, route it through existing exec approval/env/background/process semantics, split browser cloud support into the existing remote-CDP/browser-profile model or a separate PR, and leave concrete cloud provider implementations for ClawHub/npm plugins.
