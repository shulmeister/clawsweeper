## Summary

Keep open. Current main still does not implement the requested stale-allowlist CLI error path: if the primary command is present in `plugins.allow` but no plugin/alias registers that command, the missing-command helper falls through to `null`, so `runCli` does not emit the clearer stale-entry message requested in #64729. Existing validation, doctor, and security-audit paths do recognize stale/phantom `plugins.allow` entries, but they do not replace this CLI-specific UX fix. Related PR #59575 may provide a cleanup command, but it does not supersede the missing-command message issue.

## Best Possible Solution

Keep #64729 open and add a focused CLI fix in `src/cli/run-main.ts`: distinguish `plugins.allow` entries that do not correspond to a registered plugin command or manifest alias, return an actionable stale-entry message, and add resolver/startup tests for `plugins.allow: ["crons"]` with no matching provider. Coordinate wording with the existing config-validation and doctor stale-plugin warnings, and keep #59575 as a separate cleanup-command discussion.
