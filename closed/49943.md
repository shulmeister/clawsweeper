## Summary

Current main no longer matches the reported routed-zombie lifecycle. PTY exec completion is driven by the PTY adapter wait/onExit path before the exec session is marked exited, finished sessions are removed from the running registry and lose writable stdin, and new exec calls spawn fresh supervisor runs. The related discussion and PR attempts around #50114/#50124 helped narrow the cascade concern, but the current implementation prevents the reported routing path and keeps non-PTY exec runs in their own process group on the affected Linux path.

## Best Possible Solution

Close #49943 as implemented on main. Keep the current supervisor/process-registry lifecycle, where PTY sessions are finalized only after adapter wait and finished sessions cannot receive process-tool writes. If a future report shows orphaned PTY children or service-managed process-group behavior, track that as a narrower runtime bug with fresh logs and reproduction steps rather than keeping this routed-zombie issue open.
