## Summary

Current main already solves the PR's stated Cloudflare Markdown for Agents problem by sending a Markdown-preferring Accept header and handling text/markdown responses. The exact generic defaultHeaders knob is not present, and this PR has unresolved schema/cache/header-precedence review defects, so the best maintainer cleanup is to close it rather than merge it.

## Best Possible Solution

Close this PR and keep the shipped built-in Markdown-preferring web_fetch behavior from #15376. If maintainers still want a broader arbitrary-header feature, it should come as a fresh design/PR that updates strict config validation, docs, deterministic cache keys, managed-header precedence, and security guidance.
