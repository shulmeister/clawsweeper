## Summary

The report no longer matches current main: MiniMax openai-compatible transports now use strict replay/tool-result pairing logic, current tests lock that behavior, and a maintainer retest in the issue thread on OpenClaw 2026.4.9 did not reproduce the 2013 error.

## Evidence

- **MiniMax owns strict replay policy for OpenAI-compatible transport:** The MiniMax provider registers shared replay hooks, and the MiniMax hook test expects `openai-completions` transports to sanitize tool-call IDs in strict mode and apply the assistant-first ordering fix.
  - file: [extensions/minimax/provider-registration.ts:41](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/minimax/provider-registration.ts#L41)

- **OpenAI-compatible replay helper enforces strict tool-call pairing:** The shared OpenAI-compatible replay helper enables `sanitizeToolCallIds: true`, `toolCallIdMode: "strict"`, and `applyAssistantFirstOrderingFix: true` for `openai-completions` transports.
  - file: [src/plugins/provider-replay-helpers.ts:12](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/plugins/provider-replay-helpers.ts#L12)

- **Replay history rewrites assistant and tool-result IDs together:** Session replay sanitization runs `sanitizeToolCallIdsForCloudCodeAssist` over the replayed messages after OpenAI-specific normalization, so assistant tool-call IDs and matching tool-result IDs are rewritten consistently from the same transcript.
  - file: [src/agents/pi-embedded-runner/replay-history.ts:500](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/pi-embedded-runner/replay-history.ts#L500)

- **Strict transcript repair explicitly covers MiniMax-style failures:** Transcript repair moves matching tool results directly after the assistant tool-call turn and drops orphaned tool results; the inline comment explicitly notes that strict providers including MiniMax reject bad tool-result placement.
  - file: [src/agents/session-transcript-repair.ts:420](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/session-transcript-repair.ts#L420)

  - file: [extensions/minimax/index.test.ts:122](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/minimax/index.test.ts#L122)
