## Summary

Keep #61315 open. Current main still has the behavior this PR is trying to fix: channel and agent rankings are cost-first, Top Channels/Agents display cost as the primary value, and `sessions.usage` aggregates only the limited returned session set. The PR is a plausible focused bug fix, but the provided review comments identify remaining scope/limit/fallback issues that should be addressed before merge rather than closing the PR as obsolete.

## Best Possible Solution

Keep the PR open or ask the author to rebase and finish it. The best path is a scoped usage-dashboard fix that adds shared cost/token ranking semantics with regression tests, makes `sessions.usage` aggregates and UI insight denominators use a consistent full-match scope when no filters are active, preserves returned-session `limit` exactly, and keeps a backward-compatible limit warning fallback. Treat related #52521 as a separate channel-attribution issue, not a duplicate of this PR.
