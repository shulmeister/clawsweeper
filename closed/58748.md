## Summary

Current `main` already strips leaked internal runtime context from user-facing replies and rewrites raw `Exec denied (..., approval-timeout)` payloads to safe copy before delivery. The same implementation is present in the shipped `v2026.4.23` tree, so this issue appears fixed on `main` and released.

## Evidence
