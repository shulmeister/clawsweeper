## Summary

Keep open. Current main still starts the model-pricing refresh during normal gateway startup and still performs OpenRouter and LiteLLM catalog fetches with a 60s timeout. It now defers the bootstrap work to a microtask, but the requested disable surface is still absent and the open linked PR #68651 remains the implementation candidate for `models.pricing.enabled`.

## Best Possible Solution

Keep this issue open and review #68651 as the canonical implementation candidate. A complete fix should add a validated `models.pricing.enabled` config key with default behavior preserved, skip all model-pricing bootstrap/refresh network work when disabled, update config schema/help/docs/generated baselines, and include focused tests proving disabled pricing does not fetch OpenRouter or LiteLLM catalogs. The current microtask deferral is useful but does not fully satisfy the requested disable/control behavior.
