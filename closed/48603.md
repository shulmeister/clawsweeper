## Summary

Current main already addresses the narrow user problem in #48603: provider-specific rate-limit text is no longer always collapsed to the generic “API rate limit reached” copy, and the surfaced assistant/lifecycle paths use that formatter. The broader operator-diagnostics request remains tracked separately by #55451.
