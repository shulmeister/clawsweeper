## Summary

Keep open. Current `main` still has the Feishu chat routing gap described in PR #52755: `feishu_chat` always binds to the first enabled account and only checks that first account's `tools.chat` flag, while the other Feishu tools already use the shared per-execution account-routing helpers. This is not implemented on `main` yet.

## Evidence

- **Chat tool still hardcodes first enabled account:** `registerFeishuChatTools` selects `accounts[0]`, reads `resolveToolsConfig(firstAccount.config.tools)`, and builds its client with `createFeishuClient(firstAccount)`. The execute path never consults session context or request params for account routing.
