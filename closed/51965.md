## Summary

Keep open. Current main still creates non-Azure custom provider models with `input: ["text"]` and `reasoning: false`, and the regression test explicitly asserts that behavior. The linked bug #51869 is still real enough that this PR is not obsolete, but the PR needs a current-main rebase and review fixes before it is mergeable.

## Best Possible Solution

Keep the item open, but require a current-main patch in `src/commands/onboard-custom-config.ts`: either prompt for vision support or use a conservative, well-tested model-id heuristic for common OpenAI, Claude, Gemini, Qwen-VL, and similar vision models; preserve existing user-customized model entries; align reasoning defaults for non-Azure o-series/GPT reasoning models if intended; and add positive and negative regression tests covering the review-thread edge cases.
