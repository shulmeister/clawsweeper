## Summary

Current main, and released `v2026.4.23`, already address the concrete problem behind this request by cleaning up `memory-core` `dreaming-narrative-*` sessions after each run and scrubbing stale leftovers, so the Control UI session-list clutter described from `2026.4.15` is already fixed well enough without adding a generic `excludePrefixes` config.

## Evidence

- **Docs and config contract still only expose visibility modes:** The public tools config documents `tools.sessions.visibility` as only `self | tree | agent | all`; there is no `excludePrefixes` option in the current config surface.
  - file: [docs/gateway/config-tools.md:262](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/docs/gateway/config-tools.md#L262)
  - command: `nl -ba docs/gateway/config-tools.md | sed -n '260,286p'`
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **Dreaming narrative runs now delete their temporary session:** `generateAndAppendDreamNarrative()` builds a `dreaming-narrative-*` session key, then always calls `await params.subagent.deleteSession({ sessionKey })` in `finally`, so those background sessions are explicitly cleaned up after use.
  - file: [extensions/memory-core/src/dreaming-narrative.ts:857](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/extensions/memory-core/src/dreaming-narrative.ts#L857)
