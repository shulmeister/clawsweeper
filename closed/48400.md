## Summary

Current `main` already hardens embedded-runner teardown against the retry/persistence race by waiting for the agent to become idle before flushing pending tool results and only disposing the session afterward. The fix is present on `main` in commit `48b9452c0795e749bd25406b2e93213b8533fee4`, but it does not appear to be included in the latest tagged release `v2026.4.22`.
