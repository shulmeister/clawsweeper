## Summary

Keep #72210 open. Current main still produces a failed `delivery.resolved` trace for bare `delivery.mode="none"` cron runs: the delivery status path is correctly `not-requested`, but `buildCronDeliveryTrace()` still serializes `resolved.ok: false` with an error. The linked PR #72219 is the narrow fix, but it is open and unmerged.

## Best Possible Solution

Keep #72210 open until PR #72219 or an equivalent maintainer-approved change lands. The fix belongs in `buildCronDeliveryTrace()`: for explicit `delivery.mode="none"`, omit `delivery.resolved` or emit neutral skipped metadata while preserving `deliveryStatus: "not-requested"`, explicit no-deliver target context for agent `message` tool use, and existing announce/webhook trace behavior. Add a focused regression test that inspects `result.delivery` or the emitted run-log entry for the no-deliver trace shape.
