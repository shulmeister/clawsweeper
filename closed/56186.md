## Summary

Keep open. Current main still has the user-visible bug this PR targets: a failed channel config save writes `lastError`, then `handleChannelConfigSave` immediately reloads config and channels without preserving that error, so the Channels tab error surface is cleared.

## Best Possible Solution

Keep the PR open. The best maintainer path is to land the narrow error-preservation behavior after a current-main rebase and, preferably, add a focused UI/controller regression test proving that a failed `config.set` remains visible on the Channels tab after the config/channel refresh sequence.

## Evidence

- **Current handler lacks preservation:** On current main, `handleChannelConfigSave` calls `saveConfig`, then `loadConfig`, then `loadChannels` with no capture of `host.lastError` and no restoration to `host.channelsError`. The PR's core change is not present on main.
