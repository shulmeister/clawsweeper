## Summary

Keep open. Current main still contains the exact heartbeat-runner behavior this PR tries to change: exec prompts can still be user-relayable, HEARTBEAT_OK delivery is not gated off for exec completions, reasoning delivery is not gated off for exec completions, and exec replies that normalize to an empty HEARTBEAT_OK can be revived by exec fallback text. Existing tests also still assert relayable exec completion behavior, so this is not implemented on main and is not cleanly superseded by #70519, which is narrower and explicitly preserves user-relayable exec completions.

## Best Possible Solution

Maintainers should decide the async exec completion delivery contract explicitly. If exec completions are now meant to be internal-only, update `src/infra/heartbeat-events-filter.ts`, `src/infra/heartbeat-runner.ts`, and the current relay-oriented tests/docs to match this PR's guards. If relayable exec completions are still intended, close this PR with that product rationale and keep #70519 or a narrower follow-up focused only on internal/no-target Control UI transcript leakage.
