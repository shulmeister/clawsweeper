## Summary

Current main has moved sandbox extensibility to a plugin-backed backend seam, so this 29k-line core Firecracker/gVisor/vm-runner PR is no longer aligned with OpenClaw core scope. The right path is an external sandbox backend plugin built on the public sandbox plugin SDK, similar to the existing OpenShell backend.

- Closing on scope does not evaluate the full technical merit of every subcomponent in the PR; some ideas may still be worth reviving in a separate plugin repository.

## Close Comment

Closing this PR after Codex review. OpenClaw main has moved sandbox extensibility to a plugin-backed backend model: `src/agents/sandbox/backend.ts:36` provides backend registration, that seam is public in `src/plugin-sdk/sandbox.ts:24`, and sandbox resolution now goes through the registered backend factory in `src/agents/sandbox/context.ts:155`. The project scope also now explicitly says optional capability should usually ship as plugins, not core (`VISION.md:52`). There is already a concrete plugin-backed managed sandbox example in `extensions/openshell/index.ts:1` and the docs treat `openshell` as the managed backend path (`docs/gateway/openshell.md:10`). Given that direction, a very large core PR that adds gVisor/Firecracker plus a separate Go runner is out of scope for OpenClaw core. If you want to continue this work, the maintainable path is a focused external sandbox backend plugin built on the sandbox plugin SDK, with any missing SDK seams proposed separately in small additive PRs.
