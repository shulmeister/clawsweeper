## Summary

Current `main` already routes CLI-backed turns back into OpenClawŌĆÖs own session transcript before `/compact` runs, so the `"no real conversation messages yet"` skip described in #70304 no longer matches shipped/current behavior. The same implementation is present in release `v2026.4.23`.

## Evidence

- **`/compact` still only skips when the OpenClaw transcript lacks real conversation messages:** `compactEmbeddedPiSession` returns `reason: "no real conversation messages"` only when `containsRealConversationMessages(session.messages)` is false, so the bug depends on CLI turns failing to enter the OpenClaw transcript.
  - file: [src/agents/pi-embedded-runner/compact.ts:1027](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/src/agents/pi-embedded-runner/compact.ts#L1027)
  - command: `nl -ba src/agents/pi-embedded-runner/compact.ts | sed -n '1027,1035p'`
  - sha: [0e9156d20530](https://github.com/openclaw/openclaw/commit/0e9156d2053022d208c3eea1d0fa1e377ed0a154)
- **Current main persists every CLI runner turn into the OpenClaw session transcript:** After a CLI-backed run completes, `agent-command.ts` calls `persistCliTurnTranscript(...)` for `result.meta.executionTrace?.runner === "cli"`, wiring CLI output into the same session transcript `/compact` later reads.
  - file: [src/agents/agent-command.ts:1071](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/src/agents/agent-command.ts#L1071)
  - command: `nl -ba src/agents/agent-command.ts | sed -n '1071,1089p'`
  - sha: [0e9156d20530](https://github.com/openclaw/openclaw/commit/0e9156d2053022d208c3eea1d0fa1e377ed0a154)
- **CLI transcript persistence writes canonical user and assistant messages:** `persistTextTurnTranscript` resolves the canonical session JSONL via `resolveSessionTranscriptFile(...)`, opens it with `SessionManager`, and appends both the user prompt and assistant reply. `persistCliTurnTranscript` uses that path for CLI providers.
  - file: [src/agents/command/attempt-execution.ts:98](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/src/agents/command/attempt-execution.ts#L98)
  - command: `nl -ba src/agents/command/attempt-execution.ts | sed -n '98,152p;194,230p'`
  - sha: [0e9156d20530](https://github.com/openclaw/openclaw/commit/0e9156d2053022d208c3eea1d0fa1e377ed0a154)
- **Regression coverage now asserts `claude-cli` transcript write-through:** `attempt-execution.cli.test.ts` verifies that `persistCliTurnTranscript` writes a `user` entry plus an `assistant` entry whose provider is `claude-cli`, and that the persisted user text uses the transcript-visible body rather than runtime-only prompt context.
  - file: [src/agents/command/attempt-execution.cli.test.ts:351](https://github.com/openclaw/openclaw/blob/0e9156d2053022d208c3eea1d0fa1e377ed0a154/src/agents/command/attempt-execution.cli.test.ts#L351)

  - sha: [0e9156d20530](https://github.com/openclaw/openclaw/commit/0e9156d2053022d208c3eea1d0fa1e377ed0a154)
- **The fix is already in the shipped release:** The release tag `v2026.4.23` contains the same CLI transcript persistence hooks in both `src/agents/agent-command.ts` and `src/agents/command/attempt-execution.ts`, so this is not just unreleased `main` behavior.
  - file: [src/agents/agent-command.ts:1052](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/agents/agent-command.ts#L1052)

- **Issue discussion identified the right surface:** The provided issue comments already narrowed this to a CLI transcript/maintenance lifecycle gap, including the cross-reference to #68388; the current code now does that transcript write-through before `/compact` evaluates the session.
