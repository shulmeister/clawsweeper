## Summary

Current `main` already changes heartbeat exec-completion relays to embed the actual event text in the prompt, adds an empty-output fallback, and covers the behavior with regression tests. The fix appears unreleased as of `v2026.4.23`.

## Evidence

- **Heartbeat runner now passes exec event text into the prompt:** `resolveHeartbeatRunPrompt` collects matching exec-completion events into `execEvents` and passes them to `buildExecEventPrompt(execEvents, ...)`, instead of building an exec relay prompt without attached event details.
