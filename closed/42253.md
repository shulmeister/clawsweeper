## Summary

Keep open. Current main still does not implement the PR's central request: a default-off `tools.sessions.ownedAcp.enabled` exception that allows tree-scoped access to creator-owned ACP sessions. The existing visibility guard denies cross-agent targets unless visibility is `all`, config/docs only expose `tools.sessions.visibility`, and the `sessions_send(label + agentId)` path still rejects sandboxed cross-agent lookups before any owned-ACP resolution path could apply.

## Best Possible Solution

Keep the PR open, rebase it onto current main, and implement the owned-ACP exception in the current `src/plugin-sdk/session-visibility.ts` based guard plus the session tool lookup/list paths. Add the config type, Zod schema, help labels, docs, and regression tests for `sessions_list`, `sessions_history`, `sessions_send(label + agentId)`, sandboxed sends, unrelated ACP denial, and any intended `session_status` behavior; address the outstanding list-limit/spawned-cap review concerns during maintainer review.
