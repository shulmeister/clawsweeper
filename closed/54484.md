## Summary

Current main already carries Telegram inbound media downloads as `MediaPath`/`MediaPaths` and the shared reply prompt renders those fields as explicit `[media attached: ...]` lines for the agent. This covers the original image-path report and the later document/file attachment follow-up.

## Best Possible Solution

Keep the shipped implementation: Telegram owns media download/path propagation, while the shared reply prompt layer turns `MediaPath`/`MediaPaths` into agent-visible `[media attached: ...]` lines. Any future preference to also mutate the raw `Body` field should be tracked as a separate UX/API request, not this bug.
