## Summary

Current `main` already provides inline token/cost visibility during conversations: chat replies can append per-response usage/cost footers, and the web chat UI renders compact token/cost badges on assistant message groups. The exact `⚡` indicator placement from the issue was not used, but the requested functionality is implemented well enough.

## Risks / Open Questions

- The implementation surfaces usage in reply footers and web-chat badges, not inside the exact in-progress `⚡` indicator proposed in the issue.
- Estimated dollar cost is intentionally hidden for OAuth/subscription auth paths, so cost visibility is token-only there.
