## Summary

Keep PR #49669 open. Current main still has the lazy-import failure mode the PR is trying to address: `createLazy()` caches a `loader().then(...)` promise with no rejection handler, and `lazyRender()` renders `nothing` while the module remains unavailable. The PR is plausible and tied to open bug #49665, but the submitted patch is not safe to auto-close or treat as obsolete because review comments identify a real retry-loop risk from clearing `s.promise` and immediately calling `_pendingUpdate()` on persistent import failures.

## Best Possible Solution

Keep the PR open and request a revised patch: add lazy import rejection handling that logs or surfaces the failure and allows a deliberate retry without immediately triggering an unbounded `_pendingUpdate()` loop. A conservative implementation could keep a failure state, throttle or cap retries, or retry only on navigation/manual refresh, with focused coverage around `createLazy()`/`lazyRender()` behavior for rejected imports.
