## Summary

Current `main` already handles `dreaming` as a `memory-core` command alias rather than a standalone plugin id, redirects `plugins.allow` users to `memory-core`, and documents `/dreaming` as a runtime slash command with `openclaw memory` as the CLI surface. The changelog shows this shipped in `v2026.4.9`.

## Evidence

- **CLI now resolves plugin-owned command aliases:** `resolveMissingPluginCommandMessage` looks up manifest command aliases, redirects `dreaming` to parent plugin `memory-core` for `plugins.allow`, and explicitly reports that `dreaming` is a runtime slash command rather than a CLI command.
  - file: [src/cli/run-main.ts:90](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/cli/run-main.ts#L90)

- **Manifest marks dreaming as memory-core-owned alias:** `memory-core` declares `dreaming` in `commandAliases` with `kind: "runtime-slash"` and `cliCommand: "memory"`, which is the metadata the CLI/config guidance now uses.
  - file: [extensions/memory-core/openclaw.plugin.json:7](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/memory-core/openclaw.plugin.json#L7)
