## Summary

Current main already avoids the reported stale `google-vertex:default` failure path: expired/ineligible stored profiles are filtered before selection, the embedded runner falls back to an env-only auth attempt when no eligible profile remains, and `google-vertex` now resolves ADC from `GOOGLE_APPLICATION_CREDENTIALS` or the default gcloud ADC path.

## Evidence

  - file: [src/agents/auth-profiles/order.ts:66](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/agents/auth-profiles/order.ts#L66)
