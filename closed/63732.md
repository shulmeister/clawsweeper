## Summary

Keep open. Current `main` still has the reported failure mode: system events bypass freshness evaluation but still persist `updatedAt: Date.now()`, so the next real user turn can see the session as fresh and skip the documented daily reset. A direct fix was proposed in PR #63801, but that PR was closed unmerged and its patch is not present on `main`.

## Evidence

- **Current main still refreshes `updatedAt` on system events:** `initSessionState` treats heartbeat/cron/exec as system events and forces freshness to `{ fresh: true }`, but later writes `updatedAt: Date.now()` into the persisted session entry unconditionally.
  - file: [src/auto-reply/reply/session.ts:431](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/auto-reply/reply/session.ts#L431)
