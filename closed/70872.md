## Summary

#70872 reports a real bug that current main still does not fix directly, but the remaining work is now tracked by open PR #70882, which targets the exact Synology-style `params: "{}"` failure with schema-guided object/array coercion and regression tests. Close this issue as superseded by the canonical PR rather than as implemented on main.

## Best Possible Solution

Use #70882 as the canonical place to finish and land the schema-guided stringified object/array coercion for bundle MCP tool calls. Keep #70872 closed as superseded by that focused PR, and reopen only if the PR stops tracking the fix without an equivalent main-branch implementation.

## Evidence

- **Current main still forwards materialized MCP inputs unchanged:** The bundle MCP materializer exposes each tool's `inputSchema` as parameters, then calls `params.runtime.callTool(tool.serverName, tool.toolName, input)` with the raw execute input; there is no nested property coercion in current main.
  - file: [src/agents/pi-bundle-mcp-materialize.ts:113](https://github.com/openclaw/openclaw/blob/6b6dcafcee9ca86385b7c1ee2442752314d50257/src/agents/pi-bundle-mcp-materialize.ts#L113)

  - sha: [6b6dcafcee9c](https://github.com/openclaw/openclaw/commit/6b6dcafcee9ca86385b7c1ee2442752314d50257)
- **Runtime forwards record values to MCP SDK arguments:** The session MCP runtime only checks that the top-level input is a record, then passes it as MCP `arguments`; a nested `params: "{}"` value would remain a string at this boundary.
  - file: [src/agents/pi-bundle-mcp-runtime.ts:358](https://github.com/openclaw/openclaw/blob/6b6dcafcee9ca86385b7c1ee2442752314d50257/src/agents/pi-bundle-mcp-runtime.ts#L358)
  - command: `nl -ba src/agents/pi-bundle-mcp-runtime.ts | sed -n '240,380p'`
  - sha: [6b6dcafcee9c](https://github.com/openclaw/openclaw/commit/6b6dcafcee9ca86385b7c1ee2442752314d50257)
- **Top-level record helper does not coerce nested strings:** `isMcpConfigRecord` only verifies `value !== null && typeof value === "object" && !Array.isArray(value)`, so it does not parse nested stringified object or array properties.
  - file: [src/agents/mcp-config-shared.ts:3](https://github.com/openclaw/openclaw/blob/6b6dcafcee9ca86385b7c1ee2442752314d50257/src/agents/mcp-config-shared.ts#L3)
