## Summary

Current `main` already fixes this Discord inbound-image path through a different design: the gateway now rewrites offloaded attachments to opaque `media://inbound/<id>` refs and the runner resolves those refs through the media store, so a workspace sandbox bypass allowlist is no longer needed. The companion fallback-model image-capability path is also covered on `main` and shipped in `v2026.4.22`.
