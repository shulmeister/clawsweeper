## Summary

Current main already addresses the Windows browser-control `ERR_UNSUPPORTED_ESM_URL_SCHEME` repro from v2026.4.24. The issue context shows a real Windows service-start failure after browser runtime deps staged successfully, but main now disables native Jiti loading on win32 and normalizes Windows absolute plugin import paths before module loading. The relevant regression tests cover the browser bundled dist path and Windows file-URL conversion; no tag newer than v2026.4.24 is present in this checkout, so no shipped fixed release is assigned.

## Best Possible Solution

Close as implemented on current main. Keep the win32 non-native Jiti policy and Windows file-URL normalization in the shared plugin loader path; if this reproduces on a build containing current main or the next release, ask for a fresh issue with exact version, logs, diagnostics, and affected plugin/service details.
