## Summary

Current main already has hard-enforced subagent tool scoping. The capability ships as config- and agent-policy-based tool filtering rather than a new `sessions_spawn.allowedTools` parameter, but it satisfies the issue's core safety request.

## Evidence

- **Subagent tool policy is documented:** Docs state that subagents can be restricted with `tools.subagents.tools.allow` / `tools.subagents.tools.deny`, and show allow-only / deny behavior for spawned workers.
  - file: [docs/tools/subagents.md:288](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/docs/tools/subagents.md#L288)
  - command: `nl -ba docs/tools/subagents.md | sed -n '288,322p'`
- **Config reference exposes per-subagent tool policy:** Gateway config docs explicitly list `tools.subagents.tools.allow` / `tools.subagents.tools.deny` as the per-subagent tool policy for `sessions_spawn`.
  - file: [docs/gateway/config-tools.md:350](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/docs/gateway/config-tools.md#L350)
  - command: `nl -ba docs/gateway/config-tools.md | sed -n '345,352p'`
- **Runtime builds enforced child-session policy:** `resolveSubagentToolPolicy` / `resolveSubagentToolPolicyForSession` compute the spawned session's allow/deny tool policy from `cfg.tools.subagents.tools` plus stored subagent role/capabilities.
  - file: [src/agents/pi-tools.policy.ts:81](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/agents/pi-tools.policy.ts#L81)
  - command: `nl -ba src/agents/pi-tools.policy.ts | sed -n '80,125p'`
- **Runtime applies subagent policy when materializing tools:** The subagent policy is injected into the tool filtering pipeline before tools are exposed to the child session, so unavailable tools are hard-removed rather than soft-prompted away.
  - file: [src/agents/pi-tools.ts:407](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/agents/pi-tools.ts#L407)
  - command: `nl -ba src/agents/pi-tools.ts | sed -n '400,445p'`
- **Tests cover default restriction and explicit allow/deny:** Tests verify default removal of session tools from subagents and verify explicit subagent allow/deny behavior, including deny precedence.
  - file: [src/agents/pi-tools.create-openclaw-coding-tools.test.ts:216](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/agents/pi-tools.create-openclaw-coding-tools.test.ts#L216)

## Risks / Open Questions

- The shipped solution is config/agent-policy based, not a per-call `sessions_spawn.allowedTools` argument. If maintainers want spawn-time one-off overrides later, that would be a separate enhancement rather than a missing safety boundary.
