## Summary

Current `main` does not have the `web_fetch` browser-provider codepath described in the report. `web_fetch` is implemented as guarded HTTP fetch plus optional Firecrawl fallback, while Chrome lifecycle belongs to the separate `browser` plugin/tool, so the reported renderer leak does not map to a reproducible current `web_fetch` path.

## Evidence

- **`web_fetch` docs describe plain HTTP, not browser automation:** The public docs for `web_fetch` state it does a plain HTTP GET, extracts readable content, and does not execute JavaScript; JS-heavy pages should use the separate Browser tool instead.
  - file: [docs/tools/web-fetch.md:11](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/docs/tools/web-fetch.md#L11)
  - command: `nl -ba docs/tools/web-fetch.md | sed -n '1,90p'`
  - sha: [2011de69d3dc](https://github.com/openclaw/openclaw/commit/2011de69d3dc8f1a11ebc774ba4af23f54892461)
- **`web_fetch` runtime uses guarded fetch and response parsing:** `runWebFetch` calls `fetchWithWebToolsNetworkGuard`, handles an HTTP `Response`, and reads response text. There is no browser/session/process lifecycle in this codepath.
  - file: [src/agents/tools/web-fetch.ts:415](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/src/agents/tools/web-fetch.ts#L415)

- **Tool description explicitly excludes browser automation:** The `web_fetch` tool is labeled as lightweight page access 'without browser automation', and its provider fallback is resolved through `resolveWebFetchDefinition`, not through the browser plugin.
  - file: [src/agents/tools/web-fetch.ts:616](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/src/agents/tools/web-fetch.ts#L616)

- **Browser lifecycle is registered as a separate plugin/tool:** The browser extension registers its own `browser` tool, CLI, gateway method, and service. This is the current Chrome-managed surface, separate from `web_fetch`.
  - file: [extensions/browser/plugin-registration.ts:28](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/extensions/browser/plugin-registration.ts#L28)
