## Summary

Current main implements the conservative fix for the reported Mission Control/Codex quota drain. Fresh workspaces now seed a comment-only HEARTBEAT.md scaffold, interval heartbeats skip before any model call when that file is effectively empty, multi-agent heartbeats are explicit opt-in per agent, and intentional heartbeat runs have documented low-cost controls such as model override, lightContext, isolatedSession, and 0m disable.

## Best Possible Solution

Close as implemented. Keep the current heartbeat gating, default-agent-only scheduler behavior, and documented cost controls; if maintainers want an additional warning for intentionally configuring many active Codex OAuth heartbeats, that should be a separate UX enhancement.
