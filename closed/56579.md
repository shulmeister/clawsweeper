## Summary

Keep open. Current main still matches the reported ClawHub parser bug: `parseClawHubPluginSpec("clawhub:demo@")` falls through to `{ name: "demo@" }`, the install path treats that as a valid ClawHub spec, and 404 package lookup maps to `package_not_found` rather than `invalid_spec`. The linked fix PR #56582 is still open, and the later related PR #56790 is closed unmerged, so neither supersedes the issue.

## Best Possible Solution

Keep #56579 open and land a focused fix in `src/infra/clawhub-spec.ts`: return `null` when an explicit version separator is present with no version text after it, while preserving valid forms like `clawhub:demo`, `clawhub:demo@1.2.3`, and `clawhub:@scope/name`. Add regression tests in `src/infra/clawhub.test.ts` for `clawhub:demo@`, `clawhub:@scope/pkg@`, and valid scoped package parsing; PR #56582 already describes that shape of fix and can remain the canonical implementation attempt if still viable.
