## Summary

Current main no longer depends on `OPENCLAW_EXTENSIONS=telegram` to avoid the reported Telegram startup failure. Enabled bundled plugins/channels now stage or install missing runtime dependencies before import, Telegram opts into that path, and current docs describe `OPENCLAW_EXTENSIONS` as a build-time preinstall optimization rather than a required Telegram switch.

## Evidence

- **Bundled plugin loader repairs missing runtime deps before import:** `loadOpenClawPlugins` now resolves an install root, calls `ensureBundledPluginRuntimeDeps(...)`, logs any installed specs, and only then imports the plugin. On failure it records an explicit `failed to install bundled runtime deps` load error instead of silently continuing.
