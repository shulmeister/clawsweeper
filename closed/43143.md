## Summary

Keep #43143 open. Current main confirms the 10-open-PR cap is intentional, hard-coded, documented, and enforced by Barnacle, but the issue asks whether docs-only or size:XS/S batch contributors should get a higher threshold. That remaining question is a maintainer policy/product decision, not something that can be conservatively auto-closed as already fixed or non-actionable.

## Best Possible Solution

A maintainer should make an explicit policy call: keep the hard 10-open-PR limit with the existing manual override/Discord coordination path, or implement a narrow exception in `.github/workflows/labeler.yml` plus matching Barnacle messaging/tests and `CONTRIBUTING.md` docs for docs-only or size:XS/S batch PRs.
