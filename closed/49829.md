## Summary

Current main already routes the Control UI chat model picker through provider-qualified `provider/model` values, and the same implementation is present in the released `v2026.4.22` tree.

## Evidence

- **Picker submits selected option value unchanged:** The Control UI chat model `<select>` reads option values from `resolveChatModelSelectState(...)` and sends the selected value directly in `sessions.patch({ model: next })`, so the backend receives the full option value rather than re-deriving a provider from session state.
  - file: [ui/src/ui/chat/session-controls.ts:83](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/ui/src/ui/chat/session-controls.ts#L83)
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Option values are built as provider-qualified refs:** `buildQualifiedChatModelValue()` prefixes `provider/model` and `buildChatModelOptionFromLookup()` uses it for every catalog option value.
  - file: [ui/src/ui/chat-model-ref.ts:5](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/ui/src/ui/chat-model-ref.ts#L5)
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Select state prefers qualified catalog/server refs:** The selector state normalizes cached and server session models through `resolvePreferredServerChatModelValue(...)` and emits options from `buildChatModelOptionFromLookup(...)`, preventing stale-provider drift.
  - file: [ui/src/ui/chat-model-select-state.ts:34](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/ui/src/ui/chat-model-select-state.ts#L34)
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Regression tests cover non-default-provider cases:** Tests assert that the picker patches `openai/gpt-5-mini`, that DeepSeek rows resolve to `deepseek/deepseek-chat`, and that stale provider metadata is corrected instead of producing the wrong prefixed model.
  - file: [ui/src/ui/chat/session-controls.test.ts:139](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/ui/src/ui/chat/session-controls.test.ts#L139)
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Released build already contains the same implementation:** No differences appear in the relevant Control UI model-picker files between release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`) and current main.
