## Summary

Keep PR #70803 open. Current main still rejects `@hosted.lid` in the shared WhatsApp target normalizer, while adjacent WhatsApp runtime and identity paths already recognize hosted LID JIDs. The PR is a focused plugin bug fix with matching regression coverage and is not superseded by the related LID or allowFrom reports.

## Best Possible Solution

Keep the PR open for maintainer review. The likely path is to land the small WhatsApp plugin normalizer/test update, or adjust it if maintainers want stricter mapping-backed LID semantics; it should not be auto-closed as implemented, duplicate, ClawHub, or not-actionable work.
