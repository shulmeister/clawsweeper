## Summary

Issue #49844 requests a native Outlook integration in OpenClaw core, but current project direction and plugin architecture point this class of platform integration to external plugins/ClawHub rather than core.

## Evidence

- **Vision: optional capability should ship as plugins:** OpenClaw's vision says core stays lean, optional capability should usually ship as plugins, external plugins should be hosted in their own repository, and the bar for adding optional plugins to core is intentionally high.
