## Summary

Keep #58584 open. Current main has generic cron error backoff and provider/model fallback, but it does not implement the requested provider-reachability preflight before cron jobs are accepted/scheduled or before an agent run is dispatched. A dead Ollama/custom provider can still be scheduled, reach the runtime path, fail there, and only then get recorded/backed off as a normal cron failure.

## Best Possible Solution

Add a provider-aware readiness layer that cron and agent message admission can consult before starting runs. It should resolve the effective provider/model for each agent/job, run a lightweight provider-owned health probe where available, log a clear warning for unreachable providers, skip or defer affected cron executions without marking them as normal noisy run failures, retry readiness on a bounded interval such as five minutes, and add focused tests for Ollama/local-provider unreachable-to-reachable transitions plus existing cron backoff behavior.
