## Summary

Keep open. Current main still matches the reported bug: `/tools` builds its inventory from `resolveEffectiveToolInventory()`, which only enumerates `createOpenClawCodingTools()` core/plugin/channel tools, while the actual agent run path separately materializes and merges MCP server tools. The linked PR #70876 directly tracks the fix, but the provided GitHub context shows it is still open and unmerged.

## Best Possible Solution

Keep #70874 open until #70876 or an equivalent patch lands. The likely fix should extend the `/tools` inventory path to include session MCP server tools from `getOrCreateSessionMcpRuntime()` and `materializeBundleMcpToolsForRun()`, add an MCP inventory source/group, preserve graceful fallback when MCP runtime is unavailable, and cover both compact and verbose `/tools` output with regression tests.
