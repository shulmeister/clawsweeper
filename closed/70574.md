## Summary

Current main still matches the reported bug path: cross-agent spawns resolve `requesterOrigin.accountId` from the target child agent's binding, then completion routing uses that account when matching child-session bindings. The linked fix PR #70607 is still open/unmerged, and existing tests currently assert the target-account behavior for some thread-binding cases, so this needs maintainer repair rather than cleanup closure.

## Best Possible Solution

Keep #70574 open and fix the routing contract in source: preserve the parent/requester accountId for completion announce-back, while passing a separate target-agent-bound account only to child thread-binding or initial child delivery paths that genuinely need it. Add regression coverage for the Telegram/non-thread cross-agent case from the issue, same-agent spawns, and nested `streamTo="parent"` behavior, then reconcile or update the existing target-account thread-binding tests.
