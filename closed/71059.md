## Summary

Current `main` already contains the config-recovery and write-preservation changes that address this report’s failure shape: plugin-entry-only invalidity no longer triggers whole-file `.last-good` rollback during gateway startup/reload, and partial config writes merge onto the authored source snapshot instead of clobbering nested fields.

## Evidence

- **Startup recovery is now gated to avoid plugin-local whole-file rollback:** Gateway startup now calls `shouldAttemptLastKnownGoodRecovery(configSnapshot)` before restoring `.last-good`, and explicitly skips that rollback for plugin-local invalidity instead of rewriting the config file from a stale backup.
  - file: [src/gateway/server-startup-config.ts:74](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/gateway/server-startup-config.ts#L74)
  - command: `nl -ba src/gateway/server-startup-config.ts | sed -n '60,110p'`
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Recovery policy classifies plugin-entry-only failures as unsafe for whole-file restore:** `isPluginLocalInvalidConfigSnapshot(...)` returns true when validation issues are confined to `plugins.entries.*`, and `shouldAttemptLastKnownGoodRecovery(...)` refuses whole-file recovery in that case.
  - file: [src/config/recovery-policy.ts:16](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/config/recovery-policy.ts#L16)

  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Write path preserves authored nested config during partial writes:** `writeConfigFile(...)` reads the current snapshot and passes runtime/source/next config into `resolvePersistCandidateForWrite(...)`, which applies the caller patch onto the projected source snapshot instead of persisting a narrowed runtime object.
  - file: [src/config/io.ts:1624](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/config/io.ts#L1624)
  - command: `nl -ba src/config/io.ts | sed -n '1550,1715p'`
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Merge helper keeps source-owned fields on unrelated writes:** `resolvePersistCandidateForWrite(...)` creates a merge patch from runtime to next config, projects the authored source snapshot onto the runtime shape, and applies the patch there, which is the preservation logic needed to keep nested channel/auth fields from being dropped on incidental writes.
  - file: [src/config/io.write-prepare.ts:149](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/config/io.write-prepare.ts#L149)

- **Regression tests cover both stale rollback avoidance and nested-field preservation:** Tests now assert that stale last-known-good recovery is skipped for plugin `minHostVersion` skew and plugin-local reload invalidity, and that partial writes preserve source-only nested fields and normalized channel subtrees during unrelated writes.
