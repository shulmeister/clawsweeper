## Summary

Current main has moved Windows gateway management onto first-party service commands and docs (WSL2 recommended; native Windows via Scheduled Task with Startup-folder fallback), so the PM2 `reload --update-env` quirk reported here is no longer part of the documented/supported OpenClaw path. That replacement shipped in `v2026.4.23` via commit `a9797214338ba31c52c796adbb75afb16e0684a9`.
