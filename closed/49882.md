## Summary

Current `main` already suppresses the noisy user-facing tool-error case described here: non-mutating recoverable tool errors are suppressed, recovered runs with a final assistant reply do not emit an extra chat warning, and a broader `messages.suppressToolErrors` config exists for non-mutating tool warnings. The same implementation is present in released tag `v2026.4.22`.

## Evidence
