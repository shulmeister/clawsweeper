## Summary

Current `main` already covers the reported agent-CLI need: `openclaw agent --timeout` is converted into an explicit Gateway RPC timeout budget, the request is sent as `expectFinal`, and explicit `--session-id` routing is preserved instead of collapsing back to the main session. The generic `GatewayClient` still has a 30s default for non-`expectFinal` RPCs, which matches the triage comment, but that is not the active path for the agent CLI flow described in this issue.

## Evidence

- **Agent CLI maps its timeout to the Gateway request budget:** `agentViaGatewayCommand` parses `--timeout`, computes `gatewayTimeoutMs`, and calls `callGateway({ method: "agent", expectFinal: true, timeoutMs: gatewayTimeoutMs, ... })`, so agent turns are not left on the base 30s client default.
  - file: [src/commands/agent-via-gateway.ts:115](https://github.com/openclaw/openclaw/blob/a44800e929f742ca72d068e5ff3bd2a8eac2b71b/src/commands/agent-via-gateway.ts#L115)
