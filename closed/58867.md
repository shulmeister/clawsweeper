## Summary

Keep PR #58867 open. Current main still treats Feishu `@_all` as a bot mention unconditionally, so the user-facing bug this PR addresses remains present. The proposed PR is not obsolete: it adds the missing opt-in behavior and, per the provided PR context, its second commit addresses the review concern about strict `resolveFeishuReplyPolicy()` test expectations.

## Best Possible Solution

Keep the PR open, or land an equivalent Feishu plugin fix that gates `@_all` behind an explicit opt-in setting, wires that setting through schema and policy resolution, updates docs/config metadata if accepted as public config, and keeps the regression tests for default false, opt-in true, and missing bot identity.
