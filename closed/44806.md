## Summary

Current main consistently treats `OPENCLAW_HOME` as the home directory and `OPENCLAW_STATE_DIR` as the state-root override. The nested `.openclaw/.openclaw/workspace` path only appears when `OPENCLAW_HOME` is pointed at the `.openclaw` state dir itself, which does not match the documented/current contract. The PR also only special-cases workspace resolution, which would make workspace behavior diverge from agent/state path resolution on main.

## Evidence

- **Documented env contract:** `docs/help/environment.md:108` and `docs/help/environment.md:118` define `OPENCLAW_HOME` as the home-directory override and `OPENCLAW_STATE_DIR` as the state-directory override.
  - file: [docs/help/environment.md:108](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/docs/help/environment.md#L108)

- **Workspace path follows home semantics:** `resolveDefaultAgentWorkspaceDir()` always appends `.openclaw/workspace` to the resolved home dir, so supported usage is `OPENCLAW_HOME=<home>`, not `OPENCLAW_HOME=<home>/.openclaw`.
  - file: [src/agents/workspace.ts:17](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/agents/workspace.ts#L17)
