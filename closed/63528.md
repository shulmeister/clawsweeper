## Summary

Current `main` already routes `sessions_send` through the gateway `agent` run path, which starts the target session immediately rather than leaving the message parked for a later heartbeat/cron wake.

## Evidence

- **`sessions_send` now starts a target agent run directly:** The tool builds `sendParams` for the resolved target session and sends them through `callGateway({ method: "agent" ... })`; both the fire-and-forget and wait-for-reply paths use this direct run path instead of a passive queue.
