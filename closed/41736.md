## Summary

Keep open. Current main has useful mitigations for per-agent model overrides, agent-scoped auth writes, auth/status visibility, and hot reload, but it still intentionally uses a multi-location config/auth model rather than the requested single source of truth, and configured model fallback on provider/auth failure remains documented behavior.

## Best Possible Solution

Keep this issue open or explicitly consolidate it into a canonical agent/model configuration hierarchy design issue. The best path is a maintainer-owned DX design that defines the intended source of truth for model selection and auth profiles, keeps per-agent config and auth storage discoverable, preserves backward compatibility/migrations for openclaw.json and agentDir state, and decides where hard-error behavior should replace configured fallback behavior.
