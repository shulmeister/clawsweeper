## Summary

Current `main` already prevents the Discord multi-account startup burst this PR targets. The mitigation landed in the Discord plugin itself rather than as a core `channels.discord.connectStagger` config: OpenClaw now computes a per-account startup delay and sleeps before later Discord accounts start, explicitly to reduce Discord startup rate limits. That behavior is present in the shipped `v2026.4.22` tree, so this PR's requested fix is already implemented well enough on `main` via a plugin-owned solution.
