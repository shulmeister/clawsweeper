## Summary

Current `main` already implements native-`pnpm` handling for this report through the shared pnpm runner, with regression tests for both native ELF binaries and extensionless shebang shims. The PR’s extra `sanitizeEnv` path is therefore redundant, and the review discussion already identified a shim-regression risk that current `main` avoids.

## Evidence

- **Shared runner already avoids `node <native-binary>`:** `resolvePnpmRunner` only runs `npm_execpath` through Node when it is a JS file or an extensionless script with a shebang; otherwise it falls back to invoking `pnpm` directly, which covers the native-binary case described in the PR.
