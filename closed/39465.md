## Summary

Current `main` already handles the Discord DM/native-command target fallback through the newer `OriginatingTo` + conversation-identity path, so this older `src/channels/dock.ts` PR is obsolete. The same behavior is present in release `v2026.4.23`.

## Evidence

- **Threading tool context uses `OriginatingTo` before `To`:** `buildThreadingToolContext` resolves `originTo` from `OriginatingTo ?? To` and, when no channel-specific `buildToolContext` exists, injects that as `currentChannelId`. That is the current fallback path used for Discord native-command contexts.
