## Summary

The remaining work is WeChat-specific behavior in the already external `@tencent-weixin/openclaw-weixin` channel plugin. Current OpenClaw core provides the generic BOOT.md/message-tool/outbound plugin seams; the reported `contextToken` handling, stored OpenID/session policy, template-message fallback, and Tencent iLink API behavior are owned by the Weixin plugin, with no concrete missing core SDK seam identified.

## Evidence

- **WeChat runtime is external plugin owned:** The WeChat docs state that OpenClaw connects through Tencent's external `@tencent-weixin/openclaw-weixin` plugin and that WeChat login, Tencent iLink API calls, media, context tokens, and account monitoring are owned by the external plugin, not OpenClaw core.
