## Summary

Current main implements the WhatsApp outbound media path reported broken in 2026.4.8/2026.4.9. The issue discussion describes the same false-success/media-drop class linked to #62214 and #62399, and current code now preserves media through gateway send, routes media payloads through channel sendMedia, loads the WhatsApp media buffer, and builds image/video/audio/document Baileys payloads. The changelog says this shipped in v2026.4.10 for WhatsApp document and attachment sends silently dropping files while captions were delivered.

## Best Possible Solution

Close this pre-v2026.4.10 WhatsApp media-routing regression as implemented on main. Keep the shipped sendMedia/media-buffer path, and handle any fresh current-version recipient-visible WhatsApp media delivery failures in a new narrow live-reproduction issue.
