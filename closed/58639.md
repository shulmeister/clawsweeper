## Summary

Current main already implements the Gmail Pub/Sub + Tailscale path handling and integrated watcher flow that address the reported 404/no-notification setup, and the supported config/docs now differ materially from the manual setup shown in the issue.

## Evidence

- **Tailscale public path vs local gog path is handled in code:** Current main explicitly documents that Tailscale strips the public path before proxying and forces gog to listen on `/` when Tailscale is enabled, which matches the issue’s 404-at-public-path symptom.
  - file: [src/hooks/gmail.ts:83](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/hooks/gmail.ts#L83)
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Setup flow generates the split path/token configuration:** `runGmailSetup` keeps the public Tailscale path separate from the local serve path and passes `pushToken` to the public push endpoint, instead of requiring the manual script shape shown in the issue.
  - file: [src/hooks/gmail-ops.ts:138](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/hooks/gmail-ops.ts#L138)
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Built-in watcher replaces the manual extra watcher:** Gateway startup already auto-starts the Gmail watcher, and the watcher code warns when another watcher is already bound. The issue’s reproduction uses a separate scheduled `gog gmail watch serve`, which current docs now say not to run alongside the Gateway.
  - file: [src/hooks/gmail-watcher.ts:98](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/hooks/gmail-watcher.ts#L98)
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Docs now describe the supported Gmail/Tailscale shape:** The docs require `serve.path: "/"` with public `tailscale.path: "/gmail-pubsub"`, state that Gateway auto-starts `gog gmail watch serve`, and explicitly say not to run a separate watcher.
  - file: [docs/gateway/configuration-reference.md:492](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/docs/gateway/configuration-reference.md#L492)
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Issue config’s templated Gmail session key is not the supported current shape without opt-in:** Current hook docs and validation require `hooks.allowRequestSessionKey=true` plus `hooks.allowedSessionKeyPrefixes` when using the Gmail preset’s templated `sessionKey`, but the issue’s pasted config omits those settings.
  - file: [src/gateway/hooks.ts:91](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/gateway/hooks.ts#L91)
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
