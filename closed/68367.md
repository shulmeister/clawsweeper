## Summary

Current `main` no longer reproduces the reported default-path contamination. Dreaming now defaults `dreaming.storage.mode` to `"separate"`, writes Light/REM phase output under `memory/dreaming/<phase>/YYYY-MM-DD.md` unless a caller explicitly opts back into inline mode, and Session Startup still only loads `memory/YYYY-MM-DD*.md`. That matches the issue's suggested fix to keep dreaming output out of shared daily-memory bootstrap files.

## Risks / Open Questions

- Users who explicitly set `plugins.entries.memory-core.config.dreaming.storage.mode: "inline"` can still opt back into inline daily-memory blocks; the linked PR #68408 suggests follow-up hardening for that opt-in path, but the default behavior reported here is already fixed.
