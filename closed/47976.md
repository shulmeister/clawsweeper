## Summary

Current main already implements Telegram transport-layer reply threading through `channels.telegram.replyToMode`; reply-to ids are applied implicitly from the inbound message without requiring `[[reply_to_current]]`, and the behavior is covered by tests and present in release `v2026.4.22`.

## Evidence

- **Implicit current-message threading is implemented generically:** `applyReplyThreading` injects `currentMessageId` as `replyToId` when reply threading is allowed, before any explicit reply tag parsing. This is transport/runtime behavior, not dependent on the model emitting `[[reply_to_current]]`.
