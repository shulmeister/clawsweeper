## Summary

Keep open. Current main still builds Feishu post messages as a single markdown element after prefixing mentions as literal `<at user_id="...">name</at>` text, so the native post `at` element fix is not implemented. The related PRs are competing open attempts, not a clear merged or canonical superseder for automated closure.

## Best Possible Solution

Keep this open for maintainer review or manual consolidation with the other open Feishu mention PRs. The best code direction is a focused Feishu fix in `extensions/feishu/src/send.ts` that emits native post `at` elements from structured `mentions`, preserves literal `<at ...>` text in the model body, covers the payload shape with tests, and drops unrelated script/merge-conflict changes from the PR branch.
