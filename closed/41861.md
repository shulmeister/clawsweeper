## Summary

Keep #41861 open. Current main still has the exact session export template/replacement mismatch described by the PR: the checked-in HTML template contains formatter-expanded JS placeholder blocks, while `generateHtml()` still performs exact `{{JS}}`, `{{MARKED_JS}}`, and `{{HIGHLIGHT_JS}}` replacements. The latest release tag in this checkout has the same broken state, and existing tests mock or bypass the real template injection path. The PR discussion also adds useful context: review identified the additional `$`-sequence replacement hazard, and the PR addressed it with function replacers.

## Best Possible Solution

Keep this PR or a canonical equivalent open and land a consolidated fix for the session export path: restore compact JS placeholders with a formatter guard, use function replacers for JS/vendor injections so `$` sequences are preserved, and add a regression test that reads the real `template.html` instead of only a mocked compact-template string.
