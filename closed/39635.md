## Summary

Close #39635 as a duplicate of the older open #39120. Current main still lacks the requested `extraBootstrapFiles` config and still limits the bundled `bootstrap-extra-files` hook to recognized bootstrap basenames, but #39120 tracks the same remaining user problem more broadly: arbitrary workspace files in bootstrap context injection.

## Best Possible Solution

Close #39635 as duplicate/superseded by #39120 and track the implementation there. The eventual fix should add a schema-backed/defaults-and-per-agent config surface or equivalent canonical hook/API support, resolve configured files inside the workspace with existing guards, inject them after standard bootstrap files in configured order, apply existing per-file and total budgets, document the public config, regenerate schema/help artifacts, and add focused tests for arbitrary basenames, ordering, missing files, escaping paths, truncation, and override semantics.
