## Summary

Current main does not yet satisfy the requested Dreaming-specific LLM model override. The code has a partial internal `execution.model` shape, but the public `memory-core` plugin schema/docs do not expose a `dreaming.model` key, and the Dream Diary subagent call still runs without passing a provider/model override. The existing local embedding configuration mentioned in the issue comment helps memory search embeddings, but it does not solve the chat/subagent model cost path described by the issue.

## Best Possible Solution

Keep the issue open and implement the override in `memory-core`: add a documented, schema-backed Dreaming model setting, normalize it through the existing Dreaming config helpers, pass it to `generateAndAppendDreamNarrative`/`subagent.run`, and use the existing plugin subagent provider/model override authorization seam (`plugins.entries.memory-core.subagent.allowModelOverride` and optional allowlist) rather than adding a core bypass. Add focused tests for default inheritance, configured override propagation, and docs/schema alignment.
