## Summary

Close as implemented on current `main`: daily `memory/YYYY-MM-DD.md` generation is now a separate pre-compaction memory-flush path owned by `memory-core`, while embedding readiness is scoped to semantic recall/search. The reported `memory-lancedb` validation error is a separate plugin-specific config concern.

## Evidence

- **Public memory docs separate daily notes from embedding-backed search:** The current docs define `memory/YYYY-MM-DD.md` as the daily notes file, describe automatic pre-compaction memory flush as the default mechanism that saves important context to memory files, and describe embeddings only under `memory_search`.

  - file: [docs/reference/session-management-compaction.md:324](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/docs/reference/session-management-compaction.md#L324)
