## Summary

Fish Audio TTS fits OpenClaw's plugin/ClawHub extension model rather than core, and the issue already has a published community plugin linked by a commenter. Current `main` does not ship a Fish Audio provider, but the repo's vision and plugin docs explicitly direct optional speech/provider integrations to external plugins published on ClawHub or npm.

## Evidence

- **Vision keeps optional capability out of core:** OpenClaw's vision says core should stay lean, optional capability should usually ship as plugins, plugin authors should host and maintain plugins in their own repos, and plugin promotion belongs in ClawHub; the bar for adding optional plugins to core is intentionally high.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/VISION.md#L54)

It also lists speech as a first-class plugin capability via `api.registerSpeechProvider(...)`. Public docs counterpart: https://docs.openclaw.ai/plugins/building-plugins
  - file: [docs/plugins/building-plugins.md:11](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/docs/plugins/building-plugins.md#L11)
