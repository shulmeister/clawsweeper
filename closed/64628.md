## Summary

Current main already wires Ollama into the media-understanding path used by the `image` tool, and OpenClaw also auto-registers any config provider that declares an image-capable model. The issue’s reported root cause no longer matches current shipped behavior.

## Evidence

- **Ollama plugin registers media-understanding provider:** The bundled Ollama plugin explicitly calls `api.registerMediaUnderstandingProvider(ollamaMediaUnderstandingProvider)` before registering the chat provider, so Ollama is present in the media-understanding registry used by the image tool.
