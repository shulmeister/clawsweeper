## Summary

Current `main` rejects mixed-type `plugins.entries.active-memory.config.agents` before Active Memory registration or live reload, so the reported runtime crash path is not reproducible in shipped OpenClaw behavior. The PR is a reasonable defensive hardening, but it does not fix a bug that survives the existing manifest/config validation pipeline.

## Evidence

  - file: [extensions/active-memory/openclaw.plugin.json:10](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/extensions/active-memory/openclaw.plugin.json#L10)
