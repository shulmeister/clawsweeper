## Summary

Current main, the latest release tag in this checkout, and the Telegram docs/tests all show an existing supported way to disable mention gating in groups: `channels.telegram.groups.<chatId>.requireMention: false` (or `"*"`). The report's `mentionOnly` key is not the supported config surface, and I could not verify the claim that there is "no way" to disable mention filtering.

## Evidence

- **Telegram group schema accepts `requireMention`:** The released Telegram group config schema already includes `requireMention?: boolean` under `channels.telegram.groups.*`, which is the supported knob for disabling mention gating.
  - file: [src/config/zod-schema.providers-core.ts:133](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/config/zod-schema.providers-core.ts#L133)
  - command: `git show v2026.4.23:src/config/zod-schema.providers-core.ts | nl -ba | sed -n '120,245p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Public docs already show the persistent config:** The Telegram docs include a persistent config example with `channels.telegram.groups."*".requireMention: false`, and also a per-group example with `groupPolicy: "open"` plus `requireMention: false`.
  - file: [docs/channels/telegram.md:229](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/docs/channels/telegram.md#L229)
  - command: `git show v2026.4.23:docs/channels/telegram.md | nl -ba | sed -n '212,236p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Released test covers unmentioned group replies:** The Telegram bot test case named `allows all group messages when groupPolicy is 'open'` uses `groups: { "*": { requireMention: false } }` and expects `expectedReplyCount: 1` for a plain `hello` group message.
  - file: [extensions/telegram/src/bot.create-telegram-bot.test.ts:1600](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/telegram/src/bot.create-telegram-bot.test.ts#L1600)
  - command: `git show v2026.4.23:extensions/telegram/src/bot.create-telegram-bot.test.ts | nl -ba | sed -n '1598,1615p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Runtime only skips for `no-mention` when `requireMention` is true:** Current main computes mention policy from group/topic config and only logs `reason: "no-mention"` when `isGroup && requireMention && ...shouldSkip` is true, so `requireMention: false` is the bypass path in code.

That supports a discussion about defaults, but not the narrower claim that there is no existing config to disable mention filtering.
