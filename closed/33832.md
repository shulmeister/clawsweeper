## Summary

Keep #33832 open. Current main has sanitized model-call lifecycle diagnostics plus duration-only OTEL/Prometheus telemetry, but it still does not log request body size, response body size, or time-to-first-byte for Gateway model API calls. The linked implementation attempt, PR #35513, was closed unmerged.

## Best Possible Solution

Keep #33832 open until OpenClaw core adds privacy-safe model API request/response byte counts plus first-byte and total latency logging or telemetry, likely near the provider transport or existing model-call diagnostic wrapper. Add focused tests proving the fields are emitted without raw model content or secrets. If maintainers decide duration-only diagnostics/metrics are the intended replacement, document that product decision before closing.

## Risks / Open Questions

- Any implementation should preserve the current privacy boundary: byte counts and timing are useful, but raw prompts, responses, request bodies, headers, provider request IDs, tokens, and secrets should not be logged.
- Accurate time-to-first-byte likely belongs at the provider transport or stream-iteration boundary; the current diagnostic wrapper only records start and terminal completion/error duration.
