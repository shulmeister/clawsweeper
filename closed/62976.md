## Summary

Keep this issue open. Current main still hard-fails startup for plugin-local invalid config, deliberately skips last-known-good recovery for plugin-entry-only validation failures, and doctor repair still only removes stale/unknown plugin references or bundled load-path aliases rather than quarantining or disabling a known bad third-party plugin.

## Best Possible Solution

Keep this issue open and fix it in core. Add a narrow plugin-local recovery path that keeps the gateway bootable by quarantining or disabling only the bad third-party plugin entry, then extend `openclaw doctor --fix` or a plugin quarantine command to repair or quarantine `plugins.entries`, `plugins.allow`, and related `plugins.load.paths` deterministically. Coordinate with #70371/#70394 if that work lands, but do not close this issue until startup degradation and doctor/safe-mode recovery are covered by source and tests.
