## Summary

Current main and the released `v2026.4.23` code already handle `peerDependencies.openclaw` by replacing any plugin-local `node_modules/openclaw` entry with a symlink to the host OpenClaw package after install/update, with tests covering install and update behavior.

## Evidence

- **Installer rewrites plugin-local `openclaw` peer to host package:** `linkOpenClawPeerDependencies(...)` filters for the `openclaw` peer, removes any existing `node_modules/openclaw` entry with `fs.rm(..., { recursive: true, force: true })`, and recreates it as a symlink/junction to the resolved host package root.
  - file: [src/plugins/install.ts:617](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/plugins/install.ts#L617)
  - command: `nl -ba src/plugins/install.ts | sed -n '613,655p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Post-install path always applies the rewrite after dependency install/update:** `installPluginFromPackageDir(...)` captures `manifest.peerDependencies`, runs the installed dependency-tree scan, then calls `linkOpenClawPeerDependencies(...)` in `afterInstall`, so install/update flows normalize the peer after npm materialization.
  - file: [src/plugins/install.ts:790](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/plugins/install.ts#L790)
  - command: `nl -ba src/plugins/install.ts | sed -n '790,845p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Core install helper intentionally does not use `--omit=peer`:** `install-package-dir.ts` documents that omitting peer deps can strip runtime requirements, so the chosen fix is the post-install host symlink normalization rather than `npm install --omit=peer`.
  - file: [src/infra/install-package-dir.ts:246](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/infra/install-package-dir.ts#L246)
  - command: `nl -ba src/infra/install-package-dir.ts | sed -n '243,249p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Tests cover install and update normalization:** Tests assert that declaring `peerDependencies.openclaw` creates `node_modules/openclaw` as a symlink to the host package, that no link is created without the peer, and that update-mode reinstall keeps the symlinked result.
  - file: [src/plugins/install.test.ts:2380](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/plugins/install.test.ts#L2380)
