## Summary

Keep PR #42319 open. Current main has separate Google Chat auth hardening, but it does not contain the PR’s health runtime-snapshot wiring, and the PR’s own health refactor still has a concrete probe-dropping regression called out by two review comments. This is not a safe conservative cleanup close as implemented or superseded.

## Best Possible Solution

Keep the PR open or ask the contributor/maintainer to rebase it onto current main, split the Google Chat auth and health-runtime changes if both are still desired, preserve probe/audit fields in the shared snapshot helper or immediately after delegation, and add a focused health snapshot test that proves `running`/`lastStartAt` reach the `health` command without dropping existing probe output.
