## Summary

Current main already implements the OpenRouter chat-path attribution behavior through the bundled OpenRouter provider stream wrapper. The PR’s production diff is not needed for OpenRouter attribution: OpenRouter headers are defined in the provider attribution policy, the OpenRouter stream family wraps calls with `createOpenRouterWrapper`, and regression coverage asserts the headers reach stream options. The Greptile comment usefully identified that the PR’s test mock cleanup is a valid alignment, but that remaining value is test-only and does not change the shipped OpenRouter request behavior.

## Best Possible Solution

Do not merge this PR as-is. Keep the shipped route-aware OpenRouter attribution path through the bundled provider wrapper. If maintainers want the test mock alignment, land it separately as a narrow test cleanup; if someone believes an extra-params post-wrapper fallback is still missing, it should come with a current-main failing regression test that demonstrates the missing shipped behavior.
