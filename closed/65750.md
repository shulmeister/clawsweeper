## Summary

Keep this PR open. Current main still contains the clear-path history reload that the PR removes, and `loadChatHistory()` can still write returned history back into `chatMessages` after the local clear. The provided PR/issue context is a focused implementation candidate, not cleanup-close material.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best path is to remove the immediate post-reset `loadChatHistory()` call from the active `/clear` implementation, add a regression test where `chat.history` returns stale messages after `sessions.reset`, and decide whether the render-level `onClearHistory` callback should receive the same fix or be deleted if unused.
