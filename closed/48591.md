## Summary

Current `main` already implements source-based grouping for the Skills UI, and the shipped `v2026.4.22` release contains that implementation.

## Evidence

- **Source-group definitions exist in current UI:** `ui/src/ui/views/skills-grouping.ts:9` defines separate groups for `openclaw-workspace`, `openclaw-bundled`, `openclaw-managed`, and `openclaw-extra`, labeled `Workspace Skills`, `Built-in Skills`, `Installed Skills`, and `Extra Skills`.
  - file: [ui/src/ui/views/skills-grouping.ts:9](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/ui/src/ui/views/skills-grouping.ts#L9)
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)

  - file: [ui/src/ui/views/skills.ts:126](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/ui/src/ui/views/skills.ts#L126)

  - file: [ui/src/ui/views/agents-panels-tools-skills.ts:420](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/ui/src/ui/views/agents-panels-tools-skills.ts#L420)

## Risks / Open Questions

- The UI label for `openclaw-managed` is `Installed Skills`, not the issue's exact `Managed/Local` wording, but it still provides the requested source distinction.
- Skills from newer source ids like `agents-skills-personal` or `agents-skills-project` currently fall outside the four named buckets and would land in `Other Skills`; that is separate from this issue's requested bundled/workspace/managed/extra split.
