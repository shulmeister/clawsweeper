## Summary

Keep PR #69948 open. Current main has the underlying Gateway `sessions.send` behavior, but it does not expose the proposed plugin-facing `api.runtime.interSession.send` runtime namespace, types, docs, gateway binding, or test helper support. The PR is a plausible core/plugin-SDK seam rather than ClawHub-only work, and the supplied review context shows useful focused follow-up plus one remaining cache-mode concern to resolve before maintainer merge judgment.

## Best Possible Solution

Continue PR review on #69948 rather than closing it. The best path is a narrow, documented plugin runtime seam that forwards to existing `sessions.send`, updates the public `PluginRuntime` contract and API baseline, binds the Gateway runtime during plugin bootstrap, scopes calls by plugin id, maps stable errors, and includes tests for runtime availability, Gateway forwarding/idempotency, plugin-id scoping, and plugin-loader cache compatibility.
