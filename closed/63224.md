## Summary

#63224 is a newer duplicate of the already-open #59088. Both request a native Control UI workspace tree/browser to avoid SSH/manual file retrieval; #63224 adds media preview/download details, which should be folded into the canonical workspace browser/editor issue. Current main still only has the narrower core/bootstrap Files editor and explicit assistant-media attachment route, not a general workspace explorer.

## Best Possible Solution

Close #63224 as a duplicate of open #59088. Keep #59088 as the canonical Control UI workspace browser/editor issue, and when implementing it, include #63224's generated-media requirements: safe workspace tree listing, read-only/download paths, image/video previews, and text/code viewing/editing behind explicit gateway authorization and root-bound file access.
