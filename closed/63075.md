## Summary

Current main already clears Discord native slash commands when `channels.discord.commands.native` or fallback global `commands.native` is explicitly `false`. The Discord monitor resolves the native-command setting, detects explicit disable, skips deployment, then sends an empty application-command overwrite to Discord. The same implementation is present in the v2026.4.24 release tree, and the public docs now state that disabling native commands clears previously registered Discord commands.

## Best Possible Solution

Close as implemented. Keep the current Discord startup behavior that clears native commands on explicit disable; if a future report shows Discord still serving commands after the empty overwrite succeeds, handle it as a fresh Discord reconciliation/client-cache repro with API logs.
