## Summary

Current `main` already normalizes workspace-relative agent avatars into renderable Control UI avatar URLs/data before those values reach the frontend helpers. The specific raw-path inconsistency described in #70700 is implemented on `main` by commit `e8191e5b8f65f0e0544182e5ce193e096aa3354c`, but I could not tie that commit to a released tag yet.

## Evidence

## Risks / Open Questions

- There appears to be a separate authenticated-avatar/CSP problem on current `main`: `ui/src/ui/app-chat.ts:670` creates `blob:` URLs for authenticated avatar fetches, while `src/gateway/control-ui-csp.ts:47` still limits `img-src` to `'self' data:`. That aligns with the Apr 24 comment, but it is adjacent to rather than the same as the raw workspace-relative avatar normalization issue in #70700.
