## Summary

Current `main` already preloads OpenRouter capabilities for unknown dynamic models and derives `maxTokens` from the OpenRouter catalog, covering the reported silent `8192` fallback on chat/cron async paths. The fix appears to be on `main` only and not yet in the latest tagged release.

## Evidence

- **OpenRouter dynamic-model preload:** The bundled OpenRouter provider now wires `prepareDynamicModel` to `loadOpenRouterModelCapabilities(...)` before returning dynamic model metadata for unknown model ids.
  - file: [extensions/openrouter/index.ts:122](https://github.com/openclaw/openclaw/blob/15d27d1527edafe2a45e14645d93e0abe5300e80/extensions/openrouter/index.ts#L122)
  - sha: [15d27d1527ed](https://github.com/openclaw/openclaw/commit/15d27d1527edafe2a45e14645d93e0abe5300e80)
- **Catalog token limits are parsed:** OpenRouter capability parsing now prefers `top_provider.max_completion_tokens`, then `max_completion_tokens`, then `max_output_tokens`, and only falls back to `8192` if the catalog itself does not expose a limit.
  - file: [src/agents/pi-embedded-runner/openrouter-model-capabilities.ts:168](https://github.com/openclaw/openclaw/blob/15d27d1527edafe2a45e14645d93e0abe5300e80/src/agents/pi-embedded-runner/openrouter-model-capabilities.ts#L168)
  - sha: [15d27d1527ed](https://github.com/openclaw/openclaw/commit/15d27d1527edafe2a45e14645d93e0abe5300e80)
- **Async runner path waits for preload:** `resolveModelAsync(...)` awaits `prepareProviderDynamicModel(...)` and then re-runs model resolution, which is the path used by the PI runner for agent/chat execution.
  - file: [src/agents/pi-embedded-runner/model.ts:814](https://github.com/openclaw/openclaw/blob/bf0221c5b3de3f290a934a314814426c05b59b84/src/agents/pi-embedded-runner/model.ts#L814)
  - sha: [bf0221c5b3de](https://github.com/openclaw/openclaw/commit/bf0221c5b3de3f290a934a314814426c05b59b84)
- **Regression test for unknown OpenRouter models:** A model-resolution test now covers an unknown OpenRouter model, verifies the preload hook ran, and expects resolved metadata including `reasoning`, image input, and `maxTokens: 65536` instead of the old fallback.
  - file: [src/agents/pi-embedded-runner/model.test.ts:647](https://github.com/openclaw/openclaw/blob/bf0221c5b3de3f290a934a314814426c05b59b84/src/agents/pi-embedded-runner/model.test.ts#L647)
  - sha: [bf0221c5b3de](https://github.com/openclaw/openclaw/commit/bf0221c5b3de3f290a934a314814426c05b59b84)
- **Regression test for OpenRouter token fields:** Capability tests explicitly verify both top-level `max_completion_tokens` and `max_output_tokens` from the OpenRouter `/models` payload become `maxTokens` in the cached capabilities.
  - file: [src/agents/pi-embedded-runner/openrouter-model-capabilities.test.ts:40](https://github.com/openclaw/openclaw/blob/15d27d1527edafe2a45e14645d93e0abe5300e80/src/agents/pi-embedded-runner/openrouter-model-capabilities.test.ts#L40)
  - sha: [15d27d1527ed](https://github.com/openclaw/openclaw/commit/15d27d1527edafe2a45e14645d93e0abe5300e80)
- **Not in the latest release tag:** `git tag --contains 15d27d1527edafe2a45e14645d93e0abe5300e80` returned no tags, and `git merge-base --is-ancestor 15d27d1527edafe2a45e14645d93e0abe5300e80 v2026.4.23` exited non-zero, so this fix is on `main` but not in `v2026.4.23`.
