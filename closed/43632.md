## Summary

PR #43632 is superseded by the newer Claude CLI external-auth work on current main, especially the #70902 implementation recorded in the 2026.4.25 unreleased changelog. Current main registers Claude CLI as a managed external CLI auth source for the dedicated `anthropic:claude-cli` profile, normalizes it to provider `claude-cli`, and the OAuth manager can adopt the external credential before spending a copied refresh token. That addresses the central stale Claude CLI OAuth problem without merging this older PR's broader Anthropic-profile mutation and Claude CLI write-back approach, which the PR discussion already flagged as risky for multi-profile/account setups.

## Best Possible Solution

Close #43632 as superseded by the current #70902-era Claude CLI external-auth implementation. Keep the mainline design scoped to the dedicated `anthropic:claude-cli` profile and `claude-cli` provider, with identity/provider gates and runtime-only external auth overlays; any remaining two-way Claude CLI storage mutation should be proposed separately with maintainer product review.
