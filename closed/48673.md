## Summary

Keep open. Current main still emits the duplicate-plugin warning for the exact installed-global-over-bundled case this PR targets, and tests intentionally assert that warning today. The PR discussion also identified a real implementation constraint: any warning-suppression fix must preserve duplicate-state consumers in loader and config validation.

## Best Possible Solution

Keep the PR open for maintainer review or author revision. The best path is a narrow suppression design for intentional installed-plugin overrides that preserves duplicate bookkeeping used by loader and config validation, updates the manifest-registry tests for both candidate orderings, and clarifies whether the public duplicate-id docs should still say these drops are logged.
