## Summary

Current main already prevents the reported silent partial-index/no-op path. Embedding 5xx responses are retried, non-recoverable embedding failures are thrown before a file is stamped indexed, missing file rows are picked up by the next normal `openclaw memory index`, and full reindex swaps in a temp DB only after a successful build. The related #26772 checkpoint/resume issue remains separate and open.

## Best Possible Solution

Close this issue as implemented on current main and shipped in `v2026.4.24`; leave #26772 open for the broader checkpoint/resume reliability work, and open a narrower follow-up only if maintainers want stricter `memory status` health semantics around indexed/total mismatches.

## Evidence
