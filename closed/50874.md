## Summary

Current `main` already implements the requested ability to suppress Slack implicit thread mentions via `channels.slack.thread.requireExplicitMention`, with schema/docs/runtime wiring and focused tests. This PR's `requireMentionInThreads` field would add an overlapping config surface rather than restore missing behavior.

## Evidence
