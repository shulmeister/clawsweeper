## Summary

Current main implements the requested observability surface with typed plugin hooks for model calls/LLM output/agent end and subagent spawn/end lifecycle events. The remaining richer post-completion announcement customization is separately represented by related issue #22358, but the dashboard/cost/delegation tracking use case in #8354 is covered well enough on main.

## Risks / Open Questions

- These are typed plugin hooks rather than legacy internal `HOOK.md` events, so consumers should build an OpenClaw plugin for real-time dashboards.
- Subagent completion hooks do not directly include cost; consumers can correlate `runId`/session keys with `llm_output` usage and apply their own pricing table.
