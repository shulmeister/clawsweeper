## Summary

Keep open. Current main still has the catalog-helper behavior this PR changes: a bundled plugin found by pluginId is rejected when its bundled npmSpec differs from the catalog npmSpec, even when the catalog npmSpec is just the bare pluginId. The direct `openclaw plugins install acpx` path now has a separate bundled pre-NPM resolution path, so the PR may need retargeting in wording, but the stated catalog/onboarding helper fix is not implemented on main.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to either merge/port the small `npmSpec === pluginId` guard plus regression test for `resolveBundledInstallPlanForCatalogEntry`, or explicitly retarget/close it only after a maintainer decides catalog entries with bare npmSpec should remain unsupported despite the bundled pluginId match.
