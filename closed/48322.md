## Summary

Current `main` already contains Telegram preview-streaming logic that prevents the reported 1-2 character partial preview from being sent, and its tests cover short-sentence final delivery as a single message. That makes the reported `[[reply_to_current]]` workaround obsolete on current `main`.

## Evidence

- **Telegram preview now debounces the first partial send:** `DRAFT_MIN_INITIAL_CHARS` is set to 30 and passed into the Telegram draft stream, so Telegram no longer sends an initial 1-2 character preview message before the final text is ready.
  - file: [extensions/telegram/src/bot-message-dispatch.ts:87](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/extensions/telegram/src/bot-message-dispatch.ts#L87)
  - command: `nl -ba extensions/telegram/src/bot-message-dispatch.ts | sed -n '84,92p'; nl -ba extensions/telegram/src/bot-message-dispatch.ts | sed -n '338,350p'`
- **Draft stream refuses undersized first previews:** The draft stream exits early when there is no existing preview message and the rendered text is shorter than `minInitialChars`, which directly blocks the short partial preview pattern described in the issue.
  - file: [extensions/telegram/src/draft-stream.ts:318](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/extensions/telegram/src/draft-stream.ts#L318)
  - command: `nl -ba extensions/telegram/src/draft-stream.ts | sed -n '316,323p'`
- **Regression tests cover short final sentences and sub-threshold partials:** Tests assert that short partials do not send before the threshold, and that `stop()` still sends a short final sentence immediately as one message. Together these cover the reported 'first 1-2 chars break onto their own lines' mechanism.
  - file: [extensions/telegram/src/draft-stream.test.ts:610](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/extensions/telegram/src/draft-stream.test.ts#L610)
  - command: `nl -ba extensions/telegram/src/draft-stream.test.ts | sed -n '598,636p'`
- **Telegram docs describe finalizing text previews by editing the same message in place:** The docs for current `main` say text-only Telegram replies keep the same preview message and perform a final edit in place, matching the current implementation rather than a split multi-message flow.
  - file: [docs/channels/telegram.md:279](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/docs/channels/telegram.md#L279)
