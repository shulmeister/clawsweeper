## Summary

Current `main` already handles the missing `qa/` scenario-pack case without crashing: `readQaScenarioPack()` returns an empty pack when `qa/scenarios/index.md` is absent, `readQaScenarioExecutionConfig()` therefore returns `undefined`, and `discovery-eval.ts` already falls back to hardcoded refs at module load. This PR's intended fix is already present on `main` through a different implementation.

## Evidence

- **Missing-pack fallback already exists:** `readQaScenarioPack()` explicitly treats an absent `qa/scenarios/index.md` as an optional npm-distribution case and returns an empty pack instead of throwing, with an inline comment saying this avoids crashes for completion-cache updates and other consumers when `qa/scenarios/` is not shipped.
