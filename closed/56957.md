## Summary

Current main has not shipped the fix yet: WhatsApp inbound policy still falls back from `groupAllowFrom` to configured DM `allowFrom`, which can make `hasGroupAllowFrom` true and bypass the empty group allowlist. The remaining work is now tracked directly by open PR #68446, which supersedes earlier closed attempts #56978 and #60660 and targets the current `extensions/whatsapp/src/inbound-policy.ts` path identified in the latest issue comment.

## Best Possible Solution

Use #68446 as the canonical tracking and implementation path: remove the WhatsApp DM `allowFrom` fallback from `groupAllowFrom` resolution, add/keep regression coverage for `groupPolicy: "allowlist"` with DM `allowFrom` but no groups/groupAllowFrom, and align the WhatsApp group docs with the final contract.
