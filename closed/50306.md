## Summary

Current main addresses the reported LINE webhook disappearing/404 behavior. LINE still registers account-scoped webhook paths dynamically, and the Gateway now pins plugin HTTP routes to the startup registry while dispatch/auth resolve the same live HTTP-route registry, so later plugin-registry churn no longer drops channel webhook handlers until a save/reload. Regression tests cover the exact stale/changed-registry route-dispatch case, and the changelog records the webhook-route pinning fix in 2026.3.22.

## Best Possible Solution

Close as implemented and keep the current Gateway plugin HTTP-route pinning plus LINE account webhook registration. If a user on 2026.3.22 or newer still sees LINE Developer Verify return 404 with a stable public URL, that should be a fresh issue with gateway logs, LINE account/webhookPath config, tunnel uptime, and the exact OpenClaw version.
