## Summary

PR #65800 should remain open. Current main still lacks a wiki command-path policy, so `openclaw wiki ...` inherits the default `loadPlugins: "never"`; the memory-wiki status path still depends on active memory public-artifact state that lazy CLI registration does not preserve.

## Best Possible Solution

Keep PR #65800 open or reconcile it explicitly with the broader related open PR #68041. The remaining product/code path is to add a `wiki` command policy that loads runtime plugins, keep the startup-policy tests for text and JSON wiki invocations, and keep the memory-wiki healthy bridge regression test so `bridge-artifacts-missing` is not emitted when public artifacts are available.
