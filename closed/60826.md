## Summary

Keep open. Current main still writes raw `process.argv.slice(0, 8)` into config audit records on the write and observe paths, and the audit append helper serializes the record directly without applying the existing logging redaction sink. The related fix PRs #60871 and #70468 were closed unmerged, and their proposed redaction helper/tests are absent from the checked-out main.

## Best Possible Solution

Implement a centralized config-audit process-info helper that either omits argv or redacts it before every audit record is built. It should cover write, observe, and recovery observe paths, reuse the existing logging redaction patterns where appropriate, add config-audit-specific flag coverage for `--flag value` and `--flag=value`, and include regression tests in `src/config/io.audit.test.ts` or adjacent config observe tests.
