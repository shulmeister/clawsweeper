## Summary

Current main still persists unversioned CLI ClawHub installs as resolved versioned specs, and the update path reuses that persisted spec. PR #58426 remains a focused core CLI bug fix, though the latest PR head also includes an unrelated doctor test guard change that should be split or justified before merge.

## Best Possible Solution

Keep PR #58426 open. The best path is to merge or adapt the scoped CLI fix on main: persist `raw` for direct `clawhub:` installs, persist `preferredClawHubSpec` for bare-name ClawHub fallback, keep explicit `clawhub:pkg@version` installs pinned, remove the stale `formatClawHubSpecifier` CLI test-helper mock, and drop or split the unrelated doctor test guard change unless maintainers confirm it belongs in this PR. Validate with the targeted CLI install/update tests and build in a writable maintainer pass.
