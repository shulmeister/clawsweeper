## Summary

Current `main` already clears expired persisted auth-profile cooldown state before auth-profile selection, so stale backoff data no longer biases the first post-restart request. The fix is on `main` in commit `0bd8d0bba08b386ca975510235e00df2139b3e70`, but I could not confirm a released tag containing it from this checkout.

## Evidence

- **Expired cooldowns are cleared before auth-profile ordering:** `resolveAuthProfileOrder()` now calls `clearExpiredCooldowns(store, now)` before any ordering or cooldown filtering, so persisted expired cooldown state is removed before selection.
  - file: [src/agents/auth-profiles/order.ts:80](https://github.com/openclaw/openclaw/blob/0bd8d0bba08b386ca975510235e00df2139b3e70/src/agents/auth-profiles/order.ts#L80)
  - sha: [0bd8d0bba08b](https://github.com/openclaw/openclaw/commit/0bd8d0bba08b386ca975510235e00df2139b3e70)
- **Expired cooldown cleanup resets stale backoff state:** `clearExpiredCooldowns()` removes expired `cooldownUntil` / `disabledUntil` markers and resets `errorCount` plus `failureCounts` once no unusable window remains, preventing stale persisted failure state from biasing the next attempt.
  - file: [src/agents/auth-profiles/usage-state.ts:133](https://github.com/openclaw/openclaw/blob/0bd8d0bba08b386ca975510235e00df2139b3e70/src/agents/auth-profiles/usage-state.ts#L133)
