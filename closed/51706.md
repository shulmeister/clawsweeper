## Summary

Keep open. Current `main` has the runtime-model display change, but `session_status` still routes the runtime model through the selected-model path for sessions without explicit overrides. That means active fallback or channel-routed sessions can still lose the `↪️ Fallback:` line and show the wrong `🧠 Model:` value, which matches the unresolved review concern on this PR.

## Evidence

- **Session status still promotes runtime identity into the selected model slot:** `createSessionStatusTool` resolves `runtimeModelIdentity`, then for sessions without explicit overrides sets `defaultProviderForCard`/`defaultModelForCard` from that runtime identity and passes them into `buildStatusText` as both `provider`/`model` and `primaryModelLabelOverride`. That collapses selected and active model identity before rendering.
  - file: [src/agents/tools/session-status-tool.ts:510](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/agents/tools/session-status-tool.ts#L510)
  - command: `nl -ba src/agents/tools/session-status-tool.ts | sed -n '510,560p'`
  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **Fallback rendering requires selected and active models to differ:** `resolveActiveFallbackState` only marks fallback active when `selectedModelRef !== activeModelRef` and both stored fallback refs exactly match those values. If `session_status` feeds the runtime model in as the selected model, the fallback line is suppressed by construction.
  - file: [src/status/fallback-notice-state.ts:9](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/status/fallback-notice-state.ts#L9)

- **Status message uses incoming provider/model as the selected model:** `buildStatusMessage` derives `selectedProvider`/`selectedModel` from the incoming args and uses that selected ref for both the `🧠 Model:` line and fallback detection. The current status rendering path has no separate display-only runtime-model input to preserve selected-vs-active semantics.
