## Summary

Keep open. Current main still has the strict-mode behavior this PR targets: unclosed reasoning tags suppress trailing text, and existing tests still assert that strict mode drops the unclosed tail. The PR is focused on a real open bug linked to #37696, but it should not be treated as implemented or superseded because the fix is not on main and review identified a preserve-mode fallback regression that needs revision before merge.

## Best Possible Solution

Keep this PR or its linked bug open, but revise the implementation before merge: preserve strict-mode recovery for unclosed tags, keep preserve-mode closed-block semantics unchanged, and make any empty-result fallback code-aware and narrowly gated. Add regression tests for strict unclosed recovery, preserve-mode closed blocks, code-region literals, and representative channel paths such as Telegram/Discord partial delivery.
