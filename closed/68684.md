## Summary

Keep #68684 open. Current main still lacks the Docker host-gateway mapping and Docker-specific LM Studio/Ollama guidance requested by the issue; the provider onboarding defaults remain loopback-oriented. The linked PR #68702 directly tracks a plausible fix, but the provided context says it is still open/unmerged and its changes are not present in this checkout.

## Best Possible Solution

Keep the issue open until PR #68702 or an equivalent maintainer-reviewed change lands. The best fix is to add `extra_hosts: ["host.docker.internal:host-gateway"]` to the bundled Docker gateway service, document Docker host-service URLs for LM Studio and Ollama on the public Docker install page, and consider a Docker-aware hint in the LM Studio/Ollama onboarding prompts.
