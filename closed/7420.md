## Summary

Current main treats this class of lossless-context / query-delegation work as plugin-owned context-engine behavior, and the project vision explicitly keeps optional heavy orchestration out of OpenClaw core. This is better tracked as a ClawHub/plugin effort, not a core issue.

The reason is scope, not merit. Current project direction says optional capability should usually ship as plugins and stay outside core (`VISION.md:54`), and it explicitly calls out agent-hierarchy frameworks and heavy orchestration layers as things OpenClaw is not merging into core by default (`VISION.md:99`). Current main also already provides a dedicated extension path for this exact class of work: context engine plugins own ingest/assembly/compaction and can either own compaction themselves or delegate it back to runtime (`docs/plugins/architecture.md:1365`, `src/plugins/types.ts:2005`, `src/plugin-sdk/index.ts:116`, `src/config/schema.help.ts:1099`). OpenClaw’s docs further say external plugins should be published through ClawHub instead of being added to the repo (`docs/plugins/building-plugins.md:16`, `docs/plugins/building-plugins.md:125`, `docs/cli/plugins.md:56`).

Given that, this proposal belongs in the ClawHub/plugin lane rather than as a core OpenClaw issue. If you run into a concrete missing SDK seam while building this pattern, please open a focused follow-up for that specific contract gap.
