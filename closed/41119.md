## Summary

Current `main` already deduplicates repeated tool-call IDs with a separate `assignedIds` pass, preserves the first trimmed ID, and ships regression coverage for the duplicate-`edit:22` case. This PR’s proposed `usedIds.has(trimmedId)` approach matches the regression called out in review comments, so the branch is obsolete and should close as already implemented on `main`/released.
