## Summary

Current OpenClaw already supports disabling heartbeat via config with `agents.defaults.heartbeat.every: "0m"` (or the per-agent equivalent). The issue's requested capability is present in shipped docs, runtime logic, and tests; only the proposed boolean shorthand (`"heartbeat": false`) remains unsupported.

## Evidence

- **Issue discussion already identified the supported config:** The existing comment on #70668 points to `0m` and the heartbeat docs. That matches current OpenClaw behavior and directly addresses the issue's stated need to disable heartbeat via config.
- **Public heartbeat docs document config-based disable:** The shipped heartbeat docs say to set `agents.defaults.heartbeat.every` or `agents.list[].heartbeat.every`, and explicitly state: `use 0m to disable`.
  - file: [docs/gateway/heartbeat.md:53](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/docs/gateway/heartbeat.md#L53)

- **Runtime treats non-positive heartbeat intervals as disabled:** `resolveHeartbeatIntervalMs(...)` parses the configured duration and returns `null` when `ms <= 0`, which is the disable path for `0m`.
  - file: [src/infra/heartbeat-summary.ts:44](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/infra/heartbeat-summary.ts#L44)

  - file: [src/infra/heartbeat-runner.ts:1405](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/infra/heartbeat-runner.ts#L1405)
