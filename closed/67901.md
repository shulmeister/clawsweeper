## Summary

Keep open. Current main still has the reported failure shape: `openclaw subagents` is not registered as a core/sub-CLI command, and the missing-command path can still turn an absent primary command into the `plugins.allow excludes "subagents"` bundled-plugin error. Docs describe `/subagents` as a runtime slash command, not a CLI command, and there is no targeted `subagents` regression coverage.

## Best Possible Solution

Keep the issue open. The likely fix is to either add and document a real `openclaw subagents` CLI command, or more conservatively teach the missing-command guidance that `subagents` is a runtime slash command (`/subagents`) and not a plugin. In either case, restrict `plugins.allow` diagnostics to actual plugin IDs or manifest-backed aliases and add focused tests for `openclaw subagents list` under restrictive `plugins.allow`. Related #64732 can inform the generic missing-command guard, but this subagents-specific repro still needs explicit coverage or a clear maintainer product decision.
