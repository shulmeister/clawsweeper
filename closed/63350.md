## Summary

Issue #63350 is best closed as `clawhub`. Current main already exposes the generic Gateway/node/plugin surfaces Clicky would consume, while project vision and docs explicitly direct optional companion integrations, skills, and community-maintained plugins to ClawHub or external plugin repos instead of OpenClaw core.

- A Clicky plugin/client could still uncover small additive protocol or SDK gaps later; those gaps would merit separate focused issues rather than keeping this broad core-feature request open.
