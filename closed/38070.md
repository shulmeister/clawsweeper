## Summary

Agent Pack is a useful optional workspace-template/distribution idea, but it fits the existing ClawHub/native plugin path rather than OpenClaw core. Current main has no pack command or PACK.md format, and the PR adds a new core filesystem-write surface with unresolved path traversal and YAML escaping blockers.

## Best Possible Solution

Close this PR and move Agent Pack work to a ClawHub/native plugin implementation that owns the PACK.md format, CLI, install/export workflow, and safety checks. If an external implementation finds a concrete missing SDK seam, open a narrow plugin API issue or PR for that seam instead of adding the whole workspace-template system to core.
