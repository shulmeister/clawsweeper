## Summary

Keep #69206 open. Current main still generates and registers dock commands, and Telegram native command handling now routes native commands into the normal command pipeline, but I found no current handler that performs the requested dock/channel handoff for `dock:discord` or `/dock_discord`. The related closed #45596 may cover generic Telegram native dispatch, but it does not cover this dock-specific missing behavior.

## Best Possible Solution

Keep the issue open and implement a dock-command handler in the command pipeline or an equivalent generic channel handoff seam. The handler should resolve `dock:<channel>` commands from generated chat/native command metadata, update the target session delivery route or binding for the originating conversation, and add regression coverage for Telegram `/dock_discord` so it does not fall through to the model. The related generic Telegram command fallback issue #45596 should not be treated as the canonical fix for this dock-specific behavior.
