## Summary

Current main already exposes the requested config surface for `claude-cli`: users can override the bundled Claude CLI invocation with `agents.defaults.cliBackends.claude-cli.args` and matching `resumeArgs`, which is enough to pass `--bare` or equivalent flags without a core code change.

## Evidence

- **Public config shape already exposes raw CLI args:** `CliBackendConfig` includes public `args` and `resumeArgs` fields, so `claude-cli` already has a supported config surface for replacing the bundled command-line flags.
  - file: [src/config/types.agent-defaults.ts:93](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/config/types.agent-defaults.ts#L93)

  - file: [src/agents/cli-backends.ts:135](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/agents/cli-backends.ts#L135)

## Risks / Open Questions

- There is no dedicated `slimMode` or `--bare` preset on current main; users must supply full replacement `args` and `resumeArgs` themselves.
- The bundled default still uses the full Claude CLI path, so this close only covers the requested config surface, not a default-behavior change.
