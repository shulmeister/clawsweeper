## Summary

Keep open. Current main still allows the reported Control UI Agent Context regression: `agent.identity.get` resolves the selected agent through `resolveAssistantIdentity`, but that resolver still prefers global `ui.assistant` name/avatar before `agents.list[].identity` for every agent. The Agent Context panel then prefers that runtime identity over the per-agent list payload, and its avatar row still shows only `custom` for renderable image/avatar URLs or `—`, not the configured emoji. The related fix PR #54730 is closed unmerged, and #72516 is related but tracks the Chat identity race surface rather than this Agents management panel bug.

## Best Possible Solution

Keep this issue open. Land a focused fix that makes non-default agents prefer `agents.list[].identity` name/avatar/emoji over global `ui.assistant`, while preserving default-agent behavior. Update Agent Context so emoji/avatar state is shown directly instead of only `custom` or `—`, and add regression tests for default global priority, subagent per-agent name, subagent emoji/avatar, missing identity fallback, and partial fallback.
