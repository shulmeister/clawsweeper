## Summary

Keep #66515 open. Current main does not implement Yandex/YandexGPT, and this PR is the active replacement for closed PR #59768 with substantial review discussion and author fixes. Although the provider-plugin and ClawHub surfaces are sufficient for an external Yandex provider, OpenClaw’s vision also lists major model-provider support as a product priority and current docs show many core-shipped model providers, so an automated cleanup close would require maintainer product judgment rather than a high-confidence close reason.

## Best Possible Solution

Keep the PR open for maintainer direction. If OpenClaw wants YandexGPT bundled, the PR should be rebased onto current main, drop any already-main sync noise from the diff, resolve the remaining P2 onboarding/config-preservation review comments, and avoid provider-specific SDK exports unless a real shared contract requires them. If maintainers decide Yandex should not be core-shipped, ask the author to publish the provider as a ClawHub/npm plugin using the current provider-plugin SDK and then close with that explicit direction.
