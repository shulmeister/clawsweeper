## Summary

Current `main` already exposes the ACP control-plane capability plugins need through the public `openclaw/plugin-sdk/acp-runtime` SDK surface, so plugins can initialize ACP sessions and send follow-up turns without forking core.

## Risks / Open Questions

- The exact API shape differs from the issue proposal: plugins use `openclaw/plugin-sdk/acp-runtime` instead of injected `api.runtime.acp.*`, so downstream callers need to target the public SDK subpath.
