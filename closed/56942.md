## Summary

#56942 reports the Feishu `botName` per-account config being rejected after the strict schema change. Current main still uses the strict `name` field and does not include a doctor migration for legacy `botName`, so the remaining actionable work is not fixed here. The current, narrower canonical issue is #65177, which tracks the same Feishu legacy `botName` to `name` migration gap against newer releases and includes the manual workaround.

## Best Possible Solution

Close #56942 as superseded by #65177. Keep #65177 open as the canonical Feishu fix path: add a tested `openclaw doctor --fix` migration, or an explicitly accepted compatibility alias, for `channels.feishu.accounts.*.botName` to `name`, with docs/changelog aligned to the chosen compatibility behavior.
