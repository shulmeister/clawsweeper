## Summary

Current main already resolves the WebChat main-session alias mismatch at the Control UI session-default layer, using runtime snapshot defaults rather than the PR's hardcoded `main`/`agent:main:main` special case. That makes this PR obsolete and also avoids the non-default-agent/config risks raised in review.
