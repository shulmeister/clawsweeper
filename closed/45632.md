## Summary

Current `main` already handles the prompt-side failover case this PR targeted by normalizing wrapped/nested provider errors into classified failover reasons before rotation/fallback decisions, with focused tests covering the generic/abort-wrapped error shapes discussed here. The PR’s core bugfix is therefore superseded on `main`, and the rest of the branch is broader unrelated scope.

## Evidence

- **Prompt-side failover now normalizes wrapped errors before routing:** `run.ts` converts `promptError` through `coerceToFailoverError(...)`, then uses the classified reason/status for auth-profile rotation and model fallback decisions. That directly addresses the PR summary’s concern that generic outer error text could bypass prompt-side failover routing.
