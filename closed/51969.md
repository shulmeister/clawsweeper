## Summary

Keep the PR open. Current main already has some IANA timezone wording in the cron tool schema and CLI docs, but the long-form agent tool example still says only `<optional-timezone>`, so the PR is not fully redundant. The PR also needs maintainer/author cleanup before merge because its wording was challenged in review and it appears to link the unrelated #51922 build issue.

## Best Possible Solution

Keep the PR open or let a maintainer supersede it with a tiny docs patch. The best fix is to update the cron agent tool's long-form schedule example to say `schedule.tz` expects an IANA timezone identifier, add careful wording that distinguishes unsupported numeric offset strings from valid IANA identifiers such as `UTC`/`Etc/GMT+5`, and correct the PR's issue/changelog metadata before merge.
