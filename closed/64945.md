## Summary

Current main already implements the fix for this regression, and the fix was shipped in the 2026.4.11 release notes: packaged installs may still omit repo-only QA assets, but CLI startup now tolerates that and avoids crashing `openclaw completion --write-state`.

## Evidence

- **Changelog shipped fix:** The changelog explicitly lists this behavior as fixed under the `2026.4.11` release: packaged CLI startup and `openclaw completion --write-state` stop reading repo-only QA scenario markdown.
  - file: [CHANGELOG.md:1001](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/CHANGELOG.md#L1001)

- **Missing QA pack now returns empty catalog instead of throwing:** `readQaScenarioPack()` now checks for a missing `qa/scenarios/index.md` and returns an empty pack, with an inline comment calling out completion cache updates and avoiding crashes when npm distributions do not ship `qa/scenarios/`.
  - file: [extensions/qa-lab/src/scenario-catalog.ts:336](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/qa-lab/src/scenario-catalog.ts#L336)
