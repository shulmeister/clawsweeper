## Summary

Keep open. Current main still does not route ordinary Discord thread sessions through a parent channel `modelByChannel` override when `channels.discord.thread.inheritParent` is left at its default `false`. Discord preflight resolves the native parent channel id, and core can fall back through a supplied parent session key, but the reply model selection path does not receive that parent id for default-isolated thread sessions.

## Best Possible Solution

Keep this issue open and implement a narrow Discord model-routing fix: carry the resolved native Discord thread parent channel id into model override candidate resolution independently of `channels.discord.thread.inheritParent`, while preserving explicit thread-id `modelByChannel` precedence. Add regression coverage for a thread with no thread override but a parent channel override, plus a thread-specific override wins case.
