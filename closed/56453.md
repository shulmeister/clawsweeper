## Summary

Keep #56453 open. Current main still has the exact implicit `openclaw agent --to ...` default-agent mismatch the PR targets: derived `--to` keys still use `agent:main:*` when `--agent` is omitted, and the store is selected before a configured-default-agent-aware key exists. The related issue #56370 and broader PR #64108 confirm this is still actionable, but #64108 does not clearly supersede this focused PR because it is also open and has unresolved review concerns.

## Best Possible Solution

Keep the PR open or consolidate it deliberately with the broader #64108/#63992 work. The final fix should thread the configured default agent into implicit `--to` session-key derivation, derive the store path from the final resolved agent key, preserve explicit `--agent`, explicit `--session-key`, and cross-store `sessionId` behavior, and add regression coverage for a non-main default agent such as `steward`.
