## Summary

Current main already routes the reported Z.AI GLM setup through the Z.AI provider's OpenAI-compatible chat-completions path, not the Anthropic messages path that would append `/v1/messages`. The latest release tag also contains the same implementation, so the 2026.4.5 regression report is obsolete on current main.

## Best Possible Solution

Keep the shipped Z.AI plugin/model-resolution behavior. Z.AI GLM models should remain owned by the bundled Z.AI plugin, use `openai-completions` for the Z.AI coding and general endpoints, and let explicit provider config override stale registry metadata. A new issue would only be needed if someone can reproduce current main or v2026.4.24 still sending `zai/*` GLM traffic through `anthropic-messages`.
