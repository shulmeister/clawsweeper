## Summary

Current `main` no longer matches the PR’s reported bug: a restart on a not-loaded gateway does not unconditionally start a new service. The restart path now only handles an already-running unmanaged gateway or an installed-but-unloaded macOS LaunchAgent; otherwise it reports `not-loaded` and exits. This PR would also suppress the intended unmanaged restart path.

## Evidence

- **Update flow still delegates restart through daemon restart when no restart script is prepared:** `updateCommand` checks `isLoaded()` only to decide whether to prepare a restart script. If not loaded, it still passes `shouldRestart` through to `maybeRestartService`, so any behavior change must come from the daemon restart implementation rather than this PR landing on `main`.
  - file: [src/cli/update-cli/update-command.ts:1395](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/cli/update-cli/update-command.ts#L1395)
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Current daemon restart does not unconditionally start a service when not loaded:** On the not-loaded path, `runDaemonRestart()` only tries `restartGatewayWithoutServiceManager(restartPort)` and then `recoverInstalledLaunchAgent({ result: "restarted" })`. There is no unconditional start call here.
  - file: [src/cli/daemon-cli/lifecycle.ts:203](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/cli/daemon-cli/lifecycle.ts#L203)
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **No recovery falls back to 'not-loaded' instead of starting anything:** `runServiceRestart()` handles `!loaded` by invoking `onNotLoaded`; if that returns nothing, it emits the standard `not-loaded` response and returns `false` rather than starting the gateway.
  - file: [src/cli/daemon-cli/lifecycle-core.ts:398](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/cli/daemon-cli/lifecycle-core.ts#L398)
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Tests codify the current non-start behavior for not-loaded restarts:** Unit test covers restart requested for a not-loaded service and asserts the CLI reports 'Gateway service not loaded.' plus hints, matching the current non-start behavior.
  - file: [src/cli/daemon-cli/lifecycle-core.test.ts:102](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/cli/daemon-cli/lifecycle-core.test.ts#L102)
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Tests also codify the intended unmanaged restart path this PR would skip:** Current tests explicitly prefer unmanaged restart when a verified gateway listener exists without a loaded service. The PR’s `shouldRestart = false` change would suppress that supported path.
  - file: [src/cli/daemon-cli/lifecycle.test.ts:373](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/cli/daemon-cli/lifecycle.test.ts#L373)
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
