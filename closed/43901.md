## Summary

Keep open. Current main still has the per-session L1 bootstrap snapshot cache, and `resolveBootstrapFilesForRun` still uses it whenever `sessionKey` is present, so the PR’s core bug is not implemented or superseded on main.

## Best Possible Solution

Keep the PR open and refresh it against current main. The best fix remains to remove the session-keyed L1 bootstrap snapshot cache, or otherwise validate it by the same file identity used by `loadWorkspaceBootstrapFiles`; the simpler path is to always call `loadWorkspaceBootstrapFiles` and let the existing guarded L2 cache handle freshness. Add a regression test through `resolveBootstrapFilesForRun` with a `sessionKey` that modifies a bootstrap file between calls and proves the second resolve sees the new content, then update/remove reset-only snapshot-clearing tests and call sites that become obsolete.
