## Summary

Current main already guards the Slack draft-preview finalization path that matches this report: final replies are finalized in place when possible, and if the preview edit fails they fall back to normal Slack delivery instead of silently disappearing. The same logic is present in the released `v2026.4.23` tree, with targeted regression tests for failed preview finalization.

## Evidence

- **Slack draft preview finalization now falls back to normal delivery:** `dispatchPreparedSlackMessage` routes final Slack preview replies through `deliverFinalizableDraftPreview`; when preview finalization fails it logs the failure and calls normal reply delivery instead of dropping the final text.
  - file: [extensions/slack/src/monitor/message-handler/dispatch.ts:783](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/extensions/slack/src/monitor/message-handler/dispatch.ts#L783)

- **Shared preview finalizer guarantees fallback after failed final edit:** `deliverFinalizableDraftPreview` flushes the preview, tries the final edit, and if that fails discards pending preview state and invokes normal delivery before returning.
  - file: [src/channels/draft-preview-finalizer.ts:14](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/channels/draft-preview-finalizer.ts#L14)
