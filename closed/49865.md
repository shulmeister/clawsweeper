## Summary

Current main addresses the actionable parts of the Windows gateway stability cluster: SecretRefs are resolved into an active runtime snapshot and command paths can resolve from that snapshot; gateway status is now a connect/capability probe by default with read RPC only behind --require-rpc; cron errors are persisted and failure notifications/retry paths exist; ACPX is bundled as an embedded runtime backend; orphan transcript cleanup and the reported tools.profile warnings are covered. The broader gateway-wide stall class remains tracked separately by #63691 rather than by this umbrella issue.

## Best Possible Solution

Close this umbrella as implemented on current main. Keep the shipped runtime-snapshot SecretRef model, connect-first gateway status behavior, cron failure delivery/run-log behavior, embedded ACPX runtime, and session cleanup. Use #63691 for the remaining gateway-wide event-loop stall class, and ask for fresh narrow Windows repros only if one of these surfaces still fails on current builds.
