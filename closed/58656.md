## Summary

Current main already skips and retries heartbeat runs when the main lane or the resolved session lane is busy, which matches the requested defer/skip behavior for mid-turn processing.

## Evidence

- **Heartbeat skips while work is in flight:** `runHeartbeatOnce()` returns `skipped` with `reason: "requests-in-flight"` when either the main command lane or the resolved session lane has queued/active work, with the session-lane comment explicitly calling out avoiding interruption of an active streaming turn.
