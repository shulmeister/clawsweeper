## Summary

Current `main` already contains targeted fixes for the reported Windows browser/gateway failure mode: the gateway CLI path now yields one event-loop turn before starting the WebSocket handshake to avoid Windows `connect.challenge` timeouts, and browser manage commands now use longer/request-specific timeouts with regression coverage.

## Evidence

- **Gateway handshake mitigation on Windows:** `executeGatewayRequestWithScopes` explicitly yields with `setImmediate` before opening the CLI WebSocket connection, with an inline note that Windows module-loading starvation was causing `connect.challenge` handshake timeouts.
  - file: [src/gateway/call.ts:479](https://github.com/openclaw/openclaw/blob/48b9452c0795e749bd25406b2e93213b8533fee4/src/gateway/call.ts#L479)
  - command: `nl -ba src/gateway/call.ts | sed -n '450,540p'`
  - sha: [48b9452c0795](https://github.com/openclaw/openclaw/commit/48b9452c0795e749bd25406b2e93213b8533fee4)
- **Regression test for pre-connect yield:** `call.test.ts` now asserts that CLI pairing/gateway calls yield one event-loop turn before the client starts, covering the handshake-timeout mitigation path.
  - file: [src/gateway/call.test.ts:489](https://github.com/openclaw/openclaw/blob/48b9452c0795e749bd25406b2e93213b8533fee4/src/gateway/call.test.ts#L489)
  - command: `nl -ba src/gateway/call.test.ts | sed -n '470,530p'`
  - sha: [48b9452c0795](https://github.com/openclaw/openclaw/commit/48b9452c0795e749bd25406b2e93213b8533fee4)
- **Browser CLI commands now use longer manage-command timeouts:** `browser-cli-manage.ts` defines `BROWSER_MANAGE_REQUEST_TIMEOUT_MS = 45_000` and applies it to `status`, `tabs`, `profiles`, `open`, and related browser management requests instead of relying on the shorter prior behavior.
  - file: [extensions/browser/src/cli/browser-cli-manage.ts:19](https://github.com/openclaw/openclaw/blob/48b9452c0795e749bd25406b2e93213b8533fee4/extensions/browser/src/cli/browser-cli-manage.ts#L19)

  - sha: [48b9452c0795](https://github.com/openclaw/openclaw/commit/48b9452c0795e749bd25406b2e93213b8533fee4)
- **Regression tests for browser command timeout behavior:** Timeout-option tests verify that `browser start` honors an explicit parent timeout and that `status`, `tabs`, `profiles`, and `open` all use the new 45s built-in timeout.
