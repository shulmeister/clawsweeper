## Summary

Current main does not carry this PR's exact `session.maintenance.transcriptRotateBytes` / `transcriptMaxLines` API, but it now supersedes the PR's central active-transcript growth fix with an owner-authored compaction-based guard: `agents.defaults.compaction.maxActiveTranscriptBytes` triggers normal semantic compaction and, with `truncateAfterCompaction`, moves future turns onto a smaller successor JSONL instead of raw byte-splitting transcript history. The PR discussion was useful and addressed several race/performance review findings, but the product direction now present on main is narrower and safer than merging this XL raw-rotation patch.

## Best Possible Solution

Close this PR as superseded by current main's compaction-based active transcript size guard. Keep the shipped semantic compaction/successor-transcript direction, and only reopen the topic as a narrower maintainer-owned issue if OpenClaw still needs raw archival, line caps, or bulk inactive-transcript cleanup under `session.maintenance`.
