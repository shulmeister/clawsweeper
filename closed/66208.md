## Summary

This PR should close as ClawHub/plugin-scope. It adds a large bundled Microsoft 365 mail/calendar service integration, while current OpenClaw direction keeps optional commercial integrations outside core unless a missing generic plugin API is demonstrated. Current main already has the needed external plugin surfaces for tools, HTTP/webhook routes, plugin approvals, and same-chat `/approve` in Microsoft Teams; the rich M365 implementation can continue as an external ClawHub/npm plugin. This also aligns with the prior ClawSweeper decision on the related Outlook triage approvals PR #66209 and Feishu Mail/Calendar issue #32618.

## Best Possible Solution

Close this core PR and continue the M365 calendar/mail work as an external ClawHub/npm plugin using current plugin tools, HTTP routes, hooks, and plugin approvals. If richer Teams Adaptive Card callbacks are still needed beyond same-chat `/approve`, open a small generic Teams/plugin-interactive API PR with focused tests instead of bundling it with the Microsoft 365 integration.
