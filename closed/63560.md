## Summary

Current `main` already routes OpenRouter auth through provider-specific credential resolution, ignores unrelated `openai-codex` profiles, and has committed coverage showing explicit provider config auth works plus a live OpenRouter smoke test.

## Evidence

- **Auth profile lookup is provider-scoped:** `listProfilesForProvider` only returns stored profiles whose normalized provider matches the requested provider, so an unrelated `openai-codex` profile is not eligible for `openrouter` resolution.
  - file: [src/agents/auth-profiles/profile-list.ts:8](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/auth-profiles/profile-list.ts#L8)
