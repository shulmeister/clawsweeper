## Summary

Current `main` already has the Control UI and gateway pieces needed to keep image-only user messages visible after replies: the UI skips mid-run history reloads that race with optimistic rendering, the gateway persists user image uploads as transcript `MediaPath`/`MediaPaths`, and the renderer explicitly displays those history-backed user images.

## Evidence

- **UI avoids the mid-run reload race:** `handleSessionMessageGatewayEvent` now defers `chat.history` reloads while `chatRunId` is active, with an inline comment explaining that reloading mid-run races with the optimistic user-message update and delays the message card.

  - file: [src/gateway/server-methods/chat.ts:746](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/gateway/server-methods/chat.ts#L746)
