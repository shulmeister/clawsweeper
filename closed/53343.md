## Summary

Current main already provides awaited inbound gating and optional early reply through the `before_dispatch` plugin hook, which covers the capability this PR was adding via a new blocking mode on `message_received`. `message_received` remains observer-only on main, so this PR's API shape is redundant with the supported hook surface.

## Evidence
