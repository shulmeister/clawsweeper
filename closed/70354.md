## Summary

Current main already solves the OpenClaw-managed systemd lockout path by making the generated and documented gateway systemd unit use `Restart=always`, so a clean supervised SIGUSR1/update exit is relaunched instead of being left dead under `Restart=on-failure`. The linked #70466 exit-code PR is an alternate implementation, but the issue's requested pragmatic packaged-unit fix is already present in code, docs, and release content.

## Best Possible Solution

Close this issue as implemented for the supported OpenClaw-managed Linux service path. Keep the canonical generated systemd unit and public docs on `Restart=always`; handle any remaining desire to change supervised SIGUSR1 exit codes in the separate #70466 PR discussion rather than keeping this user report open.
