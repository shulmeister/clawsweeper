## Summary

Keep open. Current main still hard-codes wake hooks to the configured main session and drops wake mapping session/agent routing, while PR #70268 is a focused, recently updated fix for an open gateway bug rather than obsolete cleanup.

## Best Possible Solution

Keep #70268 open and finish normal maintainer review. The best implementation path is to land this focused gateway fix or an equivalent change that lets direct `/hooks/wake` and mapped `action: "wake"` honor explicit `sessionKey`/`agentId`, canonicalizes the target before enqueueing and heartbeat wakeup, enforces `allowedSessionKeyPrefixes` for explicit and computed wake targets, and keeps regression coverage for direct, mapped, policy-denied, and any global-scope edge cases.
