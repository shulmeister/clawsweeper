## Summary

Current main rewires Gemini CLI OAuth onto the canonical `google/*` model path with explicit `google-gemini-cli` runtime selection, keeps the auth choice grouped under Google, and preserves requested 3.1 model IDs during forward-compat cloning. The v2026.4.10 communication regression described here no longer matches current main.

## Evidence
