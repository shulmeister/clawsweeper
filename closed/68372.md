## Summary

Current `main` already truncates oversized `toolResult` messages before they are persisted to the session transcript, with tests covering an 80KB input, so the PR's core session-inflation safeguard is already implemented. The PR's own separate 51.2KB-cap patch also carried unresolved review findings, so there is not a strong reason to keep this redundant branch open.

## Evidence

- **Current guard caps tool results before persistence:** `installSessionToolResultGuard` resolves `maxToolResultChars`, truncates `toolResult` messages before persistence, and re-applies the cap on the raw append path. That means current `main` already prevents verbatim oversized tool output from being written into the session transcript.
