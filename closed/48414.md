## Summary

Current main already separates lightweight gateway reachability from true read-scope RPC checks, and it now reports missing auth context explicitly instead of only collapsing into an ambiguous unreachable result.

## Evidence

- **Gateway status now distinguishes connect vs read probes:** `probeGatewayStatus()` uses a lightweight connect probe by default and only performs a real read-scope RPC when `--require-rpc` is set, which addresses the original mismatch between a basic health check and deeper auth/RPC paths.
