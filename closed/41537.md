## Summary

Keep #41537 open. Current main has doctor-side source-config and repair coverage that preserves Telegram `dmPolicy: "allowlist"`, but the `openclaw configure` channel path still presents the DM policy select without seeding `initialValue` from the current policy. With the current option order and Clack select behavior, accepting the select default can still choose `pairing` and then persist that over an existing Telegram allowlist.

## Best Possible Solution

Keep the issue open and fix the configure path by computing the current policy before rendering the select and passing it as `initialValue`, then add a focused regression test showing an existing Telegram `dmPolicy: "allowlist"` is preserved when the user accepts the prompt default. Keep the existing doctor source-config repair behavior and coverage in place.
