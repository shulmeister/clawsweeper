## Summary

Current main still leaves #37705 actionable: the bundled Bonjour plugin can pass a 64-byte service instance label, and an overlong `OPENCLAW_MDNS_HOSTNAME` label, into `@homebridge/ciao` 1.3.6, whose DNS encoder asserts above 63 bytes. The provided discussion links fix attempts in PRs #37712 and #37714, but both were closed unmerged.

## Best Possible Solution

Keep #37705 open. Add a shared byte-aware DNS-label sanitizer in the bundled Bonjour plugin path, apply it to both service instance name and hostname before `createService()`, preserve UTF-8 boundaries with a non-empty fallback, and add focused tests for exact 63-byte labels, 64-byte labels, the reported Kubernetes-style hostname plus suffix, `OPENCLAW_MDNS_HOSTNAME`, and multibyte UTF-8. A maintainer reply should also clarify that the supported workaround is `OPENCLAW_DISABLE_BONJOUR=1` or `discovery.mdns.mode="off"`, not `CLAWDBOT_DISABLE_BONJOUR=1`.
