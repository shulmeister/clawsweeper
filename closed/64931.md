## Summary

Current shipped OpenClaw already provides the requested durable env mechanism for regenerated LaunchAgents: `openclaw update` refreshes the gateway service through the install pipeline, and that pipeline rebuilds LaunchAgent env from supported persistent sources (`~/.openclaw/.env` and config `env`). Manual plist edits are still non-durable by design, but the underlying request for a config-level persistence path is implemented.
