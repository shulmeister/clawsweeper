## Summary

Current main has several relevant primitives, but it does not implement the requested unified pre-execution planner that classifies user tasks into direct, subagent, ACP, confirmation-first, skill-loading, or channel-policy routes. The existing `activation-planner` is a plugin-loading control-plane helper, and plugin runtime APIs expose embedded agent, subagent, and task-flow primitives, not an automatic task-routing layer or `openclaw route` debug surface. This needs maintainer product/design judgment rather than conservative cleanup closure.

## Best Possible Solution

Keep the issue open for maintainer product/design judgment. A good next step would be a narrower design proposal that defines whether the planner belongs in core agent/gateway routing, a public plugin hook, or a diagnostic-only CLI surface, and explicitly maps it onto existing ACP, subagent, taskFlow, plugin activation, skill, approval, and channel-policy contracts.
