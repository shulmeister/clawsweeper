## Summary

Current main already fixes the reported attribution problem for OpenClaw-generated runtime/continuation context: the visible user prompt is kept separate from hidden runtime context, hidden context is persisted as a non-display custom message, and doctor can repair the affected duplicated transcript branch shape.

## Best Possible Solution

Close as implemented on main. Keep the shipped hidden custom-message design and doctor repair path; if users still see old affected history, direct them to run `openclaw doctor --fix` or open a fresh issue with a current reproduction.
