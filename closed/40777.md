## Summary

Keep open. Current main partially covers the reported class of leaks in the normal embedded assistant pipeline and final assistant-text extraction, but it does not implement the PR's defense-in-depth surfaces: `sanitizeUserFacingText` still does not strip reasoning tags, `normalizeReplyPayload` and streaming delivery still rely on that sanitizer, and gateway `emitChatDelta` forwards direct assistant events after only directive/internal-context cleanup. This PR is not obsolete; it needs a rebase/integration pass rather than conservative closure.

## Best Possible Solution

Keep the PR open. The best maintainer path is to rebase it onto current main and preserve the useful pieces that are still missing: add reasoning-tag cleanup to the last-mile reply sanitizer or its normalize callers, add gateway chat-delta protection for direct assistant events with split `<think>` chunks, and add focused regression tests for `normalizeReplyPayload`/streaming delivery plus direct gateway agent-event chat streaming.
