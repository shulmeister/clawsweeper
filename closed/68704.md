## Summary

Keep open. Current main does not implement the requested Gemma 4 OpenAI-compatible replay stripping: signed thinking blocks are still preserved for same-model replay and the OpenAI completions converter will rehydrate them as `reasoning_content`. The linked PR #68763 is an exact proposed fix for #68704, but it is open/unmerged and its proposed `dropReasoningFromHistory` API is not present on main.

## Best Possible Solution

Keep #68704 open and use PR #68763 as the starting point: add a typed replay policy such as `dropReasoningFromHistory` or a documented model/provider opt-in, apply it before OpenAI-compatible conversion for Gemma 4+ routes, and add focused regression coverage proving prior-turn `reasoning_content` is stripped while required same-turn tool-call reasoning behavior is preserved.
