## Summary

Current `main` already fixes the reported gateway-wide Discord crash: the Discord gateway no longer runs with `maxAttempts=0`, late `reconnect-exhausted` errors are suppressed during teardown/dispose instead of surfacing as uncaught process-killers, and the channel runtime contains per-account exits and auto-restarts them instead of letting one Discord account take down the whole gateway. The report's assumption that `channels.discord.retry` should drive WebSocket reconnects does not match current docs/config help; that setting is documented for outbound Discord HTTP/API retries only.

## Evidence

- **Gateway reconnect attempts are no longer zero:** The current Discord gateway plugin passes Carbon `reconnect: { maxAttempts: 50 }`, so the old `Max reconnect attempts (0)` path is not the current implementation.
  - file: [extensions/discord/src/monitor/gateway-plugin.ts:377](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/extensions/discord/src/monitor/gateway-plugin.ts#L377)
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Late reconnect-exhausted errors are classified and suppressed during teardown/dispose:** `createDiscordGatewaySupervisor` maps `Max reconnect attempts` errors to `reconnect-exhausted` and logs/suppresses them once lifecycle teardown/dispose has started, instead of leaving them as uncaught emitter errors.
  - file: [extensions/discord/src/monitor/gateway-supervisor.ts:76](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/extensions/discord/src/monitor/gateway-supervisor.ts#L76)

  - file: [extensions/discord/src/monitor/gateway-supervisor.test.ts:84](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/extensions/discord/src/monitor/gateway-supervisor.test.ts#L84)

  - file: [extensions/discord/src/monitor/provider.test.ts:293](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/extensions/discord/src/monitor/provider.test.ts#L293)

  - file: [src/gateway/server-channels.ts:431](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/gateway/server-channels.ts#L431)

- **Docs and config help define `channels.discord.retry` as outbound HTTP retry policy, not WS reconnect policy:** Both the Discord config UI hints and retry docs say the Discord retry block is for outbound Discord API calls / HTTP 429 handling.
  - file: [extensions/discord/src/config-ui-hints.ts:68](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/extensions/discord/src/config-ui-hints.ts#L68)

  - file: [CHANGELOG.md:1917](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/CHANGELOG.md#L1917)
