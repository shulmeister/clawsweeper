## Summary

Current main already covers the reported argument-loss path: `openai-codex/gpt-5.4` is normalized away from the generic `openai-completions` transport, and the remaining `openai-completions` stream path now repairs malformed tool-call JSON before execution with regression tests.

## Evidence

- **OpenAI Codex GPT-5.4 no longer stays on generic OpenAI transports:** Current regression coverage asserts that even if `openai-codex` is configured with `api: "openai-completions"`, `openai-codex/gpt-5.4` is rewritten back to `api: "openai-codex-responses"` on `https://chatgpt.com/backend-api`. That removes the stale transport path implicated by the March 2026 report.
