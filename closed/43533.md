## Summary

Close #43533 as superseded. Current main still has the automatic daily/idle session-memory gap, but this PR's latest implementation is not the best remaining fix: it routes stale rollovers through get-reply command fallback as command:new, has unresolved review concerns, and is now covered by newer active trackers/PRs for the same underlying bug.

## Best Possible Solution

Close this older PR as superseded and keep the remaining product/code work consolidated under #50891 plus the newer active implementation PRs, especially #61675 or the narrower #71792. The final fix should emit automatic rollover hook coverage from the stale-session path in initSessionState before reply/directive short-circuits, update session-memory registration/handler behavior, avoid misleading command:new audit semantics, and include regression tests for daily and idle rollovers.
