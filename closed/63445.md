## Summary

The request is a provider-specific auth behavior change that current main intentionally keeps out of core for remote custom providers, and the plugin SDK already exposes the hook needed to implement it as a provider plugin/ClawHub package.

## Risks / Open Questions

- No already-posted ClawHub/plugin implementation was linked in the issue, so closing assumes maintainers prefer the existing provider-plugin path over adding a generic core `no-auth` marker.
- If maintainers want a new first-party config-level `auth: none` contract for arbitrary user-defined remote providers, that would be a product decision beyond conservative cleanup.
