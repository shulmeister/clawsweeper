## Summary

The issue discussion ultimately narrowed this to the embedded `acpx` + `agentId="openclaw"` bridge path. Current `main` already routes OpenClaw ACP bridge sessions through an MCP-free ACPX delegate, has explicit regression tests for the `openclaw`/env-wrapped/resumed-session cases, and the fix is listed in the `v2026.4.23` shipped release notes.

## Evidence

- **OpenClaw ACP bridge now uses MCP-free ACPX delegate:** `AcpxRuntime` creates a distinct `bridgeSafeDelegate` with `mcpServers: []`, routes `ensureSession` by agent command, and routes follow-up `runTurn`/`getStatus` calls by persisted handle/session command. That directly matches the reported `agentId="openclaw"` ACP session path.
