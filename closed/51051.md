## Summary

Keep open. Current main still has the source-of-truth split this PR targets: `nodes status` reads `node.list`, while unfiltered `nodes list` reads only `node.pair.list`, so the reported k8s/Docker mismatch remains credible. The PR is not ready to merge as-is because review discussion flagged JSON contract and scope-regression concerns, and the branch also includes an unrelated Ollama auth change.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up, or supersede it with a focused CLI fix that makes `nodes list` reconcile `node.list` and `node.pair.list`, preserves pending and pairing-only rows, handles pairing-only tokens deliberately, keeps the JSON output contract stable or explicitly migrates it, adds regression coverage for an empty `node.pair.list` plus a paired live node in `node.list`, and removes the unrelated Ollama auth change from this branch.

## Risks / Open Questions

- Merging the PR shape described in the supplied diff could change `nodes list --json` from `PairedNode[]` to `NodeListNode[]` unless the JSON contract is preserved or intentionally documented.
