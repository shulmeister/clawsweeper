## Summary

Keep open. Current main still omits `agentDir` and `workspaceDir` from the `wrapProviderStreamFn` context, while the provider hook type and OpenAI/Codex wrapper path expect `agentDir` to detect local `openai-codex` auth. The PR is a narrow two-file fix with a regression test, and the related #65909 remains open for broader Codex native web_search reliability.

## Best Possible Solution

Keep this PR open for maintainer review and either merge it or recreate its narrow fix on main: add `agentDir` and `workspaceDir` to the `wrapProviderStreamFn` context in `applyExtraParamsToAgent`, keep the regression test, and leave #65909 open for broader Codex native web_search behavior that this PR intentionally does not claim to solve.
