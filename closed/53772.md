## Summary

Keep open. Current main has a compaction summarization model override, including local-model use, but the broader request is still not implemented: pre-compaction memory flush runs on the active run model/fallback path, startup context is injected into the active prompt rather than summarized through a separate model, and there is no general housekeeping/maintenance model override.

## Best Possible Solution

Keep this issue open and narrow the implementation target. The likely core path is to add a dedicated maintenance or memory-flush model override, route `runMemoryFlushIfNeeded` through it, and add schema/docs/tests for auth-profile handling, fallback behavior, local-model context limits, and tool/workspace capability expectations. Startup reads should be handled separately: either keep them as controlled prompt injection with existing `startupContext`, `contextInjection`, and bootstrap size settings, or define a distinct summarized-startup-context feature.
