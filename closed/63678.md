## Summary

Keep open. Current main still has the strict QMD vector parser that only accepts `Vectors:` output, while the linked bug reports healthy QMD vectors being misreported when status formatting differs. This PR is still open and focused on that exact remaining parser/test gap.

## Best Possible Solution

Keep this PR open for maintainer review, or land an equivalent patch that broadens `parseQmdStatusVectorCount` in `extensions/memory-core/src/memory/qmd-manager.ts` and adds regression tests for alternate QMD status output such as `Vectors = N`, while preserving the zero-vectors false result and the line-anchored fallback from the review discussion.
