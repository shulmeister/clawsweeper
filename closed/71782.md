## Summary

Current main already addresses the reported MEMORY.md bootstrap truncation problem. Bootstrap limits remain configurable, truncation is detected from raw-vs-injected sizes, an agent-visible `[Bootstrap truncation warning]` is appended to embedded and CLI prompt paths by default, and `/context` plus `openclaw doctor` expose user-facing raw/injected size warnings. This overlaps prior cleanup reports #53731 and #54623, and current source/docs/tests confirm the same shipped behavior.

## Best Possible Solution

Keep the current main behavior: retain configurable per-file and total bootstrap budgets, keep default once-per-signature agent-visible prompt warnings, and direct users with intentionally large MEMORY.md files to tune `agents.defaults.bootstrapMaxChars` / `agents.defaults.bootstrapTotalMaxChars` or use memory tools for on-demand recall. No additional core change is needed for this issue.
