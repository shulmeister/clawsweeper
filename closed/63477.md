## Summary

Current `main` appears to fix this Slack Socket Mode regression: inbound channel traffic is now registered on Bolt's generic `message` event, with tests covering `message`/`app_mention` handling and docs clarifying Slack channel-vs-DM access control. The fix is on `main` but I could not tie it to a released tag yet.

## Evidence

- **Current main uses Bolt's generic message event:** Slack now registers `ctx.app.event("message", ...)`, with an inline note that `message.channels` / `message.groups` are subscription labels and Bolt rejects them since v4.6. That directly matches the reported symptom: Socket Mode connected, but public-channel events never arrived.
