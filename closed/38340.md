## Summary

Keep open. Current `main` has partial overlap with this PR (`beforeDeliver` exists and helper wrappers can attach `message_sending`), but the live Discord same-surface reply path in `extensions/discord/src/monitor/message-handler.process.ts` still constructs its own dispatcher without `beforeDeliver`, so this PR's reported Discord hook gap is not shown fixed on `main`.

## Risks / Open Questions

- `main` already contains overlapping dispatcher-hook work, so this PR is no longer a clean cherry-pick and would need a rebase/adaptation to the current `src/auto-reply/dispatch.ts` architecture.
- The generic `beforeDeliver` implementation on `main` currently cancels on falsy values (`if (!deliverPayload)`), while the PR discussion/documentation expects explicit `null` cancellation only; any merge should reconcile that contract carefully.
