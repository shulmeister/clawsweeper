## Summary

Keep open. Current main still does not implement low-memory/OOM npm install recovery for the one-line installer, the linked bug #39447 remains open, and the PR discussion raises real product/security tradeoffs around automatic swap creation that need maintainer judgment rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is either a safer installer fix for #39447, such as opt-in/temporary swap or clearer OOM diagnostics and manual recovery, or a maintainer decision to close the PR in favor of a narrower implementation that avoids persistent automatic system changes. Any accepted fix should update installer docs and add focused script-level coverage where feasible.
