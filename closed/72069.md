## Summary

Keep this PR open. Current main still does not implement cross-provider fallback prompt seeding from Claude CLI session history, while this PR is a focused implementation candidate for the concrete continuity bug reported in #69973.

## Best Possible Solution

Keep this PR open for maintainer review. The likely correct path is to land this PR or an equivalent implementation that seeds a bounded Claude CLI summary/recent-turn prelude only when a fallback chain started on `claude-cli` and retries on another runtime, with regression coverage in `src/agents/command` and `src/gateway` tests.
