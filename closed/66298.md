## Summary

Keep open. Current `main` still routes `/acp ...` messages in ACP-bound sessions through ACP dispatch instead of bypassing to the command handler, and the tests explicitly lock that behavior in.

## Evidence

- **Current bypass logic still excludes `/acp` commands:** `shouldBypassAcpDispatchForCommand` only bypasses `/new` and `/reset`, then rejects any slash command that is not bang-prefixed. A bound-thread message like `/acp close` therefore does not bypass ACP dispatch on current `main`.
  - file: [src/auto-reply/reply/dispatch-acp-command-bypass.ts:48](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/auto-reply/reply/dispatch-acp-command-bypass.ts#L48)
  - command: `nl -ba src/auto-reply/reply/dispatch-acp-command-bypass.ts | sed -n '1,80p'`
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Unit test codifies the broken behavior:** `dispatch-acp-command-bypass.test.ts` currently asserts that ACP slash commands return `false` from the bypass check, including `/acp cancel`. That means the reported routing bug is still the expected behavior in the checked-in tests.
  - file: [src/auto-reply/reply/dispatch-acp-command-bypass.test.ts:18](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/auto-reply/reply/dispatch-acp-command-bypass.test.ts#L18)
  - command: `nl -ba src/auto-reply/reply/dispatch-acp-command-bypass.test.ts | sed -n '1,80p'`
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Non-bypassed command candidates are still sent into ACP dispatch:** `tryDispatchAcpReplyHook` computes `bypassForCommand` and then calls `tryDispatchAcpReply`; `tryDispatchAcpReply` immediately returns `null` only when `bypassForCommand` is true. With `/acp ...` still returning `false`, bound-session command text continues down the ACP session path.
  - file: [src/plugin-sdk/acp-runtime.ts:79](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/plugin-sdk/acp-runtime.ts#L79)

  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **OpenClaw docs still promise `/acp close` removes the binding:** The public ACP docs describe `/acp close` as the command that removes a binding from a bound ACP conversation, which matches the issue reporter's expectation and conflicts with the current bypass implementation.
  - file: [docs/tools/acp-agents.md:115](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/docs/tools/acp-agents.md#L115)
