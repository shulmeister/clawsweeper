## Summary

Keep open. Current `main` still performs the full score-all, `toSorted`, then `slice(limit)` pipeline in the current bundled `memory-core` plugin path, so PR #53665 is not implemented on main. The original PR targets the older `src/memory/manager-search.ts` path and would need rebasing/adaptation to `extensions/memory-core/src/memory/manager-search.ts`, but the optimization remains applicable and useful.

## Best Possible Solution

Keep the PR open as useful but require a small rebase/adaptation: apply the bounded top-k fallback search to `extensions/memory-core/src/memory/manager-search.ts`, add the fallback top-k tests under `extensions/memory-core/src/memory/manager-search.test.ts`, and resolve the minor review feedback before maintainer merge consideration.
