## Summary

Current main already addresses the reported subagent announce destabilization chain: completion cleanup launches announce delivery asynchronously, completion announce delivery is bounded and can fall back to queue delivery, inter-session completion wakes no longer create cancellability-hostile CLI task rows, subagent task finalization is best-effort, and the related requester_session_key NOT NULL crash path is obsolete.

## Best Possible Solution

Close this broad 2026.4.1 stability report as implemented on main and keep the current lifecycle/task-ledger design. If new logs show remaining gateway pressure from many simultaneous completion announces, that should be filed as a fresh, narrower concurrency/backpressure issue with current-version logs.
