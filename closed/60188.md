## Summary

Keep open. Current main may route many normal CLI calls through the profile config path, but the specific requested propagation is not implemented: `applyCliProfileEnv` still only sets `OPENCLAW_GATEWAY_PORT` for the `dev` profile, and there is no regression coverage showing named-profile `gateway.port` is exported or survives an already-pinned config snapshot.

## Evidence

- **Issue discussion identifies the missing env propagation:** The issue comment reports two root causes: named profiles do not set `OPENCLAW_GATEWAY_PORT`, and a process-level config cache can lock in the wrong config before profile env is applied.
- **Named profile env bootstrap still only sets a gateway port for dev:** `applyCliProfileEnv` derives `OPENCLAW_PROFILE`, `OPENCLAW_STATE_DIR`, and `OPENCLAW_CONFIG_PATH` for any profile, but only assigns `OPENCLAW_GATEWAY_PORT = "19001"` when `profile === "dev"`. It does not read `gateway.port` from the named profile config.
  - file: [src/cli/profile.ts:84](https://github.com/openclaw/openclaw/blob/02f3e9cfa25f26826e8dfd79d4bf174f2f9d8f54/src/cli/profile.ts#L84)
