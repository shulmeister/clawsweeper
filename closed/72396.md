## Summary

Current main still supports one writable plugin runtime dependency stage root, not layered read-only baseline roots plus a final writable install root. This is a concrete core plugin loader/runtime-deps feature request, not a ClawHub/plugin-only item or duplicate of the closed EACCES repair bug.

## Best Possible Solution

Keep this issue open and design it as a core plugin runtime-dependency staging enhancement: parse a documented stage-root list, resolve dependency sentinels left-to-right across immutable baseline roots plus the final writable root, install/repair only into the final writable root, register aliases for the root that satisfies each dependency, preserve deterministic cache keys across OpenClaw version and runtime-deps manifests, and add tests/docs for Docker/systemd/offline deployments. Treat closed #71730 as related EACCES repair history, not the canonical tracker for layered stage directories.

## Risks / Open Questions

- Path-list semantics need careful POSIX/Windows handling because `path.delimiter` differs and colon is currently accepted as part of the unparsed explicit stage value on POSIX.
- Loader alias registration, NODE_PATH registration, retained manifests, install locks, doctor repair, and health reporting currently assume one install root and should be updated together.
