## Summary

Current main already addresses this false-negative gateway status case: `openclaw status` uses a lighter presence-only probe, and authenticated loopback probes keep device identity attached so shared-secret loopback sessions are not stripped of read scope. The same implementation is already present in release `v2026.4.22`.

## Evidence
