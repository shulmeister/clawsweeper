## Summary

Keep #63313 open. Current main still lacks both requested behaviors: `memorySearch` has no `inputType` / `queryInputType` / `documentInputType` config path for OpenAI-compatible embeddings, and the Ollama memory embedding provider still batches by looping over `/api/embeddings` rather than calling `/api/embed`. The PR is not safely closeable as implemented, duplicate, or superseded; it still represents potentially useful product work, though its review comments need maintainer/author resolution before merge.

## Best Possible Solution

Keep the PR open or replace it with a cleaner follow-up that implements the same two product changes narrowly: add documented `memorySearch.inputType`, `queryInputType`, and `documentInputType` config/schema/types, forward effective query/document labels through OpenAI-compatible inline and batch embedding requests with correct cache-key semantics, and switch Ollama indexing to `/api/embed` with conservative fallback to the existing `/api/embeddings` path plus tests for unsupported status codes and malformed/partial batch responses.
