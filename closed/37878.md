## Summary

Current main does not implement the requested behavior. The model-usage helper still drops numeric-string costs, still treats booleans as numeric because Python bool is an int subclass, and still accepts non-finite float costs. The linked PR #37877 was closed unmerged, so it is not a canonical implemented fix.

## Evidence

- **aggregate_costs still uses raw int/float check:** `aggregate_costs()` reads `item.get("cost")`, skips anything that is not `int` or `float`, and then adds `float(cost)`. That skips valid numeric strings like `"1.75"`; because there is no bool exclusion or finite check, `True` and `float('inf')` are still accepted.
