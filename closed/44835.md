## Summary

Current main, and the latest tagged v2026.4.24 release, address the original Web UI stale form serialization problem behind the `models.0.id/name undefined` validation loop. The issue discussion also confirmed the remaining `Unknown model: openai/moonshot-v1-8k` symptom was a provider-prefix configuration mismatch, not a schema bug.

## Evidence

- **Discussion confirmed original failure mode and workaround:** In the issue thread, dsantoreis identified the validation error as a Web UI form serialization/stale state issue, the reporter confirmed clicking away to trigger blur removed the undefined-field error, then the same thread traced `openai/moonshot-v1-8k` to a provider key mismatch and the reporter confirmed changing the prefix fixed the validation loop.
- **Text inputs now patch form state on input:** Current main updates config form state on every input event for text fields, with change only trimming after the fact. That removes the reported Save-before-blur failure mode where visible ID/name values could still serialize as undefined.
