## Summary

Current `main` already implements provider-owned model catalog discovery and exposes it through `openclaw models list`, with live querying for multiple providers plus documented auto-discovery flows.

## Evidence

- **Core provider-catalog hook exists:** Core resolves provider `catalog`/`discovery` hooks and runs them as the generic mechanism for provider-owned model discovery.
  - file: [src/plugins/provider-discovery.ts:20](https://github.com/openclaw/openclaw/blob/28eb56dd21b61631ba916ea3dd0fc2afa002f31d/src/plugins/provider-discovery.ts#L20)
  - sha: [28eb56dd21b6](https://github.com/openclaw/openclaw/commit/28eb56dd21b61631ba916ea3dd0fc2afa002f31d)
- **Models CLI loads provider catalogs:** `openclaw models list` can load provider-catalog models by provider filter and static/live discovery paths, so users do not need to hand-maintain every model list just to inspect available models.
  - file: [src/commands/models/list.provider-catalog.ts:167](https://github.com/openclaw/openclaw/blob/28eb56dd21b61631ba916ea3dd0fc2afa002f31d/src/commands/models/list.provider-catalog.ts#L167)
  - sha: [28eb56dd21b6](https://github.com/openclaw/openclaw/commit/28eb56dd21b61631ba916ea3dd0fc2afa002f31d)
- **Generic OpenAI-compatible self-hosted discovery:** OpenClaw has a shared helper that queries `<baseUrl>/models` and turns the response into provider model definitions for self-hosted OpenAI-compatible providers.
  - file: [src/plugins/provider-self-hosted-setup.ts:43](https://github.com/openclaw/openclaw/blob/28eb56dd21b61631ba916ea3dd0fc2afa002f31d/src/plugins/provider-self-hosted-setup.ts#L43)
  - sha: [28eb56dd21b6](https://github.com/openclaw/openclaw/commit/28eb56dd21b61631ba916ea3dd0fc2afa002f31d)
- **Live Vercel AI Gateway catalog query:** The bundled Vercel AI Gateway provider queries `/v1/models` at runtime and falls back to a static catalog only if discovery fails.
  - file: [extensions/vercel-ai-gateway/models.ts:189](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/extensions/vercel-ai-gateway/models.ts#L189)
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Ollama auto-discovery is documented and explicit:** User-facing docs state that, when `models.providers.ollama` is not explicitly defined, OpenClaw discovers models from the local Ollama instance via `/api/tags` plus `/api/show`, and newly pulled models become available automatically.
  - file: [docs/providers/ollama.md:151](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/docs/providers/ollama.md#L151)
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Discovery-backed provider list is documented for hosted providers too:** The Vercel AI Gateway docs explicitly describe the model catalog as auto-discovered via `/v1/models`, showing the feature is not limited to one local provider.
  - file: [docs/providers/vercel-ai-gateway.md:12](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/docs/providers/vercel-ai-gateway.md#L12)
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Tests cover display-only catalog loading without live fetch:** Tests verify that provider catalog rows can be shown for `models list` without requiring a live fetch for display-only static catalog paths.
  - file: [src/commands/models/list.provider-catalog.test.ts:161](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/commands/models/list.provider-catalog.test.ts#L161)

## Risks / Open Questions

- Discovery is provider-specific rather than universal: some providers still intentionally rely on static catalogs or explicit config, so the feature is implemented as a plugin/provider capability instead of one identical flow for every backend.
