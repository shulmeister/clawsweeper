## Summary

#59739 should close as a duplicate of the older, open #7524. Current main still routes Telegram groups to isolated group/topic session keys and exposes no per-group `session: "main"`/`groupsAsMain` option, but #7524 already tracks the same remaining product work as a focused `groupScope`/group-to-main session routing request. The cross-referenced #56692 is related safety discussion for multi-agent group context, not the canonical tracker for sharing group and DM session history.

## Best Possible Solution

Close #59739 as a duplicate and keep #7524 as the canonical implementation tracker. The best fix is a core routing/config contract for selective group or channel peers to use the main or another named shared session, then wire Telegram through that shared contract with schema metadata, docs, and regression coverage for group/topic routing.
