## Summary

Current main still leaves #17761 unresolved. `trusted-proxy` auth returns immediately on proxy-auth failure, startup validation rejects `trusted-proxy` plus a shared token, Tailscale is disabled by default for this mode, credential planning excludes shared-secret credentials, and tests/docs encode the fail-closed behavior. The linked fix PR #17746 is closed unmerged, while #64122 remains related open work rather than a shipped fix.

## Best Possible Solution

Keep #17761 open until maintainers either land an intentional trusted-proxy shared-secret or Tailscale fallback, or explicitly decide the current fail-closed trusted-proxy contract is final. A complete fix should coordinate `src/gateway/auth.ts`, `src/gateway/auth-resolve.ts`, `src/gateway/credential-planner.ts`, client credential resolution, regression/e2e coverage, and `docs/gateway/trusted-proxy-auth.md`; #64122 is the current related implementation thread to continue or supersede.
