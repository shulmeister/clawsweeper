## Summary

Keep this PR open. Current main still implements Control UI Talk as an OpenAI browser WebRTC session only, while Google Live is documented and implemented for backend realtime bridges. The draft PR is a plausible implementation candidate because the missing work spans the gateway protocol, browser UI transport, and bundled Google provider contract rather than a stale cleanup or ClawHub-only request.

## Best Possible Solution

Keep this PR open as the implementation candidate. Maintainers should review whether Google Live browser Talk belongs in the bundled Google provider and Control UI, then either land a polished version with provider, gateway protocol, UI, docs, and tests, or redirect it into a narrower generic browser realtime transport API design if that is the preferred product boundary.
