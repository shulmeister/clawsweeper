## Summary

Keep open. Current `main` still delegates Codex OAuth workspace selection upstream, persists the returned OAuth credentials without any OpenClaw-side active-workspace validation/pinning, and only has a narrow fallback for `Failed to extract accountId from token`. I found no `deactivated_workspace` handling on `main`, so there is not strong evidence this issue is fixed or obsolete.

## Evidence
