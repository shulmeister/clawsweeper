## Summary

Current `main` now threads isolated-cron run timeouts through the embedded runner into guarded provider transport fetches, adds isolated-cron cold-path/bootstrap reductions, and covers `deleteAfterRun` session cleanup. That matches the exact 60-second first-call timeout pattern reported against 2026.4.2 and the zombie-session side effect, although the timeout fix appears to be unreleased as of this review.

## Evidence

- **Cron timeout now propagates from job payload into isolated embedded runs:** `runCronIsolatedAgentTurn` resolves `payload.timeoutSeconds` via `resolveAgentTimeoutMs`, and `executeCronRun` passes that `timeoutMs` into `runEmbeddedPiAgent`, so isolated cron no longer relies on a fixed 60s transport default.
