## Summary

Current `main` now feeds successful isolated cron output back into the main session as a queued system event, and the cron docs already document the supported shared-context session modes (`main`, `current`, `session:<id>`).

## Risks / Open Questions

- This is on `main` after the latest tagged release `v2026.4.23`, so released builds may still show the old behavior until the next release ships.
- The bridge queues delivered isolated cron text into the next main-session turn; workflows that need durable shared history across cron runs should still use `--session main`, `--session current`, or `--session session:<id>`.
- `pnpm docs:list` could not be run in this checkout because `pnpm` is not installed in the environment, so doc discovery was done by direct file inspection.
