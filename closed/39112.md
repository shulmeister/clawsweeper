## Summary

Keep PR #39112 open. Current main still persists zero usage for native Ollama `/api/chat` responses that omit `prompt_eval_count` / `eval_count`, and the transcript-backed dashboard/session usage path has no later estimator. The submitted patch is stale against the current plugin layout, but the underlying fix remains useful and should be ported.

## Best Possible Solution

Keep this open or replace it with an equivalent current-main fix in `extensions/ollama/src/stream.ts`: estimate native Ollama input/output usage only when counters are absent, preserve exact nonnegative counters including explicit zero, estimate from the converted prompt payload plus accumulated final output, and add focused regression coverage in the current Ollama plugin tests.
