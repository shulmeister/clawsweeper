## Summary

Close as superseded by open PR #66278. Current main still does not implement Chat Completions tool-call streaming, but #66278 is the focused canonical PR explicitly linked from the issue and states that it closes #54174 by adding OpenAI-compatible `tool_calls`, streamed `delta.tool_calls`, and tool-result follow-up handling.

## Best Possible Solution

Use #66278, or a successor with the same scope, as the canonical tracker. The best implementation path is to land Chat Completions tool-protocol support in `src/gateway/openai-http.ts` with regression coverage in `src/gateway/openai-http.test.ts`, reusing the existing Responses API pending-tool-call behavior where appropriate.
