## Summary

Keep open. Current main has Feishu interactive-card plumbing and semantic `presentation` cards through the shared `message` tool, but it does not expose the PR's requested `feishu_card` agent tool or raw Feishu schema-2.0 card send/update surface. The existing shared send path builds a constrained card from `presentation`; raw `card` payload handling exists only on edit, and the proposed `tools.card` toggle would still need current config-schema/default/test work before the PR is merge-ready.

## Best Possible Solution

Keep the PR open for maintainer/product judgment. If OpenClaw wants bundled Feishu raw card operations, update the PR against current main by adding the strict config-schema/type/default/test coverage for `tools.card`, adding account-aware execution consistent with current Feishu tool patterns, and deciding whether the tool should remain a Feishu-specific `feishu_card` tool rather than expanding the shared `message` tool with provider-native card fields. If maintainers decide raw Feishu card authoring is outside the bundled plugin scope, close later with explicit ClawHub/product rationale rather than automated cleanup.
