## Summary

Keep #69458 open. Current main has partial memory-capability lifecycle fixes for cache hits, activate:false snapshot loads, and failed registration rollback, but it still stores the active memory capability inside the scope-swapped memoryPluginState and clears that state before active scoped reloads. That means the central OC-40 behavior described by this PR, preserving the active memory capability when a different plugin scope excludes the memory plugin, is not implemented on main. Related PRs #67063 and #68041 overlap, but neither is merged/currently canonical enough to supersede this useful PR.

## Best Possible Solution

Keep this PR open or consolidate it into a canonical memory-scope fix. The best final patch should preserve the active memory capability across scoped plugin activations that exclude the memory plugin, clear it on true full resets or memory-slot disable/change, normalize the memory slot sentinel, and keep focused regression coverage in loader, memory-state, CLI scope, and memory-wiki bridge tests.
