## Summary

Current `main` already handles missing daily memory files without surfacing `ENOENT` during startup: startup-context loading skips files that cannot be opened, startup guidance now prefers runtime-provided context instead of manual rereads, and memory reads return empty text for missing `memory/*.md` files.

## Evidence
