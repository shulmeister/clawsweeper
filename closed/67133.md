## Summary

Current main and the v2026.4.24 release already cover the reported behavior: cron jobs persist last-run state and durable run history, the Gateway starts and schedules heartbeats, and memory-core self-improvement/dreaming is now reconciled as an isolated cron job instead of depending on a heartbeat-only internal hook path.

## Best Possible Solution

Keep the shipped v2026.4.24 behavior. For users still on the reported 4.11-era setup, the practical path is to upgrade and run the current doctor/migration path for stale main-session dreaming cron jobs rather than keep this core issue open.
