## Summary

Current `main` already short-circuits `message send --dry-run`, exits non-zero on send failures, and exposes `--json` output for `openclaw message`. The same code is present in the shipped `v2026.4.23` release.

## Evidence

- **Dry-run returns before delivery:** `sendMessage()` returns a dry-run result before entering either the direct-delivery branch or the gateway send call, so `--dry-run` no longer posts a real message. The same block is present in `v2026.4.23`.

  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Dry-run regression coverage:** Outbound send coverage asserts that dry-run sends skip plugin dispatch and pass `dryRun: true` into the send path.
  - file: [src/infra/outbound/outbound-send-service.test.ts:549](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/infra/outbound/outbound-send-service.test.ts#L549)
