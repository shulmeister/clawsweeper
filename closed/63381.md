## Summary

Keep open. Current `main` still matches the reported regression: Telegram native exec approvals auto-enable when approvers are resolvable, the Telegram monitor registers the native approval runtime, gateway auth/connect failures during handler startup are surfaced as startup errors, and the bootstrap layer still retries any startup failure every 1 second. There is no evidence on `main` of a guard that would stop the auth-failure retry loop described in #63381.
