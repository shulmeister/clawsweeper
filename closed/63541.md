## Summary

Current `main` no longer crashes when packaged npm installs lack `qa/scenarios/index.md`: `readQaScenarioPack()` now returns an empty fallback pack, downstream discovery code tolerates missing scenario config, and the changelog records the packaging/startup/completion-cache fix as shipped in `v2026.4.12`.

## Evidence

- **Missing QA pack now falls back instead of throwing:** `readQaScenarioPack()` returns an empty pack with a fallback kickoff message when `qa/scenarios/index.md` is absent, replacing the crash path described in the issue.
  - file: [extensions/qa-lab/src/scenario-catalog.ts:331](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/qa-lab/src/scenario-catalog.ts#L331)
  - command: `nl -ba extensions/qa-lab/src/scenario-catalog.ts | sed -n '331,346p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Registration-time discovery now tolerates missing scenario config:** `readRequiredDiscoveryRefs()` reads optional execution config and falls back to hard-coded refs via `config?.requiredFiles ?? [...]`, so command-group registration no longer requires the repo-only QA scenario pack to exist.
  - file: [extensions/qa-lab/src/discovery-eval.ts:4](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/qa-lab/src/discovery-eval.ts#L4)

- **Packaged private QA facade missing is treated as unavailable, not fatal:** The plugin-sdk QA facade catches missing packaged `qa-lab/cli.js` surfaces and returns `false` from `isQaLabCliAvailable()` instead of throwing during CLI setup.
  - file: [src/plugin-sdk/qa-lab.ts:16](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/plugin-sdk/qa-lab.ts#L16)
  - command: `nl -ba src/plugin-sdk/qa-lab.ts | sed -n '16,37p'`
