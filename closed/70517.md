## Summary

Current `main` already contains a Telegram webhook-path fix that acknowledges webhook requests before processing updates, plus update-watermark persistence and regression coverage for long-running Telegram turns. That directly matches the reported voice-message delay pattern, but the fix is newer than release `v2026.4.23`, so `fixedRelease` is unknown/not yet shipped in a tagged release.
