## Summary

PR #65804 is a focused fix for the stale subsystem file-logger cache, but the same remaining rollover work is already tracked by older canonical PR #37549, with #67206 covering the related logger-rebuild/config-path invalidation path. Current main still has the stale cache, so this should close as duplicate/superseded, not implemented.

## Best Possible Solution

Close #65804 as duplicate/superseded and keep maintainer review on #37549 for the midnight rollover fix, with #67206 covering the related `logging.file` reconfiguration invalidation path. The preferred solution should use generation/base invalidation with a regression test and avoid calling config resolution on every file log emit.
