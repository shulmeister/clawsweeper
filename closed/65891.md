## Summary

Close PR #65891 as implemented on main. Current main already ignores the generated A2UI bundle files in run-node watched-tree relevance checks, the dirty-tree path uses that gate before returning dirty_watched_tree, and a regression test covers A2UI bundle/hash dirt resolving as clean. The same implementation is present in the v2026.4.24 release commit.

## Best Possible Solution

Close PR #65891 as already shipped. Keep the current main implementation and regression test; no further core change from this PR is needed. Future generated run-node artifacts should be handled through the same explicit ignoredRunNodeRepoPaths gate with focused regression coverage.
