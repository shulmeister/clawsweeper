## Summary

Keep open. Current main still does not implement the PR's central behavior: non-Azure custom provider onboarding still writes new custom models with `input: ["text"]`, so image support for vision-capable custom model IDs remains disabled unless the user edits config. The PR may need changes because bot review raised correctness/security concerns, but the underlying bug is still real and not shipped on main.

## Best Possible Solution

Keep the PR open or ask the contributor to revise it. The best path is a core onboarding fix for #51869 that either explicitly asks whether a custom model supports image input or derives capability from a trusted provider/model metadata source, preserves deliberate text-only configs on re-onboard, and adds regression tests for non-Azure custom vision models plus text-only/customized preservation paths.
