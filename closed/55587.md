## Summary

Keep #55587 open. Current main still has the reported false-negative path: `readLaunchAgentRuntime()` marks every non-zero `launchctl print` result as `missingUnit: true` before checking whether the LaunchAgent plist exists, and the status printer still turns `missingUnit` into `Service unit not found.` plus install guidance.

## Best Possible Solution

Keep the issue open and fix `src/daemon/launchd.ts` so `readLaunchAgentRuntime()` checks `launchAgentPlistExists(env)` when `launchctl print` fails. If the plist exists, return a non-missing runtime state such as `unknown` or `stopped` with the launchctl detail preserved; only set `missingUnit` when the plist is absent. Add focused tests for both existing-plist and missing-plist print failures so `gateway status` no longer recommends reinstall during launchctl gaps.
