## Summary

Current main still has the reported bug: `runImagePrompt()` passes a hardcoded 30 second timeout through all image-description paths, while the typed/configured contract exposes `tools.media.image.timeoutSeconds`. This PR remains a useful implementation candidate because it wires the existing timeout resolver into the image tool and adds regression coverage. It should be reconciled with the older overlapping #63046 rather than auto-closed.

## Best Possible Solution

Keep this PR open and use it, or one reconciled successor, as the canonical fix for #62944: compute the image-tool timeout with the existing `resolveTimeoutMs` helper from `tools.media.image.timeoutSeconds` with a 30 second fallback, pass it through all three `describeImage`/`describeImages` branches, keep the added regression tests, and then close or supersede #63046 once maintainers choose the final patch.
