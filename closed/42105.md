## Summary

Keep open. Current main still does not expose or wire `typingTtlSeconds`, and the internal reply typing controller still falls back to its hardcoded 2-minute TTL. The PR is the canonical thread for the already-closed linked issue #41516, so conservative cleanup should not close it as stale, duplicate, or implemented; it needs PR follow-up for docs/help metadata, tests/generated schema, and precedence review before merge.

## Best Possible Solution

Keep #42105 open as the canonical PR for configurable typing TTL. The best path is to rebase it onto current main, wire `typingTtlSeconds` through agent defaults and session config into `createTypingController`, add schema/help/label/generated-schema/docs coverage, and add focused tests for default behavior, configured TTL conversion, and chosen session-vs-agent precedence.
