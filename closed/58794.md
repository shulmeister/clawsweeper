## Summary

Current `main` now guards legacy iMessage self-chat detection with `destination_caller_id`, drops ambiguous `is_from_me=true` DM rows as outbound, aligns DM echo-cache scopes, and includes regression tests for the reported echo-loop cases.

## Evidence

- **Inbound gating now avoids the DM false positive:** `resolveIMessageInboundDecision` only treats a direct thread as self-chat when `destination_caller_id` explicitly matches the normalized sender. When that field is missing or the row is otherwise just an ambiguous DM, `is_from_me=true` falls back to `{ kind: "drop", reason: "from me" }` instead of leaking inbound.
  - file: [extensions/imessage/src/monitor/inbound-processing.ts:210](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/extensions/imessage/src/monitor/inbound-processing.ts#L210)
  - command: `nl -ba extensions/imessage/src/monitor/inbound-processing.ts | sed -n '200,380p'`
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **DM echo scope is aligned on current main:** Inbound DM echo lookups use `buildIMessageEchoScope()` => `${accountId}:imessage:${sender}`, and inbound context routes DM replies to `imessage:${sender}`. Outbound delivery caches sent messages under `${accountId}:${target}`, so DM sends now remember entries under the same scope the inbound path checks.
  - file: [extensions/imessage/src/monitor/inbound-processing.ts:600](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/extensions/imessage/src/monitor/inbound-processing.ts#L600)

- **Regression tests cover the reported shapes:** The current iMessage tests explicitly cover blank/absent `destination_caller_id`, normal DM false positives, and self-chat echo drops, including `drops is_from_me outbound when destination_caller_id is blank and sender matches chat_identifier` and `uses destination_caller_id to avoid DM self-chat false positives`.
