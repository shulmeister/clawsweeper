## Summary

Current main already honors `compat.supportsStore` for `openai-completions`, so this PR's extra payload-stripping wrapper is redundant.

## Evidence

- **Current completions payload builder already gates `store` on compat:** `getCompat()` reads `model.compat.supportsStore`, and `buildOpenAICompletionsParams()` only adds `params.store = false` when that resolved flag is true.
  - file: [src/agents/openai-transport-stream.ts:1501](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/agents/openai-transport-stream.ts#L1501)
- **Current builder omits `store` unless support is enabled:** The payload builder sets `params.store = false` only inside `if (compat.supportsStore)`, which means `compat.supportsStore === false` leaves the field out entirely.
  - file: [src/agents/openai-transport-stream.ts:1694](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/agents/openai-transport-stream.ts#L1694)
- **Regression coverage exists for the no-store path:** Current tests assert that OpenAI-compatible requests omit `store` when `compat.supportsStore` is false.
  - file: [src/agents/openai-ws-stream.test.ts:1841](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/agents/openai-ws-stream.test.ts#L1841)
- **Pinned dependency already contains the same guard:** The repo pins `@mariozechner/pi-ai` 0.70.0, and inspecting that package's published `dist/providers/openai-completions.js` shows the same `if (compat.supportsStore) { params.store = false; }` logic with `getCompat()` reading `model.compat.supportsStore`.
  - file: [package.json:1586](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/package.json#L1586)

## Close Comment

Current main already honors `compat.supportsStore` for `openai-completions`, so this PR's extra payload-stripping wrapper is redundant.

What I checked:

- **Current completions payload builder already gates `store` on compat:** `getCompat()` reads `model.compat.supportsStore`, and `buildOpenAICompletionsParams()` only adds `params.store = false` when that resolved flag is true. (`src/agents/openai-transport-stream.ts:1501`)

So I’m closing this as already implemented rather than keeping a duplicate issue open.
