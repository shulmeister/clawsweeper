## Summary

Current `main` already handles `imsg` `is_from_me` payloads directly, drops normal outbound iMessage rows before dispatch, and ships regression coverage for the reported echo-loop behavior; the fix is also called out in the `v2026.4.23` changelog.

## Evidence

- **`imsg` payloads are parsed with `is_from_me` directly:** The iMessage notification parser validates and keeps the snake_case `is_from_me` field from the raw `imsg` payload, so the current code no longer depends on a later `fromMe` normalization step.
