## Summary

Keep open. Current `main` does not implement a `talk.speechLocale`-style config, iOS Talk Mode still creates `SFSpeechRecognizer()` with the default locale, and macOS Talk Mode only reuses the separate app-level `voiceWakeLocaleID` setting rather than a Talk config key.

## Evidence

- **iOS Talk Mode still uses default speech recognizer locale:** `startRecognition()` instantiates `SFSpeechRecognizer()` with no locale override, so there is no configurable non-English Talk Mode locale on iOS.
  - file: [apps/ios/Sources/Voice/TalkModeManager.swift:503](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/apps/ios/Sources/Voice/TalkModeManager.swift#L503)

- **Talk config schema has no speech locale field:** The current Talk config schema only allows `provider`, `providers`, `interruptOnSpeech`, and `silenceTimeoutMs`; no `speechLocale` or equivalent exists on `main`.
  - file: [src/config/zod-schema.ts:184](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/config/zod-schema.ts#L184)
