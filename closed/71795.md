## Summary

Keep #71795 open. Current main still has a source-level path that matches the report: normal Mattermost inbound posts enqueue a `System:`-formatted system event for the same session before dispatch, and the shared reply runner intentionally prepends queued system events into the prompt text. Discord's normal inbound message path uses separate inbound context/untrusted context and does not enqueue an inbound-message system event in the comparable handler.

## Best Possible Solution

Keep the issue open and fix it in the Mattermost plugin path. Normal inbound Mattermost user messages should not enqueue a same-turn `System:` event; sender/channel/message metadata should instead travel through finalized inbound context fields or an explicit untrusted-context/provenance shape like Discord. Add a Mattermost regression test that exercises a normal inbound post and asserts the dispatched prompt/user body does not start with `System:` while preserving clean `BodyForAgent`, command body, sender metadata, and reply routing.
