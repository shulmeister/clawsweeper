## Summary

Current main still leaves `hello-ok.auth` optional in the protocol schema, docs, and generated Swift models even though the gateway handshake now emits `auth` unconditionally. PR #68227 addresses a live protocol-contract gap. Related PR #68161 tracks the same bug, but #68227 is not safely obsolete because it includes broader schema/docs/test/Swift alignment work.

## Best Possible Solution

Keep this open. Land one canonical follow-up that removes the outer `Type.Optional` from `HelloOkSchema.auth`, regenerates/syncs TS and Swift protocol clients, updates protocol docs/examples, and updates all macOS/OpenClawKit fixtures plus regression tests so `hello-ok.auth` is consistently required while token subfields remain optional.
