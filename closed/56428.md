## Summary

Current main already covers the known silent-ignore path: gateway startup now preserves explicitly configured bundled channels like Discord even under restrictive plugin allowlists, and `channels.status` reports from that loaded channel registry, so a configured Discord channel is no longer omitted from `channels status --probe` on this path.

## Evidence

- **Gateway startup keeps explicit bundled channel config eligible:** `resolveGatewayStartupPluginIds` treats explicit bundled channel config as an allowlist bypass, so a configured bundled channel is still selected for gateway startup.
  - file: [src/plugins/gateway-startup-plugin-ids.ts:103](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/plugins/gateway-startup-plugin-ids.ts#L103)
  - command: `nl -ba src/plugins/gateway-startup-plugin-ids.ts | sed -n '81,130p'`
- **Regression test for startup inclusion under restrictive allowlists:** The startup plugin-id tests explicitly assert that bundled channel config is preserved under a restrictive `plugins.allow` list.
  - file: [src/plugins/channel-plugin-ids.test.ts:496](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/plugins/channel-plugin-ids.test.ts#L496)
  - command: `nl -ba src/plugins/channel-plugin-ids.test.ts | sed -n '496,510p'`
- **Read-only presence/status path has the same bundled-channel bypass:** `resolveConfiguredChannelPresencePolicy` also allows explicit bundled channel config to bypass restrictive allowlists for read-only discovery/status surfaces.
  - file: [src/plugins/channel-presence-policy.ts:201](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/plugins/channel-presence-policy.ts#L201)
  - command: `nl -ba src/plugins/channel-presence-policy.ts | sed -n '195,215p'`
- **Regression test for read-only discovery/status inclusion:** The presence-policy tests assert that explicit bundled channel config remains effective and discoverable in read-only scope.
  - file: [src/plugins/channel-plugin-ids.test.ts:1055](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/plugins/channel-plugin-ids.test.ts#L1055)
  - command: `nl -ba src/plugins/channel-plugin-ids.test.ts | sed -n '1055,1090p'`
- **Gateway status reports loaded channel plugins:** `channels.status` builds its response from `listChannelPlugins()`, and gateway startup loads plugins from `resolveGatewayStartupPluginIds`, so once Discord is selected for startup it appears in status/probe output.
  - file: [src/gateway/server-methods/channels.ts:162](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/gateway/server-methods/channels.ts#L162)
  - command: `nl -ba src/gateway/server-methods/channels.ts | sed -n '157,166p'; nl -ba src/gateway/server-plugins.ts | sed -n '477,499p'`
