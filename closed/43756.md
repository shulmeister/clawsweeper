## Summary

Keep open. Current main still lacks typed Slack `threadBindings` config and Slack-owned binding manager/startup wiring for `spawnAcpSessions`; the only Slack support present is generic consumption of a runtime conversation binding if some adapter has already registered one. The provided maintainer comment on #43756 also explicitly confirms the same current-main gap and notes that #35345 and #31518 were stale-closed rather than merged/completed.

## Evidence
