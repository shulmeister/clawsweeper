## Summary

Keep open. Current main still runs pre-compaction memory flush through the normal embedded prompt path with `silentExpected: true` but no `transcriptPrompt: ""` or equivalent internal-only boundary, while docs still describe the flush as silent. The linked #51421/#12590 work is about flush cadence, not this transcript-visibility leak.

## Best Possible Solution

Keep this open as the core memory-flush transcript leak tracker. The best fix is to make the pre-compaction memory flush a truly internal turn, likely by passing `transcriptPrompt: ""` from `src/auto-reply/reply/agent-runner-memory.ts` or by adding an explicit internal-run API, then add regression coverage proving the session JSONL/UI history never contains a user message with the flush prompt. Also add compaction-side sanitization for historically leaked flush prompts. Treat #51421/#12590 as the separate flush-cadence bug unless that PR is expanded to cover transcript visibility.
