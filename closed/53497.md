## Summary

Keep #53497 open. Current main still intentionally keeps the bare `openclaw/plugin-sdk` root runtime surface tiny and does not export the weixin-requested helpers from that root entry. The helpers exist on focused SDK subpaths/core surfaces, and docs/tests now steer plugin authors toward those subpaths, so this remains a real compatibility/product decision rather than an already-implemented fix. The latest maintainer discussion also explicitly keeps this issue as the canonical weixin/top-level-barrel compatibility report, with #60416 closed into it and the stringEnum cluster tracked separately in #68279.

## Best Possible Solution

Keep #53497 open as the canonical weixin top-level plugin-SDK compatibility issue. Maintainers should decide explicitly between adding legacy-compatible root exports with API baseline and contract coverage, or coordinating an `@tencent-weixin/openclaw-weixin` release that migrates to focused `openclaw/plugin-sdk/<subpath>` imports. Track stringEnum-only failures in #68279 and leave #60416 closed as a duplicate of this weixin report.
