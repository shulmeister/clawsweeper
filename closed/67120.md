## Summary

Keep #67120 open. Current main recognizes Feishu inbound audio as media and can hand attachments to the shared media-understanding path, but the Feishu channel still does not extract Feishu's `speech_to_text` field or run a channel-owned inbound audio preflight that replaces the agent-facing body before mention gating and dispatch. A maintainer comment on the issue explicitly distinguished this from the already-added media download/attachment support and asked to keep the item open.

## Best Possible Solution

Keep the issue open and implement it inside the bundled Feishu plugin. The best fix is to prefer Feishu's native `speech_to_text` field for `audio` message content when present, and add a Feishu-owned fallback audio preflight that downloads the `file_key`, calls the shared `transcribeFirstAudio` helper, and replaces the agent-facing body before mention gating/dispatch. Add focused Feishu tests for native `speech_to_text`, downloaded audio fallback, and group `requireMention` behavior so this is not only covered by the later shared media-understanding pipeline.
