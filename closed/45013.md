## Summary

Keep PR #45013 open. Current main has moved Ollama memory embeddings into the bundled Ollama plugin, but it still sends `embedQuery()` text to `/api/embeddings` unchanged and does not apply model-specific query instructions for `qwen3-embedding`, `nomic-embed-text`, or `mxbai-embed-large`. The PR is still useful, though it needs rebasing onto the current plugin file and should address the review comments about preserving literal `$` sequences in query text.

## Best Possible Solution

Keep the PR open and rework it against `extensions/ollama/src/embedding-provider.ts`: add an Ollama-local query-prefix helper, route only `embedQuery()` through it, leave `embedBatch()` document prompts unchanged, add regression tests for known models, unknown models, batch behavior, and literal `$` preservation, then decide whether docs should mention the model-specific behavior.

## Risks / Open Questions

- The PR patch targets the old `src/memory/embeddings-ollama.ts` path; current main uses `extensions/ollama/src/embedding-provider.ts`, so the work needs a rebase rather than cleanup closure.
- The proposed template replacement must avoid JavaScript replacement-token expansion for user queries containing `$&`, `$'`, `$1`, or similar sequences.
- Adding hard-coded model-prefix behavior should preserve current raw document embedding behavior and avoid changing unknown Ollama models.
