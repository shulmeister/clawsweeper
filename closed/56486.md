## Summary

Current `main` already handles the Discord stale-socket restart crash path by keeping gateway error handling attached through disconnect/teardown and explicitly suppressing late `reconnect-exhausted` events, so this PR's titled fix is no longer needed on top of `main`.

## Evidence

- **Abort/teardown path keeps listeners attached through disconnect:** `waitForDiscordGatewayStop` resolves aborts by calling `gateway.disconnect()` first and only then detaching lifecycle listeners; the inline comment states this is to catch late `error` events during disconnect instead of letting them become uncaught.
