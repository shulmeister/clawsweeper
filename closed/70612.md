## Summary

Current `main` already contains a targeted fix for the stale managed-env rewrite path behind `openclaw gateway install --force`. The install flow now reuses the existing service env only as input, parses `OPENCLAW_SERVICE_MANAGED_ENV_KEYS`, and drops previously managed keys instead of copying literal secret values back into the rewritten service config. A regression test on `main` covers that exact behavior. The fix is present on `main` after `v2026.4.23`, so there is no released tag I can prove yet.

## Evidence
