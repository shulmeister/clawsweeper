## Summary

ANI support is a valid OpenClaw channel plugin idea, but current `main` explicitly pushes new optional plugins/channels to external ClawHub or npm distribution instead of bundling them in `openclaw/openclaw`.

## Evidence

- **Project direction prefers external plugin ownership:** `VISION.md` says optional capability should usually ship as plugins, plugin authors should host and maintain plugins in their own repository, the bar for adding optional plugins to core is intentionally high, and plugin promotion belongs in ClawHub.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/VISION.md#L54)

  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Docs explicitly say new plugins do not need to be added to this repo:** `docs/plugins/building-plugins.md` tells authors to publish plugins to ClawHub or npm instead of adding them to the OpenClaw repository, and calls out channel plugins as a supported plugin type.
  - file: [docs/plugins/building-plugins.md:16](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/plugins/building-plugins.md#L16)

  - file: [docs/plugins/sdk-setup.md:486](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/plugins/sdk-setup.md#L486)
