## Summary

Keep #70238 open. Current main still has an OpenClaw-owned TUI relaunch spawn without `windowsHide`, the Windows CLI respawn path is skipped so the earlier `src/entry.ts` diagnosis is not sufficient by itself, and bundled Bonjour discovery remains enabled by default through `@homebridge/ciao`. The later PR #72332 explicitly tracks this issue but is still open/unmerged in the provided context, so the regression is not implemented on main yet.

## Best Possible Solution

Keep the issue open until a Windows-focused fix lands. The best path is to identify the exact process creating the flashing window, add `windowsHide: true` to OpenClaw-owned background spawn surfaces that can run on Windows, handle or patch the Bonjour/@homebridge/ciao path if it is the source, and add mockable regression coverage for the spawn options plus a documented temporary Bonjour-disable workaround only if needed.
