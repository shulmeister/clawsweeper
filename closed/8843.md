## Summary

Current main already solves the reported Telegram audio/document attachment download problem by auto-downloading Telegram media via getFile and exposing local MediaPath/MediaPaths in the inbound agent context. The exact file_id-in-placeholder/download-file UX from PR #38170 is not on main, but the issue explicitly allowed auto-download plus local path as an alternative.

## Evidence

- **Telegram media resolver covers audio and documents:** resolveMediaMetadata selects document and audio Telegram file refs, preserves file_name/mime_type, then resolveMedia calls getFile, downloads the file, saves it under the inbound media store, and returns the local path/content type.
