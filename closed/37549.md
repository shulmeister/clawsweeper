## Summary

PR #37549 can close as implemented on current main. The PR correctly identified a stale subsystem file-child logger bug, but main now solves the observable rollover problem more broadly by removing the cached subsystem file logger path and resolving the active rolling file at write time, with regression coverage for long-lived loggers crossing a date boundary.

## Best Possible Solution

Close this PR without merging its generation-counter implementation. Keep the broader main implementation that refreshes subsystem file children and makes the root file transport follow the active rolling file; any remaining logging rollover regression should be reported as a new current-main issue with reproduction details.
