## Summary

Keep open. Current `main` fixes the narrower OpenAI Codex `openai-codex:default` collision by deriving email or a stable fallback id from the OAuth token, but the requested feature itself is still not implemented: `openclaw models auth login` has no `--profile-id` flag and the generic provider auth context has no `profileId` plumbing.

## Evidence

- **CLI surface lacks the requested flag:** `models auth login` only defines `--provider`, `--method`, and `--set-default`. The neighboring `paste-token` command is the one that exposes `--profile-id`, matching the issue's comparison.
  - file: [src/cli/models-cli.ts:301](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/src/cli/models-cli.ts#L301)

- **Login command plumbing has no profile-id input:** `LoginOptions` has no `profileId`, and `runProviderAuthMethod(...)` calls the provider auth method without any profile-id field in the context object.
  - file: [src/commands/models/auth.ts:534](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/src/commands/models/auth.ts#L534)
