## Summary

Keep PR #46406 open. Current main already has the host=node exec behavior, but the PR’s session-level bootstrap cache invalidation half is still missing, and the lower-level workspace mtime cache is bypassed on `getOrLoadBootstrapFiles` cache hits.

## Best Possible Solution

Keep the item open, but update or supersede it with a narrower bootstrap-cache fix: preserve the current main exec implementation, add session-level staleness detection above `loadWorkspaceBootstrapFiles`, include coverage for mtime changes and missing-file-to-created transitions, then close #46406 only after that bootstrap work lands or a canonical replacement PR exists.
