## Summary

Current main already implements configurable LLM idle timeout handling for slow models: the idle watchdog is configurable via `agents.defaults.llm.idleTimeoutSeconds`, `0` disables it, unconfigured cron runs disable the watchdog entirely, and the timeout docs/tests/changelog no longer match the reported baked-in 60s behavior. Changelog entries show the timeout alignment shipped by v2026.4.11.

## Evidence
