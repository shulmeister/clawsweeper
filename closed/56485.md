## Summary

Current `main` already covers both reported problems: Control UI now sends image attachments through `chat.send`, Gateway normalizes/parses them, and the current `/new` path is a single `chat.send` call guarded by Gateway idempotency/in-flight dedupe.

## Evidence

- **Control UI `/new` path is a single send:** `dispatchSlashCommand` routes `/new` through one `sendChatMessageNow(...)` call, and `sendChatMessageNow` calls `sendChatMessage(...)` once.
  - file: [ui/src/ui/app-chat.ts:417](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/ui/src/ui/app-chat.ts#L417)

- **No client retry loop in current chat send path:** `requestChatSend` performs one `client.request("chat.send", ...)` with one generated `idempotencyKey`; there is no resend/backoff loop in the send path.
  - file: [ui/src/ui/controllers/chat.ts:232](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/ui/src/ui/controllers/chat.ts#L232)
