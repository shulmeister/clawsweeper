## Summary

The PR's repro path is obsolete on current `main`. It targets `gaxios@7.1.3` under pnpm strict/isolated linking, but the repo now forces a hoisted pnpm layout and currently resolves the Google auth chain through `gaxios@7.1.4` without any root gaxios patch. Current `main` also uses a scoped native-fetch Google auth transport where needed, with proxy/mTLS-preserving behavior and tests, so this repo-wide `globalThis.fetch` preference patch is no longer the right fix for the current tree.
