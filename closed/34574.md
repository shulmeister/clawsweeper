## Summary

Keep open. Current main still lacks the exec-specific stable outcome hash needed for this report, so repeated successful exec calls can still evade the no-progress/global circuit breaker when volatile exec metadata changes between runs. The issue also has recent user reports and an open canonical PR path for the narrow volatile-metadata fix.

## Best Possible Solution

Keep #34574 open until main lands an exec-specific stable outcome hash for successful exec results, with regression tests proving volatile exec metadata does not reset no-progress detection. Treat #41502 as the current canonical narrow fix path; keep genericRepeat critical escalation and varied-command same-result detection tracked separately through #54559 or a dedicated result-similarity follow-up.
