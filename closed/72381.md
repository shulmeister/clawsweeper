## Summary

Keep this PR open. Current main still lacks the Google Meet `realtime.agentId` config/schema surface and does not pass an agent id into Google Meet realtime consults, while the shared realtime consult runtime and Voice Call already support agent routing. The related #72372 is complementary bridge cleanup, not a superseding fix.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or refine the narrow Google Meet plugin fix: add `realtime.agentId` to config resolution and manifest/schema metadata, normalize it through the existing routing helper, pass it into `consultRealtimeVoiceAgent`, scope consult sessions by agent id, and keep the resolver plus embedded-agent invocation tests. A short Google Meet docs update for the advanced key would also be useful.
