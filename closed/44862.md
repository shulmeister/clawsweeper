## Summary

Current `main` already routes bootstrap resolution through the run’s canonical workspace, which covers the underlying subagent/bootstrap mismatch without the PR’s `agentId`-based override. The open PR’s approach is now stale relative to the current workspace-selection flow and would conflict with intentional spawned-workspace inheritance.

## Evidence

- **Workspace selection now happens before bootstrap resolution:** `src/agents/agent-command.ts:349-355` explicitly lets internal callers pin `normalizedSpawned.workspaceDir`, otherwise falls back to `resolveAgentWorkspaceDir(cfg, sessionAgentId)`. That means the run already carries the correct workspace choice before bootstrap loading starts.
