## Summary

Keep open. Current `main` still does not bypass `/acp ...` commands in ACP-bound sessions, even though the docs and existing ACP command tests expect `/acp status` and `/acp cancel` to work as local controls inside bound conversations. The related-work note pointing to PR #66407 reinforces that this is a real duplicate-worthy gap, not an already-landed fix.
