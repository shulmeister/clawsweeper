## Summary

Keep #59477 open. Current main has not implemented the requested `sessions_send` behavior: sandboxed label lookups still return the same-agent error before the A2A policy can allow the target, and later resolution still carries spawned-session restrictions. Existing docs/tests also make sandbox cross-agent visibility a policy boundary, so this needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review. Decide whether authorized A2A should bypass the sandbox spawned-session clamp for `sessions_send`, then implement that policy consistently across label and direct `sessionKey` resolution, final visibility checks, tests, and docs.
