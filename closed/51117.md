## Summary

Current main has not implemented the requested direct config behavior: Feishu still forces `disableBlockStreaming: true`, and a current test asserts that this is intentional to avoid silent reply drops. The remaining Feishu block-streaming capability is now tracked more accurately by open issue #55027, which supersedes this narrower config-report and the two closed unmerged PR attempts.
