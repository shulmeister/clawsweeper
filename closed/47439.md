## Summary

The request is useful, but the remaining work is a Telegram-specific optional routing policy for reasoning output rather than a missing OpenClaw core capability. Current main supports `/reasoning on|off|stream` and Telegram-specific reasoning/answer lanes, but those lanes deliberately deliver through the same configured Telegram account/chat. The project vision and channel-plugin docs put optional channel behavior in plugins/ClawHub unless a core API gap is shown.
