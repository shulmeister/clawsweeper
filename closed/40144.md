## Summary

Keep open. Current main still retains and evaluates per-session `toolCallHistory` across heartbeat intervals with no stale-time pruning, and heartbeats still use the main session unless `isolatedSession` is explicitly enabled. The related fix PRs in the provided context are either open/unmerged or closed unmerged, so this is not implemented or safely superseded on main.

## Best Possible Solution

Keep #40144 open until a focused mainline fix lands. The fix should prune or ignore `toolCallHistory` entries older than a conservative inactivity gap before `detectToolCallLoop` evaluates the current call, and keep `recordToolCall`/`recordToolCallOutcome` consistent with that pruning. Add regression coverage that simulates repeated heartbeat cycles separated by the stale gap, plus a control proving continuous in-turn repeats still warn or block.
