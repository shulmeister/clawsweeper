## Summary

Keep open. Current main still does not merge channel-manager runtime state into the gateway health snapshot, while channels.status does use that runtime snapshot. The PR addresses a real remaining discrepancy from #42538 rather than cleanup-close material.

## Best Possible Solution

Keep the item open and adapt the focused fix: thread the channel manager runtime snapshot into the health snapshot/cache refresh path, merge runtime fields before authoritative account/config/probe fields, and add a regression test proving health and channels.status agree on running/connected for a runtime-backed channel account.
