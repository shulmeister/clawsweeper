## Summary

Keep #38958 open. Current main at 8ba82534e6c98410e19b226f3534b285fd75d278 still lacks the remaining Feishu runtime behavior from the PR: DM display-name fallback through chatMembers, resolved human-readable group labels, root/topic-derived thread labels, Feishu-specific Unicode group display labels, and account-scoped sender/direct/group cache behavior. The related closed PRs were not merged, and #51032 only overlaps the group-name subset.

## Best Possible Solution

Keep #38958 open for maintainer review, or land an equivalent focused Feishu implementation on main: use chatMembers for DM display-name fallback with contact fallback and permission notices, scope caches by account and relevant chat/sender ids, resolve group names after admission checks while preserving resolveSenderNames opt-out semantics, preserve human-readable Feishu group labels in session display names, and derive topic/root labels only for topic-scoped sessions. If maintainers prefer #51032 for the narrower group-label work, keep a separate canonical item for the DM and topic-label behavior before closing this PR.
