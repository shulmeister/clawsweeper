## Summary

Current `main` already solves the chat-run identity problem more robustly by forcing `chat.send` runs to use the client run id from the start, so the PR's later `addChatRun()` hook is redundant.

## Evidence

- **chat.send now pins agent events to the client run id up front:** The gateway chat handler passes `runId: clientRunId` into `replyOptions` before dispatch, so downstream agent events use the chat/client id directly rather than needing a later registry mapping.
