## Summary

Keep #70554 open. Current main still implements the old memory-core dreaming behavior: a request-scoped subagent runtime error writes a local fallback narrative instead of retrying through a gateway-owned background subagent, and the plugin subagent dispatch path still cannot carry the cron run kind that the PR is trying to preserve.

## Best Possible Solution

Keep #70554 open as the active implementation path for #67152. The best maintainer path is to keep the fix narrow in memory-core plus the generic gateway subagent/run-kind seams, preserve bootstrapContextRunKind=cron through both CLI and embedded execution, avoid exposing mutable gateway runtime handles through broad public SDK barrels, and retain focused regression coverage for retry, fallback cleanup, and non-transient error handling.
