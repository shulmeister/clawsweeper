## Summary

Current main implements the Anthropic setup-token 401 fix. The original 2026.3.23-2/2026.4.5 reports were valid, and PR #54393 was useful investigation, but the current Anthropic Messages transport now detects `sk-ant-oat*` tokens, sends them as Bearer auth with the Claude/OAuth headers, routes embedded `anthropic-messages` runs through that transport, and has regression coverage for the exact header shape. The same implementation is present in the latest release tag `v2026.4.24`.

## Best Possible Solution

Close #54350 as implemented. Keep the current boundary-aware Anthropic Messages transport and regression tests; any remaining Anthropic setup-token onboarding, sync, or headless UX issues should stay tracked by narrower setup-token follow-ups rather than this request-construction 401 report.
