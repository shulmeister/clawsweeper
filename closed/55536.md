## Summary

Keep #55536 open. Current main has substantial auth-profile cooldown and model fallback behavior, but it does not implement the requested provider/model circuit breaker with configurable failure threshold/cooldown. The issue's concrete 503/service-unavailable case is specifically covered by an existing test that expects profile rotation without setting cooldown state, so repeated later requests can still retry the failing model/profile.

## Best Possible Solution

Keep #55536 open and either implement or explicitly scope a true model-level circuit breaker: persist breaker state keyed by provider/model, count consecutive failover-worthy API failures, expose threshold/cooldown/recovery config, reset on successful calls, surface state in diagnostics/models status, and integrate it with runWithModelFallback without weakening existing auth-profile rotation. PR #55736 can be considered related, but it should not be treated as the canonical fix unless it is revised to cover the provider/model breaker semantics.
