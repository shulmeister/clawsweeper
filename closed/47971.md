## Summary

Current `main` already preserves `threadId` through `resolveAnnounceTarget()` and forwards it on the `sessions_send` announce `send` call, with regression coverage for threaded session keys and fallback session metadata. The same implementation is present in release `v2026.4.22`.

## Evidence

- **Thread-aware announce target parsing is implemented:** `resolveAnnounceTargetFromKey()` parses the session conversation ref, passes `threadId` into plugin target resolution, and returns it on the announce target.
  - file: [src/agents/tools/sessions-send-helpers.ts:26](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/agents/tools/sessions-send-helpers.ts#L26)
