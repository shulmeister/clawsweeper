## Summary

Keep #69464 open. Current main at 998e37fcb3d5b937fcf70d7509b677efba939f10 still does not implement the PR's central fix: provider model `params` are not accepted/threaded onto runtime models, native Ollama still sends `options.num_ctx` from `model.contextWindow`, and the OpenAI-compatible Ollama wrapper still derives `num_ctx` from `contextWindow`/`maxTokens`. The related #44678 is closed but unmerged, and #44550 is closed in GitHub context despite the current code still lacking this wiring.

## Best Possible Solution

Keep #69464 open. The best path is to land or adapt this focused fix after maintainer review: add `params` to provider model schema/types/generated schema, merge configured params onto resolved runtime models with explicit config precedence, and make both native Ollama and Ollama OpenAI-compatible payload builders honor valid `params.num_ctx` with regression tests and docs.
