## Summary

Keep #59681 open. Current main still lets `sessions_spawn` model aliases pass through `resolveSubagentSpawnModelSelection()` as bare strings, then the native subagent spawn path writes and persists that bare value directly on the child session. The generic `sessions.patch` model resolver can resolve aliases, but this direct spawn path bypasses that resolver, so the PR's central bug is not implemented on main.

## Best Possible Solution

Keep the PR or a successor open and land a focused fix in the spawn model-selection path: resolve bare `sessions_spawn.model` and configured subagent model aliases through the existing model alias index using the effective runtime default provider, keep qualified refs and unknown strings behavior consistent with existing model resolution, and add regression tests for explicit alias overrides plus configured subagent aliases through `resolveSubagentModelAndThinkingPlan()` or `resolveSubagentSpawnModelSelection()`.
