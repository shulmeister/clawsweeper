## Summary

Keep #64963 open. Current main still lacks the PR's active reply-run guard in `runHeartbeatOnce`: heartbeat only skips on the global main lane and resolved session lane queue size, not on `resolveActiveReplyRunSessionId(sessionKey)`. The current tests cover queued session-lane work and idle execution, but not the reported window where a reply operation remains active after the lane drains. The linked root bug #64810 is still open, and the prior identical PR #64823 was closed unmerged by automation rather than superseded by a shipped fix.

## Best Possible Solution

Keep the PR open for maintainer review or merge a small equivalent fix in `src/infra/heartbeat-runner.ts`: after heartbeat preflight resolves the target `sessionKey` and before the session-lane queue check, skip with `requests-in-flight` when `resolveActiveReplyRunSessionId(sessionKey)` is truthy. Keep the regression test that seeds an active reply operation while the lane queue is idle, because that is the missing behavior boundary.
