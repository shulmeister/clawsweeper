## Summary

Keep #56297 open. Current main still passes Full-profile tools into native Ollama `/api/chat` for manually configured Ollama models unless the model is explicitly marked `compat.supportsTools: false`; the existing docs describe that as a workaround, not an automatic fix. The related PR #56332 in the supplied timeline was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open and fix the Ollama/local-model compatibility path. The likely product fix is to omit tools for Ollama models/backends known or discovered not to support tool calling, or to surface a targeted doctor/onboarding/config recommendation for `compat.supportsTools: false` on manually configured Ollama GLM/Qwen models that reject tool schemas. Add regression coverage proving Full mode does not send tool schemas when model compat disables tools, and consider capability-based Ollama discovery so missing `tools` capability becomes `compat.supportsTools: false`.
