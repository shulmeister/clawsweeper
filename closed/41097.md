## Summary

Keep #41097 open. Current main has a generic doctor warning for empty group allowlists, but it does not correctly cover the Feishu misconfiguration described by the PR: Feishu runtime checks the group ID against `channels.feishu.groupAllowFrom`, while the generic doctor warning defaults to treating `allowFrom` as a fallback. A Feishu setup with DM `allowFrom` populated but no `groupAllowFrom` can still silently block groups without the requested diagnostic. The PR as submitted is also too dirty to merge directly, so the right path is a narrow Feishu doctor/doctor-capability fix rather than closing as already implemented.

## Best Possible Solution

Keep the item open but ask for a sharply scoped replacement patch: add Feishu-specific doctor capabilities or a Feishu doctor adapter that evaluates the effective top-level/default/named account config exactly like runtime, treats `groupAllowFrom` as the required group-id allowlist, respects disabled channels/accounts, and adds focused tests for `groupPolicy: "allowlist"` with missing `groupAllowFrom` both with and without DM `allowFrom`. The current branch should be rebased or recreated without unrelated Android, skill, CI, binary, and secrets-baseline changes.
