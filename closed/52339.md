## Summary

Current `main` already fixes the reported Docker config-dir regression: `scripts/docker/setup.sh` now defaults Docker paths from `$HOME` instead of `/root`, precreates the config tree, and runs a pre-start ownership repair for the container `node` user. The same implementation is present in release `v2026.4.22`, with e2e coverage for the EACCES path.

## Evidence

- **Docker setup no longer defaults to /root:** `scripts/docker/setup.sh:235` sets `OPENCLAW_CONFIG_DIR="${OPENCLAW_CONFIG_DIR:-$HOME/.openclaw}"` and `scripts/docker/setup.sh:236` sets the workspace under `$HOME/.openclaw/workspace`, so the old `/root/.openclaw` default described in the issue is not present on current `main`.
  - file: [scripts/docker/setup.sh:235](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/scripts/docker/setup.sh#L235)
