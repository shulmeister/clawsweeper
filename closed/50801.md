## Summary

Keep PR #50801 open. Current main still treats Feishu `@_all` as a bot mention: `checkBotMentioned()` returns `true` when raw content contains `@_all`, `parseFeishuMessageEvent()` stores that as `ctx.mentionedBot`, and group `requireMention` gating then allows the message. Current tests cover mention detection but have no `@_all` regression case. The PR's provided diff directly removes the false-positive and adds targeted tests, so it is still useful work rather than already implemented on main.

## Best Possible Solution

Keep the PR open and review it as a live Feishu channel bug fix. The minimum code path to fix is `extensions/feishu/src/bot-content.ts` by removing the unconditional raw-content `@_all` success path and adding `@_all` regression coverage; if maintainers prefer an explicit opt-in `respondToAtAll` / `respondToAll` setting, reconcile that product choice with the related follow-up PRs rather than closing this as obsolete.
