## Summary

Current `main` already routes context-window/footer accounting through the final-call snapshot (`lastCallUsage` / `lastTurnTotal`) instead of the accumulated tool-loop totals, and the same logic is present in the shipped `v2026.4.23` release.

## Evidence

- **Runner keeps the latest-call prompt snapshot:** After each attempt, the runner still accumulates usage for totals, but separately stores `lastRunPromptUsage` and `lastTurnTotal` from the latest assistant call, with an inline comment that session `totalTokens` should reflect current context usage rather than accumulated tool-loop usage.
