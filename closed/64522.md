## Summary

Current `main` already addresses this report. The QA scenario catalog now treats missing packaged QA markdown as optional instead of throwing, the completion command catches subcommand registration failures so `openclaw completion --write-state` can continue, and the changelog says this shipped in `v2026.4.11`.

## Evidence

- **Missing QA pack no longer crashes packaged builds:** `readQaScenarioPack()` now returns an empty scenario pack when `qa/scenarios/index.md` is absent, with an inline comment explicitly calling out npm/package distributions and completion-cache consumers.

  - command: `nl -ba extensions/qa-lab/src/scenario-catalog.ts | sed -n '330,350p'`
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Completion generation skips failing subcommands instead of aborting:** `registerSubcommandsForCompletion()` wraps each subcommand registration in `try/catch` and logs a skip warning, which directly prevents `openclaw completion --write-state` from crashing when `qa` registration fails.
  - file: [src/cli/completion-cli.ts:55](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/cli/completion-cli.ts#L55)
