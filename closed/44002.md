## Summary

Keep open. Current main has adjacent cron/subagent fixes, but it has not implemented this PR's core current-run-window scoping for isolated cron descendant waits. The PR should not be merged as-is because it also contains an unrelated gateway restart script change with a live review concern, but the cron fix remains a plausible useful patch to rebase and narrow.

## Best Possible Solution

Keep the PR open or replace it with a maintainer-clean patch that ports only the cron isolated descendant run-window behavior: make delivery dispatch count active descendants from listDescendantRunsForRequester filtered by params.runStartedAt, pass runStartedAt into waitForDescendantSubagentSummary, add stale-only and in-window-active regression tests, and drop the unrelated gateway restart script changes.
