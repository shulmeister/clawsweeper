## Summary

Current `main` already implements this: cron isolated sessions get a human-readable `label` derived from the cron job name, and the Control UI session selector prefers `row.label` over the raw session key. The fix was shipped with the 2026.3.22 session-dropdown label work and the reporter later confirmed the behavior in 2026.3.23.

## Evidence

- **Cron runs assign a label from the job name:** `runCronIsolatedAgentTurn` sets `cronSession.sessionEntry.label = \`Cron: ${labelSuffix}\`` when the base session key is a cron session, using `job.name` when present.
