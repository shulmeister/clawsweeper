## Summary

PR #58390 adds a useful DInference model-provider plugin, but it is an optional third-party provider integration that can be shipped outside core with the current plugin/provider SDK. Current main has no DInference provider, and the PR discussion shows an additive plugin implementation with review cleanup addressed, not a missing OpenClaw core API or maintainer-protected need to bundle it.

## Best Possible Solution

Move the DInference provider to an external ClawHub/npm plugin using the current provider SDK, with the existing PR code as a starting point. Users who need it before a plugin is published can configure DInference as a custom OpenAI-compatible provider under models.providers; bundling it in core should wait for explicit maintainer product/security ownership.
