## Summary

Keep open. Current main still matches the reported QMD dirty-status bug: QMD watcher events can set an internal dirty flag, but `QmdMemoryManager.status()` always reports `dirty: false`, and `openclaw memory status` renders that value directly as `Dirty: no`.

## Best Possible Solution

Fix the QMD status contract in `extensions/memory-core`: `QmdMemoryManager.status()` should report dirty state for full managers, and status-only `openclaw memory status` needs a deterministic freshness check for `MEMORY.md`, `memory/**/*.md`, and configured QMD paths rather than always returning clean. Add QMD-focused tests for changed memory files causing `Dirty: yes` and for `status --index` behavior against a dirty QMD store.
