## Summary

Close as `clawhub`. Cortex is a useful idea, but current OpenClaw direction and public extension points already place optional memory/context-assembly systems in external plugins rather than core.

## Evidence

- **Vision scope anchor:** Repo vision says core should stay lean, optional capability should usually ship as plugins, plugin authors should host and maintain those plugins in their own repos, and memory is already a dedicated plugin slot.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/VISION.md#L54)
- **Memory is already a plugin surface:** The memory CLI docs say semantic memory is provided by the active memory plugin, with `plugins.slots.memory` selecting or disabling it.
  - file: [docs/cli/memory.md:12](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/docs/cli/memory.md#L12)
- **Context assembly is already a plugin surface:** The context-engine docs tell users to install/select a plugin engine when they want different assembly, compaction, or cross-session recall behavior.
  - file: [docs/concepts/context-engine.md:14](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/docs/concepts/context-engine.md#L14)
- **SDK supports this shape externally:** The public SDK exposes exclusive plugin slots for `registerContextEngine(...)` and `registerMemoryCapability(...)`, which matches the core of this Cortex proposal.
  - file: [docs/plugins/sdk-overview.md:195](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/docs/plugins/sdk-overview.md#L195)
- **ClawHub distribution path exists for plugins:** Plugin docs explicitly direct external plugins to publish to ClawHub or npm, with `openclaw plugins install clawhub:<package>` as the supported install path.
  - file: [docs/plugins/sdk-setup.md:521](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/docs/plugins/sdk-setup.md#L521)

## Risks / Open Questions

- If Cortex needs one missing SDK seam, a narrow plugin-enabling core PR may still be warranted.
- This is a scope decision, not a statement that the Cortex feature itself is bad or unworkable.
