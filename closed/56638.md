## Summary

Keep open. Current `main` still makes the reported surface plausible: the shared `message` tool exists and supports media/`asVoice`, but it is only in the `messaging` profile, while `tts` is not in the `coding` profile and local onboarding now defaults new setups to `tools.profile: "coding"`. That means the issue is not obsolete on current `main`; it looks like a real tool-profile exposure/UX gap rather than a closed-by-main bug.
