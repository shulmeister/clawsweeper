## Summary

Keep open. Current `main` still shows the ACPX compatibility path as intentionally ignored rather than fixed: `queueOwnerTtlSeconds` is parsed but not passed into the embedded runtime, the plugin manifest labels it as a reserved ignored field, and the only implemented command override path is `agents.<id>.command`, not a top-level ACPX backend `config.command` override. That is not strong evidence that issue #66389 is resolved.
