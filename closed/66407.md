## Summary

Keep open. Current `main` and the latest shipped release still do not bypass ACP dispatch for `/acp ...` in bound threads, and the in-tree regression tests still lock that buggy behavior in. The PR discussion also shows substantive follow-up work to cover the `commands.text: false` path, so this is still useful review material rather than a stale or already-implemented change.

## Evidence

- **Current main still excludes `/acp` from the ACP-dispatch bypass:** `shouldBypassAcpDispatchForCommand` special-cases `/new` and `/reset`, then returns `false` for any non-`!` slash command. There is no `/acp` candidate check on `main`, so `/acp close` still will not bypass ACP dispatch.
