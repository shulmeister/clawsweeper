## Summary

Current `main` no longer matches the reported inline/default dreaming failure: dreaming now defaults to `separate`, inline markdown/event writes are explicitly implemented and covered by tests, and managed dreaming cron jobs are migrated to the newer isolated `agentTurn` shape so stale cron payloads do not silently no-op.

## Evidence

- **Issue timeline:** The report immediately cross-referenced #68468 ('Change default dreaming storage.mode from inline to separate'). The current codebase follows that direction by resolving dreaming storage to `separate` by default.
  - command: `provided GitHub context`
- **Released default is separate:** `DEFAULT_MEMORY_DREAMING_STORAGE_MODE` is `"separate"`, and invalid/missing storage values fall back to that default in the dreaming config resolver. This is already present in the latest local release tag.
