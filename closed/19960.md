## Summary

The requested Meta/WhatsApp-style webhook ingress is useful, but it belongs in a ClawHub/plugin implementation rather than OpenClaw core. Current main still keeps core /hooks POST-only, while the plugin API already exposes HTTP routes, request headers, raw body readers, and runtime dispatch helpers for provider-specific webhook verification and handling.
