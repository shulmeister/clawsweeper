## Summary

PR #65376 proposes adding ATOM as a new bundled self-hosted provider plugin. Current main does not ship ATOM, but the requested capability is optional third-party provider work that fits the existing provider-plugin and self-hosted OpenAI-compatible helper surfaces. The PR and linked issue #65811 show a working extension-scoped implementation, so the best cleanup is to move that implementation to a standalone ClawHub/community plugin rather than keep this as an OpenClaw core PR.

## Best Possible Solution

Move the ATOM implementation from PR #65376 into a standalone provider plugin published through ClawHub, using the existing provider SDK and self-hosted OpenAI-compatible setup helpers. Keep OpenClaw core unchanged unless maintainers explicitly decide ATOM should become an official bundled provider or a concrete missing plugin API is identified.
