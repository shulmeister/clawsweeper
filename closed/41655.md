## Summary

Keep open. Current main supersedes the skills-watcher half with a narrower SKILL.md-only watcher, but the memory watcher still has recursive watch behavior with no depth cap, and current tests/docs preserve recursive memory indexing. The PR discussion also identified an unresolved regression risk from `depth: 2`: deeper memory files could stop marking the index dirty and serve stale search results. This is not a conservative close candidate.

## Best Possible Solution

Keep the PR open or consolidate it into a focused memory-watcher follow-up. The best fix is a redesigned memory watcher in `extensions/memory-core` that reduces FD pressure without breaking recursive memory and `extraPaths` indexing, with regression coverage for ignored large directories and deep memory files that must still trigger sync. The current SKILL.md-only skills watcher should be kept.
