## Summary

Current `main` routes Control UI tool-output rendering through guarded JSON detection with raw-text fallbacks, and the relevant renderer shipped in `v2026.4.23` (`a9797214338ba31c52c796adbb75afb16e0684a9`).

## Evidence

- **Sidebar JSON parse is guarded:** `formatToolOutputForSidebar` only attempts `JSON.parse` for `{`/`[`-prefixed text and falls back to returning the original text on parse failure, which covers multiline/special-character tool output instead of crashing the renderer.
  - file: [ui/src/ui/chat/tool-helpers.ts:11](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/ui/src/ui/chat/tool-helpers.ts#L11)
  - command: `nl -ba ui/src/ui/chat/tool-helpers.ts | sed -n '1,80p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Chat bubble JSON detection is guarded:** `detectJson` in the grouped chat renderer size-caps JSON auto-detection and wraps `JSON.parse` in `try/catch`, returning `null` on malformed content so the message renders through the normal text path instead of throwing.
  - file: [ui/src/ui/chat/grouped-render.ts:1289](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/ui/src/ui/chat/grouped-render.ts#L1289)
