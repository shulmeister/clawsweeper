## Summary

Keep open. Current main still collapses the reported model-fallback cases into `unknown`: empty/no-status provider runtime failures return `unknown`, the explicit OpenAI no-details message maps to `unknown`, and the final provider-runtime catchall also returns `unknown`. The reason unions, cooldown policy, and tests have not been expanded for `empty_response`, `no_error_details`, or `unclassified`. The only provided related hit, #49190, is about cron delivery warning state and does not supersede this core model fallback classification request.

## Best Possible Solution

Implement this in core model fallback classification: extend the failure/reason contracts with `empty_response`, `no_error_details`, and `unclassified` or an explicitly backward-compatible equivalent; update `classifyProviderRuntimeFailureKind`, failover reason classification, fallback attempt logging, transient cooldown policy, and focused tests for all three paths. Preserve useful raw error previews for unclassified messages.
