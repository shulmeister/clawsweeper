## Summary

The PR is a docs-only discoverability change for an external community plugin. Current main explicitly routes community plugin discovery to ClawHub and says not to open docs-only PRs just to add a plugin listing, so this belongs in the ClawHub/plugin distribution path rather than OpenClaw core.

## Close Comment

Closing after Codex review. This PR only adds `openclaw-ydim` to the community plugins doc, but current main now treats ClawHub as the canonical discovery surface for community plugins and explicitly says not to open docs-only PRs just for discoverability (`docs/plugins/community.md:14-16`, `docs/plugins/community.md:147-153`). That matches the repo vision that optional plugin capability should usually live in external plugins rather than OpenClaw core (`VISION.md:54-61`). If `openclaw-ydim` should be discoverable in the supported path, it should be published and maintained through ClawHub rather than carried as a docs-only PR here.
