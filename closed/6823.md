## Summary

Current main already provides the requested safety model: host `exec` is guarded by built-in exec approvals/allowlists/ask-fallback, and plugins can intercept any tool call with `before_tool_call` to either block it or require human approval. The exact YAML sketch from the issue was not added, but the underlying capability is present and shipped.

## Evidence

- **Built-in exec approvals guard host command execution:** The docs describe exec approvals as a host guardrail where commands run only when policy, allowlist, and optional user approval all agree; the effective policy is the stricter of `tools.exec.*` and `~/.openclaw/exec-approvals.json`, with deny fallback when no UI is available.
