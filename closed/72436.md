## Summary

Keep this PR open. Current main still lacks the proposed append-only `recovery-events.jsonl` session recovery sidecar and the associated inbound, route-resolution, manual `/compact`, and outbound delivery breadcrumbs. This is an open draft implementation candidate that needs maintainer review, not cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to review or narrow the proposed core implementation as a best-effort, append-only, redacted session recovery event log in `src/config/sessions`, with call sites for inbound metadata, route resolution, manual compaction, and outbound delivery plus focused tests for redaction, permissions, non-blocking failures, and no message-body logging. If maintainers prefer unifying this with adjacent passive recovery checkpoint work, this PR should be redirected or split rather than closed as already implemented.
