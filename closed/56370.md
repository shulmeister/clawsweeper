## Summary

Current main still appears to preserve the reported default/explicit routing mismatch for `openclaw agent --to ...` when no explicit `--agent` is provided and `main` is still a configured agent. The linked PR #56453 directly tracks the right fix, but it is open/unmerged in the provided GitHub context, and current source/tests still derive and assert `agent:main:main` for the implicit `--to` path.

## Best Possible Solution

Keep #56370 open until the fix lands. The best code path is to land or rework PR #56453 so `resolveSessionKeyForRequest` and CLI/gateway session-store selection resolve the configured default agent when `--to` is used without `--agent`, while preserving explicit `--agent`, explicit `--session-key`, and cross-store `sessionId` lookup behavior. Add regression coverage for a config containing both `main` and `steward` with `steward` marked `default: true`, covering both session-key derivation and the CLI gateway request path.
