## Summary

Close as superseded by PR #57717. Current main has only the seam from #57591: `"block"` is schema-valid and IR metadata is collected, but Slack delivery still coerces `markdown.tables: block` back to `code` and sends text chunks rather than Block Kit table attachments. The remaining requested work is exactly tracked by the open PR linked in the issue discussion, `feat(slack): wire Block Kit table send path` (#57717).

## Best Possible Solution

Use PR #57717, or an equivalent focused maintainer patch, as the canonical place to finish the Slack Block Kit table send path: remove the temporary block-to-code coercion for Slack, extract table metadata in the Slack formatter/send path, post one native Slack table attachment when valid, and preserve safe code-block fallback behavior for unsupported/multiple-table cases.
