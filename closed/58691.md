## Summary

Current `main` no longer treats `tools.exec.ask` / `tools.exec.security` as ignored: host exec now defaults to requested `security=full` plus `ask=off`, merges that request with host-local approvals explicitly, documents the two-layer policy, and shipped the default-YOLO fix in `v2026.4.2`.

## Evidence

- **Current requested defaults:** `resolveExecPolicyScopeSnapshot()` now uses `DEFAULT_REQUESTED_SECURITY = "full"` and `DEFAULT_REQUESTED_ASK = "off"` when `tools.exec` is unset, matching the no-prompt host-exec behavior this report expected.

That is explicit precedence logic, not the 2026.3.31 regression where the request looked ignored.
