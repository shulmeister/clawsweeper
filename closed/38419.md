## Summary

Issue #38419 is already covered by open PR #39166, which explicitly targets `acp.defaultRunTimeoutSeconds` for `sessions_spawn(runtime="acp")`. Current main still lacks that config field and only honors explicitly supplied ACP `runTimeoutSeconds`, so the remaining work belongs on the focused PR rather than this issue.

## Best Possible Solution
