## Summary

Current `main` already implements plugin-triggered approval routing: plugins can return `requireApproval` from `before_tool_call`, OpenClaw pauses execution and routes the request through `plugin.approval.*`, and the shared `/approve` plus native approval UIs handle resolution. The same behavior is already present in release `v2026.4.22`.
