## Summary

Current `main` already treats Discord `@everyone`/`@here` as satisfying guild mention gating for human/PluralKit senders, and it already keeps bot-authored broadcast mentions from counting. That makes this PR’s new gating behavior redundant on `main`; only the proposed opt-in config knob is absent, because the behavior is now unconditional instead.

## Evidence
