## Summary

Keep #67600 open. Current main still has the reported background/yield timeout bypass: `tools.exec.timeoutSec` is loaded as the default timeout, foreground exec uses it, but background/yielded exec without an explicit `timeout` passes `null` to the runtime and disables supervisor timeout enforcement. The related fix PR #67603 was closed unmerged per the provided GitHub context, and current tests now explicitly assert the buggy behavior for background exec.

## Best Possible Solution

Keep #67600 open. The likely fix belongs in `src/agents/bash-tools.exec.ts`: remove or narrow `backgroundTimeoutBypass` so `defaultTimeoutSec` applies to `background` and `yieldMs` sessions unless a per-call `timeout` explicitly overrides it. Update `src/agents/bash-tools.exec.background-abort.test.ts` to assert default-timeout enforcement for both `background: true` and `yieldMs`, while preserving the separate guarantee that aborting the tool call does not manually kill already-backgrounded sessions. The closed PR #67603 is useful prior art but should be re-reviewed against current main before reuse.
