## Summary

Current main already avoids the reported npm/Docker bundled-plugin failure through broader packaged-layout fixes: bundled discovery prefers built plugin trees, packaging rewrites bundled package entry metadata from TypeScript to JavaScript, and discovery no longer turns missing package entries into `escapes package directory` errors.

## Evidence

- **Bundled discovery prefers built packaged trees:** `resolveBundledDirFromPackageRoot()` chooses `dist-runtime/extensions` or `dist/extensions` ahead of source `extensions` for non-source-checkout installs, so packaged npm/Docker runs do not rely on source-only bundled entry paths.

  - file: [scripts/copy-bundled-plugin-metadata.mjs:23](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/scripts/copy-bundled-plugin-metadata.mjs#L23)
