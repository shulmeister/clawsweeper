## Summary

Keep #70819 open. Current main still rejects every `node_modules` symlink whose real target escapes the plugin install root, with no trusted OpenClaw peer-link exception. The PR’s core scanner fix remains useful; only the workflow-stability hunks appear to have already landed on main separately.

## Best Possible Solution

Keep the PR open and revise/rebase it onto current main. The best path is to land the scanner behavior narrowly: resolve the trusted host OpenClaw package root once per traversal, allow only exact `node_modules/openclaw` and `node_modules/.bin/openclaw` symlinks whose real target is inside that trusted host root, and keep the positive and negative regression tests. Drop or reconcile the CI workflow hunks already present on main.
