## Summary

Keep #43137 open. Current main has not implemented the central fix: the strict heartbeat schema and TypeScript config type still omit `enabled`, runtime enable checks still ignore `enabled: false`, and docs/tests only cover disabling by `every: "0m"`. The PR is stale/not merge-ready as-is because current main moved the enable check into `heartbeat-summary.ts` and the existing review comments identify scheduler and unrelated response-prefix concerns, but the underlying bug remains actionable in this repo.

## Best Possible Solution

Keep the PR open and ask for an update on current main: add `enabled?: boolean` to the heartbeat schema/type contract, update `src/infra/heartbeat-summary.ts` so defaults and per-agent `enabled:false` are actually disabled, update scheduler agent selection so disabled heartbeat blocks are not repeatedly scheduled, and add focused schema/runtime regression tests plus docs/help/metadata alignment for the new public config key. Drop or separately fix the unrelated response-prefix template changes.
