## Summary

Keep open. Current main still does not implement the PR's Feishu outbound `sendPayload` path for `payload.channelData.feishu.card`. Main has Feishu interactive-card primitives and tool-action/presentation card support, but `feishuOutbound` only wires `sendText` and `sendMedia`, while core only routes `channelData`/`interactive` payloads through native handling when the adapter exposes `sendPayload`. The closed linked issue #47890 and broader open #13175 do not supersede this concrete outbound-adapter work.

## Best Possible Solution

Keep the PR open for maintainer review. Best path is to rebase it onto current main and either implement Feishu-owned `sendPayload` for `channelData.feishu.card` using `sendCardFeishu` plus the shared text/media payload helpers, or explicitly move the use case into a Feishu `renderPresentation` implementation if raw native card payloads are no longer desired. Preserve focused tests for interactive cards, mediaUrls precedence, multi-media sends, `mediaLocalRoots`, `replyInThread`, chunking, identity headers, local-image auto-upload, and empty payload no-op behavior.
