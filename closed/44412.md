## Summary

Keep open. Current main still strips `:thread:` before cron delivery inference and then sends to the parent peer, so the Telegram direct-thread target hardening in #44412 is not implemented. The related #44351 tracks the base bug, but #44412 adds distinct hardening for casing, optional accountId session keys, and preferring the thread-suffix chatId.

## Best Possible Solution

Keep the PR open and consolidate it with #44351 into one landed fix. The final change should add Telegram-specific direct-thread extraction before the generic lowercasing/strip path in `src/agents/tools/cron-tool.ts`, preserve captured target casing, prefer the chatId from the `:thread:<chatId>:<threadId>` suffix, support optional accountId session keys, and add cron-tool regression tests for bare direct-thread, accountId direct-thread, and senderId-vs-thread-chatId cases.
