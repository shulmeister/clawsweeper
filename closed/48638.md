## Summary

Current `main` already fixes this Control UI bug class: the model picker now uses provider-qualified model refs as option values and sends the selected ref unchanged to `sessions.patch`, with regression tests covering wrong-prefix and slash-containing model-id cases. The changelog records the original multi-provider picker fix in `v2026.3.22`, with later hardening in `v2026.4.2` and `v2026.4.9`.

## Evidence

- **Picker options are provider-qualified:** `buildChatModelOptionFromLookup()` constructs each option value with `buildQualifiedChatModelValue(entry.id, provider)`, so the UI stores full `provider/model` refs instead of bare model ids.
