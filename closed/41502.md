## Summary

Keep PR #41502 open. Current main still hashes raw exec result details for no-progress detection, and exec results still carry volatile fields that can prevent the global circuit breaker from accumulating repeated identical outcomes. The linked issue #34574 remains a real tracked bug, and the PR discussion shows the author responded to review feedback by covering running exec metadata and aggregated output concerns.

## Best Possible Solution

Keep the PR open or replace it with an equivalent maintainer patch that adds exec-specific stable outcome hashing in `src/agents/tool-loop-detection.ts` plus regression tests for completed exec duration churn, running exec pid/session churn, global circuit breaker behavior, and progressing output/tail negative cases. Close #34574 only after that fix lands on main.
