## Summary

Current `main` already handles the Gemini native-vs-OpenAI endpoint mismatch described here: native Google runs serialize Gemini payloads with `functionCall`/`functionResponse`, `systemInstruction`, and `generationConfig.thinkingConfig`, and provider normalization strips a mistaken trailing `/openai` suffix for native `google-generative-ai` traffic while preserving explicit OpenAI-compatible Google endpoints when `api` is `openai-completions`. The same implementation is present in the latest shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
