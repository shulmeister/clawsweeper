## Summary

Keep open. Current main has useful point sanitizers for inbound metadata, active-memory prefixes, and `<relevant-memories>` display cleanup, but it does not implement the RFC’s central manifest-backed registry or getter for plugin-injected context tags. The remaining request is a real core/plugin-SDK contract gap, not a ClawHub-only plugin idea.

## Best Possible Solution

Keep the RFC open and implement an additive core/plugin-SDK contract: add a manifest-declared injected-context tag shape, normalize it in manifest loading, expose a sorted registry getter, document it on the public plugin manifest/hooks pages, and migrate display, compaction, dreaming/session ingestion, and memory-retain sanitizers to consume the shared registry with regression tests for current active-memory and memory-lancedb tags.
