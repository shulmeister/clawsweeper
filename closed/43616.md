## Summary

Keep open. Current main still has the malformed export HTML placeholders, while the export generator still performs exact literal replacements for contiguous `{{MARKED_JS}}`, `{{HIGHLIGHT_JS}}`, and `{{JS}}` tokens. The build copies that source template into `dist/export-html`, and the latest release tag inspected locally still has the same broken placeholder shape. The linked fix PR #43634 is open/unmerged in the provided context, so there is no shipped fix to close against.

## Best Possible Solution

Keep #43616 open until a canonical fix lands. The fix should restore or harden the real `src/auto-reply/reply/export-html/template.html` injection markers, prevent formatter drift, and add a regression test against the real template/build-copy path that verifies exported HTML contains no leftover `{{...}}` markers or bare `MARKED_JS;`, `HIGHLIGHT_JS;`, or `JS;` placeholders. The existing open fix work in #43634/#41861 should be reconciled rather than closing this issue prematurely.
