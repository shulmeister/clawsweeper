## Summary

Keep PR #40217 open. Current main still retains tool-loop history across idle heartbeat gaps, production still evaluates loop detection before recording the next call, and the linked bug #40144 plus overlapping PR #40798 remain open. The PR is a plausible fix attempt, but it still needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep PR #40217 open and consolidate it with issue #40144 and overlapping PR #40798. The best path is a rebased fix that clears or scopes loop history before detection at the real heartbeat/turn boundary, uses an activity/outcome-aware cutoff that does not disable slow-tool loop detection, resets or scopes warning dedupe consistently, and adds detect-then-record regression coverage plus aligned config/schema/docs updates.
