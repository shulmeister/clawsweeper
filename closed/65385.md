## Summary

Keep PR #65385 open. Current main already shipped the core Comfy plugin-owned config fix via #63058, but this PR still proposes distinct behavior: merging `models.providers.comfy` as a runtime override when `plugins.entries.comfy.config` is also present. That override contract is not implemented or documented on main, and the PR branch needs maintainer judgment before it is either rebased or closed as intentionally out of scope.

## Best Possible Solution

Keep the PR open for maintainer decision. If `models.providers.comfy` should intentionally remain a runtime override, rebase the PR onto current main and port only that merge/override behavior while preserving current plugin SecretRef auth and documenting the contract. If maintainers decide fallback-only is the intended compatibility boundary, close the PR manually as superseded by #63058 and note that provider override semantics are out of scope.
