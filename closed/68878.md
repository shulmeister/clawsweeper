## Summary

Keep open. Current main still has the performance shape this draft PR targets: `sessions.list` builds full `GatewaySessionRow` objects before `search`, `activeMinutes`, and `limit`, and row construction still performs transcript fallback plus per-row child-session store scans. The PR is also closely related to open issue #57715 and overlapping PR #59436, but that overlap is not enough to conservatively close this draft as duplicate without maintainer comparison of the competing implementations.

## Best Possible Solution

Keep #68878 open for maintainer review unless maintainers choose a single canonical implementation for #57715. The best path is to compare #68878 and #59436, keep or merge the stronger approach that moves cheap filtering/sorting/limit ahead of row construction and indexes child-session relationships once per request, add/retain regression coverage proving non-returned rows avoid transcript fallback, and then close the other PR as duplicate only after that choice is explicit.
