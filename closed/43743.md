## Summary

Keep #43743 open. The linked PR #43843 directly targeted this bug by adding `font-family: var(--font-body)` to `.chat-thinking`, but that PR is closed unmerged, and current main still renders assistant reasoning into `.chat-thinking` without a dedicated font-family override. Current tests cover reasoning extraction/rendering and separate CJK markdown/link behavior, but I did not find coverage or release evidence for Chinese text in the thinking/working output card itself.

## Best Possible Solution

Keep the issue open and implement a focused WebUI fix on the current `.chat-thinking` path. A conservative patch should reproduce Chinese reasoning text in the thinking/working card, add an explicit CJK-capable body font fallback or equivalent rendering fix for `.chat-thinking`, and add a targeted test or documented manual verification for Chinese reasoning output on the chat surface.
