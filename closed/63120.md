## Summary

Keep open. Current main still exposes `streamTo` in the ACP-enabled `sessions_spawn` schema without ACP-only guidance, and the executor still rejects `runtime="subagent"` calls that include `streamTo:"parent"` before reaching the subagent spawn path. There is also an open same-author implementation PR and an open maintainer replacement PR, so this should stay open until one lands or is explicitly closed by maintainers.

## Best Possible Solution

Keep this issue open until an implementation lands. The best maintainer path is to review and land the focused tolerant-runtime fix in #72331, or otherwise resolve the same-author schema-guidance PR #63121 as part of the same fix. The shipped behavior should document `streamTo` as ACP-only and prevent strict schema-following providers from breaking `runtime="subagent"` spawns when ACP-only fields are included.
