## Summary

Current `main` already contains the session-store maintenance change cross-referenced from this issue. It now enforces maintenance by default and prunes/caps oversized session stores during load, and the shipped `v2026.4.20` release notes explicitly describe this as preventing gateway OOM from accumulated cron/executor session backlogs.
