## Summary

Current `main` already fixes the stale chat avatar path described in #64998. Chat-tab agent switches now go through `switchChatSession`, which clears stale `chatAvatarUrl` state and immediately calls `refreshChatAvatar`; rendering still prioritizes `chatAvatarUrl`, so this closes the exact stale-avatar gap reported. There is also a focused test asserting that `switchChatSession` triggers avatar refresh.
