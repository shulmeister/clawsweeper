## Summary

Keep PR #64660 open. Current main overlaps with part of the streaming-usage work, but it still lacks the PR's central local-LLM plain-text tool-call promotion, timeout-compaction fallback, local endpoint capability attribution, and openai-completions-specific boundary-aware stream override. The PR discussion also contains concrete unresolved review concerns that need contributor or maintainer follow-up rather than automated closure.

## Best Possible Solution

Keep the PR open for maintainer review, but ask for a rebase and a narrower, safer implementation. The best path is provider/local-scoped plain-text tool-call promotion with tests for Gemma markers, Qwen/function_calls XML, normal JSON answers that must remain text, length/error finish reasons, raw streaming delta leakage, and text ordering. Separately decide whether generic local endpoint capability attribution and timeout message-count estimation are desired now that the transport already sends streaming usage options.
