## Summary

Current `main` fixes the reported completion-cache failure without re-adding raw `qa/` assets to the npm package: completion generation now skips broken subcommand registration, QA CLI loading is isolated behind a narrow private facade, and release tests explicitly forbid shipping `qa/scenarios/index.md`. The original 2026.4.9 packaging regression described in the issue and @martinmuron's follow-up was real, but the user-visible bug is already resolved on `main` and was shipped in `v2026.4.11` per the changelog entry for `#64648`.
