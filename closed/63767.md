## Summary

Current `main` already provides event-driven completion notifications for backgrounded `exec` sessions via `tools.exec.notifyOnExit` and `tools.exec.notifyOnExitEmptySuccess`, with system-event enqueueing plus heartbeat wake on exit. The exact per-call `notifyOnComplete` argument proposed in the issue is not present, but the underlying feature request is implemented well enough on `main`.

## Risks / Open Questions

- The exact proposed per-call API shape (`notifyOnComplete` on `exec`) is not present; the current solution is config-driven via `tools.exec.notifyOnExit` / `notifyOnExitEmptySuccess`.
- Docs still note that exec/process sessions are kept in memory and are lost on process restart, so crash-persistence is not part of the current implementation.
- This appears unreleased as of tag `v2026.4.23`, so closing is based on `main`, not on the latest shipped release.
