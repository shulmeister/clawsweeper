## Summary

Issue #70929 asks for an app-specific eNSP allowlist change in a `kc-gui`/`kc.exe` bundle that is not present on current `main`. Current OpenClaw scope and docs push optional, app-specific automation like this into external skills/plugins distributed through ClawHub or local skill overrides rather than core.

## Evidence

- **Vision keeps optional capability in plugins/skills, not core:** OpenClaw's vision says bundle-style plugins are preferred for optional capabilities, the bar for adding optional plugins to core is intentionally high, and new skills should be published through ClawHub first rather than added to core by default.
  - file: [VISION.md:64](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/VISION.md#L64)

  - file: [docs/plugins/community.md:9](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/docs/plugins/community.md#L9)
