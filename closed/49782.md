## Summary

Current main already closes the ACP parent notification loop. The original RFC root cause is obsolete: ACP turns now emit lifecycle end/error events, ACP spawns register parent-owned background task tracking, the task registry turns those lifecycle events into terminal delivery, and docs/tests cover both one-shot spawn completion and the owned-child `sessions_send` follow-up path.

## Evidence

## Risks / Open Questions

- There are still adjacent ACP delivery issues open on main (for example later mirror-message/early-finalization bugs), but they are narrower follow-up failures rather than the original missing parent-completion loop described in this RFC.
- Current implementation uses task lifecycle delivery rather than the exact subagent-registry wiring proposed in the issue, so the architectural path changed even though the requested user-visible behavior is present.
