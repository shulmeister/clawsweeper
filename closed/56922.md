## Summary

PR #56922 is superseded by the provider-transport work already on main: custom OpenAI-compatible providers, request auth/header overrides that can express Basic auth, and scoped private-network access for trusted self-hosted model endpoints are now implemented and documented. The unmerged PR branch also had multiple P1 review findings, so the right cleanup is to close this PR rather than revive it.

## Best Possible Solution

Close #56922 as superseded. Keep the shipped generic provider request transport surface on main, use custom `models.providers.<id>` entries for Owl-style OpenAI-compatible servers, and publish a dedicated Owl provider through the plugin/ClawHub path only if maintainers want a reusable packaged integration. Any broader private-network access for web_fetch or gateway input URLs should be a separate, focused, security-reviewed proposal.
