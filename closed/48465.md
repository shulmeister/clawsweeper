## Summary

Current `main` already routes proactive `message` sends through plugin-registered channels. The outbound path resolves the channel via shared plugin-aware channel resolution, falls back to the active plugin registry when the pinned/loaded map misses, and then calls the plugin's `outbound.resolveTarget(...)`. Current tests cover the stale-registry and active-registry cases that match this report.
