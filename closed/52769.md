## Summary

Keep open. Current `main` still defaults new Cron announce channels to `last`, so the reported UX bug is not implemented there. The PR also has a valid unresolved correctness issue from discussion: its proposed fallback uses `channelMeta/channelOrder`, which are derived from the loaded channel plugin registry rather than configured runtime channels, so it can auto-select and persist an unconfigured channel.
