## Summary

Current `main` already contains explicit duplicate-reply guards for repeated `text_end`/`message_end` provider events, plus regression coverage for the busy-queue path. The reported channel, WeCom, is also an external plugin surface, so any remaining WeCom-only duplication would not be an unresolved OpenClaw core cleanup item.
