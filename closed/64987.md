## Summary

Current main no longer mirrors BraveŌĆÖs plugin-scoped web search API key back into `tools.web.search.apiKey`, and the same fix is already present in shipped release `v2026.4.23`.

## Evidence

- **Runtime helper stops after writing plugin-scoped credentials:** `setResolvedWebSearchApiKey` now returns immediately after `setConfiguredCredentialValue(...)`, so providers with configured credential writers do not fall through and mutate `tools.web.search.apiKey` in the resolved runtime config.
