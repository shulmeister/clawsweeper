## Summary

Keep open. Current main still implements the reported failure: when ACP runtime spawning is available, `sessions_spawn` exposes `streamTo` in the shared schema, then rejects `streamTo:"parent"` before the native `runtime="subagent"` path can run. The related open PRs (#65282 and #68397) target the right fix direction, and the reporter validated that direction, but the fix is not present in the checked-out main or latest inspected release code.

## Best Possible Solution

Land PR #68397 or an equivalent minimal change in `src/agents/tools/sessions-spawn-tool.ts`: treat `streamTo` as ACP-only by passing it through only for `runtime="acp"`, silently drop it for native subagent spawns, and update `src/agents/tools/sessions-spawn-tool.test.ts` so `runtime="subagent"` with `streamTo:"parent"` reaches `spawnSubagentDirect` instead of returning the ACP-only error. Keep broader per-runtime schema simplification in the separate schema/provider-compat tracking work.
