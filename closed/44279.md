## Summary

Current main already handles this case by persisting unresolved allow-always approvals as durable exact-command entries instead of dropping them, and both node-host and gateway paths reuse that durable trust on rerun.

## Evidence

- **Gateway fallback persists durable approval when no resolved patterns exist:** In the gateway allow-always path, current main calls `persistAllowAlwaysPatterns(...)` and falls back to `addDurableCommandApproval(...)` when no patterns are derived, so unresolved short commands are still persisted rather than silently discarded.
