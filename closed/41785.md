## Summary

Current `main` already has native Voice Wake on app surfaces and already exposes the ingress/plugin seams a local wake daemon would use. The remaining proposal is an optional local capability that matches the repo’s documented ClawHub/plugin boundary, not OpenClaw core.

## Evidence

- **Vision keeps optional capability out of core:** `VISION.md` says core should stay lean, optional capability should usually ship as plugins, and new skills should go to ClawHub first; it also lists new core skills that can live on ClawHub under 'What We Will Not Merge (For Now)'.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/VISION.md#L54)
  - sha: [a53fea39050b](https://github.com/openclaw/openclaw/commit/a53fea39050beba21ac5468d876c37323cfdd2e6)
- **Plugin docs route new capability to external plugins:** `docs/plugins/building-plugins.md` says plugins extend OpenClaw with speech, realtime transcription/voice, media understanding, tools, hooks, and services, and explicitly says you do not need to add your plugin to the OpenClaw repo; publish to ClawHub or npm instead.
  - file: [docs/plugins/building-plugins.md:11](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/docs/plugins/building-plugins.md#L11)
  - sha: [a53fea39050b](https://github.com/openclaw/openclaw/commit/a53fea39050beba21ac5468d876c37323cfdd2e6)
- **SDK already supports daemon-style integrations:** `docs/plugins/sdk-overview.md` documents plugin registration for speech/realtime/media plus infrastructure seams including `api.registerHook(...)`, `api.registerHttpRoute(...)`, `api.registerGatewayMethod(...)`, `api.registerCli(...)`, and `api.registerService(...)`.
  - file: [docs/plugins/sdk-overview.md:73](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/docs/plugins/sdk-overview.md#L73)
  - sha: [a53fea39050b](https://github.com/openclaw/openclaw/commit/a53fea39050beba21ac5468d876c37323cfdd2e6)
- **Core already provides external wake ingress:** `docs/cli/system.md` documents `openclaw system event --text ... --mode now`, and `docs/automation/cron-jobs.md` documents `POST /hooks/wake` plus mapped hooks that transform arbitrary payloads into `wake` or `agent` actions.
  - file: [docs/cli/system.md:24](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/docs/cli/system.md#L24)
  - sha: [a53fea39050b](https://github.com/openclaw/openclaw/commit/a53fea39050beba21ac5468d876c37323cfdd2e6)
- **Wake hook behavior is covered on main:** `src/gateway/server.hooks.test.ts` verifies both direct `/hooks/wake` payloads and mapped hook payloads are queued as untrusted system events for the main session.
  - file: [src/gateway/server.hooks.test.ts:284](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/src/gateway/server.hooks.test.ts#L284)
  - sha: [a53fea39050b](https://github.com/openclaw/openclaw/commit/a53fea39050beba21ac5468d876c37323cfdd2e6)
- **Voice Wake is already native on current main:** `README.md` lists Voice Wake as a product highlight, `docs/platforms/mac/voicewake.md` documents the always-on wake-word runtime and forwarding behavior, and `apps/macos/Sources/OpenClaw/VoiceWakeForwarder.swift` prepends the recognized-voice hint and forwards the transcript into the gateway agent flow.
  - file: [apps/macos/Sources/OpenClaw/VoiceWakeForwarder.swift:7](https://github.com/openclaw/openclaw/blob/a53fea39050beba21ac5468d876c37323cfdd2e6/apps/macos/Sources/OpenClaw/VoiceWakeForwarder.swift#L7)
  - sha: [a53fea39050b](https://github.com/openclaw/openclaw/commit/a53fea39050beba21ac5468d876c37323cfdd2e6)
- **No FaceWake implementation surfaced on main:** Repo search for `FaceWake|face wake|MediaPipe|OpenCV` in `apps`, `src`, `docs`, and `README.md` returned no relevant implementation/docs hits, so this is not cleanup of an existing core feature.
  - command: `rg -n "FaceWake|face wake|MediaPipe|OpenCV" apps src docs README.md`

## Risks / Open Questions

- If maintainers specifically want an official first-party reference daemon despite the documented plugin boundary, that would be a product-scope exception rather than routine cleanup.
- The proposal bundles native Voice Wake, FaceWake, and daemon/reference-implementation ideas; closing as ClawHub assumes the reporter is satisfied with the documented external-plugin path.
