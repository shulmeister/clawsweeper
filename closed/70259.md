## Summary

Keep open. Current main still has the reported chat.history fallback gap: when a session entry has no stored thinkingLevel, the handler falls straight to the model/global resolver and never checks cfg.agents.list[].thinkingDefault. The PR is a focused bug fix that matches the documented per-agent thinkingDefault contract, and the related #70302 is a closed, unmerged companion for a different SessionRow path rather than a superseding implementation.

## Best Possible Solution

Keep #70259 open and rebase or port its narrow chat.history change onto current main: after resolveSessionAgentId, check resolveAgentConfig(cfg, sessionAgentId)?.thinkingDefault between entry.thinkingLevel and resolveThinkingDefault, then add a focused regression test for session override precedence, per-agent fallback, global/model fallback, and normalized agent session keys. Consider a separate maintainer decision for the adjacent SessionRow fallback from #70302.
