## Summary

Current main already implements the requested native multi-key behavior through the provider-agnostic auth-profile system: OpenRouter uses the normal provider auth path, multiple profiles for one provider are ordered/rotated, successful use updates lastUsed for round-robin selection, failures cool down profiles and try the next profile, and rate-limit/overload rotation can escalate to model fallback.
