## Summary

Current main still matches the reported problem. `openclaw agent` tries the Gateway path, catches any Gateway error, logs a single fallback diagnostic, then runs the embedded agent. The CLI exposes `--local` but no explicit fallback opt-in flag, tests still assert automatic fallback, docs describe the fallback as current behavior, and the embedded JSON envelope path does not add a stable transport or fallback marker. The related merged heartbeat PR #71478 explicitly left this CLI fallback issue out of scope.

## Best Possible Solution

Keep this issue open for a targeted CLI behavior change. The likely code path is `src/commands/agent-via-gateway.ts` plus option registration in `src/cli/program/register.agent.ts`, with regression coverage in `src/commands/agent-via-gateway.test.ts` and docs updates in `docs/cli/agent.md`. Maintainers should choose whether Gateway failure fails unless an explicit fallback flag is passed, or whether JSON and non-JSON output expose a clear embedded-fallback signal.
