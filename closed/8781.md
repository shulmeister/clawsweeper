## Summary

Atlas/ChatGPT-UI routing is not implemented on current `main`, and the current OpenClaw architecture and vision docs place this kind of optional browser/web integration on the external plugin or ClawHub path rather than OpenClaw core.

## Evidence

- **Vision prefers optional capability as plugins:** `VISION.md` says core should stay lean, optional capability should usually ship as plugins, and the bar for adding optional plugins to core is intentionally high.

## Risks / Open Questions

- There is still user interest here, so closing in core could hide demand unless a follow-up ClawHub/plugin issue or package is created.
- An external Atlas integration may still need small future SDK seams if plugin authors hit missing browser/web-provider hooks.
