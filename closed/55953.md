## Summary

Current `main` does not support the PR’s claimed root cause. Telegram document messages already map to `<media:document>`, and `resolveTelegramInboundBody()` assigns that type-aware placeholder before the hardcoded `<media:image>` fallback can run. The PR adds useful tests, but the specific mislabel bug described here is not confirmed against current `main`.
