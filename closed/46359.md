## Summary

Keep open. Current main partially supersedes the PR by moving Ollama discovery and native stream requests onto the shared guarded-fetch path, but the vLLM/SGLang self-hosted discovery path named in the PR still builds `${baseUrl}/models` and calls raw `fetch()` directly. The submitted PR is also stale against the current plugin architecture and review feedback identified regressions for default local provider URLs, so there is not strong evidence to close it as implemented or obsolete.

## Best Possible Solution

Keep the PR open or replace it with a fresh patch against current main. The best implementation is to update the shared self-hosted provider discovery helper to use `fetchWithSsrFGuard()`/DNS pinning with a provider-appropriate hostname policy that preserves trusted loopback/LAN defaults, add vLLM/SGLang regression tests, and leave the existing Ollama guarded-fetch implementation in place unless maintainers decide on a stricter operator-admin SSRF policy.
