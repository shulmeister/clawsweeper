## Summary

Current main only partially covers PR #68041. The loader-side memory capability preservation is present with regression coverage, but the wiki/memory CLI loading behavior requested by the PR is not implemented on main, and the wiki bridge CLI still runs the in-process sync path that depends on an active memory public-artifacts capability. The related open PRs should be reconciled by maintainers rather than auto-closing this as implemented.

## Best Possible Solution

Keep this open for maintainer reconciliation. Drop or rebase the already-implemented loader hunk, then solve the remaining wiki CLI bridge behavior with a narrow, tested runtime-loading path, likely by loading the wiki owner plus the configured active memory slot or by routing bridge import through the gateway RPC. If #67063, #69458, or #69952 becomes the chosen canonical fix, close this PR as superseded with that link.
