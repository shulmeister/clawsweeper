## Summary

Keep #67687 open. Current main still has a plausible Feishu-specific gap for the reporter's exact config shape: `groups.<chat_id>.requireMention: false` is honored only after the group passes the separate `groupPolicy`/`groupAllowFrom` access gate, and the default Feishu group policy is allowlist. With no `groupPolicy: "open"` or `groupAllowFrom`, current code returns before creating a group session, matching the reported missing `agent:main:feishu:group:<chat_id>` session. Related WhatsApp/Telegram requireMention reports do not cover this Feishu path.

## Best Possible Solution

Keep the issue open for maintainer/product handling. The likely fix is either to restore legacy compatibility by treating explicit `channels.feishu.groups.<chat_id>` entries as allowed groups under the default allowlist policy, or to add a targeted doctor/status warning plus docs/migration note explaining that `requireMention:false` only disables mention gating and `groupPolicy:"open"` or `groupAllowFrom:["oc_xxx"]` is still required. Either path should include a regression test for the exact reported config and session-creation behavior.
