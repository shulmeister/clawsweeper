## Summary

#48345 asks for a public cron control to skip startup missed-run catch-up. Current main still does not expose `skipMissedRuns` or equivalent public missed-run controls, but open #42039 already tracks the same remaining cron startup catch-up configuration work, including disabling catch-up with `maxMissedJobsPerRestart: 0` and an optional per-job skip flag.

## Evidence

- **Canonical related issue:** The provided GitHub context shows open #42039 requests exposing `maxMissedJobsPerRestart` and `missedJobStaggerMs` in `cron` config, explicitly describing duplicate messages from startup missed-job replay and proposing `maxMissedJobsPerRestart: 0` to disable missed-job catch-up. A comment on #48345 also links #42039 as the more granular version of the same request.
- **No public skip option in config type:** `CronConfig` exposes `enabled`, `store`, `maxConcurrentRuns`, retry, webhook, session retention, run log, and failure alert/destination fields; it does not expose `skipMissedRuns`, `maxMissedJobsPerRestart`, or `missedJobStaggerMs`.
