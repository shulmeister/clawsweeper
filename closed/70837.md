## Summary

Current `main` already contains post-`v2026.4.23` plugin-loader and packaged-runtime fixes that address the reported Tavily/TypeBox packaged-load failure on Windows, so this issue is implemented on `main` even though no released tag contains the fix yet.

## Evidence

- **Windows native-Jiti guard:** Current `main` disables native Jiti loading on Windows via `supportsNativeJitiRuntime()`, and `resolvePluginLoaderJitiTryNative()` therefore keeps bundled `dist/extensions/*` modules off the native path on `win32`, matching the Windows startup failure reported here.
  - file: [src/plugins/sdk-alias.ts:696](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/plugins/sdk-alias.ts#L696)

  - file: [src/plugins/sdk-alias.test.ts:873](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/plugins/sdk-alias.test.ts#L873)
