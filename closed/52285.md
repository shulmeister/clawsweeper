## Summary

Close as implemented on current main: Telegram heartbeat delivery now preserves turn-scoped route/thread metadata instead of drifting to stale shared-session state, and current regression coverage explicitly locks the Telegram heartbeat/topic cases described in the report. The fixes are documented as shipped in v2026.4.22.

## Evidence

- **Heartbeat delivery target now merges turn-scoped routing with stored session route:** `resolveHeartbeatDeliveryTarget` merges explicit turn delivery context with the session route for `target: "last"`, passes channel/to/account/thread into heartbeat resolution, and reuses the stored thread only when the route still matches.
  - file: [src/infra/outbound/targets.ts:84](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/infra/outbound/targets.ts#L84)
- **Heartbeat runner sends using resolved channel/to/thread instead of synthetic heartbeat routing:** `runHeartbeatOnce` resolves delivery from the main-session route, avoids reusing stale base-session turnSource data for isolated runs, includes `MessageThreadId` in the reply context, and delivers outbound payloads with the resolved `threadId`.
  - file: [src/infra/heartbeat-runner.ts:804](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/infra/heartbeat-runner.ts#L804)
- **Regression coverage for Telegram topic drift on heartbeat exec events:** Test proves that when session metadata drifts to the wrong Telegram topic, an exec-event heartbeat still delivers to the original Telegram topic/thread from the event delivery context.
  - file: [src/infra/heartbeat-runner.ghost-reminder.test.ts:536](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/infra/heartbeat-runner.ghost-reminder.test.ts#L536)
- **Regression coverage for Telegram isolated scheduled heartbeats:** Test proves isolated scheduled heartbeats keep Telegram topic routing and pass `MessageThreadId`/`messageThreadId` through both reply generation and outbound send.
  - file: [src/infra/heartbeat-runner.ghost-reminder.test.ts:603](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/infra/heartbeat-runner.ghost-reminder.test.ts#L603)
- **Regression coverage for shared-session thread/routing races:** Tests cover the exact failure class where `target: "last"` must keep the correct thread when the turn source matches the session route, and must drop stale thread metadata when the shared-session route has drifted.
  - file: [src/infra/outbound/targets.test.ts:819](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/infra/outbound/targets.test.ts#L819)
- **Shipped release evidence:** The changelog for `2026.4.22` includes `Heartbeat/Telegram topics` and `Gateway/sessions` fixes that match this report’s symptoms: keeping isolated heartbeats on the bound Telegram topic and preventing heartbeat/cron/exec turns from overwriting shared-session routing metadata.
  - file: [CHANGELOG.md:116](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/CHANGELOG.md#L116)
