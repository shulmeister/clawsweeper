## Summary

Keep #50052 open. Current main has improved Ollama native streaming/thinking handling, but I do not see a complete llm-task fix: the tool still forwards the model override literally, only passes Ollama thinking control when the caller explicitly supplies `thinking`, and still turns an empty/non-visible embedded result into `LLM returned empty output`. The reported provider-prefixed model parameter path also still appears unresolved for providers such as Anthropic.

## Best Possible Solution

Keep the issue open and fix it in the llm-task plugin surface: normalize or explicitly validate `provider` plus `model` when callers pass a full `provider/model` ref, default Ollama JSON-only llm-task runs to a non-thinking visible-output path or document/forward `thinking: "off"` clearly, and add focused tests for Ollama visible JSON output, thinking-only/empty output handling, and the prefixed model-parameter case. The existing native Ollama stream parser should be reused rather than adding a separate raw fetch path.
