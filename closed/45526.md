## Summary

Close. On current main, the `archive` frontmatter value is only normalized into an archive-type discriminator and then matched against explicit extractor branches; unknown values fail closed as `unsupported archive type` rather than being forwarded to `tar`. The PR discussion also identified a behavior regression in the proposed patch: it would silently reject previously-supported case-insensitive values like `Zip` and `TAR.GZ`.
