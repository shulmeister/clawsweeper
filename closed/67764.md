## Summary

Keep open. Current main has adjacent observability and plugin hooks for `llm_input`, plus Sessions and Chat transcript views, but it does not retain or expose a per-turn assembled model input in the Control UI Sessions tab. `chat.history` is a sanitized, bounded transcript view, not the exact provider input. The provided related items do not supersede this feature request.

## Best Possible Solution

Add a core debug-inspection path: capture the final assembled input around the existing `llm_input`/context assembly boundary, key it by session/run/turn, store it locally with bounded retention and sensitivity labeling, expose it through a typed Gateway RPC/schema, and add a Sessions or Chat drill-in view with tests for gating, retention, schema shape, and sanitized versus raw display behavior.

## Risks / Open Questions

- The feature would expose sensitive workspace, history, tool-result, and credential-adjacent context; implementation should be opt-in, local/debug-gated, clearly labeled, and retention-bounded.
- Exact model-input semantics differ across direct providers, CLI backends, and native Codex harnesses; the UI should label which assembled layer is being shown.
