## Summary

Current `main` already verifies structured candidate/baseline labels against `qa-suite-summary.json` run metadata before computing parity metrics, with regression coverage for mismatches and legacy summaries.

## Evidence

- **Parity gate now validates summary labels before metrics:** `buildQaAgenticParityComparison()` calls `verifySummaryLabelMatch()` for both candidate and baseline summaries before any metric computation. The helper reads `run.primaryProvider`, `run.primaryModel`, and `run.primaryModelName`, and throws `QaParityLabelMismatchError` when a structured label does not match the summary metadata, including the swapped-summary-path guidance described in the issue.
