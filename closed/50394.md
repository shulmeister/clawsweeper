## Summary

Keep #50394 open. Current main still contains the Feishu `@_all` shortcut that the PR removes, and group `requireMention` gating still trusts `ctx.mentionedBot`, so the reported behavior is not implemented on main. The PR also addresses the Greptile review gap by adding the regression test in its second commit per the provided PR context.

## Best Possible Solution

Keep the PR open and either merge #50394 after normal maintainer validation or apply the same minimal source change plus its regression test on main. The remaining product behavior belongs in the Feishu plugin: `@_all` should not set `mentionedBot=true`; explicit bot mentions should continue to work, including rich-text `post` parsing.
