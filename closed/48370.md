## Summary

Current main already treats TTS as a plugin-owned capability. The generic speech-provider extension seam exists, and repo vision explicitly says optional capabilities should usually ship as plugins rather than be added to core, so a Qwen3-TTS integration belongs as a community plugin/ClawHub item instead of an OpenClaw core issue.

## Evidence

- **Vision scopes optional capability to plugins:** VISION.md says core should stay lean, optional capability should usually ship as plugins, plugins should be hosted in their own repository, and the bar for adding optional plugins to core is intentionally high.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/VISION.md#L54)
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Speech providers are plugin capabilities:** The speech provider registry resolves providers from plugin capability ownership via the `speechProviders` key rather than a core-only table.
  - file: [src/tts/provider-registry.ts:16](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/tts/provider-registry.ts#L16)
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **SDK already documents how to add a TTS provider:** Provider plugin docs show `api.registerSpeechProvider(...)` as the extension point for adding a speech/TTS provider.
  - file: [docs/plugins/sdk-provider-plugins.md:490](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/docs/plugins/sdk-provider-plugins.md#L490)
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Manifest contract supports speech-provider ownership:** Plugin manifests can declare `contracts.speechProviders`, which is the ownership model for speech providers.
  - file: [docs/plugins/manifest.md:386](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/docs/plugins/manifest.md#L386)
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Bundled TTS providers are already implemented as plugins:** Current speech providers are registered from extension packages (`extensions/{elevenlabs,google,microsoft,minimax,openai,vydra,xai}/index.ts`), which matches the plugin-first architecture for vendor-specific TTS integrations.
  - file: [extensions/openai/index.ts:54](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/extensions/openai/index.ts#L54)

## Risks / Open Questions

- A future maintainer could still choose to bundle a Qwen3-TTS plugin for product reasons, but current main and VISION make that a product decision, not cleanup on this core issue.
- The repo includes plugin docs but no Qwen3-TTS implementation on main, so closing is based on scope/ownership rather than shipped support.
