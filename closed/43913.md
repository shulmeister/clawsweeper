## Summary

Current main implements the ACP session-control problem through the dedicated ACP/session surfaces rather than by merging ACP handling into the native `subagents` tool. `/acp sessions`, `/acp cancel`, `/acp steer`, `/acp close`, and `sessions_send` cover the linked #43496 use case, and current guidance explicitly keeps ACP harness control out of `subagents`. The PR’s bot review comments also flagged real gaps in the proposed patch that the current implementation avoids.

## Best Possible Solution

Close this PR and keep the current main direction: ACP sessions are controlled through the dedicated ACP command/session layer and `sessions_send`, while `subagents` remains scoped to native OpenClaw sub-agent runs. Any future model-facing ACP control API should be designed in that ACP/session surface with the existing requester scoping, policy, cancellation, and tests, not by reviving this patch.
