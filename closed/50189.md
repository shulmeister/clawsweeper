## Summary

Keep open for maintainer/author handling. Current main does not implement the PR’s status normalization, and the PR is not safe to close as implemented because the proposed `delivered === true => ok` rule would also mask genuine fatal cron-agent failures. No conservative auto-close reason fits.

## Best Possible Solution

Keep this PR in needs-changes/maintainer-review state rather than auto-closing it as shipped. The correct fix should classify the outcome before `applyJobResult`: distinguish successful primary user delivery plus nonfatal follow-up/render failures from fatal agent errors and real delivery failures, then apply the normalized status consistently to persisted state, finished events, run logs, failure alerts, backoff/delete behavior, and targeted regression tests. If maintainers do not want this branch, close it manually in favor of the open canonical bug #50170.
