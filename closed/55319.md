## Summary

Current `main` no longer matches the stale-cache path described in PR #55319: the channel loader already reads from the active channel registry surface, and checked-in tests cover registry swaps plus pinned-registry replacement behavior. The proposed version-keyed change is therefore obsolete relative to the PR's stated reproduction.

## Evidence
