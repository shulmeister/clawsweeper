## Summary

Current `main` no longer aborts `openclaw completion --write-state` when the QA scenario pack is unavailable, and the exact reported error is covered by a regression test.

## Evidence

- **Completion cache generation now tolerates subcommand registration failures:** `registerSubcommandsForCompletion` wraps each subcommand registration in `try/catch` and only emits a warning, so a failing `qa` subcommand no longer aborts `completion --write-state`.
  - file: [src/cli/completion-cli.ts:55](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/cli/completion-cli.ts#L55)
  - command: `nl -ba src/cli/completion-cli.ts | sed -n '45,120p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Regression test matches the reported error text:** The write-state test throws `qa scenario pack not found: qa/scenarios/index.md` from the mocked `qa` subcommand and asserts the completion cache files are still generated.
  - file: [src/cli/completion-cli.write-state.test.ts:82](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/cli/completion-cli.write-state.test.ts#L82)
  - command: `nl -ba src/cli/completion-cli.write-state.test.ts | sed -n '1,130p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **QA scenario pack is now optional in npm distributions:** `readQaScenarioPack()` returns an empty pack when `qa/scenarios/index.md` is absent, with an inline comment explaining this is required so npm distributions do not crash completion cache updates or other consumers.
  - file: [extensions/qa-lab/src/scenario-catalog.ts:331](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/qa-lab/src/scenario-catalog.ts#L331)
  - command: `nl -ba extensions/qa-lab/src/scenario-catalog.ts | sed -n '323,350p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Docs define QA seeds as repo-backed assets:** Current QA docs say the seed assets live in `qa/scenarios/index.md` and `qa/scenarios/<theme>/*.md` inside the repo, which contradicts the issue's expectation that completion should resolve them from `~/.openclaw`.
  - file: [docs/concepts/qa-e2e-automation.md:153](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/docs/concepts/qa-e2e-automation.md#L153)
  - command: `nl -ba docs/concepts/qa-e2e-automation.md | sed -n '150,175p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Release packaging intentionally excludes `qa/scenarios/index.md`:** The npm release check explicitly fails if `qa/scenarios/index.md` is present in the package, showing that missing repo QA artifacts in packaged installs is expected and must be handled gracefully.
  - file: [test/openclaw-npm-release-check.test.ts:338](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/test/openclaw-npm-release-check.test.ts#L338)

- **Issue discussion confirmed a real cross-platform regression:** Four follow-up comments on April 9-10, 2026 reported the same failure on macOS and Linux, which supports treating this as a genuine packaging/completion regression rather than a one-off local setup problem.
