## Summary

Current `main` already implements WebChat assistant image delivery: assistant/media reply payloads are converted into managed outgoing image attachments, persisted in chat history, and rendered back through Gateway media URLs. The implementation accepts image data URLs plus existing local or remote image sources, which matches the reported WebChat gap closely enough to close as implemented on `main`.

## Evidence
