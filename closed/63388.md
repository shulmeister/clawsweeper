## Summary

Current main has explicit orphaned-subagent recovery on gateway startup and restored-run startup, with bounded retry behavior and regression coverage for restart-aborted subagent runs. That addresses the reported long-delay post-reload resume behavior well enough to close as implemented on main.

## Evidence

- **Bounded orphan recovery:** Orphaned subagent recovery now uses a 5s initial delay and a bounded retry loop (`MAX_RECOVERY_RETRIES = 3`) instead of leaving interrupted runs in an open-ended paused state; the recovery path sends the synthetic gateway-reload resume message directly through `agent` RPC.
