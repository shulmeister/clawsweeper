## Summary

Keep #71379 open. Current main still does not expose `--size`, `--aspect-ratio`, or `--resolution` on `openclaw infer image edit`, while the shared image runtime and OpenAI provider already support edit-time geometry hints. The PR is addressing a real gap, but it needs maintainer review/cleanup rather than conservative auto-close.

## Best Possible Solution

Keep the PR open, narrow it to the image edit CLI/capability metadata/docs/tests surface, address the existing review comments, then land the geometry flag exposure because current main still lacks the user-visible behavior while runtime/provider contracts already support it.
