## Summary

Current main implements the requested display boundary for subagent completion announces. Ryce's comment correctly framed this as an internal-state visibility leak; current code now carries subagent completions as typed internal `task_completion` events, feeds them to the requester agent, and strips/drops internal-only runtime-context turns from Control UI/WebChat/TUI-visible history. The broader question of raw transcript persistence remains separate from this display-focused request and is already represented by #62306.
