## Summary

Current main already provides detached background-task execution and non-blocking chat, with separate task/status surfaces for background work.

## Evidence

- **Non-blocking chat send on current main:** `chat.send` registers the run and responds immediately with `{ runId, status: "started" }` before the agent work completes, which is the core behavior requested for real-time chat responsiveness.
  - file: [src/gateway/server-methods/chat.ts:2193](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/gateway/server-methods/chat.ts#L2193)

  - file: [docs/concepts/queue.md:81](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/docs/concepts/queue.md#L81)
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
