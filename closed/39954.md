## Summary

Keep PR #39954 open. Current main still has only terse profile tables and generic config help; it does not include the PR's practical clarification that `tools.profile: "messaging"` is intentionally narrow or the direct `tools.profile: "full"` example for broader command/control access. The runtime/tests confirm the distinction is real, and the related PRs are adjacent open work rather than merged replacements.

## Best Possible Solution

Keep the PR open and port its useful wording into the current `docs/tools/index.md` profile section: explain that `tools.profile: "messaging"` is intentionally narrow, that `tools.profile: "full"` is the unrestricted baseline for broader command/control, and add a compact full-profile example reconciled with the current profile table.
