## Summary

Current OpenClaw already treats `channels.discord` edits as hot-reloadable channel changes: the Discord channel registers that config prefix for reload, the gateway reload planner converts it into a Discord channel restart, the reload handler executes the stop/start cycle, startup reloads the fresh config snapshot, and docs/tests match that behavior. The shipped `v2026.4.23` release also contains this implementation.

## Evidence

- **Discord channel declares hot-reload scope:** The Discord channel plugin registers `channels.discord` under `reload.configPrefixes`, so Discord config edits are part of the channel hot-reload path.
  - file: [extensions/discord/src/shared.ts:111](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/extensions/discord/src/shared.ts#L111)
