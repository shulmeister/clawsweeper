## Summary

#38671 is the same central feature request as the open canonical tracker #18160: direct shell/exec cron payloads that bypass agent/LLM execution. Current main still has not shipped command cron support, and related PR #62789 remains open/unmerged partial implementation work, so the best cleanup is to close this duplicate while preserving the hybrid command-output discussion on #18160/#62789.

## Best Possible Solution

Close #38671 as a duplicate of #18160. Keep #18160 as the canonical core/product tracker and use #62789 or a successor PR to implement the full cron command payload contract across types, Gateway schema, normalization, runtime execution, CLI/UI/docs, tests, delivery, logging, and security policy, including the useful hybrid command-output-to-agent workflow raised here.

## Risks / Open Questions

- The hybrid mode from #38671, where command output is injected into an agent for analysis, should be explicitly carried into #18160 or PR #62789 so it is not lost during consolidation.
- Closing as duplicate does not mean the feature is implemented or rejected; direct host command execution remains security-sensitive and still needs maintainer design for command representation, cwd/env, approval/sandbox policy, timeout/termination, output limits, run logs, and delivery semantics.
