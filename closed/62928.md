## Summary

Current main already addresses the reported heartbeat/UI disappearance from 2026.4.5. Heartbeat, cron-event, and exec-event turns no longer trigger idle/daily session resets, no longer overwrite shared-session routing metadata, and Control UI/WebChat history now filters heartbeat prompts/OK acknowledgments while preserving the transcript on disk.

## Best Possible Solution

Close this as implemented. Keep the shipped heartbeat session-continuity and visible-history filtering behavior; if a current release still reproduces, track it as a new focused bug with reproduction steps, affected session key, heartbeat config, logs, and sanitized transcript samples.
