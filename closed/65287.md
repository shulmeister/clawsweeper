## Summary

Keep #65287 open. Current main does not implement Anthropic native/server-side compaction, and the mixed Anthropic prompt-cache TTL behavior requested here is not fully present. The related open #10213 tracker covers the server-side compaction portion, but it does not cover the mixed 1h stable-prefix plus 5m trailing-conversation cache policy, so closing #65287 as duplicate would lose actionable remaining scope.

## Best Possible Solution

Keep #65287 open unless maintainers split it. The implementation should add Anthropic-specific server-compaction params through the extra-params and Anthropic transport path, inject context_management.edits plus beta headers only when enabled, preserve/round-trip Anthropic compaction blocks in history and streaming, and change the Anthropic cache policy so stable system/tool/workspace prefix blocks can use 1h while trailing conversation/user/compaction content remains default 5m. If maintainers prefer narrower tracking, use #10213 for the server-compaction half and create/link a separate prompt-cache TTL issue before closing #65287.
