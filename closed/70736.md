## Summary

Current main already fixes the central reported failure: the OpenAI-compatible completions transport now sends structured `tools` in the request body when runtime tools are available, defaults proxy/local completions requests to `tool_choice: "auto"`, and parses streamed `tool_calls` back into OpenClaw tool-call blocks. The fix is recorded in the unreleased 2026.4.25 changelog, so no shipped release tag is available yet.

## Best Possible Solution

Close this issue as fixed on main. Keep the current OpenAI-compatible transport behavior, and continue recommending the native Ollama `/api/chat` provider path for the most reliable Ollama tool-calling behavior while the OpenAI-compatible mode remains a legacy/proxy path.
