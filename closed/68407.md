## Summary

Keep open. Current `main` still only bypasses ACP dispatch for `/new` and `/reset`, so the underlying bug remains. This PR moves in the right direction for `/acp`, `/status`, and `/unfocus`, but the proposed regex is too broad for no-arg commands and can reroute invalid inputs like `/status report` or `/unfocus note` away from the focused ACP session.
