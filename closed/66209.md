## Summary

Close as ClawHub/plugin-scope. The PR adds a large bundled Microsoft 365/Outlook service integration, but current OpenClaw direction keeps optional commercial integrations out of core unless a missing generic plugin API is demonstrated. Current main already has plugin distribution, tools/hooks/services/http routes, and plugin approval flows that can support an external Outlook triage plugin; any remaining Teams Adaptive Card callback need should be split into a small core/channel API PR rather than bundled with the M365 integration.

## Best Possible Solution

Move the Outlook triage assistant to an external ClawHub/npm plugin using the existing plugin APIs and plugin approval flow. If an external implementation proves a generic Teams Adaptive Card callback or action-approval SDK gap remains, split that into a small OpenClaw core/channel API PR with targeted tests.
