## Summary

Current `main` already fixes the chat context-percentage bug by computing prompt usage as `input + cacheRead + cacheWrite` before dividing by the context window, and it has regression tests covering the Anthropic-style cached-token case. The misleading cache-hit percentage from the older dashboard path is not present in the current chat metadata renderer.

## Evidence

- **Current main computes context % from full prompt usage:** `extractGroupMeta` now derives `promptTokens = input + cacheRead + cacheWrite` and uses that for `contextPercent`, which fixes Anthropic sessions where most prompt tokens arrive via `cacheRead`.
