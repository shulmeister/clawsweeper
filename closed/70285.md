## Summary

The requested cron lifecycle hooks fit the existing plugin/ClawHub extension layer, not OpenClaw core. Current main already exposes plugin hooks that can gate cron tool calls, observe successful creates, inspect/update cron state at gateway startup, and route thresholded cron failures to external webhook handlers.

## Risks / Open Questions

- The plugin route is not the same UX as a simple `cron.hooks` config block; teams would need to package and maintain a plugin/skill.
- There is no dedicated typed `cron_job_added` plugin event today, so post-create behavior would be implemented via `after_tool_call` on the `cron` tool or via external webhook flows rather than a bespoke core hook API.
