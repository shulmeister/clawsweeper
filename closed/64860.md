## Summary

Current `main` already hardens the exact failure from #64860: missing `qa/scenarios/index.md` in packaged installs no longer crashes QA scenario loading, completion-cache generation skips subcommand registration failures instead of aborting, and the private `qa` CLI is now explicitly gated to source checkouts rather than packaged installs.

## Evidence

- **Missing scenario pack now returns an empty QA pack:** `readQaScenarioPack()` explicitly treats `qa/scenarios/index.md` as optional in npm/package distributions and returns an empty pack instead of throwing, with an inline comment calling out completion-cache updates as the crash path this avoids.
