## Summary

Keep open. Current main still routes `pnpm ui:build` through `scripts/ui.js`, resolves `pnpm` to a `.cmd` path on Windows, enables `shell: true`, and passes the unquoted command path directly to `spawn`/`spawnSync`. The PR still addresses a real remaining Windows `Program Files` launcher failure, although it should likely be rebased and adjusted to use the existing `scripts/pnpm-runner.mjs`/`windows-cmd-helpers.mjs` pattern rather than adding a separate quote helper.

## Best Possible Solution

Keep the PR open but request a rebase and implementation aligned with the existing Windows-safe pnpm runner: have `scripts/ui.js` use `scripts/pnpm-runner.mjs`/`windows-cmd-helpers.mjs` or the same explicit `cmd.exe /d /s /c` pattern with `shell: false`, preserve the current full UI dependency install behavior intentionally, and add a regression test for `C:\Program Files\...\pnpm.cmd` before merging.
