## Summary

Current main, also present in v2026.4.24, already fixes the reported `/compact` cancellation loop by returning a compaction boundary instead of cancelling when no real conversation messages are found. Existing tests cover repeated empty-preparation compactions, and the runner computes `tokensAfter` and returns the operation as compacted.
