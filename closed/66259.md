## Summary

Keep open. Current main still initializes fresh QMD collection adds as `--glob` first and only retries `--mask` when QMD explicitly rejects the option. That does not cover the PR's reported QMD 2.1.0 behavior where `--glob` is accepted but the intended pattern is not honored. The PR overlaps open QMD compatibility work (#65480/#65481), but it is a focused, recent bug-fix PR with repro details, in-the-wild confirmation, and useful test coverage, so it should not be closed by automated cleanup.

## Best Possible Solution

Keep this PR open or explicitly supersede it with one canonical QMD compatibility PR after maintainer review. The best code path is to prefer `--mask` for fresh `qmd collection add` calls, preserve fallback to `--glob` only for binaries proven to need it, keep regression coverage for silent `--glob` pattern loss, and align QMD memory docs with the chosen compatibility contract.
