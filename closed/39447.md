## Summary

Keep open. Current main still has the same `install.sh` npm global install path implicated by the report, still defaults `SHARP_IGNORE_GLOBAL_LIBVIPS` to `1`, and still lacks installer-side low-memory/OOM swap recovery. The linked PR #39463 is directly relevant and proposes the missing mitigation, but it remains open/unmerged, so this is not implemented on main or safe to close as superseded.

## Best Possible Solution

Keep #39447 open until the installer either changes the sharp/libvips default, adds low-memory/OOM recovery with validation, or maintainers merge an equivalent fix such as PR #39463. The most direct product path is to land a tested installer change in `scripts/install.sh`, update installer/DigitalOcean docs, and include a focused smoke or shell-level regression check where feasible.
