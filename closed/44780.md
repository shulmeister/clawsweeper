## Summary

Keep open. Current `main` does not implement a configurable iOS Talk Mode speech-recognition locale, so this PR is not obsolete. The draft is also not ready to close out as-is: the current codebase still uses plain `SFSpeechRecognizer()` with no locale setting, but Apple documents that initializer as using the user's default language settings rather than a hardcoded `en-US`, so the PR/linked issue likely identified a real usability gap with an incorrect root-cause explanation. ([developer.apple.com](https://developer.apple.com/documentation/speech/sfspeechrecognizer?language=objc&utm_source=openai))
