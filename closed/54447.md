## Summary

Keep open. Current main still routes PDF fallback extraction through the bundled document-extract plugin, and that active PDF.js call still omits standardFontDataUrl. This PR targets the older core pdf-extract implementation, so it is stale against the current ownership boundary, but the underlying bug is not fixed on main and the PR discussion contains useful root-cause and test evidence.

## Best Possible Solution

Keep the item open or consolidate it only after maintainers pick a canonical tracker. The concrete fix should move into extensions/document-extract/document-extractor.ts: resolve the installed pdfjs-dist package root, pass a trailing-slash filesystem standardFontDataUrl into getDocument, widen the local PdfJsModule type, and add a plugin-level regression test in extensions/document-extract/document-extractor.test.ts.
