## Summary

Keep PR #46992 open. Current main does not implement the PR’s Windows PowerShell UTF-8 initialization in the agent exec runtime, and the docs still describe Windows exec Chinese mojibake as an active workaround case. The PR is not cleanly superseded: #50519 tracks the same remaining bug, while #52154 is an alternate shell-profile feature, not a default encoding fix. The PR should not be merged as-is without addressing the review concerns, but it remains useful work toward an active bug.

## Best Possible Solution

Keep the PR open as useful but not merge-ready. The best maintainer path is a focused Windows exec encoding fix that preserves PowerShell command semantics, covers both child and PTY exec paths, adds platform-independent unit coverage for the wrapper/decoder behavior, and links the work back to the canonical bug report #50519. #52154 can remain separate as an opt-in shell profile feature.
