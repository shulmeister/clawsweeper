## Summary

Keep PR #42531 open. Current main still routes memory-lancedb embeddings through the OpenAI SDK even when `embedding.baseUrl` is configured, and the existing regression coverage only verifies that `dimensions` is forwarded to the SDK path. The PR remains a plausible focused bug fix with a detailed LiteLLM/Ollama reproduction and a bot review supporting the direct `POST /embeddings` approach; it is not implemented on main or clearly superseded by the adjacent open memory-lancedb compatibility reports.

## Best Possible Solution

Keep the PR open and either land a refreshed version or fold its focused regression into a broader memory-lancedb embedding transport fix. The best implementation should explicitly handle `baseUrl` provider differences, preserve configured LanceDB vector dimensions when the provider supports them, keep native OpenAI on the SDK path unless there is a stronger reason to change it, validate empty/malformed embedding responses, and add regression tests for the baseUrl transport path.
