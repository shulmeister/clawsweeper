## Summary

Current main already resolves `OPENCLAW_GATEWAY_TOKEN` on gateway probe/RPC paths, only enables TLS when `gateway.tls.enabled === true`, and `config validate` is schema-only rather than a gateway RPC path. The issue matches behavior that is already fixed/obsolete on shipped code.

## Evidence
