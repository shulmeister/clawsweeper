## Summary

Current main already exposes a supported `claude-cli` raw-argument override path. `agents.defaults.cliBackends.claude-cli.args` and matching `resumeArgs` are schema-accepted, merged over the bundled Claude CLI backend, normalized by the Anthropic plugin, and used directly when building the spawned CLI argv. That lets users pass `--bare` without adding a new core `slimMode` field.

## Best Possible Solution

Close this as implemented by the existing CLI backend configuration surface. Keep the raw-args path documented and use narrower follow-up issues only if maintainers want a convenience `slimMode` alias or per-job/per-command sugar on top of the shipped mechanism.
