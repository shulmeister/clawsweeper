## Summary

Do not close this PR. Current main still has the reported memory-lancedb auto-capture behavior: the hook scans the whole agent_end message payload, filters the whole history, and embeds only the first three capturable texts. The PR targets a real remaining bug, although the submitted cursor approach needs revision based on the review comments around retry, cleanup, compaction, reset, and overlapping fire-and-forget hooks.

## Best Possible Solution

Keep the item open and ask for a revised memory-lancedb fix. The best implementation should skip already-processed user messages without losing retry semantics, clean session state on reset/end, handle compaction and same-length history rewrites, avoid stale writeback from overlapping fire-and-forget hooks, and add focused tests for multi-turn duplicate avoidance, no-capturable-message cursor advancement, embedding/store failure retry, reset, and compaction.
