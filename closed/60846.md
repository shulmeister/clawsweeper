## Summary

Keep #60846 open. Current main still treats skipped cron results as non-errors for failureAlert: alerts are only evaluated in the error branch, skipped results reset failure-alert state, and there is no skip counter, skipAlert/includeSkipped contract, CLI warning, UI field, or docs-backed alternate visibility path. The linked PR #60876 attempted the right class of fix but was closed unmerged and its proposed fields are not present on main.

## Best Possible Solution

Keep the issue open and implement a deliberate persistent-skip monitoring path in cron core. The likely fix belongs around `src/cron/service/timer.ts` plus the cron state/protocol/UI/docs surfaces: track consecutive skipped runs separately from errors, gate delivery with its own cooldown or an explicit `failureAlert.includeSkipped`/`skipAlert` contract, preserve the current error-alert behavior, and add regression tests for skipped runs crossing the alert threshold and for benign ok runs resetting skip state.
