## Summary

Current main already handles the mixed legacy config shape through the supported repair/setup path: top-level single-account credentials such as `botToken`, `appToken`, and `token` are moved into `accounts.default` when named accounts exist, so the default/base account becomes explicit instead of being synthesized at runtime. This also avoids the implicit-default startup risks raised in the PR review discussion.

## Best Possible Solution

Close PR #30310 and keep the shipped doctor/setup promotion behavior as the product path. Mixed legacy configs should be repaired into explicit `channels.<channel>.accounts.default` credentials rather than changing shared startup listing to synthesize an implicit default runtime; any remaining channel-specific edge cases should be tracked as focused doctor/setup repair issues.
