## Summary

Current main already addresses the reported failure mode: auth cooldowns are model-aware for rate limits, same-provider fallback candidates can probe through transient cooldowns instead of being blocked as globally unavailable, timeout failures no longer poison auth-profile state, and external chat channels now receive friendly fallback/error text instead of raw internal dumps.

## Evidence

- **Model-scoped auth cooldowns:** `isProfileInCooldown` now bypasses a cooldown when it was recorded for a different model on the same profile, and `markAuthProfileFailure` persists `cooldownModel` metadata for rate-limit failures.
  - file: [src/agents/auth-profiles/usage-state.ts:25](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/src/agents/auth-profiles/usage-state.ts#L25)
- **Cooldown metadata write path:** Cooldown updates preserve or widen `cooldownModel` based on whether the same model or a different model failed, so one model-specific 429 does not automatically block sibling models forever.
  - file: [src/agents/auth-profiles/usage.ts:586](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/src/agents/auth-profiles/usage.ts#L586)
- **Same-provider fallback probing:** Model fallback now allows one transient cooldown probe per provider run, and embedded auth-profile selection honors that probe instead of failing immediately with `No available auth profile`.
  - file: [src/agents/model-fallback.ts:809](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/src/agents/model-fallback.ts#L809)
- **Embedded runner honors cooldown probes:** When all auto profiles are in cooldown, the embedded runner can still probe one transiently unavailable profile for the current provider/model instead of skipping the whole same-provider fallback candidate.
  - file: [src/agents/pi-embedded-runner/run/auth-controller.ts:500](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/src/agents/pi-embedded-runner/run/auth-controller.ts#L500)
- **Friendly external failure text:** External chat replies now build safe user-facing messages for rate-limit/overload and generic failures, and regression tests assert that mixed fallback exhaustion no longer leaks raw `All models failed ...` detail to the user.
  - file: [src/auto-reply/reply/agent-runner-execution.ts:296](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/src/auto-reply/reply/agent-runner-execution.ts#L296)
- **Shipped release notes:** The changelog records the relevant shipped fixes: model-scoped cooldowns and user-facing countdown messaging in `2026.3.28`, and stopping raw provider/runtime error leaks to chat channels in `2026.4.1`.
  - file: [CHANGELOG.md:1978](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/CHANGELOG.md#L1978)
- **Timeout poisoning follow-up:** Embedded runs now skip auth-profile cooldown writes for timeout failures, which addresses the later comment about non-chat helper runs poisoning the primary model path.
  - file: [src/agents/pi-embedded-runner/run.ts:641](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/src/agents/pi-embedded-runner/run.ts#L641)
