## Summary

Current main already covers the reported Claude CLI/webchat history loss: gateway chat history now backfills Claude CLI session turns from the bound Claude project JSONL, and CLI runs also have explicit transcript persistence on the command path.

## Evidence

- **Gateway chat history now augments Claude CLI sessions:** `chat.history` reads the normal session transcript, then calls `augmentChatHistoryWithCliSessionImports(...)` before sanitizing/bounding history, so Claude CLI session messages are included on reload even when the local transcript is incomplete.
