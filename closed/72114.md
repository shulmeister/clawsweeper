## Summary

Keep #72114 open. Current main has adjacent runtime-deps and CLI package-update hardening, but I do not see a complete source-backed fix for the specific `update.run` cold-cache bundled runtime-deps staging plus restart/shutdown race. The related #72058 tracker and #72084 PR are still only adjacent in the supplied context; #72084 addresses stale postinstall deps and does not clearly supersede this SIGTERM/SIGKILL failure mode.

## Best Possible Solution

Keep #72114 open as a focused runtime-deps/update restart-race report. The best fix is to make gateway `update.run` package updates pre-stage or repair bundled runtime deps before restart, expose active bundled-runtime-deps staging as restart/shutdown pending work, and harden interrupted npm staging cleanup/retry. If maintainers want #72058 to own this, mark #72114 as a duplicate only after that tracker explicitly covers the `update.run`/SIGTERM/SIGKILL staging race.
