## Summary

Current main already keeps loopback token/password CLI gateway calls device-bound, which preserves operator scopes for read RPCs like `status`, `system-presence`, and `config.get`; the same protection exists in `gateway probe`, and regression coverage now asserts those local token-auth paths succeed.

## Evidence

- **CLI gateway calls preserve local device identity:** `resolveDeviceIdentityForGatewayCall()` explicitly keeps shared-auth local calls device-bound so detail RPCs such as `status`, `system-presence`, and `last-heartbeat` retain operator scopes.
  - file: [src/gateway/call.ts:196](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/gateway/call.ts#L196)
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **One-shot CLI RPC client uses that device identity:** The one-shot gateway client injects `resolveDeviceIdentityForGatewayCall()` when no explicit device identity is supplied, so normal CLI RPCs inherit the paired-device binding instead of falling back to unbound shared auth.
  - file: [src/gateway/call.ts:515](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/gateway/call.ts#L515)
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Probe path has the same fix:** `probeGateway()` keeps local authenticated probes device-bound specifically so read/detail RPCs are not scope-limited by shared-auth scope stripping.
  - file: [src/gateway/probe.ts:155](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/gateway/probe.ts#L155)
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Regression coverage matches the reported failure mode:** The integration test suite now asserts that local token-auth `status` calls succeed and that full `probeGateway()` detail RPCs (`health`, `status`, `configSnapshot`) succeed against a loopback gateway.
  - file: [src/gateway/probe.auth.integration.test.ts:9](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/gateway/probe.auth.integration.test.ts#L9)
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Shipped release contains the implementation:** The same `src/gateway/call.ts` device-binding logic is present in release tag `v2026.4.22`, proving the fix is not only on current main but also in a shipped release.
