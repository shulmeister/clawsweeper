## Summary

Current `main` no longer matches the reported startup path: Claude CLI auth is resolved as optional synthetic auth for `claude-cli` only, via local credential reads, and the codebase has no production `claude auth status` shell-out path to abort gateway startup.

## Evidence

- **No `claude auth status` call remains in production code:** A repo search over `src`, `extensions`, and `test` returns no matches for `claude auth status`, so the exact shell-out path described in the report is not present on current `main`.
  - command: `rg -n "claude auth status" src extensions test || true`
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Claude CLI auth is optional synthetic auth:** `resolveClaudeCliSyntheticAuth()` reads cached Claude CLI credentials and returns `undefined` when none are available; there is no exit/fail-closed branch in this runtime path.
  - file: [extensions/anthropic/register.runtime.ts:396](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/extensions/anthropic/register.runtime.ts#L396)
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Synthetic auth is scoped to `claude-cli` only:** The Anthropic plugin wires `resolveSyntheticAuth` only when `provider === "claude-cli"`; generic remote OpenAI-compatible provider configs are not routed through this hook.
  - file: [extensions/anthropic/register.runtime.ts:575](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/extensions/anthropic/register.runtime.ts#L575)
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Credential lookup is file/keychain based, not CLI-status based:** The Claude credential reader checks macOS keychain or the local Claude credential file and returns `null` if unavailable; it does not invoke the Claude CLI binary.
  - file: [src/agents/cli-credentials.ts:347](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/agents/cli-credentials.ts#L347)
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Current docs scope Claude login checks to the actual `claude-cli` backend:** The authentication docs describe `claude auth login` / `claude auth status --text` only as setup for reusing the local `claude-cli` backend, not as a requirement for arbitrary remote providers.
  - file: [docs/gateway/authentication.md:60](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/docs/gateway/authentication.md#L60)
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
