## Summary

Current `main` already addresses this bug by preventing orphaned `tool_result` blocks from reaching provider requests, rather than only redacting the downstream API error text. The replay sanitizer repairs or drops orphaned tool results before model calls, the live session guard avoids creating the malformed state for aborted/errored tool calls, and current tests cover the orphaned-tool-result cases.
