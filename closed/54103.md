## Summary

#54103 should close as superseded by open PR #54183. The PR directly says it closes this issue and proposes the same WhatsApp outbound send retry config, including channel-level and per-account `channels.whatsapp.retry`. Current main has adjacent fixed retry/recovery behavior, but it still lacks the requested configurable WhatsApp `retry`/`sendRetry` config surface, so #54183 is the canonical remaining work item rather than this issue.

## Best Possible Solution

Close #54103 as superseded by #54183 and keep implementation review, tests, and design discussion on that PR. If #54183 is abandoned, reopen or file a narrower issue for adding `channels.whatsapp.retry` and per-account retry config to the WhatsApp channel schema, docs, generated config metadata, and outbound send paths.
