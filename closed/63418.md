## Summary

Keep open. Current main partially implements the configured-default/UI-label pieces, but this PR still covers a real remaining gap: `sessions.list` session rows cannot use the gateway model catalog, and `chat.history` still resolves fallback thinking without the owning agent default. The merged #72324/#72407 work does not fully supersede the catalog-backed and parity parts of this PR.

## Best Possible Solution

Keep this PR open for maintainer handling, but ask for a retargeted/narrowed branch against current main. The useful remaining path is a small catalog-aware default resolver for `sessions.list` rows, plus any intended `chat.history` parity with per-agent thinking defaults; overlapping configured-default/UI pieces already on main should be dropped or reconciled while preserving contributor attribution if tests or implementation ideas are reused.
