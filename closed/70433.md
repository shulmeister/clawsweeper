## Summary

The reported Slack cron follow-up chatter is already addressed on current main: Slack non-DM verbose progress is suppressed, and cron delivery has explicit guards against stale/duplicate follow-up announcements. The issue lacks a reproducible job config, but the symptom/timing matches behavior that current code now blocks.
