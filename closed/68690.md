## Summary

Keep #68690 open. Current main has partial coverage/documentation, but the central report still has real remaining gaps: local CLI web search still reaches provider code with raw SecretRef config, and the voice-call plugin config surface does not declare SecretRef runtime collection for the reported Twilio/TTS override paths. The issue comment is useful evidence because it narrows Brave to a local-CLI/runtime-snapshot inconsistency rather than a generic Brave or migration failure.

## Best Possible Solution

Keep the issue open and narrow it into actionable work: route local `openclaw infer web search` and local TTS conversion through the command SecretRef/runtime snapshot resolver before provider code reads credentials; add voice-call `configContracts.secretInputs` plus schema/runtime tests for `twilio.authToken` and `tts.providers.*.apiKey`, or explicitly document those plugin paths unsupported; keep `hooks.token` documented as unsupported; verify the Matrix import/runtime fix with a regression test.
