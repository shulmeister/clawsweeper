## Summary

Keep open. Current main still has overwrite-only `registerMemoryCapability()` and lacks the PR's regression test, so the narrow code change is not implemented on main. The exact active-memory rationale needs maintainer confirmation because current `active-memory` does not call `registerMemoryCapability`, and related wiki CLI loading work remains separate.

## Best Possible Solution

Keep the PR open for maintainer review. If multiple memory capability registrations are a supported or tolerated path, land the narrow merge plus the regression test. If the intended contract is strictly one active memory capability, close or retarget the PR with an explicit maintainer rationale and reconcile the user-visible wiki bridge import bug through the related CLI loading work.
