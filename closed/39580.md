## Summary

Keep open. Current main still has the Linux node-service systemd unit mismatch this PR fixes: node install/uninstall wraps the gateway service with OPENCLAW_SYSTEMD_UNIT=openclaw-node, but the systemd install activation and uninstall paths still derive openclaw-gateway from OPENCLAW_PROFILE before running systemctl. The PR remains a narrow, valid bug fix, and the discussion's manual workaround to enable/restart openclaw-node.service is consistent with the current source behavior.

## Best Possible Solution

Keep the PR open and either land this minimal helper substitution after maintainer validation, or apply the same fix on main with a focused regression test around systemd install activation and uninstall using OPENCLAW_SYSTEMD_UNIT=openclaw-node. The implementation belongs in src/daemon/systemd.ts because the existing service-name resolver already provides the correct gateway fallback and node override behavior.
