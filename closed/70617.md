## Summary

Current `main` already has a separate Discord native-command authorization layer via `commands.allowFrom`, which overrides normal channel/message allowlists for command execution. The code, config types, tests, and current slash-command docs all show the requested chat-vs-command split is implemented.

## Evidence

- **Config contract defines separate command allowlist:** `CommandsConfig.allowFrom` is documented as a per-provider allowlist restricting who can use slash commands, and explicitly says it overrides the channel `allowFrom` for command authorization.
  - file: [src/config/types.messages.ts:168](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/config/types.messages.ts#L168)
  - command: `nl -ba src/config/types.messages.ts | sed -n '150,190p'`
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **Discord native commands read dedicated command allowlist:** Discord native command auth reads `cfg.commands.allowFrom`, supports provider/global entries plus `guild:<id>` entries, and treats that as a dedicated command auth source.
  - file: [extensions/discord/src/monitor/native-command.ts:157](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/extensions/discord/src/monitor/native-command.ts#L157)
  - command: `nl -ba extensions/discord/src/monitor/native-command.ts | sed -n '150,260p'`
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **Dedicated command allowlist overrides fallback message policy:** When `commands.allowFrom` is configured, Discord native command authorization uses only `commandAllowlistAuthorizer`; otherwise it falls back to guild/channel policy, owner allowlist, and member restrictions. That is the requested split between public chat policy and command policy.
  - file: [extensions/discord/src/monitor/native-command.ts:234](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/extensions/discord/src/monitor/native-command.ts#L234)
  - command: `nl -ba extensions/discord/src/monitor/native-command.ts | sed -n '220,290p'`
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **Tests cover separate Discord slash-command policy:** Tests verify Discord slash commands are authorized when `commands.allowFrom.discord` matches, rejected when it does not, and fall back to channel allowlists only when `commands.allowFrom` is unset.
  - file: [extensions/discord/src/monitor/native-command.commands-allowfrom.test.ts:124](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/extensions/discord/src/monitor/native-command.commands-allowfrom.test.ts#L124)

- **User-facing docs describe separate command authorization:** Current slash-command docs say `commands.allowFrom` is the only authorization source for commands/directives when configured, while `commands.useAccessGroups` only applies when `commands.allowFrom` is not set.
  - file: [docs/tools/slash-commands.md:69](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/docs/tools/slash-commands.md#L69)
