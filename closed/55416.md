## Summary

Keep open. Current main has moved Ollama handling into the bundled Ollama plugin, but it still only promotes native Ollama `message.tool_calls` into executable tool calls. The PR is stale against the current file layout and has real review concerns, but the requested Qwen inline JSON/function-call fallback is not implemented or cleanly superseded on main.

## Best Possible Solution

Keep the PR open as useful but require maintainer/contributor rework: port the fallback into `extensions/ollama/src/stream.ts`, validate extracted inline calls against the offered tool names, fail closed on parse errors, preserve unsafe integers, normalize tool names, and add positive plus negative tests for JSON, function-style, zero-argument, malformed, and prose/example false-positive cases. If maintainers prefer issue tracking over a stale PR, create or designate a canonical Ollama/Qwen inline-tool-call issue before closing this PR as superseded.
