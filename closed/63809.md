## Summary

Current `main` does not route `openclaw update` through the configure wizard or silently rewrite DM allowlist policy. The reported reproduction does not match the present update flow, so this item can be closed as not reproducible on current OpenClaw.

## Evidence

- **Update wizard only selects channel + restart:** `openclaw update wizard` gathers the update channel, optionally offers a git checkout for `dev`, asks whether to restart, and then calls `updateCommand(...)`. This flow does not invoke `runConfigureWizard` or channel setup.
  - file: [src/cli/update-cli/wizard.ts:24](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/cli/update-cli/wizard.ts#L24)
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Public update docs match the narrow flow:** The user-facing docs describe `update wizard` as an interactive flow to pick the update channel and confirm restart behavior; they do not describe re-running channel/configure setup during update.
  - file: [docs/cli/update.md:62](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/docs/cli/update.md#L62)
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Configure wizard is wired to `openclaw configure`:** The configure entrypoints call `runConfigureWizard({ command: "configure" ... })`; there is no corresponding update entrypoint here.
  - file: [src/commands/configure.commands.ts:7](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/commands/configure.commands.ts#L7)
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Update command's direct config write is `update.channel`:** In the update path, the explicit post-update config persistence shown here writes `update.channel` only. I did not find a direct `dmPolicy`/`allowFrom` mutation in `openclaw update`'s own config write path.
  - file: [src/cli/update-cli/update-command.ts:1326](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/cli/update-cli/update-command.ts#L1326)
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **DM policy changes are explicit and opt-in:** Channel DM policy changes are behind an explicit `Configure DM access policies now?` confirmation, and that prompt defaults to `false`, so even the configure flow does not silently reset DM policy without operator input.
  - file: [src/flows/channel-setup.prompts.ts:106](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/flows/channel-setup.prompts.ts#L106)
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Repo search found no update-to-configure linkage:** A repo search for `runConfigureWizard(` and `command: "update"` found configure callers, but no caller that routes `openclaw update` into the configure wizard.
  - command: `rg -n "runConfigureWizard\(|command: \"update\"" src -g '!**/*.snap'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
