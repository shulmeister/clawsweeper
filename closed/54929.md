## Summary

Keep open. Current main still does not implement the PR's central webchat image-turn fix: normal `chat.send` persists image media for transcript storage, but the live dispatch `MsgContext` does not receive `MediaPath`/`MediaPaths`, and reply model selection still does not promote the actual reply run to `agents.defaults.imageModel` before media handling. The PR is a plausible focused bug fix, with prior review discussion already addressing one logging note, but it needs maintainer review/rebase rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open and have a maintainer or contributor port the fix against current main: populate dispatch `MsgContext` with the persisted webchat media payload for normal dashboard image sends, and route image-bearing reply runs to a configured vision-capable `agents.defaults.imageModel` at the right point in the pipeline, ideally before media-understanding decisions so images are not redundantly summarized and then sent to a vision model. Preserve session/channel/heartbeat override precedence and update the existing chat/send and model-routing tests accordingly.
