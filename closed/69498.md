## Summary

Keep open. Current `main` adds a post-install `node_modules/openclaw` symlink for plugins that declare `peerDependencies.openclaw`, but the installer still runs `npm install --omit=dev` without `--omit=peer`, and I did not find regression coverage proving npm-sourced plugins with both normal dependencies and `peerDependencies.openclaw` avoid the duplicated OpenClaw dependency tree described in #69498.

## Evidence

- **Installer still allows npm peer auto-install:** `installPackageDir` invokes `npm install --omit=dev --silent --ignore-scripts`; there is no `--omit=peer` or equivalent opt-out in the current install path, so npm 7+ peer auto-install behavior is still reachable during plugin installs.
  - file: [src/infra/install-package-dir.ts:245](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/infra/install-package-dir.ts#L245)
  - command: `nl -ba src/infra/install-package-dir.ts | sed -n '230,255p'`
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Current mitigation is post-install symlinking:** `installPluginFromPackageDir` runs the dependency-tree scan and then calls `linkOpenClawPeerDependencies`, which deletes any existing `node_modules/openclaw` entry and replaces it with a symlink to the host OpenClaw package root.
  - file: [src/plugins/install.ts:617](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/plugins/install.ts#L617)
  - command: `nl -ba src/plugins/install.ts | sed -n '617,700p'; nl -ba src/plugins/install.ts | sed -n '815,855p'`
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Tests cover peer-only linking, not the reported npm install regression:** The added regression test asserts that a plugin with only `peerDependencies: { openclaw: "*" }` gets a `node_modules/openclaw` symlink and that no npm install runs. I did not find a test that installs a plugin with regular dependencies plus `peerDependencies.openclaw` and then asserts the nested/hoisted duplicate OpenClaw dependency tree is absent.
  - file: [src/plugins/install.test.ts:2380](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/plugins/install.test.ts#L2380)

- **Release notes describe a related partial fix:** The `v2026.4.23` changelog says plugin installs now link the host OpenClaw package into external plugins declaring `openclaw` as a peer dependency, which matches the code above, but that note does not establish that npm peer auto-install bloat is fully eliminated for npm-sourced plugins with their own runtime dependencies.
