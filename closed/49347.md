## Summary

Keep open. The published/root npm install blocker has been addressed by the related maintainer PRs, but the issue is explicitly about full source/workspace `pnpm install`, and current `main` still includes extension workspaces plus the WhatsApp/Baileys dependency path that resolves `@whiskeysockets/libsignal-node` from GitHub codeload. That leaves the reported source-install reliability problem actionable in this repo.

## Best Possible Solution

Keep the issue open and narrow it to source/workspace install reliability. The best maintainer path is to decide whether default contributor installs should avoid optional channel/plugin dependency resolution, replace or mirror the remaining GitHub codeload tarball dependency, or document a supported filtered/retry workflow for source installs. The already-merged root npm install fixes should remain separate from this source checkout issue.
