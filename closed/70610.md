## Summary

Current `main` already implements the reported Control UI switcher fix, and the same behavior was shipped in `v2026.4.23`: session options are grouped by agent, agent labels come from configured display names, and raw session keys are only used as a last-resort collision fallback.

## Evidence

- **Current switcher render path:** The Control UI session selector renders labels from `resolveSessionOptionGroups(...)` and uses the resolved option label for the selected session title instead of showing the raw session key directly.
