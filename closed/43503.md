## Summary

The requested metadata-only TTS flow is useful, but current main intentionally keeps the built-in `tts` tool as an auto-delivery primitive. OpenClaw already exposes lower-level TTS runtime seams for plugin/skill code, and the repo vision pushes channel-specific orchestration like native voice-note delivery out of core and into plugins/ClawHub.

## Evidence
