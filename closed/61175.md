## Summary

Keep #61175 open. Current main still has the confusing dual-error path: npm plugin install failures can fall through to hook-pack install/validation and format the plugin error with `Also not a valid hook pack: ...`. The recent maintainer comment also explicitly made this issue the broader canonical tracker and closed #69181 as a duplicate into it.

## Best Possible Solution

Keep #61175 open as the canonical bug. The best fix is to classify npm/path plugin failures before attempting hook-pack fallback: report plugin security-policy failures and valid-plugin validation failures alone, and only try/report hook-pack validation when the package is plausibly not a plugin or the user explicitly targets a hook-pack-looking package. Add a regression in `src/cli/plugins-cli.install.test.ts` for the normal npm plugin security-block path and keep the existing hook-pack fallback tests green.
