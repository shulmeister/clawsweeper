## Summary

Current `main` already covers this report: provider-scoped generic upstream failures such as OpenRouter `"Provider returned error"` are now classified for failover, and the model fallback loop no longer aborts early on unrecognized non-abort errors when other candidates remain.

## Evidence

- **Generic OpenRouter provider errors now classify into failover:** `classifyFailoverClassificationFromMessage` maps OpenRouter `"Provider returned error"` to `timeout`, which makes it eligible for model fallback instead of surfacing as an unclassified terminal error.
