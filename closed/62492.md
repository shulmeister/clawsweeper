## Summary

Keep #62492 open. Current main and the latest release still have the reported silent skipped-update path: `update.run` reports skipped results as `ok: true`, while the Control UI only shows feedback when `ok === false`. The related PR #66865 explicitly tracks a broader fix for this issue, but it is open and unmerged, so the focused user bug remains actionable.

## Best Possible Solution

Keep #62492 open until main changes `update.run` so skipped/error outcomes are not reported as successful UI updates, the Control UI renders every non-ok/skipped reason with actionable feedback, and regression tests cover a skipped result. If #66865 lands with those semantics, close #62492 as fixed by that PR; otherwise a narrower fix should touch `src/gateway/server-methods/update.ts`, `ui/src/ui/controllers/config.ts`, and the corresponding tests.
