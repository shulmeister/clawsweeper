## Summary

Current `main` already keeps `--json` CLI startup/plugin logs off stdout and writes `openclaw agent --json` payloads through the runtime JSON writer, so this PR’s targeted change is obsolete.

## Evidence

- **Generic JSON-mode stderr routing already exists:** CLI preaction detects JSON output mode and applies startup presentation before command execution; startup presentation routes logs to stderr when stdout must stay clean.
  - file: [src/cli/program/preaction.ts:71](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/cli/program/preaction.ts#L71)

  - file: [src/cli/command-bootstrap.ts:14](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/cli/command-bootstrap.ts#L14)
