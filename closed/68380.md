## Summary

Current `main` already handles the reported WSL2/user-D-Bus case: the systemd adapter falls back to machine-scope user control when the direct `--user` bus is unavailable, classifies missing user-bus errors separately from missing `systemctl`, and surfaces WSL-specific repair hints. The same daemon code is already present in release `v2026.4.23`.
