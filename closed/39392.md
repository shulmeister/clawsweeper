## Summary

Current checked-out main at edf40ab still does not contain the narrow Control UI agent-switch fix. The Agents overview is rendered without a keyed/remount boundary, the agent-switch handler does not reset configForm/configFormDirty, and the Primary model select still depends on select .value plus plain option rendering. A maintainer PR (#72328) is open and explicitly tracks #39392, so the issue should remain open until that PR or an equivalent fix lands on main.

## Best Possible Solution

Keep #39392 open and let the maintainer-owned #72328, or an equivalent focused patch, land on main. The fix should isolate or reset the Agents Overview model controls on selectedAgent.id changes, ideally with a regression test that switches main -> another agent -> main and verifies the displayed Primary model remains the selected agent's resolved config. Broader page-load/catalog dropdown issues can remain in separate follow-up PRs if they are not part of the narrow switch-state bug.
