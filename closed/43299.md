## Summary

Keep open. Current main has lower-level per-run provider/model override plumbing, but the requested user-facing `openclaw agent --model` flag is still not registered, documented in the CLI reference, or passed through the `openclaw agent` CLI path. The PR branch is stale against current main's refactored agent files, but it addresses a real remaining CLI/docs mismatch with downstream support in the discussion.

## Best Possible Solution

Keep the PR open or replace it with a rebased equivalent. The narrow path is to add `--model <id>` to `openclaw agent`, add `model?: string` to `AgentCliOpts`, pass it through Gateway params and `--local` embedded opts into the existing per-run `AgentCommandOpts.model` surface, and update `docs/cli/agent.md` plus CLI/gateway regression tests. Do not duplicate already-shipped lower-level override machinery unless a fresh gap is found.
