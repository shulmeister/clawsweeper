## Summary

Current `main` already routes backend gateway self-calls through the shared credential resolver, marks daemonized gateway runtimes with `OPENCLAW_SERVICE_KIND=gateway`, prefers config-first local token auth in that runtime, and sends the resolved token in the WebSocket `connect` payload. The exact `agent.wait`/`chat.history` self-call path now has direct regression coverage. I did not find a released tag containing the fix yet, so this appears implemented on `main` but not yet shipped in a tagged release.

## Evidence

- **Gateway runtime now prefers config token for local self-calls:** `resolveGatewayCredentialsFromConfig()` switches local token precedence to `config-first` when `OPENCLAW_SERVICE_KIND === "gateway"`, which is the core behavior needed for in-process gateway self-calls to keep using `gateway.auth.token`.
