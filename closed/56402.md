## Summary

Current `main` already codifies the staged plugin-SDK retirement direction this issue asked for: docs now steer plugin authors to focused `openclaw/plugin-sdk/<subpath>` imports, `compat` remains available but explicitly deprecated, the root alias is kept as a deliberately minimal barrel, first-party imports are guarded against regressing, and package exports still preserve compatibility instead of removing either surface abruptly.
