## Summary

Current main already avoids the reported failure mode: the Codex app-server client reads protocol lines from stdout only, treats stderr as plain debug text, and catches malformed JSON instead of throwing. The same implementation is present in the shipped v2026.4.23 release.

## Evidence

- **Protocol parser is attached to stdout only:** The Codex app-server client creates its readline interface from `child.stdout`, so JSON-RPC parsing is no longer fed from stderr.
  - file: [extensions/codex/src/app-server/client.ts:64](https://github.com/openclaw/openclaw/blob/d99d9eda371adc343e7e414830ec4430824c05ee/extensions/codex/src/app-server/client.ts#L64)

  - file: [extensions/codex/src/app-server/client.ts:66](https://github.com/openclaw/openclaw/blob/d99d9eda371adc343e7e414830ec4430824c05ee/extensions/codex/src/app-server/client.ts#L66)
