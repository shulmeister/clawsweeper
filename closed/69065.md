## Summary

Current main and the v2026.4.24 release resolve the actionable documentation gap by making Android’s role explicit: Android is a companion node and does not host the Gateway. The service implementation still intentionally supports managed installs only on macOS, Linux/WSL2, and native Windows, with tests covering unsupported-platform rejection. This closes the reported “no alternative documented” problem without adding Termux daemon support.

## Best Possible Solution

Keep the shipped support boundary: Android is documented as a node/companion platform, while managed Gateway hosting remains macOS, Linux/WSL2, and native Windows. If maintainers want Termux service support later, track it as a new Android/Termux platform feature rather than this regression report.
