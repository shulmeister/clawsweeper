## Summary

Current main implements the requested existing-session profile targeting well enough through per-profile `cdpUrl` endpoint forwarding, `userDataDir`, and custom Chrome MCP command/args. PR #51732 proposed a `cdpPort` field specifically, but the same user problem is now covered by configuring an existing-session profile with a concrete CDP endpoint such as `http://127.0.0.1:9223`; Chrome MCP receives that as `--browserUrl`/`--wsEndpoint` instead of using arbitrary auto-connect.
