## Summary

Current `main` already documents and implements the policy in question: the `coding` tool profile excludes `browser`, and the docs now spell out the profile contents and separate `browser` under `group:ui`. The same clarification is present in the shipped `v2026.4.22` release.

## Evidence

- **Current docs enumerate `coding` without `browser`:** The tool docs explicitly list what `tools.profile = "coding"` includes, and `browser` is not in that list.
  - file: [docs/tools/index.md:130](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/tools/index.md#L130)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Current docs place `browser` in `group:ui`, not `group:web` or `coding`:** The same page defines `group:ui` as `browser, canvas`, making the exclusion from `coding` explicit by table structure.
  - file: [docs/tools/index.md:149](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/tools/index.md#L149)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Gateway config docs match the same policy:** The gateway config reference lists `coding` as `group:fs`, `group:runtime`, `group:web`, `group:sessions`, `group:memory`, `cron`, and media tools, while `browser` appears only under `group:ui`.
  - file: [docs/gateway/config-tools.md:18](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/gateway/config-tools.md#L18)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Implementation intentionally excludes `browser` from `coding`:** `browser` is defined with `profiles: []`, and the resolved `coding` profile is built from tools tagged for `coding` plus `bundle-mcp`, so `browser` is intentionally absent.
  - file: [src/agents/tool-catalog.ts:201](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/tool-catalog.ts#L201)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Latest release already shipped the clarified docs:** `git show 00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5:docs/tools/index.md | sed -n '128,158p'` shows the same profile table and `group:ui` mapping in release `v2026.4.22`.
  - command: `git show 00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5:docs/tools/index.md | sed -n '128,158p'`
