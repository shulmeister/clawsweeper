## Summary

Keep #69209 open. Current main still does not implement the requested top-level `plugins.entries.memory-core.config.dreaming.model` override: the memory-core manifest rejects extra dreaming keys, Dream Diary subagent runs do not receive a model override, and docs still describe Dream Diary as using the default runtime model. The PR remains an active, focused implementation for the issue that ClawSweeper already closed as superseded by #69209, so it is not obsolete or safely closeable.

## Best Possible Solution

Keep #69209 open as the active implementation path. The best maintainer path is to rebase it onto current main, review the resolved fallback behavior and authorization semantics, run the targeted memory-core/host-sdk tests plus the changed gate, and land this PR or an equivalent patch with matching docs for the public `dreaming.model` config.
