## Summary

Keep open. Current `main` still does not fail fast when Discord bot identity lookup fails or returns no id, so the underlying bug described by this PR remains plausible. The PR's exact patch is stale after the April 24 provider refactor, but the behavior it wants is not implemented on `main` yet.
