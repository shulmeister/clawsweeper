## Summary

Current main already falls back to local transcript-based token estimation when provider usage stats are stale or missing, and uses that projected count to trigger preflight compaction. The same implementation is present in the shipped `v2026.4.22` release.

## Evidence

- **Transcript fallback estimates prompt tokens from the session log:** `estimatePromptTokensFromSessionTranscript()` reads the stored transcript and runs `estimateMessagesTokens(messages)`. `runPreflightCompactionIfNeeded()` uses that fallback whenever `totalTokensFresh === false` or there is no persisted total, then compares the projected count against the compaction threshold.
