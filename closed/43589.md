## Summary

Keep #43589 open. Current main has not implemented the skills watcher half of the PR: OpenClaw still depends on chokidar 5, while `ensureSkillsWatcher()` still builds `*/SKILL.md` glob watch targets and passes them to `chokidar.watch()`. The memory half is stale against current main because the old `src/memory/manager-sync-ops.ts` file no longer exists and current memory-host-sdk code has no chokidar watcher, so the PR needs rebase/narrowing rather than closure.

## Best Possible Solution

Keep the PR open, but ask for a rebase and narrow update against current main. The best path is to adapt the skills watcher to watch concrete skill roots with path filtering and depth limits, add regression tests for late-created `<skillsRoot>/<name>/SKILL.md`, and drop or separately re-evaluate the obsolete memory watcher portion unless a current memory-host-sdk watcher regression is reproduced.
