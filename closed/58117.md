## Summary

Mistral Voxtral TTS is useful but is plugin-scoped rather than a core OpenClaw issue. Current main does not register direct Mistral as a speech provider, but the speech-provider plugin API and manifest contract already support implementing it externally, and the bundled OpenRouter speech provider already exposes a Voxtral TTS model as a partial workaround.

## Best Possible Solution

Publish a Mistral/Voxtral speech provider as a ClawHub plugin using the existing registerSpeechProvider and contracts.speechProviders surfaces. If maintainers later want direct Mistral TTS bundled with the existing Mistral provider, that should be an explicit product decision rather than leaving this broad core issue open.
