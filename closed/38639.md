## Summary

Current main, also present in v2026.4.24, already covers the reported Telegram `/models` custom-provider case. The catalog path materializes `models.providers` into `models.json` before registry reads, and the shared `/models` provider-data path also synthesizes allowlisted/default custom provider entries with metadata from `models.providers`.

## Best Possible Solution

Close #38639 as implemented. Keep the current shared catalog/materialized `models.json` and synthetic allowlist behavior. If PR #62991 remains open, it should be reviewed as potentially superseded by the current shipped path or narrowed to any distinct read-only CLI edge case, not used to keep the original Telegram picker issue open.
