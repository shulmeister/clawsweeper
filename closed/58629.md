## Summary

Current main already implements the reported behavior: the Control UI `/compact` action still calls `sessions.compact`, but that RPC now treats an omitted `maxLines` as manual LLM compaction and calls `compactEmbeddedPiSession` instead of doing transcript line trimming.

## Evidence

- **Control UI compact action payload:** `ui/src/ui/chat/slash-command-executor.ts:151-168` sends `sessions.compact` with only `{ key: sessionKey }`, so the UI path does not request file trimming via `maxLines`.
