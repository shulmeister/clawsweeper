## Summary

Current main still does not implement #49939 as requested: Anthropic Opus/Sonnet 4.6 only resolve to 1M when `params.context1m === true`, and the tests explicitly keep the no-flag 4.6 path at 200k. The same remaining bug is now tracked by the newer open #66766, with #66862 carrying the focused fix discussion/PR, so #49939 can close as superseded rather than implemented. Anthropic’s March 13, 2026 announcement supports the underlying 4.6 1M request and says no beta header is required. citeturn0search0

## Best Possible Solution

Close #49939 as superseded and keep the remaining product/implementation decision on #66766 and #66862. The best fix should update the Anthropic context resolver, provider request/beta-header behavior, tests, and docs together so session budgeting, `/status`, `models list`, and actual request capability agree for Opus/Sonnet 4.6.
