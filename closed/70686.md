## Summary

Keep open. Current `main` still counts compaction checkpoint transcripts as usage-bearing session files, so the reported double-counting path is still plausible and not implemented on `main` or the latest release tag provided.

## Evidence

- **Usage-counted filter still accepts checkpoint files:** `isPrimarySessionTranscriptFileName()` returns true for any `.jsonl` file that is not an archive artifact, and `isUsageCountedSessionTranscriptFileName()` accepts all primary transcripts. A file named like `session.checkpoint.<id>.jsonl` still matches that path because checkpoints are not excluded here.
