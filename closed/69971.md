## Summary

Keep open. Current main still lacks the PR's two substantive fixes: `plugins install` still calls the direct plugin installers without a profile/config-aware `extensionsDir`, and plugin discovery still skips symlinked plugin directories. The PR is also not clearly merge-ready because the existing review comment identifies a real install-root/discovery-root alignment risk, but that is a reason to revise the PR, not close it as obsolete.

## Best Possible Solution

Keep the PR open for maintainer review and revision. The best fix is to centralize or pass a single plugin extensions root that is resolved after profile/env setup and is shared by install, discovery, list, update, and uninstall paths, then add regression tests for profile state dirs, `OPENCLAW_CONFIG_PATH`/legacy-root alignment, direct npm/path installs, and symlinked plus broken-symlink discovery.
