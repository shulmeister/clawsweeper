## Summary

Current main and the v2026.4.24 release contain the secure Google Chat add-on principal validation path, explicit rejection logging, and startup warnings for missing or malformed appPrincipal. The issue comments correctly identified the config root cause and warned against bypassing JWT validation; current code now makes that failure diagnosable without disabling auth.

## Best Possible Solution

Close as implemented: keep the shipped secure JWT validation that requires appPrincipal for Workspace add-on tokens, rely on the current rejection logs/startup warnings for diagnosis, and track any remaining setup-wizard or docs polish as a separate focused follow-up if maintainers want it.
