## Summary

Current main has not implemented the core status/status --all fix from PR #47981. The gateway-status helper and docs already classify a connected `missing scope: operator.read` probe as reachable/degraded, but the active shared status scan still sets `gatewayReachable` from `gatewayProbe?.ok === true`, so scope-limited detail-RPC failures remain classified as unreachable in the status surface. The PR is still relevant, but it needs a refreshed implementation against the current refactored status path and should split connect reachability from RPC readiness to address the existing review comments.

## Best Possible Solution

Keep the item open, but refresh the implementation: compute connect reachability in `resolveGatewayProbeSnapshot` with `isProbeReachable`, expose a separate RPC-ready signal based on `gatewayProbe.ok`, use connect reachability only for overview/connectivity messaging, keep RPC-dependent status reads gated on RPC readiness or surface explicit scope-limited unavailable states, and add regression tests for `openclaw status`, `status --all`, and JSON output with a connected `missing scope: operator.read` probe.
