## Summary

Current main already handles inbound image/video content through the media-understanding pipeline before the normal reply path. It auto-resolves configured and auth-backed multimodal providers, supports image/video capability-specific models, and preserves the default text reply path for ordinary text. The behavior is present in v2026.4.24.
