## Summary

Keep PR #47654 open. Current main still rejects unknown config keys on the normal validation/load path, while doctor-only repair can strip them explicitly. The PR’s original Telegram `inlineButtons` example is now obsolete because that key exists in schema/docs, but the broader crash-loop/tolerant-read behavior is not implemented and the PR discussion surfaced real integration risks that need maintainer review rather than cleanup closure.

## Best Possible Solution

Keep #47654 open for maintainer judgment or explicit consolidation with the canonical unknown-config-key read-mode tracker. The best fix is a deliberate tolerant startup/read path for unknown-key-only config errors that logs exact warnings and never mutates disk, while keeping strict validation for structural errors and all write/control-plane mutation paths, with regression tests for root, nested, union-backed, plugin-warning, and legacy-migration interactions.
