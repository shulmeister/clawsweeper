## Summary

Keep #72168 open. The report is fresh, concrete, independently confirmed in the issue discussion, and current main still has the reported failure shape: browser Playwright actions dynamically load `pw-ai`, `pw-session` still imports bare `playwright-core`, and `openclaw doctor --fix` still documents installing staged plugin runtime deps rather than making the root `pw-ai` chunk resolver-visible. The linked PR #72238 is the right active fix path, but the provided context says it is open/unmerged and current main does not contain a matching fix commit.

## Best Possible Solution

Keep #72168 open and use PR #72238, or an equivalent maintainer patch, as the fix path. The fix should make `playwright-core` resolvable for packaged browser Playwright actions after `openclaw doctor --fix`, preferably through a browser-local runtime shim or another resolver-visible mechanism, and should add regression coverage for packaged global/Homebrew-style installs where plugin deps are staged outside the OpenClaw package tree.
