## Summary

Current main already treats media-understanding vendors as plugin-owned capability providers, and project scope explicitly says optional capability should usually ship as plugins rather than core. Soniox is absent from main and this request's proposed core-file edits do not match the current architecture.

## Evidence

- **Vision scope:** Project vision says optional capability should usually ship as plugins, the preferred path is npm/local extension loading, and the bar for adding optional plugins to core is intentionally high.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/VISION.md#L54)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Media-understanding registry is plugin-backed:** Current media-understanding registry loads providers from plugin capability providers instead of a hardcoded core registry entry.
  - file: [src/media-understanding/provider-registry.ts:30](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/media-understanding/provider-registry.ts#L30)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Manifest metadata drives provider discovery:** Provider metadata is read from plugin manifests via mediaUnderstandingProviders contracts, so a new vendor fits the plugin manifest path.
  - file: [src/media-understanding/manifest-metadata.ts:6](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/media-understanding/manifest-metadata.ts#L6)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Public SDK supports external media-understanding providers:** The public provider-plugin docs show api.registerMediaUnderstandingProvider(...) as the intended seam for image/audio/video provider plugins.
  - file: [docs/plugins/sdk-provider-plugins.md:577](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/docs/plugins/sdk-provider-plugins.md#L577)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)

  - file: [extensions/deepgram/index.ts:5](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/extensions/deepgram/index.ts#L5)
