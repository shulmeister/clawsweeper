## Summary

Current `main` already supersedes this PR with a broader cron-payload resolution path: text-only announce delivery can prefer `finalAssistantVisibleText` for channels that opt in, while non-opt-in direct channels keep all successful text payloads instead of truncating to the last chunk.

## Evidence

- **Cron run path now resolves announce output through channel-aware payload selection:** `runCronIsolatedAgentTurn` feeds `payloads`, `finalAssistantVisibleText`, and the per-channel `preferFinalAssistantVisibleText` policy into `resolveCronPayloadOutcome`, so announce delivery no longer relies on the old single-last-chunk selection path.
