## Summary

Current main does not support `AGENTS.gpt5.md` / `SOUL.gpt5.md` as first-class bootstrap filenames, but OpenClaw already exposes hook/plugin surfaces that can do model-aware bootstrap mutation. Given the repo vision to keep optional capability in plugins/ClawHub when extension points exist, this fits better as a ClawHub/plugin hook-pack than a new core bootstrap rule.

## Evidence

- **Canonical bootstrap contract only:** Workspace bootstrap filenames are hardcoded to canonical names (`AGENTS.md`, `SOUL.md`, etc.); there is no built-in alias filename slot for `AGENTS.gpt5.md` or `SOUL.gpt5.md`.

## Risks / Open Questions

- No existing published plugin or hook pack was linked in this issue, so the closure relies on the sufficiency of current extension points rather than an already-posted implementation.
- If maintainers later want a first-party standardized model-aware bootstrap-selection contract, that should be designed as an extension API rather than added as another hardcoded filename rule.
