## Summary

#69550 reports the same remaining Control UI chat-bubble action overlap tracked by #69605 and open PR #69634. Current main is not fixed yet: the CSS padding class exists, but the renderer still does not add `has-copy` when copy/expand actions render. The linked PR #69655 correctly targeted #69550 but was closed unmerged, so the best cleanup is to close #69550 as superseded by the clearer canonical issue/PR rather than as implemented.

## Best Possible Solution

Close #69550 as superseded by the clearer canonical tracker #69605 and open PR #69634. The product fix should be the narrow UI change already proposed there: apply `has-copy` to assistant chat bubbles whenever copy/expand actions render, and add a regression test that locks that class contract for action-bearing assistant bubbles while leaving user bubbles unchanged.
