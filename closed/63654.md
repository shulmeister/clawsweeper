## Summary

Keep open. Current main still filters, strips, and suppresses qwen3.6-plus on Qwen Coding Plan endpoints, and the docs/tests still describe that as expected behavior. There are open implementation candidates, including PR #63987, which explicitly links this issue for closure, so the issue should remain open until maintainers land or reject the linked fix work.

## Best Possible Solution

Keep this issue open as the canonical Qwen Coding Plan qwen3.6-plus tracking item. Maintainers should review and either land or close the linked PRs, choosing between the narrower opt-in behavior in PR #63987 and the broader provider-truth behavior in PR #66367. If a fix lands, update the qwen plugin hooks, provider catalog tests, and public Qwen docs together; if maintainers keep the current guard, document that policy explicitly and ensure the runtime error is clear.
