## Summary

Current main already addresses the PR's central gateway RSS/startup-loading problem by computing a bounded Gateway startup plugin-id set and passing that scope into the plugin loader, backed by cold plugin registry metadata. The PR's scanDir precedence reversal is no longer the right fix and would conflict with current explicit override semantics that main preserves with tests.

## Best Possible Solution

Close #70730 and keep the current mainline startup-scoping/cold-registry implementation. If #70717 still reproduces after the next release, investigate against the current startup plugin-id selection and runtime load paths rather than merging a scanDir precedence reversal.
