## Summary

Keep PR #68859 open. Current main still does not expose `OPENCLAW_SESSION_KEY` or `OPENCLAW_AGENT_ID` to exec child processes, while the exec runtime already receives session context and passes a runtime env into gateway, PTY, and sandbox execution paths. The PR is a plausible focused fix for part of open issue #66705, but it still needs maintainer review, regression coverage, and cleanup of the unrelated UI type-only change before it is landable.

## Best Possible Solution

Keep the PR open as useful but not yet landable. The best path is to rebase it on current main, drop the unrelated UI churn, inject `OPENCLAW_SESSION_KEY` in `runExecProcess` with stale inherited values cleared when no session is bound, update `docs/help/environment.md` and `docs/tools/exec.md`, and add focused tests covering at least gateway and PTY paths, with sandbox coverage if the existing test seams make that cheap. Separately decide whether `OPENCLAW_AGENT_ID` from #66705 belongs in the same change or a follow-up.
