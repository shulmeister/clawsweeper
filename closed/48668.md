## Summary

Keep open. Current main supports Gemini embeddings for the default memory-core memory search path, but it still does not implement the PR's specific memory-lancedb capability: the memory-lancedb config, manifest, and runtime remain OpenAI-only. The PR is not merge-ready as-is based on the provided review comments, but it is still a plausible scoped feature rather than obsolete maintainer cleanup.

## Best Possible Solution

Keep the PR open for maintainer/product judgment. The best code direction is to either wire memory-lancedb through the existing memory embedding provider adapter contract so it can reuse the Google/Gemini implementation, or explicitly keep memory-lancedb OpenAI-compatible-only and document memory-core as the supported Gemini memory path. If pursued, the PR should add schema/manifest/docs/tests for provider selection, exact Gemini model handling, explicit OpenAI-compatible proxy behavior, and dimension/outputDimensionality compatibility.
