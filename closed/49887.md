## Summary

Current main already routes Slack HTTP events through the shared Bolt HTTP receiver and handles channel traffic via the generic `message` event plus `app_mention`; the latest shipped release `v2026.4.22` contains that implementation and tests covering channel/group message routing.

## Evidence

- **HTTP mode uses Bolt HTTPReceiver and shared request listener:** Slack HTTP mode constructs a Bolt `HTTPReceiver` for the configured webhook path and forwards inbound requests through `requestListener`, then registers that handler for the account webhook path.
  - file: [extensions/slack/src/monitor/provider-support.ts:129](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/extensions/slack/src/monitor/provider-support.ts#L129)
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **HTTP webhook path is registered for each Slack account:** Slack plugin HTTP routes are registered from config-only webhook paths and dispatch into the shared Slack HTTP handler registry.
  - file: [extensions/slack/src/http/plugin-routes.ts:7](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/extensions/slack/src/http/plugin-routes.ts#L7)
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Channel events are handled as generic `message` payloads, not `message.channels` event names:** Current Slack monitor code explicitly documents that Slack `message.channels`/`message.groups` subscriptions arrive as `type: "message"` payloads, registers `ctx.app.event("message")`, and separately routes `app_mention` for channels.
  - file: [extensions/slack/src/monitor/events/messages.ts:51](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/extensions/slack/src/monitor/events/messages.ts#L51)
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Tests cover channel/group message routing and channel app mentions:** Current tests assert that channel/group `message` payloads reach the message handler and that channel `app_mention` payloads are routed to the handler.
  - file: [extensions/slack/src/monitor/events/messages.test.ts:207](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/extensions/slack/src/monitor/events/messages.test.ts#L207)
