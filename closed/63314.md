## Summary

Current `main` already contains the WhatsApp `--media` fix: CLI runtime now routes media sends through the channel `sendMedia` path, outbound delivery preserves media access/local-root context, and the WhatsApp extension forwards that context into media loading. The changelog says this shipped in `2026.4.10`, which is newer than the reported broken `2026.4.8`/`2026.4.9` builds.
