## Summary

Keep this PR open. Current main still lacks the Gateway shutdown sweep for active bundled LSP runtimes, while the linked bug #72357 remains open with concrete reproduction steps. The PR is a plausible implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Review and either land or adapt this PR as the implementation candidate for #72357. The desired end state is bundled LSP stdio servers spawned/disposed with process-tree-safe cleanup, an exported active-runtime sweep invoked during Gateway shutdown with bounded concurrent MCP/LSP cleanup, and focused regression coverage in the bundled LSP runtime and Gateway close tests.
