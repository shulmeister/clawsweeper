## Summary

Keep this PR open. Current main only implements Codex-mode Computer Use setup through the Codex app-server marketplace flow; it does not provide the PR's harness-neutral `computer` plugin or a settled generic computer-use SDK. The active maintainer comment about security/permissions and SDK design makes this a product/security review item, not a conservative cleanup close.

## Best Possible Solution

Keep the PR open for maintainer product and security review. The best path is to settle the generic computer-use extension SDK, permission, and security model first, then decide whether the cua-driver implementation should be revised against that contract, published externally through ClawHub/npm, or promoted as a bundled plugin with explicit maintainer ownership.
