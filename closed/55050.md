## Summary

Current main already covers the issue’s context-restoration goal without adding a literal SESSION_START.md file: Active Memory can run a silent bounded memory sub-agent before eligible conversational replies, bare /new and /reset first turns receive runtime-loaded recent daily memory through startupContext, and typed session_start hooks exist for lifecycle tracking.

## Best Possible Solution

Keep the shipped implementation: use Active Memory for silent pre-reply memory restoration, startupContext for /new and /reset first-turn daily-memory preload, and typed plugin hooks for lifecycle-specific integrations. Do not add another core SESSION_START.md hook unless a narrower future report identifies a missing API that these shipped surfaces cannot cover.
