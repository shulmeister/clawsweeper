## Summary

Current main no longer performs the duplicate CLI plugin loads described in #47429. The reported `message send` path no longer preloads plugins in Commander preAction and instead loads them once in the message action, while `plugins list` now renders from the persisted/metadata registry snapshot rather than doing a second runtime registration pass.

## Evidence
