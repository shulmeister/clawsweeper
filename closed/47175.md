## Summary

Issue #47175 asks for a hard per-turn tool-call burst cap. The same remaining core work is already tracked by open issue #9912, which covers maxTurns/maxToolCalls limits for agent iterations. Current main still only exposes repetitive tool-loop detection and subagent timeout/concurrency settings, not the requested force-response maxToolCallsPerTurn cap, so #47175 should close as a duplicate rather than as implemented.
