## Summary

Keep PR #68651 open. Current `main` still has no `models.pricing.enabled` or equivalent documented/config-validated switch to skip the gateway model-pricing bootstrap. The linked bug #53639 remains a real core/gateway config issue: current startup still schedules the pricing refresh for non-test gateways and the refresh still fetches remote pricing catalogs whenever configured model refs exist. The PR is not obsolete, but it needs rebasing/adaptation to current `main` before merge.

## Best Possible Solution

Keep #68651 open and ask for a focused rebase onto current `main`: add `models.pricing.enabled` to `ModelsConfig`, zod schema, generated config metadata, help/labels, and user-facing docs; gate `startGatewayModelPricingRefresh` before queuing the microtask so `false` skips all bootstrap/scheduled pricing work; update tests for default/true/false behavior under the current deferred bootstrap; and fix the changelog entry with required attribution. This remains the best core fix for linked issue #53639 unless maintainers decide to solve it with a different config name or broader pricing-refresh policy.
