## Summary

Current main no longer lets Telegram-style block streaming stalls hold the embedded run until the old 600s watchdog path. Streamed block replies are decoupled from the run, channel delivery is bounded by a 15s timeout, and coverage explicitly verifies that a timed-out block reply still returns the final payload instead of hanging the run.

## Evidence

- **Streaming replies are enqueued instead of blocking the embedded run:** When block streaming is enabled, the reply delivery handler enqueues the payload into the block-reply pipeline instead of awaiting channel delivery inline.
