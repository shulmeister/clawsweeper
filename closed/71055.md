## Summary

Current `main` already replaced the Windows exec path that the issue describes: plugin/exec calls now flow through a shared runtime that executes argv directly on Windows and only falls back to `cmd.exe /d /s /c` for `.cmd`/`.bat` shims, with regression tests covering both wrapper and direct-invocation behavior. The reporter’s version-correction comment points to an older 2026.4.5-based build, which fits this being fixed on `main` but not yet in the latest tagged release.

## Evidence
