## Summary

Current main and the shipped v2026.4.24 release already normalize chat content arrays before prompt-history formatting, preventing object blocks from being inserted into agent prompts as `[object Object]`. The reporter's examples were from 2026.4.22/2026.4.23; the v2026.4.24/current implementation has source and regression-test coverage for the no-history and history/current-message cases.

## Best Possible Solution

Keep the shipped v2026.4.24/main prompt-normalization implementation. If `[object Object]` can still be reproduced on 2026.4.24 or newer with a fresh session, the best follow-up is a new focused bug report with the exact prompt, raw provider/tool payload, logs, model/provider config, and affected channel/plugin details.
