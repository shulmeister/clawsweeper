## Summary

Keep #58481 open. The report is recent, has concrete environment details plus a confirming comment and related cross-reference, and current main still does not appear to set TCP keepalive on the underlying WhatsApp/Baileys WebSocket socket. The existing `keepAlive` naming in the WhatsApp controller is an application-level heartbeat/reconnect watchdog, not OS TCP socket keepalive.
