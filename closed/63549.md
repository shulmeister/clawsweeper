## Summary

Current `main` already contains ACP stale-session recovery logic that matches this report: fresh-session preparation hides stale persisted ACP records, the manager retries session init without stale resume IDs on `ACP_SESSION_INIT_FAILED`, and regression tests cover the first-turn recovery path. The same ACP files are unchanged between `v2026.4.23` and current `main`, so the fix is shipped in that release.

## Evidence

- **ACPX runtime masks stale persisted session records after reset:** `createResetAwareSessionStore` returns `undefined` for session loads marked fresh, `prepareFreshSession()` marks a session fresh, and `close(... discardPersistentState)` re-marks it fresh. This directly addresses stale/uninitialized ACP session metadata being reused on the first prompt.
  - file: [extensions/acpx/src/runtime.ts:34](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/acpx/src/runtime.ts#L34)
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **ACP manager retries without stale persisted resume IDs:** For persistent ACP sessions with metadata but no stable session ID, the manager calls `prepareFreshSession()` before `ensureSession()`. If resume init fails with `ACP_SESSION_INIT_FAILED`, it drops stale persisted session identifiers and retries `ensureSession()` without them.
  - file: [src/acp/control-plane/manager.core.ts:1423](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/acp/control-plane/manager.core.ts#L1423)
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Regression tests cover stale-init recovery:** Tests assert that stale session-init failures are recoverable during discard resets and that `prepareFreshSession()` runs before `ensureSession()` for persistent sessions with incomplete ACP identity.
  - file: [src/acp/control-plane/manager.test.ts:1444](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/acp/control-plane/manager.test.ts#L1444)
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Regression tests cover hidden stale loads after fresh-session prep:** Tests verify stale persistent loads stay hidden until a fresh ACP record is saved, and that discard-reset close marks the session fresh again.
  - file: [extensions/acpx/src/runtime.test.ts:82](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/acpx/src/runtime.test.ts#L82)
