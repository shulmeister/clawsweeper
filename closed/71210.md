## Summary

Current `main` already fixes the reported `memory status`/`local` provider failure by resolving memory embedding providers from declared plugin capability contracts instead of relying on prior eager registry seeding. The relevant fix landed in commit `28eb56dd21b61631ba916ea3dd0fc2afa002f31d` after `v2026.4.23`.

## Evidence

- **Memory CLI/provider path now resolves by config-backed provider lookup:** `createEmbeddingProvider()` calls `getAdapter()`, and `getAdapter()` resolves via `getMemoryEmbeddingProvider(id, config)`, so CLI memory flows can resolve `local` from config/capability metadata instead of depending on a separately primed registry.
  - file: [extensions/memory-core/src/memory/embeddings.ts:46](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/extensions/memory-core/src/memory/embeddings.ts#L46)
  - command: `nl -ba extensions/memory-core/src/memory/embeddings.ts | sed -n '46,70p'`
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Runtime provider lookup falls back to declared capability providers when registry is cold:** The runtime merges registered memory embedding providers with capability-declared providers and resolves specific provider ids through `resolvePluginCapabilityProvider(...)` when nothing is registered yet.
  - file: [src/plugins/memory-embedding-provider-runtime.ts:17](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/plugins/memory-embedding-provider-runtime.ts#L17)
  - command: `nl -ba src/plugins/memory-embedding-provider-runtime.ts | sed -n '1,60p'`
  - sha: [bc21f500d4c1](https://github.com/openclaw/openclaw/commit/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2)
- **Targeted capability lookup loads the bundled owner plugin for the requested provider:** `resolvePluginCapabilityProvider(...)` discovers bundled plugins that declare the requested capability contract and loads that registry path with `activate: false`, which is the current main mechanism that makes cold-start provider lookup work without explicit pre-registration.
  - file: [src/plugins/capability-provider-runtime.ts:201](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/src/plugins/capability-provider-runtime.ts#L201)

- **`memory-core` declares ownership of the `local` embedding provider:** The bundled memory plugin manifest explicitly advertises `contracts.memoryEmbeddingProviders = ["local"]`, which is what the fallback resolver uses to find the provider on main.
  - file: [extensions/memory-core/openclaw.plugin.json:4](https://github.com/openclaw/openclaw/blob/bc21f500d4c14d8ecbacb992c39c7ff0954b04f2/extensions/memory-core/openclaw.plugin.json#L4)
