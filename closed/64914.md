## Summary

Current `main` already resolves configured `models.providers[].models[].contextWindow/contextTokens` for the session store, compaction guard, and status summary, and the same implementation is present in the shipped `v2026.4.23` release.

## Evidence

- **Session context lookup now reads configured provider model caps directly:** `resolveContextTokensForModel()` scans `cfg.models.providers[provider].models[]` for `contextTokens`/`contextWindow` before cache/default fallback, covering explicit provider+model lookups instead of dropping to `DEFAULT_CONTEXT_TOKENS`.

  - sha: [ae5c6573670b](https://github.com/openclaw/openclaw/commit/ae5c6573670b3742b156899bd2b9c36b8677f191)

  - file: [src/agents/command/session-store.ts:73](https://github.com/openclaw/openclaw/blob/ae5c6573670b3742b156899bd2b9c36b8677f191/src/agents/command/session-store.ts#L73)
