## Summary

Current main’s documented scope says optional capabilities like this should ship as external plugins, not new bundled core plugins. The existing plugin SDK already exposes the hook, HTTP-route, and CLI seams needed to implement this tracing viewer outside the repo, and main already ships `diagnostics-otel` for built-in observability export.

## Evidence

## Risks / Open Questions

- This close decision is about scope, not usefulness; the tracing viewer could still be valuable as an external ClawHub/npm plugin.
- If maintainers want a smaller core seam for external tracing plugins later, that should come as a narrow SDK proposal rather than bundling this full UI/storage feature set.
