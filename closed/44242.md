## Summary

Current main does not support this PR’s premise anymore: the only production use of createPinnedDispatcher is now wrapped by fetchWithSsrFGuard cleanup/release logic, and the checked-in tests intentionally preserve global/caller autoSelectFamily behavior instead of forcing it off.
