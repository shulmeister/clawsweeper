## Summary

Keep #39673 open. Current main has partial Linux user-systemd recovery for `Failed to connect to bus`/no-medium-style errors, but `gateway install` still has real unrecovered failure exits and still reports raw install errors without the systemd remediation hints available in status/lifecycle flows. Related #63561 is adjacent systemd-user detection work, not a clean canonical duplicate of this original install-fails regression.

## Best Possible Solution

Keep #39673 open and fix the Linux gateway install path directly. Share the existing systemd-unavailable hint rendering with `gateway install` failures, and add a focused regression test that exercises the real `service.install` / `writeSystemdUnit` / activation path when `assertSystemdAvailable`, `daemon-reload`, `enable`, or `restart` returns `Failed to connect to bus` / no-medium-style output. Keep #63561 cross-linked as adjacent systemd-user detection work unless a maintainer chooses one canonical tracker.
