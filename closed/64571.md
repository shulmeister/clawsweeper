## Summary

Current `main` already clears transient auto-failover session overrides before each turn, resets the in-memory provider/model back to the configured primary, and clears sticky runtime model fields when reverting to the default selection. Regression tests cover both the auto-heal path and preservation of user-selected overrides, and the changelog says this shipped in `v2026.4.20`.

## Evidence

- **Current model-selection logic clears auto failover overrides before each turn:** `createModelSelectionState()` detects a direct session override with `modelOverrideSource === "auto"`, applies the default selection, persists the cleared session entry, and resets the active provider/model back to the configured primary for the current turn.
