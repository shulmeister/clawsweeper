## Summary

Current `main` already routes compaction for `anthropic-vertex` through the Vertex-aware stream path instead of the generic `anthropic-messages` `/v1/messages` path, and the same simple-completion guard is present in release `v2026.4.23`.

## Evidence

- **Compaction session uses the resolved provider-aware stream before calling compact():** `compactEmbeddedPiSession()` builds the compaction session, then calls `prepareCompactionSessionAgent(...)` before `activeSession.compact(...)`, so compaction runs through OpenClaw's stream resolution layer rather than a raw default provider call.
  - file: [src/agents/pi-embedded-runner/compact.ts:906](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/pi-embedded-runner/compact.ts#L906)

- **Stream resolution special-cases anthropic-vertex:** When the model provider is `anthropic-vertex`, `resolveEmbeddedAgentStreamFn()` returns `createAnthropicVertexStreamFnForModel(params.model)` directly.
  - file: [src/agents/pi-embedded-runner/stream-resolution.ts:116](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/pi-embedded-runner/stream-resolution.ts#L116)
