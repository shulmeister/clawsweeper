## Summary

Keep open. Current `main` still ships the bundled ACPX integration, still forwards ACP runtime model overrides through the generic `model` config path, and has no clear repo-side fix for the reported per-turn Claude model reset / prompt-cache invalidation. The report is recent, specific, and still plausible even if the root cause may live in upstream `acpx` or `claude-agent-acp` rather than OpenClaw glue.

## Evidence

- **Bundled ACPX remains an OpenClaw-shipped dependency:** The bundled ACPX plugin is still present on current `main` and pins `acpx` `0.5.3`, so the integration path from the report is still part of the shipped product surface.
