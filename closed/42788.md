## Summary

Keep #42788 open, but narrow it to the remaining TUI label/header work. Current main appears to solve the active-model portion, and the Control UI label/dropdown side was already handled by #45130, but the TUI header still renders the formatted session key rather than a configured label or display name.

## Best Possible Solution

Keep the issue open and narrow it to TUI-specific session label display. The best fix is to make the TUI header and session picker prefer explicit `label`/human display metadata, preserve label separately in TUI session state if needed, and add focused TUI tests for a labeled `agent:<id>:main` session with a model override. The already-shipped active-model behavior can be referenced as complete.
