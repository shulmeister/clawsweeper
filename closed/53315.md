## Summary

Keep open. Current main still treats unknown channel config keys as fatal validation issues, so the PR is not implemented or obsolete. The PR targets a real linked startup-resilience problem, but its current broad downgrade still needs maintainer review/tests because it weakens typo detection and does not address the related heartbeat-target fatal path.

## Best Possible Solution

Keep the PR open, but request a narrower implementation before merge: preserve fatal validation for genuinely unknown channel keys, warn-and-skip only for credible stale/missing plugin-backed channel configs, add regression tests for `channels.matrix` or a fixture plugin channel missing from the registry, add a typo case such as `channels.telegrm`, and decide whether heartbeat targets referencing the skipped channel should warn instead of crash-looping.
