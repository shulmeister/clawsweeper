## Summary

Current main no longer matches the PR's premise: read/write/edit require the canonical `path` parameter at runtime, and tests explicitly reject `file`, `filePath`, and `file_path` legacy aliases. Adding `record.file` to the display-only resolver would only improve summaries for unsupported/invalid calls, so this PR is obsolete as a bug fix. The same one-line PR shape was already reviewed and closed in #60047 for the same reason.

## Best Possible Solution

Close this PR as obsolete. Keep the current canonical `path`-only read/write/edit contract unless maintainers explicitly decide to reintroduce `file` alias support end-to-end, in which case the work should update schemas, runtime validation, execution paths, display logic, and regression tests together.
