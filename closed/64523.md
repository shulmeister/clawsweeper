## Summary

Current main already treats `chat.inject` and `delivery-mirror` assistant rows as transcript-only OpenClaw messages and suppresses them from the embedded reply/broadcast path, matching the reporter’s local mitigation and covering the reported echo-loop shape.

## Evidence

- **`chat.inject` writes explicit internal assistant markers:** The shipped tree stores injected assistant transcript rows as `provider: "openclaw"` and `model: "gateway-injected"`, giving downstream code a stable way to recognize transcript-only assistant events instead of model output.
