## Summary

Current main already fixes the stock embedded-mode Anthropic tool-name rejection: bundled MCP tools are materialized with provider-safe names, embedded session creation passes the exact registered custom-tool allowlist, stock OpenClaw tools use underscore names such as `web_fetch`, and the behavior is documented and covered by tests. The related #54005 PR was closed unmerged, but current main has the equivalent product fix through the bundle MCP/provider-safe naming path.

## Best Possible Solution

Close #53990 as implemented. Keep the current provider-safe bundle MCP naming plus exact embedded session tool allowlist. If a future report shows non-bundle plugin or client-provided tool names with invalid Anthropic characters, track that as a narrower provider/plugin API compatibility issue rather than this fresh-install stock-tool bug.
