## Summary

Keep #47933 open. Current main has partial WebChat attachment support, but it is still image-scoped for user uploads and the gateway parser intentionally drops non-image payloads. Assistant-generated MEDIA files can already render as downloadable links when they are inside allowed preview roots, so the remaining gap is the broader document/archive upload and end-to-end generic file workflow requested by the issue. This matches the maintainer comment on April 25, 2026 saying the image-only path is covered separately by #51045/current main, but generic upload/download should stay tracked here.

## Best Possible Solution

Keep this issue open as the broad Control UI/WebChat file-transfer tracker. The likely product path is a core WebChat/gateway attachment contract that supports selected document/archive MIME types with explicit size limits, workspace or media-store placement, safe preview/download routing, transcript metadata, provider/model hydration rules, and tests that preserve the existing image path while adding document behavior. Narrow work such as PDF upload can proceed in a focused PR, but it should either close a scoped follow-up or be linked here as partial progress.
