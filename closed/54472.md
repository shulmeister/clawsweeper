## Summary

Current main and the shipped v2026.4.24 release have moved the macOS companion chat surface to the native SwiftUI WebChat stack and bound the Gateway `chat.history` payload before the mac UI renders it. The reported v2026.3.23 freeze on large history/system-message text is covered by the shipped native chat path, lazy list rendering, history text truncation, oversized-row placeholdering, docs, and regression/smoke coverage.

## Best Possible Solution

Keep the shipped native SwiftUI WebChat plus bounded `chat.history` implementation. If a current v2026.4.24-or-newer build still freezes on a specific transcript, track that as a new focused regression with the current app/gateway versions and a redacted failing history shape.
