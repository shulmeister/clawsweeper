## Summary

Current `main` already builds Control UI model picker values from each catalog entry's configured `provider` and forwards the selected provider-qualified value unchanged to `sessions.patch`, so the reported `ollama` vs `ollama-remote` prefix bug is already fixed on `main` and present in the shipped `v2026.4.22` code.

## Evidence

- **Picker option values use the catalog provider:** `buildQualifiedChatModelValue` prefixes with the passed provider, and `buildChatModelOptionFromLookup` calls it with `entry.provider`, producing `<provider>/<model-id>` from the model catalog rather than hardcoding an API type.
  - file: [ui/src/ui/chat-model-ref.ts:5](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/ui/src/ui/chat-model-ref.ts#L5)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Control UI sends the selected value through unchanged:** The chat header model selector reads the `<select>` value and sends `model: nextModel || null` directly in `sessions.patch`; it does not rewrite the provider prefix on submit.
  - file: [ui/src/ui/chat/session-controls.ts:234](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/ui/src/ui/chat/session-controls.ts#L234)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Tests cover provider-qualified picker values:** UI tests assert provider-qualified option values are built from catalog entries, including provider-native slash-containing ids like `openrouter/google/gemma-4-26b-a4b-it`.
  - file: [ui/src/ui/chat-model-ref.test.ts:25](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/ui/src/ui/chat-model-ref.test.ts#L25)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)

  - file: [ui/src/ui/chat/session-controls.test.ts:139](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/ui/src/ui/chat/session-controls.test.ts#L139)
