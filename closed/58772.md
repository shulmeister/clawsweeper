## Summary

Current `main` already supports no-prompt host exec (`security=full`, `ask=off`) and now ships explicit exec-policy tooling to sync config plus approvals, so the requested capability is implemented.

## Evidence

- **Runtime defaults already allow promptless host exec:** Gateway/node exec resolves unset host policy to `security="full"` and `ask="off"`, which removes per-command approval prompts unless the operator tightens policy.
  - file: [src/agents/bash-tools.exec.ts:1480](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/agents/bash-tools.exec.ts#L1480)

  - file: [src/infra/exec-approvals.ts:44](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/infra/exec-approvals.ts#L44)
