## Summary

The PR is proposing a new bundled Azure Blob Storage plugin in core, but current main explicitly steers optional commercial integrations to external community plugins distributed through ClawHub/npm rather than new bundled OpenClaw plugins.

## Evidence

- **Vision: optional capability should stay out of core:** `VISION.md:54-59` says OpenClaw has an extensive plugin API, core should stay lean, optional capability should usually ship as plugins, and plugin authors should host and maintain them in their own repository.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/VISION.md#L54)

- **Vision: commercial integrations are out of scope for core:** `VISION.md:99-107` lists “Commercial service integrations that do not clearly fit the model-provider category” under “What We Will Not Merge (For Now).” Azure Blob Storage is a storage-service integration, not a model provider.
  - file: [VISION.md:99](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/VISION.md#L99)

  - file: [docs/plugins/community.md:9](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/docs/plugins/community.md#L9)
