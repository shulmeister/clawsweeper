## Summary

Current `main` and the shipped `v2026.4.23` tag no longer route non-overflow prompt-template failures into the context-overflow recovery path. The runner only enters `[context-overflow-diag]` for explicit overflow signatures, and the classifier/test suite now keep ambiguous 400/422 cases out of that lane unless they actually match overflow markers.

## Evidence

- **Release tag only enters overflow recovery for explicit overflow matches:** In `v2026.4.23`, the runner checks `isLikelyContextOverflowError(errorText)` before creating `contextOverflowError`; the adjacent branch explicitly returns `null` for non-overflow prompt errors, so they do not enter `[context-overflow-diag]`.
