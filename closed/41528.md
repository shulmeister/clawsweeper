## Summary

Keep open. Current main partially covers `update.run` business-level errors, but it still misses real `status: "skipped"` payloads because the gateway returns them inside a successful RPC response with payload `ok: true`. The PR remains useful, though the skipped-result test should adopt Greptile's review suggestion and mock `ok: true` for the skipped case.

## Best Possible Solution

Keep the PR open and update it to handle both payload `ok: false` and `result.status === "error" | "skipped"`, with regression tests that model real gateway payloads: error as `ok: false`, skipped as `ok: true`. After that, land the narrow Control UI controller/test change rather than closing this as already implemented.
