## Summary

Current `main` already routes Control UI model-picker selections from catalog-qualified `provider/model` values instead of reconstructing them from the active session provider, and this shipped in `v2026.4.22`. The relevant files are unchanged between release `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5` and current `main` `7d1891e6e63dc35dd9dcb688ba308049e68fe695`.

## Evidence

- **Picker submits the selected qualified value directly:** The Control UI chat model dropdown uses `entry.value` for each `<option>` and passes the selected value straight to `sessions.patch`, so it does not rebuild the model string from the current session provider on change.
  - file: [ui/src/ui/chat/session-controls.ts:83](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/ui/src/ui/chat/session-controls.ts#L83)
  - command: `nl -ba ui/src/ui/chat/session-controls.ts | sed -n '70,150p'`
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Option values are built from each catalog entry's provider:** `buildChatModelOptionFromLookup()` sets option values with `buildQualifiedChatModelValue(entry.id, entry.provider)`, so a GPT entry from `openai-codex` becomes `openai-codex/<model>` rather than inheriting a stale provider like `anthropic`.
  - file: [ui/src/ui/chat-model-ref.ts:266](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/ui/src/ui/chat-model-ref.ts#L266)

  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Current override resolution corrects stale providers from catalog matches:** `resolvePreferredServerChatModelValue()` normalizes bare model ids against the catalog before falling back to the session provider, which prevents stale-provider qualification when the catalog identifies the model uniquely.
  - file: [ui/src/ui/chat-model-ref.ts:106](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/ui/src/ui/chat-model-ref.ts#L106)
  - command: `nl -ba ui/src/ui/chat-model-ref.ts | sed -n '1,220p'`
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
