## Summary

Current `main` no longer has the reported `Reports`/`Interfaces` Control UI tabs, and the replacement Usage flow now carries an explicit local-vs-UTC timezone mode from the browser into gateway date-range parsing. That makes the reported GMT+8 vs GMT+0 mismatch obsolete on current `main`.

## Evidence

- **Old tabs are gone on current main:** The Control UI navigation now exposes `overview`, `channels`, `instances`, `sessions`, `usage`, and `cron`; there is no `reports` or `interfaces` tab in the current tab model or routes.
  - file: [ui/src/ui/navigation.ts:5](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/ui/src/ui/navigation.ts#L5)

- **Usage defaults to local timezone:** The current Control UI state initializes the usage/reporting surface with `usageTimeZone = "local"`, so current main prefers browser-local time instead of UTC by default.
  - file: [ui/src/ui/app.ts:388](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/ui/src/ui/app.ts#L388)

- **UI exposes explicit Local/UTC timezone selector:** The replacement usage surface includes a timezone selector with `local` and `utc` options, rather than silently using one timezone in one tab and another elsewhere.
  - file: [ui/src/ui/views/usage.ts:596](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/ui/src/ui/views/usage.ts#L596)

- **Browser sends explicit UTC offset to gateway in local mode:** When the usage surface is in local mode, the UI sends `mode: "specific"` plus a formatted `utcOffset`; UTC mode sends `mode: "utc"`. This is the concrete fix for timezone interpretation drift.
  - file: [ui/src/ui/controllers/usage.ts:127](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/ui/src/ui/controllers/usage.ts#L127)

  - file: [src/gateway/server-methods/usage.ts:147](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/gateway/server-methods/usage.ts#L147)
