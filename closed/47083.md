## Summary

Current `main` does not fully implement this PR. The web sessions presenter still formats `totalTokens` without checking `totalTokensFresh`, and the TUI patch/apply path still copies `entry.totalTokens` blindly even though gateway/session events can carry `totalTokensFresh: false`. There is partial mitigation in gateway session-list row generation, but not enough to close this PR as already implemented.

## Evidence

## Risks / Open Questions

- The web sessions-list symptom may be partially masked today because gateway session rows already suppress stale totals in many cases, so the remaining user-visible bug is narrower than the PR body suggests.
- The TUI path still looks genuinely unfixed because patch/event updates can carry raw `totalTokens` plus `totalTokensFresh: false`, and current `applySessionInfo` does not honor the freshness flag.
- No clean reproduction run was attempted because this review was constrained to read-only inspection and the environment does not have `pnpm` available.
