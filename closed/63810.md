## Summary

Current `main` already supersedes this PR for the status-inspection path: `buildChannelsTable` now uses read-only channel plugin resolution instead of runtime registry activation, and there is a regression test proving configured external channel metadata can be read without importing full runtime. The PR’s original manifest-snapshot approach was also already flagged in review as regressing channel enumeration/security because manifest-only snapshots leave `registry.channels` empty.
