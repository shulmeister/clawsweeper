## Summary

Current main partially improves edit reliability, but it does not fully implement the issue's requested four-part behavior. OpenClaw adds recovery around the upstream edit tool and the pinned upstream package has limited fuzzy matching, but the remaining requested behavior, especially broader whitespace normalization and keyword-based read positioning, is still missing or not documented as shipped behavior.

## Best Possible Solution

Keep the issue open, but narrow it to the remaining concrete product work: decide whether edit should gain a conservative whitespace fallback beyond the pinned upstream fuzzy matcher, preserve uniqueness and non-overlap safety, add regression tests for space/tab and blank-line mismatch cases, and separately decide whether read should add a keyword locator or document offset as the line-number locator. Current recovery/error-hint behavior should be kept, but it is not a complete close for this request.
