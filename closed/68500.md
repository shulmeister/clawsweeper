## Summary

Current main keeps background exec runs supervised through child exit, and a read-only reproduction using the same detached Node spawn semantics did not leave zombie processes behind.

## Evidence

- **Background exec stays attached to a managed wait path:** `runExecProcess()` always spawns through the process supervisor, stores the returned `managedRun`, and keeps a `managedRun.wait()` promise alive even when the tool resolves early as `status: "running"` for background sessions.
  - file: [src/agents/bash-tools.exec-runtime.ts:719](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/agents/bash-tools.exec-runtime.ts#L719)
  - command: `nl -ba src/agents/bash-tools.exec-runtime.ts | sed -n '706,800p'`
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **Supervisor explicitly awaits child completion and finalizes the run:** The supervisor's `waitPromise` awaits `adapter.wait()`, then clears timers, disposes the adapter, removes the active run, and finalizes the registry on exit.
  - file: [src/process/supervisor/supervisor.ts:201](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/process/supervisor/supervisor.ts#L201)
  - command: `nl -ba src/process/supervisor/supervisor.ts | sed -n '201,238p'`
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **Child adapter resolves wait from child exit/close events:** The child-process adapter registers `child.once("exit", ...)` and `child.once("close", ...)`, then resolves its `wait()` promise from those events, which is the reaping path OpenClaw uses for non-PTY exec.
  - file: [src/process/supervisor/adapters/child.ts:244](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/process/supervisor/adapters/child.ts#L244)
  - command: `nl -ba src/process/supervisor/adapters/child.ts | sed -n '244,284p'`
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **Read-only spawn-semantics reproduction:** Ran a no-write Node reproduction with the same relevant spawn shape OpenClaw uses on Unix (`detached: true`, piped stdio, `/bin/sh -lc 'sleep 0.3'`). After the child exited, `ps` showed `shell_ps=<gone>` and `zombies=<none>`.
  - command: `node - <<'NODE' ... spawn('/bin/sh', ['-lc', 'sleep 0.3'], { detached: true, stdio: ['pipe','pipe','pipe'] }) ... ps -eo pid=,ppid=,stat=,command= ... NODE`
