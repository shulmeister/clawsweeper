## Summary

Keep #72401 open. Current main still exposes ambiguous message-tool target guidance while Discord runtime treats bare numeric IDs as channel targets, so the reported DM-prefix failure is still actionable. The exact fix is tracked by open, unmerged PR #72423, but it has not landed on main.

## Best Possible Solution

Keep #72401 open until #72423 or an equivalent main-branch change lands. The best fix is to update the model-facing `message` tool target description, and likely the shared target help text, to explicitly include Discord/Slack/Mattermost `channel:<id>` and `user:<id>` forms; add a focused schema-description or contract test if practical so the hint does not regress.
