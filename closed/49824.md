## Summary

Current `main` already routes engine-derived session totals/context into webchat and no longer computes the reported context percentage from uncached input only. The `R…` footer value in webchat is cache-read tokens, not remaining tokens.

## Evidence

- **Webchat session notice uses gateway session totals/context and suppresses stale totals:** `getContextNoticeViewModel` reads `session.totalTokens` and `session.contextTokens`, and returns nothing when `totalTokensFresh === false`.
  - file: [ui/src/ui/chat/context-notice.ts:53](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/ui/src/ui/chat/context-notice.ts#L53)
  - sha: [320d52a23e41](https://github.com/openclaw/openclaw/commit/320d52a23e41a326287826219a20f1f29f334554)

  - file: [ui/src/ui/views/chat.ts:929](https://github.com/openclaw/openclaw/blob/320d52a23e41a326287826219a20f1f29f334554/ui/src/ui/views/chat.ts#L929)
