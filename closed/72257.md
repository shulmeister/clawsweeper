## Summary

Current main fixes the reported claude-cli abort cascade by treating `claude-cli` as a CLI runtime/backend, not an embedded agent harness, and by adding direct fallback delivery for failed subagent completion announces.

## Best Possible Solution

Keep the current main behavior: `claude-cli` remains a CLI backend/runtime, not an embedded harness id, fallback providers do not inherit `claude-cli` as a harness pin, and subagent completion delivery uses direct fallback when requester wake fails. Any remaining request to change the default no-output watchdog should be tracked as a narrower timeout-policy issue.
