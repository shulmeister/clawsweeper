## Summary

Keep open. Current main still drops the configured image-understanding timeout on the MiniMax VLM path: the media runner resolves and passes a timeout budget, but the MiniMax helper has no timeout parameter and still hardcodes `AbortSignal.timeout(60_000)`. The PR targets the real remaining gap and is not implemented on main, though it should be cleaned up by removing or splitting the unrelated iMessage test before maintainer merge.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to retain the MiniMax timeout parameter threading, pass `params.timeoutMs` through both direct and unknown-model fallback MiniMax VLM routes, add the custom-timeout regression test, and split the unrelated iMessage test into a separate PR or drop it from this one.
