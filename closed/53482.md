## Summary

Keep this issue open. Current main still does not add a gateway startup diagnostic for legacy CLAWDBOT_*/MOLTBOT_* environment variables, and the provided GitHub context shows open PR #53667 is an unmerged implementation candidate with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review or replace PR #53667. The best fix is a focused gateway startup/runtime-auth diagnostic that detects CLAWDBOT_*/MOLTBOT_* keys early, emits clear OPENCLAW_* replacement guidance and doctor/migration pointers, updates misleading auth errors, and adds regression tests while preserving the intentional removal of legacy fallback behavior.
