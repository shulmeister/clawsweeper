## Summary

Keep open. Current `main` at `1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19` still implements session freshness entirely from `updatedAt`, while system events still bypass expiry checks and then advance `updatedAt`. The bug described by PR #65386 is therefore not implemented on `main`, and the PR discussion also contains unresolved follow-up review concerns that need maintainer judgment rather than cleanup closure.
