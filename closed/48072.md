## Summary

Keep open. Current main still lacks the Dockerfile pre-created `/home/node/.openclaw` directory before `USER node`, while Docker Compose/docs mount that path as the OpenClaw state/config directory and config writes still need to create/write `openclaw.json` there. The related open PR #63959 tracks the same bug, but it is not a clean superseder because its current Dockerfile patch uses mode `0755`; #48072 already incorporated the review feedback to use `0700`.

## Best Possible Solution

Keep this PR open or consolidate it with #63959/#61279 into one canonical Docker named-volume fix. The best implementation is a single Dockerfile layer before `USER node` that creates `/home/node/.openclaw` owned by `node:node` with private `0700` permissions, plus the required user-facing changelog attribution and a maintainer Docker named-volume/bind-mount verification pass.
