## Summary

Keep open. Current main still does not set WAL or synchronous=NORMAL for the bundled memory-core SQLite index. The old PR path has moved into `extensions/memory-core`, but the remaining user problem is real: the central memory DB opener only applies `busy_timeout`, and the existing test still asserts that shipped behavior. Related PR #39471 is closed/unmerged and does not supersede this open PR.

## Best Possible Solution

Keep the PR open and ask for a rebase onto current `main`. The likely implementation should set `PRAGMA journal_mode = WAL` and `PRAGMA synchronous = NORMAL` in `extensions/memory-core/src/memory/manager-db.ts` so every memory-core DB open path gets the behavior, while updating `manager.readonly-recovery.test.ts` to assert the new WAL/synchronous contract and consciously preserving or justifying removal of the shipped `busy_timeout` setting.
