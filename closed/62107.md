## Summary

Current `main` already implements a restart-sentinel continuation flow for agent-triggered gateway restarts: the supported `gateway` tool path schedules a structured restart, persists post-boot continuation state, and resumes the same session/thread after the gateway comes back. The same behavior is present in release `v2026.4.23`.

## Evidence

- **Gateway tool restart path persists continuation instead of relying on raw shell termination:** The agent-facing `gateway` tool explicitly tells agents to use `continuationMessage` when a restart still owes the user a reply, builds a restart sentinel payload, and returns the structured result from `scheduleGatewaySigusr1Restart(...)` via `jsonResult(...)` rather than surfacing a raw `SIGTERM` shell failure in the supported path.
  - file: [src/agents/tools/gateway-tool.ts:357](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/agents/tools/gateway-tool.ts#L357)

## Risks / Open Questions
