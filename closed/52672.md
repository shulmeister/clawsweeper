## Summary

Keep open. Current `main` still removes channel config in the configure wizard by deleting the raw `channels.<id>` entry directly, so this is not implemented on `main`. The linked WhatsApp restart behavior also remains plausible on current `main`: persisted WhatsApp auth on disk is treated as configured state, and the WhatsApp plugin does not expose an `onAccountRemoved` cleanup hook. This PR may still be useful, but its narrowed scope does not strongly prove the linked WhatsApp report is fully fixed, so conservative cleanup should leave it for maintainer review rather than auto-close it.
