## Summary

Keep open. Current main still implements the MS Teams federated managed-identity path as a raw ManagedIdentityCredential token pass-through, while PR #72298 proposed the missing AzureADTokenExchange/client_assertion flow. The PR is closed unmerged, and the related broad ClawSweeper reports only establish that federated/managed-identity config exists, not that this narrower token-exchange bug is fixed.

## Best Possible Solution

Keep this work tracked. The maintainer path should fix the bundled MS Teams plugin by either using the Teams SDK's federated managed-identity credential path when compatible with the pinned @microsoft/teams.apps version, or by implementing the equivalent AzureADTokenExchange client_assertion exchange with robust non-JSON/error handling and access-token caching. Update sdk tests so managed identity no longer returns the raw ManagedIdentityCredential token for the Bot Framework scope, and adjust docs if the runtime behavior changes.
