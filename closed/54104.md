## Summary

Current main implements the actionable cleanup path for this report. `google-vertex` is not a standalone plugin id; the bundled `google` plugin owns the Google provider surface and exposes `google-vertex` as provider hook/config behavior. Stale `plugins.entries.google-vertex` records are now detected and removable with `openclaw doctor --fix`, with source, tests, and docs backing that path.

## Best Possible Solution

Keep the current boundary: `google-vertex` should not be treated as a standalone plugin entry, and stale `plugins.entries.google-vertex` config should be cleaned with `openclaw doctor --fix`. Provider docs should continue steering Vertex users toward Google Cloud ADC/provider config rather than `models auth login --provider google-vertex`.
