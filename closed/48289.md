## Summary

Keep this PR open. Current main still sends the resolved Windows pnpm runner path directly to spawn/spawnSync while enabling shell:true for .cmd/.bat/.com launchers, so the linked C:\Program-space-path failure is not implemented away on main. The PR discussion is useful: the author addressed the initial unconditional-quoting concern, and the final bot review considered the conditional quoting approach safe, but main does not contain that change.

## Best Possible Solution

Keep the PR or its linked bug open. The best path is to fix scripts/ui.js for Windows pnpm.cmd paths with spaces, add a focused regression test for the generated spawn command, and consider reusing the existing Windows cmd.exe wrapper helpers used by scripts/pnpm-runner.mjs and src/process/exec.ts.
