## Summary

Keep open. Current main does not implement Gemma 4 `reasoning_content` stripping, and the PR is a plausible targeted bug fix but still needs maintainer/requested-change work. The existing replay policy only supports `dropThinkingBlocks`, which preserves the latest assistant thinking block, and provider-owned replay hooks for OpenAI/Ollama bypass the unowned fallback path where this PR proposes to add the Gemma policy.

## Best Possible Solution

Keep the PR open and request changes. The best fix is to add a provider-owned replay/sanitization contract for Gemma 4 OpenAI-compatible routes, likely in the shared OpenAI-compatible replay helper used by OpenAI/Ollama/custom providers, then wire it through history replay so all historical `thinking` blocks from Gemma 4 are stripped while preserving existing DeepSeek/Z.AI/MiniMax/Claude replay behavior. Add regression tests for `resolveTranscriptPolicy` and `sanitizeSessionHistory` with Gemma 4 model IDs on OpenAI-compatible/Ollama-style routes plus non-Gemma controls.
