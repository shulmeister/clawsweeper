## Summary

Keep PR #40443 open. Current main still lacks the explicit reload versus passive refresh split that this PR proposes: `loadConfig` has no discard option, `applyConfigSnapshot` preserves dirty form state, and the Config, Agents, and Channels reload buttons still call the same passive `loadConfig` path. The linked bug #40352 remains open, and the competing PR #40441 was closed unmerged, so this work is not implemented or superseded on main.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it onto current main, preserve the explicit reload versus passive refresh split in `ui/src/ui/controllers/config.ts`, wire Config/Agents/Channels explicit reload actions to discard pending edits, and retain focused regression coverage for form dirty state, raw dirty state, config hash-conflict recovery, and the Agents model dropdown rerender.
