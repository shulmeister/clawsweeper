## Summary

Current main no longer crashes completion-cache generation when the packaged install lacks `qa/scenarios/index.md`. The QA scenario loader now treats the repo-only pack as optional, the completion CLI skips failing subcommand registration instead of aborting, and a regression test covers the exact error string from this report. The changelog records the user-facing fix as shipped in `v2026.4.11`.

## Evidence

- **QA scenario pack is now optional in packaged installs:** `readQaScenarioPack()` returns an empty pack when `qa/scenarios/index.md` is missing, with an inline comment that npm distributions may omit the repo-only scenario directory, preventing the previous throw-on-missing path.
