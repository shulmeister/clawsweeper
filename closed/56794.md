## Summary

Keep open. Current main has improved the startup validation path so manifest channel metadata can override generated bundled channel schemas, but the doctor warning reported in the latest issue comment is still represented in source: the blocker scanner reports a disabled plugin that claims a configured channel without checking whether an enabled external/preferred plugin also owns that same channel.

## Best Possible Solution

Keep #56794 open and fix the doctor blocker logic. The scanner should resolve the effective owner set for a configured channel and suppress the disabled bundled-owner warning when an enabled external/preferred plugin owns the same channel, especially when `channelConfigs.<id>.preferOver` names the bundled plugin. Add regression coverage for `channels.feishu` with `plugins.entries.feishu.enabled=false` and an enabled external plugin claiming `feishu`, while preserving warnings for a truly disabled sole owner and `plugins.enabled=false`.
