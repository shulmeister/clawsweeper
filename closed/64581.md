## Summary

Current main already separates the SSH sandbox's remote exec workspace from the host-side workspace path in the system prompt and backend runtime, and the same behavior is present in release v2026.4.23.

## Evidence

- **Prompt now prefers sandbox exec path:** The system prompt uses `sandboxInfo.containerWorkspaceDir` as the displayed working directory when sandboxed, and adds explicit guidance that bash/exec commands must use sandbox container paths rather than the host workspace path.
  - file: [src/agents/system-prompt.ts:639](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/agents/system-prompt.ts#L639)

  - file: [src/agents/system-prompt.ts:804](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/agents/system-prompt.ts#L804)
