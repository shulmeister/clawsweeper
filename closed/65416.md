## Summary

PR #65416 is obsolete against current main. The requested video_generate buffer save cap is implemented on main by resolving generated video media limits to the configured media limit or the 16MB video cap, then passing that value to saveMediaBuffer. Main also adds regression coverage and URL fallback for oversized generated videos.

## Best Possible Solution

Close PR #65416 as implemented on current main and keep the broader maintainer implementation from 3507efa4ec1e8b7c22889ab5e2c6c9283adeb768, which preserves the requested video cap behavior, honors configured media limits, and adds hosted-URL fallback for oversized generated videos.
