## Summary

Keep this PR open. Current main still lacks the LINE config-schema guard, while the LINE runtime treats `dmPolicy: "open"` as unconditional DM acceptance. The PR head adds the missing `requireOpenAllowFrom` refinement and focused schema tests, and the review discussion is favorable apart from minor follow-up coverage/message issues that appear addressed in the current PR head.

## Best Possible Solution

Keep this PR open for maintainer review and likely landing after confirming account-level inheritance behavior and migration wording. The product-correct path is a LINE-owned schema hardening change that reuses `requireOpenAllowFrom`, keeps the focused top-level and account-level regression tests, and documents or releases the explicit migration to `allowFrom: ["*"]` for truly open DMs or `dmPolicy: "allowlist"` for restricted senders.
