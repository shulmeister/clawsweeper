## Summary

Keep open. Current main still does not include `~/.docker/bin` or `~/.orbstack/bin` in the macOS LaunchAgent service PATH, and the PR is a focused additive fix for the linked sandbox failure rather than obsolete work.

## Best Possible Solution

Keep the PR open and either merge a rebased version of its narrow additive fix with service-env regression coverage, or replace it with an equivalent core daemon change that makes Docker Desktop and OrbStack discoverable from the macOS LaunchAgent environment. This belongs in core daemon service PATH handling, not ClawHub, because `openclaw gateway install` owns the generated LaunchAgent environment.
