## Summary

Close as `clawhub`. Xiaomi MiMo native web search is a valid optional provider integration, but current OpenClaw scope and SDK docs route optional web-search/provider capabilities to plugins distributed through ClawHub or npm. Current `main` has Xiaomi as a bundled model/TTS plugin but no Xiaomi web-search registration, and existing Kimi/Gemini implementations prove model-native search tools can be implemented through the current `registerWebSearchProvider` surface without a new core API.

## Best Possible Solution

Close the core issue and encourage a Xiaomi MiMo web-search provider plugin published through ClawHub or npm. The plugin should use `api.registerWebSearchProvider(...)`, reuse `XIAOMI_API_KEY` where available, and follow the existing Kimi/Gemini model-native search patterns. If implementing that plugin exposes a concrete missing SDK capability, open a narrower plugin-API issue.
